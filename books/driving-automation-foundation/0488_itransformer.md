# iTransformer — 変数をトークンにする転置型Transformer

## ひとことで言うと

時系列予測でこれまで「時間方向」をトークンに切っていたTransformerを、発想を逆転させて「変数(チャネル)方向」をトークンにすることで、Attention が変数間の相関を、Feed-Forward が時間パターンを学ぶように役割分担させた手法です。モデル構造はそのまま、入力の並べ方(どっちを系列の単位とみなすか)を転置(invert)しただけで、当時の長期時系列予測の最高精度を更新しました。

## 直感的な理解

電力需要の予測を考えます。手元には「気温」「湿度」「曜日」「過去の需要」といった複数の変数(これを多変量時系列と呼びます。1つの時刻に複数の数値が並ぶ時系列のことです)があり、それぞれが過去96時間ぶん記録されているとします。

普通のTransformer(2021年頃の時系列向けモデル)は、各「時刻」を1つのトークン(Transformer が処理する最小単位。文章でいう単語に相当)にしていました。つまり時刻 t におけるすべての変数をまとめて1つのベクトルにし、それを96個並べて、時刻どうしの関係を Attention(後で説明します)で見ていたのです。

ところがこのやり方には不自然な点があります。「気温」と「需要」のように単位もスケールも意味もまったく違う変数を、同じ時刻だからという理由だけで1つのトークンに押し込めてしまうのです。さらに Attention は本来「並び順に意味がある(言葉なら語順)」要素どうしの関係を測る仕組みなのに、時刻トークンに使うと「位置の近い時刻ほど似ている」という当たり前の局所性すらわざわざ位置埋め込みで教え直す必要がありました。

iTransformer はこれを転置します。「変数1つ(過去96時刻ぶんの系列まるごと)」を1つのトークンにするのです。すると Attention は「気温という系列」と「需要という系列」の関係を測ることになり、これはまさに変数間の相関を見る作業になって意味が通ります。各変数の時間的なパターン(トレンドや周期)は、トークン内部を処理する Feed-Forward 層(全結合のニューラルネットワーク)に任せます。

```
従来 (時刻=トークン):
  時刻軸 → → → →
  [t1の全変数][t2の全変数]...[t96の全変数]   ← 96個のトークン
  Attention は時刻どうしを見る

iTransformer (変数=トークン):
  変数軸
  [変数Aの96時刻ぶん]   ← トークン1
  [変数Bの96時刻ぶん]   ← トークン2
  [変数Cの96時刻ぶん]   ← トークン3
  Attention は変数どうしを見る
```

## 基礎: 前提となる概念

### Attention(自己注意)とは

Attention は「各トークンが、他のどのトークンをどれくらい参照すべきか」を学習で決める仕組みです。各トークンのベクトルから Query(問い合わせ)・Key(鍵)・Value(値)という3つのベクトルを作り、Query と Key の内積で「関連度のスコア」を計算します。スコアを softmax(合計1になるよう正規化する関数)に通して重みにし、その重みで Value を加重平均します。式で書くと

Attention(Q, K, V) = softmax(QK^T / √d) V

ここで Q, K, V はそれぞれトークン数 × 次元の行列、d は次元数で、√d で割るのは内積が大きくなりすぎて softmax が極端にならないようにする調整(スケーリング)です。重要なのは Attention は集合演算であって、トークンの並び順そのものは見ない点です。だから従来の時刻トークン方式では「時刻の順序」を別途位置埋め込みで足していました。

### 多変量時系列予測のタスク設定

入力は過去 T 時刻ぶんの N 変数、つまり形 (T, N) のテーブル、出力は未来 S 時刻ぶんの N 変数 (S, N) です。長期予測(long-term forecasting)では S を 96, 192, 336, 720 など長くとって精度を競うのが慣例です。評価指標は MSE(平均二乗誤差。予測値と正解の差を二乗して平均。大きな外れに敏感)と MAE(平均絶対誤差。差の絶対値の平均。外れに比較的鈍感)を使います。どちらも小さいほど良い指標です。

## 仕組みを詳しく

iTransformer の処理を tensor の形で追います。バッチサイズを B、変数数を N、入力系列長を T とします。

1. 入力 (B, T, N) を転置して (B, N, T) にする。これで「変数」が系列の単位になります。
2. 各変数の T 次元ベクトルを、線形層(全結合)で D 次元のトークンに埋め込む(embedding)。これを変数埋め込み(variate token)と呼びます。結果は (B, N, D)。例えば N=7, T=96, D=512 なら (B, 7, 512)。
3. このトークン列に対して、標準的な Transformer エンコーダ層を L 回重ねる。各層は
   - 多頭自己注意(Multi-head Self-Attention): N 個の変数トークン間で Attention。これが変数相関を学ぶ。
   - 層正規化(LayerNorm): ここでは各変数トークンを横断して正規化するのではなく、各変数ごとに正規化する形になり、変数ごとのスケール差を吸収する。
   - Feed-Forward Network: 各トークン内部(=各変数の表現)を独立に変換し、時間パターン(周期・トレンド)を学ぶ。
4. 最後に各変数トークンを線形層で未来 S 時刻に射影し、転置を戻して (B, S, N) を出力。

位置埋め込みは不要です。変数トークンには「順序」の意味がない(変数を並べ替えても予測は変わるべきでない)ので、むしろ位置情報を入れない方が自然です。時間順序の情報はトークン内部(T 次元ベクトル)が保持しています。

注目すべきは、これがアーキテクチャの発明ではなく「入力の見方の反転」だけだという点です。だから既存の高速 Attention(FlashAttention など)や効率化技法をそのまま流用できます。

## 手法の系譜と主要論文

時系列 Transformer の流れを追うと iTransformer の位置づけが見えます。

- Informer(Zhou et al., AAAI 2021 best paper, arXiv:2012.07436): 長系列を Attention で扱う計算量 O(T^2) を ProbSparse Attention で O(T log T) に削減。時刻トークン方式の代表。
- Autoformer(Wu et al., NeurIPS 2021, arXiv:2106.13008): 系列をトレンドと季節性に分解し、Auto-Correlation 機構で周期を捉える。
- FEDformer(Zhou et al., ICML 2022, arXiv:2201.12740): 周波数領域(フーリエ)で Attention を行い計算を線形化。
- DLinear / LTSF-Linear(Zeng et al., AAAI 2023, arXiv:2205.13504): 「単なる線形層が複雑な Transformer に勝てる」と示し、時刻トークン方式 Transformer の有効性に疑問を投げかけた重要な批判論文。
- PatchTST(Nie et al., ICLR 2023, arXiv:2211.14730): 系列を小区間(patch)に切ってトークンにし、かつ各変数を独立に扱う channel-independence を導入。Transformer 復権の流れを作る。
- iTransformer(Liu et al., ICLR 2024, arXiv:2310.06625): トークンの軸を完全に変数へ反転。PatchTST の channel-independence をさらに推し進めつつ Attention で変数相関を明示的に捉える。

DLinear の批判に対し、iTransformer は「Transformer が弱かったのではなく、トークンの切り方が悪かった」という回答を与えた、と読むと系譜が理解しやすいです。

## 論文の実験結果(定量データ)

ICLR 2024 論文では、ECL(電力消費)、Traffic(道路交通量)、Weather、Solar-Energy、ETT(変圧器温度、4種)などの標準ベンチマークで、予測長 {96, 192, 336, 720} の平均 MSE/MAE を比較しています。

代表的な数値(平均 MSE、低いほど良い、論文 Table 1 より)を挙げると:

- Traffic データセット(変数数 862 と非常に多い)で、iTransformer は平均 MSE 約 0.428。PatchTST は約 0.481、DLinear は約 0.625。変数が多いほど Attention による変数相関の効果が大きく、ここで差が顕著に開きます。
- ECL(変数数 321)で iTransformer 平均 MSE 約 0.178、PatchTST 約 0.205。
- Solar-Energy(変数数 137)で iTransformer 約 0.233、これも従来比で大きく改善。
- ETT 系(変数数 7 と少ない)では差は小さく、PatchTST と拮抗。変数が少ないと Attention で相関を見る旨味が薄れるためです。

論文の主張として、iTransformer は当時の SOTA を多くの設定で更新し、特に変数数が多いデータで強い、という傾向が明確に出ています。

アブレーション(ある要素を抜いて効果を測る実験)では:

- 転置を戻して時刻トークンにすると精度が大きく劣化することを確認。役割分担(Attention=変数、FFN=時間)が効いている証拠。
- 汎化実験: 学習時に一部の変数だけ見せ、推論時に未見の変数を増やしても性能が保たれる(変数をトークンにしたことで変数数に柔軟)。
- 効率: Attention の計算量は変数数 N に依存し時刻長 T には依存しないため、長い系列でもメモリ・速度面で有利。

## メリット・トレードオフ・限界

メリット:

- 変数相関を Attention で明示的に学べる。電力・交通のように変数が多く相互依存が強いデータで効く。
- 入力長 T が長くなっても Attention のトークン数は N のままなので、長期予測でスケールしやすい。
- アーキテクチャ非依存。FlashAttention や既存 Transformer 改良をそのまま適用できる。

トレードオフ・限界:

- 変数数 N が大きいと Attention は O(N^2) になる。Traffic の N=862 のような巨大な変数では Attention 自体が重くなる。
- 変数が少ない(N=7 など)データでは旨味が小さく、単純な線形モデルや PatchTST と大差ない。
- 各変数の系列まるごとを1トークンに埋め込むため、トークン内部での細かい時間構造(短いイベント)の解像度は埋め込み層の表現力次第。patch 方式より局所パターンを取りこぼす懸念がある。
- 変数間に時間的なラグ(変数 A が変数 B に数時刻遅れて影響)がある場合、それを Attention だけで直接表現しにくい。

自動運転の文脈で言えば、複数センサ・複数物体の状態を「変数」とみなして相関を捉える発想は応用が利きますが、空間的な位置関係を持つデータ(BEV グリッドなど)にそのまま適用するのは自然でなく、時空間モデルとの組み合わせが課題になります。

## 発展トピック・研究の最前線

- 変数数が膨大なケースへのスパース Attention 適用(変数クラスタリングして階層的に Attention するなど)。
- 時系列基盤モデル(Time-Series Foundation Model)との接続。多数の異種データセットで事前学習する流れの中で、変数トークン化は変数数が可変でも扱える利点があり相性が良い。
- channel-independence(PatchTST)と channel-dependence(iTransformer)のトレードオフをデータごとに自動で切り替える研究。変数相関が有益な場面とノイズになる場面が混在するため。
- 確率的予測(点予測でなく分布を出す)や、欠損・不揃いサンプリングへの拡張。
