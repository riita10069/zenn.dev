# Mixed Precision Training (AMP)

## ひとことで言うと
Mixed Precision Training(混合精度学習、略して AMP = Automatic Mixed Precision)は、学習で扱う数値を「ビット数の少ない小さな型」と「従来どおりの大きな型」を混ぜて使うテクニックです。小さい型を使う部分は計算が速くメモリも半分で済むので、学習が速くなりGPUのメモリも節約できます。精度がシビアに効く一部の計算だけ大きな型に残すことで、最終精度をほとんど落とさずに高速化・省メモリを実現します。

## 直感的な理解
すべての計算を「電卓の表示桁を最大」でやる必要はない、という発想です。買い物のレシートの合計はきっちり計算したいけれど、途中の概算は3桁もあれば十分、という場面があります。深層学習も同じで、行列のかけ算のような「たくさん足し合わせる重い計算」は多少桁が粗くても結果に響かない一方、損失の計算や合計のように小さな差が効く部分は正確さが要ります。混合精度は「粗くてよい計算は16ビットで速く・省メモリに、正確さが要る計算は32ビットで」と自動で振り分けます。

なぜ今これが重要か。近年のモデルは巨大化し、すべてを fp32(32ビット浮動小数点)で持つと GPU メモリ(VRAM)に乗り切らず、計算も遅くなります。高解像度の画像処理や大きなバッチを限られたGPUで回すには、メモリと速度の両方を稼ぐ混合精度が事実上の標準になっています。

## 基礎: 前提となる概念
- 浮動小数点数: コンピュータが小数を表す形式。`符号 × 仮数 × 2の指数乗` の形で、ビットを「指数部」と「仮数部」に配分します。指数部が広いほど表せる数の範囲(最大・最小)が広く、仮数部が多いほど細かい桁(精度)を表せます。
- fp32(単精度): 指数8ビット + 仮数23ビット。深層学習が伝統的に使ってきた型。範囲も精度も十分。
- fp16(半精度): 指数5ビット + 仮数10ビット。範囲が狭い(正の最小はおおむね 6e-5、最大は 65504)。範囲が狭いせいで小さすぎる値が0に丸まる「アンダーフロー」や、大きすぎる値が無限大になる「オーバーフロー」を起こしやすい。
- bf16(bfloat16, Brain Floating Point): 指数8ビット + 仮数7ビット。指数が fp32 と同じ8ビットなので範囲は fp32 と同じくらい広い。そのかわり仮数が7ビットしかなく精度は粗い。範囲が広いのでアンダーフローしにくいのが最大の利点。
- TF32(TensorFloat-32): NVIDIA Ampere 以降が内部演算に使う型。範囲は fp32(指数8)、仮数は10ビット。fp32 のつもりで書いたコードを自動で TF32 に落として加速する。
- Tensor Core: Volta 世代以降の NVIDIA GPU に載る、16ビット行列積を専用に高速処理する回路。fp32 のままだと活かせず、16ビット型を使うと理論上数倍速くなる。
- 活性化(activation): 各層の途中出力。学習中はこれを逆伝播のために大量に保持するため、VRAM の主な消費源。16ビット化が最も効くのはここ。

範囲の数値例。学習が進むとネットワーク入り口に近い層の勾配は 1e-7 のような極小値になることがあります。これは fp16 の表せる最小(約6e-5)よりずっと小さく、fp16 では0に丸まって消えます(勾配消失)。一方 bf16 は範囲が広いので 1e-7 でもそのまま表せます。ここが fp16 と bf16 の運命を分けます。

精度(仮数ビット)の数値例も押さえます。仮数が `m` ビットの型では、隣り合って表せる数の相対的な刻みはおよそ `2^-m` です。fp32 は仮数23ビットなので相対刻みは約 1.2e-7、fp16 は10ビットで約 9.8e-4、bf16 は7ビットで約 7.8e-3。つまり bf16 で 1.0 のすぐ隣に表せる数は 約 1.0078 で、その間の値はどちらかに丸められます。これが「bf16 は範囲は広いが桁が粗い」の中身です。たくさん足し合わせる行列積では、各項の丸め誤差が打ち消し合いやすく粗さが目立ちにくい一方、長い累積和(running sum)や、ごく近い2数の引き算(桁落ち、catastrophic cancellation)では誤差が表に出ます。だから「足し込みの累積器(accumulator)は fp32 で持つ」のが Tensor Core の標準動作です。16ビットの入力を掛け、足し込みは32ビットで行うことで、速度と数値安定性を両立します。

## 仕組みを詳しく
AMP は「どの演算を16ビットで、どれを fp32 で行うか」を自動で振り分けます。PyTorch では `torch.autocast` というコンテキストマネージャがこの役を担い、演算の種類ごとに目標型を持つ「allowlist / blocklist(対象リスト・除外リスト)」に従って動的にキャストします。

おおまかな方針。
- 行列積・畳み込みのような「多数を足し合わせる重い演算」→ 16ビット(bf16 など)。速くて省メモリ。
- 合計、ソフトマックス(値を確率に変換する関数)、損失計算、各種正規化(Layer Normalization など)→ fp32。これらは小さな差や桁が効くので精度を残す。

最重要のしくみが「マスターウェイト(master weight)は fp32 で保持する」点です。学習は重みを `重み ← 重み - 学習率 × 勾配` で少しずつ更新しますが、この更新量はとても小さい。重みを16ビットで持つと、小さな更新が桁落ちして「足したのに変わらない」現象が起きます(0.0001 を16ビットの大きな重みに足しても丸められて消える)。そこで重み本体は fp32 で正確に保ち、forward 計算の直前だけ16ビットにコピーして使う、という二層構えにします。

典型的な学習1ステップ(bf16 の場合)。
```python
with torch.autocast(device_type="cuda", dtype=torch.bfloat16):
    out  = model(x)          # 重い演算は bf16、loss など精度要る所は fp32 に自動振り分け
    loss = criterion(out, y)
loss.backward()              # 勾配計算。bf16 は範囲が広く、Loss Scaling 不要
clip_grad_norm_(model.parameters(), 1.0)  # 勾配の暴走を防ぐクリッピング(任意)
optimizer.step()             # fp32 マスターウェイトを更新
optimizer.zero_grad()
```

fp16 を使う場合だけは、上の `backward` の前後に Loss Scaling(損失を一時的に大きくして小さい勾配のアンダーフローを防ぐ)が要ります。bf16 は範囲が広いのでこれが不要で、コードがそのぶん単純になります。詳しくは [Loss Scaling / GradScaler](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0187_loss-scaling) を参照。

メモリがどれだけ減るか、内訳で考えます。学習中の VRAM はおおまかに「モデル重み + 勾配 + オプティマイザ状態 + 活性化」です。混合精度では活性化が fp32 → 16ビットで半減し、ここが最大の節約。一方マスターウェイトは fp32 のまま、さらに16ビットのコピーも持つため、重みまわりは単純な半減にはなりません。それでも活性化が支配的な大きなモデル・高解像度では、全体として実用的なメモリに収まることが多いです。

bf16 vs fp16 の使い分け。bf16 対応GPU(NVIDIA Ampere/Ada/Hopper、Google TPU など)が手元にあるなら bf16 が第一候補です。範囲が広く Loss Scaling 不要で、学習が壊れにくい。古いGPU(例: NVIDIA T4、Pascal/Turing 世代)は bf16 非対応のことがあり、その場合は fp16 + Loss Scaling を使います。精度がシビアなモデルでは「問題の出る層だけ fp32 に固定する」部分的な除外も実務でよく使われます。

## 手法の系譜と主要論文
- Micikevicius ら「Mixed Precision Training」(2018, ICLR, arXiv:1710.03740, NVIDIA & Baidu)。fp16 + fp32 の混合精度を体系化。3本柱は (1) fp32 マスターウェイト、(2) Loss Scaling、(3) 一部演算を fp32 で実行。これが現在の AMP の原型です。
- Wang & Kanwar(2019, Google)と Kalamkar ら「A Study of BFLOAT16 for Deep Learning Training」(2019, arXiv:1905.12322, Intel/Google)。bf16 を学習に使う流れを確立。fp32 と同じ指数範囲ゆえ Loss Scaling なしで安定すると示し、fp16 の面倒さを取り除きました。TPU の bf16 はこの系譜です。
- NVIDIA「TensorFloat-32」(2020, Ampere 世代)。コードを書き換えずに fp32 演算を内部で TF32 に落として加速する仕組み。混合精度を「明示的に書かなくても一部恩恵が得られる」方向の発展。
- 量子化学習・推論への展開。Dettmers ら「LLM.int8」(2022)などで8ビット・4ビットの低精度が研究され、混合精度の考え方(精度の要る所だけ高ビットに残す)が極端な低ビットまで拡張されています。fp8(8ビット浮動小数点)を学習に使う研究も Hopper 世代以降で進行中です。

## 論文の実験結果(定量データ)
- Micikevicius et al.(2018)。ImageNet 分類(AlexNet・VGG・Inception・ResNet50 など。指標は Top-1 / Top-5 正解率)で、fp16 混合精度が fp32 と同等の精度を達成。ResNet50 などで差はおおむね 0.1〜0.3 ポイント以内、誤差の範囲。物体検出(Faster R-CNN・SSD、指標は mAP = 平均適合率、検出の当たり具合)、機械翻訳(BLEU)、音声認識(WER = 単語誤り率、低いほど良い)、言語モデリング(perplexity)など多領域で fp32 と同等を確認しました。メモリはおおむね半減、対応ハードでは学習が大きく高速化。アブレーションとして、Loss Scaling を外すと一部モデル(SSD など)で fp16 勾配の大半がアンダーフローして学習が進まないことを示し、3本柱それぞれの必要性を裏づけました。
- Kalamkar et al.(2019)。ResNet・DenseNet・物体検出・GNMT(翻訳)・DeepSpeech2(音声)・推薦・GAN など幅広いモデルで、bf16 が fp32 とほぼ同等の精度に到達し、しかも fp16 で必要だった Loss Scaling が bf16 では不要だと実証。指標は各タスクの標準指標(正解率・BLEU・WER など)で、いずれも fp32 との差は実用上無視できる範囲でした。重要な含意は「16ビットの速度・省メモリを、fp16 のチューニング地獄なしで得られる」ことです。

数値の意味。Top-1 正解率の 0.1〜0.3 ポイント差は「1000枚に1〜3枚分の判定が入れ替わる程度」で、乱数シードを変えた再学習でも生じるばらつきと同程度。つまり「精度を犠牲にせずメモリ半減・高速化」という主張が成立する根拠になっています。

## メリット・トレードオフ・限界
メリット
- 活性化が半減し、大きなモデル・高解像度・大きなバッチを同じGPUで回せる。
- Tensor Core を活かして学習が高速化(対応GPUで顕著)。
- bf16 なら Loss Scaling 不要で、ほぼ精度を落とさず導入できる。
- PyTorch では autocast で囲むだけと導入が容易。

トレードオフ・限界
- 16ビットは精度が粗く、まれに数値的に敏感なモデル(極端に小さい分散の正規化、累積和が長い演算など)で不安定化する。対処は該当箇所だけ fp32 に戻すこと。
- fp16 を選ぶなら Loss Scaling のチューニングが要り、設定ミスで学習が壊れうる。
- bf16 は仮数7ビットと精度が粗く、稀に微妙な精度差が問題になるタスクがある。
- マスターウェイトを fp32 で持つため、重み本体のメモリ削減効果は活性化ほど大きくない。むしろ重み・勾配・最適化状態がボトルネックの巨大モデルでは、別途 ZeRO/FSDP のような分割手法や量子化が必要。
- bf16 非対応の古いGPUでは bf16 の恩恵を受けられない。

## 発展トピック・研究の最前線
- fp8 学習。Hopper/Ada 世代の Tensor Core は8ビット浮動小数点に対応します。fp8 には2フォーマットあり、E4M3(指数4・仮数3、範囲は狭いが精度は相対的に高い。順伝播・重み向き)と E5M2(指数5・仮数2、範囲が広く勾配向き)を使い分けます。仮数2〜3ビットは極めて粗く、テンソルごとに動的なスケール(delayed scaling などスケール履歴を使う方式)を当てないと即破綻するため、Loss Scaling の発想をテンソル粒度へ押し進めたスケール管理が鍵です。NVIDIA の Transformer Engine などがこの fp8 混合精度を実装し、大規模言語モデルの学習で bf16 比でさらなる高速化・省メモリを報告しています。混合精度の延長線上にある最前線です。
- 巨大モデル向けのメモリ最適化との併用。混合精度は活性化を減らしますが、重み・勾配・最適化状態が主因の超大規模学習では、ZeRO/FSDP(状態を複数GPUに分割)や activation checkpointing(活性化を一部捨てて再計算)と組み合わせます。
- 量子化学習(QAT)・量子化推論。学習後に4〜8ビットへ落とす推論最適化や、学習中から低ビットを前提にする手法が活発。混合精度の「精度の要る所だけ残す」思想がより極端な形で受け継がれています。
