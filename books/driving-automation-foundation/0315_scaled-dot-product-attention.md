# スケール化内積アテンション

## ひとことで言うと

スケール化内積アテンション(Scaled Dot-Product Attention)とは、「たくさんの候補の中から、いま自分に関係の深いものを選んで重み付けで取り出す」計算です。アテンション(attention、これは「注目」という意味です)という名前のとおり、関係が強いものほど大きく注目し、その内容を多めに混ぜて取り出します。式は次の一行に凝縮されます。

```
Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) V
```

この一行が、現代の言語モデル・画像モデル・マルチモーダルモデルのほぼすべての中核になっています。GPT・BERT といった大規模言語モデルも、Vision Transformer のような画像モデルも、複数カメラの特徴を融合する自動運転の知覚モデルも、すべてこの計算を何十回も積み重ねて作られています。

## 直感的な理解

図書館で調べ物をする場面を思い浮かべてください。あなたは「自動運転の安全性について知りたい」という問い(これが query です)を持っています。書架には無数の本があり、各本の背表紙にタイトル(これが key です)が書かれています。あなたはタイトルを見比べて、問いに合う本ほど強く注目し、その本の中身(これが value です)を多めに読む、という配分をします。全部の本を同じだけ読むのではなく、関連度に応じて読む量を変えるのがアテンションです。

なぜこれが要るのか。固定の足し算や畳み込みでは「常に同じ場所を見る」しかできません。畳み込みは「隣り合う画素」という固定的な近さしか見られず、離れた位置の関係を捉えるには層を深く重ねる必要があります。しかし現実の問題では、状況ごとに見るべき場所が変わります。文章の意味理解なら「それ」という代名詞が何を指すかは文ごとに違いますし、画像の融合なら、あるマスがどの画素を参照すべきかは内容次第です。アテンションは、入力の中身に応じて注目先を毎回その場で計算するので、こうした柔軟で文脈依存の選択ができます。しかも離れた位置どうしを 1 ステップで直接つなげるため、長距離の依存関係を効率よく扱えます。これがアテンションが幅広いタスクで強い理由です。

## 基礎: 前提となる概念

まずベクトルの内積(dot product)とは、二つの数値ベクトルの要素同士を掛けて全部足した値です。たとえば [1,2,3] と [1,0,1] の内積は 1×1+2×0+3×1 = 4 です。内積は「二つのベクトルがどれだけ似た方向を向いているか」を測る量で、似た方向なら大きく、無関係なら 0 付近、逆向きなら負になります。アテンションはこの内積を「query と key の相性スコア」として使います。なぜ内積かというと、行列の掛け算としてまとめて計算でき、GPU が最も得意とする演算だからです。

次に softmax(ソフトマックス)とは、複数の実数を「合計が 1 になる確率の配分」に変換する関数です。各値を指数関数 exp で正の数にしてから、その総和で割ります。たとえば [2, 0, -1] を softmax すると、おおよそ [0.84, 0.11, 0.04] になります。大きい値ほど大きな確率を、小さい値ほど小さな確率を受け取り、合計は必ず 1 になります。アテンションでは、これを「各候補にどれだけ注目するかの重み」として使います。合計が 1 になるので、出力は value たちの加重平均になります。

最後にテンソルの形状(shape)について。本稿では B をバッチサイズ(同時に処理するデータ数)、L_q を query の個数、L_k を key/value の個数、d を 1 つのベクトルの次元数とします。形状を追うことが、アテンションの理解で最も大切です。形状さえ追えれば、どこで何が起きているかが正確に分かります。

## 仕組みを詳しく

登場人物は三つです。

- Q(query、問い合わせ): 探す側。「私はこういう情報がほしい」という手がかり。
- K(key、見出し): 探される側のラベル。「この候補はこういう性質です」という見出し。
- V(value、中身): 実際に取り出される本体の中身。

Q, K, V は通常、もとの特徴に学習可能な重み行列を掛けて作ります。つまり同じ入力ベクトルから、三つの異なる線形変換で「問い」「見出し」「中身」の三役を切り出すわけです。同じ入力から Q も K も V も作る場合を自己アテンション(self-attention)、別々の入力から作る場合をクロスアテンション(cross-attention)と呼びます。自己アテンションは系列内の要素どうしの関係を、クロスアテンションは二つの異なる系列(たとえば質問と文書、あるいは BEV クエリと画像特徴)の対応づけを担います。

形状の流れを段階的に追います。

1. Q `[B, L_q, d]`、K `[B, L_k, d]`、V `[B, L_k, d]`
2. Q と K の内積 `Q K^T` → scores `[B, L_q, L_k]`。scores[i][j] は「i 番目の query と j 番目の key の相性」。
3. `sqrt(d_k)` で割る(理由は後述)。
4. softmax を最後の次元 L_k 方向にかける → 各 query について、全 key への重み(合計 1 の確率分布)になる。形は `[B, L_q, L_k]` のまま。
5. その重みで V を加重平均 → out `[B, L_q, d]`。

具体例。d_k=4 で、ある query と 3 個の key の内積が scores=[2, 0, -1] だったとします。softmax すると おおよそ [0.84, 0.11, 0.04] となり、1 番目の value を約 8 割の重みで、2 番目を約 1 割、3 番目をほんの少しだけ混ぜて取り出します。これが「関係の深いものを多めに取り出す」の中身です。出力は L_q 個分作られ、それぞれの query が自分なりの加重平均を得ます。

ステップ 4 の前に「マスク」を足すことがあります。たとえば言語生成では未来のトークンを見てはいけないので、未来位置の scores に -∞(実装上は非常に大きな負の値)を足してから softmax にかけると、その位置の重みがゼロになります。これが因果マスク(causal mask)です。可変長の系列をまとめて処理するとき、詰め物(padding)の位置を無効化するのも同じマスクの仕組みです。

### なぜ sqrt(d_k) で割るのか

これがこの手法の名前(スケール化)の核心です。内積は要素の積を d_k 個足すので、次元 d_k が大きいほど値が大きく振れます。各要素が平均 0・分散 1 の独立な乱数だと仮定すると、内積の平均は 0、分散は d_k になり、ばらつき(標準偏差)は sqrt(d_k) に比例します。たとえば d_k=256 だと内積が ±16 程度まで簡単に振れます。

softmax は入力が大きいと出力がほぼ [1, 0, 0, ...] に張り付きます(飽和、これは「片方に偏りきって動かなくなる」状態です)。飽和した領域では softmax の勾配(学習の手がかりとなる傾き)がほぼ 0 になり、学習が進みません。そこで sqrt(d_k)=16 で割ると内積が分散 1 程度に収まり、softmax が適度に滑らかになって勾配が流れます。スケール化とは「次元が大きくても内積の大きさを一定範囲に保ち、softmax の飽和を防ぐ」ための割り算です。Vaswani らの論文も、大きな d_k で内積が増大して softmax を飽和させ、勾配を極端に小さくする点をこの割り算の理由に挙げています。割り算ひとつですが、これがないと高次元での学習が成立しないため、手法名に冠されるほど本質的です。

### マルチヘッドアテンション

multi-head(マルチヘッド、多頭)とは、Q/K/V を d 次元のまま 1 回計算するのではなく、たとえば 8 つに分けて d/8 次元ずつ 8 回並行してアテンションし、結果を連結する方法です。各ヘッドが「文法的な関係」「意味的な関係」「位置的な関係」のように別々の観点を担当でき、表現が豊かになります。1 ヘッドだと softmax で一つの加重平均しか作れず「平均化されすぎる」傾向があるのに対し、複数ヘッドは複数の注目パターンを同時に保持できます。d=256 なら 8 ヘッドで各 32 次元、という構成が典型的です。並行計算なので、ヘッドを増やしても 1 ヘッドあたりの次元が減るぶん総計算量はほぼ変わりません。

```
入力 [B, L, 256]
 → 8 ヘッドに分割: 各 [B, L, 32]
 → 各ヘッドで Scaled Dot-Product Attention
 → 8 個の [B, L, 32] を連結 → [B, L, 256]
 → 出力射影(線形層)
```

## 手法の系譜と主要論文

アテンションの起源は Bahdanau ら(ICLR 2015、arXiv:1409.0473)の機械翻訳研究です。当時の翻訳モデルは、入力文全体を一つの固定長ベクトルに押し込んでから訳していたため、長い文で情報が溢れて精度が落ちました(情報ボトルネック)。Bahdanau らは「訳語を一つ生成するたびに、入力文のどの単語に注目すべきかを毎回計算して重み付けで参照する」アテンションを導入し、長文翻訳を大きく改善しました。これがアテンションの最初の本格的な成功です。このときのスコア計算は小さなニューラルネット(加法アテンション、additive attention)でした。続く Luong ら(EMNLP 2015、arXiv:1508.04025)は、スコア計算を内積など複数方式で比較し、内積ベースの「乗法的アテンション(multiplicative attention)」が単純で有効だと整理しました。これが内積アテンションの直接の前身です。

次に Vaswani ら「Attention Is All You Need」(NeurIPS 2017、arXiv:1706.03762)が決定的な飛躍をもたらしました。提案内容は、再帰(RNN)も畳み込みも一切使わず、アテンションだけで系列を処理する Transformer です。その中核がスケール化内積アテンションで、Luong らの乗法的アテンションに sqrt(d_k) のスケール化を加えました。内積は行列積として GPU で極めて高速に計算でき、しかも RNN と違って系列を逐次処理しないので全位置を並列計算できるため、加法アテンションより速く、かつ大規模化に向いていました。さらにマルチヘッドと自己アテンションを組み合わせ、系列内の任意の二要素を 1 ステップで関連づけられるようにしました。これが現在の大規模言語モデルすべての基盤です。

その後の発展として FlashAttention(Dao ら、NeurIPS 2022、arXiv:2205.14135)があります。`Q K^T` の巨大な中間行列(L_q × L_k)を GPU の遅いメモリ(HBM、High Bandwidth Memory だが演算ユニットから見ると遠い)に丸ごと書き出すのがボトルネックだという観察から、行列を小ブロックに分けて高速なオンチップメモリ(SRAM)上で逐次計算し、巨大な中間行列を実体化しないオンライン softmax を使います。数式上の結果は厳密に同じで、計算の段取り(メモリアクセスのパターン)だけを変えた最適化です。これにより長い系列でもメモリ消費を二乗ではなく線形に抑え、数倍の高速化を実現しました。「IO-awareness(メモリ転送量を意識した設計)」という観点をアテンション最適化に持ち込んだ点が新規性です。

## 論文の実験結果(定量データ)

Vaswani ら(2017)の主実験は機械翻訳です。評価指標は BLEU スコア(機械翻訳の品質を 0〜100 で測る指標で、参照訳とどれだけ n-gram が一致するか。高いほど良い)です。WMT 2014 の英独翻訳で、Transformer(big モデル)は BLEU 28.4 を達成し、それまでの最高だった各種モデルやアンサンブルを約 2 BLEU 以上上回りました。英仏翻訳では BLEU 41.8(single model)を記録し、こちらも当時の単一モデル最高を更新しました。

特筆すべきは学習コストです。論文は、従来の最高性能モデルが要した学習計算量のごく一部(報告によると数分の一から桁違いに少ない計算)でこの精度を達成したと述べています。英独で従来手法を上回りつつ、8 枚の GPU で 3.5 日(big モデル)という当時としては効率的な学習で到達した点が、Transformer の実用的な強さを示しました。RNN と違い全位置を並列計算できることが、この学習効率の源です。

アブレーション(要素を抜く・変える実験)も豊富です。論文はヘッド数を変える実験を行い、単一ヘッド(h=1)はマルチヘッドより BLEU が約 0.9 低下し、逆にヘッドを増やしすぎても(各ヘッドの次元が小さくなりすぎて)わずかに劣化することを示しました。最適は中間のヘッド数(論文では 8 や 16)でした。また、スケール化を外して内積をそのまま softmax に入れると大きな d_k で学習が不安定になることが、この割り算の必要性を裏づけています。FlashAttention(2022)では、GPT 系モデル(GPT-2)の学習で標準実装比で約 3 倍の高速化を報告し、BERT-large の学習を当時の MLPerf 記録より約 15% 速くしたとされます。系列長を伸ばしてもメモリが線形にしか増えないため、それまで扱えなかった長い系列(数千〜数万トークン)の学習を現実的にしました。結果が数値的に同一のまま実行効率だけを改善できることを定量的に示した点が重要です。

ここで BLEU の差の意味を補足します。機械翻訳では BLEU が 1 ポイント違うと人間が読んでも訳の自然さに体感差が出る程度の改善で、2 ポイント超の改善は当時としては大きな前進でした。Transformer はそれを少ない計算で達成したからこそ、研究コミュニティ全体が一気にこのアーキテクチャへ移行しました。

## メリット・トレードオフ・限界

メリットは、注目先を入力の中身から毎回計算するため文脈に応じて柔軟であること、離れた位置の関係も 1 ステップで捉えられること(RNN のように逐次伝播せずに済む)、そして行列積として並列計算しやすく大規模化に向くことです。

トレードオフと限界の最大のものは計算量です。scores は `[L_q, L_k]` の大きさになるため、自己アテンション(L_q = L_k = N)では計算量とメモリが N の二乗で増えます。系列長 N が 1000 から 1 万になると計算が 100 倍になり、長文書・高解像度画像・長時間動画では深刻なボトルネックになります。たとえば 1 枚の高解像度画像を画素単位でアテンションさせると、画素数の二乗の注意行列が必要になり、すぐにメモリが枯渇します。また sqrt(d_k) のスケール化や適切なヘッド数、正規化(LayerNorm)を併用しないと学習が不安定になりやすく、設計の繊細さも限界の一つです。

研究上の未解決課題として、この二乗計算量をどう抑えるかが大きなテーマであり続けています。FlashAttention はメモリアクセスを最適化しましたが計算量自体は二乗のままです。後述する近似手法は計算量を線形に落とせる一方で精度を犠牲にしがちで、「精度を保ったまま線形化する」ことは依然として完全には解けていません。長文脈をどう安く扱うかは、大規模言語モデル時代の最重要課題の一つです。

## 発展トピック・研究の最前線

二乗計算量を緩和する研究は大きな潮流です。Linformer や Performer は、注意行列を低ランク近似やカーネル近似で計算量を系列長に対して線形に落としました。Longformer や BigBird は、全要素ではなく局所窓+一部のグローバルトークンだけに注目を限定するスパースアテンションで長文に対応しました。これらは精度と効率のトレードオフを探る試みで、タスクによって有効性が変わります。

画像・自動運転の文脈では、全画素を見る密なアテンションは高解像度で破綻するため、Deformable Attention(Zhu ら、Deformable DETR、ICLR 2021、arXiv:2010.04159)のように「各 query が学習で決めた少数のサンプル点だけを参照する」疎なアテンションが使われます。これは softmax で加重平均する点はそのままに、参照先を数点(たとえば 4 点)に絞ることで計算量を系列長に対して線形に減らす設計で、DETR の収束を約 10 倍速め、小物体検出も改善したと報告されています。参照点を幾何的に指定できるため、BEV(鳥瞰図)生成のようなカメラ画像と空間座標の対応づけと相性が良いことが知られています。

さらに最近では、推論時の KV キャッシュ(過去の Key/Value を貯めて再利用する)の圧縮や、グループ化クエリアテンション(Grouped-Query Attention、複数の query ヘッドで Key/Value を共有しメモリを削る)、マルチクエリアテンションなど、アテンションの効率化は大規模モデル時代の最重要課題として研究が続いています。これらは長文推論のメモリ消費を数分の一に抑えつつ精度をほぼ保つもので、最新の大規模言語モデルの多くが採用しています。スケール化内積アテンションという 2017 年の一行の式が、効率化・近似・ハードウェア最適化の膨大な研究を生み続けている、というのが現在の姿です。
