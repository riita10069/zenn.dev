# 自車運動と曲率の導出

## ひとことで言うと
データセットには「速度」や「位置」は入っていても、「カーブの曲がり具合(曲率)」や「向きの変化速度(ヨーレート)」が直接入っているとは限りません。そこで、記録されている量から差分(隣り合う時刻の引き算)を取って、これらを計算で導き出します。ただし引き算でノイズが増幅されるという厄介な性質があり、その扱いが要点です。

## 直感的な理解
運転の操作は、突き詰めると2つしかありません。アクセル/ブレーキ(=前後にどれだけ加速・減速するか=縦加速度)と、ハンドル(=どれくらい曲がるか=曲率)です。だから自車の運動を「加速度」と「曲率」の2信号の時系列で表すと、運転の本質を簡潔に記述できます。エンドツーエンド運転モデルが未来の軌跡をこの2信号で予測するのは、この簡潔さと「物理的に妥当な軌跡を出しやすい」という利点のためです。

ところが現実のデータセットは、記録している生の量がバラバラです。あるデータセットは速度・進行方向・前後加速度を直接記録していますが、別のデータセットは6自由度の自車姿勢(位置 + 向き)だけしか記録していないこともあります。どちらの場合も、最終的に必要な4信号 `[speed, acceleration, yaw(_rate/angle), curvature]` をそろえる必要があり、足りない量は計算で導出するしかありません。この「足りない物理量を、記録済みの量から数値微分で補う」のがこのトピックです。

## 基礎: 前提となる概念

### ヨーレートと曲率
- ヨーレート(yaw rate): 車の向き(ヘディング)が1秒あたり何ラジアン変わるか。単位は rad/s。直進ならほぼ0、急ハンドルほど大きい。
- 曲率(curvature): 走っている道(軌跡)の曲がり具合。半径 R の円を描いて曲がっているなら曲率は `κ = 1/R`(単位 1/m)。半径が小さい急カーブほど曲率は大きい。直進は曲率0。

この2つは関係していて、`曲率 = ヨーレート / 速度`(κ = ω/v)という近似式で結ばれます。直感的には、同じヨーレートでも速く走っていれば軌跡は緩いカーブ(曲率小)、ゆっくりなら急なカーブ(曲率大)になる、ということです。これは円運動の関係式 `v = R·ω` を `κ = 1/R = ω/v` と書き直したものです。

### 有限差分
有限差分(finite difference)とは、微分(変化の速さ)を、隣り合うサンプルの引き算で近似することです。連続的な微分 `dh/dt` を、`(h[i] - h[i-1]) / Δt` で近似します。10Hz のデータなら `Δt = 0.1` 秒です。前方差分(後ろから引く)、後方差分、中央差分(前後で引く)の3種があり、中央差分 `(h[i+1] - h[i-1]) / (2Δt)` は誤差の次数が高く(2次精度)、ノイズに少し強い性質があります。

### 角度の連続化(unwrap)と四元数
- unwrap: 角度は通常 -π〜+π で表され、たとえば +3.13 から -3.13 へ動くと、実際はほんの少し回っただけなのに数値上は約 6.28 の大ジャンプに見えます。`np.unwrap` はこの不連続を検出して連続な角度に直し、差分が暴れないようにします。これを忘れると、向きが π をまたぐ瞬間に巨大な偽のヨーレートが出ます。
- 四元数(quaternion): 回転を表す4つの数(x,y,z,w)です。姿勢が四元数で記録されている場合、ヨー角(Z軸まわりの回転)を取り出すにはオイラー角への変換が要ります。

## 仕組みを詳しく

### 速度・向きが直接ある場合
生の `heading`(進行方向の角度)と `speed` が記録されているなら、ヨーレートと曲率を次のように導出します。

```
heading = np.unwrap(heading)                      # ±π をまたぐ段差をならす
yaw_rate[i] = (heading[i] - heading[i-1]) / 0.1    # 有限差分(10Hz なので Δt=0.1)
curvature   = yaw_rate / speed                     # speed≈0 のときは 0 に保護
```

curvature では `speed` が極小(停車中)のとき `yaw_rate / speed` が0除算で発散するため、speed が閾値以下なら曲率を0に置き換える保護を入れます。

### 位置と回転だけがある場合
速度も向きの数値も無く、位置(translation)と回転(quaternion)だけのデータセットでは、2段階で導出します。

```
速度:     velocity_xy = np.gradient(translations[:, :2], t_s, axis=0)   # 位置の時間微分
          speed       = norm(velocity_xy)                               # その大きさ
加速度:   acceleration = np.gradient(speed, t_s)                        # 速度の時間微分
向き:     yaw_angle   = Rotation.from_quat(quats).as_euler("ZYX")[:, 0] # 四元数→ヨー角
ヨーレート: yaw_rate   = np.gradient(np.unwrap(yaw_angle), t_s)
曲率:     curvature   = yaw_rate / max(speed, 0.1)                      # 下限 0.1 m/s で保護
```

`np.gradient` を使う理由: 単純な前方差分と違い、`np.gradient` は実際の(不等間隔かもしれない)タイムスタンプ `t_s` を渡せて、内部は中央差分、端は片側差分で計算します。中央差分は前方差分よりノイズに少し強い性質があります。四元数からヨー角を取り出すのは `scipy` の `Rotation` を使い、`ZYX` オイラー角の最初の成分(Z軸まわり = ヨー)を取ります。

数値例: ある2フレームで位置が `(0,0)→(2.0,0.1)` m に動き、`Δt=0.1` s なら、速度はおよそ `√(2.0²+0.1²)/0.1 ≈ 20.0` m/s(=72 km/h)。同じ区間でヨー角が `0.00→0.01` rad 変わったなら yaw_rate ≈ `0.1` rad/s、曲率 ≈ `0.1/20.0 = 0.005` 1/m(半径 200 m の緩いカーブ)。

### history/future の切り出し
導出した `(T,4)` の信号列から、ある瞬間 `frame_idx` を中心に、過去64ステップを history(全4信号 → 256次元)、未来64ステップを future(うち加速度と曲率の2信号 → 128次元)として切り出します(10Hz なので各 6.4 秒)。history は「これまでどう動いたか」の文脈、future は「これからどう動くべきか」の正解です。

## 手法の系譜と主要論文

### 運動学的自転車モデル(kinematic bicycle model)
曲率を行動表現に使う発想の土台です。車両を「曲率(ハンドル)と縦加速度(アクセル/ブレーキ)で動く2輪モデル」として近似する古典的モデルで、Kong ら(IV 2015)が学習・制御文脈での妥当性を整理しています。状態 `(x, y, θ, v)` が制御入力 `(a, κ)` で `ẋ = v cosθ, ẏ = v sinθ, θ̇ = vκ, v̇ = a` と発展する、という単純で物理的に妥当なモデルです。多くのプランニング研究の出発点になっています。

### 数値微分とノイズ対策の古典
- Savitzky & Golay(Anal. Chem. 1964): 局所多項式フィッティングで平滑化しつつ微分する手法。単純な移動平均より波形の形を保ったまま微分でき、ノイズの多いセンサ信号の微分の定番です。
- Kalman(1960): カルマンフィルタ。位置・速度・加速度を運動モデルと観測ノイズの統計から同時推定し、微分由来のノイズを原理的に抑えます。自動運転の状態推定の基礎。

### エンドツーエンド運転での運動学的表現
- UniAD(Hu et al., CVPR 2023, arXiv:2212.10156): 知覚から計画まで統合し、最終的に自車軌跡を出力。
- VAD(Jiang et al., ICCV 2023, arXiv:2303.12077): ベクトル化シーン表現を用い、軌跡の滑らかさ・実行可能性を重視。

これらは「(x,y) を直接回帰するより、曲率・加速度のような運動学量で表したほうが破綻しにくい」という選択と動機を共有します。生の座標を直接予測すると、物理的にありえない軌跡(瞬間的なワープや急角度)が出やすいのに対し、制御量で表すと運動学モデルを通すことで自然と滑らかになります。

## 論文の実験結果(定量データ)

軌跡の表現方法と平滑化は、計画指標に直接効きます。

- 計画指標の意味: エンドツーエンド運転は通常 nuScenes 上で、L2 誤差(予測軌跡と正解軌跡の平均距離、メートル、低いほど良い)と衝突率(collision rate、予測軌跡が他物体と衝突する割合、低いほど良い)で評価されます。UniAD はこれらで当時の最先端を更新し、知覚から計画までの一体学習の効果を定量化しました(平均 L2・平均衝突率ともに先行手法を下回ると報告)。
- 運動学制約の効果: 運動学モデルを通して軌跡を生成する手法は、生の waypoint 回帰に比べて衝突率と物理的妥当性(曲率や加速度が車両の限界内に収まる割合)で改善する例が広く報告されています。これは「制御量で表すと破綻しにくい」ことの定量的裏付けです。
- 平滑化のトレードオフ(アブレーション的観察): 信号を強く平滑化するとノイズ起因の偽の急操作は消えますが、平滑化の窓を広げるほどラグ(遅れ)が増え、急ブレーキや回避のような反応の速い操作のピークが鈍ります。窓幅を変えた比較では、窓が狭いとノイズで曲率がギザギザ、広いと本物の急カーブのピークが削れる、という典型的なトレードオフが観測されます。最適点は速度帯やセンサ精度に依存します。

## なぜノイズが問題なのか(深掘り)

有限差分の最大の弱点は「ノイズの増幅」です。位置や角度の測定にわずかな誤差(ノイズ)が乗っていると、それを `Δt = 0.1` で割る微分操作は、その誤差を約10倍に拡大します。微分を2回重ねる加速度(位置→速度→加速度)はさらに悪化し、ノイズが約100倍に増幅されます。

数値例: 真の位置が一定速度で動いていても、GPS のジッタで位置に ±2cm の揺らぎが乗ると、速度には `±0.02/0.1 = ±0.2` m/s のギザギザが出ます。曲率は `yaw_rate/speed` で、分子のヨーレートが微分由来でノイジーなうえ、分母の speed が小さいと割り算で爆発的に増幅されます。だから停車・低速時の曲率は特に信頼できません(speed に下限を入れても、下限付近の値は依然あてになりにくい)。

対策は段階的に強められます。
1. unwrap で角度段差を除く(必須。これが無いと π またぎで破綻)。
2. speed に下限を設けて0除算を防ぐ(必須)。
3. 中央差分(np.gradient)を使う(前方差分よりノイズに強い)。
4. 差分前に移動平均や Savitzky-Golay フィルタで平滑化する(波形を保ちつつノイズ低減)。
5. カルマンフィルタで位置・速度・加速度を運動モデル込みで同時推定する(最も強力だが実装が重い)。

1〜3 は軽量で副作用が少なく、4〜5 は精度が上がる代わりにラグや実装コストが増えます。どこまでやるかは下流タスクの感度次第です。

## メリット・トレードオフ・限界

メリット
- データセットに無い量(曲率・ヨーレート)を、記録済みの位置や向きから計算で補える。
- 異なるデータセットを同じ4信号にそろえられ、混在学習が可能になる([カメラ外部パラメータとデータセット非依存設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0939_camera-extrinsics-dataset-agnostic) の運動版)。
- 曲率・加速度はハンドル/アクセルに対応する運動学的に自然な表現で、軌跡が破綻しにくい。

トレードオフと限界
- 有限差分はノイズを増幅する。微分を重ねる加速度は特に悪化する(約100倍)。
- 曲率は低速・停車時に分母の速度が小さく、値が大きく暴れる。下限保護でも完全には防げない。
- 平滑化で対処するとラグ(遅れ)が生じ、反応の速い操作を取りこぼす恐れがある。
- 角度の unwrap や四元数→オイラー角の変換は、実装ミスがあると静かに壊れる(値はそれっぽく出るが間違う)。座標系・回転順序の規約取り違えが典型。

## 発展トピック・研究の最前線

- 確率的状態推定: カルマン/拡張カルマン/粒子フィルタで、ノイズを統計的に扱いながら微分量を推定する。運動モデルを事前知識として組み込めるのが利点。
- 学習ベースの平滑化: ニューラルネットでノイズ除去と微分を同時に行い、固定フィルタのラグを減らす研究。
- 軌跡パラメータ化の選択: waypoint 直接回帰、曲率・加速度系列、ベジエ/スプライン、多項式係数など、どの表現が滑らかさと表現力を両立するかは依然活発な論点です([GAIA-1(生成的世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1214_gaia-1-generative-world-model) のような生成モデルでも行動の量子化方法が品質を左右します)。
- 物理制約の明示化: 出力を運動学モデルに通す、あるいは曲率・加速度に車両限界の制約を課すことで、安全で実行可能な軌跡だけを生成する設計。
