# データ拡張

## ひとことで言うと

データ拡張(data augmentation)とは、手元の学習データに「意味を変えない範囲の変換」を加えてデータを水増しし、モデルが余計な特徴に頼らず本質を学ぶように仕向ける技術です。たとえば猫の写真を左右反転しても猫は猫のままなので、反転した画像を「新しい学習例」として追加できます。これにより、限られたデータでも過学習(overfitting、訓練データだけに過剰に適応して新しいデータで性能が落ちる現象)を抑え、汎化性能(未知のデータに対する性能)を上げられます。

## 直感的な理解

機械学習モデルは、与えられた例を「暗記」してしまう傾向があります。たとえば、犬の写真がたまたま全部「左を向いている」データで学習すると、モデルは「左を向いている=犬」という誤った近道(ショートカット)を覚えてしまうかもしれません。右向きの犬を見せると当てられなくなります。

データ拡張は、この近道を塞ぐ作業です。学習中に犬の写真をランダムに左右反転すれば、「向き」は犬かどうかの判断に使えない情報になります。モデルは仕方なく、耳の形や毛並みといった本質的な特徴を学ばざるを得なくなります。

別の言い方をすると、データ拡張は「人間が持っている事前知識(prior)をデータの形でモデルに教える」手段です。「画像を少し平行移動しても物体のクラスは変わらない」という知識を、平行移動した画像を見せることで暗黙的に伝えています。これを不変性(invariance、入力が多少変化しても出力が変わらない性質)の学習と呼びます。

## 基礎: 前提となる概念

ここでいくつかの用語を噛み砕いておきます。

- **汎化(generalization)**: 学習に使っていない新しいデータでも正しく予測できる能力。機械学習の最終目標はこれです。
- **過学習(overfitting)**: 訓練データに過剰に適応し、訓練データでは高精度だがテストデータで精度が落ちる状態。データが少ないほど起きやすい。
- **正則化(regularization)**: 過学習を防ぐためにモデルへ制約を加える手法の総称。データ拡張は「データ側からの正則化」と位置づけられます。重みに制約をかける weight decay や、ニューロンをランダムに無効化する dropout はモデル側の正則化です。
- **テンソル(tensor)**: 多次元配列。画像は通常 `[C, H, W]`(チャンネル数、縦、横)という3次元テンソルで表され、バッチ処理時は `[B, C, H, W]`(バッチサイズを先頭に追加)になります。たとえば RGB 画像 224×224 を 32 枚まとめると `[32, 3, 224, 224]` です。データ拡張はこのテンソルに変換を加える処理です。

画像分野の代表的な変換を具体例で挙げます。

- 反転(flip): 横軸を反転。`image[:, :, ::-1]` のような操作。
- 切り抜き(crop): ランダムな位置を切り出して拡大。RandomResizedCrop が定番。
- 回転(rotation): 小さな角度(例: ±15度)で回す。
- 色変換(color jitter): 明るさ・コントラスト・彩度・色相をランダムに揺らす。
- 正規化(normalization): 画素値を平均0・分散1付近に揃える。厳密には拡張ではなく前処理ですが同じパイプラインで行います。

## 仕組みを詳しく

データ拡張は通常、学習ループの「データ読み込み(DataLoader)」段階で、画像を1枚読むたびにオンラインで(その場で)ランダム変換を適用します。同じ画像でもエポック(epoch、訓練データ全体を1周すること)ごとに違う変換がかかるため、実質的に無限のバリエーションを生成できます。

数式で見ると、元の画像 x とラベル y のペア (x, y) に対し、変換 T をパラメータ θ(変換の強さや向きをランダムに決める値)でサンプリングし、(T(x, θ), y) を学習に使います。ここで重要なのは「ラベル y が変わらない」変換を選ぶことです。たとえば「6」を180度回転すると「9」に見えるので、数字認識では回転を控えめにする必要があります。これをラベル保存性(label-preserving)と呼びます。

少し進んだ手法として、複数画像を混ぜるものがあります。

- **Mixup**(Zhang et al., ICLR 2018, arXiv:1710.09412): 2枚の画像とラベルを比率 λ で線形混合します。`x_mix = λ·x_a + (1−λ)·x_b`、`y_mix = λ·y_a + (1−λ)·y_b`。λ は通常ベータ分布から引きます。ラベルも混ぜるのが特徴で、モデルの予測を滑らかにし(過剰な自信を抑え)、決定境界を単純化する効果があると報告されています。
- **CutMix**(Yun et al., ICCV 2019, arXiv:1905.04899): 画像 A の一部矩形領域を画像 B の対応領域で置き換え、ラベルは面積比で混合します。Mixup と違い局所的な特徴が保たれます。
- **Cutout**(DeVries & Taylor 2017, arXiv:1708.04552): 画像の一部を正方形で黒く塗りつぶす。物体の一部が隠れても認識できるよう促します。

```
[元画像]      [CutMix]            [Cutout]
+--------+    +--------+          +--------+
| 犬     |    | 犬 |猫■|         | 犬 ███ |
|        |    |    |■■|          |   ███  |
+--------+    +--------+          +--------+
              ラベル=0.7犬+0.3猫   ラベル=犬のまま
```

## 手法の系譜と主要論文

データ拡張の重要性が広く認識されたのは **AlexNet**(Krizhevsky, Sutskever, Hinton, NeurIPS 2012)です。彼らは ImageNet 分類で、224×224 のランダム切り抜き・水平反転に加え、PCA による色のゆらぎ(fancy PCA、画素値の主成分方向に沿ってランダムに明るさを変える手法)を導入しました。論文では、これらの拡張がなければ「相当な過学習に苦しみ、もっと小さなネットワークしか使えなかっただろう」と述べられています。具体的には、ラベル保存変換による拡張でデータセットを実質2048倍に増やしたと記述しています。

その後しばらく、どの変換をどの強さで使うかは研究者の手作業(ヒューリスティック)でした。これを自動化したのが次の系譜です。

- **AutoAugment**(Cubuk et al., CVPR 2019, arXiv:1805.09501): 拡張ポリシー(どの変換をどの確率・強度で組み合わせるか)を強化学習で探索しました。探索空間は「16種類の変換 × 確率 × 強度」を5つの組(sub-policy)にまとめたもので、各データセットごとに最適ポリシーを見つけます。ただし探索に莫大な計算(報告では数千 GPU 時間規模)が必要でした。
- **Fast AutoAugment**(Lim et al., NeurIPS 2019, arXiv:1905.00397): 密度マッチングという考え方で探索を高速化し、ほぼ同等の精度を桁違いに少ない計算で達成しました。
- **RandAugment**(Cubuk et al., CVPRW 2020, arXiv:1909.13719): 探索そのものを廃止しました。「N 種類の変換をランダムに選び、強度 M を全変換で共通の1値にする」という、たった2つのハイパーパラメータ(N と M)だけで制御します。探索コストがほぼゼロになり、しかも AutoAugment と同等以上の精度を出せることを示しました。実用上の標準になっています。
- **TrivialAugment**(Müller & Hutter, ICCV 2021, arXiv:2103.10158): さらに単純化し、変換1つを一様ランダムに選び強度も一様ランダムにするだけ。チューニング不要で RandAugment に匹敵すると報告されました。

自動運転・3D 認識の文脈では、点群(LiDAR で得る点の集合)に対する拡張も発展しました。グローバル回転・スケーリング・並進に加え、**GT-paste / GT-sampling**(SECOND, Yan et al., Sensors 2018)では、他フレームから切り出した物体(車・歩行者)を別シーンに貼り付けて稀なクラスを増やします。BEV(鳥瞰図)ベースの知覚でもこの種の拡張が標準的に使われます。

## 論文の実験結果(定量データ)

数値で効果を見てみます。いずれも報告値で、実装やシードで多少前後します。

**RandAugment**(arXiv:1909.13719)の主な結果:
- ImageNet(1000クラス画像分類)で EfficientNet-B7 を学習し、top-1 精度およそ 85.0% を達成。これは AutoAugment 使用時とほぼ同等で、探索コストを大幅に削減しつつ精度を維持できたことを意味します。
- CIFAR-10(10クラス、5万枚の小画像)では Wide-ResNet-28-10 でテスト誤差 約2.7%。ベースライン(Cutout のみ)の約3.9%前後と比べて改善。誤差2.7%は100枚中およそ2〜3枚しか間違えないという意味です。
- 重要な観察として、データセットやモデルが大きいほど強い拡張(大きい M)が有効、という傾向が示されました。つまり最適な拡張強度はモデル規模に依存します。

**AutoAugment**(arXiv:1805.09501)の主な結果:
- CIFAR-10 で当時の最良誤差 約1.5%。
- ImageNet で top-1 精度を AmoebaNet などで約0.4〜1.3ポイント改善。小さな改善に見えますが、ImageNet の1ポイントは数千万パラメータの工夫に匹敵する価値があるとされます。
- SVHN(街頭の数字)で誤差 約1.0%。

**Mixup**(arXiv:1710.09412)の主な結果:
- ImageNet で ResNet-50 の top-1 誤差を約0.5〜1.5ポイント改善。
- 敵対的攻撃(入力に微小なノイズを加えて誤認識させる攻撃)への頑健性が向上し、ラベルノイズ(誤った正解ラベルが混ざる状況)にも強くなることが示されました。

これらに共通するメッセージは「アーキテクチャを変えずデータ拡張を工夫するだけで、誤差を数十パーセント相対的に減らせる」ことです。たとえば誤差3.9%→2.7%は相対で約30%の削減にあたります。

## メリット・トレードオフ・限界

メリットは明確です。追加のラベル付けコストゼロで汎化性能を上げられ、過学習を抑え、データが少ないドメインで特に効きます。実装も既存ライブラリ(torchvision の transforms、albumentations など)で容易です。

一方でトレードオフと限界もあります。

- **ラベルを壊すリスク**: 強すぎる変換は意味を変えてしまいます。前述の「6→9」問題のほか、医療画像で色変換を強くかけると病変が消える恐れがあります。ドメイン知識が必須です。
- **学習時間の増加**: オンライン拡張は前処理が重くなり、CPU がボトルネックになることがあります。
- **分布の不一致**: 拡張で作る画像が実世界に存在しない不自然なものになると、かえって性能を下げます(RandAugment の強度 M を上げすぎると精度が落ちるのはこの現象)。
- **タスク依存性**: 分類で効く拡張が検出やセグメンテーションで同じに効くとは限りません。物体検出では幾何変換に合わせてバウンディングボックスも変換する必要があり、実装が複雑化します。
- **根本的な限界**: データ拡張は既存データの「見え方」を変えるだけで、本質的に新しい状況(まだ一度も観測していないシーン)を生み出すわけではありません。レアな事象(自動運転での飛び出しなど)はそもそもデータに無ければ拡張でも作れません。ここが後述のシミュレーションやドメインランダム化が補完する領域です。

## 発展トピック・研究の最前線

- **学習可能・適応的な拡張**: 学習の進行に応じて拡張強度を自動調整する手法(Population Based Augmentation, Adversarial AutoAugment など)。
- **生成モデルによる拡張**: GAN や拡散モデル(diffusion model)で新しい学習画像そのものを合成する。実在しないが現実的なバリエーションを作れますが、生成物の質と多様性の担保が課題です。
- **対照学習との関係**: SimCLR(Chen et al., ICML 2020)などの自己教師あり学習では、同じ画像に異なる強い拡張をかけた2つを「同じもの」とみなして学習します。ここではデータ拡張が手法の中核そのものになります。拡張の選び方(特に切り抜きと色変換の組み合わせ)が性能を大きく左右すると報告されています。
- **3D・マルチモーダル拡張**: LiDAR 点群、カメラ画像、それらの整合性を保った同時拡張(BEV 認識での回転・反転をカメラ・点群・ラベルへ一貫適用)は自動運転で活発な研究領域です。
- **テスト時拡張(TTA, test-time augmentation)**: 推論時にも複数の拡張版を入力して予測を平均する。精度は上がりますが計算コストも増えます。
