# 深度分布スプラッティング

## ひとことで言うと

1枚のカメラ画像のピクセルには、本来「それが手前にあるのか奥にあるのか(深度)」が分かりません。そこで「このピクセルは2mかもしれないし、10mかもしれないし、50mかもしれない」という確率の分布を予測し、ありうる各距離に画像特徴を確率の重みで薄く配って3D空間にばらまき(これをスプラット splat = 飛び散らす、と言います)、最後に真上から見て(BEV へ)集めます。深度が分からないなら、ありうる全部に確率付きで置いておこう、という割り切りが核心です。これは「Lift-Splat-Shoot(LSS)」が確立した、カメラから BEV を作る主要系統の一つです。

## 直感的な理解

カメラで撮った写真は、3D世界を2Dに潰したものです。あるピクセルが画像上の (u, v) に写っていても、それがカメラから何メートル離れているか(深度)は1枚の画像だけでは原理的に決まりません。カメラから出る1本の光線(レイ)の上のどこにあってもおかしくないからです。これを「奥行きの曖昧さ」と言います。

BEV 地図(真上から見た地図)を作るには、ピクセルを正しい3D位置に置く必要があり、そのためには深度が要ります。ここで素朴に「深度を1点に決め打ちする」と、外したときに大事故になります。歩行者を10mと決め打ちして実際は5mだったら、地図上で5mずれた場所に歩行者を置いてしまう。

そこで巧妙な答えが「深度を1点に決めず、確率分布として扱う」ことです。確信があるピクセル(エッジがはっきりした近くの車のバンパーなど)は分布が鋭く尖り、確信がないピクセル(のっぺりした遠くの路面など)は分布がなだらかに広がる。特徴も分布に従って複数距離に薄く分散します。多少深度を外しても、正しい距離にもいくらか特徴が置かれているので、致命傷になりにくいのです。

LSS という名前は3段階の頭文字です。Lift(持ち上げ)= 2Dを3Dに、Splat(撒く)= 3D点をBEVに落とす、Shoot(撃つ)= BEV上で経路(自車の走る軌道候補)を予測する。

## 基礎: 前提となる概念

視錐台(view frustum, ビューフラスタム): カメラから前方に向かって扇状(ピラミッド状)に広がる、カメラが見える3D空間の範囲です。LSS では、画像の各ピクセルから出る光線上に深度ビンの数だけ点を置くので、視錐台を点群で埋め尽くす形になります。

離散深度ビン(depth bins): 連続的な深度を「4m, 4.5m, 5m, ..., 60m」のように一定間隔で区切った D 個の代表値です(原典では 4m から 45m を 1m 刻みなど、設定により D は 40〜120 程度)。連続値を予測する代わりに、どのビンの確率が高いかという分類問題に落とすことで、扱いやすく微分可能にできます。

外積(outer product): ここでは「文脈特徴ベクトル c(長さ C)」と「深度分布 α(長さ D)」を掛け合わせて [C, D] の表を作ることを指します。各深度ビン d の特徴が α_d × c になり、確率の重みで特徴がスケールされます。

ピラー集約(pillar pooling): BEV のマス(地面上の正方形)の真上に立つ柱(ピラー)を考え、その柱に落ちたすべての3D点の特徴を合計する操作です。高さ情報を潰して2Dの BEV にまとめる役割を持ちます。

pull と push: カメラ→BEV変換には大きく2つの向きがあります。pull(引っ張る)は「BEVマスから3D点を仮定して画像に投影し、そこの特徴を引っ張ってくる」(IPM や BEVFormer 系)。push(押し出す/splat)は「画像ピクセルを深度分布で3Dに広げ、BEVへ押し出す」(LSS 系)。深度分布スプラッティングは push の代表です。

## 仕組みを詳しく

LSS のパイプラインを段階的に見ます。

ステップ1: Lift(2D → 3D に持ち上げる)。各カメラ画像をバックボーン(特徴抽出ネットワーク)に通し、各ピクセルについて2つを出力します。

- 文脈特徴 c: 長さ C のベクトル(そのピクセルが何を写しているか)
- 深度分布 α: D 個の深度ビンそれぞれに対する確率(softmax で合計1に正規化)

各深度ビン d の特徴を外積で作ります。

```
そのピクセル・深度 d の特徴 = α_d × c     (長さ C のベクトルを α_d 倍)
```

形状で見ると、画像特徴が [C, H, W] のとき、深度ビン D 個に展開して [C, D, H, W] の視錐台点群ができます。各点はカメラ幾何(intrinsic / extrinsic)で計算できる既知の3D座標 (x, y, z) を持ちます。具体的な数で考えると、6カメラ・特徴解像度 16×44・D=59 だと、視錐台の点は 6 × 16 × 44 × 59 ≈ 25万点にもなり、これが計算とメモリの重さの正体です。

数値例: あるピクセルの深度分布が「10m: 0.6、20m: 0.3、その他合計: 0.1」だったとします。文脈特徴 c を 10m の位置に 0.6 倍、20m の位置に 0.3 倍、と置きます。確信が持てない場合は分布がなだらかに広がり、特徴も複数距離に薄く分散します。

ステップ2: Splat(3D点を BEV に撒く)。視錐台の全点を BEV グリッド(真上から見た格子)の対応するマスに落とし、同じマス(ピラー)に落ちた特徴を合計します。LSS は効率化のために累積和(cumulative sum)トリックを使い、点を BEV マス ID でソートして区間ごとに合計することで高速化しました(sum pooling を、累積和の差分で O(N) に落とす)。これにより [C, D, H, W] の膨大な点群を、BEV 特徴 [C, X, Y] に集約します。

ステップ3: 集約 BEV 特徴を畳み込みネットワーク(BEV エンコーダ)で処理し、最終タスク(地図セグメンテーション、3D検出、経路予測など)を解きます。

pull 方式との対比をはっきりさせます。pull(BEVFormer 系)は「BEVマス → 3D点を仮定 → 画像に投影 → 特徴を引っ張る」。深度を仮定で済ませる代わりに地面平坦などの前提や正確なキャリブレーションが要ります。push(LSS)は逆向きで「画像ピクセル → 深度分布で3Dに広げる → BEVに落とす」。深度を学習で当てる代わりに深度推定の質に依存します。

## 手法の系譜と主要論文

Lift-Splat-Shoot(Jonah Philion & Sanja Fidler, ECCV 2020, arXiv:2008.05711, NVIDIA): 原典です。問題意識は「複数カメラを、カメラの台数や配置に依存しない一貫した方法で BEV に統合したい」でした。深度を離散分布として予測し外積で持ち上げるアイデアにより、深度の不確実性を素直に表現でき、画像から BEV まで微分可能・end-to-end で学習できました。深度の正解ラベルなしで、最終タスク(地図セグメンテーション)の損失だけから深度分布が暗黙に学習される点が特徴です。

BEVDet(Huang et al., 2021, arXiv:2112.11790): LSS を3D物体検出に持ち込み、検出ヘッドと組み合わせて実用的な検出器に仕立てました。データ拡張を画像空間と BEV 空間の両方で整合的に行う工夫(BEV-space augmentation。回転・スケール・反転を BEV と画像で矛盾なく適用する)で過学習を抑え、精度を高めました。

BEVDet4D(Huang & Huang, 2022, arXiv:2203.17054): BEVDet を時間方向の2フレームに拡張しました。直前フレームの BEV 特徴を自車運動で位置合わせして連結することで、速度推定を劇的に改善しました(速度誤差 mAVE が概ね 0.9 台から 0.3 台へと大幅改善と報告)。

BEVDepth(Li et al., AAAI 2023, arXiv:2206.10092): LSS の最大の弱点だった深度の不正確さに正面から切り込みました。LiDAR(レーザー測距センサ)から得た正確な深度を教師にして、深度分布を明示的に監督学習(explicit depth supervision)します。なぜそうしたか。著者らは「暗黙学習された深度は実は質が低く、それが BEV 精度のボトルネックだ」と分析で特定したからです。さらにカメラの intrinsic/extrinsic をネットワークに入力する Camera-aware 設計を加えました。

BEVStereo(Li et al., AAAI 2023, arXiv:2209.10248): 単眼の深度推定に加え、時間的に隣り合うフレーム間の視差(temporal stereo)を使って深度をさらに磨きました。単眼だけでは曖昧な深度を、動きによって生じる多視点情報で補強する方向です。

BEVFusion(Liu et al., ICRA 2023, arXiv:2205.13542): LSS でカメラを BEV に持ち上げたうえで、LiDAR の BEV 特徴と同じ BEV 空間で融合する枠組みを示しました。カメラと LiDAR を「共通の BEV 表現」で統合する設計の代表で、splat 系がマルチモーダル融合の基盤として使われる流れを作りました。BEVPool という高速化カーネルで splat のボトルネックも解消しました。

これらに対し BEVFormer(Li et al., ECCV 2022)は深度を一切予測せず、アテンションで投影点から特徴を pull します([学習可能BEVクエリ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0974_learnable-bev-queries))。深度推定の難しさを回避できる代わりに、アテンションの計算コストがかかります。深度分布スプラッティング(push)とアテンション pull は、カメラ→BEV変換の2大系統として並び立っています。

## 論文の実験結果(定量データ)

評価は主に nuScenes(6カメラ・LiDAR を積んだ車で集めた1000シーンの自動運転データセット)で行われます。

指標の意味: mAP(mean Average Precision)は物体を正しく見つけられたかの平均適合率(高いほど良い)。NDS(nuScenes Detection Score)は位置・大きさ・向き・速度・属性の誤差を総合した0〜1の総合点。mIoU(mean Intersection over Union)は地図セグメンテーションで予測領域と正解領域の重なり率(高いほど良い)。

LSS は地図セグメンテーションで、当時の IPM ベースや単純な幾何手法を上回る mIoU を達成し、複数カメラを統一的に BEV 統合できることを示しました。

BEVDepth の効果は明確です。明示的な深度監督を加えることで、nuScenes 検出で mAP・NDS が大きく改善し、報告によりカメラのみ手法で NDS 0.60 前後(test、大型バックボーン)の高水準に達しました。決定的なアブレーションは「深度監督を外すと精度が大きく落ちる」というもので、これは「LSS の暗黙深度は実は精度が低く、正確な深度監督がボトルネック解消の鍵」という主張を定量的に裏付けます。著者らの分析では、暗黙学習された深度の正解との一致率がかなり低い(深度予測がランダムに近い領域がある)ことが示され、深度こそが BEV カメラ検出の本質的な律速段階だと結論づけられました。Camera-aware 設計と深度補正サブネットの追加で、それぞれ数ポイントの mAP 向上が積み上がることもアブレーションで示されています。

BEVDet4D は1フレームから2フレーム(時間情報追加)にすることで、速度誤差 mAVE を大幅に下げ、NDS を数ポイント押し上げました。これは「瞬間の1枚では速度が取れない」ことの裏返しで、時間情報の重要性を示します。

数値の意味: NDS が0.5から0.6に上がるのは、誤検出・位置ずれ・速度誤差が総合的に大きく減ることを意味し、自動運転の安全余裕に直結します。BEVDepth のアブレーションは「splat 方式を使うなら、深度の質を上げる(LiDAR 監督など)ことが最もコスパの高い改善」という実務的な指針を与えてくれます。

## メリット・トレードオフ・限界

メリット

- 深度を1点に決め打ちせず確率分布で扱うので、深度推定が外れても致命的になりにくい。
- カメラの台数・配置に依存しない統一的な BEV 統合ができる。
- 微分可能・end-to-end で、深度の正解ラベルなしでも一応学習できる。
- BEV 空間で LiDAR など他センサと融合しやすく、マルチモーダルの基盤になる。

トレードオフ・限界

- 深度分布が暗黙学習だと精度が頭打ちになりやすい(BEVDepth が LiDAR 監督で改善したのはこのため)。
- 視錐台の全深度ビン × 全ピクセルを扱うので計算とメモリが重い。集約処理(累積和)が GPU 並列化と相性が悪いことも実装上の課題(BEVPool 系の専用カーネルで緩和)。
- 深度ビンの粒度(何mごとに区切るか)や最大距離の設計が結果に大きく効く。
- 遠方では同じ角度誤差が大きな位置誤差になり、深度分解能が粗くなって遠くの物体の位置がぼやける。

研究上の未解決課題: 暗黙深度の質をどう上げるか(LiDAR が無い設定での自己教師深度)、視錐台集約の高速化、深度ビン設計の自動化、そして遠距離・悪天候・夜間でのロバスト性が代表的なオープン問題です。

## 発展トピック・研究の最前線

深度監督の高度化: BEVDepth の流れで、LiDAR 監督に加えて、隣接フレーム間の幾何整合性(BEVStereo のような temporal stereo / structure-from-motion 的な自己教師)で深度を磨く研究が進んでいます。

集約の効率化: 累積和トリックの代わりに、BEVPool / BEVPoolv2 など GPU 効率の良い専用カーネルで splat を高速化する実装最適化が広く使われるようになりました。BEVPoolv2 は事前計算した点-ピラー対応を使い、推論を桁違いに速くしました。

push と pull の融合: 深度分布で粗く撒いたうえでアテンションで磨く、あるいは両系統を相補的に使うハイブリッド手法が登場し、どちらか一方ではなく統合する方向が探られています。

占有・3D再構成への展開: BEV(2D)を超えて高さ方向も持つ3D占有グリッド予測へ、深度分布のアイデアが拡張されています。深度を確率で扱う発想は、ボリュメトリックな3D表現(さらには NeRF 的な体積レンダリング)と自然に接続します。
