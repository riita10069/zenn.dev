# 深度監督損失 (Depth Supervision)

## ひとことで言うと

カメラ画像から「各ピクセルが自分からどれくらい遠いか(深度)」をネットワークに予測させ、その予測を LiDAR(レーザーで距離を測るセンサー)が測った本当の距離と照らし合わせて鍛える、追加の損失(間違いの度合いを表す量)です。これにより、カメラだけで作る「真上から見た地図(BEV)」の位置精度が大きく改善します。推論時には LiDAR を使わずカメラだけで動くので、実車のセンサーコストを増やさずに精度だけ上げられるのが本質的なうまみです。深度監督は単独の研究テーマというより、カメラ式の三次元知覚パイプラインに後付けできる「ほぼ無料の精度改善レバー」として広く採用されています。

## 直感的な理解

カメラは3次元の世界を平らな写真に潰して記録します。写真には色と模様しか写っておらず、「そのピクセルが何メートル先か」という奥行きの情報は明示的には含まれていません。たとえば写真の中の小さな車は、「遠くにある大きいトラック」かもしれないし「近くにあるミニカー」かもしれません。見た目の大きさだけでは距離が一意に決まらない、これを深度の曖昧性(depth ambiguity)と呼びます。数学的に言えば、1枚の画像から3次元への逆投影は「光線」までしか決まらず、その光線上のどこに点があるかは復元できません。これがカメラ単独で三次元を扱うことの根本的な難しさです。

ところが自動運転では、まさにこの「何メートル先にあるか」が命に関わります。前方の車が10メートル先なのか30メートル先なのかで、ブレーキを踏むかどうかが変わるからです。カメラ画像を真上から見た地図(後述の BEV)に並べ直すには、各ピクセルの距離を決めないと地図上の正しい位置に置けません。距離を間違えれば、20メートル先の車を10メートル先に描いてしまい、システムは存在しない衝突危険を誤検知したり、逆に本当の危険を見落としたりします。位置の誤差は速度推定にも波及し、動き予測や計画まで連鎖的に劣化します。

深度監督損失のアイデアは単純です。「人間が見た目から距離を推測するのは難しいが、レーザーで測れば正確な距離が分かる」。だから学習のときだけ LiDAR で正解の距離を測り、それを教師にしてネットワークの深度予測を直接添削します。学習が終われば LiDAR は捨ててよく、ネットワークは「画像の見た目からそれっぽい距離を当てる」能力だけを身につけて卒業します。この「学習時だけ強い教師(privileged information、特権情報)を使い、推論時は安いセンサーで動く」という発想は、ロボティクス全般で繰り返し現れる強力なパターンです。

## 基礎: 前提となる概念

このトピックを理解するには、いくつかの言葉を押さえておく必要があります。

BEV(Bird's-Eye-View、バーズアイビュー)とは、複数のカメラ画像を「自車を中心に真上から見下ろした平面図」に並べ直した表現です。自車の周囲に半径50メートルほどのグリッド(格子)を敷き、その上に他車・歩行者・車線を配置するイメージです。なぜ真上図にするかというと、運転の意思決定(どこを走るか、誰を避けるか)は平面上の幾何で考える方が自然で、複数カメラの情報も1枚の地図に統合できるからです。

カメラパラメータ(camera parameters)とは、カメラの位置・向き(これを外部パラメータ extrinsics と呼ぶ)とレンズの焦点距離・画像中心(内部パラメータ intrinsics と呼ぶ)をまとめたものです。これらが分かっていると、3次元空間の点が画像のどのピクセルに写るか、逆に画像のピクセルがどの方向の光線かを計算できます。焦点距離 f が大きい(望遠)ほど同じ物体が大きく写るので、「見た目の大きさから距離を当てる」には焦点距離の情報が不可欠です。

投影(projection)とは、3次元の点を2次元の画像平面に写す計算です。ピンホールモデルでは、カメラ座標系の点 (X, Y, Z) は u = f·X/Z + c_x, v = f·Y/Z + c_y のように写ります(c は画像中心)。LiDAR の点(x, y, z 座標を持つ)をカメラパラメータで画像に投影すると、「この点は画像の(u, v)ピクセルに写り、距離は Z メートル」という対応が得られます。

LiDAR(ライダー)は、レーザーを照射して反射が戻る時間から距離を測るセンサーです。1回転で数万から数十万個の点(点群、point cloud)を生成し、各点の距離は誤差数センチと非常に正確です。ただし点はまばら(疎、sparse)で、遠方や黒い物体、ガラス、空には点がほとんど当たりません。1枚の画像のピクセル数(数十万〜数百万)に対し、投影された LiDAR 点はせいぜい数千〜2万程度しかないことも珍しくありません。

回帰(regression)は連続値(距離など)を当てる問題、分類(classification)は離散的なカテゴリ(犬か猫か)を当てる問題です。深度は本来連続値ですが、後述するように区間に区切って分類問題として解くことが多いです。離散化する理由は、深度の曖昧性を「1つの確率分布」として表現でき、外積で特徴を3次元に持ち上げる操作と相性が良いからです。

## 仕組みを詳しく

深度監督を Lift-Splat 系のパイプラインに組み込む流れを段階的に分解します。

ステップ1: データ準備。学習データには、カメラ画像と同時刻に撮った LiDAR 点群が含まれます。両センサーは時刻同期(同じ瞬間のデータ)とキャリブレーション(相対位置の正確な計測)が済んでいる必要があります。

ステップ2: 正解深度マップの生成。LiDAR の各点をカメラパラメータで画像平面に投影し、「画像の(u, v)ピクセルの正しい深度は18.3メートル」という対応表を作ります。これを深度マップ(depth map)と呼びます。LiDAR は疎なので、深度マップも多くのピクセルが「値なし(教師なし)」になります。実装上は、特徴マップの解像度(入力画像を 1/16 などに縮小した格子)に合わせて min-pooling で深度をダウンサンプルし、複数点が同じセルに落ちたら最も近い点を採るのが定石です。

ステップ3: ネットワークの深度予測。多くの手法は深度を連続値で回帰せず、区間(bin、ビン)に区切った分類問題として解きます。これは LSS や CaDDN が採用した離散深度(categorical depth)の考え方です。たとえば 2m〜58m を 0.5m 刻みで区切ると約112個の区間になり、各ピクセルについて「どの区間に属するか」の確率分布を出します。区切り方には均等(uniform)・対数的(log)・線形増加(LID, 遠方ほど区間を広くする)などがあり、遠方の精度要求が緩いことを反映して非均等にすることもあります。出力の形(tensor shape)は次のようになります。

```
深度予測出力: [バッチ B, 深度区間数 D=112, 特徴マップ高さ H, 特徴マップ幅 W]
画像特徴:     [バッチ B, 特徴次元 C, 特徴マップ高さ H, 特徴マップ幅 W]
```

ステップ4: Lift(持ち上げ)。各ピクセルの画像特徴ベクトル(C次元)と、そのピクセルの深度分布(D次元の確率)の外積(outer product)を取ります。すると各ピクセルは「C次元の特徴を、D個の距離候補それぞれに確率の重みで分配した」錐(frustum、視錐台)になります。形は [B, C, D, H, W]。これが「2次元の特徴を3次元空間に持ち上げる」操作です。深度分布が鋭い(1つの区間に集中)ほど特徴は1点に集まり、ぼやけているほど光線上に薄く広がります。

ステップ5: Splat(ばらまき)。持ち上げた特徴を BEV グリッドの対応するセルに集約(プーリング)し、BEV 特徴マップ [B, C, X, Y] を得ます。深度分布が正確なら、特徴は正しい BEV セルに落ちます。深度がいい加減だと、車の特徴が間違った場所にばらまかれ、BEV が歪みます。

ステップ6: 深度監督損失。ステップ2の正解深度マップを同じ区間でラベル化(one-hot)し、ステップ3の予測分布との交差エントロピー損失(cross-entropy loss、分類が外れているほど大きくなる損失)を計算します。教師のある(LiDAR 点が当たった)ピクセルだけで損失を取り、教師のないピクセルは無視します。

```
L_depth = - (1/N) Σ_{有効ピクセル} Σ_{d} y_d log( p_d )
```

ここで N は LiDAR 点が当たった有効ピクセル数、y_d は正解区間なら1そうでなければ0、p_d はネットワークが区間 d に与えた確率です。Binary cross-entropy を区間ごとに取る実装もあり、これは「深度がこの区間以上か」を各区間で判定する形に相当します。

ステップ7: 合計損失。この深度損失を、本来の3D検出損失などと足し合わせて学習します。重みは典型的に L = L_det + λ·L_depth(λ は数程度)で、深度損失が支配的にならないよう調整します。

全体像の ASCII 図:

```
カメラ画像 ──┬─→ [画像特徴抽出] ──→ 画像特徴 [B,C,H,W]
             │                                  │
             └─→ [深度予測ヘッド] ─→ 深度分布 [B,D,H,W] ─┐(外積=Lift)
                                       │                  ▼
LiDAR点群 ─→ 画像へ投影 ─→ 正解深度マップ │            視錐台特徴 [B,C,D,H,W]
                              │           │                  │(Splat)
                              ▼           ▼                  ▼
                     深度監督損失(CE) ←─ 予測深度        BEV特徴 [B,C,X,Y] ─→ 3D検出
```

数値例で効果のイメージをつかみます。深度監督なしだと、ある車のピクセルの予測深度が「25m」で実測が「18m」のように7メートルもずれることが報告されています。このずれは BEV 上で車を7メートル違う位置に描くことに相当し、検出精度を直接損ないます。深度監督を入れると、このずれが大きく縮みます。

なぜ「明示的な監督」がそれほど効くのかという理由も重要です。LSS の元論文では深度分布は検出損失からの間接的な勾配だけで学習され、深度を直接教える信号がありませんでした。そのため深度分布は「BEV 検出さえ当たればよい」という緩い制約しか受けず、実際の距離とは乖離した「都合のよい」分布に収束しがちでした。極端には、深度が完全に間違っていても、たまたま BEV 上で正しいセルに特徴が落ちれば検出は当たってしまう。BEVDepth はこれを定量的に測り、暗黙学習の深度予測が実測と大きくずれていること(報告では正解との一致がランダムに近い水準まで悪化する場合もある)を示し、明示的な監督がボトルネックを外すと主張しました。

Camera-aware の工夫も重要です。同じ「見た目の大きさ」でも、焦点距離の違うカメラでは対応する距離が変わります。BEVDepth はカメラの内部・外部パラメータを MLP で埋め込み、画像特徴をスケール変調(SE モジュール的に乗算)してから深度を推定します。これにより、焦点距離や取り付け角度の異なる複数カメラ・複数車両をまたいだ汎化が安定します。

## 手法の系譜と主要論文

深度を3次元理解に使う流れは、おおむね次のように発展してきました。

LSS — Lift-Splat-Shoot(Philion & Fidler, ECCV 2020, arXiv:2008.05711)。深度監督そのものは提案していませんが、「ピクセルごとに離散深度分布を予測し、外積で特徴を3次元に持ち上げ、BEV にばらまく」という枠組みを確立しました。後続のカメラ BEV 手法の共通基盤です。弱点は、深度分布が当てずっぽうになりやすいことでした。

CaDDN — Categorical Depth Distribution Network(Reading, Harakeh, Chae, Waslander, CVPR 2021, arXiv:2103.01100)。単眼3D検出で「離散深度分布を LiDAR で監督する」考え方を明確に打ち出しました。深度を区間分類とし、LiDAR から作った正解で交差エントロピー監督する設計は、後の BEV 手法の深度監督の直接の先祖です。区間幅を遠方ほど広げる LID(Linear-Increasing Discretization)もここで提案されました。

BEVDet / BEVDet4D(Huang, Huang, Zhu, Du, Du, 2021–2022)。LSS 系の多視点カメラ3D検出を実用的な精度・速度まで引き上げた一連の研究です。BEVDet4D は時系列(複数フレーム)を取り込み、速度推定を改善しました。これらが BEVDepth の直接の比較対象になりました。

BEVDepth(Li, Ge, Wang, Yan, Sun, Cai, Sun, AAAI 2023, arXiv:2206.10092)。本トピックの中心です。三つの貢献があります。第一に、既存の暗黙的な深度学習が実測と乖離している問題を定量分析しました。第二に、LiDAR を教師にした明示的な深度監督(Explicit Depth Supervision)を導入しました。第三に、カメラ内部・外部パラメータを深度ネットに入力する Camera-aware Depth Estimation と、効率的な BEV プーリング(Voxel Pooling の高速化)を組み合わせ、推論を速く保ったまま精度を上げました。

BEVStereo / SOLOFusion(2022–2023)。BEVDepth の後、単一画像の深度推定の曖昧性を、複数時刻のフレーム間の視差(ステレオ的手がかり)で補う方向に発展しました。SOLOFusion(Park et al. ICLR 2023)は「長期の低解像度時間ステレオ + 短期の高解像度」を組み合わせ、時間方向のステレオで深度をさらに正確にしました。BEVStereo は動的に時間ステレオの信頼度を見積もり、動く物体で破綻しないようにしました。

なお BEVFormer(Li et al. ECCV 2022)は深度を明示的に予測せず、Transformer の注意機構で BEV を直接組み立てる別系統ですが、後続研究では BEVFormer 系にも深度監督を足すと精度が上がることが報告されています。深度監督は特定アーキテクチャに限らず横断的に効く補助信号だといえます。

## 論文の実験結果(定量データ)

評価はほぼ nuScenes(ヌーシーンズ)という自動運転の標準ベンチマークで行われます。主要指標を初心者向けに説明します。

mAP(mean Average Precision、平均適合率)は、検出した3Dボックスがどれだけ正しく当たっているかの総合スコアで、0〜1(または%)で表し、高いほど良い指標です。nuScenes では中心位置の距離しきい値(0.5m, 1m, 2m, 4m)で当たり判定を取り平均します(IoU ではなく中心距離を使うのが nuScenes の特徴)。

NDS(nuScenes Detection Score)は nuScenes 独自の総合指標で、mAP に加えて位置(mATE)・サイズ(mASE)・向き(mAOE)・速度(mAVE)・属性(mAAE)の誤差をまとめたものです。0〜1で高いほど良い。研究者はこの NDS でランキングを比較します。

BEVDepth の報告によると、深度監督と Camera-aware を加えることで、同等設定のベースライン(BEVDet 系の暗黙的深度)に対し mAP・NDS がともに数ポイント改善しました。論文では nuScenes test セットで NDS がおよそ 0.60(60.0%)、mAP がおよそ 0.50 前後に達し、当時のカメラのみ手法で最高水準を記録したと報告されています(具体値は入力解像度・バックボーン・時系列フレーム数で変動します)。速度誤差 mAVE も時系列利用と相まって大きく下がりました。数ポイントの NDS 改善は、このベンチマークでは順位が大きく入れ替わる重みを持つ差で、研究コミュニティでは無視できない改善とみなされます。

アブレーション(ある要素を抜くとどうなるかの検証)も示唆的です。BEVDepth は、明示的深度監督を外すと予測深度と LiDAR の一致度が大きく落ち(報告ではしきい値内一致率が顕著に低下)、それに連動して検出 mAP が数ポイント下がることを示しました。Camera-aware の工夫(カメラパラメータ入力)を外すと、解像度やレンズの違うカメラをまたいだ汎化が悪くなります。つまり「深度監督」と「カメラ認識」は独立に効く別々の貢献だと整理できます。CaDDN でも、離散深度分布を LiDAR で監督する成分を抜くと KITTI の単眼検出 AP が落ちることが報告されており、深度監督の寄与は複数論文で一貫して確認されています。SOLOFusion の報告では、LiDAR 深度監督に加えて長期時間ステレオを入れると、深度監督単独よりさらに NDS が伸び、特に速度・遠方精度が改善することが示されました。

## メリット・トレードオフ・限界

メリット。BEV への投影精度が上がり、他車・歩行者の位置がより正確になります。推論時はカメラのみで動くため、実車のセンサーコストが増えません(LiDAR は学習時だけ使う「教師の足場」)。既存の LSS 系パイプラインに損失項を一つ足すだけで導入でき、改修が小さいのも実務上の利点です。深度分布の学習が安定すると BEV 特徴の質が上がり、その上の検出・追跡・予測すべてが恩恵を受けます。

トレードオフと限界。第一に、LiDAR とカメラが同期・較正された学習データが必須で、データ収集とキャリブレーションに手間がかかります。第二に、LiDAR の点は疎で、空・遠方・黒い物体・反射しにくい面には点が当たらず、そこは監督できません。深度監督は「点が当たった近距離・中距離」に偏った教師になりがちで、最も曖昧で重要な遠方の深度はむしろ手薄になるという皮肉があります。第三に、深度区間を細かくするほど分類クラス数 D が増え、視錐台特徴 [B,C,D,H,W] のメモリと計算が増えます。区間幅と精度はトレードオフで、ここが BEV プーリング高速化の研究動機になりました。

未解決の課題として、単眼深度の本質的曖昧性は監督だけでは消えないこと(同じ見た目でサイズの違う物体は原理的に区別できない)、LiDAR 教師のドメイン依存(あるセンサー構成で学んだ深度が別構成に移らない)、動的物体での時刻ずれ(LiDAR とカメラのわずかな時間差で動く車の深度がずれ、誤った教師になる)などが挙げられます。これらに対し、時間ステレオや自己教師あり深度(動画の連続フレームから幾何的に深度を学ぶ)を併用する研究が進んでいます。

## 発展トピック・研究の最前線

時間ステレオ(SOLOFusion, BEVStereo)は、連続フレーム間の視差から深度を補い、LiDAR 教師への依存を減らす方向です。自己教師あり深度(monodepth 系、Godard et al. の Monodepth2 など)は LiDAR を一切使わず動画の測光整合(同じ点が連続フレームで同じ色に見えるはず)から深度を学ぶ流派で、深度監督と組み合わせる試みもあります。占有予測(occupancy prediction、Occ3D / SurroundOcc など)は、深度をボクセル占有として一般化し、定義済みカテゴリにない障害物(落下物・はみ出した荷台など)も表現する近年の潮流で、深度監督の発展形と位置づけられます。さらに、LiDAR を持つ車から持たない車へ深度知識を渡すクロスモーダル蒸留(distillation)、レーダーの疎な深度を補助教師に使う研究、深度の不確実性まで出力して下流の計画に安全側で伝える研究なども現れています。
