# SDTagNet SD地図エンコーダ

## ひとことで言うと
OpenStreetMap(OSM)のような、おおまかな道路地図(SD地図)に書かれた「道路の名前」「種類」「制限速度」などのテキスト注釈を、自然言語処理(NLP)の技術でトークン(言語の最小単位)に変換し、ニューラルネットが扱えるベクトルにする地図エンコーダです。多様で非定型な地図注釈を統一的にトークン化することで、運転モデルの視覚特徴(BEV)と cross-attention(相互注意)で結びつけられるようにします。狙いは、高精細地図を整備できない地域でも、無料で全世界にある粗い地図を「事前情報」として運転モデルに与えることです。

## 直感的な理解
人間の運転手は、初めての道でもカーナビの粗い地図を見て「この先は右に分岐」「ここは制限速度40の生活道路」と先読みします。カメラだけでは見通せない遠方の道のつながりや、標識を見落としたときの制限速度を、地図が補ってくれるからです。運転モデルにも同じことをさせたい。

理想は車線をセンチ単位で記録した高精細地図ですが、これは整備コストが莫大で、世界中をカバーするのは非現実的です。一方、OpenStreetMap のような粗い地図は無料で全世界にあります。そこで「安価で広く手に入る粗い地図を、運転モデルの追加入力にできないか」が出発点です。

ここで難所があります。粗い地図の情報は「タグ」という自由なテキスト注釈の集まりで、種類が膨大かつ非定型です。これを固定の数値カテゴリに無理やり潰すと情報が落ちます。SDTagNet のアイデアは「タグをテキストとして扱い、言語モデルで埋め込む」ことです。言語モデルは未知の単語や多様な表現を意味空間に柔軟に配置できるので、初めて見るタグや珍しい注釈も扱えます。

## 基礎: 前提となる概念
地図の精度には大きく2種類あります。

HD地図(High-Definition map、高精細地図)は、車線の境界線・中心線・停止線・横断歩道などをセンチ単位で記録した詳細な地図です。Lanelet2 や OpenDRIVE がその代表フォーマットです。整備コストが非常に高いのが弱点です。

SD地図(Standard-Definition map、標準精度地図)は、カーナビ程度の粗い地図です。道路が線でつながっている程度の情報を持ちます。OpenStreetMap(OSM、誰でも編集できる世界規模の地図)が代表例で、無料で全世界をカバーします。

「タグ(tag)」は OSM のデータモデルの基本単位で、`キー=値` の組です。たとえばある道に `highway=residential`(住宅街の道)、`maxspeed=30`(制限速度30)、`name=本町通り`(固有名)、`oneway=yes`(一方通行)が付きます。数値・カテゴリ・固有名詞が混在し、種類は実質無限に増えうる非定型なテキストです。

「トークン化(tokenization)」は、テキストを言語モデルが扱える最小単位(トークン)に区切る処理です。「埋め込み(embedding)」は、各トークンを意味を表す数値ベクトルに変換することです。BERT(Devlin et al., NAACL 2019, arXiv:1810.04805)のような言語モデルは、文脈を踏まえて単語を意味空間のベクトルにします。SDTagNet はこの NLP の道具立てを地図タグに応用します。

「cross-attention(相互注意)」は、片方の集合(たとえば BEV のセル)が、もう片方の集合(地図トークン)の中から関連する情報を選んで取り込む仕組みです。各 BEV セルが「自分に関係する地図要素はどれか」を重み付きで選び、その特徴を集めます。

## 仕組みを詳しく
SD地図をベクトル分岐として処理する典型的なパイプラインは、おおむね次の4段で構成されます。

第1段、地図の取得と自車中心の切り出し。OSM の生データ(XML)を読み、自車(ego)を中心とした周辺パッチを切り出します。緯度経度(WGS84、地球を緯度経度で表す標準座標系)を、自車基準のローカル座標(前方を X、左を Y、上を Z とするような直交座標)に変換します。データ取得には Overpass API(OSM のデータをクエリで取り出すサービス)を使い、走行軌跡を囲むバウンディングボックス分のデータを取って、ローカルにキャッシュします。ここで重要な実務上の注意は、固定半径で切り出すと長いエピソード(走行区間)で地図が途中で欠けることがあるため、走行軌跡全体を囲む矩形にマージンを足した範囲を取る、という設計です。

第2段、ポリラインへのリサンプリングとタグのトークン化。道(way、点の連なりで道路を表す OSM の要素)を固定間隔の点列にリサンプリングし、各要素のタグをトークン化します。出力は要素ごとに長さが違う「ragged(不揃いの)」なデータ構造になります。

第3段、エンコーダ本体。地図要素を点単位でトークン化(point-level tokens)し、各タグのテキストを言語モデルで埋め込みます(NLP tag embeddings)。SDTagNet の特徴は、このタグエンコーダを OSM タグで事前学習しておく点です。さらに、地図要素間のつながり(グラフ構造)を固定順の識別子で表し、点の連続的な位置をサイン関数による位置エンコーディング(sine positional encoding、位置をサイン・コサインの組で表す手法。Transformer で標準的)で表します。出力は `(トークン列 [B, N, C], パディングマスク [B, N])` の形です。`B` はバッチサイズ、`N` はトークン数(サンプルごとに可変)、`C` は埋め込み次元、パディングマスクは長さをそろえるための埋め草の位置を示します。

第4段、BEV 特徴との融合。BEV グリッドの各セルを「クエリ」、地図トークンを「キー/バリュー」として cross-attention させます。メモリ効率の良い attention 実装(scaled dot-product attention の最適化版)を使えば、大きなフルグリッドでも巨大な QK 行列を一度に作らずに済みます。地図が存在しないサンプル用には学習可能な null トークンを置いて数値が発散(NaN)するのを防ぎ、ゼロ初期化のチャンネルごとゲート([ゼロ初期化残差によるマップ融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0464_umpe-residual-map-fusion))を併用して、学習開始時は「地図なし」と完全に同一の挙動から始め、徐々に地図情報を効かせます。

形状の流れのイメージです。

```
BEV セル   [B, H*W, C]   (query)
地図トークン [B, N, C]    (key, value) + mask [B, N]
   ── cross-attention ──>  地図情報で強化された BEV [B, H*W, C]
```

なぜ「タグをカテゴリ番号でなく言語埋め込みにするのか」という設計理由は、タグの語彙が開いている(未知タグが常に出る)からです。カテゴリ番号方式だと、学習時に見なかったタグは「未知」の1カテゴリに丸められて情報が落ちます。言語埋め込みなら、未知タグも構成単語の意味から近い位置に配置でき、ゼロショット的に扱えます。

## 手法の系譜と主要論文
SD地図やナビ地図を運転に使う流れには明確な系譜があります。

ベクトル地図エンコーダの源流は VectorNet(Gao et al., CVPR 2020, arXiv:2005.04259)です。地図や物体を「ポリライン(点列)の集合」として表し、グラフニューラルネットで符号化します。ラスタ画像にせずベクトルのまま扱うことで効率と精度を両立しました。地図を点列のトークンとして扱う、という発想の土台です。

SD地図を運転の事前情報に使う研究としては、SMERF(Luo et al., 2024, "Augmenting Lane Perception and Topology Understanding with Standard Definition Navigation Maps")が、SD地図を Transformer で符号化して車線知覚とトポロジー理解を強化しました。TopoNet(Li et al., 2023, arXiv:2304.05277)は車線間のトポロジー(つながり方)をグラフで推論する枠組みで、地図的な構造理解を扱います。P-MapNet(Jiang et al., 2024, arXiv:2403.10521)は SD地図と HD地図の両方を事前情報として使い、オンライン地図構築の遠方精度を改善しました。

これらの多くは、タグや道路種別を離散カテゴリとして扱います。SDTagNet(Immel et al., 2025)の差分は、それを言語埋め込みに置き換えた点です。OSM タグを言語モデルで埋め込むことで、定型カテゴリに収まらない多様な注釈や未知タグを柔軟に表現します。動機は系譜共通(安価で広範な地図事前情報の活用)で、地図品質への依存というトレードオフも共通です。新規性は「地図注釈を自然言語として扱い、NLP の表現力を地図エンコーダに持ち込んだ」ことにあります。

言語モデル側の系譜としては、BERT(Devlin et al., NAACL 2019)に代表される事前学習済み言語モデルがあり、タグの語彙的・意味的構造を捉えるための基盤技術になっています。

## 論文の実験結果(定量データ)
SD地図を事前情報に使う研究群は、地図なしのベースラインに対して明確な改善を報告しています。指標の意味を添えながら見ます。

オンライン地図構築や車線知覚の評価では、mAP(mean Average Precision、検出・分割の平均精度。高いほど良い)が主指標です。SD地図を事前情報として加える手法は、地図なし手法に対してこの mAP を有意に押し上げます。たとえば P-MapNet は、nuScenes でのオンライン HD地図構築において、SD地図事前情報を加えることで遠方領域(自車から遠いほど推定が難しい)の mAP を、報告によりおよそ十数ポイント規模で改善したとされます。遠方の改善が大きいことが重要で、これはまさにカメラだけでは見通せない範囲を地図が補うという狙い通りの効果です。

SMERF も、SD地図を組み込むことで車線検出とトポロジー推論の精度が向上することを報告しています。トポロジー推論の指標(車線のつながり方を正しく当てられたかを測る指標)が上がるのは、SD地図が「道がどうつながるか」という構造の手がかりを与えるからです。

SDTagNet については、OSM タグを言語埋め込みで扱う方式が、タグを離散カテゴリで扱う方式に比べて、同等のタスクでより高い精度を出すこと、とりわけ学習時に見なかったタグや希少なタグを含む状況で頑健であることが、著者らによって報告されています。アブレーションの観点では、(1) NLP タグ埋め込みを単純なカテゴリ埋め込みに置き換えると性能が下がること、(2) タグ情報そのものを落として道路の幾何だけにすると、制限速度や道路種別に依存する判断で性能が落ちること、が「どの要素が効いているか」を示す根拠になります。

これらの数値が意味するのは次のことです。mAP やトポロジー指標が上がるとは「道路要素を正しい場所に・正しいつながりで認識できる」ということで、ひいては運転計画の安全余裕につながります。SD地図は無料という点を踏まえると、わずかな追加計算でこれだけの改善が得られるのは費用対効果が高い、と評価できます。

## メリット・トレードオフ・限界
メリットは次の通りです。全世界で無料・広範に手に入る SD地図を運転モデルの事前情報として使えます。タグを言語埋め込みで表すので、数値・カテゴリ・固有名詞が混じる多様な注釈や未知タグも柔軟に扱えます。cross-attention で BEV 特徴と直接結びつけられ、ゼロ初期化ゲートにより「地図なし」から安全に学習を始められます。メモリ効率の良い attention 実装で、大きな BEV グリッドでも現実的に動きます。

トレードオフと限界もあります。最大の弱点は地図品質への依存です。OSM の精度・鮮度・カバレッジに性能が左右され、地図が古い・間違っている・欠けている地域では効果が出ない、あるいは誤った事前情報として害になりえます。学習済み言語タグエンコーダのダウンロードとロード、推論コストが追加されます。Overpass API での地図取得とキャッシュ管理という、運転モデル本体外のデータ取得パイプラインが必要になります。さらに、SD地図の位置精度は HD地図に劣るため、地図 BEV と画像 BEV の座標整合([地図BEVと画像BEVの座標整合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0430_map-image-bev-alignment))が緩く、融合の幾何的妥当性が HD地図ほど保証されないという根本的な限界もあります。研究上は、地図の信頼度を推定して融合の効き具合を動的に変える条件付き融合や、地図の鮮度・誤りに頑健な学習が、未解決の方向として残されています。

## 発展トピック・研究の最前線
SD地図と HD地図を組み合わせ、HD地図がある領域では精密に、ない領域では SD地図で粗く補う、という階層的な地図事前情報の研究が進んでいます。地図注釈を言語として扱う発想は、大規模言語モデル(LLM)を運転の意味理解に使う流れとも接続し、「制限速度の標識を見落としても地図のテキストから制限速度を推論する」といった、視覚と言語をまたぐ推論へ拡張されつつあります。地図の誤り検出(画像と地図が食い違う箇所を見つける)や、走行ログから SD地図を自動更新する研究も、地図品質依存という弱点への正面からの対策として活発です。
