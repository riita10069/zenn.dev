# 単輪車モデルの制御信号 (加速度・曲率)

## ひとことで言うと
車の進路を「地図上の (x, y) 座標の点列」で表すのではなく、「各時刻でどれだけアクセル/ブレーキを踏むか(加速度)」「どれだけハンドルを切るか(曲率)」という運転操作の列で表す、という軌道表現の流儀です。あとから運動学の式で積分すれば座標に戻せます。出力が「実際に走れる動き」になることが最大の利点です。

## 直感的な理解
地図に「次にここ、その次にここ」と点を打って車を動かすことを想像してください。点を自由に打てると、隣の点までの距離がバラバラだったり、いきなり横に飛んだりする「物理的にありえない経路」も簡単に作れてしまいます。人間が運転するときはそんな打ち方をしません。代わりに「ここでちょっとアクセル、ここでハンドルをこれくらい切る」という操作を連続して行い、その結果として車が進む経路が決まります。

制御信号表現はこの後者の発想です。ニューラルネットに出させるのは操作(加速度と曲率)だけで、経路は車の運動の式に従って自動的に決まります。すると、どんな操作を出しても結果は必ず「ある加速度・ある曲率で走った軌跡」になり、瞬間移動やギザギザの非物理的な経路が原理的に出てこなくなります。難しい制約を後から押し付けるのではなく、表現そのものに物理を組み込んでしまうわけです。専門的には、これを「アーキテクチャ的事前知識(architectural prior)」や「帰納バイアス(inductive bias、モデルが学びやすい方向に最初から偏らせておく仕掛け)」と呼びます。

## 基礎: 前提となる概念

軌道(これから車がたどる経路)を数値で表す方法は大きく 2 通りあります。

1. 生 Cartesian(直交座標)表現: 0.1 秒ごとの車の位置 (x, y) を並べる。例 `(0,0), (1.0, 0.02), (2.0, 0.05), ...`。直感的で、地図と照合しやすい。
2. 制御信号表現: 0.1 秒ごとの (加速度, 曲率) を並べる。物理的に実現可能な動きしか表せない。

用語を平易に説明します。
- 加速度(acceleration): 速度の変化率。プラスなら加速(アクセル)、マイナスなら減速(ブレーキ)。単位はおおむね m/s^2(メートル毎秒毎秒)。
- 曲率(curvature): 進路の曲がり具合。直進なら 0、きついカーブほど大きい。半径 R の円弧を走るなら曲率は 1/R(単位 1/m)。ハンドルの切り角に対応します。曲率が大きいほどきつく曲がります。
- ヨー角(yaw, theta): 車が向いている方向の角度。真上から見たときの「鼻先の向き」。
- ヨーレート(yaw rate): ヨー角の変化率(向きが回る速さ)。単輪車モデルでは `速度 × 曲率` に等しくなります。
- 積分(integration): 変化率(速度・加速度など)を時間にわたって足し合わせ、量(位置・速度など)を求める操作。離散的にはオイラー積分(微小時間ぶん足し算で前に進める最も単純な数値積分)が使われます。
- 運動学(kinematics)と動力学(dynamics)の違い: 運動学は「力を考えず、幾何と速度だけで動きを記述する」モデル。動力学は「タイヤにかかる力やスリップまで考える」モデル。運動学は軽くて通常走行に十分、動力学は重いが限界域で正確、という棲み分けです。

生 (x, y) 表現の弱点は、ニューラルネットが各点を自由に予測すると、隣り合う点の物理的関係を守る保証がないことです。瞬間ワープや急なギザギザを平気で出します。出力後にローパスフィルタ(高周波の暴れを削る後処理)で平滑化することはできますが、それは後付けの絆創膏で、「車が実際に出せる動きか」を根本では保証しません。そこで、車の運動を単純化した運動学モデル(動き方の数式)を通して表すと、出力が必ず実現可能な動きになります。その最も単純なものが単輪車モデル(unicycle model)です。

## 仕組みを詳しく

### 単輪車モデルとは

単輪車モデルは、車を「向きを持った 1 つの点」とみなすモデルです。状態として位置 (x, y)、ヨー角 theta、速度 v を持ちます。入力(制御)は (加速度 a, 曲率 kappa) の 2 つです。時間刻みを dt として、状態はオイラー積分で次のように更新されます。

```
v_{t+1}     = v_t + a_t * dt                  # 加速度を積分して速度
theta_{t+1} = theta_t + v_t * kappa_t * dt    # 曲率×速度で向きが回る(ヨーレート)
x_{t+1}     = x_t + v_t * cos(theta_t) * dt   # 速度と向きで位置が進む
y_{t+1}     = y_t + v_t * sin(theta_t) * dt
```

ポイントは、ニューラルネットが直接出すのは (a, kappa) の列だけ、ということです。(x, y) は上の式で積分して初めて得られます。この構造のおかげで、出力がどんな値でも「ある加速度・ある曲率で走った結果の軌跡」になり、テレポートのような非物理的な軌道が原理的に出てきません。微小な数値例で動きを見ると、向きの回転 `v*kappa*dt` は速度に比例するので、止まっている車(v=0)はいくら曲率を出しても回頭しない、という実車の挙動(その場で向きだけ変えられない)も自然に再現されます。

なお単輪車モデルは、もう少し精密な運動学的自転車モデル(kinematic bicycle model)の単純版とみなせます。自転車モデルは前輪と後輪を 1 軸ずつにまとめ、ホイールベース(前後輪間の距離)L と前輪操舵角 delta から曲率を `kappa = tan(delta)/L` のように求めます。単輪車モデルはこの操舵角と曲率の関係を曲率そのもので置き換えた、いちばん軽いモデルです。曲率を直接出力にすると、ホイールベース L という車種依存の定数に縛られないという副次的な利点もあります。

### 軌道テンソルの具体形

学習で扱う軌道テンソルは `[B, num_timesteps * num_signals]` の形が一般的です。

- `num_timesteps`(例 64): 時間ステップ数。0.1 秒刻みなら 64 ステップで 6.4 秒先まで。
- `num_signals`(例 2): 各ステップの信号数。ここでは (acceleration, curvature) の 2 つ。
- よって 1 サンプルの軌道は `64 * 2 = 128` 次元のベクトル。バッチなら `[B, 128]`。

各信号チャンネルの意味を契約で固定しておくことが重要です(出力契約については [プランナーの出力契約 (Planner Output Interface)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0921_base-planner-contract) を参照)。「各チャンネルは制御信号(加速度・曲率)であって、生 (x, y) ウェイポイントではない」と明示することで、後段の処理がチャンネルを取り違えません。たとえばチャンネル 0 と 1 を入れ替えても shape は `[B,128]` のままなので、契約で固定しておかないと「加速度のつもりで曲率を積分する」事故が静かに起きます。

### 座標への復元(可視化・採点で必要)

実際に「どこを走るか」を絵にしたり、地図のドライブ可能領域(走ってよい場所)と照合するには (x, y) に戻す必要があります。手順は前述の積分式そのままです。

1. 加速度を積分して速度とヨーを得る。
2. 速度を積分して位置を得る。
3. 位置をピクセル座標などに変換する。
4. 地図タイル上に軌道を描く、あるいは安全制約と照合する。

数値例(イメージ): 初速 5 m/s、dt=0.1 s、ある時刻に a=1.0、kappa=0.1 とします。次ステップで v=5.1、theta は `5*0.1*0.1 = 0.05` rad ぶん回り、x は約 `5*cos(0)*0.1 = 0.5` m 前進、という具合に積み上がっていきます。こうして得た (x, y) 列を使って、「許容加速度・許容曲率に収まっているか(乗り心地)」「ドライブ可能領域に入っているか(安全)」を採点できます。制御信号表現なら、加速度・曲率の上限を直接チェックするだけで乗り心地制約が課せるのも利点です。

重要なのは、この積分そのものを微分可能(differentiable)な層としてニューラルネットの中に置けることです。`cos`・`sin`・乗算・累積和はすべて微分可能なので、座標空間で測った損失(地図からのはみ出し量、衝突リスクなど)の勾配を、制御信号を出力したネットワークの重みまで逆伝播できます。これにより「物理的に妥当な操作を出す」ことと「座標空間で正しい場所を走る」ことを 1 つの学習目標で同時に最適化できます。

## 手法の系譜と主要論文

- 運動学的自転車モデル・単輪車モデルは古典的な車両運動学で、Rajamani の教科書「Vehicle Dynamics and Control」(Springer, 2012)などで標準的に扱われます。提案というより業界標準で、根拠は「低速〜中速ではこの単純モデルが実車挙動をよく近似し、計算が軽い」こと。
- Polack et al. (2017) "The kinematic bicycle model: A consistent model for planning feasible trajectories for autonomous vehicles" (IEEE Intelligent Vehicles Symposium) は、運動学的自転車モデルがどの速度・横加速度の範囲で計画用モデルとして妥当かを定量的に検討しました。横加速度がおよそ 0.5g(重力加速度の半分、約 4.9 m/s^2)を超えるとタイヤのスリップが効いて運動学モデルの誤差が無視できなくなる、という実用的な目安を与えています。
- 制御信号(曲率など)で軌道を表すプランニングは、古典的な自動運転スタックの軌道計画でも広く使われます。曲率の連続性を保つことが乗り心地と物理的実現性に直結するためです。
- 学習系の end-to-end 運転では、生ウェイポイント回帰よりも運動学的に整合した出力(加速度・操舵・曲率)を学ぶ流れが強まっています。UniAD (Hu et al., 2023, CVPR, arXiv:2212.10156) のような計画指向の統合モデルや、VAD (Chen et al., 2024, ICCV) のようなベクトル化シーン表現で軌道を出すモデルでは、最終的に物理量へ展開して安全・快適性を評価します。制御信号表現はこの「物理量での採点」を素直に可能にします。

## 論文の実験結果(定量データ)

- Polack et al. (2017) は、運動学的自転車モデルと、より精密な動力学(ダイナミクス)モデルで生成した軌道を比較し、横加速度が約 0.5g 以下の通常走行域では両者の位置誤差が小さく(数十センチ程度)、運動学モデルで計画した軌道がそのまま追従可能であることを示しました。一方、限界域(急ハンドル・高速)では誤差が急増します。これは「制御信号表現+運動学積分は通常走行では実車とよく合うが、限界域では破綻しうる」という、表現の適用範囲を定量的に裏付けます。指標は計画軌道と実車軌道の位置ずれ(meter 単位)で、これが小さいほど計画どおり走れることを意味します。
- 計画指向の統合モデル UniAD (Hu et al., 2023) は nuScenes ベンチマークで、計画の平均 L2 誤差(計画軌道と人間の運転軌道の距離、小さいほど良い)を従来手法より低減し(報告では 3 秒先で約 1.65 m から 1.03 m 前後へ)、衝突率(collision rate、計画軌道が他物体と衝突する割合)も大きく下げたと報告しています。物理整合的な軌道表現と多タスク統合が、計画の正確さと安全性の両方に効くことを示す例です。
- VAD (Chen et al., 2024) は、シーンをベクトル(地図要素・エージェント軌跡)で表しつつ自車軌道を出す構成で、UniAD と同等以上の計画精度を、ラスタ表現を使わないぶん大幅に高速(推論で数倍のスループット)に達成したと報告しています。軌道を物理整合的に扱う方針は維持しつつ表現を軽くした例として位置づけられます。
- 軌道を生成して採点する手法では、採点段で軌道を物理量(加速度・曲率)に展開し、運動学的快適性(comfort)と走行領域コンプライアンスを評価することで、総合スコア(運転品質を集約した指標、たとえば PDMS 系の指標)が向上することが報告されています。アブレーションとして、採点を外して生成のみにすると安全コンプライアンスが落ちる、という結果が「物理量での採点」の価値を支えます。

これらの数値は、制御信号表現が単なる便利な書き方ではなく、計画の実現性・安全性・快適性を担保する手段であることを示しています。

## メリット・トレードオフ・限界

メリット
- 出力が必ず物理的に実現可能な動き(積分結果)になり、非物理的なギザギザ・ワープが原理的に出ない。
- 加速度・曲率の上限を入れるだけで乗り心地(comfort)制約を素直に課せる。
- 実車のコントローラ(制御器)が受け取りやすい指令形式に近い。座標列より操作列の方がそのまま制御に渡せる。
- 運動学積分を微分可能層にできるので、座標空間の損失を制御信号の学習に逆伝播できる。

トレードオフ・限界
- 「どこを走るか」を知るには毎回積分が必要で、座標が直接見えない。可視化・地図照合のたびに復元手順が要る。
- 単輪車モデルはタイヤのスリップや高速域・動力学を無視するため、横加速度が大きい限界域(約 0.5g 超)や低 μ 路面(凍結路など滑りやすい路面)では実車とのズレが出る。
- 積分には初期値(初速・初期ヨー)が要り、各ステップの誤差が時間とともに累積しうる。遠い未来ほど誤差が大きくなる。
- 損失を座標空間で取る場合、積分を通して勾配が伝わる経路が長くなり、初期ステップの制御信号に勾配が集中して学習が不安定になることがある(勾配のスケーリングや段階的学習で緩和する)。
- 研究上の課題として、限界域でも妥当な、より高次の運動モデル(動力学を含む)を学習に組み込みつつ計算を軽く保つこと、初期速度推定の誤差に頑健にすることが挙げられます。

## 発展トピック・研究の最前線
制御信号表現の発展として、Frenet 座標系(道路の中心線に沿った縦・横の座標)で軌道を表し、道路形状に対する相対量として計画する流儀があります。これは曲がった道での表現効率が良い反面、中心線の取得に依存します。また、運動学モデルを微分可能な層としてニューラルネットに埋め込み、制御信号から座標までを学習グラフの一部にすることで、座標空間の損失(地図照合や衝突回避)を制御信号の学習に逆伝播させる手法が広がっています。さらに、限界域での妥当性を高めるため、運動学モデルと学習された残差モデル(モデル誤差を補正する小さなネット)を組み合わせるハイブリッドな運動モデルも研究されています。制御信号と滑らかな曲線表現を組み合わせ、構造的に低ジャークな指令を出す方向([ジャーク低減と軌道平滑化](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0130_jerk-reduction-smoothing))も実用上重要なテーマです。
