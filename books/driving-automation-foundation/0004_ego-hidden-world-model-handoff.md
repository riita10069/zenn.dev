# プランナの隠れ状態による「計画意図→世界モデル」の受け渡し

## ひとことで言うと

自動運転モデルの中で、軌跡を計画する部品(プランナ)が「自分はこれからこう走るつもりだ」という意図を1本のベクトルに凝縮し、未来を予測する部品(世界モデル)へ手渡す設計のことです。世界モデルは「自分がこう動いたら世界はこう変わる」を学ぶために、必ず「自分の行動(意図)」を知る必要があります。その行動情報を、プランナの再帰ネットワークの最終隠れ状態という形で受け渡すのが、近年の end-to-end 運転モデルでよく使われる素直なやり方です。

## 直感的な理解

人間が運転中に頭の中でやっていることを思い出してください。「ここでアクセルを踏めば前の車に近づくな」「右に寄せたら自転車とすれ違えるな」と、自分の操作の結果を先読みしています。この「自分が動くと世界がどう変わるか」のシミュレーションを担うのが世界モデルです。

ここで決定的に重要なのは、未来は「自分が何をするか」で激変するという点です。同じ交差点でも、自分が直進するか右折するかで1秒後の景色は全く別物になります。だから世界モデルが意味のある未来を当てるには、「自分の行動」を入力に持たねばなりません。行動を知らずに未来を予測するのは、サイコロを振って当てるのと同じです。

ところがここに難しさがあります。プランナはまだ「最終的な軌跡(数値の点列)」を出している最中であり、世界モデルを動かすタイミングでは確定した行動列がまだ手元にないことがあります。そこで「確定した数値の行動」の代わりに、プランナが内部に持っている「これからこう走るつもり」という意図の凝縮表現(隠れ状態)を渡す、というのが本トピックの発想です。隠れ状態は行動そのものではありませんが、行動を生み出す元になった意図を含んでいるので、世界モデルの条件付け信号として十分機能します。

## 基礎: 前提となる概念

いくつかの用語を、知らない人向けに噛み砕きます。

世界モデル(world model)とは、「今の状態 s と自分の行動 a を入れると、次の状態 s' を予測する」モデルのことです。これは「環境のミニチュアを頭の中に持つ」ことに相当します。s' を観測(画素)で出すか、潜在表現(抽象的な特徴ベクトル)で出すかで流派が分かれます。

再帰ニューラルネットワーク(RNN)と隠れ状態(hidden state)。系列(時間順のデータ)を扱うネットワークの一種で、入力を1ステップずつ食べながら「これまでに見たものの要約」を内部メモリに溜め込みます。この内部メモリが隠れ状態です。GRU(Gated Recurrent Unit, Cho et al., 2014)や LSTM(Hochreiter & Schmidhuber, 1997)が代表で、どちらも「ゲート」と呼ばれる仕組みで、過去の情報をどれだけ覚え、どれだけ忘れるかを学習で調整します。軌跡プランナでは、時刻 0→1→…→T と未来のウェイポイント(経路上の点)を1点ずつ生成していくのに RNN を使うことが多く、最後まで生成し終えた時点の隠れ状態には「全行程をどう走るつもりか」が凝縮されています。

条件付け(conditioning)とは、あるモデルの出力を別の情報に従わせることです。「行動 a で条件付けて未来 s' を予測する」とは、「a を入力に加えて、a が変われば s' の予測も変わるようにする」という意味です。

BEV(Bird's-Eye View, 鳥瞰図)特徴。複数カメラやセンサの情報を、真上から見た格子(グリッド)状の特徴マップに統合した表現です。形は典型的に `[B, C, H, W]`(B はバッチ数、C はチャネル数、H×W は格子の広さ)。各マスに C 個の数が入り、「今この瞬間の周囲の状況の要約」を表します。世界モデルはこの BEV 特徴を「現在の世界状態」とみなし、その未来を予測することが多いです。

## 仕組みを詳しく

典型的な end-to-end 運転モデルは、おおまかに「知覚 → 計画 → 世界モデル」の流れを持ちます。

```
カメラ/センサ → 知覚 → BEV特徴 [B, C, H, W]   (今の世界)
                          │
                          ├→ プランナ(RNN)→ 軌跡 [B, T, 2]   (これからの経路)
                          │        └→ 最終隠れ状態 ego_hidden [B, C]  (計画意図の凝縮)
                          │
                          └→ 世界モデル: BEV特徴 を ego_hidden で条件付け → 未来のBEV特徴を予測
```

プランナ側で隠れ状態が作られる流れを、数値例つきで追います。チャネル数 C を 256、生成ステップ数 T を 64(例えば 0.1 秒刻みで 6.4 秒先まで)とします。

```
h = h_0                              # 初期隠れ状態 [B, 256]
                                     #   h_0 には過去の自車運動や視覚特徴を畳み込んで初期化することが多い
for t in range(64):                  # 64時刻ぶん、軌跡を1点ずつ生成
    q = combine(h, learned_query)    # 現在のメモリと学習済みクエリから問い合わせを作る
    a = cross_attention(q, BEV)      # BEV特徴のうち、今注目すべき場所を抽出
    _, h = GRU(a, h)                 # メモリ h を更新(前のhが必要 → 逐次)
    waypoint[t] = head(h)            # この時刻のウェイポイント(座標やステア/加速)を出力
trajectory  = stack(waypoint)        # [B, 64, 2] あるいは平坦化して [B, 128]
ego_hidden  = h                      # [B, 256]  ← 64ステップ走り切った最終メモリ = 計画意図
```

ポイントは、`ego_hidden` が「64ステップの軌跡を全部生成し終えたあとの隠れ状態」である点です。各時刻でどこを見て(cross attention)どう操作するか(head)を決めていく過程で蓄積された情報が、この1本のベクトルに凝縮されています。これは「入力シーンの要約」ではなく「行動(計画)の要約」であり、世界モデルの条件付け信号として筋が良いのです。

受け渡し先の世界モデルでは、現在の世界(BEV 特徴 `[B, 256, H, W]`)に、意図 `ego_hidden [B, 256]` を混ぜ込みます。形が違う 2 つのデータをどう混ぜるかが工夫どころで、最も単純で効果的なのは「チャネル毎のバイアス(per-channel bias)」です。

```
bias        = Linear(256→256)(ego_hidden)        # [B, 256]
bias        = bias.view(B, 256, 1, 1)            # [B, 256, 1, 1]
conditioned = BEV + bias                         # ブロードキャスト加算 → [B, 256, H, W]
future      = Conv2d(...)(conditioned)           # 未来のBEV特徴を予測
```

これは「マップのどのマスにも、自車の意図ぶんの味付けをチャネル単位で足し込む」操作です。空間配置(どこに何があるか)は壊さず、チャネル方向にだけ意図を注入します。混ぜ方の詳細は [計画意図ベクトルによる未来特徴マップの条件付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0010_future-state-ego-conditioning) を参照してください。

なぜ「最終隠れ状態の 1 本のベクトル」で渡すのか。代替案として「各時刻の隠れ状態を全部渡す」「確定した軌跡座標列を渡す」などもあり得ます。しかし最終隠れ状態は、(a) 固定長で扱いやすい、(b) 全行程の意図を凝縮済み、(c) 微分可能なのでプランナと世界モデルを end-to-end に同時学習できる、という利点があり、実装の単純さと表現力のバランスが良いのです。一方で「凝縮しすぎて各時刻の細かい意図が潰れる」というトレードオフも抱えます(後述)。

## 手法の系譜と主要論文

「行動で条件付けて未来を予測する」のは、世界モデル研究の中心的なアイデアであり、長い系譜があります。

World Models(Ha & Schmidhuber, NeurIPS 2018, arXiv:1803.10122)。ゲーム環境で、視覚を VAE で潜在ベクトル z に圧縮し、MDN-RNN という再帰ネットワークが「z と行動 a から次の z を予測」する世界モデルを学びました。エージェント本体(コントローラ)はこの世界モデルの内部で「夢の中で」訓練できると示し、後続研究の出発点になりました。動機は「実環境で大量に試行錯誤するのは高コストなので、頭の中のシミュレータで学びたい」。

Dreamer 系列(Hafner et al., ICLR 2020 "Dream to Control" / ICLR 2021 "DreamerV2" / arXiv:2301.04104 "DreamerV3", 2023)。Recurrent State-Space Model(RSSM)という、決定的な再帰状態と確率的な潜在変数を組み合わせた世界モデルを使い、行動条件付きで潜在状態を長期に roll-out(展開)し、その想像上の軌跡で方策を学習します。DreamerV3 は同じハイパーパラメータで多数のドメインを解き、世界モデルベース強化学習の到達点の一つとなりました。「行動で条件付けて潜在状態を予測する」という構造は、運転の世界モデルにそのまま継承されています。

MILE(Hu et al., NeurIPS 2022, "Model-Based Imitation Learning for Urban Driving", arXiv:2210.07729, Wayve)。BEV 上に潜在的な世界モデルを学び、運転操作(行動)で条件付けて未来の潜在状態を roll-out します。模倣学習(専門家のデモを真似る学習)と世界モデルを統合した点が新しく、実車環境で試行錯誤せずに「行動の結果」を内部で予測できるようにしました。プランナの意図で未来を条件付ける本トピックの発想は、この MILE の構造に最も近いものです。

GAIA-1(Hu et al., 2023, Wayve, arXiv:2309.17080, "A Generative World Model for Autonomous Driving")。動画・テキスト・行動を入力に、運転シーンの未来を生成する大規模(約 90 億パラメータ)な生成世界モデルです。行動や言語で操れる長期の未来生成を実現しましたが、計算コストが甚大というトレードオフがあります。続編の GAIA-2(2025)はさらに多条件・多カメラ生成へ拡張されました。

JEPA(LeCun, 2022, "A Path Towards Autonomous Machine Intelligence")。未来を画素ではなく潜在表現(latent)空間で、行動を条件に予測すべきだという提言です。動機は「画素を 1 つ 1 つ当てるのは無駄が多く、抽象表現で未来を予測する方が効率的で頑健」。後述の I-JEPA や V-JEPA に具体化され、運転世界モデルが「画素ではなく特徴マップを予測し、行動の代理で条件付ける」設計の理論的支柱になっています。

これらに共通する原則は「未来予測は行動で条件付けよ」。確定した行動列が手元にない設計では、その最良の代理として「プランナの最終隠れ状態」を使う、というのが本トピックの位置づけです。

## 論文の実験結果(定量データ)

世界モデルや行動条件付けの効果は、シミュレータと実データの両方で測られてきました。指標の意味から説明します。

CARLA は都市運転のシミュレータで、Driving Score(走行スコア)という総合指標がよく使われます。これは Route Completion(指定経路をどれだけ走破できたか、%)と Infraction Penalty(信号無視・衝突などの違反に応じた減点係数)の積で、0〜100 のスコアになります。高いほど安全に目的地まで到達できることを意味します。MILE は CARLA の Town05 などのベンチマークで、当時の代表的なベースライン(例えば Roach や LBC)を上回る Driving Score を報告し、特に未知の町・天候への汎化で世界モデルの効果を示しました。報告では Driving Score が従来手法から十数ポイント単位で改善し、世界モデルの roll-out を学習に使うことが汎化に寄与すると分析されています。

アブレーション(どの要素を抜くと性能がどう変わるかの検証)の観点では、世界モデル系研究は概して「未来予測(roll-out)の損失を外すと性能が落ちる」「行動条件付けを外すと未来予測がぼやけて当たらなくなる」という結果を一貫して報告します。直感的には当然で、行動を知らない未来予測は分岐(直進か右折か)で平均化してしまい、どちらでもない非現実的な未来を出すためです。

Dreamer 系では、行動条件付き世界モデルでの想像上の学習が、実環境のサンプル効率を桁で改善することが繰り返し示されています。DreamerV3 は Atari・DMControl・Minecraft など 150 以上のタスクを単一の設定で解き、特に「世界モデル内での想像 roll-out」が少ない実データから方策を引き出す鍵だと報告されています。これは運転でも「実車での危険な試行を減らせる」という形で価値を持ちます。

GAIA-1 は生成品質を FID(Fréchet Inception Distance、生成画像と本物画像の分布の近さ。低いほど良い)などで評価し、行動・テキスト条件で一貫性のある長尺の運転動画を生成できることを示しました。ただし生成系であり、Driving Score のような閉ループ運転性能の直接評価とは指標が異なる点に注意が必要です。

注意点として、これらの数値は使うデータセット・町・天候・評価プロトコルに強く依存します。具体的な順位やポイント差は論文の設定込みで解釈すべきで、「行動条件付き世界モデルは未知環境への汎化に効く」という定性的傾向が、最も再現性のある結論です。

## メリット・トレードオフ・限界

メリット。計画意図(行動)を世界モデルに渡せるので、行動条件付きの未来予測が学べます。固定長の隠れ状態ベクトルなので扱いやすく、プランナと世界モデルを微分可能なまま end-to-end に同時学習できます。確定した行動列を待たずに済むため、計画と予測を並行して回せます。理論的にも World Models / Dreamer / MILE / JEPA の行動条件付き世界モデルの原則に沿います。

トレードオフと限界。第一に、最終隠れ状態は 1 本のベクトルへの凝縮であり、各時刻の細かい意図を別々には渡せません。情報が潰れすぎて、長期の微妙な計画(「最初はゆっくり、後半で加速」など)が未来予測に効きにくい恐れがあります。第二に、隠れ状態は「真の行動そのもの」ではなく代理(proxy)なので、条件付けの忠実度には原理的な限界があります。確定した軌跡座標を渡す設計に比べ、世界モデルが「自分が本当はどこを通るのか」を正確には知り得ません。第三に、プランナと世界モデルが密結合になり、どちらかの変更が他方に波及しやすくなります。第四に、これは世界モデル全般の未解決課題ですが、roll-out を重ねるほど誤差が累積し(compounding error)、遠い未来ほど予測が崩れます。第五に、潜在空間での予測は「何を良い潜在表現とするか」「潜在が縮退(collapse)して情報を持たなくなるのを防げるか」という難問を常に抱えます。

研究上のオープンな問いとしては、(a) 凝縮された意図ベクトル一本で十分か、各時刻の意図を構造的に渡すべきか、(b) 隠れ状態という代理ではなく確定行動を渡すべき場面はどこか、(c) 長期 roll-out の誤差累積をどう抑えるか、などが挙げられます。

## 発展トピック・研究の最前線

潜在空間での予測をさらに推し進める流れとして、V-JEPA / V-JEPA 2(Meta AI, 2024–2025)があり、動画から自己教師あり学習で世界モデル的な表現を獲得し、行動条件付きの予測やロボット制御へ応用されています。運転特化では、Vista(Gao et al., 2024)のような高解像度・行動制御可能な運転世界モデルや、Drive-WM(Wang et al., CVPR 2024)のようなマルチビュー生成世界モデルがあり、生成した未来を計画の評価に使う「生成 × 計画」の統合が進んでいます。

また、世界モデルを計画に積極的に使う Model Predictive Control(MPC)的な発想も再注目されています。複数の候補行動を世界モデルで roll-out し、最も安全・効率的な未来を生む行動を選ぶ、という枠組みです。これはマルチモーダルな候補生成([マルチモーダル軌跡予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0151_multimodal-trajectory-prediction))や非自己回帰な並列計画([非自己回帰な軌跡デコーディング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0152_non-autoregressive-trajectory-decoding))と自然に結びつきます。
