# GRU 自己回帰プランナ

## ひとことで言うと

未来の軌道を「1時刻ずつ順番に」予測する古典的なプランナです。GRU という記憶を持つ小さなニューラルネットが、直前まで予測した結果を踏まえて次の1点を出す、を繰り返します。文章を1単語ずつ生成する言語モデルと同じ「自己回帰」の考え方を、車の軌道予測に当てはめたものです。実装が単純で軽量という強みがある一方、誤差が後段に累積しやすく逐次生成で遅いという弱点があり、近年は Transformer 一括予測やフローマッチングへの置き換えが進んでいます。

## 直感的な理解

道順を口で説明する場面を思い浮かべてください。「まず10m直進、次に少し右、その次にまた直進…」と、1ステップずつ前の状態を踏まえて次を言います。前のステップを忘れて急に飛ばすことはできません。GRU 自己回帰プランナはこれをそのままモデル化します。「ここまでどう走ってきたか」という記憶を1本持ち歩き、その記憶と直前の予測から次の1点を出し、記憶を更新して次へ進みます。

この方式の素直さが利点であり弱点でもあります。利点は、時刻同士の依存関係(前の動きが次の動きを決める)を自然に表現でき、実装も軽いこと。弱点は、序盤で1回ずれると、そのずれた予測を入力にして次を出すので、誤差がどんどん積み重なること(雪だるま式)です。さらに、前の予測を待ってからでないと次を計算できないので、64時刻なら64回逐次計算が必要で、並列化しにくく遅くなります。

## 基礎: 前提となる概念

自己回帰(autoregressive)とは、「前の出力を次の入力に使う」生成方式です。系列(順番のあるデータ)を1要素ずつ作るときの基本戦略で、言語モデルが1単語ずつ文を生成するのと同じです。軌道は時刻の並びなので、最初の時刻を予測し、それを見て次の時刻を予測し…と順に作るのが自然な発想になります。

RNN(Recurrent Neural Network、再帰型ニューラルネット)とは、系列を処理するために「隠れ状態(hidden state)」という記憶を持ち回るネットワークです。各時刻で「現在の入力」と「前の隠れ状態」を受け取り、「新しい隠れ状態」と「出力」を返します。隠れ状態がこれまでの履歴を要約して運ぶので、長い系列でも文脈を踏まえた予測ができます。

GRU(Gated Recurrent Unit、ゲート付き再帰ユニット)は RNN の一種で、隠れ状態を1本持ち、ゲートと呼ばれる仕組みで「何を覚え、何を忘れ、どれだけ更新するか」を学習で制御します。具体的には更新ゲート(古い記憶をどれだけ残すか)とリセットゲート(新しい入力をどれだけ取り込むか)の2つを持ちます。素朴な RNN は長い系列で勾配が消えたり爆発したりして学習が難しいのですが、ゲートがこの問題を緩和します。

LSTM(Long Short-Term Memory)は GRU の先行手法で、ゲートが3つ(入力・忘却・出力)とセル状態を持ち、より表現力が高い反面パラメータが多くやや重いです。GRU はその簡略版で、性能は同等なことが多くパラメータが少なく計算が軽いのが利点です。

軌道表現の文脈。プランナが出す軌道は「num_timesteps 個の時刻、各時刻 num_signals 個の信号」です。信号は位置(x, y)だったり、運動指令(加速度 acc と曲率 curv)だったりします。たとえば 64 時刻 × 2 信号なら、平坦化すると `[B, 128]` です。条件として、周囲を集約した視覚特徴(BEV を集約したベクトルや、過去フレームの映像履歴)や自車の運動状態(egomotion)が入ります。

## 仕組みを詳しく

GRU 自己回帰プランナの典型的な動作を段階的に追います。num_timesteps=64、num_signals=2 とします。

1. 初期の隠れ状態 h_0 を、視覚特徴や自車状態(BEV を集約したもの、映像履歴、egomotion など)から作る。これが「どういう状況で運転を始めるか」の記憶。条件ベクトルを線形層に通して h_0 にする実装が一般的です。
2. 最初の入力(開始トークンや0ベクトル)を GRU に与え、h_0 と合わせて次の隠れ状態 h_1 と出力 y_1(時刻1の acc, curv)を得る。
3. y_1 を次の入力にして、h_1 と合わせて h_2, y_2 を得る。
4. これを 64 回繰り返し、y_1..y_64 を並べて軌道 `[B, 64*2]=[B,128]` を作る。

擬似コードで書くと次の通りです。

```
h = init_hidden(context)             # 視覚・自車情報から初期記憶 h_0
y = start_token                      # 開始入力(0ベクトル等)
outputs = []
for step in range(num_timesteps):    # 1時刻ずつ、64回
    h = gru_cell(y, h)               # 記憶を更新
    y = output_head(h)               # この時刻の (acc, curv)
    outputs.append(y)
trajectory = concat(outputs)         # [B, T*num_signals]
```

ポイントは「前の出力 y を次の入力に使う」自己回帰の構造です。隠れ状態 h が「ここまでどう走ってきたか」を運び、視覚履歴のような長い条件も初期状態 h_0 に畳み込まれます。

```
context ─→ h0
            │
 start ─→ [GRU] ─→ h1 ─→ y1 ┐
                  y1 ─→ [GRU] ─→ h2 ─→ y2 ┐
                              y2 ─→ [GRU] ─→ h3 ─→ y3  ...
   逐次: 前の y が次の入力 → 64ステップ順番に
```

学習時の注意点として teacher forcing があります。学習中は「前の予測 y を次の入力に使う」代わりに「前の正解(教師)を次の入力に使う」ことが多く、これを teacher forcing と呼びます。これは学習を速く安定させますが、推論時は正解がないので自分の予測を使わざるを得ず、「学習時に見た入力分布」と「推論時の入力分布」がずれる exposure bias(露出バイアス)を生みます。これが誤差累積の根本原因です。

## Flow Matching・一括予測との違い

| 観点 | GRU 自己回帰 | 一括予測 / Flow Matching([Euler法によるODE推論サンプリング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0218_euler-ode-sampling-inference)) |
|---|---|---|
| 生成の仕方 | 時刻を1つずつ順番に | 全時刻を同時に出力 or ノイズから変形 |
| ループの軸 | 時刻方向(T回) | なし(Transformer)/ 拡散時刻 t 方向 |
| 並列性 | 低い(前の出力待ち) | 高い(全時刻を一括処理) |
| 誤差の伝播 | 前の予測ミスが後段に累積(exposure bias) | 各ステップが全体を整える |
| 多様性(複数の未来) | 出しにくい(決定的) | 出しやすい(サンプリング) |

GRU は逐次なので、序盤の予測がずれると後段がそれを引きずる誤差累積が起きやすいのが弱点です。一括予測やフローマッチングはこの弱点を避けやすい一方、前者は時刻間の依存を構造で陽に表現せず、後者は推論で複数ステップの積分が要ります。

## 手法の系譜と主要論文

GRU 自己回帰プランナは、系列モデリングと軌道予測の歴史の中に位置します。

LSTM(Hochreiter & Schmidhuber, 1997, Neural Computation)が、ゲート機構で長期依存を扱える RNN を提案し、系列学習の勾配消失問題を緩和しました。

GRU(Cho et al., 2014, EMNLP, arXiv:1406.1078)は機械翻訳の文脈で提案され、LSTM よりゲートが少なく軽量ながら同等の性能が出ることを示し、系列モデリングで広く使われるようになりました。

自動運転・歩行者の軌道予測では、Social LSTM(Alahi et al., 2016, CVPR)が RNN で歩行者の軌道を逐次予測し、近傍エージェントの相互作用を social pooling で取り込む枠組みを示しました。以後 RNN/GRU ベースのエンコーダ・デコーダが軌道予測の定番になり、Social GAN(Gupta et al., 2018)など多様な未来を出す拡張も生まれました。

近年は一括予測・生成ベースへの移行が進んでいます。VectorNet(Gao et al., 2020, CVPR, arXiv:2005.04259)と TNT/DenseTNT は、地図とエージェントをベクトル化しグラフ・Transformer で一括処理する方式を示しました。さらに拡散・フローマッチングベースの軌道生成として、Diffusion Policy(Chi et al., 2023, RSS)や MotionDiffuser(Jiang et al., 2023, CVPR, arXiv:2306.03083)が登場し、複数モードの未来を高品質に生成できることを示しました。これらへ移行する理由は、自己回帰 RNN の誤差累積と並列性の低さ、そして多様な未来を出しにくい点を避けられるためです。

整理すると、GRU は実装が単純・軽量で逐次的な依存関係を素直に表現できる反面、長い系列で誤差が累積しやすく、生成も逐次なので遅く、複数の妥当な未来を出しにくい、という性質を持ちます。

## 論文の実験結果(定量データ)

軌道予測の標準ベンチマークでの比較から、RNN 系と一括/生成系の差が読み取れます。指標の意味も添えます。

主要指標は ADE(Average Displacement Error、予測軌道と正解軌道の各時刻の距離の平均。単位はメートル、低いほど良い)と FDE(Final Displacement Error、最終時刻だけの距離。低いほど良い)です。歩行者予測のベンチマーク(ETH/UCY)では、Social LSTM が当時の手作り手法を上回りましたが、その後の生成・Transformer 系がさらに ADE/FDE を下げました。たとえば Social GAN や Trajectron++(RNN+CVAE)、そして近年の Transformer 系は、ETH/UCY で ADE をおおむね 0.5m 前後から 0.2m 前後へと段階的に改善しています(設定により幅があり、報告による)。

車両軌道の大規模ベンチマーク(Argoverse、Waymo Open Motion Dataset)では、minADE_k(k 個の候補を出し最良のものの ADE。多様な未来を出せるかを測る)や Miss Rate(候補のどれも正解から一定距離以内に入らなかった割合。低いほど良い)が使われます。VectorNet 系の一括予測や MotionDiffuser のような生成モデルは、自己回帰 RNN 単独より minADE_k と Miss Rate を明確に下げることが報告されています。MotionDiffuser は Waymo Open Motion Dataset で、複数エージェントの同時予測において当時の最先端に匹敵する精度を、拡散ベースの一括生成で達成しました。

アブレーションの観点では、自己回帰 RNN を一括デコードに置き換えると、長い予測ホライズン(先の時刻ほど)で ADE の悪化が緩やかになることが繰り返し示されています。これは誤差累積が長期予測で効くことの直接的な証拠です。数値の意味として、ADE/FDE の 0.1m の差は車線幅(約3.5m)に対しては小さく見えますが、衝突回避や車線維持の判断では無視できず、Miss Rate の数%の改善が安全性に直結します。

## メリット・トレードオフ・限界

メリット。実装が単純で軽量、逐次的な時刻依存を自然に表現できます。1ステップずつなので「途中で止める・条件を途中で変える」といった介入がしやすく、オンライン制御に組み込みやすい場面があります。パラメータが少なく、小規模データや計算資源が限られる状況で扱いやすいのも利点です。

トレードオフと限界。自己回帰ゆえに誤差累積(exposure bias、序盤のミスが後段に波及)が起きやすいです。逐次生成なので並列化しにくく、長い軌道では推論が遅くなります。一括予測の Transformer やフローマッチングに比べ、長期依存の表現力で劣る場面があり、複数の妥当な未来(マルチモーダル性)を1本の決定的な出力で表現しにくいのも弱点です。teacher forcing と推論のギャップ(分布シフト)は学習時には見えにくく、実機で初めて顕在化することがあります。

未解決・緩和策。誤差累積を緩和する scheduled sampling(学習中に徐々に自分の予測を入力に混ぜる)や、マルチモーダル性を持たせる CVAE・GAN との組み合わせが研究されてきました。ただし根本的な逐次性の制約は残るため、近年は一括/生成系への置き換えが主流です。研究・開発の実務では、プランナの実装を名前で差し替えられる共通インターフェース(レジストリ方式)を用意し、同一データ・同一条件で複数の生成方式を比較できるようにしておくと、こうした手法選択の検証が容易になります。

## 発展トピック・研究の最前線

軌道予測・計画の最前線は、一括予測 Transformer(クエリベースで複数モードを同時出力)と、拡散・フローマッチングによる生成(複数の妥当な未来をサンプリング)に移っています。GRU 自己回帰は、これらと比較するベースラインとして、また計算資源が厳しい組み込み環境での軽量選択肢として価値が残ります。研究的には、自己回帰の素直さ(時刻依存の明示)と一括/生成の強み(並列性・多様性)を両取りする、半自己回帰(数時刻まとめて出す)やトークン化した行動の Transformer 生成などのハイブリッドが探索されています。
