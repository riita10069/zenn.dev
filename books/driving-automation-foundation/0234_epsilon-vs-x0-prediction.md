# ε予測 vs x0予測

## ひとことで言うと
拡散モデルでネットワークに「何を当てさせるか」という根本的な設計の選択肢です。ノイズだらけの入力を見せて「足したノイズを当てなさい(ε予測)」とするか、「元のきれいなデータを当てなさい(x0予測)」とするか。情報としては等価なのに、どちらをターゲットにするかで各ノイズレベルでの誤差の効き方(暗黙の重み付け)が変わり、学習の安定性と生成品質に差が出ます。拡散モデルを設計するとき最初に直面する分岐点であり、後の v-prediction や EDM の出発点でもあります。

## 直感的な理解
拡散モデルは、きれいなデータ x0 にノイズを段階的に足して最終的にほぼ純ノイズにし、その逆をたどって生成します。逆向きを担うネットワークは「ノイズの乗った入力 x_t を見て、よりきれいな状態へ一歩戻す」のが仕事です。

ここで面白いのは、x_t を作る式の中に x0 と ε の両方が入っており、係数が分かっていれば片方を当てればもう片方が計算で求まる、という点です。だから「ノイズ ε を当てる」も「元データ x0 を当てる」も、得られる情報は同じはずです。それなのになぜわざわざ選択が問題になるのか。

例えるなら、同じ試験の点数を「正解からの誤差」で測るか「正解そのもの」で測るかの違いです。情報は同じでも、採点の仕方(損失の測り方)が変わると、どの問題で頑張ることになるかが変わります。拡散では「ノイズが多い問題」と「ノイズが少ない問題」が混在していて、ε を当てさせると低ノイズの問題が難しくなり、x0 を当てさせると高ノイズの問題が難しくなる。つまり、ターゲットの選択は「どのノイズレベルの学習に力を入れるか」という暗黙の優先順位付けなのです。これが両者を理解する核心です。同じネットワーク・同じデータでも、ターゲット次第でモデルの得意・不得意なノイズ帯が変わる、ということです。

## 基礎: 前提となる概念
拡散の前向き過程は、各ノイズレベル t での入力をこう書けます。

```
x_t = sqrt(alpha_bar_t) * x0 + sqrt(1 - alpha_bar_t) * ε
```

x_t はノイズレベル t での入力、x0 は元データ、ε は加えた標準正規ノイズ(平均0・分散1)、alpha_bar_t(アルファバー)は 0 から 1 の係数で、t が進む(後ろの段階)ほど小さくなりノイズの割合が増えます。t=0 ではほぼ x0、t が最大ではほぼ ε です。係数が sqrt(alpha_bar_t) と sqrt(1-alpha_bar_t) で二乗和が 1 になるのは、ノイズを足してもデータの大きさ(分散)を保つためです(分散保存型スケジュール)。

逆向きを担うネットワークは「x_t を見てより x0 に近い状態を作る」のが仕事ですが、x_t には x0 と ε の両方が入っているので、片方を当てればもう片方は計算で出ます。

```
ε が分かれば → x0 = (x_t - sqrt(1-alpha_bar_t) * ε) / sqrt(alpha_bar_t)
x0 が分かれば → ε = (x_t - sqrt(alpha_bar_t) * x0) / sqrt(1-alpha_bar_t)
```

つまり情報としては等価です。差が生まれるのは損失の測り方です。ターゲットを ε にするか x0 にするかで、各ノイズレベルでの二乗誤差が学習全体に占める重みが変わります。具体的には、ε 予測の損失を「x0 を当てる損失」に換算すると SNR(信号対雑音比)に比例する重みが掛かり、x0 予測の損失を「ε を当てる損失」に換算すると 1/SNR の重みが掛かります。この「暗黙の重み付け」が、安定性と品質を左右する正体です。

## 仕組みを詳しく
2つの方式を具体的に見ます。

ε予測(ノイズ予測)。ネットワーク ε_θ(x_t, t) に「加えたノイズ ε」を当てさせ、損失は || ε − ε_θ(x_t, t) ||²(二乗誤差)です。直感的には、ノイズが多い段階(x_t がほぼ純ノイズ)では「乗っているノイズを当てる」ことに意味があり学習信号が安定します。一方ノイズがほとんど無い段階(x_t≈x0)では、入力にノイズの手がかりがほとんど無いのに ε を当てさせるため、わずかな入力の違いに予測が過敏に反応して暴れやすく、また x0 換算で見ると誤差の重みが大きく出て学習が不安定になります。

数値例で見ます。alpha_bar_t = 0.99(低ノイズ)のとき sqrt(1−alpha_bar_t) = 0.1。x_t はほぼ x0 で、ε の寄与は係数 0.1 と小さい。x0 を ε から復元する式 x0 = (x_t − 0.1·ε)/0.995 を見ると、予測 ε がわずかにずれても x0 換算の誤差は ε 誤差の 0.1/0.995 ≈ 0.1 倍に縮む一方、損失自体は ε 空間で測るので、x0 を当てる観点では「ほとんど見えていない小さなノイズ成分を精密に当てよ」という難題になります。要するに低ノイズ域は ε 予測にとって厳しい領域です。

x0予測(データ予測)。ネットワーク x0_θ(x_t, t) に「元データ x0」を当てさせ、損失は || x0 − x0_θ(x_t, t) ||²。こちらは逆に、ノイズが少ない段階では入力がほぼ x0 なので簡単で安定します。しかしノイズだらけの段階(x_t≈ε)では、ほぼ純ノイズから元データを当てるのは非常に難しく、予測が不安定でぼやけます。alpha_bar_t = 0.01(高ノイズ)のとき sqrt(alpha_bar_t) = 0.1 なので x_t に占める x0 成分はわずか 0.1 倍。ほぼ見えていない x0 を当てよ、という無理が生じ、ε 換算で見ると誤差の重みが極端に大きくなります。

どちらが良いか。実務では ε予測 が長く標準でした。生成品質が良くなりやすいことが経験的に分かっていたためです。before/after のイメージで言うと、ε予測 で学習したモデルは高ノイズ域の大まかな構造づくりが安定して全体に滑らかなサンプルを出しやすく、x0予測 は低ノイズ域の細部が安定するが初期の構造づくりが不安定になりやすい。ただし「両端のどちらかで困る」問題は両者に共通で残るため、後に両者を混ぜた v-prediction([v-prediction パラメータ化](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0254_v-prediction))や、各ノイズレベルの損失重みを明示的に調整する手法が提案されました。

なぜ ε予測 が経験的に良かったのか、もう一段深く言うと、ε予測 の損失は x0 換算で SNR に比例する重みを持ち、結果として中〜高ノイズ域(SNR が中程度の帯)に学習の重みが乗ります。人間の知覚に効く重要な構造(大まかな形・配置)はこの帯で決まることが多く、そこに学習資源が集中する ε予測 が、見た目の品質に有利だったと解釈できます。一方で純粋な x0 換算の重み(=変分下界の重み)で学習すると、知覚的に重要でない極低ノイズ域の超細部に過剰な重みが乗り、見た目が良くならない。DDPM が「重みを落とした単純な ε MSE」を選んだのは、まさにこの暗黙の重み付けが知覚品質に合っていたからです。

## 手法の系譜と主要論文
予測ターゲットの議論は、ε予測 の確立から、その重み付けの精緻化、そして v予測・プリコンディショニングへの一般化という流れで進みました。

Ho, Jain, Abbeel「DDPM」(NeurIPS 2020、UC Berkeley、arXiv:2006.11239)が拡散を現代的に確立した原典です。理論上は変分下界に基づく重み付き目的関数になりますが、これを「ε を当てる単純な二乗誤差(L_simple)」に再パラメータ化し、各ノイズレベルの重みを落とすと実装が簡単で品質も良いことを示しました。これが ε予測 が標準になった出発点です。動機は、単純な MSE のほうが学習が安定し、暗黙の重み付けが知覚品質に有利だったこと。トレードオフは、低ノイズ域での予測の不安定さと、対数尤度(尤度評価)では重み付き目的に劣る点が残ったことです。

Nichol & Dhariwal「Improved Denoising Diffusion Probabilistic Models」(ICML 2021、OpenAI、arXiv:2102.09672)は、DDPM を改良し、分散も学習する(L_hybrid で平均は L_simple、分散は変分下界で学習)、コサイン型のノイズスケジュールを導入する、などを提案しました。予測ターゲットや損失設計がサンプル品質と対数尤度に効くことを丁寧に検証し、ε予測 を基本としつつ重み付けやスケジュールの選び方が重要だと示しました。効果は、少ないステップでの品質と尤度の改善。特に線形スケジュールが高ノイズ端で情報をほぼ全て壊してしまい無駄な区間を作る問題を指摘し、コサインスケジュールで改善しました。トレードオフは、分散学習を加えると目的関数が複雑になり調整項が増える点です。

Salimans & Ho「Progressive Distillation」(ICLR 2022、arXiv:2202.00512)は、ε予測 も x0予測 も「端のノイズレベル」で破綻するため、両者を混ぜた v-prediction を導入しました([v-prediction パラメータ化](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0254_v-prediction))。これは ε vs x0 の二者択一の弱点を、第三の中間ターゲットで直接補うものです。少ステップ蒸留では1ステップで広いノイズ範囲をまたぐので、端の破綻が致命的になり、v の安定性が決定的に効きました。

Karras et al.「Elucidating the Design Space of Diffusion-Based Generative Models(EDM)」(NeurIPS 2022、NVIDIA、arXiv:2206.00364)は、予測対象・前処理・損失重みを統一的な設計空間として整理し、入力と出力を各ノイズレベルで正規化する「プリコンディショニング」(c_skip, c_out, c_in, c_noise の係数で出力分散を全域で1に揃える)を提案しました。これは「ε か x0 か」という二者択一を、「ネットの入出力をどうスケールすれば全域で安定するか」という一般的な枠組みに拡張したもので、ε予測・x0予測・v予測がこの枠組みの特殊ケースとして位置づけられます。EDM はさらに、学習時のノイズレベルを対数正規分布から引く重み付けを提案し、中庸の SNR 帯に学習を集中させました。

## 論文の実験結果(定量データ)
DDPM(2020)は、ε予測 + 単純 MSE という組み合わせで、CIFAR-10(32×32)において FID 約 3.17、Inception Score 約 9.46(値が大きいほど良い)を達成しました。論文は、理論的に正しい重み付き目的(L_vlb)より、重みを落とした単純な ε 予測 MSE(L_simple)のほうが見た目の品質(FID)が良いことを実験的に示しており、これが「ε予測 が標準」を決定づけました。ここでのアブレーション的知見は「目的関数の重み付けを変えると品質が変わる」こと、すなわちターゲットの暗黙の重み付けが品質を左右する、という本質的な発見です。

Improved DDPM(2021)は、コサインスケジュールと分散学習(L_hybrid)の導入により、ImageNet 64×64 や CIFAR-10 などで対数尤度(モデルがデータをどれだけうまく説明できるかの指標、bits/dim で測り小さいほど良い)と FID の両方を改善しました。特に、線形スケジュールでは高ノイズ域でサンプルの情報がほぼ消えてしまい逆向き過程で学ぶことが少ない無駄な区間が生じることを指摘し、コサインスケジュールでバランスを取ると品質が向上すると示しました。これは「ターゲット選択とノイズスケジュールはセットで、各 SNR 帯への重み配分が品質を決める」という ε vs x0 議論の延長線上の結果です。

EDM(2022)は、CIFAR-10 で当時の最高水準である FID 約 1.97(条件なし)を達成し、プリコンディショニング(出力スケールの SNR 依存正規化)と損失重みの設計が品質に大きく効くことを定量的に示しました。アブレーションでは、入出力の正規化を外したり予測ターゲットを素朴な ε/x0 に戻したりすると品質が悪化し、各ノイズレベルで出力スケールを 1 に揃えることの重要性が確認されています。これは「ε か x0 か」という素朴な選択が、より一般的には「出力をどう正規化するか」の問題であることを実証しました。EDM はサンプラ(2次の Heun 法)とノイズスケジュールも見直し、35 回程度のネット評価で高品質を達成しています。

これらの数値の意味を初心者向けに言うと、FID は生成物の本物らしさ・多様さ(小さいほど良い)、対数尤度はモデルがデータをどれだけうまく説明できるか(尤度が高いほど良い)、ネット評価回数は生成の速さ(少ないほど速い)です。一連の研究が示したのは、ターゲットの選び方ひとつ(と、それと不可分なノイズスケジュール・損失重み)が、これらの数値を数段階動かすほど効く、ということです。FID 3.17 → 1.97 のような改善は、骨格を変えずターゲットと正規化の設計だけで達成された点が重要です。

## メリット・トレードオフ・限界
ε予測 のメリットは、経験的に生成品質(FID)が良いこと、実装が単純な MSE であること、高ノイズ域で安定すること、暗黙の重み付けが知覚的に重要な帯に乗ること。トレードオフは、低ノイズ域(x_t≈x0)で予測が不安定・学習信号が過敏になること、対数尤度では劣りやすいことです。

x0予測 のメリットは、低ノイズ域で安定し最終的な細部が出しやすいこと、出力が直接データなので解釈しやすく、可視化・デバッグが容易なこと、クリッピングなど後処理を入れやすいこと。トレードオフは、高ノイズ域(x_t≈ε)で「ほぼ純ノイズから元データを当てる」のが難しく不安定なことです。

共通の限界は、どちらも「端のノイズレベル」で困ること。これを根本的に避けるには、v-prediction や EDM のプリコンディショニングのような中間的・正規化されたターゲットが要ります。さらに研究上の論点として、ターゲット選択はノイズスケジュールや損失重みと不可分であり、タスク・解像度・SNR 分布に応じて何が最適かは一意に決まらない、という難しさが残ります。たとえば高解像度・少ステップでは v や EDM 流の正規化が有利で、低解像度・多ステップでは素朴な ε 予測でも十分なことが多い、といった経験則があります。

## 発展トピック・研究の最前線
ε予測 vs x0予測 の議論は「ネットの予測ターゲットの選び方が学習の安定性を左右する」という普遍的な教訓を残しました。この問題意識は、v-prediction(両者を混ぜて全域で安定させる)、EDM プリコンディショニング(出力スケールを SNR ごとに正規化する一般枠組み)へと結実し、さらにフローマッチング/rectified flow 系(ノイズからデータへの経路の「速度」を予測する)にも通じます。いずれも「全領域で安定して学習できるターゲットを選ぶ」という同じ動機から出ています。

最前線では、損失重みを学習中に動的に調整する手法(min-SNR、soft-min-SNR など)、ターゲットとスケジュールを統一的に設計する理論、そして少ステップ生成のための蒸留・整流と相性の良いターゲット設計が活発に研究されています([Consistency Models](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0225_consistency-models))。また、画像以外のモダリティ(軌道・行動・点群・音声など)では SNR 分布やデータのスケールが画像と異なるため、どのターゲットが適切かをタスクごとに再検討する必要があり、ロボティクスや自動運転の生成モデルでもこの設計軸は重要です。「何を予測させるか」という一見素朴な問いが、生成モデルの効率と品質を握る中心的な設計軸であり続けている、というのがこのトピックの最大の学びです。
