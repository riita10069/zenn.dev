# Waymo Open Motion Dataset (WOMD)

## ひとことで言うと

WOMD は、Google 系列の自動運転会社 Waymo が公開した「車や歩行者が次の数秒でどう動くかを予測する」ための大規模データセットです。何十万もの交通シーンが、各エージェント (車・歩行者・自転車) の正確な位置・速度の時系列と、周囲の道路地図つきで収録されています。自動運転の「動作予測 (motion prediction)」「軌道予測 (trajectory prediction)」という分野で、研究者が手法を比べる共通の物差し (ベンチマーク) として使われています。さらに WOMD は、複数のエージェントがお互いに影響し合う「相互作用 (interactive) 予測」という難しい設定を初めて大規模に整備した点で画期的でした。

ここで「軌道予測」とは、ある時点までのエージェントの動き (過去 1 秒間どこをどう走ってきたか) を入力に、その後の数秒間にどこへ進むかという未来の位置の連なりを当てるタスクを指します。

## 直感的な理解

自動運転車が安全に走るには、自分の経路を決める前に「周りの車や人がこれからどう動くか」を予想しなければなりません。交差点で対向車が左折するのか直進するのか、横断歩道の歩行者が渡り始めるのか待つのか — それによって自車の取るべき行動はまったく変わります。人間の運転者は、相手の速度・向き・ウィンカー・車線の流れから無意識にこれを予測しています。

この予測には本質的な難しさがあります。未来は 1 つに決まっていません。同じ過去でも「直進するかもしれないし右折するかもしれない」のです。だから良い予測モデルは「1 本の未来」ではなく「ありうる複数の未来とそれぞれの確率」を出さなければなりません。さらに、エージェントは互いに影響し合います。狭い道で 2 台が向き合えば、片方が進めばもう片方は止まる、という依存関係があります。これを無視して各車を独立に予測すると、両方が同時に進む「ありえない未来」を出してしまいます。WOMD は、こうした多様性 (複数の未来) と相互作用 (エージェント同士の依存) を、大量の実データで学習・評価できるようにするために作られました。

## 基礎: 前提となる概念

エージェント (agent): 予測対象となる動く物体です。WOMD では車両・歩行者・自転車の 3 種類を扱います。

ポリライン (polyline): 点を順につないだ折れ線です。WOMD の地図 (車線中心線・境界線・停止線・横断歩道) は画像ではなくこのポリライン (点の連なり) として与えられます。ベクトル形式なので、解像度に縛られず精密に扱えます。各点には位置だけでなく「これは中心線か境界線か」という型情報も付きます。

マルチモーダル予測の「モード」: ここでの「モード」は確率分布の山 (取りうる選択肢) を指します。「直進」「右折」がそれぞれ 1 つのモードで、モデルは典型的に K=6 個程度のモード (候補軌道) と、各モードの確率を出力します。注意: 同じ「マルチモーダル」でも、センサの種類 (カメラ・LiDAR) を指す場合とは意味が違います。

mode collapse (モード崩壊): 学習がうまくいかず、本来分かれるべき複数の候補軌道がすべて「平均的でどっちつかずの 1 本」に潰れてしまう現象です。これを避ける損失設計が予測研究の中心課題の 1 つです。

marginal 予測と joint (interactive) 予測: marginal (周辺) 予測は各エージェントを独立に予測します。joint (相互作用) 予測は複数エージェントの未来を「お互いに整合するように」同時に予測します。後者の方がはるかに難しく、WOMD が力を入れた新設定です。

minADE / minFDE / Miss Rate / mAP (予測の評価指標): 後の実験結果のセクションで詳しく説明しますが、いずれも「複数候補のうち最良の 1 本が正解にどれだけ近いか」「終点を当てられたか」「外した割合」「確率づけが妥当か」を測るものです。

## 仕組みを詳しく

WOMD のデータ構造をバージョンの細部を捨象して説明します。

- シーン (scenario) 数は約 10 万 (走行距離にして 1,750 時間超に相当)。各シーンは 20 秒間の交通状況の切り出しです。
- サンプリングは 10 Hz (1 秒に 10 回)。20 秒のシーンは 200 タイムステップから成ります。
- 標準タスクは「過去 1 秒 (11 ステップ、現在を含む) を見て未来 8 秒 (80 ステップ) を予測」です。8 秒は予測タスクとしてかなり長く、不確実性が大きい挑戦的な設定です (多くの先行データセットは 3 秒予測でした)。
- エージェント種別は車両・歩行者・自転車。1 シーンに最大で数十のエージェントが含まれ、各シーンで「予測対象」として最大 8 体が指定されます。
- 地図は車線中心線・境界線・停止線・横断歩道・速度バンプなどに加え、信号の状態 (赤・青・黄) がポリラインとして付属します。信号は動的に変わるので、各タイムステップで状態が記録されます。

各エージェントの 1 タイムステップの状態は、おおよそ次の数値の組です。

```
state = {
  位置:   x, y, z        (メートル単位の地図座標)
  大きさ: length, width, height
  向き:   heading         (ラジアン)
  速度:   velocity_x, velocity_y
  有効性: valid           (このステップで観測できたか 0/1)
}
```

valid フラグが重要です。実データでは遮蔽やセンサの死角でエージェントが見えなくなる瞬間があり、そのステップは valid=0 になります。学習・評価ではこのフラグでマスクし、観測できなかった時刻を予測誤差に含めないようにします。

軌道予測を tensor の形で書くと、1 つの予測対象エージェントについて次のようになります。

```
入力 (過去): [11, 状態次元]   過去 1.1 秒の時系列
出力 (未来): [K, 80, 2]       K本の候補軌道、各80ステップのx,y
確率:        [K]              各候補の確率 (合計1)
```

K=6 がよく使われます。「直進」「緩く右」「急に右」「左」のように、現実的にありうる分岐を 6 本でカバーする狙いです。学習では、正解に最も近い 1 本だけを正解に引き寄せ、残りは確率だけ調整する、という損失設計 (winner-takes-all / hard assignment に近い) が多用されます。全候補を一律に平均へ引き寄せると mode collapse を起こすためです。最終時刻 (8 秒) だけ予測してから途中を補間する手法、軌道全体を一度に出す手法など、出力の作り方には流派があります。

joint (interactive) 予測ではさらに難しくなります。2 エージェントの未来を組として出すため、出力は各エージェント K 本ではなく「K 組の整合した未来ペア」になります。片方が進むなら相手は止まる、という組み合わせを 1 つのモードとして表現します。組み合わせ爆発 (各エージェント K 本なら 2 体で K^2 通り) をどう抑えるかが設計上の難所です。

## 手法の系譜と主要論文

WOMD 本体: Ettinger, Cheng, Caine, Vasudevan et al. (Waymo), "Large Scale Interactive Motion Forecasting for Autonomous Driving: The Waymo Open Motion Dataset", ICCV 2021。
- 何を提案したか: それまでで最大級 (走行時間・シーン数) の、地図つき・マルチクラス (車/歩行者/自転車)・高精度トラッキングのモーション予測データセット。さらに「複数エージェントの相互作用を同時に予測する」interactive タスクを定式化しました。
- なぜそうしたか: 既存データ (Argoverse, INTERACTION 等) は規模・多様性・地図情報が不足し、特に「エージェント同士が影響し合う場面」をきちんと評価できませんでした。実世界で安全に動くには相互作用の予測が不可欠だという問題意識です。
- 効果とトレードオフ: 大規模で多様なため、まれなシーンも学習・評価でき、予測研究のベンチマークとして定着しました。一方、データが北米の Waymo 運行エリアに偏ること、8 秒予測の本質的な不確実性が論文中でも議論されています。

母体となる知覚版 Waymo Open Dataset (Sun et al., CVPR 2020) との関係も押さえておきましょう。知覚版は 3D 検出・追跡用、WOMD はその兄弟として予測に特化して作られたものです。詳しくは [Waymo Open Dataset](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0904_waymo-open-dataset) を参照してください。

予測手法の系譜:
- VectorNet (Gao et al., CVPR 2020) が「地図と物体をベクトル (ポリライン) のグラフで表す」路線を切り開きました。それ以前は地図をラスタ画像にして CNN で処理するのが主流でしたが、解像度で情報が落ち計算も重いという問題がありました。
- MultiPath++ (Varadarajan et al., ICRA 2022, Waymo): 地図を画像化せず、車線や他エージェントの状態をベクトルのまま符号化し、要素ごとに集約 (context gating / 多層集約) してから複数候補軌道を出すアーキテクチャ。出力は固定アンカーではなく学習で得た代表的軌道 (learned anchor) の混合分布として表現します。WOMD で当時のトップ精度を達成しました。
- SceneTransformer (Ngiam et al., ICLR 2022, Waymo): 全エージェント・全時刻を 1 つの Transformer で同時に扱い、marginal と joint を同じモデルで切り替えられる統一的な定式化を提案しました。
- Wayformer (Nayakanti et al., ICRA 2023, Waymo): 自車・他車・地図・信号という多様な入力を Transformer の Attention (入力のどこに注目するかを学習する仕組み) でまとめて扱うシンプルな設計。入力ごとに専用処理を作る煩雑さを避け、統一的な Attention で強さと見通しの良さを両立しました。
- MTR (Motion TRansformer, Shi et al., NeurIPS 2022): 学習可能な「意図クエリ (intention query)」で K 本のモードを明示的に表現し、各クエリが代表的な目的地に対応するよう設計することで、モード崩壊を抑えつつ WOMD リーダーボードを更新しました。後続の MTR++ などへ発展しています。

## 論文の実験結果 (定量データ)

評価指標を平易に説明し、報告値の傾向を示します。

- minADE (minimum Average Displacement Error): K 本の予測候補のうち正解に最も近い 1 本を選び、その軌道の各時刻での位置ズレ (メートル) を平均したもの。小さいほど良い。「複数候補のうち一番当たった線がどれだけ正確か」を測ります。
- minFDE (minimum Final Displacement Error): 同じく最良候補について、最後の時刻 (8 秒後) だけの位置ズレ。終点をどれだけ当てたか。
- Miss Rate: 最良候補ですら正解から一定距離以上 (時刻ごとに定めた閾値) ずれた割合。「外した確率」。
- mAP / soft-mAP (mean Average Precision): 候補の確率づけが妥当か (自信度のキャリブレーション) まで含めて測る総合指標。当たった軌道に高い確率を、外れた軌道に低い確率を割り当てられているかを評価します。WOMD はこの mAP を主要ランキング指標としました。直進・右折といった「軌道の種類」ごとに AP を測って平均するため、まれな挙動 (急な方向転換など) を当てる力も反映されます。

WOMD の marginal 予測における代表的な傾向として、強い手法 (Wayformer・MTR 系) は 8 秒予測で minADE がおよそ 0.5〜0.6 m、minFDE がおよそ 1.1〜1.3 m 程度のレンジに収まると報告されています (バージョン・分割で差があります)。位置ズレが平均 0.5 m 台というのは、8 秒という長い予測 horizon を考えると非常に小さく、最良候補に限れば実用的な精度に達していることを意味します。一方で mAP は 0.4 前後にとどまることが多く、「最良候補は当たるが、どの候補が正しいかの確率づけ (キャリブレーション) はまだ難しい」という課題が数値に表れます。これは安全上重要で、確率を誤ると「ありそうにない未来に備えてしまう」「ありうる危険を見落とす」につながります。

MultiPath++ の論文 (ICRA 2022) は、地図をベクトルで扱う設計の効果をアブレーションで示しました。地図のベクトル符号化や、エージェント間・地図間の集約 (multi-context gating) を簡略化すると minADE/minFDE が悪化することが報告され、「ベクトル地図の明示的利用」と「文脈情報の集約」がそれぞれ精度に効いていることが定量的に確認されました。

Wayformer (ICRA 2023) のアブレーションでは、入力の融合方法を体系的に比較しました。early fusion (早い段階で全入力を 1 つの Attention に結合)・late fusion (別々に処理してから統合)・hierarchical fusion を並べ、シンプルな early fusion でも適切な Attention 設計と十分なモデル容量があれば最高水準の精度を保てることを示しました。「専用設計を増やさなくても、統一的な Attention で十分強い」という結論が、複雑化しがちな予測アーキテクチャに対する反証として価値を持ちました。さらに、計算量を抑える工夫 (latent query による系列長の圧縮) を入れても精度がほぼ落ちないことを示し、効率と精度の両立を定量化しました。

MTR (NeurIPS 2022) は intention query の有効性をアブレーションで示し、固定アンカーや単純な回帰よりも、学習可能な意図クエリで各モードを担当させる方が mAP・Miss Rate ともに改善することを報告しました。WOMD と Argoverse 2 の両方でリーダーボード上位を占めたことが、この設計の汎用性を裏づけました。

interactive (joint) 予測は marginal より一貫して難しく、同じ手法でも minADE・mAP ともに marginal 設定より悪化します。これは「複数エージェントの整合した未来」を当てる方が本質的に難しいことの定量的な裏づけであり、WOMD が interactive 設定を用意した意義そのものを示しています。

## メリット・トレードオフ・限界

メリット:
- 大規模 (約 10 万シーン) で、まれな状況も含めて学習・評価できる。
- 車・歩行者・自転車の 3 種を含み、現実の混在交通に近い。
- 詳細なベクトル地図と信号状態がつき、道路文脈に基づく予測を学べる。
- interactive (相互作用) タスクという、安全に直結する難しい設定を提供する。
- 評価サーバとリーダーボードがあり、手法を公平に比較できる。

トレードオフ・限界:
- データが Waymo の北米運行エリアに偏り、他地域・他交通文化への一般化は別途検証が必要。
- 8 秒先の予測は本質的に不確実で、単一正解との比較評価には限界がある (複数候補・確率で緩和してはいる)。単一の真値しか無いため「もう一つのありえた未来」を当てても評価上は外れになりうる。
- ライセンスは研究用途中心で、商用利用には制約がある。
- 地図・トラッキングは Waymo の内部システムで作られた擬似正解であり、完全な真値ではない (自動ラベリング由来の誤差を含みうる)。

未解決の研究課題: 確率づけ (mAP) の改善、すなわち「どの未来がどれくらいありそうか」を正しく当てること、計算量がエージェント数・時系列長に応じて増える問題への対処 (Transformer の Attention はトークン数の二乗に効きやすい)、そして予測を運転計画とエンドツーエンドに結合する方法が活発に研究されています。後継の WOMD には、より長期・カメラ付きのサブセットや、シーン生成 (sim agents) チャレンジも追加され、研究の射程が広がっています。

## 発展トピック・研究の最前線

予測と計画の境界が溶けつつあるのが最前線です。従来は「予測 → その結果を計画に渡す」と段階を分けていましたが、相手の動きは自車の動きに依存する (自車が譲れば相手は進む) ため、予測と計画を同時に解く joint prediction-and-planning や、ゲーム理論的なアプローチが研究されています。WOMD の interactive 設定はこの方向の評価基盤になっています。

もう 1 つは大規模言語モデル・基盤モデルの予測への応用です。交通シーンを言語的・記号的に記述し、汎用モデルの推論能力で意図を読む試みが現れています。また、予測の不確実性をより誠実に表現する (単なる K 本ではなく確率分布や生成過程として出す) 生成モデル系 (拡散モデルなど) の応用も進んでおり、WOMD の mAP のような確率評価がその良し悪しを測る物差しになっています。さらに WOMD の sim agents チャレンジは「予測の良さ」を「シミュレーション内で他車をどれだけ人間らしく動かせるか」という新しい角度から測り、予測とシミュレーションを橋渡ししています。
