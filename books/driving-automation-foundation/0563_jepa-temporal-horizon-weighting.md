# 未来ホライズンの重み付け

## ひとことで言うと

複数の未来時刻(たとえば 1.6秒後・3.2秒後・4.8秒後・6.4秒後)を同時に予測するモデルは、それぞれの予測について別々に答え合わせ(損失)を計算します。これら複数の損失を1つの数にまとめるとき、全部を対等に扱うか、近い未来を重く・遠い未来を軽く扱うか、という配分の設計判断が「未来ホライズンの重み付け」です。たった数個の重みを決めるだけですが、学習の安定性と予測の質、そして安全への寄与が変わります。

## 直感的な理解

天気予報を思い浮かべてください。「1時間後の天気」はほぼ確実に当たりますが、「1週間後の天気」は外れがちです。遠い未来ほど不確実性が積み重なって当てにくくなる、というのは予測一般に共通する性質です。

運転でも同じです。1.6秒後の道路はほぼ「いまの続き」で予測しやすい。だが 6.4秒後になると、信号が変わる・前の車が車線変更する・歩行者が動く、といった不確定要素が積み重なり、当てるのが格段に難しくなります。難しい予測ほど損失(誤差)は大きくなりがちです。

ここで素朴に「4つの未来の損失を全部足して平均」すると落とし穴があります。遠未来の損失が大きいせいで、学習の勾配(パラメータをどう動かすかの信号)が遠未来に引っ張られ、モデルが「当てやすく安全に直結する近未来」をおろそかにしかねません。逆に近未来だけを重視しすぎると、長期的な見通しを学べず、先読みのない近視眼的な運転計画になります。この綱引きを明示的に制御するのがホライズンの重み付けです。

## 基礎: 前提となる概念

- **ホライズン(horizon, 予測の地平線)**: どれくらい先の未来まで予測するかの時間幅です。「予測ホライズン 6.4秒」なら 6.4秒先まで見通します。これを複数点に区切って予測する(1.6秒刻みで4点など)のがマルチホライズン予測です。

- **per-step loss(時刻ごとの損失)**: 各未来時刻の予測について個別に計算した損失です。これを1つにまとめる前段の中間量です。

- **集約(aggregation)/重み付け平均(weighted mean)**: 複数の損失を1つのスカラーにまとめる操作です。単純平均は全部に重み 1、重み付け平均は時刻ごとに違う重みを掛けます。最終的な損失が backpropagation の出発点になるため、ここの設計が勾配の流れを決めます。

- **割引(discount, γ)**: 強化学習で遠い未来の報酬を係数 γ(0.9〜0.99 程度)で割り引く考え方です。`重み = γ^t`(t は未来ステップ)とすると、自然に近未来重視の指数減衰が得られます。ホライズン重み付けの近縁概念です。

- **損失スケール(loss scale)とλバランス**: 複数の損失(未来予測損失、軌道損失など)を足し合わせて学習するとき、各損失の「大きさ」が揃っていないと、大きい損失だけが勾配を支配します。係数 λ で重みづけして足すのが普通ですが、未来予測損失内部の重み付けで全体スケールが変わると、この λ バランスを毎回調整し直す羽目になります。だから後述の「正規化」が重要になります。

## 仕組みを詳しく

未来予測の損失は、まず各未来時刻ごとに別々に計算します。

```
per_step_losses = [
  MSE(予測[t+1.6s], 正解[t+1.6s]),   # 損失_1 (近未来, 当てやすい)
  MSE(予測[t+3.2s], 正解[t+3.2s]),   # 損失_2
  MSE(予測[t+4.8s], 正解[t+4.8s]),   # 損失_3
  MSE(予測[t+6.4s], 正解[t+6.4s]),   # 損失_4 (遠未来, 当てにくい)
]
```

これを `per_step`(形状 `[4]` のテンソル)にまとめ、各要素に時間方向の重み `temporal_weights` を掛けてから平均します。

```python
weighted = per_step * temporal_weights   # 要素ごとの掛け算
loss = weighted.mean()
```

重みには代表的に2つの設計があります。

1. **均等重み(equal weighting)**: `[1, 1, 1, 1]`。4つの未来を対等に扱う。最もシンプルで、強い事前知識を入れない素直な選択。
2. **近未来重視(near-horizon weighting)**: 例えば `[4, 3, 2, 1]` のように近い未来ほど大きい重み。当てやすく安全に直結する近未来を優先する。指数型 `[γ^0, γ^1, γ^2, γ^3]` もこの一種。

### 正規化(スケール保存)の工夫

重みをそのまま掛けると、損失全体のスケールが変わってしまい、他の損失との λ バランスが崩れます。これを防ぐため、重みの合計が `num_future_steps`(=4)になるよう正規化するのが定石です。

```python
weights = weights / weights.sum() * num_future_steps
```

数値例: `[4, 3, 2, 1]` は合計10なので、`[4,3,2,1] / 10 * 4 = [1.6, 1.2, 0.8, 0.4]`。合計は再び 4 となり、均等重み `[1,1,1,1]` の合計と一致します。これにより「相対配分」だけを変えて「全体の大きさ」は保つ、という綺麗な制御ができます。結果、近未来重視に切り替えても他の損失との係数を再調整せずに済みます。

さらに、重みの合計がほぼ0になる異常入力に対する防御として、`abs(total) > 1e-8` のときだけ正規化し、そうでなければスキップする実装がよく使われます。ゼロ割りで損失が NaN(非数)になるのを防ぐためです。

集約せず `[4]` の各時刻の重み付き損失をそのまま返すモード(`reduction="none"` 相当)も有用です。これは学習の監視(どの未来時刻が当たっていないか)や、推論時の [予測誤差による不確実性推定(イントロスペクション)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0697_prediction-error-introspection)(時刻ごとの異常検知)に使えます。

### before / after(勾配の流れ)

```
均等重み [1,1,1,1]:
  遠未来の大きな損失が勾配を支配 → 近未来の精度が頭打ちになりやすい

近未来重視 [1.6,1.2,0.8,0.4] (正規化済):
  近未来に勾配を厚く配分 → 当てやすい近未来を確実に学ぶ
  全体スケールは均等重みと同じなので λ バランスは不変
```

## 手法の系譜と主要論文

- **V-JEPA(Bardes et al., Meta, 2024, arXiv:2404.08471)**: 動画で複数領域・時刻を潜在空間で予測する自己教師あり学習。複数の予測対象にわたって損失を平均する基本形で、特別な時刻重み付けをせずとも強い表現が学べることを示しました。均等重みの素直な設計の代表例です。トレードオフは、難しい遠未来に勾配が引っ張られうる点。

- **Trajectron++(Salzmann et al., ECCV 2020, arXiv:2001.03093)** ほか motion forecasting 研究: 軌道予測では未来時刻ごとに不確実性が増すため、損失や評価指標を時刻ごとに分けて扱うのが定石です。評価指標 ADE(Average Displacement Error, 全時刻平均のずれ)と FDE(Final Displacement Error, 最終時刻のずれ)を分けて報告する慣習自体が、「時刻によって難しさが違う」ことを前提にしています。

- **強化学習・世界モデルの割引(Sutton & Barto, 2018 / Dreamer, Hafner et al., ICLR 2020, arXiv:1912.01603)**: 遠未来の報酬を γ で割り引く考え方は、遠未来を軽く扱う近未来重視と発想が同根です。Dreamer のような潜在世界モデルでは、ロールアウト(想像上の未来展開)の長さに応じた重み調整(λ-return など)が学習の安定化に効くことが知られています。

系譜としては、「軌道予測が時刻別評価を確立(ADE/FDE)」→「RL/世界モデルが割引で遠未来を制御」→「動画 SSL(JEPA)がマルチ時刻予測を均等平均で実用化」と発展し、現在は「均等を基本に、必要に応じ近未来重視や割引を試す」のが実務的な落とし所です。

## 論文の実験結果(定量データ)

- **V-JEPA**: Kinetics-400(動画分類)などで、ピクセル再構成型(動画 MAE)に匹敵または上回る下流性能を、より少ない事前学習計算で達成したと報告。マルチ時刻・マルチ領域の予測損失を素朴に平均する設計でも、frozen evaluation(特徴を固定して下流タスクを解く評価)で強い結果が出ることを示しました。特別な時刻重み付けが必須ではない、という存在証明です。

- **Trajectron++**: nuScenes・ETH/UCY などの歩行者・車両軌道予測で、FDE(最終時刻のずれ)が遠い予測時刻ほど大きく増えることを定量的に示しています(近未来 1秒の誤差は小さく、遠未来 4秒で数倍に増える)。これは「遠未来ほど損失が大きく、均等平均では勾配を支配しうる」という本トピックの動機を裏付ける実データです。

- **Dreamer / DreamerV2(Hafner et al.)**: Atari・連続制御で、想像ロールアウトの価値推定に λ-return(複数ホライズンの推定を指数重みで混ぜる)を用い、ホライズン長と割引の設定が最終スコアに有意に影響することをアブレーションで示しました。ホライズン重み付けの設計が性能を左右する直接的な証拠です。

定量データから読み取れる一般則は、(a) 遠未来の予測誤差は近未来より明確に大きい(よって均等平均は遠未来に偏りやすい)、(b) それでも均等平均で十分強い表現が学べる場合が多い(V-JEPA)、(c) ホライズン重み(割引)の設定はタスクによっては性能を有意に変える(Dreamer)、という3点です。「均等で始め、効かなければ重み付けを探索する」のが妥当な順序だと示唆されます。

## メリット・トレードオフ・限界

メリット
- 数個の重みだけで、近未来重視か全時刻均等かを柔軟に制御できる。
- 正規化により損失スケールが保たれ、他の損失との λ バランスを再調整せずに済む。
- 時刻ごとの損失を取り出せば、デバッグ(どの未来が当たっていないか)と推論時監視がしやすい。

トレードオフ・限界
- 最適な重みはデータ・タスク依存で決め打ちできない(ハイパーパラメータ探索が要る)。探索コストが増える。
- 近未来を重視しすぎると長期見通しが学べず、遠未来を重視しすぎると学習が不安定化する(難しい目標に勾配が支配される)。
- 重み付けは「どの時刻を頑張るか」を変えるだけで、予測そのものを賢くするわけではない。表現の質は別の工夫(良いターゲットエンコーダ、残差予測など)に依存する。
- 時刻間で損失の意味やスケールが本質的に違う場合、単純な重み付けでは揃えきれない(遠未来は本質的に分散が大きい)。

未解決の課題として、「重みを学習中に動的に調整する」適応的重み付け(不確実性に応じて自動で配分する uncertainty weighting など)の最適設計はまだ確立されていません。また、何点の未来をどの間隔で予測するか(時間サンプリング設計)とホライズン重み付けの相互作用も研究途上です。

## 発展トピック・研究の最前線

- **不確実性ベースの自動重み付け(Kendall et al., multi-task uncertainty weighting)**: 各タスク(ここでは各未来時刻)の損失を、学習可能な不確実性パラメータで自動的に重み付けする手法。手動チューニングを不要にする方向。
- **カリキュラム学習との接続**: 学習初期は近未来だけ、徐々に遠未来の重みを上げる、という時間方向のカリキュラムで学習を安定化させるアイデア。
- **行動条件付きホライズン**: 自車の行動計画(plan)の時間長とホライズン重みを揃え、計画地平線に直結する時刻を重視する設計([World Model(マルチビュー世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1028_world-action-model))。
- **長期予測の崩壊回避**: 遠未来ほどターゲット特徴が不安定になり崩壊しやすいため、ホライズン重み付けと frozen/EMA ターゲット([凍結バックボーン vs EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0556_frozen-vs-ema-target-encoder))・残差予測([TDV(時間差分・残差予測)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0583_tdv-motion-residual-prediction))を組み合わせる研究の流れ。
