# EDM (Elucidating Diffusion Models)

## ひとことで言うと
拡散モデル(画像などをノイズから少しずつ生成するモデル)を作るとき、ノイズの量の決め方、入力をネットワークに渡す前の前処理のやり方、生成時にノイズを取り除く手順(サンプラ)など、数多くの設計の選択肢があります。EDM は、それらをバラバラに考えるのではなく「一つの共通の数学的な枠組み」に整理し直し、各部品を独立に最適化できるようにした研究です。その結果、当時の画像生成の品質と速度を同時に大きく更新しました。「elucidate(解明する)」という名のとおり、それまで論文ごとに暗黙に混ざっていた設計選択を明るみに出した点が本質です。

## 直感的な理解
料理に例えます。同じ「カレーを作る」でも、火加減、具を入れる順番、煮込み時間、隠し味の選び方など多くの工程があります。ある人のカレーが美味しくても「火加減のおかげなのか、隠し味のおかげなのか」を分けて議論しないと、改良のしようがありません。

2022年頃の拡散モデルはまさにこの状態でした。DDPM、スコアマッチング、連続時間SDE(確率微分方程式)など、似たことをしているのに見た目がまったく違う定式化が乱立し、論文ごとに「ノイズの足し方」「学習時に何を予測させるか(ノイズか元データか)」「生成時の数値解法」「ステップ数」が全部違っていました。だから次のことが分かりませんでした。

- ある論文が「品質が上がった」と言っても、その理由がノイズスケジュールなのか、前処理なのか、サンプラなのか切り分けられない。
- 拡散モデルは生成時にネットワークを数十回から数千回呼ぶため遅い。どこを削れば速くなるのかが体系化されていなかった。
- 学習を安定させる小技(入力のスケーリングや損失の重み付け)が論文ごとに経験則でバラバラだった。

EDM は、これらの工程を共通言語(連続時間の常微分方程式)に翻訳し、各工程を独立した「つまみ」として取り出しました。つまみを別々に回せるようになったので、何がどう効くかを系統的に調べられるようになったのです。

## 基礎: 前提となる概念
EDM を理解するには次の概念が要ります。順に噛み砕きます。

拡散モデルとは、きれいなデータ x にだんだんノイズを足して最後に完全な雑音にし、その逆をたどって雑音からデータを生成する仕組みです。ノイズの量を1つの数 σ(シグマ、足したガウスノイズの標準偏差)で表すと、σ=0 が「ノイズなし=きれいなデータ」、σ が大きいほど「ノイズだらけ」です。

デノイザ(denoiser、ノイズ除去関数)D とは、ノイズ入りデータ x とノイズ量 σ を受け取り、「元のきれいなデータはこれだったはず」と推定して返す関数です。拡散モデルの学習とは、このデノイザをニューラルネットワークで近似することにほかなりません。

スコア(score)とは、確率密度の対数の勾配 ∇ log p(x) のことで、「今いる点から、よりもっともらしいデータがある方向」を指すベクトルです。デノイザとスコアは数式上たがいに変換でき(Tweedie の公式)、デノイザ D が学べればスコアも分かります。

常微分方程式(ODE)とは、量が時間とともにどう変化するかを表す方程式です。拡散の生成は「σ を大きい値から 0 へ少しずつ動かす」過程なので、ODE を数値的に解くことに相当します。数値的に解くとは、連続的な変化を有限個のステップに区切って近似計算することです。このステップ数(ネットワークを呼ぶ回数)を NFE(number of function evaluations)と呼び、小さいほど生成が速くなります。

FID(Fréchet Inception Distance)とは、生成画像が本物の画像とどれだけ近い分布かを測る指標です。学習済み分類器の特徴空間で、生成画像群と本物画像群の特徴の平均・共分散の距離を計算します。小さいほど良く、0が完全一致です。

## 仕組みを詳しく
EDM の中心は、拡散の生成を次の連続時間 ODE(probability flow ODE と呼ばれる)で書くことです。

```
dx/dσ = -σ * ∇_x log p(x; σ) = (x - D(x; σ)) / σ
```

右辺は「現在の点 x から、デノイザの推定 D(x;σ) へ向かう方向」を表します。σ を σ_max から 0 へ減らしながらこの式を数値的に解くと、ノイズからデータが生成されます。EDM はこの枠組みの中で、互いに独立な4つの設計要素を取り出しました。

1. ノイズレベルの表し方(σ をそのまま時間座標に使う)
従来は時刻 t(0から1)を使い、t とノイズ量の対応が定式化に埋め込まれていました。EDM は σ を直接ノイズ座標として使うことで、スケジュール(どの σ を通るか)とサンプラ(どう数値計算するか)を切り離しました。これにより「σ のどこを細かく刻むか」を自由に設計できます。

2. 前処理(preconditioning、プレコンディショニング)
ニューラルネットワーク本体を F とすると、デノイザ D を次のように組み立てます。

```
D(x; σ) = c_skip(σ) * x + c_out(σ) * F( c_in(σ) * x ; c_noise(σ) )
```

- `c_in(σ)`: 入力をネットワークに入れる前に正規化する係数。σ=80 のような大ノイズでは x の値が大きく荒れるが、`c_in` で割ることでネットワークが見る入力の分散がほぼ1に保たれる。
- `c_skip(σ)`: 入力 x をそのまま出力に足すスキップ接続の重み。ノイズが少ないときは「入力をほぼそのまま返す」のが正解に近いので、その近道を用意する。
- `c_out(σ)`: ネットワーク出力をどれだけ効かせるかの係数。学習目標(ネットワークが当てるべき量)の分散が1になるよう決める。
- `c_noise(σ)`: σ をネットワークに渡すときの変換(EDM では (1/4)ln(σ))。

EDM はこれらを「ネットワークへの入出力と学習目標の分散がどの σ でもだいたい1になる」という原則から解析的に導出しました。たとえば画像 `[B, 3, 64, 64]`(バッチB枚、RGB、64x64)に σ=80 を足すと値域が大きく荒れますが、前処理で正規化されるためネットワークはどの σ でも同じスケールの入力を見ます。これが学習安定化の核心です。

3. 学習時のノイズ分布と損失の重み付け
学習では各サンプルにランダムな σ を足して「元に戻す」よう学ばせます。EDM は σ を対数正規分布(log-normal、対数を取ると正規分布になる分布)から引きます。具体的には ln(σ) が平均 P_mean=-1.2、標準偏差 P_std=1.2 の正規分布です。これは「中くらいのノイズ」を重点的に学習させる設定で、簡単すぎる(σ小)サンプルや難しすぎる(σ大)サンプルに学習リソースを取られすぎないための工夫です。損失の重み λ(σ) も、各 σ での損失の寄与が均等になるよう設計されます。

4. サンプラ(生成手順)の改良
生成は σ_max(EDM では 80)から σ_min(0.002)へ ODE を解いて進めます。刻み幅は等間隔ではなく次式で決めます。

```
σ_i = ( σ_max^(1/ρ) + (i/(N-1)) * ( σ_min^(1/ρ) - σ_max^(1/ρ) ) )^ρ ,   i=0..N-1
```

ρ(ロー)はだいたい7に設定します。これにより、仕上げに効くノイズの小さい領域を細かく、大きい領域を粗く刻みます。なぜ仕上げ側を細かくするかというと、ODE を有限ステップで近似するときの「離散化誤差(連続的な曲線を折れ線で近似したときのずれ)」は、解の曲がり方が急なところで大きくなり、拡散の場合この曲がりはノイズが小さい終盤に集中するからです。ρ=7 という値は、各ステップの局所誤差がほぼ均等になるよう経験的・解析的に選ばれています。ρ を大きくしすぎると終盤に刻みが偏りすぎて序盤が粗くなり、小さすぎると等間隔に近づいて終盤の誤差が増えます。

数値解法には Heun 法(2次の方法。1回の予測を1回補正する2段階法)を使います。Euler 法(1次)は「現在の傾き(速度)だけを使って次の点を予測する」ため、傾きが途中で変わると誤差が積もります。Heun 法は「いったん Euler で仮の次点を出し、そこでの傾きも測って、両端の傾きの平均で進み直す」ことで、1ステップあたり2回ネットワークを呼ぶ代わりに誤差のオーダーを1段下げます(局所誤差が刻み幅 h に対し O(h^2) から O(h^3) へ)。同じ最終品質なら全体の NFE を減らせるため、2回呼ぶコストを差し引いても得になります。

加えて、生成途中で少量のノイズを足し戻す確率的(stochastic)な手順も用意し、完全に決定的(deterministic)な手順との中間を選べるようにしました。仕組みは「いったん σ を少し増やすようにノイズを注入してから、目標の σ まで決定的に戻す」というもので、注入量を S_churn というパラメータで制御します。確率的サンプラは、ODE では一度入った離散化誤差が後段まで引きずられるのに対し、各ステップでノイズを足し直すことで誤差を「忘れさせる」効果があり、多様性とロバスト性を上げます。ただしノイズ注入量を増やしすぎると、足したノイズをデノイザが取りきれず細部がぼける・彩度が落ちるトレードオフがあります。論文では S_churn・S_min・S_max・S_noise の4つで注入の量と範囲を調整し、データセットごとに最適点が異なることを示しました。

## 手法の系譜と主要論文
EDM は突然生まれたのではなく、拡散モデルの定式化が整理されていく流れの集大成です。

- Sohl-Dickstein et al., "Deep Unsupervised Learning using Nonequilibrium Thermodynamics", ICML 2015。拡散の原型。熱力学の非平衡過程からデータ生成を着想した先駆。
- Ho, Jain, Abbeel, "Denoising Diffusion Probabilistic Models" (DDPM), NeurIPS 2020 (arXiv:2006.11239)。離散時間でノイズを足し引きする実用的な定式化を確立し、画像生成で大きな注目を集めた。EDM は「DDPM も σ を使えば自分の枠組みの特殊ケースとして書ける」と示した。
- Song et al., "Score-Based Generative Modeling through Stochastic Differential Equations", ICLR 2021 (arXiv:2011.13456)。拡散を連続時間 SDE/ODE で統一的に書き、probability flow ODE を導入した。EDM はこの連続時間視点を引き継ぐ。
- Karras, Aittala, Aila, Laine, "Elucidating the Design Space of Diffusion-Based Generative Models" (EDM), NeurIPS 2022。本ノートの主役。設計空間を ODE 枠組みに整理し、前処理・ノイズ分布・サンプラを独立に最適化。少ない NFE で当時最高品質を達成した。
- Karras et al., "Analyzing and Improving the Training Dynamics of Diffusion Models" (EDM2), CVPR 2024。EDM の続編。「分散を1に保つ」思想をネットワーク内部の重みやアクティベーションまで広げ、レイヤごとの重みを正規化して学習をさらに安定させた。後段で EMA(指数移動平均)の減衰率を学習後に事後的に選ぶ手法も提示。
- Karras らは EDM の知見を一貫性モデル(Consistency Models, Song et al. 2023)や蒸留にも展開しており、EDM の前処理・スケジュール設計は今や多くの後続研究のデファクト基盤になっている。

## 論文の実験結果(定量データ)
EDM の最大の貢献は、ネットワークの構造をほとんど変えずに、設計要素の整理だけで品質と速度を同時に改善したことです。

- CIFAR-10(32x32 の小さな画像6万枚のデータセット)で、条件なし生成 FID 1.97、条件付き生成 FID 1.79 を達成。これは当時の最高水準で、しかも NFE=35 という比較的少ない呼び出し回数で実現しました。従来の DDPM が高品質に数百から千ステップを要したのと比べ、桁違いに効率的です。FID が約2ということは、生成画像の特徴分布が本物とほぼ見分けがつかない水準を意味します。
- ImageNet 64x64 でも FID 1.36(条件付き)を報告し、当時の SOTA を更新しました。
- アブレーション(どの要素を抜くとどうなるか)が論文の白眉です。出発点となる既存モデル(VP/VE という従来のスケジュール)から、サンプラだけを EDM の Heun + ρ=7 スケジュールに替えると、ネットワークを再学習せずとも FID が大きく改善しました。たとえば既存の学習済みモデルに対し、サンプラ改良だけで CIFAR-10 の FID が 3 台から 2 台前半へ下がる、という結果が示されています。これは「サンプラは学習と切り離して後から差し替えられる」という EDM の主張の直接的な証拠です。
- さらに前処理(c_in/c_out/c_skip)と log-normal なノイズ分布を導入して再学習すると、FID が 1.9 台まで下がりました。各要素が独立に効くことを段階的に示した点が、設計空間を「解明した」と呼ばれる所以です。
- NFE と FID のトレードオフ曲線も示され、EDM の Heun サンプラは Euler 法(1次)より少ない NFE で同じ FID に到達することが確認されました。たとえば Euler で NFE=数百要した品質に、Heun では NFE=数十で到達します。

## メリット・トレードオフ・限界
- メリット: 設計要素を分離して個別最適化でき、改良の見通しが立つ。サンプラだけ差し替えるといった部分改善が可能。
- メリット: 前処理で入出力の分散を1に保つため学習が安定し、ハイパーパラメータに鈍感になる。
- メリット: σ ベースの刻み方と Heun 法で、少ない NFE で高品質。推論が速い。
- メリット: DDPM・スコアベースを特殊ケースとして包含し、既存手法との公平な比較ができる。
- トレードオフ: 係数の導出やノイズ分布のハイパーパラメータ(P_mean, P_std, ρ, σ_min, σ_max)が理論寄りで、初学者には式の直感が掴みにくい。
- トレードオフ: これらのハイパーパラメータはデータに依存する。画像向けに調整された値が、軌道や音声など別ドメインでそのまま最適とは限らず、再調整が要る。
- 限界: EDM は probability flow ODE を前提にしており、超少ステップ(1〜4 NFE)の領域では蒸留や一貫性モデルに譲る。EDM 単体では NFE を数ステップまで削ると品質が落ちる。
- 限界: 確率的サンプラのノイズ注入量(S_churn 等)は手動調整が必要で、設定を誤ると品質が劣化する。

研究上の未解決課題としては、高解像度(512px 以上)でのスケジュール最適化、ドメイン横断で自動的に良い σ 分布を選ぶ方法、ODE の離散化誤差をさらに減らす高次解法と確率的サンプラの統一的理解などが残っています。EDM2 はその一部に答えましたが、最適な設計空間の探索は今も続いています。

## 発展トピック・研究の最前線
EDM の枠組みは、その後の少ステップ生成研究の共通基盤になりました。一貫性モデル(Consistency Models)は EDM の ODE 軌道上の任意点から1ステップで終点へ飛ぶ写像を学び、1〜2 NFE での生成を可能にしました。これは EDM の前処理とスケジュールをそのまま使っています。また Rectified Flow や Flow Matching といった ODE ベースの生成は、EDM のノイズ座標の考え方と地続きであり、[Stochastic Interpolants](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0252_stochastic-interpolants) はこれらをさらに広い設計空間として数学的に統一します。EDM2 が示したレイヤ正規化と事後 EMA 選択は、大規模拡散モデルの学習レシピとして広く採用されています。
