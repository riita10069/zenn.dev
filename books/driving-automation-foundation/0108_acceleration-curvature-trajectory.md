# 加速度・曲率による軌道表現

## ひとことで言うと

車がこれからどう動くかを表す「軌道（trajectory）」を、地図上の点の並び（x座標, y座標）ではなく、各時刻の「加速度（どれくらい速度を変えるか）」と「曲率（どれくらい曲がるか）」のペアで表す方法です。たとえば64ステップ × (加速度, 曲率) を10Hz（1秒に10回）で並べれば6.4秒先までの運転計画になります。位置で表すよりも値の範囲が時刻によらず安定し、出てくる軌道が物理的に滑らかになりやすい、というのが要点です。

## 直感的な理解

人に「これから30メートル先まで、どこを通るか教えて」と頼むことを考えます。答え方は2通りあります。

ひとつは「1秒後はここ、2秒後はここ、3秒後はここ…」と地図上の点（座標）を読み上げる方法。もうひとつは「最初はそのまままっすぐ、3秒後から少しブレーキ、そのあと軽く右へ」と、操作の仕方を言葉で言う方法です。

前者がwaypoint（ウェイポイント、通過点）表現、後者が制御量に近い加速度・曲率表現です。後者のほうが運転している本人の感覚に近く、「あり得ない動き」を言いにくいことに気づきます。座標を直接読み上げると「1秒後に10m先、でも2秒後はなぜか真横に5m」のような、車には不可能なジャンプを口にできてしまいます。一方で「アクセルとハンドルをこう動かす」という言い方なら、出てくる経路は自然と連続的で滑らかになります。これが加速度・曲率表現の出発点にある直感です。

## 基礎: 前提となる概念

このトピックを理解するために、いくつかの言葉を噛み砕いておきます。

軌道（trajectory）とは、これから車がたどる道筋を時間順に並べたものです。自動運転では、知覚（カメラやLiDARで周囲を見る）→ 予測（他車・歩行者の動きを読む）→ 計画（自車がどう走るか決める）という流れの最後、「計画」の出力がこの軌道にあたります。

waypoint表現とは、未来の各時刻における車の位置 (x, y) を並べる方式です。「0.1秒後に (1.0, 0.0)、0.2秒後に (2.0, 0.1) …」のように、自車を原点とした座標系（自車の現在位置を (0,0)、進行方向をx軸とする座標系。これをego frame、自車座標系と呼びます）で点を並べます。直感的で、人間が地図を見るときの感覚に近い表現です。

加速度（acceleration）とは、単位時間あたりに速度をどれだけ変えるかです。単位はm/s²（メートル毎秒毎秒）。アクセルを踏めば正、ブレーキを踏めば負の値になります。乗用車が安全・快適に出せる縦加速度はおおむね -3〜+2 m/s² 程度で、急ブレーキでも -8 m/s² 前後が物理的限界です。つまり値の範囲がはっきり決まっています。

曲率（curvature）とは、進路がどれだけ曲がっているかを表す量で、ふつう κ（カッパ）という記号で書きます。曲率は「進路が描く円の半径 r の逆数」、すなわち κ = 1/r です。直進しているときは円の半径が無限大なので κ = 0。半径100mのゆるいカーブなら κ = 0.01 [1/m]、半径10mの急カーブなら κ = 0.1 [1/m] です。ハンドルの切り具合に対応すると考えると分かりやすいでしょう。

回帰（regression）とは、連続的な数値を予測する問題のことです。軌道予測は座標や加速度・曲率という連続値を当てるので回帰問題です。これに対し「信号は赤か青か」のような離散的な選択を当てるのは分類（classification）です。

ヨー角（yaw, 向き θ）とは、自車が進行する方向の角度です。x軸方向を0とし、左回りに測ります。曲率と速度から「向きがどれだけ変わるか」が決まり、向きと速度から「どこへ進むか」が決まる、という連鎖が軌道復元の核です。

## 仕組みを詳しく

加速度・曲率表現では、軌道を次のような形のテンソル（多次元配列）で持ちます。

```
trajectory: [B, T, 2]      ← 平らにすると [B, T*2]
              │  │  └ signals: (加速度 a, 曲率 κ)
              │  └ timesteps: T ステップ (例: 64)
              └ batch: 同時に処理するサンプル数
```

たとえば10Hz（0.1秒刻み）で64ステップなら6.4秒先までを、各ステップ2個の数値（加速度・曲率）で表すので、ひとつの軌道は 64 × 2 = 128 個の数値になります。サンプリング周波数 fを上げ刻み Δt = 1/f を細かくすると軌道は滑らかになりますが、同じ時間範囲を表すのにステップ数 T が増え、計算とメモリが線形に増えます。逆に Δt を粗くすると軌道は荒くなりますが軽量化します。この時間解像度の選び方も設計判断のひとつです。

### なぜ位置ではなくこの形にするのか

waypoint表現には、深層学習で扱うときの弱点が二つあります。

第一に、値のスケールが時間とともに大きく変わります。t=0 の位置はほぼ (0,0) ですが、6秒も走れば数十メートル先になります。回帰の損失（誤差の大きさ）は値が大きいほど大きく出るので、何も工夫しないと「遠い未来の誤差」ばかりが損失を支配し、肝心の近未来が雑になります。これは時刻ごとに重みを変える [軌道損失の時間重み付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0584_temporal-loss-weighting) でも扱う問題です。加速度・曲率なら、値の範囲が車の物理的限界（縦加速度はせいぜい ±数 m/s²、曲率は普通の道路で |κ|<0.2 程度）で頭打ちになり、時刻によらずほぼ一定のスケールに収まります。学習の入出力を一定スケールに正規化しておくことは勾配の安定に直結するので、この性質は大きな利点です。

第二に、waypointをそのまま予測すると、点の並びがガタガタした物理的にあり得ない軌道になりがちです。車は瞬間移動できず、加速や操舵には限界があります。waypoint表現はその制約を表現自体に含まないので、モデルが不連続な軌道を出しても損失上は気づきにくいのです。加速度・曲率なら、後述の積分で軌道を復元するため、出力は必然的に連続で滑らかになります。

### waypointへの復元（積分）

加速度・曲率の系列から実際の道筋を得るには、初期速度 v0・初期向き θ0 を起点に、時間方向へ順に足し合わせ（積分し）ていきます。Δtは1ステップの時間（0.1秒）です。

```
速度 v[t+1]   = v[t] + a[t] · Δt
向き θ[t+1]   = θ[t] + κ[t] · v[t] · Δt      (進んだ距離 v·Δt に曲率 κ をかけると向きの変化)
位置 x[t+1]   = x[t] + v[t] · cos(θ[t]) · Δt
位置 y[t+1]   = y[t] + v[t] · sin(θ[t]) · Δt
```

これは古典制御で使われる運動学的自転車モデル（kinematic bicycle model）の離散化と本質的に同じ式です。曲率 κ は前輪の操舵角 δ とホイールベース L（前後輪の間隔）を使って κ = tan(δ)/L と結びつくので、加速度・曲率の出力はそのまま「アクセルとハンドル」に翻訳でき、車両制御へ渡しやすいという実装上の利点があります。なお、この復元式はすべて微分可能（differentiable、出力が入力に対して滑らかに変化し勾配が計算できる）な四則演算と三角関数で構成されるため、PyTorch などの自動微分フレームワーク上で「積分層」として学習グラフに組み込めます。これにより制御量で出力しながら位置空間で損失を測る、という両取りが原理的に可能になります（後述の限界も参照）。

### 数値例で感覚をつかむ

ゆるく左へカーブしながら一定速度で走る場面を考えます。

```
waypoint 表現:        加速度・曲率表現:
(0.00, 0.00)          a=0.0, κ=0.02
(2.00, 0.02)          a=0.0, κ=0.02
(4.00, 0.08)          a=0.0, κ=0.02
(6.00, 0.18)          a=0.0, κ=0.02
 ...複雑に増加          ...ほぼ一定値が並ぶ
```

waypointでは座標が非線形に増えていきますが、加速度・曲率では「ほぼ一定値の繰り返し」という、モデルにとってずっと学びやすい表現になります。学習の難しさは「予測すべきターゲットがどれだけ単純な構造を持つか」に大きく左右されるので、この差は無視できません。直感的には、ニューラルネットは「ほぼ定数を出す」ことを「時刻ごとに増え続ける座標を出す」ことより容易に学べます。

## 手法の系譜と主要論文

end-to-end運転（カメラ画像などから直接、運転の指令を出すアプローチ）の出力形式は、歴史的に「制御量を出す派」と「軌道（位置）を出す派」の間を揺れ動いてきました。

ALVINN（Pomerleau, NeurIPS 1988）は最初期のend-to-end運転で、3層の浅いニューラルネットがカメラと距離センサの画像から「ハンドル角」という制御量を直接出していました。これは当時、車に直接指令を送りたかったための自然な設計です。

DAVE-2（Bojarski et al., NVIDIA, 2016, arXiv:1604.07316）も同じ系譜で、CNNがカメラ画像から操舵角を回帰します。約72時間の走行データで学習し、実車で一定の自律走行を達成しました。制御量を直接出す方式はシンプルですが、後述の累積誤差・分布シフトに弱いことが知られていきます。

Conditional Imitation Learning（Codevilla et al., ICRA 2018, arXiv:1710.02410）は、操舵・加減速といった制御量を出しつつ、「直進・右折・左折」という高レベルの指令（command）で条件づける枠組みを示しました。制御量出力の弱点を、明示的な意図入力で補おうとする流れです。

Learning by Cheating（Chen et al., CoRL 2019）や Learning to drive from a world on rails（Chen et al., ICCV 2021, arXiv:2105.00636）あたりから、運動学モデルを介して制御と軌道を橋渡しする発想が広く使われ始めます。出力を運動学的パラメータ（加速度・曲率や速度・操舵）で表し、それを積分して軌道に直す方式は、制御量の「物理的に妥当」という長所と、軌道の「滑らか・追従しやすい」という長所を両立させる中間解です。とくに World on Rails は、運動学モデルを前提に「将来の状態が現在の行動だけで決まる（環境は固定レール上にある）」という近似で価値関数を解析的に求め、サンプル効率を大きく上げました。

一方、UniAD（Hu et al., CVPR 2023 Best Paper, arXiv:2212.10156）や VAD（Jiang et al., ICCV 2023, arXiv:2303.12077）に代表される近年のモジュラーend-to-end系は、出力を位置（waypoint）で表します。理由は、評価ベンチマークが位置で定義されているため、位置で出して位置で測るのが指標と直結するからです。

近年の大規模end-to-end運転の研究では、軌道を (x,y) waypoint ではなく加速度・曲率などの運動学的パラメータで表すと学習が安定する、と報告するものが現れています。値のスケールが安定し、積分による滑らかさが保証され、運動学的制約に沿うため、というのがその根拠で、ALVINN以来の「制御量を出す」系譜を、運動学モデルで滑らかさを担保する形に洗練させたものと位置づけられます。

## 論文の実験結果（定量データ）

軌道表現そのものを単独で比較したアブレーション（ある要素だけを抜き差しして効果を測る実験）は多くありませんが、関連する定量的知見を挙げます。

UniAD（CVPR 2023）は nuScenes データセット（ボストンとシンガポールの市街地走行を集めた、計画・予測の標準ベンチマーク）で、計画の評価指標として L2 誤差（予測軌道と実際の人間の軌道の、各時刻における位置のずれをメートルで測ったもの。小さいほど良い）と衝突率（予測軌道どおりに走ると他の物体にぶつかる割合。小さいほど良い）を報告します。UniAD は1秒・2秒・3秒先の平均 L2 誤差を約 1.03m、平均衝突率を約 0.31% にまで下げ、当時の先行手法 ST-P3（平均 L2 約 2.11m, 衝突率約 1.27%）を大きく上回りました。ここで重要なのは、評価が位置で定義されているという事実です。加速度・曲率で学習しても最終評価は位置に直して測られるため、表現を選ぶときは「学習空間」と「評価空間」の食い違いを意識する必要があります。

VAD（ICCV 2023）は同じ nuScenes で UniAD と同等以下の L2 誤差を達成しつつ、推論速度を大幅に改善したと報告します（軌道や地図要素をベクトル化することで計算が軽くなる）。VAD-Base は平均 L2 約 0.72m・衝突率約 0.22%、軽量版 VAD-Tiny は約 16.8 FPS（1秒間に処理できるフレーム数）で、UniAD の約 1.8 FPS と比べ約9倍速いとされます。これは表現の選択が精度だけでなく計算コストにも効くことを示します。

運動学的パラメータ化の効果は、主に「出力の滑らかさ」と「快適性指標（jerk = 加加速度、加速度の時間変化。乗り心地の悪さに対応する量）」で現れます。位置を直接回帰したモデルは jerk が大きく出力がガタつきやすいのに対し、加速度・曲率を積分して軌道を作るモデルは jerk が構造的に小さく抑えられる、という定性的な傾向が複数の研究で報告されています。たとえば nuPlan などの閉ループ計画ベンチマークでは、快適性（縦・横の加速度と jerk が許容範囲に収まっているか）が明示的にスコア化され、運動学制約を表現に埋め込んだ手法が快適性スコアで有利になりやすいことが観察されています。ただし「同一条件で表現だけを差し替えた厳密なアブレーション」は分野全体でまだ不足しており、ここは未解決の検証課題です。

## メリット・トレードオフ・限界

メリット
- 値のスケールが時刻によらず安定し、損失が遠未来に偏らない。
- 積分で復元するため、出力軌道が必ず連続・滑らかで、物理的に実行可能になる。
- 運動学的制約（最大加速度・最大操舵）と整合し、車両制御へ直接渡しやすい。

トレードオフ・限界
- 損失を測る空間（加速度・曲率）と、最終的に重要な空間（位置・衝突）が一致しない。曲率や加速度の小さな誤差が、積分の過程で遠方では大きな位置ずれに増幅されることがある（誤差の積分的蓄積）。たとえば一定の曲率誤差 Δκ は、距離 s 進んだとき横方向に約 (1/2)·Δκ·s² のずれを生むので、s が大きい遠方ほど二次的に効きます。
- 初期速度 v0・初期向き θ0 の推定が必要で、それがずれると復元軌道全体がずれる。
- 教師データが waypoint 形式で配られる場合、微分・差分で加速度・曲率へ逆算する前処理が必要になり、二階差分はノイズを増幅しやすいので、その過程でノイズが乗りやすい。

研究上の未解決課題として、「位置で測る評価指標」と「運動学パラメータで学ぶ表現」のギャップをどう埋めるか（例: 微分可能な積分層を学習グラフに入れて位置損失も同時に最適化する）、また表現選択の効果を厳密に切り分けるアブレーション設計が挙げられます。

## 発展トピック・研究の最前線

近年は、軌道を1本の決定論的な系列で出すのではなく、複数候補（multi-modal、複数の起こりうる運転）を確率分布として出す方向が主流化しています。MultiPath++（Varadarajan et al., ICRA 2022）や Wayformer（Nayakanti et al., ICRA 2023, arXiv:2207.05844）は混合分布で複数軌道を出し、加速度・曲率のような運動学パラメータを各モードで予測する設計とも組み合わせられます。

さらに、拡散モデル（diffusion model、ノイズから徐々にデータを生成する生成モデル）で軌道を生成する研究や、言語モデルと結合して「なぜその軌道か」を説明させる Vision-Language-Action 系の運転モデルも登場しており、そこでも出力をトークン化された運動学パラメータで表す試みが進んでいます。表現の選択は、単なる損失の安定性を超えて「生成モデルや言語モデルとどう接続するか」という設計問題に発展しつつあります。とくに運動学パラメータは値域が有界で離散トークンに量子化しやすく、自己回帰的な系列生成と相性が良いという性質が再評価されています。
