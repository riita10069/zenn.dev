# 擬似投影（pseudo-projection）フォールバック

## ひとことで言うと

本来 BEV を作るにはカメラの位置・向きの正確な情報（キャリブレーション）が必要ですが、それが手元にない場面でもとりあえずモデルを動かせるようにするための代替機構です。本物の投影行列の代わりに、学習で更新される行列を使います。ただしこれは真のカメラ幾何を学ぶものではなく、「サンプリング場所のクセを学習した固定の手がかり（空間的事前知識）」にすぎない、という性質を理解しておくことが決定的に重要です。本番運用ではなく、開発・検証・アブレーションのための足場と位置づけるべきものです。

## 直感的な理解

[ピンホール投影による3D→2D変換](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0998_pinhole-projection-3d-to-2d) で見たとおり、3D点が画像のどこに写るかを計算するには、内部パラメータと外部パラメータをかけ合わせた投影行列が必要です。これはカメラのキャリブレーション（較正＝焦点距離や取り付け位置・角度を正確に測った値）から得られます。

ところが現実には、この情報がいつも揃っているとは限りません。データセットによってはキャリブレーション値が付属していない、アーキテクチャの動作確認（テンソルの形が合うか、学習が回るか）をしたいだけで正しい幾何はまだ要らない、あるいはアブレーション（ある要素を抜いて影響を調べる実験）として「キャリブを使わないとどうなるか」を見たい、といった場面です。こうした局面で「キャリブがないと一切動かない」では開発も研究も止まってしまいます。

そこでフォールバック（fallback＝本命が使えないときの代替手段）として、キャリブがなくても動く投影の代わりを用意します。発想は単純で、「3D点を画像座標に写す行列」を固定の物理量として与える代わりに、学習可能なパラメータとして持ち、データから当てはめてしまうのです。これは「正しい地図の作り方を知らないので、とりあえず経験から各マスがだいたいどのへんを見ればよさそうかを覚える」のに近い割り切りです。

## 基礎: 前提となる概念

学習可能パラメータ（learnable parameter）。学習（誤差逆伝播）によって値が更新される変数です（PyTorch では `nn.Parameter`）。本物のキャリブは外部から与える固定値ですが、擬似投影では投影行列そのものをこの学習可能パラメータにします。小さなランダム値（たとえば標準偏差0.01程度）で初期化するのが定石で、最初から大きな値だと出力が極端な座標になり学習が不安定になるためです。

空間的事前知識（spatial prior）。「データを見る前から持っている、空間配置についての偏った思い込み」です。たとえば「前方のマスは画像中央付近を見ることが多い」といった傾向を、学習を通じて固定的に身につけたものを指します。重要なのは、これは個々のシーンの幾何を正しく解くのではなく、平均的な傾向を覚えるだけだという点です。

シグモイド関数（sigmoid）。どんな実数を入れても 0〜1 の値を返す滑らかな関数です。`σ(x) = 1 / (1 + e^{-x})` で、x が大きいほど1に、小さいほど0に近づきます。出力範囲が保証されていない値を、強制的に 0〜1 の座標へ押し込むのに使います。

可微分サンプリング（differentiable sampling）。画像のある座標から特徴を読み取る操作を、勾配が通る（学習できる）形で行うことです。Spatial Transformer Networks（後述）が確立した `grid_sample` 系の操作がこれにあたり、擬似投影が出した座標で画像を読む際にもこの仕組みを使います（[F.grid_sample による可搬な変形アテンション](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0968_grid-sample-deformable-attention) 参照）。

## 仕組みを詳しく

擬似投影はモジュールの初期化時に、学習可能なパラメータとして用意します。形状はカメラごとに `[3, 4]`、つまり本物の投影行列と同じ形にしておくと、コードの分岐が最小限で済みます。

```
pseudo_projection = 学習可能パラメータ, 形状 [V, 3, 4],  小さなランダム値 × 0.01 で初期化
   V : カメラ台数
```

投影の計算自体は本物と同じ流れ（同次座標化 → 行列積）ですが、決定的に違うのが投影後の正規化のしかたです。本物のキャリブのときはピクセル座標を画像サイズで割って 0〜1 にします。これは物理的に意味のある正規化です。一方、擬似投影は出力がどんな値になるか保証がない（unbounded＝範囲が決まっていない）ので、シグモイド関数で強制的に 0〜1 に押し込みます。

```
本物:   ref_2d = [u, v] / image_size      （物理的に意味のある正規化）
擬似:   ref_2d = sigmoid([u, v])           （範囲を強制的に 0〜1 に）
```

ここが本質的な注意点です。本物の投影は「3D空間のこの点は、幾何的に画像のここに写る」という物理的根拠があります。しかし擬似投影は「学習可能な行列 × 3D座標 → シグモイド」で座標が出てくるだけなので、3Dと2Dの対応に物理的な意味はありません。

```
本物の投影:  3D点 ──(物理的に正しい幾何)──> 画像上の正しい位置
擬似投影:    3D点 ──(学習した当てはめ)────> 画像上の「だいたいこの辺」
```

つまり擬似投影は「各 BEVマスが画像のどのあたりを見るか」という偏り（空間的事前知識）をデータから学ぶだけで、各マスと画像位置が正しく対応するわけではありません。学習が進むと、平均的なシーンに対しては「前方マスは画像中央を見る」程度のもっともらしい傾向を身につけますが、これは固定された当てはめであり、カメラ配置が変わったり物体の高さが変わったりすると正しく追従できません。さらに本質的な問題として、擬似投影の行列は入力画像に依存しない固定値なので、シーンごとに異なる物体配置を区別できません。シーンが変わってもサンプル位置は同じで、変わるのは読み取った特徴だけです。これは透視除算（深度に応じてサンプル位置が変わる本物の投影）が持つ表現力を根本的に欠いていることを意味します。

実務上の落とし穴として、ランダムなテンソルを入力にした動作確認では、擬似投影経路で損失（loss）が NaN（非数）になりやすいことが知られています。これはコードの欠陥ではなく、意味のない座標 × 意味のない入力の組み合わせで数値が不安定になりやすいことを示します。実データでの安定性は別途確認が必要で、本番運用には真のキャリブを与えることが前提です。

## 手法の系譜と主要論文

擬似投影は特定の論文が名前を付けた手法ではなく、実装上の現実的な工夫です。ただし、その背景にある「投影やサンプリング位置を学習で決める」という発想は、いくつかの確立した研究と地続きです。

対照的な立場にあるのが BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）です。BEVFormer は「正確なキャリブレーションがあること」を前提に、3D参照点を本物の投影行列で画像に落とします。この前提を置けたのは、評価に使う自動運転データセットが高精度なキャリブを提供しているからです。効果は幾何的に正しい BEV、トレードオフはキャリブが必須でキャリブのないデータには使えないことです。擬似投影はこの「キャリブ必須」というトレードオフを、開発・実験の局面で緩めるための回避策と位置づけられます。

「サンプル位置を学習で決める」発想の代表が Deformable DETR（Zhu et al., ICLR 2021, arXiv:2010.04159）です。Deformable DETR はクエリから「どこをサンプルするか」のオフセットを学習で予測します。動機は、全画素に注意を向ける計算を避けて少数の関連位置だけを見ることで、計算を軽くし収束を速めることでした。効果は高速化（学習エポック数を大幅削減）と精度向上、トレードオフはサンプル位置が学習に依存し解釈しにくいことです。擬似投影も「サンプル位置を学習で決める」点は共通しますが、決定的な違いがあります。Deformable DETR は幾何（本物の参照点）の上に小さなオフセットを足すのに対し、擬似投影は幾何そのものを学習に置き換えてしまいます。だから物理的意味が失われるのです。

サンプリング操作そのものの理論的基盤は Spatial Transformer Networks（Jaderberg et al., NeurIPS 2015, arXiv:1506.02025）にあります。STN は、画像をネットワーク内部で可微分に変換・サンプリングする仕組みを示しました。具体的には変換パラメータ（アフィン変換など）を別のサブネットワークが予測し、その変換に従ってサンプリンググリッドを生成して画像を読み直します。擬似投影が出した座標で画像から特徴を読む処理は、この可微分サンプリングを使っており、だからこそ投影行列を勾配で学習できます。投影行列をデータから学ぶというアイデアは、STN が「変換そのものを学習可能にした」延長線上にあると見ることができます。

明示的な投影サンプリングを避ける PETR（Liu et al., ECCV 2022, arXiv:2203.05625）も、思想として近い面があります。PETR は3D座標を位置埋め込みに変換して画像特徴へ注入し、サンプリング座標の明示計算を避けます。擬似投影と PETR はいずれも「投影を陽に解かずに済ませる」点で通じますが、PETR は位置埋め込みに本物のカメラパラメータを使うのに対し、擬似投影はパラメータごと学習に委ねる、という違いがあります。つまり PETR は幾何を捨てておらず、擬似投影だけが幾何を完全に手放している、という対比になります。

## 論文の実験結果（定量データ）

擬似投影そのものを主題にしたベンチマーク数値は存在しませんが、関連手法の実験が、その性質と限界を間接的に教えてくれます。

Deformable DETR は COCO 物体検出で、元の DETR が必要とした500エポックの学習を約50エポックに短縮しつつ、AP を 43〜46 程度へ向上させたと報告されました。これは「サンプル位置を学習で適応させると収束が速く精度も上がる」ことの定量的証拠です。ただしこれは本物の参照点の上にオフセットを足す設計であり、幾何の土台が残っている点が擬似投影と異なります。擬似投影には足し算する土台の幾何がそもそも無いため、この高速収束の恩恵は期待しにくいと考えられます。

キャリブレーションの有無が精度に与える影響については、BEV 認識の研究で繰り返し示されてきました。報告によると、外部パラメータに数度の誤差を入れるだけで遠方物体の mAP が目に見えて低下します。擬似投影は実質的に「キャリブ誤差が常に大きい状態」に相当するため、幾何的に正しい BEV を要求するタスク（正確な距離推定、地図合わせ）では本物のキャリブ版に対して明確に劣ると予想されます。一方で、各マスがおおよそ妥当な画像領域を見る程度の弱い事前知識でも、形状検証や粗い分類のような緩いタスクは動作します。これは「どこまでの正確さがタスクに必要か」によって擬似投影の許容範囲が変わることを意味します。

## メリット・トレードオフ・限界

メリット。カメラキャリブレーションがなくても BEV モジュールを動かせるので、開発が止まりません。テンソル形状の検証やアーキテクチャのアブレーションに使えます。本物の投影と同じ形状にしておけば、コードの分岐が最小限で済みます。学習可能なので、データの平均的な傾向に対しては多少のもっともらしさを獲得できます。

トレードオフと限界。3Dと2Dの対応に物理的な意味がないため、真の BEV にはなりません。投影行列が入力画像に依存しない固定値なので、シーンごとの違いを区別できず、透視（深度依存）の表現力を欠きます。ランダム入力との組み合わせで数値が不安定になり NaN を出しやすいです。これを本番のつもりで使うと、見た目は動くのに地図が幾何的に正しくない、という気づきにくい落とし穴になります。学習で平均的なクセを覚えるだけなので、学習時と異なるカメラ配置や視野には汎化しません。研究上の未解決課題として、「キャリブをどこまで学習で代替できるか」「弱いキャリブ（不正確な初期値）からどこまで自己較正で回復できるか」は依然オープンであり、擬似投影はその素朴な下限（lower bound）を示す存在と言えます。

## 発展トピック・研究の最前線

擬似投影の素朴さを超える方向として、self-calibration（自己較正）の研究があります。これは投影行列をランダムから学ぶのではなく、不正確な初期キャリブから出発し、画像と幾何の整合性（再投影誤差の最小化など）を手がかりに本物のキャリブへ近づけていく手法です。擬似投影が「幾何を完全に放棄して学習に丸投げ」するのに対し、self-calibration は「幾何の枠組みを残したまま、ずれた値だけを学習で直す」点で、より原理的です。

キャリブ不要の認識を目指す研究も進んでいます。複数視点の幾何的整合性を自己教師あり学習で獲得したり、カメラパラメータを推定するサブネットワークを本体に組み込んだりして、明示的なキャリブ入力なしに3D理解を行う試みです。

より広い視点では、「物理的な帰納バイアス（幾何）をどれだけ陽に組み込み、どれだけデータから学ばせるか」という、深層学習全体の中心的な問いに擬似投影は位置します。幾何を完全に与えれば正確だがデータの制約に縛られ、完全に学習に委ねれば柔軟だが汎化と安定性を失う。この連続体のどこに最適点があるかは、データ量・タスク要求・キャリブ取得コストに依存し、現在も研究が続いています。擬似投影は、その連続体の「学習に丸投げ」側の端点を体現する、教育的にも有益な参照点です。
