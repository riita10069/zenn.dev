# Map-BEV融合戦略（residual等）

## ひとことで言うと
Map-BEV 融合戦略とは、カメラから作った俯瞰図(画像 BEV)と、地図から作った俯瞰図(地図 BEV)を「どう混ぜ合わせるか」を決める部分を、プラグインのように差し替え可能にした設計のことです。混ぜ方には「残差(足し算ベース)」と「クロスアテンション(互いに注目し合う方式)」など複数の選択肢があり、名前(キー)で選べるようにレジストリ(キーと実装クラスを対応づける辞書)に登録しておきます。融合のやり方そのものをモジュール化して交換できる、というのが要点です。

## 直感的な理解
分離設計([Map Encoder分離設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0979_map-encoder-separation))により、カメラレーンと地図レーンは別々に処理され、最後に同じ形 `(B, embed_dim, H, W)` の2つの俯瞰図特徴になります。ここで残る問題が「この2つをどう1つにまとめるか」です。

混ぜ方には唯一の正解がありません。たとえば次のような選択肢があります。
- 単純に足す: `image_bev + map_bev`。軽いが、地図がいつも同じ重みで足され、調整が効かない。
- 重み付きで足す: `image_bev + α × map_bev`。地図の効き具合 α を学習で調整できる。
- 連結して畳み込む: チャネル方向に並べてから 1×1 畳み込みで混ぜる。チャネル間の相互作用を学習できるが、パラメータが増える。
- 互いに注目し合わせる: 画像 BEV の各位置が「地図 BEV のどこを見るべきか」を内容に応じて動的に計算して取り込む。表現力は高いが計算が重い。

どれが良いかはデータ・タスク・計算予算しだいで、実験して比べる必要があります。そこで「融合のやり方そのものをモジュール化して差し替えられる」ようにします。融合方式を変えるたびにモデル本体を書き換えるのではなく、レジストリのキーを変えるだけで切り替えられる。これにより、A/B 比較や系統的なアブレーション(どの方式が効くか抜き差しして検証)が一気に楽になります。

## 基礎: 前提となる概念
- 残差接続(residual connection): 出力を `output = x + F(x)` の形にする構造。`F(x)` は学習する変換、`x` はそのまま通す「近道(skip)」。深いネットワークでも勾配が消えにくくなり、学習が安定します。
- ゲート / スケール係数: 残差枝に掛ける学習可能な係数 α。`output = x + α·F(x)`。α を小さく始めると、最初は近道だけが効く(=元の挙動に近い)状態から学習を始められます。
- 恒等写像(identity mapping): 入力をそのまま出す変換。`α = 0` のとき `output = x` になり、追加した枝が一切寄与しない初期状態を作れます。
- ブロードキャスト(broadcasting): 形の違うテンソル同士の演算で、小さい方の形を自動で引き伸ばして合わせる仕組み。`(embed_dim,)` の係数を `(1, embed_dim, 1, 1)` に整形すれば、全空間位置・全バッチに同じチャネル係数を掛けられます。
- アテンション(attention): あるデータ列(クエリ)が、別のデータ列(キー)との関連度を計算し、その重みでバリュー(値)を加重平均して取り込む仕組み。数式では `softmax(QKᵀ/√d)·V`。Q はクエリ、K はキー、V はバリュー、d は次元数(√d は内積が大きくなりすぎないための正規化)。クエリとキー・バリューが別系列なら「クロスアテンション(cross-attention)」と呼びます。
- マルチヘッド(multi-head): アテンションを複数組(ヘッド)並列に計算し、異なる観点の関連を同時に捉える仕組み。例えば8ヘッドなら8通りの注目を平行して学びます。
- LayerNorm / Pre-norm: LayerNorm は特徴を正規化(平均0・分散1付近にそろえる)する層。これをアテンションや FFN の「前」に置く構成を Pre-norm と呼び、深い Transformer の学習を安定させます。
- FFN(Feed-Forward Network): Transformer ブロック内の小さな全結合ネット(`Linear → 活性化 → Linear`)。位置ごとに非線形変換を加えます。
- 系列長(sequence length): アテンションが扱う要素数。BEV を平坦化すると系列長は `H×W` になります。アテンションの計算量・メモリはこの系列長の2乗に比例します。

## 仕組みを詳しく
典型的な実装は、レジストリと複数の融合クラスから成ります。

レジストリ(差し替えの仕組み)の擬似コード:
```
MAP_FUSION_REGISTRY = {
    "residual": ResidualMapFusion,
    "cross_attn": MapCrossAttentionFusion,
}

def build_map_bev_fusion(fusion_mode, embed_dim=256, **kwargs):
    # fusion_mode が未登録なら、使える選択肢を列挙した分かりやすいエラーを出す
    return MAP_FUSION_REGISTRY[fusion_mode](embed_dim=embed_dim, **kwargs)
```
`build_map_bev_fusion("residual")` のように名前を渡すと、対応するクラスがインスタンス化されます。未知の名前には「使えるのは residual, cross_attn」と教えます。

方式1: 残差・ゲート付き足し算(ResidualMapFusion)
```
output = image_bev + alpha * map_bev
```
- `alpha` はチャネルごとに1つ持つ学習可能パラメータ(形 `(embed_dim,)`、つまり例えば256個)。`view(1, -1, 1, 1)` で `(1, embed_dim, 1, 1)` に整形し、ブロードキャストで全位置に適用します。
- 最大のポイントは `alpha` をゼロで初期化すること。学習開始時は `alpha = 0` なので `output = image_bev + 0 × map_bev = image_bev` となり、地図レーンは一切寄与しません。つまり「最初は地図なしのモデルと完全に同じ挙動」から始まります。学習が進むにつれ、役立つチャネルの `alpha` だけが0から育ち、地図情報が少しずつ取り込まれます。
- これは「ゼロ初期化残差」という安定化テクニックです。新しい枝(地図枝)をいきなり強く足すと、初期の出力分布が大きく崩れて学習が不安定になりがちですが、ゼロから始めれば既存の挙動を壊さず、安全に新情報を学べます。チャネルごとに α を持たせることで、「どのチャネルでどれだけ地図を取り込むか」を細かく学習できます。

方式2: クロスアテンション(MapCrossAttentionFusion)
- 画像 BEV をクエリ、地図 BEV をキー&バリューとして、マルチヘッドアテンション(複数の注意を並列に計算する、例えば8ヘッド)で「画像 BEV の各位置が地図 BEV のどこを見るべきか」を計算します。
- 処理の流れ(tensor shape つき):
  1. `image_bev (B, C, H, W)` を `(B, H*W, C)` に平坦化してクエリ q にする。地図も同様に kv にする。`H*W` が系列長(位置の数)になります。
  2. Pre-norm + 残差: `q = q + cross_attn(norm(q), norm(kv), norm(kv))`。
  3. FFN + 残差: `q = q + ffn(norm(q))`。
  4. `(B, H*W, C)` を `(B, C, H, W)` に戻して出力。
- 残差方式が「全位置に同じチャネル重みで地図を足す」のに対し、クロスアテンションは「位置ごとに、地図のどこを参照するかを内容に応じて動的に決める」ので表現力が高いです。その代わり `H*W × H*W` の注意行列が必要で、BEV が大きい(例: 450×300 なら `H*W = 135000`)と、注意計算は `135000² ≒ 1.8 × 10¹⁰` 要素規模になり、計算量とメモリが急増します。これが高解像度 BEV で OOM(メモリ不足)を招く主因です。

両方式とも入出力は `(B, embed_dim, H, W)` でそろっているので、モデル本体は `fused = map_bev_fusion(image_bev, map_bev)` と呼ぶだけで、中身がどちらの方式かを意識しません(疎結合、[kwargs転送によるモジュール疎結合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0973_kwargs-forwarding-design) の一例)。デフォルトを残差にするのは「最初は地図なしと同じ挙動から安全に学習を始められる」というゼロ初期化の利点を重視した設計判断です。

## 手法の系譜と主要論文
2つの方式は、それぞれ深層学習の代表的な融合・安定化手法の系譜に対応します。

- 残差接続とゼロ初期化スケール。He ら "Deep Residual Learning for Image Recognition"(2016, CVPR)が `output = x + F(x)` の残差接続で、それまで学習困難だった超深層ネット(152層など)の学習を可能にし、ImageNet で当時の最先端を達成しました。続いて ReZero(Bachlechner ら, 2020)や Fixup / SkipInit 系の研究が、「残差枝に学習可能なスカラーを掛け、それをゼロ初期化すると、最初は恒等写像から始まり学習が安定し収束も速くなる」ことを示しました。動機は「新しい枝を最初は無にして恒等写像から始めると勾配が安定する」こと。効果は学習の安定化と高速化。トレードオフは「足し算ベースなので、位置ごとに参照先を変えるような細かい制御はできない」点です。残差融合の α=0 初期化はまさにこの系譜で、チャネルごとにスカラーを持つ点が拡張になっています。

- アテンションとクロスアテンション。Vaswani ら "Attention Is All You Need"(2017, NeurIPS)が、あるデータ列(クエリ)が別のデータ列(キー&バリュー)のどこを見るべきかを学習する仕組みを確立し、機械翻訳で当時の最先端を達成しました。BEVFormer(Li ら, 2022, ECCV)はこれを自動運転の BEV 生成に応用し、空間クロスアテンションで多視点カメラ画像を BEV グリッドに集約しました。動機は「固定の足し算より、内容に応じて動的に情報を選ぶ方が表現力が高い」こと。効果は精度向上。トレードオフは「系列長の二乗に比例する計算量とメモリ」です。クロスアテンション融合は Pre-norm + 残差 + FFN という標準的な Transformer ブロック構成を踏襲しています。

- 二乗コストを抑える変形アテンション。Deformable DETR(Zhu ら, 2021, ICLR)は、全位置を見るのではなく、各クエリが学習した少数(例: 4〜8点)の参照点だけに注目する「変形アテンション(deformable attention)」を提案しました。これにより注意コストが系列長の二乗から線形近くまで下がります。BEVFormer もこれを採用し、高解像度 BEV を現実的なコストで扱えるようにしています。高解像度 BEV 融合では事実上の定石です。

- Pre-norm 構成。Xiong ら "On Layer Normalization in the Transformer Architecture"(2020, ICML)が、LayerNorm をアテンションや FFN の「前」に置く Pre-norm 構成にすると、勾配のスケールが安定し、学習率のウォームアップなしでも収束しやすくなることを理論と実験で示しました。クロスアテンション実装が norm を前置するのはこの知見に沿っています。

これらに共通する教訓は「2つの情報を統合する方法は、軽くて安定だが表現力が限られる足し算系と、表現力は高いが重いアテンション系という対極があり、設計者は精度と計算予算のトレードオフの中で選ぶ」ということです。

## 論文の実験結果(定量データ)
融合方式の選択がどれだけ効くかを、代表論文の定量結果で確認します(数値はおおよそ)。

- 残差接続の効果(ResNet)。ImageNet 分類で、残差接続のないプレーンな34層ネットは、18層ネットより学習誤差が高くなる「劣化問題(degradation)」を示しました。残差接続を入れた ResNet-34 は同じ深さで誤差が下がり、さらに ResNet-152 まで深くしても性能が向上、ImageNet で top-5 誤差 約3.57% を達成し ILSVRC 2015 で優勝しました。要点は「深くするほど良くなるはずが、残差なしでは逆に悪化する。残差はその劣化を解消する」という定量証拠です。

- ゼロ初期化スケールの効果(ReZero / Fixup)。ReZero(Bachlechner ら, 2020)は、各残差枝に α(初期値0)を掛けるだけで、正規化やウォームアップに頼らず非常に深い Transformer(数百〜千層規模)を学習でき、収束が大幅に速くなることを示しました。報告では、12層 Transformer の言語モデルで、ReZero により収束までのイテレーション数が従来比でおよそ数十%短縮された例が挙げられています。これが「新しい枝はゼロから足す」という設計の妥当性を裏づけます。

- BEV 生成におけるクロスアテンション(BEVFormer)。nuScenes で3次元物体検出を評価。指標は NDS と mAP。BEVFormer は空間クロスアテンション(多視点画像から BEV を集約)と時間自己アテンション(過去 BEV を参照)を組み合わせ、当時の純カメラ手法で最先端の NDS(約0.56)を達成しました。アブレーションでは、空間クロスアテンションを単純な投影に置き換えると検出精度が落ち、時間アテンションを外すと特に速度推定(物体の動き)の精度が悪化することが示され、「動的に参照先を選ぶ」アテンションの寄与が定量化されました。一方で、アテンションは計算が重く、解像度や参照点数を増やすと精度は上がるが速度が落ちるという、表現力とコストの明確なトレードオフも報告されています。変形アテンションの採用により、この二乗コストを実用範囲に抑えている点も重要です。

これらが示すのは、足し算系(残差)は安定性と軽さで、アテンション系は表現力で勝り、どちらを選ぶかは精度要求と計算予算で決まる、という点です。ゼロ初期化残差は「まず安全に始め、必要なだけ地図を取り込む」という保守的だが堅実な選択、クロスアテンションは「計算を払って位置依存の精緻な融合を狙う」攻めの選択にあたります。

## メリット・トレードオフ・限界
メリット:
- 融合方式を本体コードに触れずキー1つで差し替えられ、どの方式が良いかを実験で比較しやすい。
- 残差はゼロ初期化により「地図なしモデルと同じ状態」から安全に学習を始められ、計算も軽い。
- クロスアテンションは位置ごとに地図参照を動的に決められ、表現力が高い。
- 入出力の形が統一されているので、本体は融合の中身を知らずに済む(疎結合)。

トレードオフ・限界:
- 残差は足し算ベースなので、位置依存の細かい融合(ここだけ地図を強く見たい等)はできない。
- クロスアテンションは系列長(`H*W`)の二乗で計算量・メモリが増え、高解像度 BEV では重い・OOM になりうる。緩和には変形アテンション(参照点を少数に絞る)やウィンドウ分割が使われます。
- 方式が増えるほど、どれを選ぶべきかの判断と実験コストが増える。
- 両方式とも image_bev と map_bev のチャネル数・空間サイズが完全一致している前提で、ずれると動かない。
- 残差の α は学習されるが、地図が間違っているシーンでも一律に同じ α が掛かるため、「このフレームは地図を信じるな」という動的な抑制は表現できない。

研究上の未解決課題として、(1) 地図の信頼度に応じて融合の重みを動的に変える(地図が間違っていそうなときはカメラを優先する)頑健な融合、(2) 高解像度 BEV でアテンションを現実的なコストに収める効率化、(3) 足し算とアテンションの中間にある「学習可能な疎なゲート」など、表現力とコストの良い折衷点の探索、などがあります。

## 発展トピック・研究の最前線
- 変形アテンション(Deformable Attention)。Deformable DETR(Zhu ら, 2021, ICLR)由来の、全位置ではなく学習した少数の参照点だけに注目する方式。BEVFormer もこれを採用し、二乗コストを実用範囲に抑えています。高解像度 BEV 融合の定石です。
- ゲート付き・条件付き融合。地図の不確実性やシーン文脈に応じて融合の強さを変える条件付きゲート。FiLM(Perez ら, 2018, AAAI)のように、別の信号からスケールとシフトを生成して特徴を変調する手法も、入力依存の融合制御として関連します。地図が劣化・欠損する場面での頑健性を狙います。
- クロスモーダル統一 Transformer。カメラ・LiDAR・地図を1つの Transformer 内のトークンとして扱い、自己アテンションで一括融合する流れ(統一 BEV 表現)。融合方式を個別に選ぶのではなく、注意機構が自動的にモダリティ間の重み付けを学ぶ方向です。
- 残差スケールの理論。ReZero / Fixup の延長で、初期化スケールと学習率・深さの関係を理論的に解析し、正規化を使わず安定学習する設計が研究されています。融合枝のゼロ初期化はこの理論の実応用です。
