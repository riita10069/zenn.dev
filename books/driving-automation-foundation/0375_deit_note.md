# DeiT (Data-efficient Image Transformer)

## ひとことで言うと
DeiT は、Vision Transformer (ViT) が抱えていた「超大規模な非公開データがないと CNN に勝てない」という致命的な弱点を解消し、ImageNet-1K (約130万枚) という誰でも使える標準サイズのデータだけで ViT を強く学習できるようにした手法です。鍵は2つ。徹底した強力なデータ拡張・正則化のレシピと、「蒸留トークン」という新しい仕組みで CNN 教師の知識を ViT に注入するアイデアです。これにより ViT は一握りの巨大組織の技術から、研究コミュニティ全体が使える実用技術へと「民主化」されました。

## 直感的な理解
ViT は「思い込み (帰納バイアス) を捨ててすべてをデータから学ぶ」モデルでした。思い込みがない分だけ、画像とはどういうものかをゼロから学ぶのに膨大な経験 (データ) を要します。例えるなら、街のルールを何も知らない子どもに運転を教えるとき、教科書 (帰納バイアス) なしで100万回の実走だけから学ばせるようなものです。CNN は最初から「近くのものは関係する」「位置がずれても同じ物体」という教科書を持っているので、少ない経験でそこそこ運転できます。

DeiT のアイデアは2段構えです。1つは「同じ画像を毎回違う見え方で見せて、実質的な経験量を水増しする」こと (データ拡張)。1枚の画像を回転させたり色を変えたり一部を隠したりして、無限に近いバリエーションを作ります。もう1つが「すでに運転が上手な先生 (学習済み CNN) の運転を真似させる」こと (知識蒸留) です。先生の判断を見ながら学ぶと、ゼロから学ぶより遥かに速く上達します。DeiT はこの2つを組み合わせ、巨大データなしで ViT を一人前にしました。

## 基礎: 前提となる概念
まず ViT の弱点の根を押さえます。[Vision Transformer (ViT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0411_vision-transformer) は ImageNet-1K だけで学習すると同規模 CNN に負け、CNN を上回るには JFT-300M (Google 社内の約3億枚、非公開) のような超大規模事前学習が必要でした。これは潤沢な計算資源と独自データを持つ組織でないと再現できず、多くの研究者には手が届きませんでした。原因は帰納バイアスの欠如です。

次に知識蒸留 (knowledge distillation) を理解します。これは、すでに賢い「教師モデル」の出力を真似させて「生徒モデル」を効率よく学習させる手法です。Hinton らが2015年に提案しました。普通の教師あり学習では正解ラベル (例: この画像は「猫」) だけを使いますが、蒸留では教師モデルが出す確率分布 (例: 猫90%・犬8%・キツネ2%) も使います。この「猫だが少し犬にも似ている」という柔らかい情報 (ソフトラベル) には、正解ラベルにない豊かな知識が含まれており、生徒はそこから効率よく学べます。

データ拡張 (data augmentation) は、訓練画像に人工的な変換を施して見かけのデータ量を増やす技法です。過学習 (訓練データを丸暗記して未知データに弱くなること) を防ぐ最も基本的な手段で、ViT のようにデータに飢えたモデルでは特に重要になります。

## 仕組みを詳しく
DeiT の工夫は大きく2つに分けられます。

### 1. 強力な学習レシピ (データ拡張と正則化)
データが少ないと ViT はすぐ過学習します。DeiT はこれを防ぐため、多数の拡張と正則化を一度に組み合わせました。

- RandAugment / AutoAugment: 画像を自動でランダムに回転・色変化・コントラスト調整・歪ませる。
- Mixup / CutMix: 2枚の画像をピクセル単位で混ぜたり (Mixup)、片方の矩形領域を別画像に貼り付けたり (CutMix) して合成画像とラベルを作る。
- Random Erasing: 画像の一部を矩形でランダムに塗りつぶす。
- ラベルスムージング: 正解ラベルを1.0でなく0.9程度に和らげ、過信を防ぐ。
- Stochastic Depth: 学習中、Transformer の層をランダムに飛ばし、深いネットの正則化と高速化を両立する。
- 繰り返し拡張 (repeated augmentation): 同じ画像の複数の拡張版を同じバッチに入れる。

これらにより1枚の画像が毎回違う見え方で現れ、実質的なデータ量を膨らませて過学習を抑えます。学習も最適化手法 (AdamW)、学習率スケジュール、weight decay などを丁寧に設定し、1台のマシンで数日という現実的な計算量に収めました。

### 2. 蒸留トークン (distillation token) による知識蒸留
DeiT の核心の新規性がこれです。ViT には通常クラストークン (分類用の特別なトークン) が1つありますが、DeiT はそれに加えて蒸留トークンをもう1つ足します。

形状で言うと、パッチトークン196個 + クラストークン1個 + 蒸留トークン1個 = 198個のトークンを Transformer に通します。

```
[CLS][dist][p1][p2]...[p196]   198 トークン
  │     │
  │     └→ CNN 教師の予測に合わせて学習
  └──────→ 正解ラベルに合わせて学習
```

- クラストークンの出力は、正解ラベルに合わせて学習する (通常の教師あり損失)。
- 蒸留トークンの出力は、CNN 教師 (RegNetY-16GF など) が出した予測に合わせて学習する (蒸留損失)。

学習を通じて、蒸留トークンはクラストークンとは少し違う情報を捉えるようになります。実験では、学習後の2つのトークンの予測が一致する割合が完全には100%でなく、互いに相補的な見方をしていることが確認されました。推論時は両トークンの予測を組み合わせて使います。

DeiT は2種類の蒸留を比較しました。ソフト蒸留 (教師の確率分布を真似る) と、ハード蒸留 (教師が最終的に選んだ硬いラベルを真似る) です。意外にも、ViT にはハード蒸留の方が効果的でした。教師の決定 (例えばデータ拡張で猫が切れて犬に見える画像を教師が「犬」と判断したらそれを正解とする) を真似ることで、拡張とラベルの食い違いをうまく吸収できるためと解釈されています。

損失の形を言葉で押さえておきます。ソフト蒸留では、生徒と教師の出力確率分布の近さを KL ダイバージェンス (2つの確率分布のずれを測る量。ゼロなら完全一致) で測り、温度 T (確率分布をどれだけ滑らかにするかのつまみ。大きいほど分布が均され、正解以外の「2番手・3番手」の情報が際立つ) を使います。全体の損失は「正解ラベルとのクロスエントロピー」と「教師分布との KL 項」を係数 λ で混ぜた形 L = (1−λ)·L_CE(生徒, 正解) + λ·T^2·KL(生徒/T, 教師/T) になります。ハード蒸留はこれを単純化し、教師が選んだクラスを正解ラベル扱いにして L = ½·L_CE(生徒, 正解) + ½·L_CE(生徒, 教師の決定) とします。ハード版には温度 T のような調整つまみが不要で、設定が楽なうえ性能も上、という実務上ありがたい結果でした。

教師に CNN を選ぶ点も重要です。CNN は帰納バイアスを持つので少ないデータで賢く、その帰納バイアスを attention 経由で間接的に ViT へ移植している、という見方ができます。

## 手法の系譜と主要論文
- 知識蒸留の原典: Distilling the Knowledge in a Neural Network (Hinton, Vinyals, Dean; 2015, Google)。大きな教師モデルの柔らかい確率分布を小さな生徒に真似させ、性能を保ったまま軽量化する手法を提案しました。

- 出発点: [Vision Transformer (ViT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0411_vision-transformer) (Dosovitskiy et al., ICLR 2021) は大規模データ前提で、DeiT はその弱点を直接埋める研究です。

- DeiT 本体: Training data-efficient image transformers & distillation through attention (Touvron, Cord, Douze, Massa, Sablayrolles, Jegou; ICML 2021, Facebook AI Research)。提案は「強力な拡張レシピ」と「蒸留トークン」。動機は「ViT を ImageNet-1K だけで実用的に学習可能にする」こと。新規性は、蒸留を出力層の損失だけでなく専用トークンとして attention の内部に組み込んだ点です。

- 後続の改良: DeiT III (Touvron et al., ECCV 2022) は学習レシピをさらに洗練し、蒸留トークンなしでも高精度を達成。Token Labeling や CaiT (Class-Attention in Image Transformers, Touvron et al., 2021) も DeiT の系譜で、クラストークンの扱いや深層化を改良しました。

- 並行する別路線として Swin Transformer (Liu et al., ICCV 2021) が階層構造と局所注意で ViT を実用化し、CNN 側で学習レシピを近代化したのが [ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext) です。DeiT が確立した拡張レシピはこれらにも広く取り込まれています。

## 論文の実験結果 (定量データ)
評価指標は ImageNet の top-1 精度 (test 画像で予測1位が正解と一致した割合) です。

- 標準データだけで CNN 超え: ImageNet-1K のみで学習し、DeiT-B (ViT-Base 相当、約8600万パラメータ) は top-1 約81.8% を達成しました。蒸留トークンを使った DeiT-B (記号 DeiT-B⚗ と表記される) は約83.4%、さらに高解像度 (384×384) で微調整すると約85.2% に達しました。これは同等パラメータの強力な CNN (EfficientNet など) に匹敵か上回る水準で、JFT-300M のような巨大事前学習なしに到達した点が画期的でした。

- 教師の選び方: 教師に同等以上の ViT を使うより、CNN (RegNet) を使った方が生徒の精度が高くなりました。DeiT-B で、CNN 教師だと約83.0%超、ViT 教師だと約82.7%と報告され、CNN の帰納バイアスを蒸留で移す効果が定量的に確認されました。

- ハード蒸留 対 ソフト蒸留: 同じ設定で、ハード蒸留がソフト蒸留を約0.5〜1ポイント上回りました。蒸留トークンを使う方式 (DeiT 方式) は、教師の予測を単にクラストークンの損失に足すだけの素朴な蒸留より一貫して高精度でした。

- 学習効率: DeiT-B は8枚の GPU を持つ1台のマシンで約3日で学習できると報告されています。これは ViT が要求していた巨大クラスタでの長期学習と比べ桁違いに手軽で、「再現可能性」の面で大きな前進でした。

- 拡張レシピのアブレーション: RandAugment, Mixup, CutMix, Random Erasing, Stochastic Depth などを1つずつ外すと精度が低下し、特に Mixup/CutMix と繰り返し拡張を外すと数ポイント単位で悪化しました。これは「ViT を中小規模データで学ぶには強い正則化が不可欠」という主張を裏づけます。なお論文の付録には、これらを全部外した「素の ViT を ImageNet-1K で学習」した場合の数字も載っており、精度が70%台前半まで落ち込みます。つまり ViT-B 相当の構造でも、レシピ次第で約70% から約83% まで10ポイント以上動く、という事実が、当時「ViT は弱い」とされた評価の多くが構造ではなく学習設定に起因していたことを明らかにしました。

- 後続 DeiT III の数値: DeiT III (Touvron et al., ECCV 2022) は蒸留トークンを使わず、3-Augment と呼ぶ簡略化した拡張、より大きな学習率、低解像度で事前学習して高解像度で微調整する戦略を採り、ViT-L で ImageNet-1K のみ約84.9%、ImageNet-21K 事前学習を加えると約87% 前後を達成しました。これは「蒸留に頼らずレシピだけでここまで来る」ことを示し、当初の DeiT が示した方向性をさらに推し進めた結果です。

これらの数字の意味は、ViT の本来の表現力は大規模データがなくても引き出せる、ただしそれには適切な学習レシピと蒸留という「外付けの帰納バイアス供給源」が必要だ、ということです。

## メリット・トレードオフ・限界
メリット
- 超大規模な非公開データなしに、公開データだけで ViT を強く学習でき、再現性とアクセス性が高い。
- 蒸留により CNN の帰納バイアスの恩恵を ViT に取り込め、データ効率が上がる。
- 確立した拡張レシピは他の ViT・CNN にも広く流用でき、分野全体の標準的な学習手順になった。

トレードオフ・限界
- 蒸留には事前に学習済みの良い CNN 教師が必要で、生徒の上限が教師の質に左右される。
- データ拡張と正則化のハイパーパラメータが多く、組み合わせと強さの調整が手間。設定を誤ると逆に性能が落ちる。
- 蒸留トークン分だけ計算とパラメータがわずかに増える (大きな負担ではないが厳密にはゼロでない)。
- 蒸留はあくまで教師の知識のコピーであり、教師の誤りやバイアスも一緒に受け継ぐ恐れがある。

研究上の未解決課題としては、教師なしでデータ効率を上げる方向 (自己教師あり事前学習) との統合、蒸留がなぜ ViT にここまで効くかの理論的説明、そして拡張レシピの自動最適化が挙げられます。

## 発展トピック・研究の最前線
DeiT が示した「学習レシピで ViT を救う」という考え方は、その後の標準訓練手順に深く影響しました。DeiT III は蒸留に頼らずレシピだけで高精度を出し、ResNet を近代的レシピで再訓練すると精度が大きく上がる (ResNet strikes back, Wightman et al., 2021) という再評価も促しました。一方で、ラベル付き教師に依存しない方向として MAE や DINO のような自己教師あり事前学習が台頭し、現在は「大規模自己教師あり事前学習 + DeiT 由来の強い微調整レシピ」という組み合わせが主流になりつつあります。蒸留自体も、画像だけでなく言語モデルや基盤モデルの軽量化に広く使われ続けています。
