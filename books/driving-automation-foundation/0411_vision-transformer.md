# Vision Transformer (ViT)

## ひとことで言うと
Vision Transformer (ViT) は、画像を小さな四角いパッチに切り分けて「単語の列」のように並べ、もともと自然言語処理 (文章を扱う技術) で使われていた Transformer という仕組みでそのまま処理する画像認識モデルです。畳み込み (CNN の核となる、小さなフィルタを画像上で滑らせていく操作) を一切使わず、十分大きなデータで事前学習すれば従来の CNN を超えられることを示しました。これにより、長年「画像といえば CNN」という常識を覆し、画像認識の設計思想そのものを書き換えた転換点となりました。

## 直感的な理解
画像認識を「文章を読む」ことに例えてみます。文章を理解するとき、人は単語を1つずつ読みながら、離れた単語同士の関係 (「彼」が文の最初の「太郎」を指す、など) を頭の中でつなげます。Transformer は、まさにこの「どの単語がどの単語とどれくらい関係するか」を一気に計算する仕組みとして言語の世界で大成功しました。

ViT のアイデアは「画像も、小さなタイル (パッチ) に切ってしまえば、タイルの並びは単語の並びと同じように扱えるのではないか」というものです。空の写真ならば、青いタイルが何枚も連なり、その中に一羽の鳥のタイルがある。Transformer は「鳥のタイル」と「翼のタイル」を遠く離れていても直接結びつけられます。CNN は近くのピクセルしか同時に見られず、画像全体を見渡すには層を何枚も重ねて視野を広げる必要がありました。ViT は最初から画像全体を見渡せる点が本質的に異なります。

なぜわざわざこんなことをするのか。CNN には「画像は近くのピクセル同士が関係している」「物体が画像のどこに写っていても同じものと認識すべき」という前提が構造に埋め込まれています。これを帰納バイアス (inductive bias、モデルが最初から持っている思い込み) と呼びます。帰納バイアスが強いと少ないデータでも効率よく学べますが、データが大量にあるときは逆に足かせになりえます。ViT は「思い込みを捨て、すべてをデータから学ばせたら、もっと強くなれるのではないか」という賭けに出たモデルです。

## 基礎: 前提となる概念
ViT を理解するには、まず Transformer の心臓部である self-attention (自己注意機構) を知る必要があります。これは「列の中の各要素が、他のすべての要素とどれくらい関係するかを計算し、関係の強さに応じて情報を混ぜ合わせる」仕組みです。

具体的には、各トークン (列の要素。ここではパッチ1枚) から3つのベクトルを作ります。クエリ Q (自分は何を探しているか)、キー K (自分は何者か)、バリュー V (自分が持っている情報) です。あるトークンの Q と、すべてのトークンの K の内積をとると「自分が各トークンとどれくらい合致するか」のスコアが出ます。これを softmax (合計が1になるよう正規化する関数) で重みに変え、その重みで全トークンの V を加重平均します。数式で書くと

Attention(Q, K, V) = softmax(QK^T / √d) V

ここで d はベクトルの次元、√d で割るのは内積が大きくなりすぎて softmax が極端になるのを防ぐためです。この計算で、各トークンは「自分に関係の深い他のトークンの情報」を集めてきます。実際には Q, K, V を複数組 (ヘッド) 用意して別々の関係を並行に学ぶ multi-head attention を使います。

self-attention の計算量はトークン数 N の2乗 (N^2) に比例します。これが後で効いてくる重要な制約です。画像を 224×224 のピクセルそのままで扱うと N = 50176 となり、その2乗は約25億で、現実的に計算できません。だからパッチにまとめる必要がありました。

もう1つの前提が転移学習 (transfer learning) です。大量のデータでまずモデルを学習し (事前学習)、その後に目的のタスクの小さなデータで微調整 (ファインチューニング) する、という二段構えの学習法です。ViT の性能はこの事前学習のデータ規模に大きく依存します。

## 仕組みを詳しく
ViT の処理を数値例とともにステップで追います。入力画像を 224×224×3 (縦224・横224・RGB3チャンネル) とします。

1. パッチ分割: 画像を 16×16 ピクセルのパッチに切り分けます。224 / 16 = 14 なので、縦14 × 横14 = 196 個のパッチができます。各パッチは 16×16×3 = 768 個の数値の塊です。これがタイトルの「16×16 の言葉 (words)」の由来で、パッチ1枚が文章の1単語に相当します。

2. 線形埋め込み (linear embedding): 各パッチの768個の数値を、全結合層 (重み行列をかける変換) で決まった次元 D のベクトルに変換します。ViT-Base なら D=768。これで形状 (196, 768) のトークン列になります。

3. クラストークンの付加: 分類用の特別なトークン ([CLS] トークン) を学習可能なベクトルとして先頭に1つ足し、(197, 768) にします。このトークンは画像全体の要約を蓄える器として働き、最後にこれだけを見て分類します。

4. 位置埋め込み (positional embedding): Transformer はトークンの並び順を区別できません (self-attention は順序を入れ替えても同じ結果を出す)。そこで「このパッチは左上」「これは右下」という位置情報を表す学習可能なベクトルを各トークンに足します。これがないと画像をバラバラに混ぜたのと同じになります。

5. Transformer エンコーダ: self-attention 層と全結合層 (MLP) を交互に積んだブロックを複数回通します。ViT-Base なら12回。各ブロックは LayerNorm (正規化) → multi-head self-attention → 残差接続 → LayerNorm → MLP → 残差接続、という構成です。各 attention 層で197個のトークンすべてが互いの関係を計算し合い、情報をやり取りします。例えば「タイヤのパッチ」が「車体のパッチ」と強く結びつく、といった関係が学習されます。

6. 分類: 最後にクラストークンの出力ベクトルを全結合層に通し、1000クラスなどの確率を出します。

```
画像 224x224x3
   │ 16x16 パッチに分割 (14x14=196 枚)
   ▼
[p1][p2]...[p196]            各パッチ 768 数値
   │ 線形埋め込み D=768
   ▼
[CLS][e1][e2]...[e196]       197 トークン
   │ + 位置埋め込み
   ▼
┌─────────────────────────┐
│ Transformer block x12    │  各層で全トークンが相互参照
│  (self-attn + MLP)       │
└─────────────────────────┘
   │ CLS トークンの出力を取り出す
   ▼
全結合層 → クラス確率
```

ViT には複数のサイズがあります。ViT-Base (12層, D=768, ヘッド12, 約8600万パラメータ)、ViT-Large (24層, D=1024, 約3億パラメータ)、ViT-Huge (32層, D=1280, 約6億パラメータ) です。パッチサイズも16 (ViT-B/16) や14, 32 が使われ、パッチを小さくするほどトークン数が増えて精度は上がるが計算は重くなります。例えば ViT-L/16 (パッチ16) を ViT-L/14 (パッチ14) に変えると、224×224 入力でトークン数が 196 から 256 へ増え、self-attention の計算量 (N^2 に比例) は約 1.7 倍になりますが、精度は1ポイント前後上がります。これが「精度か速度か」の典型的なトレードオフ点です。

設計上の最も重要な点は、CNN が持つ局所性や並進不変性を構造にほとんど埋め込んでいないことです。パッチ化と位置埋め込みにわずかな2次元構造の知識が入るだけで、あとはすべてデータから学びます。だからこそデータ規模が性能を決めます。

補足として「パッチ埋め込みは畳み込みそのもの」という見方も重要です。16×16 のパッチを D 次元へ線形変換する操作は、カーネルサイズ16・ストライド16の畳み込み層と数学的に等価です。つまり ViT は完全に畳み込みを排除したというより、「重なりのない大きな畳み込みを最初に1回だけかけ、あとは self-attention に任せる」構造とも解釈できます。このことは、畳み込みと attention が対立概念ではなく連続的につながっていることを示唆し、後のハイブリッドモデル (CNN で浅い特徴を作ってから Transformer に渡す ViT-Hybrid や ConViT, CoAtNet 等) の発想につながりました。

## 手法の系譜と主要論文
ViT は突然生まれたわけではなく、言語モデルの成功を画像へ持ち込む流れの到達点です。

- 原典の Transformer: Attention Is All You Need (Vaswani, Shazeer, Parmar ほか; NeurIPS 2017, Google)。リカレントニューラルネットワークや畳み込みを使わず、self-attention だけで系列を処理する仕組みを提案しました。機械翻訳で当時最高性能を出し、その後の BERT, GPT などすべての大規模言語モデルの土台となりました。ViT はこの構造をほぼそのまま画像に適用しています。

- ViT 本体: An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale (Dosovitskiy, Beyer, Kolesnikov ほか; ICLR 2021, Google Research)。提案は「画像をパッチ列にして純粋な Transformer で処理する」枠組み。動機は「CNN の帰納バイアスを取り払い、データ規模で性能を伸ばせるか検証する」こと。新規性は、畳み込みをほぼ完全に排除しながら大規模事前学習で CNN を超えた点にあります。

- 画像への attention 適用の先行研究も存在しました。Non-local Neural Networks (Wang et al., CVPR 2018) は CNN に self-attention 的な大域モジュールを加え、Stand-Alone Self-Attention (Ramachandran et al., 2019) や iGPT (Chen et al., 2020) はピクセルへ直接 attention を試みました。ViT の貢献は「パッチ化により計算を現実的にし、純粋な Transformer をスケールさせた」点で、これらを実用規模へ押し上げました。

- データ不足問題を解いた後続が [DeiT (Data-efficient Image Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0375_deit_note) (Touvron et al., ICML 2021) で、蒸留とデータ拡張により ImageNet だけで ViT を学習可能にしました。

- 計算量と階層性の問題を解いたのが Swin Transformer (Liu et al., ICCV 2021) で、窓 (ウィンドウ) ごとの局所注意により計算量を N の1乗に抑え、高解像度画像や物体検出・セグメンテーションへ ViT を広げました。

- 自己教師あり事前学習との相性を活かしたのが MAE: Masked Autoencoders (He et al., CVPR 2022) で、画像の大部分を隠して復元させる事前学習で ViT をさらに強くしました。その発想を CNN へ移したのが [ConvNeXt V2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0351_convnext-v2) です。

## 論文の実験結果 (定量データ)
ViT 論文の核心の主張は「事前学習データの規模で性能が劇的に変わる」ことです。評価指標は ImageNet の top-1 精度 (test 画像で予測1位が正解と一致した割合) が中心です。

- 事前学習データの効き方: ImageNet-1K (約130万枚) だけで学習すると、ViT は同規模の CNN (ResNet ベースの BiT) に負けました。ところが ImageNet-21K (約1400万枚) で事前学習すると同等になり、JFT-300M (Google 社内の約3億枚) で事前学習すると CNN を上回りました。ViT-Huge/14 を JFT-300M で事前学習した場合、ImageNet で top-1 約88.5% を達成し、当時最高クラスの BiT-L (約87.5%) を上回りました。この「データを増やすほど ViT が伸び、ある規模で CNN を追い抜く」交差現象こそが論文最大の発見です。

- 計算効率: JFT で事前学習する場合、同じ精度に到達するのに ViT は CNN (BiT) より2〜4倍少ない計算量で済みました。帰納バイアスがない分だけ事前学習に大規模データが要りますが、いったん学べば計算効率はむしろ良いという結果です。

- アブレーション (要素を抜く実験): 位置埋め込みを完全に外すと精度が約3ポイント低下し、画像の2次元構造を表す位置情報が重要であると示されました。一方で、位置埋め込みを単純な1次元 (トークンを一列に並べた順番) にしても、2次元を明示的に表す方式とほとんど差が出ませんでした。これは ViT が位置の関係も学習で補えることを示唆します。

- attention 距離の分析: 論文は「mean attention distance (平均注意距離)」という指標を導入しました。これは各注意ヘッドが平均してどれだけ離れたパッチに注意を向けているかを、ピクセル換算した距離です。CNN の受容野に相当する概念で、小さければ局所的、大きければ大域的を見ていることを意味します。分析の結果、浅い層 (第0〜2層) でも一部のヘッドは画像全体に届く長距離 (距離100ピクセル超) の注意を持つ一方、別のヘッドは隣接パッチだけを見る局所的な振る舞いをしていました。つまり ViT は浅い層から大域と局所を同時に扱える、という CNN にない性質を持ちます。CNN は浅い層では物理的に局所しか見られず、広域を見るには層を重ねて受容野を育てる必要があります。そして層が深くなるにつれ ViT の全ヘッドの注意距離が伸び、最終層ではほぼ全ヘッドが大域を見ます。これは「浅い層は局所、深い層は大域」という CNN の階層性を、ViT が学習を通じて自発的に再発見していることを示します。

- スケーリング則: 後続研究の Scaling Vision Transformers (Zhai et al., CVPR 2022) は ViT を20億パラメータまで拡大し、ImageNet で top-1 90.45% に到達しました。モデルとデータと計算をそろえて増やすほど性能が伸びる、という言語モデルで見られた法則が画像でも成り立つことを定量的に示しました。

これらの数字が重要なのは、「画像認識の性能はアーキテクチャの工夫より、規模を増やすことで素直に伸びる」という新しい開発方針を裏づけたからです。以後、画像でも「大きく作って大量に学習する」路線が主流になりました。

## メリット・トレードオフ・限界
メリット
- self-attention により画像全体の長距離の関係を最初から捉えられる。CNN は受容野を広げるために層を重ねる必要がある。
- 帰納バイアスが弱いため、超大規模データで青天井に性能が伸びやすく、スケーリング則が綺麗に成り立つ。
- 言語 Transformer の資産 (自己教師あり学習、スケーリングの知見、マルチモーダル化) をそのまま画像へ転用できる。

トレードオフ・限界
- 計算量がトークン数 (パッチ数) の2乗で増えるため、高解像度画像をそのまま扱うのが苦手。Swin などの階層的・局所的な工夫が必要。
- 帰納バイアスが弱いので、事前学習データが少ないと CNN に大きく負ける。中小規模データで一から学ぶ用途には不向き。
- 純粋な ViT は単一解像度の特徴しか出さず、物体検出やセグメンテーションが求める多解像度 (ピラミッド状) の特徴に向かない。
- 大規模事前学習に依存するため、計算資源と巨大データを持つ組織が有利になり、再現性・公平性の面で課題がある。これを緩和したのが DeiT や自己教師あり事前学習です。

未解決の研究課題としては、attention のロバスト性 (敵対的攻撃や分布のずれへの強さ)、解釈性 (attention マップが本当に「根拠」を表すかは議論がある)、そして N^2 計算量を線形に近づける効率的 attention の設計が今も活発に研究されています。

## 発展トピック・研究の最前線
ViT 以降の発展は大きく3方向です。第一に効率化で、Swin の窓注意、Pyramid ViT (PVT)、MobileViT のようにトークン数や計算量を抑える設計が進みました。第二に学習法で、DeiT の蒸留や MAE・DINO のような自己教師あり事前学習が、大規模ラベル付きデータへの依存を下げました。第三にマルチモーダル化で、CLIP (Radford et al., 2021) は画像とテキストを同じ空間に埋め込み、ViT を言語と結びつけて、ゼロショット認識 (見たことのないクラスをテキストの説明だけで分類) を可能にしました。自動運転やロボティクスの分野では、ViT 系バックボーンが画像から鳥瞰図 (BEV) 表現を作る際の特徴抽出器として広く使われています。
