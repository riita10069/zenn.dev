# Winner-Takes-All / min-of-N 損失

## ひとことで言うと
Winner-Takes-All(勝者総取り、WTA)損失、別名 min-of-N 損失は、モデルに複数の候補(N本の軌道など)を出させ、そのうち正解に最も近い一本にだけ損失(罰)を与える学習方法です。これにより各候補が異なる妥当な未来を担当するよう自然に役割分担し、答えが複数ありえる問題で「平均してどこにも行かない出力」を避けられます。実装は驚くほど単純で、「N本出して、一番惜しい一本だけ褒める」だけです。

## 直感的な理解
答えが一つに決まらない問題(交差点で直進・左折・右折のどれもありえる)を普通の二乗誤差(MSE)で学習すると、複数の正解の平均に潰れます。これがモード平均化、つまり「左の正解と右の正解の真ん中=縁石に直進」という誰も望まない出力です。

ではどうするか。WTA のアイデアは拍子抜けするほど素朴です。モデルに候補を N 本出させ、今回の正解(たとえば左折)に一番近い一本だけを「正解へ引き寄せる」。残りの候補は今回は罰しない。こうすると面白いことが起きます。ある候補は左折のときに勝ち、別の候補は右折のときに勝つ、というふうに、各候補が「自分が得意な状況」を担当するよう自然に分業していくのです。誰も全部を担当しないので、平均化が起こりません。チームに複数の専門家を置いて、案件ごとに最も適した一人に任せるイメージです。

この「複数の出力器を用意し、最良の一つだけで学習する」枠組みは Multiple Choice Learning(MCL、複数選択学習)として Guzman-Rivera ら(NeurIPS 2012)が定式化しました。

## 基礎: 前提となる概念
損失関数(loss)とは「予測がどれだけ間違っているか」を測る数値で、学習はこれを小さくする方向に重みを更新します。

二乗誤差(MSE, Mean Squared Error)は予測と正解の差を二乗して平均したもの。MSE を最小化する一点は「正解たちの平均」になります。これがモード平均化の数学的な原因です。

argmin / argmax。argmin は「最小値を与えるインデックス(何番目か)」を返す操作。WTA では各候補の誤差のうち最小のものの番号を argmin で取り、それを勝者とします。

逆伝播(backpropagation)。損失から各重みへの勾配(どちらへ動かせば損失が減るか)を計算し、重みを更新する仕組み。WTA では勝者の損失だけを逆伝播するので、勝者の経路だけが更新され、敗者の経路は今回更新されません。

minADE / minFDE。軌道予測の評価指標で、K本の候補のうち正解に最も近い一本の平均位置誤差(minADE)・終点誤差(minFDE)です。これはまさに min-of-N の発想を評価側へ持ち込んだもので、WTA 損失で学習したモデルと評価指標が同じ「最良候補を見る」哲学で揃います。

## 仕組みを詳しく
候補を N 本(たとえば N=6)出すモデルを考えます。出力は `[B, N, D]`(B=バッチ、N=候補数、D=軌道の次元、例:64ステップ×2信号=128 なら D=128)。正解軌道は `[B, D]` が一本。

通常の MSE は全 N 本に正解との誤差を課しますが、それでは全候補が同じ正解へ寄り、結局一本に潰れます。WTA は次のようにします。
```
各候補 k の誤差を計算:   e_k = || 候補_k − 正解 ||²     (k = 1..N)
正解に最も近い候補番号:   k* = argmin_k e_k
WTA 損失:               L = e_{k*}      ← 勝者 k* の誤差だけを最小化
```
「min-of-N」という別名はこの `min_k e_k`(N本中の最小誤差)に由来します。勝者はその正解にさらに寄り、敗者は今回更新されず、別の状況で勝者になれる余地が残ります。学習データ全体を通すと、各候補が異なるモードの専門家へ分化します。

学習1回分のステップ:
```
1. モデルに N 本の候補を出させる
2. 各候補と正解の誤差 e_k を全部計算
3. 最小の e_k を持つ勝者 k* を argmin で選ぶ
4. 勝者の誤差 e_{k*} だけを逆伝播し重みを更新
```
ASCII図:
```
候補1 ──┐
候補2 ──┤   正解に最も近いのは候補3
候補3 ──┼──> 勝者! ここだけ損失を与えて引き寄せる
候補4 ──┤   (候補1,2,4,5,6 は今回更新しない=別モードを担当できる)
候補5 ──┤
候補6 ──┘
```

確信度(classification)との組み合わせ。WTA だけだと「どの候補がどれくらい確からしいか」が分かりません。そこで実務では、勝者には回帰損失(位置を正確に)を、全候補には分類損失(どれが勝者かを当てる確率を上げる)を組み合わせます。総損失はおおむね `L = e_{k*} + λ · CrossEntropy(確信度, k*)` の形です。これで「複数の候補 + 各候補の確からしさ」を出力でき、推論時は確信度が最大の一本を選べます。MultiPath / MultiPath++ はまさにこの「勝者の回帰 + 確信度の分類」を採用しています。

既知の弱点と対策。WTA には候補崩壊(mode collapse、特定の候補ばかり勝ち続け、他の候補が一度も勝てず死ぬ)という弱点があります。一度も勝てない候補は更新されず無駄になり、実効的なモード数が N より減ってしまいます。対策は「完全な勝者総取りを少し緩める」方向です。
- relaxed / evolving WTA: 勝者に大きな重み、敗者にも小さな重み ε を与えて全候補を少しずつ更新する。L = Σ_k w_k e_k(勝者 w≈1、敗者 w=ε)。
- top-k WTA: 最良の一本だけでなく上位 k 本に損失を与える。
- Stochastic MCL: 勝者割り当てに確率性を入れ、候補が固定化しないようにする。

## 手法の系譜と主要論文
原典は Guzman-Rivera, Batra, Kohli(NeurIPS 2012)「Multiple Choice Learning: Learning to Produce Multiple Structured Outputs」。複数の予測器に多様な構造化出力を出させ、最良の一つで学習する枠組みを定式化しました。動機は、曖昧な問題では「平均的な一解」より「妥当な複数解の集合」を出すほうが有用だから。これが WTA / min-of-N の起源です。

次に Lee, Purushwalkam, Cogswell ら(NeurIPS 2016)「Stochastic Multiple Choice Learning for Training Diverse Deep Ensembles」が、WTA を深層学習・確率的勾配降下(ミニバッチ学習)へ適合させ、勝者割り当てを確率化して候補崩壊を緩和しました。動機は、ミニバッチで WTA を安定に回し多様なアンサンブルを得ること。

軌道予測へは MultiPath(Chai, Sapp, Bansal, Anguelov, CoRL 2019, arXiv:1910.05449)が持ち込みました。事前定義したアンカー(典型経路の雛形)ごとに確信度(分類)を出し、正解に最も近いアンカーを勝者として、その勝者にだけオフセット回帰を課します。WTA の「勝者だけ回帰」と分類の「確信度学習」を組み合わせた代表例です。

MultiPath++(Varadarajan ら, ICRA 2022, arXiv:2111.14973)は固定アンカーの制約を外し、学習可能な候補(コンテキスト依存の anchor embedding)と min-of-N 系の学習を組み合わせ、エージェント間の情報融合を強化しました。動機は、固定アンカーでは表せない文脈依存の多峰性を扱うこと。

これらに加え、min-of-N の哲学は評価指標(minADE/minFDE/MissRate)としても定着し、学習と評価の両面で「複数候補のうちベストを見る」枠組みが軌道予測の標準になりました。MDN([Mixture Density Network 軌道予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0189_mixture-density-network-planner))が確率分布のパラメータを陽に出すのに対し、WTA は確率を陽に持たず「候補と分業」で多峰性を表す、対照的な古典解です。

## 論文の実験結果(定量データ)
MCL(2012)は構造化予測タスク(画像セグメンテーション等)で、単一予測や独立アンサンブルに比べ「オラクル誤差」(集合のどれかが当てる最良誤差)とカバレッジ(正解を集合のどれかが当てる割合)を改善したと報告しました。一つの最良解を出すより、妥当な複数解の集合を出すほうが下流タスクに有利だという主張を、オラクル指標の改善で裏付けています。

Stochastic MCL(2016)は CIFAR などの分類・セグメンテーションで、決定論的 WTA より多様性が高く、かつ候補崩壊が抑えられたアンサンブルを安定に学習できたと報告しました。確率的な勝者割り当てが「全候補が一度は勝つ」状態を保ち、実効モード数の低下を防いだことが示されています。

軌道予測では、MultiPath が nuScenes / 社内大規模データで、回帰一本のベースラインに対し minADE/minFDE を明確に下げ、多峰な軌道を確率付きで安定に出せることを示しました。アンカーごとの分類 + 勝者の残差回帰という構成が、素の MDN より学習が安定することも報告されています。MultiPath++ は Waymo Open Motion Dataset の予測ベンチマークで上位の成績を報告し、学習可能アンカーと情報融合により固定アンカー版から minADE/minFDE/MissRate を改善しました。

アブレーションの示唆は明快です。WTA の緩和(relaxed/top-k/stochastic)を外して完全勝者総取りにすると候補崩壊が起き、実効モード数が減って minADE が頭打ちになります。逆に緩和を入れると全候補が育ち多様性(カバレッジ)が上がります。また確信度の分類損失を外すと「どの候補が確からしいか」が学べず、推論時にベスト一本を選べなくなって実用 ADE(min ではなく選んだ一本の ADE)が悪化します。これは「WTA(多様性)」と「分類(選択能力)」の二本柱が両方必要であることを示しています。

## メリット・トレードオフ・限界
メリット。実装が単純(N 本出して argmin で勝者を選び、その誤差だけ最小化)。多峰な出力に自然な役割分担が生まれ、モード平均化を避けられます。MDN のような分散の発散がなく、二乗誤差ベースで比較的安定。評価指標 minADE/minFDE と学習哲学が一致するため、ベンチマーク最適化と相性が良い。

トレードオフと限界。候補崩壊(特定候補ばかり勝ち、他が死ぬ)が起きやすく、relaxed/top-k/stochastic などの緩和が事実上必須です。候補数 N を事前に決める必要があり、N より多いモードは表せません。勝者だけ更新するため敗者が長く学習されず収束が遅くなることがあります。確信度を出すには分類損失を別途組み合わせる必要があります。さらに本質的な限界として、WTA は「正解への近さ」しか見ないため、候補が多様でも各候補の確率較正(その候補が実際どれくらい起こりやすいか)は別途扱う必要があり、安全性の観点で「ありえないが正解に近い候補」を罰せない、といった課題が残ります。研究上は、候補数を可変にする、崩壊を理論的に防ぐ割り当て、そして多様性と較正を同時に保証する損失設計が未解決テーマです。

## 発展トピック・研究の最前線
WTA の崩壊問題に対し、勝者割り当てを最適輸送(optimal transport、候補と正解を最適にマッチングする数理)として解く定式化や、ハンガリアン法による集合マッチング(物体検出の DETR で使われる、候補と正解を一対一に割り当てる手法)を軌道予測へ持ち込む流れがあります。これらは「どの候補に損失を与えるか」をより原理的に決め、崩壊を抑えます。また、WTA は離散候補前提ですが、生成モデル(拡散・フローマッチング)で連続的に多峰性を表す路線が台頭し、WTA の弱点(候補数固定・崩壊)を回避しつつ多峰サンプリングを実現しています。ただし生成モデルでも「複数サンプルのうちベストを評価する」min-of-N の評価哲学は健在で、学習と評価をつなぐ概念として WTA / min-of-N は今も中心にあります。
