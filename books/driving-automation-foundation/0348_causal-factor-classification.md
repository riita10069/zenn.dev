# 因果分類 (intersection/pedestrian/traffic_light/obstacle/clear)

## ひとことで言うと

運転中の車が「今この瞬間、運転の判断を一番強く左右しているのは何か」を、あらかじめ決めた少数の種類のどれかに当てはめて答える仕組みです。たとえば「目の前に歩行者がいる」「赤信号だ」「交差点に差しかかっている」といった、運転の理由になっている支配的な要因を分類します。代表的な定式化では intersection (交差点) / pedestrian (歩行者) / traffic_light (信号機) / obstacle (障害物) / clear (特に支配要因なし) の5クラスを使います。これによって、ブラックボックスになりがちな運転モデルが「なぜその運転をしたのか」を人間が読み取りやすくなります。

## 直感的な理解

自動運転モデルが軌跡 (車の進路) を出したとき、その軌跡だけを見ても「なぜそう動いたのか」は分かりません。減速した軌跡が出ていても、それが「歩行者を見たから」なのか「赤信号だから」なのか「ただ前の車に合わせただけ」なのかは外から判別できません。これは、人間に「なぜ止まったの?」と聞いて「なんとなく」としか返ってこないようなもので、安全が問われる場面では困ります。事故が起きたとき、原因を遡れないことは実用上の致命傷になります。

そこで、運転モデルにもう1つ小さな仕事を与えます。軌跡を出すのと並行して「この場面で運転判断を最も支配しているのは何か」を、決められた選択肢から1つ選ばせるのです。選択肢を少数 (たとえば5つ) に絞るのがポイントです。選択肢が多すぎると、各選択肢に十分な学習データが集まらず、人間にも一目で分かりにくくなります。5つくらいなら、ダッシュボードに「今は traffic_light が支配的」と表示すれば、開発者も乗員も状況を即座に把握できます。

「clear (特に支配要因なし)」という受け皿クラスを用意するのも大事な工夫です。これがないと、見通しの良い直線を普通に走っているだけの場面まで無理に「交差点」や「歩行者」に押し込めることになり、分類が歪みます。clear があれば「特に何もない」を正直に表現でき、しかもデータの多数派 (運転時間の大半は何も支配要因がない) を吸収できます。

## 基礎: 前提となる概念

- エンドツーエンド (End-to-End, E2E) — カメラ画像などの入力を受け取り、いきなり軌跡や操作を出力するニューラルネットワーク。「入口 (センサ) から出口 (操作) までを1つのモデルで一気通貫に学習する」という意味です。性能は高いが、内部が不透明なのが弱点。
- ブラックボックス問題 — モデルが数百万のパラメータの計算で結論を出すため、出力を見ても判断根拠が分からない問題。安全説明やデバッグ (不具合の原因究明) を難しくします。
- 補助タスク (auxiliary task) — 本来の目的 (軌跡生成) とは別に、学習を助けるために付け足すタスク。それ自体が目的ではなく、共有表現を良くしたり解釈性を上げたりするための「おまけ」です。
- 分類 (classification) とロジット (logits) — 入力をいくつかのクラスのどれかに割り当てるタスク。モデルは各クラスへの生スコア (ロジット) を出し、最大のものを予測クラスとします。softmax を通すと合計1の確率になります。
- 解釈性 (interpretability) — モデルの判断を人間が理解できる度合い。安全クリティカルな分野で強く求められます。
- 疑似ラベル (pseudo-label) — 人手で付けた正解ではなく、別のモデル (たとえば視覚言語モデル VLM) に自動生成させたラベル。安価に大量に作れる反面、誤りを含みます。
- クラス不均衡 (class imbalance) — あるクラス (例 clear) のサンプルが他より圧倒的に多い状態。素朴に学習すると多数派ばかり当てるモデルになりやすく、対策 (クラス重み付けなど) が要ります。

## 仕組みを詳しく

クラスの集合は、順序まで含めて固定するのが定石です。たとえば次のように定義します。

```
CAUSAL_CLASSES = (
    "intersection",   # 0
    "pedestrian",     # 1
    "traffic_light",  # 2
    "obstacle",       # 3
    "clear",          # 4
)
NUM_CLASSES = 5
```

順序を固定する理由は重要です。クラスのインデックス (0=intersection, …, 4=clear) は、学習済みモデルの重みや、あらかじめ整数で保存された疑似ラベルと一対一で紐づいています。あとから順序を入れ替えると、過去に作った正解ラベルが全部ズレ、学習が静かに壊れます (エラーは出ず、ただ精度が落ちる最悪の壊れ方をします)。だから「順序は契約の一部、並べ替え禁止」という運用になります。

分類の流れを tensor の形 (shape、各次元のサイズ) で追います。バッチサイズ (一度に処理する場面の数) を B=4 とします。

1. 共有表現由来の文脈ベクトル `context` を受け取る。形は `[B, embed_dim]` = `[4, 256]`。256 は特徴を表す数値の本数 (埋め込み次元)。
2. 推論トランク (小さな MLP) を通して `reasoning_latent` `[4, 256]` に変換する。
3. 分類ヘッド (典型的には1枚の線形層) を通して `decision_logits` `[4, 5]` を得る。5 が因果クラス数。
4. 各行の最大値を取るクラスが「推定された支配的因果要因」。たとえば3番目の行のロジットが `[0.1, 0.2, 3.4, 0.1, 0.0]` なら、インデックス2 = `traffic_light` が選ばれます。softmax をかけるとこの行は traffic_light の確率がおよそ 0.90 になり、確信度としても使えます。

なぜ少数クラスに固定するのか、を設計理由として整理します。第一に、クラス数を絞ると人手アノテーション無し (VLM 疑似ラベル) でも各クラスに十分なサンプルが集まりやすい。第二に、clear という受け皿で「支配要因なし」を表現でき、分類の歪みを防げる。第三に、自由文の説明と違い、5択は機械が扱いやすく、学習信号 (損失) としても評価指標 (正解率) としても明快です。

数値例で「粗さ」の限界も見ておきます。横断歩道の赤信号で歩行者も待っている場面では、本当は pedestrian と traffic_light の両方が効いていますが、単一ラベルの分類では片方しか選べません。これが後述するトレードオフの一つです。マルチラベル化 (各クラスを独立に 0/1 判定する) で緩和できますが、評価とラベル付けが複雑になります。

クラス不均衡への配慮も実務上欠かせません。運転データの大半は clear に偏るため、何も対策しないと「常に clear と答える」だけで高い見かけ正解率が出てしまいます。これを防ぐため、交差エントロピー損失にクラスごとの重み (頻度の逆数など) をかけ、少数クラス (pedestrian など、しかし安全上は最重要) の誤りを重く罰するのが定番です。

## 手法の系譜と主要論文

運転シーンの「意味的な理由づけ」をモデルに持たせる研究は段階的に発展してきました。

- Kim and Canny, "Interpretable Learning for Self-Driving Cars by Visualizing Causal Attention" (ICCV 2017, arXiv:1703.10631)。運転モデルが画像のどこを見て判断したかを注意 (attention) で可視化し、さらにその注意領域が本当に判断の原因か (因果的か) を、領域をマスクして出力が変わるかで検証しました。動機は「ブラックボックスな運転モデルの根拠を人間が確認できるようにする」こと。前方車両や車線が判断の原因として浮かぶことを示しました。限界は、ヒートマップは出せても、本ノートのような離散の「理由ラベル」までは出さない点です。

- Kim et al., "Textual Explanations for Self-Driving Vehicles" (ECCV 2018, arXiv:1807.11546)。判断理由を自然言語の文章 (例「歩行者が横断しているので減速した」) で説明する方式。BDD-X (Berkeley DeepDrive eXplanation) データセットを構築しました。表現力は高いが、文章は曖昧で、自動評価や学習信号として扱いにくいという限界があります。離散の5クラス分類は、この曖昧さを避け、機械が扱いやすい形に落とし込んだ対極のアプローチです。

- Xu et al., "Explainable Object-Induced Action Decision for Autonomous Vehicles" (CVPR 2020, arXiv:2003.09405)。シーン中の物体 (歩行者・信号など) が、車のどの行動 (止まる・進む・曲がる) を引き起こしたかを、行動と理由のセットで出すモデルを提案。BDD-OIA データセット (行動とその理由を multi-label でアノテート) を構築しました。動機は本ノートと同じく「判断根拠を物体レベルで説明したい」こと。限界は、物体ごとの詳細アノテーションが必要でデータ作成コストが高いこと。5クラスの場面レベル分類は、これを「場面全体で支配的な1要因」に単純化した軽量版と位置づけられます。

- より新しい流れとして、視覚言語モデル (VLM) や大規模言語モデルを運転に統合する研究 (DriveGPT4, Xu et al. 2023, arXiv:2310.01412; LMDrive, Shao et al. CVPR 2024) があります。これらは自由言語での豊かな説明ができますが、推論コストが重く、出力が不安定になりやすい。離散の因果分類は、こうした VLM を「教師」に使って疑似ラベルを作りつつ、推論時は軽量な分類ヘッドで高速・安定に動かす、というハイブリッドの実用形につながります。

## 論文の実験結果(定量データ)

- Kim & Canny (ICCV 2017) は、注意マップを介在させても運転制御の予測精度がほぼ落ちない (説明性を足してもタスク性能を犠牲にしない) ことを示しました。具体的には、注意機構を挟んだモデルの操舵角予測誤差が、注意なしのベースラインと同等水準にとどまったと報告されています。さらに、注意領域をマスクすると出力が大きく変わる領域こそ「因果的」と定義し、注意の約 6 割を占めていた領域を間引いても予測がほぼ変わらない (=見ているが結論に影響しない無関係な注意) ことを示し、因果的な注意と無関係な注意を分離できることを定量的に示しました。これは「解釈を足すことのコストは小さく保てる」という、補助タスク全般に通じる重要な知見です。

- Xu et al. (CVPR 2020) は BDD-OIA で、行動予測 (4種の行動カテゴリ) と理由の説明 (21種の理由) を同時に出すマルチラベル分類を評価しました。理由を補助的に学ぶ設定が行動予測の F1 スコア (適合率と再現率の調和平均、0〜1、高いほど良い) を改善する傾向を報告しています。指標の意味としては、F1 が高いほど「見逃しも誤検出も少なく行動を当てられている」ことを表します。理由側 (explanation) の F1 も報告され、行動と理由を結びつけて学ぶことの相互強化効果を示唆しました。

- 因果分類を補助タスクとして使う一般的な期待効果は、(1) 解釈性の獲得 (推論時にほぼゼロコストで理由ラベルが得られる)、(2) マルチタスク学習による共有表現の意味的整形、の2点です。ただし、5クラス単一ラベル分類そのものの「正解率が何 % 改善した」という統一的なベンチマーク数値は、データセットとラベル品質に強く依存するため一概には言えません。特に疑似ラベルを使う場合、分類精度の上限は疑似ラベルの質に縛られる、というのが正直なところです。

- アブレーションの観点では、「理由分類の補助ヘッドを外すと主タスク (軌跡や行動予測) の性能がどう変わるか」が核心の検証です。関連研究では、関連性の高い補助タスクを足すと主タスクが改善し、無関係・矛盾するタスクだと悪化する、という結果が繰り返し観察されています (マルチタスク学習の一般則)。

## メリット・トレードオフ・限界

メリット
- モデルの判断根拠が少数の分かりやすいラベルで読め、解釈性が上がる。推論時のコストはほぼゼロ (線形層1枚)。
- クラスを固定すると、疑似ラベル付けや評価がシンプルになる。
- clear クラスで「特に理由なし」を正直に表せ、分類の歪みとデータ偏りを吸収できる。
- 補助タスクとして共有表現を意味的に整える効果が期待できる (マルチタスク学習)。

トレードオフ・限界
- 5クラスは粗い。「歩行者かつ赤信号」のように複数要因が同時に効く場面でも1つしか選べない (単一ラベルの限界)。マルチラベル化で緩和できるが、評価とラベル付けが複雑になる。
- クラス順序を契約として固定するため、後からクラスを増やす・並べ替えると過去のラベルや学習済み重みと整合しなくなる。
- 正解が人手でなく疑似ラベル (ノイズあり) の場合、分類精度の上限はその疑似ラベルの質に縛られる。
- 「支配的な1要因」という定義自体が主観的で、何を支配的とみなすかの基準がデータセット間で揺れる。
- clear への極端な偏り (クラス不均衡) を放置すると、何でも clear と答えるモデルになりかねない。クラス重み付けが必須。
- 未解決課題: 因果ラベルが本当に「因果」を捉えているか (単なる相関ではないか) の検証は難しく、介入実験 (要因を取り除いて出力が変わるか) なしには因果性を保証できません。

## 発展トピック・研究の最前線

- マルチラベル / 階層分類: 単一ラベルの粗さを補うため、複数要因の同時出力や、大分類 → 小分類の階層構造が研究されています。BDD-OIA のマルチラベル定式化がその代表例です。
- 因果性の厳密化: 注意マスキングや反事実 (counterfactual、もし要因がなかったら) 生成で、ラベルが相関でなく因果を捉えているか検証する流れ。Kim & Canny の causal attention はその先駆けです。
- VLM 蒸留: 重い VLM の自由言語推論を、軽量な離散分類ヘッドへ蒸留 (knowledge distillation) して、解釈性と推論速度を両立させるアプローチ。VLM を教師に疑似ラベルを作り、生徒の軽量ヘッドが実時間で答える構成が実用的に有望です。
- シーングラフ / 占有との接続: 因果要因を物体検出や占有グリッドと結びつけ、「どの物体が要因か」まで遡れるようにする方向。
- 言語による説明との橋渡し: DriveGPT4 のような言語生成モデルと離散分類を組み合わせ、「粗い高速ラベル + 必要時の詳細な言語説明」という二段構えの解釈性が探られています。
