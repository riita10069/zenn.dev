# Channel Projection（1x1 Conv によるチャネル次元射影）

## ひとことで言うと
画像特徴抽出器(backbone)から出てきた特徴は「チャネル数(特徴マップの厚み)」が大きく、しかも複数スケールを連結したせいで半端な数(例 1440)になっていることがあります。これを下流の部品が期待する固定値(例 256)にきっちり揃える小さな変換層を channel projection(チャネル射影)と呼びます。中身は「1x1 の畳み込み(1x1 Convolution)」というシンプルな仕組みです。空間方向(高さ・幅)はいじらず、各ピクセル位置でチャネルだけを線形変換します。

## 直感的な理解
特徴マップを「方眼紙の各マスに、色鉛筆の束が刺さっている」とイメージしてください。マスの数が空間解像度(高さ x 幅)、各マスに刺さった色鉛筆の本数がチャネル数です。channel projection は「全部のマスで同じルールに従い、1440本の鉛筆を混ぜて256本に詰め直す」操作です。マスの並び(空間)は一切動かさず、各マスの鉛筆の本数だけを揃えます。

なぜこれが要るのか。下流のモジュールは「入力は256チャネル」という前提で作られています。ところが backbone の生出力や、複数スケールを連結した特徴は、半端な大きさ(1440など)です。1440のまま渡すと形状が合わずエラーになる。だから「1440を256に変換する層」が必要で、それが channel projection です。射影(projection)とは数学で「ある次元の空間から別の次元の空間へ写す」操作を指し、ここでは1440次元のチャネルを256次元へ写すことを意味します。

## 基礎: 前提となる概念

特徴マップ(feature map)。画像をニューラルネットに通すと出てくる `(バッチ, チャネル, 高さ, 幅)` の4次元テンソル(数の塊)です。チャネルとは「その位置にどんな特徴があるかを表す数字の本数」だと思ってください。チャネル数256なら、各ピクセル位置に256個の特徴値が並んでいます。

畳み込み(Convolution)。小さな窓(カーネル)を画像の上で滑らせ、窓の中の値に重みを掛けて足す操作です。普通の畳み込みは 3x3 などの窓を使い、周囲のピクセルを混ぜて「エッジ」「模様」などを検出します。窓のサイズが受容野(その出力が画像のどれだけの範囲を見ているか)を決めます。

線形変換と非線形性。重みを掛けて足すだけの操作は「線形変換」で、いくら重ねても1つの線形変換に潰せてしまい、複雑な関係を学べません。そこで途中に「活性化関数(activation function)」という非線形な処理を挟みます。これがあるとネットワークは直線では表せない複雑な関係を学べるようになります。

## 仕組みを詳しく

典型的な実装は、たった2層の積み重ねです。

```python
self.channel_proj = nn.Sequential(
    nn.Conv2d(backbone_channels, embed_dim, kernel_size=1),  # 例 1440 -> 256
    nn.GELU()
)
```

`nn.Conv2d(in, out, kernel_size=1)` が「1x1 畳み込み」です。普通の畳み込みは 3x3 の窓で周囲を混ぜますが、1x1 は窓が1ピクセルなので空間方向(高さ・幅)には一切混ぜません。各ピクセル位置で「入力1440個の数を重み付きで足し合わせて出力256個を作る」だけです。つまり実質的に「各ピクセルに同じ全結合層(線形変換)を適用する」操作で、チャネル数だけを 1440 -> 256 に変えます。

tensor shape の before/after を具体的に書きます(空間サイズ 8x8 の場合)。

```
入力 : (B, 1440, 8, 8)   ← マルチスケール連結後
  各位置の 1440 個を、同じ重み行列 W (256 x 1440) で 256 個に線形変換
出力 : (B,  256, 8, 8)    ← 高さ・幅は不変、チャネルだけ 256 に
```

空間サイズ 8x8 はそのまま、チャネルだけが256になっているのがポイントです。重み行列 W は全ピクセルで共有されます(これが畳み込みの「重み共有」)。式で書くと、位置 (i, j) の出力ベクトル `y_ij = W x_ij + b` で、`x_ij` は入力1440次元、`W` は 256x1440 行列、`b` は256次元バイアス、これを全ての (i, j) に同じ `W, b` で適用します。

その後ろの `nn.GELU()` は活性化関数です。GELU(Gaussian Error Linear Unit)は ReLU の滑らかな版で、入力が大きいほどほぼそのまま通し、負の側を緩やかに抑えます。数式では `GELU(x) = x * Φ(x)`(Φは標準正規分布の累積分布関数)で、入力 x がどれくらい「平均より上か」に応じて滑らかにゲートをかけます。1x1 Conv だけだと単なる線形変換にとどまり表現力が乏しいため、GELU を挟んで非線形性を持たせています。

なぜ全結合層(`nn.Linear`)でなく 1x1 Conv を使うのか。データが `(B, C, H, W)` の画像形状のまま流れているので、形を崩さずチャネルだけ変換できる 1x1 Conv が自然だからです。`nn.Linear` を使うには一度 `(B, H, W, C)` に並べ替え、変換後にまた戻す手間がかかりますが、1x1 Conv ならその必要がありません。計算内容自体は両者で等価(各位置への同一線形変換)です。

なぜこの設計か。射影を「学習可能な層」にする点が肝心です。単純に1440本から先頭256本を切り捨てる、といった固定操作にすると重要な情報を捨てかねません。1x1 Conv なら「1440本のうちどの組み合わせを残すと下流のタスクに役立つか」をデータから学習できます。GELU の非線形がさらに表現力を足します。実装によっては Conv のあとに BatchNorm や LayerNorm を挟み、射影後の特徴の分布を整える(学習を安定させる)こともあります。

## 手法の系譜と主要論文

1x1 畳み込みでチャネルを操作する発想の系譜です。

Network in Network(Lin, Chen, Yan, arXiv:1312.4400, 2014)。1x1 Conv を最初に明確に提案した論文です。「1x1 Conv は各位置に小さな多層パーセプトロン(MLP)を適用するのに等しく、チャネル方向の情報を混ぜたり次元を増減したりできる」と定式化しました。従来は畳み込みといえば空間方向の特徴抽出でしたが、「チャネル方向の変換」という新しい役割を与えた点が新規性です。同論文はあわせて Global Average Pooling も提案し、その後の設計に大きな影響を与えました。

GoogLeNet / Inception(Szegedy et al., CVPR 2015, arXiv:1409.4842)。1x1 Conv を「bottleneck(ボトルネック)」として実用化しました。重い 3x3/5x5 畳み込みの前に 1x1 Conv でチャネルを減らし、計算量を大幅に削減します。効果は計算コストの削減、トレードオフは情報をいったん圧縮するので細かい情報が落ちうることです。

ResNet(He et al., "Deep Residual Learning", CVPR 2016, arXiv:1512.03385)。bottleneck ブロックで「1x1 Conv で次元を絞る -> 3x3 Conv で処理 -> 1x1 Conv で次元を戻す」という構造を標準化しました。これにより50層・101層・152層といった超深層ネットを現実的な計算量で訓練できるようになり、チャネル射影が深層化の鍵になりました。

GELU(Hendrycks & Gimpel, "Gaussian Error Linear Units", arXiv:1606.08415, 2016)。ReLU の不連続な折れ曲がりを滑らかにした活性化関数で、BERT, GPT, Vision Transformer, ConvNeXt など多くの現代モデルで標準採用されています。ReLU より滑らかで学習が安定しやすい一方、ガウス累積分布(または近似)を計算するため ReLU よりわずかに重いのがトレードオフです。

Squeeze-and-Excitation(Hu et al., CVPR 2018, arXiv:1709.01507)。1x1 Conv(全結合)でチャネルごとの重要度を入力依存に学習し、特徴マップのチャネルを動的に重み付けする機構です。固定の射影を「どのチャネルを残すか入力ごとに決める」へ一歩進めました。

## 論文の実験結果(定量データ)

GoogLeNet の bottleneck の効果が最も分かりやすい定量例です。Inception モジュール内で 1x1 Conv による次元削減を入れると、ImageNet(約120万枚・1000カテゴリの画像分類ベンチマーク)で、ネットワーク全体の演算量を大幅に抑えつつ高精度を達成しました。GoogLeNet は約500万パラメータ・約15億回の乗加算で ILSVRC 2014 の分類で top-5 誤り率約6.7%を記録し、当時はるかにパラメータの多い AlexNet(約6000万パラメータ、top-5 誤り約16%)を上回りました。1x1 Conv による次元削減を外すと演算量が数倍に膨れ上がり、同じ計算予算では訓練できなくなります。これが「1x1 Conv の計算削減効果」の直接的な実証です。top-5 誤り率とは「正解が予測上位5位以内に入らなかった割合」で、小さいほど良い指標です。

ResNet のアブレーションでは、bottleneck 設計(1x1 で絞る構造)を使うことで、同等の計算量のままより深いネットワークを訓練でき、ImageNet top-1 精度が向上することが示されています。ResNet-50 は top-1 約76%、ResNet-152 は約78%で、深さを増やしても 1x1 ボトルネックのおかげで計算量が爆発しないことが鍵でした。1x1 を使わずに全層を 3x3 にすると、同じ深さでパラメータ・演算量が数倍になり、メモリにも乗りません。

活性化関数の比較研究(Hendrycks & Gimpel, 2016)では、GELU が ReLU・ELU と比べて、複数のタスク(MNIST, CIFAR-10/100, 言語モデリング)で同等以上の精度・収束性を示しました。劇的な差ではないものの一貫して安定して良い、という結果が、後の Transformer 系での標準採用につながりました。

Squeeze-and-Excitation は ImageNet で、SE ブロック(1x1 Conv 2枚でチャネル重みを生成)を追加するだけで ResNet-50 の top-1 誤り率を約1ポイント以上改善し(SENet が ILSVRC 2017 で優勝)、追加計算量は1%未満と報告しました。「1x1 Conv でチャネルを賢く扱う」効果の定量例です。

定量的な意味の確認。1440 -> 256 の 1x1 Conv は重み行列が 256 x 1440 ≈ 37万パラメータで、8x8 = 64 ピクセル全てに適用しても演算量は控えめ(約 256 x 1440 x 64 ≈ 2360万回の乗加算)です。これに対し同じチャネル変換を 3x3 Conv でやると重みは 9 倍(約330万)に膨らみます。1x1 を選ぶことで、チャネル射影のコストを最小に抑えています。

## メリット・トレードオフ・限界

メリット。第一に、backbone の出力チャネル数が何であっても、下流が期待する固定値(256)に確実に揃えられる。第二に、1x1 Conv は空間サイズを変えないので、特徴マップの解像度(位置情報)を保ったまま厚みだけ調整できる。第三に、学習可能な層なので、単なる切り捨てではなく「どの特徴を残すか」をデータから学べる。GELU が非線形性を足し、単なる行列積以上の表現力を持つ。

トレードオフと限界。第一に、1440 -> 256 は情報の圧縮なので、細かい特徴の一部は失われうる(次元削減の宿命)。第二に、backbone を替えてチャネル数が変わると、入力チャネル数を正しく与える必要がある。これを誤ると形状エラーになります(`feature_info` などから自動算出して回避する設計もあります)。第三に、1x1 Conv は空間方向に情報を混ぜないので、空間的な文脈を取り込む役割は果たせません。あくまでチャネル次元の調整に特化した部品です。

研究上の論点として、「次元削減でどれだけ情報が失われるか」は本質的に未解決です。情報理論的にはボトルネックが情報を絞ることが汎化に役立つ(Information Bottleneck 仮説, Tishby & Zaslavsky 2015)という見方もあれば、絞りすぎると性能が落ちるという実証もあり、最適な圧縮率はタスク依存です。

## 発展トピック・研究の最前線

Depthwise Separable Convolution(MobileNet, Howard et al. 2017; Xception, Chollet 2017)は、空間方向の畳み込み(depthwise)とチャネル方向の 1x1 Conv(pointwise)を明確に分離し、計算量を劇的に削減しました。1x1 Conv がチャネル混合専用の部品として確立した好例で、MobileNet は通常の畳み込みに対し計算量を8〜9分の1に削減しながら同等精度を保ちます。

Transformer の文脈では、channel projection に相当する操作が「全結合射影(linear projection)」として至る所に現れます。Vision Transformer のパッチ埋め込み、Attention の Q/K/V 射影、Feed-Forward 層は、いずれも本質的に「チャネル(特徴)次元を変える 1x1 的な線形射影」です。1x1 Conv と 全結合層が数学的に等価であることを理解すると、CNN と Transformer の橋渡しが見えてきます。ConvNeXt(Liu et al., 2022)は ResNet を Transformer の設計思想で近代化する中で、1x1 Conv(pointwise)を MLP として再解釈し、両者の連続性を強調しました。
