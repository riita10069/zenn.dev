# egomotion信号の導出（速度・加速度・ヨーレート・曲率）

## ひとことで言うと

自車(エゴ、ego = 自分の車)が「どう動いているか」を少数の数値で表したものを egomotion(自車運動)信号と呼びます。代表的には速度・加速度・ヨーレート(向きの回り具合)・曲率の 4 つです。これらが生データに直接記録されていないことは多く、その場合は車の位置や向きの時系列から有限差分(隣り合う時刻の引き算)で計算します。この導出を、データセットが持つ情報量に応じて使い分けるのがこのトピックです。

## 直感的な理解

自動運転モデルは「自分が今どう動いているか(過去の運動)」を入力として受け取り、「これからどう動くべきか(未来の運動)」を出力します。つまり運動を数値で表す共通言語が要ります。

ところが、データの提供元によって記録されている情報がばらばらです。あるデータセットは速度計とアクセル開度を記録しているが向きの回転速度は記録していない。別のデータセットは GPS/IMU(慣性計測装置)で計算した自車の位置と姿勢(ポーズ)しか持たない。それでもモデルには毎回同じ 4 つの数値を渡したい。だから各データセットのパーサが、足りない信号を「位置・向きの時間変化」から自前で計算して 4 信号にそろえます。これにより、1 つのモデルが複数のデータセットを横断して学べるようになります([L2D LeRobotデータセットパーサ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0821_l2d-lerobot-dataset-parser) のデータセット非依存設計と直結)。

有限差分とは何かを直感で。連続的な変化率(微分)を、コンピュータ上では「隣り合う 2 時刻の値の差を時間で割る」近似で求めます。位置が 0.1 秒で 1m 進めば、速度はおよそ 1m ÷ 0.1s = 10 m/s。歩いていて 1 歩の前後で位置がどれだけ変わったかを見て歩く速さを推し量るのと同じです。これが有限差分です。

## 基礎: 前提となる概念

速度・加速度。速度(speed)は単位時間あたりの位置変化、加速度(acceleration)は単位時間あたりの速度変化です。加速度が前後方向(縦)なら「踏み込み/ブレーキの強さ」、左右方向(横)なら「カーブで体が振られる横 G」に対応します。

ヨー角とヨーレート。ヨー角(yaw、heading)は車が水平面でどちらを向いているかの角度。ヨーレート(yaw rate)はその角度の時間変化率で、車が単位時間にどれだけ回頭しているか(rad/s)を表します。右左折やカーブで大きくなります。

曲率(curvature)。その瞬間に車が描いている円の半径の逆数 κ = 1/R です(κ はカッパ、R は半径)。きついカーブ(半径小)ほど曲率は大きく、直進(半径無限)では曲率 0。重要なのは、曲率は速度に依存しない形状量だということ。同じヨーレートでも、高速ならゆるいカーブ(曲率小)、低速ならきついカーブ(曲率大)になります。だから「どんな形の経路を描くか」を速度から切り離して表現でき、予測対象として扱いやすい。

ポーズ(pose)と四元数。ポーズは位置(3 次元)+ 姿勢(向き)の組です。姿勢の表現には四元数(quaternion、回転を 4 つの数で表す形式)がよく使われます。四元数はジンバルロック(特定の姿勢で回転の自由度が縮退する問題)を避けられる利点があり、必要に応じてオイラー角(回転を 3 つの角度で表す形式)に変換して、平面走行で意味のあるヨー成分を取り出します。

中心差分と前進差分。有限差分にはいくつか種類があります。前進差分は `(f[t+1]-f[t])/dt`、中心差分は `(f[t+1]-f[t-1])/(2dt)`(dt は時間刻み)。中心差分は前後両方の隣を使うため精度が一次高く(打ち切り誤差が dt の 2 乗オーダー)、両端だけ片側差分にフォールバックします。NumPy の `np.gradient` はこの中心差分を実装し、しかも不等間隔のタイムスタンプにも対応します。

## 仕組みを詳しく

導出のやり方は、データセットがどこまで信号を持っているかで変わります。2 つの典型的なケースを示します。

ケース A: 速度と加速度を直接持つデータセット。車両状態が `[speed, heading, ..., accel_x, accel_y]` のように速度・加速度を含む場合、それらはそのまま使い、不足するヨーレートと曲率だけを計算します。等間隔サンプリング(例: 10Hz, dt=0.1s)なら単純な前進差分で十分です。

```
speed   = state[:, "speed"]                 # 既にある。そのまま
accel_x = state[:, "accel_x"]               # 既にある。そのまま
heading = np.unwrap(state[:, "heading"])    # 向き。unwrapで±πの飛びをならす
yaw_rate = np.zeros_like(heading)
yaw_rate[1:] = np.diff(heading) / 0.1       # 向きの変化率。境界はゼロ埋め
curvature = np.where(np.abs(speed) > 1e-6,  # ゼロ割回避
                     yaw_rate / speed, 0.0)
```

ケース B: ポーズ(位置 + 姿勢)しか持たないデータセット。速度も加速度も記録されていない場合は、4 信号すべてを位置と姿勢の時系列から導出します。タイムスタンプが厳密に等間隔とは限らないので、実時刻を渡せる `np.gradient`(中心差分)を使うのが頑健です。

```
velocity_xy  = np.gradient(translations[:, :2], t_s, axis=0)  # 地面方向の速度ベクトル
speed        = np.linalg.norm(velocity_xy, axis=1)            # その大きさ = 速さ
acceleration = np.gradient(speed, t_s)                        # 速さの時間微分
yaw_angle    = Rotation.from_quat(quats).as_euler("ZYX")[:,0] # 四元数→ヨー角
yaw_rate     = np.gradient(np.unwrap(yaw_angle), t_s)         # ヨー角の時間微分
speed_safe   = np.where(speed < 0.1, 0.1, speed)              # 速さの下限を固定
curvature    = yaw_rate / speed_safe                          # 曲率
```

共通の出力切り出し。どちらのケースも最終的に `(T, 4)` の `[速度, 加速度, ヨー(レート/角), 曲率]` を作り、サンプル点 `idx` の前 H ステップを過去運動履歴、後 H ステップのうち予測対象の信号を正解として切り出します(H=64 の例)。

```
history = signals[idx-64 : idx]       # (64,4) -> flatten -> (256,)
future  = signals[idx+1 : idx+1+64]   # (64,4)
target  = future[:, [1, 3]]           # 加速度(列1)と曲率(列3) -> flatten -> (128,)
```

数値例で確認します。10Hz で 1.0 → 1.05 → 1.12 m と進む位置から速度を中心差分すると、t=1 では `(1.12 - 1.0)/(2×0.1) = 0.6 m/s`。さらにこの速度列を微分すれば加速度が出ます。微分を 2 回重ねるほどノイズが増えるのが後述の難所です。

なぜ予測対象が加速度と曲率の 2 つだけなのか。この 2 つで車の縦の動き(加速・減速)と横の動き(曲がり)を表せ、運転行動として十分だからです。速度やヨー角そのものは加速度・曲率を時間積分すれば復元できる従属量なので、独立に予測する必要がありません。これは運動学的自転車モデルの発想で、制御入力(縦加速度と操舵に相当する曲率)だけで軌跡が決まるという考え方です。予測の自由度を減らすことは、学習を安定させ、出力が物理的に妥当な範囲に収まりやすくする効果もあります。

ゼロ割回避(曲率)。曲率 = ヨーレート ÷ 速度なので、停車中(速度 0)はゼロ割で無限大や NaN(非数)が出ます。速度が極小なら曲率を 0 にする、または速度に下限(例 0.1 m/s)を設けて割る、のいずれかで防ぎます。停車中は「曲がっていない」とみなすのが物理的に自然なので、どちらも妥当です。NaN が 1 つでも紛れると、その勾配が全パラメータを汚染して学習が即座に破綻するため、この防御は必須です。

ヨー角の前処理。ヨーレート計算の直前には必ず `np.unwrap` を通します。向きが ±π の境界をまたぐと差分が巨大な偽スパイク(とげ)になるためで、これは独立した重要トピックとして [np.unwrapによるヨー角の±π巻き戻し処理](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0852_yaw-rate-unwrap) で詳述します。

## 手法の系譜と主要論文

egomotion 導出は単一の論文の発明ではなく、車両運動学とデータセット設計の慣行が積み重なったものです。

運動学的自転車モデル(kinematic bicycle model)。車両の前後輪を 1 本ずつに集約した古典モデルで、状態を位置・ヨー角・速度、入力を加速度と操舵角(または曲率)で記述します(Rajamani "Vehicle Dynamics and Control", Springer 2011 が標準的教科書)。動機は「少ないパラメータで車の運動を実用十分に表せる」こと。トレードオフは、タイヤの横滑りや荷重移動といった動力学(dynamics、力を考慮した運動)を無視する単純化で、高速・急操舵では誤差が出ます。運転を「加速度 + 曲率」で表す設計はこのモデルの直系です。

KITTI(Geiger ら、Karlsruhe / Toyota、CVPR 2012)。自動運転研究の草分け的データセットで、GPS/IMU(OXTS)で同期した高精度ポーズを提供します。速度・加速度・ヨーレートは生のポーズや IMU から導出するのが通例という慣行を広めました。

nuScenes(Caesar ら、CVPR 2020、arXiv:1903.11027)。同様に ego pose を提供し、研究者が各自の必要に応じて運動量を導出します。設計思想は「全派生量を保存するより、生のポーズ + 導出ルールの方が汎用的で再現可能」というもの。トレードオフは、利用者側で微分のノイズ処理を考える必要があること。

数値微分のノイズ増幅(数値解析の一般論)。位置センサのノイズは、速度・加速度と微分を重ねるほど増幅されます。差分はノイズの高周波成分を強調するため、平滑化(移動平均、Savitzky-Golay フィルタ、カルマンフィルタ)や、精度の高い中心差分の併用が定石として知られています。Savitzky-Golay フィルタ(Savitzky & Golay, Analytical Chemistry 1964)は、局所的に多項式を最小二乗フィットして微分する手法で、単純差分よりノイズに強く微分値を得られるため、運動信号の平滑微分に広く使われます。

## 論文の実験結果(定量データ)

導出手法そのものを単体評価する論文は少ないですが、関連する定量的知見を挙げます。

中心差分 vs 前進差分の精度。数値解析の標準結果として、前進差分の打ち切り誤差は O(dt)、中心差分は O(dt²) です。dt=0.1s なら、中心差分は前進差分よりおよそ 1 桁高い精度を期待できます。一方、両者ともノイズには弱く、ノイズの標準偏差 σ・時間刻み dt のとき、差分後の速度ノイズは概ね σ/dt のオーダーに増幅されます。dt が小さいほど差分の打ち切り誤差は減るがノイズ増幅は増える、という根本的なトレードオフがあり、運転データの 10Hz(dt=0.1)はこのバランスとして実用的な点です。

ポーズ品質の影響。KITTI や nuScenes の ego pose は GPS/IMU 融合で cm〜dm オーダーの精度がありますが、トンネルや高層ビル街では GPS マルチパス(信号反射)でポーズが数 m 跳ねることが報告されています。この跳ねを素の差分にかけると非現実的な加速度(数十 m/s²、重力加速度の数倍)が出るため、外れ値除去やフィルタが実務で必要になります。導出信号の品質は元ポーズの品質に強く依存する、というのが定量的な教訓です。参考として、乗用車の通常運転の縦加速度は概ね ±3 m/s² 以内、急ブレーキでも ±8 m/s² 程度なので、これを大きく外れる値は外れ値として疑えます。

軌跡予測ベンチマークでの間接効果。Argoverse や Waymo Open Motion の予測タスクでは、入力に速度・加速度・ヨーレートを与えると、位置のみの入力に比べて短期予測の minADE(複数候補のうち最良の平均変位誤差、メートル単位、小さいほど良い)が改善する傾向が報告されています。運動信号が「次にどう動くかの慣性」を明示的に伝えるためで、egomotion を正しく導出する価値を裏づけます。

## メリット・トレードオフ・限界

メリット:
- データセットが持つ情報量に関係なく、モデルへ渡す 4 信号を統一でき、モデルがデータセット非依存になる。
- 曲率を速度で正規化して持つので、車速に依存しない曲がり具合(形状)を表現でき、予測対象として扱いやすい。
- ゼロ割を明示的に防ぐので、停車・低速シーンでも数値が壊れない。

トレードオフ・限界:
- 有限差分は位置・向きのノイズを増幅する。生データのノイズが大きいと導出信号が荒れ、学習の正解(教師信号)が汚染される。汚れた正解で学べばモデルも汚れた挙動を真似る。
- 等間隔前提の単純差分はタイムスタンプの揺らぎに弱い。実時刻を使う中心差分(`np.gradient`)の方が頑健。
- ヨー角のオイラー変換にはジンバルロックの注意点があるが、平面走行のヨー成分のみ使うなら実害は小さい。
- 未解決の課題として、素の有限差分は最適とは限らず、ノイズ特性に応じたフィルタ設計(カルマンフィルタ、平滑化スプライン)とのトレードオフ、リアルタイム処理での遅延(平滑化は過去情報を多く使うほど遅れる)が残る。平滑化を強めるほどノイズは消えるが、急ブレーキのような本物の急変も鈍ってしまう。

## 発展トピック・研究の最前線

状態推定としての egomotion。本来 egomotion は「観測から潜在的な運動状態を推定する」問題で、拡張カルマンフィルタ(EKF)や Factor Graph 最適化(GTSAM など)で IMU・GPS・ホイールオドメトリ・視覚オドメトリを融合するのが本格的なやり方です。素の有限差分はその最も軽量な近似に当たります。フィルタはノイズの統計モデルを仮定して最適に平滑化するため、単純差分より大幅に頑健ですが、計算と設計のコストが上がります。

学習ベースのオドメトリ。近年は深層学習で映像や LiDAR から直接 egomotion を回帰する手法(DeepVO, Wang ら ICRA 2017; TartanVO など)が発展し、古典的な幾何推定と融合するハイブリッドが研究されています。GPS の効かない屋内・トンネルでも動く利点があります。

予測の入力表現。egomotion をどう正規化・符号化してモデルに与えるか(絶対座標 vs 自車中心座標、速度を成分で持つか大きさ+向きで持つか)も性能に効くため、表現学習の観点で研究が続いています。自車中心(ego-centric)座標で運動を表すと、平行移動・回転に対する不変性が得られ汎化が向上することが知られています。つまり「世界のどこにいるか」ではなく「自分から見て前後左右にどう動くか」で表す方が、別の場所・別の向きでも同じ知識を再利用できます。
