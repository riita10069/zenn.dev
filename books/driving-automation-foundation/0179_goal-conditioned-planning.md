# ゴール条件付き計画（先に目標点を予測し、そこへ至る軌道を生成する2段構成）

## ひとことで言うと
「いきなり未来の軌道全体を当てる」のではなく、まず「数秒後にどこにいるか（ゴール、目標点）」を予測し、そのゴールへたどり着く軌道を後から描く、という2段階のやり方です。難しい長期予測を「どこへ向かうか（where）」と「どう向かうか（how）」に分けることで、特に長い時間先の予測が安定して正確になります。多峰性（複数のありうる未来）を「ゴールの多峰性」に肩代わりさせるのが核心です。

## 直感的な理解
軌道予測では、予測する時間が長くなるほど誤差が爆発的に増えます。3秒先ならまだしも、6秒・8秒先となると、わずかな初期のずれがどんどん広がり、しかも「右折／直進／停止」のような選択肢の枝分かれが増えて、何が起きてもおかしくなくなります。1本の軌道を直接回帰させると「右折と直進の平均」のような非現実的な軌道（モード平均化、[CVAE 軌道生成（条件付き変分オートエンコーダによる多峰な未来の生成）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0165_cvae-trajectory-generation) 参照）が出てしまう。

人間の運転を観察すると、別の見方ができます。ドライバーはまず「あの交差点を右に曲がろう」「次の出口で降りよう」と目的地・行き先を決め、それから「どう曲がるか」の細かい動きを考えます。つまり「ゴールを決める」が先で「経路を引く」が後です。

ゴール条件付き計画（goal-conditioned planning / target-driven prediction）は、この人間的な順序を機械にやらせます。多峰性の難しさを「ゴールの多峰性（右の角・直進先・左の角という複数の候補地点）」に押し込め、各ゴールが決まれば、そこへの経路は比較的素直に1本に定まる、という分解です。「どこ」の不確実性と「どう」の不確実性を切り離すのがミソです。難問を「分類問題（どのゴールか）」と「やさしい回帰問題（そこへの経路）」に分けている、と言い換えられます。

## 基礎: 前提となる概念

### 軌道予測の評価指標
このトピックを理解するには評価指標を知る必要があります。
- minADE_K（メートル）: K 本の予測のうち最良の1本の、全タイムステップ平均の位置誤差。
- minFDE_K（メートル）: 最良の1本の、最終時刻の位置誤差。長期の到達点ズレを直接測る。ゴール条件付き手法はまさにこの「最終到達点」を直接予測するので、FDE と相性がよい。
- Miss Rate（MR, ミス率）: 最良の予測の最終点が正解から一定距離（例 2m）以上外れた割合。低いほど良い。

### シーンエンコーダ（地図と他車を読む土台）
ゴールを「車線の上」に置くには、地図と周囲エージェントを数値化する仕組みが要ります。これにはベクトル表現のエンコーダ（VectorNet のように車線や軌道を折れ線ベクトルの集合として符号化するもの、あるいは LaneGCN のように車線グラフで畳み込むもの）が使われます。[LaneGCN（車線グラフ畳み込みによる動き予測）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0444_lanegcn-planning) を参照。ゴール条件付き手法はこのエンコーダの「上に乗る」予測ヘッドだと考えると理解しやすい。エンコーダが出すのは典型的に「対象エージェントの状態を要約した文脈ベクトル」と「車線セグメントごとの特徴ベクトル」で、ゴール段はこの2種を突き合わせて「どの車線地点が有望か」を採点する。

### 分類と回帰の分解という考え方
このトピックの本質は「難しい連続予測（回帰）を、やさしい離散選択（分類）と素直な回帰に分ける」ことです。分類（classification）とは「いくつかの選択肢のどれか」を当てる問題、回帰（regression）とは「連続値（座標など）」を当てる問題です。1本の軌道を端から端まで回帰させると、モードの分岐を回帰だけで表さねばならず破綻しやすい。ゴール条件付けは「どの地点をゴールにするか」を分類（または密な確率推定）で扱い、ゴールが決まった後の経路だけを回帰にするので、各段が得意な問題だけを解けるようになります。この「分解で難しさを下げる」発想は、機械学習全般で繰り返し現れる強力な設計原理です。

## 仕組みを詳しく

### 2段構成の流れ（TNT を例に）
ゴール条件付き手法は典型的に次の段階を踏みます。

第1段: ゴール候補の生成
- シーン（地図・他車）を見て、エージェントが数秒後にいそうな地点の候補をたくさん出します。
- TNT の賢い点は、ゴール候補を「車線の上」に置いたことです。地図の車線中心線に沿って多数の候補点（例: 数百〜1000個）をばらまく。車は道路の上を走るので、ゴールも道路上にあるはず。空中や建物の中にゴールを置く無駄がなくなります。

第2段: ゴールの確率推定と絞り込み
- 各ゴール候補に「ここに到達する確率」をスコアづけし、ありえない候補を捨て、有望な上位だけ（例: 上位 6〜50 個）残します。

第3段: ゴールへの軌道生成
- 残った各ゴールを条件として、現在地からそこへ至る完全な軌道（間の全タイムステップの位置）を生成します。1つのゴールにつき1本の素直な軌道が引けます。

第4段（任意）: 軌道のスコアリング・選択
- 生成した複数軌道を評価し、最終的に出力する K 本を選びます（重複や非現実的なものを NMS 的に間引く）。

### 数値・形状の具体例
```
入力     : VectorNet/LaneGCN系エンコーダのシーン特徴
ゴール候補 : 車線上に M 個の点 (例 M=1000)、各点に到達確率スコア [M]
絞り込み後 : 上位 K=6 ゴール  [6, 2]
出力     : 各ゴールにつき未来軌道 [30, 2] (30ステップ × xy)、計 [6, 30, 2] + 確率 [6]
```
擬似的な流れ:
```
goals, scores = goal_head(scene_feat)     # M個の候補と確率
topk = select_topk(goals, scores, K=6)    # 6個に絞る
for g in topk:
    traj = traj_completion(scene_feat, g) # ゴール条件つき軌道補完
```

### DenseTNT の改良点
TNT は「車線上に離散的な候補点を置く」方式でしたが、候補点の置き方（密度・粗さ）によって精度が頭打ちになる弱点がありました。候補を粗くすると正解の真のゴールが候補に含まれず、細かくすると計算が増える。DenseTNT は、離散候補に頼らず「車線上の連続的な（dense な、密な）目標確率分布」を直接推定し、そこから目標点を取り出します。さらにアンカー（あらかじめ決めた候補）に依存しない設計（anchor-free）にして、候補点の置き方への感度をなくしました。最終ゴールの選択には最適化（local maxima を選ぶ goal set predictor）を使います。

### なぜ精度が上がるのか（直感）
長期予測の誤差の大半は「どっちの方向に行くか」の枝分かれから来ます。ゴールを先に決めると、その枝分かれが第1段で確率つきに整理され、第3段の「経路を引く」は方向が決まった素直な問題になります。難問を分類（どのゴールか）と回帰（そこへの経路）に分解しているわけです。

## 手法の系譜と主要論文

- PECNet（Mangalam, Girase, Agarwal, Lee, Adeli, Malik, Gaidon, "It Is Not the Journey but the Destination: Endpoint Conditioned Trajectory Prediction", ECCV 2020）: 歩行者予測で「endpoint（終点）を CVAE で先に予測し、終点を条件に経路を埋める」手法。ゴール条件付けを VAE で実現した先駆け。終点を決めると経路が定まるという同じ発想。
- TNT（Zhao et al., "TNT: Target-driveN Trajectory prediction", CoRL 2020、Waymo）: 車両予測で「ターゲット（ゴール）を先に予測する」発想を体系化した代表。ゴール候補生成 → ゴール確率推定 → ゴール条件つき軌道生成 → スコアリング、の3〜4段構成。
- DenseTNT（Gu, Sun, Zhao, "DenseTNT: End-to-end Trajectory Prediction from Dense Goal Sets", ICCV 2021、Waymo）: TNT の後継。離散候補に頼らず密な目標確率を推定する anchor-free 設計。候補設計への感度を解消しつつ高精度を達成。
- LaneGCN / VectorNet: ゴール条件付き手法のシーンエンコーダとしてよく組み合わされる。地図と他車を表現する土台（[LaneGCN（車線グラフ畳み込みによる動き予測）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0444_lanegcn-planning)）。
- Goals as anchors の一般化: 後年の Transformer 系（mmTransformer, Wayformer, MTR）でも「アンカー軌道」「アンカー目標」という形でゴール条件付けの思想が広く取り込まれた。MTR（Motion Transformer, Shi et al., NeurIPS 2022）は学習可能な intention query（意図クエリ）でゴール条件付けを Transformer に統合し、Waymo ベンチで上位を占めた。

## 論文の実験結果（定量データ）

### ベンチマーク
主戦場は Argoverse Motion Forecasting（自動運転車が集めた都市走行データで、過去 2 秒から未来 3 秒を予測）と Waymo Open Motion Dataset です。Argoverse では K=6 本の予測で minADE / minFDE / Miss Rate を競います。

### TNT の結果
報告によれば、TNT は Argoverse forecasting で、それ以前の回帰ベース・LSTM ベース手法を上回り、当時のリーダーボード上位の成績を出しました。Argoverse test の K=6 評価で minADE はおよそ 0.9 m 弱、minFDE はおよそ 1.5 m 前後と報告され、当時の代表的な回帰手法（minFDE が 1.7〜2.0 m 級）を明確に下回りました。特に minFDE（最終到達点誤差）と Miss Rate での改善が顕著で、これは「ゴールを直接予測する」設計が最終点予測に効くことを直接示します。歩行者の Stanford Drone Dataset でも、終点を先に予測する方式が長期予測の誤差を下げることが確認されています。数値の意味を補うと、minFDE が 0.3〜0.5 m 縮むことは、3〜5 秒先の到達点予測で「車1台分（おおむね 1.8 m 幅）の半分弱」ぶん正解に近づくことに当たり、下流の計画で「相手が自車レーンに来るか隣レーンに留まるか」を分ける実用上重要な差になります。

### DenseTNT の結果
DenseTNT は Argoverse のリーダーボードで TNT を上回り、特に anchor-free 化によって「候補点の密度をチューニングしなくても」高精度が出る点が定量的に示されました。報告では Argoverse test で minFDE（K=6）が当時の上位水準（おおむね 1.0〜1.3 m）に達し、Miss Rate（2 m 閾値）も TNT 比で低減しています。離散候補方式の TNT は「候補点を細かくするほど精度は上がるが計算が増え、ある密度で頭打ちになる」という頭打ち（saturation）が観測されていましたが、DenseTNT の密な確率推定はこの頭打ちを越えました。Waymo Open Motion Dataset の 2021 Motion Prediction チャレンジでも上位の成績を収めています。

### アブレーション（どの要素を抜くとどうなるか）
TNT/DenseTNT 系のアブレーションが共通して示すのは次の3点です。
1. ゴール段を外して直接1本回帰にすると、長期の minFDE・Miss Rate が悪化する。ゴール分解が多峰性の取りこぼしを防いでいる。
2. TNT で候補点を粗くすると、真のゴールが候補に入らず誤差が増える（候補密度への感度）。DenseTNT の anchor-free 化はこの感度を消すことで改善を示した。
3. ゴール条件つき軌道補完を外して「ゴールだけ」評価すると FDE は良いが ADE（途中経路の誤差）が悪化する。経路補完段が「どう向かうか」を担っていることが確認される。

## メリット・トレードオフ・限界

メリット
- 長期予測の精度と安定性が上がる。難問を「どこへ（分類）」と「どう（回帰）」に分解できるため、特に minFDE・Miss Rate に効く。
- 多峰性をゴールの多峰性として自然に表現でき、モード平均化を避けやすい。
- ゴールを車線上に置くなど、地図の事前知識を素直に組み込める。
- 出力に「各ゴールの確率」が付くので、下流の計画で確率的に扱いやすい。

トレードオフ・限界
- 段が多く、実装・学習・推論のパイプラインが複雑になる。
- TNT 系ではゴール候補の置き方・個数の設計が精度に効くため調整が要る。DenseTNT はこれを緩和するが密な確率推定と最適化ベースの目標選択で計算が増える。
- ゴール予測が外れると、その後の軌道生成も連鎖的に外れる（誤りの伝播、error propagation）。
- 未解決の課題: ゴールは静的な「点」を仮定するが、相互作用で目的地自体が動的に変わる状況（駆け引き、譲り合い）を素直に表現しにくい。複数エージェントのゴールを同時に整合させる joint prediction も難所。

## 発展トピック・研究の最前線
- query ベースの統合: MTR や QCNet のように、学習可能なクエリでゴール条件付けを Transformer に内包する設計が主流化。離散候補のばらまきが不要になり、end-to-end に最適化できる。
- joint / scene-level 予測: 各エージェントを独立に予測するのではなく、全員のゴールと軌道を同時に整合させる（衝突しない組み合わせを出す）研究が進む。
- 計画への接続: ゴール条件付けは予測だけでなく自車の計画にも使える。「自車のゴール候補を出し、各ゴールへの軌道を生成しスコアで選ぶ」という構造は、提案＋採点型プランナー（[PDM（ルールベース予測駆動プランナー）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0197_pdm-rule-based-planner) の生成＋スコアリングや CVAE の生成＋ランキング）と発想を共有する。
- closed-loop 評価: open-loop の minFDE が良くても実走で安全とは限らない、という指摘（[PDM（ルールベース予測駆動プランナー）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0197_pdm-rule-based-planner)）を踏まえ、ゴール条件付き手法も closed-loop での検証が課題になっている。
