# 逐次フレームウィンドウ（過去・未来）アクセス

## ひとことで言うと

走行ログ(連続した映像と車両状態の記録)から、「今この瞬間」を中心にして前後の一定数のフレームをまとめて切り出す仕組みです。過去のフレームは「これまでどう走ってきたか」を、未来のフレームは「このあとどう走るべきか(正解の軌道)」をモデルに教えるために使います。切り出すときは、別々の走行(エピソード)の境界を絶対にまたがないようにします。これは、時系列を入力に取るあらゆる運転モデル、とくに未来予測を学ぶ世界モデルや自己教師あり学習の土台になる、地味だが本質的な前処理です。

## 直感的な理解

人間が運転するとき、目の前の一瞬の景色だけで操作を決めているわけではありません。「さっきまで加速していた」「今カーブの途中だ」「数秒前に対向車が来た」といった時間的な文脈を踏まえて、次にどうするかを決めます。1枚の静止画だけからでは、車が止まっているのか猛スピードなのか、右に曲がっている最中なのかすら判別できません。だからモデルにも、過去の状態の流れと現在の景色の両方を見せ、未来の軌道を当てさせます。

ここで「全部見せればいい」とはいきません。10分の走行を 10Hz(1秒に10枚)で記録すると 6000 フレームになり、これを丸ごとモデルに入れるのはメモリ的に不可能で、学習も不安定です(系列が長いほど勾配の伝播が難しく、計算量も系列長に比例または二乗で増える)。そこで、ログのある1点を「現在」と定め、その前後だけを固定長で切り出します。この切り出した区間が「フレームウィンドウ(窓)」です。窓を1フレームずつスライドさせれば、1本のログから大量の学習サンプルを作れます(スライディングウィンドウ)。

もう一つ大切なのが「エピソード境界をまたがない」という制約です。データセットは独立した複数の走行(エピソード)の寄せ集めです。エピソードAの最後とエピソードBの最初を繋いで1つの窓にすると、別の場所・別の時刻の映像が連続しているように見え、モデルは「ありえない時間のつながり」を学んでしまいます。たとえば「直進中に突然まったく違う交差点へワープした」ような系列を正解として学習させると、未来予測がめちゃくちゃになります。物理的にありえない遷移を「正解」として教え込むと、世界モデルの予測分布が壊れるのです。だから窓は必ず1つのエピソードの内側で完結させます。

## 基礎: 前提となる概念

- エピソード(episode): 1回の連続した走行記録。開始から終了まで時間的に途切れない一区間。データセットは多数のエピソードの集合。ロボット学習では「軌跡 (trajectory)」とも呼ぶ。
- フレーム(frame): ある時刻の観測一式(全カメラ画像 + 車両状態)。時刻 t のスナップショット。
- 過去ウィンドウ(history): 現在より前のフレーム列。モデルへの入力の一部。
- 未来ウィンドウ(future): 現在より後のフレーム列。教師信号(正解)に使う。
- 教師信号(supervision target): モデルに当てさせる正解。ここでは未来の運動(加速度・曲率)や未来フレームの特徴。
- 模倣学習(imitation learning): 人間の運転デモを正解として、それを真似るようにモデルを学習させる枠組み。未来軌道を当てる学習はこの一種(behavior cloning)。
- 世界モデル(World Model): 現在の状態から未来の状態を予測する内部モデル。学習には過去観測と未来観測のペアが要る。「頭の中でシミュレーションする」能力に相当。
- JEPA(Joint Embedding Predictive Architecture): 未来や欠損部分を、生の画素ではなく「画像を圧縮した特徴ベクトル(埋め込み)」で予測する自己教師あり学習方式。

## 仕組みを詳しく

10Hz 記録のデータセットを例に、過去 64・未来 64 ステップの窓を作る処理をステップで見ます。定数の意味は次の通り。

- HISTORY_TIMESTEPS = 64(過去 64 ステップ)
- FUTURE_TIMESTEPS = 64(未来 64 ステップ)
- MIN_FRAMES = 64 + 64 + 1 = 129(1サンプルに最低必要なフレーム数。過去 + 現在 + 未来)

ステップ1: エピソードの範囲を求める。
各フレームに付いた `episode_index`(どのエピソードに属するか)を読み、各エピソードがテーブルの何行目から何行目までかを `{エピソード番号: (start, end)}` の辞書にします。例: エピソード0が行 0〜299、エピソード1が行 300〜599。

ステップ2: 有効なサンプル点だけを列挙する。
あるエピソードの長さが MIN_FRAMES(129)未満なら、前後の窓を確保できないのでスキップ。確保できる場合、「現在」に選べる局所インデックスの範囲は次の通り。

```
min_frame = 64                 # これより前だと過去64フレームが取れない
max_frame = ep_len - 64 - 1    # これより後だと未来64フレームが取れない
```

エピソード長 300 なら、現在に選べるのは局所インデックス 64〜235(= 300 − 64 − 1)。各候補を `(episode_index, ep_start + frame_idx)` として登録します。ここで `ep_start` / `ep_end` を使って範囲を区切るため、窓が隣のエピソードへはみ出すことが構造的に起こりません。これが「境界をまたがない」の実装です。境界チェックを毎サンプルで行うのではなく、有効点リストを学習開始前に一度作っておく(インデックスの事前計算)のが効率的で、`__len__` は有効点の総数になります。

ステップ3: 1サンプルを取り出す。
現在フレームの行 `row` と、それがエピソード内で何番目か `sample_idx = row - ep_start` を計算し、過去 64・未来 64 を切り出します。

```
history = signals[sample_idx - 64 : sample_idx]            # (64, 4) 過去
future  = signals[sample_idx + 1 : sample_idx + 1 + 64]    # (64, 4) 未来
```

過去側は4信号(速度・前後加速度・ヨーレート・曲率)すべてで `64×4 = 256` 次元、未来側は教師に使う2信号(前後加速度・曲率)で `64×2 = 128` 次元、といった構成が一例です。未来側が教師信号です([自車軌道（ego trajectory）抽出](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0807_ego-trajectory-extraction))。

画像は現在フレームの全視点だけをデコードします。過去の視覚特徴(たとえば 896 次元)は、初期段階ではゼロ埋めのプレースホルダにしておき、世界モデル学習で過去フレームの特徴を実際に詰める、という段階的な拡張がよく取られます。これは「まず軌道予測だけで配線を通し、後から視覚的な過去文脈を足す」という漸進的開発の定石です。

数値イメージ(1サンプル、10Hz):

```
現在 row=120 (エピソード内 sample_idx=120)
 ├ 過去ウィンドウ: sample_idx 56..119  (64ステップ, 6.4秒分)
 ├ 現在: sample_idx 120 の全視点画像
 └ 未来ウィンドウ: sample_idx 121..184 (64ステップ, 6.4秒分) = 教師
```

なお、窓の各ステップが何秒に相当するかは元のフレームレートに依る点に注意します。複数データセットを混ぜるなら、各ステップの時間間隔を揃えるため間引き(ストライド)を併用します([フレームレート変換ストライド（時間ダウンサンプリング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0839_stride-for-hz-downsampling))。たとえば 10Hz の生データを 1Hz の窓にするなら、現在 t から `t ± stride, t ± 2*stride, ...` と stride=10 飛ばしで拾い、64ステップで 64 秒ぶんの粗い文脈を作る、といった使い分けになります。

## 手法の系譜と主要論文

- ALVINN(Pomerleau, "ALVINN: An Autonomous Land Vehicle in a Neural Network", NeurIPS 1989、CMU)。最初期のエンドツーエンド運転模倣。単一フレームの画像から操舵を予測する単純な3層ネットで、時間的文脈を持ちませんでした。後続研究はここに「過去の状態系列」を足していきます。歴史的出発点として重要です。

- PilotNet(Bojarski et al., "End to End Learning for Self-Driving Cars", 2016、NVIDIA、arXiv:1604.07316)。CNN で前方カメラ1枚から操舵角を直接回帰し、実車で車線維持を実証しました。これも基本は単一フレームですが、大規模実車データでエンドツーエンド模倣が動くことを示し、時系列・未来予測へ拡張する流れの土台になりました。

- ChauffeurNet(Bansal et al., "ChauffeurNet: Learning to Drive by Imitating the Best and Synthesizing the Worst", RSS 2019、Waymo、arXiv:1812.03079)。過去の自車軌跡・周辺物体・地図をラスタ画像として複数フレーム積み重ね、未来軌道を予測しました。過去ウィンドウを明示的に入力へ含めることで、単一フレームでは取れない速度・進行方向の文脈を与える発想を示しました。また、わざと悪い例(逸脱・衝突に近い状況)を合成して学習に混ぜることで(synthesizing the worst)、模倣学習が「いつも完璧なデモしか見ない」ことの脆さ(分布シフトに弱い)を補いました。

- MILE(Hu et al., "Model-Based Imitation Learning for Urban Driving", NeurIPS 2022、Wayve、arXiv:2210.07729)。世界モデルを都市運転に持ち込み、潜在状態の遷移を学習して未来を内部予測しました。学習には「過去観測の系列」と「未来観測」を時系列で切り出すデータが必須で、本トピックの窓機構はこの種の手法の前提です。

- LeCun "A Path Towards Autonomous Machine Intelligence"(2022)と I-JEPA(Assran et al., "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023、Meta AI、arXiv:2301.08243)。JEPA の考え方を提示。未来や欠損部分を画素ではなく特徴空間で予測します。運転に応用すると「未来フレームの特徴」を教師にするため、未来ウィンドウの抽出が学習の中核になります。特徴予測は「何を特徴に残すか」をエンコーダ任せにするため、対策(stop-gradient で片側の勾配を止める、EMA=指数移動平均でターゲットエンコーダをゆっくり更新する、予測器を非対称に置く)を入れないと、全サンプルが同じ定数ベクトルに潰れる崩壊(representation collapse)が起きやすい点が指摘されています。

- 因果混同の理論的整理(de Haan, Jayaraman, Levine, "Causal Confusion in Imitation Learning", NeurIPS 2019、arXiv:1905.11979)。模倣学習で、過去の自分の行動(の痕跡)を入力に含めると、モデルが本当の原因(前方の信号など)でなく「過去の行動の続き」という相関に飛びつく現象を定式化しました。過去ウィンドウ設計の落とし穴を理論面から支える重要文献です。

## 論文の実験結果(定量データ)

窓そのものを評価する論文は少なく、窓を前提にした手法の結果で「時間文脈と未来教師がなぜ効くか」を見ます。

- ChauffeurNet(RSS 2019)。過去軌跡を含む複数フレーム入力と、悪い例の合成を加えることで、シミュレーション上の走行で逸脱や衝突を大幅に減らしたと報告しています。とくに「past motion dropout(過去軌跡をわざと一部のステップで隠す)」というアブレーションが重要で、過去軌跡をそのまま全部見せると、モデルが「過去をそのまま外挿するだけ」のショートカット(inertia problem、停止していると停止し続ける)を学び、画像から状況を理解しなくなる(因果混同)ことを示しました。過去ウィンドウは有用だが、頼りすぎると逆効果という、データ設計上の重要な教訓です。

- 因果混同(de Haan et al., NeurIPS 2019)。過去の行動情報を入力に増やすほど、訓練時の予測誤差は下がるのに、実際に制御を回す(クローズドループ)と性能が落ちるという逆転現象を複数環境で定量的に示しました。「訓練ロスが低い=良い方策」とは限らないという、模倣学習の核心的な警告です。

- MILE(NeurIPS 2022)。CARLA シミュレータの標準ベンチマーク(Town/天候を変えた汎化評価)で、当時のモデルベース手法として高い運転スコア(ルート完走率と違反ペナルティを統合した Driving Score)を達成し、潜在世界モデルにより未来を内部想像しながら走ることの有効性を示しました。学習データは過去・未来を切り出した時系列で構成され、窓抽出が前提になっています。

- I-JEPA(CVPR 2023)。画像分類の線形評価(凍結特徴に線形分類器だけ載せて測る protocol)で、画素再構成型(MAE 等)に比べ、同程度以下の事前学習計算量で競争力ある精度を出し、かつ意味的な特徴を学べることを示しました。「画素を当てる」より「特徴を当てる」方が、細部のノイズ(風に揺れる葉、影のちらつき)に惑わされず本質を捉えやすいという主張の定量的裏づけです。運転の未来予測へ応用する際、未来ウィンドウの特徴を教師にする設計の動機になります。

## メリット・トレードオフ・限界

メリット

- 固定長ウィンドウなのでバッチ化(テンソル化)が容易。`(B, 64, 4)` のように形が揃う。
- エピソード境界を構造的にまたがないため、ありえない時間連続を学習する事故を防げる。
- 「現在」を1フレームずつずらして大量のサンプルを生成でき、データ拡張的な効果がある。
- 有効点を事前計算しておけば、`__getitem__` が軽くなりデータ供給が速い。

トレードオフ・限界

- 窓長が固定なので、それより長い文脈(数十秒前の出来事、目的地までの大局)は捉えられない。
- エピソードの端(前後の窓ぶん)は「現在」に選べず、短いエピソードは丸ごと捨てられ、データ量がやや減る。
- 隣り合うサンプルは窓が大きく重複し相関が強いため、シャッフルしないと似たデータが連続し、勾配のミニバッチ推定が偏る。
- 過去軌跡を入力に含めすぎると因果混同(過去の外挿に頼り画像を無視)を招く。dropout 等の対策が要る。
- 未来特徴を教師にする JEPA 系は崩壊リスクがあり、安定化の工夫が必須。
- 固定 stride 前提だと、可変フレームレートやドロップフレームのある実データで時刻がずれる(最近傍選択で緩和)。

## 発展トピック・研究の最前線

- 可変長・階層的文脈。固定窓を超え、長期記憶を持つ Transformer や状態空間モデル(Mamba 等の SSM)で長い時間文脈を線形計算量で扱う方向。
- 運転向け JEPA / 世界モデル。未来フレーム特徴の予測を運転に本格適用する研究(GAIA-1/GAIA-2 など生成的世界モデル、Vista などを含む)。生成型(画素を作る)と予測型(特徴を当てる)の両系統が競っている。
- 因果混同への対処。過去軌跡 dropout、観測ドロップ、敵対的摂動、因果発見(どの入力が本当に行動の原因かを推定)など、ショートカット学習を防ぐデータ・損失設計。
- マルチモーダル窓。カメラだけでなく LiDAR・地図・言語を同期した窓で切り出し、整合した時系列を作る。センサ間のタイムスタンプ整合(同期)が前提になる。
- クローズドループでの窓設計検証。訓練ロスでなく実走行スコアで「どれだけ過去を見せるべきか」を決める評価駆動の設計。
