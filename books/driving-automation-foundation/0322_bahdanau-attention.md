# Bahdanau Attention (additive)

## ひとことで言うと
入力された文や系列を「1個の固定サイズのベクトルに無理やり押し込む」のをやめて、出力を1ステップ作るたびに「入力のどこを今どれくらい見るか」を毎回計算しなおす仕組みです。2015年に Bahdanau らが機械翻訳のために提案し、いまの Transformer の注意機構(アテンション)の直接の祖先になりました。注意(attention)とは「全部を平等に見るのではなく、必要な箇所に重みを集中させて取り出す」操作のことです。これ1つで、長文翻訳の品質が劇的に改善し、その後10年の系列モデリングの方向を決めました。

## 直感的な理解
人間が長い英文を日本語に訳すところを想像してください。文をいったん全部読んで頭の中で1つの記憶のかたまりにしてから訳しはじめる、という人はいません。実際には、いま訳している日本語の単語に対応する英語の箇所に視線を戻しながら、一語ずつ訳していきます。「この主語を訳すときはここを見る」「この動詞を訳すときはあそこを見る」という具合に、出力の各タイミングで原文の注目箇所が動いていきます。

Bahdanau attention がやっているのは、まさにこの「視線の移動」を機械にやらせることです。それ以前の翻訳モデルは、入力文の全情報を1本の固定長ベクトル(数値の並び)に圧縮してから訳していました。短い文ならこれで足りますが、長い文では先頭の情報が薄れて消えてしまいます。注意機構は「圧縮して捨てる」のではなく「全箇所を保持しておき、出力のたびに必要な箇所だけ重み付きで取り出す」ことで、この情報の取りこぼしを防ぎます。重要なのは、どこを見るかをあらかじめ人が決めるのではなく、データから自動で学習する点です。

## 基礎: 前提となる概念

まず、当時(2014年ごろ)の機械翻訳の枠組みである **Encoder-Decoder(エンコーダ・デコーダ)** を理解する必要があります。これは2つの部品からなります。

- **Encoder(符号化器)**: 入力文(例: 英語「I love cats」)を1単語ずつ読んで、内部表現に変換する部品。
- **Decoder(復号器)**: その内部表現を受け取って、出力文(例: フランス語「J'aime les chats」)を1単語ずつ生成する部品。

両者は当時 **RNN(Recurrent Neural Network、再帰型ニューラルネット)** で作られていました。RNN とは「前の時刻の状態を次の時刻に渡しながら系列を順番に処理するネットワーク」です。各時刻に **隠れ状態(hidden state)** という数百次元のベクトルを持ち、これを更新しながら系列を読み進めます。具体的には、実用上は勾配消失に強い LSTM(Long Short-Term Memory)や GRU(Gated Recurrent Unit)という改良版 RNN が使われました。

問題は橋渡しの仕方でした。Sutskever, Vinyals, Le (2014) の **seq2seq**(sequence-to-sequence)では、Encoder が出した「最後の1本のベクトル(文脈ベクトル、context vector)」だけを Decoder に渡していました。入力文が長くなると、その全情報を1本の固定長ベクトル(例: 512次元や1000次元)に詰め込むのが無理になります。10単語の文なら何とかなっても、40単語の文を512個の数に圧縮すると、先頭のほうの情報が薄れて消えてしまうのです。これを **固定長ボトルネック(fixed-length bottleneck)** と呼びます。ボトルネック(瓶の首)は、太い流れが細い口で詰まる状態を指す言葉です。

興味深いことに、seq2seq の論文では入力文を逆順に入れると精度が上がるという裏技が報告されていました。これは「文頭どうしを近づけると、文頭の情報が薄れにくい」という固定長ボトルネックの症状そのものへの対症療法であり、ボトルネックが実在する問題だったことの傍証です。実際、当時の研究では、入力文が30単語を超えたあたりから翻訳品質が急激に落ちる現象が観測されていました。翻訳品質の指標には **BLEU スコア**(Bilingual Evaluation Understudy。機械翻訳の出力が人手の正解訳とどれだけ n-gram、つまり連続する単語の並びで一致するかを 0〜100 で測る指標。高いほど良い)が使われます。

もう1つ前提として **softmax(ソフトマックス)** を押さえます。softmax とは、複数の実数(スコア)を入力すると、それぞれを「合計が1になる正の重み(確率のようなもの)」に変換する関数です。大きいスコアほど大きい重みになります。注意機構は「どこをどれくらい見るか」を softmax で重みに変えます。

## 仕組みを詳しく

Encoder が出すのは「最後の1本のベクトル」ではなく、「入力の各時刻ごとのベクトル列」です。具体例で説明します。

入力が5単語「I love cats very much」だとします。Encoder(双方向 RNN、bidirectional RNN、つまり前向きと後ろ向きの両方向に読む RNN を連結したもの)は各単語位置に対して隠れ状態ベクトルを出します。

```
h_1, h_2, h_3, h_4, h_5   各 h_j は例えば 1000次元のベクトル
(I)  (love)(cats)(very)(much)
```

これらをまとめて **アノテーション(annotation)** と呼びます。双方向にする理由は、各 h_j に「j 番目の単語の意味」だけでなく「その前後の文脈」まで含めたいからです。前向き RNN だけだと h_j は左側の文脈しか知りませんが、後ろ向き RNN を連結すると右側の文脈も入ります。

Decoder が出力の i 番目の単語を作ろうとしている瞬間を考えます。Decoder は自分の1つ前の状態ベクトル s_{i-1}(例: 1000次元)を持っています。ここで「s_{i-1} は h_1〜h_5 のどれにどれくらい関連しているか」をスコアにします。これがアテンションの中心です。

### 1. スコア計算(additive = 加算型)

Bahdanau の方式は **additive attention(加算型注意)** または concat attention と呼ばれます。各入力位置 j について、関連度スコア e_{ij} を次の小さなニューラルネットで計算します。

```
e_{ij} = v^T · tanh( W_a · s_{i-1} + U_a · h_j )
```

記号の意味を1つずつ説明します。

- s_{i-1}: Decoder の今の状態(出力 i 番目を作る直前)
- h_j: 入力 j 番目のアノテーション
- W_a, U_a: 学習で決まる重み行列。s と h をそれぞれ共通の空間(例: 1000次元)に変換する
- v: 学習で決まる重みベクトル。最後にスカラー(1個の数)へ落とすために使う
- tanh: 値を -1〜1 に潰す非線形関数

つまり「Decoder の状態を変換したもの」と「入力位置 j を変換したもの」を**足し算**し、tanh に通し、v との内積でスコアにします。足してから非線形変換するので additive(加算型)と呼びます。ここで重要な実装上の工夫として、U_a · h_j は Decoder の時刻 i に依存しないので、出力ステップに先立って一度だけ全 j について計算しておけます。W_a · s_{i-1} だけを毎ステップ計算すればよく、これが計算を現実的にしています。対比として、のちの Transformer は内積(掛け算)を使うので multiplicative / dot-product attention(乗算型・内積型)と呼ばれます。

### 2. 重みへの正規化(softmax)

スコア e_{i1}, ..., e_{i5} を softmax に通して、合計が1になる重み α に変えます。

```
α_{ij} = exp(e_{ij}) / Σ_k exp(e_{ik})
```

例えば「cats を訳すとき」なら、結果として α が

```
α_{i1}=0.05 (I), α_{i2}=0.10 (love), α_{i3}=0.70 (cats), α_{i4}=0.05, α_{i5}=0.10
```

のようになり、「今は cats に70%注目している」と読めます。この α が **アテンション重み(attention weights)** です。Bahdanau らはこれを **soft alignment(やわらかい対応付け)** と呼びました。「hard(どれか1つだけを選ぶ)」のではなく、複数箇所に小数の重みを配る「soft」だからです。soft であることには大きな意味があります。hard な選択は「どれを選ぶか」が離散的で微分できず、勾配で学習できません。soft なら α が連続的に変化し、誤差逆伝播で end-to-end に学習できます。「対応付けと翻訳を同時に学習する」というタイトルの肝はここにあります。

### 3. 文脈ベクトル(context vector)

重み α で入力ベクトル h_j を加重平均(重み付きの足し算)します。

```
c_i = Σ_j α_{ij} · h_j      c_i は 1000次元のベクトル
```

これが「今この単語を出すために必要な、入力からの要約」です。固定の1本ではなく、出力1単語ごとに違う c_i が作られるのがポイントです。Decoder はこの c_i と自分の状態 s_{i-1}、および直前に出力した単語を使って、次の状態 s_i を更新し、次の単語の確率分布を予測します。

before/after でまとめると、

- before(seq2seq): 入力全体 → 1本の固定ベクトル → そこから全出力を生成(長文で破綻)
- after(Bahdanau): 入力 → ベクトル列を保持 → 出力1ステップごとに加重和で必要な箇所だけ抽出

なぜこの設計かというと、固定長ボトルネックの本質は「圧縮の段階で情報を捨てている」ことにあるからです。注意機構は圧縮を出力時まで先送りし、しかも出力ごとに別の圧縮(別の c_i)を許すことで、必要な情報を必要なときに取り出せるようにしています。情報理論的に言えば、Encoder の出力次元を系列長に比例して増やしたことに相当し、長文ほど効きます。

## 手法の系譜と主要論文

- **Sutskever, Vinyals, Le, "Sequence to Sequence Learning with Neural Networks" (NeurIPS 2014, arXiv:1409.3215)**。注意機構の前夜。LSTM ベースの seq2seq で英仏翻訳 BLEU 34.8(アンサンブルで 36.5)を達成しましたが、固定長ボトルネックを抱えていました。この限界が次の Bahdanau を生みます。
- **Bahdanau, Cho, Bengio, "Neural Machine Translation by Jointly Learning to Align and Translate" (ICLR 2015, arXiv:1409.0473)**。注意機構の出発点。タイトルの「Jointly Learning to Align and Translate(対応付けと翻訳を同時に学習する)」が本質を表しています。翻訳の精度を上げると同時に、副産物として「どの原語がどの訳語に対応するか」というアライメントが自動で学習されます。提案は上記の加算型注意。
- **Luong, Pham, Manning, "Effective Approaches to Attention-based NMT" (EMNLP 2015, arXiv:1508.04025)**。Bahdanau の直後に、より単純な **乗算型(dot-product / general)** のスコア関数を提案しました。`score = s^T · h`(内積)や `score = s^T · W · h`(一般化内積)で済ませる形で、加算型より計算が軽く実装も簡単です。これがのちの Transformer の Scaled Dot-Product Attention につながります。Luong はさらに、入力全体を見る **global attention** と、注目位置の周辺の窓だけ見る **local attention** を整理し、前ステップの注意結果を次の入力に混ぜる **input-feeding** も導入しました。
- **Xu et al., "Show, Attend and Tell" (ICML 2015, arXiv:1502.03044)**。注意を画像キャプション生成に持ち込み、「画像のどの領域を見ながらどの単語を生成したか」を可視化しました。soft attention と、強化学習で学ぶ hard attention の両方を提案し、注意が言語以外にも有効だと示した重要な発展です。
- **Vaswani et al., "Attention Is All You Need" (NeurIPS 2017, arXiv:1706.03762)**。Bahdanau/Luong の注意を RNN から完全に切り離し、注意だけでネットワークを組めることを示しました(詳細は [Transformer 時間方向アテンション](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0344_transformer-temporal-attention))。Bahdanau がこの系譜の出発点であり、Luong が内積化の橋渡し、Vaswani が RNN を捨てる飛躍、という流れです。

## 論文の実験結果(定量データ)

Bahdanau et al. (2015) は英仏翻訳(WMT'14)で評価しました。比較対象は注意なしの RNN Encoder-Decoder(RNNencdec)で、提案手法は注意つき(RNNsearch)です。末尾の数字は訓練に使った最大文長(30 または 50)を表します。

- 最も重要な結果は **文長に対する頑健性** です。注意なしモデル(RNNencdec-30、最大30単語で訓練)は入力文が30単語を超えると BLEU が急落しました。一方、注意つきモデル(RNNsearch-50)は50単語を超える長文でも BLEU がほとんど落ちませんでした。これは固定長ボトルネックが実際に解消されたことの直接的な証拠です。文長別の BLEU 曲線で、注意なしは右肩下がりに崩れ、注意ありはほぼ平坦という対比が論文の代表図になっています。
- BLEU スコアでは、報告によると RNNsearch-50 が全文(未知語を含む全体)でおよそ 26.75、既知語のみの文に絞るとおよそ 36.15 程度で、対応する注意なしモデル(RNNencdec-30 がおよそ 13.9 / 17.8)を明確に上回りました。当時の句に基づく統計的機械翻訳(Moses、BLEU およそ 33〜36)に純粋なニューラルモデルで初めて肉薄した点も大きな意義でした。
- **アライメント可視化のアブレーション的意義**: α_{ij} を行列として画像化すると、英語と仏語のように語順が一部入れ替わる言語対でも、対応する単語どうしが正しく結びついていることが見て取れました。たとえば「European Economic Area」が仏語で「zone économique européenne」と語順が逆転する箇所で、注意が「斜め」にずれて非単調な対応を取る様子が観察されています。これは「注意が単なる性能向上のトリックではなく、言語的に意味のある対応を学んでいる」ことの定性的証拠です。

Luong et al. (2015) は英独翻訳(WMT'14/'15)で、注意の入れ方を工夫することでさらに BLEU を伸ばしました。local attention と input-feeding を組み合わせた構成で、未知語処理を加えると WMT'15 英独でおよそ 25.9 BLEU と当時の最高水準を更新しました。指標(BLEU)の意味は前述の通りで、機械翻訳では 1〜2 ポイントの改善でも翻訳品質の体感差として無視できない、というのが当時の共通認識でした。

## メリット・トレードオフ・限界

- メリット: 固定長ボトルネックを解消し、長い入力でも先頭の情報が消えにくい。文長に対する頑健性が定量的に示された。
- メリット: 出力ステップごとに参照先を変えられるので、対応関係(アライメント)が学習でき、可視化で解釈もできる。注意の重みはモデルの「根拠」を覗く窓になる。
- メリット: soft で微分可能なので end-to-end 学習できる。後続の Transformer の基礎となり、系列処理全体の主流を作った。
- トレードオフ: 出力1ステップにつき入力全長のスコア計算が要り、計算量は系列長に比例して増える(O(出力長 × 入力長))。
- トレードオフ: RNN(再帰)ベースなので時刻を順番に処理するしかなく、並列化しづらい。GPU の並列計算能力を活かしきれない。この弱点を解消したのが Transformer。
- トレードオフ: 加算型はスコアに小さなニューラルネット(W_a, U_a, v)を持つため、内積型よりパラメータと計算がやや重い。ただし次元 d が大きいとき、内積型はスケーリング(後述の 1/√d)を入れないと softmax が飽和する問題があり、加算型は次元に対して比較的頑健という指摘もあります。実用では速度を理由に Luong の内積型に置き換わっていきました。
- 限界・未解決の論点: 「注意の重みはモデルの説明になるのか」という解釈性の議論があります。Jain & Wallace (2019, "Attention is not Explanation", NAACL) は、注意重みを別の分布に差し替えても予測がほぼ変わらない場合があることを示し、注意の可視化を過信すべきでないと指摘しました。これに対する反論(Wiegreffe & Pinter 2019, "Attention is not not Explanation", EMNLP)もあり、注意が説明になるかどうかはタスク依存で、現在も研究上の論点として残っています。

## 発展トピック・研究の最前線

加算型から内積型への移行、そして RNN を捨てた self-attention への発展が大きな流れです。内積型は行列積1回で全ペアのスコアを出せるため GPU と相性がよく、これが Transformer 隆盛の技術的基盤になりました。一方で、注意の本質である「query が key を見て value を加重和する」という枠組みは、言語・画像・点群・グラフなど分野を越えて再利用されています。自動運転の知覚でも、複数カメラの特徴を BEV(鳥瞰図)空間へ集約する際にクロスアテンションが使われるなど、Bahdanau の「クエリが必要な箇所を加重和で集める」発想は形を変えて生き続けています。最近は注意の計算量を抑える線形注意やスパース注意、注意を使わない状態空間モデル(Mamba など)との比較も活発で、「注意は本当に万能か」という問い直しも進んでいます。
