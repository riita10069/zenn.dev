# 自己教師あり事前学習

## ひとことで言うと

自己教師あり事前学習(self-supervised pretraining)とは、人手のラベル(正解の付箋)が付いていない大量のデータを使い、データ自身から自動で作れる「補助問題」を解かせて、良い初期表現(画像などをうまく数値ベクトルに変換する特徴抽出器の重み)をまず手に入れる学習戦略です。そのあとで、少量のラベル付きデータを使って本来やりたいタスク(分類・検出・運転制御など)に微調整します。「正解を人が用意するのではなく、データから自動で問題と正解を作る」ところが鍵です。自然言語処理では BERT や GPT がこの戦略で成功し、視覚分野でもここ数年で教師あり学習に匹敵あるいは凌駕する表現が得られるようになりました。

## 直感的な理解

子どもが言葉を覚える過程を思い浮かべてください。誰かが一つ一つの物に正解ラベルを付けて教えなくても、子どもは膨大な見聞きから「この形のものはよく転がる」「この音の後には大抵この出来事が続く」といった規則性を自分で見つけ、世界の構造を理解していきます。教師(ラベル)なしで、観察そのものから学んでいるわけです。

機械学習でこれを真似るのが自己教師あり学習です。なぜ必要か。深層学習は普通、入力と正解ラベルのペアを大量に与えて学びます。ところがラベル付けは非常に高コストです。たとえば自動運転で 1 シーンの 3D バウンディングボックス(車や人を 3 次元の箱で囲むアノテーション)を人手で付けると、数十分かかることもあります。一方で、ラベルの付いていないデータ(走行動画、Web 上の画像、テキスト)は、ほぼ無尽蔵に貯まります。この「無料で大量にある」データから、ラベルなしで作れる問題を解かせて良い表現を先に作ってしまえば、本タスクは少ないラベルでも高精度に届く、というのが発想です。

たとえば「画像の一部を隠して、隠した部分を当てさせる」問題なら、正解(隠した部分)はもとの画像そのものなので、人手のラベルは一切要りません。これが「自己」教師ありの意味です。データ自身が教師を兼ねている、というわけです。

## 基礎: 前提となる概念

まず表現(representation)とは、生のデータ(画素やテキスト)を、機械が扱いやすい数値ベクトル(特徴量)に変換したものです。良い表現とは、意味的に似たもの同士が近く、違うもの同士が遠くに配置されるようなベクトル空間のことです。事前学習の目標は、この良い表現を作る特徴抽出器(エンコーダ)を得ることです。良い表現が手に入れば、その上に小さな層を足すだけで多様なタスクが解ける、という汎用性が狙いです。

次に転移の二つの評価方法を押さえます。事前学習で得たエンコーダを本タスクに渡すとき、全層を本タスクのラベルで再学習するのが fine-tune(微調整)、エンコーダを凍結して上に線形分類層だけ載せて学習・評価するのが linear probe(線形プローブ)です。linear probe は「エンコーダ単体がどれだけ良い表現を作れているか」を、後段の学習力に頼らず測るための厳しい指標で、自己教師あり研究の標準的な評価法です。fine-tune は実用に近いが後段の学習で表現の粗さが補われてしまうため、表現そのものの質を見るには linear probe や k-NN(学習なしで近傍探索だけで分類)が好まれます。

最後に崩壊(collapse)という現象。自己教師あり学習で「全部の入力を同じベクトルに変換してしまう」とどんな問題でも見かけ上は解けてしまい、しかし表現としては無意味になります。たとえば「同じ画像の二つの加工を近づける」だけを目的にすると、すべてを一点に潰せば損失はゼロになりますが、何も区別できない使えない表現になります。これを表現の崩壊と呼び、これを防ぐことが多くの手法の中心的な課題です。

## 仕組みを詳しく

自己教師あり手法は大きく四系統に整理できます。違いは「何を補助問題にするか」と「どうやって崩壊を防ぐか」です。

### 対比学習(contrastive)系

同じ画像に異なる加工(切り抜き・色変え・ぼかしなど。これを augmentation、データ拡張と呼びます)を 2 回かけた 2 枚は「同じもの(正例)」、別画像は「違うもの(負例)」とみなし、正例どうしの特徴を近づけ、負例とは遠ざけます。近さは特徴ベクトルの内積(類似度)で測り、正例が近く負例が遠くなるよう InfoNCE という損失を計算します。負例を大量に押しのけることで「全部同じベクトル」への崩壊を防ぎます。崩壊回避の仕組みが「負例で押し広げる斥力」にあるのが特徴です。

ASCII で対比学習の構図を示すと次のようになります。

```
画像A ─加工1→ encoder → z1 ┐ 近づける(正例)
画像A ─加工2→ encoder → z2 ┘
画像B ─加工 → encoder → zB   遠ざける(z1,z2 から見た負例)
画像C ─加工 → encoder → zC   遠ざける(負例)
```

### 自己蒸留(self-distillation)系

student(生徒)と teacher(教師)という二つのネットを用意し、同じ画像の別々の加工を両者に見せ、student の出力が teacher の出力に近づくよう学ばせます。teacher は student の重みを指数移動平均(EMA、ゆっくり追従する平均)で更新するだけで、勾配では学習しません。負例を使わずに崩壊を防げるのが特徴で、設計はやや繊細ですが強い表現が育ちます。崩壊回避は「student と teacher の非対称性(predictor、stop-gradient、出力の中心化など)」によって担保されます。

### マスク再構成(生成的)系

画像をパッチ(小さな正方形のタイル)に分け、その大部分を隠して、残りから隠した部分を復元させます。「見える一部から全体を推測する」難しい問題を解く過程で、強い表現が育ちます。テキストの「穴埋め問題」(BERT のマスク言語モデル)の画像版にあたります。この系統は正例も負例も EMA も不要で、純粋に「隠して当てる」だけなので設計が単純なのが利点です。

### 潜在空間予測系

隠した部分を「ピクセルそのもの」ではなく「特徴空間のベクトル」で予測します。見た目の細部ではなく意味的な内容を予測対象にすることで、背景のテクスチャまで律儀に当てる無駄を避け、意味的に重要な情報に計算を集中できます。予測対象を作る側のエンコーダ(target encoder)を勾配で動かさず EMA で更新することで崩壊を防ぎます。

四系統の崩壊回避策を対比すると、対比系は「負例の斥力」、自己蒸留系は「非対称設計と stop-gradient」、生成系は「復元という難問そのもの」、潜在予測系は「target を勾配から切り離す EMA」で防いでおり、補助問題の立て方と崩壊回避策はセットで設計されている、と理解すると見通しが良くなります。

## 手法の系譜と主要論文

四系統を時系列と系譜で追うと、研究の流れが見えてきます。

対比系では、SimCLR(Chen ら、ICML 2020、arXiv:2002.05709)が枠組みをシンプルに整理し、強いデータ拡張と射影ヘッド(対比損失を計算する前の小さな全結合層)、大きなバッチが効くことを示しました。動機は「複雑な仕掛けなしでも対比学習だけで教師あり並みの表現が作れるか」を確かめることで、答えはイエスでした。ただし十分な数の負例を確保するため極めて大きなバッチ(例として 4096 枚同時)が必要というトレードオフがあります。これを解決したのが MoCo(He ら、CVPR 2020、arXiv:1911.05722)で、momentum encoder(EMA でゆっくり更新するもう一つのエンコーダ)と queue(過去のバッチの特徴をためる待ち行列)を使い、小さいバッチでも多数の負例を確保できるようにしました。MoCo の EMA エンコーダという発想は、後の自己蒸留系・潜在予測系にも受け継がれる重要な道具になりました。

自己蒸留系では、BYOL(Grill ら、NeurIPS 2020、arXiv:2006.07733)が「負例を一切使わなくても、predictor という非対称な仕掛けと teacher の移動平均で崩壊を防げる」と示しました。それまで「負例がないと崩壊する」と信じられていたため、これは画期的でした。SimSiam(Chen & He、CVPR 2021、arXiv:2011.10566)はさらに踏み込み、EMA teacher すらなくても stop-gradient(片側の勾配を止める操作)と predictor だけで崩壊を防げることを示し、崩壊回避の本質が stop-gradient による非対称性にあると整理しました。続く DINO(Caron ら、ICCV 2021、arXiv:2104.14294)は、student の出力を teacher の出力に近づける自己蒸留を Vision Transformer に適用し、ラベルなしなのに意味的に良い特徴が育つこと、そして注意マップ(モデルがどこを見ているか)が物体の輪郭を自然に捉えることを示しました。これは「自己教師ありがセグメンテーション的な構造を勝手に獲得する」という驚きの結果でした。

生成系では、MAE(He ら、CVPR 2022、arXiv:2111.06377)が大きな転機です。画像パッチの約 75% を隠し、見える 25% だけをエンコーダに通して隠した部分のピクセルを軽量なデコーダで復元します。隠す割合が大きいので難しい問題になり強い表現が育ち、しかも見えるパッチだけ処理するのでエンコーダの計算量が約 4 分の 1 に減り効率も高い、という二重の利点があります。BEiT、SimMIM、data2vec も近い系統で、テキストの BERT の成功を画像へ移植する流れの中にあります。

潜在予測系では、I-JEPA(Assran ら、CVPR 2023、arXiv:2301.08243)が、隠した領域を特徴空間で予測します。これは LeCun が提唱する JEPA(Joint Embedding Predictive Architecture)の思想に基づき、ピクセル復元の無駄を避けて意味に集中する設計です。target encoder(予測の正解を作るエンコーダ)を勾配で動かさず EMA で更新することで崩壊を防ぎます。

四系統を一言で対比すると、対比系は「負例で押しのける」、自己蒸留系は「teacher を真似る」、生成系は「隠して復元する」、潜在予測系は「特徴で予測する」となり、共通の狙いは「ラベルなしで良い表現を作る」こと、違うのは補助問題の立て方だけ、と捉えると見通しが良くなります。

## 論文の実験結果(定量データ)

評価は主に ImageNet(約 128 万枚、1000 クラス)での linear probe 精度(top-1、エンコーダ凍結+線形層、高いほど良い)と fine-tune 精度で行われます。これは「ラベルなしで学んだ表現が、教師あり学習で得た表現にどれだけ迫れるか」を測る指標です。

SimCLR(2020)は ImageNet linear probe で top-1 約 69.3%(ResNet-50)を達成し、それまでの自己教師あり手法を大きく上回り、教師あり ResNet-50(約 76%)に近づきました。バッチサイズと学習エポックを増やすほど精度が上がるという、対比学習のスケール性も示しました。アブレーションでは、色変えなどの強いデータ拡張を外すと精度が大きく落ち、データ拡張の設計が対比学習の生命線であることを定量的に示しました。MoCo v2 は同等の精度を、SimCLR が必要とした巨大バッチなしで達成し、計算資源の少ない研究者にも対比学習を開きました。

BYOL(2020)は負例なしで ImageNet linear probe top-1 約 74.3%(ResNet-50)に到達し、対比系を上回りました。アブレーションとして、predictor を外すと表現が崩壊して精度が大きく落ちることを示し、この非対称な仕掛けが崩壊回避に必須であることを定量的に証明しました。DINO(2021)は Vision Transformer(ViT)で linear probe top-1 約 80% に達し、さらに k-NN 分類(近傍探索のみ、学習なし)でも約 78% という高さを示しました。linear probe と k-NN の差が小さいことは、表現が「線形に切れるほど綺麗」に整っている証拠です。注意マップが教師なしで物体マスクを捉える定性的結果も、自己蒸留の質の高さを裏づけました。

MAE(2022)は fine-tune での強さが際立ち、ViT-Large を ImageNet で fine-tune して top-1 約 85.9%、ViT-Huge で約 87.8% を達成し、同条件の教師あり事前学習を上回りました。重要なアブレーションがマスク率で、隠す割合を 75% 付近にしたときに最良で、25% など低すぎると問題が易しくなりすぎて表現が弱くなることを示しました。これは「難しい補助問題ほど良い表現が育つ」という直感を定量的に裏づける結果です。一方で MAE は linear probe では BYOL/DINO に劣る傾向があり、「fine-tune で強いが線形分離性は低い」という生成系の特徴が現れています。I-JEPA(2023)は、ピクセル復元系より少ない事前学習計算で同等以上の linear probe 精度を達成したと報告し、特徴空間予測の計算効率の良さを示しました。とくに ViT-Huge で、MAE などピクセル再構成系より少ない GPU 時間で競争力ある精度に届いた点が新規性です。

これらの数値の意味を補足すると、linear probe top-1 が 69% から 80% へ上がることは、凍結したエンコーダ単体の表現品質が大きく向上したことを意味し、下流タスク(検出・セグメンテーション・少数ショット学習)での転移性能の向上に直結します。とくに「ラベルが 1% や 10% しかない」少データ設定での精度差は大きく、自己教師あり事前学習の本領はラベルが希少な現実の応用で発揮されます。数年で十数ポイントの改善が積み上がったことが、自己教師あり研究の急速な進歩を示しています。

## メリット・トレードオフ・限界

メリットは、高コストなラベルを大量に用意しなくても、ラベルなしデータから強い初期表現を得られることです。少量ラベルでの本タスク精度が上がり、自動運転のようにラベル付けが特に重い領域で有効です。また、得られる表現が汎用的で、複数の下流タスクに使い回せる利点もあります。

トレードオフと限界は系統ごとに異なります。対比系は負例の確保が難しく、大バッチや queue の工夫が要り、データ拡張の設計に精度が強く依存します。自己蒸留系は崩壊を防ぐ非対称設計(predictor、EMA teacher、出力の中心化など)が繊細で、ハイパーパラメータに敏感です。生成系はピクセル復元に計算を浪費しがちで、必ずしも意味理解に直結しない細部まで当てようとし、linear probe が伸びにくい傾向があります。潜在予測系は予測対象が特徴空間なので崩壊回避の設計が必須です。共通の限界として、事前学習自体に大きな計算資源とデータが必要で、補助問題で得た特徴が本タスクに最適とは限らない(タスクのズレ)という根本的な不確実性があります。たとえば分類向けに良い表現が、細かい位置を要する検出に最適とは限りません。

未解決の研究課題としては、第一に「どの補助問題が、どの下流タスクに最も良い表現を与えるか」の理論的予測がまだ立てられないこと。第二に、動画や複数センサー(カメラ+LiDAR など)のように時間・空間・モダリティをまたぐデータでの自己教師ありの最適設計が定まっていないこと。第三に、崩壊の理論的理解が手法ごとに断片的で、統一的な説明が確立していないことが挙げられます。

## 発展トピック・研究の最前線

最前線は大きく三方向です。一つは大規模化とスケール則の探求で、DINOv2(Oquab ら、2023、arXiv:2304.07193)は厳選した大規模データ(1 億枚規模をキュレーション)と自己蒸留+マスクの組み合わせで、ファインチューニングなしでも分類・深度推定・セグメンテーションなど多様な下流タスクに強い汎用視覚特徴を作り、自己教師ありが「基盤モデル(さまざまなタスクに転用できる土台モデル)」の段階に入ったことを示しました。凍結特徴のまま多タスクで競争力を持つことが、表現の汎用性の到達点を示しています。

二つ目は動画・世界モデルへの展開です。静止画ではなく時間軸を含む動画で「未来のフレームや状態を特徴空間で予測する」JEPA 系の発展(V-JEPA など)が進んでおり、これは「次に何が起きるか」を予測する力を表現に埋め込む試みです。自動運転のように時系列の予測が本質的なタスクとの親和性が高く、ピクセルではなく特徴で未来を予測する設計は、世界モデル研究と地続きです。動画は時間的な一貫性という強力な「無料の教師信号」を持つため、自己教師あり学習にとって本質的に相性が良い領域です。

三つ目はマルチモーダルへの拡張で、画像とテキスト、あるいはカメラと LiDAR といった異なるセンサー間で「同じ場面の異なる見え方」を正例とみなす対比学習(CLIP の系統)が広がっています。ラベルの代わりに「自然に対応づくモダリティのペア」を教師信号にすることで、人手アノテーションなしに意味の通った表現を学べる点が、自己教師ありの考え方の自然な延長になっています。自動運転では、複数センサーの時刻同期した観測が大量に得られるため、このマルチモーダル自己教師ありは特に有望視されています。
