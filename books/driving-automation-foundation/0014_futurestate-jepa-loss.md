# FutureState + JEPA損失

## ひとことで言うと

「自動運転モデルに、これから数秒後に世界がどう見えるかを予想させて、その予想が当たるように学習させる」仕組みです。ただし生のカメラ画像のピクセルを当てさせるのではなく、画像を抽象的に要約した「特徴(feature)」を当てさせます。これが JEPA(ジェパ, Joint-Embedding Predictive Architecture)という考え方で、未来を予想する部分(未来特徴予測器)と、その予想を当てさせる学習信号(JEPA 損失=特徴再構成損失)をセットで使います。

## 直感的な理解

人間の運転がうまいのは、「次にこうなるだろう」という予測を頭の中で常に立てているからです。前の車がブレーキランプを点けたら、車間が詰まる前に減速する。横断歩道に人が近づいたら、止まる準備をする。この「世界の動きを予測する内部モデル」を機械に持たせる研究分野を World Model(世界モデル)と呼びます。

世界モデルを持たせたい動機は2つあります。第一に、未来を予測できるモデルは、単に「ベテランの運転を真似する(模倣学習, imitation learning)」だけのモデルより、状況の因果や物理を内部で理解しやすくなります。模倣だけのモデルは「なぜそうするか」を持たないため、見たことのない状況でもろくなります。第二に、強化学習(RL, 試行錯誤で報酬を最大化する学習)や計画の土台になります。「ある行動をとったら世界がどうなるか」を頭の中でシミュレーションできると、実車で危険な試行錯誤をせずに済みます。

ここで素朴な方法は「未来のカメラ画像そのものを予測させる」ことです(ピクセル予測, pixel prediction)。しかしこれには大きな無駄があります。未来の画像には、運転に無関係な細部(木の葉の揺れ、影のちらつき、路面の模様、センサノイズ)が大量に含まれます。ピクセルを正確に当てようとすると、モデルはこうした「予測不能でどうでもいい細部」を再現する作業に労力を吸い取られ、肝心の「車や人や道がどう動くか」という抽象的なダイナミクスの学習がおろそかになります。天気予報で雲一つひとつの形まで完璧に描こうとして、肝心の「雨が降るかどうか」を外すようなものです。JEPA は「描かずに、当てるべき本質だけを当てる」という割り切りでこの無駄を断ち切ります。

## 基礎: 前提となる概念

「自己教師あり学習(self-supervised learning, SSL)」とは、人手のラベル(正解札)なしに、データ自身から学習信号を作る方法です。たとえば「画像の一部を隠して、隠れた部分を残りから当てさせる」「動画の現在から未来を当てさせる」といった具合に、データの構造そのものを正解にします。未来予測はまさに SSL であり、未来フレームさえあればラベル付けの人手が要りません。これが自動運転で重要なのは、走行ログは膨大に集まる一方、人手アノテーション(物体の枠付けなど)は高価で量が限られるからです。

「特徴空間(feature space)/埋め込み(embedding)」とは、ニューラルネットが画像を圧縮して得る抽象的なベクトルのことです。たとえば 256×256×3 のピクセル(約20万次元)を、256次元のベクトルに圧縮します。この圧縮の過程で、ノイズや無関係な細部は捨てられ、「何が写っているか、どう動いているか」という本質が残るように学習されます。

「JEPA(Joint-Embedding Predictive Architecture)」は、Yann LeCun(2022)が提唱した枠組みで、「ピクセル空間ではなく特徴空間で予測する」のが核心です。入力 x と、それと関係する別の入力 y(未来フレームやマスクされた領域)をそれぞれエンコーダで特徴に圧縮し、x の特徴から y の特徴を予測します。生成型(ピクセルを復元する)と違い、抽象空間では予測不能な細部がすでに落ちているので、モデルは本質的なダイナミクスだけに集中できます。生成モデル(VAE, GAN, 拡散モデル)が「ピクセルを作る」のに対し、JEPA は「特徴を当てる」だけで、表現学習に特化しています。重要なのは、抽象空間での予測には「すべてを定数に潰せば常に当たる」という抜け道(表現崩壊)があり、それを防ぐ仕掛けが別途必要なことです(詳細は [凍結/EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0005_ema-target-encoder))。

## 仕組みを詳しく

典型的な実装は2つの部品から成ります。未来を予測する側(未来特徴予測器)と、予測を当てさせる損失(特徴再構成損失=JEPA 損失)です。

### 未来特徴の予測器

入力は次の2つが代表的です。

1. 現在の融合特徴 形状 `[B, C, H, W]` — 現在の BEV(鳥瞰図)特徴など。`C` は埋め込み次元(256前後)、`H, W` は空間グリッド。
2. 自車意図 形状 `[B, D]` — プランナの再帰隠れ状態など、「これからどう運転するつもりか」の要約。

処理の流れの一例は次の通りです。

1. 自車意図ベクトルを全結合層で `C` 次元に投影し、`[B, C, 1, 1]` に整形して現在特徴に空間方向ブロードキャストで足す。これで「自車の運転意図」が現在特徴全体にバイアスとして注入される。
2. オプションの行動条件付け(action conditioning): 計画した軌跡も投影してもう一段バイアスとして足す。これにより「軌跡 A を走ったら世界はどう見えるか? B なら?」という反実仮想(counterfactual)ロールアウトが可能になる。これが世界モデルを「受動的な未来予測」から「行動の結果を試せる予測」へと格上げする鍵です。
3. 数層の畳み込み(例: `C → 2C → 4C` チャンネルに拡張し GELU を挟む)で未来特徴マップを生成し、チャンネル方向に分割して K 個の `[B, C, H, W]` を得る。これが +1.6s / +3.2s / ... のように複数の未来時刻の予測になる。

実装上の注意として、高解像度(例: 450×300)で `K×C` チャンネルの未来特徴を fp32 で持つと、1サンプルあたり数百 MB に達します。bf16 混合精度(16ビット浮動小数で計算してメモリと速度を稼ぐ)や空間ダウンサンプリングがほぼ必須です。

### 特徴再構成損失(JEPA 損失)

1. 予測器が出した K 個の `[B, C, H, W]` が予測特徴。
2. 正解(ターゲット特徴)は、ターゲットエンコーダ(止まった、または遅れた重みのエンコーダ。詳細は [凍結/EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0005_ema-target-encoder))を未来フレームに適用して作る。勾配は流さない(stop-gradient = `detach`)。
3. 各タイムステップで MSE(平均二乗誤差) `mean((pred - target)^2)` を計算し、`[K]` にスタック。L1(平均絶対誤差)を使う変種もある。
4. タイムステップごとの重み `temporal_weights` を掛けて総和でスケールを揃え、平均してスカラー損失にする。遠い未来ほど不確実なので重みを下げる設計が一般的(例: +1.6s に 1.0、+6.4s に 0.4 など)。

損失が「特徴空間の MSE」である点が、ピクセル予測との決定的な違いです。ピクセルを当てるのではなく、抽象表現の未来値を当てます。そして正解側に必ず stop-gradient をかける理由は、予測側とターゲット側が同時に動くと「両者で同じ定数に逃げる」共謀が起きて崩壊するからです。

### 数値例と形状の流れ

```
現在特徴 [B, 256, 56, 56]  自車意図 [B, 256]
   │  自車意図を [B,256,1,1] にして加算
   ▼
畳み込み(256→512→1024)+ GELU
   │  chunk(4) でチャンネルを4分割
   ▼
predicted: 4 × [B, 256, 56, 56]   (+1.6s,+3.2s,+4.8s,+6.4s)
   │
target: 4 × [B, 256, 56, 56]   (ターゲットエンコーダ@未来フレーム, detach)
   │  各ステップ MSE → [4] → temporal_weights → mean
   ▼
JEPA loss (スカラー)
```

この JEPA 損失は単独で使うより、模倣損失(ベテラン軌跡との誤差)に小さな重みで足す「補助損失(auxiliary loss)」として使うのが定石です。こうすればデフォルトの運転挙動を壊さずに、表現に「未来を予測する力」を上乗せできます。

## 手法の系譜と主要論文

- LeCun(2022)"A Path Towards Autonomous Machine Intelligence"(OpenReview 公開のポジションペーパー)— JEPA の原典的提言。提案は「生成(ピクセル復元)ではなく、抽象表現空間で予測する自己教師あり学習を、自律知能の中核に据える」。理由は「予測不能な細部を生成する無駄を避け、表現の本質に集中させる」。トレードオフは「表現崩壊(representation collapse, すべてを同じベクトルに潰す退化)を防ぐ工夫が別途必要」なこと。崩壊防止は [凍結/EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0005_ema-target-encoder) で扱う。

- Assran et al.(2023)I-JEPA "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture"(arXiv:2301.08243, CVPR)— 画像版 JEPA。1枚の画像内で、見えている領域(context)から、マスクした領域(target)の特徴を予測。提案は「特徴空間でのマスク予測」。新規性は「対照学習(contrastive learning)の負例集めや、強いデータ拡張に頼らず、良い表現を獲得」。トレードオフは「ターゲットエンコーダの設計とマスク戦略に性能が敏感」。

- Bardes et al.(2024)V-JEPA "Revisiting Feature Prediction for Learning Visual Representations from Video"(arXiv:2404.08471)、続く V-JEPA 2(Meta AI, 2025)— 動画版 JEPA。時間方向の未来/マスク特徴を予測する。「未来の特徴を予測する」という発想に最も近い系譜で、動きと相互作用の理解に効くと示された。V-JEPA 2 はさらにロボット行動の計画(学習した潜在空間で目標到達のための行動列を探索する)へ拡張され、JEPA が単なる表現学習を越えて「行動の世界モデル」になりうることを示しました。

- 自動運転の World Model の系譜には、ピクセル/トークン生成型が並行して存在します。GAIA-1(Hu et al., 2023, arXiv:2309.17080, Wayve)は運転動画を自己回帰生成する大規模世界モデル(画像をトークン列に変換し、言語モデルのように次トークンを予測)、DriveDreamer(Wang et al., 2023, arXiv:2309.09777)は拡散モデルで運転シーンを生成します。これらは「見える未来を作る」のが得意で、可視化やデータ拡張(レアシーンの合成)に強い反面、計算が重く学習が不安定になりがちです。対して JEPA 型(特徴空間予測)は、生成の負担を負わない分、計算と学習安定性で有利という対比があります。両者は排他ではなく、生成型で可視化・データ拡張、JEPA 型で頑健な表現学習、という役割分担も議論されています。

系譜は「LeCun の JEPA 提言 → 画像版 I-JEPA → 動画版 V-JEPA → V-JEPA 2 で計画へ拡張 → 運転ドメインへの適用(未来 BEV 特徴の予測)」と整理できます。

## 論文の実験結果(定量データ)

JEPA 系の評価は、学習した表現を凍結して下流タスク(分類・行動認識など)で測る「linear probe / frozen evaluation(線形評価)」が中心です。これは「表現がどれだけ役に立つ情報を含むか」を測ります。バックボーン(特徴抽出器)の重みは固定し、その上に線形分類器1層だけを学習して精度を見る方式で、表現そのものの良し悪しを切り分けられます。

- I-JEPA(2023)は ImageNet-1k の linear probe で、ViT-H/14 を用いておよそ 79% 前後のトップ1精度を報告しました。重要なのは効率で、強いデータ拡張に依存する対照学習系(SimCLR, BYOL など)と同等以上の精度を、より少ない事前学習エポックと拡張なしで達成した点です。アブレーションでは、(1) ターゲットを EMA エンコーダで作らず予測側と共有すると表現崩壊して精度が激減する、(2) マスク領域を大きく連続的に取る(small context, large target)ほど良い表現になる、ことが示されました。前者は [凍結/EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0005_ema-target-encoder) の必要性を裏づける結果です。

- V-JEPA / V-JEPA 2(2024-2025)は、Kinetics-400(動画行動認識)や Something-Something v2(物体操作の動きを問うデータセット)の frozen evaluation で、ピクセル生成型(masked autoencoder の動画版など)を上回る精度を報告しました。とくに「動きそのものを問う」Something-Something v2(同じ物体でも動かし方の違いをクラスにするため、外見だけでは解けない)で差が大きく、特徴空間予測が時間ダイナミクスの理解に効くことを示します。これは外見ではなく「動き」を当てる必要があり、世界モデルが目指す方向と直結します。

- ピクセル予測 vs 特徴予測の直接比較(各種 ablation 論文)では、同じバックボーンとデータで、特徴空間予測の方が下流精度で数ポイント上回りつつ、学習が安定する(損失曲線が滑らかで崩壊しにくい)ことが繰り返し観察されています。一方、特徴予測は「再構成された画像を人が見て確認する」ことができないため、デバッグや可視化がしにくいという実務上の難点があります。損失が下がっていても表現が崩壊している場合があり、損失値だけ見ても健全性が判定できません。

運転ドメイン固有の定量評価(未来 BEV 特徴予測が下流の計画指標 L2 や衝突率をどれだけ改善するか)は、データセットと評価プロトコルの標準化が進行中で、まだ確立した代表値は乏しい段階です。報告される効果は「同じ模倣損失に JEPA 補助損失を足すと、ロングテール(稀な)シーンでの頑健性が上がる」という定性的傾向が中心で、数値はおよそであり報告による差が大きい点に注意が必要です。

## メリット・トレードオフ・限界

メリット
- ピクセルではなく特徴を予測するので、運転に無関係な細部に学習資源を奪われず、本質的なダイナミクスに集中できる。
- 自己教師あり(未来フレームさえあれば人手ラベル不要)で大量の学習信号を作れる。走行ログがそのまま教師になる。
- 行動条件付けにより反実仮想ロールアウトができ、将来の RL や計画評価(候補軌跡の良し悪し判定)の土台になる。
- 補助損失として模倣学習に足す形にすれば、デフォルトの挙動を壊さず追加できる。

トレードオフ・限界
- 正解特徴を作るために、ターゲットエンコーダを未来フレームに適用するデータ前処理が要り、計算とストレージのコストがかかる。
- 特徴空間予測は表現崩壊のリスクがあり、ターゲットエンコーダ設計(凍結や EMA、stop-gradient)が別途必須([凍結/EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0005_ema-target-encoder))。
- 高解像度では未来特徴テンソルが巨大でメモリを食う(bf16 や空間縮小がほぼ必須)。
- 予測結果が抽象ベクトルなので、人が直接見て妥当性を確認できず、デバッグが難しい。
- 「特徴予測の上達」が「実際の運転性能向上」に直結する保証はなく、補助損失の重み調整が必要。重すぎると未来予測が主タスクを引っ張り、軽すぎると効果が消える。

未解決課題としては、(1) どの特徴空間(occupancy のような幾何か、意味的特徴か、生バックボーン特徴か)を予測対象にすべきかの設計、(2) 予測した未来特徴を実際の計画にどう接続するか(評価器として使うか、補助損失だけにとどめるか)、(3) 長い時間 horizon での予測の発散(誤差が雪だるま式に増えること)をどう抑えるか、が挙げられます。

## 発展トピック・研究の最前線

JEPA を「予測 + 計画」に発展させる方向では、V-JEPA 2 のように学習した世界モデルの潜在空間でロボットの行動を計画する研究が進んでいます。運転では、未来特徴予測器の上で複数の候補軌跡を反実仮想ロールアウトし、最も安全・快適な軌跡を選ぶ「model-predictive な計画」が有望です。これは [Bézier軌跡ヘッド](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0115_bezier-trajectory-head) のような速い反射層と組み合わせ、reactive(反射)と deliberative(熟考)の二層構成を作る流れにつながります。

また、予測対象を occupancy(各セルが占有されているか)や意味セグメンテーション特徴に限定して、解釈性と幾何的整合性を高める「構造化 JEPA」も研究されています。さらに、特徴空間ではなく潜在状態の確率分布を予測して不確実性を明示的に扱う方向、複数の未来仮説を保持する multi-hypothesis 世界モデルなど、「未来は一つに定まらない」という運転固有の不確実性に対応する研究が進んでいます。
