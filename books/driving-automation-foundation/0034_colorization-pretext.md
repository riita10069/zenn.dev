# colorization (彩色) プレテキスト

## ひとことで言うと

彩色 (colorization) プレテキストは、ラベルのない大量の画像だけから「役に立つ特徴」を学ぶための自己教師あり学習 (self-supervised learning) の代理タスク (pretext task) です。カラー写真から色を抜いて白黒 (グレースケール) にし、ニューラルネットワークに「もとは何色だったか」を当てさせます。空を青く、草を緑に正しく塗るには、その領域が何の物体なのかを理解していなければなりません。つまり「色当て」という建前を解く過程で、物体認識に転用できる汎用的な特徴が副産物として育ちます。色の正解はもとのカラー画像を消すだけで自動的に手に入るので、人手のラベル付けは一切不要です。

## 直感的な理解

私たちが白黒写真を見ても、たいていは違和感なく色を想像できます。バナナは黄色、消防車は赤、晴れた空は青。これができるのは、写真に写るものが「何であるか」を知っていて、その物体に典型的な色を経験的に学んでいるからです。逆に言えば、白黒画像をもっともらしく塗れるシステムは、暗黙のうちに「これはバナナだ」「これは空だ」という認識を内部に持っているはずです。

ここに自己教師あり学習の発想の核があります。「色を塗れ」という一見ただの画像変換タスクを真面目に解かせると、解くために必要な物体理解がネットワークの内部に蓄積されます。私たちが本当に欲しいのは塗られた絵そのものではなく、塗るために学習された内部の特徴抽出器のほうです。色当てはあくまで口実 (pretext) であり、それを通じて「ラベルなしで物体認識の下地を作る」ことが真の狙いです。

なぜわざわざラベルなしにこだわるのか。画像認識では「この画像は猫」「ここに歩行者がいる」といった人手のラベルが学習に必要ですが、ラベル付けは高コストで、世の中に存在する膨大な画像のごく一部にしか付いていません。一方、色付き画像そのものは無尽蔵にあります。たとえばインターネット上の写真や過去のアーカイブはほぼすべてカラーで、これを白黒化して色を当てさせる仕組みなら、ラベル付き画像が足りない領域でも、画像さえあればいくらでも事前学習 (pre-training) ができます。「無料で大量に手に入る教師信号」を作り出すこと、これが彩色プレテキストの経済的な魅力です。

## 基礎: 前提となる概念

自己教師あり学習とは、人間が付けた正解ラベルを使わず、データそのものから問題と正解を自動生成して学ぶ枠組みです。「自分で自分に問題を出して解く」ので self-supervised と呼ばれます。代理タスク (pretext task) は、その自動生成された問題のことです。最終的に解きたい本命のタスク (downstream task、下流タスク。分類・検出・セグメンテーションなど) とは別の、特徴を育てるための踏み台です。

色の表し方を理解しておく必要があります。コンピュータの画像は通常 RGB、つまり赤・緑・青の3つの強度で各画素を表します。しかし彩色タスクでは CIE Lab 色空間を使うのが標準です。Lab は色を次の3軸で表現します。

- L (Lightness): 明るさ。0 (黒) から 100 (白) で、これは白黒写真の情報そのもの。
- a: 緑 (負) から赤 (正) への度合い。
- b: 青 (負) から黄 (正) への度合い。

白黒画像はちょうど「L だけが分かっていて a と b が失われた状態」です。したがって彩色は「L から a, b を予測する」というきわめてきれいな形に定式化できます。RGB をそのまま使うと、明るさと色味が3チャンネルに混ざってしまい「明るさは既知、色味だけ未知」という構造を表現しにくいため、Lab が好まれます。Lab がもう一つ便利なのは、人間の知覚に近い (=数値上の距離が見た目の色差に近い) よう設計されている点で、後で「色の似ている/似ていない」を測るときに自然です。

転移学習 (transfer learning) も前提です。あるタスク (ここでは彩色) で学習した重みを、別のタスク (分類など) の初期値として流用する技術です。pretext task で得た backbone (画像から特徴を取り出す土台ネットワーク) を、少量のラベル付きデータで微調整 (fine-tuning) すれば、ゼロから学習するより速く高精度に到達できます。この「特徴を移植する」評価のしかたが、プレテキストタスクの良し悪しを測る共通のものさしになります。

## 仕組みを詳しく

彩色には本質的な難しさがあります。それは「正解の色は一つではない」ことです。白黒のリンゴは赤かもしれないし青リンゴかもしれない。車は赤・青・白・黒のどれでもありえます。一つの白黒入力に対して複数の妥当な色が存在する、これを多峰性 (multimodality、複数のピークを持つ分布) と呼びます。

素朴なやり方は a, b の数値を直接回帰 (regression、連続値を当てる) することです。各画素について「a はこの値、b はこの値」と予測し、二乗誤差 (各画素の予測値と正解値の差を二乗して平均したもの) を最小化します。ところがこれは多峰性のせいで破綻します。赤 (a が大) と緑 (a が小) の両方が妥当な場合、二乗誤差を最小化する答えは両者の平均、つまり「a がほぼゼロ = 無彩色のくすんだ灰色・茶色」になってしまうのです。複数の正解の平均は、どの正解とも似ていない冴えない色です。これが回帰ベースの彩色が必ずくすむ理由です。

Zhang らはこれを「回帰」ではなく「分類 (classification)」として解くことで回避しました。手順は次の通りです。

1. ab 色空間 (実際には gamut、自然画像で実際に現れる色域に限定) を 313 個の代表色のビンに離散化する。連続的な色当てを「313択のうちどれか」という分類問題に変換する。なぜ 313 かというと、ab 平面を 10 刻みの格子で区切り、自然画像に実際に現れる範囲だけ残すとおよそ 313 個のビンになるためです。
2. CNN は各画素について「313色のどれが正解か」の確率分布を出力する。形状は入力 `[B, 1, H, W]` (B はバッチ数、白黒すなわち L チャンネルのみ) から CNN を経て `[B, 313, H, W]` (各画素の313色への確率)。たとえば 256x256 入力なら出力は `[B, 313, 256, 256]` で、画素ごとに 313 次元の確率ベクトルが並ぶ。
3. これにより「この画素は赤かもしれないし緑かもしれない」という多峰性を、確率分布として素直に表現できる。平均に逃げる必要がなく、複数のピークをそのまま「赤に 0.4、緑に 0.4」のように残せる。
4. 学習時の損失は分類の交差エントロピー (予測した確率分布と正解分布の差を測る損失) だが、ここに class rebalancing (クラス再重み付け) を入れる。自然画像では地味な色 (灰色・茶色・空の青) が圧倒的に多く、鮮やかな赤や黄は希少。素のままだと損失が頻出色に支配され、ネットワークは無難な色だけ塗るようになる。そこで希少色の損失を重く、頻出色を軽く重み付けして、鮮やかな色も学ばせる。具体的には各ビンの出現頻度の逆数に近い重みを割り当てる。
5. 推論時は313色の確率分布から1色を選ぶ。単純な期待値だとまた平均化してくすむため、annealed-mean (温度パラメータ T で分布を `exp(log(p)/T)` のように尖らせてから期待値を取る) という手法で、鮮やかさともっともらしさのバランスを取る。T を小さくすると最頻色 (一点) に近づき、大きくすると平均に近づく。

before / after を整理します。

```
[before] a,b を直接回帰
  入力(白黒) ──CNN──> a,b の値 ──> 多峰性を平均化 ──> くすんだ茶/灰
                                       (赤と緑の中間 = 無彩色)

[after] 313色の分類 + 頻度リバランス + annealed-mean
  入力(白黒) ──CNN──> 313色の確率分布 ──> 鮮やかでもっともらしい色
                       (多峰性をそのまま保持)
```

学習が終わったら出力の分類層を捨て、途中の特徴抽出部 (backbone) を学習済みの初期重みとして下流タスクに転用します。なぜこの設計なのかを一言でまとめると、彩色の核心的困難は「色の曖昧さ」であり、それを正面から扱うために分類化と頻度リバランスが必要だった、ということです。

Larsson らはやや異なる設計を採りました。彩色を ab のヒストグラム (色相と彩度の分布) の予測として定式化し、画素ごとに「どの色相がどれだけ」という分布を当てさせます。Zhang の 313 ビン分類と発想は近く、いずれも「単一の色に決め打ちせず分布を予測する」点で共通します。この「分布予測」という共通解が、彩色を表現学習として機能させる鍵だったわけです。

## 手法の系譜と主要論文

彩色を表現学習に使う流れは、複数の研究が同時期に競い合いながら形成されました。

中心となるのは Zhang, Isola, Efros による "Colorful Image Colorization" (ECCV 2016, arXiv:1603.08511) です。彼らの貢献は前述の「ab 空間の313色分類化 + class rebalancing + annealed-mean」で、これによって自動着色を従来より鮮やかかつ自然にし、同時にその過程で学んだ特徴が物体認識の事前学習として有効であることを示しました。

ほぼ同時期に Larsson, Maire, Shakhnarovich の "Learning Representations for Automatic Colorization" (ECCV 2016, arXiv:1603.06668) があり、その発展が "Colorization as a Proxy Task for Visual Understanding" (CVPR 2017, arXiv:1703.04044) です。後者は彩色を「視覚理解の代理タスク」として体系的に評価し、彩色で学んだ特徴が分類・セグメンテーションへ転移することを定量的に実証しました。当時の自己教師あり手法の中で、彩色が転移性能でトップクラスであることを示した重要な仕事で、特に「アーキテクチャを揃えて公平に比較する」評価作法を整えた点に意義があります。

Zhang らはさらに "Split-Brain Autoencoders" (CVPR 2017, arXiv:1611.09842) で彩色を一般化しました。画像のチャンネルを2グループに分け、片方からもう片方を相互予測する (明度から色、色から明度の双方向)。彩色を「クロスチャンネル予測」という広い枠組みに拡張し、pretext task としての性能をさらに高めました。これは「片方向の色予測」を「双方向のチャンネル間予測」へと一般化した発展で、後年の masked modeling の「一部から残りを当てる」発想にも通じます。

この彩色の系譜は、より広い pretext task の流れの一部です。Doersch らの相対位置予測 (パッチの相対配置を当てる、2015)、Gidaris らの回転予測 (画像を回転させて角度を当てる、2018)、Noroozi らのジグソーパズル (パッチを並べ替えて元の配置を当てる、2016)、そして [欠損領域の復元 (inpainting)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0035_context-encoder-inpainting) などと並びます。いずれも「データの加工から自動で正解を作る」点で共通します。これらの幾何的・生成的 pretext task は 2018-2019 年ごろに対照学習 ([CPC](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0037_contrastive-predictive-coding)、SimCLR、MoCo など) に主役を譲り、さらに 2021-2022 年には MAE をはじめとするマスク再構成型が再興しました。彩色は、これら自己教師あり学習の歴史の初期を代表する手法です。

## 論文の実験結果 (定量データ)

Zhang らの ECCV 2016 論文は2つの軸で評価されています。

着色品質については、Amazon Mechanical Turk を使った被験者実験 (colorization Turing test) を行いました。被験者に「本物のカラー写真」と「白黒から自動着色した画像」のどちらかを見せ、本物か偽物かを判定させます。報告では、自動着色した画像のおよそ 32% で被験者が「これは本物だ」と誤判定しました。完璧なら 50% (見分けがつかない) なので、32% は当時の他手法 (素朴な回帰ベースはおよそ 1 桁台%) を大きく上回る高い水準です。これは「色の曖昧さに平均で逃げず、鮮やかにもっともらしく塗れた」ことの定量的な証拠です。

表現学習としての価値は、ImageNet と PASCAL データセットでの転移性能で測られました。彩色で事前学習した特徴を固定 (frozen) し、その上に線形分類器だけを載せて ImageNet 分類を解く線形評価 ([linear probing](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0759_linear-probing-evaluation)) では、当時の自己教師あり手法の中で上位の精度を出しました。PASCAL VOC の分類・検出・セグメンテーションへ転移した場合も、ランダム初期化や他の pretext task を上回る結果が報告されています。たとえば PASCAL VOC 2007 分類では、彩色事前学習が当時の主要な自己教師あり手法と競合し、ランダム初期化との差を大きく開けると同時に、教師あり ImageNet 事前学習との差を縮めました。Larsson らの CVPR 2017 論文では、VGG ベースで彩色事前学習した特徴が PASCAL VOC セグメンテーションの mIoU (予測領域と正解領域の重なりの平均割合、高いほど良い) で当時の他の自己教師あり手法を上回ったことが示されています。

アブレーション (構成要素を抜く検証) も重要です。class rebalancing を外すと着色は明らかにくすみ、被験者を騙せる割合が低下します。回帰ベース (313色分類でなく a,b の直接回帰) にすると、さらに無彩色寄りになり、転移性能も落ちます。つまり「分類化」と「頻度リバランス」のどちらが欠けても、着色品質と表現品質の両方が劣化することが示されました。これは「色の曖昧さへの対処が、見た目の良さと特徴の良さの両方に効く」という主張の裏付けです。

これらの指標が何を測っているかを補足します。linear probing 精度は「特徴がどれだけ線形に分離しやすい良い表現か」を測ります。高いほど、複雑な分類器を足さなくても物体カテゴリが特徴空間で綺麗に分かれている、つまり質の高い特徴だということです。転移検出/セグメンテーションの精度は、その特徴が「画像のどこに何があるか」という空間的・意味的情報まで保持しているかを測ります。着色 Turing test の合格率は、生成タスクとしての知覚的なリアルさを測る指標で、表現品質とは別軸ですが、両者が同じ設計 (分布予測 + リバランス) で同時に改善した点が論文の説得力を高めています。

## メリット・トレードオフ・限界

メリットは次の通りです。正解 (もとの色) がカラー画像から色を抜くだけで自動的に得られ、人手ラベルが完全に不要です。色を正しく当てるには物体の意味理解が必須なので、汎用特徴が育ちやすい。さらに自動着色という、それ自体が実用的な応用 (古い白黒写真や映画の色付け、医療画像の可視化補助) も同時に手に入ります。pretext task としては概念がシンプルで、追加のネットワーク (対照学習のような大きなキューや GAN) を必要としない点も実装上の利点です。

トレードオフと限界もあります。第一に、色の曖昧さは原理的に消えません。学習がうまくいっても出力は「復元」ではなく「もっともらしい色」であり、もとの実際の色とは違うことがあります。表現学習が目的ならこれは問題になりませんが、忠実な復元を期待すると裏切られます。第二に、彩色という単一の手がかりに特化するため、対照学習や後年のマスク復元型ほどには転移性能の上限が高くありません。色だけに頼ると、色に乏しいシーン (夜間、霧、白黒に近い場面) や色が物体識別に効かない領域では学習信号が弱くなります。第三に、313色分類・class rebalancing・annealed-mean といった、曖昧さに対処する設計上の工夫が必須で、素朴に実装すると性能が出ません。

研究上の未解決課題としては、「色という手がかりがどこまで物体理解の十分条件たりうるか」という根本的な限界があります。後の研究は、色のような単一の低レベル手がかりではなく、より一般的な不変性 (対照学習) や情報的に豊かな再構成 (マスク復元) のほうが、データ量とモデル規模をスケールしたときに強い表現を生むことを示しました。彩色はこの探索の初期段階を代表する、教訓的な手法と位置づけられます。

## 発展トピック・研究の最前線

彩色そのものは現在では事前学習の主流ではありませんが、その発想はいくつかの方向に生き続けています。一つは画像生成・編集の文脈で、拡散モデル (diffusion model、ノイズから徐々に画像を生成するモデル) を使った高品質な自動着色が研究されており、多峰性の扱いが「確率分布のサンプリング」という形でより自然に解けるようになりました。Zhang の 313 ビン分類が「離散化して多峰性を表す」工夫だったのに対し、拡散モデルは連続分布を直接サンプルできるため、彩色の本質的困難に対する一つの解答になっています。

もう一つは、彩色を含む複数の pretext task を組み合わせるマルチタスク自己教師あり学習の流れです。Doersch らの "Multi-task Self-Supervised Learning" (ICCV 2017) は、彩色・相対位置予測・inpainting・運動分割を同時に学ばせると単独より良い特徴が得られることを示しました。

より広い視点では、「予測対象を何にするか」という問いが自己教師あり学習の中心テーマであり続けています。彩色は色を、inpainting はピクセルを、対照学習は表現の不変性を、そして [data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) のような蒸留型は内部の潜在特徴を予測対象にします。彩色はこの「予測対象の設計史」の出発点近くに位置し、なぜ単純な低レベル予測から抽象的・情報理論的な目標へと研究が進化したかを理解するうえで、今なお参照価値があります。
