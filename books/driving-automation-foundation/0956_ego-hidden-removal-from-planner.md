# プランナーから自車隠れ状態の出力を切り離す設計

## ひとことで言うと
自動運転の end-to-end モデルでは、軌道(これからどう走るか)を出す「プランナー」という部品があります。初期の設計では、このプランナーが軌道のほかに「自車の内部状態を要約した隠れベクトル(ego hidden state)」も一緒に出力し、それを未来予測などの別の仕事に使い回していました。本稿は、その隠れ状態の出力をプランナーから切り離し、未来予測の責務を専用の別モジュールに移す、という設計判断の一般論を扱います。プランナーを「軌道を出すだけ」の単純な契約にすると、システム全体が読みやすく、改良しやすくなります。

## 直感的な理解
人間が車を運転する場面を考えてください。とっさにブレーキを踏む「反射」と、「この先の交差点はたぶん混むから車線を変えておこう」という「予測」は、頭の中でやっていることが違います。反射は速く、予測はゆっくりでよい。

ニューラルネットワークで運転を学習させるとき、初期の素朴な設計では、これを1つの部品(プランナー)に全部押し込んでいました。プランナーが「これからの軌道」を出すついでに、「今の自車状態の要約ベクトル」も出して、それを「未来がどう見えるか」を予測する別の処理の入り口にしていたのです。

これは一見効率的に見えますが、役割が混ざっています。軌道を直したいだけなのに、未来予測の都合まで考えないとプランナーを触れない。テストも「軌道」と「隠れ状態」の両方の形を確認しないといけない。この絡まりをほどくのが、本稿のテーマです。やることはシンプルで、「プランナーは軌道だけ返す」と決め、未来予測に必要な内部状態はプランナーの外で(あるいは専用モジュールで)作る、という分離です。

## 基礎: 前提となる概念
いくつか用語を噛み砕きます。

- 隠れ状態(hidden state)。ニューラルネットワークの途中に現れる中間ベクトルのことです。入力(画像など)を処理する途中で生まれる「数値の並び」で、人間には直接読めませんが、その時点までに集めた情報が圧縮されて入っています。ここでいう ego hidden state は「ego = 自車」+「hidden = 隠れ状態」で、「今この瞬間の自車の状況を要約した内部ベクトル」を指します。形は典型的には `[B, D]`(B はバッチサイズ=同時に処理するサンプル数、D は次元数、例えば 256)です。

- プランナー(planner)。観測(カメラ画像、地図、過去の動きなど)を受け取り、これからの軌道を出力する部品です。軌道は時系列で、形は例えば `[B, T, C]`(T は予測する時刻数、C は各時刻の信号数、例えば加速度と曲率の2つ)になります。

- 未来予測モジュール。「今こう見えている → これから周囲はこう見えるはずだ」を予測する部品です。生のピクセルを予測するのではなく、特徴空間(ネットワークが抽出した抽象的なベクトルの空間)で予測するのが現代の主流です。後述の JEPA がこの考え方の代表です。

- 責務分離(separation of concerns)。ソフトウェア設計の基本原則で、「1つの部品は1つの仕事だけ持つべき」という考え方です。仕事が混ざると、片方を直すともう片方が壊れる、テストが複雑になる、といった問題が生じます。

ego hidden state がなぜプランナーから出ていたか、を理解しておくと納得感が増します。未来予測モジュールは「予測の起点」となるベクトルを必要とします。プランナーは観測を全部受け取って軌道を作る過程で、まさにその「自車状況の要約」を内部に持っています。だから「ついでにそれも返してしまえ」という発想で、プランナーが ego hidden を出していたのです。便利な近道ですが、責務が混ざる代償を払っていました。

## 仕組みを詳しく
変更の前後を、関数の返り値(部品が何を出すか)で対比すると本質がはっきりします。

変更前(隠れ状態をプランナーが出していた頃)。
```
planner.forward(obs) -> (trajectory, ego_hidden)
model.forward(obs)   -> 学習時: (planner_loss, ego_hidden, future_features)
                        推論時: (trajectory, ego_hidden, None)
```
このとき ego_hidden は未来予測モジュールに渡され、未来の視覚特徴 future_features を生成し、その特徴と「本物の未来特徴」を比べる自己教師あり損失(JEPA 系の再構成損失)の計算に使われていました。返り値が3要素タプルで、しかも学習時と推論時で中身が変わる、という分かりにくい契約です。

変更後(プランナーから隠れ状態を切り離した設計)。
```
planner.forward(obs) -> trajectory                 # 形 [B, T*C]
model.forward(obs)   -> trajectory                 # 学習でも推論でも常に軌道のみ
```
プランナーは「軌道だけを返す」。未来予測は別系統に移す。返り値が常に1つになり、呼び出し側(学習ループ・推論コード・テスト)が大幅に単純化します。

なぜこの設計が良いのか、理由を段階的に積み上げます。

第一に、更新頻度が違うからです。反応的な軌道生成は高頻度(例えば 10Hz = 毎秒10回)で動かしたい。一方、視覚履歴を溜めて未来を予測する処理は低頻度(例えば 1Hz)でよい。この2つを1部品に同居させると、速い経路まで重い未来予測に引きずられて間に合わなくなります。隠れ状態の出力をプランナーから外し、未来予測を専用の低頻度モジュールに寄せると、速い経路を軽く保てます(この2バンド構成の詳細は [反応系と世界予測系を分ける2バンドアーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1007_reactive-e2e-world-action-split) を参照)。

第二に、勾配の衝突を避けられるからです。プランナーが ego_hidden を出して未来予測損失がそこに逆伝播すると、軌道を良くする学習と未来予測を良くする学習が同じ重みを取り合います。分離すれば、それぞれの目的に最適化された設計を独立に組めます。

第三に、テスト容易性です。「常に軌道だけ返す」契約なら、テストは軌道の形と中身だけ確認すればよい。隠れ状態の形・未来特徴の形まで網羅する必要がなくなります。

移行期の注意点として、未来予測モジュールがまだ完成していない段階では、未来予測の経路が一時的に「配線が切れた」状態になることがあります。モジュール自体は構築されるが forward から呼ばれず、勾配も受け取らない(いわゆるデッドコードに近い状態)。これは過渡的な状態で、専用モジュールが完成すれば配線が復活します。設計の意図とコードの実態に一時的なギャップが生じる、という点は理解しておくべきです。

## 手法の系譜と主要論文
「軌道を出す系」と「未来を予測する系」を分ける考え方は、世界モデル研究と自己教師あり学習の系譜の上にあります。

ALVINN (Pomerleau, NeurIPS 1989)。ニューラルネットで運転を学んだ最初期の研究です。カメラ画像から操舵を直接出す、いわゆる行動クローン(behavior cloning、お手本の運転を真似る教師あり学習)の原型でした。ここでは「軌道を出す」以外の責務はなく、未来予測という概念もまだありません。

ChauffeurNet (Bansal et al., RSS 2019, arXiv:1812.03079)。中間表現(俯瞰のラスター地図など)を介して運転方策を学ぶ研究で、補助的な予測タスク(他車の未来位置など)を学習に加えると頑健性が上がることを示しました。「主タスク(軌道)と補助タスク(予測)を持つ」という発想の先駆けです。

JEPA / I-JEPA (LeCun の 2022 年の構想; Assran et al., CVPR 2023, arXiv:2301.08243)。JEPA は Joint-Embedding Predictive Architecture(結合埋め込み予測アーキテクチャ)の略で、「未来や欠損した部分を、生のピクセルではなく特徴空間で予測する」自己教師あり学習の枠組みです。ピクセルを予測しようとすると、本質的でない細部(葉の揺れ、ノイズ)まで当てにいって学習が崩れる(collapse、自明な定数解に落ちる)。特徴空間で予測すれば、意味的に有用な表現を効率よく学べます。ego hidden state を起点に未来特徴を当てる、という初期設計はまさにこの JEPA の発想でした。その役割を専用の予測系に引き渡す、というのが本稿の核心です。

MILE (Hu et al., NeurIPS 2022)。運転の世界モデルで、BEV(俯瞰ビュー)上で「世界がどう変わるか」と「どう行動するか」を同時に学びます。行動方策とは別に世界予測の系を持つことの有効性を示しました。

GAIA-1 (Hu et al., 2023, arXiv:2309.17080)。運転の生成的世界モデルで、行動条件つきで未来を生成します。「もし自分がこう動いたら世界はどう変わるか」を予測する大規模モデルです。世界予測を独立した重い系として扱う、という方向性を強く後押ししました。

V-JEPA / V-JEPA 2 (Bardes et al., 2024; 2025)。JEPA を動画に拡張し、時間方向の未来予測を特徴空間で行います。運転の未来予測モジュールはこの動画版 JEPA の系譜に直接つながります。

この系譜を一言でまとめると、「最初は軌道だけ → 補助予測タスクを足す → 世界予測を独立した系に切り出す → 特徴空間での未来予測(JEPA)が主流に」という流れです。プランナーから隠れ状態を外して未来予測を専用系に寄せる判断は、この自然な流れに沿っています。

## 論文の実験結果(定量データ)
責務分離そのものは「アーキテクチャ整理」であり単一論文の手法ではありませんが、関連する研究の定量結果は分離の妥当性を裏付けます。

I-JEPA (Assran et al., CVPR 2023)。ImageNet-1K(約128万枚の画像、1000クラスの分類ベンチマーク)で、特徴空間予測の有効性を示しました。ViT-Huge/14 を使い、線形プローブ(学習済み特徴の上に線形分類器だけ載せて評価する手法、エンコーダ本体は凍結)で約 72〜73% 前後のトップ1精度を達成し、注目すべきは事前学習効率で、ピクセル再構成型の MAE が同等精度に到達するのに要する学習エポック/計算量より少ない(論文では MAE 比でおよそ数分の一の GPU 時間)と報告されています。「ピクセルではなく特徴を予測する」だけで効率が上がる理由は、ピクセル予測が背景の質感やノイズといった意味的に無価値な高周波成分まで再構成しようとして計算を浪費するのに対し、特徴空間予測は抽象表現だけを当てればよいからです。なぜこれが重要かというと、未来予測を担う系のコストを抑えられることを意味し、それが「重い予測系を低頻度で別に回す」設計を現実的にするからです。

MILE (Hu et al., NeurIPS 2022)。CARLA(運転シミュレータ)の Town05 ベンチマークで評価され、当時の最先端だった手法に対して、運転スコア(Driving Score、ルート完走率と違反ペナルティを掛け合わせた総合指標、100点満点)で大幅な改善を報告しました。報告によると Town05 の評価で従来手法(当時の代表的な模倣ベースライン)に対しておよそ +30% 規模の相対改善を示し、潜在空間(latent space)で世界の遷移を予測しながら方策を学ぶ設計の有効性を示しています。アブレーション(ある要素を抜いて効果を測る実験)では、潜在世界モデル(未来の潜在状態を予測する部分)や、確率的な潜在変数を外すと運転スコアが目に見えて低下することが示され、「予測系を持つこと」自体が運転を改善することの証拠になっています。これは、未来予測を独立した責務として切り出す価値が運転性能の数値に現れる、という直接の根拠です。

ChauffeurNet (Bansal et al., RSS 2019)。補助的な予測損失(衝突予測、他車軌道予測など)を加えると、純粋な行動クローンに比べて、ハードなテストシナリオでの失敗(脱輪・衝突)が顕著に減ったと報告されています。これは「軌道生成と予測を協調させると頑健になる」という主張を支えますが、同時に「両者をどう分離・結合するか」が設計の核心であることも示しています。

これらの数値が共通して示すのは、「未来予測は運転性能に効く。ただしそれを軌道生成と同じ部品に詰め込む必要はなく、むしろ分離した方が効率と保守性が上がる」という点です。

## メリット・トレードオフ・限界
メリット。
- プランナーの責務が「軌道を出す」一点に絞られ、契約が単純で読みやすい。返り値が常に軌道のみになり、学習ループ・推論コード・テストがすべて単純化する。
- 速い反応系と遅い推論系の役割分担が明確になり、高頻度の経路を軽量に保てる。
- 未来予測系を独立に設計・最適化でき、勾配の取り合いが起きない。

トレードオフ・限界。
- 移行の過渡期には、未来予測の経路が一時的に切れた状態になりうる。予測モジュールは構築されるが呼ばれず、勾配も来ない。設計意図とコードの実態にギャップが残る。
- 専用予測系が完成するまで、軌道生成に渡る「視覚履歴ベクトル」がプレースホルダ(ゼロ埋めなど)になり、未来予測の恩恵を受けられない期間がある。
- 2系統に分けると、両者をつなぐインターフェース(隠れ状態の次元、更新タイミング)を慎重に設計する必要があり、設計コストが増える。

研究上の未解決課題。未来予測を「どの空間で」「どの粒度で」行うのが運転性能に最適か(occupancy=占有格子か、特徴空間か、低次元か)は確立していません。また、予測系を独立に学習すると軌道系との一貫性をどう担保するか(行動条件づけ、共有エンコーダ + 勾配停止など)も活発な研究テーマです。

## 発展トピック・研究の最前線
近年は、世界モデルを大規模化して「行動条件つき未来予測」を運転計画に直接組み込む流れ(GAIA-1、V-JEPA 2 など)が強まっています。隠れ状態をプランナーから外す本稿の判断は、その大規模化を見据えた前提整備でもあります。

もう一つの最前線は、target encoder(予測対象を作る側のエンコーダ)の扱いです。JEPA 系では予測対象を固定しないと自明解への崩壊が起きるため、frozen(凍結=学習で更新しない)や EMA(指数移動平均、ゆっくり追従させる)で安定化します。BYOL (Grill et al., NeurIPS 2020) がこの「target を止めれば崩壊しない」ことを実証した代表例で、未来予測系の安定化設計の理論的な支えになっています。
