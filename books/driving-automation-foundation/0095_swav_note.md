# SwAV (Swapping Assignments between Views)

## ひとことで言うと
1枚の画像から少し違う見え方(切り抜きや色を変えた2バージョン、これを「ビュー」と呼びます)を作り、両方をいくつかの「グループ(クラスタ)」に振り分けます。そのとき「ビューAの特徴からビューBのグループ割当を当てる」「ビューBの特徴からビューAのグループ割当を当てる」と、割当を入れ替えて(swap、スワップ)予測させる自己教師あり学習です。ラベルが一切なくても、しかも従来の対照学習が必要とした大量の「負例」を持たずに、画像から良い特徴を学べるのが最大の特徴です。

## 直感的な理解
画像から特徴を学びたいけれど、正解ラベル(これは犬、これは車、といった人手のタグ)が手元にない、という状況を考えます。人間でも、似た写真をなんとなく束にまとめることはできます。「この束は犬っぽい」「この束は風景っぽい」と、名前を知らなくてもグループ分けはできる。SwAV の発想はまさにこれです。あらかじめ正解を知らなくても、データを「似たもの同士のグループ(クラスタ)」に分け、その「どのグループに属するか」を学習の手がかりにします。

ここで重要なのが「同じ写真は、切り抜きや色味を変えても、属すべきグループは変わらないはず」という当たり前の事実です。犬の写真を左半分だけ切り抜いても、白黒にしても、それは犬グループのままです。SwAV はこの不変性を使います。ビューAを見て決めたグループ割当を、ビューBの特徴から当てさせる。グループという「中間の言葉」を介すことで、画像同士を一枚ずつ突き合わせる必要がなくなります。

なぜ「割当を当てる」が良い学習信号になるのか、もう一段かみ砕きます。対照学習(後述)は「自分のビューと、同じ画像のもう片方のビューが、特徴空間で一番近い」ことを直接求めます。これは強力ですが、近い遠いの比較相手(負例)を大量に必要とします。SwAV は比較相手を「画像」ではなく「K個の固定された参照点(プロトタイプ)」に置き換えます。プロトタイプは全画像が共有する共通の物差しなので、自分のバッチに何枚あろうと、物差し自体は安定して存在します。これにより「負例の山を用意する」というコストの源が消えるのです。

## 基礎: 前提となる概念
このトピックを理解するために、いくつか言葉を噛み砕いておきます。

自己教師あり学習(self-supervised learning, SSL): 人手のラベルを使わず、データ自身の構造から「擬似的な問題」を作って学ぶ枠組みです。例えば「画像の一部を隠して当てさせる」「2つの見え方を近づける」など。ラベル付けのコストが膨大な現実では、大量の生データから事前に良い特徴(表現、representation)を学んでおき、後で少量のラベルで微調整(fine-tuning)するのが効率的です。

対照学習(contrastive learning): SSL の代表アプローチで、「同じ画像の2つの見え方(正例ペア)は特徴空間で近づけ、無関係な別画像(負例)は遠ざける」ことで学びます(→ [InfoNCE / NT-Xent ロス](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0061_info-nce-loss))。特徴空間とは、画像を数百次元のベクトルに変換した後の空間で「近い=似ている」を距離で表せます。問題は、良い性能のために大量の負例が要ること。SimCLR は一度に数千枚のバッチを、MoCo は過去の特徴をためるメモリバンクを用意しました(→ [momentum encoder / EMA target](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0078_momentum-encoder-ema))。

クラスタ(cluster)とプロトタイプ(prototype): クラスタはデータを似たもの同士でまとめた束です。各クラスタには「中心」となる代表ベクトルがあり、これをプロトタイプと呼びます。SwAV はプロトタイプを少数(代表的には K=3000 個)用意し、各画像を「どのプロトタイプに近いか」で表現します。プロトタイプはネットワークと一緒に学習で更新される、学習可能なパラメータです。形状でいえば D 次元のベクトルが K 本並んだ行列 C ∈ R^{D×K} です。

データ拡張(data augmentation): 1枚の画像から、切り抜き(crop)・左右反転・色変化(色相や明るさのゆらぎ)・ぼかしなどで複数のバリエーションを人工的に作る操作です。SSL ではこれが「同じものの別の見え方=正例ペア」を作る源になります。

最適輸送(optimal transport)と Sinkhorn: 最適輸送は「ある分布を別の分布へ、最小コストで運ぶ割当」を求める数学の枠組みです。SwAV では「B個の画像を K 個のクラスタへ、各クラスタが均等な容量を持つよう運ぶ」という輸送問題として割当を定式化します。Sinkhorn-Knopp(シンクホーン・クノップ)アルゴリズムは、輸送問題にエントロピー正則化を加えると、行と列を交互に正規化する単純な反復だけで高速に解けるという、Cuturi(2013)が機械学習に持ち込んだ手法です。SwAV はこれを GPU 上の数反復で回します。

崩壊(collapse): SSL で最も警戒される失敗で、全画像が同じ特徴(あるいは全画像が同じクラスタ)に潰れてしまう状態です。「すべて同じ」なら正例は確かに一致しますが、何も区別できず無意味です。SwAV はこれを「バッチ内をクラスタに均等に割り振る制約」で防ぎます。

## 仕組みを詳しく
SwAV の学習は5つのステップで回ります。tensor shape の例(バッチサイズ B=256、特徴次元 D=128、プロトタイプ数 K=3000)を添えます。

ステップ1: 2つのビューを作る。1枚の画像に異なるデータ拡張を2回かけ、ビュー1とビュー2を得ます。

ステップ2: 特徴に変換。共通のエンコーダ(ResNet-50 や ViT)に通し、それぞれを L2 正規化(ベクトルの長さを1に揃える操作)した特徴ベクトル z1, z2 を得ます。形状は [B, D] = [256, 128]。長さを1に揃えると、内積がそのままコサイン類似度(向きの近さ、-1〜1)になります。

ステップ3: プロトタイプとの内積。学習可能なプロトタイプ行列 C(形状 [D, K] = [128, 3000])との内積をとり、各ビューが各プロトタイプにどれだけ近いかのスコア(logits)を出します。z1·C の形状は [B, K] = [256, 3000]。これが「各画像が各クラスタにどれだけ近いか」の生スコアです。

ステップ4: 割当(コード)を計算。ここが SwAV の肝です。各ビューに対し、ソフトな割当 q1, q2(形状 [B, K]、各画像がどのクラスタにどれだけ属するかの確率的な配分)を計算します。単純にソフトマックスをとるのではなく、Sinkhorn-Knopp で「バッチ内の B 個の画像を、K 個のクラスタになるべく均等に振り分ける」という等分配制約のもとで割当を求めます。なぜ均等化が要るか。制約がないと、ネットワークは「全画像を1つのクラスタに放り込む」という自明な(崩壊した)解を選びがちです。等分配制約はこれを禁止し、各クラスタが一定の容量を持つよう強制します。Sinkhorn は行と列を交互に正規化する反復(代表的には3反復程度)で、この制約付き割当を高速に近似します。重要なのは、この割当 q はその場(オンライン)で計算され、勾配を通しません(ターゲットとして固定する)。

数式で書くと、輸送行列 Q ∈ R^{K×B} を、コスト行列 C^T Z(プロトタイプと特徴の類似度)を報酬とみなしつつエントロピー H(Q) で正則化して最大化します。max_Q Tr(Q^T C^T Z) + ε H(Q) を、行和・列和が一定という制約のもとで解く。解は Q* = Diag(u) exp(C^T Z / ε) Diag(v) という形になり、u, v が Sinkhorn 反復で求まる正規化ベクトルです。ε(イプシロン)は正則化の強さで、小さいほど割当が鋭く(ハードに近く)なります。

ステップ5: スワップ予測。名前の由来です。ビュー1のスコアにソフトマックスをかけた予測 p1 で、ビュー2の割当 q2 を当てさせます。逆も同様に p2 で q1 を当てさせます。損失は交差エントロピー(2つの確率分布の食い違いを測る指標)で、

```
L = - (1/B) Σ_b [ Σ_k q2[b,k] · log p1[b,k]  +  Σ_k q1[b,k] · log p2[b,k] ]
```

を最小化します。p1 は z1 から作るので「ビュー1の特徴」、q2 は z2 から作るので「ビュー2のクラスタ割当」。これを当てさせるのが「割当をスワップする」の意味です。「同じ画像なら、見え方が違ってもクラスタ割当は同じはず」という制約を、割当を入れ替えて学習しているわけです。

ソフトマックス予測の式は温度 τ(タウ、典型的に 0.1)を使い p1 = softmax(z1·C / τ) です。τ が小さいほど分布が鋭くなります(→ 温度の役割は [InfoNCE / NT-Xent ロス](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0061_info-nce-loss) で詳述)。

multi-crop(マルチクロップ)という工夫も SwAV の重要な発明です。標準解像度の大きな切り抜き2枚(例: 224×224)に加えて、低解像度(例: 96×96)の小さな切り抜きを複数(代表的に6枚)作ります。小さい切り抜きの特徴からは予測 p のみを作り(これらの割当 q は計算しない)、大きい切り抜きの割当を当てさせます。小さい切り抜きは計算が軽い(畳み込みの計算量はおよそ入力面積に比例するので 96² / 224² ≈ 0.18 倍)ので、安く「見え方のバリエーション」を増やせます。

実装上の小バッチ対策も知っておく価値があります。Sinkhorn の等分配はバッチ内で統計を取るため、バッチが小さいと割当が不安定になります。SwAV は直近数バッチの特徴を queue にためて Sinkhorn の母数に加えることで、小バッチでも等分配が効くようにしています。これは MoCo の queue と似た発想です(→ [momentum encoder / EMA target](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0078_momentum-encoder-ema))。

before/after で整理すると、対照学習(before)は「特徴 対 特徴」を大量に直接比較し、それゆえ大量の負例を要した。SwAV(after)は「特徴(p) 対 クラスタ割当(q)」を比較する形に置き換え、負例の山が不要になりました。

## 手法の系譜と主要論文
クラスタ型 SSL の流れを追います。

DeepCluster(Caron, Bojanowski, Joulin, Douze, ECCV 2018, "Deep Clustering for Unsupervised Learning of Visual Features", arXiv:1807.05520, Facebook AI Research)。SwAV と同じ筆頭著者 Mathilde Caron らの先行研究です。アイデアは「全データの特徴を k-means でクラスタリングして擬似ラベルを作り、それを通常の分類問題として当てる」を繰り返すもの。新規性は、クラスタリングと表現学習を交互に回す素朴で強力な枠組みを示したこと。弱点は、エポックごとに全データを再クラスタリングする重い処理が必要で、空クラスタの扱いやクラスタの並べ替え(エポック間でクラスタ番号が入れ替わる問題)など不安定さもあったことです。

SeLa(Asano, Rupprecht, Vedaldi, ICLR 2020, "Self-labelling via simultaneous clustering and representation learning", arXiv:1911.05371, Oxford VGG)。擬似ラベル付けを最適輸送問題として定式化し、Sinkhorn-Knopp で等分配制約付きの割当を解く着想を導入しました。これにより DeepCluster の「全部1クラスタに潰れる」退化を等分配で原理的に防ぎました。SwAV の Sinkhorn 利用の直接の源流です。

SwAV(Caron, Misra, Mairal, Goyal, Bojanowski, Joulin, NeurIPS 2020, arXiv:2006.09882, Facebook AI Research)。DeepCluster と SeLa の知見を統合・刷新しました。新規性は3つ。(1) 全データの再クラスタリングをやめ、Sinkhorn による割当をバッチ内でオンラインに計算する(SeLa は全データ一括だった)。(2) 割当をビュー間でスワップ予測する対照不要の目的関数。(3) multi-crop による安価なビュー増強。これにより、大バッチもメモリバンクも不要にしつつ高精度を達成しました。

その後の系譜。DINO(Caron et al., ICCV 2021)は同じ著者系統で、陽なクラスタを持たず教師-生徒の自己蒸留に進化させました。centering+sharpening という SwAV の等分配・鋭利化に通じる崩壊回避を別形で実装しています(→ [momentum encoder / EMA target](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0078_momentum-encoder-ema))。対照学習系の SimCLR / MoCo(→ [InfoNCE / NT-Xent ロス](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0061_info-nce-loss))、対照なしの BYOL / SimSiam(→ [predictor (予測ヘッド非対称性)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0083_prediction-head-predictor))と並べると、SwAV は「クラスタ割当という離散的中間表現を介する」中間的な立ち位置にあります。崩壊回避の流儀としては「負例」(SimCLR/MoCo)、「非対称性」(BYOL/SimSiam)、「正則化」(VICReg/Barlow Twins)に対し、SwAV は「等分配制約」という第4の流儀を代表します。

## 論文の実験結果(定量データ)
評価指標の意味から。SSL の標準評価は「線形評価(linear probing)」です。これは、事前学習したエンコーダの重みを凍結(固定)したまま、その上に線形分類器(1層の全結合)だけを ImageNet のラベルで学習し、Top-1 精度(最も確からしい1つが正解と一致した割合)を測る方法です。エンコーダ自体は微調整しないので、「学んだ特徴がどれだけそのまま使えるか」を純粋に測れます。

SwAV の主要結果(ResNet-50、ImageNet 線形評価、報告による)。SwAV は Top-1 で約 75.3% を達成しました。これは当時の SSL 最高水準で、完全な教師あり学習の ResNet-50(約 76.5%)に約1ポイント差まで迫る数字です。比較対象として、SimCLR は約 69.3%、MoCo v2 は約 71.1%、BYOL は約 74.3% でした。SwAV が数ポイント上回っており、ImageNet では1ポイントの差でも明確な改善とされます(線形評価は飽和域に近く、伸ばすのが難しいため)。

multi-crop のアブレーション(どの要素を抜くと性能がどう変わるかの分解実験)が示唆に富みます。報告によると、標準の2ビューのみ(multi-crop なし)では約 72% 前後にとどまり、multi-crop(2大+6小、2×224+6×96 と表記)を加えると約 75% へ、おおよそ 3〜4 ポイント向上しました。興味深いのは、multi-crop は SwAV 専用ではなく、SimCLR や DeepCluster-v2 など他手法に足しても 2〜4 ポイント程度の底上げが報告されたことです。汎用的なテクニックとして広まりました。

転移学習(他タスクへの応用)でも、検出・セグメンテーション(COCO や PASCAL VOC)で、教師あり ImageNet 事前学習を上回る転移性能が報告されました。例えば COCO の物体検出・インスタンスセグメンテーション(Mask R-CNN, R50-FPN)で、教師あり事前学習比で AP が複数点改善したと報告されています。これは「ラベルなしで学んだ特徴が、ラベルありで学んだ特徴より下流で有用」という当時インパクトのある結果でした。

データ量のスケーリングでは、SwAV を Instagram の大規模未ラベル画像(数億〜十億枚規模)で学習させると性能がさらに伸び、SSL が大規模データで素直にスケールすることも示されました。半教師あり設定(ImageNet ラベルの 1% / 10% のみで fine-tune)でも、SwAV 事前学習は当時の上位手法に匹敵する結果を出しています。

ハイパーパラメータの頑健性について。Sinkhorn の反復回数は3回程度で十分とされ、それ以上増やしても精度はほぼ飽和します。プロトタイプ数 K は 3000 が標準ですが、報告では数百〜数万のレンジで性能はおおむね頑健で、極端に小さいと表現力が足りず、過剰に大きくしても利得は小さいとされます。重要なのは「クラスタ数が真のクラス数(ImageNet なら 1000)と一致する必要はない」という点で、3000 という過剰クラスタ数でうまく働くことが、クラスタ型 SSL が意味的カテゴリそのものではなく細かい視覚的グループを学んでいることを示唆します。

## メリット・トレードオフ・限界
メリット
- 大量の負例・メモリバンク・巨大バッチが不要。クラスタ割当という中間表現を介すため、SimCLR のような数千バッチを必要としない。
- multi-crop により安価にビューを増やせ、性能が伸びる。他手法にも流用可能な汎用テクニック。
- 割当はバッチ内オンライン計算で、DeepCluster のような全データ再クラスタリングが不要。学習が1パスで回る。
- 線形評価・転移ともに当時最高水準で、教師あり事前学習を下流で上回った。

トレードオフと限界
- ハイパーパラメータが増える。プロトタイプ数 K、Sinkhorn 反復、正則化 ε、温度 τ、multi-crop の枚数・解像度など。
- 崩壊回避を等分配制約に頼る。バッチが小さいと Sinkhorn の等分配が効きにくく不安定になりうる(queue で過去のスコアをためる工夫が必要になる)。
- Sinkhorn-Knopp や最適輸送の理解が要り、実装は対照学習よりやや複雑。
- 単画像ベースで、時系列・行動・物理の情報は素直には入らない。動画や運転データの「時間方向の予測」とは設計思想が異なる。
- クラスタ数を陽に決める必要があり、データの本来の意味的粒度と一致する保証はない(過分割・過併合が起こりうる)。
- 等分配制約は「データが各クラスタにほぼ均等に分布する」という仮定を暗に置く。現実の長尾分布(クラスごとに頻度が極端に偏るデータ)ではこの仮定が破れ、頻度の低いカテゴリを過剰に膨らませる方向に働きうる。

未解決の研究課題として、クラスタ数を自動決定する方法、長尾分布での等分配制約の妥当性、動画・マルチモーダルへの自然な拡張、クラスタが学習する意味的粒度の制御などが挙げられます。

## 発展トピック・研究の最前線
SwAV 以降、SSL は大きく3系統に分岐しました。クラスタ型(SwAV, DINO)、生成・再構成型(MAE のようにマスクした画素やトークンを復元する → [MAE (Masked Autoencoders)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0070_mae_note))、そして両者を統合する方向(iBOT, DINOv2)です。特に DINOv2(Oquab et al., 2023)は DINO/iBOT の自己蒸留に大規模データキュレーションを組み合わせ、汎用視覚特徴として fine-tuning なしで多数の下流タスクに使える「基盤モデル(foundation model)」を目指しました。SwAV の等分配・オンライン割当の発想はこの流れの起点の一つです。

最適輸送と SSL の接続も活発な研究領域で、Sinkhorn 以外の輸送解法や、割当をより滑らかにする正則化、輸送計画を表現学習以外(ドメイン適応・生成モデル評価)に転用する研究が探られています。動画・運転領域では、空間クラスタリングより時間方向の予測(現在から未来の特徴を当てる → [行動条件付き潜在予測 (counterfactual rollout)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0001_action-conditioned-prediction))や再構成型のほうが相性が良い場面が多く、SwAV の直接適用は限定的ですが、multi-crop の「安価に視点を増やす」発想は高解像度映像の表現学習で引き続き有用です。また、点群・LiDAR への等分配クラスタリング SSL の応用も自動運転分野で試みられています。
