# ノイズ疑似ラベル向け label smoothing

## ひとことで言うと

正解ラベルを「100% このクラスだ」と断定する代わりに、「だいたいこのクラスだが、他のクラスの可能性も少しは残す」とぼかして学習する手法です。とくに、自動生成された疑似ラベル (たとえば視覚言語モデル VLM が付けたラベル) のように一定割合で間違っている教師信号を使うとき、モデルがその (もしかしたら誤った) ラベルに過剰な自信をもって従いすぎないようにするために使います。誤りを強く刷り込むのを防ぐ、軽量で効果的なノイズ対策です。

## 直感的な理解

学校のテストで、先生が配った模範解答が一部間違っているとします。生徒が「模範解答は絶対正しい」と思い込んで丸暗記すると、間違いごと覚えてしまい、似た問題で同じミスを繰り返します。これが機械学習で言う「ノイズラベルへの過剰追従」です。

label smoothing は、生徒に「模範解答はだいたい正しいけど、100% 鵜呑みにはするな。他の答えの可能性も頭の片隅に残しておけ」と教えるようなものです。正解とされたクラスへの確信をほんの少し下げ、他のクラスにもわずかに確率を残します。すると、たとえその「正解」が誤っていても、生徒 (モデル) は誤りを全力で刷り込むことを避けられ、誤ラベルに引きずられにくくなります。

逆に、ラベルが本当に完璧に正しいなら、ぼかさず断定して覚えたほうが学習効率は良いはずです。だから label smoothing は「ラベルにノイズがある」「モデルが過信しやすい」状況でこそ効く、という性質を持ちます。疑似ラベルを使う学習はまさにこの状況に当てはまります。

## 基礎: 前提となる概念

- one-hot ラベル — 「正解クラスだけ 1、他は全部 0」という正解の表現。5クラスで正解が3番目 (インデックス2) なら `[0, 0, 1, 0, 0]`。
- 交差エントロピー損失 (cross-entropy loss) — 分類でよく使う損失。モデルが出した確率分布と、正解の分布 (one-hot) のズレを測ります。これを小さくするよう学習すると、モデルは正解クラスの確率を 1 へ、他を 0 へ近づけようとします。
- 過信 (over-confidence) — モデルが正解クラスへ極端に大きなスコア (ロジット) を出し、自分の予測を実際以上に確からしいと思い込む状態。汎化 (未知データでの性能) を下げ、確率の信頼性 (キャリブレーション) を損ないます。
- キャリブレーション (calibration) — モデルが「確率 0.9」と言ったとき、本当に約 90% の頻度で当たっているか、という確率の確からしさ。過信したモデルはキャリブレーションが悪く、0.99 と言っても実際は 0.7 しか当たらない、といったことが起きます。
- 確証バイアス (confirmation bias) — ここでは、誤った信念を学習がさらに補強してしまう現象。誤ラベルに断定的に従うと、似た場面で同じ誤りを再生産します。
- 平滑化係数 ε (イプシロン) — ぼかしの強さを決めるパラメータ。0 で平滑化なし、大きいほど強くぼかします。典型的には 0.1 程度。

## 仕組みを詳しく

label smoothing は、one-hot ラベルを少し均す (smooth する) 操作です。平滑化係数を ε、クラス数を K とすると、ターゲット分布は次のように作られます。

- 正解クラスの目標確率: `1 - ε + ε/K`
- それ以外の各クラス: `ε/K`

具体例で計算します。K=5、ε=0.1、正解が3番目 (traffic_light, インデックス2) とすると、

- 正解クラス = `1 - 0.1 + 0.1/5 = 0.92`
- 他の4クラスそれぞれ = `0.1/5 = 0.02`
- 目標分布 = `[0.02, 0.02, 0.92, 0.02, 0.02]` (合計 1.0)

before / after で並べると、

```
before (one-hot)      : [0,    0,    1,    0,    0   ]
after  (smoothed ε=0.1): [0.02, 0.02, 0.92, 0.02, 0.02]
```

「正解は 1 ではなく 0.92 でよい」「他のクラスにも 0.02 ずつ残す」というゆるい正解になります。

実装は驚くほど単純で、多くのフレームワークでは交差エントロピー関数の引数1つで実現できます。たとえば PyTorch では `F.cross_entropy(logits, labels, label_smoothing=ε)` のように ε を渡すだけです。クラス不均衡対策の `weight` (クラスごとの重み) と併用もできます。

```
def classification_loss(logits, labels, label_smoothing=0.0, class_weights=None):
    return F.cross_entropy(logits, labels,
                           weight=class_weights,
                           label_smoothing=label_smoothing)
```

なぜこれで誤ラベル耐性が上がるのか、を損失の形で見ます。one-hot + 交差エントロピーは、正解クラスの確率 p に対しておよそ `-log(p)` の損失を課します。p を 1 に近づけるほど損失が減るので、モデルは正解クラスのロジットを際限なく大きくしようとします (過信)。もしその「正解」が誤っていれば、誤ったクラスへロジットを際限なく押し上げることになります。label smoothing では目標が 0.92 で頭打ちになり、それ以上確信を高めると損失が逆に増える (行き過ぎを罰する) ようになります。結果として、誤ラベルへの暴走的な確信形成にブレーキがかかります。

勾配の観点でも見ておきます。softmax 出力を q、目標分布を t とすると、ロジットに対する交差エントロピーの勾配はおよそ `q - t` です。one-hot では正解クラスの目標が 1 なので、q がどれだけ大きくなっても勾配 `q - 1` は 0 に達するまで正解方向へ押し続けます。label smoothing では目標が 0.92 なので、q が 0.92 を超えると勾配の符号が反転し、それ以上の過信を引き戻します。この「引き戻し」が誤ラベルへの暴走を抑える数学的な正体です。

効果を一文で言うと、「ラベルが間違っているとき、モデルが他のクラスにも確率を残すことを許すので、誤った断定を刷り込まれにくくなる」。逆に ε=0 だと誤ラベルにも全力で従わされます。この挙動は、ε>0 にすると損失値が変わり、かつ勾配は末端まで流れ続ける (学習自体は止まらない) ことをテストで確認するのが定石です。

ε の選び方には正解率とのトレードオフがあります。ノイズ率が高いほど大きめの ε が有利になりがちですが、大きすぎると正しいラベルからも学べる情報が薄まり、分類が鈍ります。経験的には 0.1 前後から始めて、検証データで調整します。

## 手法の系譜と主要論文

- Szegedy et al., "Rethinking the Inception Architecture for Computer Vision" (CVPR 2016, arXiv:1512.00567)。label smoothing を初めて広く提案した論文 (Inception-v3 の構成要素の一つ)。動機は「モデルが正解クラスへ過剰に大きなロジットを出して過信し、汎化が落ちるのを防ぐ」こと。クリーンなラベルの ImageNet ですら、ぼかすことで精度とキャリブレーションが改善することを示しました。限界として、正解が完全に正しい場合はわずかに最適から外れうる点が挙げられます。

- Müller, Kornblith, Hinton, "When Does Label Smoothing Help?" (NeurIPS 2019, arXiv:1906.02629)。label smoothing が表現に与える効果を分析。同じクラスのサンプルを潜在空間で密にまとめ、クラス間を等距離に押し広げる (タイトなクラスタを作る) ことを可視化で示しました。動機は「効く理由と効かない場面を理解する」こと。重要な副作用として、後段で知識蒸留 (知識を別モデルへ移す手法) を行う場合、クラス間の細かい関係情報 (ロジットの微妙な大小) が潰れて蒸留の質が下がることを示しました。蒸留を使わない設定ではこの副作用は問題になりません。

- Lukasik et al., "Does Label Smoothing Mitigate Label Noise?" (ICML 2020, arXiv:2003.02819)。本ノートの状況にもっとも直接対応する論文。label smoothing がラベルノイズに対して頑健性を与えうることを、理論 (損失の勾配がノイズの影響を減衰させる) と実験の両面で示しました。動機はまさに「ノイズの多いラベルでどう学ぶか」。効果はノイズ下での精度改善。限界として、ノイズが非常に大きい場合は平滑化だけでは不十分で、別のノイズ対策との併用が要る点を指摘しています。疑似ラベルはノイズを含むため、この知見が直接当てはまります。

- Zhang & Sabuncu, "Generalized Cross Entropy Loss for Training Deep Neural Networks with Noisy Labels" (NeurIPS 2018, arXiv:1805.07836)。ノイズラベル専用に設計されたロバスト損失の代表。交差エントロピー (ノイズに弱いが学習が速い) と平均絶対誤差 (ノイズに強いが学習が遅い) を1つのパラメータ q で連続的に補間します。label smoothing が「ターゲットを軟化させる」のに対し、こちらは「損失関数そのものをノイズ頑健に作り替える」アプローチで、両者は併用も比較もされます。

- 系譜としては、label smoothing は「ハードターゲットを和らげる」一連の正則化 (regularization、過学習を防ぐ工夫) の一つで、温度付き softmax を使った知識蒸留 (Hinton et al., 2015, arXiv:1503.02531) や、ノイズラベル専用手法 (Co-teaching, Han et al. NeurIPS 2018, arXiv:1804.06872; bootstrapping, Reed et al. 2015) と問題意識を共有します。label smoothing はその中で最も実装が軽く、引数1つで効くのが特徴です。

## 論文の実験結果(定量データ)

- Szegedy et al. (CVPR 2016) は ImageNet 分類で、label smoothing (ε=0.1) により top-1 / top-5 誤り率がおよそ 0.2 ポイント程度改善したと報告しています。指標の意味: top-1 誤り率は「最有力候補が外れた割合」、top-5 は「上位5候補に正解が入らなかった割合」。わずかな差に見えますが、ImageNet 規模では有意で、追加コストがほぼゼロ (引数1つ) であることを考えると費用対効果が高い改善です。

- Müller et al. (NeurIPS 2019) は複数のデータセット (ImageNet, CIFAR-10/100 等) で、label smoothing がキャリブレーション誤差 (ECE, Expected Calibration Error、モデルの言う確率と実際の正解率のズレ、小さいほど良い) を大きく下げることを定量的に示しました。たとえば温度スケーリングなしの素朴な学習で生じる高い ECE を、ε=0.1 の平滑化だけで温度スケーリング後と同等水準まで下げられる例を示しています。一方で、その状態のモデルを教師にして知識蒸留すると、生徒の精度が ε=0 の教師から蒸留した場合より下がる、というトレードオフを実験で示しています。

- Lukasik et al. (ICML 2020) は、人工的にラベルをノイズ化したデータ (一定割合のラベルをランダムに入れ替えた設定) で、label smoothing がノイズなしの素朴な学習より高い test accuracy を達成することを示しました。報告では、ノイズ率が中程度 (例 20〜40%) のとき改善が顕著で、ノイズ率が極端に高い領域 (例 70% 超) では効果が頭打ちになる、という依存性を示しています。これは「ε の最適値はノイズ率に依存する」という実務的指針の根拠になります。

- Zhang & Sabuncu (NeurIPS 2018) は、CIFAR を様々なノイズ率で汚した設定で、Generalized Cross Entropy が素の交差エントロピーより大幅に高い精度を保つことを示しました。たとえばノイズ率 40% で交差エントロピーが大きく劣化する一方、ロバスト損失は精度低下を抑えました。これは「ターゲット軟化 (label smoothing) と損失関数の頑健化は別軸の対策で、どちらもノイズに効く」ことを示し、両者の使い分け・併用の指針になります。

- アブレーションの読み方: これらの研究を通じて一貫するのは「ε=0 (平滑化なし) → ε を上げる」と、(1) キャリブレーションは単調に改善、(2) 精度はノイズ下では改善するがクリーンなデータでは ε を上げすぎると劣化、という二つの傾向です。つまり ε は「キャリブレーションと精度のトレードオフ」「ノイズ耐性と情報量のトレードオフ」を調整するつまみだと理解できます。

## メリット・トレードオフ・限界

メリット
- 引数1つを渡すだけで、ノイズ疑似ラベルへの過剰追従を抑えられる。実装コストがほぼゼロ。
- モデルの予測確率が過信しにくくなり、キャリブレーションが改善する。
- クラス重み (不均衡対策) と併用でき、ノイズ対策とクラス不均衡対策を両立できる。
- 同クラスのサンプルを潜在空間で密にまとめ、表現を整える副次効果がある。

トレードオフ・限界
- 平滑化が強すぎる (ε が大きすぎる) と、正しいラベルからも学べる情報が薄まり、分類が鈍る。
- ノイズが極端に多い場合は平滑化だけでは足りず、追加のノイズ対策 (サンプル選別、ロバスト損失など) との併用が要る。
- ε の最適値はデータのノイズ率に依存し、検証データでの調整が必要。普遍的なベスト値は存在しない。
- 後段で知識蒸留を行う場合は、クラス間の細かい関係情報が潰れて悪影響が出る (Müller et al.)。蒸留を使わなければ非該当。
- 未解決課題: ノイズが系統的 (特定クラス間で偏って起きる) な場合、一様な ε では対処しきれない。クラス依存・サンプル依存の適応的な平滑化が研究テーマです。

## 発展トピック・研究の最前線

- 適応的 label smoothing: サンプルごと・クラスごとに ε を変える手法 (難しいサンプルや不確かなラベルほど強くぼかす)。系統的ノイズへの対処として有望です。
- ロバスト損失との比較・併用: Generalized Cross Entropy (Zhang & Sabuncu, NeurIPS 2018) や対称交差エントロピー (Symmetric Cross Entropy, Wang et al. ICCV 2019) など、ノイズラベル専用に設計された損失と、label smoothing を組み合わせる研究。
- サンプル選別との組み合わせ: Co-teaching や small-loss trick (損失の小さいサンプルを信頼する) と label smoothing を併用し、ノイズの多い疑似ラベルから段階的に学ぶパイプライン。
- 蒸留との関係の再検討: label smoothing と温度付き蒸留はどちらもターゲットを軟化させる操作で、両者の相互作用 (なぜ片方が片方を阻害するか) の理論的理解が進んでいます。
- 確信度を持つ疑似ラベルとの統合: 教師 (VLM など) が出した確信度に応じて ε を変える、あるいは確信度の低い疑似ラベルを学習から間引く、といったソフトな取り扱いが探られています。
