# ベジェ (Bernstein基底) 軌道プランナ

## ひとことで言うと
車がこれからどう走るか(軌道)を、時刻ごとに一点ずつバラバラに出すのではなく、少数の「制御点」から滑らかな曲線を引いて表す方式です。ベジェ曲線(数学的には Bernstein 基底)という表現を使うことで、出てくる運転指令が「カクカクしない=滑らか」になることが計算式の上で保証されます。後段の平滑化フィルタも、学習時の平滑ペナルティ調整も不要にできるのが、この方式の最大の魅力です。

## 直感的な理解
プランナ(planner、軌道を決める部品)の仕事は「未来の軌道」を出すことです。たとえば数秒先までを多数の時間ステップに区切り、各ステップに2つの制御信号(典型的には加速度と曲率、すなわちハンドルの曲がり具合)を出す、という形を取ります。

素朴なやり方は、各ステップを1つずつ順番に予測していくもの(自己回帰, autoregressive)です。GRU(Gated Recurrent Unit、過去の情報を覚えながら系列を生成する小さなネットワーク)を使う方式が典型です。ところが1ステップずつ自由に予測すると、隣り合うステップの値が大きく跳ねることがあります。加速度が「+2, -3, +4, ...」のように激しく揺れると、その変化率(ジャーク, jerk。加速度の時間変化で、乗り心地と機械への負担を左右する量)が大きくなり、乗っていて不快なうえ、実車の制御器(コントローラ)も追従しにくくなります。

これを抑える従来の対処は2つあります。1つは出力に後処理として平滑化フィルタをかける方法。もう1つは学習時に「滑らかにせよ」というペナルティ(損失項)を足す方法です。しかしフィルタは余計な後処理で、学習時の目標と推論時の出力がずれる原因になります(train-test mismatch)。ペナルティは重みの調整が面倒で、強すぎると軌道が鈍くなり、弱すぎると揺れが残ります。しかも「滑らかにせよ」というペナルティは軟らかい制約なので、最小化の途中で必ず守られる保証はありません。

そこで「そもそも滑らかな曲線しか作れない表現で軌道を出せばいい」という発想が出てきます。それがベジェ曲線です。少数の制御点の重み付き和で曲線を定義するので、制御点が数個なら曲線は本質的になめらかになります。フィルタも平滑ペナルティも要りません。「滑らかさを後から足す」のではなく「最初から滑らかなものしか出せない」ように設計を変えるわけです。これは制約を「ソフト(ペナルティ)」から「ハード(表現自体に埋め込む)」へ移す、という設計思想の転換です。

## 基礎: 前提となる概念
ジャークと乗り心地。物理では、位置を時間で微分すると速度、速度を微分すると加速度、加速度を微分するとジャークになります。人間が不快に感じるのは主にジャーク(加速度の急変)です。エレベーターや電車が「ガクッ」とするのはジャークが大きい瞬間です。だから快適な運転とは、加速度がなめらかに変化する(ジャークが小さい)運転であり、プランナの出力もそうあってほしいのです。乗り心地の研究では、ジャークが概ね 0.9 m/s³ を超えると不快と感じられる、といった目安も使われます。

制御信号 vs 位置。軌道の表し方には大きく2通りあります。1つは各時刻の位置 `(x, y)` を直接出す方法。もう1つは「どれだけ加速し、どれだけ曲がるか」という制御信号(加速度・曲率)を出し、それを車両モデルに通して位置を計算する方法です。後者は、出力が物理的に実行可能(実際に車が出せる加減速・操舵に対応)であることを担保しやすく、滑らかさを効かせたい対象(指令の変化)に直接効かせられる利点があります。

ユニサイクル/自転車モデル。車の動きを簡単な数式で近似するモデルです。ユニサイクル(unicycle)モデルは、車を「位置と向きを持ち、前進速度と回転速度で動く点」とみなします。状態は `(x, y, θ)`(位置と向き)で、`ẋ = v cosθ, ẏ = v sinθ, θ̇ = v·κ`(κ は曲率)のように更新されます。自転車(bicycle)モデルは前後輪を考え、操舵角とホイールベースを使ってより現実に近づけます。プランナが出した制御信号(加速度・曲率)を、こうしたモデルの式に時間方向に積分(足し上げ)していくと、最終的な軌道 `(x, y)` が得られます。

凸包と凸結合。点の集合を含む最小の凸多角形を「凸包 (convex hull)」と呼びます。複数の点を「重みが全部非負で和が1」の重みで混ぜたものを「凸結合 (convex combination)」と呼び、これは必ず凸包の内側に入ります。ベジェ曲線の滑らかさ保証は、曲線上のどの点も制御点の凸結合になっている、という性質に由来します。

## 仕組みを詳しく
ベジェ曲線の核心は Bernstein 基底(バーンスタイン基底)です。n 次のベジェ曲線は、n+1 個の制御点 P_0 .. P_n を使って次の式で表されます。

```
曲線(t) = Σ_{i=0..n}  B_{i,n}(t) * P_i      (t は 0..1)
B_{i,n}(t) = C(n, i) * t^i * (1 - t)^(n - i)
```

`C(n,i)` は二項係数(n 個から i 個選ぶ組み合わせの数)です。`B_{i,n}(t)` が「時刻 t において、制御点 P_i をどれだけの重みで混ぜるか」を表す関数で、これが Bernstein 基底です。重要な性質は2つ。第一に、どの t でも全基底の和がちょうど1になること(これは二項定理 `(t + (1-t))^n = 1` から従う)。第二に、各基底が常に非負であること(t も 1-t も `[0,1]` で非負だから)。この2つから、曲線は常に制御点の「重み付き平均(凸結合)」であり、制御点の張る凸包の外に飛び出しません。これが滑らかさの保証の正体です。さらに Bernstein 多項式は無限回微分可能(C∞)なので、速度・加速度・ジャークといった微分量もすべて連続になります。ベジェ曲線の微分はまた1次低いベジェ曲線になる、という性質も計算上便利です。

実装は次のように動きます。例として、時間ステップ数 T=64、制御信号数 S=2(加速度・曲率)、制御点数 K=5(次数 n=4)とします。

1. 基底行列の事前計算。`[T, K] = [64, 5]` の行列を作ります。行が64の時間ステップ、列が5つの制御点。各要素 `B[t, i]` は「時刻 t/(T-1) における制御点 i の重み」です。標準ライブラリの組み合わせ計算関数だけで作れるので外部依存は不要です。この行列は学習しない固定値で、一度作れば毎回使い回せます。

2. 文脈ベクトルの作成。プランナは複数の入力を受け取り統合します。典型的には、自車の運動履歴(過去どう動いてきたか)、視覚履歴(過去のシーン特徴)、現在の BEV 特徴(周囲の空間表現)を、それぞれ同じ次元(たとえば256次元)に射影して足し合わせ、小さな全結合ネットに通して `[B, 256]` の文脈ベクトルにします。B はバッチサイズです。

3. 制御点の予測。線形層(制御ヘッド)が文脈から `K*S = 5*2 = 10` 個の値を出し、`[B, K, S] = [B, 5, 2]` に整形します。これが「加速度プロファイルの5制御点」と「曲率プロファイルの5制御点」です。ネットワークが直接予測するのはこのわずか10個の数だけ、という点が重要です。

4. 曲線への展開。基底行列と制御点の積で `[T, K] x [B, K, S] = [B, T, S] = [B, 64, 2]` を計算します。たった 5x2=10 個の制御点が、64x2=128 個の軌道値に滑らかに広がります。行列積1回で展開できるので計算も軽量です。

```
制御点 (5点)        Bernstein 展開        滑らかな軌道 (64ステップ)
P0 ●                                     ╱‾‾‾‾‾╲
   ╲       ──────────────────►          ╱       ╲___
P1  ●  P2 ●  P3 ●                       ╱            ‾‾‾
              ╲  P4 ●
```

ここで決定的に大事なのが出力の意味です。多くの実装では、出力は生の `(x, y)` 座標(地図上の位置)ではなく、車両ダイナミクス(ユニサイクル/自転車モデル)に渡す制御信号、すなわち `(加速度, 曲率)` のプロファイルです。だからベジェ平滑化は「加速度と曲率の時間変化」に効き、それがそのままジャーク低減・制御器に優しい・物理的に実行可能な指令につながります。位置の滑らかさではなく指令の滑らかさを保証している、という点を取り違えないことが重要です(位置だけ滑らかでも、それを実現する加減速が暴れていたら意味がありません)。

数値例で表現力を見てみましょう。制御点数 K を 3(次数2の放物線)に減らすと曲線はより単純(滑らかだが表現力は低下)になり、急なカーブや回避を表しにくくなります。逆に K を T(=64)まで増やすと各時間ステップが独立に動けてしまい、滑らかさの利点が消えて自己回帰と変わらなくなります。だから実装は通常 `2 <= K <= T` の範囲を強制し、K を「滑らかさと表現力のつまみ」として扱います。実用上は K=5〜10 程度がよく使われます。K=2 なら直線(等加速度)、K が増えるほど複雑な操作を表せる、というのが直感です。

## 手法の系譜と主要論文
軌道を多項式やスプライン(区分多項式)で表す発想は、ロボティクス・自動運転のプランニングで古くからの定番です。

古典的な経路計画。ベジェ曲線・B スプライン・クロソイド曲線(曲率が距離に比例して変化する曲線、道路設計で使われる)を使い、曲率連続な軌道を生成するアプローチが多数あります。González らのサーベイ(IEEE Transactions on Intelligent Transportation Systems, 2016)が、こうした幾何的曲線による経路計画を体系的に整理しています。狙いはジャーク最小化と実行可能性、トレードオフは制御点が少ないと鋭い回避が表現できないことです。これらは主に最適化(コスト関数の最小化)で制御点を決める非学習の手法でした。DARPA Urban Challenge(2007)に出た多くの車も、こうした多項式・スプライン軌道に依存していました。

学習ベースのエンドツーエンド軌道生成。深層学習の台頭で、シーン特徴から直接軌道を出す手法が増えました。UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)は BEV 特徴から検出・予測を経てプランニングまでを一貫して学習します。VAD(Vectorized Scene Representation for Efficient Autonomous Driving; Jiang et al., ICCV 2023, arXiv:2303.12077)は、シーンを点や折れ線のベクトル集合として効率良く表現し、軽量に軌道を出すことで速度と精度を両立させました。これらは「軌道の形を学習で獲得する」流れです。多くの学習ベース手法は軌道を単に時刻ごとの `(x, y)` 列として出しますが、そこにベジェ/スプラインのパラメータ化を組み込むと、出力次元が下がり滑らかさが保証されます。

多峰性を扱う生成モデル。1つの場面に対して「右に避けても左に避けても正解」のように複数のもっともらしい軌道が存在することを「多峰性 (multi-modality)」と呼びます。決定的に1本だけ出す方式はこれを表せません(平均を取ると「真ん中を突っ切る」危険な軌道になりうる、mode averaging 問題)。これに対し、拡散モデルやフローマッチングで軌道を生成する手法(Diffusion Policy; Chi et al., RSS 2023, arXiv:2303.04137 など)が登場しました。多峰の軌道分布を表せる強力さがある一方、反復サンプリング(ノイズから少しずつ軌道を作る繰り返し計算)が必要で推論が重いというトレードオフがあります。

この系譜を整理すると、自己回帰デコード(GRU)が「自由だが揺れうる」、拡散/フローが「多峰だが重い」、ベジェ/スプラインが「滑らかさ保証つきで軽量・決定的」という、設計上の三者三様のトレードオフが見えてきます。どれを選ぶかは、求める性質(滑らかさ・多峰性・速度)次第です。

## 論文の実験結果(定量データ)
プランニングの評価指標から説明します。代表的なのは L2 誤差(計画した軌道と人間の運転(正解)との位置のずれを、1秒後・2秒後・3秒後などで測ったメートル単位の距離、小さいほど良い)と衝突率(collision rate、計画軌道が他の車・歩行者とぶつかる割合、小さいほど良い)です。L2 が小さいほど人間に近い運転、衝突率が低いほど安全、と読みます。

オープンループ評価(記録済みデータに対して、もし自車がこう走ったらどうかを計算する評価)では、nuScenes 上で UniAD が平均 L2 誤差をおよそ 1.0m 前後まで下げ(1s/2s/3s の平均)、衝突率も従来のモジュール分割型より低減したと報告しました。VAD はこれをさらに軽量化し、ベクトル表現により推論速度を大きく上げつつ(報告では従来比でおよそ数倍速い、VAD-Tiny で 16 FPS 程度)、同等以上の L2・衝突率を達成したとされます。これは「重い密表現を使わなくても、効率的な表現と軌道生成で同等の運転品質が出せる」ことを示した点で重要です。

軌道の平滑性に直接関わる評価としては、ジャークや横加速度の統計が用いられます。ベジェ/スプラインで軌道を表す手法は、自己回帰で点ごとに出す手法に比べてジャークの分散が小さくなることが、経路計画研究で繰り返し示されています。これは指標というより設計の帰結で、「制御点が少なければ高周波の揺れが原理的に入らない」ためです。アブレーションとして制御点数を変えると、少なすぎると L2 誤差(正解への追従)が悪化し、多すぎると平滑性が失われる、という両側のトレードオフが観察されます。最良点は K を中程度(数個〜十数個)に取ったあたりに現れる、というのが経験則です。

なお、オープンループの L2・衝突率には既知の落とし穴があります。記録データでは多くの場面が「ほぼ直進」なので、単純に直進し続けるだけの素朴なベースラインでも見かけ上良い L2 が出てしまう、という指摘があります。Zhai らの研究(AD-MLP, arXiv:2305.10430)は、知覚を一切使わず自車の過去の状態だけを多層パーセプトロンに入れるだけで UniAD 級の L2 が出てしまうことを示し、オープンループ指標が知覚・推論能力をほとんど測れていない可能性を突きつけました。このため近年は、実際にシミュレータ内で運転させるクローズドループ評価(closed-loop、自車の行動が次の状況を変える評価)の重要性が強調されています。

## メリット・トレードオフ・限界
メリット。滑らかさ(低ジャーク)が数式で保証され、後段の平滑化フィルタも学習時の平滑ペナルティ調整も不要。少数の制御点を出すだけで多数の軌道値が決まるため、出力が低次元で学習が安定しやすく計算も軽い(出力次元が K×S と小さいので、過学習しにくく勾配も素直)。決定的(同じ入力なら同じ出力)で推論が1回で済み、拡散/フローのような反復サンプリングが不要。外部の数値計算ライブラリへの依存も小さく、基底は固定行列で実装が簡潔。制御点という解釈しやすいパラメータで軌道を表すので、デバッグや制約付け(制御点の範囲を限定するなど)もしやすい。

トレードオフと限界。制御点が少ないぶん、急な障害物回避のような鋭く複雑な操作は表現しにくく、表現力は制御点数で上下する。決定的なので、本質的に複数の正解がありうる場面の多峰性を表せない(そこは拡散/フローが補完候補)。出力が `(x, y)` でなく制御信号(加速度・曲率)の場合、可視化や評価では別途車両モデルで位置に積分する必要がある。さらに、ベジェ単体は「滑らかな軌道を作る」だけで、その軌道が安全か(障害物に当たらないか、車線を守るか)は別途コスト評価や制約で担保する必要があります。滑らかさと安全性は別の問題だ、という点は研究上の重要な注意です。また、グローバルな多項式表現(ベジェ)は局所的な変更が全体に波及するため、ある区間だけ鋭く曲げたい場合には B スプラインのような局所サポートを持つ表現の方が有利、という細かいトレードオフもあります。

## 発展トピック・研究の最前線
ハイブリッド表現。学習で「いくつかの候補モード」を出し、各モードをベジェ/スプラインで滑らかに表す、という組み合わせが研究されています。多峰性(複数の意図)と滑らかさ(各意図の中での快適さ)を両立させる狙いです。アンカー軌道(典型的な走り方のテンプレート)を分類で選び、その上にベジェの残差を載せる、といった設計も見られます。

微分可能な車両モデルとの結合。制御信号からベジェで軌道を作り、それを微分可能な車両ダイナミクスに通して位置を得る経路全体を一括で学習する研究があります。これにより「物理的に実行可能で、かつ人間らしい」軌道を端から端まで勾配で最適化できます。未来予測と組み合わせて行動条件つきに発展させる方向は [未来状態予測ヘッド (JEPA 的ワールドモデル分岐)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0025_future-state-jepa-head) と関連します。

クローズドループ評価への移行。前述の通りオープンループ L2 の限界が広く認識され、CARLA や nuPlan、最近の Bench2Drive といったシミュレータ上で実際に運転させる評価が主流になりつつあります。プランナの良し悪しは、最終的には「実際に走らせて事故らず快適に目的地に着けるか」(運転スコア、衝突回数、ルート完走率)で測るべき、という潮流です。この文脈では、滑らかで実行可能な指令を出すベジェ系の利点(制御器が追従しやすい)が、クローズドループでこそ効いてくる可能性が議論されています。
