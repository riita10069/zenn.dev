# Euler法によるODE推論サンプリング

## ひとことで言うと

ランダムなノイズから始めて、「いまどっちにどれだけ動くべきか」をモデルに何度か小刻みに聞きながら、少しずつノイズを本物のデータ(ここでは軌道)に変形していく推論方法です。微分方程式を小さなステップに分けて足し算で近似する Euler 法(オイラー法)を使い、たとえば10回程度の足し算でサンプルを作ります。拡散モデルやフローマッチングといった生成モデルの「本番でサンプルを作る」段階の中心となる手続きです。

## 直感的な理解

紙の上に置いた砂鉄を考えてください。磁石の周りには「この地点ではこの向きに引っ張られる」という力の矢印が全空間に広がっています。砂鉄を適当な位置に置き、矢印に沿って少しずつ動かしていくと、最終的に磁石の特定の場所に集まります。生成モデルの推論はこれにそっくりです。出発点はランダムなノイズ(適当な位置に置いた砂鉄)、矢印の場は学習済みモデルが返す「速度場」、ゴールは本物らしいデータです。

ただし矢印に沿って「滑らかに」動かすのは数学的に難しい(積分が必要)ので、実際には「現在地で矢印を1回聞いて、その向きに一定距離だけ進む」を何回も繰り返します。これが Euler 法です。1回あたりの距離を小さくして回数を増やすほど、本当の滑らかな軌跡に近づきますが、その分だけモデルを何度も呼ぶので遅くなります。この「精度 vs 速度」のバランスを取るのが推論ステップ数の設計です。

## 基礎: 前提となる概念

このトピックは生成モデルの「推論(本番でサンプルを作る)」段階の話です。学習段階の話は [外部ロス方式の学習API](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0959_external-loss-training-api) を参照してください。

速度場(velocity field)とは、「時刻 t、現在地 x のとき、どっちの方向にどれだけ進めばよいか」を返す関数です。学習済みのニューラルネット `v_theta(x, t, 条件)` がこれを近似します(theta はネットワークのパラメータ)。物理で言えば各地点に置かれた流れの矢印で、ノイズという出発点から本物のデータというゴールへ向かう流れを表します。「条件」には、自動運転なら周囲の BEV マップや過去の映像履歴・自車状態が入ります。

常微分方程式(ODE, Ordinary Differential Equation)とは、「ある量が時間とともにどう変化するか」を「変化率(微分)」で記述した式です。生成モデルの推論は次の ODE を解くことに相当します。

```
dx/dt = v_theta(x, t, 条件)
```

これは「時間 t を少し進めると、x が v_theta の分だけ動く」という意味です。t=0(ノイズ)から t=1(データ)まで解けば、出発点が終点に変わります。「解く」とは、初期値 x(t=0) から出発して各時刻の x を求めることです。

数値積分とは、ODE を厳密に解けないときに近似的に解く手法群です。最も単純なのが Euler 法(1次法)で、区間を小さく刻み、各区間では矢印が一定だと思って足していきます。より高精度な手法には Heun 法(2次)、Runge-Kutta 法(RK4、4次)などがあり、1ステップあたりの計算は増えますが同じ精度をより少ないステップで達成できます。

フローマッチング(Flow Matching)と拡散モデル(diffusion model)は、どちらも「ノイズからデータへの変換」を学習する生成モデルの枠組みです。拡散モデルは「データに少しずつノイズを足す過程」を逆向きにたどります。フローマッチングは「ノイズとデータを結ぶパス(経路)に沿った速度場」を直接学習します。両者は数学的に深く関係し、推論はどちらも上記の ODE(または確率微分方程式 SDE)を解く形に帰着します。

## 仕組みを詳しく

推論の処理を段階的に追います。軌道生成を例に、軌道次元を 128(64時刻×2信号)、推論ステップ数を10とします。

ステップ0: 条件を1回だけ準備して使い回す。ODE を解いている間、空間条件(BEV マップ)や運転コンテキスト(映像履歴・自車状態)は変わりません。変わるのは現在地 x と時刻 t だけです。そこで、BEV を平坦化・射影したキー・バリュー列や AdaLN 用の条件ベクトルを最初に1回だけ計算し、ループ内で使い回します。

```
mod_cond = 条件処理(映像履歴, 自車状態)   # 1回だけ
bev_seq  = BEV射影(bev_features)            # [B, H*W, C] を1回だけ
```

これがいわゆる「KV 列のステップ間再利用」です。BEV の平坦化と射影は数万マス分の行列演算なので、10回のステップで毎回やると10倍重くなります。1回に減らすことで ODE のループ本体(速度場 `v_theta` の呼び出し)を軽くします。再利用が成立するのは、条件が時刻 t に依存しないからです。t に依存するのは速度場の評価部分だけなので、そこだけをループで回します。

ステップ1: ノイズから開始。出発点 x を標準正規分布(平均0・分散1のガウスノイズ)からサンプリングします。形状は `[B, 128]`。

```
x = randn(B, 128, generator=generator)
```

`generator`(乱数生成器)を渡せる実装が多く、同じシードを与えると毎回まったく同じノイズから始まり、同じサンプルが出ます。評価の再現性(同じ入力で同じ結果)を保証するためです。

ステップ2: Euler 法で num_inference_steps 回だけ積分。区間 [0,1] を10等分し、刻み幅 `dt = 1/10 = 0.1` とします。各ステップで「現在の (x, t) における速度 v を聞き、x に dt×v を足す」を繰り返します。

```
dt = 1.0 / 10                         # 0.1
for step in range(10):                # 0..9
    t_val = step * dt                 # 0.0, 0.1, ..., 0.9
    v = v_theta(x, t_val, bev_seq, mod_cond)   # 現在地の矢印
    x = x + dt * v                    # 矢印の向きへ dt だけ進む
```

数値例で追います。ステップ0では t=0.0(完全なノイズ)で速度 v0 を聞き、`x ← x + 0.1*v0`。ステップ1では t=0.1 の x で速度 v1 を聞き、また 0.1 進む。これを10回繰り返すと t は 0.9 まで進み、x は本物らしい軌道に近づきます。最終的な x が出力 `[B, 128]` です。

```
t:   0.0   0.1   0.2   ...   0.9      (→ 1.0)
x:  noise  →  →  →  ...  →  trajectory
     |__dt__|__dt__|         各ステップで v_theta を1回評価
```

Euler 法の直感的なイメージは「滑り台を10段の階段で近似する」ことです。本当は連続的な滑らかな坂(真の ODE 解)ですが、10個の踏み板(ステップ)で代用します。段数が多いほど本物の坂に近づき(精度向上)、段数が少ないほど速いがガタつく(離散化誤差増)というトレードオフがあります。なぜ Euler 法を選ぶかというと、実装が足し算の繰り返しだけで単純、デバッグしやすく、後述する直線パスとの相性が良いからです。

## 手法の系譜と主要論文

「ノイズから ODE/SDE を解いてサンプルを得る」という考え方は、拡散モデルとフローマッチングの両系譜から発展しました。

DDPM(Ho et al., 2020, NeurIPS, arXiv:2006.11239)が拡散モデルを実用レベルに引き上げましたが、サンプリングに1000ステップ程度を要し非常に遅いのが弱点でした。

DDIM(Song et al., 2021, ICLR, arXiv:2010.02502)は、確率的なサンプリングを決定的な ODE 的サンプリングに変え、ステップ数を大幅に減らしても品質を保てることを示しました。決定的少ステップ推論の有用性を示した重要な先行研究です。

Score-based SDE(Song et al., 2021, ICLR, arXiv:2011.13456)は、拡散モデルが連続時間の確率微分方程式(SDE)とその対応する確率フロー ODE で統一的に書けることを示し、ODE ソルバーで効率的にサンプリングする道を開きました。

Flow Matching for Generative Modeling(Lipman et al., 2023, ICLR, arXiv:2210.02747)は、ノイズとデータを結ぶパス(特に最適輸送に基づく線形パス)に沿った速度場を、単純な回帰で学習する枠組みを提案しました。推論時に ODE を Euler 法などで解いてサンプルを得ます。本トピックの「速度場を Euler 積分」はこの枠組みそのものです。

Rectified Flow(Liu et al., 2023, ICLR, arXiv:2209.03003)は、ノイズと目標を直線で結ぶパスにすると ODE の軌跡が直線に近づき、少ないステップ(極端には1ステップ)で高品質に生成できることを示しました。直線パスを使う場合、Euler 法の少ステップ推論と相性が良い理由がここにあります。

高速ソルバーの系譜として、DPM-Solver(Lu et al., 2022, NeurIPS, arXiv:2206.00927)は拡散 ODE の半解析的な構造を利用し、10〜20ステップで高品質生成を達成しました。EDM(Karras et al., 2022, NeurIPS, arXiv:2206.00364)は2次の Heun 法を採用しノイズスケジュールを最適化することで、35ステップ前後で当時最良の生成品質を出しました。これらは「Euler より高次のソルバーで同じ品質をより少ないステップで」という方向の代表です。

## 論文の実験結果(定量データ)

ステップ数と品質・速度のトレードオフは、これらの論文で定量化されています。指標の意味も添えます。

画像生成では FID(Fréchet Inception Distance、生成画像と本物の特徴分布の距離。低いほど本物に近い)が主指標です。DDIM は CIFAR-10 で、1000ステップの DDPM がステップを50に減らすと FID が大きく悪化(品質劣化)するのに対し、DDIM では50〜100ステップでも FID をほぼ保てることを示しました。たとえば DDIM は20ステップ程度でも実用的な FID(おおむね一桁台後半〜十数)に到達し、DDPM の同ステップ数より明確に良いと報告されています。

Rectified Flow は CIFAR-10 で、直線化(reflow)した後に Euler 1ステップでも FID がおよそ4台に達することを報告しました。これは「1回モデルを呼ぶだけで高品質サンプルを出せる」ことを意味し、直線パスの強力さを示します。ステップ数を増やせばさらに改善しますが、改善幅は逓減します。

DPM-Solver は、安定拡散などの大規模拡散モデルで、従来50ステップ必要だった品質を10〜20ステップで達成できることを示しました。EDM は ImageNet 64×64 で、Heun 法ベースのサンプラーにより35ステップ前後で FID 1.36(当時の最良級)を達成し、Euler 1次法より少ないモデル評価回数で高品質に到達することを示しました。これらのアブレーションの共通点は、「ソルバーの次数を上げる」「ノイズスケジュールを最適化する」とステップ数を削減できる一方、1ステップあたりの計算は増える、というトレードオフです。

数値の意味を補足します。FID の差が「2.27 と 4.5」のように一見小さく見えても、生成品質では人間が知覚できる差になり得ます。またステップ数はそのまま推論レイテンシ(遅延)に比例します。リアルタイム制御(自動運転の軌道生成など)では、1回の推論で許される時間が数十ミリ秒のオーダーなので、10ステップか20ステップかが実機で動くかどうかを分けます。直線パス+少ステップ推論が自動運転で好まれるのはこのためです。

## メリット・トレードオフ・限界

メリット。Euler 法は実装が足し算の繰り返しで単純、理解とデバッグが容易です。条件(BEV・履歴)を1回だけ計算して使い回すため、ステップを増やしてもコストの増分は速度場の評価だけで済みます。乱数生成器により決定的で再現可能な推論ができ、評価の比較が公平になります。直線パス(Rectified Flow / 線形フローマッチング)と組み合わせると、10ステップ程度の少ステップでも実用品質が出ます。

トレードオフと限界。Euler は1次精度なので、ステップ数が少なすぎると離散化誤差が出ます。誤差はおおむねステップ幅に比例(1次)するので、半分にしても誤差は半分にしかなりません(高次法なら2乗・4乗で減る)。ステップ数を増やせば精度は上がりますが、速度場の評価回数が増え推論が遅くなります。リアルタイム制御では1ステップの遅延が効くので段数選びが重要です。高次解法(Heun, RK4, DPM-Solver, EDM)に比べ同じ段数では精度が劣るため、精度が要るならソルバーの差し替え余地があります。

未解決・実務上の課題。何ステップが最適かはパス(直線か曲線か)・学習の質・タスクに依存し、理論だけでは決められず実験的に探す必要があります。また、決定的な ODE サンプリングは多様性(同じ条件から異なるサンプルを出す能力)が SDE サンプリングより落ちることがあり、軌道生成で複数の妥当な経路を出したい場合に課題となります。SDE 項を足して多様性を回復する研究や、少数ステップでも多様性を保つ蒸留手法(consistency models など)が現在の最前線です。

## 発展トピック・研究の最前線

最近は「1〜数ステップで高品質」を狙う蒸留系が活発です。Consistency Models(Song et al., 2023, arXiv:2303.01469)は ODE 軌跡上の任意の点を直接ゴールへ写す関数を学習し、1ステップ生成を可能にしました。Progressive Distillation や Rectified Flow の reflow も同方向です。自動運転では、軌道生成の多様性(複数の妥当な未来)と少ステップ高速推論を両立させる研究、そして物理的に実現可能な軌道だけを生成するための制約付きサンプリング(ガイダンス)が課題として残っています。ソルバー側では、適応的ステップ幅(誤差を見て刻みを変える)や、学習でソルバー自体を最適化する手法も探索されています。
