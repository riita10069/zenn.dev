# Flow Matching の時刻サンプリング戦略

## ひとことで言うと

Flow Matching(フローマッチング、ノイズからデータへ連続的に変形する生成モデルの一種)の学習では、毎ステップ「変形プロセスのどの時点 t を練習するか」を1つランダムに選びます。この t をどう選ぶかが学習の効率と最終品質を左右します。t を一様(uniform、0から1まで均等)に選ぶのが教科書的な出発点ですが、近年の研究は「難しい時点を多めに、簡単な時点を少なめに練習する」よう t の分布を意図的に歪める(bias をかける)ほうが速く安定して学習できると報告しています。この時刻サンプリング戦略を扱うのがこのノートです。

## 直感的な理解

料理の練習にたとえます。材料を切る工程と、盛り付ける工程があるとして、もし両方を同じ回数だけ練習したら、難しい「切る」工程の上達が遅れます。難しい工程を重点的に練習したほうが全体の腕は早く上がる。Flow Matching の学習でも同じことが起きます。

Flow Matching は、純粋なノイズ x_0(意味のない乱数)と本物のデータ x_1(たとえば車の走行軌跡や画像)を経路で結び、その途中の1点を取り出してネットワークに「ここから x_1 へ向かう速度(進む向きと大きさ)」を当てさせます。時刻 t はこの経路上の位置で、t=0 がノイズ端、t=1 がデータ端です。経路が直線の場合、時刻 t での中間点は単純に `x_t = (1−t)·x_0 + t·x_1` と書け、その点で当てるべき真の速度は `x_1 − x_0` という定ベクトルになります。

t=1 付近(データ端)では、入力はほぼ完成品なので「あと少しでゴール」という速度を当てるのは比較的やさしい。一方 t=0 付近(ノイズ端)では、入力はほぼ無意味な乱数で、そこから「正解はどっちか」を当てるのは非常に難しい。素朴に t を一様に選ぶと、簡単な領域も難しい領域も同じ頻度で練習することになり、肝心の難所の練習量が不足します。だから難所を重点的に引く分布を設計する、というのが基本的な発想です。

ただし「どこが難所か」は経路設計とタスクに依存します。直線経路の画像生成では中間時刻(t≈0.5)が最も予測が難しいとされ、ロボットや自動運転の連続軌跡生成ではノイズ側(t≈0)が難しいとされます。後者では「ノイズ混じりの初期軌跡から、まず大局的に正しい方向を決める」段階が最も誤りやすく、ここを重点的に練習する価値が高いと経験的に報告されています。

## 基礎: 前提となる概念

このノートを理解するには4つの前提が要ります。

第一に「確率分布から値を引く(サンプリング)」とは何か。コインを投げると表か裏が出るように、ある分布を決めるとそれに従って値がばらつきます。一様分布 U(0,1) は0から1までどの値も等しく出やすい分布です。

第二に「ベータ分布(Beta distribution)」。これは0から1の範囲の値を出す分布で、2つの形状パラメータ α(アルファ)と β(ベータ)で形が決まります。確率密度は x^(α−1)·(1−x)^(β−1) に比例します。α=β=1 なら一様分布になります。α>1, β=1 にすると右肩上がり(1に近い値が出やすい)、α=1, β>1 なら左肩上がり(0に近い値が出やすい)になります。たとえば Beta(1.5, 1.0) は密度が x^0.5 に比例し、1に近い大きな値が出やすい右肩上がりの形です。期待値は α/(α+β) なので Beta(1.5,1.0) の平均は 1.5/2.5=0.6 です。

第三に「ロジット正規分布(logit-normal)」。これは「t をロジット変換 logit(t)=log(t/(1−t)) で実数全体に写し、そこに正規分布 N(m, s²) を当てて t に戻す」分布です。平均 m を0にすると山が0.5付近に集まり、中央重点(両端を薄く、真ん中を厚く)の形になります。m を負にすればノイズ側へ、正にすればデータ側へ山がずれます。後述する Stable Diffusion 3 が採用した分布です。

第四に「損失の重み付けと時刻分布の等価性」。学習損失は「時刻ごとの誤差を時刻分布で平均したもの」です。ある時刻 t を2倍の頻度で引くことと、その t の損失に2倍の重みを掛けることは、勾配の期待値の上では同じ効果になります。つまり「t の分布を歪める」操作と「損失に重みを掛ける」操作は数学的に等価で、流儀の違いにすぎません。

これらはどれも「U(0,1) を何かの関数で歪めて、特定の領域に確率を集める」道具だと思えば十分です。

## 仕組みを詳しく

時刻サンプリングの実装は、典型的には「モード切り替え式」になります。一様(ベースライン)と歪み分布(既定)を選べるようにし、歪みの強さをパラメータで調整できるようにします。

一様モード(教科書どおり、比較用):
```
t = rand(B)           # [B], 0以上1未満の一様乱数
```
バッチサイズ B のサンプルそれぞれに独立な t を1つずつ割り当てます。

ベータモード(ノイズ側重点):
```
b = Beta(alpha=1.5, beta=1.0).sample(B)   # [B], 1寄りに偏る
t = scale - scale * b                      # scale は 0.999 など 1 弱
```
段階を分解します。

1. Beta(1.5, 1.0) から b を引く。b は1に近い大きな値が出やすい(平均0.6)。
2. `t = scale·(1 − b)`。b が1寄り(大きい)なら t は0寄り(小さい)になる。つまり t がノイズ側に偏る。
3. `scale=0.999` を掛けるのは、t がちょうど 1.0(完成品そのもの、速度を学ぶ意味が薄い点)や数値的に厄介な端へ張り付かないよう、わずかに内側へ収める安全策。

数値例。Beta(1.5,1) から100個引くと多くが 0.6〜0.99 に集まります。b=0.9 なら t = 0.999×0.1 = 0.0999(ノイズ寄り)、b=0.4 なら t = 0.999×0.6 = 0.599。結果として t の集まりはノイズ端に厚く、データ端に薄くなります。

```
t の頻度（イメージ）
ノイズ端 t=0                          データ端 t=1
 ████████████▆▆▅▅▄▄▃▃▂▂▁▁           ← Beta(1.5,1) 経由（ノイズ側重点）
 ────────────────────────           ← 一様 U(0,1)（平坦）
        ▁▂▃▅███▅▃▂▁                  ← logit-normal(m=0)（中央重点）
```

α を大きくするほど偏りが強まり、α=1.0 にすると Beta(1,1)=一様に戻ります。これにより「偏りの強さ」を連続的に調整でき、極端には一様(ベースライン)へオプトアウトできます。この「ダイヤルでベースラインへ戻せる」設計は、改善が本当にサンプリング由来かを切り分ける実験設計の上で重要です。

重要な注意として、この時刻サンプラが効くのは学習時だけです。推論([Euler 法による ODE 推論（Flow Matching のサンプリング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0120_euler-ode-inference))では t を 0, 0.1, 0.2 … と等間隔に進めて ODE を解くので、このサンプラは関与しません。学習中に「どの t を多く練習したか」が、間接的に各時点の速度予測の正確さに反映され、推論品質に効いてきます。逆に言えば、推論で実際に通過する時刻(等間隔の各点)が学習で十分練習されている必要があり、偏りを強くしすぎて薄くなった領域は推論時に誤差源になります。

## 手法の系譜と主要論文

時刻サンプリングは Flow Matching/Diffusion の研究が進むにつれ「軽視されてきたが実は重要なハイパーパラメータ」として再評価された経緯があります。

出発点は Lipman, Chen, Ben-Hamu, Nickel, Le(Flow Matching for Generative Modeling, ICLR 2023, arXiv:2210.02747)です。ここでは t を U(0,1) で一様に取る前提で定式化されました。理論的には連続正規化フロー(continuous normalizing flow、ODE で表す生成モデル)の目的を、シミュレーション不要(simulation-free)で学習できる形に書き換えたもので、t の分布は「重み」として目的関数に現れますが、初期の議論では一様が標準でした。ほぼ同時期の Liu, Gong, Liu の Rectified Flow(ICLR 2023, arXiv:2209.03003)も直線経路の Flow を提案し、こちらも基本は一様サンプリングです。

次に Diffusion 側からの知見が合流します。Ho, Jain, Abbeel の DDPM(2020)やその後の研究で「ノイズが多い段階ほど学習が難しく勾配のスケールが揃わない」ことが分かり、Karras, Aittala, Aila, Laine(EDM, NeurIPS 2022, arXiv:2206.00364)が「ノイズレベルごとに損失の重みを変える」「ノイズレベルの学習分布を対数正規にする」設計を体系化しました。EDM はノイズレベル σ の学習分布を `ln σ ~ N(P_mean, P_std²)`(報告では P_mean=−1.2, P_std=1.2)とし、これは時刻分布を歪めることと等価な操作です。

決定打のひとつが Esser ら(Scaling Rectified Flow Transformers for High-Resolution Image Synthesis、通称 Stable Diffusion 3、ICML 2024、arXiv:2403.03206)です。彼らは Rectified Flow で複数の時刻サンプリング方式(一様、logit-normal、mode-shifted、cosmap など)を大規模に比較し、logit-normal(中央重点)が一様より一貫して良いと報告しました。直線経路では中間時刻が最も予測が難しく、ここを重点的に学習する効果だと説明されます。

ロボット制御・自動運転側では、Black ら(π0: A Vision-Language-Action Flow Model for General Robot Control, 2024, arXiv:2410.24164)が Flow Matching 方策(連続的な行動列を生成する方策)で、t をノイズ側に偏らせる分布(報告では Beta 系の分布)を採用し学習が安定すると述べました。自動運転の連続軌跡生成でも同様にノイズ側重点のサンプリングが使われており、低レイテンシ推論と滑らかな軌跡を両立する一要素とされます。ベータ分布によるノイズ側重点はこの系譜にあたります。

要約すると、一様(Lipman 2023 / Liu 2023)→ 損失重み付け・ノイズレベル対数正規(EDM 2022)→ 中央重点 logit-normal(SD3 2024)/ ノイズ側重点 beta(π0 系)、という流れで「t の分布をタスクに合わせて設計する」のが当たり前になりました。

## 論文の実験結果(定量データ)

SD3(Esser et al., 2024)の比較が最も体系的です。彼らは ImageNet と CC12M で複数のサンプリングスケジュールを比較しました。評価指標は主に FID(Fréchet Inception Distance、生成画像の特徴分布が本物の特徴分布にどれだけ近いかを測る距離。小さいほど良い)と CLIP スコア(画像とテキストの一致度、大きいほど良い)です。報告によると、logit-normal(平均 m=0 付近、標準偏差 s=1 付近)が一様サンプリングに対して FID を一貫して改善し、複数解像度・複数データセットで最良かそれに近い成績でした。差は数値そのものよりも「どのスケジュールでも一様が最良になることはほぼなかった」という頑健な傾向が重要です。これは「t の分布設計は無視できない」という主張の実証です。

EDM(Karras et al., 2022)は CIFAR-10 と ImageNet で、ノイズレベルの分布と損失重みを最適化することで当時の最良 FID(CIFAR-10 で条件付き約1.79、無条件約1.97 と報告)を達成しました。これは時刻(ノイズレベル)の扱い方だけで生成品質が大きく動くことを示した代表例です。ノイズレベル分布の中心 P_mean を変えるだけで FID が目に見えて上下することが、論文中のグリッド探索で示されています。

アブレーション(どの要素を抜くとどうなるか)の観点では、共通して次が観察されます。歪み分布を一様に戻すと、難所(直線経路なら中間、ロボット軌跡ならノイズ側)の予測誤差が相対的に大きくなり、同じ計算量での収束が遅くなる。逆に偏りを強くしすぎる(α を大きくしすぎる、または logit-normal の分散を小さくしすぎる)と、薄く練習した領域の仕上げ精度が落ち、推論時の該当ステップでわずかな誤差が残りやすくなります。つまり最適な偏りには「山」があり、タスクごとにチューニングが要ります。SD3 のスケジュール比較表でも、極端に中央に集中させた設定は中庸な logit-normal に劣る傾向が見られます。

## メリット・トレードオフ・限界

メリット:
- 学習しにくい領域を重点的に練習でき、同じ計算量での収束が速く安定しやすい。
- パラメータ(α や分布の平均・分散)で偏りの強さを連続的に調整でき、極端には一様へ戻せるので効果を比較検証しやすい。
- 学習可能パラメータを増やさず、サンプリングを変えるだけで効く(追加コストがほぼゼロの改善)。

トレードオフ・限界:
- 偏りを強くしすぎると薄く練習した領域の精度が落ちる。最適点はタスク依存で、探索コストがかかる。
- 「どの時点が難しいか」はモデル・データ・経路設計(直線か曲線か)で変わる。直線経路は中間が難しく、ロボット軌跡ではノイズ側が難しいなど、定石が一意でない。
- 学習時専用で、推論品質を直接は決めない(推論はソルバとステップ数で決まる)。サンプリングの恩恵は間接的で、推論で通過する時刻が薄く練習されていると逆効果になりうる。

未解決の課題として、時刻分布を学習中に適応的に動かす(難所が学習の進行で移動するのに追従する)手法はまだ発展途上です。固定の Beta や logit-normal は静的な近似にすぎません。学習初期はノイズ側が難しくても、収束が進むと難所が移動する現象が観察されており、これに追従する分布設計は研究途上です。

## 発展トピック・研究の最前線

- 適応的時刻サンプリング: 学習の進行に応じて損失が大きい時点を自動で多く引く、能動学習的な手法。各時刻の損失移動平均を保持し、それに比例して引く重要度サンプリング(importance sampling)で勾配の分散を減らす研究が進んでいます。
- 損失重み付けとの統一: 前述のとおり時刻分布を歪めることと損失に重みを掛けることは数学的に等価です。EDM 流の重み設計と Flow 流の分布設計を橋渡しし、SNR(信号対雑音比)を軸に統一的に記述する理論整理が続いています。
- 経路設計との結合: Rectified Flow の reflow(直線化の再学習)と組み合わせると、難所の位置自体が変わるため、サンプリング戦略も連動して見直す必要があります。
- ロボット・自動運転への転用: 連続行動・連続軌跡の生成では「安全側で確実に正しい初動を出す」ことが重要なため、ノイズ側重点が有効という経験則が共有されつつあります。理論的な裏付けは画像生成ほど成熟しておらず、タスク依存の実験的探索が中心です。
