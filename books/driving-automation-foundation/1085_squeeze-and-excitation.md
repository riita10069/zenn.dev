# Squeeze-and-Excitation (SENet)

## ひとことで言うと
Squeeze-and-Excitation(略して SE)は、ニューラルネットの特徴マップに対して「どのチャネル(特徴の層)が今このシーンで重要か」をネットワーク自身に判断させ、重要なチャネルを強め、不要なチャネルを弱める小さな部品です。特徴の音量つまみを、入力内容に応じて自動で上げ下げするようなもので、既存の CNN にわずかに足すだけで精度が上がるため、数えきれないバックボーンに後付けされてきました。チャネル方向の「注意 (attention)」を初めて実用的に広めたモジュールであり、現代の注意機構の出発点に位置づけられます。

## 直感的な理解
CNN は画像を処理する途中で、たくさんのチャネル(特徴の地図)を作ります。あるチャネルは縦線に、別のチャネルは肌色に、また別のチャネルは路面のテクスチャに反応する、という具合です。問題は、普通の畳み込みがこれらすべてのチャネルを対等に扱ってしまう点にあります。畳み込みのフィルタは学習が終わると固定で、入力画像が何であろうと同じ重みで反応します。

しかし実際には、シーンによって重要なチャネルは変わります。歩行者を見つけたい今このフレームでは、人の輪郭や肌色に反応するチャネルが重要で、空の青や路面の細かい模様に反応するチャネルは相対的に重要度が低いはずです。逆に、雨で滑りやすい路面を判断したいフレームでは、路面テクスチャのチャネルこそが主役になります。

人間は無意識に「今は何に注目すべきか」を切り替えています。SE はこの切り替えをネットワークに持たせる仕組みです。入力の内容を見て、チャネルごとの重要度(0〜1 の重み)をその場で計算し、特徴に掛けます。掛け算で重みづけするだけなので追加計算は非常に小さく、それでいて精度が目立って上がるため、コストパフォーマンスの良い改良として一気に広まりました。固定の畳み込みフィルタに「入力依存の動的なゲート」を一段足した、と言い換えられます。

## 基礎: 前提となる概念
- チャネル (channel): 特徴マップの「層の枚数」。各チャネルは「縦線」「赤い色」「タイヤらしい丸」など、特定パターンへの反応の地図 1 枚です。CNN は層が深くなるほどチャネル数が増え、抽象的な特徴を持ちます。
- アテンション (attention, 注意): ネットワークが「全部を均等に見るのではなく、大事なところに重みを置いて見る」仕組みの総称です。重みは入力に応じて動的に変わるのが本質で、固定の重み(普通の畳み込みフィルタ)とはここが違います。
- グローバル平均プーリング (global average pooling, GAP): 1 枚のチャネル地図(H×W の数値)全体の平均を取り、1 個の数値に潰す操作です。「このチャネルが画像全体としてどれくらい活躍したか」という大域的な手がかりを取り出します。
- 全結合層 (fully connected layer, FC): 入力のすべての要素を重み付きで足し合わせて出力を作る、最も基本的なニューラルネットの層です。ここでチャネル間の関係を学習します。
- 非線形性 (non-linearity): ReLU などの関数で、入力と出力が単純な比例関係にならないようにする部品。これがないと層を何枚重ねても 1 枚の線形変換と等価になり、複雑な関係を表せません。
- Sigmoid 関数: 任意の実数を 0〜1 に押し込む S 字の関数です。SE では最終的なチャネル重みを 0〜1 に収めるために使います。

普通の畳み込みは、出力チャネルを生成するとき各チャネルを対等に作り、シーンに応じて重要度を調整する仕組みを持ちません。SE はここに「チャネルごとの重要度を、入力内容に応じて動的に計算して掛ける」段を足します。注意したいのは、SE が新しい特徴を作るわけではない点です。あくまで既にある特徴の音量を調整する「再重み付け」であり、表現の量を増やすのではなく、表現の使い方を賢くする部品だと理解してください。

## 仕組みを詳しく
SE モジュールは 2 ステップ、名前のとおり Squeeze(絞る)と Excitation(励起する=重みを生成して掛ける)です。入力特徴マップを「チャネル C × 縦 H × 横 W」とします。

### 1. Squeeze(絞る): グローバル平均プーリング
各チャネルについて、その 1 枚の地図(H×W の数値)の平均を取り、1 個の数値に圧縮します。これでチャネルごとの「全体的にどれくらい強く反応したか」を表す C 個の数値ベクトルになります。

```
入力:  [C, H, W]   例 [256, 28, 28]
↓ チャネルごとに H×W を平均
出力:  [C]         例 [256]   (各チャネルの平均反応値)
```

なぜ平均で潰すのか。チャネルアテンションが知りたいのは「このチャネルが画像全体としてどれくらい活躍したか」という大域的な手がかりであり、空間の細かい位置は不要だからです。場所の情報は捨て、チャネルの存在感だけを残します。これにより、受容野(一度に見られる範囲)が局所に限られる畳み込みでは得にくい「画像全体の文脈(global context)」を、各チャネルに与えられます。畳み込みは局所しか見られないという根本的な制約を、たった 1 回の平均操作で補っているのがこのステップの妙味です。

### 2. Excitation(励起): 小さな全結合層 2 枚で重みを作る
この C 個のベクトルを、小さなニューラルネット(全結合層 2 枚 + 非線形)に通して、各チャネルの重み(0〜1)を作ります。

```
[C] → FC(C → C/r) → ReLU → FC(C/r → C) → Sigmoid → [C] の重み(各値 0〜1)
```

途中で C/r に絞る r を reduction ratio(削減比)と呼び、論文の標準値は 16 です。一旦次元を絞ることには 2 つの意味があります。1 つはチャネル間の関係を少ないパラメータで学習し計算量を抑えること、もう 1 つはボトルネックを作ることで「どのチャネルどうしが連動するか」という相互依存(channel interdependency)を圧縮表現として学ばせることです。間の ReLU は非線形性を入れて単純な線形写像以上の関係を表せるようにし、最後の Sigmoid で各チャネルの重みを 0〜1 に収めます。ここで Softmax(合計を 1 にする)ではなく Sigmoid を使うのは重要で、複数のチャネルが同時に高くてよい(互いに排他ではない)ため、各チャネルを独立に 0〜1 でゲートする Sigmoid が適切なのです。

### 3. 掛ける(Scale)
作った C 個の重みを、元の特徴マップの対応するチャネルに掛けます。重み 0.9 のチャネルはほぼそのまま、重み 0.1 のチャネルはほとんど消える、という具合です。

```
出力チャネル i = 入力チャネル i × 重み i   (各チャネルに同じ重みを全画素へ)
```

これがチャネル方向のアテンションです。空間(どこ)ではなく、チャネル(どの種類の特徴)に注意を向ける点が特徴で、空間にも注意する [CBAM (チャネル+空間アテンション)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1058_cbam-attention) と対比されます。

追加計算は、グローバル平均と小さな全結合 2 枚だけです。パラメータ増加量は約 2·C²/r で、ネットワーク全体で見ると ResNet-50 に足してもパラメータ増は約 +2.5%、計算量(FLOPs)増は 1% 未満に収まります。これほど安く効くため、後述のとおり EfficientNet や MobileNetV3 など多くの近代バックボーンが SE を標準部品として内蔵しています。

## 手法の系譜と主要論文

- Squeeze-and-Excitation Networks (Hu et al., CVPR 2018, 拡張版 TPAMI 2020, Momenta / オックスフォード大学, arXiv:1709.01507)。チャネル間の相互依存を明示的にモデル化し、入力に応じてチャネルを再重み付けする SE ブロックを提案しました。動機は、従来の畳み込みがチャネルを対等に扱い、文脈に応じた強調ができなかったことです。SENet(SE を組み込んだネットワーク)は ILSVRC 2017(ImageNet 大規模画像認識コンペティション)の分類部門で優勝し、注意機構を分類バックボーンに持ち込む流れを決定づけました。

- BAM / CBAM (Park et al. BMVC 2018 / Woo et al. ECCV 2018)。SE の弱点である「空間情報を捨てる」点を補い、チャネルアテンションに加えて空間アテンション(どこを強調するか)を導入しました(詳細は [CBAM (チャネル+空間アテンション)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1058_cbam-attention))。CBAM は SE の直接の発展形で、チャネル注意の段で平均プーリングだけでなく最大プーリングも併用する点で SE を拡張しています。

- MobileNetV3 (Howard et al., ICCV 2019) と EfficientNet (Tan & Le, ICML 2019) と GhostNet (Han et al., CVPR 2020)。いずれも SE を一部または全ブロックに組み込み、軽量・効率モデルでも精度を底上げしました(詳細は [MobileNet (V1/V2/V3)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0397_mobilenet) [GhostNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0386_ghostnet))。SE が「安く効く後付け部品」として広く受け入れられ、NAS で探索されるアーキテクチャの標準構成要素にまでなった証拠です。

- ECA-Net: Efficient Channel Attention (Wang et al., CVPR 2020, arXiv:1910.03151)。SE の全結合 2 枚による次元削減(C → C/r → C)が実はチャネルアテンションには逆効果だと指摘し、削減をやめて 1 次元畳み込み(カーネルサイズ k の局所的な重み共有)で隣接チャネルだけの関係を捉える、より軽く高精度な代替を提案しました。パラメータをほぼ増やさずに SE を上回る精度を出し、SE の設計を問い直した重要な後続です。

系譜としては、SE(チャネル注意)を起点に、CBAM/BAM(空間も追加)へ広がる方向と、ECA-Net(SE 自体をより効率化)へ深める方向の二つに分岐しました。さらに Transformer の自己注意([PVT (Pyramid Vision Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0402_pvt-pyramid-vit) など)は「トークン間の動的な重み付け」という意味で SE と同じ思想の延長線上にあり、SE は CNN 時代から注意機構時代への橋渡しをした歴史的部品と位置づけられます。

## 論文の実験結果(定量データ)
評価は ImageNet 分類(ILSVRC-2012、1000 クラス)で、指標は top-1 / top-5 誤り率(エラー率。小さいほど良い)です。報告された代表値の目安は次のとおりです。

```
モデル                top-1 誤り率   備考
ResNet-50            約 23.3%       ベースライン
SE-ResNet-50         約 22.3〜21.1% SE 追加で約 0.8〜1.5 ポイント改善
ResNet-101           約 22.0%       より深く重いモデル
SE-ResNet-50 (再学習) ≈ ResNet-101  浅いまま深いモデルに匹敵
ResNeXt-50           約 22.2%
SE-ResNeXt-50        約 21.1%       SE 追加で約 1.1 ポイント改善
```

ここを初心者向けに読み解きます。top-1 誤り率が約 23.3% から約 22.3% に下がる、という約 1 ポイントの改善は、ImageNet では「ほぼ無料の追加(パラメータ +2.5%、FLOPs +1% 未満)」で得られたことを考えると非常に費用対効果が高い結果です。特筆すべきは、SE-ResNet-50 が、層数が倍近い ResNet-101 に匹敵する精度を出した点で、これは「深くして計算を増やす」代わりに「チャネル注意を足す」という別の効率的な精度向上ルートがあることを示しました。Inception 系や ResNeXt など他のバックボーンに足しても一貫して改善が見られ、SE がアーキテクチャを選ばない汎用部品であることが確認されています。検出(MS COCO)でも、SE 付きバックボーンに替えると mAP が約 1〜2 ポイント向上したと報告されています。

アブレーション(要素を抜く検証)で報告された知見:
- reduction ratio r の影響: r=16 が精度と計算量の良いバランス点。r を小さくする(例 4 や 8)と精度はわずかに上がるがパラメータが増え、大きくする(例 32)と逆。精度は r にそれほど敏感ではなく、安定して効くと報告されています。
- Squeeze の方法: グローバル平均プーリングと最大プーリングを比べると、平均の方が安定して良い(CBAM は両方を併用してさらに改善)。
- Excitation の活性化: 最後を Sigmoid から ReLU や tanh に変えると性能が落ち、0〜1 のゲートにする Sigmoid が適切でした。
- SE をどの位置に入れるか: 残差ブロックの「中(残差を足す前)」に入れる構成が有効で、ネットワークの浅い層・深い層の両方で効果があるが、特に深い層でクラス特異的な再重み付けがよく働くと分析されています(浅い層では汎用的に、深い層ではクラスに応じて強く重みが変わる)。

## メリット・トレードオフ・限界
メリット
- 既存のどんな CNN にもブロック単位で後付けでき、ほぼ確実に精度が上がる。アーキテクチャを選ばない汎用性。
- 追加パラメータ・計算量が非常に小さい(ResNet-50 で約 +2.5% params、+1% 未満 FLOPs)。
- 「文脈に応じてチャネルを強調する」という汎用的な発想で、CBAM・ECA・Transformer 注意など後続研究の出発点になった歴史的意義。NAS の探索空間にも標準採用された。

トレードオフ・限界
- 空間情報をグローバル平均で完全に捨てるため、「画像のどこに注目するか」は扱えない。これが CBAM の動機になった。
- 各ブロックにプーリングと全結合が挟まることで、計算量(FLOPs)は微増でも、メモリアクセスと逐次依存(平均→FC→FC→掛け算という順番待ち)が増える。とくに軽量モデルや高速推論を狙う場面では、FLOPs の小ささに反して実機速度(レイテンシ)がやや落ちることがある。
- 後続の ECA-Net が指摘したように、全結合 2 枚による次元削減はチャネル間関係の学習をむしろ妨げる面があり、SE の設計が最適とは限らない。
- reduction ratio r の選び方で精度と計算量のバランスが変わり、わずかとはいえチューニングの自由度が増える。

## 発展トピック・研究の最前線
SE が切り開いたチャネル注意は、その後さまざまに一般化されました。CBAM/BAM は空間注意を追加し、ECA-Net は次元削減を排して効率化し、GCNet(Global Context Network)や SK-Net(Selective Kernel Networks、カーネルサイズの選択にまで注意を広げる)は大域文脈や受容野のサイズ選択にまで注意を拡張しました。Transformer の登場後は、自己注意 (self-attention) が「全トークン間の動的重み付け」という形で SE の発想を空間方向に全面展開したものと位置づけられ、CNN と注意機構を融合させたハイブリッド設計(CoAtNet など)が主流になりました。

複数センサ・複数視点の特徴を統合する文脈でも SE の考え方は生きています。「複数の入力ソース(カメラ、LiDAR など)のうちどれを今重視するか」をゲートで調整する発想は SE と同根で、マルチモーダル融合やマルチカメラ融合の重み付けにチャネル注意が応用されます。自動運転の知覚では、悪天候でカメラが信頼できないときに LiDAR チャネルを強める、といった適応的な重み付けに通じます。研究の最前線では、注意機構を「説明可能性 (interpretability)」の道具として使い、どのチャネル・どの領域が判断に効いたかを可視化する方向や、注意の計算をハードウェア効率良く実装する方向(逐次依存を減らし並列度を上げる工夫)が進んでいます。
