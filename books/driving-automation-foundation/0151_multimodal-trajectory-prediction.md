# マルチモーダル軌跡予測

## ひとことで言うと

未来の軌跡を「1 通り」ではなく「複数の候補とそれぞれの確率」として出す設計のことです。運転の未来は本質的に複数ありうる(交差点で直進も右折もありうる)ため、1 通りしか出せないモデルは候補同士の「平均」を出してしまい、どの候補でもない危険な経路(真ん中の壁に突っ込む等)を生みます。これを mode averaging 問題と呼び、それを回避するために複数の仮説を別々の山として保つのがマルチモーダル予測です。

## 直感的な理解

前方に T 字路があり、左にも右にも行けるとします。専門家(教師)の運転データには、左に行った例と右に行った例が混ざっています。

ここで「未来の軌跡を 1 本だけ出す」モデルを、二乗誤差(出力と正解の差の二乗)を最小化するように学習させると何が起きるか。数学的には、二乗誤差を最小にする 1 点の答えは「正解たちの平均」です。左の例と右の例が半々なら、最適な 1 本の答えは「左と右の中間 = 直進」になります。しかし実際の道路では直進すると正面の壁や中央分離帯に突っ込む。つまり「どの教師も実際には通らなかった、平均という名の危険経路」が出てしまうのです。これが mode averaging(モード平均化)問題です。

人間ならこう考えます。「左に行くか右に行くか、どちらもありうる。左の確率が 6 割、右が 4 割くらいか」。複数の選択肢を別々に保ち、それぞれに確からしさをつける。マルチモーダル予測はこの自然な発想をモデルに持たせ、「複数の候補軌跡 + 各候補の確率」を出させます。後段の安全判定や選択ロジックが、その候補群から安全なものを選べるようになります。

## 基礎: 前提となる概念

モード(mode)とマルチモーダル(multimodal)。確率分布の「山(峰)」のこと。modal は「山」の意。山が 1 つなら unimodal(単峰)、複数なら multimodal(多峰)。「未来の軌跡が複数通りありうる」とは「軌跡の確率分布に山が複数ある」ということです。

回帰(regression)と分類(classification)。回帰は連続値(座標など)を直接当てる問題。分類は「どのカテゴリか」を当てる問題。回帰は平均化に弱く(前述)、分類は候補ごとに独立な確率を持てるので平均化しにくい。マルチモーダル予測には両者の使い分けや組み合わせが現れます。

mode averaging 問題。1 本だけ出す回帰モデルが、複数の妥当な未来の平均を出してしまい、どれでもない非現実的な経路になる現象。マルチモーダル予測が解こうとする中心的な課題です。

Winner-Takes-All(WTA、勝者総取り)損失。複数候補のうち、正解に最も近い 1 本だけに回帰損失をかける学習法。これにより各候補が「自分が担当する未来」へと自然に分かれていきます。素朴に全候補を正解へ近づけると、結局全部が平均に潰れる(mode collapse)ので、それを避ける鍵がこの WTA です。

クエリ(query)とアンカー(anchor)。クエリは「複数の候補を生むための学習可能な種(seed)」。K 本のクエリを用意し、各クエリが 1 つの未来仮説を担当します。アンカーは「あらかじめデータから作っておいた代表的な軌跡の型」。クエリは自由に学習で形を決め、アンカーは事前に固定する、という違いがあります。

## 仕組みを詳しく

現状(ユニモーダル)の典型。
```
ego_query = Embedding(1, 256)            # 自車クエリ 1 本
→ デコーダ → trajectory [B, T, 2]        # 軌跡 1 本だけ
損失 = MSE(trajectory, 正解軌跡)          # 二乗誤差 → 平均化に弱い
```

提案(マルチモーダル)のイメージ。
```
ego_query = Embedding(K, 256)            # K 本の自車クエリ(K 個のモード)
for k in range(K):
    各クエリ k がデコーダを通る → trajectory_k [B, T, 2]
mode_head: 各モードの確率 p_k を出す       # softmax で Σ_k p_k = 1
出力: K 本の軌跡 [B, K, T, 2] と 確率 [B, K]
```
1 本のクエリを「クエリの束(bank)」にするだけで、各クエリが別々の未来仮説を担当します。クエリベースの並列計画([非自己回帰な軌跡デコーディング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0152_non-autoregressive-trajectory-decoding))とは相性が良く、クエリを「モード × 時刻」の 2 次元に拡張すれば、複数仮説を全時刻ぶん並列に出せます。

学習の鍵は「混合損失(mixture loss)」です。代表的な構成は次の通りです。

回帰側(WTA / best-of-K)。K 本のうち、正解軌跡に最も近い 1 本(勝者)だけに回帰損失をかけます。
```
k* = argmin_k distance(trajectory_k, 正解)   # 最も近い候補を選ぶ
L_reg = MSE(trajectory_{k*}, 正解)            # 勝者だけ回帰
```
これにより、各候補は「自分が一番近かったときだけ正解へ引っ張られる」ので、候補同士が別々の未来へ分かれていきます。左の教師には左担当のクエリが、右の教師には右担当のクエリが勝つように分業が進みます。

分類側(モード確率)。「どのモードが勝者だったか」を分類問題として `mode_head` に学ばせます。
```
L_cls = CrossEntropy(p, one_hot(k*))         # 勝者モードの確率を上げる
```
推論時は、最も確率の高いモードを採るか、あるいは全モードを後段(衝突判定や選択ロジック)に渡します。

before / after を対比すると、
```
before: 1 本の軌跡。分岐では平均化して危険な経路(真ん中)を出しうる。
after : K 本の軌跡 + 確率。「左60% / 右40%」のように複数仮説を保持でき、
        後段が安全に選べる。
```

設計の分岐点として、候補を「自由クエリ」で生むか「アンカー」で生むかがあります。自由クエリ(MTR 系)は柔軟だが学習で分業が崩れやすい。アンカー(MultiPath)や軌跡集合(CoverNet)は事前に山を分けておくので安定だが、集合にない動きは表せない。実務ではアンカー + オフセット回帰(アンカーで粗く山を分け、各アンカーからの微修正を回帰で出す)が安定と精度のバランスが良いとされます。

## 手法の系譜と主要論文

マルチモーダル軌跡予測は「平均化をどう避けるか」を巡って発展してきました。

Mixture Density Network(Bishop, 1994, "Mixture Density Networks")。ニューラルネットの出力を「複数のガウス分布の混合」とする古典的手法で、すべての源流です。1 つの入力に複数の妥当な出力がある回帰(逆問題など)を扱うために提案されました。各成分の平均・分散・混合重みを出力します。トレードオフは、学習が不安定でモード崩壊(全成分が同じ山に潰れる)や分散の発散が起きやすいこと。後続のすべての手法は、この不安定さをどう飼い慣らすかの工夫と言えます。

MTP(Cui et al., ICRA 2019, "Multimodal Trajectory Predictions for Autonomous Driving using Deep Convolutional Networks")。固定数のモードを出し、WTA 系の損失(最良モードに回帰 + モード分類)で学ぶ枠組みを自動運転予測に持ち込みました。「K 本出して勝者に回帰」の実用化の代表です。

MultiPath(Chai et al., CoRL 2019, Waymo, arXiv:1910.05449, "Multiple Probabilistic Anchor Trajectory Hypotheses for Behavior Prediction")。データから作った「アンカー(代表的な軌跡の型)」を用意し、各アンカーへのオフセットと確率を予測します。動機は「連続空間で直接マルチモーダルを学ぶのは難しいので、アンカーで離散化して山を分けやすくする」。効果は安定して複数モードを出せること。トレードオフはアンカー集合の質に性能が依存し、アンカーにない動きは表しにくいこと。

CoverNet(Phan-Minh et al., CVPR 2020, nuTonomy/Motional, arXiv:1911.10298, "Multimodal Behavior Prediction using Trajectory Sets")。軌跡予測を「あらかじめ用意した軌跡集合(trajectory set)からの分類問題」として解きます。動機は「回帰だと平均化しやすいので、分類にすれば各候補が独立に確率を持てる」。物理的に妥当な軌跡だけを候補にでき、平均化を構造的に回避します。トレードオフは集合の細かさと網羅性の綱引き(細かいほど候補が増え分類が難しく、粗いほど表現できない動きが出る)。

Trajectron++(Salzmann et al., ECCV 2020, arXiv:2001.03093)。条件付き変分オートエンコーダ(CVAE)で潜在変数をサンプリングし、確率的に多様な軌跡を生成。グラフで他エージェントとの相互作用も扱い、生成モデル系のマルチモーダル予測の代表です。

MTR(Shi et al., NeurIPS 2022, arXiv:2209.13508)。意図点(intention point)をクエリとして固定数の軌跡を並列に出し、局所 refine する Transformer。「クエリ束で多モードを並列生成」の現代的な到達点で、Waymo の予測チャレンジで上位を獲得しました。

要点は、平均化回避のための「複数の山を別々に保つ」工夫が、アンカー(MultiPath)・軌跡集合分類(CoverNet)・WTA 損失(MTP)・潜在サンプリング(Trajectron++)・クエリ束(MTR)へと多様化してきた、という系譜です。クエリを複数化 + 混合損失という発想は、この中で MTP/MTR 系に位置づけられ、既存のクエリベース計画に軽量に取り込めます。

## 論文の実験結果(定量データ)

評価指標を先に説明します。マルチモーダル予測では K 本の候補を出すので、「K 本のうち最良のものがどれだけ当たったか」を測る指標が標準です。minADE_K(minimum Average Displacement Error over K)は、K 本中、正解に最も近い 1 本の、全時刻平均の位置誤差(メートル)。minFDE_K(minimum Final Displacement Error)は同じく最終時刻の位置誤差。いずれも低いほど良い。Miss Rate(MR)は K 本のいずれも一定距離(例えば 2m)以内に入らなかった割合で、低いほど良い。これらは「複数候補のどれかが正解に近ければ良し」とする考え方で、まさにマルチモーダルの趣旨に沿った評価です。

MultiPath の効果。MultiPath は当時の予測ベンチマークで、ユニモーダル回帰や素朴な混合より低い minADE/minFDE を報告しました。アンカーで山を分けることで、特に分岐シーンで複数の妥当なモードを安定して出せ、平均化に起因する誤差を減らせると示されています。アブレーション的には、アンカー数を増やすと表現できるモードが増えて誤差が下がる一方、増やしすぎると学習が難しくなるトレードオフが観察されています。

CoverNet の効果。CoverNet は nuScenes 予測タスクで、回帰ベースより安定したマルチモーダル予測を示しました。軌跡集合のサイズ(粗さ)を変えると性能が変わり、適切な粒度で minADE/MR が改善することが報告されています。「回帰を分類に置き換えると平均化を避けられる」という主張を定量的に裏付けました。

WTA 損失の効果(MTP 系)。MTP 系の研究では、全モードを一律に正解へ近づける損失だと候補が平均(=単一モード)に潰れ minADE が悪化するのに対し、WTA(勝者だけ回帰)に切り替えると候補が分業して minADE_K が大きく改善する、というアブレーションが一貫して示されます。これは「混合損失の設計こそがマルチモーダル化の成否を決める」ことの直接的な証拠です。

MTR の効果。MTR は Waymo Open Motion Dataset で、当時のリーダーボード上位の mAP / minADE / Miss Rate を達成しました。クエリ束 + 局所 refine が、安定性と精度を両立できることを示しています。

指標の意味の補足。minADE_K / minFDE_K が下がるとは「複数候補のどれかが本当の未来に近い」確率が上がること、Miss Rate が下がるとは「全候補が外れる事故」が減ること。運転の安全に直結する改善です。注意点として、これらの数値はデータセット(nuScenes か Waymo か Argoverse か)と K の値・予測時間長に強く依存するため、絶対値の比較は設定込みで行う必要があります。再現性のある定性結論は「適切な混合損失 + 山を分ける機構(アンカー/集合/クエリ)で平均化を回避でき、minADE_K と MR が改善する」ことです。

## メリット・トレードオフ・限界

メリット。分岐などで複数の妥当な未来を同時に表せ、mode averaging による危険経路の生成を回避できます。各仮説に確率が付くので、後段が安全に選択・評価でき、計画の解釈性も上がります。クエリ多本化という小さな変更で導入でき、既存のクエリベース設計と相性が良い。MultiPath / CoverNet / MTR という実績ある研究系譜に乗れます。

トレードオフと限界。第一に、混合損失(WTA 等)は学習が不安定になりやすく、モード崩壊(全候補が同じになる)のリスクがあります。これを防ぐには WTA の温度・アンカー初期化・損失の重み付けの丁寧なチューニングが要ります。第二に、K 本ぶん計算が増えます。逐次デコーダのままだと特に重く、非自己回帰化([非自己回帰な軌跡デコーディング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0152_non-autoregressive-trajectory-decoding))との併用が望ましい。第三に、アンカー/軌跡集合を使う場合、その集合設計の良し悪しが性能を直接左右します(集合にない動きは表せない)。第四に、評価の落とし穴として、minADE_K は「K 本のうち最良」を測るため、K を増やすほど見かけ上良くなりますが、現実には 1 本を選んで実行するので、選択(モード確率)の質や、実際に採った 1 本の安全性を別途評価しないと過大評価になります。第五に、世界モデルへ意図を渡す設計([プランナの隠れ状態による「計画意図→世界モデル」の受け渡し](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0004_ego-hidden-world-model-handoff))と組み合わせる場合、「どのモードの隠れ状態を意図として渡すか」という下流設計の再考が必要です(全モードを渡すのか、最確モードか、確率で重み付けするのか)。

研究上のオープンな問い。(a) 候補数 K をいくつにすべきか、データ依存で適応的に決められるか、(b) モード崩壊を原理的に防ぐ損失設計はあるか、(c) 予測されたモード確率をどれだけ信頼してよいか(較正の問題)、(d) 多モード予測と安全な単一行動の選択(意思決定)をどう接続するか。

## 発展トピック・研究の最前線

マルチモーダル性の表現は、離散的な候補(アンカー/集合/クエリ)から、拡散モデルによる連続的な分布サンプリングへと広がっています。MotionDiffuser(Jiang et al., CVPR 2023)や各種 diffusion planner は、軌跡の確率分布そのものを学び、サンプリングで多様な候補を生成します。これは「山の数を固定しない」柔軟なマルチモーダル化であり、非自己回帰な並列復元とも親和的です。

また、予測と計画の境界が溶けつつあります。VADv2(Chen et al., 2024)のように、行動を確率的な語彙(vocabulary)上の分布として出し、その分布から安全な行動を選ぶ枠組みは、マルチモーダル予測を意思決定へ直結させる試みです。さらに、生成世界モデル([計画意図ベクトルによる未来特徴マップの条件付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0010_future-state-ego-conditioning))で各候補の未来を roll-out し、最も安全な未来を生む候補を選ぶ「予測 × 世界モデル × 計画」の統合が、近年の end-to-end 運転の最前線になっています。
