# 凍結バックボーン vs 微調整

## ひとことで言うと
事前学習済みのバックボーン(画像から特徴を抽出する土台ネットワーク)を新しいタスクに使うとき、そのバックボーンの重み(パラメータ)を「固定したまま」にするか、「新しいタスクに合わせて少しずつ更新する」かを選べます。前者を「凍結(freeze)」、後者を「微調整(fine-tune、ファインチューニング)」と呼びます。どちらを選ぶかで、学習の速さ・必要なメモリ・最終精度・過学習のしやすさが変わる、転移学習における最も基本的で重要な設計判断です。

## 直感的な理解
すでに運転免許を持っている人に、特定のレーシングカーの運転を教える場面を想像してください。選択肢は2つあります。1つは「運転の基礎(アクセル・ブレーキ・ハンドルの使い方)はそのまま信頼し、このマシン特有の操作だけを新しく教える」やり方。これが凍結です。もう1つは「基礎の癖も含めて、このマシンに最適化されるよう全部作り直す」やり方。これが微調整です。

教える時間(データ)が少なければ、基礎を信じて新しい部分だけ教える方が安全です。下手に基礎まで作り直すと、わずかな練習量では中途半端になり、かえって下手になる(過学習する)恐れがあります。逆に、新しいマシンが普段の運転と全く異質で、十分な練習時間があるなら、基礎から作り直した方が最終的に速く走れます。凍結か微調整かは、まさにこのトレードオフの選択です。

## 基礎: 前提となる概念
事前学習済みバックボーンは、ImageNet(約128万枚の画像で1000カテゴリ分類)などで既に「画像の見方」を学んでいます。エッジ、テクスチャ、物体の部品、物体カテゴリといった汎用的な特徴抽出器が、重みの中に詰まっています([ImageNet 事前学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0559_imagenet-pretraining))。

新しいタスク(たとえば自動運転の経路予測や、特定ドメインの画像分類)では、この重みを初期値として再利用します。これが転移学習です。問題は「この重みを訓練中にどう扱うか」です。選択肢は連続的ですが、両極端を理解すると全体像が掴めます。

凍結(freeze)とは、バックボーンの重みを訓練中いっさい更新しないことです。バックボーンは固定された「特徴抽出装置」として振る舞い、その上に乗せた新しい部分(分類ヘッドやプランナーなど)だけを学習します。

微調整(fine-tune)とは、バックボーンの重みも、新しいタスクの損失に従って勾配で更新することです。事前学習で得た知識を出発点に、タスクに合わせて全身を作り変えます。

なぜ選び分けが必要か。これを定量的に示したのが Yosinski et al.(NeurIPS 2014, "How transferable are features in deep neural networks?")です。彼らは、ネットワークの浅い層は「どんなタスクでも使える汎用特徴(エッジ・色・単純な模様。Gabor フィルタや色斑のような普遍的な特徴)」を学び、深い層は「元タスクに特化した特徴」を学ぶことを実験で示しました。だから、新タスクが元タスクに似ていてデータが少ないなら特化部分を凍結したまま使う方が安全(下手に動かすと壊れる、過学習する)、新タスクが元タスクと大きく違いデータが十分あるなら微調整して特化部分を作り替えた方が精度が出る、という使い分けが生まれます。

## 仕組みを詳しく
PyTorch での操作は実はとても単純です。各パラメータの `requires_grad`(勾配を計算するかのフラグ)を切り替えるだけです。

凍結する場合:

```python
for p in backbone.parameters():
    p.requires_grad = False     # このパラメータは更新しない
backbone.eval()                  # BatchNorm/Dropout も推論モードに固定
```

`requires_grad = False` にすると、逆伝播(backward)でこのパラメータの勾配が計算されず、オプティマイザも更新しません。さらに `backbone.eval()` にするのは、BatchNorm(バッチ正規化。ミニバッチの統計で特徴を正規化する層)が訓練中に走行統計量(running mean/var)を更新してしまうのを止めるためです。これを忘れると「重みは凍結したのに統計量だけ動く」という中途半端な状態になり、再現性や安定性を損ないます(特に小バッチや分布の偏ったデータで顕著)。同様に Dropout も推論モードで無効化されます。なお、オプティマイザに渡すパラメータも `filter(lambda p: p.requires_grad, model.parameters())` で絞ると、凍結層に無駄な最適化状態を持たせずに済みます。

微調整する場合:

```python
for p in backbone.parameters():
    p.requires_grad = True      # 更新する(通常はデフォルト)
```

実務では中間的な戦略もよく使われます。1つは段階的解凍(gradual unfreezing)です。最初の数エポックはバックボーンを凍結してヘッドだけ学習し、安定してから徐々に深い層から浅い層へと解凍していきます。学習初期に、まだランダムなヘッドから流れてくる大きく不正確な勾配がバックボーンを壊すのを防げます。もう1つは層別学習率(discriminative / layer-wise learning rate)です。バックボーンには小さい学習率(例: 1e-5)、新しいヘッドには大きい学習率(例: 1e-3)を割り当てます。さらに「浅い層ほど小さく、深い層ほど大きい」と層ごとに減衰させる layer-wise LR decay は、ViT の微調整で標準的に使われます。事前学習の知識を保ちつつ、ヘッドを速く学習させる狙いです。

メモリ・速度の具体的な違いも理解しておきましょう。凍結すると、その層の勾配と最適化状態(Adam なら各パラメータごとに保持する1次・2次モーメントの2つ)を保持しなくてよいので、学習時の VRAM 消費が大きく減ります。概算では、微調整時はパラメータ本体に加えて勾配で同サイズ、Adam の状態でさらに2倍サイズ、と1パラメータあたり4倍前後のメモリを使うのに対し、凍結層はパラメータ本体だけで済みます。backward の計算も、凍結層より手前まで勾配を伝える必要がなくなれば省け、学習が速くなります。バックボーンがモデルの大半のパラメータを占める典型的な構成では、凍結によって学習時メモリが半分近くまで減ることもあります。ただし forward(順伝播)は凍結しても必要なので、推論コスト自体は変わりません。

凍結バックボーンを使う際の実装上の注意として、後段のネットワークが期待する特徴の次元(チャンネル数や空間サイズ)と、凍結バックボーンが出力する次元が食い違うことがあります。その場合、両者を橋渡しする射影層(1×1 畳み込みや線形層で次元を合わせる小さな層)を挿入する必要があります。凍結すると当然バックボーン側で次元を合わせられないので、この射影層の設計が欠けていると、特徴を後段に接続できず学習ループが組み立てられません。凍結経路を導入するときは、この次元整合を最初に詰めておくのが定石です。

## 手法の系譜と主要論文
転移学習における凍結 vs 微調整の議論は、深層学習の初期から積み重ねられてきました。

DeCAF(Donahue et al., ICML 2014, arXiv:1310.1531)は、ImageNet で学習した CNN の中間層特徴を「そのまま(凍結して)」別タスクの汎用特徴量として使う、というアイデアを示した初期の代表です。当時はこの凍結特徴 + 単純な分類器(SVM など)だけで、多くのタスクで既存手法を上回りました。ただしタスクが元と離れると凍結では限界があることも示唆されました。

Yosinski et al.(NeurIPS 2014, arXiv:1411.1792)は、層ごとに「どこまで凍結し、どこから微調整すべきか」を経験ではなく数値で測る実験フレームワークを提案し、浅い層は汎用・深い層は特化という法則を定量化しました。あわせて、微調整が一般に凍結より精度を出すこと、しかしデータが少ないと微調整で過学習しやすいことを示しました。

ULMFiT(Howard & Ruder, ACL 2018, arXiv:1801.06146)は、自然言語処理の例ですが「段階的解凍」「層別学習率(slanted triangular learning rate)」を体系化した代表です。いきなり全層を微調整すると事前学習知識を破壊(破滅的忘却、catastrophic forgetting)するため、という動機で、少データでも安定した微調整を実現しました。凍結と微調整の中間設計を考える上で重要な参照点です。

BiT / Big Transfer(Kolesnikov et al., ECCV 2020, arXiv:1912.11370)は、大規模事前学習 + シンプルな微調整レシピ(BiT-HyperRule)で、多様な下流タスクへ強力に転移できることを示しました。事前学習の規模が大きいほど微調整が安定し、少データでも高精度に到達することを定量化した点で、現代の「大きく事前学習して微調整」という流れを後押ししました。

## 論文の実験結果(定量データ)
Yosinski et al. の実験は今も示唆に富みます。彼らは ImageNet を2つの半分(タスク A とタスク B)に分け、A で学習した重みの一部を B に移して B で学習させました。指標は分類精度です。重要な発見は2つあります。第一に、移す層を深くするほど(=特化した層まで凍結して移すほど)、A と B が異質な場合に性能が落ちること。中間層あたりを凍結して移すと数ポイント精度が下がる現象が観測され、これは深い層がタスク特化的であることと、層をまたぐ「共適応(co-adaptation)」を断ち切ることの両方が原因だと分析されました。第二に、移した層を凍結せず微調整すると、移すことで生じた性能低下が回復し、しばしばゼロから学習するより高い精度に達したこと。これは微調整の優位と、事前学習の初期化が正則化として働くことを示します。

BiT の実験では、ImageNet-21k(約1400万枚)や JFT(約3億枚)といった大規模データで事前学習したモデルを、下流の小規模データセット(例: 1クラスあたり数枚〜数十枚)に微調整したとき、データが極端に少ない領域でも高い精度を維持できることが報告されました。たとえば1クラスあたり数枚という極端な少データでも、大規模事前学習があれば実用的な精度に達し、事前学習規模が大きいほど下流で必要なデータ量が減るという関係(データ効率の向上)が定量的に示されています。

アブレーション的な観点でまとめると、「データが少ない × 元タスクと近い」なら凍結が安全で計算も軽い、「データが多い × 元タスクと遠い」なら微調整が高精度、という二軸の判断が、これらの論文の数値で裏付けられています。中間領域では段階的解凍や層別学習率が両者の良いところを取ります。

## メリット・トレードオフ・限界
凍結のメリットは、学習時メモリ(勾配・最適化状態)が減って訓練が速くなること、パラメータが少ないぶん小データでも過学習しにくいこと、事前学習の汎用特徴をそのまま保てる(破滅的忘却が起きない)こと、複数タスクで同じ凍結特徴を共有できることです。デメリットは、バックボーンを新タスクに最適化できないため上限精度が低くなりやすいこと、元タスクと新タスクが大きく異なると固定特徴が合わず性能が出ないこと(ドメインギャップが大きいと致命的)です。

微調整のメリットは、バックボーンごとタスクに最適化でき最終精度が高くなりやすいことです。デメリットは、学習時メモリ・計算が大きいこと、小データだと過学習や事前学習知識の破壊(破滅的忘却)が起きやすいこと、学習率設定がシビア(大きすぎると事前学習重みを壊し、いわゆる feature distortion が起きる)なことです。

研究上の未解決課題としては、事前学習タスクと下流タスクのドメインギャップを自動的に測り「凍結すべき層」を決める手法、破滅的忘却を抑えつつ全層を動かす正則化、分布外データ(OOD)に対する頑健性は時に全微調整より凍結や線形プローブの方が高いという報告(微調整が分布内精度を上げる一方で OOD 頑健性を損なうトレードオフ)の理論的理解が挙げられます。

## 発展トピック・研究の最前線
近年は「凍結か全微調整か」という二択を超え、パラメータ効率の良い微調整(PEFT, Parameter-Efficient Fine-Tuning)が主流になりつつあります。バックボーン本体は凍結したまま、小さな追加モジュールだけを学習する手法群です。代表に Adapter(層の間に小さなボトルネック層を挿入)、LoRA(Hu et al., ICLR 2022, arXiv:2106.09685。重み行列の更新を低ランク行列の積 BA で近似して、その小さな2行列だけ学習する。元の重み W は凍結し W+BA で推論)、Prompt Tuning / Visual Prompt Tuning(Jia et al., ECCV 2022, arXiv:2203.12119。入力側やトークン列に学習可能なトークンを足し、本体は凍結)があります。LoRA はランク r を 4〜16 程度に取るだけで、全微調整に匹敵する精度を、学習対象パラメータを元の 1% 未満に抑えて達成できると報告され、大規模モデル時代の標準になりました。これらは凍結の利点(メモリ削減・忘却回避・複数タスクで本体を共有)を保ちつつ、微調整に近い精度を引き出せるため、実用的な折衷案として急速に広がっています。凍結 vs 微調整という古典的な軸の上に、この第三の道が加わったのが現在の研究地図です。さらに、自己教師あり事前学習(MAE や DINOv2)で得た特徴は凍結したまま(線形プローブや軽い PEFT)でも非常に強く、「強い事前学習 + 凍結」という組み合わせの価値が再評価されています。
