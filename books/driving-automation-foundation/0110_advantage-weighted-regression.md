# Advantage-Weighted Regression (AWR) ポリシー損失

## ひとことで言うと

すでに集めてある「お手本データ」(エージェントが過去にとった行動の記録) から方策 (どう行動するかを決めるモデル) を学習するとき、「良い行動ほど強く真似する」ように重みをつけて回帰 (真似) させる損失です。良し悪しの判定には優位度 (advantage) という値を使い、それを指数関数で重みに変換します。模倣学習の安定さと、強化学習の「良い行動を選り好みする」性質を、たった1つの重みつき教師あり損失で両立させるのが核心です。

## 直感的な理解

専門家のデモを集めて「同じことをやれ」と教えるのが模倣学習 (Imitation Learning) です。これは単純で安定していますが、致命的な弱点があります。デモの中に「上手なプレー」と「平凡なプレー」「失敗しかけたプレー」が混ざっていると、それらを区別せず全部同じ重みで真似してしまいます。本当は「上手なものだけ強く真似し、ダメなものは軽く扱いたい」のです。

一方、純粋な強化学習は環境の中で実際に試行錯誤しながら「どの行動が得か」を学びます。しかし試行錯誤にコストや危険が伴う領域 (自動運転、医療、ロボット操作など) では、本番環境で何度も失敗させるわけにはいきません。そこで「録画済みデータだけ」で学習するオフライン強化学習 (Offline RL) が必要になります。

AWR の発想を一言でいうと、「模倣学習に、行動の質に応じた重みづけを足す」ことです。重みは優位度 (この行動は平均よりどれだけ良かったか) から作ります。良い行動なら重みが大きくなり強く真似され、悪い行動なら重みがほぼゼロで無視される。教師あり回帰の枠組みは壊さないので学習が安定し、それでいて「玉石混交のデモから玉だけ拾う」ことができます。

身近なたとえで言えば、過去の自分のプレー録画を見返して上達するスポーツ選手に近いです。すべての動きを等しく反復練習するのではなく、「あのときのフォームは良かった」という瞬間を選んで重点的に体に覚え込ませる。逆に明らかに失敗した動きは「あれは真似しないでおこう」と軽く流す。AWR の重みづけは、この「選んで反復する」を数式にしたものです。

## 基礎: 前提となる概念

いくつかの用語を、知らない人向けに噛み砕きます。

- 強化学習 (Reinforcement Learning, RL): エージェント (行動する主体) が環境の中で行動し、報酬 (reward、良い行動なら高く悪い行動なら低い数値) を受け取り、報酬の合計が最大になるような行動の選び方 (これを方策 / policy と呼びます) を学ぶ枠組みです。
- 方策 pi(a | s): 状況 s を入れると行動 a を (確率的にまたは決定的に) 返す関数。これがエージェントの「頭脳」です。確率的方策はガウス分布の平均と分散を出すことが多く、決定的方策は行動を1点で出します。
- 報酬 r とリターン (収益): 報酬は1ステップごとの良さの点数。リターンはその時点以降の報酬の合計 (遠い未来は割引率 gamma で少し割り引く)。RL の目的はリターンの期待値を最大化することです。
- Q値 Q(s, a): 状況 s でその行動 a を取り、その後は方策に従ったときに得られるリターンの見積もり。「この場面でこの操作をすると最終的にどれくらい得か」を表す数値です。
- 状態価値 V(s): 状況 s で、方策に従ったとき平均的にどれくらい得かの見積もり。「この場面そのものの良さ」です。
- 優位度 A(s, a) = Q(s, a) - V(s): その行動が「その場面の平均」よりどれだけ良かったか。プラスなら平均より良い行動、マイナスなら悪い行動です。AWR の重みはこの値から作ります。優位度を使う利点は、場面ごとの価値の高低 (V) を差し引くことで「行動そのものの良し悪し」だけを取り出せる点にあります。たとえば全体的に有利な局面では Q が一律に高くなりますが、そこから V を引くことで「その有利な局面の中でどの行動が相対的に良いか」を比較できます。

オフライン RL 特有の難しさも押さえておきます。新しく環境とやり取りできないため、「データに含まれない行動」の良し悪しが分かりません。にもかかわらず Q 関数は数学的に「データにない行動」も評価できてしまい、しばしばそれを過大評価 (overestimation) します。たとえば「アクセル全開で壁に突っ込めば Q が高い」と誤って学習してしまう。これを 分布外 (out-of-distribution, OOD) 行動の過大評価といい、オフライン RL 最大の敵です。AWR 系の手法はこの罠を、方策をデータ分布から大きく離さない設計で回避します。具体的には、重みづけ回帰は「データに実在した行動 a」だけを目標にするので、データにない行動を能動的に作り出すことがありません。

## 仕組みを詳しく

AWR のポリシー損失は次の形です。記号で示します。

```
重み w(s, a) = exp( beta * A(s, a) )            # beta は温度パラメータ
損失   L(theta) = - 平均[ w(s, a) * log pi_theta(a | s) ]
```

`log pi_theta(a | s)` は「方策がその状況 s で実際にデモがとった行動 a を出す確率の対数」です。これを大きくする = デモの行動を出しやすくする。そこに重み w をかけるので、優位度の大きい (良い) 行動ほど強く「出しやすく」されます。負号は、最大化問題を最小化問題 (損失) に変換するためのものです。

なぜ指数 exp なのか。これは理論的な裏付けがあります。「現在のデータ分布から KL ダイバージェンス (分布間の距離、片方の分布からもう片方を見たときの情報量の差) で測って一定以上離れない」という制約のもとでリターンを最大化する問題を、ラグランジュ未定乗数法で解くと、最適方策は次の形になります。

```
pi*(a|s) ∝ pi_data(a|s) * exp( A(s,a) / 温度 )
```

つまり「データ分布を、優位度の指数で重みづけし直したもの」が答えになる。AWR の損失は、この最適方策に現在の方策を射影 (近づける) するための重みつき最尤推定なのです。温度パラメータ (beta = 1/温度) は、KL 制約の厳しさを決めるラグランジュ乗数に対応します。温度が小さい (beta が大きい) ほど「最も良い行動だけを尖って真似る」、温度が大きい (beta が小さい) ほど「データ全体をなだらかに真似る」方向になります。この導出は古くは Reward-Weighted Regression (RWR, Peters & Schaal 2007) や REPS (Relative Entropy Policy Search, Peters et al. 2010) に遡り、MARWIL、AWR、CRR、AWAC、IQL に共通する数学的骨格です。

決定的方策 (1つの行動を直接吐く実装) の場合は、`log pi` の代わりに二乗誤差 (MSE) を使うのが一般的です。ガウス分布を仮定すると `-log pi` が MSE に比例するので、自然な置き換えです (分散を固定したガウスでは `-log pi(a|s) = ||a - mu(s)||^2 / (2 sigma^2) + 定数` となり、定数とスケールを除けば MSE と一致する)。

```python
# 典型的な実装 (advantage-weighted regression)
pred_actions = policy(states)                 # 方策が予測する行動 [B, action_dim]
with torch.no_grad():
    v_val = value_fn(feats).squeeze(-1)       # V(s)  [B]
    q_val = q_fn(torch.cat([feats, actions], -1)).squeeze(-1)  # Q(s, a) [B]
    advantage = q_val - v_val                 # A = Q - V  [B]
    weights = torch.exp(beta * advantage).clamp(max=100.0)     # w = exp(beta*A)

policy_loss = (weights * F.mse_loss(pred_actions, actions, reduction='none').mean(-1)).mean()
```

ステップで分解します (B はバッチサイズ = 一度に処理する件数)。

1. `pred_actions = policy(states)`: 方策に状況 (画像・センサ特徴など) を入れて行動を予測。形状 `[B, action_dim]`。
2. `advantage = Q(s,a) - V(s)`: デモがとった行動 a に対する優位度を計算。Q と V は別途学習した小さなネットワーク (MLP) で見積もります。`torch.no_grad()` で囲むのは「ここでは Q/V 側に勾配を流さない (方策だけを更新する)」ためです。Critic (価値) と Actor (方策) を分離して交互に学ぶ Actor-Critic 構成の作法です。
3. `weights = exp(beta * advantage)`: 優位度を指数で重みに変換。beta=3.0 なら、優位度が 1 増えるごとに重みが e^3 ≒ 20 倍。良い行動が圧倒的に重視されます。
4. `.clamp(max=100.0)`: 指数は急増するので重みが暴走しないよう上限でクリップ。これがないと、たまたま優位度が大きく見積もられた1サンプルが学習全体を支配します。実装によっては重みをバッチ内で正規化 (合計が 1 になるよう割る) する流儀もあります。
5. `F.mse_loss(..., reduction='none').mean(-1)`: 予測行動とデモ行動の二乗誤差を行動次元で平均し `[B]`。
6. 重みをかけて全サンプルで平均。

数値例を出します。バッチに3サンプル、優位度 `A = [+1.0, 0.0, -1.0]`、beta=3.0 とします。重みは `exp(3)=20.1`, `exp(0)=1.0`, `exp(-3)=0.05`。デモの二乗誤差がどれも 0.5 なら、損失への寄与は `20.1*0.5=10.05`, `0.5`, `0.025`。良い行動のサンプルがほぼ学習を独占し、悪い行動は無視される。これが「良い行動を選り好みして真似る」の中身です。beta を 1.0 に下げると重みは `2.72, 1.0, 0.37` となり、悪い行動も無視されなくなる代わりに選り好みが弱まります。

優位度を出すための Q/V はどう学ぶか。ここが手法ごとの最大の違いです。

- AWR (原典) は、収集済みの軌跡から TD(lambda) などで V を学び、A をモンテカルロ的に (実際に観測したリターンから V を引いて) 推定します。
- AWAC は Actor-Critic 構成で、Q を通常のベルマン更新 (詳細は [Bellman / TD 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0112_bellman-td-loss)) で学び、A = Q(s,a) - E_{a'~pi}[Q(s,a')] として推定します。
- IQL は expectile 回帰 (期待値ではなく分位点に近い量を学ぶ非対称な回帰。誤差が正のときと負のときで重みを変える) で「Q の上側の値」を V として学び、データにない行動を一切評価せずに済む設計にしています。これがオフライン RL での過大評価を根本回避する鍵です。

## 手法の系譜と主要論文

- RWR / REPS (Peters & Schaal 2007; Peters, Mülling, Altun, AAAI 2010)。提案: 報酬を指数で重みに変換して方策を回帰する Reward-Weighted Regression と、KL 制約つき方策探索 REPS。動機: 方策勾配が不安定なので、重みつき教師あり回帰として方策更新を書きたい。新規性: 「KL 制約つきリターン最大化 → 指数重み回帰」という導出をロボット制御に持ち込んだ祖先。
- MARWIL — Monotonic Advantage Re-Weighted Imitation Learning (Wang, Bapst, Heess, et al., NeurIPS 2018)。提案: デモを優位度で重みづけして模倣する定式化。「悪いデモも混ざった集合から方策を改善する」という問題意識を明確にした、深層学習時代におけるこの系譜の起点です。
- AWR — Advantage-Weighted Regression (Peng, Kumar, Zhang, Levine, 2019, arXiv:1910.00177)。提案: 報酬から優位度を見積もり `exp(beta*A)` で重みづけして行動を回帰する、それを繰り返すだけのシンプルなアルゴリズム。動機: 当時のオフポリシー RL (過去データを再利用する手法) は実装が複雑で不安定だった。回帰ベースなら安定で実装も簡単。新規性: 「教師あり回帰の反復」だけで方策改善が成り立つことを理論と実験で示した。
- CRR — Critic Regularized Regression (Wang, Novikov, Zolna, et al., NeurIPS 2020, arXiv:2006.15134)。提案: AWR をオフライン RL に特化させ、重みを `exp(A)` だけでなく「A>0 なら 1、else 0」という二値フィルタ (binary max) でも使えるようにした。動機: 純オフラインで安定に方策を抽出するため。新規性: 重み関数の選択肢を整理し、フィルタ型が大規模オフラインで頑健だと示した。
- AWAC — Advantage-Weighted Actor-Critic (Nair, Gupta, Dalal, Levine, 2020, arXiv:2006.09359)。提案: AWR を「オフラインで事前学習 → オンラインで微調整」の両方に使える Actor-Critic に組み込む。動機: オフラインデータで賢く初期化し、その後の実環境微調整を速く安定にしたい。新規性: オフライン→オンラインのシームレスな移行を実証。
- IQL — Implicit Q-Learning (Kostrikov, Nair, Levine, ICLR 2022, arXiv:2110.06169)。提案: expectile 回帰で価値を学び、抽出 (方策の取り出し) には AWR と同じ `exp(beta*A)` 重みづけ回帰を使う。動機: 分布外行動を一切評価しない設計で過大評価を根本回避する。新規性: データ内の行動だけで Q/V を学べる「implicit」な定式化。現在のオフライン RL の標準的ベースラインの一つです。

系譜をまとめると、「KL 制約から指数重み回帰を導出」(RWR/REPS) → 「優位度で重みづけして模倣」(MARWIL) → 「それだけで RL が成り立つ」(AWR) → 「オフラインに特化、フィルタ型重み」(CRR) → 「オフライン事前学習+オンライン微調整」(AWAC) → 「過大評価を構造的に回避」(IQL) と発展してきました。

## 論文の実験結果(定量データ)

評価には D4RL (Datasets for Deep Data-Driven RL, Fu et al. 2020) というオフライン RL の標準ベンチマークがよく使われます。MuJoCo 物理シミュレータの移動タスク (halfcheetah, hopper, walker2d) で、データの質を変えた複数の設定 (medium=中級者のデモ、medium-replay=学習途中の雑多なデータ、medium-expert=中級と専門家の混合) で評価し、最終的な正規化スコア (0 = ランダム方策、100 = 専門家方策) を報告します。

- IQL (Kostrikov et al. 2022) は、MuJoCo locomotion の medium / medium-replay / medium-expert を平均した正規化スコアでおよそ 77 前後を報告し、当時の有力手法 (CQL や TD3+BC) と同等以上を、より短い学習時間で達成しました。特に手間のかかる Antmaze (まばらな報酬で長い経路を計画する難タスク) の難設定 (antmaze-medium/large) で他手法を上回り、報告ではこれらの難設定で従来法を大きく引き離しました。論文の主張の核心は「expectile の分位 tau を 0.5 から 0.7、0.9 と上げるほど、データ内の良い行動に重みが寄り、性能が上がる」という点で、tau を上げすぎると逆に分散が増えて不安定になるトレードオフも示されています。
- CRR (Wang et al. 2020) は、RL Unplugged ベンチマーク (大規模オフラインデータ) の DeepMind Control Suite タスクで、当時の代表的手法 (BCQ, BEAR, ABM など) を上回る平均性能を報告しました。アブレーションでは「指数重み」と「二値フィルタ重み」を比較し、大規模で多様なデータでは二値フィルタの方が安定して高性能になる傾向が示されています。
- AWAC (Nair et al. 2020) の代表的な結果は「オフライン事前学習からオンライン微調整への移行速度」です。ロボット操作タスク (dexterous hand など) で、ゼロからオンライン学習する手法が成功にかかるサンプル数に対し、AWAC はオフラインデータで初期化することで数分の一のオンラインサンプルで同等性能に到達したと報告されています。これは「デモを優位度で重みづけして初期化する」ことの実用的価値を示す結果です。
- アブレーション (どの要素を抜くとどうなるか) の代表例: 温度 beta の影響です。beta を 0 にすると重みが全て 1 になり、ただの模倣学習 (Behavior Cloning) に退化して、デモに悪い行動が混ざる設定 (medium-replay など) で性能が大きく落ちます。逆に beta を大きくしすぎると一部サンプルが重みを独占し、勾配の分散が爆発して学習が不安定化します。論文群は beta に最適な中間値が存在することを一貫して示しており (IQL では beta=3.0 や 10.0 がタスクに応じて使われる)、これが AWR 系の最も敏感なハイパーパラメータです。

これらの数値が意味するのは、「優位度による重みづけは、デモの質が混在するほど効く」ということです。専門家だけのクリーンなデモなら模倣学習で十分ですが、現実のデータは雑多であり、そこで AWR の選り好み能力が効いてきます。実際 medium-expert (良いデータが多い) では BC との差は小さく、medium-replay (雑多) では差が大きく開くのが典型的なパターンです。

## メリット・トレードオフ・限界

メリット
- 教師あり回帰がベースなので学習が安定し壊れにくい。実装も短い (数行)。
- 「良い行動だけ強く真似る」ことで、玉石混交のデモから玉だけ拾える。
- 環境とのやり取り不要。録画データだけで方策を磨ける (試行錯誤の危険・コストを回避)。
- 模倣学習で得た重みを初期値にでき、オンライン微調整への移行が速い (AWAC の知見)。
- 方策をデータ分布から大きく離さないため、オフライン RL の過大評価に対して相対的に頑健。データにない行動を能動的に生成しない構造的安全性がある。

トレードオフ・限界
- 温度 beta の調整がシビア。大きすぎると一部サンプルが学習を支配し発散、小さすぎるとただの模倣学習に戻る。
- 指数重みが暴走しやすく、クリップ (上限) が事実上必須。クリップ値自体もハイパーパラメータ。
- 優位度 A=Q-V の精度に全面的に依存。Q/V の見積もりが悪ければ「悪い行動を良いと誤認して強く真似る」事故が起きる。
- 方策を「データに近い範囲」に閉じ込めるため、データに含まれない優れた行動を発見する能力は原理的に乏しい (デモの天井を超えにくい)。
- 重みつき回帰は事実上「データの一部しか使わない」ため、サンプル効率が悪化しうる (悪い行動の情報を捨てている)。

未解決の課題: 優位度推定の不確実性をどう重みに反映するか (推定が怪しい行動の重みを下げる)、温度 beta を自動調整する方法、そしてオフラインで学んだ方策を安全にオンライン展開する移行戦略などが、現在も活発に研究されています。

## 発展トピック・研究の最前線

- 拡散モデルや Flow Matching を方策表現に使い、AWR 的な重みづけと組み合わせる「拡散方策のオフライン RL」(Diffusion-QL, IDQL など) が登場しています。多峰な行動分布を表現できる生成モデルと、優位度重みづけの選り好みを組み合わせる方向です ([拡散モデル denoising 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0168_diffusion-denoising-loss), [Flow Matching 速度損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0124_flow-matching-velocity-loss) 参照)。IDQL (Implicit Diffusion Q-Learning) は IQL の価値推定と拡散方策の表現力を組み合わせ、重みづけサンプリングで方策を抽出します。
- 大規模な行動模倣の文脈でも、優位度や報酬で重みづけして「良いデモを厚く学ぶ」考え方が、言語モデルの選好学習 (DPO: Direct Preference Optimization など) と数学的に通じることが指摘されており、RL と教師あり学習の境界を埋める統一的視点として注目されています。DPO も KL 制約つき最適化から閉形式の重みつき損失を導く点で AWR と同じ系譜にあります。
- expectile 回帰の分位、温度、クリップといったハイパーパラメータを理論的に決める研究や、優位度の推定に分布 (不確実性) を持たせて重みを補正する研究が進んでいます。
