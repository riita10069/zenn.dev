# grid_sample バイリニア特徴抽出

## ひとことで言うと
画像や特徴マップ(数値が格子状に並んだデータ)から、「ちょうど格子の上ではない、中間の座標」の値を取り出したいときに使う操作です。たとえば「3.7行目、5.2列目の値」を、周りの4つの整数位置の値から滑らかに混ぜて求めます。しかもこの混ぜ方は微分できるので、深層学習の中で安心して使えます。空間変換やサンプリング型注意など、座標を学習で動かす仕組みの土台になる基本部品です。

## 直感的な理解
カメラ画像から作った特徴マップは、たとえば [256チャネル, 高さ8, 幅8] のように整数の格子で並んでいます。値が定義されているのは (0,0), (0,1), ..., (7,7) という整数の位置だけです。

ところが多くの処理では、整数では割り切れない連続座標の値が欲しくなります。たとえばBEV融合(複数カメラを真上から見た地図に統合する処理)では、「3次元空間のこの点をカメラに投影したら画像上の (3.7, 5.2) に落ちた。そこの特徴が欲しい」という状況が常に起こります。投影で出てくる座標は連続値(小数)なので、整数の格子からピンポイントで拾うことができません。

素朴な解決は「一番近い格子点 (4, 5) の値を使う(最近傍)」ですが、これには2つの問題があります。第一に、座標が少し動いただけで急に隣の値に飛ぶので、結果がガタガタになります。第二に、最近傍は「どの格子点を選ぶか」が階段状の不連続な選択なので、微分(勾配)が定義できず、学習で位置を調整できません。深層学習では「どこを見るか」自体を学習したい場面が多いので、この2つ目の問題が致命的です。

そこで使うのが grid_sample(グリッドサンプル)という、連続座標で滑らかに、しかも微分可能に値を取り出す操作です。深層学習フレームワークでは標準関数として提供され、`mode='bilinear'`(バイリニア=2次元の線形補間)を指定して使うのが一般的です。喩えるなら、地図上の格子状の標高データから、格子の間の地点の標高を、周囲4点の高さを傾斜に応じて混ぜて推定するようなものです。

## 基礎: 前提となる概念
補間(interpolation): 既知の点の間の未知の点の値を、周囲の値から推定することです。線形補間は2点の間を直線で結んで推定し、双線形補間(bilinear)はそれを2次元に拡張して、四角い格子の4隅の値から内部の点を推定します。「双(bi)」は「2方向(横と縦)それぞれに線形補間する」という意味で、まず横方向に2回、次にその結果を縦方向に1回、線形に混ぜる(順番は逆でも結果は同じ)操作と等価です。

微分可能性(differentiability)と勾配: 深層学習は、出力の誤差を入力やパラメータに対して微分(勾配)し、その勾配を使ってパラメータを少しずつ更新します。ある操作が微分可能であれば、その操作を通して勾配を逆向きに流せます(誤差逆伝播)。grid_sample が微分可能であることは、「サンプリング位置をどちらにずらせば誤差が減るか」という勾配を座標まで届けられることを意味し、サンプル位置自体を学習できるようになります。具体的には、出力を座標 (x, y) で微分すると、4隅の値の差(局所的な勾配)に比例した量が返り、これが「右に動けば値が大きくなる」といった情報を座標に伝えます。

正規化座標と座標規約: grid_sample はサンプリング座標を [-1, 1] の範囲で受け取る規約が一般的です。-1 が左端/上端、+1 が右端/下端で、[0,1] の正規化座標からは `座標 * 2 - 1` で変換します。座標の並びが (x, y)(列, 行)であり、行列で慣れた (行, 列) = (y, x) と逆である点も重要な作法です。この (x,y) 順の取り違えは、損失が下がらないのに原因が分からないという厄介なバグの定番原因です。

## 仕組みを詳しく
バイリニア補間の中身を数値で見ます。特徴マップから連続座標 (x, y) = (5.2, 3.7) の値を取りたいとします(x=列, y=行)。

1. 周囲の4つの整数格子点を取る: (5,3), (6,3), (5,4), (6,4)
2. 端数を重みにする: x の端数 = 0.2、y の端数 = 0.7
3. 4点を重み付き平均する。

```
値(5.2, 3.7)
 = (1-0.2)(1-0.7) * 値(5,3)   # 左上、重み 0.24
 + (0.2)(1-0.7)   * 値(6,3)   # 右上、重み 0.06
 + (1-0.2)(0.7)   * 値(5,4)   # 左下、重み 0.56
 + (0.2)(0.7)     * 値(6,4)   # 右下、重み 0.14
```

重みの合計は 0.24+0.06+0.56+0.14 = 1.0 になります。座標が下に寄っている(端数0.7)ので下側2点の重みが大きい、という直感どおりの混ざり方です。座標を連続的に動かすと重みも連続的に変わるので、出力も滑らかに変わり、座標に関して微分できます。この「微分できる」ことが学習で決定的に重要で、後段のロス(誤差)から「サンプリング位置をどっちにずらせば良くなるか」という勾配を座標まで逆伝播できます。

図で4隅と重みのイメージを示します。

```
   (5,3)─────────(6,3)      重み: 0.24      0.06
     │   ・(5.2,3.7)│              ↘      ↙
     │              │           サンプル点
   (5,4)─────────(6,4)      重み: 0.56      0.14
   下側に寄っているので下の2点が支配的
```

実装上の作法をまとめます。サンプリング座標は [-1, 1] で渡す(`sample_grid = sample_locs * 2 - 1`)。格子の最後の次元は (x, y) の順で、(行, 列) と逆なので間違えやすい。`padding_mode='zeros'` は座標が枠外([-1,1]の外)に出たら0を返す設定で、可視性マスクと組み合わせると「外に出た=寄与なし」を自然に表現できます(`'border'` だと端の値を引き伸ばすので、枠外を見たい意図とずれます)。`align_corners=False` はピクセルを「点」ではなく「面積を持つマス」として扱う座標規約で、新しいコードではこちらが推奨されます。align_corners の True/False を取り違えると、半ピクセルぶんの系統的なずれが入り、低解像度ほど影響が大きくなります。

典型的な使い方は次の形です。

```python
sample_grid = sample_locs * 2 - 1            # [0,1] → [-1,1]
feat_v = values_spatial.reshape(B, V, C, H, W)[:, v_idx]  # [B, C, H, W]
sampled = F.grid_sample(
    feat_v, sample_grid, mode='bilinear',
    padding_mode='zeros', align_corners=False
)  # [B, C, N, num_z]
```

形状(tensor shape)で追うと、入力の特徴マップは [B, C, H, W](バッチ・チャネル・高さ・幅)、サンプリング格子は [B, N, num_z, 2](BEVマス N 個 × 柱の高さ num_z 個 × 座標2)で、出力は [B, C, N, num_z] になります。つまり「BEVの各マスの各高さ点について、画像のどこを見たいか」を一気に指定し、対応する C 次元の特徴ベクトルを一括で拾ってくる処理です。N が 200×200 = 40000、num_z が 4 なら、1カメラあたり16万箇所を1回の grid_sample で補間します。GPU で一括並列処理されるため高速です。逆伝播では、これら16万点それぞれが触れた4隅の格子点に勾配が散らばって(scatter-add で)足し込まれ、入力特徴マップとサンプリング座標の両方に勾配が届きます。

## 手法の系譜と主要論文
grid_sample は単独の論文の手法というより、空間変換を微分可能にする基盤操作です。

Spatial Transformer Networks (Jaderberg, Simonyan, Zisserman, Kavukcuoglu, NeurIPS 2015, arXiv:1506.02025): 起源として最も有名な研究です。「ネットワークの中に、特徴マップを微分可能に再サンプリング(回転・拡大・歪み補正)する層を入れたい」という動機から、サンプリング格子とバイリニア補間を提案しました。なぜ微分可能性にこだわったかというと、変換のパラメータ(どう歪めるか)もロスから学習で決めたかったからです。効果は、入力の位置ずれや回転に強いネットワークが作れたこと。トレードオフは、バイリニアは滑らかな反面、細かいエッジが少しぼけることです。

FlowNet (Dosovitskiy et al., ICCV 2015, arXiv:1504.06852)とその後継 FlowNet2、ステレオ視差推定: 光学フロー(画素の移動)で一方の画像を他方へワープ(変形)させ、再構成誤差を取る処理に同じ微分可能サンプリングが使われます。座標を介して画像を再投影するあらゆる場面で grid_sample 系が現れる典型例です。

Deformable DETR (Zhu, Su, Li et al., ICLR 2021, arXiv:2010.04159) と BEVFormer (Li, Wang, Lu et al., ECCV 2022, arXiv:2203.17270): 変形可能注意(deformable attention)の中で同じ連続座標サンプリングを使います。固定の格子ではなく、ネットワークが予測したオフセット位置から特徴を拾うので、grid_sample のような微分可能補間が必須です。投影位置に学習オフセットを足してからサンプリングする設計はこの考え方の系譜にあります。なお Deformable DETR は計算効率のため bilinear をカーネル化した専用CUDA実装(MSDeformAttn)を使いますが、原理は grid_sample と同じ双線形補間です。専用カーネルは複数解像度の特徴マップを一度に扱い、メモリ往復を減らすことで標準 grid_sample より高速・省メモリになります。

関連して、3次元再構成(微分可能レンダリング)、NeRF 系のボリュームサンプリングなど、座標を介して特徴や画像を変形・再投影するあらゆる場面で grid_sample 系の補間が使われており、現代の幾何的深層学習の共通基盤になっています。

## 論文の実験結果(定量データ)
grid_sample 自体の貢献は、それを使った手法の性能として現れます。Spatial Transformer Networks は、歪んだ手書き数字データセット(distorted MNIST、回転・拡大・射影変形を加えた MNIST)で、空間変換層を入れることで誤分類率を大きく下げたことを報告しています。報告によれば、変形に対して脆弱な通常CNNに対し、STN を組み込むと誤り率が相対的に数割改善し(例えば射影変形を加えた設定で誤り率がおよそ半減する条件も報告)、ネットワークが入力を自動で整列(回転や拡大を打ち消す)できることが定量的に示されました。

Deformable DETR では、grid_sample 型のサンプリング注意により、通常の DETR が収束に約500エポックを要したのに対し約50エポック(約10分の1)で同等以上の精度に到達し、COCO 物体検出で AP 約44前後、特に小物体の AP も改善したと報告されています。これは全画素を見る代わりに少数のサンプル点を補間で拾うことで、計算量を画素数に線形なオーダーに抑えつつ、必要な位置の情報を高解像度で取れるためです。グローバル注意が画素数の二乗で計算量が増えるのに対し、サンプリング注意は「クエリ数×サンプル点数」に比例するだけなので、高解像度特徴マップを扱えるようになった点が小物体改善の本質です。

補間方式の比較研究では、最近傍補間に比べバイリニアは滑らかで勾配が安定する一方、細かいエッジがぼけるトレードオフが定量的に確認されています。高周波情報が重要なタスク(超解像など)ではバイキュービック(周囲16点を使う3次補間)やより高次の補間が選ばれることもありますが、計算コストとのバランスからバイリニアが事実上の標準です。最近傍は微分が定義できないため、座標を学習する用途では選択肢になりません。

## メリット・トレードオフ・限界
メリットは、連続座標(小数)から滑らかに値を取り出せること、微分可能なのでサンプリング位置やカメラ投影の学習に勾配を流せること、フレームワークに最適化済み実装がありGPUで大量の点を一括処理できること、`padding_mode='zeros'` で枠外を自然に0扱いでき可視性処理と相性が良いことです。

トレードオフと限界もあります。バイリニアは4点の線形混ぜなので、細かいエッジや高周波の情報が少しぼけます。座標規約([-1,1]、(x,y)順、align_corners)を間違えるとサンプリング位置が静かにずれ、損失が下がらないのに原因が分からない、という見つけにくいバグになりがちです。特徴マップの解像度(H, W)が低いと、いくら滑らかに補間しても元の情報が粗いので限界があります(投影が正確でも、特徴マップが粗ければ細部は復元できません)。また、多数の点を多数の特徴マップから拾うとメモリ帯域がボトルネックになりやすく、高解像度BEVや多カメラ・多時刻ではメモリと計算が膨らみます。勾配の観点では、サンプル点が局所4点しか見ないため、初期位置が正解から大きくずれていると勾配が「正しい方向」を指さず、局所解にはまることがあります(これが学習オフセットを小さなスケールに抑える理由の一つ)。`align_corners` の True/False や座標の正規化基準がフレームワークやバージョンで微妙に異なる点も、移植時の落とし穴です。

## 発展トピック・研究の最前線
発展方向として、より高次・高品質な微分可能サンプリング(バイキュービックや学習可能カーネル)や、エッジを保つ補間でぼけを抑える研究があります。3次元への拡張として、ボリュームから連続座標で値を引く3次元 grid_sample(占有グリッドや NeRF 系の微分可能サンプリング)が重要で、BEV を高さ方向にも解像度を持つボクセルへ拡張する占有予測タスクで多用されます。効率面では、Deformable DETR の専用カーネルのように、注意とサンプリングを融合した最適化実装や、メモリを節約する fused 実装が継続的に開発されています。さらに、サンプリング位置の不確実性を扱う確率的サンプリングや、連続関数として特徴を表現し座標を入れると値が返る暗黙表現(implicit neural representation)など、「格子の補間」から「連続関数の評価」へと一般化する流れもあり、grid_sample はその出発点として位置づけられます。混合精度学習(bf16/fp16)での grid_sample の数値安定性や、座標を高精度(fp32)で保ちつつ特徴を低精度で扱うといった実装上の工夫も、大規模学習では実務的な論点になっています。
