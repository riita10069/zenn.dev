# Temporal Memory プラグイン

## ひとことで言うと
過去の時系列(時間方向に並んだ特徴の列)を「どう要約してモデルに渡すか」という処理を、モデル本体から切り離して差し替え可能な部品にする設計パターンです。最新フレームだけを使う最小構成から、過去をゲート付き回帰ネットや注意機構で圧縮する高度な構成まで、同じインターフェース(入力と出力の約束)を守る限り何でも入れ替えられます。狙いは、時系列をどう扱うかを本体コードに触れずに実験的に比較できるようにすることです。

## 直感的な理解
1枚の静止画だけを見て運転の判断をするのは、目を一瞬開けてすぐ閉じる「パッと見」で道路を渡るようなものです。前の車がブレーキを踏み始めたのか、歩行者がこちらに近づいているのか、自分が今加速中なのか減速中なのか — これらは「変化」であって、1枚の絵には写りません。変化を捉えるには「少し前まで」の記録、すなわち時系列の履歴が要ります。

ところが履歴を素朴に全部抱え込むと、別の問題が起きます。仮にカメラが10Hz(1秒に10フレーム)で動き、過去6秒ぶんを持つと、それだけで60フレームの特徴を後段に渡すことになります。後段の軌道予測器はこの巨大な入力を処理しきれず、計算もメモリも重くなります。逆に履歴を雑に捨てると、せっかくの変化情報が失われます。

この「全部抱えると重い、捨てると情報が消える」という板挟みの解き方は1つではありません。タスクや計算予算によって最適な圧縮戦略は変わります。だからこそ「履歴の使い方」を1つに固定せず、複数の戦略を後から差し替えられる「プラグイン」として実装するのが賢いやり方です。これが Temporal Memory プラグインという設計思想です。

## 基礎: 前提となる概念
まず登場する用語を噛み砕きます。

- 特徴ベクトル(feature vector): 画像や状態を、機械学習モデルが扱いやすい数値の並び(ベクトル)に変換したもの。たとえば1枚のカメラ画像を backbone(特徴抽出器)に通すと、数百個の数値からなる1本のベクトルになります。これが「そのフレームの要約」です。
- テンソルの形(shape): 多次元配列の大きさを表す記法。`[B, T, feat]` は「Bはバッチサイズ(同時に処理するサンプル数)、Tはタイムステップ数(過去何フレーム分か)、featは1フレームあたりの特徴次元数」を意味します。たとえば `[8, 60, 896]` なら、8サンプルを同時に、それぞれ60フレームの履歴を、1フレーム896次元で持っている、という意味です。
- 文脈ベクトル(context vector): 時間方向にたたんで(要約して)得られる、過去全体を代表する1本のベクトル。形は `[B, feat]` で、時間軸Tが消えています。「過去の流れの要点」を1本に凝縮したものです。
- ダウンサンプリング(downsampling / 粗視化): データを間引いて解像度を下げること。時間方向のダウンサンプリングは「10フレームを1フレームにまとめる」のように、時間の細かさを粗くすることを指します。
- 因果性(causality): 時刻 t の判断に未来(t より後)の情報を使わないこと。運転では未来は観測できないので、時間メモリは因果的でなければなりません。これは動画認識(後付けで全フレームを使える)との大きな違いです。

そして時系列を要約する代表的な道具立てが3つあります。

1. 回帰ネット(RNN / GRU / LSTM): 系列を先頭から1ステップずつ読み、内部状態を更新していき、最後の状態に系列全体の要約を残すネットワーク。逐次的かつ因果的。
2. 畳み込み(1D Convolution): 時間方向に窓をすべらせて局所的なパターンをまとめる。ストライド(すべらせ幅)を大きくすると同時にダウンサンプリングもできる。
3. 注意機構(Attention / Transformer): 全フレーム同士の関連を一度に見て、重要なフレームに重みをつけて要約する。並列計算でき長距離依存に強い。

Temporal Memory プラグインは、これらのどれを使うかを差し替え可能にする「箱」です。

## 仕組みを詳しく
このパターンの核心は「共通インターフェース + レジストリ + 複数の実装」の3点セットです。

### 共通インターフェース(契約)
すべての時間メモリ実装は、次の同じ約束を守ります。

```
forward(visual_history, egomotion_history) -> (visual_context, egomotion_context)
  入力: visual_history    [B, T, visual_dim]   または [B, visual_dim]
        egomotion_history [B, T, egomotion_dim] または [B, egomotion_dim]
  出力: visual_context    [B, visual_dim]
        egomotion_context [B, egomotion_dim]
```

「時間方向に並んだ履歴 `[B, T, feat]` を受け取り、時間をたたんだ文脈 `[B, feat]` を返す」という契約です。入力が既に2次元 `[B, feat]`(時間軸なし)で来た場合は、そのまま素通しするフォールバックを持つのが普通です。本体は中身を一切気にせず、この契約だけを頼りに部品を呼びます。これが「疎結合」、すなわち部品同士の依存を薄く保つ設計です([kwargs転送によるモジュール疎結合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0973_kwargs-forwarding-design) と表裏一体)。

### レジストリ(名前で部品を選ぶ仕組み)
```
REGISTRY = {
    "no_memory": NoMemory,
    "one_hz":    OneHzHistoryEncoder,
    # 将来: "transformer": TransformerMemory, "mamba": MambaMemory ... と追加可能
}
build_temporal_memory(memory_mode, **kwargs)  # 文字列キーから実体を構築
```
文字列のキー(`"no_memory"` など)で実装を選べるので、設定ファイルやコマンドライン引数1つで戦略を切り替えられます。新しい戦略はレジストリに1行足すだけで、本体コードもデータ読み込みも変えずに試せます。

### 実装1: 履歴を使わないベースライン(no_memory)
パラメータを一切持たない素通しです。入力が3次元 `[B, T, feat]` なら最新フレーム `[:, -1]` だけを取り出し、すでに2次元 `[B, feat]` ならそのまま返します。

```
入力 visual_history [8, 60, 896]
  → 最新フレームを抽出 [:, -1]
  → 出力 visual_context [8, 896]
```

「履歴を使わない」という最小の基準線(ベースライン)です。一見無意味に見えますが、研究上は極めて重要です。「履歴を入れたら本当に性能が上がるのか?」を測るには、履歴なしの対照群が要るからです。これがないと改良の効果を切り分けられません。

### 実装2: 時間圧縮エンコーダ(one_hz)
「10Hzの長く細い履歴は、後段に渡すと1フレームあたりの情報が薄くて扱いづらい」という典型的な課題への対処です。合言葉は「時間方向は粗く、特徴方向は厚く」。10Hzの細長い履歴を、1Hz程度の短く厚い表現に変換します。処理は2段構成です。

段1 時間圧縮(1D Convolution):
```
Conv1d(input_dim, hidden_dim, kernel_size=10, stride=10)
```
カーネル幅10・ストライド10の重なりなし畳み込みで、10ステップを1ステップにまとめます(10Hz→1Hz)。同時に出力チャネルを `hidden_dim` に拡張して特徴を厚くします。Tが10の倍数でないときは古い側(left-trim)を捨て、最新フレームを必ず残すのが定石です。続いて GELU(なめらかな活性化関数)と LayerNorm(層正規化、数値を整える処理)を通します。
```
例: T=64, 10Hz, 圧縮率10
  [B, 64, feat] → 古い側4ステップを捨てて [B, 60, feat]
              → Conv1d で [B, hidden_dim, 6]  (約1Hzで6秒分)
```

段2 系列要約(GRU):
圧縮後の短い系列を GRU(ゲート付き回帰ネット)に通し、最後の隠れ状態を文脈ベクトル `[B, hidden_dim]` として取り出します。GRU は系列を順に読みながら「何を覚え、何を忘れるか」をゲートで制御し、最後の状態に系列全体の要約を残します。GRU の更新ゲート $z_t = \sigma(W_z x_t + U_z h_{t-1})$ は「前の隠れ状態 $h_{t-1}$ をどれだけ保持し、新しい候補 $\tilde{h}_t$ をどれだけ取り込むか」を0〜1で決めます($\sigma$ はシグモイド、$x_t$ は時刻 t の入力、$h$ は隠れ状態)。

視覚履歴(例 896次元)と自車運動履歴(例 256次元)を特徴方向に連結して合計1152次元にまとめて圧縮し、出力をまた視覚分・運動分に切り分けて返す実装もよく見られます。これは「視覚と運動は相関するので一緒に符号化したほうが効率的」という発想です。

なぜこの設計なのか。時間を粗くする(10→1)ことで後段の系列長が1/10になり計算が軽くなる一方、チャネルを厚くする(896→hidden_dim)ことで1ステップあたりの情報量を保ちます。情報を「時間軸から特徴軸へ移し替える」イメージです。

## 手法の系譜と主要論文
時系列をどう要約してモデルに渡すかは、系列モデリングの中心テーマで、長い研究の蓄積があります。

- LSTM(Hochreiter & Schmidhuber, 1997, Neural Computation)。RNN の勾配消失問題(長い系列で過去の情報が伝わらなくなる現象)を、記憶セルと入力・忘却・出力の3ゲートで解決しました。長期依存を扱える初の実用的な回帰ネットです。動機は「素朴な RNN は数十ステップ先までしか覚えられない」こと。トレードオフはパラメータが多く重いことです。

- GRU(Cho et al., 2014, EMNLP, "Learning Phrase Representations using RNN Encoder-Decoder")。LSTM を簡略化し、ゲートを2つ(更新ゲート・リセットゲート)に減らした軽量版です。多くのタスクで LSTM と同等の性能を、より少ないパラメータで出すことが後続研究で示されました。特徴圧縮後の短い系列をまとめる用途に今も広く使われます。

- ゲート付き RNN の実証比較(Chung et al., 2014, NeurIPS Deep Learning Workshop, "Empirical Evaluation of Gated Recurrent Neural Networks on Sequence Modeling")。GRU と LSTM を音楽・音声モデリングで横並び比較し、GRU が LSTM とほぼ同等の負の対数尤度を、より少ないパラメータで達成すると報告しました。「軽くて同等」という GRU 評価の根拠です。

- SlowFast Networks(Feichtenhofer et al., 2019, ICCV, FAIR)。動画認識で「ゆっくり見る経路(低フレームレート・高チャネル)」と「速く見る経路(高フレームレート・低チャネル)」の2経路に分けました。意味的な変化はゆっくり、動きの詳細は速く、と別々に扱うのが効率的という洞察です。「時間を粗く・特徴を厚く」する one_hz 流の圧縮は、この Slow 経路の発想の直系にあたります。

- Long-Term Feature Banks(Wu et al., 2019, CVPR)。長時間の文脈を「特徴の銀行(feature bank)」として保持し、現在の判断時に注意機構で参照する枠組みを提案しました。回帰ネットで逐次たたみ込む代わりに、過去の特徴を保管しておいて必要なときに引き出すという、メモリ機構の別解です。

- ChauffeurNet(Bansal et al., 2019, RSS, Waymo)。運転の模倣学習で、過去の自車状態と周辺エージェントの履歴を条件として与えると判断が安定すると報告しました。同時に「過去への過適合」、すなわち過去の軌跡をなぞるだけになってしまう危険(コピー問題、inertia problem)も指摘しています。これが、履歴なしベースライン(no_memory)を対照群として残す動機の一つです。

- Transformer 系の時間モジュール(Vaswani et al., 2017, NeurIPS 以降)。自己注意で全フレーム同士の関連を一度に見るため、長距離依存に強く並列計算もできます。近年の運転モデルは時間融合に注意機構を採る例が増えています。レジストリ方式なら、こうした新方式も本体無改変で追加できます。

- 状態空間モデル Mamba(Gu & Dao, 2023, arXiv:2312.00752)。RNN の逐次性(遅い)と Transformer の二乗計算量(長系列で重い)の両方を回避し、長系列を線形計算量・因果的に扱える新方式。運転のような長く因果的な時系列に親和性が高く、時間メモリの最新の選択肢です。

系譜をまとめると、逐次処理の RNN/LSTM/GRU から始まり、動画分野で「時間の粗視化」(SlowFast)や「特徴の保管庫」(Feature Bank)が登場し、注意機構による並列的な時間融合を経て、現在は状態空間モデルへと選択肢が広がっています。Temporal Memory プラグインは、この多様な選択肢を1つに固定せず比較可能にする「器」です。

## 論文の実験結果(定量データ)
- SlowFast(ICCV 2019)は動画行動認識ベンチマーク Kinetics-400 で、当時の単一モデル最高水準を更新しました。SlowFast R101+NL は top-1 精度(正解クラスを1位に当てる割合)で約79.8%を達成し、Slow 経路単独より明確に高い結果でした。アブレーション(要素を抜く実験)では、Fast 経路(高フレームレート・低チャネル)を取り除くと精度がおよそ1〜3ポイント低下し、「速い動きを別経路で捉える」ことの寄与が定量的に示されています。Fast 経路の計算コストは全体の約20%程度に抑えられており、「安いコストで効く」点も報告されています。「時間と特徴を分離して扱う」設計の有効性を裏づける結果です。

- GRU vs LSTM(Chung et al., 2014)では、音楽モデリング(Nottingham, JSB Chorales など)と Ubisoft 音声データセットで、GRU が LSTM とほぼ同等の負の対数尤度(値が小さいほど良い予測)を、より少ないパラメータと短い学習時間で達成すると報告されました。両者とも素朴な tanh-RNN を明確に上回り、「ゲートが効く」ことも同時に示されています。

- Long-Term Feature Banks(Wu et al., 2019)は AVA 行動検出で、短期文脈のみのベースラインに対し長期特徴銀行を足すと mAP(mean Average Precision、検出精度、高いほど良い)がおよそ数ポイント向上すると報告しました。「長い文脈を保管庫として参照する」価値の定量的根拠です。

- 運転模倣学習一般では、履歴を条件に加えると経路追従誤差(予測軌道と正解軌道のズレ)が下がる一方、ChauffeurNet が指摘するように、テスト時に履歴へ過依存すると分布外の状況(学習データにない場面)で破綻しやすくなる、という両面の報告が知られています。ChauffeurNet は過去軌跡の入力を「学習時にドロップアウト(一定確率で隠す)」する対策を導入し、コピー問題による停止スタック(止まったまま動けなくなる)を緩和できたと報告しています。履歴の効果は「平均誤差は下がるが頑健性は下がりうる」という、トレードオフを含む数値で語られます。

これらの指標の意味を補足します。top-1 精度は「最も自信のある予測が当たる割合」で高いほど良い。mAP は検出タスクの総合精度で高いほど良い。ADE(Average Displacement Error)は予測軌道と正解軌道の各時刻の距離の平均、FDE(Final Displacement Error)は終端時刻の距離で、いずれもメートル単位で小さいほど良い指標です。

## メリット・トレードオフ・限界
メリット
- 履歴の使い方を本体コードに触れずキー1つで差し替えられ、効果を実験で定量比較できる。
- 共通インターフェース(`[B,T,feat]`→`[B,feat]`)を守るため本体は中身を知らずに済む(疎結合)。
- 時間圧縮型(one_hz)は「時間を粗く・特徴を厚く」することで、薄い長尺履歴の問題を解消しつつ計算量も抑える。
- 履歴なしベースライン(no_memory)を残すことで、履歴の寄与を切り分けて検証できる。

トレードオフ・限界
- 固定圧縮率(例 10ステップ→1ステップ)の畳み込みは、その窓内の細かい時間変化を不可逆に捨てる。急ブレーキのような短時間イベントを取りこぼす恐れがある。
- GRU は逐次処理のため並列性が低く、非常に長い履歴では記憶保持と速度の両面で限界がある。
- 履歴に依存しすぎると過去をなぞるだけの挙動(コピー問題)になりうる。これは模倣学習一般の根深い弱点で、因果混同(causal confusion)とも呼ばれる。
- 時間圧縮型は履歴長Tが圧縮率以上であることを要求し、短すぎる履歴では破綻する。可変長履歴への対応は別途設計が要る。
- 運転では因果性が必須のため、未来フレームを参照する動画認識由来の手法をそのまま使えない場合がある。

## 発展トピック・研究の最前線
- 注意機構ベースの時間融合: 固定窓の畳み込みではなく、状況に応じて関連フレームに動的に重みをつける方式。長距離依存と可変長に強い。BEV 表現上で時間的自己注意を行う流れ([BEVFormer（時空間BEVクエリ）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1056_bevformer-spatiotemporal))が代表例。
- 状態空間モデル(S4 / Mamba など、2022年以降): 長系列を線形計算量で因果的に扱える新しい系列モデル。RNN の逐次性と Transformer の計算量の両方を克服しようとする最前線。
- メモリバンク/外部記憶: 過去の特徴を明示的に保管し、必要なときだけ参照する設計。長時間の文脈を扱いつつ計算を抑える方向。
- 因果性とリアルタイム制約: 運転では未来フレームを使えない(因果的でなければならない)うえ、推論遅延が安全に直結する。時間モジュールの設計は精度だけでなくレイテンシとの綱引きになる、という運転特有の制約が研究課題として残る。
