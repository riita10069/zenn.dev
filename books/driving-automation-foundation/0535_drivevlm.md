# DriveVLM

## ひとことで言うと
DriveVLM は、運転シーンを「言葉で順を追って考える(Chain-of-Thought 推論)」VLM(Vision-Language Model、画像と言葉を扱うAI)です。シーンの説明 → 状況の分析 → 段階的な計画、という人間の思考に似た手順で運転計画を立てます。さらに、賢いが遅い VLM と、速いが浅い従来プランナを組み合わせる「System1/System2 併用」の代表例で、これにより実時間性と高度な推論を両立させようとしました。「難しい場面だけ、じっくり言葉で考える脳を呼び出す」という発想が核心です。

## 直感的な理解

熟練ドライバーの頭の中を想像してください。慣れた直線道路では、ほとんど無意識に(考えずに)運転しています。ところが「工事で車線が変わり、作業員が手信号で誘導している」ような場面に出くわすと、急に意識を集中させて「あの人は止まれと言っているのか、進めと言っているのか」「向こうの車はどう動くか」とじっくり考え始めます。人間はこの2つのモードを自然に使い分けています。

この使い分けは認知科学の「2つの思考システム」(Kahneman, 2011)として知られます。System1 は速くて直感的な思考(慣れた道を無意識に運転する)、System2 は遅くて熟慮的な思考(難しい状況をじっくり考える)です。DriveVLM(Tian et al., CoRL 2024)は、このアイデアを運転システムに持ち込み、「遅いが賢い VLM(System2)」と「速いが浅い従来プランナ(System1)」を協調させることで、両者のいいとこ取りを狙いました。

なぜそんな工夫が要るのでしょうか。自動運転で本当に難しいのは、頻繁には起きない複雑なシーン(long-tail、希少な状況)です。「道路工事で作業員が手信号で誘導」「前のトラックから荷物が落ちそう」といった状況は、従来の知覚→予測→計画パイプラインが苦手とします。これらは表面的な物体検出だけでなく、状況の意味を理解し、常識で先を読む必要があるからです。

VLM はこの「意味理解と推論」が得意です。しかし VLM には致命的な弱点があります。重くて遅いことです。LLM はトークンを1つずつ生成する(自己回帰生成)ため、運転に必要な毎秒10回以上の判断には到底間に合いません。そこで「普段は速いプランナで走り、難所だけ重い VLM を呼ぶ」という役割分担が生まれます。

## 基礎: 前提となる概念

DriveVLM を理解する前提を噛み砕きます。

VLM(Vision-Language Model): 画像と言葉を同時に扱える大規模モデルです。画像を「視覚トークン」に変換して言語モデルに入力し、画像について質問したり説明させたりできます。運転では、カメラ映像を見せて「このシーンで何に注意すべきか」を言葉で答えさせる、といった使い方をします。

Chain-of-Thought(CoT、思考の連鎖)(Wei et al., NeurIPS 2022): 答えをいきなり出すのではなく、途中の考えを段階的に言葉にしてから結論に至る推論方式です。算数の文章題で「まず…次に…だから答えは」と途中式を書くのと同じで、これにより LLM の推論精度が大きく上がることが知られています。重要な発見は、モデルに「順を追って考えなさい(let's think step by step)」と促すだけで、難しい推論タスクの正答率が跳ね上がる、ということでした。DriveVLM はこれを運転計画に構造的に組み込みます。

メタアクション(meta-action): 「減速して待つ」「加速して追い越す」「停止する」といった、具体的な軌道より一段抽象的な行動方針のことです。人間も「とりあえず止まろう」と方針を決めてから、ハンドルとブレーキの細かい操作を決めます。

nuScenes(Caesar et al., CVPR 2020): 都市部の実走行データを集めた大規模データセット。6台のカメラ・LiDAR・レーダーを備え、3D 物体検出や軌道予測、計画の評価に広く使われる実車ベースのベンチマークです。

これらを踏まえると、DriveVLM は「CoT で運転計画を段階化した VLM(System2)を、速い従来プランナ(System1)と組み合わせる」構成だと理解できます。

## 仕組みを詳しく

DriveVLM の核は、CoT を運転計画に構造化した3段階の推論です。

ステップ1: Scene Description(シーン記述)。カメラ映像を見て、シーン全体を言葉で記述します。天候・時間帯・道路種別といった環境条件と、運転に影響する重要物体(前方のトラック、横断中の歩行者など)を列挙します。「何があるか」を言語化する段階です。

ステップ2: Scene Analysis(シーン分析)。列挙した重要物体それぞれについて、「いまどんな状態で、自車にどう影響するか」を分析します。たとえば「前方のトラックが減速している → 自車も減速が必要かもしれない」のように、物体の特性(C: characteristics)・現在の挙動・自車への影響を推論します。「それが何を意味するか」を考える段階です。

ステップ3: Hierarchical Planning(階層的計画)。分析結果をもとに、段階的に計画を具体化します。まず大まかな行動方針(メタアクション、例「減速して待つ」)を決め、次にそれを言葉での運転判断(decision description)に落とし、最後に具体的な軌道(waypoint の列)を出力します。粗い方針から細かい軌道へ、階層的に降りていく段階です。

```
映像 ─→ ①Scene Description  「前方にトラック、右に横断歩行者」
        ②Scene Analysis     「トラック減速中→追従減速。歩行者→停止検討」
        ③Hierarchical Plan  メタ行動「減速して停止」→判断文→軌道(waypoints)
```

なぜこの3段階に分けるのでしょうか。一気に「映像→軌道」を出させるより、間に「言葉での記述・分析」を挟んだほうが、(a) 推論精度が上がり(CoT 効果)、(b) どこで判断を誤ったかを人間が追える(各段階の自然言語出力が説明になる)からです。中間出力が読めることは、安全性検証やデバッグで大きな価値を持ちます。

そして DriveVLM-Dual という拡張版では、この遅い VLM(System2)を、従来の高速な 3D 知覚・軌道計画モジュール(System1)と組み合わせます。

- 速い従来プランナが、低遅延で常時おおまかな軌道を出し続ける(実時間性を確保)。
- 遅い VLM が、難しいシーンに対して時間をかけて高度な推論を行い、その結果(重要物体やメタアクション)を取り込んで軌道を洗練する。

これにより、普段は速いプランナで滑らかに走り、難所では VLM の深い理解で賢く対処する、という役割分担が成立します。VLM は毎フレーム走らせる必要がなく、低い頻度で呼び出せばよいので、計算負荷が現実的になります。具体的な統合では、VLM が出した重要物体やメタアクションを、速いプランナの軌道生成に「事前情報」として注入する形をとります。速いプランナはこの追加情報を手がかりに軌道を補正するため、VLM の推論が数フレームに1回しか更新されなくても、その間は速いプランナが直近の VLM 出力を参照しながら滑らかに走り続けられます。VLM の推論遅延(数百ミリ秒オーダー)が、毎時刻の制御遅延に直結しないのがこの設計の肝です。

## 手法の系譜と主要論文

DriveVLM(Tian, Gu, Li, Liu ら, "DriveVLM: The Convergence of Autonomous Driving and Large Vision-Language Models", CoRL 2024)の位置づけを系譜で整理します。

- 推論手法の源流: Chain-of-Thought Prompting(Wei et al., NeurIPS 2022)が、段階的に考えさせると LLM の推論精度が上がることを示しました。DriveVLM はこれを運転計画という構造化されたタスクに適用しました。

- 思想の源流: System1/System2 という枠組み自体は認知科学(Kahneman, 2011)由来で、AI に「速い直感」と「遅い熟慮」を分担させる発想は近年さまざまな分野で再注目されています。

- 先行する LLM 運転研究との関係:
  - DriveGPT4(Xu et al., 2023): 単一 LLM で説明と制御を出す先駆け。DriveVLM は CoT 構造化と System1/System2 併用で実時間性を補強した。[DriveGPT4](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0533_drivegpt4)。
  - LMDrive(Shao et al., CVPR 2024): 言語指示追従と CARLA 閉ループに焦点。DriveVLM は実車寄りの推論品質に焦点で、狙いが異なる。[LMDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0541_lmdrive)。

- DriveVLM の新規性: シーン記述→分析→階層計画の CoT を持つ運転特化 VLM と、それを高速従来プランナと併用する DriveVLM-Dual(System1/System2 構成)を提案したこと。さらに long-tail シーン評価用データセット SUP-AD(Scene Understanding for Planning、希少・複雑シーンを集めた注釈付きデータ)を提案し、難所の評価を可能にしたこと。

- 並走・後続の研究: DualAD をはじめとする fast-slow / dual-system 系の運転研究や、EMMA([EMMA (End-to-End Multimodal Model for Autonomous Driving)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0536_emma_note))のような大規模基盤モデルによる運転が、この後の流れを形成しています。

この系譜は、「VLM の深い推論力」と「実時間運転の制約」という相反する要求を、CoT による構造化と System1/System2 の役割分担で和解させた点に、DriveVLM の歴史的意義があることを示します。

## 論文の実験結果(定量データ)

DriveVLM は、著者らの構築した SUP-AD と、実車ベンチマークの nuScenes で評価されました。指標を初心者向けに説明します。

nuScenes での計画(planning)評価の主指標:
- L2 誤差(L2 error, メートル): モデルが予測した将来軌道が、実際に人間が走った軌道からどれだけ離れているか(典型的に1秒後・2秒後・3秒後の平均距離)。小さいほど良い。
- Collision Rate(衝突率, %): 予測軌道に沿って走ったとき、他物体と衝突する割合。小さいほど安全。

DriveVLM-Dual は、nuScenes の計画タスクで、L2 誤差と衝突率の両方を従来の計画手法に対して改善したと報告しました。報告値では、1秒/2秒/3秒の平均 L2 誤差がおよそ 0.4m / 0.7m / 1.0m 前後、平均でおよそ 0.7m 前後とされ、当時の強力な end-to-end プランナ(UniAD や VAD など、平均 L2 がおよそ 1.0m 前後)を下回る(=より正確な)水準に達したとされます(数値は論文・後続比較による)。平均衝突率も 0.1% 台と低く抑えられています。1秒後の数十センチの差は、車線内に収まるか縁石へ寄るかを分ける大きさで、この指標が小さいほど人間の運転に忠実な軌道を出せていることを意味します。

とくに VLM 単体ではなく、速い従来プランナと組み合わせた Dual 構成のほうが、軌道の滑らかさと精度が両立し、L2 誤差が下がることを示しています。これは「VLM だけだと数値精度の細かさで従来プランナに劣るが、両者を組み合わせると互いの弱点を補える」という、System1/System2 構成の有効性を裏づける数値です。

SUP-AD(long-tail シーン)での評価:
- 著者らは GPT などを審査員にした採点や、シーン記述・分析・メタアクションの妥当性評価を用い、複雑・希少シーンでの計画品質が向上することを示しました。表面的な物体検出では捉えられない「意味理解が必要なシーン」で、CoT を持つ VLM が従来パイプラインより適切な計画を立てられることを定性・定量の両面で報告しています。

アブレーション(どの要素を抜くとどうなるか)の代表的な知見:
- CoT の中間ステップ(Scene Description / Analysis)を省くと、Hierarchical Planning の品質が落ちます。「いきなり計画を出す」より「記述・分析を経る」ほうが計画が良くなる、という CoT 効果が運転でも確認されました。
- System1(速いプランナ)を外して VLM 単体にすると、実時間性が失われるだけでなく、軌道の数値精度(L2)も Dual 構成に劣ります。逆に System2(VLM)を外すと、希少・複雑シーンへの対応力が落ちます。両者がそれぞれ別の弱点を補っていることが分かります。

これらの数値が伝える本質は、「VLM を運転に使うか否か」ではなく、「VLM をどの頻度で・どう従来プランナと組み合わせるか」が実用性能を決める、ということです。

## メリット・トレードオフ・限界

メリット

- CoT により、複雑・希少シーンを段階的に深く推論でき、計画の質が上がる。
- System1/System2 併用で、実時間性(速いプランナ)と高度推論(VLM)を両立できる。VLM を低頻度でしか呼ばないので計算が現実的。
- 中間出力が自然言語なので、なぜその計画になったかを人間が追える(解釈可能性)。事故分析やデバッグに有用。
- long-tail シーン評価データセット SUP-AD を提供し、従来見過ごされてきた難所の評価を可能にした。

トレードオフ・限界

- 2系統(VLM と従来プランナ)を協調させる設計が複雑で、両者の出力をどう統合・調停するか(どちらを優先するか、どう融合するか)のチューニングが難しい。
- VLM 部分は依然として重く、車載ハードウェアで常時稼働させるのは厳しい。呼び出し頻度の設計が性能と負荷のトレードオフになる。
- CoT の中間ステップ(記述・分析)で誤ると、その誤りが最終計画まで伝播する(error propagation)。最初の物体の見落としが、計画全体を狂わせうる。
- 主にオフライン(nuScenes の録画データ)/シミュレータ評価で、実車での長期・閉ループ安全性検証は今後の課題。

研究上の未解決課題: System1 と System2 の「いつ・どちらを信じるか」の調停は本質的に難しい問題です。速いプランナと遅い VLM の判断が食い違ったとき、どちらを採用すべきか、その判断自体を誰がするのか、は明確な答えがありません。また VLM の幻覚(hallucination、もっともらしい誤り)が安全に直結する点、CoT 説明が本当にモデルの内部判断を反映しているか(忠実性)という問題も、DriveGPT4 と共通して未解決です。

## 発展トピック・研究の最前線

- fast-slow / dual-system の精緻化: VLM をいつ呼ぶか(難所をどう検出するか)、System1/System2 の出力をどう融合するかの最適化が活発な研究テーマです。
- 蒸留による軽量化: 重い VLM の推論結果(重要物体・メタアクション・シーン理解)を軽量モデルへ蒸留し、推論コストを抑えつつ System2 的な能力を持たせる方向。実用化に向けた有力路線で、自動運転で広く検討されています。詳細は [知識蒸留損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0633_knowledge-distillation-loss)。
- 大規模基盤モデルへの統合: EMMA のように、知覚・予測・計画を巨大なマルチモーダルモデルで統一する方向([EMMA (End-to-End Multimodal Model for Autonomous Driving)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0536_emma_note))。
- 評価ベンチマークの整備: SUP-AD、DriveLM、nuScenes-QA など、運転シーンの意味理解と計画を体系的に評価する基盤が整いつつあります([DriveLM (Graph VQA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0534_drivelm))。
- 閉ループでの検証: オフライン評価を超え、実時間の閉ループ環境で System1/System2 構成を検証する研究([LMDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0541_lmdrive) [CARLA(閉ループシミュレーション)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0715_closed-loop-simulation-carla))。
