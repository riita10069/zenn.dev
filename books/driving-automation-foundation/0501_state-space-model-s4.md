# S4 (Structured State Space)

## ひとことで言うと
S4 は「とても長い時系列データ」を効率よく学習するためのニューラルネットワークの部品です。たとえば 1 万ステップを超えるセンサー記録や、1 ピクセルずつ並べた画像のような長い系列でも、計算を破綻させずに「ずっと前に起きたこと」を覚えていられるように設計されています。RNN(過去を 1 ステップずつ覚えていくネットワーク)と CNN(まとめて並列計算するネットワーク)の両方の良いところを、状態空間モデルという古典的な数式の上で数学的に統合したのが核心です。学習時は CNN のように並列で速く、推論時は RNN のように一定メモリで逐次的に動かせます。

## 直感的な理解
時系列(time series、時間に沿って並んだデータ、例: 毎秒のカメラ画像や車速の系列)を扱うとき、機械学習には長年つきまとう困りごとがあります。それは「遠い過去の情報をどう覚えておくか」という問題で、専門用語では長距離依存(long-range dependency、遠く離れた時点どうしの関係)と呼びます。

具体例で考えます。自動運転車が 3 秒前に見た「赤信号」を、今この瞬間の判断に使いたいとします。1 秒間に 10 フレーム処理するなら 3 秒前は 30 ステップ前です。これくらいならまだ何とかなりますが、「30 秒前に駐車場で見た標識」のように何百ステップも前の情報が必要になると急に難しくなります。あるいは DNA 配列のように 1 本が数万〜数十万塩基もある系列、音声波形のように 1 秒で 1.6 万サンプルにもなる系列を考えると、「全部覚えておく」のはどんどん大変になります。

人間で言えば、長い小説を読みながら「序盤で出た伏線」を終盤まで覚えておくようなものです。素朴な記憶法では途中で忘れたり、全部を読み返すのに時間がかかったりします。S4 は、この「長い文脈を、忘れずに、しかも速く処理する」ための仕組みを与えてくれる部品だと思ってください。

## 基礎: 前提となる概念
S4 を理解するには、まず系列モデリングの 3 つの古典的アプローチと、その弱点を押さえる必要があります。

- RNN(Recurrent Neural Network、再帰型ニューラルネットワーク): 隠れ状態(hidden state、過去の要約を詰めたベクトル)を 1 ステップずつ更新していく方式です。理屈の上ではいくらでも過去を覚えられますが、実際には勾配消失(vanishing gradient、誤差を過去にさかのぼって伝える信号が、何十ステップもかけ算されるうちに 0 に近づいて消える現象)が起きて、遠い過去をほとんど学べません。さらに 1 ステップずつ順番にしか計算できないので、GPU での並列計算が効かず学習が遅いです。
- CNN(Convolutional Neural Network、畳み込みネットワーク): 並列計算は得意ですが、一度に見られる範囲(受容野、receptive field、出力 1 点が入力のどの範囲を見ているか)がカーネルサイズに縛られます。長距離依存を見たければ層を深く積むしかなく、効率が悪いです。
- Transformer(自己注意で全時点を見るモデル): どの時点どうしも直接つなげられて長距離依存に強い反面、系列長 L に対して計算量が L の 2 乗(O(L^2))でふくらみます。L=16000 なら注意行列が 16000×16000 ≒ 2.5 億要素にもなり、長系列では現実的でなくなります。

次に「状態空間モデル(State Space Model, SSM)」という言葉を噛み砕きます。これはもともと制御工学(ロケットや工場の自動制御)や信号処理で何十年も使われてきた古典的な数式の枠組みで、「観測できる入力と出力の裏に、目に見えない内部状態がある」と考えるものです。たとえば車の動きを「アクセル踏み込み量(入力)」と「速度計の値(出力)」だけでなく、その裏にある「実際の運動エネルギーや車輪の回転(内部状態)」で記述するイメージです。S4 はこの古典理論を、ニューラルネットワークの学習可能な層として蘇らせたものです。

## 仕組みを詳しく
S4 の核は、連続時間の状態空間モデルです。1 次元の入力 u(t)(スカラー、たとえばマイク 1 本の音圧)から出力 y(t) を作る過程を、内部状態 x(t)(N 次元のベクトル)を介して次の 2 本の式で書きます。

```
x'(t) = A x(t) + B u(t)      … 状態が時間とともにどう変化するか
y(t)  = C x(t) + D u(t)      … 状態から出力をどう読み出すか
```

ここで A は N×N 行列(状態の時間発展の仕方を決める)、B は N×1(入力を状態に取り込む)、C は 1×N(状態から出力を読み出す)、D はスカラー(入力を出力に直結させるスキップ結合)です。x'(t) は x(t) の時間微分(変化の速さ)を意味します。直感的には、x(t) は「これまでの入力の要約」を貯めておくメモリで、A がそのメモリを少しずつ書き換え、B が新しい入力を取り込み、C が必要な部分を取り出して出力にします。D は実用上 0 やスキップ接続として扱われることが多いです。

ニューラルネットで使うには、連続時間を離散化(discretize、1 ステップごとの差分式に直すこと)します。刻み幅 Δ を使い、A, B から離散版の Ā, B̄ を作ると、次のような漸化式になります。

```
x_k = Ā x_{k-1} + B̄ u_k
y_k = C x_k
```

ここが S4 の最大の発明です。この同じ計算は、まったく異なる 2 通りの形に書けます。

1. 漸化式(リカレンス)として: 1 ステップずつ x_k を更新していく形。推論時に 1 サンプルずつ入ってくる状況(自動運転やストリーミングのリアルタイム処理)に最適で、メモリは状態 1 個ぶん(N 次元)で一定です。
2. 畳み込み(コンボリューション)として: 上の漸化式を展開すると、出力 y は入力 u と「ある長いカーネル K̄」の畳み込みになります。K̄ = (C B̄, C Ā B̄, C Ā² B̄, …, C Ā^{L-1} B̄) という形で、これは長さ L の巨大な畳み込み核です。学習時はこのカーネルを一度作って FFT(高速フーリエ変換)で一気に畳み込めるので、GPU 並列でとても速いです。

つまり「学習は CNN(並列・速い)、推論は RNN(逐次・省メモリ)」という二面性を、同じパラメータで切り替えられます。これは Transformer にも純粋な RNN にもない性質です。

長距離依存に強い秘密は行列 A の選び方にあります。素朴に A をランダムに初期化すると、何十・何百ステップもかけ算するうちに状態が指数的に爆発したり 0 に消えたりして、長距離依存を学べません。これは RNN の勾配消失/爆発とまったく同じ病で、A^k(A の k 乗)の固有値が 1 より大きければ発散、小さければ消滅するという、線形漸化式の安定性の問題です。S4 は HiPPO(High-order Polynomial Projection Operator、高次多項式射影作用素)という理論に基づいた特別な構造を A に与えてこれを回避します。HiPPO の A は「過去の入力履歴を、ルジャンドル多項式という直交関数の係数として、各時刻で最適に圧縮しつづける」ように設計されています。直交関数とは「互いに重ならない(内積がゼロの)基底関数の集まり」で、フーリエ級数の sin/cos のように、関数を少数の係数の重ね合わせで表すための物差しです。直感的には「過去全体を、少数の係数で最もよく近似する圧縮器」を A が表しており、遠い過去も劣化しにくく覚えられます。HiPPO には測度(過去のどの時点を重視するか)の取り方で複数の変種があり、過去全体を一様に重視する HiPPO-LegS(scaled Legendre)が代表で、これが時間スケールに依存しない安定した記憶を与えます。

実装上の難所は、巨大カーネル K̄ を計算するために Ā のべき乗(Ā, Ā², …, Ā^{L-1})が必要になることです。素朴には L 回の行列積(各回 O(N^2))が要り、合計 O(N^2 L) で不安定かつ遅くなります。S4 はこれを、A を DPLR(Diagonal Plus Low-Rank、対角行列 + 低ランク補正、A = Λ − P Q* の形)という構造に制約し、カーネルの生成関数(各周波数成分をまとめて表す母関数)を Cauchy カーネルという行列の評価に帰着させ、周波数領域(フーリエ変換した後の空間)で一括計算することで、安定かつほぼ O(N + L) で求めるアルゴリズムを与えました。これが「Structured(構造化)」の意味で、HiPPO 行列がまさにこの DPLR 形に分解できることを利用しています。

数値例で規模感をつかみます。系列長 L=16000、状態次元 N=64 のとき、Transformer は注意行列が 16000×16000 ≒ 2.56 億要素ですが、S4 は状態が常に 64 次元のベクトルで、計算量も実質 L に比例(線形)します。メモリで言えば、Transformer は L=16000 で数 GB 規模の注意行列を 1 層ぶんだけで抱えますが、S4 の状態は 64 次元 × 特徴次元ぶんで桁違いに小さく、これが長系列での決定的な差になります。

```
        速度(学習)   長距離依存   推論メモリ   計算量
RNN:    遅い(逐次)   弱い        軽い         O(L)
CNN:    速い(並列)   狭い        ―           O(L·k)
Attn:   速い(並列)   強い        重い         O(L^2)
S4:     速い(並列)   強い        軽い(一定)  O(L)
```

## 手法の系譜と主要論文
S4 は突然生まれたのではなく、状態空間を系列モデリングに使う一連の研究の到達点です。

- Gu, Dao, Ermon, Ré, Rudra, "HiPPO: Recurrent Memory with Optimal Polynomial Projections"(NeurIPS 2020, arXiv:2008.07669)。提案: 過去の系列を直交多項式の係数として最適に圧縮しつづける行列 A の理論。動機: RNN が遠い過去を忘れる問題に数学的な最適解を与えるため。新規性: メモリの最適性を「関数近似の射影」として定式化した点。これが後続の A 初期化の理論的土台になりました。
- Gu et al., "Combining Recurrent, Convolutional, and Continuous-time Models with Linear State-Space Layers (LSSL)"(NeurIPS 2021)。提案: SSM を層として使い、RNN・CNN・連続時間の 3 視点を統合する考え方。動機: 3 つのモデル族を一つの枠組みで説明するため。新規性: S4 の直接の前身だが、A のべき乗計算が重く実用速度に課題が残りました。
- Gu, Goel, Ré, "Efficiently Modeling Long Sequences with Structured State Spaces"(S4, ICLR 2022, arXiv:2111.00396)。提案: A を DPLR に制約し、巨大カーネルを Cauchy カーネルで安定・高速に計算するアルゴリズム。動機: LSSL の計算ボトルネックを解消して SSM を実用化するため。新規性: 構造化により計算を実用速度に乗せ、長距離ベンチマークを初めて広く解いた点。
- Gupta, Gu, Berant, "Diagonal State Spaces are as Effective as Structured State Spaces (DSS)" / Gu et al., "On the Parameterization and Initialization of Diagonal State Space Models (S4D)"(ともに NeurIPS 2022)。提案: A を対角行列だけに簡略化しても性能がほぼ保てることを示した。動機: 元 S4 の低ランク補正と複素演算を取り除いて実装を簡単にするため。新規性: 対角 SSM の理論的な初期化(S4D-Lin / S4D-Inv)を与え、後の Mamba 等の土台になりました。

## 論文の実験結果(定量データ)
S4 の評価で中心になるのが Long Range Arena(LRA, Tay et al. 2021)というベンチマークです。これは「効率的 Transformer」の長距離依存能力を測るために作られた 6 タスクの集合で、テキスト分類・文書マッチング・画像のピクセル系列分類などを含み、系列長は 1000〜16000 に及びます。指標は分類精度(accuracy、正解率。高いほど良い)で、特に Path-X(16384 ステップの画像上で 2 点がつながっているか判定する超長距離タスク)は、それまでほとんどのモデルが偶然当て(2 値分類なので 50%)を超えられない難所でした。

報告された結果のポイントは次の通りです。LRA 6 タスク平均で、S4 は約 86% を達成し、当時の効率的 Transformer 群(Reformer・Linformer・Performer・Linear Transformer 等、おおむね 50〜60% 台)を 20 ポイント以上引き離しました。元の Transformer ですら平均 50% 台にとどまっていたタスク群です。タスク別では、ListOps(階層的な括弧計算、長さ約 2000)で約 59%、テキスト分類(IMDB を文字単位、長さ約 4000)で約 86%、文書マッチング(長さ約 4000 を 2 本)で約 90%、画像(CIFAR-10 を 1 ピクセル列、長さ 1024)で約 88% などと報告されています。とりわけ Path-X(16384 ステップの画像上で 2 点が線でつながっているか判定する超長距離タスク)では、多くの先行モデルが 50%(=解けていない)に張りついていたのに対し、S4 は約 88% を記録し、このタスクを意味のある精度で初めて解いたモデルとして注目されました。差の意味は明快で、「50% は 2 値分類での当てずっぽうと同じ=長距離依存を全く捉えられていない」のに対し、「88% は 1.6 万ステップ離れた 2 点の関係を実際に学習できている」ことを示します。Path-X は通常の Path タスク(長さ 1024)を 16 倍に引き伸ばした難所で、ここで偶然を超えること自体が長距離記憶の有無を切り分ける踏み絵になっています。

長系列の生波形を直接扱うタスクでも強さが示されました。Speech Commands(音声コマンド分類)で、生の音声波形(16000 サンプル)を前処理なしに入力する設定で、S4 は高い精度を保ちつつ、特徴抽出を前提とする手法に匹敵しました。画像でも、CIFAR-10 を 1 ピクセルずつ並べた長さ 1024 の系列分類(sequential CIFAR)で当時の系列モデルの最高水準を記録しています。速度面では、同等タスクで Transformer 系より大幅に高速・省メモリと報告されました。

アブレーション(どの要素を抜くと性能がどう変わるかの分解実験)の知見も重要です。A を HiPPO 由来の構造で初期化するか、ランダムに初期化するかを比べると、ランダム初期化では長距離タスクの精度が大きく落ちます。つまり「長距離依存に効いているのは A の構造化された初期化である」ことが示されました。また DSS/S4D の研究では、低ランク補正を外して A を対角だけにしても LRA の精度がほぼ維持されることが示され、「複雑な低ランク構造は必須ではなく、適切に初期化された対角 SSM で十分」という後続の流れを生みました。

## メリット・トレードオフ・限界
メリット:
- 計算量が系列長に対して線形(O(L))で、超長系列に強い。Transformer の O(L^2) という壁を回避できます。
- 学習は並列(CNN 的)で速く、推論は逐次・一定メモリ(RNN 的)という二面性を、同じ重みで両立できます。ストリーミング推論にそのまま使えます。
- HiPPO による理論的裏付けで、遠い過去の劣化が小さく長距離依存に強いです。

トレードオフ・限界:
- 線形・時不変(LTI, Linear Time-Invariant、入力の中身によらず A, B, C が常に同じ)であるため、「入力内容に応じて注目度や記憶の強さを変える」選択性がありません。重要な情報だけ残し不要を捨てる、という芸当が苦手で、これが後の Mamba 開発の動機になりました。
- 元の S4 は複素数・低ランク補正・特殊な初期化を扱い、実装が難しいです(DSS/S4D で大きく緩和されました)。
- Transformer のような「任意の 2 点を直接コピー・検索する」明示的な機構はなく、厳密な in-context 検索やコピー系タスクでは劣ることがあります。
- 未解決の課題として、安全クリティカル領域での実績の蓄積、画像など本来 2 次元・非系列的なデータへの自然な適用(スキャン順序の任意性)、長期積分での数値安定性などが挙げられます。

## 発展トピック・研究の最前線
S4 の後、状態空間モデルは爆発的に発展しました。第一の方向は単純化で、DSS/S4D による対角化が「複雑な構造は不要」という流れを作りました。第二の方向は選択性の導入で、入力依存にパラメータを変える [Mamba (Selective SSM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0491_mamba-selective-ssm) が「長距離効率と内容ベース推論の両立」を実現し、Transformer の代替候補として大きな注目を集めています。第三の方向は他領域への展開で、画像を系列とみなす Vision Mamba / VMamba、音声・ゲノム・強化学習など、長系列が問題になるあらゆる領域に SSM が広がっています。理論面では Mamba-2 の SSD(State Space Duality)が「注意機構と SSM は同じ枠組みの両面である」ことを示し、両者を統一的に理解・高速化する研究が進んでいます。さらに、SSM と注意を層ごとに混ぜるハイブリッド(Jamba など)が、純粋な SSM の検索の弱さと Transformer の長系列コストを補い合う実用解として登場しています。
