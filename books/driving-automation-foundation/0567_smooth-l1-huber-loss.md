# Smooth-L1 (Huber) 損失

## ひとことで言うと
予測した数値と正解の数値が「どれくらいズレているか」を測る道具(損失関数)の一種です。ズレが小さいときと大きいときで、ペナルティの計算式を切り替えるのが特徴です。小さいズレには二乗(L2)で滑らかに、大きいズレには絶対値(L1)でほどほどに罰を与えるので、極端に外れたデータ(外れ値)に学習が引きずられにくくなります。統計学では「Huber 損失」、深層学習の物体検出の文脈では「Smooth L1 損失」という名前で広く使われています。

## 直感的な理解
体重計が壊れていて、たまに「150kg」というありえない値を返すとします。あなたの本当の体重を推定したいとき、二乗の罰(L2)を使うと、その壊れた1点が二乗されて巨大なペナルティになり、推定値がその1点に引っ張られてしまいます。一方で絶対値の罰(L1)を使えば、壊れた値の影響は「他の正常値と同じ重みの1票」にとどまるので、まともな推定ができます。

ただし L1 には別の悩みがあります。あなたの推定値がちょうど正解にぴったり一致した瞬間、罰の「傾き」(これは「推定値を少し動かしたときに罰がどれだけ増減するか」を表す量で、学習はこの傾きをたどって進みます)が、プラス側から近づいたときは +1、マイナス側から近づいたときは −1 と、ジャンプして不連続になります。この不連続のせいで、正解の真上に来てもそこにピタッと止まれず、行ったり来たり振動しがちです。

Smooth-L1(Huber)損失は、この2つの問題を1つの式で同時に解決します。「正解の近く(小さいズレ)では L2 のように滑らかに振る舞い、正解から遠いところ(大きいズレ)では L1 のように頑健に振る舞う」よう、しきい値を境に2つの式を継ぎ目なくつなぐのです。自動運転で言えば、GPS や IMU(慣性計測装置)のノイズで一瞬だけ飛んだ位置データや、人手のラベル付けミスで全く違う場所を正解にしてしまった点があっても、それに学習が振り回されにくくなります。

## 基礎: 前提となる概念

機械学習では、モデルが出した予測と、本当の正解(教師データ)との「差」を数値1つにまとめ、その数値を小さくするようにモデルの内部パラメータを少しずつ調整していきます。この「差を表す数値」が損失 (loss)、それを計算する式が損失関数 (loss function) です。学習の正体は「損失を入力としてパラメータを微小に動かす最適化」で、その方向と勢いを決めるのが損失の勾配(微分)です。だから損失関数を選ぶことは、暗黙のうちに「どんな勾配でパラメータを動かすか」を選ぶことに等しいのです。

回帰 (regression) とは「連続的な数値そのものを当てる問題」です。たとえば「3.2秒後に車は前へ何メートル、横へ何メートル進むか」を予測するのは回帰です。これに対し「信号は赤か青か」のように離散的な選択肢を当てるのは分類 (classification) と呼び、交差エントロピーなど別の損失を使います。

回帰の基本損失は2種類あります。

- L2 損失(二乗誤差、MSE): 差を二乗する。差が 2 なら 4、差が 10 なら 100 と、大きなズレを猛烈に重く罰する。最適解は「条件付き平均」に対応する。
- L1 損失(絶対値誤差、MAE): 差の絶対値をとる。差が 2 なら 2、差が 10 なら 10 と、大きさに比例した罰。最適解は「中央値」に対応する。

外れ値 (outlier) とは、何かの拍子で混ざる、明らかに異常な極端値のことです。L2 だけを使うと外れ値1点の差が二乗で巨大化し、損失全体がその1点に支配され、モデルは「その変な1点」に合わせようと無理をしてまともなデータの予測がかえって悪化します。逆に L1 は外れ値に強い反面、原点(差ゼロ)で微分が不連続になります。Smooth-L1 はこの「L2 の外れ値弱さ」と「L1 の原点不連続」の両方を避ける折衷案として設計されました。

統計学にはこの折衷を裏づける枠組みがあります。M 推定 (M-estimation) と呼ばれ、損失関数 ρ(残差)の和を最小化する一般的な推定の枠です。L2 は ρ(x)=x²、L1 は ρ(x)=|x|、Huber はその中間です。M 推定では「影響関数 (influence function)」という量で外れ値感度を測ります。影響関数は ρ の微分 ψ(x) のことで、これが「残差1点が推定値に与える影響の大きさ」を表します。L2 は ψ(x)=2x で残差に比例して際限なく増える(だから外れ値に弱い)、L1 は ψ(x)=sign(x) で頭打ち(頑健)、Huber は残差が小さいうちは比例、大きくなると頭打ち、という設計になっています。

## 仕組みを詳しく

しきい値 (threshold) を β(ベータ、深層学習フレームワークの既定値はしばしば 1.0)とします。予測と正解の差を x = 予測 − 正解 とおくと、要素ごとの損失は次のように定義されます。

```
|x| <  β のとき(小さいズレ):  0.5 * x^2 / β
|x| >= β のとき(大きいズレ):  |x| - 0.5 * β
```

数値例(β = 1.0):

- 差 x = 0.2(小さいズレ): 0.5 * 0.2^2 = 0.02  → L2 的に小さく罰
- 差 x = 0.5: 0.5 * 0.25 = 0.125
- 差 x = 1.0(境界): 0.5 * 1 = 0.5(どちらの式でも 0.5 で一致)
- 差 x = 5.0(大きいズレ): 5.0 - 0.5 = 4.5  → L1 的に「比例どまり」で罰

同じ差 5.0 を L2 で測ると 0.5 * 25 = 12.5 です。Smooth-L1 なら 4.5 で済むので、外れ値が損失を支配しにくいことが具体的に分かります。

設計の核心は「境界 |x| = β でちょうど2つの式の値が一致し、しかも傾き(微分)も一致する」ことです。これが「Smooth(滑らか)」の意味です。式の傾きを書き下すと次のようになります。

```
|x| <  β:  傾き = x / β        (原点で 0、滑らかに増える)
|x| >= β:  傾き = sign(x) = ±1  (一定)
```

x = ±β で内側の傾き x/β = ±1 が、外側の傾き ±1 と一致するので、関数全体が一階微分まで連続(C1 連続)になります。原点では x² 由来なので傾きが連続(L1 の不連続を回避)、遠方では直線なので傾きが ±1 で頭打ち(L2 の勾配爆発を回避)になります。勾配が頭打ちになることは、学習序盤で予測が大きく外れていても、更新量が暴走しないことを意味します。

ASCII で3つの損失の形を並べると挙動の違いが一目で分かります(縦が損失、横が残差 x)。

```
損失
 |          L2  /        ← 二乗。遠方で急上昇(外れ値に過敏)
 |            /
 |   Huber __/----  L1   ← 近傍は L2 のお椀、遠方は L1 の直線
 |       _-/
 |  ___-/
 +----+---------------> x
      β(切り替え点)
```

なお Huber 損失と Smooth-L1 損失は本質的に同じものですが、スケールの慣習が異なります。古典的 Huber は外側を δ(|x| − δ/2)と書き、内側を 0.5 x² とします(δ がしきい値)。深層学習で「Smooth L1」と呼ぶときは β で割って正規化し、β を小さくすると L1 に、大きくすると L2 に連続的に近づくパラメータとして扱うことが多いです。両者の関係は β=δ で、内側だけ係数が 1/δ 違います(古典 Huber は 0.5x²、正規化 Smooth-L1 は 0.5x²/β)。論文を読むときは「δ なのか β なのか」「内側を割っているか」を確認すると混乱しません。

実装上は、要素ごとの損失をそのまま配列で返す設定(reduction を平均も合計もせず none にする)を選ぶことがあります。これは後段でステップごと・要素ごとに別々の重みを掛けたい場合(たとえば近未来の予測を遠未来より重視したい場合。[時間重み付き損失 (Temporal Decay Weighting)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0136_temporal-weighted-loss) 参照)に必要です。先に平均してしまうと、要素単位の重み付けができなくなるためです。

軌跡予測での典型的なテンソル形状を追うと、予測は `[B, T, 2]`(B=バッチ数、T=未来の時間ステップ数、2=縦横の2方向)という形をしています。Smooth-L1 を要素ごとに適用すると同じ形 `[B, T, 2]` の損失が返り、最後の次元(2方向)で平均して `[B, T]` にし、その後で時間重みを掛けて平均をとる、という流れが一般的です。

## 手法の系譜と主要論文

Huber, P. J. (1964) "Robust Estimation of a Location Parameter" (Annals of Mathematical Statistics)。統計学者ピーター・フーバーが提案した「Huber 損失」が元祖です。Huber が解こうとした問題は「データに少数の外れ値が混ざっていても、平均値などの推定が大きく狂わないようにしたい(これを頑健推定 robust estimation と呼ぶ)」というものでした。彼はノイズ分布が「ほぼ正規分布だが割合 ε だけ未知の汚染が混じる」という近傍を考え、その近傍の中で最悪ケースの分散(漸近分散)を最小化する推定量を導きました。その最適解がまさに「小さい残差は二乗・大きい残差は線形」という形になることを示したのです。これは min-max(ミニマックス)頑健性の古典的成果で、その後のロバスト統計学の出発点になりました。トレードオフは「しきい値 δ をどこに置くか(外れ値の想定割合に依存する)を人間が決めないといけない」点です。

Girshick, R. (2015) "Fast R-CNN" (ICCV 2015, arXiv:1504.08083)。物体検出(画像内の物体を四角い枠で囲い種類を当てるタスク)の有名手法で、枠の位置・大きさを回帰するのに Huber の β=1 版を「Smooth L1 loss」と名付けて使い、深層学習に広めました。Girshick が L2 を避けた理由は明確で、検出の学習序盤は予測枠が正解から大きくズレており、L2 だとその巨大な差が二乗で爆発して勾配が発散・学習が不安定になるからです。論文では「smooth L1 は L2 より外れ値に敏感でなく、勾配が ±1 で頭打ちになるため学習率の調整に神経質にならずに済む」と述べています。

Ren et al. (2015) "Faster R-CNN" (NeurIPS 2015, arXiv:1506.01497) は同じ Smooth L1 をアンカーボックス(あらかじめ用意した基準枠)のオフセット回帰に採用し、以降 SSD(Liu et al. ECCV 2016)、RetinaNet(Lin et al. ICCV 2017)、多くの 3D 物体検出器(LiDAR 点群から 3D ボックスを当てる検出器を含む)が枠座標回帰の標準損失として踏襲しました。

その後、回帰損失の系譜は2方向に枝分かれします。一つは「サンプル間の不均衡」を狙う改良で、Pang et al. (2019) "Libra R-CNN" (CVPR 2019, arXiv:1904.02701) は Smooth-L1 が「易しいサンプル(小さい残差)の勾配が小さすぎて、難しい外れ値サンプルに学習が偏る」点を指摘し、小残差域の勾配を持ち上げた Balanced L1 loss を提案しました。もう一つは「座標を別々に回帰すると物体の大小でロス重要度が偏る」点への反省で、検出枠の重なり具合 IoU(Intersection over Union)を直接最適化する GIoU(Rezatofighi et al. CVPR 2019, arXiv:1902.09630)、DIoU/CIoU(Zheng et al. AAAI 2020)へ発展しました。Smooth-L1 はこうした改良群の「比較対象となるベースライン」であり、今も多くの実装の既定値であり続けています。

## 論文の実験結果(定量データ)

物体検出の評価では mAP(mean Average Precision、平均適合率の平均。検出枠が正解とどれだけ重なり、かつ正しいクラスを当てられたかを 0〜100% で総合採点する指標。高いほど良い)が標準です。

Fast R-CNN(VGG16 バックボーン)は PASCAL VOC 2007 で mAP 約 66.9%、VOC 2012 で約 65.7% を報告しました。当時の R-CNN/SPPnet がおよそ 58〜66% 台だったのに対し、Smooth L1 による安定した枠回帰と end-to-end 学習の組み合わせで上位を占め、かつ学習速度を R-CNN の約 9 倍、推論を約 213 倍(SVD 圧縮込み)に高速化したことが報告されています。論文中の考察では、回帰損失を L2 にすると学習率を慎重に下げないと発散しやすく、Smooth L1 はその調整を不要にしたと述べられています。

Balanced L1(Libra R-CNN, CVPR 2019)は、COCO(約 12 万枚・80 クラスの大規模検出ベンチマーク)で、損失をこれだけ差し替える(他の改良も入れた総合では更に大きいが、その内訳として)ことで AP を約 0.8〜1 ポイント前後改善したと報告しています。AP は mAP を IoU しきい値 0.5〜0.95 で平均した厳しい指標で、1 ポイントの差は検出研究では十分に意味のある改善とみなされます。アブレーションの主張は「Smooth-L1 では易しいサンプルの勾配が支配的になりがちで、外れ値サンプルの学習が進みにくい。小残差域の勾配を持ち上げると両者のバランスが取れる」というものです。

ロバスト統計の観点からの定量的な直感としては、しきい値 δ を「残差の標準偏差の約 1.345 倍」に設定すると、ノイズが正規分布のときでも最小二乗推定の効率の約 95% を保ちつつ、外れ値が混じったときの頑健性を得られることが知られています(古典的な漸近相対効率 ARE の議論)。つまり「正規ノイズでの精度を 5% だけ譲って、外れ値耐性を買う」という交換が定量的に成立しているわけです。

アブレーション的な観察として、β を 0 に近づけると関数は純粋な L1 に漸近し、原点近傍で勾配が ±1 のまま振動が増える一方、β を非常に大きくすると純粋な L2 に漸近し外れ値で勾配が爆発します。多くの検出フレームワークが β=1 前後を既定にするのは、座標オフセットの典型的なスケール(正規化後でおおむね 1 程度)に合わせて「小誤差は L2、大誤差は L1」の切り替え点を残差分布の肩のあたりに置くためです。β をデータのスケールに合わせず固定したまま入力の単位だけ変える(メートル → センチメートル)と、実質的にほぼ全残差が L1 領域に入り頑健性ばかりが効いて収束が鈍る、という失敗がよく起きます。

## メリット・トレードオフ・限界

メリット
- 外れ値に頑健。大きなズレの勾配が ±1 で頭打ちになるため、ノイズ1点に学習が振り回されにくい。
- 原点でも微分が連続(C1 連続)。正解近くで学習がカクつかず、L1 のような振動を避けられる。
- L1 と L2 の「いいとこ取り」で、両者の極端な弱点を回避でき、学習率の調整がシビアになりにくい。

トレードオフ・限界
- しきい値 β(δ)を人間が決める必要がある。大きすぎると実質 L2、小さすぎると実質 L1 になり、データのスケールに合わせた調整が要る。データの単位(メートルか正規化値か)が変われば最適な β も変わる。
- 真に正確な最小二乗推定が欲しい(ノイズがきれいな正規分布)場合は、L2 より理論効率がわずかに落ちる(前述の約 5%)。
- 損失の絶対値スケールが L1・L2 と異なるため、複数損失を足し合わせるマルチタスク学習では重み調整に注意が要る。
- 二階微分が β で不連続(C2 連続ではない)ため、ニュートン法など二階情報を使う最適化とは相性がやや悪い。実用上は一階法(SGD/Adam)が主流なので問題になりにくい。
- 未解決の課題として、物体の大小やスケールによって座標誤差の重要度が一様でない問題は β 固定では解けず、IoU 系損失や Balanced L1 など、損失の形そのものを改良する研究へつながっている。

## 発展トピック・研究の最前線

回帰損失の系譜は Smooth-L1 から、(1) スケール不変性を狙う IoU/GIoU/DIoU/CIoU 損失、(2) 不確実性を同時推定する probabilistic な回帰(KL-Loss、ガウス分布の負の対数尤度)、(3) 物体の難易度でサンプル重みを変える Balanced L1 や Focal 系の回帰版、へと発展しています。軌跡予測の文脈では、点ごとの Smooth-L1 だけでなく、複数の将来仮説を出して「最も近い1本」だけを罰する minimum-over-K(min-of-N)損失や、混合ガウス分布の負の対数尤度で経路の不確実性ごと学習する方法が主流になりつつあります。Smooth-L1 はこうした高度な損失の「中の要素」として今も使われており、まず本損失の挙動を完全に理解しておくことが、より洗練された損失設計を読み解く土台になります。
