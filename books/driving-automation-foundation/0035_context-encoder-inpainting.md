# Context Encoder (inpainting)

## ひとことで言うと

Context Encoder は、ラベルのない画像だけから役立つ特徴を学ぶ自己教師あり学習 (self-supervised learning) の手法です。画像の一部 (たとえば真ん中の四角い領域) をわざと消して空白にし、ネットワークに「周りの情報から、消えた部分に何があったかを描き直させ」ます。穴を自然に埋めるには、周囲の文脈 (context) から「ここには何があるはずか」を推論する必要があり、その過程でシーンの構造を捉えた特徴が育ちます。これは後に画像分野を席巻する「マスクして復元する (masked image modeling)」系統の先駆けにあたります。

## 直感的な理解

破れた写真や、一部が塗りつぶされた絵を見せられても、私たちは「隠れている部分にはたぶんこれがある」と補完できます。机の上に置かれたコップの脚が隠れていても、コップ全体の形を想像できますし、人の顔の片目が隠れていても、もう片方から位置や形を推測できます。これができるのは、私たちが物体やシーンの構造を知っていて、見えている部分から見えていない部分を整合的に予測できるからです。

Context Encoder はこの能力をネットワークに獲得させようとします。「穴を埋めよ」という生成タスクを真剣に解くと、解くために必要な「シーンの構造理解」が内部に蓄積されます。私たちが欲しいのは埋められた絵そのものではなく、埋めるために学習された特徴抽出器のほうです。穴埋めはあくまで口実で、それを通じて物体・場面の表現を無ラベルで作るのが真の狙いです。

なぜこの発想が強力かというと、穴を埋めるのは「単にピクセルをコピーする作業」ではないからです。中央が大きく欠けていれば、周囲から直接コピーできる情報はなく、「この壁は奥に続いているはず」「この顔ならここに目があるはず」といった高度な推論が要ります。難しいタスクほど、解くために深い理解を強制する、というのが自己教師あり学習の一般原理です。小さな穴なら近傍のテクスチャをコピーするだけで埋まってしまい意味理解が不要なので、Context Encoder があえて大きな穴を選んだのは「カンニングを封じて難しくする」ためでもあります。

## 基礎: 前提となる概念

自己教師あり学習は、人手の正解ラベルを使わず、データ自身から問題と正解を自動生成して学ぶ枠組みです。代理タスク (pretext task) はその自動生成された踏み台タスクで、本命の下流タスク (downstream task、分類・検出・セグメンテーション) とは別物です。Context Encoder の pretext task は inpainting (インペインティング、画像の欠損補完) です。画像から領域を消すのはプログラムが行うので、「消した部分の元のピクセル」が自動的に正解になります。人手ラベルは不要です。

エンコーダ・デコーダ構造を知っておく必要があります。エンコーダ (encoder) は入力を低次元の圧縮表現 (特徴) に落とすネットワークで、デコーダ (decoder) はその圧縮表現から出力 (ここではピクセル) を生成するネットワークです。この「文脈をエンコーダで圧縮し、デコーダで欠損を生成する」構造から Context Encoder (文脈エンコーダ) という名が付きました。エンコーダとデコーダの間にある最も圧縮された中間表現を「ボトルネック (bottleneck)」と呼び、情報がここを通り抜ける際に余分なものが捨てられ、本質的な特徴だけが残ると期待されます。

GAN (Generative Adversarial Network、敵対的生成ネットワーク) も前提になります。生成器 (本物そっくりの画像を作る) と判別器 (本物か生成物かを見分ける) を競わせて学習する枠組みで、判別器を騙すように生成器を鍛えることで、リアルでシャープな生成物を得られます。Goodfellow らが 2014 年に提案したこの枠組みを、Context Encoder は表現学習に組み込んだ初期の代表例の一つです。

## 仕組みを詳しく

ネットワークは2部構成です。エンコーダが穴あき画像を見て文脈を圧縮表現に落とし、デコーダ (転置畳み込み、deconvolution。小さな特徴マップを拡大しながらピクセルを生成する畳み込み) がその表現から欠損領域のピクセルを生成します。処理は次の通りです。

1. 入力画像 (たとえば 128x128) から、ある領域 (たとえば中央の 64x64 の正方形) を消して空白にする。消した領域のもとのピクセルが正解。
2. 穴あき画像をエンコーダ (AlexNet 系の CNN) に通し、圧縮された特徴表現を得る。形状イメージ: `[B, 3, 128, 128]` (B はバッチ数、中央 64x64 が空白) を入力にし、エンコーダで `[B, 4000]` 程度のボトルネック特徴に圧縮する。
3. ここで重要な部品が channel-wise fully-connected layer (チャンネルごとの全結合層) です。畳み込みだけだと特徴マップの離れた位置どうしが直接つながらず、画像の左端の情報が右端の穴を埋めるのに使えません。各チャンネル内で全空間位置を結ぶこの層を挟むことで、画像全体の文脈をボトルネックに集約できます。
4. デコーダでその特徴から欠損領域のピクセルを生成する。出力は `[B, 3, 64, 64]`。
5. 生成ピクセルと正解ピクセルを比べて学習する。

ここで損失の設計が肝です。Context Encoder は2種類の損失を組み合わせました。

再構成損失 (reconstruction loss, L2): 生成ピクセルと正解の二乗誤差です。これだけだと多峰性 (複数の妥当な答えがある状況、[彩色](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0034_colorization-pretext) と同じ問題) のせいで「妥当な答えの平均」を取り、ぼやけた (blurry) 穴埋めになります。大域的な配置は合うが、エッジや質感などの細部がぼけます。

敵対的損失 (adversarial loss, GAN): 別の判別ネットワークを用意し、「埋めた領域が本物の画像か、生成された偽物か」を見分けさせます。生成側はその判別器を騙せるほどリアルな穴埋めを生成するよう促されます。判別器は「平均化されたぼやけた絵」を即座に偽物と見抜くので、生成器はシャープで自然な細部を作らざるを得なくなります。

この2つを足すことで、「大域的に正しく (L2) かつ局所的にシャープで自然 (GAN)」な穴埋めを実現しました。論文では L2 と adversarial の重みをおよそ 0.999 対 0.001 のような比で混ぜ、L2 が大域構造を、わずかな adversarial が知覚品質を担う設計にしています。before / after で整理します。

```
[before] 再構成損失 (L2) だけ
  穴あき画像 ──Enc──> 特徴 ──Dec──> ぼやけた穴埋め
                              (妥当な答えの平均化でブラー)

[after] 再構成損失 + 敵対的損失
  穴あき画像 ──Enc──> 特徴 ──Dec──> リアルでシャープな穴埋め
                              ↑ 判別器が「偽物臭さ」を罰する
```

論文は「中央の四角い穴」だけでなく、ランダムな矩形領域や格子状に散らした領域を消す方式も試しました。表現学習 (転移特徴) の目的では後者のほうが良いことを報告しています。中央固定だと「中央を埋める」という位置に特化した特徴になりがちで、ランダム化・分散化すると位置非依存のより汎用的な特徴が育つためです。なぜこの設計かを一言でまとめると、欠損補完は文脈理解を強制するから良い特徴が育ち、生成のぼやけを GAN で補うことで実用品質を確保した、ということです。学習後はエンコーダを学習済み backbone として下流タスクへ転用します。

## 手法の系譜と主要論文

中心論文は Pathak, Krähenbühl, Donahue, Darrell, Efros の "Context Encoders: Feature Learning by Inpainting" (CVPR 2016, arXiv:1604.07379) です。貢献は、画像の欠損領域を文脈から再構成する pretext task と、再構成損失 + 敵対的損失の組み合わせを提案し、欠損補完と表現学習を同時に達成したこと。2016 年当時、「文脈から欠損を予測する」発想をニューラル画像生成と結びつけた先駆的研究でした。

この「マスクして復元する」発想は、まず言語分野で大爆発します。Devlin らの "BERT" (NAACL 2019, arXiv:1810.04805) は、文中の単語 (トークン) の一部 (15% 程度) をマスクして当てる masked language modeling で、強力な汎用言語表現を学びました。Context Encoder の画像版「穴埋め」と本質的に同じ発想を、言語で大成功させたものです。言語ではトークンが離散的なので「正解候補のリストから当てる分類」として素直に解け、多峰性のぼやけ問題が起きにくいという利点がありました。これによりマスク復元型自己教師あり学習の威力が広く認知されました。

画像に戻ってきたのが He, Chen, Xie, Li, Dollár, Girshick の "Masked Autoencoders Are Scalable Vision Learners" (MAE, CVPR 2022, arXiv:2111.06377) です。画像を多数のパッチに分け、その大半 (75%) をマスクして残りから復元させます。Vision Transformer を使い、見えるパッチだけを大型エンコーダで処理し、マスク位置は小型デコーダで埋める非対称設計で、効率と性能を両立しました。GAN を使わず単純な再構成損失でも、マスク率を極端に高くすれば良い特徴が学べることを示し、現在の画像自己教師ありの主流の一つになりました。Context Encoder が「中央の大穴を L2 + GAN で埋める」だったのに対し、MAE は「多数の小穴を高マスク率で L2 だけで埋める」点が進化です。高マスク率がぼやけ問題を緩和したのは、復元タスク自体が難しくなり「平均でごまかす」余地が減ったからと解釈できます。

並行して Bao らの "BEiT" (ICLR 2022, arXiv:2106.08254) は、マスクパッチをピクセルではなく離散トークン (BERT のように) で予測する MIM (Masked Image Modeling) を提案しました。ピクセルの直接予測を避けることで、言語の masked modeling に近い「離散分類」へと持ち込んだわけです。Context Encoder → BERT → MAE / BEiT という「マスク復元」の系譜の延長線上にあります。CNN でマスク復元を成立させた [ConvNeXt V2 / FCMAE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0039_convnextv2-fcmae) や、内部特徴を予測する [data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) もこの大きな流れの分岐です。Context Encoder はこの系譜全体の最初の重要なマイルストーンです。

## 論文の実験結果 (定量データ)

Context Encoder は ImageNet と PASCAL VOC で評価されました。事前学習した特徴を PASCAL VOC 2007 の分類・検出・セグメンテーションへ転移したところ、ランダム初期化 (事前学習なし) を一貫して上回りました。報告では PASCAL VOC 2007 検出で、ランダム初期化からおよそ数ポイントの mAP (mean Average Precision、検出精度の代表指標。物体を正しく見つけた割合を平均した値で、高いほど良い) の改善が示されています。セマンティックセグメンテーションでも同様にランダム初期化を上回り、当時の他の自己教師あり pretext task (相対位置予測など) と競合する水準でした。

ただし重要な限界も同時に報告されました。当時の教師あり ImageNet 事前学習 (ラベル付きで分類を解いて得た重み) との間にはまだ明確な差があり、Context Encoder 単独では教師ありの代替には届きませんでした。これは「inpainting で良い特徴は育つが、当時の規模・設計 (AlexNet 級のエンコーダ、ImageNet 程度のデータ) ではまだ上限が低い」ことを示します。本領発揮は、Vision Transformer と大規模データ・高マスク率が整った MAE の時代 (2022) まで待つことになります。

inpainting そのものの品質も評価され、Paris StreetView などのデータで、近傍をコピーするだけの古典手法より文脈を理解した自然な穴埋めができることが示されました。

着色や後年の MAE と同様、損失設計のアブレーションが教訓的です。再構成損失 (L2) だけだと、定量的な再構成誤差 (画素単位の二乗誤差) は小さくできても、生成画像は知覚的にぼやけます。敵対的損失を加えると、L2 誤差はわずかに増えることがあっても知覚品質 (人が見たシャープさ・自然さ) が大きく改善します。これは「ピクセル単位の誤差最小化」と「人にとっての見た目の良さ」が必ずしも一致しないという、生成モデル研究の普遍的な教訓を早期に示した例です。マスク領域の取り方についても、中央固定よりランダム/分散配置のほうが下流転移で優れることが示され、後の MAE の「ランダム高マスク率」の発想につながりました。

これらの指標の意味を補足します。転移検出の mAP は、特徴が「どこに何があるか」という空間的・意味的情報を保持しているかを測ります。線形評価や fine-tuning 後の精度は、その特徴が下流タスクにとってどれだけ良い初期値かを測ります。Context Encoder の結果は「方向性は正しいが規模が足りない」という、初期 SSL に共通する状況を定量的に示したものでした。

## メリット・トレードオフ・限界

メリットは次の通りです。正解 (消したピクセル) が自動で得られ人手ラベルが不要。穴埋めには文脈理解が必須なのでシーン構造を捉えた特徴が育つ。欠損補完という実用的な画像編集応用も同時に得られる。そして何より、BERT・MAE へ連なるマスク復元型自己教師ありの先駆けという歴史的価値があります。「一部を隠して残りから当てる」というアイデアそのものが、その後の表現学習の主流の源流になりました。

トレードオフと限界もあります。第一に、ピクセル生成は本質的に難しく、再構成損失だけでは多峰性のためぼやけます。これを補う GAN は学習が不安定で、生成器と判別器のバランス調整 (どちらかが強すぎると学習が崩れる) が難しい。第二に、大きな欠損を意味的に正しく埋めるのには限界があり、複雑なシーンの中央大穴は構造が破綻しがちです。第三に、当時の転移性能は教師あり事前学習に届きませんでした。第四に、エンコーダが「穴あき画像」を見る前提なので、下流で「完全な画像」を入力したときに分布のずれ (入力統計の違い) が生じうる、という設計上の歪みもありました。後の MAE はマスクトークンをデコーダ側に追いやることでこの問題を緩和しています。

研究上の未解決課題は「ピクセル空間での再構成がどこまで良い表現を生むか」という問いです。ピクセルを忠実に再現しようとすると、意味的に重要でない高周波のノイズや質感まで予測対象になり、抽象的な意味理解が薄れる懸念があります。この弱点への一つの回答が、ピクセルではなく内部の潜在特徴を予測する [data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) や、離散トークンを予測する BEiT です。Context Encoder は「何を予測対象にすべきか」という、今も続く中心的な問いを最初に投げかけた手法と言えます。

## 発展トピック・研究の最前線

マスク復元型自己教師あり学習はその後も急速に発展しました。MAE はその簡潔さとスケーラビリティで標準的なベースラインになり、医療画像・衛星画像・動画 (VideoMAE、時間方向のパッチも含めてマスクする) など多様な領域に展開されています。CNN へ移植した [ConvNeXt V2 / FCMAE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0039_convnextv2-fcmae) は、スパース畳み込みでマスクを扱い、GRN で特徴崩壊を防ぐことで、ViT 以外でもマスク復元を成立させました。

予測対象を高度化する流れも活発です。離散トークン予測 (BEiT 系)、特徴蒸留 (data2vec)、複数ターゲットの混合などが研究されています。生成モデルの観点では、現代の inpainting は拡散モデル (diffusion model) が主流になり、多峰性を確率的サンプリングで自然に扱えるようになりました。Context Encoder が GAN で苦労したシャープさと多様性の両立が、より安定した枠組みで実現されています。

自動運転や動画理解では、空間だけでなく時間方向の欠損補完 (未来や過去のフレームを再構成する) が、シーンのダイナミクスを学ぶ手段として注目されています。Context Encoder が示した「欠けた情報を文脈から再構成すると表現が育つ」という原理は、空間・時間・モダリティを越えて一般化され続けており、自己教師あり学習の最も基礎的なアイデアの一つとして今も生きています。
