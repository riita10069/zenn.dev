# FIERY

## ひとことで言うと

複数のカメラ映像だけ(LiDAR などの距離センサーを使わず)から、車の周りを真上から見下ろした地図(BEV、Bird's-Eye View)を作り、さらに「数秒後に周りの車や人がどこへ動くか」まで予測する深層学習モデルです。しかも未来を1通りに決め打ちせず、「右折するかもしれないし、直進するかもしれない」という複数のあり得る未来を確率的に出すのが最大の特徴です。検出・追跡・予測という従来は別々だった処理を、1つのネットワークでまとめて(end-to-end に)解いた点が画期的でした。

## 直感的な理解

自動運転で安全に走るには、「今どこに何があるか」(知覚)だけでなく「次にそれがどう動くか」(予測)を知る必要があります。前の車が車線変更しそうか、歩行者が道路に飛び出しそうか。これが分からないと、止まるべきか進むべきかを安全に判断できません。

FIERY 以前の典型的なシステムには2つの分断がありました。

第一に、センサーの分断です。距離を正確に測るには LiDAR(レーザーで物体までの距離を点群として測る装置)が前提になりがちで、安価なカメラだけで3D的な周辺地図を作るのは難しいとされていました。

第二に、タスクの分断です。「物体検出 → 物体追跡 → 軌跡予測」という別々のモジュールを直列につなぐパイプライン(処理を1列に並べた構成)が主流でした。これには弱点があり、各段でエラーが積み重なります。検出ミスが追跡ミスを生み、追跡ミスが予測ミスを生む、という連鎖です。しかも各モジュールが手作りのルールに依存していました。

さらに根本的な問題として、未来は本質的に不確実です。交差点で前の車が右に行くか直進するかは、現在の情報だけでは確定できません。それなのに「1通りの未来」だけを出そうとすると、ネットワークは複数の可能性の平均をとった、ぼやけて非現実的な予測(右と直進の中間のような、実際にはあり得ない動き)を出してしまいます。FIERY はこれら全てをまとめて解こうとしました。

## 基礎: 前提となる概念

BEV(Bird's-Eye View、鳥瞰図)とは、車を真上から見下ろした俯瞰の地図表現です。前方カメラの視点ではなく、空から見た平面図に物体の位置や動きを並べます。経路計画(どう走るか決める処理)は俯瞰の地図上で考えるのが自然なので、知覚の出力を BEV にしておくと下流に渡しやすいという利点があります。

深度(depth)とは、カメラから物体までの距離です。1枚の画像からは、小さく写った車が「遠くの普通車」か「近くのミニカー」か原理的に区別できないため、深度推定はカメラ知覚の中心的な難所です。

潜在変数(latent variable)とは、直接は観測できないが、データの背後にある要因を表す隠れた変数です。FIERY では「どの未来シナリオが起きるか」を選ぶ見えないつまみとして使われます。

確率分布(probability distribution)とは、ある値が起こりやすさをどう割り振られているかの表現です。深度を「5mが60%、10mが30%…」のように分布で持つことを、確率的に持つと言います。1つに決めない代わりに不確実性を保持できます。

end-to-end(エンドツーエンド)とは、入力(生のカメラ画像)から最終出力(未来予測)までを、途中で人手のルールを挟まず、1つのネットワークで一括して学習・推論することです。中間モジュール間のエラー伝播を避けられる利点があります。

## 仕組みを詳しく

FIERY(Future Instance prEdiction in bird's-eYe view の頭文字)の処理は次の流れです。

ステップ1: 各カメラ画像から特徴抽出。車の周囲360度をカバーする6台のサラウンドカメラの各画像を、画像認識バックボーン(EfficientNet)に通して特徴マップにします。

ステップ2: カメラ特徴を BEV へ持ち上げる(Lift-Splat-Shoot 方式)。各ピクセルについて「この視線方向のどの距離に物体があるか」の深度を離散的な確率分布として予測し、その確率に応じて特徴を3D空間にばらまき(lift)、真上から見た BEV グリッドに集約(splat)します。深度を確率分布で持つことで、深度が曖昧なピクセルは複数の距離にぼんやり寄与し、はっきり分かるピクセルは特定の距離に集中します。これで1時刻ぶんの BEV 特徴ができます。

ステップ3: 時間方向に並べて自車の動きを補正。過去の数フレームぶんの BEV を、その間に自車が動いたぶん(ego-motion、自車運動)だけ座標変換で位置合わせして重ねます。これで「静止している電柱は動かず、走る車だけが BEV 上で動く」という、地面に固定された一貫した時系列 BEV になります。この位置合わせを怠ると、自車が動いたぶんまで「物体が動いた」と誤認してしまいます。

ステップ4: 時間モデルで時系列を集約し未来を生成。位置合わせ済みの時系列 BEV を、3D畳み込みや時間方向の再帰的なネットワーク(GRU 系)に通し、過去の動きの傾向を捉えて未来の複数時刻ぶんの BEV を生成します。

ステップ5: 確率的な未来(FIERY の核心)。未来の不確実性を表すために潜在変数を導入します。学習時は「実際にその後起きた未来」も入力に使って事後分布(posterior、未来を知った上での分布)を学び、推論時は「現在までの観測だけ」から事前分布(prior、未来を知らない状態の分布)を学びます。両者を近づけるよう訓練することで(変分推論の枠組み)、推論時にこの潜在変数を何回かサンプリングすると、「右折する未来」「直進する未来」といった異なる現実的なシナリオが複数出てきます。

ステップ6: インスタンス単位の出力。各 BEV マスについて「物体中心への方向(オフセット)」「未来の動き(future flow)」「占有(そこに物体がいるか)」などを出力し、これらを後処理で集約して「この車という塊が、未来にこう動く」というインスタンス(個別の物体、1台1台・1人1人)ごとの運動を得ます。検出・追跡・予測を別モジュールに分けず、この一括出力で同時に達成しているのが要点です。

出力のイメージは、BEV グリッド(例: 200×200、1マス0.5m、つまり片側50mほどの範囲)の上に、現在と未来 t+1, t+2, … 各時刻の物体占有と動きが描かれた一連の鳥瞰図が、複数シナリオぶん並ぶ形です。

```
                          ┌─ シナリオ1: 前方車が右折
カメラ6枚 → BEV特徴列 ──┤
(過去Tフレーム,         └─ シナリオ2: 前方車が直進
 ego-motionで位置合わせ)    (各シナリオ = 潜在変数の1サンプル)
```

## 手法の系譜と主要論文

Lift-Splat-Shoot (Philion & Fidler、ECCV 2020、arXiv:2008.05711)
- 提案: 各ピクセルの深度を離散確率分布で予測し、特徴を3Dに持ち上げて(lift)BEV に落とす(splat)枠組み。Shoot は計画に使う部分です。
- 動機: カメラ画像から微分可能に(誤差逆伝播できる形で)BEV を作る一般的な方法がなかった。深度を明示的に確率で持つことでこれを実現しました。
- 新規性と効果: カメラのみで end-to-end に BEV を作る礎を築きました。FIERY の BEV 生成はこれを直接の土台にしています。トレードオフは、深度予測の不確実さがそのまま BEV の品質に効くこと。

FIERY (Hu, Murez, Mohan, Dudzik, Hawke, Kendall, Cipolla、ICCV 2021、arXiv:2104.10490。著者の多くは自動運転企業 Wayve 所属)
- 提案: 「カメラのみで未来インスタンス予測を BEV で end-to-end に解く」ことを世界で初めて統合的に実現しました。
- 動機: 上記のパイプライン分断(エラー伝播)と単一未来の非現実性を一気に解消するため。
- 新規性: Lift-Splat による BEV 化 + 時系列の ego-motion 位置合わせ + 変分推論による確率的未来 + インスタンス単位出力、を1つのネットワークに統合した点。確率的潜在変数で「多様だが現実的な複数未来」を生成できることを示しました。

BEVerse (Zhang, Zhu, Du, Liu, Sima, Sun、2022、arXiv:2205.09743)と StretchBEV (Akan & Güney、ECCV 2022)
- FIERY の発展形です。BEVerse は検出・地図構築・運動予測という複数タスクを1つの BEV 表現から同時に解くマルチタスク化を進めました。StretchBEV は潜在変数を時刻ごとに展開して、より長い時間域の多様な未来生成を改善しました。
- これらは後の UniAD(検出・追跡・地図・予測・計画を統合する End-to-End システム)などへ連なる流れの一部です。「BEV で時間・タスクをまたいで一貫して扱う」という方向の系譜の中で、FIERY はその出発点に位置づけられます。

## 論文の実験結果(定量データ)

評価指標を説明します。FIERY は未来の BEV セグメンテーション(各マスが車かどうか)と、未来インスタンス予測の品質を測ります。代表指標は IoU(Intersection over Union、予測領域と正解領域の重なり具合。重なりを和集合で割った値で、1に近いほど良い)と、インスタンスの時間的一貫性を測る VPQ(Video Panoptic Quality、未来にわたって同じ物体を同じ ID で正しく予測できているか。高いほど良い)です。

FIERY は nuScenes データセット(大規模な自動運転データセット、6カメラ+LiDAR等の同期データを含む)で、カメラのみの未来予測を評価しました。当時、カメラのみで BEV の未来インスタンスを予測する直接の先行研究がほとんど無かったため、FIERY は (1) 過去フレームを使わず現在のみから外挿する単純なベースライン、(2) LiDAR を使う一部の手法、と比較されました。

主要な結果は次の通りです。現在の BEV セグメンテーションと未来予測の両方で、過去の時系列を ego-motion 補正して使う FIERY は、現在フレームのみのベースラインを未来 IoU・VPQ ともに明確に上回りました。これは「時間情報を一貫した BEV に積み上げることが未来予測に効く」ことの直接の証拠です。また、カメラのみであるにもかかわらず、未来の動的物体の運動予測で、LiDAR を使う一部の手法に匹敵する水準に達したと報告されています。

アブレーション(構成要素を抜く実験)で重要な知見が複数あります。第一に、確率的潜在変数を外して決定論的に(1通りの未来だけ)予測させると、特に分岐のある状況で予測がぼやけ、複数モードの未来を表現できなくなって品質が落ちました。確率的モデリングが「現実的で多様な未来」の鍵であることが定量的に示されています。第二に、ego-motion による時系列の位置合わせを外すと、自車運動と物体運動が混ざって性能が大きく低下しました。第三に、過去フレーム数を増やすほど未来予測の精度が上がる傾向(ただし飽和あり)が確認され、時間文脈の重要性が裏付けられました。

これらの数値が意味するのは、(1) カメラのみでも時系列を正しく扱えば未来予測は実用域に近づく、(2) 未来を確率的に扱うことは精度指標そのものを改善する(単なる見栄えの問題ではない)、という点です。

## メリット・トレードオフ・限界

メリット
- カメラのみで、検出・追跡・予測を1モデルで end-to-end に解け、モジュール間のエラー伝播がない。
- 未来を確率的に複数出すので、交差点など分岐のある不確実な状況で現実的なシナリオを表現できる。
- BEV 出力なので、下流の経路計画にそのまま渡しやすい。

トレードオフと限界
- 深度予測ベースの BEV 生成(Lift-Splat)はメモリと計算が重く、深度の誤差がそのまま BEV 上の位置ずれに直結する。遠方の物体ほど深度誤差の影響が大きい。
- 6カメラの正確なキャリブレーションと時刻同期が前提で、これがずれると時系列 BEV の一貫性が崩れる。
- 確率的サンプリングは、複数シナリオを得るために推論を複数回回す必要があり、リアルタイム性に負荷がかかる。
- 予測時間域(数秒先)を超えて長くすると、不確実性が爆発的に増えて品質が落ちる。長期予測は依然として未解決。
- 稀なシナリオ(急な飛び出しなど)はデータに乏しく、学習で確率分布として捉えきれないことがある。

## 発展トピック・研究の最前線

FIERY が示した「BEV で知覚と予測を一貫して扱う」発想は、その後の End-to-End 自動運転の中核となりました。検出・追跡・地図・予測・計画を1つのクエリベースの Transformer で統合する流れ([UniAD (Planning-oriented統合E2E)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1090_uniad_note) など)は、FIERY の「分断しない」哲学を計画まで延長したものと読めます。

BEV 生成方式の進化も活発です。FIERY の Lift-Splat は深度を明示予測しますが、深度を陽に予測せず BEV クエリがカメラ特徴に注目するアテンションベースの方式(BEVFormer 系)や、長期の時間情報を効率的に積む時系列融合([SOLOFusion](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0294_solofusion))が登場し、メモリ効率と精度を改善しています。

予測の確率モデリングも、変分推論から拡散モデル(diffusion model、ノイズから徐々にサンプルを生成する生成モデル)を使った多峰な未来生成へと発展しています。これにより、より多様で現実的な複数未来を、よりシャープに描けるようになりつつあります。さらに、占有予測(occupancy forecasting、空間の各マスが将来埋まるかを直接予測)へと出力表現を一般化し、物体カテゴリに依存しない未来予測を目指す研究も最前線にあります。
