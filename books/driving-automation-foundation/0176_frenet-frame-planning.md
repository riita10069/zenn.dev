# Frenet座標系による軌道計画

## ひとことで言うと
Frenet座標系による軌道計画とは、道路の中心線を基準にした曲がった座標系(Frenet frame)を使い、車の動きを「中心線に沿って前へどれだけ進むか(縦方向 s)」と「中心線からどれだけ横にずれるか(横方向 d)」の2つに分けて、滑らかな軌道を作る古典的な手法です。直交座標(地図上のx, y)で考えるより、道なりの運転が直感的かつ計算しやすくなります。

## 直感的な理解
普通の地図はx-y座標(東西・南北)です。しかし運転手の感覚は「この道を50m進んで、車線中央から左に0.3mずれている」というものです。道がカーブしていても、運転手は「道に沿って前へ」「道に対して横へ」で物事を考えます。

Frenet座標はまさにこの運転手の感覚を座標にしたものです。

```
        d (横方向: 中心線からの左右のずれ)
        ↑
  ======|=================== 中心線(基準線、reference path)
        |
        ●───→ s (縦方向: 中心線に沿って進んだ距離)
       自車
```

- s(縦方向、longitudinal): 基準線(道路中心線)に沿って進んだ弧長(曲線に沿った距離)。
- d(横方向、lateral): 基準線に垂直な方向への左右のずれ。

カーブした道でも、s-d座標の中では道がまっすぐ伸びているように扱えます。これが最大の利点です。なぜなら運転タスクが自然に2つに分解できるからです。
- 縦方向の問題: 前の車との車間、速度・加減速、停止線(=sをどう動かすか)。
- 横方向の問題: 車線維持、車線変更、障害物の左右回避(=dをどう動かすか)。

この分離(decoupling)により、複雑な2次元の軌道計画が、扱いやすい2つの1次元問題に近づきます。

## 基礎: 前提となる概念

基準線(reference path): 道路中心線や車線中央など、あらかじめ用意した曲線。地図やレーン情報から得ます。Frenet座標はこの線に対して定義されるので、基準線の質が結果を左右します。

弧長パラメータ s: 基準線上の点を「始点からの道沿い距離」で表す変数。基準線上の各点には接線方向(進行方向)と法線方向(垂直方向)が定まります。

ジャーク(jerk): 加速度の変化率(加速度をさらに時間微分したもの)。乗り心地はジャークが小さいほど良いことが知られます。急に踏む/急に切ると、人は不快に感じます。Werlingらの手法はこのジャークを明示的に最小化します。

ジャーク最小軌道と五次多項式: 「始点と終点の位置・速度・加速度を指定し、その間をジャーク最小で繋ぐ軌道」は、数学的に時間 t の五次多項式(quintic polynomial、tの5乗までの項を持つ式)になることが知られています。横方向 d(t) を d(t) = a0 + a1 t + a2 t² + a3 t³ + a4 t⁴ + a5 t⁵ の形で表し、6個の係数 a0..a5 を境界条件6個(始点の位置・速度・加速度と終点の位置・速度・加速度)から連立方程式で一意に解きます。記号の意味: a0..a5 が軌道の形を決める係数、tは時間。縦方向で終端速度だけ指定する場合は四次多項式(quartic)になります。

## 仕組みを詳しく

Werling et al. (2010) の手法の流れです。

1. 座標変換。現在の自車状態(x-yでの位置・速度・加速度)を、基準線を使ってFrenet座標 (s, d) とその時間微分(速度・加速度)に変換します。

2. 横方向の候補軌道生成。終端の横位置 d を複数サンプリング(例: 中心線上 d=0、左に0.5m、右に0.5m、…)し、終端到達時間 T も複数サンプリングします。各 (終端d, T) の組に対して、現在状態から終端状態へ繋ぐジャーク最小の五次多項式 d(t) を解きます。これで横方向の候補軌道が多数できます。

3. 縦方向の候補軌道生成。状況に応じて目標を変えます。
   - 高速走行(車線維持): 終端速度 ṡ を複数サンプリングし、四次多項式で s(t) を生成。
   - 追従・停止(前車や停止線あり): 終端の s 位置を複数サンプリングし、五次多項式で生成。

4. 縦×横の組み合わせと評価。縦候補と横候補を掛け合わせて2次元の候補軌道集合を作ります。各候補にコスト関数を付けます。典型的には

   C = k_j · J + k_t · T + k_d · (d_終端 − d_目標)²

   記号の意味: J は軌道に沿ったジャークの二乗積分(乗り心地の悪さ)、T は到達時間(短く速いほど良い項)、(d_終端 − d_目標)² は目標横位置からのずれ、k_* は各項の重み(設計者が調整するつまみ)。縦横それぞれにコストを定義し合算します。

5. 安全チェックと選択。コストの低い順に候補を見て、(a) 最大加速度・最大曲率・速度などの車両物理制約を超えないか、(b) 他車・障害物と衝突しないか、を確認します。制約を満たす最小コストの軌道を採用します。

6. 逆変換。選ばれたFrenet軌道を x-y 座標に戻し、低レベル制御器に渡します。

この「多数の候補をサンプリング → コストで採点 → 制約で篩(ふる)い → 最良を選ぶ」という枠組みはサンプリングベース計画と呼ばれ、実装が単純で実時間性が高いのが特長です。

## 手法の系譜と主要論文

Frenet枠そのものは19世紀の微分幾何学(Frenet-Serretの公式、曲線に沿った接線・法線・従法線の枠組み)に由来し、人名もそこから来ています。

ロボティクスでの軌道計画への本格適用は、Werling, Ziegler, Kammel, Thrun の "Optimal Trajectory Generation for Dynamic Street Scenarios in a Frenet Frame"(ICRA 2010)が決定打です。彼らはDARPA Urban Challenge(2007年の市街地自動運転コンペ)に連なる研究の流れの中で、ジャーク最小の多項式軌道とFrenet分離を組み合わせ、実時間で動的な市街地シナリオに対応できる手法を示しました。Thrunはスタンフォードの自動運転(Stanley/Junior)を率いた研究者です。

この手法は実用性が高く、オープンソースの自動運転スタック(Apollo の lattice planner や Autoware の系譜など)に広く取り入れられ、いまも比較対象(ベースライン)として頻繁に使われます。

関連系譜として、格子状に候補を張るlattice planner(Pivtoraiko & Kelly, 2005頃)、最適制御で滑らかに解くCILQR/MPC系、近年は学習ベースのプランナ(前述のエンドツーエンド運転)へと発展しています。学習ベースであってもFrenet座標で出力を表現する手法は多く、依然として基礎概念として重要です。

## 論文の実験結果(定量データ)

Werling et al. (2010) は実車実験とシミュレーションで手法を検証しました。論文は厳密な大規模ベンチマーク数値よりも、(1) 高速道路の車線維持・車線変更、(2) 市街地での合流・停止・追従、といった複数シナリオで滑らかで実時間な軌道が生成できることを示すデモが中心です。

定量面で押さえるべき点は計算時間です。手法はサンプリングと多項式の解析解(連立一次方程式)で構成されるため、当時の計算機でも1サイクルを数ミリ秒〜十数ミリ秒オーダーで回せたと報告されており、車載で要求される実時間性(おおむね10〜100ms周期、報告による)を満たします。これが、最適化を毎回フルに解く重い手法に対する実用上の優位でした。

近年のオープンソース実装(例えばFrenet Optimal Trajectoryの再実装)では、横方向の終端位置を例えば ±数メートルを0.5m刻み、終端時間を数通り、終端速度を数通りサンプリングして、合計数十〜数百候補を生成し、その中からコスト最小かつ衝突回避を満たすものを選ぶ、といった構成がよく示されます。候補数を増やすほど良い軌道を見つけやすい反面、計算量が候補数に比例して増えるトレードオフがあります。

## メリット・トレードオフ・限界

メリット。(1) 道なりの直感に合い、縦横を分離して問題を単純化できる。(2) ジャーク最小多項式は解析解で高速、かつ乗り心地が良い。(3) サンプリング+コストの枠組みは実装・調整が容易で実時間。(4) 解釈性が高く、なぜその軌道を選んだかを説明できる。

トレードオフと限界。
- 基準線への依存: 良い基準線(中心線)が必要。交差点・駐車場・基準線が無い/曖昧な場所では破綻しやすい。基準線が急カーブだとFrenet⇔x-y変換が不安定になることがある(横方向dが大きいと曲率の扱いで歪む)。
- 局所性: サンプリングした候補の中からしか選べないので、サンプリング範囲外の良い解は見つからない。候補を増やせば計算量が増える(網羅性と速度のトレードオフ)。
- 縦横分離の近似性: 本来、急ハンドルと急加減速は相互作用(タイヤの摩擦円)するが、分離してしまうとその連成を無視しがち。高ダイナミクス領域では不正確。
- 動的環境への弱さ: 他車の将来予測が外れると、選んだ軌道が後から危険になる。予測モジュールの質に強く依存する。
- 重み調整の手間: コスト関数の重み k_* を手で調整する必要があり、状況ごとの最適調整が難しい。

## 発展トピック・研究の最前線
- 最適化ベースとの融合: サンプリングで初期解を作り、MPC(モデル予測制御、将来の数秒を最適化で解く制御)で精緻化するハイブリッド。
- 学習との接続: コスト関数や基準線を学習で求める、あるいはエンドツーエンドの出力をFrenet座標で表現して安定化する。
- 不確実性の取り込み: 他車予測の不確実性を考慮した安全マージン、確率的占有を使った衝突評価。
- 交差点・非構造環境への拡張: 基準線が定義しにくい場面でのグラフ探索やlatticeとの組み合わせ。
