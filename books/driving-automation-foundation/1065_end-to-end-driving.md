# エンドツーエンド運転

## ひとことで言うと
エンドツーエンド運転(End-to-End Driving)とは、カメラやLiDARなどのセンサが取り込んだ生の入力から、最終的なハンドル角・アクセル・ブレーキ(あるいは走るべき軌道)までを、一つの学習可能なニューラルネットワークでまとめて計算してしまうアプローチです。途中の「物体検出」「車線認識」「経路計画」といった工程を人間が手で設計して繋ぐのではなく、入力から出力までを一気通貫(end-to-end、端から端まで)で学習します。

## 直感的な理解
従来の自動運転は、工場の流れ作業のように複数のモジュール(部品)を直列に並べて作ります。

```
[センサ] → [物体検出] → [追跡] → [地図構築] → [予測] → [経路計画] → [制御] → [車両]
            モジュール1  モジュール2  ...
```

各モジュールはそれぞれ独立に開発・最適化されます。これを「モジュラーパイプライン」と呼びます。利点は、各段の途中結果(例: ここに歩行者がいる、という検出結果)を人間が見て確認・デバッグできることです。

問題は、各モジュールが「自分の指標」だけを最適化してしまい、最終目標である「安全に運転する」とは必ずしも一致しないことです。例えば物体検出器は「検出の正確さ(mAP)」を上げようとしますが、検出の正確さが0.1上がっても運転がうまくなるとは限りません。逆に、本当に運転に効く情報が途中で捨てられることもあります。これを「情報のボトルネック」や「目的の不整合(objective misalignment)」と呼びます。

エンドツーエンド運転はこの流れ作業を一つの巨大な関数 f に置き換えます。

```
[センサ入力 x] → [ニューラルネット f] → [運転出力 y]
```

y(運転出力)が良くなるように、ネットワーク全体のパラメータを誤差逆伝播(backpropagation、出力の誤差を入力側へ逆向きに伝えて重みを更新する手法)で一度に学習します。これにより「最終目標に直結した特徴」がデータから自動的に獲得されることを期待します。

## 基礎: 前提となる概念

エンドツーエンド運転を理解するために、いくつかの基礎概念を押さえます。

模倣学習(Imitation Learning): 人間の運転データ(入力センサと、その時人間が取った操作)を集め、「同じ入力に対して人間と同じ出力を出せ」とネットワークに教える方法です。最も単純な形は行動クローニング(Behavior Cloning)で、これは入力 x に対する人間の行動 a* を正解ラベルとして、ネットワークの出力 a と a* の差(損失)を最小化するだけの教師あり学習です。損失の例は L = ‖a − a*‖²(二乗誤差、出力と正解の差を二乗して足したもの)です。

共変量シフト(Covariate Shift): 行動クローニングの最大の弱点です。学習データは人間が運転した「うまくいっている軌道」ばかりなので、ネットワークは道の真ん中を走る状況しか見たことがありません。ところがいざ自分で運転すると、小さなミスで少し道の端に寄ります。すると「端に寄った状態」は学習データにほとんど無く、ネットワークはどう戻ればいいか分からず、さらにずれていき、最後に道を外れます。誤差が時間とともに雪だるま式に増えるこの現象を「複利的誤差(compounding error)」とも呼びます。

模倣学習の出力形式: 出力には2系統あります。(1) 直接制御を出す(ハンドル角・速度)。(2) 軌道(trajectory、これからT秒間に通る点列、例えば0.5秒ごとの(x, y)座標)を出し、別の低レベル制御器(コントローラ)が軌道に追従する。後者の方が安定しやすく、近年の主流です。軌道の典型的な形は shape [T, 2] のテンソル(時刻数 × xy座標)です。

## 仕組みを詳しく

代表的なエンドツーエンドネットワークの構造を、入力から出力まで追います。ここでは近年のクエリベース(query-based)設計を例にします。

1. 画像エンコーダ(backbone)。複数カメラの画像(例: 6カメラ、各 shape [3, 900, 1600] = チャンネル×高さ×幅)を畳み込みネットやTransformerに通し、各画像から特徴マップを得ます。

2. BEV(Bird's-Eye-View、鳥瞰図)への変換。複数カメラの特徴を、車の真上から見た一枚の地図状の格子(例: shape [256, 200, 200] = 特徴次元×縦×横、各セルが地面の0.5m四方に対応)に投影します。視点がバラバラなカメラ画像を共通の座標系にそろえることで、後段が扱いやすくなります。

3. クエリによる情報抽出。検出したい対象(他車・歩行者)や、自車が取りたい行動を「クエリ(query、知りたいことを表すベクトル)」として用意し、Attention(注意機構、クエリが関連する特徴を選んで集める仕組み)でBEV特徴から必要な情報を吸い上げます。

4. プランニング・ヘッド。自車用のクエリ(ego query)が、周囲の物体クエリや地図情報を参照して、最終的に将来軌道 [T, 2] を出力します。

学習の損失は複数を足し合わせます。模倣損失(人間軌道との差) + 検出損失 + 地図セグメンテーション損失 など。これを「マルチタスク学習」と呼び、補助タスク(auxiliary task、本来の目的ではないが学習を助ける副タスク)が中間表現を意味のあるものに整え、純粋なブラックボックスより解釈性と性能を上げます。

## 手法の系譜と主要論文

エンドツーエンド運転の発想は意外に古く、ALVINN(Pomerleau, NeurIPS 1988)が最初期です。これはわずか数百ノードの全結合ネットで、カメラ画像から直接ハンドル角を出力する模倣学習を1980年代に実演しました。

PilotNet / DAVE-2(Bojarski et al., NVIDIA, 2016, arXiv:1604.07316)は、現代的なCNN(畳み込みニューラルネット)でこれを再現しました。単一の前方カメラ画像(入力 [3, 66, 200])から9層のCNNを通してハンドル角(出力1値)を回帰します。学習データは約72時間の人間運転。複数カメラと画像のずらしによってデータ拡張(data augmentation、わざと逸脱した状況を人工的に作って学習)を行い、共変量シフトを部分的に緩和したのが工夫でした。

その後、模倣学習の限界を埋める研究が続きます。DAgger(Ross et al., AISTATS 2011, arXiv:1011.0686)は、学習中のネットワークに実際に走らせ、危うい状況で専門家(人間や強い計画器)に正解を聞いて追加学習する手法で、共変量シフトを理論的にも緩和しました。

条件付き模倣学習 CIL(Codevilla et al., ICRA 2018)は、交差点で「右折/左折/直進」のどれを取るかが画像だけでは決まらない問題(行動の多峰性)に対し、ナビ指示を条件として与える分岐構造を導入しました。

ニューラルネットを再びモジュラーに近づけたのがUniAD(Hu et al., CVPR 2023, Best Paper, arXiv:2212.10156)です。検出・追跡・地図・予測・占有・計画を全てTransformerのクエリで繋ぎ、すべてを最終目標(プランニング)に向けて協調学習しました。「最終目標を頂点に据え、中間タスクはそれに奉仕する」という設計思想を明確にしました。

VAD(Vectorized Autonomous Driving、Jiang et al., ICCV 2023, arXiv:2303.12077)は、UniADの重いラスター(画素的)表現を捨て、地図や他車を「ベクトル(点列)」で表現することで大幅に高速化しました。

近年は大規模言語モデルを組み込むDriveGPT/LMDriveや、生成モデルで世界をシミュレートするワールドモデル(GAIA-1など)へと広がっています。

## 論文の実験結果(定量データ)

UniAD(nuScenesデータセットで評価。nuScenesは6カメラ・LiDAR付きの大規模実走行データ)の報告値です。プランニングの主指標は2つあります。

L2誤差(L2 displacement error): 予測軌道と実際に人間が走った軌道の、各時刻の位置のずれ(メートル)。小さいほど良い。UniADは平均L2をおよそ1.03m(1秒0.48m / 2秒0.96m / 3秒1.65mの平均)と報告し、先行のST-P3(平均約1.43m)を大きく下回りました。

衝突率(Collision Rate): 予測軌道で走った場合に他物体とぶつかる割合(%)。UniADは平均約0.31%で、ST-P3の約1.27%から約4分の1に低減したと報告されています。数値の意味は、3秒先まで計画した軌道が周囲とぶつからない安全性が大きく向上した、ということです。

VADは、VAD-Tiny構成でUniADと同等以上のプランニング精度を保ちつつ、推論速度を大幅に上げたと報告しています。論文ではVAD-Tinyがおよそ16.8 FPS(1秒あたりの処理フレーム数)、UniADがおよそ1.8 FPS程度とされ、約9倍前後の高速化です(報告による)。これは「ベクトル表現に変えることで重い格子計算を省ける」ことの定量的な裏付けです。

アブレーション(ablation、構成要素を一つずつ外して効果を測る実験)では、UniADは追跡・地図・モーション予測・占有予測の各補助タスクを段階的に外すとプランニング性能(L2・衝突率)が劣化することを示し、「中間タスクの協調がプランニングを助ける」という主張を支えました。なお後年、これらの評価指標(特にnuScenesの開ループ評価)は、過去軌道をそのまま延長するだけの単純なベースラインでも良い値が出てしまう、という批判(AD-MLPやBEV-Plannerの指摘、2023-2024)も出ており、数値の解釈には注意が要ります。

PilotNetは定量ベンチというより実車デモが中心で、自律走行率(autonomy、人間が介入せず自走できた時間の割合)としておよそ98%程度を報告していますが、評価条件依存であり「およそ」の値です。

## メリット・トレードオフ・限界

メリット。(1) 手設計の中間インターフェースが不要で、データから運転に効く特徴を直接学べる。(2) モジュール間で情報が失われにくい。(3) 最終目標(安全運転)に向けて全体を一貫最適化できる。

トレードオフと限界。
- 解釈性: 純粋なブラックボックスでは「なぜそう判断したか」が分かりにくく、事故時の原因究明や認証(safety certification)が困難です。UniAD系が中間タスクを明示するのは、この解釈性を取り戻す妥協でもあります。
- データ効率と希少事象: 模倣学習は「めったに起きない危険事象(ロングテール)」のデータが極端に少なく、そこで弱い。共変量シフトの根本問題も残ります。
- 評価の難しさ: 上述のとおり開ループ評価(録画データに対して軌道を当てるだけ)は実際の運転能力を過大評価しがちで、本来はシミュレータ上の閉ループ評価(自分の出力が次の状態を変える設定)で測るべきです。CARLAなどのシミュレータが使われます。
- 因果の取り違え(causal confusion): 入力の中の「楽な相関」(例: 前の時刻の自車速度)に頼り、本質的な原因(信号・歩行者)を見ない癖がつくことが知られます。

## 発展トピック・研究の最前線
- ワールドモデル統合: 将来のセンサ観測を生成的に予測し、その内部で計画する(GAIA-1, MILE)。希少事象を生成して学習データを補える可能性があります。
- 言語・基盤モデルの導入: 大規模視覚言語モデルで運転判断を言語的に説明させ、解釈性と汎化を両立させる研究(DriveVLMなど)。
- 閉ループ強化学習との融合: 模倣で初期化し、シミュレータ内の強化学習で危険回避を磨く。
- 拡散モデルによる軌道生成: 軌道を確率分布として多峰的に生成し、行動の曖昧さ(右に避けるか左に避けるか)を表現する(後述の生成モデルとの接続)。
