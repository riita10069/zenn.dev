# 軌道模倣損失（Trajectory Imitation Loss）

## ひとことで言うと

モデルが「これから車をどう走らせるか」を表す軌道(trajectory)を出力したとき、その予測を「お手本の人間ドライバーが実際に走った軌道」にどれだけ近づけられているかを測る計算式のことです。この値(損失)が小さくなるようにモデルを調整していくのが、end-to-end 運転モデルの一番中心になる学習作業です。多くの現代的な運転モデルが、この軌道回帰損失を主タスクとして採用しています。

ここで「損失(loss)」とは、機械学習で「予測がどれくらい間違っているか」を1つの数値で表したものを指します。損失が大きいほど予測がお手本からズレている、小さいほどお手本に近い、という意味です。

## 直感的な理解

自動運転モデルに「正しい運転」を教える方法はいくつかありますが、end-to-end の主流は模倣学習(imitation learning)です([模倣学習による軌道予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0561_imitation-learning-trajectory))。人間の運転データ(カメラ映像と、そのときドライバーが実際にとった軌道のペア)を大量に集め、「この映像が来たら人間はこう走った。だから君も同じように走りなさい」とモデルに真似させる学習です。

なぜ模倣なのか。自動運転で「正しい行動」を1つ1つルールで書き下すのは現実的に不可能だからです。「前に歩行者がいたら止まる」「カーブでは速度を落とす」といったルールを無数の状況について手で書くのは破綻します。一方、人間は日常的に上手に運転できています。その運転記録さえあれば「入力(映像)→ 出力(軌道)」の対応を丸ごと学べる、というのが模倣学習の発想です。

このとき必要になるのが「予測した軌道」と「人間の軌道(教師データ, ground truth, 略して GT)」のズレを数値化する仕組みです。それが軌道模倣損失です。これがなければ、モデルは「自分の予測がどれくらい良いか/悪いか」を知る手がかりがなく、学習が一切進みません。損失は backward(誤差逆伝播)の出発点であり、損失がスカラー(1つの数値)でなければ勾配が定義できない、という意味でも中心的な部品です。

具体例。人間ドライバーが「ゆるやかに左へ曲がりながら少し加速」という軌道を走ったのに、モデルが「直進して減速」を出したら、この2つの差を数値にして「これだけ間違っている、もっと左へ加速する方向に直しなさい」という信号(勾配, gradient)を作ります。

## 基礎: 前提となる概念

- 回帰(regression)損失: 連続値を当てる問題で「予測値と正解値の差」を測る損失。分類(クラスを当てる)とは別物で、軌道予測は連続値の回帰です。
- L1 損失(絶対誤差): `|予測 - 正解|`。外れ値に頑健だが、誤差ゼロ付近で微分が不連続(勾配の符号が ±1 で切り替わる)。
- L2 損失 / MSE(二乗誤差): `(予測 - 正解)²`。滑らかだが、大きな外れ値を二乗で強く罰するため外れ値に引きずられやすい。誤差が大きいほど勾配も大きくなり、学習が不安定になりうる。
- Smooth L1 / Huber 損失: 誤差が小さいときは L2 のように滑らか、大きいときは L1 のように頑健、という両者のいいとこ取り。物体検出(Fast R-CNN)由来で、座標回帰の定番([軌道損失の関数選択（Smooth L1 / MSE）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0581_smooth-l1-vs-mse-loss))。
- reduction: 損失を「要素ごとに残す(none)」か「平均する(mean)」か「合計する(sum)」かの指定。途中で時間重みをかけたい場合は一旦 none で残す。
- スカラー(scalar): backward に渡す最終的な1つの数値。要素ごとの誤差を最終的に1つにまとめてから逆伝播します。
- 正規化(normalization): 加速度(m/s²)と曲率(1/m)は単位もスケールも違うため、そのまま足すと片方が支配的になります。各信号を標準化(平均0・分散1)してから損失を測るのが普通で、これを怠ると「スケールの大きい信号ばかり最適化される」失敗が起きます。

## 仕組みを詳しく

典型的な軌道模倣損失の実装を順に分解します。

### 入力の形(tensor shape)

軌道出力ヘッド(Policy)は、1サンプルあたり `num_timesteps × num_signals` 個の数値を吐きます。よくある設定は `num_timesteps=64`, `num_signals=2`、つまり 64 ステップ × 2 信号 = 128 次元のベクトルです。

- `num_timesteps=64`: これから先を 64 個の時刻に区切る。10 Hz(1 秒間に 10 回)刻みなら、64 ステップ = 6.4 秒先までの計画を意味します。
- `num_signals=2`: 各時刻について 2 つの数値。たとえば「加速度」と「曲率」(制御に近い表現)や、「x, y 位置」(waypoint 表現)([waypoint出力 vs 制御量出力ヘッド](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0154_waypoint-vs-control-output-head))。

損失関数の forward に渡されるのは次の2つです。

- `trajectory_pred`: モデルの予測。形 `[B, 128]`(B はバッチサイズ = まとめて処理するサンプル数)。
- `trajectory_target`: 人間ドライバーのお手本。形も `[B, 128]`。

### ステップ1: 形を整える

`[B, 128]` を `[B, 64, 2]` に組み替えます(`view(B, num_timesteps, num_signals)`)。「各サンプルについて、64 の時刻それぞれに 2 信号が並ぶ」という意味の形になり、時刻ごとの扱いが容易になります。

### ステップ2: 要素ごとの誤差を計算

`loss_fn(pred, target)` で各数値の差を計算します。`reduction="none"` を指定すると平均をとらず、`[B, 64, 2]` のまま「要素ごとの誤差」が出ます。損失関数は smooth_l1(よく使われるデフォルト)か mse を選びます([軌道損失の関数選択（Smooth L1 / MSE）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0581_smooth-l1-vs-mse-loss))。

### ステップ3: 信号方向を平均して時刻ごとの誤差にする

`per_element_loss.mean(dim=2)` で、最後の次元(2 信号)を平均し、`[B, 64]` にします。これで「各サンプル・各時刻の誤差の大きさ」が1つの数になります。

### ステップ4: 時間重みをかける

`weighted = per_timestep_loss * temporal_weights` で時刻ごとに重みをかけます。`temporal_decay=0.95` なら `weights[t] = 0.95 ** t` を使い、t=0(すぐ先)は重み 1.0、t=63(6.4 秒先)は `0.95**63 ≈ 0.039` とほぼ無視に近くなります。近い未来ほど重視する設計で、これは「直後の運転の正確さが安全に直結し、遠い未来は不確実で当てにくい」ためです([軌道損失の時間重み付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0584_temporal-loss-weighting))。

### ステップ5: 全体を平均してスカラーにする

`weighted.mean()` で全部を平均し、1つの数値にします。この数値が小さくなる方向にモデルの重みを更新します。

### 数値例(before / after の感覚)

バッチ1件、ある時刻について、予測の加速度 `0.3 m/s²`、教師 `0.5 m/s²` とします。MSE なら誤差は `(0.3-0.5)² = 0.04`。曲率も同様に出して2信号を平均し、時間重みをかけ、最後にバッチ・時刻すべてで平均します。学習が進むと予測が `0.45 m/s²` に近づき、誤差は `(0.45-0.5)² = 0.0025` に縮みます。この縮小が「お手本に近づいた」ことの定量的な姿です。

```
予測軌道 [B,128] ─reshape→ [B,64,2] ─┐
教師軌道 [B,128] ─reshape→ [B,64,2] ─┤→ 要素誤差[B,64,2]
                                      → 信号平均[B,64]
                                      → 時間重み[B,64]
                                      → 全平均 → scalar loss → backward
```

### 設計上の注意点

実装で見落とされがちな点を挙げます。第一に、上で触れた信号間のスケール正規化。加速度と曲率を生のまま MSE で足すと、典型的に値域の広い側に最適化が偏ります。第二に、無効な未来ステップのマスク処理。記録の終わり際やセンサ欠損で「未来 64 ステップ」が揃わないサンプルがあり、欠損部分を損失から除外しないと意味のない誤差を学んでしまいます。第三に、時間重みの正規化。重みをかけたあと単純平均すると、重みの総和に依存して損失のスケールが変わるため、重みの和で割って基準を揃えることがあります。

## 手法の系譜と主要論文

### ALVINN(Pomerleau, 1989, NeurIPS)

自動運転に模倣学習を持ち込んだ最初期の研究。カメラ画像を入力にニューラルネットでハンドル角を直接出力させ、人間の運転データで学習しました。「知覚から制御までを1つのネットで end-to-end に学ぶ」発想の原点で、当時のルールベース手法が複雑な実環境に対応しきれなかったことが動機です。シンプルな道路での実車自律走行を示しました。トレードオフは、学習データに出てこない状況に弱いこと(後に分布シフトと呼ばれる弱点)。軌道模倣損失も原理的にこの弱点を引き継ぎます。

### Smooth L1 / Huber の出自(Fast R-CNN, Girshick 2015)

軌道のような座標・連続値の回帰で Smooth L1 が定番になったのは、Girshick, "Fast R-CNN", ICCV 2015 がバウンディングボックス回帰に Smooth L1 を使い、MSE より外れ値に頑健で学習が安定すると示したことが大きいです。Smooth L1 は誤差の絶対値が 1 を境に、内側は二乗(滑らか)、外側は線形(頑健)に切り替わる関数で、たまにある大きな誤差で勾配が爆発するのを防ぎます。軌道回帰でも同じ理由で Smooth L1 が好まれます([軌道損失の関数選択（Smooth L1 / MSE）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0581_smooth-l1-vs-mse-loss))。

### Learning by Cheating(Chen et al., CoRL 2019)

模倣学習を2段階に分ける手法。特権情報(privileged information)を見られる教師ネットを先に学習し、カメラだけの生徒ネットにその出力を模倣させます。最初からカメラだけで人間運転を真似るより学習が容易になる、という洞察。CARLA で当時の最高性能を達成しました。軌道(行動)を回帰で模倣する点は本損失と同系統です。

### UniAD(Hu et al., CVPR 2023, Best Paper)/ VAD(Jiang et al., ICCV 2023)

近年の end-to-end 運転でも、最終的な計画(軌道)の学習には L1 系の軌道回帰損失が使われます。UniAD は L1 軌道損失に衝突を避けるためのペナルティ項を加え、VAD はベクトル化した軌道・シーン表現を回帰し、さらに走行可能領域や他車との衝突を避けるための制約項を損失に組み込みます。いずれも軌道模倣損失と同系統で、「素の回帰損失に、安全や効率のための追加項をどう足すか」が違いです。

### offline 評価の落とし穴(Codevilla et al., ECCV 2018)

Codevilla et al., "On Offline Evaluation of Vision-based Driving Models", ECCV 2018 は、軌道模倣損失(open-loop の予測誤差)が小さいことと、実際にうまく運転できること(closed-loop の成功率)が必ずしも一致しないことを示しました。これは「軌道損失を下げる」最適化の限界を突いた重要な指摘で、評価指標としての軌道誤差の解釈に注意を促します。

## 論文の実験結果(定量データ)

- Fast R-CNN(Girshick 2015)は、座標回帰に Smooth L1 を使うことで MSE(L2)より学習が安定し、検出精度(mAP)が向上することを報告しました。L2 だと大きな誤差の勾配が爆発しがちで学習が不安定になるのを、Smooth L1 が誤差 1.0 を境に L1 に切り替えて抑える、という効果です。軌道回帰がこの損失を継承する直接の根拠になっています。

- UniAD(Hu et al. 2023)は nuScenes の計画タスクで、L1 軌道損失を主に、衝突回避や知覚補助タスクを組み合わせ、計画の平均 L2 誤差(予測軌道と GT 軌道の平均距離、単位はメートル)と衝突率を従来手法より低減したと報告しています(平均 L2 でおよそ 1 メートル前後の水準、衝突率も低下)。ここで「平均 L2 誤差」は予測軌道が人間軌道からどれだけ離れているかを測る最も基本的な指標で、小さいほど人間に近い軌道を出せていることを意味します。衝突率は安全性の直接指標で、軌道誤差が同程度でも衝突率が違いうるため、両方を見る必要があります。

- Codevilla et al. (2018) は、複数のモデルについて offline の予測誤差(軌道/操作の回帰誤差)と online の運転成功率の相関を調べ、両者の相関が弱い場合があることを定量的に示しました。具体的には、予測誤差が小さいモデルが必ずしも運転成功率が高いとは限らず、誤差は小さいのに実走行で破綻する例を報告しています。これは「軌道模倣損失を下げる」ことが目的の代理(proxy)に過ぎず、真の目的(安全な運転)とズレうることを示す重要なアブレーション的知見です。原因の一つは、open-loop では教師状態の上で1ステップ先を当てるだけで済むのに、closed-loop では自分の誤差が次の入力に積み重なる(分布シフト)からです。

これらの数値の含意は、「軌道模倣損失は学習を駆動する優れた信号だが、その値の小ささだけで運転性能を保証はできない。closed-loop 評価や安全項との併用が必要」という点です。

## メリット・トレードオフ・限界

メリット
- 人間の運転データさえあれば、明示的なルールを書かずに「入力 → 軌道」を学べる。
- 計算が単純で安定して学習が進む。実装も数十行で済む。
- 軌道という解釈しやすい量を直接学ぶので、出力の意味が明確で後段制御に渡しやすい。

トレードオフ・限界
- 分布シフトに弱い。教師がほとんど経験していない状況(大きく道を外れた状態など)では、お手本がないため誤った軌道を出しやすい。
- mode averaging: 複数の正解があり得る状況(左右どちらでも避けられる等)で平均を取り、どちらつかずの危険な軌道になることがある。回帰損失(L1/L2)は本質的に「正解の平均」を出すよう促すため、確率分布を出す手法(混合密度・拡散・フローマッチング系)で緩和する。
- 教師データの質に強く依存する。下手・危険な運転を学べば、モデルもそれを真似る。
- 損失値と運転性能のズレ(Codevilla の指摘): open-loop の軌道誤差を下げても closed-loop で破綻しうる。損失は代理目標に過ぎない。
- 信号スケールや欠損ステップの扱いを誤ると、見かけの損失は下がっても無意味な最適化になる。実装上の落とし穴が多い。
- 未解決寄りの課題として、安全制約(衝突・走行可能領域逸脱)を軌道損失にどう組み込むか、マルチモーダルな未来をどう表現するか、が活発に研究されています。

## 発展トピック・研究の最前線

- 安全を組み込んだ軌道損失: 衝突ペナルティ、走行可能領域からの逸脱ペナルティ、快適性(加々速度/jerk)正則化などを軌道回帰損失に足す方向(UniAD/VAD 系)。
- マルチモーダル軌道出力: 単一の回帰では mode averaging が避けられないため、複数候補(anchor)の分類+回帰、混合密度ネットワーク、拡散ポリシーによる軌道生成へ。
- closed-loop を見据えた学習: open-loop 損失の限界を踏まえ、シミュレータ closed-loop での評価・微調整、DAgger 的なデータ反復収集との併用。
- 時間重みの設計: 一律の指数減衰でなく、不確実性に応じて時刻ごとの重みを学習する方向([軌道損失の時間重み付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0584_temporal-loss-weighting))。
