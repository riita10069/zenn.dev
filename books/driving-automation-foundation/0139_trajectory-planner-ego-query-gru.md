# TrajectoryPlanner (ego query + GRU)

## ひとことで言うと
「自車を表す1個の問い合わせ票(エゴクエリ)」を使い、未来の通過点を1ステップずつ順番に予測する軌跡計画の仕組みです。各ステップで、GRU(時系列を扱う小さなネットワーク)の内部状態がBEV地図にクロスアテンションで注意を向け、その時刻のウェイポイント(通過点)を出します。BEV全体をMLPに流し込む素朴なやり方([BEVフラット化MLPのスケール問題](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0143_flatten-mlp-scaling-problem))を、BEV解像度に依存しない形に置き換えた、end-to-end 自動運転の標準的なプランナ構成です。

## 直感的な理解

軌跡計画(trajectory planning)とは「車がこれから数秒間、どこを通るか」を予測することです。出力は時系列の通過点で、例えば0.1秒ごとに6秒先まで、各点に(x, y)座標や(加速度, 曲率)を持たせます。仮に64点 × 2値なら 64 × 2 = 128個の数値です。下流の制御器は、この通過点列をたどるようにステアリングとアクセル・ブレーキを動かします。

なぜ素朴に「BEV地図全体をMLPに入れて軌跡を一気に出す」ではダメなのか。低解像度BEV(7×7など)なら成立しますが、安全のためBEVを高解像度(数百×数百マス)にすると、フラット化した入力が数千万次元に膨れ上がり、最初の全結合層だけで数十億パラメータになって破綻します([BEVフラット化MLPのスケール問題](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0143_flatten-mlp-scaling-problem))。

そこで2つの発想が鍵になります。第一に「エゴクエリ(ego query)」 ― 自車1台を表す固定サイズ(例: 256次元)の問い合わせ票を1個だけ持ち、それがBEVの必要な場所だけを能動的に見に行く。BEVが何マスあろうとクエリは256次元のままなので、解像度に依存しません。第二に「自己回帰デコード(autoregressive decoding)」 ― 未来の通過点を一気にではなく、t=1, t=2, …と1つずつ順番に予測する。前の予測を次の入力に使うので、軌跡が時間的に滑らかにつながり、運動の連続性(車は瞬間移動しない)が自然に守られます。

身近な例えで言えば、地図全体を丸暗記して一発で経路を吐くのではなく、「今いる地点から次の一歩を決め、進んだ先でまた周囲を見て次の一歩を決める」という、人間がカーナビを見ながら運転するのに近い逐次的なやり方です。

## 基礎: 前提となる概念

「RNN(Recurrent Neural Network、再帰型ニューラルネットワーク)」は、系列データを1要素ずつ処理し、「隠れ状態(hidden state)」と呼ばれる内部メモリにこれまでの文脈を持ち越すネットワークです。各ステップで「前の隠れ状態」と「今の入力」から「次の隠れ状態」を計算します。

「GRU(ジーアールユー、Gated Recurrent Unit)」は、Cho et al. が2014年(EMNLP, arXiv:1406.1078)に提案したRNNの一種です。GRUには「更新ゲート(update gate)」と「リセットゲート(reset gate)」という2つのゲートがあります。ゲートとは0〜1の値で「情報をどれだけ通すか」を制御する弁のことで、更新ゲートは過去の隠れ状態をどれだけ持ち越すか、リセットゲートは過去をどれだけ忘れて新しい入力を取り込むかを学習で決めます。これにより、素朴なRNNが抱える「勾配消失(長い系列で勾配が薄れ、遠い過去を学べない)」を緩和します。LSTM(3ゲートのRNN)を簡素化したもので、パラメータが少なく学習が速いのが特徴です。軌跡計画では「t時点までの状況を覚えたうえで、t+1の通過点を出す」という自己回帰を書くのに使います。

「クロスアテンション(cross-attention)」は、ある問い合わせ(クエリ)が別のデータ(ここではBEV特徴)のどこをどれだけ重視するかを計算し、重み付き平均で情報を吸い上げる仕組みです。エゴクエリがBEVのどこを見るかは、速度や状況で変わります(高速時は前方遠く、交差点では左右)。これを学習で獲得します。BEVは大きいので、全マスを見るフルアテンションではなく、参照点周りの数点だけ見る変形クロスアテンション([Deformable Cross-Attention](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0259_deformable-cross-attention))を使うのが効率的です。

「ウェイポイント(waypoint)」は軌跡を構成する個々の通過点です。一連のウェイポイントを時刻順に並べたものが計画軌跡で、これが下流の制御に渡されます。

「open-loop / closed-loop 評価」も最初に押さえます。open-loop は録画データに対し「人間の実走行とどれだけ近い軌跡を出したか」を測る方式。closed-loop はシミュレータ上で実際に車を走らせ、出した軌跡で次の状態が変わるのを繰り返し、衝突や逸脱が起きないかを測る方式です。後述するように、両者で結論が食い違うことがあり、評価の妥当性が研究上の論点になっています。

## 仕組みを詳しく

代表的な `TrajectoryPlanner` の構成はこうです(疑似コード)。

```python
class TrajectoryPlanner(nn.Module):
    def __init__(self, embed_dim=256, num_timesteps=64, num_signals=2):
        self.ego_query = nn.Embedding(1, embed_dim)               # 自車を表す1個のクエリ
        self.gru = nn.GRUCell(embed_dim, embed_dim)               # 自己回帰の心臓部
        self.cross_attn = DeformableCrossAttention(embed_dim, n_heads=8, n_points=4)  # BEV参照
        self.ego_state_proj = nn.Linear(ego_dim, embed_dim)       # 自車の運動状態を条件付け
        self.waypoint_head = nn.Linear(embed_dim, num_signals)    # 1点ぶんの出力
```

処理を1ステップずつ追います。形(shape)も併記します。BEVは `[B, 256, H, W]`(B はバッチ、H×W はBEVマス数)。

1. エゴクエリ(256次元)を初期状態として用意。自車の過去の運動(エゴモーション履歴、過去の速度や舵角など)を `ego_state_proj` で256次元に変換し、文脈として混ぜる。「今どれくらいの速度でどちらを向いているか」を知らないと、妥当な軌跡は出せないためです。さらに、信号・ルート・運転コマンド(直進/左折/右折)など高レベルの意図も、ここで条件として注入できます。

2. 各未来時刻 t = 1 … T(例 64)について、次を繰り返す(自己回帰ループ)。
   - GRUに現在のクエリ状態を入れ、隠れ状態(hidden state、256次元)を1ステップ更新する。
   - その隠れ状態が、変形クロスアテンションでBEVの「今見るべき場所」だけをサンプリングして参照する([Deformable Cross-Attention](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0259_deformable-cross-attention))。例えば高速時は前方遠くを、交差点では左右を見る、といった注意配分を学習で獲得する。
   - `waypoint_head`(Linear)で、その時刻の通過点(例 2次元)を出力する。
   - この出力(と更新した隠れ状態)を次のステップの入力に反映させ、ループを続ける。

3. T ステップ分の通過点を並べて、軌跡 `[B, T, 2]`(= フラットにすると `[B, 2T]`)を得る。

決定的に重要なのは、入力サイズが一切BEV解像度に依存しないことです。エゴクエリは常に256次元、GRUの隠れ状態も256次元。BEVが450×300でも200×200でも、変形クロスアテンションは「数点だけサンプリングする」のでコストはBEVマス数に線形(2乗ではない)。だから[BEVフラット化MLPのスケール問題](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0143_flatten-mlp-scaling-problem)のような次元爆発・パラメータ爆発が起きません。

学習は、予測軌跡と人間の実走行軌跡の差を最小化する模倣学習(imitation learning)で行うのが基本です。損失は各時刻のL1/L2距離の総和や、衝突を避けるための補助損失(他エージェントの予測位置と重ならないようにするコスト)を足したものです。

before / after:

- before(フラット化MLP): BEVをフラット化 → MLPで一括出力。BEV解像度に縛られ高解像度で破綻。通過点同士の時間的つながりも明示されない。
- after(TrajectoryPlanner): エゴクエリがBEVをクロスアテンションで参照しながらGRUで1点ずつ自己回帰。BEV解像度非依存。通過点が時間的に滑らかにつながる。

出力ホライズン(予測する未来の長さ)の見積もりも設計上重要です。例えば64ステップ × 10Hz = 6.4秒先まで予測する場合、時速120km(秒速約33m)なら6.4秒で約213m進みます。前方の知覚レンジ([BEV知覚レンジ設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0263_bev-perception-range))を使い切れるよう、ホライズンと知覚距離を整合させます。また、GRUの最終隠れ状態(256次元)は「自車のこれからの意図を凝縮した表現」なので、これを下流の将来状態予測(future state prediction)モジュールに渡して、計画と予測を結合する設計もよく取られます。

## 手法の系譜と主要論文

エゴクエリ + 逐次デコーダの構成は、検出のクエリ手法をプランニングに展開する流れの中で確立しました。

UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)が、この構成の直接の手本です。エゴクエリを1個用意し、各未来時刻でその状態が変形アテンションでBEV(および周辺エージェントのクエリ)を参照し、MLPでウェイポイントを出します。動機は、知覚から計画までを1つのネットワークで end-to-end に学習し、計画タスクが知覚の中間表現(BEV)を直接活用できるようにすることです。トレードオフは、自己回帰ゆえの逐次処理で推論レイテンシ(遅延)が伸びる点です。

VAD(Jiang et al., ICCV 2023, arXiv:2303.12077)は、エゴクエリが「ベクトル化されたエージェント・地図クエリ」に注意を向けて軌跡を出します。BEVを生のラスタ(画素地図)として見るのではなく、意味のある少数のベクトル要素(各エージェントの動き予測、各車線のポリライン)に圧縮してから参照する点が違いです。効果は軽量・高速で、衝突回避の制約をベクトル表現上で明示的に課しやすいこと。トレードオフはベクトル化段での情報損失です。

Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)は、`cross_attn` の中身である変形アテンションの原典で、少数点サンプリングによりエゴクエリがBEVを安く参照できる土台を与えました([Deformable Cross-Attention](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0259_deformable-cross-attention))。

近年は、自己回帰の「逐次で遅い・誤差が蓄積する」弱点を回避する流れが強まっています。Diffusion Policy(Chi et al., RSS 2023, arXiv:2303.04137)に代表される拡散モデル系や、フローマッチング系の手法は、軌跡全体を一度に(あるいは少数の反復で)生成し、マルチモーダルな未来(右折・直進という複数の妥当解)を自然に表現できます。VADv2(Chen et al., 2024, arXiv:2402.13243)は、連続的な軌跡回帰を「大量の候補軌跡(語彙)からの確率的選択」に置き換え、行動の多峰性を確率分布として扱う方向を示しました。エゴクエリでBEVを参照するという核は維持したまま、デコーダ部分が自己回帰GRUからこれらの方式へ進化しているのが現状です。

## 論文の実験結果(定量データ)

評価は主に nuScenes の planning ベンチマークで行われ、指標は2つです。

L2誤差(L2 displacement error)は、予測した軌跡と人間ドライバの実走行軌跡(正解)との、1秒・2秒・3秒先での平均ユークリッド距離(メートル)です。小さいほど良い。これは「人間と同じように走れたか」の代理指標です。衝突率(collision rate)は、予測軌跡をたどると他物体とぶつかる割合(%)で、小さいほど安全です。

報告によると、UniAD は計画の平均L2誤差を当時のモジュール分離型ベースライン(知覚・予測・計画を別々に学習する従来パイプライン)より低くし、衝突率も下げました。具体的には、3秒先の平均L2でおよそ1.6m級、平均衝突率を従来比でおよそ半減し、1%未満の水準に抑えたことが報告されています(設定により数値は変動)。重要なアブレーションとして、占有予測や動き予測などの補助タスクを外すと計画性能が悪化することが示され、「知覚タスクを共同学習することが計画を助ける」という end-to-end の主張を裏づけています。

VAD は、ベクトル化表現により UniAD と同等以上の計画精度を保ちつつ、推論速度を大幅に向上させた(報告では UniAD のおよそ数倍のフレームレート、VAD-Tiny で十数 FPS 級)ことが特徴です。これは「BEVを生で持たずベクトルに圧縮する」ことの実利を示しています。

なお、これらの open-loop 評価には強い批判があります。AD-MLP(Zhai et al., 2023, arXiv:2305.10430)は「自車の過去状態だけを入れた極めて単純なMLPでも、nuScenes の open-loop L2 と衝突率で当時の最先端と肩を並べる、あるいは上回るスコアが出てしまう」ことを示しました。これは、nuScenes の運転の多くが「直進・等速」に偏っており、過去の運動を外挿するだけで当たってしまうため、知覚を本当に使っているかを open-loop 指標が測れていないことを意味します。このため、CARLA や nuPlan などのシミュレータでの closed-loop 評価の重要性が認識され、研究の評価軸が移りつつあります。

## メリット・トレードオフ・限界

メリット
- 入力サイズがBEV解像度に依存しない(エゴクエリ256次元固定)ので、高解像度BEVでも破綻しない。
- 変形アテンションでBEV参照が線形コスト。車載リアルタイムを狙える。
- 自己回帰なので通過点が時間的に滑らかにつながり、運転状況に応じて注意先を動的に変えられる。
- GRUの隠れ状態(256次元)を下流に渡せ、計画と将来予測を自然に結合できる。

トレードオフ・限界
- 自己回帰(1ステップずつ)は逐次処理で、ステップ数ぶんのループが推論レイテンシを押し上げる。GPU並列化と相性が悪い。
- 自己回帰は誤差が蓄積する(早い時刻の予測ミスが後の時刻に伝播)。これは exposure bias とも呼ばれる古典的なRNNの課題で、学習時は正解で条件付け(teacher forcing)しても推論時は自分の予測で条件付けるため分布がずれる。
- 単峰的になりやすい。右折・直進のように複数の妥当解がある状況で、それらの平均(どっちつかずの危険な軌跡)を出しがち。マルチモーダル性は拡散・フローマッチング・候補語彙系の方が表現しやすい。
- 実装がフラット化MLPより複雑(クエリ・GRU・アテンションの組み合わせ)。

これらの弱点ゆえ、研究コミュニティではGRU自己回帰デコーダから、軌跡全体を一括生成する拡散・フローマッチング方式や、少数制御点のパラメトリック方式(ベジェ曲線など)、候補語彙からの選択方式への移行が進んでいます。一方で「エゴクエリでBEVを参照する」という核となる発想自体は、これらの新しい方式でも共通して引き継がれています。

## 発展トピック・研究の最前線

マルチモーダル軌跡生成は今最も活発な方向です。拡散モデル(Diffusion Policy 系)やフローマッチングは、ノイズから軌跡を反復的に精製することで、複数の妥当な未来を確率分布として表現できます。逐次の誤差蓄積を避けつつ、分岐シーン(分岐路、合流)を自然に扱えるのが利点です。

言語モデルとの融合も進んでいます。VLM(Vision-Language Model、画像と言語を同時に扱うモデル)を運転に持ち込み、「なぜその軌跡を選んだか」を言語で説明したり、自然言語の指示(「次の交差点を左折」)を条件として軌跡生成する研究(DriveGPT4, LMDrive など)が登場しています。エゴクエリは、こうした言語条件を受け取る窓口としても機能し得ます。

評価方法論の進化も重要なトピックです。前述の通り open-loop L2 の限界が広く認識され、closed-loop シミュレーション(CARLA の Town、nuPlan のシミュレータ)での評価、および「人間らしさ」と「安全性(衝突・違反)」を分けて測る指標設計が議論されています。プランナの真価は録画への当てはめではなく、実際に閉ループで走らせたときに崩れないかで決まる、という認識が定着しつつあります。
