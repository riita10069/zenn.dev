# 自車軌道（ego trajectory）抽出

## ひとことで言うと

走行ログには「人間ドライバーが実際にどう走ったか」が記録されています。その記録から、「今この瞬間より後、自車がどう動いたか（加速・カーブ具合の系列）」を取り出して、モデルが予測すべき正解（教師信号、ground truth）にする処理です。模倣学習（imitation learning）で「お手本」を作る工程にあたります。

## 直感的な理解

自動運転モデルに「今の状況から、これからどう走ればいいか」を学ばせたいとします。一番直接的な教え方は、上手なドライバーの実際の走りを大量に見せて「こういう状況ではこう動くんだよ」と真似させることです。これが模倣学習です。

そのためには「正解の走り方」をデータとして取り出す必要があります。走行ログには毎瞬間の速度・向き・加速度などが記録されているので、ある時刻を「現在」と決め、それより後の系列を切り出せば「この状況で人間が実際に取った未来の動き」が手に入ります。これを正解にして、モデルの予測がそれに近づくよう学習します。正解がなければ何を目指せばよいか分からないので、この抽出は学習の出発点です。料理で言えば、レシピ（モデル）を磨く前に「完成品の見本」を用意する工程です。

## 基礎: 前提となる概念

模倣学習（imitation learning）は、専門家（ここでは人間ドライバー）の行動を正解として、観測から行動への写像を教師あり学習で覚える枠組みです。最も単純な形は行動クローニング（behavior cloning）で、「この入力にはこの出力」を回帰または分類で当てます。

軌道（trajectory）をどう数値で表すかが設計の肝です。素直には「未来の (x, y) 座標の列」ですが、別の表現として「制御量に近い系列」を使う流派があります。代表が前後加速度（acceleration、アクセル/ブレーキ相当）と曲率（curvature）の時系列です。曲率とは「カーブの曲がり具合」で、`曲率 = ヨーレート ÷ 速度` と定義されます。ヨーレート（yaw rate）は1秒あたりに車の向きがどれだけ変わるかです。曲率が0なら直進、大きいほどきついカーブを意味します。

なぜ座標ではなく加速度＋曲率なのか。これは車両運動の自然な記述であり、制御量に近いため学習が安定しやすいことが知られています。また座標は積分で復元できます（加速度を積分すれば速度、さらに積分すれば位置。曲率と速度から向きが決まり、向きと速度から軌跡が描けます）。絶対座標は時間とともに誤差が積もる（ドリフトする）一方、加速度・曲率は局所的な量なのでドリフトに強いという利点もあります。

数値の安定化に関わる前提も2つあります。1つは角度の連続化です。向き（heading）は通常 -π〜π（-180度〜180度）で記録され、北を超えると +179度から -179度へ「飛び」ます。そのまま差分を取ると「358度の急回転をした」と誤認するため、まず連続値に直してから変化率を計算します（NumPy では `np.unwrap`）。もう1つはゼロ除算の回避です。曲率は `ヨーレート ÷ 速度` なので、停車付近（速度≈0）で発散します。速度が極小なら曲率を0に固定して防ぎます。

## 仕組みを詳しく

典型的な実装では、まず生の車両状態（毎瞬間の速度・向き・緯度経度・加速度などのベクトル）の時系列から、軌道表現に使う数信号を導きます。例えば速度・前後加速度・ヨーレート・曲率の4信号です。

ステップ1: 生状態から信号を導く。

```python
speed   = vehicle_states[:, 0]                       # 速度
heading = np.unwrap(vehicle_states[:, heading_idx])  # 向き。±π の折り返しを連続化
accel_x = vehicle_states[:, accel_idx]               # 前後加速度
yaw_rate[1:] = np.diff(heading) / dt                 # 向きの変化率（dt は1ステップの秒数）
curvature = np.where(np.abs(speed) > eps, yaw_rate / speed, 0)  # 曲率（ゼロ除算ガード）
# → (T, 4): [speed, accel_x, yaw_rate, curvature]
```

`dt` はサンプリング周期です。例えば 10Hz 記録なら `dt = 0.1` 秒。`np.unwrap` と `np.where(...)` のガードがここで効きます。

ステップ2: 現在時刻を中心に過去と未来を切り出す。ある時刻インデックス `i` を「現在」と決め、過去 `H` ステップと未来 `F` ステップを取り出します。

```python
history = signals[i - H : i]            # (H, 4) 過去（文脈）
future  = signals[i + 1 : i + 1 + F]    # (F, 4) 未来（教師信号の素）
```

ステップ3: 用途ごとに信号を選ぶ。過去は文脈としてモデルへ入れるので全信号を使うことが多く、未来は予測対象を絞ります。よくある絞り方が、未来から前後加速度と曲率の2信号だけを取り出すものです。

```python
egomotion_history = torch.from_numpy(history.flatten())            # (H*4,) モデル入力の文脈
trajectory_target = torch.from_numpy(future[:, [accel, curv]].flatten())  # (F*2,) 教師信号
```

未来を2信号に絞るのは、軌道を「どれだけ加減速し、どれだけ曲がるか」という最小限の自由度で表すためです。速度は加速度の積分、向きはヨーレート（≒曲率×速度）の積分で復元できるので、この2つで軌道の形は決まります。

ステップ4: 損失で教師として使う。模倣損失は、モデルの予測軌道 `(B, F, 2)`（B はバッチサイズ）と教師 `trajectory_target` を突き合わせます。回帰には外れ値に強い Smooth L1 損失（誤差が小さいときは二乗、大きいときは絶対値のように振る舞う損失）がよく使われます。さらに時間減衰の重み `weight = γ^t`（γ は 0.95 程度、t は未来ステップ）を掛け、近い未来ほど重視する設計が一般的です。近未来の予測誤差は安全に直結するためです。

数値イメージ:

```
現在 i = 120、過去 H = 64、未来 F = 64、dt = 0.1s
 過去: signals[56:120]               → flatten 256次元 → モデル入力（文脈）
 未来: signals[121:185] の[accel, curv] → flatten 128次元 → 正解
 損失: Σ_t  0.95^t · SmoothL1(pred[:,t,:], target[:,t,:])  を平均
```

この設計の効果は、座標を直接回帰させるより滑らかで物理的に妥当な軌道が出やすいことです。座標直接回帰では各時刻の点が独立にずれてギザギザになりがちですが、加速度・曲率は微分量なので連続性が自然に保たれます。

## 手法の系譜と主要論文

運転の模倣学習の原点は ALVINN（D. Pomerleau, "ALVINN: An Autonomous Land Vehicle in a Neural Network", NeurIPS 1989）です。前方画像を入力に人間の操舵を回帰する3層ニューラルネットで、当時の計算機で実車を走らせました。「人間の操作を正解にする」という発想の源流であり、教師信号抽出という工程の祖先です。

End to End Learning for Self-Driving Cars（M. Bojarski et al., NVIDIA, arXiv:1604.07316, 2016）は、前方カメラから操舵角を直接予測する畳み込みネットを実車で示し、ALVINN を深層学習で蘇らせました。教師は実走行の操舵角です。操舵そのものではなく曲率＋加速度を教師にする流派は、より車両運動に即した表現を選んだ違いで、車種やステアリング比に依存しにくいという利点があります。

ChauffeurNet（M. Bansal et al., RSS 2019, arXiv:1812.03079）は、未来の自車軌道（waypoint列）を教師に学習しつつ、模倣だけでは稀な危険状況に弱いという問題に正面から取り組みました。学習データには「事故寸前からの復帰」がほとんど含まれないため、そこへ来たモデルは何をすべきか知りません。これが共変量シフト（covariate shift、学習時と運用時で遭遇する状態分布がずれる現象）です。ChauffeurNet は意図的に擾乱（軌道を少しずらした合成状況）を加えて復帰を学ばせ、模倣学習の代表的弱点を明示しました。

Learning by Cheating（D. Chen et al., CoRL 2019, arXiv:1912.12294）は、シミュレータ内で全情報（特権情報、privileged information）を見られる教師エージェントにまず軌道を学ばせ、その出力を画像しか見られない生徒に蒸留する二段構成を提案しました。教師軌道の質が生徒の性能上限を決めることを示し、「正解軌道の作り方」が性能を左右することを裏づけています。

軌道の表現については、Trajectron++（T. Salzmann et al., ECCV 2020, arXiv:2001.03093）など動的モデルを組み込む系統が、加速度・曲率（あるいはヨーレート）といった制御量で軌道を表し、座標は積分で復元する設計を採用しています。物理的に妥当で滑らかな軌道が出やすく、座標直接回帰より破綻しにくいことが報告されています。

## 論文の実験結果（定量データ）

ChauffeurNet は社内大規模データとシミュレーションで評価され、擾乱を加えた学習（合成perturbation）を入れると、入れない素の模倣に比べて、停止線無視・道路逸脱・衝突といった失敗事例が大きく減ることを報告しています。とくに「カーブで膨らんで対向車線にはみ出す」「停止が遅れる」といった共変量シフト由来の失敗が、擾乱学習で目に見えて改善しました。素の行動クローニングはオフライン評価（記録済みデータ上の誤差）では良く見えても、クローズドループ（実際に走らせて誤差が累積する評価）で破綻するという、模倣学習の根本問題を定量的に示した点が重要です。

Learning by Cheating は CARLA シミュレータのベンチマークで、route completion（指定ルートをどれだけ完走できたか、%）と成功率を評価しました。特権教師から蒸留した生徒は、画像から直接学んだベースラインを大きく上回り、当時の CARLA ベンチで最高水準の完走率を報告しています。教師の質を上げると生徒も上がるという関係（蒸留の上限が教師で決まる）を明確に示しました。

Trajectron++ は nuScenes の歩行者・車両軌道予測で、minADE / minFDE（複数候補を出したうち最良の平均変位誤差・最終変位誤差、メートル単位で小さいほど良い）を評価し、座標を直接回帰する手法に対し誤差を低減しました。アブレーションでは、車両の運動制約（dynamics integration、加速度・曲率を積分して位置を得る部分）を外すと、物理的にありえない急な軌道が出て誤差が悪化することを示し、制御量ベース表現の有効性を裏づけています。指標の意味を補足すると、ADE は予測全体の平均的なズレ、FDE は終点でのズレで、FDE は遠い未来ほど誤差が積もるので難しい指標です。

## メリット・トレードオフ・限界

走行ログから自動で大量の正解が得られ、人手アノテーションがほぼ不要（自己教師に近い）なのが最大のメリットです。加速度＋曲率という制御量表現は車両運動として自然で、座標直接回帰より安定し滑らかな軌道が出ます。`unwrap` とゼロ除算ガードで数値的にも頑健にでき、時間減衰の重みで安全に重要な近未来を優先できます。

一方、模倣学習の宿命として共変量シフトに弱く、学習データにない逸脱状態（路肩に寄りすぎた等）からの復帰を学べません。「人間がやったこと」が必ず最適とは限らず、人間の癖や誤り（急ブレーキ、車線のふらつき）も一緒に学んでしまいます。加速度・曲率は差分・微分で導くためノイズが乗りやすく、停車付近では曲率が0固定になって情報が落ちます。未来を2信号に絞ると、車線内の微妙な横位置調整など座標表現なら拾える情報を一部捨てます。研究上の未解決課題としては、共変量シフトをデータ収集側で埋めるか（DAgger 系の対話的データ収集）、損失や表現で頑健化するか、報酬を学ぶ逆強化学習へ進むか、といった方向の選択があります。

## 発展トピック・研究の最前線

模倣学習の弱点を補う系譜として、DAgger（Ross et al., AISTATS 2011）に始まる対話的データ収集（学習途中のモデルを走らせ、間違えた状態で専門家に正解を聞き直す）があります。共変量シフトを根本から減らす考え方です。報酬を明示せず専門家の振る舞いから報酬関数を逆算する逆強化学習（inverse reinforcement learning）や、近年は大規模な走行データで軌道予測を事前学習し、運動計画と統合する end-to-end 計画（UniAD、VAD など）が最前線です。軌道表現も、単一の最尤軌道ではなく複数の候補を確率付きで出すマルチモーダル予測（人は交差点で右にも左にも行きうる、という曖昧さを表現する）が主流になっています。研究を始めるなら、まず手元のログから加速度・曲率を抽出し、積分して軌跡を再構成して元の GPS 軌跡と重ねる検証をすると、表現の妥当性と数値処理の落とし穴が体感できます。
