# Hiera

## ひとことで言うと
Hiera(ハイエラ。hierarchical =「階層的な」の略)は、画像や動画を理解するためのニューラルネットワークの一種です。画像向けの Transformer(トランスフォーマー。注意機構という仕組みで、入力の各部分どうしの関係を学ぶネットワーク)に、これまで精度向上のために後付けされてきた「ごちゃごちゃした追加部品」を全部取り払い、その代わりに賢い自己教師あり事前学習(MAE)だけで性能を引き上げた、というのが核心の発想です。結果として、部品が少ないぶん速く、しかも精度が高いという良いとこ取りを実現しました。画像・動画のセグメンテーション(画像の中の物体を切り抜く処理)で有名な SAM 2 の頭脳(バックボーン)として採用されたことでも知られています。

## 直感的な理解
家を建てるときに、装飾(彫刻や飾り柱)をたくさん付ければ立派に見えますが、その重みを支えるために構造が複雑になり、建てるのも維持するのも大変になります。階層型 Transformer の世界では、まさにこの「装飾の付けすぎ」が起きていました。精度を上げるために特殊な部品を次々と足していった結果、モデルは複雑で遅くなっていったのです。

Hiera が問いかけたのは「その装飾は本当に構造として必要なのか、それとも基礎工事(事前学習)が弱いのを補うための松葉杖にすぎないのか」という点です。Hiera の主張は後者でした。基礎工事を強くすれば(=強力な事前学習を与えれば)、装飾は外しても倒れない。むしろ装飾を外したぶん軽くて速い家になる、という仮説を実証したのが Hiera です。これは深層学習でしばしば見られる「アーキテクチャの工夫」と「学習データ・学習法の工夫」のどちらが本質かという論争に対し、後者に軍配を上げた一例として読めます。

## 基礎: 前提となる概念
まず登場人物を整理します。

「バックボーン」とは、画像を入力するとその特徴(どこに何があるか、どんな模様か)を数値の塊(特徴マップ)に変換する土台のネットワークのことです。物体検出・領域分割・俯瞰図生成といった応用タスクは、すべてこのバックボーンの出力の上に乗ります。

「ViT(Vision Transformer)」は、画像を 16×16 ピクセル程度の小さなパッチ(切れ端)に分割し、それぞれを文章中の単語のように扱って、自然言語処理で成功した Transformer をそのまま画像に当てはめたモデルです(Dosovitskiy et al., ICLR 2021)。Transformer の中心にある「自己注意(self-attention)」とは、各パッチが他の全パッチをどれだけ参照すべきかを学習で決め、関連の強いところから情報を集める仕組みです。これにより画像全体の長距離の関係を捉えられますが、計算量はパッチ数 N の2乗(O(N²))に比例して増えるという弱点があります。224×224 を 14×14=196 パッチにすれば 196² ≈ 3.8 万通りのパッチ対の重みを計算します。解像度を2倍にしてパッチ数が4倍になれば、注意の計算は16倍に膨らみます。

「自己教師あり学習(self-supervised learning)」とは、人間が付けた正解ラベル(「これは犬」など)を使わず、データそのものから問題を自動生成して学ぶ枠組みです。後述する MAE はその代表例です。ラベル付けは高価なので、ラベルなしの大量データから学べることには大きな価値があります。

ここで、素の ViT には2つの構造的な問題がありました。

問題1は「解像度が一定のまま」という点です。素の ViT は最初の層から最後の層まで、ずっと同じ解像度(同じパッチ数)で処理します。たとえば 224×224 の画像を 14×14=196 個のパッチにすると、最初から最後まで 196 トークンのままです。一方、画像認識の世界では昔から「最初は細かく(高解像度)、だんだん粗く(低解像度)し、そのぶんチャンネル数を増やす」という階層構造(ピラミッド)が有効だと知られていました。CNN(畳み込みネット。ResNet など)はまさにこれを実装しています。素の ViT にはこの階層がなく、小さい物体と大きい物体を両方うまく扱うのが苦手で、計算も重くなりがちでした。

問題2は「階層化すると飾りが増える」という点です。問題1を解くために「階層型 ViT」が次々と提案されました。Swin Transformer(Liu et al., ICCV 2021)、MViT / MViTv2(Li et al., CVPR 2021/2022)などです。これらは確かに階層を持ちますが、その代償として、相対位置バイアス(パッチ同士の位置関係を別途学ぶパラメータ)、ウィンドウ分割とシフト、専用の pooling attention(注意の中で解像度を下げる仕掛け)といった職人技的な追加部品を山ほど付け足していました。著者たちはこれらを「Bells-and-Whistles(ベルと笛 = おまけの飾り立て)」と呼んで皮肉りました。これらの部品は、実装が複雑で、特殊な CUDA カーネルを要し推論が遅くなり、なぜ効くのか直感的に説明しづらいという欠点を抱えていました。

## 仕組みを詳しく
Hiera がやったことは、大きく分けて「引き算」と「足し算」の2つです。

引き算: MViTv2 をベースに、特殊な追加部品をほぼすべて削除しました。具体的には、相対位置エンコーディング(absolute positional embedding に置き換え)、畳み込みベースのモジュール(conv projection や conv による相対位置)、専用の attention 設計などを取り除き、ごく標準的な Transformer ブロックに近づけました。位置情報は学習可能な絶対位置埋め込みで与え、解像度の縮小は単純な空間プーリング(query を pool する maxpool 的な操作)で行います。これにより、特殊カーネルなしの素直な MatMul と pooling だけでモデルが構成され、ハードウェア上で速く動きます。

足し算: その代わり「MAE(Masked Autoencoder、マスク自己符号化器)」という自己教師あり事前学習を導入しました。MAE(He et al., CVPR 2022)とは、画像のパッチをランダムに大量(典型的には75%)に隠して(マスクして)、残った25%だけを見てネットワークに「隠れた部分の見た目(画素値)を復元させる」訓練法です。穴埋め問題を大量に解かせることで、ネットワークは物体の形・文脈・空間関係を、人間のラベルなしで自力で学びます。これは言語モデルの BERT が文中の単語を隠して当てさせるのと同じ発想を、画像に持ち込んだものです。

ここが Hiera の巧妙な点です。階層型 ViT は途中で解像度が下がるので、素朴には MAE と相性が悪い(どのトークンを隠したかの対応づけが面倒で、可視トークンだけを処理する MAE の高速化が効かない)のですが、Hiera は「mask unit(マスク単位)」という考え方を導入しました。1パッチではなく、複数 stage を通しても対応が崩れない少し大きめのまとまり(例: 8×8 トークン分の領域)を1単位として隠します。すると、解像度が変わっても「隠した領域」と「見える領域」を最後まで追跡でき、可視トークンだけをまとめて処理する MAE の効率(マスクした75%を計算から外せる)が階層型でも享受できます。さらに事前学習中は最初の2 stage で「Mask Unit Attention」という mask unit 内に閉じた局所注意を使い、マスクされたトークンを処理から外して計算を節約します(深い stage では受容野が広がるので global attention に切り替えます)。

階層構造の中身を具体的な tensor shape で見てみます。224×224 の RGB 画像を入れた場合(数値は代表的な例):

```
入力画像         : [B,   3, 224, 224]
stage1 (高解像度): [B,  96,  56,  56]   細かい特徴(エッジ・小物体)
stage2           : [B, 192,  28,  28]
stage3           : [B, 384,  14,  14]
stage4 (低解像度): [B, 768,   7,   7]   抽象的な特徴(物体カテゴリ)
```

stage が進むごとに、空間サイズ(縦横)は半分、チャンネル数(C)は倍、という典型的なピラミッドです。B はバッチサイズ(一度に処理する画像枚数)。この複数解像度の出力こそが、物体検出や俯瞰図生成で必要になる「マルチスケール特徴」です([features_only マルチスケール特徴抽出](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1106_features-only-multiscale))。

なぜこの設計が成り立つのか。鍵は「飾りは弱い事前学習の代償だった」という洞察です。相対位置バイアスのような部品は、限られた教師ありデータから空間的な手がかりを学ぶための補助でした。しかし MAE は大量のラベルなし画像で空間関係そのものを学べるため、補助部品が不要になります。つまり Hiera は、構造の複雑さを事前学習の強さに置き換えた、と言えます。

## 手法の系譜と主要論文
Hiera を理解するには、画像向け Transformer の系譜を辿るのが近道です。

ViT(Dosovitskiy et al., Google, ICLR 2021, arXiv:2010.11929)が出発点です。「画像をパッチに切って Transformer に入れるだけで、大規模データなら CNN を超える」と示し、画像認識を Transformer 時代に押し上げました。ただし単一解像度で、検出など密な予測タスクには向きませんでした。

Swin Transformer(Liu et al., Microsoft, ICCV 2021, arXiv:2103.14030)は、階層構造とウィンドウ注意(計算を局所窓に限定し、窓をずらして窓間を繋ぐ shifted window)を導入し、階層型 ViT の代表になりました。検出・分割で強い一方、ウィンドウシフトという「飾り」を持ちます。Hiera はこの系譜への明確なアンチテーゼとして読めます。

MViT / MViTv2(Li et al., Meta AI, CVPR 2021/2022, arXiv:2112.01526)は、pooling attention(query/key/value を pool して解像度を段階的に下げる)と相対位置バイアスを備えた階層型で、画像・動画の両方で高精度を出しました。Hiera はこの MViTv2 を「飾りを削る土台」として選びました。

MAE(He et al., Meta AI, CVPR 2022, arXiv:2111.06377)は、Hiera の足し算の核です。「画像の大半を隠して復元させる」自己教師あり学習を、可視パッチだけをエンコーダに通す非対称エンコーダ・デコーダ設計でシンプルかつスケーラブルに実現しました。

そして Hiera(Ryali, Hu, Bolya, Fan, Feichtenhofer et al., Meta AI, ICML 2023, arXiv:2306.00989)が「MViTv2 から飾りを削り、MAE で精度を取り戻す」を実証しました。著者陣には Token Merging(ToMe, Bolya et al., ICLR 2023, arXiv:2210.09461)の研究者も含まれ、「無駄を削いで速くする」という一貫した問題意識が見えます。最後に SAM 2(Ravi et al., Meta AI, 2024, arXiv:2408.00714)が、画像・動画セグメンテーションの基盤モデルとして Hiera を画像エンコーダに採用しました。動画でほぼリアルタイムの速度が要るため、Hiera の高速かつマルチスケールな特性が好適だったのです。これは Hiera が研究室の論文に留まらず、大規模実運用モデルの土台になった重要な実例です。

## 論文の実験結果(定量データ)
Hiera 論文は、画像と動画の両方で評価しています。指標と数値の意味を初心者にも分かるように説明します。

画像分類は ImageNet-1K(約128万枚、1000カテゴリ)で、指標は top-1 精度(モデルが最も自信を持って答えた1つのラベルが正解と一致した割合。100%に近いほど良い)です。報告によると、Hiera-B(ベースサイズ、約 5,100 万パラメータ)は MAE 事前学習後の ImageNet-1K 微調整で約 84.5% の top-1 を達成し、同規模の MViTv2-B(約 84.4%)や Swin-B を上回りつつ、推論が明確に速いと示されました。論文の主張の核は「同じ精度帯なら Hiera の方が画像あたりの推論時間が短く、FLOPs(浮動小数点演算回数。計算量の指標)も少ない」という点です。著者は、複雑な相対位置や conv を捨てたことで A100 GPU 上の実測スループットが MViTv2 比でおよそ 2 倍前後速くなる構成があると報告しています。FLOPs が同じでも、特殊カーネルがなくなることで実機の壁時計時間(wall-clock)はさらに改善する、という点が重要です。

動画認識は Kinetics-400(約24万本の動画、400種類の行動カテゴリ)で評価され、Hiera は MViTv2 を上回る top-1(報告で 80% 台後半)を、より少ない計算量で達成したとされます。動画は時間方向も含むため計算が重く、速度の優位はとくに効きます。Something-Something-v2(時間的な順序が重要な行動データセット)でも強い結果が報告されています。

アブレーション(どの要素を抜くとどうなるかの検証)が Hiera 論文の白眉です。著者は MViTv2 から飾りを1つずつ外していき、それぞれが精度に与える影響を測りました。重要な発見は「教師ありスクラッチ学習(MAE なし)では、相対位置バイアスや conv を外すと top-1 が数ポイント単位で明確に落ちるが、MAE 事前学習を併用すると、同じ飾りを外しても精度が落ちない、むしろ同等以上になる」という点です。これはまさに「飾りは弱い事前学習の代償だった」という仮説の直接的な裏付けで、相対位置バイアスや畳み込みモジュールを除去しても MAE があれば性能が維持されることを定量的に示しました。

下流タスクでも検証されており、物体検出・インスタンスセグメンテーション(COCO データセット、指標は AP = average precision。検出の正確さと網羅性をまとめた値)で、同規模の階層型 ViT に匹敵または上回る結果を、より高速に得られると報告されています。

## メリット・トレードオフ・限界
メリットは、飾り部品が少ないぶん実装がシンプルで推論が速いこと(同精度帯の Swin / MViTv2 比で実測スループット優位)、MAE 事前学習によりラベルなしデータで強い初期重みが得られること、階層構造を持つのでマルチスケール特徴(検出・俯瞰図生成に必須)を自然に出せること、そして SAM 2 という大規模実運用での実績があり信頼性が高いことです。

トレードオフと限界もはっきりしています。第一に、強力な MAE 事前学習が前提です。飾りを消したぶん事前学習に依存するため、素朴にスクラッチ学習(ランダム初期化から教師ありだけで学習)すると本来の性能が出にくくなります。第二に、その事前学習には大規模データと多大な計算リソースが必要で、前段コストが大きい(復元用のデコーダを別途用意し、長時間学習する)。少データのドメインでゼロから Hiera を使うのは不利で、公開された MAE 事前学習重みの転移に頼ることになります。第三に、ViT 系全般の弱点として、CNN が持つ「画像の局所性・並進不変性」という事前知識(帰納バイアス)が弱く、データが少ない領域では CNN より不利になりやすい点が挙げられます。第四に、絶対位置埋め込みを採用するため、事前学習時と異なる入力解像度への適応(位置埋め込みの補間)に注意が要ります。

研究上の未解決課題としては、MAE 以外の自己教師あり手法(対照学習や蒸留ベース)と組み合わせたときの最適設計、動画におけるさらなる時間方向の効率化、そして帰納バイアスをどこまで削ってよいかという境界の理論的理解などが残っています。

## 発展トピック・研究の最前線
Hiera の思想「構造の複雑さを事前学習の強さに置き換える」は、その後の基盤モデル設計に影響を与えています。SAM 2 の成功は、シンプルで高速なバックボーンが動画基盤モデルの実用性を左右することを示しました。さらに進んだ方向としては、Hiera のような階層型エンコーダを画像・動画・点群といった複数モダリティへ拡張する研究、MAE と言語監督(画像とテキストの対応学習、CLIP 的な事前学習)を組み合わせて意味理解を強化する流れ、そして推論時の解像度を動的に変えて計算を節約するトークン削減(token merging / pruning)との統合などがあります。Hiera の著者の一部は token merging(ToMe)の研究者でもあり、シンプルさと速度を追求する一貫した問題意識が見て取れます。研究の現場では「新しい層を足して精度を上げる」より「事前学習を強くして層を削る」方が、推論コストの観点で実運用に効くという認識が広がっており、Hiera はその象徴的な成功例として引用され続けています。
