# DDIM (Denoising Diffusion Implicit Models)

## ひとことで言うと
DDIM は [DDPM (Denoising Diffusion Probabilistic Models)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0229_ddpm_note) の「生成が遅い」という弱点を直す手法です。DDPM が 1 つのサンプルを作るのに数百〜千回もネットワークを通す必要があったところを、DDIM は同じ学習済みネットワークをそのまま使いながら、20〜50 回程度で同等品質のサンプルを作れるようにします。しかも、毎回同じ乱数から始めれば必ず同じ結果が出る「決定論的(ランダムさのない、計算が一意に決まる)」な生成もできます。

## 直感的な理解
階段を 1 段ずつ降りるのと、エレベーターで何階か一気に降りるのを想像してください。DDPM は「1 段ずつしか降りられない階段」でした。途中の段を飛ばすと足を踏み外す(品質が崩れる)構造だったのです。DDIM は同じ建物の同じ高さを、数段ずつ飛ばして安全に降りられるようにした、いわばエレベーターです。

ポイントは、DDIM は新しい建物を建て直した(モデルを再学習した)わけではないことです。DDPM で学習済みのネットワークをそのまま流用し、「降り方(サンプリング手順)」だけを賢くしています。だから追加の学習コストなしで、ただ生成を 20〜50 倍速くできます。実用、特に車載のように計算資源が限られ低遅延が必須の環境では、この高速化が拡散モデルを使えるかどうかの分かれ目になります。

もう一つの直感は「DDPM は毎ステップでサイコロを振っていたが、DDIM は基本的にサイコロを振らない」ことです。サイコロ(乱数)を振ると、振る回数を減らした(ステップを飛ばした)ときに足すべきランダム量の見積もりがずれて品質が崩れます。乱数を振らなければ、各ステップは決まった計算になり、大きく飛ばしても破綻しにくいのです。

## 基礎: 前提となる概念
DDPM の前提を簡単に復習します。DDPM の前向き過程は、きれいなデータ x_0 に少しずつガウスノイズを足して x_t を作る過程で、「途中を飛ばして x_t を直接書ける」便利な式を持っていました。

```
x_t = sqrt(α_bar_t) · x_0 + sqrt(1 - α_bar_t) · ε
```

ここで α_bar_t は「t ステップ後にどれだけ元データが残っているか」を表す係数(1 から 0 へ減る)、ε はガウスノイズです。学習では、ネットワーク ε_θ(x_t, t) が「x_t に混ざっているノイズ ε」を予測するよう訓練されます。

マルコフ過程とは「次の状態が今の状態だけで決まる」連鎖でした。DDPM の前向き過程はマルコフで、1 ステップずつしか進めない構造です。これが遅さの根本原因でした。途中を飛ばせないのです。

周辺分布(marginal distribution)とは、各時刻 t におけるデータの散らばり方(分布)のことです。上の x_t の式は、x_0 を固定したときの x_t の分布、すなわち各 t での周辺分布を定めています。DDIM の鍵は「各時刻の周辺分布を DDPM と同じに保ったまま、ステップの繋ぎ方だけを変える」点にあります。周辺分布が同じなら、DDPM で学習したネットワーク ε_θ がそのまま使えるからです。学習で最適化される目的関数は周辺分布だけに依存し、ステップ同士をどう繋ぐか(同時分布の形)には依存しない、という観察がこの手法を可能にしています。

常微分方程式(ODE)とは、ある量の時間変化率を記述する方程式 dx/dt = f(x, t) です。初期値が決まれば解が一意に決まる(決定論的)点が、後で DDIM の見方として効いてきます。

## 仕組みを詳しく
### 非マルコフ過程の導入
Song らが立てた問いはこうです。「DDPM の学習で得たネットワークは、結局 x_t からノイズ ε を予測するものだ。その同じネットワークを再学習せずに使い回しつつ、もっと速いサンプリング手順を作れないか」。

答えは「前向き過程を非マルコフ過程として組み直す」ことでした。難しく聞こえますが、要点は「各時刻の周辺分布(上の x_t の式)を DDPM と完全に同じに保つ」よう設計するという点です。Song らは、x_t と x_0 の両方に条件付けた前向き分布 q(x_{t-1} | x_t, x_0) の族をパラメータ σ_t で定義し、どの σ_t を選んでも周辺分布が DDPM と一致するように作りました。周辺分布が同じなら ε_θ(x_t, t) はそのまま使える。違うのは「逆向きにどうたどるか」という道筋だけです。

### 決定論的サンプリング
DDIM の生成 1 ステップは次のように書けます。まず x_t とネットの予測 ε_θ(x_t, t) から、「元のきれいなデータ x_0 はたぶんこれだろう」という推定値を作ります。

```
予測 x_0 = ( x_t - sqrt(1 - α_bar_t) · ε_θ(x_t, t) ) / sqrt(α_bar_t)
```

次に、この予測 x_0 を使って次の時刻 x_{t-1}(あるいはもっと先の x_{t-k})を再構成します(σ_t=0 の場合)。

```
x_{t-1} = sqrt(α_bar_{t-1}) · (予測 x_0) + sqrt(1 - α_bar_{t-1}) · ε_θ(x_t, t)
```

ここでランダムなノイズを足さないのがポイントです。DDPM は各ステップで新たに乱数を足していました(だから確率的)。DDIM のこの式には乱数項がなく、x_t が決まれば x_{t-1} が一意に決まります。これが決定論的という意味です。式の構造を読むと「いったん x_0 を予測して、その x_0 へ向かって新しい時刻のノイズ量だけ足し直す」ことを繰り返しており、毎ステップで目標(x_0 の推定)を更新しながらそこへ近づいていく、と理解できます。

### なぜ速くできるのか
決定論にしたことで、各ステップが「予測 x_0 を経由して、いくつ先の時刻にでもジャンプできる」形になります。上の式の t-1 を、もっと小さい時刻(たとえば t-20)に置き換えても同じ形で書けるからです。たとえば t = 1000, 980, 960, … と 20 ずつ飛ばして 50 ステップだけたどっても、間を律儀に 1 ずつ進んだのに近い結果が得られます。DDPM の確率的なステップでは、飛ばすと毎回足す乱数の量がずれて品質が崩れますが、DDIM は乱数を足さないので飛ばしに強いのです。

具体的な数字: 形状 [3, 256, 256] の画像を作るとき、DDPM なら 1000 回ネットを通す必要がありますが、DDIM なら 50 回でほぼ同等、20 回でも実用的な品質が得られます。単純計算で 20〜50 倍の高速化です。

### 確率と決定論のあいだ(η パラメータ)
DDIM には η(イータ)という調整つまみがあります。これは前述の σ_t をスケールするパラメータで、η=0 で完全に決定論的(上の式)、η=1 にすると DDPM とまったく同じ確率的サンプリングになります。つまり DDIM は DDPM を特殊ケースとして含む、より一般的な枠組みです。中間の η で、決定論と多様性のバランスを連続的に選べます。

### 決定論であることの嬉しさ
決定論だと、潜在変数(latent、生成の出発点となるノイズ)と生成結果が 1 対 1 で対応します。これにより次のことができます。
- 補間(interpolation): 2 つの出発ノイズを球面線形補間(slerp)で混ぜると、中間的な生成結果が滑らかに得られる。
- 逆変換(inversion): 生成過程を逆向きにたどり、既存データに対応するノイズを求められる。これは画像編集(属性を変えて再生成する)などに使えます。
- 再現性: 同じ種から必ず同じ結果が出るので、デバッグや評価が容易になります。

### ODE としての見方
DDIM のサンプリングは、後に「拡散の生成過程を表す常微分方程式(probability flow ODE)をオイラー法(最も素朴な数値積分)で解く手続き」と解釈できることがわかりました。決定論的にしたことで、サンプリングが ODE の数値積分とみなせるようになったのです。この見方は重要で、「もっと精度の高い数値積分法(高次ソルバー)を使えば、同じステップ数でもっと正確に解ける=高品質になる」という発展(後述の EDM、DPM-Solver)に直結しました。

## 手法の系譜と主要論文
- Song, Meng, Ermon、「Denoising Diffusion Implicit Models」(ICLR 2021、arXiv:2010.02502)。非マルコフな前向き過程の族を定義し、その逆過程として DDPM と同じネットワークを使い回せる決定論的サンプラーを導出しました。動機は「DDPM の遅さの原因はマルコフ性であり、周辺分布を保ったままマルコフ性を捨てればステップを飛ばせる」という洞察です。

- Song, Sohl-Dickstein, Kingma, Kumar, Ermon, Poole、「Score-Based Generative Modeling through SDEs」(ICLR 2021、arXiv:2011.13456)。拡散を連続時間の確率微分方程式(SDE)として定式化し、それと同じ周辺分布を持つ等価な決定論的 probability flow ODE が存在することを示しました。DDIM はこの ODE の離散化の一種として理論的に位置づけられます([Score-based Generative Models (SDE定式化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0250_score-based-sde))。

- Karras, Aittala, Aila, Laine、「Elucidating the Design Space of Diffusion-Based Generative Models」(NeurIPS 2022、arXiv:2206.00364、通称 EDM)。DDIM を含む各種サンプラーを「ODE を数値的に解く手続き」として統一的に整理し、ノイズスケジュール・前処理・サンプラーを設計変数として明示しました。DDIM はオイラー法に相当すると位置づけ、Heun 法など 2 次の解法を使えば同じステップ数でより高品質になることを示しました。

- Lu, Zhou, Bao, Chen, Li, Zhu、「DPM-Solver: A Fast ODE Solver for Diffusion Probabilistic Model Sampling in Around 10 Steps」(NeurIPS 2022、arXiv:2206.00927)。拡散 ODE の半線形構造を解析的に利用した高次 ODE ソルバーで、10 ステップ前後でも高品質を達成しました。DDIM の高速化路線をさらに推し進めたもので、改良版 DPM-Solver++ とともに現在の実用システム(Stable Diffusion の既定サンプラーなど)で広く使われています。

## 論文の実験結果(定量データ)
指標は FID(Frechet Inception Distance、生成分布と本物分布の近さ。小さいほど高品質)を中心に見ます。NFE(Number of Function Evaluations)は「ネットワークを何回通したか」で、生成コストの目安です。

- CIFAR-10(32×32 の小画像)。DDIM 論文では、同じ学習済みモデルに対し、DDPM が 1000 NFE で達成する FID(約 3〜4 台)を、DDIM は約 50〜100 NFE で達成できることを示しました。さらに NFE を 20 まで減らしても実用的な FID(おおむね 6〜7 台)にとどまり、10 程度の極端な少ステップで初めて目立った劣化が出る、という傾向を報告しています。これは「ステップを飛ばしても品質が急には落ちない」という決定論サンプラーの強みの定量的な裏付けです。

- 高解像度(CelebA、LSUN など)。同様に、数十ステップで数百〜千ステップの DDPM とほぼ同等の見た目品質が得られることを示し、実用上の生成時間を 1 桁以上短縮できることを実証しました。

- 確率版との比較(η のアブレーション)。η=1(DDPM 相当)と η=0(決定論)を比べると、ステップ数が多いとき(100 以上)は両者の FID は近いものの、ステップ数を減らす(20〜50)と η=0(決定論)のほうが FID の劣化が小さいことが示されました。つまり「少ステップで使うなら決定論が有利」で、これが DDIM の少ステップ高速化の核心です。一方で多様性(生成結果のばらつき)は確率版のほうがやや高い場合があり、用途に応じて η を選ぶ余地があります。

- EDM・DPM-Solver の進展。EDM は、DDIM(オイラー法)を Heun 法など 2 次ソルバーに替えることで、同じ NFE で FID をさらに下げられることを示し、CIFAR-10 で当時最良級の FID 約 1.8〜1.9(クラス条件付き)を少 NFE で報告しました。DPM-Solver は 10〜20 NFE 程度で高品質生成を達成し、DDIM の 50 NFE 水準をさらに半減できることを報告しています。これらは「同じ学習済みモデルでも、解き方を工夫すれば生成はもっと速くできる」という流れを定量的に裏付けています。

## メリット・トレードオフ・限界
メリット
- 高速。DDPM の 20〜100 倍速く生成できる(50 ステップで同等品質)。
- 再学習不要。DDPM で学習済みのネットワークをそのまま使える。
- 決定論的生成ができ、補間や逆変換(編集)がしやすい。再現性も高い。
- η を変えるだけで DDPM と DDIM を連続的に行き来できる(柔軟)。

トレードオフと限界
- ステップ数を極端に減らす(数ステップ)と品質が劣化する。1〜2 ステップは厳しい(オイラー法の離散化誤差が大きくなる)。
- 決定論ゆえに、毎回まったく同じ入力からは同じ出力しか出ず、多様性が確率版よりやや下がることがある。
- 高品質を保つには適切なステップ数とノイズスケジュール、ソルバーの選択が必要。設定への感度が残る。
- 未解決・課題として、数ステップ(1〜4)での高品質生成は依然難しく、これを解くために蒸留(Consistency Models など)や、より良い ODE ソルバーの研究が続いています。

## 発展トピック・研究の最前線
- 高次 ODE ソルバー。EDM の Heun 法、DPM-Solver / DPM-Solver++、UniPC などが、同じステップ数でより高精度に積分し、少ステップ品質を底上げしています。
- 蒸留と一貫性モデル。多ステップ拡散を 1〜数ステップに圧縮する Progressive Distillation や Consistency Models は、DDIM の決定論的軌跡を教師信号として使うことが多く、DDIM と密接に関連します。一貫性モデルは「DDIM 軌跡上のどの点からも一気に x_0 へ飛ぶ」ことを学ぶと理解できます。
- フローマッチングとの接続。直線パスを学ぶフローマッチング([Optimal Transport 確率パス (線形OTパス)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0220_optimal-transport-path))も、推論時は ODE をオイラー法で解く点で DDIM と同じ枠組みにあり、少ステップ生成という目標を共有しています。
- 行動生成での実用。ロボットや自動運転で拡散ポリシーを使う場合、車載コンピュータで実時間に間に合わせるため、DDIM や DPM-Solver による少ステップ推論がほぼ必須の中核技術になっています。
