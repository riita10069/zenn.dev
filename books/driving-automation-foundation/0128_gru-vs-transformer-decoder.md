# 軌跡デコーダにGRUを選ぶ理由

## ひとことで言うと

未来の軌跡を1点ずつ順番に作るとき、Transformer ではなく GRU(ゲート付き回帰ユニット)を使うことがあります。理由は、軌跡が「前の点が決まらないと次の点が決まらない」という厳密な逐次依存だからです。この用途では、計算量が系列長に比例する GRU(O(T))の方が、系列長の2乗かかる Transformer の self-attention(O(T²))より素直で軽い、という設計トレードオフの話です。

## 直感的な理解

文章生成や軌跡生成のように「系列を順番に作る」タスクでは、各要素が直前までの要素に依存します。この依存をどんな部品で表現するかが設計の分かれ道です。

候補は大きく2つあります。ひとつは回帰型ニューラルネット(RNN)系の GRU で、これは時系列を1ステップずつ処理する「記憶つきの箱」です。前のステップの内部メモリを受け取り次のメモリを出すので、本質的に順番に進みます。もうひとつは Transformer デコーダで、全ステップ同士をアテンション(どのステップがどのステップに注目するか)で結びつけます。学習時は全ステップを並列に処理でき、遠い位置どうしの依存も捉えやすいのが強みです。

直感的なたとえ: GRU は伝言ゲームです。前の人から聞いた話(隠れ状態)に自分の情報を足して次の人に渡す。順番にしか進めません。Transformer は会議室で全員が全員の発言を一度に見渡す方式です。並列に処理できますが、参加者が増えると「全員×全員」の組み合わせ計算が増えます。

軌跡生成のように「左から右への一方向の連鎖」で、しかも遠い時刻間の複雑な相互作用がそれほど要らない場合、わざわざ会議室方式(Transformer)を使い、未来をカンニングしないよう覆い(causal mask)を貼って逐次に縛るより、最初から伝言ゲーム(GRU)で書く方が無駄がない、という判断が成り立ちます。

## 基礎: 前提となる概念

**RNN(Recurrent Neural Network、回帰型ニューラルネット)**: 系列を1要素ずつ処理し、内部状態(隠れ状態)を次の要素に持ち越すネットワークです。「メモリを持つネット」と考えてよいです。素朴な RNN は長い系列で勾配消失(誤差を逆伝播するうちに勾配がほぼ0になり遠い過去を学べない)という弱点があります。

**ゲート機構(gating)**: 古い記憶をどれだけ残し、新しい情報をどれだけ取り込むかを0〜1の値で調整する弁です。これにより長い系列でも記憶が消えすぎず暴れすぎず安定します。GRU は更新ゲート(update gate)とリセットゲート(reset gate)の2つを持ちます。LSTM はゲートが3つでより複雑です。

**計算量のオーダー記法(O 記法)**: 入力サイズに対し計算量がどう増えるかの表記です。O(T) は系列長 T に比例、O(T²) は T の2乗に比例。T=64 なら T²=4096 で、T が増えるほど両者の差は開きます。

**causal mask(因果マスク)**: Transformer デコーダで各位置が未来の位置を見ないようにする三角形の覆いです。系列を順番に生成する用途では未来をカンニングさせないために必須で、実装と計算が増える要因になります。

**帰納バイアス(inductive bias)**: モデルが持つ「こういう構造のデータだろう」という前提です。RNN は「逐次依存」という帰納バイアスを最初から持つので、逐次的なデータに少ないデータでも適合しやすい傾向があります。

## 仕組みを詳しく

GRU の1ステップは次の形をしています(入力256次元・隠れ状態256次元の例)。

```
入力:  x_t       [B, 256]   ← その時刻に外部から見てきた特徴の要約
状態:  h_{t-1}   [1, B, 256] ← 前ステップの記憶
ゲート計算:
  更新ゲート z_t   = σ(W_z x_t + U_z h_{t-1})        どれだけ更新するか
  リセットゲート r_t = σ(W_r x_t + U_r h_{t-1})        過去をどれだけ忘れるか
  候補状態 ñ_t     = tanh(W_n x_t + U_n (r_t ⊙ h_{t-1}))  新しく取り込む内容
  新状態 h_t      = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ ñ_t
出力:  h_t       [1, B, 256] ← 新しい記憶
```

記号の意味: σ はシグモイド(0〜1に潰す関数)、tanh は -1〜1に潰す関数、⊙ は要素ごとの掛け算、W・U は学習する重み行列です。更新ゲート z_t が1に近ければ新しい候補で大きく書き換え、0に近ければ古い記憶を維持します。リセットゲート r_t が0に近ければ過去を忘れて入力に集中します。この2つの弁で、長い系列でも記憶を適切に保てます。

軌跡デコーダでの使われ方(疑似コード):

```
h = h_0                                   # 初期記憶(過去の自車状態 + 視覚履歴から)
for t in range(64):
    query    = h.squeeze(0) + ego_query   # [B, 256]
    attended = deformable_cross_attn(query, values)  # [B, 256] 周囲を見てきた要約
    _, h     = gru(attended.unsqueeze(0), h)         # 記憶を更新
    waypoints.append(waypoint_head(h.squeeze(0)))    # [B, 2] その時刻の操作
trajectory = cat(waypoints)               # [B, 128]
ego_hidden = h.squeeze(0)                 # [B, 256] 最終記憶
```

ステップ t の `query` は `h_{t-1}` を含み、`h_t` は `attended`(その query から決めた注目先)に依存します。つまり「前が決まらないと次が決まらない」。この依存を GRU は素直に表します。

**before/after の比較**:

```
before (Transformer decoder を使ったら):
  - 64×64 のアテンション行列が必要
  - 未来をカンニングしないための causal mask が必要
  - 位置エンコーディング(各ステップの順番を教える仕掛け)が必要
  - 学習時は並列で速いが、推論時に1点ずつ生成するなら結局逐次
  - パラメータ多め

after (GRU を使う):
  - T=64 回のループだけ
  - causal mask 不要(構造的に未来を見られない)
  - 位置の順番はループ自体が表現
  - パラメータ少、実装が単純
```

「Transformer が常に最適」ではなく、O(T) で足り、依存が厳密に逐次なら軽い GRU が適切、というのが要点です。逆に、遠い時刻間の複雑な相互作用や、学習時の大幅な並列化が重要な場面では Transformer が勝ります。

## 手法の系譜と主要論文

- **GRU の提案**(Cho, van Merriënboer, Bahdanau, Bengio, EMNLP 2014, arXiv:1406.1078): 機械翻訳のエンコーダ・デコーダ枠組みの中で GRU を提案しました。LSTM より少ないゲート(3→2)で長期依存を扱い、計算とパラメータを減らすのが動機です。LSTM に匹敵する性能でより軽量、というのが新規性です。

- **GRU と LSTM の比較**(Chung, Gulcehre, Cho, Bengio, 2014, arXiv:1412.3555): ゲート付き RNN を系統的に比較し、GRU が LSTM と同等の性能を多くのタスクで示しつつ計算が軽いことを実証しました。「軽さと性能のバランスで GRU を選ぶ」根拠を与えた研究です。

- **Transformer の提案**(Vaswani et al., NeurIPS 2017, arXiv:1706.03762): RNN の逐次処理が学習の並列化を妨げ長距離依存も捉えにくかった問題を、アテンションだけで解きました。並列学習が可能で長距離依存に強く多分野を塗り替えた一方、計算とメモリが系列長の2乗 O(T²) に増えるのがトレードオフです。短く厳密に逐次な系列ではこの強みが活きにくくなります。

- **ChauffeurNet**(Bansal et al., RSS 2019, arXiv:1812.03079): 自動運転の軌跡生成で RNN(回帰型)を使い経路点を1点ずつ出す設計を採りました。軌跡は逐次依存が自然で RNN の帰納バイアスと相性が良いという判断です。一方で UniAD(Hu et al., CVPR 2023)などは Transformer デコーダを用います。どちらが良いかは系列長・並列化要求・マルチモーダル性で決まる、というのが本ノートの主題です。

系譜としては「RNN/LSTM/GRU で逐次生成(2014頃〜ChauffeurNet) → アテンションで並列化と長距離依存(Transformer, 2017) → 用途に応じて使い分け(現在)」と進み、軌跡計画では今も「逐次で短い系列なら GRU、長く複雑なら Transformer」という棲み分けが残っています。

## 論文の実験結果(定量データ)

GRU と Transformer の比較は分野ごとに様々ですが、本トピックに関わる定量的知見を挙げます。

- **GRU vs LSTM**(Chung et al. 2014): 多声音楽モデリングや音声信号モデリングのタスクで、GRU は LSTM と同等以下のパープレキシティ(言語/系列モデルの予測のうまさ、低いほど良い)を、より少ない計算で達成しました。「ゲートは要るが、ゲート数は2で足りることが多い」という実用的結論です。

- **Transformer vs RNN**(Vaswani et al. 2017): 機械翻訳 WMT2014 英→独で BLEU(翻訳の一致度、高いほど良い)28.4 を当時の最高水準で達成し、しかも学習が大幅に速い(並列化のため)ことを示しました。ただしこれは「長距離依存が重要で、入力系列が長く、学習の並列化が効く」翻訳タスクでの優位であり、64点程度の短く逐次的な軌跡には直接当てはまりません。

- **軌跡計画での実測傾向**: 軌跡デコーダを GRU から Transformer に置き換えても、開ループの L2 誤差(予測と正解の位置ずれ、メートル)はほとんど変わらないか、データ量が少ないと GRU の方が安定する、という報告が複数あります。理由は (1) 系列長 T=64 程度では O(T²) の不利が小さく、(2) 軌跡の依存が主に直前の状態に集中しており遠距離依存が薄いため、Transformer の長距離アテンションの強みが活きにくいことです。一方、推論レイテンシは「逐次生成」である限りどちらも T 回のステップが律速になり、GRU の方がステップあたりの計算が軽い分わずかに速いことが多いです。

指標の意味: L2 誤差の差が 0.05m 以内なら統計的に有意でないことも多く、その場合は「同等の精度なら軽く単純な方を選ぶ」というオッカムの剃刀的判断が働きます。GRU 採用はこの判断の典型例です。

## メリット・トレードオフ・限界

メリット:
- 計算量が O(T) で軽い(T² のアテンション計算が不要)。
- causal mask が不要で実装が単純。バグの入る余地が少ない。
- 逐次依存という軌跡の性質に帰納バイアスが合い、少ないデータでも安定しやすい。
- パラメータが少なく学習が安定しやすい。

トレードオフ・限界:
- 逐次処理なので時間方向を並列化できず、推論が遅い(T 回のループが律速)。ただしこれは自己回帰生成である限り Transformer でも同じ。
- 非常に長い系列や遠い時刻間の複雑な相互作用は Transformer より捉えにくいことがある。
- 1本の隠れ状態に情報を圧縮するため、複数の未来シナリオを同時に保持するのは苦手(マルチモーダル予測には不向き)。
- 隠れ状態がボトルネックになり、初期の入力情報が長いループの末尾で薄れる可能性がある。

未解決・発展の論点: 逐次性そのものを捨てて全軌跡を一括並列生成する非自己回帰デコード([非自己回帰な軌跡デコーディング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0152_non-autoregressive-trajectory-decoding))や、複数モードを保持できる構造(混合密度・拡散モデル)への移行が、GRU の限界を超える方向として検討されています。GRU 採用は「現時点の最小実装として素直」という位置づけで、最終形ではありません。

## 発展トピック・研究の最前線

- **State Space Model(SSM)系**: Mamba などの選択的状態空間モデルは、RNN の O(T) の効率と Transformer 並みの長距離依存能力を両立しようとする新潮流で、長い系列の逐次生成で GRU/Transformer 双方の弱点を補う候補です。
- **非自己回帰生成**: 軌跡全体を一度に出すことで逐次性のレイテンシを消し、拡散やフローマッチングでマルチモーダルも扱う方向。
- **混合密度・アンカー軌跡**: 1本の隠れ状態の限界を超え、複数の候補軌跡(モード)を明示的に保持する設計(MultiPath, TNT, DenseTNT など)。
