# 鳥瞰図の解像度と知覚範囲（前方バイアスの非対称pc_range）

## ひとことで言うと

鳥瞰図(上から見下ろした特徴マップ)を「どれだけ細かく(解像度)」「車の何メートル先まで(知覚範囲)」表すかを決める設計の話です。解像度は1セルが実空間で何メートルかを決め、粗いと物体を区別できません。知覚範囲は前後左右どこまで見るかで、運転の用途に応じて前を遠く・後ろや横を近くする「非対称(asymmetric)」な設定が有効になることがあります。解像度と範囲はVRAM(GPUメモリ)を介して密接に結合しており、限られた計算資源をどこに配分するかという最適化問題になります。

## 直感的な理解

地図アプリを思い浮かべてください。ズームアウトしすぎると、隣り合った2台の車が1つの点に潰れて区別できません。鳥瞰図でも同じで、グリッドが粗いと「車1台が1セルにすっぽり収まる」ような事態になり、近接する物体の位置関係がまったく分からなくなります。これでは精密な運転計画はできません。前の車との車間距離も、隣車線の車の正確な位置も読めなくなります。

一方で、細かくすればするほどメモリと計算を食います。だから「必要十分な細かさ」を選ぶ必要があります。やみくもに細かくすればよいわけではなく、物体を区別できる分解能があれば十分、というバランス点を探します。

もう1つの観点が「どこまで見るか」です。高速道路では前方の遠くにある減速車や落下物を早く見つけられるほど安全に止まれます。停止距離は速度の2乗で伸びるので、速いほど遠くを見る価値が高いのです。逆に後方や横は車線変更で使いますが、前方ほど遠距離は要りません。ならば限られた計算資源を前方の遠距離に厚く配分する、という非対称な設計が理にかないます。全方位を均等に遠く・細かく見るのは資源の無駄遣いになりかねません。

## 基礎: 前提となる概念

- 解像度(bev_h × bev_w): 鳥瞰図グリッドの縦横のセル数です。例えば 450×300 なら13.5万セルです。
- セルサイズ(分解能): 1セルが実空間で表す距離です。`知覚範囲の距離 / セル数` で決まります。小さいほど細かいです。
- pc_range(point cloud range, 知覚範囲): 鳥瞰図がカバーする空間を `(x_min, y_min, z_min, x_max, y_max, z_max)` のメートルで表す6要素です。LiDARの点群範囲を表す慣習的な名前ですが、カメラのみの鳥瞰図でも同じ表記を使います。
- ego座標系: 自車中心の座標です。慣例としてX=前方、Y=左、Z=上(右手系)です。論文により軸の取り方は異なります。
- 停止距離: ブレーキで止まるまでに進む距離です。空走距離(反応時間中に進む距離) + 制動距離からなり、制動距離は速度の2乗に比例します。
- VRAM(GPUメモリ): 学習・推論で中間テンソル(計算の途中結果)を置く容量です。解像度を上げると鳥瞰図の中間テンソルが大きくなり消費が増えます。
- 中間活性化(activation): 各層の出力で、逆伝播のために保持しておくテンソルです。これが解像度とともに膨らみ、実効メモリ消費を押し上げます。
- OOM(Out Of Memory): GPUメモリが足りずに学習・推論が落ちることです。

## 仕組みを詳しく

### pc_range の読み方

代表的な非対称設定の例:

```
pc_range = (-60.0, -60.0, -5.0, 120.0, 60.0, 3.0)
            x_min  y_min  z_min  x_max  y_max  z_max
```

X=前方、Y=左、Z=上として読み解くと:

- X方向(前後): -60m から +120m。前方を120m、後方を60m見ます。前を後ろの2倍遠くまで見る非対称です。
- Y方向(左右): -60m から +60m。左右それぞれ60m、対称です。
- Z方向(高さ): -5m から +3m。路面の少し下から車高の上まで。地面の起伏や高い車両を含む範囲です。

これが「前方バイアスの非対称(asymmetric)」の意味です。

### 解像度とセルサイズの検算

bev_h=450, bev_w=300 のとき、セルサイズを検算します。

```
X方向(前後): 120 - (-60) = 180m,  180 / 450 = 0.4 m/セル
Y方向(左右):  60 - (-60) = 120m,  120 / 300 = 0.4 m/セル
```

きれいに 0.4m/セルになります。0.4m/セルなら、全長4.5mの乗用車は約11セルにまたがり(4.5/0.4≈11)、車線(幅3.5m≈9セル)もはっきり区別できます。物体が複数セルにまたがることで、その向きや大きさも解像できます。

粗い設定との対比を数値で見ます。仮に知覚範囲50mを 7×7 で表すと、`50/7 ≈ 7.1 m/セル`。全長4.5mの車が1セルに収まってしまい、隣の車との位置関係を一切区別できません。7×7のような粗さは、設計値というより「バックボーン(画像特徴抽出ネット)が256×256入力を5回ダウンサンプリング(2分の1に縮小)した出力をそのまま流用した」結果として偶発的に生じることがあり、計画には粗すぎます。鳥瞰図の解像度はバックボーン出力とは切り離して明示的に決めるのが定石です。

### なぜ前方バイアスか(物理的根拠)

停止距離のうち制動距離は速度の2乗に比例して伸びます。乾いた路面で減速度を約7 m/s²とすると、制動距離は概ね次のようになります。

```
時速 50km (約14m/s):  制動距離 ≈ 14²/(2×7) ≈ 14m
時速100km (約28m/s):  制動距離 ≈ 28²/(2×7) ≈ 56m
時速120km (約33m/s):  制動距離 ≈ 33²/(2×7) ≈ 78m  (空走+反応含めると100m超)
```

時速120kmでは100m前後を見る必要が出てきます。前方120mという設定はこの要請に応えるものです。後方・側方は車線変更や合流で使いますが、相対的に近距離で足り、60mに抑えて計算資源を前方に回す判断です。後方からの追い越し車も問題ですが、自車より速い相対速度の上限が前方の障害物ほど厳しくないため、近距離でも実用上足りるという考え方です。

### 解像度とコストのトレードオフ

解像度を上げるほどセルが細かくなり精度は上がりますが、VRAMと計算量が増えます。embed_dim=256, fp16(16ビット浮動小数)で知覚範囲50mを変える例(おおよその目安):

| 解像度 | セルサイズ(50m範囲) | 鳥瞰図テンソルのVRAM | 計算量 |
|---|---|---|---|
| 50×50   | 1.0m  | 約3MB  | 低 |
| 100×100 | 0.5m  | 約13MB | 中 |
| 200×200 | 0.25m | 約50MB | 高 |

セル数は解像度の2乗で増えるので、VRAMと計算も2乗で増えます。100×100は50×50の約4倍、200×200は約16倍です。さらに鳥瞰図はネットワークの中間で多数の attention 計算を経るため、グリッドが大きいほど中間活性化が積み上がり、実効的なメモリ消費はテンソル単体の数倍に達します。表の数値はあくまで鳥瞰図テンソル1枚の見積もりで、実学習ではこれに勾配・オプティマイザ状態・各層の活性化が乗ります。

実測の感覚として、L40Sクラスのおよそ45GBのGPUで、450×300・バッチ4・bf16(16ビット浮動小数の別形式)の学習はpeak VRAMが約20GB、補助タスク(将来状態予測など)を含めると約23GBに達し、バッチ8ではOOMになる、という報告があります。高解像度の代償が直接VRAMに出ます。だからこそgradient checkpointing(活性化を保持せず逆伝播時に再計算してメモリを節約する技法)や混合精度が実務で重要になります。

## 手法の系譜と主要論文

各モデルが採用した鳥瞰図の解像度・範囲を並べると、設計思想の違いが見えます。

- BEVFormer(Li et al., ECCV 2022) / UniAD(Hu et al., CVPR 2023): 200×200、約51.2m四方、0.256m/セル。前後対称。動機はnuScenesベンチマークの標準評価範囲(中心から約51.2m)に合わせることでした。効果は他手法と直接比較しやすいことです。トレードオフは前後を区別しない対称設計で、計画より検出寄りの範囲設定である点です。
- BEVDet(Huang et al., 2021) / BEVDepth(Li et al., AAAI 2023): 128×128、0.4m/セル前後。動機は精度と計算のバランスです。0.4m/セルは多くの実装で「ちょうどよい」分解能として採用されます。
- VAD(Jiang et al., ICCV 2023): 計画指向のため前後を厚く取る非対称範囲(例えば前後を広く、左右を狭く)を採用しました。動機は運転計画では進行方向の情報が重要なことです。前方バイアスの発想と通じます。
- SparseBEV(Liu et al., ICCV 2023) / StreamPETR(Wang et al., ICCV 2023): 密なグリッドサイズではなく疎なクエリ数で効率を語る系統です。実効的に100×100相当の粗さでも疎クエリで高精度を狙います。動機は推論速度優先です。トレードオフは密グリッド前提の手法と直接比較しにくい点です。
- FB-BEV(Li et al., ICCV 2023): forward(深度lift)とbackward(クエリ投影)を組み合わせ、グリッドの「空白セル」を減らす設計です。backward projectionだけだと、どのカメラにも当たらないセルが空白になる問題を、forwardで埋めます。範囲・解像度を活かしきる工夫の一例です。

この系譜の中で、前方を大きく取った非対称設定(例えば前方120m、左右60m)は、検出ベンチマーク標準の200×200対称とは異なる「計画・高速走行重視」の独自設定として位置づけられます。標準と直接比較しにくくなる代わりに、高速での早期検知に有利になります。

## 論文の実験結果(定量データ)

解像度・範囲が性能に与える影響は、各論文のアブレーションに現れます。

BEVFormer(2022)はnuScenes検出で、鳥瞰図解像度を下げる(セルを粗くする)とmAP・NDSが低下することを報告しています。具体的には、グリッドを粗くすると小さい物体(歩行者・自転車)の検出精度が落ちやすく、これはセルサイズが物体サイズを下回らないと位置を解像できないという直感と一致します。指標mAPは検出の正確さ、NDSは位置・速度誤差も含む総合指標で、いずれも高いほど良いです。逆に解像度を上げると精度は上がるが計算量とメモリが2乗で増えるため、200×200あたりが論文の採用点になっています。

BEVDepth(2023)は、深度監督に加え鳥瞰図解像度・範囲の設定が検出精度に効くことを示し、0.4m前後のセルサイズが精度と計算のバランス点として機能することを定量化しました。報告では深度監督と適切な解像度の組み合わせでnuScenes test setのNDSを当時の上位水準(0.6前後)まで押し上げています。0.4m/セルという値が複数の独立した研究で「ちょうどよい」とされるのは、乗用車・車線・歩行者という主要対象がこの分解能で確実に複数セルを占めるためと理解できます。

知覚範囲については、範囲を遠くまで広げると遠距離物体の検出が可能になる一方、同じセル数なら遠距離のセルサイズが粗くなる(範囲 ÷ セル数が増える)トレードオフが各実装で観察されます。だからこそ「範囲を広げるなら解像度も上げる」必要があり、両者がVRAMを介して結合しています。前方バイアスはこの制約下で「全方位を遠く・細かく」は無理なので「効く方向(前方)に厚く配分する」という最適化と理解できます。VAD(2023)が計画指向で非対称範囲を採ったのは、検出最適の対称範囲が計画には最適でないという認識の表れです。

## メリット・トレードオフ・限界

メリット:
- 0.4m/セル程度の分解能で車・車線・歩行者を明確に区別でき、精密な計画が可能です。
- 前方を遠く(例えば120m)見ることで高速走行での早期検知に有利です。停止距離が速度の2乗で伸びる物理に直結した設計です。
- 後方・側方を近く(例えば60m)に抑え、計算資源を前方に集中できます。

トレードオフ・限界:
- 高解像度はVRAMを大量に消費し、バッチサイズを下げざるを得ません(高解像度・大バッチでOOM)。バッチが小さいと勾配のノイズが増え、学習が不安定になることもあります。
- 後方・側方の知覚が前方より浅いと、後方からの高速接近車や側方の死角に相対的に弱くなります。割り込みや幅寄せへの対応が課題になり得ます。
- 非対称・独自範囲は検出ベンチマーク標準(200×200対称)と直接比較しにくく、論文値との照合が難しくなります。再現性・比較可能性とのトレードオフです。
- セルサイズより小さい物体(遠方の歩行者など)は原理的に解像できません。解像度の下限が分解能の上限を決めます。

研究上の未解決課題として、「どの解像度・範囲が下流タスク(計画)に最適か」はタスク依存で、検出に最適な設定が計画に最適とは限りません。計画指向で範囲・解像度を選ぶ研究(VADの非対称範囲など)はまだ発展途上で、車速や走行環境(高速道路か市街地か)に応じて動的に変える方向も模索されています。

## 発展トピック・研究の最前線

- 適応的解像度: 前方や物体周辺だけ細かく、空きスペースは粗くする可変解像度の鳥瞰図です。一様グリッドの無駄を省きます。
- 疎表現への移行: 密グリッドをやめ、物体クエリだけで遠距離も扱います(SparseBEV系)。解像度とメモリの結合を断ち切る方向です。
- 占有予測(occupancy): 2D鳥瞰図を3Dボクセルに拡張し、高さ方向も解像します。範囲・解像度の議論がZ方向にも広がり、未知形状の障害物にも対応しやすくなります。
- メモリ最適化: gradient checkpointing、混合精度(bf16/fp16)、tiling(グリッドを分割して順に処理) で高解像度学習をGPUに収める実務技術です。
- タスク連動の範囲設計: 車速に応じて知覚範囲を動的に変える(高速なら前を遠く、低速なら近くを細かく)研究です。停止距離の物理を制御に組み込む方向です。
