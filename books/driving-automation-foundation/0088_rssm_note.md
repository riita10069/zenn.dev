# RSSM (Recurrent State-Space Model)

## ひとことで言うと
RSSM(Recurrent State-Space Model、再帰的状態空間モデル)は、「世界がこの先どう移り変わっていくか」を、画像そのものではなく小さな潜在ベクトル(数値の並び)の上で予測する仕組みです。最大の発明は、世界の状態を「必ず確実に伝わる決定論的な部分」と「ランダムにブレる確率的な部分」の2つに分けて持つことにあります。この分離のおかげで、何十ステップも先まで予測しても過去の情報が消えずに安定し、それでいて「次に何が起きるか分からない」という不確実性もきちんと表現できます。潜在空間で未来を想像する系統のワールドモデル(環境の挙動を頭の中で再現するモデル)の事実上の標準アーキテクチャになっています。

## 直感的な理解
人間が運転中に未来を予測する様子を思い浮かべてください。あなたは「10秒前に右側の信号が赤だったこと」をしっかり覚えています。これは確実な記憶で、ランダムに揺らいだりしません。一方で「前を走る車が次の交差点で右折するか左折するか」は分かりません。これは確率的な、複数の可能性が併存する情報です。

賢い予測には、この両方が要ります。確実な記憶だけだと、不確実な未来を1つの平均にぼかしてしまう。不確実性だけだと、毎瞬間ランダムに揺らいで大事な記憶が消えてしまう。RSSM は、記憶を運ぶ「決定論レーン」と、ばらつきを表す「確率レーン」を1本ずつ並べて走らせることで、この問題を一挙に解きます。これが「なぜ2経路に分けるのか」という問いの答えであり、RSSM が後続の多くの研究で再利用される理由です。

身近な対比でいうと、決定論レーンは「車のオドメーター(走行距離計)」のように単調に過去を積み上げる装置、確率レーンは「サイコロを振って次の出目を予想する装置」です。両方を1台のマシンに同居させ、毎時刻で連携させたのが RSSM だと考えると、後の説明が読みやすくなります。

## 基礎: 前提となる概念
RSSM を理解するために、いくつかの言葉を噛み砕いておきます。

潜在状態(latent state)とは、未来を予測するのに必要な情報だけを凝縮した小さなベクトルのことです。たとえばカメラ画像 64×64×3 = 12288 個の数字を、30 個程度の数字に圧縮したものです。この 30 個の中に「車の位置」「速度」「障害物の有無」といった本質が詰まっている、という考え方です。「次の予測に使わない情報(背景の雲の形など)は捨てる」のが圧縮の目的です。

状態空間モデル(state-space model)とは、「観測できる現象(画像など)の裏に、観測されない真の状態があり、その状態が時間とともに遷移していく」とみなすモデルの総称です。状態 → 次の状態 → 次の状態、と遷移していき、各状態から観測が生み出される、という構造を仮定します。古典的にはカルマンフィルタ(線形ガウスの状態空間モデル)が代表例で、RSSM はその遷移関数をニューラルネットで非線形に拡張したもの、と位置づけられます。

マルコフ性(Markov property)とは、「次の状態を決めるのに、過去のすべての履歴ではなく、いまの状態だけあれば十分」という性質です。状態がマルコフ的であってほしいのは、未来予測を「現在の状態 → 次の状態」という1ステップの繰り返しに還元できるからです。RSSM の状態が「未来予測に必要な情報をすべて含む」よう学習されるのは、このマルコフ性を成り立たせるためでもあります。

RNN(Recurrent Neural Network、再帰型ニューラルネット)とは、前の時刻の出力を次の時刻の入力に回し込むことで、過去の文脈を内部に持ち越しながら系列を処理するネットワークです。RSSM では主に GRU(Gated Recurrent Unit、ゲート付き再帰ユニット。LSTM を簡略化した RNN の一種)が使われます。「再帰的(recurrent)」という名前はここから来ています。ゲートとは「過去の記憶をどれだけ残し、新しい入力をどれだけ取り込むか」を 0〜1 の重みで制御する仕組みで、これにより長い系列でも記憶が急激に消えない(勾配消失を緩和する)ようになっています。

確率分布とサンプリング: 確率的状態は「このあたりの値になりそう」という分布として表されます。そこから実際に1つの値を取り出すことを「サンプリングする」と言います。ガウス分布(正規分布、釣り鐘型の分布)が使われることが多いです。サンプリングは普通そのままでは微分できませんが、ガウス分布なら reparameterization trick(平均 μ と標準偏差 σ を使い s = μ + σ·ε、ε はノイズ、と書き換えることで、ノイズ ε を外に出して μ・σ について微分可能にする技)で勾配を通せます。

KL ダイバージェンス(Kullback-Leibler divergence)とは、2つの確率分布がどれだけ食い違っているかを測る量です。値が小さいほど2つの分布は似ています(0 なら完全一致)。非対称(P から見た Q と Q から見た P で値が違う)で距離ではありませんが、「どれだけずれているか」の尺度として使えます。RSSM では「観測を見る前の予測」と「観測を見た後の予測」という2つの分布をこの量で近づけます。

変分推論(variational inference): 観測 o から、直接は見えない潜在状態 s の分布を推定したいとき、扱いやすい近似分布 q(s|o) を用意し、それを真の事後分布に KL で近づける枠組みです。RSSM の posterior q と prior p の構造はこの理論に根ざしており、学習目的の ELBO(変分下界)は「再構成のうまさ − KL ペナルティ」という形になります。

## 仕組みを詳しく
RSSM は、各時刻 t の潜在状態を2つの成分に分けます。

- 決定論的状態 h_t(deterministic state): GRU で時刻をまたいで受け渡される、確実に伝わる「記憶」。過去を長く保持する役割。
- 確率的状態 s_t(stochastic state): 確率分布からサンプリングされる、不確実性を表す成分。

「状態」と呼ぶときは普通この2つを連結した (h_t, s_t) を指します。下流のデコーダ・報酬予測・方策はこの連結ベクトルを入力に取ります。

### 1ステップの計算の流れ

時刻 t で行動 a_{t-1} を取り、観測 o_t を得たとき、次のように状態を更新します。記号の意味を添えて説明します。

```
1. 決定論経路(記憶の更新):
   h_t = f_GRU( h_{t-1},  s_{t-1},  a_{t-1} )
   ← 前ステップの記憶 h_{t-1}、前ステップの確率状態 s_{t-1}、直前の行動 a_{t-1}
     をまとめて GRU に通し、新しい記憶 h_t を作る。
   ← この経路は完全に決定論的。同じ入力なら必ず同じ h_t になる。記憶を絶対に失わせない。

2. 事前分布(prior、まだ観測 o_t を見ていない予測):
   ŝ_t ~ p( s_t | h_t )
   ← 記憶 h_t だけから「次の確率状態はこのあたり」というガウス分布(平均 μ_p, 標準偏差 σ_p)
     を出し、そこからサンプリング。
   ← これが「観測なしで未来を想像する」経路。実環境を動かさず夢の中で進められる肝。

3. 事後分布(posterior、観測 o_t を見た後の補正):
   s_t ~ q( s_t | h_t,  o_t )
   ← 実際の観測 o_t をエンコードした特徴も使って、より正確な確率状態
     (平均 μ_q, 標準偏差 σ_q)を出す。
   ← 学習時や、実環境から観測が得られるときに使う。
```

ここで決定的に重要なのは、prior(観測なしの予測 ŝ_t)と posterior(観測ありの補正 s_t)の2つの分布を、学習中に KL ダイバージェンス KL( q(s_t|h_t,o_t) ‖ p(s_t|h_t) ) で近づけることです。なぜこれが重要かというと、「観測がなくても、観測があったときに近い予測ができる」ようにするためです。この性質があるからこそ、計画や想像のときに観測なしで未来を長く転がせます。逆にいえば prior が posterior に追いつけないと、夢の中の予測が現実とずれ、計画が破綻します。

### 学習に使う損失

RSSM は次の3つの損失を足し合わせて学習します(おおまかに、これが ELBO に対応します)。

```
全損失 = 再構成損失 + 報酬予測損失 + β × KL損失

再構成損失   : 状態 (h_t, s_t) から元の観測 o_t をデコーダで復元したときの誤差
               (ELBO の「データを説明できているか」の項)
報酬予測損失 : 状態から報酬 r_t を予測したときの誤差
KL損失       : prior p(s_t|h_t) と posterior q(s_t|h_t,o_t) の食い違い
               (ELBO の正則化項。複雑さのペナルティ)
β            : KL損失の重み。大きすぎると posterior が観測を無視し(posterior collapse)、
               小さすぎると prior が育たず観測なし予測が当たらなくなる
```

DreamerV2 以降では KL balancing(prior 側と posterior 側で KL の学習率を変え、prior を速く posterior に追従させる)や free bits(KL が一定値未満なら罰しない)といった工夫が加わり、この β 調整の難しさを緩和しています。

### tensor shape の数値例

PlaNet 設定での典型的なサイズを示します(B はバッチサイズ、L はシーケンス長)。

```
決定論状態 h_t   [B, 200]      ← GRU の隠れ状態。大きめにして記憶を担わせる
確率状態   s_t   [B, 30]       ← サンプリングされる不確実性成分(連続ガウス版)
状態全体        [B, 230]      ← h_t と s_t を結合したもの。これが「状態」として下流に渡る
系列で扱う時     [B, L, 230]   ← 学習は L=50 程度のチャンクを切り出して時間方向に展開
```

この [B, 230] の状態から、デコーダで [B, 3, 64, 64] の画像を復元したり、スカラーの報酬を予測したりします。DreamerV2 の離散版では s_t が「32 カテゴリ × 32 グループ = 1024 次元の one-hot 束」になり、[B, 32, 32] の構造を持ちます。

### before/after で見る2経路の効き目

- 決定論経路がない(確率だけ)場合: 10ステップ先を予測すると、毎ステップのサンプリングで過去情報が薄まり、予測が崩壊する。「赤信号を見た」記憶が数ステップで消える。
- 決定論経路がある場合: h_t が GRU で過去を運ぶので、10ステップ先でも記憶が残り、一貫した予測ができる。
- 確率経路がない(決定論だけ)場合: 「対向車が右か左か」の二択を平均してぼやけた、現実にはあり得ない中間の未来になる(modeの崩壊)。
- 確率経路がある場合: 複数あり得る未来を分布として表現でき、サンプリングするたびに異なる未来を出し分けられる。

## 手法の系譜と主要論文
RSSM は単独で生まれたのではなく、ワールドモデル研究の流れの中で結晶化した部品です。

源流として Ha & Schmidhuber の World Models(NeurIPS 2018、arXiv:1803.10122)があります。これは VAE(画像圧縮)と MDN-RNN(未来予測)を別々に訓練する3段構成で、潜在空間で未来を予測する発想を世に広めました(詳細は [World Models (Ha & Schmidhuber)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0106_world-models-ha-schmidhuber))。ただし圧縮と予測が分離していて end-to-end ではなく、決定論/確率の明示的分離もありませんでした。

RSSM の初出は Hafner, Lillicrap, Fischer, Villegas, Ha, Lee, Davidson "Learning Latent Dynamics for Planning from Pixels"(PlaNet、ICML 2019、arXiv:1811.04551)です。ここで「決定論経路と確率経路を1つのモデルに併せ持つ」というアイデアが提案され、潜在ダイナミクス全体を end-to-end で学習しました(詳細は [PlaNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0082_planet_note))。著者らはこの分離を、純粋に決定論的な RNN モデルでも純粋に確率的な状態空間モデルでもどちらも不十分だった、という実証から導いています。

続く Hafner, Lillicrap, Ba, Norouzi "Dream to Control"(DreamerV1、ICLR 2020、arXiv:1912.01603)は、RSSM をほぼそのまま流用しつつ、計画方法を CEM(その場の最適化)から actor-critic(行動を出す方策ネットと価値を見積もる価値ネットを潜在空間で勾配学習)に置き換えました(詳細は [Dreamer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0047_dreamer-v1))。RSSM が微分可能であることを活かし、想像した軌跡に沿って価値の勾配を方策へ逆伝播する点が新しく、RSSM の汎用性が示された瞬間です。

Hafner, Lillicrap, Norouzi, Ba "Mastering Atari with Discrete World Models"(DreamerV2、ICLR 2021、arXiv:2010.02193)は、RSSM の確率状態を連続ガウスから「離散カテゴリ変数(32 カテゴリ × 32 グループ)」に変更しました。映像のように急な場面転換やマルチモーダル(複数の山を持つ)な未来は、連続ガウスより離散カテゴリの方が表しやすいという発見です。離散変数は微分できないため straight-through estimator(順伝播は離散サンプル、逆伝播は softmax 確率の勾配を通す近似)で学習します。

Hafner, Pasukonis, Ba, Lillicrap "Mastering Diverse Domains through World Models"(DreamerV3、arXiv:2301.04104、後に Nature 2025 掲載)は、RSSM を中核に据えたまま、symlog 変換(値域の広い量を対数的に圧縮し勾配を安定化)、報酬・価値の two-hot 離散回帰、percentile による return 正規化など多数の安定化技術を加え、ハイパーパラメータをタスクごとに調整せず固定したまま 150 以上の多様なタスクを解きました(詳細は [DreamerV3](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0049_dreamer-v3))。

並行して RSSM の GRU を別の系列モデルに置き換える研究も生まれ、Chen et al. "TransDreamer"(arXiv:2202.09481)は GRU を Transformer に替えて長期依存の捕捉を狙い、後に S4/Mamba 系の状態空間層で置き換える研究へと続きます。

## 論文の実験結果(定量データ)
RSSM の有効性は、それを使った各論文のベンチマーク結果から読み取れます。

PlaNet(ICML 2019)は DeepMind Control Suite という連続制御のベンチマーク(振り子を立てる、チーターを走らせるなど)で評価されました。ここでの指標は「エピソードあたりの累積報酬」で、値が高いほど良い性能です。PlaNet は、当時の代表的なモデルフリー手法 A3C や D4PG に匹敵する累積報酬を、報告ではおよそ 50 倍少ない環境インタラクション(環境を実際に動かした回数)で達成しました。この「50 倍少ない試行で同等」がサンプル効率の高さを示し、RSSM による潜在予測が成立していることの証拠になります。

PlaNet 論文のアブレーション(部品を抜いて効果を測る実験)では、決定論経路だけ(純粋な RNN モデル)、あるいは確率経路だけ(純粋な状態空間モデル)にすると性能が明確に低下することが報告されています。両方を併せ持つ RSSM が最も高い累積報酬を出し、「2経路の併存が本質である」という主張を定量的に裏づけました。

DreamerV2(ICLR 2021)は Atari 57 ゲームで評価されました。指標は「gamer-normalized score(人間ゲーマーのスコアを基準に正規化した相対値)の中央値」です。DreamerV2 は単一 GPU 約 1 週間(約 2 億フレーム)の学習で、人間ゲーマー水準を超える中央値スコアを達成し、同じ計算規模・データ量の Rainbow や IQN といった代表的モデルフリー手法を上回りました。このとき効いていたのが RSSM の離散化で、論文のアブレーションでは確率状態を連続ガウスに戻すとスコアが下がること、KL balancing を外しても下がることが示されています。離散カテゴリ変数の方が映像系の不確実な未来を表現しやすい、という設計判断が数値で支持されました。

DreamerV3(arXiv:2301.04104 / Nature 2025)は Atari、DMC(proprio/vision)、Atari100k、Crafter、DMLab、さらに Minecraft のダイヤモンド採掘という長期かつ疎報酬のタスクまで、合計 150 以上のタスクをすべて同一のハイパーパラメータで解きました。特に Minecraft でダイヤモンドを人手のデモデータなしにゼロから採掘できた点が広く注目されました。またモデルサイズ(パラメータ数)を増やすほど性能とサンプル効率がともに上がる、というスケーリング則的な傾向も報告されています。これらすべての中核に同じ RSSM があり、アーキテクチャの普遍性を物語っています。

## メリット・トレードオフ・限界
メリット:
- 決定論経路で長期の記憶を保持でき、長いロールアウト(想像)でも予測が崩壊しにくい。
- 確率経路で「あり得る複数の未来」という不確実性を分布として表現できる。
- prior(観測なし予測)を学習するので、観測なしで未来を想像でき、計画や夢での学習が可能になる。
- 全体が微分可能なので、想像軌跡に沿って方策の勾配を流せる(Dreamer の actor-critic 学習の前提)。
- 潜在ワールドモデルの事実上の標準で、Dreamer 系すべての基盤になっている。

トレードオフ・限界:
- GRU の逐次計算のため時間方向に並列化しにくく、Transformer ベースの系列モデルに比べ学習が遅め。近年は RSSM を Transformer や状態空間モデル(S4/Mamba 系)に置き換える研究(TransDreamer など)も現れている。
- 連続ガウス版は多峰性(複数の山)の表現に弱い。DreamerV2 の離散化はこの弱点への対処だが、離散変数は微分できないため straight-through gradient が必要で実装が複雑になる。
- KL 損失の重み β の調整が学習安定性に強く効き、チューニングが必要。重すぎると posterior が観測を無視して予測がぼやけ(posterior collapse)、軽すぎると prior が育たず観測なし予測が当たらなくなる。KL balancing/free bits はこの緩和策。
- 再構成損失に依存する設計では、報酬に無関係だが視覚的に目立つ背景に潜在が引っ張られうる。報酬や価値の予測誤差を主信号にする(再構成を捨てる)方向の研究もこの限界への応答(関連は [潜在空間予測 vs ピクセル予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0068_latent-vs-pixel-prediction))。
- 想像が長くなるほど誤差が累積し、現実とずれる(compounding error)。これはモデルベース RL 全般の本質的課題で、想像ホライズンを短く切る、価値で末端を補正する、などの工夫で抑える。

## 発展トピック・研究の最前線
RSSM の GRU を別の系列モデルに置き換える試みが盛んです。TransDreamer は GRU を Transformer に替え、長期依存の捕捉を狙いました。さらに状態空間モデル(S4、Mamba)で置き換える研究もあり、逐次計算のボトルネックを並列学習可能な形に変えつつ長期記憶を保つことを追求しています。

prior/posterior の KL を使う構造は、変分推論(観測から潜在を推定する枠組み)の文脈で理解でき、表現学習一般とも接続します。また「再構成損失を捨てて、価値や報酬の予測だけで潜在を鍛える」reconstruction-free な方向(TD-MPC2、自己予測表現を使う手法など)は、RSSM の素朴な再構成依存への代替案として研究が続いています。

運転や物流のような実世界応用では、他エージェントの意図が本質的に不確実なため、RSSM の「決定論で記憶、確率で不確実性」という分離は相性が良く、長期予測の枠組みとして参照され続けています。一方で実世界の生成は拡散モデルやトークン自己回帰の大規模生成モデル(運転動画生成など)が主流になりつつあり、RSSM はそれらと「圧縮された潜在で未来を扱う」という思想を共有しつつ、低次元・高速な制御中核として住み分けています。
