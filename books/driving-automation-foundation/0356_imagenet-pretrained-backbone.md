# ImageNet事前学習バックボーンの転移

## ひとことで言うと
バックボーン(画像を特徴ベクトルに要約する土台ネットワーク)を、目的のタスクのデータでゼロから学習させるのではなく、ImageNet という巨大な画像データセットで事前に学習済みの重みを初期値として使い、そこから少しだけ追加学習する、という転移学習(transfer learning)の手法です。汎用的な視覚特徴を再利用できるので、少ないデータと計算で高い性能に届きます。

## 直感的な理解
ニューラルネットワークをランダムな初期値から学習させると、「ここに縦のエッジがある」「これは赤い面だ」「この形は車輪だ」といった、ありとあらゆる視覚の基礎を一から覚え直させることになります。これは料理を覚えるのに、包丁の握り方から火の起こし方まで毎回ゼロから教えるようなものです。

たとえば中規模ネットワークの ResNet-50 でも約2500万個のパラメータ(調整可能な数値)を持ちます。これを目的タスク(たとえば運転シーンの認識)のデータだけで埋めるには、数十万から数百万枚規模のラベル付き画像が要ります。しかし運転データは収集もアノテーション(正解付け)も高コストで、そこまで大量には用意できません。

ここで転移学習が効きます。発想はこうです。すでに ImageNet という巨大データで鍛えられたネットワークは、「エッジ・色・テクスチャ・物体の輪郭」といった汎用的な視覚特徴をもう知っています。これらは猫や犬を見分けるために学ばれたものですが、低レベルの視覚特徴(線・角・質感)はどんな画像でも共通なので、車や道路や歩行者を見分けるのにもそのまま役立ちます。だから「ImageNet で学んだ重みを初期値にして、目的データで少しだけ仕上げ(fine-tuning、ファインチューニング)」すれば、ゼロから学ぶより圧倒的に少ないデータと時間で良い性能に届きます。

## 基礎: 前提となる概念
バックボーン(backbone): モデルの中で「画像を意味ある特徴に変換する」中心部分です。後段のタスク固有のヘッド(分類器・検出器・プランナーなど)に特徴を供給する「背骨」なのでこう呼ばれます。

事前学習(pretraining)と微調整(fine-tuning): 事前学習は本番タスクの前に大きな別データで学んでおくこと。微調整は事前学習済みの重みを初期値に、目的タスクのデータで追加学習することです。微調整では学習率(1回の更新でどれだけ重みを動かすか)を小さめにして、せっかく学んだ汎用特徴を壊さないようにするのが定石です。重みを更新せず固定したまま特徴だけ使う「特徴抽出(feature extraction)」という使い方もあります。

ImageNet: 約128万枚の学習画像・1000クラスの一般物体画像データセット(正確には ILSVRC という競技用サブセット)。深層学習全体の「土台データ」として機能してきました。より大きな ImageNet-22k(約1400万枚・2万超クラス)を事前学習に使う流派もあります。

多スケール特徴(multi-scale features): ネットワークの浅い層は高解像度で局所的な特徴(エッジなど)を、深い層は低解像度で大域的・意味的な特徴(物体全体など)を持ちます。これらを各段階から取り出したものが多スケール特徴で、検出やセグメンテーション、BEV(鳥瞰図)融合のように「細かい位置」も「広い意味」も両方欲しいタスクで重要です。

channels-first / channels-last: テンソルの軸の並びの慣習です。channels-first は `[B, C, H, W]`(チャンネルが先)、channels-last は `[B, H, W, C]`(チャンネルが後)。CNN 系は前者、Transformer 系の一部は内部で後者を扱うことがあり、混在すると形状の不一致でバグります。

## 仕組みを詳しく
事前学習バックボーンを使う典型的な構成は、画像モデルライブラリ(PyTorch エコシステムでは timm = PyTorch Image Models が定番)を介して次のように書けます。

```python
import timm
backbone = timm.create_model(
    "swinv2_tiny_window8_256",   # アーキ名
    pretrained=True,             # ImageNet 学習済み重みを読み込む
    features_only=True,          # 分類層を外し多スケール特徴を返す
)
```

要素を分解します。
- `pretrained=True`: ImageNet 等で学習済みの重みを自動ダウンロードして初期値にします。`False` ならランダム初期値(スクラッチ学習)。
- `features_only=True`: 最終分類層(1000クラスを当てる全結合層)を取り除き、途中の複数ステージの特徴マップを返すようにします。運転や検出では分類スコアではなく各解像度の特徴マップが欲しいからです。

ライブラリは各ステージのチャンネル数を `feature_info` のような属性で教えてくれるので、下流側はそれを読んで配線します。万一取得できない場合は、ダミー入力(例 `[1,3,256,256]`)を一度通して出力形状からチャンネル数を推定するフォールバックを置くのが、実装上の堅実なやり方です。

実装上の落とし穴が軸の並びです。同じ「多スケール特徴」でも、アーキによって channels-first と channels-last が混在します。

- ResNet / ConvNeXt 系: `[B, C, H, W]`(channels-first)
- Swin 系の一部実装: `[B, H, W, C]`(channels-last)

下流の融合モジュールが channels-first を前提とするなら、forward の中で形状を見て必要なら permute(軸の入れ替え)して揃えます。

```python
if f.shape[1] != expected_c and f.shape[-1] == expected_c:
    f = f.permute(0, 3, 1, 2)   # [B,H,W,C] -> [B,C,H,W]
```

この「形状自動判定 + 統一」を一箇所に集約しておくと、どのバックボーンを選んでも下流コードは同じ形状を受け取れます。バックボーンを登録テーブル(registry、名前をキーに生成関数を引ける辞書)に追加するだけで切り替えられる、という拡張性もここから来ます。

数値例: 入力 `[B, 3, 256, 256]` を Swin V2 Tiny に通すと、4ステージの多スケール特徴(おおむね `[B,96,64,64]`, `[B,192,32,32]`, `[B,384,16,16]`, `[B,768,8,8]` のような階層構造)が返り、permute で全て channels-first に揃えられます。注意点として、事前学習時の入力解像度・正規化(各チャンネルを ImageNet 統計の mean/std で標準化)に合わせないと、分布がずれて性能が落ちます。Swin V2 のように特定窓サイズ・特定解像度(例 256)で学習されたモデルは、その前提に揃えるのが安全です。

## 手法の系譜と主要論文
ImageNet(Deng et al., CVPR 2009, "ImageNet: A Large-Scale Hierarchical Image Database")。約128万枚・1000クラスのベンチマークを構築し、データ駆動の視覚研究を加速しました。「大規模で多様なベンチマークが分野を進める」という主張の実証で、事前学習文化そのものの基盤になりました。

AlexNet(Krizhevsky et al., NeurIPS 2012)。ImageNet 競技で深層 CNN が従来手法を圧倒し(top-5 誤り率を前年の約26%から約16%へ一気に下げ)、転移学習の有用性(畳み込み特徴が他タスクに移ること)を広めた転換点です。

ResNet(He et al., CVPR 2016, "Deep Residual Learning for Image Recognition")。残差接続(skip connection、入力をそのまま出力に足す近道)で「層を深くすると勾配消失で学習が劣化する」問題を解き、152層級の超深層ネットを安定学習可能にしました。ImageNet で当時最高性能を達成し、今も pretrained backbone の定番です。新しい Transformer 系に精度・効率でやや劣る場面はありますが、シンプルさと枯れた実装で根強く使われます。

Vision Transformer / Swin Transformer(Dosovitskiy et al. 2020 ViT; Liu et al. CVPR 2021 Swin; Liu et al. CVPR 2022 Swin Transformer V2, arXiv:2111.09883)。ViT は画像をパッチ列として Transformer で処理しますが、全パッチ間の自己注意は計算量が画像サイズの二乗で重くなります。Swin はウィンドウ(局所領域)内に注意を限定し、ウィンドウをずらして(shifted window)情報を交換することで、計算量を画像サイズに線形化しつつ多スケール特徴を出せるようにしました。Swin V2 はさらに高解像度・大モデルへの安定スケーリング(残差後正規化、対数化したコサイン注意など)を導入しました。検出・セグメンテーションで強い反面、実装が複雑で CNN より推論が重い場合があります。

ConvNeXt / ConvNeXt V2(Liu et al. CVPR 2022; Woo et al. CVPR 2023, "ConvNeXt V2", arXiv:2301.00808)。純粋な畳み込みネットに Transformer の良い設計要素(大カーネル、層正規化、GELU など)を取り込み、Transformer 並みの精度を CNN の効率で実現しました。V2 では FCMAE(Fully Convolutional Masked AutoEncoder、マスク自己教師あり事前学習の畳み込み版、[MAE (Masked Autoencoders)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0070_mae_note) の親戚)と GRN(Global Response Normalization)を組み合わせ、自己教師あり事前学習でさらに強化しました。

転移の効き目を定量化した重要研究が Kornblith et al.(CVPR 2019, "Do Better ImageNet Models Transfer Better?", arXiv:1805.08974)です。多数のアーキを横断的に比較し、「ImageNet 精度が高いモデルほど転移先タスクでも概ね強い」という正の相関を報告しました。ただし常に成り立つわけではなく、ドメインが大きく異なる(医療画像など)場合や、ファインチューニングよりも特徴固定(feature extraction)で使う場合に例外が出ることも示されています。

## 論文の実験結果(定量データ)
ImageNet top-1 精度(1000クラス分類でモデルの最上位予測が正解と一致する割合。高いほど良い)を物差しにすると、系譜の進歩が数値で追えます。

- ResNet-50: ImageNet top-1 でおよそ 76% 前後(標準的な学習設定)。改良された学習レシピでは 80% 近くまで報告されます。
- Swin Transformer V2 Tiny: 約 28M パラメータ規模で、報告により top-1 約 82% 前後。Swin V2 の最大モデル(SwinV2-G、約30億パラメータ)は当時 ImageNet-V2 などで最高水準を更新しました。
- ConvNeXt V2: FCMAE 事前学習 + 微調整で、Tiny 規模でも top-1 約 83% 前後、最大の Huge 規模では公開データのみで top-1 約 88% 超を報告しています。

転移の定量効果として Kornblith et al. は、ImageNet top-1 と 12 種の下流分類データセットでの精度の間に強い線形相関(報告された相関係数はおおむね 0.96 と高い)を見出しました。相関係数は -1〜1 の値で、1 に近いほど「片方が高ければもう片方も高い」という関係が強いことを意味します。0.96 という値は「ImageNet で良いモデルを選ぶ」という単純な戦略が、別タスクでも概ね有効であることを示します。一方で、ファインチューニングと特徴固定では相関の強さが変わり、ドメインギャップが大きいタスクでは事前学習の優位が縮むという注意も同時に示されています。

ResNet 論文のアブレーションでは、残差接続を外した「ただの深いネット」は層を深くするほど学習誤差が悪化(劣化現象、degradation)するのに対し、残差接続を入れると深くしても改善し続けることが示されました。具体的には 34 層のプレーンネットは 18 層より学習誤差が高くなるのに、残差版では 34 層が 18 層を上回りました。これが「深さを安全に増やせる」という主張の直接的な証拠で、その後のほぼ全アーキの土台になりました。

## メリット・トレードオフ・限界
メリット
- 少ないデータでも高性能に到達できる(汎用視覚特徴の再利用)。
- 学習が速く収束し、計算コストを大幅に削減できる。
- 登録テーブル + 形状自動判定で配線を共通化すれば、バックボーンを差し替えても下流コードを変えずに済む。

トレードオフ・限界
- ドメインギャップ: ImageNet は物体中心の整った写真ですが、運転は広域シーン・夜間・悪天候・逆光を含みます。そのまま使うと運転特有の困難条件に弱いことがあり、運転データ自体での自己教師あり事前学習([MAE (Masked Autoencoders)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0070_mae_note) など)が有効な場面があります。
- 事前学習重みのロードや巨大モデルのダウンロードに時間とストレージを要し、初期化が学習・テスト時間の支配項になることがあります。
- 正規化統計・入力解像度・窓サイズなど事前学習時の前提に合わせる必要があり、合わないと精度が落ちます。
- ライセンス・出所の確認が必要です。どのデータで学習されたか(ImageNet-1k か、より大きな ImageNet-22k や非公開データか)で性質も使用条件も変わります。
- 未解決課題: ドメイン特化の事前学習(運転・医療など)と汎用事前学習のどちらがいつ勝つか、自己教師あり事前学習と教師あり事前学習の使い分け、スケールと品質のどちらを優先すべきか、といった問いは依然として研究対象です。

## 発展トピック・研究の最前線
近年は教師あり ImageNet 事前学習から、自己教師あり(MAE、DINO、SwAV、CLIP のような言語画像対照学習)へと事前学習の主役が移りつつあります。CLIP(Radford et al. 2021)は画像とテキストの対照学習で、ゼロショット(微調整なし)でも強い汎化を示し、バックボーンの「常識」を変えました。運転分野では、走行映像での自己教師あり事前学習、BEV 特化の事前学習、基盤モデル(foundation model、巨大データで学んだ汎用モデル)からの蒸留(大モデルの知識を小モデルに移す)が活発です。アーキ面では Swin / ConvNeXt の後継、Mamba 系の状態空間モデル(系列を線形時間で処理する新方式)、ハイブリッド設計が競っています。
