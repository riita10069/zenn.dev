# 模倣学習 (Behavior Cloning)

## ひとことで言うと

模倣学習(Imitation Learning)、その最も単純な形である行動クローニング(Behavior Cloning, BC)は、「上手な人間ドライバが実際に運転したデータ」を教師にして、「同じ状況なら同じように運転する」ことをニューラルネットに真似させる学習方法です。先生のお手本を写経させるイメージで、自動運転モデルを学習させる最も基本的で安定した手段です。一方で「お手本に無い状況に弱い」という根の深い限界を抱えており、その理解が研究の出発点になります。

## 直感的な理解

「どう運転するか」をプログラムで全部書き下すのは極めて困難です。道路には無限のパターンがあり、「こういう時はこうハンドルを切る」というルールを人手で網羅するのは不可能に近い。そこで発想を変えます。「正しい運転=人間の運転」とみなし、人間の運転を大量に記録して、入力(見えている景色)と出力(その時の操作や軌道)の対応関係を機械に学ばせれば、ルールを書かずに運転を獲得できるはずです。これが模倣学習の動機です。

身近な例で言えば、教習所で教官の運転を後部座席から見て覚えるのに似ています。ただし重要な落とし穴があります。教官は常に正しい運転しか見せないので、生徒は「うっかり車線を大きくはみ出してしまったとき、どう立て直すか」を一度も見ていません。本番で初めてはみ出すと、見たことのない状況なのでパニックになり、ミスがミスを呼びます。模倣学習の最大の弱点である分布シフトは、まさにこの構造から生じます。

## 基礎: 前提となる概念

教師あり学習(supervised learning): 入力と正解のペアを大量に与え、入力から正解を当てさせる学習です。模倣学習はこの枠組みそのもので運転を扱います。入力 x = ある瞬間の観測(カメラ画像、速度などのセンサ値)、正解 y = その瞬間に人間ドライバが取った行動(ハンドル角・アクセル、または今後数秒の走行軌道)。これを (x, y) のペアとして大量に集め、「x を入れたら y を出す」ようネットワークを訓練します。学習されたものがそのまま運転方策(policy、状況→行動の対応づけ)になります。

軌道(trajectory): 行動の出力形式の一つで、「これから T ステップぶん、自車がどの位置を通るか」を点列で表したものです。たとえば 64 ステップぶんの (x, y) 座標なら、形状 (B, 64, 2) の tensor(B はバッチサイズ)。制御信号(ハンドル角)を直接出す方式より、軌道を出してから別途制御に落とす方式の方が、低レベルの車両特性に依存しにくく扱いやすいとされます。

i.i.d. 仮定の破れ: 教師あり学習は通常「データが独立同分布(i.i.d.)から来る」と仮定します。しかし運転では、いま取った行動が次の状態を決めます。モデルがミスして変な状態に入ると、その後の入力分布が学習データから外れます。これが i.i.d. 仮定の破れであり、後述の分布シフトの正体です。

## 仕組みを詳しく

行動クローニングの学習ループは、典型的に「forward → loss → backward → step」の4工程です。

ステップ1: forward(予測)。モデルにカメラ画像などの観測を入れ、未来の走行軌道を予測します。例として軌道を「64ステップ × 2次元(横位置・縦位置)」として出すと、出力は (B, 64, 2)。「これから64ステップぶん、自分はこう動くつもり」という予測です。

ステップ2: loss(お手本とのズレを測る)。予測軌道 `trajectory_pred` と人間ドライバの実軌道 `trajectory_target` のズレを測ります。実装でよく使われる二つの工夫があります。

第一に Smooth-L1 損失(別名 Huber 損失)です。ズレが小さい時は2乗誤差(MSE)のように、ズレが大きい時は絶対誤差(L1)のように振る舞います。なぜか。データには外れ値(センサノイズや異常なお手本)が混じります。MSE は大きなズレを2乗するので外れ値一個で学習が暴れますが、Smooth-L1 は大きなズレで傾きが一定になり安定します。要素ごとの損失を保持するため `reduction="none"` で計算し、後段の重み付けに渡します。

第二に時間方向の重み付け(temporal decay)です。近い未来の予測を遠い未来より重視します。`weights = temporal_decay ** t`(例 `temporal_decay=0.95`)とすると、t=0(直後)で重み 1.0、t=10 で 0.95^10 ≈ 0.60、t=63 で 0.95^63 ≈ 0.04 と未来ほど小さくなります。理由は、すぐ次にどう動くかの方が安全に直結し、遠い未来は不確実でズレても影響が小さいからです。最終的に `per_timestep_loss × temporal_weights` を平均してスカラー損失を得ます。

数値イメージ。ある時刻の予測が (1.2, 0.3)、お手本が (1.0, 0.5) なら、各成分のズレ 0.2 と 0.2 を Smooth-L1 で評価して平均し、その時刻の重み(例 0.95^t)を掛ける。全時刻・全バッチで平均したものが最終 loss です。

ステップ3: backward(誤差を逆流させる)。`loss.backward()` で、損失を小さくするには各パラメータをどちらへどれだけ動かせばよいか(勾配)が自動計算されます。

ステップ4: step(パラメータ更新)。`optimizer.step()` で勾配の向きに少しだけパラメータを動かします。これを大量の (観測, お手本軌道) ペアで繰り返すと、モデルは徐々に「人間のように走る」ようになります。

全体像。

```
観測(カメラ等) ─► 運転方策NN ─► 予測軌道 (B,64,2)
                                    │
人間ドライバの軌道(お手本) ────────┤ 軌道模倣損失
                                    │  (Smooth-L1 × 時間減衰重み)
                                    ▼
                                 loss(スカラー)
                                    │ backward
                                    ▼
                                 勾配 ─► optimizer.step()(パラメータ更新)
```

## 手法の系譜と主要論文

ALVINN (Pomerleau, NeurIPS 1988, "ALVINN: An Autonomous Land Vehicle in a Neural Network"): 模倣学習による自動運転の元祖。3層の小さなニューラルネットに前方カメラ(と距離センサ)の低解像度画像を入れ、人間の運転を真似てハンドル方向を出力させ、実車を走らせました。動機は当時から「ルールを書くより人間の運転を学ぶ方が現実的」。実走行に成功しましたが、学習した状況から少しでも外れると破綻する弱さが既に見えていました。

End to End Learning for Self-Driving Cars / PilotNet (Bojarski et al., 2016, arXiv:1604.07316): NVIDIA による深層 CNN 版。前方カメラ画像を入れ、人間のハンドル操作を直接模倣する End-to-End モデル。動機は、車線検出など中間処理を人手設計せず、データだけで運転全体を学ぶ方が良いとの仮説。明示的な車線検出なしで車線維持を実現しました。重要な工夫として、左右にずらした追加カメラの画像に「中央へ戻す操作」を正解として加え、分布シフトを部分的に緩和しました。それでもお手本に無い大きな逸脱からの復帰は学びにくいままでした。

DAgger (Ross et al., AISTATS 2011, arXiv:1011.0686, "A Reduction of Imitation Learning and Structured Prediction to No-Regret Online Learning"): 分布シフトへの理論的・実用的な特効薬。学習途中の方策を実際に走らせ、その方策が訪れた状態に対して専門家に正解行動をラベル付けしてもらい、データに追加して再学習する反復手続きです。これにより「方策が実際に出会う状態分布」で学習でき、誤差の連鎖を断ちます([強化学習による運転方策](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0599_reinforcement-learning-driving) とあわせて理解すると良い)。

ChauffeurNet (Bansal et al., RSS 2019, arXiv:1812.03079): 純粋な模倣では稀なミスからの復帰が学べない問題に対し、模倣に「合成した悪い状況(わざと逸脱させた軌道)」と「衝突・逸脱への明示的なペナルティ損失」を加えて補強しました。模倣中心ながら、報酬的なペナルティで方策を矯正する点で強化学習の発想を取り込んだ中間的手法です。模倣単独より頑健になりましたが、悪い状況を合成する設計が必要です。

Causal Confusion in Imitation Learning (de Haan et al., NeurIPS 2019, arXiv:1905.11979): 模倣学習が因果を取り違える現象を体系的に分析。観測情報が増えるほどかえって性能が下がる「因果の取り違え(causal confusion)」を実証し、介入(intervention)による因果同定で緩和できることを示しました。

これらに共通する根本的弱点が分布シフト(distribution shift)/共変量シフト(covariate shift)です。学習中は「人間が運転した良い状態」しか見ません。実走行ではモデルが少しミスしてお手本に無い状態へ入り、知らない状態でさらにミスをし、誤差が雪だるま式に増えます(compounding error、誤差の累積)。

## 論文の実験結果(定量データ)

指標の意味から。模倣学習の評価には、開ループ(open-loop)指標と閉ループ(closed-loop)指標があります。開ループは「お手本の軌道とどれだけ一致するか」を ADE(Average Displacement Error、全時刻の平均位置誤差、単位 m)や FDE(Final Displacement Error、最終時刻の位置誤差)で測ります。閉ループは「実際にモデルを走らせて何回介入(takeover)が要ったか」「衝突率」「走破距離」で測ります。重要なのは、開ループの数値が良くても閉ループで破綻することが頻繁に起きる点で、これが分布シフトの定量的な現れです。

DAgger の論文では、スーパーマリオカートの運転模倣などで、通常の行動クローニングがエピソードを重ねるほど誤差が累積して失敗率が高止まりするのに対し、DAgger は反復するたびに失敗が減り、専門家に迫る性能に収束することが示されました。理論面では、行動クローニングの誤差が時間長 T に対し最悪 O(T²) で増えうる(一回のミスがその後の状態を全部ずらすため)のに対し、DAgger は O(T) に抑えられることが示されています。この「2乗から1乗へ」が、分布シフト対策の価値を端的に表す数値です。

ChauffeurNet の報告では、純粋な模倣だけのベースラインが、わざと逸脱させたテスト場面で頻繁に脱輪・衝突したのに対し、合成した悪状況とペナルティ損失を加えた版では脱輪・衝突が大きく減り、閉ループでの頑健性が向上しました。アブレーションとして、ペナルティ損失(衝突・逸脱項)を外すと頑健性が落ち、合成擾乱を外すと復帰挙動が学べないことが示され、二つの要素が独立に効くことが確認されています。

これらの数値が重要な理由。模倣学習は「開ループ精度を上げること」と「実際に安全に走ること」が乖離しうる、という分野特有の難しさを抱えます。O(T²) という理論限界と、DAgger・擾乱合成による緩和の実験結果は、なぜ単純な BC だけでは実用に届かず、何を足せば届くのかを定量的に教えてくれます。

## メリット・トレードオフ・限界

メリット。仕組みが単純(普通の教師あり学習)で実装・学習が安定しやすい。シミュレータ不要で、記録済み運転データだけで学習できる。大量の人間運転データがあればスケールしやすい。学習が速くデバッグしやすく、エンドツーエンド運転の最初の一歩として最適です。

トレードオフと限界。分布シフト/共変量シフトに弱く、お手本に無い状態でミスが雪だるま式に増えます。専門家の性能が上限で、人間より上手くはなれません。因果の取り違え(causal confusion)が起きうる(例「ブレーキランプが点いた」を結果ではなく原因と誤学習する)。データの偏り(まっすぐ走る場面が多く危険回避が少ない)をそのまま受け継ぎます。研究上の未解決課題は、マルチモーダル性の扱いです。同じ場面で「左に避ける」「右に避ける」の二通りが正解なとき、両者を平均すると「直進して衝突」というありえない行動を学んでしまう(mode averaging)。これを避けるため、軌道を多峰分布として出すアンカー軌道や拡散モデル、混合密度ネットワークなどが研究されています。

## 発展トピック・研究の最前線

分布シフト対策として DAgger 系の対話的学習、擾乱合成と明示ペナルティ(ChauffeurNet 系)、そしてお手本を超える性能を狙う強化学習との組み合わせ([強化学習による運転方策](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0599_reinforcement-learning-driving))が主要な発展方向です。出力のマルチモーダル化(拡散方策 Diffusion Policy、軌道アンカー、混合密度)は近年の活発な領域です。また大規模運転データで事前学習し少数データで微調整する基盤モデル的アプローチ、言語と運転を結ぶ Vision-Language-Action モデル、そして模倣で初期化してからオフライン強化学習で磨く二段構えが、エンドツーエンド運転の実装パターンとして定着しつつあります。
