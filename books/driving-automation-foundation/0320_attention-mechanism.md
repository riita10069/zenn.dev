# アテンション機構

## ひとことで言うと

アテンション機構(attention mechanism)とは、たくさんの入力情報の中から「いま重要なところ」を動的に選び出し、重み付けして取り出す仕組みのことです。「アテンション」は英語で「注意・注目」という意味で、人間が文章を読むときに大事な単語だけに目を向けるのと同じように、ニューラルネットワークが入力のどこに注目するかを学習で決める、という発想です。現在の大規模言語モデルや自動運転の知覚モデルの中核となっている Transformer(トランスフォーマー)という構造の心臓部でもあります。

## 直感的な理解

「カメラを見ながら次にハンドルをどう切るか決める」という場面を考えます。前方の歩行者、左の駐車車両、信号、車線。視界には無数の情報がありますが、ブレーキを踏むかどうかを決めるのに本当に効くのは「前方の歩行者」だけかもしれません。アテンションは、この「どこを見るべきか」という重み(どれくらい注目するか)を、状況に応じて毎回計算し直す仕組みです。

もう少し機械翻訳の例で考えます。「I love cats」を「私は猫が好き」と訳すとき、「好き」という単語を作る瞬間には英語の「love」に強く注目し、「猫」を作る瞬間には「cats」に注目すべきです。固定的に全部の単語を均等に見るのではなく、出力の各時点で入力のどこに注目するかを変える。これがアテンションの直感です。

数値で言うと、入力が5個の単語ベクトルだとして、アテンションは「0.05, 0.05, 0.8, 0.05, 0.05」のような重みの配列(合計が1になる)を計算します。3番目に0.8という大きな重みが付いているので「3番目に強く注目している」という意味になります。この重みで入力ベクトルを加重平均したものが、アテンションの出力です。

## 基礎: 前提となる概念

アテンションを理解するには、いくつかの前提が必要です。

ベクトルと内積。ベクトルとは数値を並べた配列(例: [0.2, -0.5, 1.3])で、ニューラルネットワークでは単語や画像パッチを「特徴ベクトル」という数値の並びで表します。内積(dot product)とは、2つのベクトルの対応する要素を掛けて足し合わせた1つの数値です。例えば [1, 2] と [3, 4] の内積は 1×3 + 2×4 = 11 です。内積は「2つのベクトルがどれくらい似た向きを向いているか」の指標になり、似ているほど大きな値になります。アテンションはこの「内積で類似度を測る」という性質を利用します。

ソフトマックス(softmax)。任意の実数の並び(例: [2.0, 1.0, 0.1])を、合計が1になる正の数の並び(例: [0.66, 0.24, 0.10])に変換する関数です。式は softmax(x_i) = exp(x_i) / Σ_j exp(x_j) です。exp は指数関数で、大きい入力をより大きく強調します。これにより「重みの配列」として解釈できる確率分布が得られます。

再帰型ニューラルネットワーク(RNN, recurrent neural network)。系列(単語の列など)を1ステップずつ順に処理し、内部状態を引き継ぐネットワークです。アテンションが登場する前の機械翻訳はRNNが主流でした。RNNには「長い系列の最初の方の情報が、最後まで伝わりにくい」という弱点(長距離依存の問題)があり、これがアテンション誕生の動機になりました。

## 仕組みを詳しく

現代的なアテンションは「クエリ(Query)・キー(Key)・バリュー(Value)」という3つの役割で説明されます。図書館の比喩がよく使われます。クエリは「探したい本のテーマ(検索語)」、キーは「各本に貼られた見出しラベル」、バリューは「本の中身そのもの」です。検索語と各ラベルの一致度を測り、一致度が高い本ほど中身を多く取り出して混ぜ合わせる、というのがアテンションの処理です。

具体的な計算手順は次の通りです。入力の各要素から、学習可能な重み行列を掛けてクエリ q、キー k、バリュー v の3つのベクトルを作ります。

1. スコア計算: クエリ q と各キー k の内積を取る。score_i = q · k_i。これが「q と i 番目の要素の類似度」です。
2. スケーリング: スコアをキーの次元数 d_k の平方根で割る。score_i / sqrt(d_k)。これは内積が次元数に比例して大きくなりすぎ、ソフトマックスが極端な値になるのを防ぐためです。
3. 正規化: ソフトマックスをかけて重み a_i を得る(合計1)。
4. 加重和: 重み a_i でバリュー v_i を加重平均する。出力 = Σ_i a_i v_i。

これを行列でまとめて書いたのが、Vaswani らの有名な式です。

  Attention(Q, K, V) = softmax(Q Kᵀ / sqrt(d_k)) V

ここで Q はクエリを縦に積んだ行列、K はキーの行列、V はバリューの行列、Kᵀ は K の転置(行と列を入れ替えたもの)です。Q Kᵀ で全クエリと全キーの内積を一気に計算でき、結果は「クエリ数 × キー数」のスコア行列になります。d_k はキーベクトルの次元数(例えば64)です。

テンソルの形(shape)で追ってみます。系列長を n=10、特徴次元を d=512、ヘッド次元を d_k=64 とすると、Q, K, V はそれぞれ [10, 64]。Q Kᵀ は [10, 64] × [64, 10] = [10, 10] のスコア行列。これに softmax をかけても [10, 10] のまま(各行の合計が1)。最後に [10, 10] × [10, 64] = [10, 64] が出力です。「10個の出力それぞれが、10個の入力をどう混ぜたか」を [10, 10] の重み行列が表しているわけです。

ASCII 図にするとこうなります。

```
クエリ(出力位置)      キー/バリュー(入力位置)
   q1 ──┐
   q2 ──┤   内積でスコア → softmax で重み → 加重和
   q3 ──┴──> [ 0.1 0.7 0.1 0.1 ]  ×  [v1 v2 v3 v4] = 出力
                  ↑ ここに注目
```

自己アテンション(self-attention)では、Q も K も V もすべて同じ入力系列から作ります。つまり「系列が自分自身のどこに注目するか」を計算します。一方、Bahdanau らの元祖アテンションは、出力側(デコーダ)のクエリと入力側(エンコーダ)のキー・バリューという、異なる系列の間で計算するもので、これは交差アテンション(cross-attention)と呼ばれます。

マルチヘッドアテンション(multi-head attention)は、このアテンションを複数並列に走らせる工夫です。512次元を64次元×8ヘッドに分け、各ヘッドが独立にアテンションを計算してから結合します。「あるヘッドは文法的な係り受けに、別のヘッドは意味的な関連に注目する」というように、異なる種類の関係を別々に捉えられるのが利点です。

## 手法の系譜と主要論文

アテンションの起源は機械翻訳にあります。Bahdanau, Cho, Bengio による "Neural Machine Translation by Jointly Learning to Align and Translate"(ICLR 2015, arXiv:1409.0473)が最初の本格的なアテンションです。それ以前のRNNベース翻訳は、入力文全体を1つの固定長ベクトルに圧縮してから訳していましたが、長い文ではこの圧縮で情報が失われていました。Bahdanau らは、出力単語を生成するたびに入力文の各位置への重みを計算し、関連する部分を直接参照できるようにしました。重みの計算に小さなニューラルネットワーク(加法アテンション、additive attention)を使うのが特徴です。

次に Luong, Pham, Manning の "Effective Approaches to Attention-based Neural Machine Translation"(EMNLP 2015, arXiv:1508.04025)が、内積ベースの乗法アテンション(multiplicative / dot-product attention)を提案し、計算を単純化しました。現在主流の内積アテンションの直接の祖先です。

そして Vaswani ら8名による "Attention Is All You Need"(NeurIPS 2017, arXiv:1706.03762)が決定的でした。RNNを完全に捨て、アテンションだけで系列を処理する Transformer を提案しました。スケール付き内積アテンション、マルチヘッド、位置エンコーディング(系列の順序を表す情報)を組み合わせ、並列計算が可能になったことで学習が劇的に速くなりました。

画像分野へは Dosovitskiy らの Vision Transformer(ViT, ICLR 2021, arXiv:2010.11929)が持ち込み、画像を16×16のパッチに分割して「パッチの系列」として自己アテンションを適用しました。自動運転では、複数カメラの特徴を統合する DETR(Carion ら, ECCV 2020, arXiv:2005.12872)系の検出器や、BEVFormer(Li ら, ECCV 2022, arXiv:2203.17270)のように、空間クエリでマルチカメラ特徴を集約する手法の基盤になっています。

## 論文の実験結果(定量データ)

Bahdanau ら(2015)は英仏翻訳(WMT'14)で評価しました。翻訳品質の指標は BLEU(ブルー、bilingual evaluation understudy)スコアで、機械翻訳が人間の参照訳とどれだけ一致するかを0〜100で測り、高いほど良い指標です。アテンションなしの従来RNN(RNNencdec-50)が文長50単語でおよそ BLEU 17〜26 と長文で急落したのに対し、アテンション付き(RNNsearch-50)は長い文でも品質が落ちにくく、報告では BLEU 28 前後を維持しました。「長文ほどアテンションの効果が大きい」という結果は、固定長圧縮の弱点をアテンションが解消したことを示しています。

Vaswani ら(2017)の Transformer は、WMT 2014 英独翻訳で BLEU 28.4 を達成し、当時の最高記録を2以上更新しました。英仏では BLEU 41.8。注目すべきは学習コストで、当時の競合モデルが数週間規模の学習を要したのに対し、Transformer(big)は8枚のGPUでおよそ3.5日で到達したと報告されています。base モデルは12時間程度でした。アテンションの全結合的な並列計算が、RNNの逐次計算より格段に高速だったためです。

アブレーション(ablation study、構成要素を1つずつ外して効果を測る実験)では、マルチヘッドの数を変えた比較が示されています。論文の Table 3 によると、ヘッド数1(シングルヘッド)だと最良設定より BLEU がおよそ0.9低下し、逆にヘッドを増やしすぎる(32など)と各ヘッドの次元が小さくなりすぎてやや劣化しました。8ヘッド前後がバランスの良い設定だと報告されています。

## メリット・トレードオフ・限界

メリットは大きく3つです。第一に長距離依存を直接捉えられること。系列のどんなに離れた2点でも、1回のアテンションで直接結びつけられます(RNNだと多数のステップを経由する必要があった)。第二に並列計算が可能なこと。全位置のスコアを行列演算で同時に計算でき、GPUとの相性が良い。第三に解釈性。アテンション重みを見れば「どこに注目したか」がある程度わかります(ただし重みが説明そのものだとは限らない、という批判もあります)。

最大のトレードオフは計算量とメモリです。自己アテンションは全位置対全位置のスコアを計算するため、系列長 n に対して計算量・メモリが n の2乗に比例します。n=10 なら100要素ですが、n=10000 なら1億要素のスコア行列が必要になり、長い系列や高解像度画像では現実的でなくなります。これを緩和するため、スパースアテンション(一部のペアだけ計算)、線形アテンション(近似で計算量を n に比例させる)、FlashAttention(メモリ効率の良い実装)などが研究されています。

限界としては、アテンション重みが必ずしも因果的な説明にならないこと、位置情報を明示的に与えないと順序を区別できないこと(位置エンコーディングが必須)、そして大量のデータを必要とすることが挙げられます。ViT は ImageNet 規模だけだと畳み込みネットに負けることがあり、より大規模なデータでの事前学習が前提になりがちです。

## 発展トピック・研究の最前線

効率化が最大の研究テーマです。FlashAttention(Dao ら, 2022)はGPUのメモリ階層を活用してアテンションを高速・省メモリに計算し、長系列の学習を現実的にしました。Linformer や Performer は計算量を n に線形化する近似手法です。自動運転では、空間や時間に対してクエリを定義する「変形可能アテンション(deformable attention)」が、全位置ではなく数点のサンプル点だけを参照することで高解像度のマルチカメラ・時系列特徴を効率的に融合しています(BEVFormer がその代表)。また、アテンション重みが本当に「説明」になっているかという解釈性の議論、長系列での性能劣化(lost in the middle)など、未解決の課題も残っています。
