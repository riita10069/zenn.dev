# 混合ガウス NLL 損失 (GMM)

## ひとことで言うと

将来の動き (軌跡) を「複数の山を持つ確率分布」として表し、その分布のもとで正解の軌跡がどれだけ「出やすい」かを最大化する損失です。山が複数あるとは「直進する未来」「右折する未来」のように、ありうる未来が1つに定まらないことを表します。各山をガウス分布 (釣鐘型の確率分布) で表し、それらの重ね合わせ (混合ガウス) で全体を作ります。学習では「正解が出にくいこと=負の対数尤度 (NLL)」を罰として与え、これを最小化することで「正解がいずれかの山から高い確率で出る」ように分布を整えます。多峰 (山が複数) な未来を、モードごとに明示的に表現できるのが最大の特徴です。

## 直感的な理解

交差点に近づいている前方の車を想像してください。その車が次の3秒で「直進する」のか「右に曲がる」のか、外から完全には分かりません。両方とも十分にありえます。このとき「平均的な軌跡を1本だけ予測する」と何が起きるでしょうか。直進と右折の平均は「斜め前にゆっくり進む」という、現実にはありえない軌跡になってしまいます。これを平均化のワナ (regression to the mean) と呼びます。正解が複数あるのに無理に1つにまとめると、どの正解にも当てはまらない不自然な答えが出るのです。これは単なる比喩ではなく、二乗誤差 (L2 損失) を最小化する点推定が、条件付き分布の期待値 E[y|x] を返すという数学的帰結です。条件付き分布が二峰なら、その期待値は「2つの山の谷間」という、確率が低い領域に落ちます。

混合ガウス NLL はこの問題を真正面から扱います。「直進」の山と「右折」の山を別々に用意し、それぞれに「直進70%・右折30%」のような確率を割り当てます。予測は1本ではなく、確率付きの複数候補になります。これは下流の経路計画 (プランナ) にとって非常に都合がよい形です。プランナは「相手が70%で直進、30%で右折」という情報を受け取り、両方に備えた安全な行動を選べます。1本の平均軌跡しか渡されなかったら、こうした備えはできません。

なぜわざわざ確率分布の形にするのか。それは「予測の不確かさ」を数値として下流に伝えられるからです。自動運転では「自信がある予測」と「自信がない予測」を区別することが安全に直結します。分布として出せば、山の確率 (どのモードがありそうか=認識論的・意図の不確かさ) と山の広がり (そのモード内でのばらつき=測定や微小な揺らぎの不確かさ) の両方を定量的に渡せます。前者を epistemic / multimodal uncertainty、後者を aleatoric / within-mode uncertainty と呼んで区別することがあります。

## 基礎: 前提となる概念

いくつかの言葉を、知らない前提で噛み砕きます。

- ガウス分布 (正規分布): 平均のまわりに左右対称の釣鐘型に広がる確率分布です。「だいたいこの位置、ばらつきはこのくらい」を、平均 (中心) と分散 (広がりの大きさ) という2つの数で表します。1次元なら平均 μ と分散 σ²、多次元 (位置は x,y の2次元) なら平均ベクトル μ と共分散行列 Σ で表します。共分散行列とは、各次元のばらつきと次元間の相関 (x と y が連動してずれるか) をまとめた行列です。2次元なら 2×2 の行列で、対角成分が x・y それぞれの分散、非対角成分が両者の相関を表します。

- 混合ガウス (Gaussian Mixture Model, GMM): 複数のガウス分布を重み付きで足し合わせたものです。1つのガウスは山が1つしか作れませんが、足し合わせれば山が複数ある分布を表せます。各山の「混ぜる割合」を混合重みと呼び、全部足すと1になります。

- 尤度 (likelihood): 「あるモデルのもとで、実際に観測されたデータ (正解) がどれくらい出やすいか」を表す値です。尤度が大きいほど、モデルは正解をうまく説明できています。逆に「正解なのにモデルがほとんど確率を割り当てていない」と尤度は小さくなります。連続分布では尤度は確率そのものではなく確率密度ですが、「正解の出やすさ」という直感は変わりません。

- 対数尤度と NLL (Negative Log-Likelihood, 負の対数尤度): 尤度の対数をとり、マイナスをつけたものが NLL です。尤度を最大化することと NLL を最小化することは同じです。なぜ対数を使うのか。第一に、確率の掛け算 (複数時刻にわたる尤度の積など) を足し算に変えられ、数値的に安定するからです。第二に、最適化では「最小化」の形に揃えるのが扱いやすいからです。とても小さい確率 (例: 0.0001) はそのままだと桁あふれしますが、対数をとれば -9.2 のような扱いやすい値になります。混合の対数尤度は log-sum-exp という計算で安定化させるのが定石で、これは「各モードの対数を計算してから、最大値を引いて exp し、足してから log を取り、最大値を足し戻す」操作です。これで exp のオーバーフロー/アンダーフローを防げます。

- 多峰性 (multimodality): 「ありうる答えが複数ある」性質です。運転の未来予測は本質的に多峰です。同じ状況でも、ドライバーの意図や周囲の状況で結果が変わります。

## 仕組みを詳しく

将来の軌跡を表す変数を y (将来 T ステップ分の位置の系列)、観測 (入力: カメラ画像・地図・周囲エージェントの状態など) を x とします。これを K 個のガウスの混合で表します。

```
p(y | x) = Σ_{k=1..K}  π_k(x) · N( y ; μ_k(x), Σ_k(x) )
```

各記号の意味:

- K: モードの数 (例: 候補軌跡を6本出すなら K=6)。事前に決め打ちします。
- π_k(x): k 番目のモードの混合重み (そのモードが起きる確率)。全モードで足すと1になるよう、ネットワーク出力に softmax をかけて作るのが普通です。
- μ_k(x): k 番目のモードの平均軌跡。これが「そのモードでの予測軌跡そのもの」です。
- Σ_k(x): k 番目のモードの共分散。そのモード内での予測のばらつき (不確かさ) を表します。

ネットワークは入力 x を受け取り、各モードについて (π_k, μ_k, Σ_k) の3点セットを出力します。tensor shape の例を出します。バッチサイズ B、モード数 K=6、将来ステップ T=12、位置は2次元 (x,y) とすると、

```
μ : [B, 6, 12, 2]   各モードの平均軌跡
π : [B, 6]          各モードの確率 (softmax 後、行ごとに合計1)
Σ : [B, 6, 12, 2, 2] 各時刻の2x2共分散 (簡略化して対角や等方にすることも多い)
```

共分散の出力には工夫が要ります。分散は必ず正でなければならないので、ネットワークは生の値を出し、softplus や exp を通して正にします。あるいは標準偏差 σ を出して二乗します。2×2 のフル共分散を学ぶ場合はコレスキー分解 (Σ = LLᵀ の下三角行列 L を予測) を使い、対角成分を正に制約します。簡略化のため Σ を対角行列 (x と y のばらつきは独立) や等方 (σ²I、円形のばらつき) に固定する実装も多く、その場合は数値が安定しやすい代わりに細長い不確かさの形 (例: 進行方向に長い楕円) を表せません。

損失は、この分布のもとでの正解軌跡 y* の NLL です。

```
L = - log( Σ_{k=1..K}  π_k · N( y* ; μ_k, Σ_k ) )
```

これを最小化するとは「正解軌跡 y* が、いずれかのモードから高い確率で出るように分布を動かす」ことです。1つのガウスの対数密度は、定数項を除けば「マハラノビス距離 (共分散で重み付けした距離) のマイナス半分」と「分散の対数」の和です。つまり NLL は、距離を縮める力 (回帰) と、共分散を適切な大きさにする力 (不確かさの較正) を自動でバランスさせます。予測が外れているとき、分散を大きくすれば一時的に NLL は下がりますが、log Σ の項がそれを罰するので、無闇に分散を膨らませることはできません。この「正確なら分散を小さく、不確かなら大きく」という自己調整が NLL の美点です。

### モード崩壊という最大の落とし穴

ここに実装上の大問題があります。上の NLL を素朴に最適化すると、全モードが同じ正解に張り付いてしまい、せっかくの多峰性が消える モード崩壊 (mode collapse) が起きます。直感的には、各モードが「自分も正解に近づけば損失が下がる」と判断し、全員が同じ正解めがけて寄っていくためです。結果として6本の候補がほぼ同じ軌跡になり、多峰予測の意味がなくなります。学習動力学の観点では、softmax で重みを共有する全モードに勾配が分配されるため、初期にわずかでも正解に近いモードが優遇され、そのモードに重みが集中していく正のフィードバックが働くのが原因です。

これを防ぐ標準テクニックが winner-takes-all (勝者総取り, WTA) / hard assignment です。手順は次の通りです。

1. まず各モードの平均 μ_k と正解 y* の距離を測り、最も近いモード k* を選ぶ (このモードが「勝者」)。
2. 回帰損失 (μ_k* を y* に近づける) は、勝者モード k* だけ に効かせる。他のモードはこの正解では更新しない。
3. 分類損失 (どのモードが選ばれたか) で π を学習し、「この状況では k* が正解になりやすい」と覚えさせる。これは交差エントロピーで、正解ラベルは「勝者 k*」です。

こうすると各モードが別々の役割 (一方は直進担当、他方は右折担当) に分化し、多峰性が保たれます。回帰と分類を足し合わせた損失 (回帰は勝者のみ、分類は softmax 全体) は、EM アルゴリズムのハード版 (hard-EM) と数学的に対応します。E ステップが「勝者の割り当て」、M ステップが「割り当てに基づくパラメータ更新」に相当します。

数値直感を簡略化して示します。K=2、正解 y*=「右折」だとします。モード1の平均が「直進」、モード2の平均が「右折寄り」なら、勝者はモード2です。回帰損失はモード2を右折にさらに近づけ、分類損失は「この状況の正解はモード2」と π を更新します。モード1はこの場面では更新されませんが、別の場面 (直進が正解の場面) で勝者になり、直進担当として育ちます。役割分担が自然に生まれるわけです。

### アンカーによる安定化

WTA は強力ですが、初期段階では「どのモードがどの役割か」が不安定で、割り当てがぶれやすい欠点があります。これを抑えるのがアンカー (anchor) という考え方です。あらかじめ訓練データの軌跡を集めてクラスタリング (k-means など) し、「典型的な軌跡の雛形」を K 個用意します (例: 直進・左折・右折・停止…)。ネットワークは各アンカーに対する確率と、アンカーからの残差 (ズレ) だけを予測します。アンカーが「モードの種」を固定するので、割り当てが安定し、学習が崩れにくくなります。割り当ても「正解に最も近いアンカー」で決まるため、初期から一貫します。残差を学ぶことで、アンカーが粗くても最終的な軌跡は滑らかに正解に合わせられます。

アンカーをさらに進めると、軌跡の集合そのものを固定の集合 (trajectory set) とみなし、回帰すらやめて分類問題に帰着させる発想 (CoverNet) もあります。これは「数千本の候補軌跡から正解に最も近い1本を当てる」多クラス分類で、回帰の不安定さを完全に回避する代わりに、軌跡集合の解像度に精度が縛られます。

## 手法の系譜と主要論文

混合ガウス NLL は、ニューラルネットの確率出力という古い思想から、現代の軌跡予測の主力へと進化してきました。

- Mixture Density Networks (Bishop, 1994, "Mixture Density Networks", Aston University 技報)。提案: ニューラルネットの出力を「ガウス混合のパラメータ (π, μ, σ)」と解釈し、NLL で学習する枠組み (MDN) を導入。動機: 1対多の写像 (1つの入力に複数の妥当な出力がある問題、たとえば逆運動学) は、単一値を出す通常の回帰では平均化のワナに陥り扱えないため。新規性: ネットワーク出力を確率分布のパラメータとみなす、という今日まで続く発想を確立した。これが軌跡予測の GMM 損失の源流です。

- MTP — Multimodal Trajectory Predictions (Cui, Radosavljevic, Chou ほか, Uber ATG, ICRA 2019, arXiv:1809.10732)。提案: 周囲車両の将来を K 本のモード + 各モードの確率として CNN で予測し、winner-takes-all で学習。動機: 運転シーンの将来は本質的に多峰で、平均をとると非現実的な軌跡になるため。新規性: WTA を運転の軌跡予測に持ち込み、多峰予測を安定に学習する実用レシピを示した。限界: K を事前に決め打ちする必要があり、WTA は勝者以外を更新しないので学習信号が疎になりがち。

- MultiPath (Chai, Sapp, Bansal, Anguelov, Waymo, CoRL 2019, arXiv:1910.05449)。提案: データから作った軌跡アンカーを用意し、各アンカーに対する確率と残差 (アンカーからのズレ) を予測。動機: モードを完全に自由学習させると崩壊・不安定になるので、アンカーでモードの種を固定して安定させるため。新規性: アンカーベースという、後の軌跡予測の定番パターンを確立した。限界: アンカー集合の質に性能が依存し、アンカーにない挙動は表現しにくい。

- CoverNet (Phan-Minh, Grigore, Boulton ほか, nuTonomy/Motional, CVPR 2020, arXiv:1911.10298)。提案: 軌跡予測を「固定された軌跡集合からの分類」に置き換える。動機: 回帰の数値不安定さとモード崩壊を、分類化で根本的に避けるため。新規性: ダイナミクス制約を満たす軌跡集合を作り、被覆 (coverage) という基準で集合サイズを決める考え方を導入。限界: 集合の解像度が精度の上限を決める。

- Trajectron++ (Salzmann, Ivanovic, Chakravarty, Pavone, Stanford, ECCV 2020, arXiv:2001.03093)。提案: CVAE (条件付き変分オートエンコーダ) の潜在変数を離散化し、GMM の出力ヘッドと組み合わせて多峰軌跡を生成。動機: 潜在変数モデルで多峰性を扱いつつ、出力を明示的な分布として較正したいため。新規性: 離散潜在 + GMM 出力という組み合わせで、確率的かつ解釈可能な予測を実現。

- MultiPath++ (Varadarajan ほか, Waymo, ICRA 2022, arXiv:2111.14973)。提案: 入力をラスター画像でなくポリライン (折れ線) などの構造化表現で扱い、出力 GMM の予測を改良。動機: ラスター化で失われる幾何情報を活かし、より正確な多峰予測を得るため。新規性: 学習可能なアンカー埋め込みと、出力 GMM の表現を洗練。Waymo Motion ベンチで当時の上位。

- MTR — Motion Transformer (Shi, Jiang, Dai, Schiele, MPI/CUHK, NeurIPS 2022, arXiv:2209.13508)。提案: Transformer の query (DETR 系の発想) を「意図ごとの問い合わせベクトル」として使い、固定アンカーへの依存を緩めつつ GMM を出力。動機: アンカーの質への依存を減らし、長距離予測の精度を上げるため。新規性: query ベースで多峰予測を扱う流れを軌跡予測に確立し、Waymo Motion Prediction ベンチで上位を獲得。MTR++ や後続の query ベース手法群へ続く。

系譜をまとめると、MDN (確率出力の発想) → MTP (WTA で運転に適用) → MultiPath / CoverNet (アンカー・分類化で安定化) → Trajectron++ (潜在変数 + GMM) → MultiPath++ / MTR (構造化入力・query で精緻化) という流れです。一貫して「多峰な未来をどう安定に学習するか」が課題でした。

## 論文の実験結果 (定量データ)

軌跡予測の評価で最も使われる指標を、まず意味から説明します。

- ADE (Average Displacement Error, 平均変位誤差): 予測軌跡と正解軌跡の、各時刻での位置の差 (距離) を全時刻で平均した値。単位はメートル。小さいほど良い。
- FDE (Final Displacement Error, 最終変位誤差): 予測の終端 (例: 3秒後や6秒後) の位置と正解終端の距離。長期予測の正確さを測る。
- minADE_K / minFDE_K: K 本の候補のうち、正解に最も近い1本で測った ADE/FDE。多峰予測の評価ではこれが標準です。「複数候補のうち1本でも正解に近ければよい」という考え方を反映します。
- Miss Rate (MR): 終端誤差が一定のしきい値 (例: 2m) を超えた割合。低いほど良い。
- mAP (mean Average Precision): Waymo Motion などで使う、確率較正まで含めて評価する指標。「正しいモードに高い確率を割り当てているか」を測る。minADE が良くても確率がでたらめだと mAP は上がりません。

なぜ minADE/minFDE を使うのか。多峰予測では「直進と右折の両方を出した」場合、正解が右折なら右折候補で評価するのが妥当です。全候補を平均で評価すると、正しく多峰を出しているモデルがかえって不利になるためです。ただし minADE だけを追うと「とにかく候補をばらまけば1本は当たる」という抜け道があるため、mAP や Miss Rate で確率較正も同時に見るのが今の標準です。

定量例 (報告による、おおよその値):

- MTP は当時の社内・公開データで、単峰 (1本だけ) 予測に比べ多峰予測が終端誤差を明確に下げることを示しました。WTA ありとなしの比較 (アブレーション) では、WTA を外すとモード崩壊が起き、複数候補がほぼ重複して minFDE が悪化することが報告されています。これは「WTA がモードの役割分担を生む」ことの直接の証拠です。

- MultiPath は nuScenes / Argoverse 系のベンチで、アンカーなしの自由学習に比べて多峰予測が安定し、minADE/minFDE が改善することを示しました。アブレーションでは、アンカー数 K を増やすと minADE は下がる (候補が増えれば正解に近い1本を引きやすい) が、確率較正 (どのモードがありそうかの当て方) は K が大きすぎると悪化する、というトレードオフが見られます。

- CoverNet は nuScenes 予測タスクで、回帰ベース手法と競合する精度を、分類化によって安定に得られることを示しました。軌跡集合の被覆率を上げる (より細かい集合にする) と minADE が改善する一方、集合が巨大になると計算コストと確率較正が悪化する関係が報告されています。

- nuScenes prediction の代表的な数値感覚として、minADE_5 がおよそ 1.5〜2.0m、minADE_10 がおよそ 1.0〜1.5m、minFDE_1 (最尤の1本) が 7〜9m 程度というのが各手法のオーダーです (年代・手法で変動)。候補数 K を 1→5→10 と増やすと minADE が単調に下がるのは、上記の「1本でも当たればよい」という指標特性によります。

- MTR は Waymo Open Motion Dataset で、minADE・minFDE・Miss Rate・mAP の複数指標で当時の上位 (リーダーボード首位級) を記録しました。報告では minADE がおよそ 0.6m 前後、minFDE がおよそ 1.2m 前後の水準まで下がり、mAP も従来手法を上回って、query ベースが固定アンカー手法を上回ることを示しました。アブレーションでは、意図 query (motion query) を外すと多峰の表現力が落ち、Miss Rate が上がること、また dense future prediction の補助損失を加えると収束が速まり精度も向上することが示されています。

これらに共通するのは、WTA / アンカー / query のいずれかでモードの種を分けないと、多峰予測は崩壊して minFDE が悪化するという点です。逆に言えば、これらの工夫こそが GMM 損失を実用にする鍵です。また minADE と mAP は同時に良くするのが難しく、候補をばらまく戦略は minADE を良くしても mAP を悪化させる、というトレードオフが繰り返し観測されています。

## メリット・トレードオフ・限界

メリット
- モードが明示的。1回の推論で「候補軌跡 + それぞれの確率」が全部出るので、下流のプランナに渡しやすい。サンプリング型 (拡散・フロー) は全モードを見るのに複数サンプルが要るのと対照的です。推論が1回 (フィードフォワード) で済むためレイテンシが小さく、実時間制約の厳しい自動運転に向きます。
- 不確かさを分布 (混合重みと共分散) として定量化でき、安全判断に使える。
- 確率モデルとして解釈しやすく、評価指標 (尤度) も自然に定義できる。NLL は較正された不確かさを自動で促す。

トレードオフ・限界
- モード崩壊が起きやすく、WTA やアンカーなどの工夫がほぼ必須。素朴な NLL 最小化だけでは破綻します。
- モード数 K を事前に決め打ちする必要がある。多すぎると確率較正が崩れ、少なすぎると現実の多様性を捉えきれません。シーンによって妥当なモード数は変わる (直線道路では1つ、複雑な交差点では多数) のに、K は固定です。
- 各モードがガウス (単純な釣鐘型) という仮定。鋭く折れ曲がる軌跡や、複雑な形の分布は近似しきれません。時刻間の相関 (軌跡は滑らかに繋がる) を独立ガウスの積で近似すると、ギザギザのサンプルが出やすい。
- WTA は勝者以外を更新しないため学習信号が疎で、初期の割り当てが不安定になりがち。アンカーはこれを緩和しますが、アンカー集合の質に縛られます。
- minADE を下げるだけなら候補をばらまけばよいが、それは確率較正 (mAP) を犠牲にする。指標間のトレードオフが本質的に存在する。
- 未解決課題: モード数を入力に応じて動的に変える、確率較正の改善 (出した確率が現実の頻度と一致するか)、シーン整合性 (複数エージェントの予測が物理的に矛盾しないか=衝突する2軌跡を同時に高確率にしない)、ガウス仮定を超えた表現 (正規化フローや拡散との統合) などが研究の前線です。

## 発展トピック・研究の最前線

GMM 損失の発想は、より柔軟な確率モデルへ拡張されています。1つの方向は正規化フロー (normalizing flow) や拡散モデルとの統合で、ガウスの釣鐘型という制約を外し、任意形状の分布を学ぶ試みです。拡散ベースの軌跡予測 (MotionDiffuser など) は、明示的な GMM パラメータを出さずにサンプルで分布を表現し、より複雑な多峰性を捉えますが、複数サンプルが必要で推論が重いという逆のトレードオフを持ちます。もう1つは query ベースの多峰予測 (MTR 系) の深化で、固定数のアンカーに縛られず、シーンに応じて意図候補を生成する方向です。

また「確率較正」を明示的に評価・改善する研究 (出した確率が実際の発生頻度と一致するか) も重要さを増しています。安全に直結する自動運転では、minADE が小さいだけでなく「自信の度合いが信頼できる」ことが求められるためです。さらに、複数エージェントを同時に予測する joint prediction (シーン全体の整合した未来を1つの分布として表す) では、各エージェント独立の GMM では矛盾する未来 (互いに衝突する高確率軌跡) を許してしまう問題があり、シーンレベルのモード (scene-centric な混合) を扱う研究が進んでいます。エンドツーエンド自動運転では、この多峰予測を中間表現として下流のプランナに渡し、予測と計画を一体で学習する流れも主流になりつつあります。
