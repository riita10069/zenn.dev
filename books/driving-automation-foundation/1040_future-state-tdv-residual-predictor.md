# 未来状態モジュールの設定可能ホライズンと残差(TDV)予測器

## ひとことで言うと

未来状態モジュール(FutureState)は「数秒後にシーンの特徴がどうなるか」を予測する部品です。予測する未来の時刻(ホライズン)の本数を設定で変えられるようにし、さらに「未来をゼロから予測する」のではなく「現在からの変化分(残差)だけを予測する」残差予測器(時間差分=TDV 型)を任意で持たせる設計です。残差予測にすると学習が安定し、収束が速くなりやすいという利点があります。

## 直感的な理解

未来予測モジュールは、現在の融合特徴(複数カメラを統合した BEV 特徴)と、計画器の内部状態(これから走る意図の要約)を受け取り、未来の視覚特徴を予測します。ここに2つの設計上の課題があります。

課題1は「ホライズンが固定だと柔軟性に欠ける」こと。予測する未来の本数・間隔は、タスクや計算予算によって変えたいものです。市街地ではより短く密に(直近を細かく)、高速道路では長く先まで、といった調整がしたい。固定だとコードを書き換えるしかありません。そこで設定可能なホライズン(configurable horizons)が要ります。

課題2は「未来をゼロから予測するのは難しい」こと。未来の特徴を絶対値としてまるごと予測するのはネットワークにとって難題です。なぜなら未来は現在とよく似ているからです。1.6 秒後のシーンは今のシーンとほとんど同じで、変わるのは「動いた車の位置」「近づいた信号」など一部だけ。それなのに毎回フルの特徴を一から組み立てさせるのは無駄が多く、学習も不安定になります。

ここで「残差(residual、差分)を予測する」考えが効きます。「未来 = 現在 + 変化分」と分解し、ネットワークには変化分だけを予測させる。現在をベースに渡しておけば、ネットワークは「何が変わったか」だけに集中でき、予測が容易で安定します。これは ResNet(He et al., 2015)が「恒等写像からの差分を学ぶと最適化が楽になる」ことを示したのと同じ原理です。

## 基礎: 前提となる概念

- 融合特徴(fused features): 複数カメラの特徴を統合した1つの特徴マップ。BEV 形式なら `[B, C, H, W]`。
- 計画器の内部状態 / ego hidden: 軌道を生成する計画器(しばしば RNN や GRU)が持つ隠れ状態ベクトル。「これから自車がどう動くつもりか」を要約した内部表現。
- ホライズン(horizon): 何秒先までを、何本のステップで予測するか。例「+1.6s, +3.2s, +4.8s, +6.4s の4本」。
- 残差予測(residual prediction): 出力を「ベース + 差分」に分解し、差分だけをネットワークに学ばせる方式。
- 行動条件付け(action conditioning): 「自車がどの行動(軌道)を取るか」を予測の入力に加えること。これにより「行動を変えたら未来がどう変わるか」を問える。
- 反実仮想(counterfactual): 「もし別の行動を取っていたら」を仮想的に評価すること。行動条件付き予測で可能になる。
- 恒等写像(identity mapping): 入力をそのまま出力する写像。残差がゼロのとき「未来=現在」になる、という基準点。
- TD(temporal difference、時間差分): 強化学習で「現時刻の推定」と「次時刻の推定」の差分を学習信号に使う考え方(Sutton 1988)。時間方向の差分から学ぶ、という発想の源流の一つ。

## 仕組みを詳しく

### 基本の未来状態 forward

典型的な forward は次の通りです。

1. 計画器の内部状態(例 256 次元)を線形射影し、BEV 特徴のチャンネル方向のバイアスとして全空間に足し込む。「これから走る意図」を未来予測に注入する操作です。

   ```python
   bias = self.ego_proj(ego_hidden).view(-1, embed_dim, 1, 1)   # [B, 256, 1, 1]
   conditioned = fused_features + bias                          # [B, 256, H, W]
   ```

2. 任意で行動条件付け: 計画した軌道 `[B, action_dim]` を射影して第2のバイアスとして足す。これで「軌道Aを走ったら/Bを走ったら、シーンはどう変わるか」という反実仮想ロールアウトが可能になります。

3. 数層の畳み込み(例 `embed_dim → 2·embed_dim → 4·embed_dim`)で `4C` チャンネルを生成し、4分割して未来4ステップ分の特徴 `[B, C, H, W] × 4` を返す。本数 K は設定値で、4 は一例です。

この予測出力を [I-JEPA / V-JEPA(自己教師あり予測表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1074_i-jepa-v-jepa) 流の特徴予測損失につなぐと補助タスクとして機能し、前段の共有表現を鍛えます。さらに、予測誤差そのものを「モデルの不確かさ・想定外検知(out-of-distribution detection)」の信号として使える、という副次的な利点もあります。予測が大きく外れる状況=モデルが見たことのない状況、という解釈です。

### 拡張1: 設定可能なホライズン

予測する未来本数 K を引数化し、履歴長や計算予算に合わせて可変にします。低速ワールドモデル枝([JEPA 系の低速ワールドモデル分岐(予測表現を補助タスクにする設計)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1041_jepa-world-model-branch))と組み合わせる場合、履歴長 N と未来本数 K を揃える(例 N=K=4、1Hz サンプリング)設計が自然です。ホライズンを密にすると直近の力学を細かく学べる反面、出力テンソルが本数に比例して増えるため、空間圧縮([予測空間の圧縮(forecast space を切り替える調整ノブ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1036_feature-compressor-forecast-space))とのバランスが要ります。

### 拡張2: 残差(TDV)予測器

「未来 = 現在 + 予測残差」の形で予測する任意モジュールです。ネットワークは絶対的な未来特徴ではなく、現在からの差分だけを出力すればよくなります。

```
従来:    future_t = Predictor(conditioned)                    # 絶対値を直接予測
残差:    future_t = current_feature + Residual(conditioned)   # 差分だけ予測
```

現在の特徴がベースラインとして渡るので、ネットワークの仕事は「何がどう動いたか」に絞られます。学習初期、残差器の重みが小さな乱数で初期化されていれば残差はほぼゼロ、つまり「未来 ≈ 現在」という妥当な初期予測から始まります。1.6 秒後はほぼ今と同じ、という強い事前知識(prior)を構造に組み込めるわけで、これが学習安定化の本質です。残差器の最終層をゼロ初期化すると、初期予測が厳密に「未来=現在」になり、さらに安定します(LayerScale や ResNet の zero-init residual と同じテク)。

「TDV」は時間差分(temporal difference)に基づく予測という意味合いで使われる呼称です。残差・時間差分に基づく予測という考え自体は、(a) ResNet の残差学習、(b) 強化学習の TD 学習(現在の価値推定と次時刻の推定の差を使う、Sutton 1988)、(c) ビデオ予測における残差フレーム予測(現在フレーム+動き/残差)、という複数の確立した系譜に連なります。特定の単一一次文献に厳密に固定された用語ではなく、「現在を基準に差分を予測する残差予測の一般的枠組み」を指す呼び名として扱うのが安全です。

## 手法の系譜と主要論文

- He et al., "Deep Residual Learning for Image Recognition (ResNet)" (CVPR 2016, arXiv:1512.03385): 層の出力を「入力 + 残差」で学ぶ残差接続を提案。恒等写像からの差分を学ぶ方が深いネットの最適化が容易、という発見。残差予測の「差分を学ぶと楽」という発想の土台。
- Sutton, "Learning to Predict by the Methods of Temporal Differences" (Machine Learning, 1988): 時間差分(TD)学習を定式化。時間方向の推定差から学ぶという発想の古典的源流。
- Finn et al., "Unsupervised Learning for Physical Interaction through Video Prediction" (NeurIPS 2016, arXiv:1605.07157): 行動を条件にして次フレームを予測する action-conditioned video prediction を提示。未来を「現在ピクセルの変換(warp)」として予測する残差的発想と、行動条件付けの源流。
- Assran et al., "I-JEPA" (CVPR 2023, arXiv:2301.08243): 特徴空間で予測すべきという原理。未来状態の出力を特徴予測損失で活性化する根拠。
- Hu et al., "MILE" (NeurIPS 2022, arXiv:2210.07729): 自動運転で潜在ワールドモデルを学び、未来状態の予測が運転性能を支えることを示す。
- Hu et al., "UniAD" (CVPR 2023, arXiv:2212.10156): 動き予測など上流の補助タスクが計画性能を上げることを示す。未来状態を補助タスク化する裏付け。
- Bardes et al., "V-JEPA 2" (2025, arXiv:2506.09985): 行動条件付き潜在予測でロボット制御を実現。行動条件付き未来状態の最先端。

## 論文の実験結果(定量データ)

- 残差学習の効果(ResNet): ImageNet 分類で、残差接続を使わない素の深層ネットは層を深くする(34 層など)と訓練誤差すら悪化する「劣化問題(degradation problem)」が起きます。残差接続を入れた ResNet では 152 層まで深くしても劣化せず、ImageNet で当時最高水準の Top-5 誤差(約 3.57%、アンサンブル時)を達成しました。ここで重要なのは「残差にすると最適化が容易になる」という点で、これがそのまま「未来=現在+残差」にすると学習が安定する根拠になります。残差ブロックを通常の積層に戻すアブレーションで劣化が再現することが、残差の寄与を定量的に示しています。
- 行動条件付き予測の効果(Finn et al.): ロボットアームの押し動作データで、行動を条件にして次フレームを予測すると、行動を与えない予測より定量的に予測誤差(画素レベルの予測尤度)が改善することが報告されています。これは「行動を入れると未来がより確定的になり予測しやすくなる」ことの裏付けで、未来状態モジュールの行動条件付けによる反実仮想ロールアウトの土台です。
- 運転での未来状態予測の効果(MILE / UniAD): [JEPA 系の低速ワールドモデル分岐(予測表現を補助タスクにする設計)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1041_jepa-world-model-branch) に詳述。MILE は CARLA の Driving Score で潜在世界モデルの優位を、UniAD は nuScenes の計画 L2 誤差と衝突率で「上流補助タスクを抜くと計画が悪化する」アブレーションを示しています。未来状態を補助タスクとして接続する設計は、これらの定量結果に立脚します。
- 数値安定性に関する注意(一般論): 未来特徴予測を確率的・ランダムな入力でテストすると、損失が発散したり NaN になることがあります。これは設計の欠陥というより、特徴空間予測が「意味のある入力分布」を前提にしているためで、実データでの数値安定性(損失の正規化・勾配クリッピング・mixed precision の扱い)は別途検証が必要だ、という一般的な留意点です。残差予測はこの点でも有利で、出力が現在を基準にした小さな差分なので絶対値予測より発散しにくい傾向があります。

## メリット・トレードオフ・限界

メリット:
- 設定可能ホライズンで、タスク・計算予算に応じて予測の本数と間隔を柔軟に変えられる。
- 残差予測は「現在からの差分」だけを学ぶので、学習が安定し収束が速くなりやすい(ResNet と同じ原理)。ゼロ初期化と組み合わせると初期予測が「未来=現在」で始まり、さらに安定。
- 行動条件付けにより反実仮想ロールアウト(軌道ごとの未来比較)が可能で、計画と予測が連携する。MPC 的な「候補軌道を未来でロールアウトして安全なものを選ぶ」計画への道が開く。
- 活性化すれば前段の共有表現を鍛える補助信号になり、予測誤差を想定外検知に転用できる。

トレードオフ・限界:
- 高解像度 BEV では `4C` 出力が巨大(1サンプル数百MB at fp32)になり、mixed precision や空間圧縮([予測空間の圧縮(forecast space を切り替える調整ノブ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1036_feature-compressor-forecast-space))が事実上の前提になる。
- 残差予測は「現在を渡す配線」と「現在特徴との整合」が要り、実装が一段複雑になる。現在特徴のスケールと残差のスケールが合わないと加算で破綻しうる。
- 予測出力を損失に接続し、実データの未来フレームを供給する仕組みが整って初めて効果を検証できる。整備前は出力を計算しても損失に寄与しない。
- 「TDV」という呼称は複数の系譜(残差・TD 学習・残差フレーム予測)が混在し、特定の一次文献に厳密に固定されているわけではない点に注意。
- 残差は「未来が現在に近い」前提に依存する。急なシーン変化(交差点進入・割り込み)では残差が大きくなり、残差予測の優位が薄れる。

## 発展トピック・研究の最前線

- 複数ホライズンの整合性: 短いホライズンの予測を長いホライズンの予測に再帰的に積み上げる(roll-out)と誤差が累積します。各ホライズンを独立に予測するか、再帰するかは long-horizon 予測の中心的な設計選択で、教師強制(teacher forcing)とスケジュールサンプリングのトレードオフが関わります。再帰だと誤差が雪だるま式に増え、独立予測だと時間的整合(連続性)が弱くなる、という両刃。
- 行動条件付き世界モデルとしての発展: 行動を条件にした未来予測は、モデル予測制御(MPC)や計画と直結します。V-JEPA 2 のような行動条件付き潜在予測は、運転で「候補軌道ごとに未来をロールアウトし、最も安全な軌道を選ぶ」計画器への道を開きます。
- 不確かさの定量化: 残差予測の誤差分布をモデル化(例 予測分散も出力する、あるいはアンサンブルで分散を測る)すれば、想定外検知だけでなく、安全余裕の動的調整(不確かなときは保守的に運転)に使えます。
- 確率的未来予測: 未来は本質的に多峰(複数の起こりうる展開)です。決定論的な残差予測は平均的な未来に潰れがちで、生成系(拡散・VAE)や潜在変数で複数仮説を表現する方向が、割り込みや歩行者の意図のような不確実な未来を扱う研究の最前線です。
