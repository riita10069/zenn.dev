# PolarFormer (極座標BEV)

## ひとことで言うと
複数のカメラ画像から「真上から見下ろした視点(BEV: Bird's-Eye-View、鳥瞰図)」の特徴マップを作るとき、ほとんどの手法は碁盤の目のような直交格子(縦横に等間隔で区切ったマス)を使います。PolarFormer は、その格子を「自車中心からの距離 r」と「方位角 θ」で区切る極座標(polar coordinate)の格子に置き換えた手法です。カメラはレンズの中心という1点から世界を放射状(扇形)に見ているので、放射状に分割する極座標のほうがカメラの見え方と素直に一致する、という単純で強い発想に立っています。

## 直感的な理解
カメラで前方を撮ると、近くの物は大きく、遠くの物は小さく写ります。これは「奥行き(距離)」が画像の中に圧縮されているからです。さらに、画像を横方向(左から右へ)に走査すると、左端は左斜め前、中央は正面、右端は右斜め前というように「方位(角度)」が連続的に変わっていきます。つまり1枚のカメラ画像は本質的に「方位角 × 奥行き」という極座標的な構造を持っています。透視投影(perspective projection、3D の点を 1 点から見た平面に写す写像)の幾何そのものが、画像の列(縦の帯)を一定の方位角に、画像の行を一定の仰角に対応させるからです。

ところが BEV を直交格子(例: 200×200 のマス、各マスが 0.5m 四方)で表すと、この極座標構造を無理やり縦横の正方マスに押し込むことになります。カメラのある画素が見ている「1本の光線(視線、ray)」は、直交格子の上では斜めに走り、近くでは多数のマスを横切り、遠くではまばらにしか通りません。この食い違いが、特に遠距離や視野の端で「どのマスにこの画素の特徴を置くべきか」をあいまいにし、特徴をぼやけさせます。PolarFormer は「だったら BEV も角度と距離で区切ればよい」と考え、カメラ画像の構造と BEV の格子構造をそろえました。身近に言えば、扇子(角度で開く)とバウムクーヘン(距離でリング状に重なる)を組み合わせたような格子です。

## 基礎: 前提となる概念
- BEV(Bird's-Eye-View): 自車を中心とした真上視点の地図。3D 空間の特徴を高さ方向につぶして 2D の平面で扱うことで、物体検出や経路計画がやりやすくなります。直交 BEV はこの平面を x(横)と y(縦)で区切ります。
- 視点変換(view transformation): カメラ画像の特徴(画像座標 u, v にある特徴)を BEV 座標(x, y や r, θ)へ移す処理。これが「カメラのみ3D検出」の心臓部です。1 枚の 2D 画像には奥行きの情報が直接入っていないため、視点変換は本質的に「2D から 3D を当てる」ill-posed(解が一意に定まらない)問題になります。
- 直交座標と極座標: 直交座標は (x, y) で位置を表します。極座標は「原点からの距離 r」と「基準軸からの回転角 θ」で表し、(x, y) = (r cos θ, r sin θ) で相互変換できます。極座標は中心から放射状に広がる現象(波紋、レーダー、カメラの視線)を表すのに向いています。
- クロスアテンション(cross-attention): 「ある対象(クエリ)が、別の集合(キー・バリュー)の中から関係の強いものを選んで重み付きで取り込む」計算。BEV のあるセルが「自分に関係するカメラ画像の場所」から特徴を集めるのに使われます(詳しくは [Cross-Attention Fusion(クロスアテンション融合)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0318_cross-attention-fusion))。
- カメラの内部・外部パラメータ: 内部パラメータ(焦点距離・主点など、行列 K)は画像座標と視線方向の関係を、外部パラメータ(カメラの位置・向き、回転 R と並進 t)は車体座標との関係を表します。視点変換はこれらを使って「画像のどこが世界のどの方向か」を計算します。
- 2D 検出と 3D 検出: 2D 検出は画像平面上の矩形を当てるだけですが、3D 検出は「世界座標での中心位置 (x,y,z)、サイズ (w,l,h)、向き yaw、速度 (vx,vy)」まで当てます。奥行きが必要になるため格段に難しくなります。

## 仕組みを詳しく

### 直交格子 vs 極座標格子(イメージ)
```
直交格子(Cartesian): 縦横の等間隔マス       極座標格子(Polar): 角度θ×距離rで分割
+--+--+--+--+                                       __---__
|  |  |  |  |                                     /  | | |  \    θ: 方位を扇形に分割
+--+--+--+--+                                    | --+自車+-- |
|  |  |自車|  |                                    \  | | |  /    r: 距離をリング状に分割
+--+--+--+--+                                       --___--
|  |  |  |  |                  カメラ光線(放射状)が極座標の θ 軸と一直線に対応する
+--+--+--+--+
```
極座標では各セルが「角度 θ_i, 距離 r_j のくさび」として定義されます。これがカメラ画像の「列(横位置)= 方位」「奥行き = 距離」に直接対応します。

### ステップ1: カメラ特徴の抽出
N 台(典型的には 6 台)のカメラ画像(例: 各 3×900×1600)を共有のバックボーン(画像から特徴を抜き出す土台ネットワーク、ResNet-101 や VoVNet-99 など)に通し、FPN(Feature Pyramid Network、複数解像度の特徴を作る構造)でマルチスケールの特徴マップを得ます。各カメラについて、例えば 256 チャネル × H × W の特徴が複数スケール出ます。

### ステップ2: 画像列 → 極座標レイへの変換(PolarFormer の核心)
PolarFormer の鍵となる観察は「カメラ画像の各列(縦の帯)は1つの方位角の扇に対応し、列内の縦位置は仰角(高さ)に対応する」という幾何事実です。したがって、極座標 BEV のある角度のくさび(θ が固定で r が変化する1本の放射線=ポーラーレイ polar ray)は、それを写しているカメラ画像の「ある縦の帯」に幾何的に対応します。論文はこの対応を使い、画像列ごとに「方位を固定した上で奥行き r に沿った特徴系列」(polar ray representation)を作ります。クロスアテンションのクエリは、対応する画像の帯にだけ注意を向ければよく、「どの画像のどこから取るか」が幾何から明快に決まるため探索範囲が絞れて学習が安定します。

### ステップ3: 距離(奥行き)の曖昧さの扱い
1 本の光線上では「物体がどの距離 r にあるか」は 1 枚の画像だけでは決まりません(scale ambiguity、近くの小物体と遠くの大物体が同じ見え方になる)。PolarFormer は距離方向(r)に沿って複数の候補位置を並べ、クロスアテンションの重みで「もっともらしい距離」に特徴を集約します。これは極座標の「距離リング」に沿って自然に行え、明示的な深度分布推定(Lift-Splat 系)を要しません。さらに複数フレームを並べる時間版(temporal stacking)も提案され、動きの手がかりから距離・速度の曖昧さを軽減します。

### ステップ4: 極座標 BEV 上で検出
出来上がった極座標 BEV 特徴(θ × r × チャンネル)の上で 3D 検出ヘッド(DETR3D 系のクエリベース、あるいは BEV 上の畳み込みヘッド)を動かし、物体の中心位置・サイズ・向き・速度を予測します。最終出力は通常の直交座標 (x, y) に変換して返します(下流の経路計画や HD マップ照合が直交前提のため)。

数値例で形状を追うと(概念図):
```
6カメラ画像 → backbone+FPN → 6×[256, H, W] (マルチスケール)
   ↓ ステップ2-3(列=方位の対応を使ったクロスアテンション + 距離方向集約)
極座標BEV特徴 [256, Nθ, Nr]   例 Nθ=128(角度分割), Nr=128(距離分割)
   ↓ 検出ヘッド
3Dボックス {x, y, z, w, l, h, yaw, vx, vy} を直交座標で出力
```

## 手法の系譜と主要論文
- Lift-Splat-Shoot (Philion & Fidler, ECCV 2020, arXiv:2008.05711): 各画素の奥行き分布を離散ビン上の確率として推定し、特徴を 3D に持ち上げ(lift)、直交 BEV へ撒く(splat)古典的手法([Lift-Splat-Shoot (LSS)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0282_lift-splat-shoot))。直交格子前提の出発点で、後続の多くがこの直交 BEV を引き継ぎました。
- PolarNet (Zhang et al., CVPR 2020, arXiv:2003.14032) / Cylinder3D (Zhu et al., CVPR 2021, arXiv:2011.10033): LiDAR 点群のセグメンテーションで極座標(polar / cylindrical、円筒座標)表現を導入した先行研究。回転式 LiDAR の点群が放射状・遠方ほどまばらに分布する性質を、極座標分割で均す発想です。「センサーの放射状性質を極座標で扱う」というアイデアの源流であり、PolarFormer はこれをカメラに移植したと言えます。
- DETR3D (Wang et al., CoRL 2021, arXiv:2110.06922): 3D の物体クエリを画像へ逆投影してサンプリングする、クエリベース検出の先駆け。明示的な BEV 格子を持たない設計で、PolarFormer の検出ヘッド側に影響しています。
- PETR (Liu et al., ECCV 2022, arXiv:2203.05625): 3D 位置エンコーディングを画像特徴に注入し、クエリが暗黙に 3D を参照できるようにした手法。座標系をどう表現に埋め込むかという問題意識を共有します。
- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270): 直交 BEV 格子をクエリにし、空間クロスアテンション + 時間セルフアテンションで画像から特徴を集める代表手法([BEVFormer（時空間BEVクエリ）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1056_bevformer-spatiotemporal))。PolarFormer はこの「格子クエリで画像から集める」枠組みを踏襲しつつ、格子を極座標へ変えました。両者は「BEV の座標系をどう選ぶか」で対をなします。
- PolarFormer (Jiang et al., AAAI 2023, arXiv:2206.15398): 本トピック。新規性は「極座標 BEV 表現 + 画像列=方位の対応を活かしたクロスアテンション視点変換」。動機は直交格子とカメラ放射状性質のミスマッチ解消。時間版も提案しています。

## 論文の実験結果(定量データ)
評価は主に nuScenes(6 カメラ + LiDAR + レーダーを持つ、約 1000 シーン・40 万フレームの大規模自動運転データセット)のマルチカメラ 3D 検出ベンチマークで行われます。指標の意味から説明します。
- mAP(mean Average Precision): 検出した物体が正解とどれだけ一致するかの平均精度。値が高いほど良い(0〜1)。nuScenes の 3D 検出では IoU(重なり率)ではなく中心位置の距離しきい値(0.5/1/2/4 m)で一致を判定し、10 クラス・4 しきい値で平均します。
- NDS(nuScenes Detection Score): nuScenes 独自の総合スコア。mAP に加えて位置誤差(mATE)・サイズ誤差(mASE)・向き誤差(mAOE、ラジアン)・速度誤差(mAVE、m/s)・属性誤差(mAAE)をまとめた指標で、NDS = (5·mAP + Σ(1−誤差)) / 10 の形で重み付け統合します。検出の「総合的な質」を測り、値が高いほど良い。

報告によると、PolarFormer は nuScenes test split で、VoVNet-99 バックボーン + 時間集約の強い設定でおよそ NDS 0.493、mAP 0.432 前後の水準に達したとされ、同時期の DETR3D(NDS 約 0.479)や BEVFormer(NDS 約 0.569 は最強設定、標準設定では同程度帯)と競合する成績でした。重要なのは「同じバックボーン・同じ条件で比較したとき、極座標表現が直交表現を上回る」という相対比較で、論文の主張の核心は「BEV の座標系をカメラ幾何(極座標)に合わせると、視点変換が自然になり、特に遠距離・視野端で精度が改善する」という点にあります。

アブレーション(どの要素を抜くとどうなるか)では次が示されています。(1) 極座標表現を直交表現に置き換えると mAP・NDS がともに低下し、極座標設計の寄与が確認されます。(2) 「列 = 方位の対応」を使った構造化アテンションを、対応を無視した素朴な全位置アテンションに置き換えると精度が落ち、幾何的対応の活用が効いていることが裏づけられます。(3) 時間集約(複数フレームの極座標特徴の統合)を加えると、特に速度推定誤差(mAVE)が下がり NDS が向上します。逆に、極座標の角度・距離の分割数(Nθ, Nr)を粗くすると遠方の表現力が落ちる、というトレードオフも観察されます。

## メリット・トレードオフ・限界
メリット
- カメラの放射状の見え方(方位 × 奥行き)と BEV 座標系が一致し、視点変換が幾何的に自然・正確になる。特に視野端・遠距離で直交格子より整合が良い。
- 画像の列が方位に直接対応するため、「どの画像のどこから特徴を取るか」が幾何から明快に決まり、アテンションの探索範囲が絞れて学習が安定し、収束が早い。
- 放射状に情報が分布するセンサー(カメラ、回転式 LiDAR)の性質に素直に合い、深度ビンの明示推定なしに距離曖昧性を扱える。

トレードオフ・限界
- 極座標はセルの物理サイズが一様でない。中心付近は密(細かい)、遠方は粗くなるため、遠距離での空間分解能が落ちる。距離方向を対数スケールにするなどの工夫が要る場合がある。
- 下流タスク(経路計画、HD マップ照合、占有グリッド)はほぼ直交座標前提なので、最終的に直交へ変換するひと手間と補間誤差が残る。
- 直交格子向けに最適化された通常の 2D 畳み込みや検出ヘッドをそのまま流用しにくい。極座標上の畳み込みは θ が一周する周期境界(0° と 360° が隣接)を扱う必要がある。
- 評価が nuScenes 中心であり、利得はカメラ配置・視野構成に依存しうる。極端な魚眼レンズや非対称なカメラ配置での一般性は別途検証が要る。

研究上の未解決課題として、極座標と直交座標の「良いとこ取り」(中心は極座標、遠方は直交を混ぜるハイブリッド格子)、極座標 BEV を占有予測や経路計画と end-to-end でつなぐ設計、周期境界を正しく扱う極座標専用畳み込みの設計が挙げられます。

## 発展トピック・研究の最前線
- 時間集約との統合: 極座標表現でも長期の時間スタッキング(複数フレームを並べて遅延融合する)が有効で、速度推定や遮蔽への頑健性が上がります([SOLOFusion (長期時間融合)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0499_solofusion-long-term-temporal))。
- 変形アテンションとの組み合わせ: 全位置アテンションは重いため、参照点を学習で絞る変形アテンション([Deformable Attention(変形可能アテンション)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0329_deformable-attention))を極座標サンプリングに組み込み、計算量を抑える方向。
- 占有予測・end-to-end 運転への波及: BEV をどの座標系で持つかは検出だけでなく、占有グリッド予測や軌跡計画([UniAD (計画指向BEV)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1091_uniad-planning-oriented-bev))の精度・計算量にも影響します。レンジビュー(range view、距離画像)表現や円筒座標も含め、座標系選択は表現学習の設計判断として今も議論が続いています。
