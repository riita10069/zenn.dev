# オフラインRL (IQL) による方策改良

## ひとことで言うと
運転方策(どんな状況でどう運転するかの方針)を鍛える最も自然な方法は、シミュレータの中で何度も試行錯誤させ、うまくいったら報酬を与えて強化することです。これがオンライン強化学習です。しかし現実には、シミュレータが使えない・実車で試行錯誤できない、という制約がしばしばあります。オフライン強化学習(オフライン RL)は、環境と一切やり取りせず、すでに記録された運転データだけから方策を改良する枠組みで、IQL(Implicit Q-Learning)はその代表的な手法です。模倣学習(お手本のまね)で作った初期方策を、記録データの中の「良かった行動」に重みを置いて磨き直します。

## 直感的な理解
プロ棋士の対局棋譜だけを使って将棋を強くなる、という状況を想像してください。自分で相手と何局も指して経験を積む(オンライン)ことはできず、過去の棋譜を眺めるしかありません。素朴なやり方は、棋譜の手をそのまま全部まねること(模倣学習)です。でもこれには問題があります。棋譜には強い手も弱い手も混ざっているのに、全部を等しくまねてしまう。そして棋譜にない盤面に出くわすと、まねる対象がなくて急に弱くなる。

賢いやり方は、棋譜のなかで「結果的に良かった手」を見分けて、そういう手を重点的にまねることです。さらに、棋譜に一度も現れない手については「実際に試したことがないので過大評価しない」と慎重に扱う。オフライン RL、特に IQL はまさにこれをやります。データの中の良し悪しを推定し、良い行動を強調し、データにない行動には手を出さない。これにより、ただの模倣を超えて方策を改良できます。

自動運転でこれが重要なのは、実車での試行錯誤が事故に直結し許されないからです。さらに、高精細な運転シミュレータが GPU の描画 API(Vulkan など)に依存し、特定のクラウド環境では動かせないことがあります。シミュレータが封じられた状況でも、録画済みの大量の運転ログから安全に方策を改良できる、というのがオフライン RL の現実的な価値です。

## 基礎: 前提となる概念

用語を一つずつ噛み砕きます。

- 強化学習(RL, Reinforcement Learning)とは、エージェント(ここでは運転モデル)が環境のなかで行動し、結果としてもらえる報酬(良い運転なら高得点)の累積を最大化するよう、試行錯誤しながら方策を学ぶ枠組みです。
- 方策(policy, π)とは、「今の状態でどの行動を取るか」を決める関数です。自動運転なら「今のカメラ映像と速度から、次にどんな軌道(加速度・操舵)を出すか」を返します。
- 状態(state, s)・行動(action, a)・報酬(reward, r)。状態は今の状況、行動は取った操作、報酬はその結果の良し悪しの数値。
- 価値関数。状態価値 V(s) は「状態 s から先、方策に従ったときに見込める累積報酬」、行動価値 Q(s, a) は「状態 s で行動 a を取り、以降は方策に従ったときの累積報酬」です。Q が分かれば、各状態で Q が最大の行動を選べば良い方策になります。
- アドバンテージ(advantage, A(s, a) = Q(s, a) − V(s))。「その行動が、その状態での平均的な行動よりどれだけ良かったか」を表します。プラスなら平均より良い行動、マイナスなら悪い行動。
- 模倣学習(IL, Imitation Learning)とは、専門家(人間ドライバ)のお手本ログをそのまままねるように学習する方法です。出力軌道がお手本軌道に近くなるよう回帰します。これは行動クローニング(behavior cloning, BC)とも呼ばれます。
- ベルマン方程式(Bellman equation): Q(s, a) = r + γ·E[max_a' Q(s', a')]。「今の価値 = 今もらう報酬 + 割引した次状態の最善価値」という再帰関係で、Q 学習の心臓部です。γ(ガンマ、割引率、0〜1)は将来の報酬をどれだけ重視するかの係数。

IL の弱点が、なぜ RL が要るかの動機になります。IL はお手本の分布しか学べないため、お手本にない状況(例えばドライバが一度も陥らなかった、車線を少しはみ出した状態)に出くわすと、まねる対象がなく急に下手になります。これを共変量シフト(covariate shift、学習時と本番でデータ分布がずれる問題)と呼びます。さらに誤差が累積する複合誤差(compounding error)も IL の宿痾で、一度ずれると元の分布から離れ続けて破綻します。Ross らの DAgger 論文(2011)が、BC の誤差が時間ステップ T に対して T^2 のオーダーで悪化しうることを理論的に示しており、これが IL の根本的な脆さの根拠としてよく引かれます。RL は報酬を通じて「結果の良し悪し」を学べるため、これを部分的に補えます。

ここで本質的な難所が登場します。オフライン RL では環境を試せないので、データにない行動 a の価値 Q(s, a) を確かめられません。それでもニューラルネットは外挿してもっともらしい値を吐きます。ベルマン方程式の max_a' は「全行動のなかで最善」を取りますが、その候補にデータにない行動が紛れ込み、ネットがそれを過大評価すると、方策はその未検証の行動を選ぶようになり、本番で破綻します。これを分布外行動の過大評価(overestimation of out-of-distribution actions)と呼び、オフライン RL 全体の中心的な課題です。過大評価された値はベルマン更新で次々に伝播し、価値関数全体が発散することすらあります。

## 仕組みを詳しく

IQL(Implicit Q-Learning、暗黙的 Q 学習)の核心は「データにない行動を一度も問い合わせずに価値関数を学ぶ」ことです。過大評価の根本原因は、価値更新の式に出てくる max_a Q(s, a)(全行動のなかで最良のもの)を取るときに、データにない行動が選ばれてしまう点でした。IQL はこの max を、データに現れた行動だけを使った expectile 回帰(エクスペクタイル回帰)で近似します。

expectile 回帰を平易に言うと、「上振れと下振れの重みを変えた偏った回帰」です。普通の最小二乗回帰は上振れ(予測より大きい)と下振れ(予測より小さい)の誤差を等しく罰しますが、expectile τ(0.5 < τ < 1)を大きくすると上振れを許容し下振れを強く罰するようになり、結果として推定値が「データに現れた値のなかで良いほう」に寄ります。具体的には、誤差 u に対する重みを |τ − 1(u<0)| として二乗誤差に掛けます(1(·) は条件が真なら 1、偽なら 0 を返す指示関数)。τ=0.5 なら普通の平均回帰に戻ります。これが、データにない行動を問い合わせずに max を近似する仕組みです。τ を 1 に近づけるほど真の max に近づきますが、推定が不安定になります(典型的には τ = 0.7〜0.9)。

IQL は二つの価値関数を学びます。

- V(s) を expectile 回帰で学習: ターゲット Q(s, a)(データにある行動 a の価値)に対して V(s) を expectile 回帰で当てる。これにより「状態 s でデータ中の良い行動を取ったときの価値」を、行動空間を探索せずに推定する。
- Q(s, a) を通常の回帰で学習: データにある遷移 (s, a, r, s') から Q(s, a) ≈ r + γ V(s') を学ぶ。ここで次状態の価値に max_a' Q ではなく学習済みの V(s') を使うのが鍵で、これでデータにない行動を一度も評価せずに済む。

価値関数が学べたら、最後に方策を取り出します。ここで使うのがアドバンテージ重み付き回帰(advantage-weighted regression, AWR)です。

```
weight(s, a) = exp(β · A(s, a)) = exp(β · (Q(s,a) − V(s)))
loss = − E[ weight(s, a) · log π(a | s) ]
```

各記号の意味です。β(ベータ)は温度パラメータで、アドバンテージをどれだけ強調するかを決めます(典型 3.0)。良い行動(A > 0)ほど指数関数的に重みが大きくなり、悪い行動の重みは小さくなる。π(a|s) は方策が状態 s で行動 a を出す確率(連続行動なら確率密度)。つまり「良かった行動を強くまね、悪かった行動は弱くまねる」更新です。重みが暴走しないよう上限でクリップ(例: 100)するのが実務上の定石です。重要なのは、この方策抽出が「データにある行動 a だけを重み付き模倣する」形になっている点で、方策がデータ分布から外れない(過大評価された行動を選ばない)構造になっています。

連続軌道を出す回帰型方策の場合、報酬や遷移を陽に持たないなら、より単純な AWR 風の近似がよく使われます。

```python
pred = model(obs)                                  # 学習対象の方策の出力軌道
with torch.no_grad():
    baseline_pred = frozen_model(obs)              # 基準となる現方策の出力(勾配を流さない)
# アドバンテージを「基準より目標(お手本軌道)に近ければプラス」と近似
advantage = -(pred - target).pow(2).mean(-1) + (baseline_pred - target).pow(2).mean(-1)
weights = torch.exp(beta * advantage).clamp(max=100.0)
loss = (weights * (pred - target).pow(2).mean(-1)).mean()   # 重み付き回帰
```

ステップで分解します。

1. お手本の未来軌道 target との二乗誤差で「どれだけ目標に近いか」を測る。
2. アドバンテージを「基準より目標に近ければプラス」と定義する。良い行動ほど大きい。
3. weights = exp(β · advantage) で重みを作り、上限でクリップして暴走を防ぐ。
4. 重み付き回帰損失を最小化する。良かった行動を強くまね、悪かった行動を弱くまねる更新になる。学習率は IL より小さく取る(例: 3e-5)のが安全で、初期方策を壊さず微調整する意図がある。

ただしこの簡略実装は、IQL の expectile 回帰による厳密な価値学習を省き、現方策の出力を基準にしたアドバンテージで代用した近似です。報酬を別途設計せず「お手本軌道への近さ」を価値の代理にしている点も近似で、AWR / AWAC に近い形だと理解するのが正確です。厳密な IQL では報酬・遷移・終端を含む遷移タプルから V/Q を学び、より強い改良(後述の stitching)が得られる一方、実装と調整の手間が増えます。

## 手法の系譜と主要論文

オフライン RL の系譜は「分布外行動の過大評価をどう抑えるか」を軸に進化しました。

- BCQ(Fujimoto, Meger, Precup「Off-Policy Deep RL without Exploration」ICML 2019, arXiv:1812.02900)が、オフライン RL で素朴な Q 学習が破綻する原因を「データにない行動の過大評価」と明確に指摘し、生成モデル(VAE)でデータ分布に近い行動だけに方策を制限する解を示しました。これがこの分野の出発点とされます。
- CQL(Kumar, Zhou, Tucker, Levine「Conservative Q-Learning」NeurIPS 2020)は、データにない行動の Q 値をわざと低く見積もる正則化項を加え、過大評価を保守的に抑えました。「Q を下から押さえる」ことで、外挿で膨らんだ価値を構造的に潰すのが新規性です。
- AWR(Peng, Kumar, Zhang, Levine「Advantage-Weighted Regression」arXiv:1910.00177, 2019)と AWAC(Nair, Gupta, Dalal, Levine「Accelerating Online RL with Offline Datasets」arXiv:2006.09359, 2020)は、価値推定の後に「アドバンテージを指数で重み付けして回帰する」という、方策抽出のシンプルな枠組みを確立しました。実装が IL の延長で書ける点が魅力で、AWAC はオフライン初期化からオンライン微調整へ繋ぐ流れの先駆けでもあります。
- IQL(Kostrikov, Nair, Levine「Offline Reinforcement Learning with Implicit Q-Learning」ICLR 2022, arXiv:2110.06169)が現在の代表格です。expectile 回帰により、データにない行動を一度も問い合わせずに価値を学ぶというエレガントな解を提示しました。価値学習(expectile 回帰)と方策抽出(AWR)を分離した二段構成で、安定して強く、ハイパーパラメータも少なめという実用性で広く採用されています。
- サーベイ Levine, Kumar, Tucker, Fu「Offline RL: Tutorial, Review, and Perspectives on Open Problems」(arXiv:2005.01643, 2020)が、分布外行動の過大評価という共通課題と各手法の位置づけを整理しており、この分野の地図として標準的に参照されます。

自動運転への応用では、実車で試行錯誤できない制約への回答としてオフライン手法が採られ、都市運転ベンチマークや録画ログからの方策改良で IQL 系・CQL 系が比較検討されています。

## 論文の実験結果(定量データ)

評価には D4RL(Datasets for Deep Data-Driven RL, Fu et al. 2020, arXiv:2004.07219)という標準ベンチマークが広く使われます。MuJoCo の運動制御(halfcheetah, hopper, walker2d)や、ナビゲーション課題(antmaze)などから成り、データの質を変えた複数の設定(専門家データ、混合データ、ランダムデータ等)で手法を比較します。指標は正規化スコア(normalized score)で、ランダム方策を 0、専門家方策を 100 に正規化した値です。

IQL 論文(2022)の主な報告です。

- MuJoCo の locomotion タスク群では、IQL は CQL と同等以上の正規化スコアを達成しつつ、学習が安定して計算も軽い(CQL より学習時間が短いと報告)。例えば混合品質データ(medium-replay 等)で、素朴な行動クローニングを大きく上回ります。
- 特に antmaze(まばらな報酬で長い経路を継ぎ合わせる必要がある難所)で、IQL は明確に強みを見せました。データ中の良い部分軌跡をつなぎ合わせる「stitching(継ぎ接ぎ)」能力が、expectile による価値推定で発揮されるためです。antmaze-medium / large の難設定で、行動クローニングがほぼ 0 点に近いのに対し、IQL は数十点台のスコアを出すと報告されています。stitching とは、「A→B の良い断片」と「B→C の良い断片」が別々のログにあるとき、両者を価値関数経由でつないで A→C の良い経路を構成できる能力で、単純な模倣には原理的にできない芸当です。
- アブレーション: expectile τ を変えると性能が変わり、τ を 0.5(=平均、普通の回帰)に近づけると stitching 能力が落ちて性能が下がる一方、1 に近づけすぎると推定が不安定になる。最適は中間(0.7〜0.9 付近)で、これは「データ中の良い行動を重視しつつ、外挿の暴走は避ける」というバランスを定量的に示しています。方策抽出側の温度 β も、小さすぎると改良が弱く(ただの BC に近づく)、大きすぎると少数の高アドバンテージ行動に過剰適合する、というトレードオフが確認されています。

これらの結果が示すのは、「価値推定をデータ内に閉じ込めること(IQL の expectile)」と「方策抽出を重み付き回帰で安定化すること(AWR)」の組み合わせが、オフライン RL の過大評価問題に対する実用的な解になっている、ということです。簡略な AWR 風実装でも、純粋な模倣学習に比べれば「良い行動を強調する」分の改良が見込めますが、価値関数を厳密に学ばないぶん stitching のような高度な改良は得にくい、という関係になります。

## メリット・トレードオフ・限界

メリット

- シミュレータ・実車での試行錯誤が不要。記録済みデータだけで方策を改良でき、危険がない。シミュレータが描画 API の制約などで使えない環境でも適用できる。
- IL の延長として実装でき(特に AWR 風の簡略版)、既存のチェックポイントとデータローダを再利用できる。
- 過大評価を構造的に抑える(IQL は expectile でデータ外行動を問い合わせない)ため、素朴な Q 学習より安定して学習できる。

トレードオフと限界

- 記録データに無い状況の改善は本質的に難しい。共変量シフトを完全には解消できず、データ分布の外側では依然として弱い。
- 改良の上限はデータの質に縛られる。良い行動がデータに含まれていなければ、いくら重み付けしても良い方策は取り出せない(オフライン RL 共通の限界)。
- 簡略な AWR 風実装は価値関数を厳密に学ばないため、stitching のような部分軌跡の継ぎ接ぎによる改良が効きにくく、改良幅が限定的になりやすい。
- 報酬設計の問題。運転の良し悪し(安全マージン、快適性、交通ルール遵守)を直接最適化するには報酬関数の設計が必要だが、「お手本軌道への近さ」で代用すると、結局は高品質な模倣に近づくだけで、本当の意味で人間を超える方策にはなりにくい。
- 未解決の課題として、評価のオフライン性(off-policy evaluation, OPE)がある。環境を試せない以上、改良した方策が本当に良くなったかをデータだけで正しく見積もるのは難しく、過大評価のリスクが評価側にも残る。

## 発展トピック・研究の最前線

- 拡散方策(diffusion policy)とオフライン RL の融合: 行動の分布を拡散モデルで表現し、多峰的(複数の正解がある)行動を扱う Diffusion-QL(Wang et al. 2023)などが、IQL 系を D4RL の一部設定で上回る報告を出している。運転では「右に避けるか左に避けるか」のような多峰性が本質的なため、この方向は重要。
- 系列モデル化としてのオフライン RL: Decision Transformer(Chen et al. NeurIPS 2021)は RL を「望ましい報酬を条件にした系列予測」として再定式化し、価値関数を陽に学ばずに方策を得る。自己回帰モデル・大規模 Transformer の発展とともに注目が続く。
- オフラインからオンラインへのファインチューニング: オフライン RL で初期化し、少量のオンライン経験で仕上げる「offline-to-online」が、安全性と最終性能の両立策として研究されている。AWAC はこの文脈の先駆け。
- 自動運転特有の論点: マルチエージェント環境(他車・歩行者の反応)を含む閉ループ評価、長期信用割当(crash の責任がどの行動にあるか)、安全制約付き RL(safe RL)など、運転ドメイン固有の難所が研究の最前線にある。
- オフライン方策評価(OPE): 改良後の方策を実環境に出す前にデータだけで性能を見積もる手法(importance sampling 系、Fitted Q Evaluation 系)が、実運用の鍵として重視されている。
