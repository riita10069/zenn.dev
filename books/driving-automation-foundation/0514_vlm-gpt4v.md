# GPT-4V / GPT-4o

## ひとことで言うと
GPT-4V(GPT-4 with vision)と GPT-4o(omni)は、画像を読み込んで自然な文章で答えられる大規模なマルチモーダル LLM(複数の種類の入力を扱える大規模言語モデル)です。運転シーンの画像を見せると、追加学習なし(ゼロショット)で「何が起きているか」「なぜ減速すべきか」を説明できます。自動運転研究では、人手でラベルを付ける代わりにこの高品質な記述を疑似ラベル(自動生成した教師データ)として大量に作る用途や、自前モデルの出力品質を測る「物差し」として広く使われます。中身が公開されていない(クローズドな)代表的基盤モデルであり、研究の出発点でもあれば、再現性の議論で常に引き合いに出される存在でもあります。

## 直感的な理解
機械学習モデルに何かを教えるには、入力(画像)とその正解(ラベル)のペアが大量に必要です。自動運転で「なぜこの場面で車が止まったのか」という因果的な説明をラベルにしたい場合、人間が一つひとつ文章で書くと1シーンあたり数分かかり、数百万シーン規模では費用も時間も現実的ではありません。さらに、書く人によって表現がブレます。

ここで発想を変えます。世の中にはすでに膨大なテキストと画像で訓練され、交通の常識・物理・社会的ルールをかなり持っている汎用 VLM があります。これに画像を見せて「この場面で注意すべきものを挙げ、自車が取るべき行動とその理由を述べよ」と聞けば、人間が書くようなラベルが自動で得られます。つまり「すでに賢い汎用モデルに、人間アノテータの代わりをさせる」のが核心です。GPT-4V/4o は、その汎用 VLM の中でも難しい因果推論・曖昧なシーン・長文の段階的説明で長らく品質が頭一つ抜けていたため、「品質最優先で少量の高品質ラベルや評価基準を作る」場面の定番でした。

もう一つ、研究の現場では「天井(上限)を測る道具」としての役割があります。自前で作った小さなモデルがどこまで行けるかを考えるとき、最も強い汎用 VLM がゼロショットで出せる品質を比較対象に置くと、「専用学習でどれだけ差を縮められたか」「逆にどこは商用 API に頼らないと無理か」が見えてきます。

## 基礎: 前提となる概念
理解の土台になる用語を噛み砕きます。

- LLM(Large Language Model、大規模言語モデル): 大量のテキストで「次に来る単語」を予測するよう訓練された巨大なニューラルネットです。文章生成・要約・質問応答ができます。パラメータ(モデル内部の調整つまみ)が数十億〜数千億個あります。
- マルチモーダル: 「モダリティ(modality)」は入力の種類を指す言葉で、テキスト・画像・音声などがそれぞれ別モダリティです。マルチモーダルとは、これらを同時に扱えることを意味します。GPT-4V は「テキスト + 画像」、GPT-4o は「テキスト + 画像 + 音声」を扱います。
- トークン(token): モデルが処理する最小単位。文章なら単語の断片、画像なら小さなパッチ(画像片)に対応します。LLM は入力トークン列を読み、続きのトークンを1つずつ生成します。
- ゼロショット(zero-shot): その課題専用の追加学習を一切せず、事前学習で得た一般知識だけで答えること。「運転データで微調整していないのに運転シーンを説明できる」のがゼロショットの強みです。対義語は、少数の例を見せる few-shot や、専用データで追加訓練する fine-tuning です。
- VLM(Vision-Language Model): 画像と言語をまたいで理解・生成できるモデル全般を指します。GPT-4V/4o はクローズドな(内部非公開の)代表例で、対照的に重みが公開された VLM もあります。
- 疑似ラベル(pseudo-label): 人間ではなくモデルが付けた、間違いを含みうる教師データ。人間の正解(ground truth)と同等に信じてはいけません。
- 幻覚(hallucination): モデルが、入力に存在しないものをもっともらしく述べてしまう現象。VLM では「画像に無い歩行者を述べる」「赤信号を青と言う」といった形で現れます。
- 較正(calibration): モデルが出す「自信(確信度)」が、実際の正答率とどれだけ一致しているか。「90%自信あり」と言ったとき本当に9割当たるなら良く較正されている、と言います。VLM は自信過剰になりがちで、ここが疑似ラベル運用の難所です。

## 仕組みを詳しく
GPT-4V/4o は内部詳細が非公開(クローズド)で、学習データ・規模・正確なアーキテクチャは公式に開示されていません。ただしこの系統の VLM が一般にどう作られているかは、公開された先行研究から逆算できます。

構成の一般像: 視覚エンコーダ(画像を数値の特徴ベクトルに変換する部分、典型的には ViT すなわち Vision Transformer)で画像を「画像トークン」の列に変換し、それを巨大な LLM の入力に、文章トークンと混ぜて与えます。問題は「視覚側の特徴」と「言語側のトークン空間」をどうつなぐかで、ここに2つの代表的な系譜があります。

- Flamingo (Alayrac et al., NeurIPS 2022): 凍結した(重みを更新しない)視覚エンコーダと凍結 LLM の間に、学習可能な「gated cross-attention」層を差し込み、画像特徴を言語モデルの各層に注入する設計。画像と文章が交互に並んだ入力(interleaved)を扱える点が新しく、少数例(few-shot)で多様な視覚タスクをこなせることを示しました。
- BLIP-2 (Li et al., ICML 2023): 「Q-Former」という小さな橋渡しモジュールを学習し、凍結視覚エンコーダの出力から「LLM が読める少数のクエリトークン」を抽出して LLM に渡す設計。学習する部分を最小に絞ることで、巨大な視覚・言語モデルを安く接続できることを示しました。

GPT-4V/4o の正確な接続方式は非公開ですが、「強力な視覚エンコーダ + 巨大 LLM を、橋渡し機構でつなぐ」という大枠はこの系譜の延長にあると理解しておくと、挙動の見通しが立ちます。GPT-4o は画像・音声・テキストを単一モデルで end-to-end(入口から出口まで一気通貫)に扱う「omni(全部入り)」設計で、別系統を組み合わせるより応答が高速・低コストになったとされます。

運転研究での疑似ラベル生成フローを段階的に示します。

1. 走行映像から代表フレームを選ぶ。1枚のこともあれば、数フレームを並べて時間的な変化を見せることもある。
2. 構造化を促すプロンプト(指示文)を与える。例:
   「この交通シーンを説明し、自車が取るべき行動(accelerate / maintain / decelerate / stop)とその理由を JSON で出力せよ。キーは scene, hazards, action, reason。」
3. モデルが構造化テキストを返す。例:
   ```json
   {
     "scene": "片側2車線の交差点。前方信号は黄色。右から歩行者が横断中。",
     "hazards": ["横断中の歩行者", "黄信号での進入判断"],
     "action": "decelerate",
     "reason": "信号が黄で歩行者が横断しているため、停止に備えて減速する。"
   }
   ```
4. この JSON を、エンドツーエンド運転モデルの補助タスク(行動分類ヘッドや因果説明ヘッド)の教師ラベルとして使う。あるいは少数の高品質ラベルとして、安価な公開 VLM の出力品質を検証する基準にする。

なぜこの設計か。プロンプトで出力形式(JSON のキー)を縛ることで、自由文をそのまま下流で機械処理できる構造に落とし込めます。また action を有限の選択肢に固定すると、後段で分類ラベルへ写像しやすくなります。

実務上もっとも重要な注意は幻覚です。GPT-4V/4o は一定確率で、存在しない物体を述べたり、空間関係(右/左、前後)や数え上げ(歩行者が2人か3人か)を誤ります。安全クリティカルな運転ラベルにこれをそのまま使うのは危険です。定石は検証工程との組み合わせです。

- 多数決: 同じ画像で複数回生成する、あるいは複数モデルで生成し、一致した部分だけ採用する。
- グラウンディング(grounding、根拠づけ): 物体検出器など客観的な知覚出力と突き合わせ、VLM が述べた物体が実際に検出されているか確認する。
- 抜き取り人手検査: 一部を人間が検品し、誤り率を見積もって全体の信頼度を管理する。

数値例で「不一致」を扱う発想を示します。VLM が「右に歩行者あり(risk: high)」と述べたが、検出器が右側に何も検出していない場合、その疑似ラベルには低い confidence(信頼度)を付けるか破棄します。逆に両者が一致した部分には高い confidence を付けます。こうして「VLM 出力を confidence と provenance(出所、どのモデルが付けたか)付きで管理する」のが現代的な運用です。

## 手法の系譜と主要論文
GPT-4V/4o は単体の運転手法ではなく基盤モデルなので、系譜は「マルチモーダル基盤の登場 → 強力なクローズド基盤 → それを運転に橋渡しする研究」という流れで描けます。

- Flamingo (Alayrac et al., NeurIPS 2022, arXiv:2204.14198): few-shot マルチモーダル LLM の先駆。画像と文章が交互に並ぶ入力を扱い、少数例で多くの視覚タスクに適応できることを示した。これが「巨大 LLM に視覚を後付けする」設計の方向性を確立しました。
- BLIP-2 (Li et al., ICML 2023, arXiv:2301.12597): Q-Former による安価な視覚・言語接続を提案。学習コストを大幅に下げ、後続の多くのオープン VLM の土台になりました。
- GPT-4 Technical Report (OpenAI, 2023, arXiv:2303.08774): GPT-4 全体の能力を報告。視覚入力を受け付ける能力に言及。学習データ・規模・手法は競争上・安全上の理由で非公開とされ、再現性の観点では大きな制約があります。これが「クローズドな高性能基盤モデル」という前提を作りました。
- GPT-4V(ision) System Card (OpenAI, 2023): GPT-4 に視覚入力を加えた GPT-4V の能力と安全上の限界を報告。新規の手法提案というより、「強力な汎用視覚言語能力を持つが、空間関係・細かい数え上げ・医療など専門領域では誤る」という限界の開示が主眼です。研究者は「ゼロショットで広範な視覚タスクに使えるが、過信は禁物」という指針を得ました。
- GPT-4o System Card (OpenAI, 2024, arXiv:2410.21276): テキスト・画像・音声を単一モデルで扱う omni モデル。動機は「モダリティを跨ぐ統合と低遅延・低コスト化」。別系統を組み合わせる構成より高速・安価で、マルチモーダル性能も同等以上とされます。これにより、中規模(数十万〜数百万フレーム)の疑似ラベリングがコスト的に現実味を帯びました。
- GPT-Driver (Mao et al., 2023, arXiv:2310.01415): 運転の計画問題を「言語モデルへの質問応答」として定式化し、GPT-3.5/4 にシーン情報を言語で与えて軌道を言語トークンとして出力させる。動機は「LLM の常識推論を計画に直接使う」こと。GPT-4 系を運転に橋渡しする初期の代表例です。
- DriveGPT4 (Xu et al., RA-L 2024, arXiv:2310.01412): 動画を入力に、運転行動とその根拠を言語で出力する VLM を学習。学習データの一部を ChatGPT/GPT-4 で自動生成しており、「強い基盤 VLM を疑似ラベル源にして専用 VLM を作る」流れの典型です。論文自身が自動生成ラベルの幻覚リスクとフィルタの必要性を指摘しています。
- GPT-4V early evaluation for AD (Wen et al., 2023, arXiv:2311.05332): GPT-4V を運転シーンに当てて、シーン理解・因果推論・意思決定の能力と限界を体系的に検証した報告。「常識・因果は強いが、正確な距離・数え上げ・連続制御は弱い」という運転固有の弱点像を早期に整理しました。

## 論文の実験結果(定量データ)
GPT-4V/4o は単一ベンチマークの数値で語るより、「ゼロショットでどこまでできて、どこで崩れるか」を押さえるのが重要です。報告されている傾向を、指標の意味とあわせて説明します。

- GPT-4V System Card では、一般的な視覚理解(キャプション生成、図表読解、常識的なシーン記述)で高い品質を示す一方、空間関係・正確な座標推定・細かいオブジェクトカウント(数え上げ)で誤りが生じやすいと明記されています。運転では「歩行者の人数」「自車から見た物体の正確な方位」がまさにこの弱点に当たるため、座標や数の正確さを要するラベルには向きません。逆に「行動の理由づけ」のような言語的・因果的ラベルは比較的得意です。
- 運転シーン評価(Wen et al., 2023): GPT-4V は信号・標識の認識や、混雑・夜間・悪天候といった条件下でのシーン記述、危険要因の推定で人間に近い質的な妥当性を示す一方、(1) 物体間の正確な距離・相対速度の推定、(2) 連続的な車両制御量(操舵角・加速度)の直接出力、(3) 複数物体の正確な計数で系統的に誤ることが定性・定量の両面で報告されています。
- 運転 QA ベンチマークでの傾向: nuScenes 系の視覚質問応答(VQA、画像を見て質問に文章で答える課題)では、GPT-4V をゼロショットで適用すると、運転データで専用に微調整した小型 VLM に対して知覚系の細かい質問(正確な距離・個数)で劣る一方、常識・因果系の質問では強い、という分かれ方が複数の研究で報告されています。指標は回答の正解率(accuracy)や、自由文回答を判定する言語スコア(BLEU、CIDEr など、生成文が参照文にどれだけ近いかを測る指標)が使われます。BLEU/CIDEr はいずれも「生成文と人手参照文の n-gram(連続する単語列)の重なり」を測るもので、CIDEr は運転 VQA 用語の重みづけが効きやすく、この種のタスクでよく主指標に使われます。
- コスト/速度のオーダー感: GPT-4V から GPT-4o への移行で、API の応答が大幅に高速・安価になったと報告されています。これは「同じ予算でラベル付けできるフレーム数が数倍に増えた」ことを意味し、データエンジン運用の現実性に直結します。ただし数百万フレーム全件を商用 API に送る運用は、依然として費用と秘匿性(走行映像を外部に送ること)の両面で制約が残ります。

アブレーション的な観点として重要なのは「検証工程を抜くとどうなるか」です。多数決やグラウンディングを外して GPT-4V/4o の生出力をそのまま教師に使うと、下流モデルが幻覚由来の誤った因果(存在しない歩行者ゆえの減速など)を学習し、性能が劣化する報告があります。また「連鎖推論(Chain-of-Thought、知覚→予測→計画と段階的に考えさせる)」を入れると、一気に答えさせるより因果ラベルの一貫性が上がる一方、推論トークンが増えてコスト・遅延が悪化するトレードオフがあります。つまり「VLM ラベルの価値は、検証・フィルタとプロンプト設計とセットで初めて安全に引き出せる」のが共通の知見です。

## メリット・トレードオフ・限界
- メリット: ゼロショットで高品質なシーン理解・因果説明ができ、運転特化の追加学習なしに使える。
- メリット: プロンプトで自然文・構造化(JSON)出力を柔軟に制御でき、少量の高品質ラベルや評価基準を素早く作れる。GPT-4o では高速・低コスト化により中規模ラベリングも現実的。
- メリット: 「正解の物差し」として、安価な公開 VLM の出力を検証する基準に使える(高品質だが少量 → 大量は別モデル、という役割分担)。
- トレードオフ: クローズドで重みを自前 GPU に置けず、API 経由のみ。これは秘匿走行データを外部送信せざるを得ないというデータガバナンス上の制約を生む。
- トレードオフ: フレームごとに課金され、数百万フレーム規模では費用が膨大。プロンプト/応答が長いほど費用が積み増す。
- 限界: 幻覚・空間関係・数え上げ・正確な距離/速度の誤りがあり、座標や個数や制御量の正確さを要する安全クリティカルなラベルには単独では使えない。必ず検証工程が要る。
- 限界: バージョンが更新されると出力分布が変わりうる(再現性・固定が難しい)。論文の数値はそのモデル時点のものと割り切る必要がある。
- 未解決の課題: 生出力の信頼度を自動で正確に見積もる方法(よく較正された confidence)は確立されておらず、どこを人間が検品すべきかの最適な選別は依然オープンな問題です。

## 発展トピック・研究の最前線
- 二段運用(高品質 × 大量): GPT-4o で少量の高品質ラベルと評価基準を作り、大量処理は自前 GPU で動く公開 VLM に任せ、その出力を GPT-4o ラベルで検証する分業が広がっています。
- グラウンディングの強化: VLM 出力を検出器・占有グリッド・3D 知覚と突き合わせて根拠づける研究が進み、幻覚を構造的に減らす方向にあります。
- 反実仮想ラベル: 「もし右に避けていたら」のような counterfactual(反実仮想)推論を VLM に生成させ、計画の理由づけを豊かにする試みが出ています。
- 連鎖推論(Chain-of-Thought): 一気に答えさせず「知覚 → 予測 → 計画」と段階的に考えさせることで、因果ラベルの一貫性を高める手法が研究されています。
- 閉ループ運転への接続: LMDrive (Shao et al., CVPR 2024) のように、言語指示に従って実際にシミュレータ内を走らせる閉ループ運転が研究され、「言語で理解する」段階から「言語に従って制御する」段階への橋渡しが進んでいます。
