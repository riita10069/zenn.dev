# Bézier軌跡ヘッド

## ひとことで言うと

自動運転モデルが出力する「これから数秒どう動くか」という運転計画(軌跡, trajectory)を、ガタガタした出力ではなく、なめらかな曲線として出すための仕組みです。各時刻の値を全部バラバラに予測させるのではなく、少数の「操り点(control point)」だけをニューラルネットワークに予測させ、それを Bézier(ベジエ)曲線という数式で滑らかに膨らませて、数十ステップ分の運転コマンドを作ります。出力の「形」そのものを滑らかな曲線に固定することで、後処理も追加の損失項もなしに、滑らかさを構造的に(by construction)保証するのが本質です。

## 直感的な理解

なぜ「滑らかさ」がそんなに大事なのでしょうか。たとえば学習済みのニューラルネットワークに、0.1秒刻みで6.4秒先まで(64ステップ)の加速度を各時刻独立に予測させたとします。すると「0.1秒後はアクセル0.3、0.2秒後は0.9、0.3秒後は0.1…」のように、隣り合う時刻で値が激しく上下することがあります。学習データにわずかなノイズがあれば、各点の予測は互いに無関係にブレるからです。各時刻の出力に互いを縛る制約が無い限り、ネットワークはそれぞれを独立に外挿し、ノイズをそのまま吐き出します。

この激しい上下動が実車では致命的です。値が急に変わると、加速度の変化率である「ジャーク(jerk)」が大きくなります。ジャークが大きい運転は、乗っている人が「ガクンガクン」と前後に揺さぶられる不快な運転です。それだけでなく、ハンドルやアクセルを動かす実際の装置(アクチュエータ)には物理的な応答速度の限界があり、急変するコマンドには追従できません。つまり「物理的に実行不可能な計画」を出してしまうのです。

ここで発想を変えます。鉛筆で紙にぐにゃぐにゃの線を描く代わりに、画鋲を5本だけ刺して、その間をしなやかな定規(雲形定規)でなぞるイメージです。画鋲(操り点)の位置はネットワークが自由に決めますが、その間を埋める線は数式が滑らかに保証する。これが Bézier 軌跡ヘッドの直感です。操り点が5本しかない以上、出力される64ステップの曲線は構造的にカクカクできません。「滑らかにするための損失」を後付けで足すのではなく、「滑らかな関数しか表現できない出力空間」をはじめから選ぶ、という設計思想です。

## 基礎: 前提となる概念

いくつか言葉を噛み砕いておきます。

「軌跡パラメータ化(trajectory parameterization)」とは、軌跡を少数の数(パラメータ)で表す方法のことです。最も素朴なのは各時刻の値を直接並べる「waypoint(経由点)表現」で、64ステップなら64個×信号数の自由度を持ちます。対して多項式やスプライン(区分多項式)で表すと、係数や操り点という少数のパラメータだけで曲線全体が決まります。自由度を減らすことは、表現できる軌跡の集合を「滑らかなものだけ」に絞ることと同義で、これがそのまま正則化(過剰な自由度を抑えて汎化を助けること)になります。

「ジャーク(jerk)」は位置を時間で3回微分した量、つまり加速度の時間変化率です。乗り心地の良し悪しを定量化する標準的な指標で、運動計画の世界では「ジャーク最小化(jerk-minimizing)」が古典的な目標になっています。人間が快適と感じる縦方向ジャークはおよそ 2 m/s³ 以下と言われ、これを超えると不快な揺れになります。

「ユニサイクルモデル(unicycle model)」は、車を「前進速度と曲がり具合だけで動く一輪車」に単純化した運動モデルです。状態は位置 (x, y) と向き θ、制御入力は加速度 a と曲率(または角速度)κ です。曲率 κ とは「どれくらいきつく曲がっているか」を表す量で、半径 r の円を描いて回るなら κ = 1/r です。直進なら κ = 0、急カーブほど κ が大きくなります。実際の車はもっと複雑(車輪の切れ角と車体長の関係を表す自転車モデル=bicycle model など)ですが、ユニサイクルは「制御信号で運動を表す」最小の枠組みとして広く使われます。

「Bézier 曲線」は、n+1 個の操り点 P_0, ..., P_n の重み付き平均として t∈[0,1] でパラメータ化される曲線です。重みは「Bernstein 基底(Bernstein basis)多項式」で与えられます。重要な性質として、(1) 曲線は必ず操り点の凸包(convex hull, 操り点を囲む最小の凸領域)の中に収まる、(2) 操り点の数を増やすと表現力が上がるが次数も上がる、(3) 端点 P_0, P_n を必ず通る、というものがあります。この「凸包に収まる」性質が、出力の暴れを抑える数学的な根拠になります。操り点を1つ大きく動かしても、曲線全体はその影響を「ならして」受け取るため、急峻なスパイクが生まれにくいのです。

## 仕組みを詳しく

典型的な Bézier 軌跡ヘッドは、おおむね次の3段階で動きます。

### 1. コンテキスト(状況)の集約

プランナにはまず「今どういう状況か」を表す特徴が入ります。代表的な入力は3種類です。

- 鳥瞰図特徴(BEV features) 形状 `[B, C, H, W]` — 上空から見下ろした地図のような視点に整形された視覚特徴。`B` はバッチサイズ(同時処理するサンプル数)、`C` はチャンネル数(埋め込み次元、256前後が一般的)、`H, W` は空間グリッドの縦横。
- 視覚履歴(visual history) 形状 `[B, D_v]` — 過去数フレームの映像を要約したベクトル。
- 自車運動履歴(egomotion history) 形状 `[B, D_e]` — 自車(ego)の過去の速度・ヨーレート(向きの変化速度)などの要約。

これらをそれぞれ全結合層(linear projection)で共通の次元(例:256)に投影して足し合わせ、小さな MLP(多層パーセプトロン、GELU 活性化を挟む2層程度)に通して、1本の状況ベクトル `[B, 256]` にまとめます。BEV 特徴は空間方向に平均(`mean(dim=(2,3))`)して `[B, C]` に潰してから使うのが軽量で一般的です。空間平均は粗い要約ですが、計画ヘッドが必要とするのは「全体としてどんな状況か」であり、細かい空間位置は知覚段でほぼ消費済みという前提に立っています。

### 2. 操り点(control points)の予測

状況ベクトルから、全結合層が `num_controls × num_signals` 個の値を出し、`[B, num_controls, num_signals]` に整形します。よくある設定は:

- `num_controls = 5` — Bézier 曲線を定義する操り点の数(4次曲線)。
- `num_signals = 2` — 出力信号の数。`(acceleration, curvature)`(加速度, 曲率)の2チャンネルが代表的。

ここが設計上の要点です。出力を生の `(x, y)` 座標(地図上の位置)にするか、ユニサイクル制御信号 `(a, κ)` にするかで性質が変わります。制御信号で出す利点は、(1) センサ位置誤差や座標系のドリフトに頑健、(2) 下流のコントローラ(実際に車を動かす制御器)がそのまま消費できる、(3) 速度や曲率の上限といった物理制約をクリップ(値域で頭打ちにすること)で埋め込みやすい、ことです。欠点は、制御信号から実際の通過点 (x, y) を得るには車両運動モデルを別途数値積分する必要があることです。座標で出す方式はその逆で、評価指標(後述の ADE/FDE はメートル単位の位置誤差)を直接最適化できる反面、物理的実現性は保証されません。

### 3. Bernstein 基底での展開

少数の操り点を、固定された Bernstein 基底行列で多数のステップに引き伸ばします。Bernstein 基底多項式は次式です。

```
B_{i,n}(t) = C(n, i) · t^i · (1 - t)^(n - i)
  n = num_controls - 1   (例: 5 操り点なら n = 4)
  i = 0, 1, ..., n
  C(n, i) は二項係数 (n 個から i 個選ぶ組合せの数)
```

`t` は区間 [0, 1] を出力ステップ数(例:64)で等分した時間軸です。各時刻 t について B_{0,n}(t), ..., B_{n,n}(t) を並べると、「その時刻で各操り点をどんな比率で混ぜるか」を表す重みベクトルになります。これらの重みは任意の t で必ず非負で、合計が 1 になります(分割の性質, partition of unity)。だからこそ出力は操り点の凸結合となり、凸包から飛び出さないのです。すべての t について計算すれば `[num_steps, num_controls]`(例: `[64, 5]`)の基底行列が事前計算できます。二項係数は標準ライブラリで計算でき、外部依存は不要です。この行列は学習中に変わらない定数なので、一度だけ作って使い回します。

軌跡の展開は1つの行列積(テンソル縮約)で済みます。

```
control_points [B, 5, 2]  ×  basis [64, 5]  →  trajectory [B, 64, 2]
einsum("pc,bcs->bps", basis, control_points)
  p = 64 ステップ, c = 5 操り点, s = 2 信号
```

最終的に `[B, 64, 2]` を `[B, 128]` に平坦化して出力します。0.1秒刻みなら64ステップ=6.4秒先までの計画に相当します。Bernstein 基底の各列は t について滑らかな多項式なので、操り点をどう動かしても出力は必ず C^∞ 級(無限回微分可能)に滑らか、つまりジャークが構造的に有界です。

### 数値例で確かめる

操り点の加速度成分が `[0.0, 1.0, -1.0, 0.5, 0.0]` だったとします。waypoint 表現なら 0.0→1.0→-1.0 と隣接ステップで2.0も跳ねますが、Bézier 展開では t=0 で 0.0、中間で操り点を凸結合した穏やかな値を通り、t=1 で 0.0 に戻ります。隣接ステップ間の差は基底多項式の連続性により必ず小さく抑えられます。たとえば 64 等分すれば、隣接ステップ間の t の差は約 0.0156 で、滑らかな多項式の値はその刻みでわずかしか変化しません。これが「後処理なしで滑らか」の実体です。

## 手法の系譜と主要論文

軌跡を滑らかな曲線でパラメータ化する考えは、自動運転の学習型プランナよりはるかに古く、ロボティクスと最適制御の運動計画に根があります。

- 多項式/スプライン軌跡と「ジャーク最小化(jerk-minimizing trajectory)」の定式化は、運動計画の標準です。代表は Werling, Ziegler, Kammel, Thrun(ICRA 2010)"Optimal trajectory generation for dynamic street scenarios in a Frenet frame" です。これは道路に沿った曲線座標系(Frenet 座標, 道路中心線からの縦・横位置で表す座標)の上で、5次多項式(quintic polynomial)を使ってジャーク最適な軌跡を解析的に生成しました。動機は「軌跡を低次の曲線で表せば、滑らかさを曲線の次数で直接コントロールでき、少ない自由度で物理的に実現可能な経路を保証できる」ことです。5次多項式が選ばれたのは、始点と終点の位置・速度・加速度(計6条件)を厳密に満たしつつ、その制約下でジャーク二乗積分を最小化する解がちょうど5次になるからです。

- 学習型(end-to-end)プランナの主流は、長らく座標 waypoint を直接回帰する方式でした。UniAD(Hu et al., CVPR 2023 Best Paper, arXiv:2212.10156)や VAD(Jiang et al., ICCV 2023, arXiv:2303.12077)は、知覚から計画までを1本のネットで解き、最終的に未来の (x, y) waypoint を出力します。これらは表現力が高い反面、時刻間で出力が不連続になりがちで、評価時や実車展開時に平滑化の後処理(low-pass フィルタなど)を要することが多いという課題があります。

- 近年は、この後処理依存を構造で置き換える「曲線パラメータ化ヘッド」が再注目されています。Bézier/B-スプラインで軌跡を表し操り点だけを予測する設計は、低ジャーク・低計算量・物理制約の埋め込みやすさという点で、特に量産を見据えた実装で好まれます。発想自体は古典の jerk-minimizing 多項式と同じですが、操り点を最適化で解くのではなくニューラルネットに学習させる点が新しい接続です。

- 出力を座標ではなくユニサイクル制御信号 `(accel, curvature)` にする発想も広がっています。これらは「生の (x, y) より制御信号で出す方が、センサノイズと下流コントローラに対して頑健」と論じます。古典制御の「微分平坦性(differential flatness)」— 適切な出力変数を選べば軌跡と入力が代数的に決まる性質 — とも整合する考え方です。

系譜としては「最適制御の jerk-minimizing 多項式(Werling 2010)→ end-to-end の waypoint 回帰(UniAD, VAD)→ 学習ヘッドに曲線パラメータ化と制御信号契約を取り込む」という流れで理解できます。

## 論文の実験結果(定量データ)

軌跡計画の評価でよく使われる指標を、意味から説明します。

- L2 誤差 / ADE(Average Displacement Error) — 予測した各時刻の位置と正解位置のユークリッド距離の平均。FDE(Final Displacement Error)は最終時刻だけの距離。単位はメートルで、小さいほど良い。これは「どれだけ正解の道筋に近いか」を測ります。
- 衝突率(Collision Rate) — 予測軌跡が他車・歩行者と衝突する割合(%)。安全性の代理指標。
- ジャーク / 加速度の二乗平均 — 乗り心地と物理実現性の指標。小さいほど滑らか。

nuScenes(自動運転の標準ベンチマーク、米国・シンガポールの市街地走行データ)での open-loop 評価(記録済みデータに対して1回だけ予測を出し、正解軌跡と比べる評価。実際に走らせて反応を見る closed-loop と対比される)において、UniAD は平均 L2 誤差およそ 1.03m(1/2/3秒平均)、衝突率 0.31% 程度を報告しました。VAD はこれを上回り、平均 L2 をおよそ 0.72m、衝突率を 0.22% 程度に改善したと報告しています(いずれも論文の自己報告値で、評価プロトコルにより差があります)。L2 が 1.03m から 0.72m へ約 0.3m 縮むのは、3秒後の予測位置が30cm正解に近づくことを意味し、車線幅(約3.5m)に対して無視できない差です。これらは waypoint 回帰系の代表値です。

曲線パラメータ化ヘッド側の効果は、主にジャークと実車での実行可能性で現れます。一般的な比較として、waypoint 直接回帰に滑らかさ罰則(jerk penalty)を加えた構成と、Bézier/スプラインで構造的に滑らかさを保証した構成を比べると、後者は同程度の L2 誤差を保ちつつ、ジャークの二乗平均を数十%規模で低減できると報告されることが多いです。アブレーション(構成要素を抜いて影響を測る実験)の典型的な知見は次の通りです。

- 操り点の数(次数)を 5→3 に減らすと、ジャークはさらに下がるが ADE/FDE が悪化する(滑らかすぎて急な回避を表せない)。逆に 5→8 に増やすと表現力は上がるがジャークが増え、waypoint 回帰に近づく。つまり操り点数は「滑らかさ vs 俊敏さ」のトレードオフを直接制御するノブです。
- 出力を座標から制御信号 `(a, κ)` に変えると、open-loop の L2 はやや悪化することがある(座標を直接最適化していないため)が、closed-loop(実際にコントローラで走らせる)評価では実行可能性と頑健性が上がる、という乖離がしばしば観察されます。これは「open-loop 指標と実走性能は別物」という、この分野で重要な注意点を端的に示します。

数値の絶対値は評価プロトコル(ego status の入力可否など)に強く依存するため、論文間の単純比較には注意が必要です。Zhai et al.(2023)"Rethinking the Open-Loop Evaluation of End-to-End Autonomous Driving"(arXiv:2305.10430)は、自車の現在速度・加速度を入力に入れるだけで open-loop L2 が大幅に下がり、極端には視覚を一切使わず ego 状態だけの単純な MLP でも UniAD/VAD に匹敵する L2 が出ることを示しました。これは「open-loop L2 は運転知能をほとんど測れていない」という強い警鐘で、曲線ヘッドの優劣も open-loop L2 だけで判断してはいけないことを示します。

## メリット・トレードオフ・限界

メリット
- 後処理フィルタも追加損失項もなしに、滑らかさ(低ジャーク)を構造的に保証する。曲線の次数で滑らかさを直接設計できる。
- 操り点を数個予測するだけでよく、出力ヘッドが軽量で学習が安定しやすい。waypoint 直接回帰より過学習や時刻間ノイズに強い(自由度を絞ることが正則化になる)。
- ユニサイクル制御信号で出せば、実車コントローラに乗せやすく、速度・曲率の上限などの物理制約をクリップで埋め込める。
- 固定の Bernstein 基底は事前計算でき、外部ライブラリ非依存で高速。1回の行列積で展開が終わる。

トレードオフ・限界
- 表現力が操り点の数で頭打ちになる。少数の操り点では、急ハンドルの緊急回避など高周波で複雑な軌跡を正確に表せない(Bézier の凸包性質ゆえに、操り点を急に動かしても曲線は鈍る)。
- 滑らかさが強い前提なので、本来カクッと曲がるべき場面でも丸まる方向にバイアスがかかる。
- 制御信号で出すと、座標で評価したい指標(ADE/FDE)を計算するには車両運動モデルでの数値積分という追加ステップが要り、積分誤差が累積しうる。
- Bézier 曲線は端点を必ず通るが、中間の操り点は曲線が通過しない点であり、解釈や制約付けがやや非直感的。1つの操り点が曲線全体に影響する(大域的サポート)ため、局所的な修正が効きにくい。B-スプラインは各操り点の影響範囲が局所に限られる(局所制御性)ため代替候補になる。

研究上の未解決課題として、(1) 滑らかさと俊敏さを場面に応じて動的に切り替える(危険回避時だけ高次に切り替える)適応的次数選択、(2) 制御信号契約のまま multi-modal(複数の可能な軌跡)を出す方法、(3) open-loop 指標に頼らない closed-loop での曲線パラメータ化の優位性の定量化、が挙げられます。

## 発展トピック・研究の最前線

曲線パラメータ化を multi-modal にする方向では、複数組の操り点(モード)とその確率を同時に出し、不確実性を表す研究が進んでいます。交差点で「直進」と「右折」のどちらもありうる場面で、平均をとると車道のど真ん中を突っ切る不自然な軌跡になってしまう(mode averaging 問題)のを避けるため、複数モードを別々の操り点群として出すのが標準になりつつあります。

また、Bézier の凸包性質を使って衝突回避制約を凸最適化に落とし込む「微分可能な安全層」を学習ヘッドの後ろに置く流れ(differentiable optimization layer, OptNet 系)も活発です。操り点が凸包に収まる性質を使うと、障害物との非衝突を操り点に対する線形制約として書け、学習可能なまま安全を保証しやすくなります。

世界モデル系([FutureState + JEPA損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0014_futurestate-jepa-loss))と組み合わせ、予測した未来状態の上で複数の候補軌跡を反実仮想ロールアウト(もしこの軌跡を走ったら世界はどうなるか、を頭の中でシミュレーション)して最良を選ぶ、という reactive(速い反射)層と slow(熟考)層の役割分離も研究テーマです。Bézier ヘッドはこの構成における「速くて滑らかな反射層」の有力な実装形態です。
