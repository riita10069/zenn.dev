# scheduled sampling によるワープ入力の切替

## ひとことで言うと
未来予測のためにBEVを「自車が未来にどれだけ動くか」でワープする手法では、その「未来の動き」を学習時と推論時で何から取るかが問題になります。学習時は正解（データに記録された本当の未来移動量）が使えますが、推論時には正解がなく、モデルが計画した軌道（予測）を使うしかありません。この食い違いを埋めるため、学習の最初は正解を使い、学習が進むにつれて少しずつモデル自身の予測へ切り替えていく――これがscheduled sampling（スケジュールドサンプリング、計画的な切り替え）です。

## 直感的な理解
自転車の練習を思い浮かべてください。最初は親が後ろを支えてくれます（正解の補助）。いつまでも支えてもらっていると、支え無しの本番でいきなり転びます。逆に最初から完全に独りだと、まだバランスが分からず何度もこけて、上達が遅くなります。賢いやり方は、最初はしっかり支え、徐々に手を離していくことです。

機械学習でも同じ構図が現れます。未来予測モデルを訓練するとき、ワープに必要な「未来の自車移動量」を、学習中ずっと正解で与えると本番（推論時、正解が無い世界）でいきなり弱くなります。逆に最初から自分の下手な予測で与えると、序盤の学習がぐちゃぐちゃになります。scheduled samplingは「最初は正解で支え、徐々に自分の予測へ手を離す」という、まさに自転車の練習と同じ戦略です。

## 基礎: 前提となる概念
用語を平易に説明します。

- **GT（ground truth、正解データ）**: 実際にそうだった、と記録されている真の値です。ここでは「自車が時刻 t から t+h までに本当にどれだけ動いたか（未来 ego-motion）」を指します。学習データには、後から振り返れば記録できます。
- **計画軌道（planned trajectory）**: モデルが「これから自分はこう走るつもり」と予測した未来の動きです。推論時にはこれしかありません。
- **ワープ（warp）**: いまのBEVを「未来の自車座標系」へ幾何的に貼り直す操作です。これに「未来の移動量」が必要になります（[ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp)）。
- **動的残差（Δ_dynamic）**: ワープで消しきれない、他エージェントの実運動。ネットが学ぶべき対象です（[ego起因の見かけ運動と動的運動の分離](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0023_ego-induced-vs-dynamic-motion)）。
- **teacher forcing（教師強制）**: 系列予測モデルの学習で、各ステップの入力に「自分の前の出力」ではなく「正解」を使うやり方です。学習は安定しますが、本番（正解が無い）との乖離を生みます。
- **train/test 不一致（exposure bias、露出バイアス）**: 学習時に見ていた入力と、本番で与えられる入力が違うことです。違うと、本番でモデルが想定外の入力に弱くなります。teacher forcingが典型的にこれを起こします。

ワープに使う未来移動量を「正解」と「予測」のどちらにするかには、本質的なジレンマがあります。

- **学習時にGT未来 ego-motion を使う**: 値が正確できれいなので、残差ネットが学ぶべき `Δ_dynamic` の正解もクリーンになります。しかし推論時にはこのGTが存在しません（未来はまだ起きていない）。本番では計画軌道で代用するしかなく、学習時（GT）と本番（予測）で入力の分布が変わる、つまりtrain/test不一致が生じます。
- **学習時にも計画軌道（予測）を使う**: 学習と本番で同じものを使うので一貫性は完璧です。しかし学習の序盤は軌道計画ネット（プランナー）がまだ下手なので、予測軌道はデタラメです。デタラメな移動量でワープすると、残差の正解がぐちゃぐちゃになり、学習が不安定・低速になります。

つまり「正確さ（GT）」と「一貫性（予測）」がトレードオフの関係にあります。scheduled samplingはこの2つの良いとこ取りをする方法です。

## 仕組みを詳しく
学習の進行（エポックやステップ）に応じて、ワープに使う移動量を「GT」から「予測」へ徐々に切り替えます。

```
学習序盤            学習中盤               学習終盤
│ GT 100%    →    GT 70% / 予測 30%   →   予測 100%
│ (きれいな正解で  (混ぜる)               (本番と同じ条件で
│  土台を作る)                            仕上げる)
└──────────────────────────────────────────────────→ 学習ステップ
```

実装としては、各サンプル（または各ホライズン）ごとに確率 p で「予測軌道を使う」、確率 (1-p) で「GTを使う」と決め、p を学習が進むにつれて0から1へ上げていきます。p の上げ方（スケジュール）には、線形に上げる、シグモイド状に上げる、指数的に上げる、といった選択肢があります（後述の原典がこれらを比較しています）。

数値例を出します。ホライズン（予測の時間先）を {1, 2, 3, 4} 秒、1Hz（1秒刻み）で予測するとします。学習開始直後（step 0）は p=0 で全部GT。学習が半分進んだら p=0.5 で半々。終盤（収束間際）は p=1 で全部プランナーの予測。こうすると、序盤はクリーンな信号でエンコーダと残差ネットの土台を作り、終盤は本番と同じ「予測駆動」の条件に慣らせます。

重要なのは「ワープに使うのは未来の移動量 `[t, t+h]` だ」という点です。過去の ego-motion 履歴は、未来移動量を積分（足し合わせて軌跡を作る）するときの初期条件にすぎず、ワープの主たる入力ではありません。だからこの切り替えは「未来をどう見積もるか（GTか予測か）」の話です。

もう1つ重要な関連論点が target alignment（ターゲットの座標系合わせ）です。ワープした予測は「未来のego座標系」で表されます。そのため、それと比べる正解（凍結したエンコーダが作る未来フレームの特徴、いわゆるJEPAターゲット）も同じ未来ego座標系で表さないと、座標系が食い違って学習信号が壊れます。具体的には、ターゲット側にも同じ未来移動量によるワープ（あるいは座標変換）を施し、予測とターゲットが同じ「未来の自車から見た世界」を共有するようにします。scheduled samplingでGTと予測を切り替える際は、ターゲット側のワープにも同じ移動量を使う、という整合が必須です。

## 手法の系譜と主要論文
scheduled samplingは系列予測の学習法として生まれ、世界モデルのロールアウトへ応用されてきました。

- **Williams & Zipser, "A Learning Algorithm for Continually Running Fully Recurrent Neural Networks", 1989**: teacher forcing の元アイデアです。学習時に各ステップの入力へ正解を使うことで、再帰ネットの学習を安定させました。scheduled samplingはこのteacher forcingと、自分の予測を入力に使うfree-runningの中間を取る、と位置づけると分かりやすいです。

- **Bengio et al., "Scheduled Sampling for Sequence Prediction with Recurrent Neural Networks", NeurIPS 2015**: scheduled samplingを提案した原典です。系列予測（次の単語や値を順に出すモデル）で、学習時はteacher forcing、推論時は自分の予測を使うために生じる不一致（exposure bias）を指摘しました。解決として、学習中に正解と自分の予測を確率的に混ぜ、その確率を徐々に予測寄りにスケジュールします。混ぜ方のスケジュール設計がヒューリスティック（経験則）で、学習目的が途中で変わるため理論的に扱いにくいのが難点です。

- **Hu et al., "MILE: Model-Based Imitation Learning for Urban Driving", NeurIPS 2022**: 世界モデルを未来へロールアウト（連続予測）する際、自分の予測を次の入力に使う構造で、trainとtestの一貫性が問題になります。自車運動を条件に未来を展開する設計で、「未来 ego-motion でワープする」設定と近く、scheduled samplingの考えが活きる典型例です。

- **Lamb et al., "Professor Forcing", NeurIPS 2016**: teacher forcing と free-running の隠れ状態の分布を敵対的に近づける手法で、exposure bias 緩和の別アプローチです。scheduled samplingの素朴なスケジュールに対する改良の系譜として知られます。

系譜をまとめると、teacher forcing（1989）が安定した学習法を確立し、その副作用（exposure bias）を Scheduled Sampling（2015）が確率的混合で緩和し、Professor Forcing（2016）が分布マッチングで洗練し、MILE（2022）のような世界モデルのロールアウトへと応用が広がった、という流れです。

## 論文の実験結果（定量データ）
Scheduled Sampling（Bengio et al., 2015）は、画像キャプション生成（画像を見て説明文を生成する課題）や音声認識、構文解析などで評価されました。代表的なのは MS COCO の画像キャプション課題で、評価指標 BLEU-4（生成文と正解文の4語連続の一致度。0〜1または0〜100で表し、高いほど良い）や CIDEr（人間の複数参照文との合致度を測る指標）で、常にteacher forcingするベースラインを上回ったと報告されています。この論文は2015年のMS COCOキャプションチャレンジで上位の成績を収めたことでも知られ、exposure biasの緩和が実タスクの生成品質を押し上げることを示しました。

アブレーションの要点は「スケジュールの形と速さ」です。論文では、確率 p を線形・指数・逆シグモイド（inverse sigmoid）で減衰させる3パターンを比較し、急に予測寄りへ切り替えると学習が不安定になり、緩やかに切り替えるほうが安定して性能が出ることを示しました。逆に、最後までteacher forcingのまま（p を上げない）だと、学習時の数値は良くても本番（自分の予測で回す）で性能が落ちます。これはまさに「学習時のきれいさ」と「本番との一貫性」のトレードオフを定量的に表しています。

世界モデル側では、MILE（Hu et al., 2022）が CARLA（自動運転シミュレータ）の閉ループ評価で、世界モデルを潜在空間でロールアウトしながら運転する設計の有効性を示しました。閉ループ評価とは、モデルの出力が次の入力を変える（自分の運転で世界が変わる）状況で性能を測るもので、ここではexposure biasが最も顕著に効きます。学習時と推論時でロールアウトの一貫性を保つ工夫が、走行成功率（ルート完走率や違反の少なさ）に直結することが報告されています。

これらが示すメッセージは明快です。「学習と本番で入力の出どころが違う」状況を放置すると、特に自分の出力が次の入力を変える設定（生成、ロールアウト、ワープ駆動の未来予測）で性能が崩れます。scheduled samplingは、そのギャップを学習中に少しずつ埋めることで、本番性能を底上げします。

## メリット・トレードオフ・限界
メリット
- 学習序盤のクリーンな信号（GT）と、本番との一貫性（予測駆動）を両立できます。
- train/test不一致（exposure bias）を緩和し、本番でプランナーの予測がそのままワープに使われても破綻しにくくなります。
- 既存のホライズン設計（たとえば1Hz・{1,2,3,4}秒）にそのまま乗せられ、追加の機構を要しません。

トレードオフ・限界
- 切り替えスケジュール（いつ・どのくらいの速さで予測寄りにするか）はヒューリスティックで、調整が必要です。早すぎると不安定、遅すぎると不一致が残ります。
- 学習目的が途中で変わるため、収束の解析がしにくく、再現性の管理に注意が要ります。
- データ側で「未来 ego-motion」を別途用意する整備が前提です。多くのデータセットは将来の軌道目標は持っていても、ワープに必要な「フレーム間の相対移動量（並進・回転）」を直接持っていないことがあり、前処理での導出が要ります。
- target alignment（ターゲットも未来ego座標系へ揃える）を併せて実装しないと、ワープしても監督信号が噛み合いません。予測とターゲットで使う移動量を必ず一致させる必要があります。

未解決の課題として、最適なスケジュールをデータから自動で決められないか、ホライズンごとに別々のスケジュールを持たせるべきか、計画軌道の不確実性（複数仮説）をワープにどう反映するか、などが研究対象として残っています。

## 発展トピック・研究の最前線
近年の世界モデルは、自分の予測を何ステップも先まで展開する「長期ロールアウト」を行うため、exposure biasの問題がより深刻になります。これに対し、学習時から数ステップ先まで自分の予測で回して誤差累積に慣らす「マルチステップ学習」や、ロールアウトの軌跡分布を学習と推論で一致させる正則化が研究されています。また、計画軌道を点ではなく分布（複数の可能な未来）として持ち、その分布からサンプリングしてワープする確率的scheduled samplingも検討されています。生成系の世界モデル（拡散・自己回帰）では、teacher forcing相当の「正解条件付け生成」と「自己生成条件付け」のギャップを埋める手法が活発に議論されており、scheduled samplingはその古典的かつ実用的な出発点として今も参照されています。
