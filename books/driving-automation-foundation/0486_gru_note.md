# GRU (Gated Recurrent Unit)

## ひとことで言うと
GRU は、[LSTM (Long Short-Term Memory)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0490_lstm_note) が持っていた「長い系列でも昔の情報を覚えていられる」という良さを保ちつつ、ゲート(情報の蛇口)を3つから2つに減らし、セル状態と隠れ状態を1本にまとめて構造を簡略化した再帰ユニットです。部品が少ないぶんパラメータが減り学習が速くなるため、小〜中規模のデータや計算資源が限られる場面で LSTM の有力な代替として広く使われています。

## 直感的な理解
LSTM は「覚える・忘れる・出す」を3つのゲートで細かく制御する、いわば多機能で高性能な記憶装置でした。しかし機能が多いぶん部品(重み行列)も多く、その全部が本当に必要なのか、という疑問が残ります。実際、[LSTM (Long Short-Term Memory)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0490_lstm_note) の大規模アブレーション研究(Greff et al., 2017)は「忘却ゲートと出力活性化以外はかなり冗長で、簡略化の余地がある」と報告しました。

GRU はこの「冗長さ」を削ぎ落とした設計です。発想は2つ。第一に、LSTM の「忘却ゲート(古い記憶をどれだけ捨てるか)」と「入力ゲート(新しい情報をどれだけ書き込むか)」は、本来「古いものを捨てた分だけ新しいものを入れる」という連動関係にあるはずだ、ならば1つのゲートで「古いものと新しいものの配分」を決めれば足りる、と考えました。これが更新ゲートです。第二に、LSTM が長期記憶用の専用レーン(セル状態)を別建てしていたのを、隠れ状態1本に統合しました。結果としてゲートは2つ、重み行列は3組になり、LSTM のおよそ4分の3の規模になります。

比喩で言えば、LSTM が「入庫・出庫・廃棄」を別々の係員で管理する大きな倉庫だとすれば、GRU は「古い在庫を何割残し、新しい在庫を何割入れるか」を1人の係員がスライダー1本で決める小さな倉庫です。倉庫の出し入れの自由度は少し減りますが、運用が単純で速く、たいていの荷物量ならこれで十分にさばける、というのが GRU の立ち位置です。

## 基礎: 前提となる概念
GRU を理解するには [LSTM (Long Short-Term Memory)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0490_lstm_note) の前提(再帰ネットワーク、隠れ状態、勾配消失問題、ゲートとシグモイド関数)を知っておくと早いです。簡単に再掲します。

再帰ネットワーク(RNN)は系列を1ステップずつ処理し、内部メモリである隠れ状態 `h_t` を毎時刻更新するネットワークです。素朴な RNN は勾配消失(誤差を時間方向にさかのぼると同じ重み行列の繰り返し掛け算で勾配がほぼ0になり、遠い過去を学べなくなる現象)に苦しみました。LSTM はセル状態という足し算更新のレーンとゲートでこれを緩和しました。

ゲートとは、シグモイド関数 `σ`(出力を0〜1にする関数、`σ(x)=1/(1+e^{-x})`)で作る「情報を何割通すか」の蛇口です。0で遮断、1で全通し。要素ごとの掛け算 `⊙`(アダマール積)で情報量を調整します。GRU もこのゲートの考え方をそのまま使いますが、その数と配線が LSTM と違うだけです。なぜ「足し算更新+ほぼ1のゲート」が勾配消失を防ぐのかという核心は LSTM と共通なので、そこを押さえておくと GRU の動作原理は半分理解できたようなものです。

## 仕組みを詳しく
GRU は隠れ状態 `h_t` を1本だけ持ち回ります(LSTM のような別建てのセル状態はありません)。各時刻 `t` で2つのゲートを計算します。

```
更新ゲート: z_t = σ(W_z·[h_{t-1}, x_t] + b_z)   前の状態を残す割合 vs 新候補を取る割合
リセットゲート: r_t = σ(W_r·[h_{t-1}, x_t] + b_r) 候補を作るとき過去をどれだけ参照するか
```

`W_z, W_r` は学習される重み行列、`[h_{t-1}, x_t]` は前のメモリと今の入力の連結、`z_t` と `r_t` はそれぞれ0〜1の値が並んだベクトルです。次に、新しい隠れ状態の候補を作ります。

```
h̃_t = tanh(W·[r_t ⊙ h_{t-1}, x_t] + b)
```

ここでリセットゲート `r_t` が効きます。`r_t` が0に近い要素では、その部分の過去の記憶 `h_{t-1}` を無視して、今の入力 `x_t` だけから候補を作り直します(過去をリセット)。1に近ければ過去をしっかり参照します。たとえば文の切れ目で文脈をリセットしたいときに `r_t≒0` が役立ちます。最後に更新ゲートで古いものと新しいものを混ぜます。

```
h_t = (1 - z_t) ⊙ h_{t-1}  +  z_t ⊙ h̃_t
```

この最後の1本が GRU の核心です。`z_t` が0に近ければ、前の状態 `h_{t-1}` をほぼそのまま次へ流します(`(1-z_t)≒1`、`z_t≒0`)。この「素通りレーン」が、LSTM のセル状態と同じく勾配消失を防ぎます。誤差を時間方向にさかのぼるとき、`z_t≒0` の経路では `h_t ≒ h_{t-1}` なので `∂h_t/∂h_{t-1} ≒ 1` となり、勾配がほぼ減衰せずに伝わるのです。LSTM が専用のセル状態でやっていたことを、GRU は隠れ状態1本の中の「素通り項」で実現している、と理解できます。更新ゲートが「忘却ゲートと入力ゲートを1本に束ねた `(1-z, z)` の配分」になっている点が、LSTM との一番の違いです。

tensor shape の数値例を示します。入力系列 `[B, T, D]`(B=8、T=64、D=256、つまり8サンプル・各64ステップ・各ステップ256次元)を隠れ次元 256・1層・単方向の GRU に通すと、各時刻の出力 `[8, 64, 256]` と、最終隠れ状態 `h_n`(形 `[1, 8, 256]`、先頭の1は層数×方向数)が得られます。系列全体を1個の要約ベクトルにしたいときは `h_n` を `[8, 256]` に潰して使います。これは「過去の履歴を畳んで1個の文脈ベクトルにする」用途で頻出のパターンで、軌道予測などで「観測履歴を1個の状態に符号化する」のに使われます。

パラメータ数は、ゲート2つ+候補1つで重み行列が3組です。入力 D・隠れ H とすると `3 × (H×(H+D)+H)` で、LSTM の4組(`4 × ...`)に対しておよそ4分の3になります。D=256, H=256 なら GRU が約 394000、LSTM が約 525000 パラメータです。

1ステップの流れを ASCII 図にすると次の通りです。

```
h_{t-1} ──────────────────────────┐ (× (1 - z_t))
                                   ▼
x_t ─► [z_t][r_t] ─► (r_t⊙h_{t-1}) ─► tanh ─► h̃_t ─►(× z_t)─►(+)─► h_t
```

## 手法の系譜と主要論文
GRU は機械翻訳の研究の中から生まれました。

Cho, van Merriënboer, Gulcehre, Bahdanau, Bougares, Schwenk, Bengio「Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation」(2014, EMNLP)が出典です。この論文は2つの貢献をしました。1つは RNN による encoder-decoder の枠組み([Seq2Seq Encoder-Decoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0497_seq2seq-encoder-decoder) の基礎)、もう1つが GRU(論文中では明示的に「GRU」とは呼ばず、新しいゲート付き隠れユニットとして提案)です。GRU を使った encoder-decoder が、統計的機械翻訳のフレーズスコアリングを改善できることを示しました。

Chung, Gulcehre, Cho, Bengio「Empirical Evaluation of Gated Recurrent Neural Networks on Sequence Modeling」(2014, arXiv:1412.3555)は、GRU・LSTM・素朴な tanh-RNN を複数タスクで体系的に比較した、トレードオフ理解の最重要論文です。両ゲート付きユニット(GRU と LSTM)が素朴な RNN を明確に上回ること、そして GRU と LSTM はおおむね同等で、どちらが勝つかはデータとタスクに依存することを報告しました。

Jozefowicz, Zaremba, Sutskever「An Empirical Exploration of Recurrent Network Architectures」(2015, ICML)は、進化的探索(突然変異でゲート構造をランダムに変え、性能の良いものを残す)で1万以上のゲート構造を自動生成・評価し、GRU と LSTM が人間設計として優秀であることを確認しました。発見された優良な変種も GRU/LSTM と僅差で、人手設計を大きく超えるものは見つかりませんでした。同時に、LSTM の忘却ゲートのバイアスを大きめ(例えば1.0)に初期化すると性能が上がるという、今も使われる実務知見を示しました。

その後 Transformer(Vaswani et al., 2017)が再帰を注意機構で置き換え、長系列の主役は移りましたが、GRU は軽量・低遅延が要る場面(組み込み機器、リアルタイム処理、小データ、強化学習の方策ネットワークなど)で現役です。

## 論文の実験結果(定量データ)
Chung et al. (2014) は2種類のタスクで定量比較しました。1つは多声音楽モデリング(Nottingham, JSB Chorales, MuseData, Piano-midi の4データセット、評価は負の対数尤度 NLL で低いほど良い。NLL はモデルが正解系列にどれだけ高い確率を与えたかの指標で、小さいほどデータをよく説明できている)、もう1つは音声信号モデリング(Ubisoft の音声データ、同じく NLL)です。結果として、GRU と LSTM はデータセットごとに勝ったり負けたりで、明確な総合勝者はいませんでした。例えばあるデータセットでは GRU がわずかに良く、別のデータセットでは LSTM が良い、という具合です。ただし両者とも素朴な tanh-RNN を一貫して上回りました。とくに音声モデリングでは、GRU が同じパラメータ予算で LSTM と同等以上の収束速度(同じ更新回数・同じ実時間でより低い NLL へ到達)を示した点が報告されています。これは「パラメータが少ないぶん、同じ計算予算でより多く・速く学べる」という GRU の実利を裏づけます。

機械翻訳の文脈では、Cho et al. (2014) と、同時期に注意機構を導入した Bahdanau, Cho, Bengio (2015, ICLR、GRU ベースの注意付き encoder-decoder)が、WMT 英仏翻訳で BLEU(翻訳の n-gram 一致率、高いほど良い)を大きく改善しました。注意付き GRU モデルは、長い文ほど注意なしモデルに対する優位が広がることが示され、GRU が翻訳の実用モデルとして十分な表現力を持つことが確認されました(具体的な数値と長文での挙動は [Seq2Seq Encoder-Decoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0497_seq2seq-encoder-decoder) を参照)。

これらを総合すると、定量的な結論は「GRU と LSTM の性能差はタスク依存で小さく、計算予算が限られるなら GRU の方が有利になりやすい」というものです。アブレーション的に見ると、GRU からリセットゲートを抜くと短期的な依存(直前の文脈で候補をリセットする能力)の表現が落ち、更新ゲートを抜くと長期記憶が保てなくなる(`h_t = h̃_t` となり素朴な RNN に退化する)ため、2つのゲートはいずれも必要だと理解されています。Greff et al. (2017) の LSTM 側のアブレーションでも、入力ゲートと忘却ゲートを束ねる(GRU の更新ゲートに相当する coupled gate)変種が標準 LSTM とほぼ同等の性能を示しており、GRU の設計が筋の良い簡略化であることが間接的に裏づけられています。

## メリット・トレードオフ・限界
メリットは、LSTM とほぼ同等の長期依存学習能力を、より少ないパラメータ・計算量で実現できる点です。ゲートが2つで構造が単純なため、小規模データで過学習しにくく学習が速い傾向があります。ライブラリでも組み込みの GRU 層を1行で使え、実装が容易です。「過去の履歴を1個の文脈ベクトルに畳む」ような用途では、最終隠れ状態 `h_n` をそのまま要約として使えるため扱いやすいです。メモリ消費も LSTM より少なく、エッジデバイスやバッチサイズを大きく取りたい学習で有利です。

トレードオフと限界として、セル状態を分離しないぶん、極端に長く複雑な依存関係では LSTM に僅かに劣ることがあります(ただしタスク依存で、明確な優劣はない、というのが実験的なコンセンサスです)。依然として逐次計算なので、長系列では Transformer の並列処理に速度で劣ります。ゲートが少ないぶん、記憶の読み書きの自由度は LSTM より少し低く、出力ゲートに相当する「セルの中身を選んで出す」制御がない点が表現力差として現れることがあります。具体的には、隠れ状態がそのまま出力にも使われるため、「内部に保持はするが今は外に出さない」という使い分けが LSTM ほど柔軟にはできません。

研究上の課題は LSTM と共通で、逐次性に由来する遅さと、数百〜数千ステップ級の超長期依存の難しさです。これらは注意機構や状態空間モデルが補う領域です。

## 発展トピック・研究の最前線
GRU の発展としては、双方向化([双方向RNN](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0482_bidirectional-rnn))でオフライン系列ラベリングの精度を上げる、注意機構と組み合わせて [Seq2Seq Encoder-Decoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0497_seq2seq-encoder-decoder) の固定長ボトルネックを解消する、といった方向が古典的です。GRU の変種として、計算をさらに削った Minimal Gated Unit(MGU、ゲートを1つに減らす)や、リセットゲートの配線を変えた版なども提案されましたが、性能は GRU と僅差で、GRU が「簡略化の良いバランス点」であることが繰り返し確認されています。

近年は、線形時間で長系列を扱える状態空間モデル(Mamba 等)や、ゲート付き線形再帰(GLA, RWKV など)が GRU/LSTM の「一定メモリ・逐次更新」という利点を継承しつつ並列学習を可能にしており、ゲート付き再帰の系譜の現代的な到達点と言えます。GRU の「素通りレーン(更新ゲートによる残差的な状態運搬)+少数ゲート」という設計は、これら最新モデルのゲーティング機構を理解する出発点になります。実際、これらのモデルの多くは「入力依存のゲートで状態をどれだけ保持するか決める」という、GRU の更新ゲートと本質的に同じ発想を線形・並列化可能な形で再定式化したものです。
