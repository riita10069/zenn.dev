# BEVFormer 風 空間クロスアテンション(SCA)

## ひとことで言うと
上空から見た格子(BEV)の各マスが、「自分の場所を映しているカメラはどれか」を幾何計算で特定し、そのカメラ画像から情報を吸い上げる仕組みです。あるマスを見ているカメラが3台あれば、その3台から取った情報を平均します(これが 1/|V_hit| 重み付け)。これにより、別々の方向を向いた複数カメラの映像を、1枚の統一された上空図にまとめられます。カメラだけで3D空間理解をするBEV手法の中核技術です。

## 直感的な理解
複数の防犯カメラの映像から、ある部屋の真上から見た見取り図を1枚作る場面を想像してください。素朴に映像を貼り合わせると、遠近やレンズの歪みで物の位置がずれます。代わりに「見取り図のこのマスは、現実のどの位置か」を決め、「その位置はどのカメラのどこに写っているか」を逆算し、写っているカメラからだけ色を取れば、正しい位置に正しい情報を置けます。後ろを向いたカメラに前方の情報を間違って拾わせる、という事故も防げます。

これが空間クロスアテンションの発想です。BEVの各マスが「自分は現実空間のここにいる」と知っていて、その位置を各カメラ画像へ射影(投影)して「どのカメラに、どのピクセルに写っているか」を計算し、写っているカメラから特徴を集める。見えていないカメラは無視する。複数カメラに写っていれば平均する。この「物理的に正しい対応づけ」が、素朴な貼り合わせとの決定的な違いです。素朴な方法はカメラの並びに依存し歪みも残りますが、SCAは幾何に基づくので、カメラの台数や向きが変わっても一貫した上空図を作れます。

## 基礎: 前提となる概念

用語を順に説明します。

- **アテンション(attention)**: たくさんある情報の中から、いま関係のあるものだけに重みをつけて取り出すニューラルネットの仕組み。文章を読むとき、いま考えている単語に関係する別の単語に注目するイメージです。数式では、クエリ(問い)とキー(情報の見出し)の類似度で重みを作り、バリュー(情報の中身)を重み付き和で取り出します。式で書くと `Attn(Q,K,V) = softmax(QKᵀ/√d) V`。

- **クロスアテンション(cross-attention)**: 注目する側(クエリ)と注目される側(キー/バリュー)が別物のアテンション。ここではクエリがBEVの格子、注目される側がカメラ画像の特徴です。

- **空間クロスアテンション(Spatial Cross-Attention, SCA)**: BEVFormer が提案した、BEVの各格子マスがカメラ画像の「空間的に対応する場所」に注目するクロスアテンション。「空間」とは3次元の幾何対応を使うという意味です。普通のクロスアテンションが全ピクセルとの類似度を計算するのに対し、SCA は幾何で対応先を絞るので計算が桁違いに軽くなります。

- **射影(projection)**: 3次元空間の点が、カメラ画像の2次元のどこに写るかを計算する操作。カメラの校正行列(内部パラメータ + 外部パラメータ)を使います。詳細は [カメラパラメータ(キャリブレーション)入力](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0941_camera-params-calibration-input)。

- **校正行列(camera_params)**: 3次元のego座標(車基準の座標)を、カメラ画像の2次元ピクセル座標に変換する行列。形は `[B, V, 3, 4]`(B=バッチ、V=カメラ台数)。内部パラメータ(焦点距離・画像中心)と外部パラメータ(カメラの取り付け位置・向き)を合わせたものです。

- **grid_sample**: 指定した連続座標の特徴を、周囲のピクセルから補間して取り出す関数。整数でない座標(たとえば「右から12.3ピクセル目」)でも、隣接ピクセルを混ぜて滑らかに値を読めます(双線形補間)。

なぜSCAが必要か。車には複数カメラが別々の方向を向いて付いています。前・後・側方それぞれが別々の2次元画像を撮りますが、運転の判断は「車の周囲を真上から見た1枚の地図」でしたい。バラバラの画像を1枚の統一上空図にまとめる、その物理的に正しい方法がSCAです。

## 仕組みを詳しく

設定例として bev_h=200, bev_w=200, embed_dim=256, num_points_in_pillar=4, バッチ B, ビュー数 V とします。

### ステップ1: 学習可能なBEVクエリ

格子のマスは 200×200 = 40000 個。各マスに 256 次元のベクトルを割り当てます。

```python
self.bev_queries = nn.Embedding(bev_h * bev_w, embed_dim)  # [40000, 256]
```

このベクトルは学習で更新され、「このマスはカメラの何を見たいか」を表すようになります。最初はランダムですが、学習が進むと「道路面の見え方」「縁石の特徴」などに対応するクエリへ育ちます。位置エンコーディング(各マスがどこか)も足され、空間的な意味を持つようになります。

### ステップ2: 3次元の参照点(ピラーサンプリング)

各マスは地面上の1点ですが、高さ方向(上下)も考慮するため、1マスにつき高さ違いの点を num_points_in_pillar=4 個立てます。これを「ピラー(柱)」と呼びます。柱の4点は地面付近から高い位置(例: -5m〜3m など)までをカバーします。形は `[N, num_z, 3]`(N=40000 マス、num_z=4、3 は xyz 座標)。これは学習しない固定値(バッファ)として登録されます。高さ方向にサンプルするのは、物体が地面だけでなく上方にも広がる(トラックの荷台、信号機)からです。1点だけだと「地面の高さの物しか拾えない」ので、柱状に複数高さを取って3D構造を捉えます。

### ステップ3: 3次元点をカメラ画像に射影

各3次元点が、各カメラ画像のどこ(2次元のどのピクセル)に写るかを校正行列で計算します。

```
ref_2d: [B, V, N, num_z, 2]   # 各点の各カメラ上での2D位置(0〜1に正規化)
mask:   [B, V, N, num_z]      # その点がそのカメラに「写っているか」の可視マスク
```

可視マスクが核心です。3次元点を射影したとき、画像の枠外に出たり、カメラの後ろ側(深度が負)だったりすると「見えていない」のでマスクを False にします。これにより、後方のマスが前カメラの情報を拾う事故を防ぎます。深度が負(カメラの背後)の点を射影すると、数学的には画像内の座標が出てしまうことがあるので、深度の符号チェックは必須です。

### ステップ4: 見えているカメラから特徴をサンプリング

各カメラ特徴に対し、`F.grid_sample` で ref_2d の位置の特徴を補間して抜き取ります。BEVFormer の原論文はここで変形可能アテンション(deformable attention)という専用CUDAカーネルを使い、クエリから予測したオフセット先の少数の点を読みます。これに対し、移植性(どんな環境でも動くこと)を優先する実装では `grid_sample` で代用することがあります。grid_sample は標準的なPyTorch関数なので特殊なCUDAカーネルが要りません。表現力では deformable attention にやや劣る可能性がありますが、依存が軽い利点があります。

### ステップ5: 可視カメラ数で平均(1/|V_hit|)

ここがSCAの数学的核心です。あるマスを見ているカメラの集合を V_hit、その台数を |V_hit| と書きます。論文の式は次のとおり:

```
SCA(Q_p, F) = (1 / |V_hit|) × Σ_{i ∈ V_hit} Σ_{j=1}^{N_ref} DeformAttn(Q_p, P(p, i, j), F_i)
```

記号の意味: `Q_p` はマス p のクエリ、Σ は「足し合わせる」、i ∈ V_hit は「見えているカメラだけについて」、`F_i` はカメラ i の特徴、`P(p,i,j)` はマス p の j 番目の参照点をカメラ i に射影した位置。要するに「見えているカメラだけ足して、その台数で割る(平均する)」。

具体例: あるマスが3台のカメラに写っていれば3台ぶんを平均、1台にしか写っていなければそのままその1台ぶん、まったく見えていなければ寄与ゼロ。これにより、よく見える場所は多くの証拠で、見えにくい場所は少ない証拠で、と自然に重みづけされます。重なり領域(隣り合うカメラの視野が重なる部分)で値が二重に大きくならないよう正規化する役割も果たします。もし `1/|V_hit|` で割らずに単純合計すると、3台に写る重なり領域だけ特徴の大きさが3倍になり、後段の学習が不安定になります。

実装では二段構成がよく使われます。高さ方向(柱の4点)の重みは `attention_weights` を softmax で per-camera に決め、カメラ間の重みはこの 1/|V_hit| 平均で決める。この設計の重要な副産物として、カメラ間を学習重みではなく可視数の平均で扱うため、カメラ台数ぶんの固定パラメータが不要になり、台数動的化([BEV融合の num_views ランタイム動的化](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1030_bev-runtime-dynamic-num-views))がやりやすくなります。

### 時系列自己アテンション(Temporal Self-Attention)との併用

BEVFormer はこの空間方向の集約に加え、過去フレームのBEVを参照する時系列自己アテンションを持ちます。過去のBEVを自車運動ぶんだけずらして現在のBEVクエリと突き合わせることで、1フレームでは分からない「物体の速度」や「一時的に隠れた物の復帰」を捉えます。SCA(空間)と TSA(時間)の2本立てが BEVFormer の骨格です。

### 出力

V が何であれ、出力は常に `[B, embed_dim, bev_h, bev_w]` = `[B, 256, 200, 200]`。カメラ台数に依存しない統一表現です。

```
[入力] V 枚のカメラ特徴 [B, V, C, h, w]
   │  各 BEV マスを 3D に持ち上げ → 各カメラへ射影 → 可視カメラだけ grid_sample
   ▼
[集約] 1/|V_hit| 平均(見えているカメラで平均)
   ▼
[出力] 統一 BEV 特徴 [B, 256, 200, 200]   ← V に依存しない
```

## 手法の系譜と主要論文

カメラのみBEV構築の系譜です。大きく「深度を陽に持ち上げる派(LSS系)」と「クエリで吸い上げる派(BEVFormer系)」に分かれます。

- **持ち上げ派の源流: LSS / Lift-Splat-Shoot (Philion & Fidler, ECCV 2020, arXiv:2008.05711)**。各カメラピクセルに深度分布を予測し、3D点群に「持ち上げ(lift)」、BEV格子に「撒く(splat)」手法。深度を陽に推定するのが特徴で、後の多くのBEV手法の土台になりました。動機は「2D画像を物理的に正しくBEVへ落とす」、トレードオフは「深度推定の誤差がBEVに直接効く」。

- **持ち上げ派の実用化: BEVDet (Huang et al., 2021, arXiv:2112.11790)**。LSS をベースに3D物体検出を実用速度で回せるようにしたパイプライン。深度推定→BEVプーリング→検出ヘッドという構成を整理し、LSS系を検出ベンチマークに乗せました。

- **クエリ派の前身: DETR3D (Wang et al., CoRL 2021, arXiv:2110.06922)**。3D物体クエリを各カメラに射影して特徴を引く手法。BEVの密な格子は作らず疎な物体クエリで集約しますが、「3Dクエリを2D画像に射影して吸い上げる」というBEVFormerの中核アイデアの直接の前身です。

- **クエリ派の確立: BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270)**。空間クロスアテンションの出典。提案内容は「学習可能BEVクエリ + 各クエリの3次元参照点をカメラに射影 + 可視カメラだけ deformable attention で集約 + 1/|V_hit| 平均」に、過去フレームのBEVを参照する時系列自己アテンション(Temporal Self-Attention)を加えたもの。動機は「LiDARなしカメラのみで密なBEV特徴を作りたい」「複数カメラの幾何整合を取りたい」。深度を陽に持たず、アテンションで暗黙に対応づける点がLSSと対照的です。

- **射影なし派: PETR (Liu et al., ECCV 2022, arXiv:2203.05625)**。明示的な射影の代わりに、カメラの3D位置エンコーディングを特徴に足し、アテンションで暗黙に対応づける手法。SCA とは対照的で、射影行列を陽に使わない分だけ校正への頑健性の性格が違います。

- **アテンションの基礎: Deformable DETR (Zhu et al., ICLR 2021, arXiv:2010.04159)**。BEVFormer の deformable attention の基礎。注目点の周辺の少数のサンプル点だけに注目して計算を軽くする発想。SCAの `sampling_offsets`(サンプル点をクエリから予測する)はこの思想を継いでいます。

- **検出器との融合で強化: BEVFormer v2 (Yang et al., CVPR 2023, arXiv:2211.10439)**。透視ビュー(画像そのまま)の補助検出ヘッドを足してバックボーンを鍛え直し、BEVFormer の精度をさらに引き上げた続編。

- **下流統合: UniAD (Hu et al., CVPR 2023, arXiv:2212.10156)**。BEVFormer をBEVエンコーダとして採用し、その上に検出・追跡・地図・予測・計画を積む。SCAで作ったBEVが下流タスク全体の土台になることを示しました。

## 論文の実験結果(定量データ)

- **BEVFormer (nuScenes, arXiv:2203.17270)**。nuScenes は6カメラ・LiDARを積んだ車で都市を走った大規模データセット。3D物体検出の総合指標 **NDS(nuScenes Detection Score、位置・大きさ・向き・速度の誤差を総合した0〜1のスコア、高いほど良い)** で約0.56、**mAP(検出の正確さ)** で約0.42を達成。これはカメラのみ手法として当時の最高水準で、前身の DETR3D(NDS 0.4台)を明確に上回りました。BEVセグメンテーション(車線・走行可能領域を上空図で塗り分けるタスク)でも高い IoU(予測と正解の重なり率)を報告しています。

- **アブレーション(BEVFormer 論文)**。(1) 時系列自己アテンション(過去BEVの参照)を外すと NDS が数ポイント低下し、特に速度推定の誤差(**mAVE**, mean Absolute Velocity Error)が大きく悪化。動いている物体の速度は1フレームだけでは分からないので、時系列が効くことが定量的に示されました。(2) 空間クロスアテンションを単純な特徴貼り合わせに置き換えると性能が落ち、幾何に基づく可視カメラ選択が効いていると分かります。(3) BEVクエリの解像度を上げると精度が上がる一方、計算量とメモリが増えるトレードオフも示されました。

- **解像度とメモリのトレードオフ(一般的実測)**。高解像度BEV(例: 200×200 を超える格子)はメモリ・計算を大きく食います。実装によっては数百×数百の高解像度BEVを bf16(16bit浮動小数の混合精度)で回しても、バッチサイズ4程度で20GB超のGPUメモリを消費することが報告されています。車載ハードウェアでのリアルタイム推論を考えると、解像度・バッチ・精度のバランスが実務上の制約になります。

数値の意味の補足: NDS は単一指標で検出の総合力を測れる便利な指標ですが、内訳は mAP と5つの誤差項(位置 mATE、大きさ mASE、向き mAOE、速度 mAVE、属性 mAAE)の加重和です。平均なので「位置は正確だが速度が苦手」のような偏りは隠れます。だからアブレーションでは mAVE(速度誤差)など個別指標も見て、どの要素が何に効いたかを切り分けます。

## メリット・トレードオフ・限界

- **メリット**: 複数カメラを物理的に正しく1枚のBEVに統合できます。可視カメラだけ使うので、見えないカメラの誤情報を拾いません。
- **メリット**: 出力がカメラ台数に依存しないので、多データセット学習の土台になります([カメラ数が異なる多データセット学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1212_multi-dataset-training-homogeneous-batch))。
- **メリット**: grid_sample 採用版は特殊なCUDAカーネルが不要で移植性が高い(deformable attention版は速度・表現力で有利だが環境依存)。
- **トレードオフ**: 校正行列(camera_params)が必要で、ずれると射影位置が狂います。校正誤差への頑健性はSCAの弱点で、PETR系の暗黙対応づけと比較されます。数cm/数度の校正ずれでも遠方では大きなBEV位置ずれになります。
- **トレードオフ**: 高解像度BEVはメモリ・計算が重く、車載リアルタイム推論が課題。
- **トレードオフ**: grid_sample は deformable attention より注目点の自由度が低く(固定の射影点しか読まない)、表現力でやや劣る可能性。
- **限界**: |V_hit|=0(どのカメラにも写らない)マスは寄与ゼロになり、死角(車体の真下、近接の遮蔽)の情報は補えません。これはカメラ配置の物理的限界で、LiDAR融合やマップBEV([マップBEVと知覚BEVの違い](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1044_map-bev-vs-perception-bev))で補う必要があります。
- **未解決課題**: 深度の曖昧さ。SCAは深度を陽に推定しないため、遠方や薄い物体の奥行き推定が苦手な傾向。LSS系(陽な深度)とBEVFormer系(暗黙対応)のどちらが良いかは依然として研究上の論点です。

## 発展トピック・研究の最前線

- **時系列の長期化**: 単フレームBEVから、複数過去フレームを再帰的に融合する long-temporal BEV へ。速度・加速度・遮蔽復帰の推定が改善します。
- **深度の明示化との融合**: BEVFormer系のクエリ集約に、LSS系の陽な深度監督を足すハイブリッド。深度監督(LiDAR で正解深度を与える)を加えると幾何精度が上がる報告があります。
- **疎なBEV/効率化**: 全格子を密に計算する代わりに、物体がありそうな領域だけ疎に計算して車載コストを下げる方向。
- **校正非依存化**: 校正行列への依存を減らし、PETR的な学習ベースの対応づけや、校正を自己推定する仕組みと組み合わせる研究。
- **世界モデル化**: SCAで作ったBEVを潜在状態として未来へ展開し、将来のBEVを予測する世界モデル(MILE系)。計画の先読みに直結します。
