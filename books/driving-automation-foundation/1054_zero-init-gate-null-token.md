# ゼロ初期化ゲートと学習可能 null トークン

## ひとことで言うと

すでに動いているニューラルネットに新しい入力(ここでは地図情報など)を足すとき、いきなり全力で混ぜると学習が不安定になったり、入力が欠けた瞬間に計算が壊れたりします。これを防ぐ2つの安全装置が「ゼロ初期化ゲート」と「学習可能 null トークン」です。前者は「学習の最初は新しい情報を全く効かせず、必要なぶんだけ徐々に効かせる」仕組み、後者は「入力要素がゼロ個でも計算が壊れない(NaN を出さない)ようにする」ダミーの要素です。どちらも特定の応用に限らず、新しい枝を既存モデルへ安全に追加する一般的な設計パターンです。

## 直感的な理解

新しいモジュールは、学習の最初はランダムに初期化されているので、出力はほぼノイズです。このノイズを、すでにそれなりに学習が進んだネットワークの出力にいきなり足し込むと、最初の数ステップで全体の出力が大きく乱れ、損失が跳ね上がり、せっかく学習済みの部分まで巻き込んで崩れることがあります。新規モジュールは最初「ノイズ製造機」なので、いきなり信用してはいけません。そこで「最初は新しい枝の影響をゼロにし、学習が進むにつれてモデル自身が必要な分だけ蛇口を開ける」という発想が生まれます。これがゼロ初期化ゲートです。蛇口を全開で始めるのではなく、閉じた状態から少しずつひねる、というイメージです。

もう一つの問題は、入力が欠ける瞬間があることです。たとえば地図トークンは、サンプルによってはゼロ個になります(田舎で近くに地図要素がない、データ取得に失敗した等)。attention は「見にいく相手(key)が1個もない」状態を計算すると、softmax(重みを合計1に正規化する関数)の分母がゼロになり NaN(Not a Number、計算が壊れた無効値)が発生します。NaN は一度出ると勾配を通じて全パラメータに伝染し、学習が即死します(一度 NaN になったパラメータは二度と戻りません)。これを防ぐのが、常に1個は有効な相手を用意しておく null トークンです。

## 基礎: 前提となる概念

残差接続(residual connection)は、ある処理 `F(x)` の出力を入力 `x` に足し戻す `x + F(x)` という構造で、深いネットワークの学習を安定させる定番技術です(ResNet で広まりました)。`F(x)` が何もしない(ゼロを出す)なら全体は `x` をそのまま通す恒等写像(入力をそのまま出す関数)になります。勾配が `x` の経路をショートカットで通れるので、層を深くしても勾配が消えにくいのが利点です。ゼロ初期化ゲートは、この `F(x)` の前に学習可能な係数を掛けて `x + alpha * F(x)` とし、`alpha=0` から始めることで「最初は恒等写像」にする工夫です。

パラメータ(`nn.Parameter`)は、学習で値が更新される変数です。普通の重みはランダムに初期化します(対称性を破って各ユニットが別々の特徴を学ぶため)が、ゲートの `alpha` はあえてゼロで初期化します。`alpha` をゼロにしても、その内側の `F` の重みはランダムなので対称性の問題は起きず、`alpha` だけが勾配で持ち上がるという都合の良い性質があります。

attention と softmax については、各 query が複数の key との類似度を softmax で重みに変え、その重みで value を加重平均する、という点だけ押さえておけば十分です。softmax は内部で `exp(各スコア)` を全 key で合計して割るので、key が1個もないと「ゼロで割る」状態になり NaN が出ます。1個でも有効な key があれば分母は正の値になり、計算が成立します。

## 仕組みを詳しく

### ゼロ初期化ゲート

融合モジュールの出力は典型的に次の形になります。

```python
self.alpha = nn.Parameter(torch.zeros(embed_dim))  # チャンネルごと, 初期値 0
...
out = q + self.alpha * h   # q = 元の特徴(例: BEV), h = 新しい枝の出力(例: 地図文脈)
```

`alpha` は埋め込み次元(たとえば256)と同じ個数の学習可能パラメータで、ゼロで初期化されます。学習開始時は `alpha * h = 0` なので `out = q`、つまり「新しい枝を足していないのと完全に同一」の出力になります。学習が進むと、新しい情報が役立つチャンネル(256次元のうちの一部の成分)だけ `alpha` が勾配で少しずつ持ち上がり、必要なぶんだけ新情報が混ざっていきます。

ポイントは「チャンネルごと」であることです。256チャンネルそれぞれ独立に効き具合を学べるので、「新情報が効く特徴」と「効かない特徴」をモデルが自分で選別できます。スカラー1個のゲートより表現力が高い設計です(この成分ごとのスケーリングは LayerScale と同じ構造です)。

数値イメージで示すと次の通りです。

```
step 0:    alpha ≈ [0, 0, ..., 0]        → 出力 = 元の特徴(新情報の影響ゼロ)
step 数千: alpha ≈ [0.3, 0.0, 0.8, ...]  → 一部チャンネルにだけ新情報が効く
```

`alpha=0` から始めるおかげで、勾配の観点でも利点があります。`out = q + alpha * h` を `alpha` で微分すると勾配は `h` 自身に比例し、`h` を生む内側のモジュールへの勾配は `alpha` に比例します。つまり学習初期は内側モジュールへ流れる勾配が小さく抑えられ、ランダムなノイズ枝が暴れて全体を壊すことを防ぎつつ、`alpha` が「この枝は役に立つか」をまず判定する、という穏やかな立ち上がりになります。既存の学習済み挙動を一切壊さずに学習を開始でき、新しい枝を追加してもベースラインの出力が変わらないことを保証できます。

### 学習可能 null トークン

```python
self.null_token = nn.Parameter(torch.zeros(1, 1, embed_dim))
nn.init.normal_(self.null_token, std=0.02)   # 小さな乱数で初期化
...
null = self.null_token.expand(B, -1, -1)
kv = torch.cat([null, tokens], dim=1)   # 先頭に常に1個の有効 key を置く
```

入力トークン列の先頭に、必ず有効な「情報なし」を表すトークンを1個くっつけます。マスク作成時もこの null の位置は常に「参加してよい」にします。

```python
null_valid = torch.zeros((B, 1), dtype=torch.bool)   # False = 有効(マスクされない)
padding = torch.cat([null_valid, key_padding_mask], dim=1)
```

こうすると、たとえ本物のトークンが全部マスクされていても(要素ゼロ個でも)、attention には必ず null トークンという有効 key が1個あります。softmax の分母がゼロにならず、NaN が出ません。要素がない場合、各 query はこの null トークンだけを見て「情報なし」という一定のベクトルを受け取ります。`std=0.02` の小さな乱数で初期化され(Transformer の標準的な初期化スケール)、この値も学習で最適化されるので、モデルは「情報がないときに返すべき内容」を自分で学べます。単にゼロを返すのではなく、「地図が無い」という状態を表す学習された表現を返せるのがポイントです。

この2つを組み合わせると、新しい枝は「最初は完全に無害(ゼロ初期化ゲート)」で「入力が欠けても壊れない(null トークン)」状態でスタートし、学習を通じて安全に有用さを獲得していきます。両者は役割が違い、ゲートは学習の安定化(立ち上がり)を、null トークンは数値の頑健性(欠損耐性)を担います。

## 手法の系譜と主要論文

ゼロ初期化ゲートは「新しい枝を恒等写像から始める」という広く使われる設計思想で、複数の有名手法に現れます。

Fixup と SkipInit(Zhang et al., ICLR 2019, arXiv:1901.09321 ほか)は、残差枝をゼロ近傍から始めることで正規化層(BatchNorm 等)なしでも深層ネットを安定に学習できることを理論・実験で示しました。残差ブロックの最後の層をゼロ初期化すると、初期状態でブロックが恒等に近くなり、勾配爆発・消失を抑えられるという、恒等スタートの安定化効果を体系立てた先行研究です。

ReZero(Bachlechner et al., UAI 2021, arXiv:2003.04887)は、残差ブロックの出力に学習可能スカラー `alpha`(初期値0)を掛け `x + alpha * F(x)` とする方式を提案しました。動機は、学習初期を恒等写像にすると信号と勾配の伝播が等長(dynamical isometry に近い)になり、深いネットでも学習が速く安定することです。チャンネルごとのゲートはこれを成分ごとに拡張したものに相当します。

LayerScale(Touvron et al., CaiT, ICCV 2021, arXiv:2103.17239)は、各残差ブロックの出力にチャンネルごとの小さな学習可能対角行列を掛け、ごく小さい値(例 1e-4 〜 1e-6)で初期化することで、非常に深い Vision Transformer の学習を安定化しました。本稿のチャンネルごとゲートはこの LayerScale と同型の構造で、初期値をゼロにすればさらに厳密な恒等スタートになります。

LoRA(Hu et al., ICLR 2022, arXiv:2106.09685)は、大規模モデルの微調整で追加する低ランク行列 `BA` の片方(`B`)をゼロ初期化し、適応のスタートを恒等にしました。学習開始時は `BA=0` なので元の重みと同じ出力から始まり、既存重みを壊さず効率的にファインチューンできることから広く普及し、「恒等から始める」系譜の代表例になっています。

ControlNet(Zhang et al., ICCV 2023, arXiv:2302.05543)は、事前学習済み拡散モデルに新しい条件入力(エッジ画像や姿勢など)の枝を足す際、接続部に「zero convolution」(重みとバイアスをゼロ初期化した畳み込み)を置きました。学習開始時に新枝の影響をゼロにし、既存の生成能力を壊さずに条件付けを学ばせるためです。大規模事前学習モデルを安全に拡張する代表的な成功例で、本稿のゼロ初期化ゲートと発想がそのまま一致します。

null トークン側は、attention で「該当なし」を表す学習可能トークンを置く一般的なテクニックです。物体検出の DETR(Carion et al., ECCV 2020, arXiv:2005.12872)の no-object クラス(物体が無いことを明示的に表すクラス)や、各種 Transformer の learnable な `[CLS]` トークン・null embedding と同系統の発想で、「無い」という状態を学習可能な表現として明示的に持つ点が共通しています。

## 論文の実験結果(定量データ)

ReZero は、初期値0のゲートを使うことで Transformer の学習が大幅に速く安定することを示しました。報告によると、12層 Transformer の言語モデリングで、warmup(学習初期に学習率を徐々に上げる工夫)なしでも収束し、収束までのイテレーション数を従来比でおよそ56%削減したとされます。さらに正規化層を使わずに非常に深いネット(報告では数十〜100層超、実験では128層の Transformer)を学習できることを示し、恒等スタートが深層学習の安定化に効くことを定量的に裏づけました。

LayerScale は、ImageNet 分類で、深い ViT を安定に学習させ、深さを増やしたときの精度向上を引き出しました。LayerScale なしでは深いモデルが発散・劣化するのに対し、小さな初期値のチャンネルごとスケーリングを入れると安定して学習が進み、Top-1 精度を改善したと報告されています。

ControlNet のアブレーションでは、zero convolution を通常のランダム初期化畳み込みに置き換えると、学習初期に事前学習済みモデルの出力が大きく乱れ、条件付けの学習が不安定になることが示されています。zero convolution があることで、学習開始時点では元のモデルと完全に同じ出力を保ちつつ、徐々に条件制御を獲得できます。これは「ゼロ初期化ゲートが既存挙動を壊さない」という主張の直接的な実証です。

LoRA は、追加パラメータをわずか(GPT-3 175B で全体の約0.01%まで削減した例を含む)にしながら、フルファインチューンに匹敵する性能を多数のタスク(GLUE など)で達成しました。ゼロ初期化により適応開始時が恒等であることが、安定した微調整に寄与しています。これらの数値が示すのは、「恒等から始めて勾配で少しずつ開ける」設計が、安定性と効率の両面で実際に効くということです。

## メリット・トレードオフ・限界

メリットは、既存の学習済みモデルを壊さず新しい枝を追加でき学習が安定すること、入力が欠けても NaN を出さず学習が即死しないこと、そしてチャンネルごとゲートでモデルが新情報の効かせ方を自分で選べることです。さらに、学習後の `alpha` の値を見れば「どのチャンネルに新情報がどれだけ効いているか」を読み取れ、解釈性にもつながります。

トレードオフと限界としては、ゲートがゼロ初期化のため新情報が実際に効き始めるまで学習ステップを要し立ち上がりが遅いこと、新情報が常に無価値なデータではゲートがほぼゼロのまま null トークン分の計算だけが無駄になりうること、そして null トークン・マスク・ゲートと安全装置が増えるぶんコードが複雑になることが挙げられます。研究上の論点としては、ゲートの立ち上がりを速める初期化やスケジューリング(最初だけ小さな正の値から始める等)、そして恒等スタートが最適化の地形(loss landscape)をどう変えて安定化に寄与するのかの理論的理解(信号伝播理論・dynamical isometry の観点)が残されています。

## 発展トピック・研究の最前線

ゼロ初期化ゲートと null トークンの組み合わせは、マルチモーダル(複数種類の入力を扱う)モデルへの拡張で特に重要になっています。画像・LiDAR・地図・言語など複数のソースを後付けで足していく際、各ソースの枝を恒等から始めれば、どのソースを追加してもベースラインを壊さずに増設でき、さらにゲートの値を見れば「どのソースがどのチャンネルにどれだけ効いているか」を可視化できます。これはモデルの解釈性にもつながります。また、欠損入力への頑健性という観点では、null トークンを単なる NaN 回避ではなく「そのモダリティが欠けている」という明示的な信号として学習させ、モダリティのドロップアウト(学習中にわざと一部入力を欠落させて頑健性を高める手法)と組み合わせる研究も進んでいます。大規模モデルへ新機能を継続的に追加していく continual / modular な学習の文脈でも、恒等スタートのゲートは「既存能力を忘れずに新能力を足す」ための基本部品として位置づけられています。
