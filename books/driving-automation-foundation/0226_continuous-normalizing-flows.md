# Continuous Normalizing Flows (CNF) / Neural ODE

## ひとことで言うと
Continuous Normalizing Flows(連続正規化フロー、CNF)は、[Normalizing Flows](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0245_normalizing-flows) の「変換を何段も重ねる」を、段数を無限に細かくして「連続的な流れ」にしたものです。点が時間とともになめらかに動く様子を、ニューラルネットが表す「速度の場(velocity field)」で記述し、微分方程式(ODE)を解いて移動させます。この仕組みの土台が Neural ODE です。Flow Matching の登場でこの連続フローが「シミュレーションフリー」で学習できるようになり、拡散モデルと並ぶ生成の主役になりました。

## 直感的な理解
普通の [Normalizing Flows](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0245_normalizing-flows) は、可逆変換を 8 段・16 段…と離散的に重ねていました。段数が多いほど複雑な分布を表せますが、層を増やすほどメモリと計算が増えます。また「段の数」は人間が決め打ちする離散的なハイパーパラメータでした。さらに各層は「可逆かつヤコビアン行列式が軽い」という強い制約を背負わなければならず、ネットワーク設計の自由が大きく削られていました。

Neural ODE(Chen et al., 2018)の発想は「層を無限に薄くして連続にしたらどうなるか」です。ニューラルネットの残差結合 `x_{t+1} = x_t + f(x_t)`(ResNet の1ブロック、入力に補正を足して次へ渡す構造)をよく見ると、これは「微小時間で x が f だけ動く」更新で、刻みを限りなく小さくすると微分方程式 `dx/dt = f(x, t)` になります。f はニューラルネット、t は連続時間(0 から 1 など)です。つまり「層を積む」ことと「時間を進める」ことが同じになる、という視点の転換です。

身近なたとえ: 川の流れに浮かべた木の葉を考えます。川の各地点には「ここではこの向きにこの速さで流れる」という速度の場があり、葉はそれに従って連続的に動きます。CNF は、標準正規分布のランダムな点(出発地点)を、ニューラルネットが定める速度の場に沿って時間 0→1 で流すと、本物のデータ(到着地点)になる、という仕組みです。葉が決して衝突したり消えたりしない(軌道が交わらない)ので、点と点の対応は1対1のまま保たれ、自動的に可逆になります。

この見方の何が嬉しいか。
- 「層の数」が「時間をどれだけ細かく刻むか(ODE ソルバの精度)」に置き換わり、精度と計算量を学習後でも調整できる。
- メモリ効率がよい(後述の随伴法 adjoint method で、中間結果を全部保存しなくても勾配が計算できる)。
- 速度の場 f に任意のニューラルネットを使えるため、離散フローのような「可逆かつ行列式が軽い」という構造制約から解放される。流れが連続なら可逆性は自動的に成り立つからです。

## 基礎: 前提となる概念
- 常微分方程式(ODE, Ordinary Differential Equation): 「ある量の変化の速さ(微分)」を与える方程式です。`dx/dt = v` は「位置 x が時刻 t に速度 v で動く」という意味で、初期位置が決まれば全軌道が一意に決まります。
- ODE ソルバ: ODE を数値的に解く手続き。最も単純なオイラー法(Euler method)は「現在位置で速度を計算 → 小さな時間刻み Δt だけ進める → 繰り返す」を行います。`x ← x + Δt·v(x,t)`。ルンゲ・クッタ法(Runge-Kutta)はより高精度で1ステップに複数回 v を評価します。刻みを細かくするほど正確ですが計算が増えます。
- 関数評価回数(NFE, Number of Function Evaluations): ODE を解くのに速度の場 v を何回呼ぶか。これが推論コストの実体です。NFE=10 なら v を10回計算する、という意味で、後述のとおりこの値の大小が手法の優劣を決める実用指標になります。
- ベクトル場(速度の場): 空間の各点に「ベクトル(向きと大きさ)」を割り当てたもの。ここでは「位置 x・時刻 t でどう動くべきか」を返す関数 v(x,t) です。
- 発散(divergence): ベクトル場が「その点から湧き出しているか/吸い込んでいるか」を表す量で、数学的にはヤコビアン行列のトレース(対角成分の和)です。湧き出すと密度が薄まり、吸い込むと濃くなる。連続版の尤度計算で、密度の増減を表すのに使います。
- 随伴法(adjoint sensitivity method): ODE を逆向きに解くことで、中間状態を全部メモリに保存せずに勾配を計算する手法。メモリ消費を層数(=時間刻み数)に依らず一定にできます。
- 確率の輸送(transport): ある確率分布を別の分布へ「質量を保ったまま移動させる」見方。CNF は「ノイズ分布をデータ分布へ輸送する流れ」を学びます。最適輸送(optimal transport)はこの移動を最小コストで行う理論で、後述の直線経路と関係します。

## 仕組みを詳しく
中心にあるのは速度の場 v(x, t) で、これは「位置 x にいて時刻 t のとき、どの向きにどれだけ速く動くべきか」を返すニューラルネットです。

生成(サンプリング)の手順:
1. z を標準正規分布から引く(例: `(B, 2)` の2次元点)。これを時刻 t=0 の位置とする。
2. ODE `dx/dt = v(x, t)` を t=0 から t=1 まで数値的に解く。
3. t=1 に着いた位置 x が生成データ。

```
 t=0: z(標準正規の点) ●
        │ dx/dt = v(x,t) に従って流れる(速度の場)
        ▼
 t=1: x(本物の分布の点) ★
   逆向き(x→z, t=1→0)も解ける → 尤度計算が可能
```

尤度計算: 離散フローではヤコビアン「行列式」を計算しましたが、連続版では「instantaneous change of variables(瞬間的な変数変換公式)」を使い、密度の対数の変化が速度の場の発散(トレース)の時間積分で表せます。`d log p(x_t)/dt = −Tr(∂v/∂x)` を、位置の ODE と一緒に解くだけです。離散版が「行列式(全成分)」を要したのに対し、連続版は「トレース(対角だけ)」で済むのが計算上の利点です。

問題点と解決:
- 発散(トレース)の計算は次元が高いと重い(D 次元なら素朴には D 回の逆伝播)。FFJORD(Grathwohl et al., 2019)は Hutchinson のトレース推定(ランダムベクトル ε を使って `Tr(A) ≈ E[ε^T A ε]` と1回の積で近似する手法)を使ってこれを軽くし、高次元データでも CNF を実用化しました。

最大の難点は学習時です。CNF を素直に学習しようとすると、毎ステップ ODE を最後まで数値的に解いてから勾配を逆向きに流す必要があり、非常に遅く不安定でした(ODE ソルバが何十回も v を呼ぶ、その全体を通して学習する)。これを「シミュレーション(ODE 求解)が必要な学習」と呼びます。学習の1ステップごとに完全な軌道を計算するので、現実の大規模データには手が届きにくかったのです。

ここで登場するのが Flow Matching(Lipman et al., 2023)です。Flow Matching は「ODE を解かずに、速度の場 v を直接、回帰(regression、目標値に合わせる学習)で教える」方法です。考え方は次のとおり。

1. ノイズ z と本物データ x を直線で結ぶ: `x_t = (1−t)·z + t·x`(t は 0〜1)。
2. この直線上を一定速度で進むとすると、各点での「正解の速度」は `x − z`(終点 − 始点)と分かる。直線の傾きそのものです。
3. ネットワーク v(x_t, t) がこの `x − z` を出すように二乗誤差で学習する。

ODE を学習中に解く必要がない(個々の点で目標速度が解析的に分かる)ので「シミュレーションフリー」と呼ばれ、学習が劇的に速く安定しました。注目すべき理論的事実は、こうして個々のデータ点へ向かう「条件付き(conditional)」の速度を学習すると、その期待値が分布全体を正しく輸送する「周辺(marginal)」の速度の場に一致する、という点です(conditional flow matching の核心)。1点ずつの単純な目標を平均すると、全体として正しい流れになる、という保証です。これにより、解析的に書ける単純な目標を回帰するだけで、複雑な分布を生成する速度の場が手に入ります。

数値例(Flow Matching の1ステップ、軌道生成を例に):
- 本物データ x = `(B, 64, 2)`(64 ステップの軌道、各点 x,y)。
- ノイズ z = 同形状の標準正規。
- t を 0〜1 の範囲でランダムに引く(B 個)。
- 中間点 `x_t = (1−t)·z + t·x`。
- 目標速度 `target = x − z`。
- 損失 = `MSE(v(x_t, t, 条件), target)`。これを最小化するだけ。
- 推論時は z から始めて `dx/dt = v` を 5〜10 ステップ程度のオイラー法で積分して x を得る。

直線(最適輸送に近い)経路を使うため、推論時の ODE 積分が少ないステップで済むのも実用上の大きな利点です。曲がりくねった経路だと、ソルバが細かく刻まないと軌道がずれるので NFE が増えます。逆にまっすぐなら粗い刻みでも正確に終点へ着きます。

## 手法の系譜と主要論文
- Chen et al., "Neural Ordinary Differential Equations" (NeurIPS 2018、ベストペーパー、arXiv:1806.07366)。ResNet を連続時間の ODE と見なす枠組みと随伴法を提案。メモリ効率のよい連続モデルが可能に。CNF として尤度計算もできることを示した。ただし学習・推論で ODE を解くため遅い。
- Grathwohl et al., "FFJORD" (ICLR 2019, arXiv:1810.01367)。CNF の尤度計算に必要な発散(トレース)を Hutchinson 推定で軽量化。高次元データでも厳密尤度(に近い)の連続フローが現実的に。推定誤差と、依然 ODE 求解が必要なコストが残る。
- Lipman et al., "Flow Matching for Generative Modeling" (ICLR 2023, arXiv:2210.02747)。CNF をシミュレーションフリーで学習する Conditional Flow Matching を提案。直線(最適輸送)経路で結ぶことで推論時の ODE 積分が少ないステップで済む。拡散モデルと同等以上の品質を、より速くシンプルに達成。
- Liu, Gong & Liu, "Rectified Flow" (ICLR 2023, arXiv:2209.03003)。直線的な経路を学び、さらに「reflow」(生成したペアで再学習し経路をまっすぐに整える手続き)を繰り返して、ほぼ1ステップ生成を狙う手法。
- Albergo & Vanden-Eijnden, "Stochastic Interpolants" (2023)。ノイズとデータを結ぶ補間を一般的な枠組みで定式化し、ODE(決定的)と SDE(確率的)を統一的に扱えるようにした理論的整理。
- Liu et al., "InstaFlow" (2023)。Stable Diffusion を rectified flow 化し、1ステップ生成で従来の多ステップ生成に迫る品質を報告。

系譜は「離散 [Normalizing Flows](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0245_normalizing-flows) → Neural ODE(連続化)→ CNF + FFJORD(連続フローを実用化)→ Flow Matching / Rectified Flow(シミュレーションフリー学習で高速・安定化)」で、現在は拡散モデルと統合的に理解されています。

## 論文の実験結果(定量データ)
- FFJORD(2019)は CIFAR-10 で約 3.40 bits/dim(bits per dimension、ピクセルあたりの符号化ビット数。小さいほど良い)を達成し、当時の離散フロー(Glow ≈ 3.35)に迫る尤度を、可逆構造の制約なしに(任意のネットワークで)得られることを示しました。アブレーションとして、トレースの厳密計算と Hutchinson 推定を比べ、推定でも尤度がほぼ保たれることを示し、高次元への道を開きました。
- Flow Matching(2022/2023)は ImageNet 128×128 などで、拡散モデル(DDPM やスコアベース)と同等以上の FID(見た目の指標、小さいほど本物に近い)を達成しつつ、学習がより安定し、推論の関数評価回数(NFE)を減らせることを示しました。最適輸送に近い直線経路(OT path)を使うと、拡散が好む曲がった経路(variance-preserving path)より、同じ品質をより少ない NFE で出せることをアブレーションで実証しています。経路の選び方が NFE を直接左右する、という設計指針を与えた点が重要です。
- Rectified Flow / InstaFlow(2023)は、reflow を繰り返すと経路がまっすぐになり、NFE=1(1ステップ)に近い超高速生成でも実用的な FID を保てることを示しました。Stable Diffusion を rectified flow 化した InstaFlow は、1ステップ生成で従来の数十ステップ生成に迫る品質(FID で競争力のある水準)を報告しています。reflow の回数を増やすほど経路が直線化し NFE を減らせる一方、各 reflow で生成→再学習のコストがかかる、というトレードオフも示されました。

指標の意味: NFE は実用上もっとも効く量で、拡散の素朴な生成が数百〜千回の評価を要したのに対し、フロー系は数回〜1回まで下げられます。FID が同等なら NFE が小さいほど良い、という形でこれらの手法は比較されます。これが「拡散から連続フローへ」研究の重心が移った定量的な理由です。推論が速いということは、応答時間に制約のあるアプリ(対話的生成、ロボットの制御周期に間に合わせる行動生成など)で決定的に効いてきます。

## メリット・トレードオフ・限界
メリット
- 連続的な流れなので、出力(軌道など)が滑らかでなめらか。時系列・軌道生成と相性がよい。
- 厳密な尤度計算ができる(離散フローの利点を継承)。
- Neural ODE の随伴法でメモリ効率がよい(深さに依らず一定メモリ)。
- 任意のネットワークを速度の場に使える(離散フローの可逆制約から解放)。
- Flow Matching と組み合わせれば学習がシミュレーションフリーで高速・安定。

トレードオフ・限界
- 素直に学習・推論すると ODE 求解が必要で遅い(Flow Matching が学習側を緩和)。
- 推論時の生成品質が ODE ソルバのステップ数(NFE)に依存(少なすぎると粗くなり軌道が終点からずれる)。経路がまっすぐでないとステップを減らせない。
- 随伴法は理論上メモリ一定だが、逆向き ODE の数値誤差が勾配に乗ることがある(forward と backward で軌道が微妙にずれる)。
- 速度の場の発散計算など、実装が他の生成モデルよりやや込み入っている(尤度を厳密に出す場合)。Flow Matching で生成だけなら尤度計算は不要なので、この負担は回避できます。

研究上の未解決課題としては、経路設計(直線か曲線か、最適輸送をどこまで使うか、ミニバッチ内で OT カップリングを取る OT-CFM など)の最適化、超少ステップ生成での品質保持、連続フローと拡散・確率微分方程式(SDE)の統一的理解の深化などがあります。

## 発展トピック・研究の最前線
- 拡散との統一: スコアベース拡散は「確率の流れ ODE(probability flow ODE)」として書き直せ、Flow Matching と同じ土俵に乗ります。両者を一般化した stochastic interpolants の枠組みが理論の最前線で、「拡散もフローも、ノイズとデータを結ぶ補間の選び方が違うだけ」という統一像が定着しつつあります。
- 条件付き生成への応用: 速度の場に条件(クラス、テキスト、観測など)を入れることで、条件付き連続軌道や系列の生成が可能になります。ロボティクスや自動運転の軌道生成、行動方策(policy)の表現として連続フローを使う研究が活発です。複数のありうる未来を多峰性を保って生成できる点が、回帰や [GAN (敵対的生成ネットワーク)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0236_gan_note) に対する優位として注目されています。
- 超高速生成: rectified flow の reflow や consistency 系の蒸留で、1〜数ステップ生成を高品質に保つ研究が進んでいます。生成モデルを「ほぼ1回の前向き計算」に圧縮する方向です。
- 離散・多様体上のフロー: ユークリッド空間でなく、球面・回転群などの多様体(リーマン多様体)上で Flow Matching を行う Riemannian Flow Matching など、対象データの幾何に合わせた拡張が進んでいます。
