# Neural Map Prior (NMP)

## ひとことで言うと
車が同じ場所を走るたびに、その場所の「地図の記憶」を少しずつ更新して貯めておき、次に通るときはその記憶を取り出して今のセンサー情報と合わせることで、より正確な地図を作る手法です。地図を一度作って終わりにせず、走るたびに自動で育てていく、という発想が核心にあります。この記憶は人間が読める折れ線(ベクトル)ではなく「ニューラルネットが扱う数値ベクトル」の形で持つので、Neural(神経的)な Map Prior(地図の事前知識)と呼ばれます。「prior(プライアー)」とは統計や機械学習で「観測する前にあらかじめ持っている知識・思い込み」を指す言葉で、ここでは「過去の走行から学んだ、その場所の地図に関する事前知識」を意味します。

## 直感的な理解
オンライン地図構築という研究分野があります。これは「専用の測量車であらかじめ地図を作っておく」のではなく、走行中の車に付いたカメラやLiDAR(レーザーで距離を測るセンサー)の情報から、その瞬間に車線や道路境界をその場で推定する技術です。この方式には実用上ありがたい点(地図を事前に整備しなくてよい、最新の道路状況をその場で反映できる、地図の更新を待たなくてよい)がある一方で、決定的な弱点があります。

弱点は「一回きりの観測しか使わない」ことです。今この瞬間にカメラへ映ったものだけから地図を起こすので、雨で白線が滲んで見えない、前を走る大型トラックが車線を完全に隠している、夜間で路面が暗い、逆光でレンズが白飛びしている、強い影で路面のコントラストが消えている、といった状況では、その瞬間の入力が悪いだけで地図も崩れます。単一フレーム推定の精度上限は、結局その一瞬の観測品質に縛られてしまうのです。

しかし冷静に考えると、同じ道は何度も通るはずです。通勤路なら毎日、配送ルートなら一日に何度も、ロボタクシーの運行エリアなら一日に何百回も同じ交差点を通過します。昨日は晴れていて白線がくっきり見えていたかもしれない。先週はトラックがいなくて交差点全体が見えていたかもしれない。その「過去の良い観測」を毎回捨ててしまうのは、あまりにもったいない。人間のドライバーは「この交差点は右折レーンがここにある」と覚えていて、雨の日でも記憶で補います。NMP はこの「覚えておいて次に活かす」を機械にやらせる手法だと考えると本質がつかめます。

もう一つの大きな動機が、地図メンテナンスの自動化です。高精度地図(HD マップ)は通常、専用車で測量して人が編集します。道路工事、白線の引き直し、新しい横断歩道の設置のたびに測り直しが必要で、世界中の道路を常に最新に保つのは事実上不可能です。地図ベンダーが抱える最大のコストはこの「鮮度維持」です。もし「普通に走っている多数の車のログ」から地図が勝手に更新・高品質化していくなら、メンテナンスの人手コストを大幅に削れます。これは群衆の力で地図を育てる、いわゆるクラウドソーシング地図(多数の車のデータを集約して地図を作る発想)に近いものです。NMP は、その「集約して育てる」処理をニューラルネットの中に埋め込んだ、と理解できます。

## 基礎: 前提となる概念
NMP を理解するには、いくつかの土台を押さえる必要があります。順に噛み砕きます。

BEV 特徴とは、Bird's-Eye-View(バーズアイビュー、鳥瞰図=真上から見下ろした図)の特徴です。複数のカメラに映った映像を、真上から見下ろした座標系に投影し直して並べた特徴マップのことです。形は `[C, H, W]`(C はチャンネル数=特徴の種類数、H と W は鳥瞰図グリッドの縦横のマス目数)というテンソル(多次元の数値配列)で表されます。たとえば自車の周囲 60m × 30m を 0.15m 刻みのマスで切ると、H=400, W=200 程度のグリッドになり、各マスに C=256 次元くらいの特徴ベクトルが乗ります。地図構築ネットワークは、この BEV 特徴から「ここは車線」「ここは境界」という地図要素を読み取ります。カメラ画像から BEV へ投影する代表手法に LSS(Lift-Splat-Shoot, Philion & Fidler, ECCV 2020)や BEVFormer(Li et al., ECCV 2022)があり、NMP はこの BEV 特徴の段階に記憶を融合します。

attention(アテンション、注意機構)とは、複数の情報源があるとき「どれをどれだけ信じるか」をデータから学習で決める仕組みです。たとえば「今のセンサーが鮮明なら今を重視し、今が不鮮明なら記憶を重視する」という重み付けを、固定ルールではなくネットワークが状況に応じて出します。具体的には、ある特徴(query)が他の特徴群(key, value)との類似度を計算し、その類似度を重みにして value を加重平均します。NMP では「今の観測」を query、「過去の記憶」を key・value として cross-attention(異なる情報源どうしを結ぶ attention)を張ります。

GRU(Gated Recurrent Unit、ゲート付き回帰ユニット)とは、時系列を扱うニューラルネットの部品で、「古い記憶をどれだけ残し、新しい情報をどれだけ取り込むか」を学習可能なゲート(門、0〜1の開き具合を出す機構)で調整する仕組みです。新しい観測が来たときに、記憶を完全に上書きするのでも完全に無視するのでもなく、いい塩梅で更新します。NMP は地図メモリの更新にこの考え方を使います。

mAP(mean Average Precision、平均適合率の平均)とは、検出・地図構築の精度を測る代表指標です。「ネットワークが出した地図要素が、正解の地図要素とどれだけ一致しているか」を、適合率(precision、出したもののうち正しかった割合)と再現率(recall、正解のうち拾えた割合)の両方を考慮して総合した数値で、0 から 100% の間を取り、高いほど良いとされます。地図構築では、推定した車線と正解の車線がチェンファー距離(Chamfer distance、2つの点列の間の平均最近傍距離)で一定しきい値以内なら「当たり」と判定し、車線分割線・道路境界・横断歩道といった種類ごとに AP を出して平均します。しきい値は典型的に 0.5m / 1.0m / 1.5m の3段階で評価され、その平均を取ります。

## 仕組みを詳しく
NMP の中心は「グローバル地図メモリ(global map prior)」という、運行エリア全体を覆う一枚の巨大な記憶を持つことです。処理を段階的に追います。

ステップ1: グローバル地図メモリを用意する。
運転対象エリア全体を大きな格子(タイル)に区切り、各タイルに学習可能な特徴ベクトルの集まりを持たせます。これがグローバル地図 prior です。人間用の地図が「車線=この座標の折れ線」と読める形で書かれるのに対し、NMP のメモリは「その場所の特徴を表す数値の塊」というネットワーク向けの形(latent representation、潜在表現)です。学習開始時は空(初期値、たとえばゼロや学習可能な初期パラメータ)で、走行を重ねるほど中身が埋まっていきます。

```
   グローバル地図メモリ(都市全体のタイル集合)
   +----+----+----+----+
   | t1 | t2 | t3 | t4 |   各 ti は学習可能特徴ベクトル群
   +----+----+----+----+   shape [Ctile, Htile, Wtile]
   | t5 | t6 | t7 | t8 |
   +----+----+----+----+
        ↑ 自車が t6 にいる → t6 を切り出して融合
```

ステップ2: 推論時、今の位置の prior を取り出す。
車がある場所を走るとき、自車位置(GPS や自己位置推定で得た座標)に対応するタイルをメモリから切り出します。これが「その場所について過去の走行から貯めた記憶特徴」です。自車の向きに合わせて回転・切り出しを行い、現在の BEV グリッドと座標系を揃えます。

ステップ3: 今のセンサー観測と prior を融合する。
今この瞬間にマルチカメラから作った BEV 特徴 `[C, H, W]` と、メモリから取り出した prior 特徴を attention で融合します。ここで「現在の特徴を query、過去の prior を key・value」として cross-attention を張るのが典型です。直感的には「今の各マスについて、過去の記憶の対応マスを参照し、今が見えていない部分を記憶で補う」操作です。NMP では計算量を抑えるため、グリッド全体ではなく局所窓に区切った attention(GRU-attention や current-to-prior attention と呼ばれる構成)を使います。融合後の特徴から最終的な地図(車線・境界など)を出力します。

ステップ4: 走行後、prior を更新する(ここが肝)。
今回の走行で得た良い観測を、グローバル地図メモリに書き戻します。GRU 風の更新式で、メモリ特徴 `m` を新しい観測特徴 `o` の方向へ少しだけ動かします。概念的には次のような形です。

```
  z = sigmoid(Wz · [m, o])        # 更新ゲート: 今回をどれだけ取り込むか (各次元 0〜1)
  m_new = (1 - z) ⊙ m + z ⊙ o     # 古い記憶と新観測の加重平均
```

ここで `z` は 0〜1 の値を各次元に出すゲート、`⊙` は要素ごとの掛け算(アダマール積)です。`z` が小さければ記憶を保ち、大きければ今回を重視します。何度も同じ場所を走るほど、そのタイルのメモリは「多数回の観測の良いとこ取り」をした安定した prior に育ちます。これが地図メンテナンスの自動化の実体です。GRU を使う理由は、単純な平均(移動平均)だと「今回が悪い観測でも一律に取り込む」のに対し、ゲートが「今回が信頼できそうか」を文脈から判断して取り込み量を調整できるからです。

具体例で追うと分かりやすいです。ある交差点を1回目に走ったとき、雨で右折レーンの白線が見えず地図が欠けたとします。NMP はこの不完全な結果をメモリに書きます(まだ弱い prior)。2回目、晴れた日に同じ交差点を走ると今度は白線がよく見える。NMP は今回の良い観測でメモリを更新し、右折レーンが補われます。3回目以降、たとえまた雨でも、育ったメモリから右折レーンを補完できる、という具合です。観測回数が増えるほど精度が単調に上がっていく、という性質は論文でも実測されています。

## 手法の系譜と主要論文
オンライン地図構築の流れの中に NMP を位置づけます。

HDMapNet(Li, Wang, Li, et al., ICRA 2022, "HDMapNet: An Online HD Map Construction and Evaluation Framework", arXiv:2107.06307)が、マルチカメラ画像から BEV 上にセマンティックなラスタ地図(画素単位で車線か否かを塗り分けた画像)を推定し、後処理でベクトル化する初期の枠組みを示しました。これがオンライン地図構築という土俵を整備し、評価指標(チェンファー距離ベースの mAP)も提供しました。

VectorMapNet(Liu, Wang, Wang, Zhao, et al., ICML 2023)と MapTR(Liao, Chen, Wang, et al., ICLR 2023, "MapTR: Structured Modeling and Learning for Online Vectorized HD Map Construction", arXiv:2208.14437)が、ラスタを経由せず Transformer のクエリで地図要素を直接ベクトル(順序付き点列=ポリライン)として出す方式を確立し、精度と速度を大きく押し上げました。NMP の論文は、こうした「単一フレームから地図を起こす」ベース手法のいずれにも後付けできる追加モジュールとして提案された点に新規性があります。

Neural Map Prior(Xiong, Liu, Wang, Liu, Yu, Zhao, et al., CVPR 2023, "Neural Map Prior for Autonomous Driving", arXiv:2304.08481)が本トピックの本体です。提案は、学習可能なグローバル地図メモリを持ち、推論時に現在位置の prior を attention で融合し、走行後に GRU 風更新でメモリを育てる枠組みです。動機は前述の二点(一回きり観測の弱さと、メンテナンスの人手コスト)を、自己更新する prior で同時に解消することでした。

その後、prior を「自分で育てる」のではなく「外部のすでにある地図から取り込む」系統が並走します。P-MapNet(Jiang et al., RA-L 2024)は外部 SD 地図(カーナビ用の粗い地図)と過去 HD 地図をラスタとして注入し、MAE(Masked Autoencoder、穴あき入力から元を復元する事前学習)的な補完で穴を埋めます。MapEX 系(Sun et al., 2024)は既存ベクトル地図を Transformer のクエリ初期値として与えます。NMP はこれらと対照的に「外部地図に頼らず、自分の走行ログから prior を生成・更新する」点が独自で、prior をどこから持ってくるかという軸で系譜が分岐しています。詳細は [SD-map prior融合 (MapEX等)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0449_mapex-mapfusion-sd-prior) を参照してください。

時間方向の別系統として StreamMapNet(Yuan, Mao, Hou, et al., WACV 2024)は、直近フレームの BEV 特徴とクエリを次フレームへ伝播する「ストリーミング」設計で短期記憶を扱います。NMP の長期記憶(同じ場所の過去走行)と StreamMapNet の短期記憶(直前数フレーム)は補完的で、両者を統合する研究が後続しています。

## 論文の実験結果(定量データ)
NMP は nuScenes(エヌシーンズ、ボストンとシンガポールで収集された自動運転用の大規模データセットで、6カメラ + LiDAR + 5レーダーを備え 1000 シーンから成る。各シーン約20秒)で評価されました。地図要素は車線分割線・道路境界・横断歩道の3クラスで、推定と正解の一致をチェンファー距離のしきい値(0.5/1.0/1.5m)で判定して mAP を出します。mAP は前述のとおり、しきい値内で正解とどれだけ一致したかを精度・再現率込みで総合した 0〜100% の数値です。

報告によれば、NMP モジュールを HDMapNet 系のベースに乗せると mAP がおよそ 4〜5 ポイント、Transformer 系(MapTR 風)のベースに乗せても数ポイント単位で一貫して向上したとされます。たとえば横断歩道のように単一フレームでは観測の崩れやすい要素で改善幅が大きく報告されています。重要なのは「どのベース手法に付けても上がる」という汎用性で、特定の構造に依存しない追加モジュールであることを示しています。数ポイントの mAP 向上は地図構築では実用上意味のある差で、地図要素を1割前後多く・正確に拾えるオーダーに相当します。

最も示唆的なのは「観測回数を増やすほど精度が単調に上がる」というアブレーション(構成要素や条件を変えて効果を確かめる実験)です。同じ場所を1回しか走っていないときより、複数回(2回、3回……)走ってメモリが育った後の方が mAP が高くなり、prior の蓄積が直接効いていることが示されました。逆に prior 融合モジュールを外す(その瞬間の観測だけにする)と精度が落ち、融合と更新の両方が寄与していることが確認されています。また悪天候や夜間など、単一フレームでは観測が劣化する条件で、prior による補完の恩恵が相対的に大きく出ると報告されています。これらの数値は「過去の良い観測を貯めて使う」という仮説が定量的に裏づけられたことを意味します。融合方式のアブレーションでは、単純な特徴の足し算や連結よりも attention ベースの融合が優れ、更新を移動平均から GRU 風ゲートに変えると追加で改善した、という比較も報告されています。

## メリット・トレードオフ・限界
メリットを整理します。第一に、過去の良い観測を活かせるため、一回きり観測の弱さ(悪天候・遮蔽・夜間)に強くなります。第二に、走行データから prior が自動で育つので、地図メンテナンスを人手なしで進められます。第三に、既存のオンライン地図構築のどれにでも後付けできる追加モジュールとして設計されている汎用性があります。第四に、推論時のコストは「該当タイルを引いて attention で融合する」分だけで、毎フレーム重い再学習をする必要はありません。

トレードオフと限界も率直に挙げます。グローバル地図メモリの保存・配信・管理という新たな基盤が必要です。都市全体のタイル特徴をどこかに貯め、推論時に該当タイルを高速に引いてくる仕組みがいります。都市規模ではメモリ量とストレージ・通信が無視できません。次に、自車位置推定の誤差がそのまま prior のずれになります。メモリ上のどのタイルにいるかを間違えると、別の場所の記憶を融合してしまい、かえって害になります。GPS が数メートルずれる市街地キャニオン(高層ビルに囲まれて GPS が乱れる場所)ではこの問題が深刻です。さらに、工事や白線引き直しのような実世界の変化に対しては、メモリが新しい観測で更新されるまでラグがあり、古い prior が一時的に誤情報を与える危険があります。GRU 風更新のゲートが古い記憶を保ちすぎると変化に鈍く、新観測を取り込みすぎると一回きりの悪観測でメモリを汚す、という調整(可塑性と安定性のトレードオフ、stability-plasticity dilemma)の難しさもあります。

未解決の研究課題としては、変化検出(地図が変わったことを能動的に検知してメモリを素早く差し替える)、prior の不確実性の明示(この記憶はどれだけ信頼できるかをネットワークが自覚し、融合重みに反映する)、複数車両のログをプライバシーや通信コストを抑えつつ統合する分散・フェデレーテッドな更新、そしてメモリのバージョン管理(いつの観測で更新されたかを追跡し、古すぎる記憶を減衰させる)などが挙げられます。

## 発展トピック・研究の最前線
NMP 以降、時間方向の情報を地図構築に取り込む流れが広がっています。連続フレームを融合する temporal fusion(直近数フレームの BEV を時間方向にも attention で混ぜる)を組み込んだオンライン地図手法(前述の StreamMapNet など)が増え、NMP の「長期記憶(同じ場所の過去走行)」と「短期の時系列融合(直前数フレーム)」は補完的だと理解されつつあります。両者を一つの枠組みで扱う研究が後続しています。

また、地図要素どうしのトポロジー(車線がどこへつながるか、車線と信号・標識の対応)まで一体で推定する手法(TopoNet, Li et al., 2023 など、車線中心線とその接続を同時に出す系統)が登場し、prior にトポロジー情報も貯める拡張が考えられます。形状だけでなく「つながり」を記憶できれば、交差点のような構造的に複雑な場所で恩恵が大きくなります。

prior をどこから得るかという軸では、自己生成(NMP)・外部 SD 地図(P-MapNet)・外部ベクトル地図(MapEX)を組み合わせるハイブリッドが自然な発展方向です。SD 地図で接続の骨格を与え、走行ログで形状を育て、直前フレームで動的に補正する、という多段の prior 統合は実運用に近い構成として研究が進んでいます。さらに、地図を明示的に作らず潜在表現のまま下流のプランナーへ渡す end-to-end 自動運転(UniAD, Hu et al., CVPR 2023 など)の文脈でも、「場所ごとの記憶を潜在空間に貯める」という NMP の発想は記憶モジュールとして応用が検討されています。
