# FocalNet (Focal Modulation Networks)

## ひとことで言うと
FocalNet は、画像から特徴を取り出す土台ネットワーク(バックボーン)の一種です。Transformer の心臓部である自己注意(各位置が他のどの位置をどれだけ見るかを動的に計算する仕組み)をまるごと取り除き、代わりに「焦点変調(Focal Modulation)」という演算に置き換えました。焦点変調は、まず各位置の周囲の文脈を複数のスケールで集約し、次にその文脈で各位置の特徴を調整(変調)します。自己注意より軽く(計算量が画素数に対し線形)、しかも同等以上の精度を出します。

## 直感的な理解
人が混雑した交差点で信号機を探す状況を考えます。自己注意のやり方は「視界の中のあらゆる地点ペアについて、どれとどれが関係するかを総当たりで計算してから、関係の強い地点の情報を集める」というものです。総当たりなので地点数が増えると爆発的に重くなります。

焦点変調のやり方はこの順番をひっくり返します。「まず自分の周りを、近く・中くらい・遠く、と何段階かの広さで眺めて文脈をまとめる。そのあと、まとめた文脈に応じて自分の見え方を調整する」という流れです。人間の視覚にも近く、いきなり全ペアの関係を計算するのではなく、周囲をぼんやり把握してから注目先を絞り込みます。総当たり行列を作らないので、地点数が増えても計算は緩やかにしか増えません(線形)。

自己注意は「相互作用(全ペアの関連度計算) → 集約(関連度で値を集める)」の順、焦点変調は「集約(文脈づくり) → 相互作用(文脈で変調)」の順です。この順序の入れ替えが、二乗計算量を線形計算量に変える鍵です。順序を入れ替えても「位置ごとに何を重視するかを入力に応じて変える」という自己注意の動的さは、ゲート(後述)という形で保たれます。

## 基礎: 前提となる概念
いくつかの用語を順に噛み砕きます。

「自己注意(self-attention)」は、入力の各位置(トークン)に query・key・value という3つのベクトルを持たせ、ある位置の query と全位置の key の内積で関連度を計算し、softmax で正規化した重みで全位置の value を加重平均する操作です。「全位置の key との内積」が必要なため、トークン数 N に対し計算量が O(N^2) になります。N が 196(14×14 パッチ)なら 196^2 ≈ 3.8万ペア、N が 4096(高解像度)なら約1670万ペアと、二乗で爆発します。

「文脈(context)」とは、ある位置の周囲にある情報のまとまりです。歩行者を認識するとき、その位置だけ見ても人かどうか曖昧でも、周囲に横断歩道や信号があれば歩行者だと確信できます。この周囲の情報が文脈です。

「変調(modulation)」とは、ある信号を別の信号で強めたり弱めたりすることです。ラジオの変調と同じ語で、ここでは「位置ごとの特徴を、その位置の文脈に応じて要素ごとに掛け算で強調・抑制する」ことを指します。要素ごとの掛け算なので、ある特徴チャンネルを通したり止めたりするゲートとして働きます。

「焦点(focal)」という語は、近くを細かく・遠くを粗く見るという、写真のピント(焦点)に似た多段階の見方から来ています。人の視覚も、注視点の周りは細かく、周辺視野は粗く処理します。

「ハイブリッド」というカテゴリなのは、純粋な CNN でも純粋な Transformer でもなく、畳み込みで文脈を集めつつ、Transformer 的な「位置ごとに注目先を動的に変える」性質を変調という形で取り込んでいるからです。

なぜ自己注意を置き換えたいのか。理由は計算量です。O(N^2) は高解像度画像(N が数千〜数万トークン)で重すぎます。Swin Transformer は画像を窓に区切って各窓内だけで注意を計算し O(N) に近づけましたが、窓をまたぐ長距離の関係を一度のブロックでは捉えにくいという副作用がありました(窓をずらす shifted window で補うが、間接的)。焦点変調は、窓に区切らずに、文脈集約の段で広域まで一気に見るので、長距離依存を線形コストで素直に扱えます。

## 仕組みを詳しく
焦点変調は3ステップで構成されます。入力特徴を x、形状を [C, H, W] とします。各位置(画素)について次を行います。

ステップ1: 階層的文脈集約(Hierarchical Contextualization)
各位置のまわりの情報を、複数の焦点レベルで集めます。深さ方向畳み込み(チャンネルごと独立の畳み込み)を、カーネルサイズを段階的に大きくしながら L 段(典型的には L=2〜3)積み重ねます。

  z^0 = GELU( DWConv_{k0}(x) )    近傍の文脈(小カーネル、たとえば 3×3)
  z^1 = GELU( DWConv_{k1}(z^0) )   中範囲の文脈(より大きいカーネル)
  z^2 = GELU( DWConv_{k2}(z^1) )   広域の文脈(さらに大きいカーネル)
  z^{L} = GlobalAvgPool(z^{L-1})  画像全体の文脈

各段は前段の出力をさらに畳み込むので、受容野(見る範囲)が積み上がりで広がっていきます。最後にグローバル平均プーリング(画像全体の平均)を加え、最も広い文脈も用意します。これで「すぐ近く」「中くらい」「広域」「全体」という複数スケールの文脈マップ z^0, ..., z^L が手に入ります。重要なのは、各段が前段を入力にするので、L 段でカーネル k なら実効受容野が L×k 程度まで広がり、少ない計算で広い範囲を見られることです。

ステップ2: ゲート集約(Gated Aggregation)
各位置ごとに「どの焦点レベルの文脈をどれだけ使うか」を表すゲート(重み)G を、入力 x からの線形変換で計算します。G の形状は [L+1, H, W] で、各位置・各焦点レベルに1つの重みが対応します。各焦点レベルの文脈マップをこのゲートで重み付けして足し合わせ、その位置専用の変調器(modulator)を1つ作ります。

  modulator = h( Σ_{ℓ=0}^{L} G^ℓ ⊙ z^ℓ )

ここで ⊙ は要素ごとの掛け算、h は 1×1 畳み込みです。物体の細かい縁の位置では近傍の文脈(z^0)の重みが大きく、空のような一様な広い領域では広域文脈(z^L)の重みが大きくなる、というように、位置ごとに文脈の配合が変わります。ゲートが入力依存であることが、固定の畳み込みとの決定的な違いです。

ステップ3: 変調(Modulation)
作った変調器を、その位置の query 特徴(x から線形変換で作る)に要素ごとに掛け合わせます。

  y = q(x) ⊙ modulator

これで「周囲の文脈に応じてその位置の特徴を強調・抑制する」調整が完了します。全画素ペアの関連度行列を一切作らないので、計算量は画素数 N に対し線形 O(N) です。自己注意では query と key の N×N 内積が必須でしたが、焦点変調では query は変調器と1対1で掛かるだけなので、N×N の行列が消えます。

ASCII 図で流れを示します。

  x ─┬─[q proj]──────────────────────────────┐
     ├─[DWConv k0]→z0 ─[DWConv k1]→z1 ─...→zL │
     │      │            │              │      │
     └─[gate]G0,G1,...,GL─┴──────────────┘      │
                       Σ Gℓ ⊙ zℓ → [1x1] = modulator
                                              │
                                  y = q(x) ⊙ modulator ─→

このブロックを Swin / ConvNeXt と同様の4ステージ階層構造で積み上げたのが FocalNet です。Tiny / Small / Base などのサイズがあり、各サイズは対応する Swin と FLOPs・パラメータ数をそろえて比較できるよう作られています。論文は変調器のゲートマップを可視化し、ゲートが物体領域に自然に集中する(背景より物体に高い重みが立つ)ことを示し、解釈性の面でも利点があると報告しました。

なぜこの設計なのか。核心は「文脈を先に作る」ことで全ペア計算を避けつつ、「ゲートで位置ごとに文脈配合を変える」ことで自己注意の動的さ(入力に応じて注目先を変える性質)を保っている点です。固定の畳み込みだけでは入力に依存しない一律処理になりますが、ゲートが入力依存の重みを与えるので、Transformer 的な適応性が残ります。後段の論文分析では、この入力依存ゲートが性能の鍵であり、外すと精度が落ちることが示されています。

## 手法の系譜と主要論文
FocalNet は「自己注意を別の安い演算で置き換えられるのではないか」という、2021〜2022年に活発化した一連の研究の中にあります。

起点の一つは Liu et al. の Swin Transformer (ICCV 2021) です。窓分割と窓のずらし(shifted window)で自己注意の計算量を抑え、階層型 Transformer をバックボーンとして確立しました。多くの効率バックボーンが Swin を比較基準にしています。

重要な分岐点が Yu et al. の MetaFormer / PoolFormer (CVPR 2022) です。彼らは「Transformer の強さは自己注意そのものより、トークン混合層+チャンネル混合層という汎用骨格(MetaFormer)にある」と主張し、自己注意を単純な平均プーリングに置き換えても良い精度が出ると示しました。これは「自己注意は必須ではないかもしれない」という重要な気づきを与えました。

その流れで、トークン混合を何に置き換えるかの探索が進みます。Guo et al. の Visual Attention Network / LKA (2022) は大カーネル深さ方向畳み込みで注意を模しました。HorNet (Rao et al., NeurIPS 2022) は再帰ゲート畳み込みで高次相互作用を作りました。

Yang et al., "Focal Modulation Networks", NeurIPS 2022(Microsoft)はこの探索の中で、「集約してから変調する」という順序反転を提案しました。新規性は、複数スケール文脈を位置依存ゲートで配合し、その配合された文脈で query を変調するという、自己注意とは逆順だが動的さを保つ設計にあります。論文は自己注意と焦点変調を「相互作用と集約の順序が逆」という対の関係で整理し、両者を同じ枠組みで比較できるようにした点も貢献です。後継として、焦点変調は検出・分割用の高解像度バックボーン(FocalNet を DINO 検出器と組み合わせた FocalNet-DINO など)で COCO の上位記録を更新する文脈でも使われ、密予測での強さが実証されていきました。

## 論文の実験結果(定量データ)
FocalNet 論文の主要評価を、指標の意味とともに説明します。

ImageNet-1K 分類(約120万枚・1000クラス。指標は top-1 accuracy)。FocalNet-T(Tiny、約 28〜29M パラメータ、約 4.5 GFLOPs)は top-1 約 82.3% を達成し、同規模の Swin-T(約 81.3%)を約 1.0 ポイント上回りました。FocalNet-S(Small)は約 83.5%、FocalNet-B(Base)は約 83.9% で、それぞれ対応する Swin を上回りました。同じ FLOPs 帯で 1 ポイント前後の差は、ImageNet では明確な優位とみなされます。ImageNet-22K で事前学習した大型版(FocalNet-L/XL/H)はさらに高い精度に達し、特に巨大版を高解像度で使うと最上位水準に届くと報告されました。

COCO 物体検出・インスタンスセグメンテーション(約12万枚、80クラス。指標は box AP と mask AP、0〜100 で大きいほど良い)。Mask R-CNN のバックボーン差し替えで、FocalNet-T は Swin-T を box AP で約 1.5〜2 ポイント上回りました。焦点変調が広域文脈を一度に集約できることが、検出のように物体と背景・他物体の関係が効くタスクで有利に働きます。さらに DINO 検出器と組み合わせた大型 FocalNet は、当時の COCO 検出で box AP 60 超という最上位水準を報告し、密予測での到達点を示しました。box AP 60 超は、当時の SOTA 競争で記録更新に当たる水準です。

ADE20K セマンティックセグメンテーション(約2万枚、150クラス。指標は mIoU)。UperNet ヘッドで、FocalNet-T は Swin-T を mIoU で数ポイント上回りました。全画素を予測する密予測では、各位置が広い文脈を線形コストで参照できる焦点変調の設計が効きます。

アブレーション。論文の重要な分析は次の通りです。第一に、焦点レベル数 L を増やす(近・中・遠の段を増やす)と精度が上がり、ある段数(おおむね 2〜3)で飽和します。多スケール文脈の有効性を示します。第二に、グローバルプーリング(最広域文脈)を外すと精度が下がり、特に密予測で影響が大きく、広域文脈の寄与が確認されました。第三に、位置依存ゲートを外して全焦点レベルを一律に足すと精度が落ち、「位置ごとに文脈配合を変える適応性」が効いていることが示されました。これは焦点変調が単なる固定畳み込みの積み重ねではなく、入力依存の動的演算であることの裏付けです。可視化でも、ゲートマップが物体の輪郭・領域に沿って高い値を持つことが示され、定量と定性の両面で設計の妥当性が裏付けられています。

## メリット・トレードオフ・限界
メリット
- 自己注意を使わず、計算量が画素数に対し線形。高解像度・密予測(検出・セグメンテーション)に向く。
- 同規模の Swin / ViT を上回る精度。特に密予測での優位が大きい。
- 位置ごとに複数スケールの文脈を配合でき、近傍と広域を柔軟に使い分けられる。
- 変調器のゲートマップで「どの位置で・どの広さの文脈を重視したか」をある程度可視化でき、解釈の手がかりがある。これは要素積系の中では珍しい利点。
- 畳み込みベースなので実装・最適化が素直で、専用カーネルを必要としない。

トレードオフ・限界
- 焦点レベル数 L・各段のカーネルサイズなど設計パラメータが増え、調整が必要。
- 自己注意のような明示的な位置間対応(誰が誰を見たか、N×N のアテンション行列)は得られず、解釈はゲートマップ経由の間接的なものになる。
- 広域文脈に大カーネルやグローバルプーリングを使うため、構成次第ではメモリ消費が増える。
- 採用実績・事前学習済み重みの選択肢は ResNet / Swin に比べるとまだ限定的。
- 未解決の課題として、文脈集約の段構成(何段・どのカーネル)をタスクから原理的に決める指針はなく、経験的探索に依存します。

## 発展トピック・研究の最前線
焦点変調は「自己注意の代替演算」という潮流の代表例で、HorNet の再帰ゲート畳み込み、Visual Attention Network の大カーネル注意、状態空間モデル系(VMamba など)と並びます。さらに学ぶなら、まず MetaFormer の視点(トークン混合層を差し替え可能な部品とみなす)を押さえると、これら一連の手法を「同じ骨格で混合器だけ違う」と統一的に理解できます。焦点変調・大カーネル注意・SSM はすべて MetaFormer の「トークン混合器の差し替え」として見ると整理しやすいわけです。検出・分割の最前線では、焦点変調バックボーン+DETR 系検出器(DINO など)の組み合わせが高解像度密予測の強力な選択肢になっています。可視化・解釈の研究としては、変調ゲートマップが物体性(objectness)とどう対応するかの分析が、注意マップとの違いを理解する助けになります。また、文脈集約段を周波数領域フィルタや状態空間モデルで置き換えるハイブリッドも検討の余地があり、線形複雑度の混合器どうしを組み合わせる方向は今後の探索領域です。
