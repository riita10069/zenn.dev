# カメラ間空間アテンション

## ひとことで言うと

複数カメラの特徴を統合する方式のひとつです。画像の各位置について「いま見ている場所では、どのカメラの情報を重視すべきか」をニューラルネットワーク自身に学習させます。この「重視の度合いを動的に決める仕組み」がアテンション(attention、注意機構)で、ここではカメラ視点をまたいで使うので「カメラ間空間アテンション(cross-view / cross-camera spatial attention)」と呼びます。単純に全カメラを足したり連結したりするより、状況に応じた柔軟な融合ができます。

## 直感的な理解

右折しようとする運転者は、自然と右側と後方のミラーに注意を向けます。直進中なら前方に注意が集中します。つまり「いまの状況で頼るべき視界」は刻々と変わります。最も単純なカメラ融合である連結方式(concat)は、全カメラの特徴を固定の重みで混ぜるだけなので、この「状況に応じて重視先を切り替える」柔軟さを持ちません。

アテンションは、この「状況に応じてどこを重視するか」を入力データから動的に計算する仕組みです。カメラ融合に使うと、「いまこの空間位置については、前カメラを80%、右前カメラを20%重視する」といった重みをモデルが自分で算出できます。右折時には右側カメラの寄与が自動的に大きくなる、という挙動を学習で獲得できるわけです。

## 基礎: 前提となる概念

アテンションの中身を噛み砕きます。アテンションは Query(問い合わせ)・Key(鍵)・Value(値)という3つのベクトルで動きます。

- 各要素から Query・Key・Value を作る。
- ある Query と全要素の Key の内積(似ているほど大きい)を取り、softmax で「重み」に変換する。重みは合計1の確率のような分布。
- その重みで Value を加重平均する。これが出力。

噛み砕くと「自分(Query)が、他の全要素(Key)とどれだけ関係するかを測り、関係の強い要素の中身(Value)を多めに集めてくる」操作です。カメラ融合では「要素」がカメラ視点になり、各空間位置でカメラ同士が互いを参照し合います。

- セルフアテンション(self-attention): Query/Key/Value が同じ集合から作られる場合。ここではV枚のカメラ特徴が互いに注目し合う。
- マルチヘッド(multi-head): 注目の仕方を複数(例: 8通り)並列に学習して結合する手法。1種類より豊かな関係を捉えられる。
- 残差接続(residual connection)と LayerNorm: 出力に入力を足し戻し(残差)、数値のばらつきを整える(層正規化)。深い層でも学習を安定させる定番の組み合わせ(Transformer 由来)。
- 位置埋め込み(positional embedding): アテンションは要素の順番を区別しないため、「これは何番目の要素か」を表すベクトルを足して識別させる仕組み。

## 仕組みを詳しく

入力は視点ごとに整形済みの特徴 `[B*V, C, H, W] = [B*V, 256, 8, 8]` だとします。処理手順は次の通りです。

1. 形を組み替える: `[B, V, C, H, W]` に分け直し、`permute` と `reshape` で `[B*H*W, V, C]` にします。これは「各空間位置(全部で B×H×W 個)について、V 枚のカメラの C 次元特徴を並べた列」という見方です。アテンションはこの「V 個の要素の列」に対してかけます。空間位置をバッチ側に畳み込むことで、各位置ごとに独立にカメラ間アテンションを並列計算できます。

2. カメラ位置埋め込みを足す: 学習可能なパラメータ(形 `[1, V, C]`)を加えます。これは「この特徴は何番カメラ由来か」をモデルに教える印です。アテンションは順番を区別しないため、カメラの識別情報を埋め込みで明示的に与える必要があります。この発想は PETR の3D位置エンコーディングと同じ系譜です。理想的には各カメラの実際の3D姿勢(向き・位置)を反映した埋め込みにすると、幾何的な意味づけが強まります。

3. マルチヘッド・セルフアテンションをかける: 各空間位置において V 枚のカメラ特徴が互いに注目し合います(例: 8ヘッド)。出力は残差接続で安定化します。

4. フィードフォワード層(FFN): アテンションの後に `Linear → GELU → Linear` の小さなネットワークを通し、これも残差接続。LayerNorm を前後に挟みます。GELU は ReLU を滑らかにした活性化関数です。

5. カメラ方向に集約: 最後に V 個のカメラ次元を集約(例: 平均)して1つにまとめ、`[B*H*W, C]` から `[B, C, H, W]` の空間特徴に戻します。アテンションで「どのカメラを重視するか」を学習した上で集約するので、単純平均とは違い重要なカメラの寄与が強調されます。

数値例: B=2, V=7, C=256, H=W=8 なら、アテンションは 2×8×8=128 個の空間位置それぞれについて、7枚ぶんの256次元ベクトルを相互参照します。各位置で 7×7 のアテンション行列を計算するイメージです。出力は `[2, 256, 8, 8]` で、入力と同じ空間サイズを保ちます。

なぜこの設計か。各空間位置で独立にカメラ間アテンションをかけると、計算量は「位置数 × V^2」で済み、画像全体の全画素間アテンション(`(H*W*V)^2`)に比べてはるかに軽くなります。一方で、異なる空間位置にまたがる対応(あるカメラの左端と別カメラの右端が同じ物体)は直接は扱えない、という割り切りがあります。この弱点を補うのが、実世界の幾何で位置対応を取る BEV 投影方式です。

## 手法の系譜と主要論文

- Attention Is All You Need(Vaswani et al., NeurIPS 2017, arXiv:1706.03762) — アテンション・マルチヘッド機構・Transformer を提案した原典。Query・Key・Value の内積で「どの要素に注目するか」を計算する枠組みを確立。長距離の関係を直接捉えられる一方、要素数の2乗に比例する計算コストがトレードオフです。
- DETR3D(Wang et al., CoRL 2021, arXiv:2110.06922) — 3Dクエリを各カメラ画像へ投影して特徴をサンプリングし、クエリベースのマルチカメラ融合の流れを作りました。
- PETR(Liu et al., ECCV 2022, arXiv:2203.05625) — 画像特徴に3D位置エンコーディングを足し込み、明示的な幾何投影のサンプリングなしに複数カメラをクエリで横断的に扱う。カメラ位置埋め込みはこの考え方を簡略化したものです。投影演算を埋め込みに置き換えたため実装が単純な反面、深度や3D位置を学習に任せるので明示的投影方式ほど幾何的に正確とは限りません。
- BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270) — BEV クエリが3D参照点をカメラへ投影した位置の特徴に注目する空間クロスアテンションを提案。アテンションを「実世界の幾何に沿って」かける点で、純粋な特徴空間アテンションより一歩進んでいます。
- UniAD(Hu et al., CVPR 2023, arXiv:2212.10156) — 統合クエリで知覚から計画までを貫く設計。クエリ間アテンションでタスクをつなぐ思想は、カメラ間アテンションの発展形と位置づけられます。

大きな流れは「特徴空間でカメラを混ぜるアテンション → 実世界の3D幾何に沿わせたアテンション(BEVFormer)→ タスク横断のクエリアテンション(UniAD)」と進化してきた、とまとめられます。

## 論文の実験結果(定量データ)

指標は nuScenes の NDS / mAP(いずれも高いほど良い。mAP は検出一致度の平均、NDS は位置・大きさ・向き・速度誤差まで含む総合点)を用います。

- Transformer 原論文は機械翻訳(WMT2014 英独)で BLEU 28.4 を達成し、当時の最高水準を更新しました。アテンション機構の有効性を世に示した起点です。
- PETR は nuScenes 検出でおよそ NDS 0.45 前後(バックボーン・解像度により変動)を報告。3D位置エンコーディングを除くアブレーションでは精度が大きく落ち、「カメラの幾何情報を何らかの形で埋め込むこと」が融合アテンションの成否を決めると分かります。これはカメラ位置埋め込みが不可欠であることの定量的裏づけです。
- BEVFormer は nuScenes 検出で NDS 約0.517(報告による、過去フレームあり)を達成。空間クロスアテンションに加え時間アテンションを入れたアブレーションでは、特に速度推定誤差(mAVE)が改善し NDS が数ポイント上がると報告されています。
- マルチヘッド数のアブレーション(Transformer 系一般): ヘッド数を極端に1まで減らすと表現力が落ち精度が下がる一方、増やしすぎても頭打ちになる、という傾向が広く観察されています。8ヘッド前後が経験的なバランス点としてよく使われます。

これらから、カメラ間アテンションの精度を支えるのは「カメラ識別/幾何の埋め込み」と「適切なヘッド数」、そして可能なら「時間情報」だと整理できます。

## メリット・トレードオフ・限界

- メリット: 状況に応じて重視するカメラを動的に切り替えられる(連結方式の固定重みより柔軟)。
- メリット: カメラ位置埋め込みでカメラの識別・幾何情報を扱える。
- メリット: 明示的なカメラ校正(投影行列)が必須でなく、BEV 投影方式より準備が簡単。
- トレードオフ: アテンションは要素数の2乗に比例する計算コストがかかり、空間位置やカメラ枚数が増えると重くなる。
- トレードオフ: カメラ同士を「特徴空間」で混ぜるだけで、実世界の3D座標系を明示的に持たない。鳥瞰図のような幾何的に解釈しやすい表現は得られず、後段の経路計画にとっては BEV ほど扱いやすくない場合がある。
- 限界: 各空間位置で独立にアテンションするため、異なる位置にまたがるカメラ間対応(同じ物体が別位置に写る)を直接は捉えにくい。最後にカメラ方向を集約する際、特定カメラの細かい情報がぼやける可能性もある。

## 発展トピック・研究の最前線

- 幾何条件づきアテンション: BEVFormer の空間クロスアテンションのように、3D参照点を投影した位置だけに注目を限定する変形可能アテンション(deformable attention)を使うと、計算を抑えつつ幾何的に正しい対応を取れます。
- 時間アテンション: 過去フレームの特徴に注目を広げ、動き・遮蔽の補完を改善する流れ。
- 効率化: 線形アテンションやスパースアテンションで2乗コストを緩和する研究。高解像度・多カメラ・長時間履歴を扱う際に重要になります。
- クエリベースへの統合: カメラ間アテンションを独立モジュールに留めず、検出・予測・計画のクエリと一体化して end-to-end 最適化する流れ(UniAD, VAD)。
