# 軌道最適化

## ひとことで言うと
ロボットや車が通るべき経路(軌道)を、「滑らかさ」「障害物回避」「目的地への到達」といった望ましさを数値化したコスト関数として表し、それを最小にする軌道を数学的・数値的に求める計画手法です。

## 直感的な理解
目的地まで行くルートは無数にあります。その中で「ぶつからず、できるだけ滑らかで、無駄に遠回りせず、急ハンドル・急加速をしない」最良のルートを選びたい。人間なら感覚で選びますが、機械にやらせるには「良さ」を数式にする必要があります。

軌道最適化の発想はこうです。まず「悪さ(コスト)」を測る関数を作ります。障害物に近づくほど大きくなる項、軌道がガクガクするほど大きくなる項、ゴールから遠いほど大きくなる項などを足し合わせます。そして、適当な初期軌道(たとえばスタートとゴールを結ぶ直線)から出発し、「この軌道を少し変えたらコストはどっちに動くか」を計算して、コストが下がる方向へ少しずつ軌道を変形していきます。坂道でボールが谷底へ転がるのと同じで、コストの谷底にたどり着いたとき、それが最適な(少なくとも局所的に最良の)軌道です。

```
初期軌道(直線、障害物を貫通)
   ●━━━━━━━━━━━●
        ■ 障害物
        ↓ コストを下げる方向へ変形を繰り返す
   ●━╮      ╭━━●
      ╰──●──╯      滑らかに障害物を回避
```

## 基礎: 前提となる概念

「軌道(trajectory)」は、時間に沿った状態(位置・速度・姿勢など)の列、あるいは経路上の点の列です。離散的に表すなら、N 個の通過点(ウェイポイント)の集まり ξ = [q_1, q_2, ..., q_N] と書けます。各 q は関節角やxy座標などの状態ベクトルです。

「コスト関数(cost function、目的関数とも)」は軌道の悪さを実数で返す関数 C(ξ) です。良い軌道ほど C が小さい。

「最適化(optimization)」は、C(ξ) を最小にする ξ を探す作業です。多くの場合、勾配(gradient、コストが最も急に増える方向を示すベクトル。微分の多次元版)を使い、その逆方向へ進む「勾配降下法」が基本になります。

「制約(constraint)」は、守らねばならない条件です。「障害物にめり込まない」「速度の上限を超えない」「ゴールで止まる」など。制約付き最適化として定式化されます。

「局所最適(local optimum)」と「大域最適(global optimum)」の区別も重要です。谷が複数ある地形では、出発点次第で一番深い谷(大域最適)ではなく近くの浅い谷(局所最適)で止まってしまいます。軌道最適化はたいてい局所最適しか保証しません。

## 仕組みを詳しく

コスト関数はおおむね2つの部分の和で書かれます。

C(ξ) = C_smooth(ξ) + C_obstacle(ξ)

- C_smooth は滑らかさのコスト。隣り合う点の差(速度)や差の差(加速度)の二乗和で表すことが多い。急な動きを罰します。言葉で言えば「カクカクするな、ヌルッと動け」。
- C_obstacle は障害物コスト。軌道上の各点が障害物にどれだけ近いか(あるいはめり込んでいるか)を測り、近いほど大きな値を返します。

これを勾配降下で最小化します。更新式は概念的に
ξ_new = ξ_old − η ∇C(ξ_old)
ここで ∇C はコストの勾配(各点をどう動かせばコストが下がるかの方向)、η は一歩の大きさ(ステップサイズ)。これを繰り返して軌道を少しずつ整えます。手法ごとに、この更新の仕方や制約の扱い方が違います。

### CHOMP(共変ハミルトニアン最適化)
Ratliff ら(2009)。鍵は「共変勾配(covariant gradient)」という考え方です。素朴な勾配降下だと、軌道の表現の仕方(点の数や間隔)によって変形のされ方が変わってしまい、不自然にガタつきます。CHOMP は滑らかさの計量(metric)を使って勾配を整え、表現に依存せず常に滑らかに変形するよう工夫しました。さらに、障害物コストの勾配を効率よく計算できる「符号付き距離場(signed distance field、空間の各点が最も近い障害物までの距離を持つ地図)」を使い、勾配だけで最適化を回します。初期軌道が障害物を貫いていても、滑らかに押し出して回避できるのが特徴です。

### TrajOpt(逐次凸最適化)
Schulman ら(RSS 2013 / IJRR 2014)。CHOMP が勾配のみだったのに対し、TrajOpt は問題を「逐次凸最適化(sequential convex optimization、難しい非凸問題を、解きやすい凸の近似問題の連続として解く)」として定式化しました。障害物回避を不等式制約として明示的に扱い、各反復で局所的に凸近似した部分問題を厳密に解いて進みます。少ない通過点でも頑健に解け、衝突回避の保証が CHOMP より扱いやすいと報告されています。

### iLQR / DDP(動力学を活かす)
iLQR(iterative Linear Quadratic Regulator、反復線形二次レギュレータ、Li & Todorov 2004)と DDP(Differential Dynamic Programming、微分動的計画法、Mayne 1966)は、ロボットの運動方程式(動力学)を明示的に組み込む最適制御の手法です。各時刻の制御入力(アクセル・ハンドルなど)を変数とし、動力学に沿って状態が変化する制約のもとでコストを最小化します。やり方は、現在の軌道のまわりで動力学を線形近似、コストを二次近似し、後ろ向き(終端から始端へ)に最適な制御の修正量を計算、前向きに軌道を更新、を繰り返します。動力学を尊重するので、実際に車が出せる動きの軌道が得られ、モデル予測制御(MPC、未来を予測しながら制御を逐次決める枠組み)の中核としてよく使われます。

## 手法の系譜と主要論文

軌道最適化の理論的源流は最適制御(1950〜60年代、Pontryagin の最大原理、Bellman の動的計画法)にあります。Mayne(1966)の DDP がその古典です。

ロボットアーム・移動ロボットの運動計画では、まずサンプリングベースの手法(RRT、PRM など、ランダムに点を打って繋ぐ方法)が広まりましたが、得られる経路がガタついて後処理の平滑化が必要でした。これに対し「最初から滑らかな最適軌道を直接求めよう」という流れで最適化ベースが台頭します。

- CHOMP(Ratliff ら, ICRA 2009): 共変勾配+符号付き距離場。最適化ベース運動計画の代表的起点。
- STOMP(Kalakrishnan ら, ICRA 2011): 勾配を使わず確率的サンプリングで軌道を更新。微分できないコストも扱える。
- TrajOpt(Schulman ら, RSS 2013, arXiv に近い技術報告あり): 逐次凸最適化で頑健・高速。
- iLQR/DDP 系: 自律運転や脚ロボットのリアルタイム制御、MPC へ広く展開。

自動運転では、これらの考え方が車線維持・車線変更・駐車などの軌道生成に応用され、車両の運動制約(最小回転半径、加加速度=ジャークの制限)をコストや制約として組み込む形で使われています。

## 論文の実験結果(定量データ)

軌道最適化の評価は、(1)成功率(衝突せず目的に到達できた割合)、(2)計算時間、(3)軌道の滑らかさやコスト値、で測られるのが一般的です。

TrajOpt(Schulman ら, 2013)は、ロボットアームの多数の運動計画問題に対し、報告によると CHOMP や STOMP よりも高い成功率と短い計算時間を達成したとされます。特に、少ない通過点(たとえば 10〜30 点程度)でも頑健に解ける効率性が示されました。難しい狭隘環境の計画でも、逐次凸最適化が安定して衝突回避解を見つけられたと報告されています。

CHOMP(Ratliff ら, 2009)は、サンプリングベース手法が出す経路に比べ、後処理の平滑化なしに最初から滑らかで実行しやすい軌道が得られることを実機ロボットで示しました。

これらの比較で繰り返し確認される傾向は次の通りです。最適化ベースは滑らかで質の高い軌道を出せるが、初期軌道に依存し局所最適に陥りうる。サンプリングベースは大域的な探索に強いが軌道が粗い。そのため近年は両者を組み合わせ、サンプリングで大まかな経路を見つけ、最適化で磨き上げるハイブリッドが定量的にも良い結果を出すと報告されています。アブレーションでは、滑らかさコストの重みや初期軌道の与え方が成功率を大きく左右することが示されています。

## メリット・トレードオフ・限界

メリットは、滑らかで実行可能性の高い軌道を直接得られること、複数の要求(安全・快適・効率)をコスト項として柔軟に統合できること、動力学を組み込めばそのまま制御に使えることです。

トレードオフと限界:

- 局所最適問題: 初期軌道が悪いと、障害物の反対側に回るべき場面で間違った側で詰まるなど、悪い解に落ちます。大域性の保証がない。
- 非凸・非滑らかなコストの扱い: 微分できないコスト(離散的な交通ルールなど)は勾配ベース手法では直接扱いにくい。STOMP のようなサンプリング併用が要ります。
- コスト設計の難しさ: 各項の重みのバランス調整が職人技になり、状況依存です。重み付けを誤ると不自然な軌道が出ます。
- 計算時間とリアルタイム性: 高次元・長時間ホライズンでは重くなり、車載のミリ秒単位の要求と衝突します。
- 動的環境への弱さ: 周囲の車や歩行者が動く状況では、予測の不確かさをどう取り込むかが難問として残ります。

## 発展トピック・研究の最前線

近年は、軌道最適化と学習を融合する方向が活発です。コスト関数自体を人間の運転データから学習する(逆強化学習、模倣学習)、初期軌道や最適化の良い当て推量をニューラルネットで予測して収束を速める、微分可能な最適化層をネットワークに埋め込んで end-to-end に学習する、といった研究があります。

さらに、不確かさを陽に扱う確率的・ロバスト軌道最適化、多数のエージェントの相互作用を考慮するゲーム理論的計画、そして拡散モデルで軌道そのものを生成し最適化と組み合わせる手法も登場しています。自動運転では、予測(他車の未来)と計画(自車の軌道)を結合して同時に解く統合枠組みが研究の最前線です。
