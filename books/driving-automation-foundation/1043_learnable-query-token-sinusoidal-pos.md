# 学習可能クエリトークンと正弦波位置エンコーディング

## ひとことで言うと
時系列 attention で「過去フレームの列」を1本の文脈ベクトルに要約するための二つの定番部品です。学習可能クエリトークンは「列全体の要約をここに集めなさい」という専用の入れ物で、正弦波位置エンコーディング（sinusoidal positional encoding）は「これは何ステップ目のフレームか」を attention に教える時刻ラベルです。どちらも Transformer 系の時系列・集約モジュールの中核を支えます。

## 直感的な理解
attention でフレームの列を処理すると、二つの困りごとが出ます。

一つ目。attention は入力を「順番のない集合」として扱うので、フレームを並べ替えても結果が変わりません。しかし「車が近づいている」と「車が遠ざかっている」は、同じフレームの集合を時間の前後だけ入れ替えたものです。順序を捨てると、この二つを区別できなくなります。速度や加速度はまさに順序から生まれる量なので、これは致命的です。そこで各フレームに「君は3秒前、君は2秒前」という時刻の名札を付けてやる必要があります。これが位置エンコーディングです。

二つ目。T本のフレームを入れると attention は T本の出力を返しますが、下流に渡したいのは「全体を1本にまとめた要約」です。会議で全員の発言を聞いた後、議長が1人で要点をまとめるようなものです。この「議長役」を専用に1つ用意し、全フレームに注目させて要約を吸い上げさせる。これが学習可能クエリトークンです。単純に全フレームを平均してもいいのですが、それだと「どのフレームも同じくらい重要」と決めつけることになります。議長役を学習可能にすれば、「決定的な瞬間に重く、退屈なフレームに軽く」という重み付けをデータから獲得できます。

## 基礎: 前提となる概念
「トークン」とは attention に入れる1つの要素のことです。時系列なら1フレーム分のベクトルがトークンです。「クエリトークン」は、データそのものではなく、出力を集めるために用意する特別なトークンで、その中身（初期ベクトル）は学習で決まります。これは文書分類で使う CLS トークン（BERT が列全体の要約を担わせる先頭トークン）と同じ発想です。

self-attention の三役（クエリ・キー・バリュー）を思い出すと、クエリトークンは「私は列全体の要約が欲しい」というクエリを出す存在です。他の全フレームがキーとバリューを提供し、クエリトークンはそれらへの関連度を計算して、関連の強いフレームの中身を多めに集めます。結果として、クエリトークンの出力位置に「全フレームを見渡した要約」がたまります。これは attention 重みが「重み付き平均の係数」であることの直接の帰結で、係数を学習で決められる点が固定の平均との違いです。

位置エンコーディングには大きく二系統あります。固定式（sinusoidal、計算で求めるので学習不要）と学習式（learned positional embedding、各位置のベクトルを学習で獲得する）です。固定式は軽く、学習時より長い系列にも外挿（学習範囲外への当てはめ）が効きやすい一方、タスク特化の位置表現は学習式に劣ることがあります。さらに「絶対位置（何ステップ目か）」を表すか「相対位置（何ステップ離れているか）」を表すかという軸もあり、近年は相対位置を重視する方式（RoPE など）が台頭しています。

## 仕組みを詳しく

### 正弦波位置エンコーディング
各時刻に固有のベクトルを作り、入力に足します。考え方は「時刻 `pos` を、波長の異なる多数の sin/cos の値の組として表す」ことです。

```
pos = [0, 1, 2, ..., length-1]                       各時刻のインデックス
div = exp( arange(0, dim, 2) * (-log(10000) / dim) )  次元ごとに周波数を段階的に下げる
pe[:, 0::2] = sin(pos * div)   # 偶数番目の次元は sin
pe[:, 1::2] = cos(pos * div)   # 奇数番目の次元は cos
```

次元ごとに波長を変える（低い次元は短い波長＝速く回る、高い次元は長い波長＝ゆっくり回る）ことで、近い時刻は似たベクトル、遠い時刻は違うベクトルになります。これは2進数のカウンタに似ていて、低い桁は速く、高い桁はゆっくり変化することで、各時刻に一意なパターンを割り当てます。さらに重要な性質として、「何ステップ離れているか（相対位置）」が線形変換で取り出せます。`PE(pos+k)` が `PE(pos)` の回転行列による線形変換で書けるためで、これにより attention は「2ステップ前」のような相対関係を表現できます。

使い方は、入力射影後の `[B, T, d_model]`（d_model は attention の幅、たとえば512）に対し `x = x + PE(T, d_model)` と足すだけです。T=8 なら8本それぞれに「0番目」「1番目」…「7番目」の時刻ラベルが足され、attention が順序を認識できるようになります。学習パラメータを持たない（固定式で計算する）ので軽く、学習時に見たより長い系列にも外挿が効きやすいのが利点です。

なぜ 10000 という基数を使うのか。これは波長を 2π から 10000·2π まで幾何級数的に分布させ、扱える位置の範囲を十分広く取るための経験的な定数です。Vaswani et al. (2017) がこの値を採用して以来、事実上の標準になっています。時系列メモリで扱う系列長はせいぜい数十なので、この基数は余裕をもって全ステップを区別できます。

### 学習可能クエリトークン
クエリトークンは学習可能なパラメータ `[1, 1, d_model]` として持ち、小さな乱数（標準偏差0.02程度の切断正規分布）で初期化します。forward では、これをバッチ分に複製して履歴列の先頭に連結します。

```
query   = query_token.expand(B, -1, -1)   # [B, 1, d]
x       = cat([query, x], dim=1)           # [B, T+1, d]
x       = encoder(x)                       # [B, T+1, d]  self-attention 複数層
context = out_proj(x[:, 0])                # 先頭(クエリ位置)だけ取り出す → [B, d]
```

ポイントは二つです。第一に、クエリトークンの初期値は乱数ですが、学習を通じて「過去フレームのどこに注目して要約すべきか」を獲得します。固定の平均プーリング（全フレームを単純に平均する）と違い、データから集約の仕方そのものを学べます。たとえば「ブレーキランプが光った瞬間に強く注目する」ような重み付けを、誰も教えなくても学習で身につけられます。第二に、attention 後に取り出すのは先頭の1本（`x[:, 0]`）だけです。これで `[B, T+1, d]` が `[B, d]` になり、時間軸が消えます。なお、クエリトークンには位置エンコーディングを足さない（あるいは専用の固定位置を与える）ことが多く、これは「クエリは特定の時刻に属さない要約役」だからです。

数値例: T=8、d_model=512 のとき、入力射影後は `[B, 8, 512]`。位置エンコーディングを足し、クエリを先頭に付けて `[B, 9, 512]`。エンコーダ通過後 `[B, 9, 512]`、先頭を取って `[B, 512]`、最後に逆射影で元の特徴次元（たとえば視覚896＋自車256＝1152）に戻し、用途別に分割して返します。

```
[ Q  f0 f1 f2 f3 f4 f5 f6 f7 ]   ← Q=クエリトークン, f=各フレーム+位置PE
        self-attention 複数層
[ Q' f0' ... f7' ]               ← Q' に全フレームの要約が集まる
取り出すのは Q' だけ → [B, d]
```

## 手法の系譜と主要論文
- Vaswani et al. (NeurIPS 2017, "Attention Is All You Need", arXiv:1706.03762): sinusoidal positional encoding の原典。self-attention は順序不変なので位置情報を明示注入する必要がある、という問題提起と解決策をここで示しました。効果は学習なしで相対位置を表現でき長い系列に外挿しやすいこと、トレードオフは learned embedding に比べタスク特化の位置表現が得にくい場合があることです。

- Devlin et al. (NAACL 2019, "BERT", arXiv:1810.04805): 列全体の要約を集める特別な先頭トークン（CLS トークン）を導入し、その出力を分類などの下流タスクに使う設計を広めました。可変長の列を固定長の1ベクトルに、しかも学習可能な集約で落とし込める点が貢献。クエリトークンはこの発想を時系列要約に転用したものです。

- Carion et al. (ECCV 2020, "DETR", arXiv:2005.12872): 学習可能な object queries で、画像から固定数の物体を attention で直接読み出す設計を確立しました。「学習可能クエリで可変長の入力を固定数の出力へ集約する」という汎用パターンの代表例です。

- Jaegle et al. (ICML 2021, "Perceiver IO", arXiv:2107.14795): 少数の学習可能 latent クエリで、大規模・任意モダリティの入力を固定長へ圧縮する設計を一般化しました。クエリトークンの「1個のクエリで列全体を要約する」は、この系譜の最小構成に当たります。

- Su et al. (2021, "RoFormer/RoPE", arXiv:2104.09864): 回転位置エンコーディング（Rotary Position Embedding）を提案し、クエリとキーを位置に応じて回転させることで相対位置を attention スコアに自然に織り込みました。sinusoidal の「相対位置が線形変換で取れる」性質を、attention 計算の中に直接埋め込んだ後継として、長系列で標準化が進んでいます。

- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270): 学習可能な BEV クエリと位置情報を組み合わせて時系列・空間融合を行う自動運転手法。「学習可能クエリ＋位置エンコーディング」が時系列・空間融合の定番部品であることを示す実例です。

系譜をまとめると、位置エンコーディングとクエリ集約はともに Transformer（2017）に起源を持ち、BERT（CLS トークン）、DETR と Perceiver（object/latent queries）で「学習可能クエリによる固定長集約」として確立し、位置表現は RoPE 等の相対位置方式へ発展し、BEVFormer 以降は自動運転の時系列・空間融合に標準装備されるようになりました。

## 論文の実験結果（定量データ）
両部品の効果は、それぞれの原典のアブレーションで示されています。

位置エンコーディングについて、Vaswani et al. (2017) は機械翻訳ベンチマーク WMT 英独翻訳で BLEU（翻訳品質の指標、高いほど良い）を評価し、位置エンコーディングを完全に外すと翻訳品質が大きく劣化することを報告しました。論文では sinusoidal と learned positional embedding がほぼ同等の BLEU（28前後）を出すことも示し、固定式でも性能を損なわないと結論づけています。固定式を選ぶ実務的理由は、学習時より長い系列への外挿が効きやすい点です。RoFormer (Su et al., 2021) は、RoPE が長文タスクで絶対位置エンコーディングより安定して良い結果を出すことを報告し、相対位置を直接扱う優位性を示しました。

クエリ集約について、DETR (Carion et al., 2020) は COCO 物体検出で、学習可能な object queries により従来の複雑な後処理（NMS など）を排した end-to-end 検出を実現し、当時の Faster R-CNN 系と同等の AP（Average Precision、高いほど良い。COCO では AP 42〜44 級）を達成しました。Perceiver IO (Jaegle et al., 2021) は少数 latent クエリで画像・音声・点群など多様な入力を固定長へ圧縮しても、各タスクで競争力のある精度を保つことを示しました。これらは「学習可能クエリによる集約」が単純プーリングに対し表現力で優位なことを裏付けます。

時系列メモリの文脈では、クエリトークンによる集約と平均プーリングを比べると、重要フレームに重みを集中できる分だけ要約品質が上がると期待されます。ただしこの2部品は単体で運転性能を測りにくく、attention 型時系列メモリ全体の一部として評価する必要があります。現状、計算特性（軽量・追加パラメータ少）は確認しやすい一方、軌道予測精度（ADE/FDE。予測軌道と正解軌道の平均誤差・終点誤差で、低いほど良い）への寄与は学習済みモデルでの end-to-end 評価を要する段階にあります。

## メリット・トレードオフ・限界
メリット

- 位置エンコーディング: 学習パラメータ不要で軽く、順序情報を attention に与えられ、長い系列にも外挿が効きやすい。
- クエリトークン: 可変長・任意長の履歴を、学習可能な集約で固定長1ベクトルに落とせる。単純プーリングより賢い要約ができる。
- 両者とも標準的で実績のある部品で、実装が簡潔（既存の Transformer エンコーダにそのまま乗る）。

トレードオフ・限界

- sinusoidal は固定式なので、タスク特化の位置表現が要る場合は learned embedding や RoPE に劣ることがある。絶対位置中心のため、相対位置が重要な長系列では不利になりやすい。
- クエリトークンは学習パラメータを増やし、初期化や学習設定に敏感で、データが少ないと過学習しやすい。
- どちらも attention 前提なので、再帰型メモリ（GRU など）には使えず、手法選択と一体になっている。
- 単一クエリで全体を1本に潰すため、複数の異なる側面（速度・遮蔽・周辺車の意図など）を同時に保ちたい場合は表現力が足りないことがある。
- 寄与は単体では測りにくく、attention 型メモリ全体の評価が必要。

研究上の未解決課題として、位置エンコーディングの設計（絶対位置 vs 相対位置、固定 vs 学習、回転位置 RoPE など）と、クエリ数の選び方（1個で足りるか、複数クエリで多面的に要約すべきか）があります。長い系列や複雑な時間構造では、相対位置エンコーディングや複数クエリのほうが有利になる場面が報告されています。

## 発展トピック・研究の最前線
位置エンコーディングは近年、RoPE（Rotary Position Embedding、回転で相対位置を埋め込む方式）や ALiBi（attention のスコアに距離ペナルティを直接足す方式）など、長文・長系列に強い相対位置表現へと発展しています。これらは sinusoidal の「外挿が効く」利点を保ちつつ、相対位置をより自然に扱える点で大規模言語モデルや長系列時系列で標準になりつつあります。

クエリ集約の方向では、単一クエリから複数クエリ・クロスアテンションによる集約（Perceiver 系、Set Transformer）へ拡張する流れがあります。複数のクエリにそれぞれ異なる役割（速度を見るクエリ、遮蔽を見るクエリなど）を学習させれば、1本のベクトルに潰すより豊かな要約が得られる可能性があります。時系列メモリにおいても、こうした多クエリ集約が単一クエリトークンの上位互換として検討されています。集約結果を複数ベクトルで保持すれば、下流の計画モジュールが用途ごとに別々の文脈を引けるため、エンドツーエンド運転における計画精度の向上が期待されています。
