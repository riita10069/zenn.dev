# FPN (Feature Pyramid Network)

## ひとことで言うと

FPN(エフピーエヌ、Feature Pyramid Network = 特徴ピラミッドネットワーク)は、バックボーンが出す「複数解像度の特徴マップ」を賢くつなぎ合わせて、どの解像度でも「細かい位置情報」と「濃い意味情報」の両方を持たせる仕組みです。深い層が持つ意味の濃い特徴を浅い層に流し込むことで、小さい物体も大きい物体も同じように扱えるようにします。物体検出やセグメンテーションでは、バックボーンと出力ヘッドの間に挟む「ネック(neck、首)」の定番部品になっています。

## 直感的な理解

カメラ画像には大小さまざまな物体が同時に写ります。自動運転なら、目の前の大きなバスと、遠くにぽつんと写る小さな歩行者。1つの解像度の特徴だけでは、両方をうまく捉えられません([features_only マルチスケール特徴抽出](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1106_features-only-multiscale) で説明する話と同じ問題です)。

ここで「解像度」と「意味の濃さ」のトレードオフを理解する必要があります。ニューラルネットでは、入力に近い浅い層は空間解像度が高く(細かい位置が分かる)、しかし意味は薄い(「ここに何かエッジがある」程度で「それが車だ」とは分からない)。逆に深い層は空間解像度が低く粗い(位置がぼやける)が、意味は濃い(「これは車だ」と分かる)。小さい物体は浅い高解像度層にしか映らないのに、その層は意味が薄いので「それが歩行者かどうか」が判断しづらい、というジレンマがありました。

なぜ深い層ほど解像度が落ちるかというと、畳み込みネットは計算量を抑えつつ広い範囲を見るために、何段階かで縦横を半分に間引く(プーリングやストライド2の畳み込み)からです。半分に間引くと縦横の情報は4分の1になりますが、その代わり1つの値がカバーする実空間の範囲(受容野)が広がり、「広く見て総合判断する」ことができます。これが意味の濃さを生む一方、位置の細かさを失わせます。

FPN 以前の対処にはそれぞれ難点がありました。

- **画像ピラミッド**: 同じ画像を複数サイズにリサイズし、それぞれをネットワークに通して各サイズで検出する。精度は出るが、画像のたびにネットワークを何度も回すので計算が爆発的に重い。
- **各 stage 特徴をそのまま独立に使う**(SSD, Liu et al., ECCV 2016 の発想): 計算は軽いが、浅い stage の特徴は意味が薄いままなので、小さい物体の検出が苦手。

FPN の問題意識はこうです。「バックボーンは1回通すだけで、すでに複数解像度の特徴を作っている。問題は、浅い高解像度特徴に意味が足りないことだけだ。なら、深い層の濃い意味を、上(深い)から下(浅い)へ流し込んで全ての解像度を底上げすればいい」。これが**トップダウン経路(top-down pathway)**という発想で、追加コストをほぼかけずに全解像度へ意味を注入できます。

## 基礎: 前提となる概念

- **特徴マップ(feature map)**: 畳み込み層が出す `[チャンネル数, 高さ, 幅]` の3次元配列。高さ・幅が空間解像度、チャンネルが「その地点で何が検出されたか」の種類数を表します。
- **stage と stride**: バックボーンは通常、何段階かで空間解像度を半分ずつ落としていきます。各段階を stage と呼び、入力に対する解像度の縮小率を stride(歩幅)と呼びます。1/4、1/8、1/16、1/32 のように倍々で粗くなります。stride 32 とは「出力の1ピクセルが入力画像の 32×32 ピクセルに対応する」という意味です。
- **受容野(receptive field)**: 出力の1ピクセルが、入力画像のどれだけの範囲を「見て」決まっているか。深い層ほど広い。
- **lateral connection(横方向接続)**: 同じ階層の特徴を横から引き込む配線。
- **upsample(アップサンプル)**: 小さい特徴マップを大きく引き伸ばす操作。最近傍補間や転置畳み込みで実装します。
- **IoU(Intersection over Union)**: 検出した枠と正解の枠の重なり具合(重なり面積÷和集合面積)。0〜1 で、検出が正解かを判定する閾値に使います。

## 仕組みを詳しく

FPN は3つの部品でできています。具体的な tensor shape で追います(バックボーンは ResNet、入力 256×256、B はバッチサイズ)。

バックボーンの出力(ボトムアップ経路、これは普通の forward を1回通すだけ):

```
C2: [B,  256, 64, 64]   1/4    浅い・細かい・意味薄い
C3: [B,  512, 32, 32]   1/8
C4: [B, 1024, 16, 16]   1/16
C5: [B, 2048,  8,  8]   1/32   深い・粗い・意味濃い
```

**部品1: 横方向接続(lateral connection)。** 各 stage の特徴に 1×1 の畳み込みを当て、チャンネル数を共通の値(例: 256)に揃えます。1×1 畳み込みは空間サイズを変えずチャンネル数だけ変換する操作です。これで全 stage が同じチャンネル数になり、後で足し算できるようになります。

```
C2 → [B, 256, 64, 64]
C3 → [B, 256, 32, 32]
C4 → [B, 256, 16, 16]
C5 → [B, 256,  8,  8]
```

**部品2: トップダウン経路(top-down pathway)。** 一番深い C5 から始めて、空間サイズを2倍に拡大(upsample)してから、1つ浅い stage の横方向接続出力に足し込みます。これを下まで繰り返します。

```
P5 = lateral(C5)                       [B, 256,  8,  8]
P4 = lateral(C4) + upsample(P5, ×2)    [B, 256, 16, 16]
P3 = lateral(C3) + upsample(P4, ×2)    [B, 256, 32, 32]
P2 = lateral(C2) + upsample(P3, ×2)    [B, 256, 64, 64]
```

upsample で深い特徴を浅い特徴と同じ大きさにそろえ、足し算で意味を注入します。これにより、深い層の濃い意味が浅い高解像度の特徴に流れ込みます。足し算(要素ごとの加算)を使うのは、結合(concat)よりチャンネルが増えず軽い上に、「同じ意味の場所に同じ意味を上乗せする」という意図に合うためです。

**部品3: 出力の整形。** 各 P に 3×3 畳み込みを当てて、upsample で生じたボケ(エイリアシング、折り返し歪み)をならします。最近傍補間で2倍にすると同じ値が2×2に複製され格子状の人工的な縞が残るため、3×3 畳み込みでこれを平滑化します。最終的に P2〜P5(検出器によっては P6, P7 まで)という「全部が 256 チャンネルで、意味も位置情報も持つ」特徴ピラミッドが得られます。

before / after を一言で言うと、before は「浅い特徴=位置は正確だが意味が薄い、深い特徴=意味は濃いが位置が粗い」とトレードオフだったのが、after は「どの解像度も意味と位置を両立」になります。小さい物体は高解像度の P2 で、大きい物体は低解像度の P5 で、それぞれ意味情報を持った状態で検出できます。検出ヘッドは「小さい物体は P2、大きい物体は P5」のように、物体サイズに応じて適切なレベルの特徴を選んで使います。具体的には、物体の大きさ(枠の面積の平方根)を見て、どのレベルに割り当てるかを決める式が使われます。

## 手法の系譜と主要論文

- **Feature Pyramid Networks for Object Detection**(Lin, Dollár, Girshick, He, Hariharan, Belongie, FAIR, CVPR 2017)。FPN の原典。提案は上記のトップダウン+横方向接続。動機は、バックボーンが内在的に持つ多解像度特徴を、追加コストほぼゼロで全解像度に意味注入すること。効果は COCO 検出、特に小物体の精度を大きく改善し、当時の Faster R-CNN などの標準ネックになりました。トレードオフは、ネック分の少量の計算・メモリ増加と、トップダウンのみで横方向の情報統合が一方向になりがちな点です。

- **RetinaNet / Focal Loss**(Lin et al., FAIR, ICCV 2017)。FPN を1段階(one-stage)検出器のネックとして採用し、背景クラスが圧倒的に多い問題を Focal Loss で解決。FPN が二段階検出器だけでなく一段階検出器でも強力なことを示した、FPN 普及の決定打です。

- **PANet: Path Aggregation Network**(Liu et al., CUHK, CVPR 2018)。提案は、FPN のトップダウンに加えて「ボトムアップ経路」も足し、低層の正確な位置情報を上にも伝えること。動機は、FPN が深い意味を下に流す一方で、浅い位置情報を上に上げる経路が弱かったため。効果はインスタンスセグメンテーションでの精度向上。トレードオフは経路追加による計算増です。

- **NAS-FPN**(Ghiasi, Lin, Le, Google, CVPR 2019)。提案は、ネックの接続構造をニューラルアーキテクチャ探索(自動設計)で見つけること。動機は、人手のトップダウン構造が最適とは限らないこと。効果は高精度ですが、探索コストが膨大というトレードオフがあります。

- **BiFPN / EfficientDet**(Tan, Pang, Le, Google, CVPR 2020)。提案は、双方向(トップダウン+ボトムアップ)を繰り返し、各入力に学習可能な重みを付けて融合する効率的なネック。動機は、解像度ごとに寄与が違うので単純な足し算より重み付き融合が良いこと、繰り返すことで情報統合を深めること。効果は精度と効率の両立。トレードオフは構造がやや複雑になることです。

系譜をまとめると、トップダウン1方向(FPN)→ 双方向を1回追加(PANet)→ 構造を自動探索(NAS-FPN)→ 双方向を繰り返し+重み付き融合(BiFPN)、という「より豊かな多解像度統合」への進化として読めます。医療画像由来の U-Net(Ronneberger et al., MICCAI 2015)も、深い意味を浅い高解像度へ戻すという同じ原理を共有する近縁の系譜です。

## 論文の実験結果(定量データ)

評価は主に COCO データセット(約12万枚の学習画像、80 クラス)上の検出 mAP(平均適合率)で行われます。mAP は IoU 閾値 0.5〜0.95 で平均した検出精度で、0〜100(%)、高いほど良い指標です。小物体だけを見る AP_small、大物体だけの AP_large もよく報告されます。

FPN 原論文の代表的なアブレーション。Faster R-CNN のベースライン(C4 の単一解像度特徴のみを使う構成)に対し、FPN を導入すると COCO 検出の AP が数ポイント向上したと報告されています。特筆すべきは、トップダウン経路と横方向接続のどちらか一方を抜くと性能が大きく落ちる点です。論文のアブレーションでは、横方向接続を外すと AP が約10ポイント前後も低下し、トップダウン経路を外しても同様に大きく落ちると報告されています。これは「深い意味を流す経路」と「各解像度の位置情報を引き込む経路」の両方が揃って初めて効く、という設計の本質を示します。小物体の AP_small が特に改善する点も、浅い高解像度特徴へ意味を注入した効果の直接的な証拠です。なぜ横方向接続を外すと激減するかというと、トップダウンだけで上から伝わってくる特徴は何度も upsample を経て位置がぼやけており、横から引き込む高解像度の生特徴がないと細かい位置を取り戻せないからです。

BiFPN(EfficientDet)では、同等の精度を保ちながらパラメータ数と計算量(FLOPs)を従来の FPN ベース検出器より大幅に削減したと報告されています。報告では、特定の精度水準を達成するのに必要な計算量を一桁近く減らせる構成もあり、「重み付き双方向融合の繰り返し」が効率面で効くことを示しました。EfficientDet 系列は COCO で当時としては少ない計算量で高い AP を達成し、精度-効率のフロンティアを押し広げました。これらの結果は、ネックの設計が検出性能とコストの両方を大きく動かす重要要素であることを裏づけています。

## メリット・トレードオフ・限界

メリット

- バックボーンを1回通すだけで、全解像度に意味情報を注入できる。画像ピラミッドより圧倒的に軽い。
- 小物体から大物体まで、それぞれ適した解像度で意味を持った特徴を使える。
- 検出・セグメンテーションの標準ネックとして、多くのヘッドと組み合わせた実績がある。

トレードオフと限界

- ネック分の計算・メモリが追加で必要(ただし画像ピラミッドよりはるかに軽い)。
- 素の FPN はトップダウン一方向で、浅い位置情報を上に伝える経路が弱い(PANet/BiFPN が改良)。
- どの stage を入力に使うか、出力チャンネル数をいくつにするか、upsample の方式といったチューニングが必要。
- **未解決の側面**: 単純な足し算による融合は、解像度ごとの寄与の違いを無視します。BiFPN の重み付き融合や Transformer ベースのネックなど、より柔軟な統合が研究されていますが、最適な統合方式はタスク依存で確立していません。
- なお、検出ヘッドではなく BEV(俯瞰図)生成などの多解像度統合でも、FPN の「1×1 でチャンネルを揃え、解像度を合わせて融合する」発想はそのまま転用されます。多解像度特徴を 1×1 畳み込みで共通チャンネルに圧縮してから融合する設計は、FPN の横方向接続の簡易版とも読めます。

## 発展トピック・研究の最前線

FPN の発想は検出を超えて広がっています。セマンティックセグメンテーションでは Semantic FPN や、エンコーダ・デコーダ構造の U-Net(医療画像由来)が、同じ「深い意味を浅い高解像度に戻す」原理を共有します。近年は Transformer ベースの検出器(DETR 系)で、明示的な FPN を持たずにアテンションで多解像度を統合する流れも出ていますが、Deformable DETR のように依然マルチスケール特徴を活用する手法が主流です。BEV 知覚やマルチカメラ融合の文脈でも、多解像度バックボーン特徴をどう統合するかという問いに対して FPN/BiFPN が標準的な答えであり続けています。研究の最前線は「アテンションと FPN 的階層融合のどちらが、どの条件で優れるか」を見極める方向にあります。また、ViT のように単一解像度の特徴しか出さないバックボーンに対して、人工的にマルチスケール特徴を作る ViTDet(simple feature pyramid)のような研究もあり、「ピラミッドは本当に必要か」という根本的な問い直しも続いています。
