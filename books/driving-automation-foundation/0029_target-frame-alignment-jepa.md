# JEPAターゲットの座標系整合

## ひとことで言うと
将来を予測する自己教師あり学習(JEPA)では、「モデルが予測したもの」と「正解として比べる対象(ターゲット)」を、同じ座標系(同じ視点・同じ原点)で表現しないと、本当は当たっているのに位置ズレのせいで間違い扱いされてしまいます。動く自車の上では特にこの問題が深刻で、「予測とターゲットをどの座標系に揃えるか」が学習信号の質を左右する設計判断になります。

## 直感的な理解

電車に乗っているときに窓の外を撮った写真を考えてください。1.6秒前に撮った写真には「前方30mに駅の看板」が写り、いま撮った写真には同じ看板が「前方20m」に写ります。看板は1ミリも動いていないのに、自分(カメラ)が10m進んだせいで、写真の中での位置が変わったのです。

将来を予測する学習でも同じことが起きます。「いまの景色」から「1.6秒後の景色」を予測させ、実際の1.6秒後の景色と比べて学習するとします。もし予測を「いまの自分視点」で、正解を「1.6秒後の自分視点」で表してしまうと、止まっている看板ですら10mぶんズレて見え、そのズレが全部「予測の誤差」として勘定されます。モデルは正しく世界を理解しているのに、座標系の食い違いのせいで叱られ続けるわけです。本当に学んでほしいのは「他の車や歩行者がどう動くか」という難しい部分なのに、学習信号が「自分が動いたぶんの一律なズレ」で濁ってしまいます。これを正すのが座標系整合(target frame alignment)です。

もう一段かみ砕くと、自車が高速で走っているほど自車運動由来のズレは大きくなり、学習信号に占める「ノイズ」の割合が増えます。停車中なら問題は小さく、高速走行中ほど深刻、という速度依存の罠でもあります。

## 基礎: 前提となる概念

- 自己教師あり学習(SSL, self-supervised learning): 人手のラベルを使わず、データ自身から「予測課題」を作って学ぶ学習法です。たとえば「未来を予測させてその答え合わせをする」のは、未来のデータ自体が答えなので人手ラベルが要りません。
- JEPA (Joint Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ): 「将来や欠損部分を、画素ではなく特徴(意味のある数値表現)の空間で予測し、実際の特徴と比べて学習する」枠組みです。画素まで正確に当てる必要がないので、影やテクスチャのような無関係な細部に煩わされません。
- ターゲット(target): 予測の答え合わせに使う「正解」の特徴表現。多くの場合、未来フレームを固定された(frozen)エンコーダに通して作ります。固定する理由は別項(表現の崩壊防止)で詳述します。
- MSE (Mean Squared Error、平均二乗誤差): 予測と正解の差を二乗して平均した値。小さいほど予測が当たっています。座標系がズレていると、止まった物体の見かけのズレもこの値に乗ってしまいます。
- ego frame(自車座標系): 自車を原点とした座標系。前方を x 軸などと決めます。時間が経つと自車が動くので、原点も向きも時刻ごとに変わります。
- egomotion(自車運動): 自車自身がどれだけ移動・回転したか。`(dx, dy, dθ)` のように前後・左右の移動量と回転角で表せます。GPS/IMU やオドメトリ(車輪回転やセンサーから自車の移動量を積算する手法)で得られます。
- warp / resample(ワープ、再サンプリング): あるマップを別の座標系へ「描き直す」幾何変換。座標を付け替え、新しいグリッド上の各点の値を元マップから補間して埋めます。学習可能なパラメータを持たない純粋な幾何操作にでき、Spatial Transformer Network (Jaderberg et al., 2015) の grid_sample がその実装基盤です。

## 仕組みを詳しく

座標系整合の論点は、将来予測を「既知の動き」と「未知の動き」に分解するアイデア(motion-grounded residual、運動接地残差)とセットで理解すると明快です。将来の特徴マップを次のように分けます。

```
z_{t+Δ} ≈ Warp_ego(z_t)        +   Δ_dynamic
          └ 既知の自車運動で  ┘     └ 学習して予測する ┘
            現在の特徴を未来          動的な残差
            視点へ座標変換           (他車・歩行者の動き)
```

- `Warp_ego(z_t)`: 現在の特徴マップ `z_t` を、既知の自車運動 `(dx, dy, dθ)` を使って「未来の自車視点」へ resample する、パラメータを持たない幾何変換。静止している世界(看板、路面、建物)は、自車が動いただけなので、この warp でほぼ正確に未来視点へ移せます。
- `Δ_dynamic`: 静的世界の warp では説明できない残り、つまり他車や歩行者が実際に動いたぶんだけを、ネットワークが予測します。学習対象がこの残差だけに絞られるため、必要な表現容量が小さくなり学習が安定します。

ここで座標系整合が立ち上がります。`Warp_ego(z_t)` は定義上「未来のエゴ座標系」で書かれます。したがって予測 `= Warp_ego(z_t) + Δ_dynamic` も未来座標系です。すると比較対象のターゲット(未来フレームを frozen エンコーダに通した特徴)も、同じ未来座標系で表現しなければ、両者を MSE で比べる前提が崩れます。これがこの論点の核心です。

座標系を揃える具体策のイメージ:

1. 予測側は `Warp_ego` により最初から未来座標系で出る。
2. ターゲット側は未来フレームの画像から作る。未来フレームは「未来の自車が実際に見た景色」なので、これ自体がすでに未来座標系に近い。両者の原点・向きの定義を厳密に一致させる。
3. その結果、MSE が「純粋に動的な予測の良し悪し」だけを測るようにする。

数値例で確かめます。自車が1.6秒で前方に10m進んだとします。

```
座標系を揃えない場合:
  止まっている看板  予測(t 視点) = 前方30m,  ターゲット(t+1.6s 視点) = 前方20m
  → 10m の差が「誤差」として全マスに乗る(本当は完璧に予測できているのに)

座標系を揃えた場合(Warp_ego で予測を未来視点へ寄せる):
  止まっている看板  予測 = 前方20m,  ターゲット = 前方20m  → 誤差ほぼゼロ
  残る誤差は「動いた他車の予測ズレ」だけ ← これこそ学習したい信号
```

整合をどちらの座標系で取るかには選択肢があります。FIERY 流に「すべてを現在視点へ揃える」方式と、上記のように「未来視点へ揃える」方式があり、予測タスクの定義(現在から将来を見るのか、将来時点で評価するのか)に応じて選びます。重要なのは「予測とターゲットが同一の座標系」であって、どちらの座標系を選ぶか自体は設計判断です。

### warp の元データをどうするか(train/test 一貫性)

座標系整合とセットで議論されるのが「warp に使う自車運動をどこから取るか」です。

- 学習時: 実際に起きた未来軌跡(ground-truth future egomotion)を使えば、warp は正確でクリーンになります。データに記録された答えがあるからです。
- 推論時: 未来の正解はまだ存在しません。これからどう走るかを出力する planner(プランナー、自車の将来軌跡を予測するモジュール)の予測軌跡を使うしかありません。

ここで「学習は正解軌跡、推論は予測軌跡」と食い違うと、train/test 不一致(exposure bias、露出バイアス。学習時に経験しない入力分布に推論時いきなり晒される問題)が生じます。学習中に一度も経験していない「下手な軌跡で warp された入力」に推論時いきなり直面するからです。折衷案として scheduled sampling(スケジュールドサンプリング、Bengio et al. 2015 が seq2seq で提案)が使えます。最初は正解軌跡で warp し、学習が進むにつれ確率的に予測軌跡へ切り替えていく方法です。これにより座標変換の基準が学習後半で推論時条件に近づき、整合がより厳密かつ頑健になります。

## 手法の系譜と主要論文

- LeCun, "A Path Towards Autonomous Machine Intelligence" (2022, OpenReview)。JEPA を提唱した位置づけ論文。画素予測は無関係な細部まで当てさせられ非効率なので、意味のある潜在空間で将来や欠損を予測すべきだと主張しました。座標系整合の観点では、潜在空間で比べる以上「予測とターゲットが同じ前提(座標系)で書かれている」ことが暗黙の必要条件だと示唆します。
- Assran et al., "I-JEPA: Image-based Joint-Embedding Predictive Architecture" (CVPR 2023, arXiv:2301.08243)。JEPA を画像で実装。文脈ブロックからマスクされた標的ブロックの特徴を予測し、ターゲット位置を positional embedding で指定します。「予測がどの位置のターゲットに対応するか」を明示する設計は、空間的な対応づけ(=座標整合)を予測アーキテクチャに組み込む先例です。
- Jaderberg et al., "Spatial Transformer Networks" (NeurIPS 2015, arXiv:1506.02025)。微分可能な幾何変換層を提案。`Warp_ego` のような座標変換を、勾配を通せる形でネットワークに組み込む技術基盤(grid_sample / affine grid)を与えました。
- Hu et al., "FIERY: Future Instance Prediction in BEV" (ICCV 2021, arXiv:2104.10490)。サラウンドカメラから BEV を作り将来をセル単位で予測。将来 BEV を扱う際、フレーム間で自車が動くため「どの座標系で将来を表すか」を明示設計する必要があることを実務的に示しました。FIERY は過去フレームを現在の自車視点へ spatial transformer で warp して揃えてから統合する、という座標整合を内部で行っています。
- Hu et al., "MILE: Model-Based Imitation Learning for Urban Driving" (NeurIPS 2022, arXiv:2210.07729)。潜在ワールドモデルで将来 BEV を予測。自車運動と他車運動を内部状態で扱い、将来を一貫した枠組みで予測する設計は、座標系を意識した将来表現の先例です。
- Bengio et al., "Scheduled Sampling for Sequence Prediction with RNNs" (NeurIPS 2015, arXiv:1506.03099)。本来は言語生成の文脈ですが、「学習時の入力(正解)と推論時の入力(自分の予測)の食い違い」を、学習中に徐々に自分の予測へ切り替えて埋める一般的な手法です。warp の元データを正解軌跡から予測軌跡へ移す折衷策の理論的な裏付けになります。

系譜的な接続として、将来を「現在との差分」として予測する考え方(時間差分の予測)は、`Warp_ego(z_t) + Δ_dynamic` の分解と同じ精神を共有します。「全部を一から予測する」より「既知の変換 + 残差」の方が、学習すべき量が小さく安定するという発想です。これは光学フロー推定や残差学習(ResNet の恒等写像 + 残差)とも通底する、機械学習に広く現れる原理です。

## 論文の実験結果(定量データ)

座標系整合そのものを単独で測ったベンチマークは確立されていませんが、関連する設計の効果は将来 BEV 予測の文脈で定量化されています。

- FIERY (Hu et al., 2021): nuScenes での将来予測を Future IoU と Video Panoptic Quality (VPQ) で評価。過去フレームを現在視点へ warp して座標を揃える時空間統合を備えたことが、将来 IoU の向上に寄与しました。アブレーションで過去の時間統合を外すと将来予測の品質が落ちることが示され、フレーム間の座標整合を伴った履歴利用の重要性が裏づけられています。指標 IoU は「予測した占有領域と正解領域の重なり率」で、1に近いほど良いです。
- MILE (Hu et al., 2022): CARLA closed-loop で driving score を評価し、画素予測ベースや純粋な模倣学習を上回りました。自車運動を内部状態で明示的に扱い将来を一貫座標で予測する設計が、未知の町(unseen town)への汎化で特に差を生んだと報告されています。
- I-JEPA (Assran et al., 2023): 画素再構成系(MAE 等)より少ない事前学習計算で、ImageNet linear probe で競争力ある精度を達成。標的ブロックの位置をマスクトークンで明示し、文脈から予測する設計が効率的な表現学習につながることを示しました。位置の対応づけを明示することの効用を間接的に支持します。
- scheduled sampling (Bengio et al., 2015): 画像キャプション生成などで、常に正解を入力する teacher forcing のみより、推論時の自己予測入力への露出を学習中に混ぜる方が生成品質(BLEU 等、生成文と正解文の n-gram 一致を測る指標)を改善することを示しました。これは warp の元データを正解→予測へ移す折衷策が train/test 不一致を緩和しうる、という一般的な根拠になります。

定性的に重要なのは、「座標系を揃えると、誤差のうち自車運動由来の一律成分が消え、動的予測の誤差だけが残る」ため、同じネットワーク容量を本当に難しい部分(他車の運動予測)に集中できるという点です。これは指標の数値以前に、学習信号の質を根本から変えます。揃えないまま学習すると、損失の大半を自車運動由来のズレが占め、勾配が「動く物体の予測」より「静的背景の見かけの流れ」に引っ張られてしまいます。

## メリット・トレードオフ・限界

メリット
- 予測とターゲットが同じ座標系になり、MSE が「純粋に動的な予測誤差」だけを測る。学習信号がクリーンになる。
- 自車運動(既知)と他車運動(不確実)を分離でき、ネットワークは難しい後者だけに容量を割ける。
- `Warp_ego` で自車運動ぶんを先に確定させると予測が入力に固定(アンカー)され、表現の崩壊を防ぐ副次効果がある(別項参照)。

トレードオフ・限界
- warp に使う自車運動を学習時(正解軌跡)と推論時(予測軌跡)で揃えないと train/test 不一致が生じる。scheduled sampling などの追加工夫が要る。
- 座標変換(resample)の実装やフレーム定義を間違えると、かえって誤差を増やす。整合の正しさを検証する手間がかかる。補間の精度、グリッド境界の扱い(視野外に出た領域をどう埋めるか)、回転の符号など細部に落とし穴が多い。
- 学習初期はプランナーの予測軌跡が下手なので、予測軌跡で warp すると残差ターゲットが乱れる。
- 研究上の未解決課題: 自車運動の推定誤差(GPS/IMU やオドメトリのノイズ)が warp を通じてターゲットに混入する。どの座標系を基準にするのが最適か(現在視点に揃える FIERY 流か、未来視点に揃えるか)も問題依存で、確立した結論はありません。

## 発展トピック・研究の最前線

- 差分予測との統合: 将来を「既知の幾何変換 + 学習する残差」に分ける考え方は、座標整合を構造として組み込む自然な方法です。残差予測の方が学習量が小さく安定します。
- 多視点・多エージェント整合: 自車だけでなく他車視点や HD 地図座標へ揃える研究が進んでおり、協調運転(V2X、車車間/路車間通信)では複数車両の座標系をどう統一するかが新たな整合問題になります。
- 不確実性つき warp: planner 軌跡の不確実性を warp に伝播させ、ターゲット側でも不確実領域を重み下げする方向は、推論時の頑健性を高める研究テーマです。軌跡が不確かなほどターゲットを甘く採点する、という適応的な重みづけが考えられます。
- 動画 JEPA への展開: V-JEPA など時間方向の JEPA では、フレーム間の運動をどう座標づけて予測するかが本質的な設計点になり、本項の整合問題が一般の動画表現学習にも波及します。
