# モデル予測制御(MPC)

## ひとことで言うと

未来を一定時間だけ予測し、その範囲で「最も良い操作の列」を最適化問題として解き、その先頭の操作だけ実行して、また次の瞬間に予測しなおして解きなおす——この繰り返しで車やロボットを動かす制御手法です。自動運転の軌道追従(決められた経路を正確になぞる)で広く使われます。

## 直感的な理解

カーリングや車庫入れを思い浮かべてください。次の一手だけ考えて動くのではなく、「この先 3〜4 手まで読んで、その中で一番うまくいく最初の一手を打つ」。打った後、状況が変わったらまた読みなおす。これが MPC の発想です。

キーワードは「receding horizon(後退ホライズン)」です。ホライズンとは「先読みする時間窓」のこと。たとえば 2 秒先まで読むとして、0.1 秒後に 1 手進めたら、また「そこから 2 秒先」まで読みなおす。窓が常に自分の前方に後退していくので receding horizon と呼びます。なぜ全部の操作列を一度に決めて実行しないのか? 現実は予測どおりにならない(風、路面、他車の動き)からです。少し進むたびに観測しなおして補正するので、誤差や外乱に強くなります。

PID 制御(誤差に比例・積分・微分して操作量を決める古典制御)との違いも押さえましょう。PID は「目標とのずれ」に反応するだけで未来を読みません。また物理的な制約(ハンドルを切れる角度の上限、加速度の限界)を直接は扱えません。MPC は未来を予測しつつ、制約を最適化問題の「条件」として明示的に入れられる。ここが決定的に強い点です。

## 基礎: 前提となる概念

### 状態・入力・モデル

- 状態(state)`x`: その瞬間のシステムの状況を表すベクトル。車なら例えば `[位置x, 位置y, 向き θ, 速度 v]` で shape `[4]`。
- 入力/操作(control input)`u`: 自分が出せる操作。車なら `[加速度 a, 操舵角 δ]` で shape `[2]`。
- モデル(dynamics)`f`: 「今の状態と操作から次の状態がどうなるか」を表す関数。`x_{t+1} = f(x_t, u_t)`。MPC の「M(モデル)」はこれを指す。

車の典型モデルに「自転車モデル(kinematic bicycle model)」があります。前輪と後輪を 1 つずつにまとめた簡略化で、操舵角と速度から位置・向きの変化を計算します。

### 予測ホライズン

- 予測ホライズン `N`: 何ステップ先まで読むか。例: 1 ステップ 0.1 秒で N=20 なら 2 秒先まで。
- 制御ホライズン: 操作を変えてよいステップ数(予測ホライズンと同じか短く設定)。

## 仕組みを詳しく

MPC は各時刻で次の最適化問題を解きます。

```
minimize  Σ_{k=0..N-1} [ (x_k - x_ref_k)^T Q (x_k - x_ref_k) + u_k^T R u_k ]
                         + (x_N - x_ref_N)^T P (x_N - x_ref_N)
subject to  x_{k+1} = f(x_k, u_k)         (ダイナミクス制約)
            u_min ≤ u_k ≤ u_max           (操作の上下限)
            x_min ≤ x_k ≤ x_max           (状態の制約: 例 車線内)
            x_0 = 現在の観測状態
```

記号の意味を言葉で説明します。

- `Σ_{k=0..N-1}` はホライズン全体にわたる総和。各ステップのコストを足し上げる。
- `x_k - x_ref_k` は予測した状態と「目標軌道(reference)」とのずれ。これを小さくしたい。
- `Q` は状態のずれをどれだけ重く見るかの重み行列。大きいほど軌道追従を厳しくする。
- `u_k^T R u_k` は操作の大きさへのペナルティ。`R` が大きいほど「急ハンドル・急加速をしない」ように罰する。乗り心地・燃費・滑らかさに効く。
- 最後の項 `(x_N - x_ref_N)^T P ...` は終端コスト(terminal cost)。ホライズンの最後の状態を特に重視し、安定性を保証するために使う。
- ダイナミクス制約はモデル `f` どおりに状態が遷移することを強制する。
- 操作・状態の上下限が、ハンドル角の限界や車線逸脱禁止などの物理・安全制約。

解くと操作列 `u_0, u_1, ..., u_{N-1}` が得られますが、実行するのは先頭の `u_0` だけ。1 ステップ進んで状態を観測しなおし、また同じ問題を解く。これが receding horizon の核心です。

ASCII で時間の進み方を図示します。

```
時刻 t:   読む範囲 [t ........ t+N]  → u_0 を実行
時刻 t+1:    読む範囲 [t+1 ........ t+1+N]  → 新しい u_0 を実行
時刻 t+2:       読む範囲 [t+2 ........ t+2+N]  → ...
           窓が右に「後退」しながら毎回解きなおす
```

### 線形か非線形か

- モデル `f` が線形でコストが二次関数なら「線形 MPC(QP, 二次計画問題)」になり、高速かつ最適解が一意に求まる。
- 車の運動は本質的に非線形(向きが sin/cos で効く)なので、現場では非線形 MPC(NMPC)や、各時刻で線形化して QP として解く方法が使われる。

## 手法の系譜と主要論文

MPC は機械学習より歴史が古く、化学プラント制御から始まりました。

- 産業起源(1970〜80 年代): Richalet らの MPHC(1978)、Cutler & Ramaker の DMC(Dynamic Matrix Control, 1980)。石油化学プラントで多変数・制約付き制御の需要から生まれた。
- 理論的整備: Mayne et al., "Constrained model predictive control: Stability and optimality"(Automatica, 2000)が安定性・最適性の理論を体系化した代表的サーベイ。終端コスト・終端制約による安定性保証はここで明確化された。
- 自動運転への応用: Falcone et al.(IEEE Trans. Control Systems Technology, 2007)が氷雪路でのアクティブステアリングに MPC を適用した先駆例。以後、車線維持・回避・レーシングで標準的に。
- 学習ベース MPC(learning-based MPC): モデル `f` をデータから学習する、あるいは未知の外乱を学習で補正する流れ。Gaussian Process MPC(Hewing et al., 2020)はモデル誤差をガウス過程で推定して安全余裕を確保する。Aswani et al.(2013)は学習でモデルを改善しつつ安全制約を保つ枠組みを提案。
- MPC と強化学習・模倣学習の融合: MPC を「専門家(expert)」として使い、その出力を模倣してニューラルネットに高速な方策を蒸留する(MPC は計算が重いので、学習した方策でリアルタイム化する)。逆に、学習したコスト関数やダイナミクスを MPC に組み込む研究もある。
- 微分可能 MPC(Amos et al., NeurIPS 2018, arXiv:1810.13400): MPC の最適化を微分可能な層として埋め込み、コストやダイナミクスのパラメータを勾配で学習できるようにした。エンドツーエンド学習と古典制御の橋渡し。

## 論文の実験結果(定量データ)

MPC は手法であり単一の標準ベンチマークがないため、応用論文ごとの定量結果を紹介します。いずれも報告による数値です。

- Falcone et al.(2007)の氷雪路ステアリング実験では、MPC がダブルレーンチェンジ(障害物回避の標準試験)を、PID 系の単純制御が破綻する速度域でも安定に追従できたと報告されている。横方向の追従誤差と安定性で優位を示した。
- 微分可能 MPC(Amos et al., 2018)は、模倣学習タスクで、純粋なニューラルネット方策に比べて少ないデモンストレーションで高い成功率を達成したと報告。理由は MPC の構造(物理的なモデルと制約)が事前知識として効き、サンプル効率が上がるため。
- Gaussian Process MPC 系(Hewing et al., 2020)は、モデル誤差を確率的に見積もることで、名目モデルだけの MPC より制約違反の確率を抑えつつ追従性能を保てると報告。安全余裕(safety margin)を学習で適応的に調整できる点が利点。
- レーシング応用の MPCC(Model Predictive Contouring Control)では、ラップタイムを目的に組み込むことで、人間ドライバーに匹敵するライン取りが得られたとする報告がある。

アブレーション的な知見として共通するのは、(1) ホライズン N を伸ばすほど先読みが効き安定するが計算が重くなる、(2) 重み Q/R のバランスが追従精度と滑らかさのトレードオフを直接決める、という点です。

## メリット・トレードオフ・限界

メリット

- 制約(操作限界・安全領域)を最適化の条件として明示的に扱える。これは PID では難しい最大の強み。
- 多入力多出力(操舵と加減速を同時)を自然に統一的に扱える。
- 未来を読むので、先のカーブや障害物に対して先回りした滑らかな挙動になる。

トレードオフ・限界

- 計算が重い。毎時刻、最適化問題を解くため、リアルタイム性(数十 Hz)を出すには高速ソルバや線形化、あるいは学習による近似が要る。
- モデル依存。`f` が現実とずれると性能が落ちる。learning-based MPC はこの弱点をデータで埋めようとするが、学習モデルが外挿で誤ると危険。
- 重みの調整(チューニング)が職人芸になりがち。Q/R/N/P の設定で挙動が大きく変わる。
- 非凸・非線形だと局所最適に陥る、解が見つからない(infeasible)場合の扱いが必要。

## 発展トピック・研究の最前線

- リアルタイム最適化: acados, OSQP, FORCES などの高速ソルバ。組込み GPU/専用ハードでの実行。
- 確率的 MPC / ロバスト MPC: 不確実性を陽に扱い、確率的な制約(chance constraint)で「制約違反の確率を一定以下に」する定式化。
- 学習との融合: コスト関数を逆強化学習で人間の運転から学ぶ、ダイナミクスをニューラルネットで学ぶ、MPC を教師に方策蒸留する。
- 対話的・予測的プランニング: 他車の将来挙動を予測しながら相互作用を考慮する MPC(交差点・合流)。
- エンドツーエンド学習との位置づけ: ニューラルネットが直接操作を出す方式に対し、MPC は安全制約と解釈性を保てる「構造を持ったプランナ」として残り、両者のハイブリッドが研究の中心になりつつある。
