# Autoformer

## ひとことで言うと

Autoformer は「長い時系列をできるだけ遠くまで一気に予測する」ために設計された Transformer の一種です。普通の Transformer が「ある時刻の値」と「別の時刻の値」を一点ずつ比べて関連を学ぶのに対して、Autoformer は時系列をまず「ゆっくりした全体の傾き(トレンド)」と「繰り返す波(季節性・周期)」に分けたうえで、点ではなく「周期のかたまり同士」がどれだけ似ているかを使って情報を集めます。これにより、毎日・毎週といった繰り返しが強いデータで、長い先まで誤差を爆発させずに予測できるようになりました。提案当時、長期予測の主要ベンチで誤差を平均して大きく削減し、長期時系列予測 Transformer の代表的な系譜の起点の一つになりました。

## 直感的な理解

電力の消費量を考えてみます。朝に人が起きて電気を使い始め、昼に落ち着き、夕方にまた上がり、夜に下がる。これがほぼ毎日繰り返されます。さらに平日と週末では波の形が違い、夏と冬では全体の水準が上下します。このデータを「3日先まで予測したい」とき、人間が直感的にやるのは「だいたい昨日と一昨日の同じ時間帯はこのくらいだったから、明日の同じ時間帯もこのくらいだろう」という推論です。つまり「いまの時刻」と「1日前の同じ時刻」「2日前の同じ時刻」を結びつけているわけです。

普通の Transformer の自己注意(self-attention、系列の中のどの時刻同士が関連するかを学ぶ仕組み)は、原理的にはこの「1日前の同じ時刻」を見つけられるはずです。しかし実際には、ノイズの多い生の時系列を一点ずつ突き合わせると、似ている点が大量に散らばって注意がぼやけてしまいます。「昨日の朝7時」と「いまの朝7時」だけでなく、たまたま値が近い無関係な時刻にも注意が分散するのです。さらに、予測区間が長くなるほど、点ベースの注意は「どこを見ればいいか」が定まらず、出力が平らになったり振動したりしがちです。

Autoformer のアイデアは、この人間の直感をそのまま機構にすることです。第一に、データを「全体の傾き(トレンド)」と「繰り返す波(周期)」にあらかじめ分けます。傾きを取り除いた純粋な波だけを見れば、周期構造がくっきりします。第二に、「いまの波形」と「τ ステップずらした波形」がどれだけ似ているか(これを自己相関と呼びます)を計算し、よく似ている周期 τ を見つけて、その周期ぶんずらした部分系列をまとめて足し合わせます。点と点ではなく、周期のかたまり全体を集約するのです。「似ている瞬間を一点だけ探す」のではなく「似ている周期だけ抜き出してまとめて使う」という、粒度の引き上げが本質です。

## 基礎: 前提となる概念

このノートを読むのに必要な言葉を、知らない前提で一つずつ噛み砕きます。

時系列(time series)とは、時間の順番に並んだ数値の列です。例えば1時間ごとの気温 `[12.1, 12.8, 13.5, ...]` のようなものです。長期予測(long-term forecasting)とは、過去のある長さ(例: 過去96時間)を入力として、未来の長い区間(例: 先720時間)をまとめて予測するタスクを指します。先が長いほど誤差が積み重なりやすく、難しくなります。これに対し、1ステップだけ先を当てる短期予測は比較的簡単です。長期予測の難しさは「未来のどこを見ても直接の手がかりがない区間まで埋めなければならない」点にあります。

トレンドと季節性という分解は、古典的な時系列分析の基本です。トレンド(trend)は長期的にゆっくり変わる水準(例: 年々上がっていく平均気温)、季節性(seasonality)あるいは周期成分は決まった間隔で繰り返す変動(例: 毎日の昼夜サイクル)を指します。両者を足すと元の系列に戻る、と考えます。統計学の古典的手法 STL(Seasonal-Trend decomposition using Loess)などが、まさにこの分解を前処理として行ってきました。Autoformer はこの古い知恵をニューラルネットの内部に取り込んだ、と言えます。

自己注意(self-attention)は Transformer の中心部品です。系列の各時刻が「他のどの時刻に注目すべきか」を、クエリ(query)・キー(key)・バリュー(value)という3つのベクトルの内積で重みを作って決めます。系列長 L に対して全時刻ペアの重みを作るので、計算量とメモリが L の二乗(O(L²))に比例して増えます。長い系列ではこれが致命的に重くなります。L=720 なら全ペアは約52万、L=1440 なら約207万と、二乗で膨らみます。

自己相関(autocorrelation)は時系列分析の古典的な量で、系列を時間 τ だけずらしたものと元の系列がどれだけ似ているかを測ります。式では R(τ) = Σ_t X_t · X_{t-τ} のように、ずらした系列との内積で定義します。τ=24 でずらして相関が高ければ「この系列は24時間周期を持つ」と分かります。自己相関のグラフ(コレログラム)のピークが、その系列の主要な周期を教えてくれます。

高速フーリエ変換(FFT, Fast Fourier Transform)は、信号を周波数成分に分解する計算を高速に行うアルゴリズムです。自己相関はウィーナー・ヒンチンの定理(Wiener–Khinchin theorem、信号の自己相関とパワースペクトルがフーリエ変換の関係にあるという定理)により「FFT してパワースペクトルを取り、逆 FFT する」ことで、全てのずれ τ に対する値を一度に O(L log L) で計算できます。これが Autoformer の効率の鍵です。

## 仕組みを詳しく

Autoformer は二つの新しい部品からできています。系列分解(Series Decomposition)を Transformer の内部に何度も挟むことと、自己注意を Auto-Correlation(自己相関機構)に置き換えることです。

### 系列分解をモデル内部に組み込む

まず分解の中身です。トレンドは移動平均(moving average、ある窓幅で平均を取って滑らかにする操作)で取り出します。窓幅を k として、

```
trend  = AvgPool1d(padding=k//2, kernel=k)(X)   滑らかな長期傾向
season = X - trend                              滑らかさを引いた残りの波
```

`AvgPool1d` は1次元の平均プーリングで、各時刻の周囲 k 点の平均を取ります。これでギザギザした周期変動が均されて、ゆっくりした傾きだけが残ります。それを元の系列から引けば、周期成分(季節成分)が得られます。両端は系列の最初と最後の値で埋める(padding)ことで、長さを保ちます。

古典的な時系列分析でも前処理として一度だけ分解しますが、Autoformer の新規性は「これを各ブロックの中で繰り返し行う」ことです。予測を進める途中でもトレンドが少しずつ見えてくるので、その都度トレンドを抜き出して累積し、残った季節成分だけを注意機構に通します。これを「Series Decomposition Block」と呼びます。一度きりの前処理では捉えきれない「予測の途中で新たに現れるトレンド」を逐次拾えるのが利点です。

### Auto-Correlation Mechanism(自己相関機構)

自己注意の置き換えがもう一つの核です。手順は次の通りです。

1. クエリ Q とキー K(各 `[B, L, d]`、B はバッチ、L は系列長、d は特徴次元)に対して、FFT を使って全てのずれ τ に対する自己相関 R(τ) を一気に計算します。具体的には、Q を FFT、K の共役を FFT、両者を掛けて逆 FFT すると、すべての τ の相互相関が同時に得られます。FFT のおかげで O(L log L) で済みます。
2. 相関 R(τ) が高い上位 k 個の周期 τ₁, ..., τ_k を選びます。k はおよそ `c · log L`(c は定数)で設定し、選ぶ周期数も対数オーダーに抑えます。こうすることで、主要な周期だけを残し、ノイズ的な弱い周期を切り捨てます。
3. それぞれの周期 τ_i だけバリュー V を時間方向にロール(巡回シフト、`Roll(V, τ_i)`)し、相関の強さを softmax で正規化した重みで足し合わせます。

式で書くと、選ばれた周期について

```
出力 = Σ_i  softmax(R(τ_i)) · Roll(V, τ_i)
```

ここで `Roll(V, τ_i)` は「V を τ_i ステップ巡回的にずらした部分系列」、`R(τ_i)` は「その周期での似ている度合い」です。普通の注意が「点 i と点 j の重み」を作るのに対し、Auto-Correlation は「周期 τ ぶんの部分系列まるごと」を集約します。これが「点ではなく周期のかたまりを扱う」ということの正体です。多頭(multi-head)化や残差・正規化は通常の Transformer と同様に行います。

数値例で効率を実感しましょう。系列長 L=720 のとき、自己注意の全ペアは O(L²) ≈ 51万8千。一方 Auto-Correlation は FFT が O(L log L) ≈ 720 × 9.5 ≈ 6840 オーダーで、二桁以上軽くなります。長くするほど差は開きます。

### 全体の形状で整理

```
入力: 過去系列 X_past [B, L, d]
       ↓ 系列分解で初期トレンド・季節を作る
Encoder: 季節成分に Auto-Correlation → 残差 → 再分解(トレンドは捨て、季節を磨く)を N 層
Decoder: 季節成分に Auto-Correlation と cross 相関 → 再分解
          トレンド成分は各層で抽出して累積加算していく
出力: (Decoder の季節予測) + (累積したトレンド予測) = 最終予測 [B, L_pred, d]
```

設計の理由は明快です。季節成分は「周期のずれ」で集約するのが自然なので Auto-Correlation に通し、トレンド成分は注意で混ぜると逆に乱れるので、足し算で素直に積み上げる。役割をはっきり分けることで、長い先まで安定するわけです。デコーダの初期入力は、過去系列の後半を切り出してトレンドは平均値・季節はゼロで初期化し、各層でこれを少しずつ精緻化していく、という設計になっています。

## 手法の系譜と主要論文

時系列予測に Transformer を持ち込む流れは2019〜2020年前後から本格化しました。

最初の壁は自己注意の O(L²) の重さでした。Li らの LogSparse Transformer(Li et al., NeurIPS 2019, arXiv:1907.00235)が、注意を疎にして長系列に対応する初期の試みを示しました。続いて Zhou らの Informer(Zhou et al., "Informer: Beyond Efficient Transformer for Long Sequence Time-Series Forecasting", AAAI 2021 best paper, arXiv:2012.07436)が、クエリの中で本当に効いているものだけを選ぶ ProbSparse 注意で O(L log L) に落とし、デコーダを一度に全予測を出す生成的(generative)な形にして、長系列を Transformer に入れる道を開きました(→ [Informer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0334_informer))。

次に Wu らの Autoformer(NeurIPS 2021)が登場します。動機は「Informer は速くなったが、依然として点ベースの注意で周期構造を直接は捉えていない」という指摘でした。新規性は本ノートで述べた系列分解の内部組み込みと Auto-Correlation で、点ではなく周期単位の依存を扱える点が新しい。著者は清華大学の Haixu Wu, Jiehui Xu, Jianmin Wang, Mingsheng Long らです。

その後継が同グループの FEDformer(Zhou, Ma, Wen, Sun, Jin et al., "FEDformer: Frequency Enhanced Decomposed Transformer for Long-term Series Forecasting", ICML 2022, arXiv:2201.12740)です。Autoformer の系列分解の発想を引き継ぎつつ、注意を周波数領域(フーリエ・ウェーブレット)で行い、ランダムに選んだ少数の周波数モードだけ使うことで計算量を O(L) まで落としました。Autoformer の直接の発展形です。

ここで流れが一度ひっくり返ります。Zeng らの DLinear(Zeng et al., "Are Transformers Effective for Time Series Forecasting?", AAAI 2023, arXiv:2205.13504)は、時系列を分解してから「ただの一層の線形回帰」をかけるだけのモデルが、Informer や Autoformer や FEDformer を多くのベンチで上回ると示し、「複雑な Transformer は本当に必要か」という議論を巻き起こしました。皮肉なことに、この結果は Autoformer が広めた「分解」という前処理の有効性を逆に裏付けるものでもありました。

これに答えたのが Nie らの PatchTST(ICLR 2023)です。分解も自己相関も使わず、時系列をパッチに切ってチャネル独立に Transformer へ入れる素直な方式で、再び Transformer が線形モデルを上回ると示しました(→ [PatchTST](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0336_patchtst))。さらに Liu らの iTransformer(ICLR 2024, arXiv:2310.06625)は変数そのものをトークンにする方向を提案し、議論は続いています。Autoformer はこの一連の論争の出発点に位置し、「分解」という発想を後続のほぼすべての手法に残しました。

## 論文の実験結果(定量データ)

Autoformer の評価は、長期予測で標準的に使われる6種のベンチマークで行われました。ETT(電力トランスの温度、ETTh1/h2/m1/m2 の4変種)、Electricity(321顧客の電力消費)、Traffic(862本の道路センサーの占有率)、Weather(21の気象指標)、Exchange(8カ国の為替レート)、ILI(インフルエンザ様疾患の患者数)です。性質がバラバラ(周期が強いもの・弱いもの、変数が多いもの・少ないもの)で、汎用性を測るのに適しています。

評価指標は MSE(Mean Squared Error、平均二乗誤差)と MAE(Mean Absolute Error、平均絶対誤差)です。MSE は予測と正解の差を二乗して平均したもので、大きく外したときに強くペナルティがかかります。MAE は差の絶対値の平均で、外れ値に頑健です。どちらも小さいほど良く、その差が小さいほど予測が正解に近いことを意味します。長期予測ではこの値が「先720ステップまで予測したときに、平均してどれだけずれるか」を表すので、小さい差でも実用上の信頼性に直結します。なお入力は通常 z-score 正規化されているので、MSE/MAE は無次元の相対量として比べられます。

論文の中心的な主張は、6ベンチ・複数の予測長を平均して、当時の最良手法に対し相対的に約38%の MSE 削減を達成した、というものです。具体的には、予測長を 96, 192, 336, 720 と伸ばしたとき、Informer などの従来手法は先が長くなるにつれ MSE が急激に悪化したのに対し、Autoformer は劣化がゆるやかでした。例えば Electricity や Traffic のような周期性の強いデータで差が顕著で、報告された数値では予測長720の設定で従来 Transformer 系より MSE が数十パーセント低い、という結果が並んでいます。Exchange のような周期の弱いデータでも、移動平均によるトレンド分離が効いて健闘しています。一方 ILI(系列が短く周期が複雑)では、長期予測の相対的な改善幅は他のデータほど大きくありませんでした。

アブレーション(構成要素を抜いて効果を確かめる実験)も重要です。Auto-Correlation を普通の自己注意に置き換えると、特に長い予測長で性能が落ちました。これは「周期単位の集約」が長期予測の安定性に効いていることを示します。また系列分解ブロックを取り除くと、トレンドと季節が混ざったまま注意にかかり、やはり誤差が増えました。逆に言えば、この二つの部品がそれぞれ独立に効いていることが確認されたわけです。なお Auto-Correlation で選ぶ周期数 k(およそ log L)を変えると性能が変わり、少なすぎると主要周期を取りこぼし、多すぎるとノイズを拾うというトレードオフが見られました。さらに、入力長を伸ばすと(過去をより多く与えると)予測精度が一般に向上する傾向が報告され、これは後の PatchTST が「長い入力を活かす」方向を強調する伏線にもなっています。

## メリット・トレードオフ・限界

メリットは三つに整理できます。第一に、系列分解で周期と傾向を分離するため、長い予測でも周期を安定して捉えられること。第二に、Auto-Correlation により点ではなく周期単位の依存を直接扱えること。これは予測がのっぺりと平らになる「過剰平滑化」を避けるのに効きます。第三に、FFT による O(L log L) の計算量で、長系列にスケールすること。

トレードオフと限界もはっきりしています。まず Auto-Correlation は「明確な周期性がある」データで真価を発揮する設計です。周期が弱い、あるいは不規則なデータ(為替のようにランダムウォークに近いもの)では、自己相関のピークがはっきり立たず、利点が薄れます。実際、論文のベンチでも Exchange のような非周期的データでは他のデータほどの圧倒的な優位は出ていません。

次に、移動平均の窓幅(分解カーネルサイズ)がハイパーパラメータとして効きます。窓が小さすぎると周期がトレンド側に漏れ、大きすぎると季節成分にトレンドが混ざります。データごとの調整が必要で、固定窓は複数周期が重なるデータ(日次+週次+年次)を一つの窓で扱いきれません。FEDformer 以降、複数カーネルの分解や学習可能な分解が提案されたのはこの限界への対応です。

さらに研究上の未解決課題として、後発の DLinear が「分解+線形」というはるかに単純なモデルで対抗できると示したこと、PatchTST が分解も自己相関も使わずに上回ったことから、「Auto-Correlation という凝った機構が、その複雑さに見合う本質的な利得を持つのか」という問いが残っています。複数変数(多変量)間の相互作用を Autoformer が陽に扱っていない(変数ごとの相関を明示的にはモデル化しない)点も議論があり、チャネルの扱い方は iTransformer 以降も決着していません。

## 発展トピック・研究の最前線

Autoformer 以降、長期予測の研究は大きく三つの方向に分かれて進んでいます。

一つ目は周波数領域のさらなる活用です。FEDformer がフーリエ領域で注意を取る方向を示したあと、周波数成分を直接学習する軽量モデル(FreTS など)が多数提案され、計算量と精度の両立が追求されています。自己相関も突き詰めれば周波数の話なので、Autoformer はこの潮流の入口でもありました。

二つ目は「Transformer をやめる、あるいは大幅に簡素化する」方向です。DLinear の衝撃を受けて、TimesNet(Wu et al., ICLR 2023, arXiv:2210.02186)のように主要周期で系列を二次元化して二次元畳み込みで扱う手法や、TiDE・TSMixer のようなシンプルな MLP ベースの手法が再評価されました。Autoformer が広めた「分解」という前処理は、これらの非 Transformer 系にも広く取り込まれています。

三つ目は基盤モデル化です。大量の異種時系列で事前学習し、ゼロショットで未知のデータを予測する時系列の基盤モデル(foundation model、例: TimesFM, Moirai, Chronos, Lag-Llama など 2023〜2024)が登場し、個別タスクごとに学習する従来の枠組みを超えようとしています。この文脈で、Autoformer の Auto-Correlation のような明示的な周期機構が必要か、それとも巨大データと素朴なアーキテクチャで十分かは、現在進行形の論点です。

自動運転や時系列センサ処理の観点では、Autoformer の「滑らかな大局成分と細かい変動を分けて扱う」という発想自体は、軌道予測(滑らかな進路+周期的なふらつき)や状態推定の設計ヒントとして一般に応用が考えられます。周期構造を陽にモデル化する考え方は、時系列予測の枠を超えた汎用的な設計原理になっています。
