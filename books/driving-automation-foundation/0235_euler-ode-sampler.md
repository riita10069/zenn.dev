# Euler ODE サンプラ

## ひとことで言うと
「今いる場所から、次に進むべき向き(速度)をニューラルネットに聞いて、その向きに少しだけ進む」という操作を何回か繰り返し、ランダムなノイズからまともなデータ(画像・軌道・音声など)を作り出す、最も単純な生成手続きです。生成モデルの常微分方程式(ODE)を一次のオイラー法(Euler method)で数値的に解くことに相当します。実装は更新式1行ですが、生成モデルのサンプリングを理解する出発点であり、より高度な解法([Heun / 2次サンプラ (EDM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0239_heun-sampler) や [DPM-Solver](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0232_dpm-solver))を学ぶための基準点(ベースライン)になります。

## 直感的な理解
霧の中で目的地に向かって歩くところを想像してください。完全な道順は見えませんが、足元のコンパスが「今この場所では、こっちへ進むのが正しい」という向きだけは教えてくれます。あなたにできるのは、コンパスの示す向きに数歩進み、立ち止まって新しい向きを確認し、また数歩進む、の繰り返しです。これがオイラー法です。

歩幅が小さければ(ステップ数が多ければ)、こまめに向きを直すので道を大きく外しません。歩幅が大きいと、向きが途中で変わってもそれに気づかず進んでしまい、目的地からずれます。このずれが「離散化誤差」です。生成モデルでは「ネットに向きを聞く回数(ステップ数)」が計算コストそのものなので、できるだけ少ない歩数で正確に目的地に着きたい、という綱引きが常にあります。オイラー法はその最も素朴な解法であり、すべての高速サンプリング研究の比較対象になります。

## 基礎: 前提となる概念
生成モデルの多くは、答えを一発で出すのではなく、ランダムなノイズから出発して少しずつ整えて答えに近づける、という発想をとります。この「整える過程」は数学的には ODE で書けます。

```
dx/dt = v_θ(x, t, c)
```

- `x` は今の状態(整えている途中のデータ)。
- `t` は仮想的な「整え時間」で、0 から 1(あるいは逆向き)に進みます。片端が純ノイズ、もう片端が完成データです。
- `v_θ` はニューラルネットが出力する速度場(velocity field)。今の x をどの向きにどれだけ動かせば答えに近づくかを教えます。`θ`(シータ)は学習で決まる重みパラメータの記号です。
- `c` は条件(condition)。クラスラベル、テキスト、周囲環境の特徴など、生成をその状況に合わせるための入力です。条件がなければ無条件生成になります。

この ODE を手で解いて答えを書き下すことはできません。`v_θ` はニューラルネットなので式が複雑すぎるからです。そこで数値積分、つまりコンピュータで少しずつ近似的に解く方法が要ります。その最も単純なものがオイラー法です。なお、この ODE は拡散モデルの [Probability Flow ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0246_probability-flow-ode) や Flow Matching の確率パスと結びついており、「生成 = ODE を数値積分する」という見方は今日の生成モデルの共通言語になっています。

数値積分という言葉を補足します。微分方程式 `dx/dt = v` は「変化の速さ」を与えているだけで、各時刻での x の値そのものは与えていません。x の値を知るには、速さを時間方向に足し合わせる(積分する)必要があります。多くの場合この積分は数式では解けないので、時間を細かい区間に区切り、各区間で近似的に足していきます。これが数値積分で、区切り方と各区間の近似のしかたによって精度と計算量が変わります。

## 仕組みを詳しく
オイラー法の考え方は「今の速度がそのステップ幅のあいだ一定だと仮定して、その向きに小さなステップだけ進む」です。ステップ幅を `Δt` とすると、更新式は次の1行です。

```
x_next = x + Δt · v_θ(x, t, c)
```

これを始点から終点まで N 回繰り返します。等間隔なら `Δt = 1/N` です。具体的な流れを shape つきで追います。Flow Matching 系の軌道生成を例に、N=10、`Δt = 0.1` の場合を考えます。

1. ノイズから出発する。`x = randn(B, D)` で標準正規分布(平均0・分散1)から引きます。形は `[B, D]`。B はバッチ数、D は出力次元です(例えば将来64時刻×2信号=128次元なら D=128)。
2. 条件を一度だけ計算しておく。環境特徴や履歴などの条件 c は x や t に依存しないので、ループの外で一度だけ前処理し、毎ステップ使い回します(計算節約)。
3. ループを N=10 回回す。各ステップ step = 0,1,...,9 で
   - `t_val = step · Δt`(0.0, 0.1, ..., 0.9)
   - `v = v_θ(x, t_val, c)` で今の速度を求める
   - `x = x + Δt · v` で更新する
4. ループ後の x が完成データ。形は `[B, D]` のまま。

before/after で言うと、step 0 の x は完全なランダムノイズ(意味をなさない数値の羅列)です。ステップを重ねるごとに、条件 c が示す構造(道路や障害物、テキストの意味など)に沿って x が現実的なデータへ整っていきます。1ステップでネットを1回だけ呼ぶので、ネット呼び出し回数(NFE, Number of Function Evaluations)はステップ数そのものに一致します。これがオイラーの最も大きな実用上の利点です。

なぜオイラー法でうまくいくのでしょうか。鍵はパスの直線性です。Flow Matching では学習時に「ノイズ x_0 から正解 x_1 へ向かう直線パス」を教え込みます。途中点を `x_t = (1-t)·x_0 + t·x_1`、目標速度を `x_1 - x_0`(t によらない定数)として、ネットがこの速度を出すよう平均二乗誤差(MSE)で回帰します。真の速度がほぼ一定なら、オイラー法の「速度一定」という仮定がよく当たり、少ないステップでも誤差が小さくなります。これが「10 ステップ程度で動く」理由です。逆に拡散モデルの PF-ODE はパスが大きく曲がるため、純粋なオイラーでは多くのステップが要り、高次ソルバの恩恵が大きくなります。

誤差の理論も押さえます。オイラー法は1次の手法です。1ステップあたりの誤差(局所打ち切り誤差, local truncation error)は `Δt^2` のオーダー、N ステップ積み上げた全体の誤差(大域誤差, global error)は `Δt` のオーダー、つまりステップ数 N に反比例して減ります。ステップを倍にすれば誤差はおよそ半分、という素直な関係です。なぜ局所が `Δt^2` なのに大域が `Δt` なのかというと、誤差の出るステップが約 `1/Δt = N` 回あるので、`Δt^2 × N = Δt` となるからです。後述の2次ソルバ([Heun / 2次サンプラ (EDM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0239_heun-sampler))はこれを大域 `Δt^2` に改善します。

オイラーの拡張も理解しておくと深まります。拡散の SDE をそのまま離散化すると Euler-Maruyama 法という確率版のオイラーになり、毎ステップでガウスノイズを足し戻します。これは確率的サンプリングに相当し、決定論的オイラー(PF-ODE 版)とは「ノイズを足すかどうか」だけが違います。同じネットで両方を切り替えられるのが、ODE と SDE を統一的に扱う枠組みの強みです。

## 手法の系譜と主要論文
- オイラー法そのものは、18世紀の数学者 Leonhard Euler に由来する古典的な数値解法です(Euler, 1768 ごろの著作)。特定の生成モデル論文ではなく、数値解析の標準教科書(Hairer, Nørsett & Wanner, "Solving Ordinary Differential Equations I")に載る基礎です。最も単純な代わりに精度は低めで、より精密な Runge-Kutta 系や予測子修正子法の出発点という位置づけです。
- Song et al., "Score-Based Generative Modeling through Stochastic Differential Equations"(ICLR 2021, arXiv:2011.13456): 拡散モデルが SDE で書け、同じ周辺分布を持つ決定論的 ODE([Probability Flow ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0246_probability-flow-ode))が存在することを示しました。これにより「拡散の生成 = ODE を数値積分する」という見方が確立し、オイラーのような ODE ソルバが拡散サンプリングに使えるようになりました。
- Lipman et al., "Flow Matching for Generative Modeling"(ICLR 2023, arXiv:2210.02747): ノイズからデータへの直線的な確率パスを直接学習する Flow Matching を提案。パスが直線に近いほど ODE の曲がりが小さく、オイラーのような低次ソルバでも少ステップで高品質に積分できる、という設計の理論的裏付けを与えました。
- Liu et al., "Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow"(ICLR 2023, arXiv:2209.03003): 学習済みの流れを「再フロー化(reflow)」して経路をさらに直線化し、究極的には1ステップ生成に近づける手法を提案。直線性とオイラー法の相性をさらに突き詰めたものです。
- Albergo & Vanden-Eijnden, "Building Normalizing Flows with Stochastic Interpolants"(ICLR 2023, arXiv:2209.15571): ノイズとデータをつなぐ補間(interpolant)を一般的に設計する枠組みを提示し、Flow Matching とほぼ同時期に「決定論的に分布を流す ODE」を別角度から定式化しました。オイラーが効く条件(滑らかで曲がりの小さい速度場)を設計する自由度を与えます。
- Karras et al., "Elucidating the Design Space of Diffusion-Based Generative Models"(EDM, NeurIPS 2022, arXiv:2206.00364): 1次オイラーよりも2次のヘウン法のほうが同じネット評価回数で離散化誤差が小さいことを実験的に示し、オイラーを「基準となる最も単純な解法」と位置づけました([Heun / 2次サンプラ (EDM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0239_heun-sampler))。

## 論文の実験結果(定量データ)
評価には FID(Fréchet Inception Distance、生成画像と本物画像の特徴分布のずれ。小さいほど高品質)がよく使われます。FID は本物画像と生成画像をそれぞれ Inception ネットの特徴空間に写し、両者の平均と共分散の差を距離化したもので、CIFAR-10(32×32の小画像)では FID 2〜3 台が当時の最先端、10 を超えると目に見えて崩れる、という尺度感です。

EDM(Karras et al., NeurIPS 2022)のアブレーションが、オイラーの限界を最も明快に示しています。同じ学習済みモデル・同じノイズスケジュールでも、ソルバを1次オイラーから2次ヘウンに替えるだけで、同じ NFE での FID が顕著に改善します。逆に言えば、オイラーで同じ FID に到達するには、より多くの NFE が必要になります。報告では、オイラーで数十〜百ステップ要する品質に、ヘウンは20ステップ前後(NFEで35〜79程度)で到達します。具体例として、CIFAR-10 で NFE を絞った領域(数十回)では、オイラーの FID がヘウンより明確に高く(=悪く)出ることが、ステップ数を横軸にした曲線で示されています。

Flow Matching(Lipman et al., ICLR 2023)では、パスを直線(最適輸送に対応する条件付きパス)に取ると、拡散の曲がった分散保存パスに比べて、同じステップ数でのサンプル品質が高く、オイラーでも少ステップで良好という結果が示されています。ImageNet 64×64 などで、Flow Matching が同等または少ない関数評価回数で拡散ベースラインに匹敵する FID を達成しています。Rectified Flow(Liu et al., ICLR 2023)では、再フロー化を進めると1ステップ(NFE=1)でも実用的な FID(CIFAR-10 でおよそ数台、reflow と蒸留の組み合わせでさらに改善)に到達できることが示され、オイラーの単純さを保ったまま極端な高速化が可能なことを実証しました。これらはいずれも「オイラーの精度はパスの直線性に強く依存する」という共通のメッセージを定量的に裏づけています。

数値の意味を補足すると、NFE はネットを呼ぶ回数で、これがほぼそのまま生成1サンプルあたりのレイテンシと計算量に対応します。FID を一定に保ったまま NFE を半分にできれば、推論コストが半分になるということで、リアルタイム用途では決定的な差です。オイラーは「NFE = ステップ数」と最小なので、1ステップあたりが軽い分、パスが直線なら最もコスト効率が良くなります。

## メリット・トレードオフ・限界
メリットは、まず実装が極めて単純なことです。更新式が1行で、理解しやすくバグを入れにくい。次に、条件 c をループ外で一度だけ計算して使い回せるため、ステップを増やしてもネット呼び出し以外のオーバーヘッドが小さい。そして Flow Matching の直線パスとの相性がよく、10 ステップ程度で実用品質が出るため、自動運転やロボット制御のような低レイテンシ用途に向きます。NFE がステップ数に一致するので、計算予算を見積もりやすいのも実務上の利点です。

トレードオフと限界は次の通りです。第一に、1次の手法なので離散化誤差が大きめで、ステップを大きく刻むと結果が荒れます。第二に、同じ品質を得るのに2次のヘウン([Heun / 2次サンプラ (EDM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0239_heun-sampler))や構造特化の [DPM-Solver](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0232_dpm-solver) より多くのステップを要する場合があります。第三に、パスが大きく曲がる(拡散の PF-ODE のような)状況では、少ステップだと曲がりを追えず大きくずれます。第四に、決定論的 ODE なので、SDE サンプリングのような確率的多様性や誤差の自己補正効果は得られません。同じノイズ・同じ条件なら毎回同じ出力です(これは再現性の長所でもあり、多様性の短所でもあります)。

失敗例としてよく知られるのは、ステップ数を欲張って減らしすぎたケースです。直線パスでないモデルにオイラーを少ステップで適用すると、軌道がカクついたり、画像なら色ずれ・構造の崩れが出ます。とくに分散爆発型(VE)スケジュールのように高ノイズ領域で速度が急変する設定では、序盤のステップで大きく踏み外し、後段で取り返せないという破綻が起きやすいです。研究上の課題は「どれだけ直線化すればオイラーで何ステップまで減らせるか」を理論的に保証することで、Rectified Flow や蒸留はその方向の答えの一つです。

## 発展トピック・研究の最前線
オイラーの限界を超える方向は二つあります。一つは「解法を高度化する」方向で、2次のヘウン([Heun / 2次サンプラ (EDM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0239_heun-sampler))、半線形構造を使う DPM-Solver([DPM-Solver](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0232_dpm-solver))、適応ステップや高次 Runge-Kutta などです。もう一つは「パスを直線化してオイラーでも済むようにする」方向で、Rectified Flow の再フロー化、Consistency Models による1〜数ステップ蒸留、Consistency Trajectory Models などです。実用システムでは、レイテンシ予算とのトレードオフでこの二つを組み合わせます。低レイテンシが最優先の現場では、まず学習側で直線パスを採り、推論はオイラーの少ステップで回す、という設計が広く使われています。

さらに踏み込むと、オイラーの大域誤差 `O(Δt)` という遅い収束は、ステップを等間隔にとる前提に由来します。EDM が示したように、ノイズレベルに対して非線形にステップを配置する(高ノイズ側を細かく、低ノイズ側を粗く)だけで、同じ NFE での誤差が下がります。つまりオイラーという解法を変えなくても、ステップの配置を工夫するだけで品質が改善する余地があり、これはソルバ研究と相補的なテーマです。誤差解析にもとづく最適ステップ配置や、軌道の局所的な曲がり具合に応じてステップ幅を変える適応積分は、単純なオイラーの寿命を延ばす実用的な工夫として研究が続いています。
