# BEVDet4D型 特徴キュー連結

## ひとことで言うと

前フレームのBEV（真上から見た平面図、Bird's-Eye-View）の特徴を、自車の動いたぶんだけ座標をそろえてから、現在のBEV特徴と「チャネル方向に横へ並べて連結」し、畳み込み（Conv）で混ぜる時間融合手法です。注意機構（attention）を使わないぶん実装が単純で計算が軽い。基本は隣り合う2フレームだけを混ぜます。この手法が残した最大の教訓は「過去BEVを自車運動で整合してから連結することが決定的に重要」という点で、これは時間融合全般に通じる発見になりました。

## 直感的な理解

BEVFormer型（時間 self-attention）は表現力が高い一方、実装が複雑で計算も重め、というハードルがあります。BEVDet4D（Huang & Huang, 2022）は「もっと単純で軽い時間融合はできないか」という問題意識から生まれました。

着眼点はこうです。物体の速度を知るのに必要なのは「前と今で位置がどれだけずれたか」だけです。だとすれば、前のBEVと今のBEVを単純に隣に並べて畳み込みに見せれば、Convが「ずれ」のパターンを検出できるはず、という発想です。難しい注意機構を使わなくても、Convが空間的な差分を学べばよい。動画の2コマを重ねて差分を取ると動いた部分が浮かび上がるのと同じ理屈です。これが「キュー連結（feature queue concatenation）」の動機です。

ただし、ここに落とし穴があります。前のBEVと今のBEVは「別々の自車位置を中心にした座標」で描かれているので、そのまま重ねると静止物まで全部ずれて見えます。BEVDet4Dの核心的な貢献は、この「座標をそろえる（整合する）」工程の重要性を明確に示したことにあります。整合してから連結すれば、静止物は重なり、動く物だけがずれた、きれいな差分入力をConvに渡せます。

## 基礎: 前提となる概念

用語を噛み砕きます。

- チャネル（channel）: 特徴マップの「奥行き方向の層」のこと。256チャネルなら各マスに256個の数字が並びます。
- 連結（concatenation, concat）: 2つの特徴マップを奥行き方向に積み重ねること。256チャネルと256チャネルを連結すると512チャネルになります。情報を混ぜず、ただ横に並べる操作です。足し算（加算融合）と違い、元の2つの情報を両方そのまま保持できるのが利点です。
- キュー（queue）: 先入れ先出しの記憶列。直近の数フレームを溜めておく置き場です。
- Conv（畳み込み, convolution）: 近所のマスをまとめて見て新しい特徴を作る、画像処理の基本演算。連結した512チャネルにConvをかけると、Convの重みが「前と今の差分」を抽出するように学習されます。受容野（receptive field、一度に見られる範囲）の中に収まるずれしか捉えられない点が後述の限界につながります。
- BEVDet: BEVDet4Dの土台になった単一フレームのカメラBEV検出器（Huang et al., 2021, arXiv:2112.11790）。LSS（Lift-Splat-Shoot）流に画像をBEVへ持ち上げ、BEV上で検出します。BEVDet4Dはこれに時間次元（4D）を足した拡張です。
- 自車運動整合（ego-motion alignment）: 前BEVを現在の自車中心座標に座標変換すること。後述の通り、この手法の成否を分ける要です。

## 仕組みを詳しく

処理をステップで分解します。

1. 過去BEVのキュー保持: 直近フレームのBEV特徴をキューに溜めます。基本は前フレーム `B_{t-1}` の1枚。形状は例 `[B, 256, H, W]`（たとえば `[B, 256, 128, 128]`）。

2. 自車運動整合（この手法の最重要発見）: BEVDet4Dの論文は「前BEVを現在の自車中心に座標変換する整合が決定的に重要」と指摘しました。整合をしないと、自車自身の動きと物体の動きが混ざってしまい、速度推定が破綻します。整合は、前BEVを自車の並進・回転ぶんだけ平行移動・回転させてから重ねます（`grid_sample` で双線形補間）。論文はさらに、整合後に残る空間的な微小ずれを吸収する補助的な位置合わせ（追加の畳み込み層）を入れて学習を安定させました。なお BEVDet4D は、速度を「位置の差」として素直に学ばせるため、あえて時間差で正規化しない（位置オフセットのまま回帰する）設計上の工夫も行っています。

3. チャネル連結: 整合済みの `B'_{t-1}` と現在の `B_t` を、チャネル方向に連結します。

   ```
   B_t     : [B, 256, H, W]
   B'_{t-1}: [B, 256, H, W]   (整合済み)
             │ concat (チャネル方向)
             ▼
   連結     : [B, 512, H, W]
   ```

4. Conv 融合: 連結した512チャネルに畳み込みをかけ、256チャネルへ戻します。`[B, 512, H, W] → Conv → [B, 256, H, W]`。このConvが「前と今の差分（＝動き）」を表す特徴を抽出します。

5. 下流タスク: 融合後BEVで3D検出などを行います。BEVDet4Dは特に「速度推定誤差（mAVE）」を大きく下げたことで知られます。

数値例: 前車が0.5秒で1.0m前進。整合後の `B'_{t-1}` では前車はマス上で「数マス前」にいます。連結してConvにかけると、Convは「同じ物体が数マスぶん移動した」パターンを捉え、速度ベクトルを表す特徴を出力できます。整合をサボると、前車も自車の前進ぶんずれて見え、速度を誤ります。

設計の妙は「速度推定をConvに学ばせるなら、入力をできるだけ整った状態にしておく」という発想です。整合によって静止物は完全に重なり、動く物だけがずれる状態にしておけば、Convは「ずれ＝速度」というシンプルな対応を学べます。整合しないと、静止物も動く物も両方ずれて、どれが本当の物体運動か分からなくなります。これは「モデルに難しいことをさせる前に、前処理で問題を簡単にしておく」という機械学習の一般的な良い設計指針の好例です。

## 手法の系譜と主要論文

- BEVDet（Huang et al., 2021, arXiv:2112.11790）: 単一フレームのカメラBEV検出器。LSS流の画像→BEV持ち上げと、BEV上でのデータ拡張（BEV空間での回転・スケーリング・反転。画像空間とBEV空間の両方で拡張する点が新しかった）を導入し、カメラのみ3D検出の基礎を作りました。BEVDet4Dの土台です。

- BEVDet4D（Huang & Huang, 2022, arXiv:2203.17054）: 本手法の出典。BEVDetに時間次元を足した「4D」拡張。提案は「前フレームBEVを自車運動整合→チャネル連結→Convで融合」。動機は注意機構を避け、単純・軽量に時間手がかりを得ること。新規性は「整合の重要性の発見」と「速度推定の劇的改善」です。最大の教訓は「自車運動整合の有無で結果が激変する」点で、これは後続のあらゆる時間融合手法が前提とする常識になりました。

- BEVDepth（Li et al., AAAI 2023, arXiv:2206.10092）: BEVDet系の弱点だった「深度推定が不正確」を、LiDARから得た深度で明示的に監督して改善した手法。BEVDet4Dと組み合わせて使われ、深度監督が検出精度を大きく押し上げることを示しました。時間融合（4D）と深度監督（Depth）は直交する改善で、両方足すと相乗効果があります。

- BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）: 対比対象。時間 self-attention で融合する重め・高表現力の手法。BEVDet4Dは「もっと軽くしたい」側の代表です。詳細は [BEVFormer型 再帰的BEV融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0481_bevformer-recurrent-fusion)。

- SOLOFusion（Park et al., ICLR 2023, arXiv:2210.02443）: 「BEVDet4D型の隣接2フレームは短すぎる。長期の観測を使うべき」と主張し、長期コストボリュームで深度を改善しました。BEVDet4Dの「短期限定」という弱点への一つの回答です。

論点は「単純・軽量（BEVDet4D） vs 表現力・長期（BEVFormer / SOLOFusion / VideoBEV）」の対比に集約されます。

## 論文の実験結果（定量データ）

- BEVDet4D（2022）: nuScenes での速度推定誤差 mAVE（mean Absolute Velocity Error、検出物体の速度ベクトルの平均誤差 m/s、低いほど良い）を、単一フレームのBEVDetから劇的に改善しました。報告では、時間融合なしのBEVDetがmAVEおよそ0.9 m/s前後だったのに対し、BEVDet4DはmAVEをおよそ0.3〜0.4 m/s台へと半分以下に下げています。これは「1フレームでは速度をほぼ当てられない」状態から「実用的に速度が読める」状態への転換を意味します。総合指標のNDS（nuScenes Detection Score、検出精度・各種誤差を総合した0〜1の指標、高いほど良い）もmAVE改善に引っ張られて数ポイント向上しました。NDS の計算式では速度誤差が一つの重み付き構成要素なので、mAVE の半減はNDS全体を直接押し上げます。

  最も重要なアブレーションが「自車運動整合の有無」です。整合を外すと、速度推定の利得がほぼ消え、mAVEが整合ありに対して大きく劣化したと報告されています。これは「ただ過去BEVを連結するだけでは効かず、整合してこそ効く」ことの直接的な証拠で、時間融合全般に通じる教訓になりました。整合に使う運動情報の質（オドメトリ精度）が低いと、整合自体が誤差源になることも示唆されています。

- BEVDepth（AAAI 2023）: 深度のLiDAR監督を加えることで、位置誤差 mATE（mean Absolute Translation Error）を下げ、NDSをさらに押し上げました。時間融合（BEVDet4D）と深度監督を併用した構成が、当時のカメラのみ手法で高い水準を記録しています。

- SOLOFusion（ICLR 2023）: BEVDet4Dの隣接2フレーム融合に対し、長期（最大16フレーム程度）の観測を使うことで、nuScenes test でNDS 54%超を達成。フレーム数を2→8→16と増やすほど検出・速度推定が改善することをアブレーションで示し、BEVDet4Dの「短期限定」が性能上限になっていることを定量的に裏づけました。長期は深度精度（mATE）の改善に特に効きます。

定量的な核心は「整合つきの2フレーム連結だけでmAVEが半減する」「ただし整合を外すと利得が消える」「フレーム数を増やせばさらに伸びる余地がある」の3点です。BEVDet4Dは最小コストで時間融合の大きな利得が得られることを示し、後続の長期化研究（SOLOFusion / VideoBEV）の出発点になりました。

## メリット・トレードオフ・限界

メリット:
- 実装が単純。連結とConvだけで、注意機構が不要。既存の単一フレームBEV検出器（BEVDet等）に小さな追加で導入できる。
- 計算・メモリのオーバーヘッドが小さい。時間融合を低コストで試す最初の一歩に向く。車載のリアルタイム推論とも相性が良い。
- 速度推定の改善効果が実績として明確（BEVDet4DのmAVE半減）。
- 深度監督（BEVDepth）など他の改善と直交し、足し合わせやすい。

トレードオフと限界:
- 基本は隣接2フレームのみ。長期履歴へ自然には伸びない（長い隠れの追跡などには不向き）。キューを伸ばしても連結チャネル数が線形に増え、Convでまとめにくくなる。
- 自車運動整合が必須で、サボると速度推定が破綻する（この手法の最大の教訓）。整合はオドメトリ精度に依存する。
- 注意機構ほどの柔軟な対応づけはできず、速い・複雑なダイナミクスでは表現力が劣る。Convの受容野（一度に見られる範囲）を超えるずれは捉えにくく、高速で移動する物体や低フレームレートでは大きく外れたマスを対応づけられない。
- 連結＋Convは固定的な融合で、フレームごとに「どちらをどれだけ信用するか」を動的に切り替えにくい（attention はそれが得意）。
- 未解決の課題: 長期化（SOLOFusion / VideoBEV が部分的に回答）、整合誤差への頑健性、Convの局所性ゆえに大きな運動を扱う際の受容野設計（変形可能畳み込みや大カーネルの導入など）。

## 発展トピック・研究の最前線

- 長期化: SOLOFusion の長期コストボリュームや、再帰メモリによる多フレーム保持で、2フレーム限定の制約を超える流れ。
- 連結とattentionのハイブリッド: 軽量なConv連結で粗く融合し、重要部分だけattentionで精緻化する折衷案。計算予算に応じて使い分ける設計。
- 疎な表現への移行: 密なBEVを毎フレーム保持するコストを避け、物体クエリの伝播（StreamPETR / Sparse4D 系）で時間を扱う方向。BEVDet4Dの「密なBEVをまるごと連結する」路線への対案です。
- 整合の精緻化: オドメトリ誤差を学習で補正し、整合のずれに頑健な融合を狙う研究。整合残差を吸収する学習可能なアラインメント層の高度化。
- 深度監督との併用: BEVDepth 系の明示的深度監督と時間融合を組み合わせ、位置・速度の両方を底上げする構成が実務の標準になりつつあります。
