# VADv2 確率的プランニング

## ひとことで言うと
自車の進む道筋を1本だけ予測するのではなく、「ありえる道筋の見本帳(語彙)」をあらかじめ数千本用意し、その一つひとつに「これくらいありえる」という確率を割り当てる形で計画する手法です。運転の不確かさ(どっちに進むか決めきれない状況)を確率分布としてそのまま表現できるのが特徴で、軌道計画を「巨大な選択肢集合上の分類問題」として解くのが核心です。VADv2 は [VAD ベクトル化シーン+プランニング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0210_vad-vectorized-planning)(VAD)の後継で、VAD の弱点だった「単峰の出力(1本しか出せない)」を確率分布へ拡張しました。

## 直感的な理解
レストランで「今日のおすすめは何ですか」と聞かれて、店員が「これ1品です」と言い切るより、「人気順に並べた数千品のメニューで、上位はこれとこれです」と確率付きで答えてくれたほうが、あなたはアレルギーや予算で安全に選べます。VADv2 の発想はこれと同じです。軌道を1本に決め打ちせず、数千本のメニュー(軌道語彙)に確率を振り、あとから「衝突しない」「道をはみ出さない」という条件で安全な1本を選べるようにします。

なぜこれが必要か。現実の運転は本質的に不確実で多峰的です。交差点で「直進も右折もありえる」、前方の障害物を「左から避けるか右から避けるか」。正解は1つに決まりません。にもかかわらず軌道を1本だけ出すモデルを模倣学習(お手本の運転を真似る学習)で訓練すると、学習データの複数の正解が平均化され、どれでもない中途半端な軌道になります(モード平均化)。VADv2 はこれを「確率分布を学ぶ」ことで根本的に避けます。連続空間で1本を回帰させる代わりに、空間をあらかじめ数千個の代表に切り分けておき「どれか」を確率で答えさせる。これにより平均化の罠を構造的に逃れます。

## 基礎: 前提となる概念
模倣学習(imitation learning)とは、人間など専門家(エキスパート)のお手本の行動を真似るように学習する方式です。E2E 自動運転では、人間の運転ログの軌道を正解として、モデルがそれに近い軌道を出すよう学習します。素直にやると複数の正解が平均化されるのが弱点で、VADv2 はこの弱点に確率分布で対処します。

カテゴリ分布(categorical distribution)とは、有限個の選択肢それぞれに確率を割り当てた分布です。サイコロ(6面それぞれに確率)が典型例です。VADv2 は「軌道語彙の各本」を選択肢とするカテゴリ分布を学びます。出力は語彙の本数ぶんの実数を softmax(全要素を正にして合計1に正規化する関数)に通したもので、合計が1になる確率の列です。

離散化(discretization)とは、連続的な空間(無限にある軌道)を有限個の代表で刻むことです。利点は扱いが分類になって安定すること、欠点は代表に無いものは表現できない(離散化誤差)ことです。語彙が細かいほど誤差は小さくなりますが、選択肢が増えて計算とスコアリングが重くなります。

多峰性(multimodal)は前述の通りありえる答えが複数の山に分かれている状態です。連続空間で多峰分布を直接モデル化する(混合ガウスなど)のは山の数を決めにくく学習が不安定ですが、離散化してしまえばカテゴリ分布を学ぶだけで山の数を気にせず多峰性を表現できます。これが VADv2 が離散語彙を選ぶ最大の理由です。

ベクトル化シーン表現は [VAD ベクトル化シーン+プランニング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0210_vad-vectorized-planning) で説明した、世界を点と線で軽量に表す方式です。VADv2 はこれを VAD から受け継ぎます。

## 仕組みを詳しく
ステップ1: 軌道語彙(planning vocabulary)を作る。
学習データの大量の本物軌道を集め、代表パターンに圧縮した「軌道の見本帳」を作ります。VADv2 ではこの語彙が非常に大きく、数千本規模(報告で約4096本)です。1本の軌道は各時刻の (x, y) を並べたもので、語彙全体は `[N, T, 2]`(N は語彙数、T は時刻数)という巨大な集合です。語彙が大きいほど表現できる軌道が細かくなります。これは [アンカーベース軌道デコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0157_anchor-based-trajectory-decoder) のアンカーや [CoverNet 軌道分類](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0163_covernet-classification-planning) の軌道集合を、桁違いに大規模化したものと位置づけられます。語彙は学習を通して固定です(推論ごとに作り直しません)。

ステップ2: ベクトル化シーンを作る(VAD 譲り)。
カメラ画像 → BEV 特徴 → ベクトル化マップ(車線などのポリライン)+ ベクトル化エージェント(他車の状態と進路)を抽出します。世界を軽量な点と線で表す部分です。

ステップ3: 確率分布の学習(VADv2 の核心)。
シーン情報(地図・他車・自車状態・ナビ指示)を条件として、語彙の各軌道に確率を割り当てます。実装上は、各語彙軌道を埋め込みベクトルに符号化し、シーン特徴との注意機構で各語彙の「適合度スコア」を出し、softmax で確率 `[N]` に変換します。これが「どの軌道がどれくらいありえるか」のカテゴリ分布です。

学習では、お手本の軌道(エキスパート軌道)に近い語彙ほど高確率になるよう学習します。単純に「一番近い1本だけ正解」とするのではなく、エキスパート軌道との距離に応じて目標確率を作ります(近いものほど高い目標確率、距離をやわらかい分布に変換)。これにより似た軌道同士が滑らかに確率を分け合い、複数のありえる選択肢に自然に確率が分散します。これが多峰性の表現です。

```
目標確率の作り方(イメージ)
  エキスパート軌道との距離 d_i を計算
  目標確率 ∝ exp(-d_i / τ)   （τ は温度。小さいほど鋭く一点集中、大きいほどなだらか）
  → 近い語彙ほど高い目標確率。複数の近い語彙に確率が分散 → 多峰
  学習は予測分布 p とこの目標分布 q の交差エントロピー(KL)を最小化
```

ステップ4: 推論(実際に走るとき)。
分布が得られたら、最大確率の軌道(argmax)を選んで実行することも、安全フィルタを通したうえで選ぶこともできます。たとえば「確率上位の数本のうち、衝突しない・道からはみ出さないものを選ぶ」という後処理を確率分布の上で行えます。これは1本しか出さない手法では不可能な、確率的計画ならではの柔軟さです。VADv2 ではさらに、シーン理解(scene token)と語彙のスコアにルールベースの事前知識(交通規則など)を組み合わせるカスケード機構も導入しています。

```
before (VAD):  出力 [T, 2] の軌道1本 → 多峰性を平均化、選択の余地なし
after (VADv2): 出力 [4096] の確率分布 → ありえる道筋に確率分散、上位から安全な1本を選べる
```

なぜカテゴリ分布なのか。連続軌道空間で多峰分布を直接モデル化するのは難しい(混合ガウスは山の数を事前に決める必要があり、モード崩壊や数値不安定が起きやすい)からです。語彙で空間を離散化しておけば「どの語彙か」を学ぶだけでよく、山の数を気にせず多峰性を表現でき、学習も分類タスクとして安定します。代償は離散化誤差で、語彙を大きくして緩和します。これは連続生成で多峰性を扱う [DiffusionDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0171_diffusiondrive) とちょうど対照的な設計判断で、「離散の確率分布」対「連続の生成過程」という2つの潮流の片翼を代表します。

## 手法の系譜と主要論文
確率的・離散化計画の系譜を追います。

CoverNet (Phan-Minh et al., nuTonomy/Aptiv, CVPR 2020, arXiv:1911.10298)
軌道予測を「十分細かい軌道集合上の分類」に落とし込んだ先駆けです。回帰を捨てて分類だけで多峰予測を実現でき、集合サイズを上げると Miss Rate が下がることを示しました。VADv2 の大語彙分類はこの路線を E2E 計画へ持ち込んだものです。詳細は [CoverNet 軌道分類](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0163_covernet-classification-planning) を参照。

VAD (Jiang et al., Horizon Robotics/華中科技大学, ICCV 2023, arXiv:2303.12077)
ベクトル化シーン表現で軽量・高速な E2E 計画を確立しましたが、出力が単峰で多峰性が弱いのが課題でした。VADv2 はこの土台を受け継ぎつつ出力を確率分布に拡張します。

VADv2 (Chen, Jiang, Xu, et al., Horizon Robotics/華中科技大学, 2024, arXiv:2402.13243)
E2E 計画を「大規模軌道語彙上のカテゴリ分布(確率的計画)を学ぶ問題」として定式化し、ベクトル化シーンを条件にその分布を出力します。動機は運転の本質的不確実性・多峰性を決め打ち1本でなく確率分布で表すことで、前身 VAD の単峰の弱点を解消することです。語彙サイズを数千本へスケールさせたことが、アンカー手法との最大の量的差分です。

Hydra-MDP (Li et al., NVIDIA, 2024, arXiv:2406.06978)
VADv2 の「語彙上の分布学習」をさらに発展させ、複数の評価指標(衝突回避・進捗・快適性など)を教師として multi-head distillation(複数の教師ヘッドからの蒸留)で学ぶ手法です。各語彙軌道に対し「衝突しないか」「進捗するか」などのメトリックごとのスコアを予測させ、それらを統合して最終軌道を選びます。NAVSIM チャレンジ(2024)で上位を獲得し、語彙ベース計画が単なる模倣を超えて多目的最適化に拡張できることを示しました。これは「人間の真似」から「人間が達成したい複数の目的を直接最適化する」への重要な一歩です。

同時期の [DiffusionDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0171_diffusiondrive) は語彙という離散集合ではなく拡散モデル(ノイズから軌道を徐々に生成)で多峰性を扱う別アプローチで、両者は「離散の確率分布」対「連続の生成過程」の対比で読むと理解が深まります。

## 論文の実験結果(定量データ)
評価設定を説明します。VADv2 はクローズドループ寄りの評価を重視しました。クローズドループ評価とは、モデルの予測を実際に環境へ適用して状態を更新し続け、最後まで安全に走れたかを測る方式です(対してオープンループは人間ログとの一致だけを測る)。CARLA シミュレータの Town05 ベンチマークがその代表で、Driving Score(走行スコア、ルート完走率と違反ペナルティを掛け合わせた総合指標、高いほど良い)と Route Completion(完走率)が主要指標です。Driving Score は「どれだけ進めたか × どれだけ違反を避けたか」の積なので、進んでも違反すれば下がり、安全でも進まなければ上がらない、バランスの取れた指標です。

報告では、VADv2 は CARLA Town05 ベンチマークで当時のカメラ入力 E2E 手法として最高水準のクローズドループ性能を達成しました。Driving Score でベースライン(VAD を含む既存手法)を上回り、特に多峰性が問われる交差点・分岐で、決め打ち単峰の VAD より違反・スタックが減ったことが示されています。確率分布が得られることで安全制約と組み合わせた軌道選択が可能になり、これがクローズドループ性能の向上に直結しました。

アブレーションの要点。
- 語彙サイズの影響: 語彙を小さくする(離散化を粗くする)と性能が低下し、約4096本程度で良いバランスになることが示されました。これは「離散化誤差を下げるには大語彙が要るが、大きすぎると分類が難しくスコアリングも重くなる」という [CoverNet 軌道分類](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0163_covernet-classification-planning) と同じトレードオフの再確認です。
- 目標確率の作り方: 「最近傍1本だけ正解」とする hard label より、距離に応じたやわらかい目標分布(soft label)のほうが性能が良く、滑らかな確率分散が多峰性の獲得に効くことが示されました。τ(温度)が小さすぎると単峰に潰れ、大きすぎると分布がぼやけて argmax の信頼性が落ちる、という中庸が良いという結果です。
- 確率的計画 vs 単峰回帰: 同じバックボーンで出力だけ単峰回帰に置き換えると、多峰シーンで性能が落ち、確率分布定式化の寄与が確認されました。
- カスケード機構: シーン理解と確率分布を組み合わせる段を外すと、ルール違反が増えました。

指標の意味の補足: Driving Score がベースライン比で数ポイント上がることは、クローズドループでは「最後まで違反なく走り切れるエピソードが増える」ことを意味し、オープンループの L2 誤差が数センチ縮むのとは比較にならない実用的インパクトがあります。なぜなら一度の重大違反(衝突・逆走)でエピソードのスコアが大きく削られるためです。クローズドループでは誤差が時間とともに蓄積する(compounding error、ある時刻の小さなズレが次の入力分布をずらし、さらに大きなズレを呼ぶ)ので、オープンループの一致度では測れない真の運転能力が問われます。

## メリット・トレードオフ・限界
メリット
- 運転の不確実性・多峰性を確率分布としてそのまま表現でき、モード平均化を防ぐ。
- 分布が得られるので、上位候補から安全(衝突回避・道路逸脱回避)なものを選ぶ後処理が可能。ルールベース知識との融合もしやすい。
- カテゴリ分布なので山の数を気にせず学習が安定する(連続多峰モデルの不安定さを回避)。
- ベクトル化シーン(VAD 譲り)の軽量さと、確率的計画の柔軟さを両立。
- Hydra-MDP のように複数指標の蒸留へ自然に拡張でき、模倣を超えた多目的最適化に進める。

トレードオフと限界
- 語彙が離散なので語彙に無い軌道は出せない(離散化誤差。語彙を大きくして緩和するが計算・メモリが増える)。非定型な緊急回避は語彙外になりやすい。
- 数千本の語彙すべてにスコアを付けるため計算・メモリのコストがかかる。
- 目標確率の作り方(距離→確率変換の温度 τ など)に調整が要る。
- ベクトル化シーン抽出の誤りは確率分布にもそのまま伝播する。
- 語彙が学習データ分布に依存するため、別環境・別速度域では語彙の再構築が要ることがある。
- 未解決課題: 語彙をシーンに応じて適応的に変える、離散語彙と連続生成の長所統合(拡散との融合)、模倣を超えた安全性保証の組み込み。

## 発展トピック・研究の最前線
語彙ベース計画は2024年以降の NAVSIM 系チャレンジで主力の一つになり、Hydra-MDP のように複数指標の蒸留で多目的化する方向、[DiffusionDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0171_diffusiondrive) のように離散語彙(アンカー)を拡散の初期化に使い連続生成と融合する方向、の両面で発展しています。VADv2 の「離散分布で多峰性を安定に学ぶ」という核は、後者でもアンカー付き拡散の出発点として生き続けています。後続では、語彙を Vision-Language モデルと組み合わせて高レベルの意図(「あの車を追い越す」など言語的指示)で条件付けする研究も現れています。

評価面では CARLA クローズドループや NAVSIM 擬似クローズドループが標準化されつつあり、オープンループ L2 誤差だけで計画能力を語らない流れが定着しました。VADv2 がクローズドループ重視で評価された意義は、この潮流の先取りとして大きいです。一方、CARLA のシミュレーション分布と実世界の分布の差(sim-to-real gap)や、語彙設計の自動化はまだ研究途上です。
