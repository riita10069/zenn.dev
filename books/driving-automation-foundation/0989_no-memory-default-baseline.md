# NoMemory（時系列メモリ無しのベースライン）

## ひとことで言うと
自動運転や動画認識のモデルが「過去の映像や動きの履歴」をいっさい使わず、いまこの瞬間の1フレームだけを見て判断する、一番シンプルな構成です。新しく追加した「過去を覚える仕組み（時系列メモリ）」が本当に効くのかを公平に比べるための基準点（ベースライン）であり、同時に「履歴を使う機能を入れても既存の動作を壊さない」ための安全策にもなります。研究の文献では「single-frame baseline」「per-frame model」「monocular（単眼・単フレーム）detector」などと呼ばれます。

## 直感的な理解
あなたが運転中に、ほんの一瞬だけ目を開けて1枚の写真を撮り、すぐ目を閉じる、という状況を想像してください。その1枚から「前の車がどれくらいの速さで走っているか」「歩行者が自分に近づいているのか遠ざかっているのか」を当てられるでしょうか。答えはほぼ不可能です。速度とは「位置が時間とともにどう変わるか」であって、1枚の写真には時間の変化が写っていないからです。距離や向きは1枚からでも推定の余地がありますが、速度や加速度といった「時間微分」の量は、原理的に1枚では決まりません。

ところが現実のモデルは、まさにこの「1枚だけ見て判断する」ことから出発しました。理由は単純で、1枚を入力にするのが一番作りやすく、計算も軽く、学習データも揃えやすいからです。この素朴な構成が NoMemory です。NoMemory は、後から「複数フレームを見る仕組み」を足したときに「本当に良くなったのか」を測るための物差しになります。物差し無しに「良くなった」と言っても、それが履歴のおかげなのか、ただパラメータ（学習する重み）が増えてたまたま当たっただけなのか、区別できません。

専門用語の補足です。「ベースライン」とは「比較の出発点になる、一番素朴な手法」という意味です。新しい手法の良し悪しは、ベースラインからどれだけ改善したかという差分で測ります。差分が出てはじめて「この工夫が効いた」と科学的に主張できます。良いベースラインの条件は、強すぎず弱すぎず、誰が再現しても同じ結果になるほど単純で、比較対象と「履歴の有無以外は同条件」であることです。NoMemory はこの条件をほぼ完璧に満たします。

## 基礎: 前提となる概念
このトピックを理解するには、いくつかの前提を押さえておくと楽です。

第一に「フレーム」です。カメラは連続した静止画を高速に撮り続けます。その1枚1枚をフレームと呼びます。自動運転のカメラはおおむね毎秒10〜30枚（10〜30 fps、frames per second）を出します。連続するフレームを並べると動画になり、フレーム間の差から「動き」が読み取れます。逆に言えば、1枚を切り出した瞬間に「動き」の情報は消えます。

第二に「特徴ベクトル（feature vector）」です。生のカメラ画像はそのままでは扱いにくいので、ニューラルネットワーク（学習する関数の集まり）が画像を「数百次元の数字の並び」に圧縮します。これが特徴ベクトルで、「このフレームに何が写っているか」を機械が扱える形に要約したものです。たとえば896次元のベクトルとは、896個の実数で1フレームを表す、という意味です。

第三に「テンソルの形（tensor shape）」です。テンソルとは多次元の数値配列で、形は各次元の大きさを並べたものです。たとえば `[B, T, D]` という形は、`B` がバッチサイズ（同時に処理するサンプル数、たとえば4）、`T` が時間ステップ数（過去フレームの本数、たとえば64）、`D` が1ステップあたりの特徴次元（たとえば896）を表します。`[4, 64, 896]` なら「4サンプル、それぞれ過去64フレーム、各フレーム896次元」という意味です。

第四に「時系列メモリのインターフェース」です。履歴を扱うモジュールは多くの実装で共通の入出力契約を持ちます。典型的には、入力が `[B, T, D]`（時間方向に並んだ履歴）で、出力が `[B, D]`（時間をたたんだ1本の文脈ベクトル）です。`T` が消えている点が肝心で、下流のプランナ（軌道を出す部分）は時間軸のない要約だけを受け取ります。時系列メモリの仕事は、この「`[B, T, D]` から `[B, D]` への圧縮」を担うことだと言えます。NoMemory もこの契約を守る、最も単純な実装にすぎません。

## 仕組みを詳しく
NoMemory はこの圧縮を「最も単純なやり方」で行います。やることは2通りだけです。

1. 入力が3次元 `[B, T, D]`（履歴付き）なら、最新の時刻（最後のフレーム）だけを取り出す。Python のインデックス記法では `history[:, -1]` と書き、`-1` は「末尾」を意味します。結果は `[B, D]`。過去 T-1 枚は捨てます。
2. 入力が2次元 `[B, D]`（そもそも時間軸が無い）なら、そのまま素通し（パススルー）する。

数値例で追いましょう。`[4, 64, 896]`（バッチ4、過去64フレーム、特徴896次元）が入ってきたとき、最新の1フレームを取ると `[4, 896]` になります。64フレーム中、最新の1フレームだけが残り、残り63フレームは完全に無視されます。ASCII 図で書くと次の通りです。

```
入力 history:  [ f0  f1  f2  ...  f62  f63 ]   形 [B, 64, 896]
                                       └── これだけ採用
出力 context:  [ f63 ]                          形 [B, 896]
```

NoMemory の決定的な特徴は、学習する重み（パラメータ）が一切ゼロだということです。掛け算も足し算もせず、ただ末尾を選ぶだけなので、計算も実質ゼロです。多くのベンチマーク実装で、NoMemory はパラメータ数0、1回あたりの遅延が数十マイクロ秒（0.0数ミリ秒）と報告されます。比較対象となる学習型の時系列メモリ（GRU 圧縮型やアテンション型）はパラメータ数が数百万から数千万に達するので、NoMemory がいかに軽量かが分かります。これにより NoMemory は「計算コストの絶対的な下限」という意味でも基準になります。どんな時系列手法も、遅延とパラメータでこれより下に行くことはできません。

なぜ「末尾を選ぶだけ」で済ませるのか。それは NoMemory が「履歴を使わないとはどういうことか」を厳密に定義するためです。もし平均や重み付けをすると、それ自体が一種の時系列処理になってしまい、純粋な単フレーム比較になりません。たとえば過去64フレームを単純平均すれば、動いている物体はブレて写り、「動きの大きさ」がわずかに漏れ込みます。あくまで「最新の1枚だけ」を採ることで、履歴ゼロという条件を曖昧さなく実現します。この厳密さこそが、ベースラインとしての価値を担保します。

実装上もう一つ重要なのは「決定性」です。NoMemory は乱数も学習も使わないため、同じ入力には必ず同じ出力を返します。比較実験で「履歴あり」と「履歴なし」を切り替えるとき、片方が確率的に揺れると差の解釈が難しくなります。NoMemory が完全に決定的であることで、観測された差はすべて履歴融合の効果だと帰属できます。

## 手法の系譜と主要論文
NoMemory そのものは新しい発明ではなく、時系列以前のモデルが当たり前に取っていた構成です。歴史的には、画像1枚を入力とする物体検出やセグメンテーションのモデル（2D 認識）がそのまま自動運転の知覚に流用され、「単フレーム認識」が標準でした。その限界が明らかになるにつれて、履歴を足す研究が次々登場します。NoMemory はその「足す前の状態」を指す言葉として、比較のために残り続けています。

- FCOS3D (Wang et al., ICCVW 2021, arXiv:2104.10956): 1枚のカメラ画像から3次元の物体ボックスを直接回帰する代表的な単フレーム手法です。距離・大きさ・向きはそれなりに当てられる一方、速度推定は本質的に苦手で、単フレーム検出器の到達点と限界を同時に示しました。多くの時系列手法がこの種の単フレーム検出器を出発点（ベースライン）として引用します。

- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270): 過去フレームの BEV（bird's-eye-view、鳥瞰図。カメラ画像を真上から見た2次元の地図状の特徴に変換したもの）特徴をキューに保持し、現在フレームが過去を temporal self-attention で参照する手法を提案しました。論文の ablation（要素を抜き差しして効果を測る実験）で、temporal な融合を外した「単フレーム版」、つまり事実上の NoMemory と比べ、速度推定と遮蔽（手前の物体に隠れて一時的に見えなくなること）への頑健性が向上したと報告しています。NoMemory が原理的に不利な点を、対照実験で浮き彫りにした例です。

- BEVDet4D (Huang & Huang, 2022, arXiv:2203.17054): 直前フレームの特徴を空間的に位置合わせして連結するだけで速度推定誤差が劇的に下がることを示しました。これは「単フレームでは速度がほぼ推定できない」ことの定量的な裏付けであり、NoMemory の弱点を最も直接的に示した研究の一つです。

- SOLOFusion (Park et al., ICLR 2023, arXiv:2210.02443): 短期高解像度と長期低解像度の履歴を組み合わせ、履歴を長くするほど精度が伸びることを示しました。裏を返せば、単フレーム（NoMemory）からの伸びしろが非常に大きいことを意味します。

- StreamPETR (Wang et al., ICCV 2023, arXiv:2303.11926): 過去の物体クエリを次フレームへ伝播させる物体中心の時系列融合を提案し、長期履歴を安価に扱えることを示しました。ここでも単フレームベースラインが比較対象として使われています。

これらに共通する論旨は「単フレーム（NoMemory）は軽く単純だが、速度・加速・遮蔽の扱いで原理的に不利。だから履歴融合を足す価値がある」というものです。逆に言えば、履歴融合を足したのに NoMemory に勝てないなら、その融合手法は設計が悪いと判断できます。NoMemory はこの判定の物差しになります。

## 論文の実験結果（定量データ）
時系列融合の論文では、NoMemory に相当する単フレームベースラインとの比較が必ず示されます。指標の意味から説明します。

3D 物体検出ではよく nuScenes データセット（大規模な実走行データ。6カメラ＋LiDAR＋レーダで都市部を走行収録し、約140万枚のカメラ画像に3次元ボックスを付与）が使われ、総合指標 NDS（nuScenes Detection Score）と平均精度 mAP（mean Average Precision）が報告されます。NDS は検出の正確さに加え、位置・大きさ・向き・速度・属性の誤差をまとめた総合点で、0〜1（または0〜100%）で高いほど良い指標です。速度誤差は mAVE（mean Average Velocity Error、m/s。低いほど良い）で測ります。

BEVDet4D は、単フレームの BEVDet と比べて mAVE をおよそ半分以下に下げたと報告されています（報告によれば 0.9 m/s 級から 0.4 m/s 級へ）。これは「1フレームでは速度をほぼ当てられないが、2フレームを位置合わせして使うだけで速度誤差が半減する」という、NoMemory の限界を端的に示す数字です。mAVE が 0.9 から 0.4 に下がるとは、たとえば時速 10km（約 2.8 m/s）で動く相手の速度推定が、半分以上ずれていた状態から、まずまず信頼できる状態へ変わる、という意味です。NDS も単フレーム比でおよそ数ポイント（数%）向上します。BEVFormer は nuScenes test で NDS 0.569、mAP 0.481 を報告し、temporal self-attention を外したアブレーションでは速度・遮蔽関連の指標が悪化することを示しました。SOLOFusion は履歴長を伸ばすほど NDS が上がり続け、長期履歴版で当時の最高水準（NDS 0.5 台後半〜0.6 台）に達したと報告しています。

なぜこの差が重要かというと、自動運転では「前の車が止まっているか動き出したか」の判断が安全性に直結し、それは速度推定の正確さそのものだからです。mAVE が半減するということは、衝突回避や追従制御の信頼性が大きく改善することを意味します。NoMemory はこの改善幅をゼロ点として可視化する役割を果たします。アブレーションの読み方としては、「ある手法から時系列要素だけを抜いて NoMemory 相当にすると mAVE がどれだけ悪化するか」を見れば、その手法が履歴から得ている価値の大きさが分かります。

## メリット・トレードオフ・限界
メリット

- パラメータ0・遅延ほぼゼロで、計算コストの絶対的な下限を与える。どんな時系列手法も「これより軽くはならない」基準点になる。
- 実装が数行で、バグの入りようがなく、完全に決定的なので、比較の信頼できる基準になる。
- デフォルトに据えれば、履歴を使う機能を追加しても既存の学習・推論パイプラインの振る舞いを変えない安全策になる（後方互換）。履歴が無い状況（システム起動直後など）でも必ず動く。

トレードオフ・限界

- 速度・加速度・進行方向の変化など「時間を見ないと分からない情報」を原理的に捉えられない。
- 遮蔽（一瞬隠れた物体）を過去フレームで補完できない。物体が1フレームだけ隠れると、その瞬間に存在ごと見失う。
- 判断が1フレームの揺らぎ（ちらつき、検出のフリッカ）に左右されやすく、フレーム間で予測がブレやすい。閉ループ制御では小刻みなハンドルやブレーキの揺れにつながる。
- 用意した履歴の情報をまるごと捨てるため、計算が軽い代わりに上限性能が低い。

研究上の未解決課題として、「ベンチマークで NoMemory に対する遅延・パラメータの優劣は容易に測れるが、運転の良し悪し（軌道誤差や閉ループの安全性）への寄与は、学習済みモデルとオープンループ／クローズドループ評価が揃わないと測れない」という問題があります。オープンループ評価とは録画データに対して予測を当てる評価、クローズドループ評価とはシミュレータの中で実際に運転させて結果を見る評価です。NoMemory が「運転性能の」ベースラインとして真に機能するには、同条件で学習させた時系列手法との end-to-end 比較が前提になります。

## 発展トピック・研究の最前線
単フレームベースラインを起点に、時系列融合は大きく二系統へ発展しました。一つは BEV 空間で過去を位置合わせして融合する流れ（BEVFormer、BEVDet4D、SOLOFusion）、もう一つは物体やトークンを単位に過去を伝播させる流れ（StreamPETR、Sparse4D 系）です。近年はさらに、エンドツーエンド運転（知覚から計画まで一気通貫で学習する UniAD、VAD など）の文脈で、時系列メモリが軌道予測の精度に直接効くことが示されつつあります。NoMemory はこれらすべての研究で「履歴を足す前」の参照点として登場し続けます。

もう一つの最前線は「どれだけ過去を見るべきか」の最適化です。長い履歴ほど精度は上がるが計算も増えるため、固定長キューや効率的なメモリ伝播が研究されています。NoMemory（履歴長ゼロ）は、この長さ–精度トレードオフ曲線の左端の点に当たります。曲線の左端を厳密に固定しておくことで、「履歴を1フレーム足すごとに何が得られるか」という限界効用を、研究者は定量的に議論できるようになります。
