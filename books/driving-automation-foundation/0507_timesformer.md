# TimeSformer

## ひとことで言うと
動画を、畳み込みを一切使わず Transformer の注意機構(attention)だけで理解するモデルです。鍵は「空間の注意」と「時間の注意」を別々の手順に分ける(divided attention、分離注意)こと。全部を一度に注意で結ぶより計算をずっと軽くしつつ、動画の時空間の関係を捉えます。動画認識を畳み込みから注意機構へ移行させた初期の代表例のひとつです。

## 直感的な理解
まず Transformer と注意機構を平易に説明します。Transformer は文章や画像を「小さな断片(トークン)の集まり」として扱い、注意機構で「どの断片がどの断片とどれくらい関係するか」を全ペアについて計算する仕組みです。たとえば文章なら、「それ」という単語がどの名詞を指すかを、文中の全単語との関係を測って決めます。

画像版の ViT(Vision Transformer)は、画像を例えば16×16画素のパッチに切り、各パッチを1トークンとして、すべてのパッチどうしの関係を見ます。畳み込みのように「近所だけ」を見るのではなく、最初から画像全体を見渡せるのが強みです。離れた2か所の関係(画面の左端と右端の物体の対応など)を1段で捉えられます。

これを動画にそのまま広げると問題が起きます。動画は「パッチ × フレーム数」ぶんのトークンになります。1フレームが14×14=196パッチ、8フレームあれば 196×8=1568 トークン。注意機構は全トークン対の関係を計算するので、計算量はトークン数の2乗、つまり 1568²≒246万ペアにもなります。フレームを増やすほど爆発的に重くなり、長い動画は扱えません。

TimeSformer の問いはタイトル通り "Is Space-Time Attention All You Need?"(動画理解に時空間の注意だけで十分か?)です。畳み込みを一切使わず、しかも全トークンを一度に結ぶ重い注意を避けて、どう効率よく時空間を捉えるか。その答えが「空間と時間に注意を分ける」divided attention でした。

## 基礎: 前提となる概念
用語を噛み砕きます。

トークン(token)とは、Transformer が処理する最小単位のベクトルです。画像なら1パッチ、文章なら1単語(に近いもの)が1トークンです。各トークンは固定次元(例えば768次元)のベクトルで表されます。

自己注意(self-attention)とは、各トークンが他の全トークンとの関連度(注意の重み)を計算し、関連の強いトークンの情報を重み付きで集めて自分を更新する操作です。関連度は query と key というベクトルの内積で測ります。各トークンから query・key・value という3種のベクトルを作り、query と key の内積で重みを決め、その重みで value を足し合わせて新しいトークンを得ます。計算量はトークン数 M に対して M² に比例します(全ペアを見るため)。これが大量トークンで重くなる根本原因です。

位置埋め込み(position embedding)とは、各トークンに「どこの・いつの断片か」を表す学習可能なベクトルを足すことです。注意機構自体は順序や位置を区別しない(トークンの並べ替えに対して結果が変わらない)ので、位置情報を明示的に注入します。動画では「どのフレームの、どの空間位置か」を埋め込みます。

クラストークン(class token)とは、分類のために先頭に置く特別なトークンで、全トークンの情報を集約し、最終的にこれを分類器に通してラベルを予測します。

マルチヘッド注意(multi-head attention)とは、注意を複数の独立した「ヘッド」で並行して計算し、それぞれが異なる関係(ある頭は色、別の頭は動きなど)を捉える仕組みです。

## 仕組みを詳しく
入力動画を ViT 流にパッチに切ります。各フレームを16×16のパッチに分け、線形変換でトークン(例えば768次元のベクトル)にします。各トークンに、空間位置と時間位置を表す位置埋め込みを足します。

TimeSformer はいくつかの注意の組み方を比較し、divided space-time attention(分離時空間注意)を推奨しました。1つの Transformer ブロックの中で、注意を2段に分けます。

1. 時間注意(temporal attention)。あるパッチ位置(例えばフレーム内の座標 (3,5))について、全フレームにわたる同じ位置のトークンだけを集めて、その間で注意を計算します。「画面のこの場所が、時間とともにどう変わったか」を見る段です。8フレームなら、この注意は8トークン間だけの計算で済みます。

2. 空間注意(spatial attention)。次に、同じフレーム内の全パッチの間で注意を計算します。「このフレームの中で、どの場所どうしが関係するか」を見る段です。1フレーム196パッチなら、この注意は196トークン間の計算です。

なぜ分けると軽いのか、数値で比較します。一括(joint)注意なら全1568トークン対で 1568²≒246万ペア。分離だと、時間注意は位置ごとに 8² のペアを196位置ぶん(196×64≒1.3万)、空間注意はフレームごとに 196² を8フレームぶん(8×38416≒30.7万)。合計でも一括の1/7以下に収まります。フレーム数 F とパッチ数 N に対し、一括が (N×F)² なのに対し、分離は N²×F + F²×N で済む、というのが効率の正体です。F や N が大きいほど差が開きます。

```
joint:    (N·F)²            = (196·8)² ≒ 246万   ← 全トークン総当たり
divided:  N²·F + F²·N        = 196²·8 + 8²·196 ≒ 32万 ← 空間内 + 時間軸ごと
```

ステップでまとめると、各 Transformer ブロックは「時間注意 → 空間注意 → 全結合層(MLP)」の順に処理し、これを多数のブロックで重ねます。各注意の後には残差接続と層正規化(LayerNorm)が入り、勾配が流れやすくなっています。最終的にクラストークンの出力を分類器に通して動作ラベルを予測します。畳み込みは一切登場しません。

tensor shape のイメージ。入力 [バッチ, 3, 8フレーム, 224, 224] → パッチ化 → トークン列 [バッチ, 1+1568, 768](先頭の1はクラストークン)→ 各ブロックで時間注意と空間注意を交互に適用 → 出力 [バッチ, クラス数]。

論文は他にも複数の注意設計を比較しました。「空間だけ注意(各フレーム独立、時間を見ない)」「一括時空間注意(全トークン総当たり)」「軸ごと(axial)注意(高さ・幅・時間を別々に注意)」「sparse local + global(局所窓と疎な大域を組み合わせ)」などです。結果として divided が精度と効率のバランスで最良だと結論づけました。divided は「空間と時間は別の軸であり、それぞれ独立に注意を割り当てた方が効率的かつ十分に表現力がある」という仮説の実証です。重要なのは、時間注意を入れることで、各空間位置がフレームをまたいで情報をやり取りし、運動を捉えられる点です。空間だけ注意ではこの情報のやり取りが起きず、動画を「フレームの寄せ集め」としてしか見られません。

## 手法の系譜と主要論文
Dosovitskiy et al., "An Image is Worth 16x16 Words (ViT)", ICLR 2021。TimeSformer の土台。画像をパッチに切って Transformer で処理する発想そのもので、TimeSformer は ViT の事前学習重みを初期値に再利用します。ViT 自体が「畳み込みなしでも大規模データがあれば画像認識で CNN を超える」ことを示し、その流れを動画へ持ち込んだのが TimeSformer です。

Bertasius, Wang, Torresani, "Is Space-Time Attention All You Need for Video Understanding? (TimeSformer)", ICML 2021。本トピックの原典。提案は畳み込みなしの動画 Transformer と、複数の注意設計の比較に基づく divided space-time attention。動機は一括時空間注意がトークン数の2乗で重すぎることの解消です。

Arnab et al., "ViViT: A Video Vision Transformer", ICCV 2021。TimeSformer とほぼ同時期。空間用と時間用の Transformer を段階的に重ねる「分解(factorised)」構成を複数提案し、divided 注意と近い思想を別の形で実現しました。ViViT は factorised encoder(空間 Transformer の出力を時間 Transformer に渡す)など4つのモデルを比較し、時空間の分解方法の設計空間を体系的に探りました。両者は「時空間注意をどう分解すれば効率と精度が両立するか」を競った代表例です。

Liu et al., "Video Swin Transformer", CVPR 2022(詳細は [Video Swin Transformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0510_video-swin-transformer))。全トークンに注意を計算するのをやめ、局所的な3D 窓の中だけで注意を計算する方式。TimeSformer の分離注意とは別方向(窓の局所化)で重さを解いた後継です。

Tong et al., "VideoMAE", NeurIPS 2022。動画 Transformer を自己教師あり(マスク再構成。動画の大部分のパッチを隠して残りから復元させる)で事前学習する手法。TimeSformer 系がデータ飢餓に弱い点を、ラベルなし大量動画での事前学習で緩和し、その後の標準になりました。VideoMAE は動画の時間的冗長性ゆえ 90% という極端に高いマスク率が有効だと示しました。

## 論文の実験結果(定量データ)
指標を説明します。Kinetics-400/600 の top-1 精度は分類正解率です。Something-Something V2(SSv2)は「何かを左に動かす」「何かを倒す」など、見た目より時間的順序が決定的なデータセットで、時間モデリングの良し悪しがはっきり出ます。SSv2 は同じ物体・背景でも動きの向きや順序でラベルが変わるため、「背景や物体名で当てる近道」が効かず、純粋な時間理解が問われます。

Kinetics-400 で、TimeSformer-L(高解像度・多フレーム設定、96フレーム入力)は top-1 で約 80.7% を報告しました。3D CNN の代表である SlowFast(約 79%台)に匹敵し、注意ベースで畳み込みを使わずにここに達したことが注目されました。標準設定の TimeSformer でも約 78% で、I3D 系を上回ります。

注意設計のアブレーションが論文の核です。Kinetics-400 では、空間だけの注意(時間を見ない)が約 76.9%、一括時空間注意が約 77.4% に対し、divided space-time attention が最高でした。Kinetics は背景や物体で当てやすいため差は小さめですが、時間が決定的な SSv2 では差が劇的に開きます。SSv2 で空間だけ注意は約 36% 程度しか出ないのに対し、divided 注意は約 59-62% に達し、20ポイント以上の差がつきました。これは「時間注意を分離して明示的に入れることが、時間順序の理解に決定的」という強い証拠です。一括注意は理論上は最も表現力が高いはずですが、同じ計算予算では多フレームを扱えず、メモリ制約からフレーム数を減らさざるを得ないため、実効的には divided に劣りました。つまり「理論上の表現力」より「同じメモリでどれだけ長いクリップを見られるか」が実性能を決めた、という実務的に重要な教訓です。

効率の検証も示されました。フレーム数を8→32→96 と増やすと、一括注意は早々にメモリで破綻する一方、divided 注意は現実的な計算量で長いクリップを扱え、フレーム増加に伴って精度が伸び続けました。高解像度・長クリップで精度が伸びるスケーリングの良さは、Transformer 系の大きな利点です。事前学習の重要性も明確で、ImageNet 事前学習なしに動画だけで学ぶと大きく精度が落ち(数十ポイント単位で低下)、データ量が性能を強く左右することが確認されました。学習・推論コストの面でも、TimeSformer は SlowFast 系より学習が速いケースが報告され、注意ベースの効率の良さが実証されています。

## メリット・トレードオフ・限界
メリット。畳み込みなしで動画の時空間関係を end-to-end に学習できます。分離注意により一括注意の計算爆発(トークン数の2乗)を大幅に回避し、長めのクリップを現実的な計算量で扱えます。ViT の画像事前学習を初期値に再利用でき、解像度やフレーム数を増やすほど精度が伸びるスケーリングの良さがあります。SSv2 の結果が示すように、時間順序が本質のタスクで強いのが特長です。注意機構は受容野が初段から全体に及ぶため、離れたフレーム・離れた空間位置の関係を浅い層で捉えられる点も利点です。

トレードオフと限界。大量データと事前学習が前提で、小データでは過学習しやすく、3D CNN より初期の収束が遅いことがあります。パッチ単位のため、細かい局所運動の解像度は畳み込みに劣る場合があります。分離注意は近似であり、空間と時間が強く絡む現象(同じ瞬間に空間的にも時間的にも複雑に相関するイベント)を一括注意ほど厳密には捉えられない可能性があります。注意機構自体はメモリ消費が大きく、フレーム数を大きく増やすと依然として重くなります。divided 注意も時間注意がフレーム数 F の2乗を含むため、数百フレームの長尺ではこの項が再び効いてきます。研究上の課題として、時間と空間の注意の順序や重み付けの最適化、長尺動画(数分)への拡張、そして事前学習データ依存をどう減らすか(自己教師あり化)が継続的なテーマです。

## 発展トピック・研究の最前線
分離注意の系譜は ViViT の factorised encoder/attention と並んで、動画 Transformer の設計空間を切り開きました。その後、局所窓注意の Video Swin が効率と局所性で対抗し、自己教師あり事前学習の VideoMAE がデータ飢餓を緩和し、Multiscale Vision Transformers(MViT/MViTv2)が階層的に解像度を変える設計で精度と効率を両立しました。最前線では、画像と動画を統一的に扱うマルチモーダル基盤モデル、テキストと動画を結ぶ video-language モデル(動画に自然言語で問い合わせる)、そして長尺動画の効率的な注意(線形注意・状態空間モデル Mamba 系など、注意のコストを2乗から線形に落とす試み)が研究されています。

自動運転など実時間が求められる領域では、空間理解と時間理解をモジュールとして分離する設計思想が、TimeSformer の divided attention が示した「空間と時間を分けると効率的」という知見と同じ方向にあります。複数フレームを1つのバックボーンに統合する際、分離注意は計算爆発を避ける有力な指針になります。とりわけ、空間バックボーンを各フレームに適用し、時間方向だけを軽い注意で結ぶ構成は、リアルタイム性と表現力のバランスを取りやすく、実応用で好まれます。
