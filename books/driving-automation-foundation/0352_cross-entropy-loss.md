# クロスエントロピー損失

## ひとことで言うと
クロスエントロピー損失とは、「モデルが予測した確率分布」が「正解」とどれだけずれているかを測る、分類問題でいちばん基本になる物差し(損失関数)です。正解クラスに高い確率を出せば損失は小さく、低い確率しか出さないと損失が大きくなります。

ここで「損失関数」とは、モデルの出力がどれだけ間違っているかを1つの数値で表したもので、学習はこの数値を小さくする方向にモデルの重み(パラメータ)を少しずつ調整していく作業のことです。

## 直感的な理解
クイズで「自信のある答えに大きく賭ける」状況を想像してください。正解に大きく賭けて当たれば得点、外せば大きく失点する。クロスエントロピーはこの賭けの罰金表です。モデルが正解クラスに 99% の確率を割り当てて当たればほぼ罰なし、逆に正解クラスにわずか 1% しか割り当てなかったのに実はそれが正解だった、というときは巨額の罰金が科されます。

なぜ「賭け方」で罰を変えるのか。それは、モデルに単に「正解クラスを最も高くする」だけでなく「確信度まで含めて誠実であれ」と要求したいからです。自信過剰な間違いを重く罰することで、モデルは確率を正直に出すよう学びます。この性質は厳密には proper scoring rule(真の確率を出すのが最も得になる採点規則)と呼ばれ、確率を歪めずに学ばせる理論的根拠になっています。

## 基礎: 前提となる概念

**分類 (classification)**: 「この画像は犬か猫か鳥か」のように、入力を決まった種類(クラス)のどれかに振り分ける問題。自動運転でも「注意すべき主因は歩行者か信号か障害物か」「このピクセルは路面か車線か歩道か(セマンティックセグメンテーション)」といった判断は分類として扱えます。

**logits(ロジット)**: モデルの最後の層が出す、softmax 前の生の数値の並び。各クラスへの「確信度の生スコア」で、正にも負にもなり得ます。

**softmax(ソフトマックス)**: logits を「合計が1になる確率」に直す変換。各数値を指数関数 `exp()` に通してから全体の合計で割ります。大きい値ほど大きい確率になり、全部足すと1になります。`softmax(x)_i = exp(x_i) / Σ exp(x_j)`。

**確率分布**: 各クラスへの確率の並び。たとえば3クラスなら `[0.7, 0.2, 0.1]` のように、各値が0以上で合計1。

**one-hot ベクトル**: 正解クラスだけ1、他は0のベクトル。正解が0番クラスなら `[1, 0, 0, 0, 0]`。「正解の確率分布」とみなせます。

**エントロピー**: ある確率分布の「予測しづらさ・不確実性」を測る量。`H(p) = −Σ p_i log p_i`。サイコロのように全クラスが等確率なら最大、ほぼ1クラスに確率が集中していれば小さい。クロスエントロピーはこのエントロピーの「2分布版」です。

## 仕組みを詳しく

クラスが5個、ある1サンプルのモデル出力(logits)が次だったとします。

```
logits = [2.0, 0.5, -1.0, 0.0, 1.0]   # 形状 [5]
```

**ステップ1: softmax で確率に変換。** 各要素を `exp()` に通します。

```
exp = [7.39, 1.65, 0.37, 1.00, 2.72]
合計 = 13.13
p   = [0.563, 0.126, 0.028, 0.076, 0.207]   # 合計 1.0
```

**ステップ2: クロスエントロピー = 正解クラスの確率の負の対数。** 正解が0番(値2.0を出していたクラス)なら、

```
loss = -log(p[0]) = -log(0.563) = 0.574
```

`log` は自然対数(底 e)。確率が1.0に近いほど `-log` は0に近づき、小さいほど大きく(罰が大きく)なります。正解クラスの確率が 0.9 なら loss = -log(0.9) = 0.105、0.1 なら loss = -log(0.1) = 2.303 と、外すほど急激に罰が増えます。

これが「負の対数尤度 (negative log-likelihood, NLL)」とも呼ばれる理由です。尤度(ゆうど)とは「そのモデルのもとで正解が起きる確率」のこと。それを最大化したいので、符号を反転して「最小化する損失」にしたのがクロスエントロピーです。つまり、クロスエントロピー最小化 = 最尤推定(maximum likelihood estimation)であり、これがこの損失が「原理的に正しい」と言える根拠です。

**一般形(数式)**: 正解の確率分布を `q`(通常 one-hot)、予測確率分布を `p` とすると、

```
H(q, p) = − Σ_i q_i · log(p_i)
```

`q` が one-hot なら、正解クラス以外の項は `q_i = 0` で消え、`−log(p_正解)` だけが残ります。これが上の計算です。記号の意味: `Σ_i` は全クラスにわたる和、`q_i` は正解分布のクラス i の値、`p_i` はモデルの予測確率。

**ステップ3: バッチでは平均をとる。**

```python
loss = F.cross_entropy(logits, labels)
# logits: [B, num_classes]  (softmax 前の生の値を渡す)
# labels: [B]               (各サンプルの正解クラス番号 0..num_classes-1)
```

重要な注意: PyTorch の `F.cross_entropy` には softmax 前の logits を渡します。関数の内部で softmax と log をまとめて計算します。自分で softmax をかけてから渡すと二重にかかって壊れます。なぜ内部でまとめるかというと、`log(softmax(x))` を別々に計算すると値が極端に大きい/小さいとき桁あふれ(オーバーフロー/アンダーフロー)を起こすため、log-sum-exp という数値的に安定な式でまとめて計算するからです。`F.cross_entropy = F.nll_loss(F.log_softmax(...))` という関係を覚えておくと、二項分類向けの `binary_cross_entropy_with_logits`(sigmoid を内蔵)との使い分けも整理できます。

**勾配がきれいになる。** softmax + クロスエントロピーの組み合わせでは、logits に対する勾配が `予測確率 − 正解one-hot` という単純な引き算になります。上の例なら `[0.563−1, 0.126, 0.028, 0.076, 0.207] = [−0.437, 0.126, 0.028, 0.076, 0.207]`。正解クラスは確率を上げる方向、それ以外は下げる方向に押されます。この単純さが学習を安定させ、計算も速くします。

**セグメンテーションでの形**: 画像の各ピクセルをクラス分類するセマンティックセグメンテーションでは、`logits: [B, num_classes, H, W]`、`labels: [B, H, W]` という形で、全ピクセルにわたってクロスエントロピーの平均をとります。自動運転の BEV セグメンテーション(走行可能領域・車線の予測)でも同じ枠組みです。

## 手法の系譜と主要論文

クロスエントロピーは特定の論文で「提案」されたというより、情報理論と最尤推定の枠組みから自然に導かれる古典的な損失です。

**Shannon (1948), "A Mathematical Theory of Communication"** — 情報理論の創始。エントロピー(情報の量・不確実性の尺度)とクロスエントロピー(ある分布を別の分布で符号化したときのコスト)の概念がここから生まれました。クロスエントロピー損失は「正解の分布を、モデルの予測分布で表現しようとしたときに余分にかかる符号コスト」を意味し、予測が正解と一致すれば余分コストはゼロです。`H(q,p) = H(q) + KL(q‖p)` と分解でき、最小化は KL ダイバージェンス(2分布の隔たり)の最小化と等価です。

**Bridle (1990, NIPS)** — softmax 出力を確率として解釈し、最尤推定で学習する枠組みを整理。これによりニューラルネットの分類出力を確率として扱い、負の対数尤度で学習する現在の標準形が定着しました。動機は「ネットの出力を確率として一貫して扱いたい」、効果は「勾配が素直になり学習が安定する」こと。

**Goodfellow, Bengio, Courville (2016), "Deep Learning"** — 回帰で二乗誤差を使うのと同様に、分類では「出力を確率分布とみなした最尤推定 = クロスエントロピー」が原理的に正しい選択だと整理。二乗誤差を分類に使うと出力が飽和した領域で勾配が小さくなり学習が遅いのに対し、クロスエントロピーは外したとき大きな勾配が出るため学習が速いと説明されています。

**Szegedy et al. (2016, CVPR), label smoothing** — クロスエントロピーの過信問題への対策として label smoothing を導入(Inception-v3 の論文)。正解 one-hot を `[0,1,0]` ではなく `[0.033, 0.933, 0.033]` のように少しならして、モデルが特定クラスに確率を1に押し付けすぎるのを防ぎます([ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing))。

**Müller et al. (2019, NeurIPS), "When Does Label Smoothing Help?"** — 平滑化がなぜ効くかを可視化で解明。同一クラスの表現を密なクラスタにまとめ、校正(確信度の正しさ)を改善する一方、知識蒸留の教師として使うとクラス間の微妙な情報を潰す副作用があると定量的に示しました。

## 論文の実験結果(定量データ)

**学習速度の比較(教科書的知見)**: 分類タスクで二乗誤差(MSE)とクロスエントロピーを比較すると、クロスエントロピーのほうが収束が速い。理由は、シグモイド/softmax が飽和した(確信が極端な)領域で、MSE は勾配が小さくなり学習が止まりがちなのに対し、クロスエントロピーは `予測−正解` の勾配が飽和の影響を受けにくいから。これは深層学習で分類にクロスエントロピーが標準採用された最大の実務的理由です。

**label smoothing の効果(Szegedy et al. 2016, Müller et al. 2019)**: ImageNet 分類で label smoothing を適用すると、top-1 精度がわずかに向上(報告により約0.2〜0.5ポイント程度)し、より重要なのはモデルの校正(calibration、出力確率の信頼性)が改善したこと。Müller et al. は、平滑化が同一クラスの表現をクラスタにまとめ過信を抑える一方、知識蒸留の教師としては情報を潰してしまう副作用があると定量的に示しました。指標の意味: 校正とは「確信度80%と言ったとき本当に約80%当たるか」で、安全性が問われる応用では精度以上に重要です。

**現代ネットの過信(Guo et al. 2017)**: "On Calibration of Modern Neural Networks" は、深く大きくなった現代のネットワークが、精度は高いのに著しく過信(出力確率が実際の正解率を大きく上回る)になることを ECE(Expected Calibration Error、確信度と実正解率のずれの平均)という指標で定量化しました。クロスエントロピーで素朴に学習すると過信が進むこと、そして temperature scaling(出力 logits を温度 T で割る簡単な後処理)で校正が大きく改善することを示しています。自動運転のように「確信度を信じて判断を委ねる」場面では、この校正の悪さが安全リスクに直結します。

これらが意味すること: クロスエントロピーは強力だが「正解クラスを1に押し付ける」性質ゆえ過信を招きやすく、平滑化やクラス重み付け、後処理での校正補正で補うのが実務の定石、という結論につながります。

## メリット・トレードオフ・限界

メリット
- 分類の損失として理論的に正しく(最尤推定・KL 最小化に一致)、勾配が `予測確率 − 正解` と単純なので学習が安定し速い。
- ライブラリ一発で書け(`F.cross_entropy`)、softmax と数値安定化(log-sum-exp)を内部でやってくれる。
- 重み付け(class weight)や平滑化(label smoothing)など、不均衡・ノイズ対策の拡張をそのまま受け取れる。

トレードオフ・限界
- 過信を招きやすい。正解クラスの確率を1に押し付けようとするため、校正が悪くなりがち([ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing)・temperature scaling で緩和)。
- クラス不均衡や易しい例の多さに弱い。そのままでは多数派・易しい例に引っ張られる([クラス重み付き損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0349_class-weighted-loss) / [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) で補正)。
- 正解ラベルが間違っていると、その間違いを忠実に学んでしまう(ノイズ耐性が低い)。`−log(p)` がラベルノイズに対して無限大まで罰を出すため、誤ラベルが学習を強く歪める。擬似ラベルを使う場面では特に注意。
- 未解決の課題: ノイズ耐性のある分類損失(対称クロスエントロピー、GCE など)、不均衡下での最適な重み付け、確率校正と精度の両立は、いまも活発な研究テーマ。

## 発展トピック・研究の最前線
- [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) (Lin et al. 2017): 易しい例の損失を抑え、難しい例に集中させるクロスエントロピーの変種。物体検出の不均衡問題で標準化。
- ノイズ耐性損失: Generalized Cross Entropy (Zhang & Sabuncu 2018)、Symmetric Cross Entropy (Wang et al. 2019) など、ラベルノイズに頑健な変種。MAE(平均絶対誤差)はノイズに頑健だが学習が遅いという、CE との中間を埋める。
- 校正の研究: temperature scaling(Guo et al. 2017)など、学習後に出力確率の信頼性を補正する手法。Focal Loss が副次的に校正を改善するという報告(Mukhoti et al. 2020)もある。
