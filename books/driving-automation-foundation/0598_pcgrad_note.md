# PCGrad (勾配射影)

## ひとことで言うと

複数の仕事を1つのモデルで同時に学ぶと、あるタスクが「重みをこちらへ動かしたい」と引っ張る方向と、別のタスクが引っ張る方向が真逆になり、互いに足を引っ張り合うことがあります。PCGrad(Projecting Conflicting Gradients、競合勾配の射影)は、ぶつかり合う勾配のうち「相手と逆向きにぶつかっている成分」だけを幾何的に削り取り、競合を取り除いてから重みを更新する手法です。論文では gradient surgery(勾配の手術)とも呼ばれます。

## 直感的な理解

2人で台車を押す場面を想像してください。2人が同じ向きに押せば台車は速く進みます(協調、内積が正)。横に並んで違う向きに押せば、力の一部は打ち消し合うものの台車はとりあえず進みます(直交に近い)。ところが2人が正面から押し合えば、台車は全く動かず力が無駄になります(競合、内積が負)。マルチタスク学習でタスクの勾配を素朴に足し合わせると、この「正面から押し合う」状況が頻繁に起き、どちらのタスクも改善しない方向に進んでしまうのです。

PCGrad のアイデアは「2人が正面から押し合っているとき、相手とぶつかっている分だけ自分の押す向きを横にずらそう」というものです。ずらした後の力は相手の力と直交する(垂直になる)ので、自分は前に進めるし、相手の進みを邪魔しません。これを全タスクのペアで行い、整えた勾配を足して更新します。

なぜこれが必要かというと、損失の大きさを揃えるだけの手法(不確実性重み付けや GradNorm)では、向きの衝突は解決できないからです。仮に全タスクの勾配の大きさを完璧に揃えても、向きが真逆なら相変わらず打ち消し合います。PCGrad は「大きさ」ではなく「向き」という別の軸に切り込んだ点が決定的に新しいのです。

## 基礎: 前提となる概念

勾配(gradient)とは、損失をネットワークの各重みで微分して得られるベクトルで、「各重みをどの方向にどれだけ動かせば損失が最も速く下がるか」を示します。重みと同じ次元(数百万次元になることもある)を持ち、学習はこのベクトルの逆方向へ少しずつ進みます。

2つのベクトルの関係は内積(dot product、対応する成分を掛けて足した値)で測れます。内積が正なら2ベクトルは鋭角(おおむね同じ向き)、ゼロなら直交(垂直)、負なら鈍角(おおむね逆向き)です。マルチタスクでタスク i とタスク j の勾配 g_i, g_j の内積 g_i · g_j が負のとき、両者は競合(gradient conflict)していると言います。

この競合が引き起こすのが負の転移(negative transfer)です。マルチタスク学習は本来「あるタスクで学んだ表現が別タスクにも役立つ(正の転移)」ことを期待しますが、勾配が競合すると逆に「あるタスクの学習が別タスクを悪化させる」ことが起きます。タスクを増やしたのに精度が下がる、という現象の主因の1つです。

射影(projection)はベクトルの操作で、あるベクトルを別の方向へ「影として落とす」ことです。g_i を g_j 方向に射影すると、g_i のうち g_j に沿った成分が取り出せます。PCGrad はこの成分が「逆向き(負)」のときだけ、それを g_i から差し引きます。

不確実性重み付け([不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall))や GradNorm([GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))は損失や勾配の大きさ(スケール)を調整する系統で、向きには手を付けません。PCGrad は向きに着目するので、これらと役割が異なり併用できます。

## 仕組みを詳しく

中心操作はベクトルの射影による競合成分の除去です。手順をステップで示します。

ステップ1。各タスク i の損失から勾配 g_i を計算します。共有パラメータについてタスクごとに別々に逆伝播し、g_i を個別に保持します。

ステップ2。タスクのペア (i, j) について内積 g_i · g_j を調べます。負なら競合あり、ゼロ以上なら競合なしで何もしません。

ステップ3。競合している場合だけ、g_i から g_j 方向の(負の)成分を引き去ります。

```
g_i ← g_i − ( (g_i · g_j) / ||g_j||^2 ) · g_j     （g_i · g_j < 0 のときのみ）
```

この操作の幾何的な意味は「g_i を、g_j に直交する平面へ射影する」ことです。括弧内の係数は g_i の g_j 方向成分の大きさで、それを g_j 単位ベクトル分だけ引くと、残るのは g_j と直交する成分だけになります。直交していれば、g_i に沿って動いてもタスク j の損失を(一次近似で)悪化させません。

ステップ4。これを全タスクのペアについて、タスクの順序をランダムにシャッフルしながら繰り返します。順序をランダム化するのは、後で述べる順序依存性を緩和するためです。

ステップ5。整えた各タスク勾配を足し合わせ、その合計で共有パラメータを更新します。

```
g_PC = Σ_i g_i^(修正後)
θ ← θ − η · g_PC      （η は学習率）
```

数値例で確認します。タスク A の勾配 g_A = (2, 0)、タスク B の勾配 g_B = (−1, 1) とします。内積は 2·(−1)+0·1 = −2 で負、つまり競合しています。

```
g_A · g_B = -2,  ||g_B||^2 = (-1)^2 + 1^2 = 2
g_A の修正: g_A − (-2/2)·g_B = (2,0) + (-1,1) = (1, 1)
検算: 修正後 g_A=(1,1) と g_B=(-1,1) の内積 = -1+1 = 0 → 直交した
```

g_A を (2,0) から (1,1) に修正することで、B と直交する方向に変わりました。これで A を進めても B を悪化させません。同様に g_B についても A 方向の競合成分を取り除きます。

```
競合あり (内積<0):
   g_A →→        g_A に沿うと g_B が悪化
   g_B ↖

PCGrad 後:
   g_A の「g_B と逆向き成分」を除去 → g_B と直交する g_A' に
   g_A' ↗  g_B ↖   (互いに直交し、打ち消し合わない)
```

PCGrad は重み付けではなく勾配そのものを書き換える「手術」です。だから不確実性重み付けや GradNorm でスケールを揃えつつ、PCGrad で向きを整える、という併用が自然に成立します。

## 手法の系譜と主要論文

PCGrad の出典は Tianhe Yu, Saurabh Kumar, Abhishek Gupta, Sergey Levine, Karol Hausman, Chelsea Finn による "Gradient Surgery for Multi-Task Learning"(NeurIPS 2020, arXiv:2001.06782)です。著者らは Stanford / Google Brain / Robotics at Google に在籍し、強化学習のマルチタスク・メタ学習(Meta-World ベンチマーク)を主要な動機としていました。論文の理論的貢献は、マルチタスク最適化が難しくなる「悲劇的三要素(tragic triad)」を明示したことです。すなわち (a) 勾配の競合(内積が負)、(b) 勾配の大きさの大きな差(dominating gradients、支配的勾配)、(c) 損失地形の高曲率(curvature)。この3つが重なると、素朴な勾配和は最悪のタスクを大きく悪化させると示しました。PCGrad は主に (a) を、競合成分の射影で直接解きます。

前史として、損失のスケールを揃える系統が先行していました。Kendall et al.(CVPR 2018, [不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall))は不確実性で、Chen et al. の GradNorm(ICML 2018, [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))は勾配ノルムでスケールを調整します。PCGrad はこれらとは直交する「向き」の問題に焦点を当て、相補的な立ち位置を取りました。

後続では一般化が進みました。Bo Liu らの CAGrad(Conflict-Averse Gradient descent, NeurIPS 2021, arXiv:2110.14048)は、平均損失の降下を保証しつつ最悪タスクの改善を最大化する更新方向を制約付き最適化で求める、PCGrad の発想を最適化問題として定式化した手法です。Aviv Navon らの Nash-MTL(ICML 2022, arXiv:2202.01017)はタスクをプレイヤーと見立ててナッシュ交渉解で勾配を統合します。MGDA(Multiple Gradient Descent Algorithm, Sener & Koltun NeurIPS 2018)はパレート最適性の観点から共通降下方向を求める古典で、PCGrad 系の理論的背景になっています。

## 論文の実験結果(定量データ)

PCGrad 論文は教師あり学習と強化学習の両方で評価しています。

強化学習では Meta-World という、ロボットアームで50種類のマニピュレーション課題(ボタン押し、ドア開け、物体移動など)を1つのポリシーで学ぶベンチマークを使いました。評価指標は成功率(success rate、課題を達成できた割合)です。報告によると、PCGrad を SAC(Soft Actor-Critic)に組み込むことで、素朴なマルチタスク SAC に対して成功率が大幅に向上しました。具体的には MT10(10タスク)で平均成功率がベースラインを十数ポイント上回り、最も難しい MT50(50タスク)では素朴な MTL-SAC が伸び悩む(報告では成功率が低位にとどまる)のに対し、PCGrad は明確に高い成功率に到達したとされています。タスク数が多いほど勾配の競合が起きやすいので、PCGrad の効果が顕著に出る設定です。著者らはマルチタスクとメタ強化学習(MAML や PEARL と組み合わせる設定)の両方で改善を示し、勾配競合の除去が学習効率(同じ環境ステップ数でより高い成功率に達するサンプル効率)も押し上げることを報告しました。

教師あり学習では、マルチタスク画像認識(MultiMNIST のように複数桁を同時に分類する設定や、CIFAR-100 をタスク分割したマルチタスク版、NYUv2 のシーン理解)で評価し、等重みベースラインや既存の重み付け手法に対して各タスクの平均性能を改善しました。NYUv2 の3タスク(セグメンテーション・深度・法線)では、全タスクをまとめた相対指標 Δ_MTL で等重みベースラインや MGDA を上回り、さらに PCGrad をタスク特化アーキテクチャ(MTAN など)の上に重ねがけすると追加の改善が得られることが示されました。とりわけ、タスク間の負の転移が顕著な設定で、PCGrad が最悪タスクの劣化を防ぐことが示されています。

アブレーションとして重要なのは、悲劇的三要素のどれが効くかの分析です。論文は、競合が存在しても (b) 支配的勾配や (c) 高曲率が無ければ性能劣化は小さいが、3つが揃うと深刻になることを示し、PCGrad が競合除去を通じてこの悪条件を緩和することを実証しました。また、PCGrad を他のアーキテクチャ(タスク特化の構造)や他の重み付け手法と組み合わせても追加の改善が得られることが示され、「向きの調整」が独立した有効因子であることを裏付けています。

指標の意味を補足します。成功率はロボット課題では最も解釈しやすい指標で、数パーセントの差でも実用上は大きな違いです。マルチタスク認識の平均精度の向上は、「タスクを増やしても各タスクが犠牲にならない」ことを意味し、これこそマルチタスク学習が目指す本来の利点です。

## メリット・トレードオフ・限界

メリットは次の通りです。損失のスケールではなく勾配の向きの競合(負の転移の主因)を直接取り除けること。不確実性重み付けや GradNorm などスケール系手法と役割が異なるため併用できること。強化学習を含む幅広いマルチタスク設定で効果が報告されていること。実装はオプティマイザのラッパとして書け、タスクごとの勾配を取り出して射影し合算するだけで原理は単純なこと。

トレードオフと限界もあります。各タスクの勾配を別々に保持・計算する必要があり、メモリと計算が増えます。タスク数を T とすると逆伝播を T 回行い、射影はペア数 O(T^2) に比例するので、タスクが多いほどコストが重くなります。次に、ペアごとの射影は順序依存性があり、ランダム化で緩和するものの結果は厳密には順序に依存します。また勾配の大きさ(支配的勾配)の問題は PCGrad 単体では解かないので、スケール調整は別手法に頼る必要があります。さらに、そもそも競合が少ない問題では恩恵が小さく、計算コストだけが増えることがあります。理論的にも、PCGrad は局所的な一次の競合を除くだけで、損失地形の曲率(三要素の (c))には直接対処しないため、必ずしも大域的に最良の更新方向を与える保証はありません。

未解決の研究課題は、計算コストの削減(勾配のサンプリングや低ランク近似)、向きと大きさを統一的に扱う最適化の定式化、そしてタスク数が非常に多い場合のスケーラビリティです。CAGrad や Nash-MTL はこの方向の発展ですが、こちらも最適化のコストが課題として残っています。

## 発展トピック・研究の最前線

PCGrad は「マルチタスク最適化を、勾配ベクトルの幾何の問題として捉える」流れの起点になりました。後続の CAGrad は「平均損失を必ず下げつつ、最悪タスクの改善を最大化する」更新方向を凸最適化で求め、PCGrad より理論保証が強い代わりに毎ステップ小さな最適化問題を解きます。Nash-MTL はゲーム理論的公平性を持ち込み、各タスクが交渉で合意する更新を計算します。FAMO(Fast Adaptive Multitask Optimization, Liu et al. 2023)は、こうした手法の計算コストを下げ、勾配を陽に保持せずに損失の低下を均衡させる軽量化を狙います。

一方で、批判的検証も進んでいます。Kurin らや Xin らの研究(2022 前後)は、十分にチューニングした単純な等重みやスケール正規化が、多くのベンチマークで高度な勾配操作手法と同等に強いと報告し、「ベンチマーク依存で万能の勝者はいない」という現状認識を示しました。それでも PCGrad が提示した「勾配の競合」という概念枠組みは、マルチタスク・連続学習(continual learning、新しいタスクを学ぶと古いタスクを忘れる破滅的忘却の問題)・メタ学習などで広く参照され続けています。連続学習における勾配射影(古いタスクを悪化させない方向へ新タスクの勾配を射影する手法群)は PCGrad と同じ幾何的発想を共有しています。
