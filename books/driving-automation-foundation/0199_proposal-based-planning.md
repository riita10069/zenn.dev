# プロポーザルベース計画

## ひとことで言うと
車や歩行者がこれからどう動くかを一発で当てに行くのではなく、まず「ありえそうな行き先や道筋」をたくさん候補(proposal = 提案)として吐き出し、それぞれに「これくらいありえる」というスコアを付け、良いものを選んで残す、という段階的な手法です。たくさん出して、採点して、選ぶ。この三拍子が本質です。一本に賭けないので、未来が複数に枝分かれする状況(交差点など)を自然に扱えます。

## 直感的な理解
人間が運転計画を立てるとき、いきなり「0.1秒後にハンドルを何度切る」とは考えません。まず「あの右折レーンに入ろう」「前の車の後ろに付こう」という大まかな行き先を決め、それから具体的な動き方を埋めます。プロポーザルベース計画はこの二段構えを機械で再現したものです。

なぜ一発で当てに行かないのか。未来予測には「多峰性(multimodality)」という厄介な性質があります。多峰とは、ありえる未来が一つの山ではなく複数の山に分かれていること。交差点では直進も右折も左折もどれも妥当です。ここで「未来の座標を一本だけ数値で出す」回帰(regression、数値を直接予測すること)を素直に学習すると、左折の正解と右折の正解の平均、つまり「どこにも行かない、まっすぐ縁石に突っ込む軌道」を出してしまいます。これを「モード平均化(mode averaging)」と呼びます。複数の正解の真ん中を取ると、どの正解でもない最悪の答えになるのです。

プロポーザルベースの発想は、この平均化を構造的に避けます。「一本に決めず、複数候補を別々に保持し、それぞれを採点する」。左折候補と右折候補が別オブジェクトとして並存するので、平均化が起こりようがありません。さらに「採点(スコアリング)」という形を取ることで、各候補が「どれくらいありえるか」を数値で扱えます。これを評価可能性(後から候補同士を比較・検証できる性質)と呼び、自動運転では特に重要です。なぜなら、後段で「この候補は衝突する」「これは信号無視だ」といった安全フィルタを候補ごとに掛けられるからです。

## 基礎: 前提となる概念
理解に必要な土台を、知らない前提で順に押さえます。

軌道(trajectory)とは、ある物体の位置を時刻順に並べたものです。たとえば「6秒先まで0.5秒刻み」なら12個の点で、各点が (x, y) 座標を持つので `[12, 2]` という形(tensor shape、テンソルの各次元のサイズ)になります。

回帰と分類の違い。回帰は数値そのものを当てる問題(座標を直接出す)、分類は選択肢の中からどれかを選ぶ問題(「左折」「直進」「右折」のどれか)です。プロポーザルベースは両方を混ぜます。候補を「選ぶ(分類的なスコアリング)」一方で、候補の中身は「数値の軌道(回帰的)」です。

region proposal(領域提案)という言葉は物体検出から来ています。画像のどこに物体があるかを当てる課題で、二段階検出器と呼ばれる Faster R-CNN(Ren et al., NeurIPS 2015)は「画像のあちこちに大量の候補枠を出し → 各枠を採点して本物の物体だけ残す」という流れを取りました。proposal という用語と「大量に出して採点して絞る」というパイプライン思想は、ここが起源です。プロポーザルベース計画は、この検出の二段階思想を「枠」から「軌道」へ移植したものだと理解すると一気に腑に落ちます。

NMS(Non-Maximum Suppression、非最大値抑制)も検出からの借用です。スコアの高い候補同士が近すぎて重複しているとき、一番スコアの高いものだけ残して近いものを消す後処理を指します。これがないと「ほとんど同じ軌道が6本」のような多様性のない出力になってしまいます。

ゴール(goal / target)とは軌道の最終到達点のことです。プロポーザルベースの多くは「まずゴールだけ予測 → ゴールへの道筋を埋める」という分解を取ります。ゴールという1点を絞るほうが、軌道全体(12点×2次元)を一度に当てるより問題が易しいからです。

## 仕組みを詳しく
代表例として TNT(Target-driveN Trajectory prediction、「ターゲットに駆動された軌道予測」)の三段階を分解します。

ステップ1: ターゲット予測(target prediction)。
まず「最終的にどこへ行き着くか」の候補を大量に出します。地図上の道路(車線中心線など)に沿って点を密にばらまきます。たとえば候補点 N=1000 個。各点について「ここがゴールである確率」と「その点からの微小なズレ(オフセット)」を予測します。出力は各候補点の確率 `[N]` と微調整 `[N, 2]`。なぜ点を道路に沿って置くのか。車は道路の外には基本行かないので、地図に沿わせるだけで「ありえない候補」を最初から排除できるからです。ここで確率上位の点(たとえば上位50点)を選びます。

ステップ2: 軌道補完(motion / trajectory completion)。
選ばれた各ゴールに対し「現在地からそのゴールまでどう進むか」という完全な軌道を一本ずつ生成します。ゴールが固定されているので、軌道全体をゼロから当てるより遥かに楽です(終点が決まったお絵かきの方が簡単なのと同じ)。各ゴールにつき `[T, 2]`(T=時刻数)が出るので、50ゴールなら `[50, 12, 2]`。

ステップ3: スコアリングと選択(scoring + selection)。
生成した複数の軌道それぞれに「これがどれくらいありえるか」のスコアを付けます。スコアは軌道全体の文脈(他車との整合、地図との整合、自然さ)を見て決めます。スコア上位を残しつつ、近すぎる軌道は NMS で間引いて、最終的に多様で良質な数本(たとえば6本 `[6, 12, 2]`)を残します。

数値例で流れをまとめると、
```
ゴール候補 N=1000        →  [1000] の確率 + [1000,2] のオフセット
上位50点を選択          →  50ゴール
各ゴールへ軌道生成       →  [50, 12, 2]
スコアリング + NMS      →  最終6本  [6, 12, 2]
```

なぜこの三段構造が効くのか。問題を「行き先の選択(離散的・採点しやすい)」と「行き先までの補完(条件付きで易しい)」に分けることで、それぞれが解きやすい部分問題になるからです。一気に `[12,2]` を回帰するより、各段の責務が明確で学習も安定します。

DenseTNT(Gu et al., ICCV 2021)はこの「ゴール候補を離散点としてばらまく」部分を改良しました。TNT はゴールを地図上の離散点として置くため、点の密度が精度の上限を決めます。密にすれば精度は上がるが計算は重くなる、というジレンマです。DenseTNT は「密なゴール確率の地図(dense goal probability map)」を学習し、点単位ではなく地図上の確率分布として行き先を表します。さらに「どのゴールを正解ラベルにするか」を後付けで最適に割り当てるオンライン最適化(目的関数を解いて教師信号を作る)を導入しました。これにより、事前に固定したアンカーやゴールセットに頼らない anchor-free(アンカー無し、事前定義の雛形に縛られない)に近い形で、高精度を達成しました。

## 手法の系譜と主要論文
proposal の思想自体は物体検出の Faster R-CNN(Ren, He, Girshick, Sun, NeurIPS 2015)に遡ります。「大量に候補を出し、採点して絞る」という二段階の枠組みがここで確立しました。

軌道予測へ本格的に持ち込んだのが TNT(Zhao, Gao, Lan, Sun, Sapp, Varadarajan ら, CoRL 2020, arXiv:2008.08294)です。動機は明快で、軌道全体を一気に予測するのは難しく多峰性も扱いにくいので、「まず行き先を絞る」という人間に近い階層分解で問題を易しくしたかった。新規性は target(ゴール)を明示的に予測の中心に据えた点(target-driven)にあります。

次に DenseTNT(Gu, Sun, Zhao, ICCV 2021, arXiv:2108.09640)が、TNT の離散ゴールが作る精度上限を、密なゴール確率と教師割り当ての最適化で取り払いました。これが「goal-based prediction」系譜のひとつの到達点です。

同時期に、ゴールを介さず候補を直接出して採点する系統も発展しました。VectorNet(Gao et al., CVPR 2020)が地図と軌道をベクトル(線分の集合)として符号化する基盤を作り、その上に TNT/DenseTNT が乗りました。さらに後年、「軌道の語彙(vocabulary、よく出る軌道パターンの辞書)上の確率分布を学ぶ」というスコアリングの一般化や、複数候補を出してから絞り込む拡散ベースの計画(アンカー付き拡散)など、proposal 思想は形を変えて生き続けています。スコアリング段は安全性チェックを後付けしやすいという実用的利点があり、エンドツーエンド計画の中でも「候補を出して評価する」という骨格は繰り返し再発明されています。

## 論文の実験結果(定量データ)
評価指標を先に説明します。軌道予測では minADE / minFDE がよく使われます。

minADE(minimum Average Displacement Error)は、モデルが出した K 本の候補のうち正解に最も近い一本を選び、その軌道の各時刻における正解との距離を平均した値(単位はメートル)です。「K本のうちベストが平均何メートルずれているか」を測ります。小さいほど良い。minFDE(minimum Final Displacement Error)は同じく最良候補の、最終時刻だけの距離。終点をどれだけ当てられたかを測ります。Miss Rate(MR)は、最良候補の最終地点が正解から一定距離(通常2.0m)以上ずれた割合で、「大外しした割合」を表します。これらは複数候補を出すモデルを公平に測るための「min-of-N」型の指標で、proposal ベースと相性が良いです。

TNT は Argoverse / nuScenes / Waymo / INTERACTION といった主要な予測ベンチマークで、当時の最先端と肩を並べるか上回る精度を報告しました。Argoverse Motion Forecasting(車の3秒予測)で、回帰一本やナイーブな多峰手法に対し minADE/minFDE を明確に下げています。報告によると、ゴール予測を介すことで特に minFDE(終点誤差)の改善が大きく、「行き先を先に絞る」分解の効果が終点精度に直結することが示されました。

DenseTNT は Argoverse Motion Forecasting Challenge で当時の首位級の成績を報告しました。K=6(6本予測)設定の minFDE と Miss Rate で従来手法を上回り、特に Miss Rate の低減(大外しが減る)が顕著でした。これは密なゴール確率が「ありえる行き先を取りこぼしにくい」ことの直接の表れです。

アブレーション(どの要素を抜くとどうなるかの検証)では、ゴール予測段を外して直接軌道を回帰すると多峰性が失われ minFDE が悪化すること、NMS や多様性確保の後処理を外すと似た軌道ばかりになり「6本のうちベスト」が伸びないこと(minADE が頭打ち)が報告されています。DenseTNT 側では、教師ラベルの最適割り当てを単純な最近傍割り当てに戻すと精度が落ち、割り当て最適化が効いていることが示されました。これらは「候補生成の質」「採点の質」「多様性の確保」の三つが揃って初めて性能が出る、というプロポーザルベースの本質を裏付けています。

## メリット・トレードオフ・限界
メリット。多峰な未来を複数候補としてそのまま保持でき、モード平均化を構造的に回避できます。各候補にスコアが付くため評価可能性が高く、後段で安全制約のフィルタを候補ごとに足せます。ゴールを先に絞る階層分解で問題が易しくなり、終点精度(minFDE)が伸びやすい。候補を動的に生成するので、後述の固定集合方式より柔軟です。

トレードオフと限界。第一に、多段パイプライン(生成 → スコアリング → 選択)で構成が複雑になり、各段の損失設計と学習が大変です。第二に、ゴール候補の密度や候補数が精度と計算量の真っ向トレードオフになります(TNT の弱点、DenseTNT が緩和したのもここ)。第三に、NMS など後処理のしきい値調整が必要で、勾配が流れずエンドツーエンド最適化と相性が悪い部分が残ります。第四に、生成段が貧弱だと、どれだけスコアリングが優秀でも良い軌道は出てきません。候補集合に無いものは選べない、という意味で生成段が精度の天井になります。研究上の未解決課題としては、スコアリング段が「正解への近さ」を学習しているだけで「安全性・社会的妥当性」を本当に評価できているか、候補多様性を学習で陽に保証する方法、そして後処理を学習可能な形に統合するエンドツーエンド化が挙げられます。

## 発展トピック・研究の最前線
スコアリングの一般化として、軌道を語彙(よく出るパターンの辞書)へ量子化し、その語彙上の確率分布を直接学ぶ流れがあります。これは「採点」を「分類」に帰着させ、大規模な候補集合をエンドツーエンドで扱う方向です。また、候補生成を生成モデル(拡散・フローマッチング)に置き換え、ノイズから多様な候補を引いてから採点・選択するハイブリッドも登場しています。これは proposal 思想(出して採点して選ぶ)と生成モデルの表現力を両取りする試みです。さらに、Transformer ベースの集合予測(物体検出の DETR のように、固定数のクエリが各候補を担当し NMS を不要にする)を軌道予測へ持ち込み、後処理を学習に溶かし込む研究も進んでいます。
