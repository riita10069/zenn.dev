# 点群の深層学習

## ひとことで言うと
点群(point cloud)とは、LiDAR(ライダー、レーザー光を飛ばして物に当たって戻る時間から距離を測るセンサのことです)が出力する3次元の点の集まりです。各点は (x, y, z) という3次元座標と反射強度(どれだけ強く光が返ってきたか)を持ち、1回のスキャンで10万点ほどになります。点群の深層学習とは、この「順序のない大量の点」をニューラルネットで処理し、物体検出やセグメンテーションをする手法のことです。自動運転で「前の車まで正確に何メートルか」を知るための中心技術です。

## 直感的な理解
カメラ画像は色や見た目はよく分かりますが、奥行き(距離)が直接は分かりません。写真に写る前方の車が遠くにある大きな車なのか、近くにある小さな車なのかは、画像だけでは原理的に曖昧です(スケール曖昧性)。一方 LiDAR は光の往復時間から距離を直接測れるので、各点が「空間のどこにあるか」を物理的に確定できます。自動運転では距離の正確さが安全に直結する(止まれるか止まれないかを分ける)ため、点群を扱う技術が重要になります。

ところが点群は画像とは性質がまったく違い、画像用に発達した CNN(Convolutional Neural Network、格子状に並んだ画素に畳み込み演算をするネットワーク)をそのまま使えません。難しさは主に3つです。第一に順序がないこと(後述)。第二に密度が不均一なこと(近くは点が密、遠くはまばら)。第三に規則的なグリッドでないこと(画像のように等間隔の格子に値が並んでいない、点が空間に散らばっている)。点群の深層学習の歴史は、この3つの難しさをどう乗り越えるかの工夫の歴史です。

## 基礎: 前提となる概念
順序不変性(permutation invariance)が最重要概念です。点群は点の集合であり、点を並べ替えても表す形は同じです。{点A, 点B, 点C} と {点C, 点A, 点B} は同じ物体を表します。ところがニューラルネットの普通の入力は「並び順」を持つので、入力をシャッフルすると出力が変わってしまう。これでは困るので、並べ替えても結果が変わらない演算(順序不変な演算)を使う必要があります。N個の点を入力したとき、その順番をどうシャッフルしても出力が一致しなければなりません。

順序不変を実現する鍵は対称関数(symmetric function)です。入力の順序によらず同じ値を返す関数で、代表例が max(最大値)、sum(合計)、mean(平均)です。max([3,1,2]) も max([1,2,3]) も 3 で同じ。点群処理の多くは、各点を独立に変換したあと、この対称関数で全点を集約することで順序不変性を得ます。理論的には、PointNet 論文が「十分大きな MLP と max pooling の組み合わせは、任意の集合関数を近似できる」ことを証明しており、これがこのアプローチの正当性を保証しています。

MLP(多層パーセプトロン、全結合層を重ねたネットワーク)、voxel(ボクセル、3次元空間を立方体の格子セルに区切った1マス)も用語として押さえておきます。voxel は画像のピクセルを3次元に拡張したものと思えばよいです。受容野(receptive field、出力の1点が入力のどれだけの範囲を見ているか)という概念も後で使います。

## 仕組みを詳しく
点群処理には大きく3つの系統があります。それぞれ「順序のなさ」をどう乗り越えるかが違います。

点を直接扱う系。PointNet(Qi ら、CVPR 2017)は、各点を独立に小さな MLP に通して特徴を出し、最後に全点にわたって max pooling(各特徴次元ごとに最大値だけを取る操作)をします。max は対称関数なので順序不変性を達成します。具体的には、点数 N、各点が1024次元の特徴を持つ tensor [N, 1024] に max pooling をかけると、N が消えて [1024] という1本のグローバル特徴になります。N が何であろうと、並びがどうであろうと結果は同じです。PointNet はさらに T-Net という小ネットで入力点群を学習的に回転補正(空間変換、向きのずれを直す)し、向きのずれに頑健にしました。

PointNet++(Qi ら、NeurIPS 2017)は、PointNet が苦手だった局所構造(細かい形)を捉えるため、近い点をグループ化して階層的に処理します。まず farthest point sampling(FPS、できるだけ離れた代表点を選ぶ手法)で代表点を選び、その周りの近傍点をまとめて小さな PointNet にかけ、これを段階的に繰り返して、細かい形から大きな形まで多段階で特徴を集めます。画像 CNN の「畳み込みを重ねて受容野を広げる」発想を点群に持ち込んだものです。これにより、PointNet が見られなかった「点の局所的な配置パターン」を捉えられるようになりました。

voxel 化系。VoxelNet(Zhou & Tuzel、CVPR 2018)は、空間を voxel に区切り、各 voxel の中に入った点を VFE(Voxel Feature Encoding)という小ネットで1本の特徴にまとめ、その後 voxel を3次元の格子とみなして 3D 畳み込みをかけます。これで「不規則な点群」を「規則的な格子」に変換し、CNN 的な処理に持ち込みます。SECOND(Yan ら、Sensors 2018)はこの 3D 畳み込みを sparse convolution(点が入っている voxel だけ計算する疎な畳み込み)にして大幅に高速化しました。LiDAR 空間の大半(9割以上)は空っぽなので、空のマスを計算しないだけで劇的に速くなります。

柱(pillar)化系。PointPillars(Lang ら、CVPR 2019)は、z(高さ)方向を区切らず、地面に垂直な「柱(pillar)」に点をまとめます。3次元を縦方向につぶして実質2次元の格子にするので、軽い 2D 畳み込みで処理でき非常に高速です。地面を例えば0.16m四方の柱に分け、各柱の点をまとめて特徴化し、[柱の数, 特徴次元] を擬似画像 [チャンネル, 縦, 横] に並べ替えて 2D CNN に渡します。3D conv を完全に避けられるのが速度の鍵で、自動運転の 3D 物体検出で広く使われています。

数値例で粒度の違いを掴みます。LiDAR の検出範囲を前後左右80m、上下数mとすると、0.1m voxel では空間に数千万マスができますが、実際に点が入るのは数万〜十数万マスだけです。voxel 系が sparse 化を必須とする理由がここにあります。pillar 系は高さを畳むことで、例えば前後左右の格子だけ(数十万マス程度)に減らし、計算をさらに軽くします。

## 手法の系譜と主要論文
PointNet と PointNet++(Qi ら)の貢献は、点群を「いったん画像や格子に変換しなくても」深層学習で直接扱えると示したことです。それまでは点群を格子化してから処理するのが当たり前で、その変換で情報が落ちていました。効果は順序不変な処理の確立と、それを支える理論保証。トレードオフは各点を独立に見るぶん局所の細かい形を捉えにくい点で、これを PointNet++ が階層化で補いました。

VoxelNet(Zhou & Tuzel)は、点群を学習可能な特徴で voxel 化する end-to-end な枠組みを示しました。精度は高い一方、3D 畳み込みは計算とメモリの負担が大きいのがトレードオフで、SECOND の sparse conv はまさにそこを解消する改良です。SECOND はまた、回転角の予測に sine-error loss を導入するなど検出特有の工夫も加えました。

PointPillars(Lang ら)は、z 方向を畳んで 2D にすることで速度を最優先しました。効果はリアルタイム動作に十分な高速性、トレードオフは高さ方向の細かい情報をある程度捨てている点です。それでも実用上は精度と速度のバランスが良く、定番になりました。

PointRCNN(Shi ら、CVPR 2019)は、点群から直接物体候補(proposal)を生成する 2段階検出器で、点ベースアプローチの検出性能を押し上げました。PV-RCNN(Shi ら、CVPR 2020)はさらに進んで、voxel 系の効率と点系の柔軟な位置精度を組み合わせる(voxel 特徴を keypoint に集約する)ハイブリッドで、当時の検出 SOTA を更新しました。系譜としては「点系と voxel 系は対立ではなく融合できる」ことを示した位置づけです。

3系統を「精度・速度・メモリ」で大まかに並べると、点直接系は柔軟だが大規模点群でやや重く、voxel 系は精度が出るがメモリを食い(sparse 化で緩和)、pillar 系は最速だが高さ情報を一部犠牲にする、という整理になります。現在はこれらを組み合わせるハイブリッドが主流です。

## 論文の実験結果(定量データ)
3D 物体検出の主要ベンチマークは KITTI と nuScenes です。評価指標は 3D mAP(mean Average Precision、複数の物体クラスにわたって、予測した 3D 枠が正解と IoU 一定以上で重なる割合を総合したもの)で、0〜100% で高いほど良い。KITTI は車・歩行者・自転車を Easy/Moderate/Hard の難易度別に評価し、車は IoU 0.7 を一致閾値とします。nuScenes は10クラス・360度・多様な天候を含み、NDS(nuScenes Detection Score、mAP に距離・速度・向きの誤差を加味した総合指標)を主指標とします。

PointPillars 論文の KITTI 実験では、車クラスで当時の高精度手法に匹敵する 3D mAP を達成しつつ、推論速度を桁違いに高速化したことを報告しています。具体的には、当時の voxel 系手法が数〜十数 FPS だったのに対し、PointPillars は報告でおよそ 60 FPS 超(LiDAR のスキャン周期 10〜20Hz を大きく上回る速度)を実現しました。「z 方向を畳んで 2D 化すると、車検出の精度をほとんど落とさず一桁速くなる」というアブレーション的知見が、その後の実用システムの設計を方向づけました。

SECOND は KITTI で、VoxelNet の dense な 3D 畳み込みを sparse conv に置き換えるアブレーションにより、精度を保ちつつ学習・推論をともに数倍高速化したことを示しました。PV-RCNN は KITTI の車 Moderate で 80% 台後半の 3D AP を達成し、点系・voxel 系単独を上回ることで「両者の融合が効く」ことを定量的に示しました。PointNet 系は ModelNet40(3D 形状分類)で当時の最高精度(約 89% accuracy)に並び、点群を直接扱える初の実用的枠組みであることを示しました。PointNet++ はこれを約 90〜91% へ引き上げ、局所構造を捉える階層化の効果を裏付けました。これらの数値が示す共通の教訓は、「点群処理は精度と速度とメモリが強く綱引きしており、どの表現(点/voxel/pillar)を選ぶかがその配分を直接決める」という点です。

## メリット・トレードオフ・限界
メリットは、LiDAR 点群が距離を直接・高精度に測れるため、奥行き推定が苦手なカメラを補えること、3D 物体検出での位置精度が高いことです。夜間や逆光でもアクティブセンサ(自ら光を出す)であるため安定して距離が取れます。これはカメラ(受動センサ、外部の光に依存)が苦手とする条件です。

トレードオフは、LiDAR がセンサ自体高価で、点群は順序のなさ・不均一さ・非格子性ゆえに専用処理が要ることです。手法ごとに精度と速度とメモリが綱引きの関係にあり、用途に応じて系統を選ぶ必要があります。カメラのみの設計に対しては、センサ追加のコストと複雑さ、キャリブレーション(センサ間の位置合わせ)の手間が増します。

限界・課題として、点群は色やテクスチャの情報を持たない(信号機の色や標識の文字が読めない)ため、カメラとの相補性が本質的です。また遠距離では点が極端にまばらになり(80m 先の歩行者は数点しか当たらない)、小さな物体や遠くの歩行者の検出が難しくなります。雨・雪・霧では反射ノイズが乗りやすく、舞い上がる水しぶきを物体と誤認する問題もあります。研究の未解決課題には、点群とカメラの効果的な融合、ラベル付けが高コストな 3D アノテーション(人手で3D枠を付ける作業)を減らす自己教師あり/弱教師あり学習、時系列点群からの動き推定(scene flow)、ドメイン間(LiDAR の機種や設置が違う環境)での汎化などがあります。

## 発展トピック・研究の最前線
センサ融合が大きな潮流です。BEVFusion(Liu ら、NeurIPS 2022)は、カメラ画像と LiDAR 点群をそれぞれ BEV(鳥瞰図)空間の特徴へ落とし込み、その共通の BEV 表現上で融合します。動機は、画像が色・意味に強く点群が幾何・距離に強いという相補性を、同じ座標系で素直に統合することです。ここで点群を BEV へ落とす処理として、本稿で説明した pillar/voxel 化が直接効いてきます。BEVFusion は nuScenes の 3D 検出・BEV セグメンテーションで、単一センサ手法を上回る精度(検出で NDS が大きく向上)を報告し、融合の有効性を定量的に示しました。重要なのは、片方のセンサが故障しても残りで動作するロバスト性(頑健性)も同時に得られる点です。

さらに近年は、点群用の Transformer(Point Transformer、Zhao ら ICCV 2021 などの系譜)で局所注意(近傍点だけにアテンションをかける)により細かい形状を効率的に捉える方向、点群の自己教師あり事前学習(Point-MAE など、点の一部をマスクして復元させて表現を学ぶ)、点群と言語・画像をまたぐマルチモーダル基盤モデル、そして 4D(時系列)点群から動きや占有(occupancy、空間の各セルが埋まっているか)を予測する手法が研究の最前線です。占有予測は「未知の物体でも空間が埋まっているかどうかは分かる」ため、ラベルにない障害物への安全性の観点で注目されています。
