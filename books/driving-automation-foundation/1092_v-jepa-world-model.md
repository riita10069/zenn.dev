# V-JEPA（潜在空間世界モデル）

## ひとことで言うと
V-JEPA(ブイ・ジェパ)は、動画の隠した部分を「ピクセル(画素)そのもの」で当てるのではなく、「特徴(意味を表す抽象的な数値ベクトル)」のレベルで予測することで、ラベルなしの動画から自分で学ぶ(自己教師あり)モデルです。「世界が次にどうなるか」を抽象空間で予測する、という考え方を体現しており、再構成系の自己教師あり学習に対する有力な対抗アプローチとして研究コミュニティで注目されています。「将来の状態を、画素ではなく特徴空間で予測する」という発想は、計画や予測を扱う多くのシステムの思想的な土台になっています。

## 直感的な理解
あなたが運転中、前方の駐車車両の陰から子どもが飛び出すかもしれない、と身構える場面を考えてください。このときあなたの頭の中で起きているのは「次の瞬間の風景を1画素ずつ正確に思い描く」ことではありません。「子どもが出てくるかもしれない」という意味レベルの予測です。風景の細部(車のボディの反射、アスファルトの粒、葉っぱの揺れ)を正確に当てる必要はまったくなく、むしろそれに気を取られるのは有害です。

V-JEPA の核心はこの直感そのものです。学習機械に未来や隠れた部分を予測させるとき、画素という最も細かいレベルで当てさせると、本質的に予測不可能で意味のない細部にまで労力を割かせてしまう。だから予測は「意味のレベル(特徴空間)」で行うべきだ、というわけです。言い換えると、世界の不確実な部分(揺れる葉の正確な位置など)は捨て、決定的で意味のある構造(子どもが出てくる/出てこない)だけを予測対象にする、という割り切りです。

## 基礎: 前提となる概念
自己教師あり学習(Self-Supervised Learning、SSL)とは、人間がラベル(これは猫、これは車)を付けなくても、データ自身を使って学ぶ枠組みです。ラベル付けは高コストなので、ラベルなしの大量データから表現(特徴)を学べることには大きな価値があります。代表的な手口が「データの一部を隠して、隠した部分を当てさせる」やり方で、文章の穴埋め問題(BERT が有名)を画像や動画でやるイメージです。

埋め込み(embedding)あるいは表現(representation)とは、画像や動画を意味のレベルに圧縮した数値ベクトルです。生のピクセルが「絵の具の量」だとすれば、埋め込みは「この絵が何を描いているか」を数値化したものに近いです。良い表現とは、見た目が違っても意味が同じものは近くに、意味が違うものは遠くに配置されたベクトル空間を指します。

エンコーダ(encoder)は入力を埋め込みに変換するニューラルネット、プレディクタ(predictor、予測器)はある埋め込みから別の埋め込みを予測するニューラルネットです。V-JEPA のエンコーダには Vision Transformer(ViT、画像をパッチに分けて Transformer で処理する画像版 Transformer)が使われます。

世界モデル(world model)とは「現在の状態と行動から、次の状態がどうなるかを予測する内部モデル」を指します。これがあれば、実際に動く前に頭の中で「こう動いたらこうなる」とシミュレーションでき、計画に使えます。古典的にはピクセル空間で次フレームを描く生成的世界モデルが多かったのに対し、V-JEPA は「次状態を特徴空間で予測する世界モデル」を志向します。

## 仕組みを詳しく

### ピクセル再構成の何が無駄なのか
最初に流行った自己教師ありは「画像の一部をマスク(隠す)して、隠した画素を復元させる」方式でした。MAE(Masked Autoencoder、マスク自己符号化器、He et al. CVPR 2022)が代表で、画像の75%をランダムに隠して残り25%から元画素を復元させます。

ところがピクセル単位の復元には根本的な無駄があります。壁の細かい模様、葉1枚1枚の形、映像のノイズなど、本質的に予測不可能で意味理解にほとんど寄与しない細部まで、モデルは正確に当てる努力を強いられます。「道路をどう走るべきか」を学びたい場面で、隣のビルの窓ガラスの反射を1画素単位で再現する必要はありません。再構成損失はこうした無関係な高周波細部に大きな勾配を割くため、サンプル効率(同じ性能に到達するのに必要なデータ量)で不利になりがちです。これは「予測誤差をピクセルの2乗誤差で測ると、目立つが意味のない高周波成分の誤差が支配的になる」という、生成モデル一般に共通する問題でもあります。

### LeCun の JEPA 構想
2022年、Yann LeCun(Meta AI のチーフ AI サイエンティスト)が構想論文「A Path Towards Autonomous Machine Intelligence」で JEPA(Joint-Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)を提唱しました。骨子は「予測はピクセル空間ではなく表現(埋め込み)空間で行うべきだ」というものです。抽象空間で予測すれば、予測不可能で無意味な細部を自然に無視でき、意味のある構造だけを学べます。同論文はこれをエネルギーベースモデル(状態の組み合わせの「ありえなさ」をスカラのエネルギーとして測り、ありそうな組には低いエネルギーを与える枠組み)と組み合わせ、自律機械知能の設計図として提示しました。生成(generative)で全部を当てにいくのではなく、予測(predictive)で意味だけを当てる、という対比が思想の中心にあります。

### V-JEPA の処理の流れ
V-JEPA(2024、Meta AI の Bardes ら、arXiv:2404.08471)は JEPA を動画に適用したものです。流れは次の通りです。

1. 動画クリップ(例えば16フレーム)を小さなパッチ(画像の小片、例えば時空間で16×16ピクセル×2フレームの塊)に分け、時空間のトークン列にする。
2. その一部を時空間的に大きくマスクする。ランダムにばらまくのではなく、連続したブロック(空間的に大きな領域を全フレームにわたって)を丸ごと隠すのが効くと報告されています。歯抜けが小さいと近傍からの単純補間で当てられてしまい、意味的な予測を学ばないためです。
3. 見えている部分(コンテキスト)を「コンテキストエンコーダ」に通し、特徴ベクトルにする。
4. その特徴から「プレディクタ(予測器)」が、隠した部分の特徴ベクトルを予測する。プレディクタには「どの位置を予測するか」を示す位置トークン(マスクトークン)を与える。
5. 一方、隠した部分の正解は「ターゲットエンコーダ」に元の(隠していない)動画を通して得る。
6. 学習目的は「予測した特徴」と「ターゲットエンコーダが出した特徴」を近づけること。損失はピクセルの L2 ではなく、特徴ベクトル同士の L1 距離。

### 形(tensor shape)のイメージ
- 入力動画: `[16フレーム, 3, 224, 224]`
- パッチ化後トークン列: 例 `[N, 1024]`(N=トークン数、1024=特徴次元)
- マスク後の見える部分: `[N_visible, 1024]`
- プレディクタの出力(隠した部分の予測特徴): `[N_masked, 1024]`
- ターゲット特徴(正解): `[N_masked, 1024]`
- 損失: この2つの `[N_masked, 1024]` の距離

```
ピクセル復元(MAE):  見える部分 ─> デコーダ ─> [N_masked, 画素 3x16x16=768]  ← 画素を当てる
特徴予測(V-JEPA):  見える部分 ─> プレディクタ ─> [N_masked, 1024]            ← 意味を当てる
                                                  ↑近づける↓
                  隠した部分の元動画 ─> ターゲットエンコーダ ─> [N_masked,1024]
```

ピクセル復元では出力が「画素そのもの」ですが、V-JEPA では出力が「特徴(意味)」である点が決定的に違います。予測の「正解」がモデル自身(ターゲットエンコーダ)の出力なので、何を予測すべきかという目標自体が学習とともに動く、という独特の構造を持ちます。

### 崩壊(collapse)を防ぐ工夫
特徴空間で予測する手法には「全部同じベクトルを出せば距離はゼロになる」というズル(表現の崩壊、representation collapse。モデルが手抜きをして入力によらず同じ答えを出す現象)の危険があります。コンテキストエンコーダとターゲットエンコーダが同じ重みで、両方に勾配を流すと、両者が定数ベクトルへ縮退してしまいます。損失だけ見れば崩壊は最適解(損失ゼロ)なので、何も対策しないと素直にそこへ落ちます。

V-JEPA はターゲットエンコーダを、コンテキストエンコーダの重みの EMA(Exponential Moving Average、指数移動平均。`θ_target ← m·θ_target + (1−m)·θ_context` のように毎ステップ少しずつ追従する「ゆっくり版コピー」、係数 m は0.99台後半)として更新し、ターゲット側には勾配を流さない(stop-gradient)ことで崩壊を防ぎます。ターゲットが少し遅れて動く非対称性が、定数縮退を避けて意味のある目標を与え続けます。この設計は BYOL(Grill et al. NeurIPS 2020、対比的な負例ペアなしで崩壊を防いだ先駆け)や DINO、I-JEPA(画像版)から受け継いだものです。対比学習(contrastive、正例と負例を引き離す)が大量の負例を要したのに対し、EMA + stop-gradient は負例なしで崩壊を回避できるのが利点です。

## 手法の系譜と主要論文

- LeCun, "A Path Towards Autonomous Machine Intelligence", 2022(構想論文)。生成的なピクセル予測ではなく、表現空間での予測(JEPA)とエネルギーベース世界モデルを中心に据えた自律機械知能の設計図。ピクセル予測は予測不可能な細部に縛られ効率と汎化で不利、という問題意識を理論的に整理した。後続の I-JEPA / V-JEPA の支柱。

- Grill et al., "BYOL", NeurIPS 2020(arXiv:2006.07733)。負例を使わずに、オンラインネットワークがターゲットネットワーク(EMA)の出力を予測する形で崩壊を回避できることを実証。JEPA 系の崩壊回避設計の直接の祖先。

- Assran et al., "I-JEPA", CVPR 2023(Meta AI、arXiv:2301.08243)。静止画で JEPA を実装。1枚の画像の見える領域(コンテキストブロック)から、隠した複数の大きな領域(target blocks)の特徴を予測。ピクセル復元(MAE)より意味的な表現を効率よく学ぶことを狙い、計算効率と下流性能で良好な結果を出した。手作りのデータ拡張(色変換や切り抜き)に頼らない点も特徴。

- Bardes et al., "V-JEPA", Meta AI 2024(arXiv:2404.08471)。I-JEPA を動画に拡張し、時空間マスクで「動きを含む将来表現」を特徴空間で予測。動画は時間方向の予測が本質で、ピクセル復元の無駄が静止画よりさらに大きいことが動機。

- Assran et al., "V-JEPA 2", Meta AI 2025(arXiv:2506.09985)。100万時間規模の動画で事前学習し、さらに行動条件付き(action-conditioned)の予測器(V-JEPA 2-AC)を少量のロボットデータで追加学習。世界モデルとして「現在の状態表現と行動から次の状態表現を予測する」方向を強め、ロボット操作の計画にほぼゼロショットで使えることを示した。表現学習から行動計画への橋渡しを進めた。

これらに共通するのは「再構成(generative)ではなく予測(predictive)を特徴空間で行う」という設計哲学で、MAE 系・対比学習系と並ぶ自己教師ありの第三の柱として位置づけられています。

## 論文の実験結果(定量データ)

V-JEPA(2024)は動画理解のベンチマークで評価されました。代表的なのが Kinetics-400(K400、約24万本の動画で400種類の人間の行動を分類)や Something-Something v2(SSv2、「物を左に動かす」など時間順序が決定的に効く行動のデータセット)です。評価方式は frozen evaluation(エンコーダの重みを凍結したまま、その特徴の上に軽い分類器=attentive probe だけを学習して性能を測る方法。エンコーダを微調整しないので、表現そのものの良さを測れる)が中心に用いられました。

報告によれば、V-JEPA は SSv2 のような「時間的ダイナミクスが重要なタスク」で frozen 評価の精度が高く、報告された SSv2 の frozen 精度はおよそ70パーセント前後で、動画版 MAE などのピクセル再構成系や画像事前学習系を上回りました。SSv2 は「左から右か右から左か」を当てる必要があり、見た目(appearance)だけでは解けず動き(motion)の理解が要るため、時間予測を特徴空間で学んだ V-JEPA の強みが出やすいタスクです。これは「特徴予測が動きの意味を捉えている」ことの定量的な裏づけとされています。K400 でも競争力のある精度を、画像系の強力な事前学習に匹敵する水準で達成したと報告されています。

サンプル効率の面でも、V-JEPA は同等の下流性能に到達するのに必要な事前学習計算量・データ量が再構成系より少ないと報告されています。アブレーションでは、(1)マスクを連続した大ブロックにすること(歯抜けマスクだと性能が落ちる)、(2)損失を画素ではなく特徴空間で取ること(画素損失版に切り替えると下流精度が低下)、(3)EMA ターゲットによる崩壊回避、のそれぞれが性能に寄与することが示されました。とくにマスク戦略の効果は大きく、ランダムな小領域マスクでは近傍補間で解けてしまい意味的予測が育たないことが確認されています。

V-JEPA 2(2025)では、行動条件付きの世界モデル(V-JEPA 2-AC)を使い、ロボットアームのピック&プレースなど操作タスクで、追加のタスク固有学習をほとんど行わずに計画ができることが報告されました。具体的には、目標状態を画像で与え、行動候補を内部の予測器で展開して目標表現に最も近づく行動列を選ぶ(モデル予測制御的なプランニング)ことで、未知の物体・環境でも一定の成功率を達成したとされます。これは表現学習で得た「次状態の予測能力」が、そのまま行動計画に転用できることを示す結果で、自己教師あり表現と計画をつなぐ重要な実証です。

## メリット・トレードオフ・限界

メリット側。予測不可能・無意味な画素の細部を無視でき、意味のある構造だけを効率よく学べます。自己教師ありなのでラベルなしの大量データ(運転ログや動画アーカイブなど)から学べます。「将来の表現を予測する」枠組みは、計画(将来どうなるかに基づく判断)と本質的に相性が良く、世界モデルとして使えます。対比学習と違い大量の負例バッチを要しない点も実装上の利点です。

トレードオフと限界。表現の崩壊を防ぐ仕掛け(EMA ターゲット、stop-gradient)が必須で、EMA の係数や学習率を誤ると学習が崩壊します。出力が抽象特徴なので「学習がうまくいっているか」を直接目視できず、評価に下流タスク(凍結特徴の線形評価や microscopic な probe)が必要で、表現の良し悪しが間接的にしか見えません。予測対象がターゲットエンコーダ自身の出力であるため、「何を予測するか」がモデル内部で動く目標になり、理論的な解析(なぜ崩壊しないか、何を学んでいるか)が再構成系より難しいという課題もあります。さらに、凍結したエンコーダと予測器・下流タスクの間で特徴次元を合わせる射影層が実務上は必要になるなど、接続まわりの実装が地味に効いてきます。生成系と違い「予測結果を人間が見て確認する」ことができないため、安全クリティカルな用途では検証性の低さが論点になります。

## 発展トピック・研究の最前線
JEPA の枠組みは画像(I-JEPA)、動画(V-JEPA)、行動条件付き世界モデル(V-JEPA 2)へと拡張されてきました。最前線の論点は、(1)行動を条件に入れて真の「行動→次状態」予測器を学び、それを計画(MPC やサンプリングベースのプランニング)に直接使う方向、(2)複数モダリティ(画像・音・言語)を共通の埋め込み空間で予測する統合、(3)崩壊回避を EMA に頼らず理論的に保証する手法、(4)予測の不確実性(複数のありうる未来)を表現に持たせるマルチモーダル予測(現状の JEPA は決定的な点予測に近く、本質的に分岐する未来をひとつにまとめてしまう傾向がある)です。

自動運転や具現化 AI(embodied AI)の文脈では、「画素を再構成する世界モデル」と「特徴空間で予測する世界モデル」のどちらが計画に有利かが活発に議論されています。特徴予測派は効率と汎化を、画素再構成派は解釈可能性と直接的な検証可能性を強みとして主張します。将来状態を特徴空間で予測してそれを補助損失に組み込む設計は、まさにこの系譜の実践であり、予測器と本体ネットワークの出力次元整合や、予測損失を主タスク(模倣・計画)とどう重み付けするかが実装上の核心になります。予測損失が強すぎると主タスクを邪魔し、弱すぎると表現学習の恩恵が出ない、というバランスの調整が研究上の勘所です。
