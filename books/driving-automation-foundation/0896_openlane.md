# OpenLane / OpenLane-V2

## ひとことで言うと
OpenLane は、道路の車線を「画像上の 2D の線」ではなく「地面上の 3 次元の曲線」として正確にラベル付けした大規模データセットです。続編の OpenLane-V2 は、個々の車線に加えて「車線どうしのつながり方(どの車線からどの車線へ進めるか)」や「信号・標識といった交通要素が、どの車線にかかるか」という関係(トポロジー)まで扱う、より高度なベンチマーク(性能を測る共通の物差し)です。車線を点として見るのではなく、運転に使える「道路ネットワーク」として理解させることを目指します。

## 直感的な理解
人間が交差点で右折するとき、頭の中では「いま中央レーンにいる。右折するには右の 2 本目に移り、その先で右に分岐する車線へ入る。あの信号は自分の進む車線に対するものだ」という関係を一瞬で読み取っています。この「どの車線がどこへ続き、どの信号が自分にかかるか」という関係の理解が、運転判断の土台です。

ところが従来の車線検出の多くは、カメラ画像の上にペンで線を引くように「2D の線」を見つけるだけでした。問題は2つあります。第一に、車は 3 次元空間を走るのに、2D の線では「車線が地面のどこを通るか」「上り坂・カーブでどう曲がるか」が分かりません。坂やカーブでは画像上の見え方と実際の地面上の位置が大きくずれます。第二に、線が見えても「線と線のつながり」「線と信号の対応」が分からなければ、経路が引けません。OpenLane は第一の問題(3D 化)を、OpenLane-V2 は第二の問題(関係=トポロジーの理解)を解こうとしたベンチマークです。

なぜ「関係」がそこまで重要なのか。地図アプリで考えると分かりやすい。地図は単に道路の画像ではなく、「この交差点でこの道とこの道がつながっている」というグラフ(つながりの情報)を持つから経路探索ができます。自動運転も同じで、車線を見つけるだけでなく「走れる道のグラフ」を作れて初めて経路が引けます。OpenLane-V2 は、この「走れる道のグラフをカメラからその場で作る」課題を初めて大規模に定式化しました。

## 基礎: 前提となる概念
- 2D 車線検出 vs 3D 車線検出: 2D は画像の画素座標 (u, v) 上の線。3D は車両座標系のメートル単位 (x, y, z) 上の曲線。3D なら高さ(坂)も表せ、地図や経路計画にそのまま使えます。2D の線を「地面は平ら」と仮定して 3D に持ち上げると、坂やカーブで大きな誤差が出ます。
- BEV(Bird's-Eye-View、鳥瞰図): 上空から地面を見下ろした地図のような表現。車線は地面に貼り付いているため、BEV 上で扱うと 3D 位置が素直に表せます。カメラの斜め視点(perspective)を BEV へ変換するのが近年の車線・地図推定の定石です。
- IPM(Inverse Perspective Mapping、逆透視変換): カメラ画像を「地面は完全に平ら」と仮定して真上から見た図に変換する幾何手法。安価ですが、坂・段差・カーブで仮定が崩れて誤差が出ます。学習ベースの BEV 変換はこの限界を補います。
- ポリライン(polyline): 折れ線。車線や道路境界を、数十個の点を順につないだ線として表現します。曲線を点列で近似する素朴で扱いやすい形式です。
- 中心線(centerline): 各車線の真ん中を通る、進行方向を持つ有向の線。経路計画は車線境界より中心線を辿るほうが自然です。OpenLane-V2 はこの中心線を主役に据えます。
- トポロジー(topology): 要素どうしのつながり方。ここでは「車線→車線」(進める/進めない)と「車線↔交通要素」(信号がこの車線にかかる)の関係。位置が合っていても、つながりが間違っていれば経路は引けません。
- グラフ(graph): ノード(節点)とエッジ(辺)で関係を表す構造。車線・交通要素がノード、つながりがエッジになり、道路全体が 1 つのグラフになります。
- HD マップ(High-Definition Map): 車線レベルの精度を持つ高精細地図。従来は事前に作って車載しますが、オンライン HD マッピングは走行中にその場で作ります。事前地図の維持コストを下げられる利点があります。
- Transformer / 注意機構(attention): 入力のどの部分にどれだけ注目するかを学ぶ仕組み。車線推定では、画像特徴を BEV へ変換する際や、要素間の関係を推論する際に使われます。

## 仕組みを詳しく
OpenLane(初代、ECCV 2022):
- 母体は Waymo Open Dataset。その走行データに 3D 車線のラベルを新たに付与。
- 規模はおよそ 20 万フレーム、累計 88 万本以上の車線。大規模な 3D 車線専用データとして当時最大級。
- 各車線は 3 次元の点列(地面上を通る曲線を点でサンプリングしたもの)で表現。車線の種類(白実線・破線・黄線など 14 カテゴリ)も付く。シーン属性(交差点か直進かなど)も付与。
- これにより「カメラ画像から直接 3 次元空間中の車線曲線を予測する」課題(3D Lane Detection)を学習・評価できる。

3D 車線データの形:

```
2D 車線: lane = [(u1,v1), (u2,v2), ...]        画素座標
3D 車線: lane = [(x1,y1,z1), (x2,y2,z2), ...]  車両座標 [メートル]
1 フレーム = 車線ごとに点列を持つリスト + 各車線の種類ラベル
```

OpenLane-V2(NeurIPS 2023、別名 OpenLane-Topology):
- 母体は Argoverse2 と nuScenes。全周囲カメラ(7 台前後)のデータを使う。前方単眼から全周囲へ広げた点が初代との大きな違い。
- 扱う要素が 3 種類に拡張:
  1. 車線中心線(Centerline): 有向の線。Bezier 曲線や点列で表現。
  2. 交通要素(Traffic Element): 信号機・道路標識など、画像上の 2D ボックス + 属性(色・種別)。
  3. トポロジー関係: 「車線→車線」のつながりと、「車線↔交通要素」の対応。
- 出力はグラフ。ノードが車線・交通要素、エッジがつながり。すなわち「シーンの道路グラフを作る」課題で、オンライン HD マッピング(走りながら高精細地図を作ること)の統一ベンチマークとして位置づけられます。

評価指標が独特です。位置精度だけでなく、つながりの正しさまで測ります。
- 車線(centerline)の検出精度を DET_l、交通要素の検出精度を DET_t という mAP 系の指標で測る。車線検出はフレシェ距離(2 つの曲線がどれだけ似ているかを測る距離)に基づくマッチングで評価する点が画像検出と異なります。
- トポロジーの正しさを TOP スコア(車線間 TOP_ll、車線-要素間 TOP_lt)で測る。これは予測したグラフのエッジが正解とどれだけ一致するかを、グラフのエッジ予測の適合率・再現率として表します。
- これらを統合した OLS(OpenLane-V2 Score)で総合評価。OLS は各サブ指標の重み付き平均です。

この指標設計の意味は重要です。「線は見えたが、つなぎ方を間違えた」モデルは運転に使えません。検出精度だけ測るとそういうモデルを高く評価してしまいますが、TOP/OLS を入れることで「正しくつなげたか」まで罰せられます。指標が研究の方向(関係推論を解け)を定義している好例です。ベンチマーク設計とは「何を測るかを決めることで、コミュニティに何を解かせるかを定義する」営みでもあるのです。

## 手法の系譜と主要論文
- 3D-LaneNet(Garnett et al., ICCV 2019): カメラ画像から直接 3D 車線を回帰する先駆け。ネットワーク内で BEV へ投影して車線を予測する「dual-pathway」発想を導入。3D 車線という課題設定そのものを切り拓いた。
- Gen-LaneNet(Guo et al., ECCV 2020): 仮想 BEV(virtual top-view)表現と幾何的に整合する 2 段階設計で 3D-LaneNet を改良。坂での誤差を抑え、3D 車線研究の基盤を固めた。
- PersFormer / OpenLane(Chen et al., ECCV 2022): カメラの斜め視点の特徴を Transformer(Perspective Transformer)で BEV へ変換し、その BEV 上で 3D 車線を予測。2D と 3D の車線を統一的に扱える設計。同時に OpenLane ベンチマークを公開。BEV を使う理由は、車線が地面にあるため地面座標で扱うと 3D 位置を素直に表せるから。代償は BEV 変換の計算コストとカメラ校正(キャリブレーション)精度への依存。
- HDMapNet(Li et al., ICRA 2022): オンラインで HD マップを作る初期手法。BEV 上でセグメンテーションして後処理でベクトル化する。OpenLane-V2 のトポロジー推論の前段に位置づく。
- MapTR(Liao et al., ICLR 2023): 地図要素をポリラインの集合としてエンドツーエンドに予測。点の順序不変性(同じ車線を逆順の点列で表しても等価)を扱う「permutation-equivalent」モデリングで高速・高精度を実現し、オンライン地図推定の強力なベースラインに。後処理不要が利点。
- OpenLane-V2 / TopoNet(Wang et al., NeurIPS 2023 D&B): 車線検出を関係推論(トポロジー推論)へ拡張した初の大規模統一ベンチマーク、および付属ベースライン TopoNet(車線グラフをグラフニューラルネットで推論)。自動運転に必要なのは個々の線ではなく「走れる経路のネットワーク」だから。
- TopoMLP(Wu et al., ICLR 2024): トポロジー推論を、凝った構造ではなく検出を強くしたうえでシンプルな MLP で関係を予測する設計で高いスコアを達成。「良い検出器があれば関係推論は単純でも解ける」という知見を与えた。
- LaneGAP / その後の系譜: 車線をパスの集合として扱う表現や、地図要素を言語のように系列生成する手法など、表現方法の探索が続く。

系譜としては「2D 車線 → 3D 車線(BEV 化)→ 車線 + 関係(グラフ化)→ 統一 HD マッピング」と、扱う対象が線から構造へ、構造から運転に直結するネットワークへと深まってきた流れです。

## 論文の実験結果(定量データ)
OpenLane(PersFormer, ECCV 2022):
- ベンチマーク規模: 約 20 万フレーム、累計 88 万本超の車線、14 種の車線カテゴリ。3D 車線検出の評価は F-Score(適合率と再現率の調和平均、0〜100、高いほど良い)と、X/Z 方向の位置誤差(near 0〜40m と far 40〜100m 領域別の平均誤差、小さいほど良い)で測る。位置誤差を near/far に分けるのは、遠方ほど予測が難しく、遠方誤差が運転(高速での先読み)に効くためです。
- PersFormer は、当時の 2D ベースの手法を BEV 経由の 3D 予測に置き換えることで、3D 車線 F-Score を大きく改善し、特に遠方(far)の位置誤差を縮めたことが報告されています。アブレーションでは、Perspective Transformer による学習ベースの BEV 特徴変換を素朴な IPM(地面平面を仮定した幾何変換)に置き換えると、坂やカーブのある区間で誤差が増えることが示され、学習ベースの視点変換の有効性を裏付けます。IPM は「地面は平ら」と仮定するため坂で破綻するという限界を、注意機構による学習で補えるという知見です。さらに、2D と 3D を同時に学習するマルチタスク設計が単独学習より両者を底上げする傾向も示されました。

OpenLane-V2(NeurIPS 2023):
- 評価は OLS(総合)、車線・交通要素の mAP(DET_l, DET_t)、トポロジーの TOP_ll / TOP_lt。Argoverse2 由来と nuScenes 由来の 2 つのサブセット(subset_A / subset_B)で構成。
- ベースライン(TopoNet など)の OLS は、公開当初は相対的に低い水準(おおむね数十点台)で、特に TOP(つながり)スコアが検出スコアより大きく低い(TOP がしばしば 1 桁〜十数点に留まる一方で DET は数十点)ことが報告されました。これは「線は検出できるが、つながりを当てるのははるかに難しい」という課題の所在を定量的に示します。TOP が低いと、検出精度が高くても運転に使えるグラフにならないため、OLS が伸びません。
- アブレーション的知見として、(1) 車線-車線トポロジーと車線-要素トポロジーを同時に解く設計のほうが、別々に解くより整合的なグラフが得られる、(2) 交通要素の検出と車線検出を共有特徴で同時学習するほうが単独学習より相互に精度が上がる、(3) TopoMLP の知見では、検出性能を上げると下流のトポロジースコアも連動して上がる(関係推論のボトルネックは関係モジュールより検出の質にある場面が多い)、といった点が示されています。

これらの数値が示す本質は、「3D 化は BEV 学習でかなり解けてきたが、関係推論(つながり)は依然として未解決の難問」という研究の現在地です。TOP スコアの低さは、コミュニティに残された伸びしろを定量的に可視化したものと言えます。

## メリット・トレードオフ・限界
メリット:
- 3D 車線(地面上の実 3 次元座標)を大規模にラベル付けし、坂・カーブに強いデータ。88 万本超という規模は 3D 車線専用データとして突出。
- OpenLane-V2 は車線・交通要素・関係まで含み、運転判断に直結する道路グラフを扱える。
- 評価指標(TOP/OLS)がつながりの正しさまで測るため、「線は見えてもつなぎ方を間違える」モデルを正しく低評価できる。ベンチマークが研究方向を正しく規定している。
- Waymo / Argoverse2 / nuScenes という確立した母体の上に構築され、検出・予測・地図の他研究と接続しやすい。

トレードオフ・限界:
- グラフ(関係)予測は単なる検出より難しく、手法・評価ともに複雑。TOP スコアの伸び悩みが示すとおり未解決領域が大きい。
- 3D 車線ラベルはカメラ校正・センサー精度・地図整合に依存し、ラベル作成コストが高く、ラベル自体に誤差が混入しうる。
- 母体データセットのライセンス・地域の制約を引き継ぐ。地域多様性は母体に依存する。
- 研究上の未解決課題: 交差点のような複雑トポロジー、見通しの悪い遠方や遮蔽下での車線推定、地図の時間的一貫性(走行中に推定が揺れない、temporal consistency)、ロングテールな道路構造(環状交差点・特殊分岐)などが残る。

## 発展トピック・研究の最前線
- オンライン HD マッピングのベクトル化: MapTR 系のように、後処理なしでエンドツーエンドにポリラインを出す方向が主流化。時系列を使って複数フレームの観測を統合し、地図推定を安定化させる手法(StreamMapNet 系)も登場。
- トポロジー推論の改良: 車線グラフを言語モデル的に系列生成する手法、グラフニューラルネットで関係を明示的に推論する手法、検出を強化して関係を単純化する手法(TopoMLP)などが競合。
- エンドツーエンド運転への統合: 車線・トポロジー理解を知覚から計画まで一気通貫の運転モデルに組み込む流れ(UniAD のように知覚・予測・計画を 1 つのネットワークで結ぶ統合設計)。「どの車線へ進めるか」の理解は経路計画の前提です。
- BEV 表現の共通基盤化: 車線・占有・物体検出をすべて BEV 上で統合する設計が、タスク横断の標準になりつつあります。LiDAR を補助に使う多モーダル化([PandaSet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0898_pandaset) のような併載データが素材)も進行中です。
- 標準定義地図(SD Map)の事前知識利用: ナビ用の粗い地図を事前知識として与え、トポロジー推論を補助する研究。事前地図ゼロからの推定の難しさを、安価な粗地図で緩和する発想です。
