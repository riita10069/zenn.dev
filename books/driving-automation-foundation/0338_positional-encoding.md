# Positional Encoding

## ひとことで言うと
自己注意(Self-Attention)は系列を「順番のない集合」として扱ってしまい、何番目かが分かりません。そこで各時刻に「これは何番目」という位置情報を、ベクトルとして足し込んでやる仕組みが Positional Encoding(位置エンコーディング)です。これがないと Transformer は「過去と未来の順番」すら区別できません。正弦波を使う固定方式、データから学習する方式、2時刻の差を扱う相対方式、回転で埋め込む方式があり、選択がモデルの長文脈能力を左右します。

## 直感的な理解
トランプのカードを5枚、机に「順番に」並べたとします。あなたに「この5枚を見て次のカードを当てて」と言われたら、並んだ順序が決定的なヒントになります。ところが、もし5枚をシャッフルして渡されたら、順序の情報は失われ、ただの「5枚の集合」になってしまいます。

自己注意の素の状態は、まさにこの「シャッフルされた集合」を見ているのと同じです。各時刻が他の時刻と内積を取って関連度を測りますが、その計算には「何番目か」が一切入っていません。だから「10, 20, 30 という加速の系列」と「30, 20, 10 という減速の系列」を、注意機構は同じものとして扱ってしまいます。

Positional Encoding は、各カードに「これは1枚目」「これは2枚目」という番号札を貼る操作です。ただし整数の番号をそのまま貼るのではなく(整数を直接足すと値が際限なく大きくなり、距離100と101の区別が相対的に潰れます)、位置ごとに固有のパターンを持つベクトルを足し込みます。こうすることで、注意機構が内積を取るときに「位置の違い」も自然に効いてくるようになります。

## 基礎: 前提となる概念

理解の鍵は **permutation equivariance(順序同変性)** です。これは「入力の順序を入れ替えると、出力も同じように入れ替わるだけで、本質的な処理は変わらない」という性質です。

[Transformer 時間方向アテンション](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0344_transformer-temporal-attention) の自己注意は、各時刻のスコアを Query と Key の内積で計算します。この計算には「何番目の時刻か」という位置の情報がどこにも入っていません。そのため、入力時刻の順番を入れ替えると、出力も対応して入れ替わるだけで、各要素の中身の処理は変わりません。数式的には、置換行列 P に対して Attention(PX) = P · Attention(X) が成り立つ、ということです。

具体例で確かめます。系列「A, B, C」と、並べ替えた「C, B, A」を自己注意に入れると、A という要素が他の要素を見るときのスコアは位置に依存しないため、周囲の要素が同じなら A の出力は同じになります。つまり「A が1番目か3番目か」という情報がモデルに伝わりません。

これは時系列では致命的です。

- 速度の系列 [10, 20, 30] km/h(加速中)と [30, 20, 10] km/h(減速中)は、順序が逆なら全く違う運転状況なのに、位置情報がないと区別できない。
- 言語でも「犬が猫を追う」と「猫が犬を追う」は語順で意味が逆になる。

RNN は1ステップずつ順番に読むので位置情報が自然に入りますが、Transformer は並列に全部見るぶん、順序を別途与える必要があります。これを埋めるのが Positional Encoding です。なお畳み込み(CNN)も局所的な位置関係(カーネルの中での相対位置)を暗に持つため、位置を陽に足さなくてもある程度順序を扱えますが、自己注意にはその構造すらない、という違いがあります。

## 仕組みを詳しく

やることは単純で、入力の各時刻の特徴ベクトル(形状 `[L, d]`、L=系列長、d=次元)に、位置を表すベクトル PE(同じ形状 `[L, d]`)を足し算します。

```
入力に与えるもの = 単語/時刻の埋め込み X[pos]  +  PE[pos]
```

### 正弦波(sinusoidal)方式

Vaswani らの元論文は、学習しない固定の正弦波を使いました。位置 pos、次元 i に対して、

```
PE[pos, 2i]   = sin( pos / 10000^(2i/d) )
PE[pos, 2i+1] = cos( pos / 10000^(2i/d) )
```

噛み砕くと、各次元ごとに「波の周期」を変え、低い次元は短い周期(細かく上下)、高い次元は長い周期(ゆっくり上下)の正弦・余弦の値を入れます。これにより、

- 位置 pos ごとに固有のパターン(指紋のようなもの)ができる。
- 異なる周期を重ねるので、幅広い間隔を表現でき、近い位置と遠い位置が滑らかに変化する。
- 値は常に -1〜1 に収まり、どんなに長い系列でも発散しない。

具体例: d=4 とし、pos=0,1,2 の PE を書くと(概略)

```
pos=0: [ sin(0), cos(0), sin(0), cos(0) ] = [0, 1, 0, 1]
pos=1: [ sin(1), cos(1), sin(1/100), cos(1/100) ] ≈ [0.84, 0.54, 0.01, 1.00]
pos=2: [ sin(2), cos(2), sin(2/100), cos(2/100) ] ≈ [0.91, -0.42, 0.02, 1.00]
```

このように位置ごとに違うベクトルになり、足し込むことで「順序」が情報として埋め込まれます。

正弦波方式のうれしい性質は、ある位置 pos+k の PE が pos の PE の **線形変換(回転)** で表せることです。三角関数の加法定理(sin(a+b)、cos(a+b) を sin a, cos a, sin b, cos b で書ける公式)から導けるため、各周期の成分はその周期に応じた回転行列で pos から pos+k へ移れます。モデルが「k だけ離れている」という相対的な位置関係を線形写像1つで学びやすくなる、というのが設計意図です。また、学習データより長い系列にも **外挿(extrapolation、訓練で見ていない長さでも値を計算できる)** が可能です。10000 という底の数は、最も短い周期(2π)から最も長い周期(2π×10000)までの広い波長帯をカバーするための設計値です。

### 学習埋め込み(learned)方式

もう1つは、位置ごとに学習可能なベクトルを用意し、データから最適化する方式(BERT、GPT-2 などが採用)。位置 0, 1, 2, ... それぞれに対応するベクトルを学習パラメータとして持ちます。柔軟ですが、訓練で見た最大長を超えると対応するベクトルが存在せず使えない(外挿できない)弱点があります。

### 相対位置・回転(発展形)

後続研究では「絶対位置」ではなく「2時刻の差(相対位置)」を注意スコアに直接組み込む方式が広まりました。動機は「言語でも時系列でも、本質的なのは絶対的な番号より『どれだけ離れているか』だ」という観察です。代表が **RoPE(Rotary Positional Embedding、Su et al. 2021)** で、Query/Key を位置に応じて回転させることで相対位置を内積に自然に反映します。位置 m のベクトルを角度 mθ だけ回転させると、位置 m の Query と位置 n の Key の内積が、回転の差 (m−n)θ だけに依存する形になり、絶対位置が打ち消されて相対位置だけが残ります。Value には回転をかけないため加重和の中身は保たれます。長系列や時系列予測で広く使われています。

## 手法の系譜と主要論文

- **Vaswani et al., "Attention Is All You Need" (NeurIPS 2017, arXiv:1706.03762)**。正弦波方式を提案・採用。自己注意の順序同変性を補うのが動機です。
- **Gehring et al., "Convolutional Sequence to Sequence Learning" (ICML 2017, arXiv:1705.03122)**。Transformer とほぼ同時期の CNN ベース翻訳で、学習する絶対位置埋め込みを使いました。位置埋め込みという発想自体はこの前後で共有されていました。
- **Shaw, Uszkoreit, Vaswani, "Self-Attention with Relative Position Representations" (NAACL 2018, arXiv:1803.02155)**。絶対位置ではなく相対位置を注意の Key/Value に加える方式を提案。「言語や時系列では絶対位置より相対距離が本質」という考えに基づきます。
- **Dai et al., "Transformer-XL" (ACL 2019, arXiv:1901.02860)**。相対位置エンコーディングを使い、長い文脈をセグメントをまたいで保持する仕組み(recurrence + relative PE)を導入し、長距離依存の扱いを改善しました。
- **Su et al., "RoFormer: Enhanced Transformer with Rotary Position Embedding" (RoPE, 2021, arXiv:2104.09864)**。回転で相対位置を埋め込む方式。長系列の外挿性に優れ、現在の大規模言語モデル(LLaMA, GPT-NeoX, PaLM など)や時系列 Transformer の標準的選択肢の1つになりました。
- **Press, Smith, Lewis, "ALiBi: Train Short, Test Long" (ICLR 2022, arXiv:2108.12409)**。位置エンコーディングを足す代わりに、注意スコアに距離に比例したペナルティ(ヘッドごとに固定の傾き m × 距離)を直接加える方式で、訓練長を大きく超える外挿に強いと報告しました。
- 時系列 Transformer の文脈では、[Informer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0334_informer) が位置エンコーディングに加えて **タイムスタンプ(月・日・時・曜日など)を埋め込む temporal embedding(時刻埋め込み)** を導入し、周期性(1日周期、1週間周期など)を明示的にモデルへ与えています。

## 論文の実験結果(定量データ)

Vaswani et al. (2017) は、正弦波方式と学習埋め込み方式を機械翻訳(WMT 2014 英独)で比較しました。指標は BLEU(訳の n-gram 一致度、高いほど良い)です。

- 両方式の BLEU はほぼ同等(おおむね差は 0.1 程度以内)でした。にもかかわらず正弦波を採用した理由が重要で、「学習埋め込みは訓練最大長を超えると使えないが、正弦波は任意の長さに外挿できるため」と明記されています。つまり性能が同じなら汎用性の高いほうを選ぶ、という設計判断です。これは「位置エンコーディングの選択は性能だけでなく外挿能力で決まる」という分野共通の教訓になりました。

Shaw et al. (2018) は相対位置表現を導入し、英独翻訳で絶対位置の正弦波を上回る BLEU(報告ではおよそ 0.3〜1.3 ポイントの改善)を示しました。相対位置のほうが翻訳に効くという定量的根拠です。

RoPE(Su et al. 2021)は言語モデリングと長文タスク(中国語 LM、GLUE など)で評価され、収束が速く、訓練より長い系列でも性能劣化が緩やかであること(外挿性)を示しました。ALiBi(Press et al. 2022)はさらに踏み込み、長さ1024で訓練したモデルを長さ2048以上で評価しても、正弦波や学習埋め込みより **perplexity**(言語モデルの予測の不確かさを測る指標、低いほど良い)の悪化がはるかに小さいと報告しました。たとえば訓練長の数倍の系列でも perplexity がほとんど劣化しない例が示され、「相対位置・距離ベースのほうが外挿に強い」という流れを定量的に裏づけています。なお RoPE の外挿も無限ではなく、訓練長を大きく超えると劣化するため、周波数を調整する後処理(後述)が併用されます。

## メリット・トレードオフ・限界

- メリット: 自己注意に順序情報を与え、時系列・言語で順番を区別できるようにする(Transformer に必須)。
- メリット(正弦波): 学習不要・追加パラメータゼロで、訓練より長い系列にも外挿できる。
- メリット(相対/RoPE/ALiBi): 2時刻の距離を直接扱え、長距離・長系列に強い。RoPE は Q/K を回すだけで実装も軽い。
- トレードオフ: 足し算方式(絶対位置)は位置情報と中身の情報が同じベクトル空間で混ざるため、表現が互いに干渉する余地がある。
- トレードオフ(学習埋め込み): 訓練最大長を超えると使えない。
- トレードオフ: どの方式を選ぶかでモデルの外挿能力・周期表現力が変わるので、データの性質(周期性の有無、必要な系列長)に合わせた選択が要る。
- 限界・未解決の論点: 多変量時系列では「位置(時刻)」だけでなく「どの変数(チャネル)か」という別軸の情報も必要で、空間・チャネル方向の位置エンコーディングをどう設計するかは依然オープンな課題です。また RoPE や ALiBi の外挿も無限ではなく、極端に長い系列では劣化が報告されており、長文脈の位置表現は活発な研究領域です。不等間隔・欠損のある時系列では「pos = 整数の番号」という前提自体が崩れます。

## 発展トピック・研究の最前線

絶対位置から相対位置、そして回転(RoPE)や距離バイアス(ALiBi)へという流れが大きな潮流です。現在の大規模言語モデルはほぼ RoPE 系を採用し、さらに RoPE の周波数(回転角の底)を調整して長文脈に拡張する手法(Position Interpolation、NTK-aware スケーリング、YaRN など)が実用されています。これらは「訓練長 2k のモデルを 32k や 128k で使う」ための定番テクニックになりました。時系列分野では、時刻埋め込み(年・月・日・時・曜日というカレンダー情報の注入)に加え、欠損や不等間隔のサンプリングをどう位置として表現するか、観測時刻そのものを連続値として扱う continuous-time の位置表現(Neural ODE 系や時間カーネル)などが研究されています。位置エンコーディングは地味に見えて、Transformer の長文脈能力と外挿性を左右する中心的な要素であり続けています。
