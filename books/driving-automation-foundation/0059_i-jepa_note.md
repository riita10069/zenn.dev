# I-JEPA (Image JEPA)

## ひとことで言うと
I-JEPA は、1枚の画像の一部分(コンテキスト=手がかり)を見て、別の隠された部分の「特徴」を当てるように学習する画像版 JEPA です。ピクセルを一切復元しないのに強い表現が学べることを大規模に実証した、JEPA を世に知らしめた最初の主要モデルです。手作りの画像加工(色変換や反転といったデータ拡張)にも頼らず、特徴空間で予測するだけで意味のある表現が得られる、という点が画期的でした。2023年に Meta AI の Assran らが発表しました。

## 直感的な理解
ジグソーパズルを思い浮かべてください。完成図の一部だけを見せられて、「残りのピースにはだいたい何が描かれているか」を当てる遊びです。空の青が広がっている部分を見れば、隣も空だろうと推測できます。地面と空の境界が見えれば、その先に何があるかも見当がつきます。このとき私たちは、ピースの絵を1ドットずつ正確に描き出しているわけではなく、「ここは空」「ここは木」という意味のレベルで補完しています。

I-JEPA がやっているのはまさにこれです。画像の一部を手がかりに、隠れた領域の中身を「意味の特徴」として予測します。重要なのは、隠れた領域の正確なピクセル(葉の一枚一枚、影の微妙なグラデーション)を当てる必要がないことです。「ここは木の葉が茂っている」という特徴さえ合っていればよい。だから予測不能な細部に振り回されず、意味の理解に集中できます。理論的な母体は [JEPA (Joint-Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0064_jepa-architecture) と [LeCunの自律機械知能構想](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0069_lecun-jepa-vision) にあり、I-JEPA はそれを画像で「本当に役立つ表現が学べる」と実証した最初の一歩です。

## 基礎: 前提となる概念
このトピックを理解するには、いくつかの用語が必要です。

パッチ(patch)とは、画像を格子状に切り分けた小片です。たとえば 224×224 の画像を 16×16 ピクセルごとに切ると、14×14 = 196 枚のパッチになります。

Vision Transformer(ViT)は、画像パッチを言語の単語のように1つずつトークン(処理単位)として扱い、Transformer(系列全体の関係を一度に見るネットワーク)で処理する画像認識モデルです。各パッチを特徴ベクトルに変え、自己注意(self-attention、どのパッチがどのパッチに注目すべきかを動的に計算する仕組み)でパッチ同士の関係を計算します。

当時(2022〜2023年)の自己教師あり学習には2つの主流があり、I-JEPA はその両方の弱点を避けようとしました。

一つは不変性ベース(invariance-based)です。同じ画像を2通りに加工(切り抜き・色変換・反転など。これを data augmentation、データ拡張と呼ぶ。人間が手で決める画像のいじり方)し、その2つが似た特徴になるよう学習します(SimCLR, DINO, BYOL など)。問題は、どう加工するかという人間の事前知識(手作りのバイアス)に性能が強く依存し、特定のタスクに偏りやすいことです。たとえば色変換を強くかけて学習すると「色を無視する」特徴になり、色が手がかりになるタスク(花の種類の識別など)で弱くなります。

もう一つは生成的(generative)です。隠したピクセルを復元します(MAE など)。問題は、予測不能な細部にコストを払うため、線形プロービングのような「特徴の素の良さ」を測る評価では表現がやや弱くなりがちなことです。

I-JEPA は「画像加工に頼らず、かつピクセル復元もせず、特徴空間で予測するだけで良い表現が学べるか」を問い、できると示しました。

## 仕組みを詳しく
手順をステップで追います。

1. 画像を [224, 224] とし 16×16 パッチに分けると 14×14 = 196 個のパッチになる。
2. そのうち1つのコンテキストブロック(連続したパッチ群、画像の85〜100%程度のスケールから1つサンプリングし、後述のターゲット領域を取り除いたもの)を選び、コンテキストエンコーダ(ViT)に通して特徴を得る。これが手がかりになる。
3. 同じ画像から複数のターゲットブロック(隠して当てる対象。論文では概ね4個の長方形領域、各々画像の15〜20%程度のスケール)を選ぶ。各ターゲットブロックの「正解特徴」は、ターゲットエンコーダ(これも ViT)に画像全体を通してから、該当位置のパッチ特徴を取り出して作る。
4. 予測器(predictor、小さく細い ViT)が、コンテキストの特徴と「どの位置を当てるか」を示す位置トークン(マスクトークン、当てたい場所の座標情報を埋め込んだ学習可能なベクトル)を受け取り、各ターゲットブロックの特徴を予測する。
5. 損失は「予測した特徴」対「ターゲットエンコーダが出した正解特徴」の L1 ないし L2 距離(要素ごとの差の絶対値ないし二乗の和)。ピクセルは一切登場しない。

```
画像 [224,224,3]
   ├─ コンテキスト(一部パッチ) ─[context ViT]─▶ 特徴
   │                                              │
   │                                  ─[predictor + 位置トークン]─▶ 予測特徴 [N_target, D]
   │                                                                       │
   └─ 全体 ─[target ViT(EMAで更新, 勾配なし)]─▶ ターゲット特徴 [N_target, D]
                                                                           │
                                          loss = distance(予測特徴, ターゲット特徴)
```

ここで N_target は当てるパッチ数、D は特徴次元(ViT-Huge なら 1280、ViT-Large なら 1024)です。

崩壊(全パッチが同じ定数特徴になる失敗、[JEPA (Joint-Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0064_jepa-architecture) 参照)を防ぐ要が、ターゲットエンコーダの更新方法です。ターゲットエンコーダはコンテキストエンコーダの重みを EMA(指数移動平均、ゆっくり追従させる更新。係数は学習が進むにつれ 0.996 から 1.0 へ上げていくスケジュールが使われる)でコピーするだけで、ターゲット側には勾配を流しません(stop-gradient)。これにより「予測器がターゲットを定数へ引きずる」ことができず、崩壊を回避します。

設計上の核心的な工夫が、ターゲットブロックを「十分に大きく、空間的にまとまった領域」に選ぶことと、コンテキストブロックからターゲットと重なる領域を取り除くことです。もしターゲットが小さくバラバラなパッチなら、隣のパッチを見ればほぼ分かってしまい(局所的な補間で済んでしまい)、意味のある予測になりません。また、コンテキストにターゲット領域が含まれていると「答えを見ながら答える」ことになり学習になりません。大きく、まとまった、重ならないブロックを当てさせることで、初めてモデルは「シーン全体の意味」を使う必要に迫られます。これは MAE がランダムに細かく75%を隠すのとは対照的な、ブロック単位のマスキング戦略です。ここが I-JEPA の性能を最も左右する設計判断でした。

## 手法の系譜と主要論文
I-JEPA の原論文は、Mahmoud Assran, Quentin Duval, Ishan Misra, Piotr Bojanowski, Pascal Vincent, Michael Rabbat, Yann LeCun, Nicolas Ballas, "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023(arXiv:2301.08243)です。提案は「画像加工に頼らずパッチの特徴を特徴空間で予測する」こと。動機は、不変性ベースの手作りバイアスを減らしつつ、生成系のピクセル復元の無駄も避けたい、という両取りでした。

この前提として、LeCun の構想論文(2022, [LeCunの自律機械知能構想](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0069_lecun-jepa-vision))が JEPA という設計思想を提示し、I-JEPA はその画像での最初の実証という位置づけになります。崩壊回避の構造(EMA ターゲット + predictor + stop-gradient)は BYOL(Grill ら, 2020)由来で、不変性ベースの強敵が DINO(Caron ら, ICCV 2021、教師-生徒の蒸留で注目度マップが物体輪郭に対応するという顕著な性質を示した)、生成系の代表が He らの MAE(CVPR 2022)です。I-JEPA はこれらを直接の比較ベースラインに据えました。

後続で最も重要なのが Bardes らの V-JEPA(Video-JEPA, 2024)です。I-JEPA を動画に拡張し、「動画の一部の時空間ブロックから他の時空間ブロックの特徴を予測」します。同じ画像内の空間予測だった I-JEPA に対し、V-JEPA は時間方向の予測まで含めるので、世界の「動き」を表現します。さらに V-JEPA 2 / V-JEPA 2-AC(2025, [V-JEPA 2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0101_v-jepa-2))が規模を拡大し、行動条件付けでロボット制御まで到達しました。自動運転で「未来の状態を予測する」発想はこの V-JEPA 系の延長にあります。

## 論文の実験結果(定量データ)
評価の中心は線形プロービング(linear probing)です。事前学習したエンコーダの重みを凍結し、その出力特徴の上に1層の線形分類器だけを学習させて、ImageNet-1k(約128万枚・1000クラス)で分類精度を測ります。特徴に意味が詰まっていれば、単純な分類器でも高精度が出ます。

I-JEPA の報告によれば、ViT-Huge(約6.3億パラメータ)を ImageNet-1k で事前学習した場合、線形プロービングのトップ1精度(正解を1位に当てた割合)で概ね70%台後半に達し、生成系の MAE を上回りました。とくに重要なのは学習効率で、I-JEPA は ViT-Huge を 16 枚程度の A100 GPU で 72 時間ほどという、MAE や iBOT が同等精度に必要とする計算量に対しおよそ数分の1の GPU 時間で到達したと報告されています。これは「重いピクセル復元目標を捨てたぶん、収束が速い」という JEPA の主張を裏づけます。さらに、入力解像度を 448 ピクセルに上げた ViT-Huge では線形プロービング精度がさらに伸び、生成系・不変性ベースの強力なベースラインに肉薄ないし凌駕したと報告されています。

加えて、低ショット(各クラス少数枚だけで分類器を学ぶ設定、たとえば ImageNet の 1% ラベルのみ)でも生成系より頑健だったこと、物体カウント(Clevr/Count)や深度順序(Clevr/Dist)といった空間推論を要するタスクで不変性ベース手法(DINO など)より優位だったことが報告されています。これは「I-JEPA の特徴は単なるクラスの意味だけでなく、位置・数・前後関係といった空間的な構造も保持している」ことを示唆します。逆に、画像レベルの意味分類に最適化された不変性ベース手法は、こうした空間推論で取りこぼしがある、という対比が見えます。

アブレーションで決定的だったのがマスキング設計です。ターゲットブロックを小さくしたり数を減らしたりすると線形プロービング精度が大きく低下し、十分大きくまとまった領域を複数(おおむね4個)当てさせる設定で最良になったと報告されています。コンテキストとターゲットを重複させない処理を外すと、自明な補間ができてしまい性能が落ちました。また EMA 係数を速くしすぎると崩壊に近づき性能が落ちる、という観察もあり、崩壊回避の繊細さを定量的に示しています。予測器の深さ・幅についても、コンテキストエンコーダより細く浅い predictor が良いという非対称性が観察されています。

## メリット・トレードオフ・限界
メリットは、第一に人手の画像加工バイアスに頼らず汎用的な表現が学べること(色や向きを潰さないので空間推論に強い)。第二にピクセル復元をしないので学習が速く、ぼやけない意味的特徴になること。第三に EMA + stop-gradient という比較的単純な仕組みで崩壊を防げることです。

トレードオフと限界は、ターゲットブロックの選び方(大きさ・数・スケール)というハイパーパラメータに敏感で、設計の良し悪しが性能を大きく左右すること。損失が特徴距離なので学習中の良し悪しを直接見れず、下流評価が必要なこと。そして細粒度のピクセル情報が必要なタスク(高精細な領域分割、細部のテクスチャ識別)では、ピクセルを丁寧に学ぶ生成系に劣る場合があることです。さらに、ターゲットエンコーダが捨てた情報は当てられないため、表現の上限がターゲット側の質に縛られる、という JEPA 共通の限界も持ちます。fine-tuning(全層を下流タスクで再学習する評価)では、I-JEPA と MAE の差は線形プロービングほど大きくないことも知られており、「素の特徴の良さ」と「微調整後の到達点」は別物だ、という注意が必要です。

未解決の課題として、I-JEPA の特徴がどのタスクで不変性ベース(DINO 系)に勝ち、どこで負けるのかの体系的な理解がまだ発展途上であること、マスキング戦略を手作りでなく自動で最適化できないか、ターゲットエンコーダが捨てる情報を制御して下流タスクに必要な情報を残せないか、といった点が研究されています。

## 発展トピック・研究の最前線
I-JEPA の直接の発展が動画への拡張で、これは自動運転の「未来予測」と直結します([V-JEPA 2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0101_v-jepa-2))。また、崩壊回避を非対称性に頼らずに行う VICReg や SimSiam との比較は、JEPA がなぜ崩壊しないのかを理解する上で重要です([Stop-gradient (勾配停止)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0658_stop-gradient), [VICReg](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0661_vicreg_note), [SimSiam](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0093_simsiam))。応用面では、I-JEPA の凍結エンコーダをターゲットとして使い、別ドメイン(俯瞰特徴の未来予測など)で予測器だけを学習する、という転用が研究の流れとして見られます([行動条件付きワールドモデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0003_action-conditioned-world-model))。さらに、I-JEPA の系統に位置・スケール条件をより柔軟に与える発展や、点群・音声・マルチモーダルへの JEPA 化(画像以外のドメインで同じ枠組みを使う試み)が広がっており、「特徴空間予測」が表現学習の汎用的なレシピになりつつあるのが現状です。
