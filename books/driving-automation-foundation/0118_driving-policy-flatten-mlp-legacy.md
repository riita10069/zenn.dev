# flatten+MLP型の計画ヘッドとその構造的限界

## ひとことで言うと

特徴マップを1本の長いベクトルに潰し(flatten)、全結合ネット(MLP)に通すだけで軌跡や操作量を出す、最も素朴な計画ヘッドです。実装が簡単で小さい特徴マップなら動きますが、特徴マップを高解像度にするとパラメータ数が空間サイズの2乗で爆発し、現実的に学習も推論もできなくなります。「なぜ単純な方法ではダメで、attention型に移るのか」を理解するための、教科書的な反面教師です。この限界を知ることが、なぜ近年の計画ヘッドが query + attention 設計へ移行したのかを腑に落とす近道になります。

## 直感的な理解

ニューラルネットで「特徴から答えを出す」最後の一手として、一番に思いつくのが「全部つなげる」方法です。手元の特徴を全部1列に並べて、それを全結合層(入力すべてと出力すべてを線で結ぶ層)に放り込めば、原理上はどんな関数でも近似できます(万能近似定理)。実際、画像分類でも長らく最後はflatten→全結合でした。素朴で、実装が短く、最初に動かすには都合がいいのです。

ところが自動運転の計画ヘッドでは、入力が「車の周囲を表す2次元の地図(鳥瞰図特徴マップ)」です。これを1列に潰すと2つの問題が起きます。1つは、地図が細かくなるほど列が天文学的に長くなり、それを受ける全結合層の重みが爆発すること。もう1つは、潰した瞬間に「どのセルが隣同士か」という空間の構造が失われ、ネットが位置関係をゼロから覚え直さなければならなくなることです。

身近に喩えると、街の地図を理解するのに、地図を細かく切り刻んでバラバラのカードにし、順番だけ覚えてつなぎ直すようなものです。元の地図をそのまま見れば一目で分かる「この交差点はあの公園の隣」という関係を、毎回手探りで復元しなければなりません。元の2次元配置にあった豊かな手がかりが、flattenの瞬間に捨てられてしまうのです。

## 基礎: 前提となる概念

- 全結合層(fully connected / linear layer): 入力ベクトルのすべての要素と出力ベクトルのすべての要素を重みでつなぐ層です。入力N次元・出力M次元なら、重み行列はN×M個のパラメータを持ちます。N、Mが大きいとこの積が一気に巨大化します。
- flatten(平坦化): 多次元テンソル `[C, H, W]` をバッチ以外の軸でまとめて1本の長いベクトル `[C×H×W]` にする操作です。空間の隣接関係は保持されません。
- MLP(多層パーセプトロン): 全結合層を複数重ね、間に非線形活性化(GELU, ReLUなど。線形変換だけでは表せない複雑な関数を表現するために挟む)を入れた最も基本的なニューラルネットです。
- パラメータ数: 学習で更新される重みの総数です。多いほどメモリと計算を食い、過学習(訓練データに過剰適合し未知データで性能が出ないこと)もしやすくなります。
- 帰納バイアス(inductive bias): モデルの構造に組み込まれた「世界はこうなっているはず」という仮定です。畳み込みなら「近くの画素は関係が強い(局所性)」「同じ模様はどこにあっても同じ意味(並進不変性)」、attentionなら「関連の強い場所を動的に選ぶ」という仮定を持ちます。良い帰納バイアスはデータ効率(同じ精度に必要なデータ量)を上げます。
- 鳥瞰図特徴マップ: 車を真上から見た座標系の特徴です。形 `[B, C, H, W]`。詳細は[鳥瞰図特徴融合（BEVFormer風 空間クロスアテンション）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0299_bev-fusion)を参照してください。
- ウェイポイント / 制御量: 軌跡を構成する位置や、加速度・曲率などの操作量のことです。

## 仕組みを詳しく

典型的なflatten+MLP計画ヘッドのforwardは次の流れです。入力を鳥瞰図特徴 `fused_features: [B, 256, 8, 8]`、過去の視覚要約 `visual_history`、自車運動 `egomotion_history` とします。

1. チャンネル削減: `Conv2d(256, 3, ...)` で256チャンネルを3チャンネルに圧縮します。`[B, 256, 8, 8] → [B, 3, 8, 8]`。flatten後の次元を抑えるための前処理です。
2. flatten: バッチ以外を1本に潰します。`[B, 3, 8, 8] → [B, 192]`(3×8×8=192)。
3. 連結: 視覚ベクトル192 + 視覚履歴 + 自車運動 を結合し、例えば1344次元にします。
4. MLP: `fc1(1344→1344) → GELU → Dropout → fc2(1344→1164) → GELU → Dropout → fc3(1164→128)`。Dropoutは訓練中に一部のユニットをランダムに無効化して過学習を抑える手法です。
5. 出力: 軌跡 `[B, 128]`(64ステップ × 2信号 = 加速度と曲率、10Hzで6.4秒先まで)。履歴学習用の圧縮視覚ベクトルを副次的に出すこともあります。

8×8の小さなグリッドではこれで成立します。問題は解像度を上げたときです。

### なぜ高解像度で破綻するか(具体的な数値)

flatten後の次元数は `チャンネル数 × H × W` で決まります。

```
 8× 8, 3ch :   3 ×   8 ×   8 =        192 次元  → 問題なし
 8× 8, 256ch:  256 ×   8 ×   8 =     16,384 次元  → まだ何とか
450×300, 256ch:  256 × 450 × 300 = 34,560,000 次元  → 約3456万
```

最初の全結合層 `fc1` が `34,560,000 → 34,560,000` だとすると、重み行列は

```
34,560,000 × 34,560,000 ≈ 1.2 × 10^15 (約120京) パラメータ
```

になります。1パラメータをfp32(4バイト)で持つだけで約4.8エクサバイト。保存も計算も完全に不可能です。チャンネルを3に削っても `3 × 450 × 300 = 405,000` 次元あり、最初の層だけで数百億〜兆パラメータに達します。比較として、大規模言語モデルでも数百億〜数千億パラメータ規模であることを思えば、たった1層でその数百倍に達する異常さが分かります。

つまりflatten+MLPは「入力の空間サイズが大きくなるとパラメータが空間サイズの2乗で増える」という構造的欠陥を持ちます。畳み込みやattentionがパラメータ数を入力サイズと切り離している(重み共有・疎なアクセス)のと対照的です。

### 空間構造の破壊という第2の欠陥

数値爆発を仮に許容できたとしても、flattenはもう1つの問題を抱えます。`[H, W]` を1列にすると、隣接していたセルが列の上では遠く離れた要素になります。例えば同じ列にある上下のセルは、行幅W個分だけインデックスが離れます。全結合層はどの入力要素も平等につなぐので、「このセルとあのセルが空間的に隣」という情報を一切使えません。ネットはデータからその関係を毎回学び直す必要があり、データ効率が悪化します。

ASCII図で示すと、元の2次元配置:

```
A B C        flatten後の1列:
D E F   →    A B C D E F G H I
G H I        (AとDは縦に隣だが、列では3つ離れる)
```

全結合層から見ると、AとB(横の隣)も、AとD(縦の隣)も、AとI(対角の遠く)も、すべて区別のつかない「ただの別々の入力」です。畳み込みなら3×3カーネルが自動的にA,B,D,Eをまとめて見る(局所性の帰納バイアス)のに、flattenはその構造を全部捨ててしまいます。

### 解像度を変えるたびに作り直し

flatten後の次元が解像度で変わるため、MLPの入力次元も変わります。鳥瞰図を `8×8` から `100×100` に上げたら `fc1` を別物に作り直し、重みを最初から学習し直さねばなりません。解像度非依存ではない、という運用上の欠点です。attention型なら正規化座標(0〜1)でアクセスするため、解像度を変えてもヘッドはそのまま使えます。

これらの理由から、高解像度の鳥瞰図に移行する際([鳥瞰図の解像度と知覚範囲（前方バイアスの非対称pc_range）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0302_bev-resolution-asymmetric-pc-range))、flatten+MLP型は使えず、必要な場所だけ見る[軌跡計画ヘッド（ego query + 再帰デコード + deformable cross-attention）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0138_trajectory-planner)(deformable cross-attention型)に置き換えられるのが定石になっています。

## 手法の系譜と主要論文

flatten+MLP型のエンドツーエンド運転は、自動運転ディープラーニングの原点に連なります。

歴史をさかのぼると、ALVINN(Pomerleau, NeurIPS 1988)が、3層ニューラルネットに前方画像を入れてステアリングを出す先駆けでした。当時の計算機ではごく小さな画像しか扱えませんでしたが、「画像から操作を直接学ぶ」という発想自体はこの時点で存在しました。

NVIDIA PilotNet(Bojarski et al., 2016, "End to End Learning for Self-Driving Cars", arXiv:1604.07316)は、前方カメラ画像を5層の畳み込みで処理し、最後に全結合層を重ねてステアリング角を1つの値として直接回帰しました。それまでの自動運転は車線検出・経路計画・制御を人手で設計してつなぐのが主流でしたが、PilotNetは「画像から操作量まで一気に学習する」end-to-end学習を実証しました。動機は人手の特徴設計を排すことでした。効果は単一カメラでの車線維持の実走デモです。トレードオフは、出力が単純な操作量(ステア1値)に限られ、空間構造を保った推論や多数物体の関係理解、長期の軌跡計画が難しい点でした。flatten+MLP型の計画ヘッドはこの素朴な回帰の直系です。

これに対し、空間構造を保ったまま必要箇所を参照する設計が登場します。Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)は attention を少数の参照点に絞り、高解像度でも計算量を抑えました。BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)は学習可能なクエリが幾何で当たりを付けた場所だけに attention して鳥瞰図を作り、UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)はその上に query+attention の計画ヘッドを載せました。これらは「解像度を上げても破綻しない」という、まさにflatten+MLPの弱点を克服する設計です。計画ヘッドがflatten+MLPからattention型に移った流れは、この系譜に沿っています。

## 論文の実験結果(定量データ)

PilotNet(2016)は定量ベンチマークというより実走デモが中心で、報告では人間の介入なしに自律走行できた時間の割合(autonomy)を指標とし、限定的なルートで98%程度の自律率を達成したとされます。指標の意味は「全走行時間のうち人が手を出さずに済んだ割合」で、高いほど良いです。autonomy = (1 - 介入回数 × 介入1回あたりの復帰時間 / 総走行時間) のように定義されました。ただし対称的な多物体シーンや複雑な交差点での評価ではなく、車線維持という限定タスクでの数値である点に注意が必要です。

flatten+MLP型の限界を逆照射するのが、後続のattention型の実験です。前述の通りUniAD(2023)はnuScenesオープンループ計画で平均L2誤差約1.03m・平均衝突率約0.31%を達成し、空間構造を保つ設計が高解像度・多物体シーンで効くことを示しました。Deformable DETR(2021)はCOCO検出で標準DETRの約10分の1の学習エポックで同等以上のAPを出し、密な全結合的処理を疎な参照に置き換える効果を定量化しました。これらは「flattenで構造を捨て全結合で総当たりする」方式が、表現力でも効率でも劣ることの間接的な証拠です。

直接的なアブレーションとしては、計画ヘッドをflatten+MLPに差し替えると、(1)高解像度では学習自体が回らない(パラメータ爆発でメモリ不足になる)、(2)低解像度に落とせば回るが、グリッドが粗すぎて物体を区別できず計画精度が頭打ちになる、という二者択一に陥ることが知られています。例えば鳥瞰図を7×7や8×8まで落とせばflatten+MLPでも動きますが、その解像度では1セルが実空間で数メートルに相当し([鳥瞰図の解像度と知覚範囲（前方バイアスの非対称pc_range）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0302_bev-resolution-asymmetric-pc-range)参照)、車1台が1セルに潰れて近接物体を区別できません。この二者択一がattention型への移行を不可避にしています。

## メリット・トレードオフ・限界

メリット(小規模・初期実装で選ばれる理由):
- 実装が極めて単純です。最小の動くend-to-endパイプラインを最短で立ち上げられ、配線が正しいか(損失が下がるか)の検証に向きます。
- 特徴マップが小さい(7×7、8×8程度)うちは十分機能し、デバッグも容易です。
- 全結合は理論上どんな関数も近似でき、構造的制約が少ないです。問題設定が小さければこの自由度が利点になります。

トレードオフ・限界(高解像度で廃止される理由):
- パラメータ数が空間サイズの2乗で増え、高解像度化で次元爆発します(450×300で約120京パラメータ)。
- flattenで空間構造が壊れ、位置関係をゼロから学ぶためデータ効率が低いです。
- 解像度を変えると入力次元が変わり、ヘッドを作り直して再学習が必要です(解像度非依存でない)。
- 単一ベクトルへの回帰なので、多峰性(複数の妥当な軌跡)を表現できません。

研究上の含意として、この反面教師は「計画ヘッドに必要なのは表現力の高い汎用近似器ではなく、空間構造を尊重し疎にアクセスする帰納バイアスだ」という教訓を与えます。万能近似定理が「原理上できる」ことを保証しても、現実的なデータ量・計算量で学習できるかは帰納バイアスが決めるのです。

## 発展トピック・研究の最前線

- 帰納バイアスの設計: 畳み込み(局所性)、attention(動的な参照)、グラフ(関係性)など、どの構造的仮定を入れるかが性能とデータ効率を左右します。タスクに合った帰納バイアスを選ぶことが、闇雲にパラメータを増やすより重要です。
- トークン化された計画: 鳥瞰図をパッチ/トークンに分け、Transformerで処理する設計です。flattenの欠点(構造破壊と次元爆発)を避けつつ系列モデルの強みを使います。
- 効率的attention: 計画ヘッドでも線形attentionやdeformable attentionで計算量を抑える研究が続いています。
- 単純さの再評価: 一方でAD-MLP(Zhai et al., 2023, arXiv:2305.10430)のように「自車運動だけのMLP」が一部ベンチマーク(nuScenesオープンループ)で強いという指摘もあり、何が本当に効いているかの見極めが研究テーマになっています。これは「画像を見ないMLPが強い」のではなく「ベンチマークが慣性で当てられる偏りを持つ」ことを示すもので、flatten+MLPの復権ではない点に注意が必要です。
