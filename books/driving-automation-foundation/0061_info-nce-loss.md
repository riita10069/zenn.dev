# InfoNCE / NT-Xent ロス

## ひとことで言うと
「同じものから来たペア(正例)」を近づけ、「無関係なもの(負例)」を遠ざけるように特徴を学ぶための損失関数(loss、モデルの間違い具合を数値化して小さくしていく目標)です。仕組みは「たくさんの候補の中から正解のペアを選ぶクイズ」に近く、正解を選べる確率を上げるよう学習します。温度(temperature)という1つのつまみが効き目を大きく左右し、対照学習で最も重要なハイパーパラメータの一つです。

## 直感的な理解
似た画像を近くに、違う画像を遠くに並べたい、と考えます。しかし「近くに」だけを目標にすると、ずるい解が生まれます。全部の画像を同じ1点に置けば、どんなペアも「近い」が達成できてしまう。これは崩壊(collapse)と呼ばれ、何も区別できない無意味な状態です。

防ぐには「離す圧力」が要ります。InfoNCE / NT-Xent の賢いところは、「正例を近づける」と「負例から離す」を別々の損失にせず、1本の式に綺麗にまとめた点です。形は分類問題そのもの。「目の前の特徴に対し、たくさんの候補のうちどれが正例(同じ画像の別ビュー)か当てよ」という多肢選択クイズにして、正解を選べる確率を最大化します。正解を選ぶには、正例と高い類似度を持ち、かつ紛らわしい負例とは低い類似度を持つ必要がある。こうして近づける力と離す力が同時に働きます。

このクイズ形式が効く理由をもう少し。仮に「正例の類似度を直接最大化する(回帰)」だと、いくらでも大きくできてしまい、相対的な比較になりません。クイズ形式(softmax)は「正例が、すべての候補の中で相対的に一番らしい」ことを求めるので、自動的に正規化がかかり、暴走しにくいのです。

## 基礎: 前提となる概念
コサイン類似度: 2つのベクトルの「向きの近さ」を測る量です。両ベクトルを L2 正規化(長さを1に揃える)してから内積をとると、値は -1(正反対)〜 1(完全に同じ向き)に収まります。対照学習では特徴を正規化してこれを類似度に使うのが標準です。正規化することで「ベクトルの長さ」という学習を不安定にしがちな自由度を消し、純粋に「向き」だけで似ている度合いを測れます。

ソフトマックスと交差エントロピー: ソフトマックスは複数のスコアを「合計1の確率分布」に変換する関数です。交差エントロピー(cross entropy)は、正解の分布とモデルの予測分布の食い違いを測る損失で、正解クラスに高い確率を割り当てるほど小さくなります。InfoNCE はこの交差エントロピーを、正例を正解クラスとみなして使っているだけ、と理解できます。

正例と負例: 1枚の画像にデータ拡張(切り抜き・色変化など)を2回かけて作った2ビューが「正例ペア」。同じバッチ内の他の画像から来たビューはすべて「負例」として扱います。

相互情報量(mutual information): 2つの確率変数がどれだけ情報を共有しているかを測る量です。「画像のあるパッチ」と「別のパッチ」が同じ物体の一部なら、両者は情報を共有しています。InfoNCE はこの相互情報量の下界(lower bound、これより下がらない保証値)を最大化していると解釈でき、ここに理論的な裏付けがあります。具体的には、負例を N 個使うと I(x;c) ≥ log N - L_InfoNCE という関係があり、負例が多いほど下界が締まります。

NCE(Noise-Contrastive Estimation): Gutmann & Hyvärinen(2010)の古典手法で、「正しいデータと、わざと混ぜたノイズを識別する分類器を学習することで、確率モデルを推定する」枠組みです。InfoNCE はこの「ノイズ(=負例)と対比して学ぶ」発想の系譜にあり、名前もここから来ています。

## 仕組みを詳しく
設定: バッチに N 枚の画像があり、各画像から2つの拡張ビューを作るので合計 2N 個の特徴ベクトルができます。各ベクトルは L2 正規化済みとします。

ある特徴 z_i に対して、その正例は「同じ画像から来たもう片方のビュー z_j」です。残りの 2N-2 個はすべて負例として扱います。

NT-Xent 損失(z_i から見た形):

```
sim(u, v) = u·v / τ            # 正規化済みの内積(コサイン類似度)を温度 τ で割る
L_i = - log(  exp(sim(z_i, z_j)) / Σ_{k≠i} exp(sim(z_i, z_k))  )
```

分子は「正例ペアの類似度」、分母は「自分以外すべて(正例1つ+負例たち)との類似度の合計」です。これはソフトマックス分類の交差エントロピーで、「2N-1 個の候補の中で、正例 z_j を正解として選ぶ確率」を最大化しています。バッチ全体ではこれを全ペアで平均します。

数値例: τ=0.1、正例の内積(正規化前のコサイン類似度)が 0.9、負例たちの内積がどれも 0.2 のとき。τ で割ると sim はそれぞれ 9.0 と 2.0。exp(9.0)≈8103、exp(2.0)≈7.39。負例が 100 個あれば分母 ≈ 8103 + 100×7.39 = 8842。正例を選ぶ確率 ≈ 8103/8842 ≈ 0.916。L = -log(0.916) ≈ 0.088。正例がしっかり選べていて損失が小さい状態です。逆に正例の類似度が低ければ確率が下がり、損失が大きくなって学習が進みます。

温度 τ(タウ)の役割。τ は類似度を割るスケールで、小さいほど類似度の「差」を強調します。

- τ が小さい(例 0.05): 少しの類似度差が大きな確率差になり、最も紛らわしい負例(hard negative、正例とよく似た負例)を強く突き放す。学習は鋭くなるが、過敏で不安定になりやすく、意味的に近い「実は仲間」のサンプルまで無理に引き離す副作用(class collision の悪化)もある。
- τ が大きい(例 0.5): 差がなだらかになり、すべての負例を穏やかに均等に扱う。崩壊しにくく安定するが、ハードな負例への集中が弱まり、区別の鋭さが鈍る。

勾配のレベルで見ると一段はっきりします。z_i に関する勾配は、各負例 z_k に対し「その負例を正例と誤認している確率」に比例した重みで効きます。τ が小さいほどこの重みがハード負例(類似度の高い負例)に集中し、易しい負例はほぼ無視されます。Wang & Liu(CVPR 2021)は、対照損失が本質的に「ハードネス aware(紛らわしい負例ほど強く罰する)」な性質を持ち、温度 τ がそのハードネスへの集中度を制御することを定量的に示しました。τ を下げるほど勾配がハード負例に集中する一方、極端に下げると「意味的に近い真の仲間」まで強く突き放す uniformity-tolerance のトレードオフが悪化します。

理論的な見方として、Wang & Isola(ICML 2020)は InfoNCE が「alignment(正例同士を近づける)」と「uniformity(特徴を超球面上に一様に散らす)」という2つの性質を同時に最適化していると分解しました。alignment は正例ペアの距離を小さく、uniformity は全特徴が球面上に均等に散らばる(=情報を最大限保つ)ことを促します。uniformity が崩壊を防ぐ役割を担い、温度はこの2つのバランスにも効きます。彼らは alignment と uniformity を直接損失として書き下しても、InfoNCE と同等の性能が出ることを実験的に確認しました。

before/after: 温度なし(τ=1)だと類似度差が弱く効きづらく学習が遅い。温度導入(after)で、ハードな負例への重み付けが調整可能になり性能が安定・向上します。

## 手法の系譜と主要論文
InfoNCE(Oord, Li, Vinyals, 2018, "Representation Learning with Contrastive Predictive Coding (CPC)", arXiv:1807.03748, DeepMind)。系列データ(音声・テキスト・画像のパッチ列)で「未来の表現を予測する」文脈に対照損失を導入しました。新規性は、生成的に細部まで予測する代わりに「本物の未来(正例)と無関係なサンプル(負例)を識別するクイズ」に置き換えた点。動機は、生成は難しく無駄な細部まで再現させられるが、識別なら学習しやすいこと。InfoNCE という名は NCE(Noise-Contrastive Estimation)に由来し、相互情報量の下界を最大化する解釈を持ちます。

NT-Xent(Chen, Kornblith, Norouzi, Hinton, ICML 2020, "A Simple Framework for Contrastive Learning of Visual Representations (SimCLR)", arXiv:2002.05709, Google Brain)。同じ式に "Normalized Temperature-scaled Cross Entropy" の名を付けました。SimCLR の貢献は、複雑な仕掛けを足すのではなく、(1) 強いデータ拡張(特に crop と color distortion の組み合わせ)、(2) L2 正規化、(3) 温度スケーリング、(4) 大きなバッチ(=大量の負例)、(5) 投影ヘッド(projection head、特徴の後段に置く MLP)を丁寧に設計するだけで強くなると示したこと。SimCLR v2 ではさらに大きなモデルと蒸留を組み合わせ、半教師あり設定で大きく伸ばしました。

MoCo(He, Fan, Wu, Xie, Girshick, CVPR 2020)は同じ InfoNCE を使いつつ、負例を「メモリバンク(キュー)+モメンタムエンコーダ」で供給し、大バッチを不要にしました(→ [momentum encoder / EMA target](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0078_momentum-encoder-ema))。対比として BYOL / SimSiam(→ [predictor (予測ヘッド非対称性)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0083_prediction-head-predictor))は「負例を一切使わない」設計で、InfoNCE を捨てても崩壊を避けられることを示しました。SwAV(→ [SwAV (Swapping Assignments between Views)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0095_swav_note))は特徴同士の直接対照をやめ、クラスタ割当の予測に置き換えました。さらに、画像とテキストをまたぐ対照学習として CLIP(Radford et al., 2021)が InfoNCE をマルチモーダルに拡張し、ゼロショット分類の基盤となったのも、この損失の系譜の重要な分岐です。

## 論文の実験結果(定量データ)
評価指標: SSL では「線形評価(linear probing)」が標準です。事前学習したエンコーダを凍結し、その上に線形分類器だけを ImageNet ラベルで学習して Top-1 精度を測ります。エンコーダを微調整しないので「学んだ特徴の素の質」を測れます。

SimCLR(ResNet-50、ImageNet 線形評価、報告による)。Top-1 約 69.3%。前世代手法より大きく上回り、SSL が教師あり(ResNet-50 で約 76.5%)に近づける可能性を示しました。SimCLR が示した重要なアブレーション:

- バッチサイズ依存: バッチを 256 から 4096〜8192 へ増やすと精度が数ポイント単位で向上。InfoNCE は負例数(=バッチ内の他サンプル数)が多いほど効くため、大バッチが有利という性質が露わになりました。逆に小バッチでは負例不足で性能が出にくい。学習エポックを伸ばすほど大バッチの優位は縮むが、消えはしないと報告。
- 投影ヘッドの効果: 損失を「投影ヘッド後の特徴」で計算し、下流評価は「投影ヘッド前の特徴」で行うと、ヘッドを使わない場合より約 10 ポイント以上 Top-1 が向上したと報告。投影ヘッドが「対照タスク特有の情報(色や切り抜き位置を当てる手がかり)」を吸収し、エンコーダ側により一般的な特徴を残す効果があると解釈されます。
- データ拡張の重要性: crop と color distortion の組み合わせが特に効き、片方だけだと大きく性能が落ちると報告。色を変えないと「同じ色味」という手抜きの手がかりで正例を当ててしまい、表現が貧弱になります。色ヒストグラムというショートカットを潰すのが color distortion の役割です。
- 温度感度: τ を 0.1〜0.5 で検証し、最適値(おおむね 0.1〜0.2)から外れると Top-1 が数ポイント変動。温度が最も効くつまみの一つと確認されました。

学習時間とデータ量のスケーリングでは、長く学習し大きなモデルにするほど SSL のほうが教師あり学習よりラベル効率の利得が伸びる傾向も示されました。ImageNet ラベルの 1% だけで fine-tune した半教師あり設定でも、SimCLR v2 は大幅な改善を報告しています。CPC(InfoNCE 原典)側でも、音声話者識別・音素認識、画像、テキストで汎用に効くことが報告され、負例数を増やすほど相互情報量下界が締まり性能が上がる関係が確認されています。

## メリット・トレードオフ・限界
メリット
- 「正例を近づけ負例から離す」を1本の式で実現し、uniformity 項が働くため崩壊を避けやすい。
- 相互情報量の下界最大化、alignment/uniformity という理論的解釈がある。
- 温度 τ という1つのつまみで、ハードな負例への集中度を調整できる。
- 実装が素直で、分類の交差エントロピーとほぼ同じコードで書ける。
- マルチモーダル(CLIP)まで素直に一般化でき、汎用性が高い。

トレードオフと限界
- 多数の負例が必要。SimCLR は大バッチ(数千)、MoCo はメモリバンクで対処するが、いずれも計算・メモリのコストがかかる。
- 温度 τ に敏感で、調整を誤ると性能が大きく落ちる。
- 負例の質に左右される。無関係すぎる負例は学習信号が弱く、似すぎる負例(実は同じ意味のクラス)は class collision を起こし誤って引き離す。
- false negative 問題: ラベルがないので「たまたま同じ犬の別写真」を負例にしてしまい、本来近づけるべきものを遠ざける。これが教師ありとの性能差の一因とされる。
- 相互情報量下界の解釈には注意も。下界がタイトでない(緩い)場合があり、「相互情報量を上げているから良い」より「適切な inductive bias を入れているから良い」という説明のほうが妥当という議論(Tschannen et al., 2020)もある。
- 単画像ベースが基本で、時系列・行動・物理の情報は別途設計が必要。

## 発展トピック・研究の最前線
false negative や class collision を緩和する方向として、Supervised Contrastive Learning(Khosla et al., 2020、ラベルがある場合に同クラスを全部正例にする)、debiased contrastive learning(Chuang et al., 2020、負例に紛れた正例の影響を統計的に補正)、hard negative mining(Robinson et al., 2021、紛らわしい負例を重点サンプリング)などが提案されています。負例を完全になくす方向が BYOL/SimSiam(→ [predictor (予測ヘッド非対称性)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0083_prediction-head-predictor))とクラスタ型 SwAV(→ [SwAV (Swapping Assignments between Views)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0095_swav_note))で、これらは「負例なしでも崩壊しない」ことを示し対照学習の前提を問い直しました。動画・運転領域では、空間的な正例ペアより時間方向の予測(現在から未来の特徴を当てる → [行動条件付き潜在予測 (counterfactual rollout)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0001_action-conditioned-prediction))を対照損失に組み込む研究もあり、InfoNCE の「正解の未来を当てる」という CPC 由来の発想がここで再び生きてきます。点群・LiDAR や軌跡予測でも、対比すべき正負ペアの定義をタスクに合わせて設計する研究が続いています。
