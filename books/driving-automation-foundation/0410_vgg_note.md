# VGG

## ひとことで言うと
VGG は、3×3 という非常に小さな畳み込みフィルタだけを大量に積み重ねた、構造が極めてシンプルで均質な CNN (畳み込みニューラルネットワーク) です。2014年に登場し、「フィルタを小さくして層を深くするほど精度が上がる」という設計原則を明快に示しました。今では画像分類の性能では新しいモデルに譲りますが、その素直で安定した特徴の出方から、知覚損失 (perceptual loss、人間の見た目に近い形で画像の似ている度合いを測る損失) の特徴抽出器として現在も現役で使われています。

## 直感的な理解
CNN の設計を「視力検査の道具をどう組むか」に例えてみます。一度に広い範囲を見る大きなレンズ (大きなフィルタ) を1枚使うか、小さなレンズ (小さなフィルタ) を何枚も重ねて同じ範囲をカバーするか、という選択があります。直感的には大きなレンズ1枚で済ませたくなりますが、VGG は「小さなレンズを何枚も重ねた方が、同じ視野でもパラメータが少なく、しかも途中で何度も処理を挟めるので賢くなる」ことに気づきました。

具体的には、7×7 の大きなフィルタ1枚と、3×3 の小さなフィルタ3枚を重ねたものは、画像のほぼ同じ広さ (受容野、一度に見える範囲) を見ます。ところがフィルタの重みの数は、7×7=49個に対し、3×3を3枚で27個 (=3×3×3) と少なくて済みます。しかも3枚重ねれば間に活性化関数 (非線形な変換) が3回入り、より複雑な模様を表現できます。VGG はこの「小さいフィルタを深く重ねる方が効率的で強い」という原則を、実験で徹底的に検証したモデルです。

## 基礎: 前提となる概念
畳み込み (convolution) は、小さなフィルタ (重みの並んだ小窓) を画像の上で少しずつ滑らせ、各位置でフィルタと画像の重なり部分を掛けて足す操作です。これにより「縦のエッジ」「特定の色の塊」といった局所的な模様を検出します。同じフィルタを画像全体で使い回すので、物体が画像のどこにあっても同じ模様を捉えられます (並進不変性)。

受容野 (receptive field) は、出力の1点が入力画像のどれだけ広い範囲を見ているかを表します。畳み込みを重ねるほど受容野は広がり、ネットワークの深い層ほど大きな物体や大域的な構造を捉えられます。

プーリング (pooling)、特にマックスプーリングは、小さな窓 (例えば2×2) の中で最大値だけを残して残りを捨て、特徴マップの縦横を半分に縮める操作です。情報を要約しつつ計算量を減らし、小さな位置ずれに強くする働きがあります。

活性化関数、ここでは ReLU (入力が負ならゼロ、正ならそのまま通す関数) は、各層に非線形性を与えます。これがないと何層重ねても1つの線形変換に潰れてしまい、複雑な模様を表せません。

## 仕組みを詳しく
VGG-16 (重みを持つ層が16層の版) の処理を数値例で追います。入力は 224×224×3。

1. 3×3 畳み込み (チャンネル64) を2回 → 224×224×64。続いて2×2 マックスプーリングで 112×112×64。
2. 3×3 畳み込み (128) を2回 → プーリングで 56×56×128。
3. 3×3 畳み込み (256) を3回 → プーリングで 28×28×256。
4. 3×3 畳み込み (512) を3回 → プーリングで 14×14×512。
5. 3×3 畳み込み (512) を3回 → プーリングで 7×7×512。
6. これを平らに並べ (7×7×512 = 25088個)、全結合層 (4096 → 4096 → 1000) を通してクラス確率を出力。

```
224x224x3
  3x3 conv x2 → pool → 112x112x64
  3x3 conv x2 → pool →  56x56x128
  3x3 conv x3 → pool →  28x28x256
  3x3 conv x3 → pool →  14x14x512
  3x3 conv x3 → pool →   7x7x512
  flatten → FC 4096 → FC 4096 → FC 1000 → クラス確率
```

設計の特徴は徹底した均質性です。畳み込みは全部3×3、ストライド (滑らせる歩幅) は1、パディング (周囲の縁取り) で解像度を保ち、プーリングは全部2×2。解像度を半分にするたびにチャンネル数を倍にする、というルールを最後まで一貫させています。なぜチャンネルを倍にするかというと、解像度が下がって空間方向の情報が減る分を、より多くの種類の特徴 (チャンネル) で補い、各層の計算量をおおよそ一定に保つためです。

「小さいフィルタを深く積む方が得」という主張を数値で確かめておきます。3×3 を2枚重ねると受容野は 5×5 相当、3枚重ねると 7×7 相当になります (1段ごとに受容野は両側に1ずつ広がるため)。同じ入出力チャンネル C で受容野 7×7 を実現する場合、7×7 フィルタ1枚のパラメータは 7×7×C×C = 49C^2。一方 3×3 を3枚なら 3×(3×3×C×C) = 27C^2 で、約45%少なくて済みます。加えて3枚の間に ReLU が2回入るので、1枚では作れない非線形な (折れ曲がった) 決定境界を表現できます。この「同じ受容野・少ないパラメータ・高い非線形性」という三拍子こそ VGG の核心の主張です。

VGG には層数の異なる版があり、VGG-11, VGG-13, VGG-16, VGG-19 が代表です。VGG-16 で約1億3800万パラメータ、VGG-19 で約1億4400万パラメータと、当時としても非常に重く、その大半は最初の全結合層 (7×7×512×4096 ≈ 1億) が占めます。この巨大な全結合層が VGG の最大の弱点でした。

### なぜ今も知覚損失で使われるか
画像生成・超解像 (低解像度画像を高解像度化する技術)・スタイル変換では、生成画像と正解画像の「似ている度合い」を測る必要があります。ピクセルの差を直接比べる (二乗誤差、L2) と、生成画像が1ピクセルずれただけで大きな誤差になり、人間の見た目とずれた評価になります。結果として生成画像がぼやけがちになります。

そこで、学習済み VGG に生成画像と正解画像の両方を通し、中間層が出す特徴マップ同士の差を損失にします。VGG の中間特徴は「エッジ」「テクスチャ」「物体らしさ」といった人間の知覚に近い情報を素直に階層的に表すため、ここで近ければ人間にも似て見える、という発想です。これが知覚損失 (perceptual loss) です。VGG が選ばれるのは、構造が単純で特徴が層ごとにきれいに整理され、再現性が高く、誰もが同じ重みを使える共通基準になっているためです。

## 手法の系譜と主要論文
- VGG 本体: Very Deep Convolutional Networks for Large-Scale Image Recognition (Simonyan & Zisserman; ICLR 2015, Oxford 大学の Visual Geometry Group。VGG はこのグループ名の略)。提案は「3×3 固定の均質な深層 CNN」。動機は「ネットワークの深さが精度に与える影響を、他の条件をそろえて純粋に調べる」こと。新規性は、大きなフィルタを小さなフィルタの積み重ねで置き換える原則を体系的に検証した点です。

- 直前の世代 [AlexNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0369_alexnet) (Krizhevsky et al., 2012) は層が浅く (8層)、フィルタが大きく不均質 (11×11, 5×5, 3×3 が混在) でした。VGG はその設計を「3×3 を深く積む」へと整理・統一しました。

- 並行して GoogLeNet / Inception (Szegedy et al., CVPR 2015) も同じ ImageNet 2014 で競い、こちらは Inception モジュールという並列構造で効率を追求しました。VGG は単純さで、GoogLeNet は効率で対照的でした。

- 翌年の ResNet (He et al., CVPR 2016) は「残差接続 (skip connection、入力を出力に足し込む近道)」で VGG より遥かに深い100層超を学習可能にし、VGG の「単純に深くするだけでは限界がある (深すぎると学習が壊れる)」という問題を解決しました。これにより分類の主役は ResNet 系に移りました。

- 知覚損失として VGG を使った代表例が Perceptual Losses for Real-Time Style Transfer and Super-Resolution (Johnson, Alahi, Fei-Fei; ECCV 2016) で、VGG 中間特徴の差を損失に使う方式を広めました。超解像の SRGAN (Ledig et al., CVPR 2017) や多くの画像生成モデルもこの VGG 知覚損失を採用しています。

## 論文の実験結果 (定量データ)
評価指標は ImageNet (ILSVRC) の top-5 エラー率です。これは予測の上位5位以内に正解が含まれない割合で、当時の主要指標でした。低いほど良い指標です。

- 深さの効果: VGG は11層から19層まで深さだけを変えて比較しました。深くするほどエラーが下がり、VGG-16 と VGG-19 が最良で、ImageNet 検証セットで top-5 エラー約7.3%前後 (複数モデルのアンサンブルや多スケール評価でさらに下がる) を達成しました。これは AlexNet 世代 (top-5 エラー約16%) から大幅な改善です。ILSVRC 2014 の分類部門で2位、物体位置推定 (localization) 部門で1位となりました (分類1位は GoogLeNet)。

- フィルタサイズの検証: 1×1 畳み込みや、3×3 を積む代わりに大きなフィルタを使う構成と比較し、3×3 を深く積む構成が一貫して最良でした。これは「小さいフィルタを深く」という主張の直接の裏づけです。

- 多スケール評価: 学習・推論時に画像を複数の解像度で処理して結果を平均すると、単一スケールよりエラーがさらに下がりました (約1ポイント程度)。物体の大きさのばらつきに対応する効果です。

- アブレーション的観察: ネットワークを深くしても、ある深さを超えると改善が頭打ちになりました。著者らは初期化の工夫 (浅いネットを先に学習し、その重みで深いネットを初期化する) で深いネットの学習を安定させており、これは「残差接続なしでは深さに限界がある」という、翌年 ResNet が解決する課題を示唆していました。後年、BatchNorm (層ごとに出力の分布を正規化する仕組み) を各畳み込みの後に入れた VGG-BN が登場し、この段階的初期化なしでも深い VGG を安定に学習できるようになりました。これは「VGG が深さで詰まった原因の一部は最適化の難しさであり、正規化で緩和できる」ことを後から裏づけた例です。

- LRN の無効性: AlexNet が使っていた LRN (Local Response Normalization) を VGG に入れても精度がほとんど改善せず、むしろメモリと計算を増やすだけ、と報告されました。これが LRN が以後の設計から姿を消す決定打の一つになりました。

これらの数字が重要なのは、「アーキテクチャを単純で均質にしたまま深さを増やすだけで精度が伸びる」という、設計の指針を初めて明快に示した点にあります。

## メリット・トレードオフ・限界
メリット
- 構造が極めて単純で均質なため、理解しやすく実装・改造しやすい。教科書的な明快さがある。
- 3×3 を深く積むという原則は後続 CNN すべてに影響を与えた基礎。
- 中間特徴が人間の知覚に近く安定しているため、知覚損失の特徴抽出器として今も広く有用。

トレードオフ・限界
- パラメータ数・計算量が非常に大きく、特に最初の全結合層が全パラメータの大半を占めて非効率。
- 残差接続がないため、単純に層を増やすと勾配が伝わりにくくなり学習が難しくなる。深さの実用上限が低い。
- 分類精度・効率では ResNet 以降のモデルに大きく劣り、現代のバックボーンとしては使われない。
- メモリ消費が大きく、組み込みやモバイル用途には不向き。

## 発展トピック・研究の最前線
VGG の「均質な積み重ね」という思想は、後の設計に二通りの形で受け継がれました。一方は ResNet・DenseNet のように接続構造を工夫して深さの限界を突破する流れ、もう一方は MobileNet・EfficientNet のように同じ精度をより少ない計算で得る効率化の流れです。VGG 自体は分類タスクの第一線を退きましたが、知覚損失・スタイル変換・GAN の評価など「人間の見た目に近い特徴空間が必要な場面」では今も標準的に使われ続けています。最近は知覚損失の特徴抽出器を VGG から ViT 系や自己教師あり学習で得た特徴に置き換える試み (LPIPS の発展形など) も進んでおり、何を「人間の知覚に近い特徴」とみなすかは依然として研究テーマです。
