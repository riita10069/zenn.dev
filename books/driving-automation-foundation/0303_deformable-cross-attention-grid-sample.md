# grid_sampleによるdeformable cross-attention

## ひとことで言うと

「特徴マップ(画像をニューラルネットに通して得られる数値の格子)の必要な場所だけを、学習で決めた座標からピンポイントに取ってくる」仕組みを deformable attention(変形可能アテンション)と呼びます。本来この仕組みは高速化のために専用の GPU プログラム(CUDA カーネル)を書くのが定石でしたが、深層学習フレームワークに標準で備わるサンプリング関数(PyTorch なら `torch.nn.functional.grid_sample`、以下 `F.grid_sample`)だけで同じことが書けます。専用カーネルを書かない代わりに、どんな環境でもそのまま動く可搬性(ポータビリティ)と、コードの読みやすさを得る、という設計の選択です。

## 直感的な理解

アテンション(attention)とは「入力のどこに注目すべきか」を学習で決める仕組みです。たとえば文章を読むとき、人は全単語を均等に見ず、文脈上重要な単語に視線を集中させます。ニューラルネットでこれを真似たのがアテンションです。

ところが素朴なアテンションは「注目候補すべてとの関連度」を総当たりで計算します。注目候補が N 個あれば、ある問い合わせ(query)につき N 回の内積計算が必要で、問い合わせも N 個あれば全体で N×N 回、つまり計算量が要素数の2乗(O(N²))に膨らみます。鳥瞰図(上から見た地図状の特徴マップ)のようにマス目が多い場面、たとえば 450×300 = 135,000 マスもあると、この総当たりは現実的な時間で終わりません。

ここで発想を変えます。「全部見るのをやめて、本当に見たい少数の点(たとえば4〜8点)だけ見る」。さらにその点の位置を固定せず、問い合わせの内容に応じてずらせる(deform=変形できる)ようにすれば、対象に合わせて柔軟に視線を動かせます。これが deformable attention です。図書館で全棚を端から読むのをやめ、目録から数冊だけ抜き出して読むようなものです。

問題は「学習で決めた小数点座標(たとえば横 137.4 番目、縦 88.9 番目)の特徴をどう取り出すか」です。特徴マップは整数番地のマスにしか値を持ちません。小数番地の値は近所のマスから補間(interpolation、間を埋めて推定する)して作ります。この補間を微分可能(誤差を逆向きに流して学習できる)に行うのが `F.grid_sample` であり、これが deformable attention を専用カーネルなしで書ける鍵です。

## 基礎: 前提となる概念

**特徴マップ(feature map)**: 画像をニューラルネット(畳み込み層など)に通すと、`[チャネル数 C, 高さ H, 幅 W]` という3次元の数値の塊が出てきます。これを特徴マップと呼びます。バッチ(複数サンプルをまとめて処理する単位)を含めると `[B, C, H, W]` です。各空間位置 (h, w) に C 次元のベクトルがあり、その位置に「何があるか」を符号化しています。

**双線形補間(bilinear interpolation)**: 小数座標 (x, y) の値を求めるとき、その点を囲む整数番地4マスの値を、距離に応じた重みで混ぜます。たとえば x=137.4 なら、137 番地に 0.6、138 番地に 0.4 の重みをかける、という具合に縦横それぞれで線形に混ぜるので「双(2方向)線形」です。重みが座標の連続関数なので、出力を座標で微分でき、座標を予測する層まで勾配が流れます。

**誤差逆伝播(backpropagation)**: ニューラルネットは出力の誤差を入力側へ逆向きに伝え、各重みを少しずつ直して学習します。このとき途中の全演算が微分可能でないと勾配が途切れて学習できません。だから「座標を予測して、その座標から特徴を取る」処理が微分可能であることが決定的に重要です。

**`F.grid_sample` の座標規約**: この関数は座標を `[-1, +1]` の範囲で指定します。-1 が左端/上端、+1 が右端/下端、0 が中央です。一方モデル内部では `[0, 1]` の正規化座標(画像幅・高さで割って 0〜1 にした座標)で扱うことが多いので、`grid = loc * 2 - 1` という単純な変換で `[0,1] → [-1,1]` に直してから渡します。

**サンプリング時の境界処理**: 座標が枠外に出たとき何を返すかを `padding_mode` で指定します。`'zeros'` なら枠外はゼロを返し、枠外サンプルが悪さをしないようにできます。`align_corners` はピクセルを「点」とみなすか「面積を持つマス」とみなすかの座標解釈の違いで、解像度を変えても座標の意味がずれないよう `False`(ピクセル中心基準)を選ぶことが多いです。

## 仕組みを詳しく

deformable cross-attention(クロスアテンション=問い合わせ側と参照側が別のテンソルである場合)を `F.grid_sample` で実装する典型的な流れを、軌跡計画ヘッドを例に追います。入力は `query: [B, C]`(現在の問い合わせ、C は埋め込み次元、たとえば 256)と `values: [B, C, H, W]`(参照する特徴マップ、たとえば鳥瞰図)です。

1. **参照点を予測**: `ref = reference_point(query).sigmoid()`。小さな全結合層 `reference_point: Linear(C, 2)` が query から2次元座標を出し、`sigmoid` で `[0,1]` に収めます。形は `[B, 2]`。これが「だいたいこの辺を見る」という基準座標です。

2. **オフセット(ずれ)を予測**: `offsets = sampling_offsets(query).reshape(B, K, 2) * offset_scale`。`sampling_offsets: Linear(C, 2K)` が基準点の周りに K 点(たとえば K=8)をどれだけずらすかを出します。`offset_scale`(たとえば 0.1)を掛けることで、ずれを基準点近傍に制限します。形は `[B, K, 2]`。

3. **各点の注目度を予測**: `attn_w = attention_weights(query).softmax(-1)`。`attention_weights: Linear(C, K)` が K 点それぞれの重みを出し、`softmax`(合計が1になるように正規化する関数)を通します。形は `[B, K]`。

4. **サンプル座標を組み立て**: `sample_locs = ref.unsqueeze(1) + offsets`(基準点に各オフセットを足す)。これを `[-1,1]` に変換し `[B, K, 1, 2]` に整形します。`F.grid_sample` は `grid` を `[B, H_out, W_out, 2]` の形で要求するので、K 点を `H_out=K, W_out=1` として並べます。

5. **特徴を取る**: `sampled = F.grid_sample(values, sample_grid, mode='bilinear', padding_mode='zeros', align_corners=False)`。形は `[B, C, K, 1]`。K 点それぞれの位置で双線形補間された C 次元特徴です。

6. **重み付き合計**: `attended = (sampled.squeeze(-1) * attn_w.unsqueeze(1)).sum(dim=-1)`。K 点を注目度で混ぜて1つの特徴 `[B, C]` にし、最後に `output_proj: Linear(C, C)` を通します。

数値例で形の流れを追うと、`B=4, C=256, H=450, W=300, K=8` のとき:

```
query          [4, 256]
ref            [4, 2]          ← sigmoid で [0,1]
offsets        [4, 8, 2]       ← *0.1 で近傍に制限
attn_w         [4, 8]          ← softmax で合計1
sample_grid    [4, 8, 1, 2]    ← [-1,1] 座標
sampled        [4, 256, 8, 1]  ← grid_sample 出力
attended       [4, 256]        ← 8点の重み付き和
```

**なぜ `offset_scale` が要るのか**: 学習初期は `sampling_offsets` の出力が大きく暴れ、サンプル点が枠外へ飛んだり1点に密集したりして学習が発散しやすくなります。`offsets * 0.1` でずれを基準点の半径10%程度に抑えると序盤から安定します。長距離の注目が必要なら基準点 `ref` 自体を動かせばよいので、オフセットを抑えても表現力は失われません。これは「粗い位置(ref)+ 細かい変形(offsets)」という役割分担です。

**オリジナルの Deformable DETR との対応**: Deformable DETR の本来の定式化はマルチヘッド(複数の独立した注目を並列に持つ)かつマルチスケール(複数解像度の特徴を同時に参照)で、効率のため専用 CUDA カーネル(論文では `MSDeformAttn` と呼ばれる演算子)を使います。`F.grid_sample` を使う簡略版はこれを単一ヘッド・単一スケールに落とし込んだものに相当し、表現力を一部犠牲にして可搬性を取った実装と位置づけられます。両者の数学的な違いを明確にしておくと、オリジナルは各ヘッド `m`・各スケール `l` ごとに参照点 `p` 周りの `K` 点をサンプルし、`MSDeformAttn(z_q, p) = Σ_m W_m [ Σ_l Σ_k A_{mlqk} · W'_m x_l(p + Δp_{mlqk}) ]` と書けます。ここで `A` はアテンション重み(`Σ_k A = 1`)、`Δp` は学習されるオフセット、`W_m`/`W'_m` は射影行列です。`F.grid_sample` 版はこの内側のサンプリング `x(p + Δp)` を双線形補間で素直に実装し、ヘッド数を 1、スケール数を 1 に固定したものとみなせます。マルチヘッドやマルチスケールが欲しければ、`grid_sample` を頭数・スケール数だけループで呼べば原理的には再現できますが、その分だけ Python レベルのループが増え、専用カーネルの一括処理に対して速度で不利になります。

## 手法の系譜と主要論文

- **Spatial Transformer Networks**(Jaderberg, Simonyan, Zisserman, NeurIPS 2015, arXiv:1506.02025): 「微分可能なサンプリング」を最初に定式化した研究です。画像内の対象を自動で切り出し・変形するために、サンプリング格子(grid)を生成し、双線形補間で特徴を取り出す層(grid generator + sampler)を提案しました。任意座標からの双線形サンプリングが誤差逆伝播できることを示したのがこの論文で、後年の `grid_sample` API の理論的基礎です。これがなければ deformable attention を標準関数だけで書くことはできません。

- **Deformable Convolutional Networks**(Dai et al., ICCV 2017, arXiv:1703.06211): 畳み込みのサンプリング位置を固定格子から学習可能なオフセット付きに拡張しました。「注目位置を学習でずらす」という deformable の核心思想を畳み込みに導入した研究で、後の deformable attention の前身です。

- **Deformable DETR**(Zhu, Su, Li, Wang, Li, Dai, ICLR 2021, arXiv:2010.04159): DETR(物体検出を集合予測として解く Transformer)が全画素アテンションのせいで収束が遅く小物体に弱かった問題を、deformable attention で解決しました。各 query が特徴マップ全体ではなく、学習で決めた少数の参照点周り(各ヘッド K=4点が典型)だけにアテンションします。効率的な実装に専用 CUDA カーネルを使う点がトレードオフで、`F.grid_sample` 実装はこのカーネル依存を回避する代替です。

- **BEVFormer**(Li, Wang, Lu, Xie, Dai, Lu, Zhou, Qiao, ECCV 2022, arXiv:2203.17270): deformable attention を多カメラの鳥瞰図生成に応用しました。3D参照点を各カメラ画像に投影し、その投影位置周りだけを deformable にサンプルする spatial cross-attention を提案しています。多視点融合での deformable サンプリングの代表例です。

この系譜は「固定サンプリング → 学習可能オフセット(Deformable Conv) → アテンションへの統合(Deformable DETR) → 多視点幾何への応用(BEVFormer)」と進み、底流に一貫して「Spatial Transformer の微分可能サンプリング」が流れています。

## 論文の実験結果(定量データ)

deformable attention の効果を最も明快に示すのは Deformable DETR です。物体検出の標準ベンチマーク COCO(約12万枚の画像に物体の枠とクラスを付けたデータセット)で評価されます。指標は **AP(Average Precision、平均適合率)** で、予測した枠が正解とどれだけ重なり(IoU)正しいクラスかを、複数のしきい値で平均した0〜100の総合精度です。高いほど良い指標です。

- 元の DETR は COCO で約 42〜43 AP に達するのに **500 エポック**(学習データを500周なめる)もの学習を要しました。Deformable DETR は **50 エポック**、つまり約10分の1の学習量で同等以上(論文の標準設定で 43.8 AP、改良版の iterative bounding box refinement と two-stage を加えると 46〜52 AP)に到達しました。収束の約10倍高速化が最大の成果です。なぜ速くなるかというと、元の DETR のアテンションは初期化時に全画素へほぼ一様に薄く注目を配るため、「対象の周りに注目を集中させる」状態に学習で到達するまで何百エポックもかかります。deformable は最初から参照点近傍の少数点しか見ないので、無駄な探索空間が小さく、収束が速くなります。
- 小物体の精度 AP_S(small、面積が 32×32 ピクセル未満の物体だけの AP)で特に改善が大きく、論文では元 DETR の AP_S 約 20.5 に対し Deformable DETR は約 26.4 と報告されています。全画素アテンションでは高解像度の細部に注目が割けなかったのに対し、複数解像度をまたいで参照点周りを見る deformable では小物体を捉えやすくなったためです。
- アブレーション(ある要素を抜いて効果を確かめる実験)では、マルチスケール入力(複数解像度を同時に参照)を単一スケールに落とすと AP が約 1.7 ポイント低下し、特に小物体で顕著に落ちると示されています。サンプリング点数 K(論文では各ヘッド 4 点が標準)を減らすと精度がなだらかに下がり、増やしても 4〜8 点付近で飽和します。つまり「少数点で十分だが、少なすぎると表現力不足、多すぎても見返りが小さい」という飽和的な関係があります。さらにアテンション重みを学習せず一様固定にすると精度が落ちることから、「どの点を重視するか」を学習することが効いていると分かります。

BEVFormer は自動運転の nuScenes(6カメラ + LiDAR を備えた都市走行データセット)で評価され、カメラのみの3D物体検出指標 **NDS(nuScenes Detection Score、検出精度に速度・属性誤差を加味した総合スコア、0〜1で高いほど良い)** で当時のカメラ単体手法の上位に入りました。deformable な spatial cross-attention により、全画素アテンションより大幅に軽い計算で多視点融合を実現した点が評価されました。

`F.grid_sample` 版は専用カーネル版に比べ、同じサンプル点数なら速度・メモリ効率で劣ることがあります(関数呼び出しのオーバーヘッドや、マルチヘッドを明示ループで書く必要があるため)。一方で、点数が少なく単一スケールであれば差は小さく、可搬性の利得が上回る、というのが実装上の判断になります。

## メリット・トレードオフ・限界

メリット:
- 専用 CUDA カーネルが不要で、CPU でも GPU でもそのまま動く。ビルド環境やハードウェア世代に縛られず、再現性とテスト容易性が高い。
- `F.grid_sample` は微分可能なので、サンプリング座標を予測する層まで end-to-end で学習できる。
- 全要素アテンション(O(N²))より圧倒的に軽い。K 点だけ見るので問い合わせあたり O(K) で済む。

トレードオフ・限界:
- 専用カーネル版より速度・メモリ効率で劣る場合がある。マルチヘッド・マルチスケールを Python ループで展開すると遅くなりやすい。
- 単一ヘッド・少点数の簡略実装はマルチヘッド deformable attention ほどの表現力がない。
- 学習初期の不安定さを `offset_scale` で抑える必要があり、この値の調整がハイパーパラメータとして残る。大きすぎると発散、小さすぎると注目範囲が狭くなりすぎる。
- 双線形補間は近傍4点しか使わないため、急峻な特徴の境界では補間がぼやける。高周波の細部を正確に拾うには解像度を上げる必要がある。

未解決の研究課題としては、サンプリング点数・ヘッド数・オフセット範囲を入力に応じて動的に変える適応的サンプリングや、`grid_sample` のメモリ効率を専用カーネル並みに引き上げるコンパイラ最適化(`torch.compile` 等)の活用が挙げられます。

## 発展トピック・研究の最前線

- **DAT(Deformable Attention Transformer、Xia et al., CVPR 2022, arXiv:2201.00520)**: 画像分類・検出のバックボーン自体に deformable attention を組み込み、データ依存で注目位置を選ぶ汎用ビジョン Transformer を提案しました。deformable の応用が検出ヘッドからバックボーンへ広がった例です。
- **疎なクエリ設計の流れ**: BEVFormer のような密な鳥瞰図クエリに対し、Sparse4D や StreamPETR は疎な3Dクエリで効率を上げる方向を探っています。deformable サンプリングはこうした疎クエリ手法でも特徴抽出の中核を担います。
- **時系列への拡張**: 単一フレームではなく過去フレームの特徴も参照点周りでサンプルする temporal deformable attention により、速度や遮蔽の推定が改善することが報告されています。
