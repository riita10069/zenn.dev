# PatchTST

## ひとことで言うと

PatchTST は長い時系列を遠くまで予測するための Transformer です。新しいことは二つだけで、(1) 時系列を1点ずつ入れるのをやめ、連続した数十点をまとめた「かたまり(パッチ)」を一つの入力単位にする、(2) 複数の変数(チャネル)を混ぜず、一つずつ独立に処理する。この二つの素直な工夫だけで、Informer や Autoformer や FEDformer のような複雑な機構を上回る予測精度を出しました。当時「複雑な Transformer は単純な線形モデルに負ける」と言われていた流れに対し、「Transformer は使い方さえ正しければ時系列に強い」と示し直した、転換点となる手法です。

## 直感的な理解

文章を読むとき、人間は一文字ずつ意味を取っているわけではありません。「天気」「明日」「晴れ」といった単語のかたまりで意味を捉えます。一文字ずつ「天」「気」と見ても情報が薄く、関連を見つけにくい。これと同じことが時系列にも言えます。

従来の時系列 Transformer は、各時刻の値(複数変数なら、その瞬間のスナップショット)を一つのトークンとして自己注意にかけていました。これは「一文字ずつ注意する」ようなもので、一点だけでは「数ステップにわたる上下動のパターン」という意味のある形を持ちません。注意が拾える情報が薄くなり、しかもトークン数が系列長そのままなので計算も重くなります。

PatchTST のアイデアは、画像認識で大成功した Vision Transformer(ViT)から来ています。ViT は画像を 16×16 ピクセルの小さなパッチに切り、各パッチを一つのトークンにして Transformer に入れました。「画像が16×16の単語でできているなら、時系列も連続する数十点の単語でできているはずだ」。論文のタイトル「A Time Series is Worth 64 Words(時系列は64の単語に値する)」が、まさにこの ViT のタイトル「An Image is Worth 16x16 Words」のもじりになっています。64 という数は、典型的な設定でパッチ化後のトークン数が数十オーダーになることを指しています。

もう一つの工夫は、変数を独立に扱うことです。電力データなら多数の変数があるとき、従来は全変数を一点にまとめて一トークンにしていました。PatchTST は逆に、各変数を独立した一次元時系列として、同じ Transformer に別々に通します。変数間の相関を無理に学ばせるとノイズや過学習を招くことが多いので、いっそ独立に扱うほうが頑健だ、という発見です。これを「チャネル独立(channel independence)」と呼びます。

## 基礎: 前提となる概念

必要な言葉を順に説明します。

トークン(token)とは Transformer が処理する最小の入力単位です。言語なら単語や部分単語、画像ならパッチ、PatchTST では「時系列のパッチ」が一トークンになります。Transformer の自己注意は、トークン同士の関連を学ぶので、何を一トークンにするかが性能を大きく左右します。トークンの選び方は、Transformer 系手法における最も重要な設計判断の一つです。

自己注意の計算量は系列(トークン列)の長さ L に対して O(L²)、つまり L の二乗に比例します。L=512 なら 512² ≈ 26万ペアの重みを計算します。L を減らせば二乗で軽くなるので、トークン数を減らすことは効率に直結します。

チャネル(channel)あるいは変数(variate)とは、多変量時系列における各系列のことです。例えば気象データなら気温・湿度・気圧がそれぞれチャネルです。多変量予測では、これらをどう扱うかが設計の分かれ目になります。チャネル混在(channel-mixing)は全変数を一トークンにまとめる方式、チャネル独立(channel-independence)は各変数を別々に処理する方式です。前者は変数間の相関を直接使えますが、変数ごとにスケールや性質が違うと過学習しやすい。後者はその逆です。

正規化(normalization)も予測の前処理で重要です。時系列は時間とともに平均や分散がずれる(分布シフト、distribution shift)ことが多く、入力ごとに平均・標準偏差で正規化し、出力で元に戻す RevIN(Reversible Instance Normalization、Kim et al., ICLR 2022)という手法がよく併用されます。PatchTST も同種のインスタンス正規化を使い、これが精度に効くことが知られています。

自己教師あり学習(self-supervised learning)とは、ラベルなしのデータから「データの一部を隠して当てさせる」ような問題を自分で作って学習する枠組みです。BERT(言語、マスク単語予測)や MAE(画像、マスクパッチ復元)が代表で、大量のラベルなしデータで事前学習し、少量のラベル付きタスクに転移します。

## 仕組みを詳しく

PatchTST は三つの処理からできています。パッチ化、チャネル独立、線形予測ヘッドです。任意で自己教師あり事前学習も加えられます。

### パッチ化(Patching)

長さ L の一変数時系列を、長さ P のパッチに区切ります。隣り合うパッチをどれだけずらすかをストライド S で決めます。

```
元: x[1], x[2], ..., x[L]            L=512(1点ずつだと512トークン)
パッチ長 P=16, ストライド S=8 とすると
パッチ数 N = floor((L - P)/S) + 1 ≈ 63
各パッチは16点をまとめた1トークン  →  トークン数が 512 から 約64 に激減
```

S < P なら隣のパッチが重なり、S = P なら重ならない区切りになります。効果は二つあります。第一に、トークン数が約 L/S に減るので、自己注意が O(L²) から O((L/S)²) へ大きく軽くなります。上の例ならペア数は約26万から約4千へ、二桁減ります。これにより、入力長を 336 や 512 のように長く取っても現実的なコストで回せます。第二に、各トークンが「16点ぶんの局所的な形」という意味のある単位になり、注意が拾える情報が濃くなります。各パッチは長さ P のベクトルを線形層で d 次元に埋め込み、位置埋め込み([Positional Encoding](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0338_positional-encoding))を足して、通常の Transformer Encoder に入ります。

### チャネル独立(Channel Independence)

M 個の変数を持つ時系列 `[B, M, L]`(B はバッチ)を、変数の軸をバッチに畳み込んで `[B·M, L]` のように扱い、各変数を独立した一次元系列として同じ Transformer に通します。重みは全変数で共有します。

なぜ独立にするのか。変数間の相関を無理にモデルに学ばせると、変数ごとにスケールや性質が違う場合にノイズや過学習を招きやすい。一方で一変数ずつ処理し、重みは共有すれば、変数に共通するパターン(周期や形状)は学べるうえ、頑健で過学習しにくくなります。「相関は捨てるが共通の形は学ぶ」という割り切りです。重み共有のおかげで、変数が増えてもパラメータ数は増えず、むしろ実質的な学習サンプル数(変数 × 時系列窓)が増えて学習が安定します。

### 予測ヘッド

各チャネルについて、全パッチの表現 `[N, d]` を平らに並べて(flatten)、一つの線形層で予測したい長さ L_pred の値をまとめて出力します。自己回帰(一点ずつ予測して入力に戻す)をしないので速く、誤差が累積しません。これは長期予測で先が長くなるほど効いてきます。自己回帰だと一点の誤差が次の入力を汚し、先に行くほど雪だるま式に悪化しますが、一括出力ならその連鎖が起きません。

### 自己教師あり事前学習(任意)

パッチの一部をランダムにマスク(隠す)して、その中身を復元させる事前学習を提案しました。ラベルなしの大量の時系列で表現を学んでおき、少量データのタスクに転移できます。これは BERT のマスク単語予測、MAE のマスクパッチ復元と同じ発想で、パッチ化したからこそ自然に適用できる利点です。点単位でマスクすると簡単に近傍補間で当てられてしまいますが、パッチ単位でマスクすると「形」を理解しないと復元できないので、より良い表現が学べます。

### 形状で追う

```
入力: [B, M, L]                              M変数、各長さL
 正規化(RevIN系) →
 パッチ化: [B, M, N, P]                        N パッチ、各長さP
 チャネル独立に展開: [B·M, N, P]
 パッチ埋め込み: [B·M, N, d] + 位置埋め込み
 Transformer Encoder ×層: [B·M, N, d]
 flatten + 線形ヘッド: [B·M, L_pred]
 チャネルを戻す + 逆正規化: [B, M, L_pred]
```

## 手法の系譜と主要論文

時系列予測に Transformer を入れる試みは、効率化(LogSparse, NeurIPS 2019 → Informer, AAAI 2021)→ 周期分解(Autoformer, NeurIPS 2021 → FEDformer, ICML 2022)と、機構をどんどん複雑にする方向で進んでいました(→ [Informer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0334_informer) / [Autoformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0321_autoformer))。注意をどう疎にするか、どう周波数領域で取るか、という工夫が競われていたのです。

ここに冷や水を浴びせたのが Zeng らの DLinear(Zeng et al., "Are Transformers Effective for Time Series Forecasting?", AAAI 2023, arXiv:2205.13504)です。時系列を分解して一層の線形回帰をかけるだけのモデルが、これら複雑な Transformer を多くのベンチで上回り、「Transformer は本当に時系列に有効なのか」という問いを突きつけました。論文タイトルそのものが挑発的で、コミュニティに大きな衝撃を与えました。

PatchTST(Nie, Nguyen, Sinthong, Kalagnanam; ICLR 2023、IBM Research と Princeton の共同)はこの問いに「Transformer の使い方が悪かっただけだ」と答えました。新規性はパッチ化とチャネル独立という二つの素直な工夫で、これだけで Informer/Autoformer/FEDformer も DLinear も上回りました。複雑な注意改造ではなく、入力の作り方を変えることで勝った点が重要です。パッチ化の起源は Dosovitskiy らの ViT(Dosovitskiy et al., "An Image is Worth 16x16 Words", ICLR 2021, arXiv:2010.11929)で、PatchTST はそれを時系列に持ち込んだ形です。

その後、チャネルの扱い方をめぐって議論が続きます。Liu らの iTransformer(Liu et al., ICLR 2024, arXiv:2310.06625)は、PatchTST のチャネル独立とは逆に、変数(チャネル)そのものを一トークンにして変数間の注意を取る方式を提案し、変数相関が重要なデータで強さを示しました。PatchTST が「変数は独立に」と言ったのに対し、iTransformer は「変数間の関係こそ注意で学べ」と主張したわけで、どちらが良いかはデータ依存というのが現在の理解です。両者を組み合わせる(時間方向と変数方向の両方に注意を取る)手法も後続で多数提案されています。

## 論文の実験結果(定量データ)

評価は長期予測の標準ベンチで行われました。Weather(21の気象指標)、Traffic(862本の道路センサー)、Electricity(321顧客)、ETT 系列(ETTh1/h2/m1/m2、電力トランスの温度)、ILI(インフルエンザ様疾患)などです。予測長は 96, 192, 336, 720(ILI は 24〜60)と複数設定し、入力長も長め(例: 336 や 512)を使います。入力を長くできるのはパッチ化で計算が軽いからで、これが精度向上の一因でもあります。

指標は MSE と MAE です。MSE は予測と正解の差を二乗して平均した値で、大きく外すほど重くペナルティがかかります。MAE は差の絶対値の平均で外れ値に頑健です。どちらも小さいほど良く、その差は「平均してどれだけ予測がずれるか」を表します。入力は正規化されているので無次元の相対量として比較できます。

論文の主な結果として、PatchTST は教師あり設定で、Weather・Traffic・Electricity・ETT の多くの予測長で当時の SOTA(state-of-the-art、最高性能)を更新しました。特に Traffic や Electricity のような変数が多く周期性の強いデータで効果が大きく、FEDformer や Autoformer に対して MSE で二桁パーセント(報告例では Traffic などで 20%超)の削減が示された設定もあります。DLinear に対しても、多くの設定で同等以上、特に長い予測長で優位を示しました。これは「複雑な Transformer は線形モデルに負ける」という DLinear の結論を、Transformer 側から覆した点で意義が大きい結果です。

自己教師あり事前学習の効果も検証されました。マスクパッチ復元で事前学習してから微調整(fine-tuning)すると、ゼロから教師あり学習するより精度が上がり、さらに別のデータセットへ転移(transfer)させても良好な性能を保つと報告されています。これはパッチ化が表現学習と相性が良いことを示し、後の時系列基盤モデルの設計を後押ししました。

アブレーションでは、パッチ化を外して一点トークンに戻すと性能が落ち、トークン数増加による計算コストも跳ね上がりました。チャネル独立を外してチャネル混在にすると、多くのデータで過学習気味になり精度が低下しました。つまりパッチ化とチャネル独立はそれぞれ独立に効いていることが確認されています。一方で、パッチ長 P を変えると性能が変わり、データの周期に近い長さのとき良くなる傾向が見られました。入力長を伸ばすほど精度が上がる(過去をより多く活かせる)点も確認され、これは従来 Transformer が入力を伸ばすと逆に悪化していた弱点を克服したことを意味します。

## メリット・トレードオフ・限界

メリットは、パッチ化でトークン数を削減して計算を軽くしつつ、局所的な形を一トークンに凝縮できること。チャネル独立で過学習しにくく頑健で、重み共有により学習効率も良いこと。複雑な注意改造なしに従来手法を上回り、設計がシンプルで再現しやすいこと。そしてパッチマスク復元による自己教師あり事前学習が自然に行えることです。シンプルさゆえに実装・調整がしやすく、ベースラインとして広く使われています。

トレードオフと限界もあります。第一に、チャネル独立は変数間の強い相互作用を取りこぼす恐れがあります。変数同士が因果的に強く絡むデータ(片方が原因でもう片方が結果になるようなもの)では、iTransformer 系のように変数間注意を取る方式のほうが良い場合があります。第二に、パッチ長とストライドがハイパーパラメータで、データの周期と合わないと性能が出にくい。第三に、パッチ内の点の順序は埋め込み線形層に潰されるため、パッチ内部の細かい時間順序情報は失われやすく、非常に短い周期の微細構造を捉えにくいことがあります。

研究上の未解決課題として、チャネル独立 vs チャネル依存の最適なバランスは依然はっきりしていません。データの変数間相関の強さ・サンプル数・予測長に応じて最適解が変わり、汎用的な指針は確立していません。また DLinear の存在が示すように、「そもそもパッチ化 Transformer の表現力が、より単純なモデルに対してどれだけ本質的な優位を持つか」は、データや評価設定(入力長・正規化の有無)に強く依存し、一般的な結論には至っていません。ベンチマーク自体が小さく汎化を測りきれていない、という批判もあります。

## 発展トピック・研究の最前線

PatchTST 以降、長期予測研究はパッチ表現を出発点にさらに広がりました。一つは前述の iTransformer に代表される「チャネルをどう扱うか」の探索で、変数トークン化や変数間注意とパッチ化を組み合わせる方向(Crossformer など、時間軸と変数軸の両方に注意を取る手法)が研究されています。

もう一つは時系列の基盤モデル(foundation model)です。大量の異種時系列で事前学習し、未知のデータをゼロショットで予測するモデル(TimesFM, Moirai, Chronos, Lag-Llama, MOMENT など 2023〜2024)の多くが、入力をパッチ化して扱います。PatchTST のパッチ化は、この基盤モデル路線の標準的な部品として定着しました。マスク復元の事前学習も、これらの大規模事前学習の素地になっています。言語モデルの成功(トークン化+大規模事前学習)を時系列に移植する流れの、最初の重要な一歩が PatchTST だったと位置づけられます。

非 Transformer 系でも、TimesNet(Wu et al., ICLR 2023)のように周期で系列を二次元化して畳み込む手法や、TSMixer のような MLP ベース手法など、パッチや周期の概念を活かしたモデルが多数現れています。

なお「連続した小片をまとめてトークンにする」というパッチ化の発想自体は、画像のパッチ化(ViT)、音声のフレーム化、自動運転で映像や履歴を局所窓に切る処理など、系列を扱うあらゆる分野で共通する基本設計です。系列が長く一点あたりの情報が薄いときに、まとめて意味のある単位にするという考え方は、時系列予測を超えて広く応用できます。
