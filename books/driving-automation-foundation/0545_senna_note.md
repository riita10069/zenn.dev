# Senna (VLM + E2E 二系統)

## ひとことで言うと
Senna は、画像と言葉が両方分かる賢いAI(VLM)に「次は減速して直進維持する」という高レベルの方針を言葉で決めさせ、別の専用モデル(画像から軌道を直接出す End-to-End モデル)にその方針に従う具体的な軌道を作らせる、役割分担型の運転システムです。賢い意味理解と精密な数値生成という相反する2つの能力を、別々のモジュールに分業させるのが核心です。じっくり考える賢い系(遅い)と反射的に動く速い系(速い)を分ける「二系統(System1/System2)」設計の代表例の1つです。

## 直感的な理解
運転には2種類の頭の使い方があります。1つは「あの工事の看板があるから先で車線が減りそうだ。あの歩行者は渡りたそうにしている。だから今は減速して様子を見よう」という、意味を理解してじっくり考える判断です。もう1つは「では実際にハンドルを何度切り、何メートル先まで何秒かけて進むか」という、座標と数値で正確に出す操作です。

人間は前者を意識的に(ゆっくり)、後者をほぼ無意識に(素早く)こなしています。問題は、AIでこの2つを1つのモデルにまとめてやらせようとすると、どちらかが犠牲になることです。言葉が得意な大規模AIに「3.2秒後にx=12.4, y=1.1へ行け」と座標を喋らせると、桁を間違えたり不正確になりがちです。逆に画像から軌道だけを高速に出すモデルは、座標の精度は高いのに「なぜその軌道なのか」を説明できず、見慣れない状況で意味を取り違えます。

Senna のアイデアは単純です。「考える担当」と「操作する担当」を分けて、それぞれを得意分野に専念させる。賢いAI(VLM)には数値を喋らせず、言葉のレベルの方針(減速する・左に寄る、など)だけを決めさせる。座標の精密な生成は、それが得意な専用モデルに任せる。この分業を仲介する仕組みが Bridge(橋渡し)です。比喩を避けて言えば、これは「意思決定の抽象度を、各モジュールの数値表現能力に合わせて分割する」という設計判断です。

## 基礎: 前提となる概念
理解に必要な用語を、知らない人向けに噛み砕きます。

VLM(Vision-Language Model、ビジョン言語モデル)とは、画像と言葉を一緒に扱える大規模AIのことです。たとえば画像を入力すると「これは雨の交差点で、右に駐車車両があり、左から自転車が来ている」と言葉で説明できます。LLM(大規模言語モデル、文章を理解・生成するAI)に「目」を付けたものと考えてよいです。内部構造は、画像をパッチに分割して特徴ベクトルに変える視覚エンコーダ(典型的には Vision Transformer)、その視覚特徴を言語モデルが読める空間に射影する projector(投影層)、そして言語を生成する LLM 本体、の3段からなります。世界の常識(工事中は車線が減る、雨だと滑る)を大量のテキストとペア画像から学んでいるのが強みです。

E2E プランナー(End-to-End Planner)とは、生のセンサ入力(カメラ画像など)から、将来走るべき軌道(座標の列)を直接出力するモデルです。End-to-End とは「入口(画像)から出口(軌道)まで一気通貫で、途中を人間が手作業で設計しない」という意味です。出力は典型的には将来3秒分の (x, y) 座標を0.5秒刻みで6点並べた tensor shape [6, 2] のようなものになります。代表的な実装系統には、ベクトル化したシーン表現で計画する VAD(Jiang et al., ICCV 2023)のような手法があります。

meta-action(メタアクション)とは、運転の方針を少数の離散的な言葉のカテゴリで表したものです。「メタ」は「一段上の・抽象的な」という意味で、具体的な座標の一段上にある抽象的な指示です。連続値の座標ではなく「減速」「左に寄る」といった有限個の選択肢なので、言語モデルが扱いやすい形になります。離散カテゴリにすることには情報理論的な利点もあります。連続値の予測は誤差が無限に小さくなり得る難しい回帰問題ですが、有限カテゴリの予測は分類問題になり、softmax で確率分布として扱え、学習が安定します。

conditioning(条件づけ)とは、モデルの出力をある指示・制約に沿わせることです。「減速」という条件を与えると、E2E プランナーは減速を満たす軌道の中から最適なものを選ぶようになります。条件なしで自由に出すのではなく、上位の方針に従わせるわけです。実装上は、meta-action を埋め込みベクトル(embedding、カテゴリを固定長の数値ベクトルに変換したもの)に変え、それを E2E プランナーの中間特徴に連結(concat)したり cross-attention で注入したりします。

System1/System2 とは、心理学者カーネマンの二重過程理論に由来し、System1 が速くて自動的な思考、System2 が遅くて熟慮的な思考を指します。運転AIでは VLM が System2(遅い系)、E2E が System1(速い系)に対応します。詳しくは [System1/System2 (Kahneman 二重過程理論)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0546_system1-system2-kahneman) と [Fast-Slow 二系統運転アーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0537_fast-slow-thinking-driving) を参照してください。

## 仕組みを詳しく
Senna の核心は「VLMに数値の軌道を喋らせず、言葉の高レベル運転判断(meta-action)を喋らせ、それを別モデルが軌道に変換する」ことです。

meta-action の設計は、運転判断を2つの直交する軸に分けます。

- 横方向(lateral): 左に寄る / 直進維持 / 右に寄る、といった操舵の方向。
- 縦方向(longitudinal): 加速 / 速度維持 / 減速 / 停止、といった速度の扱い。

VLM はこの組み合わせ(例「直進維持 × 減速」)を言葉で出し、さらに「前方に駐車車両があり接近中のため」という理由の説明文も添えます。横と縦を直交軸に分けるのは組合せ爆発を防ぐ工夫です。仮に「左に寄りながら減速」「右に寄りながら加速」を1つの巨大なフラットなカテゴリ集合として持つと、組合せは横3×縦4=12個ですが、軸ごとに分ければ3+4=7個のロジットを学べばよく、各軸が独立に学習できてデータ効率が上がります。座標ではなく言葉なので VLM の得意分野に問題を落とし込め、しかも理由が出るので解釈性(なぜそう動いたかが人間に分かること)が高まります。

処理の流れを段階的に示します。

```
複数カメラ画像 + ナビ指示(例: 次の交差点を直進)
   │
   ▼
[VLM] ─── 画像とナビを理解し、運転知識で方針を言語化
   │  出力(言葉):
   │   横: 直進維持
   │   縦: 減速
   │   理由: 前方に駐車車両があり接近中のため
   ▼
[Bridge] ─── meta-action を E2E が使える条件(埋め込み等)に変換
   │
   ▼
[E2E プランナー] ─── meta-action で条件づけて
                     具体的な軌道(座標列)を生成
   │  出力(数値):
   │   将来3秒の (x,y) を6点  tensor shape [6, 2]
   ▼
車両制御へ
```

ここで決定的に重要なのは、E2E プランナーが自由勝手に軌道を出すのではなく、VLM が決めた meta-action に条件づけられて出すことです。これにより「賢い方針(VLM)」と「精密な実行(E2E)」が一直線につながります。VLM が「減速」と言えば、E2E は減速を満たす軌道空間の中で最適点を探す、という役割分担です。

なぜ VLM に座標を直接出させないのか。これが Senna の設計思想の中心です。言語モデルはトークン(語の断片)を1つずつ生成する仕組みで、連続値を高精度に出すのが構造的に苦手です。座標をテキストにして「12.4」と書かせると、トークナイザが "12" と ".4" のように数字を分割するため桁の関係を学びにくく、桁落ちや数値の不安定さが出ます。さらに自己回帰生成(1トークンずつ逐次に出す)は遅延が大きく、6点×2座標を数十トークンかけて出すと車載の実時間制約(典型的には10Hz、すなわち100ms 以内)を満たせません。一方、責務を「言語による判断」と「数値による生成」にきれいに分けると、各モジュールが得意分野に専念でき、汎化(未知状況への強さ)と精度が両立しやすくなります。

学習面では、まず VLM を運転向けの質問応答データで訓練します。具体的には、シーン説明(今どんな状況か)、重要物体の認識(注意すべき対象は何か)、推奨 meta-action(この状況でどう運転すべきか)といった大量のQAペアを用意し、VLM に「運転に必要な常識と判断」を教え込みます。これを planning-oriented QA と呼びます。汎用の画像言語データで事前学習された VLM を、運転 QA でファインチューニング(微調整)する2段階学習が典型です。その後、VLM の出力する meta-action で E2E プランナーを条件づけ、軌道予測を学習します。VLM は汎用知識を保持しているため、運転データが少ない珍しい状況でも常識で補える、という利点が生まれます。

## 手法の系譜と主要論文
LLM/VLM を運転に使う研究の流れの中で、Senna の立ち位置を時系列で追います。

第一世代は「LLM/VLM に直接、軌道や制御を喋らせる」アプローチでした。GPT-Driver (Mao, Qian, Ye, Wang, 2023, arXiv:2310.01415) は、運転計画を言語モデリング問題として定式化し、GPT-3.5 に座標列をテキストで出力させました。知覚結果を言語化してプロンプトに入れ、軌道をトークン列として生成させる発想です。DriveGPT4 (Xu et al., 2023, arXiv:2310.01412) はマルチモーダル LLM で低レベル制御信号(速度・操舵)を出し、同時に行動の説明も生成しました。LMDrive (Shao et al., CVPR 2024, arXiv:2312.07488) は言語指示に従う閉ループ運転を CARLA シミュレータで実証しました。これらは解釈性と言語指示追従という新しい価値を示しましたが、共通の弱点が「言語モデルに数値を喋らせると不正確で、しかも自己回帰生成は遅い」ことでした。

第二世代は「VLM と高速な数値モデルを組み合わせる」二系統アプローチへ進みます。DriveVLM-Dual (Tian et al., CoRL 2024, arXiv:2402.12289) は、VLM が Chain-of-Thought(段階的推論)で scene description → scene analysis → hierarchical planning と粗い軌道を出してから、Fast 分枝(従来型の高速プランナー)がそれを精緻化・高頻度化する構成です。VLM の遅さを Fast 分枝で補い、実時間性を確保する考えです。VLM が低頻度(数Hz)で粗い計画を更新し、高速モジュールが高頻度(10Hz以上)で実行する、という非同期協調が核心です。

Senna (Jiang et al., 2024, arXiv:2410.22313, Huazhong University of Science and Technology と Horizon Robotics ら) は、この二系統をさらに純化しました。DriveVLM-Dual が VLM に粗い軌道まで出させるのに対し、Senna は VLM の出力を「言葉の meta-action」に絞り、座標生成を完全に E2E 側(Senna-E2E、VAD 系のプランナー)へ寄せます。責務分離をより徹底することで、各モジュールの得意分野への専念を最大化し、汎化と精度の両立を狙ったのが新規性です。VLM 側を Senna-VLM と呼び、planning-oriented QA データセットで訓練します。

この系譜の背後には、運転を System1/System2 に分けるという大きな思想([Fast-Slow 二系統運転アーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0537_fast-slow-thinking-driving))と、VLM の常識でロングテール(滅多に起きない稀な状況)に対処したいという動機があります。

## 論文の実験結果(定量データ)
Senna は主に nuScenes(代表的な自動運転ベンチマークデータセット。ボストンとシンガポールで収集された約1000シーン・約40000サンプルの実走行データ)で評価されました。

計画性能の主指標は L2 誤差(予測軌道と人間の実際の軌道の、各時刻での距離のズレをメートルで測ったもの。小さいほど人間の運転に近い)と衝突率(collision rate、予測軌道が他物体とぶつかる割合。小さいほど安全)です。L2 誤差が小さいことは「人間の軌道を正確に再現できている」ことを、衝突率が低いことは「安全な軌道を出せている」ことを意味します。nuScenes では将来1秒・2秒・3秒の L2 をメートルで報告し、その平均を見るのが慣例です。報告によると、Senna は VLM による高レベル判断を加えることで、座標だけを直接出すベースラインの E2E プランナー(VAD など)に比べて平均 L2 誤差と平均衝突率の両方を改善し、当時の SOTA(state-of-the-art、最高性能)水準の計画性能を達成したとされます。とくに meta-action による条件づけが、人間の意図(ここで減速すべき、ここで車線変更すべき)に沿った軌道を選ばせる効果を持つと報告されています。

汎化(クロスドメイン転移)の評価も重視されました。あるデータセットで学習したモデルを別の都市・別の収集環境のデータで試すと、通常は性能が大きく落ちます。Senna は VLM の常識が効くため、こうした未知ドメインでの zero-shot/few-shot 転移において性能の落ち込みが小さく、ベースラインより大きな優位を示したことが報告されています。これは「VLM の事前学習知識が、運転データだけでは届かない汎化を補う」という設計意図を裏づける結果です。具体的には、別データセットへ少量のデータで適応させたとき、Senna の方がはるかに小さい L2 で済んだと報告されています。

アブレーション(構成要素を抜いて効果を確かめる実験)の観点では、(1) meta-action による条件づけを外して E2E が自由に軌道を出すようにすると計画性能が落ちる、(2) VLM の planning-oriented QA 事前学習(シーン記述・重要物体認識・行動予測の各 QA)を省くと判断の質が落ちる、(3) 重要物体を抽出する設計を外すと、シーン理解が弱まり meta-action の精度が下がる、といった傾向が示されています。これは「VLM の判断層」と「専用の運転知識学習」の両方が寄与していることを意味します。なお、これらの数値はデータセットや評価プロトコルの違い(自車運動を入力に含めるか、地面真値の知覚を使うか等)で大きく変動するため、絶対値ではなく「同一条件のベースラインに対する相対的な改善」として読むのが妥当です。

## メリット・トレードオフ・限界
メリット:

- VLM は言語による判断、E2E は数値による軌道生成、と各モジュールが得意分野に専念でき、賢さと精度を両立しやすい。
- meta-action と理由の説明文が出るため、なぜその運転をしたかが人間に分かる(解釈性・デバッグ性・事故時の説明責任)。
- VLM の常識のおかげで、ロングテールや別ドメインへの汎化が効きやすい。
- 離散 meta-action は分類問題として学べるため、連続軌道回帰を VLM に強いるより学習が安定する。

トレードオフと限界:

- 2段構成ぶんの学習・推論コストとメモリ。VLM は重く(数B〜数十B パラメータ)、車載での実時間動作には量子化・蒸留・低頻度実行(VLM を数Hz、E2E を10Hz超で非同期に回す)などの工夫が要る。
- meta-action という中間表現がボトルネックになりうる。カテゴリが粗いと微妙なニュアンス(「あの子供がボールを追いそうだから、わずかに右に余裕を取る」)を表現できず、VLM の賢さが E2E に伝わりきらない。これは情報理論でいう情報ボトルネックそのもので、横3×縦4=12通り程度の離散値では、連続的に変化する状況の細部を運べない。
- VLM の判断が誤ると、その誤った方針に E2E が忠実に従う(誤りの伝播)。VLM 自体の幻覚(hallucination、もっともらしい嘘の生成)も持ち込まれる。
- VLM の推論遅延を吸収する仕組み(遅い系の出力が間に合わない瞬間に速い系をどう動かすか。[Fast-Slow 二系統運転アーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0537_fast-slow-thinking-driving) の遅延問題)が別途必要。

未解決の研究課題として、meta-action の表現粒度をどう最適化するか(離散カテゴリのまま豊かにするか、連続的な意図埋め込みにするか)、VLM の判断の正しさをどう検証・保証するか、二系統間の責務境界をどこに引くのが最適か、VLM の幻覚を安全層でどう止めるか、などが残っています。

## 発展トピック・研究の最前線
さらに深く学ぶ方向として、いくつかの流れがあります。1つは、VLM の出力を meta-action のような離散ラベルではなく、連続的な「意図埋め込み(latent intent)」として E2E に渡し、表現のボトルネックを緩める研究です。離散カテゴリの代わりに VLM の隠れ状態(hidden state)を直接 E2E にトークンとして渡すことで、より豊かな情報を運ぶ試みがあります。もう1つは、VLM 側に反実仮想推論([Counterfactual Reasoning (反実仮想)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0530_counterfactual-reasoning-driving))を持たせ、「この meta-action を選んだら何が起きるか」を考えさせることで、より因果に基づいた判断にする方向です。

また、二系統設計の根本にある「速い系と遅い系をどう協調させるか」は、VLA(Vision-Language-Action モデル、知覚から行動までを1つの大きなモデルで扱う流れ)との対比でも議論されています。Senna のように責務を明示的に分けるか、それとも1つの大規模モデルに統合するか、という設計の二択は今も研究コミュニティで活発に検討されています。統合派は誤りの伝播やボトルネックを避けられる一方で、実時間性と数値精度の両立という Senna が回避した難問に正面から向き合う必要があります。閉ループ評価(実際にシミュレータで走らせて性能を測る)の重要性も近年強調されており、nuScenes のような開ループ L2 だけでは過小評価・過大評価が起きることが指摘されています。
