# 時系列メモリ (Temporal Memory) — 過去フレームをどう要約するか

## ひとことで言うと
過去のフレーム(視覚の見え方と自車の運動の履歴)を1本の「文脈ベクトル」にまとめ直す部品です。「過去をどうまとめるか」の方式を差し替えられるのが特徴で、何もせず最新フレームだけ使う方式と、細かい高頻度の履歴を低頻度の濃い表現に圧縮してから要約する方式などを切り替えられます。この部品は「過去を要約するエンコーダ」であって「未来の軌道を出すプランナ」ではない、という役割分担が肝心です。

## 直感的な理解
運転は「いまこの瞬間」だけでは決められません。前の車がさっきから減速し続けているのか、自分がいま加速の途中なのか、横の歩行者がどちらに動いているのか。こうした判断には「少し前からの流れ」が要ります。1枚の静止画を見ても、その車が止まっているのか時速60kmで走っているのかは分かりません。連続したコマを並べて初めて速度や加速が見えます。これは物理的にもその通りで、速度は位置の時間微分、加速度はさらにその微分です。1枚の画像には微分の情報が原理的に含まれないため、時間方向の情報が必須になります。

モデルにこの「流れ」を渡すのが時系列メモリの役割です。素朴には過去のコマを全部渡せばよさそうですが、ここに落とし穴があります。たとえばカメラが 10Hz(1秒に10コマ)で動くと、数秒ぶんの履歴は時間方向に長く、各コマの特徴は薄い「細長い」表現になりがちです。細長い履歴をそのまま使うと、ステップごとに判断がチラつく(decision flicker、隣り合うフレームで出力がガタガタ揺れる)原因になります。チラつきは乗り心地を悪くするだけでなく、急なハンドル操作として安全性も損ないます。そこで「過去のまとめ方」を選べるようにし、目的に応じて粗く要約するか細かく残すかを切り替えます。

## 基礎: 前提となる概念
入力テンソルの形を `[B, T, feat]` と書きます。B はバッチサイズ(同時に処理するサンプル数)、T は時間ステップ数(過去何コマぶんか)、feat は各コマの特徴次元(1コマを何個の数値で表すか)です。たとえば `[8, 64, 896]` なら、8サンプルそれぞれについて、過去64コマ、各コマ896次元の視覚特徴がある、という意味です。

ダウンサンプリング(downsampling)とは、時系列の時間解像度を下げる操作です。10コマを1コマにまとめれば、時間方向の長さが1/10になります。逆に各コマを表す特徴の次元数(チャンネル数)を増やすことを「特徴方向に濃くする」と表現します。「時間方向は粗く、特徴方向は濃く」とは、長さを縮める代わりに各点の情報量を増やすことです。情報量の総量を保ったまま形を変える、という考え方です。

RNN(Recurrent Neural Network、回帰結合型ニューラルネット)は、系列を1ステップずつ順に読み、内部状態(記憶)を更新しながら最後にまとめのベクトルを出すネットワークです。素朴な RNN は長い系列で勾配消失(誤差信号が遡るほど小さくなり、遠い過去を学べない)を起こすため、これを解決するために LSTM(Hochreiter & Schmidhuber 1997)や GRU が提案されました。GRU(Gated Recurrent Unit; Cho et al., EMNLP 2014, arXiv:1406.1078)は、「どれだけ過去を覚え、どれだけ忘れるか」をゲート(門、0〜1の係数)で制御し、長い系列でも勾配が消えにくいよう工夫されたものです。LSTM より少ないゲート(更新ゲートとリセットゲートの2つ)で構成され、パラメータが少なく軽量です。

1次元畳み込み(Conv1d)は、時間軸に沿って小さな窓(カーネル)をスライドさせ、窓内の値を重み付き和にまとめる操作です。カーネル幅とストライド(窓を動かす歩幅)を同じ値にすると、窓が重ならず、たとえば10コマごとに1つの出力に「プール」できます。これが時間圧縮の道具になります。重み付き和なので、単純な平均より「どの瞬間を重視するか」を学習できる点が利点です。

## 仕組みを詳しく
典型的な設計では、複数の要約方式を文字列キーで切り替えられるレジストリ(部品の登録表)にします。たとえば次のような構造です。

```
REGISTRY = {
    "no_memory": NoMemory,            # 最新フレームだけ使う
    "one_hz":    OneHzHistoryEncoder, # 10Hz の履歴を 1Hz に圧縮して要約
}
```

ビルダ関数が文字列キーから実体を組み立て、共通インターフェース(抽象基底クラス)が「`[B, T, feat]` の履歴を受け取り `[B, feat]` の文脈を返す」という約束を定めます。視覚と自車運動の2系列を同時に扱う設計が一般的です。共通インターフェースを切ることで、新しい要約方式を足しても呼び出し側を変えずに差し替えられます。

方式1: 最新フレームのみ(no_memory、ベースライン)。
パラメータを持たない素通しです。履歴が3次元 `[B, T, feat]` なら最新の1コマ `history[:, -1]` を、すでに2次元 `[B, feat]` ならそのまま返します。「過去は使わず最新だけ」という比較用の下限(lower bound)です。研究では、新しい機構を提案するときに「それを抜いた最小構成」と比べて初めて効果が言えるため、こうした下限の用意が重要です。実装上の注意として、ビルダが全方式に視覚次元などの引数を一律に転送するため、この方式はそれらを受け取って無視する仕掛け(`**kwargs`)を持ちます。これがないと既定構成での組み立てに失敗します。

方式2: 高頻度履歴の圧縮要約(one_hz)。
「10Hz の細長い履歴を 1Hz の太い表現に圧縮してから要約する」方式です。中核は2段階です。

1. 時間圧縮。重なりなしの Conv1d(カーネル幅=ストライド=subsample_ratio、たとえば10)で、10ステップごとの窓を1ステップにプールしつつ、特徴次元を hidden_dim まで広げます。履歴は古い→新しいの順とみなし、T が ratio の倍数でないときは先頭(最も古い)側の余りを切り捨て、最新フレームが必ず残るようにします。例として T=64・ratio=10 なら、余り4を捨てて 60→6 コマ(約1Hz で約6.4秒ぶん)に圧縮します。GELU(滑らかな活性化)と LayerNorm(正規化)で整えます。

2. 系列要約。圧縮した約1Hz の系列を GRU に通し、最後の隠れ状態 `[B, hidden_dim]` を文脈として取り出します。

視覚履歴と自車運動履歴を扱う場合、両者を特徴次元で連結して1本(たとえば visual 896 + egomotion 256 = joint 1152 次元)にし、一括で圧縮した後、視覚ぶん(先頭896次元)と運動ぶん(残り256次元)に切り戻して返す設計が使われます。2系列をまとめて圧縮することで、視覚と運動の時間的な対応(同じ瞬間の見え方と動き)を一緒に学べます。別々に処理すると「この見え方とこの運動が同時に起きた」という対応が壊れやすいため、合わせて畳み込むことに意味があります。

数値例: `[B, 64, 896]` の10Hz視覚履歴に one_hz を適用すると、64コマ →(余り4切り捨て)→ 60コマ → 6コマ(約1Hz・6.4秒)→ GRU で `[B, hidden_dim]` の1ベクトルになります。no_memory なら同じ入力から64コマ目だけを取って `[B, 896]` を返します。

```
[B,64,896]  ──Conv1d(k=s=10)──>  [B,6,hidden]  ──GRU──>  [B,hidden]   (one_hz)
[B,64,896]  ──最新1コマ抽出──>  [B,896]                                (no_memory)
```

なぜこの設計なのか。第一に、要約方式をレジストリで差し替えることで、過去をどう扱うかという研究上の選択肢を1行で切り替えて比較・再現できます。第二に、時間圧縮を先にかけてから RNN に通すことで、長い系列を直接 RNN に流す場合の計算コストと勾配消失を避けつつ、各点の特徴容量を確保します(60ステップを直接 RNN に流すより、6ステップに圧縮してから流すほうが速く、勾配も遡りやすい)。第三に、最新フレームを必ず残す切り捨て方は、「いまこの瞬間」の情報を失わないための安全策です。制御では最新の観測が最も重要なので、これを捨てないことが効きます。

## 手法の系譜と主要論文
時系列を畳み込みでダウンサンプルしてから RNN で要約する構成は、音声・行動認識で定番化しました。WaveNet(van den Oord et al., 2016, arXiv:1609.03499)は、膨らませた(dilated)因果畳み込みで時間方向の受容野を指数的に広げる発想を示し、長い音声系列を扱えるようにしました。膨張畳み込みとは、カーネルの参照点を1個飛ばし・2個飛ばしと間引いて広い範囲を少ない層数で見る技法で、層を重ねるたびに受容野が2倍ずつ広がります。TCN(Temporal Convolutional Network; Bai, Kolter, Koltun 2018, arXiv:1803.01271)は、因果畳み込み+膨張+残差接続だけで、系列モデリングにおいて LSTM/GRU と同等以上の性能を出せることを多数のベンチマークで示し、「系列だから RNN」という固定観念に一石を投じました。狙いは長い系列を扱いやすい長さに圧縮しつつ受容野を広げることで、トレードオフは粗くしすぎると速い変化を取りこぼすことです。

GRU 自体は Cho et al.(EMNLP 2014, arXiv:1406.1078)が機械翻訳の文脈で提案し、LSTM(Hochreiter & Schmidhuber 1997)より少ないゲートで同等の性能を出せる軽量な選択肢として広まりました。LSTM が記憶セルと3ゲート(入力・忘却・出力)を持つのに対し、GRU は記憶セルを隠れ状態に統合し2ゲートに簡略化しています。

自動運転の時系列融合では、BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)が前フレームの BEV(鳥瞰図、上から見下ろした特徴マップ)特徴を時間的自己アテンションで取り込み、停止物と移動物の区別や遮蔽(occlusion、物陰)下の追従を改善しました。ここで紹介した履歴圧縮方式は、BEV 特徴そのものではなく履歴ベクトルを圧縮する軽量版という位置づけで、BEV の時間融合と相補的です。固定の畳み込みで圧縮するか、アテンションで「どの過去フレームが重要か」を場面ごとに選ぶか、というのが設計の分かれ目です。詳しくは [BEV (鳥瞰図) によるマルチビュー融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0933_bev-view-fusion) を参照してください。

「履歴が細すぎる/判断がチラつく」問題への対処として時間解像度を落とす発想は、制御や強化学習でいう行動の時間的一貫性、あるいはフレームスキップ(frame-skipping)とも通じます。DQN(Mnih et al., Nature 2015)は同じ行動を数フレーム繰り返すことで学習を安定・高速化しました。粗い時間粒度は安定する反面、反応の遅れというトレードオフを伴います。

## 論文の実験結果(定量データ)
TCN(Bai et al., 2018)は、コピー記憶課題・順序付き MNIST(画像を1ピクセルずつ系列として読ませる課題)・多種の系列モデリングタスクで、同程度のパラメータの LSTM/GRU を多くのケースで上回りました。たとえば順序付き MNIST(Permuted MNIST)では、TCN が約 97% 以上の精度を出し、ベースラインの LSTM/GRU(おおむね 88〜96% 程度)を上回りました。長距離依存を要する課題ほど畳み込みの広い受容野が効くことを示しています。指標は分類精度や系列予測誤差で、要するに「長い系列の中の離れた手がかりをどれだけ正しく結びつけられるか」を測っています。さらに、RNN の逐次処理に比べ畳み込みは並列計算できるため、学習が速いという実務的な利点も報告されました。

BEVFormer(2022)は nuScenes で評価され、時間的自己アテンション(過去フレームの BEV を参照する仕組み)を入れると、3D 物体検出の総合指標 NDS(nuScenes Detection Score、検出精度と速度・属性推定をまとめた指標。高いほど良い)が向上しました。報告では NDS が約 51.7% に達し、時間情報を使わない単フレーム版に対し数ポイント高い値です。特に物体の速度推定誤差(mAVE、mean Average Velocity Error。小さいほど良い)が大きく下がりました。速度は1フレームでは原理的に測れない量なので、時間融合の効果が最も鮮明に出る指標です。アブレーションとして時間的自己アテンションを外すと、移動物の検出・速度推定が悪化することが示されています。

履歴の時間粒度に関する一般的な知見として、時間解像度を下げると出力の分散(チラつき)が減り軌道が滑らかになる一方、急ブレーキ直後のような速い遷移の追従が遅れる、というトレードオフが繰り返し観測されます。subsample_ratio のような圧縮率は、この滑らかさと反応性のバランスを決めるハイパーパラメータで、運転では「滑らかさで乗り心地を、反応性で安全性を」担保するため、両者の妥協点を実験で探します。

## メリット・トレードオフ・限界
メリットは、要約方式を文字列1つで差し替えられ、実験の切り替えと再現が容易な点です。高頻度履歴を低頻度の濃い表現に圧縮する方式は、決定のチラつきを抑えつつ各ステップの特徴容量を確保できます。視覚と自車運動を連結して一括圧縮すれば、見え方と動きの時間的対応を一緒に学べます。最新フレームのみの方式はパラメータゼロで軽く、比較の下限として使えます。GRU は LSTM より軽く、圧縮後の短い系列ならコストは小さく済みます。

トレードオフと限界もあります。最新フレームのみの方式は過去を一切使わないため、減速の継続や接近の傾向といった時間的手がかりを取りこぼします。圧縮要約方式は時間を粗くするぶん、1Hz より速い急変(急ブレーキ直後の挙動など)の細部が潰れうるので、圧縮率のチューニングが効きます。さらに、履歴が圧縮率より短いと圧縮方式は成立せず(最低1窓ぶん必要)、短い履歴では使えません。GRU は系列を逐次処理するため並列化しにくく、長い系列では Transformer 系の時間融合に対して速度面で不利になることもあります。圧縮を固定の畳み込みで行うと、場面に応じて重要な過去フレームを選ぶといった適応的な要約はできない、という表現上の限界もあります。たとえば「3秒前に飛び出した歩行者」が今の判断に決定的でも、固定畳み込みは均等に圧縮してしまい、その瞬間を強調できません。

## 発展トピック・研究の最前線
さらに深く学ぶ方向として、第一に Transformer ベースの時間融合があります。BEVFormer のように時間的自己アテンションで「どの過去フレームを参照するか」を学習で決める方式は、固定畳み込みより適応的です。第二に、状態空間モデル(State Space Model)系、たとえば S4 や Mamba(Gu & Dao, 2023, arXiv:2312.00752)は、長い系列を線形時間で効率的に扱える新世代の系列モデルとして注目され、RNN の逐次性と畳み込みの並列性の利点を選択的に併せ持ちます。Mamba は入力依存のゲートで「何を覚え何を捨てるか」を動的に決めるため、固定畳み込みの弱点を補う方向です。第三に、再帰的な隠れ状態を持つ「メモリ」として明示的に記憶を保持する Recurrent Memory Transformer 系の研究があります。第四に、運転特有の課題として、可変フレームレート(センサーごとに頻度が違う)や非同期センサーの時間整合を扱う研究、また連続時間でのモデリング(Neural ODE など、不均一な時刻のサンプルを自然に扱う)が進んでいます。
