# 対照的自己教師あり学習

## ひとことで言うと

対照的自己教師あり学習(Contrastive Self-Supervised Learning)とは、人間が貼ったラベル(正解)を一切使わずに、「同じものから作った2つの見え方は似た特徴を持つべきで、別のものとは違う特徴を持つべき」というルールだけを手がかりに、画像などのデータから役に立つ表現(特徴ベクトル)を学ぶ手法です。

## 直感的な理解

まず2つの言葉を噛み砕きます。

自己教師あり学習(self-supervised learning)とは、データ自身から「擬似的な正解」を自動で作り出して学習する方法です。人間がラベルを付ける必要がありません。たとえば文章の一部を隠して「隠した単語を当てよ」という問題はデータだけから作れます。画像でも同じことができます。

対照的(contrastive)とは「比べる」という意味です。ここでは「似ているはずのペア(正例)」と「似ていないはずのペア(負例)」を比べて、前者を近く、後者を遠くに配置するよう学習します。

直感を絵にします。1枚の犬の写真があるとします。これを2通りに加工します。片方は左右反転して色を少し変える、もう片方は一部を切り出して拡大する。すると見た目は違うが「中身は同じ犬」の2枚ができます。この2枚を「正例ペア」と呼び、特徴空間(各画像を点で表す高次元の空間)で近くに来てほしい、と要求します。一方、まったく別の猫や車の写真は「負例」で、遠くに来てほしいと要求します。

```
特徴空間(イメージ: 2次元に潰した図)

   犬A視点1  ●─近づける─● 犬A視点2     ← 正例ペア(同じ画像由来)
                  │
                遠ざける
                  │
        猫 ○      車 ○      ← 負例(別の画像)
```

この「近づける・遠ざける」を大量の画像で繰り返すと、ラベルなしでも「犬らしさ」「物体の形」といった意味的な特徴を捉えた表現が育ちます。育った表現は、後で少量のラベルを足すだけで分類や検出に転用でき、自動運転のような大量の無ラベル映像が手に入る分野で特に価値があります。

## 基礎: 前提となる概念

- 表現(representation)/埋め込み(embedding): 入力画像を、ニューラルネットワークが扱いやすい数値ベクトル(たとえば128次元や2048次元)に変換したもの。良い表現とは「意味の近いものが近くに来る」ベクトルのことです。
- エンコーダ(encoder): 画像を表現に変換するネットワーク本体。多くは ResNet や Vision Transformer。
- データ拡張(data augmentation): 元データに、反転・切り抜き・色変換・ぼかしなどの変換を加えて見え方を増やす操作。対照学習では「同じ画像から2つの見え方を作る」ために必須です。
- 正例(positive)/負例(negative): 似ているべきペアが正例、似ていないべきペアが負例。
- 温度パラメータ(temperature、τ): 後述する損失関数で、似ている・似ていないの判定の「鋭さ」を調整するつまみ。
- 線形評価(linear probing): 学習した表現の良さを測る定番手順。エンコーダを凍結(重みを固定)し、その上に1層だけの線形分類器を載せてラベルで学習し、分類精度を見る。表現が良ければ、1層足すだけで高精度が出ます。

## 仕組みを詳しく

中心にあるのが InfoNCE 損失(Information Noise-Contrastive Estimation、Oord ら 2018)です。SimCLR ではこれを NT-Xent(Normalized Temperature-scaled Cross Entropy)と呼びます。式は次の形です。

L = -log( exp(sim(z_i, z_j)/τ) / Σ_k exp(sim(z_i, z_k)/τ) )

記号を言葉で説明します。

- z_i, z_j: 同じ画像から作った2つの見え方の表現ベクトル(これが正例ペア)。
- sim(a, b): 2ベクトルの類似度。通常はコサイン類似度(cosine similarity、ベクトルの向きの近さ。-1 から 1 の値で、1 なら同じ向き)。
- τ(温度): 小さいほど類似度の差を強調します。SimCLR では 0.1 から 0.5 あたりがよく使われ、0.1 前後が良いと報告されています。
- 分母の Σ_k: 正例 z_j と、たくさんの負例 z_k 全部にわたる和。
- 全体の意味: 「正例どうしの類似度が、すべての候補の中でいちばん高くなってほしい」。これは実質「z_i から見て、正しい相手 z_j を負例の群れの中から当てる分類問題」になっています。当たれば損失が小さく、外せば大きくなります。

ここで決定的に重要なのが「負例をどう、どれだけ集めるか」です。負例が多いほど学習は難しく、その分良い表現が育つことが経験的に知られています。この負例の調達方法で、主要手法が枝分かれします。

第一の系統、SimCLR(Chen ら 2020)は、巨大なミニバッチ(一度に処理するデータの束)を使い、バッチ内の他のすべての画像を負例とします。バッチサイズ N なら、各サンプルに対して 2N-2 個の負例が自動的に手に入ります。仕組みは単純ですが、たくさんの負例を確保するには大きなバッチ(報告では 4096 以上)と多数の GPU/TPU が必要という弱点があります。SimCLR は2つの追加の発見でも有名です。(1) エンコーダ出力にそのまま損失をかけるのではなく、間に投影ヘッド(projection head、2層程度の小さな MLP)を挟み、その出力で対照損失を計算すると性能が大きく上がる。(2) 強いデータ拡張(特に「ランダム切り抜き」と「色の歪み」の組み合わせ)が決定的に効く。

第二の系統、MoCo(Momentum Contrast、He ら 2020)は、負例を「キュー(queue、待ち行列)」に溜める発想です。過去のミニバッチで計算した表現を辞書(dictionary)として保持し、新しいバッチが来たらそれらを負例として使い、最も古いものを捨てます。これにより、バッチサイズを大きくしなくても膨大な負例(報告では 65536 個規模)を使えます。ただし、辞書の中の表現は古いエンコーダで作られたもので「鮮度」がばらつくと学習が不安定になります。そこで MoCo はモメンタムエンコーダ(momentum encoder)を導入します。これは負例側のエンコーダを、勾配ではなく、メインのエンコーダ重みの指数移動平均(ゆっくり追従する平均)で更新する仕組みです。

θ_k ← m · θ_k + (1 - m) · θ_q

θ_k が負例側(key)エンコーダ、θ_q がメイン(query)エンコーダ、m はモメンタム係数(0.999 など 1 に近い値)。m が 1 に近いほど key エンコーダはゆっくり変化し、辞書の中の表現の一貫性が保たれます。

```
MoCo の構造(概略)

  画像 ─┬─ query エンコーダ(勾配で更新)─→ q
        │
        └─ key エンコーダ(EMA で更新)──→ k+   ← 正例
                                         [キュー: k-, k-, ...]  ← 過去の負例
  損失: q を、k+ を当てる問題として InfoNCE
```

その後、負例を一切使わない自己教師あり手法も登場しました。BYOL(Bootstrap Your Own Latent、Grill ら 2020)や SimSiam(Chen と He 2021)は、正例ペアだけを使い、「予測ネットワークと停止勾配(stop-gradient、片側の勾配を止める操作)」という工夫で、すべての表現が一点に潰れてしまう自明解(collapse、崩壊)を回避します。負例が要らないので小バッチでも回しやすい利点があります。クラスタリングを使う SwAV(Caron ら 2020)も同系統です。これらは「対照的」とは厳密には呼ばないこともありますが、対照学習が切り開いた自己教師あり表現学習の延長線上にあります。

## 手法の系譜と主要論文

- Oord, Li, Vinyals, "Representation Learning with Contrastive Predictive Coding"(CPC, arXiv:1807.03748, 2018): InfoNCE 損失を提示。対照学習の理論的な土台。
- Wu, Xiong, Yu, Lin, "Unsupervised Feature Learning via Non-Parametric Instance Discrimination"(CVPR 2018): 「各画像を1クラスとみなす」インスタンス識別という発想と、負例を溜めるメモリバンクの原型。
- He, Fan, Wu, Xie, Girshick, "Momentum Contrast for Unsupervised Visual Representation Learning"(MoCo, arXiv:1911.05722, CVPR 2020): モメンタムエンコーダとキューで負例問題を解決。
- Chen, Kornblith, Norouzi, Hinton, "A Simple Framework for Contrastive Learning of Visual Representations"(SimCLR, arXiv:2002.05709, ICML 2020): 大バッチ・強い拡張・投影ヘッドで対照学習をシンプルかつ強力に。
- Chen, Fan, Girshick, He, "Improved Baselines with Momentum Contrastive Learning"(MoCo v2, 2020): SimCLR の知見(投影ヘッド・強い拡張)を MoCo に取り込み改善。
- Grill ら, "Bootstrap Your Own Latent"(BYOL, NeurIPS 2020): 負例なしでも崩壊しないことを示し、常識を覆す。
- Chen, He, "Exploring Simple Siamese Representation Learning"(SimSiam, CVPR 2021): 停止勾配だけで崩壊回避できることを単純化して示す。
- Caron ら, "Unsupervised Learning of Visual Features by Contrasting Cluster Assignments"(SwAV, NeurIPS 2020): クラスタ割り当ての対照で負例を不要に。

この系譜の延長として、世界モデル系の JEPA(Joint Embedding Predictive Architecture)は、ピクセルを再構成せず表現空間で予測する点で、対照学習や BYOL の思想を受け継いでいます。

## 論文の実験結果(定量データ)

評価の定番は ImageNet(約128万枚、1000クラスの画像データセット)での線形評価です。ResNet-50 というエンコーダを使い、自己教師ありで学習した後に線形分類器だけを足したときの Top-1 精度(最も確率の高い1クラスが正解と一致する割合)で比較します。比較の基準として、同じ ResNet-50 をラベル付きで普通に教師あり学習すると Top-1 はおよそ 76 パーセント前後です。

- MoCo(初代, 2020): ResNet-50 線形評価でおよそ 60.6 パーセント(報告値)。当時、教師ありと無教師の差を大きく縮めた成果として注目されました。
- SimCLR(2020): ResNet-50 でおよそ 69.3 パーセント(報告値)。大バッチと強い拡張、投影ヘッドの効果でさらに前進。アブレーション(要素を一つずつ外して効果を測る実験)では、投影ヘッドを外すと数ポイント低下し、色の歪みを含む強い拡張を弱めると大きく低下することが示されました。
- MoCo v2(2020): SimCLR の知見を取り込み、200 エポック学習でおよそ 67.5 パーセント、800 エポックでおよそ 71.1 パーセント(報告値)。SimCLR の大バッチを必要とせず同等以上を達成した点が実用上の利点です。
- BYOL(2020): 負例なしでおよそ 74.3 パーセント(報告値)と、教師ありにかなり近づきました。

数値の読み方を補足します。線形評価は「凍結した表現の上に1層足すだけ」なので、表現そのものの良さを直接測っています。60 パーセントから 74 パーセントへの上昇は「ほとんど分離されていない特徴」から「ほぼ教師ありに匹敵する分離した特徴」への進歩を意味します。

転移学習(別タスクへの応用)でも価値が示されています。自己教師あり表現を初期値にして物体検出(COCO データセットなど)を学習すると、報告によっては教師あり事前学習と同等かそれ以上の検出精度(平均適合率 AP)が得られ、特にラベルが少ない設定での優位が一貫して観察されています。半教師あり設定(ラベル 1 パーセントや 10 パーセントだけ使う)では、自己教師あり事前学習が大きく効くことが SimCLR などで報告されています。

## メリット・トレードオフ・限界

メリットは、人手のラベルなしに大量データから表現を学べることです。自動運転のように生映像が膨大に集まる一方でアノテーション(ラベル付け)が高価な領域では、これは決定的に重要です。学習した表現は下流の検出・セグメンテーション・分類へ転用でき、特に少ラベル設定で強みを発揮します。

トレードオフと限界もあります。

第一に、データ拡張への強い依存です。性能は「どんな拡張をどの強さで使うか」に敏感で、ある領域で効く拡張が別領域では有害なこともあります。たとえば色の歪みは一般物体では有効ですが、色そのものが意味を持つタスク(信号機の色など)では情報を壊す危険があります。

第二に、計算コストです。SimCLR の大バッチ要件は多数の GPU/TPU を要求します。MoCo はこれを緩和しますが、それでも長いエポック数の学習が必要です。

第三に、崩壊(collapse)のリスクです。対照損失で負例が不足したり、BYOL/SimSiam で停止勾配などの工夫を誤ると、すべての入力が同じ表現に潰れて何も学べなくなります。なぜ BYOL が負例なしで崩壊しないのか、その理論的理解は活発な研究対象でした。

第四に、何を「同じ」とみなすかの限界です。対照学習は「同じ画像由来=似ているべき」と仮定しますが、本当は別画像でも意味的に同じもの(別々に撮った2匹の犬)を負例として遠ざけてしまう「偽の負例(false negative)」問題があります。これは表現を歪める潜在的な原因です。

## 発展トピック・研究の最前線

マルチモーダルへの拡張: CLIP(Radford ら 2021)は画像とテキストを対照学習で結びつけ、「画像とその説明文を正例、無関係な組を負例」とすることで、ゼロショット分類(学習時に見ていないクラスを文章で指定して分類)を可能にしました。対照学習がモダリティをまたぐ整列に有効なことを示した重要な事例です。

マスク再構成系との対比: その後、Masked Autoencoder(MAE、He ら 2021)のように「画像の大部分を隠して復元する」生成的な自己教師あり学習が台頭し、対照学習と双璧をなしています。対照系は意味的な分離に強く、マスク系は密な予測タスク(セグメンテーション等)に強い、といった使い分けが議論されています。

予測アーキテクチャ(JEPA): ピクセル再構成も明示的な負例も使わず、表現空間で未来や隠した部分を予測する I-JEPA や V-JEPA は、対照学習と BYOL の思想を統合・発展させたものとして注目されています。世界モデルへの接続点です。

理論的理解: 対照損失が表現空間で「整列(alignment、正例を近づける)」と「均一性(uniformity、表現を球面上に広く散らす)」という2つの性質を同時に最適化していることが示され(Wang と Isola 2020)、なぜ良い表現が得られるかの理解が進みました。
