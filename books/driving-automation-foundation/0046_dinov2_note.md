# DINOv2

## ひとことで言うと
[DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) と [iBOT](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0060_ibot_note) の学習方法を、徹底的にキュレーション(選別・整理)した約1億4千万枚の画像で大規模に回し、「微調整(ファインチューニング)せずにそのまま貼り付けるだけで多くのタスクに効く、汎用の画像特徴抽出器」を作ったものです。言語の世界で言えば「画像版の基盤モデル(foundation model)」を目指した取り組みで、新しいアルゴリズムの発明というより、既存の良い手法を大規模・安定・高品質に回しきった工学的総合力に本質があります。

## 直感的な理解
自然言語処理では、BERT や GPT のように「大量のテキストで一度だけ事前学習しておけば、あとは少しの調整で(あるいは調整なしで)多様なタスクに使える」基盤モデルが定着しました。一度作れば翻訳にも要約にも質問応答にも使える、汎用の言語理解エンジンです。画像でも同じものを作りたい、というのが大きな動機です。

ここで実用上の決定的なキーワードが「微調整なし(frozen、重みを凍結したまま)」です。普通は事前学習したバックボーンを、各タスクごとに追加学習(微調整)して適応させます。しかしタスクごとに微調整するのは手間もコストもかかり、タスクの数だけバックボーンのコピーが要ります。DINOv2 が目指したのは「特徴抽出器の重みは1つに固定し、その上に小さな分類器や回帰器を1個載せるだけ」で高性能が出る状態です。これが実現すれば、たった1つの凍結バックボーンを、分類・深度推定・セグメンテーション・画像検索など何十ものタスクに使い回せます。

研究の難しさは、モデルとデータを大きくすると学習が壊れやすくなる点にあります。自己教師あり学習は崩壊しやすく、巨大モデルではメモリも計算も爆発します。DINOv2 はそれを地道なデータ選別と実装最適化で乗り越えたところに本質があり、「テキストの助けを借りずに、純画像の自己教師あり学習だけで基盤モデルが作れる」ことを実証しました。

## 基礎: 前提となる概念
- 基盤モデル(foundation model): 大量のデータで広く事前学習し、多様な下流タスクに転用できる大規模モデル。一度作れば多くの用途に使い回せる。
- キュレーション(curation): データの選別・整理。インターネットから無造作に集めた画像(uncurated data、雑多で品質も内容もばらばら)は、人気の物に偏りがち(ロングテールの希少な概念が極端に少ない)で、そのまま学習すると表現の質が落ちる。良質で多様な分布になるよう選び抜く工程がキュレーション。
- 凍結特徴(frozen features): バックボーンの重みを固定したまま取り出す特徴。微調整(全層を再学習)と対比される。凍結で高性能なら、その特徴が真に汎用的である証拠になる。
- 弱教師あり(weakly-supervised): CLIP のように、画像に付随するテキスト(キャプション、altテキスト)を弱い教師信号として使う方法。人手のクラスラベルより安価だが、テキストという外部情報に依存する。DINOv2 はこのテキストすら使わない純画像学習。
- 知識蒸留(distillation): 大きな教師モデルの知識を小さな生徒モデルに移す技術。DINOv2 では巨大モデルから配布用の小モデルを作るのに使う。
- 自己蒸留と MIM(Masked Image Modeling): 詳細は [DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) と [iBOT](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0060_ibot_note) を参照。DINOv2 の学習目的はこの2つの組み合わせ。

## 仕組みを詳しく
DINOv2 は3つの柱で成り立っています。

1. データのキュレーション(自動選別パイプライン): まずインターネットから収集した約12億枚の未選別画像を出発点にします。これをそのまま使うと質が低いので、独自の自動パイプラインで選別します。具体的には、ImageNet など「お手本となるキュレーション済みデータセット」の画像に特徴が似ているものを、画像同士の類似度(自己教師ありで学習した埋め込みのコサイン類似度)で検索して拾い、近重複(near-duplicate、ほぼ同じ画像)を除去します。こうして約1億4千万枚(LVD-142M と命名)の質の高い学習用データを構築しました。思想は「量と質の両立」です。テキストもクラスラベルも使わず、画像の特徴空間での類似度だけでキュレーションする点が特徴です。

2. 学習目的: 2つの損失を同時に使います。
   - DINO のグローバルな自己蒸留([DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) 参照): 画像全体レベルで、生徒の CLS トークン分布を EMA 教師の分布に近づける。画像全体の意味を学ぶ。
   - iBOT のパッチレベルのマスク予測([iBOT](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0060_ibot_note) 参照): マスクしたパッチの教師出力(オンライントークナイザのソフトラベル)を生徒が当てる。局所の細部を学ぶ。
   - さらに KoLeo 正則化(Kozachenko-Leonenko エントロピー推定に基づく項)を加えます。これはバッチ内の各特徴ベクトルについて「最も近い別の特徴までの距離」をなるべく大きくするよう促す項で、特徴が一部に固まらず空間を広く均等に使うことを促します。最近傍探索を使う検索タスクの性能を高めます。
   - 加えて、解像度を学習の最終段階でだけ高く(例 224 → 518)する短い高解像度学習を入れ、密な予測タスク(画素単位の予測)の性能を底上げします。最初から高解像度で全学習すると計算コストが莫大なので、最後だけ上げるのが工夫です。

3. 大規模学習の安定化と高速化: モデルを大きくする(最大 ViT-g、約11億パラメータ)と学習が壊れやすくなります。これを、FlashAttention(注意計算を省メモリ・高速にする実装)、効率的な確率的深さ(stochastic depth)、シーケンス長を揃える実装最適化、FSDP(Fully Sharded Data Parallel、モデルの重み・勾配・オプティマイザ状態を複数 GPU に分割して載せる仕組み)などで乗り越えます。最後に、巨大な ViT-g で学んだ知識を、小さなモデル(ViT-S/B/L)に蒸留して配布します。これにより、小さくて速いモデルでも巨大モデルに近い性能が得られます。論文は「ゼロから小モデルを自己教師あり学習する」より「大モデルから蒸留する」方が小モデルの性能が高いことを示しました。

出力イメージ: ある画像(例 224×224)を凍結した DINOv2-ViT-L/14 に通すと、(224/14)^2 = 256 パッチ + CLS。画像全体を要約した1本のベクトル(CLS、1024次元)と、パッチごとの特徴マップ(16×16 = 256 パッチ × 1024次元、つまり [256, 1024])が得られます。前者は分類や画像検索に、後者は深度推定やセグメンテーションのような「画素単位の予測」に使えます。tensor shape では、入力 [B,3,224,224] → CLS [B,1024] + patch tokens [B,256,1024]。

## 手法の系譜と主要論文
- 構成要素その1: Caron et al., DINO([DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note)、ICCV 2021, arXiv:2104.14294)。画像レベルの自己蒸留。
- 構成要素その2: Zhou et al., iBOT([iBOT](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0060_ibot_note)、ICLR 2022, arXiv:2111.07832)。パッチレベルのマスク予測蒸留。
- 比較相手の弱教師あり基盤モデル: Radford et al., CLIP(ICML 2021)。画像-テキスト対照学習でゼロショット分類を実現しましたが、テキストという外部教師に依存します。DINOv2 は「テキストなしの純画像学習でも CLIP に匹敵し、密なタスクでは上回れる」ことを目標にしました。OpenCLIP(Cherti et al., CVPR 2023)など大規模 CLIP 系も比較対象です。
- DINOv2 本体: Oquab et al., "DINOv2: Learning Robust Visual Features without Supervision"(2023, arXiv:2304.07193, Meta AI)。(1) 自動データキュレーションによる LVD-142M、(2) DINO + iBOT + KoLeo の学習、(3) 大規模学習の安定化・高速化、(4) 巨大モデルから小モデルへの蒸留、の総合パッケージ。動機は、テキストを使わない純画像自己教師ありで、微調整不要の汎用特徴という基盤モデルを実現すること。
- 後続の改良: Darcet et al., "Vision Transformers Need Registers"(ICLR 2024, arXiv:2309.16588, Meta)。DINOv2 の注意マップやパッチ特徴に現れる不自然な高ノルムのトークン(アーティファクト、特定のパッチが異常に大きな値を持ち、そこに画像全体の情報を詰め込んでしまう現象)を、「レジスタトークン(register token、計算用の捨て置きトークンを数個追加し、そこにモデルが余計な情報を逃がせるようにする)」を加えることで除去できると示しました。特徴マップが滑らかになり密なタスク性能が向上します。改良版 DINOv2(DINOv2 with registers)にはこの register が取り込まれています。

## 論文の実験結果(定量データ)
DINOv2 の評価は「凍結した特徴の上に最小限のヘッドを載せる」原則で行われ、その汎用性が示されました。

- ImageNet 分類(線形評価、凍結): ViT-g/14 で Top-1 約 86.5%。自己教師あり手法として当時の最高水準で、弱教師あり(テキスト併用)の OpenCLIP 系に匹敵します。微調整した教師あり ViT に肉薄する数値です。比較として、それまでの純画像自己教師ありの凍結線形評価は ViT-L 級で 80% 台前半が相場であり、大きく上回りました。
- 深度推定(凍結特徴 + 軽いヘッド): NYUv2 と KITTI(屋内/屋外の深度データセット)で、凍結 DINOv2 特徴の上に簡単な深度ヘッドを載せただけで、CLIP など他の凍結特徴を明確に上回りました。指標 RMSE(予測深度と正解の二乗平均平方根誤差、小さいほど良い)で他の自己教師あり・弱教師あり特徴より良好。画素単位の幾何的タスクで純画像学習が強いことを示す重要な結果で、テキスト学習の CLIP が苦手とする幾何情報を純画像学習がよく捉えることを意味します。
- セマンティックセグメンテーション(凍結特徴, ADE20K): 凍結特徴に線形ヘッドを載せただけの設定で mIoU(各クラスの予測領域と正解領域の重なり率の平均、高いほど良い)が CLIP 系を上回りました。バックボーンを凍結したまま、密な空間予測でここまで出るのは特徴の品質の高さの証拠です。
- 画像検索・インスタンス認識: Oxford/Paris(ランドマーク検索)などで凍結特徴のまま高い検索精度。KoLeo 正則化が特徴を空間に均等分布させた効果が出ています。
- 頑健性(robustness): ImageNet-A/R/Sketch(敵対的に選んだ難画像・スタイル変化・スケッチなどの分布外データ)でも高い精度を保ち、特徴が表面的な見た目に過学習せず汎用的であることを示しました。
- アブレーション・スケーリング: 学習データを uncurated(未選別)のまま使うと、同じデータ量でもキュレーション済み(LVD-142M)より性能が劣ることを示し、キュレーションの価値を定量化しました。また iBOT 損失や KoLeo を外すと密なタスクや検索が低下し、各構成要素の寄与を確認しています。モデルサイズ ViT-S(2100万) → B(8600万) → L(3億) → g(11億)と大きくするほど一貫して性能が上がる、きれいなスケーリング則が観察されました。
- register の効果(Darcet et al. 2024): レジスタトークン追加により、特徴マップのアーティファクトが消え、注意マップが解釈しやすくなり、密なタスク(セグメンテーション・深度)の性能がわずかに向上しました。

## メリット・トレードオフ・限界
メリット:
- 微調整なし(凍結)で多様なタスクに高性能。1つのバックボーンを何十ものタスクに使い回せる。
- テキスト等の外部教師に頼らない純画像学習で、画素レベルの密なタスク(深度・セグメンテーション)に特に強い。幾何的・空間的な手がかりを良く捉える。
- 大モデルから小モデルへ蒸留した軽量版(ViT-S/B/L)が配布され、現実的なサイズでも恩恵を受けられる。

トレードオフと限界:
- 学習に必要な計算資源とデータ量が莫大で(数千 GPU 日規模)、ゼロから再現するのは事実上困難。配布された重みを使うのが現実的。
- 配布重みは固定で、特殊ドメイン(例 車載カメラの見え方、夜間、悪天候、医療画像、衛星画像)に特化させたい場合は結局追加学習(微調整やアダプタ)が要ることがある。LVD-142M は一般的なウェブ画像中心で、特殊ドメインの分布をカバーしきれない。
- register 導入前のバージョンでは注意マップにアーティファクトが残るなど、細部の癖がある。
- データキュレーションのパイプライン自体に「お手本データセット(ImageNet 等)への偏り」が入り込む可能性があり、何が良い分布かを ImageNet が暗黙に定義してしまう循環的な限界がある。真に未知の分布への汎化は依然オープンな課題。
- テキストとの結びつきを持たないため、ゼロショット分類(言語で指定したクラスを学習なしで分類)は単体ではできず、別途言語との接続層が必要。

## 発展トピック・研究の最前線
DINOv2 の凍結特徴は、多くの応用研究のデファクトな視覚バックボーンになりつつあります。エンドツーエンドの知覚・予測パイプラインでは、凍結した視覚バックボーンの上にタスク固有のヘッド(検出・占有予測・軌道予測など)を載せる構成が一般的で、その凍結バックボーンに DINOv2 を採用する研究が増えています。深度・幾何への強さは、単眼深度推定(Depth Anything は DINOv2 を教師特徴として活用)、3D 再構成、ロボティクスの視覚基盤としても活用されています。

研究の最前線では、(1) DINOv2 特徴をテキストや言語モデルと後付けで結びつける(凍結 DINOv2 + 軽い接続層で視覚言語タスクに使う、LLaVA 系のビジョンエンコーダ選択など)流れ、(2) register の理論的理解と他モデルへの一般化、(3) 動画・3D・マルチモーダルへの自己蒸留の拡張、(4) ドメイン特化のための効率的な追加学習(LoRA などのアダプタ)、(5) より大規模・多ドメインへ拡張する後継研究(DINOv3 系の取り組み)などが活発です。DINOv2 は「純画像の自己教師あり学習だけで、テキストの助けなしに基盤モデルが作れる」ことを実証した点で、視覚基盤モデル研究の里程標になっています。
