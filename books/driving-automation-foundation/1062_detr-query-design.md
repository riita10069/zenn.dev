# DETR系クエリ設計

## ひとことで言うと
DETR(デター、DEtection TRansformer)は「画像の中に何がどこにあるか」を、たくさんの手作りルールや後処理に頼らず、Transformer という仕組みと「クエリ」と呼ばれる小さなベクトルの集合だけで一気に予測する設計です。クエリとは「この物体を担当して情報を集めてこい」という一人の探偵のようなもので、固定本数の探偵が画像を分担して見ます。この「物体をクエリ(ベクトル)で表す」という考え方は単なる検出器を超えて、検出・追跡・地図・予測・計画といった複数の処理段を一本でつなぐ共通の言葉になりました。ここではその基礎を、機械学習をまったく知らない人にも分かるところから、研究で議論できるレベルまで積み上げて説明します。

## 直感的な理解
画像から「車がここ、人がそこ」と枠を当てる作業を想像してください。素朴にやろうとすると、画像の至るところに「ここに物があるかも」という仮の枠を何万個も敷き詰め、それぞれについて当たり外れを判定し、最後に重なった枠を間引く、という手順になります。これは「とりあえず大量にばら撒いてから整理する」やり方で、ばら撒き方や整理の仕方に人間が決める設定が大量に紛れ込みます。

DETR の発想はまったく逆です。「画像にせいぜい数十個しか物はないのだから、最初から数十人の担当者(クエリ)を用意し、各自に画像を見に行かせて『私の担当は左下の車です』と報告させればよい」というものです。ばら撒きも間引きも要りません。担当者が自分で画像のどこを見るかを決め、報告を出す。この単純さが、検出を「端から端まで一本の学習」にまとめあげる鍵になりました。「端から端まで一本(end-to-end)」というのは、入力(画像)から最終出力(検出結果)までを、途中に学習できない手作業の部品を挟まずに、ひとつのニューラルネットとして勾配で最適化できる、という意味です。

## 基礎: 前提となる概念
いくつかの言葉を先に噛み砕いておきます。

物体検出とは、画像の中から「ここに車がある」「ここに人がいる」と、物体を囲む長方形(バウンディングボックス、bounding box。物体を囲む四角い枠のこと)とその種類(クラス)を当てるタスクです。

CNN(Convolutional Neural Network、畳み込みニューラルネット)は、画像から特徴を抽出する定番の仕組みです。画像を少しずつ圧縮しながら「エッジ」「模様」「物体らしさ」といった抽象的な特徴を取り出します。

Transformer は、系列(順番に並んだ要素の列)を処理するニューラルネットの一種で、アテンション(attention、注目機構)という仕組みを核にします。アテンションとは「ある要素が、他のどの要素をどれくらい重視して情報を吸い上げるか」を重み付きで決める計算です。具体的には、各要素から「問い合わせ(query)」「鍵(key)」「中身(value)」の3つのベクトルを作り、query と key の内積で「どれくらい関係が深いか」を測り、その重みで value を加重平均します。自分自身の列の中で行えばセルフアテンション、別の列を参照すればクロスアテンション(cross-attention)と呼びます。DETR でいう「クエリ」はこのアテンションの query に相当する役割を担います。

テンソル(tensor)は多次元配列のことです。本稿では `[100, 256]` のように「100行×256列の数値表」を表す記法を使います。

これらを踏まえて、DETR がどう物体検出を解くかを見ます。

## 仕組みを詳しく

### DETR 以前のやり方と、その面倒くささ
DETR 以前の主流(Faster R-CNN、YOLO など)は、おおまかに次の手順でした。

1. 画像の至るところに、あらかじめ決めた大量の候補枠(アンカー、anchor。「ここに物があるかもしれない」という当たりをつけた仮の枠)を敷き詰める。1枚の画像で数千〜数万個。
2. 各候補枠について「物があるか/何のクラスか/枠をどうずらすか」を予測する。
3. 同じ物体に重複した枠が大量に出るので、NMS(Non-Maximum Suppression、ノンマキシマム・サプレッション。重なった枠のうちスコア最大の1つだけ残し他を消す後処理)で間引く。

問題は、アンカーの大きさ・縦横比・密度、NMS のしきい値といった人間が手で決める設定(ハイパーパラメータ)が大量にあり、データセットごとに調整が要ることでした。さらに NMS は微分できない(学習で勾配を流せない)操作なので、検出器全体を一本の学習で最適化できず、パイプラインが分断されていました。微分できないとは、出力を少し変えたときに損失がどう変わるかという「傾き」が定義できず、誤差逆伝播(出力の誤りを入力側のパラメータへ伝えて更新する仕組み)で学習信号を流せない、ということです。

### クエリとは何か
DETR でいうクエリ(オブジェクトクエリ、object query)は、長さ256くらいの数値ベクトルです。DETR では例えば100本用意します。形で書くと `[100, 256]` のテンソルです。

重要なのは、この100×256個の数値が「学習可能なパラメータ」だという点です。最初はランダムですが、学習が進むと各クエリが「画像の左下の小さい物体を探す担当」「中央の大きい物体を探す担当」のように、それぞれ役割を自然に獲得します。誰も手で割り振らないのに、データから分業が立ち上がります。これは可視化研究で実際に確認されており、原論文の付録では各クエリが出力した枠の中心座標を散布図にすると、クエリごとに特定の位置・スケール帯に偏った分布を示すことが報告されています。つまり100本それぞれが暗黙の「縄張り」を持つわけです。

### 処理の流れ(ステップ分解)
入力画像を1枚 `[3, 800, 1200]`(3チャンネル=RGB、高さ800、幅1200ピクセル)とします。

1. CNN(バックボーン、典型的には ResNet-50)で画像を特徴マップ `[2048, 25, 38]` 程度に圧縮する。
2. 1×1畳み込みでチャンネルを256に落とし、空間方向を並べ直して `[950, 256]`(950 = 25×38、各位置を1トークンとみなす)の系列にする。これに位置エンコーディング(各トークンが画像のどこかを示す情報)を足し、Transformer エンコーダで自己注意を数層かけて画像内の関係を整理する。これがデコーダから参照されるキー/バリュー(参照される情報)になる。
3. 100本のオブジェクトクエリ `[100, 256]` をデコーダに入れる。デコーダ内で各クエリは、まずクエリ同士でセルフアテンション(「君が左の車なら、私は右の車を担当しよう」という重複回避の相談)を行い、続いて画像特徴 `[950, 256]` にクロスアテンションして自分の担当領域の情報を吸い上げる。これを6層繰り返す。
4. 各クエリの出力ベクトル `[256]` を、小さな全結合層(FFN、Feed-Forward Network)に通して「枠の座標4つ(中心x, 中心y, 幅, 高さ)」と「クラス(車/人/… + 特別な『物体なし(no-object, ∅)』クラス)」に変換する。
5. 結果として `[100, 4]` の枠と `[100, クラス数+1]` のスコアが一発で出る。NMS 不要。

```
画像 [3,800,1200]
  └─ CNN ─> 特徴 [2048,25,38] ─1x1conv+flatten─> トークン列 [950,256]
                                                      │ (key/value)
クエリ [100,256] ─> Transformer Decoder(self-attn + cross-attn ×6) ─> [100,256]
                                                      │
                                          FFN ─> 枠 [100,4] + クラス [100,C+1]
```

デコーダを6層重ねる理由は、各層が「アテンションで情報を集める→自分の枠の推定を少し直す」を繰り返し、層を経るごとに枠が正解へ収束していくからです。原論文は各層の出力にも損失を掛ける補助損失(auxiliary loss)を入れており、これが収束を助けることを報告しています。

### 集合予測とハンガリアンマッチング
100個の予測と正解(例えば画像に車3個)をどう対応づけて学習するか。順序に意味のない「集合」を直接出すこの問題設定を集合予測(Set Prediction、セット予測)と呼びます。「集合」とは要素の並び順に意味がない集まりのことで、検出結果は「左の車・右の人」と並べても「右の人・左の車」と並べても同じ意味なので、順序付きの系列予測とは性質が違います。DETR はハンガリアン法(Hungarian algorithm、2つの集合の要素を1対1で総コスト最小に割り当てる古典アルゴリズム、計算量はおよそ要素数の3乗)で、100個の予測と3個の正解を最適に1対1対応させます。対応した予測には「正解の枠とクラスに近づけ」、残り97個には「物体なし ∅ と答えよ」と教えます。

マッチングのコストは、クラス確率と枠の一致度(L1距離 + GIoU、Generalized IoU。枠の重なり具合を測る指標で、まったく重ならない場合も連続的に値が変わるよう通常の IoU を拡張したもの)を組み合わせて計算します。この「1正解につき必ず1予測が割り当たる」という制約こそが、重複枠を構造的に防ぎ、NMS を不要にしている本質です。NMS という後処理で重複を消すのではなく、損失の設計レベルで「1物体=1予測」を強制しているわけです。逆に言えば、もし2つのクエリが同じ物体を当てると、片方しか正解に割り当てられず、もう片方は「∅と答えよ」と罰せられるので、自然に役割分担が学習されます。

### クエリ受け渡しでモジュール境界を置き換える
DETR が真に効いたのは、検出を超えて「処理段の接続様式」を変えた点にあります。従来のシステムは「検出 →(枠の数値リストを渡す)→ 追跡 →(IDのリストを渡す)→ 予測 → 計画」のように、各段の間で人間が定義した中間表現(枠の座標、トラックID等の数値リスト)をやり取りしていました。この中間表現は情報を落とします。枠だけ渡すと、その物体の見た目・姿勢・周囲との関係といった文脈が失われます。

DETR 系の設計では、各物体を「クエリ(リッチな256次元ベクトル)」として表し、それをそのまま次段へ渡します。すると後段は、枠の数値だけでなく「その物体が何で、どう動いていて、周囲とどう関係するか」を含んだ豊かな表現を受け取れます。統合型の自動運転モデルは、この「クエリを共通通貨にして処理段を数珠つなぎにする」設計を全タスクに広げました。研究コミュニティではこの様式を「query-based」「クエリの伝播(query propagation)」と呼びます。重要なのは、中間表現を数値リストではなく学習可能なベクトルにすることで、後段の損失から前段へも勾配が流れ、システム全体が最終目的に向けて協調学習できるようになる点です。

## 手法の系譜と主要論文

- Carion, Massa, Synnaeve, Usunier, Kirillov, Zagoruyko, "End-to-End Object Detection with Transformers (DETR)", ECCV 2020(Facebook AI、現 Meta AI、arXiv:2005.12872)。アンカーと NMS を廃し、学習可能クエリ + Transformer + ハンガリアンマッチングによる集合予測で検出を end-to-end に学習する元祖。手設計の後処理がパイプラインを分断し調整コストを生んでいた問題への回答。

- Zhu, Su, Lu, Li, Wang, Dai, "Deformable DETR", ICLR 2021(arXiv:2010.04159)。クエリが画像全体ではなく、少数の参照点(reference points)まわりの数点だけに注目する「変形可能アテンション(deformable attention)」を導入。DETR の全画素アテンションが計算量大(系列長の2乗に比例)かつ収束が遅かった問題への対処。さらに複数解像度の特徴(multi-scale)を扱えるようにし、小物体検出を改善した。

- Meng et al., "Conditional DETR"(ICCV 2021、arXiv:2108.06152)、Liu et al., "DAB-DETR"(ICLR 2022、arXiv:2201.12329)。クエリに明示的な空間情報(参照点や4次元のアンカーボックス x,y,w,h)を持たせて収束を速める一連の改良。元の DETR のクエリは中身が抽象的すぎて「どこを見るべきか」を学ぶのに長い時間がかかっていた。DAB-DETR はクエリを「動的アンカーボックス」として解釈し直し、枠のサイズに応じてアテンションのガウシアン的な広がりを調整することで、クロスアテンションを空間的に絞り込んだ。

- Li et al., "DN-DETR"(CVPR 2022、arXiv:2203.01305)。学習時にノイズを加えた正解枠をデコーダに与えて「ノイズ除去(denoising)」を補助課題として課し、ハンガリアンマッチングの不安定さ(エポックごとに割り当て先が変わって学習が揺れる問題)を緩和して収束を高速化。マッチングを介さず正解と直結した学習信号を追加で与えるのが要点。

- Zhang et al., "DINO: DETR with Improved deNoising anchOr boxes"(ICLR 2023、arXiv:2203.03605)。DAB + DN を統合し、対比的ノイズ除去(contrastive denoising、正例と負例のノイズ枠を対にして「どこまでが同じ物体か」の境界を鋭くする)や mixed query selection を加え、COCO 検出で当時の最先端を達成。DETR 系がアンカーベース手法を性能でも上回った転換点。

- Hu, Yang, Chen, et al., "Planning-oriented Autonomous Driving (UniAD)", CVPR 2023(best paper、arXiv:2212.10156)。検出・追跡(TrackFormer)・オンライン地図(MapFormer)・動作予測(MotionFormer)・占有予測・計画を、すべてクエリの受け渡しで一本につなぐ。各段の出力クエリが次段の入力クエリになる。最終目的の「計画(どう走るか)」に向けて全タスクを協調させ、中間表現の情報損失を避ける狙い。DETR のクエリ思想を自動運転の統合アーキテクチャに昇華した代表例。

- Wang et al., "DETR3D"(CoRL 2021、arXiv:2110.06922)、Cheng et al., "Mask2Former"(CVPR 2022、arXiv:2112.01527)。前者はカメラ画像群から3次元の枠をクエリで直接予測(2D 特徴へ逆投影してサンプリング)、後者は検出・インスタンス・パノプティックの各セグメンテーションを「マスクをクエリで予測する」一枚岩の設計に統一。クエリ設計が2D 検出の枠を超えて汎用の予測言語になったことを示す。

## 論文の実験結果(定量データ)

DETR(2020)は COCO データセット(約12万枚の画像に物体を注釈した検出ベンチマークの定番)で評価されました。指標は AP(Average Precision、平均適合率。検出の正確さを0〜100で測り、高いほど良い。適合率と再現率のトレードオフ曲線の下面積を、複数の重なり判定しきい値で平均したもの)です。報告では ResNet-50 バックボーンで AP 42.0 を達成し、同条件の Faster R-CNN(おおむね AP 40前後)と同等以上でした。とくに大きい物体での AP(AP_L)は Faster R-CNN を明確に上回りましたが(報告でおよそ +7〜8 ポイント)、小さい物体(AP_S)は逆に数ポイント劣りました。これは全画素アテンションの実効解像度が低く、小さな物体の細部を拾いにくいためと説明されています。

最大の弱点は学習の遅さで、DETR は500エポック(データセットを500周)という長大な学習を要しました。Deformable DETR(2021)はこれを約50エポック、つまり約10分の1に短縮し、同時に AP も約43〜46へ改善、小物体 AP_S も大きく持ち上げました。変形可能アテンションが計算を参照点近傍に絞ることで、収束の速さと小物体性能の両方を改善した、というのがアブレーション(要素を抜き差しして寄与を測る実験)の結論です。multi-scale 特徴を外すと小物体性能が落ちることも示されています。

DN-DETR(2022)はノイズ除去補助課題により、同じエポック数での AP を約1.5〜2ポイント押し上げ、学習曲線の初期立ち上がりを速めました。DINO(2023)は ResNet-50・約12エポックで AP 49前後、Swin-L 大型バックボーンでは COCO test-dev で AP 63超を報告し、DETR 系が後処理ベースの強力な検出器を性能でも追い抜いたことを示しました。注目すべきは、DETR が500エポックかかっていた水準を、改良版が12エポック程度で大きく超えた点で、これはクエリへの空間事前情報の付与とマッチング安定化が「収束の遅さ」という DETR 最大の弱点を解消したことを意味します。

統合型自動運転モデル(UniAD、2023)は nuScenes(自動運転の大規模マルチセンサデータセット、約1000シーン)で評価され、検出・追跡・地図・予測・計画を個別に最適化した構成と比べ、計画の指標で改善を報告しました。具体的には、計画軌跡の平均変位誤差(L2、予測軌跡と正解の距離、メートル単位)が複数手法より低く、衝突率(collision rate、計画軌跡が他物体とぶつかる割合)も低減したと報告されています。報告された平均 L2 はおよそ1メートル前後、衝突率は1パーセント未満の水準で、タスクを分離した構成より良好でした。アブレーションでは、上流タスク(追跡・地図・予測)を順に外していくと計画性能が段階的に悪化することが示され、「クエリで全タスクをつなぎ計画に奉仕させる」設計の有効性が裏づけられました。一方で、この恩恵は学習の組み立て(段階的な事前学習)に強く依存することも報告されており、単純に全部を同時学習すると不安定になります。なお後続研究では UniAD の計画評価指標の妥当性(自車の過去運動だけで衝突率がほぼ説明できてしまう等)に批判も出ており、評価方法論自体が研究対象になっています。

## メリット・トレードオフ・限界

メリット側。アンカー・NMS など手設計の後処理を排除でき、検出を端から端まで一本で最適化できます。各物体を豊かなベクトル(クエリ)で表すため、後段への情報損失が少なく、処理段の接続の共通言語になります。集合予測なので重複出力が構造的に出ません。

トレードオフと限界。元祖 DETR は収束が極端に遅く学習コストが高い(改良版で大幅緩和)。小さい物体に弱い傾向があり、これは実効解像度とクエリ本数の制約に起因します。ハンガリアンマッチングはエポックごとに割り当てが変わるため学習が不安定になりやすく(DN-DETR がこの問題に取り組んだ)、いまも研究テーマです。クエリの中身は人間に解釈しにくく、デバッグや安全性検証が難しいという根本課題は残ります。固定本数のクエリ(例100本)を超える数の物体が同時に存在する稠密シーンでは取りこぼしが起き、本数を増やすと計算が重くなるというトレードオフもあります。さらに、ハンガリアンマッチングは1対1割り当てゆえに、正解に割り当たる予測が1個に限られ「正例が少なく学習信号が薄い」問題があり、近年の H-DETR や Group DETR は学習時だけ1対多割り当てを併用してこれを補っています。

## 発展トピック・研究の最前線
DETR 系は静止画検出からセグメンテーション(Mask2Former はマスク予測をクエリで統一)、3D 検出(DETR3D、PETR はカメラ画像から3次元枠をクエリで直接予測)、動画の物体追跡(TrackFormer、MOTR はトラックをクエリの時間伝播として表現し、前フレームの物体クエリを次フレームへ持ち越すことで ID を維持)へ広がりました。これらに共通するのは「予測対象を固定本数のクエリの集合として表し、損失をマッチングで定義する」という一貫した設計言語です。

自動運転では、クエリを空間グリッド上の鳥瞰図特徴(BEV、Bird's-Eye-View)に対するクロスアテンションで運用する流れが主流化しました(BEVFormer はその代表で、BEV グリッドの各セルを学習可能クエリとして周囲カメラの特徴を集約します)。行動(軌跡)生成側でも、軌跡を「行動トークンのクエリが空間特徴を参照して情報を吸い上げる」形で構成する設計が広く使われており、これは DETR のクロスアテンション思想と地続きです。未解決の最前線としては、クエリの解釈可能性と安全性保証、可変数の物体への対応、マッチングの安定化と学習信号の密化、そして検出から計画までを真にひとつの目的関数で一貫学習する end-to-end 化の信頼性向上が挙げられます。
