# マルチフレームBEV特徴の時間融合

## ひとことで言うと

カメラ映像から作る「真上から見た平面図」（BEV: Bird's-Eye-View、鳥の目線で地面を見下ろした俯瞰図）の特徴を、1フレーム（1時刻ぶんの画像のまとまり）ずつ独立に処理するのではなく、「少し前の時刻に見たBEVの情報」を現在の予測に持ち込む仕組みの総称です。過去を覚えておくことで、物の動き（速度）や、いったん物陰に隠れた対象を扱えるようになります。BEVの時間融合は、3D物体検出・地図認識・軌跡予測・end-to-end運転のすべてで標準的な構成要素になっています。

## 直感的な理解

運転は本質的に時間の流れの中で起きます。前の車が右に動いているか左に動いているかは、1枚の写真では分かりません。2枚以上を見比べて初めて「動いている方向と速さ」が分かります。スポーツ中継の一時停止画像を見て、ボールが飛んでいるのか止まっているのかを当てるのが難しいのと同じです。物理的に言えば、速度は位置の時間微分なので、最低2時刻の観測がないと推定不可能です。1フレームの入力からは原理的に速度は出てきません。

もう一つの例が「隠れ（オクルージョン、occlusion）」です。大型トラックの陰に歩行者が一瞬隠れたとき、人間は「さっきまでそこにいたから、まだそこにいるはず」と推測します。これは過去の記憶があるからこそできる判断です。1フレームだけを見るモデルは、隠れた瞬間に歩行者を見失います。

時間融合は、この「過去の記憶」をモデルに持たせる仕組みです。ナイーブに過去の生画像を全部覚えるのは重すぎる（カメラ6台×数フレーム×高解像度を毎回処理し直すことになる）ので、研究の主流は「すでに処理済みのBEV特徴マップを覚える」方向に進みました。BEV特徴は、シーンを構造化した中間表現なので、過去の処理結果を無駄にせず積み上げられます。

## 基礎: 前提となる概念

用語を噛み砕きます。

- フレーム: ある時刻の入力画像一式のこと。複数カメラなら複数枚で1フレームです。nuScenes ではキーフレームが2Hz（0.5秒間隔）、生のカメラは12Hz程度です。
- BEV特徴マップ: カメラ画像をBEV格子（真上視点の格子）に並べ直し、各マスに特徴ベクトル（たとえば256次元）を持たせたテンソル。形状は `[B, C, H, W]`（B=バッチ、C=チャネル、H・W=格子の縦横、たとえば200×200）。
- 時間融合（temporal fusion）: 過去フレームの情報と現在の情報を混ぜ合わせること。
- self-attention（自己注意）: 複数の要素が互いに「どれをどれだけ参照するか」を学習して重み付けし、情報を混ぜる仕組み。時間方向に広げると、現在のBEVマスが過去のBEVマスを参照できます。
- 自車運動整合（ego-motion alignment / compensation）: 自車（ego）が動いたぶん、過去のBEVの座標をずらして現在の自車中心座標に合わせる処理。時間融合で最も重要な前処理です。
- オドメトリ（odometry）: 自車がどれだけ移動・回転したかの推定値。車輪の回転やIMU（慣性計測装置）、GNSS（衛星測位）から求めます。整合にはこのオドメトリ（時刻t−1からtへの剛体変換行列、並進3＋回転3の6自由度）が必須です。
- 疎な表現と密な表現: 密（dense）はBEV格子全マスに特徴を持たせる方式、疎（sparse）は物体がありそうな少数の点（クエリ）だけに特徴を持たせる方式。後者は軽い反面、空きマスの情報を落とします。

なぜ整合が決定的に重要かを直感で説明します。前フレームのBEVは「0.5秒前の自車を中心にした座標」で描かれています。自車はその間に前進・旋回しているので、そのまま現在のBEVに重ねると、地面も建物も静止物まで全部ずれて見えます。整合をしないと、自車自身の動きと物体の動きが区別できなくなり、速度推定が破綻します。逆に整合を正しく行えば、静止物はぴったり重なり、動く物だけがずれる、という理想的な入力が作れます。

## 仕組みを詳しく

時間融合の一般的な流れを、手法によらない共通の骨格として示します。

1. 過去BEVの保存: 前フレームで作ったBEV特徴を記憶します。形状は典型的に `[B, C, H, W]`（たとえば `[B, 256, 200, 200]`）。手法によって「直前1枚だけ」か「直近数枚のキュー」か「再帰的に積み上げた1枚」かが変わります。

2. 自車運動整合: 前フレームと現在フレームで自車が動いているので、オドメトリから求めた並進・回転の変換行列を使って、過去のBEVを現在の自車中心座標へ座標変換します。実装は、現在BEVの各マスの実世界座標を求め、その点を過去座標系へ写す逆変換を掛け、過去BEV上の対応する位置を双線形補間（PyTorchの `grid_sample` など）で取り出す、という流れが標準です。

   ```
   前フレームBEV(過去の自車中心) ──ego運動で座標変換──> 現在の自車中心に整合
   ```

3. 融合: 整合した過去BEVと現在BEVを混ぜます。混ぜ方が手法ごとの違いです。
   - (a) 時間方向 self-attention（BEVFormer型）: 現在のBEVクエリが過去BEVを参照する。表現力が高い。
   - (b) チャネル方向に連結してConv（BEVDet4D型）: 単純で軽量。
   - (c) 物体クエリだけ伝播（StreamPETR / Sparse4D型）: 密な特徴マップでなく疎な物体クエリを伝える。メモリ効率が高い。
   - (d) 長期の再帰メモリ（VideoBEV型）: 多数フレームの文脈をメモリ爆発なく保持。
   - (e) 長期コストボリューム（SOLOFusion型）: 多フレームの視差から深度そのものを直接改善。

4. 下流タスク: 融合後のBEVを使って3D検出・地図認識・軌跡予測などを行います。

数値例で効果を見ます。ある車が0.5秒で1.0m前進していたとします。
- before（時間融合なし）: 現フレームの位置しか分からず、静止（速度ゼロ）と区別できない。
- after（時間融合あり）: 過去の位置と現在の位置の差から「秒速2m前進」と推定できる。これが速度推定です。

設計上の根本的なトレードオフは「何をどれだけ覚えるか」です。フル解像度のBEV特徴マップを覚えれば表現力は高いがメモリを食う。疎な物体クエリだけ覚えれば軽いが空間情報が落ちる。短期（2フレーム）なら軽いが長い隠れに弱い。長期なら強いが複雑でメモリ管理が要る。どの軸を取るかが手法選択の本質です。この4軸（密/疎、短期/長期、attention/conv、特徴マップ/クエリ）の組み合わせで、ほとんどの手法を整理できます。

## 手法の系譜と主要論文

BEV時間融合の系譜を時系列で描きます。

- BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）: 前フレームのBEV特徴マップを時間 self-attention で現在のBEVクエリに融合する時空間トランスフォーマー。1フレームずつ再帰的に過去を伝えます（recurrent）。表現力が高く実績豊富で、後述のUniADが基盤として採用しました。トレードオフはフル特徴マップの保存コストと、deformable attention（注目点をずらしながら見る効率的な注意機構）の実装の重さ、自車運動データへの依存です。詳細は [BEVFormer型 再帰的BEV融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0481_bevformer-recurrent-fusion)。

- BEVDet4D（Huang & Huang, 2022, arXiv:2203.17054）: 自車運動整合後に過去と現在のBEVをチャネル連結し、Convで融合する軽量手法。注意機構を使わず実装が単純です。「整合の有無で結果が激変する」という重要な教訓を残しました。トレードオフは基本2フレーム限定で長期履歴に弱いこと。詳細は [BEVDet4D型 特徴キュー連結](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0480_bevdet4d-queue-concat)。

- SOLOFusion（Park et al., ICLR 2023, arXiv:2210.02443）: 「隣接2フレームは短すぎる」と主張し、長期（最大16フレーム程度）の画像観測からコストボリューム（複数視点の画像特徴を様々な深度で照合した類似度のかたまり）を作り、時間視差で深度推定を改善する手法。融合の目的が「速度」ではなく「深度精度」である点で他と補完的です。さらに「短期・高解像度」と「長期・低解像度」を組み合わせると効率が良いという設計指針を示しました。トレードオフは長期フレームを保持・処理する計算コスト。

- StreamPETR（Wang et al., ICCV 2023, arXiv:2303.11926）: 密なBEV特徴マップではなく、軽量な物体クエリだけを「運動を考慮した層正規化（motion-aware layer normalization、自車運動と経過時間でクエリを補正）」とともにフレーム間伝播させる手法。メモリ効率が非常に高く、長期の時間モデリングを安価に実現します。トレードオフは物体中心の疎なクエリが前提で、密な空間BEVクエリとは構造が異なること。

- Sparse4D / Sparse4Dv2（Lin et al., 2022-2023, arXiv:2211.10581 / arXiv:2305.14018）: BEV格子そのものを作らず、3D anchor を画像の複数視点・複数時刻にサンプリングして集約する完全に疎な4D検出器。BEVを経由しないことでメモリと計算を抑えつつ、時間集約で高精度を出します。「密なBEVは本当に必要か」という問い直しの代表です。

- VideoBEV（Han et al., 2023, arXiv:2303.05970）: BEVFormer型の「1フレーム再帰」では過去情報が世代を経るごとに薄れる弱点を指摘し、temporal embedding で長期文脈をメモリ爆発なく多数フレームに保持する再帰融合を提案。BEVFormer型の限界を補う位置づけです。

- UniAD（Hu et al., CVPR 2023, arXiv:2212.10156）: 検出・追跡・地図・予測・計画を1つに統合した「Planning-oriented」な大型モデル。BEVエンコーダとしてBEVFormerをそのまま採用し、時間融合済みBEVを全タスクの共通土台にしました。CVPR 2023 Best Paper。これは「時間融合BEVが下流タスク統合の基盤になる」ことを示した代表例です。

系譜を一言でまとめると、「フル特徴マップを覚える表現力重視（BEVFormer）」と「軽さ・疎さ・長期効率を追う改良（BEVDet4D / StreamPETR / Sparse4D / SOLOFusion / VideoBEV）」が並走し、それらが統合フレームワーク（UniAD）の基盤として収束していった、という流れです。近年は「そもそも密なBEVを持たない疎なクエリ伝播」が効率面で優勢になりつつあります。

## 論文の実験結果（定量データ）

時間融合の効果は、主に nuScenes ベンチマークの速度推定誤差と検出精度で測られます。指標の意味を添えます。

- mAVE（mean Absolute Velocity Error）: 検出した物体の速度ベクトルの平均誤差（m/s）。低いほど良い。時間融合が最も劇的に効く指標です。1フレームのみのBEVDetはmAVEがおよそ0.9 m/s前後と高い（速度をほぼ当てられない）のに対し、時間融合を入れたBEVDet4DはmAVEをおよそ0.3〜0.4 m/s台へと半分以下に下げたと報告されています。時間情報なしには速度は原理的に推定できないので、この差は本質的です。秒速0.9mの誤差は時速にして約3km/hの誤差であり、追従制御や衝突予測には致命的なレベルです。

- NDS（nuScenes Detection Score）: 検出精度（mAP）・位置誤差・サイズ誤差・向き誤差・速度誤差・属性誤差を総合した0〜1の指標。高いほど良い。mAVE はこの構成要素の一つなので、速度推定の改善はNDS全体を直接押し上げます。BEVFormerは時間 self-attention を入れることでNDSを単一フレーム版から数ポイント改善し、test セットでNDS 56.9%を達成しました。アブレーションで時間 self-attention を外すとmAVEが大きく悪化し、時間情報の寄与が定量的に確認されています。

- SOLOFusion（ICLR 2023）: 長期フレーム融合により、nuScenes test でNDS 54%超を達成し、当時のカメラのみ手法で高水準を記録。短期融合（2フレーム）に対し、フレーム数を2→8→16と増やすほど検出・速度推定が改善することをアブレーションで示しました。深度推定の改善が効いており、mATE（位置誤差）の低下が顕著です。

- StreamPETR（ICCV 2023）: クエリ伝播ベースの時間融合で、nuScenes test でNDS 63%超（強いバックボーンと追加学習込み）を達成。密なBEV特徴を保持せずにこの精度を出した点が、メモリ効率の高さを示しています。長期のクエリ伝播がmAVEと検出精度の両方を改善することを報告しました。8フレーム程度のクエリ伝播でも、密なBEV方式より大幅に軽い計算で同等以上の精度が出ます。

これらに共通する定量的教訓は、「速度推定（mAVE）は時間融合の有無で2倍以上変わる」「自車運動整合を外すと整合ありに対して性能が大きく劣化する」という2点です。後者はBEVDet4Dのアブレーションが最も明確に示しており、整合なしでは時間融合の利得がほぼ消えると報告されています。また「フレーム数を増やすほど伸びるが、収穫逓減があり、長期化はコストと精度のトレードオフ」という点も複数論文で一致しています。

## メリット・トレードオフ・限界

メリット:
- 物体の速度推定が可能になる（同じ物を複数フレームで見るため）。原理的に1フレームでは不可能なことが可能になる。
- 隠れ（オクルージョン）への耐性。少し前に見えていた物を覚えていられる。
- 軌跡予測の改善。動きの傾向を時間で捉えられる。
- BEV特徴の安定化。時間方向の平滑化でフレームごとのノイズ（フリッカー、検出のちらつき）が減り、追跡が安定する。
- 深度推定の改善（SOLOFusion型）。複数時刻の視差は単眼の深度推定より信頼性が高い。

トレードオフと限界:
- 自車運動データ（オドメトリ）が必須。整合を誤ると静止物まで動いて見え、逆効果になる。低速・停止時はオドメトリ誤差の相対影響が大きくなる。
- 過去フレームの保存ぶんメモリと計算が増える（手法により差が大きい）。密なBEVマップを多数保持すると数GB規模になることもある。
- アーキテクチャ変更の規模が手法依存。クエリ伝播型は構造変更が大きい。
- ストリーミング推論との整合。学習時は固定長シーケンス、推論時は無限ストリームなので、再帰状態の初期化やリセットの扱いに注意が要る。
- 未解決の課題: 長期（数秒以上）の文脈を効率よく保持しつつ、誤った過去情報を「忘れる」仕組み（誤検出の自己訂正、追跡のドリフト抑制）は依然として難しい。動的に変化するシーンで過去BEVをどこまで信用するかの不確実性扱いも研究途上です。

## 発展トピック・研究の最前線

- クエリベース統合（疎な表現）への移行: 密なBEVグリッドを保持するコストを避け、StreamPETR / Sparse4D のように疎な物体クエリで時間を扱う流れが強まっています。
- 時空間の同時モデリング: 検出だけでなく、未来予測（occupancy flow forecasting、各マスの占有が将来どう動くか）まで一体で行うモデル。過去から未来へ時間を双方向に扱います。
- 4D Occupancy: 3D占有グリッドに時間軸を加え、各voxelの占有と運動（flow）を同時に予測する研究。
- end-to-end運転との統合: UniAD系のように、時間融合BEVを検出・予測・計画の共通基盤とし、計画性能を最終指標に据えて全体を最適化する流れ。
- 世界モデル（world model）との接続: 過去BEVから未来BEVを生成・予測する生成モデル（GAIA、DriveDreamer 系）で、時間融合を「予測・想像」へ拡張する研究。
