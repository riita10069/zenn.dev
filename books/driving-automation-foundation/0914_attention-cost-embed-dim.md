# 埋め込み次元と注意機構の計算コスト

## ひとことで言うと
ニューラルネットが扱う「特徴ベクトルの長さ(埋め込み次元 embed_dim、記号では d)」を大きくすると、注意機構(attention)の計算コストはその2乗に比例して増えます。たとえば d を 256 から 1440 へ上げると、注意の中核である線形射影の計算とパラメータが約 31 倍(1440² / 256² ≒ 31.6)になります。だから「表現力を上げたいから次元を大きく」と安易にすると、推論が劇的に遅くなりメモリも食います。多くの実用設計が d=256 前後を選ぶのは、この2乗コストとのバランスゆえです。

## 直感的な理解
注意機構を「会議で誰の発言を重視するか決める仕組み」だと考えてください。各参加者(トークン)は「自分の意見(特徴ベクトル)」を持っています。注意は「自分が今知りたいこと(query)」と「各参加者が提供できる情報(key)」を照らし合わせ、関連の強い相手の発言(value)を重く取り込みます。

このとき各参加者の意見ベクトルが長い(d が大きい)ほど、意見を query/key/value に変換する処理が重くなります。変換は行列の掛け算で、その行列は d×d のサイズを持つので、d を2倍にすると行列の要素数(=計算量)は4倍になります。d を 256 から 1440 へ約 5.6 倍にすると、5.6² ≒ 31 倍です。会議の人数(トークン数)を増やしていないのに、一人ひとりの意見を長くしただけで処理が 31 倍になる、というのが直感です。

車載の自動運転モデルは数十ミリ秒で1フレームを処理しなければなりません。同じ精度なら軽い方が望ましく、無駄に大きい次元はレイテンシとメモリを食うだけです。だから「どこまで次元を削れるか」は実用上きわめて重要な設計判断になります。表現力と速度はトレードオフですが、後述のスケーリング則が示すとおり、次元を増やして得られる精度は次元に対して劣線形(収穫逓減)なのに、コストは2乗で増えます。この非対称性こそが「むやみに次元を上げない」規律の根拠です。

## 基礎: 前提となる概念
- 埋め込み次元(embedding dimension, embed_dim, d): 特徴を表すベクトルの要素数(長さ)です。各画素や各トークンを d 個の数字で表します。長いほど表現力は上がりますが、計算もメモリも増えます。モデルの「幅(width)」とも呼ばれます。

- 線形層(Linear / 全結合層): 入力ベクトルに重み行列を掛けて出力ベクトルを作る層です。入力次元 d_in、出力次元 d_out なら、重み行列は d_in × d_out 個の要素を持ちます。入力も出力も d なら d×d = d² 個です。掛け算の回数(計算量)も、トークン1個あたり概ね d² に比例します。

- FLOPs(浮動小数点演算回数): 計算量の指標で、掛け算と足し算の総回数です。小さいほど速い・軽い。レイテンシ(処理時間 ms)と概ね相関しますが、メモリ帯域やキャッシュの効き方で実測は乖離することもあります。

- 注意機構(attention): 多数の要素の中から「今どれを重視すべきか」を学習する仕組みです。各要素を Query(Q、何を探すか)、Key(K、何を提供できるか)、Value(V、提供する中身)の3つに変換し、Q と K の内積で「関連度スコア」を計算し、softmax(合計が1になるよう正規化)で重みにして、それで重み付けした V の和を出力します。Self-Attention は同じ集合の中での参照、Cross-Attention は「ある集合(query 側)が別の集合(key/value 側)を参照する」注意です。複数カメラの視点を融合する場面では Cross-Attention がよく使われます。

- トークン(token): 注意が扱う1つ1つの要素です。文章なら単語、画像なら画素やパッチ、複数視点融合ならカメラごとの特徴です。その個数を N と書きます。

- マルチヘッド注意(Multi-Head Attention, MHA): d 次元を h 個のヘッドに分け、各ヘッドが d/h 次元で独立に注意を計算してから結合する仕組みです。次元を増やさずに「異なる観点の注意」を並列に持てるため、表現を多様化する安価な工夫として広く使われます。

## 仕組みを詳しく
注意のコストが d の2乗に比例する理由を分解します。注意の中心は4つの線形変換です(入力 X のトークン数 N、次元 d)。

```
Q   = X · W_Q    (W_Q は d×d)
K   = X · W_K    (W_K は d×d)
V   = X · W_V    (W_V は d×d)
out = Attn · W_O (W_O は d×d)
```

W_Q, W_K, W_V, W_O はいずれも d×d の重み行列で、合計 4·d² 個のパラメータを持ちます。トークン数 N に対し、これら射影の計算量は O(N·d²) のオーダーです。d を 256 から 1440 へ上げると d² は次のように変わります。

```
256²  =    65,536
1440² = 2,073,600
比     = 2,073,600 / 65,536 ≒ 31.6 倍
```

つまりトークン数が同じでも、埋め込み次元を 1440 にするだけで射影部分の計算とパラメータが約 31 倍になります。

連結融合の例でも同じ比が現れます。複数視点を連結してから 1×1 畳み込みで融合する設計を考えます。`Conv1×1(num_views × embed_dim, embed_dim)` の重みは「入力チャンネル × 出力チャンネル」なので、

```
embed_dim = 1440, num_views = 8:  8 × 1440 × 1440 ≒ 16.6M パラメータ
embed_dim =  256, num_views = 8:  8 ×  256 ×  256 ≒  0.52M パラメータ
```

256 にするだけでこの1層が約 32 分の1 に縮みます(やはり d² 比 31.6 に一致)。

注意機構にはもう1つコスト項があります。トークン同士の総当たりで作るスコア行列(N×N)に由来する O(N²·d) の項です。これは N(視点や画素の数)に効く別軸のコストで、d とは独立に効きます。整理すると、

```
射影コスト:    O(N · d²)   ← d を大きくすると2乗で効く
スコア行列:    O(N² · d)   ← N を大きくすると2乗で効く
```

どちらが支配的かは N と d の大小で決まります。閾値は概ね N ≈ d のときで、N < d なら射影(d²)が、N > d ならスコア行列(N²)が支配します。実務では、画素単位の注意(N が画素数で巨大、たとえば 64×64=4096)では N² が支配的、視点融合のように N が小さく(たとえば N=64)d が大きいと d² が支配的、というように、どちらがボトルネックかは設定で変わります。埋め込み次元削減が効くのは後者の状況で、画素単位注意では d を削るより N を削る工夫(スパース注意・トークン削減)の方が効きます。

数値例で before/after を見ます。N=64(8視点 × 8トークン相当)、Multi-Head Attention の場合:

```
before: d=1440 → 射影 4·d² = 約 8.3M パラメータ/層、計算は N·4·d² に比例
after : d= 256 → 射影 4·d² = 約 0.26M パラメータ/層
削減率: 約 31.6 分の1
```

メモリの観点も重要です。注意の中間で作るスコア行列は N×N×h(ヘッド数)の大きさを持ち、これが活性化メモリを食います。さらに各射影の重みと出力(N×d)もメモリを使います。d を削ると重みと出力テンソルのメモリが d 比(線形)〜 d² 比(重み)で減るため、レイテンシだけでなく学習時のメモリ(バッチサイズの上限)にも効きます。車載の実時間制約に加え、学習リソースの制約からも次元削減の意味は大きいのです。

## 手法の系譜と主要論文
- Vaswani et al., "Attention Is All You Need", NeurIPS 2017(arXiv:1706.03762)。Transformer と Multi-Head Attention の原典で、注意の計算量を O(N²·d) と整理しました。射影の重みが d×d であること、複数ヘッドに分割すること(ヘッド数 h、各ヘッド次元 d/h)もここで定義されました。多頭化は d を増やさずに表現を多様化する工夫である点も重要です。元論文は d_model=512、h=8(各ヘッド64次元)を基本としました。

- Carion et al., "End-to-End Object Detection with Transformers (DETR)", ECCV 2020(arXiv:2005.12872)。物体検出を Transformer で解き、埋め込み次元 d=256 を採用しました。この 256 は、その後 Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)、BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)、UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)、PETR(Liu et al., ECCV 2022, arXiv:2203.05625)、VAD(Jiang et al., ICCV 2023, arXiv:2303.12077)へと事実上の標準次元として受け継がれました。256 が「表現力」と「車載で回せる計算量」のバランス点として経験的に良いとされてきたためです。NLP 系の大規模言語モデルが d=4096 以上を使うのと対照的で、視覚・運転系がコスト制約から小さい次元を選ぶ傾向がよく分かります。

- Dosovitskiy et al., "An Image is Worth 16x16 Words (ViT)", ICLR 2021(arXiv:2010.11929)。画像 Transformer の埋め込み次元(ViT-Base で 768、ViT-Large で 1024、ViT-Huge で 1280)とモデルサイズ・精度の関係を示しました。次元を上げれば精度は上がりうるが、計算とデータ要求も増えるという一般的なスケーリングのトレードオフを裏付けます。ViT は大規模データ(JFT-300M)で事前学習しないと中規模データでは CNN に負ける、という結果も示し、「大きい次元は大量データを要求する」ことを実証しました。

- Kaplan et al., "Scaling Laws for Neural Language Models", 2020(arXiv:2001.08361)。モデル幅(次元)・深さ・データ・計算の間の冪乗則(power law)を定量化し、「次元を上げると性能は上がるが収穫逓減し、計算は急増する」ことを理論的・経験的に示しました。安易な次元拡大が割に合わない理由の一般的な根拠です。

- Dao et al., "FlashAttention", NeurIPS 2022(arXiv:2205.14135)。注意の N² メモリ問題を、スコア行列を全部メモリに展開せずタイル単位で計算する IO 効率の良いアルゴリズムで解決しました。d² ではなく N² 軸の話ですが、「コストを下げる手段は次元削減だけではない」ことを示す代表例として系譜に置けます。

系譜としては、原典 Transformer が O(N²·d) を定義し、DETR が検出向けに 256 を定着させ、その後の自動運転系がこぞって 256 を踏襲した、という流れです。次元削減そのものは新規手法ではなく、「標準次元に合わせる」という規律として広く共有されています。一方で N² 軸の効率化(FlashAttention、変形可能注意、線形注意)は活発な研究領域で、d 軸と N 軸は独立に最適化されてきました。

## 論文の実験結果(定量データ)
指標の意味から。物体検出の精度は mAP(平均適合率の平均、大きいほど良い)で測ります。推論の重さは FLOPs(浮動小数点演算回数)やパラメータ数、レイテンシ(ms)で測ります。

- DETR(ECCV 2020)は COCO で、d=256 という比較的小さい次元でも、ResNet ベースの強力な検出器 Faster R-CNN と同等の mAP(DETR-R50 で約 42 mAP、長く学習した DC5 構成で 43〜44 級)を達成したと報告しています。これは「巨大な次元を使わなくても、注意機構の設計が良ければ高精度が出る」ことの実証で、256 標準化の根拠になりました。なお DETR は小物体(AP_S)が弱く、これが Deformable DETR で改善される動機になります。

- Deformable DETR(ICLR 2021)は同じ d=256 を保ちつつ、注意の参照を少数の点に限定する変形可能注意で、DETR の学習を約 10 倍速く収束させ(500 エポック → 50 エポック級)、小物体の AP も改善したと報告しています。アブレーションでは、変形可能注意を外すと収束が大幅に遅くなり、参照点数を減らしすぎると精度が落ちる、という形でコストと精度のトレードオフが現れます。ここでの教訓は「次元を上げて精度を稼ぐより、注意の構造(N 軸の使い方)を工夫する方が費用対効果が高い」ことです。

- スケーリング則(arXiv:2001.08361)の観察として、モデル幅を増やすと損失は冪乗則で緩やかに下がる(収穫逓減)一方、計算コストは幅の2乗で増える、という非対称性が定量化されています。つまり次元を 256→512 にして得られる精度向上はわずかでも、計算は約4倍になります。この「わずかな精度向上に対して計算が2乗で増える」という関係が、d を不必要に大きくしない最大の理由です。

- ViT(ICLR 2021)の比較では、ViT-Base(d=768)から ViT-Large(d=1024)へ次元を上げると ImageNet 精度は上がりますが、大規模事前学習データがある条件下に限られ、データが少ないと過学習で逆に劣化することが示されました。次元拡大は「データと計算の裏付けがあって初めて報われる」という条件付きの利得であることを示す例です。

定量データの読み方として重要なのは、「精度向上は次元に対して劣線形(収穫逓減)、計算は次元の2乗(超線形)」という非対称性です。だから車載のように計算予算が固定の場面では、次元を削って浮いた予算を「より良い注意構造」や「より多くのトークン」に回す方が合理的、という結論になります。

## メリット・トレードオフ・限界
メリット
- d を小さく(例: 256)すると注意の射影と 1×1 畳み込みが d² 比で軽くなり、推論レイテンシとメモリが大幅に減る。学習時のバッチサイズ上限も上がる。
- 業界標準 256 にそろえると、論文の構成・ハイパーパラメータ・事前学習の知見を流用しやすい。実装の再現性とデバッグの容易さも上がる。

トレードオフ・限界
- 次元を下げると表現力が落ちる可能性がある。タスクやデータ量によっては 512 の方が精度が出る場面もあり、最適点はタスク依存。データ量が多いほど大きい次元が報われやすい。
- 大きな連結特徴(1440 や 3840)を 256 へ射影するのは情報圧縮であり、有用な情報の一部が失われうる。圧縮の仕方(1×1 畳み込み+活性化、ボトルネックの設け方)が精度を左右する。
- d² コストは射影の話で、トークン数 N に効く N² コストは別問題。次元を削っても N が大きいタスク(高解像度の画素単位注意)では依然として重い。ボトルネックがどちらかを見極めずに d だけ削っても効かないことがある。
- 多頭注意ではヘッド数 h と各ヘッド次元 d/h の配分も性能に効くため、単に d を変えるだけでなくヘッド構成も再調整が要る。d を 256 に下げてヘッド数 8 を保つと各ヘッド 32 次元になり、ヘッドの表現力が落ちる場合がある。

未解決の課題として、「タスク・計算予算ごとの最適次元を自動決定する」方向(幅の NAS、適応的次元、Mixture-of-Experts による条件付き計算)は研究途上で、256 が常に最適という保証はありません。

## 発展トピック・研究の最前線
- 線形注意・効率的注意: Linformer、Performer、FlashAttention などは N² 項を削る系統で、d² 項とは別軸の高速化です。両者を組み合わせると、次元削減と系列長削減の両方で効率化できます。FlashAttention はとくに、近似なしで N² メモリを線形メモリに落とす点が実用上強力です。
- 変形可能注意・スパース注意: Deformable DETR 系は「全部見ない」ことで N² を抑え、結果的に d を上げる余地を作ります。BEV 系の空間クロスアテンションもこの系統です。
- 蒸留と量子化: 大きい次元の教師から小さい次元の生徒へ知識蒸留(knowledge distillation)する、あるいは重みを低ビット化する(int8/int4 量子化)ことで、表現力をなるべく保ちつつ車載コストを下げる流れがあります。
- 条件付き計算(MoE): 全トークンに大きい次元を使うのではなく、入力に応じて一部のパラメータだけを使う Mixture-of-Experts により、実効的な計算を抑えつつ実効容量を上げる方向も研究されています。
