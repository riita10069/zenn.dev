# iBOT

## ひとことで言うと
画像の一部(パッチ)を隠して当てさせる「マスク学習」と、教師と生徒のコピーで真似させる「自己蒸留」を1つにまとめた事前学習です。最大の特徴は、隠したパッチの「正解」を、別途用意した固定の辞書ではなく、学習中に一緒に育つ教師ネットワーク(これをオンライントークナイザと呼ぶ)が動的に作る点です。画像レベルと局所レベルの両方の表現を同時に学べるため、汎用視覚基盤モデルの中核部品の一つになりました。

## 直感的な理解
言語モデルの BERT は「文中の単語を隠して当てる」穴埋め(マスク予測)で大成功しました。例えば「今日は良い [MASK] ですね」の [MASK] を「天気」と当てる、という訓練です。このとき隠した単語の正解は「辞書の何番目の単語か」という明確な番号で表せます。言語には辞書(語彙、vocabulary)という有限の記号集合があるからです。

これを画像でやろうとすると壁にぶつかります。「隠したパッチの正解は何か?」という問いに、明確な答えがありません。画像のパッチは連続的な色の集まりで、「辞書の何番目」という離散的な正解が存在しないからです。pixel(画素)をそのまま予測する手もあり([SimMIM](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0092_simmim_note) や MAE がこの道)、それも有効ですが、「色の細部を再現する」低レベルなタスクに寄りやすく、意味的な表現が育ちにくいという懸念がありました。

iBOT の発想は「正解の番号を、固定の辞書から引くのではなく、表現学習と一緒に育つ教師ネットワークに毎回作らせればよい」というものです。教師は [DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) と同じく生徒の EMA(過去の平均)なので、学習が進むにつれて教師=トークナイザも賢くなります。固定辞書のように途中で時代遅れになることがなく、常に「今の表現にとって最良の正解」を提供し続けます。言い換えれば「正解の作り手」と「正解を当てる側」が一緒に成長していく、自己改善型の穴埋め学習です。

## 基礎: 前提となる概念
- マスク画像モデリング(Masked Image Modeling, MIM): 画像のパッチの一部を隠し、隠した部分を予測させる自己教師あり学習。言語の BERT(マスク言語モデリング)の画像版。
- トークナイザ(tokenizer): 生データを離散的な記号(トークン)の列に変換する装置。言語では「文字列 → 単語ID列」。画像では「画像 → パッチごとの記号ID」を作るもの。
- dVAE(discrete Variational Auto-Encoder): 画像を有限個の離散コード(視覚的な記号)に圧縮するオートエンコーダ。BEiT が使った固定トークナイザの実体。
- CLS トークンとパッチトークン: ViT の出力には、画像全体を要約する CLS トークン1個と、各パッチに対応するパッチトークンが複数ある。CLS は「森(全体)」を、パッチは「木(局所)」を表す。
- ソフトラベル(soft label): 「正解は3番」のような確定的なラベル(ハードラベル)ではなく、「3番が70%、5番が20%、…」という確率分布の形の正解。蒸留ではこのソフトな分布を真似させる。連続的な情報を含むので学習信号が濃い。
- 自己蒸留と EMA: [DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) を参照。生徒が backprop で更新され、教師は生徒の指数移動平均でゆっくり追従する。

## 仕組みを詳しく
iBOT は "image BERT pre-training with Online Tokenizer"(オンライントークナイザを用いた画像版 BERT 事前学習)の略です。[DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) と同じく生徒・教師の2ネットワーク(教師は生徒の EMA)を使い、2種類の損失を同時に最適化します。

1. 画像から2つのビュー(拡張した見え方)を作り、それぞれをパッチに区切ります(例: 224×224 を 16×16 ピクセルごとに区切ると 14×14 = 196 パッチ)。
2. 生徒に入れるビューでは、パッチの一部(マスク率はブロック単位で決め、おおむね 30〜50%)をマスクトークン(学習可能な特別な埋め込み)に置き換えます。教師には、マスクしていない元のビューを入れます。
3. 損失その1(画像全体の自己蒸留 = DINO 由来): 生徒と教師それぞれの CLS トークンについて、生徒の出力分布を教師の出力分布に近づけます。これは [DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) とまったく同じ仕組みで、画像レベルの意味を学ばせます。クロスビュー(生徒のビュー1の CLS を、教師のビュー2の CLS に合わせる)で行います。
4. 損失その2(パッチレベルのマスク予測 = iBOT 固有): 生徒がマスクされたパッチ位置に出力した分布を、教師が「マスクされていない同じパッチ」を見て出した分布に近づけます。ここで教師の出力こそが「そのパッチの正解(ソフトラベル)」です。教師は生徒の EMA なので、学習が進むにつれて教師(=トークナイザ)も賢くなります。これが「オンライントークナイザ(online tokenizer、学習しながら一緒に更新される記号化器)」の意味です。固定辞書も別工程の dVAE も不要になりました。
5. 崩壊(全パッチが同じ答えになる無意味解)を防ぐために、[DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note) と同じセンタリングとシャープニングを CLS とパッチの両方の射影ヘッドに適用します。CLS 用とパッチ用で射影ヘッドを共有するか分けるかは設計の選択肢で、論文は共有が有効と報告しています。

before/after で言うと、BEiT は「先に dVAE で辞書を作って固定 → 隠したパッチがどの辞書番号か(ハードな番号)を当てる」でした。iBOT は「辞書(教師)も表現も同時にゆっくり育てる → 教師が今出すソフトな分布に合わせる」に変わりました。固定辞書は学習の途中で「表現が成長したのに正解だけ古いまま」というズレを生みますが、オンライン方式はこのズレが原理的に起きません。

tensor shape 例: ViT-B/16 に 224×224 を入れると 196 パッチ + CLS = 197 トークン、各 768 次元。マスクで例えば 75 パッチを置換。教師は元画像の 196 パッチ全てに対し分布を出し、そのうち生徒がマスクした 75 箇所の分布をターゲットにする。CLS 射影 [B,768]→[B,K]、パッチ射影は各パッチ独立に [B,196,768]→[B,196,K](K は例えば 8192)。

ASCII図:

  ビュー1 ─maskあり─> 生徒 ─┬─ CLS  ──> 教師ビュー2のCLSに近づける(DINO損失)
                            └─ maskパッチ ──> 教師の同位置パッチ分布に近づける(MIM損失)
  ビュー2 ─maskなし─> 教師(EMA) ─> CLS分布 / 全パッチ分布(=ソフトな正解, stop-grad)

CLS の蒸留が「木を見て森も見る」の森を、パッチの蒸留が木を担当します。両方を同時に学ぶことで、画像全体の意味(分類向き)と局所の細部(検出・密予測向き)の両方に強い特徴が得られます。

## 手法の系譜と主要論文
- 比較対象・固定トークナイザ方式: Bao et al., "BEiT: BERT Pre-Training of Image Transformers"(ICLR 2022, arXiv:2106.08254, Microsoft)。DALL-E の dVAE を流用して画像を離散トークンに変換し、その番号を予測ターゲットにしました。MIM を ViT で初めて大きく成功させた一方、固定外部トークナイザという制約を残しました。iBOT はこれを「オンライン化で置き換えられる」と示した相手です。
- 直接の前身・自己蒸留: Caron et al., DINO([DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note)、ICCV 2021)。iBOT は DINO の自己蒸留を「画像全体(CLS)」だけでなく「パッチ単位」にも拡張したものとみなせます。
- iBOT 本体: Zhou et al., "iBOT: Image BERT Pre-Training with Online Tokenizer"(ICLR 2022, arXiv:2111.07832, ByteDance / Johns Hopkins ほか)。自己蒸留をパッチレベルのマスク予測に応用し、教師ネットワーク自身をオンライントークナイザとして使う MIM を提案。
- 並行する画素再構成系 MIM: He et al., "Masked Autoencoders Are Scalable Vision Learners" / MAE(CVPR 2022, arXiv:2111.06377)や Xie et al., SimMIM([SimMIM](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0092_simmim_note)、CVPR 2022)。これらは「隠したパッチの画素そのもの」を再構成します。iBOT は画素ではなく「教師の特徴分布」を予測ターゲットにする点が異なり、より意味的・高レベルなターゲットを使う流派です。MAE は「マスク率を 75% と高くし、可視パッチだけをエンコーダに通す」効率設計で、巨大 ViT の事前学習を安価にした点で対照的です。
- 後継・統合先: Oquab et al., DINOv2([DINOv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0046_dinov2_note)、2023)。DINO の画像レベル蒸留と iBOT のパッチレベル蒸留を両方採用しており、iBOT は大規模汎用基盤モデルの中核構成要素として生き続けています。後の特徴予測系(I-JEPA など、特徴空間で隠れた領域を予測する手法)にも「画素でなく特徴を予測する」という思想が受け継がれています。

## 論文の実験結果(定量データ)
評価は ImageNet(約128万枚、1000クラス)を中心に行われます。

- 線形評価(凍結特徴の上に線形分類器1層): ViT-B/16 の iBOT 特徴を凍結し線形分類器を載せると、ImageNet Top-1 で約 79.5%。より大きな ViT-L/16 では約 81.0%。DINO や BEiT を上回りました。
- k近傍法(k-NN): ViT-B/16 で約 77.1%。線形分類器すら学習せずに高精度が出るのは特徴がクラスごとにきれいに分離している証拠で、iBOT が画像レベルの意味も良く捉えていることを示します(画素再構成系 MIM はこの k-NN が弱い傾向があり、ここが特徴予測系の強み)。
- 微調整(fine-tuning、全パラメータを下流タスクで再学習): ViT-B/16 を ImageNet 分類で微調整すると約 84.0%、ViT-L/16 で約 86.3%。BEiT の同条件を上回りました。
- 物体検出・セグメンテーション(局所特徴が効くタスク): COCO データセットでの物体検出(指標 AP^box、検出ボックスの平均適合度)とインスタンスセグメンテーション(AP^mask)、ADE20K でのセマンティックセグメンテーション(指標 mIoU、各クラス領域の予測と正解の重なり率の平均)で、当時の自己教師あり手法を上回りました。特にパッチレベルの蒸留損失を加えたことで、密な予測タスクでの改善が顕著でした。
- アブレーション(どの要素を抜くとどうなるか): MIM(パッチ蒸留)損失を外して CLS の DINO 損失だけにすると、分類はそれなりでも検出・セグメンテーションが明確に低下しました。逆に「オンライントークナイザ(EMA 教師)」を「固定トークナイザ(dVAE)」に差し替えると、表現が成長しても正解が古いままになるため、線形評価・k-NN ともに数ポイント低下しました。これがオンライン化の効果を直接示すアブレーションです。マスク率やマスクのパターン(ブロック状にまとめて隠すか、ランダムに散らすか)も性能に影響し、ブロック状マスクがやや有利でした。
- 頑健性: ImageNet-A/C/R などの分布外・破損データでも当時の手法より高いロバスト性を示し、画像レベルと局所レベルを同時に学ぶ表現が表面的特徴に過学習しにくいことを確認しました。

## メリット・トレードオフ・限界
メリット:
- 固定の外部トークナイザ(dVAE)が不要。教師が一緒に賢くなるオンライン方式で、表現と正解を同時最適化できる。「表現が育ったのに正解が古い」というズレが原理的に起きない。
- 画像全体(CLS)と局所(パッチ)の両方を同時に学ぶため、分類から検出・密予測まで幅広く強い特徴になる。画素再構成系が弱い k-NN/線形評価でも高い。
- 画素ではなく特徴分布を予測ターゲットにするため、低レベルな色の再現に引っ張られず意味的な表現が育ちやすい。汎用基盤モデルの実用的な中核部品になった。

トレードオフと限界:
- 生徒・教師の2ネットワークと2つの損失を同時に回すため学習設定が複雑で、メモリと計算コストが画素再構成系 MIM(MAE 等)より高い。MAE のように「可視パッチだけ通す」効率化が使いにくい(教師が全パッチを見る必要がある)。
- 崩壊防止(センタリング・シャープニング)、マスク率、温度、CLS とパッチの損失の重みなど、調整すべきハイパーパラメータが多く繊細。設定ミスで学習が崩れやすい。
- 効果は主に ViT で検証されており、CNN への適用は素直ではない(パッチ単位のトークン構造が前提)。
- 予測ターゲットが教師の出力という「動く標的」であるため、学習初期の教師がまだ未熟な段階での信号の質や、教師と生徒の更新速度のバランスの理論的理解は完全には確立されていない。

## 発展トピック・研究の最前線
iBOT が確立した「自己蒸留 + マスク予測」の組み合わせは、その後の自己教師あり学習の主流レシピになりました。特に [DINOv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0046_dinov2_note) はこの2損失をそのまま採用し、データ規模とモデル規模を押し上げることで微調整不要の汎用視覚特徴を実現しています。

研究の流れとしては、(1) 予測ターゲットを画素にするか特徴にするか(MAE 系 vs iBOT 系)の比較と統合、(2) マスク戦略の最適化(どこをどれだけ隠すと最も学習が進むか、難しいパッチを選んで隠す手法など)、(3) 特徴空間で隠れた領域を予測する系列(I-JEPA など、画素も離散トークンも使わず潜在表現で予測する)、(4) 動画への拡張(時間方向のマスク予測)、(5) マルチモーダルへの展開などが活発です。iBOT のオンライントークナイザという発想は「学習対象自身を教師にする」自己改善型学習の一般的な設計原理として、画像以外の領域にも影響を与えています。
