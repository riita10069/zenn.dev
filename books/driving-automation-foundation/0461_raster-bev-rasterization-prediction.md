# ラスタBEV地図(予測向け)

## ひとことで言うと
自動運転車のまわりの「地図」と「他の車や歩行者」を、真上から見下ろした多チャンネルのカラー画像のようなものに描き込み、その絵を画像認識ニューラルネットワーク(CNN)に入力して「この先どう動くか」を予測する、軌跡予測の古典的な入力表現です。地図・物体・信号・運動履歴という、本来はバラバラの形式で持っている情報を、CNN が一番得意とする「画像」という1つの形にそろえてしまうのが核心のアイデアです。2018〜2019年ごろに業界の標準的な手法となり、その後ベクトル表現に主役を譲りましたが、すべてのモダンな予測手法の比較の出発点になっています。

## 直感的な理解
人間が運転中に「前の車はこの先どう動くか」を予測するとき、頭の中では周囲の状況を一望できる俯瞰図のようなものを描いています。「あの車は右折レーンにいて右ウィンカーを出している、だから右に曲がる」「信号が赤だから止まるはず」というように、地図のレイアウトと物体の状態を重ね合わせて考えています。

機械にこれをやらせたいとき、最初に立ちはだかる壁は「入力の形式がバラバラ」という問題です。物体の位置は座標の数値、道路は線分の集合、信号は赤・青・黄という状態記号です。これらを同時に1つのニューラルネットワークに食わせるのは設計が難しい。そこで「全部を1枚の俯瞰絵に焼き込んでしまえば、画像認識で大成功している CNN をそのまま転用できる」と気づいたわけです。画像認識の分野では 2012年の AlexNet 以降、CNN が圧倒的な性能を示していました。その成熟した武器を予測に持ち込むための一番手っ取り早い橋渡しが、このラスタ化(ピクセル絵への変換)でした。

## 基礎: 前提となる概念

軌跡予測(trajectory prediction または motion forecasting)とは、ある物体が将来たどる位置の系列を予測することです。「軌跡」とは、時間とともに動く位置の並び、たとえば「0.5秒後は (12.0, 3.1)、1.0秒後は (14.2, 3.0) …… 6秒後まで」という座標の列です。自動運転では自車(ego vehicle、自分の車)以外の周辺エージェント(他車・歩行者・自転車)について、典型的には 3〜8秒先までを予測します。

CNN(Convolutional Neural Network、畳み込みニューラルネットワーク)は、小さなフィルタを画像上で滑らせて局所的なパターンを抽出することを繰り返すネットワークです。これは「近くのピクセル同士は関係が強い」「同じパターンは画像のどこにあっても同じ意味」という画像特有の性質を活用しており、画像認識で非常に強力です。

BEV(Bird's-Eye View)とは「鳥の視点」、つまり真上から見下ろした図のことです。地図を真上から見ると、車線・交差点・物体の位置関係が平面上のレイアウトとしてそのまま絵に描けます。

ラスタ化(rasterization)とは、線・点・多角形といった図形を、ピクセル(画素)の格子に塗り分けて画像にすることです。逆に、線や点を数値座標のまま保持する表現をベクタ(ベクトル)表現と呼びます。ラスタBEV地図とは、真上から見た地図・物体をピクセルの絵に焼き込んだもの、ということになります。

## 仕組みを詳しく

典型的な作り方をステップで分解します。

### ステップ1: 自車中心・進行方向そろえの座標系を決める

予測する瞬間の自車位置を画像の中心(あるいは下端中央)に置き、自車の進行方向を画像の上向きに固定します。こうすると、どの場面でも「自分が常に絵の中央で前を向いている」状態に正規化され、CNN にとって学習が楽になります。これを agent-centric / ego-centric な座標系と呼びます。回転と平行移動を施して世界座標を自車基準に変換する処理が前段に入ります。

### ステップ2: 物理範囲と解像度を決める

「前方80m × 左右40mを、1ピクセル=0.2mで描く」と決めると、画像サイズは次のようになります。

- 横: 40m ÷ 0.2m = 200ピクセル
- 縦: 80m ÷ 0.2m = 400ピクセル

つまり 400×200 ピクセルの絵です。1ピクセルが何メートルに対応するかを解像度と呼びます。0.1m など細かくするほど精密ですが、画像が大きくなって計算が二乗で重くなります。範囲と解像度のこのトレードオフは、後述する地図タイルのズーム設計とまったく同じ問題です([地図タイル/ズームレベル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0448_map-tile-pyramid-zoom))。

### ステップ3: 情報の種類ごとにチャンネルを分けて描く

ここがラスタBEVの肝です。普通のカラー画像は赤・緑・青の3チャンネルですが、ラスタBEV地図では情報の種類ごとに専用のチャンネルを作ります。チャンネルとは、同じ大きさの絵を何枚も重ねたときの「1枚分」です。例えば次のように割り当てます。

```
ch 0 : 走行可能領域 (道路なら1、それ以外0)
ch 1 : 車線中心線
ch 2 : 横断歩道・停止線
ch 3 : 赤信号がかかる車線
ch 4 : 自車の現在位置と形 (矩形を塗る)
ch 5 : 周辺車両の現在位置
ch 6 : 周辺車両の 0.2秒前 の位置
ch 7 : 周辺車両の 0.4秒前 の位置
...   (過去 N ステップ分を別チャンネルに)
```

最終的な入力テンソル(多次元配列)の形は、例えば `(チャンネル数, 高さ, 幅) = (11, 400, 200)` になります。これを CNN に入れると、「赤信号の手前にいる車は止まりやすい」「カーブする車線の上の車は曲がる」といった関係を、画像のパターンとして学習できます。

過去の位置を時間ごとに別チャンネルにする、あるいは色の濃さ(時間が古いほど薄く)で表現することで、「速い・遅い・止まっている」という運動情報も絵に入れられます。後者の濃淡で履歴を表す工夫は ChauffeurNet が採用しました。

### before/after のイメージ

- before(生データ): 車Aの座標 (x=12.3, y=-4.1)、速度 8m/s、向き 30度。道路は線分の集合 [(0,0)-(50,0), (50,0)-(50,30), ...]。信号=赤。これらがバラバラの形式。
- after(ラスタBEV): 上記すべてが `(11, 400, 200)` のテンソル1つに焼き込まれる。CNN から見れば「ただの多チャンネル画像」。

### 出力側の設計

入力をラスタ化した後、CNN が特徴を抽出し、最後の全結合層などで将来軌跡を回帰(数値を予測)します。ただし軌跡予測は本質的に多峰性(multi-modal)を持ちます。同じ状況でも「直進」「右折」という複数の正解があり得るのです。1本だけ回帰すると、直進と右折の平均という「車線の真ん中を突っ切る」あり得ない軌跡を出してしまいます。これを mode averaging 問題と呼びます。この問題への対処が、後述する MultiPath や CoverNet の主題になります。

## 手法の系譜と主要論文

### IntentNet (Casas, Luo, Urtasun, Uber ATG, CoRL 2018)

ラスタBEV系の初期の代表が IntentNet です。LiDAR 点群を BEV にボクセル化したものと、HD Map をラスタ化したものを同じ BEV 平面に重ね、CNN で検出と予測を同時に行いました。物体の意図(直進・右折・停止など離散カテゴリ)と軌跡を一緒に出すことで、純粋な運動予測より精度を上げられることを示しました。「地図を BEV ラスタに重ねて CNN で食う」という設計の原型の一つです。

### ChauffeurNet (Bansal et al., Waymo, 2018, arXiv:1812.03079)

論文タイトルは "ChauffeurNet: Learning to Drive by Imitating the Best and Synthesizing the Worst"。予測というより自車の運転方策そのものを学習する研究ですが、入力表現としてラスタBEVを徹底的に使った点で系譜の中心です。

- 提案: 道路・物体・信号・意図経路・自車履歴をそれぞれ別チャンネルのトップダウン画像に描き、CNN(FeatureNet)+ 再帰的な AgentRNN で将来の運転軌跡を1ステップずつ生成する。
- 動機: 複雑な手作りパイプラインを避け、強力な CNN に絵として渡して一貫学習したかった。さらに絵にすると、人工的に危険な状況(車線逸脱、衝突直前)を描いて「悪い例」を合成し、それを避けるよう学習させやすい(タイトルの "Synthesizing the Worst")。
- 新規性: 純粋な模倣学習だけでは長い経路で誤差が累積して破綻する(共変量シフト)問題に対し、合成した摂動データと補助損失(車線維持・衝突回避・進行方向整合など)を組み合わせて頑健化した点。

### MultiPath (Chai et al., Waymo, CoRL 2019, arXiv:1910.05449)

論文タイトルは "MultiPath: Multiple Probabilistic Anchor Trajectory Hypotheses for Behavior Prediction"。

- 提案: ラスタBEVを CNN で処理する点は同じだが、出力を「あらかじめ用意した代表軌跡(アンカー)それぞれの確率と、各アンカーからのオフセット(微修正)」として予測する。アンカーは訓練データの軌跡を k-means などでクラスタリングして得る。
- 動機: mode averaging 問題を回避し、多峰の将来を確率付きで明示的に出すため。
- 新規性: 物体検出の anchor box の発想を軌跡予測に持ち込み、回帰を「どのアンカーが正しいかの分類 + 微小な残差回帰」に分解した点。これにより1回の前向き計算で複数の確率付き軌跡を出せる。

### CoverNet (Phan-Minh et al., nuTonomy/Aptiv, CVPR 2020)

MultiPath と同時期に、ラスタBEV入力のまま予測を完全な分類問題に落とした手法です。物理的に実現可能な軌跡の集合(trajectory set)をあらかじめ列挙し、「どの軌跡か」を当てる多クラス分類にしました。回帰の不安定さを避けつつ多峰性を扱える点が利点で、ラスタBEV系の出力設計のもう一つの方向を示しました。

### その後の転換: VectorNet (Gao et al., Waymo, CVPR 2020, arXiv:2005.04259)

ラスタBEVの限界(解像度に縛られる、無駄な空白ピクセルが多く計算が重い、車線の接続関係が暗黙化する)を根本から解消するため、地図と軌跡を「ポリライン(点の並び)」のままグラフとして扱う VectorNet が登場し、以後はベクトル系が主流になりました([Argoverse HD Map](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0438_argoverse-hd-map))。ラスタBEVはその比較対象・原点として現在も論文で必ず触れられます。

## 論文の実験結果(定量データ)

軌跡予測の主要指標を先に説明します。いずれも予測位置と正解位置の距離(メートル)で、小さいほど良い指標です。

- minADE_k(minimum Average Displacement Error): k 個の予測候補のうち最も良いものについて、予測軌跡全体と正解軌跡の各時刻の距離を平均したもの。「複数出した予測のうち一番当たったものが、平均どれだけずれたか」。
- minFDE_k(minimum Final Displacement Error): 同じく最良候補の、最終時刻だけのずれ。「ゴール地点をどれだけ当てられたか」。
- Miss Rate: 最終地点のずれが一定距離(例 2m)を超えた割合。

MultiPath(CoRL 2019)は、自社の大規模データと nuScenes 予測ベンチマークで評価されました。報告によれば、単一軌跡を回帰するベースライン(CNN で1本だけ出す方式)に対し、アンカーによる多峰出力で displacement error を大きく下げています。特に右左折のような分岐シーンで、平均化によるあり得ない軌跡が消え、複数仮説のうち最良候補(minADE/minFDE)が改善しました。アブレーション的には、アンカー数を増やすほど minADE は下がる一方、確率較正(どの仮説が起きやすいか)の質が重要になる、という傾向が示されています。

CoverNet(CVPR 2020)は nuScenes 予測チャレンジで評価され、軌跡集合の密度(用意する候補数)を増やすほど minADE が下がること、固定集合より動的に実現可能性で絞った集合の方が効率的であることを報告しています。当時の nuScenes リーダーボードで、回帰ベースの手法と競合する水準を、分類だけで達成した点が注目されました。

### 受容野(receptive field)という構造的な弱点

ラスタBEVの限界を理解する鍵が「受容野」です。受容野とは、CNN の出力1ピクセルが入力のどれだけの範囲を見て計算されているかの広さです。畳み込みは局所フィルタの積み重ねなので、受容野は層を重ねるほど広がりますが有限です。1ピクセル0.2mで描いた絵で、50m先の合流車線の情報が現在の車に影響するには、ネットワークは画像上で250ピクセル離れた2点を1つの計算に取り込めるだけの受容野を持たねばなりません。これを確保するには深い層・大きなカーネル・大きなストライドが要り、計算量が膨らみます。逆に言えば「遠く離れた地点同士の長距離依存」をラスタCNNで扱うのは構造的に苦手で、これがベクトル系(地図要素間を注意機構やグラフで直接つなぐ)へ移った最大の動機です。注意機構は任意の2要素を距離に関係なく1ホップで結べるため、受容野の制約から原理的に自由になります。

そして系譜全体を一望できるのが VectorNet(CVPR 2020)の比較実験です。VectorNet 論文は、同じ自社データ・Argoverse 予測データで、ラスタBEV + ResNet 系の CNN を直接の対抗馬として比較しました。報告では、ベクトル表現はラスタ表現に対して、より少ない計算量(モデルパラメータと FLOPs を約1桁削減できるケースが示される)で同等以上の予測精度(displacement error の低減)を達成しました。アブレーションでは、ラスタ化の解像度を粗くすると遠方・細部の精度が落ちること、逆に細かくすると計算が重くなることが定量的に示され、「ラスタは解像度と計算量のトレードオフから逃れられない」という限界が明確化されました。これがベクトル系移行の決定打になります。

## メリット・トレードオフ・限界

メリット
- 画像認識で実績のある CNN をそのまま流用でき、実装が早い。
- 地図・物体・信号・運動履歴を1つのテンソルに統合でき、入力が簡潔。
- 過去の運動をチャンネルや濃淡で自然に表現できる。
- 危険シーンの人工合成(摂動)が画像操作として容易(ChauffeurNet)。

トレードオフ・限界
- 解像度に精度が縛られる。1ピクセル0.2mなら、それ未満の差は原理的に表現できない。
- 範囲を広げる/解像度を上げると画像が二乗で大きくなり計算が重い。しかも大半は空白ピクセルで無駄が多い(疎なシーンを密なグリッドで持つ非効率)。
- 車線の接続関係(どの車線からどこへ行けるか)が暗黙化し、CNN がパターンとして学び直す必要がある。長距離の依存関係(数十m先の合流)は局所フィルタでは捉えにくい。
- 遠方の物体ほど画像端で小さく描かれ、情報が薄くなる。
- 回転不変性が乏しい。同じ形状の道路でも向きが変わると別パターンに見えるため、データ拡張やヨー正規化に依存する。

未解決・研究的な論点としては、ラスタとベクトルの良いとこ取り(両者を併用するハイブリッド)、ラスタ表現でも長距離依存を捉えるための注意機構の併用、ラスタBEV を中間表現として明示的に持つことの解釈性の利点などがあり、用途次第で今もラスタBEVが選ばれる場面はあります(例: センサー融合をBEV平面で行いたい場合)。

## 発展トピック・研究の最前線

ラスタBEV以後の流れは大きく2系統あります。1つはベクトル系(VectorNet → LaneGCN → TNT/DenseTNT → mmTransformer → SceneTransformer → Wayformer)で、ポリラインと注意機構で地図と運動を符号化する方向です。もう1つは、知覚から予測・計画までを単一のBEV特徴空間で end-to-end に解く流れ(FIERY, BEVFormer 系、UniAD など)で、ここでは学習されたBEV特徴マップが事実上「学習済みのラスタBEV」として復権しています。手作りで多チャンネルを描く代わりに、カメラ/LiDARからニューラルネットがBEV特徴を生成する点が違いますが、「BEV平面上で予測を解く」という思想は ChauffeurNet 系から連続しています。
