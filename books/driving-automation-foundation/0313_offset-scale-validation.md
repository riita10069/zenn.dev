# deformable attentionのoffset_scale制御と検証

## ひとことで言うと

変形可能注意(deformable attention)では、注目すべき基準点から実際にどれだけ離れた場所を見に行くか、その移動量(オフセット)の大きさを `offset_scale` という1つの係数で制御します。これをコードに固定で埋め込むのをやめて外から指定できるようにし、さらに「有限かつ非負の数」であることを起動時に検証する、という設計の話です。小さなつまみですが、ここが暴れると学習がそもそも収束しないため、安定性を左右する勘所になります。

## 直感的な理解

普通の注意機構(全位置を見て重みづけする)は、画像のように位置が何千・何万とある対象では非常に重くなります。位置数を N とすると計算量が N の2乗で増えるからです。変形可能注意は「全部を見るのをやめて、基準点の周りの少数の場所だけ、しかも『どこを見るか』を学習で決めて参照する」方式です。これで軽くて柔軟になります。

ここで「どこを見るか」はネットワークが予測するオフセット(基準点からのズレ)で決まります。問題は、学習の初期にこのオフセットが大きく暴れることです。重みがランダムに初期化された直後は、ネットワークはまだ「どこを見るべきか」を知らないので、予測するオフセットはほぼでたらめです。ズレが大きすぎると、サンプリング位置がグリッドの外へ飛んだり、毎回ばらばらの場所を見たりして、勾配(学習の手がかり)が不安定になり、学習が発散します。

直感的なたとえで言うと、初心者が望遠鏡で星を探すとき、いきなり大きく振り回すと視野から外れて何も見えません。最初は基準の位置からほんの少しずつ動かすほうが安定して目標を捉えられます。`offset_scale` は、この「動かす量を最初から控えめに抑える」ためのつまみです。小さな係数(典型的には 0.1)を掛けて、基準点からせいぜいグリッドの 10% 程度しかずれないように制限します。

## 基礎: 前提となる概念

- 注意機構(attention): クエリ(問い合わせ)が複数のキー/バリュー(候補)に対して「どれをどれだけ重視するか」を重みとして計算し、バリューを重み付き平均する仕組みです。普通の注意は全候補に対して重みを計算するので、候補数の2乗で計算量が増えます。

- 変形可能注意(deformable attention): 全候補ではなく、基準点の周りの少数(例 4 点)のサンプル点だけを参照する注意です。サンプル点の位置は固定ではなく、クエリから予測したオフセットで動かせるので「変形可能」と呼ばれます。計算量が候補数(画像なら全画素数)に依存せず、参照点数だけで決まるので、高解像度でも軽いのが利点です。

- 基準点(reference point): 各クエリが「だいたいこのあたりを見ればよい」という出発点です。BEV(鳥瞰図、真上から見下ろした地図状の表現)を作る用途では、BEV グリッドの各セルに対応する 3D 点をカメラ画像へ射影して求めます。検出用途ではクエリごとに学習で決まる正規化座標を使います。

- サンプリングオフセット(sampling offset): 基準点からどれだけずらした場所を実際にサンプリングするか。ネットワークが学習で予測します。サンプリングには `grid_sample`(指定座標の特徴を双線形補間で取り出す微分可能な操作)を使うので、オフセットを通して勾配が流れ、「どこを見るべきか」自体を学習できます。

- offset_scale: 予測したオフセットに掛ける係数です。オフセットの「効きの強さ」を一律にスケール(拡大縮小)します。多くの実装では、サンプリング位置を `[0, 1]` に正規化した座標空間で扱うので、`offset_scale=0.1` なら「正規化座標で最大 10% ぶんしか動かない」という意味になります。

なぜ制御と検証が両方必要か。制御が必要なのは、最適なオフセット範囲がタスク・解像度・カメラ配置で変わる調整つまみだからです。0.1 を埋め込んだままだと、実験のたびにソースを書き換えることになります。検証が必要なのは、もし負の値や `inf`(無限大)/`nan`(非数、計算が壊れたときに出る「数でない値」)を渡してしまうと、サンプリング位置の計算が静かに壊れ、原因の分かりにくい学習失敗になるからです。起動時に弾けば、原因不明の発散を未然に防げます。

## 仕組みを詳しく

検証関数の典型例

```python
def validate_offset_scale(offset_scale):
    if not isinstance(offset_scale, (int, float)) or isinstance(offset_scale, bool):
        raise ValueError(...)            # 数値でない・bool は弾く
    if not math.isfinite(offset_scale) or offset_scale < 0:
        raise ValueError(...)            # inf / nan / 負 を弾く
```

ポイントを3つ。

- `bool` を明示的に弾く。Python では `True == 1`、`isinstance(True, int) == True` なので、うっかり `True` を渡しても通ってしまう罠があります。`True` がスケール 1.0 として動いてしまうと、意図せず大きなオフセットを許してしまうので、明示的に拒否します。`False` も 0.0 として通ってしまい、これは「オフセット無効化」として動くため意図と違えば気づきにくい。
- `math.isfinite` で `inf` と `nan` の両方を弾く。これらを掛けるとサンプリング座標が `nan` 汚染し、その座標で `grid_sample` した結果も `nan`、損失も `nan`、勾配全体が `nan` になって学習が即死します。`nan` は一度生まれると伝播し、どこで生まれたか後から追うのが極めて困難なので、入口で止めるのが鉄則です。
- 負の値を弾く。負だとオフセットの符号が反転して意図しない方向にずれます。0 は許容します(後述、オフセット無効化という有効なアブレーション設定なので)。

コンストラクタで受け取り、検証してから保持します。

```python
def __init__(self, ..., offset_scale=0.1):
    validate_offset_scale(offset_scale)
    self.offset_scale = offset_scale
```

forward での使われ方

```python
offsets = self.sampling_offsets(queries)          # 各クエリが予測した生オフセット
offsets = offsets.reshape(B, N, num_points, 2)    # (x, y) のズレに整形
offsets = offsets * self.offset_scale             # ここでスケールを掛けて抑える
sample_locs = reference_points + offsets          # 基準点 + 抑えたオフセット
feats = F.grid_sample(value, sample_locs, ...)    # その位置の特徴を双線形で取得
```

数値例(offset_scale=0.1 のとき)

- ネットワークが生オフセット `0.8`(正規化座標で 80% ぶん)を予測しても、`0.8 × 0.1 = 0.08`、つまり基準点から 8% しかずれない。基準点を中心に狭い範囲を探る挙動になり、初期から安定する。
- 仮に `offset_scale=1.0` なら、基準点から最大でグリッド全域に飛べてしまい、学習初期に不安定になりやすい。
- `offset_scale=0.0` なら常に基準点ちょうどをサンプリング(オフセット無効化)。これは「オフセットを抜くと性能がどうなるか」を見るアブレーションとして有効なので、非負の 0 は許可する。

重要な性質として、`offset_scale` が制限するのは「基準点からのズレ」だけで、基準点そのものが指す範囲は制限しません。だから基準点を遠方に置けば、オフセットを小さく抑えても長距離の注意は依然として可能です。スケールは「局所的な微調整の幅」を絞るだけで、大域的な参照能力は犠牲にしません。この性質があるからこそ、初期の安定性(オフセット小)と最終的な表現力(基準点の自由配置)を両立できます。`offset_scale` は融合モジュールへの引数として外から渡せるようにしておくと、実験ごとにソースを触らず変えられます。

## 手法の系譜と主要論文

- Deformable Convolutional Networks(Dai et al., ICCV 2017, arXiv:1703.06211): 畳み込みのサンプリング格子を、学習したオフセットで変形できるようにした先駆けです。固定の 3x3 格子では捉えにくい形状の変化(回転・スケール・非剛体変形)に適応できることを示しました。「オフセットを学習で予測する」発想の源流で、オフセットの初期化(0 から始める)と範囲制御が安定性の鍵になることもここで顕在化しました。

- Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159): その発想を注意機構へ持ち込み、deformable attention を提案しました。基準点の周りの少数サンプル点(既定で各ヘッド 4 点)をオフセット付きで参照することで、DETR の重い全位置注意を置き換え、計算量を大幅に下げつつ収束を約 10 倍速めました。オフセットを直接予測すると初期に不安定になりやすいため、原論文ではサンプリングオフセットを予測する線形層の重みを 0 初期化し、バイアスをサンプル点が基準点の周囲に放射状・均等に並ぶよう初期配置するなど、初期化で慎重に抑えています。`offset_scale` による一律スケーリングは、この「オフセットの効きを抑える」工夫を1つの係数で素朴に実現したものと位置づけられます。

- Vision Transformer with Deformable Attention(DAT, Xia et al., CVPR 2022, arXiv:2201.00520): 変形可能注意を画像分類・検出のバックボーンそのものに組み込み、データ依存に注目領域を変えられる汎用バックボーンとして提案しました。ここでもオフセットの範囲制御(スケール係数によるクランプ)が学習安定性に効くことが報告されています。

- BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270): 空間 cross-attention と時間 self-attention の両方に deformable attention を用い、BEV クエリが各カメラの少数点を参照して BEV を構築します。複数カメラ・複数時刻にまたがる参照では、基準点の射影誤差やオフセットの暴れが組み合わさって不安定になりやすく、サンプリング位置の安定化は実務上きわめて重要です。単一ヘッドに簡略化した実装でも、`offset_scale` のような抑制つまみは安定学習に効きます。

これらに共通する効果は「全位置を見ないので軽い」「基準点で大域を、オフセットで局所を担える」点、トレードオフは「オフセットの自由度が学習を不安定にしうるため、スケールや初期化での抑制が必要」な点です。`offset_scale` はまさにそのトレードオフを1つのつまみにしたものです。

## 論文の実験結果(定量データ)

Deformable DETR(ICLR 2021)は COCO 物体検出で、DETR が必要とした 500 エポック(データ全体を 500 周する学習)を約 50 エポックに短縮しつつ、box AP(検出の正確さを 0〜100 で測る指標、高いほど良い)を約 43.8 から約 46.2 へ改善したと報告しています。約 10 分の 1 の学習で、しかも精度が上がるという二重の改善です。とくに小物体 APs が DETR より大きく改善し、これはマルチスケールの少数点参照が細かい物体の局所特徴を捉えられることに由来します。この速さと精度は、変形可能注意が「見るべき少数の点」に集中できることに由来し、その安定動作はオフセットの初期化・スケーリングに支えられています。

アブレーションでは、サンプリング点数(reference あたり何点参照するか)を 1→4→8 と増やすと精度が上がる一方で計算量も増えること、マルチスケール(複数解像度の特徴をまたいで参照する)を入れると小さい物体の検出が改善することが示されています。とくにオフセット予測層を 0 初期化しない、あるいはスケーリングを外して大きなオフセットを初期から許すと、学習初期の発散や収束の遅延が起きやすいことが、実装上の経験として広く共有されています。これは「offset_scale を大きくする/外す」アブレーションに相当し、安定性が崩れる側の典型例です。逆に `offset_scale=0`(オフセット完全無効)にすると、固定点サンプリングと同じになり deformable の柔軟性が失われて精度が頭打ちになります。

BEVFormer(ECCV 2022)は nuScenes で 3D 物体検出の総合指標 NDS(検出精度と位置・速度・属性の正しさを総合した 0〜1 の値、高いほど良い)約 0.517、mAP 約 0.416 を達成しました。複数カメラの特徴を deformable attention で BEV へ集約する構成で、サンプリング位置の安定化(基準点の正確な射影とオフセットの抑制)がこの精度を支えています。基準点の射影が誤っていればオフセットをいくら抑えても無意味になる、という依存関係も実務上の重要な含意です(スケールは局所のズレしか直さない)。

## メリット・トレードオフ・限界

メリット

- 学習の安定性を1つの数で調整できる。実験のたびにソースを書き換えなくてよい。
- 不正値(bool / inf / nan / 負)を起動時に即 `ValueError` で弾き、原因不明の学習失敗を未然に防ぐ。
- 0 を許容するので「オフセットを抜く」アブレーションが簡単に行える。
- 基準点の範囲は制限しないので、抑制しても長距離の注意能力は保たれる。

トレードオフ・限界

- 小さすぎると局所的な位置補正がほとんど効かず、deformable の柔軟性を活かせない。大きすぎると不安定。最適値はタスク・解像度・カメラ配置に依存し、要チューニング。
- 全サンプル点・全ヘッドに同じ係数を一律適用する単純な仕組みで、原論文のような初期化やヘッド・点ごとのきめ細かい制御とは別物。簡略実装では表現の自由度がそのぶん限定される。
- スケールはあくまで「効きの抑制」であり、基準点そのものの精度(カメラ射影や 3D 点配置の正しさ)が悪ければサンプリング全体がずれる。offset_scale はその誤差を補正しない。
- 未解決の課題として、オフセット範囲を固定スケールではなく学習可能・位置依存にする(例: 場所ごとに許容ズレを変える)、あるいはアニーリング(学習が進むにつれスケールを徐々に緩める)で初期安定と後半の柔軟性を両立する、といった方向は素朴な単一係数では扱えない。

## 発展トピック・研究の最前線

- Deformable attention は検出だけでなく、BEV 知覚・占有グリッド(オキュパンシー)予測・世界モデルなど、空間を高解像度で扱う近年の自動運転モデルの基本部品になっている。サンプリング安定化はそのすべてに共通の前提。
- 初期化の工夫(0 初期化、基準点周りへの均等配置)とスケーリングは目的が重なるため、両者をどう組み合わせるかは実装の流儀が分かれる。原論文の初期化を踏襲しつつ追加のスケールで二重に抑える設計も一般的。
- サンプリング座標が範囲外へ飛ぶのをスケールだけでなくクランプ(範囲への切り詰め、`clamp(-1, 1)` など)で守る設計もよく併用される。クランプが実際に効いているかをテストで検証する(不正値で確実に例外が出る、範囲外座標が必ず内側へ丸められる)ことが、静かなバグを防ぐ実務上の定石になっている。
- 効率面では、変形可能注意のサンプリング+集約を専用 CUDA カーネルで融合実装するのが一般的で、これにより `grid_sample` を素朴に呼ぶより大幅に高速・省メモリになる。研究の最前線では、この専用カーネル依存をなくして純 PyTorch で動かしつつ速度を保つ実装(FlashAttention 的な発想の deformable 版)も模索されている。
