# ResNet (残差ネットワーク)

## ひとことで言うと
ResNet(Residual Network、残差ネットワーク)は、層をとても深く積み重ねたニューラルネットワークでも学習が破綻しないようにした、画像認識の定番設計です。秘密は「スキップ接続(skip connection)」という近道で、ある層の入力を数層先へそのまま足し込む仕組みです。これにより、層を深くするほど精度が下がってしまうという当時の難問を解決し、50層・101層・152層といった超深層ネットワークを実際に学習できるようにしました。画像から特徴を取り出すバックボーン(backbone、土台のネットワーク)として今も広く使われ、新しい手法の良し悪しを測る基準(ベースライン)としても標準的な存在です。

## 直感的な理解
ニューラルネットワークは「層(layer)」を重ねるほど複雑なパターンを表せる、と期待されます。層とは、入力に計算を施して次へ渡す処理のひとまとまりです。直感的には、層が多いほど表現できることが増えるので、深いネットワークは浅いネットワークより少なくとも同じくらいは賢くなれるはずです。なぜなら、深いネットワークの追加した層が「何もしない(入力をそのまま通す)」を学べば、浅いネットワークと全く同じ振る舞いになるからです。理屈の上では、深くして悪くなる理由はないはずでした。

ところが2015年頃まで、現実には逆の現象が観測されていました。層を深くしすぎると、かえって精度が下がるのです。しかもこれは「過学習(訓練データに合わせすぎて未知データで悪化する現象)」ではありません。過学習なら訓練データ上では良くなるはずですが、観測されたのは訓練データ上の誤差すら大きくなる、という現象でした。最適化そのものがうまくいっていないのです。つまり「賢くなる能力はあるのに、その状態にたどり着けない」という、学習アルゴリズム側の問題でした。

「追加層が何もしないを学べば浅いネットと同じになれるはず」なのに、それすら学習できない。この不可解な現象を劣化問題(degradation problem)と呼びます。ResNetは、この劣化問題を「層に、何もしない変換(恒等写像)を学ぶのを簡単にしてやれば解決する」という発想で乗り越えました。難しい変換をゼロから学ばせるのではなく、「入力からの差分だけ」を学ばせる、という見方の転換です。

## 基礎: 前提となる概念
理解の土台として、いくつかの言葉を噛み砕きます。

畳み込み(convolution)とは、小さなフィルタ(たとえば3×3のマス)を画像の上で滑らせ、各位置で局所的なパターン(エッジや模様)を検出する演算です。CNN(畳み込みニューラルネットワーク)は、この畳み込みを多数積み重ねた構造で、画像認識の標準です。フィルタの重みが学習対象で、何を検出するかをデータから自動で覚えます。

勾配(gradient)とは、「重みをどちらに、どれだけ動かせば誤差が減るか」を表す方向と大きさの情報です。学習は、出力側で計算した誤差を入力側へ逆向きに伝える「誤差逆伝播(backpropagation)」で、各層の勾配を求めて重みを更新します。これは連鎖律(合成関数の微分は各段の微分の掛け算)で各層の寄与を計算します。

勾配消失/勾配爆発とは、この逆伝播の途中で勾配が層を通るたびに小さくなりすぎる(消失)、または大きくなりすぎる(爆発)現象です。連鎖律で各層の微分を掛け合わせるため、1未満の値が続けば指数的に縮み、1超の値が続けば指数的に膨らみます。層が深いほど、入力に近い層へ届くころには勾配がほぼ0になったり発散したりして、その層がうまく学習できなくなります。劣化問題の主因の一つがこれです。

恒等写像(identity mapping)とは、「入力をそのまま出力する」変換、つまり「何もしない」変換のことです。劣化問題の核心は「深いネットが、追加層に恒等写像を学ばせることすら難しい」という点にありました。普通の層で恒等写像を作るには、重みを微妙に調整して入力をそっくり再現する必要があり、これが意外に難しいのです。

バッチ正規化(Batch Normalization)も前提として重要です。各層の出力をミニバッチ単位で平均0・分散1付近に整える処理で、学習を安定させ、深いネットの収束を助けます。ResNetの各残差ブロックの中で標準的に使われます。

## 仕組みを詳しく
普通の層は、入力 x を受け取って目標の出力 H(x) を直接学ぼうとします。ResNet では代わりに、層が学ぶ対象を「入力との差分(残差、residual)」にずらします。

```
出力 = F(x) + x
```

- F(x) は数層(畳み込みなど)の処理で、これに残差 H(x) − x だけを学ばせます。
- + x の部分がスキップ接続(ショートカット)で、入力 x を加工せずそのまま先へ足す近道です。

なぜこれが効くのか、2つの理由があります。第一に、もし「この層では何もしないのが最適」なら、F(x) = 0 を学べばよく、これは重みを0に寄せるだけなので簡単です。つまり恒等写像を作るのが容易になります。深くしても少なくとも浅いネットと同じことができる、という当初の期待が、構造として保証されるわけです。第二に、+ x のおかげで、逆伝播の勾配がこのショートカットを通って奥の層まで素直に流れます。足し算の微分は1なので、勾配はショートカットを通るとき減衰せずそのまま通過します。勾配が層を通るたびに減衰するのではなく、近道を通って直接届くので、勾配消失が起きにくくなります。

ResNet50 の構造をざっくり追います。入力画像は (3, 224, 224) のような (チャンネル, 高さ, 幅) です。

- まず7×7畳み込みとプーリングで解像度を1/4に落とします(stemと呼ぶ最初の処理)。
- そのあと4つの「ステージ(stage)」があり、各ステージは複数の残差ブロックの積み重ねです。ResNet50 ではブロック数が各ステージ 3, 4, 6, 3 の計16ボトルネックブロックで、3層×16+stem1層+分類層1層でおよそ50層になります(これが「50」の由来)。ステージを進むごとに解像度は半分、チャンネル数は倍に増えます。
- ResNet50 では各残差ブロックが「ボトルネック(bottleneck)」型です。1×1畳み込みでチャンネルを減らす → 3×3畳み込みで処理する → 1×1畳み込みでチャンネルを戻す、の3層構成です。1×1で一度細くして計算を節約してから重い3×3を通すので、計算量を抑えつつ深くできます。これに対し ResNet18/34 は3×3を2つ重ねる「basic block」を使います。
- 各ステージ末尾の特徴マップのチャンネル数は、stem出力を含めて 64, 256, 512, 1024, 2048 の5段です。

入力224×224のときの各段の解像度の流れ:

```
stem 後 : 56×56  (チャンネル   64)  ← 細かいエッジ
stage1  : 56×56  (チャンネル  256)
stage2  : 28×28  (チャンネル  512)
stage3  : 14×14  (チャンネル 1024)
stage4  :  7×7   (チャンネル 2048)  ← 意味的な物体
```

「解像度を下げながらチャンネルを増やす」ことで、浅い段は細かい模様(エッジなど)、深い段は意味のある物体(車・人など)を捉える、階層的な特徴になります。この設計は「空間的な細かさ」を「特徴の抽象度」に少しずつ変換していくと解釈できます。この複数段を同時に取り出して使う方法は [features_only マルチスケール特徴抽出](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0361_resnet50-features-only) を参照してください。

## 手法の系譜と主要論文
ResNet以前の代表が VGG (Simonyan & Zisserman, "Very Deep Convolutional Networks for Large-Scale Image Recognition", ICLR 2015, arXiv:1409.1556) です。VGGは3×3畳み込みを単純に積むだけのシンプルな設計で、16層・19層まで深くして高精度を達成しました。「小さな3×3を重ねれば大きな受容野を効率よく作れる」という設計原則を確立した点で重要です。しかしスキップ接続が無いため、さらに深くすると劣化問題に直面し、それ以上深くできませんでした。ResNetはこの限界を破った点に意義があります。

原典は He, Zhang, Ren, Sun, "Deep Residual Learning for Image Recognition", CVPR 2016 (arXiv:1512.03385) です。提案はスキップ接続つきの残差学習で、動機は劣化問題の解消、効果は超深層の学習を現実的にしたことでした。ImageNet分類で当時の最高精度を更新し、152層という当時としては桁外れに深いネットを学習させ、ILSVRC 2015(ImageNetの大規模画像認識コンペ)で優勝しました。この論文は被引用数が極めて多く、深層学習史で最も影響力のある論文の一つです。トレードオフは、非常に深くするとパラメータと計算が増え、メモリ・速度コストが上がることです。ResNet50はその中で精度と計算のバランスが良い定番サイズ(約2,500万パラメータ、約4 GFLOPs @224²)として定着しました。

続編が He et al., "Identity Mappings in Deep Residual Networks", ECCV 2016 (arXiv:1603.05027) です。残差ブロック内の活性化関数(ReLU)とバッチ正規化の順番を見直し(pre-activationと呼ぶ、BN→ReLU→Conv の順にする構成)、ショートカットをできるだけ素通り(純粋な恒等写像)に保つほど勾配が流れやすい、と理論と実験の両面から示しました。これにより1001層という極端に深いネットでも学習が安定し精度が改善することを実証しました。これは「スキップ接続の経路に余計な変換を挟まないほど良い」という設計指針を確立しました。トレードオフは実装が少し変わる点です。

その後 Vision Transformer 系([Swin Transformer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0408_swin-transformer) / [Swin Transformer V2 (SwinV2)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0363_swin-v2-tiny))が登場し、大規模データでは精度で上回る場面が増えましたが、ResNetは今も軽量で扱いやすいCNNバックボーンの基準として現役です。新手法は、しばしば「ResNet50比でどれだけ良いか」を物差しに語られます。さらに ConvNeXt (Liu et al., CVPR 2022, arXiv:2201.03545) は、ResNetを出発点に Transformer の設計思想を取り込み、純CNNのまま最前線の精度に戻したことで、ResNetの骨格が今も有効であることを逆説的に証明しました。

## 論文の実験結果(定量データ)
原論文(He et al., 2016)の核心的な実験は、CIFAR-10(3万2千枚の小さな画像分類データセット)とImageNet(約120万枚・1000クラスの大規模画像分類データセット)での比較です。

劣化問題の証拠として、論文は通常のCNN(plain net、スキップ接続なし)で20層と56層を比較し、56層のほうが訓練誤差・テスト誤差ともに大きいことを示しました。深くしただけで悪化する、という劣化問題の直接的な証拠です。同じ条件にスキップ接続を入れたResNetでは、56層のほうが20層より誤差が小さくなり、「深くするほど良くなる」という本来期待された挙動が回復しました。さらにCIFAR-10では110層まで深くしても安定して学習でき、1202層という極端な深さでも発散せず学習できる(ただし過学習気味になる)ことを示しました。これは「深さの上限」を一気に押し上げた結果です。

ImageNet分類では、ResNetの深さを増やすほど誤差が下がりました。報告された top-5 誤差(正解が予測上位5つに入らない割合)は、ResNet-34 でおよそ 8.6%、ResNet-50 でおよそ 7.1%、ResNet-101 でおよそ 6.4%、ResNet-152 でおよそ 5.7% でした。比較対象のVGGは深くできず、ResNet-152の単一モデルがVGGを明確に上回りました。アンサンブル(複数モデルの予測を平均する手法)では top-5 誤差 3.57% を達成し、ILSVRC 2015で優勝しました。top-5誤差は「上位5つの予測のどれかが正解なら成功」とする緩めの指標で、1000クラスという難しさを踏まえた標準指標です。当時の人間の top-5 誤差が約5%と言われており、ResNet-152のアンサンブルがそれを下回ったことは大きな話題になりました。数%の差でも、当時の最先端競争では決定的な差でした。

ImageNet以外でも、ResNetを物体検出(COCOデータセット)のバックボーンに使うと検出精度(mAP、平均適合率)が大きく向上し、汎用バックボーンとしての強さが示されました。論文では COCO 検出で当時の最先端を更新し、ResNet系バックボーンが分類以外のタスクにも転移することを実証しました。アブレーション的には「スキップ接続を外すと劣化問題が再発する」ことが繰り返し確認されており、性能の源泉が確かにスキップ接続にあると裏づけられています。pre-activation 論文(2016)では、CIFAR-10 で1001層 ResNet がオリジナルの構成より誤差を下げ(報告でおよそ4.6%付近)、ショートカットを純粋な恒等写像に保つことの効果を定量的に示しました。

## メリット・トレードオフ・限界
メリット。

- 超深層でも安定して学習でき、高い精度が出ます。
- 構造が枯れていて実装・学習が容易です。事前学習済み(ImageNetなどで先に学習させた)重みが豊富に公開されており、すぐ転用できます(転移学習・ファインチューニングの標準的な出発点)。
- CNNなので局所的な模様の検出に強く、入力画像サイズの制約が緩いです(後述のViTのような窓サイズの縛りがありません)。任意解像度の入力を扱いやすく、検出・セグメンテーションのバックボーンに向きます。
- 多くの新手法のベースラインとして使われており、比較の基準として信頼できます。

トレードオフ・限界。

- 同程度の計算量の最新Transformer系より、大規模データのタスクでは精度で劣ることがあります。
- 畳み込みの受容野(一度に見られる範囲)は基本的に局所的で、画像全体にまたがる長距離の関係を捉えるのは苦手です(層を深く積むことで間接的に広げています)。遠く離れた2物体の関係を1ステップで捉えるのは Transformer の自己注意のほうが得意です。
- ResNet50で約2,500万パラメータあり、軽量モデルよりは重いです。エッジデバイスでの推論には工夫(蒸留・量子化・枝刈りなど)が要ることがあります。
- 未解決の論点として、CNNの帰納バイアス(局所性・並進不変性という前提)が、大規模データではむしろ表現力の天井になりうる、という議論があります。Transformer系はこの前提を持たない分、十分なデータがあれば上回る、というのが近年の流れです。逆にデータが少ない場面では、CNNの帰納バイアスが効いてResNetが勝つこともあります。

## 発展トピック・研究の最前線
ResNetを起点に、効率と精度を改良する系譜が多数生まれました。ResNeXt(グループ畳み込みで「カーディナリティ」という新しい次元を導入)、Wide ResNet(深さより幅を増やす)、DenseNet(全層を密に結合する)などです。EfficientNet系は、深さ・幅・解像度をバランスよくスケールさせる「複合スケーリング」の方針で、ResNetより少ない計算量で高精度を達成しました。

近年は、CNNとTransformerの良いところを取り込む流れが顕著です。ConvNeXt(Liu et al., CVPR 2022)は、ResNetを出発点にTransformerの設計思想(大きなカーネル7×7、レイヤ正規化、GELU活性化、ステージ比率の調整など)を一つずつ取り入れ、純CNNでありながらSwin Transformerに匹敵する精度(ImageNet で同規模 Swin と互角)を達成しました。これは「ResNetの基本骨格は今も有効で、細部を現代化すれば最前線で戦える」ことを示した重要な結果です。

研究者として深掘りするなら、スキップ接続が損失地形(loss landscape、重み空間での損失の形)を滑らかにして最適化を容易にする、という理論的解析(Li et al., "Visualizing the Loss Landscape of Neural Nets", NeurIPS 2018)が面白い方向です。この論文は、スキップ接続のないネットの損失地形が険しく多数の谷だらけで最適化が難しいのに対し、ResNetでは滑らかな盆地状になり最適化が容易になる様子を可視化しました。スキップ接続がなぜ効くのかを、最適化の幾何という観点から理解できます。また、ResNet を「微分方程式の離散化(各残差ブロックが時間ステップ)」とみなす Neural ODE の視点も、残差構造の理論的理解を深める方向として知られています。
