# 最大マージン計画損失 (Cost Volume)

## ひとことで言うと

最大マージン計画(Maximum Margin Planning)は、「どこを通ると良いか/悪いか」を表すコストマップ(cost volume)を学習させる手法です。専門家(人間)が実際に通った経路のコストが、ほかのどの候補経路よりも明確に低くなるよう、しかも単に低いだけでなく「一定の差(マージン)をつけて」低くなるように学習します。これにより、コストマップ上で一番安いルートを選べば自然に人間らしい運転になる、という仕組みです。逆強化学習(報酬/コストをデータから学ぶこと)を、敵対的ではなくマージン最大化という凸最適化で解く方法だと位置づけられます。

## 直感的な理解

自動運転の経路計画には大きく2つの流派があります。

1. **軌跡を直接出す**: モデルが「これから通る点列」を直接出力する(回帰型。[行動クローニング損失 (Behavior Cloning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0111_behavior-cloning-loss) がこれ)。
2. **コストマップを学んで探索する**: 空間の各場所・各候補軌跡に「コスト(通る悪さ)」を割り当て、コストが最小のものを選ぶ。

2番目の利点は **解釈性と安全性** です。「なぜこの経路を選んだか」を「コストが一番低かったから」と説明でき、コストマップを見れば「障害物の場所は高コスト、走行可能領域は低コスト」と人間が確認できます。出力前に複数候補のコストを比べて最適なものを選ぶので、明らかに危険な軌跡を弾きやすい。検証や安全評価の観点で、ブラックボックスな直接回帰より好まれる場面があります。

問題は「このコストマップをどう学習させるか」です。コスト自体には正解ラベルがありません(「この地点のコストは3.7」というデータは存在しない)。あるのは「人間がここを通った」という事実だけです。

そこで使うのが最大マージン計画の発想です。「正解そのものは分からないが、専門家が選んだ経路は、ほかのどの経路よりコストが低いはずだ」という制約からコスト関数を逆算します。直接回帰が「真似する」のに対し、コスト学習は「なぜその経路が良いのかの評価基準を学ぶ」点で一段抽象度が高く、その評価基準を新しい場面に当てはめれば未知のレイアウトでも妥当な経路を選べる、という汎化の利点があります。

## 基礎: 前提となる概念

- **BEV(鳥瞰図、Bird's-Eye-View)**: 真上から見下ろした平面マップ。`H×W` のグリッドに区切って各セルに値を持たせる表現が、コストマップの土台になります。
- **コスト/報酬の関係**: コストは「負の報酬」と考えてよい。コストが低い=報酬が高い。よってコストを学ぶことは逆強化学習(IRL)そのものです([GAIL (敵対的模倣学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0177_gail-adversarial-imitation) と同じ問題意識を別の解き方で解く)。
- **マージン**: 2つのものを区別するときの「余裕の幅」。SVM(サポートベクターマシン、マージン最大化で分類する古典的手法)が、正解と不正解の境界をできるだけ離して引くのと同じ発想です。
- **ヒンジ損失 `[x]_+`**: `max(0, x)` のこと。制約が満たされていれば0、破った分だけ正の値で罰する。SVMの損失そのものです。劣勾配(subgradient)で最適化できる点が重要です。
- **構造化予測(structured prediction)**: 出力が単なるラベルではなく「軌跡」のような構造を持つ予測問題。最大マージン計画は、構造化SVM(Tsochantaridis et al., JMLR 2005)を計画に応用したものと理解できます。
- **特徴量カウント(feature expectation)**: ある軌跡が通った各セルの特徴(障害物近接度、車線中央度など)を足し合わせたベクトル。Abbeel & Ng(ICML 2004)の見習い学習(apprenticeship learning)は「専門家と方策の特徴量カウントを一致させれば、真の報酬が特徴の線形和である限り性能も一致する」ことを示し、コスト学習の理論的支柱になっています。

## 仕組みを詳しく

### コストマップ(cost volume)とは

空間や軌跡候補に対するコストの集まりです。BEVを `H×W` のグリッドに区切り、各セルにコスト値を持たせます。時間方向も含めて `T×H×W` の3次元にして「t秒後にこのセルにいるコスト」を表すと、これが "volume"(体積)と呼ばれる所以です。モデル(典型的にはCNN)がセンサ入力からこのコストマップを出力します。たとえば 3秒先までを 0.5秒刻みで6時刻、BEVを 200×200 セル(各セル0.25m なら 50m×50m 相当)とすると cost volume は `[6, 200, 200]` の tensor になります。

ある候補軌跡 `τ`(点列)のコストは、その軌跡が通るセルのコストを足し合わせて定義します。

```
C(τ) = Σ_{各時刻 t} cost_map[ t, τ の位置(x_t, y_t) ]
```

tensor で言えば、`cost_map` が `[T, H, W]`、軌跡 `τ` が `[T, 2]` で、各時刻の座標でマップを索引して合計する、という操作です。索引(離散セル参照)は微分できないので、実装では双線形補間(bilinear sampling)で滑らかに値を引き、コストマップへ勾配が流れるようにします。

### マージン損失の式

専門家軌跡を `τ_E`、ほかの候補軌跡を `τ`、専門家と `τ` の「ずれの大きさ」を `m(τ)`(専門家から離れた軌跡ほど大きいペナルティ関数)とします。学習で課す制約は次の通りです。

```
C(τ_E) ≤ C(τ) - m(τ)      ……すべての候補 τ について
```

意味は「専門家の軌跡のコストは、ほかのどの候補のコストよりも、少なくとも `m(τ)` 以上は低くあれ」です。なぜマージンを入れるか。単に「専門家が一番安い」だけだと、ほぼ同じコストの危ない軌跡が紛れ込んでも区別できないからです。マージンを設けることで「専門家から大きく外れた軌跡ほど、はっきり大きなコスト差をつける」ようになり、コスト関数が頑健になります。マージン関数 `m(τ)` は典型的には専門家との L2 距離の和(loss-augmentation)で、SVM の「分類マージン」を構造化出力に拡張したものです。

### 実際の損失(ヒンジ損失の形)

制約を破った分だけ罰する形にすると、最小化する損失は次のようになります。

```
L = [ C(τ_E) - min_τ ( C(τ) - m(τ) ) ]_+
```

`min_τ` の中で「コストが低く、かつ専門家から遠い」最悪の候補(最も制約を破りやすい候補。loss-augmented inference の解)を1本選び、その候補と専門家の差を罰します。制約が満たされていれば(専門家が十分安ければ)損失0、破っていればその超過分が損失になります。劣勾配は「専門家軌跡が通るセルのコストを下げ、最悪候補が通るセルのコストを上げる」向きに流れます。

### 数値例

候補が3本あり、コストマップから計算したコストが

- 専門家軌跡: `C = 5.0`
- 候補A(専門家にそっくり、m=0.2): `C = 5.5`
- 候補B(大きく外れて危険、m=3.0): `C = 6.0`

候補Aの制約: `5.0 ≤ 5.5 - 0.2 = 5.3` → 満たす(損失0)。
候補Bの制約: `5.0 ≤ 6.0 - 3.0 = 3.0` → 破る。超過 `5.0 - 3.0 = 2.0` が損失。

この場合、損失2.0を下げる方向にコストマップを更新します。具体的には「候補B(危険軌跡)が通るセルのコストをもっと上げる」「専門家軌跡が通るセルのコストを下げる」よう勾配が流れます。結果、危険な経路ほど高コストなマップが学習されます。

### 軌跡の集合をどう用意するか

すべての候補軌跡を考えるのは無理なので、実際は有限のサンプル軌跡集合(数千〜数万本の、車両運動学的に妥当な走行候補)を事前生成し、その中で上記のマージン制約を課します。クロソイド曲線(曲率が距離に比例する、ハンドルを一定速度で切ったときの軌跡)や等加速度モデルで候補を撒く方式が一般的です。ここがこの手法の生命線で、良い軌跡が候補集合に入っていないと、どれだけコストマップが正確でも選べません。「argmin の中身が候補集合に縛られる」という構造的制約です。

## 手法の系譜と主要論文

### Abbeel & Ng (2004, ICML, "Apprenticeship Learning via Inverse Reinforcement Learning")

最大マージン計画の直前の源流。報酬が特徴の線形和なら、専門家と特徴量カウントを一致させた方策は専門家と同等の性能になることを示し、コスト/報酬学習に理論的な土台を与えました。

### Ratliff, Bagnell & Zinkevich (2006, ICML, "Maximum Margin Planning")

最大マージン計画の原典です。

- 提案: 専門家のデモから、その経路を最適にするコスト関数を、マージン制約付きの構造化予測問題として学習する枠組み。コストは特徴量の線形和で表し、劣勾配法(subgradient)で最適化。
- 動機: 従来のIRLは計算が重く解が一意に定まらない(同じデモを説明する報酬が無数にある不良設定問題)。マージン最大化(SVM的)にすることで凸最適化として安定に解け、専門家に近い経路を強く優先できる。
- 新規性: 計画問題(最短経路探索を含む)を構造化マージン学習として定式化し、内側に「最良経路探索(プランナ)」を組み込んだまま学習できる点。

### Zeng et al. (2019, CVPR, "End-to-end Interpretable Neural Motion Planner", NMP)

最大マージン計画を深層学習・自動運転に持ち込んだ代表例です。

- 提案: LiDAR点群とHDマップから、3次元の cost volume(時空間コストマップ)をCNNで出力し、サンプリングした多数の軌跡候補をこのコスト上で評価して最小コスト軌跡を選ぶ、解釈可能な end-to-end プランナ。学習に max-margin loss を使い、専門家軌跡のコストが他候補より(マージン付きで)低くなるようコストマップを学ぶ。同時に物体検出も補助タスクとして学習。
- 動機: 軌跡を直接回帰する end-to-end は中身がブラックボックスで安全検証が難しい。コストマップを介せば「なぜその軌跡か」が可視化でき、検出という解釈可能な中間タスクも持てる。
- 新規性: 古典的な線形特徴の max-margin planning を、深層特徴で表現した時空間 cost volume に乗せ、知覚から計画まで一貫して微分可能にした点。

### P3 / MP3 (Sadat et al., ECCV 2020 / Casas et al., CVPR 2021)

NMPの発展系。P3 は「意味的占有(semantic occupancy)」を中間表現にして、解釈可能なコストと安全制約を結合。MP3 は地図なし(HDマップに頼らない)で、オンライン地図推定・動き予測・コスト評価を統合しました。いずれもコスト+サンプル評価+マージン学習という骨格を引き継ぎつつ、中間表現と安全項を強化した流れです。

### 関連: ChauffeurNet (Bansal et al., 2019, RSS)

直接マージン損失ではないものの、軌跡計画に複数の補助損失(衝突・道路逸脱など)を組み合わせる思想は近く、[衝突/安全制約損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0161_collision-safety-loss) と合わせて読むと理解が深まります。

## 論文の実験結果(定量データ)

- **NMP (2019)**: 大規模な実世界運転データ(北米都市部、数千時間規模の自社収集データ)で評価。指標は (1) 計画の人間らしさを測る L2 軌跡誤差(専門家軌跡との距離、メートル。最大3秒先まで時刻ごとに評価)、(2) 安全性を測る衝突率(planned collision rate)。NMP は、軌跡を直接回帰する模倣ベースラインや ego-motion forecasting に比べ、L2 誤差を同等以下に保ちつつ、各予測時刻での衝突率を顕著に下げたと報告しました。とくに3秒先など遠い地平線で、コストマップを介して候補を比較する構造が、危険軌跡の選択を抑える効果が大きく出ています。物体検出も同時学習することで、検出性能を犠牲にせず計画を改善できることも確認(マルチタスクの相乗効果)。
- **P3 (2020)**: nuScenes 等で、意味的占有を介すことで、模倣のみより衝突率と道路逸脱を低減。占有という中間表現の解釈性と安全性の両立を示しました。アブレーションでは、占有を介さず直接コストを出すと衝突率が悪化することを示し、解釈可能な中間表現の価値を裏づけています。
- **MP3 (2021)**: HDマップを使わない設定で、オンライン推定した地図と動き予測を使い、地図利用手法に迫る計画品質を達成。マップ依存を外しても max-margin/コスト型の枠組みが機能することを実証しました。

指標の意味を補足します。**L2 軌跡誤差(ADE 系)** は予測経路と専門家経路の距離(メートル)で、小さいほど人間に近い。**衝突率** は計画軌跡が他物体の予測占有と重なる割合で、安全性の直接尺度として小さいほど良い。max-margin 系の価値は、L2 をBC並みに保ちつつ衝突率を下げられる点にあり、これは「人間らしさ」(L2)と「安全」(衝突)を別々のコスト項として加算で扱えること、そして複数候補から選ぶ argmin 構造が極端に危険な候補を弾けることの帰結です。

## メリット・トレードオフ・限界

メリット
- **解釈性が高い**。コストマップを見れば「どこが危険か」を人間が確認でき、安全検証しやすい。
- 複数候補をコストで比較して選ぶので、明らかに危険な軌跡を選びにくい。
- マージン制約により、専門家から外れた軌跡ほどはっきり高コストになり頑健。
- [衝突/安全制約損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0161_collision-safety-loss) などの追加コスト項を足しやすい(コストの加算で表現できる)。
- 凸最適化(線形コストの場合)として安定に学べ、見習い学習の理論で性能保証もある。

トレードオフ・限界
- **候補軌跡を事前サンプリングする必要があり、サンプル集合の質に性能が依存する**(良い軌跡が候補にないと選べない)。これが最大の弱点。
- 「コスト最小=最適」という前提が、複数の妥当解がある状況やマルチモーダルな選択では弱い(最良候補がほぼ同コストで競合し、選択が不安定になる)。
- マージン関数 `m(τ)` や候補生成の設計など、調整箇所が多い。
- 直接回帰型(BC)より計算とパイプラインが複雑。
- 学習信号は「専門家が最良」という相対比較なので、専門家自身が最良でない(人間の運転が必ずしも最適でない)場合、上限が専門家に縛られる。

研究上の未解決課題は、候補サンプリングを学習可能にする(微分可能サンプリングや候補生成自体の学習)、マルチモーダルなコスト極小を扱う、そしてコスト関数の学習を closed-loop の安全性に直結させる(オフライン損失と実走行性能の乖離を埋める)ことです。

## 発展トピック・研究の最前線

最大マージン計画は「コスト+候補評価+マージン学習」という古典的骨格ですが、近年は中間表現(占有・地図・動き予測)を強化する方向(P3/MP3)と、コスト型を生成型と融合する方向(拡散モデルで候補を生成しコストで選ぶハイブリッド、あるいは学習可能な軌跡アンカーを置く方式)に進んでいます。また、コスト学習の本質である逆強化学習は、敵対的アプローチ([GAIL (敵対的模倣学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0177_gail-adversarial-imitation))や最大エントロピーIRLとも理論的に地続きで、「報酬/コストをデータから学ぶ」という共通問題の異なる解法として比較されます。最大エントロピー版では argmin(硬い選択)を softmin(確率的選択)に置き換えると、マージン損失が占有測度マッチングと連続的につながることも知られています。安全制約をコストに足し込む設計は [衝突/安全制約損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0161_collision-safety-loss) と一体で、解釈可能で安全な計画の主要な研究潮流を形成しています。
