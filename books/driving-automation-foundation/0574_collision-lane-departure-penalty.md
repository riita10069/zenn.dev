# 衝突・車線逸脱ペナルティ項

## ひとことで言うと

自動運転モデルが出した運転計画(未来軌道)が「他の車にぶつかりそう」「車線からはみ出しそう」なときに、損失(モデルの間違い具合を表す数値)を余分に大きくして罰する追加の項のことです。お手本の運転をそっくり真似る通常の損失だけでは安全までは保証されないため、安全に直結する行動に直接ペナルティをかけ、学習を安全な方向へ誘導します。模倣損失が「平均的に上手か」を測るのに対し、ペナルティ項は「危険な瞬間がないか」を測ります。

## 直感的な理解

人間に運転を教えるとき、「上手な人と同じハンドルさばきを真似なさい」とだけ言って済ませる教官はいません。それに加えて「絶対に前の車にぶつけるな」「白線をはみ出すな」という、結果に対する強い禁止を教えます。前者は模倣、後者が制約です。

機械学習の運転モデルも、放っておくと「お手本の平均にだけ近づく」よう学びます。ところがここに落とし穴があります。軌道は時刻ごとの未来位置が連なった長い列で、損失はその全体の平均誤差で測られます。すると、6 秒ぶんの軌道のうち 60 ステップはお手本どおりでも、たった 4 ステップだけ障害物に突っ込む計画が、平均に埋もれて罰せられない、ということが起きます。平均的には優等生でも、一瞬の致命的ミスを見逃すのです。

そこで「ぶつかる」「はみ出す」という安全違反だけを取り出して、別枠で重く罰する補助項を足します。これが衝突ペナルティ・車線逸脱ペナルティです。発想としては、最適化において「禁止区域」を地形のくぼみ(谷)ではなく急な崖として作り込み、最適化が崖に近づくほど強い反発力を受けるように損失地形を整形する、と捉えると分かりやすいです。

## 基礎: 前提となる概念

理解に必要な用語をかみ砕きます。

- 模倣学習(imitation learning): 人間の運転ログ(お手本)と同じ行動を出すようモデルを訓練する方式。最も基本的な end-to-end 運転の学習法で、行動クローン(behavior cloning)とも呼ばれます。
- 主損失と補助損失(auxiliary loss): 主損失は学習の中心的な目的(ここでは「お手本を真似る」)。補助損失はそれを補う追加の目的(ここでは「安全違反を避ける」)。最終損失は両者の重み付き和です。
- 軌道の表現: 軌道は (x, y) の未来位置を並べたウェイポイント列で表すこともあれば、加速度・曲率といった制御量の列で表すこともあります。制御量表現は学習が安定しやすい一方、距離を計算するにはいったん運動学(車がどう動くか)で積分して (x, y) に戻す必要があります。
- 運動学積分: 加速度・曲率・初期速度から、時間刻み dt(たとえば 0.1 秒, 10Hz)で逐次積分し、各時刻の自車位置 (x, y) と向きを求める計算。自転車モデル(bicycle model)など簡易な車両運動学がよく使われます。
- HD マップ(高精細地図): 車線の境界線や中心線をセンチメートル精度で持つ地図。車線逸脱の判定に使います。
- 符号付き距離(signed distance): ある点が領域の内側か外側かを符号(内側なら負、外側なら正など)で表した距離。車線の内外判定に便利で、滑らかに微分できる量として扱えます。
- ヒンジ損失(hinge loss): `max(0, x)` の形で、しきい値を超えたぶんだけ罰し、超えなければ 0 にする損失。安全マージン違反だけを罰するのにちょうど合う形です。

模倣だけの弱点を整理すると次の 3 点です。第一に、お手本データには事故を起こした運転がほぼ含まれないため、モデルは「衝突するとどれだけ悪いか」を一度も教わりません(分布外の状態に対する罰の不在)。第二に、軌道全体の平均誤差を小さくしても、ごく一部のタイムステップで障害物に突っ込む計画は平均に埋もれて罰せられません。第三に、二乗誤差(MSE)は「数センチ車線をはみ出す」のと「中央をきれいに走る」のを連続的にしか区別せず、安全か危険かという離散的に重大な境界を強く意識しません。ペナルティ項はこの 3 点をいずれも補います。

## 仕組みを詳しく

ペナルティ項は典型的に次の流れで計算します。まず予測軌道を (x, y) の未来位置列に展開します。たとえば 64 ステップを 0.1 秒刻み(10Hz)で積分すれば、6.4 秒先までの自車の予定位置 64 点 `ego_xy[0..63]` が得られます。

衝突ペナルティ(collision penalty)

1. シーン中の他車・歩行者など各物体の予測未来位置を用意する(検出 + 動き予測モジュール、または学習時は正解 GT から)。
2. 各未来時刻 t で、自車の予定位置とすべての物体位置の距離を測る。
3. 距離が安全マージン(例: 2.0 m)より近いステップにペナルティを与える。

```
collision_loss = 0
for t in range(64):
    for obj in objects:
        d = distance(ego_xy[t], obj_xy[t])
        if d < safe_margin:                    # 例: 2.0 m
            collision_loss += (safe_margin - d) ** 2   # 近いほど重く罰する
```

距離が安全マージンより遠ければ 0、近いほど二次関数的に大きく罰します。`d` が 1.5 m でマージン 2.0 m なら `(0.5)^2 = 0.25`、0.5 m なら `(1.5)^2 = 2.25`、衝突寸前の 0.1 m なら `(1.9)^2 = 3.61` と、危険なほど急激に増えます。この「不感帯(マージン外は 0)+ 違反域での急増」という形状が、安全な領域では何も言わず、危険域だけ強く押し戻す挙動を生みます。

車線逸脱ペナルティ(lane departure penalty)

1. HD マップから自車線の左右境界線を取得する。
2. 各時刻 t で、自車の予定位置が境界の内側にあるかを符号付き距離で測る。
3. 境界をはみ出した分(外側に出た距離)だけを罰する。

```
departure_loss = 0
for t in range(64):
    over = signed_distance_outside_lane(ego_xy[t])  # 内側なら<=0、はみ出すと>0
    if over > 0:
        departure_loss += over ** 2
```

最終損失は、主損失とこれらを重み付きで足し合わせます。

```
total = traj_loss + lambda_c * collision_loss + lambda_d * departure_loss
```

`lambda_c`, `lambda_d` は各ペナルティの強さを決める係数(ハイパーパラメータ)です。大きすぎると安全を気にしすぎて動けなくなり(過度に保守的、止まりがち)、小さすぎると効きません。実務ではバランス探索が必要で、学習初期は係数を小さくして徐々に強める(warm-up)設計や、模倣損失が十分下がってからペナルティを足し始める段階的学習(curriculum)も使われます。

before(ペナルティ無し): お手本に平均的に近ければ、たまに障害物と重なる計画でも損失は小さい。
after(ペナルティ有り): 同じ計画でも衝突ステップで損失が跳ね上がり、最適化が「ぶつからない軌道」へ押し戻される。

微分可能性についての注意があります。`if d < margin` のような条件分岐そのものは微分できませんが、`max(0, margin - d)^2`(ヒンジ二乗)のように書けば、違反域では滑らかに微分でき、勾配が軌道デコーダまで流れます。距離計算自体も (x, y) について微分可能なので、運動学積分を微分可能に実装すれば、ペナルティの勾配は元の制御量(加速度・曲率)の予測まで遡って効きます。ここが設計の肝で、「罰したい現象(衝突)」と「勾配を流したい変数(制御量の予測)」を微分可能な経路でつなぐことで、ペナルティが学習信号として機能します。

ペナルティには大きく二つの実現様式があります。ソフト制約(soft constraint)は上記のように損失に足し込み、最適化が「なるべく避ける」よう誘導します。ハード制約(hard constraint)は衝突する軌道を候補集合から物理的に除外する、あるいは後段の安全フィルタで安全集合へ射影し直す方式で、「絶対に出させない」ことを目指します。ソフトは勾配で滑らかに学習を導けるが違反を完全には防げず、ハードは違反を防げるが滑らかな学習信号になりにくい、という相補関係にあります。

## 手法の系譜と主要論文

安全制約を損失に明示的に組み込む流れは、コストマップ系から幾何制約系へ、さらに制御理論的な安全層へと進化してきました。

- NMP(Zeng et al., "End-to-end Interpretable Neural Motion Planner", CVPR 2019)が先駆けです。空間に学習可能なコスト場(cost volume)を作り、障害物や車線逸脱が起きる領域のコストを高くしておき、その上で最小コストの軌道を選ぶ枠組みを提案しました。max-margin 損失で「人間の軌道のコスト < サンプルした他の軌道のコスト」を満たすよう学習します。これは「ペナルティを場として学習する」最初期の形でした。
- ST-P3(Hu et al., "ST-P3: End-to-end Vision-based Autonomous Driving via Spatial-Temporal Feature Learning", ECCV 2022)は、知覚・予測・計画を時空間特徴で統合し、計画段階で占有(occupancy)を使った衝突回避コストを取り入れました。視覚のみ(LiDAR なし)で end-to-end の計画まで通した点が新しく、nuScenes と CARLA の両方で評価されました。
- UniAD(Hu et al., "Planning-oriented Autonomous Driving", CVPR 2023, Best Paper)は、検知・追跡・地図・占有予測・動き予測・プランニングを 1 つのネットワークに統合し、最終のプランニング損失に模倣項(L1 系)に加えて collision-aware な項(占有グリッドとの重なりを罰する)を入れます。提案理由は、各サブタスクを最終目的(安全な運転計画)に向けて協調させること。タスクを個別に最適化するのではなく、最終のプランニングに資する向きで全タスクを学習する「planning-oriented」という設計思想を打ち出しました。
- VAD(Jiang et al., "VAD: Vectorized Scene Representation for Efficient Autonomous Driving", ICCV 2023)は、シーンをベクトル(点や線の集合)で表し、その幾何を使って ego-agent collision 制約・ego-boundary(車線境界)制約・ego-lane direction(車線方向)制約という複数の安全制約損失を明示的に定義します。ラスタ(画素グリッド)表現より幾何的距離を直接・軽量に計算できるのが利点で、これが後述の大幅な高速化につながりました。
- BEV-Planner(Li et al., "Is Ego Status All You Need for Open-Loop End-to-End Autonomous Driving?", CVPR 2024)は、安全制約損失そのものより評価のあり方に切り込んだ批判的研究ですが、衝突率を測る際の自車・他車の形状近似(点か矩形か)や時間粒度がスコアを大きく動かすことを示し、ペナルティ設計の前提を問い直しました。
- VADv2 や近年の生成系プランナ(軌道を多峰の分布として扱う手法)では、安全制約をハード制約(衝突軌道を候補から除外)とソフト制約(ペナルティで罰する)の両面から扱う設計が広がっています。さらに制御理論側からは、制御バリア関数(Control Barrier Function, CBF; Ames et al., 2019)を学習方策の後段に置き、安全集合からの逸脱を保証付きで防ぐ研究が合流しつつあります。

## 論文の実験結果(定量データ)

評価で中心となる指標は次の 2 つで、いずれも nuScenes ベンチマークで報告されます。

- L2 誤差(L2 error): 予測軌道と正解軌道の位置ずれをメートルで測る。1 秒後・2 秒後・3 秒後の平均で表すことが多い。小さいほど正確。これは「お手本にどれだけ近いか」(模倣の質)を測ります。
- 衝突率(collision rate): 予測軌道に沿って自車を動かしたとき、他のエージェントや障害物と衝突する割合(%)。1s/2s/3s での平均で表す。小さいほど安全。これは「危険な瞬間がないか」を直接測ります。L2 が小さくても衝突率が高い、ということが起こり得る点が重要です。

報告されている代表的な数値を挙げます(nuScenes、3 秒平均、出典により集計法が異なるため傾向値として読んでください)。

- ST-P3 は nuScenes で 3 秒平均の衝突率を 1% 前後のオーダーまで下げ、占有を用いた衝突コストの有効性を示しました。
- UniAD は、衝突率の平均で従来手法(ST-P3 など)を大きく下回り、当時の最先端を更新しました。論文では平均衝突率がおよそ 0.3% 程度のオーダーまで下げられたと報告され、collision-aware な設計と planning-oriented な統合学習が衝突回避に効いたとされます。
- VAD は、ベクトル表現と複数の安全制約損失により、UniAD と同等以上の計画安全性(同オーダーの衝突率)を保ちつつ、推論速度を大幅に高速化しました。VAD-Base はおよそ 4〜5 倍の高速化(数 FPS から十数 FPS のオーダー)を達成したと報告されています。安全と速度を両立した点が新規性です。

アブレーション(どの要素を抜くとどうなるか)も明快です。VAD はインスタンス衝突制約・境界逸脱制約・車線方向制約を一つずつ外す実験を行い、各制約を抜くと衝突率や逸脱が悪化することを示しました。特に衝突制約を外すと衝突率が目に見えて上がり、「ペナルティ項が直接安全に効いている」ことの定量的な裏付けになっています。UniAD も、占有を使った衝突コストを外すと衝突率が悪化することを報告しています。

ここで指標の意味を補足します。衝突率の 0.3% という値は「1000 回の運転計画のうち約 3 回は衝突する計画を出す」という意味で、実用にはまだ遠い水準ですが、ペナルティの有無で 2 倍以上変わることが多く、その差は安全性能として決定的に重要です。一方、L2 誤差だけを追うと「平均的に上手だが時々ぶつかる」モデルを良しとしてしまうため、両指標を併記する評価が標準になっています。

ただし重大な注意があります。BEV-Planner(CVPR 2024)は、nuScenes の open-loop 評価で自車の過去運動だけを入力にした単純なモデルが、カメラと安全制約を使う複雑な手法に匹敵する L2 誤差・衝突率を出してしまうと報告しました。これは「nuScenes の軌道は直進が多く、過去の延長で当ててしまえる」というデータセット側の偏りと、衝突率の測り方(形状近似や時間粒度)に数値が敏感であることの両方を暴いたものです。つまりペナルティ項が「効いて見える」数値が、評価設計のアーティファクトである可能性まで疑う必要がある、というのが現在の到達点です。

## メリット・トレードオフ・限界

メリット

- 安全に直結する失敗(衝突・逸脱)を損失で直接罰せるため、模倣だけより安全方向に学習が寄る。
- 事故データがほとんど無い模倣学習の弱点を、幾何的な制約で補える。
- 平均に埋もれがちな「一瞬の致命的ミス」を局所的に拾える。

トレードオフ・限界

- 計算に他物体の未来位置・車線境界・運動学積分が必要で、依存先(検出・予測・HD マップ)の品質に結果が左右される。検出が物体を見落とせば、衝突ペナルティはそもそも発火しません。
- 重み係数のチューニングが難しい。強すぎると過度に保守的(止まりがち、急ブレーキ多発)になり、弱すぎると効かない。
- 学習時に正解 GT の他物体位置を使い、推論時には自前の検出・予測を使うため、学習と本番で条件がずれる(GT-gap)。
- 制御量(加速度・曲率)表現のままでは距離が計算できず、(x, y) への運動学積分が必要で、その変換に数値誤差が乗る。
- 衝突率指標そのものに測定上の癖があります。占有グリッドの解像度や、自車・他車を点で近似するか矩形で近似するかで数値が変わり、論文間の単純比較を難しくしています(BEV-Planner の指摘)。

未解決の課題として、ハード制約とソフト制約のバランス、相互作用(他車が自車の行動に反応する)を考慮した衝突予測、そして「保守的すぎて流れに乗れない」過剰防衛をどう避けるか、が研究の焦点になっています。さらに、open-loop の衝突率が閉ループ(実走行)の安全と乖離する問題が大きく、ペナルティの効果を閉ループでどう検証するかが本質的に残されています。

## 発展トピック・研究の最前線

- 占有予測ベースの衝突回避: 物体を個別に検出せず、空間の占有確率を予測してその上で衝突コストを定義する方向。検出漏れに強く、未知物体にも反応できる。
- 相互作用認識プランニング: 他エージェントが自車の計画に反応する(ゲーム理論的)前提で衝突を評価する研究。固定された未来位置を仮定する素朴なペナルティの限界を超えようとしています。
- 微分可能シミュレーション・閉ループ評価: open-loop の衝突率は実走行の安全と必ずしも一致しないという批判があり、シミュレータ内で実際に走らせて衝突を測る閉ループ評価へ重心が移りつつあります。
- 安全層(safety filter)/ 制御バリア関数(CBF): 学習した軌道を後段で安全集合に射影し、衝突軌道を物理的に出させない手法。ソフトなペナルティと組み合わせる設計が増えています。Ames et al. の CBF は、安全集合の不変性(一度安全なら安全のまま留まる)を制約として保証する枠組みで、学習ベースのプランナに数学的な安全保証を後付けする橋渡しになっています。
