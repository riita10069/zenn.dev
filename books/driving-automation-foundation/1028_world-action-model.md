# World Model(マルチビュー世界モデル)

## ひとことで言うと

「世界がこの先どう変わるか」をモデルの内部に予測させる仕組みです。運転の文脈では、複数カメラの映像から「次の瞬間に世界がどう見えるか」を学習表現の空間で予測させ、その予測能力を運転判断の質に活かします。未来予測を鳥瞰図(BEV)空間で行う設計から、各カメラのマルチビュー特徴で直接行う設計へ移す流れがあり、本ノートはその構成と移行の理由を扱います。

## 直感的な理解

熟練ドライバーは、目の前の映像を見ながら無意識に「あの車はこのまま進む」「あの歩行者は止まりそうだ」と未来を予測しています。この予測能力こそが、安全で滑らかな運転の核です。同じ能力をニューラルネットワークにも持たせたい、というのが世界モデル(world model)の動機です。

世界モデルとは「現在の状態と自分の行動から、未来の状態を予測するモデル」です。これを学習させると2つの恩恵があります。第一に、未来を予測できる特徴は「いま何が起きているか」を深く理解した特徴です。その良質な特徴を、どう走るかを決める軌道計画と共有すれば、計画の質も底上げされます。第二に、予測が外れたときに「想定外のことが起きた」と気づけるので、これを安全シグナル(自己点検、introspection)に使えます。

たとえるなら、世界モデルは「頭の中で先読みする力」です。チェスの強い人が数手先を読むように、運転モデルにも「この先どうなるか」を内部でシミュレートさせ、その先読みで現在の判断を研ぎ澄ます、というわけです。

## 基礎: 前提となる概念

特徴抽出器(backbone): 画像を、意味が凝縮された特徴ベクトルの並び(特徴マップ)に変換するニューラルネットワークです。Swin Transformer や ConvNeXt、ResNet などがよく使われます。

マルチビュー: 1台のカメラではなく、車体の周囲に取り付けた複数台(たとえば前後左右7台)のカメラ画像をまとめて扱うことです。形状は `[B, N, 3, H, W]`(バッチ、カメラ台数N、RGBの3チャンネル、高さ、幅)で表します。

BEV(Bird's Eye View、鳥瞰図): 複数カメラの映像を、幾何変換とカメラパラメータで「上から見下ろした地図」状の格子特徴に統合した表現です。空間的に解釈しやすい反面、生成に重い幾何処理が必要で、テンソルも `[B, C, 数百, 数百]` と大きくなります。

行動条件つき予測(action-conditioned prediction): 「自分がこの行動を取ったら、世界はこう変わる」という、自分の行動を入力に含めた未来予測です。受動的に「世界がどうなるか」を当てるだけでなく、能動的に「自分の行動次第で未来がどう分岐するか」を扱える点が、計画にとって本質的です。

表現崩壊(representation collapse): 予測する側と正解を作る側を両方とも自由に学習させると、「両方が常に同じ定数を出せば予測誤差はゼロ」という抜け道が生まれ、特徴が無意味な定数に潰れてしまう現象です。これを防ぐため、正解側は勾配を止める(stop-gradient)か、予測側の重みを緩やかに追従させる EMA(指数移動平均)で更新する、という非対称設計を取ります。

## 仕組みを詳しく

運転向けエンドツーエンドモデルに世界モデルを組み込む典型的なパイプラインは、おおよそ次の流れです。

```
N台のカメラ画像 [B, N, 3, H, W]
   → Backbone(特徴抽出器)              : 各カメラの特徴
   → Feature Fusion(BEV 統合)          : 鳥瞰図特徴 [B, C, Hbev, Wbev]
   → Map Encoder/Fusion(ナビ地図と融合)
   → Temporal Memory(過去履歴の圧縮)
   → Trajectory Planner(GRU / Flow Matching 等) : 軌道 + ego_hidden [B, C]
   → World Model(未来予測, ego_hidden で条件づけ) : 未来特徴 × 数時刻
```

世界モデルの予測器の核は「ego_hidden による条件づけ」です。`ego_hidden` は軌道計画器(たとえば再帰ネットワーク GRU)が出す数百次元の隠れ状態で、「モデルがこれから走ろうとしている意図」を要約しています。これを未来特徴予測に注入すると、「自分がこの軌道を走ったら、世界はこう見えるはず」という行動条件つきの予測になります。実装の骨子は次のようになります。

```
bias       = ego_proj(ego_hidden).view(B, C, 1, 1)   # 意図を per-channel バイアス化
conditioned = current_features + bias                 # 現在特徴に意図を加える
future      = predict_future(conditioned)             # 未来特徴を出力
future_per_timestep = split(future, num_timesteps)    # 複数時刻に分割
```

さらに、計画した軌道そのもの(`[B, num_timesteps × num_signals]`)を2つ目のバイアスとして注入する設計もあります。これが効くと「軌道A vs 軌道Bで世界はどう変わるか」という反実仮想(counterfactual)のロールアウトが可能になり、いわゆる World *Action* Model(行動つき世界モデル)になります。

学習は、予測した未来特徴と、実際の未来フレームを frozen backbone(学習で動かさない固定の特徴抽出器)に通して作った正解特徴とを、MSE や L1 で比較します。frozen を使う理由は前述の表現崩壊を防ぐためで、これは BYOL や JEPA で確立された非対称設計です。各未来時刻に重みを付け、遠い未来ほど不確実なので損失を割り引く工夫([未来ホライズンの重み付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0563_jepa-temporal-horizon-weighting))も併用されます。

BEV から「マルチビュー特徴で直接予測する」設計へ移る動機は実務的です。BEV 特徴は `[B, C, 数百, 数百]` と巨大で、未来を複数時刻ぶん予測すると正解・勾配中間結果まで含めてメモリが破綻します([JEPAターゲットのメモリ圧縮戦略](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0577_jepa-memory-budget-compression))。また BEV 変換には重い幾何処理とカメラパラメータが要り、未来予測のためだけにこの経路を通すのは過剰です。そこで「未来予測は、BEV に投影する前のマルチビュー特徴(各カメラごとの特徴)で行えばよい」と整理すると、予測対象が軽くなり、未来フレームの正解も各カメラ画像を backbone に通すだけで作れ、データ供給(過去履歴と未来正解を逐次フレームで渡す仕組み)とも素直につながります。

## 手法の系譜と主要論文

世界モデルの系譜は、強化学習・生成モデル・自己教師あり表現学習が合流しています。

概念の出発点の一つが Ha & Schmidhuber の "World Models"(2018, arXiv:1803.10122)です。観測を圧縮する VAE、時系列を予測する RNN、その上で行動を決めるコントローラを分け、「潜在の中で先読みして方策を学ぶ」枠組みを提示しました。Dreamer 系(Hafner et al., 2020-2023)はこれを発展させ、潜在空間の中での想像(imagination)ロールアウトで強化学習する手法を確立しました。

運転特化では MILE(Hu et al., Wayve, NeurIPS 2022, arXiv:2210.07729)が、潜在世界モデルがピクセル予測ベースラインを運転性能で上回ることを実証しました(詳細は [MILE(運転向け潜在世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1215_mile-latent-world-model))。GAIA-1(Hu et al., Wayve, 2023, arXiv:2309.17080)は動画・行動・テキストを統合した自己回帰的な生成世界モデルで、行動条件つきの未来動画生成を実現しました。行動条件づけと反実仮想ロールアウトという発想は、ここに直接の源流があります。トレードオフは生成モデルゆえの計算量の大きさです。

自己教師あり表現学習の文脈では、LeCun の JEPA 提案(2022)を起点に、I-JEPA(Assran et al., CVPR 2023, arXiv:2301.08243)が画像で、V-JEPA(Bardes et al., 2024, arXiv:2404.08471)が動画で、「ピクセルではなく学習表現の空間で未来・欠損を予測する」枠組みを定式化しました。未来特徴を表現空間で予測する設計はこの系統に属します。

計画指向のエンドツーエンド設計では UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)が、知覚・予測・計画を一つのネットに統合し、検出・追跡・地図などの補助タスクが共有 BEV 表現を良くして最終的な計画性能を上げることを示しました。「世界モデルを補助タスクとして本線の表現を底上げする」狙いと同じ思想です。

## 論文の実験結果(定量データ)

MILE(arXiv:2210.07729)は CARLA(運転シミュレータ)の評価で、当時の最先端手法を上回る運転スコア(ルート完走率や違反の少なさを統合した指標、大きいほど良い)を報告しました。重要なのは、潜在世界モデルがピクセル予測ベースラインを運転性能で上回るというアブレーションで、「未来を表現空間で予測する」選択の妥当性を直接示しています。

GAIA-1(arXiv:2309.17080)は、約4700時間規模の実走行データから学習し、テキストや行動の指示に応じた多様で長尺の未来動画を生成できることを示しました。定量的なフレーム品質に加え、「同じ現在から行動を変えると未来が分岐する」という行動条件性を定性的・定量的に示し、行動つき世界モデルの実在性を裏付けました。

UniAD(arXiv:2212.10156)は nuScenes(都市運転の大規模データセット)で、知覚・予測・計画を統合した結果、計画の平均 L2 誤差(予測軌道と実際の軌道の距離、メートル、小さいほど良い)と衝突率(計画軌道が他物体とぶつかる割合、小さいほど良い)の両方を、各タスクを別々に解く構成より改善しました。アブレーションでは、補助タスク(追跡・地図・動作予測)を一つずつ外すと計画性能が下がることが示され、「補助タスクが本線を底上げする」効果を定量的に証明しています。世界モデルを補助タスクとして使う発想の数値的根拠です。

I-JEPA(arXiv:2301.08243)は ImageNet-1k の線形評価で、ピクセル再構成型の自己教師あり学習に匹敵する精度を、はるかに少ない事前学習計算量で達成しました。表現空間で予測する効率の良さを示す結果で、未来特徴予測の前提を支えます。

## メリット・トレードオフ・限界

メリット:
- 未来予測という補助タスクで、軌道計画と共有する特徴の質が上がる。
- 行動条件づけにより「もし別の軌道なら」という反実仮想が扱え、計画の検討材料になる。
- 予測誤差をそのまま安全シグナル(イントロスペクション)に流用できる。
- 補助タスクとして任意に有効/無効を切り替えられる設計なら、既存の挙動を壊さず段階導入できる。

トレードオフ・限界:
- 未来正解の供給に逐次フレーム(過去履歴と未来)が要り、データパイプラインが複雑化する。
- frozen/EMA ターゲットやメモリ圧縮など、表現崩壊やメモリ破綻を防ぐ付帯機構が必須で実装量が増える。
- BEV を未来予測経路から外すと、空間的に解釈しやすい予測表現は得にくくなる(マルチビュー特徴は人間には読みにくい)。
- 補助損失の重み(λ)を誤ると、軌道精度を犠牲にして未来予測へ過適合する恐れがある。

未解決の課題は、未来の本質的な多義性(同じ現在から複数の妥当な未来があり得る)をどう扱うかです。決定論的な MSE 予測は複数未来の「平均」を当てに行きぼやけます。確率的世界モデルや生成的世界モデルがこの緩和策ですが、計算量との両立が研究上の論点です。

## 発展トピック・研究の最前線

生成的世界モデルの大規模化: GAIA 系のように、運転を動画生成として捉え、行動やテキストで条件づけて長尺の未来を生成する方向が活発です。生成された未来をシミュレータ代わりに使い、稀な危険シーンを合成してデータ拡張する応用も進んでいます。

予測と計画の統合: 世界モデルの予測を計画の中に明示的に折り込み、「複数の候補軌道それぞれで未来をロールアウトし、最も安全・滑らかな軌道を選ぶ」モデル予測制御(MPC)的な使い方が研究されています。行動条件づけはこの方向の前提条件です。

潜在トークン化とスケール則: BEV 格子の代わりに、物体中心の潜在スロットや少数の潜在トークンで世界を表し、Transformer で時系列予測する設計が増えています。言語モデルで見られたスケール則(データ・計算を増やすと性能が予測可能に伸びる)が運転世界モデルでも成り立つか、が大きな関心事です。
