# CoAtNet (畳み込みと注意の統合)

## ひとことで言うと
CoAtNet(コートネット、"Convolution and Attention" を縮めた名前)は、画像認識モデルで「畳み込み(CNN の部品)」と「自己注意(Transformer の部品)」を、どちらか一方ではなく、原理から導かれた順番で組み合わせたモデルです。Google の研究で、データが少ないときは CNN のように、データが多いときは Transformer のように振る舞い、双方の長所を両立させました。論文タイトルの "for All Data Sizes"(あらゆるデータ規模に向けて)が主張を端的に表しています。発表当時、JFT-3B という超大規模データを使った設定で ImageNet top-1 約 90.88% という世界記録を樹立した点でも注目されました。

## 直感的な理解
画像 AI には長らく二つの流派がありました。CNN は「近所だけを見る職人」です。少ない経験でも筋のよい仕事をしますが、現場全体を見渡す視野は狭い。Transformer は「全員と会議する調整役」です。視野は広く、データが豊富なら抜群の表現力を発揮しますが、経験が浅いと当てずっぽうになりがちです。

ここで素朴な疑問が湧きます。「両方を雑に混ぜれば最強では」。しかし雑に混ぜると、どちらの良さも中途半端になります。CoAtNet の貢献は、混ぜ方を二つの原理から導いたことです。第一に、注意の計算式の中に畳み込みの「近所重視」を足し込んで一体化する。第二に、画像処理の浅い段(高解像度でトークンが膨大)では計算の軽い畳み込みに任せ、深い段(低解像度でトークンが少ない)で初めて注意を使う。この順番が、汎化性(少データでも当たる力)と容量(データが多いほど伸びる上限)を両立させる鍵だと、実験で体系的に示しました。

ここで強調したいのは、CoAtNet が「思いつきで設計を試した」のではなく「畳み込みと注意の数式を並べて比較し、共通の枠組みに統一してから、どこをどう混ぜるべきかを演繹した」点です。設計を職人芸ではなく理屈に落とし込もうとした論文で、これがハイブリッドバックボーン研究の到達点の一つとされる理由です。

## 基礎: 前提となる概念
用語を順に噛み砕きます。

畳み込み(convolution)は、画像の小さな窓を順にスライドさせて近所のピクセルだけから特徴を作る処理です。「近くが大事(局所性)」「画像を少しずらしても同じ物体は同じ(平行移動同変性)」という前提を構造として持ちます。窓の重み(カーネル)を画像のどこでも使い回す重み共有が、この同変性の源です。

自己注意(self-attention)は、各トークン(画像パッチなどの単位)が他の全トークンとの関係の強さを計算し、関係が深い相手の情報を重く取り込む仕組みです。各トークンから Query(問い合わせ)、Key(鍵)、Value(値)を作り、Query と Key の内積で関係の強さを測り softmax で正規化して Value を加重平均します。視野は画像全体に届きますが、計算量はトークン数 N の二乗 O(N^2) です。畳み込みと違い、重みが「内容に応じて」その場で変わるのが特徴で、これが高い表現力の源泉です。

汎化性(generalization)とは、学習に使っていない未知のデータでも正しく予測できる力です。帰納バイアス(モデルにあらかじめ組み込まれた前提)が正しいほど、少ないデータでも汎化します。CNN の局所性バイアスはこれを助けます。

容量(capacity)とは、モデルが表現できる関数の複雑さの上限です。容量が大きいほど、データが豊富なときに細かい規則まで覚え込めます。自己注意は重みを内容に応じて柔軟に変えられる(入力依存の動的な重み)ため、容量が大きい側です。一方、畳み込みは重みが固定(入力に依らない静的な重み)なので、容量はやや小さい代わりにデータ効率が高い。

この二つはしばしばトレードオフの関係にあります。CNN は汎化性が高いが容量が頭打ち、Transformer は容量が大きいが少データでは汎化しにくい。CoAtNet(Dai, Liu, Tan, Le; Google Research, NeurIPS 2021)の問題意識は、この二者を場当たり的でなく原理から両立させることでした。

MBConv とは、MobileNet 由来の軽い畳み込みブロックで、深さ方向分離畳み込み(チャネルごとに空間方向の畳み込みをかけてから 1x1 でチャネルを混ぜる)を中心に、いったんチャネルを増やしてから戻す inverted bottleneck 構造と、入出力を足す残差接続を持ちます。EfficientNet の主要部品でもあります。CoAtNet が畳み込み段に MBConv を選んだのは、MBConv の中の 1x1 conv → depthwise conv → 1x1 conv という構造が、Transformer の FFN(全結合の中間層)と数式的に似た「拡張してから戻す」形をしており、注意ブロックと並べたときに構造が揃って扱いやすいからです。

## 仕組みを詳しく
CoAtNet は二つの設計判断からなります。

### 判断1: 深さ方向畳み込みと自己注意を「足し算で統合」する
CoAtNet は、畳み込みと相対位置付き自己注意を一つの式に統合します。ある位置 i が位置 j を見るときの重みを比べます。

- 純粋な注意: `A[i,j] ∝ exp(query_i · key_j)`(内容の類似で決まる、全位置に届く)
- 純粋な深さ方向畳み込み: `w[i - j]`(相対位置 i-j だけで決まる、内容に依らない固定重み、近傍に限定)
- CoAtNet(統合版、相対注意): `A[i,j] ∝ exp(query_i · key_j + w[i - j])`(内容と相対位置を足してから正規化)

ここで `w[i-j]` は相対位置ごとに学習される実数(相対位置バイアス、relative attention bias)です。記号の意味を言葉で言うと、`query_i · key_j` は「内容がどれだけ似ているか(内容項)」、`w[i-j]` は「位置がどれだけ近いか・どの方向か(位置項)」を表し、これらを足してから softmax で正規化します。結果として「近いものは内容に関わらず少し重視する(畳み込み的)」かつ「内容が関係するものはさらに重視する(注意的)」を一つの注意で実現します。畳み込みの帰納バイアスを注意に注入した、と理解できます。

なぜ「足し算で」統合するのが筋がよいのか。論文は、純粋な畳み込みは相対位置だけで重みが決まるので入力に対する一般化は強いが内容に応じた柔軟さがなく、純粋な注意は内容に応じて柔軟だが位置の常識を持たない、と整理します。両者を softmax の中で足すと、位置項 `w[i-j]` が全データで共有される事前知識として効きつつ、内容項が各サンプルごとに調整を加えるという、二つの良さが分離せずに同居します。これが「足し算統合」の理屈です。

### 判断2: 階層の浅い側に畳み込み、深い側に注意を置く
CoAtNet は 5 段(S0〜S4)の階層を持ち、解像度を順に半分にしていきます。入力 `[B, 3, 224, 224]` の例:

- S0(stem): 通常の畳み込み → `[B, 64, 112, 112]`
- S1: MBConv(畳み込み) → `[B, 96, 56, 56]`(トークン相当 3136)
- S2: MBConv(畳み込み) → `[B, 192, 28, 28]`(784)
- S3: Transformer(相対注意) → `[B, 384, 14, 14]`(196)
- S4: Transformer(相対注意) → `[B, 768, 7, 7]`(49)

著者は S2/S3/S4 のどこから注意を始めるか、つまり畳み込み段(C)と注意段(T)の並べ方を総当たりで比較し、C-C-T-T(S1/S2 が畳み込み、S3/S4 が注意。S0 を除いて数える表記)が最良と示しました。理由は二つです。

1. 自己注意は計算量 O(N^2)。浅い層は解像度が高くトークンが膨大(56x56=3136)で、ここで注意を使うと計算が爆発します。だから浅い層は計算の軽い MBConv に任せ、トークンが十分減った深い層(14x14=196、7x7=49)で注意を使う。S3 の 196 トークンなら全トークン注意でも約 3.8 万関係で現実的です。
2. 汎化の観点でも、浅い層では局所的なエッジ・模様を捉えれば十分で、畳み込みの局所性がむしろ汎化を助けます。深い層では物体同士の大域的な関係が重要になり、ここで注意が活きます。

この「畳み込み → 注意」の順番が、CoAtNet の汎化性と容量の両立を生みます。なお各段の境界では解像度を半分に落としチャネルを倍にする、CNN 流の縮約(ダウンサンプリング)を行います。CoAtNet には大きさ違いのモデル(CoAtNet-0 から CoAtNet-7 まで)があり、段の繰り返し数とチャネル幅を増やしてスケールさせます。

```
入力 224x224
  S0 conv  112x112  64ch
  S1 MBConv 56x56   96ch   ← 畳み込み(C)
  S2 MBConv 28x28  192ch   ← 畳み込み(C)
  S3 相対注意 14x14 384ch  ← 注意(T)
  S4 相対注意  7x7  768ch  ← 注意(T)        並びは C-C-T-T が最良
```

## 手法の系譜と主要論文
- EfficientNet(Tan & Le, ICML 2019, arXiv:1905.11946): MBConv を主部品とする純畳み込みネット。深さ・幅・解像度を複合スケーリング(compound scaling)し、当時のパラメータ効率の最前線。汎化に強いが容量が頭打ちという、CoAtNet が乗り越えたい片側を代表します。
- ViT(Dosovitskiy et al., ICLR 2021, arXiv:2010.11929): 注意のみで容量は高いが、巨大データなしでは汎化しにくい。CoAtNet が両立させたいもう片側です。
- BiT / NFNet などの大規模 CNN: 大規模事前学習で CNN を強くする路線。CoAtNet はこれらとも比較対象になります。
- CvT(Wu et al., ICCV 2021, arXiv:2103.15808): 射影を畳み込みで行う初期ハイブリッド。CoAtNet は「ブロック単位で畳み込み段と注意段を分ける」点が異なります。CvT が各注意ブロックの内部に畳み込みを混ぜるのに対し、CoAtNet は段で役割を分業させます。
- CoAtNet(Dai, Liu, Tan, Le; Google Research, NeurIPS 2021, arXiv:2106.04803): 足し算統合と C-C-T-T 階層を提案。あらゆるデータ規模で強いことを実証。
- MaxViT(Tu et al., ECCV 2022, arXiv:2204.01697): CoAtNet の後継的位置づけ。同じ Google チームの流れで、段で分けるのではなく、全段で局所(Block 注意)と大域(Grid 注意)を線形計算量で両立させる方向に進化させました。CoAtNet の「深い段でしか大域を見られない」制約を解いた格好です。

## 論文の実験結果(定量データ)
主舞台は ImageNet 系の分類で、指標は top-1 accuracy(最も確信度の高い予測が正解と一致した割合)。データ規模を変えて「少データでも多データでも強い」ことを示したのが要点です。

- ImageNet-1k のみ(約 120 万枚)で学習: CoAtNet-3(約 1.68 億パラメータ)が top-1 約 84.5%。同規模帯の ViT 系・純 CNN 系を上回りました。小型の CoAtNet-0(約 2500 万)でも約 81.6% と、パラメータ効率が高いことを示しています。
- ImageNet-21k(約 1300 万枚)で事前学習 → ImageNet-1k に転移: CoAtNet-3 が top-1 約 87.6%。
- JFT-300M(約 3 億枚)で事前学習: CoAtNet-4 で top-1 約 89.1%。
- JFT-3B(約 30 億枚という非公開の超大規模データ)で事前学習: 最大の CoAtNet-7(約 24 億パラメータ)が ImageNet top-1 約 90.88% を報告し、発表当時の最高記録を更新しました。同条件で大型 ViT(ViT-G/14、約 18 億パラメータ、約 90.45%)を上回り、しかも事前学習の計算量(TPU 時間)を抑えて到達した点が、容量側でも勝てる根拠とされました。

この「少データの 84.5% から超大データの 90.88% まで一貫して強い」という挙動こそが "for All Data Sizes" の実証です。多くのモデルはどちらか一方の規模に最適化されがちで、両端で同時に最前線というのは当時珍しいものでした。論文は精度をデータ規模に対してプロットした曲線を示し、CoAtNet の曲線が ViT の曲線を小規模側でも大規模側でも上回って通ることを可視化しています。

アブレーション(どの要素を抜くと何が起きるか)も豊富です。

- 注意を始める段を変えた比較で、C-C-T-T が最良。注意を浅くから入れすぎる(C-T-T-T)と計算が重くなるうえ少データでの汎化が落ち、逆に注意を遅らせすぎる(C-C-C-T)と大データでの伸びが鈍る、という双方向の劣化を示しました。最適点が中間にあることを定量的に裏づけています。ここは「両端を試したら真ん中が良かった」という単なる経験則ではなく、汎化と容量のトレードオフから予測される結果が実験で確認された、という構成になっています。
- 相対位置バイアス `w[i-j]` を外す(純粋な内容のみの注意に戻す)と精度低下。畳み込み由来の位置バイアスを注意に足す統合が効いていることを示します。
- データ規模を増やしたときの精度の伸び方を ViT と並べ、CoAtNet のほうが小規模で高く、大規模でも追い抜かれないことを曲線で示しました。

数値の意味を補足すると、ImageNet top-1 で 90% 前後の領域では 0.1〜0.3 ポイントの差が当時の記録更新に相当しました。飽和に近い高精度帯では誤りの残りが減るほど 1 件の改善が難しくなるため、0.4 ポイント(ViT-G の 90.45% → CoAtNet-7 の 90.88%)の差は大きな意味を持ちます。84.5%(1k のみ)という数字も、外部データなしでこの水準は当時の上位です。

## メリット・トレードオフ・限界
メリット
- 少データ(ImageNet-1k)でも多データ(JFT-3B)でも高精度という、規模をまたいだ汎用性。
- 浅い段を畳み込みにすることで、高解像度入力でも計算量を抑えられる。注意を 196/49 トークンの低解像度段に閉じ込めるのが効く。
- 畳み込みの局所バイアスが汎化を助け、注意の大域性が表現力の上限を引き上げる。両者の良さを構造として束ねている。
- 設計が原理から導かれているため、なぜそうするかの説明がつき、スケールアップの指針が立てやすい。

トレードオフ・限界
- 段ごとに畳み込みブロックと注意ブロックを切り替える設計で、単純な積み重ね(ResNet など)より実装・調整が複雑。
- 注意を始める最適な段はデータ規模に依存し、自動では決まらない。新しいデータ規模では手動探索が要る。C-C-T-T は ImageNet 級で最適でも、別ドメインで最適とは限りません。
- 浅い段が畳み込みのため、最初の段階では大域的な関係を直接捉えられない。深い段に到達するまで大域情報が出てこないのは、後の MaxViT が解こうとした弱点です。
- 注意段を含むため、純粋 CNN より推論時のメモリ使用が増えやすい。とくに高解像度転移時に顕著。
- JFT-3B のような非公開超大規模データに依存する最高記録は再現が難しく、公開データだけでの追試には限界がある。研究の再現性という観点では弱点です。

研究上の未解決課題として、畳み込みと注意の配分・段境界をデータとタスクに応じて自動最適化する一般原理は確立しておらず、依然として経験則とアブレーションに頼っています。また「足し算統合」が最良の混合方法かどうか(乗算やゲート機構との比較)も完全には決着していません。

## 発展トピック・研究の最前線
CoAtNet が整理した「段で畳み込みと注意を分ける」設計は、その後「一つのブロック内で局所と大域を両立させる」方向(MaxViT の Block/Grid 注意)へと発展しました。両者は同じ Google の流れで、CoAtNet → MaxViT は「大域注意を深い段に限定する制約を、線形計算量の多軸注意で全段に解放する」という連続した進化として読めます。別系統では、注意の中でプーリングして解像度を下げる MViT/MViTv2 が動画まで含めて階層化を推し進めています。

さらに ConvNeXt(Liu et al., CVPR 2022)のように「純畳み込みのまま Transformer の設計思想(大きなカーネル、LayerNorm、GELU、少ない活性化)だけ取り込む」逆方向の流れもあり、ハイブリッドが必須か否かという論争を生みました。ConvNeXt が注意なしで CoAtNet 級の精度に迫ったことは、「CoAtNet の強さは注意そのものより、設計の整理にあったのではないか」という問いを残しました。

最前線では、自己教師あり事前学習(MAE, BEiT など)との組み合わせ、検出・セグメンテーションへの統一バックボーン化、相対位置バイアスの効率実装(FlashAttention 系との両立)、実機スループットの最適化が継続テーマです。加えて、自己注意を線形時間の状態空間モデル(Mamba)や効率的注意に置き換えたうえで畳み込みと混ぜる「次世代ハイブリッド」も現れており、CoAtNet が定式化した「局所処理と大域処理をどの段でどう混ぜるか」という設計問題は、部品が変わっても中心的な論点であり続けています。
