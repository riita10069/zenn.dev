# Flow Matching 速度損失

## ひとことで言うと

ランダムなノイズから本物のデータ (たとえば車がこれから進む軌跡) へ「まっすぐ移動する速度」を学習する生成手法です。学習では、ノイズと正解を直線でつないだ途中の点で「どっちへどれだけ動くべきか (速度)」を予測させ、その予測と正解速度の二乗誤差を最小化します。拡散モデルと同等の多峰な分布を生成できながら、学習がより単純で、推論ステップ数が少なく速いのが特徴です。

## 直感的な理解

「これから数秒間どう進むか」という軌跡を予測したいとします。難しいのは正解が1つに決まらないことです。交差点で「直進してもよいし右折してもよい」場面では、ありうる軌跡が複数あります (これを多峰性 = マルチモーダル、山が複数ある分布と呼びます)。

普通の回帰 (1つの軌跡を直接予測して MSE で合わせる) はこれが苦手です。「直進」と「右折」の両方が正解だと、平均をとった「まっすぐ突っ込む (どちらでもない危険な軌跡)」を出してしまいます。これは平均化問題 (mode averaging) と呼ばれ、安全が要る領域では致命的です。

そこで生成モデル (確率分布そのものを学習し、そこからサンプリングする手法) を使います。代表が拡散モデル ([拡散モデル denoising 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0168_diffusion-denoising-loss)) ですが、生成時に何十〜何百ステップもノイズ除去を繰り返す必要があり遅いという弱点がありました。

Flow Matching (フローマッチング) は、この多峰生成能力を保ちつつ、より速く・素直に学習する方法です。発想は「ノイズの分布から、データの分布へ、点を連続的に流して (flow) 運ぶ速度場 (各点での進む向きと速さ) を学ぶ」こと。たとえるなら、川の流れ (速度場) を学んでおけば、上流 (ノイズ) に葉っぱを落とせば下流 (データ) に流れ着く、というイメージです。学んだ速度場に従って点を運ぶ作業 (ODE を解く) が生成にあたります。

## 基礎: 前提となる概念

- 速度場 / ベクトル場 (velocity field) v(x, t): ある時刻 t (生成の進行度 0〜1)、ある点 x にいるとき「次にどの方向へどれだけ動くか」を表すベクトル。これを学習します。
- 確率フロー (probability flow): ノイズの分布を、時間とともに連続的に変形してデータの分布に一致させる「流れ」。速度場に従って点を動かすと実現されます。各時刻 t での点の分布 p_t を 確率パス (probability path) と呼び、p_0 がノイズ、p_1 がデータです。
- 常微分方程式 (ODE): `dx/dt = v(x, t)` という形の式。「各時刻での速度が決まれば、初期位置からの軌道が一意に決まる」というもの。生成時はこれを数値的に解きます。
- オイラー法 (Euler method): ODE を解く最も単純な方法。微小ステップ dt で `x ← x + dt * v(x, t)` を繰り返す数値積分です。より高精度なソルバ (中点法、Runge-Kutta、Heun 法など) を使えば少ないステップで精度を上げられます。
- 条件 (conditioning) c: 「どんな状況か」を表す入力。自動運転なら周囲環境の特徴・自車運動履歴など。同じノイズでも条件が違えば違う出力へ流れるよう学びます。

拡散モデルとの関係も押さえます。拡散モデルは「スコア (確率密度の対数の勾配 ∇ log p(x))」を学びますが、これは数学的には確率フロー ODE の速度場と1対1に対応します (Song et al. 2021 の確率フロー ODE が橋渡しになる)。つまり Flow Matching と拡散モデルは「同じ確率フローを、違う量を通して学んでいる」近縁の手法です。Flow Matching は速度場を直接 MSE で回帰するぶん、目的関数がシンプルになります。

## 仕組みを詳しく

最も使われる「直線パス (linear / rectified flow)」では、ノイズ x_0 と正解データ x_1 を直線で結びます。時刻 t (0〜1) における途中点は:

```
u_t = (1 - t) * x_0 + t * x_1        # t=0でノイズ、t=1で正解
```

この直線上を一定速度で進むので、その点で「進むべき速度」は単純に:

```
target_velocity = x_1 - x_0           # 始点から終点へのベクトル (定数)
```

学習は、ネットワーク v_theta に (途中点 u_t, 時刻 t, 条件 c) を入れて速度を予測させ、上の正解速度との二乗誤差を最小化するだけです。

```
速度損失 L = E_{x_0, x_1, t}[ || v_theta(u_t, t, c) - (x_1 - x_0) ||^2 ]
```

ここで重要な理論的事実があります。各サンプルペア (x_0, x_1) ごとの「条件付き速度場」(上の x_1 - x_0) を回帰すると、その期待値として「真の周辺速度場」(分布全体を正しく流す速度場) が学習される、ということが Lipman et al. (2023) で証明されました。直感的には、1つの途中点 u_t には複数のペアが交差して通りうるので、ネットワークはそれらの平均速度を学びます。その平均がちょうど分布全体を運ぶ正しい速度場になるのです。これが Conditional Flow Matching の核心で、「サンプルごとの単純な目標を回帰するだけで、分布全体のフローが学べる」こと、しかもシミュレーション (実際にフローを走らせること) なしで学べる (simulation-free) ことを保証します。これは拡散の連続版 (スコアマッチング) が抱えていた計算の重さを回避します。

学習データの作り方を擬似コードで示します。

```python
B = trajectory_target.shape[0]
x_0 = torch.randn_like(trajectory_target)          # ノイズ x_0 ~ N(0, I)
t   = sample_timesteps(B)                          # 時刻 t を [0,1) からサンプル
u_t = (1.0 - t).unsqueeze(-1) * x_0 + t.unsqueeze(-1) * trajectory_target  # 直線上の途中点
target_velocity = trajectory_target - x_0          # 正解速度 = x_1 - x_0
v_pred = v_theta(u_t, t, c)                         # ネットワークの速度予測
loss = F.mse_loss(v_pred, target_velocity)
```

軌跡を生成対象にする場合、形状はたとえば `[B, num_timesteps * num_signals]`。num_timesteps=64 (将来 64 ステップ)、num_signals=2 (例: 加速度と曲率) なら `[B, 128]` です。

時刻 t のサンプリングには工夫の余地があります。教科書どおりなら一様分布 U(0,1) ですが、ロボット・自動運転の行動生成では「ノイズ寄り (低 t) の速度場が最も学びにくく、そこを重点的にサンプルすると安定・高精度になる」という知見があります。そのため Beta 分布などで t をノイズ寄りに偏らせる実装がしばしば使われます (pi-0 系の流儀)。画像生成の Stable Diffusion 3 では逆に中間の t を厚くする logit-normal 分布が効くと報告されており、対象ドメインで最適な時刻重みが違うことが分かっています。いずれも「学習が難しい領域に学習資源を集中する」ための工夫です。

速度ネットワーク v_theta の典型的な内部構成 (軌跡生成の場合):
1. 途中点 u_t を時系列形 `[B, T, signals]` に整形し、各時刻を1つの「行動トークン」として埋め込み次元 (例 256) に射影 → クエリ `[B, T, 256]`。
2. 環境特徴 (たとえば俯瞰ビュー特徴 `[B, C, H, W]`) を空間方向に平坦化して `[B, H*W, C]` のキー/バリューにし、クロスアテンション (クエリが環境のどこに注目するかを計算する仕組み) で空間情報を取り込む。
3. 時刻 t を sinusoidal embedding (正弦・余弦で数値を多次元化) → MLP に通し、条件と足して AdaLN 変調 (gamma, beta の affine 変換でアテンション出力をスケール・シフトする。画像生成モデル DiT のやり方) を適用。
4. 速度ヘッドで各トークンを速度 `[B, T, signals]` に戻す。

生成 (推論) 時: 新しいノイズを1つサンプルし、`dx/dt = v_theta(x, t, c)` を t=0→1 までオイラー法で積分します。直線パスは経路がまっすぐなので、たとえば 10 ステップ程度の少ない刻みでも十分な品質が出ます。環境特徴と条件は1回だけ計算して各ステップで使い回せば軽量です。拡散モデルの数十〜数百ステップに比べ大幅に速いのが利点です。

数値直感: ノイズ x_0=[3.0, -2.0]、正解 x_1=[1.0, 0.5] なら正解速度は [1.0-3.0, 0.5-(-2.0)] = [-2.0, 2.5] (定数)。t=0.5 の途中点は [(0.5*3+0.5*1), (0.5*-2+0.5*0.5)] = [2.0, -0.75]。ネットワークはこの途中点と t=0.5 と条件を見て [-2.0, 2.5] を当てにいきます。生成時はノイズ x=[3.0,-2.0] から始め、10 ステップなら dt=0.1 で `x ← x + 0.1 * v(x, t)` を 10 回繰り返し、最終的に x_1 付近に到達します。

## 手法の系譜と主要論文

- Flow Matching for Generative Modeling (Lipman, Chen, Ben-Hamu, Nickel, Le, ICLR 2023, arXiv:2210.02747)。提案: 拡散モデルのようにスコアを学ぶ代わりに、確率フローの速度場を直接回帰で学ぶ Conditional Flow Matching。動機: 拡散の学習はノイズスケジュール設計が複雑でサンプリングも遅い。速度場を直接 MSE で学べばシンプルかつ任意のフローを設計できる。新規性: 「条件付き速度場の回帰の期待値が周辺速度場になる」ことを証明し、シミュレーションフリーな学習を可能にした。
- Rectified Flow (Liu, Gong, Liu, 2022/2023, arXiv:2209.03003)。提案: ノイズとデータを直線で結ぶフロー (linear interpolation) を学び、できるだけまっすぐな (rectified) 経路にする。動機: 経路が直線に近いほど少ないステップ (極端には1ステップ) で生成できる。新規性: reflow という反復手順 (学習済みフローで作ったペアで再学習する) で経路をさらに直線化し、1〜数ステップ生成を実現。直線パス `u_t = (1-t)x_0 + t x_1` はこの流儀です。
- Stochastic Interpolants (Albergo & Vanden-Eijnden, ICLR 2023, arXiv:2303.08797)。提案: ノイズとデータをつなぐ補間 (interpolant) を一般化し、拡散・Flow Matching を統一的に扱う枠組み。動機: 様々な生成手法が同じ補間の特殊ケースだと示す。新規性: 決定的 (ODE) と確率的 (SDE) のサンプリングを補間設計から自由に選べる。
- Stable Diffusion 3 / MM-DiT (Esser et al., 2024, arXiv:2403.03206)。提案: rectified flow + Transformer (MM-DiT) を大規模テキスト画像生成に適用し、時刻サンプリングの重みづけ (logit-normal) を体系的に比較。動機: Flow Matching を商用級の高解像度画像生成にスケールさせる。新規性: 時刻重みづけや補間方式の大規模アブレーションを公開し、rectified flow が拡散より高品質・高速だと実証した。
- pi-0 (Black et al., 2024, arXiv:2410.24164)。提案: ロボットの行動生成器を Flow Matching で構成し、視覚言語モデルと組み合わせた汎用操作方策。動機: 多様な操作タスクの多峰な行動を、少ない推論ステップで生成する。新規性: 大規模ロボットデータで Flow Matching 方策をスケールさせ、時刻サンプリングをノイズ寄りに偏らせる実装上の工夫を広めた。自動運転の軌跡生成も同じ枠組みで議論されています。

系譜としては、「拡散のスコアを速度場の回帰に置き換える」(Flow Matching) → 「経路を直線にして高速化」(Rectified Flow) → 「補間設計で統一」(Stochastic Interpolants) → 「大規模画像生成へ」(SD3) / 「ロボット・運転の行動生成へ実用化」(pi-0 系) と進んできました。

## 論文の実験結果(定量データ)

- Flow Matching (Lipman et al. 2023) は画像生成ベンチマーク ImageNet (解像度 64 や 128) で評価され、生成品質の指標 FID (Fréchet Inception Distance、生成画像と本物画像の分布の近さ。小さいほど良い) で、同規模の拡散モデルと同等以上を達成しました。さらに重要なのは、ODE ソルバの関数評価回数 (NFE: Number of Function Evaluations、=ネットワーク呼び出し回数) を抑えても品質が保たれる点で、拡散より少ない NFE で同等 FID に到達したと報告されています。NFE は推論コストの直接の指標なので、これは「同じ品質をより速く出せる」ことを意味します。
- Rectified Flow (Liu et al. 2023) は、reflow を 1〜2 回適用することで、CIFAR-10 などで NFE=1 (1ステップ生成) でも実用的な FID (報告では 1ステップで FID 数前後) を達成できることを示しました。素の Flow Matching が数〜十数ステップを要するのに対し、経路の直線化が推論ステップ削減に直結することの定量的証拠です。
- Stable Diffusion 3 (Esser et al. 2024) は、複数の補間方式 (rectified flow, EDM, cosine 拡散など) と時刻重みづけを大規模に比較し、rectified flow + logit-normal の時刻重みが最も低い検証損失と高い画像品質を与えると報告しました。これは「補間方式と時刻サンプリングが品質を強く左右する」ことの大規模な実証です。
- アブレーションの代表例は「時刻サンプリング分布」と「推論ステップ数」です。一様サンプリングに対し、学習が難しい領域 (ドメインに応じてノイズ寄り、または中間) を厚くサンプルすると、同じ学習量で生成誤差が下がることが報告されています。推論ステップ数については、直線パスでは 4〜10 ステップ程度で品質がほぼ飽和し、それ以上増やしても改善が小さいという結果がよく示されます。これは拡散モデル (ステップを大きく減らすと品質が崩れる) との明確な違いです。

指標の意味: FID は生成データと実データの特徴分布の距離で、小さいほど「本物らしく、かつ多様」を意味します。NFE はネットワークを何回呼ぶかで、推論レイテンシに比例します。Flow Matching の価値は「FID を保ったまま NFE を下げる」点にあり、実時間性が要る制御・運転で特に効きます。

## メリット・トレードオフ・限界

メリット
- 多峰な分布を表現でき、「直進か右折か」を平均でつぶさずに生成できる。
- 拡散モデルより推論が速い (直線パスゆえ少ステップで足りる)。
- 学習目的が単なる速度の MSE で、拡散の複雑なノイズスケジュール設計が不要。シミュレーションフリーで学習効率が良い。
- ODE ベースなので決定的サンプリングが可能 (同じノイズ・条件なら同じ出力)。安全が要る用途で再現性を担保しやすい。
- クロスアテンション + AdaLN により、空間・状況情報を保ったまま条件付き生成できる。

トレードオフ・限界
- 推論はノイズからの積分なので、ノイズが違えば出力も変わる (確率的)。決定的な再現には乱数シード固定が要る。
- ODE の刻み数 (推論ステップ数) と時刻サンプラの設定が品質に効き、調整が要る。
- 学習・推論ともに、単純な1点回帰より計算が重い。
- 直線パスは理論上は最適でない場合があり (真の最適輸送経路は曲がりうる)、reflow なしでは1ステップ生成は難しい。多数のペアが交差する領域では平均速度がぼやけ、reflow で直線化しないと品質が落ちる。

未解決の課題: 最適輸送 (optimal transport) に基づくより良い補間の設計、1〜2ステップ生成の品質をさらに上げる蒸留手法、条件付き生成での「条件をどれだけ強く効かせるか」(classifier-free guidance の Flow Matching 版) の理論などが研究の最前線です。

## 発展トピック・研究の最前線

- 最適輸送 Flow Matching (OT-CFM, Tong et al. 2023): サンプルペアをランダムに結ぶのではなく、ミニバッチ内で最適輸送でペアリングして経路の交差を減らし、より直線化する。学習・推論をさらに効率化する方向です。
- Mean Flow / 一貫性モデルとの融合: 数ステップ生成を保証する一貫性 (consistency) の考え方を Flow Matching に持ち込み、1〜2ステップで高品質を狙う研究 (Consistency FM, MeanFlow など) が進んでいます。
- Vision-Language-Action モデル: 視覚言語モデルの出力を条件に、Flow Matching で連続的な行動・軌跡を生成する大規模方策 (pi-0 系) が、ロボットと自動運転の両分野で主流になりつつあります。生成と意思決定 ([Advantage-Weighted Regression (AWR) ポリシー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0110_advantage-weighted-regression)) を橋渡しする方向としても注目されています。
