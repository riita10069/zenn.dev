# I3D (Inflated 3D ConvNet)

## ひとことで言うと
画像認識でよく使う2D 畳み込み(縦×横を見るフィルタ)を、時間軸を足して3D(縦×横×時間)に「膨らませる(inflate, インフレート)」ことで、動画を直接扱えるようにした手法です。さらに、画像で学習済みのネットワークの重みをそのまま3D に引き伸ばして再利用するという賢い初期化の工夫で、ゼロから学ぶより速く高精度に到達しました。動画認識に「画像の事前学習資産を持ち込む」道を開いた、時空間畳み込みの代表的な研究です。

## 直感的な理解
2D 畳み込みを思い出します。画像認識の CNN は、小さなフィルタ(例えば3×3 の窓)を画像の上で滑らせ、「縦と横の近所の画素パターン」を捉えます。猫の耳の形や車のタイヤの丸みなど、空間的な模様を見るのが得意です。

しかし動画には「時間」という3つ目の軸があります。「手を上げる」と「手を下げる」は、どのフレームも見た目はよく似ていて、違いは時間方向の順序にしかありません。2D 畳み込みは1枚ずつしか見ないので、この時間順序を直接は捉えられません。1枚を見るだけでは、その手が上がる途中なのか下がる途中なのか区別できないのです。

2017年当時、動画認識には主に2つのやり方がありました。ひとつは各フレームを2D CNN で処理して後で平均する方法で、これは時間順序を無視します。もうひとつは Two-Stream(見た目+光学フローの2系統)で、動きを捉えますが光学フロー(画素の移動ベクトル)を事前計算する手間がかかります。

I3D は「いっそ畳み込み自体を時間込みの3D にすればよい」と提案しました。しかし3D CNN には大問題がありました。学習する重み(パラメータ)が一気に増え、動画データだけでゼロから学ぶには当時データが足りなかったのです。画像の世界には ImageNet(数百万枚のラベル付き画像)という巨大な学習資産があるのに、3D ではそれを使えない、というのが壁でした。I3D の核心は、この壁を「膨張」で乗り越えたことにあります。

## 基礎: 前提となる概念
用語を順に噛み砕きます。

畳み込み(convolution)とは、小さなフィルタ(重みの集まり)を入力の上で位置をずらしながら掛け合わせて足す操作です。「近所のパターンに反応する検出器」を画像全体に適用するイメージです。重みは学習で決まります。フィルタを1種類使うと特徴マップが1枚でき、Cout 種類使えば Cout 枚の特徴マップ(チャンネル)が得られます。

転移学習・事前学習(transfer learning / pre-training)とは、大量データで学んだ知識を別タスクに使い回すことです。ImageNet で「物の見え方」を学んだネットワークは、その重みが「エッジ・テクスチャ・物体パーツ」の良い検出器になっているので、それを初期値として別タスクを学ぶと、ゼロから始めるより速く高精度に到達します。深層学習で最も効く実用テクニックのひとつです。

光学フロー(optical flow)とは、隣り合うフレーム間で各画素がどちらにどれだけ動いたかを表すベクトル場で、運動を明示的に表す入力です。RGB が「何が写っているか」を表すのに対し、フローは「何がどう動いたか」だけを抽出した、見た目を捨てた純粋な運動表現です。

クリップ(clip)とは、動画から切り出した短い連続フレーム列(例えば64フレーム)です。長い動画はクリップ単位で処理し、後で結果を集約します。

ブートストラッピング(bootstrapping)とは、「すでにある良い知識から立ち上げる」ことです。ここでは、画像で学んだ2D の重みを3D の初期値として流用することを指します。

## 仕組みを詳しく
膨張(inflation)を具体的に説明します。画像用の優秀なネットワーク(I3D では Inception-v1、別名 GoogLeNet)を土台にします。その中の各2D フィルタは、例えば [出力チャンネル, 入力チャンネル, 3, 3] という形(縦3×横3 の窓)をしています。

これを時間軸を足して [出力チャンネル, 入力チャンネル, 3, 3, 3] という3D フィルタ(時間3×縦3×横3)に作り変えます。形状だけ言えば、2D フィルタを時間方向に N 回コピーして重ね、N で割って平均化します。N=3 ならフィルタを3枚重ねて 1/3 倍します。

なぜコピーして N で割るのか。明確な理由があります。「同じ画像を N フレーム並べた退屈な動画(=止まった映像)」を3D ネットワークに入れたとき、その出力が、元の2D ネットワークに1枚入れたときの出力と一致するようにするためです。割り算で正規化しておくと、止まった動画では時間方向の合計が元の1枚ぶんと等しくなり、ImageNet で学んだ「物の見え方」の知識がそのまま3D ネットワークに引き継がれます。数式で言えば、止まった動画では各時刻の入力が同一なので、時間方向に N 個の同じ重み(各 1/N)を掛けて足すと、合計が元の重み1個ぶんと等しくなる、という仕掛けです。膨張がなければ3D の重みは乱数からのスタートになり、動画データだけでこの良い初期値に到達するのは困難でした。

処理のステップをまとめます。

1. ImageNet で学習済みの2D Inception-v1 を用意する。
2. すべての2D 畳み込みとプーリングを3D に膨張させ、フィルタ重みを上記のやり方でコピー&正規化して初期値にする。
3. この3D ネットワークを Kinetics(大規模動画データセット)で動画として学習させる。入力は例えば64フレーム × 224×224 のクリップで、tensor の形は [バッチ, 3, 64, 224, 224](チャンネル3, 時間64, 縦224, 横224)です。
4. RGB ストリームと光学フローストリームの2本を別々に膨張・学習し、最後に予測を融合する(Two-Stream の良さも取り込む)。

```
2D フィルタ [Cout, Cin, 3, 3]
      │ 時間方向に3回コピーして 1/3 倍(正規化)
      ▼
3D フィルタ [Cout, Cin, 3, 3, 3]
入力 [B, 3, 64, 224, 224] ──3D conv の積層──▶ 時間も空間も徐々に縮む
```

形状の変化を追うと、3D 畳み込みは時間軸も含めて畳み込むので、層を進むにつれて時間方向の長さ(64)も空間と同様に少しずつ縮みながら、各層が「数フレームにまたがる動きパターン」を捉えるようになります。「手が上がっていく」のような時間順序が、ここで初めて畳み込みフィルタに直接学習されます。最終層で時空間を大域プーリングし、分類器で動作ラベルを予測します。I3D では序盤の数層だけ時間方向のプーリングを控えめにし、時間解像度を急に潰さないよう配慮しています。これは「早く時間を潰すと細かい運動が消える」ためです。

この論文のもう一つの大きな貢献が Kinetics データセットです。数十万本の YouTube クリップに動作ラベルを付けた大規模データで、「3D CNN をゼロから学ぶにはデータが足りない」という当時の根本問題を緩和しました。論文タイトルの "Quo Vadis"(ラテン語で「どこへ行くのか」)は、ある映画のワンシーンを例に「次に何が起きるか予測するには動きの理解が要る」という問題提起を込めたものです。1枚の静止画では「次に何が起きるか」は分からず、時間方向の文脈が要る、という動画認識の本質を象徴的に示しています。

## 手法の系譜と主要論文
時空間畳み込みの流れを整理します。

Tran et al., "C3D: Learning Spatiotemporal Features with 3D Convolutional Networks", ICCV 2015。3×3×3 の3D フィルタを最初に広めた研究。表現は豊かですがゼロ学習でデータ不足に苦しみ、Sports-1M で学んでも当時の Two-Stream に精度で及ばない面がありました。C3D の特徴は、汎用的な動画特徴抽出器として後段の単純な分類器(SVM など)に渡せた点で、実用性で広く使われました。

Simonyan & Zisserman, "Two-Stream Convolutional Networks", NeurIPS 2014。RGB と光学フローの2系統。I3D はこの Two-Stream 構成を3D 化して取り込みました。空間ストリームが見た目、時間ストリームがフロー由来の運動を担当する分業の原型です。

Carreira & Zisserman, "Quo Vadis, Action Recognition? (I3D)", CVPR 2017。本トピックの原典。提案は (1) 2D を3D へ膨張させる I3D アーキテクチャ、(2) ImageNet 事前学習重みを3D へ移植する初期化、(3) 大規模動画データセット Kinetics。動機は「3D CNN は表現力が高いが学習データ不足で実力を出せない」問題を、画像の知識転用と大規模データで解くこと。

Tran et al., "A Closer Look at Spatiotemporal Convolutions (R(2+1)D)", CVPR 2018。3D の 3×3×3 畳み込みを、空間2D(1×3×3)と時間1D(3×1×1)の2段に分解する手法。計算を軽くしつつ非線形性を増やす狙いで、同じ計算量でより高精度を達成しました。分解により、空間フィルタと時間フィルタの間に活性化関数(非線形)が1つ増えるため、表現力が上がるというのが要点です。I3D の「重い3D」を分解で効率化する流れで、後の [Video Swin Transformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0510_video-swin-transformer) の時間軸分離にもつながります。

Xie et al., "Rethinking Spatiotemporal Feature Learning (S3D)", ECCV 2018。I3D の3D を部分的に分離・間引いて軽量化した後継。下層は2D、上層だけ3D にすると速くて精度も保てる(top-heavy 設計)、という知見を示しました。これは「低レベルの特徴は時間情報が少なくても抽出でき、時間統合は高レベルで行うのが効率的」という洞察です。

Feichtenhofer et al., "SlowFast Networks", ICCV 2019(詳細は [SlowFast Networks](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0498_slowfast-networks))。I3D 系の3D CNN を二経路に発展させ、フレームレートの異なる経路で意味と運動を分担しました。

## 論文の実験結果(定量データ)
指標を説明します。UCF-101 と HMDB-51 は、それぞれ101・51カテゴリの比較的小規模な動作認識データセットで、指標は分類精度(正解率)です。小規模ゆえ、いかに良い事前学習を持ち込めるかが性能を分けます。Kinetics-400 は約24万本・400カテゴリの大規模データで、事前学習の供給源かつ評価ベンチマークの両方の役割を果たします。

I3D の主要な発見は「Kinetics で事前学習してから小規模データに転移すると大きく伸びる」ことでした。Kinetics 事前学習を経た I3D(RGB + フローの2ストリーム融合)は、UCF-101 で約 98.0%、HMDB-51 で約 80.7% を報告し、当時の最高精度を大きく更新しました。事前学習なし(ImageNet 膨張のみ、Kinetics 学習なし)に比べ、Kinetics 事前学習はこれらの精度を十数ポイント押し上げました。具体的には、HMDB-51 で ImageNet 膨張のみのモデルが 50% 台だったところ、Kinetics 事前学習で 80% 台に跳ね上がる、という劇的な差が報告されています。これは「動画でも大規模事前学習が画像と同じく決定的に効く」ことを示した重要な結果で、その後の動画認識研究の方向を決定づけました。

膨張(inflation)初期化のアブレーションも示されました。ImageNet 重みから膨張で初期化した I3D は、3D を乱数から学んだ場合より一貫して高精度かつ収束が速く、Kinetics 上の精度で明確な差が出ました。膨張は単なる便利技ではなく、3D CNN を現実的に学習させる鍵だったわけです。乱数初期化では同じデータ量で I3D の性能に届かず、画像で蓄えた「物の見え方」の事前知識が3D の学習を強力に支えていることが裏付けられました。

ストリームの寄与もアブレーションで分解されました。RGB 単独より、RGB と光学フローの2ストリームを融合する方が高精度です。Kinetics 学習後でもフローの追加は数ポイントの上積みをもたらし、生の RGB だけでは運動の手がかりが完全には学びきれていないことが示唆されました(後の SlowFast はこのフロー依存を Fast 経路で置き換えます)。Kinetics-400 そのものでの I3D の top-1 はおよそ 71-72% 程度(RGB+フロー融合)と報告され、これが以後の動画モデルの比較基準になりました。RGB 単独だと約 68% 程度で、フロー追加が数ポイント効いた、という分解です。

## メリット・トレードオフ・限界
メリット。時間順序を畳み込みで直接学習でき、見た目が似た動作(手を上げる/下げる)を区別できます。膨張により ImageNet の画像知識をそのまま3D へ転用でき、学習が速く高精度になります。Kinetics と組み合わせて、動画認識の転移学習の標準的な土台になりました。構造が素直(2D CNN をほぼそのまま3D 化)で理解しやすい点も普及を助けました。

トレードオフと限界。3D 畳み込みは計算量・メモリが2D より大幅に重く、64フレームのクリップを処理するだけでも GPU メモリを圧迫します。3D 畳み込みのパラメータ数と計算量は、フィルタの時間幅 N 倍になるため、2D の素朴な拡張ではコストが跳ね上がります。1クリップ単位の処理なので、動画全体の長い時間文脈(数十秒の物語的な流れ)は別途集約が必要です。光学フローを併用する設定ではフローの事前計算コストが残り、リアルタイム用途では足かせになります。後続の分離型(R(2+1)D, S3D)に比べると同じ計算量での効率で見劣りします。研究上の課題として、3D 畳み込みは受容野が局所的で、長距離の時空間関係を捉えるには層を深くする必要があり、ここから注意機構ベースの動画モデル([TimeSformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0507_timesformer)、[Video Swin Transformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0510_video-swin-transformer))への移行が起きました。

## 発展トピック・研究の最前線
I3D 以後、効率化は「3D の分解」と「注意への置換」の2方向に進みました。分解では R(2+1)D・S3D が空間と時間を分けて畳み込み、計算を抑えつつ非線形性を増やしました。注意への置換では、畳み込みを使わず Transformer の注意で時空間を結ぶ TimeSformer や ViViT、局所窓注意の Video Swin が登場し、長距離関係と大規模事前学習の相性の良さで3D CNN を上回っていきます。一方で I3D が確立した教訓「画像の大規模事前学習を動画へ転用する」「膨張で初期化する」は今も生きており、画像 ViT 重みを動画モデルへ膨張・転用する手法(画像 Swin → Video Swin、画像 ViT → ViViT/TimeSformer など)に直接受け継がれています。

自動運転など、各フレームを画像バックボーンで処理し時間を後段で集約する「2D 空間 + 1D 時間」の分離設計も、I3D 系の効率化(R(2+1)D の空間・時間分離)と同じ思想の延長にあります。膨張の発想は、画像で大量に蓄えた事前知識を捨てずに時系列へ持ち込む一般的な道具であり、動画に限らず「静的なモデルを動的なタスクへ拡張する」場面で繰り返し再発見されています。
