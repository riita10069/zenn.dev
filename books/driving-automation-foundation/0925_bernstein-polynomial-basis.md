# Bernstein多項式基底

## ひとことで言うと
ベジェ曲線という滑らかな曲線を作るための「数学のレシピ」です。少数の制御点を、各時刻でどんな重みで混ぜれば滑らかな曲線になるかを決める係数の表(行列)で、これさえ用意しておけば制御点から全時刻の曲線を行列1回で計算できます。係数は二項係数とべき乗だけで書けるので、特別な科学計算ライブラリに頼らず標準ライブラリだけで実装できます。

## 直感的な理解
5人で1つの音(和音)を出す合唱を想像してください。各時刻で「誰がどれだけ大きい声を出すか」のバランスが決まっていて、曲の始めは1人目だけ、終わりは5人目だけ、途中はみんなが滑らかに混ざります。このバランス表が Bernstein 基底です。5人が「制御点」、各時刻の声量配分が「重み」で、重みは時刻に対してなめらかに移り変わるので、出てくる和音(=軌道)もなめらかになります。

なぜ表(行列)にして持っておくのか。合唱の声量配分は曲の間ずっと同じ譜面で決まっているので、毎回その場で計算し直す必要はありません。最初に一度だけ譜面(基底行列)を書いておき、毎時刻はそれを見るだけ。計算量も無駄がなく、譜面自体は学習で変わる対象ではないので勾配も要りません。

## 基礎: 前提となる概念
言葉を噛み砕きます。

- 基底 (basis): 「これらを混ぜれば目的のものが作れる」という部品セット。ここでは「各制御点が、各時刻にどれだけ寄与するか」を表す重みの集まりです。
- 多項式 (polynomial): t の何乗かを足し合わせた式。例: 1 + 2t + 3t^2。次数 (degree) はいちばん高いべきの数。
- Bernstein 多項式 (Bernstein polynomial): ベジェ曲線を定義する、決まった形の多項式たち。パラメータ t を 0 から 1 まで動かしたとき、各制御点の影響度を表します。
- 二項係数 C(n, k) (binomial coefficient): 「n 個から k 個を選ぶ組み合わせの数」。パスカルの三角形に現れる数で、Python なら標準ライブラリ math.comb(n, k) で計算できます。
- パラメータ t: 軌道に沿った進み具合を 0(始点)から 1(終点)で表す数。実時刻 0〜T 秒を t = 0〜1 に正規化して使います。

なぜ重い外部ライブラリ(scipy など)を避けて標準ライブラリで書けるのか。Bernstein 基底や二項係数は科学計算ライブラリに関数として用意されていますが、この計算自体は二項係数とべき乗の掛け算だけで完結します。たった数行のためにライブラリ依存を増やすと、インストールが重くなり、バージョン衝突のリスクも生まれ、軽量な実行環境(エッジデバイスなど)に載せにくくなる。だから「math.comb とべき乗で十分」という判断が合理的です。これは数値計算の実装で広く見られる「依存最小化」の判断です。

## 仕組みを詳しく
時刻数 P = 64、制御点数 C = 5 で説明します。次数は n = C − 1 = 4 です。i 番目の制御点 (i = 0, 1, ..., n) の、パラメータ t における重みを表す Bernstein 基底の定義式は次です。

```
B_{i,n}(t) = C(n, i) * t^i * (1 - t)^(n - i)
```

各記号の意味を言葉で。C(n, i) はべき乗の組み合わせ係数(山の高さを調整する定数)。t^i は「終点に近いほど効く成分」(i が大きい制御点ほど t が大きいとき効く)。(1−t)^(n−i) は「始点に近いほど効く成分」。これらの積で、i 番目の制御点が時刻 t でどれだけ効くかが決まります。

行列として一気に作る手順(ブロードキャストで [64, 5] を生成):

```
n    = num_controls - 1                                 # = 4
t    = linspace(0.0, 1.0, num_points).unsqueeze(1)      # [P=64, 1]  時刻を 0..1 で64等分
i    = arange(num_controls).unsqueeze(0)                # [1, C=5]   制御点番号 0..4
comb = tensor([comb(n, k) for k in range(num_controls)]).unsqueeze(0)  # [1, 5]
basis = comb * (t ** i) * ((1.0 - t) ** (n - i))        # [64, 5]
```

ステップで噛み砕きます。

1. t = linspace(0, 1, 64): 時刻を 0 から 1 まで等間隔に 64 個並べる。0 が始点、1 が終点。形は [64, 1]。
2. i = arange(5) = [0, 1, 2, 3, 4]: 制御点の番号。形は [1, 5]。
3. comb = [C(4,0), C(4,1), C(4,2), C(4,3), C(4,4)] = [1, 4, 6, 4, 1]: n=4 の二項係数(パスカルの三角形の5行目)。形は [1, 5]。
4. ブロードキャスト(形の小さい方を自動で引き伸ばして要素ごとに演算する仕組み)で [64, 5] を一気に作る。t^i は [64,1] と [1,5] が掛かって [64,5] に、(1−t)^(n−i) も同様。最後に comb を掛けて完成。

できた行列の各行は「ある時刻における5個の制御点の重み」です。Bernstein 基底には2つの重要な性質があります。

- 1の分割 (partition of unity): 各行(各時刻)の5個の重みを足すと必ず 1 になる。たとえば t=0 の行は [1, 0, 0, 0, 0](始点は最初の制御点に完全一致)、t=1 の行は [0, 0, 0, 0, 1](終点は最後の制御点に一致)、真ん中あたりは5点が滑らかに混ざる。
- 凸包性 (convex hull property): 重みがすべて非負で合計1なので、曲線は必ず制御点の凸包(制御点を囲む最小の凸領域)の中に収まる。つまり曲線が制御点の範囲外に暴れて飛び出すことがない。これは安全領域内に軌道を閉じ込めたいロボティクスで非常に有用な性質です。

数値例。制御点を加速度成分だけ見て [0, 1, 2, 1, 0] と置くと、t=0 で 0、t=1 で 0、途中で滑らかに 2 付近まで上がって戻る、山なりのなめらかなプロファイルが得られます。各時刻の値は「5個の制御点をその行の重みで加重平均したもの」なので、隣の時刻と急にジャンプできず、必ずなめらかにつながります。具体的に t=0.5 の行を計算すると、n=4 では重みは [C(4,0)·0.5^0·0.5^4, C(4,1)·0.5^1·0.5^3, C(4,2)·0.5^2·0.5^2, C(4,3)·0.5^3·0.5^1, C(4,4)·0.5^4·0.5^0] = [1·0.0625, 4·0.0625, 6·0.0625, 4·0.0625, 1·0.0625] = [0.0625, 0.25, 0.375, 0.25, 0.0625] となり、足すと 1(1の分割の確認)、中央の3点目が最大の重み 0.375 を持ちます。この行を制御点 [0,1,2,1,0] に内積すると 0·0.0625 + 1·0.25 + 2·0.375 + 1·0.25 + 0·0.0625 = 1.25。t を 0 から 1 まで動かしながら同じ内積を繰り返せば、山なりのプロファイルが描けます。

微分の構造も知っておくと役立ちます。次数 n のベジェ曲線を t で微分すると、次数 n−1 のベジェ曲線になり、その制御点は隣り合う元の制御点の差分 n·(P_{i+1} − P_i) で与えられます。つまり「ベジェの導関数もまたベジェ」で、速度・加速度・ジャークといった高階の時間微分が同じ基底の枠内で閉じて計算できます。これが「ジャークを直接ペナルティに入れたい」「終端速度を制約したい」といった要求を、制御点の線形条件として綺麗に書ける理由です。出力が加速度・曲率のプロファイルなら、その時間微分(=ジャーク)は元の制御点の一次差分に基底を掛けたものなので、ジャーク最小化が制御点に関する二次形式の最小化になり、解析的・微分可能に扱えます。

実装上、この行列は一度計算したら学習中ずっと変わらないので、固定テンソルとしてモデルに保持します(register_buffer のように「学習しないがモデルと一緒に GPU へ移動し、勾配は持たない」テンソルとして登録するのが定石。詳細は [register_buffer(PyTorch)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1009_register-buffer))。保存ファイルに含めない設定にすれば、どうせ再計算できる固定値でチェックポイントを太らせずに済みます。使うときは [ベジェ曲線トラジェクトリプランナー](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0116_bezier-trajectory-planner) の通り einsum で制御点 [B, 5, 2] に掛け、軌道 [B, 64, 2] を得ます。

## 手法の系譜と主要論文
Bernstein 多項式は応用数学・計算幾何の古典です。

- Bernstein (1912): 連続関数を多項式で一様に近似できるという Weierstrass の近似定理に、構成的(具体的に多項式を組み立てる)証明を与える文脈で導入されました。主張は「この形の多項式の重ね合わせで任意の連続関数をいくらでも近づけられる」。これが Bernstein 基底の出発点で、近似理論の基礎です。
- Bezier と de Casteljau (1960年代): フランスの自動車メーカーで、Pierre Bezier と Paul de Casteljau がそれぞれ独立に、車体曲面設計のためにこの基底を実用化しました(いわゆるベジェ曲線)。動機は、デザイナーが制御点をドラッグするだけで直感的に滑らかな曲線を作れること。効果は CAD・フォント(TrueType/PostScript)・グラフィックスでの爆発的普及。トレードオフは、制御点を増やすと次数が上がり、計算が重く数値的に不安定になりうること。実務では低次のベジェを区分的につなぐ B-spline が好まれます。
- de Casteljau のアルゴリズム: 基底を陽に計算せず、制御点間を再帰的に線形補間してベジェ点を評価する数値安定な方法。一方、時刻が固定で「全時刻ぶんを一括で行列計算したい」場合は、基底行列を直接作る方式のほうが GPU の行列積(einsum)と相性が良い。用途で選び分けます。
- ロボティクス・運転への応用: Werling et al. (2010, ICRA) の minimum-jerk 多項式軌道のように、滑らかさ(低ジャーク)を保証する軌道生成に多項式/ベジェ基底が使われます。さらに Bernstein 基底の凸包性を利用して衝突回避制約を凸制約として扱う研究(quadrotor やマニピュレータの軌道計画)があり、「軌道を安全領域内に数学的に閉じ込める」のに活用されています。

## 論文の実験結果(定量データ)
Bernstein 基底そのものは「定理」であって学習手法ではないため、ベンチマーク数値ではなく数学的性質と、それを使う系の効果で評価されます。

- 近似能力: Weierstrass の近似定理の Bernstein 構成では、次数 n を上げるほど近似誤差が 0 に収束します。ただし収束は遅く(誤差は概ね 1/√n のオーダー)、実用では「滑らかさ保証」が主目的で、近似精度を稼ぐためにむやみに次数を上げることは少ない。これが「制御点を5個程度に抑える」実務判断の数学的背景です。
- 数値安定性: 高次ベジェ(制御点を多くする)では、両端付近で t^i や (1−t)^(n−i) が極端に小さい/大きい値になり、浮動小数点の桁落ちで不安定化することが知られます。制御点 5〜7 程度では実用上問題になりにくいが、十数個を超えると顕著になる、というのが経験則です。区分 B-spline はこの問題を低次の連結で回避します。
- 軌道生成での効果: minimum-jerk 軌道(Werling et al., 2010)系の評価では、多項式/ベジェ表現を使うことでジャーク二乗積分(乗り心地の悪さの指標、小さいほど良い)が、区分線形や素朴な点列に比べ系統的に小さくなることが示されます。指標「ジャーク二乗積分」は加速度の変化の激しさを累積したもので、小さいほど乗員が感じるカックンが少ない。

## メリット・トレードオフ・限界
メリット。

- 標準ライブラリの二項係数計算だけで実装でき、重い外部依存が要らない。インストールが軽く、バージョン衝突がなく、軽量な実行環境にも載せやすい。
- 固定値なので一度計算してテンソルに持つだけ。毎ステップの計算コストが実質ゼロ、勾配も不要でメモリも節約できる。
- 1の分割と凸包性という数学的性質により、曲線が暴れず必ず滑らかで、制御点の凸包内に収まる。安全領域の保証に直結する。
- 全時刻を行列1回で展開でき、GPU の einsum と相性が良く高速。

トレードオフ・限界。

- 制御点数を増やすと多項式の次数が上がり、両端付近で数値的に不安定になりうる(高次ベジェの一般的弱点)。実用上は制御点を少なく保つ。
- 単一の高次ベジェは局所制御が苦手。1つの制御点を動かすと曲線全体が影響を受けるため、局所的に形を直したい用途には区分 B-spline のほうが向く。
- 時刻数・制御点数を固定前提で行列を作るため、可変長の軌道を扱うには基底を作り直す必要がある。
- 近似精度の収束が遅い。形の表現力を稼ぐ目的には不向きで、あくまで滑らかさ保証と低次元化が主たる利点。

## 発展トピック・研究の最前線
- B-spline と NURBS: 低次ベジェを区分的につなぎ、連結点での連続性(C^1, C^2 など、何階微分まで滑らかかを表す)を制御する一般化。局所制御に強く、CAD やロボティクスの軌道計画で主流。NURBS は重み付きで円弧なども厳密に表せる拡張。
- 凸最適化との結合: 凸包性を使い、制御点に線形/凸制約を課して「軌道を安全回廊内に閉じ込める」定式化。学習で制御点の初期推定を出し、凸最適化で安全制約を満たすよう微修正するハイブリッドが研究されています。
- 微分可能性の活用: 基底行列が固定の線形写像なので、制御点に関して軌道は完全に微分可能。end-to-end 学習で「軌道全体の損失」を制御点に逆伝播でき、生成モデルや模倣学習にそのまま組み込める。
- 他基底との比較: フーリエ基底(周期運動向き)、多項式基底(数値不安定)、B-spline 基底(局所制御)など、用途に応じた基底選択は依然として設計上の論点。Bernstein 基底は「滑らかさ保証 + 凸包性 + 実装の軽さ」のバランスが良い選択肢として位置づけられます。
- 凸包性を使った安全保証つき計画: quadrotor やマニピュレータの軌道生成では、Bernstein 基底の凸包性を活用し「軌道が必ず安全回廊(自由空間)の中に収まる」ことを、制御点が安全領域の凸多面体に含まれるという線形制約として課す研究があります(Mellinger & Kumar (2011) の minimum-snap 軌道や、その後の MIQP/凸最適化ベースの計画系)。連続な軌道全体の安全性を、有限個の制御点に対する有限個の制約に落とせるのが本質的な利点で、無限次元の関数空間での制約充足を有限次元の凸最適化に縮約できます。学習でラフな制御点を出し、凸最適化で安全制約を満たすよう微調整するハイブリッドは、学習の柔軟さと最適化の保証を両立する現実的な設計です。
- ラショナル/有理ベジェと NURBS: 各制御点に重みを与えた有理ベジェは、円弧や楕円といった円錐曲線を厳密に表現でき、通常の多項式ベジェでは近似しかできない曲線を正確に扱えます。曲率一定のカーブ(定常円旋回)を厳密に出したい運転計画では有理表現が有利な場面があります。
