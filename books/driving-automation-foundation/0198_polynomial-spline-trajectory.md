# 多項式/スプライン軌道パラメータ化

## ひとことで言うと
軌道を「点をたくさん並べたもの」ではなく、「少数の係数で表せるなめらかな曲線の式」として表現する古典的なやり方です。3〜5次の多項式や、スプライン(区間ごとの多項式をつないだ曲線)、ベジェ曲線(制御点で形を決める曲線)の係数を数個決めるだけで、連続的でなめらかな軌道全体が一意に決まります。点を直接出すより出力次元が小さく、滑らかさが式の形そのものから保証されるのが利点です。

## 直感的な理解
誰かに円を描いてもらうとき、「円周上の点を100個プロットして」と頼むと、点はガタガタになりがちです。代わりに「中心と半径を教えて」と頼めば、たった2つの情報で完璧に滑らかな円が一意に決まります。少数のパラメータで形を表す方が、安定して滑らかなのです。

軌道計画も同じです。軌道を0.1秒刻みの (x, y) 点を60個で表すと、120個の数値(自由度)があります。自由度が多いと表現力は高いものの、困ったことが起きます。

- 隣り合う点の間でカクカクする。実車は急加速・急ハンドルできないので不自然。
- 滑らかさを保証するには、別途「隣の点との差を小さくする」後処理やペナルティが要る。
- 最適化や回帰で120個全部を独立に動かすと計算が重く、解が暴れやすい。

ここで「実車の軌道は数秒程度の短い区間では、せいぜい数回曲がるくらいの単純な形しかしていない」という事実に注目します。だったら120個の数値は過剰で、もっと少ないパラメータで十分です。5次多項式なら係数6個で、その区間の位置・速度・加速度・躍度(ジャーク)まで含めて連続でなめらかな曲線が一意に決まります。

多項式・スプライン表現の狙いは2つです。(1) 少数パラメータで軌道を表して計算と回帰を軽くすること。(2) 多項式は何度でも微分できるので速度・加速度・ジャークが自動的に連続になり、滑らかさが式の形そのものから保証されること。

## 基礎: 前提となる概念

多項式(polynomial)とは、`a0 + a1*t + a2*t^2 + ...` のように、変数 t の累乗に係数を掛けて足した式です。最も高い累乗を次数(degree)と呼び、5次なら t^5 まで含みます。係数の個数が自由度で、5次多項式は係数6個です。

微分(derivative)とは「変化の速さ」を求める操作です。位置を時間で微分すると速度、速度を微分すると加速度、加速度を微分するとジャーク(jerk、躍度=加速度の変化率)になります。ジャークは乗り心地と安全に直結する量で、ジャークが大きいと「ガクッ」とした不快で危険な動きになります。多項式は微分しても多項式のままなので、これらが全て式で連続に得られます。

スプライン(spline)とは、長い曲線を複数の区間に分け、各区間を低次の多項式で表し、区間の境目で値や微分がなめらかにつながるようにしたものです。製図でしなやかな板(雲形定規・自在定規)を使って曲線を描いた道具が語源です。境目で何階微分まで一致させるかを連続性のクラス(C0=値だけ、C1=1階まで、C2=2階まで)と呼びます。

ベジェ曲線(Bezier curve)は、制御点(control point)という少数の点の重み付き和で表す曲線です。元は自動車の車体設計(ルノーの de Casteljau、シトロエンの Bézier)で生まれました。制御点を動かすと曲線全体が滑らかに変形します。

Frenet 座標系(フレネ座標)は、軌道を「世界座標の (x, y)」ではなく「道路の中心線に沿った距離 s」と「中心線からの横ずれ d」の2軸で表す座標系です。道路に沿った運動を素直に記述できます。直線・カーブを問わず「どれだけ進んだか」と「中心からどれだけずれたか」で表せるので、車線維持や追い越しを自然に書けます。

二次計画(QP, Quadratic Programming)とは、目的関数が二次式・制約が線形不等式の最適化問題で、凸(谷が1つ)なので大域最適解が高速に求まります。軌道の滑らかさ(ジャークの二乗)を二次の目的にし、障害物回避や速度上限を線形制約にすると QP で解けます。

## 仕組みを詳しく

最も基本的な多項式軌道から説明します。時刻 t における位置 x(t) を5次多項式で表します。

```
x(t) = a0 + a1*t + a2*t^2 + a3*t^3 + a4*t^4 + a5*t^5
```

微分すると速度・加速度・ジャークが全て式で出ます。

```
速度     x'(t)   = a1 + 2 a2 t + 3 a3 t^2 + 4 a4 t^3 + 5 a5 t^4
加速度   x''(t)  = 2 a2 + 6 a3 t + 12 a4 t^2 + 20 a5 t^3
ジャーク x'''(t) = 6 a3 + 24 a4 t + 60 a5 t^2
```

係数 a0〜a5 は6個あるので、始点と終点で6個の条件(始点の位置・速度・加速度、終点の位置・速度・加速度)を指定すると、連立方程式を解いて係数が一意に決まります。これで「始点と終点をなめらかにつなぐ唯一の5次曲線」が手に入ります。なぜ5次かというと、両端で位置・速度・加速度を指定すると6条件になり、ちょうど6係数の5次多項式に対応するからです。ジャーク最小という観点でも5次が自然に出てきます(後述)。

スプラインは、1本の高次多項式だと係数を大きくすると曲線が端で激しく振動する(Runge 現象=高次多項式が端点付近で大きく波打つ現象)のを避けるため、各区間を低次(例 3次)に保ったまま全体をなめらかにつなぎます。境目で位置・1階微分・2階微分を一致させると、C2 連続の三次スプラインになります。

ベジェ曲線は、n+1 個の制御点に対し、ベルンシュタイン多項式(Bernstein polynomial)という基底関数を重みにして表します。次数 n のベルンシュタイン基底は `B_i(u) = C(n,i) * u^i * (1-u)^(n-i)`(C は二項係数、u は0〜1の正規化時刻)です。制御点が5個なら、軌道上の各時刻の値は「5個の制御点を、その時刻でのベルンシュタイン基底値で混ぜたもの」になります。

テンソルで表すと、基底行列が `[P, C]`(P=出力点数, C=制御点数)、制御点が `[B, C, S]`(B=バッチ, S=信号チャネル数)のとき、`einsum("pc,bcs->bps")` を計算して `[B, P, S]` の軌道が出ます。ベジェの利点は、(1) 制御点を動かすだけで曲線全体が動く、(2) 曲線が必ず制御点の凸包(制御点を囲む最小の領域)の中に収まるので暴れない、(3) 低次の混ぜ合わせなので滑らかさが構造的に保証される、ことです。

数値例で凸包性を実感します。制御点を縦に並べた直線的な配置にすれば、ベジェ曲線は決してその外側に飛び出しません。これは「ネットが多少おかしな制御点を出しても、軌道が暴走しない」という安全余裕を構造的に与えます。物体検出の bounding box のように、軌道の存在範囲が制御点で囲めるのです。

Frenet 座標と組み合わせる古典手法も重要です。s 方向(縦、進む距離)と d 方向(横、ずれ)それぞれを時間の多項式で表し、終端条件を少しずつ変えた候補軌道を多数生成し、コスト(滑らかさ・速度逸脱・衝突など)で最良を選びます。例えば「3秒後に横ずれ d=0(中央)」「d=−0.5(やや右)」など複数の終端 d を試し、それぞれに5次多項式を当て、コスト最小を採用します。

なぜ「ジャーク最小なら多項式になる」のか。最適制御の理論で、ジャークの2乗の積分 ∫(x''')^2 dt を最小化する軌道を変分法(オイラー・ラグランジュ方程式)で解くと、その解が多項式になることが数学的に示せます。具体的には、位置を指定する縦方向の問題で躍度最小なら5次、速度を指定する問題なら4次の多項式が解になります。つまり多項式は「滑らかさを最適化した結果として自然に出てくる形」であり、恣意的な選択ではありません。これが Werling らの手法の理論的根拠です。

## 手法の系譜と主要論文

Werling, Ziegler, Kammel, Thrun の "Optimal Trajectory Generation for Dynamic Street Scenarios in a Frenet Frame"(ICRA 2010)は、この分野の古典です。提案は、Frenet 座標で横方向 d(t) と縦方向 s(t) をそれぞれ5次(quintic、位置指定)と4次(quartic、速度指定)の多項式で表し、終端条件を変えた候補軌道を多数生成し、ジャーク最小化を含むコスト関数で最良を選ぶことです。なぜ多項式かというと、ジャーク最小軌道が数学的に多項式になることが示せるからです。効果として、なめらかで快適、かつ衝突回避できる軌道がリアルタイム生成できることを示しました。この「Frenet サンプリングプランナ」は今も多くの研究・量産・オープンソース実装の基礎になっています。

McNaughton et al.("Motion Planning for Autonomous Driving with a Conformal Spatiotemporal Lattice", ICRA 2011)は、多項式スパイラル(曲率を多項式で表す)で道路に沿った格子(lattice)状の候補を生成し、時空間で探索する手法を示しました。多項式パラメータ化を探索ベースの計画と組み合わせた代表例です。

Baidu Apollo の EM Planner(Fan et al., "Baidu Apollo EM Motion Planner", 2018, arXiv:1807.08048)は、量産自動運転スタックで多項式・スプライン表現を使った代表例です。経路(path)と速度(speed)を交互(EM=Expectation-Maximization 的)に最適化し、各段階で動的計画法(DP)で粗い解を作り、その周辺を区分多項式(piecewise polynomial spline)で表して二次計画(QP)で滑らかに仕上げます。提案理由は、滑らかさ制約と障害物制約を扱いやすい凸最適化として定式化するためです。効果は量産レベルのリアルタイム性と滑らかさの両立です。

近年の学習ベースプランナでも、ネットに直接60個の点を出させる代わりに、少数の多項式・ベジェ係数や制御点を回帰させる設計が広く使われます。理由は、出力次元が小さくなって学習が安定し、滑らかさがタダで付いてくるからです。重要な応用として、ベジェ平滑化を (x, y) 座標ではなく制御信号(加速度・曲率)のプロファイルに掛ける設計があります(→ [運動学自転車モデル拘束デコード](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0186_kinematic-bicycle-decoding))。「制御の出し方を少数の制御点で滑らかにする → 加速度・曲率がカクつかない → ジャークが小さく実車に優しい指令になる」という二段構えで、後処理フィルタも学習可能な滑らかさペナルティも不要にできます。

## 論文の実験結果(定量データ)

評価には、軌道の滑らかさ指標(ジャークの大きさ、加速度の変化)、計画誤差(ADE/FDE、→ [軌道評価指標 ADE/FDE/collision](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0793_trajectory-eval-metrics-ade-fde))、計算時間(レイテンシ)、そして衝突率や快適性が使われます。

Werling らの手法は、ジャーク最小という最適性により、サンプリングした候補の中から「乗り心地が良く安全な」軌道を選べることを示しました。重要なのは、多項式パラメータ化のおかげで、各候補軌道がそもそも滑らかさを保証された状態で生成される点です。これにより、後処理で平滑化する手間なしに、リアルタイム(当時のハードで数十ミリ秒オーダー)で数百本規模の候補生成・評価ができました。

Apollo EM Planner は、実車量産スタックで動作するレベルのリアルタイム性(計画周期、概ね100ミリ秒オーダー内に収まる計算時間)を達成し、多項式 QP による滑らかな軌道が高速道路・市街地で実用に耐えることを示しました。DP で大域的に粗探索し、QP で局所的に滑らかに仕上げる二段構成が、解の質と速度を両立させた鍵です。

学習ベース手法での一般的なアブレーション知見。
- 出力を「生の60点回帰」から「少数のベジェ制御点・多項式係数の回帰」に変えると、出力次元が大幅に減り(例 120次元 → 制御点5個×2次元=10次元)、学習が安定し、生成軌道のジャークが構造的に小さくなります。滑らかさペナルティ項を別途学習する必要が消えます。
- 制御点数(またはベジェ次数)を増やすと表現力が上がりますが、ある点を超えると改善が頭打ちになり、低次のままで都市運転の軌道はほぼ表現できると報告されます。一方で次数を上げすぎると Runge 現象(端での振動)のリスクが増します。
- ベジェ平滑化を座標でなく制御信号(加速度・曲率)に掛ける設計では、最終軌道のジャークがさらに小さくなり、下流コントローラへの指令が滑らかになります。

これらの指標の意味。ジャークが小さいほど乗り心地が良く、急な動作による制御の追従誤差や安全リスクが減ります(快適性は NAVSIM の PDM Score でも明示的なサブ指標、→ [Hydra-MDP 多教師蒸留計画](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0182_hydra-mdp-multitarget))。出力次元が小さいほど、係数空間のわずかな誤差が軌道に与える影響を見通しやすく、学習が安定します。

## メリット・トレードオフ・限界

メリット。
- 少数の係数・制御点で軌道全体を表せるので、出力次元が小さく学習・最適化が軽い。
- 微分が式で得られ、速度・加速度・ジャークが連続。滑らかさが式の形から保証される。
- ベジェは制御点の凸包内に収まるので暴れにくく、後処理フィルタが不要。
- 古典最適化(QP)と相性がよく、量産実績がある。制御信号に掛ければ低ジャーク指令が構造的に得られる。

トレードオフと限界。
- 表現できる軌道の形に上限があり、複雑な回避を低次係数で近似すると荒くなる。
- 高次の単一多項式は端で振動(Runge 現象)しやすい。スプラインで緩和するが区間設計が要る。
- Frenet ベースは道路中心線が前提で、中心線のない場面(交差点内・駐車場・自由空間)に弱い。中心線が急カーブだと Frenet→世界座標の変換がゆがみ、横ずれ d の解釈が崩れることもある。
- 学習で係数を直接回帰すると、係数空間でのわずかな誤差が軌道の大きなズレに化けることがある(係数の感度問題)。特に高次係数は軌道全体に効くため、誤差が増幅されやすい。制御点回帰(ベジェ)は各点が局所的に効くので感度がやや穏やか。

## 発展トピック・研究の最前線

研究の方向として、(1) 多項式・スプラインのパラメータ化を運動学拘束(→ [運動学自転車モデル拘束デコード](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0186_kinematic-bicycle-decoding))や語彙ベース選択(→ [軌道語彙(クラスタアンカー)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0205_trajectory-vocabulary-anchors))と組み合わせる流れ、(2) B-スプラインや NURBS など、より制御しやすい基底の利用、(3) 微分可能な最適化層(differentiable QP、例えば OptNet 系)をネットに組み込み、Apollo 流の最適化を学習と end-to-end でつなぐ流れ、(4) 自由空間(中心線がない場面)に強い、グラフベースや拡散ベースの生成と多項式平滑化のハイブリッド、があります。少数パラメータで滑らかさを構造的に保証するという思想は、学習と古典最適化を橋渡しする中心的な道具として今も価値を持ち続けています。生成型(拡散・自由回帰)が表現力で勝り、パラメータ型(多項式・スプライン)が滑らかさと解釈性・計算効率で勝る、という住み分けの中で、両者を組み合わせる設計が現在の主流になりつつあります。
