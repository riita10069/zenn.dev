# LSTM (Long Short-Term Memory)

## ひとことで言うと
LSTM は、素朴な再帰ネットワーク（系列を順に処理するニューラルネット）が「長い系列で昔の情報を忘れてしまう」という致命的な弱点を、長期記憶の専用レーンである「セル状態」と、情報の出し入れを学習で制御する複数の「ゲート」によって解決した手法です。長期記憶（Long-term memory）と短期記憶（Short-term memory）の両方をうまく扱えることから、Long Short-Term Memory（長い・短期記憶）と名付けられました。1997年に提案されて以来、Transformer が登場するまでの約20年間、系列を扱う深層学習のほぼ全分野を支えた金字塔です。

## 直感的な理解
人間が長い文章を読むことを考えてみます。「昨日、田中さんが買ってきたケーキを、今日になって私は食べた」という文の「食べた」の主語を理解するには、文頭の「私は」を覚えておく必要があります。間に「田中さん」「ケーキ」など別の名詞が挟まっても、本当に必要な情報（主語）は最後まで保持し、不要になった情報は捨てる、という記憶の取捨選択が要ります。

素朴な再帰ネットワーク（RNN, Recurrent Neural Network）はこの「長く覚えておく」が極端に苦手でした。理由は後で詳しく述べますが、簡単に言うと「記憶を毎ステップ同じ計算で上書きし続ける」ため、20ステップも前の情報になると痕跡がほぼ消えてしまうのです。LSTM が出した答えは明快です。記憶を上書きで壊さず、足し算で更新する専用の通り道(セル状態)を1本用意し、その通り道に対して「どれを忘れるか」「どれを新しく書き込むか」「どれを外に出すか」を、データから学んだゲートで調整する。これだけで、何十ステップも離れた情報を保てるようになりました。

身近な比喩を続けると、セル状態は「ベルトコンベアの上を流れていく荷物」です。コンベアは特別な操作をしなければ荷物をそのまま運び続けます。ゲートは「コンベアの脇に立つ係員」で、係員が「この荷物はもう要らないから降ろす(忘却)」「新しい荷物を載せる(書き込み)」「今の荷物の一部を取り出して別の作業に渡す(出力)」を判断します。素朴な RNN にはこのコンベアがなく、荷物を毎回ぐちゃぐちゃに混ぜ直していた、というイメージです。

## 基礎: 前提となる概念
LSTM を理解する前に、いくつかの前提を噛み砕いておきます。

再帰ネットワーク(RNN)とは、系列データ(時間順に並んだデータ、例えば文章・音声・センサ時系列)を、1ステップずつ処理しながら「隠れ状態」と呼ばれる内部メモリを更新していくネットワークです。隠れ状態 `h_t`(時刻 t のメモリ)は、前の時刻のメモリ `h_{t-1}` と今の入力 `x_t` から計算されます。素朴な RNN は `h_t = tanh(W·[h_{t-1}, x_t] + b)` のように、毎時刻同じ重み行列 `W` を掛けてメモリを作り直します。ここで `tanh` は値を -1〜1 に収める関数、`[h_{t-1}, x_t]` は前のメモリと今の入力を横に連結したベクトル、`b` はバイアス(下駄)です。重要なのは「全ての時刻で同じ `W` を使い回す」点で、これが後述の勾配問題の元凶になります。

勾配消失問題とは、この素朴な RNN を学習するときに起きる現象です。ニューラルネットの学習は「誤差逆伝播」、つまり出力の誤差を入力側へさかのぼって各重みをどう直すかを計算することで進みます。RNN では誤差を時間方向にさかのぼる必要があり、これを BPTT(Backpropagation Through Time、時間方向の誤差逆伝播)と呼びます。このとき、誤差が1ステップさかのぼるたびに同じ重み行列(の関連項)を繰り返し掛けることになります。掛ける値が1より小さければ、20回も掛ければ `0.9^20 ≒ 0.12`、`0.5^20 ≒ 0.000001` のようにほぼ0になります(勾配消失)。逆に1より大きいと `1.5^20 ≒ 3325` のように発散します(勾配爆発)。勾配が0になると「遠い過去の重みをどう直せばよいか」の情報が届かず、長期依存(離れた時刻同士の関係)を学べません。Hochreiter は1991年の修士論文でこの現象を理論的に分析し、誤差の伝播が時間的な距離に対して指数的に減衰することを示しました。これが LSTM 開発の直接の動機です。

シグモイド関数 `σ(x) = 1/(1+e^{-x})` は、どんな実数を入れても出力を0〜1の範囲に押し込む関数です。LSTM ではこれを「ゲート」、すなわち「情報を何割通すか」を表す蛇口の開き具合として使います。0なら完全に遮断、1なら全通しです。tanh 関数は出力を-1〜1にする関数で、新しい情報の中身(正負の値)を作るのに使います。ゲートに `σ` を、書き込む情報の中身に `tanh` を使い分けるのがゲート機構の基本パターンで、これは GRU や注意機構のゲートにも受け継がれる考え方です。

## 仕組みを詳しく
LSTM は隠れ状態 `h_t`(短期記憶、外に出力する値)に加えて、セル状態 `c_t`(cell state、長期記憶の本体)という2本目のベクトルを持つ点が最大の特徴です。各時刻 `t` で、入力 `x_t` と前の隠れ状態 `h_{t-1}` を連結したものから、3つのゲートと1つの候補を計算します。

```
忘却ゲート: f_t = σ(W_f·[h_{t-1}, x_t] + b_f)   セル状態のうち何を忘れるか
入力ゲート: i_t = σ(W_i·[h_{t-1}, x_t] + b_i)   候補のうち何を書き込むか
出力ゲート: o_t = σ(W_o·[h_{t-1}, x_t] + b_o)   セルのうち何を出力に出すか
候補情報:   g_t = tanh(W_g·[h_{t-1}, x_t] + b_g) 今回書き込みたい新情報の中身
```

各記号の意味を言葉にすると、`W_f, W_i, W_o, W_g` は学習で決まる重み行列、`b_*` はバイアス、`[h_{t-1}, x_t]` は前のメモリと今の入力を横に連結したベクトルです。`f_t, i_t, o_t` はそれぞれ0〜1の値が並んだベクトル(蛇口の開き具合の集合)、`g_t` は-1〜1の値が並んだ「今回書き込みたい中身」です。

そして、セル状態と隠れ状態を次のように更新します。

```
c_t = f_t ⊙ c_{t-1}  +  i_t ⊙ g_t      古い記憶を一部忘れ、新しい情報を一部足す
h_t = o_t ⊙ tanh(c_t)                   セルの中身を選んで外に出す
```

`⊙` は要素ごとの掛け算(アダマール積)です。ここが LSTM の核心です。`c_t` は `c_{t-1}` に対して「忘却ゲートを掛けて、新情報を足す」だけで作られます。素朴な RNN のように重み行列を掛けて作り直すのではありません。足し算が主体だと、誤差を時間方向にさかのぼるときの掛け算の連鎖が起きにくく(忘却ゲートが1に近ければ勾配がほぼそのまま伝わる)、勾配が遠くまで減衰せずに届きます。この「記憶を素直に運ぶレーン」を、原論文は constant error carousel(一定誤差の回転木馬, CEC)と呼びました。誤差が減衰も増幅もせずぐるぐる回り続ける、という比喩です。数式で見ると、セル状態の勾配の伝播は `∂c_t/∂c_{t-1} = f_t`(おおむね)で、`f_t≒1` なら 1 が連続して掛かるだけなので減衰しません。素朴 RNN の `tanh` の導関数(最大 1、ふつう 0.x)が連続して掛かるのと対照的です。

tensor shape の数値例を挙げます。隠れ次元 H=128、バッチサイズ B=32 とすると、`h_t` も `c_t` も形は `[32, 128]`、各ゲート `f_t, i_t, o_t, g_t` もすべて `[32, 128]` です。素朴な RNN は重み行列が1組ですが、LSTM はゲート3つ+候補1つで重み行列が4組あります。入力次元 D、隠れ次元 H とすると、おおよそのパラメータ数は `4 × (H×(H+D) + H)` で、素朴な RNN(`1 × (H×(H+D)+H)`)の約4倍です。具体的に D=128, H=128 なら、素朴 RNN が約 33000、LSTM が約 132000 パラメータで、ちょうど4倍ほどになります。

1ステップ内部の流れを ASCII 図にすると次のようになります。

```
c_{t-1} ──►(× f_t)──►(+ i_t⊙g_t)──► c_t ──►tanh──►(× o_t)──► h_t
                                       │
x_t, h_{t-1} ─► [f_t][i_t][o_t][g_t]   └────────────────────► (次のステップへ c_t も渡す)
```

なぜこの設計なのか。素朴な RNN は「記憶を保つこと」と「記憶を更新すること」を同じ1本の隠れ状態でやろうとして、毎回上書きする破壊的更新になっていました。LSTM はこれを「保つ専用のレーン(セル状態、足し算更新)」と「読み書きを制御するゲート」に分離した、と理解すると本質が見えます。

実装上の重要な小技として、忘却ゲートのバイアス `b_f` を大きめ(例えば 1.0 や 2.0)に初期化すると学習初期から `f_t≒1` となり、記憶が壊れにくくなって長期依存の学習が安定します。これは Jozefowicz et al. (2015) が実験的に確立した知見で、多くのライブラリ実装でデフォルトとして採用されています。

## 手法の系譜と主要論文
LSTM は一度に完成したわけではなく、約20年かけて形が整っていきました。

Hochreiter と Schmidhuber による原論文「Long Short-Term Memory」(1997, Neural Computation 9(8))が出発点です。当時、勾配消失問題は Hochreiter 自身の1991年の修士論文で理論的に指摘されていました。原論文はセル状態と入力ゲート・出力ゲートを導入し、誤差が減衰せずに流れる constant error carousel によって長期依存を学べることを示しました。ただし当初の提案には忘却ゲートがありませんでした。

Gers, Schmidhuber, Cummins「Learning to Forget: Continual Prediction with LSTM」(2000, Neural Computation)が忘却ゲートを追加しました。忘却ゲートがないと、セル状態に情報が際限なく溜まり続けて飽和し、特に終わりのない連続予測タスクで破綻することが分かったためです。忘却ゲートで「古くなった記憶を能動的に捨てる」ことが可能になりました。今日「LSTM」と言うとき、普通はこの忘却ゲート付きの版を指します。

Gers らはさらに「peephole connection(覗き穴接続)」(2000, 2002)も提案しました。これはゲートの計算にセル状態 `c_{t-1}` 自体も入力する変種で、リズムやタイミングを精密に測るタスク(例えば一定間隔のイベント生成)で効果がありました。

Graves と Schmidhuber は LSTM を双方向化(BiLSTM)し音声認識・手書き文字認識に応用しました(2005、詳しくは [双方向RNN](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0482_bidirectional-rnn))。これにより LSTM が実用的な精度を出すことが広く知られるようになりました。

Greff, Srivastava, Koutník, Steunebrink, Schmidhuber「LSTM: A Search Space Odyssey」(2017, IEEE TNNLS)は、LSTM の各部品(各ゲート、peephole、活性化関数など)が本当に必要かを、複数タスク・約5400回の実験で徹底検証したサーベイ的研究です。トレードオフを理解する上で必読です。

その後、Cho らの [GRU (Gated Recurrent Unit)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0486_gru_note)(2014)が LSTM をゲート2つに簡略化し、Vaswani らの Transformer(2017)が再帰そのものを注意機構で置き換えて、長系列処理の主役は LSTM から Transformer へ移りました。ただし LSTM は今も省メモリ・低遅延・小データの場面で現役です。

## 論文の実験結果(定量データ)
Greff et al. (2017) のアブレーション(部品を1つずつ抜いて性能変化を見る実験)が、各ゲートの重要度を最もよく示しています。彼らは音声認識(TIMIT、音素誤り率で評価)、手書き文字認識(IAM Online)、多声音楽モデリング(JSB Chorales)の3タスクで、標準 LSTM(vanilla LSTM)と8つの変種を比較しました。報告された主な知見は次の通りです。

忘却ゲートを除去すると、3タスク全てで性能が大きく悪化しました。とくに連続予測的なタスクで顕著で、忘却ゲートは LSTM の最重要部品だと結論づけられています。出力活性化(`tanh(c_t)`)を除去した場合も、ほぼ全タスクで明確に悪化しました。この2つが「抜くと最も困る」要素です。一方で、peephole 接続を除去しても性能はほとんど変わらず(平均的にはむしろ僅かに簡略化が有利な場合すらあり)、入力ゲートと忘却ゲートを1つに束ねたり(後述の GRU 的な簡略化に相当する coupled input-forget gate)、入力活性化を除いたりしても、性能低下は比較的小さいと報告されました。これは「LSTM はかなり冗長で、簡略化の余地がある」ことを示し、GRU のような軽量版が成立する根拠になっています。

ハイパーパラメータの感度分析も重要です。学習率(learning rate)が最も結果を左右する要素で、隠れ層のサイズ(これも効くが学習率ほどではない)や運動量(momentum、ほとんど効かない)よりも支配的でした。さらに学習率と隠れ層サイズの間に強い相互作用はなく、ほぼ独立に調整できると報告されています。これは実務上きわめて重要で、LSTM を回すときはまず学習率を対数スケールで丁寧に探し、momentum の調整に時間をかけすぎない、という指針を与えます。

歴史的な性能例としては、Graves と Schmidhuber の BiLSTM が TIMIT フレーム単位音素分類で当時の手法を上回ったこと、Sutskever らの LSTM Seq2Seq が WMT'14 英仏翻訳で BLEU(翻訳の n-gram 一致率、高いほど良い)約34.8 を達成しフレーズベース統計翻訳(約33)を上回ったこと([Seq2Seq Encoder-Decoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0497_seq2seq-encoder-decoder) 参照)などが、LSTM が長期依存を実際に学べることの定量的証拠としてよく引かれます。合成タスクの「adding problem(系列中の指定2点の値を足す)」や「copy task(長い系列の先頭を末尾で再現する)」では、素朴 RNN が 10 ステップ程度で破綻するのに対し、LSTM は 100〜1000 ステップ規模の依存を学べることが原論文系の研究で示されています。

## メリット・トレードオフ・限界
メリットとして、忘却ゲートとセル状態により素朴な RNN より遥かに長い依存関係を学べます。ゲートで「覚える・忘れる・出す」を学習で調整するため、タスクに応じた記憶ができます。素朴な RNN なら数ステップが限界の依存も、LSTM は数十〜百ステップ規模まで扱えると報告されています。長年の実績があり、ライブラリ実装(各種フレームワークの組み込み LSTM 層)も成熟していて、cuDNN のような GPU カーネルで高度に最適化されています。

トレードオフと限界も明確です。重み行列が4組あり、素朴な RNN や GRU よりパラメータが多く計算も重めです。依然として逐次計算(時刻を1つずつ順に処理する)なので、長系列では Transformer のような並列処理に速度で劣ります。GPU を使っても系列方向には並列化できないため、系列長 T に比例した遅延が出ます。小規模データではパラメータが多いぶん過学習しやすく、GRU の方が良い結果になることもあります。さらに、勾配消失を緩和はしますが完全に消すわけではなく、数百〜数千ステップの超長期依存はやはり苦手で、ここは注意機構や状態空間モデルの方が有利です。固定サイズの隠れ状態に全履歴を圧縮するため、長い文脈の細部を全て保持することは原理的にできず、これは注意機構が「全時刻を保持して必要時に参照する」アプローチで解決した点です。

未解決・発展的な課題として、逐次性ゆえの学習・推論の遅さは本質的で、近年の状態空間モデル(S4, Mamba など)はこの逐次性を保ちつつ並列学習を可能にしようとする試みです。LSTM 自体を現代的に再設計する xLSTM(Beck et al., 2024)のような研究も登場しており、「再帰の利点(一定メモリ・低遅延)を保ったまま Transformer に対抗する」方向は今も活発な研究領域です。

## 発展トピック・研究の最前線
LSTM の発展方向は大きく3つあります。1つ目は軽量化で、[GRU (Gated Recurrent Unit)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0486_gru_note) がセル状態と隠れ状態を統合しゲートを2つに減らした代表例です。2つ目は文脈活用の拡張で、[双方向RNN](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0482_bidirectional-rnn) が前後両方向の文脈を使えるようにしました。3つ目は再帰そのものの置き換えで、[Seq2Seq Encoder-Decoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0497_seq2seq-encoder-decoder) に注意機構が組み込まれ、最終的に Transformer が再帰を不要にしました。

現在の最前線では、Transformer の弱点(系列長の2乗に比例する計算量・メモリ)を逆手に取り、線形コストで長系列を扱える再帰系モデルが再注目されています。状態空間モデル Mamba(Gu & Dao, 2023)や、LSTM を行列メモリ(matrix memory)・指数ゲーティング・並列学習対応に拡張した xLSTM(Beck et al., 2024)はその代表で、「LSTM の精神を現代のハードウェアで蘇らせる」流れと言えます。xLSTM は言語モデリングのスケーリング実験で、同規模の Transformer や Mamba と競合する性能を報告しており、再帰系の復権を象徴しています。LSTM を学ぶことは、これら最新モデルの設計思想(ゲーティング、状態の運搬、メモリの読み書き)を理解する土台になります。
