# CoverNet 軌道分類

## ひとことで言うと
車がこれからどう動くかを「未来の座標をゼロから計算して当てる問題」ではなく、「あらかじめ大量に用意した軌道の見本カタログの中から、一番それらしい一枚を選ぶ問題」として解く手法です。つまり運転計画を回帰(数値を直接出す)ではなく分類(選択肢から選ぶ)で扱います。カタログに載っている軌道がすべて物理的に実現可能なものなら、何を選んでも「ありえない軌道」は絶対に出てこない、というのが最大の狙いです。

## 直感的な理解
回帰で未来の座標を自由に出させると、二つの困りごとが起きます。一つは多峰性(ありえる未来が複数の山に分かれること)による平均化。交差点で直進・右折・左折のどれもありえるとき、それらの平均をとった「どこにも行かない軌道」を出してしまいます。もう一つが物理的にありえない軌道です。回帰は数値を制約なく出せるので、「時速200kmで瞬間的に90度曲がる」ような、車の運動として不可能な軌道を平然と吐くことがあります。

CoverNet はこの両方を発想の転換で同時に潰します。「未来を計算で作る」のをやめ、「物理的に実現可能な軌道だけを集めた有限の集合を先に作り、その中から選ぶ」。カタログの中身が最初から実現可能なものだけなら、出力は構造的に必ず妥当になります。レストランのメニュー(作れる料理しか載っていない)から選ぶようなもので、メニューにない無茶な注文(物理破綻軌道)は原理的に出せません。

名前の CoverNet は「軌道空間を cover(覆う・カバーする)集合」から来ています。ありえる軌道の空間を、有限個の代表軌道で漏れなく覆おう、という思想です。

## 基礎: 前提となる概念
軌道(trajectory)は時刻ごとの位置の並びです。6秒先を0.5秒刻みなら12点、各点 (x,y) で `[12, 2]`。

回帰と分類。回帰は「数値をそのまま当てる」(座標を出す)、分類は「選択肢のどれかを選ぶ」(N個の候補のどれか)。回帰の出力空間は連続で無限、分類は離散で有限です。CoverNet の核心は出力空間を連続から有限の集合へ切り替えたことにあります。有限になると softmax で確率分布を作れて学習が安定し、しかも集合の中身を吟味すれば妥当性を担保できます。

運動学的・動力学的制約(kinematic / dynamic constraints)。車にはハンドルの最大切れ角、加速度・減速度の限界、最高速度といった物理的な制約があります。これを破る軌道は危険であり無意味です。CoverNet の「動的集合」はこの制約を集合生成の段階で守らせます。

softmax と交差エントロピー。softmax は実数のベクトルを「全部正で合計1」の確率分布に変える関数。交差エントロピー損失は「正解カテゴリの予測確率を上げる」分類の標準的な損失で、正解クラスの確率が低いほど大きな罰を与えます。

量子化誤差(quantization error)。連続な対象を有限の代表点で近似するときに必ず生じる「ぴったりには表せない誤差」のこと。CoverNet では「正解の軌道が集合の中の一番近い軌道とどれだけずれるか」がこれにあたり、集合の細かさが精度の上限を決めます。

## 仕組みを詳しく
ステップ1: 軌道集合(trajectory set)を作る。CoverNet の特徴は集合を二通りで作れる点です。

固定集合(fixed set): 学習データ全体から代表的な軌道を抜き出し、互いに近すぎる軌道は一つにまとめ、距離が一定以上(イプシロン ε 以上)離れた代表だけ残します。これで軌道空間を細かさ ε で覆う集合ができます。ε を小さくすると軌道本数が増えて細かく、大きくすると粗くなります。規模は数千本〜数万本です。ε と精度・本数の関係は、「覆い(cover)」を ε で作るという CoverNet の名の由来そのものです。

動的集合(dynamic set): ここが最も独創的な部分です。車の現在の速度・進行方向から出発し、車の運動モデル(物理法則に基づく動きの式)に従って到達可能な軌道だけをその場で生成します。時速60kmで走る車は次の数秒で急停止はできても瞬間的に真横へは動けません。動的集合は今の状態から物理的に届く範囲だけを候補にするので、低速時は候補が少なく密、高速時は遠くまで届く候補、というように状況に応じて中身が変わります。これにより常に「今この車にとって実行可能な軌道」だけが選択肢になります。

ステップ2: 分類で選ぶ。入力は自車周辺をラスタ画像(真上から見た地図状の鳥瞰図、道路・他車・自車を色分けして描いた画像)として与え、CNN(畳み込みニューラルネット、画像処理に強い構造)に通します。出力は集合の各軌道に対するスコアで、集合が N 本なら `[N]`。softmax で確率分布にします。最大確率の軌道が最有力の未来、上位 k 本がありえる候補トップ k です。

ステップ3: 損失計算。学習時は集合の中から「正解軌道に一番近い一本」を正解ラベルとみなし、交差エントロピー損失で学習します。近さは各時刻の位置距離の平均などで測ります。

数値例:
```
6秒先を0.5秒刻み(12点)、集合 N=2206 本、入力ラスタ [3, 500, 500]
CNN → 出力 [2206] のスコア → softmax → 確率 [2206]
argmax(最大の位置)が選ばれる軌道のインデックス
学習ラベル = 正解に最も近い集合内軌道のインデックス
```

回帰との before / after 比較:
```
before(回帰):  出力 [12,2] の座標を直接予測
               → 物理的にありえない軌道も出る / 平均化で潰れる
after(CoverNet): 出力 [N] の確率
               → 集合に入るのは実現可能な軌道だけ→破綻しない
               → 複数候補を確率付きで自然に出せる
```

## 手法の系譜と主要論文
多峰な軌道予測の流れの中で、CoverNet は「分類で計画する」系譜の代表です。

先行して MultiPath(Chai, Sapp, Bansal, Anguelov, CoRL 2019, arXiv:1910.05449)が「事前定義したアンカー軌道(典型経路の雛形)ごとに確率を出し、選ばれたアンカーを残差(オフセット)回帰で微調整する」枠組みを提示しました。分類でモードを選び、回帰で微調整する折衷です。

CoverNet(Phan-Minh, Grigore, Boulton, Beijbom, Wolff, CVPR 2020, arXiv:1911.10298)は、ここから残差回帰を捨て純粋な分類に振り切りました。最大の新規性は動的集合、すなわち「現在の運動状態から到達可能な軌道だけをその場で生成する」考え方で、これにより出力が必ず物理的に妥当になることを保証しました。動機は、当時の回帰ベースや GAN ベース(敵対的生成で多様な軌道を作る手法)が物理破綻や学習不安定に悩んでいたことです。分類なら出力空間が有限で安定し、集合の中身を制御すれば妥当性も担保できます。

CoverNet 以降、「軌道の語彙(vocabulary、よく出る軌道パターンの辞書)上の確率分布を学ぶ」という発想がより大規模・エンドツーエンドへ発展しました。語彙を数千本に増やし、ラスタ画像ではなくベクトル地図やセンサ特徴から直接スコアリングする方向です。CoverNet の「分類で計画する/集合で妥当性を保証する」骨格は、この語彙ベース計画の直接の祖先と言えます。MultiPath の残差回帰路線と CoverNet の純分類路線、この二つの対比が「分類で計画する」設計の二大選択肢です。

## 論文の実験結果(定量データ)
評価は nuScenes prediction ベンチマーク(Caesar et al., CVPR 2020)で行われました。指標を説明します。

minADE_k は K 本の候補のうち正解に最も近い一本を選び、その各時刻の位置誤差を平均した距離(メートル)。K=5 や K=10 で報告され、小さいほど良い。MissRate_k は最良候補の終点が正解から一定距離(2m など)以上外れた割合。OffRoadRate は予測軌道が走行可能領域(道路)から外れた割合で、これが「物理・地図妥当性」を測る CoverNet にとって象徴的な指標です。

CoverNet は nuScenes で、当時の回帰ベースや物理ベースのベースラインに対し競争力のある minADE/MissRate を、より低い OffRoadRate(道路逸脱がほぼゼロに近い)で達成したと報告されています。回帰系が時に道路を外れる軌道を出すのに対し、集合の中身を走行可能なものに限った CoverNet は構造的に道路外へ出にくい、という設計の効果が数値で表れました。これは「精度を多少譲っても妥当性を保証する」という CoverNet の価値を端的に示します。

アブレーション(要素を抜く検証)では二つが重要です。第一に、軌道集合の細かさ ε を粗くする(本数を減らす)と量子化誤差が増え、最良候補ですら正解に寄れず minADE が悪化します。逆に ε を細かくすると minADE は改善しますがクラス数(本数)が爆発し、分類が難しくなって学習・推論が重くなります。報告では数百本から数千本へ増やすと minADE が改善する一方、改善は逓減し、ある密度で頭打ちになることが示されました。第二に、固定集合より動的集合のほうが OffRoadRate と物理妥当性で優れる傾向が示され、「今の運動状態に合わせて候補を作る」ことの効果が確認されました。

## メリット・トレードオフ・限界
メリット。集合の中身を実現可能な軌道に限れば、物理的にありえない軌道を構造的に出せなくなります(妥当性の保証)。これは安全性を最重視する場面で大きな価値です。分類問題なので学習が安定し、softmax の確率として多峰性を自然に表現でき、複数候補をそのまま出せます。動的集合を使えば車の現在の運動状態に応じて候補が自動で適応します。

トレードオフと限界。最大の限界は量子化誤差で、集合に無い軌道は絶対に出せず、集合の細かさが精度の上限になります。細かくしようとすると軌道本数が爆発し、分類クラス数が増えて計算が重くなる(ε と精度・計算量の真っ向トレードオフ)。軌道集合の作り方(ε の値、運動モデルの選択)が性能に直結し、設計の良し悪しが結果を左右します。残差で微調整しないため、アンカー+残差型(MultiPath)より「ぴったりの軌道」へは寄せにくい。研究上の未解決課題としては、集合の本数を増やさずに表現力を上げる方法(階層的・条件付きの集合)、集合の中身を学習でデータ駆動に最適化する方法、そして分類の確率が本当に「起こりやすさ」として較正(キャリブレーション、予測確率と実際の頻度が一致すること)されているかの保証が挙げられます。

## 発展トピック・研究の最前線
CoverNet の「有限集合上の分類」は、後年の大規模語彙ベース計画へ直結します。数千本規模の軌道語彙を用意し、ベクトル地図やセンサ特徴から直接スコアリングしてエンドツーエンドで学習する手法が、量子化誤差と本数爆発のジレンマを、より賢い語彙設計と学習で緩和しようとしています。また、CoverNet 型の「妥当性を集合で保証する」発想と、生成モデル(拡散・フローマッチング)の「連続空間から滑らかにサンプリングする」発想を組み合わせ、生成した候補を実行可能性で篩(ふるい)に掛けるハイブリッドも研究されています。較正の観点では、分類確率を実際の発生頻度へ合わせる温度スケーリングや、コンフォーマル予測(統計的に保証された候補集合を作る手法)を計画へ持ち込む動きもあります。
