# Bezier/パラメトリック軌跡表現

## ひとことで言うと
軌跡（これからの動き）を、全時刻の値を1つずつ予測するのではなく、ベジェ曲線（Bezier curve）の少数の制御点（control points）というコンパクトなパラメータで表す設計です。たとえば64時刻ぶんの動きを、5個の制御点だけ予測し、それを滑らかな曲線に展開します。すると出力が自動的になめらか（急なガタつき＝ジャークが小さい）になり、出力ヘッドが予測すべき数字の個数も激減します。滑らかさを「後処理のフィルタ」や「損失への罰則項」で外から押し付けるのではなく、表現の構造そのものに組み込むのが要点です。

## 直感的な理解
道路を走る車の軌跡を「次の3秒間、0.05秒刻みで車がどこにいるか」と表すとします。これは60点の座標列です。この60点を独立に予測すると、1点目は前に、2点目は急に横に、3点目はまた前に、という物理的にあり得ない暴れた軌跡が出やすくなります。車は質量を持つ剛体なので、瞬間的にワープしたり折れ曲がったりできません。にもかかわらず、座標を1点ずつ独立に出すニューラルネットには「隣の点と整合せよ」という制約がどこにも入っていないので、平気で暴れます。

ここで、製図で曲線を引くときの「雲形定規」を思い浮かべてください。数本のピン（制御点）を置くと、その間を滑らかにつなぐ曲線が一意に決まります。ピンを動かせば曲線全体が連動して変わり、決して途中でカクカク折れません。ベジェ表現はこの発想です。「60点を直接決める」のではなく「5本のピンを置く」。ピンの数が少ないので、表現できる形は『滑らかな曲線』に限定され、暴れる自由度がそもそも存在しません。これは「制約を後から課す」のではなく「暴れる選択肢を最初から表現空間から消す」という考え方で、専門的には帰納バイアス（inductive bias、モデルが解の形について最初から持つ思い込み）を設計で注入する、と言います。

なぜ自動運転でこれが効くのか。プランナ（経路計画器）の出力は最終的に車両のコントローラ（実際にハンドルとアクセルを動かす制御器）に渡されます。コントローラは滑らかな指令しか追従できません。ガタガタの軌跡を渡すと追従できず、乗員には「カックンカックン」の不快な揺れ（ジャーク）として伝わります。出力が定義から滑らかなら、この問題が根本から消えます。

## 基礎: 前提となる概念

### 軌跡とは何か、どう表すか
軌跡（trajectory）は時間とともに変化する状態の列です。最も素朴には2次元座標 (x, y) を時刻ごとに並べた `(T, 2)` のテンソルです（T は時刻数、テンソルとは多次元の数値配列のこと）。しかし自動運転では座標を直接出すより「制御信号」を出す流儀があります。代表的なのが自転車モデル（unicycle / bicycle model、車を自転車のように1本の軸で近似した運動モデル）で、車の動きを「加速度 a」と「曲率 κ（カーブの曲がり具合、旋回半径の逆数で、まっすぐなら0、急カーブほど大きい）」の2信号で表します。各時刻の (a, κ) を初期状態から積分すると (x, y, 速度, 向き) が一意に復元できます。座標を直接出すと「物理的にあり得ない加速」を出してしまいがちですが、(a, κ) を出すと積分の過程で物理的整合性が保たれやすい、という利点があります。これを動力学積分（dynamics integration）と呼びます。

### ジャークとは
ジャーク（jerk）は加加速度、すなわち加速度の時間微分です。位置を1回微分すると速度、2回で加速度、3回でジャークです。人間が不快に感じる揺れの主因は加速度そのものより、その急変であるジャークだと知られています。エレベータが滑らかに加速・減速するよう設計されているのはジャークを抑えるためです。軌跡の滑らかさを定量化する標準的な指標がジャークの二乗積分 ∫|d³x/dt³|² dt で、これが小さいほど滑らかです。乗り心地の研究では、ピークジャークが 0.9 m/s³ を超えると多くの人が不快と感じる、といった報告があり、定量的な設計目標になります。

### パラメトリック曲線とは
パラメトリック曲線（parametric curve）は、少数のパラメータで形が決まる曲線です。直線は2点（始点・終点）で決まり、円は中心と半径で決まります。ベジェ曲線は n+1 個の制御点で決まる多項式曲線です。重要な性質は「自由度がパラメータ数に制限される」こと。60点の座標列は60×2の自由度を持ちますが、5制御点なら5×2の自由度しかありません。残りは曲線の式が滑らかに埋めます。次数（曲線の多項式の最高次）が n のベジェ曲線は、最大で n−1 回まで曲がる向きを変えられる、という直感的な制約があり、これが「表現力の上限」を決めます。

### ベルンシュタイン基底とは
ベジェ曲線の数学的な核は「ベルンシュタイン基底多項式（Bernstein basis polynomial）」です。これは Sergei Bernstein が1912年に導入した、区間 [0,1] 上の多項式の重み関数の集まりです。制御点が n+1 個（次数 n）のとき、i 番目の基底は

```
B_{i,n}(t) = C(n,i) · t^i · (1−t)^(n−i)      (i = 0, 1, ..., n,  t ∈ [0,1])
```

ここで C(n,i) は二項係数（n 個から i 個選ぶ組み合わせの数）、t は曲線上の位置を表す正規化された時刻（始点が t=0、終点が t=1）です。意味は「時刻 t において、各制御点をどれだけ混ぜるか」の重みです。たとえば n=4（制御点5個）なら基底が5本あり、どの時刻でも5本の重みの合計は必ず 1 になります（partition of unity、単位の分割という性質）。さらに各基底は常に0以上なので、曲線は制御点の凸結合（重み付き平均）になり、必ず制御点を囲む凸包（convex hull）の内側に収まります。この凸包性は「制御点が安全領域内なら軌跡も安全領域内」という保証に使え、運動計画で重宝されます。曲線は制御点 P_i の重み付き和

```
C(t) = Σ_i B_{i,n}(t) · P_i
```

として作られます。実装上は、T 個の時刻 × (n+1) 個の制御点ぶんの基底行列 `[T, n+1]` を事前計算して固定行列として保持し、制御点との行列積を取るだけです。基底は入力に依存しない定数なので、ニューラルネットの中では学習しないバッファ（PyTorch でいう register_buffer、勾配が流れず保存だけされるテンソル）として持つのが一般的です。

## 仕組みを詳しく

軌跡デコーダ（特徴から軌跡を生成する部分）としてベジェ表現を使う典型的な実装は、次の流れです。文脈ベクトル（シーンを要約した特徴）から制御点を回帰し、固定基底で軌跡に展開します。

具体的な設定例として、時刻数 T=64、信号数 S=2（加速度・曲率）、制御点数 num_controls=5、特徴次元 D=256 を考えます。

ステップ1: 文脈の集約。シーンの情報源（俯瞰特徴 BEV `[B, 256, H, W]`、過去の視覚履歴 `[B, 896]`、自車の運動履歴 `[B, 256]` など。B はバッチサイズ）をそれぞれ線形層で `[B, 256]` に揃えて足し合わせ、小さな全結合ネットワーク（MLP、2層＋GELU 活性化など）に通して文脈ベクトル `[B, 256]` を作ります。BEV は空間方向に平均プーリングして `[B, 256]` に潰すのが簡便な方法です。

ステップ2: 制御点の回帰。文脈ベクトルを線形層（control head）に通し、`num_controls × num_signals = 5 × 2 = 10` 個の値を出力し、`[B, 5, 2]` の制御点テンソルに整形します。意味は「加速度プロファイル用の5制御点」と「曲率プロファイル用の5制御点」です。

ステップ3: 基底展開。事前計算したベルンシュタイン基底 `[T, num_controls] = [64, 5]` と制御点 `[B, 5, 2]` をテンソル積（einsum で `"pc,bcs->bps"`）して `[B, 64, 2]` の信号プロファイルを得ます。p は時刻インデックス、c は制御点インデックス、s は信号インデックスです。

ステップ4: 整形。`[B, 64, 2]` を `[B, 128]` に平坦化して返す、あるいは自転車モデルで積分して座標列 (x, y) に変換します。

数値例で before/after を見ます。

```
before（各時刻独立予測）:
  自由パラメータ = 64 × 2 = 128 個。隣接時刻が無関係に動ける。
  → 加速度プロファイル例: [3.0, -2.8, 3.1, -2.9, ...] （暴れる、ジャーク大）

after（ベジェ5制御点）:
  自由パラメータ = 5 × 2 = 10 個。残り118次元は基底が滑らかに補間。
  → 加速度プロファイル例: [1.2, 1.4, 1.5, 1.4, 1.1, ...] （滑らか、低ジャーク）
```

設計上の重要な選択が「何に対して滑らかさをかけるか」です。座標 (x, y) に直接ベジェをかける流儀もありますが、加速度・曲率プロファイルにかける流儀のほうが、ジャーク（加速度の変化率）を直接抑えられ、かつ自転車モデルの物理制約と整合します。ベジェの便利な性質として、曲線の微分（速度・加速度・ジャーク）もまたベジェ曲線になり、その制御点は元の制御点の差分で閉じた式で書けます。だからジャークの大きさを制御点だけから解析的に評価でき、損失に組み込むのも容易です。制御点数の選び方は表現力と滑らかさのトレードオフで、少なすぎると急なレーンチェンジが表現できず、多すぎると暴れる自由度が戻ってきます。実務では妥当性チェック（制御点数は2以上、かつ時刻数以下）を入れます。

ASCII でイメージを示します。

```
制御点（5本のピン）        ベジェ曲線（展開後の軌跡）
   P1                          ___
      P2                     _-    -_
         P3      →         _-        \
      P4                  /           \_
   P5                    /              \
（暴れずに滑らかな1本の曲線として補間される）
```

## 手法の系譜と主要論文

ベジェ／ベルンシュタイン表現自体は計算幾何の古典で、Paul de Casteljau（1959、Citroën 社内）と Pierre Bézier（1962、Renault 社内）が自動車ボディの曲面設計のために独立に開発しました。皮肉にも今また自動運転の軌跡設計に戻ってきています。基礎のベルンシュタイン基底は Sergei Bernstein が1912年にワイエルシュトラスの近似定理の構成的証明のために導入したものです。

軌跡予測・計画を「低次元の代表で表す」発想の系譜は次の通りです。

- 古典的運動計画。Werling et al.（ICRA 2010）の Frenet 座標系（道路の中心線に沿った座標系）での5次多項式（quintic polynomial）軌道生成が長く標準でした。なぜ5次か。始点と終点で位置・速度・加速度の6つの境界条件を満たすには6係数（5次）が必要で、5次多項式はジャークの二乗積分を最小化する解になる（最小ジャーク軌道）という最適制御の定理に基づきます。ベジェ・Bスプライン・多項式はいずれも「微分可能で滑らかさを次数で調整でき、連続曲率を保証しやすい」点を共有します。

- MultiPath（Chai et al., CoRL 2019, arXiv:1910.05449）。提案: あらかじめ訓練データからクラスタリングで作った代表軌跡（anchor、アンカー）の集合を用意し、ネットワークは「どのアンカーが尤もらしいか」の確率と、各アンカーからの微小オフセットを予測する。動機: 未来は多峰（左折も右折もあり得る）なので、1本の平均軌跡を出すと「どちらでもない直進」というモード平均化（mode averaging）の失敗が起きる。アンカーで離散化することでこれを防ぐ。新規性: 連続回帰問題を「分類＋微調整」に変換した点。

- TNT（Target-driveN Trajectory prediction; Zhao et al., CoRL 2020, arXiv:2008.08294）。提案: まず到達目標点（target）の候補を離散サンプリングしてスコア付けし、選ばれた目標へ向かう軌跡を生成する。動機: アンカー集合の質に依存する MultiPath の弱点を、目標点ベースで緩和。DenseTNT（Gu et al., ICCV 2021, arXiv:2108.09640）はアンカー不要の密な目標点予測へ発展させ、ヒューリスティックなアンカー設計を取り除きました。

- パラメトリック曲線＋動力学。Trajectron++（Salzmann et al., ECCV 2020, arXiv:2001.03093）は制御信号（加速度・操舵）を予測して動力学で積分する dynamics integration を導入し、物理整合性を保証しました。MTR（Motion Transformer; Shi et al., NeurIPS 2022, arXiv:2209.13508）は学習可能なクエリ（motion query）でアンカーと回帰を統合し、Waymo の予測ベンチマークで当時の首位を取りました。

ベジェ表現は、アンカー（離散的な型から選ぶ）とは異なり「連続パラメータ（制御点）で軌跡をコンパクト化する」点が特徴ですが、「軌跡を低次元で表す」という動機を共有します。近年は両者を組み合わせ、アンカーごとにベジェ制御点を回帰するハイブリッドも見られます。

## 論文の実験結果（定量データ）

軌跡予測の標準ベンチマークは Argoverse（自動運転車の走行ログ、約32万シーン）と nuScenes、評価指標は次の通りです。

- minADE（minimum Average Displacement Error）: K 本予測したうち最良の軌跡について、全時刻の予測位置と正解位置の平均ユークリッド距離（メートル）。小さいほど良い。K 本のうち最良だけを採点するのは「複数の妥当な未来のどれか1つが当たればよい」という多峰予測の評価思想です。
- minFDE（minimum Final Displacement Error）: 最終時刻だけの位置誤差（メートル）。終点がどれだけ合うかを見る。
- MR（Miss Rate）: 最終位置が正解から2mを超えて外れた予測の割合。安全に直結するため重視される。

報告されている代表的な数値（Argoverse motion forecasting、K=6、約3秒予測）:

- MultiPath は素朴な単峰回帰ベースラインに対し minFDE と MR を明確に改善し、特にモード平均化を避けることで「直進と右折の中間」のような無意味な予測を減らしました。アブレーションでアンカー数を減らすと多峰性を取りこぼし MR が悪化します。
- TNT は Argoverse で minFDE およそ1.4m台、MR 十数%台と報告され、当時のリーダーボード上位を記録しました。アブレーションでは「目標点予測を外すと性能が大きく落ちる」ことが示され、目標条件付けの寄与が定量化されています。
- DenseTNT はアンカー集合への依存を除いて MR をさらに下げ（Argoverse で MR がベースラインから数ポイント低下）、ヒューリスティックなアンカー設計が不要になることを示しました。
- MTR は Waymo Open Motion Dataset で soft mAP と minADE/minFDE を当時最高水準に押し上げ、クエリベースの統合の有効性を示しました。

パラメトリック／動力学積分の効果については、Trajectron++ が nuScenes 予測で、座標を直接回帰する版に比べ動力学積分版のほうが物理的に妥当（acceleration/jerk の制約違反が減少）かつ精度も改善することを報告しています。一般に「制御信号を滑らかな曲線で表す」と、生の座標回帰に比べジャーク指標が桁で改善する一方、minADE は曲線の自由度が足りないと若干悪化しうる、というトレードオフが繰り返し観測されています。たとえば制御点を5から3に減らすとジャークはさらに下がるが急な回避が表現できず minFDE が悪化する、といった具合で、これが「制御点数は表現力と滑らかさの妥協点」であることの実証です。

数値の意味を補足します。minFDE が 0.3m 改善するというのは、3秒後の予測位置が30cm正解に近づくということで、車線幅が約3.5mであることを思えば「隣の車線か自車線か」の判定精度に直結する重要な差です。MR が数%下がるのは、致命的に外れる予測の頻度が減るということで、安全性の観点で minADE 以上に重視されます。

## メリット・トレードオフ・限界

メリット:
- 出力が定義から滑らか（低ジャーク）。後処理のローパスフィルタも、損失への滑らかさ罰則項も不要で、滑らかさが構造として保証される。
- 軌跡を少数パラメータ（例10個）で表すコンパクト表現。学習しやすく、出力ヘッドが軽量。
- ベジェの凸包性により「制御点が安全領域内なら軌跡も安全領域内」という幾何的保証が得られ、安全制約と接続しやすい。
- 1回の前向き計算で軌跡が出るため決定的（同じ入力で同じ出力）で、生成系より推論が速く再現性・検証が容易。
- 加速度・曲率プロファイルを直接滑らかにすると、車両コントローラに優しい物理的に妥当な指令になりやすい。

トレードオフ・限界:
- 決定的な1点出力なので多峰な未来（左折も右折もあり得る）を表現できず、モード平均化の罠が残る。これを解くにはアンカー併用や混合密度出力（複数のベジェを確率付きで出す）、生成モデル（拡散・フローマッチング）への移行が必要。
- 制御点が少ないと急なレーンチェンジや障害物回避を表現しきれない。表現力と滑らかさのトレードオフは原理的に消えない。
- シーン特徴を1ベクトルに潰してから制御点を回帰する実装では、障害物の正確な空間位置の一部が失われる。空間を保つ attention 系デコーダと対照的。
- 未解決の課題として、制御点数を状況に応じて動的に変える（直線では少なく、複雑な操舵では多く）適応的次数選択や、ベジェ表現の上で安全制約（衝突回避・運動学制約）を厳密に満たす最適化との接合は研究途上です。

## 発展トピック・研究の最前線
近年は「軌跡をパラメトリック表現で出す」発想が拡散モデル系プランナにも取り込まれ、拡散の出力空間をベジェ係数にしてサンプルを最初から滑らかにする試みがあります。また、ベジェ制御点の上で微分可能な衝突回避コスト・運動学制約を最適化する「学習＋最適化のハイブリッド」（differentiable optimization layer、最適化問題を1層として埋め込み勾配を通す手法）も活発です。アンカー軌跡をベジェで表し、アンカーごとに制御点をオフセット回帰するマルチモーダル化も自然な発展方向です。さらに、Bスプラインや有理ベジェ（NURBS）への一般化で、より長い軌跡を区分的に滑らかに、かつ局所的に編集可能に表す研究も進んでいます。
