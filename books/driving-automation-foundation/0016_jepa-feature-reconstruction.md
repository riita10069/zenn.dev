# JEPA風特徴再構成損失

## ひとことで言うと

JEPA風の特徴再構成損失(feature reconstruction loss)は、未来予測モジュールが出した「未来のシーン特徴」と、実際に未来に起きた映像から作った「正解の特徴」を比べて、ズレ(誤差)を測る損失関数です。比べるのは画像のピクセルではなく特徴(抽象的な数値表現)で、これがJEPA(LeCunが2022年に提唱した自己教師あり学習の設計思想)の流儀です。この損失を小さくするように学習すると、モデルは未来を抽象的に見通す力を身につけます。「絵の正確さ」ではなく「要点の当たり具合」を採点する損失だと考えると直感に合います。

## 直感的な理解

損失関数(loss function)とは、モデルの出力が「どれだけ間違っているか」を1つの数値で表す関数です。学習とは、この数値が小さくなる方向にモデルの重み(パラメータ)を少しずつ調整する作業のことです。だから「何を損失にするか」は、モデルが何を学ぶかを決める根幹です。料理にたとえれば、損失は「味見の基準」です。基準が「見た目の派手さ」なら見た目重視の料理人になり、「味の深さ」なら味重視になります。何を採点するかが、育つ能力を決めます。

未来を予測するモデルを学習させたいとき、素朴には「未来の画像を予測させて、本物の未来画像とピクセル単位で比べる」方法(ピクセル再構成損失)が考えられます。しかしこれには大きな問題が二つあります。第一に、未来は本質的に不確実です。「前の車がブレーキを踏むか踏まないか」は完全には予測できません。ピクセルで正解を1枚に決めて比べると、モデルは「あり得る未来の平均」を出そうとしてボヤけた画像を予測しがちです(平均化のボケ)。これは、起こりうる複数の未来をすべて満たそうとして、どれにも当たらない中間のぼやけた画を出す現象です。第二に、ピクセルには運転に関係ない細部(空の雲、影、路面の質感)が大量に含まれ、そこを合わせる学習に労力が吸われます。そこでJEPAの発想では、ピクセルではなく特徴空間で予測・比較し、運転に関係する抽象情報に焦点を当て、どうでもいい細部や予測不能なノイズは特徴抽出の段階で捨てます。

## 基礎: 前提となる概念

特徴空間(feature space): 画像をニューラルネットで抽象的な数値表現に変換した空間です。高い層の特徴は「車がいる」「車線がある」「空きスペースがある」といった意味的な情報を表し、雲の形や路面の質感のような枝葉は捨てられています。

MSE(Mean Squared Error=平均二乗誤差): 予測と正解の差を要素ごとに2乗して平均した値です。小さいほど近いことを意味します。式は MSE = mean((pred - target)^2)。2乗するので大きなズレを強く罰し、符号の影響を消します。確率的に言えば、MSEを最小化することは「ガウス分布を仮定したときの平均(条件付き期待値)を当てにいく」ことに相当し、これが多峰的な未来で平均化ボケを生む数学的な理由です。

凍結(frozen)と detach(勾配を切る): 凍結とは、あるネットワークの重みを学習中に更新しないことです。detach とは、計算グラフ(順伝播の記録)からその部分を切り離し、そこを通って勾配(学習の更新信号)が流れないようにすることです。stop-gradient とも呼ばれます。未来予測の正解側はこの二つを使い、「正解は動かさず、予測側だけを正解に近づける」ようにします。

崩壊(collapse、representation collapse): 自己教師あり学習で起こりうる失敗で、予測も正解も全部同じ値(たとえば全部ゼロ)にすると損失が0になってしまう詐欺的な解に逃げる現象です。何も学ばずに損失だけ下げられてしまうので、これを防ぐ工夫が自己教師学習の核心になります。崩壊には「完全崩壊(全部同じ定数)」と「次元崩壊(特徴が一部の次元に潰れる)」があります。

EMA(指数移動平均、Exponential Moving Average): あるネットワークの重みを、別のネットワークの重みにゆっくり追従させて更新する方法です。target = m * target + (1-m) * online のように混ぜます(m は0.99〜0.999のように1に近い係数)。ターゲットを少しずつ動かすことで安定させつつ、固定よりも質の高いターゲットを得る狙いがあります。

## 仕組みを詳しく

JEPA風特徴再構成損失の典型的な実装を、入力・処理に分けて説明します。

入力は二つです。第一に予測特徴、未来予測モジュールが返した複数の未来特徴のタプル/リスト。各 `[B, C, H, W]`(C は特徴次元、H×W は空間グリッド)。第二に正解特徴、同じ形のターゲット特徴のタプル/リスト。これは凍結したバックボーン(学習で重みを更新しない画像エンコーダ)を、実際の未来フレーム(+1.6s, +3.2s, +4.8s, +6.4s など)に適用して作ります。「凍結」かつ「detach」なのが要点で、正解側は動かさず予測側だけを正解に近づけます。

処理ステップは次のとおりです。第一に個数チェック。予測も正解も同じ未来ステップ数(例: 4)あるか確認し、違えばエラーにします。第二に各未来時刻ごとにMSEを計算します。`torch.mean((pred - target) ** 2)`。形が一致しなければエラーにします。第三に、時刻数ぶんのMSEを `[num_future_steps]` のテンソルに積みます。第四に時刻ごとの重み付け。時間方向の重みを掛けます。たとえば「近い未来ほど重視」したければ `[2.0, 1.5, 1.0, 0.5]` のように指定でき、何も指定しなければ全部1にします(近い未来ほど予測が確実で学習信号として信頼できる、という考え方)。第五に重みの正規化。重みの合計でスケールを保つよう正規化します(weights / total * num_future_steps)。これで重みを変えても損失全体の大きさが大きくブレません(損失のスケールが変わると学習率の調整がやり直しになるため)。合計がほぼ0という壊れた指定のときはNaN(非数)を防ぐため正規化をスキップする安全装置を入れるのが堅実です。第六に集約。`reduction="mean"` なら重み付き平均をとってスカラーを返し、`reduction="none"` なら時刻ごとの損失をそのまま返します(どの未来時刻が難しいか診断するログ用)。

数値例: ある時刻の予測特徴と正解特徴が全要素で平均0.04だけずれていたら、その時刻のMSEは 0.04^2 = 0.0016。4時刻ぶん同様に出して、時間重みを掛けた平均をとる、というイメージです。

なぜ正解側を凍結バックボーンで作るのか。もし正解側のエンコーダも一緒に学習させてしまうと、モデルは予測も正解も全部同じ値にする崩壊(collapse)に逃げられます。正解側を固定すれば、予測側だけが本物の未来情報に合わせざるを得ず、崩壊を防げます。これはJEPA系で決定的に重要なテクニックで、「予測側(online)と正解側(target)を非対称にする」という設計思想の核です。

ASCII図で全体像を示します。

```
  未来の実フレーム ──→ [凍結バックボーン] ──(detach)──→ target_features
                                                            │
  現在のシーン ─→ [未来予測モジュール] ──→ predicted_features  │
                                              │              │
                                              └─── MSE ───────┘ ← この差を最小化
                                              (予測側だけ更新、正解側は固定)
```

## 手法の系譜と主要論文

LeCun (2022) "A Path Towards Autonomous Machine Intelligence"(JEPA構想の原典)。提案: 入力と予測対象をそれぞれエンコーダで埋め込み、埋め込み空間で予測する(Joint Embedding Predictive Architecture)。動機: 生成(ピクセル復元)は不確実性に弱く非効率。新規性: 「生成しない自己教師」という設計思想を、自律的な機械知能への道筋として体系化した点。トレードオフ: 崩壊を防ぐ工夫(凍結ターゲット、stop-gradient=勾配停止、EMAなど)が必須で、設計が繊細。

この崩壊を防ぐ系譜には先行研究があります。Grill et al. (NeurIPS 2020) の BYOL(arXiv:2006.07733)は、負例(別のサンプルを「違うもの」として押し離す対照学習の要素)を使わずに、EMAで更新するターゲットネットワークと stop-gradient だけで崩壊を防げることを示し、JEPA系のターゲット設計の基礎になりました。SimSiam(Chen & He, 2021)はさらに EMA すら外し、predictor と stop-gradient だけで崩壊を防げることを示しました。一方 He et al. (CVPR 2022) の MAE(Masked Autoencoder, arXiv:2111.06377)は、画像の一部(典型的に75%)を隠してピクセルを復元する生成型で、JEPAが「特徴で予測する」のと対比される代表例です。Bardes et al. (ICLR 2022) の VICReg(arXiv:2105.04906)は、stop-gradient/EMA に頼らず「分散を一定以上に保つ項」と「次元間の共分散を抑える項」を明示的に損失に入れて崩壊を防ぐ別系統で、崩壊回避の選択肢を広げました。

Assran et al. (2023) "I-JEPA"(画像版JEPA、CVPR 2023, arXiv:2301.08243)。提案: 画像の一部を隠し、隠した領域の特徴を予測する。target encoder は予測側(online)のEMAで更新。動機: ピクセル復元を避けつつ強い表現を学ぶ。効果: ラベルなしで強い画像表現を、生成型より少ない計算で獲得。トレードオフ: マスクの作り方(大きめのブロックを複数隠す戦略)やEMA係数に敏感。

Bardes et al. (2024) "V-JEPA"(動画版、arXiv:2404.08471)。動画の時空間マスクを特徴空間で予測。「未来フレームの特徴を予測する」発想に最も近く、運転の未来予測の直接の先行例です(詳細は [V-JEPA (動画用JEPA表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0100_v-jepa_note))。

JEPA風特徴再構成損失は、これらの「特徴空間で予測ズレを測る」考えを、運転の未来予測タスクに当てはめたものです。MSEで素直に測り、凍結バックボーンでターゲットを作る点でJEPAの核を踏襲しています。一方、I-JEPA/V-JEPAのようなEMAターゲットや手の込んだマスク戦略までは導入しないシンプルな実装(凍結バックボーンによる未来フレーム特徴をターゲットにする)も実務ではよく採られます。EMAは質が高い反面、係数調整が繊細で実装も複雑なので、まず凍結で足場を作るのは妥当な選択です。研究の系譜で言えば「凍結ターゲット(最も単純)→ EMAターゲット(BYOL/JEPA)→ 明示正則化(VICReg)」という崩壊回避の選択肢があり、どれを採るかは安定性と実装コストのトレードオフです。

## 論文の実験結果(定量データ)

指標の意味から説明します。自己教師あり学習の質はよく「線形評価(linear probing)」で測ります。学習した特徴を凍結し、その上に単純な線形分類器だけを載せて下流タスク(例: ImageNet分類)の精度を測る方法で、特徴そのものの良さを反映します。精度が高いほど、特徴が有用な情報を素直に持っていることを意味します。線形分類器しか足さないので、もし特徴が良くなければ精度は上がらず、特徴の質を直接反映します。

I-JEPA(2023)の報告では、ImageNet の線形評価で、ピクセル復元型の MAE と競合する精度を、事前学習の計算量を大幅に削減して達成しました(同等精度に必要なGPU時間がMAEより大きく少ないと報告)。これは「ピクセルを描かない」ことの効率上の優位を定量的に示します。アブレーションでは、ターゲットをEMAで更新せず固定すると性能が落ちることが示され、ターゲット安定化の重要性が裏付けられました。

V-JEPA(2024)の報告では、動画の特徴空間予測で学んだ表現が、動作認識(Kinetics-400 や Something-Something-v2)などの下流タスクで、対照学習型やピクセル予測型を上回る、または匹敵する精度を達成しました。とくに凍結評価(frozen evaluation、特徴を固定したまま評価)で強く、学んだ特徴が追加学習なしで有用であることを示しています。とりわけ運動の理解が問われる Something-Something-v2 で生成型を上回った点は、「特徴予測のほうが動きの抽象を捉えやすい」ことの示唆として重要です。BYOL(2020)の報告は、負例を使わずEMAと stop-gradient だけで、対照学習に匹敵する ImageNet 線形評価精度(ResNet-50 で約74%、より大きなモデルで70%台後半)を達成し、「崩壊を防ぐにはEMAターゲットと勾配停止で十分」という重要な発見を定量的に確立しました。

これらの結果が共通して示すのは、「ピクセルを正確に復元しなくても、特徴空間で予測するだけで運転や認識に有用な表現が学べる」こと、そして「崩壊を防ぐターゲット設計(凍結またはEMA、あるいはVICReg型の正則化)が性能の鍵」ということです。MSEという素朴な損失でも、ターゲット設計さえ正しければ機能します。逆に言えば、損失の形(MSEか)よりも、何をターゲットにし、どうやって崩壊を防ぐかのほうが本質的だ、というのがこの分野の教訓です。

## メリット・トレードオフ・限界

メリットは四つです。第一に、ピクセル予測の「平均化ボケ」や「不要な細部に労力を取られる」問題を避けられます。第二に、凍結ターゲットで崩壊を防ぎつつ、ラベル不要の自己教師信号が得られます(未来フレームは時間が経てば自動的に手に入る)。第三に、時刻ごとの重み(temporal weights)で「近い未来を重視」など柔軟に調整でき、NaN防止の安全装置も入れられます。第四に、reduction="none" で時刻別の損失を観測でき、どの未来時刻が難しいか診断できます(遠い未来ほど損失が大きい、といった傾向を可視化できる)。

トレードオフと限界もあります。ターゲットの質が凍結バックボーン頼みで、バックボーンが運転に重要な情報を捨てていると予測ターゲットも貧弱になります。予測側とターゲット側の次元・空間解像度を厳密に合わせる必要があり、出力次元が一致しない場合は間に射影層(線形層で次元を合わせる)を足す配管が要ります。MSEは「平均的にずれを減らす」損失なので、稀だが重大な未来(急な飛び出し等)を軽視しがちです。そして自己教師損失は学習を補助するものであり、それ単体で運転がうまくなるわけではありません。模倣学習(軌跡を人間のお手本に合わせる損失)やRLとの組み合わせが前提で、複数の損失をどう重み付けするか(マルチタスク学習のバランス)が新たな調整対象になります。

研究上の未解決課題としては、運転に最適なターゲットエンコーダの選び方、不確実な未来を扱う確率的損失(分布間の距離や対数尤度で測る)への拡張、複数の自己教師信号と模倣損失の重み配分、そして特徴空間で測った損失の値が下流の運転性能とどれだけ相関するか(損失が下がっても運転がうまくならないことがあり得る)の検証、が挙げられます。

## 発展トピック・研究の最前線

発展方向の一つは、ターゲットをEMAで更新する I-JEPA/V-JEPA 流の高度化です。固定凍結より質の高いターゲットが得られますが、EMA係数とマスク戦略の調整が繊細になります。もう一つは確率的予測で、未来を1点ではなく分布として予測し、MSEではなく分布間の距離や対数尤度で測ることで平均化ボケを緩和する方向です。VICReg のように分散・共分散の正則化項を足して崩壊をより積極的に防ぐ路線、あるいはコサイン類似度ベースの損失(BYOL が採用)に置き換える路線もあります。さらに、この自己教師損失を世界モデルRL([World Modelシミュレーション学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0031_world-model-simulation))に組み込み、予測した未来から報酬を計算するループを閉じることが、実用化に向けた大きなテーマです。

研究者としての着眼点は、崩壊を防ぐ最小限の仕掛けは何か(凍結で足りるか、EMAやVICRegのような分散・共分散正則化が要るか)、そして自己教師で学んだ未来予測が実際に下流の運転性能を上げるかという因果関係の検証です。後者は特に重要で、自己教師の損失だけを見て満足せず、「その表現が本当に計画を良くしたか」をクローズドループ評価まで含めて確かめる姿勢が、研究者には求められます。
