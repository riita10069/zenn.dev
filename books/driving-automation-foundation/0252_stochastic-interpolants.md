# Stochastic Interpolants

## ひとことで言うと
生成モデルは「でたらめなノイズ」から「本物らしいデータ」へ変換する仕組みです。Stochastic Interpolants(確率的補間子)は、2つの分布(たとえばノイズの分布とデータの分布)の間をなめらかにつなぐ「橋」を、数式で自由に設計するための一般的な理論枠組みです。この枠組みを使うと、拡散モデル(スコアベース)も Flow Matching も、その橋の引き方の違いとして同じ言葉で説明でき、しかも生成時に「決定的に作るか・確率的に作るか」を後から連続的に選べます。

## 直感的な理解
川の両岸を想像してください。片方の岸が「ノイズの分布」、もう片方が「データの分布」です。生成とは、ノイズ岸からデータ岸へ渡ることに相当します。

橋の架け方には自由があります。まっすぐ最短距離で渡す橋(直線補間)もあれば、ゆるくカーブを描く橋(三角関数補間)もあります。さらに、橋を渡るときに「決められた一本道をまっすぐ歩く」(決定的=ODE)か、「ふらふらとランダムに揺れながら渡る」(確率的=SDE)かも選べます。

2020年代前半、生成モデルにはこの「橋の架け方」と「渡り方」が違う複数の流派が乱立していました。スコアベース拡散はランダムに揺れながら渡る方式、Flow Matching はまっすぐ歩く方式で、やっていることは近いのに別物として扱われていました。Stochastic Interpolants は「橋の形を α, β, γ という関数で指定する」「渡り方は同じ橋に対して決定的にも確率的にもできる」と整理し、すべてを一つの設計空間にまとめました。

## 基礎: 前提となる概念
理解に必要な概念を噛み砕きます。

分布(distribution)とは、データがどんな値をどんな頻度で取るかを表す確率の地図です。生成モデルは、簡単な分布(ガウスノイズ)から複雑な分布(実画像など)への写像を学びます。

ODE(常微分方程式)とは、量が時間とともにどう変化するかを表す方程式で、ランダムさを含みません。初期値が決まれば結果は一意に決まる(決定的)。

SDE(確率微分方程式)とは、ODE にランダムな揺らぎ(ブラウン運動)を加えた方程式です。同じ初期値でも結果が毎回ばらつく(確率的)。

速度場(velocity field)b(x,t) とは、各位置 x と時刻 t で「次にどちらへどれだけ動くか」を指すベクトルの地図です。これを ODE に従って積分すれば、点が時間とともに動いてノイズからデータへ運ばれます。

スコア(score)s(x,t) とは、その時刻の確率密度の対数の勾配 ∇ log p(x,t) で、「よりもっともらしいデータがある方向」を指すベクトルです。これは SDE による確率的生成で使われます。

連続方程式(continuity equation)とは、確率の「質量保存則」です。各時刻 t での分布 p(x,t) と速度場 b(x,t) は、∂p/∂t + ∇·(p b) = 0 という関係を満たさねばなりません。これは「分布が湧き出したり消えたりせず、速度場に沿って運ばれる」ことを保証します。Stochastic Interpolants の理論的な核は、補間子を決めると、この連続方程式を満たす速度場が一意に定まる点にあります。

## 仕組みを詳しく
中心となるのは「補間子(interpolant)」という関数です。時刻 t を 0 から 1 まで動かしながら、ノイズ x_0 とデータ x_1 を混ぜた中間状態 x_t を作ります。

```
x_t = α(t) * x_0 + β(t) * x_1 + γ(t) * z
```

- α(t): 始点 x_0(ノイズ)の重み。t=0 で1、t=1 で0になるよう選ぶ。
- β(t): 終点 x_1(データ)の重み。t=0 で0、t=1 で1になるよう選ぶ。
- γ(t): 追加の独立ノイズ z をどれだけ混ぜるかの重み。両端で0にする。
- z: 補間の途中だけ加える独立な標準正規ノイズ。これが「stochastic(確率的)」の由来。

ポイントは α・β・γ を自由に選べることです。

- α(t)=1-t、β(t)=t、γ(t)=0 にすると、ノイズとデータを直線でつなぐだけ。これは Flow Matching / Rectified Flow に相当します。
- α(t)=cos(πt/2)、β(t)=sin(πt/2) のような三角関数にすると、拡散モデルに近い橋になります。
- γ(t) を 0 でなくすると、途中にランダムさが入り、確率的な経路になります。

この補間子 x_t が決まると、理論から2つの学習対象が導けます。

1. 速度場 b(x,t): 連続方程式を満たすベクトル場。これを学習して ODE を解けば、ノイズからデータを決定的に生成できます。理論的には、b(x,t) は「x_t=x という条件のもとでの時間微分 ẋ_t の条件付き期待値」として書け、ニューラルネットワークで回帰できます。
2. スコア s(x,t): その時刻の確率密度の対数勾配。これを学習して SDE を解けば確率的に生成できます。

重要なのは、同じ補間子から速度場とスコアの両方が得られることです。これにより、生成時に決定的な ODE と確率的な SDE を自由に行き来できます。具体的には、生成用の SDE を

```
dx = [ b(x,t) + ε(t) * s(x,t) ] dt + sqrt(2 ε(t)) dW
```

と書くと、拡散係数 ε(t) を 0 にすれば純粋な ODE(決定的)、大きくすれば強い確率的経路になります。つまり ε(t) が「決定的⇔確率的」を連続的に回すダイヤルです。同じ学習結果から推論時に振る舞いを変えられるのが、この枠組みの実用的な価値です。

学習目標は単純です。補間子 x_t は閉じた式なので、各 t でランダムに x_0, x_1, z をサンプルして x_t と「真の速度 ẋ_t(時間微分、α'(t)x_0 + β'(t)x_1 + γ'(t)z)」を作り、ネットワークがそれを当てるよう二乗誤差で回帰します。tensor shape の例で言うと、データが `[B, D]`(バッチB、次元D)なら、x_0, x_1, x_t, 目標速度すべて `[B, D]` で、ネットワークは `(x_t, t) → [B, D]` を出力します。スコアも同様に z の条件付き期待値として別ヘッドで回帰するか、速度から導出できます。

この理論の数学的価値は、Flow Matching とスコアベース拡散が「補間子と、そこから決定的に作るか確率的に作るかの選択」という1つの設計空間の中の点として説明できると示したことです。さらに、補間子が両端で正しい境界条件(t=0でノイズ、t=1でデータ)を満たせば、生成された分布が厳密にデータ分布へ収束することが理論的に保証されます。

## 手法の系譜と主要論文
この枠組みは、生成モデルが「拡散」と「フロー」に分岐したあと、それを再統合する流れの中で生まれました。

- Song et al., "Score-Based Generative Modeling through SDEs", ICLR 2021 (arXiv:2011.13456)。拡散を連続時間 SDE で書き、対応する probability flow ODE も導いた。確率的・決定的生成の両立可能性を最初に示した。
- Chen et al., "Neural Ordinary Differential Equations", NeurIPS 2018。ODE をニューラルネットで定義する Continuous Normalizing Flow の基礎。速度場で生成するという発想の源流。
- Albergo & Vanden-Eijnden, "Building Normalizing Flows with Stochastic Interpolants", ICLR 2023 (arXiv:2209.15571)。補間子の基本アイデアを提示し、2分布をつなぐ確率過程から速度場を学べることを示した。
- Lipman, Chen, Ben-Hamu, Nickel, Le, "Flow Matching for Generative Modeling", ICLR 2023 (arXiv:2210.02747)。条件付き速度場を直接回帰する Flow Matching を提案。Stochastic Interpolants の特殊ケース(γ=0、適切な α,β)に相当。
- Liu, Gong, Liu, "Flow Straight and Fast: Rectified Flow", ICLR 2023 (arXiv:2209.03003)。ノイズとデータを直線でつなぎ、経路を反復的にまっすぐにして少ステップ生成を狙う。これも補間子の枠組みの一例。
- Albergo, Boffi & Vanden-Eijnden, "Stochastic Interpolants: A Unifying Framework for Flows and Diffusions", 2023 (arXiv:2303.08797)。本ノートの主役。補間子の一般化と、そこから速度場・スコアの両方を導く統一理論。決定的生成と確率的生成を連続的に切り替えられることを示した。
- Ma et al., "SiT: Exploring Flow and Diffusion-based Generative Models with Scalable Interpolant Transformers", ICCV 2023 (arXiv:2401.08740)。Stochastic Interpolants を Transformer(DiT 系)で大規模に実装し、補間子・予測対象・サンプラの選択が品質に与える影響を体系的に検証した実装研究。

## 論文の実験結果(定量データ)
Stochastic Interpolants は理論枠組みであるため、定量評価の多くは後続の実装研究(特に SiT)で行われました。

- SiT(Ma et al., 2023)は、画像生成の標準ベンチマークである ImageNet 256x256 で、同一アーキテクチャ(DiT、Diffusion Transformer)を拡散の定式化から Stochastic Interpolants ベースの定式化に置き換えるだけで FID(生成画像が本物とどれだけ近いかを測る指標。小さいほど良い)を改善したと報告しました。具体的には DiT-XL/2 の FID が拡散ベースで約9台だったのに対し、補間子ベースの SiT-XL/2 では約8台前半へ、さらに分類器なしガイダンスを併用して FID 2.06 まで到達したと報告されています。アーキテクチャを変えずに定式化(橋の引き方と渡り方)だけで品質が上がる点が、設計空間としての枠組みの有効性を示します。
- アブレーションとして、SiT は「予測対象(速度を直接予測 vs ノイズを予測 vs スコア)」「補間子(直線 vs 三角)」「サンプラ(ODE vs SDE)」を個別に切り替えて比較しました。報告では、直線補間+速度予測+適度な確率性を持つ SDE サンプラの組み合わせが安定して良く、決定的 ODE 単体よりも SDE を併用した方が FID が下がる傾向が示されました。これは「同じ学習結果に対し、推論時に確率性ダイヤル ε(t) を調整して品質を最適化できる」という Stochastic Interpolants の主張の実証です。
- 確率性の量と品質の関係も調べられ、ε(t) を 0(純 ODE)から増やしていくと、ある範囲までは FID が改善し、過剰にすると逆に悪化する、という凸的な傾向が報告されています。つまり「ほどほどの確率性」が最良で、これは EDM の確率的サンプラの知見とも整合します。

元の理論論文では、低次元の合成分布(混合ガウスなど)で、補間子から導いた速度場・スコアが理論どおりに正しい分布へ収束することを数値的に確認しています。

## メリット・トレードオフ・限界
- メリット: Flow とスコアベース拡散を1つの理論で説明でき、比較・設計がしやすい。
- メリット: 同じ学習結果から決定的(ODE)生成と確率的(SDE)生成を切り替えられる。推論時に品質と多様性のバランスを調整できる。
- メリット: 補間子 α, β, γ を自由に選べるため、データに合わせた橋の設計ができる。直線補間を選べば経路が短く少ステップ生成に向く。
- メリット: 補間子が閉じた式なので学習目標(速度の回帰)がシンプルで、シミュレーションフリー(生成過程を回さずに学習できる)。
- トレードオフ: 理論が抽象的で、最適な補間子やノイズ量 ε(t) はデータ依存で実験的に探す必要がある。
- トレードオフ: 自由度が高いぶん、初学者は「何を選べば良いか」の指針を別途持つ必要がある。
- トレードオフ: 確率的経路(γ≠0 または ε>0)はノイズ管理が増え、決定的経路より生成の制御が難しい場合がある。
- 限界: 直線補間で経路が交差すると速度場が平均化され曲がってしまい、結局多ステップを要する。これを直すのが Rectified Flow の反復直線化だが、追加の学習コストがかかる。
- 限界: 高次元・複雑な条件付け(テキストや画像条件)での最適補間子は未解明で、経験的探索に頼る部分が大きい。

## 発展トピック・研究の最前線
近年は、Flow Matching/Stochastic Interpolants が大規模画像・動画生成のデファクト定式化になりつつあります(Stable Diffusion 3 や多くの動画モデルが Rectified Flow 系を採用)。研究の最前線は、(1) 少ステップ・1ステップ生成のための一貫性蒸留やショートカットモデルとの統合、(2) 任意の2分布間(ノイズ→データだけでなくデータ→データ、画像翻訳など)を結ぶ補間子設計、(3) 条件付き生成での最適輸送(optimal transport)に基づく x_0 と x_1 のペアリングで経路を直線に近づける手法、(4) 確率性ダイヤル ε(t) を学習的・適応的に決める方法、などです。理論面では [EDM (Elucidating Diffusion Models)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0233_edm-framework) が整理した拡散側の設計空間と、Flow 側を含めたこの枠組みが収束しつつあり、生成モデルの設計が「確率過程の橋をどう架けてどう渡るか」という共通言語で語られるようになっています。
