# TDV (時間差による表現学習)

## ひとことで言うと
TDV（Temporal Difference for Visual representation、時間差による視覚表現学習）は、動画の連続するフレームの「差分（変化したところ）」を予測させることで、ラベルなしで画像から良い表現を学ぶ自己教師あり学習の手法です。「次フレームそのもの」ではなく「フレーム間のdelta（変化分）」に注目するのが特徴で、未来予測モデルで広く使われる「deltaを予測する」発想に論文的な後ろ盾を与えます。

## 直感的な理解
人間は赤ちゃんのころ、誰かに「これは猫」「これは机」とラベルを付けてもらわなくても、世界を見ているだけで物の概念を獲得します。重要な手がかりの1つが「動き」です。一緒に動く部分は同じ物体だ、動かない背景と動く物は別だ、といったことを、時間を通して観察するだけで学べます。

機械学習でも同じことをしたい、というのがこのトピックの出発点です。自動運転や画像認識では、本来「この画像には車が写っている」といった正解ラベルを人手で大量に付けて学習します。しかしラベル付けは高コストで、運転データは膨大です。そこで「正解ラベルなしで、データの構造そのものから学習信号を作れないか」というのが自己教師あり学習（self-supervised learning）の動機です。

動画はその素材として理想的です。「時間」という無料の構造があるからです。「フレームtを見て、フレームt+1を予測せよ」という問題は、誰もラベルを付けなくても自動で作れます。ところが、素朴に「次フレームのピクセルそのものを当てさせる」やり方には落とし穴があります。動画の隣り合うフレームはほとんど同じなので、「前フレームをそのままコピーする」だけで予測誤差をかなり小さくでき、ネットは何も意味のあることを学びません。これをショートカット学習（楽な抜け道に逃げること）と呼びます。TDVは、フレームそのものではなく「変化したところ」を予測対象にすることで、この抜け道をふさぎます。コピーしても差分は当たらないので、ネットは「何がどう動いたか」を捉えざるを得なくなります。

## 基礎: 前提となる概念
理解に必要な言葉を順に噛み砕きます。

- **エンコーダ（encoder）**: 画像を、扱いやすい特徴ベクトル（数百次元の数列）に圧縮するニューラルネットです。`z_t = Encoder(画像_t)` と書きます。
- **自己教師あり学習（self-supervised learning, SSL）**: 人間がラベルを付けず、データ自身から「正解」を作って学習する枠組みです。たとえば「画像の一部を隠して当てさせる」「2つの拡大表示が同じ画像由来か当てさせる」などがあります。学習後のエンコーダは、少量のラベルで下流タスク（物体検出など）にすぐ適応できる良い表現を持ちます。
- **対照学習（contrastive learning）**: 似たペア（同じ画像の別アングルなど）を近づけ、違うペア（別画像）を遠ざけるSSLの一系統です。大量の「違うペア（負例、negative）」が必要なのが難点でした。
- **表現崩壊（representation collapse）**: SSLでよく起きる失敗で、ネットが「全部同じベクトルを出す」という手抜き解に落ちることです。これでも一部の損失はゼロにできてしまうので、対策が要ります。
- **時間差分（temporal difference）**: ここでは2フレームの特徴の差 `Δz = z_{t+Δ} - z_t` を指します。「次の瞬間、表現がどう変わるか」を表します。

## 仕組みを詳しく
注意として、以下はTDVというアプローチの一般的な枠組みの説明です。考え方は次のステップです。

1. 動画から時刻 t と t+Δ の2フレームを取り出す。
2. それぞれをエンコーダに通して特徴 `z_t`、`z_{t+Δ}` を得る。各々たとえば256次元のベクトル、あるいは空間構造を保った `[C, H, W]` の特徴マップ。
3. ネットに「`z_t` から、時間差分 `Δz = z_{t+Δ} - z_t` を予測させる」。つまり「次の瞬間、表現がどう変わるか」を当てる課題にする。
4. 予測した `Δz_pred` と実際の `Δz` を比べて損失（たとえば二乗誤差）を計算し、エンコーダを更新する。

ここで重要なのは、「次フレーム `z_{t+Δ}` を直接当てる」のと「差分 `Δz` を当てる」のは、数学的には情報量が同じに見えても、学習のしやすさが全く違うという点です。前者は「ほとんど変わらない大きなベクトル」を再現する課題で、コピーへ逃げやすい。後者は「変わった小さい部分」を当てる課題で、変化＝運動に注意が向きます。これは「大きくて予測可能な部分」と「小さくて重要な部分」を分ける発想（[ego起因の見かけ運動と動的運動の分離](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0023_ego-induced-vs-dynamic-motion)）と同じ精神です。

数値で感覚をつかみます。隣接2フレームの特徴がほぼ同じで、各次元の値が0付近で動いているとします。`z_{t+Δ}` の典型的な大きさ（ノルム）が10だとすると、`Δz` の大きさは0.5程度かもしれません。前者を当てる課題では、ネットが「ほぼz_tをコピー」すれば誤差0.5で済み、改善の動機が弱い。後者を当てる課題では、その0.5の中身（どの次元がどちらへ動いたか）を当てねばならず、運動の構造を学ぶ強い圧力がかかります。

ただし注意点があります。生の動画では「変化分」にカメラ（撮影者・自車）の動きによる見かけの変化が混ざります。たとえば自車が前進しただけで、止まっている物の特徴も画面上で移動し、差分として現れます。これを分離しないと、「動いた物」を学んでいるつもりで「自分が動いただけ」を学んでしまいます。そこで、ego由来の変化を先に幾何計算で取り除き（[ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp)）、残った「本物のダイナミクスの差分」だけをdeltaとして予測する、という前処理を組み合わせるのが、TDVの発想を自動運転へ持ち込むときの定石です。

## 手法の系譜と主要論文
時間差分・予測ベースの自己教師ありの系譜を時系列で描きます。

- **van den Oord et al., "Representation Learning with Contrastive Predictive Coding (CPC)", 2018**: 系列データの未来を潜在空間で予測する対照学習を提案しました。「未来を予測する課題から良い表現が生まれる」という路線の起点の1つです。負例を使う対照損失だったため、負例設計に依存する弱点がありました。

- **Grill et al., "Bootstrap Your Own Latent (BYOL)", NeurIPS 2020**: 片方のネット（オンライン）が、もう片方（ターゲット、ゆっくり更新されるEMA = 指数移動平均のコピー）の出力を予測します。負例を一切使わずに表現崩壊を避けられることを示し、SSLに衝撃を与えました。「なぜ崩壊しないのか」の理解が難しく、EMAや予測器（predictor）の存在が効くことが後の研究で分析されました。

- **Feichtenhofer et al. / 動画SSLの諸研究（2021前後）**: 動画でフレーム順序の当て、再生速度の予測、未来の特徴予測など、時間構造を使うSSLが多数提案されました。共通テーマは「時間こそ無料の教師信号」です。

- **LeCun, "A Path Towards Autonomous Machine Intelligence (JEPA)", 2022**: ピクセルではなく潜在空間で未来を予測すべき、という思想を整理しました。TDVの「差分を潜在で予測」と相性が良く、近年の世界モデル設計の土台になっています。

- **Assran et al., "I-JEPA: Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023**: JEPAの思想を画像で具体化し、潜在空間でマスクされた領域を予測することで、ピクセル再構成より効率的に強い表現が学べることを実証しました。

- **Daithankar et al., "You Don't Need Strong Assumptions: Visual Representation Learning via Temporal Differences (TDV)", arXiv:2606.15956**: 本トピックの主役論文です。時間差分を予測対象にすることで、強い前提や複雑な設計なしに良い視覚表現が学べると主張します。タイトルの「強い仮定はいらない」は、対照学習の負例設計やBYOLの慎重なEMA調整のような重い前提を置かなくても、時間差分という素直な信号だけで十分だ、という主張を表します。差分にカメラ運動・ノイズが混ざる点が課題として残り、運動補償や前処理が要る場面があります。（注: この情報は2026-06-30時点。arXiv番号の形式から比較的新しい論文であり、細部の数値は原典を参照してください。）

系譜をまとめると、CPCが「未来予測で表現を学ぶ」路線を開き、BYOLが「負例なしでも崩壊を避けられる」ことを示し、JEPA/I-JEPAが「潜在空間で予測」を思想・実装として固め、TDVが「予測対象を差分にすれば前提が軽くなる」と主張する、という流れです。

## 論文の実験結果（定量データ）
表現学習の良し悪しは、通常「線形評価（linear probe）」で測ります。これは、SSLで学んだエンコーダを凍結（重みを固定）したうえで、その上に線形分類器だけを1層載せて分類精度を測る方法です。エンコーダ自体は分類用に微調整しないので、「表現がどれだけ素のままで使えるか」を測れます。

BYOL（Grill et al., 2020）は ImageNet（1000クラス・約128万枚の画像分類ベンチマーク）の線形評価でTop-1精度74.3%を達成しました。Top-1精度とは「最も確信したクラスが正解だった割合」です。当時の対照学習の代表SimCLRが約69.3%だったのに対し、負例を一切使わずにそれを上回った点が画期的でした。アブレーションでは、予測器（predictor）を外す、あるいはターゲット側のEMA更新を止めて即座に重みをコピーすると、表現が崩壊して精度が大きく落ちることが示されています。これは「崩壊回避には予測器とゆっくりしたターゲット更新が効く」ことの定量的裏付けです。

I-JEPA（Assran et al., 2023）は、ピクセル再構成型のSSL（MAE など）と比べて、同等以下の計算量でImageNetの線形評価精度を上回ることを報告しました。要点は「ピクセルを細部まで再構成するより、潜在空間で予測するほうが、意味的に役立つ表現を効率よく学べる」という結果です。これはTDVが「差分（＝潜在の変化）を予測する」路線を選ぶ理由と一致します。

これらの数値が示す共通のメッセージは、「再構成よりも予測、ピクセルよりも潜在、そして適切な崩壊回避策」が表現の質を押し上げる、ということです。TDVはこの流れの中で「予測対象を時間差分にすれば、余計な前提なしに同じ恩恵が得られる」と位置づけられます。差分を予測することは、コピーへの逃げ道をふさぐ意味で、暗黙の崩壊回避にもなっています。

## メリット・トレードオフ・限界
メリット
- ラベル不要で、動画の時間構造から自動的に学習信号を作れます。
- 「差分を当てる」ことで、ネットが運動・ダイナミクスに敏感な表現を学びます。
- 次フレーム丸ごと再構成のショートカット（コピーへ逃げる）を避けやすく、軽い前提で済みます。

トレードオフ・限界
- 差分にはカメラ・自車運動による見かけの変化が混ざります。分離しないと「自分が動いただけ」を学んでしまうため、ego運動の補償（[ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp)）と組み合わせるのが定石です。
- 隣接フレームの差分は小さく、センサノイズに埋もれやすい。サンプリング間隔（何秒先のフレームと比べるか）やノイズ処理の設計が結果を左右します。間隔が短すぎると差分がノイズだらけ、長すぎると変化が大きすぎて予測が難しくなります。
- 表現崩壊（全部 `Δz=0` に潰れる等）のリスクは残ります。BYOL/JEPA的な崩壊回避の工夫（予測器、ストップグラディエント、EMAターゲット、あるいはワープによる非ゼロの構造的正解）が要ります。

未解決の課題として、最適なサンプリング間隔の自動決定、長時間の差分（複数ステップ先）をどう扱うか、差分予測で学んだ表現が下流の制御・計画タスクへどこまで転移するか、などが研究対象です。

## 発展トピック・研究の最前線
TDVの考え方は、世界モデルの潜在予測と密接に結びついています。近年は、動画をトークン（離散シンボル）化して自己回帰的に未来を生成する系（ビデオ生成系の世界モデル）や、潜在拡散で未来を生成する系が台頭しており、いずれも「差分・変化をどう予測するか」が核心です。自動運転では、egoアクションを条件として与えたうえで残りのダイナミクスを潜在で予測する設計が標準化しつつあり、TDVのような「軽い前提の差分予測」はその前段の表現学習として有望視されています。研究の最前線では、差分予測と崩壊回避の理論的理解、マルチモーダル（カメラ＋LiDAR）への拡張、長時間ホライズンでの誤差累積の抑制が議論されています。
