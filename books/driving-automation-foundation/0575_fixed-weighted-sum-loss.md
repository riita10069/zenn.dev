# 固定重み和損失 (Weighted Sum)

## ひとことで言うと

1つのモデルに複数のことを同時に学ばせたいとき、それぞれの「間違い度合い(損失)」を、人が決めた固定の係数でかけ算してから足し合わせるだけ、という最もシンプルな統合方法です。「メインの仕事を 1.0、おまけの仕事を 0.1」のように手で重みを決めて全体を1つのスカラー値にまとめ、それを下げるように学習します。英語では unitary scalarization(単一のスカラー化)や linear scalarization(線形スカラー化)とも呼ばれ、マルチタスク学習のあらゆる手法が比較の出発点(ベースライン)に置く、分野の標準的な初期設定です。

## 直感的な理解

料理にたとえると、いくつもの材料(各タスクの損失)を1つの鍋(合計損失)に入れて煮込むようなものです。塩を入れすぎれば塩味だけが立ち、他の味が消えます。固定重み和では、各材料の量を人が「塩は少なめ、出汁は多め」と最初に決め打ちします。シンプルで、味の調整が直感的に分かるのが利点です。

なぜわざわざ複数の損失を足すのでしょうか。多くの応用では「本命の予測目標」が1つあり、それだけを学ばせると、モデルが表面的なパターンを真似るだけで、深い理解(物体の配置、シーンの幾何、因果関係など)が浅いまま正解を出してしまうことがあります。たとえば自動運転で「人間と同じ運転軌跡を出力する」ことだけを学ばせると、見た目の動きは真似られても「なぜその行動が安全なのか」を理解しないまま正解に当たってしまう、という事態が起こり得ます。そこで「本命ではないが、学ばせると本命の性能も上がるおまけの学習目標」=補助タスク(auxiliary task)を足します。物体検出、深度推定、未来の状態予測、セグメンテーションなどが典型例です。これらを少しだけ混ぜると、モデルは本命にも補助にも役立つ共通の特徴を学ぶよう促されます。

問題は「複数の損失をどう1つにまとめるか」です。勾配降下で最小化する対象は最終的に1つの数値である必要があるので、合体は避けられません。最も単純で確実な合体が、固定係数をかけて足す固定重み和です。

## 基礎: 前提となる概念

損失(loss)とは、モデルの予測が正解からどれだけずれているかを表す数値で、小さいほど良い予測です。学習はこの損失を勾配降下(損失の勾配と逆向きに少しずつ重みを動かす)で下げていく作業です。

マルチタスク学習(multi-task learning、MTL)では、共有エンコーダ(複数タスクで共用する特徴抽出器、shared backbone)と、タスクごとの出力ヘッド(task-specific head)を持つモデルを、複数の損失で同時に学びます。各損失を1スカラーにまとめる必要があり、その最も単純な方法が重み付き和です。

正則化(regularization)とは、モデルが学習データに過度に適合(過学習、overfitting)するのを防ぎ、未知のデータでも通用する一般的な表現を学ばせる工夫の総称です。補助タスクを少しだけ足すことには、この正則化の効果があります。直感的には、複数の目標を同時に満たさねばならないという制約が、1つの目標だけに都合のよい(=過剰適合した)解を選ばせない圧力として働く、と理解できます。

なぜ係数を「固定」にするのか。係数を学習中に動かす凝った手法(動的重み平均 [Dynamic Weight Averaging (DWA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0588_dynamic-weight-averaging)、勾配ノルムを揃える [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm)、不確実性を学習する [不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall)、勾配の向きを直す [PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))もありますが、それらは挙動が読みにくく、学習が不安定になったときに原因の切り分けが難しくなります。新しい問題に取り組む初期は「まず確実に動く・挙動が予測できる」ことが最優先なので、固定係数で始めるのが定石です。

## 仕組みを詳しく

基本の式は次の通りです。主タスクの損失を L_main、補助タスクの損失を L_aux1, L_aux2... とすると、

```
L_total = L_main + λ1 · L_aux1 + λ2 · L_aux2 + ...
```

λ(ラムダ)が人が手で決める固定係数です。主タスクの係数は普通 1.0 に固定し、補助タスクには小さい値(0.1 や 0.01)を割り当てます。すべてのタスクが等しく重要な場合は全係数を 1.0 にする「等重み(equal weighting / unitary scalarization)」が特別なケースになります。

数値例で見ます。主タスク(たとえば軌跡予測)の損失が L_main = 2.0、補助タスク(たとえば物体検出)の損失が L_aux = 8.0、係数 λ = 0.1 とすると、

```
L_total = 2.0 + 0.1 × 8.0 = 2.0 + 0.8 = 2.8
```

ここに重要なポイントがあります。もし λ = 1.0 にすると L_total = 2.0 + 8.0 = 10.0 となり、補助タスクが損失の大きさで支配的になります。すると勾配の大きさも補助タスク由来の成分が大きくなり、学習は補助タスクを下げることばかりに引っ張られ、本来一番大事な主タスクがおろそかになります。補助タスクはあくまで本命の手助けなので、低い係数(0.1 程度)に抑えるのが定石です。

なぜ低係数が効くのか。前述の通り、補助損失を小さく足すと正則化として働きます。モデルは「主タスクだけでなく補助タスクにも役立つ共通特徴」を学ぶよう促され、結果として主タスクの汎化性能(未知データへの強さ)が上がります。係数が大きすぎると補助タスクが主役を食い、小さすぎると効果が消えるので、適度な小ささが重要です。

スケールの違いに注意が必要です。固定重み和の最大の難所は、損失ごとに値の桁(スケール)が違うことです。たとえば回帰損失(連続値の予測誤差、平均二乗誤差など)が 2.0 付近、分類損失(クロスエントロピー)が 0.001 付近だと、足してもほぼ回帰損失しか効きません。だから係数 λ は「タスクの重要度」だけでなく「スケールを揃える役目」も兼ねています。実務では学習初期に各損失の値の桁を観察し、桁が揃うように λ を調整するのが基本です。

```
L_main (スケール~2.0, 係数1.0)
   +
L_aux1 (スケール~8.0, 係数λ1=0.1) → 寄与 0.8
   +
L_aux2 (スケール~0.001, 係数λ2=100?) → スケール補正が必要
   ↓
L_total （1つのスカラー）→ 勾配降下で最小化
```

この「桁合わせ × 重要度」を人手でやるのが固定重み和であり、それを自動化したいという欲求が動的・不確実性ベースの手法を生みました。なお、勾配の観点で見ると、合計損失の勾配は各タスク勾配の係数付き線形和 ∇L_total = ∇L_main + Σ λ_k ∇L_aux_k になります。つまり固定重み和は「タスク勾配を係数で重み付けして足す」ことと等価で、後述の勾配操作系手法(PCGrad など)はこの足し算そのものに介入して向きを整える点で異なります。

## 手法の系譜と主要論文

固定重み和そのものは手法というより、ほぼ全てのマルチタスク論文が比較対象に使う分野標準の初期設定です。重要なのは「固定だと不便だから自動化したい」という流れを生んだ点なので、その動機を語った代表論文を系譜として挙げます。

最初の転機は Alex Kendall, Yarin Gal, Roberto Cipolla の "Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics"(CVPR 2018, arXiv:1705.07115)です。著者らは Cambridge 大学に在籍していました。固定重み和の最終性能が係数 λ の手動チューニングに極端に敏感で、係数をグリッドサーチ(総当たりで試す)するのは時間も計算資源も莫大だと問題提起しました。そして各タスクの観測ノイズ(homoscedastic uncertainty、入力に依存しないタスク固有の不確かさ)を学習可能パラメータとして導入し、それで重みを自動決定する手法を提案しました。回帰タスクと分類タスクを統一的に扱う重み付け式を確率モデル(ガウス尤度・ソフトマックス尤度の最尤推定)から理論的に導いた点が新規性です。

同時期に Zhao Chen らの GradNorm(ICML 2018, [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))が、損失ではなく勾配ノルムを揃える方向から自動化に挑みました。翌年、Shikun Liu, Edward Johns, Andrew Davison の "End-to-End Multi-Task Learning with Attention"(CVPR 2019, arXiv:1803.10704)が、固定重み和をベースラインとしつつ、損失の下降率から重みを動的に決める動的重み平均([Dynamic Weight Averaging (DWA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0588_dynamic-weight-averaging))を提案しました。彼らは固定重みの限界を「単純だが最適値を探すのが面倒で、学習が進むと最適な重みが変わってしまう」と明確に指摘しています。

さらに後、勾配の向きの競合に着目する PCGrad(Yu et al. NeurIPS 2020, [PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))や、最適化として定式化する CAGrad(Liu et al. NeurIPS 2021)、Nash-MTL(Navon et al. ICML 2022)へと発展しました。これらに共通する出発点は「固定重み和はシンプルで強力なベースラインだが、係数探索のコストと、学習進行で最適重みが変わる問題がある」という認識です。

ところがその後、揺り戻しが起きます。Vitaly Kurin らの "In Defense of the Unitary Scalarization for Deep Multi-Task Learning"(NeurIPS 2022, arXiv:2201.04122)や Derrick Xin らの "Do Current Multi-Task Optimization Methods in Deep Learning Even Help?"(NeurIPS 2022)は、複数の高度な手法を共通条件で再評価し、「正則化と学習率さえ適切なら、単純な固定重み和が高度な手法に匹敵する」と報告しました。固定重み和は系譜の起点でありながら、いまも終点(最強ベースライン)であり続けているのです。

## 論文の実験結果(定量データ)

固定重み和に関して論文が示す最も重要な定量的事実は「最終性能が係数 λ に強く依存する」ことです。

Kendall らは、深度推定・セマンティックセグメンテーション・インスタンスセグメンテーションを同時に学ぶ実験(CityScapes など屋外運転シーン、各ピクセルにラベルを付ける密予測タスク群)で、固定重みの組み合わせを変えると最終性能が大きく振れることを示しました。報告によると、3タスクの相対重みを連続的に変化させたときの性能は、適切な比率の付近でのみ良くなる凸的な形(山なり)を描き、外れた比率では特定タスクの性能が大きく劣化しました。これは「人手チューニングの当たり外れが大きい」ことの直接的な証拠です。そのうえで、提案の不確実性重み付けは手動チューニングなしで、固定重み和の手探りで見つけた最良値に匹敵かそれ以上の性能(報告ではセグメンテーション IoU と深度誤差の両方で改善)を達成したとされています。評価指標は深度なら誤差(小さいほど良い)、セグメンテーションなら mIoU(平均 IoU、予測領域と正解領域の重なり率、大きいほど良い)です。

Liu らも NYUv2(屋内 RGB-D の3タスク:セグメンテーション・深度・法線)で、固定の等重みをベースラインに置き、動的重み平均がそれを上回ることを示しました。全タスクの相対性能をまとめた指標 Δ_MTL(各タスクのベースライン比を平均した値)で、等重みは中位、動的重み平均と不確実性重み付けがそれを上回る、という序列が報告されています。これも「固定はそこそこ強いが、自動調整に劣る場面がある」ことを示します。

アブレーション的な知見として広く知られるのは、補助タスクの係数を 0 から徐々に上げると、ある範囲では主タスク性能が向上(正則化効果)し、上げすぎると主タスクが補助に食われて劣化する、という逆 U 字の関係です。この「適度な低係数が最良」というカーブは多くのマルチタスク・補助タスク研究で繰り返し観察されており、固定重み和を使ううえでの実践的な指針になっています。

一方で重要な反証もあります。Kurin らと Xin らの再現研究(NeurIPS 2022)は、適切な正則化(weight decay、Dropout)と丁寧な学習率調整を施した固定重み和(あるいは等重み)が、Multi-MNIST、CityScapes、NYUv2、CelebA、Meta-World などのベンチマークで GradNorm・PCGrad・MGDA・Uncertainty Weighting といった高度な手法と統計的に同等か、むしろ上回る場面が多いと報告しました。Kurin らは特に、高度な手法の見かけの優位の一部は「ベースラインの正則化不足」に由来すると指摘しています。つまり「固定重み和は古いから弱い」のではなく、係数と正則化さえ妥当なら依然として非常に強いベースラインだ、というのが現在の理解です。

## メリット・トレードオフ・限界

メリットは次の通りです。実装が最も単純(足し算とかけ算だけ)で、バグが入りにくいこと。挙動が完全に予測可能で、係数を上げ下げした効果が直感的に分かること。追加の学習パラメータと追加の逆伝播がゼロなので学習が安定し、計算コストが最小で、デバッグの土台に最適なこと。十分にチューニングすれば高度な手法に肉薄する(あるいは上回る)強さがあること。

トレードオフと限界もあります。係数 λ を人手で探す必要があり、タスク数が増えると組み合わせ爆発(N タスクなら N 次元のグリッド)で探索が大変になること。学習が進むと本来かけるべき最適な重みが変化するのに、固定なので追従できないこと(動的重み平均が解こうとする問題)。損失ごとのスケール差を係数で吸収しないと、一部の損失に学習が支配されること。タスク間で勾配が衝突している(片方を下げると片方が上がる、負の転移)場合、固定重み和では成分が打ち消し合って解決できず、勾配射影など別系統の手法が必要なこと。

未解決の実務課題は、係数探索の効率化(少数の試行で良い比率を見つけるベイズ最適化など)、スケール自動正規化との併用、そして「いつ固定で十分で、いつ自動手法に移るべきか」の判断基準です。一般的な進め方としては、まず固定重み和で確実に動く土台を作り、損失のスケール差や学習速度の偏り、負の転移といった具体的な症状が観測されてから、その症状に合った自動手法([Dynamic Weight Averaging (DWA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0588_dynamic-weight-averaging) / [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm) / [不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall) / [PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))へ段階的に移行するのが堅実です。補助タスクを「低係数で少しだけ足す」設計は、まさにこの堅実路線を体現したもので、リスクを最小化しつつ正則化の恩恵を取りに行く実用的な選択です。

## 発展トピック・研究の最前線

固定重み和の発展は「人手の係数決めをどう自動化・効率化するか」に集約されます。第一の方向は本稿で触れた自動重み付け(不確実性・勾配ノルム・損失動態・勾配幾何)です。第二の方向はメタ学習で、Auto-Lambda(Liu et al. 2022, arXiv:2202.03091)のように検証性能を最大化するよう重み付け方策そのものを上位最適化(bilevel optimization)で学習する試みがあります。第三の方向はスケール正規化の標準化で、損失を学習初期の値で割って桁を自動的に揃える前処理(loss normalization)を入れると、固定重み和でも係数探索が大幅に楽になることが経験的に知られています。

研究コミュニティの現在の到達点は、やや逆説的です。高度な手法が次々登場した一方で、複数の厳密な再現研究が「適切にチューニングした単純な固定重み和は依然として最強クラスのベースライン」と結論づけました。したがって実務でも研究でも、固定重み和は「捨てる対象」ではなく「最初に必ず立てるべき、そして最後まで比較対象であり続ける」基準線として扱うのが正しい姿勢です。新手法を提案するときは、十分にチューニングした固定重み和に本当に勝てているかを問うことが、健全な評価の前提になっています。これはマルチタスク最適化に限らず、機械学習全般で「強いベースラインを正しく立てる」ことの重要性を象徴する事例として語られます。
