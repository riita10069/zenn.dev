# HorNet (gnConv / 再帰的ゲート畳み込み)

## ひとことで言うと
HorNet は、画像から特徴を取り出す土台ネットワーク(バックボーン)の一種です。Transformer の自己注意が暗黙に行っている「特徴どうしの掛け合わせ(高次の空間相互作用)」を、畳み込みと要素ごとの掛け算(ゲート)を再帰的に繰り返すだけで効率よく再現する、gnConv (recursive gated convolution) という演算を提案しました。注意機構を一切使わずに、同規模の Transformer 型・CNN 型バックボーンと同等以上の精度を、計算量を画像サイズに対してほぼ線形に保ったまま達成した点が新しさです。

## 直感的な理解
ある画像認識タスクを考えます。「車のタイヤ」という特徴が画面のある位置にあり、「車体」という特徴が別の位置にあるとき、この2つが同時に強いときだけ「ここに車がある」と確信したい、という状況がよくあります。これは「特徴Aと特徴Bが両方強いときだけ反応する」という関係で、足し算では表現できません。足し算は「AかBのどちらかが強ければ反応する」しか作れないからです。両方が強いときだけ反応させるには、AとBを掛け算する必要があります。掛け算は片方がゼロに近ければ出力もゼロに近づくので、「両方そろったときだけ通す門(ゲート)」として働きます。

Transformer の自己注意がなぜ強いのか、という問いに対し、HorNet の著者たちは「自己注意は本質的に特徴どうしの掛け算(高次の相互作用)を行っているからだ」という仮説を立てました。query と key を掛けて関連度を出し、その関連度を value にさらに掛ける、という二重の掛け算がその正体です。それなら、自己注意という重い仕組みを丸ごと持ち込まなくても、掛け算という核心だけを安い畳み込みの世界で組み立て直せば、同じ強さが得られるはずだ、というのが gnConv の発想です。台所道具にたとえるなら、高機能だが高価な調理ロボット(自己注意)の機能のうち、実際に効いている「材料を混ぜて反応させる」工程だけを、安いハンドミキサー(畳み込み+ゲート)で再現したようなものです。

ここで「次数(order)」という言葉が重要になります。普通の畳み込みや全結合は、入力特徴を重み付きで足すだけなので、入力に対して1次(線形)の演算です。自己注意は query・key・value という3つの量が掛かるので、入力に対して2次以上の非線形性(高次)を持ちます。この「掛け算の重なり具合=次数」こそが表現力の源だ、というのが HorNet の中心仮説です。gnConv はこの次数を、畳み込みとゲートだけで任意に上げられる仕組みを与えます。

## 基礎: 前提となる概念
gnConv を理解するために、いくつかの用語を順に噛み砕きます。

「畳み込み(convolution)」とは、小さな重み行列(カーネル、たとえば 3×3)を画像の上で滑らせ、各位置で近傍の画素値とカーネルの要素ごとの積の和を取る操作です。近くの画素どうしの関係(局所的な空間情報)を集めるのが得意です。「受容野(receptive field)」とは、出力の1点が入力のどれだけの範囲を見ているかの広さです。3×3 を1層だと 3×3 しか見えませんが、層を重ねると受容野は広がります。逆に言えば、CNN が広域を見るには深く積む必要があり、これが「CNN は大域文脈が弱い」と言われる理由です。

「深さ方向畳み込み(depth-wise convolution)」とは、チャンネル(色や特徴の次元)ごとに独立な畳み込みを行うものです。通常の畳み込みは全チャンネルを混ぜますが、深さ方向畳み込みは各チャンネルを別々に処理するので計算が軽く、空間方向の情報集約だけを安価に行えます。大カーネル(7×7 など)を使っても計算量が抑えられるのが利点で、ConvNeXt 以降の効率 CNN の定番部品です。

「空間相互作用(spatial interaction)」とは、画像のある位置の特徴が、別の位置の特徴とどう影響し合うかという関係のことです。先述の「次数」は、その相互作用に何個の特徴が掛け合わされているかを表します。

「ゲート(gating)」とは、2つの信号を要素ごとに掛け算することです。LSTM の入力ゲート・忘却ゲートや、GLU (Gated Linear Unit) で古くから使われてきた、情報の選別機構です。GLU は y = (W1 x) ⊙ σ(W2 x) のように、片方の枝でもう片方を「どれだけ通すか」制御します。gnConv はこのゲートを再帰的に重ねて次数を上げる点が新しいわけです。

なぜ自己注意の高次相互作用を畳み込みで置き換えたいのか。理由は計算量です。自己注意は画像内の全位置ペアの関連度を計算するため、トークン数(画素数)を N とすると計算量が O(N^2) になります。高解像度画像(たとえば自動運転の前方カメラのような大きな画像)では N が膨大になり、二乗の重さが致命的になります。具体例として、解像度を縦横2倍にすると N は4倍、自己注意の計算は16倍に膨らみます。畳み込みは N に対して線形 O(N) なので、高次の表現力を保ったまま線形計算量にできれば理想です。

## 仕組みを詳しく
gnConv (g は gated、n は order の次数) の処理を段階的に追います。入力特徴を x、形状を [C, H, W](チャンネル C、縦 H、横 W)とします。n 次(n-order)の gnConv を例にします。

ステップ1: 入力射影
1×1 畳み込み(全画素に同じ線形変換をかける操作。空間方向には混ぜない)で、入力を複数のグループに分割します。n 次の場合、ゲート用の系列 p_0, p_1, ..., p_{n-1} と、最初に掛け合わせる基底 q_0 を作ります。チャンネル数は段階的に倍増する形(C_0, C_1, ..., C_{n-1} を C/2^{n-1}, C/2^{n-2}, ..., C のように)に割り当てられます。低次の段ほど少ないチャンネルで処理されるので無駄な計算を抑えます。総チャンネルが Σ C_k = C(1 + 1/2 + 1/4 + ...) < 2C に収まるため、段数を増やしてもチャンネル予算がほぼ一定に保たれるのがミソです。

ステップ2: 深さ方向畳み込みによる空間混合
基底を深さ方向畳み込み(大きなカーネル、たとえば 7×7)で空間混合します。これで「近傍の空間情報」が取り込まれます。あるいは GFNet 由来のグローバルフィルタ(周波数領域で広い受容野を得る仕組み)を使えば、画像全体に及ぶ広域の空間情報を線形コストで取り込めます。これが「広い範囲を見る」という自己注意の効果を代替する部分です。掛け算の前に必ず空間混合を挟むことが、自己注意の「離れた位置の値を掛け合わせる」性質を再現する鍵です。

ステップ3: 再帰的ゲーティング
ここが核心です。漸化式で書くと次のようになります。

  p_{k+1} = f_k( g_k ) ⊙ p_k    (k = 0, 1, ..., n-1)

ここで g_k は空間混合済みの基底、⊙ は要素ごとの掛け算(ゲート)、f_k は段ごとの 1×1 畳み込み(チャンネルを次段の次元に合わせる線形変換)です。1段ごとにゲート(掛け算)が1回入るので、相互作用の次数が1ずつ上がります。p_0 から始めて n 回ゲートを掛けると、入力特徴が n+1 個掛け合わさった (n+1) 次の相互作用に到達します。

数値例で次数の上がり方を見ます。
- n=1(1段): p_1 = f_0(g_0) ⊙ p_0。特徴2個の積 → 2次相互作用。これは GLU とほぼ同じです。
- n=2(2段): さらに f_1(g_1) を掛ける → 特徴3個の積 → 3次相互作用。
- n=3(3段): 4個の積 → 4次相互作用。

つまり段数 n を1増やすだけで次数が1上がり、しかも各段のチャンネルを半分ずつにしているため、総計算量は段数を増やしてもほぼ一定(線形)に収まります。自己注意は固定で2次相互作用しか作れないのに対し、gnConv は次数を自由に上げられる点で、むしろ自己注意の一般化とも言えます。論文は「自己注意は gnConv の n=2 の特殊例とみなせる」という整理を与えています。

ステップ4: 出力射影
最終段 p_n を 1×1 畳み込みでまとめ、出力します。

ASCII 図で1ブロックの流れを示します。

  x ─[1x1 proj]─┬─ q0 ─[7x7 dwconv]─┐
                ├─ p0 ───────────────⊙─ p1 ─[f1]─⊙─ p2 ─ ... ─[1x1 out]─→
                ├─ (g1) ──────────────┘            ↑
                └─ (g2) ───────────────────────────┘

このブロックを Swin / ConvNeXt と同じ4ステージ階層構造(解像度を 1/4 → 1/8 → 1/16 → 1/32 と段階的に下げ、チャンネルを増やす)に積み上げたのが HorNet バックボーンです。各ステージの初め(高解像度のステージ)は局所的な大カーネル深さ方向畳み込み版 gnConv、後段(低解像度のステージ)はグローバルフィルタ版 gnConv を使い分け、局所と広域の両方を捉えます。Tiny / Small / Base / Large の4サイズが用意され、ConvNeXt や Swin とパラメータ数・FLOPs をそろえて比較できるよう設計されています。

なぜこの設計なのか。要点は「掛け算の前に必ず空間混合(畳み込み)を挟む」ことで、自己注意が持つ「広い文脈を掛け合わせる」性質を、二乗計算量を払わずに再現している点です。各段でチャンネルを半減させるのは、高次の段ほどチャンネルを減らして計算量の爆発を防ぐためで、これにより次数を上げても線形コストが保たれます。

## 手法の系譜と主要論文
gnConv は突然生まれたわけではなく、いくつかの研究の流れの合流点にあります。

第一の流れは「自己注意の本質は何か」という分析です。Vaswani et al. の Transformer (NeurIPS 2017) 以降、自己注意の強さの源泉を問う研究が続きました。Hua et al. の Gated Attention Unit / FLASH (ICML 2022) は、ゲート機構と注意を組み合わせ、線形計算量で注意に迫る表現力を出しました。HorNet はこの「ゲート=掛け算による高次相互作用」という見方を、純粋に畳み込みの枠内へ持ち込みました。

第二の流れは「CNN による Transformer の再評価」です。Liu et al. の ConvNeXt (CVPR 2022) は、Swin Transformer の設計上の工夫(大カーネル、LayerNorm、GELU、stage 比率の調整など)を ResNet 系 CNN に移植し、注意機構なしでも Swin に匹敵する精度を出せると示しました。ただし ConvNeXt は「設計の真似」であり、Transformer の数学的本質まで再現したわけではありません。HorNet の著者たちは「ConvNeXt が設計を移植したのに対し、自分たちは Transformer の数学的核心(高次相互作用)そのものを畳み込みで再構成した」と位置づけています。

第三の流れは「広い受容野を線形コストで得る」工夫です。同じ著者グループの GFNet (Rao et al., NeurIPS 2021) は、フーリエ変換で周波数領域に移してグローバルな重み(学習可能なフィルタ)をかけ、逆変換で戻すことで、O(N log N) という安いコストで画像全体に及ぶ受容野を得ました。HorNet はこのグローバルフィルタを高次段に採用しています。

そして Rao et al., "HorNet: Efficient High-Order Spatial Interactions with Recursive Gated Convolutions", NeurIPS 2022(清華大学・北京大学ほか)が gnConv を提案し、これら3つの流れを「相互作用の次数」という統一的な視点で整理しました。後続としては、再帰ゲーティングの考え方は他の効率バックボーン(Focal Modulation、Visual Attention Network、InternImage など)と同じく「自己注意を安い演算で置き換える」大きな潮流の中に位置づけられます。さらに後年は状態空間モデル(VMamba 等)が線形複雑度の主役候補として加わり、gnConv はその系譜の重要な中間点として参照されています。

## 論文の実験結果(定量データ)
HorNet 論文の主要な評価を、指標の意味とともに説明します。

ImageNet-1K 分類(約120万枚・1000クラスの画像分類ベンチマーク。指標は top-1 accuracy = 最も確信したクラスが正解だった割合)。HorNet-T(Tiny、約 22.4M パラメータ、約 4.0 GFLOPs)は top-1 約 82.8% を達成しました。これは同規模の Swin-T(約 81.3%)、ConvNeXt-T(約 82.1%)を上回ります。1〜1.5 ポイントの差は、ImageNet のこの規模では「同じ計算量帯で頭一つ抜ける」意味のある改善です。HorNet-B(Base、約 87M パラメータ)では約 84.2〜84.3% となり、Swin-B(83.5%)・ConvNeXt-B(83.8%)を上回りました。ImageNet-22K(約1400万枚)で事前学習した HorNet-L(Large)は 384×384 入力でおよそ 87% 前後に達したと報告されています。事前学習データを増やすほど高次相互作用の容量が活きる、という大型化に強い傾向も示されました。

COCO 物体検出・インスタンスセグメンテーション(約12万枚、80クラス。指標は AP = average precision、検出した枠とマスクの正確さを総合した平均適合率。box AP が物体枠、mask AP がピクセル単位の領域。0〜100 の値で大きいほど良い)。Cascade Mask R-CNN のバックボーンを差し替えた比較で、HorNet-T は box AP 約 51.7、mask AP 約 44.8 を記録し、同設定の Swin-T・ConvNeXt-T を 1〜2 ポイント上回りました。検出は分類より高次の空間関係(物体の部分どうしの関係、物体と背景の関係)が効くタスクなので、高次相互作用を狙った gnConv の利点が分類より大きく出る傾向が示されました。

ADE20K セマンティックセグメンテーション(約2万枚、150クラスの画素単位ラベル付け。指標は mIoU = mean Intersection over Union、予測領域と正解領域の重なり率をクラス平均したもの。100% が完全一致)。UperNet ヘッドで、HorNet-T は mIoU 約 49.2(single-scale)を達成し、Swin-T(約 44.5)を 4〜5 ポイント上回りました。密予測(全画素を予測するタスク)では広い文脈の活用が決定的で、グローバルフィルタ版 gnConv の効果が顕著に出ています。

アブレーション(構成要素を抜いて効果を確認する実験)。論文の重要な分析は次の通りです。第一に、次数を変える実験では、2次・3次・5次と上げると精度が上がり、ある程度(おおむね3次前後)で飽和します。これは「高次相互作用が確かに効いている」直接的な証拠です。次数を1次(掛け算なし、ただの畳み込み相当)に落とすと精度が明確に下がります。第二に、グローバルフィルタを除いて局所カーネルだけにすると、特に密予測タスク(検出・分割)で精度が落ちます。広域受容野の寄与を示しています。第三に、チャンネル半減の設計を外す(全段同じチャンネル)と計算量が増えるだけで精度はほとんど上がらず、効率設計の妥当性が確認されました。

## メリット・トレードオフ・限界
メリット
- 自己注意の二乗計算量を払わずに、Transformer の本質(高次の空間相互作用)を畳み込みで実現できる。計算量は画素数に対しほぼ線形。
- 同規模の Swin / ConvNeXt と同等以上の精度。特に検出・セグメンテーションのような高次関係が効く密予測で優位が大きい。
- 純 CNN 系なので実装・最適化が素直で、各種フレームワーク・ハードウェアに載せやすい(専用カーネル不要)。
- 再帰段数 n で相互作用の次数を、カーネルサイズやグローバルフィルタで受容野を、それぞれ独立に調整できる設計の柔軟さ。自己注意を n=2 の特殊例として包含する一般性も理論的に魅力。

トレードオフ・限界
- 次数 n・カーネルサイズ・局所/グローバルの配分など、調整すべきハイパーパラメータが増える。
- 自己注意のように「位置 i が位置 j を見ている」という明示的な対応関係(アテンションマップ)が出ないため、何を根拠に判断したかを直接可視化しにくい。要素ごとの掛け算は解釈が間接的。
- グローバルフィルタ版はフーリエ変換を伴うため、入力解像度が学習時と変わるとフィルタの補間が必要で、可変解像度への対応に一手間かかる。
- 要素ごとの積を再帰的に重ねるため、活性化の値域が広がりやすく、学習時の数値安定性(正規化・初期化)に配慮が要る場合がある。
- 採用実績・エコシステム・事前学習済み重みの選択肢は、ResNet や Swin に比べるとまだ薄い。
- 未解決の課題として、再帰ゲーティングが「なぜ・どの程度の次数で十分か」の理論的な指針は経験則に留まっており、タスクごとに最適次数を原理的に決める方法は確立していません。

## 発展トピック・研究の最前線
gnConv の「ゲート(掛け算)で高次相互作用を作る」発想は、自己注意を安価な演算で代替する2022年前後の大きな潮流の一翼です。同時期の Focal Modulation(文脈を先に集めて変調する)、Visual Attention Network の大カーネル注意、後発の状態空間モデル系(VMamba など)と並び、「Transformer 一強への対抗軸」を形成しました。さらに深く学ぶなら、ゲート機構の起源である GLU・LSTM、広域受容野を線形コストで得る周波数領域フィルタ(GFNet・FFT 系)、そして「次数」という概念を多項式ネットワーク(polynomial networks, Π-Nets, Chrysos et al.)の観点から眺める研究が、gnConv の理論的背景を補強してくれます。多項式ネットワークの研究は「掛け算を重ねると関数の表現クラスがどう広がるか」を理論的に扱っており、gnConv の経験的成功に数学的な裏付けを与える方向です。実応用では、高解像度入力で自己注意が重すぎる場面(衛星画像、医用画像、運転シーンの大判カメラ)が gnConv 系の主戦場です。
