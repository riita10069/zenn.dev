# Gradual Unfreezing

## ひとことで言うと
Gradual Unfreezing (段階的凍結解除) とは、事前学習済みモデルを新しいタスクに微調整 (fine-tuning) するときに、最初は出力に近い「上層」だけを学習対象にし、学習が進むにつれて入力に近い「下層」へ少しずつ学習対象を広げていく手法です。一度に全層を動かすのではなく、上から順番に「解凍」していくことで、事前学習で身につけた汎用知識が一気に壊れる破滅的忘却 (catastrophic forgetting) を防ぎます。

## 直感的な理解
事前学習済みモデルを別タスクに使うとき、出力側には新しい head (タスク固有の出力層) を付け足すのが普通です。たとえば 1000 クラスの画像分類で学習したモデルを別の分類に転用するなら、最後の分類層を付け替えます。この新しい head は学習前なのでランダムな初期値、つまり中身はデタラメです。

学習の最初の瞬間を想像してください。デタラメな head が出す予測は当然大きく外れます。すると「正解からこれだけズレている」という誤差信号 (勾配) も非常に大きく、荒っぽくなります。この荒い勾配を、事前学習で丁寧に整えられた backbone (本体) の全層に一気に流すと、汎用的で価値の高い下層まで巻き込んで大きく書き換えてしまい、せっかくの事前学習の成果が台無しになります。

たとえるなら、新人 (head) がまだ仕事を覚えていないのに、その新人の判断でベテラン部署 (backbone 下層) の長年のやり方まで一斉に変更させてしまうようなものです。まず新人を一人前にしてから、徐々に組織全体の微調整に入る方が安全です。Gradual Unfreezing はこの「まず上を整え、徐々に下へ」を実現します。

## 基礎: 前提となる概念
凍結 (freeze) とは、ある層の重みを学習中に更新しない (固定する) ことです。多くのフレームワークでは、その層のパラメータに「勾配を計算しない」という印 (PyTorch なら `requires_grad = False`) を付けると、勾配が流れず重みが変わらなくなります。逆に凍結解除 (unfreeze) は、その印を外して `requires_grad = True` に戻し、学習対象に含めることです。重要なのは、凍結しても順伝播 (入力から出力へ計算を流すこと) は普通に行われる点で、凍結層は「計算には使うが学習では更新しない」状態になります。

エポック (epoch) とは「訓練データを 1 周ぶん全部見ること」です。10 エポック学習するとは、訓練データ全体を 10 回繰り返し見ることを意味します。Gradual Unfreezing では「何エポックごとに 1 層ずつ解凍するか」というスケジュールを決めます。

backbone (バックボーン) とは、事前学習済みの特徴抽出本体のことです。画像なら畳み込みや Transformer の積み重ねで、入力から段階的に抽象的な特徴を取り出します。head はその特徴を受け取って最終的な答え (分類・回帰など) を出す、タスク固有の小さな部分です。

なぜ下層を後回しにするのかには、転移学習の古典的な知見が背景にあります。Yosinski ほか (NeurIPS 2014) は、ニューラルネットの下層ほど汎用的でタスクをまたいで転用しやすい特徴 (Gabor フィルタや色斑のような普遍的な特徴) を学び、上層ほどタスク固有になることを実験で示しました。つまり下層は「壊したくない共通財産」であり、Gradual Unfreezing が下層を最後に回す設計はこの知見に根拠を持ちます。

## 仕組みを詳しく
L 層のネットワークを考え、手順をステップに分けます。

ステップ 1 (最初の数エポック): 最上層 (出力に最も近い層) と新しい head だけを凍結解除し、それ以外は全部凍結します。head が新タスクにある程度フィットするまで学習します。

```
[凍結] 層1 ... 層10  [凍結]
[学習] 層11, 層12, head  ← ここだけ動かす
```

ステップ 2 (次の数エポック): 1 つ下の層も凍結解除します。

```
[凍結] 層1 ... 層9
[学習] 層10, 層11, 層12, head
```

ステップ 3 以降: これを繰り返し、エポックが進むごとに 1 層 (または数層) ずつ下へ凍結解除を広げます。最終的に全層が学習対象になります。

```
[学習] 層1 ... 層12, head  ← 最終的に全部解凍
```

実装イメージ (概念コード):

```python
# 最初は backbone を全凍結、head だけ学習
for p in backbone.parameters():
    p.requires_grad = False

# エポックが進むたびに上の層から1ブロックずつ解凍
def unfreeze_top_block(epoch):
    blocks = list(backbone.blocks)        # 下→上に並んだブロック列
    n = min(epoch + 1, len(blocks))       # 解凍する数を徐々に増やす
    for blk in blocks[-n:]:               # 上から n 個
        for p in blk.parameters():
            p.requires_grad = True
```

before/after で言うと、before は「最初から全層 unfreeze で fine-tuning」、after は「上層から段階的に unfreeze」です。学習の安定性が大きく変わります。具体的な数値感覚としては、head だけ学習する初期段階では更新されるパラメータが全体の数パーセントに過ぎず、勾配の暴れがそこに閉じ込められます。解凍が進むにつれて学習対象が増え、最終的に 100% のパラメータが更新対象になりますが、その頃には head が安定しているので勾配も穏やかになっています。

設計上の注意がいくつかあります。第一に、凍結中の層を含むモデルでは、正規化層 (たとえば BatchNorm) の統計をどう扱うかに気をつける必要があります。凍結したつもりでも BatchNorm の running statistics (推論時に使う移動平均の平均・分散) は重みではなく順伝播のたびに更新されるため、凍結層を学習モードのままにしておくと統計が新タスクのデータで書き換わり、下層の出力分布が静かに変化してしまいます。これを防ぐには凍結層を推論モード (`eval`) に固定する配慮が要ります。第二に、オプティマイザ (重みを更新する仕組み) が凍結中のパラメータをどう扱うかも要確認で、解凍したタイミングでそのパラメータを最適化対象に追加し直す実装が必要なことがあります。Adam のように各パラメータの勾配の移動平均 (モーメント) を保持するオプティマイザでは、新たに解凍したパラメータのモーメントをゼロから始めるか引き継ぐかで挙動が変わります。

Gradual Unfreezing は discriminative learning rate や layer-wise LR decay としばしば併用されます。前者・後者は「その層をどれだけ強く動かすか (学習率)」を制御し、Gradual Unfreezing は「いつその層を動かし始めるか (時間軸)」を制御するもので、役割が補完的です。ULMFiT では、解凍した層に対してさらに層別の小さい学習率を当てることで、二重に下層を保護していました。

## 手法の系譜と主要論文
Howard & Ruder, "Universal Language Model Fine-tuning for Text Classification (ULMFiT)", ACL 2018 (arXiv:1801.06146) が Gradual Unfreezing を提案・命名した論文です。事前学習した言語モデル (大規模テキストで次単語予測を学んだ 3 層 LSTM) を文書分類に転移する手法 ULMFiT の構成要素の 1 つとして導入されました。提案理由は、全層を一度に fine-tuning すると catastrophic forgetting が起き、転移の利点が失われるから。「最後の層から始めて、エポックごとに 1 層ずつ下へ unfreeze していく」という具体的な手順を示しました。ULMFiT はこれを discriminative fine-tuning (層別学習率) と slanted triangular learning rates (急に上げてゆっくり下げる学習率スケジュール) と組み合わせて使い、3 つの工夫の相乗効果で性能を出しました。

この設計の理論的背景には Yosinski, Clune, Bengio, Lipson, "How transferable are features in deep neural networks?", NeurIPS 2014 (arXiv:1411.1792) があります。彼らは AlexNet の各層を別タスクへ転移する実験を行い、下層の特徴は汎用的でそのまま転用でき、上層に行くほどタスク固有になることを定量的に示しました。さらに、転移した層を凍結したまま使う方が、下手に全層を動かすより安定する場合があること、また転移によって生じる「層間の共適応 (co-adaptation) の断絶」が性能低下の原因になりうることも報告しました。Gradual Unfreezing が「下層を後回しにする」設計は、この「下層は汎用で壊したくない」という知見に根ざしています。

歴史的な流れとしては、まず単純な「特徴抽出器として凍結して使う (feature extraction、backbone を完全に固定して head だけ学習)」が古くからあり、次に「全層を一度に fine-tuning する」が普及し、その中間として「段階的に解凍する」Gradual Unfreezing が安定性と性能のバランスを取る手法として位置づけられました。現在では、巨大モデル時代には全層を更新せずアダプタや LoRA など一部だけ学習する流れも強まっており、Gradual Unfreezing は「忘却を防ぐ古典的かつ堅実な選択肢」として参照されます。

## 論文の実験結果(定量データ)
評価指標を説明します。文書分類では error rate (誤分類率、低いほど良い) がよく使われます。たとえば error 5.0% は 100 件中 5 件を間違えることを意味し、4.0% に下がれば誤りが 2 割減ったことになります。

ULMFiT は 6 つの文書分類ベンチマーク (IMDb 映画レビューの感情分類、AG News や DBpedia のトピック分類、Yelp レビュー、TREC 質問分類など) で評価され、当時の最高性能を更新しました。報告によれば、IMDb で error rate を従来手法から相対的に大きく削減し (報告では誤り 4.6% 程度まで低下、従来比でおおよそ 2 割前後の誤り削減)、複数データセットで SOTA を達成しました。重要なアブレーションとして、Gradual Unfreezing を入れずに全層を一度に fine-tuning すると性能が落ち、特に学習が不安定になる (検証誤差が暴れる、あるいは発散する) ことが示されました。逆に、最後の層を凍結したまま fine-tuning するのも、Gradual Unfreezing を組み込んだ完全版に劣りました。これは「凍結しすぎても解凍しすぎても良くなく、段階的が最良」という、本手法の核心を裏づける結果です。

ラベルの少ない設定での頑健性も大きな成果でした。ULMFiT は、ごく少量のラベル付きデータ (たとえば 100 件程度) でも、はるかに多くのデータ (数万件規模) でゼロから学習した従来手法に匹敵する精度を出せることを示しました。これは事前学習知識を Gradual Unfreezing で壊さず保てたことの直接的な恩恵で、データが乏しいほど「忘却を防ぐ」価値が際立つことを意味します。

Yosinski ほか (2014) の実験では、ImageNet を二分割した転移課題で、転移した層を凍結したまま使ったときと fine-tuning したときの精度差を層位置ごとに測りました。結果、下層 (1〜2 層目) を転移しても性能はほぼ落ちないが、中間層を凍結したまま使うと共適応の断絶で精度が下がること、そしてその下がった分は fine-tuning で回復できることを定量的に示しました。下層を不用意に動かすと性能が落ちる一方、適切に転移すればゼロから学習するより良くなる、という Gradual Unfreezing の前提を裏づける結果です。

## メリット・トレードオフ・限界
メリット:
- catastrophic forgetting を防ぎ、事前学習知識を守りながら新タスクへ適応できる。
- 学習初期の荒い勾配が下層を壊すのを避けられ、学習が安定する。
- 追加の重みパラメータは不要で、学習スケジュールの工夫だけで実現できる。
- ラベルが少ない設定で特に頑健性が高い。

トレードオフ・限界:
- 解凍スケジュール (何エポックで何層解凍するか) という設計判断が増える。
- 段階的に進めるぶん総学習エポックが増え、学習時間が延びやすい。
- 効果はタスク・モデル・データ量に依存し、十分なデータがあれば最初から全層 fine-tuning でも遜色ないことがある。
- 凍結層の正規化統計やオプティマイザ状態の扱いを誤ると、意図せず下層が動いてしまうバグを生む。
- 解凍のたびに学習対象が変わるため、学習率スケジューラの設計が複雑になり、warmup を解凍ごとに入れ直すなどの工夫が要る場合がある。

未解決の課題として、最適な解凍ペース (1 層ずつか数層ずつか、何エポックごとか) をデータから自動決定する確立した方法はなく、実務では経験則に頼ります。また、勾配の暴れを抑えるという目的に対しては、適切な warmup と層別学習率を併用すれば全層同時 fine-tuning でも十分安定する場合が多く、Gradual Unfreezing が必須かどうかはタスク依存です。

## 発展トピック・研究の最前線
巨大事前学習モデルの時代になり、Gradual Unfreezing の発想は parameter-efficient fine-tuning (PEFT) へと形を変えて受け継がれています。Adapter (Houlsby ほか 2019、arXiv:1902.00751、各層に小さな追加モジュールだけ挿入して学習) や LoRA (Hu ほか 2021、arXiv:2106.09685、低ランク行列だけ学習) は、本体を凍結したまま少数のパラメータだけ動かす点で「忘却を防ぐ」目的を共有します。Gradual Unfreezing が「時間方向に解凍範囲を広げる」のに対し、PEFT は「空間的に学習対象を絞る」アプローチと言えます。

また、学習率を層ごとに連続減衰させる layer-wise LR decay は、Gradual Unfreezing の「下層を慎重に扱う」狙いを、ハードな凍結ではなくソフトな学習率調整で実現する近縁手法です。凍結が「学習率を 0 にする」極端な場合と見れば、両者は連続的につながっています。現在の Vision Transformer の fine-tuning では、明示的な Gradual Unfreezing より LLRD が主流になっていますが、両者は「下層の事前学習表現を守る」という同じ哲学を共有しています。さらに継続学習 (continual learning、複数タスクを順に学んでいく設定) の研究でも、忘却を防ぐための凍結・部分更新の戦略として Gradual Unfreezing の系譜が参照されています。
