# JEPA系 vs 生成系世界モデルの対比

## ひとことで言うと
「未来を予測する世界モデル」には大きく2系統あります。画素まで作り直す生成系(GAIA-1 / Vista など)と、画素は作らず抽象的な特徴ベクトルだけを予測する JEPA 系です。どちらにも明確な長所短所があり、世界モデルを設計するときに「何を予測対象にするか」という最初の分岐点になります。本ノートはその2系統を比較する地図です。

## 直感的な理解

絵を描いて未来を予測する人と、言葉で要点だけメモして予測する人を想像してください。

絵を描く人(生成系)は、未来のシーンを細部まで描き起こします。木の葉の一枚一枚、影の落ち方、路面の質感まで。出来上がった絵は誰が見ても分かりますが、描くのに膨大な時間がかかり、しかも「次の瞬間に葉がどう揺れるか」など本質的にどうでもいい細部にも筆を費やします。

要点メモの人(JEPA 系)は、「前方に歩行者、自車は減速すべき」といった抽象的な要点だけを書き留めます。葉の揺れは書きません。素早く、本質に集中できますが、メモそのものは絵のように直接見て楽しめるものではなく、解釈するには別の知識が要ります。

世界モデルの2系統はこの比喩そのものです。未来の画像全体を生成するのか、未来の「意味的な特徴」だけを予測するのか。この選択が、計算コスト・可視化のしやすさ・予測の頑健さ・評価方法のすべてを左右します。LeCun が JEPA を提唱した動機をひとことで言えば「予測できないものを予測しようとして容量を無駄にするな」です。路面に映る雨粒の正確な配置を当てることに意味はなく、当てるべきは「前方が滑りやすい」という意思決定に効く抽象量だ、という主張です。

## 基礎: 前提となる概念

いくつかの土台を噛み砕きます。

- 世界モデル(world model): 「今の状態 + 行動 → 未来の状態」を予測するネットワーク。
- 生成(generation): 出力として実際のデータ(ここでは画像・動画のピクセル)を作ること。
- 特徴 / 表現(feature / representation): ニューラルネットの中間層が出す、入力を要約した数値ベクトル。人間にはそのままでは読めないが、意味的な情報を凝縮しています。
- エンコーダ(encoder): 入力(画像など)を特徴に変換するネット。
- 自己教師あり学習(self-supervised learning, SSL): ラベルなしデータから、データ自身を手がかりに表現を学ぶ方式。「画像の一部を隠して残りから当てる」などが典型。
- 表現崩壊(representation collapse): SSL で起こりがちな失敗で、エンコーダが入力を無視して常に同じ出力(定数)を返してしまう状態。「予測対象が予測器の出力と一致すればよい」という目的だと、両方とも定数にすれば誤差ゼロでズルできてしまうため起きます。これを防ぐ工夫が JEPA 系の肝です。
- 拡散モデル(diffusion model): ノイズから徐々にノイズを取り除いて画像を作る生成方式。現在の高品質画像・動画生成の主流。

JEPA(Joint-Embedding Predictive Architecture, 結合埋め込み予測アーキテクチャ)は、Yann LeCun が 2022 年に提唱した世界モデルの設計思想で、「画素を生成せず、抽象表現空間で予測する」ことを核とします。「結合埋め込み(joint-embedding)」とは、context と target の両方を同じ埋め込み空間に写してから、その空間内で予測する、という意味です。

## 仕組みを詳しく

生成系の処理の流れは次の通りです。

```
過去フレーム + 行動 --> [生成モデル] --> 未来フレーム(画素) [B, 3, H, W]
loss = 画素レベルの再構成誤差、あるいは拡散モデルのノイズ予測誤差
```

拡散モデルや Transformer ベースのトークン生成(画像を離散トークン列に変換してから次トークンを予測)が使われます。出力は人間が見られる動画です。拡散ベースは推論時に数十回のノイズ除去ステップを踏むため、1フレーム生成に多くの計算を要します。

JEPA 系の処理の流れはこうです。

```
context --> [encoder]        --> z_context --> [predictor] --> ẑ_target (予測した特徴)
target  --> [target encoder] --> z_target  (stop-gradient で勾配を止める)
loss = || ẑ_target - z_target ||   (特徴空間での距離)
```

ポイントは2つの工夫です。第一に target encoder の出力に stop-gradient をかける(勾配を流さない)こと。第二に target encoder を、本体エンコーダの重みの指数移動平均(EMA, exponential moving average)でゆっくり更新すること。この非対称な構造により、両方が定数に潰れる表現崩壊を防ぎます。なぜ崩壊が防げるかを直感で言うと、target 側がゆっくり遅れて動く「動く標的」になるため、context 側が単純な定数へ逃げても標的に追いつけず、入力に依存した有意な表現を学ばざるを得なくなるからです。出力は特徴ベクトルで、人間が直接見ることはできません(見たければ別途デコーダを後付けする必要があります)。

両者を対比表で整理します(初心者向けに数値感も添えます)。

| 観点 | 生成系 (GAIA-1 / Vista) | JEPA系 (I-JEPA / V-JEPA) |
|---|---|---|
| 予測対象 | 画素(例: [B,3,288,512]) | 特徴(例: [B,N,768]) |
| 計算コスト | 高い(拡散の多ステップ生成等) | 比較的低い |
| 出力の可視化 | できる(動画が見える) | できない(別途デコーダ要) |
| 細部 | 質感・影まで生成 | 予測不要な細部は無視 |
| 崩壊リスク | 低い(画素という明確な正解がある) | あり(stop-grad + EMA で回避) |
| 評価 | FVD など生成品質指標が使える | 線形評価・下流タスク精度で測る |
| 未来の多峰性 | 確率的生成で複数の未来を出せる | 平均化でぼやけやすい(課題) |

「予測不要な細部に容量を浪費しない」という JEPA 系の主張の数値的根拠は、後述する計算効率の実験に表れます。

## 手法の系譜と主要論文

JEPA 系の系譜:
- Yann LeCun "A Path Towards Autonomous Machine Intelligence" (2022): 世界モデルは抽象表現空間で予測すべきで、画素生成は非効率という構想を提示。動機は「予測不能な細部に容量を浪費しない」。これは指針(白書)であって実装ではありません。
- I-JEPA (Mahmoud Assran et al., CVPR 2023, arXiv:2301.08243): 画像版の実装。1枚の画像の一部(context)から別の領域(target)の特徴を予測する。少ない計算で強い表現を学べることを示した。MAE(画素再構成)と違い、context を一つの大きめブロックに、target を複数の意味的に意味のある領域にする「マルチブロックマスキング」が新規性です。
- V-JEPA (Adrien Bardes et al., 2024, Meta): 動画版。時空間的にマスクした領域の特徴を予測し、動画理解の表現を学ぶ。トレードオフは出力が特徴のため人間が直接検証しづらいこと。
- V-JEPA 2 (Meta, 2025): V-JEPA を大規模化し、ロボット行動の計画にも使える行動条件付き世界モデルへ発展。JEPA の発想が「表現学習」から「制御に使える世界モデル」へ橋渡しされた例で、JEPA 系も行動条件付きロールアウト([反実仮想ロールアウト](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1033_counterfactual-rollout))が可能だと示しました。

生成系の系譜:
- GAIA-1 (Anthony Hu et al., Wayve, 2023, arXiv:2309.17080): 運転動画の世界モデル。動画・テキスト・行動を入力に未来運転動画を生成。約 90 億パラメータ規模で、行動条件付きの多様な未来運転を生成できることを示した。トレードオフは巨大で計算が重いこと。
- DriveDreamer (Xiaofeng Wang et al., ECCV 2024, arXiv:2309.09777): HD マップや物体ボックスなどの構造化条件を与えて運転シーンを生成。条件付けにより制御性を高めた。トレードオフは条件データの整備が必要なこと。
- Vista (Shenyuan Gao et al., NeurIPS 2024, arXiv:2405.17398): 高解像度・長時間・行動制御可能な運転世界モデル。拡散ベースで、反実仮想的な運転生成を高画質で実演。トレードオフは推論が重いこと。
- Genie (Jake Bruce et al., DeepMind, ICML 2024, arXiv:2402.15391): 行動可能な仮想世界を、行動ラベルなしの動画だけから教師なしで学ぶ生成系。潜在行動を自動発見する点が新しい。トレードオフは計算規模。

この2系統は対立というより、目的による使い分けです。表現学習や効率的な計画には JEPA 系、人間が検証したいシナリオ生成やデータ拡張には生成系、という分担が見えてきています。なお Dreamer 系([モデルベース強化学習 (MBRL)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0077_model-based-rl))は「潜在で予測しつつ画素再構成も学習信号に使う」中間型であり、純粋な2分法では割り切れないことにも注意が要ります。

## 論文の実験結果(定量データ)

- I-JEPA (CVPR 2023): ImageNet の線形評価(エンコーダを凍結し、その特徴に線形分類器だけを乗せて分類精度を測る方法。表現の良さの代理指標)で、画素を再構成する MAE(Masked Autoencoder)などと同等以上の精度を、はるかに少ない事前学習計算量で達成したと報告。具体的には、同程度の精度に到達するのに必要な GPU 時間が数倍効率的だったとされ、「画素を当てない方が表現学習として効率的」という JEPA の主張を数値で裏づけました。線形評価精度が高いほど、エンコーダが「分類に使える本質的な情報」を捉えていることを意味します。
- V-JEPA (2024): Kinetics-400(動画行動認識)や Something-Something v2(細かい時間的動作の認識)などの凍結評価で、ピクセル再構成ベースの動画 SSL を上回る、または同等の性能を、より少ない学習サンプルで達成。特に「動きの理解」を要するタスク(Something-Something v2 は物体の動かし方そのものを当てるため、見た目より動きが効く)で強いことが示されました。凍結評価とは、学習済みエンコーダを固定したまま小さな分類器だけを乗せて測る方法で、表現がどれだけ汎用的かを見る指標です。
- GAIA-1 / Vista(生成系): これらは FVD(Fréchet Video Distance, 後述)や生成画質で評価されます。Vista は先行の運転動画生成手法に対し FVD を改善し、より長く・高解像度で・行動に追従した動画を生成できることを報告しています。FVD は値が小さいほど「本物の動画分布に近い」ことを意味し、生成系世界モデルの標準的な品質指標です([運転世界モデルの評価指標](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0767_navsim-driving-world-model-eval))。

注意すべきは、JEPA 系と生成系は同じ指標で直接比較できないことです。JEPA 系は画素を作らないので FVD で測れず、生成系は特徴の線形評価をそのまま受けにくい。この「評価軸が違う」こと自体が、2系統の本質的な差を物語っています。公平に比べたいなら、最終的には下流タスク(運転性能、closed-loop 安全性)で揃えて測るしかありません。

## メリット・トレードオフ・限界

JEPA系:
- メリット: 計算が軽い、予測不要な細部を無視でき本質に集中できる、表現学習として強い、ノイズ・無関係な細部に頑健。
- トレードオフ・限界: 出力を直接可視化できない、表現崩壊を防ぐ工夫(stop-grad + EMA)が必須で学習レシピがデリケート、生成品質指標が使えない、未来の多峰性を表現する手立てが乏しく予測がぼやけやすい。「何を target にするか(マスク設計)」が性能を大きく左右し、設計依存が強い。

生成系:
- メリット: 人間が見て検証できる、FVD など確立した評価指標が使える、リアルなシナリオ生成やデータ拡張に使える、確率的生成で複数の未来を出せる。
- トレードオフ・限界: 計算が非常に重い、予測不能な細部にまで容量を割く、推論が遅く実時間計画には不向き、巨大モデルの学習コストが高い、生成のリアルさ(FVD が良い)が必ずしも下流の運転性能に直結しないこと。

研究上の未解決課題として、「JEPA 系の効率と、生成系の可視化・検証性を両立できないか」という問いがあります。たとえば JEPA で学んだ表現に軽量デコーダを後付けして必要なときだけ可視化する、あるいは生成系を潜在空間に圧縮して軽くする(latent diffusion 的アプローチ)、といった折衷が模索されています。

## 発展トピック・研究の最前線

- ハイブリッド世界モデル: 潜在空間で予測(JEPA 的)しつつ、必要に応じて軽量デコーダで画素に戻す。Dreamer 系([モデルベース強化学習 (MBRL)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0077_model-based-rl))はこの中間に位置します。
- 行動条件付き世界モデル: どちらの系統も、行動を条件に未来を変える([反実仮想ロールアウト](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1033_counterfactual-rollout))機能を持たせる方向が運転応用で重要。生成系は「行動を変えた未来動画」、JEPA 系は「行動を変えた未来特徴」を出します。V-JEPA 2 はこの方向で JEPA を制御へ拡張した例です。
- 評価方法論の発展: 系統をまたいで公平に比較するため、最終的に下流タスク(運転性能、closed-loop 安全性)で測るべきという議論が強まっています([運転世界モデルの評価指標](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0767_navsim-driving-world-model-eval))。
- スケーリング: 生成系は GAIA-1 のように数十億パラメータへ大規模化が進む一方、JEPA 系は効率で勝負する方向で、両者のスケーリング法則の違いが研究テーマになっています。
- 潜在行動の自動発見: Genie のように、明示的な行動ラベルなしで「世界を操作する軸」を学ぶ方向は、ラベル取得が難しい実世界データで特に有望です。
