# ConvNeXt V2

## ひとことで言うと
ConvNeXt V2 は、純粋な畳み込みニューラルネットワーク(CNN)である ConvNeXt を、最新の自己教師あり事前学習に対応させて強化したものです。具体的には「画像の一部を隠して当てさせる」マスク予測型の事前学習を CNN でも回せるようにした FCMAE(Fully Convolutional Masked Autoencoder)と、特徴チャンネルの偏りを直す GRN(Global Response Normalization)という小さな部品を組み合わせ、純 CNN のまま Vision Transformer 系の自己教師ありモデルに正面から対抗できる性能を出しました。鍵は、アーキテクチャと事前学習を別々ではなく一緒に設計し直した(co-design)点にあります。

## 直感的な理解
前作 ConvNeXt(V1)は「設計を Transformer 風に近代化すれば純 CNN でも強い」ことを示しました。しかし当時、ラベルなしの大量画像から賢く学ぶ「自己教師あり事前学習」の最有力手法 MAE は Transformer 専用で、CNN にうまく適用できませんでした。料理に例えると、最高の食材調達ルート(MAE)が手に入ったのに、自分の厨房(CNN)はその食材を受け入れる設備がなかった、という状況です。ConvNeXt V2 がやったのは、厨房そのものを食材に合わせて改修し(GRN の追加)、調理法も CNN 用に作り直す(FCMAE)ことでした。結果、純 CNN でも自己教師ありの恩恵をフルに受けられるようになりました。

なぜこれが重要かというと、自動運転を含む実応用では「人手でラベルを付けた画像」は高価で量が限られる一方、「ラベルのない走行画像」は無尽蔵に集まるからです。ラベルなしデータから強い特徴を学べる能力は、そのまま実用上の競争力になります。

## 基礎: 前提となる概念
言葉を順に噛み砕きます。

- バックボーンとは、生のピクセルから意味のある特徴を取り出す土台部分のことです。入力は例えば 256x256x3(縦・横・色3チャンネル)の数値の塊で、ここから「車のエッジ」「路面のテクスチャ」といった特徴を抽出します。
- 自己教師あり学習とは、人間が「これは猫」とラベルを付けなくても、データ自身から学習信号を作る方法です。ラベル付けのコストをかけずに大量データから学べます。
- マスクオートエンコーダ(MAE、He et al., CVPR 2022)とは、画像の大部分(MAE では約 75%)を隠して(マスクして)、残った部分から隠した部分を復元させる事前学習です。「画像の4分の3を隠しても4分の1から復元できる」という発見が、視覚の自己教師あり学習を大きく前進させました。MAE は Transformer 向けに設計されていました。Transformer はトークン(パッチ)の集合を順序不問で自由に扱えるので、残ったパッチ「だけ」をエンコーダに入れて計算量を 1/4 に減らせたためです。
- スパース畳み込み(sparse convolution)とは、値が存在する場所だけで計算し、空(マスクされた)場所はスキップする畳み込みです。特に submanifold sparse convolution は、入力が存在する位置にだけ出力を生成し、穴を勝手に膨張させない方式で、後述の FCMAE の核になります。
- チャンネルとは特徴マップの「種類」を表す次元です。96 チャンネルなら 96 種類の異なる模様検出器が並んでいるイメージで、多様なチャンネルが生きているほど表現力が高いと言えます。
- 特徴崩壊(feature collapse)とは、多くのチャンネルが似た値になったり、ごく一部だけが強く反応して残りがほぼゼロ(死んだ状態)になったりして、実効的な表現の多様性が失われる現象です。GRN が直そうとする問題です。

## 仕組みを詳しく
ConvNeXt V2 の貢献は大きく2つ、FCMAE と GRN です。両者は互いに必要で、片方だけでは効きません。ここが co-design の意味です。

### 1. FCMAE(Fully Convolutional Masked Autoencoder)
Transformer 向けの MAE は、残ったパッチだけを並べてエンコーダに入力できました。しかし CNN はフィルタを画像全面に滑らせる前提なので、一部だけ隠すと「穴の開いた画像」になり、フィルタが穴をまたいで計算してしまい、隠した情報が漏れたり(マスクされた領域にも畳み込みが値を書き込んでしまう)、学習が崩れたりします。マスクを単なるゼロ埋めで実装すると、層を重ねるうちに穴がだんだん埋まっていき、復元タスクが自明になってしまうのです。

FCMAE はこれをスパース畳み込みで解決します。処理の流れを数値例で追います。

```
入力 224x224x3
  │ 32x32 のパッチに分割し、約 60% をランダムにマスク
  ▼
見えている約 40% の領域だけを submanifold スパース畳み込みで処理
  │ (マスク領域はスキップ。穴をまたぐ漏れ・穴の浸食が起きない)
  ▼
エンコーダ(ConvNeXt 本体)が見えている部分から特徴を抽出
  ▼
軽量なデコーダ(単一の ConvNeXt ブロック程度)が隠した部分のピクセルを復元
  ▼
復元誤差(実ピクセルとの差、マスク領域のみで計算)を最小化
```

正解ラベルは一切要りません。「画像自身」が教師になります。submanifold スパース畳み込みにより、ViT の MAE と概念的に等価な「見えている部分だけから隠れた部分を予測する」設定が CNN でも成立します。デコーダは ViT-MAE が用いた重い Transformer デコーダではなく、軽量な単一 ConvNeXt ブロックで十分なことも示されました。ファインチューニングや本番推論時にはスパース畳み込みを通常の密な畳み込みに戻して使うため、推論コストは通常の ConvNeXt と変わりません。

### 2. GRN(Global Response Normalization)
FCMAE で事前学習してみると、ConvNeXt の特徴チャンネルが「ごく少数のチャンネルだけが強く反応し、残りが死んでいる」という特徴崩壊を起こすことが分かりました。著者らはこれをチャンネル間のコサイン距離(特徴の類似度)で定量化し、深い層ほど崩壊が進むことを可視化しています。死んだチャンネルが多いほど実効的な表現力が落ちます。

GRN はこれを直す軽量な正規化層で、各 ConvNeXt ブロックの MLP(逆ボトルネックの拡張側)の後に1つ挿入します。処理は3ステップです。形状を (B, H, W, C)、B=バッチ、H/W=縦横、C=チャンネルとして示します。

```
1. グローバル集約: 各チャンネルの特徴マップ全体の大きさ(L2 ノルム)を計算
   (B, H, W, C) → (B, 1, 1, C)  各チャンネルの「総活性量」 G_c
2. 正規化(分割正規化): G_c を全チャンネルの平均的な大きさで割り、相対的な強さ N_c に変換
   N_c = G_c / mean_c(G_c)
3. キャリブレーション: 元の特徴 x に N_c を掛けて再重み付けし、
   学習可能なスケール γ・バイアス β と残差で調整
   出力 = γ · (x · N_c) + β + x
```

効果は、強すぎるチャンネルが相対的に抑えられ、弱いチャンネルが持ち上がり、チャンネル間の多様性(コントラスト)が回復することです。生物の神経で観測される側方抑制(強く発火した細胞が近傍を抑える)に着想を得た仕組みです。追加パラメータは γ と β の各 C 個だけで、数行で書ける軽量な層ですが、FCMAE と組み合わせたときに効果が最大化されます。論文のアブレーションでは、教師あり学習に GRN だけ足してもほとんど伸びず、FCMAE ありで GRN を足すと大きく伸びることが示されており、これが「co-design(両者を一緒に設計する)」という主張の根拠です。なお V1 にあった Layer Scale は GRN と役割が重なるため V2 では取り除かれています。

この2つを組み合わせ、Atto(約 370 万パラメータ)から Huge(約 6.5 億パラメータ)まで幅広くスケールさせたのが ConvNeXt V2 です。小型版 `convnextv2_tiny` のステージ別チャンネルは ConvNeXt 同様 96, 192, 384, 768 で、入力 256x256 ならおおよそ 64x64 → 32x32 → 16x16 → 8x8 の4段特徴を出します(features_only 指定でこれらを取り出して下流タスクへ渡せます)。ConvNeXt は channels-first(B, C, H, W)で特徴を返す点が、channels-last 系の Swin と扱いが分かれるところです。

## 手法の系譜と主要論文
- 2016年(CVPR): He らの ResNet が残差接続で超深層 CNN を可能にし、系譜の起点となりました。
- 2017年: Graham らの submanifold sparse convolution が、スパース入力を効率的に畳み込む基盤を与え、FCMAE の実装を支えました。
- 2022年(CVPR): Liu らの ConvNeXt([ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext))が、ResNet を Transformer 風に近代化し純 CNN で Swin を上回りました。ただし MAE 型自己教師ありには未対応でした。
- 2022年(CVPR): He らの MAE(arXiv:2111.06377)がマスク予測型自己教師あり事前学習を確立。ただし Transformer 専用でした。同時期の SimMIM(Xie et al.)も同系統で、[Swin Transformer V2 (SwinV2)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0363_swin-v2-tiny) の大規模学習を支えました。
- 2023年(CVPR): Woo, Debnath, Hu, Chen, Liu, Kweon, Xie らの ConvNeXt V2(arXiv:2301.00808、Meta AI と KAIST)が本トピック。動機は「CNN を MAE 型自己教師ありに対応させる」こと。新規性は FCMAE(スパース畳み込みで CNN にマスク予測を可能化)と GRN(特徴崩壊の修復)、そして両者の co-design です。

## 論文の実験結果(定量データ)
- ImageNet-1K 分類(top-1 accuracy、最も確からしい予測が正解と一致する割合): ConvNeXt V2-Huge は ImageNet-22K 中間学習を併用した最良構成(512x512)で約 88.9% に達し、当時の純 CNN として最高クラスの精度を記録しました。小型の Tiny は、教師あり学習版 ConvNeXt V1-T(約 82.1%)に対し、FCMAE 事前学習版で約 83.0% へと明確に上回りました。
- スケーリングの一貫性: Atto(370 万、約 76.7%)から Huge(6.5 億)まで、全サイズで FCMAE 事前学習が教師ありスクラッチ学習を上回りました。特に Atto/Femto のような極小モデルでも自己教師ありが効く点は、ViT-MAE が小型では効きにくいのと対照的で、「自己教師ありが小型 CNN にも効く」ことを示し、エッジ向け小型モデルにも恩恵が及ぶ実用上重要な結果です。
- 下流タスク: COCO 物体検出・ADE20K セグメンテーションでも、FCMAE 事前学習版が教師あり版より高い box AP / mIoU を示し、事前学習で得た特徴が分類以外にも転移することを確認しています(ConvNeXt V2-L で ADE20K mIoU 約 54〜55 級)。

最重要のアブレーションは GRN と FCMAE の相互依存です。教師あり学習に GRN を足してもほぼ効果なし(約 +0.1 以下)、FCMAE だけだと特徴崩壊で頭打ち、FCMAE + GRN にして初めて大きく伸びる(報告では合わせて約 +0.7〜0.8 ポイント、かつ崩壊指標が大幅に改善)、という結果が報告されています。これは「アーキテクチャ(GRN)と事前学習法(FCMAE)を別々に最適化しても駄目で、一緒に設計してこそ性能が出る」という論文の中心主張を数値で裏付けるもので、深層学習における co-design の好例としてよく引用されます。GRN を入れると死んだチャンネルが減り、チャンネル間のコサイン類似度が下がる(=多様性が回復する)ことも可視化されています。

## メリット・トレードオフ・限界
メリット:
- ラベルなし画像から FCMAE で事前学習でき、ViT の MAE に匹敵する性能を純 CNN で得られます。走行ログのような大量の無ラベル画像を活かせます。
- CNN なので局所性・並進不変性という帰納バイアスを持ち、ViT より小さなデータでも比較的安定して学習できます。
- 純 CNN は推論時のメモリ・速度が読みやすく、車載 GPU などエッジ環境への展開で扱いやすい。FCMAE のスパース化は事前学習時だけなので、推論コストは V1 と同じです。
- GRN は軽量(追加パラメータは 2C のみ)で、既存の ConvNeXt 構造をほぼそのまま流用できます。

トレードオフ・限界:
- FCMAE のスパース(submanifold)畳み込みは実装が複雑で、自前で事前学習するハードルは高い(実務では公開済みの事前学習重みを使うのが通例)。
- 大規模スケール(Huge など)では事前学習の計算コストが大きい。
- ViT のように超大規模データや画像テキスト混在で青天井に伸びるかという点では、CNN の帰納バイアスがかえって上限になる場面が指摘されています。
- GRN の効果は FCMAE と組でこそ最大化され、単独では限定的なため、他の学習設定への移植では効果が読みにくいことがあります。

## 発展トピック・研究の最前線
ConvNeXt V2 は「自己教師あり事前学習はバックボーンの種類(CNN か Transformer か)を選ばない」ことを示し、マスク予測型の事前学習を視覚基盤モデルの標準的な構成要素へ押し上げました。SwinV2 の SimMIM、BEiT 系のトークン予測など、同時代の自己教師あり手法と並んで、現在は多くの視覚モデルが何らかのマスク予測で事前学習されます。特徴崩壊を防ぐ正規化(GRN)の発想は、対照学習や生成モデルでの表現の多様性維持にも通じる一般的な論点で、後続のネットワーク設計でも参照されています。研究の最前線では、画像・テキストを横断するマルチモーダル自己教師あり、動画や時系列を含むマスク予測、そして自動運転の文脈では大量の無ラベル走行データからの事前学習が、ラベル効率とドメイン適応の鍵として注目されています。未解決の課題としては、CNN の帰納バイアスがスケール上限を作る境界がどこか、車載のリアルタイム制約下で巨大事前学習モデルをどう圧縮(蒸留・量子化)して載せるか、が挙げられます。
