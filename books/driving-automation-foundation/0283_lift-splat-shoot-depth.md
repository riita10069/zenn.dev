# LSS(明示的深度)によるBEV化

## ひとことで言うと

LSS(Lift, Splat, Shoot)は、カメラ画像から鳥瞰図(BEV、真上から見た地図)を作る代表的な方法です。「画素ごとに、その点が自車からどのくらい遠いか(深度)」をモデルに予測させ、その予測を頼りに各画素の特徴を 3D 空間に持ち上げ(lift)、上から見た地図に貼り付ける(splat)のが特徴です。深度を明示的に予測する点が、深度を予測しない暗黙投影方式(BEVFormer 系)との大きな違いで、カメラ BEV 化の二大潮流の一方の出発点になりました。

## 直感的な理解

写真を1枚見て「この物体は何メートル先か」を当てるのは、人間でも難しいことがあります。遠くにある大きなトラックと、近くにある小さなおもちゃの車が、写真上では同じ大きさに写ることがあるからです。1枚の画像だけからは奥行きが一意に決まりません。これを専門的には「不良設定問題(ill-posed problem、答えが一つに定まらない問題)」と呼びます。

ところが、BEV(真上から見た地図)を作るには、各画素が実世界のどこにあるかを知る必要があり、そのためには奥行きが要ります。ここで LSS の発想が効いてきます。「奥行きが一意に決まらないなら、無理に1つの値に決め打ちせず、確率分布として持てばよい」というものです。

たとえば、ある画素について「10m が確率40%、15m が35%、20m が25%」のように、複数の深度候補に重みを振り分けます。確信が持てる画素(地面のはっきりした模様など)は1つの深度に重みが集中し、曖昧な画素(のっぺりした壁など)は複数に薄く広がります。こうしておけば、深度推定の不確かさを潰さずに下流へ渡せて、複数カメラや時系列の情報と合わさったときに正しい位置が自然に浮かび上がりやすくなります。

## 基礎: 前提となる概念

**深度(depth)** とは、カメラから対象までの距離(奥行き)です。ステレオカメラ(2眼)や LiDAR があれば測れますが、単眼カメラ1枚では本質的に曖昧です。

**視錐台(frustum)** とは、カメラから前方に向かって広がる四角錐状の視野空間のことです。画像の各画素は、カメラから前方へ伸びる1本の「光線(ray)」に対応します。その光線上のどこに物体があるか(=深度)は1枚では分からないので、光線上に深度候補を何段も並べて点群にします。

**外積(outer product)** とは、ここでは「深度ごとの確率」というベクトルと「特徴」というベクトルを掛け合わせて、両者の全組み合わせの表を作る操作です。たとえば長さ D の深度確率と長さ C の特徴から、D×C の表ができます。

**スプラッティング(splatting)** とは、ばらばらに散らばった 3D の点を、規則正しいグリッド(BEV のマス目)に投げ込んで集計する操作です。「水滴を地面に落として、同じマスに落ちた分を合計する」イメージです。

## 仕組みを詳しく

LSS の名前は3段階の頭文字 Lift / Splat / Shoot から来ています。形(tensor shape)を追いながら説明します。カメラ画像から取り出した特徴マップを `[V, C, H, W]`(V=カメラ枚数、C=チャンネル、H×W=特徴マップの縦横)とします。

**1. Lift(持ち上げ)。** 各画素について、深度の候補を離散的に並べます。たとえば 4m から 45m を 1m 刻みで D=41 段階に区切ります。モデルは各画素ごとに「どの深度がどれくらい確からしいか」という確率分布(長さ D、合計1になるようソフトマックスで正規化)を予測します。同時に、画素ごとの特徴ベクトル(長さ C)も持ちます。この2つの外積(深度確率 × 特徴)を取ると、各画素は「D 個の深度それぞれに、確率で薄めた特徴」を持つ視錐台になります。形は `[V, C, D, H, W]` に膨らみます。確信が持てない画素は複数の深度に薄く広がり、確信できる画素は1つの深度に集中する、という性質がここで生まれます。

```
1画素の中身(D=41 の例):
 深度  : 4m  5m  ... 10m ... 15m ... 45m
 確率  : .01 .01 ... .40 ... .35 ... .00   (合計1.0)
 特徴  : [256次元のベクトル c]
 → 各深度に「確率 × c」を置く。確率40%の10mに最も濃く、35%の15mに次に濃く
```

**2. Splat(貼り付け)。** Lift で得た視錐台の各点を、カメラの校正情報(内部行列・外部行列)で実世界の 3D 座標に変換します。3D→画像は曖昧さなく計算できるので、ここも一意です。変換した点を BEV グリッド(たとえば 200×200 のマス目)に投げ込み、同じマスに複数の点が落ちたら合計します。この集計は、点群を扱う PointPillars(Lang et al., CVPR 2019)由来の効率的な手法(マスごとの sum pooling を累積和で高速化する cumsum trick)で実装され、全カメラぶんを1枚の BEV 特徴マップ `[C, X, Y]` にまとめます。

**3. Shoot(撃ち出す)。** 得た BEV 特徴の上で、自車が取りうる複数の走行軌跡(あらかじめ用意したテンプレート軌跡)を評価し、最も良い軌跡を選びます。これが経路計画にあたる部分です。LSS の元論文は知覚から運動計画まで一気通貫で提案した点が新しく、「Shoot」はその計画部分の名前です。

深度を確率分布で持つことの効果を具体例で言うと、ある画素の深度が 10m か 30m か曖昧な場合、決め打ちすると間違った位置に特徴を置いてしまいますが、確率で両方に薄く置けば、別のカメラや次のフレームの情報と整合した位置だけが強く残り、矛盾する位置は他から支持されず消えていきます。

## 手法の系譜と主要論文

- **Lift, Splat, Shoot(Philion & Fidler, ECCV 2020, arXiv:2008.05711)** が本ノートの主題です。任意台数・任意配置のカメラから、校正情報だけで end-to-end に BEV 表現と運動計画を学習する枠組みを提案しました。画素深度を離散確率分布として扱う点が中核の新規性です。

- **BEVDet(Huang et al., arXiv:2112.11790, 2021)** が LSS の lift-splat を 3D 物体検出に応用し、データ拡張(BEV 空間での回転・スケールなど)と効率化を加えて実用的な検出精度を達成しました。LSS 系を検出タスクで広めた仕事です。

- **BEVDepth(Li et al., AAAI 2023, arXiv:2206.10092)** が LSS 最大の弱点である「深度予測の不正確さ」に正面から取り組みました。LiDAR の点群を投影して得た正解深度を教師信号にし、深度予測ヘッドを明示的に学習させます(depth supervision)。さらにカメラ校正を意識した深度補正(camera-aware)を加えました。「明示的深度の質が BEV の質を左右する」ことを定量的に証明した点で重要です。

- 対照として **BEVFormer(Li et al., ECCV 2022)** は深度を明示的に予測せず、BEV クエリの空間クロスアテンションで複数カメラを統合する暗黙投影方式です。深度誤差の直接の影響を避ける別解として並び称されます(詳細は [BEVFormer風の暗黙的投影](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0273_bevformer-deformable-attention))。

## 論文の実験結果(定量データ)

LSS 系の評価も主に nuScenes(6カメラと LiDAR を積んだ車で都市を走った大規模データセット)で行われます。指標は **mAP**(検出の当たり具合、0〜1、高いほど良い)と **NDS**(位置・大きさ・向き・速度・属性の誤差まで総合したスコア、高いほど良い)です。

LSS の元論文自体は BEV セグメンテーション(地図上で「道路」「車」などの領域を塗り分けるタスク)で評価され、当時のカメラのみ手法として高い IoU(Intersection over Union、予測領域と正解領域の重なり率、高いほど良い)を報告しました。

検出タスクでの数値は後続が示します。BEVDet は nuScenes で当時のカメラ検出として競争力ある NDS を出し、BEVDepth はそこから NDS をおよそ 0.60 前後まで押し上げたと報告されています。とりわけ重要なのは BEVDepth のアブレーションで、深度の教師信号(depth supervision)を抜くと mAP・NDS が明確に低下することが示されました。これは「LSS 系では深度予測の正確さが性能のボトルネックである」という主張の直接の裏付けです。深度の誤差を測る独自の指標で見ると、教師ありにすることで深度誤差が大きく減り、それが検出精度の改善とよく相関していました。

メモリ・計算の面では、視錐台 `[V, C, D, H, W]` を作るため D を増やすほど負荷が二乗的に上がります。BEVDet 系の後続研究では、この lift-splat の集計を高速化する CUDA 実装(BEVPoolv2 など)が提案され、推論速度が数倍改善した、という報告があります。

## メリット・トレードオフ・限界

メリット。深度という人間にも解釈しやすい中間表現を持つため、デバッグや誤り分析がしやすく、LiDAR による教師あり(BEVDepth)で精度を直接底上げできます。lift-splat の集計はカメラ枚数に対して素直に拡張でき、任意配置の複数カメラを1枚の BEV にまとめやすい設計です。

トレードオフと限界。第一に、深度予測が外れると BEV 上の物体位置が大きくずれます。深度精度がそのまま性能のボトルネックになりやすく、これが BEVDepth が生まれた動機でした。第二に、全画素×全深度段階の視錐台を作るためメモリと計算が重く、深度段数 D を増やすほど負荷が上がります。第三に、単眼からの深度推定は本質的に不良設定問題で、確率分布で緩和しても完全には解消せず、特に遠方(深度が大きく、わずかな誤差が大きな距離誤差になる領域)で精度が落ちます。第四に、地面付近に偏った深度分布になりやすく、空中の物体や急な高低差で弱さが出ることがあります。

## 発展トピック・研究の最前線

深度を改善する方向では、BEVDepth に続いて時間方向の深度整合(連続フレームの一貫性で深度を補正する)や、ステレオ的な手がかりを使う研究が進みました。集計の高速化(BEVPool 系の CUDA 最適化)は実用化に直結します。また、LSS の forward projection と BEVFormer の backward projection を組み合わせ、両者の良いところを取る融合手法も提案されています。さらに、BEV(平面)を超えて 3D 占有グリッド(occupancy、空間をボクセルで区切って物の有無を予測する)へ持ち上げる流れがあり、明示的深度方式は高さ方向を素直に扱える利点から、この占有予測との相性が議論されています。LiDAR と融合する BEVFusion 系では、LSS でカメラを BEV に持ち上げてから LiDAR の BEV 特徴と結合する構成が広く使われています。
