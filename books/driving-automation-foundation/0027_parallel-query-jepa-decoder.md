# 並列クエリ型JEPAデコーダ

## ひとことで言うと
「これから数秒後に世界がどう見えるか」を、未来の各時刻ぶんのベクトルを一気に同時に予測する仕組みです。1秒後・2秒後・3秒後…を順番に1個ずつ作る(自己回帰)のではなく、未来の時刻ごとに用意した「問い合わせ役(クエリ)」を並べ、過去の記憶に一括で注意を向けて、全時刻ぶんの未来特徴を並列に出力します。逐次計算の遅さと、予測誤差が後の時刻ほど積み重なる問題の両方を避けるのが狙いです。

## 直感的な理解
未来の予測を「数珠つなぎ」でやるか「同時に」やるか、を考えてみてください。

数珠つなぎ(自己回帰、autoregression)とは、まず1秒後を予測し、その予測を入力に足してから2秒後を予測し、さらにそれを足して3秒後…と、前の出力を次の入力に使って順番に伸ばすやり方です。文章を書くときに「1単語ずつ次の単語を予測する」言語モデルと同じ発想です。直感的で、各ステップが直前の文脈に正しく依存できるという強みがありますが、2つの弱点があります。

第一に、N個の未来を作るのに計算をN回回さなければならず、待ち時間(レイテンシ)が積み上がります。自動運転は遅延に敏感で、判断が0.1秒遅れるだけで、時速60km/hなら約1.7m進んでしまいます。第二に、誤差が雪だるま式に膨らみます。1秒後の予測が少しズレていると、それを入力に使う2秒後はもっとズレ、3秒後はさらに…と先の時刻ほど誤差が累積します。これは exposure bias(露出バイアス)と呼ばれる、自己回帰モデルに共通する難点です。学習時は各ステップに「本物の過去」を与えるのに、推論時は「自分の不正確な予測」を与えるため、学習と推論の条件がズレて先の時刻が崩れていきます。

そこで「同時に」やる発想が出てきます。未来の各時刻に「私は+1.6秒後の特徴が欲しい」「私は+3.2秒後が欲しい」という小さな問い合わせ役(クエリ)を最初から並べておき、それぞれが過去の記憶を一度だけ参照して、全時刻ぶんを並列に出力します。N回ぶんの計算を1回にまとめ、誤差の連鎖も断ち切ります。これが並列クエリ型デコーダ(parallel query decoder)です。

## 基礎: 前提となる概念
いくつかの用語をほどいておきます。前提知識ゼロでも読めるように丁寧に説明します。

attention(注意機構)とは、たくさんの情報の中から「今の自分に関係が深いものを重視して取り込む」仕組みです。具体的には、自分を表すベクトル query(クエリ、問い合わせ)を作り、参照したい情報それぞれを key(キー、見出し)と value(バリュー、中身)のペアで表します。query と各 key の内積(似ている度合い)を計算し、次元の平方根で割ってスケールを整え、softmax(合計が1になるように正規化する関数)で「合計1の重み」に変換し、その重みで value を加重平均します。これにより「関係が深い情報ほど強く混ぜる」ことが実現できます。式で書けば Attention(Q,K,V) = softmax(QK^T / √d) V で、d は key の次元数、QK^T が「全クエリ × 全キー」の類似度行列です。

self-attention(自己注意)は query・key・value がすべて同じ集合から作られる場合で、cross-attention(交差注意)は query が一方、key と value がもう一方から作られる場合を指します。ここでは未来クエリが query 側、過去の記憶が key/value 側になります。query の本数(=未来時刻数N)と key/value の本数(=履歴の長さ)は別々でよく、出力は必ず query と同じ本数だけ出ます。これが「N本のクエリを入れると、ちょうどN個の未来が出てくる」並列デコーダの根幹です。

JEPA(Joint-Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)とは、「ピクセルそのものを当てる」のではなく「特徴空間(ニューラルネットが画像から抽出した抽象的なベクトルの空間)で、文脈から対象の表現を予測する」自己教師あり学習(self-supervised learning、人間がラベルを付けなくてもデータ自身から学ぶ枠組み)の総称です。なぜ特徴空間なのか。ピクセルを直接当てさせると、葉っぱの揺れや影のちらつきといった「予測しても意味のない細部」まで合わせさせられ、計算が無駄になり表現も汚れます。特徴空間で予測すれば、抽象度の高い「何がどう動くか」だけに集中できます。未来予測を補助タスク(主タスクの運転を助けるために一緒に学ぶ副次的な学習目標)にすると、モデルは「物がどう動くか」という世界のダイナミクスを内部に獲得でき、それが運転の判断に役立つ、という狙いです。

埋め込み(embedding)とは、画像や単語といった生のデータを、意味を保ったまま固定長の数値ベクトルに変換したものです。以降「フレーム埋め込み」は1枚の画像をベクトル化したもの、「特徴」もほぼ同義で使います。

## 仕組みを詳しく
典型的な構成を、tensor の形(shape、軸ごとの大きさ)と数値例で追います。記号は次のとおりです。B はバッチサイズ(同時に処理する系列の本数)、N は予測する未来ステップ数、D は1フレーム埋め込みの次元数、V はカメラ台数、context は過去の記憶(エンコード済みの視覚履歴、または過去N個のフレーム埋め込み)です。

例として N=4(未来時刻 +1.6s / +3.2s / +4.8s / +6.4s)、D=256、V=7 を取ります。

ステップ1: N個の未来クエリを用意する。各クエリは学習可能なベクトルで、「何秒後を指すか」を区別させるため時間位置埋め込み(temporal positional embedding、時刻ごとに割り当てた位置ベクトル)を足します。バッチ方向に展開して shape は `[B, N, D]`。位置埋め込みがないと4個のクエリが見分けられず、全部同じ未来を出してしまうので必須です。直感的には、4本のクエリは「最初は同じ初期値だが、それぞれ違う時刻ラベルを背負って出発する社員」のようなもので、ラベルがあるからこそ別々の仕事をします。

ステップ2: 過去の記憶 context に対して cross-attention を1層(あるいは数層)かける。

```
dec = CrossAttention(query=future_queries, key=context, value=context)  # [B, N, D]
```

中身を言葉にすると、各未来クエリが「過去のどのフレームや記憶を見れば自分の時刻を予測できるか」を重みで選び、加重平均します。たとえば+1.6秒後を担うクエリは直近のフレームを強く見て、+6.4秒後を担うクエリは少し長い動きの傾向を見る、といった配分を学習で獲得します。N個のクエリは互いに独立に、同時に計算されます。デコーダを多層にするなら、クエリ同士の self-attention(未来時刻どうしの整合を取る)と cross-attention(過去を見る)を交互に積む DETR 流の構成も使えますが、最小構成は cross-attention 1層です。

ステップ3: 線形射影(linear projection、行列を掛けて次元を合わせる操作)で、ターゲット側の特徴空間に合わせます。たとえばカメラごとに平均した pooled 特徴に揃えるなら shape は `[B, N, V*C]` のようになります。

ASCII図で自己回帰と並列クエリを並べます。

```
自己回帰(逐次):
  pred(+1.6s) → 入力に足す → pred(+3.2s) → … → pred(+6.4s)   N回直列、誤差累積、レイテンシ N×

並列クエリ:
              ┌→ query(+1.6s) ┐
  context ──→ ┤  query(+3.2s) ├─ cross-attn(1回) → [pred+1.6, +3.2, +4.8, +6.4] 同時、1×
              │  query(+4.8s) │
              └→ query(+6.4s) ┘
```

計算量の数値例。cross-attention 1層のコストはおよそ N × (context長) × D に比例します。N=4、context長=28(4フレーム×7カメラ)、D=256 なら 4×28×256 ≈ 2.9万回ぶんの内積で、現代の GPU では誤差の範囲のコストです。これに対し自己回帰でデコーダを N=4 回展開すると、各回が直前に依存して直列に並ぶため、計算総量が同程度でも壁時計時間(実際に待つ時間)は約4倍になります。レイテンシ予算がミリ秒単位の実車では、この直列性こそが致命的です。

なぜこの設計なのか。第一に、未来時刻と出力の対応が固定(クエリ k が必ず時刻 t_k を出す)なので、後述の DETR のような「どのクエリがどの正解に対応するか」のマッチング処理が不要になり、学習が単純になります。第二に、cross-attention 1層+線形射影だけなので軽量で、推論が速い。代替案として GRU(Gated Recurrent Unit、系列を逐次処理する小さな回帰型ネットワーク)で N ステップ展開する方法もあり、パラメータ数自体は小さいのですが、逐次依存が残るためレイテンシ面で並列クエリが好まれます。

## 手法の系譜と主要論文
この発想は、検出と自己教師あり予測の2つの系譜が合流したものです。

出発点は DETR(DEtection TRansformer、Carion et al., 2020, ECCV, arXiv:2005.12872)です。それ以前の物体検出は、大量の候補枠を出して非最大抑制(NMS、重複する枠を消す後処理)で間引く方式が主流でした。DETR は「学習可能なオブジェクトクエリを N 個並べ、Transformer デコーダで画像特徴に cross-attention して N 個の物体を並列に出力する」発想に切り替え、逐次処理も NMS も不要にしました。この「集合を一度に出すクエリ」の考え方が、未来時刻を一度に出す本トピックのクエリと同型です。ただし DETR は「どのクエリがどの物体に対応するか」が事前に決まらないため、二部マッチング(Hungarian matching、予測と正解を最適に1対1対応させる手法)が必要でした。未来予測では時刻と出力の対応が固定なのでこの複雑さが消えます。

DETR の弱点(収束に長いエポックを要する、小物体に弱い)に答えたのが Deformable DETR(Zhu et al., 2021, ICLR, arXiv:2010.04159)です。各クエリが画像特徴の全位置を見るのではなく、少数の「注目すべき点」だけを学習的に選んで見る deformable attention を導入し、学習を約10分の1のエポックに短縮しました。これは「クエリが context のどこを見るか」をスパースにして効率化する発想で、未来予測デコーダでも履歴が長大なときに有効な発展方向です。

並行して自己教師あり学習の側で JEPA が立ち上がります。I-JEPA(Image-JEPA、Assran et al., 2023, CVPR, arXiv:2301.08243)は、画像の一部(コンテキスト)から別の領域(ターゲット)の特徴を、ピクセルではなく特徴空間で予測する predictor(予測器)を提案しました。本トピックのデコーダは、まさにこの「特徴を予測する predictor」の時間版に当たります。続く V-JEPA(Video-JEPA、Bardes et al., 2024, arXiv:2404.08471)は JEPA を動画へ拡張し、時空間のマスク領域の特徴を予測させて「動き」の表現を獲得しました。未来予測デコーダの損失設計は、この V-JEPA の特徴回帰の流れを汲みます。

軌道予測(他車の未来経路を当てるタスク)では、クエリ並列の発想が一足早く標準化しました。Wayformer(Nayakanti et al., 2022, arXiv:2207.05844)や MTR(Motion TRansformer、Shi et al., 2022, NeurIPS, arXiv:2209.13508)は、「複数の学習可能なモードクエリ」で多様な未来軌道を並列に出します。MTR は意図ごとのクエリで Waymo の動作予測を改善し、クエリ並列が「単一の未来を当てる」より「ありうる複数の未来を出す」のに向くことを示しました。本トピックの時間クエリは、これの「時刻方向版」と見ることもできます。

運転ドメインの世界モデルでは、GAIA-1(Hu et al., 2023, Wayve, arXiv:2309.17080)や MILE(Hu et al., 2022, NeurIPS, arXiv:2210.07729)が「未来予測を学ぶと運転に有用な表現が得られる」ことを示しました。ただし GAIA-1 は離散トークンを自己回帰的に生成する重い生成モデルです。並列クエリ型デコーダは、この「未来予測で表現を学ぶ」狙いは引き継ぎつつ、生成を並列・軽量にしてレイテンシを抑える方向の割り切りと言えます。

## 論文の実験結果(定量データ)
DETR は COCO 物体検出ベンチマーク(80カテゴリ・約12万枚の画像で検出精度を測る標準ベンチ)で、当時の標準だった Faster R-CNN と同等の精度を達成しました。報告では DETR が約 42 AP(Average Precision、検出の総合精度指標。0〜100で高いほど良く、複数の重なり閾値での平均適合率を測る)で、同等規模の Faster R-CNN(約42 AP)とほぼ並びます。重要なのは精度そのものより「手作りの候補枠生成も NMS も使わず、純粋なクエリ集合予測で同等に届いた」点で、これが「並列クエリで集合を出す」設計の有効性を示しました。一方で大きな弱点として、収束に約500エポック(Faster R-CNN の数倍)を要し、小物体の AP が低い傾向が報告されました。アブレーション(ある要素を抜くと性能がどう落ちるかを見る分析)では、デコーダ層を1層だけにすると AP が数ポイント落ち、複数層のクエリ間 self-attention が重複検出の抑制に効くことが確認されています。

Deformable DETR はこの収束問題に答え、約50エポックで DETR を上回る 43〜46 AP に到達し、特に小物体 AP を大きく改善しました。学習時間を1桁縮めたことが、その後のクエリ型検出器の普及を後押ししました。

I-JEPA は ImageNet の線形評価(linear probing、凍結した表現の上に線形分類器だけを学習して精度を測る、表現の質の標準的な測り方)で、ピクセル再構成型の手法より少ない計算で高い精度を出しました。報告では、ViT-Huge 規模で線形評価トップ1精度 約77〜79% に届き、同等のデータと計算量で MAE(Masked Autoencoder、ピクセルを再構成する自己教師あり手法)を上回りつつ、事前学習を約10分の1の計算で済ませたとされます。特徴空間での予測が「無駄な細部を当てない」ぶん効率的であることを示した結果です。V-JEPA は動画の動作認識(Kinetics-400 や Something-Something-v2 などで動画が何の行動かを当てるタスク)の凍結特徴評価で強い結果を示し、特徴空間の時空間予測が動きの表現獲得に有効であることを裏づけました。これらは「未来を特徴空間で予測する」設計選択の一般的な妥当性の根拠になります。

軌道予測側では、MTR が Waymo Open Motion Dataset でその年の挑戦課題で上位を占め、複数モードのクエリ並列が多峰な未来の表現に有効であることを定量的に示しました。評価には minADE(複数予測のうち最も正解に近いものの平均変位誤差。小さいほど良い)や Miss Rate(どの予測も正解から一定距離以上外れた割合)が使われ、単一モード予測より一貫して改善が報告されています。

## メリット・トレードオフ・限界
メリットは明快です。N時刻を同時生成するので逐次レイテンシがなく、自動運転の遅延要件に有利です。自己回帰の誤差累積(exposure bias)を避けられ、cross-attention 1層+線形射影のみで軽量です。時刻と出力が固定対応なのでマッチング不要で学習も単純です。

トレードオフと限界もあります。第一に、各未来時刻が独立に予測されるため、時刻間の連続性(なめらかに繋がる動き)を陽に強制しません。+1.6秒と+3.2秒の予測が物理的に矛盾していても、それを罰する仕組みが標準では入りません。必要なら時刻間の差分を罰する平滑化損失や、クエリ間 self-attention で時刻の整合を取る層を足します。第二に、N が固定です。可変長の未来や任意時刻の問い合わせを出すには設計変更(時刻を連続値で条件付けるなど)が要ります。第三に、予測がターゲットの特徴空間に正しく射影できているか(次元・スケールの整合)の確認が必要で、ここがズレると損失が下がらないか、逆に簡単に下がりすぎて崩壊(representation collapse、予測もターゲットも定数に潰れて損失だけ0になる失敗)します。

研究上の未解決課題として、未来は本来一意に決まらない(他車がどう動くかは確率的)のに、単一の予測を出す決定的なデコーダは「複数のありうる未来」を1つに平均してぼかしてしまう問題があります。確率的な未来(多峰性、複数のピークを持つ分布)をどう扱うかは活発な研究テーマで、MTR のように「モード×時刻」の二次元クエリへ拡張する方向が考えられます。

## 発展トピック・研究の最前線
並列クエリの発想は検出を越えて広がっています。動作予測では、複数の未来候補をクエリで並列に出す手法(MTR、Wayformer など)が標準化しつつあり、「1つの未来を当てる」のではなく「ありうる複数の未来とその確率を出す」方向が主流です。これを特徴予測デコーダに持ち込めば、多峰性の課題に応えられる可能性があります。クエリを「時刻 × モード」の格子にし、モードごとに分岐した未来特徴を並列出力する設計が自然な拡張です。

また、世界モデル(world model、環境のダイナミクスを学んで頭の中でシミュレーションする仕組み)の文脈では、特徴空間で未来を予測する JEPA 系と、ピクセル/トークンで生成する拡散モデル・自己回帰モデル系が競っています。前者は軽くて計画に使いやすく、後者は人間が見て検証できる映像を生成できる、という棲み分けが議論されています。並列クエリ型デコーダはこの「軽い特徴予測」陣営に属し、いかに豊かな表現を安く学ぶかが焦点になります。さらに、Deformable DETR 流のスパース注意を時間方向に持ち込み、長大な履歴のうち各未来クエリが見るべき少数のキーだけを選ぶことで、長期予測のコストを抑える方向も有望です。
