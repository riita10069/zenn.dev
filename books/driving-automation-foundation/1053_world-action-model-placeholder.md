# World / Action Model（低頻度推論ブランチ）

## ひとことで言うと
World / Action Model（世界・行動モデル）とは、低頻度（例えば1Hz）でマルチビュー画像を見て、それを「視覚履歴」という1本のベクトルに圧縮し、さらに「少し先の世界がどう見えるか」を特徴空間で予測する、二系統アーキテクチャの推論（System2）側モジュールです。圧縮した視覚文脈を高頻度の反射系統に供給して時間的な状況理解を補い、同時に未来予測の誤差を自己教師あり学習の信号として使い、ラベルなしで世界の動き方を学びます。

## 直感的な理解
反射的に動く制御系統は、入力の1つとして「過去の視覚特徴を圧縮したベクトル（視覚履歴）」を受け取ることがよくあります。なぜ視覚履歴が欲しいかというと、運転は今この瞬間の1枚だけでは判断できないからです。「さっきまで赤信号だった」「前の車がだんだん減速している」「右からゆっくり近づいてくる自転車がいる」といった時間的な文脈は、過去の画像列を見ないと分かりません。

ところが、この時間的文脈を毎フレーム重く処理するのは無駄です。シーンの大局的な理解（どんな環境で、何が起きていて、どう変化しているか）は1秒に1回まとめて行えば十分なことが多い。そこで、低頻度でまとめて重い視覚理解を行い、その成果を1本のベクトルに固めて反射側に渡す——この役を担うのが World / Action Model です。

さらにこのモジュールには「将来の特徴を予測し、それを学習の信号にする」狙いがあります。人間の正解ラベル（どこに進むべきか）が無くても、「数フレーム後の世界の見え方」を当てさせれば、世界がどう動くかという物理的・因果的な構造そのものを学べます。これが「世界モデル」と呼ばれる所以であり、Yann LeCun が提唱する JEPA の核心です。直感的には、未来を予測できるということは世界の動き方を理解しているということだ、という前提に立っています。

## 基礎: 前提となる概念
「自己教師あり学習（self-supervised learning, SSL）」を理解しましょう。通常の教師あり学習は人間がラベル（正解）を付けたデータで学びます。自己教師ありは、データ自身から自動で「問題と答え」を作り、ラベル付けの手間なしに学ぶ方式です。例えば「画像の一部を隠して残りから当てさせる」「過去のフレームから未来のフレームを当てさせる」など。運転データは大量に集まる一方で人間のラベル付けは高コストなので、自己教師ありは非常に魅力的です。

「特徴空間（feature space）」と「ピクセル空間」の違いも重要です。ピクセル空間は生の画像そのもの（各画素のRGB値）。特徴空間は、ニューラルネットがその画像を要約した抽象ベクトルの空間です。「車が前進している」「信号が赤」といった意味は特徴空間に表れますが、雲の細かい形や葉の揺れといった予測不能な細部はピクセル空間にしかありません。未来予測を特徴空間で行えば本質だけ学べ、ピクセル空間で行うと予測不能な細部に労力を取られます。

「凍結（frozen）」とは、あるネットワークのパラメータを学習で更新しない（`requires_grad=False`）ことです。これには2つの意味があります。1つは計算が軽くなること（逆伝播しない）。もう1つは、後述する collapse を防ぐために予測ターゲットを安定させることです。

「collapse（崩壊）」は自己教師あり学習特有の失敗です。「予測した特徴」と「実際の特徴」を近づけるよう学習すると、何の制約もなければ「全部を同じ定数ベクトルにする」のが最も簡単な解になります。常に同じベクトルを出せば予測誤差はゼロですが、何の情報も持たない無意味な表現です。これが collapse で、これを防ぐ工夫が自己教師あり手法の主戦場です。

## 仕組みを詳しく
World / Action Model の典型的なパイプラインは次の通りです。

```
multi-cam画像 @1Hz
  → Vision Backbone（凍結・反射側と共有可）→ 各視点の特徴マップ
  → View Pooling → フレームごとに1本の埋め込み
  → Temporal Encoder → 視覚履歴ベクトル（反射側へ供給）
  → Future Predictor → 未来フレームの特徴を予測
  → 実際の未来フレームを凍結Backboneに通した特徴と比較
  → Future Feature Prediction Loss（JEPA損失）
```

コンポーネントごとに噛み砕きます。

第1に Vision Backbone（凍結して再利用）。反射系統と同じ backbone を使いつつ、`requires_grad=False` にして学習で更新しないのが一般的です。凍結の理由は3つあります。(a) 反射側が軌跡やBEVのために backbone を学習している場合、世界モデルの予測損失も同じ backbone を動かすと2つの目的が綱引きして表現が壊れる。(b) JEPA は予測ターゲット側が固定されていないと collapse するため、凍結 backbone が安定したターゲット特徴空間を与える。(c) backbone を逆伝播しないので計算が軽い。

第2に View Pooling。複数視点の特徴マップを、1フレームあたり1本の埋め込みベクトルにまとめます。軽い cross-attention（1つのクエリで全視点を集約する attention pooling）がよく使われ、BEV のような幾何投影を省いて軽量化することもあります。

第3に Temporal Encoder。各フレームの埋め込みを時間方向に圧縮します。GRU（ゲート付き回帰型ユニット、系列を逐次に要約する RNN の一種）や temporal attention-pool が使われます。この出力が反射系統に渡す視覚履歴ベクトルになり、しばしば「ゼロ埋めのプレースホルダ」だった入力を意味ある情報に置き換えます。

第4に Future Predictor。過去の文脈から「少し未来のマルチビュー特徴」を予測する小さなデコーダです。

第5に Future Feature Prediction Loss（JEPA）。予測した未来特徴と、凍結 backbone が実際の未来フレームから作った特徴とを比べ、そのずれ（典型的には L2 距離やコサイン距離）を損失にします。最重要なのは「ピクセルではなく特徴空間で比べる」点です。

数式で書くと、コンテキスト（過去）から予測した未来特徴を $\hat{z}_{t+k} = g_\phi(\text{Enc}(x_{\le t}))$、実際の未来フレームを凍結ターゲットエンコーダに通した特徴を $z_{t+k} = \bar{f}_\theta(x_{t+k})$ とすると、損失は $\mathcal{L} = \| \hat{z}_{t+k} - \text{sg}(z_{t+k}) \|^2$ です。ここで $g_\phi$ は予測器、$\text{Enc}$ はコンテキストエンコーダ、$\bar{f}_\theta$ は凍結（または EMA 更新の）ターゲットエンコーダ、$\text{sg}$ は stop-gradient（その項に勾配を流さない操作）を表します。stop-gradient とターゲット凍結が collapse を防ぐ鍵です。なぜなら、ターゲット側に勾配を流すと「ターゲットも予測も一緒にゼロベクトルへ寄せる」近道が開いてしまうからです。

## 手法の系譜と主要論文
世界モデルの源流は Ha と Schmidhuber の "World Models"（NeurIPS 2018）です。観測を VAE で圧縮し、RNN で未来の潜在状態を予測し、その潜在空間上で行動方策を学ぶ枠組みを示しました。「環境の内部モデルを学び、その中で計画・学習する」という発想を確立しました。

次に自己教師あり表現学習の系譜です。Grill らの BYOL（"Bootstrap Your Own Latent", NeurIPS 2020, arXiv:2006.07733）は、ネガティブサンプル（対比学習で使う「異なる画像」）を一切使わず、オンラインネットワークがターゲットネットワーク（オンラインの EMA = 指数移動平均で緩やかに更新するコピー）の出力を予測する形で、collapse を回避しつつ強い表現を学べることを示しました。stop-gradient と非対称な予測器の組み合わせが鍵という知見は、後続すべてに継承されています。

そして LeCun の "A Path Towards Autonomous Machine Intelligence"（2022）が JEPA（Joint-Embedding Predictive Architecture）を中核理論として提示しました。生成的（ピクセル再構成）より特徴空間での予測のほうが効率的で頑健、という主張です。これを画像で具体化したのが Assran らの I-JEPA（"Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023, arXiv:2301.08243）で、画像の一部（コンテキスト）から別の部分（ターゲット）の特徴を予測します。ピクセル再構成や手作りのデータ拡張に頼らず意味的な表現を効率良く学べることを示しました。さらに Bardes らの V-JEPA（2024）が動画に拡張し、時間方向の予測（過去フレームから未来・マスク領域の特徴を予測）で運動・ダイナミクスの表現を学べることを示しました。World / Action Model はこの V-JEPA の発想を、マルチビュー運転データと低頻度処理に適用した位置づけです。

自動運転の生成的世界モデルとしては Wayve の GAIA-1（Hu et al., 2023, arXiv:2309.17080）が代表的で、行動条件付きで未来の運転映像そのものを生成します。これはピクセル空間で予測する対極の選択で、解釈性・データ生成に強い一方、計算が重く予測不能な細部にも労力を割きます。JEPA 系は逆に特徴空間で軽く本質だけを狙う選択であり、両者は世界モデルの2大潮流を成しています。

## 論文の実験結果（定量データ）
I-JEPA（Assran et al., 2023）は ImageNet-1K（1000クラス・約128万枚の画像分類ベンチマーク）で評価されました。自己教師ありで学んだ表現を、線形評価（学習済み特徴の上に線形分類器だけを足して分類精度を測る、表現の良さの標準指標）したところ、ピクセル再構成系の MAE（Masked Autoencoder）などと同等以上の精度を、はるかに少ない事前学習エポック数（おおむね数分の1の計算量）で達成したと報告されています。具体的には ViT-Huge を用いて ImageNet で 70%台後半の線形評価精度を、競合手法より少ない GPU 時間で得たとされ、「特徴空間予測は計算効率が良い」という JEPA の主張を裏付けました。これが World / Action Model が特徴空間予測を選ぶ実証的根拠です。

V-JEPA（Bardes et al., 2024）は Kinetics-400（動画行動認識ベンチマーク）や Something-Something-v2（物体の動きの方向まで区別する難しい行動認識）で評価され、動画から自己教師ありで学んだ表現が、凍結評価（backbone を凍結したまま下流タスクを解く）で強い性能を示しました。特に、動きの細かい区別を要する Something-Something-v2 で、ピクセル予測系より動的特徴の表現が優れることが報告されています。これは「未来特徴予測がダイナミクスを捉える」ことの直接的証拠で、運転のような動的シーンへの応用を後押しします。

BYOL（Grill et al., 2020）の重要なアブレーションは「ターゲットネットワークの EMA を外して同一ネットワークにすると collapse する」という結果です。EMA（あるいは凍結）によるターゲットの安定化が collapse 回避に必須であることを定量的に示し、World / Action Model が backbone を凍結する設計の根拠になっています。凍結はこの EMA をさらに単純化した（移動平均すら止めた）選択で、反射側がすでに backbone を学習している状況では安定したターゲットを得やすい一方、ターゲット表現が下流タスクに最適化されないという制約も持ちます。

V-JEPA の具体的な数値感としては、Something-Something-v2 の凍結評価（attentive probing）で 70% 前後の top-1 精度を報告し、同規模のピクセル予測系（動画版 MAE 等）を動きの理解を要するタスクで上回ったとされます。Kinetics-400 でも凍結バックボーンのまま 80% 前後の行動認識精度を達成し、「ファインチューンせずとも特徴がそのまま使える」ことを示しました。凍結評価で強いということは、学んだ表現が特定の下流タスクに過適合せず汎用的であることを意味し、これは反射側に渡す視覚文脈として再利用する用途と相性が良い性質です。

なお、自動運転の文脈での世界モデルの効果は、下流の計画 L2 誤差や衝突率（[System1/System2分離（Reactive/Reasoning）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1017_system1-system2-architecture) 参照）で測られますが、JEPA 的な世界モデルを運転計画に接続して定量改善を厳密に示した研究はまだ発展途上であり、報告される数値は手法・データセット依存が大きい点に注意が必要です。一方で生成的世界モデル側では、GAIA-1（Hu et al., 2023）が大量の運転動画から行動・テキスト条件付きで数十秒スケールの一貫した未来映像を生成できることを示し、世界モデルが運転シーンの構造（車両の運動、信号、道路レイアウト）を内部に獲得しうることの定性的証拠を与えています。特徴空間予測（JEPA）と生成（GAIA）は、計算効率と解釈性・データ生成能力のトレードオフを両端で代表しています。

## メリット・トレードオフ・限界
メリットは、低頻度で重い視覚理解を行いその成果を軽い反射側が再利用できる計算効率、凍結 backbone と小さなヘッドだけ学習する追加コストの小ささ、ラベル不要の自己教師ありで世界の動き方を学べること、そして視覚履歴のプレースホルダを意味ある時間文脈に置き換えられることです。

トレードオフと限界は重大です。第一に collapse のしやすさで、backbone 凍結・stop-gradient・非対称予測器などの安定化が必須です。第二に、backbone の凍結方式に設計上の選択が残ります。専用の凍結コピーを持つか、反射側のライブ backbone を stop-gradient で共有するか、EMA で緩やかに更新するか、それぞれ安定性・メモリ・表現品質のトレードオフが異なります。第三に、低頻度ゆえ視覚履歴は最大で1更新周期ぶん（例えば1秒）古くなりえます。第四に、特徴空間予測は「何を予測しているか」が人間に見えにくく、解釈性で生成的世界モデルに劣ります。

研究上の未解決課題としては、(1) 特徴空間予測の表現が本当に下流の運転計画を改善するかの厳密な検証、(2) collapse を確実に防ぎつつ表現の情報量を最大化する正則化（VICReg のような分散・共分散正則化との組み合わせ）、(3) マルチビュー間の幾何的整合をどこまで明示的にモデル化すべきか、(4) 低頻度処理の遅延が安全性に与える影響の定量評価、などがあります。

## 発展トピック・研究の最前線
最前線では、JEPA 系の正則化を強化する VICReg（分散・不変・共分散の3項で collapse を防ぐ）や、行動を条件に入れた action-conditioned world model（自車の行動を入力に未来を予測し、計画と一体で学ぶ）への拡張が進んでいます。また、生成的世界モデル（GAIA 系）と特徴空間予測（JEPA 系）を融合し、粗い予測は特徴空間で軽く、必要なときだけ生成で詳細化するハイブリッドも検討されています。運転特化では、マルチビュー＋時間＋行動を同時に扱う大規模世界モデルを大量の無ラベル走行データで事前学習し、少量のラベルで計画にファインチューンする「基盤モデル的世界モデル」が有力な方向です。さらに、世界モデルを使った closed-loop 評価（学習した世界モデル内で方策を仮想走行させて安全性を測る）は、実車・シミュレータに頼らない評価手段として注目されています。
