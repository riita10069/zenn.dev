# Deformable Cross-Attention

## ひとことで言うと
注意機構(アテンション)の効率版です。「BEV地図や画像のどこを見るか」を全部のマスから探すのではなく、まず大まかな基準点(参照点)を決め、そこから「学習で覚えた少しのズレ(オフセット)」だけ動かした数点だけを見ます。その数点の特徴をバイリニア補間(`grid_sample`)で取ってくるので、地図がどれだけ大きくても計算量が候補数に線形で済みます。高解像度の特徴を扱うための、現代の知覚・計画モデルの基幹部品です。

## 直感的な理解

普通のアテンション(フルアテンション)は「全部を見て重み付けする」仕組みです。これは表現力が高い反面、候補が増えると計算量が爆発します。例えば200×200=4万マスのBEVに対し、4万個のクエリがそれぞれ4万個の候補を見ると、4万 × 4万 = 16億ペアの計算が必要です。450×300なら180億ペアを超え、メモリも計算時間も現実的でなくなります。アテンションの計算量は系列長Nに対しO(N²)で増えるため、空間解像度を上げると二次関数的に重くなるのです。

しかし冷静に考えると、ほとんどのペアは無駄です。BEVのあるマスにとって本当に関係するカメラ画像上の位置は、そのマスを各カメラに投影した「だいたいここ」という近傍だけのはずで、画像の反対側まで総当たりする意味はありません。物体検出でも、あるオブジェクトクエリが見るべきは画像全体ではなく、その物体がいそうな一帯だけです。

この「本当に関係ある場所は各クエリの近傍の数点だけ」という洞察を形にしたのが変形アテンション(deformable attention)です。「変形(deformable)」とは、固定の格子ではなく、学習でズレ(オフセット)を加えて見る場所を動的に変形させることを指します。クエリごとに「参照点(reference point)」を起点に、学習した数個のオフセット先だけをサンプリングして情報を集めます。これは画像認識における可変形畳み込み(deformable convolution)のアイデアをアテンションに移植したものとも言えます。

## 基礎: 前提となる概念

「アテンション(attention)」は Transformer(Vaswani et al., NeurIPS 2017, arXiv:1706.03762)の中核です。クエリ(Q、欲しい側)・キー(K、各候補の見出し)・バリュー(V、各候補の中身)の3つを使い、クエリと各キーの類似度をソフトマックス(合計1になるよう正規化する関数)で重みにして、バリューの重み付き和を取ります。式で書くと softmax(QK^T/√d)V です。ここで d は次元、√d はスケーリングで、内積が大きくなりすぎてソフトマックスが飽和し勾配が消えるのを防ぐためのものです。フルアテンションでは、Q がN個、K/V がN個あると、QK^T だけでN×Nの行列ができ、これが計算量とメモリのボトルネックになります(O(N²))。

「マルチヘッド(multi-head)」は、Q/K/Vを複数の小さな部分空間(ヘッド)に分け、各ヘッドで独立にアテンションを計算してから結合する仕組みです。各ヘッドが「色の手がかり」「形の手がかり」のように別々の観点で注意を向けられるため、表現力が上がります。

「クロスアテンション(cross-attention)」は、QとK/Vが別々のデータから来る場合を指します。ここでは Q が「BEVクエリ」や「エゴクエリ」、K/V が「カメラ画像特徴」や「BEV特徴」です。セルフアテンション(QもK/Vも同じデータ)と対になる概念です。

「バイリニア補間(bilinear interpolation)」は、整数でない座標(例: 行3.7, 列5.2)の値を、周囲4つの整数格子点の値から距離に応じて加重平均で求める方法です。連続的な座標でサンプリングでき、しかも微分可能(座標に関して勾配が流れる)なので、オフセットを学習できます。この微分可能サンプリングの源流は Spatial Transformer Networks(Jaderberg et al., NeurIPS 2015, arXiv:1506.02025)で、PyTorch では `F.grid_sample` が担います。

## 仕組みを詳しく

変形クロスアテンションは、次の3部品でできています。

1. 参照点(reference point): クエリが「だいたいこのへんを見ればいい」という基準位置。BEVの場合、各マスの3D実世界座標をカメラ画像へ投影して得る。検出の場合、クエリが予測した物体中心。
2. 学習オフセット(learned offset): 参照点から「実際に見る点」へのズレ。クエリ自身から線形層で予測する小さな数値で、学習で最適化される。「参照点はこの位置だが、実際は少し右上を見たほうが良い」を学ぶ。
3. アテンション重み(attention weight): 各サンプリング点をどれくらい重視するか。これもクエリから直接予測し、ソフトマックスで正規化する。フルアテンションのようにキーとの内積で計算するのではなく、クエリだけから直接出すのが変形アテンションの特徴(これがO(N²)を回避する核心)。

処理を形(shape)つきで追います。N をクエリ数、P を1クエリあたりのサンプリング点数(典型 4)とします。

ステップ1。各クエリから、見る点のオフセットと重みを予測する。
```python
offsets = self.sampling_offsets(queries)        # [B, N, P*2]
offsets = offsets.reshape(B, N, P, 2) * offset_scale  # ズレを小さく抑え学習安定化(例 0.1)
attn_weights = self.attention_weights(queries).softmax(dim=-1)  # [B, N, P] 各点の重視度
```
各クエリにつき P 個の点だけを見ます。N=40000, P=4 なら 16万点。フルアテンションの 40000 × 40000 = 16億ペアに比べ桁違いに少ない。

ステップ2。参照点(投影済み)にオフセットを足し、実際に見る位置を決める。
```python
sample_locs = ref_points + offsets    # [B, N, P, 2]  参照点 + 学習ズレ
sample_grid = sample_locs * 2 - 1     # grid_sample の [-1, 1] 座標系へ変換
```

ステップ3。その位置の特徴をバイリニア補間で拾う。
```python
sampled = F.grid_sample(
    feat, sample_grid, mode='bilinear',
    padding_mode='zeros', align_corners=False
)  # [B, C, N, P]
```
`mode='bilinear'` が「整数でない座標の値を周囲4ピクセルから滑らかに補間する」指定。`padding_mode='zeros'` は画像外を指したら0を返す指定です。

ステップ4。拾った特徴を重みで合算する。マルチカメラなら、見えているカメラ全体で正規化する。
```python
weighted = (sampled * attn_weights.unsqueeze(1)).sum(dim=3)  # 重み付き合算
output = weighted / visible_count.clamp(min=1.0)             # 可視カメラ数で平均
```

計算量の比較を数字で(N=135000, P=4 の場合):

- フルアテンション: クエリ数 × 候補数 = 135000 × 135000 ≈ 182億回。候補が増えると2乗で増える。
- 変形アテンション: クエリ数 × サンプリング点数 = 135000 × 4 = 54万回。候補が増えても点数は固定なので線形。

before / after:

- before(フルアテンション): 全候補を総当たりで見る。O(N²)で高解像度は破綻。
- after(変形アテンション): 参照点周りの数点だけ見る。O(N×P)で線形。高解像度でも回る。

なぜ重みをキーとの内積でなくクエリ直接予測にするのか、という設計の急所も押さえます。フルアテンションが重い根本原因は QK^T のN×N行列です。変形アテンションは「どの点を見るか(オフセット)」も「どれだけ重視するか(重み)」もクエリだけから出すことで、このN×N行列を一切作りません。代償として、見る場所をデータ全体との比較ではなくクエリの予測に賭けることになり、参照点が大きく外れていると重要情報を取りこぼします。

実際の Deformable DETR では、これに「マルチスケール(複数解像度の特徴マップから同時にサンプリング)」と「マルチヘッド(複数の観点)」が加わります。1つのクエリが、低解像度から高解像度までの複数特徴マップ上で、それぞれ参照点周りの数点を見る。これにより、大きな物体は粗い特徴、小さな物体は細かい特徴、と使い分けられます。なお `offset_scale`(オフセットを抑える係数)が必要なのは、学習初期にオフセットが暴れて参照点から遠く飛ぶと、無関係な場所を見てしまい学習が発散しやすいためで、初期は参照点近傍に留めるのが安定化の定石です。

なお「参照点 + 学習オフセット + バイリニア補間」というパターンは、BEVを作るとき(各BEVクエリがカメラを参照)にも、軌跡を出すとき([TrajectoryPlanner (ego query + GRU)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0139_trajectory-planner-ego-query-gru) でエゴクエリがBEVを参照)にも同一の部品で使い回せます。新たな依存ライブラリを足さずに実装できるのも実務上の利点です。

## 手法の系譜と主要論文

DETR(Carion et al., ECCV 2020, arXiv:2005.12872)が出発点です。物体検出を集合予測として定式化し、学習可能クエリとフルアテンションで実現しました。しかしフルアテンションゆえに収束が極端に遅く(500エポック規模)、計算量の都合で低解像度の特徴しか使えず、小物体の検出が苦手でした。

Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)が、まさにこの2つの弱点を変形アテンションで解決しました。各クエリの参照点周りの少数点だけをサンプリングし、計算量を線形化。これにより高解像度・マルチスケール特徴を扱えるようになり、収束も大幅に高速化しました。変形アテンションの直接の原典であり、本ノートの中核です。

BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)が、変形アテンションを2方向に展開しました。空間クロスアテンション(Spatial Cross-Attention、各BEVクエリが対応する3D点を各カメラに投影し周辺をサンプリング)と、時間セルフアテンション(Temporal Self-Attention、過去BEVを参照)です。マルチカメラ・時系列をすべて変形アテンションで線形コストにつなぎ、BEV知覚の標準形を作りました。

UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)は、変形アテンションをBEVエンコーダから下流の計画ヘッドまで一貫して使い、知覚から計画までを線形コストのアテンションで貫きました。これにより、高解像度BEVを保ったまま end-to-end 学習が現実的になりました。

Deformable Attention Transformer(DAT, Xia et al., CVPR 2022, arXiv:2201.00520)は、検出・知覚を越えて、画像分類バックボーンのセルフアテンションそのものに変形サンプリングを持ち込み、計算効率と精度を両立しました。技術的源流としては、微分可能サンプリングを導入した Spatial Transformer Networks(Jaderberg et al., NeurIPS 2015, arXiv:1506.02025)と、アテンション機構そのものを定義した Transformer(Vaswani et al., NeurIPS 2017, arXiv:1706.03762)の交点に位置するのが、この変形アテンションだと言えます。

## 論文の実験結果(定量データ)

Deformable DETR の評価は COCO 物体検出データセット(80カテゴリ、約12万枚の訓練画像)で行われました。指標は mAP(mean Average Precision、IoU しきい値 0.5〜0.95 にわたる平均適合率、高いほど良い)です。IoU(Intersection over Union)は予測枠と正解枠の重なり面積の割合で、これがしきい値を超えれば正解とみなします。

報告によると、Deformable DETR は元の DETR と同等以上の mAP(ResNet-50 バックボーンで約43〜46 AP)を達成しながら、訓練エポック数を約500から約50へ、つまり約10分の1に短縮しました。さらに DETR が苦手だった小物体(AP_small)で大きく改善しました(DETR の小物体APが20台前半だったのに対し、Deformable DETR は20台後半へ)。これは「高解像度・マルチスケール特徴を扱えるようになった」直接の効果で、小さな物体は高解像度特徴がないと見えないためです。

アブレーションでは、サンプリング点数 P を増やすと精度が上がるが計算量も増えるトレードオフが示され、P=4 程度が実用的なバランス点とされました。また、マルチスケール特徴を変形アテンションで束ねる構成が、単一スケールより明確に優れることが報告されています。iterative bounding box refinement(各デコーダ層で枠を少しずつ修正する)や two-stage 版を足すと、さらに数ポイント AP が上がることも示されました。

BEVFormer のアブレーション(nuScenes、NDS と mAP)では、空間クロスアテンションでフルアテンションと変形アテンションを比べると、精度はほぼ同等のまま計算量とメモリを大幅削減できることが示されました。つまり「全部見なくても関係ある数点で足りる」という前提が、検出だけでなくBEV知覚でも成立することが実証されています。時間セルフアテンションを抜くと NDS が数ポイント低下し、特に速度推定(mAVE)が悪化することも報告されています。

## メリット・トレードオフ・限界

メリット
- 計算量が候補数に線形(O(N×P))。高解像度BEVや高解像度画像でも現実的に回る。
- 各クエリが「見るべき場所」を学習で動的に決められる(状況依存の注意配分)。
- バイリニア補間サンプリング一発で実装でき、新しい依存ライブラリが不要。BEV生成にも軌跡計画にも同じ部品を流用できる。
- マルチスケール特徴を自然に束ねられる。

トレードオフ・限界
- サンプリング点が少ない(例 4点)と、参照点が外れたとき重要情報を見落とす。重みをデータ全体と比較せずクエリ予測に賭けるため、参照点の初期品質に敏感。
- 参照点の精度(=BEVではカメラ校正の正確さ)に依存する。校正がずれると見る場所そのものがずれる。
- `grid_sample` のバイリニア補間は、一部の車載アクセラレータ(NPU等)で最適化されにくく、デプロイ時のレイテンシ要因になり得る。FlashAttention のような高度な最適化カーネルが効きにくい(疎な散らばったメモリアクセスのため)。
- 単一ヘッドに簡略化すると、マルチヘッドのフル実装ほどの表現力は出ない。

未解決の課題としては、サンプリング点・オフセットの初期化と学習安定性(`offset_scale` の調整が必要なのはこの難しさの表れ)、ハードウェアフレンドリーな近似実装、そして参照点が信頼できない場合(校正誤差・動的物体)への頑健化が挙げられます。

## 発展トピック・研究の最前線

変形アテンションは検出・BEV知覚を越えて広がっています。セグメンテーション分野では Mask2Former などが変形アテンションを用い、占有グリッド予測(Occupancy)でも3Dボクセルクエリが変形サンプリングで特徴を集めます。

一方、ハードウェア最適化の観点からは、`grid_sample` を避けて純粋な行列積で近似する手法や、ウィンドウ分割で局所性を保ちつつ効率化する手法(Swin系)との比較・併用も研究されています。車載デプロイを見据えると「精度を保ちつつ、いかにアクセラレータに乗せやすい形にするか」が実務上の最前線です。

さらに広い視点では、変形アテンションは「全体を見ずに、関連箇所だけを動的に選んで見る」という疎なアテンション(sparse attention)の一系統に位置づけられます。長系列を扱う言語モデルの効率化(局所窓・ルーティング・選択的注意)とも問題意識が共通しており、「N²の壁をどう破るか」という普遍的なテーマの、空間データ版の解だと捉えると見通しが良くなります。変形アテンションの設計は、この制約と表現力のせめぎ合いの中で進化を続けています。
