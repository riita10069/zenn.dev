# Swin Transformer V2 (SwinV2)

## ひとことで言うと
Swin Transformer V2 は、画像を小さな窓(ウィンドウ)に区切ってその中だけで自己注意(どの部分がどれと関係が深いかを計算する仕組み)を行い、計算量を抑えた階層型の画像用 Transformer です。前身の Swin V1 が抱えていた「モデルを大きくすると学習が不安定になる」「学習時と違う解像度で使うと性能が落ちる」という2つの弱点を、scaled cosine attention(向きだけで関係を測る注意)・residual post-normalization(残差後の正規化)・log-spaced continuous relative position bias(対数間隔で連続的に与える相対位置)という3つの工夫で直したものです。高解像度・大規模化への耐性が大きく改善され、視覚 Transformer を約30億パラメータ級まで安定して学習する道を開きました。

## 直感的な理解
ビルを高くしていくと、低い建物では問題にならなかった揺れや基礎の沈下が無視できなくなります。ニューラルネットワークでも同じことが起きます。層を浅く小さく組んでいるうちは平気でも、層を深く幅を広くして「巨大化」させると、ある層の出力の大きさ(活性値)が層を進むごとにじわじわ膨らみ、最後には学習が発散してしまう、という現象が起きます。Swin V1 を巨大化したときにまさにこれが起き、V2 はその「高層化に耐える基礎工事」を施したモデルだと考えると分かりやすいです。

もう一つの問題は解像度です。低解像度の画像で訓練したモデルを、本番では高解像度で動かしたい、という要求はよくあります。ところが V1 は窓内の位置関係(相対位置)を特定のサイズに紐づけて学習していたため、サイズが変わると位置の意味がずれて性能が落ちました。V2 はこの位置の与え方を「サイズに依存しない連続的な関数」に置き換え、解像度をまたいでも意味が滑らかに引き継がれるようにしました。

自動運転のように高解像度・多視点のカメラ画像を扱い、しかもモデルをスケールさせたい場面では、この「大規模化への安定性」と「解像度耐性」が直接効いてきます。

## 基礎: 前提となる概念
V2 を理解するには、まず V1([Swin Transformer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0408_swin-transformer))の骨格を押さえる必要があります。要点だけ再掲します。

- 窓ベース自己注意(W-MSA): 特徴マップを 8x8 などの窓に区切り、自己注意を窓の中だけで計算する。これで計算量が画像サイズに線形になる。
- シフト窓(SW-MSA): 1層おきに窓の区切りを半分ずらし、窓をまたぐ情報も段階的に伝える。
- 階層構造: ステージを進むごとに解像度を半分・チャンネルを倍にし、4段のマルチスケール特徴を作る。Tiny のチャンネルは 96, 192, 384, 768。

さらに新しく出てくる言葉を噛み砕きます。

- 活性値(activation)とは、各層が出力する数値そのものの大きさのことです。これが層ごとに大きく膨らむと、勾配(学習で重みを更新する量)も暴れ、学習が不安定になります。V2 の論文では、深い層と浅い層で活性の振幅が数百倍異なる例が示されています。
- コサイン類似度とは、2つのベクトルの「向きの近さ」だけを測る指標で、−1 から 1 の範囲に収まります。ベクトルの大きさには影響されません。内積(ドット積)は向きと大きさの両方に影響されるため、片方のベクトルが極端に大きいと値が暴走しますが、コサインなら常に有界です。
- Pre-Norm と Post-Norm とは、Transformer ブロックの正規化(LayerNorm)を残差接続の「前」に置くか「後」に置くかの違いです。Pre-Norm(x + f(LN(x)))は学習しやすいが活性が累積しやすく、Post-Norm(LN(x + f(x)))は各層で出力が抑えられる代わりに学習が難しいとされてきました。
- 自己教師あり事前学習とは、人間のラベルなしでデータ自身から学習信号を作る前処理のことです。V2 の論文では SimMIM(マスクした画像の画素を当てさせるマスク予測型)を併用し、巨大モデルを少ないラベルで学習しています。

## 仕組みを詳しく
V2 の核心は3つの工夫です。順に、何の問題をどう解いたかを見ます。

### 1. Scaled Cosine Attention(スケール化コサイン注意)
V1 は Query と Key の内積で注意スコアを測っていました。内積はベクトルの大きさに引きずられるため、ある位置の Key が極端に大きいとそこへ注意が集中し、注意の分布が一部に独占されます。これが大規模モデルでの活性暴走と学習不安定の主因でした。

V2 は注意スコアを次のように置き換えます。

```
注意スコア(i, j) = cos(q_i, k_j) / τ + B_ij
```

ここで cos(q_i, k_j) は Query ベクトル q_i と Key ベクトル k_j のコサイン類似度(向きだけの近さ)、τ(タウ)は学習可能な温度パラメータで各ヘッド・各層ごとに持ち(下限を 0.01 に設定し、注意が鋭くなりすぎないようにする)、B_ij は後述の相対位置バイアスです。コサインは常に [−1, 1] に収まるので、入力の大きさがどう暴れても注意スコアが有界に保たれます。これにより、注意が特定位置に独占される現象が抑えられ、層が深く幅広くなっても学習が安定します。

### 2. Residual Post-Normalization(残差後の正規化)
V1 は各サブブロックの「前」に LayerNorm を置く Pre-Norm 方式でした。Pre-Norm は学習しやすい一方、残差接続で足し込まれる出力の大きさが層を進むごとに累積し、深い層では活性が非常に大きくなる傾向があります。V2 は LayerNorm を残差を足した「後」に移す Post-Norm に変えます。

```
V1 (pre-norm):  z = x + MSA(LN(x)),  out = z + MLP(LN(z))
V2 (post-norm): z = x + LN(MSA(x)),  out = z + LN(MLP(z))
```

これにより各ブロックの出力が毎回正規化され、足し込まれる量の振幅が一定に抑えられます。論文の解析では、Pre-Norm の Swin-L は深い層で活性が浅い層の数百倍に膨らむのに対し、Post-Norm にするとこの膨張がほぼ消え、層をまたいでほぼ一定に保たれることが示されています。Post-Norm は一般に学習が難しいとされますが、scaled cosine attention と組み合わせることで安定性と学習しやすさを両立しています。

### 3. Log-spaced Continuous Relative Position Bias(対数間隔の連続相対位置バイアス)
V1 は窓内の相対位置ごとに学習可能なバイアスの表(テーブル)を持っていました。窓サイズが固定なら問題ありませんが、サイズが変わると表のサイズも変わり、双線形補間でつなぐしかなく、これが精度低下の原因でした。

V2 は2段階で置き換えます。第一に、相対位置のバイアスを表ではなく小さな2層 MLP(メタネットワーク、ReLU 挟み)で「相対座標(Δx, Δy)→ バイアス値」と連続的に計算します。これにより任意の窓サイズに対応でき、未知のサイズへも MLP が滑らかに値を生成します。第二に、相対座標を対数スケールに変換してから MLP に入れます。

```
Δx̂ = sign(Δx) · log(1 + |Δx|)    (Δy も同様)
```

対数を取る理由は、低解像度から高解像度へ転用するとき、座標の差が大きく外挿(訓練時に見ていない範囲への拡張)になるのを防ぐためです。線形座標のままだと「8x8 窓で学んだものを 24x24 窓で使う」と相対座標の範囲が約3倍に広がり大きな外挿になりますが、対数を取ると入力空間の広がりが圧縮され(例えば線形で +8 が必要な外挿が、対数空間では約 +0.43 で済む)、外挿の度合いが大幅に小さくなります。結果として、低解像度で事前学習したモデルを高解像度のファインチューニングや推論に転用しても性能が落ちにくくなります。

### 大規模化を支える周辺の工夫
論文では上記3点に加え、巨大モデルを実際に回すための実装的な工夫も導入されています。GPU メモリを節約する activation checkpointing、複数 GPU にモデルを分割する Zero-Redundancy Optimizer、注意計算を低精度で行う逐次的自己注意計算、そして SimMIM 自己教師あり事前学習です。これらが揃って初めて SwinV2-G(約30億パラメータ)を 1536x1536 という超高解像度で学習できました。

### 階層構造と Tiny 構成の数値
階層は V1 同様で、ステージごとに解像度を半分・チャンネルを倍にします。小型版 `swinv2_tiny_window8_256` の名前は次の意味です。
- tiny: 小型版。総パラメータはおよそ 2800〜3500 万(構成・分類ヘッドの有無で変動)。
- window8: 自己注意を取る窓のサイズが 8x8。
- 256: 想定入力サイズ 256x256。

入力 256x256 のとき、各ステージの特徴マップはおおよそ 64x64(96ch)→ 32x32(192ch)→ 16x16(384ch)→ 8x8(768ch)になります。多くのライブラリ(timm 等)では features_only という指定でこれら4段を取り出せ、検出・分割や鳥瞰図変換など下流タスクへ渡せます。注意点として、入力はパッチ・窓に割り切れるサイズである必要があり、規定外のサイズはあらかじめリサイズしてから流すのが一般的です。

## 手法の系譜と主要論文
- 2021年(ICCV): Liu らの Swin V1(arXiv:2103.14030)が窓注意・シフト窓・階層構造で Transformer を汎用バックボーン化([Swin Transformer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0408_swin-transformer))。
- 2021〜2022年(arXiv→CVPR 2022): 同チームの Swin V2(arXiv:2111.09883)が本トピック。動機は「視覚モデルを言語モデル並みの数十億パラメータへスケールしたい。だが V1 はそのまま大きくすると壊れる」というもの。新規性は上記3工夫に加え、自己教師あり事前学習 SimMIM の併用で巨大モデルのラベル要求を減らした点です。
- 関連: He らの MAE(arXiv:2111.06377, CVPR 2022)、Xie らの SimMIM(arXiv:2111.09886, CVPR 2022)はマスク予測型の自己教師あり事前学習の系譜で、V2 の大規模学習を支えます。CNN 側の対抗は ConvNeXt / ConvNeXt V2([ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext) / [ConvNeXt V2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0351_convnext-v2))で、同じ「設計と事前学習の共設計」という問題意識を持ちます。後続の視覚基盤モデル(EVA, InternImage など)も、SwinV2 が確立した「安定化 + マスク事前学習でスケールする」路線を継承しています。

## 論文の実験結果(定量データ)
- ImageNet-1K / ImageNet-V2 分類: SwinV2 の巨大版 SwinV2-G(約30億パラメータ)は、ImageNet-V2(分布シフトへの頑健性を測る別撮りテストセット)で top-1 約 84.0% を達成し、当時の最高記録を更新しました。top-1 とは「最も確からしい予測が正解と一致した割合」で、V2 はこの一般化性能で先行モデルを上回りました。ImageNet-1K の通常 val でも約 90.2% に到達しています。
- COCO 物体検出: box AP(検出枠と正解枠の重なりを複数しきい値で平均した指標)で test-dev 約 63.1、mask AP(インスタンス分割)約 54.4 を達成し、当時の最高水準を更新しました。
- ADE20K セマンティックセグメンテーション: mIoU(予測領域と正解領域の重なりのクラス平均)で val 約 59.9 を達成。
- Kinetics-400 動画分類: top-1 約 86.8% を達成し、画像だけでなく動画でも有効性を示しました。

最も重要なメッセージはデータ・計算効率です。SwinV2-G は、同等以上の精度を出した先行の巨大モデルに比べ、約 40 分の1 のラベル付きデータと約 40 分の1 の訓練計算で到達したと報告されています。これは scaled cosine attention と Post-Norm による安定化、log-spaced 相対位置による解像度転移、そして SimMIM 事前学習が相乗した結果です。

アブレーションも示唆的です。scaled cosine attention を外して内積注意に戻すと、大型モデルでは学習中に活性が発散し、安定して学習できなくなります(小型でも約 +0.1〜0.4 ポイントの精度向上が報告)。また、相対位置を線形座標のまま(対数なし)にすると、低解像度で事前学習し高解像度でファインチューニングしたときの精度が、対数版より明確に低下します。たとえば 256→512 のような解像度転移で、線形パラメータ化や V1 のバイアス補間より連続対数版が高い top-1 を示しました。これらは「3工夫はそれぞれ別の問題(安定性・解像度転移)を独立に解いており、巨大化・高解像度化のどちらにも欠かせない」ことを意味します。

## メリット・トレードオフ・限界
メリット:
- 窓分割で計算量が画像サイズに線形。高解像度・多視点入力に向く。
- V1 比で学習が安定し、数十億パラメータ級まで発散せずに学習できる。
- 解像度を変えても性能が落ちにくく、低解像度事前学習→高解像度本番という実運用パターンに強い。
- マルチスケール特徴を出すため、CNN 系と同じ配線で差し替えでき、検出・分割・BEV 変換に使いやすい。

トレードオフ・限界:
- 窓・パッチに割り切れる入力が必要で、サイズ制約がある(リサイズで対応するのが通例)。
- V1 比で工夫が増え、実装・デバッグの難度が上がる。温度 τ の下限設定、Post-Norm と cosine attention の組み合わせなど学習の安定化に細かな調整が要る。
- 純 CNN(ResNet 等)より構造が複雑で、推論レイテンシやメモリの読みづらさが増す場面がある。コサイン正規化は内積より演算が増え、小型モデルでは速度面の利点が薄いこともある。
- 小データでは Transformer 一般の過学習傾向が残り、強い事前学習や十分なデータ量が前提になりやすい。

## 発展トピック・研究の最前線
SwinV2 が示した「安定化 + 自己教師あり事前学習でスケールする」という路線は、その後の大規模視覚基盤モデルへ受け継がれました。マスク予測型の自己教師あり(MAE, SimMIM, BEiT 系)はバックボーンの種類を問わず標準的な前処理になり、ConvNeXt V2 の FCMAE もこの流れに連なります。一方で、注意の計算をさらに効率化する線形注意や FlashAttention、注意を畳み込みで置き換える大カーネル CNN(InternImage の変形可能畳み込み)など、効率と精度のトレードオフを攻める研究が続いています。自動運転では、複数カメラの特徴を時系列で統合し鳥瞰図空間へ持ち上げる知覚モデルのバックボーンとして SwinV2 がよく採用され、高解像度耐性とマルチスケール出力がそのまま活かされています。研究上の未解決課題としては、巨大モデルの推論コストを車載のような制約環境でどう下げるか(蒸留・量子化・疎化)、そして注意の有界化(コサイン化)がもたらす表現力の上限がどこにあるか、が挙げられます。
