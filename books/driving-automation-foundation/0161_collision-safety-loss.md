# 衝突/安全制約損失

## ひとことで言うと

衝突/安全制約損失は、モデルが予測した「これから進む軌跡」が、他の車や歩行者にぶつかりそうだったり、走ってよい範囲(車線・歩道境界など)を飛び出したりすると、その分だけ罰金(ペナルティ)を加える補助的な損失です。「人間に似ているか」だけを見る模倣損失だけでは安全が保証されないので、明示的に「危ないことをするな」と教える役目を持ちます。模倣損失が「お手本に似せる」ものなら、安全損失は「やってはいけない一線を引く」ガードレールです。

## 直感的な理解

行動クローニング([行動クローニング損失 (Behavior Cloning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0111_behavior-cloning-loss))は「人間と同じ軌跡を出せ」と教えるだけです。これには重大な落とし穴があります。

- **データの偏り**。事故シーンや危機回避のデータは圧倒的に少ない。人間ドライバーはほとんど安全に運転しているので、「ぶつかりそうなときどうするか」をモデルはほとんど学べない。
- **モード平均化**。模倣損失は「平均的に人間っぽい」軌跡を出すよう学習する。複数の避け方がある場面で、右に避ける軌跡と左に避ける軌跡の平均を取ると、ちょうど真ん中=障害物に突っ込む軌跡を出してしまうことがある(MSE を最小化する解が条件付き期待値=谷間に落ちるため)。

つまり模倣損失だけだと、「だいたい人間っぽいが、ときどき危険」なモデルになりかねません。安全は自動運転で最優先なので、「人間らしさ」とは別の軸で「物理的に危険な軌跡には大きなペナルティを課す」明示的な損失が必要になります。これが衝突/安全制約損失です。模倣が「正しい例から学ぶ」のに対し、安全損失は「物理法則・交通規則から導いた制約を直接コードする」点で、データに依存しない知識を注入する役割を持ちます。

## 基礎: 前提となる概念

- **予測軌跡**: 未来の各時刻における自車位置の列 `τ = {(x_t, y_t)}`。これを評価対象にします。
- **占有(occupancy)/占有予測**: 周囲の物体が「未来の各時刻にどのセルを占めるか」を表すマップ。`[T, H, W]` の確率場で表すことが多い(FIERY などが代表)。
- **走行可能領域(drivable area)**: 道路・車線など走ってよい領域。BEV上で「1=走ってよい、0=ダメ」のマップにする。HDマップやセグメンテーションから得る。
- **ジャーク(jerk)**: 加速度の変化率(加速度の時間微分)。急加減速・カクつきの指標で、乗り心地と安全に関わる。
- **微分可能(differentiable)**: 損失で学習するには、ペナルティが軌跡の座標について滑らかに変化し勾配が流せる必要がある。0/1の硬い衝突判定は勾配が流れないので、滑らかな近似に置き換えます(後述)。
- **open-loop / closed-loop**: open-loop はログ再生上で1回だけ予測を出して真値と比べる評価、closed-loop は予測で車を動かして次の状況を作り続ける評価。安全損失の効果は本来 closed-loop で測るべきだが、データセット評価の多くは open-loop なので、この差が後述の限界につながります。

## 仕組みを詳しく

予測軌跡 `τ` に対し、安全に関わるいくつかのペナルティ項を足し合わせます。代表的なのは次の3種類です。

### 1. 衝突ペナルティ(collision loss)

他の交通参加者(車・歩行者・自転車)の予測占有領域と、自車の予測軌跡が重なったら罰します。

- 周囲の物体それぞれについて、未来の各時刻にどこを占めるか(占有マップ)を予測する。
- 自車軌跡上の各点が、その時刻の他物体の占有領域に入っていたらペナルティ。距離が近いほど大きくする例が多い。

数値イメージ: t=1.0秒後に前方車両が `y=10m` を占め、自車予測も `y=9.5m` まで進むなら、車間 `0.5m` は危険。安全マージン(たとえば3m)を切った分 `3.0 - 0.5 = 2.5` をペナルティに換算します。

### 2. 走行可能領域ペナルティ(drivable area / off-road loss)

予測軌跡が車線や走行可能領域からはみ出したら罰します。

- BEV上に「走ってよい=1、ダメ=0」のマップを用意する(HDマップや学習したセグメンテーションから)。
- 自車軌跡の各点がダメな領域に入っていたらペナルティ。歩道・対向車線・路肩への逸脱を抑える。

### 3. その他の安全・快適制約

- 速度・加速度・ジャークの上限を超えたら罰する → 急加速・急ブレーキを抑え、乗り心地と安全を確保。
- 横方向加速度(向心加速度)の制限 → 急ハンドルを抑える。
- 進行方向(heading)と車線方向の一致 → 逆走や斜行を抑える。

### 全体の合わせ方

これらは普通、主損失である模倣損失に **低い係数で足します**([固定重み和損失 (Weighted Sum)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0575_fixed-weighted-sum-loss) の方式)。

```
L_total = L_imitation + λ_col · L_collision + λ_road · L_offroad + λ_smooth · L_smooth
```

模倣で「人間らしい軌跡」を学びつつ、安全項で「危険な逸脱を矯正する」二段構えです。係数 `λ` が大きすぎると、安全すぎて動けない(過度に保守的)軌跡になり、小さすぎると効きません。バランス調整が鍵で、係数を自動調整する研究(動的重み付け [Dynamic Weight Averaging (DWA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0588_dynamic-weight-averaging))もここに関わります。実装上は、安全損失が効くのは「危険な軌跡を出したとき」だけなので、安全なシーンでは値が0になり、学習の大半では模倣損失が主導する、という非対称な効き方をします。

### 微分可能にする工夫

「ぶつかった/ぶつからない」の0/1判定はカクカクで勾配が流れません。実際は「障害物との距離に応じてなめらかに増えるソフトなペナルティ」にします。よくある形は、安全マージン `d_safe` を距離 `d` が下回るほど滑らかに増える関数です。

```
L_collision = Σ_t  softplus( d_safe - dist(ego_t, obj_t) )
```

`softplus(x) = log(1 + exp(x))` は `max(0, x)` を滑らかにしたもので、`d_safe` を切るとなだらかに立ち上がり、安全側では限りなく0に近づきます。これにより「もう少し離れれば損失が下がる」という勾配が軌跡の各点 `(x_t, y_t)` へ伝わり、軌跡が安全側へ押し戻されます。ガウシアンの占有(障害物を確率場で表す)と軌跡の重なり積分で表す方式も同様の効果を持ちます。後者は他物体の予測の不確かさ(占有確率)を自然に取り込めるので、確信度の高い障害物ほど強く避ける、という挙動になります。

## 手法の系譜と主要論文

### ChauffeurNet (Bansal et al., 2019, RSS, "ChauffeurNet: Learning to Drive by Imitating the Best and Synthesizing the Worst")

安全補助損失の先駆け。模倣に加えて、衝突損失・道路逸脱(on-road)損失・幾何(車線追従)損失を明示的に足し、さらに「わざと悪い状況を合成して(trajectory perturbation)回復を学ばせる」工夫を導入。「最良を模倣し、最悪を合成する」という副題が思想を端的に表しています。模倣だけでは学べない危険回復(分布シフトで陥る状況)を、補助損失と合成データで補う流れの出発点です。

### FIERY (Hu et al., 2021, ICCV, "FIERY: Future Instance Prediction in Bird's-Eye View")

衝突損失の入力を供給する側の代表。カメラ画像から BEV 上で「未来のインスタンス占有」(どの車がどこをどう動くか)を確率的に予測する枠組み。安全損失は他物体の未来占有を必要とするので、こうした占有予測の精度が安全制約の土台になります。

### ST-P3 (Hu et al., 2022, ECCV, "ST-P3: End-to-end Vision-based Autonomous Driving via Spatial-Temporal Feature Learning")

カメラのみの end-to-end 自動運転で、知覚→予測→計画を時空間特徴で一貫学習。計画段階で、サンプリングした軌跡候補を、学習した cost に加えて「他エージェントとの衝突」「走行可能領域からの逸脱」コストで評価し、最も安全で人間らしい軌跡を選びます。模倣だけでは安全が担保されない、という問題意識への明確な回答です。

### UniAD (Hu et al., 2023, CVPR, "Planning-oriented Autonomous Driving"、CVPR Best Paper)

検出・追跡・マッピング・動き予測・占有予測・計画を1つのネットワークに統合し、すべてを「最終的な計画に役立つように」学習する設計。計画の損失に、模倣項に加えて衝突回避コスト(他エージェント占有との重なりを罰する項)を組み込みます。タスクを別々に最適化すると誤差が伝播し最終計画が安全でなくなるので、計画を中心に据えて安全制約を計画損失に直接入れる、という思想です。

### VAD (Jiang et al., 2023, ICCV, "VAD: Vectorized Scene Representation for Efficient Autonomous Driving")

シーンをベクトル表現(占有マップのラスタではなく、車線や物体を点列・ベクトルで表す)で軽量化しつつ、計画に「自車-他車の衝突制約」「自車-境界の逸脱制約」「自車-車線の方向制約」という明示的な制約損失を導入。計算を大幅に軽くしながら安全制約で衝突率を下げました。占有マップ(ラスタ)ではなくベクトルで制約を書く設計が特徴です。

これら共通の設計思想は「模倣損失だけでは安全は保証できない。衝突・逸脱を罰する明示的なコスト項を計画損失に足すことで、構造的に危険を避ける」です。

## 論文の実験結果(定量データ)

評価は主に nuScenes(都市部の実走行データセット)の open-loop 計画タスクで行われ、指標は2つです。**L2 軌跡誤差(メートル)**: 計画軌跡と人間軌跡の距離(1秒・2秒・3秒先で測り平均)。小さいほど人間に近い。**衝突率(Collision Rate, %)**: 計画軌跡が他物体と衝突する割合。小さいほど安全。安全損失の価値は「L2 を悪化させずに衝突率を下げられるか」で測られます。

- **ST-P3 (2022)**: カメラのみで、平均 L2 を約1m前後に保ちつつ、衝突率を従来のカメラ手法より低減。安全コストの導入が衝突を抑えることを示しました。
- **UniAD (2023)**: nuScenes 計画で当時の最高水準。報告された平均 L2 は約1.0m前後、平均衝突率はおよそ0.3%水準と、先行手法(衝突率がより高い)を明確に下回りました。各中間タスク(検出・占有・動き予測)を統合し計画向けに学習することの効果を、アブレーションで一つずつ示しています。具体的には、動き予測モジュールや占有予測モジュールを外すと計画の衝突率が悪化することを示し、「安全な計画には周囲の未来占有を正しく持つことが効く」と定量的に裏付けました。
- **VAD (2023)**: ベクトル表現で推論を大幅高速化(報告ではUniADの数倍のFPS、おおむね9〜10倍速いVAD-Tiny構成も提示)しつつ、平均 L2 と衝突率を同等以下に維持。明示的な衝突・境界・方向制約が、軽量化しても安全性を保つ鍵だと示しました。

ただし重要な注意があります。後続研究(BEV-Planner / nuScenes 計画評価の再検討、Zhai et al. 2023)で、これらの衝突率は自車運動が単純な直進主体のシーンに偏ること、評価プロトコル(衝突判定の解像度や自車サイズの扱い)次第で数値が大きく動くこと、さらに「自車の過去軌跡(ego status)だけを入力にした単純なモデルでも同等の L2 と衝突率が出てしまう」ことが指摘されました。つまり「衝突率0.3%」のような数字は手法間比較の目安にはなるが、絶対的な安全保証ではなく、知覚を使わないショートカットでも達成しうる、という読み方が研究者には求められます。これは open-loop 指標の限界([行動クローニング損失 (Behavior Cloning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0111_behavior-cloning-loss) でも触れたオフライン損失と実走行性能の乖離)の一例で、近年は NeuroNCAP のような closed-loop の衝突評価が併用されつつあります。

## メリット・トレードオフ・限界

メリット
- 安全を明示的に教えられる。模倣損失では学べない「危険回避」を補える。
- データ偏り(事故シーンが少ない)を補完し、希少な危険状況への頑健性を上げる。物理・規則由来の知識なので、データになくても効く。
- モード平均化による「真ん中に突っ込む」軌跡を抑制できる。
- コスト型プランナ([最大マージン計画損失 (Cost Volume)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0188_max-margin-planning-loss))とも組み合わせやすい(コストの加算で表現できる)。

トレードオフ・限界
- 係数が大きすぎると過度に保守的(動けない・流れに乗れない)になり、小さすぎると効かない。バランス調整が難しい。
- 他エージェントの占有予測や走行可能領域マップの精度に安全性が依存する(予測が外れれば安全保証も崩れる)。予測の不確かさを無視すると過信になる。
- ペナルティを微分可能にするための近似(ソフト距離関数)が、実際の物理的衝突判定と完全には一致しない。学習で衝突損失が下がっても、硬い判定では衝突する余地が残る。
- 学習時の損失を下げても、実走行(closed-loop)での安全が自動的に保証されるわけではない(分布シフトの影響、評価プロトコル依存)。

研究上の未解決課題は、open-loop の衝突率と closed-loop の実安全性のギャップを埋めること、評価指標の標準化、そして「過度に保守的にならず、かつ確実に危険を避ける」係数バランスの原理的な決め方です。

## 発展トピック・研究の最前線

近年は、安全制約を後付けの補助損失としてではなく、計画の構造そのものに組み込む方向が進んでいます。制御理論の制御バリア関数(Control Barrier Function, CBF)を学習に統合して安全を保証付き(forward invariance、危険集合に入らないことを数学的に担保)で扱う研究、占有を陽に予測してそこへの侵入を罰する occupancy-aware planning、そして closed-loop シミュレータ(衝突を実際に起こして評価する環境)で安全損失の効果を検証する流れです。評価面では、open-loop 指標(L2・衝突率)の限界が広く認識され、closed-loop 成功率・介入率・衝突回数での評価が標準になりつつあります。安全損失はコスト型計画([最大マージン計画損失 (Cost Volume)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0188_max-margin-planning-loss))や分布マッチング型模倣([GAIL (敵対的模倣学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0177_gail-adversarial-imitation))と組み合わせて、解釈可能で安全な end-to-end 運転の中核を担い続けています。
