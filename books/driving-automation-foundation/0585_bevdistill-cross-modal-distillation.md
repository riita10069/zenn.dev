# BEVDistill (クロスモーダル蒸留)

## ひとことで言うと
LiDAR(レーザーで距離を直接測るセンサー)を使う「先生モデル」が持っている正確な3D知識を、カメラだけを使う「生徒モデル」に教え込む(蒸留する)学習技法です。推論(運用)時はカメラだけで動かしますが、学習のときに LiDAR の知識を借りることで、カメラのみの3D検出の精度を底上げできます。鍵は「LiDAR とカメラの特徴を BEV(真上視点)という共通の座標系でそろえてから知識を移す」ことです。

## 直感的な理解
3D 物体検出で「どこに、どんな大きさの、どの向きの物体が、どれくらいの速度であるか」を当てるには、奥行き(距離)の情報がとても重要です。LiDAR はレーザーの往復時間で距離を直接測るので、奥行きが正確で 3D 検出が得意です。一方カメラは安価で高解像度・色情報を持ちますが、1 枚の 2D 画像から 3D を復元するのは原理的に難しく(奥行きが一意に定まらない ill-posed 問題)、奥行きの推定が苦手です。

実車では「高価で機械的に壊れやすい LiDAR を載せず、カメラだけで LiDAR 並みに検出したい」という強い動機があります。そこで「学習のときだけ LiDAR の力を借り、運用ではカメラだけで動く」という発想が生まれます。これは、優秀な家庭教師(LiDAR モデル)が試験前に生徒(カメラモデル)を教え込み、本番の試験(推論)では生徒が一人で解く、という構図に似ています。先生は本番には出てきません。家庭教師の利点は、正解(○×だけ)を教えるのでなく「なぜそうなるか、どこに注目すべきか」という豊かな中間情報まで伝えられる点にあります。

## 基礎: 前提となる概念
- モダリティ(modality): 情報の種類のこと。ここでは LiDAR(点群)とカメラ(画像)という、性質の異なる入力を指します。「クロスモーダル(cross-modal)」は、この異なる種類の間をまたぐことを意味します。
- 点群(point cloud): LiDAR が返す「3D 空間の点の集まり」。各点は座標(x, y, z)と反射強度を持ちます。点群はまばら(空間のごく一部にしか点がない、特に遠方ほど)で色情報がありません。
- 知識蒸留(knowledge distillation): 大きく賢いが重い先生モデルの「ソフトな出力」(正解ラベルよりも豊かな確信度の分布など)を、軽い生徒モデルに学ばせて性能を移す技法。Hinton らが 2015 年に提案しました。先生の出力(logits)を温度パラメータ T で割ってなめらかにし(softmax(z/T))、生徒がそれを真似ることで「正解クラス以外の確信度の構造(dark knowledge=暗黙知)」まで伝わるのが要点です。FitNets(Romero ら 2015)はこれを「中間特徴も真似させる(feature distillation)」へ拡張しました。BEVDistill はこの両系統を BEV 空間で統合します。
- BEV(Bird's-Eye-View): 真上から見下ろした視点の特徴マップ。LiDAR もカメラも、最終的には同じ BEV 平面上の特徴に変換できます。座標が同じグリッドにそろうので、両者を「同じ真上地図」としてセル単位で比較できます。これがクロスモーダル蒸留を可能にする土台です。
- DETR 系の集合予測: 物体を「クエリの集合」として一括予測する Transformer ベースの検出枠組み(DETR, Carion ら 2020)。予測と正解を二部マッチングで 1 対 1 に対応づけるため、先生と生徒のクエリも物体単位で対応づけられます。
- 前景・背景の不均衡: 画像や BEV の大部分は「物体のない背景」で、検出に効く「物体のある前景」はごく一部です。素朴に全画素を均等に扱うと、背景に学習が引きずられます。

## 仕組みを詳しく
クロスモーダル蒸留の難しさの本質は「LiDAR 特徴(点群由来)とカメラ特徴(画像由来)は分布も性質も違うので、単純に『先生の特徴に生徒の特徴を合わせろ』では噛み合わない」ことです。点群は幾何が正確だがまばら、画像は密だが奥行きが曖昧で、特徴空間の意味が一致しません。BEVDistill はこれを「BEV という共通の土俵」で解き、さらに知識を2つのレベル(密な特徴レベルと、疎な物体レベル)で移します。

### 準備: 共通の BEV 表現にそろえる
- 先生(LiDAR): 点群を 3D エンコーダで処理して LiDAR-BEV 特徴を作る(例: 256×128×128 = チャネル × グリッド縦 × グリッド横)。
- 生徒(カメラ): 複数カメラ画像をバックボーン+視点変換してカメラ-BEV 特徴を作る(例: 同じ 256×128×128)。

両者が同じ BEV グリッドにそろうので、マス(セル)ごとに対応づけて比較できます。

### 蒸留1: 密な特徴蒸留(dense feature distillation)
BEV のセルごとに「先生の特徴」と「生徒の特徴」を近づけます(L2 距離など)。素朴に全セルを一律に近づけると問題が起きます。理由は2つ。
1. LiDAR-BEV はまばらで、何もないセルや遠方ではそもそも先生も情報を持たない(ノイズを真似ることになる)。
2. そういう無意味なセルまで生徒に真似させると、学習が乱れ前景の学習が薄まる。

そこで BEVDistill は「物体がありそうな前景セルを重視する」重み付けを行います。具体的には、各セルの重要度を物体の存在確率(ガウシアン的な前景マスク)に応じて重み付けし、前景に近いセルほど蒸留損失を強くかけます。これにより「前景・背景の不均衡」と「LiDAR のまばらさ」に同時に対処します。

### 蒸留2: インスタンス蒸留(instance distillation)
特徴だけでなく「物体(インスタンス)単位」でも知識を移します。先生(LiDAR)が検出した各物体について、その物体に対応するクエリ・予測(位置・サイズ・向き・速度に加え、分類の確信度の分布)を、生徒(カメラ)の対応する予測に近づけます。ここで DETR 系の集合予測が活き、先生と生徒のクエリを二部マッチング(ハンガリアンマッチング: 予測と正解を 1 対 1 に最適に割り当てる組合せ最適化)で対応づけ、物体レベルで「先生はこの物体をこう捉えている」というソフトな知識を伝えます。正解ラベル(ハードな 0/1)よりも、先生の確信度分布(ソフトな情報)を学ぶのが蒸留の核です。

### 学習の全体像
生徒モデルの損失は次の合計です。
```
L = L_det(検出の正解ラベルとの誤差: 分類 + 回帰)
  + α · L_dense(前景重み付き特徴蒸留)
  + β · L_inst(インスタンス蒸留)
```
α, β は各蒸留損失の重み(ハイパーパラメータ)。学習時のみ LiDAR 先生を併走させ(先生は事前学習済みで固定、勾配を流さない)、推論時には先生を捨ててカメラ生徒だけで動かします。これで「学習で LiDAR を借り、運用はカメラだけ」が成立し、推論コストは一切増えません。

### before/after の直感
```
蒸留なし: カメラBEV特徴は奥行きがあいまい → 遠方や遮蔽で検出ミス・位置ずれ
蒸留あり: 学習中、LiDAR先生が「ここに車がある(正確な距離)」と密特徴と物体予測の
        両面で教える → カメラBEV特徴が「あるべき形」へ寄り、奥行き感が改善
```

## 手法の系譜と主要論文
- Knowledge Distillation (Hinton, Vinyals & Dean, 2015, arXiv:1503.02531): 蒸留の起源。大モデルのソフト出力を温度でなめらかにし小モデルに学ばせる。動機はモデル圧縮(同性能で軽量化)でした。
- FitNets (Romero et al., ICLR 2015, arXiv:1412.6550): 出力だけでなく中間特徴(hints)も真似させる feature distillation を提案。BEVDistill の密特徴蒸留の源流。
- MonoDistill / LIGA-Stereo(2021–2022): LiDAR の知識を画像系 3D 検出へ移す先行研究。BEVDistill は「BEV を共通空間にする」点と「密特徴 + インスタンスの二段」で整理した点が新しい。
- BEVDepth (Li et al., AAAI 2023, arXiv:2206.10092): LiDAR 点群を投影した正解深度でカメラの奥行き推定を直接監督してカメラ BEV 検出を改善。BEVDistill が「BEV 特徴・物体予測を真似させる(表現を移す)」のに対し、BEVDepth は「奥行きそのものを正解として教える(監督信号を足す)」ので、狙いは近いが教え方が異なる相補的アプローチ。
- BEVFusion (Liu et al., ICRA 2023, arXiv:2205.13542): カメラ BEV と LiDAR BEV を統一空間で融合する手法。「BEV を共通土俵にする」発想を共有しますが、こちらは推論時に両モダリティを使う融合であり、蒸留(推論時はカメラのみ)とは目的が異なります。
- BEVDistill (Chen et al., ICLR 2023, arXiv:2211.09386): 本トピック。新規性は「BEV 空間での前景重み付き密特徴蒸留 + DETR 系インスタンス蒸留」の二段構成と、それをモダリティ非依存に設計した点。
- UniDistill (Zhou et al., CVPR 2023, arXiv:2303.15083): BEV 蒸留を任意のモダリティ間(LiDAR→カメラ、カメラ→LiDAR、融合→単一など)に一般化したフレームワーク。BEVDistill の発想を多方向へ拡張した後続研究。

## 論文の実験結果(定量データ)
評価は nuScenes のマルチカメラ 3D 検出で行われます。指標の意味:
- mAP(mean Average Precision): 検出と正解の一致度の平均。nuScenes の 3D では中心距離のしきい値(0.5/1/2/4 m)で一致判定し、10 クラス・4 しきい値で平均します。高いほど良い。
- NDS(nuScenes Detection Score): mAP と各種誤差(位置 mATE・サイズ mASE・向き mAOE・速度 mAVE・属性 mAAE)を統合した総合スコア。検出の総合的な質を測り、高いほど良い。

報告によると、BEVDistill を BEVFormer 系のカメラ生徒に適用すると、蒸留なしの同じ生徒に対して mAP・NDS がともに明確に向上します。論文では nuScenes val で、BEVFormer ベースラインに対して mAP・NDS をおよそ数ポイント(報告では mAP で +2〜3 ポイント、NDS で同程度規模)押し上げ、test split では当時のカメラのみ手法の中で上位、NDS でおよそ 0.51〜0.52 前後の水準に達したと報告されています。数ポイントの mAP/NDS 改善は、追加センサーを推論時に増やさず学習だけで得られる利得としては大きく、実運用コストを増やさない点に価値があります(同じ改善を解像度やバックボーン拡大で得ようとすると推論コストが跳ね上がる)。

アブレーションでは、(1) 密特徴蒸留だけ、(2) インスタンス蒸留だけ、(3) 両方、を比較し、両方を組み合わせたときに最も改善することが示されています(二段が相補的)。また、前景重み付けを外して全セル一律で蒸留すると改善幅が縮む、または悪化することが報告され、「前景に絞る」設計が効いていることが裏づけられます。これは LiDAR のまばらさ・背景の多さという BEV 特有の性質に対処した結果です。さらに、先生の質(LiDAR 検出器の強さ)が高いほど生徒の改善も大きい傾向が示され、蒸留が「先生の上限」に強く依存することも示唆されています。

## メリット・トレードオフ・限界
メリット
- 推論時はカメラのみで動くのに、学習時に LiDAR の正確な3D知識を借りて精度を底上げできる(運用コスト・レイテンシはカメラのまま)。
- BEV を共通土俵にすることで、性質の違う LiDAR とカメラの特徴を素直に比較・蒸留できる。座標が一致するので対応づけが明快。
- 前景重視の密特徴蒸留 + 物体単位のインスタンス蒸留で、検出に効く知識を選んで移せる。二段が相補的。

トレードオフ・限界
- 学習に「同期したカメラと LiDAR のペアデータ」と、学習済みの LiDAR 先生モデルが必要で、データ収集・時刻同期・キャリブレーション・先生学習のコストが高い。
- 先生(LiDAR)の弱点や偏り(雨・霧・反射で苦手な対象、ガラスや黒い物体での点の欠落、色情報がない、遠方でまばら)も生徒に移りうる。先生が苦手な状況では蒸留が逆効果になる場合がある。
- 蒸留の重み(α, β)やマッチング・前景重みの設計が繊細で、調整を誤ると改善しない、または学習を不安定にする。
- 検出タスク向けの設計で、占有予測や経路計画など別タスクへそのまま流用するには損失設計の作り替えが要る。

研究上の未解決課題として、(a) 先生のラベルノイズや欠落をどう扱うか(信頼度に応じた蒸留)、(b) 時間情報を含む蒸留(複数フレームの動きの一貫性の知識を移す)、(c) ラベルなしデータでの自己蒸留・半教師蒸留、(d) 占有・運転方策(end-to-end)への蒸留拡張、が挙げられます。

## 発展トピック・研究の最前線
- 双方向・多モダリティ蒸留: UniDistill のように方向を限定せず、レーダー・地図など他の手がかりも含めて知識を移す流れ。
- 時間蒸留: 長期時間集約([SOLOFusion (長期時間融合)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0499_solofusion-long-term-temporal))の文脈で、先生が持つ「動きの一貫性」を生徒に移す試み。
- 特徴再構成・予測との接続: 将来状態予測([未来特徴予測 (FutureState型モジュール)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0012_future-state-prediction))のような自己教師タスクと蒸留を組み合わせ、ラベルに頼らず BEV 表現を鍛える方向。蒸留損失の設計(どの特徴を、どの重みで近づけるか)は、表現学習一般に通じる重要な研究テーマです。
