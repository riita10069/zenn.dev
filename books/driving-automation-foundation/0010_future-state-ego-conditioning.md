# 計画意図ベクトルによる未来特徴マップの条件付け

## ひとことで言うと

未来を予測する世界モデルが、「今の世界を表す鳥瞰図(BEV)特徴マップ」に「自車の計画意図ベクトル」をチャネル毎の下駄(バイアス)として足し込み、そこから一定の時間刻みで複数枚の未来特徴マップを予測する仕組みのことです。形の違う 2 つのデータ(3 次元の特徴マップと 1 本のベクトル)を、空間構造を壊さずに混ぜる軽量な条件付け技法であり、その源流は画像生成・VQA で確立した FiLM(Feature-wise Linear Modulation)にあります。

## 直感的な理解

世界モデルが未来を当てるには「自分がこれから何をするか」を知る必要があります。同じ交差点でも、止まるか進むかで 1〜2 秒後の景色は別物になるからです。そこで「今の世界(BEV 特徴)」に「自分の意図(計画意図ベクトル)」を教え込みたい。

ところが、この 2 つはデータの形が全く違います。BEV 特徴は `[B, C, H, W]` の 3 次元グリッド(地図のように縦横に広がり、各マスに C 個の数が乗っている)。一方、計画意図は `[B, C]` のただの 1 本のベクトル(空間的な広がりを持たない)。地図に 1 本の数列を「足す」には工夫が要ります。

直感的な比喩で言えば、料理の味付けに似ています。鍋(BEV 特徴)の中身は場所ごとに具材が違いますが、そこに「今日は少し辛めにしたい」という方針(意図)を、全体に均一に振りかける塩こしょう(チャネル毎のバイアス)として加える。具材の配置(空間構造)はそのままに、全体の風味(チャネルの値)だけを意図に合わせて底上げする。これがチャネル毎バイアスによる条件付けです。

## 基礎: 前提となる概念

特徴マップ(feature map)とチャネル(channel)。畳み込みネットワークの中間出力で、形は `[B, C, H, W]`。H×W が空間グリッド、C がチャネル(各空間位置で何種類の特徴を見ているか)。例えば C 番目のチャネルが「車らしさ」、別のチャネルが「路面の白線らしさ」を表す、というように、チャネルごとに異なる意味的な軸を担うイメージです。

ブロードキャスト(broadcast)。形が違うテンソル同士の演算で、小さい方を自動で大きい方の形に複製してから計算する仕組みです。`[B, C, 1, 1]` と `[B, C, H, W]` を足すと、`[B, C, 1, 1]` が H×W の全マスにコピーされてから加算されます。つまり「全マスに同じ C 次元の下駄を足す」が 1 行で書けます。

活性化関数(activation function)。線形演算(畳み込みや全結合)の間に挟む非線形関数で、これがないとネットワークは何層重ねても 1 層の線形変換と等価になってしまいます。GELU(Gaussian Error Linear Unit, Hendrycks & Gimpel, 2016)や ReLU が代表で、表現力を生む要です。

条件付け(conditioning)。あるモデルの出力を外部情報に従わせること。ここでは未来予測を「自車の意図」に従わせます。条件付けの入れ方には、入力に結合(concatenate)する、注意機構(attention)で混ぜる、正規化層のパラメータを外部情報から作る(FiLM や Conditional BatchNorm)、などの流派があります。本トピックはそのうち最も軽量な「チャネル毎の加算バイアス」です。

潜在空間での未来予測。未来を画素(RGB の値)で当てるのではなく、抽象的な特徴マップ(潜在表現)で当てる流儀です。画素を細部まで当てるのは無駄が多く、運転に関係ない背景の揺らぎまで予測しようとして崩れがちです。特徴空間なら「意味的に重要な変化」だけを扱えます。

## 仕組みを詳しく

世界モデルのヘッドが行う処理を、ステップに分解します。B をバッチ、C を 256、H×W を BEV の広さとします。

ステップ 1: 意図をチャネル毎の下駄に変換する。
```
bias = Linear(256 → 256)(ego_hidden)   # [B, 256]   計画意図ベクトルを線形変換
bias = bias.view(B, 256, 1, 1)          # [B, 256, 1, 1]  空間の高さ・幅を1にする
```
ここで全結合層 `Linear(256→256)` を一枚かませるのは、意図ベクトルの空間と特徴マップのチャネル空間を「翻訳」するためです。意図の生の値をそのまま下駄にするのではなく、学習で「どのチャネルをどれだけ底上げすべきか」を覚えさせます。

ステップ 2: ブロードキャスト加算で混ぜる。
```
conditioned = fused_features + bias     # [B, 256, H, W] + [B, 256, 1, 1] → [B, 256, H, W]
```
`bias` は空間方向が 1×1 なので、H×W の全マスに同じ 256 次元の下駄が足されます。マップのどのマスにも、自車の意図ぶんの味付けがチャネル単位で加わる、という操作です。

ステップ 3: 畳み込みで未来へ変換する。
```
x = Conv2d(256 → 512)(conditioned)
x = GELU(x)                             # 非線形性を入れる
x = Conv2d(512 → 1024)(x)               # [B, 1024, H, W]
```

ステップ 4: チャネル方向に切り分けて複数の未来時刻にする。
```
futures = x.chunk(4, dim=1)             # 4 × [B, 256, H, W]
# これらが例えば t+Δ, t+2Δ, t+3Δ, t+4Δ の未来特徴マップ
```
出力 1024 チャネルを 4 等分すると、それぞれ 256 チャネルの未来特徴マップ 4 枚になります。これらが一定の時間刻み(例えば 1.6 秒刻みで 6.4 秒先まで)の未来を表します。

なぜ「チャネル毎バイアス」を選ぶのか。理由は 3 つあります。第一に、空間構造を壊さない。意図を空間に貼り付けて掛け算したり、巨大な全結合で空間ごと混ぜると、「どこに何があるか」という地図情報が崩れたりパラメータが爆発します。加算による下駄なら各マスは「元の特徴 + 共通の意図」となり、配置を保ったまま意図を反映できます。第二に、軽量。`Linear(256→256)` 一枚分のパラメータしか増えません。第三に、BEV 解像度に依存しない。H, W が変わっても同じコードが動きます。

メモリの注意点を具体的な数値で見ます。BEV 解像度を 450×300、未来 4 枚 × 256 チャネル、float32(1 要素 4 バイト)とすると、中間の 1024 チャネル出力だけで概算 `450 × 300 × 1024 × 4 バイト ≈ 550 MB / サンプル`に達します。バッチを増やすとすぐに GPU メモリを食い尽くすので、実運用では混合精度(bf16 や fp16、1 要素 2 バイトで半減)や、世界モデルに入れる前の BEV 特徴の空間ダウンサンプリングがほぼ必須になります。これは順伝播の入出力インタフェースには影響しません。

## 手法の系譜と主要論文

特徴マップを外部情報で変調する技法の系譜を辿ります。

Conditional Batch Normalization(de Vries et al., NeurIPS 2017, "Modulating early visual processing by language")。バッチ正規化のスケールとシフトのパラメータを、外部情報(言語)から動的に生成する手法です。質問文に応じて視覚特徴の処理を初期段階から変調できることを示し、条件付けを「正規化パラメータの注入」として実現する流れの起点になりました。

FiLM(Perez et al., AAAI 2018, "Feature-wise Linear Modulation", arXiv:1709.07871)。上記を一般化し、特徴マップにチャネル毎のスケール(掛け算 γ)とシフト(足し算 β)を加えるだけ、という極めて単純な条件付けを提案しました。`FiLM(x) = γ(z) ⊙ x + β(z)`(z は条件、⊙ はチャネル毎の掛け算)。動機は「軽量に外部情報で特徴を変調したい」。少ないパラメータで強力な条件付けができることを Visual Question Answering(画像に関する質問応答)で示しました。本トピックのチャネル毎バイアスは、FiLM のうち「シフト β(足し算)だけ」を使った形と見なせます。スケール γ を足せば完全な FiLM になります。トレードオフは「空間ごとに別々の変調はできない(全マス共通の γ, β)」点です。

SPADE(Park et al., CVPR 2019, "Semantic Image Synthesis with Spatially-Adaptive Normalization", arXiv:1903.07291)。FiLM の弱点である「空間一様」を克服し、空間位置ごとに異なる γ, β を持たせて、セマンティックマップから写実的な画像を生成しました。これは「空間ごとに違う意図を効かせたい」場合の発展形にあたります。

JEPA(LeCun, 2022, "A Path Towards Autonomous Machine Intelligence")。未来は画素ではなく潜在表現(特徴)空間で予測すべきという提言です。何を「良い潜在表現」とするかの設計が難しいというトレードオフはありますが、世界モデルが「画素ではなく特徴マップを予測する」設計の理論的支柱です。

I-JEPA(Assran et al., CVPR 2023, "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", arXiv:2301.08243)。画像版 JEPA で、画像の一部の特徴から別領域の特徴を予測します。画素再構成より意味的で抽象的な表現を学べることを示し、「未来(他領域)を特徴空間で予測する」発想の具体例です。

MILE(Hu et al., NeurIPS 2022, arXiv:2210.07729)。BEV 上で行動条件付きに未来潜在を roll-out する運転世界モデル。本トピックの「計画意図で未来特徴を条件付ける」のは、この行動条件付き潜在予測の単純化版にあたります。

要約すると、本トピックは「FiLM(のβだけ)で条件付け」×「JEPA/I-JEPA 的に特徴空間で未来を予測」×「MILE 的に行動(意図)で条件付け」を組み合わせた、軽量で実装容易な世界モデルヘッドです。

## 論文の実験結果(定量データ)

FiLM の効果。FiLM 論文は CLEVR(物体の属性や関係を問う合成 VQA データセット)で、当時の最先端を上回る精度を報告しました。報告では全体精度が約 97.7%(従来の強いベースラインが 90% 台前半)に達し、特に「複数物体の比較」「数える」といった構成的推論を要する難問で大きく改善しました。アブレーションでは、FiLM 層を取り除く・スケール γ かシフト β の片方を外すと精度が下がることが示され、軽量な変調が推論性能の鍵だと裏付けられています。γ と β の両方があると最も良く、片方だけでも素朴な結合(concatenation)より良いという結果は、「チャネル毎バイアスだけでも一定の効果がある」という本トピックの設計判断を支持します。

潜在予測の効果(I-JEPA)。I-JEPA は ImageNet の線形評価(学習済み表現の上に線形分類器だけを載せて測る、表現の良さの標準テスト)で、画素再構成型の自己教師あり手法より少ない計算量で高い精度を達成し、特に大きなモデル・大きな画像で効率の良さが際立つと報告されました。これは「未来を画素ではなく特徴で予測する」方針が、効率と表現品質の両面で有利だという証拠です。

世界モデル全体の効果(MILE)。前述の通り MILE は CARLA の閉ループ運転で Driving Score を従来手法から改善し、特に未知の町・天候への汎化で、行動条件付き潜在 roll-out が効くことを示しました。アブレーションでは、未来予測の損失や行動条件付けを外すと汎化性能が落ちる傾向が一貫して報告されています。

指標の意味の補足。Driving Score は Route Completion(経路走破率 %)× Infraction Penalty(違反による減点係数)で 0〜100、高いほど安全に走破。線形評価精度は「凍結した表現がどれだけ意味的か」を測る代理指標で、高いほど良い表現。CLEVR 精度は「構成的な視覚推論がどれだけできるか」。これらが改善するということは、それぞれ「運転がより安全に」「表現がより意味的に」「条件付けがより正確に効いている」ことを意味します。

## メリット・トレードオフ・限界

メリット。空間構造を壊さずに意図を注入できます(チャネル方向のみの変調)。加算による下駄なのでパラメータが少なく実装が単純で、デバッグも容易です。「未来を特徴空間で予測する」点で JEPA/MILE 系の世界モデル設計に沿い、画素予測より効率的です。BEV 解像度に依存せず動くため、解像度を変えてもコードが共通です。

トレードオフと限界。第一に、チャネル毎バイアスは全マスに同じ下駄を足すため、空間ごとに違う意図の効かせ方ができません(FiLM のスケール項 γ もない、シフトだけの簡易版)。「右折するから右側だけ未来を変えたい」のような空間選択的な条件付けは表現しにくく、必要なら SPADE 的な空間適応や注意機構への拡張が要ります。第二に、高解像度では未来複数枚の出力が極めて重く(450×300 で 1 サンプル数百 MB)、混合精度やダウンサンプリングがほぼ必須です。第三に、これは設計全般の課題ですが、未来予測を学習の損失に正しく接続しないと、条件付けの良し悪しを検証できません。特徴空間の予測損失(feature reconstruction loss)は「予測した未来特徴」と「実際に時間が進んだときの特徴」を一致させる形で組みますが、潜在が縮退(collapse、全部同じ値に潰れて損失だけ下がる)する罠があり、stop-gradient やターゲットエンコーダの工夫(I-JEPA や BYOL 系の知見)が必要になります。

研究上のオープンな問い。(a) シフトだけで十分か、スケールや空間適応を足すべきか、(b) 何時刻先まで・どの時間刻みで未来を予測するのが計画に有用か、(c) 特徴空間の予測損失をどう設計すれば崩壊せず、かつ計画性能に効くか。

## 発展トピック・研究の最前線

条件付けの最前線は、FiLM のような単純な線形変調から、cross-attention による柔軟な条件注入(拡散モデルの条件付けで主流)、さらにはトークン単位の生成世界モデル(GAIA-1 系)へと広がっています。運転世界モデルでは、Drive-WM(Wang et al., CVPR 2024)や Vista(Gao et al., 2024)のように、行動条件付きで高解像度・マルチビューの未来を生成し、それを計画の評価に使う流れが進んでいます。

特徴空間での未来予測そのものも、V-JEPA / V-JEPA 2(Meta, 2024–2025)が動画の自己教師あり世界モデルとして発展させ、行動条件付き予測やロボット制御へ応用しています。本トピックの「軽量なチャネル毎バイアス + 特徴空間予測」は、これら大規模手法のミニマルな出発点として理解すると見通しが良くなります。
