# ポーズからのego運動算出 (finite differencing)

## ひとことで言うと

ego(エゴ)運動とは、自分の車そのものの動き、すなわち速度・加速度・向きの変化(ヨーレート)・カーブの曲がり具合(曲率)のことです。運転記録には自車の位置や向き(ポーズ)が時刻つきで並んでいるので、それを「隣り合う時刻の差を時間間隔で割る」という計算で速度や角速度に変換します。この「差を取って変化率を出す」操作が finite differencing(有限差分、微分の離散近似)です。こうして作った数値が、運転モデルの入力(過去の運動履歴)と正解(未来の目標軌道)になります。

## 直感的な理解

自動運転モデルを学習するには、「今どう動いているか」「これからどう動くべきか」を数値で表す必要があります。ところがデータセットが直接くれるのは、多くの場合「自車がいつ・どこに・どっちを向いていたか」という生のポーズ列であって、速度・加速度・曲率がそのまま入っているとは限りません。

人間に置き換えると分かりやすいです。地図上に自分の足跡(0.1秒ごとの位置)が点々と打ってあるとします。点と点の間隔を見れば「どれくらい速く歩いたか(速度)」が分かります。間隔の広がり方を見れば「加速しているか減速しているか(加速度)」が分かります。足跡の向きの変わり方を見れば「どれくらい急に曲がったか(ヨーレート・曲率)」が分かります。やっていることは、まさにこの「隣の足跡との差を見る」作業です。連続時間なら微分ですが、データは0.1秒ごとの飛び飛びの値なので、微分の代わりに「隣との差 ÷ 時間間隔」で近似します。これが有限差分です。

## 基礎: 前提となる概念

用語を一つずつ噛み砕きます。
- ポーズ(pose): 自車の位置(x, y, z)と向き(回転)をまとめたもの。「いつ、どこに、どっちを向いて居たか」。回転は後述の四元数やオイラー角で表す。
- 速度(velocity): 単位時間あたりの位置の変化。位置を時間で微分したもの。大きさだけ取れば「速さ(speed)」。
- 加速度(acceleration): 単位時間あたりの速度の変化。速度を時間で微分したもの。アクセル/ブレーキの強さに対応。
- ヨー(yaw): 車の上下軸まわりの向き(進行方向の角度)。「右を向いた/左を向いた」の角度。
- ヨーレート(yaw rate): ヨーの変化率。1秒あたり何ラジアン回ったか。ハンドルの切り具合と速度で決まる回転の速さ。
- 曲率(curvature): カーブの曲がり具合。半径 R の円を描くなら曲率は 1/R。直進なら 0、きついカーブほど大きい。場所や速度に依存しない、道の形そのものを表す量。
- ラジアン: 角度の単位。1周 = 2π ラジアン。π ラジアン = 180度。
- 四元数(quaternion): 3次元の回転を4つの数(w, x, y, z)で表す方法。オイラー角(ヨー・ピッチ・ロール)で起きるジンバルロック(特定姿勢で回転の自由度が縮退して向きが一意に決められなくなる不具合)を避けられるため、ロボティクスで広く使われる。

なぜ生のポーズではなく、これらの運動学量(kinematic features、運動の幾何だけで決まる量)に変換するのか。理由は汎化(学んだことが未知のデータでも通用すること)です。絶対座標「緯度35.6度・経度139.7度」はその場所でしか意味を持ちませんが、「時速30kmで曲率0.05のカーブを曲がっている」は世界中どこでも同じ意味を持ちます。場所に依存しない動きの表現にすることで、モデルが学んだことが別の道でも通用しやすくなります。これは画像認識で「絶対的な画素位置」より「エッジの向き」のような相対的特徴の方が汎化するのと同じ発想です。

## 仕組みを詳しく

有限差分の基本式は単純です。ある量 f が時刻ごとに `f[0], f[1], f[2], ...` と並んでいて、時間間隔が `dt` のとき、変化率(後退差分)はこう近似します。

```
変化率[i] ≒ (f[i] - f[i-1]) / dt
```

10 Hz なら `dt = 0.1` 秒です。なお差分には後退差分(`f[i]-f[i-1]`)・前進差分(`f[i+1]-f[i]`)・中心差分(`(f[i+1]-f[i-1])/2`)があり、中心差分は誤差が小さい(2次精度)反面、未来の値を使うので因果的でない(その時刻までの情報だけでは計算できない)欠点があります。データセットによって「最初から速度・加速度が入っている」場合と「ポーズしかなく自分で差分する」場合があり、実装は3パターンに大別できます。

### パターンA: ポーズしか無い場合(最も基本的な有限差分)

自車ポーズ列(位置と向き)だけが与えられるとき、すべてを差分で作ります。

```python
# 位置 (x, y) の系列から速度・加速度
dx = np.diff(x) / dt           # 速度のx成分
dy = np.diff(y) / dt           # 速度のy成分
speed = np.sqrt(dx**2 + dy**2) # 速さ
accel = np.diff(speed) / dt    # 加速度(speedをもう一度差分)

# 向き(heading)の系列からヨーレート・曲率
heading = np.unwrap(headings)  # 角度の折り返しを連続化(後述)
yaw_rate = np.zeros_like(heading)
yaw_rate[1:] = np.diff(heading) / dt
curvature = np.where(np.abs(speed) > 1e-6, yaw_rate / speed, 0.0)
```

ここに2つの必須の工夫があります。

1. `np.unwrap`(角度の連続化)。角度は通常 -π から +π でぐるっと一周します。179度から-179度へ変わるのは実際には「2度回った」だけなのに、生の数値では「358度の急変」に見えます。差分するとこの偽の急変が巨大なヨーレートに化けてしまう。`unwrap` は折り返しをまたぐ点に ±2π を足し引きして連続値に直し、差分が暴れないようにします。
2. ゼロ除算ガード。曲率は ヨーレート ÷ 速度 です。停車中(速度 ≈ 0)に割ると無限大やノイズの増幅が起きるので、速度が `1e-6` 以下なら曲率を 0 にします。停まっている車に「曲がり具合」は意味がないので妥当な扱いです。

### パターンB: 速度・向きは有るが角速度系は差分する場合

車両状態に speed・heading・前後加速度が最初から入っているなら、速度と加速度はそのまま使い、ヨーレートと曲率だけ向きの差分で作ります。差分の対象が減るぶんノイズも減ります(差分回数が少ないほどノイズ増幅が少ない)。

### パターンC: すべて入っているが高レートな場合(間引き)

高頻度(例 100 Hz)で位置・四元数・速度・加速度・曲率が入っているデータでは、差分は不要で、目的のレート(例 10 Hz)へ間引き(ダウンサンプリング)します。

```python
df_10hz = df.iloc[::10]                              # 10行ごとに1行残す
speed = np.sqrt(vx**2 + vy**2)                       # 速度の大きさ
yaw = rotation.as_euler("ZYX")[:, 0]                 # 四元数→ヨー角に変換
```

単純な `::10` 間引きには落とし穴があります。元の時刻が等間隔でない、欠損行がある、といった場合に時刻の不整合を見落とすと、後段の差分が不正確になります。間引き前にタイムスタンプの単調増加と間隔の均一性を検査するのが堅牢な実装です(この健全性チェックを省くと、まれな時刻飛びが大きな偽加速度を生む)。

### 数値例

時刻 0.0, 0.1, 0.2 秒で向き(heading)が 0.00, 0.05, 0.12 ラジアンだったとします。

```
yaw_rate[1] = (0.05 - 0.00) / 0.1 = 0.5  rad/s
yaw_rate[2] = (0.12 - 0.05) / 0.1 = 0.7  rad/s
```

速度が 10 m/s なら曲率は:

```
curvature[2] = 0.7 / 10 = 0.07  (1/m)  → 半径 1/0.07 ≒ 14m のカーブ
```

半径14mはかなりきつい右左折相当だと直感的に分かります。このように、生の系列を運転モデルが扱いやすい物理量へ翻訳しているのです。

### 出力テンソルの形(典型例)

過去・未来の窓([逐次フレーム読み込み (sequential loading)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0836_sequential-frame-loading))と組み合わせて、次のような固定長テンソルにまとめるのが一般的です。

```
egomotion_history  (256,)  = 64 ステップ × 4 信号 [speed, accel, yaw_rate, curvature]
trajectory_target  (128,)  = 64 ステップ × 2 信号 [accel, curvature]
```

未来側を「加速度と曲率」の2つに絞るのは設計上のポイントです。モデルは各時刻で(どれだけ加減速し、どれだけ曲がるか)= (加速度, 曲率)を予測し、それを時間積分すれば速度・向き・位置、すなわち軌道が復元できます。これは制御入力(アクセルとハンドルに近い量)に近い表現で、運動学的に一貫した軌道を出しやすくする狙いがあります。逆に位置 (x, y) を直接予測させると、各時刻が独立に誤差を持ち、物理的にあり得ない「ワープ」(連続する2点が車では到達不可能な距離で離れる)を生みやすくなります。

## 手法の系譜と主要論文

有限差分そのものは数値解析の基礎(微分の離散近似)で、特定論文に帰属しません。運転・予測の文脈での扱いと、ノイズ対策の系譜を挙げます。

- 運動学量による状態表現。VectorNet(Gao ほか, CVPR 2020)や Trajectron++(Salzmann ほか, ECCV 2020)は、エージェントの過去軌跡から速度・向きの変化を特徴量として使います。生の絶対座標より「動きそのもの」の方が場所に依存せず汎化しやすいためです。共通の課題として、有限差分は微分なのでノイズ(位置の小さな測定誤差)を増幅します。位置が±数cmガタつくと、1回差分した速度はそれ以上にガタつき、2回差分した加速度はさらに荒れます。これは数学的に、微分が高周波成分を強調する(周波数 ω の正弦波を微分すると振幅が ω 倍になる)ことの帰結です。
- 平滑化・物理モデルによる対処の系譜。差分ノイズ増幅への古典的対処が Kinematic Bicycle Model(運動学的自転車モデル、Kong ほか, IV 2015, "Kinematic and dynamic vehicle models for autonomous driving control design")です。車を前後2輪に簡略化した運動方程式で、ステアリング角と速度から物理的に妥当な軌道を生成・平滑化します。これにより、生差分のジッタ(細かい揺れ)を物理的制約で抑えられます。
- 時系列推定によるフィルタリング。カルマンフィルタやその非線形版(EKF/UKF、拡張/無香料カルマンフィルタ)で位置・速度・加速度を同時推定すると、観測ノイズを統計的に抑えつつ滑らかな運動量が得られます。差分の前処理として広く使われます。最小限の安定化(`unwrap` + ゼロ除算ガード)だけで済ませる実装も多いですが、その場合は出力がやや荒くなるトレードオフを受け入れることになります。

## 論文の実験結果(定量データ)

有限差分は単独で評価される対象ではないため、それを使った下流タスクと、ノイズ・平滑化の効果という観点で定量を述べます(値は各論文・関連報告に基づくおよその目安)。

- 入力特徴の質が予測精度に効くことは、軌道予測の標準指標 ADE/FDE(予測軌道の平均/最終地点の正解からのずれ、メートル、小さいほど良い)で測られます。Trajectron++(ECCV 2020)は nuScenes で、運動学に整合した動力学を組み込む(出力を加速度・ヨーレートのような制御量で表し積分する)ことで、座標を直接回帰する素朴な手法より物理的にあり得ない軌道(急なワープなど)を減らし、誤差を改善したと報告しています。これは「加速度・曲率で表して積分する」設計の妥当性を裏づけます。
- ノイズ増幅の実害は前処理アブレーションで現れます。位置観測に測定誤差が乗るデータで生の二階差分(加速度)をそのまま使うと、加速度の分散が観測ノイズの増幅で跳ね上がり、学習が不安定になることが一般に観測されます。平滑化(バイクモデル/カルマン)を挟むと、加速度系列の高周波ノイズが抑えられ、下流の予測誤差と学習安定性が改善する、という報告が複数あります。
- 角度処理の重要性は定性的に明確です。`unwrap` を省くと、±π 境界をまたぐフレームで偽の巨大ヨーレート(数百 rad/s 相当の外れ値)が混入し、そのサンプルが勾配を支配して学習を壊します。これは「1行のバグが学習全体を不安定にする」典型例で、前処理の正しさが定量結果に直結することを示します。なぜ勾配を支配するかというと、損失関数(誤差の大きさ)が外れ値で巨大になり、その1サンプルの更新がモデル全体を引きずるからです。

数値の意味の補足: ADE/FDE は「予測がどれだけ現実からずれたか」を距離で測る指標で、例えば6秒先予測の FDE が数メートル改善することは、交差点やカーブで車線をはみ出すかどうかの差に相当し、安全性の観点で重要です。

## メリット・トレードオフ・限界

メリット
- 生のポーズや速度から、運転モデルが扱いやすい物理量(速度・加速度・ヨーレート・曲率)を一律に作れる。
- 場所に依存しない運動量なので絶対座標より汎化しやすく、教師信号として素直。
- データセットごとに持っている量が違っても(ポーズだけ/速度入り/高レート全部入り)、同じ次元・同じ意味のテンソルに統一でき、モデル側をデータソースに非依存にできる。
- 加速度・曲率で表して積分すれば運動学的に一貫した軌道が復元でき、物理的に破綻した出力を抑えやすい。

トレードオフ・限界
- 有限差分は微分なので測定ノイズを増幅する。位置がガタつくと速度・加速度はもっと荒れる。一階より二階差分(加速度)が、さらに角速度系がノイズに敏感。
- 角度の折り返し(±π)や停車時のゼロ除算など、素朴に差分すると壊れる箇所があり、`unwrap` やガードが必須。これらは「分かっていれば1行」だが、抜けると静かに学習を壊す。
- 高レートからの単純間引き(`::10`)は、欠損行や時刻の不均一を見落とすと不正確になる。間引き前のサンプリング健全性チェックが要る。
- 平滑化を入れない最小実装では出力が荒くなりうる。実データでの数値安定性(差分由来の外れ値が loss を発散させないか)は別途検証が必要で、平滑化・フィルタリングの有無は精度と計算コストのトレードオフになる。未解決寄りの課題として、観測ノイズ特性が異なる複数データセットを混ぜて学習するときの、運動量の正規化・整合の取り方が挙げられます。

## 発展トピック・研究の最前線

近年は、ポーズの前処理を学習に統合する方向が進んでいます。観測ノイズを陽にモデル化する微分可能カルマンフィルタ(differentiable Kalman filter、フィルタの中をニューラルネットと一緒に逆伝播で学習する)や、ニューラル ODE(常微分方程式をニューラルネットで表し連続時間ダイナミクスを直接学ぶ)は、離散差分の限界を超えようとする試みです。出力表現としては、加速度・曲率のような制御量で予測して運動学/動力学モデルで積分する「ダイナミクス制約付きデコーダ」が、物理的に妥当な軌道を保証する標準的設計になりつつあります。マルチセンサー(GPS+IMU+ホイールオドメトリ)を融合して高精度な ego 運動を得るオドメトリ/SLAM(自己位置推定と地図構築)の知見も、教師信号の品質向上に直結します。
