# affine_grid と grid_sample による微分可能ワープ

## ひとことで言うと
画像や特徴マップを「平行移動・回転・拡大縮小」させる操作を、PyTorch の `affine_grid` と `grid_sample` という2つの関数だけで実現できます。しかもこの操作は微分可能(differentiable)、つまり「どう動かしたら損失が減るか」の勾配(各重みをどちらへ動かすべきかの傾き)がワープを通り抜けて backbone(特徴抽出ネット)まで届くので、学習を止めずに幾何変換を組み込めます。BEVワープ([ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp))や空間変換を実装する実用的な道具です。

## 直感的な理解
方眼紙に絵が描いてあって、それを右へ少しずらした新しい方眼紙を作りたいとします。素朴には「各マスの絵を右隣のマスへコピー」すればよさそうですが、もし「2.5マスぶん右へ」のように半端な量だとどうでしょう。2.5マス先には元の絵がぴったり来ないので、左右のマスの絵を混ぜて中間の絵を作る必要があります。これが補間です。

`affine_grid` と `grid_sample` は、この「半端な量でもなめらかにずらす」作業を機械にやらせる道具です。さらに重要なのは、この作業が「微分できる」こと。微分できるとは、出来上がった絵を見て「もっと右にずらせば損失が減る」と分かったとき、その情報を絵を描いた元の段階(ニューラルネット)まで遡って伝えられる、という意味です。ずらす操作を学習の輪の中に組み込めるのです。

## 基礎: 前提となる概念

- **特徴マップ(feature map)**: 縦横のマス目に数字ベクトルが並んだもの。形 `[B, C, H, W]`(B バッチ、C チャンネル数、H 高さ、W 幅)。画像は C=3 の特殊な場合。

- **補間(interpolation)**: 近くの値を混ぜて中間の値を作ること。**バイリニア補間(bilinear interpolation)** は、参照したい位置の上下左右4マスの値を、距離に応じた重みで混ぜる方式。位置が4マスのちょうど中央なら4つを均等に、片方へ寄っていればその近いマスを重く混ぜます。

- **微分可能と自動微分**: PyTorch は計算をたどって勾配を自動計算します(自動微分)。`for` ループで添字をずらして値をコピーするような自作処理は、添字操作が離散的で勾配が通らないことが多く、その先のネットを学習できません。「微分可能な」演算で書くことが、学習信号を流す条件です。

- **アフィン変換(affine transformation)**: 平行移動・回転・スケール・せん断を1つの行列で表せる線形変換。2次元なら 2×3 の行列で表します。

特徴マップを空間的に動かしたい場面は多くあります。たとえば「今の BEV を、自車が動いた未来の座標系へ貼り直す(ワープ)」。素朴な自作ループには2つの致命的問題があります。(1) 移動量が整数マスとは限らず、半端な量には補間が要る。(2) 自作のインデックス操作は勾配が通らず学習が止まる。`affine_grid` と `grid_sample` は、この補間つき空間変換を1セットで提供し、内部が全部微分可能なので両方を同時に解決します。

## 仕組みを詳しく

2段構えです。

### ステップ1: affine_grid で「どこから値を取ってくるか」の地図を作る

アフィン変換を 2×3 の行列 `theta` で指定します。

```python
import torch
import torch.nn.functional as F

# theta: [B, 2, 3] のアフィン行列
# 例: 平行移動だけ(回転なし)。tx, ty は正規化座標(-1〜1)での移動量
theta = torch.tensor([[[1.0, 0.0, tx],
                       [0.0, 1.0, ty]]])      # shape [1, 2, 3]

grid = F.affine_grid(theta, size=(B, C, H, W), align_corners=False)
# grid: [B, H, W, 2] — 出力の各マス(h,w)について「入力のどの正規化座標から
#       サンプリングするか」を表す (x, y) が入っている
```

ここで超重要なのが正規化座標(normalized coordinates)という考え方です。これらの関数はピクセル単位ではなく、画像全体を -1 から +1 へ正規化した座標で動きます。左端が -1、右端が +1、中央が 0。だから「ピクセルで何マス動かすか」を必ずこのスケールへ換算してから `theta` へ入れます。換算を忘れると桁が合わず壊れます。

回転を入れる場合は左上 2×2 が回転行列になります(θ をヨー角として)。

```
[ cosθ  -sinθ  tx ]
[ sinθ   cosθ  ty ]
```

補足として、`affine_grid` は「出力座標 → 入力座標」の逆向きマッピング(inverse warping)を作ります。出力の各マスについて「入力のどこから値を持ってくるか」を指すので、出力に穴が空かない(全マスが埋まる)のが利点です。順方向に値をばらまく forward warping だと穴や重なりが生じやすいため、補間つきリサンプリングでは逆向きが標準です。

### ステップ2: grid_sample で実際に値を貼り直す

```python
warped = F.grid_sample(z_t, grid, mode="bilinear",
                       padding_mode="zeros", align_corners=False)
# z_t:     [B, C, H, W] 入力特徴
# warped:  [B, C, H, W] ワープ後の特徴(shape は同じ)
```

`grid_sample` は出力の各マスについて「grid が指す入力座標」から値を取り、バイリニア補間して埋めます。`padding_mode="zeros"` は参照先が画像外(動かした結果、今は見えていない場所)なら 0 で埋める設定です。`border`(端の値を引き伸ばす)や `reflection`(折り返す)も選べます。

### 数値例(前進 8m のワープ)

BEV が H=450(前後方向)、1マス0.4mとします。自車が8m前進 = 20マスぶん景色が手前へずれる。これを正規化座標に直すと、全体450マスが 2.0(-1〜1)に対応するので、20マス = `20/450 * 2 ≈ 0.0889`。これを `theta` の平行移動成分へ入れ `grid_sample` を通すと、止まっている景色が一様に20マスぶん移動した特徴が得られます。

### 勾配が通る、とはどういうことか

`grid_sample` のバイリニア補間は「4近傍の重み付き和」なので、出力は入力 `z_t` の値について微分可能です。式で書くと、出力位置の値は `out = (1-a)(1-b)·v00 + a(1-b)·v10 + (1-a)b·v01 + ab·v11`(`v00..v11` は4近傍の入力値、`a, b` はサブピクセル位置の小数部)という形で、これは `v00..v11` についても、サンプリング座標 `a, b` についても線形/双線形に微分できます。したがってワープ後に計算された損失の勾配は `warped` → `z_t` → さらにその先の backbone まで逆流します。一方 `theta`(移動量)を学習しないパラメータフリー運用にする場合(既知の ego 移動量を使うだけ)は、そこへ勾配を流す必要はありません。逆に STN のように `theta` を学習する場合は、grid と補間を通して `theta` についても微分可能です(ただし `theta` 経由の勾配は不安定になりやすい)。

なぜ `theta` 学習が不安定になりやすいかも噛み砕くと、座標 `a, b` についての勾配は「隣接ピクセルの値の差」に比例します(`∂out/∂a = (1-b)(v10-v00) + b(v11-v01)`)。つまり平坦な領域では勾配がほぼゼロで信号が消え、エッジ付近でだけ大きくなる。この空間的に偏った勾配が、変換パラメータの学習を局所解に陥りやすくします。既知量で `theta` を与えるパラメータフリー運用は、この厄介な勾配地形を最初から避けられる利点があります。

## 手法の系譜と主要論文

- **STN (Jaderberg et al., NeurIPS 2015, arXiv:1506.02025)**: `affine_grid` + `grid_sample` の元になった発想を提案。ネットの中に「入力を自動で平行移動・回転・拡大縮小して整える微分可能なモジュール」を埋め込めることを示した。動機は、CNN が位置・向きのズレに弱い問題をデータから変換を学んで吸収させること。効果は分類精度の向上。トレードオフは変換パラメータの学習が不安定になりやすいこと。

- **Appearance Flow (Zhou et al., ECCV 2016, arXiv:1605.03557)**: 入力ビューから別視点の画像を、ピクセルをどこから持ってくるかの流れ場(flow)で再サンプリングして合成。微分可能ワープでビュー合成を学ぶ初期例。

- **SfMLearner (Zhou et al., CVPR 2017, arXiv:1704.07813)**: 推定したカメラ運動と深度でフレームをワープし「次のフレームを再構成」する自己教師あり奥行き・自己運動学習。微分可能ワープで測光損失(photometric loss、再構成画像と実画像の画素差)の勾配を深度・運動推定ネットへ流す枠組み。BEVワープと同じ道具立てで、動く物体やオクルージョンでワープ前提が崩れる点も共通します。

- **Deformable Convolution (Dai et al., ICCV 2017, arXiv:1703.06211)**: サンプリング位置自体を学習でずらす発想。grid_sample 的なサンプリングを畳み込みの各点へ一般化したもので、微分可能リサンプリングの応用系として系譜上重要です。

ego-warp のようなパラメータフリー運用は、STN の機構だけ借りて変換は既知量で与える割り切りで、学習の不安定さを避けつつ幾何変換を確実に組み込む実用的な選択です。

## 論文の実験結果(定量データ)

- **STN**: 歪み MNIST(回転・スケール・並進を加えた手書き数字)で、空間変換モジュールを挿入すると分類誤り率が大幅に低下(歪み条件によってはベースラインの誤り率の数分の1)。CUB-200 鳥分類でも、複数のSTNが体の部位へ自動でズームすることで精度向上を報告。微分可能な空間変換がCNNの幾何脆弱性を補えることを示した一方、`theta` 学習の不安定性も指摘されています。

- **SfMLearner**: KITTI 奥行きベンチで、ラベル(正解深度)を一切使わずに、当時の教師あり手法に迫る精度を報告。指標は絶対相対誤差(Abs Rel、低いほど良い)などで、ステレオや LiDAR の正解なしに単眼動画だけから奥行きを学べることを示した点が画期的。動的物体・オクルージョン領域では誤差が増え、それを explainability mask で除外する工夫が必要だったことも報告され、ワープ前提の限界を定量的に示しました。

- **Deformable ConvNets**: COCO 物体検出・PASCAL VOC セグメンテーションで、通常の畳み込みを変形畳み込みへ置き換えると mAP / mIoU(高いほど良い)が一貫して向上。学習可能なサンプリング位置(微分可能リサンプリング)が幾何変動への適応に効くことを示しました。

これらは、微分可能ワープ/リサンプリングが「幾何変動の吸収(STN)」「ラベルなし学習信号の伝播(SfMLearner)」「特徴の幾何適応(Deformable)」と、用途を問わず実効性を持つことを定量的に裏づけます。同時に、ワープ前提が崩れる動的領域での誤差増という限界も共通して示されています。

## メリット・トレードオフ・限界

メリット
- 補間つきの空間変換と自動微分が1セットで手に入る。自作ループの罠(非整数移動・勾配が通らない)を回避できる。
- 学習を止めずに幾何変換を挟める(勾配が backbone まで届く)。
- shape を変えない(`[B,C,H,W]` → `[B,C,H,W]`)ので既存パイプラインへ差し込みやすい。
- GPU で高速(内部は最適化済みカーネル)。

トレードオフ・限界
- 正規化座標(-1〜1)への換算を間違えやすい。ピクセル移動量をそのまま入れると桁が合わず壊れる。
- `align_corners` の True/False で1ピクセルぶんの座標解釈がずれる。生成側と揃えないと微妙にずれる(よくあるバグ)。中身を噛み砕くと、`align_corners=True` は「正規化座標 -1 と +1 が端ピクセルの中心に対応する」解釈、`False` は「-1 と +1 が端ピクセルの外縁(セルの境界)に対応する」解釈で、両者は `(N-1)/N` 倍ぶんスケールが違います。H=450 なら約 0.22% のずれで、見た目には気づきにくいが、ワープを自己回帰で何十回も繰り返すと累積して無視できなくなります。`affine_grid` と `grid_sample` で同じ設定を使うこと、そして座標換算式と整合させることが必須です。
- `padding_mode="zeros"` だと動かして見えなくなった領域が 0 になり、そこは情報がない(残差で補えないと欠損のまま)。
- バイリニア補間は値をなめらかに混ぜるので、鋭いエッジ(物体境界)がわずかに鈍る。回転を繰り返すと累積でぼける。
- `theta` を学習させる場合(STN流)は勾配が不安定になりやすい。既知量で与えるパラメータフリー運用なら回避できる。

## 発展トピック・研究の最前線

`grid_sample` は深度・フローを使った3次元ワープ(forward-backward consistency による自己教師あり、複数フレームのコストボリューム構成)、ビュー合成(NeRF 系の微分可能レンダリングの前段)、特徴アライメント(動画の時間整合)など、微分可能幾何の基盤演算として広く使われ続けています。一方で補間によるエッジのぼけや、視野外領域の欠損は本質的な限界で、これを補うために学習する残差(本トピックの `Δ_dynamic`)、複数解像度のワープ、占有を併用するハイブリッドなどが研究されています。`align_corners` 由来の座標規約バグは実装上の定番の落とし穴で、テストを規約非依存の不変条件(移動ゼロなら恒等、平行移動で中身が動く、勾配が流れる)で書くのが堅実です。
