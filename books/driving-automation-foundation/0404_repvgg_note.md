# RepVGG (構造的再パラメータ化)

## ひとことで言うと
RepVGG は、学習するときと推論(実際に使う)ときでネットワークの構造を変える手法です。学習時は精度が出やすい複数の枝(分岐路)を持つ構造で訓練し、推論時はその複数の枝を数学的に1本の畳み込みへ畳み込んで、VGG のような「3×3畳み込みをひたすら積んだだけ」の単純な1本道(single-path)に変換します。この変換を構造的再パラメータ化(structural re-parameterization)と呼び、これによって「学習しやすさ」と「推論の速さ」を両立させます。

## 直感的な理解
ネットワーク設計には2つの相反する要求があります。

一方で、学習時は分岐がある構造の方が精度が出やすいことが分かっています。代表が ResNet の残差接続(skip connection、入力をそのまま出力に足す近道)です。これがあると、層を深くしても学習の手がかりとなる勾配が消えにくく、安定して学習できます。

他方で、推論時は分岐のないまっすぐな1本道の方が速いです。分岐があると、各枝の途中結果を一時的にメモリに保持して後で足し合わせる必要があり、メモリアクセスが増えて GPU の演算ユニットが遊んでしまうからです。VGG のように「3×3畳み込みと活性化関数だけを積む」単路構造は、現代の GPU やアクセラレータが最も得意とする形です。

従来はどちらかを諦めるしかありませんでした。分岐を使えば精度は上がるが推論が遅い。単路にすれば速いが学習が難しい。RepVGG の発想は「学習と推論で構造を変えてしまえば両取りできる」というものです。学習用構造と推論用構造が数学的に完全に等価になるよう設計すれば、推論時に単路へ変換しても精度は1ビットも落ちません。料理にたとえれば、調理中は便利な専用器具をいくつも使うが、完成品(推論)はシンプルな一皿として出す、しかも味は完全に同じ、という関係です。

## 基礎: 前提となる概念
いくつかの用語を順に噛み砕きます。

「畳み込み(convolution)」は、小さな重み行列(カーネル)を画像上で滑らせ、近傍画素との積和を取る線形操作です。線形であることが後で決定的に効きます。線形とは「入力を定数倍したら出力も同じ定数倍になり、入力の和には出力の和が対応する」性質で、複数の線形操作を足し合わせたものは、別の1つの線形操作にまとめられます。

「残差接続(residual / skip connection)」は、ある層の出力に入力をそのまま足す近道です。出力 = F(x) + x の形で、深いネットワークで勾配が消える問題(層を遡るうちに学習信号が小さくなって学習が進まなくなる現象)を緩和します。He et al. の ResNet (2016) で導入され、深層学習の精度を一段押し上げました。

「バッチ正規化(Batch Normalization, BN)」は、層の出力を平均0・分散1付近に整える処理で、学習を安定・高速化します。重要な性質として、推論時の BN は固定の線形変換(各チャンネルに定数を掛けて定数を足すだけ)になります。学習中は各ミニバッチの統計で正規化しますが、推論時には学習中に集めた移動平均の統計を使うため、入力に依らない固定の式になるわけです。

「単路 / 多枝(single-path / multi-branch)」は、データの通り道が1本か分岐するかの違いです。

「FLOPs と実速度(latency / throughput)」は別物です。FLOPs は浮動小数点演算の回数(理論計算量)、実速度は実際にかかる時間です。分岐構造はメモリアクセスや小演算のオーバーヘッドで、同じ FLOPs でも実速度が遅くなりがちです。これは「計算強度(arithmetic intensity、1バイトのメモリ転送あたり何回計算するか)」が低いと、GPU が演算より先にメモリ帯域で詰まる(memory-bound になる)ためです。RepVGG が狙うのは FLOPs ではなく実速度の改善です。

「再パラメータ化(re-parameterization)」は、同じ関数を別のパラメータ表現で書き直すことです。RepVGG では、複数枝のパラメータを等価な1枝のパラメータへ書き換えます。

## 仕組みを詳しく
中心は「学習用ブロック」と「推論用への変換」です。

学習時のブロック構造
各層は、同じ入力 x に対して次の3つの枝を並列に計算し、結果を足し合わせます。

  y = ReLU( BN( Conv3x3(x) ) + BN( Conv1x1(x) ) + BN( x ) )

1. 3×3畳み込み枝(メインの枝)
2. 1×1畳み込み枝(補助の枝。チャンネルごとの線形変換)
3. 恒等枝(identity、入力をそのまま通す枝。入出力のチャンネル数と解像度が一致するときのみ)

各枝の後にバッチ正規化が付きます。この3枝は ResNet の残差構造に似ており、勾配を流しやすくして学習を安定させ、精度を上げます。3枝はそれぞれ受容野や役割が異なる(3×3は近傍、1×1はチャンネル混合、恒等は素通し)ので、暗黙のアンサンブル(複数の異なるモデルの平均をとる効果)のように働きます。

推論時への変換(構造的再パラメータ化)
推論前に、この3枝を1個の3×3畳み込みへ統合します。鍵は「全ての枝が線形演算であること」です(活性化 ReLU は足し合わせの後にあるので、足し合わせる部分はすべて線形)。線形変換は足し合わせても1つの線形変換にまとめられます。手順は3段階です。

第一に、BN を直前の畳み込みに吸収します。推論時の BN は y = γ(z - μ)/σ + β という固定の線形式です(μ,σ は学習中に集めた平均・標準偏差、γ,β は学習パラメータ)。これは畳み込みの重み W とバイアス b に対し、W' = (γ/σ) W、b' = (γ/σ)(b - μ) + β と書き換えれば、畳み込みの中に溶け込みます。これで各枝が「BN なしの畳み込み」になります。

第二に、全ての枝を 3×3 畳み込みの形にそろえます。
- 1×1畳み込みは、3×3カーネルの中央だけに値が入り周囲がゼロの 3×3 畳み込みとみなせます(zero-padding でカーネルを膨らませる)。
- 恒等枝は、各チャンネルが自分自身だけを中央で1倍に通す 3×3 畳み込み(中央が1、他がゼロ、チャンネル間はゼロ)とみなせます。これは入力をそのまま出力する単位行列的なカーネルです。

第三に、3つの 3×3 重みを単純に足し合わせます。畳み込みは線形なので、同じ入力に各枝を適用して足すことと、重みを足してから1回畳み込むことは数学的に同一です。

  W_merged = W3x3' + W1x1_padded' + W_identity'
  b_merged = b3x3' + b1x1' + b_identity'

数値イメージ。ある出力チャンネルの3枝が次の 3×3 重みを出すとします。
- 3×3枝: 任意の 3×3 行列 W3
- 1×1枝を 3×3 に埋めたもの: 中央だけ非ゼロの W1
- 恒等枝を 3×3 に埋めたもの: 中央が1で他ゼロの Wi
変換後は W = W3 + W1 + Wi(BN吸収済み)。バイアスも同様に合算します。

変換後の推論ネットワークは「3×3畳み込み + 活性化(ReLU)」だけがまっすぐ並ぶ VGG 型単路になります。出力は学習時と完全一致(数学的に等価)なので精度は変わりません。

ASCII 図で before / after を示します。

  [学習時]                          [推論時(変換後)]
   x ─┬─[Conv3x3]─[BN]─┐             x ─[Conv3x3 merged]─[ReLU]─→
      ├─[Conv1x1]─[BN]─┤(+)─[ReLU]→
      └─[Identity]─[BN]─┘

効果は明確です。学習時は3枝で精度を稼ぎ、推論時は単路なので GPU 上で高速・省メモリに動きます。なぜ単路が速いのか改めて言えば、分岐の中間結果を保持する必要がなく(メモリアクセス削減)、演算が3×3畳み込みという単一種類に揃うので、GPU の演算ユニットを高い稼働率で回せるからです。NVIDIA の cuDNN などは 3×3 stride-1 畳み込みに専用の最適化(Winograd 変換など)を持っており、その最速経路だけを使えることも実速度に効きます。

## 手法の系譜と主要論文
RepVGG は2つの古典の良いとこ取りです。

Simonyan & Zisserman の VGG (ICLR 2015) は、3×3畳み込みだけを深く積むという極めて単純な設計で高精度を出し、長くベースラインとして使われました。しかし分岐構造を持つ後発モデルに精度で抜かれ、単路は時代遅れと見なされていました。

He et al. の ResNet (CVPR 2016) は残差接続を導入し、深いネットワークの学習を可能にして精度の天井を引き上げました。以降ほぼ全ての高精度 CNN が何らかの分岐を持つようになりました。残差接続には「複数経路の暗黙アンサンブル」という解釈(Veit et al., 2016)もあり、RepVGG の3枝もこの効果を学習時だけ借りていると見られます。

Ding et al., "RepVGG: Making VGG-style ConvNets Great Again", CVPR 2021(清華大学・MEGVII ほか)は「学習時に分岐の利点を得て、推論時に単路の速度を得る」再パラメータ化を提案し、VGG 型単路を実用精度・実速度の両面で復権させました。新規性は、BN 吸収と枝の 3×3 化という具体的な等価変換手順を確立した点です。

この「学習用構造と推論用構造を分離する」思想はその後広がりました。同じ著者グループの RepLKNet (CVPR 2022) は、31×31 のような超大カーネルを小カーネル枝の再パラメータ化で安定に学習する手法を示しました。RepOptimizer は再パラメータ化を最適化器の側に移しました。Vasu et al. の MobileOne (Apple, CVPR 2023) は、モバイル端末の実レイテンシを直接最適化対象にし、学習時に過剰分岐(over-parameterization)を持たせて推論時に単純化する設計で、iPhone 上で1ミリ秒級の推論を達成しました。これらはいずれも RepVGG の「構造を学習と推論で切り替える」共通思想の発展です。

## 論文の実験結果(定量データ)
RepVGG 論文の主要評価を、指標の意味とともに説明します。

ImageNet-1K 分類(約120万枚・1000クラス。指標は top-1 accuracy = 最有力予測が正解だった割合)と、実速度(NVIDIA 1080Ti 上のスループット、images/sec = 1秒間に処理できる画像数。大きいほど速い)。論文の主張は「同じ精度ならより速い、同じ速度ならより高精度」です。

代表的な数値として、RepVGG-A0 は top-1 約 72.4%、RepVGG-B0 は約 75.1% などの軽量帯から、RepVGG-B2 は約 78.8%、グループ畳み込みを足した RepVGG-B3g4 などは約 80% 前後まで届きます。比較では、RepVGG-B2 は ResNet-50 / ResNet-101 と同等以上の精度を、1080Ti 上で大幅に高いスループット(報告ではおよそ 1.5〜2 倍前後の速さ)で達成しました。RepVGG-A0 は ResNet-18 より高精度かつ高速、というように軽量帯でも優位が示されています。重要なのは、FLOPs が同等でも実速度で RepVGG が勝つことが多い点です。これは単路化によるメモリアクセス削減と演算密度向上の効果で、FLOPs だけ見ていると見落とす実用上の差です。論文は EfficientNet などとの比較でも、EfficientNet は FLOPs こそ小さいが実速度は遅いことを指摘し、「FLOPs より実速度を見るべき」という主張を数値で裏付けました。

アブレーション。論文の重要な分析は次の通りです。第一に、3枝(3×3・1×1・恒等)のうち枝を抜くと学習後の精度が下がります。特に恒等枝(残差に相当)を抜くと学習が不安定になり精度が明確に落ち、残差的近道の重要性が確認されました。1×1枝を抜いても精度がやや下がります。報告では、3枝そろえた構成が単路学習に対して 1〜2 ポイント程度の精度上乗せをもたらしました。つまり多枝はアンサンブル的に効いており、単に飾りではありません。第二に、再パラメータ化の前後で出力が完全一致することを実測で確認し、変換が数学的に厳密であることを示しました。第三に、学習を最初から単路で行うと(同じ単路構造でも)精度が出ず、「多枝で学習してから単路に変換する」ことが本質であると示しました。これは RepVGG の核心的主張の実証です。

## メリット・トレードオフ・限界
メリット
- 学習時は分岐構造で精度を稼ぎ、推論時は単路で高速・省メモリという両取りができる。
- 推論構造が3×3畳み込みと活性化だけなので、GPU・アクセラレータが最も得意とする形で実速度が出る。
- 変換は数学的に完全等価で、精度を1ビットも落とさない。
- FLOPs ではなく実レイテンシ・スループットを改善するため、車載リアルタイム推論や低レイテンシ要件と相性が良い。
- 再パラメータ化の思想は他手法(MobileOne, RepLKNet 等)へ広く応用できる。

トレードオフ・限界
- 学習時は3枝分のメモリと計算が必要で、学習自体は重くなる。
- 推論前に再パラメータ化の変換ステップが1回必要で、デプロイ手順が一段増える。
- 単路構造はパラメータ効率(同じ精度に必要なパラメータ数)では分岐を残したモデルにやや劣る場合がある。
- 構造が単純な分、Transformer 系のような長距離依存・大域文脈の表現力は限定的。広い受容野を得るには深く積むしかない。
- 学習後に構造が変わるため、量子化や追加学習(fine-tune)の扱いに注意が要る。BN を吸収済みの単路を量子化すると、分岐時の数値分布と変わり精度劣化が起きやすく、量子化を意識した変換や再学習が必要になることがある。具体的には、3枝を足し合わせた重みは値の分布が広がりやすく、8ビット量子化の刻み幅に収まりにくい。これは再パラメータ化系の共通の実務上の落とし穴です。

## 発展トピック・研究の最前線
構造的再パラメータ化は「学習時の表現力と推論時の効率を分離する」一般的な道具で、応用範囲は広がり続けています。RepLKNet は超大カーネルの安定学習に、MobileOne はモバイル実レイテンシ最適化に、それぞれ展開しました。さらに学ぶなら、まず「線形演算は融合できる」という原理(Conv-BN 融合は推論最適化の基本テクニックで、TensorRT などの推論エンジンが自動で行う)を押さえると、RepVGG が手作業の再パラメータ化を学習構造設計にまで拡張したものだと理解できます。研究の最前線では、量子化に強い再パラメータ化(QARepVGG, Chu et al. 2022 など、足し合わせ後の重み分布を量子化フレンドリーに保つ枝設計)が課題として残っており、エッジ・車載アクセラレータへの実展開で重要になります。QARepVGG は BN の位置や枝設計を見直し、再パラメータ化後の重み・活性の分散を抑えて INT8 量子化での精度劣化を大幅に減らしました。FLOPs と実レイテンシの乖離をどう設計指標に組み込むか、というハードウェア対応設計(hardware-aware design)の議論も、RepVGG が提起した重要な視点で、近年の効率モデル設計の前提になっています。
