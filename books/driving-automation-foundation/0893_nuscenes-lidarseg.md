# nuScenes-lidarseg / Panoptic

## ひとことで言うと

LiDAR セグメンテーション拡張とは、ある運転データセットの LiDAR(レーザー距離計)点群の「1 点 1 点」に、それが何であるか(車・歩行者・道路・植物…)のラベルを付けた拡張です。種類で塗り分けるだけのものをセマンティック版、さらに「同じ種類でも個体を区別する」情報を足したものをパノプティック版と呼びます。後者では「車を 1 台ずつ数える」「同じ車を連続フレームで追い続ける」ことまでできます。

ここで「点群(point cloud)」とは、LiDAR が周囲に反射したレーザーの戻りから作る、3 次元空間に散らばった大量の点の集まりです。各点は (x, y, z) 座標と反射の強さ(intensity, レーザーが戻ってきた強度。素材で変わる)を持ちます。

## 直感的な理解

物体を囲む 3D の箱(バウンディングボックス)だけでは答えられない問いがあります。「路面はどこからどこまでか(走行可能領域)」「縁石・歩道・植え込みのような、数えないが知っておきたい背景はどこか」「いま見ているこの 1 点は車の点なのか地面の点なのか」。箱は地面のような広がりのあるものを表せず、輪郭も粗い四角でしか近似できません。トレーラーや曲がった連結バスのように箱に収まらない形状もあります。

自動運転では、走行できる路面の正確な形や障害物の細かい輪郭を知ることが安全に直結します。これを得るのがセグメンテーション(segmentation, 領域分割)です。点群にラベルを付けることで「ここまでが走れる路面、ここから先は縁石」という境界を点の精度で得られます。さらに、車が 2 台並んで停まっているとき両方を同じ「車」ラベルにしてしまうと「2 台ある」ことが分からないので、個体ごとの区別(インスタンス)を組み合わせる必要が出てきます。たとえば駐車場で出庫を予測するには、隣り合う車をきちんと別個体として認識しなければなりません。

## 基礎: 前提となる概念

セグメンテーションには 3 つの段階があります。

- セマンティックセグメンテーション(semantic segmentation): 各要素に「何であるか」のクラスラベルを付ける。点群版なら各点に「車」「路面」などの種別を付ける。個体は区別しない。
- インスタンスセグメンテーション(instance segmentation): 数えられる物体(things)を個体ごとに分ける。「1 番の車」「2 番の車」。
- パノプティックセグメンテーション(panoptic segmentation): Kirillov et al., CVPR 2019 が統一概念として提唱。数えない背景(stuff, 路面・空・植生)はセマンティックに塗り、数える物体(things, 車・人)はインスタンスに分ける、という両者を 1 枚に統合したもの。すべての点に「クラス + (物体なら)個体 ID」が付く。

things と stuff の区別が鍵です。things は「数える対象(可算名詞的)」で個体 ID を持ち、stuff は「数えない広がり(不可算名詞的)」で個体 ID を持ちません。この区別がパノプティックの設計思想そのものです。なぜ統合するかというと、従来はセマンティック研究とインスタンス検出研究が別々に発展しており、「シーン全体を漏れなく・重複なく説明する」一枚の出力を作る共通の物差しが無かったからです。

LiDAR の物理的性質も理解が必要です。回転式 LiDAR は複数のレーザービーム(たとえば 32 本)を縦に並べ、回転させて全周をスキャンします。点は車から放射状に広がり、遠いほど隣の点との間隔が開いて疎(まばら)になります。32 ビームのような中解像度センサーでは、20 m 先の歩行者は数十点、50 m 先では数点しか当たらず、何であるかの判別が難しくなります。この「遠いほど疎」という不均一さが、点群処理を画像処理と根本的に違う問題にします。画像は画素が均一な格子に並びますが、点群は密度が場所によって何桁も変わります。

## 仕組みを詳しく

代表的な LiDAR セグメンテーション拡張(nuScenes-lidarseg)のスペック:

- 対象: 各キーフレーム(ラベル付き瞬間, 2 Hz)の 32 ビーム LiDAR 点群。
- ラベル: 点ごとに 32 クラス(評価では背景や稀少クラスを統合して 16 クラスに集約)。車・トラック・バス・歩行者・自転車などの things と、走行可能路面・歩道・地形・植生・建物などの stuff を含む。
- 規模: 約 14 億点(約 1.4 billion points)に手作業でラベル付け、約 4 万キーフレーム。
- Panoptic 版はこれに個体 ID と時間方向の追跡(panoptic tracking)を追加。

データの形を具体的に書くと、1 キーフレームの点群が N 点(32 ビーム LiDAR なら 3 万点前後)あるとき:

```
points       = [N, 4]    # 各行 (x, y, z, intensity)
sem_labels   = [N]       # 各点のクラス番号 (例: 0=noise, 1=car, 24=drivable_surface, ...)
inst_ids     = [N]       # Panoptic 版のみ。things クラスの点には個体ごとの番号、stuff は 0
```

things と stuff の扱いを ASCII で示すと:

```
   走行路面(stuff)                車1(thing, id=3)   車2(thing, id=4)
  ················               ▓▓▓▓               ▒▒▒▒
  ················               ▓▓▓▓               ▒▒▒▒
  → 全部同じ "drivable" ラベル    → 同じ "car" でも id で区別
```

これを解くモデルの典型は、点群を入力に各点のクラス(と個体)を出力する 3D セグメンテーションネットワークです。

```
入力:  [N, 4] の点群
出力:  [N, クラス数] のクラス確率 → argmax で各点に最尤クラスを割り当て
       (Panoptic 版はさらに各点の個体埋め込み or 中心へのオフセットを出してクラスタリング)
```

点群を扱う表現には大きく 3 流派があります。(1) 点を直接ニューラルネットに入れる point-based(PointNet 系)、(2) 空間を小箱に区切る voxel-based、(3) 球面や鳥瞰へ投影して 2D 画像のように扱う projection-based(レンジ画像など)。それぞれ精度と速度のトレードオフがあり、LiDAR の放射状・疎という性質にどう適応するかが性能を分けます。

評価指標を初心者向けに説明します。

- mIoU(mean Intersection over Union, 平均 IoU): セマンティックの基本指標。各クラスについて IoU =(予測領域と正解領域の重なりの点数)÷(両者の和集合の点数)を計算し、全クラスで平均する。1 に近いほど良い。たとえば mIoU 0.75 は「平均して予測と正解が 75 % 重なっている」を意味する。クラス平均なので、点数の少ない稀少クラスも大クラスと同じ重みで効くのがポイント。
- PQ(Panoptic Quality): パノプティックの総合指標で、PQ = SQ × RQ と分解できる。SQ(Segmentation Quality)は正しく対応づいたセグメント同士の平均 IoU で「形がどれだけ合っているか」、RQ(Recognition Quality)は F1 スコア的な量で「個体を取り違えず過不足なく検出できたか(TP/(TP + 0.5FP + 0.5FN))」を測る。両者の積なので、形が合っていても検出漏れが多ければ PQ は下がる。IoU 0.5 を超えたセグメント対だけを正解対応(TP)とみなす設計。
- PAT(Panoptic Tracking) / LSTQ などの追跡指標: 時間をまたいで同じ個体に同じ ID を保てたか(ID の一貫性)を測る。追跡が途切れて ID が入れ替わるとペナルティになる。Panoptic nuScenes はパノプティックを時間方向へ拡張し、瞬間の品質と追跡の一貫性を同時に評価する枠組みを提案した。

## 手法の系譜と主要論文

点群セグメンテーションのベンチマークと手法は、互いに刺激し合って発展してきました。

- SemanticKITTI: Behley, Garbade et al., "SemanticKITTI: A Dataset for Semantic Scene Understanding of LiDAR Sequences", ICCV 2019。KITTI の 64 ビーム LiDAR シーケンスに点ごとのセマンティックラベル(評価 19 クラス、約 43 億点)を付けた先駆的データセット。点群セグメンテーション研究の出発点になった。
- パノプティックの概念: Kirillov, He, Girshick, Rother, Dollár, "Panoptic Segmentation", CVPR 2019。画像で things+stuff を統合する PQ 指標を定義し、別々だった 2 分野を 1 つのタスクにまとめた。後に LiDAR へ波及。
- PolarNet: Zhang et al., "PolarNet: An Improved Grid Representation for Online LiDAR Point Clouds Semantic Segmentation", CVPR 2020。点群を俯瞰の極座標(リング状)グリッドに投影し 2D 的に処理する。LiDAR 点が放射状・遠方ほど疎という性質に合わせ、デカルト格子より各格子セル内の点数を均す。3D 畳み込みより軽量でオンライン処理に向く。
- Cylinder3D: Zhu et al., "Cylindrical and Asymmetrical 3D Convolution Networks for LiDAR Segmentation", CVPR 2021。点群を円柱状(極座標)のボクセル(3D の小箱)に区切り、非対称 3D 畳み込み(車のような細長い物体に合わせた異方性カーネル)で処理。放射状分布に合わせた分割で精度を伸ばした。トレードオフは 3D 畳み込みの計算・メモリの重さ。
- Panoptic nuScenes: Fong, Mohan, Hurtado, Zhou, Caesar, Beijbom, Valada, "Panoptic nuScenes: A Large-Scale Benchmark for LiDAR Panoptic Segmentation and Tracking", IEEE RA-L 2022。nuScenes 点群に大規模なパノプティックラベルと、時間方向の panoptic tracking ベンチマークを与えた。lidarseg(セマンティック)を土台に個体区別と追跡を加えた点が新規性。

系譜は「セマンティックの大規模ラベル(SemanticKITTI, 2019)」→「things+stuff を統合する PQ 指標(Kirillov, 2019)」→「LiDAR の放射状分布に合わせた効率的表現(PolarNet 2020, Cylinder3D 2021)」→「パノプティック + 追跡の大規模ベンチマーク(Panoptic nuScenes, 2022)」と進みました。

## 論文の実験結果(定量データ)

- nuScenes-lidarseg のセマンティックセグメンテーションでは、Cylinder3D が公開当時の主要手法を上回り、検証セットで mIoU 約 76 % 台を報告しました。mIoU 76 % は「16 クラス平均で予測と正解が 7 割超重なる」水準で、走行路面のような大きく安定したクラスは 95 % 超に達する一方、自転車・工事用品のような小さく稀なクラスは 50 % を下回ることが多く、平均がそれらに引き下げられます。同じ手法を 64 ビームの SemanticKITTI(mIoU 約 67 %)と比べると数値が違いますが、これはクラス定義・センサー解像度・地域が異なるためで、データセット間の絶対値比較には注意が要ります。
- PolarNet は SemanticKITTI と nuScenes-lidarseg の両方で評価され、3D 畳み込みベースに匹敵する精度を、推論速度を保ったまま達成しました。極座標グリッド化によりデカルト格子より各セルの点数偏りが小さくなり、遠方クラスの IoU 低下を緩和したと報告しています。
- Panoptic nuScenes 論文では、複数のベースライン(セマンティックネットワーク + 物体検出を後段で組み合わせた手法など)を PQ で比較し、当時の最良で PQ 約 60 % 台を報告。PQ = SQ × RQ の分解では SQ(形の一致)は高め(80 % 台)でも RQ(個体検出の過不足)が PQ を律速する、つまり「形は合っているが個体の検出漏れ・取り違えが残る」ことが課題だと示されました。追跡(tracking)では遠方・遮蔽時に ID が途切れやすいことが定量的に確認されています。

アブレーションの観点では、(1) things と stuff を分けずに一律セマンティックで扱うと、隣接する同種物体(並んだ車など)が融合して PQ の RQ 成分が大きく落ちる、(2) 極座標/円柱表現をデカルト格子に戻すと遠方クラスの IoU が顕著に下がる、(3) 複数フレームを集約せず単一フレームだけだと疎な遠方点でクラス誤りが増える、という傾向が各論文で報告されています。これらは「LiDAR の放射状・疎という物理的性質に表現を合わせること」と「things/stuff を区別すること」が精度の本質的要因であることを示しています。

## メリット・トレードオフ・限界

メリット:
- 点ごとの精密なラベルで、走行可能領域や物体輪郭を点の精度で学べる。箱では表せない地面・縁石・植生を区別できる。
- パノプティック版は意味 + 個体を統合し、同種物体の区別・連続追跡まで扱える。
- 母体データセット(カメラ・地図・自車運動)と座標が揃い、マルチモーダル融合(カメラ特徴と LiDAR 特徴の統合)に使える。
- mIoU / PQ という確立した指標と公開ベンチマークがあり、手法を公平に比較できる。

トレードオフ・限界:
- 点ごとの手作業ラベルは作成コストが極めて高く、ラベル頻度は 2 Hz と粗い。
- 中解像度 LiDAR(32 ビーム級)では遠方点群が疎で、小物体・遠方の区別が難しい。64 ビームの SemanticKITTI より遠方性能が出にくい。
- 母体データセットを前提とする拡張で、単独では使えない。
- 3D セグメンテーションは計算・メモリが重く、車載のリアルタイム運用には最適化が要る。

研究上の未解決課題として、PQ を律速する RQ(個体の過不足)の改善、ドメインギャップ(あるセンサー・地域で学んだモデルが別環境で大きく劣化する問題。32 ビーム ↔ 64 ビームのビーム数差は典型的な障壁)、そして稀少クラスの長い裾(class imbalance, クラスごとの点数が桁違いに偏る問題)があります。これらは点群セグメンテーションの一般化を阻む中心的な難問です。

## 発展トピック・研究の最前線

最近は、カメラ画像と LiDAR を融合してセグメンテーション精度を上げる手法、Transformer ベースで点群を直接扱う手法(点をトークンとして自己注意で処理)、複数フレームの点群を時間方向に集約して疎さを補う手法が進んでいます。また、点群セグメンテーションの正解を BEV(Bird's Eye View, 俯瞰)セグメンテーションの弱い教師として転用する流れもあり、カメラだけで俯瞰の領域分割を学ぶ研究で LiDAR ラベルが教師(または擬似ラベルの検証)として活用されています。

さらに、自己教師あり学習(ラベルなしの大量点群で事前学習し、少量ラベルで微調整)や、画像基盤モデルの知識を点群へ蒸留する手法によって、高コストな手作業ラベルへの依存を減らす方向が活発です。最近はオープンボキャブラリ化(あらかじめ決めた固定クラスに縛られず、テキストで指定した任意のカテゴリを点群上で見つける)や、占有予測(occupancy prediction, 空間を細かい格子に区切り各セルが占有されているか・何のクラスかを予測する)へ発展し、パノプティックの「漏れなく説明する」思想がボリュメトリックな表現へ広がっています。
