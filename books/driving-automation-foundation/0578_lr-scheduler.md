# 学習率スケジューラ

## ひとことで言うと

ニューラルネットワークを学習させるとき、「1回の更新でパラメータをどれくらい動かすか」を決める数字を「学習率(learning rate)」と呼びます。学習率スケジューラ(LR scheduler)とは、この学習率を学習の進み具合に合わせて自動で変化させる仕組みのことです。最初は大きく動かして速く学び、後半は小さく動かして精密に仕上げる、という調整を1つの学習の中で実現します。固定の学習率より高い最終精度が得られることが多く、現代のディープラーニングではほぼ標準の構成要素になっています。

## 直感的な理解

山を下って谷の最も低い地点(loss が最小の点)を探す状況を想像してください。最初、自分が山のどこにいるか分からないので、大股で速く下りたい。しかし谷底に近づいたら、大股のままでは谷底を行ったり来たり跨いでしまい、いつまでも一番低い点に止まれません。歩幅を細かくして、そっと最下点に収めたい。

この「歩幅」が学習率です。学習の段階によって最適な歩幅が違うのに、固定学習率は1つの歩幅で全行程を通します。早く下りるには大股が要り、谷底に収めるには小股が要る。両立できないのが固定学習率の根本的な弱点です。スケジューラは「序盤は大股、終盤は小股」と歩幅を時間とともに切り替えることで、この矛盾を解きます。

具体例で示します。学習率を 1e-3(0.001)で固定したとします。序盤は loss が順調に下がりますが、ある程度下がると歩幅が大きすぎて loss が 0.5 付近で上下にブレるだけになり、それ以上下がらなくなります。ここで学習率を 1e-4 に下げると、また loss が下がり始めて 0.3 まで到達できる。こうした現象は実務で非常によく観察され、「下げると一段下がる」挙動はスケジューラの効果を端的に示しています。学習曲線(横軸が step、縦軸が loss のグラフ)を見ると、学習率を下げた瞬間に loss が階段状にカクッと落ちる「クリフ(cliff)」が現れることがあり、これは「下げる前は歩幅が大きすぎて谷底付近で振動していた」証拠だと読めます。

## 基礎: 前提となる概念

学習率を理解するには、まず「勾配降下法(gradient descent)」を知る必要があります。ニューラルネットは大量の「重み(weight)」と呼ばれる数値の集まりです。学習とは、この重みを少しずつ正しい値に近づける作業です。近づける手がかりが「勾配(gradient)」で、これは「重みをどの方向にどれだけ動かせば誤差(loss)が減るか」を表す矢印のような量です。

重みの更新は、単純化すると次の式です。記号の意味を添えます。

```
新しい重み = 今の重み − η × g
  η (イータ) : 学習率。動かす歩幅の大きさ。
  g          : 勾配。動かす方向と大きさの手がかり。
```

η が大きすぎると歩幅が大きすぎて最適点を通り過ぎ、loss が下がるどころか発散(どんどん増大)します。小さすぎると一歩が小さすぎていつまでも目的地に着かず、学習が極端に遅くなります。「ちょうどいい η」は学習の段階によって変わる、というのがスケジューラを必要とする本質的な理由です。

もう一つの前提が「epoch」と「step(iteration)」です。学習データ全体を1巡することを1 epoch、その中で1バッチ(まとめて処理する数十枚程度のデータ)を1回処理することを1 step と呼びます。スケジューラは epoch 単位で動かすものと step 単位で動かすものがあり、近年は滑らかさのため step 単位が主流です。

さらに、なぜ学習率が「単独で」効くのではなく他の要素と絡むのかも押さえておきます。Adam/AdamW のような適応的オプティマイザは、各パラメータごとに過去の勾配の統計から「実効的な学習率」を内部で調整します。スケジューラはこの上にさらに「全体の倍率」を時間でかける役割です。つまり実効学習率 = スケジューラが出す η × オプティマイザの内部調整、という二段構えで動いています。この関係を理解すると、後述する warmup がなぜ Adam で特に効くか(Adam の二次モーメント推定が初期に不安定だから)が見えてきます。

## 仕組みを詳しく

スケジューラは「現在の step(または epoch)」を受け取り、「今この瞬間に使うべき学習率」を返す関数だと考えると分かりやすいです。代表的な種類を、基準学習率(初期値)を `base_lr = 1e-3` として数値例とともに説明します。

### 1. Step Decay(階段状に下げる)

決まった回数ごとに学習率を一定割合(典型的には 0.1 倍)で下げます。

```
30 epoch ごとに 0.1 倍する場合:
  epoch 0-29   : lr = 1e-3
  epoch 30-59  : lr = 1e-4
  epoch 60-89  : lr = 1e-5
```

階段状のグラフになります。実装が単純で挙動が読みやすい反面、「いつ下げるか」を人間が経験で決める必要があります。

### 2. Cosine Annealing(コサインカーブで滑らかに下げる)

学習率をコサイン関数に沿って base_lr から最小値 lr_min まで滑らかに下げます。総ステップ数を `T`、現在のステップを `t` として、

```
lr(t) = lr_min + 0.5 × (base_lr − lr_min) × (1 + cos(π × t / T))
```

`t=0` で cos(0)=1 なので lr=base_lr、`t=T` で cos(π)=−1 なので lr=lr_min。途中は滑らかな下り坂です。

```
lr
1e-3 ┤●
     │  ●●
     │     ●●
     │        ●●
     │           ●●●
lr_min┤              ●●●●●●●
     └───────────────────────→ step
     0                        T
```

階段のような急変がないため学習が安定しやすく、近年最もよく使われるスケジュールの一つです。終盤に学習率がほぼ 0 になることで、谷底に静かに収束する効果があります。なぜ「コサイン」かという問いには明快な理論的必然性はなく、「最初はゆっくり下げ、中盤で速く下げ、終盤でまたゆっくり 0 に収束する」という滑らかな S 字の下り坂が経験的に良かった、というのが実情です。線形減衰(linear decay)も同程度に良いとする報告が多く、大規模言語モデルでは線形 + 最後だけ急減衰が好まれることもあります。

### 3. Warmup(立ち上げ)

学習のごく最初だけ、学習率をゼロ(または極小)から base_lr まで徐々に上げる工夫です。

```
warmup を 500 step とする場合:
  step 0   : lr = 0
  step 250 : lr = 5e-4
  step 500 : lr = 1e-3 (ここから本来のスケジュールに合流)
```

なぜ最初を低くするのか。学習開始直後は重みがランダムな初期値で、勾配が極端に大きかったり方向がデタラメだったりします。そこへいきなり大きな学習率を掛けると、最初の数ステップで重みが壊れて発散することがあります。最初はそっと小さく動かして助走をつけるのが warmup です。Transformer 系のモデルや Adam 系オプティマイザでは事実上必須として知られます。Adam では二次モーメント(勾配の二乗の移動平均)が最初の数ステップでまだ正確に推定できておらず、その間の実効学習率が暴れやすいため、warmup でその期間を低速にしてやり過ごす、という説明が広く受け入れられています。

実務では「Warmup + Cosine」がよく使われます。最初に warmup で線形に立ち上げ、その後 cosine で滑らかに 0 近くまで下げる構成です。

### 4. ReduceLROnPlateau(頭打ちになったら下げる)

検証 loss を監視し、一定 epoch 改善しなくなったら学習率を下げる、データ駆動の方式です。総ステップ数を事前に決めなくてよい利点がある一方、「何 epoch 改善しなければ下げるか(patience)」という別のハイパーパラメータが要ります。

### 実装イメージ

多くのディープラーニングフレームワークにスケジューラのモジュールがあり、オプティマイザに取り付けて毎 step(または毎 epoch)`step()` を呼ぶだけで学習率が自動更新されます。

```python
optimizer = AdamW(model.parameters(), lr=1e-3)
scheduler = CosineAnnealingLR(optimizer, T_max=total_steps)
for batch in loader:
    loss = compute_loss(batch)
    loss.backward()
    optimizer.step()
    scheduler.step()   # ここで学習率を次の値へ更新
```

呼ぶ順番に注意が要ります。一般に `optimizer.step()` の後に `scheduler.step()` を呼びます(古い実装では順番が逆だと最初の学習率がスキップされる警告が出ることがあります)。また学習を途中から再開するとき、スケジューラは「今が何 step か」という内部カウンタを持つため、その状態も保存・復元しないと再開後に学習率が食い違います([チェックポイント(保存・再開)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1161_checkpointing-save-resume))。

## 手法の系譜と主要論文

学習率スケジューリングは「人手の経験則」から「理論と大規模実証に裏打ちされた標準手法」へと発展してきました。

- He et al., "Deep Residual Learning for Image Recognition", CVPR 2016(ResNet, arXiv:1512.03385)。深い画像認識ネットワークの学習で、学習率を一定 epoch ごとに 0.1 倍する step decay を採用しました。素朴な手法ながら、これだけで非常に深いネットワークが安定学習でき、当時の最高精度を達成しました。「下げるタイミングは人手で決める」という限界が、後の自動的なスケジュールへの動機になります。

- Smith, "Cyclical Learning Rates for Training Neural Networks", WACV 2017(arXiv:1506.01186)。学習率を上げ下げ周期的に動かす発想を導入し、加えて「学習率を少しずつ上げながら loss を観察して適切な範囲を見つける LR range test」を提案しました。学習率を勘ではなく実験で決める方法論の先駆けです。

- Loshchilov & Hutter, "SGDR: Stochastic Gradient Descent with Warm Restarts", ICLR 2017(arXiv:1608.03983)。学習率をコサインカーブで下げ、下げきったら base_lr に戻す(restart する)ことを繰り返す手法を提案しました。狙いは、学習率を大きく戻すことで悪い谷(局所解)から抜け出し、より良い谷を探させること。Cosine Annealing がここから広く普及しました。

- Vaswani et al., "Attention is All You Need", NeurIPS 2017(Transformer, arXiv:1706.03762)。warmup を前提とするスケジュール(最初に線形に上げ、その後ステップ数の逆平方根に比例して下げる)を提案しました。Transformer は学習初期が特に不安定で warmup なしだと発散しやすいためです。

- Smith & Topin, "Super-Convergence: Very Fast Training Using Large Learning Rates", 2017(arXiv:1708.07120)。One-Cycle Policy を提案。学習率を一度大きく(通常より一桁高く)まで上げてから下げる1周期のスケジュールで、特定条件下では通常の数分の一の epoch 数で同等精度に達する「super-convergence」を報告しました。大きな学習率が正則化の役割を果たすという見方を与えています。

- Goyal et al., "Accurate, Large Minibatch SGD", 2017(arXiv:1706.02677)。大きなバッチで学習するとき、学習率をバッチサイズに比例して線形にスケールさせ、最初に warmup を入れることで大規模分散学習でも精度を落とさないことを示しました。warmup の有効性を大規模に実証した代表例です。

- 近年は大規模言語モデルの文脈で「WSD(Warmup-Stable-Decay)」が注目されています(例: Hu et al., MiniCPM, arXiv:2404.06395)。warmup の後に学習率を一定に保ち、最後だけ急減衰させる構成で、安定期の任意の途中チェックポイントから追加学習しやすいという運用上の利点があります。

## 論文の実験結果(定量データ)

- SGDR(Loshchilov & Hutter, 2017)は CIFAR-10/CIFAR-100 画像分類で検証されました。CIFAR-10 のエラー率(100 枚中何枚間違えるかの割合で、低いほど良い)で、warm restart を使った Wide ResNet が当時のベースラインを下回るエラーを達成し、同じ計算予算でより速く良い解に到達することを示しました。複数の restart 時点のモデルをアンサンブル(平均)することで、単独モデルより数値上さらにエラーが下がる効果も報告されています(おおむね 1 ポイント前後の改善)。

- Attention is All You Need(Vaswani et al., 2017)は英独翻訳 WMT 2014 で BLEU(翻訳品質の指標で、参照訳との一致度。高いほど良い)28.4 を達成し、当時の最高を更新しました。論文中で warmup ステップ数を 4000 とするスケジュールが指定されており、後続研究の追試で「warmup を外すと学習が発散しやすくなる」ことが繰り返し報告されています。warmup は精度の数値そのものより、学習が完走できるかどうかという土台に効きます。

- Goyal et al.(2017)は ImageNet 分類で、ResNet-50 をバッチサイズ 8192(256 GPU)で学習しても、バッチ 256 の標準学習とほぼ同等の精度(top-1 約 76 パーセント前後)を保ったまま、学習を 1 時間程度に短縮できることを示しました。鍵は「学習率を線形スケール + 最初に warmup」です。warmup を外すと大バッチ学習の初期で精度が崩れることがアブレーションとして示されており、warmup がどの要素として効いているかが明確になっています。

- Super-Convergence(Smith & Topin)は CIFAR-10 で、通常のステップ減衰が数百 epoch を要するのに対し、One-Cycle で大幅に少ない反復(報告では桁違いに少ない場合もある)で同等以上の精度に達した例を示しました。ただし効果は条件依存が強く、再現には学習率の上限探索(LR range test)が前提になります。

数値の意味を補足します。エラー率や top-1 精度は「全データのうち正解した割合」を測る指標で、1 パーセントの差でも大規模なベンチマークでは順位を左右する重要な差です。BLEU は機械翻訳で 1 ポイントの改善が体感できる品質差に相当するとされます。スケジューラはこれらの最終指標を底上げすると同時に、「同じ計算予算でより速く到達する」「そもそも発散させない」という形でも効きます。

## メリット・トレードオフ・限界

メリット
- 学習初期は大きく動かして速く、後半は小さく動かして精密に、を1つの学習で両立できる。
- 多くのモデルで固定学習率より最終精度が高い。同じ計算予算での収束も速い。
- warmup を入れると学習初期の発散を防げる(特に Transformer 系・Adam 系で重要)。

トレードオフ・限界
- 「どのスケジュール」「総ステップ数 T」「warmup の長さ」など調整すべきハイパーパラメータが増える。
- 設定を誤ると(例: 早く下げすぎる)かえって精度が落ちる。総ステップ数を実際の学習長と合わせ損ねると cosine が途中で 0 に達して学習が止まる、あるいは 0 に届く前に学習が終わって終盤の精密化が効かない、といった事故も起きる。
- 学習率を操作する手法であって、データ品質や損失設計が根本的に悪い場合の解決にはならない。
- 未解決の課題として、最適なスケジュールはモデル・データ・バッチサイズに依存し、理論から一意に導くことは難しい。実務では今も探索やヒューリスティックに頼る部分が残ります。学習率を自動最適化する研究(hypergradient descent、適応的スケジュール、学習率を不要にする D-Adaptation/Prodigy 系)も進んでいますが、決定打には至っていません。

## 発展トピック・研究の最前線

- One-Cycle Policy / Super-Convergence(Smith & Topin)。学習率と運動量(momentum)を逆位相で同時に動かすことで極端に速い収束を狙う。
- WSD(Warmup-Stable-Decay)スケジュール。途中チェックポイントから追加学習しやすい性質を理由に、計算予算が事前に固定しにくい大規模学習で採用が進む。
- バッチサイズと学習率の関係は活発な研究領域で、「learning rate を上げる代わりにバッチサイズを増やす」(Smith et al., 2018, "Don't Decay the Learning Rate, Increase the Batch Size")という見方もあります。両者は学習ダイナミクスの観点で等価に近い操作だと整理されています。
- 学習率を人手で決めない方向の研究: D-Adaptation(Defazio & Mishchenko, 2023)や Prodigy は、最適点までの距離を推定して学習率を自動設定しようとします。
- スケジューラはオプティマイザの内部状態と密接に絡むため、再開時にスケジューラの状態も保存する必要があります([チェックポイント(保存・再開)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1161_checkpointing-save-resume))。
