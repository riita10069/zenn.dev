# シフト型タイムステップスケジュール

## ひとことで言うと

フローマッチング(ノイズから少しずつ本物のデータを作り出す生成方式)では、生成を何回かのステップに分けて進めます。このとき、ステップを時間軸 [0,1] 上に均等に置くのではなく、`t_shifted = (alpha * t) / (1 + (alpha-1) * t)` という式でわざと歪ませて配置するのがシフト型タイムステップスケジュールです。ステップ数が少ないとき(64 回ではなく 4〜8 回しか回さないとき)に生成品質を底上げするための、追加学習がいっさい要らない推論時のテクニックです。alpha=1.0 のときは均等配置と完全に一致するので、安全に試せます。

## 直感的な理解

料理のたとえで考えます。生地(ランダムノイズ)を整形して完成品(きれいなデータ)にするのに、オーブンの温度を時間とともに少しずつ変えていくとします。じっくり 64 段階で焼けばムラなく焼けますが、時間がかかります。4 段階で焼こうとすると、各段階のジャンプが大きく、特に「まだ生地がドロドロの初期段階」で大ざっぱに動かすと、最終的な形が大きく崩れます。

シフト型スケジュールは、この「焼き加減のチェックポイントをどこに置くか」を最適化する発想です。崩れやすい区間を避け、仕上げに近い区間にチェックポイントを密集させることで、少ない段階数でも破綻を防ぎます。重要なのは、焼く対象(モデル)も焼き方の道具(学習済みパラメータ)もいっさい変えず、ただ「いつ様子を見るか」のスケジュールだけを変える点です。だから再学習が不要で、すでに動いているモデルにそのまま被せられます。

なぜこれが現場で重要かというと、生成モデルの推論回数(関数評価回数、NFE = Number of Function Evaluations と呼びます。「モデルを何回呼ぶか」という意味です)が、そのままレイテンシと計算コストに直結するからです。自動運転やロボットの制御のように低遅延でリアルタイムに動かしたい用途では、NFE を 64 から 4〜8 へ落とせるかどうかが死活問題になります。そこで「少 NFE でも品質を保つ」工夫が強く求められます。

## 基礎: 前提となる概念

まず「フローマッチング (Flow Matching)」を、知らない人向けに説明します。これは画像・音声・軌道などを生成する手法の一種で、考え方の核は「2 つの確率分布をつなぐ連続的な経路を学ぶ」ことです。出発点は完全なランダムノイズ(標準正規分布 N(0,I) からサンプリングした点)で、これを時刻 t=0 とします。到達点はデータ分布(本物らしいサンプル)で、時刻 t=1 とします。

その間を、直線的に補間します。データ点を x_1、ノイズ点を x_0 とすると、途中の点は x_t = (1-t)·x_0 + t·x_1 という単純な式で定義できます。これは x_0 から x_1 へ向かう一直線上を、t に比例して進む点です。このとき、各点 x_t が「進むべき向きと速さ」、すなわち速度場 (velocity field) は v = x_1 - x_0 という時間に依らない一定ベクトルになります。

モデルは、ノイズだらけの点 x_t と時刻 t を入力に、この速度場 v を予測するように学習します。学習が終われば、推論時は次の常微分方程式 (ODE) を解きます。

```
dx/dt = v_theta(x, t)      x(0) = ノイズ, x(1) = 生成結果
```

`v_theta` は学習済みモデルが出す速度場、theta はそのパラメータです。この ODE を解く=ノイズから出発して速度場の向きに少しずつ進み、t=1 まで積分すれば生成サンプルが得られる、というわけです。

ここで「ステップ数」とは、この積分を何分割で近似するか(離散化の細かさ)です。一番単純なオイラー法 (Euler method) では、区間 [0,1] を N 分割し、各区切り点 t_i でモデルを呼んで速度を求め、`x ← x + v·(t_{i+1} - t_i)` と進めます。N が大きいほど刻みが細かく積分誤差(離散化誤差)が小さく、N が小さいほど誤差が大きくなります。タイムステップスケジュールとは、この区切り点 {t_0, t_1, ..., t_N} をどこに置くかの決め方です。

関連語の補足。「拡散モデル (Diffusion Model)」はフローマッチングと近縁の生成手法で、ノイズを徐々に足す/取り除く確率過程を学びます。フローマッチング(特に直線経路版)は拡散モデルを ODE の視点で整理・単純化したものと見なせ、両者は数学的につながっています。シフト型スケジュールの発想は拡散モデル側の「ノイズスケジュール設計」の研究から来ているので、両分野を行き来します。

## 仕組みを詳しく

通常の均等 (uniform) スケジュールでは、ステップ数 N に対し時刻を等間隔に置きます。N=4 なら:

```
t = [0.0, 0.25, 0.5, 0.75, 1.0]   (区間境界。間が 4 ステップ)
```

シフト型では、この均等な各点に変換をかけます:

```
t_shifted = (alpha * t) / (1 + (alpha - 1) * t)
```

`alpha` は歪み具合を決めるハイパーパラメータ(人間が事前に決める設定値)です。alpha>1 で後半寄り、0<alpha<1 で前半寄りになります。

alpha=3.0 のときの数値例:

| 元の t | 計算 | t_shifted |
|---|---|---|
| 0.00 | (3·0)/(1+2·0) | 0.000 |
| 0.25 | 0.75/1.5 | 0.500 |
| 0.50 | 1.5/2.0 | 0.750 |
| 0.75 | 2.25/2.5 | 0.900 |
| 1.00 | 3.0/3.0 | 1.000 |

元の t=0.25 が 0.5 に、t=0.5 が 0.75 に押し上げられています。ステップ点が「後半 (t が 1 に近い側)」に密集する形です。視覚化すると:

```
uniform :  0----0.25----0.5----0.75----1     (等間隔)
shifted :  0--------0.5----0.75--0.9---1     (後半に密集)
            ↑大ジャンプ    ↑細かく刻む
```

最初の 1 ステップで t=0 から 0.5 まで一気に進む大胆なジャンプを許し、その後を細かく刻みます。これは「ノイズ支配的な初期は粗く、答えに近い後半は丁寧に」という配分です。

恒等性の確認。alpha=1.0 を入れると:

```
t_shifted = (1·t) / (1 + 0·t) = t
```

変換が恒等写像になり、均等スケジュールと完全に一致します。この性質のおかげで「デフォルトを uniform、シフトを純粋な追加オプション」として実装でき、alpha=1.0 にすれば既存挙動とビット単位で同じになります。新機能を入れても回帰(後退)を起こさない設計が組めるわけです。

数式の素性も押さえておきます。この式は実は単調増加で全単射(0→0, 1→1, 途中も一意に対応)です。logit 空間(対数オッズ空間)でのシフトとして導出され、ノイズと信号の比 (signal-to-noise ratio) を log スケールで一定量だけずらす操作に対応します。Stable Diffusion 3 系の文献では、シフト量を sigma 空間(ノイズ標準偏差)で表す等価な書き方 `sigma_shifted = alpha·sigma / (1 + (alpha-1)·sigma)` も使われ、本質は同じです。

なぜ後半に寄せると効くのか、もう一段深く。少 NFE のオイラー積分では、各ステップの局所誤差が次ステップに持ち越され累積します。t≈0 付近は速度場の予測が不確かでスケールも大きく、ここで粗いステップを踏むと方向を大きく外しやすい——と素朴には思えますが、実際は逆方向の事情も効きます。直線経路フローマッチングでは t=0 付近の速度場は比較的なだらかで、むしろ t→1 付近で軌道が各データ点へ「分岐・収束」して速度場が急変します。その急変区間にステップを密に置く方が積分誤差を抑えられる、というのが後半寄せが効く一因です。最適な alpha はデータの次元・解像度・NFE に依存するため、決め打ちではなく探索すべき量になります。

## 手法の系譜と主要論文

時系列で系譜を追います。

第一段階: Rectified Flow (Liu, Gong & Liu, "Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow", ICLR 2023, arXiv:2209.03003)。ノイズからデータへの経路を「できるだけ直線にする」ことで、少ステップ生成を可能にする基礎理論を提示しました。reflow という再帰的な手続きで経路をまっすぐにし、極端には 1 ステップ生成も狙えると示しました。シフト型スケジュールは「経路を直線化する」のではなく「直線に近い経路上でのサンプリング点の配置を最適化する」ので、両者は補完的です。

第二段階: Stable Diffusion 3 (Esser et al., "Scaling Rectified Flow Transformers for High-Resolution Image Synthesis", ICML 2024, arXiv:2403.03206)。高解像度画像ほどノイズスケジュールを後ろ側にシフトすべきだという「resolution-dependent timestep shift(解像度依存のタイムステップシフト)」を提案しました。理由は、解像度が上がると 1 ピクセルあたりの情報量・ノイズの相対影響が変わり、最適なノイズ配分(どの t でどれだけノイズを残すか)がずれるためです。具体的には、解像度の平方根に比例してシフト量を増やす式を与え、これにより高解像度生成の FID(後述)が改善しました。トレードオフはシフト量という追加ハイパーパラメータの調整が必要になることです。

第三段階(本トピックの直接の出典): GoalFlow (Xu et al., "GoalFlow: Goal-Driven Flow Matching for Multimodal Trajectories Generation for Autonomous Driving", arXiv:2503.05689, 2025)。自動運転の軌道生成にフローマッチングを使い、複数の軌道候補を生成してゴール地点との整合性やドライバブルエリア(走行可能領域)で再ランキングする手法です。その推論部に `t_shifted = (alpha·t)/(1+(alpha-1)·t)` のシフトスケジュールを取り入れ、画像生成で培われた知見を軌道生成へ移植しました。アブレーションで「ステップ数が少ないほどシフトの効果が大きい」ことを定量的に示しています。

この系譜は「直線化(経路そのものを良くする)→ サンプリング点配置(良い経路の刻み方を最適化する)→ ドメイン移植(軌道生成へ応用)」という流れとして整理できます。

## 論文の実験結果(定量データ)

評価指標の意味から説明します。

FID (Fréchet Inception Distance) は画像生成の品質指標で、生成画像群と本物画像群の特徴分布の距離を測ります。小さいほど本物に近く、良い。Stable Diffusion 3 の論文では、タイムステップシフトを解像度に応じて入れることで、高解像度(例: 1024×1024)生成時の FID が、シフト無し(均等)に比べて目に見えて改善したと報告されています。とくに少サンプリングステップ設定で差が顕著でした。シフトは大きなモデル・高解像度ほど効きが大きい、という傾向も観察されています。

軌道生成側の指標。NAVSIM や Waymo といった自動運転ベンチマークでは、PDMS (Predictive Driver Model Score)、ADE (Average Displacement Error)、FDE (Final Displacement Error) などを使います。PDMS は安全性・進捗・快適性を総合したスコアで高いほど良い、ADE は予測軌道と正解の各時刻の距離の平均(単位 m、小さいほど良い)、FDE は最終時刻の距離(同)です。GoalFlow の文献では、シフト型スケジュールのアブレーションとして、サンプリングステップ数を変えながら品質を測り、ステップ数が小さいほど(例えば数ステップ)シフト有りが無し(均等)を上回る幅が大きく、ステップ数を十分に増やすと両者の差が縮まる、という傾向が示されています。これは「シフトは少ステップ時の積分誤差を吸収する補正であり、十分な刻みがあれば差は飽和する」という直感と一致します。

注意として、シフトの効果は alpha の選び方に強く依存し、不適切な alpha はむしろ品質を落とすことがアブレーションで観察されます。最適 alpha は NFE・モデル・データに応じて探索する必要があり、汎用の「最良値」は存在しません。報告される数値はその探索後のベスト設定であることが多い点に留意します。

## メリット・トレードオフ・限界

メリット:

- 追加学習がいっさい不要。学習済みモデルに対し、推論時の時刻配置を変えるだけで使える。
- alpha=1.0 で従来挙動と完全一致するため、既存パイプラインを壊さず安全に opt-in できる。
- ステップ数が少ないほど効果が大きく、低レイテンシ・少 NFE 運用(車載組み込み、リアルタイム制御など)と相性が良い。
- 実装が数行で済み、どんなフローマッチング/拡散サンプラにも被せられる汎用性がある。

トレードオフ・限界:

- alpha という新ハイパーパラメータが増え、適切に調整しないと逆に品質が落ちうる。最適値はデータ・解像度・NFE 依存で一般解が無い。
- 効果は少ステップ時に大きいが、十分なステップ数(例: 64)があると均等との差は飽和し、ほぼ無くなる。
- あくまでサンプリング点の配置最適化であり、モデルや学習データの質を改善するものではない。生成器が悪ければシフトしても良くはならない。
- ある分野(画像)で効いた alpha が別分野(軌道)でそのまま効く保証はなく、ドメインごとの再探索が要る。

未解決の課題: 最適 alpha を「探索なしで」予測する理論はまだ確立していません。解像度依存式のような経験則はありますが、軌道生成のような構造の異なるデータでの最適シフト則は研究途上です。また、より進んだ高次 ODE ソルバ(Heun 法、DPM-Solver 等)や蒸留(distillation)による少ステップ化と、シフトをどう組み合わせるのが最善かも開かれた問いです。

## 発展トピック・研究の最前線

シフトは「サンプリングの離散化を改善する」系統の一手法であり、隣接領域として次があります。高次 ODE ソルバ(オイラーより誤差の小さい積分公式)や、DPM-Solver 系の専用ソルバは、同じ NFE でより正確に積分します。蒸留 (distillation) は、多ステップ教師モデルの出力を少ステップ生徒モデルに教え込み、1〜4 ステップ生成を実現するアプローチで、Consistency Models や進行的蒸留が代表です。これらはモデルを再学習する点でシフトと異なりますが、品質と速度のトレードオフを攻める目的は同じで、シフトと併用もできます。

近年は、サンプリングスケジュール自体を学習で最適化する研究(各 t を固定式でなくデータ駆動で配置する)や、データ点ごとに適応的にステップを置く適応ソルバも登場しています。シフト型スケジュールはその中で「最も軽量・解釈可能・追加学習不要」な選択肢として、まず試す価値の高いベースラインの位置を占めています。
