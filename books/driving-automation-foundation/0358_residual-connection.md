# 残差接続(skip connection)

## ひとことで言うと

ニューラルネットワークのある層(ブロック)の出力 F(x) に、その層への入力 x をそのまま足し込み、出力を F(x) + x にする仕組みです。この「そのまま足すための近道」を skip connection(スキップ接続、飛び越し接続)と呼びます。残差(ざんさ)とは「差分」のことで、層は最終的な出力そのものではなく「入力からの差分 F(x)」を学習する、という意味でこの名前が付いています。この単純なアイデアが、それまで不可能だった 100 層を超える深いネットワークの学習を可能にし、画像認識・自然言語処理・音声・自動運転のほぼすべての近代モデルの基礎部品になりました。

## 直感的な理解

階段を 100 段登ることを考えます。普通のネットワークは、1 段目で得た情報を 2 段目に渡し、2 段目の出力を 3 段目に渡し……と、必ず全段を経由して情報を運びます。途中の段で少しずつ情報が劣化すると、100 段目に届くころには元の情報がほとんど残りません。学習の手がかりである勾配も同じで、出力側から入力側へ各段を逆に伝わるうちに、掛け算が繰り返されて消えてしまいます。

残差接続は、各段の横に「エスカレーター」を併設するようなものです。情報も勾配も、各段の処理(F)を通る経路だけでなく、エスカレーター(+x の近道)を通ってそのまま下まで届きます。だから 100 段あっても元の情報と勾配が失われません。さらに「この段では何もしないのが最適」という場合、普通の段は「入力をそのまま出す」ことを苦労して学ばねばなりませんが、残差接続なら処理 F をゼロにするだけで自動的に素通りになります。深いネットワークに「使わない層を安全に素通りさせる」自由を与えた、というのが本質です。言い換えると、層を足すことが「最悪でも何もしない(性能を下げない)」を保証する構造になっているため、設計者は安心して層を深くできるようになりました。

## 基礎: 前提となる概念

まず層(レイヤー)とは、入力の数値ベクトルを別のベクトルに変換する処理の単位です。画像認識では「畳み込み(画像の局所パターンを抽出する演算)+ 正規化 + 活性化関数(非線形変換)」をひとかたまりにしたブロックがよく使われます。これらを何十個も積み重ねたものが深いネットワークです。深くするほど、単純な辺やテクスチャから、物体の部品、物体全体、という具合に抽象度の高い特徴を段階的に組み立てられる、というのが深層化の動機です。

次に誤差逆伝播(backpropagation)とは、ネットワークの学習方法です。出力と正解の差(損失)を計算し、その損失を各層のパラメータで微分して勾配を求め、出力側から入力側へ向かって順に勾配を伝えていきます。このとき各層の勾配は連鎖律(微分の掛け算ルール)で次々に掛け合わされます。

ここで勾配消失(gradient vanishing)という問題が起きます。各層で勾配が 0.5 倍に縮むとすると、10 層を遡るうちに 0.5 の 10 乗 = 約 0.001 倍まで小さくなります。100 層なら天文学的に小さくなり、入力に近い層の勾配がほぼゼロになって、その層のパラメータが更新されなくなります。層が深いほど掛け算の回数が増えるので、深いネットほど起きやすい現象です。逆に各層で勾配が 2 倍に増えると今度は爆発(gradient explosion)し、値が発散して学習が壊れます。深層化はこの「縮みすぎ」と「増えすぎ」の両側の崖の間を綱渡りする難しさを抱えていました。

さらに ResNet 論文が指摘した劣化問題(degradation)という別の現象もあります。これは過学習(訓練データに合わせすぎて未知データで悪化する)とは違い、層を深くすると訓練データに対する誤差すら悪化するという、より根本的な問題です。56 層のプレーンなネットが 20 層のものより訓練誤差が高い、という直感に反する結果が観測されていました。過学習なら訓練誤差は下がるはずなので、これは「最適化が難しくて深いネットが学習しきれていない」ことを示す現象でした。

## 仕組みを詳しく

通常の層は y = F(x) という形で、入力 x を変換して出力 y を作ります。残差接続では出力を次のようにします。

```
y = F(x) + x
```

ここで F は「畳み込み + 正規化 + 活性化」などをまとめたひとかたまりの処理(残差関数)です。この足し算が学習にどう効くかは、勾配を見ると分かります。出力 y を入力 x で微分すると、

```
dy/dx = d(F(x) + x)/dx = F'(x) + 1
```

となります。重要なのは末尾の「+ 1」です。たとえ F'(x) が小さく 0 に近くても、勾配は最低でも 1 が残ります。backward 時に勾配がこのブロックを通っても 0 倍にはならず、1 倍ぶんは必ず下流(入力側)へ素通りします。これが skip connection による近道経路で、勾配消失を防ぎます。L 個のブロックを積んだとき、最深部から最浅部への勾配は各ブロックの (F'+1) の積になりますが、その積を展開すると「1 だけを選んだ経路(完全な素通り)」が必ず一項含まれるため、勾配が完全にゼロになることはありません。

before / after を数値で見てみます。skip がない 10 層で各層 0.5 倍なら、入力側に届く勾配は約 0.001 倍です。skip がある場合、各ブロックの勾配は (F'(x) + 1) がおよそ 1.0 を保てるので、10 層を経ても 1.0 の 10 乗 がおよそ 1.0 のオーダーに残り、入力側の層もきちんと更新されます。

もう一つの利点が「恒等写像(こうとうしゃぞう、入力をそのまま出すこと)を学習しやすい」点です。もし「この層は何もしないのが最適」だとしたら、通常の層は y = x を作るために重みを苦労して調整する必要があります。残差接続なら F(x) = 0 を学ぶだけで y = x が達成できます。重みをゼロに近づけるのは、特定の写像を学ぶよりずっと簡単です。最適化の観点では、解きたい関数 H(x) を直接学ぶより、その「恒等写像からのずれ」F(x) = H(x) - x を学ぶほうが、解が恒等写像に近いとき残差が小さくなり最適化が容易になる、という再定式化(reformulation)が残差学習の核心です。

### 形状を合わせる必要がある

足し算なので、F(x) と x のテンソル形状は一致している必要があります。たとえば x が [B, 256, 56, 56](バッチ B、チャネル 256、縦横 56)なら F(x) も同じ [B, 256, 56, 56] にします。チャネル数や解像度が変わるブロックでは、x 側にも 1x1 畳み込みやストライド付き畳み込みを挟んで形状を合わせてから足します(これを射影付き skip、projection shortcut と呼びます)。ResNet 論文はこの射影 shortcut と、ゼロ詰めで次元を合わせる恒等 shortcut を比較し、射影のほうがわずかに精度が高いが恒等のほうがパラメータゼロで軽い、というトレードオフを報告しています。

### ボトルネックブロック

ResNet-50 以上の深いモデルでは、計算量を抑えるためボトルネック構造を使います。これは 1x1 畳み込みでチャネルを減らし(例 256→64)、3x3 畳み込みで処理し、再び 1x1 でチャネルを戻す(64→256)三層構成で、その全体に skip を掛けます。チャネルを絞ってから処理することで、同じ深さでも計算量を大きく削減できます。

```
入力 x [B,256,H,W]
  ├─ 1x1 conv → 64ch → 3x3 conv → 64ch → 1x1 conv → 256ch  =F(x)
  └────────────────────────────────────────────── x (skip)
出力 y = F(x) + x
```

このボトルネックのおかげで、ResNet-50 は VGG-19 より深い(50 層 対 19 層)にもかかわらず、計算量(FLOPs)は VGG-19 の数分の一に収まります。深さと計算量を切り離せたことが、超深層化を実用的にした隠れた立役者です。

## 手法の系譜と主要論文

起点は He, Zhang, Ren, Sun の "Deep Residual Learning for Image Recognition"(CVPR 2016、通称 ResNet、arXiv:1512.03385)です。提案内容は上で述べた y = F(x) + x の残差ブロックを積み重ねるアーキテクチャです。動機は劣化問題の観測で、著者らは「深いネットでも少なくとも浅いネット+恒等写像と同等の性能は出せるはず」と考え、その恒等写像を構造として与えました。新規性は、ゲートなどの複雑な仕掛けを使わず「ただ足すだけ」で深層化を実現した点です。

これに先立つ Srivastava らの Highway Networks(2015、arXiv:1505.00387)は、残差接続にゲート(0〜1 の重み)を付けて y = T(x)·F(x) + (1 - T(x))·x のように足し合わせを学習で調整しました。LSTM のゲート機構を順伝播ネットに持ち込んだ発想です。ResNet はこのゲートを省き、単純に 1 で足すだけにしたことで学習が安定し実装も軽くなりました。「複雑なゲートは要らなかった」という発見が ResNet の貢献の一つで、同時期の独立な工夫が、より単純な形に収束した好例です。

He らはさらに Pre-activation ResNet(ECCV 2016、"Identity Mappings in Deep Residual Networks"、arXiv:1603.05027)で、正規化と活性化をブロックの前に置く順序変更を提案しました。元の ResNet は足し算の後に活性化(ReLU)を置いていましたが、これだと skip 経路にも活性化が乗って完全な恒等写像にならず、勾配がわずかに減衰します。Pre-activation はこの ReLU を残差枝の内側に移し、skip 経路を「足すだけ」の純粋な恒等経路にしたことで、勾配が一切減衰せず素通りできるようにして、1000 層を超えるネットの学習を可能にしました。残差接続がなぜ効くのかの理論的な理解が深まった重要な後続研究です。

Veit ら(NeurIPS 2016、arXiv:1605.06431)は「ResNet は多数の浅いパスのアンサンブル(寄せ集め)のように振る舞う」という解釈を与えました。L 個の残差ブロックがあると、信号が通れる経路は各ブロックで「F を通る/skip で飛ばす」の二択なので 2 の L 乗通りあり、ResNet はそれらの暗黙の集合として機能している、という見方です。実際に学習済み ResNet からブロックを一つ削除しても精度がほとんど落ちないという実験が、この「冗長な多数パス」の解釈を支持しました。これは残差接続の効きの理由を別の角度から照らした研究です。

Huang らの DenseNet(CVPR 2017、arXiv:1608.06993)は、足し算ではなく過去の全層の出力を連結(concat)して渡す方式で、特徴の再利用を促しました。連結なのでチャネル数が層ごとに増えメモリは食いますが、各層が前のすべての層に直接つながるため勾配の通り道がより豊富になり、少ないパラメータで高い精度を出しました。CIFAR で ResNet と同等精度を約 3 分の 1 のパラメータで達成したことが報告されています。

そして Transformer(Vaswani ら、NeurIPS 2017、arXiv:1706.03762)の各サブ層も「x + Sublayer(x)」という残差接続を使い、層正規化(LayerNorm)と組み合わせて深い Transformer の学習を支えています。原理は ResNet とまったく同じで、残差接続が畳み込みネットを超えて現代の標準部品になったことを示しています。後に LayerNorm を残差枝の手前に置く Pre-LN 方式が、深い Transformer の学習安定化に効くことが分かり、これは画像での Pre-activation ResNet と同じ発見の言語版でした。

## 論文の実験結果(定量データ)

ResNet 論文の中心的な実験は ImageNet(約 128 万枚、1000 クラスの画像分類データセット)での分類精度です。評価指標は top-1 / top-5 誤り率(予測の最有力候補/上位 5 候補に正解が入らない割合、低いほど良い)です。

劣化問題を示す決定的な比較として、論文は CIFAR-10(6 万枚の小画像)と ImageNet の両方で「プレーンネット(skip なし)を深くすると訓練誤差が悪化する」ことを定量的に示しました。プレーンな 34 層は 18 層より訓練誤差・検証誤差ともに高く、深くしたのに性能が落ちるという現象が再現されました。同じ層数の ResNet(skip あり)は逆に 34 層が 18 層より良く、深層化の恩恵を正しく受けられることを示しました。具体的には ImageNet の検証 top-1 誤り率で、プレーン 34 層が 18 層より約 1〜数ポイント悪化するのに対し、ResNet 34 層は ResNet 18 層を約 2.8 ポイント上回りました。この符号の逆転こそが skip connection の効果を切り分けて示すアブレーションです。

ImageNet の最終結果では、ResNet-152(152 層)が当時としては極めて深いネットの学習に成功し、top-5 誤り率 約 4.49%(単一モデル)、アンサンブルで約 3.57% を達成して ILSVRC 2015 で優勝しました。当時の代表的な VGG(19 層)の top-5 誤り率が約 8% 前後だったことと比べると、誤り率が半分以下に下がった計算で、これは画像認識の歴史的な飛躍でした。この 3.57% は当時報告されていた人間の top-5 誤り率(約 5% とされる)を下回り、「機械が画像分類で人間を超えた」という象徴的な数字として広く引用されました。

CIFAR-10 では、skip により 110 層、さらに 1202 層という極端に深いネットまで学習できることを実証しました。1202 層は過学習で 110 層より検証精度がわずかに落ちますが、訓練自体は破綻せず収束しており、「深さの限界が勾配ではなくデータ量側に移った」ことを示す象徴的な結果です。アブレーション(要素を抜く実験)の観点では、skip を外したプレーン版が同じ層数で必ず劣化することが、残差接続そのものの寄与を切り分けて証明しています。Pre-activation ResNet(2016)はさらに CIFAR-10 で 1001 層を安定学習させ、誤り率を元の ResNet より下げて、skip 経路を純粋な恒等写像に近づける効果を定量的に示しました。

## メリット・トレードオフ・限界

メリットは、深いネットを安定して学習できること、恒等写像を安価に表現できること、勾配が下流まで届くことです。この三つにより、層数を増やすことが素直に表現力の向上につながるようになりました。学習の収束も速くなり、同じ計算予算でより高精度に届きます。

トレードオフは、skip 経路ぶんのメモリ(足し算のために入力 x を保持する必要がある)と加算の計算がわずかに増えること、そして F(x) と x の形状を合わせる管理が必要になることです。とくにメモリは見落とされがちで、backward のために各残差ブロックの入力を保持する必要があり、これが深いネットのメモリ消費の一因になります(勾配チェックポインティングで一部を再計算して節約する手法が併用されます)。とはいえ計算コストの増加はごくわずかで、得られる学習安定性の利益が圧倒的に大きいため、現代の深層モデルではほぼ標準装備です。

限界・未解決の論点もあります。残差接続は勾配消失を緩和しますが完全に消すわけではなく、極端に深いネット(1000 層超)では正規化や初期化の工夫を併用しないと依然不安定です。また「なぜ残差接続がこれほど効くのか」の理論的説明には複数の説(前述の Veit らのアンサンブル的振る舞い、損失地形の平滑化、暗黙の浅いパス集合など)があり、決定版の理解はまだ研究途上です。損失地形の研究では、skip connection が損失関数の凹凸を滑らかにして局所最適に陥りにくくする、という可視化も報告されています。さらに、足し算ベースの残差はチャネル次元を増やせない(同形状でないと足せない)という制約があり、これを連結で回避する DenseNet との優劣はタスクやメモリ予算に依存します。

## 発展トピック・研究の最前線

残差接続の発展は多方向に広がっています。Wide ResNet(Zagoruyko & Komodakis、2016)は「深さより幅(チャネル数)」を増やす方向を探り、16 層程度でも幅を広げれば 1000 層 ResNet を上回れることを示しました。ResNeXt(Xie ら、2017)は残差ブロック内を複数の並列経路に分ける(グループ畳み込み)ことで効率を上げ、同じ計算量で精度を改善しました。EfficientNet(Tan & Le、2019)は深さ・幅・解像度を協調的にスケールする複合係数を提案し、いずれも残差ブロックを土台にしています。

Transformer 系では、深い層を安定させるための残差スケーリング(DeepNorm、ReZero、LayerScale など、残差の足し込みに学習可能な係数やゼロ初期化を入れる手法)が活発に研究されており、これは Highway のゲートをより洗練させた形と見ることもできます。とくにゼロ初期化した係数で残差枝を足す手法(ReZero、LayerScale)は、学習開始時に新しい枝の寄与をゼロにして既存の挙動を壊さず、徐々に有用な枝だけを育てる安全な学習を実現します。これにより数百層〜千層規模の Transformer の安定学習が可能になり、大規模言語モデルや大規模視覚モデルの深層化を支えています。残差接続という 2016 年の素朴なアイデアが、いまも深層モデルの安定化研究の中心にあり続け、形を変えながらほぼすべての近代アーキテクチャに受け継がれています。
