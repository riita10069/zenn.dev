# 反応型E2E統合モデル (マルチカメラ融合・計画・未来状態予測)

## ひとことで言うと
これは、車の周りに付いた複数のカメラ映像、進路を示すナビゲーション地図、過去の動きの履歴を入力として受け取り、「これからどう走ればよいか」という未来の走行軌跡(車がたどる点の列)を一気に出力するニューラルネットワークの設計類型です。途中で「物体検出」「車線認識」のように人間が決めた中間タスクへ明示的に分けず、入力から軌跡まで一本のネットで結ぶので「End-to-End(エンドツーエンド、E2E)」と呼びます。さらに、バックボーンや融合方式や計画器といった内部部品を文字列キーで差し替えられる「交換可能な部品構成」で組むのが、実用志向のE2E統合モデルでよく採られる作り方です。

## 直感的な理解
自動運転を作るとき、昔は「目(知覚)」「予想(予測)」「判断(計画)」を別々の人(モジュール)が担当するリレー方式でした。引き継ぎのたびに情報が削られ(たとえば知覚は物体を四角い箱に丸めて渡すので、見た目の細かい手がかりは消える)、しかも各段が別々の目標で最適化されるため全体最適になりにくい弱点があります。E2Eは、目から判断までを1人の人(1つのネット)が通しで担い、人間の運転を真似て学ぶ方式です。情報を途中で捨てず、最終目標(うまく運転すること)から逆算して内部表現を学べるのが利点です。

ここで扱う「反応型(reactive)」E2Eは、特に「今この瞬間のセンサに即座に反応して軌跡を出す」速い反応エンジンを指します。人間の運転でいえば、地図を見て大まかな道筋を考える「遅い思考」と、目の前に飛び出した自転車に反射的にブレーキを踏む「速い反応」のうち、後者に相当します。認知科学でいう「システム1(速い・直感的)とシステム2(遅い・熟慮的)」の二分法に対応づけると分かりやすく、多くの実用設計は、この速い反応器を高頻度(たとえば10Hz=1秒に10回)で回し、別に低頻度でじっくり世界を理解する遅い側(world model 的なモジュール)を将来つなぐ二層構成を想定します。反応器が常時安全運転を担い、遅い側が長期の予見や難所の判断を補助する、という分業です。

## 基礎: 前提となる概念
- 模倣学習(imitation learning): 人間の運転ログを正解とし、同じ軌跡・操作を出すようネットを学習させる方法。E2E自動運転の基本です。弱点は、学習で見なかった状態に陥ると挙動が未定義になること(共変量シフト、covariate shift)。
- バックボーン(backbone): 画像から特徴を抜き出す、既製の画像認識ネット。ResNet(畳み込みの定番)、Swin Transformer(画像を窓に区切って attention)、ConvNeXt(畳み込みを Transformer 風に近代化した設計)などが使われます。ImageNet などで事前学習済みの重みを再利用することが多く、ゼロから学ぶより少ないデータで高精度になります。
- 特徴融合(feature fusion): 複数台のカメラから得た特徴を1つの表現にまとめる処理。後述の BEV に投影してまとめる方式や、単純に連結(concat)する方式、attention で混ぜる方式があります。
- BEV(Bird's-Eye-View、鳥瞰図): 世界を真上から見た格子表現。複数カメラの斜め視点を共通の俯瞰座標に直すことで、距離や位置を素直に扱えます。画像から BEV への投影は、各画素の奥行きを推定して3D点を作り俯瞰格子に落とす方式(LSS = Lift-Splat-Shoot)や、BEV側のクエリで画像を引きにいく方式(BEVFormer)があります。
- 時系列メモリ(temporal memory): 過去の映像や自車運動の履歴を圧縮して文脈ベクトルにする部品。今だけでなく直前までの流れを使って判断します。これが無いと、静止画1枚では他車の速度が原理的に分からず、止まっている車か動いている車かを区別できません。
- 計画器(trajectory planner): 融合特徴と文脈から、未来の (x, y) 列(waypoint)を生成する部品。回帰(平均的な1本を直接出す)、生成モデル(flow matching や拡散で複数候補を出す)、曲線パラメータ(Bezier 曲線の制御点を出す)など複数の作り方があります。
- 自己教師あり学習(self-supervised learning): 正解ラベルなしで、データ自身から学習信号を作る方法。たとえば「映像の一部を隠して、隠した部分を当てさせる」。後述の未来状態予測はこの一種です。

## 仕組みを詳しく
代表的な反応型E2E統合モデルは、薄い最上位ラッパー(将来 world model を足す余地を残す層)の中に本体ネットを1つ抱える二層構造で組まれることが多いです。本体の forward(順伝播、入力を出力に変換する1回の計算)は次の枝に分かれます。tensor shape を添えて追います。記号: B=バッチサイズ、V=カメラ台数(例:7)、C=チャンネル数(RGBなら3)、H/W=画像の高さ/幅。

1. カメラ枝(Camera branch)
   - 入力 `camera_tiles` は `(B, V, 3, H, W)`。複数カメラ画像を束ねた形です。
   - これを `(B*V, 3, H, W)` に並べ替えてバックボーンに通し、各カメラの特徴マップを得ます。V台を一括でバックボーンに流すのは、台数ぶんのループを避けてGPUを効率よく使うためです。
   - 特徴融合で、V 台ぶんの特徴を1つの BEV(例:450×300 の格子)にまとめ、画像由来の BEV 特徴(image_bev)を作ります。各カメラの視野が重なる部分は、複数カメラの情報が同じ BEV セルに集約されます。

2. 地図枝(Map branch)
   - 入力はナビ地図(進むべき道を描いた画像)で `(B, 3, H_map, W_map)`。これは「次の交差点で右」のような走行意図を空間的に与える役割です。
   - 地図エンコーダで同じ BEV 格子サイズの特徴に変換し、地図由来の BEV 特徴(map_bev)を得ます。

3. 融合(Map-BEV fusion)
   - 画像由来 BEV と地図由来 BEV を1枚に重ね、融合特徴(fused_features)を作ります。残差(residual、地図特徴を足し込む)方式などがあり、地図を「補正項」として画像主体の特徴に注入します。

4. 時系列メモリ(Temporal Memory)
   - 過去映像履歴と自車運動履歴を圧縮し、視覚文脈・自車運動文脈の2つのベクトルにします。最小構成では「履歴をそのまま使う(no-memory、現フレームだけ)」、必要なら RNN(再帰ネット)や attention 系に差し替えます。自車運動履歴(過去の速度・操舵)は、動的に滑らかな運転を続けるための慣性情報になります。

5. 軌跡計画(Trajectory Planner)
   - 融合特徴と2つの文脈を計画器に渡し、未来軌跡を出力します。例えば `num_timesteps=64`、`num_signals=2`(x,y)なら、出力は `(B, 64*2)` = 長さ128のベクトル(64ステップぶんの (x,y))になります。64ステップを0.5秒刻みで読めば32秒先まで、0.1秒刻みなら6.4秒先までの予定経路を表す、という解釈です。

6. 未来状態予測(Future-state prediction)
   - 計画器の内部状態を条件に、未来の視覚特徴を予測するモジュールを併設する設計があります。これは画素ではなく「特徴(中間表現)」を予測して自己教師あり損失に使う JEPA 系の考え方の取り込みを狙うものです。画素まで当てると照明や葉のゆれのような予測不能な細部に学習が引きずられますが、特徴レベルなら「次に何が起きるか」の本質だけを学べます。安定化と損失接続が研究上の論点になります。詳細は [未来特徴予測 (FutureState型モジュール)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0012_future-state-prediction) / [V-JEPA（潜在空間世界モデル）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1092_v-jepa-world-model)。

学習時と推論時で出力の意味を切り替える設計が実用上の要点です。学習モードでは計画器の内部で損失(回帰なら模倣 MSE = 予測軌跡と正解軌跡の二乗誤差、flow matching なら velocity MSE = 速度場の二乗誤差)まで計算してスカラーの損失を返します。こうすると学習ループ側は「どの計画器を使っているか」を気にせず同じコードで回せます。推論時は軌跡そのものを返します。出力の「契約(どんな型を返すか)」を学習/推論で分けることで、計画器の交換が学習ループに波及しません。

最大の設計上の特徴は「交換可能な部品構成(component registry、部品を名前で登録しておく仕組み)」です。バックボーン・融合・地図エンコーダ・時系列メモリ・計画器をすべて文字列キーで選べるようにし、新方式をレジストリに登録すれば本体コードを書き換えずに切り替えられます。これにより、後述の主要論文の知見(UniAD 的な計画頂点設計、VAD 的な軽量化、JEPA 的な自己教師)を部品単位で実験・比較できます。研究の現場では「どの部品が効いているか」を切り分けたいので、この差し替え可能性はアブレーション実験のコストを大きく下げます。

## 手法の系譜と主要論文
この種の統合モデルは単一論文の再現ではなく、複数のE2E研究の良いところを部品として取り込める器です。思想的な親を時系列で追います。

- LSS(Philion & Fidler, "Lift, Splat, Shoot", ECCV 2020, arXiv:2008.05711): 各カメラ画素の奥行き分布を推定して3D点群(lift)を作り、BEV格子に落とし込む(splat)。多視点画像→BEV 融合の基礎技術で、カメラ枝の image_bev 生成の代表的なやり方です。
- ST-P3(Hu et al., "ST-P3: End-to-end Vision-based Autonomous Driving via Spatial-Temporal Feature Learning", ECCV 2022, arXiv:2207.07601): 知覚・予測・計画を時空間特徴で一貫して扱う初期のビジョンE2E。BEV を中間表現に使い、占有予測を経て計画する点が image_bev/map_bev 構成と近いです。
- UniAD(Yihan Hu et al., "Planning-oriented Autonomous Driving", CVPR 2023, Best Paper, arXiv:2212.10156): 知覚・予測・計画を1つの Transformer に統合し、計画タスクを頂点に据えて各モジュールをクエリで連結。「最終目的(計画)から逆算して中間表現を選ぶ」発想は、中間タスクを固定せず軌跡を直接出す方針と通じます。トレードオフは巨大で学習が重く、各サブタスクの教師ラベル(検出箱や地図)が要る点です。
- VAD(Bo Jiang et al., "VAD: Vectorized Scene Representation for Efficient Autonomous Driving", ICCV 2023, arXiv:2303.12077): シーンをベクトル化(物体や車線を点列で表す)して密 BEV 計算を減らし、UniAD より高速・高性能を達成。実用志向の軽量E2Eの代表で、反応型統合モデルが目指す「軽さと実用性」の手本です。詳しくは [VAD (Vectorized Scene Representation)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1094_vad-vectorized-scene)。
- I-JEPA(Mahmoud Assran et al., "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", CVPR 2023, arXiv:2301.08243)/ V-JEPA(Bardes et al., 2024): 隠した部分の「特徴」を予測して自己教師あり学習する JEPA の代表(I は画像、V は動画)。未来状態予測モジュールはこの思想の取り込みを狙います([V-JEPA（潜在空間世界モデル）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1092_v-jepa-world-model) 参照)。
- Flow Matching(Yaron Lipman et al., "Flow Matching for Generative Modeling", ICLR 2023, arXiv:2210.02747): ノイズから目標分布への連続的な流れ(速度場)を学ぶ生成モデルの枠組み。計画器の一方式として、複数の起こりうる軌跡(マルチモーダルな未来)を表現するのに使えます。拡散モデルより学習・サンプリングが単純なのが利点です。

これらは共通して「中間タスクで情報を捨てない」「最終的な運転性能で評価する」方向を向いており、反応型統合モデルはその設計言語を差し替え可能な部品として実装した点に独自性があります。

## 論文の実験結果(定量データ)
構成部品の出自となる主要論文の数値を、初心者向けに意味を添えて挙げます。

- UniAD は nuScenes の open-loop 計画評価で、計画の L2 誤差(予測軌跡と人間の実走行の平均距離、メートル、低いほど良い)と衝突率(Collision Rate、低いほど良い)を、当時のビジョンベースライン(ST-P3 など)に対し改善したと報告しました。平均 L2 はおよそ1.0m前後から大きく下げ、衝突率も低減しています。論文の主張の核は「計画を頂点にして全タスクを一体最適化すると、各タスクを別々に最適化するより最終計画が良くなる」ことです。UniAD のアブレーションでは、計画の上流に予測・占有などのモジュールを段階的に足すほど計画性能が上がる(各モジュールが計画に寄与する)ことが示されました。逆に言えば、知覚や予測の教師ラベルが全部必要で重い、というのがコストです。
- VAD は同じ nuScenes 計画評価で、UniAD と同等以上の L2・衝突率を保ちつつ、推論速度を数倍(報告ではおおむね UniAD の数倍速い、FPSで顕著)に上げ、ベクトル化が「軽量化と性能の両立」に効くことを定量的に示しました。密 BEV の重い格子演算を、物体・車線をベクトルで持つことで置き換えた成果です。
- ST-P3 は、知覚・予測・計画を時空間で一貫学習することで、各段を分離した構成より計画が改善することを nuScenes / CARLA で示し、ビジョンのみのE2Eの初期の足場になりました。
- 計画器の選択(回帰 vs 生成モデル)については、一般に回帰型(平均的な1本を出す)は安定だが多峰性(右にも左にも行きうる状況)を1本に潰す「平均化(mode averaging)」の傾向があり、最悪の場合「右にも左にも行ける」が「真ん中=障害物に突っ込む」になりかねません。flow matching や拡散モデル系は複数候補を表せる反面、学習・サンプリングのコストと安定化が課題、という傾向が各種研究で報告されています。

ここで頻出する指標の意味を補足します。L2 誤差は「予測した軌跡が人間の走った軌跡からどれだけずれたか」を測り、小さいほど人間らしい運転です。ただし open-loop(過去ログ上の再生)評価なので、実際に走らせたときの挙動を完全には反映しません。特に「自車の過去の動きだけからかなり当てられてしまう」ショートカット問題が指摘されており、L2 が良くても閉ループで安全とは限りません。衝突率は計画軌跡が他物体と重なる割合で、安全性の代理指標です。研究では絶対値よりベースライン比の改善方向と、速度・メモリのトレードオフ、できれば閉ループ評価を併読するのが妥当です。

## メリット・トレードオフ・限界
メリット
- 部品が文字列キーで差し替え可能で、新手法の実験コストが低い。バックボーン・融合・計画器を独立に比較でき、どの部品が性能に効くかを切り分けやすい。
- 学習/推論で出力契約を切り替え、損失計算を計画器内に閉じ込めることで、学習ループが計画器の種類に依存しない。回帰でも生成モデルでも同じ訓練コードが回ります。
- カメラ・地図・履歴・未来予測まで、E2Eに必要な枝が一通り揃う。新しいセンサや表現を枝として足しやすい拡張性があります。

トレードオフ・限界
- 高解像度の密 BEV(例:450×300)はメモリを多く食い、バッチサイズや解像度が GPU 容量に強く制約される。スパース表現([SparseDrive (スパース表現E2E)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1084_sparse-drive))に比べ計算が重く、車載のリアルタイム制約では不利になりえます。
- 未来状態予測(自己教師)の損失をどう接続するかは研究的に難しく、効果が安定して出ない場合がある。特徴次元の整合(予測出力と参照特徴の次元合わせ)に射影層が要ることがあり、また予測器と対象エンコーダが同じになると「全部ゼロを出せば損失ゼロ」という自明解(representation collapse)に陥る危険があります。
- 遅い側(world model 的モジュール)を併設する二層構想は、つなぎ方や学習信号の設計、二者の更新頻度の合わせ方が未解決の論点になりやすい。
- E2E 全般の弱点として、判断根拠が説明しづらく(ブラックボックス)、学習データに無いまれな状況での挙動保証が難しい。InterFuser のような明示的安全制約は標準では持ちません([InterFuser (安全制約付き解釈可能E2E)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1075_interfuser))。模倣学習由来の共変量シフトにも弱いです。

## 発展トピック・研究の最前線
- スパース化への移行: 密 BEV をやめてスパースなクエリ融合に置き換えると軽量化できる([SparseDrive (スパース表現E2E)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1084_sparse-drive))。融合方式を文字列キーで差し替える設計なら、スパース融合モードの追加が構造的に容易です。
- 地図のベクトル化: ナビ地図をラスタ画像でなくポリライン(ベクトル)で入れると効率と精度が上がりうる([VectorNet (ベクトル地図/エージェント符号化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0467_vectornet) / [VAD (Vectorized Scene Representation)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1094_vad-vectorized-scene))。
- 世界モデルとの統合: 低頻度で未来を想像する world model を反応器に接続し、長期の予見と短期の反射を分業させる研究が活発です([V-JEPA（潜在空間世界モデル）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1092_v-jepa-world-model))。GAIA-1 や DriveDreamer のような生成的 world model を「想像エンジン」として使う方向もあります。
- 安全層の追加: 中間表現から物体密度マップを補助出力させ、軌跡に明示的安全フィルタをかける拡張で、E2Eに解釈性・安全性を足す方向([InterFuser (安全制約付き解釈可能E2E)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1075_interfuser))。
- 計画器の生成モデル化: 拡散モデルや flow matching で多峰なマルチモーダル軌跡を扱い、open-loop だけでなく閉ループでの頑健性を高める流れ。VLM(視覚言語モデル)で意図を言語化してから軌跡を出す研究も増えています。
