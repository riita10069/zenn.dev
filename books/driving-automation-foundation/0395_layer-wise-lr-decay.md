# Layer-wise LR Decay (LLRD)

## ひとことで言うと
Layer-wise Learning Rate Decay (LLRD、層別学習率減衰) とは、事前学習済みのニューラルネットを新しいタスクに合わせて微調整 (fine-tuning) するときに、「入力に近い層ほど学習率を小さく、出力に近い層ほど大きく」設定し、隣り合う層ごとに一定の比率で連続的に減衰させる手法です。学習率 (learning rate) とは「重みを 1 回の更新でどれだけ大きく動かすか」を決める数字で、大きいほど大胆に・小さいほど慎重に重みが変わります。LLRD は、層ごとにこの学習率を変える discriminative learning rate の考え方を、数グループではなく全層に細かく当てはめたものです。Transformer (アテンション機構を積み重ねた深いネットワーク) 系のモデルである ViT (Vision Transformer) や BERT を転移するとき、最終精度をはっきり押し上げることが知られています。

## 直感的な理解
事前学習 (pretraining) とは、大量のデータでモデルにあらかじめ汎用的な知識を身につけさせておくことです。たとえば ImageNet という 1400 万枚規模の画像データセットで学習したモデルは、「エッジ・色・テクスチャ・物体の部品」といった画像認識の基礎を既に持っています。この知識を別のタスク (たとえば運転シーンの認識) に流用するのが転移学習 (transfer learning) です。

ここで困ることがあります。深いネットワークでは、入力に近い「下層 (lower layer)」は色やエッジのような汎用的でどんなタスクにも効く特徴を抽出し、出力に近い「上層 (upper layer)」はタスク固有の判断に近い特徴を扱う、という役割分担があります。下層が持つ汎用知識は壊したくない財産です。ところが fine-tuning ですべての層に同じ大きな学習率を使うと、新しいタスクのデータに引っ張られて下層の汎用特徴まで大きく書き換わってしまい、せっかくの事前学習の価値が失われます。これを破滅的忘却 (catastrophic forgetting) と呼びます。

逆に、忘却を恐れて全層に小さな学習率を使うと、今度はタスク固有の判断を担う上層の適応が遅く、新しいタスクへの当てはまりが頭打ちになります。

LLRD はこのジレンマを「層ごとに学習率を変える」ことで解きます。下層ほど学習率を小さくして汎用知識を守り、上層ほど学習率を大きくしてタスクに合わせて積極的に動かす。会社の改革にたとえると、長年積み上げた基礎方針 (下層) はそっと触り、最前線の現場運用 (上層) は大胆に変える、という配分です。

## 基礎: 前提となる概念
LLRD を理解するには、いくつかの前提を押さえる必要があります。

層 (layer) とは、ネットワークの 1 段分の計算ブロックです。Transformer なら「self-attention (自己注意、入力の各要素が互いにどれだけ関連するかを計算する仕組み) + feed-forward (全結合) のひとまとまり」が 1 層 (1 block) で、これを 12 段や 24 段積み重ねます。ResNet なら畳み込みブロックが層に相当します。

勾配 (gradient) とは、「損失 (loss、予測と正解のズレを測る値) を小さくするには各重みをどちらにどれだけ動かせばよいか」を示す微分の情報です。学習は「勾配を計算 → その方向に学習率ぶんだけ重みを動かす」を繰り返します。重みの更新量はおおざっぱに「学習率 × 勾配」です。だから学習率が小さければ更新は小さく、層はほとんど変わりません。

パラメータグループ (parameter group) とは、深層学習フレームワークで「このまとまりの重みにはこの学習率を使う」と指定できる単位です。LLRD は層ごとにパラメータグループを作り、層番号に応じた学習率を割り当てることで実装します。最適化アルゴリズム (optimizer、勾配から重みの更新量を決める仕組み) は、各パラメータグループに別々の学習率を持たせられるようになっているのが普通です。

埋め込み層 (embedding layer) とは、生の入力 (画像のパッチや単語トークン) を最初に数値ベクトルに変換する最下層です。LLRD では最も入力に近いので、最も小さい学習率が割り当てられます。

## 仕組みを詳しく
L 層のネットワークを考えます。一番上の層 (出力に最も近い L 層目) の学習率を基準 `base_lr` とし、減衰率 (decay rate) を `α` (アルファ、0 < α < 1、たとえば 0.65 や 0.75) とします。n 層目 (下から数えて n 番目) の学習率は次式で決めます。

```
lr(n) = base_lr × α^(L − n)
```

ここで `L − n` は「その層が最上層から何段下にあるか」を表す距離で、最上層なら 0、1 段下なら 1、最下層なら `L − 1` です。L 層目 (最上層) は `α^0 = 1` 倍で基準のまま、1 つ下の層は `α` 倍、さらに下は `α^2` 倍……と、入力に近づくほど学習率が指数的に小さくなります。

12 層の Transformer で `base_lr = 1e-3`、`α = 0.65` の場合の具体例:

```
層12 (最上層・出力側): 1e-3 × 0.65^0  = 1.00e-3
層11                : 1e-3 × 0.65^1  = 6.50e-4
層10                : 1e-3 × 0.65^2  = 4.23e-4
 …
層 1 (最下層・入力側): 1e-3 × 0.65^11 ≈ 7.5e-6
埋め込み層           : 1e-3 × 0.65^12 ≈ 4.9e-6
```

最下層の学習率は最上層の約 200 分の 1 まで小さくなります。新しく付けた分類 head (タスク固有の出力層) は事前学習されておらずランダム初期値なので、通常は基準 `base_lr` のまま、あるいはそれ以上の学習率で素早く学習させます。

before/after で見ると、減衰なしは「12 層すべて lr=1e-3」、LLRD ありは「層ごとに 1e-3 から 5e-6 までなだらかに減衰」という違いです。減衰率 `α` が 1 に近いほど全層がほぼ同じ学習率 (= 減衰なし) に、0 に近いほど下層がほとんど凍結 (固定) に近づきます。

なぜ「指数的に」減衰させるのかには理由があります。層を 1 段下るごとに「事前学習知識の価値」と「壊れやすさ」が一定割合で増す、という経験的な観察に対し、一定比率の掛け算 (指数減衰) が素直に対応するからです。線形に減衰させる変種もありますが、深いネットワークでは指数減衰が広く使われます。指数減衰は「下層に行くほど急速に学習率が小さくなる」性質を持ち、たとえば 24 層モデルで `α=0.75` なら最下層は最上層の `0.75^23 ≈ 0.0013` 倍、つまり実質ほぼ凍結に近い扱いになります。深いモデルほど下層が強く保護される、という直感に合致します。

実装は概念的にシンプルです。層ごとにパラメータグループを作り、上式の `lr(n)` を割り当てるだけです。

```python
# 概念コード: 下から上へ並んだ blocks に層別 lr を割り当てる
param_groups = []
num_layers = len(blocks)
for i, blk in enumerate(blocks):          # i=0 が最下層
    scale = decay ** (num_layers - 1 - i) # 上層ほど 1 に近い
    param_groups.append({"params": blk.parameters(),
                         "lr": base_lr * scale})
param_groups.append({"params": head.parameters(), "lr": base_lr})
optimizer = torch.optim.AdamW(param_groups)
```

実務上の注意点がいくつかあります。第一に、weight decay (重みを小さく保つ正則化) は層別にしないことが多く、bias や正規化層のパラメータは weight decay から除外する、というレシピが論文ごとにあります。第二に、層別倍率を「学習率そのもの」として持たせるか、「基準学習率に掛ける係数 (lr scale)」として持たせるかで、学習率スケジューラ (warmup や cosine decay) との組み合わせ方が変わります。一般には各パラメータグループに `lr_scale` を持たせ、スケジューラが出す共通の倍率に層別倍率を掛ける実装が安全です。こうすると warmup や cosine decay が全層に一様にかかりつつ、層間の相対比が保たれます。第三に、層の並び順 (どれが下層か) を取り違えると減衰の向きが逆になり、汎用知識を守るどころか壊してしまうため、実装時には埋め込み層が最小・head が最大になっていることを必ず確認します。

## 手法の系譜と主要論文
LLRD の発想は段階的に育ってきました。

Howard & Ruder, "Universal Language Model Fine-tuning for Text Classification (ULMFiT)", ACL 2018 (arXiv:1801.06146) が源流です。ここで discriminative fine-tuning (層別の学習率) という概念が提案されました。彼らは深い言語モデル (3 層 LSTM ベースの AWD-LSTM) を文書分類に転移する際、層ごとに学習率を変えることを提案し、典型的には「1 つ上の層の学習率を 2.6 倍」(言い換えると下層を約 1/2.6 倍ずつ減衰) というルールを使いました。動機は、下層の汎用言語知識を守りつつ上層をタスクに適応させること。これが「層ごとに学習率を変える」という考えを明確に確立しました。

Clark, Luong, Le, Manning, "ELECTRA: Pre-training Text Encoders as Discriminators Rather Than Generators", ICLR 2020 (arXiv:2003.10555) は、効率的な事前学習 (ある単語が本物か別モデルが差し替えた偽物かを当てる replaced token detection という課題) を提案した論文ですが、下流タスクへの fine-tuning で layer-wise LR decay を明示的なハイパーパラメータとして採用し、減衰率を 0.8 前後に設定しました。深い Transformer を安定して微調整するレシピの一部として LLRD を定着させました。

Bao, Dong, Wei, "BEiT: BERT Pre-Training of Image Transformers", ICLR 2022 (arXiv:2106.08254) は、画像をパッチに区切ってマスクし、欠けた部分の離散トークンを当てる自己教師あり学習で ViT を事前学習する手法です。下流の画像分類への fine-tuning で layer-wise learning rate decay を採用し、付録でその有効性を報告しました。これにより「LLRD は NLP だけでなく Vision Transformer の転移でも効く」ことが示され、画像分野に広まりました。

He, Chen, Xie ほか, "Masked Autoencoders Are Scalable Vision Learners (MAE)", CVPR 2022 (arXiv:2111.06377) も、画像の大部分 (約 75%) をマスクして元画素を再構成する自己教師あり事前学習で ViT を学習し、fine-tuning する際の標準レシピに LLRD を組み込みました。MAE は減衰率 0.65〜0.75 程度を ViT のサイズに応じて使い分けており (ViT-Base/Large で 0.65、ViT-Huge で 0.75 前後)、現在の Vision Transformer 転移の事実上の標準設定になっています。

このように、ULMFiT (NLP・概念提唱) → ELECTRA/BERT 系 (NLP で定着) → BEiT/MAE (Vision に拡張) という流れで、LLRD は大規模 Transformer を fine-tuning するための定番テクニックになりました。

## 論文の実験結果(定量データ)
評価指標をまず説明します。画像分類では top-1 accuracy (正解率) を使います。これは「モデルが最も確信したクラスが正解と一致した割合」で、ImageNet なら 1000 クラスから 1 つを当てる難問なので、1 ポイントの差でも研究上は大きな意味を持ちます。NLP では GLUE (複数の言語理解タスクを集めたベンチマーク) のスコアなどが使われます。

MAE の論文では、ImageNet-1K (1000 クラス・約 128 万枚) で ViT-Large を fine-tuning する際、LLRD の減衰率を変えたアブレーション (どの要素を抜く・変えるとどうなるかを調べる実験) が報告されています。報告によれば、層別減衰を入れることで top-1 accuracy が明確に改善し、ViT-Large では減衰なしと比べておよそ 1 ポイント前後の差が出ています。1000 クラス分類で 1 ポイントは「1 万枚のうち 100 枚を余計に正解できる」差で、大規模モデルの fine-tuning では大きな意味を持ちます。さらに重要なのは依存性の形で、減衰率を 1.0 (= 減衰なし、全層同じ学習率) にすると精度が明確に下がり、適切な減衰率 (ViT-Large でおよそ 0.65〜0.75) で最大になる、という山なりの関係が示されています。減衰率が小さすぎる (たとえば 0.5) と下層が学習されず未適応になり、これも精度を落とします。効果はモデルが大きいほど顕著で、ViT-Huge では LLRD の有無による差がさらに広がると報告されています。

BEiT でも同様に、fine-tuning 時に layer-wise LR decay を使うことが安定した精度向上につながると報告されています。ELECTRA では GLUE の各タスクで層別減衰を含むレシピが当時の最高性能更新に寄与しました。

定量的な含意をまとめると、(1) LLRD の効果は大きく深いモデルほど大きい、(2) 減衰率には最適値があり山なりの依存性を持つ、(3) 自己教師あり事前学習 (BEiT/MAE) のように下層表現が強力なモデルほど、それを守る LLRD の恩恵が大きい、という 3 点が実験から読み取れます。逆に、教師ありで大量データを使って事前学習したモデルや、下流タスクのデータが事前学習分布に近い場合は、減衰の効果がやや薄まる傾向も観察されています。

## メリット・トレードオフ・限界
メリット:
- 深く大きい Transformer を安定して fine-tuning でき、最終精度が上がりやすい。
- 下層の汎用的な事前学習表現を壊さずに、上層をタスクへしっかり適応させられる。
- 層ごとに連続的に減衰するので、粗いグループ分けより自然で調整しやすい。
- 追加の重みは不要で、学習率の割り当てを変えるだけで実装でき、推論時の計算コストはまったく増えない。

トレードオフ・限界:
- 減衰率 `α` という新たなハイパーパラメータが増え、タスク・モデルごとに調整が要る。最適値は山なりで、外すと逆効果になりうる。
- 効果はモデル規模に強く依存し、小さいモデルや浅いネットワークでは恩恵が薄い。
- 層数ぶんのパラメータグループを作るため、実装がやや煩雑で、層の並び順 (どれが下層か) を間違えると逆効果になる。
- 学習率スケジューラ (warmup や cosine decay) と組み合わせるとき、層別倍率とスケジューラの倍率の掛け合わせ方を正しく扱う必要がある。

未解決の課題として、最適な減衰率をデータやモデル構造から自動で決める方法はまだ確立しておらず、実務では試行錯誤に頼る面が残っています。また LLRD が暗黙に仮定する「層の深さ = 抽象度の単調な序列」は、残差接続やスキップ接続が複雑に絡む現代のアーキテクチャでは必ずしも成り立たず、純粋な深さ順の減衰が最適かどうかは議論の余地があります。

## 発展トピック・研究の最前線
LLRD は「全層を更新しつつ更新量を層で変える」アプローチですが、近年は parameter-efficient fine-tuning (PEFT、ごく一部のパラメータだけを学習する効率的な微調整) が台頭しています。代表が LoRA (Low-Rank Adaptation、Hu ほか 2021、arXiv:2106.09685) で、元の重みは凍結したまま小さな低ランク行列だけを学習します。LLRD が「全層を慎重に動かす」のに対し、LoRA は「ほとんど動かさず一部だけ足す」発想で、目的 (忘却を防ぎつつ適応する) は共通しています。両者は併用も可能で、LoRA を入れる層の選び方や、LoRA のランクを層ごとに変える「層別ランク割り当て」は LLRD の精神を引き継いだ研究です。

また LLRD は gradual unfreezing (時間軸で下層を後から学習対象にする) と相補的です。LLRD は「各層をどれだけ強く動かすか (学習率)」を、gradual unfreezing は「いつその層を動かし始めるか (タイミング)」を制御します。ULMFiT は両者を併用しました。さらに discriminative learning rate は LLRD のより粗い (数グループ版) 一般形で、LLRD はその細粒度版と理解できます。最近は「学習中に減衰率を動的に調整する」「層ごとの勾配ノルムを見て自動で学習率を割り当てる」といった適応的手法も提案されており、ハイパーパラメータ調整の負担を減らす方向の研究が進んでいます。
