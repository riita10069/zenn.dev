# カテゴリ深度分布 (depth bins)

## ひとことで言うと
カメラ画像の各ピクセルが「カメラからどれくらい離れているか(深度)」を、1つの数値でズバリ当てるのではなく、「2m の可能性 30%、3m の可能性 50%、4m の可能性 20%」のように、いくつかの距離候補(ビン)への確率分布として予測する手法です。これにより「この場所の奥行きには自信がない」という不確実性をそのまま数値で表現できます。1枚または複数のカメラ画像から3次元の世界、特に鳥瞰図(上空から見下ろした地図のような表現)を組み立てるとき、深度推定の曖昧さを正面から扱える点が本質です。

## 直感的な理解
人間は片目をつぶって写真を1枚見ただけでも、なんとなく「これは近い、あれは遠い」と判断できます。しかしこの判断は完全ではありません。例えば、おもちゃの車の写真と本物の車を遠くから撮った写真は、画像上ではまったく同じ大きさに写りえます。つまり「画像上の大きさ」と「実際の距離」は1対1で決まらない、という根本的な曖昧さがあります。これを単眼深度の曖昧性(monocular depth ambiguity)と呼びます。物理的には、1枚の画像はカメラの光線(レイ)に沿った世界を投影したもので、そのレイ上のどの距離に点があったのかという情報がそもそも失われています。これは「逆問題」と呼ばれる、答えが一意に定まらない問題です。

ここで、もしモデルに「この物体は 3.0m 先です」と一点で答えさせると、間違ったときに大きく外します。一方で「2m が 10%、3m が 60%、4m が 30%」と幅をもたせて答えさせれば、モデルは自分の自信のなさを表現できますし、後段の処理がその不確実性を考慮して使えます。霧の中の遠い物体なら分布が広がり、目の前のはっきりした壁なら分布が一点に尖る、という具合です。カテゴリ深度分布はこの「幅をもった答え」を機械学習で扱えるようにする道具です。

なぜわざわざ確率分布にするのか、もう一つ実用上の理由があります。カメラ画像を鳥瞰図(BEV: Bird's-Eye View)に変換するには、各ピクセルが3次元空間のどこに対応するかを決めねばなりません。深度が1点に固定されていると、その1点が間違っていたとき特徴が完全に違う場所に置かれてしまいます。確率分布にしておけば、複数の候補位置に薄く特徴をばらまけるので、学習の途中で正しい位置へ徐々に集中させることができ、しかも全体が微分可能(誤差を逆向きに流して学習できる)になります。回帰だと一点が動くだけですが、分布だとどのビンに重みを移すかを連続的に調整できるため、勾配が安定して流れるのです。

## 基礎: 前提となる概念
ここで使う用語を一つずつ噛み砕きます。

深度(depth)とは、カメラのレンズ中心からその点までの距離(メートル)のことです。厳密にはレイに沿った直線距離ではなく、カメラの光軸方向(画像面に垂直な Z 軸)への射影距離を指すことが多く、これを z 深度と呼びます。カメラの幾何では、画像上のピクセル位置 (u, v) と深度 z が分かれば、カメラの内部パラメータ(焦点距離 fx, fy、画像中心 cx, cy をまとめた行列 K)を使って、カメラ座標系の3次元点 (X, Y, Z) を一意に復元できます。式で書くと X = (u - cx) z / fx、Y = (v - cy) z / fy、Z = z です。ここで分かるのは、深度 z さえ手に入れば残りは単純な掛け算で3次元復元できる、ということです。だからこそ深度推定が3次元化の心臓部になります。

回帰(regression)と分類(classification)の違いも要です。回帰は「3.2m」のような連続値を直接出す方式です。分類は「複数の候補(クラス)のうちどれか」を確率で出す方式です。深度を回帰で解くと、損失関数(予測と正解のズレを測る関数、典型的には二乗誤差 L2)が複数の正解候補の平均を取りに行く性質があり、物体の手前(近い)と背景(遠い)の境界で、どちらでもない中間の距離を出してしまいがちです。これを境界のにじみ(depth bleeding)と呼びます。分類にすると、近い候補と遠い候補を別々のビンとして保持できるので、平均化によるにじみが起きにくくなります。これは画像のセグメンテーションが回帰でなく分類で解かれるのと同じ理由で、不連続なジャンプ(物体エッジでの深度の段差)を表現しやすいのです。

ソフトマックス(softmax)は、複数の実数を合計が1になる確率に変換する関数です。深度ビンごとに出した生のスコア(ロジット)を z_i とすると、ビン i の確率は exp(z_i) / Σ_j exp(z_j) で計算され、すべて足すと1になります。これが「確率分布」を作る仕組みです。

ビン(bin)とは「入れ物」「区間」のことです。連続した深度の範囲(例えば 1m から 60m)を区切った各区間を1つのビンとします。例えば 1m 刻みなら [1,2), [2,3), …, [59,60) で約 59 個のビンになります。分布から1つの代表深度が欲しいときは、各ビンの中心値をその確率で重み付けして足す期待値(soft argmax)を使うのが一般的で、d̂ = Σ_i p_i · c_i(c_i はビン i の中心深度)と書けます。

## 仕組みを詳しく
処理の流れを段階的に追います。まずカメラ画像を畳み込みニューラルネットワーク(CNN、画像から特徴を抽出する標準的な仕組み)に通して特徴マップを得ます。テンソル(多次元配列)の形は典型的に次のとおりです。

- 画像特徴: [B, C, H, W]
  - B = バッチサイズ(同時に処理する画像の枚数、例えば6台分で6)
  - C = チャネル数(特徴の種類の数、例えば64)
  - H, W = 縮小された特徴マップの高さ・幅(例えば 16 × 44)

次に、各ピクセルごとに深度分布を予測します。

- 深度分布: [B, D, H, W]
  - D = 深度ビンの数(例えば 59)

ここで [B, D, H, W] の各 (b, :, h, w) ベクトルにソフトマックスをかけ、D 個のビンの確率の合計が1になるようにします。具体例で見ます。あるピクセル (h, w) の分布が次のようだとします。

```
深度ビン:   1m   2m   3m   4m   5m   6m  ...
確率:      0.02 0.05 0.30 0.50 0.10 0.03 ...   (合計1.0)
```

これは「ここは 4m 先の可能性が最も高い(50%)が、3m の可能性も 30% ある」という意味です。輪郭のはっきりした近い壁なら 4m に 0.95 のように一点へ集中し、霧の向こうの遠方や平坦でテクスチャの乏しい領域では複数ビンに広く分散します。この分布の鋭さ・広がりこそが深度の不確実性そのものです。分布のエントロピー H(p) = -Σ_i p_i log p_i を計算すれば、不確実性を1つのスカラーとして取り出すこともでき、下流の追跡や計画に「ここは信頼できない」と伝えられます。

続いて、この深度分布と画像特徴を組み合わせて3次元の「特徴つき点群」を作ります。Lift-Splat-Shoot(LSS)という代表的な枠組みでは次の手順を踏みます。

1. 画像特徴 [B, C, H, W] を用意する。
2. 各ピクセルで深度分布 [B, D, H, W] を得る。
3. 各ピクセルで、特徴ベクトル(長さ C)と深度分布(長さ D)の外積を取る。つまり「そのビンの確率 × 特徴ベクトル」を D 個分作る。結果は [B, C, D, H, W] という大きなテンソルになる。
4. この計算の意味は「確率の高い深度には特徴を濃く、低い深度には薄く配置する」こと。確率 0.5 のビンには特徴が半分の強さで、確率 0.02 のビンにはほぼゼロの強さで置かれる。
5. D 個の深度それぞれの3次元位置(カメラ幾何で計算可能)に特徴を打ち上げ(lift)、BEV グリッドへ振りかけて合計する(splat)。同じグリッドセルに落ちた特徴は足し合わされ、上から見た特徴地図(BEV特徴)になる。

数値例で外積の大きさを実感します。C=64, D=59, H=16, W=44, B=6 とすると、錐台特徴 [B,C,D,H,W] は 6×64×59×16×44 ≈ 1.6 億要素です。float32 なら約 640MB で、これが「深度を分布にすると計算とメモリが重い」と言われる直接の理由で、後述のトレードオフに直結します。

なぜビンの区切り方を工夫するのか、も重要です。等間隔で区切ると、近距離も遠距離も同じ刻みになりますが、安全上は近い物体ほど正確な距離が欲しく、遠い物体は多少の誤差が許されます。そこで CaDDN は線形増加離散化(LID: Linearly Increasing Discretization)を提案しました。これは近くを細かく、遠くを粗くビン幅を広げていく方式です。ビン番号 i の深度境界を次のように二次的に増やすと、刻みが i に比例して広がります。

```
d_i = d_min + (d_max - d_min) · i(i+1) / (D(D+1))
```

ここで d_min, d_max は深度範囲の最小・最大、D はビン数です。等間隔(UD: Uniform Discretization)や対数間隔(SID: Spacing-Increasing Discretization、DORN が採用)という選択肢もあり、どれを選ぶかが精度に効きます。SID は対数スケールなので近距離をさらに細かく刻みますが、近距離の量子化が細かすぎてラベルノイズに敏感になることもあり、LID は SID と UD の中間の挙動を狙ったものと理解できます。

## 手法の系譜と主要論文
カテゴリ深度分布は、深度推定そのものの研究と、BEV 変換の研究という2つの流れが合流して生まれました。

DORN (Fu et al., CVPR 2018, arXiv:1806.02446) は「Deep Ordinal Regression Network for Monocular Depth Estimation」で、深度推定を回帰ではなく順序つき分類(ordinal regression)として解く先駆けでした。深度を SID(対数的に広がるビン)で離散化し、「この深度はビン k より大きいか」という K-1 個の二値判定の集合として学習しました。普通の分類ではビン同士の順序関係(2m より 3m が近い、という連続性)が無視されますが、順序回帰はそれを保つ点が巧妙です。これにより深度推定の精度が当時の KITTI 深度ベンチマークで大きく向上し、「深度は分類した方がよい場合がある」という発想を広めました。これがカテゴリ深度分布の思想的な源流です。

LSS: Lift-Splat-Shoot (Philion & Fidler, ECCV 2020, arXiv:2008.05711) は「Lift, Splat, Shoot: Encoding Images from Arbitrary Camera Rigs by Implicitly Unprojecting to 3D」で、深度を離散ビンの確率分布として予測し(Lift)、その確率で重み付けした特徴を3次元へ打ち上げ、BEV に集約(Splat)する微分可能な枠組みを確立しました。動機は、車両周囲を覆う複数台カメラの画像を共通の鳥瞰図へ統合することでした。重要なのは、深度の正解ラベルを一切使わず、最終タスク(BEV セグメンテーション)の学習信号だけで深度分布が自然に立ち上がる暗黙的(implicit)深度学習だった点です。手軽な一方で、深度分布はぼやけがちでした。

AdaBins (Bhat et al., CVPR 2021, arXiv:2011.14141) は単眼深度推定の文脈で、ビンの幅を固定せず画像ごとに Transformer で適応的に決める手法を提案しました。同じシーンでも屋内の近距離主体の画像と屋外の遠距離主体の画像ではビン配置の最適解が違う、という洞察です。固定ビンへの依存を緩める研究方向の代表例です。

CaDDN (Reading et al., CVPR 2021, arXiv:2103.01100) は「Categorical Depth Distribution Network for Monocular 3D Object Detection」で、本トピックの名前の由来です。LSS と同じく深度を分布として予測しますが、LiDAR(レーザーで距離を測るセンサー)由来の正解深度で分布を直接教え込む明示的(explicit)深度監督を導入しました。暗黙学習のぼやけを断ち切り、単眼3D検出の精度を引き上げました。

BEVDepth (Li et al., AAAI 2023, arXiv:2206.10092) は明示的深度監督を複数カメラの大規模設定で体系化し、カメラの内部・外部パラメータをネットワークに注入する Camera-aware の工夫や、計算を高速化する Voxel Pooling を加えました。これによりカメラのみ手法の深度品質と検出精度をさらに押し上げ、カテゴリ深度分布+明示監督が標準的な構成として定着しました。

## 論文の実験結果(定量データ)
評価指標の意味から説明します。

KITTI 3D 検出ベンチマークでは AP(Average Precision、平均適合率)を使います。これは「正しく検出できた割合と、誤検出の少なさ」をまとめた指標で、適合率と再現率の曲線の下の面積として計算され、高いほど良いです。3次元のボックスが正解とどれだけ重なるか(IoU、Intersection over Union)が閾値(車は 0.7)を超えたものを正解とみなします。難易度は Easy / Moderate / Hard に分かれ(物体の大きさ・遮蔽・切れ具合で決まる)、Moderate がランキングの基準です。

CaDDN は KITTI の car クラスで、テストセットの 3D AP(Moderate)が約 19.2% に達したと報告しました。これは発表当時の単眼3D検出手法の中で最高水準で、先行する Pseudo-LiDAR 系や M3D-RPN(約 9.7% Moderate)と比べて 2 倍近い前進でした(数値は報告による)。深度分布をシャープにしたこと、すなわち明示的深度監督が効いたことが、この向上の主因です。

アブレーション(ある要素を抜くと性能がどう変わるかの検証)も示唆的です。CaDDN の研究では、(a) 深度を明示的に監督せず暗黙学習に任せる、(b) ビンの区切りを LID から等間隔(UD)に変える、といった改変で精度が落ちることが報告されています。特に深度監督を外すと分布がぼやけ、3D 位置の精度が顕著に低下しました(報告では数ポイント単位の AP 低下)。これは「カテゴリ分布という形だけでなく、正解で教えることが効いている」ことを意味します。LID から UD への変更でも AP が下がり、近距離分解能の重要性が裏づけられました。

BEV 変換側では、BEVDepth が nuScenes(車両周囲360度を6カメラ+LiDAR で記録した大規模データセット、約 1000 シーン・140 万枚のカメラ画像)で、深度を LiDAR で明示監督するとカメラのみ検出の総合指標 NDS(nuScenes Detection Score)が大きく改善することを示しました。BEVDepth はカメラのみで NDS 0.60 前後(ResNet 系の高解像度・時間融合あり設定、報告による)に到達し、深度監督なしの構成に対して明確な優位を示しました。NDS は検出の正確さ(mAP)・位置誤差・スケール誤差・向き誤差・速度誤差・属性誤差などをまとめた総合点で、1に近いほど良い指標です。BEVDepth のアブレーションでは、深度監督を加えるだけで予測深度の絶対誤差が顕著に下がり、それが mAP・mATE(位置誤差)の改善へ直結することが定量的に示されました。

## メリット・トレードオフ・限界
メリットは次のとおりです。深度の不確実性を確率分布として明示的に持てるので、自信のない場所を広い分布として扱えます。すべての処理が微分可能なので、最終タスクの誤差から end-to-end で学習できます。分類なので近い候補と遠い候補を別々に保持でき、回帰で起きる境界のにじみを避けやすくなります。分布のエントロピーから不確実性スコアを取り出して下流へ渡せる、という副産物もあります。

トレードオフと限界もはっきりしています。第一に計算とメモリです。ビン数 D だけ出力チャネルが増え、錐台特徴 [B,C,D,H,W] が巨大になります(前述の約 1.6 億要素の例)。第二にビン設計依存です。最大距離・ビン数・離散化方式(UD/SID/LID)が精度を左右し、データセットごとにチューニングが要ります。第三に学習方法のジレンマです。暗黙学習(LSS)は楽だが深度がぼやけ、明示学習(CaDDN/BEVDepth)はシャープだが LiDAR ラベルの収集コストがかかります。第四に原理的限界です。単眼である限り、テクスチャの乏しい面や遠方では分布が広がりやすく、LiDAR ほどの深度精度には届きません。第五にキャリブレーションの問題で、ソフトマックスが出す「50%」が本当に 50% の的中率に対応している保証はなく、過信した分布を出しがちです。

未解決の課題としては、ラベルなしで深度をシャープにする自己教師あり手法の高度化、ビンを固定せずデータに応じて適応的に決める手法(AdaBins 系)、確率分布のキャリブレーションの保証(温度スケーリングなどの後処理を超えた学習時の工夫)、そして遠方ビンの分解能不足の解消などが研究の焦点です。

## 発展トピック・研究の最前線
明示的深度監督を一般化し、時系列の多視点情報をステレオ視として使って深度を精緻化する流れ([BEVStereo](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0274_bevstereo))や、深度分布を陽に持たずに学習可能なクエリで3次元から画像へ逆向きにサンプリングする Transformer 系(BEVFormer など)との対比が研究の中心です。後者は錐台テンソルの巨大化を避けられる一方、幾何の明示性が下がるという別のトレードオフを持ちます。また、深度分布のエントロピー(分布の広がり)を不確実性スコアとして下流の追跡や計画に渡す研究、深度ラベルを LiDAR ではなく構造から復元する(structure-from-motion)疑似ラベルで代替する研究も進んでいます。さらに近年は、明示的な深度ビンを介さず3次元占有(occupancy)を直接予測する手法や、深度を確率体積(probabilistic volume)としてマルチビューステレオの理論と接続する研究も活発で、カテゴリ深度分布は「微分可能な3次元化」という大きな問題系の一つの解として位置づけ直されつつあります。
