# IQL Expectile 回帰損失

## ひとことで言うと
IQL(Implicit Q-Learning、暗黙的Q学習)の expectile(エクスパクタイル)回帰損失とは、過去に集めたログだけ(新たに環境で試行錯誤しない=オフライン)から「ある状況がどれだけ価値があるか」を、楽観しすぎず保守的に見積もるための損失です。普通の平均ではなく「上側の残差を重く、下側を軽く」と非対称に回帰することで、データに実在する良い行動の価値を拾いつつ、データに無い未知の行動を過大評価しない、という芸当を実現します。これがオフライン強化学習で最も厄介な「分布外行動の過大評価」問題を、構造的に回避する鍵になります。

## 直感的な理解
プロのドライバーの運転記録を山ほど集めたとします。あなたはそれを見るだけで(自分で運転して事故るリスクは負わずに)上手な運転を学びたい。ここで素朴に「各状況で一番良かった結果を真似よう」とすると、罠にはまります。記録に無い操作についても「もしこうしていればもっと良かったはず」と勝手に夢想し、その根拠のない高評価を信じて危険な真似をしてしまうのです。

IQL の発想は「記録に無い行動については一切評価を下さない。記録に実在する行動の中で、結果が良かった方に注目する」というものです。expectile 回帰は、この「実在する結果のうち良い方に注目する」を数学的にきれいに行う道具です。平均(全部を等しく見る)でも最大(一番良いものだけ見る、外れ値に弱い)でもなく、その中間の「良い方に少し寄せた値」を取ります。なぜ最大ではなく expectile なのか。最大は1つの幸運な記録(外れ値、ノイズ)に振り回されますが、expectile は良い方に寄せつつ全データのばらつきを使うので頑健です。

## 基礎: 前提となる概念
強化学習(RL)の用語を平易に。状態(state, s)はいまの状況(カメラ画像や車速など)、行動(action, a)はとる操作(ハンドル角・アクセル)、報酬(reward, r)は各時刻の良さの点数です。割引率 γ(0〜1、たとえば0.99)は「将来の報酬を現在からどれだけ重視するか」の係数で、遠い未来の報酬ほど γ を何度も掛けて割り引きます(なお、この γ は Focal Loss の集中度 γ とは無関係の別物です)。

価値関数 V(s) は「状態 s から方策に従って進んだとき、この先に積み上がる割引報酬の期待値」、行動価値関数 Q(s,a) は「状態 s で行動 a を取り、その後は方策に従ったときの期待値」です。RL のゴールは Q が高くなる行動を選ぶ方策(policy、状態を入力に行動を出す関数)を学ぶことです。アドバンテージ A(s,a) = Q(s,a) - V(s) は「その行動が平均よりどれだけ良いか」を表します。

オフライン RL は、シミュレータや実車で新たに試行錯誤せず、あらかじめ集めた固定データセット(専門家の運転ログなど)だけで方策を改善する設定です。実車での試行錯誤は危険で高価なので、自動運転で特に重要です。

ここで核心の落とし穴。通常の Q 学習の更新は `Q(s,a) ← r + γ max_{a'} Q(s', a')` と、次状態 s' で最良の行動 a' を選んだときの Q を使います。この max が問題です。max はデータに一度も現れていない行動(out-of-distribution action、分布外行動)まで候補に含めて最大値を取ろうとします。学習されていない行動の Q は当てずっぽうで、しばしば過大に出ます(関数近似器は訓練していない入力に対して任意の値を返しうる)。その幻の高評価を `max` が拾い、ブートストラップ(自分の推定を自分の目標に使う連鎖)で雪だるま式に増幅され、方策がデータに無い危険な行動へ暴走します。これが分布外行動の過大評価(overestimation)で、オフライン RL 最大の敵です。

IQL の狙いは「Q の更新から max を排除し、データに実在する行動だけで価値を推定する」ことです。その推定に expectile 回帰を使います。

## 仕組みを詳しく
expectile(エクスパクタイル)は、分位点(quantile)回帰の二乗誤差版だと思うと掴みやすいです。普通の最小二乗回帰は残差(予測と実測の差)を符号によらず対称に二乗して平均し、結果として平均値を当てに行きます。expectile 回帰は残差の符号で重みを変え、非対称にします。

残差 `diff = q - v`(観測された Q 値から価値予測 v を引いた値)に対し、重み付き二乗誤差を最小化します。τ(タウ)は 0〜1 の expectile パラメータです。

```
weight = τ        (diff > 0 のとき, Q が予測 v より大きい = 上振れ)
weight = 1 - τ    (diff <= 0 のとき, 下振れ)
loss   = mean( weight * diff^2 )
```

τ = 0.5 だと上下が同じ重みで、ただの最小二乗(平均を当てる)に戻ります。τ を 0.5 より大きくすると、上振れ(diff>0、Q が予測より高い=良い行動)の残差を重く罰するので、v はデータ中の良い行動寄りの値、つまり「この状態で実際に取られた行動のうち良い方の価値」に引き上げられます。τ→1 で最良値に近づき(攻めた見積もり)、τ→0.5 で平均的(保守的)になります。実用では τ=0.7〜0.9 がよく使われます。

数値例。ある状態で観測された行動の Q 値が `[1.0, 1.0, 5.0]`(2回は普通、1回だけ良い)だったとします。

```
τ=0.5 (=平均)  → v ≈ 2.33   (素直な平均)
τ=0.7         → v ≈ 3.5 付近 (良い方に寄る)
τ=0.9         → v ≈ 4.7 付近 (ほぼ最良値に寄る)
```

決定的に重要なのは、この「上側を取る」操作を、データに無い行動を一切持ち出さず、観測済みの Q 値のばらつきだけから実現している点です。`max_{a'}` を使わないので、分布外行動の過大評価が原理的に起きません。これが IQL の「暗黙的(implicit)」の意味で、最良行動を明示的に選ばずに、価値推定の非対称性を通じて暗黙に最良に近い価値を得ています。理論的には、τ→1 の極限で expectile は「データに存在する行動に限定した上での max」、すなわち in-support max に収束し、これが分布外を見ない安全な最大値推定になります。

IQL 全体は3つの更新を交互に回します。形状の感覚として、状態 s から特徴抽出器が特徴 feat を出し、価値ネット V は `[B,1]`、Q ネットは `[feat ⊕ action] → [B,1]` を出すとします(B はバッチサイズ)。

1. V 更新(expectile 回帰、本トピックの中心): `V(s)` を `Q(s,a)` の上側 expectile に近づける。Q はターゲットネットワーク(更新を緩やかにした Q のコピー)を使い、勾配が V→Q へ逆流しないよう止める。

```
vf_loss = expectile_loss(target_q(s,a) - v_pred, tau)    # 上の式そのもの
```

2. Q 更新(通常の二乗誤差): `Q(s,a)` を `r + γ V(s')` に近づける。max を使わず、次状態の価値 V を使うのがポイント。V がすでに「データ内の良い行動の価値」を表しているので、max なしでも改善方向の情報が伝わります。

```
q_target = r + γ * V(s')        # next state value, max を使わない
qf_loss  = mse(q_pred, q_target)
```

3. 方策更新(Advantage-Weighted Regression, AWR): アドバンテージ `A = Q(s,a) - V(s)` が大きい(=平均より良かった)行動を、`exp(β·A)` の重みで重点的に模倣する。β は温度(逆温度)で、大きいほど「良い行動だけを尖って真似る」。

```
advantage   = Q(s,a) - V(s)
weights     = exp(β * advantage).clamp(max=100)   # 発散防止にクリップ
policy_loss = mean( weights * mse(pred_action, action) )
```

このように「価値の見積もり(V, Q)」と「方策」を、すべてデータ内の行動だけで完結させながら改善します。Q 学習と違い、方策抽出が重み付き模倣(回帰)なので、これも分布外行動を生み出しません。学習・抽出のどの段階でもデータの外に出ない、という一貫性が IQL の安定性の源です。なお IQL は方策を「価値学習の後段」に切り出しており、価値関数を一切いじらずに方策だけ後から抽出できる(同じ V,Q から β を変えて複数方策を作れる)という実装上の柔軟性もあります。

## 手法の系譜と主要論文
統計的な源流は Newey & Powell の "Asymmetric Least Squares Estimation and Testing"(Econometrica 1987)で、expectile という非対称最小二乗の概念を計量経済学で導入しました。分位点回帰(残差の絶対値を非対称に重み付け)の二乗誤差版にあたり、いたるところ微分可能で勾配学習と相性が良い性質があります。

オフライン RL の系譜では、まず分布外行動の過大評価をどう抑えるかが中心課題でした。Kumar, Zhou, Tucker, Levine の Conservative Q-Learning(CQL、NeurIPS 2020、arXiv:2006.04779)は、分布外行動の Q を明示的に押し下げる正則化項を加えて保守性を担保しました。学習した Q が真の価値の下界になることを理論的に保証する一方、押し下げの強さの調整が難しく、悲観的になりすぎて性能が出ないことがある、という弱点がありました。Fujimoto & Gu の TD3+BC(NeurIPS 2021、arXiv:2106.06860)は「Q 学習に行動クローン(模倣)項を足すだけ」という極小の修正で強い性能を出し、複雑な保守性正則化が必須でないことを示しました。

この流れの上に Kostrikov, Nair, Levine の "Offline Reinforcement Learning with Implicit Q-Learning"(ICLR 2022、arXiv:2110.06169)が登場します。新規性は「Q 更新で分布外行動の max を取らず、価値 V を expectile 回帰でデータ内行動の上側として推定し、方策は AWR で抽出する」枠組みを、分布外行動に一度も触れずに完結させた点です。CQL のように分布外 Q を見て押し下げる必要すらなく、そもそもデータの外を見ない。方策抽出も AWR(Peng et al. 2019、advantage-weighted regression、arXiv:1910.00177)という重み付き模倣で、これもデータ内に閉じています。動機は「分布外の過大評価を構造的に避けつつ、CQL のような過度の悲観も避ける」両立でした。

## 論文の実験結果(定量データ)
IQL は D4RL(Datasets for Deep Data-Driven RL)というオフライン RL の標準ベンチマークで評価されました。D4RL はロコモーション(halfcheetah, hopper, walker2d などの歩行ロボ制御)、AntMaze(迷路をアリ型ロボが進む、報酬がゴール到達時のみという疎報酬+長期計画が要る難タスク)、Adroit(ロボハンド操作)、Kitchen(複数サブタスクを順にこなす操作)などを含み、各タスクにデータ品質の異なるデータセット(専門家のみ medium-expert、中級 medium、探索的 random など)が用意されています。指標は正規化スコア(ランダム方策を0、専門家を100に正規化したスコア、高いほど良い)です。

ロコモーション系では、IQL は CQL やその他の当時の手法と同等以上のスコアを、より単純な実装と速い学習で達成しました。最も差が出たのが AntMaze です。AntMaze は疎報酬で、良い軌跡の断片を繋ぎ合わせる「ステッチング(stitching)」能力(別々の不完全な軌跡をつないで最適経路を構成する)が要求されます。ここで IQL は当時の手法を明確に上回るスコアを報告し、特に難しい antmaze-large 系で大きな改善を示しました。これは「max を取らずに価値を伝播させる」設計が、疎報酬での長期的な価値伝播にも有効であることの実証です。Kitchen や Adroit でも当時の最先端に匹敵する結果を出しました。

アブレーションで最も重要なのが expectile パラメータ τ です。τ=0.5(=ただの平均、SARSA 的な on-data 価値)では性能が伸びず、τ を 0.7、0.9 と上げるほど(特に AntMaze で)スコアが向上することが示されました。これは「上側 expectile を取ること」自体が性能の源であり、単なる平均回帰では不十分だという直接の証拠です。ただし τ を1に近づけすぎると in-support max ですらサンプル数の少ない状態で1つの上振れ Q に引っ張られ、データのばらつきやノイズに振り回されて不安定化する傾向も報告され、τ には最適点がある(攻めすぎも保守的すぎもダメ)という非単調性が見られます。方策側の温度 β も同様にアブレーションされ、大きすぎると重み `exp(β·A)` が一部サンプルに集中して発散しがちなため、クリップが実装上要ることが示されています。

計算効率の面でも、IQL は CQL のような重い正則化計算(分布外行動のサンプリングと log-sum-exp 評価)を持たず、教師あり回帰の組み合わせだけなので学習が速く安定、という実用上の利点が定量的に報告されています。

## メリット・トレードオフ・限界
メリット。実車・シミュレータでの試行錯誤なしに過去ログだけで方策を改善でき、安全・低コスト。Q 更新で分布外行動の max を取らないため、オフライン RL 最大の敵である過大評価を構造的に回避する(押し下げ正則化のような後付けの対症療法が要らない)。expectile の τ で「平均的(τ=0.5)」から「保守的に良い方を狙う(τ→1)」まで連続調整でき、実装も重み付き二乗誤差の数行で済む。CQL 等の明示的悲観に比べ安定で実装が軽い。疎報酬・要ステッチングのタスクにも強い。価値学習と方策抽出が分離しており、同じ価値関数から β を変えて方策を作り直せる。

トレードオフと限界。第一に、データ分布内に良い行動が含まれていないと改善できません。IQL はデータの外を見ない設計なので、性能上限がデータの質に縛られます(専門家データが無ければ専門家級にはなれない)。第二に、τ(価値の楽観度)と β(方策の重点度)のチューニングが要り、τ は大きすぎると不安定、β は大きすぎると重みが発散します(クリップが定番)。第三に、AWR による方策抽出は重み付き模倣なので、データ内の行動の補間しかできず、データ点間を大胆に外挿する真の最適化はしません。第四に、報酬が代理(proxy reward、本来測りたい品質の近似)だと、最適化先が本当に望む挙動とずれます。第五に、expectile はあくまで「期待値の非対称版」であって分位点そのものではないため、目指している「条件付き max」との対応は τ→1 の極限近似であり、有限の τ では厳密な最大値とは一致しません。

研究上の未解決課題は、τ をデータから自動決定する方法、expectile 回帰の保守性の理論的な特徴づけ(なぜ AntMaze のステッチングに効くのかの完全な理解)、そしてオフラインで学んだ方策をオンラインで安全に微調整する際の expectile の役割です。

## 発展トピック・研究の最前線
IQL の後継として、価値推定を expectile から別の非対称関数に置き換える研究が続いています。Extreme Q-Learning(XQL、Garg et al., ICLR 2023、arXiv:2301.02328)は、報酬や価値の誤差が極値分布(Gumbel 分布)に従うという仮定を置き、Gumbel 回帰を使って `max` を陽に取らずに対数和指数(soft max)的な最大値推定を行い、IQL の expectile を別の理論枠組みから一般化しました。これにより τ のような分位点パラメータを使わずに保守的な最大値を得られます。

オフラインからオンラインへの移行(offline-to-online fine-tuning)は活発な領域です。オフラインで IQL 等により安全な初期方策を学び、その後オンラインで少量の試行錯誤を許して仕上げる際、価値推定が保守的すぎると改善が遅く、楽観的すぎると初期に崩れる(オフラインからオンラインへ切り替えた瞬間に性能が落ちる distribution shift)、というジレンマがあり、τ のスケジューリングや切り替えが研究されています。

自動運転・ロボティクスへの応用では、模倣学習([模倣学習 (Behavior Cloning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0560_imitation-learning))で初期方策を学んだ後、同じログデータを使って IQL 的なアドバンテージ重み付けで「平均的な模倣」を「良い行動寄りの模倣」へ磨き上げる、という後段学習(post-training)の使い方が一般的です。模倣だけだとデータ中の良い行動も悪い行動も等しく真似てしまうのに対し、価値推定で良い行動を選り分けられるのが利点です。報酬は実データから滑らかさ(ジャークの小ささ)や前進・車線維持といった代理指標で構成することが多く、この代理報酬の設計品質が最終的な運転品質を左右する、という点が応用上の最大の論点です。研究コミュニティでは、報酬を人手で設計せず軌跡の好ましさから学ぶ(preference-based RL)方向と IQL を組み合わせる試みも進んでいます。
