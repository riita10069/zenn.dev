# TransDreamer

## ひとことで言うと
TransDreamer は、Dreamer という有名な世界モデル型 RL の「記憶を担う中身」を、RNN(順番に1ステップずつ状態を更新する再帰型ネット)から Transformer(系列全体を一度に見られるアテンション機構)に置き換えたものです。これにより、長い時間にわたる出来事の関係(長期依存)をうまく覚えられるようにしました。世界モデルのアーキテクチャを RNN から Transformer へ移す潮流の、初期の代表例です。

## 直感的な理解

世界モデル型 RL(環境のミニチュアを学習し、その想像の中で方策を鍛える、[IRIS (Transformer世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0063_iris-transformer-world-model) 参照)の代表が Dreamer 系です。Dreamer の心臓部は RSSM(Recurrent State-Space Model)という仕組みで、ここに RNN(具体的には GRU というゲート付き RNN)が使われています。

RNN は「1ステップ前の隠れ状態」を次に渡して時間を進めるので、原理的には過去全体の情報を持ち越せます。リレーでバトンを渡し続けるイメージです。しかし実際には次の弱点があります。

- 長期依存が苦手: 数百ステップ前の重要な出来事(たとえば「鍵を取った」)を、ずっと後(扉を開ける場面)まで隠れ状態の中に保持し続けるのは難しい。情報が薄まって消えていきます(勾配消失)。バトンに書いた情報が、何百人もリレーするうちに掠れていくようなものです。
- 逐次計算で遅い: RNN は時刻 t の計算が t-1 の結果に依存するので、原理的に並列化できない。学習が遅い。

一方 Transformer は、系列の任意の時刻どうしをアテンション(注意機構、関連する過去を直接参照する仕組み)で結びつけられます。100ステップ前でも1ステップ前でも「直接見に行ける」ので長期依存に強く、しかも学習時は系列全体を並列処理できます。自然言語処理(NLP)で実証済みのこの強みを世界モデルに持ち込んだのが TransDreamer です。

## 基礎: 前提となる概念

- RNN と GRU: RNN は系列を1ステップずつ処理し、隠れ状態に過去を要約します。GRU(Gated Recurrent Unit)はゲート機構で「何を覚え何を忘れるか」を制御する RNN の一種です。
- アテンション機構: 各時刻が「過去のどの時刻をどれだけ参照するか」を重み(クエリとキーの内積から計算)として求め、関連する情報(バリュー)を直接取り込む仕組み。Transformer の中核です。
- 因果マスク(causal mask): 自己回帰予測で「未来を見ない」ようにアテンションを制限する仕組み。時刻 t は t 以前だけ参照します。
- 状態空間モデル(state-space model): 観測の背後にある隠れ状態の時間発展をモデル化する枠組み。RSSM はこれを決定的成分と確率的成分に分けて学習します。
- 想像内学習(learning in imagination): 学習した世界モデルの中で仮想的にロールアウトし、その仮想軌跡で方策(actor-critic)を鍛えること。実環境への接触を減らせます。

## 仕組みを詳しく

### RSSM のおさらい(Dreamer の世界モデル)

RSSM は各時刻の状態を2つに分けて扱います。

- 決定的状態 `h_t`: 過去全体を要約した「記憶」(RNN が更新する部分)。
- 確率的状態 `z_t`: その時刻の不確実性を表す確率変数(観測から推定する部分)。

Dreamer はこの `(h_t, z_t)` の流れの上で、報酬予測・観測再構成・方策(actor-critic)を学習します。なぜ決定的成分と確率的成分に分けるかというと、決定的成分が長期の文脈を安定して運び、確率的成分が観測ごとの不確実性(画面のどこに何が出るか分からない)を吸収するという役割分担のためです。DreamerV2 ではこの `z_t` を離散カテゴリ変数(複数のone-hotベクトルの組)にして、Atari のような多峰な不確実性を表現しやすくしました。

### TransDreamer の変更点: TSSM(Transformer State-Space Model)

TransDreamer は RSSM の「記憶を更新する RNN」を Transformer に置き換えた TSSM(Transformer State-Space Model)を提案しました。

- RNN の逐次更新 `h_t = GRU(h_{t-1}, ...)` をやめる。
- 代わりに、過去の状態列 `(z_1, a_1), ..., (z_{t-1}, a_{t-1})` に対して Transformer のアテンションを適用し、現時刻の決定的状態 `h_t` を計算する。
- このとき因果マスクを使い、時刻 t は t 以前だけを参照する。

つまり「記憶 `h_t`」を、RNN の積み重ねではなく「過去の全状態への重み付き参照」として作ります。確率的状態 `z_t` の扱いや、報酬・観測の予測、想像内 actor-critic 学習という Dreamer の骨格は維持し、変えたのは「記憶の作り方」だけです。

```
RNN版(RSSM):
  h_1 → h_2 → h_3 → ... → h_t   (1本の鎖。遠い過去は薄れる)

Transformer版(TSSM):
  h_t = Attention( query=今, keys/values=過去の全状態 )
  (遠い過去でも必要なら直接強く参照できる)
```

### 学習の流れは Dreamer と同じ

世界モデル(TSSM)を学習 → 想像の中(imagination)でロールアウト → その中で actor-critic を学習、という Dreamer の枠組みはそのまま使います。変えたのは「記憶の作り方」だけ、と理解すると分かりやすいです。

学習時は、収集済みの軌跡(過去のプレイ記録)を系列としてまとめて Transformer に流せるので並列化が効きます。一方、想像時には過去のトークン/状態を保持しながら1ステップずつ次状態を生成するため、各ステップで過去系列全体へのアテンションが走ります。これが後述の計算コストの源です。

### 具体的な効果が出る場面

論文では、過去の手がかりを長く覚えていないと解けない記憶系タスク(例: 一度見た情報を後で使うナビゲーション、隠れた目標の記憶)で、RNN ベースの Dreamer より明確に強いことを示しました。短期依存中心のタスクでは大差ないこともありますが、長期記憶が要るほど Transformer 化の効果が出ます。これは「長期依存を直接参照できる」という Transformer の理論的強みが、実タスクで顕在化する条件を示した点で示唆的です。逆に言えば、Transformer 化は「常に良い」のではなく「記憶要求が高い課題で効く」道具だと位置づけられます。

## 手法の系譜と主要論文

### 前提: PlaNet / Dreamer / DreamerV2 と RSSM (Hafner et al., 2019–2021)

PlaNet(ICML 2019)が RSSM を導入し、画像から潜在ダイナミクスを学んで計画(CEM などの軌道最適化)する枠組みを示しました。Dreamer(ICLR 2020)は「観測を潜在状態に圧縮し、その潜在空間の中で想像して RL を回す」枠組みを確立し、勾配を想像軌跡に通して方策を直接学べるようにしました。DreamerV2(ICLR 2021)は離散潜在を導入して Atari で単一 GPU・人間超えを達成しました。TransDreamer はこの確立された土台の「記憶モジュール」だけを差し替えた、という位置づけです。

### TransDreamer (Chen, Wu, Yoon, Ahn, 2022, arXiv:2202.09481)

RSSM の RNN を Transformer に置換した TSSM を中核とする世界モデル型 RL を提案。動機は、RNN の長期依存の弱さと逐次計算の遅さを、アテンションの直接参照と並列性で解決することでした。長期記憶を要するタスクで Dreamer(DreamerV2)を上回る性能を示し、学習時の並列計算の利点も報告しました。世界モデルの記憶機構を Transformer 化した初期の明示的な提案として位置づけられます。Transformer 自体は Vaswani et al. "Attention Is All You Need" (2017) に由来し、その長期依存・並列性の利点を世界モデルへ輸入した格好です。

### 並行する Transformer 世界モデルの潮流

同じ「世界モデルを Transformer 化する」流れには、トークン化路線の IRIS([IRIS (Transformer世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0063_iris-transformer-world-model), Micheli et al., ICLR 2023)、Transformer-XL 流の TWM(Robine et al., ICLR 2023)、確率的潜在を持つ STORM(Zhang et al., NeurIPS 2023)があります。IRIS が「画像を離散トークン列にして言語モデル流に予測する」路線なのに対し、TransDreamer は「RSSM の状態空間モデルを維持したままアテンションを入れる」路線で、アプローチが異なります。前者は1フレームを多数のトークンに展開する(系列が長くなる)のに対し、後者は1ステップを1つの圧縮状態にまとめる(系列が短い)違いがあります。両者は「RNN を脱して Transformer の長期依存・並列性を世界モデルに取り込む」という共通目標を、異なる設計で追求しました。

## 論文の実験結果(定量データ)

TransDreamer の評価は、記憶を要する制御タスクと標準的な制御ベンチマークで行われました。指標と意味を補足しつつ要点を述べます。

- 記憶系タスク(Hidden Order Discovery など、過去に見た情報を後で使う必要があるタスク): TransDreamer は DreamerV2 を明確に上回る最終性能・学習速度を報告。これらのタスクでは「数十〜数百ステップ前の観測を保持できるか」が成否を分け、Transformer の直接参照が効きます。ここでの性能指標は累積報酬(エピソードで得た得点の合計)で、高いほど良い。RNN 版は記憶期間が伸びるほど性能が崩れる一方、TSSM はなだらかに保たれることが示されました。
- DMC(DeepMind Control)など連続制御ベンチマーク: 記憶要求が低めのタスクでは DreamerV2 と同等程度。長期記憶が不要な場面では Transformer 化の利得が小さく、追加の計算コストに見合わないことがあると示されました。これは「Transformer 化が万能ではなく、タスクの記憶要求に依存する」という重要な知見です。
- 計算面: 想像時に各ステップで過去全系列へのアテンションが必要なため、RNN のような一定サイズの隠れ状態に畳み込めず、系列長に対してメモリ・計算が増大します(系列長の二乗に比例)。学習時は系列を並列処理できる利点と、この想像時コストがトレードオフになります。

数値の絶対値は実装・タスクに依存しますが、論文の主張は明快です。「長期依存が支配的なタスクで Transformer 世界モデルは RNN 世界モデルを上回り、短期依存タスクでは利得が薄れる」。これが後続の Transformer 世界モデル研究(IRIS, TWM, STORM)の出発点になりました。

## メリット・トレードオフ・限界

メリット
- 長期依存・長期記憶に強い(遠い過去を直接参照できる)。
- 学習時に系列の並列計算が可能で学習が速くなりうる。
- Dreamer の確立された枠組み(想像内 RL)をほぼそのまま活かせる(差し替えが局所的で、既存知見を継承できる)。
- NLP 由来の Transformer スケーリング知見を流用できる。

トレードオフ・限界
- アテンションの計算・メモリが系列長の二乗で増え、長いロールアウトで重い。これが想像ホライズンの実用上限を縛ります。
- RNN のような固定サイズの要約状態に畳み込めず、過去系列の保持が必要(想像時のメモリ増)。RNN は隠れ状態1個で済むので、リアルタイム制御では RNN が有利な場面もあります。
- 短期依存中心のタスクでは RNN 版に対する優位が小さく、コストに見合わないことがある。
- 世界モデル型 RL 共通の課題(モデル誤差が方策に伝播、長期予測の誤差累積 [長期予測ホライズンと誤差累積](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0795_vista-temporal-horizon))は残る。
- 未解決の研究課題: 長期依存の強みを保ちつつ計算量を線形に抑える方法(状態空間モデル系の効率的アーキテクチャ、リカレント圧縮との組み合わせなど)が探索されています。

## 発展トピック・研究の最前線

- 効率的な長系列アーキテクチャ: Transformer の二乗コストを避けつつ長期依存を保つため、線形アテンションや、Mamba 系の状態空間モデル(選択的状態空間モデル、系列長に線形でかつ長期依存を扱える)を世界モデルに使う研究が出ています。「RNN の効率」と「Transformer の長期依存」を両立させる方向です。
- トークン化路線との統合: TSSM のような圧縮状態ベースと、IRIS のようなトークン化ベースを組み合わせ、長期記憶と鮮明な予測を同時に得る試み。
- 部分観測・記憶ベンチマークの整備: 「どれだけ長く何を覚えられるか」を厳密に測るベンチマーク(記憶容量・記憶期間を分離して評価する POPGym など)が整備され、世界モデルの記憶能力を診断できるようになっています。
- 自動運転との接点: 運転は「数秒〜数十秒前の状況(前方車の挙動、信号の変化、過去の他車の動き)を覚えておく」長期依存が本質的に重要です。世界モデルで運転環境を学び、その中で安全に方策を評価・改善する closed-loop シミュレーションを考える際、TransDreamer は「記憶をどう持つか」の設計選択肢として参照価値があります。死角に消えた歩行者を覚えておくような場面では、RNN より明示的な長期参照が効く可能性があります。
