# ConvGRUによる空間BEV履歴

## ひとことで言うと
過去の特徴マップ(自動運転では鳥瞰図 BEV の特徴)を1枚1枚捨てずに、ConvGRU という「空間構造を保ったまま記憶する小さな再帰ネットワーク」に溜め込み、マス目(セル)ごとに「ここの物体は前のフレームからどう動いたか」を捉えられるようにする仕組みです。グローバルベクトルを足すだけの軽い条件付けより重いぶん、ずっと細かい動きの情報を扱えます。

## 直感的な理解

1枚の静止画像(1フレーム)だけを見ても、物体の速度は分かりません。前方に車が写っていても、その瞬間の1枚からは止まっているのか時速60kmで走っているのか区別できないのです。速度を知るには、最低でも2フレーム、現実的には数フレームを比べて「位置がどれだけずれたか」を見る必要があります。これは人間でも同じで、ストロボのように一瞬だけ見せられた景色から相手の速さを当てるのは難しいですが、しばらく眺めていれば自然に分かります。

将来の状況を予測したいモデルにとって、この速度情報は本質的です。「いま隣の車が時速40kmで右に動いている」と分かれば「1秒後はここにいるはず」と予測できますが、速度が分からなければ予測のしようがありません。さらに加速度(速度の変化)まで読めれば、「ブレーキを踏み始めた」「加速して割り込んでくる」といった意図の手がかりも得られます。これらは原理的に複数フレームの比較からしか出てきません。

ここで素朴な方法として「過去を1本のグローバルベクトルに圧縮して足す」やり方があります(別項の per-channel バイアス)。これは「シーン全体がだいたいどんな動き方をしているか」は伝えられますが、「このマスにいる車だけが速い」といった位置ごとに違う動き(per-cell motion、セルごとの運動)は表現できません。圧縮の段階で「どこに」という空間情報が潰れてしまうからです。ConvGRU はこの限界を超えるために、過去を「空間の形を保ったまま」記憶し、セル単位で時間変化を追えるようにします。

## 基礎: 前提となる概念

- 再帰ニューラルネットワーク(RNN, Recurrent Neural Network): 時系列を1ステップずつ処理し、「隠れ状態(hidden state)」という記憶を持ち越しながら更新していくネットワークです。前のステップの記憶が次のステップに引き継がれるので、過去の文脈を踏まえた処理ができます。
- 隠れ状態(hidden state): RNN が持ち越す記憶です。普通の RNN ではこれが1本のベクトル(形 `[B, D]`)で、空間構造を持ちません。
- GRU (Gated Recurrent Unit、ゲート付き回帰ユニット): RNN の改良版で、「どれだけ覚え続け、どれだけ忘れ、どれだけ上書きするか」をゲート(門、0〜1の重み)で調整します。素朴な RNN が長い系列で勾配消失(学習信号が薄れて遠い過去を覚えられなくなる現象)を起こしやすいのに対し、ゲートによって記憶を選択的に残せます。LSTM の簡略版で、ゲートが2つ(更新ゲートとリセットゲート)とパラメータが少なく計算も軽いのが特徴です。
- 勾配消失(vanishing gradient): 誤差を逆方向へ伝える際、各ステップで小さい数を掛け続けると信号が指数的に小さくなり、遠い過去まで届かなくなる現象。ゲート機構はこれを和らげます。
- 畳み込み(convolution): 画像処理の基本操作で、近傍のマス目だけを見て小さなフィルタを掛ける処理です。空間的な局所構造(エッジ、テクスチャ、物体の一部)を捉えるのに向きます。全結合と違い、空間の位置関係を保て、パラメータ数も入力サイズに依存しません。
- BEV (Bird's Eye View、鳥瞰図): カメラ画像を処理して作る「上から見下ろした地図状の特徴マップ」です。形は `[B, C, H, W]`。自動運転で「どこに何があるか」を表すのに広く使われ、複数カメラの情報を1つの俯瞰平面に統合できる利点があります。
- ego frame(自車座標系): 自車を原点とした座標系。時間が経つと自車自身が動くので原点が動きます。

## 仕組みを詳しく

### GRU と ConvGRU の違い

ConvGRU (Convolutional GRU、畳み込み GRU)は、GRU の中の「掛け算(全結合)」をすべて「畳み込み」に置き換えたものです。これにより隠れ状態が1本のベクトルではなく「空間マップそのもの」(形 `[B, D, H, W]`)になります。つまり ConvGRU は「マップの形をした記憶」を持てます。

数式のイメージ(畳み込みを `*`、要素ごとの積を `⊙`、シグモイドを `σ` とする。`x_t` は時刻 t の入力マップ、`h_t` は隠れ状態マップ):

```
更新ゲート  z_t  = σ(W_z * x_t + U_z * h_{t-1})       どれだけ新情報で上書きするか
リセットゲート r_t = σ(W_r * x_t + U_r * h_{t-1})       過去をどれだけ参照するか
候補状態    h~_t = tanh(W * x_t + U * (r_t ⊙ h_{t-1}))  今ステップの新しい記憶候補
新しい記憶  h_t  = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ h~_t    古い記憶と候補をゲートで混合
```

ポイントは `W_z, U_z, W_r, U_r, W, U` がすべて畳み込みカーネルであることです。各マス目の記憶更新が「自分とご近所のマス」を見て行われるので、「このマスの値が前フレームから隣のマスへ移った」という局所的な動きを記憶の中で表現できます。ゲート `z_t, r_t` も空間マップなので、「画面のこの領域は更新が速く、あの領域は静的」という空間ごとに異なる時間ダイナミクスを学べます。`z_t` が1に近いセルは「ここは速く変わるので新情報で上書きしろ」、0に近いセルは「ここは変わらないので過去を維持しろ」を意味します。

全結合版 GRU だと隠れ状態を1点に潰すため、空間のどこで何が起きたかは消えます。畳み込み版は空間の格子をそのまま記憶として持ち越すので、「左前方のセル群が右下のセル群へ流れている」といった運動場(motion field)を内部に保持できるのが本質的な違いです。

### 履歴への適用

手順を分解します。

1. 各時刻 `t` で、カメラ画像から現在の特徴マップ `x_t`(形 `[B, C, H, W]`、例 `[4, 256, 450, 300]`)を作る。
2. ConvGRU に `x_t` を入力し、隠れ状態 `h_t`(形 `[B, D, H, W]`)を更新する。`h_t` は「これまでのマップの流れ」を空間的に保持した記憶。
3. この `h_t` を将来予測モジュールに渡し、セルごとの運動情報を使って将来マップ(または静的ワープでは説明できない動的残差 `Δ_dynamic`)をより正確に予測する。

グローバルベクトル方式が `[B, C]` という空間を潰した記憶だったのに対し、`h_t` は `[B, D, H, W]` と空間を保つので、「どのマスがどう動いているか」をそのまま下流に渡せます。

```
グローバルベクトル方式:  過去 → [B, C]            「全体としてどう動いているか」
ConvGRU 方式:           過去 → [B, D, H, W]       「各マスがどう動いているか」
```

### メモリの重さと圧縮

問題はメモリです。隠れ状態が空間マップ `[B, D, H, W]` で、しかも畳み込みの中間活性も空間サイズぶん発生するため、グローバルベクトル方式に比べ桁違いに重くなります。高解像度 BEV(例 450×300)では、将来予測だけでも VRAM を強く圧迫し(混合精度 bf16 でも batch 4 で 20GB 級になりうる)、ConvGRU を素直に足すと簡単に OOM(Out Of Memory、メモリ不足)します。さらに学習では時間方向に展開した全ステップの活性を逆伝播のために保持するため、系列長に比例してメモリが増えます。

緩和策としてチャネル方向の圧縮がよく使われます。`C = 256` を `D = 64` のように線形射影で絞ってから ConvGRU に通せば、記憶のメモリを 1/4 に減らせます。ほかに、空間解像度のダウンサンプリング(粗いグリッドで動きだけ追う)、勾配チェックポインティング(逆伝播時に活性を再計算してメモリと計算を交換する)、隠れ状態を時間方向に短く打ち切る truncated BPTT(Backpropagation Through Time、時間方向の逆伝播を数ステップで切る)などが定石です。いずれも「メモリを減らすほど表現の細かさが落ちる」というトレードオフを伴います。

## 手法の系譜と主要論文

- Shi et al., "Convolutional LSTM Network: A Machine Learning Approach for Precipitation Nowcasting" (NeurIPS 2015, arXiv:1506.04214)。LSTM の全結合を畳み込みに置き換えた ConvLSTM を提案し、降水ナウキャスト(レーダー画像から数十分後の降雨を予測)に適用しました。空間構造を保った時系列予測の原点的な論文です。動機は、気象のように「空間パターンが時間とともに動く」現象を、空間を潰さずに予測することでした。
- Ballas et al., "Delving Deeper into Convolutional Networks for Learning Video Representations" (ICLR 2016, arXiv:1511.06432)。GRU の全結合を畳み込みに置き換えた構造(GRU-RCN、いわゆる ConvGRU)を動画表現学習に導入しました。LSTM より軽い GRU を畳み込み化することで、行動認識でフレーム特徴の時間統合を効率化しました。複数階層の特徴に ConvGRU を適用する設計も提案しています。
- Hu et al., "FIERY: Future Instance Prediction in Bird's-Eye View from Surround Monocular Cameras" (ICCV 2021, arXiv:2104.10490)。サラウンドカメラから BEV を作り、その将来をインスタンス(物体)単位・セル単位で確率的に予測しました。動機は、自動運転では「どのセルにどの物体がいつ来るか」が衝突回避に直結するからです。過去フレームを現在の自車視点へ warp して揃えてから ConvGRU 系の時間統合を行う、空間時系列の再帰予測を BEV に持ち込んだ代表例です。
- Hu et al., "MILE: Model-Based Imitation Learning for Urban Driving" (NeurIPS 2022, arXiv:2210.07729)。潜在空間にワールドモデル(世界の動き方の内部モデル)を持ち、将来の BEV を再帰的に予測してから方策を学びました。画素ではなく潜在特徴の空間で将来を予測することで、意味のある変化だけを学べるという利点を示しました。
- Li et al., "BEVFormer: Learning Bird's-Eye-View Representation from Multi-Camera Images via Spatiotemporal Transformers" (ECCV 2022, arXiv:2203.17270)。再帰の代わりに temporal self-attention で過去 BEV を統合する潮流の代表。ConvGRU/ConvLSTM 系から Transformer 系時空間メモリへの移行点として系譜上重要です。

系譜を一言でまとめると、「畳み込みで空間を保ったまま時系列を扱う」アイデアが気象予測(ConvLSTM)で生まれ、動画認識(ConvGRU)で軽量化され、自動運転 BEV(FIERY, MILE)で将来予測に応用され、近年は Transformer の注意機構(BEVFormer)へ主役が移りつつある、という流れです。

## 論文の実験結果(定量データ)

- ConvLSTM (Shi et al., 2015): 降水ナウキャストで、全結合 LSTM や当時の運用アルゴリズム(ROVER)より高い予測スコア(critical success index などの降雨検出指標、雨と判定すべき領域をどれだけ当てたかを測る)を達成しました。「畳み込み化により空間相関を保つこと」が時系列予測精度に効く、という核心を最初に定量的に示しました。
- FIERY (Hu et al., 2021): nuScenes(都市運転のサラウンドカメラ・LiDAR データセット)で将来インスタンスセグメンテーションを評価しました。指標は Future Video Panoptic Quality (VPQ) と将来 Intersection-over-Union (IoU)。VPQ は「将来の各時刻で、物体の形と ID(同一物体の追跡)をどれだけ正しく当てたか」を測る指標です。FIERY は LiDAR を用いる従来手法と競合する、あるいは上回る将来予測精度をカメラのみで達成したと報告されました。アブレーションでは、過去フレームを時間方向に統合する再帰部を外すと将来 IoU が明確に低下し(報告では数ポイント規模)、時系列メモリの寄与が確認されています。
- MILE (Hu et al., 2022): CARLA(運転シミュレータ)の closed-loop 評価で、driving score(ルート完走度と違反の少なさを総合したスコア、満点に近いほど良い)を測りました。MILE は画素予測ベースの世界モデルや模倣学習ベースラインを上回る driving score を達成し、特に学習時に見ていない町(unseen town)への汎化で差が大きいと報告されました。潜在空間での再帰的な将来予測が、運転方策の質に直結することを示した結果です。
- BEVFormer (Li et al., 2022): nuScenes の3次元物体検出で、temporal self-attention により単一時刻ベースラインから NDS(nuScenes Detection Score)を数ポイント引き上げ、特に速度推定の誤差(mAVE)を大きく改善しました。これは「時間情報が速度推定に効く」という ConvGRU 系の主張を、別の時間統合手段でも追認した結果です。

これらに共通するのは、「空間構造を保ったまま過去を記憶する機構」を入れると将来予測系・速度推定系の指標が一貫して改善し、その機構を抜くアブレーションで性能が落ちる、という傾向です。速度・運動という単一フレームでは原理的に得られない情報を、空間メモリが供給していることの裏付けと言えます。

## メリット・トレードオフ・限界

メリット
- セルごとの運動(per-cell motion)を捉えられる。物体ごとの速度・加速度を空間位置に紐づけて表現できる。
- 空間構造を保ったまま記憶するので、将来予測モジュールに位置を伴った情報を渡せる。
- 単一フレームでは原理的に得られない速度情報を供給でき、将来予測の精度向上が期待できる。ゲートにより空間ごとに異なる時間ダイナミクス(動く領域と静止領域)を学べる。

トレードオフ・限界
- メモリと計算が重い。隠れ状態が空間マップ `[B, D, H, W]` で、高解像度ではすぐ OOM の危険がある。学習では時間展開ぶんの活性保持が効いてさらに重い。
- 緩和のためチャネル圧縮やダウンサンプリングをすると、そのぶん表現の細かさが落ちる。
- 再帰構造なので学習・推論ともに逐次性が出やすく、並列化しにくい。長期予測では誤差が累積する(自己回帰的に予測を積み重ねるほどズレが増える)。
- グローバルベクトル方式のように1行差し込むだけでは済まず、配線・初期化・メモリ対策のチューニングの手間が増える。
- 研究上の未解決課題: 動く自車の上で記憶を更新する場合、隠れ状態自体を自車運動に合わせて座標変換(warp)してから次フレームを統合すべきか、という設計問題があります。座標系を揃えないと「自車が動いたぶんの見かけの流れ」を物体の運動と取り違えます(別項の座標系整合と直結する論点)。

## 発展トピック・研究の最前線

- Transformer ベースの時空間メモリ: 近年は ConvGRU/ConvLSTM の代わりに、過去の BEV を temporal self-attention で統合する手法(BEVFormer, Li et al., ECCV 2022)が主流になりつつあります。再帰の逐次性を避けつつ長距離の時間依存を扱える反面、参照する過去フレーム数に応じてメモリ・計算が増える別の問題が出ます。
- 明示的な運動分解との併用: 将来予測を「既知の自車運動による静的ワープ + 学習された動的残差」に分けると、空間メモリは動的残差の予測だけに集中でき、学習が安定します(別項の座標系整合)。記憶が「他車の運動」だけを表現すればよくなり、自車運動由来の見かけの流れを学ばずに済みます。
- 世界モデルの潮流: 潜在空間で将来を再帰予測する世界モデル(MILE のほか、GAIA-1 など生成系)は、運転方策の学習基盤として研究が活発です。空間メモリはその中核部品の一つで、生成 Transformer や拡散モデルとの組み合わせも進んでいます。
- 隠れ状態のワープ統合: 自車運動でメモリそのものを毎フレーム座標変換してから新観測を融合する設計は、移動するエージェント上で空間メモリを正しく蓄積する鍵で、占有予測(occupancy forecasting)系の研究で発展しています。
