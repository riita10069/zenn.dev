# 周辺エージェント動き予測損失

## ひとことで言うと

自車の周りにいる他の車・歩行者・自転車(これらをまとめてエージェントと呼びます)が、これから数秒のあいだにどこへ動くかを予測させ、その予測が実際の動きとどれだけ合っていたかを測る損失です。「相手がどう動くか」を当てられると、自車の計画(プランニング)が安全になります。未来が一通りに決まらない(複数の可能性がある)ことを正しく扱うのがこの損失設計の核心で、ここを誤ると予測は「どの未来にも当たらない無難な平均」に潰れてしまいます。

## 直感的な理解

運転で一番怖いのは「他人の動き」です。前の車が急ブレーキするか、歩行者が飛び出すか、隣の車が車線変更してくるか。これらを予測せずに自車の軌道だけ決めると、相手が動いた瞬間に破綻します。人間のドライバーは無意識に「あの車は曲がりそうだ」「あの歩行者は渡りそうだ」と予測しながら運転しています。これを機械にやらせるのが動き予測(motion forecasting / trajectory prediction)です。

動き予測が難しい本質的な理由は、未来が一つに決まらないこと、すなわちマルチモーダル性(multimodality)です。交差点に近づく車は「直進」「右折」「左折」のどれもありえます。ここで素朴に「1本の正解軌道に近づけよ」と学習させると、ネットワークは三つの可能性の平均を取り、「どれでもない、中途半端に直進ぎみで少し曲がる線」を出してしまいます。これをモード平均化(mode averaging)あるいはモード崩壊と呼びます。平均軌道はどの実際の挙動にも当たらず、しかも物理的にありえない経路(車線をまたぐ斜め線、縁石を突っ切る線など)になりがちで、計画にとってむしろ有害です。だから良い動き予測は「複数の候補軌道」と「それぞれの起こりやすさ(確率)」を出す必要があります。これはちょうど、天気予報が「明日は雨70%、晴れ30%」と確率付きで複数の可能性を述べるのに似ています。「平均的な天気」を述べても役に立たないのと同じ理屈です。

E2E 自動運転では、この動き予測を補助タスク(本来の計画を助ける中間課題)として組み込みます。周辺エージェントの未来を予測する能力が育つと、自車計画モジュールが「相手の未来」を踏まえて安全な軌道を引けるようになります。

## 基礎: 前提となる概念

軌道(trajectory)は時間に沿った位置の列です。未来軌道は「これから T 秒間を一定刻み(例 0.5 秒)で表した (x, y) 点列」です。6秒先まで0.5秒刻みなら12点になります。過去の軌道(observation / history)は逆に「これまでの数秒の (x, y) 点列」で、これが予測の主要な手がかりになります。

エージェント(agent)は自車以外の交通参加者(車・歩行者・自転車・バス等)です。動き予測ではエージェントごとに過去の軌道を入力し、未来の軌道を出力します。

文脈(context)とは予測の手がかりになる周辺情報で、道路地図(車線の形・接続)、信号、他エージェントの位置・速度などです。道路は車の動きを強く制約する(車線に沿って動く)ので、地図情報は予測精度を大きく左右します。

ベクトル化表現(vectorized representation)とは、地図や軌道を画像(ラスタ)ではなく「点を結ぶ矢印(ベクトル)の集まり」として表す方法です。車線の中心線を折れ線(polyline)で表し、各線分をベクトルとして扱います。これにより細かい道路形状を潰さず、計算も軽くできます。対する旧来のラスタ化(地図を画像として描いて CNN に入れる)は、細い車線や遠方が解像度で潰れる弱点がありました。

グラフニューラルネットワーク(GNN)は、要素(ノード)と関係(エッジ)からなるグラフ上で情報を伝播させて学ぶネットワークです。エージェントと車線の相互作用を表現するのに向いています。近年は Transformer の注意機構が同じ役割を担うことが多くなっています。

## 仕組みを詳しく

動き予測損失は典型的に次の要素からなります。

要素1: 複数候補(K本)の軌道を出す。各エージェントについて K=6 本の未来軌道を予測します。tensor shape のイメージ:

```
予測軌道:   [エージェント数 A, モード数 K=6, 時間ステップ T=12, 座標2]
モード確率: [エージェント数 A, モード数 K=6]
（K本それぞれが「直進」「右折」などの可能性を担い、確率がその起こりやすさ）
```

要素2: 損失は二部構成です。

回帰損失(regression loss)。K本のうち「正解に一番近い1本」だけを選び(Winner-Takes-All、勝者総取り)、その軌道と正解軌道の点ごとの差を L1 や Huber 損失で測ります。なぜ最良の1本だけかというと、全候補を正解に近づけると前述のモード崩壊が起きるからです。「外した候補は罰しない」ことで、各候補が別々のモード(直進・右折・左折)に分かれて棲み分けます。これは k-means のクラスタ割り当てに似た発想で、各候補が異なる「未来のクラスタ」を担当するよう自然に分化します。

分類損失(classification loss)。正解に一番近かった候補の確率を上げ、他を下げる交差エントロピーです。これで「どの候補がどれくらい確からしいか」を学びます。計画はこの確率を使って「起こりやすい未来」を重視できます。

```
L_motion = L_reg( 最良候補 k*, 正解 ) + λ · L_cls( モード確率, k* が正解 )
   k* = argmin_k  Σ_t || 予測_{k,t} − 正解_t ||   （最終点距離 or 全点平均距離で選ぶ）
```

ここで k* は正解に最も近い候補の番号、L_reg はその候補だけの点ごと回帰誤差、L_cls は k* を正解ラベルとした分類損失、λ は二項のバランス係数です。

混合密度ネットワーク(MDN)的な別流派もあります。各候補を点列ではなくガウス分布の系列(各時刻に平均と分散)として出し、損失を負の対数尤度にする方式です。Trajectron++ や MultiPath はこの確率モデル寄りで、予測の不確実性(楕円の広がり)まで陽に表現できます。WTA は実装が単純で広く使われ、MDN は不確実性を計画に渡しやすい、というトレードオフです。

数値例。交差点で前方車について6本の候補(直進2・右折2・左折2)を出したとします。実際にはその車は右折しました。回帰損失は6本のうち右折に最も近い1本にだけ点ごと距離を計算し、分類損失はその右折候補の確率を上げます。これにより「直進・左折候補も将来用に残しつつ、今回は右折を当てた」という学習ができます。次に同じような場面で車が直進すれば、今度は直進候補が選ばれて磨かれます。こうして K 本がそれぞれ別のモードへ自然に分化します。

エージェントの動きと文脈をどう表現するかも損失の効きを左右します。VectorNet は車線や軌道を「ベクトルの集まり」として表し、まず各折れ線内をまとめる局所グラフ(subgraph)、次に折れ線同士をつなぐ大域グラフ(global interaction graph)という階層 GNN で相互作用を処理しました。この表現は後の損失(WTA + 分類)とよく噛み合い、軽量で高精度な予測を可能にします。

```
過去の軌道(観測) ─┐
道路地図(車線)   ─┼─→ [シーン符号化: GNN/Transformer] ─→ K=6本の未来候補 + 各確率
他エージェント    ─┘                              │
                                                  ▼
                            正解に最も近い1本を選ぶ(Winner-Takes-All)
                                                  │
                          回帰損失(その1本) + 分類損失(候補の確率) = 動き予測損失
```

## 手法の系譜と主要論文

動き予測の損失設計は、マルチモーダル性をどう扱うかで進化してきました。

MultiPath(Chai, Sapp, Bansal, Anguelov, CoRL 2019, arXiv:1910.05449, Waymo)。あらかじめ用意した代表軌道(アンカー軌道)への分類+各アンカーからの残差回帰でマルチモーダル性を表現しました。「離散モード(どのアンカーか)+連続残差(そのアンカーからのズレ)」という分解は、後続の損失設計に強く影響しました。アンカーを使うことでモード崩壊を構造的に避けられる一方、アンカー集合がデータ分布に合わないと性能が頭打ちになる弱点があります。

VectorNet(Gao, Sun, Zhao, Shen, Anguelov, Li, Schmid, CVPR 2020, arXiv:2005.04259, Waymo)。地図とエージェント軌道をベクトル化エンティティとして表現し、階層グラフネットで相互作用をモデル化しました。動機は「従来のラスタ画像方式は計算が重く細かい道路形状が潰れる」こと。軽量かつ高精度を実現し、ベクトル化表現の標準を作りました。補助タスクとして一部のノードをマスクして復元させる自己教師も提案しています。

LaneGCN(Liang, Yang, Hu, Chen, Casas, Urtasun, ECCV 2020, arXiv:2007.13732, Uber)。車線グラフを明示的に作り、車線間・車線とエージェント間・エージェント同士の四種の相互作用を畳み込みで扱いました。地図構造の活用を深めた系譜で、Argoverse で当時最高精度を達成しました。

Trajectron++(Salzmann, Ivanovic, Chakravarty, Pavone, ECCV 2020, arXiv:2001.03093)。未来を点列ではなく確率分布(条件付き変分オートエンコーダ CVAE による潜在変数モデル)として出す系統で、不確実性を陽に表現し、動的制約(車は横滑りしない等)も組み込めます。WTA とは別の「分布として学ぶ」流派の代表です。

UniAD MotionFormer(Hu et al. CVPR 2023, Best Paper, arXiv:2212.10156)。E2E 自動運転 UniAD の動き予測モジュールです。検出・追跡で得たエージェントクエリ、地図クエリ、自車クエリを Transformer の注意機構で相互作用させ(agent-agent, agent-map, agent-goal の三種の注意)、シーン全体を一度に予測します。各エージェントに複数モードの未来を出します。論文の主張は「動き予測を最終目的の計画と end-to-end でつなぐと、計画の安全性指標(衝突率など)が改善する」。従来の「知覚→予測→計画を別々に学ぶ」分業を、勾配でつないだ点が新規性です。

MTR / QCNet(2022–2023)。MTR(Motion Transformer, Shi et al. NeurIPS 2022)は「動きクエリ(意図ごとの学習可能クエリ)+ 局所注意 + 反復的な軌道精緻化」で長距離予測を改善し、Waymo Open Motion を更新しました。QCNet(Zhou et al. CVPR 2023)はクエリ中心・相対座標(各エージェントを原点に置くシーン中心ではない表現)で並進・回転不変性を得て、ストリーミング予測も可能にしました。これらが Waymo / Argoverse 2 の近年の主役です。

## 論文の実験結果(定量データ)

動き予測の標準ベンチマークは Argoverse / Argoverse 2、Waymo Open Motion、nuScenes です。指標を初心者向けに説明します。

minADE(minimum Average Displacement Error)は、K本の予測のうち最良の1本について、全時間ステップの予測点と正解点の平均距離(メートル)です。小さいほど良い。「予測の点列が全体としてどれだけ正解の線に沿っていたか」を測ります。K=6 で測ることが多く、minADE_6 のように書きます。

minFDE(minimum Final Displacement Error)は、最良候補の最終時刻の点と正解最終点の距離(メートル)です。小さいほど良い。「行き着く先(数秒後の到達点)をどれだけ当てたか」を測り、計画にとって特に重要です。

Miss Rate(MR、見逃し率)は、最良予測でも最終点が一定距離(例 2m)を超えて外れたサンプルの割合で、低いほど良い。「致命的に外した割合」を表します。

mAP / brier-minFDE(予測版)は、モード確率と当たり判定を組み合わせた総合スコアで、Waymo / Argoverse 2 で使われます。確率の校正(自信の付け方の正しさ)まで評価する点が単純な距離指標と違います。

VectorNet は、当時主流だったラスタ+CNN 方式に対し、おおむね 70% 少ない計算量でより良い minADE / minFDE を達成し、ベクトル化の有効性を示しました。MultiPath は単一モード予測に比べてマルチモーダル指標(minADE over K)を明確に改善し、「平均化を避けて複数候補を出す」ことの効果を定量化しました。MTR・QCNet 系は Waymo / Argoverse 2 のリーダーボードで minADE・minFDE・MR・brier-minFDE を相次いで更新しています(例えば Argoverse 2 で minFDE が 1.2m 台まで縮むなど、年々改善が続いています)。

UniAD では、動き予測の指標(minADE・minFDE・MR)で従来の分業パイプラインに匹敵しつつ、最終目的である計画の衝突率・L2 誤差が改善した点が核心です。論文のアブレーションは、動き予測モジュール(=動き予測損失)を外すと計画の衝突率が悪化することを示し、「周辺エージェントの未来を予測する能力が自車計画の安全性に直結する」ことを定量的に裏づけました。また Winner-Takes-All と多モード設計の重要性は複数論文で一貫しており、単一モード回帰に退化させるとモード崩壊で minADE が大きく劣化する(候補を1本に絞ると minADE が倍以上に悪化する報告もある)ことが繰り返し示されています。

## メリット・トレードオフ・限界

メリット。相手の未来を踏まえた安全な自車計画が可能になります。マルチモーダルに予測することで、急な右折や飛び出しといった低頻度だが危険な未来を取りこぼしにくくなります。検出・追跡で作った表現を再利用でき、E2E パイプラインに自然に組み込めます。確率付きの複数候補は、計画側で「最悪ケースに備える」リスク考慮にそのまま使えます。

トレードオフと限界。未来軌道の正解ラベルには長時間の追跡アノテーションが要り高コストです。候補数 K や予測時間 T を増やすと計算とメモリが増えます。上流(検出・追跡)の誤りがそのまま予測誤りに伝播します(誤差の連鎖、cascading error)。そして Winner-Takes-All の弱点として、選ばれなかった候補は学習信号を受け取らないため、まれにしか起きないモードがいつまでも下手なまま固定される(モード崩壊・候補の死、mode collapse / dead modes)という問題があります。さらに、距離指標(minADE)は最良1本だけを評価するので、確率の付け方が雑でもスコアが良く見える「評価のごまかし」が起き、brier 系指標や校正の議論につながります。

未解決の課題としては、長期予測(8秒以上)の急速な精度劣化、希少だが致命的な挙動(逆走・飛び出し)の予測、社会的相互作用(譲り合い・割り込み)の双方向性をどう表現するか、そして予測の不確実性を計画にどう安全側に伝えるか(過信が事故を生む)が挙げられます。

## 発展トピック・研究の最前線

近年は予測と計画の境界が溶けつつあります。自車の計画意図を条件にして周辺の反応を予測する条件付き予測(conditional / interactive prediction)、ゲーム理論的に相互作用を解く手法、拡散モデル(diffusion)で多様な軌道を生成する流派(MotionDiffuser など)が現れています。表現面では、エージェント中心の相対座標で並進・回転不変性と汎化を高める QCNet 系、目標点(goal)提案+精緻化の MTR 系が主流です。損失面では WTA の死モード問題への対処(上位数本を罰する soft 割り当て、EM 風の割り当て)、不確実性を陽に出す分布損失、そして「予測の良さ」と「計画の安全さ」を直接結ぶ計画認識型の予測損失(downstream の計画性能を見て予測を学ぶ、PiP / DIPP 系)が活発な研究領域です。共同予測(joint prediction、複数エージェントの未来を相関を持たせて同時にサンプリングする)は、エージェント間の整合した未来を出せる点で計画に有利とされ、Waymo Interaction Prediction などで重視されています。
