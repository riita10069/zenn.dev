# 車線検出

## ひとことで言うと
車線検出(lane detection)とは、カメラ画像や BEV(Bird's Eye View、鳥瞰図、つまり真上から見下ろした地図のような視点のことです)から、道路上の白線・黄線や走行できるレーンの形を機械に見つけさせる知覚(perception、センサ情報から世界を理解するタスク)です。「ここがあなたの走るべきレーンですよ」という線を、ピクセルの集合や点の列、あるいは数式(曲線)として出力します。車を道路上に正しく保つための、最も基礎的な環境認識の一つです。

## 直感的な理解
人間が運転するとき、無意識に「自分の車線の左右の白線」を見て、その内側に車を保ち続けています。自動運転でも同じで、車がどこを走るべきかを決めるには、まず車線がどこにあるかを知る必要があります。具体的に車線情報が要る場面は、レーンキープ(車線中央を保って走る)、車線変更(隣のレーンが空いているか、どこへ移ればよいか)、交差点やカーブでの経路選択(分岐するレーンのどれをたどるか)などです。

車線検出が他の知覚タスク(物体検出など)と違って厄介なのは、出力すべき対象の「個数が一定しない」ことです。画像分類なら出力は常にクラス1個ですが、車線は場所によって2本だったり5本だったりします。さらに、線がかすれて消えかかっている、雪や落ち葉で隠れている、夜間で照明が乏しい、前の車に隠れている、カーブで大きく曲がり遠くほど細く写る、といった見た目の変動が極めて大きい。「細長く、薄く、本数が変わり、強い文脈推論(見えない部分を周囲から推測すること)が要る」という性質が、車線検出を独立した研究分野にしています。

たとえると、虫食いだらけの点線をつなぐパズルです。一部しか見えなくても「ここは1本の連続した線だ」と人間は補完できますが、機械にこの「連続性の常識」を持たせるのが難しいのです。

## 基礎: 前提となる概念
セマンティックセグメンテーション(semantic segmentation)とは、画像の各ピクセルに「これは車線」「これは路面」「これは空」のようにクラスのラベルを付けるタスクです。出力は入力画像と同じ大きさのラベルマップになります。車線検出の古典的アプローチはこの枠組みを使います。

回帰(regression)とは、連続値(座標や係数のような数値)を直接予測することです。分類が「どのクラスか」を当てるのに対し、回帰は「いくつか」を当てます。車線を点の座標や多項式の係数として出すのは回帰です。

anchor(アンカー)とは、あらかじめ用意した「候補の形」のことです。物体検出では候補の枠(box anchor)、車線検出では候補の直線(line anchor)を多数置き、各候補を実際の車線に合うよう微調整(refine)します。ゼロから線を描くより、近い候補を選んで直すほうが学習が安定するという発想です。

IPM(Inverse Perspective Mapping、逆透視変換)も押さえます。カメラ画像は遠近法で奥が縮んで写りますが、路面が平らだと仮定すれば、画像を真上から見た BEV に幾何変換できます。BEV にすると平行な車線が画像上でも平行に写り、距離も正しくなるため車線が扱いやすくなります。ただし「路面は平ら」という仮定が坂道で崩れるのが弱点です。

評価指標も押さえます。車線検出でよく使う F1 スコアは、予測した車線が正解とどれだけ一致したかを測る指標で、precision(適合率、予測した車線のうち正しかった割合)と recall(再現率、正解の車線のうち当てられた割合)の調和平均です。調和平均なので、片方だけ高くても F1 は伸びず、両方をバランスよく上げる必要があります。一致判定には、予測線と正解線の重なり(IoU、Intersection over Union、共通面積÷和集合面積)が一定以上かを使います。F1 は 0〜100% で、高いほど良い。

## 仕組みを詳しく
車線をどう「数値として」表すかという出力表現に、代表的な3流派があります。tensor の形を添えて説明します。

1. セグメンテーション方式(各ピクセルを分類する)。入力画像が高さ288、幅800、RGB 3チャンネルなら入力 tensor は [3, 288, 800]。出力は各ピクセルが「車線か否か」を表すマップ [288, 800] で、車線部分が1、それ以外が0です。車線ごとに区別したいときは [車線数+背景, 288, 800] のように層(クラスチャネル)を持たせます。長所は形が自由でどんな曲がり方も表現できること、短所は後処理(ピクセルの塊を1本の線につなぐクラスタリングや曲線フィッティング)が必要なことです。

2. 点列方式(各車線をポイントの列で表す)。1本の車線を、画像の各行ごとに「その行で車線が横方向のどこにあるか」を示す x 座標の並びで表します。縦に18本の代表行を取り、各行の x 座標を出すなら1車線あたり18個の数値、車線が最大4本なら出力 tensor は [4, 18]。row-wise classification(各行で車線がどの列セルにあるかを分類する)という変種もあり、これは回帰より学習が安定し高速です。横方向を W 個のセルに離散化し「どのセルか + 車線が無い」を分類する形にします。

3. 多項式・曲線方式(車線を数式で表す)。車線の形を x = a·y³ + b·y² + c·y + d のような3次多項式で近似し(画像では縦 y を変数に取ると曲線が表しやすい)、係数 a, b, c, d の4数値だけを出力します。1車線がわずか数個の数値で表せるので非常にコンパクトで、tensor は [車線数, 係数の数] と極小。短所は、複雑にうねる線(工事区間の急な蛇行など)を低次多項式では表しきれないことです。

数値例で違いを実感します。1枚の画像で4本の車線を表すとき、セグメンテーションなら 288×800 ≈ 23万個のピクセル値、点列なら 4×18 = 72 個、多項式なら 4×4 = 16 個の数値で済みます。出力の桁数がそのまま「表現力」と「後処理の重さ」のトレードオフに対応しています。

## 手法の系譜と主要論文
SCNN(Spatial CNN、Pan ら、AAAI 2018)は、セグメンテーション方式の代表で、特徴マップ上で行方向・列方向に情報を順に伝播させる畳み込みを提案しました。動機は、車線が細長く連続した構造を持つため、通常の畳み込み(局所しか見ない)では遠く離れた部分の文脈をつなげにくいこと。隣の行へ特徴を順次足していくことで、途切れた線や前の車に隠れた線を文脈で補完しました。同時に大規模データセット CULane を公開し、この分野のベンチマークを整備した功績も大きいです。

LaneNet(Neven ら、IV 2018)は、セグメンテーションを2枝に分けました。1つは「ここは車線か否か」を出す枝(binary segmentation)、もう1つは各ピクセルに埋め込みベクトル(embedding、意味的な座標のようなもの)を割り当てる枝です。同じ車線のピクセルは埋め込みが近く、違う車線は遠くなるよう discriminative loss で学習し、後でクラスタリングして車線を1本ずつ分離します。動機は、単純なクラス固定セグメンテーションでは車線本数を事前固定する必要があり、本数可変の現実に対応できないこと。本数可変に対応できた一方、クラスタリングという後処理が残るのがトレードオフでした。

UFLD(Ultra Fast Lane Detection、Qin ら、ECCV 2020)は、ピクセル単位のセグメンテーションをやめ、各代表行で「車線がどの列セルにあるか」を分類する row-wise 方式にしました。処理を行単位の軽い分類に落とすことで、報告で 300 FPS 超(1秒間に300枚以上を処理)という極めて高速な推論を実現しました。さらに「行方向の分類は大域的に見渡せるので、局所しか見ないセグメンテーションより文脈に強い」という利点も主張しました。速度と精度のバランスを大きく変えた一手です。

PolyLaneNet(Tabelini ら、ICPR 2020)は多項式回帰方式の先駆けで、CNN で画像から各車線の多項式係数を直接出します。シンプルで高速ですが、後発の Transformer 系より精度面で見劣りすると報告されています。

LSTR(Lane Shape Prediction with Transformers、Liu ら、WACV 2021)は、Transformer(系列を注意機構=各要素が他の要素を参照する仕組みで処理するネットワーク)を使い、車線を多項式のパラメータとして直接回帰しました。画像から「車線は何本で、それぞれの係数はいくつか」を一気に出力する end-to-end(端から端まで1つのネットで完結する)方式で、後処理のクラスタリングを廃したのが大きな効果です。DETR(物体検出を集合予測として解く Transformer)の発想を車線に持ち込んだ位置づけです。一方、多項式表現ゆえ非常に入り組んだ形状の表現力には限界があります。

CLRNet(Cross Layer Refinement Network、Zheng ら、CVPR 2022)は、line anchor を粗い位置から細かい位置へ段階的に refine していく方式です。低解像度の高レベル特徴(意味が濃いが位置が粗い)で大まかな位置を決め、高解像度の低レベル特徴(位置は細かいが意味が薄い)で細部を詰める、という多層をまたいだ修正(cross-layer refinement)を行います。さらに ROIGather で各車線候補の局所特徴と画像全体の大域特徴を統合し、新しい line IoU loss で線同士の重なりを直接最適化します。動機は、車線が遠くの薄い手がかりと全体の文脈の両方を見ないと当てられないこと。複数ベンチマークで当時の SOTA(state-of-the-art、最高精度)を達成しました。トレードオフは anchor 設計やマルチステージ構造による実装の複雑さです。

3D 化の系譜も重要です。3D-LaneNet(Garnett ら、ICCV 2019)は、2D 画像から直接 3D の車線(高さを持つ)を end-to-end で推定する先駆けで、平面仮定に頼る IPM の限界を超えようとしました。PersFormer(Chen ら、ECCV 2022)は Transformer で透視ビューと BEV を結ぶ attention を学習し、大規模 3D 車線データセット OpenLane を整備しました。坂やうねりのある実道路では 2D だけでは不十分だ、という問題意識がこの系譜を生みました。

## 論文の実験結果(定量データ)
ベンチマークの中心は CULane と TuSimple です。CULane は夜間・混雑・影・曲線・矢印標示など9つの難条件シナリオを含む大規模データセット(約13万枚)で、評価は F1 スコア(線を幅30ピクセルに太らせ IoU 0.5 以上を一致とみなす)。TuSimple は高速道路中心で比較的素直なデータセットで、accuracy(各行で予測 x 座標が正解の許容範囲に入った点の割合)を主指標とし、誤検出率(FP)と見逃し率(FN)も併記します。

CLRNet 論文は、CULane で当時の最高 F1 を報告し、バックボーンに ResNet や DLA を使った構成でおよそ 79〜80% 台の F1 を達成しました。これは SCNN(報告でおよそ 71% 台)を約8ポイント、初期の anchor 系手法を数ポイント上回る水準です。F1 が数ポイント上がることは、夜間や混雑のような難条件で見逃し・誤検出が目に見えて減ることを意味し、安全に直結します。CLRNet のアブレーションでは、cross-layer refinement を外すと F1 が明確に下がること、line IoU loss を通常の距離損失に戻すと精度が落ちることが示され、各部品の寄与が定量化されています。

UFLD は、TuSimple で 95% 台の accuracy を保ちつつ、ResNet-18 ベースで報告 300 FPS 超という速度を示しました。これは「行単位分類に落とすと、精度をほとんど落とさず一桁速くできる」というアブレーション的な知見を与えます。LSTR は Transformer による end-to-end 化でクラスタリング後処理を不要にしつつ、TuSimple で 96% 台の accuracy と高速推論(報告で 400 FPS 超)を両立しました。PolyLaneNet も TuSimple で 90% 台前半の accuracy を高速に出しますが、CULane のような難条件では多項式表現の限界から後発手法に劣ると報告されています。

これらを総合すると、出力表現の選択が「精度・速度・後処理の有無」を直接決めることが定量的に見て取れます。セグメンテーション系は柔軟だが後処理が重く、row-wise/多項式系は軽量だが表現力に上限があり、anchor + refinement 系は精度が高いが実装が複雑、という綱引きです。3D 系では、OpenLane 上で 2D 手法を 3D に素朴に持ち上げた場合より、PersFormer のように BEV を陽に扱う手法が高さ方向の精度で明確に優れることが報告されています。

## メリット・トレードオフ・限界
メリットは、車線という人間にも理解できる中間表現が得られ、挙動を説明・デバッグしやすいこと、レーンキープや車線変更の判断に直接使えることです。誤りが起きたとき「どの車線をどう間違えたか」を可視化でき、検証(validation)しやすい点も実用上大きい利点です。

トレードオフは、出力表現ごとに一長一短があること(前述)です。さらに、明示的な車線検出を中間表現として挟む設計(モジュラー型パイプライン、知覚→地図→計画と段階を分ける従来型)では、車線という固定表現が性能の天井になる場合があります。検出器が見落とした車線の情報は下流に伝わらないからです。これに対し end-to-end(画像から運転行動を直接学ぶ)設計では、車線を明示的に持たず、auxiliary task(補助タスク、本来の目的とは別に学習中だけ解かせる副次タスク)として学ばせて特徴抽出器に良い手がかりを与えるだけにとどめる選択肢があります。

限界として、2D 画像上の車線検出は、勾配のある坂や3次元的なレーン形状を平面と誤認しやすい問題があります(IPM の平面仮定の崩れ)。これに対応するのが OpenLane などの 3D 車線データセットで、平面でなく高さ方向も持つ車線を扱います。また、地図に車線情報がある領域では、検出に頼らず高精度地図(HD map)を参照する設計もあり、検出と地図のどちらに頼るかは設計判断です。研究の未解決課題には、悪天候・夜間でのロバスト性、地図無し領域での 3D 車線推定、ドメインギャップ(学習した地域と異なる国や標示規格での性能低下)、検出結果の不確実性(自信のなさ)をどう下流の計画に伝えるか、などがあります。

## 発展トピック・研究の最前線
近年は BEV 上での車線・走行可能領域の認識が主流化しています。複数カメラの画像を BEV へ投影し、その鳥瞰図上で車線をセグメンテーションする方式(BEV segmentation)は、複数カメラをまたいだ一貫した車線地図を直接得られる利点があります。画像を個別に処理して後で貼り合わせるより、最初から共通座標で扱うほうが整合性が高いのです。

さらに、車線を含む地図要素(横断歩道、停止線、道路境界など)をベクトル(点の連なり)として直接予測する online HD map construction(VectorMapNet, MapTR などの系譜、2022〜)が活発で、車線検出と地図生成の境界が溶けつつあります。これらは検出から「運転に使える構造化地図の生成」へと問題設定を押し広げる流れです。MapTR は地図要素を順序付き点集合として階層的に表現し、リアルタイムでベクトル地図を組み立てます。最前線では、こうしたオンライン地図を時系列で蓄積・更新する手法、地図とプランニングを一体で学ぶ end-to-end 運転モデル(車線を陽に出さず内部表現として持つ)へと研究が進んでいます。
