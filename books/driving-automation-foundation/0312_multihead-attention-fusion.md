# Multi-Head Attention融合

## ひとことで言うと
Multi-Head Attention(マルチヘッドアテンション、多頭注意)とは、「ある情報が別の情報のどこにどれだけ注目すべきか」を計算する仕組み(アテンション)を、複数の独立した視点(ヘッド)で並列に行い、結果をまとめる方法です。異なる種類のカメラ特徴・地図特徴・センサ特徴を融合する場面で、各位置が相手のどこを参照するかを内容に応じて動的に決められる、表現力の高い融合手段です。

## 直感的な理解
会議で発言を理解する場面を想像してください。ある一言を理解するために、あなたは過去の発言のうち関係する部分に注目します。注目先は固定ではなく、今の発言の内容によって変わります。さらに、あなたは同時に複数の観点で聞いています。「話の論理は通っているか」「感情はどうか」「以前の主張と矛盾しないか」。これらは別々の注意の払い方で、1つにまとめると粗くなります。Multi-Head Attention はこの「内容に応じて注目先を変える」「複数の観点を並列に持つ」という人間の聞き方をそのまま機械に与えたものです。

なぜこれが要るのかというと、特徴どうしを混ぜるとき、単純な足し算や連結([Early Fusion vs Late Fusion](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0331_early-vs-late-fusion))は「位置を固定で対応づける」混ぜ方だからです。しかし実際には「この位置に最も関係する相手側の情報はどこか」が場所ごとに変わります。例えばカメラ由来の鳥瞰図(BEV)の「ここに何か物体らしきものが見える」というマスにとって有用な地図情報は、必ずしも真上の同じマスとは限りません。キャリブレーション誤差や、少し離れた車線境界が効くこともあります。この動的な対応づけを実現するのがアテンションです。

## 基礎: 前提となる概念
アテンションは Query(クエリ、質問役)、Key(キー、見出し役)、Value(バリュー、中身役)の3つで動きます。図書館で本を探すたとえがよく使われます。あなたの探したい主題がクエリ、各本の背表紙のタイトルがキー、本の中身がバリューです。クエリとキーの一致度で「どの本をどれだけ読むか」を決め、その重みで中身を混ぜて読む、というイメージです。

処理は3段です。クエリ Q とキー K の内積(似ているほど大きい値)を取り「Q が K にどれだけ注目すべきか」のスコアを作ります。スコアを softmax で正規化して合計1の重み(注目度)にします。その重みで Value V の加重平均を取ります。これが出力です。式で書くと `Attention(Q,K,V) = softmax(QKᵀ / √d) · V` です。ここで Q,K,V は行列で、各行が1つのトークン(BEV のマスなど)のベクトル、d は1ベクトルの次元です。softmax とは複数の数値を「合計1の確率分布」に変換する関数で、大きい値ほど大きい重みになります。√d で割るのは、次元 d が大きいと内積が大きくなりすぎて softmax が一点に尖りすぎ、勾配が消えるのを防ぐスケーリングです。例えば d=64 のとき、内積の典型的な大きさは √64=8 倍程度に膨らむので、これで割って正規化します。

self-attention は Q,K,V すべてを同じ入力から取り「自分自身の中で関係を見る」使い方です。cross-attention(クロスアテンション)は Q を片方の特徴から、K と V をもう片方の特徴から取り「相手を見る」使い方です。異なるソースを融合する場面ではクロスアテンションを使い、片方をクエリ、もう片方をキー・バリューにします。「クエリ側が主役で、キー・バリュー側から欲しい情報を引いてくる」という非対称な関係になります。

## 仕組みを詳しく
マルチヘッド化の意味を説明します。1つのアテンションだけだと「車線の位置を見たい」「交差点かどうかを見たい」「進行方向の連続性を見たい」といった複数の見たい関係性を1つの重みに混ぜてしまい、表現力が落ちます。そこで埋め込み次元(例 256)を複数のヘッド(例 8個)に分け、各ヘッドを 32 次元(256÷8)にします。各ヘッドが独立に Q,K,V アテンションを行い、8個の出力(各32次元)を最後に連結して 256 次元に戻し、出力用の線形層を通します。8個のヘッドが別々の関係性(別々の部分空間)を学習できるのが利点です。各ヘッドは元の埋め込みを学習可能な線形層で別々の 32 次元部分空間に射影してから注目するので、互いに異なる「見方」を担当できます。

クロスアテンション融合を tensor shape で追います。入力は2枚の BEV、片方を「主特徴 BEV」、もう片方を「補助特徴 BEV」とします(どちらも [B, 256, H, W])。

```
main_bev [B,256,H,W] ─permute+reshape→ q  [B, H*W, 256]   (クエリ)
aux_bev  [B,256,H,W] ─permute+reshape→ kv [B, H*W, 256]   (キー兼バリュー)

q  = LayerNorm(q)        # pre-norm: 安定化のため先に正規化
kv = LayerNorm(kv)
attn_out = MultiheadAttention(num_heads=8)(q, kv, kv)   # 8ヘッドのクロスアテンション
q = q + attn_out         # 残差接続(元のqを足し戻す)
q = q + FFN(LayerNorm(q))# FFN(256->512->256, GELU)+残差

出力: q を [B,256,H,W] に戻す
```

各構成要素を平易に説明します。permute+reshape は [B,256,H,W] の特徴マップを「H*W 個の 256 次元トークン列」に並べ替える操作で、空間グリッドをトークンの系列とみなすことでアテンションにかけられるようにします。LayerNorm(層正規化)は各ベクトルを平均0・分散1に整える前処理で、学習を安定させます。pre-norm はアテンションの前に正規化する流儀で、深いネットでも勾配が安定しやすい(post-norm より学習率を上げやすい)ことが知られています。残差接続(q = q + attn_out)はアテンションの結果を元の特徴に足す操作で、アテンションが有用な情報を見つけられなくても元の特徴がそのまま残るため学習初期から壊れにくく、勾配も深層まで流れます。FFN(Feed-Forward Network、位置ごとの全結合)は各マスを独立に 256→512→256 と変換して非線形性を加えます。アテンションが「マス間の混合」、FFN が「各マス内の加工」を担うという分業です。

計算量に注意が要ります。H*W 個のクエリそれぞれが H*W 個の相手トークン全部を見るので、計算量とメモリは (H*W)² に比例します。例えば BEV が 50×50 = 2500 トークンなら 2500² = 625万、100×100 = 10000 トークンなら 10000² = 1億で、解像度を2倍にするとコストは16倍に膨れます。これがアテンション融合の主なコスト要因であり、大きな BEV では深刻になります。とくに学習時はこの (H*W)² サイズの注目行列を中間結果として保持するため、メモリが GPU の限界に達しやすいのが実務上の最大の壁です。

なぜこの設計かというと、固定対応づけ(連結・加算)では場所ごとに変わる最適な参照先を表現できず、単一ヘッドでは複数種類の関係を同時に学べないからです。マルチヘッド+残差+pre-norm+FFN という構成は、表現力と学習安定性を両立させる Transformer ブロックの標準形として広く検証されてきました。

## 手法の系譜と主要論文
Transformer (Ashish Vaswani, Noam Shazeer, Niki Parmar, Jakob Uszkoreit, Llion Jones, Aidan Gomez, Łukasz Kaiser, Illia Polosukhin、NeurIPS 2017, arXiv:1706.03762)。Scaled Dot-Product Attention と Multi-Head Attention、それに残差接続・LayerNorm・FFN を組んだアーキテクチャを提案しました。動機は、当時主流の RNN が系列を順番に処理するため遅く、遠い要素間の関係も伝わりにくかったことです。アテンションなら全要素を一度に見て任意の要素間の関係を直接取れ、並列計算もしやすくなります。マルチヘッドは複数の異なる関係性を同時に学ぶための工夫です。

DETR (Nicolas Carion ら、ECCV 2020, arXiv:2005.12872)。物体検出にクロスアテンションを持ち込みました。学習可能なオブジェクトクエリが画像特徴を参照して物体を直接予測する構図で、「クエリが別ソースの特徴を参照する」という融合パターンの代表例です。これにより手設計の後処理(非最大抑制など)を排した end-to-end 検出が可能になりました。

Deformable DETR (Xizhou Zhu ら、ICLR 2021, arXiv:2010.04159)。DETR の弱点だった収束の遅さと高解像度での計算コストを、全トークンでなく少数の参照点だけを見る「変形可能アテンション」で解決しました。二乗コストを回避する代表的な手法で、後の BEV 知覚(BEVFormer など)の基盤になりました。

BEVFusion (Zhijian Liu ら、ICRA 2023, arXiv:2205.13542 / Tingting Liang ら、NeurIPS 2022, arXiv:2205.13790)。カメラ BEV と LiDAR BEV を共通の BEV 空間で統合する研究群です(同名で異なるチームの論文が2本ある)。「ある BEV に別ソースの BEV を融合する」という構図はこの系譜に位置づけられ、融合の場として BEV 空間を使う設計が標準化しました。BEV 空間で混ぜることで、カメラと LiDAR のように性質の異なるセンサを共通座標で扱えます。

比較として、より軽量なゲート融合([学習ゲーティング融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0307_learnable-gating-fusion))は (H*W)² のコストを避けますが、場所ごとの動的な対応づけはできません。アテンション融合とゲート融合は表現力と計算コストのトレードオフの両端にあり、用途に応じて使い分けます。

## 論文の実験結果(定量データ)
Transformer の原論文は機械翻訳ベンチマーク WMT 2014 の英独・英仏翻訳で評価され、BLEU(翻訳の良さを 0〜100 で測る指標で、参照訳との n-gram 一致度が高いほど良い)で当時の最高性能を更新しました。報告では英独で BLEU 28.4 を達成し、それまでの最良手法(アンサンブル含む)を上回りました。英仏では BLEU 41.8 を、しかも当時の最良モデルの数分の1の学習コストで達成しています。これは全要素を並列処理できるアテンションの効率を示します。

アブレーションとして、原論文はヘッド数を変える実験を行いました。単一ヘッドだと8ヘッド構成より BLEU が約0.9ポイント低下し、逆にヘッドを増やしすぎても(各ヘッドの次元が小さくなりすぎて)精度が頭打ちまたは低下することが示されました。これはマルチヘッドが「複数の関係性を分担する」という設計意図どおりに効いていること、かつ最適なヘッド数が存在することを意味します。同論文は √d のスケーリングを外すと学習が不安定になることも示しており、各設計要素が実証的に裏付けられています。

融合タスクでの定量例としては、BEVFusion が nuScenes の 3D 物体検出で、カメラ単独や LiDAR 単独より大きく mAP(mean Average Precision、複数クラス・複数閾値で平均した検出精度)と NDS(nuScenes Detection Score、検出精度に加え位置・速度・属性の誤差も総合した nuScenes 独自の指標)を向上させたことが挙げられます。報告ではカメラと LiDAR を BEV 空間で融合することで mAP が単一センサ比で数ポイントから十数ポイント向上し、悪条件(夜間・雨)での頑健性も改善しました。とくに片方のセンサが苦手な状況(カメラは暗所、LiDAR は遠方の小物体)を他方が補えることが、融合の本質的な利点として定量的に示されています。数値の意味として、検出 mAP の数ポイントは見落としや誤検出の確率に直結し、安全マージンに影響する実用上重要な差です。

## メリット・トレードオフ・限界
メリットは、場所ごとに「相手のどこを参照するか」を内容に応じて動的に決められること(固定対応づけより柔軟)、複数ヘッドで複数種類の関係性を並列に学習でき表現力が高いこと、残差・pre-norm・FFN で学習が安定しやすいことです。クロスアテンションは入力ソースの空間サイズが違っても(クエリ数とキー数が違っても)扱えるため、解像度やモダリティの異なる特徴の橋渡しに使いやすい利点もあります。

トレードオフと限界として、計算・メモリが系列長(H*W)の2乗に比例し BEV が大きいと重くなります。パラメータが多くデータが少ないと過学習しやすいこと、そして軽量なゲート融合が持つ「初期は相手を無視し徐々に取り込む」ような安定性は持たず初期から相手特徴に強く影響される(初期化や学習率に敏感)ことも挙げられます。アテンションは順序情報を持たないため、空間グリッドをトークン化する際には位置エンコーディング(各位置がどこかを示す追加情報)を与えないと「どのマスがどこか」を区別できない点にも注意が要ります。研究上の未解決課題は、二乗コストを下げる効率的アテンション(線形アテンション、変形可能アテンション、局所窓アテンションなど)をどう融合に最適化するか、そしてアテンションが何を見ているかの解釈性・検証性をどう担保するかです。

## 発展トピック・研究の最前線
アテンション融合の最前線は、効率化と適用範囲の拡大の2方向です。効率化では、変形可能アテンション(Deformable DETR、BEVFormer)が全トークンでなく少数の参照点だけを見て二乗コストを回避し、幾何事前知識で参照範囲を絞る手法([Geometry-guided Kernel Transformer (GKT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0333_gkt-geometry-kernel-transformer))も同じ系統です。線形アテンション(softmax を近似してコストを系列長に線形化)や FlashAttention(Tri Dao ら、NeurIPS 2022, arXiv:2205.14135、注目行列を中間保存せず GPU メモリ階層を意識して計算する正確な実装)のようなメモリ効率実装も大きな BEV を扱う鍵になっています。適用範囲では、時系列の過去フレームをクロスアテンションで取り込む時空間融合、複数センサ(カメラ・LiDAR・レーダ・地図)を共通空間で混ぜるマルチモーダル融合、さらに知覚から予測・計画までを単一の Transformer で end-to-end に貫く設計が進んでいます。アテンションは「異種情報を内容に応じて結びつける」汎用的な接着剤として、知覚パイプラインの中心であり続けています。
