# 地図特徴の残差融合 (ゲート付き残差注入)

## ひとことで言うと

カメラ映像から作った「シーンの鳥瞰図特徴」に、地図から作った「地図の鳥瞰図特徴」を、学習で調整できる重みをかけて足し込む仕組みです。重みをゼロから始めることで、学習初期は地図を完全に無視して画像だけで普通に学習し、地図が役に立つと分かった分だけ少しずつ取り込みます。新しい情報源(地図)を既存のネットワークに「壊さずに安全に注入する」ための、汎用的で軽量な融合パターンです。

## 直感的な理解

すでにそれなりに運転できるモデルに、後から地図という新しい入力を足したいとします。素朴に二つの特徴を混ぜると、地図側のネットワークがまだ何も学習できていない初期段階では、地図特徴はただのランダムなノイズです。それを強く混ぜると、せっかく学習が進んでいる画像側まで巻き込んで学習全体が不安定になります。

人間に例えるなら、運転に慣れた人へ「この新しいカーナビも参考にして」と渡したとき、最初は信用せず自分の目で運転し、ナビが正確だと分かった範囲だけ徐々に頼るようになる、という賢い態度です。ゲート付き残差融合は、この「最初は信じない、有用さが分かった分だけ取り込む」をパラメータの初期化一つで自動的に実現します。鍵は「足す量をゼロから始める」という一点です。足す量がゼロなら入力がそのまま素通りするので、地図ブランチが付いていても付いていないのと同じ挙動になり、何も壊しません。

## 基礎: 前提となる概念

BEV 特徴(Bird's-Eye-View): 真上から見下ろした視点の格子状特徴マップです。形状は (B, C, H, W) の4次元 tensor で、B はバッチサイズ、C はチャンネル数(各マスが持つ特徴ベクトルの次元、例 256)、H×W は鳥瞰格子の縦横(例 200×100)。カメラから作る BEV を「シーン BEV(画像 BEV)」、地図から作る BEV を「地図 BEV」と呼び分けます。

残差接続(residual connection): 画像認識の ResNet(He et al., CVPR 2016)で有名になった構造で、出力を `y = x + F(x)` と書きます。入力 x をそのまま通す経路(恒等写像、identity)を確保し、ネットワークが学ぶのは「x からの変化分 F(x)」だけにします。これにより深いネットワークでも勾配(学習信号)が末端まで届きやすくなり、勾配消失という学習困難が解消されました。

ブロードキャスト(broadcast): 形の違う tensor 同士の演算で、PyTorch などが自動的に次元を繰り返して形を揃える仕組みです。長さ256のベクトルを (1, 256, 1, 1) に整形すると、(B, 256, H, W) の tensor に対し「256チャンネルそれぞれに対応する値を、全バッチ・全格子マスへ同じ値で」掛けられます。

学習可能パラメータ(learnable parameter): 学習(誤差逆伝播)によって値が更新される数です。これをどう初期化するかが学習の安定性を大きく左右します。

## 仕組みを詳しく

中心となる演算は数式一本で書けます。

```
output = image_bev + alpha * map_bev
```

- `image_bev`: カメラ画像から作った BEV 特徴。形状 (B, C, H, W)。例 (4, 256, 200, 100)。
- `map_bev`: 地図から作った BEV 特徴。`image_bev` と同じ形状でなければならない。
- `alpha`: チャンネルごとに1個ずつ用意した「混ぜる強さ」を表す学習可能パラメータ。長さ C(=256)のベクトル。

ステップ1: alpha をゼロで初期化する。

```python
self.alpha = nn.Parameter(torch.zeros(embed_dim))
```

全てゼロで始めるのが核心です。学習開始時点では `alpha * map_bev` が完全にゼロなので `output = image_bev + 0 = image_bev` となり、地図ブランチが無いモデルと完全に同一の振る舞いになります。地図エンコーダが初期にノイズを撒いて本体学習を壊すことが原理的にありません。

ステップ2: ブロードキャストのための整形。

```python
gate = self.alpha.view(1, -1, 1, 1)   # 形状 (1, 256, 1, 1)
```

`alpha` は長さ256の1次元ベクトル、足す相手 `map_bev` は4次元 (B, 256, H, W)。そのままでは掛けられないので形を (1, 256, 1, 1) にします。ブロードキャストにより、256チャンネルそれぞれに対応する `alpha` が全バッチ・全格子マスへ同じ値で掛かります。例: チャンネル17番の `alpha[17]=0.8` なら、map_bev のチャンネル17番だけが 0.8 倍されて image_bev のチャンネル17番に足されます。チャンネルごとに重みが独立なので、「このチャンネルの地図情報は役立つから強く」「このチャンネルは要らないからゼロのまま」とモデルが選り好みできます。

ステップ3: 残差として加算する。

```python
return image_bev + gate * map_bev
```

出力形状は入力と同じ (B, 256, H, W)。後段(プランナや軌道ヘッド)は融合済み BEV をそのまま使えます。

学習が進むとどうなるか。誤差逆伝播を通じて、「地図のこのチャンネルを足すと予測が良くなる」なら、その `alpha[c]` が自然に大きくなります。役に立たないチャンネルの `alpha` はゼロ付近にとどまります。人手で「地図をどれだけ信じるか」を決め打ちする必要がなく、データから自動で決まります。

before / after の数値イメージ。学習開始時、image_bev のあるマスの値が 0.50、map_bev が 0.90、alpha が 0.0 なら output は 0.50(地図無視)。学習後、有用と判明して alpha が 0.6 になれば output は 0.50 + 0.6×0.90 = 1.04 となり、地図が寄与します。逆に有害なチャンネルでは alpha が 0.0 付近に留まり、地図は無視され続けます。

## 手法の系譜と主要論文

ResNet (He et al., CVPR 2016, arXiv:1512.03385): 残差接続 `y = x + F(x)` を提案し、勾配消失を恒等写像経路で解決して152層という超深層の学習を可能にしました。ゲート付き残差融合は「F(x) の代わりに alpha 倍した地図特徴を足す」応用で、`alpha=0` 初期化により「最初は恒等写像=画像 BEV そのまま」という安全な開始点を作る点が ResNet 思想の継承です。

Fixup Initialization (Zhang et al., ICLR 2019, arXiv:1901.09321): 正規化層(BatchNorm 等)に頼らず、残差ブランチの最後の層をゼロ初期化することで、各残差ブロックが恒等写像から始まるようにし、深いネットワークを安定に学習できることを示しました。「残差の寄与をゼロから始める」という思想の理論的土台です。

ReZero (Bachlechner et al., UAI 2021, arXiv:2003.04887): 各残差ブロックを `y = x + α·F(x)` とし、スカラー α を初期値ゼロの学習可能パラメータにしました。これだけで Transformer や深層 CNN の学習が安定・高速化し、収束エポックが大きく短縮されることを報告。ゲート付き残差融合の per-channel alpha は、この ReZero ゲートをスカラーからチャンネル単位ベクトルに拡張したものと位置づけられます。

ControlNet (Zhang et al., ICCV 2023, arXiv:2302.05543): 学習済み拡散モデルへ新しい条件(エッジ画像や姿勢など)を注入する際、接続部に zero-convolution(重み・バイアスをゼロ初期化した畳み込み)を置きました。学習初期は新条件の寄与がゼロなので、巨大な事前学習モデルを一切壊さずに新機能を足せます。地図融合のゼロ初期化ゲートと完全に同じ発想で、別分野でも有効性が実証されています。

BEVFusion (Liu et al., ICRA 2023, arXiv:2205.13542): カメラと LiDAR の BEV 特徴を共通 BEV 空間で融合する代表的手法。二つのセンサ特徴を連結(concat)してから畳み込みで混ぜます。残差融合との違いは、BEVFusion が concat+畳み込みで混ぜ方の表現力を高める(その分パラメータと計算が増える)のに対し、残差融合は地図特徴を「足す」だけで軽量、かつ初期に寄与ゼロにできる点です。

## 論文の実験結果(定量データ)

ゲート付き残差融合そのものは単独の論文より、上記の構成要素の効果として定量化されています。

ReZero の報告では、12層 Transformer の学習で、標準的な初期化に比べ収束が大幅に速くなり、また 100 層を超える深い Transformer を追加の正規化なしで安定に学習できたとされます。スカラーゲートのゼロ初期化が「学習初期の信号爆発を防ぐ」ことが、収束速度の改善として測られたものです。

Fixup の報告では、正規化層を一切使わずに、ImageNet 分類や機械翻訳で正規化ありと同等の精度に到達できたとされます。これは「残差ブランチをゼロから始める」初期化単独の寄与を示します。

ControlNet の報告では、zero-convolution を通常の畳み込み(ランダム初期化)に置き換えると学習初期に出力が崩壊しやすく、収束も不安定になることが示されています。これは「ゼロ初期化ゲートが、新規入力を既存モデルへ安全注入する鍵」であることのアブレーション的証拠です。

融合方式の比較については、BEVFusion 系の研究で、単純加算・concat・attention の各融合を比べると、表現力(到達精度)は attention・concat が高く、計算コストと初期安定性は加算が優れる、というトレードオフが一般的に報告されます。指標の意味としては、ここでの精度は下流タスク(検出 mAP や運転計画の誤差)で測られ、融合の良し悪しが最終性能にどう効くかを表します。

これらが重要な理由。「新しい情報源を足すと往々にして学習が壊れる」という実務的な落とし穴に対し、ゼロ初期化ゲートが分野横断(画像分類・拡散モデル・センサ融合)で一貫して有効だと示されている点が、この手法の信頼性の根拠です。

## メリット・トレードオフ・限界

メリット。alpha ゼロ初期化により学習初期は地図ブランチが本体学習を一切邪魔しません(安全な開始)。チャンネルごとに混ぜる強さをデータから自動学習でき、人手チューニング不要です。残差加算は連結より軽量で、チャンネル数が増えず後段の計算も増えません。地図エンコーダ無しの構成へ自然に縮退でき(alpha=0 で同一挙動)、後方互換が取りやすい。

トレードオフと限界。混ぜ方がチャンネル単位のスカラー倍に限られ、空間的に「どの場所の地図を画像のどこへ効かせるか」は制御できません。image_bev と map_bev の空間サイズ (H, W) が完全一致している必要があり、事前に BEV 解像度を揃える前処理が前提です。学習済み alpha は推論時に固定なので、「この場面では地図が間違っていそうだから信頼度を下げる」という状況依存の動的調整はできません。それを実現したければ、地図と画像を attention で対応づける cross-attention 融合や、入力に応じてゲートを動的計算する手法(条件付きゲート)が候補になります。研究上の未解決課題は、地図が現実とずれている(古い・誤っている)場面での頑健性で、静的なチャンネルゲートはこの誤りを場面ごとに無効化する手立てを持ちません。

## 発展トピック・研究の最前線

空間的な選択性を持たせる方向(地図 BEV とシーン BEV を deformable cross-attention で位置対応づける)、ゲートを入力依存にする方向(現フレームの不確実性からゲート値を動的に出力する)、地図の信頼度自体を推定して融合重みに反映する方向が考えられます。また、地図を BEV 特徴として注入するのではなく、ベクトル地図のトークンを Transformer のクエリとしてプランナに直接与える「マップトークン化」も、エンドツーエンド運転(VAD, UniAD 系)で主流になりつつあります。ゼロ初期化ゲートの発想自体は LoRA や Adapter といった大規模モデルの効率的微調整にも通じ、「既存能力を壊さず新機能を足す」一般原理として広く再利用されています。
