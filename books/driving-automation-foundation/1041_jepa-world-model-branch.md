# JEPA 系の低速ワールドモデル分岐(予測表現を補助タスクにする設計)

## ひとことで言うと

運転モデルに「ゆっくり動く、世界の先読みをする枝」を追加する設計です。高速(例 10Hz)で動く運転判断の枝とは別に、低速(例 1Hz)で過去数フレームの映像をまとめ、「数秒後にシーンがどう見えるか」を潜在空間(意味の空間)で予測します。この予測タスクの学習信号が、運転判断側の特徴抽出も同時に賢くします。これは JEPA(Joint-Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)の原理を、エンドツーエンド運転に補助タスクとして組み込む一般的な構成です。

## 直感的な理解

反応的(reactive)なエンドツーエンド運転モデルは、今この瞬間のマルチカメラ画像を高速に処理し、すぐに軌道を出します。反射神経のように速いですが、「これから何が起きるか」を内部に明示的には持っていません。

人間の運転を考えると、私たちは目の前だけでなく「数秒先のシーンの予測」を頭の中で常に走らせています。前の車が減速しそう、あの歩行者は渡りそう、という先読みです。この「世界が時間とともにどう変化するか」を内部に持つモデルをワールドモデル(world model、世界モデル)と呼びます。

ここでの設計思想は、この先読みを別個の重い計画器として作るのではなく、「補助タスク(auxiliary task)」として学習に足すことです。補助タスクとは「本来やりたいこと(=軌道予測)とは別に、ついでに解かせる学習課題」のこと。軌道の損失だけで学習すると、画像特徴を作る前段(バックボーンとビュー融合)に届く学習信号が弱くなりがちです。なぜなら損失はネットワークのずっと下流(出力側)にあり、勾配がたくさんの層を逆向きに伝わるうちに薄まる(勾配消失に近い現象)からです。そこで「数秒後の特徴を予測せよ」という別の課題を直接前段に与えると、前段に濃い学習信号が届き、「シーンがどう変化するかを予測できるだけの情報を含む特徴」を作るよう促せます。これが共有表現の質を上げ、結果として軌道予測も良くなる、という理屈です。

そして「なぜ画像(ピクセル)を予測せず、特徴(潜在空間)を予測するのか」。未来のピクセルを予測しようとすると、影の位置・テクスチャ・圧縮ノイズといった運転に無関係な細部まで再現させられ、無駄が多い。これを解決するのが JEPA で、「予測を潜在空間で行う」ことで意味のある変化だけを捉えます。詳細は [I-JEPA / V-JEPA(自己教師あり予測表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1074_i-jepa-v-jepa)。

## 基礎: 前提となる概念

- エンドツーエンド運転(end-to-end driving): カメラ画像などの生の入力から、知覚・予測・計画を1つのニューラルネットでまとめて学習し、直接ハンドル・アクセルや走行軌道を出す方式。従来の「検出→追跡→予測→計画」をモジュールに分けるパイプライン型と対比されます。
- バックボーン(backbone): 画像から特徴マップを抽出する基幹ネットワーク(ResNet、Swin Transformer、ConvNeXt など)。
- ビュー融合(view fusion): 複数カメラの特徴を1つの統合表現(連結、クロスアテンション、あるいは鳥瞰図 BEV)にまとめる処理。
- 軌道(trajectory): 自車がこれから走る位置の系列(例 数秒先までの (x, y) 点列)。
- 模倣損失(imitation loss): 人間の運転ログを正解として、モデルの軌道がそれにどれだけ近いかを測る損失。エンドツーエンド運転の主損失。
- 補助損失(auxiliary loss): 主損失とは別に足す損失。前段の表現を鍛えたり、学習を正則化したりする目的で使う。
- 潜在空間(latent space): エンコーダが作る抽象特徴の空間。ピクセル空間の対義語。
- 速い枝・遅い枝(fast/slow branch): 高頻度で反応する経路と、低頻度で先読みする経路を分ける二系統設計。生物の反射(脊髄反射)と熟慮(大脳)の役割分担の比喩で語られることがある。

## 仕組みを詳しく

低速ワールドモデル枝の典型的な構成要素を分解します。具体的な数値は一例であり、解像度・次元・ホライズン本数は設計者が選ぶ自由度です。

### 1. FrameEncoder(1フレームをベクトルに圧縮)

入力は1時刻のマルチカメラフレーム `[B, V, 3, H, W]`(B はバッチ、V はカメラ台数、3 は RGB)。処理はバックボーン → 最終特徴マップ → 全空間平均(global average pooling、GAP)→ 線形射影で、出力は1フレームあたりの埋め込みベクトル `[B, D]`(D は例えば 224〜256 次元)です。マルチカメラの場合は各カメラを符号化してから平均、あるいはアテンションで融合します。

```
[B, V, 3, H, W] → backbone → GAP → 各視点を融合 → Linear → [B, D]
```

バックボーンは反応側の枝と共有するのが定石です。別々のバックボーンを持つとパラメータが倍増し、かつ表現が分裂して補助タスクの「前段を鍛える」効果が薄れるためです。

### 2. RollingHistoryBuffer(過去フレームの FIFO バッファ)

低速側で来る各フレームの埋め込みを、サイズ N(例 4)の循環バッファ(FIFO、先入れ先出し)に溜めます。読み出すと古い順→新しい順に連結して `[B, N·D]`(例 `[B, 4·224] = [B, 896]`)を返します。この連結ベクトルがそのまま反応側のプランナへの「視覚履歴」入力になります。バッファが埋まる前は先頭をゼロ詰めします。

```
tick: emb_t → push → buffer=[e0,e1,e2,e3] → concat → [B, 896] → 反応プランナへ
```

### 3. future_predictor(未来の特徴を予測)

連結した視覚履歴 `[B, N·D]` から、K 個(例 4)の未来埋め込み `[B, D]` を予測する小さな予測器(MLP や軽量 Transformer)です。

```
[B, 896] → Linear → GELU → Linear → [B, 4·224] → 4分割 → [B, 224] × 4
```

### 4. target エンコーダ(正解の作り手、frozen)

予測した未来特徴を「実際の未来フレームの特徴」と比べる必要があります。その正解は、frozen(凍結=学習で更新されない)なバックボーンのコピーが、実際の未来フレーム(例 +1.6s, +3.2s, +4.8s, +6.4s)を符号化して作ります。この target は勾配を遮断(stop-gradient、detach)します。理由は表現崩壊を防ぐためで、詳細は [I-JEPA / V-JEPA(自己教師あり予測表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1074_i-jepa-v-jepa)。target を frozen にするか EMA(指数移動平均)にするかは設計上のトレードオフで、frozen は単純だが学習後半で古びる、EMA は新鮮だが減衰率の調整が要ります。

### 5. jepa_loss(損失)

予測と正解(detach 済み)の特徴空間での距離を測ります。距離は L1(絶対値の平均、V-JEPA に倣う)・L2(I-JEPA)・smooth-L1(Huber、外れ値に頑健)から選びます。これを軌道の模倣損失に重み付きで足します:

```
L_total = L_imitation + λ · L_jepa
```

λ は補助損失の重み。大きすぎると軌道精度を犠牲にし、小さすぎると補助効果が消えるため、調整が要ります。研究実装では λ を 0.1〜1.0 程度から探索し、学習曲線で「模倣損失を悪化させずに表現指標が改善する」点を探すのが典型です。

### オプトイン設計と推論時の扱い

この枝は「デフォルトでオフ、明示的に有効化したときだけ動く」オプトイン(opt-in)設計にするのが安全です。オフのときは従来の反応的挙動と完全に同一になります。オンのとき、forward は推論時には軌道だけを返し、学習時には軌道と未来予測の両方を返して2損失を計算します。シーケンス(走行区間)間ではバッファをクリアして、別の走行の履歴が混ざらないようにします。実データで未来フレームを供給できない初期段階では、ランダムテンソルで2損失(模倣+JEPA)の1ステップが通ることを確認し、本格学習はデータパイプラインが未来フレームを書き出せるようになってから行う、という段階的な進め方が現実的です。

## 手法の系譜と主要論文

- LeCun, "A Path Towards Autonomous Machine Intelligence" (2022): JEPA という枠組みを提唱した position paper(立場表明論文)。「ピクセルを予測するのではなく抽象表現を予測する世界モデル」という哲学の出発点で、本設計の動機の源流。エネルギーベースモデルと階層的・行動条件付き JEPA(H-JEPA)という長期構想を提示した。
- Grill et al., "BYOL" (NeurIPS 2020, arXiv:2006.07733): 学習可能な predictor と EMA で緩やかに更新する target という非対称設計で崩壊を防ぐ。JEPA の target エンコーダ設計の原型。
- Assran et al., "I-JEPA" (CVPR 2023, arXiv:2301.08243): 画像で、文脈領域から対象領域の表現を潜在空間で予測する手法を実証。ここでの未来特徴予測ロスと target エンコーダ設計の直接の基盤。
- Bardes et al., "V-JEPA" (2024, arXiv:2404.08471): 動画版 JEPA。時間方向の予測を扱い、損失を I-JEPA の L2 から L1 に変更。運転の未来予測で L1 を採るのはこの知見に倣う。
- Bardes et al., "V-JEPA 2" (2025, arXiv:2506.09985): 100 万時間超の動画で事前学習し、行動条件付き予測でロボット操作のゼロショット計画にまで踏み込んだ。低速枝を「行動を入力に取る世界モデル」へ拡張する方向の最先端。
- Hu et al., "Model-Based Imitation Learning for Urban Driving (MILE)" (NeurIPS 2022, arXiv:2210.07729): 自動運転で潜在ワールドモデルを学習し、ピクセル予測ベースラインを上回ることを示した。「運転で特徴空間予測が効く」直接の根拠。
- Hu et al., "Planning-oriented Autonomous Driving (UniAD)" (CVPR 2023, arXiv:2212.10156): 知覚系の補助損失が共有 BEV 表現の質を上げ、最終的な計画性能を改善することを示した。「補助タスクが前段表現を良くする」主張の裏付け。

系譜の流れを一言で繋ぐと、BYOL が「非対称設計で崩壊を防ぐ」原理を確立し、LeCun がそれを「世界モデルは潜在空間で予測すべき」という哲学に格上げし、I-JEPA / V-JEPA が画像・動画で実証し、MILE / UniAD が「運転で潜在予測・補助タスクが効く」ことを定量的に示した、という積み上げになっています。

## 論文の実験結果(定量データ)

主要な定量的根拠を、指標の意味とともに示します。

- MILE(CARLA シミュレータの CARLA Leaderboard で評価): 運転性能の代表指標は「Driving Score(走行スコア)」で、これは「ルート完走率(どれだけ目的地まで進めたか)」と「違反ペナルティ(衝突・信号無視などの減点を 0〜1 の係数で表現)」を掛け合わせた総合指標です。MILE は潜在空間で世界の力学を学ぶ手法で、CARLA の新規 town(学習に使わなかった町)での汎化を含め、当時の模倣学習・行動クローニング系のベースラインを上回る Driving Score を報告しました。重要なのは、潜在ワールドモデルを学んだ表現が、明示的な BEV セグメンテーションのラベルが乏しい状況でも運転性能を支えた点で、「潜在予測が運転に効く」ことの定量的裏付けになっています。
- UniAD(nuScenes データセットで評価): nuScenes は約 1000 シーン・多数の都市走行を含む大規模データセットです。UniAD は知覚・予測・計画を1つのネットに統合し、計画の評価指標である「L2 誤差(予測軌道と正解軌道の平均距離、メートル単位、小さいほど良い)」と「衝突率(collision rate、予測軌道で他物体に衝突する割合、%、小さいほど良い)」の両方で当時の最先端を更新しました。報告では平均 L2 誤差を従来手法から有意に下げ(おおむね 1m 前後の領域でベースライン比の改善)、平均衝突率も低減しています。論文のアブレーションが本トピックにとって重要で、知覚・予測の各補助タスク(検出・追跡・地図・占有・動き予測)を1つずつ外していくと、最終的な計画の L2 誤差と衝突率がいずれも悪化することが示されています。つまり「上流の補助タスクが共有表現を鍛え、下流の計画を助ける」という主張が、各タスクを抜いたときの劣化量で定量的に裏付けられています。これは「JEPA 未来予測を補助タスクとして足す」設計の根拠そのものです。
- I-JEPA / V-JEPA の効率と表現品質の数値は [I-JEPA / V-JEPA(自己教師あり予測表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1074_i-jepa-v-jepa) に詳述。要点は「潜在予測はピクセル予測より学習効率が良い(同じ精度に少ない計算で到達)」「frozen/EMA の非対称性を外すと表現が崩壊して下流精度が大きく落ちる」というアブレーション結果です。

## メリット・トレードオフ・限界

メリット:
- 前段(バックボーン / 融合)に濃い学習信号を与え、共有表現の質を上げうる補助タスク。
- 潜在空間予測なので、運転に無関係な細部を学ばずに済む。
- バックボーンを共有しパラメータ増を抑制。frozen/EMA target で崩壊を回避。
- オプトイン設計で、オフ時は既存の反応的挙動と完全互換。段階的に導入・検証できる。
- 予測誤差そのものを「想定外検知(out-of-distribution / 不確かさ)」の信号に転用できる可能性がある。予測が大きく外れる状況=モデルが訓練分布で見ていない状況、という解釈。

トレードオフ・限界:
- 実データの未来フレーム供給(データパイプライン側の未来フレーム書き出し・時刻整合)が前提で、これが未整備だと学習信号が作れない。
- 低速の別枝・未来フレーム保持で計算・メモリが増える。未来 BEV 特徴は高解像度だと巨大になり、空間圧縮や予測空間の調整([予測空間の圧縮(forecast space を切り替える調整ノブ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1036_feature-compressor-forecast-space))が事実上の前提になる。
- 補助損失の重み λ のチューニングが性能を左右する。λ が大きいと模倣損失と綱引きになり軌道が劣化する。
- frozen target は学習後半で古びる懸念があり、EMA との設計トレードオフが残る。
- 補助タスクが下流の運転性能を本当に上げるかは、最終的には実データでの A/B 評価(補助損失あり・なしでの L2 誤差・衝突率比較)でしか確かめられない。「効くはず」という理屈と「実際に効いた」は別物である点が研究上の未解決課題。

## 発展トピック・研究の最前線

- 行動条件付き世界モデル: 予測に「自車の行動(軌道)」を入力として与えると、「軌道Aを取ったら世界はどう変わるか」という反実仮想(counterfactual)ロールアウトができ、予測と計画が双方向に連携します。V-JEPA 2 のロボット応用がこの方向の先端です。詳細は [未来状態モジュールの設定可能ホライズンと残差(TDV)予測器](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1040_future-state-tdv-residual-predictor)。
- 生成系ワールドモデルとの対比: GAIA-1(Wayve, 2023, arXiv:2309.17080)や Vista(Gao et al., NeurIPS 2024, arXiv:2405.17398)はピクセル/トークンを生成する重い世界モデルで、可視化や long-horizon(長い時間範囲)のロールアウトに強い一方、計算コストが高い。JEPA 系は潜在予測で軽量だが、何を学んだかを目で見て確かめにくい。両者の使い分け(可視化と計画には生成系、表現学習と効率には JEPA 系)が研究テーマです。
- 残差予測による安定化: 未来をゼロから予測せず「現在+変化分」に分解する残差予測は学習を安定させます。詳細は [未来状態モジュールの設定可能ホライズンと残差(TDV)予測器](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1040_future-state-tdv-residual-predictor)。
- 階層 JEPA(H-JEPA): LeCun の構想では、短いホライズンを細かく予測する下位 JEPA と、長いホライズンを粗く予測する上位 JEPA を積み上げ、長期計画を扱う。運転で「数秒先の車線変更」と「数分先のルート選択」を異なる時間スケールで予測する方向に対応する。
