# MaxViT (Multi-Axis Vision Transformer)

## ひとことで言うと
MaxViT(マックスビット)は、画像認識モデルで「近くだけを密に見る注意」と「画像全体に薄く広く張った注意」を交互に重ね、近所と遠くの両方を効率よく見られるようにした Transformer です。普通の ViT は画像全体の注意を計算すると非常に重くなりますが、MaxViT はその計算量を画像サイズに比例(線形)まで抑えながら大域性を保つ点が特徴です。畳み込み(MBConv)も各ブロックに組み込んだ、純粋な ViT と CNN の中間に位置する階層型バックボーンで、分類・検出・セグメンテーション・画像生成まで一つの設計で横断的に強いことを示しました。

## 直感的な理解
ViT の自己注意は「全員が全員と会議できる」ので視野は無限ですが、参加者(画像のトークン)が増えると会議の組み合わせが爆発します。これを抑える先行手法 Swin は「同じ部屋の人とだけ会議する」窓注意で計算を軽くしましたが、部屋の外の人とは直接話せず、何層も重ねてようやく情報が部屋をまたいで伝わります。つまり大域性が間接的で遅い。Swin で画像の端から端まで情報を届かせるには、窓のサイズと層数に応じて何段も積む必要があり、浅い段では受容野(一つの出力が見ている入力の範囲)が画像全体に届きません。

MaxViT の発想は、会議を二種類に分けて交互に開くことです。一つは「同じ町内の人だけの会議」(Block 注意、局所)。もう一つは「全国の各町から一人ずつ集めた会議」(Grid 注意、大域)。後者は画像全体に均等に散らばった代表だけを集めるので参加者は少なく計算は軽いのに、画像の端から端までの関係(空と路面、左右の車線など)を一回の会議で直接つかめます。この二種の会議を交互に開けば、近所も遠方も、しかも軽い計算で見渡せます。最初のブロックから大域が見えるという点が、層を積んで待つ Swin との決定的な違いです。

## 基礎: 前提となる概念
用語を順に噛み砕きます。

トークン(token)とは、画像を細かく区切ったときの一つの単位(パッチ、あるいは特徴マップ上の一点)です。自然言語処理の単語に相当します。

受容野(receptive field)とは、ある出力の値が、入力画像のどれだけの範囲の情報に依存しているかです。畳み込みは層を重ねるほど受容野が少しずつ広がりますが、なかなか画像全域には届きません。全トークン注意は一発で受容野が画像全域です。MaxViT は Grid 注意によって、計算を抑えたまま受容野を最初から画像全域に広げます。

自己注意(self-attention)は、各トークンが他トークンとの関係の強さを計算し、関係が深い相手の情報を重く取り込む仕組みです。各トークンから Query, Key, Value を作り、Query と Key の内積で関係を測り softmax で正規化して Value を加重平均します。計算量はトークン数 N の二乗 O(N^2) で、N が増えると急速に重くなります。具体例で言うと、画像を 56x56 = 3136 トークンに分けると、全トークン間注意は 3136 x 3136 ≒ 約 980 万回の関係計算になります。高解像度の画像(自動運転のカメラ画像、医用画像、衛星画像など)では非現実的です。

線形計算量とは、トークン数 N に対して計算が N の一乗(N に比例)で済むことです。O(N^2) を O(N) に落とせれば、解像度を上げても破綻しません。Block 注意も Grid 注意も、グループあたりの参加者数を固定(たとえば 49)にすることで、グループ数が増えるだけになり、全体では N に比例します。

窓注意(windowed attention)は、画像を小さな窓に区切り窓内だけで注意を計算する手法で、Swin Transformer(Liu et al., ICCV 2021)が代表です。窓が小さいので計算は線形ですが、窓の外は見えません。Swin は窓をずらす(shifted window)工夫で隣の窓と少しずつ情報を混ぜ、層を重ねて画像全体に情報を行き渡らせます。

MBConv は MobileNet 由来の軽い畳み込みブロック(深さ方向分離畳み込み+inverted bottleneck+残差)で、CoAtNet(Dai et al., 2021)でも採用された局所処理部品です。畳み込みは空間順にスライドするため、暗黙の位置情報も持ち込めます。MaxViT が MBConv を各ブロックに残したのは、CoAtNet 譲りの「浅い局所処理は畳み込みが効率的かつ位置情報も担える」という知見の継承です。

MaxViT(Tu, Talbott, Hossain, Wu, Zhang, et al.; Google Research, ECCV 2022)は「一つのブロックの中で、局所と大域の両方を直接かつ線形計算量で見たい」という動機で生まれました。CoAtNet が「深い段でしか大域を見られない」制約を持っていたことへの回答でもあります。

## 仕組みを詳しく
MaxViT の核は Multi-Axis Self-Attention(多軸自己注意、Max-SA)です。二種類の注意を組み合わせます。例として `[B, C, H, W] = [B, 192, 28, 28]` の特徴マップで考えます。

### 1. Block Attention(ブロック注意=局所)
特徴マップを固定サイズの窓(例: 7x7)に格子状に区切ります。28x28 なら 4x4 = 16 個の窓ができ、各窓は 7x7 = 49 トークン。注意は各窓の内部 49 トークンの間だけで計算します。これは近所だけを密に見る局所注意です。計算量は窓サイズ(49)で決まり、画像が大きくなっても窓あたりは一定なので、全体では O(N) の線形です。テンソル操作としては、空間を `(高さの窓数 Gh, 窓内高さ P, 幅の窓数 Gw, 窓内幅 P)` に並べ替え、窓内次元 (P, P) を注意の対象軸に取ります。

### 2. Grid Attention(グリッド注意=大域)
ここが MaxViT の発明です。今度は格子を「ブロックではなくグリッド(網目)」で取ります。28x28 を 7x7 のグリッドに割り当て直すと、各グループは画像全体に均等に散らばった 4x4 = 16 トークンの集まりになります。隣り合うトークンではなく、画像の端から端まで一定間隔で間引いたトークン同士で注意を計算します。

たとえると、Block は「同じ町内の人だけで会議」、Grid は「全国の各町から一人ずつ集めて会議」です。Grid は画像全体に薄く広く張った注意なので、遠く離れた領域の関係を 1 ブロックで直接捉えられます。しかも各グループのトークン数は間引かれて少ないので計算は軽く、こちらも O(N) の線形です。テンソルとしては Block と同じ並べ替えの軸の取り方を入れ替えるだけ(窓内軸 (P,P) ではなくグリッド軸 (Gh,Gw) を注意対象にする)で実装でき、追加コストがほとんどないのも巧妙な点です。Axial Attention が「行」「列」という二つの軸に注意を分けたのを、MaxViT は「局所窓内」「大域グリッド」という二つの軸に分けた、と理解できます。これが "Multi-Axis(多軸)" の名の由来です。

```
特徴マップ 8x8(数字は同じグループに入るトークン)

Block Attention(近所をまとめる)     Grid Attention(間引いて散らす)
 A A B B C C D D                       1 2 1 2 1 2 1 2
 A A B B C C D D                       3 4 3 4 3 4 3 4
 A A B B C C D D                       1 2 1 2 1 2 1 2
 E E F F G G H H   ...                 3 4 3 4 3 4 3 4   ...
同じ文字=同じ窓で注意                  同じ数字=同じグリッドで注意
近接トークン同士                        画像全体に散ったトークン同士
```

### ブロック構成
MaxViT は一つの MaxViT ブロックを次の順で並べます。

1. MBConv(畳み込み。局所処理に加え暗黙の位置情報を埋め込む。条件付き位置エンコーディング conditional position encoding の役割も果たすため、明示的な位置埋め込みが不要)
2. Block Attention(局所注意、相対位置バイアス付き)
3. Grid Attention(大域注意、相対位置バイアス付き)

この三つを 1 セットとし、各 Stage(階層)で繰り返します。階層は解像度を半分ずつ下げチャネルを増やし(`[B,64,56,56]` → `[B,128,28,28]` → `[B,256,14,14]` → `[B,512,7,7]`)、CNN と同じ多段特徴を出します。重要なのは、最初の Stage(高解像度)から既に Grid 注意で画像全体を見られること。Swin のように層を重ねて大域伝達を待つ必要がありません。これが「最初のブロックから受容野が画像全域に届く」という MaxViT の主張です。モデルサイズは MaxViT-T(Tiny)/S/B/L/XL があり、各 Stage の繰り返し数とチャネル幅でスケールします。

## 手法の系譜と主要論文
- Axial Attention(Ho et al., 2019, arXiv:1912.12180; Wang et al., Axial-DeepLab, ECCV 2020, arXiv:2003.07853): 注意を行方向と列方向に分けて順に適用し、二次元の全域注意を線形近似する。MaxViT の "多軸(multi-axis)" の発想の源流です。Axial が行・列という直交する軸に分解したのに対し、MaxViT は局所・大域という直交する軸に分解した、という対応関係があります。
- ViT(Dosovitskiy et al., ICLR 2021, arXiv:2010.11929): 全トークン注意で大域性は完璧だが O(N^2) で高解像度に弱い起点。
- Swin Transformer(Liu et al., ICCV 2021, arXiv:2103.14030): 窓注意で線形計算量を実現。ただし大域性は窓ずらしの積み重ねに依存し間接的。MaxViT はこの弱点を Grid 注意で直接補いました。Swin の shifted window が「窓を半分ずらして隣と繋ぐ」という間接的な大域化なのに対し、MaxViT は最初から大域グリッドを一本入れます。
- CoAtNet(Dai et al., NeurIPS 2021, arXiv:2106.04803): 段ごとに畳み込みと注意を切り替えるハイブリッド(C-C-T-T)。MaxViT は MBConv を各ブロックに残しつつ、全段で局所も大域も持つ点が前進です。同じ Google チームの連続した研究で、CoAtNet の「深い段でしか大域が見えない」制約を解いたのが MaxViT という関係です。
- MaxViT(Tu, Talbott, Hossain, Wu, Zhang, et al.; Google Research, ECCV 2022, arXiv:2204.01697): MBConv + Block 注意 + Grid 注意のブロックを提案。高解像度でも線形計算量で局所と大域を同時に捉える。

## 論文の実験結果(定量データ)
MaxViT は分類だけでなく検出・セグメンテーション・生成まで横断的に評価されました。一つの設計で多タスクに強いことを示すのが論文の狙いの一つです。

画像分類(ImageNet、指標は top-1 accuracy = 最確信予測が正解と一致した割合):
- ImageNet-1k のみ学習(224x224): MaxViT-T(約 3100 万パラメータ)で top-1 約 83.6%、MaxViT-S(約 6900 万)で約 84.5%、MaxViT-B(約 1.2 億)で約 85.0%、MaxViT-L(約 2.1 億)で約 85.2%。
- 512x512 の高解像度ファインチューニングで MaxViT-L は top-1 約 86.7% に到達。
- ImageNet-21k 事前学習 → 1k 転移: MaxViT-L が top-1 約 88.4%、最大の MaxViT-XL では約 88.5〜88.7% を報告。当時の上位に並びました。
- パラメータ・FLOPs に対する精度効率で、同規模の Swin や CoAtNet を上回ると報告されています。たとえば同程度の FLOPs 帯で MaxViT-T が Swin-T を約 2 ポイント上回るといった比較が示されています。

物体検出・インスタンスセグメンテーション(COCO データセット、指標は AP = Average Precision、検出の正確さを総合した値で高いほど良い): MaxViT バックボーンを Cascade Mask R-CNN などに載せた構成が、同規模 Swin を上回る box AP / mask AP を報告しました。高解像度入力でも線形計算量で済む利点が検出で活きます。

セマンティックセグメンテーション(ADE20K、指標は mIoU = mean Intersection over Union、予測領域と正解領域の重なり率の平均で高いほど良い)でも強い結果を示しました。

画像生成(GAN)への応用でも、MaxViT ベースの生成器が当時の FID(Frechet Inception Distance、生成画像と実画像の分布の近さを測る指標で低いほど良い)で競争力を示し、バックボーンとしての汎用性を裏づけました。分類向けに設計したブロックが生成にも使えることは、設計の一般性の証拠とされます。

アブレーション(どの要素を抜くと何が起きるか):
- Grid 注意を外して Block 注意だけにすると精度低下。大域注意の寄与を示します。Block だけでは Swin に近い「局所しか直接見えない」状態に戻ります。
- Block 注意を外して Grid だけにしても低下。局所の密な注意も必要で、両者が補完的だと分かります。Grid は画像全体を疎にしか見ないため、近接する細部の相互作用が抜け落ちます。
- MBConv を外すと精度低下かつ位置情報が弱まる。MBConv が局所特徴と暗黙の位置エンコーディングを担っていることを示します。MBConv なしだと別途位置埋め込みが必要になります。
- 窓サイズ・グリッドサイズを変える実験で、入力解像度との整合(窓サイズが解像度を割り切れること)が性能に効くことが示されました。

数値の意味を補足すると、ImageNet top-1 で 83〜85% の領域での 0.5〜1.0 ポイント差は、同等パラメータ・FLOPs 下での差なので効率の優位として意味があります。とくに「同じ計算量で Swin より高い」という主張が、線形計算量を保ったまま大域性を足せたことの実利です。Block と Grid を両方持つことの追加コストはほぼゼロ(軸の取り方を変えるだけ)なので、得られる精度向上はほぼ純利益と見なせます。

## メリット・トレードオフ・限界
メリット
- 局所(Block)と大域(Grid)を一つのブロックで両立。最初のブロックから受容野が画像全域に届き、情報伝達が速い。
- 計算量が画像サイズに線形。高解像度入力(検出・セグメンテーション・大判画像)に強い。
- 階層型なので検出・セグメンテーションなど下流タスク向けの多段特徴を出せる。生成にも応用可能で汎用性が高い。
- Block/Grid の切り替えがテンソルの並べ替えだけで実装でき、追加パラメータがほぼない。

トレードオフ・限界
- MBConv + Block + Grid の三段構成で、純粋 ViT や ResNet より実装が複雑。
- 窓サイズ・グリッドサイズが入力解像度に依存し、割り切れない解像度ではパディングや調整が必要。任意解像度への適応にやや手間がかかる。検出のように入力サイズが可変なタスクで運用上の注意が要ります。
- Grid 注意は画像全体を疎にサンプルするため、近接する微細構造の相互作用は Block 注意頼みになり、両者の役割分担に依存する。Grid のサンプリング間隔が粗いと、その中間スケールの構造を取りこぼす可能性があります。
- 推論メモリと実装の手間が純 CNN より増える。実機スループットはカーネル実装(並べ替えと注意の融合)に左右されやすく、理論 FLOPs と実測速度が乖離しやすい。

研究上の未解決課題として、Block と Grid の比率や順序、窓サイズの自動決定は経験則に依存しており、解像度・タスクをまたいだ最適設計の一般原理は確立していません。また Grid の固定間隔サンプリングが最良の大域化かどうか(学習可能なサンプリングや変形可能な注意との比較)も開かれた問いです。

## 発展トピック・研究の最前線
MaxViT が示した「一ブロック内で多軸に局所と大域を両立する」考えは、効率的な高解像度ビジョンの一つの到達点です。系譜としては Swin の窓注意、CoAtNet の段分離ハイブリッドを統合・発展させた位置づけで、後続の効率的注意(疎注意、線形注意、状態空間モデルなど)とも比較されます。

最前線では、自己教師あり学習(MAE 系)との組み合わせ、検出・セグメンテーションのための統一バックボーン化、動画への時間軸拡張(局所・大域に加えて時間軸を多軸に足す)、任意解像度入力への適応、そして実機での高速化(カーネル融合や構造の簡素化、FlashAttention 系との統合)が継続的な研究テーマです。

横長で遠近の関係が重要な画像(前方カメラ画像など、消失点付近の遠方と手前の路面の関係)では、Grid 注意のように画像全体を疎に見る機構が理論上相性がよく、応用研究の対象になっています。一般に、画像内に大域的な幾何構造(地平線、車線の収束、シーンの左右対称性)があるドメインでは、最初の段から大域関係を直接見られる MaxViT 型の設計が有利になりやすいと考えられます。
