# MAE (Masked Autoencoders)

## ひとことで言うと
画像の大部分(典型的には75%)を隠して見えなくし、残ったわずかな部分だけから「隠した部分のピクセルは何だったか」を当てさせる訓練方法です。人間が正解ラベルを付ける必要がなく、画像そのものを問題と答えに使えるので、大量の画像で「画像を理解する能力」を安く鍛えられます。これを自己教師あり事前学習(self-supervised pre-training)と呼びます。

## 直感的な理解
ディープラーニングの画像モデルは、性能を出すために大量の「ラベル付きデータ」を欲しがります。ラベルとは「この画像は猫」「これは車」といった正解の付箋です。しかし人間が手で正解を付ける作業は高コストで、枚数に限界があります。

そこで「ラベルがいらない学習」が求められました。発想は自然言語処理(NLP)から来ています。BERT(2018)というモデルは、文章の単語の一部を隠して「隠した単語を当てる」訓練(マスク言語モデリング)だけで、文章理解力を飛躍的に高めました。正解はもとの文章自身にあるので、人手のラベル付けが不要です。穴埋め問題を大量に解くうちに言語の構造を体得する、というイメージです。

「同じことを画像でやれないか」というのが MAE の出発点です。ただし画像はそのまま単語化できません。MAE は画像を小さなパッチ(タイル状の断片)に切り、その大半を隠し、残りから復元させる形で「画像版 BERT」を実現しました。背景には、ViT(Vision Transformer、画像をパッチ列として Transformer で処理するモデル)の登場で「画像をパッチの列として扱う」土台がすでに整っていたことがあります。

## 基礎: 前提となる概念
自己教師あり学習(self-supervised learning): データ自身から「正解」を自動生成して学ぶ枠組みです。穴埋め(マスク予測)や、同じ画像の2つの見え方を近づける対照学習(contrastive learning)などがあります。

ViT(Vision Transformer): 画像を固定サイズのパッチに分割し、各パッチを単語のように埋め込みベクトルにして Transformer に流すモデルです。Transformer は自己注意(self-attention、系列内の各要素が他の全要素を参照して重み付き和を取る機構)で要素間の関係を学びます。

エンコーダ / デコーダ: エンコーダは入力を内部表現(特徴)に変換する部分、デコーダはその内部表現から元の形(ここではピクセル)を復元する部分です。

線形評価(linear probing): 事前学習で得た特徴を固定したまま、その上に線形分類器(1層の全結合)だけを学習して性能を測る評価です。特徴そのものの良さ(どれだけ素直に分類できる形になっているか)を測ります。これに対しファインチューニングは全パラメータを目的タスクで追い込む評価です。

冗長性(redundancy): 画像は隣り合うピクセルが似ているため情報が空間的に重複しています。だから一部を隠しても周囲から推測できてしまう、という性質です。

## 仕組みを詳しく
MAE の処理を、例として 224×224 のカラー画像で順に追います。

ステップ1(パッチ分割): 画像を 16×16 ピクセルのパッチに切ります。224 ÷ 16 = 14 なので、14×14 = 196 個のパッチができます。各パッチは「画像の小さなタイル1枚」です。

ステップ2(高マスク率): 196 個のうち約75%(= 147 個)をランダムに捨て、残り25%(= 49 個)だけを残します。「マスク率75%」が MAE の特徴的な設計です。BERT は単語のマスク率15%程度ですが、画像はピクセルが空間的に冗長なので少し残すだけでは穴埋めが簡単すぎて学習になりません。だから大胆に75%隠して、難しい問題にします。

ステップ3(エンコーダ): ここが効率化の核心です。エンコーダ(ViT)には「残った49個のパッチだけ」を入れ、隠した147個は入力しません。Transformer の計算量は入力系列長にほぼ比例(自己注意は二乗)するので、25%しか入れないと計算が大幅に軽くなり、巨大なエンコーダでも安く訓練できます。出力は49個ぶんの特徴ベクトルです。

ステップ4(デコーダ): エンコーダ出力の49個に、隠した147箇所を表す共通の「マスクトークン」(中身は同じ学習可能ベクトル)を差し込み、合計196個の系列に戻します。各位置に「画像のどこか」を示す位置埋め込み(position embedding)を足し、小さめの Transformer デコーダに通して各パッチのピクセル値を予測します。デコーダはエンコーダより軽量(浅く狭く)で、事前学習が終わったら捨てる前提です。

ステップ5(損失): 予測したピクセルと本物のピクセルの差を、マスクした部分についてのみ二乗誤差(MSE)で測り、これを小さくするよう学習します。見えていた49パッチは損失に入れません(復元する意味がないため)。正規化済みピクセル(各パッチ内で平均・分散を揃えたもの)を目標にすると性能が上がることも報告されています。

形状で追うと: 画像 `[3,224,224]` → パッチ列 `[196, 768]`(768 = 16×16×3)→ マスク後 `[49, 768]` → エンコーダ出力 `[49, D]` → マスクトークン補完 `[196, D]` → デコーダ → 予測ピクセル `[196, 768]`。損失はこのうちマスクされた147個ぶんだけ。

事前学習後はデコーダを捨て、エンコーダだけを残します。このエンコーダは「画像の本質をよく捉える特徴抽出器」になっており、ラベル付きデータで分類・検出・セグメンテーションなどの本番タスクに少量データで微調整します。

```
[元画像] → パッチ化 → 75%マスク
   見える25% ──► [重いエンコーダ ViT] ──► 特徴
                                            │ + マスクトークン + 位置埋め込み
                                            ▼
                          [軽いデコーダ] ──► ピクセル予測 ──(マスク部のみMSE)── [元ピクセル]
```

## 手法の系譜と主要論文
BERT(Devlin et al., NAACL 2019, "BERT: Pre-training of Deep Bidirectional Transformers")。マスク予測の元祖(言語版)です。文の単語の一部を `[MASK]` に置換して当てさせる事前学習で、文章理解の多数のベンチマークを塗り替えました。「穴埋めだけで強い表現が育つ」というマスクモデリングの威力を確立しました。

BEiT(Bao et al., ICLR 2022, "BEiT: BERT Pre-Training of Image Transformers")。画像をピクセルではなく、事前に学習した離散トークナイザ(dVAE)で「視覚の単語」に変換し、マスクしたパッチのトークンを当てさせる手法です。MAE はこれを「離散トークンではなくピクセルを直接予測する」と単純化した、と位置づけられます。

MAE(He et al., CVPR 2022, arXiv:2111.06377, "Masked Autoencoders Are Scalable Vision Learners"、Kaiming He ら、Facebook AI Research)。提案は3点。(1)高マスク率(75%)、(2)非対称設計(重いエンコーダは見えるパッチだけ処理、軽いデコーダだけが全位置を扱う)、(3)ピクセル直接再構成。動機は「画像は冗長なので大胆に隠さないと学習が易しすぎる」「見えるパッチだけ処理すれば巨大モデルでも安く回せる」。効果は ViT-Huge のような巨大モデルを効率的に事前学習でき、微調整後に高精度を達成、かつスケーリングが素直に効くことです。

SimMIM(Xie et al., CVPR 2022, "SimMIM: A Simple Framework for Masked Image Modeling")。MAE とほぼ同時期の同種手法で、マスクトークンをエンコーダにも入れる単純な構成と、線形1層という極めて軽いデコーダ(予測ヘッド)でも有効であることを示しました。「凝った設計がなくてもマスク画像モデリングは効く」という頑健性の証拠です。

ConvNeXt V2 / FCMAE(Woo et al., CVPR 2023, "ConvNeXt V2")。MAE の発想を畳み込みネットに移植した FCMAE(Fully Convolutional Masked AutoEncoder)で CNN を自己教師あり事前学習し、GRN(Global Response Normalization)と組み合わせて精度を底上げしました。マスク画像モデリングが Transformer 専用ではないことを示しました(詳細は [ImageNet事前学習バックボーンの転移](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0356_imagenet-pretrained-backbone))。

対比として、対照学習系(SimCLR、MoCo、BYOL、SwAV)は「同じ画像の2つの見え方を近づける」アプローチで、MAE の「再構成」とは思想が異なります。MAE の利点は、負例(後述)や複雑なデータ拡張・特殊な正則化を必要とせず設計がシンプルな点です。

## 論文の実験結果(定量データ)
MAE 論文の主要数値(ImageNet-1k、ViT バックボーン、top-1 精度 = 最上位予測が正解と一致する割合、高いほど良い):

- ViT-Large をスクラッチ(事前学習なし)で学習すると top-1 はおよそ 82% 台に留まりますが、MAE で自己教師あり事前学習してから微調整すると約 85.9% に達したと報告されています。事前学習の効果が数ポイントという大きな差で現れました。
- ViT-Huge では微調整後 top-1 約 87.8% を達成し、当時 ImageNet-1k のみで学習した手法として最高水準でした。外部の巨大ラベル付きデータに頼らずここまで届く点が画期的でした。
- マスク率のアブレーションが象徴的です。マスク率を変えて性能を調べると、最適は 75% 前後で、低すぎる(問題が易しい)と性能が伸びず、高すぎる(情報が足りない)とやや落ちる、という凸の関係が示されました。言語の 15% より遥かに高いマスク率が画像では最適、という反直感的な発見です。
- 非対称設計の効果: 見えるパッチだけをエンコーダに入れることで、同じ計算予算で 3 倍以上の高速化が得られ、ViT-Huge の学習が現実的になったと報告されています。
- デコーダのアブレーション: デコーダは浅く狭くても微調整性能はほとんど落ちず、デコーダの設計は線形評価により効く(微調整には影響が小さい)ことが示されました。

転移評価では、COCO 物体検出・インスタンスセグメンテーション(物体ごとに領域を塗り分ける課題)、ADE20K セマンティックセグメンテーション(各ピクセルにクラスを付ける課題)で、教師あり事前学習を上回るスコアが報告され、MAE 表現が分類以外の密予測タスク(画素単位で出力を出すタスク)にも有効であることが示されています。具体的には ViT-Large で COCO の検出指標(AP、平均適合率。検出の正確さを 0〜100 で測り高いほど良い)が教師あり事前学習比で数ポイント改善、ADE20K の mIoU(平均 Intersection-over-Union。予測領域と正解領域の重なり率の平均、高いほど良い)でも改善が報告されています。一方で線形評価(特徴固定)では対照学習系(MoCo v3 など)にやや劣る場面があり、MAE は「微調整前提で真価を発揮する」手法だと論文自身が位置づけています。これはピクセル再構成が「見た目の細部」も拾うため、特徴がそのままでは線形分離しにくい形になりがちだから、と解釈されています。

学習効率の数値も印象的です。マスク率75%により系列長が4分の1になるため、ViT-Large の事前学習が、同等の精度を狙う対照学習や全パッチを使うマスク手法に比べて壁時計時間で3倍以上速いと報告されました。「巨大モデルを安く事前学習できる」ことが MAE のスケーラビリティの源です。

## メリット・トレードオフ・限界
メリット
- ラベル不要。画像そのものが教師になるので大量データを安く活用できる。
- 高マスク率 + 非対称設計で、巨大エンコーダでも計算が軽い(見えるパッチだけ処理)。
- 設計がシンプル。負例・複雑なデータ拡張・特殊な正則化が不要で再現しやすい。
- スケーリングが素直に効く(モデル・データを大きくするほど良くなる)。

トレードオフ・限界
- 微調整前提。線形評価では対照学習系に劣ることがある。特徴をそのまま固定して使う用途には向きにくい。
- ピクセル再構成は細部の見た目に引っ張られ、必ずしも高次の意味理解に直結しない場合がある。
- デコーダ・マスクトークン・位置埋め込みなど事前学習専用の部品が必要(本番では捨てる)。
- 主に ViT 向けに設計され、CNN へは FCMAE のような工夫(マスク領域を疎畳み込みで扱う)が必要。
- 未解決課題: 何を再構成目標にすべきか(生ピクセル vs 特徴 vs 離散トークン)、動画・3D・マルチモーダルへの最適なマスク戦略、再構成型と対照型・予測型([JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))の使い分けは依然として研究対象です。

## 発展トピック・研究の最前線
MAE の発想は静止画を超えて広がっています。VideoMAE / MAE-ST は時空間パッチをマスクして動画表現を学び、AudioMAE は音声スペクトログラムに適用、Point-MAE は点群に拡張しました。再構成目標を「ピクセル」から「特徴」に変える流れ(I-JEPA など [JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))は、ピクセル細部のぼやけ問題を回避し意味的な予測を狙うもので、MAE の正統な発展系の一つです。運転分野では、ラベル付けが特に高コストな走行映像から自己教師あり事前学習する候補として MAE 系が有力で、BEV やマルチカメラ・LiDAR へのマスクモデリング適用が研究されています。
