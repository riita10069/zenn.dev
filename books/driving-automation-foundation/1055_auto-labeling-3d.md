# オートラベリング (auto-labeling)

## ひとことで言うと
人間が手作業でデータに正解(ラベル)を書き込む代わりに、計算資源をたっぷり使った強力なモデルに自動でラベルを付けさせる手法です。特に3次元空間での物体の位置・大きさ・向き(3Dバウンディングボックス、以下「3D箱」)を、LiDAR(レーザーで周囲までの距離を測るセンサー)が出す点群に自動付与するのが代表例です。人手ラベルを桁で減らし、大規模なラベル付きデータを現実的なコストでそろえるための実務基盤になっています。狙いは「人間に匹敵する、場合によっては人間以上に安定した正解」を、人手をほとんど使わずに作ることです。

## 直感的な理解
データ駆動の機械学習では、教師あり学習(正解付きデータでモデルを訓練する方式)が依然として主力です。ところが自動運転の3D知覚では、その「正解」を作るコストが極端に高い。

3D箱を人手で付ける作業を想像してください。LiDARが吐き出すのは、空間に散らばった数万〜数十万個の点(各点が x, y, z 座標を持つ「点群 point cloud」)です。アノテーター(ラベルを付ける作業者)は、この点の塊を3次元ビューアでぐるぐる回しながら「ここに車が1台」「向きはこっち」「縦横高さはこのくらい」と直方体を1個ずつ合わせていきます。1フレーム(ある瞬間の1スキャン)に車・人・自転車が10〜20個あれば、数分〜十数分かかります。

数で実感してみます。代表的な大規模 LiDAR データセットには3D箱が1000万個オーダーで含まれると報告されています。仮に1箱あたり安く見積もって30秒で付けられたとしても、1200万個 × 30秒 ≈ 10万時間。1人が毎日8時間働いても約34年分です。これを人手だけでまかなうのは非現実的です。しかも自動運転車は1台が1日に数百〜数千キロ走り、その点群は際限なく増え続けます。人手は線形にしかスケールしないので、データ量の増加に追いつけません。

そこで「走行が終わったあと(オフライン)に、時間をかけてよいから巨大なモデルに自動でラベルを付けさせよう」という発想が出てきます。これがオートラベリングです。点群そのものは機械が大量に集められるので、ボトルネックはあくまで「正解を付ける人手」なのだ、という問題意識が出発点です。

## 基礎: 前提となる概念
理解の土台となる用語を、知らない前提で噛み砕きます。

- LiDAR と点群: LiDAR は周囲にレーザーを照射し、戻ってくるまでの時間から距離を測るセンサーです。1回転スキャンするごとに、空間に当たった点の集合(点群)が得られます。点は遠くほど疎(まばら)になり、遮蔽(別の物体の影)に入った部分には点が来ません。例えば50m先の歩行者には点が数個しか当たらないこともあり、それだけでは人か標識柱か区別が困難です。
- フレームとシーケンス: ある瞬間の1スキャンが「フレーム」、それを時系列に並べたものが「シーケンス(連続フレーム列)」です。10Hzなら1秒で10フレーム。
- オンボード (onboard) モデル: 実車に積んで走行中にリアルタイムで動かすモデル。遅延制約が厳しく(例えば100ミリ秒以内に答えを出さねばならない)、未来の情報は使えません(まだ起きていないから)。精度には原理的な限界があります。
- オフボード (offboard) モデル: 走行後にサーバー上で動かすモデル。リアルタイム制約がなく、シーン全体の過去・未来の時間情報をすべて使えます。だから同じ点群でも、ずっと高精度な箱を出せます。
- 擬似ラベル (pseudo-label): モデルの予測を「仮の正解」として扱うこと。オートラベリングの核心はまさに「オフボードモデルの高精度な予測を擬似ラベルとして採用する」ことにあります。
- 半教師あり学習 (semi-supervised learning): 少量のラベル付きデータと大量のラベルなしデータを併用してモデルを鍛える枠組み。擬似ラベルはその主要な道具立ての一つです。

このオンボード/オフボードの非対称性こそが、オートラベリングが成立する根拠です。オフボードは「カンニングし放題(未来を見られる、何度でも計算できる)」なので、オンボードより良い答えを出せる。その良い答えを正解として配って、軽量なオンボードモデルを鍛えるわけです。重要なのは「教師は速くなくてよい」という割り切りで、これによりオフボード側は何十フレームも束ねた巨大入力や、何度も推論を繰り返す重い処理を許容できます。

## 仕組みを詳しく
代表手法 3DAL (3D Auto Labeling, Qi et al., CVPR 2021) を例に、処理を段階で分解します。

### ステップ1: シーン全体を一気に見る
走行中のオンボードモデルは「今この瞬間」の1フレームしか見られません(せいぜい数フレームのバッファ)。オフボードモデルは、ある車が映っている全フレーム(例えば5秒間=50フレーム)をまとめて扱えます。これにより、ある瞬間だけ点が少なくて見えにくくても、前後フレームの情報で補完できます。これは「未来を見る」ことに相当し、原理的にオンボードでは不可能な情報利得です。

### ステップ2: 物体ごとに点群を集めてトラック化する
まず通常の3D検出器(例: PointPillars や CenterPoint)を各フレームに走らせて箱の候補を出し、同じ物体を時間方向につなげて「トラック(軌跡)」にします。例えば「フレーム1〜50に出てくるこの1台」を1本のトラックとしてまとめます。3DAL はこのトラック単位で後段の処理を行います。トラック化(マルチオブジェクトトラッキング)自体が前段に必要で、ここでIDを取り違えると後段の重ね合わせが破綻する、という依存関係があります。

### ステップ3: 物体中心で点群を重ね合わせる(object-centric refinement)
3DAL の中核的な工夫です。物体は動いている前提で、その物体の中心座標系に各フレームの点を変換してから重ね合わせます。

- 静止物(駐車中の車、標識): 全フレームの点をそのまま重ねれば、点密度が一気に上がり、形がくっきり出ます。
- 動く物体(走行中の車): フレームごとに位置が違うので、まず物体の動き(軌跡)を推定し、「その物体から見た座標系」に各フレームの点を揃えてから重ねます。すると、ばらばらだった点が1つの密な車形に集約されます。

数値イメージで言うと、1フレームでは車に点が50個しか当たっていなくても、50フレーム分を物体中心で重ねれば数千点の密な点群になり、車の正確な寸法・向きを推定しやすくなります。3DAL はさらに「軌跡を整える静的ブランチ/動的ブランチ」を分け、運動量(物体は急に瞬間移動しない、寸法は時間で変わらない)を考慮して箱を時間方向に滑らかにします。

ASCII でイメージすると以下です。

```
オンボード(1フレーム):        オフボード(50フレーム重ね合わせ後):
   ・   ・                          ・・・・・・・・・
      ・        →(疎すぎて          ・・・[ 車 ]・・・   →(密で形が明瞭、
   ・      箱が不確か)               ・・・・・・・・・      箱が高精度)
```

### ステップ4: 高精度な箱を最終ラベルとして出力する
重ね合わせた密な点群から、専用ネットワーク(box scoring / refinement network)が各フレームの最終的な3D箱(位置 (x,y,z)・サイズ (l,w,h)・向き(ヨー角 θ))を出します。これが人手ラベルの代わりになります。実装上は、各フレームの箱を独立に出すのではなく、トラック全体で一貫させる(寸法を共有し、軌跡を平滑化する)ことで時間方向のがたつきを抑えます。

## 手法の系譜と主要論文
オートラベリングは「擬似ラベル/自己訓練」という古い思想が、3D知覚という重い問題に応用されて結実したものです。系譜をたどります。

### 知識蒸留 (Hinton, Vinyals & Dean, NeurIPS Workshop 2015)
「高精度だが重いモデル(教師)の知識を、軽いモデル(生徒)に移す」枠組み。教師の出力(ソフトな確率分布)を生徒の学習目標に使います。オートラベリングは「オフボードの正確さ」を「オンボードの速さ」に移す手段であり、思想的にこの蒸留の延長線上にあります。

### 自己訓練/Noisy Student (Xie et al., CVPR 2020)
画像分類で、教師モデルがラベルなし画像に擬似ラベルを付け、ノイズ(データ拡張・dropout・stochastic depth)を加えた生徒を訓練し、生徒を新しい教師にして反復する手法。ImageNet で当時の最高精度 88.4% top-1 を達成し、ラベルなしの追加データ(JFT-300M)が効くことを示しました。ここで「教師が間違えると、その間違いが正解として伝播し、モデルが誤りを強化する(確証バイアス confirmation bias)」という危険が広く認識されました。オートラベリングで低信頼ラベルを除外したり人手抜き取り検査をするのは、この危険への対処です。

### FixMatch (Sohn et al., NeurIPS 2020)
半教師あり学習で「弱い拡張で擬似ラベルを作り、高信頼なものだけ採用して、強い拡張をかけた入力でその擬似ラベルに合わせる」一貫性正則化を確立。擬似ラベルに信頼度しきい値を設ける設計の代表で、3D 自動ラベリングの「低信頼を捨てる」運用の理論的後ろ盾になっています。

### 3DAL / Offboard 3D Object Detection (Qi et al., CVPR 2021)
3D 点群シーケンス向けのオートラベリングを定式化した代表作。上述の object-centric な多フレーム重ね合わせで、オフボードの自動ラベルが人手品質に迫ることを示しました。動機は、大規模 LiDAR に人手で箱を付けるのが非現実的で、かつオンボードでは時間情報を使い切れず精度が頭打ちになること。

### Auto4D (Yang et al., 2021, Uber ATG)
箱の「4D」(3D空間 + 時間)を一貫して整える方向の研究。トラック全体で寸法を1つに固定し(物体の大きさは時間で変わらないという制約)、軌跡を滑らかにすることで時間方向の一貫性を高めました。「object size branch」と「motion path branch」を分ける設計が特徴で、3DAL と並ぶ初期の重要研究です。

### スケーラブル擬似ラベル (Caine et al., 2021)
半教師あり3D検出として、少量の人手ラベル + 大量の擬似ラベルで検出器を底上げする方向。教師・生徒を別アーキテクチャにし、擬似ラベルのソフトさ(信頼度の連続値)を活かす設計を検討。オートラベリングを「ラベル効率」の観点で位置づけ直した流れです。

現在の潮流: 近年はカメラ・LiDAR・地図を統合したマルチモーダルなオフボード自動ラベリング、視覚言語モデル(VLM)や基盤モデルを使った「ゼロショット/弱教師」自動ラベリング、占有グリッド(occupancy)やシーンフロー(各点の運動)まで自動付与する方向へ広がっています。

## 論文の実験結果(定量データ)
ここでは指標の意味から説明します。3D 検出の主要指標は mAP / mAPH です。AP (Average Precision) は「検出の正確さ(precision)と網羅性(recall)のバランス」を1つの数字にまとめたもので、0〜100% で大きいほど良い。Waymo の mAPH はこれに「向き(heading)の正しさ」で重み付けしたもので、箱の向きまで合っていないと得点が下がります。検出が正解とみなされる条件は IoU(箱の重なり率、Intersection over Union)が閾値(車なら0.7)以上であることです。さらに難易度 LEVEL_1(点が5個以上当たる比較的見える物体)/ LEVEL_2(点が1個以上、ほぼ見えない難物体も含む)に分けて報告されます。

3DAL の報告(Waymo Open Dataset, 車クラス)では、おおよそ次のような傾向が示されています。

- 自動ラベルの品質: オフボード自動ラベルは、人間アノテーター同士のばらつき(human-level variance、2人の人間が同じ物体に付けた箱のズレ)と同程度の品質に達した、と報告されています。言い換えると「自動ラベルと人手ラベルのズレ」が「人間どうしのズレ」と同水準で、実質的に人手品質に並んだということです。
- オンボード単一フレーム検出器との比較: 同じ点群でも、シーケンス全体を使う 3DAL はオフボードで大幅に高い mAPH を達成し、報告では難物体・遠距離・遮蔽下(LEVEL_2)で特に差が開きました(時間情報の補完が効くため)。
- 蒸留効果: 3DAL の自動ラベルを使って学習した生徒検出器が、少ない人手ラベルしか使わない場合より性能を改善。ラベルなしデータを擬似ラベル化して足すと検出器が底上げされることを示しました。Caine et al. の半教師あり設定でも、わずかな人手ラベル(全体の数%〜10%程度)に大量の擬似ラベルを足すと、フルラベルに近い検出性能が出ることが報告されています。

アブレーション(どの要素を抜くとどうなるか)の知見:
- 多フレーム重ね合わせを外して単一フレームに戻すと、点密度が下がり箱精度が明確に低下する。これは「時間情報こそが効いている」ことの裏付けです。重ねるフレーム数を増やすほど密度が上がり精度が上がりますが、フレームを増やしすぎると古い時刻の位置推定誤差が混入して飽和・劣化する、というトレードオフも観測されています。
- 動的物体の運動補償(物体中心座標への変換)を外すと、走行中の車のように動く対象で点がぶれて重ならず、寸法・向きの推定が悪化する。静止物では影響が小さいことから、補償が動物体に効いていると分かります。

なお、Auto4D 系の研究では「トラック全体で寸法を1つに固定する」制約を入れると時間方向の箱のがたつき(フレーム間での寸法の揺れ)が減り、軌跡の滑らかさ指標が改善することが報告されています。

## メリット・トレードオフ・限界

メリット
- 人手ラベルのコスト・時間を桁で削減できる。
- オフボードならではの時間情報で、人手以上に安定した箱が得られる場合がある(特に遮蔽・遠距離)。
- 大規模ラベル前提の手法(知識蒸留、半教師あり、大規模事前学習)を実用化できる。
- スケールする: 計算資源を足せばラベル生産量が増える。人手のように線形の人件費でなく、クラウド計算で弾力的に増減できる。

トレードオフ・限界
- 確証バイアス: 自動ラベルの誤りがそのまま学習に伝播する。教師が苦手な対象(混雑する歩行者群、極端な遮蔽、高速移動、希少クラス)では誤りが蓄積しやすい。信頼度フィルタや人手抜き取り検査が必須。
- 計算・保存コスト: シーン全体を処理するため重く、走行後のバッチ処理が前提。トラック単位で何度も推論するため、1走行あたりの計算は単一フレーム検出の何倍にもなる。
- 偏りの固定化: 「自動ラベルで学んだモデルを、また自動ラベリングに使う」と、教師の盲点(検出できなかった対象)が永久に学習されないまま固定される危険がある。
- 評価の難しさ: 自動ラベルそのものに誤りがあるため、「自動ラベルを正解にした評価」は楽観的になりがち。最終評価には人手の保留検証セット(自動ラベルが一切混ざらない正解)が要る。
- 前段依存: トラッキングのID取り違えや、初段検出の見落としがそのまま伝播する。物体を一度も検出できなければ、いくらフレームを重ねてもラベルは生まれない。

未解決の課題: 希少だが安全上重要な事象(long-tail、例えば道路上の落下物や予測外の歩行者挙動)は、教師モデルがそもそも見落とすため自動ラベリングで取りこぼしやすい。ここを能動学習(モデルが自信のないデータを優先的に人手へ回す)やマイニングで補う研究が続いています。

## 発展トピック・研究の最前線
- マルチモーダル自動ラベリング: LiDAR に加えカメラ・レーダー・地図を融合し、見えにくい対象(遠方・小さい物体)の自動ラベル精度を上げる。カメラの見た目情報が「点が少なくても人だと分かる」補完を与える。
- 基盤モデル/VLM 活用: 大規模事前学習モデルやオープン語彙検出(SAM, Grounding DINO 系)を使い、人手定義のクラスに縛られない弱教師・ゼロショットの自動ラベリング。「珍しい物体」をテキスト指定で拾える可能性。
- 占有・シーンフローへの拡張: 箱だけでなく、空間の占有状態(occupancy)や各点の運動(scene flow)まで自動付与する方向(End-to-End な運転モデルが中間表現として欲しがる信号)。
- 不確実性つき自動ラベル: 各ラベルに信頼度を付け、学習側でその信頼度を使って損失を重み付けし、誤伝播を抑える。
- ループ運用(データエンジン): 自動ラベル → 学習 → 性能の悪い領域をマイニング → そこだけ人手 → 再学習、という継続的改善ループ。Tesla の Data Engine や各社の closed-loop データ運用の中核思想。
