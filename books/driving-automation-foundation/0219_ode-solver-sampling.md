# ODEソルバによる軌道サンプリング

## ひとことで言うと
Flow Matching (フローマッチング) という生成モデルでは、学習済みネットワークが「いまの状態を見て、データらしくなる方向へどれだけ速く動くべきか」という速度ベクトル場を出力します。推論ではランダムなノイズから出発し、その速度に従って少しずつ前へ進む計算を何回か繰り返し、最終的なデータ (たとえば車の軌道) を作ります。この「少しずつ進む」計算が ODE ソルバ (常微分方程式の数値解法) で、代表的なのはオイラー法とホイン法です。本トピックは生成モデルの「学習」ではなく「推論をどう実行するか」を扱います。

## 直感的な理解
紙地図の上で、目的地は分からないが「いまいる地点ではこっちへ歩け、向きはこれ」という矢印だけが各地点に描いてある、という状況を想像してください。矢印 (速度場) に従って一歩進み、新しい地点でまた矢印を読み、また一歩進む。これを繰り返せば、矢印しか知らなくても目的地にたどり着けます。生成モデルの推論はこれと同じです。最初の地点はランダムなノイズで、各地点の矢印はニューラルネットが教えてくれます。

なぜ「最終地点を一発で出す」のではなく「矢印に従って何度も進む」必要があるのか。それは、現実の変換 (ノイズ→本物のデータ) が一直線では表せない曲がった道だからです。曲がった道を一本の直線で近似すると大きくずれます。途中の地点で何度も向きを確認し直すことで、曲がりに追従できる。確認の回数 (ステップ数) を増やすほど正確になりますが、毎回ネットワークを呼ぶので遅くなります。この「速さ vs 正確さ」のつまみが ODE ソルバの本質です。

自動運転の軌道生成でこれが効くのは、「次にどう動くか」の正解が一つに決まらないからです。交差点では直進も右折もありうる。決定的に 1 本だけ出すモデルは、複数の正解の平均 (どっちつかずの中途半端な軌道) を出してしまいがちです。生成モデルはノイズを変えるたびに違う軌道を引けるので、この多峰性 (複数の山がある分布) を素直に扱えます。その「引く」操作の実体が ODE 積分です。

## 基礎: 前提となる概念

まず Flow Matching を説明します。これは「ランダムなノイズ」を「本物らしいデータ」へ変換する生成モデルの一種です。発想はこうです。完全なノイズ x0 (正規分布から引いたランダムなテンソル) と、本物のデータ x1 の間を、まっすぐな直線でつなぎます。

```
x_t = (1 - t) * x0 + t * x1     (t は 0 から 1 の連続値)
```

ここで t は「変換の進み具合」を表すパラメータです。t=0 ならノイズそのもの、t=1 なら本物そのもの。この直線上を一定速度で進むと考えると、各点での「進むべき速度」は単純に終点マイナス始点、つまり x1 - x0 です (直線なので速度は途中で変わりません)。

学習では、ネットワーク v_theta (theta は学習可能なパラメータ) に「いまノイズ混じりの状態 x_t と時刻 t を見たとき、進むべき速度はどちらか」を当てさせます。教師信号は x1 - x0、損失はその二乗誤差 (予測した速度と正解速度の差を二乗して平均したもの) です。式で書くと、

```
L = E[ || v_theta(x_t, t) - (x1 - x0) ||^2 ]
```

E は期待値 (たくさんのサンプルにわたる平均) を表します。学習はこれだけで終わります。拡散モデル (Diffusion Model) のように複雑なノイズスケジュールや時刻ごとの重み付けを設計しなくてよいのが Flow Matching の利点です。

ここで重要なのが「常微分方程式 (ODE: Ordinary Differential Equation)」という言葉です。ODE とは「ある量の時間変化率 (微分) が、その量と時刻の関数で与えられている」という形の方程式です。Flow Matching の場合、

```
dx/dt = v_theta(x, t, c)
```

と書けます。dx/dt は「x が時刻 t に対してどれだけ速く変化するか」、c は条件 (画像特徴・自車状態・地図など、生成を方向づける外部情報) です。日本語で言えば「x の変化率はネットワーク v_theta が出す速度で決まる。出発点 x(0) はノイズ。では x(1) はいくつになるか」を求める問題です。ニューラルネットが中に入っているので手計算では解けません。だからコンピュータで近似的に解く。それが ODE ソルバ (数値積分器) で、本トピックの主役です。

なぜ「ODE」であって「SDE (確率微分方程式)」でないのか、も押さえておきます。拡散モデルのサンプリングには、各ステップでランダムなノイズを足す確率的なやり方 (SDE) と、ノイズを足さず決まった軌跡を辿る決定的なやり方 (ODE) があります。Song et al. (2021) が示したのは、同じ分布を生む決定的な「確率フロー ODE (probability flow ODE)」が必ず存在するということです。Flow Matching はこの ODE 版を最初から直接学習している、と理解できます。ODE は同じノイズから始めれば常に同じ結果を出す (再現性がある) ので、安全が絡む応用では扱いやすい一方、出力の多様性は SDE より控えめになります。

## 仕組みを詳しく

最も単純な ODE ソルバが「オイラー法 (Euler method)」です。区間 [0, 1] を N 個の小さなステップに分け、各ステップで「現在の速度 × 刻み幅」だけ前へ進めます。

```
dt = 1 / N
x  = ノイズサンプル (正規分布から)
for step in range(N):
    t = step * dt
    v = v_theta(x, t, c)      # いまの位置・時刻での速度をネットワークに聞く
    x = x + dt * v            # その速度で少しだけ進む
return x                      # t=1 に到達した x が最終出力
```

数値例。N=10 なら dt=0.1 で 10 回ループします。各ステップでネットワークを 1 回呼ぶので、推論コストは「ネットワーク呼び出し N 回」です。この呼び出し回数を NFE (Number of Function Evaluations、関数評価回数) と呼びます。N を大きくすると刻みが細かくなり積分が正確になりますが、その分 NFE が増えて遅くなります。N を小さくすると速いですが、曲がった軌道を直線で近似する誤差 (離散化誤差、truncation error) が増えます。これが本質的なトレードオフです。

ここで Flow Matching の利点が効きます。学習時に「ノイズと本物をまっすぐな直線で結ぶ」ように設計したので、解くべき ODE の軌跡もできるだけ直線に近くなります。直線に近い軌跡は粗いステップ (N=5〜20 程度) でも正確に積分できる。画像拡散モデルが当初は何百ステップ必要だったのに対し、Flow Matching が少ステップで済むのはこの理由です。これを徹底的に追求したのが後述の Rectified Flow です。

次に「ホイン法 (Heun's method)」。オイラー法は「区間の始点の速度だけ」で進むので、速度が途中で変わるとずれます。ホイン法はまず始点の速度で仮に一歩進み (予測子、predictor)、進んだ先の速度も計算して、両者の平均で本当に進みます (修正子、corrector)。

```
v1 = v_theta(x, t)             # 始点の速度
x_pred = x + dt * v1           # 仮に一歩進む
v2 = v_theta(x_pred, t + dt)   # 進んだ先での速度
x  = x + dt * (v1 + v2) / 2    # 始点と終点の速度を平均して進む
```

ホイン法は 1 ステップでネットワークを 2 回呼ぶので NFE が 2 倍ですが、同じ精度ならオイラー法より少ないステップで済むことが多く、軌道が曲がっているときに有利です。数値解析の言葉で、オイラー法は「1 次精度」(ステップ幅 dt を半分にすると誤差がおよそ半分)、ホイン法は「2 次精度」(dt を半分にすると誤差がおよそ 1/4)、さらに高精度な RK4 (4 次のルンゲ・クッタ法、1 ステップ 4 回呼び) は誤差が dt の 4 乗で減ります。ただし RK4 は NFE も 4 倍です。

ASCII で 1 次と 2 次の違いを描くと、曲がった真の軌跡に対し、

```
真の軌跡:    ●━━╮
                 ╰━━●
オイラー(1次):   ●─────●   始点の向きで突っ切る → 曲がりを見落としずれる
ホイン(2次):     ●──╮
                    ╰──●   両端の向きの平均で進む → 曲がりに追従
```

「精度の次数」と「同じ NFE 予算でどちらが得か」の関係は実用上きわめて重要です。NFE 予算がきつい (たとえば 4〜10 程度しか呼べない) 領域では、刻みが粗くなるので 2 次ソルバの離散化誤差の小ささが効き、ホインが勝ちやすい。逆に NFE を潤沢に取れるなら、オイラーでも刻みが十分細かくなって両者の差は消えます。車載のように「呼び出し回数を絞りたい」場面ほど、高次ソルバの恩恵が大きいわけです。

条件付き生成では、ネットワークに毎ステップ条件 c も渡します。ここで重要な最適化があります。条件のうちステップ間で変わらない部分 (画像/BEV 特徴のエンコードや、時刻に依存しないモジュレーション条件) は、ループの外で一度だけ計算し、N 回のステップで使い回します。こうすると毎ステップのコストが「速度を出す軽いヘッドの計算」だけになり、推論が大幅に速くなります。多くの実装で採られる定石です。tensor shape の例を挙げると、出力 x の形は [B, trajectory_dim] で、軌道を「64 ステップ × (加速度, 曲率) の 2 信号」で表すなら [B, 128] になります。B はバッチサイズです。

数値例で積分の途中を追ってみます。1 次元の簡単な ODE dx/dt = v で、始点 x=0.0、N=4 (dt=0.25)、各ステップでネットワークが順に v=1.0, 0.8, 0.6, 0.4 を返したとします。オイラー法では x は 0.0 → 0.25 → 0.45 → 0.60 → 0.70 と進みます。各行が「いまの x に dt*v を足す」操作です。ホイン法ならさらに各ステップで終点側の速度も読み、平均を取るぶん真の積分に近づきます。実際のネットワークでは v が状態 x にも依存するので、進んだ先で速度が変わる効果がもっと顕著に出ます。

## 手法の系譜と主要論文

ODE ソルバそのものは古典数値解析です。オイラー法 (Leonhard Euler, 18 世紀)、ホイン法、ルンゲ・クッタ法 (Carl Runge 1895, Wilhelm Kutta 1901) が標準で、誤差解析も確立しています。生成モデルへの応用が新しい流れです。

(1) Song, Sohl-Dickstein, Kingma, Kumar, Ermon, Poole, "Score-Based Generative Modeling through Stochastic Differential Equations" (ICLR 2021, arXiv:2011.13456)。拡散モデルのサンプリングを SDE として定式化し、それと同じ分布を生む決定的な確率フロー ODE が存在することを示しました。これにより「拡散モデルの推論 = ODE を解く」という視点が確立し、ODE ソルバの選び方が品質と速度を左右するようになりました。

(2) Karras, Aittala, Aila, Laine, "Elucidating the Design Space of Diffusion-Based Generative Models" (EDM, NeurIPS 2022, arXiv:2206.00364)。拡散モデルの設計空間を整理し、サンプリングにホイン法ベースの 2 次ソルバを使うと少ないステップでも高品質になることを実験的に示しました。「ソルバ選択は品質を決める設計要素である」と明確にした点が重要です。

(3) Lipman, Chen, Ben-Hamu, Nickel, Le, "Flow Matching for Generative Modeling" (ICLR 2023, arXiv:2210.02747)。確率経路に沿った速度場をシミュレーション無しで回帰する学習法 (Conditional Flow Matching) を提案。特に最適輸送 (Optimal Transport) に基づく直線経路を使う版を示しました。直線にする動機は、推論時の ODE 積分が少ないステップで済みサンプリングが速くなるからです。

(4) Liu, Gong, Liu, "Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow" (ICLR 2023, arXiv:2209.03003)。同時期に独立に「経路を直線化する」アイデアを提案。さらに学習を繰り返して経路を一段ずつまっすぐにする「reflow」という手続きを導入し、最終的に 1 ステップ (NFE=1) でも高品質な生成を狙えることを示しました。Flow Matching と Rectified Flow は表現は違えど中身が近く、「直線経路 + ODE 積分」の現代的潮流を作りました。

(5) 蒸留による少ステップ化。Song, Dhariwal, Chen, Sutskever, "Consistency Models" (ICML 2023, arXiv:2303.01469) は、確率フロー ODE の軌跡上の任意の点を一気に始点 (=データ) へ写す関数を学び、1〜数ステップ生成を可能にしました。ODE 積分を「逐次に解く」から「一発で写像する」へ置き換える発想で、少ステップ化研究の中心になっています。

(6) ロボティクス・運転応用。Chi et al., "Diffusion Policy" (RSS 2023, arXiv:2303.04137) は行動 (action) の生成に拡散を使い、推論で反復ソルバを回しました。Black et al. の pi-0 (Physical Intelligence, 2024, arXiv:2410.24164) は Flow Matching ベースのアクション生成を VLA (Vision-Language-Action) に組み込み、高頻度の連続制御を生成しました。自動運転の軌道生成でも拡散/Flow Matching を採用する研究が増えています。共通する動機は「行動・軌道の分布は多峰なので、決定的な 1 出力より分布をサンプリングしたい」こと。共通する課題は「推論が複数ステップ必要で、10〜数十 Hz のリアルタイム制約との両立にステップ数調整が要る」ことです。

## 論文の実験結果(定量データ)

Flow Matching / Rectified Flow 系の実験で繰り返し報告されるのは「少ない NFE で高品質」です。画像生成での品質指標の定番は FID (Fréchet Inception Distance) で、生成画像の分布と本物画像の分布の距離を測ります。値が小さいほど本物に近く、良い。

Lipman et al. (2023) は ImageNet 64x64 などで、同等品質を従来の拡散モデルより少ない関数評価回数で達成したと報告しています。主張の核は「直線に近い経路を学ぶと、オイラー法でもステップ数を減らせる」ことです。

Liu et al. (Rectified Flow, 2023) は CIFAR-10 で、reflow を適用したのち蒸留すると NFE=1 (1 回のネットワーク評価) で FID がおよそ 4 前後まで下がったと報告しています。比較として、素朴な拡散モデルが同等 FID に到達するには数十〜数百ステップを要します。これは「経路をまっすぐにすると ODE が易しくなり、極端には 1 ステップで済む」ことの定量的な裏付けです。

Karras et al. (EDM, 2022) は CIFAR-10 で当時の最高水準の FID (無条件生成でおよそ 1.79) を達成し、その際にホイン法ベースの 2 次ソルバが少ステップ高品質に寄与することを示しました。アブレーション (ある要素を抜くとどうなるかの検証) では、同じ NFE 予算でオイラー法 (1 次) よりホイン法 (2 次) が低い FID を出す、つまり「同じ計算量なら 2 次ソルバが得」という傾向が示されています。報告では NFE を 35〜79 程度に絞っても高品質を保ち、逆に NFE を十分大きく取れば両ソルバの差は縮みます。これは数値解析の理論 (高次ソルバは粗いステップで有利、細かくすれば差が消える) と整合します。

Consistency Models (Song et al., 2023) は CIFAR-10 で 1 ステップ生成の FID がおよそ 3.5 前後、2 ステップでさらに改善し、数百ステップの拡散に匹敵する品質を「1〜2 回の評価」で出せることを示しました。これは推論コストを 2 桁減らせることを意味し、リアルタイム応用への道を開きました。

運転・ロボ応用では、Diffusion Policy (Chi et al., 2023) が複数の操作タスクで成功率を従来手法 (明示的方策・暗黙的方策・行動クローニング) 比でおよそ平均 +20 ポイント以上改善したと報告し、多峰な行動を扱える生成方策の有効性を示しました。ただし推論ステップ数 (拡散の反復) が遅延に直結するため、DDIM などの加速ソルバや少ステップ化が実運用上の鍵になる、という議論が続いています。これは軌道生成プランナでも同じで、ステップ数を変えたときの遅延と滑らかさの実測は別途ベンチマークで確認します ([プランナー遅延/滑らかさベンチマーク](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0696_planner-latency-smoothness-benchmark))。

## メリット・トレードオフ・限界

メリット
- 速度場さえ学習できれば、ノイズから多様で滑らかなデータ (軌道) を生成できる。多峰な分布を素直に扱える。
- ステップ数 N で「速度と精度」を推論時に自由に調整できる (再学習不要)。
- Flow Matching/Rectified Flow の直線経路は積分が容易で、少ステップでも品質が高い。
- 条件のうち不変部分を使い回すことで、ステップ単価を下げられる。
- ODE (決定的) なので、同じノイズシードで同じ出力が再現でき、デバッグ・安全評価がしやすい。

トレードオフ・限界
- 1 サンプル生成にネットワークを N 回呼ぶので、決定的な 1 ショット出力より遅い。
- オイラー法はステップを粗くすると離散化誤差が出る。曲がった軌跡で顕著。
- ホイン/RK4 は精度が上がるが NFE が増え、実装も複雑になる。
- 確率的サンプリングなので、再現性確保には乱数シード管理が要る。
- 蒸留で 1〜数ステップ化すると、追加学習コストがかかり、品質がわずかに劣化したり多様性が落ちたりするリスクがある。
- 未解決の課題: リアルタイム制約 (例 10〜数十 Hz) 下で品質を保ったまま NFE をどこまで減らせるか。安全が絡む運転では「サンプルごとに出力が揺れる確率性」をどう制御・保証するか (決定論との折り合い) が研究上の論点。

## 発展トピック・研究の最前線
- 少ステップ化・蒸留: Consistency Models、Consistency Trajectory Models、Distribution Matching Distillation、reflow による経路直線化。NFE を 1〜4 まで削る方向が活発。
- 適応ステップソルバ: 誤差を見て刻み幅を自動調整する適応 RK (Dormand-Prince など)。固定ステップより効率的だが、車載のような決め打ち遅延が求められる環境では固定ステップが好まれる。
- SDE vs ODE: 確率的サンプリング (SDE) は多様性が増えるが遅延が読みにくく、決定的 (ODE) は再現性と速度に優れる。タスクに応じた使い分けが議論されている。
- 条件付け強度の制御: classifier-free guidance に相当する誘導を Flow Matching でどう入れ、guidance を上げたときの軌道の鋭さ・多様性のトレードオフをどう扱うか。
- 行動・軌道生成への特化: 制御周波数に合わせた action chunking (まとまった行動列を一度に生成) と少ステップソルバの組合せ、運動学的実行可能性を満たす制約付き生成。
