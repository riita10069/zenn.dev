# DreamerV3

## ひとことで言うと

DreamerV3 は、「ハイパーパラメータ(学習設定の調整つまみ)をタスクごとに変えずに、たった1つの固定設定のままで150以上の多種多様なタスクを解く」ことを実証した汎用ワールドモデル強化学習です。連続制御・Atari・3Dナビゲーション・ロボット、そして「Minecraft でダイヤモンドを掘り当てる」という極めて難しい長期・まばら報酬タスクまで、同じ設定で達成しました。さらに「モデルを大きくするほど性能が上がり、しかもデータ効率まで良くなる」というスケール則を示し、ワールドモデルの汎用性と拡張性を確立しました。2025年に Nature 誌に掲載されています。

## 直感的な理解

これまでの強化学習エージェントは、タスクが変わるたびに人間が職人芸で設定を合わせ直す必要がありました。報酬の大きさが0.01のタスクもあれば10000のタスクもある。観測が画像のこともセンサー値(proprioception)のこともある。報酬が毎ステップもらえるタスクもあれば、Minecraft のように何千ステップも行動してやっと1回もらえる「まばらな報酬(sparse reward)」のタスクもある。この多様さに毎回手で合わせるのは現実的ではなく、再現性も損なわれます。

DreamerV3 のアイデアは、「入力や報酬がどんなスケール・分布でも、自動で扱いやすく整えてしまえばよい」というものです。料理に例えると、どんな食材が来ても下ごしらえ(正規化・変換)を自動で行うので、レシピ(ハイパーパラメータ)を変えずに同じ手順で調理できる、という発想です。一つ一つの工夫(symlog、twohot など)は地味な「数値を整える前処理」ですが、それらを丁寧に積み重ねることで、設定固定のまま幅広いタスクで破綻しない頑健性が生まれます。

## 基礎: 前提となる概念

### ハイパーパラメータと汎用性

ハイパーパラメータは学習率や損失の重みなど、学習前に人間が決める設定値です。タスクごとに最適値が変わるため、従来は調整(grid search 等)が必要でした。「単一設定であらゆるタスクを解く」とは、このチューニングを不要にすることで、RL の再現性・実用性に関わる長年の課題への回答です。チューニングには大量の計算と試行錯誤が要るため、これを消せる意義は大きいです。

### スケール則

スケール則(scaling law)は「モデルや計算を大きくするほど性能が予測可能に向上する」経験則です。大規模言語モデルで有名ですが、強化学習では「大きくすると不安定になる/過学習する」ことも多く、世界モデル RL でスケール則が成り立つかは未知でした。DreamerV3 はここに肯定的な答えを与えました。

### 想像ロールアウトと RSSM

基本骨格は [RSSM (Recurrent State-Space Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0088_rssm_note) の離散潜在 + 想像ロールアウト上での actor-critic 学習で、[DreamerV2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0048_dreamer-v2) を引き継ぎます。学習済み世界モデルの中で未来を想像し、その中で方策を磨く点は V1/V2 と共通です([Dreamer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0047_dreamer-v1) 参照)。新規性は骨格ではなく「あらゆるスケール・分布を吸収する頑健化」の側にあります。

## 仕組みを詳しく

新しいのは「あらゆるスケール・分布の入力と報酬を自動で扱いやすく整える」一連の頑健化テクニックです。

### 1. symlog 変換(対称対数変換)

報酬や観測の大きさがタスクで桁違いに違う問題に対し、symlog を使います。

```
symlog(x) = sign(x) · ln(1 + |x|)
```

sign(x) は符号(プラスかマイナスか)、ln は自然対数、|x| は絶対値です。大きな値は対数で圧縮し、小さな値はほぼそのまま、符号は保ちます。報酬が10000でも0.01でも symlog を通せば扱いやすい範囲に収まります。デコーダや報酬ヘッドは symlog 空間で予測し、予測時は逆変換 symexp(x) = sign(x)·(exp(|x|)−1) で元のスケールに戻します。これにより、出力スケールをタスクごとに調整する必要がなくなります。

### 2. 報酬・価値の twohot エンコーディング

価値や報酬を「1つの実数」として回帰(regression)するのではなく、あらかじめ用意した代表値(bin、symlog 空間に等間隔で 255 個など)それぞれの確率として予測する分類問題に変換します。twohot は、ある値を隣り合う2つの bin に按分して表す方式です(例: 値3.4 を bin=3 に 0.6、bin=4 に 0.4 と割り当てる)。これにより報酬スケールが極端でも安定して学習でき、二乗誤差回帰の不安定さ(外れ値で勾配が爆発する、スケールに敏感)を避けられます。分類のクロスエントロピー損失はスケール不変に近く扱えるのが利点です。

### 3. percentile によるリターン正規化

actor 学習で、想像したリターン(累積報酬)を、バッチ内の上位95パーセンタイルと下位5パーセンタイルの差で割って正規化します(ただし差が1未満なら割らない安全装置付き)。これにより、報酬が大きいタスクでも小さいタスクでも、まばらでも密でも、actor へ流れる勾配のスケールが揃い、探索の度合い(エントロピー正則化との釣り合い)が安定します。外れ値に引っ張られにくいのもパーセンタイルを使う理由です。

### 4. free bits による KL 正則化

[RSSM (Recurrent State-Space Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0088_rssm_note) の KL 損失に下限(free bits、例: 1 nat)を設け、KL がその下限を下回ったら罰しません。これにより「潜在が情報を捨てすぎて posterior と prior が一致し、表現が崩壊する」のを防ぎつつ、易しいタスクで KL 項が過剰に効いて表現が痩せるのも防ぎます。

### これらの統合効果

個々のテクニックは「数値を整える前処理」で地味ですが、合わせると「報酬スケール・観測の種類・報酬頻度がどんなに違っても同じ設定で安定学習できる」という汎用性が生まれます。論文の主張は「賢い新アルゴリズムではなく、丁寧な頑健化の積み重ねが汎用性を生む」という点にあり、これが DreamerV3 の本質的貢献です。

### tensor のイメージ

```
観測             → symlog → エンコーダ → 離散潜在 s_t [B, 32, 32]
決定論状態 h_t   [B, hidden]   (モデルサイズで 256〜数千)
報酬・価値ヘッド  → twohot 分布(255 bin, symlog 空間)
actor            → 想像リターンを percentile 正規化して更新
```

モデルサイズは XS から XL まで複数用意され、同じハイパラのままパラメータ数だけを変えてスケール則を測ります。

## 手法の系譜と主要論文

### 系譜上の位置

[PlaNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0082_planet_note)(潜在計画)→ [Dreamer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0047_dreamer-v1)(潜在想像で方策学習 + 解析的勾配)→ [DreamerV2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0048_dreamer-v2)(離散潜在で Atari 制覇)→ DreamerV3(単一設定で汎用 + スケール則)という発展の到達点です。骨格は V2 とほぼ同じで、V3 の貢献は「タスク横断で破綻しない頑健化」と「スケールしても安定する設計」に集約されます。

### Hafner, Pasukonis, Ba, Lillicrap "Mastering Diverse Domains through World Models" (arXiv:2301.04104, 2023; Nature 2025)

既存の Dreamer はタスクごとにハイパラ調整が必要で汎用性に欠けました。symlog・twohot・パーセンタイル正規化・free bits などスケールや分布の違いを自動吸収する仕組みを統合すれば、設定固定で何でも解けるはず、という仮説を検証しました。提案手法は単一設定で多領域を解き、Minecraft ダイヤ採掘を人間データやカリキュラムなしで初達成し、モデルサイズに対するスケール則を実証しました。2025年に Nature 誌へ掲載され、世界モデル研究の代表例として広く参照されています。

### 比較対象: VPT (Baker et al., NeurIPS 2022)

Minecraft では、VPT が大量の人間プレイ動画(約7万時間)で模倣事前学習してからダイヤ採掘を達成しました。DreamerV3 は人間データに一切頼らず、報酬だけからゼロで達成した点で対照的であり、世界モデルの長期探索能力を際立たせました。

## 論文の実験結果(定量データ)

### 多領域ベンチマーク(150超タスク、単一設定)

評価は連続制御(DMC: proprioception と vision)・Atari 100k と Atari 200M・DMLab・Crafter・Minecraft・BSuite・ProcGen など8つ以上のベンチマーク群、合計150超のタスクです。決定的なのは「すべて同じハイパーパラメータで」解いた点です。多くのベンチマークで既存の専用手法に匹敵または上回り、Atari 100k(2時間相当・40万フレームでサンプル効率を測る低データ条件)では当時の有力手法群(EfficientZero など特化手法を除く一般手法)と競合し、Crafter のような手続き生成サバイバル環境(22 の達成項目をどれだけ解けるか)でも最高水準を更新したと報告されています。

数値の意味を補足すると、各ベンチマークは指標が異なります(累積報酬、人間正規化スコア、達成項目数など)が、ポイントは「個別最適化なしで横断的に強い」ことです。従来は1つのベンチマークに特化した手法が乱立し、ある手法が別領域では全く動かないことが普通でした。単一設定での横断的な強さは質的に新しい成果でした。

### Minecraft ダイヤモンド

DreamerV3 は、人間のデモ(お手本)やカリキュラム(段階的課題設定)なしで、ゼロから Minecraft のダイヤモンドを採掘した初のアルゴリズムになりました。ダイヤ採掘は「木を切る→木材から道具を作る→石を掘る→かまどで鉄を精錬する→鉄の道具を作る→さらに深く掘る→ダイヤ」という12段階前後の長い手順を要し、報酬が極めてまばらで、ブロックの世界がランダム生成される超難問です。報告では、約1億環境ステップ規模の学習で、複数の独立した試行のうちかなりの割合(全シードで定期的に成功)でダイヤ採掘に到達したとされます。これを汎用設定のまま、しかも人間データなしで達成したことが大きな話題になりました。

### スケール則のアブレーション

世界モデルとエージェントのパラメータ数を増やす(XS の数百万級 → XL の2億級)ほど、最終性能とデータ効率(同じスコアに到達するのに要するステップ数)がともに単調に向上することが示されました。「大きいモデルほど少ないデータで高いスコアに届く」という、言語モデルで知られる傾向が世界モデル RL でも成り立つことを示した点が重要です。さらに頑健化テクニックの除去実験では、symlog を外すと報酬スケールが極端なタスクで学習が崩れ、twohot を実数回帰に戻すと外れ値で不安定化し、free bits を外すと潜在崩壊で性能が落ち、パーセンタイル正規化を外すと報酬スケールの異なるタスク間で探索が不安定になる、と各工夫がそれぞれ寄与することが確認されています。「どれか1つを抜くと一部タスクで破綻する」という形のアブレーションが、汎用性が個々の頑健化の合わせ技で成り立っていることを裏付けます。

## メリット・トレードオフ・限界

メリット: 単一ハイパラ設定で150超の多様なタスクを解く高い汎用性。symlog・twohot・パーセンタイル正規化・free bits などでスケール差・報酬頻度差・観測種別の違いに頑健。モデルサイズに対するスケール則(大きいほど高性能かつ高効率)を実証。Minecraft ダイヤ採掘をデモなし・ゼロから初達成し、長期・まばら報酬タスクへの探索力を示した。実装が公開され再現性が高い。

トレードオフ・限界: 想像ロールアウト型 RL の宿命として、世界モデルの予測誤差が long-horizon(長期)で蓄積するリスクは残る。多数の頑健化テクニックの組み合わせで実装が複雑であり、各工夫の寄与の理論的理解はまだ経験則の域を出ない。確率的で予測困難な実世界(他者の意図が読めない交通、敵対的なエージェントなど)では、Minecraft のような環境ほどうまくいくとは限らない。Minecraft でも探索には1億ステップ規模を要し、決して「速い」わけではない。安全クリティカルな自動運転にそのまま適用するには、想像内の誤差蓄積が事故に直結しうるため安全性検証が不可欠です。

## 発展トピック・研究の最前線

DreamerV3 は「世界モデルは特定タスク専用の道具ではなく、汎用知能の基盤になりうる」という展望を示しました。以降、世界モデルを Transformer や状態空間モデルで構成する研究(Robine らの TWM, Zhang らの STORM, Micheli らの IRIS)、拡散モデルで観測を生成する世界モデル(Alonso らの DIAMOND)、より大規模なデータでの事前学習、行動条件付き・テキスト条件付きの動画世界モデル(DeepMind の Genie、Google の GameNGen など)へと拡張が進んでいます。DreamerV3 の頑健化(symlog, twohot)は他の RL アルゴリズムにも取り込まれ、デファクトのテクニックになりつつあります。

自動運転・ロボティクスでは「実機を危険にさらさず想像で学ぶ」という思想が魅力的で、closed-loop シミュレーションと組み合わせる方向が活発です。一方で、長期想像の誤差蓄積をどう抑えるか、不確実性を考慮して信頼できない予測領域では想像を打ち切る/ペナルティを与える仕組み、頑健化テクニックの理論的理解、安全保証付きの想像学習が今後の中心課題です。世界モデルが「物理的に正しい予測」をどこまで内在化できるかは、生成系世界モデル([潜在空間予測 vs ピクセル予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0068_latent-vs-pixel-prediction) 参照)との統合を含めて活発に議論されています。
