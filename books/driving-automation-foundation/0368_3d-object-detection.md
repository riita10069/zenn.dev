# 3D物体検出

## ひとことで言うと

3D物体検出とは、センサ(LiDARやカメラ)から得たデータをもとに、周囲にある物体を「3次元空間の箱(3Dバウンディングボックス)」として位置・サイズ・向きまで含めて推定する技術です。自動運転車が「前方12mに、進行方向に対して15度傾いた長さ4.5mの車が、時速30kmで近づいている」と理解するための土台になります。

ここで「バウンディングボックス」とは物体を囲う直方体のことです。2次元の画像処理では物体を平面の四角形(x, y, 幅, 高さ)で囲いますが、3D物体検出では奥行きを含む直方体(中心座標 x, y, z、サイズ w, l, h、向き θ)で囲います。

## 直感的な理解

人間は両目で見た景色から、無意識に「あの車はだいたい10mくらい先にいる」と奥行きを推定しています。これは脳が「両目の視差」「物体の見かけの大きさ」「地面との接地点」などの手がかりを統合しているからです。

機械にこれをやらせるとき、大きく2つの入力源があります。

- LiDAR(ライダー): レーザーを周囲に照射し、反射が返ってくるまでの時間から距離を測るセンサです。出力は「点群(point cloud)」と呼ばれる、空間に散らばった大量の点((x, y, z) 座標と反射強度を持つ)の集合です。1スキャンで10万〜数十万点が得られます。奥行きが正確(誤差数cm)ですが、点がまばらで色情報がありません。
- カメラ: 普通のRGB画像です。色やテクスチャ(模様)が豊富で安価ですが、画像そのものには奥行き情報がありません。1枚の写真だけから「これは小さい近くの車か、大きい遠くの車か」を区別するのは原理的に曖昧です。

3D物体検出のアルゴリズムは、これらの入力から「物体がどこにいくつあり、どの向きでどんな大きさか」を一気に当てる問題を解きます。

## 基礎: 前提となる概念

3D物体検出を理解するために、いくつかの前提概念を噛み砕いておきます。

点群(point cloud): LiDARが返す N 個の点の集合です。tensor の形でいうと、おおよそ `[N, 4]`(各点が x, y, z, intensity を持つ)です。N はスキャンごとに変動し、点の並び順に意味はありません(順序不変性、permutation invariance)。この「順序がない」「数が可変」という性質が、画像のような格子状データを前提とした通常のCNN(畳み込みニューラルネットワーク)をそのまま使えない理由です。

ボクセル(voxel): 3次元空間を立方体の格子で区切った一つ一つのマス目のことです。2次元のピクセルを3次元に拡張したものです。点群をボクセルに割り当てると、まばらな点群を規則的な格子データに変換でき、3D畳み込みが使えるようになります。ただし大半のボクセルは空(物体がない)なので、計算が無駄になりやすいという問題があります。

BEV(Bird's Eye View、鳥瞰図): 真上から見下ろした2次元の地図のような表現です。自動運転では物体はほぼ地面の上に並んでおり、高さ方向の重なりが少ないため、上から見たBEV平面で物体を扱うと検出が簡単になります。3次元の問題を実質2次元の問題に落とせるのが利点です。

アンカー(anchor): 検出時にあらかじめ用意しておく「典型的な物体の箱のテンプレート」です。例えば「車は幅1.6m 長さ3.9m」といった既定サイズの箱を空間に敷き詰め、各位置でその箱を微調整して物体に合わせます。一方、後述のCenterPointのように「アンカーを使わずに物体の中心点を直接当てる」アンカーフリーな手法もあります。

評価指標 mAP と NDS: 検出精度は主に mAP(mean Average Precision、平均適合率の平均)で測ります。これは「検出した箱が正解とどれだけ重なっているか」を様々なしきい値で評価して平均した値で、0から1(または0〜100%)で高いほど良い指標です。自動運転データセット nuScenes では、距離誤差・サイズ誤差・向き誤差・速度誤差なども含めて統合した NDS(nuScenes Detection Score)という指標も使われます。

## 仕組みを詳しく

3D物体検出のパイプラインは、入力に応じて大きく次の系統に分かれます。

LiDAR系(点群を入力とする)の典型的な流れは次の通りです。

```
点群 [N,4]
  → エンコード(点群を格子状の特徴に変換: ボクセル化 or 柱状化)
  → 2D/3D CNN バックボーンで特徴抽出
  → BEV特徴マップ [C, H, W]
  → 検出ヘッド(箱の中心・サイズ・向き・クラスを回帰)
  → 3Dボックス出力
```

ここで「エンコード」の方式が手法ごとの個性になります。

PointPillars(後述)では、点群をBEV平面の格子で区切り、各格子(これを「pillar = 柱」と呼ぶ。高さ方向は分割せず1本の柱にまとめる)の中の点をまとめて1つの特徴ベクトルにします。すると点群が `[C, H, W]` という普通の2D画像のような形になり、高速な2D CNNがそのまま使えます。ボクセルのように高さ方向に細かく区切らないので3D畳み込みが不要で、推論が速いのが売りです。

CenterPoint(後述)では、検出ヘッドの考え方を変えます。アンカーを敷き詰める代わりに、BEV平面上で「物体の中心がここにある」という確率の濃淡を表すヒートマップを予測します。ピークの位置が物体中心で、そこから箱のサイズ・向き・(時系列があれば)速度を回帰します。向きが任意の角度を取る車両では、固定サイズのアンカーを多数の角度で用意するより、中心点を直接当てる方が素直で精度が出やすい、という発想です。

カメラ系(画像を入力とする)はより難しい問題です。画像には奥行きがないため、それを補う工夫が要ります。

FCOS3D(後述)は、2D物体検出器FCOSを3Dに拡張し、画像の各ピクセルから「そのピクセルに写る物体の中心までの2D位置ずれ」と「深度(奥行き)」「3Dサイズ」「向き」を直接回帰します。深度を回帰で当てにいく素朴かつ強力な方式です。

BEVDet(後述)は、複数カメラ画像をいったんBEV(鳥瞰図)特徴に変換してから検出します。各カメラ画像の各ピクセルについて「奥行きの確率分布」を推定し、それを使って画像特徴を3D空間に持ち上げ(lift)、地面に投影し(splat)、BEV特徴マップを作ります。この「Lift-Splat-Shoot」という考え方は、複数カメラの情報を統一された鳥瞰座標系に統合できるため、その後の検出や複数センサ融合がやりやすくなります。

数式の感覚をつかむために、検出ヘッドが回帰する量を言葉で説明します。1つの物体について、ネットワークは以下を出力します。

- 中心座標 (x, y, z): メートル単位の3次元位置。
- サイズ (w, l, h): 幅・長さ・高さ。実際にはlog(対数)で回帰することが多く、これは「2倍の大きさ」と「半分の大きさ」を対称に扱い、学習を安定させるためです。
- 向き θ: 物体の進行方向の角度。sin θ と cos θ の2つに分けて回帰することがよくあります。これは θ=0度と θ=360度が同じ向きなのに数値が大きく離れる「角度の不連続」を避けるためです。
- クラス確率: 車・歩行者・自転車などの分類スコア。

## 手法の系譜と主要論文

3D物体検出の流れを年代順に追うと、技術の進化が見えてきます。

VoxelNet(Zhou & Tuzel, CVPR 2018, arXiv:1711.06396): 点群をボクセルに区切り、各ボクセル内の点を小さなニューラルネット(VFE層)で特徴化し、3D畳み込みで処理した初期の代表作です。エンドツーエンドで学習できることを示しましたが、3D畳み込みが重く低速でした。

SECOND(Yan et al., Sensors 2018): VoxelNetの3D畳み込みを「疎な畳み込み(sparse convolution)」に置き換え、空のボクセルを計算しないことで大幅に高速化しました。空間的に疎な点群の性質を活かした重要な工夫です。

PointPillars(Lang et al., CVPR 2019, arXiv:1812.05784): 前述のpillar(柱)エンコードで3D畳み込みを排除し、2D CNNだけで処理することで高速化と高精度を両立しました。実用システムで広く使われる定番になりました。

PointNet / PointNet++(Qi et al., CVPR 2017 / NeurIPS 2017): 点群を格子化せず、点そのものを直接扱う系統の出発点です。順序不変性を満たす対称関数(max pooling)を使って点群全体の特徴を作る発想は、後続の多くの手法に影響しました。

FCOS3D(Wang et al., ICCVW 2021, arXiv:2104.10956): カメラ単体(monocular)での3D検出を、2D検出器FCOSの自然な拡張として定式化しました。nuScenesのカメラ部門コンペで上位の成績を収め、カメラベースのベースラインとして広く参照されます。

CenterPoint(Yin et al., CVPR 2021, arXiv:2006.11275): 中心点ヒートマップによるアンカーフリー検出と、2段階での精緻化を導入しました。後述のように当時のnuScensベンチマークを大きく更新しました。

BEVDet(Huang et al., arXiv:2112.11790, 2021)とその発展BEVDet4D: 複数カメラをBEVに統合する設計で、カメラベース手法の精度を引き上げました。BEVDet4Dは時系列(過去フレーム)を取り込み、速度推定を大きく改善しました。

BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270): Transformerの注意機構(attention)でカメラ特徴をBEVに集約する手法で、カメラベースBEV検出の流れを決定づけました。

系譜を一言でまとめると、(1) LiDARではボクセル→疎畳み込み→柱→アンカーフリー中心点へと「軽量化と素直さ」を追求し、(2) カメラでは単眼回帰→複数カメラのBEV統合→Transformerへと「奥行きの曖昧さをどう克服するか」を追求してきた、という二つの流れがあります。

## 論文の実験結果(定量データ)

代表的な定量データを、数値の意味とともに紹介します。なお具体値は論文・ベンチマーク版数で変動するため「報告による」値として扱ってください。

PointPillars(CVPR 2019): KITTIデータセット(自動運転検出の古典的ベンチマーク)で、当時の手法に匹敵する精度を、推論速度およそ60Hz超(報告による)という高速さで達成しました。速度の意味は重要です。自動運転では数十ミリ秒以内に検出を終えないと制御が間に合わないため、「精度と速度の両立」がそのまま実用性を意味します。

CenterPoint(CVPR 2021): nuScenesテストセットで NDS およそ65%、mAP およそ58%(報告による、LiDARの2段階版)を達成し、それまでのアンカーベース手法を明確に上回りました。Waymo Open Datasetでも当時の最高水準を更新しました。アブレーション(構成要素を一つずつ外して効果を測る実験)では、アンカーベースのヘッドをCenterPointのヘッドに差し替えるだけで複数mAPの改善が報告されており、「アンカーフリー中心点表現そのものの効果」が確認されています。

FCOS3D(ICCVW 2021): nuScenesのカメラ単独検出で、当時のカメラ部門で上位(コンペ1位の報告)を獲得しました。ただしカメラ単独のNDSはLiDAR手法より大きく低く(報告でNDS 40%台)、これは「画像から奥行きを当てる難しさ」を定量的に示しています。LiDARの65%前後と比べると、奥行きセンサの有無がいかに効くかが分かります。

BEVDet / BEVDet4D: 複数カメラ統合でカメラ単独より精度を上げ、特にBEVDet4Dは時系列導入で速度誤差(mAVE)を大きく下げました(報告で速度誤差がおよそ半減)。速度誤差が下がる意味は、複数フレーム間の物体移動を捉えられるようになったため、瞬間の画像だけでは原理的に分からない「動き」を推定できるようになった、ということです。

これらの数値から読み取れる重要な事実は次の2点です。第一に、同条件ならLiDAR系がカメラ系より精度で優位です(奥行きが直接測れるため)。第二に、カメラ系は時系列・複数視点・Transformerなどの工夫でその差を縮めつつあり、しかもコストが圧倒的に安いため、コストと精度のトレードオフの上で重要な選択肢になっています。

## メリット・トレードオフ・限界

センサごとのトレードオフを整理します。

LiDAR系の利点は奥行き精度の高さと、明るさ(昼夜)に頑健なことです。限界は、(1) センサが高価、(2) 点群が遠距離で急激にまばらになり遠方の小さな物体(歩行者など)を取りこぼしやすい、(3) 雨・霧・雪でレーザーが散乱して点が欠ける、(4) 色やテクスチャがないため標識の文字や信号の色は読めない、ことです。

カメラ系の利点は安価で高解像度、色・テクスチャが豊富なことです。限界は、(1) 単眼では奥行きが原理的に曖昧、(2) 夜間・逆光・トンネル出入口など照明条件に弱い、(3) 深度推定誤差が遠距離で増大する、ことです。

ボクセル/柱表現の限界として、格子の解像度を上げれば精度は上がるが計算量とメモリが急増する、という「解像度と計算コストのトレードオフ」があります。pillarのように高さ方向を潰すと、高さの細かい情報(例: トラックの荷台の上の物体)を失う可能性があります。

検出ヘッドの限界として、向き θ の推定は180度反対向き(車が前向きか後ろ向きか)の取り違えが起きやすく、専用の補正が必要になることがあります。

共通の未解決課題として、(1) ドメインギャップ(あるデータセットで学習したモデルが別の都市・気候・センサ構成では精度が落ちる)、(2) まれな物体・形状(横転車両、特殊車両など学習データに少ないもの)への弱さ、(3) 遠方・遮蔽(occlusion、物体が他の物体に隠れる)下での検出、が挙げられます。

## 発展トピック・研究の最前線

センサ融合(sensor fusion): LiDARの正確な奥行きとカメラの豊富な意味情報を組み合わせる方向です。BEV空間で両者の特徴を結合するBEVFusion(Liu et al., 2022)などが代表で、単一センサより安定した精度が報告されています。融合の難しさは、センサ間の時刻ずれ・座標系のずれ(キャリブレーション誤差)をどう吸収するかにあります。

時系列・4D化: 過去フレームを取り込むことで速度推定と遮蔽への頑健性を高める流れです。BEVDet4DやBEVFormerの時系列版がこの方向です。

Transformerベースと query方式: DETR系(物体を「クエリ」として表現し、注意機構で当てる)を3Dに持ち込んだDETR3D・PETRなどがあり、アンカーやNMS(非最大抑制、重複検出を消す後処理)を不要にする方向が進んでいます。

知覚と予測・計画の統合(end-to-end): 検出を独立した部品にせず、軌道予測や経路計画まで一つのネットワークで解く流れ(UniADなど)も登場し、検出はその一部に組み込まれつつあります。

自己教師あり・基盤モデル: 大量のラベルなしデータで事前学習し、少ないラベルで高精度を出すアプローチも研究されています。3Dラベル付けは非常に高コスト(1シーンあたりの手作業が膨大)なため、ラベル効率は実務上きわめて重要です。
