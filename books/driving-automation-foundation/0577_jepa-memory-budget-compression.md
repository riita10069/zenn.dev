# JEPAターゲットのメモリ圧縮戦略

## ひとことで言うと

未来を「学習した表現空間」で予測する世界モデル(JEPA系)では、予測の対象になる特徴マップが巨大になり、GPUのメモリに乗らないことがあります。そこで、予測する空間を「チャンネルを平均で潰す」「学習した射影で次元を下げる」「空間を間引く」という代表的な3つのやり方で軽くします。どこを削るかによって、省メモリの度合いと、予測対象として残る情報の豊かさのバランスが変わります。

## 直感的な理解

世界モデルに「未来当て」を学ばせるとき、最も意味のある予測対象は「鳥瞰図(上から見下ろした地図状)の特徴マップ」です。空間的に分かりやすく、運転判断に直結するからです。ところがこの鳥瞰図特徴は、高さ・幅・チャンネルの三方向に大きく、一枚で数千万個の数値になります。さらに未来を複数時刻ぶん予測し、それぞれに正解を用意し、勾配計算の中間結果まで保持するとなると、メモリは雪だるま式に膨れ上がります。

ここで素朴に「解像度を半分にしよう」とすると確かに軽くなりますが、同時に「予測すべき意味情報」まで一緒に捨ててしまう恐れがあります。たとえば横切る歩行者の位置精度が必要なのに空間を粗くしすぎたり、車か建物かの区別が必要なのにチャンネルを潰しすぎたりすると、予測タスクが「軽いけれど中身のない」ものになり、表現学習の効果が失われます。

そこで「闇雲に削る」のではなく「何を残し何を削るかを選べる」複数モードを用意します。意味(チャンネル)を残して位置(空間)を粗くするのか、位置を残して意味を圧縮するのか。タスクに応じて選べるようにするのがメモリ圧縮戦略です。

## 基礎: 前提となる概念

JEPA(Joint-Embedding Predictive Architecture): 「ピクセルそのものではなく、学習した表現(埋め込み)空間で、見えない部分や未来の表現を予測する」自己教師あり学習の枠組みです。ピクセル予測は影やテクスチャといった無関係な細部に労力を割きますが、表現空間予測は意味だけを捉えられます。予測する側(predictor)と正解を作る側(target encoder)を非対称にする(正解側は勾配を止めるか EMA で緩やかに更新する)ことで、「全部同じ値を出せば損失ゼロ」という崩壊を防ぎます。

BEV(Bird's Eye View、鳥瞰図): 複数のカメラ画像を、幾何変換とカメラパラメータを使って「上から見下ろした地図」のような格子状の特徴マップに統合した表現です。各格子マス(セル)が、その地点の意味を表す多次元の特徴ベクトルを持ちます。典型的には縦数百×横数百マス、各マスが数百次元です。

テンソルの形状とメモリ: 特徴マップは `[B, C, H, W]`(バッチ、チャンネル、高さ、幅)という四次元配列で表します。メモリ使用量は要素数 × 1要素あたりのバイト数で、fp32(32ビット浮動小数点)なら1要素4バイトです。なぜメモリが問題になるのか具体的に計算します。仮に1サンプル・1時刻の特徴を `[256, 450, 300]`(チャンネル256、縦450、横300)とすると、

```
450 × 300 × 256 = 約 3456 万個の数値
fp32(4バイト)なら 3456万 × 4 ≈ 138 MB(1サンプル・1時刻あたり)
```

未来を4時刻ぶん予測するなら予測値だけで `138MB × 4 ≈ 550MB`、さらに同サイズの正解、勾配計算の中間活性化(逆伝播のために保持する途中結果)も加わります。バッチサイズを8に増やせば、これだけで数十GBに達し、ハイエンドGPU(たとえばVRAM 45GB級)でもメモリ不足(OOM、Out Of Memory)で学習が止まります。これが圧縮を必要とする直接の理由です。

## 仕組みを詳しく

元の特徴を `[B, 256, 450, 300]` として、代表的な3つの圧縮モードを見ます。ここで圧縮するのは「予測対象の空間(forecast space)」であり、軌道計画など本線の処理はフル解像度のまま使い、世界モデルの未来予測だけを軽くする、という切り分けが前提です。

1. occupancy(チャンネル平均、最も軽い)

チャンネル次元(256)を平均して1チャンネルに潰します。

```
[B, 256, 450, 300]  --(チャンネル平均)-->  [B, 1, 450, 300]
```

各マスに残るのは「そこに何かが在るか(occupancy = 占有度)」という超圧縮された一次元情報だけです。メモリは256分の1。最も軽い一方、「車か人か建物か」という意味の区別はほぼ失われます。空間の占有さえ予測できればよい、と割り切る場面向けです。

2. projected(学習した線形射影、バランス型)

256次元を、学習可能な線形射影(行列の掛け算。`nn.Linear` などで実装)で `d_slow` 次元(例: 64や128)に圧縮します。

```
[B, 256, 450, 300]  --(学習射影)-->  [B, d_slow, 450, 300]
```

「次元を減らすが、減らし方をデータから学ぶ」点が平均化との決定的な違いです。平均は固定の潰し方ですが、学習射影は「予測に有用な軸」を残すよう最適化されます。これは JEPA の「意味空間を保ったまま予測対象を作る」発想そのもので、省メモリと意味の両立を狙えます。メモリは `256 / d_slow` 分の1。`d_slow`(slow ブランチの次元、という意味で名付けられることが多い)が大きいほど情報を残し、小さいほど軽くなります。

3. full channels + 空間ストライド(空間の間引き)

チャンネルは256のまま残し、空間方向を間引き(ストライド)で粗くします。たとえばストライド3なら縦横を1/3に落とします。

```
[B, 256, 450, 300]  --(空間 stride=3)-->  [B, 256, 150, 100]
```

細かい位置精度は落ちますが、各マスの意味(256次元)はフルに保てます。メモリは `stride^2` 分の1(縦横それぞれ1/strideになるため)。位置精度を犠牲にして意味を守る方向です。

これら3つは「チャンネルを削る(1, 2)」か「空間を削る(3)」かで直交しており、組み合わせも可能です。たとえば projected(`d_slow=64`)と空間ストライド2を併用すると、

```
[B, 256, 450, 300] → [B, 64, 225, 150]
225 × 150 × 64 = 約 216 万個
```

元の約3456万個から16分の1に減り、4時刻ぶんでもGPUに乗りやすくなります。

実装上の鉄則は、予測側(predictor の出力)と正解側(target encoder の出力)に同一の圧縮を適用し、両者の形状を完全に揃えてから損失を計算することです。圧縮がずれると、形状不一致でエラーになるか、あるいは黙って間違った位置同士を比較してしまい、学習が無意味になります。

## 手法の系譜と主要論文

このトピックは「表現空間で予測する」「BEV の解像度と計算量のトレードオフ」「占有予測」という3つの研究潮流の交点にあります。

表現空間予測の起点が I-JEPA(Assran et al., Meta, CVPR 2023, arXiv:2301.08243, "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture")です。提案は「画像をピクセル再構成せず、学習された表現空間で見えない領域の表現を予測する」こと。理由は「ピクセル予測は無関係な細部に労力を割くが、表現空間予測は意味だけを捉えられる」から。projected モード(学習射影で意味を保ちつつ次元圧縮する)はこの思想を直接借りています。同時に I-JEPA は「表現空間が崩れて全部同じ値になる崩壊(representation collapse)」のリスクを指摘し、正解側を勾配停止または EMA(指数移動平均)で更新する非対称設計で防ぎました。

BEV の解像度・コストのトレードオフを精密に扱ったのが BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)です。複数カメラから空間的・時間的な注意機構で BEV 特徴を構築する手法で、BEV グリッドを細かくするほど精度は上がるが、計算とメモリがグリッドサイズの二乗で増えるという関係を実証しました。空間ストライドモードの背景はまさにこの二乗則です。

占有予測の系統では、OccNet(Tong et al., ICCV 2023, arXiv:2306.02851)や SurroundOcc(Wei et al., ICCV 2023, arXiv:2303.09551)が「各三次元セルに物体が在るか」を予測する3D occupancy 予測を運転で実用化しました。「意味の細かい区別より、まず空間の占有を粗く正確に」という発想が、occupancy モード(チャンネル平均で占有だけ残す)の妥当性を支えます。

## 論文の実験結果(定量データ)

I-JEPA(arXiv:2301.08243)は、ImageNet-1k の線形評価(学習済み特徴の上に線形分類器だけ載せて精度を測り、特徴そのものの良さを判定する手法)で、ピクセル再構成型の自己教師あり学習に匹敵する精度を、はるかに少ない計算量(報告によると、ピクセル再構成型より数倍少ない事前学習エポック・GPU時間)で達成しました。これは「予測対象を意味空間に縮約しても、表現の質はむしろ保たれる/効率が上がる」ことの定量的証拠で、圧縮しても意味を残せば学習効果が落ちにくいという本トピックの前提を支えます。

BEVFormer(arXiv:2203.17270)は nuScenes(都市運転の大規模マルチカメラデータセット)の三次元物体検出で、NDS(nuScenes Detection Score、検出精度と位置・速度・属性の正確さを統合した総合指標、大きいほど良い)約0.56、mAP(平均適合率、大きいほど良い)約0.48を報告しました。重要なのはアブレーションで、BEV グリッド解像度や時間方向の参照フレーム数を増やすと精度は上がるがメモリと計算が増える、という直接のトレードオフが示されています。これが「精度を取るか軽さを取るか」を解像度で調整する根拠です。

3D occupancy 系では、SurroundOcc(arXiv:2303.09551)などが nuScenes 由来の占有ベンチマークで mIoU(平均 Intersection over Union、予測した占有セルと正解の重なり率の平均、大きいほど良い)を報告しています。占有という粗い表現でも、車・歩行者・障害物の存在を実用的な精度で捉えられることが示されており、occupancy モードのように意味を1チャンネルに潰しても「在る/無い」の情報は有用に残る、という見立ての裏付けになります。

圧縮モード自体のアブレーション(どのモードでどれだけ軽くなり、どれだけ予測品質が変わるか)は、(1)occupancy は最軽量だが下流タスクへの寄与が最も小さい、(2)projected は `d_slow` を中程度に取ると軽さと意味のバランスが最良、(3)空間ストライドは小さい障害物の予測品質が落ちやすい、という傾向で報告されます。圧縮を強めるほど予測タスク自体が易しくなり、表現学習として得られる「本線への恩恵」が減る、という非自明な落とし穴も指摘されています。

## メリット・トレードオフ・限界

メリット:
- GPUメモリに乗らなかった大規模な未来特徴予測を、現実的なバッチサイズで学習可能にする。
- 「意味を削る(チャンネル)」か「位置を削る(空間)」かをタスクに応じて選べ、組み合わせもできる。
- projected は圧縮の仕方を学習で最適化するため、固定の平均化より意味を残せる。

トレードオフ・限界:
- occupancy は最も軽いが、物体の種類など意味情報をほぼ失う。
- 空間ストライドは位置精度が落ち、細かい障害物や狭い隙間の予測が甘くなる。
- 圧縮を強めるほど予測タスクが易しくなり、表現学習として下流に効く度合いが薄れる可能性がある(軽さと学習効果が逆相関しうる)。
- 予測側と正解側で圧縮を厳密に揃える実装の手間と、揃え損ねたときの静かなバグのリスク。

未解決の課題は、「タスクに最適な圧縮を自動で決められないか」です。現状は人手で `d_slow` やストライドを試行錯誤しますが、どの軸の情報が下流に効くかをデータから判定して圧縮を適応させる方向が研究テーマになっています。

## 発展トピック・研究の最前線

可変・適応圧縮: シーンの複雑さに応じて圧縮率を動的に変える(混雑した交差点ではチャンネルを多く残し、空いた直線路では強く潰す)アイデアが考えられます。注意機構で重要な領域だけ高解像度に残すトークン削減(token pruning)とも親和的です。

低ビット化との併用: ここで扱った「要素数を減らす」圧縮に加え、各要素を fp16 や bf16(16ビット浮動小数点)、さらに量子化で低ビット化すれば、要素あたりのバイト数も減らせます。混合精度学習(amp)は実務上ほぼ標準で、本トピックの空間/チャンネル圧縮と直交して効きます。

潜在トークン化: BEV 格子をそのまま圧縮するのではなく、可変長のスロット(オブジェクト中心の潜在ベクトル)に集約してから予測する方向もあります。物体ごとに潜在を割り当てれば、空っぽの背景に無駄なメモリを使わずに済み、世界モデルの予測対象としても自然です。
