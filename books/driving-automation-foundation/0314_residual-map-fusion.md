# Residual Map Fusion(残差ゲート地図融合)

## ひとことで言うと
すでにそこそこ働いているニューラルネットワークに、新しい情報源(ここでは地図)を「最初はまったく効かせず、学習が進むにつれて少しずつ効かせる」という安全な足し方で混ぜ込む設計です。中心となる式は `出力 = 元の特徴 + alpha × 新しい特徴` で、効き具合 `alpha` を 0 から学習で育てます。`alpha` を 0 で初期化することがこの設計の核で、学習開始時には新しい情報がいっさい影響しないため、これまで学習してきた経路を壊さずに新機能を後付けできます。これは画像認識・拡散モデル・大規模言語モデルにまたがって繰り返し再発見されてきた、汎用的で頑健な「枝の足し方」です。

## 直感的な理解
運転に十分慣れた人のところに「カーナビの案内」という新しい情報源が加わる場面を想像してください。最初からナビを全面的に信じると、ナビがまだ調整中で誤案内を出していた場合、せっかく身につけた運転感覚まで乱されます。賢いやり方は「最初はナビをほぼ無視し、ナビが正しいと分かってきた範囲だけ、徐々に頼る割合を増やす」ことです。残差ゲート地図融合がやっているのはまさにこれです。

機械学習でこれが切実な問題になる理由は、学習の初期段階にあります。地図を処理する部分(map encoder、地図を特徴ベクトルに変換するネットワーク)は学習開始直後にはまだ何も学んでおらず、出力はほぼ意味のないランダムなノイズです。一方でカメラ側の経路は、事前学習済みバックボーン(大量の画像であらかじめ訓練された特徴抽出器)などのおかげで、すでにそれなりに機能していることが多い。この状況で「地図の特徴とカメラの特徴をいきなり対等に足す」と、機能している側にノイズを注入することになり、学習が不安定化したり、収束が大幅に遅れたりします。これは新しい枝(branch)を既存ネットワークに接ぎ木するとき普遍的に起きる問題で、「初期化ショック」と呼べる現象です。

身近な比喩を離れて正確に言えば、問題の本質は「最適化の出発点」にあります。ニューラルネットの学習は損失(誤差)を下げる方向にパラメータを少しずつ動かす作業ですが、出発点が悪い(損失が高く、勾配の方向も荒れている)と、そこから抜け出すのに時間がかかったり、悪い局所解にはまったりします。新しい枝を 0 から始めれば、出発点は「枝を付ける前と完全に同じ」になり、最適化の良い出発点を保てます。

## 基礎: 前提となる概念
この設計を理解するには四つの前提が要ります。

第一に「残差接続(residual connection)」です。`出力 = x + F(x)` という形で、入力 `x` をそのまま通す経路(スキップ接続)と、変換を加える経路 `F(x)` を足し合わせる構造です。「残差」とは「元の値からの差分」のことで、ネットワークには「最終的な答え」そのものではなく「元の値にどれだけ足し引きすべきか」という差分だけを学習させます。差分がゼロでよいなら `F(x)=0` を学べばよく、これは恒等写像(何もしない変換)を表現しやすいことを意味します。深いネットワークで勾配が消える(層を遡るうちに勾配がほぼ 0 になり学習が進まない)問題を、この構造が大幅に緩和しました。

第二に「ゲート(gate)」です。情報をどれだけ通すかを決める弁のことで、ここでは新しい枝 `F(x)` に掛ける係数 `alpha` がそれにあたります。`alpha=0` なら新しい枝は完全に閉じており、`alpha` が大きくなるほど新しい情報が強く流れ込みます。重要なのは `alpha` が固定値ではなく、学習で更新される可変パラメータである点です。

第三に「BEV(Bird's-Eye-View、鳥瞰図)特徴」です。カメラの斜め視点の画像や地図を、真上から見下ろした座標系(縦横の格子=グリッド)に変換した特徴マップを指します。自動運転では複数の情報源を共通の俯瞰座標に揃えてから融合するのが定石で、カメラから作った BEV と地図から作った BEV を同じグリッド上で足し合わせられるようにしておきます。

第四に「チャネル」です。特徴マップは `[バッチ, チャネル, 高さ, 幅]` の四次元テンソルで持たれ、チャネルとは「特徴の種類」を表す次元です。あるチャネルは「車線らしさ」を、別のチャネルは「障害物らしさ」を表す、というように役割が分かれているのが普通です(完全に明示的に分離しているわけではないが、学習を通じて各チャネルが特定の概念に反応するようになる)。後で出てくる「チャネルごとのゲート」を理解するには、この「チャネル=特徴の種類」というイメージが土台になります。

## 仕組みを詳しく
典型的な実装は次のように書けます。

```python
self.alpha = nn.Parameter(torch.zeros(embed_dim))  # 例: 256個のゼロ

def forward(self, image_bev, map_bev):
    gate = self.alpha.view(1, -1, 1, 1)   # (1, 256, 1, 1) に変形
    return image_bev + gate * map_bev
```

ここで最も大事な設計判断が二つあります。

一つ目は「alpha をスカラ(1個の数)ではなくチャネルごとのベクトル(例: 256個)にする」ことです。alpha をチャネルごとに持てば、「地図が役立つ種類の特徴(車線・停止線・交差点形状など)では地図を強めに効かせ、地図が無関係な特徴では効かせない」という選択的な融合ができます。これがスカラ 1 個のゲートより細やかで、表現力が高い理由です。この「チャネルごとに重みを学習で決める」という発想は、後述の Squeeze-and-Excitation(SENet)が一般化した「チャネル単位の再重み付け」と同じ系統にあります。

`view(1, -1, 1, 1)` で形を `(1, 256, 1, 1)` に変えるのは、ブロードキャスト(形の異なるテンソル同士を、サイズ 1 の次元を自動で引き伸ばして計算する仕組み)で各チャネルに対応する alpha を掛けるためです。テンソル形状で追うと次のようになります。

```
image_bev: [B, 256, 450, 300]   (B=バッチ, 256=チャネル, 450×300=BEV解像度)
map_bev:   [B, 256, 450, 300]   (image_bev と同じ形である必要がある)
gate:      [1, 256, 1, 1]        (チャネルごとの効き具合)

gate * map_bev → [B, 256, 450, 300]   (チャネル c は alpha[c] 倍される)
image_bev + (gate * map_bev) → [B, 256, 450, 300]
```

before(学習開始時): alpha が全要素 0 なので、出力は `image_bev` そのもの。地図の影響は完全にゼロで、地図なしで学習する場合とビット単位で同じ挙動になります。

after(学習後): 例えば車線を表すチャネルの alpha が 0.4 に育てば、そのチャネルでは地図の車線情報が 4 割の重みで足されます。地図と相性の悪いチャネルでは alpha が 0 付近にとどまります。学習された alpha のベクトルを観察すると、「どの特徴に地図が貢献したか」が読み取れる、という副次的な解釈性も得られます。

二つ目の設計判断は「足し算(加法的融合)であること」です。連結(concatenation)してから変換する方式([Concat Fusion(連結融合)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0317_concat-fusion))と違い、足し算は入力と出力の形が変わらず、チャネル数も増えません。後段のネットワークはそのまま使え、計算もメモリも軽い。半面、足し算は「カメラと地図の積(相互作用)を直接表現できない」という表現力の上限を持ちます。「地図が車線を示す、かつカメラがその車線を遮蔽物で隠している、ならば地図を優先する」というような条件分岐的・乗法的な参照は、純粋な加法では表せません。そうした柔軟さが必要なら、クロスアテンション型の融合([Cross-Attention Fusion(クロスアテンション融合)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0318_cross-attention-fusion))のほうが適します。

実装上の制約として、`image_bev` と `map_bev` は同じ解像度・同じチャネル数でなければなりません。形が違えば要素ごとの足し算が成立しないからです。したがって地図 BEV を作る段階で、カメラ BEV のグリッド解像度とチャネル次元に揃えておく前処理が必須になります。地図はラスタ画像(車線や境界を色で塗った画像)やベクトルポリライン(線分の頂点列)で与えられることが多く、それを CNN や小さなエンコーダで上記の形の特徴マップに変換します。

## 手法の系譜と主要論文
この融合方式は単一の論文の発明ではなく、複数の確立したアイデアの合流点です。系譜をたどると設計思想が見えてきます。

出発点は ResNet(He, Zhang, Ren, Sun、CVPR 2016、"Deep Residual Learning for Image Recognition")です。残差接続 `output = x + F(x)` を導入し、それまで学習困難だった超深層ネットワーク(152 層など)の安定学習を可能にしました。`image_bev + 何か` という骨格はここに直接由来します。ResNet は、ネットワークが「恒等写像」を表現しやすくすることが深層学習の鍵だと示した点で歴史的に重要です。

次に「新しい枝をゼロで初期化して安全に足す」という発想の系譜です。Stochastic Depth(Huang et al.、ECCV 2016)は学習中に残差ブロックを確率的に丸ごと無効化(=枝の出力を 0 にする)し、これが強力な正則化と高速学習につながることを示しました。「枝を 0 にしても残差接続のおかげでネットワークが壊れない」という観察が、後の「枝を 0 から育てる」発想の土台になっています。ReZero(Bachlechner et al.、UAI 2021、"ReZero is All You Need: Fast Convergence at Large Depth")は、各残差ブロックに学習可能なスカラ係数を掛け、それを 0 で初期化することで、Transformer や深層 CNN の収束を大幅に速め、層を非常に深くしても安定して学習できることを示しました。残差ゲート地図融合の `alpha=0` 初期化はこの思想とほぼ同型です(ReZero がスカラなのに対し、ここではチャネルごとのベクトルに拡張している点が違い)。

チャネルごとに係数を持つ発想は Squeeze-and-Excitation(Hu et al.、CVPR 2018)が一般化しました。SENet はグローバルプーリングで各チャネルの代表値を取り、小さなネットワークでチャネルごとの重み(0〜1)を出して特徴マップを再重み付けします。「チャネル単位で情報の流れを調整する」という発想が、残差地図融合の「チャネルごとの alpha」と同じ系統にあることが分かります。

より最近では ControlNet(Zhang, Rao, Agrawala、ICCV 2023)の「zero-convolution(ゼロ初期化した畳み込み)」が、巨大な事前学習済み拡散モデルに新しい制御入力(輪郭・姿勢・深度など)を後付けする際、ゼロ初期化のおかげで「学習開始時点では元モデルと完全に同一の出力」を保証し、事前学習を壊さずに条件制御を学習できることを実証しました。これは「強い事前学習済みモデルへ安全に機能を増設する」という残差地図融合とまったく同じ動機です。

地図そのものを BEV に融合する流れは HDMapNet(Li et al.、ICRA 2022)や MapTR(Liao et al.、ICLR 2023)などの高精細地図(HD map)系研究で扱われてきました。これらは「地図は極めて強い事前知識だが、カメラと座標系・スケール・時刻を正確に合わせないと、ずれた情報がかえって性能を落とす」という共通の教訓を残しています。残差地図融合はそのリスクを alpha のゼロ初期化と選択的ゲートで緩和する設計と理解できます。

## 論文の実験結果(定量データ)
この方式は複合的なテクニックなので、関連論文が報告する定量結果を要素ごとに見るのが理解の近道です。

ResNet は ImageNet 分類(1000 クラスの画像認識ベンチマーク)で、残差接続なしの「プレーンな」34 層ネットが 18 層ネットより精度が悪化する(劣化問題)のに対し、残差接続を入れた ResNet-34 は ResNet-18 より明確に良くなり、ResNet-152 で当時の最高水準の top-5 誤差(アンサンブル時およそ 3.57%)を達成したと報告しています。top-5 誤差とは「モデルが上位 5 候補を挙げて、その中に正解が一つも無かった割合」で、低いほど良い。残差接続を抜くと深くするほど悪化する、という対照が「残差が効いている」直接の証拠です。

ReZero は、ゼロ初期化スカラを入れることで深い Transformer や CNN の収束イテレーション数が大幅に短縮されたと報告しています。報告によれば 12 層 Transformer の収束がおよそ数倍速くなり、レイヤ正規化(LayerNorm)なしでも非常に深い構成(例として 100 層超)を学習できたとされます。教訓は「枝をゼロから始めると、最適化の初期不安定がなくなり収束が速い」という定量的な裏付けです。

ControlNet は、ゼロ初期化(zero-convolution)を「ランダム初期化」に置き換えるアブレーション(ある要素を抜いたり替えたりして効果を測る実験)を行い、ゼロ初期化版が学習初期から有害なノイズを元モデルに注入しないため、品質を保ったまま条件制御を獲得できることを示しました。ランダム初期化に替えると初期の生成品質が崩れる、という対照が「ゼロ初期化ゲートの価値」を定量・定性の両面で支持しています。

SENet は ImageNet 分類で、SE ブロックを追加するだけで ResNet-50 の top-5 誤差をおよそ 1 ポイント前後改善し、わずかな追加計算で一貫した精度向上を示しました。これは「チャネルごとの再重み付けは小さなコストで効く」ことの定量的な裏付けで、チャネルごと alpha の妥当性を支えます。

地図融合そのものの効果については、HD map を事前情報として与える BEV 知覚研究が、地図ありで車線・走行可能領域のセグメンテーション IoU(Intersection over Union、予測領域と正解領域の重なりを 0〜1 で測る指標。1 に近いほど良い)が向上する一方、地図とセンサの位置合わせ(ローカライズ)誤差が数十センチ以上あると改善が打ち消される、あるいは悪化することを繰り返し報告しています。つまり「地図融合の効きは、座標整合の精度に強く依存する」ことが定量的に知られています。残差地図融合はこの依存性そのものを消すわけではありませんが、alpha が悪い整合の影響を受けるチャネルを 0 付近に抑える方向で、緩衝材として働きうる点が利点です。

## メリット・トレードオフ・限界
メリット側を整理します。alpha のゼロ初期化により、地図なし学習と完全に同一の挙動から安全に出発でき、既存の学習済みカメラ経路を壊しにくい。チャネルごとに効き具合を学習するため、有用な地図情報だけを選択的に取り込め、学習後の alpha から「どの特徴に地図が効いたか」という解釈も得られます。パラメータは alpha ベクトル 1 本だけで、計算もメモリも極めて軽い。実装が単純で検証しやすいのも実務上の強みです。

トレードオフと限界も率直に挙げます。第一に、alpha が育つまで地図効果が出ないため、地図が本来即座に効くべき状況でも立ち上がりが遅れる場合があります。第二に、`image_bev` と `map_bev` を同じ解像度・チャネル数に揃える前処理が必須で、その整合自体に手間と誤差の余地があります。第三に、加法的融合の表現力の上限です。カメラと地図の複雑な相互作用(乗法的・条件分岐的な参照)は表現しにくく、そこはクロスアテンション型に劣ります。第四に、座標ずれをそのまま足し込むリスクです。alpha は緩衝にはなりますが、地図とカメラの幾何整合が大きくずれていれば、ずれた情報を足してしまう根本問題は残ります。第五に、チャネル単位のゲートは「空間のどこでも同じ効き具合」になるため、「交差点だけ地図を強く、直線路では弱く」のような空間依存の調整はこのままではできません。

研究上の未解決課題としては、(1) チャネル単位より細かい「空間位置ごと・状況ごとに動的に変わるゲート」をどう安全に学習するか、(2) alpha を時系列や走行シーン(高速道路か市街地か)で条件付けして適応的にする方法、(3) 地図の信頼度(古い地図、工事による不一致)を融合時にどう反映するか、などが挙げられます。これらは「強い事前知識をいつ・どれだけ信じるか」という、知覚と地図融合の本質的なテーマに連なります。

## 発展トピック・研究の最前線
ゼロ初期化ゲートの考え方は、いまや「事前学習済みモデルへの安全な機能追加」の標準技法として広がっています。大規模言語モデルや拡散モデルへのアダプタ追加(LoRA など、低ランク行列を足し込む手法でも、片方の行列をゼロ初期化して出力側を 0 から始める設計が一般的)に同じ思想が見られ、「巨大モデルを壊さずに新能力を接ぎ木する」という現代的な要求に応えています。

地図融合の最前線では、固定の HD map を足すだけでなく、知覚モデル自身が地図を生成・更新する「オンライン地図構築(online mapping、MapTR シリーズなど)」が進んでおり、生成した地図 BEV を知覚へ戻す双方向の設計が研究されています。また、地図の信頼度を明示的に推定して融合重みに反映する不確実性考慮型の融合や、ゲートを空間位置・チャネル・時刻すべてで動的に決めるアテンション型ゲート(SENet 的なチャネル注意と空間注意を組み合わせた CBAM 系の発想)も探索されています。残差地図融合はこれら高度な融合の最もシンプルで頑健な出発点として位置づけられ、「まずゼロ初期化ゲートで安全に入れ、必要に応じてアテンション型へ発展させる」という設計の入口になります。
