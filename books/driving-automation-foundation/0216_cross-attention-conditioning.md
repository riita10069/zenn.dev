# クロスアテンション条件付け

## ひとことで言うと

クロスアテンション (cross-attention) とは、「あるデータ列が、別のデータ列の中から自分に関係する情報を選んで取り込む」仕組みです。たとえば生成中のノイズだらけの軌道 (未来の動きの候補) が、カメラやセンサから作った周囲環境の地図表現を参照し、「この場所には壁がある、ここは空いている」といった空間情報を時刻ごとに取り込む、という使い方をします。テキストから画像を生成する拡散モデルでも、画像の各部分がテキスト中の関係する単語を参照するのに使われています。条件 (周囲環境やテキスト) を生成に流し込む、汎用で強力な機構です。

## 直感的な理解

軌道を生成するには、車の周囲がどうなっているかを知る必要があります。前に車がいれば減速、空いていれば直進、というように、出力する軌道は「周囲の状況」という条件に依存します。この「条件を与えて生成を制御する」ことを条件付け (conditioning) と呼びます。

一番素朴な条件付けは、周囲環境を1本の固定長ベクトル (たとえば256次元) にぎゅっと圧縮 (プーリング、pooling = 平均などで小さくまとめる) して、生成器に足し込む方法です。しかしこれには大きな問題があります。周囲環境は本来「空間的」な情報です。鳥瞰特徴マップ (Bird's Eye View, BEV = 真上から見た地図状の特徴) は「縦 H × 横 W のマス目に、各マスの状況が C 次元のベクトルで入っている」3次元データです。これを1本のベクトルに平均化すると、「左に障害物」「右が空いている」といった場所ごとの違いが消えてしまいます。平均すると左右の情報が混ざって「なんとなく障害物がある」程度になり、どちらに避ければよいか分からなくなるのです。

クロスアテンションは、この空間構造を保ったまま条件を取り込む仕組みです。軌道の各時刻の点が、地図のどのマスに注目すべきかを自分で選びます。「3秒後の自分の位置あたりのマスを重点的に見る」といった、点ごと・場所ごとに細かく条件を取り込めます。これは「全マスを平均する」のではなく「関係するマスだけ重く読む」、という動的な情報選択です。比喩を避けて言えば、出力側の各要素が条件側の各要素との関連度を計算し、関連度を重みとした加重平均で条件を読み込む、という操作です。

## 基礎: 前提となる概念

アテンションの基本用語を噛み砕きます。アテンションは3つの役割からなります。

- クエリ (Query, Q): 「私はこういう情報が欲しい」という問い合わせ。
- キー (Key, K): 「私はこういう情報を持っている」という見出し。
- バリュー (Value, V): 実際に取り込まれる中身。

仕組みは「Q と各 K の似ている度合い (内積) を計算 → スケーリングしてソフトマックスで重み (合計1) に変換 → その重みで V を加重平均する」です。数式では

Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) V

ここで d_k はキーの次元で、sqrt(d_k) で割るのは内積が次元に応じて大きくなりすぎ、ソフトマックスが鋭くなりすぎる (勾配が消える) のを防ぐためです。Q に近い K を持つ V ほど多く取り込まれます。Q が n 個、K/V が m 個あれば、softmax の中身は n×m の「関連度行列」になり、各行 (各クエリ) が m 個のバリューへの重み配分を表します。

「セルフ (self) アテンション」は Q・K・V がすべて同じ列から来るもの (列の内部で情報をやり取り)。「クロス (cross) アテンション」は Q が一方の列、K と V が別の列から来るもの (列をまたいで情報を取り込む) です。ここではクエリが軌道、キー/バリューが空間特徴、という対応になります。

マルチヘッド (multi-head): Q・K・V を複数の小さな部分空間 (ヘッド) に分け、別々にアテンションして結合します。1つのヘッドが「障害物の位置」、別のヘッドが「走行可能領域」といった異なる観点を同時に見る、と理解すると直感的です。

条件付けの3大流派を区別しておくと見通しが良くなります。
- 連結 (concatenation): 条件ベクトルを入力に貼り付ける。最も素朴。
- FiLM / AdaLN (Adaptive Layer Normalization): 条件からスケール gamma とシフト beta を作り、特徴を正規化したものに `normed * (1 + gamma) + beta` を掛け足す。FiLM (Perez et al., AAAI 2018) が一般化し、DiT が拡散に持ち込んだ。全体に一様に効く条件 (時刻やクラスラベルなど大域条件) を入れるのに向く。
- クロスアテンション: 構造を持つ条件 (テキスト列・空間マップ・他エージェント) を、場所ごと・要素ごとに細かく取り込む。

役割分担の定石は「大域条件は AdaLN、構造を持つ条件はクロスアテンション」です。

## 仕組みを詳しく

具体的な tensor shape で処理を追います。バッチサイズ B、未来時刻数 T (例として64)、各時刻の信号数 (加速度と曲率など) 2、特徴次元 C (例として256)、空間グリッドが H×W とします。

ステップ1 — 空間特徴をトークン列に変換:

```
bev_features:  [B, C, H, W]     (チャネルが先、空間が後ろ)
flatten(2):    [B, C, H*W]      (H と W を1本に潰す)
transpose:     [B, H*W, C]      (空間トークンが並ぶ列に)
線形射影:      [B, H*W, C]      (K/V 用に変換)
```

H=45, W=30 なら H*W=1350個の「空間トークン」になります。各トークンは地図上の1マスに対応する C 次元ベクトルです。これがキーとバリューになります。条件が時刻に依存しない場合 (環境は生成プロセスを通して固定)、この K/V への変換は1回だけ計算して全サンプリングステップで使い回せます (計算節約)。空間位置の情報が flatten で失われるので、位置埋め込み (positional encoding) を足してマスの座標をモデルに伝えるのが通例です。

ステップ2 — ノイズ軌道をクエリに変換:

```
u_t:           [B, T*2]         (T時刻 × 2信号 をフラットにしたもの)
reshape:       [B, T, 2]        (時刻ごとに分ける)
線形射影:      [B, T, C]        (各時刻の (acc, curv) を C 次元クエリに)
```

これで「T個のクエリ (未来の各時刻)」ができます。

ステップ3 — クロスアテンション:

```
queries:  [B, T, C]      (T=64)
keys:     [B, H*W, C]    (1350)
values:   [B, H*W, C]    (1350)
関連度:   [B, T, H*W]    (= 64 × 1350 の重み行列、各行が softmax で和1)
出力:     [B, T, C]
```

各時刻のクエリ (T個) が、H*W個の空間マスの中から自分に関係するマスを選んで加重平均で情報を取り込みます。たとえば「2秒後」のクエリは、2秒後に到達しそうな地図領域のマスに大きな重みを置きます。出力は時刻ごとに C 次元、合計 `[B, T, C]`。

ステップ4 — 大域条件は AdaLN で注入:

空間情報以外の条件 (過去の映像特徴の要約、自車の運動履歴、生成プロセスの時刻 t など) は、別ルートで AdaLN を通して入れます。これらから gamma と beta を作り、アテンション出力を正規化したものに掛け足します。場所ごとに違う条件はクロスアテンション、全体に効く条件は AdaLN、という役割分担です。

ステップ5 — 出力ヘッド:

```
head:      [B, T, C] → [B, T, 2]
reshape:   [B, T*2]
```

各時刻の出力 (拡散ならノイズ、フローマッチングなら速度ベクトル) が得られます。

なぜこの設計か: 軌道点数 T (数十) は小さく、空間マス数 H*W (千以上) は大きい。クエリ側を軌道 (少数)、キー/バリュー側を空間 (多数) にすると、計算量はおよそ T × (H*W) に比例します。逆向き (空間がクエリ、軌道が K/V) にすると、出力が H*W 個 (千以上) になってしまい、生成したいのは軌道なのに無駄が多い。生成したい少数の対象をクエリにするのが自然です。

## 手法の系譜と主要論文

Vaswani ら "Attention Is All You Need" (NeurIPS 2017): アテンション機構そのものを提案。元々は機械翻訳で、デコーダ (出力側) がエンコーダ (入力側) の単語列を参照する部分がクロスアテンションです。RNN のように情報を1本のベクトルに詰め込むと長文で情報が落ちる (情報ボトルネック) ため、各出力位置が入力のどこでも直接参照できるようにしたのが核心。長距離依存を直接扱えますが、計算量がキー数の2乗 (正確には Q長 × K長) に比例するのが弱点です。

Perez ら "FiLM" (AAAI 2018): 条件からアフィン変換 (スケールとシフト) を作って特徴に掛ける条件付けを一般化。視覚的質問応答で、質問文から作った FiLM パラメータが画像特徴を変調することで推論を導けることを示しました。DiT の AdaLN はこの直系で、「大域条件はアフィン変調で十分」という流れの源流です。

Rombach ら "High-Resolution Image Synthesis with Latent Diffusion Models (LDM)" (CVPR 2022、いわゆる Stable Diffusion の基礎): 拡散モデルの U-Net にクロスアテンションを差し込み、テキストなど任意の条件を画像生成に注入する汎用機構を提案。テキスト・レイアウト・他画像など多様な条件を、生成器の各層で柔軟に取り込めるようにしたのが新規性です。1つのアーキテクチャでテキスト→画像、レイアウト→画像など多様な条件付き生成が可能になりました。「ノイズ生成対象がクエリ、条件特徴がキー/バリュー」という発想は、軌道生成へそのまま転用できます。

Peebles & Xie "Scalable Diffusion Models with Transformers (DiT)" (ICCV 2023): 拡散モデルのバックボーンを U-Net から Transformer に置き換え、時刻やクラス条件を AdaLN (正確には学習で初期化を調整した adaLN-zero、残差ゲートの初期値を0にして学習開始時は恒等写像にする) で注入する設計を確立しました。「大域条件は AdaLN、空間/系列条件はアテンション」という今日の標準的な分担はこの系譜です。スケール (モデルサイズ・計算量) を増やすほど生成品質が素直に伸びることも示しました。

軌道・行動生成への応用としては、Chi ら "Diffusion Policy" (RSS 2023) が観測を条件に行動系列を拡散生成し、ロボット操作で観測条件付け (FiLM やクロスアテンション系) が有効であることを示しました。自動運転の軌道拡散でも、BEV 空間特徴をキー/バリューに用いるクロスアテンション条件付けが広く採用されています。条件側を少数の潜在トークンに要約してから注入する Perceiver IO (Jaegle et al., ICLR 2022) 系も、大きな空間条件を扱う際の計算削減策として関連します。

## 論文の実験結果 (定量データ)

LDM (Rombach et al., 2022) はテキスト→画像で FID (Fréchet Inception Distance、生成画像と本物画像の分布の近さを測る指標。低いほど良い) を主要指標に評価しました。クロスアテンションによる条件付けで、テキスト条件に忠実かつ高品質な生成を、ピクセル空間拡散より大幅に少ない計算量 (潜在空間で動かすため、解像度を空間的に8倍縮めるとピクセル数は約64分の1) で達成しています。条件忠実度は CLIP スコア (生成画像とテキストの一致度) でも測られ、クロスアテンション層を持つ構成が条件追従性で優位でした。

DiT (Peebles & Xie, 2023) は ImageNet 256×256 のクラス条件付き生成で、最大構成 DiT-XL/2 が FID 2.27 という当時の最良水準を報告しました。重要なアブレーションは「条件注入方式の比較」で、(a) 入力に条件を連結、(b) クロスアテンションで注入、(c) AdaLN、(d) adaLN-zero を比べ、adaLN-zero が最も良い FID を与えました。具体的には連結方式やクロスアテンション方式より adaLN-zero が明確に低い FID を示し、しかも追加パラメータ・計算が少ない、という結果です。これは「クラスラベルのような大域条件は AdaLN が効率的」という結論で、逆に言えば「テキストや空間特徴のような構造を持つ条件にはアテンションが要る」という今日の使い分けの根拠になっています。さらに DiT は計算量 (Gflops) を増やすほど FID が単調に下がるスケーリング則を示し、Transformer ベース拡散の有望さを定量的に裏づけました。

クロスアテンションのコスト面の数値感覚: キー数 H*W に対し計算量とメモリがほぼ線形〜 (Q長を掛けて) 増えます。空間解像度を H×W = 45×30 (1350) から 90×60 (5400) に上げると、アテンションのコストはおよそ4倍。条件の細かさ (解像度) と計算コストは直接トレードオフの関係にあります。実装上は、関連度行列 `[B, T, H*W]` を顕在化させるとメモリを食うため、FlashAttention のような融合カーネルで行列を物理的に作らず計算するのが現代の標準です。

## メリット・トレードオフ・限界

メリット:

- 空間構造 (場所ごとの違い) を潰さずに条件付けできる。プーリングで平均化する素朴な方法より、障害物の左右や前後を区別できる。
- 生成対象の各要素 (軌道なら各時刻) が、自分に関係する領域を自動で選んで参照できる (動的な注目)。
- LDM/DiT で実績のある汎用機構で、条件を追加・拡張しやすい (テキスト・地図・他エージェントなど別々のクロスアテンションを足せる)。
- アテンション重みを可視化すると「どこを見て決めたか」が観察でき、解釈性の手がかりになる (ただし重みが必ずしも因果的な根拠とは限らない点には注意)。

トレードオフと限界:

- 計算量とメモリがキー数 (H*W) に比例して増える。空間解像度を上げると重くなる (上記の通り)。
- アテンション重みの学習に十分なデータが要る。少データだと「どこを見るべきか」をうまく学べず、注目がぼやける (uniform に近づく) ことがある。
- 単純な足し込み条件付けより実装が複雑で、ヘッド数・次元などのハイパーパラメータが増える。
- 条件が弱い場合に生成器が条件を無視してしまう「条件の取りこぼし」が起きうる (これを補うのが classifier-free guidance などの誘導手法)。
- セルフアテンションと違い、関連度がクエリとキーの食い違う特徴空間で計算されるため、初期化や正規化を誤ると学習開始時に勾配が不安定になりやすい (adaLN-zero のようなゲート初期化が効くのはこのため)。

## 発展トピック・研究の最前線

計算量削減では、キー数を抑えるための線形アテンション、近似アテンション (FlashAttention によるメモリ効率化)、潜在トークンを介して条件を圧縮する Perceiver 系 (少数の学習可能クエリで大きな条件を要約してからクロスアテンション) が研究されています。空間解像度を上げたいが計算を抑えたい、という軌道生成のニーズに直結します。

条件をどれだけ効かせるかの制御では、classifier-free guidance (条件あり/なしの予測を外挿して条件追従を強める手法) が拡散・フロー生成の標準になっており、クロスアテンション条件付けと併用されます。複数条件 (地図 + 他エージェント + 交通規則) を別ルートで注入し、推論時に各条件の強さを個別に調整する compositional な誘導も進んでいます。アテンションマップを直接編集して生成内容を操作する研究 (prompt-to-prompt 等) もあり、クロスアテンションが「制御の取っ手」になりうることを示しています。

アーキテクチャ面では、DiT 系の Transformer バックボーンが拡散/フロー生成の主流になり、軌道生成でもセルフアテンション (時刻間の整合) とクロスアテンション (環境条件) を交互に積む構成が一般的です。さらに、空間特徴そのものをどう作るか (BEV エンコーダの設計、マルチビューカメラからの持ち上げ) が、条件付けの質を左右する上流の研究テーマとして連動しています。研究上の未解決課題としては、条件忠実度と多様性のトレードオフ (条件を強く効かせると出力が画一化する)、長い条件列でのアテンションの効率と精度の両立が中心にあります。
