# Frozen Backbone

## ひとことで言うと
他の大規模データで事前学習済みの「画像から特徴を取り出す部品 (バックボーン)」の重みを、学習中いっさい変えずに固定したまま使い、その後ろに付けた小さな部品 (ヘッド) だけを学習させるやり方です。バックボーンを「すでに完成した目」として使い回すことで、計算とメモリを節約しつつ、少ないデータでも安定して学習できます。転移学習における2大流儀のうち「特徴抽出 (feature extraction)」と呼ばれる側にあたり、もう一方の「微調整 (fine-tuning)」と対をなします。

## 直感的な理解
熟練の写真家がいて、どんな写真を見ても瞬時に「ここに人がいる」「これは縁石だ」と見抜けるとします。あなたのタスクが「その写真からどう運転すべきか決める」ことなら、写真家の目を一から鍛え直す必要はありません。写真家の目はそのまま借りて、あなたは「見抜いた情報から判断を下す部分」だけを学べばよい。Frozen Backbone はまさにこれです。

ニューラルネットワークで画像を扱うとき、最初に「画像 → 特徴 (エッジ・模様・物体らしさなどの抽象的な手がかり)」へ変換する大きな部品が必要です。これをバックボーン (backbone, 背骨) と呼びます。バックボーンをゼロから学習させるには膨大なデータと計算が要ります。ImageNet (約128万枚のラベル付き画像) などの大規模データで学習済みのバックボーンは、すでに汎用的な視覚特徴を獲得しています。これを使い回すのが転移学習で、その中でもバックボーンの重みを一切更新せず固定するのが Frozen Backbone です。

なぜ「固定する」ことにあえて意味があるのか。一見すると、せっかく手元のタスク用データがあるなら全部使ってバックボーンも調整した方が良さそうに思えます。しかし手元のデータが少ないと、バックボーンまで動かすと「少ないデータの細部を丸暗記する (過学習)」方向に汎用特徴が壊れてしまう。固定すれば壊しようがない、というのが核心です。

## 基礎: 前提となる概念
- バックボーン: 特徴抽出を担う基幹ネットワーク。畳み込み系の ResNet・ConvNeXt、Transformer 系の ViT・Swin Transformer などが代表例。
- 重み (パラメータ): 学習で決まる内部の数値。「凍結 (freeze)」とはこの数値を更新対象から外すこと。
- 勾配 (gradient): 損失 (誤差) を各重みで微分した値で、「その重みをどちらにどれだけ動かせば誤差が減るか」を示す。学習はこの方向に重みを少し動かす。凍結部分では勾配を流す・貯める必要がない。
- 順伝播 (forward) と逆伝播 (backward): 入力から出力へ計算するのが順伝播、出力側の誤差を入力側へさかのぼって各層の勾配を求めるのが逆伝播。凍結部分でも順伝播は必要 (特徴を作るため) だが、その手前まで逆伝播を流す必要はない。
- ヘッド: バックボーンの後ろに付くタスク専用の小部品。凍結方式ではここだけ学習する。
- linear probing (線形評価): 凍結したバックボーンの出力に「線形分類器 (重み付き和を取るだけの1層)」だけを載せて評価する手法。事前学習済み特徴の良さそのものを測る標準的なものさし。なぜ線形か――もし特徴がすでに「クラスごとにきれいに分かれている (線形分離可能)」なら、単純な直線 (超平面) で切り分けられるはず、という発想です。

転移学習には大きく2つの流儀があります。
1. Fine-tuning (微調整): 事前学習済みの重みを初期値とし、下流データで全体を再学習する ([Fine-tuning](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0384_fine-tuning))。
2. Frozen Backbone (凍結バックボーン): バックボーンを固定し、後ろのヘッドだけ学習する。

Frozen Backbone が選ばれる理由は主に次の3点です。
- 学習データが少ない: バックボーン全体を再学習すると、少ないデータに過度に適合して汎用特徴を壊しかねない。固定すれば壊れない。
- 計算・メモリを節約したい: 固定部分は勾配計算と重み更新が不要なので学習が速く、GPU メモリも少なくて済む。
- 安定した特徴が欲しい: 同じ入力には常に同じ特徴が出る (再現性が高い)。後段の設計・検証がしやすく、後述する自己教師あり学習の「動かないターゲット」としても都合が良い。

## 仕組みを詳しく
通常の学習では、ネットワーク全体のすべての重みについて勾配を計算し、少しずつ更新します。Frozen Backbone では、バックボーン部分だけこの更新を止めます。深層学習フレームワークでは典型的に次を行います。

- 各パラメータの勾配計算フラグをオフにする (PyTorch なら `param.requires_grad = False`)。これで最適化器が更新対象から外す。最適化器に渡すパラメータ集合自体から除く実装もある。
- バックボーンを評価モード (`.eval()`) にし、学習時特有の挙動である Dropout (一部のユニットをランダムに無効化する正則化) や [Batch Normalization](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0619_batch-normalization) の統計更新を止め、推論と同じ安定した振る舞いにする。ここを忘れると、凍結したつもりでも BatchNorm の移動平均だけが入力に応じて動き続け、「出力が静かに変わる」という分かりにくいバグになる。
- 必要なら勾配を貯めないモード (`torch.no_grad()`) のもとで順伝播を行い、活性値の保存 (逆伝播のために通常はメモリに残す中間結果) すら省いて、さらにメモリを削る。

### データの流れ (tensor の形で)
カメラ画像 `[B, 3, 256, 256]` (B はバッチサイズ。たとえば B=8、RGB 3チャンネル、256×256 ピクセル) を例にします。

```
入力:  [B, 3, 256, 256]
        │  frozen backbone (重み固定、requires_grad=False)
        ▼
多段特徴 (features-only モード):
        [B,  96, 64, 64]   ← 浅い: エッジ・色
        [B, 192, 32, 32]
        [B, 384, 16, 16]
        [B, 768,  8,  8]   ← 深い: 物体らしさ
        │  ここまで勾配は流れない (重み不変)
        ▼
   学習対象ヘッド (融合・予測など)  ← ここだけ勾配が流れ、更新される
```

1. 凍結バックボーンに入力 → 複数スケールの特徴マップが出る。途中の多段特徴だけを返すモード (features-only) では上記のような4段の特徴が返る (具体的なチャンネル数・解像度はバックボーンにより異なる)。
2. ここまでは重みが固定なので、勾配は流れず重みも変わらない。
3. この特徴を後段の学習対象ヘッドに渡し、ヘッドの重みだけを誤差で更新する。

before/after で言うと、学習を何ステップ回しても「バックボーンの重み」は1ステップ目とまったく同じ (before == after) で、変わるのはヘッドの重みだけです。なぜこれで十分なのか――Sharif Razavian et al. (2014) や Donahue et al. (2014) が示したように、ImageNet で学んだ中間特徴は別タスクにとっても「線形分離可能なほど良い」表現になっているからです。賢い目が出した特徴は単純なヘッドでも十分使える、という経験則が前提にあります。

### 凍結を使う典型シーン: 自己教師あり学習のターゲット
凍結バックボーンは、自己教師あり学習 (ラベルなしデータから学ぶ枠組み) で「予測のターゲット (正解側)」を作るのにも使われます。代表が JEPA (Joint-Embedding Predictive Architecture: 画素ではなく特徴空間で未来や欠損を予測する考え方) です。未来フレームや隠した領域の「正解特徴」を凍結した (あるいは EMA = 指数移動平均でゆっくり追従する) ターゲット側エンコーダで抽出し、予測側はそれに合わせるよう学習します。

なぜターゲットを固定するのか。もし予測側とターゲット側が同時に自由に学習すると、両者が結託して「どんな入力にも同じ一定ベクトルを出す」自明解に落ちられます。これを表現崩壊 (representation collapse) と呼びます。損失はゼロになりますが特徴は無価値です。ターゲットを止める (stop-gradient) ことで、片側だけが追いかける非対称な関係を作り、崩壊を防ぎます。

実務上の注意として、凍結バックボーンの出力次元と後段が期待する次元が合わないことがよくあります。features-only で取り出した特徴 (例 `[B, 768, 8, 8]`) と、後段が想定する埋め込み次元 (例 256) が食い違う場合、1×1 畳み込みや線形層といった射影層 (projection) を1枚挟んで次元を揃えるのが定石です。射影層は学習対象に含めます。

## 手法の系譜と主要論文
- Donahue et al., "DeCAF", ICML 2014。ImageNet 学習済み CNN の中間活性を固定特徴として取り出し、別タスクで使う発想を広めた。深層特徴の汎用性を初めて広く示した。
- Sharif Razavian et al., "CNN Features off-the-shelf", CVPRW 2014。固定特徴 + 線形分類器だけで当時の専用 SOTA を上回ると示し、凍結特徴抽出の強さを決定づけた。論文タイトルの "off-the-shelf (棚から出してそのまま)" という言い回しがこの流儀の代名詞になった。
- Yosinski et al., "How transferable are features in deep neural networks?", NeurIPS 2014。凍結すると深い層ほど性能が落ちる (元タスク特化のため) ことを定量化。凍結の限界と fine-tuning の必要性を明確化した。
- Chen et al., "SimCLR", ICML 2020 / He et al., "MoCo", CVPR 2020。事前学習の質を測る標準として linear probing (バックボーン凍結 + 線形ヘッド) を確立し、「凍結したまま使える特徴をどう作るか」を中心課題にした。
- Kumar et al., "Fine-Tuning can Distort Pretrained Features and Underperform Out-of-Distribution", ICLR 2022。凍結 (線形評価) の方が分布外で fine-tuning より強くなりうることを示し、凍結と微調整を二段で組む LP-FT を提案。
- Assran et al., "I-JEPA", CVPR 2023。画素ではなく特徴空間で予測する JEPA を画像に適用。凍結 (EMA) ターゲットエンコーダの設計が崩壊回避と学習効率の鍵であることを示した。

系譜としては「固定特徴は使える (2014)」→「凍結すると深層で限界がある (2014)」→「凍結特徴の質を linear probing で測り、自己教師ありで質を底上げ (2020-)」→「凍結ターゲットを使った特徴空間予測 (2023-)」と進んでいます。

## 論文の実験結果(定量データ)
Sharif Razavian et al. (2014) は、ImageNet で学習した OverFeat の固定特徴に線形 SVM を載せただけで、Pascal VOC 2007 物体分類 (mean Average Precision = mAP, 物体の有無を当てる平均適合率) や各種シーン・細粒度認識で当時の専用手法に匹敵・凌駕すると報告。「再学習なしの off-the-shelf 特徴」が強力なベースラインになることを示しました。

Yosinski et al. (2014) は、転移した層を凍結したまま使うと top-1 accuracy (最も確率の高い予測が正解だった割合) が層の深さとともに低下する一方、fine-tuning すれば回復することを定量化。これは「凍結は省コストだが精度の上限では fine-tuning に劣りうる」という凍結 vs 微調整のトレードオフを数値で裏づけたものです。

自己教師あり学習の標準評価 linear probing では、凍結バックボーン + 線形ヘッドだけの設定で、ImageNet top-1 accuracy が手法と規模により70%台前半〜後半に達します。SimCLR (2020) は ResNet-50 で約69.3%、より大きいモデル (ResNet-50 4x) で76.5%、MoCo v2 は ResNet-50 で約71.1% と報告。これは「凍結特徴に1層足すだけ」でこれだけ当てられることを意味し、凍結バックボーンの実用性を裏づけます。

Kumar et al. (2022) は、分布内 (ID) では fine-tuning が linear probing を上回るが、分布外 (OOD) では逆転しうることを示しました。一例として、事前学習特徴が良いほど、全層 fine-tuning は最初に動く下層がランダム初期化のヘッドに引きずられて特徴を歪め、OOD 精度を落とす。先に凍結で線形ヘッドだけ学習し収束させてから fine-tune する LP-FT が、ID と OOD の双方で最良に近い結果になると報告。「いつ凍結すべきか」を分布シフトの観点から定量化した重要な結果です。

アブレーション的な含意として、凍結特徴の精度は (1) どの層から取り出すか、(2) 事前学習タスクが下流とどれだけ近いか、に強く依存し、ドメインが離れるほど fine-tuning との差が開きます。

## メリット・トレードオフ・限界
メリット
- 学習が速い: 固定部分の勾配計算・重み更新が不要。
- メモリ節約: 勾配や最適化器の状態 (Adam なら各重みごとに1次・2次モーメントの2つ) を凍結部分について持たなくてよい。
- 少データに強い: 汎用特徴を壊さないため過学習しにくい。
- 安定・再現性: 同じ入力に常に同じ特徴。後段の検証や、自己教師あり学習の動かないターゲットとして都合が良い。

トレードオフ・限界
- 精度の上限が下がりうる: バックボーンを下流に合わせて調整できないため、ドメインのずれが大きいと固定特徴が最適でない。
- ドメインギャップに弱い: ImageNet (昼間の鮮明な写真中心) と、夜間・雨・魚眼などの特殊な分布は見た目が異なり、固定特徴がそのまま最良とは限らない。
- 次元・解像度の不一致: 凍結出力と後段が期待する次元が合わず、射影層など追加部品が必要になることがある。
- 中間的選択肢が無視されがち: 「全部凍結」と「全部学習」の間に、後段の数層だけ解凍する部分凍結や段階的解凍 ([Gradual Unfreezing](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0388_gradual-unfreezing))、層ごとに学習率を変える ([Discriminative / Layer-wise Learning Rates](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0380_discriminative-learning-rates)) がある。完全凍結はその一端にすぎない。
- 未解決の課題: どの層を凍結し、どの層を解くと最も費用対効果が高いかの自動決定は依然オープン。分布シフトの種類によって解くべき層が変わる (surgical fine-tuning, Lee et al. 2023) という知見もあり、一律のレシピは存在しない。

## 発展トピック・研究の最前線
- linear probing vs fine-tuning の使い分け: 上記 Kumar et al. (2022) の LP-FT のように、まず凍結で線形ヘッドを学習し、その後 fine-tune する二段構えが OOD ロバスト性で有効。
- 自己教師あり凍結特徴の高品質化: MAE (He et al. 2022)・DINO (Caron et al. 2021)・I-JEPA など、凍結したまま強い特徴を作る事前学習の進展。これらは linear probing 精度を年々押し上げている。
- 凍結基盤モデル + 軽量アダプタ: 巨大な凍結バックボーンに小さな学習部品 (アダプタ・LoRA・プロンプト) だけを足す PEFT 的アプローチが、計算効率と性能の両立で主流化しつつある。凍結は「全部固定」、PEFT は「ほぼ固定 + 少しだけ可変」と捉えると連続的に理解できる。
