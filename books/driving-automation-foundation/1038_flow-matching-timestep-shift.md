# フローマッチングのシフト時刻スケジュール

## ひとことで言うと

フローマッチングという生成手法は、でたらめなノイズから本物のデータへ「時刻 t=0 から t=1 まで」少しずつ進めて作ります。推論を速くするためにステップ数を減らす(たとえば 50 ステップを 5 ステップに)と、一度に進む区間が荒くなって生成物の質が落ちます。シフト時刻スケジュールは、推論時に時刻 t を `t_shifted = (αt)/(1+(α-1)t)` という式で歪め、ステップの取り方を「本当に丁寧さが要る区間」に偏らせることで、少ないステップでも質を保つ工夫です。学習はそのまま変えず、推論での時刻の刻み方だけを変える、追加コストほぼゼロの改良です。

## 直感的な理解

ジグソーパズルを組むことを想像してください。最初の数ピース(枠や角)を置く段階は大ざっぱでよく、適当に進めても大勢に影響しません。しかし完成間際、細部のピースをはめる段階は一手一手が効いてきます。もし手数(ステップ)が限られているなら、序盤に手数を使いすぎず、細部の効く終盤に手数を多く残したい。シフト時刻スケジュールがやっているのは、まさにこの「手数の配分」です。

生成モデルでも同じことが観察されます。生成の序盤(t が小さく、ノイズに近い側)は全体の方向がざっくり決まる段階で、終盤(t が 1 に近く、完成間際)で細かい形が固まることが多い。時刻 t を等間隔(uniform)に刻むと、本当に丁寧さが要る終盤に十分なステップを割けません。そこで「重要な区間に刻みを多く配る」よう t を歪めるのがシフト時刻です。車載チップのように計算予算が厳しく、どうしても少ステップで動かさなければならない状況で効いてきます。

## 基礎: 前提となる概念

順に噛み砕きます。

- 生成モデル(generative model): ノイズなどの単純な入力から、本物らしいデータ(画像・軌道・音声など)を作り出すモデルの総称。
- フローマッチング(Flow Matching): ノイズ(でたらめな点)を出発点に、学習した「速度の場(velocity field, 各位置でどっちへどれだけ動くべきか)」に従ってデータへ運ぶ生成手法です。時刻 t は 0(完全なノイズ)から 1(完成データ)まで進みます。形式的には、ノイズ点 x0 とデータ点 x1 を直線でつなぎ、その途中点 xt = (1-t)x0 + t x1 における速度 x1 - x0 をニューラルネットに予測させて学習します。推論時はこの速度に従って t=0 から数値積分で進めます。
- 速度の場(velocity field): 「いまこの位置・この時刻にいるなら、次はこの向きにこれだけ動け」を返す関数。学習したネットワークがこれを担います。
- ステップ数(NFE, Number of Function Evaluations): t を何回に分けてネットワークを評価し動かすか。50 ステップなら細かく正確だが遅い。5 ステップなら速いが各ステップで大きく動くので数値積分の誤差が出やすい。推論コストはほぼこのステップ数に比例します。
- 拡散モデル(diffusion model): フローマッチングと近縁の生成手法で、ノイズを少しずつ除去してデータを作ります。両者は数学的に深くつながっており、サンプリング高速化の工夫も共通点が多いです。

時刻スケジュール(time schedule)とは「推論時にどの t たちでネットワークを評価するか」の取り決めです。等間隔に {0, 0.25, 0.5, 0.75, 1.0} と取るのが素朴な uniform スケジュール。シフトはこれを非等間隔に置き換えます。

## 仕組みを詳しく

推論時、本来 t を 0〜1 で等間隔に取るところを、各 t を次の式で別の値へ写します(α は歪みの強さを決める定数、α>1)。

```
t_shifted = (α · t) / (1 + (α - 1) · t)
```

性質を数値で確認します(例として α=3)。

- t=0 → t_shifted = 0(端は不動)
- t=1 → t_shifted = (3)/(1+2) = 1(端は不動)
- t=0.5 → (3·0.5)/(1+2·0.5) = 1.5/2 = 0.75
- t=0.25 → (0.75)/(1+0.5) = 0.5
- t=0.75 → (2.25)/(1+1.5) = 0.9

つまり等間隔だった t = {0, 0.25, 0.5, 0.75, 1.0} は、写像後に {0, 0.5, 0.75, 0.9, 1.0} となり、終盤(1 寄り)に点が密集します。区間ごとの刻み幅で見ると、序盤は粗く(0→0.5 を 1 ステップで飛ぶ)、終盤は細かく(0.9→1.0 を丁寧に進む)なります。同じ 5 ステップでも、丁寧さが要る終盤に多くの評価が配られるわけです。α を大きくするほど偏りが強まります。

```
等間隔 t : |. . . . .|   (0, 0.25, 0.5, 0.75, 1)
shifted  : |.  .  . ..|   (終盤に密集; 0, 0.5, 0.75, 0.9, 1)
            0          1
```

なぜこの形の式なのか。この変換は単調増加(t が増えれば t_shifted も増える)で、両端 0 と 1 を固定し、中間を一方向にずらす性質を持ちます。逆向き(序盤を密に)したければ α<1 にすればよく、α=1 なら恒等写像(uniform と同じ)に戻ります。連続でなめらかなので、刻みの密度が急にジャンプせず、数値積分が安定します。

最も重要なのは「学習を一切変えない」点です。学習済みの速度場はそのまま使い、推論時にどの t で速度場を評価するか(刻み方)だけを変えます。実装上も `timestep_schedule = "uniform" | "shifted"` のような小さなフラグの追加で済み、デフォルトを uniform にしておけば既存の挙動と完全に同じ(後方互換)です。リスクが低い改良である理由はここにあります。

## 手法の系譜と主要論文

少ステップでも質を保つ、という目標に対して、研究は「学習で経路を直線化する」流れと「推論手順を工夫する」流れの二系統で発展してきました。

- DDIM (Song et al., ICLR 2021, arXiv:2010.02502): 拡散モデルで「学習を変えずにサンプリング手順だけ工夫して少ステップ化する」流れの出発点。決定論的なサンプラを導入し、数十ステップでの高速生成を可能にしました。シフト時刻はこの「推論側を工夫する」思想のフローマッチング版と位置づけられます。
- Flow Matching for Generative Modeling (Lipman et al., ICLR 2023, arXiv:2210.02747): フローマッチングの基礎理論。確率の流れを時刻 t に沿って学ぶ枠組みを定式化しました。t の刻み方(スケジュール)を変える自由度がこの枠組みに最初から内在しており、シフト時刻はその自由度を使い切る工夫です。
- Rectified Flow (Liu, Gong, Liu, ICLR 2023, arXiv:2209.03003): ノイズからデータへ「まっすぐ」運ぶ経路を学ぶことで、極端には 1 ステップ生成すら可能にする理論。これは学習側で経路を直線化するアプローチです。シフト時刻は学習に手を入れず推論側だけで似た目的を達成する、補完的な工夫と言えます。
- Stable Diffusion 3 (Esser et al., 2024, arXiv:2403.03206): 大規模画像生成で、解像度に応じて時刻スケジュールをずらす timestep shift を明示的に議論しました。高解像度画像ほどノイズの効き方が変わるため、シフト量を解像度依存で調整します。シフト時刻が画像生成の実運用でも有効である裏付けです。
- GoalFlow (Xu et al., arXiv:2503.05689, 2025): 軌道生成にフローマッチングを使い、推論時にこのシフト時刻スケジュールを採用しました。アブレーションで「シフトの効果は低ステップ数のときに最も大きい」ことを示しています。少ステップ・低レイテンシが要る車載計画への応用として、シフト時刻を軌道生成に持ち込んだ代表例です。

系譜をまとめると、DDIM が「推論手順の工夫で速くする」を切り開き、Lipman らがフローマッチングの土台を作り、Rectified Flow が「学習で直線化」を、SD3 や GoalFlow が「推論時の時刻シフト」を実用に乗せた、という流れです。

## 論文の実験結果(定量データ)

定量評価で押さえるべき点を、関連研究の報告から整理します。

- 軌道生成での評価指標は ADE(Average Displacement Error, 予測軌道と正解軌道の各時刻の距離の平均。単位はメートル、小さいほど良い)や、計画ベンチマークの複合スコア PDMS(NAVSIM)が使われます。GoalFlow は NAVSIM navtest で PDMS 約 90.3 を報告し、当時の最上位級でした。シフト時刻はこの推論を少ステップで回しても品質を保つ役割を担います。
- ステップ数依存性が肝心です。GoalFlow 系のアブレーションでは、多ステップ(たとえば 20 ステップ以上)ではシフトの有無で差がほとんど出ない一方、少ステップ(数ステップ)に絞ると、uniform だと ADE/PDMS が目に見えて悪化するのに対しシフトを入れると劣化が抑えられる、という傾向が報告されています。これは直感(序盤は粗くてよく終盤が効く)と整合します。数値の意味としては「同じ計算予算(同じステップ数)で、より良い軌道が出る = 車載のレイテンシ制約下で性能を落とさずに済む」ということです。
- 画像生成側でも、Stable Diffusion 3 (arXiv:2403.03206) は適切な時刻シフトを入れることで、特に高解像度での FID(Fréchet Inception Distance, 生成画像の質を測る指標。小さいほど本物に近い)が改善したと報告しています。解像度が上がるほどシフト量を強めるのが有効、という解像度依存の効果も示されました。
- Rectified Flow (arXiv:2209.03003) は、経路を直線化する reflow という追加学習を重ねることで、1〜2 ステップ生成でも FID をかなり低く保てることを示しました。これは「学習側」での少ステップ化であり、推論側のシフト時刻と組み合わせるとさらに少ステップ化を押し進められる可能性を示唆します。

要するに、シフト時刻の効果は「ステップ数を絞ったときの品質劣化を緩和する」ところに集中して現れ、ステップ数が潤沢ならほとんど差が出ません。だからこそ、組込みのように少ステップが必須の場面で価値が高い改良です。

## メリット・トレードオフ・限界

メリット

- 学習不要。学習済みモデルにそのまま後付けできる。
- 少ステップ推論の質を保てるため、車載・組込みの低レイテンシ要件に効く。
- デフォルトを uniform にすれば完全後方互換で、リスクの低い追加。
- 連続でなめらかな写像なので数値積分が安定し、α=1 で恒等に戻せる安全性がある。

トレードオフ・限界

- α という新しいハイパーパラメータが増え、適切な値を経験的に探す必要がある。最適 α はステップ数・対象データに依存します。
- 効果はステップ数依存。多ステップではほとんど差が出ず、メリットが消える。
- α を不適切にすると、丁寧さが要る区間からむしろ刻みを奪ってしまい、質が下がる可能性がある。
- この一次元的なシフト式が最適なスケジュールである保証はない。本来の最適スケジュールは速度場の曲率(経路がどこで急に曲がるか)に依存するはずで、固定式はその近似にすぎません。

研究上の未解決課題として、(1) スケジュールをデータや時刻ごとに学習で最適化する(固定式をやめる)アプローチ、(2) シフト時刻と高次の数値積分法(Heun 法など)を組み合わせて少ステップ品質をさらに上げる、(3) 軌道生成特有の構造(終端の到達点が効く)を踏まえたタスク特化のスケジュール設計、などが考えられます。

## 発展トピック・研究の最前線

少ステップ生成の最前線は、推論時の工夫だけでなく蒸留(distillation)に向かっています。Consistency Models (Song et al., ICML 2023) や蒸留系の手法は、多ステップのサンプラを 1〜数ステップで模倣するよう生徒モデルを学習させ、品質を保ったまま劇的に高速化します。シフト時刻が「学習を変えない安価な改良」であるのに対し、蒸留は「追加学習を払ってでも速くする」アプローチで、両者は予算と要求品質に応じて使い分けられます。

また、フローマッチングは画像だけでなくロボット制御・行動生成にも広がっています。π0 のようなロボット方策(policy)はフローマッチングで連続的な行動列を生成しており、リアルタイム制御のために少ステップ推論が必須です。ここでもシフト時刻のような推論時スケジュール最適化が現実的な手段として注目されています。時刻スケジュールという小さな自由度が、生成品質と計算コストの境界線をどこまで押し下げられるか、が研究の関心事です。
