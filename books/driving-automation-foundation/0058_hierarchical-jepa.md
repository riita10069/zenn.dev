# 階層JEPA (H-JEPA)

## ひとことで言うと
階層 JEPA(Hierarchical JEPA, H-JEPA)とは、未来予測を「短い時間の細かい予測」と「長い時間のざっくりした予測」など複数の層に分けて積み重ねる構想です。上の層ほど抽象的で長期の計画を、下の層ほど具体的で短期の制御を担い、上の層が下の層に目標(サブゴール)を与えることで、長期計画と短期制御を橋渡しします。Yann LeCun が 2022 年に提唱した、自律機械知能(autonomous machine intelligence)の中核アーキテクチャの一つです。

## 直感的な理解

自動運転車が「3km 先の目的地に着く」という計画を立てるとします。これを 1 ステップ 0.1 秒の細かい予測だけで表そうとすると、3km ぶんの全アクション(ハンドル・アクセルの操作列)を 0.1 秒刻みで何万ステップも先まで予測しないといけません。これは現実的に不可能です。遠い未来ほど不確実性が雪だるま式に増え、0.1 秒刻みの細かい予測は数秒先でほぼ無意味になります。

逆に、粗い予測だけにすると「次の瞬間ハンドルを何度切るか」という実際の制御に必要な細かさが出せません。

人間の運転を考えると、私たちは無意識に複数の時間スケールを同時に扱っています。

- 長期(分単位): 「次の交差点を右折して、その先の高速に乗る」という抽象的な計画。具体的なハンドル角は考えていない。
- 中期(秒単位): 「前の車との車間を保ちつつ右車線に移る」という戦術。
- 短期(0.1 秒単位): 「今この瞬間ハンドルを 2 度右に、アクセルを少し緩める」という具体的な制御。

LeCun はこの「複数の抽象度・時間スケールの予測を階層的に積む」構造を機械にも持たせるべきだと提案しました。これが H-JEPA です。

## 基礎: 前提となる概念

JEPA(Joint Embedding Predictive Architecture)とは「ピクセルではなく特徴空間で未来を予測する」自己教師あり世界モデルの枠組みです(詳しくは [JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))。H-JEPA はこれを縦に積み重ねたものです。

世界モデル(world model)とは「今の状態とこれからとる行動から、未来の状態を予測するモデル」です。これがあると、実際に行動する前に頭の中で結果をシミュレーションして計画できます。

時間抽象(temporal abstraction)とは「複数の細かいステップを 1 つのまとまり(たとえば右折という出来事)として扱う」考え方です。上位の層が 1 ステップを大きくまとめることで、少ないステップ数で遠い未来まで届きます。

サブゴール(subgoal)とは、最終目標までの途中目標です。上位の層が「30 秒後に右車線にいる」というサブゴールを決め、下位の層がそれを実現する具体的な操作列を埋めます。

階層的強化学習(Hierarchical Reinforcement Learning)とは、行動を「上位ポリシーが選ぶ大きな方針」と「下位ポリシーが実行する具体行動」に分ける枠組みで、H-JEPA と発想を共有します。

## 仕組みを詳しく

H-JEPA は JEPA を縦に積み重ねた構造です。各層が異なる時間スケールと抽象度を担当します。

```
            ┌──────────────────────────────────────────┐
レベル2(高) │ 抽象表現 s2 ─► predictor2 ─► ŝ2(数秒〜分先)  │  長期・抽象・粗い時間刻み
            └──────────────┬───────────────────────────┘
                           │ 上の層が下の層に「目標(サブゴール)」を与える
            ┌──────────────▼───────────────────────────┐
レベル1(低) │ 具体表現 s1 ─► predictor1 ─► ŝ1(数百ms先)   │  短期・具体・細かい時間刻み
            └──────────────────────────────────────────┘
```

ポイントは 2 つです。

1. 時間スケールの違い。上の層は「次に何ステップ進むか」を 1 単位で大きくまとめます。下の層が 1 ステップ = 0.1 秒なら、上の層は 1 ステップ = 数秒に相当します。これにより上の層は少ないステップ数で遠い未来まで届きます。

2. 抽象度の違い。上の層の特徴 s2 は、下の層の特徴 s1 よりさらに圧縮された「意味のかたまり」です。s1 が「車線中央に対する横方向のずれ」のような具体量を持つのに対し、s2 は「右折を完了した」のような出来事レベルの抽象を持ちます。

計画の流れ(推論時)はトップダウンです。まず上の層が長期の抽象的なサブゴール(例: 「30 秒後に右車線にいる」)を決め、それを下の層に条件として渡し、下の層が具体的な短期アクション列を埋めます。学習時は各層が JEPA と同じく特徴空間での予測誤差(MSE)で学習し、ターゲット側は凍結/EMA([凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))で崩壊を防ぎます。

数値例で時間スケールの効果を示します。6.4 秒先まで予測したいとき、

- 単層で 0.1 秒刻み → 64 ステップ予測が必要。遠い側はほぼ予測不能で、誤差も累積して発散しやすい。
- 2 層 H-JEPA: 下層 0.1 秒刻みで直近 1.6 秒(16 ステップ)、上層 1.6 秒刻みで 4 ステップ(6.4 秒)→ 各層は短いステップ数で済み、上層は粗くて良いので不確実性に耐えられる。

この「上層を粗い時間刻みにする」発想は、多ステップ未来特徴予測([多ステップ未来特徴予測と予測ホライズン設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0019_temporal-feature-prediction-horizon))が粗い刻みで遠い未来まで届かせる設計と通じます。単層でも刻みを粗く取れば H-JEPA の下位レイヤ的な効果を部分的に得られ、その上に長期のルート計画レイヤを積めば H-JEPA に近づきます。

## 手法の系譜と主要論文

H-JEPA は「特徴空間予測」と「階層的時間抽象」という 2 つの流れの合流点にあります。

特徴空間予測の流れ。LeCun の「A Path Towards Autonomous Machine Intelligence」(2022)が JEPA([JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))を提唱し、その階層版として H-JEPA を構想しました。提案内容は「世界モデルを階層化し、各層が異なる抽象度・時間スケールで未来を予測する。上の層が下の層にサブゴールを与えることで長期計画と短期制御を橋渡しする」。動機は「単一スケールでは遠い未来の不確実性に対処できず、抽象的計画と具体的制御を同時に扱えないから」です。これは構想段階の提案であり、当時は完全な実装・検証が存在しませんでした。

階層的時間抽象の流れ。Sutton, Precup, Singh の Options framework(Artificial Intelligence, 1999)が源流です。上位ポリシーが「オプション(一定期間続く行動方針)」を選び、下位ポリシーが具体行動を実行する構造で、H-JEPA の階層的時間抽象と発想を共有します。後続の FeUdal Networks(Vezhnevets et al., ICML 2017)は、上位の Manager が抽象的な方向(サブゴール)を低頻度で出し、下位の Worker が高頻度で具体行動をとる、という深層 RL での階層を実装し、Montezuma's Revenge のような長期報酬の難タスクで優位を示しました。これは H-JEPA のトップダウンなサブゴール受け渡しと構造が酷似します。

世界モデルの流れ。Ha & Schmidhuber の World Models(2018)、Hafner et al. の Dreamer 系(PlaNet 2019, Dreamer 2020, DreamerV2 2021, DreamerV3 2023)が潜在空間でのロールアウトを行い、想像の中で方策を学びます。ただしこれらは基本的に単一時間スケールです。Director(Hafner et al., 2022)は Dreamer に階層を入れ、上位が潜在サブゴールを出し下位がそれを追う構成で、長期タスクの性能を上げました。H-JEPA はこれらを「特徴空間予測(JEPA)+ 多時間スケール階層」として統合する方向性に位置づけられます。

これらを並べると、H-JEPA は「JEPA の特徴空間予測」と「階層的時間抽象(Options / FeUdal / Director)」を一つの自己教師あり世界モデルとして統合しようとする構想だと整理できます。

## 論文の実験結果(定量データ)

H-JEPA そのものは構想段階のため、独立した大規模なベンチマーク結果は確立していません。ここでは発想を共有する関連手法の定量結果から、階層化の効果を読み取ります。

FeUdal Networks(Vezhnevets et al., 2017)は、報酬がまばらで長期の探索を要する Atari の Montezuma's Revenge で、階層なしの A3C ベースラインが事実上スコア 0 付近に留まるのに対し、Manager-Worker の階層構造でスコアを大きく改善したと報告しました。指標はゲームスコア(高いほど良い)で、これは「長期にわたる一貫した行動が必要なタスクで、抽象的サブゴールを与える上位層が探索を導く」効果を示します。

Director(Hafner et al., 2022)は、Dreamer に潜在サブゴールの階層を加えた手法で、報酬が遠い(エピソード末尾でしか得られない)長期タスク群において、階層なしの Dreamer を上回る成功率を報告しました。とくに「ゴールまでの中間サブゴールを潜在空間で立てられる」ことが、長い時間地平のタスクで効いたとされます。

DreamerV3(Hafner et al., 2023)は単一スケールながら、Minecraft でダイヤモンドを採掘するという長期・複合タスクを、ハイパーパラメータ調整なしで人間の事前知識なしに達成したと報告し、世界モデルによる計画の到達点を示しました。これは「単一スケール世界モデルがどこまで行けるか」の基準線であり、H-JEPA のような階層化が「ここからさらに長期へ」を狙う動機を与えます。

これらの数値の意味は「報酬や情報が遠い長期タスクほど、階層的に時間を抽象化する構造が単一スケールを上回る」という傾向です。H-JEPA はこの傾向を特徴空間予測の枠組みで実現しようとする構想だと読めます。

## メリット・トレードオフ・限界

メリット。長期の抽象計画(どこへ向かうか)と短期の具体制御(今ハンドルをどう切るか)を 1 つの枠組みで両立できます。上の層が粗い時間刻みなので、遠い未来でも少ないステップ数で予測でき、誤差累積と不確実性の爆発を抑えられます。人間の階層的な運転判断に構造が近く、解釈しやすい面があります。

トレードオフと限界。構想が先行しており、JEPA 単層(I-JEPA, V-JEPA)に比べると、H-JEPA としての大規模実装・検証はまだ発展途上です。層間のサブゴール(上が下に何を渡すか)の表現設計が難しく、誤ると上下の層が噛み合わず、上位が下位に実現不能な目標を出す/下位が上位の目標を無視する、といった失敗が起こります。単層より構造が複雑で、学習・デバッグの難易度が上がります。研究上の未解決課題として、サブゴール表現をどう自動的に学ぶか(人手で決めると汎用性を失う)、各層の時間スケールをどう決めるか、層をいくつ積むのが最適かが定まっていない点が挙げられます。

## 発展トピック・研究の最前線

H-JEPA に近づく具体的な動きとして、世界モデルの階層化(Director)、行動条件付き特徴予測の大規模化(V-JEPA 2, 2025)、潜在サブゴールの自動発見(教師なしでサブゴールを見つける研究)があります。自動運転では、ルート計画(分単位)・戦術判断(秒単位)・制御(0.1 秒単位)を別レイヤに分けるアーキテクチャが、長期の運転意図と即時の制御を結ぶ手段として検討されています。特徴空間予測([JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))と多ステップ予測([多ステップ未来特徴予測と予測ホライズン設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0019_temporal-feature-prediction-horizon))を土台に、上位レイヤを積む方向が現実的な発展経路です。サブゴールの解釈可能性、層間の整合性の保証、複数層の同時学習の安定化が今後の焦点です。
