# 反応系と世界予測系を分ける2バンドアーキテクチャ

## ひとことで言うと
自動運転の end-to-end モデルを、更新頻度の異なる2つの系に分ける設計のことです。1つは「速い反応系」で、カメラと地図を見て高頻度(例えば 10Hz=毎秒10回)で軌道を出します。もう1つは「遅い世界予測系」で、長い視覚履歴を溜めて低頻度(例えば 1Hz)で未来を予測します。人間でいうと「とっさの反射」と「じっくり考える脳」を分けたイメージです。速い系を軽く保ったまま、重い未来予測を別系統に載せられるのが利点です。

## 直感的な理解
運転中、あなたの脳は2種類の処理を同時にこなしています。1つは「前の車が急ブレーキした、すぐ踏む」という即応。もう1つは「この交差点は混みそうだ、車線を変えておこう」という先読み。前者は 0.1 秒以内に反応しないと事故になる。後者は 1 秒くらいかけてゆっくり考えても間に合う。

ニューラルネットワークで運転全体を1つの巨大なモデルにすると、この2つが同居します。すると問題が起きます。第一に、先読み(重い計算)が即応(軽くあるべき計算)を引きずって、10Hz に間に合わなくなる。第二に、役割が混ざってどこを直せばいいか分からなくなる。第三に、未来予測のための重い計算が毎フレーム走ってメモリを圧迫する。

そこで、システムを2つの帯域(バンド)に分けます。速い反応系は毎フレーム軽く回し、遅い世界予測系は数フレームに1回だけ重く回す。世界予測系が作った「これまでの流れの要約」を反応系に渡してやれば、反応系は軽いまま先読みの恩恵を受けられる。これが2バンドアーキテクチャの核心です。

## 基礎: 前提となる概念
用語を順に噛み砕きます。

- 更新頻度(Hz、ヘルツ)。1秒間に何回処理を回すかです。10Hz なら 0.1 秒ごと、1Hz なら 1 秒ごと。反応系を高頻度、予測系を低頻度にするのがこの設計の肝です。

- BEV(Bird's-Eye View、俯瞰ビュー)。複数のカメラ画像を、真上から見下ろした地図のような格子(グリッド)に統合した表現です。形は例えば `[B, D, H, W]`(B はバッチ、D は特徴次元、H×W はグリッド解像度、例えば 450×300)。複数カメラの情報を1つの座標系に揃えられるので、運転の中間表現として広く使われます(詳細は [BEV Fusionを標準融合方式に統一](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0300_bev-fusion-as-standard))。

- 時系列メモリ(temporal memory)。過去数フレームの情報を圧縮して保持する部品です。「さっき横に車がいた」という文脈を覚えておくのに使います。

- 自己教師あり学習(self-supervised learning)。人間がラベルを付けなくても、データ自身から学習信号を作る方法です。例えば「未来のフレームを当てる」というタスクは、未来のフレームそのものが正解になるので、人手のラベルが要りません。

- JEPA(Joint-Embedding Predictive Architecture)。未来や欠損部分を、生のピクセルではなく特徴空間で予測する自己教師あり学習の枠組みです。世界予測系の中核技術になります。

- frozen / EMA。frozen は「凍結=学習で重みを更新しない」、EMA は「指数移動平均=ゆっくり追従させる」。JEPA で予測対象(target)を作るエンコーダをこれらで安定化させ、予測が自明な定数に崩壊するのを防ぎます。

これらを組み合わせると、反応系は「カメラ → BEV → 地図融合 → 時系列メモリ → プランナー → 軌道」という処理鎖になり、予測系は「視覚履歴 → frozen エンコーダ → 集約 → 未来特徴予測(JEPA)」という別の鎖になります。

## 仕組みを詳しく
データの流れで両系を説明します。記号は B=バッチ、V=カメラ台数(例 V=7)、N=過去フレーム数とします。

速い反応系(高頻度、例えば 10Hz)。
```
1. カメラ画像 [B, V, 3, H, W]
     → backbone(画像特徴抽出器)で各ビューを特徴化
2. 特徴融合 → BEV グリッド [B, D, 450, 300]
3. 地図エンコーダがナビ地図を同じ BEV グリッドに変換し、重ね合わせる
4. 時系列メモリが視覚履歴・自車運動履歴 [B, T, D] を文脈ベクトルに圧縮
5. プランナーが軌道 [B, T*C] を出力(T=予測時刻数、C=信号数)
```
この鎖は軽量で、毎フレーム回せます。プランナーは「軌道だけを返す」単純な契約です(その理由は [プランナーから自車隠れ状態の出力を切り離す設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0956_ego-hidden-removal-from-planner) を参照)。

遅い世界予測系(低頻度、例えば 1Hz)。
```
入力: 低頻度の多視点ストリーム [B, N, V, 3, H, W]
     (N は過去フレーム数。N=6 なら 6 秒ぶんなど)
1. frozen backbone で各フレームを特徴化
     凍結する理由は2つ:
       (a) 反応系の backbone 学習と勾配が衝突しないようにする
       (b) JEPA の予測対象(target)を固定し、自明解への崩壊を防ぐ
2. ビュー方向の attention pool(V 台ぶんの特徴を1ベクトルに集約)
3. 時間方向の attention pool(N フレームを1つに圧縮)
     → encoded_visual_history [B, D]
4. この履歴ベクトルを反応系の visual_history 入力に渡す
5. 未来予測器(JEPA decoder)が未来 N フレームの特徴を予測し、
   frozen backbone が作る本物の未来特徴と比べる損失 L_jepa を計算
```
総損失は `L = L_traj + λ * L_jepa` のように、軌道損失と JEPA 損失の重み付き和になります(λ は両者のバランスを取るハイパーパラメータ)。軌道損失の一本化については [損失計算をプランナーから切り離し外部の軌道損失に一本化する設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0566_planner-loss-decoupling) を参照。

効率の工夫が重要です。backbone が凍結されていて、各フレームの特徴がそのフレームだけに依存するなら、低頻度で回すとき「新しい 1 フレームだけ」を通し、残り N−1 フレームの特徴はキャッシュ(計算済みの結果を再利用)できます。これで予測系のコストを大幅に下げられます。仮に毎回 N フレーム全部を通すと N 倍の計算になりますが、キャッシュを使えば実質 1 フレーム分で済みます。

なぜこの分割が設計として優れているのか。反応系の周期(0.1 秒)に予測系の重い計算を載せないので、リアルタイム性が守られます。予測系が数フレームに1回しか動かなくても、その出力(これまでの流れの要約)はゆっくりとしか変わらない情報なので、反応系は古めの要約を使っても問題ありません。つまり「速く変わる情報は速く、ゆっくり変わる情報はゆっくり」という、情報の時間スケールに合わせた自然な分割になっています。

## 手法の系譜と主要論文
2バンド分離と、遅い系での未来予測は、複数の研究系譜の合流点にあります。

認知科学的な比喩として、Kahneman の「Thinking, Fast and Slow」(2011) の System 1(速い直感)/ System 2(遅い熟考)が引き合いに出されます。ただし、本稿の分割は比喩ではなく具体的で、「更新頻度の違い(10Hz の反応 vs 1Hz の推論)」で役割を切り分けている、と理解するのが正確です。

MILE (Hu et al., NeurIPS 2022)。BEV 上で世界モデル(未来予測)と方策(行動)を同時に学ぶ研究です。観測の予測と行動の学習を結びつけることで、予測能力と運転性能を両立しました。「予測系を持つこと」の有効性を運転で実証した代表例です。

GAIA-1 (Hu et al., 2023, arXiv:2309.17080)。運転の生成的世界モデルで、行動条件つきの未来を生成します。「もし自分がこう動いたら世界はどう変わるか」を学ぶ巨大モデルです。世界予測を重い独立系として扱う方向性を強く後押ししました。未来予測を計画軌道で条件づける(action-conditioning)という発想はこの系譜です。

JEPA / I-JEPA (LeCun 2022; Assran et al., CVPR 2023, arXiv:2301.08243)、V-JEPA(Bardes et al., 2024)。特徴空間で未来・欠損を予測する自己教師あり学習です。ピクセル予測の崩壊を避け、意味的に有用な表現を効率よく獲得します。遅い系の未来予測はこの JEPA 系で、frozen backbone により予測対象を固定します。V-JEPA は動画に拡張され、時間方向の未来予測を扱うので、運転の世界予測系に直接つながります。

BYOL (Grill et al., NeurIPS 2020, arXiv:2006.07733)。target encoder を止める(EMA で更新する)ことで、負例(negative sample)なしでも表現が自明解に崩壊しないことを示しました。「frozen または EMA の target なら崩壊しない」という、予測系を安定化させる設計の理論的根拠です。

系譜をまとめると、「行動と予測を結合する世界モデル(MILE)→ 大規模生成世界モデル(GAIA-1)→ 効率的な特徴空間予測(JEPA / V-JEPA)」という流れがあり、それを「速い反応系」と「遅い予測系」に物理的に分けて実装するのが現代的な構成です。

## 論文の実験結果(定量データ)
MILE (Hu et al., NeurIPS 2022)。CARLA シミュレータの Town05 ベンチマークで評価されました。評価指標の中心は Driving Score(運転スコア、ルート完走率に違反ペナルティを掛けた総合指標、100 点満点)と Route Completion(ルート完走率、%)です。MILE は当時の最先端を上回る運転スコアを報告し、特に Town05 Long(長く難しいルート)での改善が目立ちました。アブレーションでは、世界モデル(未来予測)部分を外すと運転スコアが低下することが示され、予測系を持つこと自体が運転を良くする証拠になっています。この「予測を抜くと性能が落ちる」という結果が、わざわざ別系統を用意する理由を裏付けます。

I-JEPA (Assran et al., CVPR 2023)。ImageNet-1K で、ViT-Huge を使い線形プローブ(学習済み特徴の上に線形分類器だけ載せる評価)で約 70%超 のトップ1精度を、ピクセル再構成型(MAE)より少ない事前学習計算で達成したと報告されています。重要なのは効率です。少ない計算で良い表現が得られるなら、未来予測系を低頻度で回す設計が現実的になります。

V-JEPA(Bardes et al., 2024)。動画での自己教師あり学習で、Kinetics-400(動画行動認識のベンチマーク)や Something-Something-v2(時間的な因果・順序の理解を要するベンチマーク)などの下流タスクで、エンコーダを凍結したまま(frozen evaluation、attentive probe を載せるだけ)評価し、ピクセル再構成型の動画 MAE を上回る精度を報告しました。特に Something-Something-v2 では時間構造の理解が問われるため、特徴空間予測の優位(report ではおよそ +5〜10 ポイント規模)が顕著です。続く V-JEPA 2(2025)はさらに大規模化し、ロボット操作への転移や行動条件つき予測まで踏み込んでいます。動画から時間的な構造を特徴空間で学べることを示しており、運転の未来予測に直接応用できる結果です。

BYOL (Grill et al., NeurIPS 2020)。ImageNet 線形プローブで約 74.3% のトップ1精度(ResNet-50)を達成し、SimCLR など負例を使う対照学習に匹敵しながら、負例(negative sample)を一切使わずに崩壊せず高性能を出せることを実証しました。崩壊(collapse)とは、すべての入力を同じ定数ベクトルに写すと予測誤差がゼロになってしまう自明解のことです。BYOL のアブレーションは、target を EMA で止める(momentum 係数 τ を 0.99 付近にする)という1点が崩壊回避の鍵で、τ=0(target を online と同一にする=止めない)にすると精度が劇的に崩れることを示しました。これは遅い予測系で frozen/EMA を使う判断の直接の根拠です。

数値が共通して語るのは、「未来予測は運転に効く(MILE)」「特徴空間予測は効率が良い(I-JEPA, V-JEPA)」「target を止めれば崩壊しない(BYOL)」の3点で、これらが2バンド設計の3本柱です。

## メリット・トレードオフ・限界
メリット。
- 速い反応(高頻度)と遅い推論(低頻度)を分け、リアルタイム性を保ったまま重い未来予測を載せられる。
- 役割が明確で、どちらを直すか分かりやすく、独立に改良・テストできる。
- frozen backbone + キャッシュで予測系のコストを抑えられる。
- 未来予測を、反応経路の速度を犠牲にせず追加できる。

トレードオフ・限界。
- 予測系が未完成の間は、反応系に渡る視覚履歴がプレースホルダ(ゼロ埋め)になり、未来予測の恩恵が出ない過渡期がある。
- 2系統に分けると、両者をつなぐインターフェース(履歴ベクトルの次元、更新タイミング、同期方法)を合わせる設計コストが増える。
- frozen backbone は安定だが、反応系で改善された特徴を取り込めない。共有エンコーダ + 勾配停止(shared + stop-grad)との間にトレードオフがある。

研究上の未解決課題。予測系の出力をどの空間(occupancy、特徴、低次元)で表現するのが運転性能に最適かは未確立です。また、反応系と予測系をどう同期させるか(予測の更新が反応に伝わる遅延の扱い)、行動条件づけをどこまで入れるか、も活発に研究されています。

## 発展トピック・研究の最前線
最前線は、世界モデルの大規模化(GAIA 系)と、その出力を計画に直接フィードバックする閉ループ化です。予測系が「自分の行動でどう未来が変わるか」を返せれば、反応系はそれを見て軌道を選べます。これは行動条件つき世界モデルと計画の融合であり、現在最も注目される方向の1つです。

もう一つは、frozen と EMA の選択や、target encoder の更新速度のチューニングです。BYOL や JEPA の知見を運転の動画ストリームに適用するとき、どの程度の慣性で target を追従させるかが安定性と表現品質を左右します。
