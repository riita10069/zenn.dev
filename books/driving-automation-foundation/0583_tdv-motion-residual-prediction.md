# TDV(時間差分・残差予測)

## ひとことで言うと

未来の世界を「ゼロから全部描き直す」のではなく、「いまの世界はだいたいそのまま残るから、変化する分(差分・デルタ)だけを予測する」やり方です。予測の負担を「変わる部分」だけに集中させることで、出力のばらつきが小さくなって学習が安定し、動いているものを自然と捉えやすくなります。「TDV」は Temporal Difference(時間方向の差分)を予測値(Value / Vector)として扱う、という発想を縮めた呼び方です。

## 直感的な理解

ある一場面を思い浮かべてください。交差点の手前で信号待ちをしていて、目の前を1台の車が横切っている。1.6秒後、何が変わっているでしょうか。空は同じ、建物も同じ、路面も同じ、停まっている車も同じです。変わるのは横切る車の位置と、ほんの少し近づいてくる手前の景色くらいです。つまり「次の瞬間の世界」のほとんどは「いまの世界」とそっくりで、本当に変化するのはごく一部です。

それなのに、未来を「白紙から」予測させると、ニューラルネットワーク(人間の脳を真似た、数値を変換する関数の積み重ね)は「変わらない大部分」までもう一度ゼロから再現しようとします。これは、ほとんど同じ2枚の絵を渡されて「2枚目を丸ごと描き直せ」と言われるようなもので、貴重な学習能力(モデルが調整できる重みの容量)を無駄遣いします。

そこで発想を変えます。1枚目をコピーしておいて「変わった部分だけ上書きせよ」と指示すれば、モデルは差分(上書きすべき場所と量)だけを考えればよくなります。この「差分だけを予測する」のが残差予測(residual prediction)であり、時間方向に対して行うのが TDV です。差分の大きい領域はそのまま「動いている領域」なので、モデルは放っておいても動的な物体に注目するようになります。

## 基礎: 前提となる概念

順に用語を噛み砕きます。

特徴マップ(feature map): 画像をニューラルネットワークの途中まで通したときの中間表現です。生のピクセル(色の数値)ではなく、「ここにエッジがある」「ここは車らしい」といった抽象的な情報がチャンネル(特徴の種類)ごとに並んだ三次元の数値の塊で、形状は `[チャンネル数, 高さ, 幅]`(バッチを含めて `[B, C, H, W]`)で表します。`B` はバッチサイズ(一度に処理するサンプル数)です。

残差(residual): 「全体 − ベースライン(基準値)」のことです。たとえば気温を予測するとき、「明日の気温」を当てるより「今日との差」を当てる方が簡単なのと同じです。未来特徴の場合、ベースラインを「現在特徴」に取ると次のように分解できます。

```
未来特徴 = 現在特徴 + (未来特徴 − 現在特徴)
                       └── この差分(motion delta)だけを予測する = 残差予測
```

自己教師あり学習(self-supervised learning): 人間が正解ラベルを付けなくても、データ自身の構造から学習信号を作る方法です。動画なら「過去のフレームから未来のフレームを当てる」というタスクを自動で作れます。未来は時間が進めば必ず観測できるので、ラベル付けのコストなしに大量の学習信号が得られます。TDV はこの「未来当て」を残差形式で行う自己教師あり手法です。

なぜ差分予測が学習を安定させるのか。機械学習では、出力の値が小さくゼロ近傍に集中しているほど、勾配(重みをどちら方向に更新すべきかの情報)が暴れにくく、学習が安定します。ほとんどの位置で差分が「ほぼ0」になる残差は、まさにこの「ゼロ近傍に集中した分布」を予測対象にするので、安定するわけです。

## 仕組みを詳しく

通常の予測と残差予測を、形状つきで並べて比較します。現在特徴を `f_t`(形状 `[B, C, H, W]`、たとえば `[B, 256, H, W]`)、`k` ステップ先の未来特徴を `f_{t+k}` とします。`Predictor` は未来を予測するニューラルネットワークで、`context` は予測に使う追加情報(たとえば「自車がこれから取る行動」を要約したベクトル)です。

通常予測(未来を直接出力):
```
predicted_{t+k} = Predictor(f_t, context)      # 未来特徴そのものを出力
```

残差予測(TDV、差分を出力して足し戻す):
```
delta_{t+k}     = Predictor(f_t, context)      # 変化分(差分)を出力
predicted_{t+k} = f_t + delta_{t+k}            # 現在特徴に足し戻す
```

学習の損失(loss、予測と正解のずれを測る量)は両方とも `MSE(predicted_{t+k}, target_{t+k})`(平均二乗誤差)で同じですが、ネットワークが内部で生成するものが「絶対値」か「差分」かが決定的に違います。

数値イメージを出します。ある画素位置・あるチャンネルの値が、現在 `f_t = 0.83`、実際の未来 `target = 0.85` だとします。

| | 通常予測 | 残差予測(TDV) |
|---|---|---|
| モデルが当てる値 | 0.85(絶対値) | 0.02(差分 delta) |
| 出力レンジ | 値の全域に散らばる | ほぼ 0 付近に集中 |
| 動く領域 | 0.85 のような大きい値 | delta が大きい(例 0.30) |
| 止まる領域 | 値はやはり大きい | delta ≈ 0 |

止まっている領域では `delta ≈ 0` を出せばよく、動いている領域だけ `delta` が大きくなります。つまりモデルの「予測努力」が動的領域に自然と配分されます。

設計上の重要ポイントが2つあります。

ベースラインに何を使うか。最も単純なのは現在特徴 `f_t` をそのまま使う「ゼロ次予測」で、「物体は止まっている」という前提に相当します。より高度には「物体は等速直線運動する」と仮定し、過去2フレームから速度を推定してベースライン自体を動かす方法もあります(`baseline = f_t + (f_t − f_{t-1})`)。等速ベースラインは、定速で流れる背景や定速走行する周辺車に対して残差をさらに小さくできます。

残差予測器のゼロ初期化。学習開始時に `delta ≈ 0` となるよう、予測器の最終層の重みをゼロ(またはごく小さい値)で初期化するのが定石です。こうすると学習の出発点が「未来 ≒ 現在」という安全で妥当な挙動になり、いきなり大きく外れて勾配が爆発する事故を防げます。これは深層残差ネットワーク ResNet が「恒等写像(入力をそのまま出す)」から学び始める思想とまったく同じです。

## 手法の系譜と主要論文

残差・差分を学習対象にする発想は、複数の分野が合流して生まれています。

残差学習の源流は ResNet です(He, Zhang, Ren, Sun, "Deep Residual Learning for Image Recognition", CVPR 2016, arXiv:1512.03385)。それまで層を深くするほど精度が頭打ち・劣化する「劣化問題」がありました。提案は、層に求める出力 `H(x)` を直接学ばせる代わりに、残差 `F(x) = H(x) − x` を学ばせ、出力を `x + F(x)` とする「スキップ接続」です。恒等写像を初期値にできるため、152層という当時としては桁外れの深さでも学習が崩れませんでした。TDV の「現在 + 差分」は、この空間方向の残差を時間方向に焼き直したものと言えます。

動画予測の分野では、ピクセルやフレーム間差分・運動(モーション)を明示的に予測する系統が古くからあります。Finn, Goodfellow, Levine の CDNA("Unsupervised Learning for Physical Interaction through Video Prediction", NeurIPS 2016, arXiv:1605.07157)は、未来フレームをゼロから生成せず「現在フレームのピクセルをどう動かす(変換する)か」を予測しました。動かす指示=運動を予測する、という点で TDV と発想が通じます。Lee らの SAVP("Stochastic Adversarial Video Prediction", 2018, arXiv:1804.01523)など確率的動画予測も「変化の不確実性」を扱い、変化に焦点を当てる流れを強めました。オプティカルフロー(隣接フレーム間の各ピクセルの移動ベクトル)を予測する研究群も、本質的には「動いた分=差分」を予測しています。

近年は、ピクセルではなく潜在・特徴空間で予測する自己教師あり学習が主流になりました。I-JEPA(Assran et al., Meta, CVPR 2023, arXiv:2301.08243)は「画像をピクセル再構成せず、学習された表現空間で見えない領域の表現を予測する」枠組みです。V-JEPA(Bardes et al., 2024, arXiv:2404.08471)はこれを動画に拡張しました。TDV の新規性は、こうした潜在予測アーキテクチャに「ゼロから予測」ではなく「現在 + 学習デルタ」という残差構造を持ち込み、変化の予測に集中させた点にあります。

## 論文の実験結果(定量データ)

残差・差分構造が効くことを示す代表的な定量結果を挙げます。

ResNet(arXiv:1512.03385)は ImageNet(約128万枚・1000クラスの画像分類ベンチマーク)で、残差なしの素朴な34層ネットが18層ネットより悪化する(劣化問題)のに対し、残差つき34層は18層を上回りました。最終的に ResNet-152 は ImageNet 検証セットで top-5 誤り率(正解が予測上位5位に入らない割合)約3.57%を達成し、当時の優勝記録を更新しました。ここで重要なのは「同じ深さでも残差を入れるだけで誤差が下がる」というアブレーション(ある要素を抜く/入れて効果を比較する実験)で、残差構造そのものの寄与を示しています。

動画予測では、フレームを直接生成するモデルより、運動・変換を予測するモデルの方が、予測フレームの鮮明さや SSIM(構造的類似度、1に近いほど元画像に似ている指標)・PSNR(ピーク信号対雑音比、dB、大きいほど良い)で有利になる傾向が、CDNA 以降の一連の研究で繰り返し報告されています。特に静的な背景が支配的なシーンでは、差分予測の方が背景のぼやけ(直接生成で起きがちな「平均化によるボケ」)を抑えられます。

潜在予測の系統では、I-JEPA が ImageNet-1k の線形評価(学習済み特徴の上に線形分類器だけ載せて精度を測る方法。特徴の良さを直接測る)で、ピクセル再構成型の自己教師あり学習(MAE など)に対して、同等以上の精度をはるかに少ない計算量で達成したことが報告されています。これは「予測対象を意味空間に置き、無駄な細部を捨てる」効果の定量的裏付けで、TDV が依拠する前提を支えます。

残差予測そのものの効果は、未来予測タスクで「残差あり/なし」を切り替えたアブレーションで評価されます。報告によると、残差を入れると(1)学習初期の損失が低い水準から始まり収束が速い、(2)予測誤差マップが動的物体の周辺に集中し静的領域では小さい、という2点が観察されます。逆に、シーンが急変するサンプル(急ハンドル・カメラの大きな揺れ)では残差の利得が縮む、という限界も同時に報告されています。

## メリット・トレードオフ・限界

メリット:
- 予測対象がゼロ近傍の差分になり、出力の分散が下がって学習が安定し、収束が速くなる。
- 差分が大きい=動く領域なので、モデルの予測努力が自然に動的物体へ配分され、運動表現が良くなる。
- ゼロ初期化と組み合わせると「未来 ≒ 現在」という安全な初期挙動から学習が始まり、初期の発散を防げる。

トレードオフ・限界:
- 急変シーン(カメラの大揺れ、急ハンドル、シーンの切り替わり)では差分が大きく、残差の利点が薄れる。「未来は現在に近い」という前提が崩れる領域では効きにくい。
- ベースラインの取り方(静止前提か等速前提か)に性能が依存する。等速ベースラインは過去フレームを要し、推定誤差も入り込む。
- 「現在特徴をそのまま足す」近道があるため、モデルが手抜きをして常に `delta ≈ 0` を出す縮退に陥るリスクがある。損失設計(動的領域を重み付けする)や正則化で防ぐ必要がある。

未解決の課題として、差分の不確実性(同じ現在から複数の未来があり得る、いわゆる multi-modality)を残差予測でどう扱うかがあります。MSE は平均を当てに行くため、複数あり得る未来の「平均」というぼやけた差分を出しがちです。確率的な残差予測や、後述する潜在空間での予測がこの緩和策として研究されています。

## 発展トピック・研究の最前線

潜在空間 × 残差の組み合わせが現在の最前線です。ピクセルではなく学習表現の空間で残差を予測すれば、無関係な細部を捨てつつ「意味の変化分」だけを学べます。世界モデル(自分の行動を条件に未来を予測するモデル)に残差予測を組み込み、行動を変えたときの「差分の差分」を見ることで反実仮想(もし別の行動をしたら世界はどう変わるか)を扱う研究も進んでいます。

もう一つの方向は、ベースラインの高度化です。単純な「現在をコピー」から、シーンフロー(三次元空間での各点の運動)や自己運動補償(カメラ=自車の動きをまず打ち消す)を取り入れ、残差をさらに純粋な「他者の運動」に絞る試みです。これにより、自車が動くことで生じる見かけの変化と、周辺物体が本当に動く変化を切り分けられます。
