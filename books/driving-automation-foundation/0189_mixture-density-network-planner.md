# Mixture Density Network 軌道予測

## ひとことで言うと
Mixture Density Network(混合密度ネットワーク、MDN)は、ニューラルネットワークの出力を「一つの数値」ではなく「複数のガウス分布(正規分布)を混ぜた確率分布のパラメータ」にする手法です。軌道予測では「次に車がどこへ行くか」を一本の線ではなく「直進70%・左折20%・停止10%」のように、複数の可能性とその確からしさ、さらに各可能性の不確かさ(広がり)まで含めて確率分布として出力できます。

## 直感的な理解
普通の回帰ニューラルネット(入力から数値を当てるネット)を二乗誤差(MSE)で学習すると、「正解たちの平均」を出すように育ちます。答えが一つに決まる問題ならそれでよいのですが、運転の未来予測は一つに決まりません。

具体例。前方に分岐があり「左へ行く」か「右へ行く」のどちらも妥当だとします。学習データには左の例と右の例が混ざっています。一本の軌道を MSE で学習すると、左と右の平均、つまり「まっすぐ分岐の壁に突っ込む軌道」を出してしまう。どちらの正解にも当てはまらない最悪の出力です。これがモード平均化問題(複数の山の平均を取ってしまう問題)。
```
学習データ:  左へ ↖    右へ ↗
MSE回帰の出力:      ↑(平均=壁に直進、誰も望まない)
MDNの出力:    ↖ 50% と ↗ 50%(両方の山を保持)
```
この問題を根本から避けるため「出力を一点ではなく確率分布にする」発想が生まれました。それが MDN です。複数の山(モード)を持つ分布を表現できるので、左と右を別々の山として残せます。

## 基礎: 前提となる概念
確率分布。ある値がどれくらい起こりやすいかを表す関数です。ガウス分布(正規分布)は中心(平均 μ)と広がり(分散 σ²)で決まる釣鐘型の分布で、最もよく使われます。

混合(mixture)。複数の分布を重み付きで足し合わせること。山が一つでは表せない多峰な分布を、山を複数足すことで表現します。

ガウス混合モデル(Gaussian Mixture Model, GMM)。K 個のガウス分布を混合重み π で足したもの。`p(y) = Σ_k π_k · N(y | μ_k, σ_k²)`。これが MDN の出力する分布です。

尤度(likelihood)。ある分布のもとで観測データ(正解)が出てくる確率(密度)のこと。「この分布は正解をどれくらいよく説明するか」を測ります。大きいほど良い分布。

対数尤度・負の対数尤度(NLL)。尤度は掛け算で小さくなりすぎるので対数をとって扱います。学習では「尤度を最大化=負の対数尤度を最小化」します。NLL が MDN の損失です。

softmax / softplus。softmax は出力を正で合計1の確率に変える関数(混合重み π に使う)。softplus や exp は出力を必ず正にする関数(分散 σ² に使う、分散は負になれないため)。

## 仕組みを詳しく
出力する分布: GMM。MDN はネットワーク出力を K 個のガウスのパラメータにします。各ガウスは三つで決まります。
- 混合重み π_k(その山がどれくらい起こりやすいか、全部足すと1)
- 平均 μ_k(その山の中心。軌道予測なら「この経路」)
- 分散 σ_k²(その山の広がり=不確実性の大きさ)

K=3 で軌道(128次元 = 64ステップ×2信号)を予測するなら、出力次元はおおむね:
```
混合重み:  [B, 3]            ← softmax で合計1に
平均:      [B, 3, 128]       ← 各山ごとの軌道
分散:      [B, 3]  または各次元ごと [B, 3, 128]   ← softplus で正に
```
混合重み π_k は softmax、分散 σ_k² は exp/softplus を通して正に保ちます。

学習: 負の対数尤度(NLL)。損失は二乗誤差ではなく、正解軌道 y を出力した混合分布がどれだけ高い確率で説明できるかです。
```
損失 L = − log( Σ_k π_k · N(y | μ_k, σ_k²) )
        (N はガウスの確率密度)
```
学習が進むと「正解 y の近くにどれかのガウスの山が高い重み π_k で乗る」ように π, μ, σ が調整されます。重要なのは平均化が起きないこと。左の正解には左の山が、右の正解には右の山が割り当たり、両方の山が共存します。MSE のように一点へ潰れません。NLL が log の中で和(Σ_k)を取っているため、「どれか一つの山が正解を説明できれば損失が下がる」構造になっており、これが分業を生む数理的な仕掛けです。

推論: サンプリングか最尤の山。予測時は二通り。
1. 混合重み π_k に従って山を一つ選び、その山からサンプリング(多様な未来を複数出せる)
2. 最も重みの大きい山の平均 μ_k をそのまま採用(一番ありそうな一本)
軌道予測では「上位 K 本の候補と確信度」を出力に使い、後段で最良の一本を選ぶ使い方が一般的です。

よくある落とし穴。MDN は学習が不安定になりがちです。分散 σ が小さくなりすぎると `log N` が発散して損失が爆発したり(一点に確率質量が集中する数値的破綻)、特定の山ばかりに重みが集中して他の山が死ぬモード崩壊が起きます。実務では σ に下限を設ける、重みのエントロピー正則化(重みが偏りすぎないよう罰する)を入れる、log-sum-exp で数値安定に計算する、といった工夫が要ります。この学習の難しさが、後の世代で [Winner-Takes-All / min-of-N 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0602_winner-takes-all-loss) や拡散・[Conditional Flow Matching (CFM)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0214_conditional-flow-matching) へ移る一因になりました。

## 手法の系譜と主要論文
原典は Bishop(1994, 技術報告、著書 Pattern Recognition and Machine Learning でも詳説)「Mixture Density Networks」。ニューラルネットの出力をガウス混合のパラメータにし NLL で学習する枠組みを提案しました。動機は、1対多(同じ入力に複数の妥当な出力がある)問題で MSE の平均化を避けること。これにより多峰な条件付き分布を表現でき、不確実性も同時に出せます。

歩行者・車両の軌道予測へは Social-LSTM / Social-GAN 系の研究を経て GMM 出力が広く使われました。代表が MultiPath(Chai, Sapp, Bansal, Anguelov, CoRL 2019, arXiv:1910.05449)。素の MDN の不安定さを避けるため、事前にアンカー軌道(典型経路の雛形)を用意し、各アンカーごとに確率と微小なオフセット(μ にあたる補正)を出します。出力空間をアンカーで離散化することで学習を安定させつつ多峰性を表現する、MDN の実用的な改良です。

MultiPath++(Varadarajan ら, ICRA 2022, arXiv:2111.14973)は固定アンカーの制約を緩め、学習可能なアンカーとエージェント間情報融合で精度を向上させました。

Trajectron++(Salzmann, Ivanovic, Chakravarty, Pavone, ECCV 2020, arXiv:2001.03093)はグラフ構造(エージェント間の相互作用をグラフで表す)と GMM 出力を組み合わせ、複数エージェントの多峰な未来をダイナミクス(運動)制約付きで予測しました。MDN 系の出力を実シーンの相互作用へ拡張した代表例です。系譜としては「素の MDN(柔軟だが不安定)→ アンカーで離散化して安定化(MultiPath)→ 学習可能アンカー・グラフ・相互作用へ拡張(MultiPath++/Trajectron++)→ 生成モデルへ移行(拡散・フローマッチング)」という流れで理解できます。

## 論文の実験結果(定量データ)
評価指標。NLL(負の対数尤度、分布が正解をどれだけ説明するか、小さいほど良い)、minADE_K / minFDE_K(K本候補のうち正解に最も近い一本の平均位置誤差・終点誤差、メートル)、MissRate(最良候補の終点が一定距離以上外れた割合)を使います。

Bishop の原典は玩具的な逆問題(同じ入力に複数の出力がある人工データ)で、MSE 回帰が平均化で破綻するのに対し MDN が複数の枝を正しく分布として捉えられることを示しました。多峰性を表現できることの原理的な実証です。

MultiPath は nuScenes / 社内大規模データセットで、回帰一本のベースラインに対し minADE/minFDE を明確に下げ、多峰な軌道を確率付きで安定に出せることを報告しました。素の MDN を直接学習するより、アンカーで離散化したほうが学習が安定し精度も出ることが示されています。MultiPath++ は Waymo Open Motion Dataset の予測ベンチマークで上位の成績を報告し、学習可能アンカーと情報融合で固定アンカー版から minADE/minFDE/MissRate を改善しました。Trajectron++ は ETH/UCY(歩行者)や nuScenes(車両)で、決定論的手法や単峰手法に対し NLL と minADE/minFDE を改善し、ダイナミクス制約を入れることで物理的に妥当な多峰予測ができることを示しました。

アブレーションの示唆。ガウスの数 K を1にする(単峰に戻す)と多峰性が失われ、分岐シーンで minFDE が悪化します。分散 σ の安定化(下限・正則化)を外すと NLL が発散し学習が崩壊する報告があり、MDN の数値的脆さが裏付けられています。MultiPath ではアンカーを外して素の MDN に戻すと学習が不安定化し、アンカーによる離散化が安定性に寄与していることが示されました。これらは「MDN は表現力は高いが、そのまま使うと不安定で、離散化や正則化という補助輪が要る」という MDN の本質を定量的に示しています。

## メリット・トレードオフ・限界
メリット。多峰な未来を複数のガウスの山として陽に表現でき(左折・右折・直進を分けて出せる)、各予測に不確実性(分散 σ)と確からしさ(重み π)が付くので、後段で確信度に応じた判断ができます。出力が確率分布として解釈でき、NLL で素直に学習できる。較正された確率を直接得られる点は、安全判断において WTA より有利です。

トレードオフと限界。学習が不安定(分散の発散、特定の山への重み集中=モード崩壊)で、σ の下限・正則化・log-sum-exp などの安定化が事実上必須。山の数 K を事前に決める必要があり、想定より多くのモードがあると表しきれない。高次元・長尺の軌道では、ガウス一個で経路全体を表すのが粗くなりやすい(MultiPath のアンカー化はこの緩和策)。さらに、各時刻を独立ガウスで扱うと時間方向の相関(滑らかな軌道であること)を表しにくく、共分散を入れると次元が爆発する、という構造的ジレンマがあります。研究上の未解決課題は、可変数のモードを扱う方法、高次元軌道で表現力と安定性を両立する分布族、そして時間相関を効率的に表す出力構造の設計です。

## 発展トピック・研究の最前線
MDN の「分布のパラメータを陽に出す」路線は、近年「分布から直接サンプリングする生成モデル」へと主役が移りつつあります。拡散モデルやフローマッチングは、ガウス混合のように分布の形を事前に固定せず、ノイズから軌道を生成する過程を学ぶことで、より柔軟な多峰分布を、しかも MDN の数値的脆さなしに表現できます。一方で MDN は「確率密度を陽に評価できる(尤度が計算できる)」という生成モデルにない利点を持ち、異常検知(尤度が低い=想定外の状況)や不確実性の定量化では今も価値があります。最前線では、MDN 的な陽な分布出力と生成モデルのサンプリングを組み合わせるハイブリッド、ガウス混合の代わりに正規化フロー(可逆変換で複雑な分布を表す手法)を出力に使うアプローチ、そして較正された不確実性を計画の安全マージンへ直結させる研究が進んでいます。
