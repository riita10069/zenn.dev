# FutureState による将来視覚特徴予測

## ひとことで言うと
いま見えている場面の特徴から「1.6秒後、3.2秒後、4.8秒後、6.4秒後にはどんな特徴になっていそうか」を複数枚まとめて予測する部品です。実際に未来を当てに行く練習をモデルにさせることで、「世界はこの後どう動くか」という感覚(世界モデル)を身につけさせます。人間が正解ラベルを付けなくても、未来のフレームそのものが正解になるので、自己教師あり学習として成立します。予測対象を生画像ではなく抽象的な特徴にする点が肝で、これにより運転に効く構造だけを効率よく学べます。

## 直感的な理解
良い運転は先読みでできています。前の車が減速し始めたら自分も減速の準備をする。横から来る車が交差点で止まらなそうなら身構える。歩行者が縁石に近づいたら飛び出しを警戒する。どれも「この後こうなりそうだ」という予測に基づく判断です。

ところが、運転の正解ラベル(人間が実際にどうハンドルを切りアクセルを踏んだか)だけで学習すると、モデルは「人の操作を真似る」ことしか学びません。なぜその操作をしたのか、つまり世界がどう動いているのかを理解しないまま、表面だけコピーします。これは模倣学習の弱さで、訓練データにない状況に出会うと脆く崩れます。

ここで「未来を予測させる」補助課題を足します。補助タスク(auxiliary task)とは、本来の目的(運転操作の模倣)とは別に、学習を助けるために追加で解かせる課題のことです。未来予測の正解は、未来のフレームを実際にネットワークに通せば自動で手に入ります。人間のラベル付けが一切要りません。これが自己教師あり学習(self-supervised learning、データ自身から正解を作る学習)です。追加の人手なしで、モデルに「世界の動き方」という豊かな学習信号を与えられます。直感的には、運転を覚えるだけでなく「次に何が起きるか」を当てるゲームを並行してやらせることで、表現が世界の物理に根ざしたものになる、という狙いです。

## 基礎: 前提となる概念

いくつかの言葉をほどきます。

特徴空間(feature space / 表現空間)とは、生のピクセルではなくネットワークが抽出した中間表現が住む空間です。`[B, C, H, W]` のような特徴マップが1点を表します。同じ場面でも、ピクセル空間では照明やノイズで大きく動きますが、良い特徴空間ではそうした無関係な変動が抑えられ「意味」が安定して表されます。

世界モデル(world model)とは、「いまの状態と取る行動が与えられたとき、次の状態がどうなるか」を予測する内部モデルです。物理シミュレータの学習版だと思えばよい。世界モデルがあれば、実際に動く前に頭の中で結果を試せます。

自己教師あり学習は、ラベルなしデータから「データの一部を隠して残りから当てさせる」などの方法で学習信号を作る枠組みです。言語モデルの「次の単語を当てる」や、画像の「マスクした領域を復元する」がその例です。

行動条件づけ(action-conditioning)とは、未来予測を「どんな行動を取るか」で条件づけることです。同じ現在でも、ブレーキを踏めば未来Aに、アクセルなら未来Bになる。行動で条件づけた予測ができると、反実仮想(counterfactual、もし別の行動を取ったらどうなるか)を頭の中で比較でき、計画に直接使えます。

表現の崩壊(representation collapse)とは、特徴空間で予測を学ぶときに起きる病理です。予測器も対象も同じネットワークが作る表現なので、何も考えず「すべての特徴を同じ定数ベクトルにする」と予測誤差がゼロになってしまう。これは損失だけ下がるが何も学んでいない退化解で、これを防ぐ工夫が自己教師あり表現学習の中心課題です。

stop-gradient(勾配を止める操作)とは、ある経路の出力を「定数とみなして」逆伝播で勾配を流さない操作です。後述するように、予測の対象側にこれをかけることで、対象が予測器に迎合して両方が同時に崩壊するのを防ぎます。

## 仕組みを詳しく

典型的な将来特徴予測ヘッドは、現在の融合特徴 `[B, C, H, W]` を入力に取り、複数の時刻の未来特徴を一気に出します。代表的な実装は次の流れです(数値は一例)。

第1段に、チャンネルを拡張する畳み込みを置きます。`Conv2d(C → 2C, kernel=3, stride=1, padding=1)`。3×3 畳み込みで空間サイズを保ったまま(padding=1)チャンネルを倍増させます。続いて GELU などの非線形活性化を挟みます。第2段でさらに `Conv2d(2C → 4C)` とチャンネルを増やします。最後に `torch.chunk(out, chunks=4, dim=1)` でチャンネル方向を4等分し、それぞれ `[B, C, H, W]` の4枚の未来特徴に切り分けます。なぜチャンネルをいったん4倍に膨らませてから割るのかというと、4つの異なる未来時刻にそれぞれ専用の「容量」を割り当てたいからです。1つの共有特徴を流用するより、時刻ごとに別チャンネル群を学ばせるほうが、近い未来と遠い未来で異なる挙動(遠いほど不確実)を表しやすくなります。

```
現在特徴        [B, C, H, W]
  └ Conv C→2C, GELU
                [B, 2C, H, W]
  └ Conv 2C→4C
                [B, 4C, H, W]
  └ chunk(4, dim=1)
未来特徴4枚     [B, C, H, W] × 4
   = t+1.6s, t+3.2s, t+4.8s, t+6.4s の予測
```

4枚はそれぞれ 1.6秒後・3.2秒後・4.8秒後・6.4秒後の予測です(1.6秒間隔で6.4秒先まで)。間隔と地平線(horizon、どこまで先を見るか)は設計値で、運転計画に必要な時間スケール(数秒先)に合わせて選ばれます。地平線を長く取るほど計画に有用ですが、遠い未来ほど本質的に予測不能で誤差が増えるため、どこで打ち切るかは性能とのトレードオフです。

なぜ「特徴」を予測して「画像そのもの」を予測しないのか。これが核心です。ピクセルを当てに行くと、葉っぱの揺れ、影の動き、センサノイズといった予測不能で運転に無関係な細部まで再現しようとして、計算と容量を浪費します。抽象化された特徴空間で予測すれば、「物体がどこへ動くか」という運転に効く構造だけを学べます。これが JEPA(Joint-Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)の中心思想です。たとえば前方の木の葉のざわめきはピクセルでは大量の変化ですが、運転判断にはまったく無関係であり、特徴空間ではほぼ無視されます。

より進んだ設計では、行動条件づけを加えます。計画モジュールが持つ「これからどう走るつもりか」の要約(たとえば GRU の隠れ状態、長さ256程度のベクトル)を、未来予測のチャンネルごとのバイアスとして足し込む。こうすると「軌道Aで走ったら未来はこう、軌道Bならこう」という反実仮想予測ができ、単なる受動的な未来予測から、計画に使える能動的な世界モデルへ進化します。

損失(loss、学習の良し悪しを測る関数)は、予測した未来特徴 `ẑ_{t+k}` を、実際に未来フレームを通して得た特徴 `z_{t+k}` に近づけるよう設計します。たとえば `L = Σ_k || ẑ_{t+k} − sg(z_{t+k}) ||²` のような特徴再構成損失です。ここで `sg` は stop-gradient で、対象側を固定して崩壊を防ぐための工夫です。これがないと、予測器と対象が手を取り合って両方を定数にする退化解に落ちます。実用上は、対象側のネットワークを予測器の指数移動平均(EMA、exponential moving average、過去の重みをゆっくり混ぜた緩慢に動くコピー)にして、対象がゆっくりしか変化しないようにする手法(BYOL や I-JEPA で確立)も併用されます。

## 手法の系譜と主要論文

将来特徴予測は、世界モデルと自己教師あり表現学習という2つの大きな流れの合流点にあります。

Ha & Schmidhuber, "World Models"(NeurIPS 2018)は、環境の未来を予測する内部モデル(VAE で観測を圧縮し、RNN で潜在状態の遷移を予測)を学習し、その上で行動を学ぶという世界モデルの古典です。「未来を予測できる表現は、良い意思決定の土台になる」という考えを確立しました。Hafner et al. の Dreamer 系(DreamerV2 2020, DreamerV3 2023)はこれを発展させ、学習した世界モデルの中で「想像」して方策を学ぶことで高いサンプル効率を達成しました。DreamerV3 は同一のハイパーパラメータで150以上の多様なタスクを解いたと報告され、世界モデルの汎用性を示しました。

LeCun, "A Path Towards Autonomous Machine Intelligence"(2022)は、知能の中核に「ピクセルではなく表現空間で未来を予測するモデル」を据えるべきだと論じ、JEPA という枠組みを提唱しました。理由は、ピクセル再構成が予測不能な細部に計算を浪費するのに対し、表現空間ならば意味のある構造だけを予測できるからです。Assran et al., "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture"(I-JEPA, CVPR 2023)はこれを画像で実証しました。将来特徴予測ヘッドは、まさに運転の特徴空間での未来予測であり、JEPA の運転・時間方向版と言えます。

運転に世界モデルを持ち込んだ系譜では、Hu et al., "FIERY: Future Instance Prediction in Bird's-Eye View from Surround Monocular Cameras"(ICCV 2021)が、BEV 表現上で未来の他車インスタンスを確率的に予測し、未来予測を運転認識の主タスクに据える有効性を示しました。Hu et al., "Model-Based Imitation Learning for Urban Driving"(MILE, NeurIPS 2022)は、運転を世界モデル(未来の潜在状態を予測する内部モデル)と組み合わせて学習し、未来特徴の予測が運転方策の質を上げることを示しました。行動条件づけで未来を予測する設計は、この行動条件つき世界モデルの直接の系譜です。さらに Hu et al., "GAIA-1: A Generative World Model for Autonomous Driving"(Wayve, arXiv:2309.17080, 2023)は、運転シーンを離散トークンに変換して生成的に未来を予測する大規模世界モデルを示し、生成系の最前線を切り開きました。

## 論文の実験結果(定量データ)

世界モデルや未来予測の補助タスクが効くことは、複数の研究で定量化されています。

I-JEPA(Assran et al., 2023)は、ImageNet という画像分類データセットで自己教師あり事前学習の質を評価しました。線形評価(linear probing、凍結した特徴の上に線形分類器だけ載せて精度を測る、表現の良さの定番指標)で、ピクセル再構成型の手法(MAE など)と同等以上の精度を、はるかに少ない計算量(報告によると同等精度に到達する事前学習のGPU時間が数分の一)で達成したと報告されています。重要な含意は「表現空間で予測するほうが、ピクセル復元より効率的に良い表現が学べる」という点で、これが特徴空間予測を選ぶ根拠です。

MILE(Hu et al., 2022)は、CARLA シミュレータの closed-loop 評価(実際に車を走らせて成績を測る、静的なデータセット評価より厳しい)で、Driving Score(ルート完走率と交通違反ペナルティを掛け合わせた総合スコア、0〜100、高いほど良い)を測りました。世界モデルを組み込んだ MILE は、当時の模倣学習ベースラインを大きく上回り、特に未知のルート・天候への汎化で差が出たと報告されています。アブレーションでは、未来潜在状態の予測を外すと性能が落ちることが示されており、「未来を予測する内部モデルが運転方策を底上げする」ことの直接の証拠です。Driving Score の意味は、ルートを完走しても信号無視や衝突があるとペナルティ係数で減点される、という安全性込みの総合点です。

FIERY(Hu et al., 2021)は nuScenes で未来インスタンスセグメンテーションを評価し、Future IoU(予測した未来の占有領域と正解の重なり、高いほど良い)と VPQ(Video Panoptic Quality、時間方向の一貫性も含めた品質指標、高いほど良い)で、未来予測を明示的にモデル化することの優位を示しました。

これらの読み方は次の通りです。第1に、未来予測という補助タスクはラベルなしで成立するのに、本来の運転タスクの汎化を底上げする。第2に、ピクセルより特徴空間で予測するほうが計算効率と表現品質の両面で有利。第3に、closed-loop の汎化(未知環境)でこそ差が出るので、静的データセットの数値だけ見ていると過小評価しやすい、という点です。

## メリット・トレードオフ・限界

メリットの第1は、人間のラベルなしで豊かな学習信号を得られること。未来フレームが自動的に正解になるので、世界の動き方を学ばせ、模倣の表面的コピーを補正できます。

メリットの第2は、特徴空間で予測するためピクセル再構成より軽く、運転に効く構造を学びやすいこと。

メリットの第3は、行動条件づけを入れれば「この軌道なら未来はこうなる」という反実仮想予測ができ、計画に直接使えること。

トレードオフの第1はメモリです。チャンネルを大きく拡張する設計(C→2C→4C)はパラメータと中間活性化が増え、特に高解像度 BEV(数百×数百のグリッド)では1サンプルで数百MB級になり、混合精度(bf16/fp16)やダウンサンプリングが事実上必須になります。たとえば 256ch × 450 × 300 を4倍に膨らませると、中間テンソル1枚だけで数百MBに達します。

トレードオフの第2は崩壊への脆さです。特徴空間での予測は退化解(全部同じベクトル)に落ちやすく、stop-gradient、対象側の指数移動平均(EMA target)、分散・共分散正則化(VICReg など)といった崩壊防止策を正しく設計しないと、損失だけ下がって何も学ばない状態になります。崩壊は損失曲線が急にゼロ付近に張り付くことで気づけますが、対策の組み合わせは経験依存です。

トレードオフの第3は補助タスクの重み調整です。補助損失の重みが大きすぎると本来の運転タスクの学習を邪魔し、小さすぎると効果が出ません。この重みは経験的なチューニングが要ります。

未解決の課題としては、未来の不確実性をどう扱うかがあります。未来は本質的に一意に決まらない(歩行者が右に行くか左に行くか分からない)ため、決定論的な1点予測は複数の可能性を平均化したぼやけた未来を出しがちです(mode averaging と呼ばれる現象)。確率的な未来予測(複数の未来分布)や、生成モデルで多峰性を表す研究は進行中ですが、計画にどう接続するかは未解決です。

## 発展トピック・研究の最前線

第1の最前線は、生成的世界モデルです。GAIA-1(Wayve, 2023)や近年の動画拡散モデルを使った運転世界モデル(GenAD など)は、行動条件づきで高品質な未来動画を生成し、シミュレータやデータ拡張に使う方向へ進んでいます。生成された未来は人間が目で見て妥当性を確かめられる利点があります。

第2は、トークン化された世界モデルです。観測を離散トークンに変換し、Transformer で次トークンを予測する(言語モデルと同じ枠組み)アプローチで、スケーリング則(モデルとデータを増やすほど性能が予測可能に上がる経験則)の恩恵を受けやすいとされます。

第3は、計画と世界モデルの一体化です。学習した世界モデルの中で軌道候補を rollout(展開)し、最も安全・効率的な軌道を選ぶ model-based planning が、エンドツーエンド運転の有力な方向として研究されています。Dreamer 系で確立した「想像の中で方策を学ぶ」考えを運転に持ち込む流れです。
