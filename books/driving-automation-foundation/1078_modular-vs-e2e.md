# モジュラー vs E2E アーキテクチャ

## ひとことで言うと

自動運転ソフトを「検出する係・予測する係・計画する係」のように仕事ごとの部品に分けて作るのがモジュラー(modular)方式、画像などの入力から運転の出力(ハンドル・アクセルや走行経路)までを1個のニューラルネットで一気に学習するのがE2E(エンドツーエンド、end-to-end)方式です。どちらが正しいという話ではなく、それぞれに得意・不得意があり、設計上のトレードオフを理解することがこの分野の出発点になります。

## 直感的な理解

自動運転車は「カメラや LiDAR(レーザー距離センサ)が捉えた周囲の状況」から「次にどう動くか」を決めなければなりません。この巨大な問題を、どう小分けにしてプログラムにするかが設計の根本的な分かれ道です。

身近なたとえで言うと、料理のレシピを考えてください。モジュラー方式は「材料を切る人・炒める人・盛り付ける人」と工程を分け、それぞれの工程の品質を別々にチェックできます。どこで失敗したかが一目で分かります。一方 E2E 方式は「材料を入れたら完成品が出てくる調理ロボット」で、中の工程は見えませんが、工程間の受け渡しロスがなく、最終的な味だけを目標にまるごと最適化されます。前者は検査しやすいがつなぎ目で情報が痩せ、後者は効率的だが中身がブラックボックスになる、という対比です。

## 基礎: 前提となる概念

歴史的にまず広まったのがモジュラー方式です。人間のエンジニアが問題を段階に分けます。典型的な流れは次のとおりです。

```
センサ入力 → [知覚 Perception] → [予測 Prediction] → [計画 Planning] → [制御 Control] → 車を動かす
              物体を検出・追跡        他車の将来を予測      自車の経路を決める     ハンドル・アクセル
```

各用語を噛み砕きます。知覚(perception)は「周りに何があるか」を検出・追跡すること。予測(prediction)は「他の車や歩行者がこれからどう動くか」を見積もること。計画(planning)は「自車がどんな経路・速度で進むか」を決めること。制御(control)はその計画を実現するハンドル角・アクセル量を出すこと。各段階(モジュール)は独立したチームが作り、独立したデータでテストでき、出力も人間が読める形(「前方10mに歩行者、速度1m/s」など)で受け渡します。多くの実車用自動運転スタックがこの方式の代表です。

E2E 方式の語彙も押さえます。E2E とは「入力から出力までの全経路を、途中に人工的な区切りを置かず、1つの微分可能なネットワークで学習する」こと。微分可能(differentiable)とは、出力の誤差を入力側まで勾配(gradient、誤差を減らす方向)として伝えられるという意味で、これにより全パラメータを最終目的に向けて一括で更新できます。模倣学習(imitation learning、人間の運転を真似る教師あり学習)や強化学習で学習します。

## 仕組みを詳しく

両方式の中身を、具体的な tensor の形(shape)を交えて対比します。tensor とは多次元配列のことで、ここでは [チャンネル, 高さ, 幅] のように次元を並べて書きます。

モジュラー方式の知覚の例(数値で): 複数台(たとえば8台)のカメラ画像(各 3×900×1600 のテンソル、3 は RGB の色チャンネル、残りは縦横ピクセル)を検出器に通し、出力は「3D ボックスのリスト」、たとえば

```
[{class:"car", x:12.3, y:-1.2, z:0.0, w:1.8, l:4.5, h:1.5, yaw:0.1, score:0.92}, ...]
```

という人間が読める構造化データです。これを予測モジュールに渡し、各ボックスの将来軌跡(例: 3秒先まで 0.5 秒刻みの (x, y) 列)を出し、計画モジュールがそれを避ける自車経路を最適化します。各段で「何を考えているか」が明示的に見えるのが特徴です。

完全 E2E 方式の例(数値で): 同じ複数カメラ画像を1つのネットに入れ、出力は運転指令そのもの、たとえば「(加速度 a, 曲率 κ) を 10Hz で数十ステップぶん」という (T, 2) のテンソルです。途中に「歩行者ボックス」のような明示的な中間表現は必須ではなく、ネット内部の特徴ベクトル(例: [B, C, H, W] の BEV 特徴、BEV は Bird's-Eye-View=上空からの俯瞰視点)として暗黙に持ちます。学習は人間ドライバの運転ログ(模倣学習)や報酬(強化学習)で、出力が人間に近づくよう全パラメータを一括更新します。

中間に位置するのが、UniAD や ST-P3 に代表される「統合だが中間表現を残す」設計です。1つのネットの中に検出・追跡・予測の小モジュールを積みますが、それらを「最終目的(計画)のための補助タスク」と位置づけ、全体を一緒に学習します(詳しくは [Planning-oriented設計思想](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1081_planning-oriented-paradigm))。E2E の「つなぎ目なし」とモジュラーの「中身が見える」の良いとこ取りを狙います。

整理すると、設計の軸は2つあります。

```
              区切りが多い ←─────────────→ 区切りが無い
中間表現が    モジュラー型              （該当少）
人間可読      (検出→予測→計画を別々に)

中間表現が    UniAD/ST-P3 型           完全E2E
ネット内部    (1ネット+補助タスク,      (画像→操舵を直接,
              中身も観察可能)          中身は不可読)
```

モジュラーは「多区切り・可読」、完全 E2E は「無区切り・不可読」、UniAD 系は「1ネットだが中間表現も観察可能」という配置になります。

## 手法の系譜と主要論文

- ALVINN(Pomerleau, 1989, NeurIPS, "ALVINN: An Autonomous Land Vehicle in a Neural Network"): カーネギーメロン大の初期作。前方カメラとレーザー測距の入力から操舵方向を出す小さなニューラルネットを学習し、E2E 自動運転の源流とされる。当時の計算資源では本格運用に届かなかったが「画像から操舵を直接学ぶ」という発想を最初に実証した。

- NVIDIA PilotNet(Bojarski et al., 2016, arXiv:1604.07316, "End to End Learning for Self-Driving Cars"): 前方カメラ画像からハンドル角を直接回帰する完全 E2E を提案し、深層学習時代に E2E を再注目させた。動機は「人間が手で設計した特徴(車線・物体)を介さず、データから直接運転に効く特徴をネットに学ばせる」こと。効果は最小限の人間設計で車線維持ができたこと。トレードオフは、なぜそう操舵したかが説明できず(ブラックボックス)、未知状況での安全保証が難しいこと。

- ST-P3(Hu et al., 2022, ECCV, arXiv:2207.07601, "ST-P3: End-to-end Vision-based Autonomous Driving via Spatial-Temporal Feature Learning"): 知覚・予測・計画を時空間 BEV 特徴で一気通貫に学ぶ E2E を提案。動機は「各段の特徴を共有して段間の整合を取る」こと。占有予測などの中間タスクを解釈用に残す折衷型。

- UniAD(Hu et al., 2023, CVPR best paper, arXiv:2212.10156, "Planning-oriented Autonomous Driving"): 完全 E2E の不透明さとモジュラーの情報落ちの両方を避けるため、1ネットに検出・追跡・地図・予測・占有予測を積み、全部を計画に奉仕させる統合設計を提案。効果は nuScenes で各指標と衝突率を同時改善。トレードオフはモデルが重く、全タスクの教師ラベルが必要なこと。詳細は [Planning-oriented設計思想](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1081_planning-oriented-paradigm)。

- Codevilla et al., "Conditional Imitation Learning"(CIL, 2018, ICRA, arXiv:1710.02410)と "Exploring the Limitations of Behavior Cloning"(2019, ICCV): E2E 模倣を「行き先(右折/左折/直進)の指令で条件づける」ことで交差点での曖昧さを解いた重要作と、その後に「データを増やしても汎化が頭打ちになる・初期化の乱数だけで性能が大きくぶれる」という行動クローニングの限界を実験的に暴いた反省的研究。完全 E2E の限界論の出発点。

- Transfuser(Prakash et al., CVPR 2021, arXiv:2104.09224)と Interfuser(Shao et al., CoRL 2022): カメラと LiDAR を Transformer で融合する E2E を CARLA クローズドループに最適化した系譜。中間に「物体・信号の補助予測ヘッド」を持たせて安全性を稼ぐ折衷で、完全 E2E と統合型の橋渡しにあたる。

- Chen et al., 2024(IEEE TPAMI, arXiv:2306.16927, "End-to-End Autonomous Driving: Challenges and Frontiers", サーベイ): モジュラーと E2E の長短を体系的に整理。E2E は因果混同(causal confusion、見かけ上の相関を学んでしまう。後述)や分布シフト(学習時と違う状況で弱い)に弱いと指摘。モジュラーは検証しやすいが累積誤差と目標不整合に弱いとまとめている。

## 論文の実験結果(定量データ)

E2E と中間表現つき統合設計の効果を定量的に裏づけたのが UniAD です。評価には nuScenes(都市運転の公開大規模データセット、複数カメラ・LiDAR・3Dアノテーションつき)を使い、最終的なプランニング性能を主に2つの指標で測りました。

- L2 誤差(L2 distance): 予測した自車軌跡と実際の人間の軌跡の、各時刻の位置のずれ(メートル)。小さいほど人間に近い。
- 衝突率(collision rate): 予測軌跡で走った場合に他物体とぶつかる割合(%)。安全性の代理指標で、これが最も重視される。

UniAD の報告では、3秒先までの平均衝突率を、当時の有力な E2E 手法(ST-P3 など)と比べておよそ半分以下に下げました。具体的には、平均衝突率で ST-P3 が 1% 前後だったのに対し UniAD は 0.3% 前後という水準が報告されています(評価条件により数値は変動)。平均 L2 誤差も同時に改善しました。重要なのは「検出・追跡・予測の中間タスクを全部入れて一緒に鍛えると、最終目的(プランニングの安全性)が改善した」という点で、これが統合設計の存在意義を支える定量的根拠です。

ただし注意が必要です。サーベイ(Chen et al., 2024)や後続研究は、nuScenes の L2 誤差という指標がオープンループ評価(録画と比べるだけで、自分の判断が次の状況を変えない評価)に依存しており、過去の自車運動だけを見る単純な MLP でも見かけ上良いスコアが出てしまうことを指摘しました。つまり「L2 が小さい=本当に運転がうまい」とは限らず、E2E モデルの真価はクローズドループ評価(自分の判断が次状況を変える、実走に近い評価)でこそ測るべきだ、という議論につながりました(詳しくは [オープンループ vs クローズドループ評価](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0725_open-loop-vs-closed-loop-eval))。この指標の限界は、E2E 研究を読むうえで必ず意識すべき落とし穴です。

因果混同(causal confusion)の具体例も研究で示されています。E2E モデルに「直前の自車速度」を入力に含めると、モデルは「前のフレームと同じ速度を続ければだいたい正解」という近道を学び、本来見るべき信号機や先行車をほとんど見なくなることがあります。オープンループでは数値が良く見えても、実際にはほぼ何も見ていない。これは E2E が「見かけの相関」を学んでしまう弱さの典型例です。de Haan, Jayaraman & Levine(NeurIPS 2019, "Causal Confusion in Imitation Learning", arXiv:1905.11979)はこの現象を体系化し、より多くの情報(過去フレーム)を入力に加えるほど性能が逆に悪化する「inertia(慣性)問題」を示しました。直感に反して入力を増やすと悪くなる、というこの結果は E2E 模倣の本質的な落とし穴です。

オープンループ指標の脆さは数値でも裏づけられています。Zhai et al.(AD-MLP, 2023, arXiv:2305.10430)は、カメラ画像を一切使わず自車の過去の運動状態(速度・加速度・舵角の履歴)だけを 3 層程度の MLP に入れた極端なベースラインが、nuScenes の平均 L2 誤差や衝突率で UniAD・VAD と同等かそれ以上のスコアを出すことを報告しました。これは「nuScenes の運転は等速・直進が支配的で、過去の運動を延長するだけで録画に当てはまってしまう」というデータセットの偏りを暴いたもので、E2E モデルがオープンループで良く見えても本当に周囲を見ているとは限らないことの最も鋭い証拠になりました。以後、評価は NAVSIM(Dauner et al., NeurIPS 2024)のような非反応的だが反実仮想を含む擬似クローズドループや、CARLA の完全クローズドループへ移りつつあります。

## メリット・トレードオフ・限界

モジュラー方式
- メリット: 各部品を独立に開発・テスト・検証できる。出力が人間可読で、事故時に「どの段で間違えたか」を追える(解釈性・安全保証がしやすい)。実車スタックで実績が豊富。各モジュールに既存の知見やルールを組み込みやすい。
- トレードオフ・限界: 段間で情報が痩せ、エラーが伝播・累積する(検出器が歩行者を1つ取り落とすと、その先は存在を知りようがない)。各モジュールの最適化目標(検出器なら mAP=平均適合率)が最終目的(安全運転)とずれる。手設計の区切りが現実の複雑さを取りこぼす。

E2E 方式
- メリット: つなぎ目の情報落ちがなく、全体が最終目的のために一括最適化される。手設計の特徴が不要で、データから運転に効く表現を学べる。データとモデルを増やすほど伸びるスケーリングの恩恵を受けやすい。
- トレードオフ・限界: ブラックボックスで解釈・デバッグ・安全保証が難しい。大量の運転データが必要。因果混同や分布シフトに弱く、まれな危険シーン(ロングテール)の学習が難しい。失敗時に原因を特定しにくい。

統合設計(UniAD 系)は両者の中間で、中間表現を残して解釈性を保ちつつ最終目的に向けて一括最適化しますが、全中間タスクの教師ラベルが必要でモデルが重い、というコストを払います。

研究上の未解決課題として、(1) E2E の解釈性・検証可能性の確保(安全認証のために何を保証できるか)、(2) ロングテールの希少事象への頑健性、(3) オープンループ指標の信頼性とクローズドループ評価の標準化、(4) 因果混同の体系的な抑制、が残っています。

## 発展トピック・研究の最前線

近年は大規模言語モデル・視覚言語モデル(VLM)を運転に持ち込む流れが強まっています。運転判断を言語で説明させる、あるいは VLM の常識推論を E2E パイプラインに組み込むことで、ブラックボックス性の緩和とロングテール対応の両立を狙う研究が活発です。また、World Model(環境の遷移を学習したシミュレータ)を E2E 学習と組み合わせ、危険なく多様な状況を生成して頑健性を高める方向も注目されています。モジュラーと E2E の二項対立は溶けつつあり、「微分可能で一括最適化できるが、内部に検証可能な中間構造を持つ」設計へ収斂しつつあるのが現在の潮流です。
