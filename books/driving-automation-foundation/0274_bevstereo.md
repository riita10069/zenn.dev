# BEVStereo

## ひとことで言うと
BEVStereo は、カメラ画像から鳥瞰図(BEV)を作って3次元物体を検出する手法で、深度(奥行き)推定の精度を上げるために「時系列の多視点ステレオ」という幾何の手がかりを使います。連続する2フレーム(時刻が少し違う画像)を、自車が動いたことで視点がずれた「左右の目」とみなし、人間が両眼で奥行きを感じるのと同じ原理(ステレオ視)で深度を測ります。さらに、動く物体や模様の乏しい場所などステレオが破綻しやすい状況を見抜いて、そこでは単眼の深度推定に切り替える適応的な仕組み(動的時系列ステレオ)を入れることで、LiDAR の正解ラベルなしでも深度の質を底上げしました。

## 直感的な理解
片目を閉じて指を顔の前に立て、左右の目で交互に見ると、指が背景に対して左右に飛んで見えます。近い指ほど大きく飛び、遠い壁はほとんど動きません。この「見える位置のズレ(視差)」の大きさから脳は距離を割り出しています。これがステレオ視です。

自動運転車に左右2台のステレオカメラを並べる代わりに、BEVStereo は「同じカメラの少し前のフレーム」を使います。自車が前に進めば、時刻 t-1 と t でカメラの位置がずれるので、この2枚はちょうど左右の目のように使えます。近い物体は2フレーム間で大きく動き、遠い物体はあまり動きません。このズレから深度を幾何的に計算するのです。LiDAR の正解を必要としないのが、深度を直接教え込む [BEVDepth](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0270_bevdepth) との大きな違いです。学習用の LiDAR が手に入らない、あるいは LiDAR の校正ずれが心配な現場で価値が出ます。

ただし、この「過去のフレームを使うステレオ」には固有の落とし穴があります。信号待ちで自車が止まっていると視点がずれず、視差がゼロに潰れてステレオが効きません。前を走る車のように動く物体は、2フレームで勝手に位置が変わるのでステレオの前提(同じ静止物が2視点に写る)を壊します。空や白い壁のように模様のない領域は、対応点を一意に決められません。BEVStereo はこうした失敗を見越して逃げ道を用意した点に賢さがあります。

## 基礎: 前提となる概念
いくつかの言葉を噛み砕きます。

BEV (Bird's-Eye-View) は鳥が真上から見下ろした俯瞰視点で、複数カメラの情報を自車中心の地図に統合した表現です。距離や位置関係を扱いやすく、運転判断につなげやすいのが利点です。

深度 (depth) は各ピクセルが「カメラから何メートル遠いか」を表す量です。2次元画像を3次元に戻すための鍵で、カメラ BEV 手法の精度はこの深度の質でほぼ決まります。これを実証したのが [BEVDepth](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0270_bevdepth) です。

視差 (disparity) は、2つの視点から同じ点を見たときの「画像上での位置のズレ」です。近い点ほど視差が大きく、遠い点ほど小さい。視差 disp と深度 Z、ベースライン(視点間距離)B、焦点距離 f の間には `Z = f · B / disp` という反比例の関係があり、視差を測れば深度が逆算できます。時系列ステレオではベースライン B が自車の移動量にあたります。

ステレオ視 (stereo vision) は、視差から深度を求める方法の総称です。左右2枚の画像で「同じ点がどこに写っているか(対応点)」を探し、そのズレから距離を出します。

エピポーラ幾何 (epipolar geometry) は、2視点の幾何関係を表す枠組みです。片方の画像のあるピクセルに対応する点は、もう片方の画像では1本の直線(エピポーラ線)上にしか存在しません。対応点探索を「面全体」から「1本の線」に絞れるので計算が軽くなります。時系列ステレオでは、カメラ運動が分かればこの線を引けます。

多視点ステレオ (Multi-View Stereo, MVS) は、2枚に限らず複数の視点画像から3次元を復元する古典的研究分野です。深層学習版の代表が MVSNet (Yao et al., ECCV 2018) で、後述するコストボリュームを微分可能な形で学習に組み込みました。BEVStereo のステレオ部分はこの系譜の考え方を時系列・BEV 文脈に持ち込んだものです。

時系列ステレオ (temporal stereo) は、左右カメラの代わりに「同じカメラの連続フレーム」をステレオの2視点として使う方式です。自車運動でできる視点差を利用します。

## 仕組みを詳しく
BEVStereo は、幾何的に正確だが状況依存で失敗もするステレオ深度と、常に出せるが精度は劣る単眼深度を、状況に応じて使い分けて融合します。

### ステレオ深度の基本: コストボリューム
ステレオ視の中心にあるのがコストボリューム (cost volume) です。これは「ある深度を仮定したとき、現フレームのピクセルと前フレームの対応ピクセルがどれだけ一致するか」を、複数の深度候補ごとに並べたテーブル(3次元のテンソル)です。

具体的な作り方を述べます。現フレームの特徴マップ `F_t` を固定し、深度候補 d ごとに、前フレームの特徴マップ `F_{t-1}` を「深度 d を仮定したときの幾何(ホモグラフィ/warp)」で現フレーム視点に変形(warp)します。warp した `F_{t-1}^{(d)}` と `F_t` の類似度(内積や差の絶対値など)を取ると、本当の深度のときに最も似て、外れた深度では似なくなります。これを全深度候補で並べたものがコストボリュームです。

```
コストボリューム C[d, h, w]:
  d = 深度候補のインデックス(例 0..D-1)
  (h, w) = 現フレーム特徴マップ上の位置
  値 = 深度 d を仮定したときの「現フレーム特徴 F_t」と
       「前フレームを warp した特徴 F_{t-1}^{(d)}」の一致度(group-wise correlation 等)

  → 各 (h, w) について d 方向で softmax を取り、期待値や分布として深度を推定
```

### BEVStereo の適応的な工夫
時系列ステレオを実用にするために、BEVStereo は中心的な工夫を入れました。

1. 動的な時系列ステレオ (Dynamic Temporal Stereo / dynamic depth candidates)。すべての深度候補(例 D=100 以上)を毎回均一に試すのは計算もメモリも重いので、まず単眼深度ネットで粗い深度分布を出し、その分布が示す「有望な深度範囲」の周りだけに少数の候補(例 数個〜十数個)を動的に配置して、そこを細かく(高解像度に)調べます。深度候補の位置をピクセルごとにデータ依存で動かすので「動的」と呼びます。これにより、限られた計算量でも精度の高い深度を出せます(全候補を一様に張る MVS の素朴版よりはるかに軽い)。

2. 信頼度に応じた融合。ステレオが効く場所(模様があり、自車が十分動き、対象が静止物)ではステレオ深度を重視し、ステレオが失敗しやすい場所(模様なし、自車が静止中、対象が動く物体)では単眼深度に頼る、という重み付けをネットワークが学習します。各ピクセルでステレオ・単眼それぞれの信頼度(confidence)を推定し、信頼度で加重平均して最終的な深度分布を作ります。停止時はそもそも有効なベースラインが取れないので融合の重みが単眼側に倒れます。

3. これにより、ステレオが使えない場面でも破綻せず、使える場面では幾何の正確さを取り込めます。最悪でも単眼相当の精度に落ちるだけで、決定的に壊れることを防ぎます。なお最終的な深度分布は、LiDAR 監督([BEVDepth](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0270_bevdepth) 流)を併用することもできます(BEVStereo は両者を組み合わせる設定も持つ)。

### 全体の流れ

```
現在フレーム t + 前フレーム t-1(各カメラ) + カメラ運動(自車の移動量・ego-motion)
   │
   ▼ 画像エンコーダ(重み共有)
画像特徴(F_t と F_{t-1})
   │
   ├─ 単眼深度推定(monocular DepthNet)──── 粗い深度分布 ─┐
   │                                                       │ 候補配置に使う
   ▼ 動的時系列ステレオ(候補を有望範囲に配置 → warp → 相関)│
コストボリューム → ステレオ深度分布                          │
   │                                                       │
   ▼ 信頼度に応じて融合(stereo / mono を confidence で加重)◀┘
   │
   ▼ 融合した高品質な深度分布 [N, D, H, W]
   │
   ▼ LSS 視点変換 → BEV 特徴 → BEV エンコーダ → CenterPoint ヘッド
3D 物体 box(位置・サイズ・向き・速度・クラス)
```

### 数値イメージ
自車が0.5秒で約5m進む場面を考えます。これがステレオのベースライン B ≈ 5m に相当します。5m先の物体と50m先の物体では、`Z = f·B/disp` の関係からフレーム間の視差が大きく異なり、近距離ほどステレオがよく効いて正確な深度が出ます。逆に信号待ちで自車が止まっていると B ≈ 0 となり視差がほぼ0に潰れ、`Z = f·B/disp` の式が機能せず対応点を絞り込めずステレオが破綻するので、その間は融合の重みが単眼側に寄り、破綻を避けます。前を走る車のように動く物体も、ステレオの静止仮定を破る(2フレームで物体自体が動く)ため信頼度が下がり、単眼に頼る挙動になります。

## 手法の系譜と主要論文
BEVStereo は2つの系譜の交点に立っています。

ひとつは古典的・深層的な多視点ステレオ。MVSNet (Yao, Luo, Li, Fang, Quan; ECCV 2018, arXiv:1804.02505) が微分可能なコストボリュームを深層学習に組み込み、ステレオ深度を end-to-end で学習できるようにしました。後続の R-MVSNet / CasMVSNet などが「粗から細へ(coarse-to-fine)」のカスケード深度推定で計算を減らす発想を確立し、BEVStereo の動的候補配置はこの coarse-to-fine の思想を時系列・自動運転に適用したものと読めます。

もうひとつはカメラ BEV 検出の深度改善の流れ。LSS (Philion & Fidler, ECCV 2020, arXiv:2008.05711、[Lift-Splat-Shoot (LSS)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0282_lift-splat-shoot)) が深度ベース視点変換の土台を作り、BEVDepth (Li et al., AAAI 2023, arXiv:2206.10092、[BEVDepth](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0270_bevdepth)) が「深度が検出のボトルネック」と診断して LiDAR 監督で深度を治しました。

そして BEVStereo (Li, Sun, Liang, Han, Zhang, Pi, Li, Zhang, Wang, Li; AAAI 2023, arXiv:2209.10248「BEVStereo: Enhancing Depth Estimation in Multi-view 3D Object Detection with Temporal Stereo」) が、BEVDepth と同じ問題意識(深度の質を上げる)を共有しつつ、手段を LiDAR 監督ではなく時系列ステレオに置きました。新規性は、時系列ステレオの固有の失敗ケース(静止・無模様・動的物体)を、動的な深度候補配置と信頼度融合で克服し、メモリ・計算を抑えながら実用的に機能させた点です。BEVStereo と BEVDepth は姉妹手法(同じ研究グループ)であり、両者を組み合わせる(幾何 + LiDAR 監督)発想もその後に現れました。

時間方向を使う別の系譜として [BEVDet4D](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0272_bevdet4d) がありますが、こちらは時間を主に「物体の速度推定」に使うのに対し、BEVStereo は時間を「深度の精緻化」に使う、という目的の違いがあります。さらに後続の SOLOFusion (Park et al., 2022) は、短期の高解像度ステレオ深度と長期の低解像度時系列融合を統合し、BEVStereo の短期ステレオを長期履歴と組み合わせて性能を伸ばしました。

## 論文の実験結果(定量データ)
評価は nuScenes ベンチマークで行われました。nuScenes は6カメラ・LiDAR・レーダーを積んだ車で集めた大規模データセット(約1000シーン、各20秒)で、カメラ3次元検出の標準評価舞台です。

指標の意味を確認します。mAP (mean Average Precision) は検出の正確さ(正しい位置・クラスで見つけられたか)を、3次元中心位置の距離しきい値で TP 判定して平均した値で、高いほど良い。NDS (nuScenes Detection Score) は mAP に位置(mATE)・サイズ(mASE)・向き(mAOE)・速度(mAVE)・属性(mAAE)の誤差を統合した総合指標で、検出が「どれだけ精密に当たったか」まで含むため実用的な総合点として重視されます。時系列を使うと特に mAVE(速度誤差)が下がるため NDS が大きく上がる傾向があります。

論文の報告によれば、BEVStereo は LiDAR 監督に頼らずに、カメラのみ3次元検出で当時の高水準を達成しました。代表的な数値を挙げます。

- ResNet-50 バックボーン・256x704 入力・2フレーム時系列の設定で、nuScenes val で mAP およそ 0.35、NDS がおよそ 0.50 前後と報告されています。これは同条件で時系列ステレオを使わない単眼深度ベース(BEVDepth 単フレーム相当)を数ポイント上回ります。
- 大きなバックボーン(V2-99 等)と高解像度・長い時系列を組み合わせた設定では、nuScenes test split で NDS が約 0.61、mAP が約 0.525 に到達したと報告されています。これは LiDAR 深度監督を使う BEVDepth とほぼ肩を並べる水準で、「LiDAR なしでも幾何で同等の深度品質に届く」ことを示します。

アブレーション(構成要素を抜く実験)では次が示されました。

- 時系列ステレオを外して単眼深度のみにすると精度が落ち(報告でおよそ -2〜-3 NDS 規模)、ステレオの寄与が確認されます。
- 動的な深度候補配置(粗く絞ってから少数候補を細かく調べる)を、均一に多数候補を試す方式に置き換えると、計算量・メモリが大きく増えるうえ精度はほとんど変わらない、というトレードオフの悪化が示されました。動的配置が「効率と精度の両立」に効いていることが分かります。
- 信頼度融合を外して常にステレオを使うと、停止時や動的物体で深度が破綻し、全体精度が下がります。融合が頑健性に寄与していることが裏づけられます。特に低速・停止シーンの多いシーケンスで効果が顕著です。

総じて、「時系列ステレオは正しく扱えば LiDAR なしでも深度を改善できる。ただし失敗ケースへの適応的対処が不可欠」というのが実験の結論です。

## メリット・トレードオフ・限界
メリット

- LiDAR の正解ラベルなしで、時系列ステレオの幾何によって深度を正確にできる。データ取得が画像だけで済み、LiDAR-カメラ校正ずれの影響も受けない。
- ステレオが失敗しやすい場面(静止・無模様・動的物体)に適応的に対処し、破綻を避ける。最悪でも単眼相当に落ちるだけ。
- 特別なステレオカメラ機材が不要(連続フレームを使うだけ)。
- 動的候補配置でコストボリュームを軽くしており、全候補一様 MVS より大幅に省メモリ。

トレードオフ・限界

- 前フレームの保持・処理とコストボリュームの warp/相関計算で、計算量とメモリが増える(単眼ベースより重い)。
- 自車運動と「対象が静止」という仮定に依存するため、停止時や動的物体ではステレオ効果が原理的に限定される(融合で緩和するが完全には消えない)。
- 単眼とステレオの融合重みを学習する必要があり、設計が複雑になる。学習も不安定になりやすい。
- カメラ運動(自己位置推定 ego-motion)の誤差がコストボリュームの幾何 warp に直接効くため、運動推定が不正確だと深度も劣化する。GPS/IMU やオドメトリの質に依存。
- 近距離ほどステレオが効き、遠距離は視差が小さくステレオの恩恵が薄い(遠方は単眼頼みに戻る)。

研究上の未解決課題としては、動的物体に対するステレオの正しい扱い(物体ごとの運動を補償するシーンフロー)、運動推定誤差への頑健化、ステレオと LiDAR 監督・自己教師あり深度の最適な統合、長期時系列とのスケーラブルな結合などが残っています。

## 発展トピック・研究の最前線
BEVStereo 以降、時系列をどう深度に効かせるかは活発な研究テーマであり続けています。

長期時系列の活用。直前1フレームだけでなく、数フレーム〜数秒の履歴を BEV 空間でアラインして積み上げると、検出と速度推定がさらに安定します。SOLOFusion はその代表で、長期低解像度の履歴と短期高解像度のステレオを組み合わせます。長期時系列を使う検出器ではメモリと計算をどう抑えるか(BEV 特徴のキャッシュと再利用)が課題です。

幾何と監督の統合。時系列ステレオ(幾何)と LiDAR 明示監督(教師)、さらに自己教師あり深度(連続フレームの再投影測光誤差を最小化する monodepth 系)を組み合わせ、それぞれの長所を取る方向。

クエリベースとの対比。BEVStereo は「深度を当てて特徴を前向きに持ち上げる」明示的な投影ですが、3次元クエリから画像にサンプリングしに行く後ろ向き(query-based)な時系列融合手法(BEVFormer の temporal self-attention、StreamPETR など)も並行して発展しており、両者の比較が続いています。

動的物体のシーンフロー。ステレオの静止仮定を緩め、物体ごとの運動(シーンフロー)を推定して補償することで、動く物体でもステレオを効かせる試みが研究されています。占有予測(occupancy)や4D 表現でも、時系列ステレオ由来の信頼できる深度が土台として再利用されています。
