# SwinV2マップバックボーン

## ひとことで言うと
地図の画像から「どこに道があり、どこが車線で、どこに横断歩道があるか」といった特徴を数値の塊として抜き出す土台のネットワークに、Swin Transformer V2 の Tiny という種類を使う構成の話です。ラスタ地図(画像として描かれた地図)を入力に、階層的で計算効率の良い注意機構(attention)で特徴を抽出します。Vision Transformer の弱点だった「高解像度での計算量爆発」をウィンドウ分割で解決した系列の代表格です。

## 直感的な理解
地図画像を理解するには、細い線(車線境界)のような局所的な細部と、交差点全体の形のような大局的な配置の両方を見る必要があります。細部だけ見ても「ここが右折レーン」とは分からず、全体だけ見ても「どこに線があるか」が分かりません。Swin Transformer は、まず小さな窓の中で細かく注意を払い、層を重ねるごとに窓をずらしたり領域をまとめたりして、だんだん広い範囲を見るように設計されています。人が地図を読むとき、まず近くの線を追い、次に少し引いて全体の構造を把握するのに似ています。

なぜわざわざ Transformer を使うのか。画像認識には長らく CNN(畳み込みニューラルネットワーク)が使われてきましたが、Transformer は「どの部分とどの部分が関係しているか」を直接計算できる柔軟さがあります。地図のように「離れた2本の車線が1つの交差点に属する」といった長距離の関係を捉えるのに向いています。ただし素朴な Transformer は計算量が画像サイズの2乗で増えるため高解像度に弱く、その弱点を実用的に解決したのが Swin です。

## 基礎: 前提となる概念

地図をモデルに渡す方法は大きく2つあります。ベクトル地図(vector map、車線や境界線を点の列・線分として数式的に表す形式)と、ラスタ地図(raster map、それを普通の画像として描き起こした形式)です。ラスタ地図なら「画像」なので、画像認識で実績のあるネットワークをそのまま流用できます。本ノートが扱うのはラスタ地図のエンコードです。

画像から特徴を抜き出す土台をバックボーン(backbone)と呼びます。大きく2系統あります。CNN 系(ResNet, ConvNeXt など)はご近所のピクセルを小さなカーネルで少しずつ見て積み上げます。Transformer 系は画像を小さなパッチ(タイル)に切り、パッチ同士の関係を注意機構で捉えます。

注意機構(attention)とは、各要素(ここではパッチ)が他の全要素を「どれだけ重視すべきか」を計算し、重み付き平均で情報を集める仕組みです。クエリ(query、問い合わせ役)、キー(key、照合役)、バリュー(value、中身)という3つのベクトルを作り、クエリとキーの内積で類似度を出し、それで value を重み付けします。全パッチが全パッチを見る素朴な自己注意(self-attention)は、パッチ数 N に対して計算量が N の2乗です。画像が大きくなると N が増えるので、解像度の2乗で計算が膨らみます。これが Vision Transformer(ViT, Dosovitskiy et al., ICLR 2021)の高解像度での弱点でした。

トークン(token)は Transformer が扱う最小単位で、ここでは1パッチ=1トークンです。各トークンは固定長のベクトル(埋め込み)で表されます。

## 仕組みを詳しく

ラスタ地図エンコーダで Swin V2 Tiny を使う典型的な生成は次の形です(timm のようなモデルライブラリ経由)。

```python
backbone = timm.create_model(
    "swinv2_tiny_window8_256",
    pretrained=False,      # 事前学習重みを使わずランダム初期化
    features_only=True,    # 各ステージの中間特徴を全部返す
    in_chans=in_channels,  # 通常3 (RGB地図)
)
```

モデル名 `swinv2_tiny_window8_256` を分解します。

- swinv2: Swin Transformer V2。
- tiny: 小型構成(パラメータ約 28M)。Base(約 88M)や Large(約 197M)より軽く、計算資源を抑えたいときの選択。
- window8: 注意計算を 8x8 のウィンドウ(窓)単位で行う。これが Swin の肝。
- 256: 想定入力解像度が 256x256。地図画像をこの解像度に揃えて渡すのが前提。

Swin の中身を初心者向けに順を追います。

ステップ1 パッチ分割。256x256 の画像を小さなパッチ(Swin は 4x4 ピクセル)に切り、各パッチを線形変換で1トークンにします。256/4 = 64 なので、最初は 64x64 = 4096 トークンが並びます。

ステップ2 ウィンドウ注意(Window-based Multi-head Self-Attention, W-MSA)。全トークンが全トークンを見るのではなく、8x8 のウィンドウ内のトークンだけで注意を計算します。各ウィンドウは独立に処理されるので、計算量は画像サイズに対して2乗ではなく一次(線形)で済みます。これが高解像度でも軽い理由です。具体的には、ウィンドウサイズを M(ここでは 8)とすると、計算量は ViT の O(N^2) に対し O(N·M^2) になり、M を固定すれば N に線形です。たとえば N=4096 なら、全域注意は約 1670 万ペアを計算しますが、8x8=64 トークンのウィンドウ 64 個なら 64×(64^2)≈26 万ペアで済み、約 64 分の 1 に減ります。

ステップ3 シフトウィンドウ(Shifted Window, SW-MSA)。ウィンドウを固定したままだと窓の境界をまたいだ情報が混ざりません。そこで層ごとにウィンドウを半分(M/2)だけずらし、隣の窓と情報が行き来できるようにします。Swin のブロックは「W-MSA の層」と「SW-MSA の層」を交互に並べる構造で、これで局所と局所をまたいだ情報伝播を両立します。Shifted Window が名前の由来です。

ステップ4 階層化(ステージとパッチマージ)。数ブロックごとに「パッチマージ(patch merging)」で隣接 2x2 のトークンをまとめ、解像度を半分にしてチャンネル数を倍にします。CNN のダウンサンプリングに相当します。結果、ステージごとに 96 → 192 → 384 → 768 チャンネルといった階層特徴が得られます。浅いステージは高解像度で細部、深いステージは低解像度で大局を捉えます。

`features_only=True` を付けると、最終出力だけでなく各ステージの中間特徴をすべて返します。ラスタ地図では細い線も大局配置も使いたいので全段を取り、後段で統一・結合・チャンネル射影します(融合の入口)。

形の流れを整理します。

- 入力: (B, 3, 256, 256)
- Swin 各ステージ出力(例): (B, 96, 64, 64), (B, 192, 32, 32), (B, 384, 16, 16), (B, 768, 8, 8)
- プーリングで空間統一後: 全ステージ (B, C_i, 8, 8)
- チャンネル方向結合後: (B, 1440, 8, 8)  (96+192+384+768)
- チャンネル射影後: (B, 256, 8, 8)

なお、Swin 実装の出力は (B, H, W, C) のチャンネル末尾(channels-last)形式で返ることがあり、後段の Conv 系処理に渡す前に `permute(0, 3, 1, 2)` でチャンネル先頭(channels-first)に直す整合作業が入ることがあります。

`pretrained=False`(ランダム初期化)を選ぶ理由は、ラスタ地図画像が ImageNet の自然画像とは見た目がかなり違う(道路の塗り分け図のようなもの)ため、ImageNet 事前学習の恩恵が必ずしも大きくない、という判断があります。一方で十分な学習データがない場合は、事前学習の方が有利になることもあり、ここはデータ規模とのトレードオフです。

## 手法の系譜と主要論文

ViT (Dosovitskiy et al., ICLR 2021, arXiv:2010.11929)。「An Image is Worth 16x16 Words」。画像を 16x16 パッチに切り、純粋な Transformer で分類する手法を確立しました。新規性は「畳み込みをほぼ捨てても、大規模事前学習があれば CNN に勝てる」と示したこと。限界は、全パッチ全結合の自己注意で計算量が解像度の2乗になり、検出やセグメンテーションのような高解像度・密予測タスクに不向きだった点です。

Swin Transformer (Liu et al., ICCV 2021, arXiv:2103.14030, Microsoft Research Asia)。「Hierarchical Vision Transformer using Shifted Windows」。ウィンドウ注意とシフトウィンドウで計算量を画像サイズに線形化し、CNN のような階層的特徴ピラミッドを Transformer で実現しました。動機は、密予測には高解像度かつ階層的な特徴が要るが ViT は計算量が爆発するため。これにより検出・セグメンテーションのバックボーンとして広く使われるようになりました。トレードオフは、ウィンドウ分割という設計制約が入り純粋な全域注意より柔軟性が下がること、実装がやや複雑になることです。

Swin Transformer V2 (Liu et al., CVPR 2022, arXiv:2111.09883)。本ノートが扱う V2 です。主な改良は3点。(1) 残差接続の後に Layer Norm を置く post-norm への変更、(2) ドット積の代わりにコサイン類似度ベースのスケールド注意(cosine attention)、(3) ログ間隔の連続相対位置バイアス(log-spaced continuous position bias)。動機は、V1 を巨大化・高解像度化すると学習が不安定になり、低解像度で学習して高解像度で使う「解像度転移」が難しかったため。効果は、最大 30 億パラメータ(SwinV2-G)まで安定に学習でき、低解像度学習→高解像度適用の転移がしやすくなったこと。トレードオフは、これらの工夫は巨大モデルでこそ効き、Tiny のような小型では恩恵が相対的に小さい点です。

比較対象として ConvNeXt (Liu et al., CVPR 2022, arXiv:2201.03545)。CNN を Transformer 風の設計思想(大きなカーネル、LayerNorm、GELU、少ない活性化など)で現代化し、Swin と同等以上の精度を純 CNN で達成しました。これは「Transformer の優位は注意機構そのものより、学習レシピと設計の現代化に多くを負う」ことを示し、バックボーン選択を Transformer 系と CNN 系で差し替え比較する意義を裏付けます。地図/画像のどちらにどちらが適するかは入力の性質とデータ規模で変わるため、差し替え可能にしておくのは合理的です。

地図の応用としては HDMapNet (Li et al., ICRA 2022, arXiv:2107.06307) が、カメラやセンサ特徴を BEV へ投影してから地図要素(車線・横断歩道など)を推定します。BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270) は時空間クロスアテンションで BEV 表現を構築し、ここで階層型バックボーンのマルチスケール出力がキー/バリューとして使われます。地図そのものを画像として符号化する流れと、地図を出力として推定する流れの両方で、強力な画像バックボーンが基盤になります。

## 論文の実験結果(定量データ)

Swin V1 の ImageNet-1K 分類では、Swin-T(Tiny、約 28M パラメータ、計算量約 4.5 GFLOPs)が top-1 精度およそ 81.3% を達成しました。これは同程度のパラメータ数の ResNet-50(約 76.2%)や DeiT-S(約 79.8%)を上回ります。top-1 精度とは「最上位の予測が正解だった割合」で、大きいほど良い指標です。Swin-B(Base、約 88M)では約 83.5%、Swin-L を ImageNet-22K で事前学習すると約 87% に達します。

Swin の真価は密予測タスクで顕著です。物体検出ベンチマーク COCO で、Swin をバックボーンにした構成は box AP(平均適合率、検出の正確さを 0〜100 で測る指標、大きいほど良い)で、当時の ResNeXt 系バックボーンを数ポイント上回り、Cascade Mask R-CNN との組み合わせで box AP 51〜58 程度を報告しました。セマンティックセグメンテーションの ADE20K では、Swin が mIoU(平均 Intersection over Union、予測領域と正解領域の重なり具合の平均、大きいほど良い)でおよそ 53.5%(Swin-L, UPerNet)に達し、従来手法を更新しました。これらは「ウィンドウ注意による線形計算量で高解像度を扱えるからこそ、密予測で勝てる」ことの実証です。

Swin V2 では、解像度転移とスケーリングの実験が中心です。低解像度(例 192x192 や 256x256)で事前学習したモデルを、より高い解像度(例 384x384 や 512x512、巨大モデルでは 1536x1536)で微調整・推論する際、V1 では精度が崩れやすかったのに対し、V2 の改良(cosine attention と log-spaced position bias)で安定しました。最大構成 SwinV2-G(約 30 億パラメータ)は、ImageNet-V2 で top-1 約 84.0%、COCO test-dev で box AP 約 63.1、ADE20K で mIoU 約 59.9 と、当時の複数ベンチマークで最高水準を記録しています。アブレーションでは、post-norm と cosine attention を外すと巨大モデルで学習が発散しやすくなることが示されており、これらが安定化に効いていることが分かります。一方 Tiny 規模では V1 と V2 の差はわずか(ImageNet top-1 で 1 ポイント未満)で、安定化の恩恵は主に Base 以上で表れます。これは「小型では V2 特有の工夫の価値が限定的」という前述のトレードオフの定量的裏付けです。

ConvNeXt 論文の比較では、ConvNeXt-T が Swin-T と同等の ImageNet top-1(約 82.1%)を、純 CNN で達成しています。これにより「Tiny クラスでは Transformer か CNN かよりも、設計と学習レシピが効く」ことが定量的に示されました。

## メリット・トレードオフ・限界

メリット
- ウィンドウ注意で高解像度でも計算量が線形に抑えられ、地図の細い線を捉えやすい。
- 階層的な多段特徴が得られ、全段(features_only)を融合に使える。
- モデルライブラリ(timm 等)で簡単に差し替え可能で、ConvNeXt や ResNet との比較検証がしやすい。
- 長距離依存(離れた車線が同じ交差点に属する等)を注意機構で捉えやすい。

トレードオフ・限界
- 入力が 256x256 を想定し、地図画像をこの解像度に合わせる必要がある(情報の粒度がこれに縛られる)。高解像度が要るなら別の window/解像度設定が必要。
- ランダム初期化(pretrained=False)は、ドメインが自然画像と離れている利点がある反面、学習データが少ないと特徴抽出が弱くなる。
- Transformer は一般に CNN よりデータ要求量が多い傾向があり、小規模データでは過学習しやすい。
- V2 の主要な工夫(安定化・解像度転移)は Tiny では恩恵が限定的で、計算コストに見合わない場合がある。
- ウィンドウ分割は局所性の仮定であり、極端に大局的な依存を1層で捉えるには複数ブロックを要する。

## 発展トピック・研究の最前線

Swin の後継・周辺には多くの発展があります。Swin V2 が示したスケーリングと解像度転移の安定化は、その後の大規模ビジョン基盤モデルの設計に影響しました。一方で「ウィンドウ注意は本当に必要か」という問いから、グローバル注意を効率化する手法(線形注意、疎な注意、FlashAttention の実装最適化など)や、ConvNeXt のように畳み込みで対抗する流れも並行して進んでいます。

自動運転の文脈では、画像バックボーンの出力を BEV 空間へ持ち上げる(lift する)研究が活発です。BEVFormer(Li et al., ECCV 2022)は、画像特徴に対して時空間のクロスアテンションを行い BEV 表現を構築します。ここでバックボーンの階層特徴がマルチスケールのキー/バリューとして使われ、Swin 系の階層出力との相性が良いです。地図エンコードの最前線では、ラスタ地図を画像バックボーンで符号化する方式と、ベクトル地図をグラフ/トランスフォーマで符号化する方式の使い分け・統合が論点で、両者を同一アーキテクチャで切り替え可能にする設計(ラスタ化エンコーダとベクトル化エンコーダの両対応)が研究・実装の両面で模索されています。バックボーンを差し替え可能にしておくことは、こうした探索を低コストに回すための実務的な前提になっています。
