# 行動クローニングによるE2E運転

## ひとことで言うと

行動クローニング(behavior cloning)とは、人間が運転したログ(カメラ映像とそのときのハンドル・アクセル操作)を教師データにして、「この映像を見たら、人間ならどう操作したか」をニューラルネットワークに直接まねさせる学習方法です。E2E(end-to-end、エンドツーエンド)運転とは、カメラ画像などの生の入力から、ハンドル角や速度といった制御出力までを1つのネットワークで一気通貫に出す方式を指します。従来のように「物体検出→車線認識→経路計画→制御」と段階を分けて作り込むのではなく、人間の運転を丸ごと模倣する、最もシンプルで歴史の古いE2E学習がこの行動クローニングです。

## 直感的な理解

子供が大人の運転を後ろの席からずっと見て育ち、見様見真似でハンドル操作を覚えるようなものです。「この景色のときはこう曲がっていた」というパターンを大量に覚え込みます。技術的には、入力(カメラ画像)と出力(そのとき人間が切ったハンドル角)のペアを膨大に集め、画像を入れたら人間と同じハンドル角を返すように関数を学習する、ただの教師あり学習(正解ラベル付きデータからの学習)です。

シンプルゆえに強力ですが、根本的な弱点があります。教師は「人間が普通にうまく運転したログ」しか含みません。すると、いったん車が道の端に寄るなどの「人間ならやらない失敗状態」に陥ると、その状態の正解データが学習に存在しないため、モデルは何をすべきか分からず、さらに失敗を重ねて破綻します。これを分布シフト(distribution shift)あるいは共変量シフト(covariate shift)と呼びます。学習時に見た状況の分布と、実際に運転中に出会う状況の分布がずれていく、という意味です。誤差が誤差を呼ぶこの連鎖が、行動クローニング最大の課題です。

## 基礎: 前提となる概念

教師あり学習(supervised learning)。入力と正解ラベルのペアからなるデータで、入力→正解を予測する関数を学習する、機械学習の最も基本的な枠組みです。行動クローニングは「画像→操作」を学ぶ教師あり学習そのものです。

回帰(regression)と損失関数。ハンドル角のような連続値を予測することを回帰と言います。予測と正解のずれを測るのが損失関数で、よく使う平均二乗誤差(MSE, mean squared error)は (予測 - 正解)² の平均です。値が小さいほど人間の操作に近い。学習はこの損失を小さくするように重みを調整します。

模倣学習(imitation learning)。専門家(人間)の振る舞いをまねして方策(policy、状況→行動の対応づけ)を学ぶ分野の総称で、行動クローニングはその最も単純な一種です。報酬を設計して試行錯誤で学ぶ強化学習と対比されます。模倣学習は報酬設計が要らず、専門家データさえあればよいのが利点です。

## 仕組みを詳しく

基本パイプラインは次の通りです。まず人間が運転し、前方カメラ画像 x と、その瞬間のハンドル角(またはステアリング・スロットル・ブレーキ)y をペアで記録します。次に畳み込みニューラルネットワーク(CNN、画像処理に適したネットワーク)に画像 x を入れ、出力 ŷ がなるべく人間の y に一致するよう、損失 L = (ŷ - y)² を最小化して重みを学習します。推論時は、走行中のカメラ画像をそのままネットに通し、出てきたハンドル角で車を操作します。

NVIDIA の PilotNet(Bojarski ら, 2016)を具体例にします。入力は前方カメラ画像を 66×200 ピクセルにリサイズしたもの。ネットワークは正規化層 + 畳み込み5層 + 全結合3層で、最終出力はハンドルの逆半径(1/曲率、まっすぐなら0)という1つの数値です。テンソルの形で追うと、入力 [3, 66, 200](3はRGBチャンネル)→ 畳み込みで空間を圧縮しながらチャンネルを増やし → 平坦化 → 全結合 → 出力 [1]。たった1枚の画像から1つの操舵量を出す、驚くほど単純な構成です。

```
カメラ画像 [3,66,200]
    │  畳み込み5層(特徴抽出)
    ▼
特徴マップ → 平坦化
    │  全結合3層
    ▼
ハンドル角(逆半径)[1]
```

分布シフトへの実践的な対策が2つ重要です。1つ目はデータ拡張による回復行動の教え込みです。PilotNet は中央カメラに加え左右にもカメラを付け、左カメラ画像には「右に戻れ」、右カメラ画像には「左に戻れ」という補正済みラベルを与えました。これにより、車が道の端に寄った状況とそこからの復帰操作を擬似的に学習でき、自己回復能力を持たせられます。

2つ目は DAgger(データ集約、Dataset Aggregation)というアルゴリズム(Ross ら, 2011)です。学習したモデルに実際に運転させ、モデルが訪れた状態(失敗ぎみの状態を含む)に対して「専門家ならどう操作するか」を後から人間がラベル付けし、そのデータを学習に追加して再学習します。これを繰り返すと、モデルが実際に出会う状態分布の正解データが揃い、分布シフトが大幅に緩和されます。理論的にも、素朴な行動クローニングの誤差が軌道長 T に対して T² で悪化しうるのに対し、DAgger は T に線形に抑えられることが示されています。

## 手法の系譜と主要論文

最初期は Pomerleau の ALVINN(Autonomous Land Vehicle in a Neural Network, NeurIPS 1989)です。3層のニューラルネットに前方カメラとレーザー測距の低解像度入力を与え、進むべき方向を出力させました。1989年当時の計算機でこれを実現し、実車を時速数キロで自律走行させた、E2E運転の元祖です。当時のネットワークは入力 30×32 程度の極めて小さなもので、現代から見れば玩具のようですが、発想は今と同じでした。

時代を飛ばして、深層学習の隆盛とともに NVIDIA の Bojarski ら "End to End Learning for Self-Driving Cars"(2016, arXiv:1604.07316)が PilotNet を発表し、現代的なCNNでE2E運転が実用レベルで動くことを示して大きな注目を集めました。続く解説論文 "Explaining How a Deep Neural Network Trained with End-to-End Learning Steers a Car"(2017, arXiv:1704.07911)では、ネットが画像のどこを根拠に判断しているか(車線や道路境界に反応していること)を可視化しました。

理論面では Ross, Gordon, Bagnell の DAgger(AISTATS 2011, arXiv:1011.0686)が、行動クローニングの分布シフト問題を定式化し、反復的データ集約で解決する枠組みを与えました。模倣学習を語るうえで欠かせない論文です。

その後の発展として、CARLA(運転シミュレータ)上での条件付き模倣学習(Codevilla ら, ICRA 2018, arXiv:1710.02410。「次の交差点で右折」のような高レベル指令を条件に加える)、模倣学習の限界を分析した "Exploring the Limitations of Behavior Cloning for Autonomous Driving"(Codevilla ら, ICCV 2019, arXiv:1904.08980)、そして知覚から計画までを学習で統合する近年のE2E自動運転(UniAD など, CVPR 2023)へと系譜が続きます。行動クローニングは今もこれら高度な手法の土台、あるいは初期化として使われ続けています。

## 論文の実験結果(定量データ)

PilotNet(Bojarski ら, 2016)の主な指標は自律走行率(autonomy)で、「人間の介入なしに走れた時間の割合」を表します。論文の定義は、autonomy = (1 − 介入回数 × 6秒 / 走行時間)× 100% で、1回の介入を6秒のロスとみなして計算します。報告では、実車テストでおよそ98%の自律走行率を達成し、ニュージャージーからの一般道・高速の走行で、ほとんど人間の介入なしに走れたとされています。学習データはおよそ72時間分の運転映像でした。なお、これは限定的な条件下の報告であり、あらゆる状況での安全性を保証するものではない点に注意が必要です。

Codevilla ら(2019, ICCV)の限界分析は、行動クローニングの弱点を定量的に示した重要な研究です。CARLA シミュレータで、学習データ量を増やすと性能が一旦上がるものの、ある量を超えると頭打ち、あるいは悪化することがあると報告しました。また、同じ学習設定でも乱数の初期値(シード)を変えるだけで成功率が大きくばらつき、運転成功率の標準偏差が無視できない大きさになることを示しました。さらに、過去の数フレームを入力に加えると、モデルが「直前の自分の動き」をショートカット的に模倣してしまう慣性問題(inertia problem、止まっていると止まり続ける)が起きることも指摘しました。これらは「データを増やせば解決する」という素朴な期待への警鐘になりました。

DAgger(Ross ら, 2011)は、スーパーマリオのゲームAIなどの実験で、素朴な行動クローニングが訪れたことのない状態で破綻するのに対し、反復的データ集約により大幅に安定した方策が得られることを示しました。誤差の理論限界が軌道長に線形に抑えられるという結果が、実験的にも裏付けられています。

## メリット・トレードオフ・限界

メリットは圧倒的なシンプルさです。報酬設計が不要で、人間の運転ログさえあれば教師あり学習として学習でき、実装も理解も容易です。中間表現を設計せずに済むため、人間が見落とす微妙な手がかり(路面状態や周囲の雰囲気)も学習しうる、というE2E共通の魅力もあります。

トレードオフと限界は深刻です。第一に分布シフト。前述のとおり失敗状態の正解が無く、誤差が累積して破綻しやすい。第二に因果の混同(causal confusion)。例えば「ブレーキランプが点いている前車」と「自分が減速する」という相関を学んだモデルが、原因と結果を取り違えることがあります。第三にロングテール(稀なケース)への弱さ。データに乏しい珍しい状況(動物の飛び出し、特殊な工事現場)では適切に振る舞えません。第四に解釈性と検証の難しさ。1つのブラックボックスなので、なぜその操作をしたかの説明や、安全性の保証が難しい。第五に、人間以上にはなれないこと。専門家の模倣なので、教師の人間が下手なら、あるいは人間が経験しない状況では性能が頭打ちになります。

## 発展トピック・研究の最前線

分布シフトへの対策として、DAgger 系の反復データ集約、シミュレータで失敗状態を大量生成して回復行動を学ばせる手法、そして模倣学習と強化学習を組み合わせて専門家を超える方策を探す研究が進んでいます。因果混同を避けるため、因果構造を明示的に扱う模倣学習も提案されています。近年は、軌道(将来の経路点の列)を出力させる軌道予測型の行動クローニングや、Transformer・拡散モデル(diffusion policy)で多峰的な人間行動(同じ状況でも複数の妥当な選択がある)を表現する手法が注目されています。さらに、知覚・予測・計画を学習で統合しつつ、解釈可能な中間表現(鳥瞰図上の占有や軌道)を経由させることで、E2E の性能と検証可能性を両立させようとする統合型自動運転が最前線です。行動クローニングはこれらの出発点として、依然として基礎的かつ重要な位置を占めています。
