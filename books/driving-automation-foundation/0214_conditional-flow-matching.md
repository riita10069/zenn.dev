# Conditional Flow Matching (CFM)

## ひとことで言うと
Conditional Flow Matching（条件付きフローマッチング、CFM）は、ノイズの分布からデータの分布へと確率の塊を「流す」ためのベクトル場——空間の各点で「どっちへどれだけの速さで進むべきか」を表す速度の地図——を、シミュレーションなしの単純な二乗誤差回帰だけで学習する生成モデルの枠組みです。拡散モデルが数百〜千ステップかけて少しずつノイズを除くのに対し、CFM はノイズとデータをほぼ直線で結ぶ経路を学べるため、数ステップ〜十数ステップで生成できます。理論的には拡散モデルを含む一般化であり、近年の大規模画像・動画・ロボット行動・走行軌道生成の標準的な土台になっています。

## 直感的な理解
たくさんの砂粒（=確率の塊）が、時刻 t=0 ではランダムに散らばっている（標準ガウスノイズの分布）とします。時刻 t=1 には、その砂粒たちが「犬の画像」や「ありえそうな未来の走行軌道」の形に整列していてほしい。各砂粒が「今この瞬間、どっちへどれくらいの速さで動けばよいか」を教えてくれる矢印の地図があれば、ノイズから出発した砂粒を地図に従って動かすだけでデータが作れます。この矢印の地図がベクトル場（速度場）です。

問題は、この地図をどうやって学習するかです。素直に考えると「t=0 のノイズ分布から地図に従って t=1 まで実際に流してみて、できあがった分布がデータ分布と一致するように地図を直す」という方法になります。ところがこの「実際に流してみる」というのが常微分方程式（ODE）を数値積分する作業で、学習のたびに大量の計算が必要で、しかも積分誤差が積み重なって不安定になります。CFM の発明は、この「実際に流してみる」作業を学習から完全に追放したことです。地図を直接、いきなり、二乗誤差で当てにいけるようにしました。これが「シミュレーションフリー（simulation-free）」という言葉の意味です。

ここで強調したいのは、「学習したいもの（データ全体を正しく運ぶ速度場）」は手で書けないのに、「学習に使う教師信号（個々のデータ点を運ぶ自明な速度）」は紙と鉛筆で書ける、という非対称性です。CFM はこの2つが二乗誤差の意味で同じ勾配を生むことを証明し、書ける方を当てにいくだけで書けない方が学習できる、という橋を架けました。

## 基礎: 前提となる概念

CFM を理解するには、いくつかの前提を順に押さえる必要があります。前提知識ゼロから積み上げます。

確率分布の補間: 分布 p（ノイズ、例えば標準ガウス分布 N(0, I)。I は単位行列で「各次元が独立に分散1」を意味する）と分布 q（データ）があるとき、その途中をなめらかにつなぐ分布の列 p_t（t は 0 から 1 まで動く時刻）を考えます。p_0 = p（ノイズ）、p_1 = q（データ）です。この p_t を「確率パス（probability path）」と呼びます。生成とは「ノイズの分布 p_0 から出発して、この確率パスをたどって p_1 のデータ分布に到達する」ことに他なりません。

連続正規化フロー（CNF, Continuous Normalizing Flow）: 確率パスを「ベクトル場 v(x, t) に従って点を動かす」という形で実現する枠組みです。点 x の動きは常微分方程式 dx/dt = v(x, t) で書かれます。これは「位置 x にいる点は、毎瞬間 v(x, t) の速度で動く」という意味です。ベクトル場 v が決まれば、初期分布 p_0 をどう流すと p_1 になるかが一意に決まります。逆に「p_0 を p_1 に運ぶ v」を見つけられれば生成器が手に入ります。CNF は Chen らの Neural ODE（NeurIPS 2018, arXiv:1806.07366）に始まる枠組みで、ニューラルネットでベクトル場を表現します。

連続の方程式（continuity equation）: 確率パス p_t とベクトル場 v_t が「整合している」とは、流体力学の連続の方程式 ∂p_t/∂t + div(p_t v_t) = 0 を満たすことを言います。div は発散（divergence、その点から確率がどれだけ湧き出す/吸い込まれるか）です。直感的には「確率は途中で消えたり湧いたりせず、ただ流れて移動するだけ」という保存則です。この方程式を満たす v_t を「p_t を生成するベクトル場」と呼びます。求めたいのはこの v_t です。

従来 CNF の問題: 従来の CNF は、学習のたびに ODE を順方向・逆方向に数値積分し、最尤推定（生成された分布がデータにどれだけ合うか、対数尤度）を最大化していました。対数尤度を計算するには、変数変換に伴うヤコビアン行列式の対数 log|det ∂f/∂x| が必要で、これを連続時間で扱うと ∫ div(v) dt という積分になり、これ自体 ODE を解く作業になります。つまり「学習1回ごとに ODE を解く」ため計算が極端に重く、大規模データでは事実上使えませんでした。CFM はこの最尤推定を捨て、ベクトル場を直接回帰する方針に切り替えます。

最適輸送（OT, optimal transport）: 分布 p を分布 q に「総移動コスト最小」で運ぶ写像・経路を求める数学。コストを移動距離の二乗にした場合、最適な運び方は各点が直線で目的地へ進む形になります。CFM が「直線パスを選ぶと速い」と言うとき、背後にあるのがこの OT の直感です。

## 仕組みを詳しく

CFM の核心は「周辺ベクトル場は直接学べないが、条件付きベクトル場なら自明に書ける。そして両者を二乗誤差で結ぶと勾配が一致する」という数学的な仕掛けです。順を追って説明します。

ステップ1: 条件付き確率パスを定義する。各データ点 x_1 を1つ固定し、その x_1 を終点とする条件付きパス p_t(x | x_1) を設計します。最もよく使われるのが、ノイズ x_0 と固定したデータ x_1 を直線で結ぶ補間です。

```
x_t = (1 - t) * x_0 + t * x_1        （x_0 ~ N(0, I)）
```

t=0 で x_0（ノイズ）、t=1 で x_1（データ）になります。この直線上を等速で進むので、各時刻の条件付き速度はずっと一定で、

```
条件付き速度 u_t(x | x_1) = x_1 - x_0
```

たったこれだけです。終点から始点を引いた一定ベクトルが、その砂粒が進むべき速度になります。これは時刻 t に依存しません。なお拡散モデルに対応する条件付きパスを選ぶこともでき（x_t = ᾱ_t x_1 + σ_t x_0 の形）、その場合の条件付き速度は係数の時間微分から決まるやや複雑な式になります。CFM の偉さは「補間の形を自由に選べる一般枠組み」である点で、直線だけでなく拡散相当の曲がったパスも同じ理屈で扱えます。

ステップ2: 周辺ベクトル場と条件付きベクトル場の関係。本当に学習したいのは「データ全体を集めた周辺確率パス p_t を生成する周辺ベクトル場 u_t(x)」です。これは個々の条件付きパスを x_1 について混ぜ合わせたもので、

```
u_t(x) = E_{x_1}[ u_t(x | x_1) * p_t(x | x_1) / p_t(x) ]
       = ∫ u_t(x | x_1) * p_t(x | x_1) * q(x_1) / p_t(x) dx_1
```

という重み付き平均で書けます。この周辺速度を直接計算するには分母の p_t(x)（周辺密度、すべての x_1 を混ぜた密度）が必要で、これは積分が解けず扱えません。ここが従来手法の壁でした。

ステップ3: CFM 定理（二乗誤差なら勾配が一致する）。Lipman らが示した鍵となる定理は、次の2つの損失が θ（ネットワークの重み）に関して同じ勾配を持つ、というものです。

```
周辺 Flow Matching 損失:  L_FM(θ)  = E_{t, x ~ p_t} || v_θ(x, t) - u_t(x) ||²
条件付き Flow Matching 損失: L_CFM(θ) = E_{t, x_1 ~ q, x ~ p_t(·|x_1)} || v_θ(x, t) - u_t(x | x_1) ||²
```

L_FM は扱えない（u_t(x) が計算できない）が、L_CFM は自明に計算できます（u_t(x | x_1) = x_1 - x_0 と分かっているから）。証明の核は「二乗を展開すると、両損失は v_θ の線形項と二乗項で構成され、定数差を除いて一致する」ことを期待値の積分順序を入れ替えて示す点にあります。両者の θ 勾配が等しいので、L_CFM を最小化すれば、求めたかった周辺ベクトル場 u_t(x) を学習したことになります。これが CFM の心臓部です。「条件付き（個々の x_1 ごとの自明な速度）を当てにいくだけで、全体として正しい速度場が学べる」という等価性です。

ステップ4: 実際の学習ループ。これ以上ないほど単純です。

1. データ x_1 をミニバッチで取る（shape 例 [B, D]、B はバッチサイズ、D は次元）。
2. ノイズ x_0 ~ N(0, I) を引く（同じ shape [B, D]）。
3. 時刻 t を一様乱数 U(0,1) で引く（shape [B]）。
4. 中間点 x_t = (1-t) x_0 + t x_1 を計算する（[B, D]、t は次元方向にブロードキャスト）。
5. 目標速度 target = x_1 - x_0 を計算する（[B, D]、時刻によらず一定）。
6. ネットワーク v_θ(x_t, t) を評価し、損失 mean(|| v_θ(x_t, t) - target ||²) を最小化する。

ODE は一切解いていません。直線を1本引いて、その上の1点で速度を当てさせるだけ。これがシミュレーションフリーの正体です。

数値例: D=2 で x_0 = (1.0, -0.5)、x_1 = (3.0, 2.0)、t=0.25 とすると、x_t = 0.75·(1,-0.5) + 0.25·(3,2) = (1.5, 0.125)、target = (3-1, 2-(-0.5)) = (2.0, 2.5)。ネットワークは点 (1.5, 0.125) と時刻 0.25 を入力に (2.0, 2.5) を出すよう学習します。別の t を引けば同じ x_0, x_1 のペアでも別の x_t になり、target は同じ (2.0, 2.5) のままです（直線上はどこでも速度一定だから）。

ステップ5: 生成（サンプリング）。学習後はノイズ x_0 ~ N(0, I) から出発し、ODE dx/dt = v_θ(x, t) を t=0 から t=1 まで数値積分します。最も単純なオイラー法なら、刻み幅 Δt = 1/N（N は推論ステップ数）で、

```
x ← x + Δt * v_θ(x, t),   t ← t + Δt        （これを N 回）
```

直線輸送を学んでいるなら生成パスもほぼまっすぐで、N=10 程度の粗い刻みでも崩れにくいのが特徴です。拡散の数百〜千回と比べて桁違いに軽くなります。

条件付き生成（条件変数 c の注入）: 画像なら「テキスト」、ロボット行動なら「観測画像と状態」、走行軌道なら「鳥瞰図（BEV, Bird's-Eye-View）特徴・自車運動履歴・周囲物体の状態」などの文脈 c をベクトル場に入れ、v_θ(x, t, c) とします。時刻 t は正弦波埋め込み（sinusoidal embedding、実数 t を異なる周波数の sin/cos の組で連続ベクトルに変換するもの）で渡し、c はクロスアテンション（生成中の特徴が条件特徴を参照する注意機構）や AdaLN（Adaptive Layer Normalization、条件からスケール γ とシフト β を作って特徴を変調する DiT 由来の手法）で注入するのが定石です。なお t を一様分布ではなく、中間時刻を厚めにサンプリングする logit-normal 分布や、ノイズ寄りに偏らせた分布で引く工夫もよく使われます。速度場が学びにくい領域を重点的に学習させるためです。

## 手法の系譜と主要論文

- Chen, Rubanova, Bettencourt, Duvenaud (2018, NeurIPS)「Neural Ordinary Differential Equations」(arXiv:1806.07366): ニューラルネットでベクトル場を表し、ODE として連続変換を行う CNF の起点。理論的に美しいが学習が ODE シミュレーション依存で重く、CFM が解決すべき問題を定義した存在です。
- Sohl-Dickstein ら (2015, ICML) と Ho ら (2020, NeurIPS, [DDPM（拡散モデル基礎）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0230_ddpm-denoising-diffusion)) の拡散モデルが「ノイズを段階的に除く生成」の出発点。Song & Ermon のスコアマッチングと合わせ、後に連続時間の確率微分方程式（SDE）/ ODE として統一的に理解されました。CFM はこの連続時間の見方を「最尤推定なしのベクトル場回帰」へ純化したものです。
- Lipman, Chen, Ben-Hamu, Nickel, Le (2023, ICLR)「Flow Matching for Generative Modeling」(arXiv:2210.02747): 本トピックの中心。条件付き Flow Matching の目的関数と、周辺/条件付きベクトル場の勾配一致定理を提示。動機は従来 CNF の ODE シミュレーション依存を取り除くこと。拡散の確率パスだけでなく、OT に対応する直線パスも選べることを示し、直線パスのほうが生成が速く品質も良いと報告しました。
- Liu, Gong, Liu (2023, ICLR)「Flow Straight and Fast: Rectified Flow」(arXiv:2209.03003) と Albergo & Vanden-Eijnden (2023, ICLR)「Building Normalizing Flows with Stochastic Interpolants」(arXiv:2209.15571): ほぼ同時期に独立に「直線補間でベクトル場を学ぶ」考えに到達。三者は数学的にほぼ同じ枠組み（補間 + ベクトル場回帰）で、合わせて「flow matching / stochastic interpolant 系」と総称されます。詳細は [Rectified Flow](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0248_rectified-flow)。
- Tong, Fatras, Malkin ら (2024, TMLR)「Improving and Generalizing Flow-Based Generative Models with Minibatch Optimal Transport」(arXiv:2302.00482): CFM を一般化し、ミニバッチ内で最適輸送を解いて x_0 と x_1 を賢く対応づける OT-CFM を提案。ランダム対応よりパスがまっすぐになり少ステップ生成が改善します。
- Albergo, Boffi, Vanden-Eijnden (2023)「Stochastic Interpolants: A Unifying Framework」(arXiv:2303.08797): 補間に確率的な「橋」項を足し、決定論的 ODE と確率的 SDE のサンプラを連続的につなぐ統一理論を提示。flow matching と拡散を1つの枠で扱えるようにしました。
- Esser ら (Stable Diffusion 3, 2024, ICML)「Scaling Rectified Flow Transformers」(arXiv:2403.03206): flow matching を大規模テキスト→画像に適用し、学習時刻のサンプリング（logit-normal などの非一様スケジュール）が品質に効くことを大規模に検証。CFM が研究から実用の主流へ移った象徴です。
- Lipman ら (2024)「Flow Matching Guide and Code」(arXiv:2412.06264): flow matching の体系的な解説とリファレンス実装をまとめた総説。離散・多様体・確率的補間まで含む現在の到達点を整理しています。

## 論文の実験結果(定量データ)

評価指標の意味を先に押さえます。FID（Fréchet Inception Distance）は生成画像の分布と本物画像の分布の近さを測る指標で、Inception ネットの特徴空間で両分布をガウスと見なし距離を測ります。小さいほど本物に近い（良い）。NFE（Number of Function Evaluations）は1サンプル生成に必要なネットワーク呼び出し回数で、小さいほど速い。ODE ソルバの刻み数とほぼ一致します。NLL（負の対数尤度、bits/dim、1画素あたり何ビットで符号化できるか）はデータをモデルがどれだけよく説明できるかで、小さいほど良い。straightness（まっすぐさ）は生成軌跡が直線からどれだけ外れるかの量で、E∫|v(x_t,t) - (x_1 - x_0)|² dt として定義され、0 なら完全直線です。

- Lipman ら (2023): ImageNet 128×128 で、OT 直線パスを使う Flow Matching は拡散ベースの確率パスより少ない NFE で同等以下の FID に到達したと報告。同じネットワークでも、確率パス（拡散相当）より OT パスのほうが収束が速く、サンプリングの NFE を削っても FID の劣化が緩やかでした。CIFAR-10 では bits/dim でも競争力ある値を達成。アブレーション的な要点は「条件付きパスを拡散型から OT 直線型に変えるだけで、刻みを粗くしても品質が崩れにくい」という定量的裏付けです。
- Tong ら (2024): OT-CFM は素朴な CFM（独立サンプリング）に比べ、同じ NFE 設定での FID と straightness が改善。少 NFE（例えば NFE が一桁）の領域で差が大きく、ミニバッチ OT による対応づけが直線化に効くことを示しました。アブレーションでは「OT 対応づけを外して独立対応に戻すと straightness が悪化し、少ステップ FID が劣化する」ことが確認されています。低次元の合成データ（2D の8つのガウス→市松模様など）では、独立対応の生成軌跡が交差して曲がるのに対し OT-CFM の軌跡がほぼ平行な直線になる様子が可視化され、直感を裏づけています。
- Stable Diffusion 3 (Esser ら, 2024): flow matching + 非一様な学習時刻サンプリング（logit-normal で中間時刻を厚めにサンプル）が、一様サンプリングや拡散の損失重み（EDM, v-prediction など）より一貫して良い FID / CLIP スコア（テキストと画像の整合度）を与えたと大規模に報告。複数の定式化を同条件で比較したアブレーションで rectified flow + logit-normal が総合最良と結論し、モデルとデータをスケールするほど flow matching の優位が安定することを示しました。

これらの数値の意味するところは共通しています。「ノイズとデータを直線で結ぶ確率パスを選び、それをシミュレーションなしで回帰学習すると、拡散と同等以上の品質を、より少ない生成ステップで得られる」という点です。

## メリット・トレードオフ・限界

メリット
- 学習が単純で安定。中間点で速度を二乗誤差回帰するだけで、ODE シミュレーションも変分下限の導出もヤコビアン計算も不要。
- 生成が速い。直線パスを選べば数ステップ〜十数ステップで済み、拡散の数百ステップから桁違いに軽い。
- 確率パスの設計が自由。拡散相当の曲がったパスから OT 直線パスまで、補間の形を選べる一般枠組み。拡散は CFM の特殊例として包含される。
- マルチモーダルな出力を自然に表現でき（初期ノイズの引き方で複数の妥当な出力をサンプリングできる）、条件付け（テキスト、BEV、運動履歴）も素直に組み込める。

トレードオフ・限界
- ノイズとデータの素朴な独立対応では条件付きパス（直線）同士が空間内で交差し、その交差点で周辺ベクトル場が平均化されて生成パスが曲がる。完全な直線化には OT 対応づけ（OT-CFM）や reflow（[Rectified Flow](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0248_rectified-flow)）が必要。
- 1回の生成にランダム性（初期ノイズの引き方）が入るため、同条件でも出力が変わる。評価の再現には乱数シード固定が必要。
- 学習時刻のサンプリング分布（一様か、Beta か、logit-normal か）、補間の形、ガイダンス強度など調整余地のあるハイパーパラメータが残る。
- 未解決の課題として、条件付き生成における誘導（classifier-free guidance に相当する理論）の整備、離散データ・多様体上のデータへの拡張、超少ステップ（1〜2ステップ）での品質と多様性の両立、OT をミニバッチで近似することの誤差解析などが活発に研究されています。

## 発展トピック・研究の最前線
- 確率的補間（stochastic interpolant）として一般化され、決定論的 ODE だけでなく確率的 SDE サンプラとの橋渡しが整理されつつある（Albergo らの一連の研究）。同じ学習済みモデルで ODE と SDE を切り替え、品質と多様性をトレードオフできます。
- 多様体・離散空間への拡張（Riemannian Flow Matching、Discrete Flow Matching）。回転群 SO(3) 上の姿勢、文章やグラフなど非ユークリッド・離散データへの適用が進んでいます。
- ロボット・自動運転の行動生成への応用。Vision-Language-Action モデル（π0 など）が flow matching で連続行動を生成し、軌道プランナでも拡散デコーダの後継として採用が広がっています。マルチモーダルな未来を表しつつ少ステップで生成できる点が車載のリアルタイム制約に合います。
- 蒸留と組み合わせた超少ステップ化（[Rectified Flow](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0248_rectified-flow) の reflow、consistency model 系、mean flow）で、1〜2ステップ生成の品質を詰める研究。
- ガイダンス理論の整備。拡散の classifier-free guidance を flow matching へ移植し、条件遵守と多様性のバランスを制御する手法が提案されています。
