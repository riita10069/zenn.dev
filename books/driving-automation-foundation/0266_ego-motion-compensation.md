# 自車運動補償（ego-motion alignment）

## ひとことで言うと

過去のフレームで作った「鳥瞰図(真上から見た地図のような特徴表現)」は、その時の自車の位置・向きを中心に描かれています。自車は動いているので、過去の地図をそのまま今の地図に重ねると位置がずれます。自車運動補償とは、「過去の地図を、今の自車の位置・向きに合わせて回転・平行移動してから重ねる」前処理のことです。これをやって初めて、過去と現在で同じ静止物が地図上の同じ場所に揃い、時間融合(過去情報の活用)が正しく機能します。時系列BEVのほぼすべての手法が、融合方式の違いによらずこの補償を共通の土台として使っています。

## 直感的な理解

歩きながら地図を描く人を想像してください。5秒前に「自分の真正面5mに信号機」と描き、今また地図を描くと、5m進んだ今は信号機は「自分のすぐ目の前」に来ています。2枚の地図はどちらも「自分を中心」に描いているので、同じ信号機が違う場所に描かれます。さらにカーブを曲がっていれば、地図全体が回転もしています。

この2枚をそのまま重ねたら、1つの信号機が2か所に写って見えます。コンピュータも同じで、過去BEVと現在BEVを座標のずれを直さずに重ねると、同じ物が別々の場所に二重に現れ、モデルは「これは別物だ」と誤解します。時間融合(self-attentionやconcat)は「同じ物が同じ位置にいる」ことを暗黙に期待しているので、ずれたまま渡すとノイズを学習させることになり、むしろ性能が下がります。だから重ねる前に、過去の地図を「今の自分の視点」に揃え直す必要があります。これが自車運動補償です。揃え直すのに必要なのは「5秒前から今までに、自分がどれだけ前へ進み、どれだけ向きを変えたか」という自車の移動量だけです。

## 基礎: 前提となる概念

- 時間融合(temporal fusion): 過去フレームの情報を現在の予測に持ち込む処理。1枚の画像では分からない「動き」や「一時的な隠れ」を補うために行います。融合の方法には self-attention(自己注意)やチャネル結合(concat)、コストボリューム照合などがあります。

- 自車中心座標系(ego frame): BEV特徴は普通この座標系で表されます。地図の中心(0, 0)が常に自車の位置、地図の上方向が常に自車の進行方向です。「自車から見た相対的な地図」であり、自車が動けば座標系そのものが動きます。

- self-attention(自己注意): 同じデータの中で要素どうしを関連づけ、重みづけして混ぜ合わせる仕組み。時間融合では「過去BEVの各点」と「現在BEVの各点」を関連づけます。関連づけが正しく働くには、同じ物が同じ位置にいる必要があります。

- ego-motion(自車運動): 自車そのものの移動量。前進量 Δx、横移動量 Δy、向きの変化 Δθ などで表します。GPS、IMU(慣性計測装置。加速度と角速度を測るセンサ)、ホイールオドメトリ(車輪の回転量から進んだ距離を推定)などから得ます。これらを融合した推定結果を「ego-pose」と呼びます。

- 剛体変換と同次変換行列(rigid transformation / homogeneous matrix): 物の形を変えずに「回転 + 平行移動」だけ行う変換が剛体変換です。これを1つの行列でまとめて書いたのが同次変換行列です。回転 R と並進 t を1つの行列に詰め込み、点の座標に掛けるだけで変換できます。

- 微分可能サンプリング(grid_sample / STN): 格子上の特徴を任意の(小数を含む)座標から取り出す操作を、勾配が流れる形(微分可能)で行う仕組み。Spatial Transformer Networks(Jaderberg et al., 2015)が原典で、運動補償はこの操作の上に成り立ちます。

## 仕組みを詳しく

必要なのは「過去フレームの自車座標系」から「今の自車座標系」への座標変換です。段階で分解します。

ステップ1: フレーム間の自車の動きを求める。過去から今までに自車が「前に Δx、横に Δy、向きが Δθ 回転した」という量を、GPS/IMU/オドメトリから得ます。2次元の剛体変換は3×3の同次変換行列で書けます。

```
T = | cosθ  -sinθ   Δx |
    | sinθ   cosθ   Δy |
    |  0      0      1  |
```

この行列1つに「θ だけ回転して、(Δx, Δy) だけ平行移動する」操作がまとまっています。同次座標(点を (x, y, 1) と3要素で書く流儀)を使うと、回転と平行移動を1回の行列掛け算で同時に適用できるので便利です。実装では「過去フレームのworld→ego変換」と「現フレームのego→world変換」を合成して相対変換を作ります。

ステップ2: 過去BEVの各セルの座標を変換する。過去BEVは `[B, C, H, W]`(例 `[4, 256, 200, 200]`、Cはチャネル数、H×WはBEVグリッドの縦横)という格子状の特徴マップです。今の座標系の各格子点 (x, y) が、過去座標系ではどこ (x', y') だったかを、上の行列の逆変換で計算します(出力格子を基準に入力をどこから取るかを決める、いわゆる inverse warping)。

ステップ3: その位置から過去BEVを取り出す(リサンプリング)。(x', y') は格子の真上に乗るとは限らず、格子のあいだの小数座標になります。そこで周囲4点から双線形補間(bilinear interpolation, 近い4点から距離の比率で値を作る方法)で値を取り出します。深層学習フレームワークでは `grid_sample` という関数1つで「変換 + 補間」をまとめて行え、しかも微分可能なので学習で勾配が流れます。

ステップ4: 揃った過去BEVと現在BEVを融合する。これでようやく、同じ信号機が両方の地図で同じ位置に来ます。あとは self-attention やチャネル結合で融合します。

before / after で言うと、補償前は「過去の信号機が地図の (100, 80)、今の信号機が (100, 100)」と20セルずれています。BEVの解像度が 0.5 m/セルなら、これは実空間で10mのずれです。補償後は「過去も今も (100, 100)」に揃い、attention が同じ物体だと正しく認識できます。

重要な注意: これは「世界に対して静止している物(道路、建物、信号、車線)」を揃える処理です。動いている車などは自車の動きとは独立に動くので、運動補償だけでは完全には揃いません。補償で静止物のずれを消し、残った「動的物体のずれ」を attention やネットワークが学習で吸収する、という役割分担になります。むしろこの残差ずれこそが「物体がどっちに動いているか」の手がかりであり、速度推定に使われます。つまり補償は「動きの手がかりを消す」のではなく「静止背景という基準を揃えて、動きを純粋な残差として浮かび上がらせる」ための処理だと理解すると正確です。

## 手法の系譜と主要論文

自車運動補償は単独の論文というより、時系列センサ処理に共通する基礎技術です。代表的な文脈を挙げます。

- ICP / 点群レジストレーション(Besl & McKay, IEEE TPAMI 1992 ほか): LiDAR(レーザーで周囲の3D点群を測る装置)の世界では、異なる時刻のスキャンを ICP(Iterative Closest Point)で位置合わせするのが古くからの定石です。BEVの運動補償は「異なる時刻のセンサ観測を共通座標系へ揃える」というこの発想の、画像特徴版とみなせます。LiDARでは点群そのものを揃え、BEVでは特徴グリッドを揃える、という違いです。LiDARでは、自車が高速移動中に1スキャンする間の歪み(motion distortion)を補正する「de-skewing」も同じ ego-motion を使います。

- Spatial Transformer Networks (Jaderberg et al., NeurIPS 2015, arXiv:1506.02025): 特徴マップに対する空間変換を微分可能に行う一般枠組み。運動補償の grid_sample 実装はこの理論の直接の応用で、変換を固定値で与えるか学習で出すかが違うだけです。

- BEVDet4D (Huang & Huang, arXiv:2203.17054, 2022): カメラBEVで時間融合を実用化した初期の代表。現在と過去のBEV特徴を ego-motion で揃えてからチャネル方向に結合(concat)し、畳み込みで融合します。論文は、この「揃える」前処理によって速度推定の誤差が大きく下がることを定量的に示しました。揃えずに concat しても時間情報を活かせない、という点が強調されています。詳細は [BEVDet4D型 特徴キュー連結](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0480_bevdet4d-queue-concat)。

- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270): 直前フレームのBEVを保持し、現在のBEVクエリと temporal self-attention で融合します。その前段で、過去BEVを現在の ego frame に揃えてから attention をかけます。運動補償なしでは時間 self-attention がそもそも成り立たない、という前提が置かれています。詳細は [BEVFormer型 再帰的BEV融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0481_bevformer-recurrent-fusion)。

- SOLOFusion (Park et al., ICLR 2023, arXiv:2210.02443): 時間ステレオで深度を当てる手法。過去フレームを各深度候補で現フレームへ投影する際、ego-motion を使います。ここでは運動補償が「2視点のベースラインを定義する」役割を担い、移動量がずれると深度照合そのものが崩れます。詳細は [SOLOFusion型 時間ステレオ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0500_solofusion-temporal-stereo)。

- StreamPETR (Wang et al., ICCV 2023, arXiv:2303.11926): 密マップでなく物体クエリを伝播する手法でも、過去クエリの位置を ego-motion で現フレーム座標へ移し、さらに運動を正規化に組み込みます。グリッドでもクエリでも、揃える必要は共通であることを示す例です。詳細は [StreamPETR型 オブジェクト中心伝播](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0503_streampetr-object-propagation)。

これらに共通するのは「時間融合のどの方式(attention融合・concat融合・ステレオ深度・クエリ伝播)を選んでも、過去を現在の座標系に揃える前処理は前提になる」という点です。揃え方が違うだけで、揃える必要そのものは普遍的です。

## 論文の実験結果(定量データ)

自車運動補償は単体で評価されるより、「あり/なし」のアブレーションで効果が示されます。代表的な知見を、指標の意味とともに説明します。

- mAVE(mean Average Velocity Error, 速度誤差、m/s、小さいほど良い): BEVDet4D系の報告では、過去BEVを ego-motion で揃えてから融合すると、揃えない場合に比べて速度誤差が大きく(報告例ではおよそ半減レベルで)改善します。揃えないと「静止物のずれ」と「動的物体のずれ」が混ざってしまい、速度の手がかりが汚れるためです。揃えることで、残差ずれが純粋に物体の動きを反映し、速度推定が安定します。

- mAP / NDS(検出の正確さ・総合スコア、大きいほど良い): 運動補償を入れると、時間融合による検出精度向上がフルに得られます。補償なしの時間融合は、二重像によるノイズで、むしろ単一フレームより性能が伸びにくい(場合によっては劣化する)ことが報告されています。「時間融合 = 補償 + 融合」であり、補償を抜くと融合の効果が損なわれる、というのがアブレーションの一貫した結論です。

- 補間による劣化: grid_sample の双線形補間は高周波の情報(細かいエッジ)をわずかに鈍らせます。再帰的に毎フレーム補間を繰り返すと、長期では特徴がぼやける傾向があり、長期再帰融合([VideoBEV型 長期再帰融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0511_videobev-longterm-fusion))で整合精度が重要になる一因です。

注意として、効果の大きさは ego-motion の取得精度に依存します。高精度なGPS/IMUが使える設定では補償が綺麗に効き、オドメトリだけでドリフト(誤差の蓄積)が大きい設定では補償後にも残差が残ります。nuScenes のように高精度な ego-pose がアノテーションされたデータでは補償の効果が綺麗に出ますが、安価なセンサ構成では効果が割り引かれます。

## メリット・トレードオフ・限界

メリット:

- 静止物が時間方向に正しく重なるので、self-attention や concat 融合が正しく機能する。
- 物体の動きや遮蔽の履歴を、座標ずれに邪魔されず学習できる。残差ずれが速度の手がかりになる。
- 剛体変換という確立された理論に基づくので挙動が予測しやすく、追加の学習パラメータがほぼ不要。軽量で、どの時間融合方式にも差し込める。

トレードオフ・限界:

- 正確な自車移動量(ego-motion)が必要。GPS/IMU/オドメトリのドリフトや誤差が、そのまま補償のずれになる。
- 動的物体は自車運動だけでは揃わない。残差はモデルが吸収する必要があり、混雑シーンや高速で交差する物体では負担が増える。
- grid_sample の補間で特徴がわずかに鈍る(高周波情報の劣化)。長期で繰り返すと累積する。
- フレーム間隔が長い(自車が大きく動く)と、過去BEVのうち現在の視野外へ出た領域は使えず捨てるしかなく、有効情報が減る。地面が平らでない(坂・段差)場合、2D剛体変換の前提が崩れて高さ方向にずれが残る。

研究上の未解決課題としては、(1)ego-motion 自体をネットワークで推定し補償と同時最適化すること(ラベルやセンサのドリフトに頑健にする)、(2)補間劣化を避ける高精度な特徴リサンプリング、(3)静的・動的を分離して補償する(静的は剛体変換、動的はフロー推定で別々に揃える)アプローチ、(4)3D(高さ方向や非平面)を考慮した補償などが挙げられます。

## 発展トピック・研究の最前線

自車運動補償は、時系列BEV知覚のほぼすべての手法に埋め込まれた「縁の下の力持ち」です。研究の最前線では、補償を陽な前処理として固定するのではなく、学習可能な変形場(deformable な整合)として柔らかく扱う方向や、静的背景と動的物体を分離してそれぞれ別の運動モデルで揃える(scene flow 推定と組み合わせる)方向が探られています。また、世界モデル系では「自車運動を条件として未来のBEVを予測する」ために、補償の逆操作(現在から未来への ego-motion 適用)が使われ、補償の理解は予測・計画とも地続きになっています。LiDARの点群レジストレーションとカメラBEVの運動補償を統一的に扱うマルチモーダル整合、さらに ego-motion をオンラインで自己推定する自己教師あり手法も活発なテーマです。
