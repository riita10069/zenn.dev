# BLIP-2 (Q-Former)

## ひとことで言うと
BLIP-2 は「すでに賢い画像エンコーダ」と「すでに賢い大規模言語モデル (LLM、大量の文章で学習させた言語モデル)」を、両方とも凍結したまま (重みを再学習せず固定したまま)、その間に小さな橋渡しモジュール「Q-Former」だけを挟んで学習する手法です。橋だけ学べばよいので、巨大な部品を作り直すより圧倒的に安く「画像を見て言葉で答えるモデル」が作れます。VLM の歴史で「凍結部品+軽い橋」という実用的なレシピを確立した、転換点となる仕事です。

## 直感的な理解
画像を見て言葉で答えるモデル (VLM) を一から作るのは非常に高コストです。視覚エンコーダ (画像をベクトルに変える部品) も LLM もそれぞれ巨大で、両方を end-to-end (端から端まで全部いっぺん) に学習すると、必要な計算資源とデータが膨大になります。

一方で世の中には、画像をよく理解する視覚エンコーダ (CLIP の ViT など、[CLIP (画像-言語対比学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0513_vlm-clip) 参照) も、言葉をよく操る LLM (OPT, FlanT5 など) も、それぞれ別々にすでに存在します。問題は「この2つは別々の場所で別々に育ったので、共通の言語 (表現形式) を持っていない」ことです。例えるなら、優秀な写真家と優秀な作家がいるが、お互いの言葉が通じない。間に「通訳」を1人雇うのが BLIP-2 の発想です。写真家 (視覚エンコーダ) も作家 (LLM) も再教育せず、通訳 (Q-Former) だけを育てます。

さらに難所が「モダリティギャップ (modality gap、画像という形式と言葉という形式の隔たり)」です。画像特徴は数百〜数千個のパッチ特徴という大量・冗長な情報で、LLM が扱う言葉のトークン (語の単位) とは性質が違います。これをそのまま LLM に流すと量が多すぎ、意味もかみ合いません。BLIP-2 はこのギャップを少数の「通訳トークン」で埋めます。

なぜ凍結が嬉しいのか、もう一段かみ砕きます。LLM を画像つきで丸ごと再学習すると、せっかく言語で身につけた膨大な知識が壊れる (破滅的忘却、catastrophic forgetting と呼ぶ) リスクがあり、しかも数十億〜数百億の重みを更新するメモリと計算が要ります。凍結すれば、その知識を温存したまま、橋を通じて画像情報を「言葉のように」差し込めます。

## 基礎: 前提となる概念
- 凍結 (frozen): 学習中にそのモジュールの重みを一切更新しないこと。勾配 (重みをどちらへ動かせば損失が減るかの情報) を流さないので計算もメモリも節約でき、元の能力を保てます。
- クロスアテンション (cross-attention): あるトークン列 (query、問い合わせ側) が、別のトークン列 (key/value、参照される側) を参照して情報を取り込む注意機構。Q-Former では「クエリトークン」が「画像パッチ特徴」を参照します。注意機構とは「どこをどれだけ見るか」を重みで表す仕組みです。
- ソフトプロンプト (soft prompt): 自然言語の単語ではなく、連続値ベクトルとして LLM に与える「指示」。Q-Former の出力は、画像内容を表すソフトプロンプトとして LLM 先頭に差し込まれます。単語に対応しない「画像の意味を込めた仮想トークン」と考えるとよいです。
- ITC / ITM / ITG: 後述する3つの学習目的の略。Image-Text Contrastive (対比)、Image-Text Matching (一致判定)、Image-grounded Text Generation (画像条件つき生成)。
- アテンションマスク (attention mask): どのトークンがどのトークンを見てよいかを制御する0/1の表。同じネットワークでも、マスクを切り替えると「対比向き」「生成向き」など異なる振る舞いを学ばせられます。

## 仕組みを詳しく
中心は Q-Former (Querying Transformer、問い合わせ型 Transformer) という小さなモジュールです。

1. 凍結した画像エンコーダ (例: CLIP/EVA-CLIP ViT-g) が画像を [257, 1408] のようなパッチ特徴に変換する (257 = パッチ数+CLS トークン、1408 = 次元)。この大量の特徴をそのまま LLM に入れたくない。
2. Q-Former は「学習可能なクエリトークン」を例えば32個だけ持つ。形状 [32, 768]。これは「画像から何を抜き出すか」を学ぶ質問係。最初はランダムな値で、学習を通じて「言葉にしやすい要点を抜く」よう育ちます。
3. この32個のクエリが、クロスアテンションで画像のパッチ特徴を覗き込み、画像の要点を32本のベクトルに凝縮する。出力は [32, 768]。
4. この32本を線形層で LLM の入力次元 (例: 2560) に射影し、LLM の文章トークンの前にソフトプロンプトとして差し込む。LLM はそれを言葉の続きとして読み、回答を生成する。

つまり 257個の冗長な画像特徴 → 32個の濃縮トークンへ、Q-Former が情報を絞り、かつ LLM が読める形へ変換しています。トークン数を 257 → 32 に減らすことは、後段 LLM の計算量 (自己注意の計算は系列長の2乗で効く) を大きく下げる効果もあります。例えば後段に画像由来トークンが32個か576個かで、長文を扱うときの計算負荷が桁違いになります。

学習は2段階です。

第1段階 (Vision-Language 表現学習): 画像エンコーダと Q-Former だけを結び、3つの目的を同時に学ぶ。
- ITC (画像-テキスト対比): 正しい画像-文章を引き寄せ、間違った組を引き離す ([CLIP (画像-言語対比学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0513_vlm-clip) と同じ発想)。このときクエリとテキストは互いを見られないマスクにし、画像由来のクエリ表現とテキスト表現を別々に作って比較する。
- ITM (一致判定): 画像と文章が合っているか否かを2値分類する。クエリとテキストが相互に見えるアテンションを使い、細かい整合性まで判定させる。
- ITG (画像条件つき生成): クエリに込めた画像情報をもとにキャプションを生成する。この時クエリはテキストを見られないアテンションマスクにして、「画像の情報だけで文を作る」よう強制する。これによりクエリが「文を作るのに十分な画像情報」を抱え込むようになる。

3つの目的でアテンションマスクを巧みに切り替えることで、同じ Q-Former が「対比」「判定」「生成」のすべてに使える表現を学びます。これにより、クエリが「言葉に変換しやすい画像の要点」を抜くようになります。これは前身 BLIP のマルチタスク思想を、橋の中に凝縮した設計です。

第2段階 (生成学習): Q-Former の出力を凍結 LLM に繋ぎ、LLM が正しい文章を生成するよう Q-Former (と射影層) だけを学習する。LLM も画像エンコーダも凍結。第1段階で「言語に揃った画像表現」を先に作っておくことが、この接続をスムーズにします。

学習されるのは Q-Former (と射影層) だけで、これは約1.9億パラメータ程度。視覚エンコーダ (10億級) も LLM (数十億〜数百億級) も触らないため、学習コストが劇的に小さくなります。

具体例 (運転シーン): 交差点の画像を Q-Former が32トークンに圧縮 → 凍結 LLM (FlanT5) がそれを文脈に「信号は黄色で、左から歩行者が横断中です。減速して停止に備えるべきです」と生成。橋だけ学べば、LLM の言語知識をそのまま使って状況説明ができます。

## 手法の系譜と主要論文
- Frozen (Tsimpoukelli et al., NeurIPS 2021, DeepMind, arXiv:2106.13884): 「凍結 LLM に画像特徴をソフトプロンプトとして差し込む」発想の先駆け。視覚エンコーダだけ学習し LLM は凍結。few-shot (少数例) でマルチモーダルに振る舞えると示した。BLIP-2 の「凍結 LLM + 学習する橋」の直接の祖先です。
- BLIP (Li et al., ICML 2022, Salesforce, arXiv:2201.12086): BLIP-2 の前身。Web のノイジーなキャプションをモデル自身で作り直し浄化する CapFilt (captioner で説明文を生成し直し、filter で質の悪いものを除く) を提案。動機は「Web キャプションは質が低い」。理解・生成の両タスクを底上げした。
- Flamingo (Alayrac et al., NeurIPS 2022, DeepMind, arXiv:2204.14198): 凍結 LLM に Perceiver Resampler (大量の画像特徴を固定数のトークンに要約する部品) と gated cross-attention 層を差し込み、画像と文章を交互に入力できる VLM を提案。動機は「少数例 (few-shot) で新タスクに適応できる汎用 VLM を作る」。few-shot で多数のベンチを更新したが、80B と巨大で学習・推論が重い。BLIP-2 はこの「凍結部品を活用」の流れを、より軽い Q-Former で再構成したと位置づけられます。
- BLIP-2 (Li et al., "BLIP-2: Bootstrapping Language-Image Pre-training with Frozen Image Encoders and Large Language Models", ICML 2023, Salesforce, arXiv:2301.12597): 凍結エンコーダ+凍結 LLM+学習する Q-Former という構成を提案。動機は「巨大部品を再学習せず、橋だけ学べば安く高性能な VLM が作れる」。新規性は2段階の Q-Former 学習でモダリティギャップを埋めた点。
- InstructBLIP (Dai et al., NeurIPS 2023, Salesforce, arXiv:2305.06500): Q-Former を「指示に応じて画像特徴を抜く (instruction-aware)」よう拡張し、26 のデータセットを指示形式に統一して学習。動機は「橋自体も指示を見て抜き方を変えるべき」。クエリだけでなく指示テキストも Q-Former に入れて、質問に応じた特徴抽出を可能にしました。これは [LLaVA (Visual Instruction Tuning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0515_vlm-llava) (単純な線形/MLP 橋+指示データ) と対をなす設計思想で、「橋を賢くする派」と「橋を単純にしてデータで勝負する派」が同時期に並走しました。

## 論文の実験結果 (定量データ)
- ゼロショット VQAv2: BLIP-2 (ViT-g + FlanT5XXL) はゼロショット VQAv2 でおよそ 65.0% を達成。VQA (Visual Question Answering、画像への質問応答) の精度は「人間の回答と一致した割合」で、画像を見て質問に正しく答えられた率です。これを Flamingo80B (約 56.3%) と比べると、約 8.7 ポイント高い。注目すべきは学習可能パラメータが Flamingo の約 1/54 (BLIP-2 は約 1.9 億 vs Flamingo は約 100 億超) しかない点で、「凍結部品+軽い橋」が効率の点で圧勝したことを示します。
- キャプション/検索: NoCaps や COCO キャプションでも当時の上位水準を示し、画像-テキスト検索でも BLIP 比で改善しました。第1段階で対比・一致・生成を同時に学んだ表現が、検索 (対比由来) と生成 (生成由来) の両方に効くことを裏づけます。
- 学習効率: 視覚エンコーダと LLM を凍結するため、学習対象は Q-Former の約 1.9 億パラメータのみ。end-to-end で全体を学習する従来法に比べ、必要な計算資源とデータが大幅に減りました。論文は同等以上の性能をはるかに少ない学習可能パラメータで達成したと強調しています。
- アブレーション (第1段階の有無): 第1段階 (表現学習) を省いて第2段階だけで学習すると、生成性能が明確に劣化し、学習も不安定になることが報告されています。これは「Q-Former がまず言語に揃った画像表現を学ぶ」工程が、後段の LLM 接続に不可欠であることを示します。逆に言えば、橋を単に挟むだけでは不十分で、橋を言語空間に揃える事前学習が肝心だという教訓です。
- LLM の選択: 凍結 LLM に OPT 系 (素の言語モデル) を使うと素直なキャプション生成、FlanT5 系 (指示チューニング済み) を使うと指示追従や推論的な質問応答が伸びる、という傾向が示されました。橋は同じでも、繋ぐ LLM の性質がそのまま VLM の性格に出ます。これは「VLM の言語能力は接続先 LLM に強く規定される」という、その後も繰り返し確認される一般則の早い実証例です。

## メリット・トレードオフ・限界
- メリット: 視覚エンコーダと LLM を凍結するため学習が安く速い。既存の強力な部品を再利用でき、LLM の言語知識を破壊せずに済む。
- メリット: 画像特徴を32トークンに圧縮するので、LLM へ渡す入力が短く推論も軽い。長文対話でも画像分の負荷が一定。
- メリット: 橋 (Q-Former) が能動的に要点を抜くので、単純な特徴流し込みより情報整理が効く。
- トレードオフ: Q-Former が抜けなかった細部は LLM に伝わらない。32トークンというボトルネック (情報の通り道の狭さ) が、細かい読み取りや計数の弱さの原因になりやすい。「画面端の小さな標識の文字」のような情報は32本に押し込む過程で失われがちです。
- トレードオフ: 凍結 LLM の知識・癖・バイアスをそのまま引き継ぐ。安全用途では幻覚 (hallucination、画像にない物をもっともらしく述べる現象) に注意。
- 限界・未解決: 2段階学習でレシピが複雑になり再現の手間が増える。また、その後の研究 ([LLaVA (Visual Instruction Tuning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0515_vlm-llava) など) は「Q-Former のような複雑な橋より、線形/MLP の単純な橋を大量のデータで学ぶ方が高解像度や細部に強い」と示し、Q-Former 型は次第に主流から外れていきました。圧縮の効率と細部保持のトレードオフは依然として設計上の論点です。

## 発展トピック・研究の最前線
- 橋のリサンプラ系譜: Q-Former、Perceiver Resampler (Flamingo)、近年の MLP コネクタ (LLaVA) は「画像トークンをどれだけ・どう圧縮するか」という同じ問いへの異なる答えです。圧縮しすぎると細部を失い、圧縮しないと推論が重い。このトレードオフの最適点は解像度や用途で変わります。動画など極端にトークンが多い設定では、再び Q-Former 型の圧縮が見直される動きもあります。
- instruction-aware な特徴抽出: InstructBLIP の「質問に応じて見るべき画像領域を変える」発想は、後の VLM が画像トークンと指示を早期に相互作用させる設計 (early fusion、初期段階で混ぜる) へ繋がりました。
- 凍結か微調整か: BLIP-2 は「凍結」を売りにしましたが、その後は計算が許せば視覚エンコーダも一緒に微調整する方が性能が高い、という揺り戻しもあり、「どこを凍結しどこを学ぶか」は今も実務上の重要判断です。データ量が少ない領域 (運転・医療) では凍結が安全側、潤沢なら微調整が高性能、という使い分けが一般的です。
- トークン圧縮の再評価: LLaVA 系の長い画像トークンが推論コストを押し上げる中、学習可能なクエリで再要約する Q-Former 的アプローチが効率化文脈で再注目されています。歴史は単線でなく、用途ごとに最適点が揺れ動いています。
