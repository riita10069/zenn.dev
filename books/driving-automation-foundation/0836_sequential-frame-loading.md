# 逐次フレーム読み込み (sequential loading)

## ひとことで言うと

逐次フレーム読み込みとは、連続した運転記録の中から「ある一瞬」を中心に据え、その前の数十フレーム(過去)と後ろの数十フレーム(未来)を窓のように切り出して、1つの学習サンプルにする処理です。例えば 10 Hz(1秒10フレーム)で過去64フレーム+未来64フレームを取ると、過去6.4秒と未来6.4秒のひとまとまりになります。過去は「これまでどう動いてきたか(履歴=モデルの入力)」、未来は「これからどう動くべきか(教師信号=正解)」として使います。時系列を扱う運転・予測モデルのデータ準備に共通する中核手法です。

## 直感的な理解

自動運転モデルに「今この瞬間の画像」を1枚だけ見せて運転させようとすると、うまくいきません。運転は時間的な文脈に強く依存するからです。具体例で考えます。

- まったく同じ前方画像でも、直前まで加速していたのか減速していたのかで、次にすべきアクセル/ブレーキ操作は変わります。1枚の画像にはこの「直前の流れ」が写っていません。
- 前方の車が「止まりかけ」なのか「動き出し」なのかは、複数フレームにわたる位置変化を見ないと区別できません。静止画では両者が同じに見えます。
- 自分が今カーブの入口なのか出口なのかは、これまでたどってきた軌跡(履歴)から判断できます。

つまりモデルには「過去の文脈」を時系列として与える必要があります。一方で学習には「正解」も要ります。人間の運転をまねる模倣学習(imitation learning)では、「この瞬間から先、人間ドライバーが実際にどう動いたか」が正解になります。これが未来側のフレームです。だから「今この瞬間」を中心に、過去と未来を両方とも一定長で切り出すのです。それを行うのが逐次フレーム読み込みです。

## 基礎: 前提となる概念

用語を噛み砕きます。
- フレーム: ある一瞬のデータ1枚分。記録レートが 10 Hz なら 0.1 秒ごとに1フレーム。フレームには画像・速度・自車位置など、その瞬間の全情報が紐づく。
- エピソード(またはクリップ): 1回の連続した記録。途中で途切れない一続きのドライブ。長さ T フレーム。
- 履歴(history): 学習サンプルの「入力」になる過去側の系列。
- 教師信号/ターゲット(target, ground truth): 学習サンプルの「正解」になる未来側の系列。ground truth(グラウンドトゥルース)は「絶対的に正しいとみなす答え」という意味の用語。
- 模倣学習: 専門家(ここでは人間ドライバー)の行動データを教師にして、その振る舞いを再現する方策(policy、状況に応じて行動を決める関数)を学ぶ枠組み。「人間がした操作=正解」とみなす。
- スライディングウィンドウ(sliding window): 一定長の窓を時系列上で少しずつずらしながら、各位置で切り出す手法。動画処理や時系列学習の定番。
- データ拡張(data augmentation): 同じ素材から少しずつ違うサンプルを作り、データ量と多様性を水増しして過学習を抑える技法。

過学習(overfitting)とは、モデルが訓練データの細部を丸暗記してしまい、新しいデータでは性能が出ない状態です。例えるなら、過去問の答えだけを暗記して試験本番(初見の問題)で解けない状態。時系列の窓切り出しでは、後述するサンプル間相関がこれを招きやすいので注意が要ります。

## 仕組みを詳しく

中心となる時刻のインデックスを `sample_idx` とします。過去 H ステップ、未来 F ステップを取るとして、典型的な値は H = F = 64、レート 10 Hz です。

```
H (history)  = 64   # 過去6.4秒 (10Hz)
F (future)   = 64   # 未来6.4秒 (10Hz)
最低必要フレーム数 = H + F + 1 = 129   # 1サンプルを作るのに要る連続フレーム
```

ステップ分解:

1. エピソード(1回の連続記録)の全フレームを用意する。長さ T フレーム。
2. 「有効な中心位置」の範囲を決める。前に H フレーム、後ろに F フレーム必要なので、中心は `H <= sample_idx <= T - F - 1` の範囲に限られる。両端ぎりぎりのフレームは中心にできない(過去か未来が足りないため)。
3. 各有効中心位置について、過去 `[sample_idx-H : sample_idx]`(H フレーム)と未来 `[sample_idx+1 : sample_idx+1+F]`(F フレーム)を切り出す。
4. 過去側からは「履歴」を、未来側からは「教師信号(目標軌道)」を作る。

ASCII で窓のイメージ(エピソード上を窓がスライドしていく):

```
エピソードのフレーム列(長さT):
[0][1][2] ............................................ [T-1]
              |<-- 過去H=64 -->|  ^  |<-- 未来F=64 -->|
                                sample_idx(現在)
sample_idx を H から T-F-1 まで1ずつずらして、サンプルを量産する
```

サンプルを「フレームごと」に作ると、1エピソードから多数のサンプルが取れます。擬似コードはこうなります。

```python
min_frame = H                  # 64
max_frame = T - F - 1          # T - 65
for frame_idx in range(min_frame, max_frame + 1):
    samples.append((episode_idx, episode_start + frame_idx))
```

つまり長さ T のエピソードからは `T - (H + F)` 個のサンプルが生まれます(H=F=64、T=200 なら 72 個)。窓は1フレームずつずらすので、隣り合うサンプルは大きく重なります(オーバーラップ)。これは学習データを増やす効果がある一方、近いサンプル同士は相関が高いという注意点もあります(後述)。

切り出した中身の典型的な使われ方とテンソル形状:
- 過去側の車両運動 → `egomotion_history`。例えば 64 ステップ × 4 信号(速度・加速度・ヨーレート・曲率)= shape (64, 4) または平坦化して (256,)。各信号の作り方は [ポーズからのego運動算出 (finite differencing)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0812_egomotion-finite-differencing) を参照。
- 未来側の車両運動 → `trajectory_target`。例えば 64 ステップ × 2 信号(加速度・曲率)= (64, 2) または (128,)。これが模倣学習の正解。
- 現在時刻のマルチカメラ画像 → 例えば shape (カメラ数, 3, 256, 256) のテンソル。先頭が画像枚数、3 が RGB チャンネル、後ろ2つが画素の縦横。

設計上の重要点は「両側とも常に完全に埋まる(パディング不要)」ことです。エピソード両端のフレームを最初から中心候補にしないので、足りない分をゼロで埋める処理が要りません。これは実装をシンプルにし、ゼロパディングが学習に与える悪影響(存在しないデータをモデルに見せてしまい、「停止していた」と誤学習する)も避けられます。

データ拡張として「サンプル時刻をランダムに選ぶ」手法もよく使われます。学習時に `sample_idx` を有効範囲内でランダムに振ると、同じクリップから時間的に少しずれたサンプルを取れ、過学習を抑えられます。決定的に全フレームを使うか、ランダムサンプリングするかは、データ量と計算予算で選びます。データが潤沢なら全フレーム、限られた計算予算でエポックを回したいならランダム、というのが目安です。

複数のセンサーソース(例えばカメラと自車ポーズ)で系列長がずれることがあります(カメラは 10 Hz だがポーズは 20 Hz、あるいは記録開始のずれ)。その場合は「短い方に合わせる」のが安全です。`usable = min(カメラ長, ポーズ長)` として、窓がどちらのソースでも範囲外に出ないように有効中心範囲を決めます。これを怠ると、片方のソースで添字が範囲外になりクラッシュしたり、ずれた時刻のペアを学習させてしまいます。

## 手法の系譜と主要論文

時系列の窓で過去文脈と未来軌道を作る発想は、end-to-end 運転(センサー入力から運転操作まで一気通貫で学ぶ枠組み)と軌道予測の双方で標準的な作法として確立してきました。

- ChauffeurNet(Bansal, Krizhevsky, Ogale; Waymo, RSS 2019, "ChauffeurNet: Learning to Drive by Imitating the Best and Synthesizing the Worst")。過去の状態系列から未来の軌道(航跡点列)を回帰する模倣学習を提案。動機は、1フレームだけでは運動の文脈が足りず安定した運転方策を学べないこと。過去履歴を入れることで滑らかで一貫した軌道を生成できた一方、模倣学習は人間データの分布に強く依存し、まれな状況(分布外, out-of-distribution、訓練データにほとんど出てこない状況)に弱いという根本的トレードオフを抱えます。同論文はこれに対し「最悪のケースを合成して学ばせる」摂動データ拡張(教師軌道を意図的にずらして立て直し方を学ばせる)を導入しました。
- 軌道予測の系譜では、VectorNet(Gao ほか, Waymo, CVPR 2020)と Trajectron++(Salzmann ほか, ECCV 2020)が「過去 H ステップ → 未来 F ステップ」という入出力設計を採用しています。狙いは過去のエージェント運動から確率的な未来軌道を予測すること。過去窓を長く取るほど文脈が増え予測が安定する一方、計算量が増え、古すぎる過去はかえってノイズになります。過去・未来を有限長に区切るのは、このバランスを取った設計選択です。
- 動画認識一般でも、連続フレームを固定長クリップに切る「クリップサンプリング」は標準で、I3D(Carreira & Zisserman, CVPR 2017, "Quo Vadis, Action Recognition?")などが採用しています。運転データへの適用が本トピックで、さらに「未来側を教師信号にする」点が運転・予測に特有です。動画認識ではクリップは分類ラベル1個(走っている/歩いている等)に対応しますが、運転では未来そのものが密な回帰ターゲット(各時刻の連続値の系列)になります。

## 論文の実験結果(定量データ)

各手法は、窓設計(履歴長・未来長)が性能に効くことを実験で示しています。値は各論文・関連報告に基づくおよその目安です。

- 軌道予測の標準評価指標は ADE(Average Displacement Error, 予測軌道と正解軌道の各時刻の距離の平均)と FDE(Final Displacement Error, 最終時刻での距離)です。いずれもメートルで、小さいほど良い。これらは「予測がどれだけ正解からずれたか」を測るので、ADE/FDE の改善はそのまま予測精度の向上を意味します。例えば6秒先予測で FDE が3m から2m に減れば、交差点で隣の車線にはみ出すかどうかの差に相当します。
- Trajectron++(ECCV 2020)は nuScenes と pedestrian データセット(ETH/UCY)で、過去の運動履歴とエージェント間相互作用を取り込むことで、当時のベースライン比で予測誤差を大きく下げたことを報告しています。アブレーション(構成要素を1つずつ抜いて効果を測る実験)では、過去履歴を取り除く(現在状態だけにする)と FDE が悪化することが示され、履歴窓の寄与が定量的に確認されています。
- VectorNet(CVPR 2020)は Argoverse 予測ベンチマークで、過去軌跡をベクトル(折れ線)として符号化し、ラスタ化(画像化)するベースラインより少ない計算量(報告ではメモリと FLOPs を大幅削減)で同等以上の精度を達成したと報告しています。ここでも入力の過去窓長は性能に効くハイパーパラメータとして扱われます。
- ChauffeurNet(RSS 2019)は閉ループ評価(実際にモデルで運転させて、その結果がさらに次の入力になる評価。誤差が蓄積するため厳しい)で、過去履歴ありの方が軌道の滑らかさと安定性が高いことを示しました。さらに摂動拡張を抜くアブレーションでは、ドリフト(少しずつ車線中心からずれる現象)からの復帰が悪化し脱輪が増えることが報告されています。これは「人間データの分布だけでは復帰挙動を学べない」という模倣学習の限界を定量的に裏づけた結果です。なぜなら人間は普通ドリフトしないので、復帰のデータがほとんど無いからです。

これらに共通する教訓は、(a) 過去窓を持たせると精度・安定性が上がる、(b) しかし長くしすぎると計算が増え寄与が頭打ちになる、(c) 未来を教師にする模倣学習は分布外に弱く、データ拡張や摂動が要る、という3点です。

## メリット・トレードオフ・限界

メリット
- 過去文脈と未来正解を同時に作れ、模倣学習の入出力がきれいに揃う。
- 両端を除外することでパディング不要、欠損のない固定長サンプルになる。実装が単純で学習も安定する。
- 窓を1フレームずつずらすので、1エピソードから大量のサンプルが取れる(データ効率が良い)。
- 窓設計(H, F, レート)をデータセット間で共通化すれば、入出力次元が揃い、ローダーを差し替えてもモデルを無改変で使える。

トレードオフ・限界
- 隣り合うサンプルが大きく重なるため、サンプル同士の相関が高い。シャッフルが不十分だと似たサンプルが固まり、ミニバッチ(一度に学習する小さな塊)の勾配が偏って学習が不安定になる。エピソード単位のシャッフルや、ストライド(窓のずらし幅)を空けるサンプリングが対策になる。
- 窓長が固定(例 6.4 秒)なので、それより長い時間スケールの文脈(数十秒先の合流判断など)は捉えられない。
- エピソードが最低フレーム数(H+F+1)未満だと1サンプルも作れず捨てられる。短いクリップが多いデータでは無駄が出る。
- 未来を教師信号にする模倣学習なので、人間の運転が最適とは限らないことがそのままモデル性能の上限になる。分布外の場面(事故寸前の復帰など)はデータにほとんど含まれず学べない。これは ChauffeurNet が摂動拡張で、近年の研究が強化学習やシミュレーションで補おうとしている未解決領域です。

## 発展トピック・研究の最前線

近年は、固定長窓を超える方向の研究が進んでいます。可変長やマルチスケールの履歴を Transformer の自己注意(系列内の各要素が他のどの要素を重視するかを学習する仕組み)で扱い、関連する過去だけを選択的に参照する手法が広がりました。また未来側を単一の正解軌道ではなく複数モード(右折・直進など分岐しうる未来)として確率的に出すマルチモーダル予測が標準になりつつあります。指標も minADE/minFDE(複数予測のうち最良のものとの誤差)へ拡張されています。模倣学習の分布外問題に対しては、シミュレータ上での閉ループ学習や、専門家データへの摂動・反実仮想(counterfactual、起きなかったが起こり得た状況)合成が活発です。窓サンプリング自体も、単純な等間隔ではなく、難しい場面(急ブレーキ・割り込み)を重点的にサンプルするカリキュラム/重み付けサンプリングが効果を上げています。
