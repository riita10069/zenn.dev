# LLaVA (Visual Instruction Tuning)

## ひとことで言うと
LLaVA (Large Language and Vision Assistant) は「画像を見ながら人間の指示に従って答える」ように LLM を訓練したモデルです。画像エンコーダと LLM を、ごく単純な1〜2層の変換だけで繋ぎ、「画像つきの指示と理想的な回答」のデータで教え込むことで、対話的に画像について説明・推論できる VLM を作ります。構造の単純さと再現の容易さから、研究コミュニティの事実上の標準ベースライン (基準となる比較対象) になりました。

## 直感的な理解
[CLIP (画像-言語対比学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0513_vlm-clip) や [BLIP-2 (Q-Former)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0512_vlm-blip2) のような VLM は「画像を分類する」「短いキャプションを出す」ことはできても、「この画像を見て、なぜこの車は止まるべきかを段階的に説明して」のような、人間の自由な指示に対話的に従う能力 (instruction following、指示追従) が弱いという課題がありました。

テキストのみの世界では、GPT 系の LLM が「指示チューニング (instruction tuning、人間の指示と模範回答のペアで微調整すること)」によって飛躍的に使いやすくなりました。命令文と理想の応答を大量に見せると、モデルは「人間が望む形で答える」癖を身につけます。LLaVA の発想はシンプルで「これを画像つきでもやろう」です。つまり「画像+指示文+模範回答」というデータで LLM を微調整すれば、画像を理解しつつ指示に従う VLM になるはずだ、という狙いです。

最大の障害は「画像つき指示データが世の中に存在しない」ことでした。LLaVA はここを巧みに解決します。鍵は「画像を見ていないテキスト LLM に、画像の文字情報だけ渡して会話を捏造させる」という逆転の発想です。世の中には画像のキャプションや物体の座標を集めたデータセットがあるので、その「文字で書かれた事実」を強い LLM (当時の GPT-4 のテキスト版) に渡し、「この情景についての対話を作れ」と頼めば、人手をほぼ介さず大量の指示データが作れる、という着想です。

## 基礎: 前提となる概念
- 指示チューニング (instruction tuning): 「指示 → 望ましい応答」のペアでモデルを微調整し、人間の意図に沿って振る舞わせること。タスクごとの専用学習なしに、文章で頼めば従うようになります。
- 画像トークン: 視覚エンコーダが出した画像特徴を、LLM が単語と並べて読めるベクトル列に変換したもの。LLaVA では各パッチがそのまま1トークンになります。
- 射影層 (projection / connector、コネクタ): 画像特徴を LLM の埋め込み次元 (語ベクトルの大きさ) に合わせる変換。LLaVA 初代は単層線形、LLaVA-1.5 は2層 MLP (多層パーセプトロン、線形変換と非線形関数を重ねたもの) です。
- Vicuna: LLaMA (Meta の公開 LLM) を会話データで微調整したオープンな対話 LLM。LLaVA の言語側として使われます。
- self-instruct (自己指示生成): 強いモデルに指示データそのものを生成させて学習に使う手法。LLaVA はこれの「画像版」を実装しました。

## 仕組みを詳しく
まずデータ作りが独特です (GPT-4 を使った自己生成、self-instruct の画像版)。

1. 既存の画像キャプション・物体検出ボックスのデータセット (COCO) を用意する。
2. 画像そのものではなく、その画像の「キャプションの文字列」と「物体の座標リスト (bounding box)」をテキストとして GPT-4 (当時は画像未対応のテキスト版) に渡す。
3. GPT-4 に「この情景についての質問と答え (conversation)」「詳細な説明 (detailed description)」「複雑な推論 (complex reasoning)」の3種類を文章で大量生成させる。GPT-4 は画像を見ていないが、テキストの情報から会話を捏ね上げる。
4. こうして約15.8万件の「画像 ↔ 指示 ↔ 回答」データを人手をほぼ介さず作る。この「データを LLM に作らせる」工程こそ LLaVA の中核的貢献です。

次にモデル構造はとてもシンプルです。

- 凍結した CLIP ViT-L/14 が画像をパッチ特徴 [576, 1024] に変換する (576 = 24×24 パッチ、入力336px・パッチ14px の場合)。
- 射影層 (初代は1層線形、LLaVA-1.5 は2層 MLP) でこれを LLM の埋め込み次元へ変換し、[576, 4096] のような「画像トークン列」にする。
- この画像トークン列を、ユーザの指示文トークンの前に連結して LLM (Vicuna/LLaMA) に入れる。LLM はそれを言葉の文脈として読み、回答を生成する。

[BLIP-2 (Q-Former)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0512_vlm-blip2) の Q-Former が「32トークンに能動的に絞る橋」だったのに対し、LLaVA は「576トークンをほぼそのまま線形変換で流す橋」です。橋が単純な分、(a) 情報の取りこぼしが少ない、(b) 学習・実装が容易、という違いがあります。代償として LLM に入る画像トークンが長く (576本)、推論が重くなります。

学習も2段階です。
- 第1段階 (特徴アライメント、feature alignment): 射影層だけを学習し、画像トークンが LLM の言語空間にざっくり揃うようにする (画像エンコーダも LLM も凍結)。約59.5万件の画像-キャプションペアを使い、「画像トークン → 既存キャプション」を予測させて橋を整える。
- 第2段階 (指示チューニング、visual instruction tuning): 射影層と LLM を、前述の指示データで微調整する。これで「指示に従って画像を説明・推論する」能力が宿る。視覚エンコーダは凍結のままが基本。

具体例: 入力画像 (交差点) +指示「この状況で運転手は何に注意すべき?」→ LLaVA は576個の画像トークンを文脈に、「左から歩行者が横断中で、前方の信号は黄色です。減速し停止に備えるべきです」のように文章生成します。橋が単純なので「どこに何が」という空間情報がパッチ単位でそのまま LLM に届くのが、こうした位置を伴う説明に効きます。

## 手法の系譜と主要論文
- LLaVA (Liu et al., "Visual Instruction Tuning", NeurIPS 2023, Microsoft/UW-Madison, arXiv:2304.08485): GPT-4 で画像指示データを自動生成し、CLIP 視覚特徴を線形射影で LLM に繋いで指示チューニングする枠組みを提案。動機は「画像つき指示追従データが無いなら、テキスト LLM に作らせればよい」。新規性は、複雑な橋を捨て、データ生成側に工夫を寄せた点です。
- MiniGPT-4 (Zhu et al., 2023, KAUST, arXiv:2304.10592): ほぼ同時期に、BLIP-2 の Q-Former + 凍結 Vicuna に1層の射影を足して整え、対話能力が出ると示した。LLaVA とともに「凍結部品 + 軽い接続 + 少量の高品質対話データ」で VLM が作れることを広めました。
- InstructBLIP (Dai et al., NeurIPS 2023, Salesforce, arXiv:2305.06500): [BLIP-2 (Q-Former)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0512_vlm-blip2) の Q-Former を「指示に応じて画像特徴を抜く」よう拡張し、多数のデータセットを指示形式に統一して学習。LLaVA (単純な橋+指示データ) と対をなす設計思想で、同時期に「橋を賢くする派」と「橋を単純にしてデータで勝負する派」が並走しました。
- LLaVA-1.5 (Liu et al., 2023, arXiv:2310.03744): 射影層を1層→2層 MLP に、入力解像度を 336px に、学習データに学術 VQA (OKVQA, OCR-VQA, A-OKVQA など) を追加した改良版。動機は「単純な改良の積み重ねで大幅に性能が伸びる」。公開データのみ・1日弱の学習で多数のベンチを当時の SOTA (最高水準) 級に押し上げ、VLM 研究の標準ベースラインになりました。
- LLaVA-NeXT (LLaVA-1.6, Liu et al., 2024): 高解像度画像をタイル (AnyRes、画像を格子状に分割し各タイルを個別にエンコードして結合) に分割し、細部・OCR・図表の弱さを大幅に改善。LLaVA 系が抱えていた「低解像度だと小さな文字や遠方物体を読めない」問題への直接的な回答で、後続の高解像度 VLM 設計の雛形になりました。後続の LLaVA-OneVision (2024) は画像・複数画像・動画を統一的に扱う方向へ拡張しています。

## 論文の実験結果 (定量データ)
- マルチモーダル対話/推論: LLaVA は GPT-4 を採点者にした対話評価 (LLaVA-Bench) で、テキスト版 GPT-4 を基準 (100%) としておよそ 85% 前後の相対スコアを報告しました。これは「画像を見ない GPT-4 が捏造した会話で学んだだけのモデルが、対話品質で意外なほど健闘した」ことを示します。
- ScienceQA: 科学の図表つき選択式問題 ScienceQA で、LLaVA は単体で約 90.9%、GPT-4 と組み合わせる工夫で約 92.5% を達成し、当時の SOTA を更新しました。図やグラフを読んで理科の問題を解く課題で、視覚と言語の統合的推論を測ります。
- LLaVA-1.5 の伸び: LLaVA-1.5 (13B) は射影層の MLP 化・解像度向上・VQA データ追加というシンプルな変更で、11 のベンチで広く改善し、多くで SOTA 級に到達。例えば VQAv2 で約 80%、GQA (推論を要する VQA) で約 63%、TextVQA (画像内文字の読解) で約 61% などが報告されました。公開データのみ・8×A100 約1日という低コストで達成した点が再現性の観点で高く評価されました。
- アブレーションの教訓: 射影層を線形1層から2層 MLP にすると性能が一貫して向上し、入力解像度を 224 → 336 に上げると細部を要するタスクで明確に伸びました。学術 VQA データを混ぜると VQA 系ベンチが上がる一方、入れすぎると自由対話の自然さが落ちる、というデータ配合のトレードオフも観察されています。逆に LLaVA-NeXT が示したのは「解像度を上げずに細部タスクを伸ばすのは困難」で、AnyRes タイル化で TextVQA/DocVQA など OCR/図表系ベンチが大きく改善しました。これらは「橋の容量」と「入力解像度」が VLM 性能の主要な律速 (ボトルネック) であることを示すアブレーションです。
- 幻覚の測定: object hallucination (画像に無い物を述べる現象) を測る POPE (Li et al., EMNLP 2023) では、LLaVA-1.5 は前世代や一部の Q-Former 型より幻覚が少ないと報告され、単純な橋でも適切なデータ設計で幻覚を抑えられることが示されました。

## メリット・トレードオフ・限界
- メリット: 構造が単純 (線形/MLP の橋) で実装・再学習が容易。標準ベースラインとして再現性が高く、研究の比較基準になっている。
- メリット: 画像特徴を絞らず流すため、細部情報の取りこぼしが Q-Former 型より少ない。パッチ単位の空間情報がそのまま届く。
- メリット: 指示チューニングにより自由な質問に対話的に答えられ、因果的・段階的な推論を引き出しやすい。
- トレードオフ: 画像トークンが長く (576本〜、AnyRes ではさらに数千本)、入力が伸びて推論コスト (時間・メモリ) が上がる。動画や多数画像では特に深刻。
- トレードオフ: 自己生成 (GPT-4 由来) データのバイアス・誤りを継承し、存在しない物を述べる幻覚が起きやすい。安全クリティカルな用途では検証が必須。
- 限界・未解決: 凍結 CLIP の限界 (細かい計数・空間関係) をそのまま引き継ぐ。また初代は低解像度ゆえ OCR や遠方物体に弱く、これは LLaVA-NeXT のタイル化で部分的に解決されましたが、トークン爆発 (トークン数が数千本に膨れること) という新たなコストを生みました。データの質 (どんな指示・応答で学ぶか) が性能を大きく左右するため、データ設計自体が研究テーマになっています。

## 発展トピック・研究の最前線
- 高解像度処理: AnyRes/タイル化、動的解像度、トークン圧縮 (重要でない画像トークンを間引く、または併合する) など、「細部を保ちつつトークンを増やしすぎない」ための研究が盛んです。運転や動画のように入力が大きい領域では律速になります。
- データ中心の改善: LLaVA の核心は「データ生成」にあり、より多様で高品質な指示データ (推論連鎖 chain-of-thought を含む、領域特化の) を作る取り組みが続いています。運転や医療など領域特化の QA データ生成もこの系譜です。
- 幻覚の抑制: POPE などのベンチが整備され、対比的な学習・根拠提示・検証用の追加学習で抑える研究が進んでいます。安全が問われる用途では特に重要です。
- 動画・複数画像への拡張: LLaVA-OneVision などは単一画像から動画・複数画像へ統一的に拡張し、時系列を扱える方向へ進んでいます。連続フレームの理解は運転シーン解析と直結します。
- 単純さの勝利という教訓: LLaVA は「凝った橋より、単純な橋+良いデータ+十分な解像度」という方向性を示し、その後の多くのオープン VLM ([Qwen-VL](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0516_vlm-qwen-vl) を含む) がこのレシピの要素を踏襲しています。
