# EfficientNet

## ひとことで言うと
EfficientNet は、画像から特徴を取り出す畳み込みニューラルネットワークを「賢く大きくする」ための設計則とモデル群です。ネットワークを大きくする方法には「深さ(層数)」「幅(チャンネル数)」「入力解像度(画像の大きさ)」の3つがありますが、EfficientNet はこの3つを1つの決まった比率でまとめて増やす「複合スケーリング(compound scaling)」を提案しました。土台となる小型モデル B0 自体も自動構造探索(NAS)で見つけ、そこから B1〜B7 を体系的に拡大します。同じ計算量で従来より一段高い精度を出せるのが核心です。

## 直感的な理解
モデルの精度を上げたいとき、定石は「モデルを大きくする」ことです。しかし大きくする方向が3つあり、どれか1つだけを極端に増やしてもすぐ頭打ちになります。たとえば層だけを4倍に深くしても、入力画像が小さいままでは細部の情報がそもそも足りず、深さを活かせません。幅だけ広げても、抽象化を担う深さが足りなければ高レベルな概念を学べません。

EfficientNet の著者の気づきは「3つの軸はバランスが大事で、互いに釣り合った比率で同時に増やすべき」というものです。直感的には、入力画像を大きくする(解像度↑)なら、増えたピクセルを処理して受容野を保つために層も増やし(深さ↑)、細部を表す特徴の本数も増やす(幅↑)必要がある、ということです。風船を膨らませるとき縦だけ伸ばすといびつになるのと同じで、3方向をバランス良く膨らませてこそ綺麗に大きくなります。問題は「どんな比率で増やすのが最適か」で、それを実験から1組の定数として求めたのが複合スケーリングです。

## 基礎: 前提となる概念
3つのスケーリング軸を確認します。深さは層の数で、複雑なパターンを学べますが深すぎると勾配が流れにくくなり(残差接続があっても飽和する)、リターンが減ります。幅は各層のチャンネル数(検出する特徴の本数)で、きめ細かい特徴を多く持てますが、広いだけでは抽象化が浅くなります。解像度は入力画像の縦横ピクセル数で、細部が見えるようになりますが、計算量が縦横の積、つまり解像度の2乗で増えます。

次に depthwise convolution(深さ方向畳み込み)です。通常の畳み込みは全入力チャンネルを混ぜながら空間処理をしますが、depthwise はチャンネルごとに独立した畳み込みを行い、チャンネル間は混ぜません。これは前述のグループ化畳み込みでグループ数をチャンネル数まで増やした極限です。空間処理を depthwise が担い、チャンネル混合は別の1×1畳み込み(pointwise)が担う「分業(depthwise separable convolution)」により、通常畳み込みのおよそ 1/k²(k はカーネルサイズ)程度まで計算量を削減できます。

Squeeze-and-Excitation(SE、Hu ら CVPR 2018)も重要部品です。これは「どのチャンネルが今の入力にとって重要か」を学習で判断し、チャンネルごとに重み付けする小さな仕組みです。具体的には、各チャンネルの空間平均(squeeze、[C,H,W]→[C])を取って1本のベクトルにし、小さな全結合層を2段(C→C/r→C、r は縮約率)通して0〜1の重要度(excitation)を出し、元の特徴マップにチャンネルごとに掛けます。チャンネル方向の注意機構(attention)の一種で、わずかな計算追加で精度を上げます。

最後に NAS(Neural Architecture Search、ニューラル構造探索)です。これはネットワークの構造(各層の種類・カーネルサイズ・チャンネル数など)を人手ではなく自動で探索する技術です。探索空間を定め、候補モデルを学習・評価して、精度や速度を報酬に良い構造を選びます。EfficientNet の土台 B0 はこの NAS で見つけられました。

## 仕組みを詳しく
複合スケーリングは、1つの複合係数 φ(ファイ)で3軸の拡大率を一括制御します。

- 深さ = α^φ
- 幅 = β^φ
- 解像度 = γ^φ

ここで α・β・γ は定数です。畳み込みの計算量(FLOPs)は、深さに比例し、幅の2乗に比例し、解像度の2乗に比例します。そこで著者らは「φ を1増やすと総 FLOPs がおよそ2倍になる」という制約

  α × β² × γ² ≈ 2、 ただし α≥1, β≥1, γ≥1

を課しました。この制約のもとで、小さなネットワーク(B0)上のグリッドサーチ(総当たり探索)を行い、α≈1.2、β≈1.1、γ≈1.15 を見つけました。あとは φ を 0, 1, 2, … と増やすだけで、3軸がこの比率でバランス良く同時に拡大されます。たとえば φ=1 なら深さ1.2倍・幅1.1倍・解像度1.15倍、φ=2 ならそれぞれ2乗倍(深さ1.44倍など)、という具合です。なお α・β・γ は小さな B0 で1回だけ探索すればよく、大きなモデルでは探索コストをかけずに φ を上げるだけ、というのが計算面での巧妙さです(大きなモデルで直接探索すると非現実的なコストになる)。

```
複合係数 φ を1段上げると…
  深さ × 1.2     (層が少し増える)
  幅   × 1.1     (チャンネルが少し増える)
  解像度 × 1.15  (入力画像が少し大きくなる)
  → 計算量はおよそ2倍、精度はバランス良く向上
B0(224px) → B3(300px) → B5(456px) → B7(600px)
```

複合スケーリングを適用する土台が EfficientNet-B0 です。B0 は MnasNet と同じ探索枠組みの NAS で、推論時の計算量(FLOPs、目標は約400M)を報酬に組み込んで最適化して見つけられました。その主部品が MBConv(Mobile Inverted Bottleneck Convolution)で、MobileNetV2 由来です。MBConv の流れは次の5段です。

1. 1×1 畳み込みでチャンネルを一度「広げる」(expand、たとえば6倍)
2. depthwise 畳み込み(チャンネルごとに独立した3×3または5×5)で空間特徴を取る
3. Squeeze-and-Excitation でチャンネルごとの重要度を重み付け
4. 1×1 畳み込みでチャンネルを元の細さに「戻す」(project、ここは活性化関数を入れない linear bottleneck)
5. 入力と出力を足す残差接続(入口と出口のチャンネル数・空間サイズが一致するときだけ)

通常の Bottleneck が「絞ってから処理して戻す」のに対し、MBConv は「広げてから depthwise で軽く処理して絞る」という逆ボトルネック(inverted residual)構造です。チャンネルを広げた中間で空間処理をすると表現力が上がり、入口と出口を細くすることでブロック間を流れるデータ量(とメモリ)を抑えられます。残差接続は細い入口・出口どうしを結ぶので、メモリ効率も良くなります。最後の project を活性化なしの線形にするのは、低次元(細い)空間で ReLU を通すと情報が壊れやすい、という MobileNetV2 の知見によるものです。

B0 から φ を上げて B1, B2, …, B7 まで作ると、入力解像度は 224px → 600px、パラメータは約530万 → 約6600万、FLOPs は約0.39B → 約37B へと段階的に増えます。

## 手法の系譜と主要論文
原論文は Tan・Le「EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks」(ICML 2019、arXiv:1905.11946、Google Brain / Google Research)です。動機は「3軸を場当たり的・職人技で増やすのをやめ、原理に基づいてバランス良くスケールしたい」ことでした。貢献は2つで、(1) 複合スケーリング則の提案と、3軸が独立でなく相互依存するという経験的裏付け、(2) NAS で見つけた効率的ベースライン B0 の提供です。

土台となった部品の系譜が重要です。Sandler ら「MobileNetV2」(CVPR 2018)が逆ボトルネック構造(inverted residual + linear bottleneck)を提案し、Tan ら「MnasNet」(CVPR 2019)が実機(モバイル端末)の推論レイテンシも報酬に含めた platform-aware NAS を提案しました。Hu ら「Squeeze-and-Excitation Networks」(CVPR 2018、ILSVRC 2017 優勝)がチャンネル注意を加えました。EfficientNet はこれら3つを統合し、その上にスケーリング則を載せた、という位置づけです。

続く Tan・Le「EfficientNetV2」(ICML 2021、arXiv:2104.00298)は、初代の弱点を直接修正しました。第一に、初代の大きな depthwise 畳み込みは初期層では GPU/TPU をうまく使えず学習が遅いことが分かり、初期ステージを通常の畳み込みを使う Fused-MBConv(expand の1×1 と depthwise を1つの通常3×3にまとめた版)に置き換えました。第二に、学習時に画像解像度を段階的に上げ、それに合わせて正則化(Dropout, データ拡張)も段階的に強める progressive learning を導入し、学習を大幅に高速化しました。また NAS の報酬を FLOPs ではなく実機の学習スループットへ寄せ、訓練効率を直接最適化しました。系譜は MobileNetV2 / SE / MnasNet → EfficientNet → EfficientNetV2 と整理できます。

## 論文の実験結果(定量データ)
主評価は ImageNet-1K(120万枚、1000クラス)で、指標はトップ1精度(top-1 accuracy、最有力候補が正解する割合、高いほど良い)です。最大モデル EfficientNet-B7 はトップ1精度 約84.3%(トップ5 約97.0%)を達成し、当時の最高精度モデルだった GPipe(約84.3%)と同等の精度を、約8.4分の1のパラメータ(B7 約6600万 対 GPipe 約5.5億)と約6.1分の1の計算量で実現しました。さらに、CPU 推論で GPipe より約6倍速いと報告されています。小型側でも、EfficientNet-B1 が ResNet-152 と同等のトップ1精度(約78.8%)を約7.6分の1のパラメータで、また B4 が ResNeXt-101 と同等精度を約半分以下の計算量で出すなど、精度/効率比のフロンティアを軒並み更新しました。

最も核心的なアブレーションは複合スケーリングの効果検証です。同じ B0 を土台に、(a) 深さだけ、(b) 幅だけ、(c) 解像度だけ、(d) 複合(3軸同時)でそれぞれ FLOPs を増やしたとき、(a)(b)(c) はどれも約80%付近で精度が頭打ちになるのに対し、(d) の複合スケーリングだけがそれを超えて精度を伸ばし続けました。同じ FLOPs 増加に対して、複合スケーリングは単一軸スケーリングより最大で約2.5ポイント高い精度を出すと報告されています。これが「バランス良く同時に増やすべき」という主張の直接的な根拠です。クラス活性化マップ(CAM)の可視化でも、複合スケーリングしたモデルは対象物体により集中した、意味のある領域に反応することが示されました。

転移学習(ImageNet で学んだ重みを別データセットに応用する)でも強さを示しました。CIFAR-100(約91.7%)、Flowers(約98.8%)、Stanford Cars など8つのデータセットのうち5つで当時の最高精度を更新し、しかも転移時のパラメータは平均で約9.6分の1という効率でした。

EfficientNetV2 のアブレーションでは、Fused-MBConv を初期ステージ(浅い側の数ステージ)に使うと学習が速くなる一方、全ステージに使うと逆に遅くパラメータも増えるという非自明な結果が示され、初期だけ通常畳み込み・後段だけ depthwise という配置が最適だと結論づけられました。progressive learning と合わせて、V2 は同等精度で初代の約5〜11倍速く学習でき、ImageNet トップ1で約85.7%(V2-L、ImageNet-21k 事前学習で約87%超)に達したと報告しています。

## メリット・トレードオフ・限界
メリットは、深さ・幅・解像度をバランス良く同時に拡大でき精度/計算量比が非常に高いこと、1つの複合係数 φ でモデルの大小を体系的に切り替えられること、B0〜B7 と幅広いサイズが用意され用途に応じて選べることです。FLOPs あたりの精度では長くベンチマークの基準でした。

トレードオフと限界もはっきりしています。第一に、depthwise 畳み込みは理論 FLOPs は小さくても、演算密度(計算量あたりのメモリアクセス、arithmetic intensity)が低いため、GPU/TPU のメモリ帯域がボトルネックになり、FLOPs の割に実機の学習・推論が遅いことがあります。第二に、高解像度入力(B5 以降は 456px〜600px)は順伝播で生じる中間データ(アクティベーション)のメモリ消費が非常に大きく、大きなバッチが組めません。学習効率に直結する実務的な制約で、V2 が学習解像度を制限したのもこのためです。第三に、初期層の大きな depthwise が学習を遅くする弱点があり、これは V2 で Fused-MBConv により改善されました。第四に、SE や逆ボトルネックなど構造がやや複雑で、ResNet 系の素直さに欠け、一部の推論ハードウェア(特に SE の global pooling が苦手なアクセラレータ)では最適化が難しいことがあります。

研究上の教訓として、EfficientNet は「FLOPs を最適化しても実機の速度・メモリは最適化されない」というギャップを浮き彫りにしました。これが後続研究で、FLOPs ではなく実機レイテンシやメモリ、学習スループットを直接目標にする設計(V2 自身、RegNet、ハードウェア対応 NAS)へ向かわせる契機の1つになりました。

## 発展トピック・研究の最前線
EfficientNet は「モデルスケーリングを科学にする」という研究の流れを決定づけました。その後 RegNet(Radosavovic ら, CVPR 2020)は、単一モデルではなく「設計空間」全体を統計的に分析し、良いモデルが従う設計則(幅や深さがステージごとに線形に増える、最適なグループ幅が存在する、など)を量子化された設計パラメータとして導出する方向へ進めました。複合スケーリングの思想は Vision Transformer のスケーリング研究(モデル幅・深さ・パッチ数・データ量を同時に論じる)にも引き継がれ、データ量・モデルサイズ・計算量のバランスを論じるスケーリング則(power-law)の系譜につながっています。

実用面では、EfficientNet は物体検出のバックボーンとしても展開され、EfficientDet(Tan ら, CVPR 2020)が EfficientNet バックボーンと双方向特徴ピラミッド(BiFPN)を組み合わせ、検出のバックボーン・ネック・解像度を同じ複合スケーリングで同時に拡大することで、検出でも精度/効率フロンティアを更新しました(COCO で当時最高水準の mAP を大幅に少ない計算量で達成)。研究の最前線では、depthwise の実機効率の悪さを踏まえ、通常畳み込みと depthwise を混ぜる Fused 設計、ハードウェア対応 NAS、そして CNN と注意機構を融合したハイブリッド設計(CoAtNet、MobileViT など)へと関心が広がっています。
