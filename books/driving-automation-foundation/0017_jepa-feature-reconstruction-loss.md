# JEPA特徴再構成ロス

## ひとことで言うと
「これから世界がどう見えるか」をモデルに予測させて学習させたいとき、未来の画像そのもの(ピクセル)を当てさせるのは難しすぎます。そこで、画像をいったん特徴(抽象的な数値ベクトル)に変換してから、予測した特徴と本物の特徴の差を測ります。これが JEPA 流の特徴再構成ロスです。予測すべき対象を「予測可能な抽象構造」だけに絞ることで、学習が安定します。

## 直感的な理解
窓の外の景色を見て「10 秒後どうなっているか」を予想してください。前の車がどこにいるか、信号が変わるか、といった大きな流れはかなり当てられます。一方で、隣の木の葉が次にどう揺れるか、対向車のナンバーの数字、影の細かい形まで正確に当てるのは不可能です。これらは予測不能なノイズです。

もしモデルに「未来の画像のピクセルを全部当てろ」と要求すると、モデルはこの予測不能な細部まで無理に当てようとして、せっかくの計算能力(容量)をムダ遣いします。結果、肝心の「車がどう動くか」「信号が変わるか」という運転判断に効く構造をかえって学び損ねます。人間が未来を予想するときも、ピクセル単位ではなく「車・人・信号といった抽象的なレベル」で予想しているはずです。JEPA はこの直感を学習に持ち込み、「ピクセルではなく特徴で予測せよ」と主張します。

## 基礎: 前提となる概念

用語を平易に説明します。
- 世界モデル(world model): 「今こう動いたら次に世界がこう変わるか」を予測するモデル。本ノートの文脈では、数秒先(例 +1.6 / 3.2 / 4.8 / 6.4 秒)の視覚特徴マップを予測する部品を指します([FutureState 世界モデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0013_future-state-world-model))。
- 損失(loss): モデルの予測が正解からどれだけずれているかの数値。これを小さくするよう学習します。
- ピクセル空間(pixel space): 画像を「各ピクセルの色の値(RGB)」として扱う空間。
- 特徴空間(feature space): 画像をニューラルネット(バックボーン)に通して得た、抽象的な数値ベクトルの空間。細部のノイズが落ち、意味的な構造が残ります。
- バックボーン(backbone): 画像から特徴を抽出する大きなニューラルネット。CNN(畳み込みネット)や ViT(Vision Transformer)が使われます。
- 自己教師あり学習(self-supervised learning): 人手のラベルなしで、データ自身から教師信号を作って学習する方法。たとえば「画像の一部を隠して残りから当てる」など。
- 凍結(frozen): 学習中に重みを更新しない(勾配を流さない)こと。固定された変換として使います。
- detach / stop-gradient(勾配の切断): あるテンソルを通る勾配の逆流を止める操作。正解側に勾配を流さないために使います。
- EMA(指数移動平均): ターゲット側の重みを、学習側の重みのゆっくりした平均としてコピーしていく更新法。`θ_target ← m·θ_target + (1-m)·θ_online`(m は 0.99 などのモメンタム)で、ターゲットが緩慢に追従します。
- 表現崩壊(representation collapse): モデルが手抜きをして「何を入れても同じベクトルを出す」状態に陥ること。差がゼロになり損失も下がるが、何も学べていない退化解です。

未来予測を「ピクセルを当てる」課題にすると、前述のとおり予測不能な細部に容量を奪われます。JEPA(Joint-Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)の発想は、入力も予測対象もいったん特徴(埋め込み)に変換し、特徴空間で予測の良し悪しを測ることです。特徴は既に細部の予測不能ノイズを捨て、シーンの抽象的な意味(物の配置・動きの大局)を残しているので、当てるべきものが「予測可能な抽象構造」だけになり、学習が安定します。この発想は LeCun が 2022 年の構想論文「A Path Towards Autonomous Machine Intelligence」で自律知能の中核に据えたもので、I-JEPA / V-JEPA はその具体化です。

## 仕組みを詳しく

特徴再構成ロスの典型的な処理は次のとおりです。

1. 世界モデルが予測した未来特徴マップ群を受け取る。タプル/リストで、各要素は `[B, C, H, W]` の 4 次元テンソル。
   - B はバッチサイズ、C はチャンネル数(埋め込み次元、例 256)、H×W は俯瞰図(BEV, bird's-eye-view)グリッドや特徴マップの空間解像度。
   - 予測する時刻数が複数なら、`num_future_steps`(例 4)= +1.6 / 3.2 / 4.8 / 6.4 秒の 4 枚、のように複数要素になる。
2. 同じ形のターゲット特徴マップ群を受け取る。これは凍結バックボーンを未来フレームの実画像に適用して作る。
   - 凍結バックボーンに実際の +1.6 秒後などの画像を入れて特徴を取り出し、それを正解とする。ターゲットは detach して勾配を切る。こうすると正解側が学習で動かず、安定した目標になる。
3. 各時刻について MSE(平均二乗誤差)を計算: `mean((pred - target)^2)`。
4. 各ステップの損失を `[num_future_steps]` に積み、時刻ごとの重み(temporal weights)を掛ける。
5. reduction が "mean" なら重み付き平均のスカラーを返す。"none" ならステップ別損失(例 `[4]`)を返してログ用に使う。

実装上の数値の振る舞いに関わる勘所:

- 形が合わないと例外を出す。予測とターゲットの shape 不一致、要素数の不一致を厳格にチェックする。気づかぬうちに別物を比較する事故を防ぐため。
- 時刻別重みは「重み付き平均が素の MSE と同じスケールに乗る」よう正規化する。たとえば `weights = weights / total * num_future_steps` とすると、重みを変えても損失の大きさの尺度がぶれない。学習率や他の損失とのバランスを取り直さずに重みだけ調整できる。
- ゼロ和ガード: 重みの合計が極小(例 1e-8 以下)だと割り算で NaN(非数)になるため、その縮退ケースでは正規化をスキップして与えられた重みをそのまま使う。損失が静かに壊れるのを防ぐ安全装置。

時刻別重みの意味: 近い未来(+1.6 秒)は予測しやすく、遠い未来(+6.4 秒)は本質的に難しい。重みで「どの時刻を重視するか」を調整でき、たとえば遠い未来を軽くして近い未来を重視する設定が可能です。逆に「遠い未来も当てる力」を伸ばしたいなら遠方重視にする、というカリキュラム的な使い方もできます。

ピクセル空間と特徴空間の違いを数値イメージで:

```
ピクセル一致を強制: 予測すべき次元 = H*W*3(色)で、大半が予測不能ノイズ → 学習が細部に振り回される
特徴空間で比較:     予測すべき次元 = [C,H,W] の抽象特徴で、運転に効く構造だけ残る → 安定
```

ここで一番デリケートなのが表現崩壊の回避です。もし正解側のバックボーンも学習で一緒に更新してしまうと、モデルは「予測も正解も全部同じ定数ベクトルにする」という手抜き解に落ちて損失をゼロにできてしまいます。これでは何も学べません。凍結バックボーン+detach はこの崩壊を簡単かつ確実に防ぐ手段で、正解側が固定なので「全部同じ」に縮退する誘惑が生じません。JEPA 系の研究では、正解側を完全凍結する代わりに EMA でゆっくり動かす方法も使われますが、いずれにせよ「正解側に勾配を流さない(stop-gradient)」ことが崩壊回避の必須要件である点は共通です。SimSiam(Chen & He, 2021)は、EMA すら使わず stop-gradient だけで崩壊を防げることを示し、崩壊の本質が「ターゲット側に勾配を流さないこと」にあると明らかにしました。

なぜ stop-gradient が効くのか、直感を補足します。予測側と正解側の両方に勾配が流れると、両者は「互いに歩み寄って一番楽な一致点(=定数)」へ滑り落ちます。片側を止めると、正解側は動かない的になり、予測側は「動かない的に当てる」課題を解かざるを得ず、定数に縮退する近道が塞がれます。

## 手法の系譜と主要論文

- BYOL (Grill et al., 2020, NeurIPS, arXiv:2006.07733)。負例(押し離す対象)なしで、生徒ネットワークが教師ネットワーク(生徒の EMA)の出力を予測する自己教師あり学習。stop-gradient と EMA ターゲットだけで表現崩壊を回避できることを示し、後の「予測ターゲットを固定/緩慢に動かす」設計の基礎になりました。
- SimSiam (Chen & He, 2021, CVPR, arXiv:2011.10566)。BYOL から EMA を取り除いても、stop-gradient さえあれば崩壊しないことを示し、崩壊回避の核心が stop-gradient であることを明確化しました。
- MAE (He et al., 2022, CVPR, arXiv:2111.06377)。対比対象として重要。画像の大半(例 75%)をマスクし、ピクセル空間で欠損部を再構成する自己教師あり学習。単純で強力ですが、予測対象がピクセルなので細部再構成に容量を使うのが弱点。JEPA はこの「ピクセルで再構成する」点を「特徴空間に移す」ことで改善しようとします。
- I-JEPA (Assran et al., 2023, CVPR, arXiv:2301.08243; Meta AI、LeCun の構想に基づく)。画像のマスクした領域を、ピクセルではなく特徴(埋め込み)空間で予測する。動機は MAE のようなピクセル再構成が予測不能な高周波細部に容量を割く問題の回避。ターゲットエンコーダは文脈エンコーダの EMA とし、stop-gradient で崩壊を防ぎます。
- V-JEPA (Bardes et al., 2024, Meta AI, arXiv:2404.08471)。動画版 JEPA。動画の時空間的に欠損した部分を特徴空間で予測する。動きの予測に有効で、「未来の視覚特徴を予測する」という設定はこの動画 JEPA の発想に最も近いです。
- 自動運転の世界モデル系(ピクセル/トークンで未来を生成する大規模生成モデル群)は、未来をピクセルやトークンで生成するため計算が重く、細部に引きずられます。JEPA 流の特徴予測は「運転判断に効く抽象構造だけを予測する」軽量な代替として位置づけられます。

系譜をまとめると、ピクセル再構成(MAE, 2022)→ 特徴空間での予測と崩壊回避(BYOL 2020・SimSiam 2021 の知見を取り込んだ I-JEPA 2023)→ 動画・未来への拡張(V-JEPA 2024)→ 自動運転の未来特徴予測への応用、という流れです。

## 論文の実験結果(定量データ)

- I-JEPA (Assran et al., 2023) は ImageNet の線形評価(特徴を固定し線形分類器だけ学習して精度を測る評価)で、ピクセル再構成型の MAE を上回る精度を、より少ない学習計算量で達成したと報告しています。論文では、ViT-Huge を用いた構成で MAE 比で大幅に短い事前学習エポック数でも高い線形評価精度(70% 台後半〜)に達し、計算効率の良さを示しました。指標の線形評価精度は「特徴がどれだけ意味的に分離しているか」を測り、高いほど良い表現です。重要な含意は「ピクセルを当てない方が、同じ計算で意味的に強い特徴が得られる」ことです。
- アブレーションとして、予測ターゲットを特徴空間からピクセル空間に戻すと、同条件で線形評価精度が低下することが示されており、「特徴空間で予測する」こと自体が性能の源泉であると裏付けられています。また、ターゲットエンコーダの stop-gradient/EMA を外すと表現崩壊が起き、損失は下がるのに評価精度が壊れる、という結果が、崩壊回避の工夫が必須であることを定量的に示します。
- V-JEPA (Bardes et al., 2024) は、動画特徴のフリーズ評価(凍結した特徴での下流タスク評価)で、ピクセル再構成型の動画自己教師あり手法に対し、動作認識など複数ベンチマーク(Kinetics-400、Something-Something-v2 など)で上回る精度を、少ない学習データ・計算で達成したと報告しています。特に「動きの予測に効く特徴」が得られることが、未来予測への応用の根拠になっています。
- MAE (He et al., 2022) はファインチューニング(全層を下流タスクで再学習)では非常に強い一方、特徴を凍結したままの線形評価では JEPA 系に劣る、という対比がよく引かれます。これは「ピクセル再構成は強い初期化を与えるが、特徴そのものの意味的な分離は特徴空間予測に分がある」という棲み分けを示します。
- BYOL (Grill et al., 2020) は、負例を一切使わずに ImageNet 線形評価で当時の対照学習(SimCLR など)に匹敵・凌駕する精度(ResNet-50 で約 74%、より大きなバックボーンで 79% 超)を達成し、「ターゲット固定+stop-gradient」だけで強い表現が学べることを示しました。これが JEPA 系のターゲット設計の出発点です。

これらの数値は、未来予測の損失をピクセルではなく特徴空間で取ることが、効率と表現品質の両面で有利であることを支えています。

## メリット・トレードオフ・限界

メリット
- 予測不能なピクセル細部に振り回されず、運転判断に効く抽象構造の予測に集中できる(学習が安定し、計算効率も良い)。
- ターゲットを凍結バックボーン+detach にすることで、JEPA で問題になりがちな表現崩壊を簡単に避けられる。
- 時刻別重みで近い/遠い未来の重視度を制御でき、ゼロ和ガードで NaN 事故を防げる。
- 既存の予測パイプラインに任意適用しやすく、前向き計算の契約を壊さずに足せる(補助損失として後付けできる)。

トレードオフ・限界
- ターゲットを作る凍結バックボーンの質に依存する。バックボーンが弱いと正解自体が弱く、学んだ未来特徴も弱くなる。
- 特徴空間の損失は「人間が見て分かるピクセル一致」ではないので、デバッグや解釈が難しい。予測がどれだけ正しいかを直感的に確かめにくい。
- 未来フレームの実画像(+1.6 秒後など、時間的に整合の取れたデータ)が学習に必要で、データ整備のコストがかかる。
- 高解像度の俯瞰特徴(例 450×300 グリッド)では `[B, num_future_steps*C, H, W]` 相当の特徴が巨大になり、メモリ負荷が高い(bf16 など混合精度が前提になりやすい)。
- 特徴空間で MSE を取ると、特徴の各次元のスケールに結果が引きずられる。スケールの大きい次元が損失を支配して、意味的に重要だがスケールの小さい次元が無視される恐れがあり、正規化や白色化の工夫が要ることがある。
- 研究上の未解決課題として、特徴空間で何が予測され何が捨てられているかの定量的な解釈、凍結ターゲットと学習可能ターゲット(EMA)のどちらが下流タスクに良いかの条件依存性、未来予測特徴が実際の運転性能(衝突率・計画精度)にどれだけ寄与するかの直接評価が挙げられます。

## 発展トピック・研究の最前線
発展方向として、特徴空間予測に行動条件付け(自分がどの軌道を選ぶかで未来特徴がどう変わるか)を組み込み、世界モデルとして反実仮想(もし別の行動を取ったら)を扱う方向があります([行動条件付きロールアウト (counterfactual)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0002_action-conditioned-rollout))。また、ターゲットを単一バックボーンの特徴ではなく、複数視点や時系列を統合した表現にする、あるいは予測の不確実性(分散)も同時に学んで「予測しにくい部分は緩く採点する」ことで予測不能ノイズの扱いを洗練させる研究があります。崩壊回避についても、stop-gradient/EMA に加えて、特徴の分散や共分散を明示的に制約する手法(VICReg など)の系統があり、どの正則化が未来予測に最も効くかは未確定です。自動運転では、JEPA 流の特徴予測を補助損失として知覚・計画と同時学習し、表現の時間的一貫性を高めて計画の安定化につなげる試みが進んでいます。
