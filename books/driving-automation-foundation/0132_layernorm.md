# Layer Normalization

## ひとことで言うと
Layer Normalization(レイヤー正規化、略して LN)とは、1つのサンプル(事例)について、その特徴次元の並びを「平均0・分散1」にそろえ、そのあと学習可能な2つのパラメータ gamma(スケール、拡大縮小)と beta(シフト、ずらし)で調整しなおす処理です。正規化(normalization、これは「数の大きさのばらつきを一定の基準にそろえること」という意味です)の一種で、Transformer の標準部品として、ほぼすべての現代的な大規模モデルに使われています。

## 直感的な理解
ニューラルネットワークは層を重ねるほど、途中で流れる数値の大きさが層ごとにバラバラになりがちです。あるところでは数値が0.01付近、別のところでは100付近、ということが起きます。大きさがそろっていないと、学習で各層が「お互いに合わせて動く」のが難しく、収束が遅くなったり不安定になったりします。これを内部共変量シフト(internal covariate shift、層の入力分布が学習中に動いてしまう現象)と呼ぶこともあります。

たとえるなら、声の大きさがバラバラな合唱団です。一人だけ極端に声が大きいと、全体のバランスが崩れ、指揮者(optimizer)はその一人に引きずられて他をうまく調整できません。各層の入口で数値の大きさをそろえてやれば、後ろの層は安定した範囲の入力を受け取れ、学習がスムーズになります。これが正規化の狙いです。LN はその中でも「1サンプル内で特徴方向にそろえる」やり方で、特に Transformer(アテンション中心の構造)と相性が良いものです。

## 基礎: 前提となる概念
平均(mean)と分散(variance)を確認します。平均は数値の中心、分散はばらつきの大きさ(各値が平均からどれだけ離れているかの二乗の平均)を表します。「平均0・分散1にそろえる」とは、データの中心を0に移し、ばらつきの幅を1にそろえることです。これを標準化(standardization)と呼びます。標準化すると、元が [2, 4, 6] でも [200, 400, 600] でも同じ [-1.22, 0, 1.22] になり、絶対的な大きさの違いが消えて「相対的な形」だけが残ります。

テンソルの「次元(軸)」の理解が要点です。Transformer の中間表現は典型的に [B, T, D] という形をとります。B はバッチ内のサンプル数、T は系列長(トークン=処理の単位の数)、D は特徴次元(各トークンを表すベクトルの長さ)です。「どの軸で平均・分散を取るか」が正規化手法の違いそのものです。LN は最後の D 軸(特徴方向)だけで統計を取り、B と T はまたぎません。つまり [B, T, D] のテンソルなら B×T 個のベクトルそれぞれについて独立に D 個の値の平均・分散を計算します。

学習可能パラメータとは、誤差逆伝播で更新される重みのことです。LN の gamma と beta はそれぞれ D 次元のベクトルで、学習可能です。正規化で一律にそろえた後、各特徴次元ごとに「もう少し強調」「少しずらす」という微調整を学習で獲得します。これにより、正規化が情報を消しすぎないようにしています。極端には、gamma と beta が学習によって正規化を打ち消す値になることも可能で、モデルは「正規化を効かせる/効かせない」を自分で選べます。

## 仕組みを詳しく
式は次のとおりです。x は1サンプル(正確には1トークン)の特徴ベクトル、mean と var はそのベクトル内で計算した平均と分散、eps は0割りを防ぐ小さな値(例: 1e-5)です。

```
y = gamma * (x - mean) / sqrt(var + eps) + beta
```

具体例。x = [2.0, 4.0, 6.0] とします。
- 平均 mean = (2+4+6)/3 = 4.0
- 分散 var = ((2-4)^2 + (4-4)^2 + (6-4)^2)/3 = (4+0+4)/3 ≈ 2.667、その平方根 ≈ 1.633
- 正規化後 = [(2-4)/1.633, (4-4)/1.633, (6-4)/1.633] ≈ [-1.225, 0.0, 1.225]

これで平均0・分散1になりました。before [2, 4, 6] → after 約 [-1.22, 0, 1.22] です。最後に gamma と beta をかけて足します。初期値は gamma=1, beta=0 なので最初は正規化結果そのままですが、学習で調整されます。

重要なのは「どの方向で平均・分散を取るか」です。LN は1サンプル(1トークン)の特徴次元(上の例なら3要素)の中だけで計算します。バッチ内の他のサンプルや、系列内の他のトークンは一切参照しません。だからバッチサイズが1でも、系列長が可変でも、まったく同じ計算が成立します。この「他サンプルに依存しない」性質が、推論時(1件ずつ処理する場面が多い)に学習時とまったく同じ計算でよいという実装上の大きな利点を生みます。

正規化手法の家族を軸の違いで整理すると理解が深まります。[N, C, H, W](画像の N=バッチ、C=チャネル、H/W=空間)を例にとると、BatchNorm は (N, H, W) をまたいで C ごとに統計を取り、LayerNorm は (C, H, W) をまたいでサンプルごとに取り、InstanceNorm は (H, W) をまたいでサンプル×チャネルごとに取り、GroupNorm はチャネルをグループに分けてグループ×サンプルごとに取ります。同じ「正規化」でも、またぐ軸が違うだけでまったく性質の異なる層になるのです。

## 手法の系譜と主要論文
主引用は Ba, Kiros, Hinton "Layer Normalization"(arXiv:1607.06450, 2016)です。提案の動機は、当時広く使われていた Batch Normalization が、バッチ内の統計に依存するため、系列を扱う RNN やバッチが小さい場面で扱いにくかったことです。サンプル内特徴方向で統計を取れば、バッチの大きさや他サンプルに依存せず、1事例だけでも同じ計算ができ、系列長が可変でも安定します。効果は学習の安定化と収束の速さ。トレードオフは、各特徴次元が本来そろえないほうがよい意味(絶対的な大きさそのものが情報を持つ場合)を持つときでも一律にそろえてしまう点で、それを gamma/beta で取り戻す設計です。

Batch Normalization(Ioffe & Szegedy, ICML 2015)との違いを押さえます。BatchNorm(BN)は同じ特徴チャネルについてバッチ方向(複数サンプルをまたいで)平均・分散を取ります。LN は1サンプル内の特徴方向で取ります。BN はバッチが小さいと統計が不安定になり(バッチサイズ2や4では推定が荒れる)、推論時は学習中に貯めた移動平均を使うなど扱いが煩雑です。LN はバッチサイズに依存せず推論も学習も同じ式なので、Transformer や系列長が変わる場面、バッチを大きく取れない場面で有利です。逆に、畳み込み主体の画像分類では BN のほうが効くことも多く、構造に応じた使い分けが必要です。

Transformer での標準採用は Vaswani ら "Attention Is All You Need"(NeurIPS 2017)からです。アテンション(各トークンが他のトークンを重み付きで参照する仕組み)と残差接続(入力を出力に足す経路)を組み合わせたブロックに LN を入れることで、深い系列モデルの学習を安定させました。これが LN を一気に普及させた決定打です。

Pre-LN と Post-LN(Xiong ら "On Layer Normalization in the Transformer Architecture", ICML 2020, arXiv:2002.04745)。Transformer の1ブロックは「アテンション+残差」「FFN(全結合の小ネット)+残差」からなります。Post-LN は残差を足した後に LN をかける元祖の配置(原論文の配置、`x + Sublayer(x)` の後に LN)、Pre-LN はサブ層に入る前に LN をかける配置(`x + Sublayer(LN(x))`)です。Xiong らは理論と実験で、Post-LN は学習初期に出力近くの層で勾配が大きくなりすぎ、学習率のウォームアップ(最初は小さく徐々に上げる)が必須なのに対し、Pre-LN は勾配がよく振る舞いウォームアップ無しでも安定して学習できることを示しました。現代の大規模モデルの多くが Pre-LN を採用するのはこの理由です。

RMSNorm(Zhang & Sennrich, NeurIPS 2019)は、LN から平均の引き算(re-centering)を省き、二乗平均平方根(RMS、各要素を二乗して平均し平方根を取った値)で割るだけに簡略化した変種です。`y = gamma * x / sqrt(mean(x^2) + eps)` の形になります。平均をそろえる re-centering 効果はほとんど学習に寄与せず、分散をそろえる re-scaling 効果が主だ、という観察に基づきます。計算が軽く(平均の計算が不要)、近年の大規模言語モデルで広く採用されています。

## 論文の実験結果(定量データ)
LN 原論文は、RNN 系のタスクで効果を示しました。代表例として、画像説明文の検索(image-text ranking)、質問応答、機械翻訳、手書き生成などで、LN 無しに比べて収束が速まり最終性能も改善することを報告しています。特に attentive reader 型の質問応答で、LN ありは検証誤差の収束が明確に速くなりました。BN が苦手だった RNN において、LN は時間ステップごとに同じ正規化を安定して適用でき、これが効きました。論文は「LN は学習を数倍速く収束させる」ことを複数タスクの学習曲線で示しています。

Xiong ら(Pre-LN vs Post-LN)の実験は示唆に富みます。機械翻訳(WMT)と言語モデルで、Post-LN Transformer はウォームアップを外すと学習が発散または大きく悪化する一方、Pre-LN Transformer はウォームアップ無しでも収束し、学習を大幅に簡略化・高速化できました。アブレーションとして、層が深くなるほど Post-LN の出力層付近の勾配ノルムが大きく不安定になることが理論計算と実測の両方で示され、「深い Transformer ほど Pre-LN の恩恵が大きい」という定量的傾向が確認されています。実用面では、ウォームアップのステップ数調整という面倒なチューニングを Pre-LN が不要にした意義は大きいです。

実用的なアブレーションの観点では、Transformer から LN を完全に除くと学習が極めて不安定になる(発散しやすい)ことが繰り返し報告されており、LN は性能向上の飾りではなく、深い Transformer を学習可能にするための必須部品である、というのが共通理解です。RMSNorm を採用した近年の大規模言語モデルでは、LN と同等の性能を保ちつつ正規化部分の計算を数十%削減できたと報告されており、「平均の引き算は実はほぼ不要」という当初の主張が大規模実験で裏付けられています。

## メリット・トレードオフ・限界
メリットは、バッチサイズに依存せず学習が安定し収束が速まること、推論と学習で同じ式が使え実装が単純なこと、Transformer や可変長系列で扱いやすいことです。分散学習でバッチを複数 GPU に分けても、LN は各サンプル内で完結するため GPU 間の統計同期が要らない利点もあります(BN は同期が必要になりがち)。

トレードオフは、正規化で各特徴の絶対的な大きさという情報がいったん失われる点で、これは gamma/beta(あるいは後述の条件付き正規化の外部注入)で必要な分を復元します。畳み込み主体の画像ネットでは BN のほうが効くこともあり、構造に応じた使い分けが必要です。

限界・課題として、LN は各層で平均・分散の計算と除算が入るため、計算量とメモリ帯域(GPU が値を読み書きする速度の制約)の負担が無視できず、超大規模モデルでは RMSNorm のような簡略版でコストを削る動きがあります。また、どの軸で正規化すべきかは構造依存で、画像では LN を空間全体に取るかチャネルごとに取るかで挙動が変わるなど、正規化軸の最適設計は今も経験則に頼る部分が残ります。なぜ正規化がこれほど学習を助けるのかという理論的理解も、当初の「内部共変量シフトを抑えるから」という説明が後の研究で疑問視され、「損失地形を滑らかにするから」など複数の仮説が並立する未決着の問題です。

## 発展トピック・研究の最前線
条件付き正規化が重要な発展です。AdaLN(Adaptive Layer Normalization)は、gamma と beta を固定の学習パラメータにせず、時刻や条件ベクトルから動的に生成して外から注入します。実装上は LN を `elementwise_affine=False`(gamma/beta を内部に持たない)にしておき、`normed * (1 + gamma) + beta` の形で外部条件由来の (gamma, beta) を与えます。拡散モデル(diffusion model、ノイズを徐々に除去して画像を生成するモデル)や条件付き生成で、ノイズの時刻やクラス条件をモデルに伝える主要な手段として使われています。Peebles & Xie の DiT(Diffusion Transformer, ICCV 2023)は、AdaLN を使った AdaLN-Zero(残差をゼロ初期化して安定化)が他の条件注入法より優れることを ImageNet 生成で定量的に示しました。この発想の源流は FiLM(Feature-wise Linear Modulation, Perez ら, AAAI 2018)で、条件からスケールとシフトを生成して特徴を変調するという考え方を一般化したものです。

近年は、正規化を一切置かずに深い Transformer を安定学習させる試み(正規化フリーの初期化・残差スケーリング、ReZero や Fixup などの系譜)や、正規化と残差接続の相互作用の理論的解析も進んでおり、「なぜ正規化が学習を安定させるのか」の理解そのものが研究の最前線です。さらに、正規化層をシンプルな要素ごとの非線形関数(tanh など)で置き換えても深いネットが学習できるという報告も出ており、正規化の本質が何かをめぐる議論は続いています。
