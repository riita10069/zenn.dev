# 軌道損失の時間重み付け

## ひとことで言うと

軌道は「数秒先までの計画」ですが、すぐ先（0.1秒後）の予測とずっと先（6秒後）の予測を、損失計算で同じ重さに扱うべきかという論点です。「近い未来ほど重く、遠い未来ほど軽く」扱う時間重み付け（temporal weighting）を入れると、当てにくい遠未来の誤差で学習が支配されるのを防ぎ、安全上重要な近未来の精度に学習を集中させられます。

## 直感的な理解

天気予報を思い浮かべてください。今日と明日の予報はよく当たりますが、1週間先の予報はあてになりません。それでも「7日後の予報が外れた」ことを今日の予報と同じ重さで反省していたら、明日の予報の精度を上げる努力がおろそかになります。当てやすい近い予報を確実に当てることに力を注ぐべきです。

運転の軌道予測もこれと同じです。0.1秒後に車がどこにいるかはほぼ一通りに決まりますが、6秒後は信号が変わる・前車が止まる・歩行者が出るなど展開が枝分かれし、本質的に当てられません。しかも車は数十ミリ秒ごとに軌道を作り直すので、実際に従うのは予測の冒頭だけ。遠未来の誤差を近未来と同じ重さで叱ると、モデルは当てられない遠未来を無理に当てようとして近未来を犠牲にします。これを防ぐのが時間重み付けです。

## 基礎: 前提となる概念

軌道予測の不確実性は時間とともに増大する、という性質がこのトピックの土台です。理由は2つあります。

第一に分岐の増大。未来は時間が経つほど可能性が枝分かれします。1秒後の状態はほぼ確定でも、数秒後は環境（他のエージェントの行動、信号）に依存して多数の妥当な未来が生じます。1本の正解で学ぶ模倣学習では、この多数性が「正解が定まらない」誤差として効いてきます。これは [軌道損失の関数選択（Smooth L1 / MSE）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0581_smooth-l1-vs-mse-loss) で触れた mode averaging（複数の妥当な未来の平均に潰れる問題）が、遠未来ほど深刻になるという形でも現れます。

第二に実運用の重要度差。自動運転は高頻度で再計画します。後述する receding horizon（後退ホライズン）という考え方で、長い予測区間を計算しても実際に車へ送るのは最初のごく一部だけ。だから近未来の精度は即座に安全に直結し、遠未来の精度は次の瞬間に上書きされるので相対的に重要度が低いのです。

評価指標 ADE / FDE。軌道予測の精度はよく ADE と FDE で測ります。ADE（Average Displacement Error, 平均変位誤差）は全時刻の位置ずれ（m）の平均、FDE（Final Displacement Error, 最終変位誤差）は最終時刻だけの位置ずれです。FDE は遠未来（終端）を重視する指標なので、後述するように近未来重視の時間重み付けと相性が悪い場合があります。複数候補を出すときは、最良の候補で測る minADE / minFDE（K本出して最も当たった1本で評価）が使われます。

## 仕組みを詳しく

時間重み付けの典型的な実装は、時刻 t に応じて指数的に減衰する重みをかけることです。減衰率を d（0 < d ≤ 1、たとえば 0.95）とすると、

```
d == 1.0 のとき:  weights = [1, 1, 1, ..., 1]       全時刻を平等に
d  < 1.0 のとき:  t = [0, 1, 2, ..., T-1]
                  weights = d ** t                   = [1, d, d², ..., d^(T-1)]
```

d=0.95、10Hz（0.1秒刻み）、64ステップの場合、重みは次のようになります。

| 時刻 t | 実時間 | 重み 0.95^t |
|---|---|---|
| 0 | 0.0s | 1.000 |
| 10 | 1.0s | 0.599 |
| 20 | 2.0s | 0.358 |
| 40 | 4.0s | 0.129 |
| 63 | 6.3s | 0.039 |

すぐ先（t=0）は重み1.0、6.3秒先（t=63）は約0.039と、およそ25分の1に割り引かれます。指数減衰のほか、線形減衰（重み = 1 − t/T）や、近未来だけ重くして中盤以降は一定にするステップ状の重みなど、減衰の形そのものも設計対象です。

損失計算では、時刻ごとの誤差テンソル `[B, T]` にこの重みを掛けてから平均します。

```
weighted_loss = per_timestep_loss * weights        # [B, T] * [T] (broadcast)
loss = weighted_loss.mean()
```

d=1.0 を渡せば全時刻が重み1になり、時間重み付けを無効化（平等扱い）できます。つまりこの機能はオン/オフ可能なハイパーパラメータ（学習前に人間が決める設定値）として扱え、A/B 比較がしやすいのが実務上の利点です。

### 数値例

ある学習ステップで、近未来（t=0〜5）はぴったり当たり、遠未来（t=55〜63）が大きくずれたとします。時間重みなしなら遠未来の大きな誤差で損失全体が膨らみ、勾配は「遠未来を直せ」と強く引っ張られます。時間重みありなら遠未来の誤差は約0.04倍に縮むので、損失も勾配も「まず近未来を完璧に」という方向を優先します。減衰率 d を小さくするほど近未来偏重が強まり、1.0 に近づけるほど平等に戻ります。

### 設計上の注意

重みの正規化をどうするかで損失のスケールが変わります。`weights.sum()` で割って正規化すれば、d を変えても損失の絶対値スケールが揃い、学習率を変えずに済みます。正規化しないと d を小さくするほど損失の総量が小さくなり、実効的な学習率が下がる副作用が出ます。これは見落としやすい落とし穴で、「d を変えたら学習が遅くなった」現象の正体がこの実効学習率の変化であることがあります。

## 手法の系譜と主要論文

強化学習の割引率 γ（Sutton & Barto, Reinforcement Learning: An Introduction）。強化学習では未来の報酬に割引率 γ（典型 0.9〜0.99）を掛け、遠い未来を割り引いて評価します。遠未来は不確実で、近い結果ほど確実かつ重要だから、という動機です。割引は無限ホライズンの報酬和を有限に収束させる数学的役割も持ちます。軌道損失の時間減衰はこの発想を教師あり学習に持ち込んだものと見なせ、減衰率の値域も近いです。

モデル予測制御（MPC）と receding horizon（Mayne et al., Automatica 2000 のサーベイほか）。制御理論の MPC は、長い予測区間（horizon）を計算しても実際に適用するのは最初の1ステップだけで、次の瞬間に再計算します（receding horizon、後退ホライズン）。MPC のコスト関数でも終端コスト（terminal cost）と段階コスト（stage cost）を区別し、区間内の重み付けを設計します。「実際に従うのは予測の冒頭だけ」という前提が、近未来重視の時間重み付けの理論的正当化になります。

軌道予測ベンチマークの慣行（Social LSTM: Alahi et al., CVPR 2016 / Trajectron++: Salzmann et al., ECCV 2020, arXiv:2001.03093 / Wayformer: Nayakanti et al., ICRA 2023, arXiv:2207.05844）。歩行者・車両の軌道予測分野では、学習時は全時刻をほぼ平等に扱いつつ、評価で ADE と FDE を分けて報告するのが一般的です。明示的な時間減衰重みを置かない研究も多いですが、これは評価指標が全時刻（ADE）や終端（FDE）を平等・終端重視で測るためで、近未来重視の運転計画とは前提が異なります。運転計画のように高頻度再計画する用途では、時間重みを入れる設計が自然に支持されます。

不確実性に基づく重み付け（Kendall et al., CVPR 2018, arXiv:1705.07115）。固定の減衰ではなく、各出力の予測分散（不確実性）を学習し、不確実な出力ほど自動的に重みを下げる枠組みです。元はマルチタスク学習で「どのタスクをどれだけ重視するか」を学習する手法ですが、軌道の時刻方向にも転用でき、「当てにくい遠未来は分散が大きいので自然に軽く扱われる」という、固定減衰の理論的な一般化を与えます。

## 論文の実験結果（定量データ）

時間重み付けそのものを単独で切り分けたアブレーションは多くありませんが、関連する定量的知見を挙げます。

軌道予測の精度は時間とともに劣化することが、ベンチマーク数値から読み取れます。たとえば nuScenes や Waymo Open Motion Dataset の予測タスクでは、1秒先の FDE は数十cm〜1m程度でも、6〜8秒先の FDE は数m〜10m規模に膨らむのが普通です。Wayformer（ICRA 2023）は Waymo Open Motion Dataset で当時最高水準の minADE/minFDE を報告しましたが（8秒予測で minADE はおよそ 0.5m台）、それでも長ホライズン側で誤差が大きく増える傾向は変わりません。この「遠未来ほど誤差が大きい」事実が、時間重み付けの動機を数値で裏づけます。誤差がホライズン後半で急増する以上、平等重みのままだと損失の大半を後半が占めることになります。

強化学習における割引率の効果は古くから定量化されており、γ を 1 に近づけると長期計画性が増す一方で学習の分散（バリアンス）が増え収束が遅くなる、小さくすると近視眼的（myopic）になるが学習が安定する、というトレードオフが一般に観察されます。軌道損失の減衰率 d も同型のトレードオフを持ち、d を小さくしすぎると「先の交差点を見越した早めの減速」のような長期計画性が落ちる、という定性的な帰結が予想されます。

評価指標との不整合に注意が必要です。もしベンチマークが FDE（終端誤差）を重視する場合、時間重みで近未来に寄せた学習は終端精度を下げ、指標スコア上は不利に出ます。逆に閉ループ（実際に走らせる）評価や安全性指標では、近未来重視が有利に働きやすい。つまり時間重み付けの是非は「どの指標で評価されるか」に強く依存し、開ループ指標と閉ループ性能の乖離（AD-MLP, Zhai et al., 2023, arXiv:2305.10430 が、自車の過去状態だけで開ループ L2 が当たってしまうと指摘）とも絡む論点です。学習で何を重視するかと、評価で何が測られるかを一致させないと、改善したつもりが指標上は後退する、という事故が起こりえます。

## メリット・トレードオフ・限界

メリット
- 当てにくい遠未来の誤差で勾配が支配されるのを防ぎ、安全上重要な近未来の精度に学習資源を集中できる。
- receding horizon で高頻度に再計画する実運用（実際に従うのは冒頭だけ）と整合する。
- d=1.0 で無効化でき、効果を実験的に検証しやすい。

トレードオフ・限界
- 減衰を強くしすぎると遠未来をほぼ学ばず、長期的な計画性（先を見越した減速・車線変更）が落ちる。
- 評価指標が遠未来も平等に測る（ADE）あるいは終端重視（FDE）の場合、指標スコアが不利になりうる。
- 最適な減衰率はデータセット・タスク・評価指標依存で、追加のハイパーパラメータ調整が必要。正規化を忘れると実効学習率が変わる副作用もある。

研究上の未解決課題として、時間重み付けと評価指標の整合（学習の重みと評価の重みをどう一致させるか）、不確実性に応じて重みを動的に決める方法（固定の指数減衰ではなく予測分散に基づく重み付け）、そしてマルチモーダル予測のもとで「遠未来は1本に潰さず分布で評価する」設計との組み合わせが挙げられます。

## 発展トピック・研究の最前線

固定の指数減衰の代わりに、予測の不確実性を陽に推定し、確からしさに応じて重みを変える方向があります。たとえば各時刻の予測分散を出し、分散が大きい（当てにくい）時刻は損失を割り引く、不確実性重み付け（uncertainty weighting, Kendall et al., CVPR 2018 のマルチタスク重み付けに着想）です。これは固定減衰を「データから学ぶ減衰」へ一般化したものと見なせます。また遠未来を1本の軌道で叱るのをやめ、複数候補の分布として評価する（min-of-N 損失や混合分布の負対数尤度）ことで、遠未来の多峰性を時間重みとは別の方法で扱う研究も進んでいます。さらに拡散モデルや自己回帰トークン生成で軌道を作る最近の手法では、損失が交差エントロピーやスコアマッチングに変わるため、時間重み付けもトークン位置ごとの重みとして再解釈されつつあります。長期計画性を保ちながら近未来精度を確保するには、「近未来は厳密に、遠未来は分布として緩く」測る損失設計が一つの有望な方向です。
