# Deformableクロスアテンションのスパースサンプリング

## ひとことで言うと

deformable(変形可能)クロスアテンションは、広大な特徴マップの全マスを見るのではなく、クエリごとに「ここを見るべき」という少数の点を学習で選び、そこだけをサンプリングして集約する仕組みです。これにより計算量とメモリを劇的に減らせます。代償として、選ばれなかったマスは「そのクエリにとって存在しないのと同じ」になり、特定の入力(例えば一部のカメラ)が最終出力に影響しないことが起こりえます。

## 直感的な理解

巨大な図書館で調べ物をする場面を想像してください。すべての本を1ページずつ全部読む(密なアテンション)のは確実ですが、時間が無限にかかります。現実には「目次を見て、関連しそうな数冊の、特定の数ページだけ」を読みます(疎なサンプリング)。後者は圧倒的に速い一方、読まなかったページに重要な情報があっても気づけません。

deformable アテンションはこの「数ページだけ読む」を学習でやる仕組みです。各クエリ(質問者)が「まずこの辺り(参照点)を見て、内容に応じて少しずらした数点だけ精読する」と決めます。鳥瞰図(BEV)のグリッドのように数万マスもある対象では、全マスを見る密なアテンションは計算もメモリも破綻します。だから疎にサンプリングする。これは合理的な設計判断ですが、「読まなかったページ=未サンプリングのマス」には情報が届かない、という原理的な副作用がついてきます。あるカメラの映像がたまたま未サンプリングのマスにしか反映されないと、そのカメラをいじっても出力が変わらない、ということが起きるのです。

## 基礎: 前提となる概念

アテンション(attention): 多数の情報の中から、今必要な箇所に重みをつけて参照する仕組み。クエリ(質問する側)、キー(各位置の見出し)、バリュー(各位置の中身)からなり、クエリとキーの一致度で重みを決め、バリューを加重平均します。

密なアテンション(dense attention): クエリが全位置(N個)を見る方式。自己注意では計算量がクエリ数 × N、メモリも N×N のアテンション行列を持つため、画像やBEVのように N が数万になると爆発します。Transformerの自己注意はこの形です。

クロスアテンション(cross-attention): クエリ側とキー・バリュー側が別物の場合のアテンション。BEVでは「BEVクエリ(出力したいマス)」が「カメラ特徴(入力)」に注目して情報を引きます。

deformable(変形可能): もともとは Deformable Convolution(Dai et al., ICCV 2017)に由来する概念で、「固定の格子位置ではなく、学習したオフセットだけずらした位置を参照する」発想。これをアテンションに持ち込んだのが deformable attention です。

サンプリング点(sampling points): 各クエリが実際に参照する少数の座標。num_points 個(例えば4点や8点)だけ持ち、全位置ではなくここだけを見ます。

参照点(reference point): 各クエリが「だいたいこの辺り」と最初に定める基準座標。BEVでは、3Dの基準点をカメラ画像へ射影して得た2D座標などを使います。

双線形補間(bilinear interpolation): 整数格子の特徴から、任意の小数座標の特徴を、周囲4点の重み付き平均で推定する操作。サンプリング座標が格子の隙間に来ても特徴を取り出せます。実装では grid_sample 関数が使われます。微分可能なので、サンプリング位置に関する勾配が流れ、オフセットを学習できます。

## 仕組みを詳しく

deformable クロスアテンションの典型的な処理を、ステップで分解します。

```python
offsets = sampling_offsets(queries)            # クエリから「どこをずらして見るか」を予測
offsets = offsets.reshape(B, N, num_points, 2) # (B, N, P, 2): 各クエリ P 点分の (dx, dy)
offsets = offsets * offset_scale               # 縮小して学習を安定化(後述)

sample_locs = ref_points + offsets             # 参照点 + オフセット = サンプリング座標
sample_grid = sample_locs * 2 - 1              # [0,1] 正規化座標を grid_sample の [-1,1] へ
sampled = F.grid_sample(feat, sample_grid, mode="bilinear", align_corners=False)

attn_w = softmax(attention_weights(queries))   # 各サンプリング点の重み
out = (sampled * attn_w).sum(dim=points)        # 少数点を加重平均してクエリ出力に
```

1. 参照点。各クエリ(出力したいBEVマスや検出クエリ)が「だいたいこの辺りを見る」基準位置 `ref_points` を持つ。BEVなら3D基準点をカメラへ射影した2D座標。
2. オフセット予測。クエリの内容に応じて、基準位置から「どれだけずらして見るか」を `sampling_offsets`(小さな線形層)が予測する。形状 `(B, N, num_points, 2)`。N はクエリ数、num_points は1クエリあたりのサンプリング点数(少数、例えば4〜8)。
3. スケーリング。予測オフセットに `offset_scale`(例えば0.1)を掛けて小さくする。学習初期に大きくずれて特徴マップの外へ飛び出し、勾配が消えて発散するのを防ぐ安全装置です(詳細は [offset_scale バリデータ — ハイパーパラメータの入力検証と早期失敗](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0991_offset-scale-validator))。
4. サンプリング。参照点 + オフセットの座標から `grid_sample` で双線形補間して特徴を取り出す。
5. 集約。取り出した少数点の特徴を、別途予測したアテンション重みで加重平均し、そのクエリの出力にする。ここで重みは softmax で正規化され、クエリとキーの内積から計算する代わりにクエリから直接予測する点が、標準アテンションとの大きな違いです。これによりキー全体との内積が不要になり、計算が N に依存しなくなります。

ここで決定的なのは num_points が「少数」だという点です。N個のクエリそれぞれが、特徴マップ全体のうちごく一部の座標しか触りません。計算量は密な「クエリ数 × 全マス N」から「クエリ数 × num_points」へ落ちます。BEVのマス数を例えば 200×200 = 40,000 とすると、密なら各クエリが4万マスを見ますが、deformable なら数点。これは2〜4桁の削減です。multi-scale 特徴を使う場合も、各スケールから少数点ずつサンプリングするだけで済むため、高解像度特徴を扱っても計算が破綻しません。

副作用が本ノートの核心です。「あるカメラビュー v が、どのクエリのサンプリング点にも投影されない」状況が起こりえます。その場合、ビュー v は中間の融合特徴マップには寄与していても、deformable サンプリングを使う下流(プランナや検出ヘッド)の出力には測定可能な変化を与えません。数値例でいえば、200×200 のグリッドのうち下流が実際に触るのが数百〜数千点だとすると、ある周辺カメラの担当領域がまるごと未サンプリングになることがあります。特に学習前のランダム初期化では、オフセットが意味のある場所へまだ収束しておらず、この見落としが顕著です。これが、最終出力でビュー寄与を測るテストを不安定にする理由です([融合特徴レベルでのビュー寄与テスト](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0963_fused-feature-level-view-contribution-test))。

before/after を表で整理します。

| | 密なアテンション | deformable(疎) |
|---|---|---|
| 各クエリが見るマス | 全 N マス | 学習で選んだ num_points 点 |
| 計算量 | クエリ数 × N | クエリ数 × num_points(N に非依存) |
| メモリ | アテンション行列 N×N で大 | サンプリング点分のみで小 |
| 未参照マスの存在 | なし | あり(本ノートの核心) |
| 入力の出力への寄与 | 必ず反映 | 反映されないことがある |

なぜこの設計が選ばれるのか。密なアテンションは N が大きいと計算量が N の2乗(自己注意)あるいは N に比例(クロス)で増え、高解像度の知覚では現実的に回りません。deformable は「重要な場所は少数で、それを学習で当てられる」という仮説に賭けて、計算を N に依存しない形へ落とします。多くのタスクでこの賭けは当たりますが、賭けである以上「見落とし」は原理的に残ります。さらに、収束が速いという副次効果もあります。密なアテンションは初期に全位置へ薄く広がった注意から少しずつ鋭くしていくため学習に時間がかかりますが、deformable は最初から少数点に絞るため、勾配が集中して早く学習が進みます。

## 手法の系譜と主要論文

疎サンプリングの系譜は、畳み込みの変形からアテンションの変形へと進みました。

Deformable ConvNets (Dai et al., ICCV 2017, arXiv:1703.06211)、DCNv2 (Zhu et al., CVPR 2019, arXiv:1811.11168)。畳み込みのサンプリング位置を固定格子から学習オフセットでずらす発想を導入。「参照位置を内容に応じて動かす」という deformable の原型です。DCNv2 はさらに各サンプリング点に変調重み(modulation)を加え、不要な点を抑制できるようにしました。

DETR (Carion et al., ECCV 2020, arXiv:2005.12872)。Transformerで物体検出をエンドツーエンド化。ただし密なアテンションのため収束が非常に遅く(数百エポック)、高解像度特徴の扱いが重いという課題を残しました。これが疎化の動機です。

Deformable DETR (Zhu et al., ICLR 2021, arXiv:2010.04159)。各クエリが全位置ではなく学習した少数のサンプリング点だけを見る deformable attention を提案。DETRの収束を約10分の1に短縮し、計算とメモリを大幅削減しました。multi-scale 特徴も自然に扱えます。本ノートの疎サンプリングの直接の理論的基盤であり、同時にその限界(見落とし)もここに内在します。

BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270)。deformable attention を多カメラBEVへ適用。空間クロスアテンションでBEVクエリが各カメラの射影位置周辺の少数点だけをサンプリングし、時間クロスアテンションで過去BEVを位置合わせして融合します。多カメラ統合を現実的な計算量で実現した代表例で、BEV系プランナや知覚の疎サンプリングはこの系譜に連なります。

その後、Sparse4D (Lin et al., 2022以降, arXiv:2211.10581) や PETR系(Liu et al., ECCV 2022, arXiv:2203.05625)など、疎なクエリでさらに効率化する流れが続きます。Sparse4D は3Dアンカーから複数点をサンプリングして時系列に集約し、PETR は3D位置エンコーディングを画像特徴に注入することで明示的なサンプリングを避けつつ密な参照を保ちます。系譜の一貫した教訓は、「計算効率のために参照を疎にすると、必ずどこかを見落とす。だから疎化の利得と見落としのリスクは設計上トレードオフする」ということです。

## 論文の実験結果(定量データ)

Deformable DETR は COCO 物体検出(画像認識の標準ベンチマーク、約12万枚の学習画像、指標は mAP = mean Average Precision、検出の適合率と再現率を統合した0〜100の値、高いほど良い)で評価されます。報告によると、Deformable DETR は DETR とおおむね同等以上の mAP(およそ43〜46程度の水準)を達成しつつ、学習エポックを500級から50級へ約10分の1に短縮しました。さらに小物体の検出(multi-scale 特徴を疎に扱えるため)で明確な改善を示しました。数値の意味は、mAP が同等なら「精度を落とさず」、エポックが10分の1なら「学習コストが10分の1」ということです。小物体は画素数が少なく高解像度特徴が必要ですが、密アテンションでは高解像度を扱うと計算が破綻するため、疎化が小物体精度の改善に直結しました。アブレーションでは、サンプリング点数 num_points を増やすと精度がわずかに上がる一方で計算が増え、ある点数(報告では4点前後)で頭打ちになることが示されています。「少数点で十分当てられる」という仮説の定量的裏付けです。

BEVFormer は nuScenes(6カメラ、約1000シーン)の3D物体検出で評価され、主指標は NDS(nuScenes Detection Score、検出精度と位置・速度・属性誤差を統合した0〜1、高いほど良い)と mAP です。報告では NDS およそ 0.517、mAP およそ 0.416 を達成し、当時のカメラのみ手法を上回りました。アブレーションでは、deformable な空間クロスアテンション(全画像を密に見るのではなく射影位置周辺の少数点を見る)が、計算量を抑えつつ精度を保つことを示しています。また時間クロスアテンションを外すと速度推定誤差 mAVE(mean Average Velocity Error、低いほど良い)が悪化し、過去フレームの疎な参照が動き推定に効くことが確認されています。

これらが示すのは、第一に、疎サンプリングは精度をほぼ落とさず計算・学習コストを桁で削減できること。第二に、サンプリング点数には収穫逓減(diminishing returns)があり、少数で十分なこと。第三に、しかし「見る場所を絞る」設計である以上、絞られた領域の情報は原理的に下流へ届かず、これがテストやデバッグで「特定入力が効いていないように見える」現象を生むこと、です。効率と網羅性は同じ設計判断の両面なのです。

## メリット・トレードオフ・限界

メリット。巨大な特徴マップ(高解像度画像や数万マスのBEV)でもアテンションを現実的な計算量・メモリで回せます。各クエリが内容に応じて見る場所を学習で適応的に選べるため、固定窓の畳み込みより柔軟です。収束も速く、学習コストを大きく削減できます。

トレードオフと限界。第一に、選ばれないマスは無視されるため、特定の入力(カメラビューや領域)の寄与が下流出力に出ないことがあり、これが偽の「使われていない」判定を招きます。第二に、サンプリング点数や `offset_scale` の設定が学習の安定性・網羅性に影響します。点数が少なすぎれば見落としが増え、`offset_scale` が大きすぎれば初期に発散し、小さすぎれば探索範囲が狭くなります。第三に、テスト・デバッグ時に「融合の問題」と「サンプリングの問題」を意識的に切り分ける必要があり、観測点を中間特徴に置く工夫が要ります([融合特徴レベルでのビュー寄与テスト](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0963_fused-feature-level-view-contribution-test))。第四に、参照点の精度(BEVなら射影行列のキャリブレーション)がサンプリング位置を左右し、キャリブレーション誤差がそのまま見る場所のズレになります。

未解決の課題として、見落としを定量的に監視・保証する仕組み(網羅性メトリクス)、安全に関わる領域を必ずサンプリングさせる制約付き deformable、点数を入力に応じて動的に変える適応的サンプリングなどが、研究上のオープンな方向として残っています。安全クリティカルな自動運転では「重要な領域を確実に見ている」ことの保証が求められるため、疎化と保証の両立は応用上の大きな論点です。

## 発展トピック・研究の最前線

疎サンプリングの効率化はその後も進み、Sparse4D 系はクエリ自体を疎にして時系列の3D検出を効率化、PETR系は位置エンコーディングで明示的なサンプリングを避けつつ3D空間を表現します。BEV以外でも、画像生成・点群処理・動画理解で「全位置ではなく学習した少数点を見る」発想が広く使われています。一方で、deformable の見落とし問題に対しては、サンプリングのエントロピーや被覆率を観測して網羅性を測る試みや、安全クリティカルな領域へのサンプリングを誘導する手法が探られています。効率(疎にする)と網羅性(見落とさない)のバランスをどう自動調整するかは、知覚モデルが安全性要件を満たす上での中心的な研究課題であり続けています。
