# Waymo Open End-to-End Driving データセット

## ひとことで言うと

Waymo(Google 系の自動運転企業)が公開した、end-to-end 自動運転モデル(知覚・予測・計画を1本のニューラルネットでつなぎ、カメラ画像から直接「どう走るか」を出すモデル)を学習・評価するためのデータセットです。車体を取り囲む複数カメラの映像、自車がどう動いてきたかの履歴、「次の交差点で右折せよ」といった高レベルの行き先指示(ルーティングコマンド)、そして人間ドライバが実際に走った正解の将来軌道がセットになっています。「カメラ画像と行き先指示を入れたら、人間が運転したのと同じ将来経路を出せるか」を直接測れるのが最大の特徴です。

## 直感的な理解

end-to-end 自動運転モデルを訓練するには、教師あり学習(正解つきのデータからモデルを学ばせる方式)の原則どおり「入力」と「正解」のペアが大量に要ります。入力はカメラ映像など車が外界から得る情報、正解は「人間ならこの後どこを通ったか」という将来の経路です。モデルに入力を見せ、人間の経路を真似させる(模倣学習、imitation learning)のが基本の枠組みです。

ここで問題になるのが、世の中の運転データセットはそれぞれ欠けているものがある点です。あるデータセットは正解の将来軌道はあるが他車の3次元位置(バウンディングボックス、物体を囲む立方体の枠)が付いていないので「予測経路が他車と衝突しないか」を測れない。別のデータセットは多様な実映像はあるが正解の将来軌道がそもそも無く、モデルが落ちずに動くかの確認(forward pass、入力から出力まで一度計算を通すこと)にしか使えない。

研究で本当に欲しいのは「正解の将来軌道が付いていて、予測のズレを数値で計算でき、かつ規模が大きく多様」なデータセットです。Waymo が 2025 年の E2E Driving Challenge 向けに公開したこのデータセットは、まさにその穴を埋めるために設計されています。身近な例で言えば、教習所で教官の運転(正解)を録画し、生徒の運転(モデル予測)とどれだけズレたかを採点するための、巨大で標準化された問題集だと思ってください。Waymo が特に強調するのは、ありふれた直進ではなく「ロングテール」と呼ばれる、めったに起きないが対処を誤ると致命的な場面(緊急車両、複雑な工事帯、横断する動物、無秩序な交差点など)を意図的に多く集めている点です。

## 基礎: 前提となる概念

このトピックを理解するために、いくつかの言葉を噛み砕きます。

- 軌道(trajectory): 時間とともに変化する自車の位置の列。たとえば 0.5 秒刻みで今後 5 秒先までの (x, y) 座標を並べたもの。「これからどこを通るか」を表す。座標系は通常、現在の自車を原点にした自車中心座標(ego-centric coordinate)で、x が進行方向、y が横方向。
- ego(自車)と ego state: ego は「自分の車」を指す業界用語。ego state(自車状態)は速度・加速度・ヨー角(車体の向き、回転角)・ヨーレート(向きの変化率)など、自車の運動状態を表す数値の集まり。
- ルーティングコマンド(routing command): カーナビが出すような高レベルの指示。「直進」「右折」「左車線へ移れ」など。カメラ画像だけでは交差点で曲がるべきか直進すべきか決められない(分岐の曖昧さ、behavioral ambiguity)ため、人間の意図を補う入力として与える。離散ラベルで与える方式と、数十メートル先の目標点(goal point)の座標で与える方式がある。
- open-loop 評価と closed-loop 評価: open-loop は「正解の過去」を入力に与え、未来を1回だけ予測してそのズレを測る方式。モデルの予測は実際の車には反映されない。closed-loop は予測した行動で実際にシミュレータ上の車を動かし、その結果また観測してまた予測する、を繰り返す方式。後者の方が現実に近いが構築コストが高い。
- ADE / FDE: 予測軌道の正確さを測る指標。ADE(Average Displacement Error、平均変位誤差)は予測点と正解点の距離を全時刻で平均したもの。FDE(Final Displacement Error、最終変位誤差)は予測の終点と正解の終点だけの距離。どちらも単位はメートルで、小さいほど良い。詳しくは [軌跡品質メトリクス](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0730_trajectory-quality-metrics) を参照。
- RFS(Rater Feedback Score): Waymo の E2E チャレンジで採用された独自の主指標。後述するが、単純な幾何的ズレ(ADE)ではなく「人間の評価者が許容できるか」を反映した点数で、複数の正解を許す設計になっている。

## 仕組みを詳しく

このデータセットが1サンプルとして提供する典型的な中身を、E2E モデルの入力に対応づけて段階的に説明します。多くの E2E モデルは入力を次のようなテンソル(数値が格子状に並んだ多次元配列)として受け取ります。

1. マルチカメラ画像。車体まわりに配置された複数カメラの映像。Waymo の車両は前方・側方・後方を含むおよそ8視点をカバーします。前処理として各画像を一定解像度(たとえば 256×256 ピクセル)にリサイズし、ImageNet 統計で正規化(各色チャンネルから学習済みデータセットの平均を引き標準偏差で割る操作。事前学習済みバックボーンが期待する入力分布にそろえる)してテンソル化します。形は `[B, V, 3, H, W]`。B はバッチサイズ(一度に処理する場面数)、V はビュー数(カメラ台数)、3 は RGB の3色、H・W は画像の縦横。たとえば 8 カメラ・256×256 なら `[B, 8, 3, 256, 256]`。

2. 自車状態履歴(ego state history)。過去数フレームにわたる自車の速度・加速度・ヨーなどの時系列。形は `[B, T, D]`(T は過去の時刻数、D は状態ベクトルの次元)。これは「今どれくらいの速さで、どう動いてきたか」という運動の文脈で、将来軌道の質を強く左右します。実際、ego state だけでもそれなりに良い軌道が出てしまうことが知られており(後述する AD-MLP)、カメラを本当に活かせているかの検証が欠かせません。

3. ルーティングコマンド。高レベルの行き先指示。離散ラベル(直進/左折/右折のいずれか)で与える方式と、数十メートル先の目標点の座標で与える方式があります。これをモデル内部でどう埋め込むか(離散ならワンホット表現や埋め込みベクトル、目標点なら座標を直接特徴に連結)は設計上の選択になります。

4. 正解将来軌道(ground-truth future trajectory)。人間ドライバが実際に走った将来の位置列 `[B, F, 2]`(F は予測する将来時刻数、2 は x と y)。Waymo の設定では将来 5 秒程度を予測対象とします。これが教師信号で、モデル予測との距離が ADE/FDE になります。なお、モデルによっては軌道を直接 (x, y) で出さず、(加速度, 曲率) のような制御量で出すものもあり、その場合は評価時に積分して (x, y) に変換してから正解と比較します。

評価の流れを ASCII 図で示すと次のようになります。

```
[8カメラ画像] ─┐
[自車状態履歴] ─┼─→ [E2Eモデル] ─→ [予測軌道 (x,y)列]
[ルーティング ] ─┘                          │
                                            ↓ L2距離を時刻ごとに計算 / RFSで採点
[正解将来軌道 (x,y)列] ──────────────────→ ADE / FDE / RFS
```

なぜこの最小構成(カメラ + ルーティング + 正解軌道)なのか。E2E モデルの評価思想は「中間の物体検出精度ではなく、最終的に出てくる運転経路の良さで測るべき」というものです。従来のモジュラー型(知覚→予測→計画を別々のモジュールで作る方式)では各段の精度を個別に測れましたが、E2E では中間表現が陽に存在しないこともあるため、最終出力の軌道が人間とどれだけ一致するか(模倣の質)を直接ベンチする設計になっています。

### RFS(Rater Feedback Score)という工夫

単純な ADE には根本的な弱点があります。運転には「正解が一つではない」場面が多いことです。前の遅い車を追い越すとき、左から抜くか右から抜くか、どちらも安全で妥当です。しかし人間ドライバが左から抜いた1本の軌道だけを正解とすると、右から抜くモデルは「大きく外した」と不当に減点されます。これを multi-modality(複数の妥当な未来が並存する性質)の問題と呼びます。

Waymo の E2E チャレンジは、これに対処するため Rater Feedback Score を導入しました。人間の評価者(rater)があらかじめ複数の妥当な軌道を許容範囲つきで用意し、モデルの予測がその許容範囲にどれだけ収まるかで採点します。許容ゾーンの中に入れば高得点、安全上問題のある逸脱(逆走、車線逸脱、衝突方向)は強く減点されます。つまり「人間の1本に幾何的に近いか」ではなく「人間の評価者が許せる運転か」を測る、より運転の本質に近い指標です。

## 手法の系譜と主要論文

軌道予測・計画ベンチマークの系譜を時系列で描きます。

- nuScenes(Caesar et al., CVPR 2020, arXiv:1903.11027)。Motional 社が公開した大規模マルチモーダルデータセット。6 カメラ・LiDAR・レーダ・HD マップ・3D 物体アノテーションを 1000 シーン(各 20 秒)に付与。これが後年の planning ベンチの事実上の標準入力源になりました。動機は「都市部の多様なシーンを多センサで網羅し、知覚から予測までを支える」こと。

- ST-P3(Hu et al., ECCV 2022, arXiv:2207.07601)と UniAD(Hu et al., CVPR 2023, arXiv:2212.10156, Best Paper)。nuScenes 上で「カメラ画像から直接 planning(将来軌道)まで」を測る枠組みを確立しました。UniAD は知覚・予測・計画を1つのネットワークに統合し、planning の L2 誤差(ADE に相当)と衝突率(predicted trajectory が他車のボックスと重なる割合)を主指標に据えました。これにより「ADE/FDE + 衝突率」という評価のテンプレートが普及します。

- VAD(Jiang et al., ICCV 2023, arXiv:2303.12077)。シーンをベクトル(車線や物体を点列・ベクトルで表現)で扱い計算を軽くしつつ、UniAD と同等以上の planning 性能を出しました。

- AD-MLP(Zhai et al., 2023, arXiv:2305.10430)。極めて重要な批判的研究で、「カメラを一切使わず、自車の過去状態(ego status)だけを多層パーセプトロン(MLP、最も単純な全結合ニューラルネット)に入れるだけで、nuScenes の planning ADE が当時の最先端 E2E 手法に匹敵してしまう」ことを示しました。理由は、運転は慣性が支配的で「直前の速度・加速度をそのまま延長する」だけで多くの場面が当たるから。これは「ADE が低い=知覚を活かしている」とは限らないという警鐘で、以降のベンチでは ego-status だけのベースラインとの比較が必須とされました。BEV-Planner(Li et al., 2024, arXiv:2312.03031)も同様に nuScenes open-loop 評価の偏りを定量的に示しています。

- Waymo Open Dataset E2E Driving Challenge(2025)。上記の流れを、より大規模で地域・状況の多様性が高い Waymo データに拡張した公開ベンチマーク。カメラ + 自車状態 + ルーティングから将来軌道を予測し、RFS で順位づけします。nuScenes より規模・多様性が大きく、かつロングテール場面を意図的に集めているため、単純な慣性外挿(直前の動きの延長)では当たらず、知覚・ルーティングを本当に使えているかを問う場として機能します。

注: このデータセットの正確なカメラ台数の内訳、サンプリングレート(何 Hz で記録されるか)、提供フォーマット、評価サーバの指標定義は公式ドキュメントが最終的な正となります。本ノートは E2E 運転データセット一般の構成と公開ベンチの慣行に基づいて記述しています。

## 論文の実験結果(定量データ)

具体的な数値の感覚をつかむため、同系統の planning ベンチで広く引用される nuScenes 上の代表値を挙げます(報告値は論文・設定により幅があります)。

- UniAD(CVPR 2023)は nuScenes planning で平均 L2 誤差(1秒・2秒・3秒先の平均)およそ 1.0 m 前後、平均衝突率およそ 0.3% 前後を報告しました。ここで L2 誤差 1.0 m とは「3 秒先までを平均して予測位置が正解から約 1 メートルずれる」という意味です。市街地走行で 1 メートルのズレは車線内に収まるかどうかに関わる無視できない差で、0.1 m 単位の改善が論文の貢献として競われます。

- VAD(ICCV 2023)は UniAD と同等以下の L2 誤差(報告でおよそ 0.7〜0.9 m 台)を、推論速度を大幅に上げつつ達成したと報告しました。「同じ正確さをより速く」が貢献の核です。

- AD-MLP(2023)のアブレーション(ある要素を抜いて影響を測る実験)が衝撃的でした。カメラ・LiDAR・地図をすべて捨て、自車の過去状態だけを入力にしても、nuScenes の平均 L2 がおよそ 0.3〜0.5 m 台と、当時のカメラ入力 E2E 手法より良い値が出た、と報告されています。これは「nuScenes の open-loop 評価では慣性で当たってしまい、知覚の寄与が見えにくい」ことを定量的に暴いたもので、その後のベンチ設計を変えました。

Waymo E2E チャレンジでは、こうした教訓を踏まえ、より多様で「単純な慣性外挿では当たらない」ロングテール場面を含め、かつ RFS で複数正解を許すことで、知覚・ルーティングの活用度をより公正かつ厳しく評価しようとしています。2025 年チャレンジの規模は、報告によれば数千シーン(各シーンが数秒の運転断片)規模で、上位手法は RFS のスコアを競いました。実際のリーダーボード上位手法の数値は公式の最新結果を参照してください。指標の意味は次の通りです。ADE が小さい=平均的に正解経路に近い、FDE が小さい=遠い未来でもズレない、衝突率が低い=出した経路が他車と物理的にぶつからない、RFS が高い=人間評価者が許容できる運転に近い、です。

## メリット・トレードオフ・限界

メリット

- 正解の将来軌道が付くため、ADE/FDE/RFS を直接計算できる。正解軌道が無いデータセットでは forward pass の成否しか分からないのに対し、予測の正確さを数値化できる。
- マルチカメラ + 自車履歴 + ルーティングと、E2E モデルが必要とする入力が一通りそろう。
- 規模が大きく地域・状況が多様で、かつロングテール場面を意図的に集めているため、汎化性能(訓練時に見ていない場面への強さ)の評価に向く。
- 公開ベンチなので他手法と同じ土俵で比較でき、再現性・公平性が高い。
- RFS により multi-modality(正解が一つでない)問題を緩和し、ADE 単独より運転の本質に近い評価ができる。

トレードオフ・限界

- 本質的に open-loop 中心の評価では、予測が実際の車に反映されないため誤差の蓄積(下記)を捉えにくく、実走行(closed-loop)性能を過大評価しがち。
- 各モデルの入力フォーマット(カメラ台数・解像度・正規化、ルーティングの与え方)に合わせる前処理・パーサ実装が必要で、データセット間の差異を吸収する手間が残る。
- 軌道を制御量(加速度・曲率)で出すモデルでは積分時の初速の取り方で誤差が変わるなど、評価実装の細部が数値に効く。
- RFS は人間評価者の用意した許容ゾーンに依存するため、ゾーンの設計品質が指標の妥当性を左右する。
- 利用にはライセンス・規約の確認が必要(研究用途の条件など)。

未解決の課題として、open-loop の幾何誤差と closed-loop の安全性・快適性の相関が弱い問題は本質的に残っており、シミュレータでの閉ループ評価や、衝突率・違反率といった安全指標との併用が研究の焦点になっています。

## 発展トピック・研究の最前線

- closed-loop ベンチへの移行。CARLA(オープンソースの運転シミュレータ)や nuPlan、Waymo のシミュレーション環境を使い、予測した行動で実際に車を動かして評価する流れが強まっています。open-loop ADE だけでは見抜けない「誤差の蓄積(時間とともにズレが雪だるま式に増える現象。compounding error)」を捉えられます。模倣学習特有の covariate shift(モデルが少しズレた状態に入ると訓練で見ていない状況に陥り、さらにズレる連鎖)は closed-loop でしか露見しません。
- マルチモーダル軌道予測。1つの正解に回帰するのではなく「右折する/直進する」など複数の可能な未来を確率つきで出す方向。minADE(複数候補のうち最も当たったものの ADE)などの指標が併用され、RFS とも親和性が高い。
- 大規模基盤モデル・VLM(視覚言語モデル、画像と言語を同時に扱うモデル)の活用。シーンを言語で説明させたり、ルーティングを自然言語で与える研究が進み、Waymo の EMMA(End-to-End Multimodal Model for Autonomous driving, 2024, arXiv:2410.23262)のように大規模 VLM をそのまま運転に転用する研究も登場しました。Waymo のような大規模データがその学習源になっています。
