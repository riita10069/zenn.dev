# CvT (Convolutional Vision Transformer)

## ひとことで言うと
CvT は、画像を理解する深層学習モデルの一種です。文章翻訳などで成功した Transformer(トランスフォーマー)という仕組みを画像に応用した ViT(Vision Transformer)に、昔ながらの画像処理技術である「畳み込み(convolution)」を二か所に組み込んだものです。これにより、ViT の弱点だった「学習に巨大なデータが必要」「大きな画像で計算が重い」という二つの問題を同時に和らげ、ImageNet-1k という比較的小さなデータだけで高精度を出せるようにしました。研究の系譜の中では「畳み込みと自己注意のハイブリッド」という大きな流れの初期を代表する一つです。

## 直感的な理解
なぜ畳み込みを Transformer に入れたいのか。たとえてみます。Transformer の自己注意は「教室の全員が、他の全員と一斉に会話できる仕組み」です。誰とでも直接話せるので、遠くの席の人(画像でいえば遠く離れた領域)とも一発でつながれます。これは強力です。しかし、隣の席の人と話すのと、教室の反対側の人と話すのを、最初は完全に同じ重みで扱います。つまり「隣同士は似た意味を持ちやすい」という、画像にとって当たり前の常識を最初は知りません。だから大量の例を見せて、その常識をゼロから学ばせる必要があります。これが「データ飢餓(data-hungry)」の正体です。

一方、畳み込みは「自分の机の周り 3x3 マスの近所だけを見る」やり方です。最初から「近くが大事」という前提を持っています。だから少ない例でもすぐに筋のよい特徴をつかめます。CvT のアイデアは単純で「Transformer の全員会話の強みは残しつつ、入口と途中に畳み込みを置いて、近所重視という常識をタダで持たせよう」というものです。これにより、巨大データを用意しなくても、画像の自然な構造を最初から活用できます。

もう一段だけ踏み込んでおきます。ViT が画像を「重ならないパッチに切る」やり方は、隣り合うパッチの境界で情報が断ち切られるという副作用を持ちます。たとえば一本のエッジが二つのパッチの境界をまたいで走っている場合、ViT は最初の段階ではその連続性を保持しません。自己注意で後から関係を学べはしますが、最初から構造として持っている畳み込みのほうが、この種の連続性を扱うには素直です。CvT が「重なりを持つ畳み込みでトークンを作る」のは、まさにこの境界断絶を埋めるための処置だと理解すると腑に落ちます。

## 基礎: 前提となる概念
順に用語を噛み砕きます。

畳み込み(convolution)とは、画像の小さな窓(たとえば 3x3 ピクセル)を順にスライドさせ、その窓内のピクセルだけを見て一つの特徴値を計算する処理です。窓が小さいので「近所のピクセル同士の関係」だけを捉えます。この「近くだけを見る性質」を局所性(locality)と呼びます。窓の中の重み(カーネル、フィルタとも言う)は画像のどこに窓を置いても同じものを使い回します。この使い回しを重み共有(weight sharing)と呼び、これがあるおかげで「画像のどこにあっても同じ模様は同じ模様として検出される」という性質、すなわち平行移動同変性(translation equivariance)が生まれます。

帰納バイアス(inductive bias)とは、モデルにあらかじめ組み込まれた前提知識のことです。畳み込みは「近いものは関係が深い(局所性)」「画像を少しずらしても同じ物体は同じ物体(平行移動同変性)」という二つの強い帰納バイアスを構造として持っています。帰納バイアスが正しければ、モデルは少ないデータでも正解にたどり着きやすくなります。逆に帰納バイアスがタスクに合わないと、かえって性能の天井を下げる足かせにもなります。このバイアスの強さと表現力の上限のバランスが、CNN と Transformer の本質的な対立軸です。

CNN(Convolutional Neural Network、畳み込みニューラルネット)は畳み込みを積み重ねたモデルで、AlexNet(2012)、VGG、ResNet などが代表です。長年、画像認識の主役でした。

ViT(Vision Transformer、Dosovitskiy et al., ICLR 2021)は、画像を 16x16 ピクセルのパッチ(小片)に切り分け、各パッチを線形変換で一つのトークン(自然言語処理でいう単語のような単位)に変え、Transformer の自己注意で全パッチ同士の関係を一度に計算します。畳み込みを一切使わず Transformer だけで画像が扱えることを示した点が画期的でした。しかし畳み込みの帰納バイアスを捨てたため、JFT-300M(3 億枚)のような超大規模データで事前学習しないと CNN(ResNet 系)に勝てませんでした。論文中の有名な観察として、ImageNet-1k(120 万枚)だけで学習すると ViT は同規模 ResNet に劣り、データを増やすにつれて逆転する、という曲線が示されています。

自己注意(self-attention)とは、各トークンが「自分は他のどのトークンとどれくらい関係が深いか」を計算で求め、関係が深い相手の情報を重く取り込む仕組みです。具体的には各トークンから Query(問い合わせ)、Key(鍵)、Value(値)という三種のベクトルを線形変換で作り、Query と Key の内積で関係の強さ(注意重み、attention weight)を測り、その重みを softmax で正規化してから Value を加重平均します。「車」というトークンが「タイヤ」「道路」のトークンと強く結びつく、といった関係を学習します。

計算量の注意点として、自己注意の計算量はトークン数 N の二乗 O(N^2) に比例します。画像を細かく分けてトークンが増えるほど、注意は急速に重くなります。N=196(14x14)なら約 3.8 万の関係計算ですが、N=3136(56x56)になると約 980 万になり、80 倍以上に跳ね上がります。これが高解像度で ViT が苦しむ理由です。

## 仕組みを詳しく
tensor とは「数字が並んだ多次元の箱」で、画像特徴は `[B, C, H, W]` = (バッチ枚数, チャネル数, 縦, 横) の形で表します。CvT は ViT に対して二種類の畳み込みを差し込みます。

### 1. Convolutional Token Embedding(畳み込みトークン埋め込み)
従来の ViT は画像を重ならないパッチに切り分けてトークンにしました。CvT は代わりに「畳み込み」でトークンを作ります。

- 入力画像: `[B, 3, 224, 224]`(3 は RGB)
- Stage 1 の畳み込み: カーネル 7x7、ストライド(窓のずらし幅)4、パディングあり → 出力 `[B, 64, 56, 56]`
- これをトークンの列とみなすと、56x56 = 3136 個のトークン、各トークン 64 次元

ここでカーネルが 7x7 とストライド 4 なので、隣り合うトークンを作るときに窓が一部重なります(オーバーラップ)。重なる分だけ隣同士が情報を共有し、局所的なつながりが自然に保たれます。これが普通の ViT の「重ならないパッチ分割(カーネル=ストライド=16 で重なりゼロ)」との決定的な差です。各 Convolutional Token Embedding のあとには Layer Normalization が置かれ、続くトランスフォーマーブロックに渡されます。

さらに CvT は階層構造(hierarchy)を持ち、三つの Stage を経るごとに解像度を下げ、チャネルを増やします。

- Stage 1: `[B, 64, 56, 56]`(トークン 3136、低次元)
- Stage 2: `[B, 192, 28, 28]`(トークン 784)
- Stage 3: `[B, 384, 14, 14]`(トークン 196、高次元)

各 Stage の入口でまた Convolutional Token Embedding を適用して解像度を半分にします。CNN(ResNet など)と同様、浅い層は細かいエッジや模様、深い層は意味的なまとまり(物体らしさ)を捉えます。元祖 ViT が最初から最後まで 14x14=196 トークンの単一スケールだったのと対照的で、CvT は CNN 流のピラミッド型を採用しています。これは下流タスク(物体検出やセグメンテーション)で複数解像度の特徴マップが欲しいデコーダに渡しやすいという実用的な利点になります。

### 2. Convolutional Projection(畳み込み射影)
自己注意では各トークンから Query, Key, Value を作ります。通常の ViT はこれを線形変換(各トークンを独立に処理する全結合)で作りますが、CvT は深さ方向分離畳み込み(depthwise separable convolution、チャネルごとに別々に空間方向の畳み込みをかけてから 1x1 で混ぜる軽い畳み込み)で作ります。これにより、Q/K/V を作る段階でも近所の情報が混ざり、局所性が入ります。深さ方向畳み込みとは、通常の畳み込みが全チャネルをまたいで重みを持つのに対し、各チャネルが自分の中だけで空間畳み込みをする軽量版で、パラメータと計算が大幅に少ない方式です。

さらに重要なのが計算削減です。Key と Value 側ではストライド 2 の深さ方向畳み込みを使い、トークン数を縦横それぞれ半分=全体で 1/4 に減らします。

- before(通常): Query 28x28=784 と Key 784 の注意 → 784 x 784 ≒ 61 万の関係計算
- after(CvT): Query 784 と Key 196 の注意 → 784 x 196 ≒ 15 万の関係計算(約 4 分の 1)

Query 側は解像度を保つので、出力のトークン数(=その Stage の解像度)は変わりません。Key/Value だけ間引くこの方式を spatial reduction とも呼び、後続研究にも引き継がれます。CvT の論文ではこの設計を Convolutional Projection と呼び、線形射影版・通常畳み込み版・深さ方向分離版を比較して、深さ方向分離版が精度と計算量のバランスで最良だと示しています。

```
通常の ViT 射影              CvT の Convolutional Projection
tr  token ──[Linear]──> Q     token map ─[depthwise conv s=1]─> Q (解像度そのまま)
tr  token ──[Linear]──> K              ─[depthwise conv s=2]─> K (1/4 に縮約)
tr  token ──[Linear]──> V              ─[depthwise conv s=2]─> V (1/4 に縮約)
```

### 位置埋め込みが不要になる
ViT は「各パッチが画像のどこにあるか」を別途、位置埋め込み(positional embedding)として足す必要がありました。CvT は畳み込み自体が位置情報を内包します。畳み込みは窓を空間順にスライドさせ、ストライドや隣接関係で各出力の位置が一意に決まるため、明示的な位置情報を持たせなくても場所が分かるのです。CvT は位置埋め込みを完全に省きました。これは、入力解像度を変えたときに位置埋め込みを補間し直す手間が消えるという実用上の利点になります(物体検出のような高解像度入力に転用しやすい)。

なお CvT は ViT 同様に classification token(分類用の特別なトークン、画像全体を要約する役割を持つ学習可能なベクトル)を最終 Stage にだけ付け、その出力を分類ヘッドに渡します。浅い Stage では cls トークンを置かず、最終段でだけ導入するのは、浅い段では空間的な局所処理に集中させ、大域要約は最後にまとめて行うという役割分担の表れです。

## 手法の系譜と主要論文
この分野は「畳み込みと注意をどう混ぜるか」という大きな問いを共有しています。

- ViT(Dosovitskiy, Beyer, Kolesnikov, et al.; Google, ICLR 2021, arXiv:2010.11929): 畳み込みなしで画像が扱えることを示した起点。ただしデータ飢餓で、ImageNet-1k 単独では同規模 ResNet に劣る。
- DeiT(Touvron, Cord, Douze, et al.; Facebook, ICML 2021, arXiv:2012.12877): 知識蒸留(teacher の CNN から学ぶ distillation token)で ViT を ImageNet-1k だけで学習可能にした。ただし教師モデルを別途用意するコストがある。CvT はアーキテクチャ自体に局所性を入れるという、教師なしの別解を示した点で対照的です。
- CvT(Wu, Xiao, Codella, Liu, Dai, Yuan, Zhang; Microsoft, ICCV 2021, arXiv:2103.15808): Convolutional Token Embedding と Convolutional Projection の二点で局所性・階層性・位置不変性を取り込み、データ効率を改善。
- PVT(Pyramid Vision Transformer, Wang, Xie, Li, et al.; ICCV 2021, arXiv:2102.12122): ほぼ同時期に階層構造と Key/Value のトークン削減(spatial-reduction attention, SRA)を導入。CvT と問題意識を共有しますが、PVT が線形射影で削減するのに対し、CvT は射影そのものを畳み込みで行う点が独自です。
- Swin Transformer(Liu, Lin, Cao, et al.; Microsoft, ICCV 2021, arXiv:2103.14030): 窓注意で局所性と線形計算量を実現。CvT が「射影に畳み込み」なのに対し、Swin は「注意の範囲を窓に限定」というアプローチの違いがあります。同じ Microsoft 発ですが設計思想が異なり、結果的に Swin のほうが下流タスクのデファクトとして広く普及しました。
- CeiT, LeViT, ConViT など同時期のハイブリッド群: それぞれ feed-forward に畳み込みを入れる、推論高速化に振る、ソフトな畳み込みバイアスを注意に与える、といった派生を提示。
- これらの後、CoAtNet(Dai et al., 2021)が「段ごとに畳み込みブロックと注意ブロックを分ける」体系化を、MaxViT(Tu et al., 2022)が「全段で局所と大域を線形計算で両立」を提案し、ハイブリッドの設計空間が整理されていきます。

この系譜の中で CvT の歴史的役割は「ViT の三つの構成要素(トークン化・射影・位置情報)それぞれに畳み込みで手を入れると何が起きるか」を最初期に丁寧に分解して見せたことにあります。後続の多くのハイブリッドは、この三分割のどれかを引き継いでいます。

## 論文の実験結果(定量データ)
評価の主舞台は ImageNet-1k 分類です。指標は top-1 accuracy(最も確信度の高い予測クラスが正解と一致した割合。100% に近いほど良い)。この分野では 1 ポイント未満の差でも有意とされます。

- CvT-13(約 2000 万パラメータ、4.5 GFLOPs 程度): ImageNet-1k のみで top-1 約 81.6%。
- CvT-21(約 3200 万パラメータ): ImageNet-1k のみで top-1 約 82.5%。同規模帯の DeiT-B(約 8600 万パラメータ、約 81.8%)より少ないパラメータで上回りました。つまり「JFT のような外部巨大データなし、教師蒸留なし」でこの精度に届いた点が要点です。比較として、ResNet-50(約 2500 万、約 76〜79%)、同規模の純 ViT-S/B との優位もあわせて主張されています。
- ImageNet-22k(約 1400 万枚)で事前学習し ImageNet-1k に転移した CvT-W24(大型版、約 2.77 億パラメータ)は top-1 約 87.7% を報告。当時の上位に並びました。
- 高解像度(384x384)でファインチューニングするとさらに 1 ポイント前後上がる、という ViT 系共通の挙動も示されています。

アブレーション(どの要素を抜くと何が起きるか)も論文で示されています。これが CvT の主張の核心です。

- Convolutional Token Embedding を通常のパッチ分割(重なりなし)に戻すと精度が低下(報告では 0.8〜1 ポイント前後)。重なりを持つ畳み込み埋め込みが効いていることを示します。
- Convolutional Projection を通常の線形射影に戻すと精度が低下。とくに Key/Value のストライド畳み込みは、精度をほぼ保ったまま計算量を約 30% 削減できると報告されました。精度を犠牲にせず軽くできるのが価値です。ストライドを 1 のまま(縮約なし)にすると精度はわずかに上がるが計算が増える、というトレードオフも示され、ストライド 2 が良い妥協点だとされています。
- 位置埋め込みを足しても精度がほとんど変わらないことを確認(差はほぼゼロ、報告で 0.1 ポイント未満)。畳み込みが位置情報を担っているため、位置埋め込みは冗長だと実証しました。これは「位置埋め込みを外せる」という設計上の主張を裏づけ、後の解像度可変応用の根拠になります。

数値の意味を補足すると、top-1 で 0.7 ポイント(DeiT-B → CvT-21)というと小さく見えますが、パラメータ数を 8600 万から 3200 万へ約 1/3 に減らしながら勝っている点が本質です。ImageNet で 1 ポイントを稼ぐのは年単位の研究努力に相当することが多く、同じ精度をより小さなモデルで出せることは、推論コストとメモリの観点で実利が大きいのです。

## メリット・トレードオフ・限界
メリット
- ImageNet-1k だけ(JFT 不要、蒸留不要)で高精度。データ効率が良い。少データ領域での汎化に強い。
- 位置埋め込みが不要で、入力解像度の変更に強い。検出やセグメンテーションへ転用しやすい。
- 階層構造のおかげで多段の特徴マップを出せ、下流タスクのデコーダ(FPN など)に渡しやすい。
- Convolutional Projection による Key/Value 削減で、精度を保ちつつ計算量を抑えられる。

トレードオフ・限界
- 純粋な ViT より設計が複雑で、畳み込み層の追加実装・調整が要る。Stage 数・カーネルサイズ・ストライドといったハイパーパラメータが増える。
- 大域注意を一部制約する(Key/Value を間引く)ため、JFT-3B 級の超大規模データでは、制約のない大型 ViT に逆転される可能性がある。容量の天井は ViT のほうが高いという見方です。帰納バイアスは少データで効く一方、無限のデータの前ではむしろ足かせになりうる、という CNN/Transformer 論争の縮図がここにあります。
- depthwise 畳み込みはパラメータあたりの計算は軽いものの、メモリ帯域律速になりやすく、理論 FLOPs の割に実機(GPU)の速度が伸びにくいことがある。FLOPs と実測スループットが乖離する典型例です。
- 事前学習済み重みのエコシステムは、後発で広く使われる Swin や ConvNeXt ほど充実していない場面がある。デファクト化の差が実務上の採用しやすさに影響します。

未解決の課題としては、畳み込みと注意の「最適な配分」をデータ規模ごとに自動決定する方法は確立しておらず、依然として経験的なハイパーパラメータ探索が必要です。また、CvT が獲得した局所性が「本当に畳み込みでなければ得られないのか、適切な学習則や事前学習で純 ViT でも代替できるのか」は、後の MAE 系自己教師あり学習の成功を踏まえると、なお論争的な問いとして残っています。

## 発展トピック・研究の最前線
CvT が示した「畳み込みで局所性を取り戻す」流れは、その後の階層型ハイブリッドへ発展します。CoAtNet は段単位で畳み込みと注意を分離し、MaxViT は一つのブロック内で局所(Block)と大域(Grid)の注意を両立させました。MViT/MViTv2 は注意の中でプーリングして解像度を下げる別系統の階層化を提案し、とくに動画へ展開しました。一方、ConvNeXt(Liu et al., CVPR 2022)は逆に「純粋な畳み込みのまま Transformer の設計思想だけ取り込む」方向で、ハイブリッドに頼らず同等精度を達成し、「本当に注意は必須か」という問いを投げかけました。

研究の最前線では、自己教師あり学習(MAE, BEiT など)との組み合わせで事前学習を強化する流れ、検出・セグメンテーション・動画への統一バックボーン化、推論速度の実機最適化(構造再パラメータ化、深さ方向畳み込みの融合カーネル)が継続テーマです。さらに最近は、自己注意そのものを線形時間の状態空間モデル(Mamba 系)や効率的注意で置き換える動きもあり、「局所性は畳み込みで、大域性は効率的シーケンスモデルで」という新しいハイブリッドの構図が現れています。CvT が問うた「局所性をどう構造に埋め込むか」という主題は、バックボーンの形が変わっても生き続けています。
