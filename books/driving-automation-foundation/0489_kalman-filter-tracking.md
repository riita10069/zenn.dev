# カルマンフィルタによる追跡

## ひとことで言うと

カルマンフィルタ(Kalman filter)とは、ノイズだらけの観測から、物体の本当の状態(位置や速度)を時々刻々と推定し続ける古典的なアルゴリズムです。観測が多少ブレても、物理の予測と組み合わせて滑らかで確からしい推定を出します。多物体追跡(multi-object tracking、MOT)では、各物体に1つのカルマンフィルタを割り当て、フレームをまたいで「同じ物体」を追い続ける定番手法です。

## 直感的な理解

暗い部屋で、転がるボールを断続的に光らせて位置を測る場面を想像してください。光るたびに位置は分かりますが測定は不正確でブレます。一方、ボールは物理法則に従うので「さっきの位置と速度から次の位置はだいたいここ」と予測できます。

カルマンフィルタは、この2つの情報源を賢く混ぜます。物理からの予測(ある程度信頼できる)と、新しい観測(ノイズあり)を、それぞれの不確かさに応じた重みで加重平均します。予測が不確かなら観測を重視し、観測がノイジーなら予測を重視する。これを毎ステップ繰り返すことで、どちらか単独より正確な推定が得られます。「予測してから観測で修正する」、このサイクルが本質です。

## 基礎: 前提となる概念

押さえるべき言葉:

- 状態(state): 推定したい量。追跡では位置と速度をまとめたベクトル。たとえば x = [位置x, 位置y, 速度x, 速度y]ᵀ(ᵀは縦ベクトルを表す転置記号)。
- 観測(measurement): センサが返す値。検出器が出す物体の位置など。状態の一部しか直接は測れないことが多い(位置は見えるが速度は見えない、など)。
- 共分散行列(covariance): 推定の不確かさを表す行列。対角成分が各変数の分散(ばらつき)、非対角が変数間の相関。これが「どれだけ自信があるか」を数値化します。
- 状態遷移モデル: 状態が時間でどう変わるかの物理モデル。等速運動なら「新しい位置 = 古い位置 + 速度 × 時間」。

カルマンフィルタには前提があり、線形なシステムとガウス分布(正規分布)のノイズを仮定します。この仮定の下では、ベイズ的に最適な推定であることが理論的に保証されます。非線形な場合は拡張カルマンフィルタ(EKF)やアンセンテッドカルマンフィルタ(UKF)で近似します。

## 仕組みを詳しく

カルマンフィルタは2段階を交互に繰り返します。記号の意味を言葉で説明します。

予測ステップ(predict): 物理モデルで状態を1ステップ先へ進める。
```
x̂⁻ = F x̂        … 状態の予測(F は状態遷移行列。等速運動の物理を表す)
P⁻ = F P Fᵀ + Q  … 不確かさの予測(Q はプロセスノイズ。物理モデルの不完全さ)
```
x̂(エックスハット)は状態の推定値、上付き⁻は「観測で直す前」を表します。P は共分散(不確かさ)。予測すると、時間が経つぶん不確かさ P は増えます(Q が足される)。

更新ステップ(update): 新しい観測 z で予測を修正する。
```
K = P⁻ Hᵀ (H P⁻ Hᵀ + R)⁻¹   … カルマンゲイン
x̂ = x̂⁻ + K (z - H x̂⁻)        … 状態の修正
P  = (I - K H) P⁻              … 不確かさの縮小
```
H は観測行列(状態のうち何が観測できるかの対応)、R は観測ノイズ、z は実際の観測。(z - H x̂⁻) は「予測した観測」と「実際の観測」の差で、イノベーション(innovation、新情報)と呼びます。

最重要は K(カルマンゲイン)で、予測と観測のどちらをどれだけ信じるかの重みです。観測ノイズ R が大きい(観測が当てにならない)と K は小さくなり、予測を重視。逆に予測の不確かさ P⁻ が大きいと K は大きくなり、観測を重視。観測を取り込むと不確かさ P は縮みます(自信が増す)。この自動的なバランス調整がカルマンフィルタの美しさです。

### 多物体追跡への応用: SORT

検出器(画像から物体ボックスを出す、関連: [インスタンスセグメンテーション](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0392_instance-segmentation))は各フレームで物体を見つけますが、フレーム間で「どれとどれが同一物体か」は教えてくれません。これを結びつける(データ関連付け、data association)のが追跡です。

SORT(Bewley et al., "Simple Online and Realtime Tracking", ICIP 2016, arXiv:1602.00763)はこれを最小構成で実現しました。

```
各フレームで:
1. 既存の各トラックをカルマンフィルタで「次の位置」へ予測
2. 検出された新ボックスと、予測ボックスの IoU(重なり)を全ペア計算
3. ハンガリアン法で最適にマッチング(検出 ↔ 既存トラック)
4. マッチした検出でカルマンフィルタを更新
5. 余った検出 → 新トラック誕生 / 余ったトラック → 一定回数で消滅
```

ハンガリアン法(Hungarian algorithm)は、割り当てコストの総和を最小にする組み合わせを求める古典的最適化アルゴリズム。ここでは「予測と検出が重ならないほど高コスト」とし、全体で最も妥当な対応付けを選びます。SORT は見た目情報を一切使わず、位置と動きだけで追跡します。

### DeepSORT: 見た目を足す

SORT は速いが、物体が一時的に隠れて再出現すると別物として扱い、ID が入れ替わる「ID switch」が頻発します。DeepSORT(Wojke et al., "Simple Online and Realtime Tracking with a Deep Association Metric", ICIP 2017, arXiv:1703.07402)は、各物体の見た目を表す特徴ベクトル(外見埋め込み、appearance embedding)を深層ネットで抽出し、位置の近さに加えて「見た目の似ているか」も関連付けの手がかりにしました。これで遮蔽後の再同定(re-identification)が改善し、ID switch が大きく減ります。

## 手法の系譜と主要論文

- 原典: Kalman, "A New Approach to Linear Filtering and Prediction Problems", 1960。アポロ計画の航法にも使われた、制御・信号処理の基礎。
- 非線形拡張: 拡張カルマンフィルタ(EKF)、アンセンテッドカルマンフィルタ(UKF, Julier & Uhlmann 1997)、粒子フィルタ(particle filter)。
- 検出ベース追跡: SORT(ICIP 2016, arXiv:1602.00763)→ DeepSORT(ICIP 2017, arXiv:1703.07402)。
- 近年: ByteTrack(Zhang et al. ECCV 2022, arXiv:2110.06864)— 低信頼度の検出も捨てずに2段階で関連付ける。OC-SORT(Cao et al. CVPR 2023)— カルマンフィルタの遮蔽時の弱点を補正。これらは「カルマン+データ関連付け」の枠組みが今も主流であることを示しています。

## 論文の実験結果(定量データ)

評価には MOT Challenge(MOT16/17/20)が標準で、主指標は MOTA(Multi-Object Tracking Accuracy)。これは見逃し(false negative)・誤検出(false positive)・ID switch の3種の誤りをまとめて1つにした指標で、高いほど良い(最大100%)。IDF1 は同一 ID をどれだけ保てたかを測る指標で、追跡の一貫性を見ます。

SORT(2016)は、当時としては MOTA で競争力ある水準を、桁違いの速度(報告でおよそ 数百 fps)で達成し、「単純な手法でも検出器が良ければリアルタイム追跡が成り立つ」ことを示しました。ただし ID switch が多いのが弱点でした。DeepSORT(2017)は同じ枠組みに外見特徴を足し、報告によると ID switch をおよそ 4 割以上削減し、IDF1 を改善したとされています。MOTA は検出性能に強く依存するため SORT と大きくは変わらない一方で、IDS と IDF1 という「一貫性」の指標で差が出た点が重要です。

ByteTrack(ECCV 2022)は MOT17 で MOTA・IDF1 ともに当時の最高水準(報告で MOTA 80% 前後、IDF1 77% 前後)を達成し、「低信頼度の検出も一度マッチングに回す」という単純な工夫だけで大きく伸びることを示しました。これらの数値の読み方として、MOTA は検出器の良し悪しに引きずられやすく、IDF1 や ID switch のほうが「追跡アルゴリズムそのものの巧拙」をよく表す、という理解が実務では重要です。

## メリット・トレードオフ・限界

メリット:
- 軽量・高速・オンライン(過去だけ見れば動く)で、車載のリアルタイム用途に向く。
- 不確かさを共分散として明示的に持つので、予測の信頼度が分かる。
- 物理に基づくので、検出が一瞬抜けても予測で補える(短時間の遮蔽に頑健)。

トレードオフ・限界:
- 線形・ガウス仮定。急な方向転換や複雑な運動(歩行者の不規則な動き)では等速モデルがずれる。EKF/UKF や学習ベースで補う。
- ノイズ行列 Q, R の調整が難しい。値次第で滑らかすぎたり追従が遅れたりする。
- データ関連付けの脆さ。密集・長時間遮蔽では IoU だけだと取り違える(DeepSORT の外見特徴で緩和)。
- 検出器依存。追跡精度は土台の検出性能に大きく左右される。

## 発展トピック・研究の最前線

- 学習ベース追跡: Transformer で関連付け自体を学ぶ(TrackFormer、MOTR など)、end-to-end の流れ。
- 3D・センサ融合: LiDAR・レーダ・カメラを統合した3D MOT。鳥瞰図(BEV)上でカルマンフィルタを回す自動運転向け構成。
- 不確実性とアンサンブル: 複数の運動モデルを混ぜる Interacting Multiple Model(IMM)、推定の信頼度を下流の判断に渡す(関連: [アンサンブル学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0175_ensemble-methods))。
- 異常な動きの検出: 追跡軌跡から逸脱挙動を見つける(関連: [異常検知](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0734_anomaly-detection))。
