# 軌道語彙(クラスタアンカー)

## ひとことで言うと
自動運転で「これから車がどう進むか」を決めるとき、座標を一から数値で計算して出す(回帰)のではなく、あらかじめ用意した「代表的な進み方の見本帳」から最も良いものを選ぶ、という方法があります。この見本帳が軌道語彙(trajectory vocabulary)です。中身は、過去の運転データに含まれる実際の軌道を大量に集めてグループ分け(クラスタリング)し、各グループの代表として取り出した数千本の典型軌道です。これらをアンカー(anchor=錨、基準となる候補)と呼びます。計画を「自由作文」ではなく「選択問題(分類問題)」に変えるための部品で、近年の End-to-End 計画の多くがこの考え方に立っています。

## 直感的な理解
あなたが外国語で道案内をする場面を想像してください。文法を一から組み立てて自由に話す(回帰に相当)のは難しく、間違えやすい。一方、よく使う言い回しを100通り暗記しておき、状況に合うものを選んで使う(分類に相当)なら、確実で速い。軌道語彙はこの「定型句集」に当たります。

なぜ「選ぶ」方が良いのか、もう少し踏み込みます。運転には本質的に「複数の正解」がある状況が頻繁にあります。前方の障害物を右からも左からも避けられる、交差点で直進も右折もできる、といった具合です。これをマルチモーダル(multi-modal=複数の妥当な選択肢がある)と呼びます。

ここで回帰の致命的な弱点が出ます。回帰は二乗誤差などを最小化するので「最も誤差が小さくなる1つの数値」を出そうとし、複数の正解があると、その平均を取ってしまいます。「右に避ける軌道」と「左に避ける軌道」の平均は「まっすぐ障害物に突っ込む軌道」です。これを mode averaging(モード平均化)問題と呼びます。学習データには右避けも左避けも含まれるので、回帰はその間を取って、どちらでもない危険な軌道を学んでしまうのです。これは数学的には「条件付き分布が二峰性なのに、二乗誤差最小は条件付き平均を返す」ために起きます。

選択(分類)方式ならこの問題が起きません。「右避け候補を選ぶ確率0.5、左避け候補を選ぶ確率0.5」と、2つの選択肢を分離したまま表現でき、どちらか一方(確率の高い方、または安全な方)を選べば衝突しません。これがマルチモーダルを自然に扱えるという、軌道語彙の最大の利点です。

## 基礎: 前提となる概念

回帰(regression)と分類(classification)の違いを押さえます。回帰は連続値(実数)を直接予測する問題です。例えば「未来の x 座標は3.2メートル」と数値を当てます。分類は、あらかじめ決めた有限個の選択肢の中から正解を選ぶ問題です。例えば「候補A・B・Cのうちどれか」を選びます。軌道計画を分類として解くには、選択肢となる軌道候補の集合が必要で、それが軌道語彙です。

クラスタリング(clustering)とは、似たデータを自動でグループにまとめる手法です。代表格が k-means(ケイミーンズ)です。これは「データ点を k 個のグループに分け、各グループの中心(平均)を求める」アルゴリズムで、(1) ランダムに k 個の中心を置く、(2) 各データ点を最も近い中心のグループに割り当てる、(3) 各グループの平均を新しい中心にする、(2)と(3)を変化が止まるまで繰り返す、という手順で動きます。距離は通常ユークリッド距離(各点の差の二乗和の平方根)を使います。軌道語彙では、数十万本の軌道を k=数千 個のクラスタに分け、各中心を代表軌道(アンカー)とします。

ソフトマックス(softmax)は、複数の数値(スコア)を「合計が1になる確率分布」に変換する関数です。候補 i のスコア s_i に対し、exp(s_i) を全候補の exp(s) の和で割ります。これで「候補 #37 を選ぶ確率0.4、#512 を選ぶ確率0.3、…」という分布が得られます。

クロスエントロピー損失(cross-entropy loss)は、分類で「正解クラスの予測確率を高めよ」と教える損失です。正解クラス c の予測確率 p_c に対し −log(p_c) を最小化します。正解の確率が1に近いほど損失は0に近づき、低いほど大きく罰します。

アンカーという用語は、物体検出の「アンカーボックス」(あらかじめ用意した代表的な矩形候補)から来ています。「自由に位置を回帰する」より「代表候補からのずれを回帰する」方が安定する、という発想が軌道計画にも持ち込まれたものです。

## 仕組みを詳しく

軌道語彙の作り方と使い方を段階的に説明します。準備は計算機の中でオフラインに1回だけ、推論は走行中に毎回行います。

準備フェーズ(オフライン、1回だけ)。

ステップ1: 軌道を集める。
訓練データの各シーンから、自車の未来軌道を取り出します。例えば未来3秒を0.5秒刻みで6点 `[(x1,y1),...,(x6,y6)]` で表すと、1本の軌道は `6×2 = 12` 次元のベクトルになります。これを数十万〜数百万本集めます。座標は自車中心の座標系(ego-centric、自車の現在位置を原点・進行方向を前にとる)に揃えるのが普通で、こうすると「右に曲がる」という形そのものでクラスタリングできます。世界座標のまま集めると、同じ「右折」でも交差点の位置ごとに別物になってしまい、語彙がぼやけます。

ステップ2: クラスタリングする。
集めた軌道ベクトルを k-means で V 個のクラスタに分けます(例 V=4096 や 8192)。各クラスタの中心(平均軌道)が代表軌道=アンカーになります。結果として V 本の代表軌道が得られ、直進・緩い右カーブ・急な左折・減速停止・加速追い越し…といった典型的な進み方が網羅されます。テンソルとしては `[V, T, 2]`(語彙サイズ V、各 T 点、各点2次元)で、これを固定の見本帳としてモデルに持たせます。VADv2 では計画語彙として 4096 本程度、Hydra-MDP では 8192 本がよく使われます。

推論フェーズ(走行中、毎回)。

ステップ3: ネットワークが各候補のスコアを出す。
カメラ等から作った環境特徴と自車状態を入力に、ネットは語彙の各候補 i に対して「この状況でこの候補がどれだけ良いか」のスコア `s_i` を出します。出力は `[V]` 次元です。ソフトマックスを通せば候補ごとの確率分布になります。実装上は、各アンカー軌道を埋め込み(embedding)に変換し、環境特徴との内積や attention でスコアを計算する構成が多く使われます。

ステップ4: 候補を選ぶ。
最もスコアの高い候補をそのまま採用するか、上位候補に小さな補正(オフセット回帰=代表軌道からのずれを少しだけ回帰で微調整)を足します。VADv2 のように確率分布として保持し、Hydra-MDP のように複数指標で採点し直して選ぶ(→ [Hydra-MDP 多教師蒸留計画](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0182_hydra-mdp-multitarget))こともできます。

学習。人間が実際に通った軌道に最も近いアンカーを正解クラスとして、クロスエントロピー損失で「正解に近い見本を選べ」と教えます。生の座標を当てる回帰ではなく、見本帳の中から正解を当てる分類問題に変換しているのが要点です。多くの実装では「分類(どの候補か)」と「微小なオフセット回帰(選んだ候補をどう微調整するか)」を組み合わせます。VADv2 では、正解1本だけでなく「人間軌道に近い複数候補に距離に応じた soft なラベルを割り当てる(distance-based soft label)」工夫で、近い候補を一律に不正解とする粗さを和らげています。

数値例で効果を実感します。前方の障害物を右か左で避ける状況。回帰なら平均化で直進(衝突)しがちですが、語彙方式なら右避けアンカー #37 に確率0.5、左避けアンカー #88 に確率0.5を割り当て、両方の選択肢を保持できます。確率の高い方、または安全指標で勝つ方を選べば衝突しません。

```
        回帰(平均化)              語彙(分類)
障害物 ■                      障害物 ■
   ↑ 直進=衝突                右↗  左↖  両モードを保持
  / \                          選択 → 安全な方を取れる
右   左(平均が中央に潰れる)
```

語彙サイズの設計はトレードオフです。候補が少なすぎると細かい運転を表現できず、多すぎると候補ごとのスコア計算でコストとメモリが増えます。また、何秒先までを何点で表すか(時間ホライズンと刻み)も語彙の質を左右する設計判断です。k-means 自体も、初期値依存(k-means++ で緩和)や、頻度の高い直進ばかりにアンカーが集中して稀な機動(急回避・Uターン)が代表されにくい、という偏りを持つため、語彙構築の工夫が研究対象になっています。

## 手法の系譜と主要論文

軌道アンカーの源流は動き予測(motion prediction)の分野にあります。

MultiPath(Chai, Sapp, Bansal, Krishnan 他、Waymo、CoRL 2019、arXiv:1910.05449)は、周辺車両の未来予測で k-means によるアンカー軌道集合を導入した先駆けです。提案は「アンカーで離散化し、各アンカーに確率と微小オフセット(各時刻のガウス分布)を割り当てる」という形でマルチモーダル予測を安定化することでした。動機は、回帰が複数モードを平均化して潰す問題を避けることです。効果として、複数の妥当な未来を確率付きで出せるようになりました。トレードオフはアンカーのカバレッジ依存です。

TNT(Zhao et al., "Target-driveN Trajectory prediction", CoRL 2020)は、アンカーを「軌道全体」ではなく「目標点(終端の到達候補)」として離散化し、目標を選んでから軌道を補完する2段構えを提案しました。離散化の粒度を終端だけに絞ることで効率と精度を両立させた発展です。

MultiPath++(Varadarajan 他、Waymo、ICRA 2022、arXiv:2111.14973)は、固定アンカーを学習可能な潜在アンカー(learned latent anchors)に拡張し、入力依存でアンカーを適応させました。固定 k-means アンカーの「データ分布が変わると合わなくなる」弱点を緩める発展です。

計画(planning)側への持ち込みが VADv2(Jiang, Chen 他、2024、"VADv2: End-to-End Vectorized Autonomous Driving via Probabilistic Planning", arXiv:2402.13243)です。軌道語彙(planning vocabulary)を明示的に導入し、軌道を語彙から確率的に選ぶ「確率的計画」を提案しました。動機は、計画が本質的にマルチモーダルで、確率分布で表すのが自然であり、回帰の平均化問題を避けるためです。効果は計画性能の改善と、確率という形での不確実性表現です。トレードオフは語彙サイズと品質への依存です。

そして Hydra-MDP(Li 他、CVPR 2024、arXiv:2406.06978、→ [Hydra-MDP 多教師蒸留計画](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0182_hydra-mdp-multitarget))が、同じ語彙ベースに複数の評価指標の教師で全候補を採点する多教師蒸留を足し、NAVSIM 上位を獲得しました。これにより軌道語彙は「単に選ぶ」だけでなく「多面的に採点して選ぶ」土台として確立しました。

## 論文の実験結果(定量データ)

動き予測ベンチマーク(Waymo Open Motion Dataset, Argoverse)では、マルチモーダル指標 minADE / minFDE / Miss Rate が使われます。minADE は「K本(例 K=6)の予測のうち、正解に最も近い1本の全時刻平均位置誤差」、minFDE は同じく「最も近い1本の終点誤差」、Miss Rate は「最も近い予測でも終点が一定距離(例2m)以上外れた割合」です。単一予測の ADE と違い、複数の妥当な未来を出すモデルを正しく評価できます。MultiPath / MultiPath++ はアンカー方式により、回帰ベースより minADE / minFDE を改善したと報告されています。MultiPath++ は提案当時 Waymo Open Motion Dataset の動き予測リーダーボードで首位を占めました(mAP・minADE で当時の最上位)。

計画側では、VADv2 が CARLA の Town05 ベンチマークなどの閉ループ評価で、回帰ベースのプランナに対し運転スコア(Driving Score)・経路完走率(Route Completion)を改善したと報告しています。特に、確率分布として複数モードを保持することで、回帰では潰れていた分岐状況(右折/直進など)での挙動が改善します。

アブレーションの一般的知見。
- 語彙サイズ V を増やすと、ある点までは性能(計画スコアや minADE)が改善し、その後は頭打ちになります。V が小さすぎると粗い軌道しか出せず、大きすぎると計算コストだけ増えます。計画では V=数千〜1万程度が良い帯域として報告されます。
- アンカーへの分類のみ(オフセット回帰なし)だと、選んだ代表軌道の粒度が限界になり、細かい微調整ができません。オフセット回帰を足すと終点誤差(FDE)が改善します。
- 正解1本だけのハードラベルより、近い複数候補への soft label(VADv2)の方が学習が安定し、計画スコアが上がると報告されています。
- 固定 k-means アンカー(MultiPath)を学習可能アンカー(MultiPath++)にすると、データ分布への適応で minADE / minFDE がさらに下がると報告されています。

これらの指標の意味。ADE / FDE が小さいほど予測が人間(正解)に近いことを示しますが、計画では「人間と同じか」だけでなく「衝突しないか・進めるか」も重要です。だからこそ Hydra-MDP のような多指標採点が語彙の上に乗る価値があります。

## メリット・トレードオフ・限界

メリット。
- 計画を安定した分類問題に変換でき、回帰の学習不安定を避けられる。
- マルチモーダル(複数の妥当な選択肢)を確率分布として自然に表現でき、平均化による危険な出力を防ぐ。
- 実データ由来なので「ありえる軌道」だけが候補になり、物理的に無理な出力が出にくい。
- Hydra-MDP のような多指標採点・選択の土台になる。離散候補なので各候補を独立に評価しやすい。
- 候補のスコアが確率として出るので、不確実性(どれくらい迷っているか)を下流に伝えられる。

トレードオフと限界。
- 語彙に無い動き(学習データに無かった特殊な回避)は表現できない。カバレッジの上限がそのまま性能の上限になる。
- 語彙サイズを増やすと候補ごとのスコア計算でコストとメモリが増える。
- クラスタ数・時間ホライズン・刻みなど語彙の設計が性能を左右し、調整(チューニング)が要る。k-means は頻出する直進にアンカーが偏り、稀な機動が代表されにくい。
- 固定 k-means アンカーは訓練データの分布に依存する。運用ドメインが変わると(別の都市・別の交通文化)語彙が合わなくなる。学習可能アンカーで緩和できるが、その分学習が複雑になる。
- 純粋な選択だけでは粒度が足りず、滑らかな微調整にはオフセット回帰の併用が要ることが多い。

## 発展トピック・研究の最前線

固定アンカーの限界を超える方向として、(1) 学習可能・入力依存のアンカー(MultiPath++ 系)、(2) アンカーを階層的に粗から細へ絞り込む coarse-to-fine 設計、(3) 拡散モデル(diffusion)で連続的にサンプリングしつつ語彙的な多様性を担保する手法、が研究されています。また、Hydra-MDP のように語彙を「採点して選ぶ」土台に使う流れと、VLM(視覚言語モデル)ベースのプランナが言語で軌道意図を出してから語彙に落とす流れも交差しつつあります。語彙の構築方法そのものも、単純な k-means から、意味的多様性(右折・合流・回避など機能でバランスを取る)を考慮した構築や、運動学的に実現可能なアンカーだけを残す構築(→ [運動学自転車モデル拘束デコード](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0186_kinematic-bicycle-decoding))へと洗練が進んでいます。離散選択と連続生成のどちらが本質的に優れるかは未解決で、両者を混ぜるハイブリッドが現在の主戦場です。
