# World Models (Ha & Schmidhuber)

## ひとことで言うと
カメラ画像のような高次元の観測を小さな数字の並びに圧縮し、その圧縮された世界が「次にどうなるか」を予測する小さなモデルを作り、エージェント(行動を決める主体)はその予測された「頭の中の世界=夢」だけで練習できる、という研究です。2018 年に David Ha と Jürgen Schmidhuber が発表し、現代の「ワールドモデル(世界モデル)」研究のスタート地点として広く引用されます。ここでいうワールドモデルとは、環境(ゲームや運転シーンなど)が次にどう変化するかを予測する、いわば頭の中の物理シミュレーターのことです。

## 直感的な理解
あなたが目を閉じて、これから運転する道のりを頭の中で思い描く場面を想像してください。実際に車に乗っていなくても、「ここを右折して、信号で止まって、その先で合流して……」と、頭の中の道路で何度でも予行演習できます。実際の道を走らなくても、頭の中のシミュレーションで練習が進む。これがこの研究の核心です。

機械にこれをやらせるには、2つの能力が要ります。1つは「目に映る大量のピクセルを、要点だけに圧縮する」こと。もう1つは「その要点が次にどう変化するかを予測する」こと。この2つが揃えば、エージェントは現実で危険な失敗を繰り返さなくても、自分の頭の中で安全に何度でも練習できます。Ha & Schmidhuber はこれを VAE と RNN で具体化しました。

人間の脳の予測符号化(predictive coding)の比喩もよく使われます。私たちは感覚入力を丸ごと処理しているのではなく、「次に何が来るか」を絶えず予測し、予測と実際のズレ(誤差)だけに注意を向けている、という仮説です。World Models はこの「世界の内部モデルを持ち、未来を予測する」という脳の働きを、機械学習の部品で素朴に再現した研究だと見ることもできます。

## 基礎: 前提となる概念
強化学習(Reinforcement Learning、報酬を最大化する行動を試行錯誤で学ぶ枠組み)でエージェントをカメラ画像から直接訓練しようとすると、困りごとがあります。

第一に、観測が大きすぎます。64×64 の RGB 画像は 64 × 64 × 3 = 12288 個の数字です。方策(「この状況ならこう動く」というルール)に毎ステップこの 12288 次元を直接食わせると、学習すべきパラメータが膨大になり、学習が遅く不安定になります。第二に、実環境での試行回数が高くつきます。ロボットや車では何百万回も失敗しながら試すのは時間・費用・安全の面で現実的ではありません。

人間は運転中、目に映る全ピクセルを意識しているわけではありません。「前の車が近い」「右にカーブがある」といった要点だけを抽象化し、さらに「このままアクセルを踏めば追突しそうだ」と数秒先を予測しています。Ha & Schmidhuber は、この「要点への圧縮」と「未来の予測」を機械にやらせれば、エージェントは現実で試さなくても自分の予測した世界の中で練習できるはずだ、と考えました。

理解に必要な用語を噛み砕きます。

VAE(Variational Autoencoder、変分オートエンコーダ)とは、画像を小さな数字の並び(潜在ベクトル z)に圧縮し、そこから元画像を復元できるように学習するニューラルネットです。普通のオートエンコーダと違い、z を「点」ではなく「平均と分散を持つ確率分布」として学習するので、潜在空間が滑らかになり、近い z は近い画像になります。これにより z の世界で連続的に未来をたどれるようになります。

RNN(Recurrent Neural Network)は過去の情報を内部状態として持ち越しながら系列を処理するネットで、ここでは LSTM(長期記憶を保ちやすいゲート付き RNN)が使われます。

MDN(Mixture Density Network、混合密度ネットワーク)は、出力を「1つの値」ではなく「複数のガウス分布を重ね合わせた分布(混合分布)」として出す仕組みです。未来が一通りに決まらない状況を、複数の山を持つ分布で表せます。

進化戦略(evolution strategy)は、微分を使わずパラメータをランダムに少しずつ変えて成績の良いものを残していく最適化手法です。報酬がカクカク(微分不能)でも最適化でき、並列化しやすいのが利点です。

## 仕組みを詳しく
このモデルは大きく3つの部品でできており、頭文字をとって V・M・C と呼ばれます。役割が明確に分かれているのが設計上の特徴です。

### V: Vision(視覚) — VAE で画像を圧縮する

VAE を使って各フレームを圧縮します。

```
入力: 64×64×3 の画像(12288 次元)
   │ VAE エンコーダ
出力: 潜在ベクトル z(論文では 32 次元)
```

12288 個の数字を 32 個に圧縮します。約 384 分の 1 です。この 32 次元の z が「今この瞬間の世界の要約」になります。VAE はこの 32 次元から元の画像をできるだけ忠実に復元するよう訓練されるので、32 個の数字に画像の重要情報が詰め込まれます。V はランダム行動で集めたフレームを使って単独で先に訓練されます。

### M: Memory(記憶) — MDN-RNN で未来を予測する

M は時間方向を担当します。RNN(具体的には LSTM)に MDN を組み合わせて MDN-RNN と呼びます。MDN は「次に起こることを1つの値ではなく、複数の山を持つ確率分布として出力する」仕組みです。なぜ確率にするかというと、未来は一通りに決まらないからです。たとえばゲームで敵が左右どちらに動くか分からないとき、平均を取って「真ん中」と予測すると現実にはあり得ない曖昧な未来になってしまいます。複数の可能性で表すことでこれを避けます。

```
入力: 現在の潜在ベクトル z_t(32 次元)、行動 a_t、RNN 内部の隠れ状態 h_t
出力: 次の潜在ベクトル z_{t+1} の確率分布  →  P(z_{t+1} | a_t, z_t, h_t)
```

温度パラメータ τ(タウ)で予測のばらつき具合を調整でき、τ を上げると混合分布がより広がり、予測世界がより不確実(難しい)になります。この τ が後述の「夢の中での過適応」を抑える鍵になります。M は V が出した z の系列を教師にして訓練されます。

### C: Controller(制御) — 行動を決める小さな部品

C はエージェント本体、つまり「今の状況でどの行動を取るか」を決める部分です。驚くべきはこれが極端に小さいことです。

```
入力: z_t(32 次元)と h_t(RNN 隠れ状態、256 次元)を結合した 288 次元
出力: 行動 a_t(CarRacing では 3 次元: ハンドル・アクセル・ブレーキ)
構造: 単純な1層の線形写像。パラメータは 288 × 3 + 3 = 867 個だけ
```

V と M で世界を圧縮・予測してあるおかげで、行動を決める C 自体はこれほど小さくて済みます。この小さな C を CMA-ES(Covariance Matrix Adaptation Evolution Strategy、進化戦略の一種)で進化させます。微分を使わないので、報酬がカクカクしていても最適化できます。パラメータが 1000 個未満なので進化戦略でも十分扱えるサイズに収まっているのが要点です。

### 「夢の中で学習する」とはどういうことか

M(MDN-RNN)は z_{t+1} を予測できるので、実環境を一切使わず「z だけの世界」を回し続けられます。手順はこうです。

```
1. ある z_t から C が行動 a_t を出す
2. M が (z_t, a_t, h_t) から次の z_{t+1} をサンプリングし、h を更新する
3. z_{t+1} を新たな z_t とみなして 1 に戻り、繰り返す
```

実際のピクセル画像を一度も生成しなくても、潜在空間 z の系列だけで「もしこう動いたらこうなる」をシミュレートできます。VizDoom(火の玉を避けるゲーム)実験では、この夢の中だけで C を訓練し、その C を本物のゲームに持っていくと高得点が取れることを示しました。これが論文タイトル "Recurrent World Models Facilitate Policy Evolution"(再帰的世界モデルが方策の進化を助ける)の意味です。

ここで「報酬」と「終了(死亡)」も M に予測させる点が重要です。火の玉に当たって死ぬかどうかを M が予測できないと、夢の中ではいつまでも死なず、現実に転移できないからです。

## 手法の系譜と主要論文
この研究の思想的な源流は Schmidhuber の長年の構想にあります。1990 年代から彼は「環境を予測するモデル(world model)と、それを使って行動を決めるコントローラを分ける」というアイデア(後の "On Learning to Think"、arXiv:1511.09249 などにまとまる)を提唱していました。本研究はこの構想を、当時実用になった VAE と LSTM で具体化したものと位置づけられます。

本体は David Ha, Jürgen Schmidhuber "Recurrent World Models Facilitate Policy Evolution"(NeurIPS 2018、オンライン版 "World Models"、arXiv:1803.10122)です。提案は、観測を VAE で圧縮(V)、時間発展を MDN-RNN で予測(M)、行動を小さな線形 C で決め進化戦略で最適化、という3部品構成。動機は、高次元観測を直接扱う強化学習の非効率さを、圧縮 + 予測 + 小さな方策で克服することでした。インタラクティブなオンライン論文(distill 形式)として公開され、「夢の中の運転」を読者が体験できるデモが話題になりました。

直系の後継が Hafner らの一連の研究です。PlaNet(ICML 2019、arXiv:1811.04551、[PlaNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0082_planet_note))は V と M を1つの RSSM([RSSM (Recurrent State-Space Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0088_rssm_note))に統合して end-to-end 学習にし、潜在空間で CEM 計画しました。Dreamer(ICLR 2020、arXiv:1912.01603、[Dreamer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0047_dreamer-v1))は World Models の「夢の中で学ぶ」を勾配学習の actor-critic で実用水準に押し上げ、DreamerV2([DreamerV2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0048_dreamer-v2)、arXiv:2010.02193)、DreamerV3([DreamerV3](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0049_dreamer-v3)、arXiv:2301.04104)と続いて Atari や Minecraft で人間超えのサンプル効率を達成しました。

生成系の系譜も豊かです。映像生成側では、運転動画を制御つきで作る [GAIA-2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0054_gaia-2_note) や、高忠実度の運転未来予測を狙う [Vista](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0104_vista_note)、行動条件で運転動画と方策を生む [DriveDreamer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0052_drivedreamer) が、いずれも「観測を圧縮した潜在空間で未来を生成・予測する」枠組みを受け継いでいます。汎用の対話型環境生成では Genie([Genie](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0055_genie_note))が、行動で操作できる「遊べる世界」を大量の動画から学ぶ形で World Models の発想をインターネット規模に拡張しました。

## 論文の実験結果(定量データ)
2つの環境で評価されました。

CarRacing-v0 は、上から見た車でコースを走り、訪れたタイルの数で報酬が決まるタスクです。当時このタスクで「解いた」とみなされる基準は 100 試行平均で 900 点超でした。World Models は、実環境で V/M を学習し C を CMA-ES で最適化する標準設定で、100 試行平均でおよそ 906 点(報告値、標準偏差つき)を達成し、このタスクで初めて安定して 900 点超を報告した手法になりました。比較対象の従来手法(DQN ベースなど)はこの水準に届いておらず、圧縮表現の上で 1000 パラメータ未満の小さな線形コントローラを進化させるだけで高難度の視覚制御が解けることを示した点が大きな意義です。アブレーション的観察として、入力を z だけにして h(記憶)を外すと性能が落ち、時間方向の記憶が効いていることが示されています。

VizDoom(Take Cover、火の玉を避け続けるタスク)では、より野心的な実験として「M が作る夢の中だけで C を訓練し、その C を本物の環境に転移する」ことを行いました。報告では、夢の中で学んだ方策が実環境でも基準(生存時間 750 ステップ)を上回って長く生き延び、夢のみの学習で実環境に通用することを実証しました(夢で学んだ方策が実環境で平均 1100 ステップ超といった水準)。

ただし重要なアブレーション的観察として、夢の不確実性 τ を適切に上げないと、エージェントが「M の予測の穴(現実にはあり得ない楽な状況)」を突いてズルをしてしまい、夢では高得点でも実環境で通用しない方策に収束する問題が報告されています。τ を上げて夢を意図的に難しく(より不確実に)すると、この過適応が抑えられ実環境での頑健性が上がる、という知見です。これは生成モデルの誤差をエージェントが悪用する「world model exploitation」の最初期の明確な指摘で、後続研究が真剣に対処する課題になりました。

## メリット・トレードオフ・限界
メリット:
- 高次元観測(画像)を低次元(32 次元)に圧縮することで、行動を決める部分を極小(数百パラメータ)にでき、進化戦略のような微分不要の最適化が使える。
- 学習した世界モデルの中(夢)で何度でも安全・高速に試行でき、実環境の試行回数を節約できる。
- 3部品(V/M/C)が役割分担しており、各部品を別々に理解・改良しやすい。
- MDN と温度 τ により、未来の不確実性を陽に扱い、その大小を制御できる。

トレードオフ・限界:
- 世界モデルが現実を正確に再現できないと、エージェントがその誤差を悪用してしまう(夢の中だけで強く現実で弱い)。τ で不確実性を足す回避策が要る。
- VAE の圧縮で細部が落ちるため、細かい情報が勝敗を分けるタスクには向きにくい。圧縮が報酬と無関係な情報(背景)を優先してしまうこともある。
- V → M → C を段階的に訓練する設計で、3つを同時に最適化していない。再構成が良くても報酬に効く情報を逃すミスマッチが起こりうる。後継の Dreamer はこの分離を解消し、潜在空間で end-to-end に学習して性能と汎用性を上げた。
- 確率分布の出力(MDN)や温度調整など、設計の手間とハイパーパラメータが増える。
- 評価したタスクは比較的単純(低解像度・短時間)で、実世界スケールへの直接適用はこの時点では未検証。

## 発展トピック・研究の最前線
World Models が提示した「圧縮 + 予測 + 計画(または夢での学習)」の枠組みは、その後 RL を超えて拡散モデルや Transformer ベースの大規模生成と結びつきました。運転分野では、生成された動画そのものを学習・評価データとして使う方向(危険シーンの合成、反実仮想シナリオ生成)へ発展しています。汎用環境生成では、行動で操作できる「遊べる世界」を動画から学ぶ研究(Genie 系)が、World Models の発想をインターネット規模のデータに拡張した形と言えます。

一方で「夢を悪用する」問題、すなわち生成モデルの誤差にエージェントや評価が引っ張られる課題は今も中心的です。生成データだけで学習・評価を閉じず実データと併用する、生成の物理整合性を高める、不確実性を陽に扱う、生成環境を定期的に実環境で校正する、といった方向で研究が続いています。また「世界モデルを持つこと自体が知能の中核ではないか」という思想的主張(LeCun の JEPA 構想や Schmidhuber の long-standing な主張)とも接続し、世界モデルは生成 AI の一分野を超えて、自律エージェント研究の基盤概念になっています。
