# カメラ校正 (内部/外部パラメータ)

## ひとことで言うと
カメラで撮った写真の中の「ピクセルの場所」と、現実世界の「3次元の場所」を対応づけるための数値の組を求める作業が「カメラ校正(calibration)」です。これには2種類のパラメータがあります。内部パラメータ(intrinsics)は「そのカメラのレンズと撮像素子の性質」(焦点距離・画像中心・歪みなど)、外部パラメータ(extrinsics)は「そのカメラが車のどこにどの向きで付いているか」です。この2つが揃って初めて、画像のピクセルを車を中心とした3D座標へ、あるいはその逆へ変換できます。BEV(俯瞰)投影、複数カメラの貼り合わせ、地図との重ね合わせ、すべての前提になる基礎です。

## 直感的な理解
カメラ画像は、3次元の世界を2次元のピクセルに「潰した」ものです。潰す過程で奥行きの情報が失われるので、写真を見ただけでは「画像の右上に写っている車は、自車から見て前方右に何メートルか」がわかりません。

ところが自動運転では、複数カメラの映像を1枚の俯瞰図(BEV)に貼り合わせたり、カメラで見た物体を地図上の位置に対応づけたりする必要があります。これには「このピクセルは3D空間のどの方向に対応するか」「このカメラは車のどこに付いているか」という情報が要ります。それを数値として測り、行列の形で持っておくのがカメラ校正です。校正がなければ、画像はただの色の格子であって、空間的な意味を持ちません。

内部と外部を分ける理由は、原因の切り分けです。レンズの性質はカメラ固有で、車のどこに付けても変わらない(内部)。一方、取り付け位置と向きは、同じカメラでも前に付けるか後ろに付けるかで全く変わる(外部)。分けておけば、カメラを交換したら内部だけ、取り付けを変えたら外部だけ更新すればよくなります。物理的に独立な要因を独立なパラメータに分けておくのが、保守と再利用の両面で合理的なのです。

## 基礎: 前提となる概念
ピンホールカメラモデルという理想化を使います。これは「レンズを無限小の穴(ピンホール)とみなし、世界の点から穴を通った光がまっすぐ撮像面に届く」という幾何モデルです。実際のレンズには厚みも歪みもありますが、まずこの理想モデルで考え、歪みは後から補正します。この理想化のおかげで、3D 点から画像への投影が「行列の掛け算」という線形代数の操作で書けるようになります。

座標系がいくつか登場します。

- 世界座標(world / 基準フレーム): 車や地図を基準にした3D座標。たとえば「自車の後輪中心を原点とし、前方を X、左を Y、上を Z」とする。
- カメラ座標(camera): そのカメラを原点とした3D座標。普通は「光軸方向を Z、右を X、下を Y」とする。
- 画像座標(image / pixel): 撮影された画像のピクセル位置 (u, v)。左上が原点。

同次座標(homogeneous coordinates)という表現を使います。3D点 (X, Y, Z) を、末尾に1を足した (X, Y, Z, 1) という4次元で表す方法です。なぜこうするかというと、回転と平行移動を1つの行列の掛け算でまとめて書けるようになり、さらに最後に割り算をすることで「遠くのものは小さく写る」という遠近投影(透視投影)を線形代数で扱えるからです。同次座標は全体を定数倍しても同じ点を表す(スケール不変)という性質があり、これが投影の「最後に割る」操作の根拠になっています。

## 仕組みを詳しく
内部パラメータは3×3の行列 K(カメラ行列、intrinsic matrix)で表します。

```
    | fx   0  cx |
K = |  0  fy  cy |
    |  0   0   1 |
```

- fx, fy: 焦点距離をピクセル単位で表したもの。「どれくらいズームしているか」。大きいほど望遠で、同じ物体が大きく写る。fx と fy が分かれているのは、撮像素子のピクセルが厳密には正方形でないことがあるためで、通常はほぼ等しい。
- cx, cy: 主点(principal point)。光軸が画像面と交わる点のピクセル座標で、おおむね画像の中心。幅1920・高さ1080の画像なら (960, 540) 付近。
- 0 が多いのは、理想ピンホールでは軸が直交しスキュー(画素軸の傾き)がないと仮定するためです。

外部パラメータは「世界座標 → カメラ座標」への回転 R(3×3)と並進 t(3×1)で、これをまとめた4×4の同次変換行列 T で表します。

```
    | R(3x3)  t(3x1) |
T = |   0  0  0    1  |
```

回転 R は「カメラがどちらを向いているか」、並進 t は「カメラがどこにあるか」を表します。R は直交行列(行列式が +1 の回転)で自由度は3(ロール・ピッチ・ヨー)、t も自由度3なので、外部は計6自由度です。校正ファイルが逆向きの「カメラ → 基準フレーム」の行列を持っている場合は、逆行列を取って「基準フレーム → カメラ」に直してから使います。ここで向きの取り違えは投影の大事故につながるので、どちらの変換を持っているかを必ず確認します。

最終的に欲しいのは投影行列 P(3×4)で、内部と外部を掛け合わせて作ります。

```
P = K @ T_world_to_cam[:3, :]    # (3,3) @ (3,4) = (3,4)
```

この P を使うと、世界座標の3D点 `(X, Y, Z, 1)` を画像ピクセル `(u, v)` に変換できます。

```
[u', v', w'] = P @ [X, Y, Z, 1]      →      u = u'/w',  v = v'/w'
```

最後に w'(奥行きに相当)で割るのがポイントです。これが「遠くのものほど画面中心に寄って小さく写る」という遠近効果を生みます。数値例で確かめます。fx=fy=1000, cx=960, cy=540 のカメラの正面 5m・右 1m・上下0の点 (カメラ座標で X=1, Y=0, Z=5) を投影すると、u' = 1000·1 + 960·5 = 5800、w' = 5 なので u = 5800/5 = 1160。同じ右 1m でも奥行き 10m なら u' = 1000·1 + 960·10 = 10600、w' = 10 で u = 1060。遠いほど主点 960 に寄る(中心に近づく)ことが数値で確認できます。逆に、ピクセルから3Dへ戻すには深度 w' が必要で、単眼カメラでは深度が未知のため「光線(ray、そのピクセルが対応する3D空間の半直線)」までしか復元できません。深度を別途与えるか、LiDAR や複数視点で補います。

実務で見落としがちな工夫が内部行列のスケーリングです。校正は元の解像度(たとえば1920×1080)で測られますが、ニューラルネット backbone に入れる前に画像をリサイズ・中央切り抜き(CenterCrop)します。すると元の K のままではピクセル位置がずれます。そこで、リサイズ倍率に合わせて fx, fy, cx, cy をスケールし、切り抜きぶんだけ cx, cy を平行移動させて補正します。

- Resize で横にスケール scale_x がかかれば `fx *= scale_x`, `cx *= scale_x`(縦も同様)。
- CenterCrop でオフセット分 `cx -= offset_x`, `cy -= offset_y`。

この補正を忘れると投影が数十ピクセル単位でずれ、BEV 変換が破綻します。地味ですが必須の処理で、データ拡張(画像の左右反転や random crop)を入れるたびに K も同じ幾何変換で更新しなければならない、という落とし穴があります。

レンズ歪み(distortion)も内部パラメータの一部です。広角レンズでは画像の端が樽型(barrel distortion、外側ほど縮む)に曲がります。これは半径方向の歪み係数 (k1, k2, k3) と接線方向の係数 (p1, p2) で表され、ピンホールモデルに当てはめる前に歪みを除去(undistort)します。魚眼レンズ(視野角がほぼ180度に達するもの)ではこの多項式モデルが破綻するため、等距離射影など別のモデルを使います。

出力は、典型的には形状 `(V, 3, 4)` のテンソル(V=カメラ台数)で、各カメラの投影行列をまとめたものです。これは1シーンで不変(取り付けが変わらない限り各フレームで同じ)なので、シーンごとに1回だけ計算してキャッシュするのが普通です。

## 手法の系譜と主要論文
カメラ校正の理論は古典的な多視点幾何に根ざしています。

Zhang(Zhengyou Zhang, Microsoft Research, TPAMI 2000, "A Flexible New Technique for Camera Calibration")が、実用上の決定打になりました。チェスボードの平面パターンを別々の角度から数枚撮るだけで、カメラの内部パラメータとレンズ歪みを推定できる手法です。動機は、それ以前は精密な3D校正治具が必要で手間とコストが大きかったことです。各撮影でパターンの平面と画像の対応からホモグラフィ(平面どうしの射影変換、3×3行列)を解き、複数枚の制約を集めて K を閉形式で初期推定し、最後に再投影誤差を最小化する非線形最適化(Levenberg-Marquardt 法)で精密化します。今でも OpenCV の `calibrateCamera` の理論的土台です。トレードオフは、平面パターンの撮影が必要で、走行中に自動で校正し続けるオンライン校正とは別物である点です。

Hartley & Zisserman("Multiple View Geometry in Computer Vision", 2nd ed. 2004)は、ピンホールモデル・射影幾何・複数視点の幾何を体系化した教科書で、校正・三角測量・エピポーラ幾何(2台のカメラの間に成り立つ幾何制約)の標準リファレンスです。校正を深く学ぶならここが基盤になります。直接線形変換(DLT)による P の推定や、基礎行列・基本行列の推定など、現代の SLAM や SfM(Structure from Motion)の土台がここにまとまっています。

BEV 投影系の代表として Lift-Splat-Shoot(Philion & Fidler, NVIDIA, ECCV 2020, "Lift, Splat, Shoot: Encoding Images from Arbitrary Camera Rigs by Implicitly Unprojecting to 3D")があります。これはまさに内部・外部パラメータを使って各カメラのピクセルを共通の BEV 格子へ投影します。各ピクセルに対し深度の離散的な確率分布を予測し(Lift)、その確率に応じて特徴を3D空間へばらまき(Splat)、BEV 格子に集約します。校正が正確であることが前提で、校正がずれると BEV がぼやけます。深度が未知という単眼の弱点を「確率分布として持つ」ことで回避した点が巧妙です。

その後の BEVFormer(Li et al., Nanjing/上海AI Lab, ECCV 2022, "BEVFormer: Learning Bird's-Eye-View Representation from Multi-Camera Images via Spatiotemporal Transformers")は、校正された投影を使って BEV 上のクエリ(BEV 格子の各セルに対応する学習可能なベクトル)が各カメラの該当ピクセルを attention で参照する構成へ発展しました。BEV セルを 3D 点として複数の高さで持ち、各カメラに投影してそのピクセル特徴を集める(deformable attention)という流れで、正確な内部・外部パラメータを空間変換の土台として要求します。

## 論文の実験結果(定量データ)
校正そのものの品質は、再投影誤差(reprojection error、3D 点を投影し直したときの画像上のピクセルずれ)で測ります。単位はピクセルで、良好な校正なら平均1ピクセル未満が目安です。Zhang の手法は、合成データと実画像の両方で、それ以前の治具ベース手法に匹敵する精度を、はるかに簡便な撮影で達成したと報告しています。合成実験では既知の真値に対して焦点距離・主点・歪み係数を高精度に復元できることが示されています。

校正の重要性は下流タスクの精度ではっきり出ます。Lift-Splat-Shoot は nuScenes で、複数カメラから BEV のセマンティックセグメンテーション(各 BEV セルが道路か車か等を分類)を行い、当時の単眼ベースの手法より IoU(Intersection over Union、予測領域と正解領域の重なり率、大きいほど良い)を改善しました。ここで効いているのが、複数カメラを校正に基づいて一つの BEV 格子へ正しく融合できる点です。BEVFormer は nuScenes の 3D 物体検出で、当時のカメラのみ手法の NDS / mAP を明確に押し上げ、さらに時間方向(過去フレームの BEV)を取り込むことで速度推定や遮蔽物の検出を改善したと報告しています。

校正誤差の影響を調べたアブレーション的な研究では、外部パラメータ(取り付け角度)に数度の誤差を入れると BEV 上の物体位置が大きくずれ、検出性能が顕著に落ちることが報告されています。たとえばピッチ角(俯角)が1〜2度ずれるだけで、遠方の地面投影が数メートル単位でずれ得ます。簡単な見積もりとして、俯角の誤差 Δθ ラジアンは距離 d の地面点で約 d·Δθ の横ずれを生むので、d=50m・Δθ=1度(≈0.017rad)なら約 0.85m、2度なら約 1.7m の誤差になります。遠くの点ほど小さな角度誤差が大きな位置誤差に増幅されるためで、校正がクリティカルな前提であることを定量的に示しています。

これらの数値が示す本質は、「校正の質が、その上に積むあらゆる空間処理(BEV 融合・地図重ね合わせ・3D 検出)の精度の天井を決める」という点です。校正がずれていれば、どんなに賢いモデルを載せても下流はずれます。

## メリット・トレードオフ・限界
メリットは次の通りです。

- 校正があれば、画像ピクセルと3D世界を正確に対応づけられ、複数カメラの BEV 合成や地図との重ね合わせが可能になる。
- 内部と外部を分けることで、カメラ交換時は内部だけ、取り付け変更時は外部だけ更新すればよい。保守が楽になる。
- 一度求めれば(取り付けが変わらない限り)再利用でき、シーンごとに計算し直す必要がない。投影行列をキャッシュでき計算も軽い。

トレードオフと限界は次の通りです。

- 校正がずれると下流の投影・BEV がすべてずれる。精度のクリティカルな前提になる。とくに遠方では角度誤差が増幅される。
- 走行中の振動・温度変化・荷重・サスペンションの沈み込みで校正が少しずつずれる。本来はオンライン校正(走りながら自動補正)が望ましいが、自動運転中に安定して行うのは難しい。
- リサイズ・クロップ・データ拡張など前処理を変えるたびに K の再スケールが要る。忘れるとバグの温床になり、しかも「少しだけずれる」ため発見しにくい。
- 単眼カメラでは深度が未知のため、ピクセルから3Dへ一意に戻せない(光線までしか定まらない)。深度推定や複数視点・LiDAR の併用が要る。
- 校正値のパースや座標系の取り違え(world→cam と cam→world の混同、軸の定義違い)があると、ダミーの単位行列に近い投影で代用することになり、特徴連結のような融合では一応動いても BEV 投影では意味のない結果になる。校正が機能の前提条件になる典型例で、初期の動作確認で見落とされやすい不具合です。

## 発展トピック・研究の最前線
オンライン自動校正(走行中にレーン・消失点・他センサとの整合から外部パラメータを推定し続ける)が実用上の重要課題です。LiDAR とカメラの間の外部校正を自動化する手法(両者で同じ物体・エッジが重なるように姿勢を最適化する)、複数カメラ間の相対姿勢を画像特徴のマッチングから推定する手法などが研究されています。

学習ベースの校正・自己校正も進んでいます。深層学習で1枚の画像から焦点距離や歪みを推定したり(消失点や直線構造を手がかりにする)、ネットワークの中に微分可能な投影層を組み込んで校正のずれを学習で吸収したりする方向です。さらに、校正の不確実性を明示的に扱い、ずれに頑健な BEV 融合を目指す研究もあります。校正を「固定の既知値」ではなく「推定すべき不確実な量」として扱う流れが最前線です。

近年の BEV 認識モデルでは、校正パラメータをネットワークに条件として与える(カメラの姿勢を埋め込みベクトルにして入力する)ことで、多様なセンサ構成や取り付け位置に汎化させる試みも進んでいます。これは校正を前処理として外に置くのではなく、モデルの一部として統合する方向であり、[センサ構成 (camera/LiDAR/radar)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1083_sensor-suite-config) の異なる装備差を一つのモデルで吸収する流れとも接続します。
