# Bellman / TD 損失

## ひとことで言うと

「この場面でこの行動をとると将来どれくらい得か」を表す Q値 を学習するための損失です。Q値の予測と、「今もらった報酬 + 次の場面の価値」という目標との差を二乗して小さくします。この目標が一部に自分自身の予測を含むのが特徴で、未来を待たずに1歩ごとに学習を進められます。これを時間差分 (TD: Temporal Difference) 学習 と呼び、現代の強化学習のほぼすべての価値ベース手法の土台です。

## 直感的な理解

将棋やチェスを学ぶ場面を想像してください。一局が終わるまで「この手が良かったか」が分からないとしたら、学習はとても遅くなります。実際には人間は「この手を指したら盤面が有利になった気がする」という、一手ごとの感触で学んでいます。TD 学習はこの「一手ごとに学ぶ」を数学にしたものです。

具体的には、「ある局面の価値」と「一手進めた次の局面の価値」が食い違っていたら、その差 (TD誤差) を使って今の見積もりを修正します。最終結果を待たなくても、隣り合う2つの予測のつじつまを合わせ続ければ、やがて全体が正しくなる。これが TD のアイデアです。

なぜこれが必要か。強化学習で最大化したいのはリターン (将来の報酬の合計) ですが、これを直接観測するには未来が全部終わるまで待たねばなりません。長い系列のタスクでは、これでは学習が遅すぎて使い物になりません。TD 学習は「未来を待たず、1歩先の見積もりで今を更新する」ことでこの問題を解きます。これはモンテカルロ法 (エピソードの最後まで実際の報酬を観測してから更新する方法) との対比で理解すると鮮明です。モンテカルロは正解を観測するのでバイアスがない代わりに、長い系列の確率変動をすべて含むので分散が大きく学習が遅い。TD は1歩先の見積もりで代用するので速い代わりに、見積もりの誤りが混じる (バイアスがある)。この バイアスと分散のトレードオフ が TD 学習の本質です。

## 基礎: 前提となる概念

用語を平易に説明します。

- 報酬 (reward) r: 行動の直後にもらえる「良さの点数」。
- 割引率 (discount factor) gamma: 0〜1 の値で、将来の報酬をどれだけ重視するか。1 に近いほど遠い未来まで重視し、0 に近いほど目先だけを見る。数学的には、gamma < 1 が無限和の収束を保証する役割も持ちます。
- リターン (return): その時点以降にもらえる報酬の (割引つき) 合計。`R_t = r_t + gamma*r_{t+1} + gamma^2*r_{t+2} + ...`。強化学習の目的はこの期待値を最大化することです。
- Q値 Q(s, a): 状況 s で行動 a をとり、その後は方策に従ったときのリターンの見積もり。行動価値関数とも呼びます。
- 状態価値 V(s): 状況 s の良さ (方策に従ったときのリターンの見積もり)。
- 方策 (policy) pi: 状況に対しどの行動をとるかのルール。

中心となるのが ベルマン方程式 (Bellman equation) です。直感はこうです。「ある場面の価値 = 今すぐもらえる報酬 + 次の場面の価値 (を gamma で割り引いたもの)」。リターンの定義を1項だけ展開すると、リターンが「最初の報酬 + 残りのリターン」に分解できることから自然に出てきます。式で書くと:

```
Q(s, a) = E[ r + gamma * V(s') ]        (s' は次の状況)
```

右辺の `r + gamma * V(s')` を TD目標 (TD target) と呼びます。ここが肝心で、TD目標は一部に「自分の予測 V(s')」を含んでいます。確定した正解ではなく、自分の見積もりを目標に使うこの仕組みを ブートストラップ (bootstrap、自分の足で自分を持ち上げる) と呼びます。これにより未来を待たずに更新できる反面、初期の不正確な見積もりが目標に混じるという弱点も生みます (後述)。

オンポリシーとオフポリシーの区別も重要です。オンポリシー (SARSA など) は「今学んでいる方策が実際にとった次行動」で TD目標を作ります。オフポリシー (Q学習など) は「次状態でとりうる最良行動」で目標を作るので、データを集めた方策と学習する方策が違っても学べます。過去データの再利用 (経験再生) はオフポリシーだからこそ可能です。

## 仕組みを詳しく

TD損失 (Bellman損失) は次の二乗誤差です。

```
TD誤差 delta = Q(s, a) - ( r + gamma * V(s') )    # 予測 と TD目標 の差
損失   L = 平均[ delta^2 ]                          # その二乗を最小化
```

`delta` が TD誤差 (TD error)。これを 0 に近づける = 予測 Q値 と「報酬+次の価値」が整合する、ということです。

典型的なオフライン RL の実装は次のようになります (Q を学び、別途学んだ V を次状態の価値に使う流儀)。

```python
# Q-function update (Bellman / TD loss)
q_input = torch.cat([feats, actions], dim=-1)
q_pred  = q_fn(q_input).squeeze(-1)                 # 予測 Q(s, a)  [B]
with torch.no_grad():
    next_v   = value_fn(next_feats).squeeze(-1)     # V(s')  [B]
    q_target = rewards + gamma * next_v             # TD目標 = r + gamma * V(s')
qf_loss = F.mse_loss(q_pred, q_target)              # 二乗誤差
```

ステップで分解します (B はバッチサイズ)。

1. `feats`: 状況を表す特徴ベクトル。画像なら CNN/ViT バックボーンが出した特徴 (例 `[B, 512]`)。
2. `q_input = cat([feats, actions])`: 特徴と行動を連結。Q値は「状況と行動の両方」に依存するので両方入れます。
3. `q_pred = q_fn(q_input)`: Q関数 (小さな MLP) が予測した Q値 `[B]`。
4. `next_v = value_fn(next_feats)`: 次の状況の価値 V(s')。`torch.no_grad()` で囲むのは TD目標側に勾配を流さない (目標を固定的に扱う) ため。これがないと予測も目標も同時に動いて学習が不安定になります。
5. `q_target = rewards + gamma * next_v`: TD目標。gamma=0.99 が典型 (実効的な「見通す未来の長さ」は 1/(1-gamma) ≒ 100 ステップに相当)。
6. `qf_loss = mse_loss(q_pred, q_target)`: 予測と TD目標の二乗誤差。これを下げる方向に Q関数を更新します。

数値例。ある1サンプルで報酬 r=2.0、次状況の価値 V(s')=10.0、gamma=0.99 なら TD目標 = 2.0 + 0.99*10.0 = 11.9。Q の予測が 9.0 なら TD誤差は 9.0 - 11.9 = -2.9、損失への寄与は 2.9^2 = 8.41。学習が進むと予測が 11.9 に近づき、誤差が縮みます。

なぜ no_grad で目標を固定するのか (重要)。Q関数の出力が「予測」にも「目標 (の元になる価値)」にも現れると、目標自体が学習で動いてしまい、犬が自分の尻尾を追うように発散しがちです。DQN ではこれを防ぐため ターゲットネットワーク (少し前のパラメータのコピー) を別に持ち、目標はそちらで計算します。コピーの更新は、数千ステップごとに丸ごと同期する方式と、`theta_target ← tau*theta + (1-tau)*theta_target` のように毎ステップ少しずつ追従させる ソフト更新 (Polyak 平均、tau は 0.005 程度) の2方式があります。簡易な実装では `no_grad` で目標側を切り離し、かつ V を別ネットワークにすることで近い効果を得ます。

オンライン Q学習 (DQN) では、次状態の価値を `max_a' Q(s', a')` (次状態でとりうる最良行動の Q) で置き換えます。これが Q学習の「貪欲に最良を見積もる」性質ですが、後述するように max が過大評価を生む原因にもなります。

TD(lambda) と n-step return にも触れておきます。1ステップ TD (上記) と、エピソード終端まで見るモンテカルロの中間として、n ステップ先まで実報酬を使い、それ以降をブートストラップする `n-step return` があります。`R^(n)_t = r_t + gamma r_{t+1} + ... + gamma^(n-1) r_{t+n-1} + gamma^n V(s_{t+n})`。n を大きくするほどバイアスが減り分散が増える。TD(lambda) はこの n をいろいろ変えた目標を `lambda` で幾何平均し、バイアスと分散をなめらかに調整します。報酬がまばらなタスクでは n-step や lambda を使うと情報伝播が速くなります。

## 手法の系譜と主要論文

- TD学習の原典 — Sutton (1988) "Learning to Predict by the Methods of Temporal Differences"、および教科書 Sutton & Barto "Reinforcement Learning: An Introduction"。提案: 最終結果を待たず、連続する予測どうしの差 (TD誤差) で価値関数を更新する。動機: モンテカルロ法 (エピソード終了まで待つ) は遅く分散が大きい。1歩先で更新すれば速く、オンラインで学べる。新規性: ブートストラップという考え方を確立し、強化学習の中核アルゴリズム (TD(0)、TD(lambda)、SARSA) の基礎を作った。Q学習自体は Watkins (1989) が、オフポリシーで最適行動価値に収束する更新則として導入しました。
- DQN — Deep Q-Network (Mnih et al., Nature 2015, "Human-level control through deep reinforcement learning")。提案: Q関数をディープニューラルネットで近似し、Atari ゲームを生のピクセルから人間並みにプレイ。Bellman損失 (Q予測と TD目標の二乗誤差、実際には外れ値に強い Huber 損失) を最小化する。動機: 高次元の画像入力に手作り特徴なしで対応する。新規性: 関数近似で不安定だった TD 学習を、ターゲットネットワークと経験再生という2つの安定化技法で実用化した。
- Prioritized Experience Replay (Schaul, Quan, Antonoglou, Silver, ICLR 2016)。提案: 経験再生のサンプリングを一様ではなく TD誤差の大きさに比例させる。動機: 学べることの多い (誤差の大きい) 遷移を優先的に学べば効率が上がる。新規性: TD誤差を「学習価値の指標」として再利用する発想。
- Double DQN (van Hasselt, Guez, Silver, AAAI 2016)。提案: 通常の DQN が Q値を過大評価する問題に対し、行動の選択と評価を別ネットで分ける。動機: max 演算が「たまたま高く見積もられた行動」を選びがちで、過大評価が累積する。新規性: 「選択は最新ネット、評価はターゲットネット」と分けるだけで過大評価を大幅に抑制。
- Distributional RL — C51 (Bellemare, Dabney, Munos, ICML 2017)。提案: Q値を1つの数ではなくリターンの確率分布として学び、ベルマン更新を分布版に拡張する。動機: 同じ平均でも分布の形に情報があり、それを学ぶと表現が豊かになる。新規性: 分布版ベルマン作用素を導入し Atari の最高水準を更新した。
- IQL — Implicit Q-Learning (Kostrikov, Nair, Levine, ICLR 2022, arXiv:2110.06169)。提案: オフライン RL で「データにない行動を一切評価しない」ために、max の代わりに expectile 回帰で V を学ぶ。動機: オフラインで max を取ると分布外行動を過大評価して破綻する。新規性: Bellman 目標の `V(s')` をデータ内行動だけから安全に推定する方法を与えた ([Advantage-Weighted Regression (AWR) ポリシー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0110_advantage-weighted-regression) と組み合わさる)。

## 論文の実験結果(定量データ)

- DQN (Nature 2015) は、49 本の Atari 2600 ゲームを共通の単一アーキテクチャ・共通ハイパーパラメータで学習し、そのうち過半数 (報告では 29 本) で人間のテストプレイヤーの 75% 以上のスコアを達成、多くのゲームで人間を上回りました。指標は「ゲームスコアを人間スコアで正規化した値」で、100% が人間と同等を意味します。深層強化学習ブームの起点となった結果です。
- DQN のアブレーションが特に重要です。論文は、2つの安定化技法 (ターゲットネットワークと経験再生) をそれぞれ外した場合のスコア低下を報告しています。たとえば Breakout や Seaquest などのゲームで、経験再生を外すとスコアが数分の一から十分の一に低下し、ターゲットネットワークを外しても大幅に悪化しました。これは「素の Bellman/TD 損失は関数近似と組み合わせると容易に発散する」ことの定量的証拠で、安定化が飾りではなく必須であることを示しています。
- Double DQN (AAAI 2016) は、通常の DQN が学習中に Q値を真の値より大きく見積もる (論文中のグラフで、推定 Q値が実際のリターンを系統的に上回る) ことを実測で示し、Double DQN がこの乖離を縮めると同時に、Atari 全体の中央値スコアを向上させたと報告しました。過大評価が単なる理論的懸念でなく、実性能を損なうことの証拠です。
- Rainbow (Hessel et al., AAAI 2018) は、Double DQN、優先度つき経験再生、分布的 RL (C51)、ノイズ付き探索 (NoisyNet)、Dueling アーキテクチャ、multi-step return という6つの改良を統合し、Atari 57 ゲームで当時最高の正規化スコア (中央値で人間の数倍) を達成しました。アブレーションでは6要素を1つずつ外し、優先度つき経験再生と multi-step return の寄与が特に大きいことを示しています。これは「Bellman 目標の作り方 (multi-step) と、どの遷移を学ぶか (優先度) が性能を強く左右する」ことの定量的証拠です。
- IQL (ICLR 2022) の D4RL ベンチマーク結果については、expectile の分位 tau を上げる (V をより楽観的にする) ほど、データ内の良い行動を捉えて性能が上がる一方、上げすぎると不安定になるトレードオフが報告されており、Bellman 目標の `V(s')` をどう推定するかが性能を直接左右することを示しています。

指標の意味: Atari の「正規化スコア」は、ランダム方策を 0%、人間プレイヤーを 100% としたときの相対値で、複数ゲームを横断して比較できるよう設計されています。D4RL の正規化スコアも同様 (0=ランダム、100=専門家) で、これらにより「どのゲーム/タスクでどれだけ強いか」を一つの数で議論できます。

## メリット・トレードオフ・限界

メリット
- 未来全部を待たず、1歩先の見積もりで価値を更新できる (学習が速い、オンライン可能)。
- Q値が手に入ると「どの行動が良いか」を比較でき、優位度を通じて方策改善に直結する ([Advantage-Weighted Regression (AWR) ポリシー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0110_advantage-weighted-regression))。
- 損失は単なる二乗誤差で実装が簡単。
- モンテカルロ法に比べて分散が小さい (1ステップ分の確率変動しか含まない)。

トレードオフ・限界
- ブートストラップによるバイアス。目標が自分の (初期は不正確な) 予測を含むため、誤りが伝播し発散しうる。バイアス (TD) と分散 (モンテカルロ) のトレードオフは TD(lambda) や n-step で調整します。
- 関数近似 + ブートストラップ + オフポリシーの3つが揃うと理論的に発散しうる (deadly triad と呼ばれる)。安定化の工夫 (目標の固定、ターゲットネット、経験再生) がほぼ必須。
- max を取る設計で Q値の過大評価が起きやすい。Double DQN や IQL のような工夫で回避する。
- 割引率 gamma の設定が結果に影響する (大きすぎると見積もりが不安定、小さすぎると近視眼的)。
- 報酬がまばら (sparse) なタスクでは、TD 目標の情報が伝わるのに多くのステップを要し、学習が遅くなる。

未解決の課題: deadly triad を理論的に保証つきで回避する方法、関数近似下での収束保証、まばらな報酬での効率的な価値伝播 (n-step return や後付け報酬整形、Hindsight Experience Replay など) は、現在も研究が続いています。

## 発展トピック・研究の最前線

- 分布的強化学習 (Distributional RL, C51/QR-DQN/IQN, Bellemare et al. 2017〜): Q値を1つの数ではなくリターンの分布として学ぶ。Bellman 更新を分布版にすると性能が上がることが示され、Atari の最高水準を更新しました。分位点回帰版 (QR-DQN, IQN) はリスク考慮の制御にも使えます。
- オフライン RL の保守的手法 (CQL, IQL): データにない行動の Q を意図的に低く見積もる (CQL の正則化項)、あるいはそもそも評価しない (IQL の expectile) ことで、過大評価を抑える方向が主流になっています。
- TD 学習の連続制御版: 連続行動では max が取れないため、TD3 (van Hasselt 系の過大評価対策を連続行動に拡張、2つの Q の小さい方を使う clipped double Q) や SAC (エントロピー正則化つき soft Bellman) が標準で、いずれも Bellman/TD 損失を核に持ちます。
