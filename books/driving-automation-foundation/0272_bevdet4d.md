# BEVDet4D

## ひとことで言うと
BEVDet4D は、カメラだけで3次元物体検出を行う BEVDet に「時間」の軸を加えた拡張です。今のフレームだけでなく少し前のフレームの鳥瞰図(BEV)も一緒に使うことで、物体がどれくらいの速さでどちらに動いているか(速度)を推定できるようにし、検出の安定性も高めました。名前の「4D」は、3次元の空間(XYZ)に時間 T を足した4次元を意味します。鍵となる発想は、自車の動きの分だけ前フレームを正確に位置合わせし、速度推定という難問を「2フレーム間のわずかな位置ズレを見る」という易しい問題へ変換した点にあります。

## 直感的な理解
自動運転では、周りの車や歩行者の位置だけでなく速度を知ることが極めて重要です。前の車が減速しているのか、歩行者が道路へ飛び出してくるのか、それを判断するには動きの情報が要ります。ところが1枚のスナップショットだけからは原理的に速度を測れません。止まった写真を1枚見ても、その車が速いのか止まっているのか分からないのと同じです。速度を知るには、異なる時刻の情報を比べて「どれだけ動いたか」を見る必要があります。

人間も2枚の写真を見比べて「この車はこっちに動いた」と判断します。BEVDet4D はまさにこの「時間をまたいで比べる」仕組みをネットワークへ組み込みました。ただし素朴に2フレームを比べるだけでは落とし穴があります。自車自身が動いているので、止まっている標識ですら前後のフレームで別の位置に写ってしまう。BEVDet4D の巧妙さは、この自車の動きをまず打ち消してから比べる点にあります。電車の窓から外を見ると、止まっている建物が後ろへ流れて見えるのと同じで、観測者(自車)の運動を差し引かないと真の動きが分からないのです。

## 基礎: 前提となる概念
エゴモーション(ego-motion、自車運動)とは、自分の車自身がフレーム間でどれだけ移動・回転したかを表す量です。車載のオドメトリ(走行距離計)や IMU(慣性計測装置)、あるいは地図上での自己位置推定から得られます。これがあると、前フレームの座標系を現フレームの座標系へ正確に変換(剛体変換、平行移動 t と回転 R の組)できます。3次元点 p は p' = R p + t で現フレーム座標へ移ります。

時間融合(temporal fusion)とは、複数の時刻の特徴を組み合わせて1つの予測に使うことです。最も単純な融合はチャネル方向の連結(concatenate、特徴を横に並べてくっつける)で、ネットワークに「2時刻分の情報をまとめて処理しなさい」と渡す形です。より高度なものには再帰的な状態更新(過去を1つの隠れ状態に畳み込む)や attention による参照があります。

速度(velocity)は、位置の時間変化です。BEV 上での物体の位置ズレ Δp と、フレーム間の時間差 Δt から v = Δp / Δt として求まります。だから2フレームの位置ズレを正しく測れれば、速度が出ます。

nuScenes の評価指標 mAVE(mean Absolute Velocity Error、平均絶対速度誤差、単位 m/s)が、速度推定の良さを測る中心指標です。物体ごとの予測速度と正解速度の差の絶対値を平均したもので、小さいほど良いです。これと位置誤差 mATE・向き誤差 mAOE などをまとめた総合点が NDS(nuScenes Detection Score、大きいほど良い)で、速度誤差は NDS にも組み込まれているため、速度が当たると総合点も上がります。

## 仕組みを詳しく
基本は BEVDet と同じ4モジュール(画像エンコーダ → LSS視点変換 → BEVエンコーダ → 検出ヘッド)ですが、視点変換と BEVエンコーダの間に、前フレームの BEV 特徴を融合する仕組みを差し込みます。

1. 現在フレーム(時刻 t)の各カメラ画像から BEVDet と同じく BEV 特徴を作る。形は [B, C, X, Y]。
2. 1つ前のフレーム(時刻 t-1)でも同様に BEV 特徴を作って保持しておく。
3. ここが核心。t-1 の BEV と t の BEV は、自車が動いた分だけ座標がズレている。そこでエゴモーションを使い、t-1 の BEV を t の座標系へ位置合わせ(アライメント、剛体変換)する。実装上は BEV グリッドを変換後の座標で再サンプリング(grid sample、双線形補間)します。
4. 位置合わせした t-1 の BEV と現在の t の BEV をチャネル方向に連結する。形は [B, 2C, X, Y]。
5. これを BEVエンコーダ以降へ流し、検出ヘッドが位置・サイズ・向きに加えて速度も出力する。

なぜ位置合わせが要るのか、具体例で見ます。自車が 0.5 秒で 5m 前進したとします。アライメントなしだと、止まっている標識が BEV 上で 5m 後ろへズレて見え、これを「標識が後ろに動いた」と誤解しかねません。アライメントすると標識のズレは0になり、本当に動いている前方車だけがズレとして残ります。BEVDet4D の重要な発見は、自車運動の分だけ正確に合わせると、静止物の見かけの動きがゼロになり、残った動きが純粋に物体自身の動きになる、という点です。これで速度推定の学習が「絶対速度を当てる」という難問から「2フレーム間の微小な位置ズレを見る」という易しい問題へ単純化されます。論文ではこの単純化が学習を安定させ、速度誤差を大きく下げる鍵だと位置づけられています。

```
時刻 t-1 BEV          時刻 t BEV
[静止標識]──┐         [静止標識]
            │ エゴモーションで t 座標系へ整列
            ▼
整列後 t-1 ≒ t の静止物は重なる
→ 残るズレ = 物体自身の動き = 速度
```

設計上の細かい工夫もあります。前フレームの特徴は視点変換の直後(BEVエンコーダに入る前)で融合します。エンコードする前の生に近い BEV を比べた方が位置のズレ情報が残っていて速度推定に有利だからです(エンコード後だと畳み込みで空間情報がにじみ、微小なズレが潰れる)。また、論文では速度を直接ネットワークに当てさせるより、2フレーム間の位置オフセットとして学習させる方が素直だと示唆されています。さらに、2フレームの時間間隔が大きすぎると動きが大きくなり対応づけが難しく、小さすぎると位置ズレが微小すぎてノイズに埋もれます。適切な間隔(数百ミリ秒程度)が要ります。

## 手法の系譜と主要論文
BEVDet (Huang et al., 2021, arXiv:2112.11790) が単一フレームのカメラ BEV 検出を確立しました([BEVDet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0271_bevdet_note))。これが土台です。

BEVDet4D (Huang & Huang, 2022, arXiv:2203.17054)「BEVDet4D: Exploit Temporal Cues in Multi-camera 3D Object Detection」が、位置合わせした前フレーム BEV を連結する時間モジュールを加え、カメラのみで速度推定を実用水準にしました。動機は、LiDAR ベース手法が速度推定で先行しており、カメラのみ手法でそこへ追いつくことでした。2フレーム連結という最小限の追加で大きな効果を出した点が評価されています。

BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270) は、時系列を時間方向の self/cross-attention で融合する別系統で、過去の BEV を再帰的に参照します。時間情報の活用という問題意識を共有しつつ、融合の機構(明示的なアライメント+連結 vs. attention による暗黙の対応づけ)が異なります。

BEVDepth (Li et al., AAAI 2023, arXiv:2206.10092) は深度を LiDAR で明示監督し、時間融合とは独立した改善として組み合わせ可能です([BEVDepth](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0270_bevdepth))。SOLOFusion (Park et al., ICLR 2023, arXiv:2210.02443) は、短期(高解像度・近接フレーム)と長期(低解像度・多数フレーム)の時間融合を組み合わせ、2フレームに留まらず十数フレームの長い時間窓を効率的に使う方向を切り開きました。StreamPETR (Wang et al., ICCV 2023, arXiv:2303.11926) は、過去の検出をクエリとして次フレームへ伝搬する「ストリーミング」型で、BEV を毎回作り直さず物体単位で時間情報を運ぶ効率的な系統を示しました。BEVDet4D の「2フレーム BEV 連結」から、「多フレーム長期融合」「クエリ伝搬」への発展を示す代表例です。

## 論文の実験結果(定量データ)
nuScenes で評価します。BEVDet4D は、土台の BEVDet に対して mAVE(速度誤差、小さいほど良い)を大幅に下げました。報告によれば、単一フレームの BEVDet では速度推定が苦手で mAVE が大きかった(0.8〜0.9 m/s 程度)のに対し、BEVDet4D は時間融合により mAVE をおよそ半分前後(0.4 m/s 台)まで削減し、総合指標 NDS も明確に改善しました(validation で NDS が数ポイント単位、おおむね 0.04〜0.05 程度上昇したと報告)。速度誤差が NDS の構成要素であるため、mAVE の改善がそのまま総合点を押し上げた格好です。

アブレーションが設計の正しさを裏づけます。BEVDet4D の研究では、(a) エゴモーションによる位置合わせをしない、(b) 前フレームを融合する位置を変える、(c) 速度を直接回帰する形にする、といった改変が比較されました。位置合わせを外すと、静止物の見かけ移動と物体運動が混ざって速度誤差が大きく悪化することが示され、アライメントが効果の主因だと確認されました。また融合をエンコード前の生 BEV で行う方が、エンコード後で行うより位置ズレ情報が保たれ有利だと報告されています。速度を絶対値で当てさせるより位置オフセットとして学習させる方が安定する点も裏づけられました。これらは「時間情報を単に足すのではなく、自車運動を補償してから比べる」ことの重要性を定量的に示しています。

## メリット・トレードオフ・限界
メリットは、カメラのみで物体の速度を推定でき自動運転の予測・計画に必要な動き情報が得られること、前フレーム融合で検出の時間的一貫性(フレーム間でブレない)が増すこと、BEVDet をほぼそのまま拡張でき既存パイプラインへ組み込みやすいこと、追加コストが前フレーム1枚分の保持で済み比較的軽いことです。

トレードオフと限界は次のとおりです。前フレームの保持・処理で計算量とメモリが増えます。エゴモーション情報が必要で、その誤差(オドメトリや IMU のドリフト、タイムスタンプのズレ)が位置合わせを狂わせ速度推定に響きます。そして2フレーム程度の短い時間窓では、長い時間スケールの動き(ゆっくり加減速する車、長時間オクルージョンされる歩行者)の文脈を捉えにくいです。これは SOLOFusion 系の長期融合が補う点です。また、前フレームでオクルージョン(他物体に隠れる)していた物体は対応づけられず、出現直後の物体の速度は不安定になります。

未解決の課題としては、長期時間融合の計算効率と精度の両立、エゴモーション誤差への頑健化、そして移動物体が前フレームでオクルージョンしていた場合の速度推定の安定化が挙げられます。

## 発展トピック・研究の最前線
BEVDet4D の2フレーム融合は、より長い時間窓を効率的に使う研究(SOLOFusion の短期長期分離、StreamPETR のクエリ伝搬など)へと発展しました。長期融合は遠方・低速物体の速度推定や、一時的なオクルージョンからの復帰に効きますが、過去フレームをどこまで保持しメモリ・計算とどう釣り合わせるかが設計の焦点です。時間情報を速度推定だけでなく深度精緻化に使う流れ([BEVStereo](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0274_bevstereo)、連続フレームをステレオ視として扱う)も並走します。BEVDet4D が時間を速度推定に使うのに対し、ステレオ系は時間を深度精緻化に使う、という対比が両者の理解を助けます。さらに、検出だけでなく将来の軌道予測や占有予測(occupancy forecasting)、end-to-end の運転計画へ時間情報を伸ばす研究が、BEV ベースの時間モデリングの最前線になっており、BEVDet4D が示した「自車運動を補償してから時間を比べる」という原則は、これら後続のほぼ全てに共通する基礎になっています。
