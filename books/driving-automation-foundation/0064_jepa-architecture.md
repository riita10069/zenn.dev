# JEPA (Joint-Embedding Predictive Architecture)

## ひとことで言うと
JEPA は「欠けた部分や未来を、生のピクセル(画像の点ひとつひとつの色の値)ではなく、特徴(feature、内容を凝縮した数値の列)の空間で予測する」という自己教師あり学習の枠組みです。画像や動画を1点ずつ復元するのではなく、「だいたいこういう内容になるはず」という抽象的な表現を当てにいきます。予測しても意味のない細部を最初から捨てられるので、ぼやけない・意味のある表現が学べる、というのが核心の主張です。この設計思想は2022年にヤン・ルカン(Yann LeCun)が提唱し、その後 I-JEPA・V-JEPA という実装によって画像・動画で実証されました。

## 直感的な理解
人間がコップを机のふちに置いて手を離したらどうなるかを想像するとき、私たちは「コップが落ちて割れる」という結果のレベルで予測します。落下中にコップの取っ手がどの角度を向くか、破片がどの正確な位置に飛び散るか、床のタイルにどう影が落ちるかまでは想像しません。なぜなら、そういう細部は予測しても役に立たず、しかも一意には決まらないからです。

ところが、もし「未来の画像をピクセル単位で正確に描け」とAIに要求したらどうなるでしょうか。破片の正確な位置や影の形まで当てさせることになります。これらは原理的に予測不能なので、AIは「全部の可能性を平均したぼやけた絵」を出すしかなくなります。平均すれば二乗誤差(予測と正解の差の二乗)の罰が最小になるからです。確率論的には、複数の正解 y のどれが来るか分からないとき、二乗誤差を最小にする1点の予測は「すべての可能な y の平均」になります。だから出力は全候補の平均=ぼやけた絵になるのです。結果として、AIは予測できない細部を無理に再現することに能力を浪費し、肝心の「コップが落ちる」という意味の理解がおろそかになります。

JEPA はこの問題を、予測する場所を変えることで回避します。ピクセル空間ではなく、内容を要約した特徴の空間で予測するのです。特徴の中では「コップが落下した」という情報は残しつつ、「破片の正確な位置」のような予測不能な細部は最初から落とされています。落とされた情報は当てる必要がないので、ぼやけ問題そのものが起きません。自動運転に置き換えれば、「1.6秒後に前方の車がどの位置にいるか」という意味は予測したいが、「そのとき街路樹の葉がどう揺れているか」は予測したくない、という選別を自動でやってくれる枠組みです。

## 基礎: 前提となる概念
JEPA を理解するには、いくつかの用語を押さえておく必要があります。

自己教師あり学習(self-supervised learning)とは、人間が正解ラベルを付けなくても、データ自身の一部を隠して残りから当てさせることで学習する方式です。たとえば文章の一部の単語を隠して残りから当てる(BERT)、画像の一部を隠して復元する、といったやり方です。大量のラベルなしデータをそのまま使えるため、近年の表現学習の主役になっています。

エンコーダ(encoder)は、画像や動画のような高次元の入力を、数百〜数千次元の特徴ベクトル(数値の並び)に圧縮するネットワークです。たとえば 224×224×3 = 150,528 個の数(ピクセル)を 256 個や 1024 個の数に潰す関数だと思ってください。この潰した先の空間を埋め込み空間(embedding space)と呼びます。

世界モデル(world model)は、「現在の状態と自分の行動から、未来の状態を頭の中でシミュレーションする仕組み」です。JEPA はこの世界モデルを、ピクセルではなく特徴空間で実装しようという試みだと位置づけられます。

生成的(generative)手法とは、隠した部分のピクセルそのものを復元させる手法のことです。代表が Masked Autoencoder(MAE)で、これは画像の75%を隠して残り25%から元のピクセルを復元させます。JEPA はこの生成的手法へのアンチテーゼとして提案されました。

対照学習(contrastive learning)も押さえておきましょう。これは「同じ画像の2つの見え方は近く、別の画像とは遠く」を学ぶ手法です(SimCLR など)。「近づける」だけだと全部が1点に潰れる崩壊が起きるので、「別の画像(負例、negative)は遠ざける」という反発力で崩壊を防ぎます。JEPA は負例を使わない非対照型(non-contrastive)に分類され、崩壊回避を別の仕組み(後述の非対称性)で行います。

## 仕組みを詳しく
JEPA は3つの部品でできています。

1. コンテキストエンコーダ(context encoder): 手がかりとなる入力 x(例: 画像の一部、あるいは現在のシーン)を特徴 s_x に変換する。
2. ターゲットエンコーダ(target encoder): 予測したい対象 y(例: 画像の別の一部、あるいは未来のシーン)を特徴 s_y に変換する。
3. 予測器(predictor): s_x と「何を予測するかの手がかり z(当てたい位置の情報や、取った行動など)」を受け取り、s_y の予測 s_y_hat を作る。

学習では予測 s_y_hat と本物の特徴 s_y の距離(典型的には L2 距離、つまり要素ごとの差の二乗を足して平方根を取った値)を小さくします。重要なのは、損失(loss、予測と正解のズレを表す数値。小さいほど良い)をピクセルではなく特徴ベクトル同士で測る点です。

```
x (手がかり)  ──[context encoder]──▶ s_x  ─┐
                                            ├─[predictor(+手がかり z)]─▶ s_y_hat
y (対象)      ──[target encoder] ──▶ s_y  ─┘            │
              (stop-grad, EMA更新)                      ▼
                            loss = distance(s_y_hat, s_y)  を最小化
```

数値例で形のイメージをつかみます。画像を 256 次元の特徴に潰すエンコーダなら、入力 x は [224, 224, 3] のピクセル(15万個の数)から s_x = [256] のベクトル(256個の数)になります。予測も [256] 対 [256] の比較で済み、15万個のピクセルを当てる必要はありません。比較すべき数が桁違いに少なく、しかもその256個には「意味」だけが残っている、というのが利点です。

ここで誰もが直面するのが、表現の崩壊(representation collapse)という失敗です。これは、ターゲットエンコーダが入力に関係なく常に同じ定数ベクトル(たとえば全部ゼロ)を出すように学習してしまう現象です。定数なら予測器も定数を出せば距離はゼロになり、損失は完璧に小さくなります。しかし特徴は何の情報も持たない空っぽのベクトルになり、まったく役に立ちません。距離を小さくするという目標が、皮肉にも「全員が同じ無意味な答えに逃げ込む」インセンティブを生むのです。これを情報理論で言えば、特徴の持つ相互情報量(入力との結びつきの強さ)がゼロになる、エントロピーが消えた状態です。

崩壊を防ぐ標準的な工夫が、非対称性(asymmetry)の導入です。具体的には、ターゲットエンコーダの重みを、コンテキストエンコーダの重みの指数移動平均(EMA、exponential moving average。過去の値を少しずつ混ぜてゆっくり更新する平均。たとえば新しいターゲット重み = 0.999×古いターゲット重み + 0.001×コンテキスト重み)で更新し、ターゲット側には勾配(loss を下げる方向の微分情報)を流しません。これを stop-gradient と呼びます。こうすると、予測器がターゲットを定数へ「引きずろう」としても、ターゲットエンコーダは予測器の都合では動かない(勾配が届かない)ため、両者がそろって定数に逃げることができなくなります。この EMA + stop-gradient + 予測器という3点セットは、BYOL(Grill ら, NeurIPS 2020)が「負例なしでも崩壊しない自己教師あり学習ができる」と最初に示した構造で、JEPA はこれを「予測を特徴空間で位置・行動条件付きに行う」方向へ発展させたものと見ることができます。崩壊回避は JEPA 学習の最大の難所で、ここを外すと学習は一見成功しているように見えて(損失は下がっているように見えて)表現は無意味、という最悪の事態に陥ります。

なぜピクセルでなく特徴空間なのか、をもう一段深く考えます。情報理論的には、予測対象に含まれる「予測不能なノイズ」をどう扱うかの違いです。ピクセル復元では、ノイズ(葉の揺れ、センサーノイズ)も損失に含まれるため、モデルはノイズを表現する容量を割かざるを得ません。特徴空間では、ターゲットエンコーダが先にノイズを捨てた表現を作るので、予測器はノイズを当てる義務から解放されます。これが「予測しても無駄な情報を構造的に無視できる」という JEPA の本質的な優位です。LeCun はこれをエネルギーベースモデル(energy-based model)の言葉で定式化し、「もっともらしい (x, y) のペアには低エネルギーを、ありえないペアには高エネルギーを割り当てる」関数を学ぶ問題として整理しました。崩壊は「すべてのペアに低エネルギーを割り当ててしまう退化解」に相当し、EMA や正則化はこの退化解を避けるための仕掛けとして位置づけられます。

## 手法の系譜と主要論文
JEPA という名前と設計思想を提示した原典は、Yann LeCun の "A Path Towards Autonomous Machine Intelligence"(2022, Meta AI/OpenReview に公開されたポジションペーパー)です。LeCun は「機械に世界モデルを持たせたい。しかし生成的にピクセルを当てる方式は予測不可能な細部に苦しむ。だから抽象表現の空間で予測する Joint-Embedding Predictive Architecture を中心に据えるべきだ」と主張しました。これは実験論文ではなく構想論文で、JEPA はそこで提案された複数モジュール(知覚・世界モデル・アクター・コスト・設定器)の中心に位置づけられます([LeCunの自律機械知能構想](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0069_lecun-jepa-vision) で詳述)。

技術的な系譜をさかのぼると、JEPA の崩壊回避の原型は非対照型自己教師あり学習にあります。Grill ら BYOL(2020)が EMA ターゲット + predictor + stop-gradient で負例なし学習を実現し、Chen ら SimSiam(2021)が「EMA すら無くても stop-gradient だけで崩壊が防げる」ことを示しました。一方、Bardes ら VICReg(2022)や Zbontar ら Barlow Twins(2021)は、非対称性に頼らず、分散・共分散への明示的な正則化で崩壊を防ぐ系譜を作りました。JEPA はこれらの「joint-embedding(2つの入力を同じ空間に埋める)」アーキテクチャに、predictor を介した非対称な予測と位置・行動条件 z を加えた発展形です。

この思想を画像で初めて大規模に実証したのが Assran らの I-JEPA(CVPR 2023, [I-JEPA (Image JEPA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0059_i-jepa_note))です。1枚の画像の一部から隠れた部分の特徴を予測するだけで、ピクセル復元なしに強い表現が学べることを示しました。次に Bardes らの V-JEPA(2024)が、これを動画に拡張し、時間方向の予測(=動きの理解)まで学習させました。さらに V-JEPA 2 / V-JEPA 2-AC(2025, [V-JEPA 2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0101_v-jepa-2))が規模を拡大し、行動を入力に加えてロボット制御にまで接続しました。

思想的な対比軸として重要なのが、Kaiming He らの MAE(Masked Autoencoders Are Scalable Vision Learners, CVPR 2022)です。これは画像の75%を隠してピクセルを復元させる生成的手法の代表で、「隠す割合を高くすると、断片的な手がかりから全体を補う意味理解が必要になる」という発想で高い転移性能を達成しました。JEPA 派は「MAE はピクセルという予測不能な細部にコストを払っている。特徴空間で予測すれば、その無駄を省ける」と批判します。両者の優劣はタスク依存で、現在も活発に比較研究が続いています。

## 論文の実験結果(定量データ)
JEPA 系の有効性を定量的に示した中心が I-JEPA です。評価には主に線形プロービング(linear probing)という方法が使われます。これは、事前学習したエンコーダの重みを凍結(固定)し、その出力特徴の上に「線形分類器」という最も単純な分類器(1層だけの全結合層)を1つ載せて、ImageNet(約128万枚・1000クラスの画像分類ベンチマーク)で分類精度を測るものです。特徴そのものの良さを測る標準的な物差しで、特徴に意味が詰まっていれば単純な分類器でも高精度が出ます。この「凍結+線形」という設定が重要なのは、もし特徴の上に多層のネットワークを載せて学習させると、特徴が悪くても後段が補ってしまい、特徴自体の良さが測れなくなるからです。

I-JEPA の報告によれば、ViT-Huge(約6.3億パラメータの大型 Vision Transformer)を ImageNet で事前学習した場合、線形プロービングで概ね70%台後半のトップ1精度(正解クラスを1位に当てた割合)を達成しつつ、同等性能の生成系手法(MAE など)より少ない学習エポック数・計算量で到達したと報告されています。論文では、I-JEPA は iBOT や MAE などのベースラインに対し、同じ精度に到達するまでの GPU 時間がおよそ数分の1で済んだとされ、これは「ピクセル復元という重い目標を捨てたぶん、学習が速く効率的になる」という JEPA の主張を裏づける結果です。

アブレーション(ある要素を抜くと性能がどう落ちるかを調べる実験)で重要だったのが、ターゲットブロックの大きさとマスキング戦略です。当てさせる領域を小さく散らばったパッチにすると、隣を見ればほぼ分かってしまい、線形プロービング精度が大きく低下したと報告されています。十分大きくまとまった領域(画像の15〜20%程度を占める長方形ブロックを複数)を当てさせることで、初めて「シーン全体の意味」を使う必要が生まれ、表現が意味的になります。また EMA の係数(ターゲットエンコーダの追従の遅さ)も敏感で、追従が速すぎると崩壊しやすく(特徴の分散が小さくなる方向へ進む)、遅すぎると学習が進みにくい、というトレードオフが観察されています。崩壊が起きているかどうかは、特徴ベクトルの次元ごとの分散(rank)をモニタすることで早期に検知できる、というのが実務上の知見です。

## メリット・トレードオフ・限界
メリットとしては、第一に予測不可能なピクセル細部を当てなくて済むので出力がぼやけないこと。第二に計算が軽いこと(15万ピクセルではなく数百〜千次元の比較)。第三に、行動や位置を手がかり z として与えれば「行動条件付きの予測」へ自然に拡張でき、世界モデルや制御に発展させやすいことです。

トレードオフと限界は明確です。表現崩壊が起きやすく、EMA 更新・stop-gradient・予測器の非対称設計など崩壊回避の工夫が必須で、これらのハイパーパラメータに学習が敏感です。損失が「特徴同士の距離」なので、学習中の良し悪しを損失値だけでは判断しづらく(ピクセルなら描いた絵を見れば分かるが、特徴の良し悪しは下流タスクで測るしかない)、評価コストがかかります。さらに、予測の質はターゲットエンコーダの質に上限を縛られます。ターゲットが捨てた情報は原理的に当てられないため、もしターゲットエンコーダが下流タスクに必要な情報まで捨てていると、表現が貧しくなります。これは「ショートカット問題」とも関係し、ターゲットがあまりに圧縮的だと、予測器が自明な低次元の関係だけを学んで止まってしまうことがあります。

未解決の課題として、(1)崩壊回避の理論的な保証がまだ弱く、なぜ EMA + stop-gradient で崩壊が防げるのかの完全な説明が定まっていないこと(BYOL の崩壊回避は batch normalization が暗黙の負例として効いている、という説と、predictor の力学による、という説が議論され続けている)、(2)細粒度のピクセル情報が要るタスク(高精細な領域分割など)では生成系に劣る場合があること、(3)不確実な未来(複数の可能性)を1つの特徴ベクトルに潰すと表現が平均化されてしまうため、潜在変数で多峰性をどう表すかが研究途上であること、が挙げられます。

## 発展トピック・研究の最前線
崩壊回避の仕組みは、JEPA だけでなく自己教師あり学習全般の中心テーマです。stop-gradient 一本で崩壊を防ぐ SimSiam、分散・共分散に明示的な正則化をかけて崩壊を防ぐ VICReg、特徴次元間の相関を抑える Barlow Twins など、非対称性に頼らない崩壊回避の系譜があり、JEPA の理解にはこれらの比較が有益です([Stop-gradient (勾配停止)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0658_stop-gradient), [VICReg](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0661_vicreg_note), [SimSiam](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0093_simsiam))。

応用の最前線は、JEPA を世界モデルとして使い、行動条件付けで制御につなげる方向です。観察動画から学んだ特徴予測器に行動を入力し、未来特徴を予測してモデル予測制御(MPC)で計画する、という流れが V-JEPA 2-AC で示されました。自動運転の文脈では、「未来の俯瞰特徴(BEV 特徴)を予測し、それを軌道計画と結びつける」という設計が、この系譜の自然な延長として研究されています([行動条件付きワールドモデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0003_action-conditioned-world-model))。さらに、不確実な未来を扱うために潜在変数を予測器に入れる確率的 JEPA や、複数の時間スケールを積み重ねる階層型 JEPA(H-JEPA)の実装が、構想段階から実証段階へ移りつつあるのが現状です。
