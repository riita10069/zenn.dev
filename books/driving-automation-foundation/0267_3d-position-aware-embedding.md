# 3D位置認識埋め込み

## ひとことで言うと
2次元のカメラ画像の各点に、「この点はカメラから見て3次元空間のだいたいこのあたり(の光線上)を見ています」という3次元の位置情報を、数のベクトルにして付け足す仕組みです。これにより、平面の画像特徴が「自分の3D位置」を意識できるようになり、明示的な投影サンプリングなしに3次元物体検出や BEV(俯瞰図)推論ができます。これはカメラのみ3D知覚における鍵概念で、とくに [PETR (Position Embedding Transformation)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0287_petr_note) の中核を成します。

## 直感的な理解
カメラ画像は平面(2次元)です。各画素は色や明るさを持ちますが、「自分が3次元空間のどこを見ているか」という情報は明示的には持っていません。人間は経験から奥行きを推測しますが、ニューラルネットワークに3次元の位置を扱わせるには、その位置を数値として教えてあげる必要があります。

同じ問題は自然言語処理にもありました。Transformer(注意機構=どこに注目するかを学習で決めるネットワーク)は、入力された単語の順番をそのままでは認識できません。アテンションが置換不変(入力を並べ替えても同じ出力になる性質)なので、語順の情報が計算で消えてしまうからです。そこで各単語に「あなたは何番目の単語です」という位置埋め込み(positional encoding)を足して順番を教えます。位置を数値として注入することで、ネットワークは順序を扱えるようになります。

3D位置認識埋め込みは、これの3次元版です。画像の各点に「あなたが対応する3次元空間の座標はここです」を数値ベクトルにして足し込みます。ただし、素朴に [x, y, z] の3つの数をそのまま足すだけでは、ネットワークが細かい位置差を区別しにくいことが知られています。理由は後で詳しく説明しますが、簡単に言うと「生の座標値は変化がなめらかすぎて、ニューラルネットが高い空間周波数(細かい位置の違い)を学びにくい」からです。そこで周波数符号化(後述)で表現力の高いベクトルにしてから足します。

## 基礎: 前提となる概念
3つの前提を噛み砕きます。

第1に、視錐台(frustum)です。1台のカメラが見ている空間は、四角錐を途中で切ったような立体(ピラミッドの先を切った形)になります。これを視錐台と呼びます。画像の各画素は、この視錐台の中をカメラから奥へ伸びる1本の光線(ray)に対応します。1枚の画像だけでは、その光線上のどこに物体があるか(奥行き)は決まりません。同じ画素値でも、近くの小さい物体か遠くの大きい物体かの区別が原理的に付かないからです。

第2に、カメラ校正です。3次元の点を画像のどこに写すかは、2種類のパラメータで決まります。intrinsic(内部パラメータ)は焦点距離やレンズ中心などカメラ自身の性質で、3次元のカメラ座標を画像のピクセル座標に対応づけます。extrinsic(外部パラメータ)はカメラが車のどこにどの向きで付いているかで、世界座標とカメラ座標を対応づけます。これらを使うと、画像座標と3次元座標を行き来できます。逆投影(画素→光線)には intrinsic の逆行列を、座標系の統一には extrinsic を使います。

第3に、周波数符号化(positional encoding / Fourier features)です。座標値そのものではなく、sin と cos を複数の周波数で適用した値を並べてベクトルにします。なぜこれが効くのかは、近年「ニューラルネットは低周波の関数を優先して学ぶ性質(spectral bias、スペクトルバイアス)があり、生の座標を入れると高周波の細部を学びにくい。sin/cos で高周波成分を明示的に入力に混ぜると細部が学べる」という形で理論的にも整理されました(Tancik et al., NeurIPS 2020)。

## 仕組みを詳しく
ステップで分解します。記号: V=カメラ台数(6カメラ構成が典型)、C=特徴次元(典型的に256)、H・W=特徴マップの縦横、D=深度候補の数(典型的に64)。

ステップ1: 視錐台の3D点を作る
- 各画素に対応する光線上に、深度候補を D個(例: 1m から 60m を線形または非線形に)離散的に並べます。これで各画素に D個の3D点が割り当たります。形状の感覚は [V, D, H, W, 3](末尾の3は x,y,z)。これはまだカメラ座標系です。
- なぜ複数深度を張るか: 1枚の画像では奥行きが定まらないので、「光線上のどこにいるか」を1点に決めずに、ありうる候補をまとめて埋め込みに含めてしまう、という割り切りです。実際に物体がどの深度かは、後段のアテンションが画像内容と突き合わせて判断します。深度を「決め打ちしない」ことで、深度推定の誤差連鎖を避けられます。

ステップ2: 共通座標系へ変換
- カメラの intrinsic と extrinsic を使い、上の3D点を全カメラ共通の座標系(典型的には自車中心の座標系)に変換します。これで、別々のカメラの点が同じ物差しの上に乗り、6台のカメラの特徴を1つの3次元空間として扱えるようになります。前方カメラの点と後方カメラの点が、同じ自車座標 (x,y,z) で表せるわけです。

ステップ3: 周波数符号化
- 座標 (x, y, z) をそのまま使う代わりに、各成分に複数周波数の sin/cos を適用します。たとえば x について [sin(2⁰πx), cos(2⁰πx), sin(2¹πx), cos(2¹πx), ..., sin(2^{L-1}πx), cos(2^{L-1}πx)] と並べ、y, z も同様にして連結します。低い周波数は大まかな位置、高い周波数は細かい位置差を表します。L が周波数の段数(バンド数)です。
- 数値例: x が 10m と 10.1m のとき、生の値の差は 0.1 と小さく、ネットワークには違いがほとんど見えません(全結合層を1回通すと埋もれる)。しかし高周波の sin/cos を通すと、たとえば cos(2⁵π·10) と cos(2⁵π·10.1) は大きく違う値になり、その 0.1 の差が大きな振動の違いとして現れ、近接した位置でも区別できるようになります。

ステップ4: 位置埋め込みネットワークでC次元化し、画像特徴に足す
- 符号化したベクトルを小さな MLP(多層パーセプトロン、数層の全結合ネットワーク)に通して、画像特徴と同じC次元にそろえます。D個の深度ぶんの埋め込みを集約して、画素ごとに1本のC次元ベクトルにまとめます。
- それを画像特徴に足し算します(実装によっては結合)。結果が「3D位置認識特徴(3D position-aware features)」。形状は [V, C, H, W] のまま、中身に位置の意味が乗ります。

before/after の感覚: before は「色・テクスチャ・形は分かるが、自分の3D位置を知らない特徴」。after は「色・テクスチャ + 自分が3次元空間のどの光線上にいるかを知っている特徴」。この after があるからこそ、オブジェクトクエリが幾何投影なしに3D位置を推論でき、デコーダは普通のグローバルアテンションだけで済みます。クエリと特徴の内積が「内容の一致度 + 位置の一致度」を同時に測るようになる、と理解すると本質が掴めます。

## 手法の系譜と主要論文
位置を数値ベクトルで注入する、という発想の系譜を追います。

Attention Is All You Need (Vaswani et al.; NeurIPS 2017)
- sin/cos による位置符号化(positional encoding)を導入。系列内の単語の絶対位置を、複数周波数の sin/cos で表したベクトルとして特徴に足す。3D位置認識埋め込みはこの考えを3次元座標へ拡張したものです。1次元の「順番」を3次元の「座標」に置き換えた、と見るのが分かりやすい。

NeRF (Mildenhall, Srinivasan, Tancik, Barron, Ramamoorthi, Ng; ECCV 2020, arXiv:2003.08934)
- 3次元座標を高周波の sin/cos で符号化してから MLP に入れると、シーンの細部(高周波成分)を鮮明に再構成できることを示した。符号化なしでは出力がぼやけることを実例で示しており、3D位置埋め込みの周波数符号化と同じ動機(生座標では細部が学べない)を共有します。

Fourier Features (Tancik, Srinivasan, Mildenhall, Fridovich-Keil, Raghavan, Singhal, Ramamoorthi, Barron, Ng; NeurIPS 2020, arXiv:2006.10739)
- 「MLP は低周波の関数を優先して学ぶ(spectral bias)ため、入力に Fourier 特徴を加えると高周波関数の学習が劇的に改善する」ことを、Neural Tangent Kernel(無限幅ネットの学習を解析する理論)で裏付けた。周波数符号化が経験則ではなく理論的に妥当であることを示した論文です。

PETR (Liu, Wang, Wang, Zhang, Sun; ECCV 2022, arXiv:2203.05625)
- 提案: 視錐台の3D座標を共通座標系へ変換し、周波数符号化して MLP でC次元化、画像特徴に注入する 3D position embedding(3D位置認識埋め込み)。
- 動機: 画像特徴に最初から3D位置を持たせれば、[DETR3D](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0278_detr3d_note) のような毎回の投影サンプリングが不要になり、デコーダは普通のグローバルアテンションでよくなる。
- 新規性: NLP/NeRF の位置符号化を、多視点カメラの3D検出という具体的なタスクに「視錐台 + 共通座標系変換 + 周波数符号化」という形で落とし込み、それが実際に機能することを示した点。

PETRv2 (Liu et al.; ICCV 2023, arXiv:2206.01256)
- 過去フレームの3D位置埋め込みを ego-pose で現在系へ整列して時系列を入れ、さらに画像特徴の内容で位置埋め込みを調整する feature-guided 版に拡張。位置埋め込みを「純粋な幾何」から「特徴に適応する幾何」へ進化させました。詳細は [PETRv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0289_petrv2_note)。

## 論文の実験結果(定量データ)
3D位置認識埋め込みそのものの効果は、PETR のアブレーションで最もはっきり示されます。評価は nuScenes(6カメラのサラウンド自動運転データ、10クラス3D注釈)で、指標は mAP(中心距離ベースの平均精度、高いほど良い)と NDS(位置・寸法・向き・速度・属性の誤差を合成した総合スコア、0〜1、高いほど良い)です。指標の詳細は [DETR3D](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0278_detr3d_note) の実験節を参照してください。

PETR 論文の主要なアブレーションの示すところ:
- 3D位置埋め込みを取り除き、2次元の位置符号化(画像内の u,v 座標だけ)に置き換えると、mAP・NDS が大きく低下します。つまり「3次元の」位置を特徴に焼き込むことが、性能の中核であると確認されました。位置情報を2Dに留めると、デコーダは奥行きの手がかりを失い、3D 検出の精度が崩れます。3D 埋め込みなしでは検出がほとんど成立しない、というのが報告の趣旨です。
- 周波数符号化を外して生座標を直接 MLP に入れると精度が下がる、という観察も報告されています。spectral bias の理論と整合し、近接位置の区別が難しくなるためと解釈できます。
- 深度候補 D の範囲・粒度を変える実験では、範囲が物体の存在する距離帯をカバーしていることが重要で、狭すぎると遠方を取りこぼします。D の値そのものには頑健性があり、ある程度の点数を確保すれば性能は安定する傾向が示されています。

PETR 全体の代表値はおよそ、ResNet-50 系で mAP 0.339・NDS 0.403、大規模バックボーン(VoVNet-99)+高解像度で nuScenes test NDS 0.481・mAP 0.434 前後です。これらは 3D位置認識埋め込みを土台に達成されており、この埋め込みが無ければ成立しません。

## メリット・トレードオフ・限界
メリット
- 画像特徴が3D位置を自前で持てるので、デコーダ側は投影サンプリング不要のシンプルなアテンションでよい。実装が素直になる。
- 周波数符号化により近距離・遠距離の位置差を細かく表現でき、ネットワークが高周波の空間的な区別を学習しやすい。
- 複数カメラを共通座標系の位置埋め込みでそろえられ、視点統合が自然に行える。明示的な後処理ルールなしに6台が1空間に溶け込む。
- 1枚の画像で奥行きが定まらない問題を、光線上の複数深度をまとめて埋め込むことで回避し、最終判断を後段のアテンションに委ねられる。深度を決め打ちしないので誤差連鎖が起きない。

トレードオフ・限界
- 深度候補の範囲・粒度(視錐台設計)が性能に直結し、タスクや距離帯に応じた調整が必要。
- 位置埋め込みは校正(intrinsic/extrinsic)が正しい前提で意味を持つ。校正がずれると位置の意味そのものがずれ、精度が落ちる。実車の振動・経年による校正ずれは現実的なリスク。
- 生座標でなく高次元化するため、メモリ・計算がやや増える。とくに D 個の深度候補ぶんの埋め込みを持つコストがある。
- 純粋な幾何の埋め込みは画像内容を考慮しないため、暗所や遠方など不確かな領域でも一律の位置重みを与えてしまう。これを画像特徴で調整しようとしたのが PETRv2 の feature-guided 位置エンコーダです。

研究上の未解決課題としては、深度を陽に分布として学ぶべきか(LSS 系のように)暗にアテンションへ委ねるべきか、周波数符号化の最適な周波数帯をどう選ぶか(高すぎるとノイズ、低すぎると区別不足)、校正誤差にどう頑健化するか、といった論点が残ります。

## 発展トピック・研究の最前線
- 特徴適応型の位置埋め込み: 画像特徴の内容で位置埋め込みを調整し、不確かな領域の重みを下げる方向([PETRv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0289_petrv2_note) の feature-guided position encoder)。
- 利用先としての検出器: この埋め込みを土台にした end-to-end 多視点3D検出([PETR (Position Embedding Transformation)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0287_petr_note))と、その時系列・多タスク拡張([PETRv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0289_petrv2_note))。
- 設計思想の対比: 位置を特徴に「焼き込む」本手法に対し、3Dクエリを画像へ「投影してサンプリングする」[DETR3D](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0278_detr3d_note) や、BEV グリッドを陽に構築する [空間クロスアテンション (SCA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0316_spatial-cross-attention) という対照的な路線がある。三者を比べるとカメラのみ3D知覚の設計空間が見渡せる。
- 理論面: Fourier 特徴と spectral bias の理解は、座標を入力に持つあらゆるネットワーク(NeRF、暗黙関数 implicit function、座標MLP、3D Gaussian Splatting の前段など)に共通する基盤であり、3D知覚を超えて応用が広い。座標をどう符号化するかは、3D 深層学習に通底する設計問題です。
