# 潜在空間予測 vs ピクセル予測

## ひとことで言うと

「未来を予測する」と一口に言っても、予測のターゲットには大きく2つの流派があります。1つは「未来の映像を1ピクセルずつ描き直す」やり方(ピクセル予測、生成的アプローチ)、もう1つは「映像の意味を圧縮した抽象的な数値ベクトルだけを当てにいく」やり方(潜在空間予測、非生成的アプローチ)です。後者は、雨粒の形や葉の揺れといった「予測しても意味のない細部」を無視できるため、「物体がどう動くか」「シーンがどう変化するか」という本質だけを知りたい場面で有利になります。この設計選択は、世界モデル(world model)・自己教師あり表現学習・自動運転の知覚予測に共通する根本的な分岐点であり、近年の表現学習の最大の論点の1つです。

## 直感的な理解

人間が運転や歩行をするとき、私たちは「目の前の景色のすべてのピクセルを脳内で再描画」しているわけではありません。「前の車がブレーキランプを点けた、たぶん減速する」「右から子どもが飛び出しそうだ」という、行動に関わる本質だけを予測しています。雨粒が次の瞬間どこに落ちるかや、街路樹の葉が風でどう揺れるかは、予測してもほぼ当たらないし、当たったところで判断に何の役にも立ちません。

機械に未来予測を学ばせるときも同じ問いに直面します。学習信号(損失)を「映像全体のピクセル誤差」にすると、モデルは予測不可能な細部を必死に当てようとして学習リソースを浪費します。情報理論の言葉でいえば、自然画像のピクセルが持つ情報量(エントロピー)の大半は、物体の意味とは無関係なテクスチャ・照明・ノイズに費やされています。そこで「映像をいったん意味の本質に圧縮し、その本質の未来だけを予測する」という発想が出てきます。これが潜在空間予測です。LeCun の比喩を借りれば、「世界の予測不可能な部分を表現するためにモデルの容量を使うべきではない」のです。

## 基礎: 前提となる概念

### 潜在空間とは

「潜在空間(latent space)」とは、画像などの高次元データをニューラルネットワークで圧縮した、低次元の数値ベクトルが住む空間のことです。「潜在(latent)」は「表に出ていない、隠れた」という意味で、ピクセルの裏に隠れた「意味の本質」を表す数値だと思ってください。たとえば 1920×1080×3 ≈ 622万次元の画像を、エンコーダ(encoder: 画像を圧縮するネットワーク)で 256〜1024次元のベクトルに圧縮します。この少数の数値に「前方に車がいる」「自車は直進中」といった意味が凝縮され、葉の揺れのような細部は圧縮の過程で捨てられます。

### 生成モデルと識別モデル

ピクセル予測は「生成モデル(generative model)」の一種です。生成モデルはデータそのもの(ピクセルの確率分布 p(x))をモデル化し、新しいサンプルを描き出せます。一方、潜在空間予測は「予測対象を抽象表現に置き換える」点で、データを描き出さない非生成的アプローチです。両者の中間に「潜在を生成するが画素には戻さない」設計や「潜在空間で拡散させる(latent diffusion)」設計もあり、境界はグラデーションになっています。

### エネルギーベースの視点(EBM)

LeCun の枠組みでは、予測を「ある入力 x と出力 y の組がどれだけ整合的か」を測るエネルギー関数 E(x, y) として捉えます。生成モデルは E を確率に正規化しようとする(分配関数の計算が重い)のに対し、JEPA はエネルギーが低い領域だけを学べばよく、正規化を回避できます。「y を直接生成しない」という設計はこのエネルギーベースモデル(energy-based model, EBM)の発想に根ざしています。

### 自己教師あり学習

ラベル(正解タグ)を人間が付けずに、データ自身の構造から学習信号を作る枠組みを「自己教師あり学習(self-supervised learning, SSL)」と呼びます。「画像の一部を隠して残りから当てる」「現在から未来を当てる」といったタスクで、人手のラベルなしに表現を獲得します。潜在 vs ピクセルの議論は、この SSL の目的関数をどこで定義するか(画素空間か特徴空間か)という問題でもあります。

## 仕組みを詳しく

両者の処理の流れを tensor shape(テンソルの形: 数値配列の各次元のサイズ)で比較します。バッチサイズを B=8、画像を 224×224 RGB とします。

### ピクセル予測の流れ

```
入力画像          [8, 3, 224, 224]   ← 現在のフレーム
   │ エンコーダ
潜在ベクトル z_t   [8, 256]
   │ ダイナミクスモデル(時間発展を予測)
未来潜在 z_{t+1}   [8, 256]
   │ デコーダ(decoder: ベクトルから画像を再構成)
予測画像          [8, 3, 224, 224]   ← ここまで戻す
   │
損失 = 予測画像 と 実際の未来画像 のピクセルごとの差(例: MSE)
```

ポイントは最後にデコーダで画像へ戻すこと。損失はピクセル単位(3×224×224 = 150,528 個の数値の差)で計算されます。

### 潜在空間予測の流れ(JEPA 系)

```
現在画像          [8, 3, 224, 224]
   │ コンテキストエンコーダ(学習対象、勾配が流れる)
現在潜在 z_t       [8, 256]
   │ 予測器(predictor)
予測した未来潜在 ẑ_{t+1}  [8, 256]

未来画像          [8, 3, 224, 224]   ← 実際の1秒後(または隠したブロック)
   │ ターゲットエンコーダ(別系統、勾配を流さない = stop-gradient)
本物の未来潜在 z_{t+1}     [8, 256]

損失 = ẑ_{t+1} と z_{t+1} の差(256次元ベクトル同士の距離、例: smooth L1)
```

ポイントは画像に戻さないこと。損失は256次元の特徴空間で計算され、150,528 次元ではなく256次元の比較なので、無意味な細部はそもそも損失に含まれません。予測器(predictor)は通常コンテキストエンコーダより小さい補助ネットワークで、「文脈表現から欠落部分の表現へ写像する」役割だけを担います。

### 数値例で効果を実感する

雨のシーンで1秒後を予測する場合を考えます。前方の車は「2メートル前進した」という予測すべき重要情報がある一方、雨粒は「ランダムに違う位置に降る」という予測不可能・無意味な情報です。ピクセル予測モデルは、雨粒の位置が違うだけで巨大なピクセル誤差を被ります。150,528 ピクセルのうち雨粒部分が数万ピクセルあれば、MSE 損失の大半が「予測不可能な雨粒」由来になり、肝心の「車の前進」の学習信号がノイズに埋もれます。確率的に言えば、ピクセル予測モデルはぼやけた平均画像(複数の可能な未来の平均)を出力して損失を下げる方向に陥りやすく、これが生成系で予測映像がぼやける根本原因です。潜在空間予測なら、エンコーダが「雨粒は本質でない」と学習して圧縮時に捨てるため、256次元ベクトルには「車が2m前進」という情報だけが残り、そこを正確に予測する方向へ学習が進みます。

### なぜ崩壊するか、なぜ防げるか

潜在空間で予測損失だけを最小化すると、エンコーダが「すべての入力を同じ定数ベクトルに潰す」とズルをして損失をゼロにできてしまいます。ẑ も z もすべて同じ値なら差は常にゼロです。これを「表現崩壊(representation collapse)」と呼びます。崩壊を防ぐ代表的な工夫は次の系統です。

第一に、ターゲットエンコーダをコンテキストエンコーダと分離し、ターゲット側へは勾配を流さない(stop-gradient)。これにより「両方を同時に定数へ潰す」近道が断たれます。第二に、ターゲットエンコーダの重みをコンテキスト側の EMA(exponential moving average: 指数移動平均、例 θ_target ← τ·θ_target + (1−τ)·θ_context、τ≈0.996)でゆっくり更新し、教師信号が急激に定数へ潰れないようにする(BYOL/I-JEPA 系)。第三に、分散・共分散の正則化を明示的に加える(VICReg 系)。分散項は各次元の標準偏差が一定値を下回らないよう罰則を与え(崩壊を直接防ぐ)、共分散項は次元間の相関を減らして情報の冗長を抑えます。第四に、対照学習(contrastive)で負例(別画像の表現)を遠ざけ、全部が同じ点に集まれないようにする。これらは「崩壊を防ぐ」という同じ目的への異なる処方箋であり、どれを選ぶかが手法の系統を分けます。

## 手法の系譜と主要論文

### 自己教師あり表現学習の源流(2018–2020)

潜在で予測する発想の源流の一つは対照学習(contrastive learning)です。Chen らの SimCLR(ICML 2020, arXiv:2002.05709)、He らの MoCo(CVPR 2020, arXiv:1911.05722)は「同じ画像の2つの拡張(crop, color jitter など)は近く、別画像は遠く」を潜在空間で学習しました。負例を大量に必要とする(SimCLR は巨大バッチ、MoCo はキューでまかなう)のが弱点でした。Grill らの BYOL(NeurIPS 2020, arXiv:2006.07733)は負例を一切使わず、stop-gradient と予測器と EMA ターゲットだけで崩壊を防ぐ非対照型を示し、「潜在予測でも崩壊しない設計が可能」ことを実証して大きな衝撃を与えました。Chen と He の SimSiam(CVPR 2021)は EMA すら外し、stop-gradient だけで崩壊が防げることを示して、何が崩壊防止に本質的かの理解を深めました。

### ピクセル再構成側: MAE(2021)

He らの Masked Autoencoder(MAE, CVPR 2022, arXiv:2111.06377)は、画像の 75% をマスクして残り 25% から「マスク部分のピクセル」を再構成する事前学習です。エンコーダは見えている 25% のパッチだけを処理するので計算が軽く、ViT を高効率に学習できます。ImageNet 微調整で当時最高水準を達成しました。MAE はピクセルを教師信号とするため崩壊しにくく解釈も容易ですが、低レベルなテクスチャ復元に容量を割く点が後の I-JEPA に批判されます。

### LeCun "A Path Towards Autonomous Machine Intelligence"(2022)

Meta の Yann LeCun(2018年チューリング賞)が2022年に公開したポジションペーパーで、JEPA(Joint-Embedding Predictive Architecture: 結合埋め込み予測アーキテクチャ)という設計思想を提唱しました。生成的に画素を描き直すのではなく、入力と未来の両方を埋め込みに変換し、埋め込み同士で予測する非生成的 SSL を将来の自律知能の中核に据えるべきだ、という主張です。階層的 JEPA(H-JEPA)による多時間スケールの予測、世界モデルと行動計画の統合などのロードマップも示しました。

### Assran et al. "I-JEPA"(Meta AI, CVPR 2023, arXiv:2301.08243)

画像版 JEPA の最初の本格的実装。1枚の画像から、空間的にまとまった大きなコンテキストブロックを取り、そこから隠した複数のターゲットブロックの「特徴表現」を予測します(ピクセルではない)。生成も対照学習もデータ拡張(手で設計した crop/color)も使わず、マスク戦略だけで意味表現を獲得する設計です。

### Bardes et al. "V-JEPA"(Meta AI, 2024, arXiv:2404.08471)

動画版。動画の時空間ブロック(連続フレームにまたがる3Dの塊)をマスクし、特徴空間で予測します。動画特有の「時間的に予測可能な構造」を画素を描かずに学ぶ点が新規で、動作理解系のベンチマークで強い転移性能(凍結特徴での評価)を示しました。2025年には行動条件を加えた V-JEPA 2 へと発展し、ロボット操作の世界モデルとして物理的予測に踏み込んでいます。

### Dreamer 系のハイブリッド(Hafner et al., 2019–2023)

強化学習の世界モデルとして潜在空間でロールアウト(想像上の未来展開)する流派です。詳しくは [Dreamer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0047_dreamer-v1) [DreamerV2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0048_dreamer-v2) [DreamerV3](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0049_dreamer-v3) を参照。Dreamer は潜在空間で予測しますが、学習安定化のために画像再構成損失も補助的に併用するハイブリッドで、純粋な JEPA とは異なります。後継の DreamerV3 では再構成を維持したまま symlog 等で頑健化しており、「再構成は崩壊防止と表現の接地に有効」という立場を取ります。

### 生成系世界モデル: GAIA-1(Wayve, 2023, arXiv:2309.17080)

自動運転企業 Wayve は逆に、未来の運転映像を実際に生成する世界モデル GAIA-1 を発表しました。約 90 億パラメータ規模で、4,700 時間の運転データから学習し、画像・テキスト・行動を離散トークン化して自己回帰 Transformer で次のトークンを予測、拡散デコーダで動画に戻します。動機は「人間がシーンを目で確認でき、レアケースのデータ拡張やシミュレーションに使える」という実用性です。生成派の代表として、潜在派の効率と対をなす位置にあります。

## 論文の実験結果(定量データ)

### I-JEPA vs MAE(ImageNet, 線形評価)

線形評価(linear probing: 凍結した特徴の上に線形分類器1層だけを載せて精度を測る、表現の良さの代表指標)で、I-JEPA は ViT-Huge/14 で ImageNet-1k のトップ1精度およそ 72–77% 帯を達成し、同等の事前学習計算量で MAE を上回ったと報告されています。とくに重要なのは計算効率で、I-JEPA はピクセルを復元しないためエンコーダがターゲット表現だけ扱えばよく、MAE や iBOT より少ない GPU 時間で同等以上の表現を獲得したとされます(論文では収束までの事前学習エポックが大幅に少ないと報告)。「ピクセル復元という重い作業を省いても、むしろ意味的に良い表現が得られる」という、潜在派にとって象徴的な結果です。低レベル特徴に依存しないため、物体カウントや深度予測といった低密度ラベルタスクへの転移でも MAE を上回ったと報告されています。

### V-JEPA(動画下流タスク)

V-JEPA は Kinetics-400(動作分類)や Something-Something v2(時間的推論を要する動作認識、物体の見た目より「どう動いたか」が効く)で、凍結特徴の評価(attentive probe)において画素再構成型(VideoMAE 等)や画像事前学習を転用した手法を上回る転移性能を示しました。とくに「時間順序を入れ替えると解けない」タイプのタスク(SSv2)で優位という報告は、画素を描かずとも時間的因果構造を捉えられることを示します。凍結バックボーンのまま複数の画像・動画タスクへ強く転移する点が、汎用表現としての強みです。

### アブレーション(何を抜くとどうなるか)

潜在予測系の実験で繰り返し確認されるのは次の点です。第一に stop-gradient や EMA ターゲットを外すと表現崩壊が起き、線形評価精度がほぼランダム水準(ImageNet なら 0.1% 近辺)まで落ちます(BYOL/SimSiam 系の典型的アブレーション)。SimSiam は「stop-gradient こそが崩壊防止の本質で、予測器はそれを助ける」ことをアブレーションで示しました。第二に I-JEPA ではマスク戦略が決定的で、ターゲットを十分大きく(画像面積の 15–20% 程度)・空間的にまとまったブロックにし、コンテキストを広く取ると性能が大きく改善し、小さくバラバラなマスクだと低レベルな手がかり(隣接ピクセルの補間)に頼って意味表現が育ちません。第三に MAE 系では予測対象を生ピクセルから正規化ピクセルや HOG 特徴・離散トークンに変えると精度が変動し、「何を予測対象にするか」が表現の質を直接左右することが示されています。

## メリット・トレードオフ・限界

潜在空間予測のメリット: 予測不可能・無意味な細部を無視でき学習信号が本質に集中する。損失計算が低次元で軽く事前学習が高速。得られる表現が抽象的で制御・分類・検出など下流タスクに転用しやすい。マルチモーダルな未来(複数の可能性)を平均でぼかさずに扱いやすい。

潜在空間予測の限界: 表現崩壊のリスクがあり stop-gradient・EMA・正則化など安定化の工夫が必須で、ハイパラ(EMA の τ、分散項の重み等)に敏感。予測結果を人間が目で見て確認できずデバッグが難しい。「何を捨てるべきか」をエンコーダが誤ると重要情報(小さいが安全に効く物体など)まで捨てる恐れがある。さらに、評価指標が「特徴距離」になるため、何を学べているかの解釈が難しく、ベンチマーク設計自体が研究課題として残ります。

ピクセル予測のメリット: 予測映像を直接見られ解釈性・検証性が高い。データ拡張やシミュレーション用の映像生成(closed-loop sim、レアケース合成)に使える。明確な教師信号があり崩壊しにくい。テキストや行動で条件付けて「もしこうしたら」を可視化できる。

ピクセル予測の限界: 予測不可能な細部に学習リソースを奪われ、複数の未来の平均としてぼやけがち。高解像度・長尺の動画生成は計算コストが極めて高い(GAIA-1 は数十億パラメータ規模)。生成品質の高さ(FID 等)が必ずしも下流制御性能の高さを意味しない、という乖離も指摘されています。

未解決の課題として、「潜在派と生成派のどちらが世界モデルとして優れるか」は決着していません。生成は検証・データ生成・人間との対話に強く、潜在は効率と転移に強い。両者を統合する(潜在で計画し、必要時だけ画素を生成して検証する)ハイブリッドが現実的な方向と見られています。また、潜在表現が「行動に必要な情報をすべて含んでいるか」を保証する理論はまだ乏しく、安全クリティカルな応用では「捨てた情報が本当に不要だったか」の検証が課題です。

## 発展トピック・研究の最前線

近年は潜在拡散モデル(latent diffusion, Rombach et al. 2022)が両者の橋渡しになりつつあります。VAE で潜在に圧縮してから拡散で生成すると、画素生成の利点(検証可能性)を保ちつつ計算を潜在空間に押し込められるためです。Stable Diffusion 系の動画版や、自動運転の世界モデル(GAIA、DriveDreamer、Vista など)はこの路線です。自動運転の世界モデルでは、潜在予測でコンパクトに未来を展開しつつ、評価や可視化のときだけデコードする設計が増えています。

また JEPA を行動条件付き(action-conditioned)に拡張し、「ある行動を取ると潜在がどう変わるか」を学ぶ研究(V-JEPA 2 など)は、世界モデルと表現学習の融合点として活発です。崩壊の理論的理解(なぜ stop-gradient が効くのかの解析、White ら・Tian らの研究)、予測対象(どの抽象度の特徴を当てるべきか)の設計、潜在予測の評価指標の確立が、今後の中心的な研究テーマです。最終的には「効率の潜在」と「検証の生成」をタスクに応じて使い分け・統合する設計論に収束していくと見られています。
