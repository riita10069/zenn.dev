# Focal Loss

## ひとことで言うと
Focal Loss(フォーカルロス、焦点損失)とは、すでに正しく分類できている「易しい例」の損失への寄与を自動的に下げ、まだ間違えている「難しい例」に学習を集中させる損失です。物体検出のように「背景ばかりで対象がごくわずか」という極端なクラス不均衡で威力を発揮します。通常のクロスエントロピーに `(1 - p)^γ` という減衰係数を掛けるだけで、当たっている例の信号を弱め、外している例の信号を残します。focal は「焦点を当てる」という意味です。

## 直感的な理解
1万人の生徒を採点する先生を想像してください。9990人はもう完璧に解けていて、10人だけが難問でつまずいています。先生の労力(=損失と勾配)が9990人の○つけに吸い取られてしまうと、本当に指導が要る10人の声がかき消されます。Focal Loss は「もう満点に近い生徒の答案は流し見し、つまずいている生徒に時間を集中する」採点方針です。

なぜこれが要るのか。物体検出では画像の大半が「背景」で、その背景の大多数は「明らかに背景」と一目で分かる易しい例です。易しい背景例は1つ1つの損失は小さくても、数が万単位なので合計すると圧倒的になり、本当に学びたい「まれな物体」や「紛らわしい難例」を押し流します。[クラス重み付き損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0349_class-weighted-loss) のクラス重み付けは「背景クラスを軽く、物体クラスを重く」とクラス単位で補正しますが、それでも「簡単に背景と分かる例」が大量にある問題は残ります。Focal Loss はクラスではなく「その例がどれだけ易しいか」で重み付けすることで、この残った問題を直接叩きます。

## 基礎: 前提となる概念
まず物体検出と密な分類(dense classification)。物体検出は画像のどこに何があるかを当てる課題です。多くの検出器は画像上に候補位置(アンカー、anchor)を密に何万個も置き、各候補について「物体か背景か」「どのクラスか」を分類します。1段階検出器(SSD、YOLO、RetinaNet など)はこの分類を候補の事前絞り込みなしに密に行うので速い一方、背景候補が桁違いに多い極端な不均衡にさらされます。1画像あたり背景アンカーが10万のオーダーに対し物体アンカーは数個〜数十個、という比率は珍しくありません。

次にクロスエントロピーと p_t という記法。クロスエントロピーは正解確率を上げる損失で、ここでは正解クラスにモデルが付けた確率を p_t と書きます(正解に自信があるほど p_t は1に近い)。二値分類なら、正解が「物体」のとき p_t はモデルが出す物体確率 p、正解が「背景」のとき p_t は `1-p` です。通常のクロスエントロピーは `CE = - log(p_t)` です。詳しくは [クロスエントロピー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0352_cross-entropy-loss) を参照してください。

最後に「易しい例/難しい例」の定義。p_t が1に近い例は、モデルがすでに高い確信で正解できている=易しい例。p_t が小さい例は、モデルがまだ正解できていない=難しい例です。Focal Loss の着眼点はまさにこの p_t の大きさです。

## 仕組みを詳しく
Focal Loss は通常のクロスエントロピー `- log(p_t)` に減衰係数(変調因子、modulating factor)`(1 - p_t)^γ` を掛けます。γ(ガンマ)は集中度を決めるハイパーパラメータで、原論文の定番は γ = 2 です。

```
CE  = - log(p_t)
FL  = - (1 - p_t)^γ * log(p_t)
```

この `(1 - p_t)^γ` が肝です。γ = 2 での効き目を数値で見ます。

```
易しい例   p_t = 0.9 : (1-0.9)^2 = 0.01   → 損失を 100分の1 に減衰
やや易しい p_t = 0.7 : (1-0.7)^2 = 0.09   → 約 11分の1
難しい例   p_t = 0.3 : (1-0.3)^2 = 0.49   → 約半分(ほぼ通常通り)
すごく難   p_t = 0.1 : (1-0.1)^2 = 0.81   → ほぼそのまま
```

よく当たっている例ほど損失をぐっと小さくし、当たっていない難例はほぼそのまま残します。結果、大量の易しい背景例が総損失を支配することを防ぎ、勾配が難例に集中します。γ=0 にすると `(1-p_t)^0 = 1` なので、Focal Loss はただのクロスエントロピーに戻ります。つまり Focal Loss はクロスエントロピーの連続的な一般化で、γ を上げるほど易しい例の抑制が強くなります。

実務では、クラス頻度の不均衡を直接補うクラス重み α と組み合わせた α-balanced 版がよく使われます。

```
FL = - α_t * (1 - p_t)^γ * log(p_t)
```

α_t は正解が物体なら α、背景なら `1-α`(α=0.25 が定番)で、頻度の偏りを補正します。ここで重要なのは α と γ が別々の病を治すことです。α はクラス頻度の不均衡(背景が多い)を、γ は例の易しさの偏り(易しい背景が多い)を補正します。両方あって初めて密な検出の不均衡が御せます。

検出では各クラスを独立に sigmoid で「そのクラスか否か」を判定する二値版(sigmoid focal loss)で実装するのが一般的です(softmax の多クラス版より勾配の振る舞いが扱いやすく、クラス間で勾配が干渉しない)。

なぜこの設計なのか。当初の素朴な対策は「背景候補を間引いて物体候補と数を揃える」ハードネガティブマイニング(OHEM など、損失の大きい難例だけ選んで学習)でした。しかしこれは勾配の不安定や情報の捨てすぎ(易しい例を完全に無視)を招きます。Focal Loss は全候補を使ったまま、易しい例の寄与を連続的に下げるので、サンプリングのヒューリスティクスなしに同じ効果を滑らかに得られる、という点が設計上の優位です。

加えて原論文は学習初期の不安定を防ぐ「prior 初期化」も提案しています。背景が圧倒的に多いため、最終分類層のバイアスを「物体の事前確率が小さい(例: π=0.01)」と仮定した値で初期化し、学習開始直後に膨大な背景例が生む巨大な損失で発散するのを防ぎます。Focal Loss の効果はこの初期化とセットで初めて安定する、という点は実装上の重要な注意です。

## 手法の系譜と主要論文
原典は Lin, Goyal, Girshick, He, Dollár の "Focal Loss for Dense Object Detection"(ICCV 2017、RetinaNet の論文、arXiv:1708.02002)です。動機は当時の検出器の二極構造にありました。2段階検出器(まず候補を数千に絞り、次に分類する Faster R-CNN 系)は高精度だが遅い。1段階検出器(候補を絞らず密に分類する SSD/YOLO 系)は速いが精度で劣る。著者らは1段階が劣る根本原因を「前景・背景の極端なクラス不均衡」と突き止め、Focal Loss でそれを解けば1段階でも高精度になると主張しました。提案手法を載せた RetinaNet は、シンプルな1段階構造のまま当時の2段階検出器を精度で上回りつつ高速、という結果を出しました。

その後 Focal Loss は検出を超えて広く転用されます。医療画像など前景ピクセルが極端に少ないセグメンテーションでは、領域の重なりを測る Dice loss と組み合わせた Focal-Dice / Focal-Tversky などが使われ、いずれも「易しい大多数のピクセルを抑えて難しい境界に集中させる」という同じ動機です。

校正の文脈で重要なのが Mukhoti, Kulharia, Sanyal, Golodetz, Torr, Dokania の "Calibrating Deep Neural Networks using Focal Loss"(NeurIPS 2020、arXiv:2002.09437)です。彼らは Focal Loss が不均衡対策だけでなく、出力確率の校正(確信度の正直さ)を改善する効果を持つことを示しました。Focal Loss が暗黙にエントロピー最大化(過信を抑える)として働く、より正確には予測分布と一様分布の KL 距離の上界を最小化する正則化として解釈できるという理論的説明を与え、サンプルごとに γ を変える適応的 Focal Loss(FLSD: Focal Loss with sample-dependent γ)も提案しました。これはラベルスムージングと並ぶ校正手法として Focal Loss を位置づけ直した重要な仕事です。

検出ではさらに Li らの Generalized Focal Loss(NeurIPS 2020、arXiv:2006.04388)が、離散的なクラス分類を連続的な品質推定(検出ボックスの良さ、IoU との結合)へ拡張し、分類と局在化の品質を統一的に扱えるようにしました。Quality Focal Loss と Distribution Focal Loss の2部品からなり、現代の検出器(YOLO 系の後期版など)に広く取り込まれています。損失設計の一般化としては Leng らの PolyLoss(ICLR 2022、arXiv:2204.12511)が、クロスエントロピーと Focal Loss を多項式展開の一系列として統一的に捉え、最初の項の係数を1つ足すだけで両者を超える設計(Poly-1)を示しました。

## 論文の実験結果(定量データ)
RetinaNet の原論文は COCO という大規模物体検出ベンチマーク(80カテゴリ、十数万枚)で評価しています。評価指標は AP(Average Precision、平均適合率。検出の正確さと網羅性を IoU しきい値0.5〜0.95で平均した総合スコアで、高いほど良い。IoU は予測ボックスと正解ボックスの重なり率)です。

最も示唆的なのは γ のアブレーションです。γ=0(=通常のクロスエントロピー)では、密なアンカーを使った1段階検出は学習が易しい背景に支配されてうまく収束しませんが、γ を 0 → 0.5 → 1 → 2 → 5 と上げるにつれ AP が改善し、γ=2 付近で最良になりました。原論文では γ=2、α=0.25 の組が最良の設定として報告されています。α と γ を組み合わせた効果も示され、片方だけより両方を使うほうが良い結果でした。比較対象としてハードネガティブマイニング(OHEM)も実験され、Focal Loss が OHEM を数 AP 上回ったことが報告されています。

総合性能では、RetinaNet(ResNet-101-FPN バックボーン)が COCO test-dev で当時の主要な1段階検出器(SSD、YOLOv2 など)と2段階検出器(Faster R-CNN 系)の双方を上回る AP(おおよそ39 AP 前後、当時として最高水準)を、推論速度を保ったまま達成しました。「1段階は精度で2段階に勝てない」という当時の常識を覆した点が、この数値の歴史的意義です。

校正の側では Mukhoti et al. (2020) が、CIFAR-10/100 や SVHN、20 Newsgroups 上で、クロスエントロピー学習のモデルが ECE(期待校正誤差、小さいほど確信度が正直)で数%の自信過剰を示すのに対し、Focal Loss(特に適応的 γ の FLSD)がラベルスムージングや温度スケーリングに匹敵またはそれ以上に ECE を下げ、しかも精度を犠牲にしないことを示しました。重要なアブレーションは「γ を大きくするほど校正は良くなるが、大きすぎると今度は確信が足りなくなる(過小自信になる)」という非単調性で、適応的に γ を選ぶ意義を裏づけています。彼らは「予測が易しくなる(p_t が大きくなる)につれ γ を下げる」ヒューリスティクスを提案しました。

## メリット・トレードオフ・限界
メリット。極端なクラス不均衡(背景が圧倒的に多い密な検出など)で、易しい多数派に総損失が支配されるのを防げる。「クラス」ではなく「例の易しさ」で重み付けするので、クラス重みでは取りこぼす「同一クラス内の易/難の偏り」に効く。γ=0 で通常のクロスエントロピーに戻る連続的な一般化なので、既存損失からの移行が容易。さらに(Mukhoti et al.)校正改善というおまけの効果もあります。

トレードオフと限界。第一に γ・α の2つのハイパーパラメータ調整が要ります(γ=2, α=0.25 が出発点)。第二に、ラベルノイズがある環境では深刻な落とし穴があります。Focal Loss は p_t が小さい例を「学ぶべき難例」として強調しますが、誤ラベルもまた p_t が小さくなる(モデルが正しく予測しても誤った正解と食い違う)ため、ノイズを難例と取り違えて過剰に学習する危険があります。ノイジーな環境では γ を控えめにする、[ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing) と併用してノイズの罰を和らげる、といった配慮が要ります。第三に、モデルの確率 p_t が信頼できない学習初期は効き方が読みにくく、前述の prior 初期化など別の工夫と併用することが多いです。第四に、多クラス softmax 向けの標準実装は検出の二値版ほど整備されておらず、用途に応じて実装を調整する必要があります。第五に、不均衡が穏やか(背景が支配的でない)なタスクでは、Focal Loss の恩恵は小さく、ときに通常のクロスエントロピーより劣ることもあります。万能薬ではありません。

研究上の未解決課題は、γ をデータやサンプルから自動・適応的に決める方法、ノイズと真の難例を区別する仕組み、そして不均衡補正と校正改善という2つの効果を理論的に統一して理解することです。

## 発展トピック・研究の最前線
適応的・サンプル依存の Focal Loss が一つの方向です。Mukhoti et al. の FLSD のように、各サンプルや学習段階に応じて γ を変えることで、不均衡補正と校正のバランスを動的に取ります。

損失関数の統一的理解も進んでいます。PolyLoss(Leng et al. ICLR 2022)はクロスエントロピー・Focal Loss を多項式 `Σ ε_j (1-p_t)^j` の系列として表現し、最初の数項の係数を調整するだけで両者を包含・凌駕できることを示しました。これは「Focal Loss は無数にある損失設計の一点にすぎない」という見方を与え、ImageNet・COCO・3D検出で軽微な係数調整による一貫した改善を報告しています。

検出の最前線では、分類と局在化の品質を分離せず統一的に扱う Generalized/Varifocal Focal Loss、ラベルの曖昧さ(ボックス境界の不確かさ)を連続分布として扱う Distribution Focal Loss などが現代の検出器に組み込まれています。Varifocal Loss は正例と負例で非対称に重み付けし、IoU を予測ターゲットに使う点で Focal Loss を検出向けに磨いたものです。

応用面では、ノイジーな擬似ラベル(視覚言語モデルやルールベースのラベラが生成したラベル)で長尾分類器を訓練する設定で、Focal Loss・[クラス重み付き損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0349_class-weighted-loss)・[ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing) を組み合わせる構成が広く使われます。それぞれが難易度・頻度・ラベルノイズという別々の軸を補正するため相補的ですが、Focal Loss とラベルノイズの相性の悪さ(難例とノイズの混同)があるため、この三者の重みの取り方自体が実務上のチューニング対象になります。研究コミュニティでは、難例とノイズを確信度の時間変化(学習を進めても下がり続けるか)で区別しようとする試みなどが続いています。
