# TCN (Temporal Convolutional Network)

## ひとことで言うと

時系列データ(時間の順に並んだデータ、例えば1秒ごとの車の速度の列)を扱うとき、長らく定番だったのは RNN や LSTM という「1ステップずつ順番に処理する」モデルでした。TCN はそれを「畳み込み(フィルタをスライドさせて特徴を取り出す計算)」だけで置き換えたモデルです。順番に処理しなくてよいので学習が速く、工夫しだいで長い過去まで一度に見渡せます。2018年に Bai らが「系列なら RNN」という当時の常識を実証的に揺さぶり、畳み込みでも(むしろ多くの場面でそれ以上に)系列が扱えると示したことで広く知られるようになりました。

## 直感的な理解

自動運転で「過去6.4秒(10Hzで64フレーム)の状態から次の進路を決めたい」とします。RNN だと、時刻1の状態を処理して隠れ状態を作り、それを時刻2に渡し、また時刻3へ…と64回の計算を必ず順番にこなす必要があります。GPU は本来「たくさんの計算を同時に」やるのが得意なのに、この順次処理ではその強みが活きません。系列が長いほど、また学習データが多いほど、この逐次性がボトルネックになります。

畳み込みなら話が変わります。畳み込みは「あるフィルタを系列の上でスライドさせ、各位置で同じ計算をする」操作で、全位置を同時に計算できます。64フレームを一気に処理できるので、データが多いほど RNN との速度差が効いてきます。さらに、RNN が長い過去を覚えるのに苦労する(後述の勾配の問題)のに対し、畳み込みは積み方を工夫すれば長距離も安定して見渡せます。

ただし時系列には一つ守るべきルールがあります。「未来をカンニングしてはいけない」。時刻 t の予測に t+1 以降の値を使ってしまうと、推論時にはまだ存在しない未来を使うことになり、ずるをした学習になります。これを防ぐのが「因果畳み込み」です。TCN はこの因果性を守りながら、間隔を広げた畳み込みで遠くまで見て、残差接続で深くしても安定させる、という三つの工夫の組み合わせです。「畳み込みは局所しか見られない」という弱点を、間隔を倍々に広げる工夫で克服したのが本質です。

## 基礎: 前提となる概念

順を追って前提を説明します。

RNN(Recurrent Neural Network、再帰型ニューラルネット)は、前の時刻の計算結果(隠れ状態)を次の時刻に渡しながらデータを1ステップずつ処理するモデルです。LSTM(Long Short-Term Memory)や GRU(Gated Recurrent Unit)はその改良版で、ゲート(門)という仕組みで重要な情報を選んで覚えます。共通する弱点が二つあります。一つは並列化できないこと(時刻 t の計算に t-1 の結果が要る)、もう一つは長距離の依存を覚えにくいことです。

勾配消失・勾配爆発(vanishing / exploding gradient)とは、ニューラルネットを学習する際に誤差を後ろ向きに伝える計算(誤差逆伝播)で、勾配(学習に使う微分の値)が層やステップを経るごとにどんどん小さくなったり大きくなったりして、学習が不安定になる現象です。RNN は系列が長いほどこの経路が長くなり、深刻になります。LSTM のゲートはこれを緩和しますが、完全には解決しません。

畳み込み(convolution)は、小さなフィルタ(重みの集まり)を入力の上でスライドさせ、各位置でフィルタと入力の積和を取って特徴を出す操作です。画像では二次元、時系列では一次元(時間方向)に行います。カーネルサイズ(kernel size)はフィルタが一度に見る点の数です。例えば k=3 なら3点を見ます。

受容野(receptive field)とは、ある出力が「入力のどれだけの範囲を参照できるか」を表す広さです。受容野が64なら、その出力は過去64ステップ分の情報を使って計算されています。長い依存を捉えるには受容野を広げる必要があります。受容野が足りないと、それより遠い過去はそもそも見えません。

残差接続(residual connection、skip connection)は、ある層の入力をその層の出力に足し込む構造です。これがあると「最悪でも何もしない(恒等写像)」を学べるので、層を深く積んでも性能が劣化しにくくなります。He らの ResNet(CVPR 2016)で確立された手法で、深いネットの学習を可能にした立役者です。

## 仕組みを詳しく

TCN は三つの部品の組み合わせです。

### 1. 因果畳み込み(causal convolution)

通常の一次元畳み込みは、ある時刻の出力を計算するとき前後を両方見ます。時系列予測ではこれが問題で、時刻 t の出力に t+1 以降が混ざると未来のカンニングになります。因果畳み込みは、出力時刻 t に対して入力の t 以前だけを参照するようにずらした畳み込みです。実装上は、入力の左側(過去側)にだけゼロを詰める(パディングする)ことで、出力長を保ちつつ未来を見ないようにします。詳しくは [Dilated Causal Convolution](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0484_dilated-causal-convolution) を参照してください。

### 2. dilated convolution(拡張畳み込み)で受容野を指数的に広げる

因果畳み込みを普通に積むと、L 層で見られる過去はせいぜい `1 + L·(k-1)`(k はカーネルサイズ)程度で、層数に比例してしか伸びません。64ステップ見たいなら多数の層が要り、現実的ではありません。深くしすぎると今度は勾配の問題が出ます。

そこで dilation(拡張率)を使います。フィルタが参照する点の間隔を 1, 2, 4, 8, ... と層ごとに倍々で広げます。すると層数 L に対して受容野が2のべき乗で増えます。具体的には、カーネルサイズを k、各層の dilation を `d=2^i`(i は層番号)としたとき、L 層積めば受容野はおよそ `1 + (k-1)·(2^L − 1)` まで広がります。

数値例で見ます。k=3、dilation を 1, 2, 4, 8, 16 と5層積むと、受容野は `1 + 2·(1+2+4+8+16) = 1 + 2·31 = 63` ステップ。たった5層で64フレーム弱の過去を見渡せます。RNN なら64ステップ分の逐次計算が要るところを、5層の並列計算で済むわけです。受容野が層数に対して指数的に伸びるので、長くしたければ層を少し足すだけで足ります(例: 8層で約510ステップ)。

```
層0 (d=1):  ●─●─●          隣接3点
層1 (d=2):  ●─・─●─・─●      2つおき
層2 (d=4):  ●───・───●…      4つおき
…倍々で間引いて見ることで、少ない層で遠くまで届く
```

### 3. residual block(残差ブロック)で深くしても安定させる

層を深くすると勾配が伝わりにくくなります。TCN は各ブロックを「dilated causal conv → 重み正規化(weight normalization)→ 活性化(ReLU)→ ドロップアウト」を2回重ね、その上で入力をそのまま出力に足し込む残差接続にしています。入力と出力のチャネル数が違うときは 1×1 畳み込みで合わせます。残差があるので深くしても劣化しにくくなります。重み正規化とドロップアウトは学習の安定と過学習抑制のために入っています。

### 形状で追う

バッチサイズ B、入力チャネル C_in、系列長 T のテンソル `[B, C_in, T]` を入力します。各 dilated causal conv 層は左パディングで長さを保ち、`[B, C, T]` を `[B, C, T]` に通します。最終的に `[B, C_out, T]` が出て、各時刻 t の出力ベクトルが「その時刻までの過去全体」を要約した特徴になります。系列全体を一つにまとめたいなら最後の時刻 T−1 のベクトルを取り出します。設計の理由は、こうすれば任意長の系列を同じ重みで処理でき、各時刻の出力が因果性を保ったまま過去を要約する点にあります。系列全体を一度に並列処理できるので、長さ T が大きいほど RNN に対する学習速度の優位が際立ちます。

## 手法の系譜と主要論文

時間方向の畳み込みで系列を扱う発想は、いくつかの源流が合流して TCN という形に結晶しました。

直接の源流は van den Oord らの WaveNet(van den Oord et al., 2016, arXiv:1609.03499、→ [WaveNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0517_wavenet))です。音声の生波形を1サンプルずつ自己回帰生成するために、dilated causal convolution を導入し、dilation を倍々にして受容野を指数的に広げるアイデアを確立しました。数千サンプルという超長系列を現実的な層数で扱えることを示した点が画期的でした。

同じ頃、Lea らの "Temporal Convolutional Networks for Action Segmentation and Detection"(Lea et al., CVPR 2017)も TCN という呼称を使い、動画の行動区分(どの区間がどの動作か)に時間畳み込みを適用しました。Bai らの汎用版とはやや別系統ですが、「時間方向の畳み込みで系列を扱う」流れを後押ししました。Gehring らの畳み込み機械翻訳(ConvS2S, Gehring et al., ICML 2017)も、畳み込みで系列を並列処理する流れの一つです。

これらを「分類も回帰も含む汎用の系列モデリング道具」として整理し直し、名前を広めたのが Bai, Kolter & Koltun(2018)です(著者の Zico Kolter, Vladlen Koltun は機械学習・コンピュータビジョンの著名研究者)。当時「系列なら RNN」という固定観念が強かったため、それを実証的に揺さぶる狙いで、多様なベンチマークで TCN と LSTM/GRU を同程度のパラメータ数で正面から比較しました。論文の主張は控えめで、「TCN を新提案する」というより「畳み込み系を真剣な比較対象として検討すべきだ」という問題提起の色が濃いものでした。

その後、Transformer(Vaswani et al., NeurIPS 2017)が自己注意で系列を扱う第三の道を開き、長系列の時系列予測では Informer/Autoformer/PatchTST 系へと主役が移りました(→ [Autoformer](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0321_autoformer) / [PatchTST](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0336_patchtst))。とはいえ TCN は軽量で安定し、推論が速いことから、組込み機器やストリーミング処理、計算資源が限られる現場では今も実用的な選択肢として使われ続けています。

## 論文の実験結果(定量データ)

Bai らは、性質の異なる多様な系列タスクで TCN と標準的な RNN(LSTM, GRU)を同程度のパラメータ数で比較しました。「公平な比較」を強く意識し、ハイパーパラメータ探索の予算も揃えています。主なタスクは次の通りです。

加算問題(adding problem)と複製記憶(copy memory)は、長距離の依存を覚える能力を測る合成タスクです。加算問題は系列中の指定された2点の数を足す課題、複製記憶は「系列の最初に出た情報を、長い空白の後で正確に再現する」課題で、いずれも長距離記憶のテストとして有名です。TCN は長い系列長(数百〜千ステップ)でも安定して低い誤差を保ったのに対し、LSTM/GRU は系列が長くなると性能が崩れる傾向が報告されました。これは dilation による広い受容野が、長距離依存を直接捉えられることを示します。

逐次 MNIST / Permuted MNIST(28×28 画像のピクセルを1点ずつ、784長の系列として読み込み分類する課題。Permuted は順序をランダムに入れ替えてより長距離の依存を要求する)では、TCN が LSTM/GRU を上回る分類精度を出しました。例えば逐次 MNIST で TCN は99%前後の精度を達成し、同条件の RNN を上回ったと報告されています。Permuted MNIST では差がさらに開く傾向でした。

ポリフォニック音楽のモデリング(Nottingham, JSB Chorales など、複数の音が同時に鳴る楽譜の予測)や、文字レベル・単語レベルの言語モデリング(PennTreebank, WikiText-103, text8 など)でも、多くの設定で TCN が同規模の RNN と同等以上のスコア(言語モデルでは perplexity、低いほど良い)を示しました。perplexity とは「モデルが次の記号をどれだけ迷わず予測できるか」を表す指標で、値が低いほど予測が確信を持っていて良いことを意味します。直感的には「平均して何択で迷っているか」に相当し、perplexity が 100 なら100通りで迷っている、20 なら20通り、という具合です。

論文全体の結論は、「汎用の系列タスクにおいて、畳み込み系の TCN は再帰系(RNN)と少なくとも同等、多くの場合それ以上の性能を、より速い学習と安定した勾配で達成する」というものでした。アブレーション的な観察として、dilation を取り除くと受容野が足りず長距離タスクで性能が落ちること、残差接続を外すと深いネットで学習が不安定になることが示され、各部品の必要性が裏付けられています。また TCN は学習中のメモリ使用が RNN より大きい(全時刻の中間表現を保持するため)一方、勾配の経路が安定しているという定性的観察も述べられています。なお、受容野は設計時の層数と dilation で固定されるため、想定より長い依存は捉えられないという限界も同論文で明記されています。

## メリット・トレードオフ・限界

メリットは四つです。第一に、全時刻を並列に計算できるので RNN より学習が速い(特に GPU で顕著)。第二に、dilation により少ない層数で長い過去を見渡せる(受容野が層数に対して指数的に伸びる)。第三に、勾配の経路が各時刻でほぼ一定の深さなので、RNN より勾配消失・爆発に強く学習が安定する。第四に、残差接続のおかげで深く積んでも劣化しにくい。

トレードオフと限界もはっきりしています。受容野が固定で、設計時に決めた長さより遠い依存は捉えられません(RNN は理論上は無限の過去を持てる)。推論時に長い履歴をメモリに保持する必要があり、逐次1ステップだけ持てばよい RNN に比べると、ストリーミング推論には工夫(過去の中間表現のキャッシュなど)が要ります。学習時のメモリ使用も RNN より大きくなりがちです。dilation・層数・カーネルサイズの設計次第で性能が大きく変わるため、ハイパーパラメータ調整が必要です。

研究上の論点として、自己注意(Transformer)が登場して以降、「固定受容野の畳み込み」と「全時刻を見られる注意」のどちらが系列モデリングに本質的に優れるかは、タスクと系列長と計算資源に依存し、一般的な結論はありません。注意は任意の2点を直接結べる柔軟さがある一方 O(L²) と重く、畳み込みは O(L) と軽い一方で受容野が固定です。短中距離で計算効率を重視するなら TCN、超長距離や柔軟な依存が必要なら注意系、という使い分けが実務的な落としどころになっています。

## 発展トピック・研究の最前線

TCN の発想はいくつかの方向に発展しています。一つは因果畳み込みと自己注意の融合で、畳み込みで局所を効率よく要約しつつ注意で長距離を補う、ハイブリッドな系列モデルが多数提案されています(畳み込みで局所特徴を抽出してから注意に渡す構成は、音声認識の Conformer などにも見られます)。

もう一つは、近年再注目されている状態空間モデル(state space model)系です。S4(Gu et al., ICLR 2022, arXiv:2111.00396)やそれを発展させた Mamba(Gu & Dao, 2023, arXiv:2312.00752)は、長系列を線形時間で扱い、TCN が苦手とする「固定受容野」の制約を、連続時間の状態方程式という別の道具で克服しようとしています。興味深いことに、これらは畳み込みとしても再帰としても解釈できる二面性を持ち、「畳み込みでも純粋な注意でもない第三の長系列モデル」として、TCN が切り開いた「RNN 以外で系列を扱う」流れの最前線にあります。

時系列予測の文脈では、TimesNet(Wu et al., ICLR 2023, arXiv:2210.02186)のように系列を主要周期で二次元化して二次元畳み込みをかける手法など、畳み込みを長期予測に活かす研究も続いています。SCINet など、ダウンサンプリングと畳み込みを組み合わせた予測手法も提案されています。

実応用の観点では、TCN の「並列学習・安定・軽量・低遅延推論」という性質は、自動運転の状態履歴処理、センサ信号のリアルタイム分類、音声・生体信号処理など、計算資源や遅延に制約のある現場で今も価値を持ちます。映像や履歴を時間方向に圧縮・要約する処理は、TCN が確立した「畳み込みで時間軸を扱う」系譜の上にあると言えます。
