# RSSM (Recurrent State-Space Model)

## ひとことで言うと
RSSM は、世界の状態を「次が確実に決まる部分(決定論的)」と「確率的にしか分からない部分(確率的)」の2つに分けて表す、再帰的な潜在状態のモデルです。これがあると、行動する主体(エージェント。たとえばロボットや運転車)は、現実で実際に試す前に頭の中で「この操作をしたら次はどうなるか」を想像できます。画像のような高次元の観測を、毎ステップ小さな数値ベクトルへ圧縮し、その小さな空間の中で何十ステップも先まで予測を転がせるのが価値です。世界モデル(環境のふるまいを学習した内部シミュレータ)の系譜、特に Dreamer 系のプランニングの心臓部にあたる仕組みです。

## 直感的な理解
あなたが暗い部屋を歩いていると想像してください。前に進めば壁にぶつかる、という「ほぼ確実に決まる結果」もあれば、ドアの向こうに何があるかという「開けてみないと分からない結果」もあります。賢く動くには、この2種類を区別して頭の中に地図を持つ必要があります。確実な部分はしっかり記憶として運び、不確実な部分は「いくつかの可能性」として保持しておく。RSSM がやっているのはまさにこれです。

なぜ2種類に分ける必要があるのか。純粋に確率的なモデル(状態が毎ステップ独立にサイコロを振るような設計)は、情報を長く覚えておくのが苦手です。何ステップも先まで記憶を運ぼうとすると、毎回の確率的なゆらぎで過去の情報が薄れてしまう。具体的には、サンプリングを繰り返すたびにノイズが注入されるので、10 ステップ前に観測した「壁の位置」のような確定情報が、途中のランダム性に埋もれて消えてしまうのです。逆に純粋に決定論的なモデル(RNN だけ)は記憶は得意ですが、「未来は1通りに決まる」と決めつけてしまい、本当は不確実な未来(他の車が右に行くか左に行くか分からない、など)を表現できません。決定論モデルが分岐する未来を学習させられると、複数の正解を平均した「ぼやけた未来」(2つの車の中間にゴーストが現れるような非物理的な予測)を出してしまいます。RSSM の発想は「いいとこ取り」です。決定論的な経路で記憶を確実に運びつつ、確率的な経路で未来の多様性・不確実性を表す。Hafner らはこの2つを組み合わせることで、長期の予測と不確実性表現の両立ができることを示しました。

## 基礎: 前提となる概念
RSSM を理解するために、いくつかの言葉を噛み砕いておきます。

- 世界モデル (world model): これは「環境の動きを学習した内部シミュレータ」という意味です。「今の状態で行動 a をとると、次の状態と報酬(良い行動への点数)はこうなる」を予測できるモデルです。現実で何度も試さず、頭の中で先読みできるのが価値です。
- 潜在状態 (latent state): これは「画像のピクセルそのものではなく、シーンの本質を圧縮した小さなベクトル(数字の並び)」という意味です。たとえば 12288 個の数字(64×64 のカラー画像)を 30 個程度の数字に圧縮し、その小さな空間で未来を予測します。これにより計算が桁違いに軽くなります。
- 再帰 (recurrent): これは「1つ前の状態を引き継いで次の状態を作る仕組み」という意味です。RNN(リカレントニューラルネット)や GRU(ゲート付き再帰ユニット。LSTM を簡略化した記憶ユニットで、何を覚え何を忘れるかをゲートで制御する)が代表で、時間の流れを内部記憶でつなぎます。
- 決定論的 (deterministic) と確率的 (stochastic): 決定論的は「入力が同じなら出力も必ず同じ」。確率的は「同じ入力でも複数の結果がありうる(分布で表す)」という意味です。
- 分布 (distribution): これは「ある値がどのくらいの確からしさで起きるか」を表す確率の形のことです。たとえばガウス分布(正規分布)は釣鐘状の山で「平均のあたりが起きやすく、離れるほど起きにくい」を表します。
- KL ダイバージェンス (Kullback-Leibler divergence): これは「2つの確率分布がどれだけ違うか」を測る量です。0 なら完全に同じ、大きいほど違う、という非対称な距離のようなものです。記号で `KL(q‖p)` と書き、「真の分布 q を、近似 p で置き換えたときに失う情報量」と読みます。
- 変分推論 (variational inference): これは「直接計算できない複雑な確率(ここでは潜在状態の事後分布)を、扱いやすい分布で近似し、その近似を良くしていく」枠組みのことです。VAE(変分オートエンコーダ)の理論的土台でもあり、RSSM の学習はこの枠組みの上に立っています。

## 仕組みを詳しく
RSSM の状態は2つの変数からなります。

- 決定論的状態 `h_t`: GRU で更新される隠れ状態。記憶を確実に運ぶ「背骨」です。
- 確率的状態 `s_t`: `h_t` を条件に、分布(ガウス分布やカテゴリ分布)からサンプリングされる確率変数。未来のゆらぎを担います。

完全な状態は `(h_t, s_t)` のペアで、これを合わせて「モデル状態」と呼びます。1ステップの更新は次の流れです(`a_t` は行動、`o_t` は観測=画像など)。

1. 決定論的更新: `h_t = f(h_{t-1}, s_{t-1}, a_{t-1})`。前の状態と行動から、記憶の背骨を GRU で更新します。ここに観測 `o_t` は入りません。だから記憶の背骨は「観測なしでも転がせる」のです。
2. 事前分布 (prior): `ŝ_t ~ p(s_t | h_t)`。観測を見る前に「次の確率的状態はこのへんだろう」と予測した分布です。これが「想像」に使う部分です。
3. 事後分布 (posterior): `s_t ~ q(s_t | h_t, o_t)`。実際の観測 `o_t` をエンコーダで特徴に変えて取り込み、補正した分布です。学習時はこちらの状態を使います。
4. 観測の再構成: `s_t, h_t` から元の観測 `o_t` を復元しようとする(デコーダ)。これで潜在状態が画像の中身を保つよう学習されます。同時に報酬 `r_t` も `(h_t, s_t)` から予測します(報酬予測ヘッド)。
5. 学習目標: 観測の再構成誤差を小さくしつつ、事前分布 prior が事後分布 posterior に近づくように学ぶ(KL ダイバージェンスを最小化)。prior が posterior に近づく=観測を見なくても次を当てられる、ということです。

ここが RSSM の肝です。学習後は「観測なしでも prior `p(s_t|h_t)` だけで次状態を生成」できます。つまり実際にセンサーを使わず、頭の中だけで `状態→行動→次状態→…` と何ステップも転がせます。これを imagination rollout(想像のロールアウト)と呼びます。

学習で最適化する目的関数は、変分下限(ELBO、evidence lower bound)と呼ばれる形になります。式の骨格は次の通りです。

```
L = Σ_t [  − log p(o_t | h_t, s_t)        ← 観測の再構成(small ほど良い)
           − log p(r_t | h_t, s_t)        ← 報酬の予測
           + β · KL( q(s_t|h_t,o_t) ‖ p(s_t|h_t) ) ]  ← prior を posterior に近づける
```

言葉で言うと「(1) 各時刻で観測と報酬をうまく予測できること」と「(2) 観測を見ないで予測した prior が、観測を見て補正した posterior に近いこと(KL を小さく)」の和です。(1) が潜在状態に世界の中身を詰め込ませ、(2) が「観測なしで予測する力」を鍛えます。係数 `β` は再構成と予測のバランスを取るつまみで、これが大きすぎると潜在が情報を捨てて崩壊(posterior collapse)し、小さすぎると prior が posterior に追いつけません。

数値イメージで実感してみましょう。画像 `[3,64,64]`(=12288 次元)をエンコーダで圧縮し、`h_t` を 200 次元、`s_t` を 30 次元とします。世界モデルはこの 230 次元の小さな空間で未来 15 ステップ程度を高速に想像し、各想像状態から報酬を予測して、どの行動列が良いかを評価します。ピクセル空間で未来を描くより桁違いに軽い計算です。DreamerV2 以降では `s_t` を「32 個のカテゴリ変数 × 各 32 値」のような離散表現(32×32=1024 次元の one-hot の束)にして安定性を上げています。

ASCII 図(想像ロールアウト):

```
 h_{t-1},s_{t-1}  --a-->  h_t  --prior--> s_t  --a--> h_{t+1} --prior--> s_{t+1} ...
      |                    |                          |
   (記憶の背骨)         報酬予測 r̂_t               報酬予測 r̂_{t+1}   観測は一切使わない
```

なぜこの二経路設計が効くのか。決定論経路 `h_t` は微分可能な連続写像なので、勾配(誤差をどう減らすかの向き)が時間をさかのぼってきれいに流れ、長期の情報を運べます。一方、確率経路 `s_t` は分布からのサンプリングなので、reparameterization trick(再パラメータ化。ガウスなら `s = μ + σ·ε` と書き、ノイズ `ε` を外から足してサンプリングを微分可能にする技法。離散なら straight-through 推定や Gumbel-Softmax を使う)で勾配を通します。この組み合わせで「長く覚える」と「分岐する未来を表す」を同時に実現しているのです。学習時は posterior 経由、想像時は prior 経由という二刀流が、観測ありの正確な学習と観測なしの自由な想像を両立させます。

## 手法の系譜と主要論文
- Doerr, Daniel, Schiegg, Nguyen-Tuong, Schaal, Toussaint, Trimpe, "Probabilistic Recurrent State-Space Models" (PR-SSM), ICML 2018, arXiv:1801.10395。「Recurrent State-Space Model」という用語と、確率的な再帰潜在ダイナミクスを変分推論で学ぶという発想の先行例です。低次元の力学系を対象にしており、ここではまだ高次元画像は扱っていません。

- Hafner, Lillicrap, Fischer, Villegas, Ha, Lee, Davidson, "Learning Latent Dynamics for Planning from Pixels" (PlaNet), ICML 2019, arXiv:1811.04551。深層 RSSM を提案し、画像から直接学習する形に発展させた事実上の原典です。決定論的 GRU 経路と確率的経路を併せ持つことで、ピクセルからの長期予測を可能にしました。学習した世界モデルの中で行動列を最適化する手法(CEM, cross-entropy method による online プランニング。候補の行動列をたくさん試し、報酬の高い上位だけを残して分布を更新することを繰り返す最適化)を採用。動機は「画像から直接、軽い潜在空間で計画したい」。さらに overshooting(複数ステップ先まで一度に予測誤差を取る正則化)で長期の予測精度を高めました。

- Hafner, Lillicrap, Ba, Norouzi, "Dream to Control" (Dreamer / V1), ICLR 2020, arXiv:1912.01603。RSSM を使い、その場プランニングではなく方策(policy)と価値(value)を想像の中で学習する枠組みへ発展。RSSM が Dreamer 系の共通土台であることを確立しました。詳細は [Dreamer 世界モデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0050_dreamer-world-model)。

- Hafner et al., "Mastering Atari with Discrete World Models" (DreamerV2), ICLR 2021, arXiv:2010.02193。確率的状態を連続ガウスから「カテゴリ分布(離散の組)」に変更。離散の確率的状態のほうが画像系で予測が安定すると示しました。加えて KL バランシング(prior を posterior に寄せる力と posterior を prior に寄せる力を別々の係数で制御する)を導入し、posterior collapse を防ぎました。

- Hafner, Pasukonis, Ba, Lillicrap, "Mastering Diverse Domains through World Models" (DreamerV3), 2023 / Nature 2025, arXiv:2301.04104。symlog 変換(値の大きさを対数的に圧縮しつつ符号を保つ)や two-hot エンコーディング、free bits(KL に下限を設けて潰れを防ぐ)等で、ハイパーパラメータをいじらず多様なドメインで動くようにしました。RSSM の基本骨格は維持しています。詳細は [Dreamer 世界モデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0050_dreamer-world-model)。

RSSM の設計思想は後続にも広く影響し、「決定論+確率の二経路」「観測なしで転がせる想像」は世界モデルの定番要素になりました。運転分野では BEV(俯瞰図)潜在で都市運転を想像する MILE(Hu et al., NeurIPS 2022)などが、RSSM 的な潜在ダイナミクスを応用した例として読めます([MILE(運転向け潜在世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1215_mile-latent-world-model))。

## 論文の実験結果(定量データ)
PlaNet(ICML 2019)は DeepMind Control Suite(物理シミュレータ上の連続制御タスク群。指標は1エピソードの累積報酬で、最大 1000 に近いほど良い)で評価されました。報告によれば、cheetah-run や walker-walk、cup-catch などのタスクで、当時の代表的なモデルフリー手法 A3C や D4PG に匹敵する性能を、はるかに少ない環境とのやり取り(おおむね 50 倍少ないエピソード数、約 100 万フレーム規模)で達成しました。「環境とのやり取りが少なくて済む」ことをサンプル効率と呼びます。これが重要なのは、実機ロボットや運転のように現実で試すのが高コスト・危険な領域では、試行回数こそがボトルネックだからです。

アブレーション(どの要素を抜くとどうなるかの検証)も RSSM の核心を示しています。PlaNet 論文では、確率的状態だけのモデル(SSM)と決定論的状態だけのモデル(RNN ベースの決定論モデル)を比べ、両方を併せ持つ RSSM が一貫して最良でした。確率経路だけだと長期の情報保持に失敗してロールアウトが崩れ、決定論経路だけだと不確実性を表せず予測が過信的になる、という形で、二経路それぞれの役割が定量的に裏づけられました。とりわけ確率経路なしのモデルは、未来が分岐するタスクで「平均的にぼやけた」予測になり、再構成画像がにじむことが視覚的にも示されています。

DreamerV2 では Atari 55 ゲームで、人間正規化スコア(human-normalized score。人間のスコアを 1.0、ランダムを 0 とした相対値)の中央値が約 1.64 に達し、単一 GPU・単一モデル・約 2 億フレームで初めて人間並み(中央値で人間超え)を達成しました。このとき確率的状態をガウスからカテゴリ分布に替えるアブレーションが効いており、カテゴリ表現を外すと多くのゲームでスコアが大きく落ちることが示されています。離散表現が画像系の予測安定性に寄与する、という定量的根拠です。なぜ離散が効くかについては、画像の世界では「物が在る/無い」「ドアが開いた/閉じた」のような本質的に離散的なモードが多く、カテゴリ分布のほうがこうした多峰的な未来を素直に表せるから、と論文は議論しています。

## メリット・トレードオフ・限界
メリット
- 決定論経路で長期記憶、確率経路で未来の不確実性、を同時に成立させられる。
- 学習後は観測なしで未来を想像でき、現実の試行回数を大幅に減らせる。
- 小さい潜在空間で予測するので計算が軽く、長いロールアウトが現実的(数百ステップでも秒オーダーで生成できる)。
- 確率的状態が「他の車が右か左か」のような複数の未来を分布で表せる。

トレードオフ・限界
- 再構成ベースだと、報酬や運転に無関係な背景の細部まで復元しようとして容量・計算を浪費する。小さく映る遠方の歩行者のように「画素は少ないが判断に決定的」な対象が、再構成損失では軽視されやすい。これが、再構成を避けて潜在空間で予測する JEPA 系([JEPA系 vs 生成系世界モデルの対比](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0066_jepa-vs-generative-world-models))が登場した動機の一つです。
- 何ステップも想像を重ねると予測誤差が蓄積する(compounding error、複利的に誤差が膨らむ現象)。誤った想像状態の上でさらに想像を重ねるため、後半ほど現実から乖離する。
- 確率的状態の分布の形(ガウスかカテゴリか)や KL の係数で学習の安定性が大きく変わり、調整が要る。posterior collapse(潜在が情報を運ばなくなる崩壊)は典型的な失敗例。
- 潜在状態が解釈しづらく、何を表しているか人が読み取りにくい。研究上は、潜在状態の意味づけ(disentanglement)や、安全性が要る領域での検証可能性が未解決の課題です。

## 発展トピック・研究の最前線
RSSM 以降の世界モデルは大きく2方向に進んでいます。1つは「再構成をやめる」方向で、TD-MPC2(Hansen et al., 2024)のように報酬予測に必要な情報だけを潜在に残す設計や、DINO-WM のように自己教師あり特徴量の空間で予測する設計が現れています。これらは「画素を復元する代わりに、判断に効く特徴だけを予測する」点で、再構成の浪費を避けます。もう1つは「生成の表現力を上げる」方向で、Transformer ベースの世界モデル(IRIS, TWM など。観測を離散トークンに変えて自己回帰予測する)や拡散モデルによる世界モデル(GAIA-1 系の運転動画生成、[GAIA-1(生成的世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1214_gaia-1-generative-world-model))が、長期・高解像度の想像を狙っています。RSSM の「決定論+確率」「潜在で転がす」という骨格は、これら新世代の比較基準(ベースライン)として今も生き続けています。研究上の生きた論点は、(a) 何を予測対象にするか(画素か潜在か報酬か)、(b) compounding error をどう抑えるか(複数ステップ予測の正則化、teacher forcing の割合制御)、(c) 部分観測下で潜在に「世界の十分統計量」をどこまで詰め込めるか、です。
