# 軌道損失の関数選択（Smooth L1 / MSE）

## ひとことで言うと

予測した軌道と教師軌道の「ズレの大きさ」を1つの数値（損失）にするとき、その数値化の式に何を使うかの選択です。代表候補が MSE（二乗誤差）と Smooth L1（フーバー損失）で、違いは主に「大きなズレ（外れ値）にどれだけ過敏に反応するか」です。回帰タスクでは外れ値に頑健な Smooth L1 が定番として広く使われます。

## 直感的な理解

テストの採点方法を考えてみます。

採点方式MSE（二乗）は、間違いの大きさを二乗して減点します。1点ずれたら1点減点、5点ずれたら25点減点。大きく外した1問が、全体の点数を一気に台無しにします。生徒（モデル）は「とにかく大ハズレだけは絶対に避けよう」と、その1問に全力を注ぐようになります。

採点方式Smooth L1 は、小さなずれは二乗で、大きくずれた分は「ずれた分だけ」リニア（直線的）に減点します。5点ずれても減点は5点弱。大ハズレ1問に引きずられず、全問をバランスよく見られます。

運転データには、人間の急ハンドルやセンサノイズ、ラベル付けミスで、ときどき極端な値（外れ値）が混じります。MSE方式だとモデルがその異常な1点に引きずられ、大多数の普通の運転が雑になります。Smooth L1 方式ならその引きずられが抑えられる、というのが直感です。

## 基礎: 前提となる概念

回帰（regression）とは連続値を予測する問題で、軌道予測（座標や加速度・曲率という連続値を当てる）はこれにあたります。回帰では「予測値 pred と正解値 target の差 e = pred − target」を1つの損失に集約しますが、その集約の式で学習挙動が変わります。

外れ値（outlier）とは、大多数のデータから極端に離れた値です。運転ログでは、緊急回避の急操舵、GPSやIMU（慣性計測装置）のスパイクノイズ、自動ラベル付けの失敗などで生じます。

勾配（gradient）とは、損失を小さくするためにモデルのパラメータをどちらにどれだけ動かせばよいかを示す量です。学習は勾配に沿ってパラメータを更新します（勾配降下法）。損失が外れ値に過敏だと勾配も外れ値に支配され、更新の方向がその1点に引っ張られます。勾配の大きさが極端に振れると、1ステップでパラメータが飛びすぎて学習が発散（損失が無限大やNaNへ向かう）することもあります。

頑健性（robustness）とは、少数の異常なデータに学習結果が大きく乱されない性質です。外れ値に頑健な損失とは、外れ値があっても勾配が暴れにくい損失を指します。統計学では、この頑健性を「影響関数（少数の汚染データが推定量に与える影響）が有界に保たれるか」で測ります。

## 仕組みを詳しく

### MSE（Mean Squared Error, 平均二乗誤差）

各要素の誤差を二乗します。損失は `e²`、勾配は `2e`（誤差に比例）。

- 誤差 0.1 → 損失 0.01。とても小さい。
- 誤差 5.0 → 損失 25.0。二乗で極端に大きい。勾配も 10 と大きく、強く引っ張る。

特徴は「大きな誤差を二乗で強調する」こと。外れ値があると損失も勾配も一気に膨らみ、その1点を直そうと学習が強く引っ張られます。誤差が小さい領域では勾配も小さくなる（2e → 0）ので、最適値近くで滑らかに収束する利点もあります。統計的には、MSE を最小化することは「誤差が等分散の正規分布（ガウス分布）に従う」という仮定のもとでの最尤推定に一致します。実走ログのように誤差が裾の重い分布（極端な値が出やすい分布）だと、この仮定が崩れて MSE は不利になります。

### Smooth L1（Huber 損失）

小さな誤差では MSE のように二乗、ある閾値 β（しきいち。多くの実装でデフォルト β=1.0）を超えると L1（絶対値）のように線形に切り替わる、ハイブリッドな式です。

```
|e| < β のとき:  0.5 · e² / β       （二乗。原点付近は滑らか）
|e| ≥ β のとき:  |e| − 0.5 · β       （線形。外れ値で勾配が一定に張りつく）
```

- 誤差 0.1 → ほぼ二乗（0.005）。小さい誤差にはMSEと同様に敏感で、微調整がきく。
- 誤差 5.0 → 線形（5.0 − 0.5 = 4.5）。MSEなら25だったのが4.5。勾配も β を超えると ±1 で一定になり、外れ値が勾配を支配しなくなる。

この切り替えにより、原点付近では二乗の滑らかさ（微分が連続で収束が安定）、外れ値には線形の頑健さ（勾配が暴れない）、という両方の良さを併せ持ちます。なお L1（絶対値誤差）単体だと原点で微分が不連続（左から −1、右から +1）になり、最適値付近で振動しやすいので、Smooth L1 はその不連続を二乗で丸めた版とも言えます。閾値 β は「どこから外れ値とみなすか」の境界で、β を小さくするほど多くの誤差を線形扱い（頑健寄り）に、大きくするほど MSE に近づきます。

### 数値例での比較

教師=0.5、予測=0.6（普通のズレ e=0.1）と、教師=0.5、予測=8.0（外れ値 e=7.5）が混ざったバッチを考えます（β=1.0）。

```
                普通のズレ e=0.1   外れ値 e=7.5     比率
MSE             0.01               56.25           約 5600 倍
Smooth L1       約 0.005           約 7.0          約 1400 倍
勾配(MSE)        0.2                15.0            外れ値が勾配を支配
勾配(SmoothL1)   0.1                1.0             外れ値の勾配が頭打ち
```

MSEでは外れ値が普通のズレの約5600倍の損失を持ち、勾配も75倍差でバッチ全体がこの1点に支配されます。Smooth L1 では損失比が約1400倍に、勾配比は10倍に縮み、普通のズレも学習に効きます。

## 手法の系譜と主要論文

Huber 損失（Huber, 1964, "Robust Estimation of a Location Parameter", Annals of Mathematical Statistics）。統計学者 Peter Huber が、最小二乗法（MSE相当）が外れ値に弱いという古典的問題を解くために提案した頑健回帰の損失です。外れ値の影響を線形に抑えつつ原点付近では二乗の滑らかさを保つ、という設計思想がここで確立しました。Smooth L1 はこの Huber 損失を β でスケールした一種です。Huber 損失は「正規分布の中心とラプラス分布（裾の重い分布）の裾を貼り合わせた」分布の最尤推定に対応すると解釈できます。

Fast R-CNN（Girshick, ICCV 2015, arXiv:1504.08083）。物体検出のバウンディングボックス位置回帰に Smooth L1 を広めた研究です。論文中で「L2(MSE)損失は外れ値に過敏で勾配が爆発しやすく学習が不安定になるため、外れ値に頑健な Smooth L1 を使う」と明記しています。以降、検出・回帰タスクの事実上の標準になりました。

UniAD（Hu et al., CVPR 2023 Best Paper, arXiv:2212.10156）。end-to-end運転の計画に L1 損失と衝突ペナルティ（collision penalty、他物体とぶつかる軌道に追加の罰を与える項）を組み合わせました。L1系を選ぶ理由は外れ値への頑健性、衝突ペナルティを加える理由は「人間と似ているが危険な軌道」を回帰だけでは防げないためです。回帰損失だけでは「人間の軌道に近いか」しか測れず、安全性は別の項で陽に与える、という設計思想がここに表れています。

VAD（Jiang et al., ICCV 2023, arXiv:2303.12077）。軌道や地図要素をベクトル化（vectorized、点列や折れ線として疎に表現）して学習します。密なラスター表現（画像のように格子で表す）より計算が軽く高速、という動機で、損失は L1 系を基調とします。

multimodal trajectory loss（Cui et al., ICRA 2019, arXiv:1809.10732 ほか）。1本の正解しかない回帰で複数候補を出すと「中間的な軌道」に潰れる問題（後述の mode averaging）に対し、複数モードを出して「最も近いモードだけ」を回帰する loss（min-of-N / winner-takes-all、最も良かった候補だけに勾配を流す）が提案されました。損失関数の選択（L1/MSE）とは別の軸ですが、軌道回帰の損失設計を語るうえで欠かせない系譜です。

## 論文の実験結果（定量データ）

Fast R-CNN（ICCV 2015）は PASCAL VOC 物体検出で、回帰損失を L2 から Smooth L1 に替えることで学習の安定性が向上し、学習率を大きく取れるようになったと報告しています。L2 では大きな誤差で勾配が爆発し発散しやすかったのに対し、Smooth L1 では勾配が一定に張りつくため安定する、という定性的説明が、以降の回帰タスクで Smooth L1 が標準になる根拠になりました。

UniAD（CVPR 2023）は nuScenes で、計画の1〜3秒平均 L2 誤差を約 1.03m、平均衝突率を約 0.31% に抑えました。アブレーションでは、知覚・予測・占有・計画を一体で学ぶ統合設計の各タスクヘッド（物体追跡 TrackFormer、地図 MapFormer、動き予測 MotionFormer、占有予測 OccFormer）を外すと計画性能が劣化することが示されています。とくに動き予測や占有予測のモジュールを外すと衝突率が悪化し、周囲エージェントの未来を考慮することが安全な計画に効くと分かります。また衝突ペナルティを加えると衝突率が下がる一方、L2 誤差はわずかに増えることがあり、「人間への近さ（L2）」と「安全性（衝突率）」がトレードオフしうることを示します。これは損失項の重み調整が性能に直接効く例です。

VAD（ICCV 2023）は nuScenes で UniAD と同等以下の L2 誤差（VAD-Base で平均約 0.72m）を保ちつつ、推論を大幅に高速化しました（VAD-Tiny で約 16.8 FPS、UniAD の約 1.8 FPS に対し約9倍）。損失関数そのものは L1 系を基調としており、Smooth L1 / L1 系が運転計画でも実用的な既定値であることを裏づけます。

外れ値の多寡による使い分けについては、データセットがクリーン（ラベルノイズが少ない、たとえば高精度な合成・シミュレーションデータ）なら MSE の方が大きな誤差を強く罰して精度を追い込めることがあり、ノイズが多い実走ログでは Smooth L1 が安定するという経験則が広く共有されています。「どちらが良いか」はデータの外れ値分布に依存し、厳密には A/B（同じ条件で損失だけ替えて比較）で検証すべき項目です。

## メリット・トレードオフ・限界

Smooth L1（回帰の既定値として広く使われる）
- メリット: 外れ値・ノイズに頑健で学習が安定。原点付近は滑らかで微分が素直、収束しやすい。大きな学習率を許容しやすい。
- トレードオフ・限界: 閾値 β というハイパーパラメータが増える（β の選び方で挙動が変わる）。本当に大きく直すべき誤差も線形でしか効かないため、クリーンなデータでの純粋な精度最大化では MSE に劣ることがある。

MSE
- メリット: 大きな誤差を強く罰するので、外れ値の少ないきれいなデータでは高精度に追い込める。式が単純でパラメータが要らない。誤差が正規分布のとき統計的に最適（最尤推定）。
- トレードオフ・限界: 外れ値1点に損失・勾配が支配され学習が不安定。実走ログのノイズに弱く、発散しやすい。

共通の限界（損失関数選択では直らない問題）
- mode averaging（モード平均化）。1本の正解しかない回帰で、状況に複数の妥当な行動がある（交差点を右にも左にも行ける）場合、L1 も MSE も「全候補の平均」という、どちらつかずで実際には危険な中間軌道に潰れがちです。L1 は中央値、MSE は平均値へ潰れる傾向の違いはありますが、どちらも単峰の出力では多峰の正解を表現できません。これは損失式の選択では解決できず、複数モード出力＋min-of-N 損失や確率分布出力（混合分布・拡散モデル）といった別の設計が必要です。

## 発展トピック・研究の最前線

研究の最前線では、回帰損失そのものより「出力をどう表現し、どう損失を設計するか」に焦点が移っています。複数候補を出す混合密度ネットワーク（mixture density network、出力を正規分布の重み付き和で表す）、winner-takes-all で最近傍モードだけ回帰する損失、軌道を離散トークン化して交差エントロピー（分類の損失）で学ぶ自己回帰生成、拡散モデルのスコアマッチング損失などです。また安全性を陽に入れる衝突・道路逸脱ペナルティや、人間らしさを測る学習済み評価器（learned metric）を損失に組み込む方向もあります。Smooth L1 vs MSE は基礎の一里塚であり、その上に「マルチモーダル性」「安全制約」「生成モデル」という層が積み重なっているのが現状です。
