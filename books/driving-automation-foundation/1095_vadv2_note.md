# VADv2 (確率的計画E2E)

## ひとことで言うと
VADv2 は、ベクトル化 End-to-End 自動運転手法 VAD の後継で、自動運転の「次の動き」を1本の決まった軌道として出すのではなく、「たくさんの動きの候補と、それぞれの確からしさ(確率)」として出す手法です。これにより、状況によって複数の妥当な走り方がある場面でも、より自然で安全な判断ができます。2024 年に発表され、運転シミュレータ CARLA の閉ループ評価で当時の最高性能を更新しました。

## 直感的な理解
前を遅い車が走っているとします。あなたなら「右から追い越す」「左から追い越す」「後ろで車間を保って待つ」のどれを選んでもよく、どれも正解です。運転には「答えが1つに決まらない」場面があふれています。これを多峰性(multi-modality、もっともらしい答えの山が複数あること)と呼びます。「峰(モード)」とは確率分布のなかで確からしさが盛り上がっている山のことです。

ここでもし機械に「1本の正解軌道を出せ」と教師あり学習(お手本との誤差を小さくする学習)させると、困ったことが起きます。右に避けた例と左に避けた例を大量に見せると、二乗誤差を最小にする答えは両者の平均、つまり「まっすぐ障害物に突っ込む軌道」になってしまうのです。これを mode averaging(モードの平均化)あるいは regression-to-the-mean と言い、End-to-End 計画の有名な失敗です。平均された軌道はどの正解にも当てはまらず、しばしば最悪の選択になります。

VADv2 の発想はシンプルです。「1本に決めるのをやめ、ありうる行動の見本帳を用意して、それぞれにどれくらい確からしいか点数(確率)を付けよう」。右に避ける見本にも左に避ける見本にも確率を割り当てられるので、平均で潰れることがなくなります。連続値を当てる回帰問題を、選択肢から選ぶ分類に近い問題へ作り変えたわけです。

## 基礎: 前提となる概念
回帰と分類: 回帰とは、出力が連続値(例: 軌道の x, y 座標)で、それを直接予測すること。二乗誤差で学習すると平均化が起きやすい。分類とは、あらかじめ決めた選択肢の中から「どれか」を確率付きで選ぶこと。選択肢ごとに独立に確率を出せるので、複数の山を同時に表現できます。VADv2 は連続的な軌道計画を、離散候補の上の分類へ置き換えます。

ベクトル化シーン表現: 前身の VAD 譲りで、世界を密な画像(ラスタ)ではなく、車線のポリライン(折れ線)や他車の位置・向き・将来候補軌道といった「線と点のベクトル」で表す方式です。軽くて速く、計画の条件として扱いやすい利点があります。

開ループと閉ループ: 開ループ(open-loop)評価は、過去の運転ログに対し「自分ならどう走ったか」を予測し、人間の実軌道との距離で測る方式で、車は実際には動きません。閉ループ(closed-loop)評価は、予測した行動でシミュレータ内を実際に走らせ、その結果また観測が変わる、という相互作用を伴う厳しい評価です。1つのミスがその後の状況を変えて連鎖する(分布シフトと呼びます)ため、閉ループの方が実運転に近く難しい。VADv2 の主戦場は閉ループです。

softmax: 複数の数値(スコア)を、すべて正で合計が1になる確率の集合に変換する関数。大きいスコアほど大きな確率になります。VADv2 が候補軌道に確率を割り当てる際に使います。

KL ダイバージェンス / クロスエントロピー: 2つの確率分布がどれだけ違うかを測る量。VADv2 は「ネットワークが出した分布」を「お手本から作った目標分布」へ近づけるとき、この種の分布間距離を損失(最小化したい量)に使います。

## 仕組みを詳しく
VADv2 の核心は probabilistic planning(確率的計画)です。

ステップ1: 行動の候補集合(planning vocabulary、計画語彙)を用意する。大規模な運転データから、実際に人間が取った軌道を大量に集め、クラスタリング(似たもの同士をまとめる)で代表的なものを抽出して「行動の見本帳」を作ります。規模は論文ではおよそ 4096 本。「直進」「ゆるく右」「急に左」「減速して停止」…といったテンプレートの集まりです。各候補は未来 T 秒の点列、例えば `[N_t, 2]`(N_t 個の時刻ステップ、各点が x, y)。これを N=4096 本そろえて `[N, N_t, 2]` とします。語彙は学習前に一度作って固定し、学習中も推論中も同じものを使います。

ステップ2: 各候補に確率を割り当てる。ネットワークは1本の軌道を回帰するのではなく、N 個の候補それぞれに「この行動を取る確からしさ」のスコアを出し、softmax で `[N]` 個の確率(合計1)に変換します。これは事実上、4096 クラスの分類です。右に避ける候補にも左に避ける候補にも確率を配れるので、多峰性が自然に表現できます。

ステップ3: ベクトル化シーン情報を条件にする。確率の割り当ては、VAD 譲りのベクトル化シーン表現(車線ポリライン、他車ベクトル、ナビ指令、自車状態など)を条件として行われます。学習する対象は条件付き確率 `P(action | scene)`、すなわち「今この状況なら、この候補がどれくらい確からしいか」です。各候補軌道は埋め込み(数値ベクトル化)され、計画用 Transformer のなかでシーン特徴と注意機構(cross-attention)で照合してスコアが計算されます。語彙が大きいので、この「候補×シーン」の照合を効率よく行うことが実装上の要点です。

ステップ4: 学習の目標(蒸留に近いソフトラベル)。教師信号は単純な「人間が取った1本に確率1、残り全部0」というハードラベルではありません。VADv2 は、各候補軌道に対し「人間軌道との近さ」や「衝突・境界逸脱などのルール適合」に基づくソフトな目標分布を作り、ネットワークの予測分布をそこへ近づけます(分布間の差を測る損失で学習)。人間軌道に近い候補ほど高い目標確率、危険な候補ほど低い目標確率、という連続的な教師を作るわけです。これにより、人間のお手本だけでなく安全規則の知識も分布の形に織り込めます。1本のお手本では捨てられていた「次に良い行動」の情報まで使える点が効いています。

ステップ5: 推論時の選び方。走行時は、確率が最も高い候補を選ぶ(argmax)か、上位候補から安全フィルタ(ルールで危険な候補を除外する仕組み)を通して選び、必要なら微調整します。確率分布が手元にあるので「次に確からしい代替行動」も分かり、危険時に二番手へ切り替えるといった運用がしやすいのが実用上の強みです。

数値例で mode averaging との違いを示します。

```
状況: 前方に停止車。右回避と左回避が半々に正解。

単一軌道回帰の出力:    (右と左の平均) → まっすぐ前進 → 衝突。NG
VADv2 の出力(分布):  P(右回避)=0.45, P(左回避)=0.45,
                       P(直進)=0.02, P(減速停止)=0.08, …
                       → 山が2つ残り、どちらかを選べる。OK
```

## 手法の系譜と主要論文
MultiPath(Chai ら、Waymo、CoRL 2019、arXiv:1910.05449)。あらかじめ用意したアンカー軌道(代表軌道)の集合に対し、各アンカーの確率と、そこからの微小オフセット(残差)を予測する軌道予測手法。「候補集合に確率を割り当て、多峰性を表現する」という VADv2 の根本アイデアの源流のひとつです。当時は他車の予測に使われましたが、VADv2 はこの考えを自車の計画(プランニング)へ持ち込みました。計画は予測と違い「自分が選んで実行する」ため、確率分布が安全フィルタや意思決定に直結する点が新しい意味を持ちます。

VAD(Jiang ら、ICCV 2023、arXiv:2303.12077)。ベクトル化シーン表現と幾何制約による高速 End-to-End。ただし計画は基本的に単一軌道の回帰で、多峰性を強くは扱いませんでした。VADv2 はこの計画部分を確率分布へ置き換えた直接の発展形であり、シーン表現の部分は VAD をほぼ受け継ぎます。

VADv2(Chen, Jiang, Zhang ら、Horizon Robotics・華中科技大学など、2024、arXiv:2402.13243)。計画を「行動語彙上の確率分布」として学習する確率的計画を提案。動機は単一軌道回帰の mode averaging を断ち、純カメラ End-to-End で閉ループに強くすること。CARLA Town05 閉ループでの強さを示しました。

Hydra-MDP(Li ら、NVIDIA、2024、arXiv:2406.06978)。VADv2 と同時代の流れで、複数のルールベース教師(衝突回避・進行・快適性・到達など)から「マルチターゲットの知識蒸留」で候補軌道のスコアを学ぶ手法。語彙ベースの確率的計画というアイデアを発展させ、シミュレーション指標を直接最適化する方向を示しました。NAVSIM ベンチマークで上位を取り、語彙スコアリング型計画を一気に主流化しました。

NAVSIM(Dauner ら、NeurIPS 2024、arXiv:2406.15349)。開ループ L2 の偏りへの反省から作られた疑似閉ループの計画ベンチマーク。短時間だけ非反応的にシミュレートし、衝突なし・走行可能領域順守・到達進度・快適性を合成した PDMS スコアで評価します。VADv2/Hydra-MDP 系の語彙スコアリング計画がここで標準的ベースラインとなりました。

## 論文の実験結果(定量データ)
VADv2 は nuScenes(実車ログのベンチマーク)と CARLA(運転シミュレータ)の両方で評価され、当時の最高性能を報告しました。

CARLA は、街・天候・交通を仮想的に再現し、実際に車を走らせて評価できるシミュレータです。VADv2 の主な舞台は CARLA の Town05 における閉ループ評価でした。ここで使う代表的な指標が2つあります。

Route Completion(ルート完走率、指定経路のうち何パーセント進めたか、高いほど良い)と、Driving Score(運転スコア、完走率に違反・事故のペナルティを掛け合わせた総合点、高いほど良い)。たとえば信号無視や衝突があるたびにペナルティ係数が掛かるため、Driving Score は「どれだけ進めたか」と「どれだけ安全だったか」を同時に評価します。閉ループでは1つの判断ミスがその後の状況を悪化させ連鎖するため、これらの指標は実運転の質をよく反映します。

報告では、VADv2 はカメラのみの入力で、CARLA Town05 の閉ループ設定(Town05 Long/Short)において当時のカメラ手法を上回る Driving Score を達成し、SOTA(state-of-the-art、その時点の最高性能)を更新したとされています(絶対値は評価バージョン依存)。とくに、それまで強かった一部の特権情報(地図・周辺の真値・ルールを多く与えた)手法に対して、純粋なベクトル化 End-to-End で競争力を示した点が重要でした。閉ループで強いということは、「お手本に似ている」だけでなく「実際に走らせても破綻しにくい」ことを意味します。VAD(単一軌道回帰)を同条件で閉ループに置くと Driving Score が明確に下がることも示され、確率化が閉ループ性能の鍵だと裏づけられました。

アブレーションの要点。確率的計画(分布出力)を単一軌道回帰に戻すと、多峰的な場面で性能が落ちることが示され、確率化の寄与が確認されました。行動語彙の規模 N を小さくしすぎる(例えば数百本)と、表現できる行動の粒度が粗くなり、最適行動を取りこぼして性能が低下します。逆に N を増やすと表現力は上がるものの、候補×シーンの照合コストとメモリが増えるトレードオフが報告されています。論文は N=4096 付近を実用的なバランスとして選びました。また、安全規則をソフト目標分布へ織り込む工夫(衝突・境界に基づく目標確率の調整)を外すと、衝突・違反が増える方向に悪化しました。

## メリット・トレードオフ・限界
メリット。多峰性(複数の妥当な行動)を自然に表現でき、mode averaging を避けられます。確率分布が出るので、安全ルールや不確実性に基づく判断(自信のないときは保守的に減速する等)と組み合わせやすい。回帰より分類に近いため学習が安定しやすく、閉ループ評価で破綻しにくい傾向があります。ソフトラベル蒸留により、ルールベースの知識をデータ駆動の枠組みへ無理なく注入できる点も実用的です。

トレードオフと限界。行動語彙(候補集合)の設計が性能を左右し、語彙にない動き(想定外の鋭い回避や微妙な連続操作)は出しにくい。候補が離散なので、ぴったり最適な連続軌道とはわずかにずれ、後段の微調整が要ることがあります。候補数 N を増やすほど計算・メモリが増え、リアルタイム性とのバランスを取る必要があります。語彙は学習データの分布に依存するため、収集データに偏り(直進が多い等)があると、まれな行動(急な回避、Uターン)の確率推定が苦しくなります。また閉ループで強いとはいえ、CARLA と現実の差(sim-to-real gap、シミュレータと実世界のずれ)は依然として未解決の課題で、CARLA で良いスコアが実車の安全性をそのまま保証するわけではありません。

## 発展トピック・研究の最前線
語彙ベースの確率的計画は、その後「複数のルールベース教師からスコアを蒸留する」方向(Hydra-MDP など)や、シミュレーション指標(PDMS)を直接最適化する方向へ発展しました。評価面では、開ループ L2 の偏りへの反省から、NAVSIM のような疑似閉ループ・大規模多様シーンの計画ベンチマークが整備され、語彙スコアリング型の計画が標準的なベースラインになっています。

さらに近年は、固定語彙の表現力の限界を超えるため、拡散モデル(diffusion、ノイズから徐々に軌道を生成する手法)で連続的かつ多峰な軌道分布を直接生成するアプローチ(DiffusionDrive など)や、大規模言語・視覚モデル(VLM)を計画に結合して常識推論や指示理解を持ち込む流れも活発です。確率的計画は「計画を確率分布として扱う」という基盤的な考え方として、これら新しい手法の出発点に位置づけられています。固定語彙と連続生成の中間(語彙を文脈に応じて動的に作る、粗い語彙から細かく精緻化する)を探る研究も進んでいます。
