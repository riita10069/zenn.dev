# U-Net (拡散モデル用)

## ひとことで言うと
U-Net(ユーネット)は、画像を一度ぎゅっと縮めて要点をつかみ、それをまた元の解像度まで広げ直す、アルファベットの「U」の字の形をしたニューラルネットワークです。縮める途中の情報を、広げる側へ「ショートカット(スキップ接続)」で直接渡すのが特徴で、細部を失わずに画像全体を処理できます。拡散モデルでは、この U-Net が「ノイズの乗った画像から、乗っているノイズを予測する」中心部品として長年標準的に使われてきました。Stable Diffusion をはじめ多くの実用モデルの骨格でもあります。

## 直感的な理解
拡散モデルは、ノイズだらけの画像を見て「いま乗っているノイズはこれだ」と推定し、少しずつ取り除いて画像を作ります。この「ノイズ推定係」に求められる仕事を考えると、二つの相反する要求が出てきます。

一つは「この画像全体として何が写っているのか」という大局的な理解です。猫を生成している途中なら、ノイズの下に猫の輪郭があるはずで、それを踏まえてノイズを推定しなければなりません。猫の左耳の位置は右耳と整合していなければならず、これには画像全体を見渡す必要があります。もう一つは「ピクセル単位の細かい正確さ」です。出力は入力と同じ大きさの「ノイズの画像」で、ヒゲ1本、毛並みの方向まで細かく当てる必要があります。

ここでジレンマが生じます。大局を理解するには、画像を縮めて抽象化し、広い範囲を一度に見られるようにする必要があります。ところが縮めると、細部の正確な位置情報が失われてしまいます。逆に細部を保とうと高解像度のまま処理すると、計算が爆発するうえ、1ピクセルが見られる範囲(受容野)が狭く大局がつかめません。U-Net はこのジレンマを「縮めて大局をつかむ道(エンコーダ)」と「広げて細部を復元する道(デコーダ)」の両方を持ち、縮める途中の細部をショートカットで復元側に直送することで解決します。これが U-Net が拡散の骨格として選ばれた本質的な理由です。

## 基礎: 前提となる概念
U-Net を理解するための言葉を噛み砕きます。

畳み込み(convolution)とは、小さなフィルタ(たとえば 3×3 の窓)を画像の上で滑らせ、各位置で局所的なパターン(エッジ、テクスチャなど)を抽出する演算です。近所のピクセルしか見ませんが、層を重ねると徐々に広い範囲を見られるようになります。畳み込みには「平行移動しても同じパターンを同じように検出する」という性質(並進等変性)があり、これは画像処理に都合のよい帰納バイアス(事前の構造的仮定)です。同じフィルタを画像全体で使い回すので、パラメータ数が少なく少データでも学習しやすい利点があります。

ダウンサンプリング(縮小)とは解像度を半分に落とす操作(プーリングやストライド2の畳み込み)で、アップサンプリング(拡大)はその逆で解像度を倍に戻す操作(転置畳み込みや最近傍補間 + 畳み込み)です。受容野(receptive field)とは、出力の1ピクセルが入力のどれだけ広い範囲の情報を見て決まっているか、という範囲のことです。縮めるほど1ピクセルが広い範囲を代表するようになり、受容野が広がります。

セグメンテーションとは、画像の各ピクセルが「細胞」「背景」などどのクラスに属するかを塗り分けるタスクで、入力と同じ解像度のピクセルごとの予測を出します。U-Net はもともとこのために生まれました。拡散のノイズ予測も「画像 → 同サイズの画像(=ノイズ)」という同じ構図なので、そのまま流用できたのです。この「入出力の解像度が一致する dense prediction」という点が、U-Net が拡散にぴったりはまった理由です。

## 仕組みを詳しく
U-Net の処理を、U字の形に沿って追います。

U字の左半分、エンコーダ(縮める道)。入力画像、たとえば [B, 3, 256, 256] を、畳み込みとダウンサンプリングの繰り返しで、だんだん小さく・チャンネル数を多くしていきます。

```
[B, 64, 256, 256] → [B, 128, 128, 128] → [B, 256, 64, 64] → [B, 512, 32, 32]
```

解像度が下がるほど受容野が広がり、「画像全体として何が写っているか」という大局的・意味的な情報が圧縮されます。チャンネル数を増やすのは、解像度を捨てた分の表現力を確保するためです(空間を 1/4 に減らすたびにチャンネルを2倍にしてバランスを取るのが定石)。

U字の底、ボトルネック。最も縮んだ中央部で、画像全体を見渡した抽象的な特徴が得られます。拡散用 U-Net では、この付近(低解像度の層、典型的には 16×16 や 8×8)に自己注意(self-attention、離れた位置どうしの関係を直接計算する仕組み)層を入れるのが一般的です。畳み込みだけでは受容野に限界があり、画像の端から端までの整合性を取りにくいので、解像度が下がってトークン数が減った(注意の計算量はトークン数の2乗なので低解像度なら安い)ところで注意を入れて長距離の関係を補うのです。

U字の右半分、デコーダ(広げる道)。ボトルネックの特徴を、アップサンプリングと畳み込みで少しずつ元の解像度まで戻し、最終的に [B, 3, 256, 256](予測したノイズ)を出します。

スキップ接続、U-Net の心臓部。縮める途中(エンコーダ)の各解像度の特徴を、広げる側(デコーダ)の同じ解像度の地点へショートカットで直接渡し、チャンネル方向に連結します。なぜ効くのか。縮める過程で「境界の正確な位置」「細かいテクスチャ」は失われがちです。デコーダがボトルネックの抽象情報だけから復元すると細部がぼやけます。スキップ接続でエンコーダ側の細かい情報を直接届けることで「意味はボトルネックから、細部はスキップから」と両取りでき、シャープな出力が得られます。before/after で言えば、スキップなしだと輪郭がにじむ画像、スキップありだと輪郭のはっきりした画像になります。勾配の通り道も増えるので、深いネットワークでも学習が安定します。

拡散特有の追加、時刻と条件の注入。拡散の U-Net には画像以外の入力があります。一つは時刻 t(ノイズ除去の何ステップ目か)で、t を正弦波埋め込みでベクトル化し、各残差ブロックに足し込みます(より進んだ実装では、t から動的にスケール・シフトを生成して特徴を変調します。FiLM や adaptive group normalization と呼ばれる形)。これでネットワークは「ノイズが多い段階か少ない段階か」を区別して振る舞いを変えられます。もう一つは条件 c(テキストなど)で、U-Net の中間にクロスアテンション層を挟み、テキスト埋め込みに注目しながらノイズ除去します。これで「赤い車」などの指示を反映できます。

時刻の注入を、固定の加算ではなく時刻から動的に生成した係数で正規化を変調する形(adaptive group normalization)にしたのが、後の DiT の AdaLN([DiT (Diffusion Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0231_dit-diffusion-transformer) 参照)へつながる発想です。U-Net 時代の素朴な加算が、Transformer 時代の AdaLN へ洗練されていった、という連続性で理解すると見通しがよくなります。条件注入の歴史は「どこに、どのくらい賢く外部情報を流し込むか」の探索史でもあります。

## 手法の系譜と主要論文
U-Net の歴史は、医療画像から始まり拡散の標準骨格へ、そして Transformer への置き換えへと進みます。

Ronneberger, Fischer, Brox「U-Net: Convolutional Networks for Biomedical Image Segmentation」(MICCAI 2015、フライブルク大学、arXiv:1505.04597)が原論文です。エンコーダ/デコーダ + スキップ接続による「画像 → 同サイズ画像」の対称ネットワークを提案しました。動機は、医療画像は学習データが少なく(annotated 画像が数十枚しかない)、かつ出力はピクセル単位の予測が必要、という制約の中で意味と細部を両立させることでした。強いデータ拡張(弾性変形)と組み合わせ、少データで高精度を実現しました。

Ho, Jain, Abbeel「Denoising Diffusion Probabilistic Models(DDPM)」(NeurIPS 2020、UC Berkeley、arXiv:2006.11239)が、ノイズ除去を繰り返して画像を生成する拡散モデルを現代的に定式化し、そのノイズ予測ネットワークとして時刻埋め込み・自己注意付きの U-Net(PixelCNN++/Wide ResNet 系の改良版)を採用しました。これにより U-Net は拡散の標準骨格になりました。

Dhariwal & Nichol「Diffusion Models Beat GANs on Image Synthesis」(NeurIPS 2021、OpenAI、arXiv:2105.05233)は、U-Net の徹底的なアブレーション(チャンネル幅、注意層をどの解像度に置くか、注意ヘッド数、適応的グループ正規化、残差ブロックの BigGAN 風アップ/ダウンサンプリングなど)と分類器ガイダンスにより、拡散が GAN を生成品質で上回ることを示しました。この論文の改良 U-Net(ADM)は以後のベースラインになりました。

Rombach et al.「High-Resolution Image Synthesis with Latent Diffusion Models」(Stable Diffusion、CVPR 2022、arXiv:2112.10752)は、ピクセル空間ではなく VAE で圧縮した潜在空間で U-Net を動かすことで計算量を激減させ、クロスアテンションでテキスト条件を注入する形を確立しました([Latent Diffusion Models (LDM / Stable Diffusion)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0242_latent-diffusion-models))。U-Net + クロスアテンション + 潜在拡散という組み合わせが、テキスト画像生成の事実上の標準になりました。

その後 Peebles & Xie の DiT(ICCV 2023)が U-Net を Transformer に置き換え、スケーラビリティで上回ることを示しました([DiT (Diffusion Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0231_dit-diffusion-transformer))。現在は超高解像度・動画・大規模では DiT 系が主流になりつつありますが、計算資源が限られる場面や中小規模、エッジ推論では U-Net も依然として実用的で、両者は使い分けの関係にあります。

## 論文の実験結果(定量データ)
原 U-Net 論文(2015)は、ISBI 細胞追跡チャレンジの電子顕微鏡画像セグメンテーションで、わずか 30 枚程度の学習画像(強い弾性変形データ拡張を併用)でも当時の最高精度を達成し、warping error などの指標で他手法を上回って優勝しました。データが乏しい医療画像でも機能する点が高く評価され、これが U-Net 普及の原点になりました。

DDPM(2020)は、生成品質の指標 FID(値が小さいほど本物に近い)で、CIFAR-10(32×32 の小画像)において FID 約 3.17、Inception Score 約 9.46 を達成し、当時の多くの GAN に匹敵または上回りました。U-Net バックボーンと「ノイズ予測の単純な二乗誤差(simplified objective)」の組み合わせが、安定した学習で高品質生成を実現できることを示し、現在の拡散ブームの起点になりました。トレードオフは、生成に 1000 ステップ前後必要で遅い点で、この遅さが後の高速化研究([v-prediction パラメータ化](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0254_v-prediction) による蒸留や [Consistency Models](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0225_consistency-models) など)を生みました。

Dhariwal & Nichol(2021)は、改良 U-Net(ADM)+ 分類器ガイダンスにより、ImageNet 256×256 の条件付き生成で FID 約 4.59、ImageNet 512×512 で FID 約 7.72(いずれもガイダンスあり)を達成し、拡散が GAN(BigGAN-deep など)を品質で上回ることを定量的に示しました。アブレーションでは、注意層を 32×32・16×16・8×8 の複数解像度に置くこと、注意ヘッド数を増やすこと、適応的グループ正規化で時刻を注入することなどが FID 改善に効くと報告されました。トレードオフは、モデルが大きく(数億パラメータ)計算コストが高い点です。

これらの数値が示す大局は明快です。U-Net は、30 枚の医療画像から大規模 ImageNet 生成まで、「画像 → 同サイズ画像」の幅広いタスクで一貫して機能する堅実な骨格であり、拡散モデルが GAN を追い抜く時代の主力でした。FID という同じ物差しで、データ規模もタスクも大きく異なる設定にスケールできたことが、この骨格の汎用性を物語ります。

## メリット・トレードオフ・限界
メリットは、第一に「画像 → 同サイズ画像」のタスクに最適で入出力解像度が一致すること、第二にスキップ接続により意味(大局)と細部(高解像度)を両取りでき、シャープな出力が得られること、第三に畳み込み主体ゆえ並進等変性など画像向きの帰納バイアスを持ち、比較的少データでも強いこと、第四に拡散での実績が豊富で時刻・条件注入の手法が確立しており、実装・最適化の知見が蓄積していることです。

トレードオフと限界としては、まず畳み込みの受容野が限られ、画像全体にまたがる超長距離の関係は捉えにくく、自己注意の追加が必要になること(純粋な畳み込みだけでは大局整合が弱い)。次に、チャンネル数・注意層の位置・ブロック設計・アップ/ダウンサンプリング方式など手作りの要素が多く、Transformer ほど素直にスケールせず「どこを大きくすれば伸びるか」が読みにくいこと。さらに高解像度をピクセル空間で直接扱うと計算・メモリが重く、潜在空間での運用([Latent Diffusion Models (LDM / Stable Diffusion)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0242_latent-diffusion-models))が前提になりがちなこと。そして拡散の生成自体が多ステップで遅く、別途高速化手法が要ることです。

研究上の論点としては、U-Net と DiT のどちらをいつ使うべきか(データ規模・解像度・計算予算に応じた選択。一般に小規模・少データなら U-Net、大規模なら DiT が有利との経験則)、スキップ接続の情報を Transformer 系でどう代替・補完するか(U-ViT のようにスキップを Transformer に持ち込む試みもある)、ハイブリッド(畳み込み + 注意)設計の最適点はどこか、といった問いが残っています。

## 発展トピック・研究の最前線
U-Net の発想は拡散を超えて広く生きています。スキップ接続でマルチスケールの情報を統合するという考えは、物体検出・セグメンテーションの特徴ピラミッドネットワーク(FPN)などにも通じます。拡散の文脈では、U-Net と Transformer の良いとこ取りを狙う研究(U-ViT、DiffiT、Hybrid 設計で畳み込みは局所、注意は大局を担う)や、潜在空間でのより効率的な骨格設計が進んでいます。U-Net 内部のスキップ接続の特徴をキャッシュして再利用することで推論を高速化する手法(DeepCache 等)も登場しました。

条件注入の進化も見どころです。U-Net 時代の「時刻を加算」「条件をクロスアテンション」という素朴な形が、DiT の AdaLN(正規化のスケール・シフトを条件で動的生成)へと洗練され、さらに画像とテキストを対称に扱うマルチモーダル Transformer(MM-DiT)へ発展しました。この一連の流れを「条件をどう注入するか」という軸で追うと、生成モデルのアーキテクチャ史が一本の線でつながって見えます。空間的に密な制御(ControlNet のように U-Net の各解像度へ条件特徴を注入する手法)では、U-Net のマルチスケール構造がむしろ強みになる場面もあり、骨格選択はタスク依存だという理解が現在の到達点です。
