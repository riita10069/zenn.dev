# ビデオトークナイザ (VQ/離散潜在)

## ひとことで言うと
ビデオトークナイザは、動画のフレーム(連続的な数値だらけの画像)を「単語のような少数の離散コード(整数)」に圧縮する変換器です。画像を整数の列にできれば、文章を扱う言語モデルと同じ仕組み(Transformer)で動画を予測・生成できるようになります。トークン化世界モデルや動画生成モデルの土台となる前処理であり、その品質がモデル全体の予測品質の上限を決めます。

## 直感的な理解

画像や動画は、本質的に「連続値の巨大な配列」です。256×256 の RGB 画像はピクセル値が 256×256×3 = 196,608 個あり、それぞれが 0〜255 の値を取ります。動画ならこれが毎フレーム続きます。これをそのまま系列モデル(Transformer)で予測しようとすると、入力が巨大すぎて計算が破綻し、しかもピクセル単位の予測は分布が複雑(各ピクセルは連続値)で扱いにくい、という問題があります。

一方、言語モデル(GPT など)が大成功したのは、入力が「有限種類の離散トークン(単語)の列」だからです。語彙が5万語なら、次の単語は「5万択の分類問題」になり、扱いやすい。さらに、分類として予測すると「確率分布の中から特定の選択肢を選ぶ」ことができるため、回帰のように平均化されてぼやけることがありません。たとえば「次のフレームでキャラが右にも左にも行きうる」場合、回帰だと両者の中間(ぼやけた二重像)を出してしまいますが、分類なら「右」を確率0.5で選ぶといった鮮明な予測ができます。

そこで「画像・動画も、語彙(コードブック)を持つ離散トークンの列に変換できれば、言語モデルの強力な道具立てをそのまま使えるのではないか」という発想が生まれました。これを実現するのが VQ(Vector Quantization、ベクトル量子化)系のトークナイザです。トークン化世界モデル(IRIS、[IRIS (Transformer世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0063_iris-transformer-world-model))やインタラクティブ世界モデル(Genie、[潜在行動モデル (Latent Action Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0067_latent-action-model))は、まずこのトークナイザで動画を離散化してから系列モデルを回します。

## 基礎: 前提となる概念

- オートエンコーダ: 入力を小さな表現に圧縮し(エンコーダ)、そこから元を復元する(デコーダ)ニューラルネット。圧縮を通じてデータの本質を学びます。
- 潜在表現(latent representation): エンコーダが作る中間の圧縮表現。連続値なら「連続潜在」、整数なら「離散潜在」。
- コードブック(codebook): あらかじめ用意した有限個の代表ベクトルの辞書。各潜在ベクトルをこの中の最も近い1語に置き換えます。語彙(vocabulary)とも呼ばれます。
- 量子化(quantization): 連続値を有限個の代表値に丸める操作。VQ はベクトル単位でこれを行います。
- 微分可能性: ニューラルネットは勾配で学習しますが、「最近傍を選ぶ」量子化は段差のある操作で微分できません(どのコードを選んでも微小変化で出力が動かない)。これを回避する近似(後述の straight-through estimator)が必要です。

## 仕組みを詳しく

中核は VQ-VAE(Vector Quantized Variational Autoencoder)です。普通のオートエンコーダに「量子化(離散化)」を足したものです。

### ステップ分解

1. エンコード: 画像 `x : [3, 256, 256]` を畳み込みネットで圧縮し、空間的な特徴マップ `z_e : [D, 16, 16]` を得る。16×16 の各格子点が D 次元のベクトル。
2. 量子化(VQ): あらかじめ用意したコードブック `{e_1, ..., e_K}`(K 個の D 次元ベクトル、例 K=1024)を持つ。各格子点ベクトル `z_e[i,j]` を、コードブックの中で最も近い `e_k`(ユークリッド距離最小)に置き換える。記録するのは「どの番号 k を選んだか」という整数だけ。
   - 結果: `[D, 16, 16]` の連続特徴 → `16×16 = 256` 個の整数(各 0〜1023)= トークン列
3. デコード: 選ばれたコードベクトル列を畳み込みネットで元の解像度に戻し、画像 `x_hat` を再構成する。
4. 学習: 再構成誤差(`x` と `x_hat` の差)に加え、量子化のための2つの損失を足す。
   - codebook loss: コードブックのベクトルを、割り当てられた特徴に近づける。
   - commitment loss: エンコーダ出力を、選ばれたコードに近づける(エンコーダが辞書から離れすぎないように)。係数 β で重み付けします。
   - 量子化は微分不可能なので、勾配は straight-through estimator(量子化を恒等関数とみなして勾配をそのまま通すトリック。デコーダ側の勾配を、量子化を素通りさせてエンコーダに渡す)で逆伝播させます。

```
画像 [3,256,256]
  └─エンコーダ→ 特徴 [D,16,16]
       └─最近傍探索(コードブックK語)→ トークン [16,16] = 整数256個
            └─コードベクトルに戻す [D,16,16]
                 └─デコーダ→ 再構成画像 [3,256,256]
```

### 数値で見る圧縮率

- 元: 256×256×3 = 196,608 個の連続値
- トークン化後: 256 個の整数(各 log2(1024)=10ビット)= 2,560 ビット

桁違いに小さくなり、しかも「有限種類の記号列」になりました。これで動画予測が「次トークンを当てる分類問題」に変わります。圧縮率を上げると系列が短くなって系列モデルが軽くなりますが、その分、再構成品質が落ちる(細部が失われる)というトレードオフがあります。系列モデルの計算量は系列長の二乗に効くため、トークン数の削減は計算を大きく節約します。

### 動画特有の拡張

静止画トークナイザを動画に使う方法は大きく2つあります。第一に、各フレームを独立にトークン化し、時間方向の予測は系列モデル(世界モデル)に任せる方式。第二に、3D 畳み込みなどで時間方向も同時に圧縮し、時空間トークンを作る方式(隣接フレームの冗長性を圧縮できて効率的だが、フレーム単位のランダムアクセスが難しくなる)。Genie はフレームを時空間トークンに変換し、その上で潜在行動モデルとダイナミクスモデルを回します。動画は隣接フレームがよく似ているため、時間方向の圧縮で大きな効率が得られる一方、シーン転換やフレーム間で不連続な動きの再現が難所になります。

## 手法の系譜と主要論文

### VQ-VAE (van den Oord, Vinyals, Kavukcuoglu, NeurIPS 2017)

連続潜在を離散コードブックに量子化するオートエンコーダ VQ-VAE を提案。動機は、言語のように離散的で意味のある表現を画像・音声でも得ること、そして当時の系列生成モデル(PixelCNN/WaveNet)との相性の良さでした。離散潜在で高品質な再構成・生成を実現し、通常の VAE で起きがちな posterior collapse(潜在が無視され、デコーダが入力を見ずに生成してしまう問題)を回避しました。一方で、量子化が微分不可能で straight-through 近似が必要なこと、コードブックの一部しか使われない codebook collapse(死語問題)が起きやすいこと、量子化誤差で細部が失われることが課題として残りました。

### VQ-VAE-2 (Razavi et al., NeurIPS 2019)

コードブックを階層化(粗い/細かい2段の潜在)し、高解像度で鮮明な生成を可能にしました。当時 GAN の代表だった BigGAN に迫る多様で高精細な画像生成を尤度ベースで達成し、「離散潜在でも高解像度生成ができる」ことを示しました。階層化で構成が複雑になり、学習・サンプリングが重くなる代償があります。

### VQGAN (Esser, Rombach, Ommer, CVPR 2021)

VQ-VAE に敵対的損失(GAN, Generative Adversarial Network、本物そっくりかを判定する識別器との競争)と知覚損失(perceptual loss、事前学習ネットの特徴空間での近さ)を加え、同じトークン数でも見た目が格段に鮮明になりました。純粋な再構成損失(画素のL2誤差)だとぼやけるのを、知覚的・敵対的損失で補い、少ないトークンで高解像度を表現できるようにしたのが新規性です。これがトークン化生成の実用化を後押しし、DALL-E 系や Latent Diffusion(Stable Diffusion の前身)、世界モデルの事実上の標準部品になりました。GAN 学習の不安定さ、複数の損失項のチューニングコストが代償です。

### ViT-VQGAN (Yu et al., ICLR 2022) と量子化の改良

ViT-VQGAN は畳み込みの代わりに Vision Transformer をエンコーダ/デコーダに使い品質と効率を改善し、コードを因子分解した低次元空間で量子化することでコードブック利用率も上げました。さらに量子化方式そのものの改良が進みました。FSQ(Finite Scalar Quantization, Mentzer et al., ICLR 2024)は各次元を少数のレベル(例: 各次元を5〜8段階)に独立量子化することでコードブックを暗黙化し、明示的なコードブックの学習なしに、codebook collapse をほぼ起こさず高い利用率を達成する簡潔な方式です。MAGVIT-v2(Yu et al., ICLR 2024)は Lookup-Free Quantization(LFQ、コードブックを持たず符号ビットで量子化)で語彙を大きく(数万〜)拡張し、動画トークナイザで拡散モデルを上回る生成品質を示しました。これらは「コードブック利用率」と「語彙サイズ」を同時に伸ばす現代的な系譜です。

### 世界モデルでの利用: IRIS / Genie

IRIS([IRIS (Transformer世界モデル)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0063_iris-transformer-world-model))と Genie([潜在行動モデル (Latent Action Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0067_latent-action-model))は、いずれも入口にこの種のビデオ/画像トークナイザを置きます。トークン化なしでは、その後の自己回帰 Transformer による予測が成立しません。トークナイザの品質(量子化誤差の小ささ、コードブック利用率)が世界モデルの予測品質の上限を決めます。トークナイザが再現できないものは、どんなに優れた系列モデルでも予測できないからです。

## 論文の実験結果(定量データ)

トークナイザの品質は主に再構成品質と生成品質、コードブック利用率で測ります。指標の意味を補足します。

- 再構成 FID / rFID: 再構成画像と元画像の分布の近さを測る指標(Frechet Inception Distance、事前学習ネットの特徴分布の距離)。低いほど良い。VQGAN は VQ-VAE 系に対し、同じトークン数で再構成 FID を大幅に下げ(=見た目が鮮明)、これが下流の生成品質を押し上げました。ImageNet で VQGAN は 256×256 画像を 16×16=256 トークンに圧縮しても鮮明に再構成できることを示しました。
- コードブック利用率: K 語のうち実際に使われる割合。VQ-VAE 系では数%しか使われない codebook collapse が頻発しましたが、EMA(指数移動平均)更新、未使用コードのリセット、FSQ や LFQ の導入により、利用率がほぼ 100% 近くまで改善することが報告されています。利用率が低いと実効的な語彙(表現力)が落ち、再構成・生成が劣化します。FSQ は明示的コードブックを持たないため、設計上 collapse がほぼ起きません。
- MAGVIT-v2 (動画生成・圧縮ベンチマーク, Kinetics や UCF-101 など): LFQ による大語彙トークナイザで、同条件の拡散モデルを上回る動画生成品質(FVD などの動画版 FID)と圧縮性能を報告。「言語モデル(トークン自己回帰)が拡散に勝てる」ことを、トークナイザの品質向上によって示した点が重要です。論文タイトルの "Language Model Beats Diffusion" はこの主張を端的に表しています。
- アブレーションの一般的傾向: 知覚損失や GAN 損失を外すと再構成がぼやけ、量子化レベル(語彙サイズや空間圧縮率)を上げ下げすると「系列長(=系列モデルの計算量)」と「再構成品質」が直接トレードオフします。世界モデルでは、トークナイザの再構成品質を上げると予測フレームの細部が保たれ、方策・予測性能が向上することが繰り返し観察されています。

## メリット・トレードオフ・限界

メリット
- 連続画像を有限の離散記号列に変換でき、言語モデル(Transformer, 自己回帰生成)の道具をそのまま使える。
- 大幅な圧縮で系列が短くなり、系列モデルの計算が現実的になる。
- 離散表現は構造が明確で、サンプリングや確率モデリング(分布からの選択)が扱いやすく、鮮明な生成につながる。
- トークン列という共通形式により、画像・動画・音声・行動を同じ系列モデルで扱う「マルチモーダル統合」が容易になる。

トレードオフ・限界
- 量子化誤差により細部が失われる(再構成の上限がそのまま世界モデル・生成の上限を決める)。トークナイザが消した遠方の小物体は、系列モデルでも復元できない。
- codebook collapse(辞書の一部しか使われない)が起きやすく、利用率向上の工夫(EMA 更新、コードブックリセット、FSQ、LFQ 等)が要る。
- 量子化が微分不可能で straight-through 近似が必要、学習が繊細(損失バランスや学習率に敏感)。
- 鮮明さを出すには GAN/知覚損失など追加学習が必要で、チューニングコストが高く、GAN 学習は不安定になりやすい。
- 動画では時間的一貫性(フレーム間のちらつき、flicker)が課題で、フレーム独立トークン化だと時間方向の冗長を活かせず、時空間トークン化だと計算が重くなる。未解決の課題として、高圧縮・高品質・時間整合を同時に満たすトークナイザの設計が続いています。

## 発展トピック・研究の最前線

- コードブックなし量子化: FSQ / LFQ のように明示的なコードブックを持たず、量子化を単純なスカラー操作や符号化で実現する方式が主流になりつつあります。実装が単純で利用率が高く、語彙を大きくできます。
- 大語彙トークナイザ: 語彙を数万〜数十万に拡張し、表現力を上げる流れ。MAGVIT-v2 が代表で、動画生成で拡散モデルを上回りました。
- 連続トークン vs 離散トークン: そもそも量子化せず連続潜在のまま系列モデルに渡す手法(拡散ベース、連続オートリグレッション、潜在拡散)も台頭しており、「離散化は本当に必要か」という問いが研究の最前線です。世界モデルでは IRIS(離散)対 DIAMOND(連続拡散)の対比がこれに対応します。
- 1次元トークナイザ・効率化: 2次元格子ではなく、画像を意味的に圧縮した少数の1次元トークン列に変換する研究(TiTok など)も登場し、極端な圧縮率で品質を保つ方向が探られています。
- 自動運転への応用: 運転シーンの生成・予測やデータ拡張、鳥瞰図(BEV、上空から見下ろした俯瞰表現)の離散トークン化と系列予測など、トークン化世界モデルへ進む際の必須前処理として位置づけられます。運転映像は時間的整合と細部(遠方の小さな物体、信号機の色)の保持が特に重要で、トークナイザ設計の難所になります。安全に直結する小物体を量子化で消さない設計が課題です。
