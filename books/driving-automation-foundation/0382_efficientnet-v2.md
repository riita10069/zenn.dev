# EfficientNetV2

## ひとことで言うと
EfficientNetV2 は、画像から特徴を取り出すニューラルネットワーク(画像認識の「土台」となる部品)です。前世代の EfficientNet は「同じ計算量でとても高精度」という長所がありましたが、実際に学習させると時間がかかるという弱点を抱えていました。V2 はその遅さの原因を分析し、初期層を「Fused-MBConv」という GPU で速く動く部品に置き換え、学習中に画像サイズと正則化の強さを段階的に上げる「progressive learning(段階的学習)」を導入することで、精度を保ったまま学習を最大で十数倍速くしました。同じ精度の他モデルよりパラメータも数倍少なく済みます。

## 直感的な理解
ニューラルネットの研究では、長らく「推論時にどれだけ速いか」「どれだけ高精度か」が主な競争軸でした。しかし現場では、モデルを一から学習させる(train する)ための時間とコストも決定的に重要です。大きなモデルを ImageNet(後述する約 128 万枚の画像データセット)で学習させるのに、GPU や TPU を何十台も使って何日もかかることが珍しくありません。学習が遅いと、ハイパーパラメータを変えて試す回数が減り、研究のスピードそのものが落ちます。さらに気候負荷や電気代の観点でも、学習時間は無視できないコストです。

EfficientNetV2 が解こうとした問題は、まさに「学習が遅い」ことです。たとえるなら、燃費の良い車(EfficientNet V1)を作ったが、その車は組み立てに異様に時間がかかる工場で作られていた、という状況です。V2 は車の燃費(推論効率・精度)を保ったまま、工場の組み立て時間(学習時間)を大幅に短縮しました。鍵は2つあります。1つは「序盤に向かない部品を、序盤に向く部品へ差し替える」こと。もう1つは「学習の進み具合に合わせて教材の難易度を上げていく(小さい画像から大きい画像へ)」ことです。

## 基礎: 前提となる概念
理解に必要な用語を順に噛み砕きます。

CNN(畳み込みニューラルネットワーク)とは、画像を小さな窓(フィルタ)でスキャンしながら「ここに縦線がある」「ここにタイヤらしい丸がある」といった特徴を段階的に拾い上げる仕組みです。画像認識の標準的な道具で、EfficientNet も CNN の一種です。

畳み込み(convolution)とは、小さなフィルタ(例えば 3×3 の数値の表)を画像の上を滑らせ、重なった部分で掛けて足す計算を繰り返す操作です。CNN の計算量のほとんどはここに集中します。

depthwise(深さ方向)畳み込みとは、各チャネル(特徴マップの層)を、それぞれ専用のフィルタで独立に処理し、チャネルをまたいで混ぜない畳み込みです。普通の畳み込みより掛け算回数(FLOPs、浮動小数点演算の回数)が桁違いに少ないのが売りです。EfficientNet V1 が多用した部品 MBConv の中核がこれです。

MBConv とは「Mobile inverted Bottleneck Convolution」の略で、「1×1 畳み込みでチャネルを広げる(expand)→ depthwise 3×3 で空間方向を処理 → Squeeze-and-Excitation でチャネルの重要度を調整 → 1×1 畳み込みで元のチャネル数に戻す(project)」という流れの部品です。MobileNetV2 が提案した逆ボトルネック構造を踏襲しています([MobileNet (V1/V2/V3)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0397_mobilenet) を参照)。

NAS(Neural Architecture Search、ニューラルネット構造の自動探索)とは、層の種類・数・チャネル数などの組み合わせ候補の中から、最も良い構造を機械が自動で見つける技術です。EfficientNet V1 のベースモデル B0 はこの NAS で発見されました。

複合スケーリング(compound scaling)とは、EfficientNet V1 が提案した「深さ(層数)・幅(チャネル数)・解像度(入力画像サイズ)を、一定の比率で同時にバランス良く大きくする」拡大法です。B0 を起点に B1〜B7 へとこの規則で拡大します。係数 φ を1つ動かすだけで、深さは d=α^φ、幅は w=β^φ、解像度は r=γ^φ と連動して増えます(α,β,γ はグリッド探索で求めた定数で、α·β²·γ²≈2 という制約を満たすよう決めます。これは φ を 1 上げるごとに総 FLOPs が約2倍になることを意味します)。

FLOPs と実機速度は別物だ、という点も重要な前提です。FLOPs は理論上の掛け算回数で、紙の上の指標です。一方、実際の速度はメモリの読み書き回数(MAC、Memory Access Cost)や GPU/TPU の並列性にも左右されます。depthwise 畳み込みは FLOPs が小さくても、計算密度が低い(演算器あたりに運ぶデータが多く、演算ユニットを使い切れない)ため、実機では理論ほど速くなりません。この食い違いが V2 の出発点になります。

## 仕組みを詳しく
V2 の工夫は大きく3つです。

第1は Fused-MBConv の導入です。V1 の MBConv は「1×1 拡張 → depthwise 3×3 → SE → 1×1 戻す」でしたが、Fused-MBConv は最初の「1×1 拡張」と「depthwise 3×3」を、1つの普通の(密な)3×3 畳み込みに融合(fuse)します。

```
V1 MBConv:        1x1拡張 → depthwise 3x3 → SE → 1x1戻す
Fused-MBConv:     ┌─ 3x3 (拡張と空間畳み込みを融合した普通の畳み込み) ─┐ → 1x1戻す
```

普通の畳み込みは計算が密で GPU/TPU の演算ユニットを使い切るため、層が浅くチャネルが少ない初期段階では depthwise 版より実機で速くなります。具体的な数値で見ると、論文の分析では、ステージ1〜3 のような低チャネル領域では Fused-MBConv に置き換えると FLOPs もパラメータも増えるのに学習スループット(1秒あたり処理画像数)が上がる、という逆転が起きます。一方、全層を Fused に置き換えると今度は FLOPs とパラメータが過剰に膨らみ、後半の高チャネル層では遅くなります。そこで V2 は「序盤の数ステージだけ Fused-MBConv、後半は従来の MBConv」という配置を、training-aware NAS(学習速度も評価指標に含めた探索)で最適に決めました。論文の構成表では、おおむねステージ 1〜3 が Fused-MBConv、ステージ 4 以降が MBConv となっています。

第2は progressive learning(段階的学習)です。学習の序盤は小さい画像(例えば 128 ピクセル四方)と弱い正則化で速く回し、学習が進むにつれて画像サイズを大きく(例えば 300 ピクセル四方)し、同時に正則化も強めていきます。ここで V2 が見つけた重要な知見は「画像サイズと正則化の強さは連動させるべき」という点です。

具体的には学習を 4 つのフェーズに分け、各フェーズで入力解像度と正則化強度(データ拡張 RandAugment の強さ・Dropout の確率・Mixup の係数)を段階的に引き上げます。

```
フェーズ   入力解像度   RandAugment強度   Dropout/Mixup
  1        128px       弱                弱
  2        ~186px      やや弱            やや弱
  3        ~244px      中                中
  4        ~300px      強                強
```

なぜ連動が必要かというと、小さい画像は情報量が少なく過学習しにくいので強い正則化は不要(むしろ精度を下げる)、大きい画像は情報量が多く過学習しやすいので強い正則化が要る、という非対称性があるからです。画像サイズだけ上げて正則化を据え置くと、序盤に学んだ特徴と終盤に学ぶ特徴の難易度がちぐはぐになり精度が落ちます。逆に小さい画像に強い正則化をかけると、もともと少ない情報をさらに削ってしまい学習が進みません。両者を同時に上げることで、序盤は高速・終盤は高精度を両立できます。

第3はステージごとに異なるスケーリングと、画像サイズ上限を控えめにする工夫です。V1 は B7 で 600 ピクセルもの大画像を使い、これが GPU/TPU メモリを圧迫して大きなバッチを組めず、学習を遅くしていました。V2 は学習時の解像度上限を 480 ピクセル程度に抑え、さらに後半ステージほど層を厚くする非一様なスケーリングを割り当てました(V1 の複合スケーリングは全ステージを一律に拡大していた)。結果として EfficientNetV2-S / M / L という系列が作られました。tensor shape の感覚で言えば、入力が [B, 3, 384, 384] のとき、ステージを経るごとに空間は半分ずつ縮み、チャネルは増えていき、最終的に [B, 1280, 12, 12] 程度の特徴マップを得て、global average pooling で [B, 1280] のベクトルにしてから分類層へ渡します。

## 手法の系譜と主要論文
出発点は Tan & Le「EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks」(ICML 2019、Google、arXiv:1905.11946)です。NAS で発見した小さな基礎モデル B0 を、複合スケーリングで B1〜B7 へ拡大し、当時の ImageNet で最先端の精度効率(同じ FLOPs での精度)を達成しました。B0 の素地となったのが同じグループの MnasNet(Tan et al., CVPR 2019, arXiv:1807.11626)で、これはモバイル端末上の実測レイテンシを報酬に組み込んだ NAS でした。

EfficientNetV2(Tan & Le, ICML 2021, arXiv:2104.00298)は V1 の弱点である学習速度を正面から扱いました。新規性は3点、すなわち Fused-MBConv、学習時間も最適化目標に含めた training-aware NAS、そして progressive learning です。動機は明快で「V1 は精度効率は良いが学習が遅い。学習速度そのものを最適化したい」でした。著者らは V1 の遅さの原因を3つに特定しています。すなわち(1)非常に大きな画像での学習はメモリを食いバッチサイズが小さくなる、(2)depthwise 畳み込みは序盤の層で遅い、(3)全ステージ一律のスケーリングが最適でない、です。V2 はこの3つにそれぞれ対処しました。

progressive learning の発想の系譜には、Touvron らの「Fixing the train-test resolution discrepancy」(NeurIPS 2019、arXiv:1906.06423)があります。これは「学習時と評価時で最適な解像度が違う(データ拡張のランダムクロップにより、学習時に見るオブジェクトは評価時より大きく見える)」ことを指摘し、解像度の扱いが精度に効くことを示しました。V2 はこれを学習中に動かすという形へ発展させ、かつ正則化と連動させた点に独自性があります。データ拡張の RandAugment(Cubuk et al., 2019)や mixup(Zhang et al., ICLR 2018)も、正則化スケジュールの構成要素として取り込まれています。

その後、Vision Transformer(ViT, Dosovitskiy et al., ICLR 2021)系や ConvNeXt(Liu et al., CVPR 2022)など、純粋な CNN を超える、あるいは CNN を Transformer の知見で再設計する流れが主流になっていきます。EfficientNetV2 は「純 CNN で学習効率と精度効率を両立する」系統の到達点の1つとして位置づけられます。

## 論文の実験結果(定量データ)
評価は主に ImageNet(1000 クラス、約 128 万枚の学習画像)の top-1 accuracy(画像 1 枚に対しモデルの第1候補が正解と一致する割合)で行われます。高精度域では、この指標が 1 ポイント上がるのはかなり大きな改善です。84% を 85% にするには、誤りの 16% のうち1ポイント分(全体の約 6%)を新たに正解にする必要があり、残るのは難しいサンプルばかりだからです。

報告された主な結果は以下のとおりです(ImageNet1k 学習)。

```
モデル              ImageNet top-1   パラメータ   学習速度の傾向
EfficientNet-B7     約 84.3%         66M          基準(遅い)
EfficientNetV2-S    約 83.9%         約 22M       V1比で大幅に高速・小型
EfficientNetV2-M    約 85.1%         約 54M       高速
EfficientNetV2-L    約 85.7%         約 120M      高速
```

論文の主張で最も強いのは学習速度です。EfficientNetV2 は、同等以上の精度に到達するまでの学習時間を V1 比でおよそ数倍〜十数倍短縮したと報告されています(著者らは TPUv3 上で「最大 11 倍速い学習・最大 6.8 倍小さいパラメータ」と表現)。さらに、より大きな学習データ(ImageNet21k、約 1300 万枚・2万クラス)で事前学習したうえで ImageNet1k へ転移すると、V2-L は top-1 で約 87% 台に達し、当時の大型 ViT(ViT-L/16 など)に匹敵しつつ学習は数倍速いと報告されました。転移学習(CIFAR-10/100、Flowers、Cars などへの fine-tune)でも、V1 や ViT を上回る精度を、より短い学習時間で達成しています。

アブレーション(構成要素を抜いて効果を測る実験)も示されています。Fused-MBConv を初期層に使わず全層 MBConv にすると、FLOPs は減るが実機学習速度が落ちます。逆に全層 Fused-MBConv にすると、FLOPs が増えすぎて遅くなります。「序盤だけ Fused」が実速度の最適点だ、という分業が定量的に裏づけられました。progressive learning については、解像度だけ段階的に上げて正則化を固定すると精度が頭打ちになり、両者を連動させたときに最も高い精度(報告では約 +0.7 ポイント)が得られることが示されています。固定解像度学習に対して、progressive learning は学習時間を大きく削りながら最終精度をむしろ向上させる、という二重の利得が確認されました。これにより「速度向上の主因は Fused-MBConv の配置と progressive learning の2つ」という主張が支えられています。

## メリット・トレードオフ・限界
メリットは、初期層 Fused-MBConv により実機での学習・推論が速いこと、progressive learning で学習時間を大幅短縮しつつ最終精度を保てること、V1 より少ないパラメータで同等以上の精度が出ること、S / M / L とサイズ展開があり用途で選べることです。多くの学習フレームワーク(timm など)に実装が含まれ、事前学習済み重みも入手しやすく、転移学習のバックボーンとして扱いやすい点も実用上大きな利点です。

トレードオフと限界もあります。第1に、progressive learning は「いつ解像度を上げ、いつ正則化を強めるか」というスケジュール設計が必要で、固定設定より運用の手間が増えます。学習ループ側が解像度や正則化を動的に変更できる作りでないと、この恩恵を受けられません(逆に言えば、推論時には progressive learning は関係せず、固定解像度でのレイテンシ・精度がそのまま効きます)。第2に、Fused-MBConv は普通の畳み込みなので FLOPs は depthwise より大きく、「FLOPs は増えたが実機速度は速い」という、紙の指標では捉えにくい特性を持ちます。FLOPs だけでモデルを比較する文化が根強いため、誤解されやすい点です。第3に、ImageNet 系の自然画像で最適化されており、医療画像や俯瞰(鳥瞰)特徴のような分布の異なるドメインでそのまま最適とは限りません。第4に、純 CNN ゆえ、大規模事前学習(数億〜数十億枚)で性能が伸び続ける Transformer 系と比べると、超大規模データ領域での伸びしろは限定的という指摘もあります。

## 発展トピック・研究の最前線
EfficientNetV2 以降、画像バックボーンの主戦場は CNN 対 Transformer のせめぎ合いに移りました。ConvNeXt(Liu et al., CVPR 2022)は ViT の設計要素(大きなカーネル 7×7、LayerNorm、GELU、少ない活性化と正規化など)を CNN に取り込み、純 CNN でも Transformer 級の精度が出ることを示しました。一方で MaxViT や CoAtNet のように畳み込みと自己注意を混ぜたハイブリッドも登場し、CoAtNet は ImageNet で 88〜90% 台に到達しています。

progressive learning の考え方は、学習効率化の文脈で今も発展しています。カリキュラム学習(易しい教材から難しい教材へ)の一種と見ることもでき、解像度・正則化を動かすという発想は、近年の大規模事前学習でも「序盤は低解像度で速く、終盤に高解像度で仕上げる」という形(FixRes 的な fine-tune、可変解像度学習)で広く使われています。training-aware NAS のように「探索の評価指標に学習コストや実機レイテンシを直接入れる」流れも、ハードウェア対応 NAS(hardware-aware NAS)として継続しています。実務では、EfficientNetV2 は速度と精度のバランスが良いベースラインとして、検出・セグメンテーションのバックボーンや、計算資源の限られた現場での選択肢として今も広く使われています。
