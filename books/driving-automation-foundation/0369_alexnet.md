# AlexNet

## ひとことで言うと
AlexNet は、2012年の ImageNet という画像認識コンペで、それまでの手法を圧倒的な差で打ち破った CNN (畳み込みニューラルネットワーク) です。この勝利が「ディープラーニング (深層学習) の時代」の幕開けとなりました。GPU を使った大規模学習、ReLU という活性化関数、Dropout という過学習対策など、今では当たり前になった道具を実用レベルで世に広めた点で、現代の深層学習すべての出発点と言える歴史的なモデルです。

## 直感的な理解
2012年以前、コンピュータに画像を分類させるには、人間が「画像のどこに注目すべきか」を手作業で設計していました。エッジの向きや色の勾配を数式で取り出す職人技です。これは「料理のレシピを人間が事細かに書く」ようなもので、書ける範囲を超えた賢さには到達できませんでした。

AlexNet がやったのは「注目すべき特徴も含めて、全部データから自動で学ばせる」ことです。料理人に何百万皿も作らせて、自分で美味しい作り方を見つけさせるようなものです。これ自体のアイデア (ニューラルネットワーク) は何十年も前からありましたが、3つの壁がありました。学習に膨大な計算が要る、層を深くすると学習が止まる、データが足りず丸暗記してしまう、です。AlexNet は「大量データ (ImageNet)」「速い計算機 (GPU)」「学習を止めない工夫 (ReLU)」「丸暗記を防ぐ工夫 (Dropout・データ拡張)」をちょうど良いタイミングで全部そろえ、この3つの壁を同時に突破しました。その結果が、コンペでの歴史的な圧勝でした。

## 基礎: 前提となる概念
ImageNet は約120万枚の訓練画像を1000クラスに分類する大規模データセットで、毎年コンペ (ILSVRC) が開かれていました。評価指標は top-5 エラー率 (予測の上位5位以内に正解が含まれない割合、低いほど良い) です。2012年以前、人手設計の特徴量 (SIFT や HOG といったエッジや勾配を数式で取り出す手法) を使った手法のエラー率は26%前後で停滞していました。

畳み込み (convolution) は、小さなフィルタを画像上で滑らせ、各位置で局所的な模様 (エッジ・色の塊など) を検出する操作です。これを重ねることで、浅い層はエッジ、中間層は部品、深い層は物体全体、と階層的に複雑な特徴を捉えます。

活性化関数は各ニューロンの出力をどう変換するかを決める関数で、ネットワークに非線形性を与えます。これがないと何層重ねても1つの線形変換に潰れ、複雑なパターンを表せません。

過学習 (overfitting) は、モデルが訓練データを丸暗記してしまい、未知のデータに弱くなる現象です。パラメータが多くデータが相対的に少ないと起きやすく、AlexNet のように数千万パラメータを持つモデルでは深刻な問題になります。

## 仕組みを詳しく
AlexNet の構造を数値例で追います。入力は 224×224×3 (論文では227×227として扱う実装も多い)。

1. 第1畳み込み層: 11×11 という大きなフィルタを96枚、ストライド (滑らせる歩幅) 4で適用 → 約55×55×96。続いてマックスプーリングで縮小。
2. 第2畳み込み層: 5×5 フィルタ256枚 → マックスプーリング。
3. 第3〜5畳み込み層: 3×3 フィルタを重ねる (384, 384, 256チャンネル)。
4. 全結合層: 4096 → 4096 → 1000 を通してクラス確率を出力。

```
224x224x3
  11x11 conv (96, stride4) → pool → 55x55x96 付近
   5x5  conv (256)         → pool
   3x3  conv (384)
   3x3  conv (384)
   3x3  conv (256)         → pool
  flatten → FC 4096 → FC 4096 → FC 1000 → クラス確率
```

合計8層 (畳み込み5 + 全結合3)、パラメータ数は約6000万でした。重要なのは構造そのものより、これを「実際に学習させて勝たせた」技術的工夫です。

### 鍵となった3つの工夫
1. ReLU 活性化関数: 当時主流だった tanh や sigmoid は、入力が大きいと勾配 (学習の手がかりとなる傾き) がほぼゼロになり、深い層で学習が止まる「勾配消失」を起こしました。ReLU (Rectified Linear Unit、入力が負ならゼロ・正ならそのまま) は正の領域で勾配が1のまま消えないため、学習が数倍速くなりました。論文では、同じ精度に到達する学習が tanh より約6倍速いと報告されています。これが深い CNN を現実的な時間で学習可能にしました。

2. GPU での並列学習: 当時の GPU (GTX 580, メモリ3GB) は容量が小さかったため、AlexNet はネットワークを2枚の GPU に分割し、一部の層でのみ両 GPU 間で情報をやり取りする工夫をしました。これにより CPU では何週間もかかる学習を約5〜6日に短縮し、「大規模な深層学習は GPU でやる」という今に続く標準を作りました。

3. 過学習対策: 約6000万パラメータは訓練データを丸暗記しやすいため、2つの対策を導入しました。
   - データ拡張: 画像をランダムに切り抜き・左右反転し、さらに RGB の主成分に沿って色を揺らすことで、見かけのデータ量を大きく増やしました。これだけで過学習を大幅に抑えたと報告されています。
   - Dropout: 学習中、全結合層のニューロンをランダムに約半分、一時的に無効化します。これにより特定のニューロンの組み合わせに依存しすぎず、頑健な特徴が学べます。Dropout がないと深刻な過学習が起きたと報告されています。

補助的に LRN (Local Response Normalization、近傍チャンネル間で出力を正規化する仕組み) も使われましたが、これは後の研究 (VGG など) で効果が薄いとされ、以後ほとんど使われなくなりました。

「なぜ2012年だったのか」を押さえると、AlexNet の歴史的意味がより鮮明になります。CNN の基本構造 (LeNet) は1990年代に既にあり、ReLU や Dropout の着想も単独では先行例がありました。欠けていたのは3つの条件が同時にそろうことでした。第一に大規模なラベル付きデータ (ImageNet は2009年公開、約120万枚というそれまでにない規模)。第二に汎用 GPU 計算環境 (CUDA が2007年に登場し、研究者が GPU を行列演算機として使えるようになった)。第三にそれらを組み合わせて深いネットを最後まで学習させるノウハウ (ReLU で勾配消失を避け、Dropout とデータ拡張で過学習を抑える) です。AlexNet は新発明の塊というより「機が熟した要素を正しく組み合わせて初めて成立させた」研究であり、だからこそ一度成立すると瞬く間に分野全体へ波及しました。

## 手法の系譜と主要論文
- 前史: ニューラルネットワークによる手書き数字認識の LeNet (LeCun et al., 1998) が CNN の原型で、畳み込みとプーリングの基本構造を確立していました。AlexNet はこれを ImageNet 規模のカラー画像へ大きくスケールさせたものです。

- AlexNet 本体: ImageNet Classification with Deep Convolutional Neural Networks (Krizhevsky, Sutskever, Hinton; NeurIPS 2012, トロント大学。第一著者名から AlexNet と呼ばれる)。提案は「GPU で学習する深層 CNN + ReLU + Dropout + データ拡張」。動機は「大規模データと GPU で深い CNN を実用化し、人手設計の特徴量に基づく従来手法を超える」こと。新規性は、これらの要素を統合して実際に巨大データで学習を成功させ、圧勝で実証した点です。

- この設計の不均質さ (大きなフィルタと巨大全結合層) を整理したのが [VGG](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0410_vgg_note) (Simonyan & Zisserman, ICLR 2015) で、フィルタを3×3に固定し均質に深くする原則を確立しました。

- さらに ResNet (He et al., CVPR 2016) が残差接続で100層超を可能にし、AlexNet の8層から飛躍的に深くしました。

- AlexNet が広めた ReLU・Dropout・データ拡張・GPU 学習は、その後の [ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext) や [Vision Transformer (ViT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0411_vision-transformer) を含むほぼすべての深層モデルの土台になっています。

## 論文の実験結果 (定量データ)
評価指標は ImageNet (ILSVRC) の top-5 エラー率と top-1 エラー率です。

- コンペでの圧勝: ILSVRC 2012 で AlexNet は top-5 エラー約16.4% (テストセットでは約15.3%) を達成し、2位の従来手法 (約26.2%) に約10ポイントの大差をつけて優勝しました。10ポイントという差は、当時の指標が年に1〜2ポイントずつ改善していた状況からすると桁外れで、これが研究コミュニティと産業界に衝撃を与えました。

- ReLU の効果: 同じネットワークで活性化関数だけを ReLU と tanh で比べると、ReLU は訓練誤差をある水準まで下げる速度が約6倍速いと報告されました。これは深い CNN を実用的な時間で学習できるかどうかを左右する差でした。

- 過学習対策のアブレーション: Dropout を外すと過学習が顕著に悪化したと報告されています。データ拡張も同様で、これらを外すとテスト精度が大きく落ちました。約6000万パラメータを120万枚で学習する以上、正則化が不可欠だったことが定量的に裏づけられました。

- 重ね合わせプーリング: 窓を一部重ねるマックスプーリング (overlapping pooling) は、重ねない場合よりエラーをわずかに下げ (top-5で約0.3ポイント)、過学習もしにくくなったと報告されました。

これらの数字が重要なのは、単に精度が高かったからではなく、「人手で特徴を設計する時代が終わり、データと計算で表現を学ぶ時代が始まった」という方向転換を、誰の目にも明らかな大差で示したからです。この1回の勝利が、その後10年以上の研究の流れを決定づけました。

## メリット・トレードオフ・限界
メリット
- ディープラーニングの実用性を世界に証明した歴史的意義。一つの結果が分野全体の方向を変えた。
- ReLU・Dropout・データ拡張・GPU 学習という、今も使われる基礎技術の普及点。
- 構造が単純で、CNN の基本 (畳み込み → プーリング → 全結合) を学ぶ教材として明快。

トレードオフ・限界
- 11×11 など大きなフィルタや巨大な全結合層で、パラメータ効率・計算効率が悪い。
- 残差接続がなく深くできないため (8層止まり)、表現力に限界がある。
- 2 GPU への手動分割や LRN など、現代では不要・非推奨となった工夫が含まれる。
- 分類精度は現代のモデルに大きく劣り、実用バックボーンとしては使われない。

## 発展トピック・研究の最前線
AlexNet 以降、CNN は「より深く、より効率的に」進化しました。VGG が均質な深層化を、GoogLeNet が並列モジュールによる効率化を、ResNet が残差接続による超深層化を実現し、その後 EfficientNet が精度と計算量のバランスを系統的に最適化しました。やがて [Vision Transformer (ViT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0411_vision-transformer) が畳み込みを使わない路線を開き、さらに [ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext) のように Transformer 時代の知見を CNN に逆輸入する流れも生まれました。AlexNet が普及させた ReLU・Dropout・データ拡張・GPU 学習という4本柱は、これらすべての世代を貫く共通基盤として今も生きています。歴史を振り返るとき、AlexNet は「なぜ今のモデルがこの形をしているのか」を理解するための出発点として外せない存在です。
