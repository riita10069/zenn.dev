# プランナーの整理 (GRU削除)

## ひとことで言うと
車がこれからどう動くか(軌道)を決める部品を「プランナー」と呼びます。プランナーには複数の作り方があり、歴史的に何種類も併存しがちですが、そのうちメモリを多く食い処理が遅い「GRUプランナー(系列を1ステップずつ生成する回帰型)」を削除し、軌道全体を一括で出せる「Flow Matchingプランナー」と「ベジエ(Bezier)プランナー」に絞る、という整理のパターンを扱います。なぜ逐次生成が不利なのか、一括生成の2方式がどう補完し合うのかを掘り下げます。

## 直感的な理解
プランナーが出すのは「これから数秒間、どう走るか」の計画です。これには2つの作り方の発想があります。

ひとつは「1歩ずつ書いていく」やり方。まず1ステップ目の操作を決め、それを踏まえて2ステップ目を決め、と順番に積み上げます。人が文章を一文字ずつ書くのに似ています(これがGRU=回帰型のやり方)。直感的で因果関係が自然ですが、64ステップ書くなら64回順番に処理せねばならず、前のステップが終わるまで次に進めません。遅く、しかも途中経過を全部覚えておく必要があるのでメモリも食います。

もうひとつは「全体を一気に描く」やり方。軌道を丸ごとひとつの曲線や、ノイズから滑らかに変形させる流れとして表し、64ステップを並列に・一度に出します。前のステップを待たないので速く、覚えておく中間状態も少なくて済みます。ベジエ曲線(数点の制御点で滑らかな曲線を描く)とFlow Matching(ノイズから目標へ変形を学ぶ生成モデル)がこちらです。プランナーの整理は、この「遅い逐次型」を捨てて「速い一括型」2つに絞る判断です。

## 基礎: 前提となる概念

**プランナー(planner、計画器)**: 車の周囲の情報(BEV特徴、地図、過去の動きの履歴など)を受け取り、「これからのN ステップぶんの操作」を出力するモジュール。「どう走るか」を出す最後の頭脳にあたります。

**制御信号としての出力**: プランナーの出力は、生のxy座標とは限りません。多くの設計では、加速度(acceleration)と曲率(curvature、ハンドルの曲がり具合)という制御信号を出し、それを車両運動モデル(unicycle dynamics=一輪車近似など)に通して実際の軌跡に変換します。こうすると物理的に実行可能な軌道だけが出る、という利点があります。出力の形はたとえば `(B, num_timesteps * num_signals)`、num_timesteps=64、num_signals=2(加速度と曲率)です。

**GRU(Gated Recurrent Unit)**: ゲート付き回帰型ユニット。系列を1ステップずつ順番に処理するニューラルネット。LSTMを簡略化したもので、過去の状態を「隠れ状態」として持ち回ります。

**自己回帰(autoregressive)**: 前のステップの出力を次のステップの入力に使う生成方式。1ステップ目が決まらないと2ステップ目に進めないため、本質的に逐次的(並列化できない)です。

**計算グラフ(computational graph)**: 勾配を逆向きにたどって学習する(誤差逆伝播)ために、ネットワークが行った計算を記録したもの。逐次処理が長いほどこのグラフが長くなり、保持するためのメモリ(VRAM)が増えます。

**生成モデル(generative model)**: データの分布そのものを学び、そこから新しいサンプルを生み出すモデル。軌道を「もっともらしい軌道の分布」として扱い、そこからサンプリングする発想です。Flow Matchingはこの一種です。

## 仕組みを詳しく

残す2つのプランナーが具体的に何をするのかを見ます。前提として、どのプランナーも出力フォーマットは同じ(加速度・曲率の系列)に統一しておくのが定石です。そうすれば損失計算やプランナーの差し替えが共通化できます。

### ベジエプランナー

ベジエ曲線(Bezier curve)は、少数の「制御点」で滑らかな曲線を定義する数式です。処理は次の通り。

1. 入力(BEV特徴・視覚履歴・自車状態)をまとめてひとつの文脈ベクトルにする。
2. そこから少数の制御点を予測する。たとえば num_controls=5 個、各制御点は num_signals=2 次元。
3. 事前計算したBernstein基底(ベジエ曲線の数式の重み行列、形状 `[64, 5]`)を掛けて、5点を64ステップの軌道へ一気に展開する。

数式で書くと、64ステップの軌道 `T` は `T = B @ P` で得られます。`B` は形状 `[64, 5]` のBernstein基底行列(時刻ごとに5つの制御点をどう重み付けるか)、`P` は形状 `[5, 2]` の制御点。行列積ひとつで全64ステップが同時に出るため、逐次ループがありません。

この設計の利点は、軌道が制御点の線形結合になるため、加速度・曲率のプロファイルが構造的に滑らかになることです。急なガクつき(高ジャーク)が出にくく、後処理の平滑化フィルタも、滑らかさのための追加損失も要りません。「滑らかさが数式で保証される」のがベジエの強みです。

### Flow Matchingプランナー

Flow Matchingは、ノイズから目標軌道へ「滑らかに変形させる流れ(速度場)」を学ぶ生成モデルです。

- **学習時**: 目標軌道 `x_1` とランダムノイズ `x_0` を直線でつなぎ、`x_t = (1-t)*x_0 + t*x_1` とします。`t` は0〜1の進み具合。この直線上を進む速度は単純に `x_1 - x_0`(始点から終点への差)。ネットワーク `v_θ(x_t, t, c)`(`c` は条件=BEV特徴など)が、この速度を当てるようにMSE(平均二乗誤差)で学習します。「どの中間地点でも、目標へ向かう正しい向きと速さを答えられるように」訓練するわけです。
- **推論時**: 新しいランダムノイズから出発し、常微分方程式 `dx/dt = v_θ(x, t, c)` を `t=0→1` までEuler法(微分を小刻みに足し込む数値解法)で積分して軌道を得ます。積分ステップ数はたとえば10ステップ。GRUの64回逐次に比べてはるかに少ない反復で済みます。
- 速度ネットワークの中身は、軌道を num_timesteps 個の「行動トークン」として扱い、平坦化したBEVグリッド(H×W のキー/バリュー)にcross-attentionで条件付けし、時刻 `t` と条件は AdaLN(適応的層正規化、DiTというモデルで使われる注入方式)で与える、という構成がよく使われます。

Flow Matchingの強みは、軌道分布が多峰(マルチモーダル=「右に避ける」と「左に避ける」のように複数のもっともらしい解がある)でも、それを表現できることです。回帰で平均を取ると「両者の中間=正面衝突」になってしまう問題を、生成モデルなら回避できます。

### なぜGRUを外すのか(まとめ)

GRUには2つの実務的弱点があります。第一に遅い。64ステップの軌道を作るには64回の逐次計算が必要で、並列化できません。第二にメモリを食う。逐次的に状態を持ち回るため学習時の計算グラフが長くなり、VRAM消費が大きくなります。ベジエ(行列積1回)もFlow Matching(数ステップの積分)も、この逐次ループを持たないため、速度・メモリの両面で有利です。

```
GRU:          step1 → step2 → ... → step64   (64回、順番待ち)
Bezier:       制御点 → 行列積1回 → 全64ステップ (並列)
Flow Matching: ノイズ → 約10ステップ積分 → 全64ステップ (少反復)
```

## 手法の系譜と主要論文

- **Alahi et al., "Social LSTM" (CVPR 2016)**。歩行者の軌道を、各人にLSTM(GRUの親戚)を割り当てて逐次予測し、近くの人どうしの相互作用を「social pooling」で取り込んだ先駆的研究。多くの軌道予測がこのRNN/GRUによる逐次生成を踏襲しました。動機は系列の自然なモデル化。効果は実装が素直で履歴を扱いやすいこと。限界は逐次性ゆえの遅さと、長系列での勾配・メモリの扱いづらさ。プランナーからGRUを外す判断は、まさにこのトレードオフが効いた結果です。

- **Lipman et al., "Flow Matching for Generative Modeling" (ICLR 2023, arXiv:2210.02747)**。連続正規化フロー(Continuous Normalizing Flow)を、シミュレーション不要(simulation-free)で学習する枠組みを提案。動機は、拡散モデルより単純な目標(直線に沿った速度を当てるだけ)で生成モデルを学べること。効果は安定して速い学習・推論。限界は推論時に数値積分が要ること。

- **Liu et al., "Rectified Flow" (ICLR 2023, arXiv:2209.03003)**。ノイズとデータを直線で結ぶ「直線化」によって、少ない積分ステップでも高品質に生成できることを示した。Flow Matchingの推論を高速化する理論的背景で、「10ステップ程度で十分」という実用設定を支えます。

- **Black et al., "π0 (pi-zero): A Vision-Language-Action Flow Model for General Robot Control" (2024, arXiv:2410.24164)** およびその後継。ロボットの行動をFlow Matchingで生成し、学習時刻 `t` をBeta分布で偏らせる(難しい低 `t` 側を多くサンプルする)工夫を導入。`t = 0.999 - 0.999·Beta(1.5,1)` のような時刻サンプリングはこの系譜です。運転・操作の行動生成にFlow Matchingが有効であることを示した代表例です。

- **Peebles & Xie, "DiT: Scalable Diffusion Models with Transformers" (ICCV 2023, arXiv:2212.09748)**。拡散モデルの骨格をTransformerにし、時刻と条件をAdaLN(適応的層正規化)で注入する設計を確立。Flow Matchingプランナーの速度ネットワークがAdaLNで時刻・条件を入れる作りは、ここから来ています。

- **ベジエ/多項式による軌道表現**。多くのモーションプランニング研究が、Bernstein/ベジエや多項式で軌道を少数係数にパラメータ化してきました。動機は、低ジャークで実行可能な滑らかな軌道を保証すること。効果は制御器に優しい指令。限界は、表現力が制御点数に縛られ、急な機動を表しにくいこと。

系譜としては、「RNN/GRUによる逐次軌道生成(2016〜)」が長く主流でしたが、「拡散・Flow Matchingによる一括生成(2022〜)」と「曲線パラメータ化による滑らか保証」が台頭し、近年のプランナーは後者2系統に収束しつつあります。

## 論文の実験結果(定量データ)

軌道予測・計画の主要指標は次の通りです。意味も添えます。
- **ADE / minADE(Average Displacement Error)**: 予測軌道と正解軌道の各時点の距離の平均(メートル)。低いほど良い。minADEは複数候補のうち最良のものを採る。
- **FDE / minFDE(Final Displacement Error)**: 最終時点(数秒後)の距離。終端のずれを測る。低いほど良い。
- **衝突率(collision rate)**: 計画した軌道が他物体と衝突する割合(%)。低いほど良い。

定量結果の例:
- **Social LSTM (CVPR 2016)**: 歩行者軌道予測のETH/UCYデータセットで、相互作用を考慮しない単純なLSTMや線形外挿より明確に低いADE/FDEを達成。RNN系が軌道予測のベースラインを確立しました。一方、後年の研究では、回帰ベースのRNNは多峰性を表現できず、衝突を避ける複数経路の片方に平均化してしまう弱点が繰り返し指摘されています。
- **Flow Matching / Rectified Flow (ICLR 2023)**: 画像生成で、拡散モデルと同等以上の品質を、推論ステップ数を1桁減らして達成。Rectified Flowは数ステップ(極端には1ステップ)でも高品質を示し、これが軌道生成における「10ステップ程度の少反復で十分」という設定の根拠になっています。
- **π0系のロボット方策 (2024)**: Flow Matchingで生成した行動が、多様で実行可能な操作を高い成功率で生み出すことを実機・シミュレーションで示しました。多峰な行動分布(複数の正しいやり方)を扱える点が、回帰ベースを上回る要因として報告されています。
- **アブレーションの一般論**: 拡散・Flow系のプランナーでは、推論ステップ数を減らすと衝突率やADEが悪化するトレードオフが共通して観測されます。たとえば10ステップから2ステップに減らすと、計算は速くなるが終端誤差が増える、という関係です。ベジエ系では、制御点数を増やすと表現力(急機動への追従)が上がるが滑らかさの保証が弱まる、という逆方向のトレードオフが見られます。

数値の意味として、ADEの0.1mの差は、車線内の横位置のずれや障害物との余裕に対応し、衝突率の数%の差は安全性に直結するため、これらの小さな差は実用上重要です。

## メリット・トレードオフ・限界

メリット:
- 逐次ループが消え、軌道生成が速い(並列または少反復)。リアルタイム制御に有利。
- 学習時の計算グラフが短くなり、VRAM消費が減る。
- 維持するプランナーが2つに減り、コードとテストが単純になる。
- ベジエは構造的に滑らか、Flow Matchingは多峰(複数のもっともらしい軌道)を表現しやすい、と役割が補完的。「滑らかさ重視」と「多様性重視」を使い分けられる。

トレードオフと限界:
- Flow Matchingは推論で数値積分が要り、ステップ数を減らすと精度(衝突率・終端誤差)が落ちる。速度と精度のトレードオフを抱える。
- ベジエは制御点数で表現力が決まり、急な回避機動などは苦手。滑らかさと俊敏さが両立しにくい。
- GRUのような明示的な逐次因果構造を持つ予測が選べなくなる。長期依存を厳密に逐次モデル化したいタスクでは不利になりうる。
- 未解決の課題として、生成モデル系プランナーの「安全性の保証」(出力分布の裾に危険な軌道が紛れ込まないか)や、推論ステップ数の最小化と安全性の両立は、依然として研究途上です。

## 発展トピック・研究の最前線
拡散・Flow Matchingによる行動生成は、ロボット操作のDiffusion Policy(Chi et al., 2023)を起点に爆発的に広がり、自動運転計画にも波及しています。最前線では、(1)蒸留(distillation)や一貫性モデル(consistency model)で推論を1〜数ステップに圧縮する高速化、(2)安全制約を生成過程に組み込むguided sampling(危険領域を避けるよう誘導する)、(3)視覚言語モデルと結合して高水準の意図(「あの車に道を譲る」)を条件にする行動生成、が活発です。ベジエ系も、制御点を動的に増減させたり、運動学的制約を曲線パラメータに直接埋め込む方向で進化しています。両系統を組み合わせ、Flow Matchingで多様な候補を出し、ベジエ的な平滑化制約で実行可能性を担保するハイブリッドも検討されています。
