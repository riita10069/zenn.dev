# ResNet-50 バックボーン

## ひとことで言うと
ResNet は、層をどれだけ深くしても学習が破綻しないようにする「残差接続 (ショートカット配線)」という発明で、深層学習の流れを変えた CNN です。中でも ResNet-50 は、精度・速度・安定性のバランスがよく事前学習重みも豊富なため、画像認識の世界で最も広く使われる「定番のバックボーン」であり、新しい手法の良し悪しを測る基準点 (ベースライン) として今も現役です。

## 直感的な理解
直感に反する事実から始めます。「層を深くすればするほど賢くなるはず」と思いますよね。ところが 2015 年以前、ある程度以上に層を深くすると、かえって精度が下がる現象が観測されていました。しかもこれは、訓練データに合わせすぎて未知データで失敗する「過学習」とは別物で、訓練データに対してすら誤差が増えるという不思議な現象でした。これを劣化問題 (degradation) と呼びます。

考えてみると変な話です。深いネットワークは、浅いネットワークの上に「何もしない層 (入力をそのまま出力する恒等写像)」を積めば、原理的には浅いネットワークと同じ性能を出せるはずです。にもかかわらず実際にはそうならなかった。つまり「何もしない」という単純なことすら、深いネットワークには学習が難しかったのです。

ResNet の発想は逆転的です。「層に何もかも一から作らせるのではなく、入力との差分 (残差) だけを学ばせよう」。各層の出力に、その層の入力をショートカット配線でそのまま足し込む。すると、もし最適解が「何もしない」なら、層は差分をゼロにすればよく、これは一からゼロ写像を作るよりずっと簡単です。この小さな工夫だけで、100 層を超える深さでも安定して学習できるようになりました。後から振り返ると、この「差分を学ぶ」という発想は単なる工学的工夫を超えて、深層学習の最適化に対する理解そのものを変えた転換点でした。

## 基礎: 前提となる概念
勾配 (gradient): 学習とは、誤差 (予測と正解のずれ) を小さくするように重みを少しずつ調整する作業です。「重みをどちらにどれだけ動かせば誤差が減るか」を示すのが勾配で、これを出力側から入力側へ逆向きに伝える (誤差逆伝播; backpropagation) ことで全層の重みを更新します。

勾配消失 (vanishing gradient): 層が深いと、勾配を逆向きに伝えるうちに掛け算が繰り返され、値がどんどん小さくなって入力近くの層にほとんど届かなくなります。届かなければ更新されず、入力側の層が学習されないまま放置されます。劣化問題の主因の一つです。逆に値が爆発する勾配爆発もありますが、深層 CNN では消失のほうが厄介でした。

残差接続 (residual / skip connection): 層の入力 x を、層の変換 F(x) の出力にそのまま足し、`出力 = F(x) + x` とする近道配線です。誤差逆伝播のとき、この足し算を通って勾配が掛け算を経ずに入力側へ素通りできるので、勾配消失が大幅に緩和されます。数式で見ると、出力を入力で微分したとき `∂(F(x)+x)/∂x = F'(x) + 1` となり、F' が小さくても「+1」の項が必ず残るため勾配がゼロに潰れません。これが深いネットワークを学習可能にした核心です。

バックボーン (backbone): 生の画像を意味的な特徴に変換する大きな本体部分です。分類・検出・セグメンテーション・運転計画など、頭 (ヘッド) を付け替えて多様なタスクに使い回します。

事前学習重みとレシピ: バックボーンは ImageNet などで先に学習した重みを配るのが普通ですが、同じ「ResNet-50」でも、どんな学習手順 (レシピ) で得た重みかによって精度がかなり変わります。この点は後述しますが、初学者が再現性で最もつまずくポイントなので頭の隅に置いてください。

## 仕組みを詳しく
ResNet-50 の処理を、入出力の形 (tensor shape) とともに追います。

```
入力 1枚のカメラ画像: [B, 3, 256, 256]   (3 = RGB)
   stem (7x7 畳み込み + maxpool) で 1/4 に
ステージ1: [B,  256, 64, 64]   解像度 1/4
ステージ2: [B,  512, 32, 32]   解像度 1/8
ステージ3: [B, 1024, 16, 16]   解像度 1/16
ステージ4: [B, 2048,  8,  8]   解像度 1/32
```

解像度を半分にするごとにチャンネル (特徴の種類) を倍増させていくのは CNN の伝統で、「空間的にざっくり、意味的に豊か」へ情報を圧縮します。分類ヘッドを外し、各ステージの特徴マップを丸ごと返す「features only」モードでバックボーンとして使うのが標準的です。検出やセグメンテーションでは、ステージ 1〜4 の異なる解像度の特徴を特徴ピラミッド (FPN) でまとめ、小さい物体は高解像度・大きい物体は低解像度の特徴で捉えます。

「50」はおおよその層数で、ResNet-50 は「ボトルネックブロック」という単位を積み重ねた構成です。ボトルネックは 1×1 → 3×3 → 1×1 の 3 つの畳み込みを残差接続でくるんだもので、最初の 1×1 でチャンネルを減らし、3×3 で空間的な計算をし、最後の 1×1 で元に戻します。次元を絞ってから空間計算をすることで、計算量を抑えつつ深さを稼ぎます。たとえばステージ 3 の入力 1024 チャンネルを 256 に絞って 3×3 を掛け、また 1024 に戻す。仮に 3×3 を 1024 チャンネルのまま掛けると計算量が 16 倍ほどになるので、この「絞り」が深い CNN を現実的なコストに収める鍵です。

レイアウトの注意点: ResNet は CNN の慣習どおり特徴を `[B, C, H, W]` (チャンネルが 2 番目) で返します。Swin Transformer 系は `[B, H, W, C]` (チャンネルが最後) で返すため、複数のバックボーンを差し替え可能にするなら、後段に渡す前に形を統一する変換を挟むのが定石です。形が同じでもチャンネル軸の位置が違うと意味が崩れて学習が進まない、という見つけにくいバグの原因になります。

残差ブロックの改良も押さえておきます。原典の ResNet は `出力 = ReLU(F(x) + x)` のように足した後に活性化を置きましたが、続く研究 (He et al., ECCV 2016) で正規化と活性化をブロックの「前」に置く pre-activation 構成のほうが、より深い構成で勾配がさらに素直に流れ、わずかに精度が上がると示されました。pre-activation では足し算の経路 (恒等写像) に活性化や正規化が一切挟まらないため、勾配が文字どおり最初から最後まで素通りできます。細かいですが、残差接続の本質が「足し算による勾配の素通り」にあることを裏づける結果です。

## 手法の系譜と主要論文
- ResNet (He et al., "Deep Residual Learning for Image Recognition", CVPR 2016, arXiv:1512.03385): 残差接続を提案した原典。深さの劣化問題を「表現力の不足」ではなく「最適化の難しさ」の問題と捉え、近道を作れば最悪でも恒等写像を学べるので性能が落ちないと考えた。この問題の切り分けこそが論文の知的核心。
- Identity Mappings in Deep Residual Networks (He et al., ECCV 2016, arXiv:1603.05027): pre-activation により 1000 層級まで安定学習できることを示し、残差接続の理論的理解を深めた。なぜ恒等写像の経路をきれいに保つことが重要かを定式化した。
- ResNeXt (Xie et al., CVPR 2017, arXiv:1611.05431): ボトルネックの 3×3 をグループ畳み込みに置き換え、「深さ・幅」に加えて「カーディナリティ (並列経路の数)」という新しい拡張軸を導入した。同じ計算量でも経路を増やすほうが精度が上がることを示した。
- ResNet strikes back (Wightman et al., 2021, arXiv:2110.00476): アーキテクチャは変えず、最新の学習レシピ (強いデータ拡張、長いスケジュール、ラベルスムージング、Mixup/CutMix 等) だけで ResNet-50 の精度を大きく引き上げられることを示した。同じ「ResNet-50」でも重みのレシピ次第で数ポイント精度が変わるという、ベンチマーク比較上きわめて重要な注意喚起。
- 後継として ConvNeXt (Liu et al., CVPR 2022) は、まさにこの ResNet を出発点に Transformer の設計を移植して近代化したもの。ResNet-50 を選ぶ意味は、こうした新手法と同じ土俵で比較できる確固たる基準点を持てる点にある。

## 論文の実験結果 (定量データ)
指標の意味: ImageNet-1K は約 128 万枚・1000 クラスの画像分類ベンチマークで、top-1 精度 (最も確信したクラスが正解の割合) で評価します。

- 原典の ResNet は、ImageNet 分類で 152 層という当時破格の深さを安定学習し、トップ精度を更新しました (ILSVRC 2015 優勝)。ResNet-50 の素の top-1 はおよそ 76% 前後と報告されています (原典の標準レシピ、90 エポック)。
- 劣化問題のアブレーションが論文の白眉です。残差接続のない素朴な深層 CNN (plain network) では、34 層が 18 層より訓練誤差すら高くなる (深いほど悪い) のに対し、残差接続を入れた ResNet では 34 層が 18 層より一貫して低い誤差になりました。「残差接続を抜くと深さの利益が消える」というこの対比が、残差接続の効果を最も雄弁に語っています。重要なのは、これが検証誤差ではなく訓練誤差で起きている点で、過学習ではなく純粋に「最適化が難しくなった」ことを示します。
- ResNet strikes back の報告では、改良レシピだけで ResNet-50 の top-1 が約 80% 超 (報告では最良で 80.4% 程度) へ引き上げられました。アーキテクチャ同一でレシピだけでこれだけ動くため、「新バックボーンが ResNet-50 を ○% 上回った」と言うときは、比較した ResNet-50 がどのレシピの重みかを必ず確認する必要があります。これは「素の 76%」と「強化レシピの 80%」では結論が逆転しうる、という研究上の落とし穴です。
- 下流タスクでも、ResNet-50 は COCO 物体検出や各種セグメンテーションのバックボーンとして長年の標準で、Faster R-CNN・Mask R-CNN・DETR など主要検出器のベースライン構成として無数の論文に登場します。たとえば Faster R-CNN + ResNet-50-FPN は COCO box AP がおよそ 37〜38 程度という、長く参照され続けてきた基準値を持ちます。この「比較可能性の高さ」自体が ResNet-50 の最大の価値です。

## メリット・トレードオフ・限界
- メリット: 枯れていて挙動が予測しやすく、学習が安定。事前学習重みが豊富で再現性が高い。軽量・高速で、比較ベースラインに最適。
- メリット: 残差接続という単純で頑健な仕組みのため、転移学習・ファインチューニングが素直に効きやすい。実装・デプロイのエコシステムが最も成熟しており、量子化・枝刈り・各種推論ランタイムでの動作実績が豊富。
- トレードオフ: 受容野 (一度に見られる範囲) が局所的で、画像全体にまたがる長距離依存の把握は自己注意を持つ Transformer 系に劣りやすい。
- トレードオフ: 同じパラメータ数あたりの到達精度は、近代化された Swin や ConvNeXt にやや及ばないことが多い。最新の SOTA を狙う本番モデルというより、信頼できる基準点という位置づけ。
- 限界・注意: 前述のとおり「resnet50」の精度はレシピ依存。どの重み (どのライブラリのどのタグ) を使ったかを記録しないと、実験の比較が成立しなくなる。これは初学者が最も陥りやすい再現性の落とし穴で、論文を読むときも「どの ResNet-50 か」を確認する癖をつけるとよい。

## 発展トピック・研究の最前線
- 残差接続の遍在化: 残差接続はもはや ResNet 固有ではなく、Transformer のブロックにも、拡散モデルの U-Net にも、ほぼすべての深層モデルに組み込まれている基礎技術です。「深いネットワークには近道を入れる」は現代深層学習の常識になりました。残差接続を「アンサンブルの暗黙的な集合」とみなす解釈 (Veit et al., 2016) など、その働きの理論的理解も進んでいます。
- アーキテクチャ探索との接点: ResNeXt (グループ畳み込みで幅を稼ぐ)、Wide ResNet (層を浅く幅広に)、RegNet (Radosavovic et al., CVPR 2020、設計空間を統計的に探索) など、ResNet を起点とした派生が数多くあり、効率と精度のトレードオフを探る研究の土台になっています。
- ベースラインとしての役割の進化: 近年の論文は ResNet-50 を「改良レシピ込み」で比較することが増えており、純粋なアーキテクチャ比較と学習レシピの寄与を切り分けることの重要性が広く認識されるようになりました。これは ResNet strikes back が提起した問題意識が定着した結果です。
- 自動運転知覚での実用: 軽量・高速で安定という特性から、車載のリアルタイム推論や、新しい知覚手法の効果を切り分ける比較対象として今も選ばれます。マルチカメラ 3D 検出の多くの論文が ResNet-50 を「標準バックボーン」として採用し、新手法はまずこの構成で効果を示すのが慣例です。複数バックボーンを同一の学習設定で回せば、アーキテクチャの違いが運転タスクの精度・速度にどう効くかを定量的に切り分けられます。
