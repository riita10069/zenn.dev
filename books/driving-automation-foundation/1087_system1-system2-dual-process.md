# System 1 / System 2 二重過程の自動運転モデル

## ひとことで言うと

人間の思考に「速くて反射的な System 1」と「遅くてじっくり考える System 2」の 2 種類があるという認知科学の考え方を、自動運転モデルの設計に持ち込んだ枠組みです。普段の運転は速い経路にまかせ、難しい場面だけ熟考する経路を働かせることで、リアルタイム性と賢さを同時に手に入れようとします。

## 直感的な理解

車を運転していて、まっすぐな道を一定速度で走っているときを思い浮かべてください。あなたはほとんど何も「考えて」いません。手は無意識にハンドルを微調整し、目は前方をなんとなく追い、判断は反射のように出てきます。これが System 1 的な処理です。

ところが、初めて来た複雑な交差点で、工事の誘導員が手信号を出していて、しかも対向車が譲ってくれているのか進んでくるのか分からない、という状況になると、あなたは一瞬立ち止まって「あの人は止まれと言っているのか進めと言っているのか」「対向車は減速しているか」と意識的に推論します。これが System 2 的な処理です。遅いけれど、慣れない状況に対応できます。

自動運転モデルにも、まったく同じ二つの要求があります。一つは速さ。車は 10〜20 Hz（1 秒に 10〜20 回）の頻度で判断を出し続けないと、制御がガクガクして危険です。もう一つは賢さ。交差点での譲り合い、工事規制、突然の飛び出しのような、まれだが難しい状況（これを long-tail、ロングテールと呼びます。「めったに起きないが種類が無数にある稀少ケースの裾野」という意味です）では、立ち止まって理由づけする処理が要ります。

毎フレーム重い推論を回せば賢いが遅すぎる。軽い反射だけにすれば速いが難所で間違える。この矛盾を解く設計思想が、二重過程（dual-process）です。要点は「速さと賢さはトレードオフだが、両方をいつも最大化する必要はない。ほとんどの瞬間は速さだけでよく、賢さは選択的に投入すればよい」という資源配分の発想にあります。

## 基礎: 前提となる概念

二重過程理論（dual-process theory）は心理学の理論です。心理学者 Daniel Kahneman が一般向け書籍 "Thinking, Fast and Slow"（2011）で広めました（用語自体は Stanovich と West が整理したものを Kahneman が普及させた）。要点はこうです。

- System 1: 速い・自動的・努力不要・並列的・直感的。慣れた運転操作や、見慣れた顔の認識のような処理。誤りやすいが安い。
- System 2: 遅い・意識的・努力が要る・逐次的・論理的。込み入った状況での熟慮や、桁の多い暗算のような処理。正確だが高くつく。

重要な但し書きとして、これは脳に物理的に二つの装置があるという主張ではなく、人間の判断を説明するための「モデル（説明の枠組み）」です。この区別を機械システムに移すときも、文字どおり二つの神経回路を作るというより「速い経路と遅い経路に役割分担する」という設計メタファとして使います。実際、二重過程理論は心理学内部でも単純化のしすぎだという批判があり、System 1 と System 2 がきれいに分かれない場面も多いと指摘されています。工学に持ち込むときも、この二分法は厳密な真理ではなく、有用な設計の補助線として扱うのが健全です。

機械学習側の前提語も押さえます。レイテンシ（latency）とは「入力を与えてから出力が返るまでの時間」で、車載では数十ミリ秒のオーダーが要求されます。仮に判断が 200 ミリ秒遅れれば、時速 60 km の車は約 3.3 メートル進んでしまうため、レイテンシは安全に直結します。推論（inference、ここでは reasoning の意味）とは、観測から直接出力を回帰するのではなく、中間的な仮説（誰が優先か、数秒後に何が起きるか）を立てて結論を導く処理を指します。世界モデル（world model）とは、現在の観測から「次に世界がどうなるか」を予測する内部モデルで、System 2 的な「先読み」を担う候補です。

## 仕組みを詳しく

二重過程を機械システムに落とし込む典型的な分担を、tensor shape を交えて説明します。

System 1 側（高速・常時稼働）が担うこと。カメラ画像や鳥瞰特徴（BEV, Bird's-Eye-View、上空から見下ろした地図状の特徴表現）から特徴を抽出し、即座に軌跡を出します。軌跡は `[B, num_timesteps * num_signals]` のような形で、B はバッチ数、num_timesteps は予測する未来の時間刻み数、num_signals は各時刻で出す量（x 座標, y 座標, 向き等）です。10〜20 Hz で回り、レイテンシが小さいことが絶対条件です。エンドツーエンド運転モデルの「画像から軌跡を出す主経路」がこれに相当します。

System 2 側（低速・必要時のみ稼働）が担うこと。「今この場面の支配要因は何か」「数秒後に世界はどう変わるか」をじっくり推論します。出力は軌跡そのものではなく、System 1 を条件づける文脈ベクトルや潜在表現です。たとえば `[B, D]`（D は潜在次元、256 など）の reasoning latent や、世界モデルが予測する未来の特徴マップです。System 2 は最終出力を出さず「ヒント」を出す、という分担が肝です。コストを抑えるため、高頻度では回さず低頻度（たとえば 1 Hz）や難所だけ起動します。この「System 2 はオプショナルな条件づけだけを供給する」という非対称な役割分担は、システム全体を後方互換に拡張する設計（[kwargs 経由のオプショナル入力でプランナ I/F を不変に保つ設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0993_optional-producer-kwargs-planner-interface) 参照）と自然に噛み合います。

両者の接続方式は主に二通りです。

カスケード（縦続）型。System 2 の出力（reasoning latent や予測表現）を System 1 に足し込んで条件づけます。両経路を一つの計算グラフでつなぎ、勾配を上流（System 2）まで流して同時に学習することもできます。利点は協調が密になること、欠点は System 2 を毎フレーム計算しがちで高コストになりやすいことです。

非同期・低頻度型。System 2 を System 1 とは別の頻度で走らせ、System 1 は「最新の System 2 結果」を参照します。重い推論を毎フレームやらずに済むのが利点ですが、System 2 の結果が古くなり、急変する状況に追従できないリスクがあります。

ASCII で全体像を描くとこうなります。

```
   カメラ/センサ ─┬─→ [System 1: 高速反射] ──(10-20Hz)──→ 軌跡 [B, T*S]
                  │            ↑ 条件づけ
                  └─→ [System 2: 低速熟慮] ──(1Hz or 難所のみ)──→ latent/未来予測 [B, D]
```

なぜこの分担が合理的か。計算予算の配分問題として見ると分かります。常に賢い推論を回す予算はない。だが、ほとんどのフレームは「直進し続ける」ような自明な判断で、賢さは要らない。賢さが必要なのは稀なフレームだけ。ならば、安い処理を常時、高い処理を選択的に、と配分すれば期待コストを最小化しつつ難所性能を確保できる、というのが二重過程設計の経済的な根拠です。具体的に数で考えると、System 2 が 1 フレームあたり System 1 の 50 倍の計算量を要しても、10 フレームに 1 回しか起動しなければ平均計算量は System 1 のおよそ 6 倍に収まり、毎フレーム起動（50 倍）より圧倒的に安く済みます。

## 手法の系譜と主要論文

心理学的土台は Kahneman "Thinking, Fast and Slow"（2011）です。System 1/2 という二分法を一般に定着させました。

これを AI 全般の課題提起に翻訳したのが Yoshua Bengio の NeurIPS 2019 招待講演 "From System 1 Deep Learning to System 2 Deep Learning" です。主張は「現在の深層学習は System 1（パターン照合・知覚）は得意だが、System 2（逐次的推論・因果・分布外への汎化）が弱い」。動機は、独立同分布でない状況や新しい組み合わせへの一般化（OOD generalization, systematic generalization）の脆さです。影響は大きく、その後の推論・因果・モジュール構造研究の方向づけになりました。限界は、System 2 を具体的にどう実装するかの処方箋までは示していない点です。

自動運転への直接的な適用例として、Tian et al. "DriveVLM: The Convergence of Autonomous Driving and Large Vision-Language Models"（2024, arXiv:2402.12289）があります。DriveVLM-Dual は、VLM（Vision-Language Model、画像と言語を統合的に扱う大規模モデル）による遅い意味推論を System 2 とし、従来型の高速プランナを System 1 とし、VLM の出力（シーン記述・行動意図）を高速経路に注入しました。動機は明確で、VLM 単体は推論が重く実時間性を満たせず、高速経路単体は long-tail に弱いから。新規性は、両者を協調させる dual-system 設計を実車スケールで示した点です。同系統では言語による運転推論を扱う Sima et al. "DriveLM: Driving with Graph Visual Question Answering"（2024, ECCV, arXiv:2312.14150）が、運転判断を一連の質問応答グラフ（知覚→予測→計画を質問の連鎖でつなぐ）として表現し、推論過程を明示化する方向を示しました。NVIDIA の Hydra-MDP / Alpamayo 系の取り組みも知られています。

「未来をじっくり予測する System 2」の方向では、Yann LeCun "A Path Towards Autonomous Machine Intelligence"（2022, OpenReview）が提唱した JEPA（Joint-Embedding Predictive Architecture）系の世界モデルが代表です。観測の未来をピクセルではなく特徴空間（埋め込み空間）で予測します。動機は、画素を当てにいくと不確実な細部に計算を浪費し脆くなるから、意味のある表現空間で予測する方が効率的で頑健だ、というもの。効果は予測的・熟慮的表現の獲得。トレードオフは学習の不安定さ（表現が一点に潰れる collapse のリスク）と計算コストです。

## 論文の実験結果（定量データ）

DriveVLM（Tian et al. 2024）は、独自に構築した SUP-AD（Scene Understanding for Planning）データセットと、公開ベンチマーク nuScenes で評価しました。nuScenes は屋外運転シーンの大規模データセットで、自動運転研究の標準ベンチマークの一つです。プランニング性能の指標は、予測軌跡と人間の実際の走行軌跡との L2 距離（メートル単位の位置誤差。小さいほど良い）と、衝突率（collision rate、予測軌跡で障害物にぶつかる割合。小さいほど良い）です。この L2 距離は「モデルが出した経路が、その場面で人間が実際に走った経路にどれだけ近いか」を測るもので、典型的には未来 1〜3 秒の平均で評価します。DriveVLM-Dual は、VLM の遅い推論を高速経路に統合することで、平均 L2 誤差と衝突率の双方で従来手法を上回ったと報告されています。特に複雑な交差点や稀なシーンでの改善が大きく、これは「難所でこそ System 2 が効く」という設計の主張を裏付けます。アブレーションとして、VLM 経路を外して高速経路のみにすると long-tail シーンでの性能が落ち、逆に VLM のみで実時間制御しようとすると周波数が足りず実用にならない、という二系統の相補性が示されています。

JEPA 系の代表例 I-JEPA（Assran et al. "Self-Supervised Learning from Images with a Joint-Embedding Predictive Architecture", 2023, CVPR, arXiv:2301.08243, Meta AI）は、画像表現の自己教師あり学習で、生成的手法（ピクセルを復元する MAE 等）より少ない計算量で高い線形プローブ精度（学習した表現の良さを、表現の上に線形分類器だけを乗せて測る指標）を達成しました。ImageNet で、ピクセル再構成ベースの手法に対し、同等以上の精度をより効率的な学習（より少ない事前学習エポックや GPU 時間）で得られると報告されています。これは「特徴空間での予測（System 2 的な先読み）が、画素再構成より効率的かつ頑健」という主張を支える定量的証拠です。

これらの数値が共通して示すのは、速い経路だけでは long-tail で取りこぼし、遅い経路だけでは間に合わない、両者を組み合わせると両立できる、という二重過程設計の核心です。重要なのは「System 2 を足すと平均性能が少し上がる」のではなく「滅多に起きないが致命的な難所での失敗を減らす」点にあり、平均 L2 のわずかな改善の裏に、稀少ケースでの大きな改善が隠れていることが多い、という読み解きが研究上は欠かせません。

## メリット・トレードオフ・限界

メリット。速さ（高頻度の反射）と賢さ（難所での熟慮）を役割分担で両立できます。重い推論を常時走らせないので、平均的な計算コストとレイテンシを抑えられます。System 2 を完全オプションとして設計すれば、それを切ってもデフォルトの安定した高速挙動を保てるため、段階的な導入と検証が容易です。

トレードオフと限界。第一に、二系統をどう協調させるか（注入の仕方・頻度・タイミング）の設計が難しく、ここに研究の多くが費やされます。第二に、非同期型では System 2 の結果が古くなり、急変する状況に追従できない「鮮度（staleness）」の問題が生じます。1 Hz で更新する reasoning が、0.5 秒前のシーンを前提にしていれば、その間に飛び出してきた歩行者には対応できません。第三に、VLM 等を System 2 に使うと、その出力のハルシネーション（もっともらしい誤り）や遅延が新たな弱点になります。言語モデルが「歩行者はいません」と自信を持って誤れば、高速経路がそれを信じて事故につながりえます。第四に、全体の複雑さが増し、検証・デバッグ・安全保証の負担が上がります。安全クリティカルな車載システムで「System 2 がいつ起動し、その結果がいつ反映されたか」を保証可能にすることは、未解決の工学課題です。

研究上の未解決問題として、「System 2 をいつ起動すべきか」を判断するメタ制御（どのフレームが難所かを安く見抜く仕組み）の設計が挙げられます。常時起動はコスト過大、起動が遅れれば手遅れ。この「考えるべきときを知る」能力自体が研究対象になっています。理想は「難所を、難所に突入する前に安く予見する」ことですが、それ自体がもう一つの予測問題であり、鶏と卵の関係にあります。

## 発展トピック・研究の最前線

近年は、大規模言語モデルの「思考の連鎖（chain-of-thought、答えに至る中間推論ステップを明示的に書き出させる手法）」や推論時計算（test-time compute、推論時に思考ステップを増やして性能を上げる）が、まさに System 2 の人工的実装として注目されています。自動運転でも、VLM に運転シーンを言語で説明させ段階的に推論させる研究が増えています。一方で、言語を介さず潜在空間で熟慮する latent reasoning や、世界モデルによる明示的な未来ロールアウト（複数の行動候補を内部でシミュレートして比較する）も、System 2 の有力な実装路線です。System 1 と System 2 を別々に学習するのか同時に学習するのか、勾配を共有するのか遮断するのか、といった学習戦略の選択も活発に研究されています。

評価方法そのものも論点です。open-loop 評価（記録データに対する軌跡予測誤差）では、二重過程の真価である「難所での回避」が平均指標に埋もれやすく、closed-loop 評価（シミュレータで実際に走らせ、衝突や逸脱を測る）でないと System 2 の寄与が見えにくい、という問題が知られています。そのため、長tail シーンを意図的に集めたベンチマークや、稀少ケースに重みを置いた評価指標の設計が、二重過程モデル研究の前進に不可欠だと考えられています。
