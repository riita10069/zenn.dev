# Dynamic Weight Averaging (DWA)

## ひとことで言うと

複数のタスクを1つのモデルで同時に学ぶとき、「最近あまり損失が下がっていない(=学習が遅れている)タスク」に自動で重みを多めにかけて、学習のバランスを取る手法です。人が係数を手で決める固定重み和([固定重み和損失 (Weighted Sum)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0575_fixed-weighted-sum-loss))と違い、各タスクの損失の下がり具合(下降率)を毎エポック観察して重みを動的に調整します。それでいて計算は「過去2回の損失の割り算とソフトマックス」だけなので、追加の学習パラメータが一切要らず、とても軽量です。

## 直感的な理解

複数の科目を並行して勉強している学生を想像してください。数学の成績はぐんぐん伸びているのに、英語の成績がここ最近ほとんど変わらないとします。賢い勉強配分は「伸びている数学はそこそこにして、停滞している英語にもっと時間を割く」ことです。DWA はまさにこれを自動でやります。最近の損失の下がり方を見て、停滞しているタスク(損失があまり下がっていないタスク)に多めの重みを割り当て、順調なタスクの重みを下げます。

なぜ単純な固定重みではダメなのでしょうか。学習は時間とともに状況が変わるからです。学習の序盤はタスク A が遅れていても、中盤になると A は追いつき、今度はタスク B が遅れる、ということが普通に起きます。固定の係数では「今どのタスクが遅れているか」に追従できません。DWA は損失の履歴を見て、その時点で遅れているタスクへ自動的に重みを移すので、固定重みのこの弱点を埋めます。

そして DWA が広く使われる最大の理由が「軽さ」です。後述する勾配ノルムを測る手法(GradNorm)や学習パラメータを増やす手法(不確実性重み付け)と違い、DWA は損失の値を覚えて割り算するだけ。既存の学習コードに数行足すだけで導入でき、学習を不安定にするリスクがほとんどありません。

## 基礎: 前提となる概念

マルチタスク学習(multi-task learning、1つのモデルで複数の予測目標を同時に学ぶこと)では、合計損失を各タスク損失の重み付き和として作ります。

```
L_total(t) = Σ_k λ_k(t) · L_k(t)
```

ここで k はタスクの番号、λ_k はタスク k の重み、t は学習の時刻です。問題は λ_k をどう決めるかです。固定重み和では人が定数を決めますが、タスクごとに学習速度が違うという厄介な性質があります。簡単なタスクは早く損失が下がりきり、難しいタスクはなかなか下がりません。簡単なタスクの勾配が学習を占有すると、難しいタスクが置き去りになります。

エポック(epoch)とは、学習データを一通り使い切る1周のことです。DWA は重みをエポック単位で更新します(ミニバッチごとではなく)。これは損失のノイズを平均化して安定させるためです。

ソフトマックス(softmax)は、複数の実数値を「合計が1になる正の比率(確率のような値)」に変換する関数です。値が大きいほど大きな比率になります。DWA はこれを使って、生の下降率を正規化された重みに変換します。温度(temperature)というパラメータ T で、変換の「メリハリ」を調整できます。

DWA の競合手法として、勾配ノルムを揃える GradNorm([GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))、不確実性を学習する Kendall らの手法([不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall))、勾配の向きを直す PCGrad([PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))があります。DWA はこの中で最も軽量で、損失の値だけを使う点が特徴です。

## 仕組みを詳しく

DWA の核心は「各タスクの損失の下降率(どれだけ速く下がっているか)を見る」ことです。下降率が小さい(あまり下がっていない)タスクは遅れているとみなして重みを上げ、急速に下がっているタスクは順調とみなして重みを下げます。3ステップで実装できます。

ステップ1。タスク k の相対的な下降率 r_k を、過去2エポックの損失の比で測ります。

```
r_k(t) = L_k(t−1) / L_k(t−2)
```

L_k(t−1) は1エポック前のタスク k の損失、L_k(t−2) は2エポック前の損失です。

- ほとんど下がっていなければ L_k(t−1) ≈ L_k(t−2) なので r_k ≈ 1.0(遅れている)。
- 急速に下がったなら L_k(t−1) < L_k(t−2) なので r_k < 1.0(順調)。

つまり r_k が大きいほど「最近停滞している=もっと重みが必要」という指標になります。比を使うので、各タスクの損失の絶対的なスケールには依存しにくい(スケール非依存)という利点があります。

ステップ2。生の下降率 r_k を、温度付きソフトマックスで重みに変換します。

```
λ_k(t) = K · exp( r_k(t) / T ) / Σ_j exp( r_j(t) / T )
```

K はタスク数で、これにより全タスクの重みの合計が常に K に保たれます(平均すると 1.0)。T は温度で、大きいほど全タスクの重みが均等(差が小さい)に近づき、小さいほど下降率の差が強調されます。論文では T=2 を使っています。学習の最初の2エポック(t=1, t=2)はまだ過去の損失が揃わないので、r_k を 1 に初期化し、全タスク均等(λ_k=1)で始めます。

ステップ3。得た重みで合計損失を作り、いつも通り最適化します。

```
L_total(t) = Σ_k λ_k(t) · L_k(t)
```

数値例で確認します。2タスク(A と B)で、過去2エポックの損失が

- タスク A: 8.0 → 7.6(r_A = 7.6/8.0 = 0.95、よく下がっている)
- タスク B: 3.0 → 2.99(r_B = 2.99/3.0 = 0.997、ほぼ停滞)

T=2 でソフトマックスを計算すると、exp(0.95/2)=exp(0.475)≈1.608、exp(0.997/2)=exp(0.4985)≈1.646。合計 3.254。K=2 なので

```
λ_A = 2 · 1.608 / 3.254 ≈ 0.988
λ_B = 2 · 1.646 / 3.254 ≈ 1.012
```

停滞しているタスク B の重みがわずかに大きくなりました。差は小さいですが、損失の停滞が続けば r_B は 1 に張り付き、A が下がり続けて r_A が小さくなるほど差は開きます。これがエポックを通じて「遅れているタスクへ重みを移す」挙動になります。

```
損失履歴を覚える（過去2エポック）
   ┌── L_k(t-2), L_k(t-1) ──┐
   │                        │
   ▼                        ▼
 r_k = L_k(t-1)/L_k(t-2)   （下降率：大きい＝停滞）
   │
 softmax(r_k / T) × K       （合計を K に正規化）
   │
 λ_k で重み付き和 L_total = Σ λ_k L_k
```

軽量さの根拠は明確です。追加で学習するパラメータがゼロで、必要なのは過去2エポック分の損失を覚えることと割り算・ソフトマックスだけ。勾配ノルムの計算も、学習可能パラメータの追加もないので、学習が不安定になりにくく、既存コードに数行足すだけで導入できます。

## 手法の系譜と主要論文

DWA の出典は Shikun Liu, Edward Johns, Andrew J. Davison による "End-to-End Multi-Task Learning with Attention"(CVPR 2019, arXiv:1803.10704)です。著者らは Imperial College London に在籍していました。この論文の主役は実は MTAN(Multi-Task Attention Network)というアーキテクチャで、共有エンコーダの各層にタスクごとの注意(attention)モジュールを付け、各タスクが共有特徴から必要な部分だけを選び取れるようにする構造です。DWA はその学習用の損失重み付け法として、いわば副産物的に提案されました。

DWA を提案した動機は明確に「軽量化」です。先行する重み付け手法、すなわち Chen らの GradNorm(ICML 2018, [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))は勾配ノルムを計算する必要があり、Kendall らの不確実性重み付け(CVPR 2018, [不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall))は学習可能パラメータを追加します。著者らは「損失の値だけ見れば学習速度のバランスは十分取れる、もっと軽くできる」と考え、損失の下降率という最小限の情報だけで重みを決める DWA を設計しました。

系譜の中で DWA は「損失ベースの軽量近似」という独自の位置を占めます。GradNorm が勾配(学習の実体)を見るのに対し、DWA は損失(勾配の結果)を見ることでコストを大幅に下げています。一方、向きの競合を扱う PCGrad(Yu et al. NeurIPS 2020, [PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))とは問題の軸が異なり、DWA は向きの衝突は解けません。後続の研究では、これら軽量手法と高度な勾配操作手法を比較するベンチマーク(Vandenhende et al. の MTL サーベイなど)で、DWA が「コストパフォーマンスの良いベースライン」として頻繁に参照されます。

## 論文の実験結果(定量データ)

論文の主要な評価は NYUv2 で行われました。NYUv2 は屋内シーンの RGB-D データセットで、3タスク(セマンティックセグメンテーション・深度推定・表面法線推定)を同時に学びます。評価指標は、セグメンテーションが mIoU(平均 IoU、予測領域と正解領域の重なり率の平均、大きいほど良い)と画素精度、深度が絶対誤差・相対誤差(小さいほど良い)、法線が角度誤差(度、小さいほど良い)です。

報告された結果の要点は次の通りです。アーキテクチャ MTAN は、共有部分が大きいベースライン(全層を全タスクで共有する split 構造や、各タスクを別々に学ぶ single-task 構造)に対して、同等以下のパラメータ数で全タスクの性能を改善しました。損失重み付けの比較では、DWA は等重み(全タスク固定で λ=1)と不確実性重み付けの中間〜同等の性能を示し、追加パラメータなしで安定して良い結果を出しました。具体的には、3タスクの相対性能(各タスクのベースライン比を平均した指標 Δ_MTL)で、DWA は等重みを上回り、不確実性重み付けと拮抗する数値が報告されています。MTAN + DWA の組み合わせは、セグメンテーションの mIoU、深度の絶対・相対誤差、法線の角度誤差(平均/中央値、および誤差が指定角度以内に収まった画素の割合)のいずれでも single-task ベースラインに匹敵かそれ以上を達成し、「1モデルで3タスクを同時に解いても各タスクが犠牲にならない」というマルチタスクの理想に近い結果を示しました。

アブレーションとして重要なのは、重み付け手法の比較と温度 T の影響です。論文は同一アーキテクチャ上で等重み・不確実性・DWA を比較し、DWA が等重みを上回ることを示しました。これは「学習進度に応じて重みを動かすこと自体に価値がある」ことの証拠です。温度 T については、T を大きくすると重みが均等に近づき DWA の効果が薄れ、小さくしすぎると重みがぶれて不安定になるため、中庸の T=2 が選ばれています。

指標の意味を補足します。mIoU はセグメンテーションの定番指標で、数ポイントの向上は物体境界の誤りが目に見えて減ることを意味します。法線の角度誤差が下がることは、面の向き推定が正確になり3次元理解が改善することを意味します。マルチタスクでは Δ_MTL のような「全タスクをまとめた相対指標」が本質で、DWA は無コストでこの総合指標を底上げできる点が評価されました。ただし、絶対的な最高性能を狙う設定では、より重い手法やタスク特化のアーキテクチャに譲ることもある、というのが正直な位置づけです。

## メリット・トレードオフ・限界

メリットは次の通りです。追加の学習パラメータがゼロで、学習が安定し、実装が数行で済むこと(軽量)。学習の進行に合わせて重みが自動追従し、固定重み和の「最適重みが時間で変わる」弱点を解消すること。損失の比を使うのでスケールにある程度ロバストなこと。調整すべきハイパーパラメータが温度 T ひとつだけなこと。

トレードオフと限界もあります。損失の値しか見ないので、勾配の向きの衝突(片方を下げると片方が上がる負の転移)は解決できません。これには PCGrad のような勾配操作が必要です。過去2エポックの損失に基づくため、損失がノイジー(ばらつく)だと重みもぶれます。初期の損失スケール差が極端だと、最初の数エポックは不安定になりえます。さらに「下降率が大きい(=停滞している)タスクほど重要」という暗黙の仮定が常に正しいとは限りません。単に学ぶ価値の低い、あるいはノイズで下がりにくいだけのタスクに過剰な重みを与えてしまう危険があります。

未解決の課題としては、損失ノイズに対するロバスト化(下降率の平滑化やエポック内推定)、向きの競合との統合、そして「停滞=重要」仮定が破れるケースの自動検出が挙げられます。

## 発展トピック・研究の最前線

DWA は「損失の動態(ダイナミクス)から重みを決める」という発想の代表で、これを発展させた手法がいくつもあります。Loss-Balanced Task Weighting(LBTW)や、検証損失の改善率を見る手法、不確実性と下降率を組み合わせるハイブリッドなどです。最近では、こうした手作りのルールを使わず、重み付け方策をメタ学習(学習のための学習)で獲得する Auto-Lambda(Liu et al. 2022)のような方向も出ています。

実務的な知見として、マルチタスクのサーベイ(Vandenhende et al. "Multi-Task Learning for Dense Prediction Tasks", TPAMI 2021)や複数の再現研究は、「どの重み付け手法が最良かはデータセットとタスク構成に強く依存し、十分にチューニングした単純手法が高度な手法に肉薄する」と一貫して報告しています。その中で DWA は「導入コストが極小で、等重みより安定して良い」という再現性の高い特性から、新しいマルチタスク設定を試すときの最初の一手として今も広く使われています。研究の最前線は、損失動態・勾配幾何・メタ学習を統合し、タスク構成に自動適応する重み付けへ向かっています。
