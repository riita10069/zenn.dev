# モデル設定から前処理を自動導出する (timm の resolve_model_data_config)

## ひとことで言うと
画像を AI モデルに入れる前には「サイズを揃える」「色の数値を決まった範囲に直す (正規化)」といった下準備 (前処理) が要ります。この前処理の値を人間が手で書くと、モデルが期待する値とズレて精度が静かに落ちる事故が起きがちです。`timm` というライブラリの `resolve_model_data_config` は「モデル自身が持っている設定」から前処理を自動で導き出し、このズレを根本から防ぎます。一般化すると「前処理はモデルと一体で管理し、人間に転記させない」という設計原則の実装です。

## 直感的な理解
学習済みモデルは「特定の眼鏡」をかけて世界を見るように訓練されています。たとえば「画像は必ず 256×256 にして、色は ImageNet の平均と標準偏差で標準化したもの」という前提で重みが調整されている。推論や追加学習のとき、別の眼鏡 (違うサイズ、違う正規化) で見せると、モデルは「見たことのない統計分布の画像」を渡されて混乱し、性能が落ちます。

問題は、この「正しい眼鏡の仕様」がモデルごとに違うことです。人間が各モデルの論文や設定を読んで前処理コードに書き写すのは、間違いの温床です。モデルを差し替えるたびに前処理も直さねばならず、直し忘れれば気づかぬうちに精度が劣化します。発想の転換は単純で、「眼鏡の仕様をモデル自身に持たせ、モデルに聞けば前処理が分かる」ようにすればよい。これが `resolve_model_data_config` の核心です。

## 基礎: 前提となる概念

- timm (ティム / PyTorch Image Models): Ross Wightman 氏が 2019 年頃から開発している、何百種類もの画像認識モデルとその学習済み重みを統一インターフェースで使えるライブラリです。今日の画像認識研究の事実上の標準実験基盤になっています。
- backbone (バックボーン): 画像を受け取って「特徴 (画像の意味を表す数値ベクトルやマップ)」に変換するネットワーク本体です。Swin Transformer V2、ConvNeXt V2、ResNet-50 などがよく使われます。
- 前処理 (preprocessing) / transform: モデルに入れる前に画像へ施す変換です。代表的なのはリサイズ (寸法をモデルの期待値に合わせる)、中央切り抜き (center crop)、テンソル化、正規化 (各ピクセル値から平均を引いて標準偏差で割る) です。
- 正規化 (normalization): 画像のピクセル値 (0〜255 を 0〜1 にしたもの) を、学習時に使ったのと同じ統計量で標準化することです。ImageNet で学習したモデルは `mean=[0.485, 0.456, 0.406]`, `std=[0.229, 0.224, 0.225]` という決まった値で正規化されています。この値は ImageNet 訓練画像の RGB チャンネルごとの平均と標準偏差です。
- 補間 (interpolation): リサイズで画素を補う方式。bilinear (双線形) や bicubic (双三次) があり、モデルの学習時と揃えるのが望ましいです。

問題を具体的に。Swin Transformer V2 の `swinv2_tiny_window8_256` は入力 256×256 を前提にしています。これは「窓 (window) サイズ 8 で 256×256 を処理する」と名前自体が宣言しているモデルです。誤って 224×224 でリサイズして渡すと、窓のサイズと特徴マップの寸法が噛み合わず、最悪エラー、よくても精度低下になります。一方 ResNet-50 は伝統的に 224×224。モデルごとに正解が違うわけです。

## 仕組みを詳しく

典型的なコードは次の 4 ステップです。

```python
backbone = timm.create_model(backbone_name, pretrained=False)
data_config = timm.data.resolve_model_data_config(backbone)
transform = timm.data.create_transform(**data_config, is_training=False)
del backbone
```

1. `timm.create_model(backbone_name, pretrained=False)`: 指定した名前のモデルを作ります。`pretrained=False` なので学習済みの重み (数百 MB) はダウンロードしません。ここで欲しいのは重みではなく「設定」だけだからです。設定は軽いので無駄なダウンロードを避けられます。

2. `timm.data.resolve_model_data_config(backbone)`: モデルが内部に持っているメタ情報 (`pretrained_cfg` / 旧称 `default_cfg`) から前処理設定を辞書として取り出します。典型的には次のキーが入ります。

```python
{
    "input_size": (3, 256, 256),     # チャンネル数, 高さ, 幅
    "mean": (0.485, 0.456, 0.406),   # 正規化の平均 (RGB)
    "std": (0.229, 0.224, 0.225),    # 正規化の標準偏差 (RGB)
    "crop_pct": 0.9,                 # リサイズ後にどの割合を中央切り抜きするか
    "interpolation": "bicubic",      # リサイズの補間方式
}
```

つまり「このモデルはどんな画像を期待するか」をモデル自身が教えてくれます。`swinv2_tiny_window8_256` なら `input_size` は `(3, 256, 256)`、ResNet-50 系なら `(3, 224, 224)` といった具合です。

3. `timm.data.create_transform(**data_config, is_training=False)`: 上の設定を展開して、実際に画像へ適用できる変換オブジェクトを組み立てます。これは「リサイズ → 中央切り抜き → テンソル化 → 正規化」を順に行う `Compose` です。`is_training=False` は「データ拡張 (ランダム反転や色変えなど) はしない、推論/検証用の決定的な変換にする」という指定です。学習時にランダム拡張を入れたいなら `True` にし、`auto_augment` などの追加引数を渡します。

4. `del backbone`: 設定を取り終わったらモデル本体は不要なのでメモリから消します。

こうして作った `transform` を、各カメラフレームに適用します ([動画 (.mp4) からの効率的なフレームデコードとセンサー時刻同期](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0832_physical-ai-av-sdk-video-decode) のステップ3)。結果、どの画像も backbone がぴったり期待する `(3, 256, 256)` の正規化済みテンソルになります。

### before / after で見る効果

- before (手書き前処理): backbone を `swinv2_tiny_window8_256` から `convnextv2_tiny` に変えたら、前処理コードの `input_size` や `mean/std` も人間が書き換える必要がある。書き換え忘れると精度が静かに低下する。`224` と `256` を取り違えても例外が出ないことすらあり、バグに気づきにくい。
- after (自動導出): `backbone_name="convnextv2_tiny"` と引数を変えるだけ。前処理は自動でそのモデル用に切り替わり、コードのどこも触らなくてよい。複数 backbone のベンチマーク比較が公平かつ安全になる。

### 数値例: 正規化のズレが何を起こすか

ピクセル値 0.5 (中間グレー) を考えます。正しい正規化 `(0.5 - 0.485) / 0.229 ≈ 0.066`。もし誤って ImageNet ではなく `mean=0, std=1` (正規化なし) で渡すと、モデルには `0.5` がそのまま入ります。`0.066` を期待する層に `0.5` が来れば、各層の活性値がずれて伝播し、出力は大きく狂います。畳み込みや正規化層は入力の統計分布に敏感なので、この小さなズレが層を重ねるごとに増幅されます。

## 手法の系譜と主要論文

前処理とモデルの整合は、論文の「手法」というより再現性 (reproducibility) と公平な比較の問題として論じられてきました。

- ImageNet 正規化の慣習 (AlexNet: Krizhevsky et al., NeurIPS 2012 以降): 画像分類モデルは ImageNet 訓練データの平均・標準偏差で正規化するのが事実上の標準になりました。学習時とテスト時で同じ統計を使うことが前提で、これがズレると「分布シフト (distribution shift)」が起きて性能が落ちることが繰り返し報告されています。モデルが「自分はどの統計で学習されたか」を記録し、それを忠実に再現する仕組みが望まれた背景です。
- timm / PyTorch Image Models (Wightman, 2019-, GitHub): timm はモデルごとに `pretrained_cfg` (既定設定) を持たせ、前処理をモデルと一体で管理する設計を広めました。狙いは「モデルを差し替えても前処理が自動追従し、ベンチマークが公平に比較できる」ことです。多くの研究が timm を実験基盤に使い、前処理の食い違いによる不公平な比較を避けています。`resolve_model_data_config` はその中核 API です。
- ConvNeXt (Liu et al., CVPR 2022, arXiv:2201.03545): ResNet を近代化して Transformer に匹敵させた研究で、timm を実験基盤に使いました。学習レシピ (前処理・拡張・正則化) を揃えて初めて、アーキテクチャ単体の貢献を切り分けられることを丁寧に示しました。前処理の統一が「公平な比較」の前提だという好例です。
- train-test resolution discrepancy / FixRes (Touvron et al., NeurIPS 2019, arXiv:1906.06423): 学習時とテスト時で画像解像度が違うと精度が変わることを定量的に示した研究です。「前処理の解像度をモデルと正しく合わせることが性能に直結する」ことの実証で、解像度を自動で合わせる仕組みの価値を裏づけます。

## 論文の実験結果 (定量データ)

指標は基本的に ImageNet 分類の Top-1 精度 (画像 1 枚に対してモデルの第 1 候補が正解と一致した割合、%。高いほど良い) です。

- FixRes (Touvron et al., 2019) は、ResNet-50 を 224×224 で訓練し 224 でテストした場合より、テスト解像度を上げて適切に微調整すると Top-1 が約 1〜2 ポイント改善することを示しました。逆に言えば「学習とテストで解像度・前処理がずれていると、それだけで 1〜2 ポイントを損する」ことを意味します。1〜2 ポイントは、ImageNet では新しいアーキテクチャ 1 世代分に相当する大きさで、無視できません。
- crop_pct (中央切り抜きの割合) の設定も精度に効きます。timm のモデルごとに最適値が異なり、これを取り違えると数百分の一〜数十分の一ポイントの差が出ます。`resolve_model_data_config` はこの値もモデルから取得するため、人間が忘れる余地がありません。
- 正規化統計のズレ (アブレーション): ImageNet 正規化を完全に外して `mean=0, std=1` で推論すると、ImageNet 学習済みモデルの Top-1 は数十ポイント崩落します (正常に動かないレベル)。これは「正規化はオプションではなく必須」であり、自動導出の価値が最も劇的に出る箇所です。
- 解像度ミスマッチ (Swin V2 の窓構造の例): 窓ベースの Transformer は入力解像度が窓構成と噛み合わないと、特徴マップの分割が破綻して精度が大きく落ちるか、そもそも実行時エラーになります。`input_size` をモデルから取ることで、こうした構造起因のミスを未然に防げます。

## メリット・トレードオフ・限界

メリット
- 前処理がモデルに自動追従するので、backbone を変えても前処理コードを書き換えずに済み、転記ミスによるズレ事故を根絶できる。
- 学習時とテスト時の前処理が一致し、再現性と公平なベンチマーク比較が保たれる。
- `pretrained=False` で設定だけ取るため、重みダウンロードの無駄がない。

トレードオフ・限界
- 複数台のカメラに同一の `transform` を適用すると、カメラごとに違う前処理 (画角補正・ホワイトバランス差の補正など) をしたい場合は別途工夫が要る。
- モデルの `pretrained_cfg` が想定する前処理 (ImageNet 由来の統計) が、対象ドメイン (自動運転の広角カメラ、夜間、悪天候) に最適とは限らない。ドメインに合わせた正規化統計を再計算して上書きする余地は残る。
- `is_training=False` 固定だと、このパスでは画像のデータ拡張が効かない。学習時の空間的拡張を入れたい場合は別経路が要る (時間方向の拡張はサンプル切り出しで別途行える)。
- 未解決の課題として、ImageNet 統計を自動継承するのが最善か、それともドメイン固有統計に置き換えるべきかは、ドメインギャップの大きさ次第で答えが変わり、一般的な指針は確立していません。

## 発展トピック・研究の最前線
- ドメイン適応的正規化: 対象データセットで mean/std を再計算する、あるいはバッチ正規化統計を対象ドメインで再推定する手法。ImageNet 統計とのギャップが大きい医療画像・衛星画像・自動運転で効きます。
- 学習レシピの標準化と公平比較: ConvNeXt が示したように、アーキテクチャの貢献を切り分けるには前処理・拡張・正則化を厳密に揃える必要があり、これを支えるのが設定の自動管理です。
- 解像度可変モデル: 入力解像度を柔軟に変えられるアーキテクチャ (FlexiViT 系) の研究。前処理とモデルの結合をさらに緩める方向です。
