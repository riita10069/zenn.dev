# 未来特徴予測 (FutureState型モジュール)

## ひとことで言うと

未来特徴予測モジュール(FutureStateと呼ばれる類のもの)は、「今のシーンの特徴」と「これから自車がどう動くつもりか(プランナーの意図)」を入力にして、一定刻み(例: 1.6秒ごと)で数ステップ先までの未来のシーン特徴を予測する小さなニューラルネットです。未来の画像そのものではなく、画像を抽象化した「特徴」を予測するのがポイントで、これがJEPA風の自己教師あり学習の予測ターゲットになります。運転AIに「数秒先を見通す感覚」を持たせるための部品です。

## 直感的な理解

運転を上手にこなすには、「今どう見えているか」だけでなく「このまま進むと数秒後にどうなるか」を内部で見通す力が役立ちます。人間も「あの車は前に割り込んできそうだ」「カーブの先はこうなっているはず」と未来を予測しながら運転しています。先を読めるからこそ早めに減速したり進路を譲ったりできるわけです。AIにこの「未来を見通す感覚」を持たせる手段が、未来予測モジュールです。

ただし「未来の生のカメラ画像をピクセル単位で正確に描く」のは非常に難しく、計算も重く、しかも運転に本当に必要な情報(他車の位置、車線、走れる空間)以外の枝葉(空の雲の形、路面の細かい模様)まで再現しようとして無駄が多くなります。そこで多くの手法は、ピクセルではなく特徴空間(feature space。画像をニューラルネットで抽象的な数値表現に変換した空間)で未来を予測する方針を取ります。これはJEPA(後述)の思想です。「絵を正確に描く」のではなく「要点だけを抽象的に当てる」のだ、と考えると直感に合います。試験でいえば、風景を一筆一筆そっくりに写生するのではなく、「ここに車がいて、車線はこう曲がっている」という要約メモを当てにいく、というイメージです。

## 基礎: 前提となる概念

特徴(feature)・特徴空間: ニューラルネットは生画像を、層を重ねるごとに「より抽象的な数値表現」へ変換します。最終的に得られる多次元配列が特徴で、その表現の住む世界が特徴空間です。低い層では「エッジ・色」、高い層では「車・車線・空きスペース」といった意味的な情報を表します。

tensor shape の読み方: シーン特徴はよく `[B, C, H, W]` の形で表されます。B はバッチサイズ(同時処理するサンプル数)、C はチャンネル数(特徴の次元、例: 256)、H と W は空間の縦横グリッドです。BEV(Bird's-Eye-View=鳥瞰図)特徴なら H×W は地面を上から見たグリッド解像度に対応します。たとえば `[8, 256, 450, 300]` は「8サンプル、256次元の特徴を、縦450×横300のマス目それぞれが持っている」という意味です。

行動条件付き(action-conditioned): 予測モデルに「自分がこれから取る行動(軌跡)」も入力する設計です。行動を入れないと「ただ流れに任せた未来」しか予測できませんが、行動を入れると「この行動を取ったらこうなる」という反実仮想(もしも)の予測ができます。これが [World Modelシミュレーション学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0031_world-model-simulation) のRL構想の土台になります。

凍結バックボーン(frozen backbone): 学習中に重みを更新しない(勾配を流さない)画像エンコーダです。バックボーンとは、画像から特徴を取り出す土台のネットワーク(ResNet や Swin Transformer など)を指します。未来予測の「正解」を作るときにこれを使い、正解側を固定することで学習の崩壊(後述)を防ぎます。

自己教師あり学習(self-supervised learning): 人手のラベルなしで、データ自身から教師信号を作る学習です。未来予測の場合、「未来のフレームは時間が経てば自動的に手に入る」ので、それを正解として使えます。ラベル付け(人が一枚一枚に「車」「歩行者」と注釈を付ける作業)のコストがかからないのが大きな利点です。

## 仕組みを詳しく

未来特徴予測モジュールの典型的な構造を、入力・処理・出力に分けて説明します。

入力は三つです。第一に融合済みのシーン特徴 `[B, C, H, W]`。複数カメラ画像などを処理・融合した、シーンの抽象表現です。C は特徴次元(例: 256)、H×W は空間グリッド(BEVなら例えば数百×数百)。第二にプランナーの隠れ状態 `[B, D]`(例: 256次元)。これは軌跡プランナー(GRUのような再帰型や、Flow Matching のような生成型)の最終隠れ状態で、「プランナーがこの先どう運転するつもりか」という意図を要約したベクトルです。第三に(オプションで)平坦化した計画軌跡 `[B, action_dim]`。行動条件付き予測のときだけ使います。

処理ステップは次のとおりです。

第一に、プランナーの意図を特徴に注入します。隠れ状態を線形層 `Linear(D, C)` でチャンネル数 C に射影し、`[B, C, 1, 1]` の形にして、シーン特徴の全空間位置に同じ値を足します(ブロードキャスト)。式で書けば `conditioned = fused_features + bias` のような形です。これで「どんな運転をするつもりか」がシーン特徴全体に染み込みます。なぜ足し算かというと、チャンネル方向の一様なバイアスとして意図を流し込むのが軽量で安定だからです(より凝った設計では FiLM のようなスケール+シフト変調や cross-attention を使うこともあります)。

第二に、オプションで行動を注入します。軌跡が渡されたら、同様に線形層で C 次元のチャンネルバイアスに変換して足します。これが反実仮想予測のための行動条件付けです。行動次元を指定せずに構築したのに軌跡を渡すとエラーにする、といった安全装置を入れるのが堅実な設計です(設定と実行時入力の不整合を早期に検出するため)。

第三に、未来を予測します。畳み込み層を数段重ねて特徴を変換します。たとえば `Conv2d(C → 2C, 3x3)` → GELU(滑らかな活性化関数) → `Conv2d(2C → 4C, 3x3)` のように、最終出力を `[B, 4C, H, W]` にします。この 4C は「4つの未来時刻ぶんの特徴」をひとまとめにしたものです(時刻数は設計次第)。空間構造を保ったまま予測したいので、全結合ではなく畳み込みを使うのが自然です。

第四に、時刻数ぶんに分割します。`torch.chunk(future, chunks=4, dim=1)` でチャンネル方向に4つに割り、それぞれ `[B, C, H, W]` の未来特徴にします。各々が +1.6s, +3.2s, +4.8s, +6.4s に対応する(=6.4秒先まで1.6秒刻み)、といった時間割り当てになります。

出力は、時刻数ぶんのテンソルのタプルで、各 `[B, C, H, W]` です。

メモリの注意点が重要です。空間解像度が高い(例: 数百×数百のBEVグリッド)と、`4C` チャンネルの出力は1サンプルあたり fp32 で数百MBに達し非常に重くなります。だから実装では混合精度(bf16/fp16。数値の精度を一部下げてメモリと速度を稼ぐ手法)や、予測前の空間ダウンサンプリングが事実上前提になります。フル解像度で未来予測をONにすると、ピーク使用VRAMが目に見えて増え、バッチサイズを大きくするとメモリ不足(OOM)になりやすい、という実測傾向が一般に報告されます。

数値例: C=256、H×W=450×300、未来4時刻なら出力は `[B, 1024, 450, 300]`。1サンプル(B=1)あたり 1024×450×300×4バイト ≈ 553MB。バッチ4なら約2.2GB、これに中間活性化や勾配が乗るため、フル解像度では容易に20GB級になります。L40S(45GB)クラスのGPUでも、フル解像度かつ未来予測ONでバッチを上げるとOOMする、というのがこのオーダー計算から予想できます。

## 手法の系譜と主要論文

未来特徴予測の設計思想は、二つの系譜の交点にあります。

一つはJEPA(Joint Embedding Predictive Architecture)系です。LeCun (2022) "A Path Towards Autonomous Machine Intelligence" が原典で、提案は「未来や欠損部分を、ピクセルではなく特徴(埋め込み)空間で予測せよ」。動機は「ピクセル予測は無駄な細部にとらわれ、抽象的な理解が育ちにくく、不確実性に弱い」こと。効果は「抽象表現が安定して学べる」。トレードオフは「ターゲット特徴の作り方(凍結バックボーンやEMAターゲットの質)に性能が依存する」点です。これを画像に具体化したのが Assran et al. の I-JEPA(CVPR 2023, arXiv:2301.08243)、動画に拡張したのが Bardes et al. の V-JEPA(2024, arXiv:2404.08471)で、後者の「動画の時空間マスクを特徴空間で予測する」発想が「未来フレームの特徴を予測する」未来特徴予測に最も近いです(詳細は [JEPA風特徴再構成損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0016_jepa-feature-reconstruction) と [V-JEPA (動画用JEPA表現学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0100_v-jepa_note))。

もう一つは世界モデル系です。Ha and Schmidhuber (2018) "World Models"(arXiv:1803.10122)と Hafner et al. の Dreamer シリーズは、行動条件付きで次状態を予測する世界モデルを学び、その中で計画・RLを行いました。未来特徴予測モジュールに行動条件付け(action_dim)が備わるのはこの系譜で、行動を変えたら未来がどう変わるかを予測できるからこそRLの土台になります。自動運転特化では Hu et al.(Wayve)の MILE(NeurIPS 2022, arXiv:2210.07729)がBEV状態の世界モデルを示し、現実の運転ログから学んだ潜在状態で将来のBEVセマンティクスを想像できることを実証しました。

対照的な路線として、Wayve の [GAIA-1 (運転生成World Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0053_gaia-1_note) は未来運転シーンを動画として生成しますが、未来特徴予測は生成ではなく特徴予測に寄せる点が異なります。生成は見た目がリアルですが重く、特徴予測は軽く運転に有用な抽象に集中できる、という棲み分けです。系譜を一文でまとめると、「ピクセルを描く生成型(World Models / GAIA)」と「特徴で予測する非生成型(JEPA)」の二つの大きな潮流があり、未来特徴予測は後者に世界モデルの行動条件付けを足したハイブリッドだ、と位置づけられます。

## 論文の実験結果(定量データ)

未来特徴予測そのものを直接評価する標準ベンチマークは確立途上ですが、土台となるJEPA系・世界モデル系の定量結果から効果が読み取れます。

I-JEPA(2023)の報告では、ピクセル再構成型の自己教師(MAE=Masked Autoencoder)に比べ、同程度の表現品質をはるかに少ない計算で達成しました。具体的には、ImageNet の線形評価(学習した特徴の上に単純な線形分類器を載せて精度を測る、表現の良さの定番テスト)で、MAEと競合しつつ事前学習を大幅に高速化したと報告されています(同等水準の表現に必要なGPU時間がMAEより数倍少ないオーダー)。これは「ピクセルを描かず特徴を当てる」ことの計算効率上の利点を示します。

V-JEPA(2024)の報告では、動画から特徴空間予測で学んだ表現が、動作認識(Kinetics-400 や Something-Something-v2 など)の下流タスクで、ピクセル予測型や対照学習型の自己教師を上回る、または匹敵する精度を達成しました。とくに「凍結した特徴の上で評価する(frozen evaluation)」設定で強く、学んだ特徴がそのまま追加学習なしで有用であることを示しています。アブレーションでは、ターゲット側のエンコーダの作り方(EMAで安定化するかどうか)が性能を左右し、ターゲット設計の重要性が裏付けられました。

世界モデル系の MILE(2022)では、未来のBEVセマンティクスを内部で想像的に展開でき、その想像がCARLA上のクローズドループ運転計画性能(Driving Score)の向上に寄与することが示されました。これは、運転に意味のある未来を特徴/状態空間で予測する価値の定量的な根拠です。観測を与えず潜在状態だけで複数秒先まで妥当に展開できた点が、内部予測が実際に機能している証拠になっています。

実装上の定量的な注意としては、未来予測をONにすると学習時のメモリと計算が増えるトレードオフが一般に観測されます。フル解像度のBEVで未来4時刻を予測すると、ピークVRAMが数GB単位で増加し、バッチサイズの上限が下がる、という実測傾向は多くの実装で共通します。前節の数値例のとおり、出力テンソルだけで1サンプル数百MBになるためです。

## メリット・トレードオフ・限界

メリットは四つです。第一に、ピクセルでなく特徴を予測するので、運転に有用な抽象的な未来理解が学べ、計算も画像生成より軽い。第二に、プランナーの意図(隠れ状態)で条件付けるので、「この運転をしたらこうなる」という意図と整合した未来予測になります。第三に、行動条件付け(action_dim)で反実仮想予測ができ、世界モデルRLの土台になります。第四に、未来予測は学習時の自己教師にのみ使い、推論時(実走時)には計算しない設計にできるので、実走コスト(レイテンシ・VRAM)をゼロにできます。

トレードオフと限界もあります。フル解像度ではメモリが非常に重く、混合精度やダウンサンプリングが事実上必須で、バッチを大きくできません。予測ターゲット(凍結バックボーンの特徴)の品質と次元整合に性能が左右されます。とくに、ターゲットを作る凍結バックボーンの出力次元と、未来予測モジュールの出力次元が一致しないことがよくあり、間に射影層(線形層で次元を合わせる)を挟む配管が必要になります。たとえば features_only で取り出したバックボーン特徴のチャンネル数と、未来予測の出力チャンネル数 C が食い違うと、そのままではMSEが計算できません。さらに、未来は本質的に不確実(他車の意図は読み切れない)なので、MSEのような平均的な損失だと「あり得る未来の平均」に寄り、稀だが重大な未来(急な飛び出し)を軽視しがちです。

研究上の未解決課題としては、ターゲット特徴の最適な作り方(凍結 vs EMA、どのバックボーンを使うか)、不確実な未来を表現する確率的・多峰的予測の導入、長時間先の予測精度の維持、そして自己教師で学んだ未来予測が実際に下流の運転性能を上げるかという因果関係の検証、が挙げられます。

## 発展トピック・研究の最前線

発展方向の一つは、ターゲットエンコーダをEMA(指数移動平均。予測側の重みをゆっくり追従させてターゲットを安定化)で更新する I-JEPA/V-JEPA 流の高度化です。単純な凍結バックボーンより質の高いターゲットが得られる可能性があります。もう一つは確率的・多峰的な未来予測で、ありうる複数の未来を分布として予測し、平均化のボケを避ける方向です(条件付き拡散モデルや混合密度ネットワークで複数の未来候補を出す)。さらに、未来特徴予測を世界モデルRL([World Modelシミュレーション学習](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0031_world-model-simulation))と統合し、予測した未来から報酬を計算して方針を磨く完全なループを閉じることが、実用化に向けた大きなテーマです。

研究者としての着眼点は、未来予測の質をどう測るか(下流の運転性能との相関、想像ロールアウトの長期安定性)と、予測が確実な領域と不確実な領域を区別して扱う仕組み(不確実性推定)の設計です。特徴空間で予測する以上「正解特徴が本当に運転に必要な情報を保持しているか」という前提そのものを検証する評価指標の整備も、今後の重要な研究課題です。
