# 視覚履歴による動的残差の条件付け

## ひとことで言うと
他の車が次にどこへ動くかを予測するには「速度」が要りますが、写真1枚を見ても速度は分かりません（止まっているのか動いているのか区別できない）。速度は「過去の数フレーム」を見て初めて分かります。視覚履歴による条件付け（visual-history conditioning）とは、過去の視覚的な文脈（過去フレームから作った特徴）を未来予測モジュールに与えることで、動的残差 `Δ_dynamic`（他エージェントの動き）をより当てやすくする工夫です。

## 直感的な理解
一枚の写真を見せられて「この車は次の瞬間どこへ行くか」と問われたら、答えに困るはずです。前に進むのか、止まっているのか、後退するのか、写真1枚からは原理的に分かりません。ところが、同じ車を写した2〜3枚の連続写真を見れば、「右へ動いている」「だんだん減速している」と一目で分かります。

人間にとって当たり前のこの事実は、機械にとっても同じです。未来予測モデルに「他者の動き（動的残差）」を当てさせたいなら、過去の数フレームを見せて速度の手がかりを与えなければなりません。ところが、未来予測モジュールが「いまの1フレーム」だけを入力に取る設計だと、原理的に速度が分からず、他者の動きを当てる課題が解けません。そこで「過去フレームから作った時系列の文脈」を注入してやろう、というのがこのトピックの核心です。

## 基礎: 前提となる概念
言葉を順に噛み砕きます。

- **速度と単一フレームの限界**: 速度（velocity）とは「位置が時間とともにどう変わるか」、すなわち位置の時間微分です。微分は最低2点（2時刻）が無いと定義できません。だから単一フレームから速度は出せません。加速度（acceleration、速度の変化）はさらに1点多く、最低3フレームが要ります。
- **BEV（鳥瞰図）特徴マップ**: 真上から見た地図状の特徴で、たとえば `[B, 256, 450, 300]`（B=バッチ、256=チャンネル、450×300=空間グリッド）のテンソルです（[ego起因の見かけ運動と動的運動の分離](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0023_ego-induced-vs-dynamic-motion)）。
- **動的残差（Δ_dynamic）**: ego起因の見かけ運動をワープで消した後に残る、他エージェントの実運動。未来予測モジュールが学ぶべき対象です。
- **条件付け（conditioning）**: ある計算に外部の情報を「条件」として与え、出力を方向づけることです。たとえば「プランナーの意図」や「アクション」を未来予測モジュールに与えると、その意図に沿った未来が出ます。
- **チャンネルごとのバイアス（per-channel bias）**: 特徴マップの各チャンネル（256次元の各成分）に、空間方向には一様な「下駄（一定値）」を足す条件付けの最も簡単な形です。全マスに同じ値を足すので、空間方向の細かい違いは表現できません。
- **時間メモリ（temporal memory）**: 過去フレームを一定の頻度（たとえば1Hz）でエンコードし、`[B, T, feat]`（T=過去フレーム数）の履歴として保持する仕組みです。これが「視覚履歴」の供給源になります。

未来予測モジュールが現在の融合特徴とプランナーの意図だけを入力にしていると、過去の視覚文脈を一切持ちません。その結果、原理的に速度が見えず、動的残差を当てられません。注意すべきは、ここで使いたいのは「視覚（シーン）の履歴」であって、自車運動（ego-motion）の履歴ではない点です。自車の動きはワープ（[ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp)）が既に扱っているので、履歴側は「周りのシーンがどう動いてきたか」を担当する、という役割分担になります。

## 仕組みを詳しく
この条件付けは、コストと表現力の異なる2段階（tier）で設計するのが定石です。

### Tier 1: グローバルなチャンネルごとのバイアス

- 過去フレームの履歴 `[B, T, feat]` を時間方向に集約（平均やアテンションなど）して、1本の「視覚文脈ベクトル」`visual_ctx`（shape `[B, feat]`）にまとめる。
- それを線形層でBEVのチャンネル数（たとえば256次元）に射影し、`[B, 256, 1, 1]` に reshape する。
- BEV特徴 `[B, 256, 450, 300]` に、このベクトルをチャンネルごとのバイアスとして足し込む（空間方向450×300には同じ値をブロードキャスト、つまり全マスに同じ下駄を履かせる）。

これは、プランナーの意図を注入する一般的なパターンとまったく同じ形です。疑似コードで書くと次のようになります。

```python
bias = visual_proj(visual_ctx).view(B, C, 1, 1)   # [B, 256, 1, 1]
conditioned = fused_features + bias                # 全マスに同じ下駄
```

Tier 1は安価で、既存の条件付け構造にそのまま差し込めます（ドロップイン）。シーン全体の運動傾向、たとえば「全体的に交通が流れている／詰まっている」程度の文脈を伝えられます。ただし、マスごと（per-cell）の細かい運動は伝わりません。全マスに同じ下駄を足すだけなので、「このマスの物体は右へ、あのマスの物体は左へ」という方向別の情報は表現できないのです。

### Tier 2: 空間的なBEV履歴 + ConvGRU

- 過去のBEVマップ（空間構造を保ったまま）を時系列で並べ、ConvGRU（Convolutional GRU、空間構造を保ったまま時系列を処理する再帰ネット）に通す。
- これにより「このマスの物体は右上方向へ動いてきた」というマスごとの運動（per-cell motion）を表現できる。
- 当然、空間×時間ぶんメモリが重くなる。チャンネル方向に圧縮（次元削減）すれば軽くできます。

数値で効果のイメージを示します。前方の車が過去3フレームで毎フレーム5マス（=1秒で1m相当、1マス20cm想定なら）右へ動いてきたとします。Tier 2なら「このマス周辺は右向きの速度がある」と未来予測モジュールに伝わり、残差ネットは未来でもそのぶん右にずらした残差を出せます。Tier 1だと「全体的に動きがある」程度しか伝わらず、どのマスがどちら向きか、までは出せません。

設計の指針として、まずTier 1で「過去文脈を入れる効果があるか」を安価に確かめ、効果が見えてからTier 2で空間的な精緻化に進む、という段階的アプローチが合理的です。Tier 1は既存構造への足し算なので、入れても壊れにくく、効果測定の出発点に向いています。

## 手法の系譜と主要論文
過去文脈で将来運動を条件づける発想は、BEVの将来予測研究と条件付け機構の研究が合流したものです。

- **Ballas et al., "Delving Deeper into Convolutional Networks for Learning Video Representations", ICLR 2016**: ConvGRU（畳み込み版GRU）を動画表現学習に使った代表例です。全結合のGRUと違い、空間構造を保ったまま時間方向の依存を扱えます。Tier 2の道具立てそのものです。フレーム数ぶんメモリと計算が増えるのがコストです。

- **Perez et al., "FiLM: Visual Reasoning with a General Conditioning Layer", AAAI 2018**: 外部の文脈で特徴をチャンネル方向に変調（スケール γ とバイアス β を掛け足しする、`γ ⊙ x + β`）する条件付けの一般形を提案しました。Tier 1のチャンネルバイアス注入は、このFiLMのバイアスのみ版に当たります。安価ですが、空間方向の細かさは出せないトレードオフがあります。

- **Hu et al., "FIERY: Future Instance Prediction in Bird's-Eye View", ICCV 2021**: 過去の複数フレームからBEVを時系列で集約し、未来のインスタンス運動を確率的に予測しました。過去文脈なしには将来運動を当てられない、という前提を実証した重要論文です。時系列集約のメモリと計算コストがトレードオフです。

- **Hu et al., "Planning-oriented Autonomous Driving (UniAD)", CVPR 2023（CVPR 2023 Best Paper）**: 知覚・追跡（tracking）・運動予測（motion forecasting）・占有予測・計画を1つのネットに統合し、いずれも時系列特徴の上で行いました。過去の軌跡履歴から将来を予測する設計で、「時系列文脈が将来予測の質を上げる」という主張の強力な裏付けになっています。

系譜をまとめると、ConvGRU（2016）が「空間構造を保った時系列処理」の道具を、FiLM（2018）が「文脈でチャンネルを変調する」一般形を提供し、FIERY（2021）が「過去文脈からBEVの将来運動を予測する」枠組みを確立し、UniAD（2023）がそれを統合的な運転モデルへ昇華した、という流れです。視覚履歴による条件付けは、この道具と枠組みを未来予測モジュールへ接続するものです。

## 論文の実験結果（定量データ）
FIERY（Hu et al., 2021）は nuScenes（6カメラ＋LiDARの大規模自動運転データセット）で、未来のBEVセマンティックセグメンテーションを評価しました。指標は Future IoU（予測領域と正解領域の重なり、1に近いほど良い）と Future PQ（Panoptic Quality、インスタンスを個体として正しく追えているかも含む指標）です。FIERYは過去複数フレームを集約することで、2秒先予測でおおむね30%台後半のIoUを達成しました。重要なのはアブレーションで、過去フレームの集約を外して単一フレームから将来を予測しようとすると、IoUが大きく低下します。これは「速度情報が単一フレームに無いため将来運動を当てられない」という、本トピックの大前提を定量的に示しています。

UniAD（Hu et al., 2023）は、運動予測の指標 minADE（minimum Average Displacement Error、複数の予測軌道のうち最良のものと正解との平均位置誤差。メートル単位、小さいほど良い）や minFDE（最終地点での誤差）で、過去の追跡履歴を使うことで予測精度が上がることを示しました。さらに、知覚から計画までを一気通貫で学習すると、計画の衝突率（collision rate）が下がるという結果を報告し、CVPR 2023のBest Paperに選ばれました。時系列文脈を上流の予測に効かせると、下流の計画の安全性まで改善する、という波及効果がここで示されています。

条件付け機構そのものについては、FiLM（Perez et al., 2018）が視覚的推論ベンチマーク CLEVR で、チャンネル変調による条件付けが質問内容に応じて画像特徴を柔軟に方向づけられることを示し、高い正答率を達成しました。これは「外部文脈で特徴を変調する」というTier 1の設計が、安価ながら有効であることの一般的な裏付けです。一方、バイアスのみ（空間一様）の条件付けは、空間的に異なる操作が必要な課題では精度が頭打ちになることも知られており、ここがTier 1の限界とTier 2の必要性を示しています。

これらの数値の総合的なメッセージは、「過去文脈は将来運動予測に不可欠であり、その注入は安価なグローバル変調（Tier 1）から始めて、必要なら空間的な精緻化（Tier 2）へ進むのが効率的」ということです。

## メリット・トレードオフ・限界
メリット
- 単一フレームでは観測不能な速度・加速度の手がかりを与え、動的残差の予測精度を上げられます。
- Tier 1は既存の条件付け（プランナー意図の注入など）と同じパターンで、安価かつドロップインで入ります。
- 時間メモリの基盤が既にあれば、新規にメモリ機構を作らずに済みます。

トレードオフ・限界
- Tier 1はグローバルバイアスなので、マスごとの方向別運動は表現できません（全マス同じ下駄）。混雑の有無くらいは伝わっても、誰がどちらへ動くかは伝わりません。
- Tier 2はper-cell運動を扱えますが、空間×時間でメモリが重くなります。高解像度BEV（たとえば450×300）の将来予測はもともと重く、単精度（fp32）では1サンプルあたり数百MB級になることもあるため、半精度（bf16）化、ダウンサンプリング、チャンネル圧縮が前提になります。
- 過去フレームを保持・エンコードするぶん、入力データ量とパイプラインの複雑度が増えます。
- 履歴に含まれるego運動の情報はワープと役割が重複しうるので、「視覚／シーン文脈」に絞る設計判断が要ります。混ぜると二重計上になり、学習が混乱します。

未解決の課題として、過去を何フレーム・どの頻度で保持するのが最適か、Tier 1とTier 2をどう組み合わせるか（粗いグローバル文脈と細かい空間文脈の併用）、遮蔽で過去に見えなかった物の速度をどう扱うか、などが残っています。

## 発展トピック・研究の最前線
近年のBEV知覚は「時間融合（temporal fusion）」を標準装備にしており、過去のBEVを現在の自車座標系へ整列（align）してから融合する設計が広く使われています。代表例として、複数フレームのBEVをアテンションで融合する手法や、過去の特徴を再帰的に繰り越すストリーミング型の設計があります。これらはまさに本トピックのTier 1/Tier 2を高度化したもので、ego整列（自車運動でのワープ）と視覚履歴の融合を組み合わせています。研究の最前線では、長時間の履歴を効率よく圧縮するメモリ機構、遮蔽下での速度推定、そして履歴を使った将来予測を計画・制御へ一気通貫で繋ぐエンドツーエンド設計が活発に探られています。
