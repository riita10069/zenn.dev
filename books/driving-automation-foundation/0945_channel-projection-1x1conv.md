# 1x1 Conv チャネル射影

## ひとことで言うと
1×1 畳み込み(1x1 Conv)は、画像の縦横サイズは一切変えずに、各ピクセルが持つ「数値の本数(チャネル数)」だけを増減させる、とても軽い変換です。マルチスケール特徴を連結して膨らんだチャネル数を、後段が扱いやすい埋め込み次元へ圧縮するときの定番部品です。

## 直感的な理解
たくさんの計器が並んだダッシュボードを想像してください。1440 個のメーターがあって全部を一度に見るのは大変です。そこで「速度計と回転計を組み合わせて1つの新しい指標を作る」「燃料計と航続距離計から1つの実用指標を作る」というように、生のメーターを賢く組み合わせて 256 個の意味のある指標に作り直したいとします。1×1 conv がやっているのはまさにこれで、各ピクセル位置で「たくさんの生チャネルを賢く混ぜ合わせて少数の有用なチャネルに作り直す」操作です。縦横の場所は触らず、各場所のチャネルの組み合わせだけを学習します。

なぜこの操作に独立した名前と役割が与えられているのかというと、現代のネットワークは「空間を混ぜる処理(畳み込みやアテンション)」と「チャネルを混ぜる処理(1×1 conv)」を分けて積み重ねる設計が主流だからです。1×1 conv は後者の純粋な担い手で、空間には一切手を出さず、チャネル方向の情報の組み替えだけに専念します。

## 基礎: 前提となる概念
特徴マップは「縦×横×チャネル」の3次元の数値の塊です。チャネルとは、各ピクセル位置にぶら下がっている数値ベクトルの長さのことです。例えば 64×64×256 なら、64×64 の各位置に長さ256のベクトルがある、という意味です。各チャネルは「縦エッジに反応する特徴」「赤い領域に反応する特徴」のように、それぞれ異なるパターンへの反応の強さを表していると考えられます。

普通の畳み込み(convolution)は、3×3 などの窓を画像上で滑らせ、窓内の複数ピクセルを重み付き和でまとめる操作です。窓が大きいほど隣のピクセルの情報が混ざります。これに対し 1×1 畳み込みは窓が1ピクセルなので、隣のピクセルは一切混ざらず、各位置のチャネル方向だけを変換します。「空間方向の受容野が1」と言い換えてもよいです。

全結合層(Linear)は入力ベクトル全体に重み行列を掛ける操作です。1×1 conv は「各ピクセル位置に対して同一の全結合層を独立に適用する」のと数学的に等価です。つまり空間の各点に同じ小さな変換を共有して掛けます。なぜ全結合でなく畳み込みの形で書くのかというと、特徴マップは縦横の空間構造を持ったままなので、空間を保ちつつ各位置に同じ変換を適用したいからです。重みは全位置で共有されるため、画像サイズが変わってもパラメータ数は変わりません。

埋め込み次元(embed dimension)とは、後段の処理が前提とする固定のチャネル数のことです。Transformer 系の融合・BEV 化・クロスアテンションはたいてい単一の埋め込み次元(例 256 や 512)で動くよう設計されており、入力をその次元に揃える必要があります。複数のソースやスケールの特徴は元のチャネル数がバラバラなので、後段に渡す前に共通の埋め込み次元へ「射影(projection)」する必要があり、その軽量な手段が 1×1 conv です。

## 仕組みを詳しく
1×1 畳み込みがやっているのは「各ピクセル位置で、入力チャネルベクトル(長さ Cin)に重み行列 W(Cout × Cin)を掛けて出力チャネルベクトル(長さ Cout)を作る」ことです。式で書くと、位置 (i,j) について `out[i,j] = W · in[i,j] + b` で、W と b は全位置で共有されます。典型的には直後に活性化関数を挟みます。

```python
channel_proj = nn.Sequential(
    nn.Conv2d(backbone_channels, embed_dim, kernel_size=1),  # 例: 1440 -> 256
    nn.GELU()                                                # 非線形活性化
)
```

tensor shape で追うと(BV はバッチ×ビュー、空間を 8×8、入力 1440・出力 256 とする)次の通りです。

```
入力:  [BV, 1440, 8, 8]
        ↓ 1x1 Conv2d(1440 -> 256)  ← 各 (8,8) 位置で 1440次元ベクトルに W(256x1440) を掛ける
       [BV, 256, 8, 8]
        ↓ GELU                      ← 各要素に非線形を適用
       [BV, 256, 8, 8]
```

重みの大きさは Cout × Cin = 256 × 1440 ≒ 37万パラメータ(プラス bias 256)です。窓が 1×1 なので、同じ入出力チャネルの 3×3 conv と比べて重みは9分の1で済みます。空間方向に窓を持たないぶん、計算もメモリも軽いのが特徴です。計算量(FLOPs)は H×W×Cin×Cout に比例し、上の例だと 8×8×1440×256 ≒ 2360万回の積和で、これは 1 フレームの処理としてはごく軽量です。

GELU(Gaussian Error Linear Unit)は活性化関数で、入力が大きいほどほぼそのまま通し、負の小さい入力は滑らかに弱める非線形です。ReLU(負を0にする)に似ていますが滑らかで、Transformer 系でよく使われます。これを挟むことで、単なる線形圧縮ではなく非線形な特徴変換になります。なお活性化を挟まない素の 1×1 conv も「純粋な線形射影」としてよく使われ、用途によって使い分けます(残差ブロックの次元合わせなどでは活性化なしのことが多い)。

直感的には、1×1 conv は「1440本の生の特徴から、後段に有用な 256 本の組み合わせ特徴を作り直す」役割です。チャネル間の相関を学習する効果があり、単なる次元削減以上の意味を持ちます。なぜこの設計かというと、連結後の高次元特徴をそのまま後段に渡すと重み行列が巨大になり計算・メモリが膨れるうえ、後段が前提とする埋め込み次元に揃わないからです。1×1 conv は軽量にこの両方を解決します。

数値で効果を示すと、仮に後段のアテンションが特徴次元 D に対して D² に比例するメモリを使うとすれば、1440 をそのまま渡すと 1440² ≒ 207万、256 に圧縮すれば 256² ≒ 6.5万で、およそ32倍の差になります。圧縮の有無が後段の重さを大きく左右することが分かります。チャネルを増やす方向(拡張、例 256→1024)にも使え、Transformer の FFN や ResNet の bottleneck では「絞って広げる」「広げて絞る」を 1×1 conv で組み合わせます。

## 手法の系譜と主要論文
Network in Network (Min Lin, Qiang Chen, Shuicheng Yan、ICLR 2014, arXiv:1312.4400)。畳み込みの間に「1×1 畳み込み」を挟む手法(論文では mlpconv と呼ぶ)を提案しました。各位置で小さな多層パーセプトロンを適用する発想です。動機は、普通の畳み込みは線形フィルタなので各位置でのチャネル間の非線形な相互作用を表現しにくいことでした。これにより 1×1 conv が「チャネル次元の変換・圧縮・拡張」を担う基本部品として定着しました。同論文はグローバル平均プーリング(各チャネルの空間平均1値を取る)も提案しており、現代 CNN の土台のひとつです。

GoogLeNet / Inception (Christian Szegedy ら、CVPR 2015, arXiv:1409.4842)。重い 3×3 や 5×5 畳み込みの前に 1×1 conv を置いてチャネルを減らす「次元削減ボトルネック」を導入しました。動機は、チャネルが多いまま大きなカーネルを掛けると計算が爆発することで、先に 1×1 でチャネルを絞れば計算量を大幅に削減できます。「ボトルネックで後段を軽くする」思想は、連結で膨れた特徴を埋め込み次元へ圧縮する用途とまったく同じです。

ResNet (Kaiming He ら、CVPR 2016, arXiv:1512.03385)。bottleneck ブロックで 1×1 conv をチャネルの圧縮(reduce)と復元(restore)の両方に使いました。256 チャネルを 1×1 で 64 に絞り、3×3 を 64 チャネルで行い、再び 1×1 で 256 に戻す構成です。深いネットワークでも計算量を抑えるための標準構成で、1×1 conv が現代 CNN の基本部品であることを象徴します。

MobileNet (Andrew Howard ら、2017, arXiv:1704.04861)。depthwise separable convolution を導入し、空間方向の畳み込み(depthwise)とチャネル方向の 1×1 conv(pointwise)を完全に分離しました。ここで 1×1 conv は「チャネルを混ぜる唯一の場所」という明確な役割を担い、計算量削減の主役になります。

SENet (Squeeze-and-Excitation Networks; Jie Hu ら、CVPR 2018, arXiv:1709.01507)。1×1 conv 的なチャネル変換でチャネルごとの重要度を動的に学習し再重み付けします。チャネル方向の操作が精度向上に直結することを示し、ImageNet 分類で当時の最高性能を更新しました。

系譜としては「1×1 conv の発明(Network in Network) → 計算削減のボトルネックとして活用(Inception/ResNet) → チャネル混合専用部品としての確立(MobileNet) → チャネル重要度の動的制御(SENet) → 任意の次元整合・チャネル混合の汎用部品として定着」と読めます。

## 論文の実験結果(定量データ)
1×1 conv は単体で評価される部品ではなく、それを使ったアーキテクチャの効率改善として効果が現れます。GoogLeNet (Inception v1) は ImageNet(1000 クラスの画像分類ベンチマーク、約128万枚で学習)で、当時の VGG など大型モデルと同等以上の精度を、はるかに少ないパラメータ数と計算量で達成しました。報告では Inception v1 のパラメータはおよそ 500 万から 700 万程度で、同時期の VGG-16(約1億3800万パラメータ)の20分の1以下です。この効率の中心にあるのが 1×1 conv によるボトルネックで、大きなカーネルの前にチャネルを絞ることで計算量(FLOPs)を桁で削減しています。

ResNet の bottleneck ブロックも同様で、1×1 で 256 チャネルを 64 に絞り、3×3 を 64 チャネルで行い、再び 1×1 で 256 に戻す構成により、3×3 を 256 チャネルでそのまま行う場合に比べて計算量を大幅に削減しつつ ImageNet の分類精度(ResNet-152 で top-5 誤り率 4.49%、当時の最高性能)を達成しました。top-5 誤り率とは「正解が予測上位5位以内に入らなかった割合」で、低いほど良い指標です。アブレーション的に見ると、ボトルネックを使わず同じ深さを素朴に積むとパラメータと計算が現実的でない量に膨れ、学習も不安定になります。

SENet は、既存のネットワーク(ResNet など)に 1×1 conv ベースのチャネル再重み付けモジュールを差し込むだけで、ImageNet の top-5 誤り率をさらに下げ(ILSVRC 2017 分類タスクで優勝)、追加計算がごくわずかであることを示しました。これは「チャネル方向にどの特徴を強調するか」を制御することの効果を定量的に裏付けています。

数値の意味を補足すると、パラメータ数や FLOPs が桁で減ることは、同じ計算資源でより深い・より広いネットワークを学習できることを意味し、最終的な精度向上の余地に直結します。1×1 conv は「精度を保ったまま後段を軽くする」ための、目立たないが効果の大きい部品です。

## メリット・トレードオフ・限界
メリットは、空間サイズを変えずチャネルだけを変換でき後段の次元要件に合わせられること、窓が 1×1 なので重みが小さく計算・メモリが軽いこと、チャネル間の相関を学習でき活性化と合わせて非線形な特徴再構成ができることです。重みが全位置共有なので入力解像度に依存せず、可変サイズ入力にもそのまま使えます。

トレードオフと限界として、空間方向の情報は一切混ぜない(隣ピクセルを見ない)ので位置的な文脈づけはできません。それは普通の畳み込みやアテンションの役割です。また大きな圧縮(例 1440→256)は情報の損失を伴い、何を捨てるかは学習任せで人間には解釈しにくいです。各位置に同じ変換を掛けるだけなので、位置ごとに異なる処理はできません(位置依存の処理が要るなら別の機構が必要)。研究上の論点としては、圧縮率をどこまで攻められるか(圧縮しすぎると後段の表現力が頭打ちになる)、そしてチャネル混合をグループ化して軽量化する group convolution / depthwise separable convolution(MobileNet 系)のような効率化との使い分けが挙げられます。

## 発展トピック・研究の最前線
1×1 conv の発想は、軽量化研究で多方向に展開しています。depthwise separable convolution(MobileNet)は空間方向の畳み込みとチャネル方向の 1×1 conv を分離して計算量を激減させます。group convolution / channel shuffle(ShuffleNet)はチャネルをグループに分けて 1×1 conv を軽くしつつ、グループ間の情報をシャッフルで混ぜます。Squeeze-and-Excitation(SENet)は 1×1 conv 的なチャネル変換でチャネルごとの重要度を動的に再重み付けします。Transformer 時代には、各トークンに同じ全結合を掛ける FFN(位置ごとの線形変換)が 1×1 conv とまったく同じ役割を果たしており、CNN と Transformer をまたいで「チャネル方向の点ごとの変換」という共通原理が貫いています。近年の ConvNeXt のような「Transformer の設計を CNN に逆輸入した」モデルでも、チャネル混合は 1×1 conv(=pointwise)が担っており、空間混合とチャネル混合を分離する設計思想は現在の標準になっています。
