# 物体検出の損失関数

## ひとことで言うと

物体検出(画像のどこに何があるかを四角い枠=バウンディングボックスで囲って当てるタスク)を学習させるには、「何があるか」を当てる分類損失と「どこにあるか」を当てる位置回帰損失の2つを組み合わせます。この記事では、その組み合わせ方と、focal loss・GIoU・集合予測(set prediction)といった重要な工夫を、なぜ必要になったかの歴史とともに説明します。

## 直感的な理解

物体検出器が答えるべきことは2つあります。1つは「この枠の中身は犬か、猫か、車か、それとも背景(何もない)か」というクラス分類。もう1つは「枠の位置・大きさ(中心座標 x, y と幅 w, 高さ h)はこれで合っているか」という位置の回帰(連続値の予測)です。

学習では、この2つのズレをそれぞれ損失として測り、足し合わせて全体を最小化します。

```
L = L_分類 + λ * L_位置
```

`λ`(ラムダ)は2つの損失のバランスを取る重み。位置損失は背景には適用しません(背景には枠がないので「どこ」を問えない)。この素朴な枠組みは正しいのですが、実際にやると2つの厄介な問題にぶつかります。それが「クラス不均衡」と「位置の測り方の不整合」で、focal loss と GIoU はそれぞれの解決策です。

## 基礎: 前提となる概念

まず IoU(Intersection over Union)を定義します。予測した枠と正解の枠について、

```
IoU = (重なっている面積) / (合わせた全体の面積)
```

完全一致なら 1.0、全く重ならなければ 0。検出の良し悪しを測る基本指標で、「IoU が 0.5 以上なら正解とみなす」といった使い方をします。

次に評価指標 AP / mAP。AP(Average Precision)は、検出の正確さ(precision=検出したもののうち正しい割合)と網羅性(recall=正解のうち検出できた割合)のトレードオフ曲線の下の面積で、1.0 が完璧。mAP(mean AP)は全クラスの AP の平均。物体検出論文の主役の指標です。COCO データセットでは複数の IoU 閾値(0.5 から 0.95)で平均した厳しい mAP を使います。

回帰損失でよく使う Smooth L1 損失も定義しておきます。予測と正解の差 `d` に対し、

```
|d| が小さいとき: 0.5 * d²     (二乗で滑らか)
|d| が大きいとき: |d| - 0.5    (直線で外れ値に頑健)
```

小さな誤差には敏感に、大きな誤差(外れ値)には鈍感に振る舞う、両者の良いとこ取りの損失です。

## 仕組みを詳しく

### 二段階検出と分類+回帰の基本形(Faster R-CNN)

Faster R-CNN (Ren et al., 2015) は二段階(two-stage)検出器の代表です。まず Region Proposal Network が「物体がありそうな枠の候補」を多数出し(第一段)、次に各候補を分類し位置を微調整します(第二段)。損失は分類にクロスエントロピー(=正解クラスの予測確率の対数の負、確率が高いほど損失が小さい)、位置回帰に Smooth L1 を使い、両者を足します。位置回帰はアンカー(あらかじめ用意した基準枠)からのズレ量を予測する形をとります。

### クラス不均衡と Focal Loss(RetinaNet)

一段階(one-stage)検出器は候補生成と分類を一気にやるので速いのですが、深刻な問題があります。画像のほとんどの領域は「背景(物体なし)」なので、学習サンプルの圧倒的多数が簡単な背景になります。簡単な例が大量にあると、その小さな損失の積み重ねが全体を支配し、肝心の難しい物体例の学習が埋もれてしまう。これがクラス不均衡(class imbalance)です。

Lin et al. (ICCV 2017, arXiv:1708.02002) の Focal Loss はこれを解きました。通常のクロスエントロピー `CE = -log(p_t)`(`p_t` は正解クラスの予測確率)に対し、

```
FL = -(1 - p_t)^γ * log(p_t)
```

`(1 - p_t)^γ` という係数を掛けます。`γ`(ガンマ、典型的に2)は調整パラメータ。直感は、すでに正しく自信を持って分類できている例(`p_t` が1に近い)では `(1 - p_t)^γ` がほぼ0になり損失をほとんど消す、一方まだ自信のない難しい例(`p_t` が小さい)では係数が1に近く損失をほぼそのまま残す、という仕組みです。これにより「簡単な背景の山」に学習が埋もれず、難しい例に集中できます。

### 位置の測り方の不整合と GIoU

Smooth L1 で枠の座標を回帰する素朴な方法には欠陥があります。座標の誤差を小さくしても、それが必ずしも IoU(本当に評価したい重なり)の改善に直結しない。さらに、2つの枠が全く重なっていない場合、IoU は常に 0 になり「どれだけ離れているか」の情報が消えて勾配が出ません(学習が進まない)。

Rezatofighi et al. (CVPR 2019, arXiv:1902.09630) の GIoU(Generalized IoU)はこれを補正します。2つの枠を囲む最小の枠 `C` を考え、

```
GIoU = IoU - (|C \ (A∪B)|) / |C|
```

`|C \ (A∪B)|` は「囲み枠 C のうち、2つの枠 A, B のどちらにも属さない隙間の面積」。重なっていなくても、枠同士が遠いほどこのペナルティが大きくなり、GIoU は -1 から 1 の値を取ります。損失は `1 - GIoU`。これにより重ならない場合でも「近づけよ」という勾配が出ます。後継の DIoU/CIoU(Zheng et al., AAAI 2020)はさらに中心距離やアスペクト比も考慮し収束を速めました。

### 集合予測と二部マッチング(DETR)

従来の検出器は、大量のアンカーや候補枠を出した後に NMS(Non-Maximum Suppression=重複した枠を後処理で間引く手続き)が必須でした。これは手作りの後処理で、調整が面倒です。

Carion et al. の DETR (ECCV 2020, arXiv:2005.12872) は発想を変え、検出を集合予測問題として定式化しました。Transformer が固定個数(例えば100個)の予測を一度に出し、それを正解の集合と一対一で対応づける。この対応づけにハンガリアン法(二部マッチング=N個の予測とM個の正解を、総コスト最小になるよう過不足なくペアにする最適割当アルゴリズム)を使います。マッチング後、ペアになった予測にだけ分類損失と位置損失(L1 と GIoU の組み合わせ)を課します。

これにより「1つの物体には1つの予測だけが対応する」ことが学習で保証され、NMS が不要になりました。損失全体は、二部マッチングのコストと、マッチ後の分類・GIoU・L1 損失の和です。

## 手法の系譜と主要論文

- Ren et al. Faster R-CNN (NeurIPS 2015, arXiv:1506.01497)。分類クロスエントロピー + Smooth L1 回帰の二段階検出の基礎を確立。
- Lin et al. Focal Loss / RetinaNet (ICCV 2017, arXiv:1708.02002)。クラス不均衡を損失関数で解決し、一段階検出を二段階に匹敵させた。
- Rezatofighi et al. GIoU (CVPR 2019, arXiv:1902.09630)。評価指標 IoU と整合した回帰損失。
- Zheng et al. DIoU/CIoU (AAAI 2020, arXiv:1911.08287)。収束を速める改良。
- Carion et al. DETR (ECCV 2020, arXiv:2005.12872)。集合予測と二部マッチングで NMS とアンカーを排除。
- Zhu et al. Deformable DETR (ICLR 2021, arXiv:2010.04159)。DETR の収束の遅さを改善。

自動運転の3D検出(BEV 上で車や歩行者を箱で囲う)でも、これらの損失は形を変えて使われます。分類 + 3Dボックス回帰(7次元: 中心 xyz、寸法 lwh、向き)の組み合わせが基本で、DETR 由来の集合予測は BEV 検出(DETR3D, BEVFormer 等)に広く採用されています。

## 論文の実験結果(定量データ)

Focal Loss / RetinaNet (Lin et al., 2017) は COCO データセットで評価され、RetinaNet が COCO test-dev で mAP 39.1(ResNeXt-101-FPN バックボーン)を達成しました。これは当時の二段階検出器(Faster R-CNN 系)を上回り、かつ一段階検出器の速度を保ったものです。アブレーションでは `γ=0`(=普通のクロスエントロピー)から `γ=2` に変えるだけで AP が大幅に向上し、focal loss の係数そのものが効いていることが示されました。

GIoU (Rezatofighi et al., 2019) は、Faster R-CNN と YOLOv3 の回帰損失を L1/MSE から GIoU 損失に置き換える実験を行い、PASCAL VOC と COCO の両方で mAP が一貫して改善しました。特に高い IoU 閾値(厳しい基準)での AP 向上が顕著で、GIoU が「枠をより正確に合わせる」効果を持つことが定量的に示されています。

DETR (Carion et al., 2020) は COCO で、確立された Faster R-CNN ベースラインと同等の mAP 約42を達成し、NMS やアンカー生成といった手作り部品を完全に排除しました。特に大きな物体での AP は Faster R-CNN を上回り、一方で小さな物体の検出は当初劣るという明確なトレードオフが報告されました。また学習に約500エポックという非常に長い時間を要する欠点も明らかになり、これが Deformable DETR など後続研究の動機になりました。

これらの数値の意味を整理すると、focal loss は「不均衡を損失設計で解く」、GIoU は「評価指標と損失を一致させる」、DETR は「後処理を学習に内包する」という、それぞれ異なる軸での前進だと言えます。

## メリット・トレードオフ・限界

分類 + 回帰の組み合わせという基本形は堅牢で、ほぼ全ての検出器の土台です。focal loss は不均衡に強く一段階検出の実用化を後押ししました。GIoU 系は座標回帰を評価指標に近づけ追加コストもほぼゼロ。DETR の集合予測は後処理ハイパーパラメータ(NMS の閾値など)を消し、パイプラインを簡潔にしました。

トレードオフも明確です。focal loss は `γ` と `α`(クラス重み)の調整が必要で、データセットによって最適値が変わります。GIoU は重ならない枠での勾配を救うものの、枠が完全に内包される配置などでは依然として勾配が弱く、これが DIoU/CIoU の改良動機になりました。DETR は前述のとおり収束が極端に遅く、小物体に弱い。二部マッチングは学習初期に「どの予測がどの正解を担当するか」が不安定で、これが遅さの一因です。

根本的な未解決課題として、損失関数が最適化する量(座標誤差や分類確率)と、最終的に人間が気にする量(衝突を避ける、見逃さない)の間には依然ギャップがあります。自動運転では「遠くの小さな歩行者を1人見逃す」コストが「近くの車の枠が数ピクセルずれる」コストより圧倒的に大きいのに、標準的な損失はこれを均等に扱いがちです。安全性を考慮した損失設計は活発な研究領域です。

## 発展トピック・研究の最前線

近年は分類と位置品質を統合する損失(Generalized Focal Loss, GFL, Li et al., NeurIPS 2020)が現れ、「分類スコアが高くても枠の品質が低い予測」を抑えます。また DETR 系は DINO(Zhang et al., 2022)などで COCO mAP 60超に達し、収束の遅さも大幅に改善されました。

3D・BEV 検出では、不確実性を出力に持たせる確率的検出や、時系列情報を使った検出(複数フレームから動きを見て検出を安定化)が進んでいます。LiDAR とカメラを融合する場合、各モダリティの損失バランスがまた新たなマルチタスク的課題になります。
