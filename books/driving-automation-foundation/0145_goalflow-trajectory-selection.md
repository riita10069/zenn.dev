# 軌道選択モード (nearest / mean)

## ひとことで言うと
プランナーが出したK本の軌道候補に点数を付けた後、「どう1本に決めるか」の2つの代表的なやり方です。nearest は最高得点の1本をそのまま選ぶ(argmax)方式、mean は得点に応じてK本を滑らかに混ぜ合わせる(softmax加重平均)方式です。GoalFlow という論文がこの2モードを明示し、広く参照されています。どちらも学習を必要としません。

## 直感的な理解
複数の候補に点数を付けたら、最終的に車を走らせる軌道は1本に決めなければなりません。ここで決め方に2つの哲学があります。

1つは「いちばん点が高い1本だけを使う」やり方です。最も評価された軌道をそのまま実行するので、直感的で確実です。もう1つは「上位の候補を点数に応じて混ぜる」やり方です。1本に絞らず、良い候補同士の中間を取ります。

レストランで料理を選ぶ場面に例えると、前者は「一番食べたいものを1つ頼む」、後者は「食べたいもの上位3つを少しずつ盛り合わせる」に似ています。後者は良いとこ取りに見えますが、組み合わせ次第では破綻します。前方の障害物を「右から避ける軌道」と「左から避ける軌道」がどちらも高得点だったとき、それらを平均すると「真ん中=障害物に直進」という最悪の合成が生まれます。どちらの選択が良いかは状況に依存し、これが nearest と mean のトレードオフの核心です。

## 基礎: 前提となる概念
まず argmax です。これは「最大値を取るインデックス(番号)を返す」操作です。スコアが [2.0, 1.0, 0.5] なら argmax は 0(最初の要素が最大)を返します。nearest モードはこの argmax で選ばれた候補をそのまま採用します。決定的(deterministic)で、同じスコアなら必ず同じ候補を選びます。

次に softmax です。これは複数の実数(スコア)を「合計が1になる確率のような重み」に変換する関数です。各スコア s_i に対して exp(s_i) / Σ_j exp(s_j) を計算します。高いスコアほど大きな重みになりますが、低いスコアもゼロにはならず少しだけ重みが残ります。mean モードはこの softmax 重みで候補を加重平均します。

そして温度(temperature)です。softmax にはスコアを割る温度パラメータ T を付けられます。softmax(s / T) と書きます。

- 温度が小さい(例 0.1): スコアの差が拡大され、重みが鋭くなって最高得点にほぼ全振りします。mean が nearest に近づきます。
- 温度が大きい(例 10): スコアの差が圧縮され、重みがなだらかになって全候補をほぼ均等に混ぜます。

温度は「選択をどれだけ尖らせるか/なだらかにするか」の連続的なつまみです。この温度付き softmax は、知識蒸留(Hinton et al., NIPS 2014 Workshop)で出力分布の鋭さを制御する道具として広く知られるようになりました。

最後にマルチモーダルという概念です。運転状況はしばしば複数の正しい解を持ちます(右からも左からも避けられる、など)。これを軌道分布のモード(峰)と呼びます。複数モードがあるとき、平均を取ると峰と峰の谷間(どの解でもない無効な軌道)に落ちる危険があります。これがモード平均化(mode averaging)の問題で、mean モードが抱える本質的なリスクです。

## 仕組みを詳しく
入力は合算スコア `combined` [B, K]と候補軌道 `trajectories` [B, K, dim]です(B はバッチサイズ、K は候補数、dim は軌道の次元)。

nearest モード(argmax):

```
best_idx = combined.argmax(dim=1)                      # 各バッチで最高得点の番号 [B]
trajectory = trajectories[arange(B), best_idx]         # その1本 [B, dim]
```

各バッチ要素について、K個のスコアのうち最大のインデックスを取り、そのインデックスの軌道を選びます。出力は選ばれた候補そのものです。

mean モード(softmax加重平均):

```
weights = softmax(combined / temperature, dim=1)       # [B, K]
trajectory = (trajectories * weights.unsqueeze(-1)).sum(dim=1)   # [B, dim]
```

各候補に重みを掛けて足し合わせ、1本の軌道に合成します。

数値例。K=3 の候補スコアが [2.0, 1.0, 0.5] のとき。

- nearest: argmax = 0 なので候補0をそのまま採用。
- mean(温度1.0): softmax([2.0, 1.0, 0.5]) ≈ [0.59, 0.24, 0.17]。最終軌道 = 0.59×候補0 + 0.24×候補1 + 0.17×候補2。候補0寄りだが他も少し混ざる。
- mean(温度0.1): softmax([20, 10, 5]) ≈ [0.9999, ...]。ほぼ候補0だけ。nearest と実質同じ。

ASCII図で整理します。

```
combined [B,K]   trajectories [B,K,dim]
   │
   ├── nearest:  argmax(combined) ──▶ その1本を取り出す
   │
   └── mean:     softmax(combined/T) を重みに ──▶ K本を加重平均で合成
```

選択ロジックは学習可能パラメータを持たず、勾配を流さない(no-grad)文脈で実行できます。座標で持つ軌道なら mean の加重平均はそのまま幾何的な平均になりますが、(加速度, 曲率)など制御入力で持つ軌道を平均する場合は、制御列の平均が必ずしも座標列の平均と一致しない点に注意が要ります。

## 手法の系譜と主要論文
最尤モードを選ぶか期待値(平均)を取るかという論点は、軌道予測の研究で長く扱われてきました。MultiPath(Chai et al., Waymo, CoRL 2019)は、軌道を固定のアンカー(代表的な軌道テンプレート)集合上の確率分布として表し、各アンカーの確率と残差を予測しました。複数モードを明示的に持ち、最尤モードを選ぶ(argmax 相当)か全モードを保持するかを区別できます。Trajectron++(Salzmann et al., Stanford, ECCV 2020)は、CVAE(条件付き変分オートエンコーダ)で軌道分布を潜在変数として表し、サンプリングや最尤推定で出力を取り出しました。これらの研究が共有する教訓は、「平均はモード間で破綻しうるので、安全には最尤モード(nearest 相当)を選ぶ方が無難な場面が多い」ということです。

エンドツーエンドプランニングでこの2モードを明示的に並べたのが GoalFlow(Xing et al., 上海交通大学/Bosch 他, arXiv:2503.05689, 2025)です。フローマッチングで複数軌道を生成・採点し、最終軌道を選ぶ際に「最良候補を選ぶ(nearest)」か「候補を統合する(mean)」かを選択できるようにしました。動機は、生成モデルのマルチモーダルな出力をどう1本に落とすかが最終性能を直接左右するからです。GoalFlow は NAVSIM ベンチマークで当時最高水準の PDMS を達成しました。

温度付き softmax による重み付けは、知識蒸留(Hinton et al., NIPS 2014 Workshop)で広く知られた、出力分布の鋭さ制御の道具です。mean モードの温度はこの考え方をそのまま選択に応用したもので、nearest 寄り〜均等混合まで連続的に振る舞いを変えられます。

## 論文の実験結果(定量データ)
GoalFlow(arXiv:2503.05689)は NAVSIM で PDMS およそ 90 点台前半を報告し、選択戦略を含む設計全体で当時の拡散ベース手法を上回りました。選択モードに関する直接のアブレーションでは、最良候補を選ぶ方式と統合する方式で最終 PDMS が変わり、ベンチマークによって最適なモードが異なることが示されています。一般的傾向として、候補の質が揃っていて似た軌道が多い状況では mean が滑らかさで有利になり、候補が明確に分岐する(右避け/左避け)状況では nearest が安全側になります。

軌道予測の文脈でも、最尤モード選択と期待値の違いは指標に表れます。たとえば minADE(複数予測のうち最も正解に近いものの平均変位誤差)は複数モードを保持する方式で良くなりますが、単一の最終予測を出す場面では最尤モードを選ぶ方が衝突率などの安全指標で安定する、という報告が複数あります。これは、平均が谷間に落ちて物理的に無効な軌道を生むリスクを反映しています。

温度の効果については、温度を下げるほど mean は nearest に漸近し、両者の性能差が縮まることが原理的に分かります。実務では、候補が分岐しうる状況で mean の温度を上げすぎると危険な合成が増えるため、温度を低めに設定して nearest 寄りにするか、最初から nearest を既定にする実装が安全とされます。

## メリット・トレードオフ・限界
メリットは次のとおりです。nearest は決定的で軌道が安定し、最も高評価の候補をそのまま使えて実装も単純です。mean は良い候補を活かして滑らかな軌道を作れ、わずかな差の候補を捨てずに済みます。温度パラメータで mean の振る舞いを nearest 寄り〜均等混合まで連続的に調整できます。どちらも学習不要・0パラメータです。

トレードオフと限界は明確です。mean は性質の異なるモード(右に避ける vs 左に避ける)を平均すると、危険な中間(障害物へ直進)を生む恐れがあります。これがモード平均化問題です。nearest は2位以下を完全に無視するため、僅差で最良候補が外れだった場合に救えません。mean の温度はチューニングが必要なハイパーパラメータです。そして両モードに共通する本質的な限界として、採点(DAC や快適度)の質に完全に依存し、採点が誤れば選択も必ず誤ります。未解決の課題としては、状況に応じて nearest と mean を自動で切り替える(分岐があるときは nearest、似た候補が多いときは mean)適応的選択や、モードを潰さずに代表軌道を選ぶクラスタリングベースの選択(複数モードを保持してから各モードの代表を出す)などが研究されています。

## 発展トピック・研究の最前線
さらに深く学ぶ方向として、選択を学習可能にするアプローチがあります。スコアリング関数を固定の幾何/物理計算ではなく、学習した評価器(value model や critic)に置き換えれば、選択の精度が上がる可能性があります。GoalFlow 自身も完全版では学習済みの走行可能領域分類器やゴール地点語彙を使って採点を高度化しています。また、複数モードを単純な平均で潰さず、まずモードをクラスタリングして各クラスタの代表を選ぶ手法(non-maximum suppression 的な選択)は、mean の破綻と nearest の取りこぼしの中間を狙う有望な方向です。さらに、選択の不確実性を下流に伝える(1本に決め切らず分布のまま制御層へ渡す)研究や、世界モデルで各候補を仮想ロールアウトしてから選ぶ手法も進んでおり、「採点して選ぶ」という枠組み自体がより洗練されつつあります。
