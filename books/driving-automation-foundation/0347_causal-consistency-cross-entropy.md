# 因果整合性損失 (Causal Consistency Cross-Entropy)

## ひとことで言うと
因果整合性損失とは、自動運転モデルに「いまのシーンで最も注意すべき要因は何か(交差点なのか、歩行者なのか、信号なのか…)」を当てさせ、その答えが視覚言語モデル (VLM) の付けた擬似ラベルと合っているかをクロスエントロピーで測る補助の損失です。本命のハンドル・アクセル予測とは別に、モデルに「なぜそう運転するか」という理由(因果)を意識させるための追加学習です。

## 直感的な理解
運転の上手い人間は「止まる」だけでなく「なぜ止まるか」を分かっています。赤信号だから、子どもがいるから、前が詰まっているから。この理由の理解があるから、見たことのない状況でも応用が利きます。

一方、人間の運転ログをただ真似する模倣学習だけのモデルは、「画面の見た目」と「操作」の表面的な対応を覚えるだけで、理由を理解しないまま動くことがあります。多くの場面ではそれで動きますが、見たことのない危険なシーンで脆くなります。「赤信号でブレーキ」ではなく「夕方の赤っぽい画面でブレーキ」を覚えていたら、夕焼けで誤作動しかねません。これは因果混同 (causal confusion) と呼ばれる、模倣学習の根深い問題です(de Haan et al. 2019)。模倣学習は「ブレーキランプの光と減速」のような、相関はあるが因果が逆だったり見かけだけだったりする手がかりに引っかかりやすいのです。

そこで、本命の運転予測に加えて「このシーンで支配的な因果要因は何か」を当てる小さな分類タスクを併走させ、共有される内部表現に「因果を意識した情報」をにじませる。これが因果整合性損失の狙いです。

## 基礎: 前提となる概念

**模倣学習 (imitation learning)**: 人間(専門家)の運転ログ「この画像のときこう操作した」のペアを集め、それを真似るよう学習する手法。教師あり学習として実装され、自動運転の行動予測の基本です([模倣学習 (Behavior Cloning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0560_imitation-learning))。

**因果混同 (causal confusion)**: 入力に「結果」と「原因」の両方が写っているとき、モデルが誤って結果を手がかりにしてしまう現象。古典例は、ダッシュボードのブレーキ表示灯を見せると運転模倣の精度が落ちる(モデルが「灯が点いたから減速する」と覚え、実際の障害物を見なくなる)というもの。因果を意識させる補助損失は、この混同を緩和する一つの方策です。

**長尾(ロングテール)シーン**: 「めったに起きないが起きると重大」なまれな状況。飛び出す歩行者、赤信号での交差点進入リスクなど。走行データ全体では頻度が低いが、安全上は最重要。模倣学習だけだと頻度の高い「ただの直進」ばかり上手になり、まれで重要なシーンの判断が育ちにくいという偏りが生じます。

**System 1 / System 2**: 心理学者 Daniel Kahneman の二重過程理論。System 1 は反射的・高速な処理(いつもの直進)、System 2 は熟考型のゆっくりした処理(危ない交差点で何が支配的かを考える)。因果推論ヘッドは、この System 2 に相当する熟考の小さな部品を併設する発想です。

**補助タスク (auxiliary task)**: 本命タスクとは別の、ついでに学ばせるタスク。直接の目的ではないが、共有される表現の質を高めるために足します。

**VLM (Vision Language Model, 視覚言語モデル)**: 画像を見て言葉で説明できる AI(GPT-4V、LLaVA など)。各シーンに「最も注意すべき要因は?」と尋ね、その答えをラベルに使います。

**擬似ラベル (pseudo-label)**: 人間ではなく機械(ここでは VLM)が付けた仮の正解。安く大量に作れる反面、間違いを含むノイジーなラベルだという前提を忘れてはいけません。

## 仕組みを詳しく

処理は3段階です。

**ステップ1: 因果推論ヘッドが分類する。** プランナ(軌跡を出す部分)が持つ文脈ベクトル `ego_hidden`(形状 `[B, 256]`、B はバッチサイズ、256 はベクトル次元)を入力に、小さな MLP(全結合のネット)が次を出します。

```
reasoning_latent : [B, 256]   # 推論の潜在ベクトル(プランナに戻して条件付けに使える)
decision_logits  : [B, 5]     # 5クラスへの生スコア(softmax 前)
```

5クラスの順序は固定の約束です(クラス定義は設計次第ですが、典型例)。

```
0: intersection  (交差点)
1: pedestrian    (歩行者)
2: traffic_light (信号)
3: obstacle      (障害物)
4: clear         (特に支配的要因なし)
```

**ステップ2: VLM 擬似ラベルとのクロスエントロピーを計算する。** これが因果整合性損失の本体です。

```python
loss = F.cross_entropy(
    decision_logits,                  # [B, 5]
    labels,                           # [B]  VLM が付けた 0..4 の整数
    weight=class_weights,             # 希少クラスを重く(任意)
    label_smoothing=label_smoothing,  # 過信を抑える(任意)
)
```

たとえば `decision_logits[i] = [3.0, 0.0, 0.5, -1.0, 1.0]` で VLM ラベルが `0`(交差点)なら、softmax 後の交差点の確率が1に近づくほど loss が小さくなります。中身は [クロスエントロピー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0352_cross-entropy-loss) を参照。

**ステップ3: 本命の損失に小さく足す。**

```
total = imitation_loss + λ * causal_consistency_loss
```

λ(重み)は小さめに設定します。擬似ラベルはノイジーなので、ここに学習を引っ張られすぎると本命の運転性能が落ちます。あくまで「補助」「整合性チェック」として効かせるのが要点です。

**設計上の重要な約束: 勾配を上流まで流す。** このヘッドの勾配は `ego_hidden` を作る上流(トランク=共有の特徴抽出部)まで戻る必要があります。途中に `torch.no_grad` を置いてはいけません。理由は、ヘッド単体が賢くなっても意味がなく、共有表現そのものを「因果を意識した表現」に育てたいからです。勾配が上流に届くからこそ、補助タスクが本体の特徴を形作れます。これがいわゆるカスケード設計(補助タスクの効果が本体に流れ落ちる)です。逆に言えば、ヘッドを `detach` した入力で学習させると、補助損失は「飾り」になり、本命表現に何の影響も与えません。

**擬似ラベルを soft target にする発展形**: 単一クラス(hard label)ではなく、VLM が出す確率分布そのものを目標にすると、`F.cross_entropy(logits, soft_target)` ではなく `−Σ soft_target · log_softmax(logits)` の形で「VLM の自信のなさ」も学習に反映できます。ノイズの多い擬似ラベルでは、soft target のほうが過学習を防げることがあります([知識蒸留損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0633_knowledge-distillation-loss) と同じ原理)。

## 手法の系譜と主要論文

この損失は複数の研究系統を1つにまとめたものと理解できます。

**Kahneman, "Thinking, Fast and Slow" (2011)** — System 1 / System 2 の二重過程理論。反射的処理と熟考的処理を分けるという発想が、「速い軌跡生成」と「遅い因果推論ヘッド」の役割分担の根拠です。近年の自動運転でも「fast-slow」「dual-process」アーキテクチャとして繰り返し参照されます。

**de Haan et al. (NeurIPS 2019, arXiv:1905.11979)** — "Causal Confusion in Imitation Learning"。模倣学習が「結果」を「原因」と取り違える現象を明確に定式化し、入力の手がかりが多いほど(=情報が増えるほど)模倣性能がかえって悪化し得るという逆説を示しました。因果を明示的に学ばせる補助信号の必要性を理論的に裏づける、この分野の基礎文献です。

**Jaderberg et al. (ICLR 2017, arXiv:1611.05397)** — "Reinforcement Learning with Unsupervised Auxiliary Tasks" (UNREAL)。本命タスクに補助予測タスク(ピクセル変化予測・報酬予測など)を足すと、共有表現が良くなり本命性能が上がることを示しました。提案は補助タスク追加、動機は表現学習の強化、効果は学習効率と性能の向上、トレードオフは補助タスクの設計次第で逆効果にもなること。因果整合性損失はこの「補助タスクで共有表現を育てる」系統の自動運転版です。

**DriveGPT4 (Xu et al. 2023, arXiv:2310.01412)** ほか VLM 系: 視覚言語モデルにシーンを説明させ、その出力を運転の理由付けや教師信号に使う研究系統。Wayve の LINGO-1/2 も同系統で「運転中に何を見てなぜそう動くか」を言語化します。提案は人手アノテーションの代わりに VLM で安く大量にラベルを作ること、効果はスケール、トレードオフは生成ラベルのノイズと事実誤り(hallucination)です。だからこそ平滑化や低い損失重みでの利用が推奨されます。

系譜としては「二重過程理論(認知科学)→ 因果混同の定式化(模倣学習)→ 補助タスク学習(深層強化学習)→ VLM 擬似ラベル(基盤モデル時代)」という流れの合流点に位置づけられます。

## 論文の実験結果(定量データ)

因果整合性損失そのものは個別論文の標準ベンチマークを持ちませんが、構成要素の効果は各系統で報告されています。

**因果混同の定量化 (de Haan et al. 2019)**: 著者らは、運転模倣のベンチマークで「過去の自分の行動を示す手がかり(前フレームの操作など)を入力に足すと、模倣性能が悪化する」ことを示しました。情報が増えれば性能が上がるはず、という直感に反する結果で、モデルが本物の原因(障害物・信号)ではなく相関する手がかりに依存していた証拠です。介入(intervention)を使って正しい因果構造を選ぶと性能が回復することも示し、因果を意識させる学習の価値を定量的に裏づけました。

**補助タスクの効果 (UNREAL, Jaderberg et al. 2017)**: DeepMind Lab の3D迷路タスクで、補助タスク(ピクセル制御・報酬予測)を加えた UNREAL は、ベースの A3C に対して最終性能と学習速度の両方を大きく改善したと報告されています。報告によれば、学習に必要なデータ量(サンプル効率)が大幅に改善したケースもあり、平均性能も向上。意味するところは「直接の報酬と無関係な予測タスクでも、共有表現を鍛えれば本命が伸びる」ということで、因果整合性損失の理論的支柱です。

**VLM 擬似ラベルの効果と限界**: VLM による説明・タグ付けは、人手では届かないスケールでデータに意味注釈を付けられる一方、シーンによっては事実と異なる説明を自信満々に出す(hallucination)ことが繰り返し報告されています。このため、擬似ラベルを教師に使う研究では「ラベルはノイジー」という前提のもと、label smoothing や信頼度フィルタ(VLM の確信度が低いサンプルを除外)、低い損失重みが定石になっています。

これらを踏まえた設計上の含意: 因果整合性損失は λ を小さく、label_smoothing で過信を抑え、class_weights で希少クラスを補正する、という「ノイズと不均衡を同時に受け止める」構成になっています。アブレーション的に言えば、λ を上げすぎると本命の運転模倣性能(軌跡誤差)が悪化し、補助の意味が損なわれる、というのが想定される失敗モードです。逆に λ をゼロにすると因果の手がかりが表現に残らず、長尾シーンでの汎化が落ちると予想されます。

## メリット・トレードオフ・限界

メリット
- 本命の運転模倣に「因果を意識する」補助信号を足し、長尾シーンでの判断材料になる表現を育てられる。因果混同の緩和に寄与する。
- 任意・非破壊(本体の forward や返り値契約を変えない)に設計でき、不要なら外せる。
- 人手アノテーション不要(VLM 擬似ラベル)で大量のシーンに適用できる。
- クロスエントロピーに平滑化・クラス重みを足せる設計で、ノイズと不均衡に同時に対処できる。
- 生 logits をそのまま下流に渡すのではなく、信頼度(softmax 最大値)と由来(provenance)を添えた構造体として渡す設計にすれば、下流が「この理由付けをどれだけ信じるか」を判断できる。

トレードオフ・限界
- 擬似ラベルがノイジーで間違いを含む。重み λ を上げすぎると本命性能を損なう。VLM の偏りがそのまま教師信号の偏りになる。
- 5クラスへの粗い分類なので、現実の複雑な因果(複数要因が同時に効く、など)を取りこぼす。
- 補助タスクと本命タスクの勾配が干渉(衝突)し得る。両者の勾配方向が逆を向くと学習が不安定になる。マルチタスク学習の勾配整合(PCGrad など)が必要になる場合がある。
- 補助タスクが本当に本命性能を上げるかはデータ・λ 依存で、丁寧なアブレーション検証が要る。
- 未解決の課題: 「支配的因果要因」という単一ラベルでシーンを表すこと自体の限界。マルチラベル化、連続的な注意度、説明文(自由言語)による教師など、より豊かな因果表現が研究方向。

## 発展トピック・研究の最前線
- Chain-of-thought 風の運転推論: VLM に「観察→理由→行動」の連鎖を生成させ、それを教師や中間表現に使う流れ。
- 信頼度を考慮した擬似ラベル学習: VLM の確信度を重みにする、または不確実なラベルを soft target として扱う手法。
- マルチモーダル因果: 単一クラスでなく複数要因の同時表現、シーングラフによる関係の明示化。
- 因果介入による頑健化: de Haan et al. の流儀で、訓練中に手がかりを意図的に除去・改変し、見かけの相関に依存しない表現を強制するアプローチ。
