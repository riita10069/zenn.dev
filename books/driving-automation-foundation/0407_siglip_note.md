# SigLIP

## ひとことで言うと
SigLIP(Sigmoid Loss for Language Image Pre-training)は [CLIP 画像エンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0372_clip-image-encoder) の改良版です。CLIP は画像とテキストを対応付ける学習で「バッチの中のどれが正しいペアか」を softmax(全候補の中から1つを選ぶ確率計算)で当てさせていました。SigLIP はこれを「各ペアごとに、正しいペアか否かを独立に Yes/No で判定する」sigmoid 損失に置き換えます。この一見小さな変更で、巨大バッチが要らなくなり、小〜中規模のバッチでも安定して高精度に学習できるようになりました。資源効率が大きく改善し、公開された画像エンコーダは近年の多くのオープン VLM(視覚言語モデル)で CLIP に代わって採用が進んでいます。

## 直感的な理解
クラスの席替えを想像してください。CLIP のやり方は「各生徒(画像)に対し、クラス全員(バッチ内の全文)の中から相棒(正しい文)を1人選ばせる多択クイズ」です。全員を見渡さないと「誰が一番相棒らしいか」の確率が決まりません。だから比較相手を増やす=クラスの人数(バッチ)を増やすしかなく、人数が少ないと「数人の中から選ぶだけ」で問題が易しくなってしまいます。さらに「全員の答案を集めて正規化する」という作業が、人数が増えるほど集計係(分散学習の通信)を圧迫します。

SigLIP のやり方は「各ペアごとに、この2人は相棒か?を○×で答えさせる」ものです。1組ずつ独立に判定するので、クラス全員を一度に見渡す必要がありません。人数が少なくても各○×問題の難しさは変わらず、判定は意味を保ちます。これが「巨大バッチが要らなくなる」直感です。集計係も「全員を集めて1回で正規化」する必要がなくなり、組ごとにバラバラに採点できるので通信が軽くなります。

## 基礎: 前提となる概念

- 対照学習と CLIP: [CLIP 画像エンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0372_clip-image-encoder) は N 組の(画像, 文)を1バッチにまとめ、NxN のコサイン類似度行列を作り、softmax で「各画像にとって正しい文はどれか」を当てさせます。
- softmax: 複数候補のスコアを確率に変換する関数。分母に全候補のスコア合計(指数和)を含むため、ひとつの確率を出すのにバッチ全体の情報が必要です。これが「全候補を集めないと損失が決まらない」性質の根源。
- sigmoid: 1つのスコアを 0〜1 の確率に変換する関数。他の候補に依存せず単独で計算できます。二値分類(Yes/No)に使います。
- 負例数とバッチサイズ: 対照学習では負例が多いほど精度が上がります。CLIP では1画像あたりの負例数 = バッチサイズ−1。だから精度を上げるには巨大バッチ(CLIP は 32768)が必要でした。
- all-gather(全収集通信): 分散学習で複数の機材(GPU/TPU)が持つベクトルを全機材に集める通信処理。softmax の正規化に必要で、機材が増えるほど通信量とメモリが重くなります。
- クラス不均衡(class imbalance): 正例と負例の数が大きく偏ること。NxN の行列では正例は対角の N 個だけ、負例は N(N−1) 個で圧倒的多数になります。

巨大バッチは(1)大量の GPU/TPU メモリと多数の機材を要し、(2)分散学習で全ベクトルを集める all-gather 通信が重く、(3)小規模チームには手が出にくい、という問題を生みます。SigLIP は「損失の形を変えれば、この巨大バッチ依存を断ち切れる」と気づいた研究です。

## 仕組みを詳しく

### softmax 損失 vs sigmoid 損失

CLIP(softmax)の考え方:
- 1枚の画像に対し「バッチ内 N 個の文のどれが正解か」を1つ選ぶ多クラス分類。
- 確率はバッチ内全文の類似度で正規化(=分母に全て入る)。バッチ全体を見ないと損失が決まらない。
- 行方向(画像→文)と列方向(文→画像)の2方向で計算するため、行・列それぞれで別々の正規化が必要。

SigLIP(sigmoid)の考え方:
- すべての(画像 i, 文 j)の組を独立に「対応するペアか?(Yes/No)」の二値分類にする。
- 対角(i=j、本物のペア)は Yes(ラベル +1)、非対角(i≠j、別物)は No(ラベル −1)。
- 各ペアに sigmoid を使った二値ロジスティック損失を計算し合計する。正規化が不要なので方向の区別も消える(対称的)。

1ペアあたりの損失:

```
L_ij = log( 1 + exp( z_ij · ( −t' · sim(I_i, T_j) + b ) ) )
```

- sim はコサイン類似度。z_ij は正例なら +1、負例なら −1。
- t' は温度に相当する学習可能スケール、b は学習可能バイアス。
- 全体損失は全ペアの平均: (1/N) Σ_i Σ_j L_ij。

各ペアの損失が他ペアに依存しないことがポイントです。softmax の「全候補で正規化」が消えるので、損失を計算するのにバッチ全体を一度に集める必要がありません。

### バイアス項 b はなぜ要るか

NxN ペアのうち正例は対角の N 個だけで、負例は N(N−1) 個と圧倒的多数です(N=1000 なら正例1000 対 負例約100万)。素朴な sigmoid 損失だと学習初期に「全部 No」と答えるだけで損失がほぼ下がってしまい、正例の信号が大量の負例の損失に埋もれます。SigLIP は学習可能バイアス b を大きな負の値(例 −10)で初期化し、初期に sigmoid の出力を「No 寄り」に補正しておくことで、この極端な不均衡で損失が暴れる(正例側の勾配が消える)のを防ぎます。これは焦点損失(focal loss; Lin et al., ICCV 2017。物体検出で多数の背景と少数の物体の不均衡に対処)などクラス不均衡対策と同じ思想で、「最初から不均衡を前提に出力分布を寄せておく」工夫です。温度 t' も log スケールで初期化され、両者とも学習で適切な値へ動きます。

### なぜ小バッチでも安定するか

- 損失がペア単位で独立 → バッチが小さくても各損失が意味を保つ(○×問題の難しさはバッチ人数に依らない)。
- 分散学習で「全ベクトルを集めて巨大行列を一度に作る」必要が薄れ、メモリ・通信が軽い。SigLIP は機材間でテキスト表現を少しずつ持ち回り(chunked / シャッフル方式)、各機材が自分の担当ペアだけ局所的に損失を計算する効率的実装を提案しました。これにより N² の計算を機材数で分割でき、メモリ上に巨大な NxN 行列を一度に置かずに済みます。
- バイアス項 b で初期の正負不均衡に頑健。

### 出てくるもの

SigLIP も CLIP 同様、画像エンコーダ(ViT)とテキストエンコーダを持ち、学習後は画像エンコーダを汎用視覚バックボーンとして取り出せます。公開された SigLIP / SigLIP 2 の画像エンコーダは、近年の多くのオープン VLM で CLIP に代わって採用されており、VLM の「目」のデファクトの一つになりつつあります。

## 手法の系譜と主要論文

### SigLIP — Zhai et al., ICCV 2023「Sigmoid Loss for Language Image Pre-Training」

著者は Google DeepMind(Xiaohua Zhai ら)。softmax 対照損失を sigmoid 二値損失に置き換え、学習可能な温度 t' とバイアス b を導入。動機は CLIP の巨大バッチ依存・all-gather 依存を取り除き、小〜中バッチでも安定・高精度に学習すること。新規性は「対照学習の損失をペア独立な二値問題に再定式化した」点で、これにより負例数とバッチサイズの結びつきが緩み、分散効率も改善しました。CLIP の系譜の中で「規模で押す」のとは別の「効率で勝つ」分岐を切り開きました。

### SigLIP 2 — Tschannen et al., 2025「Multilingual Vision-Language Encoders ...」

SigLIP に複数の改良を統合した後継。(1)自己教師あり補助目標(SILC/TIPS 由来の自己蒸留・マスク予測)を追加し局所特徴(密な空間表現)を強化、(2)多言語データで学習し言語カバレッジを拡大、(3)解像度可変対応(NaFlex により異なる解像度・アスペクト比を1モデルで扱う)、(4)キャプション生成(decoder)の補助損失。動機はゼロショット分類だけでなく、密予測(セグメンテーション・位置特定)や多言語・多解像度への汎用性を高めること。VLM の視覚エンコーダとして接続したときの下流性能を意識した設計になっています。

### 位置づけ

SigLIP は [CLIP 画像エンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0372_clip-image-encoder) の系譜の中で「損失の形を改良して資源効率を上げた」分岐に当たります。CLIP が「規模で押す」方向(ALIGN, OpenCLIP のスケーリング則)を示したのに対し、SigLIP は「同じ規模でもっと効率良く、あるいは小規模でも CLIP に並ぶ」方向を示し、両者は補完的です。実務的には、損失を sigmoid に替えるだけで CLIP のレシピの大半をそのまま流用できるため、移行コストが低いことも普及を後押ししました。

## 論文の実験結果(定量データ)

中心指標はゼロショット ImageNet 分類 Top-1 精度です(学習に ImageNet ラベルを使わず、クラス名の文との類似度で分類)。

- バッチサイズ感度: CLIP(softmax)はバッチを小さくすると精度が顕著に落ちますが、SigLIP(sigmoid)は小バッチでの低下が小さく、特に小バッチ域(数千以下)では SigLIP が CLIP を上回ると報告されました。一方で両者とも巨大バッチで精度が頭打ちになり、SigLIP の検証では性能は約 32k 付近で飽和し、それ以上バッチを増やしても改善はわずかでした。これは「対照学習に青天井の巨大バッチが必須」という従来の前提を覆し、「適度なバッチで十分」という実務的指針を与えました。なお論文は極端な検証として 100 万規模のバッチまで試し、それでも 32k からの改善が小さいことを確認しています。
- 効率: SigLIP は限られた機材で短時間に強いモデルを学習できることを示しました。論文では比較的少数の TPUv4 チップで数日規模という現実的な計算量で CLIP 同等以上のゼロショット精度に到達した例が報告されています(具体値は構成により異なるため、報告によると「同等精度をより少ない計算・機材で」というのが要点です)。
- 精度水準: SigLIP の代表的な ViT モデル(SigLIP-L 等)はゼロショット ImageNet で CLIP 同等以上(80% 前後の構成も)を達成し、画像→文・文→画像の検索(retrieval)タスクでも高い性能を示しました。
- SigLIP 2: ゼロショット分類で SigLIP / 同規模 CLIP を上回り、加えてセマンティックセグメンテーション・深度推定など密予測、および多言語検索でも改善。VLM の視覚エンコーダとして接続したときの下流性能(視覚的質問応答 VQA など)でも CLIP 系より良い結果が報告されています。

アブレーションの要点。(a)バイアス項 b を外す、または不適切に初期化すると学習初期が不安定化し最終精度が下がる。正負不均衡の補正が効くことを示します。(b)温度 t' を固定値ではなく学習可能にすると精度が上がる。(c)sigmoid と softmax を同条件で比較すると、巨大バッチでは同等、小〜中バッチでは sigmoid が優位、という一貫した傾向が得られました。この「どこで優位が出るか」を明示したのが SigLIP の実務的価値です。

## メリット・トレードオフ・限界

メリット:
- sigmoid 損失により巨大バッチが不要。小〜中バッチでも安定・高精度。
- 分散学習の all-gather・メモリ・通信負荷が軽く、少ない機材で学習可能。資源が限られる環境に向く。
- バイアス項により学習初期の正負不均衡に頑健。
- 公開画像エンコーダは VLM の視覚側基盤として CLIP に置き換わりつつある。CLIP のレシピからの移行コストが低い。

トレードオフと限界:
- 温度 t'・バイアス b など調整項が増える。初期化を誤ると不安定化しうる。
- 対照学習一般の限界(データ品質・拡張設計への依存、細かい数え上げの苦手さ、専門概念の弱さ、構成性の弱さ)は CLIP と同様に残る。
- 損失をペア独立にした分、「バッチ全体での相対比較(全候補を見渡したうえでの順位付け)」がもたらす情報は弱まる。巨大バッチで CLIP が追いつく領域では明確な優位はない。
- 事前学習自体は依然として大規模データを要する(利用は公開重みで容易)。
- 単純なキャプション学習由来の弱い局所性(画素単位の密な表現)は SigLIP 1 でも残り、SigLIP 2 が補助目標で改善したものの、密予測の最前線では専用手法に及ばない場合がある。

## 発展トピック・研究の最前線

SigLIP は「対照損失の再設計で効率を上げる」流れの代表で、その後 SigLIP 2 が自己教師あり補助目標・多言語・可変解像度を統合しました。視覚言語事前学習の最前線では、(1)損失と目標の多重化(対照+生成+自己蒸留のハイブリッド、EVA-CLIP などとの比較)、(2)データキュレーション(DataComp 等で「どのデータを選ぶか」が規模以上に精度を左右することが定量化された)、(3)密特徴の質の向上(セグメンテーション・対応点・位置特定)、(4)VLM の視覚エンコーダとしての最適化(言語モデルへの接続を前提にした学習や解像度設計)が活発です。実務的には「巨大バッチを避けたい」「限られた GPU で視覚言語特徴を得たい」「VLM の目として接続したい」場合、SigLIP 系が第一候補になりつつあります。未解決課題は CLIP と共通で、細かい計数・専門ドメイン・構成性・バイアスの扱い、そして密予測性能の底上げです。
