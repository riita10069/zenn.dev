# VideoBEV型 長期再帰融合

## ひとことで言うと

カメラ映像から鳥瞰図(真上から見た地図のような特徴表現)を作る3D知覚で、過去フレームの情報をどう長く覚えるかは難問です。「1つ前の鳥瞰図だけを覚える」単純な再帰だと、古い情報が世代を経るごとに薄れて消えます。長期再帰融合とは、「時間埋め込み(temporal embedding)」という目印を使い、たくさんのフレームにわたる長期の文脈を、メモリを爆発させずに保ち続ける手法です。再帰のメモリ効率(常に一定メモリ)と、長期保持(古い情報が消えない)の両取りを狙います。並列にN枚積む方式と同等の精度を、メモリ一定で達成することがゴールです。

## 直感的な理解

時系列の情報を覚える方法には、大きく2つの極があります。

ひとつは「並列(parallel)」方式。過去Nフレームを全部そのまま保存しておき、現在のフレームを作るときに全部をまとめて参照します。情報はすべて生で残るので強力ですが、覚えるフレーム数Nに比例してメモリと計算が増えます。N=16 にすればメモリも計算も16倍で、すぐ破綻します。

もうひとつは「再帰(recurrent)」方式。「過去をまとめた1つの記憶」だけを持ち、現在フレームが来るたびにその記憶を少しずつ更新します。覚えるフレーム数に関係なくメモリは一定で、とても効率的です。ところが弱点があります。記憶は毎フレーム新しい情報と混ぜ直されるため、古い情報がだんだん薄まって消えていくのです。

この希釈は「伝言ゲーム」とそっくりです。最初の人が言った言葉は、何人もを経由するうちに別物になります。10フレーム前に見た歩行者の情報は、1フレームごとに新情報と混ざり、10回も混ぜ直されると元の姿はほぼ残りません。5秒間ずっと陰にいた歩行者や、ゆっくりした動きの傾向を覚えたいとき、この希釈は致命的です。古典的なRNNが「勾配消失」で長期依存を学べなかったのと同じ構造の問題が、BEV記憶の世界でも起きます。

長期再帰融合は「再帰の効率(メモリ一定)を保ったまま、希釈を防いで長期情報を保つ」という、両極のいいとこ取りを目指します。鍵は、記憶に混ぜ込む情報に「これは何フレーム前のものか」という時間の目印を付けることです。

## 基礎: 前提となる概念

- BEV(Bird's-Eye-View, 鳥瞰図): 真上から見下ろした地図のような表現です。カメラ画像の特徴を「地面の上のどこか」へ投影してできる、`[チャネル, 高さ, 幅]` の格子状の特徴マップです。3D知覚や経路計画はこのBEV上で行うと扱いやすくなります。

- 時間融合(temporal fusion): 過去フレームの情報を現在の予測に持ち込む処理全般です。1枚の画像では分からない「動き」「一時的な隠れ」を補うために使います。

- 再帰(recurrent)とメモリ一定: 「過去の要約 h を1つだけ持ち、新フレーム x が来たら h ← f(h, x) と更新する」形です。RNN(再帰型ニューラルネット)と同じ構造で、何ステップ回しても h のサイズは変わらないのでメモリは一定です。画像格子に対する再帰の古典は ConvLSTM(Shi et al., 2015)で、隠れ状態を空間マップとして持ちます。

- 並列(parallel)とメモリ増大: 過去Nフレームを生のまま保持し、attention などでまとめて参照します。情報損失はないがメモリ・計算が N に比例します。BEVDet4D の多フレーム concat や SOLOFusion の長期コストボリュームがこの系です。

- 自車運動補償(ego-motion alignment): 過去の記憶BEVは過去の自車位置を中心に描かれているので、現在の自車中心座標へ回転・平行移動で揃えてから融合します。これをしないと同じ静止物が地図上の別の場所に二重に現れます。詳細は [自車運動補償（ego-motion alignment）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0266_ego-motion-compensation)。

- 時間埋め込み(temporal embedding): 「この情報は何フレーム前/何秒前のものか」を表すベクトルです。位置埋め込みが「空間のどこか」を表すのに対し、時間埋め込みは「時間軸のどこか」を表します。これを情報に付けておくと、モデルは「古い情報」を「古いという目印付き」で区別して扱え、新情報に単純上書きされて消えるのを防げます。

## 仕組みを詳しく

長期再帰融合の骨格を段階で示します。

ステップ1: 再帰的なメモリ更新。1フレームごとに「過去をまとめた記憶BEV」を更新していきます。形は例えば `[B, C, H, W] = [4, 256, 200, 200]` 程度で、これがフレームをまたいで一定サイズのまま保たれます。覚えるフレーム数に関係なくメモリが一定、という再帰の利点をここで確保します。並列方式なら N=16 でこのテンソルが16枚必要ですが、再帰なら常に1枚です。

ステップ2: 自車運動整合。記憶BEVを現在の自車中心座標へ座標変換(回転 + 平行移動 + grid_sample による補間)してから現フレームと融合します。これは長期になるほど誤差が積もりやすく、整合の精度が結果を左右します。

ステップ3: 時間埋め込みの付与。ここが核心です。記憶を更新するとき、混ぜ込む各情報に「何フレーム前由来か(または何秒前か)」を表す時間埋め込みを足します。これにより、古い情報も「古いという目印付き」で保持され、新情報による単純な上書き・希釈を受けにくくなります。モデルは「5秒前のこの特徴」と「0.5秒前のこの特徴」を取り違えずに使い分けられます。

```
単純再帰:  h ← f(h, x_t)                    古い情報は混ぜ直しで薄れる
長期再帰:  h ← f(h ⊕ TE(過去), x_t ⊕ TE(now)) 時間目印で古さを区別して保持
            (TE = temporal embedding, ⊕ は足し込み)
```

時間埋め込みを入れることで、融合モジュール(attention や畳み込み)は「これは古い情報だから重みを調整する」「これは新しいから優先する」といった、時間軸を意識した混ぜ方を学習できます。希釈が起きる根本原因は「全フレームを区別なく等価に混ぜる」ことなので、区別の手がかりを与えるのが処方箋です。

ステップ4: 長期文脈の活用。結果として、多数フレーム(長い時間窓)にわたる文脈を、メモリ一定のまま下流タスクへ渡せます。長い隠れの追跡や、ゆっくりした動きの傾向把握に強くなります。

希釈の比較を概念図で示します。

```
単純再帰(時間埋め込みなし):
  情報の強さ
   100│■
      │ ■
      │  ▪
      │   ·            10フレーム後にはほぼ消える
      └──────────────> フレーム

長期再帰(時間埋め込みあり):
   100│■  ■  ■  ■      時間目印付きで長く保持される
      │
      └──────────────> フレーム
```

数値例: ある歩行者が3秒間(0.5秒間隔なら約6フレーム)車の陰に隠れたとします。単純再帰では6回混ぜ直すうちに歩行者の記憶が薄れ、再出現時に「新しい物体」と誤認しやすい。長期再帰は「6フレーム前に見た歩行者」という時間目印付きで保持し、再出現を「同じ歩行者の継続」として扱いやすくなります。

## 手法の系譜と主要論文

- ConvLSTM (Shi et al., NeurIPS 2015, arXiv:1506.04214): 空間マップを隠れ状態として持つ再帰の古典。降水ナウキャスティングのために提案されましたが、「空間構造を保ったまま時間方向に再帰する」枠組みはBEV記憶の再帰更新の概念的な祖先です。

- BEVDet4D (Huang & Huang, arXiv:2203.17054, 2022): 現在と直前1フレームのBEVを ego-motion で揃えてチャネル方向に結合(concat)し、畳み込みで融合する軽量な2フレーム手法。時系列でBEVを揃えて速度推定を大きく改善できることを示し、後続の起点になりました。詳細は [BEVDet4D型 特徴キュー連結](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0480_bevdet4d-queue-concat)。

- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270): 直前フレームのBEVを保持し、現在のBEVクエリと temporal self-attention で再帰的に融合します。「1つ前のBEVだけを覚える単純再帰」の代表で、長期再帰融合が改善の対象とする基準です。詳細は [BEVFormer型 再帰的BEV融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0481_bevformer-recurrent-fusion)。

- BEVFormer v2 (Yang et al., CVPR 2023, arXiv:2211.10439): BEVFormerを多フレームへ拡張し、過去BEVを複数並べる(並列寄りの)時間融合を採用しました。性能は伸びる一方、保持フレーム数ぶんのコストがかかり、「並列の限界」を示す対照例になります。

- SOLOFusion (Park et al., ICLR 2023, arXiv:2210.02443): 「長期観測を使う」問題意識を共有する別アプローチ。長期フレームを並列に保持してコストボリューム(深度を当てる照合表)を作ります。VideoBEVが「再帰で長期」を狙うのに対し、SOLOFusionは「並列で長期」を狙う対比になります。詳細は [SOLOFusion型 時間ステレオ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0500_solofusion-temporal-stereo)。

- VideoBEV (Han et al., arXiv:2303.05970): 本トピックの中心。タイトルは "Exploring Recurrent Long-term Temporal Fusion for Multi-view 3D Perception"。提案の新規性は、単純再帰の情報希釈を temporal embedding による長期保持で克服したこと。動機は、並列方式のメモリ爆発を避けつつ(再帰のメモリ一定を維持)、希釈で失われていた長期文脈を取り戻すことです。「並列でN倍のメモリを払わずに、並列に匹敵する長期文脈を得る」点が貢献です。

- StreamPETR (Wang et al., ICCV 2023, arXiv:2303.11926): 同時期に「効率的な長期時間モデリング」を別の切り口(密マップでなく物体クエリの伝播)で解いた手法。VideoBEVが密BEVの再帰を磨くのに対し、こちらはスパースなクエリ伝播。効率と表現力の異なる極として対比されます。詳細は [StreamPETR型 オブジェクト中心伝播](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0503_streampetr-object-propagation)。

系譜の論点は「再帰(メモリ一定だが希釈リスク) vs 並列(情報保持だがメモリ増大)」で、長期再帰融合は再帰側に立って希釈を克服する立場です。

## 論文の実験結果(定量データ)

評価は nuScenes(6カメラ・360度の自動運転ベンチマーク)が中心です。指標を説明します。

- NDS(nuScenes Detection Score): 検出の総合スコア。mAP(検出の正確さ)に加え、位置・サイズ・向き・速度・属性の各誤差を統合した0〜1の値で、大きいほど良い。

- mAVE(mean Average Velocity Error): 速度推定の平均誤差(m/s)。小さいほど良い。時系列融合が最も効く指標です。

報告されている傾向(VideoBEV論文より、おおよその値):

- VideoBEVは同等バックボーン・解像度の単純再帰(BEVFormer型)や2フレーム手法(BEVDet4D型)に対して、NDSで数ポイント程度の向上を報告しています。大型バックボーン設定で nuScenes test の NDS は 0.6 前後に達したと報告され、当時の再帰系として高水準です。特に長期文脈が効く設定で差が開きます。

- 並列方式(SOLOFusion型の長期並列融合)とほぼ同等の精度を、再帰のメモリ一定で達成できる点が強調されます。「並列だと保持フレーム数ぶんメモリが増える」のに対し、VideoBEVは長期化してもメモリがほぼ一定なので、長い時間窓を実用的なコストで使えます。これが論文の核心的な主張です。

- 速度誤差 mAVE の改善が顕著。長期の文脈が「物体が一定方向にどれだけ動き続けているか」の推定を安定させるためで、単純再帰比でも mAVE がさらに下がる傾向が報告されています。

- アブレーション: temporal embedding を外すと長期文脈の恩恵が消え、単純再帰に近い性能まで落ちることが示されます。これが「希釈を防いでいるのは時間埋め込みである」という主張の根拠です。一方で、融合に使うフレーム数を増やすほど効果は逓減し、ある長さ(報告ではおよそ8フレーム前後)で頭打ちになる傾向も報告されています。

注意点として、長期文脈の恩恵の大きさはデータセットに強く依存します。動的物体が多く系列が長いシーンでは効きますが、動きが乏しく短い系列では単純再帰との差は小さくなります。論文でも「単純再帰に対する優位はデータセット依存」という趣旨が述べられており、この手法は「まず短期の時間融合を入れ、希釈が問題だと確認してから導入を検討する」発展段階の手法と位置づけられます。

## メリット・トレードオフ・限界

メリット:

- メモリを一定に保ったまま長期の時間文脈を使える(再帰の効率 + 長期保持)。
- 並列方式に匹敵する精度を、はるかに小さいメモリで得られる可能性がある。
- 長い隠れの追跡や、ゆっくりした動きの傾向把握に強い。
- 単純再帰の「情報希釈」を時間埋め込みで明示的に克服する。

トレードオフ・限界:

- temporal embedding の導入で仕組みが複雑になり、学習・実装の難度が上がる。
- 長期文脈の恩恵はデータセット依存。短い系列・動きの少ないシーンでは効果が出にくく、過剰投資になりうる。
- 自車運動整合の誤差が長期で積もりやすく、整合の精度がより重要になる。整合がずれると長期記憶ほど壊れやすい。grid_sample の補間が再帰のたびに特徴をわずかに鈍らせ、累積する。
- 効果がフレーム数に対して逓減する。どこまで遡るべきかの最適点はタスク依存で自明でない。
- 学習時に長い系列を展開して勾配を流すため、学習メモリと時間が増える(BPTTの一般的な負担)。

研究上の未解決課題としては、(1)時間埋め込みが暗黙に学習する「忘却の速さ」をタスクに応じて適応させること、(2)再帰記憶の中で「静的な舞台」と「動く物体」を別の時定数で扱うこと、(3)長期整合のドリフト(自車位置誤差の蓄積)をモデル内で補正することなどが挙げられます。

## 発展トピック・研究の最前線

長期再帰融合は、世界モデル(world model)系の研究と接続します。過去の観測を圧縮した内部状態を再帰的に更新し、そこから未来を予測する枠組みは、再帰記憶 + 時間埋め込みの発想と地続きです。状態空間モデル(Mamba 系)による長系列の効率的な再帰も、希釈と計算量のトレードオフを別角度から解く候補として注目されています。また、密なBEV記憶を持つ再帰系と、軽量な物体クエリを持ち回るスパース系([StreamPETR型 オブジェクト中心伝播](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0503_streampetr-object-propagation))は、効率と表現力の異なる極として比較され続けています。両者を組み合わせ「静的な舞台は粗い長期再帰BEVで、動く物体は精密なクエリで」と役割分担するハイブリッドが、近年の有望な方向です。
