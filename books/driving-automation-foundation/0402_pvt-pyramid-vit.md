# PVT (Pyramid Vision Transformer)

## ひとことで言うと
PVT(Pyramid Vision Transformer)は、画像を扱う Transformer(注意機構ベースのニューラルネット)を、物体検出や領域分割のような「画素レベルの細かい予測(密予測)」に使えるよう作り変えたバックボーンです。CNN のように特徴の解像度を段階的に下げて「ピラミッド状」の多解像度特徴を出すこと、そして注意計算を安くする「空間縮小注意(SRA)」を導入したことが核心です。これにより、検出・分割の世界で長年標準だった CNN バックボーン(ResNet など)を、純粋な Transformer で初めて置き換えられるようになりました。

## 直感的な理解
2020 年に登場した元祖 Vision Transformer(ViT)は、画像分類で CNN に匹敵する精度を出して衝撃を与えました。注意機構で画像全域の離れた場所どうしの関係を一度に捉えられる点が魅力です。ところが ViT を物体検出や領域分割にそのまま使おうとすると、二つの壁にぶつかりました。

一つは解像度の問題です。検出や分割では、遠くの小さな歩行者から目の前の大きなトラックまで、さまざまな大きさの物体を扱う必要があり、細かい高解像度の特徴と大局的な低解像度の特徴の両方が要ります。CNN は処理を進めるにつれ解像度を段階的に下げる「ピラミッド」を自然に作りますが、ViT は最初に画像を 16×16 のパッチに切った後、最後まで同じ低解像度(1/16)のまま処理するため、このピラミッドを持ちません。1/16 という粗さは、小さな物体を捉えるには荒すぎます。

もう一つは計算量の問題です。自己注意は全パッチ対全パッチの関係を計算するため、パッチ数の 2 乗で計算が増えます。高解像度画像はパッチが多く、注意計算が爆発的に重くなって現実的に回りません。例えば 1/4 解像度のパッチを使おうとすると、ViT 比でパッチ数が 16 倍、注意計算は 16² = 256 倍に膨らみます。

PVT はこの二つを、CNN を真似たピラミッド構造と、注意を安くする空間縮小注意で同時に解決しました。「Transformer に CNN の良いところ(多解像度・段階的縮小)を取り入れる」発想です。

## 基礎: 前提となる概念
- Transformer / 自己注意 (self-attention): 入力を小さな単位(トークン)に分け、各トークンが他のすべてのトークンとどれだけ関係するかを計算し、関係の強い相手から情報を集める仕組みです。Query(問い合わせ)、Key(鍵)、Value(値)という 3 種のベクトルを使い、Query と Key の内積で関係の強さ(注意の重み)を測り、その重みで Value を加重平均します。画像全域の長距離関係を一度に捉えられるのが長所です。
- トークン (token): Transformer が扱う最小単位。画像では小領域(パッチ)1 個に対応します。N 個のトークンがあるとき、自己注意は N×N の関係行列を作ります。
- ViT (Vision Transformer, Dosovitskiy et al., ICLR 2021): 画像を 16×16 などのパッチに切ってトークンとみなし、Transformer に通す手法。分類で CNN に匹敵しました。
- 密予測 (dense prediction): 物体検出や意味的領域分割など、画像の各画素/各位置に対して細かく答えを出すタスクの総称。分類のように画像 1 枚に 1 ラベルではなく、位置ごとに答えが要ります。
- 特徴ピラミッド (feature pyramid): 解像度の違う特徴を複数段(高解像度で細かい/低解像度で大局的)持つこと。大小さまざまな物体を扱う検出・分割では事実上必須で、FPN(Feature Pyramid Network)などのヘッドはこのピラミッドを前提に設計されています。
- 計算量が O(N²): N 個のトークンに対し全ペアの関係を計算するため、計算量がトークン数の 2 乗で増えること。N が増えると急激に重くなります。
- 位置埋め込み (positional embedding): Transformer は本来トークンの順序・位置を知らないため、各トークンに「自分は何番目か」を表すベクトルを足す仕組み。固定長で学習すると、入力解像度が変わったとき破綻しやすいという弱点があります。

## 仕組みを詳しく

### 1. ピラミッド構造(4 ステージで解像度を段階的に下げる)
CNN(ResNet)が 4 段階で特徴を縮小していくのを真似て、PVT も 4 ステージで構成します。各ステージの最初に「パッチ埋め込み(patch embedding)」で隣接するトークンをまとめ、解像度を下げます。

```
入力画像 224×224
Stage1: 特徴 56×56  (画像の 1/4,  チャネル少なめ、トークン 3136 個)
Stage2: 特徴 28×28  (1/8,   トークン 784 個)
Stage3: 特徴 14×14  (1/16,  トークン 196 個)
Stage4: 特徴 7×7    (1/32, チャネル多め、トークン 49 個)
```

これにより、CNN バックボーンが出すのと同じ「複数解像度の特徴ピラミッド」が得られ、既存の検出器(RetinaNet, Mask R-CNN, DETR など)や分割ヘッド(Semantic FPN など)の上にそのまま差し込めます。高解像度の Stage1 で細部を、低解像度の Stage4 で大局を捉えます。解像度が下がるにつれチャネル数を増やすのも CNN の定石どおりで、情報量(空間 × チャネル)をおおむね保ちます。

### 2. 空間縮小注意 (Spatial-Reduction Attention, SRA)
自己注意の O(N²) 問題を緩和する工夫です。SRA は、注意を計算する前に Key と Value のトークン数だけを畳み込みで減らします。

具体的には、縮小率 R(例 8)を使い、Key/Value 側の特徴を R×R のブロックにまとめて 1 個のトークンに圧縮します(ストライド R の畳み込みで実装)。Query 側はそのままなので、出力の空間的な細かさ(=Query の数)は保たれ、関係計算の相手(Key/Value)だけを粗くする、という非対称な工夫です。

```
通常の注意: Query N個 × Key N個      → 計算量 ∝ N²
SRA:        Query N個 × Key (N/R²)個   → 計算量 ∝ N²/R²   (R=8 なら 1/64)
```

直感的には、「出力の各位置は細かく出すが、その位置が周囲を見渡すときは少し粗い解像度で全体を眺める」イメージです。地図を見るとき、自分の現在地は番地レベルで把握しつつ、街全体は区画レベルのざっくりした地図で眺める、という感覚に近いです。高解像度でトークンが多い Stage1 では R を大きく(例 8)して計算を強く削り、トークンが少ない Stage4 では R を小さく(例 1)します。これで高解像度ステージでも注意計算が現実的なコストに収まり、ピラミッドの最上段(高解像度)を持てるようになりました。

### 3. PVTv2 の改良
続編 PVTv2(Wang et al., 2022)は 3 点を改良しました。

- 重なりありパッチ埋め込み (overlapping patch embedding): パッチを少し重ねて切る(ストライド < カーネルサイズの畳み込み)ことで、パッチ境界をまたぐ局所構造(連続性・エッジ)を保ち、ViT が苦手だった CNN 的な局所感度を取り戻しました。v1 はパッチを重ねずに切るため境界で情報が途切れていました。
- 線形複雑度の注意 (Linear SRA): Key/Value を畳み込みで縮小する代わりに、固定サイズ(例 7×7)へ平均プーリングで圧縮することで、Key/Value のトークン数を入力解像度によらず一定にし、計算量をトークン数に比例(線形 O(N))させました。これで超高解像度でもスケールします。
- 畳み込みフィードフォワード (convolutional FFN): Transformer ブロック内の全結合部(FFN)に depthwise 畳み込みを挟み、トークンの位置関係を暗黙的に扱えるようにしました。これにより、画像サイズが変わると破綻しがちな固定の位置埋め込みを不要にしています(検出・分割では入力解像度が様々なのでこれは実用上重要)。

これらは「Transformer に CNN の帰納バイアス(局所性・平行移動への頑健さ)を後から足す」改良で、純 Transformer の弱点を補いつつ大域注意の長所は残す、という方向性です。

## 手法の系譜と主要論文

- Vision Transformer (Dosovitskiy et al., ICLR 2021, Google, arXiv:2010.11929)。画像をパッチ列として Transformer に入れ、大規模データ(JFT-300M)で事前学習すれば分類で CNN に匹敵することを示した起点。ただし単一低解像度(1/16 のまま)・O(N²) で、密予測には不向きでした。

- Pyramid Vision Transformer (Wang et al., ICCV 2021, 南京大学 / 香港大学 / 商湯科技, arXiv:2102.12122)。ViT を密予測に使えるよう、CNN 流のピラミッド多解像度構造と、注意を安くする空間縮小注意(SRA)を導入。純 Transformer バックボーンとして初めて検出・分割で ResNet 系を一貫して上回りました。「畳み込みなしの汎用バックボーン」を標榜した点に新規性があります。

- Swin Transformer (Liu et al., ICCV 2021, Microsoft, arXiv:2103.14030)。PVT とほぼ同時期(2021 ICCV)の対抗馬で、注意を「窓(局所領域、例 7×7 パッチ)」に限定して O(N²) を窓サイズに対する一定コストに抑え、窓を半分ずらす(shifted window)ことで隣の窓をまたぐ大域情報も段階的に伝える、という別アプローチでピラミッド特徴を出します。PVT(Key/Value を空間縮小して大域を粗く見る)と Swin(注意を局所窓に限定し、ずらしで大域を繋ぐ)は、どちらも「ViT の O(N²) 問題を解いてピラミッド特徴を出す」同じ目的への双璧で、以後の階層型 ViT バックボーンの二大潮流になりました。

- PVTv2 (Wang et al., Computational Visual Media 2022, arXiv:2106.13797)。重なりパッチ・線形注意・畳み込み FFN で局所性と効率を改善。v1 が局所構造と位置情報の扱いで CNN に劣り計算もまだ重かった点を補い、線形計算量とさらなる精度向上を達成し、Swin と互角以上の評価を得ました。

## 論文の実験結果(定量データ)
評価は ImageNet 分類(top-1 精度、大きいほど良い)、MS COCO 物体検出(mAP = mean Average Precision、大きいほど良い)、ADE20K 意味的分割(mIoU = mean Intersection over Union、領域ごとの正解との重なり率の平均、大きいほど良い)で行われました。報告された目安は次のとおりです。

```
分類 (ImageNet, top-1)
ResNet-50    ≈ 25.6M params, 4.1 GFLOPs, top-1 約 76〜78%
PVT-Small    ≈ 24.5M params, 3.8 GFLOPs, top-1 約 79.8%
PVTv2-B2     ≈ 25.4M params, 4.0 GFLOPs, top-1 約 82.0%
Swin-T       ≈ 28M  params, 4.5 GFLOPs, top-1 約 81.3%
```

ここを初心者向けに読み解きます。同程度のサイズ(パラメータ数・FLOPs)の ResNet-50 と比べると、PVT-Small は分類 top-1 で約 2〜4 ポイント高く、PVTv2-B2 にいたっては約 82.0% と大きく上回ります。top-1 で 2 ポイント以上の差は ImageNet では明確な改善です。PVTv2-B2 は同サイズの Swin-T(約 81.3%)も上回り、純 Transformer 系の中でも競争力があることを示しました。

検出・分割ではバックボーンの差がさらに大きく出ます。論文の報告では、RetinaNet 検出器(1 段検出器)のバックボーンを ResNet-50 から PVT-Small に替えると、COCO の box mAP がおよそ 36 台から 40 前後へと約 4 ポイント向上します(mAP は検出の総合精度で、1〜2 ポイントの向上でも実用上意味があり、4 ポイントは大きい)。Mask R-CNN(2 段検出器、インスタンス分割つき)でも box / mask 両方の mAP が一貫して向上します。PVTv2-B2 ではさらに伸び、ADE20K の意味的分割でも Semantic FPN ヘッドの mIoU が ResNet 系を明確に上回ったと報告されています。これらは「Transformer の長距離注意が、物体の全体像や周辺文脈を捉えるのに効く」ことを示しています。とくに大きな物体や、文脈(周囲との関係)で判断すべき領域で効果が顕著です。

アブレーション(要素を抜く検証)の主な知見:
- 空間縮小注意 (SRA) を抜く(=通常の全注意にする)と、高解像度ステージ(Stage1, 2)で計算量が爆発し、メモリにも乗らず学習が現実的でなくなる。SRA はピラミッド最上段(高解像度)を可能にする要です。
- ピラミッド構造を持たず単一解像度(元祖 ViT 風)にすると、検出・分割で大きく性能が落ちる。多解像度が密予測に必須であることの裏づけ。
- PVTv2 のアブレーションでは、重なりパッチ・畳み込み FFN・線形注意のそれぞれが精度と効率に寄与し、特に畳み込み FFN による局所性と位置情報の補強が小さい物体の検出に効いたと報告されています。重なりパッチも境界付近の小物体検出に貢献しました。

## メリット・トレードオフ・限界
メリット
- 多解像度ピラミッド特徴を出すので、既存の検出/分割ヘッド(FPN, RetinaNet, Mask R-CNN, DETR など)にそのまま差し込め、小さい物体も大きい物体も扱える。
- 空間縮小注意で、高解像度ステージでも注意計算を現実的なコストに抑えられる(v2 では線形計算量に)。
- 自己注意により、画像全域の離れた場所どうしの関係を CNN より得意に捉える。文脈理解が要るシーンに強い。

トレードオフ・限界
- SRA で Key/Value を粗くするため、最も細かい長距離関係(遠くの細部どうしの精密な対応)は一部失われる(粗い解像度で全体を眺めることのコスト)。
- 純 Transformer は CNN より大量データ・強い正則化・データ拡張や事前学習を要し、小規模データだと過学習しやすい。ImageNet 規模以上の事前学習がほぼ前提(v2 は畳み込みを混ぜて帰納バイアスを足すことでこの要求をいくぶん緩和)。
- 同じ精度帯では、最適化が進んだ CNN 実装より実機速度(レイテンシ)が劣る場合がある。注意機構はメモリ帯域を多く使い、ハードウェアによっては FLOPs ほど速くならない。組込み・車載のような制約環境では特に注意が必要。
- 局所窓ベースの Swin と比べ、どちらが優れるかはタスク・解像度・実装に依存し一概に言えない。設計選択は実測比較を要する。

## 発展トピック・研究の最前線
PVT と Swin が確立した「階層型(ピラミッド)Vision Transformer」は、その後の検出・分割バックボーンの主流になりました。注意の効率化はさらに進み、Twins(局所注意と大域注意を交互に重ねる)、PVTv2 の線形注意、Focal Transformer(細かい近距離 + 粗い遠距離の注意を併用)、CSWin(十字形の窓注意)など、さまざまな「安い注意」の変種が提案されています。一方で、CNN 側も ConvNeXt のように Transformer の設計知見(大きなカーネル、層正規化 LayerNorm、活性化を減らす、逆ボトルネック)を取り込んで巻き返し、CNN と Transformer の境界は曖昧になりつつあります。

近年は CNN と注意を組み合わせたハイブリッド(CoAtNet、MaxViT など)が精度と効率の両面で強く、純粋などちらか一方にこだわらない設計が一般化しました。複数カメラ・複数視点の特徴を鳥瞰図(BEV, Bird's-Eye View)空間に統合して自動運転の知覚を行う近年の手法群(BEVFormer など)は、まさに PVT が解いた「高解像度での注意の計算コスト」と「多解像度特徴の活用」を前提として組み立てられています。BEVFormer は画像特徴ピラミッドに対して空間・時間の交差注意を掛けて BEV 表現を作りますが、その下支えとして高解像度でも回る効率的な注意が不可欠であり、PVT の問題意識は自動運転の知覚スタックに直結しています。研究の最前線では、こうした大域注意を保ちつつ車載リアルタイム制約に収める軽量化・蒸留・量子化、そして注意を線形時間で近似する手法(線形注意、状態空間モデル系)が活発なテーマです。
