# Pillow の Image.Resampling enum 化対応と画像補間

## ひとことで言うと
画像を拡大・縮小するときに「ピクセルの隙間をどう埋めるか(補間方法)」を指定する書き方が、Python の画像ライブラリ Pillow の新しいバージョンで変わりました。古い書き方 `Image.BILINEAR` は非推奨になり、新しい書き方 `Image.Resampling.BILINEAR` が正式になりました。表面的には「1行を書き換えるだけ」の API 互換修正ですが、その背景には画像補間という、機械学習の前処理で避けて通れない重要なテーマがあります。

## 直感的な理解
方眼紙に描いた絵を、別のサイズの方眼紙に写し取る場面を想像してください。元の絵が10マス四方、新しい紙が5マス四方なら、新しい紙の1マスは元の絵のどのマスに対応するでしょうか。きっちり1対1には対応せず、新しい1マスが元の2マス分にまたがります。このとき「またがった元のマスの色をどう混ぜて1色にするか」を決めるのが補間(interpolation, resampling)です。

これは見た目の問題にとどまりません。機械学習では、撮影サイズがバラバラの画像をモデルの入力サイズ(例: 224×224)にそろえる前処理が必ず入ります。ここでどの補間を選ぶかで、入力画像の細部の保たれ方が変わり、結果としてモデルの精度にまで影響します。「ただのリサイズ」と侮ると、学習時と推論時で補間方法が食い違って精度が落ちる、といった落とし穴にはまります。後述するように、この前処理のずれは ImageNet 分類で測定可能な精度差を生むことが実証されています。

## 基礎: 前提となる概念
- Pillow(ピロー): Python で画像を読み書き・加工する定番ライブラリ。`PIL` という名前で import します(`from PIL import Image`)。歴史的経緯で、配布パッケージ名は Pillow、import 名は PIL です。
- リサイズ(resize): 画像を別の大きさに変えること。`img.resize((W, H), 補間方法)` のように呼びます。
- 補間 / リサンプリング(interpolation / resampling): 元のピクセルと新しいピクセルの位置がずれるとき、間の色を計算する方法。代表的な方式を挙げます。
  - 最近傍法(NEAREST): 一番近い元ピクセルの色をそのまま使う。最速だがギザギザ(ジャギー)が出る。セグメンテーションのラベル画像など「色を混ぜてはいけない」場面では逆にこれが正解。
  - 双線形補間(BILINEAR): 周囲4ピクセルを距離に応じて加重平均する。滑らかで軽く、実用上のバランスが良い。
  - 双三次補間(BICUBIC): 周囲16ピクセルを使い、より滑らか。やや重い。
  - LANCZOS: sinc 関数に基づく高品質な縮小向けフィルタ。縮小で最も鮮鋭さを保ちやすいが最も重い。
- enum(列挙型, enumeration): とり得る値が決まった定数の集まりに、名前と型を与えたもの。曜日を裸の整数 `0,1,2` でなく `Weekday.MONDAY` と扱う仕組みです。これは「この値は補間方法の一種だ」と型として明示でき、エディタの補完が効き、型チェッカーも「ここに渡せるのは Resampling のどれか」と判定できる、という意味を持ちます。
- マジックナンバー(magic number): コード中に直書きされた、意味の説明がない裸の数値。`resize(size, 2)` の `2` のような書き方は、`2` が双線形を指すと知らなければ読めず、誤って別の数を書いても気づけません。enum 化はこのマジックナンバーを名前付き定数に置き換える改善です。

## 仕組みを詳しく
API がどう変わったかを整理します。

- 旧来(Pillow 9 より前): `img.resize(size, Image.BILINEAR)` のように、`Image` モジュール直下の定数 `Image.BILINEAR`(中身は整数 2)を渡していました。これらは裸の整数で、型としては「ただの int」でした。
- Pillow 9.1.0(2022年)以降: これらの定数は `Image.Resampling` という enum にまとめられ、`Image.Resampling.BILINEAR` と書くのが正式になりました。`Image.BILINEAR` も後方互換で当面は動きますが「非推奨(deprecated)」となり、将来のバージョンで削除予定とされています。

書き換えは1行です。

```python
# before
img = img.resize(image_size, Image.BILINEAR)

# after
img = img.resize(image_size, Image.Resampling.BILINEAR)
```

処理の中身は before / after で完全に同じです。`Image.Resampling.BILINEAR` の内部値は依然として整数 2 で、`Image.BILINEAR` と同じものを指します。たとえば 1000×800 の画像に `image_size = (512, 512)` を渡すと、出力は 512×512 になり、隣り合う元ピクセルの色を距離で加重平均した滑らかな結果が得られます。生成されるピクセルは1ビットも変わりません。変わるのは次の3点だけです。

1. 非推奨警告(DeprecationWarning)が出なくなり、ログがきれいになる。
2. enum を介すことで「これは補間方法である」と型として明示され、静的型チェッカー([mypy の段階的導入向け緩和設定 (gradual typing)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1129_mypy-incremental-adoption-config) 参照)が「ここに int リテラルを渡している」と曖昧に扱わずに済む。
3. 将来 Pillow が旧 `Image.BILINEAR` を削除しても壊れない(前方互換)。

双線形補間の中身を数式で確認します。新しいピクセル位置 (x, y) が、元画像の整数格子点 (i, j), (i+1, j), (i, j+1), (i+1, j+1) の間にあり、小数部を dx, dy(0以上1未満)とすると、出力の色 P は次のように計算されます。

```
P = (1-dx)(1-dy) * I(i,   j)
  +   dx (1-dy)  * I(i+1, j)
  + (1-dx) dy    * I(i,   j+1)
  +   dx  dy     * I(i+1, j+1)
```

ここで I(i, j) は元画像の格子点 (i, j) の色値、dx・dy は「新しい点が右下方向へどれだけ寄っているか」を表す重みです。4つの重みの合計は常に1になり、近い格子点ほど大きく寄与します。最近傍法はこの式で「最も近い1点の重みを1、残りを0」にした極端な場合に相当します。

数値例で確かめます。4つの隣接ピクセルの輝度値が左上 I(i,j)=100、右上 I(i+1,j)=200、左下 I(i,j+1)=120、右下 I(i+1,j+1)=220 で、新しい点が dx=0.5, dy=0.5(ちょうど真ん中)にあるとします。すると重みはすべて 0.25 になり、P = 0.25×(100+200+120+220) = 0.25×640 = 160。最近傍法なら最も近い1点(たとえば 100)を選ぶので、双線形が中間色 160 を作るのに対し最近傍は元の値をそのまま使う、という違いが具体的に見えます。

なぜ実務で双線形がよく選ばれるかというと、最近傍より滑らか(ジャギーが出ない)で、双三次や LANCZOS より軽い、という「中庸」だからです。地図線や軌跡を描いた画像のように「滑らかさはほしいが計算は軽くしたい」前処理で特に好まれます。一方、用途によって最適解は変わるので、補間方法は見た目の好みではなくタスクに応じて選ぶべき設計判断だ、という認識が重要です。

## 手法の系譜と主要論文
これは単一の論文発の手法というより、画像処理の古典的理論と、それを実装するライブラリの API 進化が交わる場所です。

理論面では、補間は信号処理のサンプリング理論に根ざします。最近傍・双線形・双三次は多項式補間の次数を上げた系列で、Gonzalez & Woods "Digital Image Processing"(画像処理の標準教科書)では、双線形は「最近傍より滑らかだが双三次より軽い、実用的な中庸」と位置づけられています。より高品質な LANCZOS フィルタは sinc 関数(理想的な低域通過フィルタ)を有限窓で近似するもので、Duchon "Lanczos Filtering in One and Two Dimensions"(J. Applied Meteorology, 1979)が古典的な定式化です。sinc に基づくため、縮小時にエイリアシング(本来の細かい模様が偽の縞模様= モアレに化ける現象)を抑えつつ鮮鋭さを保てる一方、計算量が大きいというトレードオフを持ちます。

ライブラリ進化の面では、Pillow が補間定数を整数から enum へ移したのは、型安全性とコードの可読性を高める一般的なリファクタリングの一例です。裸の整数定数(マジックナンバー)は「2 が何を意味するか」がコードから読み取れず、誤って別の整数を渡しても気づけません。enum 化は、こうした曖昧さを型で縛る方向の改善で、Python エコシステム全体で見られる潮流です(標準ライブラリも `enum` モジュールの導入後、`socket` の定数群など多くの定数が enum 化されました)。

機械学習側では、前処理が精度に効くことを定量化した研究群が重要です。Touvron らの "Fixing the train-test resolution discrepancy" (NeurIPS 2019) は、学習と評価で解像度や前処理がずれると分類精度が体系的に落ちることを示し、補間を含む前処理の整合性に研究者の注意を向けさせました。さらに高品質化の方向として、Dong らの SRCNN (ECCV 2014) 以降の超解像研究が、固定カーネル補間を学習ベース手法で置き換える系譜を切り拓きました。

## 論文の実験結果(定量データ)
補間方法・前処理の選択が機械学習の精度に与える影響は、複数の経験的研究で定量化されています。

画像分類では、Touvron らの FixRes 系研究(NeurIPS 2019)が代表的です。彼らは、学習時のデータ拡張(ランダムクロップ等)と評価時の中央クロップで、物体がモデルに見える「実効的な大きさ」がずれることを指摘し、評価解像度を調整するだけで精度が改善することを示しました。指標は ImageNet の Top-1 精度(1000クラス分類で最上位予測が正解と一致する割合)で、解像度・前処理の不整合を補正するだけで Top-1 が1〜2ポイント程度、設定によってはそれ以上動きうることが報告されています。1〜2ポイントというと小さく聞こえますが、ImageNet では最新手法どうしの差がこの程度であることも多く、前処理の食い違いだけで「モデルを1世代ぶん損する」に等しい影響があるという意味です。補間方法の食い違い(学習は双線形、評価は双三次など)も、この「train-test の前処理ずれ」の一種として精度に効きます。

セグメンテーション(画像を画素単位で領域分けするタスク)では、補間の選び方が品質を決定的に左右します。入力画像には滑らかな双線形が適する一方、クラスラベルを格納したマスク画像に双線形を使うと、隣接クラスの整数ラベル(例: クラス3とクラス5)が平均されて存在しないラベル(クラス4)が生まれてしまいます。ここでは必ず最近傍法を使う、というのが定石で、これを誤ると評価指標 mIoU(mean Intersection over Union, 予測領域と正解領域の重なり = 交差を、和集合で割った値の全クラス平均。0から1で、1に近いほど良い)が目に見えて劣化します。アブレーション的に言えば「マスクを双線形でリサイズする」という一見些細な誤りだけで、mIoU が数ポイント単位で落ちる事例が報告されています。

超解像では、Dong らの SRCNN (ECCV 2014) が固定補間に対する学習ベース手法の優位を定量化しました。指標は PSNR(ピーク信号対雑音比, dB。値が大きいほど元画像に近い)で、双三次補間をベースラインとして、Set5・Set14 などの標準ベンチマークで数 dB の改善が報告されています。PSNR が +1dB 上がるごとに復元誤差(平均二乗誤差)が体系的に小さくなることを表し、双三次という古典的補間に対し学習ベース手法が明確に優位なことを示します。

これらの数字の意味は、「リサイズの補間は見た目の好みではなく、タスクに応じて選ぶべき設計判断であり、選び方を誤ると測定可能な精度低下を招く」ということです。enum 化はこの選択を型として明示し、誤った定数を渡しにくくする副次的な効果も持ちます。

## メリット・トレードオフ・限界
enum 化対応そのもののメリット:
- 非推奨警告が消え、ログがきれいになる。
- enum 経由で型が明確になり、静的解析が通る。コードを読んだ人が「2」ではなく「BILINEAR」と意図を読み取れる。
- 将来 Pillow が旧定数を削除しても壊れない。

トレードオフと限界:
- 実行時の挙動・画質は一切改善しない。純粋に互換・型・可読性のための修正であり、精度には影響しない。
- 新しい書き方は古い Pillow(9.1.0 より前)には存在しないため、極端に古い Pillow を使う環境では逆に動かない。依存バージョンの下限に注意が要る。

補間方法そのものの限界(より本質的な研究課題):
- 双線形・双三次は固定のカーネル(重みの形)を使うため、エッジ(輪郭)をまたいで色を混ぜてしまい、輪郭がぼける。これは「内容に依存しない補間」の原理的な限界です。
- 縮小時には、補間の前にローパスフィルタ(なまし)をかけないとエイリアシングが出る。Pillow の `resize` は内部でこれを考慮しますが、自前で補間を実装したり `align_corners` などの設定を誤ると見落としがちな落とし穴です。

## 発展トピック・研究の最前線
固定カーネル補間の限界を超える方向として、学習ベースの超解像(super-resolution)があります。SRCNN(Dong ら, ECCV 2014)以降、畳み込みニューラルネット、続いて GAN ベース(SRGAN)、近年は拡散モデルベースの手法が発展し、輪郭をまたいだぼけを抑えつつ、知覚的にも自然な高解像度画像を生成できるようになっています。固定カーネルが「内容を見ずに一律に混ぜる」のに対し、学習ベースは「この輪郭は保ち、この平坦部は滑らかに」と内容適応的に振る舞える点が本質的な違いです。

再現性の観点では、機械学習フレームワークごとに「同じ双線形」と称しても実装の細部(`align_corners` の扱い、アンチエイリアスの有無、座標系の取り方)が異なり、結果が微妙にずれる問題が知られています。PyTorch・TensorFlow・Pillow・OpenCV の双線形リサイズが完全には一致しないことは経験的によく指摘され、学習と推論で異なるライブラリを使うと、モデルの重みは同じなのに精度が再現しない、という厄介なバグの原因になります。前処理のライブラリと補間設定を学習・推論で厳密にそろえる、という再現性確保の実践は、研究の信頼性を保つうえで地味ながら重要なテーマです。enum による明示は、こうした「どの補間を使っているか」をコード上で取り違えにくくする小さな一歩でもあります。
