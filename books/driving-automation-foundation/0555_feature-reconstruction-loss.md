# 特徴再構成ロス(未来特徴予測の補助損失)

## ひとことで言うと

運転や行動予測のモデルでは、「これから数秒後に景色がどう変わるか」を内部で予測させると、現在のシーンを理解する力そのものが鍛えられることが知られています。特徴再構成ロス(feature reconstruction loss)は、モデルが予測した「未来の特徴」と、実際にその時刻に観測された「本物の未来の特徴」を比べ、その差(主に MSE = 平均二乗誤差)を小さくするように学習させる補助的な損失です。生の画素ではなく抽象化された特徴空間で比べる点が肝で、これによって意味のある変化だけを学ばせ、影や質感のような無関係な細部を無視できます。本来解きたいタスク(例: 軌道予測)とは別の「おまけ課題」ですが、共有する前段の表現を底上げすることで本タスクの精度も上げる狙いがあります。

## 直感的な理解

学習中の生徒にたとえます。最終目標が「正しいハンドル操作を決める」ことだとしても、いきなり操作だけを練習させると、なぜその操作が正しいのかの理解(=世界がどう動くか)が育ちません。そこで「この後この車はどう動くと思う?」「次の交差点で何が起きそう?」と予測させる練習を併走させると、生徒は世界の因果を理解せざるを得なくなり、結果として操作判断も上達します。特徴再構成ロスはこの「予測練習」をモデルにさせる仕組みです。

もう一つの直感は、学習信号(勾配)の届き方です。本タスクの損失はモデルの一番出力側で計算され、その学習信号が前段のバックボーンまで届くには多くの層を逆向きに通る必要があります。途中で信号が薄まり、前段に十分な指導が届きにくくなります。前段に近い場所で「未来を予測してみろ」という課題を直接与えれば、前段に濃い学習信号が届き、「この後どう変化するか予測できるくらい情報量の多い特徴を作れ」という明確な指示になります。

## 基礎: 前提となる概念

用語を一つずつ噛み砕きます。

- 損失 (loss): モデルの出力が「どれだけ間違っているか」を1つの数値で表したもの。学習はこの数値を小さくする方向にパラメータを少しずつ動かす作業です。
- MSE (Mean Squared Error, 平均二乗誤差): 予測値と正解の差を2乗して平均した値。差が大きいほど急激に大きくなり、最も基本的な損失です。
- 補助タスク (auxiliary task): 本来解きたい課題(例: 軌道予測)とは別に、おまけで解かせる課題。直接の目的ではないが、解かせることで本体の表現が良くなります。
- 未来特徴予測モジュール: 現在のフレームの特徴を受け取り、一定時間後(例: 1.6秒後、3.2秒後、4.8秒後、6.4秒後)の特徴マップを予測する部品。10Hz(1秒10フレーム)で撮影しているなら、1.6秒後はおよそ16フレーム先です。
- 凍結 (freeze) / detach: パラメータや勾配を固定し、学習で動かさないこと。`detach()` は「この値については勾配を流さない」という指示です。
- 表現崩壊 (representation collapse): 予測と正解の両方を学習可能にすると、モデルが「すべてを同じ定数ベクトルにすれば損失ゼロ」というズルを見つけ、特徴が無意味な一点に潰れる現象。これを防ぐため正解側(ターゲット)を固定します(詳しくは [表現崩壊(representation collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0600_representation-collapse))。

なぜ画素ではなく特徴で比べるのか。未来の画像を画素単位で当てさせると、モデルは歩道のタイル模様や雲の形、カメラノイズまで再現させられ、容量と学習労力の大半を無関係な細部に費やします。さらに未来は本質的に不確実(歩行者が右に行くか左に行くか分からない)なので、画素を一点に決め打ちさせると平均的でぼやけた無難な画像しか出せません。特徴空間には既に「車」「歩行者」「進行方向」といった意味だけが残り細部は捨てられているので、意味的に妥当な未来を当てればよく、これらの問題を回避できます。この思想は JEPA(詳しくは [JEPA(潜在空間での未来予測)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0590_jepa-latent-prediction))に由来します。

## 仕組みを詳しく

### 全体の流れ

```
現在フレーム t:
  image_t --Backbone--> Fusion --予測モジュール--> 予測特徴[t+1.6s], [t+3.2s], [t+4.8s], [t+6.4s]

実際の未来フレーム t+16 (1.6秒後):
  image_{t+16} --凍結Backbone--> Fusion --> 目標特徴[t+1.6s]
  ...(他の時刻も同様)

loss = MSE( 予測特徴 , 目標特徴.detach() )
```

`detach()` で目標側の勾配を止めるのは、表現崩壊を防ぐためです。正解側を学習で動かせてしまうと、予測と正解が手を取り合って同じ定数に潰れてしまいます。

### 実装の中身をステップで

1. 入力: 予測特徴(予測モジュールが出した複数時刻ぶんの特徴マップ、各 `[B, C, H, W]`)と、目標特徴(凍結バックボーンが未来フレームから作った同じ形のもの)。
2. 形状チェック: 時刻数が揃っているか、各ペアの形が一致するかを確認。違えばエラーにします。
3. 各時刻ごとの MSE: `torch.mean((pred - target) ** 2)` を時刻ぶん計算します。
4. 時間重み付け: 各時刻に重みを掛けられるようにします。近い未来を重視したいなら `[2, 1, 1, 1]` のように設定します。近い未来ほど予測しやすく(=学習信号が安定)、遠い未来ほど不確実性が高いという性質を反映できます。
5. 重みの正規化: 重みの合計が時刻数(例: 4)になるよう割り戻すことで、重み付きでも素の MSE と同じスケールに保ちます。これがないと重みを変えるたびに損失全体のスケールが変わり、他の損失とのバランス取りが狂います。
6. ゼロ和ガード: 重みの合計がほぼ0(例: 1e-8 未満)だと割り算で NaN(非数)になるため、その場合は正規化をスキップします。数値事故を防ぐ実務上の工夫です。
7. reduction: `"mean"` なら最終的に1つのスカラー、`"none"` なら時刻ごとの損失を返します(時刻別にログを取って分析するため)。

具体的な数値例。予測と目標がどちらも `[B=4, C=256, H=8, W=8]` で、ある時刻の各要素の差の2乗平均が 0.04 だったとします。重みが全て1なら、その時刻の損失は 0.04。各時刻の平均をとって最終損失になります。

### 全体損失への組み込み

```python
total_loss = task_loss + lambda * feature_reconstruction_loss
```

`lambda`(ラムダ)は本タスクと補助タスクのバランスを取る重みです。典型的には 0.5〜1.0 程度が目安とされます。大きすぎると未来予測ばかり頑張って本タスク(軌道精度など)が落ち、小さすぎると補助信号が無視されます。実装では等重み(lambda=1相当)から始め、検証性能を見て調整する例が多いです。

### L1 損失という選択肢

MSE の代わりに L1 損失(差の絶対値の平均)を使う実装もあります。MSE は外れ値の影響を2乗で強調するため、まれな大きな予測ミスに学習が引っ張られます。L1 は外れ値に頑健ですが、勾配が一定なので最後の細かい収束がやや粗くなります。動画系の JEPA で L1 が有効と報告されたことを受け、両方を切り替えられるようにする設計が一般的です。

### 推論時の使い道(イントロスペクション)

学習後、この予測誤差を推論時に計算すると「モデルの不確かさ」の指標になります。予測誤差が大きいフレームは「モデルが予想しなかった状況」を意味し、工事現場・事故・未経験の天候・突然飛び出す歩行者などの異常検知(out-of-distribution 検知)に流用できます。学習に使った補助損失を、運用時の安全監視にそのまま転用できるのは実用上の大きな利点です。

## 手法の系譜と主要論文

- BYOL (Grill et al., NeurIPS 2020, arXiv:2006.07733): 自己教師あり学習で「予測側」と「ターゲット側」を非対称にし、ターゲットを EMA(指数移動平均、予測側の重みをゆっくり真似る更新)で動かすことで、ネガティブサンプル(他画像との対比)無しでも表現崩壊を防げることを示しました。本ロスが「ターゲットを凍結/EMA にして detach する」設計の理論的根拠です。

- JEPA / I-JEPA (LeCun の構想 2022; Assran et al., CVPR 2023, arXiv:2301.08243): 未来や欠損部分を画素ではなく学習済み潜在空間で予測する枠組み。本ロスが「画素ではなく特徴を MSE で合わせる」のはこの思想そのものです(詳しくは [JEPA(潜在空間での未来予測)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0590_jepa-latent-prediction))。

- V-JEPA (Bardes et al., 2024, arXiv:2404.08471): JEPA を動画に拡張し、時間方向に隠した部分の潜在を予測。L1 損失も有効と報告しました。動画では時間方向の予測が自然に「未来予測」になるため、時系列タスクと相性が良いという示唆を与えます。

- MILE (Hu et al., NeurIPS 2022, arXiv:2210.07729): 自動運転向けに潜在世界モデルを学習し、画素予測ベースラインを上回ることを示しました。運転では意味的変化(車がどう動くか)が重要で、路面のテクスチャ細部は不要だという主張の実証です。

- UniAD (Hu et al., CVPR 2023, Best Paper, arXiv:2212.10156): 検出・追跡・地図セグメンテーションといった知覚補助タスクを加えると、共有する BEV 表現の質が上がり、最終的なプランニング性能も上がることを示しました。本ロスも同じ「補助タスクで共有表現を鍛える」発想です。

系譜としては、自己教師あり表現学習(BYOL → I-JEPA → V-JEPA)で確立した「潜在空間で予測・崩壊回避」の技術を、自動運転の世界モデル(MILE)やマルチタスク統合(UniAD)が下流タスクの性能向上として取り込んだ流れです。特徴再構成ロスはその合流点に位置します。

## 論文の実験結果(定量データ)

- BYOL: ImageNet 線形評価(学習済み特徴の上に線形分類器だけ載せて精度を測る)で Top-1 約74.3%を達成し、ネガティブサンプルを使う SimCLR(約69.3%)を上回りました。ネガティブ無しでも崩壊せず高精度という点が画期的で、本ロスの「ターゲット固定で十分」という設計の裏付けです。

- I-JEPA: ViT-Huge を ImageNet で自己教師あり学習し、線形評価で高精度を、画素再構成型の MAE より大幅に少ない計算量(事前学習エポックあたりの計算が小さい)で達成しました。意味のある特徴を効率よく学べることを示しています。

- MILE: CARLA シミュレータの closed-loop 評価(実際に走らせて評価する厳しい設定)で、潜在世界モデルが画素予測ベースラインや模倣学習単独を上回る運転スコアを記録しました。アブレーションでは、潜在空間での予測を外す(=画素で予測する)と性能が落ちることが示され、「特徴空間で予測する」ことの寄与が定量的に確認されています。

- UniAD: nuScenes ベンチマークで、知覚・予測の補助タスクを段階的に追加するアブレーションを行い、補助タスクを足すごとに最終プランニングの衝突率(collision rate)と平均変位誤差(L2 距離)が改善することを示しました。具体的には、全モジュールを統合した構成が、プランニングだけの構成に比べ衝突率を大きく下げました。これは「補助損失が共有表現を鍛え、本タスクを助ける」という本ロスの仮説を最も直接的に支持する結果です。

指標の意味を補足します。線形評価の Top-1 は「特徴がどれだけ意味を捉えているか」の代理指標で、特徴が良いほど単純な線形分類器でも高精度になります。衝突率と L2 距離は運転計画の質を測り、衝突率が低く L2 が小さいほど安全で正確な軌道です。補助損失でこれらが改善することが、特徴再構成ロスを使う価値の核心です。

## メリット・トレードオフ・限界

メリット

- バックボーンや融合モジュールといった前段に濃い学習信号が届き、共有表現の質が上がります。
- 画素ではなく特徴空間で予測するので、無意味な細部の再現に労力を割きません。
- 推論時の予測誤差を不確かさ・異常検知に流用できます。
- 完全にオプトイン(任意有効化)で実装すれば、無効時は既存挙動と完全に同一に保て、後方互換性を壊しません。

トレードオフ・限界

- 学習時に「未来フレーム」が必要になり、連続フレームのデータ整備とメモリ・計算コストが増えます。単独画像のデータセットには使えません。
- 凍結バックボーンを別に持つぶんメモリを食います(EMA 版はさらにコピーの更新コストが加わります)。
- 重み lambda のチューニングが必要で、間違えると本タスク精度が犠牲になります。
- 予測と目標の特徴次元が一致しないことがあり、その場合は射影層(次元を合わせる学習可能な変換)が必要になります。特に、現在側のバックボーンと未来側のバックボーンが別物だと、出力チャンネル数が食い違いやすいです。
- 研究上の未解決課題として、未来の本質的な不確実性(マルチモーダルな未来)を MSE/L1 という単一点予測でどこまで扱えるかがあります。MSE は複数あり得る未来の平均に収束しがちで、確率分布として未来を予測する方が原理的に望ましい、という議論が続いています。

## 発展トピック・研究の最前線

- 確率的・マルチモーダル未来予測: MSE は「あり得る未来の平均」を予測してしまうため、拡散モデルや潜在変数で複数の未来候補を表現する方向(例: 拡散ベースの世界モデル)が研究されています。
- 世界モデルとしての発展: 予測モジュールを行動(操作)で条件づける(action conditioning)と、「この操作をしたら未来はどう変わるか」を予測する世界モデルになり、計画やシミュレーションに使えます。GAIA-1 や DriveDreamer など運転向け生成世界モデルがこの系譜です。
- EMA ターゲットとの組み合わせ: ターゲットを完全凍結にするか EMA で緩やかに更新するかで、表現の鮮度と安定性のトレードオフがあります(詳しくは [凍結バックボーン vs EMAターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0556_frozen-vs-ema-target-encoder))。
- 時間スケールの設計: 何秒先を何点予測するか(近接重視か長期重視か)は下流タスク次第で、時間重み付けの最適化自体が研究テーマになり得ます。
