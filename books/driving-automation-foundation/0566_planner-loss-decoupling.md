# 損失計算をプランナーから切り離し外部の軌道損失に一本化する設計

## ひとことで言うと
損失(loss)とは、モデルの予測が正解からどれだけズレているかを表す数値で、学習はこれを小さくする作業です。初期の設計では、軌道を出す各プランナーが「自分用の損失計算」を内部に抱えていました。本稿は、その内部損失を取り除き、損失計算をモジュールの外(学習ループ)にある共通の「軌道損失」に一本化する、という設計判断の一般論を扱います。すべてのプランナーが同じ形式で軌道を出すなら、損失も1つの共通関数で済みます。これによりプランナーを自由に差し替えても、学習ループも損失も変えずに済むようになります。

## 直感的な理解
レストランの厨房を想像してください。料理人(プランナー)は料理(軌道)を作るのが仕事です。ところが初期の設計では、料理人が「自分の料理を自分で採点する」ことまでやっていました。和食の料理人は和食の採点基準で、フレンチの料理人はフレンチの基準で、それぞれ自前の採点表を持っている。

これは問題です。料理人を入れ替えるたびに採点基準も変わり、店全体で味の方針(辛さの好み、塩加減の方針)を統一できません。お客さんへの提供時と採点時で、料理人が出すものの意味が変わる(あるときは料理、あるときは点数)、というのも分かりにくい。

そこで、採点係(損失モジュール)を厨房の外に1人置きます。料理人は料理を作ることだけに専念し、できあがった料理は全部この採点係が同じ基準で採点する。料理人をどう入れ替えても、採点の仕組みは1つ。味の方針も採点係が一元管理できる。これが損失計算の一本化です。前提として、すべての料理人が「同じ皿の形」で料理を出す(=すべてのプランナーが同じ形式で軌道を出す)ことが必要です。

## 基礎: 前提となる概念
用語を噛み砕きます。

- 損失関数(loss function)。予測と正解の差を1つの数値に変換する関数です。この数値が小さいほど予測が良い。学習はこの数値を下げる方向に重みを調整します。

- 模倣損失(imitation loss)。お手本(人間が運転した正解軌道)に予測軌道を近づける損失です。行動クローン(behavior cloning)の中核で、予測とお手本の差を L1 や L2 で測ります。

- L2 損失 / L1 損失 / Smooth L1。L2(二乗誤差、MSE)は差を二乗するので大きなズレに敏感ですが、外れ値に引きずられやすい。L1(絶対値誤差)は外れ値に頑健だが原点で微分が不連続。Smooth L1(Huber 損失)は、小さい誤差は二乗・大きい誤差は線形にして、両者の良いとこ取りをします。

- 関心の分離(separation of concerns)。「軌道を出す」関心と「軌道を採点する」関心を別の部品に分ける、というソフトウェア設計の原則です。

- 交換可能性(interchangeability)。プランナーを別の方式(例えば回帰型 ↔ 生成型)に差し替えても、周辺コードを変えずに動く性質です。

なぜ初期設計でプランナーが損失を持っていたか。プランナーの方式ごとに「最も自然な学習目的」が違うからです。回帰型のプランナーなら予測軌道とお手本の差(模倣損失)、生成型(Flow Matching など)なら速度場の予測誤差(velocity MSE)、というように。だから「各プランナーが自分の学習目的を内部に持つ」のは一見理にかなっていました。問題は、それが責務を混ぜ、契約を複雑にしたことです。

## 仕組みを詳しく
初期設計の問題を具体的に挙げます。

- 学習ループが「今どのプランナーが入っているか」を知らないと損失の扱いが変わり、呼び出し側が複雑になる。学習時だけ返り値の中身が「軌道」ではなく「スカラーの損失」になる、という分かりにくい契約が生じる。
- 損失のロジックがプランナーごとに散らばり、損失関数の種類・重みづけ・時間減衰といった方針を一箇所で管理できない。
- プランナーは本来「軌道を出す」のが仕事で、損失計算は学習側の関心。責務が混ざっている。

一本化の前提は、プランナーの出力を統一することです。異なる方式のプランナー(例えば生成型の Flow Matching と、制御点で曲線を表す Bezier 型)が、どちらも同じ形 `[B, T*C]`(B=バッチ、T=予測時刻数、C=信号数)の軌道を返すように揃えます。出力が同形式なら、損失も1つの共通関数で計算できます。

一本化後の流れ。
```
1. プランナーの forward(obs) は推論専用になり、軌道 [B, T*C] だけを返す
   (損失も内部状態も返さない)
2. 学習ループが、予測軌道と正解軌道(target)を
   共通の軌道損失関数 TrajectoryLoss に渡して損失を計算する
```

共通の軌道損失関数の中身を、典型的な実装例で具体的に見ます。
```
1. 予測と正解を [B, T, C] に整形する(例: T=64 ステップ × C=2 信号)
2. 要素ごとの誤差を計算。損失関数は Smooth L1 が既定(MSE も選べる)
3. 信号方向(例: 加速度・曲率)で平均し、各時刻の損失にする
4. 時間方向に重みをかける。temporal_decay=0.95 が既定なら重みは 0.95^t
   (t=0,1,2,...)。近い未来ほど重く、遠い未来ほど軽くなる
5. 全体を平均してスカラーの損失を返す
```

数値例で時間重みを理解します。T=4、temporal_decay=0.95 なら重みは `[0.95^0, 0.95^1, 0.95^2, 0.95^3] = [1.0, 0.95, 0.9025, 0.857...]`。t=0(0.1 秒後)の誤差は等倍、t=3 の誤差は約 0.86 倍に割り引かれます。temporal_decay=1.0 にすれば全ステップ均等です。なぜ近い未来を重くするか。直近の操作ミスの方が即座に危険につながるからです。遠い未来の軌道は不確実性が高く、多少ズレても再計画で修正できますが、0.1 秒後の操作は今すぐ安全に直結します。

この一本化により、プランナーを差し替えても(回帰型 ↔ 生成型)、学習ループも損失も一切変える必要がありません。プランナーは「軌道を出す」、損失モジュールは「軌道を採点する」と、責務がきれいに分かれます。

注意すべき設計上の論点として、プランナー方式に固有の学習特性があります。Flow Matching は本来「速度場の MSE」で学習する手法です。これを外部の軌道損失(最終的な軌道の差)だけで学習させると、生成過程の中間状態への学習信号が変わります。出力契約を揃えれば外部損失で扱えますが、各方式が本来想定する学習目的と外部軌道損失の整合は、設計時に検討すべき点です。

## 手法の系譜と主要論文
軌道の模倣損失と、損失をモデル外に置く設計の流れを追います。

ALVINN (Pomerleau, NeurIPS 1989)。ニューラルネットで運転を学んだ最初期の研究で、行動クローンの原型です。予測した操舵とお手本の差を最小化する、素朴な教師あり損失を使いました。損失は学習の外側にあり、ネットワークは出力を出すだけ、という今に通じる構図でした。

NVIDIA PilotNet (Bojarski et al., 2016, arXiv:1604.07316)。カメラ画像から操舵角を直接回帰する end-to-end 運転で、操舵角の L2 損失で学習しました。損失は単一で外部にある、というシンプルな形を実用規模で示しました。

CoverNet (Phan-Minh et al., CVPR 2020)。軌道予測を「あらかじめ用意した軌道集合からの分類」として定式化し、分類損失(クロスエントロピー)を使いました。これは損失の選択(回帰 vs 分類)が出力表現と密接に絡むことを示す例で、「出力表現を決めれば損失も決まる」という関係を浮き彫りにします。

TransFuser (Chitta et al., ICCV 2021, arXiv:2104.09224)。マルチモーダル融合で軌道を出す end-to-end 運転で、軌道の L1/L2 損失を外部で計算します。複数センサを融合してもプランナー出力は軌道に揃え、外部損失で学ぶ、という構成の代表例です。

Flow Matching (Lipman et al., ICLR 2023, arXiv:2210.02747)。生成モデルの学習法で、本来は「速度場の MSE(velocity MSE)」という方法固有の損失を持ちます。これを軌道生成プランナーに使うと内部損失が必要に見えますが、最終出力を軌道に揃えれば外部の単一損失でも扱えます。「方法固有の損失」と「共通の軌道損失」の境界を考える上で重要な例です。

系譜の要点は、「損失は単一・外部に置くのが基本(ALVINN, PilotNet, TransFuser)だが、出力表現や生成方式によって方法固有の損失が現れることもある(CoverNet, Flow Matching)」という緊張関係です。出力契約を軌道に統一すると、この緊張を外部一本化の側に倒せます。

## 論文の実験結果(定量データ)
損失一本化はアーキテクチャ整理であり単一論文の手法ではありませんが、背景にある損失設計の研究は定量結果を持ちます。

NVIDIA PilotNet (Bojarski et al., 2016)。実車・シミュレータで評価され、操舵角の L2 損失だけで、人間の介入なしに走行できる時間の割合(autonomy 指標、`autonomy = (1 - 介入回数 × 6秒 / 総走行時間) × 100%` という式で算出。1回の介入を人間が復帰させるのに約6秒かかるとみなす)で報告されました。論文では実走行でおよそ 98% 程度の autonomy を報告し、単一の外部損失でも実用的な end-to-end 運転が学べることを示した点が重要です。この 98% という数値の意味は、「1000 秒走って介入が数回以内」に相当し、当時としては素朴な単一損失でここまで到達できたことが驚きでした。

TransFuser (Chitta et al., ICCV 2021、および拡張版 TPAMI 2022)。CARLA の Longest6 / Town05 ベンチマークで、Driving Score(運転スコア、Route Completion × Infraction Penalty、100 点満点)と Infraction(違反、衝突・信号無視などの頻度)で評価されました。LiDAR と複数カメラの Transformer 融合 + 外部の軌道(waypoint)損失で、当時の純カメラ手法や late-fusion 手法を大きく上回り、拡張版では Longest6 で Driving Score がおよそ 70 前後(同条件のベースライン群がしばしば 30〜50 台にとどまるのに対し)を報告しています。アブレーションでは、補助タスク(深度・セマンティック・BEV セグメンテーション予測)を外部損失として加えると運転スコアと違反指標が悪化する(=補助損失を入れると改善する)ことを示しており、「出力は軌道に統一し、損失は外部で複数組み合わせる」設計の有効性を裏付けます。

CoverNet (Phan-Minh et al., CVPR 2020)。nuScenes(約 1000 シーン、6 自由度アノテーション付きの大規模運転データセット)の軌道予測で評価され、minADE_k(k 個の予測候補のうち最も正解に近いものの平均変位誤差、メートル)などの指標で報告されました。回帰ではなく「あらかじめ数千本用意した軌道集合からの分類(クロスエントロピー損失)」で扱うことで、minADE_5 などで当時の回帰ベースラインに匹敵しつつ、物理的にありえない軌道(横滑り・非現実的な急旋回)を出しにくくなる利点を示しました。何を測っているかというと、minADE は「5 本出して一番当たった1本がどれだけ正解に近いか(メートル単位、小さいほど良い)」で、分類化により候補がすべて実現可能な運動学に縛られる点が効きます。損失の選び方(回帰 vs 分類)が出力表現と一体で予測の質を変える、という具体例です。

時間重みづけの効果は多くの予測・計画研究で確認されており、近未来を重く評価すると、直近の操作精度(短期 ADE)が改善する傾向が報告されています。なぜ重要かというと、短期の精度は安全性に直結し、長期の軌道は再計画で補正できるため、限られた学習容量を近未来に配分するのが合理的だからです。

## メリット・トレードオフ・限界
メリット。
- 損失が1箇所に集約され、損失関数・時間重み・温度などを一元管理できる。
- プランナーの責務が「軌道を出す」だけになり、契約が単純(返り値が常に軌道)。
- プランナーを差し替えても学習ループ・損失を変えずに済む(交換可能性)。
- テストが単純化し、削除された内部損失のテストを抱えずに済む。

トレードオフ・限界。
- プランナー方式に固有の学習特性(例: Flow Matching の velocity MSE)を、外部の軌道損失だけで完全に再現できるかは設計に注意が要る。方式の学習特性と外部損失の整合を取る必要がある。
- 損失を外に出した分、学習ループ側に「どの損失をどう組み合わせるか」の責務が集中する。複数の損失(軌道 + 自己教師あり予測など)を足すとき、重み λ のチューニングが学習ループ側の負担になる。

研究上の未解決課題。軌道の表現(回帰の生値、制御点、分類候補、生成過程)ごとに最適な損失が異なるため、「出力を統一して外部損失に一本化する」ことと「各方式の表現力を活かす」ことのバランスは設計依存です。また、複数損失の重み付け λ を自動調整する手法も活発な研究テーマで、代表例として Kendall・Gal・Cipolla の不確実性重み付け(CVPR 2018、各タスクの観測ノイズ分散 σ² を学習し `L = Σ (1/2σ_i²) L_i + log σ_i` で重みを自動決定)や、Chen らの GradNorm(ICML 2018、各損失項の勾配ノルムを揃えるよう重みを更新)があります。前者は nuScenes 風のマルチタスク設定で手動チューニングした固定重みと同等以上の性能を、グリッドサーチなしで達成したと報告しており、損失を外部に一本化しておくとこうした自動重み付け機構を後付けしやすい、という実務上の利点につながります。

## 発展トピック・研究の最前線
近年は、軌道予測をマルチモーダル(複数の未来候補を出す)に扱い、それに応じた損失(勝者総取り損失=winner-takes-all、混合密度ネットワークの負対数尤度など)を使う流れが主流です。これらも基本は「プランナーは候補を出す、損失は外部で採点する」という分離の上に成り立ちます。

もう一つの最前線は、軌道損失に加えて自己教師あり予測損失(JEPA 系の特徴再構成など)を総損失に足す構成です。総損失 `L = L_traj + λ * L_jepa` のように複数目的を組み合わせるとき、損失を外部に一本化しておくと、新しい損失項を足すのが容易になります(関連は [反応系と世界予測系を分ける2バンドアーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1007_reactive-e2e-world-action-split))。
