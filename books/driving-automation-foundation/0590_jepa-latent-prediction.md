# JEPA(潜在空間での未来予測)

## ひとことで言うと

JEPA(Joint Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ)は、「次に何が起きるか」「隠された部分に何があるか」を、生の画素ではなく一度抽象化した数値表現(潜在空間)で予測する自己教師あり学習の枠組みです。画素を細部まで描き直すのではなく、「意味的に何が変わるか」だけを当てに行きます。これにより、影や質感やノイズのような予測しても意味のない細部を無視し、限られた計算資源を本質的な意味の理解に集中できます。Yann LeCun が「人間のような世界モデルを学ぶ道筋」として提唱し、画像版の I-JEPA、動画版の V-JEPA として具体化されました。生成モデル(画素を描く)とも対比学習(他サンプルとの対比)とも異なる第三の道として注目されています。

## 直感的な理解

天気予報にたとえます。明日の空を予測するのに、雲一粒一粒の形まで正確に描こうとする人はいません。「明日は雨が降りそう」「気温は20度くらい」といった意味レベルで当てれば十分です。雲の細部を完璧に描くのは不可能ですし、できても天気予報の役に立ちません。JEPA は同じ発想で、未来を「意味レベル(雨か晴れか)」で当て、「画素レベル(雲の形)」では当てません。

なぜ画素予測ではダメなのか。未来の画像を画素単位で当てさせると、モデルは歩道のタイル模様、雲の細かい形、カメラノイズまで再現しなければなりません。これらは運転判断にほぼ無関係なのに、モデルの容量と学習労力の大半を食います。さらに未来は本質的に不確実(歩行者が右に行くか左に行くか分からない)なので、画素を一点に決め打ちさせると、あり得る未来の平均をとった「ぼやけた無難な画像」しか出せなくなります。JEPA は潜在空間で予測することでこの両方を回避します。潜在空間には既に意味だけが残り細部は捨てられているので、意味的に妥当な未来を当てればよく、不要な細部にも不確実性にも煩わされにくいのです。

## 基礎: 前提となる概念

用語を噛み砕きます。

- 潜在空間 (latent space) / 埋め込み (embedding): 画像や文章を、ニューラルネットが扱いやすい数値ベクトルに圧縮した「内部表現」です。1枚の画像が例えば各位置 256 次元のベクトル群に変換され、その数値の並びが「このシーンの意味」を表します。
- 自己教師あり学習 (self-supervised learning): 人手のラベル(正解の付箋)無しで、データ自身から学習信号を作る方法。「画像の一部を隠して残りから当てる」「未来フレームを当てる」のように、データの構造そのものを問題に変えます。大量のラベルなしデータを活用できるのが強みです。
- 生成的予測 (generative prediction): 未来や欠損の画素そのもの(画像)を作り出す予測。MAE(Masked Autoencoder)などが代表で、隠した画素を復元します。
- 対比学習 (contrastive learning): 同じ画像の別バージョンは近く、別画像は遠くなるように埋め込みを学ぶ方法。多数のネガティブサンプル(別画像)が必要になりがちです。
- 表現崩壊 (representation collapse): 予測側と正解側の両方を学習可能にすると、「すべてを同じ定数ベクトルにすれば損失ゼロ」というズルが見つかり、特徴が無意味な一点に潰れる現象。JEPA はこれを防ぐ非対称設計が必須です(詳しくは [表現崩壊(representation collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0600_representation-collapse))。

JEPA は生成的でも対比的でもない第三の道です。画素を描かないので生成の無駄が無く、ネガティブサンプルも要らないので対比学習の大量サンプル問題も避けられます。代わりに「正解側を崩壊させない工夫」が必要になります。

## 仕組みを詳しく

JEPA は3つの部品からなります。

1. コンテキストエンコーダ (context encoder): 観測できる部分(現在のフレーム、画像の見えている領域)を潜在ベクトルに変換します。これは普通に学習(勾配が流れる)します。
2. ターゲットエンコーダ (target encoder): 予測したい部分(未来のフレーム、隠した領域)を潜在ベクトルに変換します。こちらは勾配を流さない(detach する)か、EMA(指数移動平均)でコンテキスト側の重みをゆっくり真似て更新します。学習可能にすると崩壊するため、この非対称性が要です。
3. 予測器 (predictor): コンテキストの潜在ベクトルから、ターゲットの潜在ベクトルを予測します。比較的小さなネットワークで、「位置の手がかり(どこを予測するか)」を受け取って対応する潜在を出します。

学習は「予測器が出した潜在ベクトル」と「ターゲットエンコーダが出した実際の潜在ベクトル」を近づける(MSE や L1 を最小化する)ことで行います。

```
観測          予測したい部分
[現在/見える部分] -> context encoder -> z_ctx
                                          |
                                          v
                                      predictor -> z_hat  --比べる(MSE/L1)--+
                                                                            |
[未来/隠した部分] -> target encoder(凍結/EMA) -> z_tgt --detach-------------+
```

I-JEPA の例では、1枚の画像の一部を「コンテキスト」、別の複数領域を「ターゲット」とし、コンテキストの潜在からターゲット領域の潜在を予測させます。生成系(画素を描く MAE)と違い、画素を一切再構成しないのが要点です。

形状の具体例。コンテキストが `[B, N_ctx, D]`、ターゲットが `[B, N_tgt, D]`(N はパッチ数、D は埋め込み次元)のとき、predictor は各ターゲット位置の埋め込み `[B, N_tgt, D]` を出力し、ターゲットエンコーダの `[B, N_tgt, D]` と要素ごとに比較します。数値例として D=768、ターゲットを4領域・各16パッチ取るなら N_tgt=64 で、予測も正解も `[B, 64, 768]` のテンソルになり、その差の二乗平均が損失です。

崩壊を防ぐ仕組みを掘り下げます。もしターゲットエンコーダも勾配で更新できたら、両者が「常にゼロベクトルを出す」ことを学べば損失はゼロになり、何も意味のない解に落ちます。これを防ぐのが (a) ターゲット側の勾配停止、(b) EMA によるゆっくりした更新(予測側が先に動き、ターゲットが遅れて追う非対称性が崩壊を抑える)、(c) 予測器という追加の非対称な部品、の組み合わせです。BYOL がこの非対称設計でネガティブ無しの崩壊回避を実証し、JEPA がそれを予測タスクへ一般化しました。

## 手法の系譜と主要論文

- LeCun の JEPA 構想 ("A Path Towards Autonomous Machine Intelligence", 2022): 知能の実現には「世界モデルを潜在空間で予測的に学ぶ」ことが鍵だと主張しました。生成モデルは不確実な未来を一点に決め打ちして失敗するため、潜在空間で抽象的に予測すれば不確実性を吸収できると考えました。これは枠組みの提案で、具体実装が後続です。

- BYOL (Grill et al., NeurIPS 2020, arXiv:2006.07733): 予測側とターゲット側を非対称にし、ターゲットを EMA で更新することでネガティブ無しでも崩壊しないことを示しました。JEPA の崩壊回避設計の直接の祖先です。

- I-JEPA (Assran et al., CVPR 2023, arXiv:2301.08243): 画像で JEPA を実装。画像の一部から他領域の潜在表現を予測させ、ラベル無しで良い表現を学習しました。画素再構成型(MAE)より少ない計算で線形評価の高精度を達成しました。ターゲットエンコーダの設計(EMA 係数など)に敏感で、崩壊回避の工夫が必須というトレードオフも報告しています。

- V-JEPA (Bardes et al., 2024, arXiv:2404.08471): 動画に拡張し、時間的に隠した部分の潜在を予測。L1 損失も有効と報告しました。動画は時間方向の予測が自然に「未来予測」になるため、自動運転のような時系列タスクと相性が良いという示唆を与えます。

- V-JEPA 2 (Assran et al., 2025): 大規模動画事前学習から、行動条件づけの世界モデルへ発展させ、ロボット操作の計画にゼロショットで使える方向を示しました。「潜在世界モデルで計画する」という LeCun の構想を実機タスクへ近づけた最新の到達点です。

- MILE (Hu et al., NeurIPS 2022, arXiv:2210.07729): 自動運転で潜在世界モデルを学習し、画素予測ベースラインを上回りました。運転では意味的変化(車の動き)が重要で路面の細部は不要だという、JEPA 的予測の自動運転での有効性を実証した重要な事例です。

系譜をまとめると、BYOL(崩壊回避の非対称設計)→ I-JEPA(画像での予測アーキテクチャ確立)→ V-JEPA / V-JEPA 2(動画・行動条件・計画への拡張)という流れで、並行して MILE などが自動運転ドメインで「潜在予測が効く」ことを示し、両者が世界モデル研究で合流しています。

## 論文の実験結果(定量データ)

- I-JEPA: ViT-Huge/14 を ImageNet で自己教師あり学習し、線形評価(特徴の上に線形分類器だけ載せる)で高精度を達成。重要なのは効率で、画素再構成の MAE に比べ、同等以上の表現品質をおよそ数分の一の事前学習計算で得たと報告されています。これは「画素を描かないぶん無駄が無い」という主張の定量的裏付けです。また、低レベルの細部(物体数え上げや深度予測のような局所タスク)でも MAE に競合する結果を示し、抽象化しても局所情報が失われすぎないことを確認しました。

- V-JEPA: Kinetics-400(動画行動認識)や Something-Something-v2(時間的推論を要する行動認識)で、凍結評価(特徴を固定して分類器だけ学習)において強い性能を示しました。特に時間的な因果が重要な Something-Something で画素予測系を上回り、「動画の意味的・時間的構造を捉えられる」ことを示しました。

- MILE: CARLA シミュレータの closed-loop 評価で、潜在世界モデルが模倣学習単独や画素予測ベースラインを上回る運転スコアを記録しました。アブレーションで潜在予測を画素予測に置き換えると性能が落ちることが示され、潜在空間で予測することの寄与が確認されています。

- BYOL: ImageNet 線形評価で Top-1 約74.3%を達成し、ネガティブサンプルを使う対比学習(SimCLR 約69.3%)を上回りました。ネガティブ無しでも崩壊しない非対称設計の有効性を実証し、JEPA 系全体の前提を支えています。

指標の意味を補足します。線形評価・凍結評価の精度は「学習した特徴がどれだけ意味を捉えているか」の代理指標で、特徴が良いほど単純な分類器でも高精度になります。事前学習計算量の比較は「同じ品質を得るのにどれだけ計算が要るか」を測り、JEPA の効率性の核心です。closed-loop 運転スコアは実際に走らせた質を測り、open-loop(録画に対する予測誤差)より厳しく、世界モデルの実用性を直接評価します。

## メリット・トレードオフ・限界

メリット

- 意味のない画素細部(影・ノイズ・テクスチャ)を予測しなくて済み、計算と容量を節約できます。
- 未来の不確実性に強いです。一点の画素に決め打ちせず潜在空間で抽象的に当てるため、ぼやけた無難画像問題を避けられます。
- ラベル無し(自己教師あり)で良い表現を学べ、大量のラベルなしデータを活用できます。

トレードオフ・限界

- ターゲットを学習可能にすると表現崩壊が起きるため、凍結/EMA などの非対称設計が必須です(→ [表現崩壊(representation collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0600_representation-collapse))。設計が崩れると学習が無意味な解に落ちます。
- 潜在空間で予測するため、人間が「何を予測したか」を画像として直接見られず、解釈性が下がります。デバッグや検証がしにくくなります。
- EMA 係数・予測器の容量・マスク(隠す領域)の取り方など、ハイパーパラメータに敏感で調整が難しいです。
- 評価の難しさ: 潜在空間の予測品質そのものを直接測る良い指標が確立しておらず、下流タスクの精度で間接的に評価せざるを得ません。これは研究上の未解決課題です。
- 真にマルチモーダルな未来(複数のあり得る展開)を、決定的な潜在予測でどこまで表現できるかは依然として議論があり、確率的な拡張が模索されています。

## 発展トピック・研究の最前線

- 行動条件づけ世界モデル: 予測を行動(操作)で条件づけると「この操作をしたら未来の潜在はどうなるか」を予測でき、計画やシミュレーションに使えます。V-JEPA 2 やロボット制御研究がこの方向です。
- 自動運転への適用: 潜在世界モデルを使って「危険そうな未来」を内部で先読みし、計画や異常検知に活かす研究が進んでいます。GAIA-1 や DriveDreamer のような生成世界モデルと、JEPA 流の潜在予測のどちらが運転計画に有利かは未確定の論点です。
- 階層的・長期予測: 数秒先まで安定して予測するには時間階層(粗い長期予測と細かい短期予測の組み合わせ)が必要とされ、階層 JEPA の研究が始まっています。
- マルチモーダル JEPA: 画像・動画だけでなく、点群やテキストを含む複数モダリティを共通潜在で予測する拡張も模索されています。
