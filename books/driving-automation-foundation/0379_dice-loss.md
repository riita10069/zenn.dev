# Dice 損失

## ひとことで言うと
画像の各ピクセルを「対象物(前景)」か「それ以外(背景)」かに塗り分けるタスク(セグメンテーション)で、予測した塗り分けと正解の塗り分けが「どれだけ重なっているか」を直接最大化する損失関数です。対象物が画面のごく一部しか占めないとき(クラス不均衡が激しいとき)に特に強く、医療画像や自動運転の小さな物体の領域分割で広く使われます。重なり度合いを測る古典的な Dice 係数を、ニューラルネットの学習に使えるよう微分可能な形に直したものです。

## 直感的な理解
セグメンテーションを学習させる素朴な方法は、ピクセルを1個ずつ「正解クラスか否か」で採点すること(クロスエントロピー損失)です。ところがこれは、前景が小さいときに破綻します。

たとえば 512×512 ピクセル(約26万ピクセル)の前方カメラ画像で、歩行者が占めるのはせいぜい 2000 ピクセルだとします。前景:背景 = 2000:260000 ≒ 1:130 という極端な偏りです。このとき「全部背景だ」と予測するだけで、26万ピクセルのうち25万8千を正解にでき、ピクセル正解率は 99% を超えます。しかし肝心の歩行者は1人も検出できていません。クロスエントロピーの合計は背景の正解で稼がれてしまい、モデルは「とりあえず背景と言っておけば損失が下がる」と学習してしまうのです。自動運転では歩行者の見落としは致命的なので、この挙動は許されません。

そこで発想を変えます。「ピクセルを1個ずつ数える」のではなく「予測した前景領域と正解の前景領域が、面積としてどれだけ重なっているか」を測りたい。重なりだけに注目すれば、背景がいくら多くてもスコアは前景の当たり具合だけで決まります。この重なり度合いを測る古典的な指標が Dice 係数で、それを最大化する形に直したのが Dice 損失です。「面積の一致を直接最適化する」という発想は、最終的に評価したい指標(IoU や Dice)そのものを学習目標にしてしまう、という意味でも自然です。

## 基礎: 前提となる概念
- セグメンテーション(segmentation、領域分割): 画像の各ピクセルを「道路」「車」「背景」などのクラスに分類するタスク。出力は入力画像と同じ解像度のラベルマップ(各ピクセルにクラス番号が付いた画像)になる。
- 前景(foreground)と背景(background): 関心のある対象物が前景、それ以外が背景。二値セグメンテーションでは前景=1、背景=0。
- クラス不均衡(class imbalance): あるクラスのピクセル数が他クラスより極端に多い/少ない状態。前景が小さいほど不均衡が激しい。
- クロスエントロピー損失(cross-entropy loss): 各ピクセルが正解クラスである確率の対数を取って和を最小化する分類損失。ピクセルごとに独立に評価するため、ピクセル数の多いクラスに支配されやすい。
- IoU(Intersection over Union、和集合に対する積集合の比): 2領域の重なり面積を、2領域を合わせた面積で割った値。セグメンテーション評価の代表指標。Dice はこれと数学的に近い親戚。
- F1 スコア / 適合率・再現率: 適合率(precision、予測した前景のうち本当に前景だった割合)と再現率(recall、本当の前景のうち拾えた割合)の調和平均が F1。Dice 係数はこの F1 と数学的に同一である。
- 微分可能(differentiable): 勾配(各パラメータを少し動かしたとき損失がどう変わるか)を計算できること。ニューラルネットの学習は勾配降下に頼るので、損失は微分可能でなければならない。

## 仕組みを詳しく
Dice 係数は、2つの領域 A(予測した前景)と B(正解の前景)の重なりを次で測ります。

```
Dice = 2 * |A ∩ B| / (|A| + |B|)
```

`|A ∩ B|` は両方が前景と言っているピクセル数(重なり)、`|A|`・`|B|` はそれぞれの前景ピクセル数です。値は 0(全く重ならない)から 1(完全一致)まで。「重なった部分」を「両者の面積の平均」で割っている、と読むと直感的です。重なりが大きいほど1に近づきます。

なお Dice 係数は二値分類の F1 スコアと数学的に同一です。IoU との関係は `Dice = 2·IoU / (1 + IoU)` で、単調増加の対応がつきます(IoU が上がれば Dice も必ず上がる)。したがって Dice を最大化することは IoU を最大化することとほぼ等価です。

学習に使うには微分可能にする必要があります。予測を 0/1 の硬い値ではなく、シグモイドやソフトマックスを通した確率 p(0〜1の連続値)として扱い、掛け算で「ソフトな重なり」を表します。これを「ソフト Dice」と呼びます。

```python
def dice_loss(pred, target, eps=1.0):
    # pred:   モデルの前景確率 (シグモイド後), 形状 [B, H, W], 値 0〜1
    # target: 正解マスク (0 or 1),            形状 [B, H, W]
    pred = pred.flatten(1)        # [B, H*W]
    target = target.flatten(1)    # [B, H*W]

    intersection = (pred * target).sum(-1)          # ソフトな重なり [B]
    denom = pred.sum(-1) + target.sum(-1)           # |A| + |B|     [B]
    dice = (2 * intersection + eps) / (denom + eps) # [B], 範囲 0〜1
    return (1 - dice).mean()                        # 損失なので 1 から引く
```

ステップで追います。

1. 予測確率 `pred` と正解 `target` を `[B, H*W]` に平らに伸ばす。
2. `pred * target` を全ピクセルで合計 → ソフトな重なり。両方が前景に近い(どちらも1に近い)ピクセルだけが寄与する。
3. `pred.sum + target.sum` で面積の合計を出す。
4. Dice 係数を計算。`eps`(平滑化項、例: 1.0)は分母が 0 になるのを防ぎ、前景がゼロの画像でも安定させる。分子にも足すと「前景が無い画像で予測も無ければ Dice=1(完璧)」と扱える。
5. `1 - dice` で損失に変換(完全一致なら 0)。

数値例。3ピクセルの極小例で、正解 `target=[1,1,0]`、予測 `pred=[0.9, 0.8, 0.2]` とします。

```
intersection = 0.9*1 + 0.8*1 + 0.2*0 = 1.7
denom        = (0.9+0.8+0.2) + (1+1+0) = 1.9 + 2.0 = 3.9
dice(eps=0)  = 2*1.7 / 3.9 = 3.4/3.9 ≈ 0.87
loss         = 1 - 0.87 = 0.13
```

ここがポイントです。Dice 損失は前景ピクセル数で正規化されているので、背景がいくら多くても損失の大きさは前景の当たり具合で決まります。「全部背景」と予測すると intersection=0、dice≈0、loss≈1 と最大の罰を受け、手抜きが効かなくなります。これが不均衡に強い理由です。

勾配の性質も理解しておく価値があります。クロスエントロピーは各ピクセルで局所的に勾配が定まる(あるピクセルの損失はそのピクセルの予測だけで決まる)のに対し、Dice 損失は分母に「画像全体の前景面積の和」を含むため、あるピクセルの勾配が画像全体の前景量に依存します(非局所的)。具体的には、ソフト Dice の前景確率 p_i に関する偏微分はおよそ `∂L/∂p_i = -2 · (g_i · denom - 2 · intersection) / denom²`(g_i は正解値)の形になり、denom が小さい(前景が小さい)ほど 1 ピクセルあたりの勾配が大きくなります。これにより小さな前景に勾配が集中し不均衡に強くなる一方、前景が極端に小さい/存在しない画像では分母が小さく勾配が暴れやすい、という副作用も生まれます。実務では学習初期の安定のためクロスエントロピーと足し合わせた複合損失(`loss = ce + dice`、いわゆる Combo Loss)がよく使われます。

## 手法の系譜と主要論文
Dice 係数自体は生態学・分類学で古くから知られていました。Dice (1945) と Sørensen (1948) が独立に「2つの種の集合の類似度」を測る指標として提案したもので、当初は機械学習とは無関係の集合類似度でした(英語圏では Sørensen–Dice coefficient とも呼ばれる)。

深層学習の損失関数としてこれを再発見し広めたのが V-Net(Milletari, Navab & Ahmadi, 3DV 2016, arXiv:1606.04797)です。前立腺の 3D MRI セグメンテーションを対象とし、臓器が体積のごく一部しか占めずクラス不均衡が深刻という問題に直面しました。クロスエントロピーでは前景が学習されないため、Dice 係数を直接最適化する目的関数(soft Dice)を導入し、不均衡データでも前景を捉えられることを示しました。これが医療画像分野で「Dice を損失にする」流れの出発点になりました。

その後、複数クラスへの拡張として Generalised Dice Loss(Sudre et al., DLMIA 2017, arXiv:1707.03237)が提案されました。クラスごとに「そのクラスの面積の逆数の二乗」で重み付けすることで、小さいクラスほど重みを大きくし、複数の不均衡クラスを同時に扱えるようにしました。これは Crum et al. の Generalised Tanimoto coefficient の発想を損失に持ち込んだものです。

さらに偽陽性(false positive、誤って前景と言う)と偽陰性(false negative、見落とし)のバランスを制御したい要求から Tversky Loss(Salehi et al., MLMI 2017, arXiv:1706.05721)が登場しました。Dice の分母を `TP + α·FP + β·FN` の形に一般化し、α と β で重みを変えられます(α=β=0.5 で Dice に一致)。医療では見落としを特に嫌うため β を大きく取ります。続く Focal Tversky Loss(Abraham & Khan, ISBI 2019, arXiv:1810.07842)は Tversky 指標に Focal 機構(`(1 - TI)^γ` で難しいサンプルを強調)を組み合わせ、極小病変の検出を改善しました。

別系統として Lovász-Softmax(Berman, Triki & Blaschko, CVPR 2018, arXiv:1705.08790)があります。これは IoU 損失(Jaccard 損失)のサブモジュラ性を利用し、IoU を凸下界(Lovász 拡張)で直接代理最適化するもので、Dice の近似ではなく IoU そのものに対するより厳密な代理損失を与えます。これらは現在も医療画像・小物体セグメンテーションの標準的な選択肢として並存しています。

## 論文の実験結果(定量データ)
V-Net 論文では PROMISE2012(前立腺 MRI ベンチマーク)で評価しました。評価指標は Dice 係数そのもの(高いほど良い、最大1)と、境界面の距離指標(Hausdorff 距離など、低いほど良い)です。V-Net は Dice 係数およそ 0.87 前後を達成し、当時の手法と比べて競争力のある精度を、しかも 3D ボリュームを一気に処理する効率で得たと報告されています。重要なのは「Dice を損失にしたことで、不均衡の強い前立腺領域を再重み付けのハイパーパラメータ調整なしに学習できた」という定性的知見です。クロスエントロピーでは前景クラスへの重み付けを手で調整しないと前景が消える問題があり、Dice 損失はその調整を不要にしました。

Tversky Loss 論文では多発性硬化症(MS)の病変セグメンテーション(MRI)で評価し、Dice 損失と比べて Dice スコアと再現率(sensitivity)を改善したと報告されています。具体的には β を大きく(偽陰性を重く)取ることで、Dice 単体に比べ病変の見落としが減り、F2 的な指標で優位になりました。これは「前景が小さく見落としコストが高いタスクでは、適合率と再現率のバランスを損失レベルで調整できることが効く」という知見を定量化したものです。

その後の比較研究(セグメンテーション損失のサーベイ的論文群)では、前景比率が数%を切るような強い不均衡タスクで、クロスエントロピー単体に比べ Dice 系損失が IoU/Dice 指標を数ポイント〜十数ポイント押し上げる例が多数報告されています。一方で前景が比較的大きく境界の精度が重要なタスクでは、Dice 単体は境界がやや甘くなり、クロスエントロピーとの複合(`CE + Dice`、Combo Loss)が最も安定するという結果が繰り返し示されています。アブレーションの典型は「Dice を外してクロスエントロピーだけにすると小物体の再現率が大きく落ちる」「eps(平滑化項)を外すと前景なし画像で損失が発散する」「Lovász-Softmax にすると Dice より mIoU が 1〜2 ポイント高くなるが学習が遅くなる」というものです。

## メリット・トレードオフ・限界
メリット
- クラス不均衡(前景が小さい)に非常に強い。前景ピクセル数で正規化されるため、背景の量に左右されない。
- 評価指標(IoU/Dice)を直接最適化するので、学習目標と最終評価が一致する。
- 「全部背景」のような手抜き解を許さない。
- クラス重みのハイパーパラメータ調整がほぼ不要(V-Net の主張)。

トレードオフ・限界
- 前景が極端に小さい/存在しない画像で勾配が不安定になりやすい(eps とクロスエントロピー併用で緩和)。分母が小さいと勾配が爆発する。
- 領域の重なりを重視するため、細かい輪郭・境界の精度が甘くなることがある。境界が重要なら境界損失(Boundary Loss)や CE との併用が要る。
- 多クラスでは単純な Dice では各クラスの不均衡を扱いきれず、Generalised Dice などの拡張が必要。
- soft Dice は非凸で最適化地形が複雑。学習初期の不安定さは未解決の実務課題で、warmup や複合損失で対処するのが通例。
- 偽陽性と偽陰性を等価に扱うため、応用によっては Tversky への置き換えが必要。
- バッチやサンプルをまたいで Dice を集計する単位(画像ごとに Dice を取って平均するか、バッチ全体で1つの Dice を取るか)で値と勾配が変わり、再現性に注意が要る。

## 発展トピック・研究の最前線
損失関数の選択は「タスクの不均衡度と境界の重要度で決まる」という理解が定着し、Combo Loss、Boundary Loss(境界の距離を直接損失にする)、Lovász-Softmax(IoU を凸近似で直接最適化)、Unified Focal Loss(Focal・Tversky・Dice を1つの枠組みに統合)など、Dice の弱点を補う系統が研究されています。Transformer ベースのセグメンテーション(Mask2Former, Cheng et al., CVPR 2022, arXiv:2112.01527 など)では、マスク予測の損失として Dice とクロスエントロピー(またはバイナリ CE)の加重和が標準的に採用されており、Dice 系損失は現在も生きた基盤技術です。自動運転の俯瞰図(bird's-eye-view)セグメンテーションや占有予測(occupancy)では、小さく偏った前景(車・歩行者・障害物)を学ぶのに Dice 系が有力候補になり、しばしば Focal Loss と組み合わせて使われます。研究の最前線では「どの損失が最良か」は単一の答えがなく、データの不均衡度・境界の重要度・キャリブレーション(予測確率の信頼性)の要求に応じて選ぶ、という設計知が蓄積されつつあります。
