# 軌跡計画ヘッド（ego query + 再帰デコード + deformable cross-attention）

## ひとことで言うと

自動運転モデルの一番末端で「これから自車が数秒間どう動くか」という未来の軌跡を出力する部品です。車の周りを上空から見下ろした地図のような特徴マップ(鳥瞰図)の中から、運転判断に効きそうな場所だけを学習で選んで参照し、未来を1ステップずつ生成していきます。中心となる道具立ては3つあります。自車を表す問い合わせベクトル(ego query)、時系列を1点ずつ生成する再帰ネットワーク、そして見る場所を入力に応じて動的にずらせる注意機構(deformable cross-attention)です。これらを組み合わせることで、特徴マップの解像度に依存せず、なめらかで物理的に妥当な軌跡を効率よく出せます。

## 直感的な理解

人間が運転するとき、視界の全部を均等に注視しているわけではありません。前方の先行車、合流してくる車、横断しそうな歩行者など、「いま運転判断に効くポイント」に視線を素早く移しながら、次にどう動くかを連続的に決めています。信号が赤になりそうなら前方の信号を、車線変更するならミラーの中の後続車を、というように、目的に応じて注視点が動きます。

ニューラルネットの計画ヘッドにも同じ発想が要ります。車の周囲を表す特徴マップは大きく(縦横数百セル × 各セル数百次元)、その全部を平等に処理するのは無駄が多く非効率です。そこで「自分はどこを見るべきか」という問い(query)を持たせ、その問いに応じて特徴マップの一部だけを参照(attention)し、参照した情報から次の動きを決める。これを未来に向かって何度も繰り返せば、なめらかな軌跡が描けます。

なぜ「1ステップずつ」生成するのか。未来の軌跡は連続したもので、t秒後の位置は t-0.1秒後の位置と速度から自然に決まります。直前の状態を引き継ぎながら次を出す再帰的な生成は、この連続性を素直に表現できます。「いま減速し始めたなら、次の瞬間も減速の続き」という物理的な滑らかさが、状態を持ち越す再帰構造から自動的に生まれます。

## 基礎: 前提となる概念

このノートを読むのに必要な用語を、知らない人向けに噛み砕きます。

- 鳥瞰図(BEV: Bird's-Eye-View): 車を真上から見下ろした座標系で表した特徴マップのことです。複数カメラの情報を1つの地図に統合したもので、形は `[B, C, H, W]` と表します。Bはバッチ(同時に処理するサンプル数)、Cはチャンネル(各セルが持つ特徴ベクトルの次元数)、H×Wがグリッドの大きさです。作り方は[鳥瞰図特徴融合（BEVFormer風 空間クロスアテンション）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0299_bev-fusion)を参照してください。
- query(問い合わせベクトル): 「自分はどんな情報が欲しいか」を表すベクトルのことです。attention機構が「どこを見るか」を決める起点になります。
- key / value: attentionにおける情報源です。queryと照合される索引が key、実際に取り出される中身が value です。
- attention(注意機構): query(問い)と key/value(情報源)を照合し、関連の強い場所から情報を重み付きで取ってくる仕組みのことです。Transformerの中核で、「どこに注目するか」を学習で決めます。
- cross-attention: query と、それとは別物(ここでは鳥瞰図特徴)との間で行う attention のことです。自分自身の中で行う self-attention と区別されます。
- deformable(変形可能): 見る場所を固定せず、入力に応じて動的にずらせる、という意味です。Deformable DETR(Zhu et al., 2021)が画像物体検出で導入しました。
- 自己回帰(autoregressive): 1つ前の出力を次の入力に使って順番に生成するやり方のことです。文章生成や時系列予測で広く使われます。
- 再帰ユニット(RNN / GRU / LSTM): 過去の状態を「隠れ状態(hidden state)」として持ち越しながら時系列を処理する小さなネットワークです。GRU(Gated Recurrent Unit、Cho et al., 2014)はゲート(情報の流れを開閉する仕組み)で取捨を制御する軽量版です。
- ウェイポイント(waypoint): 軌跡を構成する1点のことです。位置 `(x, y)` で表すこともあれば、加速度・曲率などの制御量で表すこともあります。
- 学習可能(learnable): 誤差逆伝播(backpropagation、出力の誤差を入力側へ伝えて各重みをどう変えれば誤差が減るか計算する手続き)で、訓練中にその値そのものが更新されることを指します。固定値ではありません。
- オープンループ / クローズドループ評価: オープンループは「予測軌跡を実際の運転に反映させず、人間の運転記録と比べるだけ」の評価です。クローズドループは「予測軌跡で実際に車(またはシミュレータ内の車)を動かし、その結果生じた新しい状況でまた予測させる」評価です。後者の方が実用性能に近いですが計算コストが高いです。

## 仕組みを詳しく

典型的な実装では、計画ヘッドは鳥瞰図特徴 `bev_features: [B, C, H, W]` と自車運動履歴 `egomotion_history: [B, D]` を受け取り、未来の軌跡を返します。代表的な設定として、チャンネル数 C=256、未来ステップ数 64、各ステップ 0.1秒(10Hz)で 6.4秒先まで、というものがあります。以下、この設定を例に段階を追います。

### ステップ1: ego query の用意

`ego_query` は要素1個・次元256の学習可能ベクトル(実装上は `nn.Embedding(1, 256)` のように作る)です。これは「自車」を表す問い合わせで、「鳥瞰図のどこを見て運転を決めればよいか」を訓練を通じて獲得します。最初はランダムな値ですが、損失を下げる方向に更新されるうちに「運転判断に効く場所を引き出す問い」へと育っていきます。物体検出のDETRにおける object query(各 query が「1つの物体」に対応する学習可能ベクトル)の自車版だと考えると分かりやすいです。物体検出では複数の query を並べて多数の物体を拾いますが、計画では「自車1つ」を表す query が1本あれば足ります。

### ステップ2: 再帰ユニットの初期化

GRUの隠れ状態 `h` を、自車運動履歴から初期化します。

```
h = ego_state_proj(egomotion_history).unsqueeze(0)   # [1, B, 256]
```

`ego_state_proj` は線形層(例えば `Linear(D, 256)`)で、自車の速度・加速度・操舵などの運動状態をGRUが扱える次元に射影します。「いまどんな運動状態か」を出発点にすることで、生成される軌跡が現在の運動と連続します。これをしないと、停止状態から急に高速の軌跡が生成されるような物理的に不連続な出力が起きやすくなります。

### ステップ3: 64ステップの自己回帰ループ

ここが中心です。num_timesteps=64 回、次を繰り返します。

各ステップ t で:

```
query    = h.squeeze(0) + ego_q              # [B, 256]  現在の隠れ状態に ego query を加算
attended = deformable_cross_attn(query, bev) # [B, 256]  鳥瞰図から関連特徴を取得
_, h     = gru(attended, h)                  #           取得特徴で隠れ状態を1ステップ更新
waypoints.append(waypoint_head(h))           # [B, 2]    この時刻のウェイポイントを出力
```

`waypoint_head` は `Linear(256, num_signals)` で、num_signals=2 のときは2つの信号、例えば加速度(acceleration)と曲率(curvature, ハンドルの曲がり具合)を出します。位置 `(x, y)` を直接出す設計もありますが、制御量を出して運動モデル(自転車モデルなどの車両ダイナミクス)で積分して位置に変換する方が、物理的に実現可能ななめらかな軌跡になりやすいという利点があります。逆に位置を直接出すと、車両が物理的に追従できない急な横移動を含む軌跡が出てしまうことがあります。

最後に64ステップ分を連結すると `64 × 2 = 128` 次元のベクトルになり、これが軌跡 `trajectory: [B, 128]` です。GRUの最終隠れ状態 `ego_hidden: [B, 256]` も返し、これは下流の補助タスク(将来状態の予測など、[特徴空間で未来を予測するヘッド(JEPA 風 future-state head)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0011_future-state-jepa-loss)参照)が「自車が世界をどう理解したか」の要約として使えます。

数値例で流れを追うと、最初のステップでは隠れ状態は現在の運動状態そのものです。query で鳥瞰図の「すぐ前方」を見て「先行車が近い→減速」を読み取り、加速度を負に出す。次のステップではその隠れ状態を引き継ぎ、もう少し先を見て…という具合に、視線と判断が前方へ進んでいきます。減速を始めた状態が隠れ状態に記憶されるため、後続ステップでもその減速トレンドが維持されます。

### ステップ4: deformable cross-attention の中身

「鳥瞰図のどこを見るか」を動的に決める部分です。固定の格子を見るのではなく、query に応じてサンプリング点をずらします。

```
ref_point = reference_point(query).sigmoid()              # [B, 2]   基準点(0〜1の正規化座標)
offsets   = sampling_offsets(query).reshape(B, K, 2) * s  # [B, K, 2] 基準点周りのK個のずれ
attn_w    = attention_weights(query).softmax(-1)          # [B, K]   各点への注目度(合計1)
sampled   = grid_sample(values, ref_point + offsets)      # [B, K, C] 双線形補間で特徴取得
attended  = (sampled * attn_w).sum(dim=1)                 # [B, C]   重み付き合計
```

各記号の意味:
- `reference_point(query).sigmoid()`: query から「鳥瞰図上のどこを見るか」の基準点を予測し、sigmoid(値を0〜1に圧縮する関数)で正規化座標にします。0が左端・1が右端というふうに、グリッドの絶対サイズに依存しない座標で表すのがポイントです。
- `sampling_offsets`: 基準点の周りにK個(例えば8個)のサンプリング点をどれだけずらすかを予測します。`offset_scale` s(例えば0.1)を掛けてずれを基準点周辺に制限し、学習初期の発散を防ぎます。s が大きすぎると初期にサンプリング点が画面外へ飛んで勾配が消え、学習が進まなくなります。
- `attention_weights`: K点それぞれにどれだけ注目するかの重みです。softmax(合計が1になるよう正規化する関数)を通すので、K点の注目度の合計は必ず1になります。
- `grid_sample`: K個の連続座標で鳥瞰図特徴を双線形補間(近傍4セルを距離で重み付け平均)して取り出します。標準演算なのでカスタムCUDAカーネル不要で可搬です。詳しくは[grid_sampleによるdeformable cross-attention](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0303_deformable-cross-attention-grid-sample)を参照してください。

この仕組みのおかげで、計画ヘッドは鳥瞰図の全部(H×W個のセル)を平等に処理する代わりに、K個の点だけを見ます。H×Wが13万を超えても見る点はK個のままで、計算量がグリッドサイズに依存しません。これがdeformable attentionの本質的な利点で、後述する旧来の[flatten+MLP型の計画ヘッドとその構造的限界](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0118_driving-policy-flatten-mlp-legacy)がグリッドサイズの2乗でパラメータ爆発するのと根本的に異なります。

### なぜこの設計なのか

- 解像度非依存: 鳥瞰図を高解像度にしても、計画ヘッドはサンプリング点の正規化座標を変えるだけで動きます。ヘッドを作り直す必要がありません。
- 疎なアクセス: 全セルを見る密な attention は計算量がグリッドサイズに比例して増えますが、deformable はK点固定です。グリッドが大きくなっても見る点数は変わりません。
- 連続性: 再帰デコードで直前の状態を引き継ぐため、軌跡が物理的になめらかになります。
- 情報の集約点: 最終隠れ状態 `ego_hidden` が「自車視点のシーン要約」になり、補助タスクや解釈の足がかりになります。

## 手法の系譜と主要論文

計画ヘッドの設計は、いくつかの研究の流れが合流したものです。順を追って系譜を描きます。

時系列予測のRNN系譜から。Social-LSTM(Alahi et al., CVPR 2016)は歩行者の軌跡を、各人にLSTMを割り当て、隠れ状態を近傍と共有(social pooling)しながら1点ずつ将来位置を生成する枠組みを提案しました。これは「1ステップずつ未来位置を生成する」自己回帰デコードの原型です。動機は人の動きの社会的相互作用(周囲の人を避ける、流れに乗る)を捉えることでした。トレードオフは逐次生成で並列化しにくい点です。再帰ユニットの軽量化としてGRU(Cho et al., EMNLP 2014, arXiv:1406.1078)が登場し、LSTMより少ないゲートで同等の時系列モデリングを実現したため、軌跡デコーダでよく使われます。

注意機構の系譜から。Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)は、物体検出のTransformerにおいて、全画素を見る密な attention の代わりに各 query が少数の参照点周りだけをサンプリングする「deformable attention」を導入しました。動機は、密な attention が高解像度画像で計算量と収束時間を爆発させることでした。本ノートで使うサンプリングオフセット + attention重みの構造は、この論文がそのまま起源です。

エンドツーエンド運転への統合。BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)は鳥瞰図エンコーダを提供し([鳥瞰図特徴融合（BEVFormer風 空間クロスアテンション）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0299_bev-fusion))、UniAD(Hu et al., CVPR 2023, arXiv:2212.10156)はその上に知覚・予測・計画を1本のネットワークで貫く設計を提案しました。UniADでは「計画を最終目的とし、すべての中間タスクをそれに資するよう最適化する(planning-oriented)」という思想のもと、ego query(自車トークン)を周辺エージェントや地図のトークンに attention させて軌跡を出します。動機は「タスクを別々に最適化すると誤差が累積し、計画に効く情報が途中で失われる」ことでした。本ノートの ego query 方式はこの系譜にあります。UniAD はその設計でCVPR 2023のベストペーパーを受賞し、planning-oriented という考え方を広めました。

VAD(Jiang et al., ICCV 2023, arXiv:2303.12077)はシーンを密なラスタ(画素グリッド)ではなくベクトル(線・点の集合、例えば車線を折れ線で、エージェントを軌跡ベクトルで)で表し、ego query がそれに attention して計画する方式で、計算を大幅に軽くしました。続くVADv2(Chen et al., 2024, arXiv:2402.13243)は、単一軌跡を回帰するのではなく、あらかじめ用意した多数の軌跡候補(語彙)の上に確率分布を学習する「probabilistic planning」を導入し、後述する多峰性(複数の妥当な軌跡)の問題に応えました。

現在の流れとして、自己回帰の逐次性を嫌い、全ウェイポイントを一度に出す非自己回帰デコードや、複数候補軌跡を出してスコアリングするモード予測(VADv2など)、拡散モデル(ノイズから徐々に軌跡を洗練する生成モデル)で軌跡を生成する手法が研究されています。

## 論文の実験結果(定量データ)

評価でよく使われる指標を、まず意味から説明します。

- L2誤差(平均変位誤差、ADE): 予測した軌跡の各時刻の位置と、実際に人間が運転した位置とのユークリッド距離(直線距離)の平均(単位はメートル)です。小さいほど良いです。1秒後・2秒後・3秒後の値や、それらの平均(avg)で報告されることが多いです。例えばL2が1mなら、3秒後に実際の運転位置から平均1mずれている、という意味です。
- 衝突率(collision rate): 予測軌跡通りに動いたとき、シーン中の他物体と衝突する割合(%)です。小さいほど安全です。この指標が重要なのは、L2が小さくても衝突する軌跡は実用にならないためです。L2は「人間の運転にどれだけ似ているか」を測りますが、衝突率は「安全か」を測る別の軸です。

nuScenes(自動運転の標準ベンチマーク。ボストンとシンガポールで集めた1000シーン、各20秒)でのオープンループ計画評価では、UniAD(Hu et al., 2023)が当時の代表的ベースラインを上回りました。報告によれば、平均L2誤差は約1.03m、平均衝突率は約0.31%で、ST-P3(Hu et al., ECCV 2022, arXiv:2207.07601)など先行手法(平均L2で1.5m超、衝突率0.7%前後)を明確に下回りました。UniADのアブレーション(どの要素を抜くと性能がどう落ちるか)では、知覚・予測の中間モジュールを計画に接続したことが計画品質に効くことが示され、特にモーション予測(他車の未来予測)を計画 query に渡す経路を切ると衝突率が悪化することが報告されています。これは「他車がどう動くかを計画ヘッドが参照できること」が安全性に直結する証拠です。

VAD(Jiang et al., 2023)は、UniAD同等以上の計画精度(報告で平均L2約0.72m、衝突率も同程度に低い)を保ちつつ、ベクトル表現により推論速度を大きく改善しました。報告ではUniADに対しておよそ9倍前後の高速化(FPSで比較、UniADが約1.8 FPSに対しVADが約16 FPS)とされ、計画品質と速度を両立できることを示しました。動機が「密表現の計算コスト削減」だったことを裏付ける結果です。

Deformable DETR(Zhu et al., 2021)はCOCO物体検出で、標準DETRが学習に500エポックを要したところを、同等以上のAP(検出精度の指標、Average Precision)を約50エポック、すなわちおよそ10分の1の学習量で達成しました。deformableなサンプリングが収束を劇的に速めることを示した結果で、計画ヘッドにこの機構を持ち込む根拠になっています。

なお、近年これらのnuScenesオープンループ評価には批判もあります。AD-MLP(Zhai et al., 2023, arXiv:2305.10430)は、カメラ画像を一切使わず自車の運動状態(過去の速度・加速度・操舵)だけを単純なMLPに入れても、nuScenesのL2・衝突率で上記の手法に匹敵する数値が出てしまうことを示しました。具体的には、画像を見ない自車運動のみのMLPが平均L2で当時のSOTA級と同程度の値を記録しました。これはベンチマークが「自車の慣性(過去の動きの延長)だけで当てられる」偏りを持つことを意味し、オープンループ指標の解釈には注意が要るという重要な指摘です(後述の限界に関わります)。

## メリット・トレードオフ・限界

メリット:
- 鳥瞰図の解像度に依存しません。高解像度化してもサンプリング座標(正規化された0〜1)を変えるだけで動きます。
- deformable attention は見る点をK個に固定するため、全セルをflattenする方式より圧倒的に省パラメータ・省メモリです([flatten+MLP型の計画ヘッドとその構造的限界](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0118_driving-policy-flatten-mlp-legacy)との対比)。
- ego query + attention は近年の主流設計で、検出・予測・計画の知見(DETR系の query 設計)を取り込みやすいです。
- 再帰デコードで運動の連続性が自然に保たれます。

トレードオフ・限界:
- 自己回帰デコードは逐次実行なので推論レイテンシ(応答までの時間)が長くなりやすいです。64ステップを順番に回すと並列化できず、安全クリティカルな車載では遅延が問題になり得ます。非自己回帰デコードはこの解決策です。
- 自己回帰は途中の予測誤差が後段に蓄積しやすいです(誤差累積、exposure bias)。訓練時は正解を見て学ぶが推論時は自分の出力を入力に使うため、ズレが増幅されやすいのです。scheduled sampling(訓練中も時々自分の出力を入力に混ぜる)などで緩和されます。
- deformable attention は学習初期に基準点が定まらず不安定になりやすく、`offset_scale` などの調整やウォームアップ(学習率を最初は小さくする)が要ります。
- 単一の軌跡を回帰する設計は、分岐(右に行くか左に行くか)のような多峰性(複数の妥当解)を表現できません。回帰損失は複数の正解の平均に引っ張られるため、平均化されて「真ん中を突っ切る」非現実的な軌跡が出ることがあります。モード予測や生成モデルはこの課題への応答です。
- オープンループ評価の信頼性。上記AD-MLPの指摘の通り、過去の運動だけで当たってしまうベンチマーク偏りがあり、クローズドループ評価(実際にシミュレータ内で運転させる、CARLA等)での検証が研究の必須事項になりつつあります。

## 発展トピック・研究の最前線

- 非自己回帰・並列デコード: 全ウェイポイントを1回のforwardで出します。レイテンシと誤差累積を同時に改善します。トランスフォーマーデコーダで全ステップを並列に出す設計が増えています。
- マルチモーダル軌跡: 複数候補を出して確率やコストでスコアリングします(VADv2, 2024)。多峰性を表現でき、回帰の平均化問題を回避します。
- 拡散モデルによる計画: ノイズから軌跡を逐次的に洗練する生成的アプローチです。多峰性を自然に扱え、複数の妥当な軌跡を分布として出せます。
- 言語・VLM(視覚言語モデル)との統合: 大規模視覚言語モデルに「なぜその軌跡か」を説明させたり、運転判断の根拠を言語で出す研究が進んでいます。解釈可能性と汎化の両面で期待されます。
- クローズドループ評価への移行: nuPlan、CARLA、Bench2Driveなど、実際に運転させて事故率・進行率・快適性を測る評価系への移行が進んでいます。オープンループの慣性バイアス問題を回避するためです。
