# DenseNet

## ひとことで言うと
DenseNet(Densely Connected Convolutional Network、密に結合した畳み込みネットワーク)は、画像から特徴を取り出す畳み込みニューラルネットワークの一種です。普通のネットワークは層を1段ずつ順番に通すだけですが、DenseNetは「ある層の出力を、その後に続く全ての層へ直接つなぐ」という密な配線をします。これにより、過去の層で作った特徴を後段で何度でも使い回せて、勾配(学習の手がかり)も全層に直接流れます。結果として、ResNetより少ないパラメータで同等以上の精度を出せます。

## 直感的な理解
深いネットワークでよく起こる問題は2つあります。1つは、過去の層が苦労して作った有用な特徴が、次々と層を通るうちに少しずつ変形・上書きされ、深い層に着くころには失われてしまうこと。もう1つは、学習の手がかりである勾配が、出力から入力へ遡るうちに弱まり、入力に近い層がほとんど学習されなくなることです。

DenseNetの発想は単純で大胆です。「失われるなら、最初から全部つないでおけばいい」。各層の出力を、後ろに続く全ての層へ配線として直接届けます。すると、どんなに深い層からでも過去の特徴に直接アクセスでき、逆に勾配もどの層からでも入力側へ近道で届きます。

身近な例で言えば、リレー方式の伝言ゲーム(隣の人にだけ伝える)では、最後の人に着くころには内容が歪んでしまいます。DenseNetは全員が全員に直接話す会議方式で、誰の発言も歪まずに全員へ届きます。各人が新たに付け加える情報は少しだけ(これを成長率と呼びます)ですが、全員の発言が共有されるので、少ない発言量でも豊かな結論にたどり着けます。

## 基礎: 前提となる概念
畳み込み層と特徴マップの基本を確認します。画像をネットワークに通すと各層は [チャンネル数 C, 高さ H, 幅 W] の3次元データ(特徴マップ)を出します。チャンネルは「検出している特徴の本数」です。畳み込み層は、前の層の特徴マップを受け取り、重みをかけて新しい特徴マップを作ります。

次に残差接続(residual connection)です。これは ResNet(He ら, CVPR 2016)が導入した仕組みで、ある変換 F の出力に入力 x をそのまま足して y = F(x) + x とします。足し算によって、勾配が逆伝播するときに「F を通る経路」と「足し算をそのまま遡る経路」の2本ができ、後者を通って勾配が弱まらずに入力側へ届きます。これが深いネットワークの学習を可能にしました。DenseNetは、この「足す」を「連結する(横に束ねる)」に置き換えた発展形だと理解すると分かりやすいです。

連結(concatenation)とは、複数の特徴マップをチャンネル方向に積み重ねて1つにまとめる操作です。たとえば [16, H, W] と [12, H, W] を連結すると [28, H, W] になります。足し算は要素ごとに値を加えて情報を混ぜてしまいますが、連結は元の特徴を一切変えずに並べて保持するので、後段の層が「どの過去特徴をどう使うか」を自由に選べます。この違いが DenseNet と ResNet の本質的な分かれ目です。情報理論的に言えば、足し算は情報の重ね合わせ(あとから分離できない)であり、連結は情報の保存(あとから取り出せる)です。

勾配消失(vanishing gradient)は前述の通り、深い層で勾配が弱まり学習が止まる現象です。DenseNet はこれを「全層が損失関数と入力の両方に近くなる」配線で緩和します。L 層のネットワークなら、普通の直列接続では入力から出力まで L 本の経路しかありませんが、密結合では層 i と層 j (i<j) の間に直接の経路があり、組み合わせは L(L+1)/2 通りに増えます。

## 仕組みを詳しく
DenseNetの中心単位は Dense Block(密ブロック)です。ブロック内の各層は、自分より前にある全ての層の出力を連結したものを入力として受け取ります。第 l 層の出力を x_l とすると、

  x_l = H_l([x_0, x_1, …, x_{l-1}])

ここで [·] は連結、H_l はその層の変換(後述)です。ResNet の x_l = F(x_{l-1}) + x_{l-1} と比べると、足し算ではなく全ての過去出力を連結している点が違います。

各層が新しく出すチャンネル数を成長率 k(growth rate)と呼び、典型的には k=12 や k=32 と小さく設定します。最初にブロックへ入る特徴のチャンネル数を c0 とすると、4層のブロックでは次のように入力チャンネルが k ずつ増えます。

- 層1: 入力 c0、出力 k(=12)
- 層2: 入力 c0 + 12、出力 12
- 層3: 入力 c0 + 24、出力 12
- 層4: 入力 c0 + 36、出力 12

重要なのは、各層が新たに作るのはわずか k チャンネルだけという点です。ResNet が各ブロックで数百チャンネルを丸ごと作り直すのに対し、DenseNet は「全員の知識を共有し、各自はほんの少しだけ新情報を足す」方式なので、パラメータが非常に少なくて済みます。著者らはこれを「集合知(collective knowledge)」というメタファーで説明しています。特徴マップそのものがネットワーク全体の共有状態(global state)であり、各層はそこに少しずつ書き込むだけです。

```
入力 ──┬──→ 層1 ──┬──→ 層2 ──┬──→ 層3 ──┬──→ 層4
        │          │          │          │
        └──────────┴──────────┴──────────┘
       (前の全層の出力を、後ろの各層へ連結して渡す)
```

各層 H_l の中身は、効率版(DenseNet-B)では「Batch Normalization → ReLU → 1×1畳み込み(4k チャンネルに絞るボトルネック) → BN → ReLU → 3×3畳み込み(k チャンネル出力)」の順です。ここで BN と ReLU を畳み込みの前に置く「pre-activation」配置を採っているのが特徴で、これは ResNet の改良版(He ら 2016 の identity mappings)に倣ったものです。先頭の1×1ボトルネックは、入力チャンネルが連結で膨らんでも計算量を抑える役割を持ちます。たとえば入力が 256ch でも、まず 4k=128ch に絞ってから 3×3 をかけるので、3×3 の負担は入力チャンネル数に依存せず一定に抑えられます。

連結を続けるとチャンネル数が増え続け、特徴マップの空間サイズ(H, W)も縮みません。そこで Dense Block と Dense Block の間に Transition Layer(遷移層)を挟みます。これは「1×1畳み込みでチャンネルを圧縮(compression、係数θ=0.5なら半分に) → 2×2 average pooling で縦横を半分に」する層で、データ量を抑えて次のブロックへ渡します。圧縮を入れた版を DenseNet-C、ボトルネックと圧縮の両方を入れた版を DenseNet-BC と呼びます。

ImageNet 用の DenseNet-121 は、成長率 k=32、4つの Dense Block(層数 6, 12, 24, 16)で構成され、総パラメータは約800万個です。比較対象の ResNet-50 が約2560万個であることを考えると、3分の1以下のパラメータで同等の精度を出すことになります。より大きな DenseNet-201 でも約2000万個と、同等精度の ResNet-101(約4500万個)の半分以下です。

## 手法の系譜と主要論文
原論文は Huang・Liu・van der Maaten・Weinberger「Densely Connected Convolutional Networks」(CVPR 2017、arXiv:1608.06993、Best Paper Award 受賞)です。著者らの動機は、ResNet で観察された現象に根ざしています。Huang らは先行研究「Deep Networks with Stochastic Depth」(ECCV 2016)で、ResNet の層を学習中にランダムに丸ごとスキップ(恒等変換に置換)しても精度がほとんど落ちず、むしろ正則化として精度が上がることを発見しました。これは「ResNet には冗長な層が多く、各層が独立に機能できる一方で、層どうしの情報共有が不十分」という洞察を生みました。ならば最初から全層を密につないで特徴を共有させればよい、というのが DenseNet の出発点です。

DenseNet の貢献は3点に整理できます。第一に勾配消失の緩和。全層が損失関数に近道で接続されるため、Inception の補助分類器に似た「深い監督(deep supervision)」の効果が自然に生まれます。第二に特徴の再利用(feature reuse)。連結により過去の特徴がそのまま保持され、後段が選んで使えます。第三にパラメータ効率。成長率 k を小さく保つことで、各層が学ぶべき情報量が減り、モデルが小型になります。

この系譜は ResNet → Stochastic Depth → DenseNet という流れで理解できます。DenseNet 自体は ResNet ほど主流バックボーンの座を占めませんでしたが、その「特徴再利用」「全層接続」の思想は、後の CSPNet(Cross Stage Partial Network、Wang ら CVPRW 2020)や、密な特徴融合を行う検出・セグメンテーションのネック(neck)設計、さらには現代のスキップ接続多用アーキテクチャに広く浸透しました。

メモリ問題への対処として Pleiss らの「Memory-Efficient Implementation of DenseNets」(2017、arXiv:1707.06990)も重要です。連結した特徴を毎層メモリに保持する代わりに、共有メモリ領域を使い回し、必要時に連結と BN を再計算する gradient checkpointing を使い、メモリ消費を二次オーダーから線形オーダーへ大幅に削減しました(計算時間は約15〜20%増)。

## 論文の実験結果(定量データ)
小規模ベンチマークでは CIFAR-10 / CIFAR-100(各5万枚訓練、10/100クラスの32×32画像)と SVHN(街路の数字)で評価されました。指標は分類誤り率(error rate、低いほど良い)です。DenseNet-BC(L=190, k=40)は CIFAR-10 でデータ拡張ありの誤り率 約3.46%、CIFAR-100 で 約17.18% を達成し、当時の ResNet 系(ResNet-1001 など)を、はるかに少ないパラメータで上回りました。具体的には、約1000万〜1500万パラメータの ResNet-1001 系と同等以上の精度を、DenseNet-BC は約3分の1以下のパラメータで実現したと報告されています。

ImageNet(120万枚、1000クラス)では、DenseNet-201 が単一クロップでトップ1誤り率 約22.6%、トップ5誤り率 約6.3% を記録しました。重要なのはパラメータと計算量の対比で、DenseNet-201(約2000万パラメータ)が ResNet-101(約4500万パラメータ)と同等のトップ1精度を、約半分のパラメータと約半分の計算量(FLOPs)で達成した点です。著者らは精度を縦軸、パラメータ数や FLOPs を横軸に取った曲線で、DenseNet が常に ResNet の左上(同じ精度を少ない計算で)に位置することを示しました。たとえば DenseNet-121(約2.9 GFLOPs)が ResNet-34(約3.6 GFLOPs)を上回るトップ1精度を、より少ない計算量で出しています。

アブレーションでは、ボトルネック(B)と圧縮(C)の効果が検証されています。DenseNet-BC は、素の DenseNet より少ないパラメータでありながら同等以上の精度を出し、これら2つの効率化が精度を犠牲にせずモデルを小型化することを示しました。また成長率 k を大きくすると精度は上がるがパラメータも増えるという素直なトレードオフも報告されています。さらに、学習済み DenseNet の重みを可視化すると、Transition Layer や後段の層が前段(特に直前ではない遠い層)の特徴にも有意な重みを割り当てていることが確認され、特徴の再利用が実際に起きている証拠とされました。一方で Transition Layer の一部は前段特徴に小さい重みしか割り当てておらず、「不要な特徴を捨てている」=特徴選択も同時に働いていることが示されました。

## メリット・トレードオフ・限界
メリットは、勾配が全層に直接流れて深いネットワークでも学習が安定すること、過去の特徴を連結で保持・再利用でき少ないパラメータで高精度が出ること、各層が成長率 k だけ新情報を足す設計でモデルが小型になることです。パラメータ効率と精度効率の両方で、当時の ResNet を明確に上回りました。

トレードオフと限界が、まさに DenseNet の最も教育的な点です。「パラメータは少ないのにメモリは食う」という一見矛盾した特性を持ちます。理由は、連結のために中間特徴マップを学習中ずっとメモリ上に保持し続ける必要があり、しかも素朴な実装では各層が連結結果を新しいメモリに毎回コピーするため、メモリ消費が層数の二乗オーダーで増えるからです。パラメータ数(モデルの重みの個数)とアクティベーションメモリ(順伝播で生じる中間データの量)は別物で、DenseNet は前者が小さくても後者が大きい典型例です。これは高解像度の入力や大きなバッチを扱う場面で深刻な制約になります。Pleiss らの再計算手法はこれを緩和しますが、その分計算時間が増えるトレードオフがあります。

もう1つの限界は実行速度です。連結が多く、メモリアクセスのパターンが ResNet のような単純な加算より断片的なため、同じ FLOPs でも GPU 上での実効スループットが出にくい傾向があります。FLOPs の少なさが実機の速さに直結しない、という現代CNN設計の課題をここでも見て取れます。これらの実用上の扱いにくさが、DenseNet が ResNet ほど広く主流バックボーンにならなかった背景にあります。

## 発展トピック・研究の最前線
DenseNet が提起した「アクティベーションメモリ vs パラメータ数」というトレードオフの理解は、現代の大規模モデル学習でますます重要になっています。gradient checkpointing(再計算によるメモリ削減)は、いまや大規模 Transformer の学習でも標準的に使われる技術で、その先駆けの1つが DenseNet のメモリ効率実装でした。

特徴再利用の思想は CSPNet として実用化され、物体検出の YOLO 系バックボーン(CSPDarknet など)に広く採用されました。CSPNet は特徴マップをチャンネル方向に2つに分け、片方だけを密結合(または ResNet 風ブロック)に通してからもう片方と連結することで、密結合の表現力を保ちつつ、勾配経路の重複を断ち切って計算量とメモリを削減します。また、密な多解像度特徴融合の考え方は、セグメンテーションや医療画像で広く使われる U-Net 系のスキップ接続や、HRNet の高解像度並列保持にも通じています。研究の最前線では、こうした密接続の利点を、計算とメモリの効率を保ちながらいかに取り込むかが継続的なテーマであり、また近年では Transformer の dense な自己注意も「全要素を全要素に繋ぐ」という意味で DenseNet と同じ思想の延長線上にあると見ることができます。
