# ConvNeXt V2 / FCMAE

## ひとことで言うと

ConvNeXt V2 は、画像認識用の畳み込みネットワーク (CNN) を、画像の一部を隠して復元させる自己教師あり学習 (MAE 型) で事前学習できるように作り直したモデルです。そのために「全畳み込みマスクオートエンコーダ (FCMAE)」という学習法と「GRN」という小さな正規化部品を新しく入れました。マスク復元型の事前学習はもともと Vision Transformer 向けに作られていましたが、ConvNeXt V2 はそれを CNN でも成立させ、CNN の計算効率の良さとマスク復元の強い表現学習を両立させた点に意義があります。

## 直感的な理解

良い特徴抽出器を無ラベルで作りたい、というのが自己教師あり学習の目標です。画像の大部分を隠して残りから復元させる MAE (Masked Autoencoder) はそのための強力な方法ですが、当初は Transformer 専用でした。一方で CNN は、Transformer より計算が軽く、高解像度画像でもメモリが膨らみにくい (注意機構の計算量が画素数の二乗で増えないため) など実用上の利点があります。だから「CNN でも MAE のような事前学習がしたい」という自然な需要がありました。

ところが MAE をそのまま CNN に持ち込もうとすると壊れます。理由を直感的に言うと、Transformer は「隠したパッチを処理から完全に取り除く」ことができますが、畳み込みは「画像全体を密に滑って計算する」ので、一部が抜けた歯抜けの入力をうまく扱えないのです。隠した穴をゼロで埋めると、畳み込みはその「ゼロの境界」をはっきりした手がかりとして拾ってしまい、「ここからここまでが隠された領域だ」というカンニングペーパーを得てしまいます。すると復元タスクが簡単になりすぎ、意味のある特徴が育ちません。さらに学習が進むと、見えている部分と「ゼロで埋めた偽の部分」が混ざった統計を学んでしまい、本番 (穴のない画像) との分布のずれも生じます。これらをマスクパターンの漏れと呼びます。ConvNeXt V2 はこの漏れを塞ぐことから始まります。

## 基礎: 前提となる概念

畳み込みネットワーク (CNN) は、画像を小さな窓 (カーネル) で少しずつずらしながら見て特徴を取り出すネットワークです。ResNet や ConvNeXt がこの仲間です。同じカーネルを画像全体で使い回す (重み共有) ため、パラメータが少なく、局所的なパターンを効率よく捉えられます。

ConvNeXt (Liu et al., CVPR 2022, arXiv:2201.03545) は Meta が出した CNN で、Transformer の設計の良いところ (7x7 の大きなカーネル、層正規化 LayerNorm、GELU 活性、チャンネルを一度広げて戻す逆ボトルネック、ダウンサンプリングの分離など) を CNN に取り込み、純粋な畳み込みでも Vision Transformer に匹敵する精度を出したモデルです。V2 はその改良版です。

MAE (Masked Autoencoder) は、画像の大部分 (例: 75%) を隠して残りから隠した部分を復元させる学習法です。He らの CVPR 2022 論文 (arXiv:2111.06377) が ViT 向けに提案しました。ViT (Vision Transformer) は画像をパッチに区切って Transformer で処理するモデルです。MAE は見えるパッチだけを大型エンコーダで処理し、マスク位置は小型デコーダで埋める非対称設計で、効率と性能を両立させました。

特徴崩壊 (feature collapse) / 次元崩壊 (dimensional collapse) も前提です。学習した特徴の多くのチャネルが似たような反応をしてしまい、表現の多様性が失われる現象です。表現がごく少数の方向に潰れると、いくらパラメータがあっても実質的な情報量が少なくなり、性能が出ません ([次元崩壊 (dimensional collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0043_dimensional-collapse) 参照)。

## 仕組みを詳しく

ConvNeXt V2 は2つの工夫からなります。この2つを一緒に設計した (co-design) のが論文の核心で、片方だけでは効果が薄く、両方そろって初めて MAE 事前学習が CNN で機能します。

### FCMAE (Fully Convolutional Masked Autoencoder、全畳み込みマスクオートエンコーダ)

入力画像を格子に区切り、たとえば 60% のマスク率でランダムに隠します。224x224 の画像を 32x32 のブロックに区切ると 7x7 = 49 ブロックになり、その約 60% を隠します。マスクは最も粗い解像度で生成し、それを各段階の解像度にアップサンプルして適用することで、ダウンサンプリングしても情報が漏れないようにします。

ここで肝になるのがスパース畳み込み (sparse convolution) です。これは「見えている画素だけを計算に入れ、隠した穴は計算からスキップする」畳み込みです。ViT の MAE が隠したパッチをトークン列から取り除くのと同じ効果を CNN で実現します。これにより、穴の境界をモデルが手がかりにしてしまう漏れを防ぎ、ゼロ埋めによる分布のずれも避けられます。学習時はスパース畳み込みで歯抜け入力を処理し、fine-tuning や推論時は通常の密な畳み込みに戻せるよう設計されています (スパースと密で数学的に等価になるよう構成されている)。

エンコーダ (特徴を取り出す前半、ここが ConvNeXt 本体) で見えている部分から特徴を作り、デコーダ (後半、ConvNeXt ブロック1つ程度の軽量なもの) で隠した部分のもとピクセルを復元します。MAE が大型 Transformer デコーダを使うのに対し、FCMAE は単一ブロックの非常に軽いデコーダで十分なことを示しました。損失は隠した領域のみで計算し、復元誤差 (平均二乗誤差, MSE。隠した画素について予測値と正解値の差を二乗平均したもの) を最小化します。事前学習が終わったら、デコーダとスパース化は捨て、エンコーダ (ConvNeXt V2 本体) だけを下流タスク (分類・検出・密予測など) に転用します。

### GRN (Global Response Normalization、全域応答正規化)

FCMAE で学習させると特徴崩壊が起きました。論文の分析では、ConvNeXt の逆ボトルネック (チャンネルを一度大きく広げる部分) で、多くの特徴チャネルが互いに似た反応をして冗長になり、いくつかのチャネルだけが過剰に強く反応し、残りが死んでしまう現象が観察されました (特徴のコサイン類似度が高くなる)。すると表現の多様性が失われ性能が出ません。

GRN はこれを直す小さな層で、ConvNeXt ブロックの中 (チャンネルを広げた直後あたり) に差し込みます。各チャンネルの特徴マップ (shape `[H, W]`) があるとき、動作は3ステップです。

1. 各チャンネルごとに空間方向の大きさ (L2 ノルム、二乗和の平方根) を測り、チャンネルごとに1つの値 `g_c` を得る。チャンネル数を C とすると `[C]` のベクトル。
2. それを全チャンネルの平均 (または平均的な大きさ) で割って相対化する (あるチャンネルが他と比べてどれだけ強いか)。`n_c = g_c / mean(g)`。
3. もとの特徴に `n_c` を掛けて、強すぎるチャンネルを抑え弱いチャンネルを相対的に持ち上げる。さらに学習可能なスケール γ とバイアス β を足し、残差で元の入力に戻す。

効果として、チャンネル間の競争 (feature competition) が促され、どのチャンネルもそれぞれ違う特徴を担当するようになり、崩壊が防げます。GRN はパラメータがごくわずか (γ と β の2本のベクトルだけ) で計算も軽いのに、効果が大きい点が美点です。GRN は ConvNeXt V1 にはなく V2 で追加された目玉です。なぜ co-design が必要かを一言でまとめると、スパース畳み込みでマスク漏れを塞いだ結果として新たに表面化した特徴崩壊を、GRN が抑えるという相補的な関係にあるからです。

## 手法の系譜と主要論文

中心論文は Woo, Debnath, Hu, Chen, Liu, Kweon, Xie の "ConvNeXt V2: Co-designing and Scaling ConvNets with Masked Autoencoders" (CVPR 2023, Meta AI + KAIST, arXiv:2301.00808) です。貢献は FCMAE + GRN の co-design で、MAE 事前学習を CNN に持ち込むとマスク漏れと特徴崩壊で性能が出ないという二重の問題を、同時に解決しました。

系譜の上流には2つの源流があります。一つは He らの MAE (CVPR 2022, arXiv:2111.06377) で、ViT で「75% マスク + 非対称な大型エンコーダ・小型デコーダ」を提案しました。高マスク率にすると復元が難しくなり強い特徴が育つこと、見えるパッチだけ処理すれば高速なことが要点です。ただし ViT 専用で CNN に直接使えず、まさにそこを ConvNeXt V2 が埋めました。もう一つは Liu らの ConvNeXt V1 (CVPR 2022, arXiv:2201.03545) で、Transformer の設計知見を CNN に注入したアーキテクチャです。ConvNeXt V2 はこの V1 に GRN を追加し、FCMAE で事前学習できるようにした発展形です。さらに遡れば「マスクして復元する」発想の原点は [Context Encoder](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0035_context-encoder-inpainting) (CVPR 2016) で、ConvNeXt V2 はその系譜を現代の CNN で完成させたものと言えます。

比較対象として Bao らの "BEiT" (ICLR 2022, arXiv:2106.08254) は、ピクセルでなく離散コードを復元する MIM (Masked Image Modeling) で、トークナイザ (画像をコードに変換する別モデル、dVAE など) が必要という違いがあります。予測対象を内部の潜在特徴にする [data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) とも対比されます。FCMAE はこれらと違いトークナイザも教師ネットワークも要らず、ピクセル MSE だけで完結する簡潔さが特徴です。CNN でマスク事前学習を試みた先行研究 (SparK など、同じくスパース畳み込みを用いた) もありますが、ConvNeXt V2 は GRN による崩壊対策とアーキテクチャの co-design、そして Atto から Huge までの一貫したスケーリング検証で完成度を高めた点が独自の位置づけです。

## 論文の実験結果 (定量データ)

ConvNeXt V2 は ImageNet-1K と ImageNet-22K を中心に評価されました。最大の主張は「FCMAE 事前学習が、教師ありで学習した ConvNeXt V1 を一貫して上回る」ことです。

サイズ展開: Atto (約 350 万パラメータ) から Huge (約 6.6 億パラメータ) まで幅広いサイズで検証され、すべてのサイズで V2 (FCMAE 事前学習) が対応する V1 (教師あり) を上回りました。たとえば ConvNeXt V2-Huge を ImageNet-22K で事前学習し 512x512 解像度で fine-tuning した場合、ImageNet-1K top-1 精度でおよそ 88% 台後半という、当時の CNN として非常に高い水準を報告しています。top-1 精度は 1000 クラスのうち最も確からしい1つを正解とする割合で、高いほど良い指標です。小型の Atto/Femto サイズでも、教師あり V1 比で明確な改善が示され、「FCMAE は大型モデルだけでなく小型でも効く」ことが強調されました。これは MAE が主に大型モデルで効果を発揮しがちだったのに対する強みです。

アブレーションが特に教訓的です。論文は GRN なしで FCMAE を回すと特徴崩壊が起き、精度が伸びないことを示しました。逆に、教師あり学習に GRN を足すだけでは効果が限定的です。FCMAE と GRN を両方そろえたときに最大の改善が出る、というのが co-design 主張の定量的裏付けです。具体的には、ある中間サイズのモデルで、教師あり V1 から FCMAE + GRN にすることで top-1 が数ポイント改善したと報告されています。マスク率についても、ViT の MAE の 75% より低い 60% 前後が CNN では良いことが示され、アーキテクチャによって最適なマスク率が異なることが分かりました。これは「同じマスク復元でも、計算の局所性 (CNN) と大域性 (ViT) の違いで適切な難易度が変わる」ことを示唆します。

転移性能も評価されています。COCO 物体検出/インスタンスセグメンテーションや ADE20K セマンティックセグメンテーションへ転移すると、FCMAE 事前学習した ConvNeXt V2 は教師あり初期化を上回る mAP / mIoU (mean Intersection over Union、予測領域と正解領域の重なりの割合の平均。高いほど良い) を示しました。これは、FCMAE で学んだ特徴が分類だけでなく、空間的な位置情報を要する密な予測タスクにも有効であることを意味します。

## メリット・トレードオフ・限界

メリットは次の通りです。CNN でも MAE 型の自己教師あり事前学習ができ、ラベルなしの大量画像から特徴を学べます。ViT より計算・メモリ効率がよく、高解像度の画像 (大きな入力) でも扱いやすい。GRN で特徴崩壊を抑え、表現の多様性を保てます。ピクセル MSE だけで完結し、トークナイザや教師ネットワークが不要なので、BEiT や data2vec より仕組みが簡潔です。

トレードオフと限界もあります。第一に、スパース畳み込みの実装がフレームワーク依存で扱いづらく、環境によっては学習パイプラインの構築にハードルがあります。第二に、MAE 系全般の弱点として、事前学習に大量データと長い学習時間 (数百〜千エポック規模) が必要です。第三に、「ConvNeXt V2 というアーキテクチャを使うこと」と「FCMAE で自前事前学習すること」は別物で、多くの場合は既存の事前学習済み重みをロードして使うのが現実的です。自前で FCMAE を回すには相応の計算インフラが要ります。

研究上の未解決課題として、「ViT と CNN でなぜ最適なマスク率や設計が異なるのか」の理論的理解はまだ完全ではありません。また、GRN は特徴崩壊への対症療法的な側面があり、なぜ FCMAE が崩壊を誘発するのか (逆ボトルネックとマスク復元の相互作用) の根本原因の解明、そしてより原理的な崩壊回避法の探索は今後の課題です。マスク復元 (ピクセル再構成) が最良の予測対象かという、[data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) が投げかけた問いも CNN の文脈で開いたままで、ピクセル MSE は意味的に無関係な高周波成分まで予測対象にしてしまう弱点を抱えたままです。

## 発展トピック・研究の最前線

ConvNeXt V2 は「CNN は時代遅れではなく、適切な事前学習を与えれば ViT と十分に戦える」ことを示した点で、CNN 復権の一つの到達点でした。その後、マスク復元型事前学習は CNN・ViT を問わず標準的な選択肢になり、ハイブリッド (畳み込みと注意機構を混ぜた) アーキテクチャや、解像度・スケールに応じた最適なマスク戦略の研究が続いています。

予測対象の高度化も進んでいます。ピクセル再構成 (FCMAE) ではなく、内部の潜在特徴を予測する [data2vec](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0041_data2vec) 型や、離散トークンを予測する BEiT 型を CNN に適用する試みもあります。高解像度認識や密予測の文脈では、CNN の効率性とマスク復元の表現力の組み合わせは依然として実用的な魅力があり、大量の無ラベル映像から特徴を学ぶ事前学習の有力な選択肢の一つです。学習した表現の質を測るには、損失値ではなく [linear probing / kNN 評価](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0759_linear-probing-evaluation) や下流タスクの転移性能で確認するのが定石です。GRN が示した「チャンネル間の競争を明示的に促す」という正規化の発想は、特徴崩壊が問題になる他の自己教師あり手法 ([次元崩壊](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0043_dimensional-collapse) が起きうる蒸留型・対照型) への示唆としても参照されています。
