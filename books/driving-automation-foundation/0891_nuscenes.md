# nuScenes

## ひとことで言うと

nuScenes (ニュー・シーンズ) は、自動運転会社 Motional (旧 nuTonomy / Aptiv) が公開した、車に積んだ複数のカメラ・LiDAR (レーザー距離計)・レーダーをすべて同時に記録した代表的なデータセットです。車のまわり 360 度をくまなく捉えており、物体検出・追跡・BEV (鳥瞰図) 知覚・エンドツーエンド運転など、自動運転の多くの研究でベンチマーク (性能を比べる共通の試験問題) として使われています。特に「複数カメラの映像を俯瞰図の上に統合して周囲を把握する」BEV 知覚という研究分野の土台を作ったデータセットとして重要です。

ここで「BEV (Bird's-Eye View、鳥瞰図)」とは、空から車を真上に見下ろしたような俯瞰の座標系のことです。複数カメラの映像を、この俯瞰図の上に統合して物体の位置を把握する手法が近年の主流で、nuScenes はその研究の中心的な検証場になっています。

## 直感的な理解

自動運転車は、前方だけでなく後方・側方も含めた全周囲を同時に認識する必要があります。追い越しや車線変更では斜め後ろの車を、交差点では左右から来る歩行者を見落とせません。これを実現するには、複数のセンサを車体の各方向に取り付け、それらを時刻同期して扱う「マルチモーダル・全周囲」のデータが要ります。

なぜ複数の種類のセンサ (マルチモーダル) を組み合わせるのでしょうか。カメラは色や見た目・標識の文字を捉えるのが得意ですが、距離が分かりにくく、暗所や逆光に弱いです。LiDAR は正確な距離 (3D 形状) を測れますが、色が分からず、雨や霧で乱れます。レーダーは電波なので悪天候に強く、相手の速度を直接測れますが、解像度が低く形が分かりません。三者を組み合わせると、互いの弱点を補い合えます。さらに、人間が頭の中で前後左右を 1 枚の俯瞰図にまとめて状況を把握するように、機械にも全周のカメラ映像を 1 枚の俯瞰図 (BEV) に統合させるのが自然で、nuScenes はそのための全周囲データを提供しました。

## 基礎: 前提となる概念

モダリティ (modality): センサの種類のことです。カメラ・LiDAR・レーダーがそれぞれ別のモダリティで、これらを束ねて使うのがマルチモーダルです。

キーフレーム (keyframe): ラベルが付いている代表的な瞬間です。nuScenes ではこのキーフレームで 6 カメラ + LiDAR + レーダーがすべて時刻同期しており、ここを基準に学習・評価します。キーフレーム間の中間フレーム (sweep) はラベルなしで記録され、速度推定や時間融合の補助に使えます。

BEV グリッド: 俯瞰座標を格子状のマス目に区切ったものです。例えば自車を中心に前後左右 50 m を 0.5 m 刻みで区切ると 200×200 のマス目になり、各マスに「ここに車がいるか」「ここは車線か」を割り当てます。これが BEV 知覚の出力空間です。

3D バウンディングボックス: 物体を囲む 3 次元の直方体で、中心 (x, y, z)・寸法 (w, l, h)・向き (ヨー角) で表します。nuScenes はこれに加えて各物体の速度や属性 (歩行者が立っている/座っている、車が止まっている/動いている) もラベル付けしています。

mAP と NDS: mAP (mean Average Precision) は検出の「どれだけ正しく漏れなく当てたか」を測る古典的指標です。ただし nuScenes の mAP は IoU ではなく BEV 平面上の中心距離 (0.5 / 1 / 2 / 4 m の 4 閾値) で当たり判定し、それらを平均します。さらに 3D 検出では、位置が合っていても向きや速度を間違えると危険です。そこで nuScenes は NDS (nuScenes Detection Score) という独自の総合指標を作り、mAP に加えて 5 つの True Positive 誤差 — 位置 (mATE)・大きさ (mASE)・向き (mAOE)・速度 (mAVE)・属性 (mAAE) — をまとめて点数化しました。単純な mAP では測れない品質を織り込んだのが NDS です。

## 仕組みを詳しく

nuScenes の主なスペックを段階的に見ていきます。

- シーン数は 1000。各シーンは約 20 秒の運転の切り出しです。
- 撮影地はボストン (米・右側通行) とシンガポール (左側通行)。右側通行と左側通行の両方を含み、交通文化の多様性があります。
- センサ構成 (全周囲):
  - カメラ 6 台 (前・前左・前右・後・後左・後右)、各 1600×900 RGB、12 Hz 記録。6 台で水平 360 度をカバー。
  - LiDAR 1 台 (天井、32 ビーム、20 Hz、全周 360 度の点群)。
  - レーダー 5 台 (電波で速度と距離、悪天候に強い、13 Hz)。
  - GPS / IMU で自車の位置・姿勢を記録。
- アノテーション (正解ラベル) は 2 Hz (0.5 秒ごと) で、3D 直方体を手作業で付与。23 物体カテゴリ、合計約 140 万個の 3D ボックス。属性も付きます (検出評価では 10 カテゴリに集約)。

1 つのキーフレームの中身を具体的に書くと次のようになります。

```
keyframe = {
  画像:    6 枚 (各 1600 x 900 の RGB)
  点群:    LiDAR の (x, y, z, 反射強度, ring index) が約 3 万点
  レーダー: 5 台分の (位置, 相対速度) 点群
  自車姿勢: ego_pose (地図上の位置と向き)
  ラベル:  各物体の 3D box {中心 x,y,z, 大きさ w,l,h, 向き, 種別, 速度, 属性}
}
```

BEV 知覚モデルの典型的な処理を追うと、まず 6 枚の画像 `[6, 3, 900, 1600]` を入力として受け取り、各画像を CNN で特徴量に変換します。次にカメラの取り付け位置・向き (キャリブレーション) を使って、各画像特徴を俯瞰グリッド (例えば `[200, 200]`) 上に投影します。ここで難しいのは、カメラ画像は 2D なので「そのピクセルが地上のどこにあるか (奥行き)」が一意に決まらないことです。Lift-Splat-Shoot 系の手法は、各ピクセルに奥行きの確率分布を持たせて 3D 空間に「持ち上げ (lift)」、俯瞰グリッドに「ばらまく (splat)」ことでこの曖昧さを吸収します。最後に俯瞰グリッド上で 3D ボックスを検出 (shoot) します。

評価では、検出した各ボックスを正解と照合し、位置誤差・寸法誤差・向き誤差・速度誤差・属性誤差をそれぞれ測り、NDS に統合します。位置だけ合っていて速度を外したボックスは NDS で減点されるため、「動いている車を止まっていると誤認する」ような危険な誤りがスコアに反映されます。NDS は概念的には `NDS = (1/10)[5·mAP + Σ(1 - min(1, 各TP誤差))]` のように、mAP に重みを置きつつ 5 つの誤差を加味した加重平均です。

nuScenes には車線・横断歩道・歩道などのベクトル地図 (map expansion) も付属し、自車の地図上の位置と合わせれば「車線に沿った予測」も可能です。地図・自車姿勢・速度ラベルが揃っているため、知覚だけでなく予測・計画まで連続して研究できるのが強みです。派生として nuScenes-lidarseg (点群の意味ラベル)、CAN bus (車両の生信号)、nuImages (2D 画像の大規模アノテーション) も公開されています。

## 手法の系譜と主要論文

データセットの系譜では、nuScenes は KITTI (Geiger et al., CVPR 2012) の制約を解消する形で登場しました。KITTI はカメラが主に前方向きで全周囲をカバーせず、データ量が小さく天候・時間帯の多様性に乏しく (主に晴天・昼間)、レーダーを含みませんでした。

nuScenes (Caesar, Bankiti, Lang et al., CVPR 2020, "nuScenes: A multimodal dataset for autonomous driving") はこれらを解決しました。6 台のカメラで全周をカバーし、LiDAR とレーダーも積み、夜間や雨天を含む多様な状況を、ボストンとシンガポールで収集しました。3D 検出向けに NDS という総合評価指標を新設したのも貢献の 1 つです。同年に公開された Waymo Open Dataset ([Waymo Open Dataset](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0904_waymo-open-dataset)) がマルチ LiDAR と大規模ラベルに強いのに対し、nuScenes はレーダーと HD マップを持ち全周をカメラで覆う点が異なります。

このデータセットを土台に多数の重要手法が生まれました。
- LSS (Lift-Splat-Shoot, Philion & Fidler, ECCV 2020): 各カメラ画素に深度の確率分布を持たせて 3D に持ち上げ、俯瞰グリッドにばらまいて複数カメラを 1 枚の BEV 特徴に統合する。カメラだけで距離が曖昧な問題を、深度を確率で扱うことで吸収します。後続の BEV 手法の基礎になりました。トレードオフは、深度推定が不正確だと BEV がぼやけることです。
- DETR3D (Wang et al., CoRL 2021): 3D の物体クエリを各カメラ画像へ投影して特徴を引き出す、NMS (重複検出の後処理) 不要の検出器。明示的な BEV グリッドを作らずに 3D 検出します。トレードオフは、投影点が画像外に出ると情報が取れないことです。
- BEVFormer (Li et al., ECCV 2022): BEV グリッドの各マスを「クエリ」とし、Transformer の Attention (spatial cross-attention) で複数カメラから、また temporal self-attention で過去フレームから情報を集める。過去を使うことで速度推定や遮蔽に強くなります。nuScenes 3D 検出で当時のトップ級でした。トレードオフは Attention の計算が重いことです。
- BEVDepth (Li et al., AAAI 2023): LiDAR を教師として深度を明示的に学習し、カメラのみ BEV の精度を底上げしました。LSS の弱点 (曖昧な深度) を直接補強したものです。
- BEVFusion (Liu et al., ICRA 2023): カメラ BEV 特徴と LiDAR BEV 特徴を同じ BEV 空間で統合する汎用融合フレームワーク。

これらはいずれも nuScenes が共通の試験問題を提供したからこそ、公平に比較・発展できました。

## 論文の実験結果 (定量データ)

nuScenes 3D 検出のスコアは主に NDS と mAP で報告されます。両者とも 0〜1 (百分率で 0〜100) で、大きいほど良い指標です。

カメラのみの BEV 手法の伸びは劇的でした。初期の代表 FCOS3D (単眼 3D 検出、Wang et al., 2021) は nuScenes 検証セットで NDS がおよそ 0.37 程度でした。DETR3D は複数カメラを統合して NDS をおよそ 0.42〜0.43 へ、BEVFormer は時間情報を加えて NDS をおよそ 0.47〜0.52 のレンジへと押し上げました (バックボーンや入力解像度で差があります)。つまり「全周囲のカメラを俯瞰図で統合し、さらに時間方向の情報を使う」ほど検出が良くなるという流れが、数値で一貫して確認されています。

LiDAR ベースやマルチモーダル融合はさらに高く、CenterPoint (LiDAR) は NDS が 0.65 前後、カメラと LiDAR を融合する BEVFusion (Liu et al., ICRA 2023) は NDS が 0.70 を超える水準を報告しました。これは「正確な距離を測れる LiDAR を加えると、カメラのみより大きく検出が改善する」ことを示します。カメラのみで NDS 0.5 前後、LiDAR を足して 0.65〜0.70 という差は、奥行き情報の有無がいかに 3D 検出を左右するかを定量的に物語っています。

アブレーションで特に示唆に富むのは BEVFormer の時間融合です。過去フレームを使う temporal self-attention を取り除くと、速度推定の誤差 (mAVE、平均速度誤差) が顕著に悪化し、それに伴って NDS が下がることが報告されました。NDS は速度誤差を織り込む指標なので、「過去フレームを見ることで物体の速度を当てやすくなり、それが総合スコアに効く」という因果が数値で追えます。1 枚の画像だけでは物体が止まっているか動いているか分かりませんが、複数時刻を見れば動きが読めるため、時間情報が速度推定に直結するのです。同様に BEVDepth は、LiDAR を教師にした深度監督を入れると mAP・NDS がともに上がることを示し、「カメラ BEV のボトルネックは深度推定にある」ことを定量的に突き止めました。

NDS という指標設計そのものの価値も実験的に裏づけられています。mAP だけで順位づけすると、向きや速度を誤った危険な検出器が高く評価されかねません。NDS は位置・寸法・向き・速度・属性の誤差をまとめて点数化することで、「安全に意味のある検出品質」をより忠実に反映します。これにより研究コミュニティの最適化対象が、単なる検出率から「運転に使える検出」へと方向づけられました。

## メリット・トレードオフ・限界

メリット:
- 6 カメラで全周囲をカバーし、実走に近い知覚研究ができる。
- カメラ・LiDAR・レーダーのマルチモーダルで、融合手法を試せる。
- ボストンとシンガポール、昼夜・雨天を含み多様性が高い。
- 地図・自車姿勢・速度・属性ラベルが付き、知覚から予測・計画まで連続して扱える。
- NDS という洗練された評価指標と公開リーダーボードがあり、比較が公平。

トレードオフ・限界:
- 3D ボックスのラベルが 2 Hz と粗く (カメラ 12 Hz・LiDAR 20 Hz より低い)、高頻度の動き解析にはやや不向き。
- LiDAR が 32 ビームと当時としても控えめで、遠距離・小物体の点群が疎い (Waymo の 64 ビーム級主 LiDAR と比べると密度が低い)。
- 1000 シーンは登場時こそ大きかったが、後発の超大規模データ (10 万シーン級の予測データなど) に比べると小さい。
- 2 都市・特定の運行エリアに偏り、地域一般化は別途検証が必要。
- mAP の当たり判定が IoU でなく中心距離のため、寸法のずれに鈍く、他データセットの IoU ベース mAP と直接は比較できない。

未解決の研究課題: カメラのみで LiDAR 融合にどこまで迫れるか (センサコスト削減のため)、ラベルが粗い・少ない条件での自己教師あり学習、そして検出・追跡・予測・計画を一貫して学ぶエンドツーエンド化が活発に議論されています。nuScenes 上では、検出から計画までを 1 つのモデルで解く UniAD (Hu et al., CVPR 2023, best paper) のような統合手法も登場しました。

## 発展トピック・研究の最前線

最前線の 1 つは、知覚を単独タスクとしてではなく運転計画まで一貫して学ぶエンドツーエンド化です。UniAD (CVPR 2023) は検出・追跡・地図・予測・占有・計画を 1 つのネットワークに統合し、最終目標 (安全な計画) から逆算して各タスクを学習させることで、モジュールを別々に最適化するより計画性能 (衝突率・経路追従誤差) が改善することを nuScenes 上で示しました。これは「中間タスクの精度より最終的な運転の良さを直接最適化すべき」という思想の代表例です。一方で、こうした nuScenes 上の open-loop 計画評価 (記録された自車軌跡との比較) は、自車が止まっていれば誤差ゼロになるなどの落とし穴が指摘されており、評価方法そのものが研究テーマになっています。

もう 1 つは BEV 表現を占有格子 (occupancy grid、空間を 3D のマス目に区切り各マスが物体で埋まっているかを推定) へ拡張する流れです。これにより、事前に定義したカテゴリに当てはまらない未知の障害物 (落下物など) も「埋まっている空間」として検出でき、安全性が高まります。nuScenes 派生の Occ3D などのベンチマークがこの方向を支えています。BEV 知覚の標準ベンチマークとしての nuScenes の役割は、検出から占有・計画へとタスクが広がるなかでも続いています。
