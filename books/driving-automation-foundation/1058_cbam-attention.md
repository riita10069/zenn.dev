# CBAM (チャネル+空間アテンション)

## ひとことで言うと
CBAM(Convolutional Block Attention Module)は、CNN の特徴マップに対して「どの種類の特徴(チャネル)が重要か」と「画像のどの場所(空間)が重要か」の両方を、軽い注意機構で順番に判断して強調する部品です。[Squeeze-and-Excitation (SENet)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1085_squeeze-and-excitation)(SE)がチャネルだけに注意したのに対し、CBAM は「種類」と「場所」の二段構えで特徴を磨き上げます。既存の CNN にわずかに足すだけで、特に物体検出のように「どこに何があるか」が問われるタスクで精度が上がります。

## 直感的な理解
SE は「今このシーンではどの種類の特徴(チャネル)を強調すべきか」を判断する仕組みでした。これは強力ですが、空間の位置をまったく考慮しません。SE はチャネルを潰すときに画像全体を平均してしまうため、「画面のどこに注目すべきか」という情報が消えてしまうのです。チャネル全体に同じ重みを掛けるので、同じチャネルなら画面の左上も右下も同じ扱いになります。

しかし自動運転や物体検出のような場面では、場所が決定的に重要です。同じ「人の輪郭らしさ」というチャネルが反応していても、それが空のあたりで反応していれば背景の雲や模様による誤検出かもしれず、横断歩道のあたりで反応していれば本物の歩行者である可能性が高い。つまり、同じ種類の特徴でも、どこに出ているかで意味が大きく変わります。

CBAM は、人間が物を見るときの二段階の注意に似ています。まず「何に注目するか(人を探そう)」を決め、次に「画面のどこを見るか(右下の方が怪しい)」を決める。CBAM もまずチャネル(何)を絞り、次に空間(どこ)を絞ることで、SE の弱点を補い、より文脈に合った特徴強調を実現します。種類のフィルタと場所のフィルタを直列に重ねる、と覚えると分かりやすいです。

## 基礎: 前提となる概念
- アテンション (注意): ネットワークが大事なところに重みを置いて見る仕組み。重みは入力に応じて動的に変わります。
- チャネルアテンション: 「どの種類の特徴を強調するか」を決める注意。例えば今は人の輪郭のチャネルを重視する、といった重み付けです。SE が代表例で、出力は [C,1,1] 形の重み(チャネルごとに 1 個)です。
- 空間アテンション: 「画像の縦横どの位置を強調するか」を決める注意。例えば画面右下の歩行者がいそうな領域を重視する、といった位置ごとの重み付けです。出力は [1,H,W] 形の重み(位置ごとに 1 個)です。
- 平均プーリング / 最大プーリング: ある軸方向に値をまとめる操作。平均は「全体の平均反応」、最大は「最も強い反応」を取り出します。平均は全体傾向に、最大は局所的に突出した手がかりに敏感で、両者は相補的な情報を持ちます。
- 受容野 (receptive field): 1 つの出力が入力画像のどれだけ広い範囲を見て決まるか。畳み込みのカーネルサイズで決まり、CBAM の空間注意では 7×7 の畳み込みがこれを決めます。

[Squeeze-and-Excitation (SENet)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1085_squeeze-and-excitation) はチャネルごとの重要度を動的に調整して成功しましたが、空間情報をグローバル平均で潰すため「どこに注目するか」は扱えませんでした。CBAM はチャネルと空間の両方に注意を向けることで、この弱点を補います。重要なのは、チャネル注意と空間注意がそれぞれ別の軸(チャネル軸 C と空間軸 H×W)を潰して相手の軸の重みを作る、という対称的な構造になっている点です。

## 仕組みを詳しく
CBAM は 2 つのサブモジュールを直列(順番)に適用します。入力特徴を「チャネル C × 縦 H × 横 W」とします。

### 1. チャネルアテンション(何を強調するか)
SE と似ていますが、平均だけでなく最大値も使うのが違いです。

1. 各チャネルについて、H×W を平均プーリング(全体の平均反応)と最大プーリング(最も強い反応)の両方で 1 個ずつに圧縮します。最大プーリングを足すのは、平均では薄まってしまう「局所的に強い手がかり」を拾うためです。例えば小さな信号機が画面の片隅に強く反応していても、平均では埋もれますが最大なら残ります。
2. それぞれを、重みを共有する小さな全結合層(MLP、中間で C/r に絞る)に通します。共有することでパラメータを節約し、平均経路と最大経路を同じ尺度で評価します。
3. 2 つの出力を足し合わせ、Sigmoid で 0〜1 のチャネル重みにします。
4. 元の特徴マップの各チャネルに掛けます。

```
[C, H, W]
├ 平均プーリング → [C] → 共有MLP ┐
└ 最大プーリング → [C] → 共有MLP ┘ → 足す → Sigmoid → [C,1,1] チャネル重み → 入力に掛ける
```

### 2. 空間アテンション(どこを強調するか)
チャネルアテンションを掛けた後の特徴に対して行います。

1. 各画素位置について、全チャネルを平均と最大で 1 個ずつに圧縮します。結果は「平均地図 [1,H,W]」と「最大地図 [1,H,W]」の 2 枚です。チャネル方向を潰すことで、各位置の「総合的な活躍度」を取り出します。
2. この 2 枚を連結して [2,H,W] にし、7×7 の畳み込み 1 枚に通して [1,H,W] の空間重み地図を作り、Sigmoid で 0〜1 にします。7×7 という大きめのカーネルは、各位置の重要度を決めるのに周囲の文脈もある程度見るためです(隣だけ見て決めるのではなく、少し広い近傍を考慮する)。
3. 元の特徴マップの全チャネルに、この空間重みを位置ごとに掛けます。

```
[C, H, W]
├ チャネル方向に平均 → [1, H, W] ┐
└ チャネル方向に最大 → [1, H, W] ┘ → 連結 [2,H,W] → 7×7 conv → Sigmoid → [1,H,W] 空間重み → 入力に掛ける
```

### 形状と順序でまとめる
```
入力 [C,H,W]
 → チャネルアテンション(重み [C,1,1] を全空間にブロードキャストして掛ける)→ [C,H,W]
 → 空間アテンション  (重み [1,H,W] を全チャネルにブロードキャストして掛ける)→ [C,H,W](出力)
```

「種類で絞ってから、場所で絞る」二段フィルタだと理解すると分かりやすいです。論文はこの「チャネル → 空間」の順を推奨しています(逆順や両者を並列に適用する構成も試し、この順が最良だったと報告)。追加計算は SE よりやや多い程度で、依然として非常に軽量です。なお全体は通常、残差的に元の特徴に作用させる(過度に消しすぎないよう恒等成分を残す)実装と組み合わせて使われます。

## 手法の系譜と主要論文

- Squeeze-and-Excitation (Hu et al., CVPR 2018, arXiv:1709.01507)。CBAM の直接の前身で、チャネルのみのアテンション(詳細は [Squeeze-and-Excitation (SENet)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1085_squeeze-and-excitation))。空間を扱えない弱点が CBAM の出発点になりました。

- BAM: Bottleneck Attention Module (Park et al., BMVC 2018, arXiv:1807.06514)。CBAM と同じ著者グループ(KAIST / Lunit / Adobe)による兄弟手法で、チャネル注意と空間注意を並列に計算して足し合わせ、ネットワークのボトルネック(ダウンサンプリング位置、ステージの切れ目)にだけ挿入します。空間注意には拡張畳み込み(dilated convolution、フィルタの間隔を空けて受容野を広げる工夫)を使い、少ない計算で広い範囲を見ます。

- CBAM: Convolutional Block Attention Module (Woo et al., ECCV 2018, arXiv:1807.06521)。BAM の並列統合に対し、CBAM はチャネル注意と空間注意を直列に適用し、すべての畳み込みブロックに挿入できる軽量モジュールにしました。平均プーリングだけでなく最大プーリングも併用する点が SE/BAM との違いです。CBAM(直列・全ブロック)と BAM(並列・ボトルネックのみ)は、二種の注意をどう組み合わせ(直列か並列か)どこに置くか(全ブロックかボトルネックのみか)という設計選択肢を体系化しました。

- Coordinate Attention (Hou et al., CVPR 2021, arXiv:2103.02907)。CBAM の空間注意がチャネルを潰して位置情報を弱めてしまう点を改良し、縦方向と横方向に別々にプーリングして座標(位置)情報を保ったままチャネル注意に埋め込む、モバイル向けの効率的な後継です。

系譜としては、SE(チャネル注意)→ BAM/CBAM(空間注意を追加、組み合わせ方を探索)→ Coordinate Attention(位置情報を保ちつつ軽量化)という流れで、その後 Transformer の自己注意([PVT (Pyramid Vision Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0402_pvt-pyramid-vit) など)が空間方向の注意をより一般的・大域的に扱う形で発展していきます。CBAM はこの過渡期に「軽量 CNN 注意」の代表として広く使われました。

## 論文の実験結果(定量データ)
評価は ImageNet 分類(top-1 / top-5 誤り率、小さいほど良い)と物体検出(MS COCO / PASCAL VOC、指標は mAP = mean Average Precision、大きいほど良い)で行われました。報告された目安は次のとおりです。

```
分類 (ImageNet, top-1 誤り率)
ResNet-50           約 24.6%   ベースライン
SE-ResNet-50        約 23.1%   チャネル注意のみ
CBAM-ResNet-50      約 22.7%   チャネル + 空間
ResNet-101          約 23.4%   ベースライン(より深い)
CBAM-ResNet-101     約 21.5%   CBAM 追加
MobileNet           約 31.4%
CBAM-MobileNet      約 29.0%   軽量モデルでも約 2.4 ポイント改善
```

ここを初心者向けに読み解きます。CBAM を ResNet-50 に足すと top-1 誤り率がベースラインから約 1.9 ポイント下がり、しかも SE(チャネルのみ)よりさらに約 0.4 ポイント改善しています。この「SE を上回る差」が空間注意の貢献分です。1 ポイント未満の差は小さく見えますが、追加コストがごくわずかであること、そして MobileNet・WideResNet・ResNeXt など多様なバックボーンで一貫して同じ傾向が出たことから、信頼できる改善とみなされています。とくに軽量な MobileNet で約 2.4 ポイントと大きく効いた点は、表現力が乏しいモデルほど「限られた特徴を賢く使い分ける」注意の恩恵が大きいことを示唆します。

検出タスクではより効果が顕著で、論文では CBAM をバックボーンに組み込むと MS COCO の mAP がベースライン比でおよそ 1〜2 ポイント、VOC でも明確に向上したと報告されています。これは「どこに物体があるか」を当てる検出では、空間注意が直接的に効くためで、CBAM の設計動機(SE が場所を扱えない)と整合します。さらに Grad-CAM(判断根拠を可視化する手法)による解析で、CBAM 付きモデルの方が対象物体の領域に正しく注目している(背景に惑わされにくい)ことも示されました。

アブレーション(要素を抜く検証)で報告された主な知見:
- プーリングの種類: チャネル注意で平均だけでなく最大プーリングを併用すると、平均のみ(SE 相当)より精度が上がる。両方を使う設計が有効。
- 配置順序: チャネル → 空間の順が、空間 → チャネルの順や両者並列より良かった。
- 空間注意のカーネル: 空間注意の畳み込みは 3×3 より 7×7 の方が良く、ある程度広い受容野が空間重みの推定に有効。
- どちらか一方を抜く: チャネル注意だけ、空間注意だけより、両方を組み合わせた方が良い。二種の注意は相補的。

## メリット・トレードオフ・限界
メリット
- チャネル(何)と空間(どこ)の両方に注意でき、SE より文脈に即した強調ができる。
- 既存 CNN にブロック単位で後付けでき、特に物体検出で効果が出やすい。場所が効くタスクに向く。
- 追加計算量が小さく、軽量バックボーンから大型まで幅広く適用できる汎用性。Grad-CAM で見ると注目領域が改善し、説明可能性の観点でも有利。

トレードオフ・限界
- SE より計算とパラメータがやや多い(空間注意の 7×7 畳み込みと、追加の最大プーリング経路の分)。
- 空間アテンションは 7×7 畳み込みベースで受容野が局所的なため、画像全域にまたがる長距離の関係は捉えにくい。そこは自己注意ベースの Transformer 系([PVT (Pyramid Vision Transformer)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0402_pvt-pyramid-vit) など)が得意とする領域です。
- 空間注意がチャネルを潰す際に位置情報が弱まる(Coordinate Attention が突いた弱点)。
- チャネル → 空間の適用順や、直列/並列(BAM)、挿入位置といった設計の自由度が増える分、最適点を見つけるチューニングが要る。また逐次に二つの注意を通すためメモリアクセスと逐次依存が増え、軽量モデルでは FLOPs の小ささに反してレイテンシが伸びることがある。
- SE と同じく、注意はあくまで既存特徴の再重み付けであり、新しい情報を生むわけではない。下流タスクによっては効果が小さい場合がある。

## 発展トピック・研究の最前線
CBAM 以降、チャネルと空間の注意をどう統合し効率化するかが研究テーマになりました。Coordinate Attention(Hou et al., CVPR 2021)は、空間注意の際に縦方向と横方向のプーリングを別々に行うことで位置情報を保ちつつ軽量化し、モバイル向けに最適化しました。Triplet Attention は 3 つの軸(C-H、C-W、H-W)間の相互作用を捉え、軸をまたぐ依存を安く扱います。一方で、Transformer の自己注意が空間方向の長距離依存を全面的に扱えるようになると、CBAM のような局所的な畳み込みベース空間注意は、軽量 CNN の補強という役割に落ち着きました。

現在の主流は CNN と注意機構のハイブリッドで、局所性を持つ畳み込みと大域性を持つ注意を組み合わせる設計(CoAtNet、MobileViT、MaxViT など)が高い精度と効率を両立しています。CBAM が示した「チャネルと空間という二つの軸に分けて注意を考える」という枠組みは、こうした現代的設計の基礎的な考え方として今も参照されます。研究の最前線では、注意マップを判断根拠の可視化(説明可能性、安全クリティカルなシステムでの根拠提示)に使う応用や、検出・分割といった密予測タスクで注意をどこにどう挿入すると効くかの系統的な分析、そして注意の逐次依存を減らして推論を並列化・高速化する実装研究が続いています。
