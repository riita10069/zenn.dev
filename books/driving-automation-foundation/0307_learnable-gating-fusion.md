# 学習ゲーティング融合

## ひとことで言うと

2つ以上の特徴ストリーム(情報の流れ)を混ぜるとき、「どちらをどれだけ取り込むか」を固定の比率ではなく、学習で決まる重み(ゲート)で制御する融合方法です。ゲートとは「門の開き具合」のことで、0なら閉(その情報を通さない)、1なら全開(そのまま通す)を意味します。この門の開き具合をネットワークが学習で自動調整するので、有用な情報ストリームは強く取り込み、邪魔なものは絞り込む、という賢い混ぜ方ができます。特にゲートを0から学習させる「ゼロ初期化」を組み合わせると、新しいモジュールを足しても学習の出だしを壊さず、安定して訓練を始められます。

## 直感的な理解

料理に例えます。スープ(基準となる特徴)に新しい調味料(追加の特徴)を加えるとき、いきなり大量に入れると味が壊れます。だから少しずつ味見しながら、効果のある調味料は多めに、合わないものはほぼ入れない、と量を調整しますよね。学習ゲーティングはこの「味見しながら量を決める」をネットワーク自身にやらせる仕組みです。

ニューラルネットワークでは、複数の情報源(センサーや処理経路)から特徴を集めて1つにまとめる場面が頻繁にあります。最も単純なまとめ方は単純加算(A + B)です。しかしこれには根本的な弱点があります。

- 単純加算は「常に1:1で混ぜる」ことを意味します。もし片方のスケール(数値の大きさ)がもう片方より極端に大きいと、小さい方の有用な情報が埋もれてしまいます。
- 学習の初期段階では、新しく追加した経路(例えば地図情報を処理するモジュール)はまだランダムな値しか出さず、意味のないノイズです。それを最初から1:1で足すと、もともと安定して学習できていた基準経路まで巻き添えで壊れます。
- そもそも「どちらをどれだけ信用すべきか」は、特徴の種類(チャネル)ごと、状況ごとに違うはずです。固定比率ではこの違いを表現できません。

学習ゲーティングはこれら全てに答えます。混ぜる比率そのものを学習可能なパラメータにすることで、ネットワークが訓練データから「最適な混ぜ具合」を自分で見つけ出します。

## 基礎: 前提となる概念

このトピックを理解するために、いくつかの用語を噛み砕いておきます。

特徴マップ(feature map)とは、ニューラルネットワークの途中で計算される中間データのことです。画像を扱うネットワークでは、典型的に [B, C, H, W] という4次元のテンソル(多次元配列)で表されます。B はバッチサイズ(同時に処理する画像の枚数)、C はチャネル数(特徴の種類の数、例えば256)、H と W は特徴マップの縦横のサイズです。「チャネル」というのは、各位置で「色」「エッジ」「テクスチャ」のように異なる種類の情報を並列に持っている、その本数のことだと思ってください。

パラメータ(parameter)とは、学習で値が更新される変数のことです。ニューラルネットの重みはすべてパラメータです。ある変数を「学習対象」と宣言すると、その変数は誤差逆伝播(backpropagation、出力の誤差を入力側へ伝えて各重みをどう直すか計算する仕組み)の対象になり、訓練のたびに少しずつ更新されます。

ブロードキャスト(broadcast)とは、形の違うテンソル同士を計算するとき、小さい方を自動的に引き伸ばして大きい方に合わせる仕組みです。例えば形 [256] のベクトルを形 [B, 256, H, W] のテンソルに掛けると、ベクトルの各値がチャネル方向に当てはめられ、すべての B・H・W の位置で同じチャネルには同じ重みが掛かります。

恒等写像(identity mapping)とは、入力をそのまま出力する関数 f(x) = x のことです。「何もしない」処理ですが、深いネットワークの学習を安定させる上で極めて重要な役割を持ちます。

シグモイド関数(sigmoid)とは、任意の実数を 0〜1 の範囲に押し込む関数 σ(z) = 1 / (1 + e^(−z)) です。ゲートを「0〜1の比率」として表したいとき頻繁に使われます。z が大きければ 1 に、小さければ 0 に近づき、z=0 で 0.5 になります。

## 仕組みを詳しく

学習ゲーティング融合にはいくつかの形がありますが、最も基本的なのは「加法的ゲーティング(additive gating)」です。基準となる特徴に、追加の特徴をゲート重みで調整して足し込みます。

```
output = base + gate ⊙ extra
```

ここで base は基準ストリーム、extra は追加ストリーム、gate は学習可能な重み、⊙ は要素ごとの掛け算(アダマール積)です。gate の形には選択肢があり、それぞれパラメータ数と表現力が変わります。

- スカラ1個: ストリーム全体を一律に何倍するか。最もパラメータが少なく(1個)、表現力は低い。
- チャネルごとのベクトル(形 [C]): チャネルごとに別々の重み。どの種類の特徴を取り込むかを選択的に制御できる。パラメータは C 個(例: 256)。
- 空間ごと・入力依存の動的ゲート: 入力に応じてゲート値を毎回計算する。最も柔軟だが計算コストが高い。

チャネルごとのベクトル版を、具体的なテンソル形状で追ってみます。BEV(鳥瞰図特徴)を2経路融合する典型例で考えます。

```
base  [B, 256, H, W]   (基準の特徴、例えばカメラ由来)
extra [B, 256, H, W]   (追加の特徴、例えば地図由来)
alpha [256]            → reshape → [1, 256, 1, 1]   各チャネルに1つの重み

出力[b, c, h, w] = base[b, c, h, w] + alpha[c] * extra[b, c, h, w]
```

alpha を [1, 256, 1, 1] に変形してからブロードキャストで掛けることで、チャネル c ごとに「追加特徴のチャネル c をどれだけ足すか」を alpha[c] が制御します。バッチ b や空間位置 h, w には依存しない、つまり画像のどこでも同じチャネルには同じ重みが掛かる、というのがこの方式の特徴です。

ここで決定的に重要なのが「ゼロ初期化」です。alpha を0から学習開始すると、訓練の出だしでは次のようになります。

```
学習開始時 (alpha = 0):
  output = base + 0 * extra = base
  → 追加ストリームは一切寄与しない。
    extra を処理するモジュールが存在しないモデルと完全に同一の出力。

学習が進むと:
  有用なチャネルの alpha が正方向に育ち、そのチャネルの extra を強く取り込む。
  役に立たないチャネルの alpha は0付近に留まり、無視され続ける。
```

before/after を数値で見ます。仮に学習後、車線境界に対応するチャネル42の alpha が 0.8、空を表すチャネル200の alpha が 0.02 に収束したとします。すると地図の車線境界情報はしっかり取り込まれ、地図の無意味な背景は事実上無視される、という選択的な融合が実現します。

なぜゼロ初期化がこれほど効くのか。深いネットワークを訓練するとき、最も安定するのは「最初は恒等写像に近い」状態です。新しいモジュールがいきなりノイズを注入すると、それまで安定していた勾配の流れが乱れ、損失が発散したり収束が遅れたりします。ゼロ初期化のゲートは「最初は何もしない、必要なぶんだけ徐々に開く」を保証するので、追加モジュールを安全に導入できます。これが残差接続(residual connection)やゲーティングが深層学習で広く使われる根本理由です。

加法的ゲーティングと並んで重要なのが「凸結合(convex combination)型ゲーティング」です。これは2つを足して1になる重みで混ぜます。

```
output = gate ⊙ A + (1 - gate) ⊙ B,   0 ≤ gate ≤ 1
```

gate を sigmoid 関数で作れば、「A を gate ぶん、B を残りぶん」という比率混合になります。後述の Highway Networks がこの形を使いました。加法的ゲーティングとの違いは、こちらが「どちらかを増やせばもう片方が減る」トレードオフ型なのに対し、加法的は「基準は常にそのまま、追加分だけ調整」という非対称型である点です。タスクが「2つの源のどちらを信じるか」という相互排他的な性質なら凸結合、「主たる源に補助情報をどれだけ足すか」という非対称な性質なら加法的、と使い分けます。

動的ゲートの作り方も具体化しておきます。入力依存ゲートは、典型的には特徴を圧縮した記述子から小さなネットワークでゲート値を出します。例えば squeeze-and-excitation 型では、特徴 [B,C,H,W] を空間平均で [B,C] に潰し(グローバル平均プーリング)、全結合層 C→C/r→C(r は縮約率、例えば16)で通し、最後に sigmoid を掛けて [B,C] のゲートを得て、元の特徴にチャネル方向ブロードキャストで掛けます。縮約率 r はパラメータ数を抑えつつ表現力を保つトレードオフのつまみで、大きくするほど軽量・低表現になります。

計算コストの観点では、固定の学習ゲーティング融合は極めて軽量です。掛け算と足し算が1回ずつ、特徴マップの要素数(B×C×H×W)に比例するだけで、クロスアテンション融合([Multi-Head Attention融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0312_multihead-attention-fusion))のような (H·W)² に比例する重い計算は一切要りません。これがゲーティングを実務で好んで使う大きな理由です。

## 手法の系譜と主要論文

Highway Networks (Srivastava, Greff, Schmidhuber、2015、arXiv:1505.00387)
- 提案: 各層で「変換した出力 H(x)」と「素通りの入力 x」を、学習ゲート T(x) で混ぜる。式は y = H(x)·T(x) + x·(1 − T(x)) で、T は入力 x から計算される sigmoid 出力(0〜1)です。T が transform gate、(1−T) が carry gate と呼ばれます。
- 動機: 当時、層を深くするほど勾配消失(誤差が入力側へ伝わるうちに小さくなりすぎて学習できなくなる現象)でネットワークが訓練できなくなる問題がありました。LSTM のゲート機構を順伝播ネットに持ち込み、情報を素通りさせる経路を学習で作ることで、これを克服しようとしました。
- 新規性と効果: 学習可能ゲートで「混ぜ具合そのものを学ぶ」発想の源流です。100層を超えるネットワークの訓練を可能にしたことを示しました。トレードオフはゲート計算用に追加パラメータと計算が要ること。

Deep Residual Learning (ResNet; He, Zhang, Ren, Sun、CVPR 2016、arXiv:1512.03385)
- 提案: Highway のゲートを「常に開(carry gate = 1)」に固定した究極の単純化で、y = F(x) + x という残差接続を導入しました。
- 動機: Highway はゲート計算がオーバーヘッドになる。ゲートを学習せず常に素通りさせるだけでも深層化の恩恵が得られるのではという仮説です。
- 効果: ImageNet で152層という当時前例のない深さを達成し、トップ5エラー率を大幅に改善しました。学習ゲーティングの「ゲートを定数に固定した特殊ケース」と位置づけられます。加法的ゲーティングの output = base + alpha·extra は、基準を素通りさせ追加分にだけゲートを掛ける構造で、ResNet と Highway のちょうど中間にあたります。

Gated Linear Units (GLU; Dauphin, Fan, Auli, Grangier、ICML 2017、arXiv:1612.08083)
- 提案: 線形変換した特徴をもう一つの sigmoid ゲートで掛ける y = (X·W) ⊙ σ(X·V) という形を、言語モデリングで導入しました。
- 効果: ゲーティングが系列モデリングの勾配伝播を改善し、当時の言語モデルで競争力ある性能を示しました。後の Transformer 系(GLU を取り入れた GeGLU/SwiGLU など)に影響しています。

Squeeze-and-Excitation Networks (SENet; Hu, Shen, Sun、CVPR 2018、arXiv:1709.01507)
- 提案: チャネルごとの重要度(0〜1)を入力から動的に予測し、特徴チャネルを再重み付けします。具体的には特徴を空間方向に平均圧縮(squeeze)し、小さな全結合層で各チャネルの重み(excitation)を計算して掛けます。
- 動機: チャネルによって重要度が違うのに一律に扱っていた。入力に応じてチャネルを選択的に強調・抑制したいという発想です。
- 効果: ImageNet 2017 で優勝。わずかな計算追加で多くのネットワークの精度を底上げできることを示しました。チャネルごとゲートという点で学習ゲーティングと同じ系譜ですが、SE は入力依存で毎回動的に計算するのに対し、固定の学習ベクトル版は学習後は1組の値で固定、という違いがあります。

ReZero is All You Need (Bachlechner, Majumder, Mao, Cottrell, McAuley、UAI 2021、arXiv:2003.04887)
- 提案: 残差ブランチにゼロ初期化のスカラ alpha を掛けます。output = x + alpha · F(x) で、alpha の初期値は0です。
- 動機: 深いネットワークや Transformer の学習を安定・高速化したい。出だしを恒等写像にすれば勾配が素直に流れます。
- 効果: 数千層規模のネットワークや深い Transformer の収束を大幅に高速化しました。ゼロ初期化ゲートの有効性を理論・実験の両面で裏付けた重要論文です。

LayerScale (CaiT; Touvron, Cord, Sablayrolles, Synnaeve, Jégou、ICCV 2021、arXiv:2103.17239)
- 提案: 深い Vision Transformer の各残差ブランチに、ごく小さい値(例: 1e-4 や 1e-6)で初期化したチャネルごとの学習ベクトルを掛けます。
- 効果: 深い ViT の訓練を安定させ精度を改善。ゼロに近い小さな初期値でゲートを開き、チャネルごとに寄与を調整するという、加法的ゲーティングのチャネルベクトル版に直接対応する設計です。

この系譜をまとめると、「ゲートで情報を選択的に流す」(Highway/GLU)→「ゲートを定数に固定して単純化」(ResNet)→「ゼロ近傍初期化で安全に導入」(ReZero/LayerScale)→「チャネルごとに動的選択」(SENet)という流れで、学習ゲーティングは深層学習の安定化と選択的融合の中核技術として確立してきました。

## 論文の実験結果(定量データ)

Highway Networks は、ゲートなしの素朴な深層ネットでは層を深くすると訓練誤差がむしろ増える(最適化が破綻する)のに対し、ゲートを入れると100層級でも訓練誤差が下がり続けることを示しました。これは「ゲートが最適化の難しさそのものを緩和する」直接の証拠です。

ResNet は ImageNet 分類で、プレーンな34層ネット(残差接続なし)が18層ネットより誤差が高い(深いのに悪化する劣化問題)のに対し、残差接続を入れた ResNet-34 は ResNet-18 より明確に良く、さらに152層まで深くしてトップ5エラー率を当時最高水準(報告では検証セットで 5%前後)まで下げました。指標はトップ5エラー率、つまり「正解が予測上位5位以内に入らなかった割合」で、低いほど良い指標です。残差接続(ゲートを1に固定したゲーティング)という構造変更だけで深層化の恩恵が取り出せたことが要点です。

SENet は ImageNet で、SE ブロックを足すだけでベースラインの ResNet-50 のトップ5エラー率をおよそ1ポイント弱改善し(報告では ResNet-50 比でトップ1エラーが約1.5ポイント改善)、追加計算はモデル全体の数パーセント未満に収まることを報告しました。アブレーション(構成要素を抜く実験)では、squeeze(空間圧縮)を外すとチャネル重みの推定が悪化し効果が減ること、excitation の活性化に sigmoid を使うのが ReLU や tanh より良いことが示されています。「アブレーション」とは、ある部品を取り除いて性能がどう落ちるかを見ることで、その部品が本当に効いているかを検証する実験手法です。

ReZero は、12層 Transformer の言語モデリングで、標準の初期化に比べて収束に必要なステップ数をおよそ半分以下に短縮できたと報告しています。また Transformer の層正規化(LayerNorm)やウォームアップ(学習率を最初ゆっくり上げる工夫)に頼らずとも、ゼロ初期化ゲートだけで安定して深い層を訓練できることを示しました。

LayerScale は ImageNet で、深い ViT(例えば36層級)を素朴に訓練すると精度が頭打ちまたは悪化するのに対し、LayerScale を入れると深さに応じて精度が伸び続け、最終的にトップ1精度(正解が予測1位だった割合)を改善することを示しました。アブレーションでは初期値が大きすぎると安定化効果が薄れ、十分小さい値(1e-4 など)で最も安定することが報告されています。

これらの結果に共通するメッセージは、「ゲートの初期値を恒等写像に近づけるほど、深いモデルの訓練が安定し最終精度も上がる」という点です。学習ゲーティング融合でゼロ初期化を採るのは、この一連の実証に裏打ちされた設計判断です。

## メリット・トレードオフ・限界

メリット
- ゼロ初期化と組み合わせれば、新しいモジュール(追加センサーや地図など)を足しても既存の学習を壊さず、安定して訓練を始められる。
- 計算が掛け算と足し算だけで非常に軽い。アテンションの (H·W)² のような重い計算が不要で、リアルタイム推論に向く。
- チャネルごとベクトル版なら「どの種類の特徴が有用か」をチャネル単位で取捨選択できる。
- 解釈性がある。学習後の alpha を見れば、どのストリーム・どのチャネルがどれだけ取り込まれているかが読み取れる。

トレードオフと限界
- 固定ゲート版(学習後に値が固定されるタイプ)は、場所ごと・入力ごとに動的に重みを変えられない。「晴れの日は地図を信用し、霧の日はカメラを信用する」といった状況依存の切り替えはできない。これには SENet のような入力依存ゲートが必要。
- 加法的ゲーティングは「同じ位置の特徴同士を足す」だけなので、2つのストリーム間に空間的なズレ(位置合わせ誤差)があっても補正できない。空間的な対応づけが要る場合はクロスアテンションが必要。
- ゲートが過剰に大きく育つと基準特徴を相対的に薄めてしまうリスクがある。正則化(重みが大きくなりすぎないようにする工夫)や学習率調整が要る場合がある。
- ゲートが学習の早い段階で「閉じたまま」に固定されてしまい(0付近から動かなくなる)、追加ストリームが永遠に使われなくなる失敗もある。これは追加ストリーム側の特徴がそもそも有用な勾配を返せていないときに起きやすく、デバッグが難しい。一度ゲートが0に張り付くと、そのストリームに勾配がほとんど流れず学習が進まないので、初期化を完全な0ではなくごく小さな正値にする、補助損失で追加ストリームを直接監督する、といった対処が取られる。

研究上の未解決課題としては、「どのゲート形(スカラ/チャネル/空間/入力依存)がどのタスクで最適か」を事前に予測する理論がまだ確立していない点が挙げられます。また、複数(3つ以上)のストリームを同時にゲーティングするとき、ゲート間の相互作用(softmax で正規化して総和を1にするのか、各々独立に開くのか)をどう設計するかも探索の余地が大きい領域です。

## 発展トピック・研究の最前線

学習ゲーティングは Mixture-of-Experts(MoE、複数の専門家ネットワークを入力に応じてゲートで選ぶ仕組み)の中核でもあります。GShard(Lepikhin et al., 2020, arXiv:2006.16668)や後続の大規模モデルでは、トークンごとにどの専門家を活性化するかをゲートが決め(典型的には top-1 や top-2 のスパースなゲーティング)、計算量を一定に抑えつつモデル容量(総パラメータ数)を大きく増やす技術として再注目されています。融合の文脈では、モダリティ(センサー種別)ごとに専門家を割り当て、状況に応じて重み付ける「モダリティゲーティング」が、頑健なマルチモーダル融合の有力アプローチになっています。

もう一つの最前線は「動的・条件付き計算」です。ゲートを入力依存にして、信頼できない情報源を実行時に動的に遮断する設計は、センサー故障や悪天候に強い自動運転知覚の鍵とされています。例えば、片方のセンサーが汚染されたらそのゲートを自動で閉じる、という頑健化です。各情報源の不確実性を推定し、不確実なほどゲートを絞る「不確実性重み付け融合」も同じ系統にあります。

さらに、ゲートの初期化と正則化の理論研究も進んでいます。ゼロ初期化がなぜ効くのかを、訓練初期の損失地形(loss landscape)の平坦さや勾配の素直さの観点から定量的に説明する研究が続いており、より深いモデルを安定に訓練する処方箋が整理されつつあります。
