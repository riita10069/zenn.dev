# 凍結バックボーン vs EMAターゲットエンコーダ

## ひとことで言うと

「未来や隠された部分の特徴を予測する」タイプの自己教師あり学習では、予測の正解を作る側のネットワーク(ターゲットエンコーダ)を、予測する側と同じように自由に学習させてはいけません。両方を自由に動かすと、ネットワークは「入力に関係なく全部同じ値を出す」というズルをして損失だけを下げてしまうからです。これを防ぐため、正解側を非対称(asymmetric)に扱います。代表的な選択肢が2つあります。(1) 事前学習した重みを一切動かさない「凍結(frozen)」、(2) 予測側の重みをゆっくり真似させる「EMA(指数移動平均)」。この2方式の仕組み・効果・選び方を、研究の系譜とともに掘り下げます。

## 直感的な理解

学習を「生徒(予測側)が先生(正解側)の答えに近づける」プロセスだと考えてみます。普通の学校なら先生の答えは固定された正しいものですが、もし先生が「生徒の答えに合わせて自分の答えをこっそり書き換えてもよい」としたらどうなるでしょうか。生徒と先生は示し合わせて「両方とも『42』とだけ書く」ことに決めれば、答えは常に一致し、テストは満点です。しかし何も学んでいません。これがまさに、予測側と正解側を両方学習可能にしたときに起きる崩壊です。

防ぐ方法は単純で、「先生が生徒に合わせて答えを変えられないようにする」ことです。やり方の一方の極が「先生の答えを最初から最後まで一字も変えない」凍結で、もう一方が「先生は生徒の進歩をゆっくり取り入れるが、急には変わらない」EMAです。どちらも「正解が予測側に瞬時に擦り寄る」ことを禁じている点で共通しています。この非対称性こそが、ネガティブサンプル(後述)を使わずに崩壊を防ぐ鍵だと、2020年前後の一連の研究が明らかにしました。

## 基礎: 前提となる概念

いくつかの用語を、知らない人向けに噛み砕きます。

- **エンコーダ(encoder)/バックボーン(backbone)**: 入力(画像や動画フレーム)を受け取り、意味の凝縮された数値ベクトル(特徴, feature)に変換するニューラルネットワークのことです。ResNet や Vision Transformer (ViT)、Swin Transformer などが使われます。「バックボーン」は特に、システムの土台となる特徴抽出器を指す呼び方です。

- **ターゲットエンコーダ(target encoder)**: 予測の「正解」(ターゲット, target)を作る側のエンコーダです。たとえば未来フレームや隠された画像領域を入力し、予測すべき目標特徴を出力します。これに対して予測する側を「オンライン(online)エンコーダ」や「コンテキスト(context)エンコーダ」と呼びます。

- **凍結(frozen)**: パラメータ(重み)を学習で一切更新しないことです。実装上は勾配計算を止めます(`requires_grad = False`)。事前学習済みの状態にピン留めするイメージです。

- **EMA(Exponential Moving Average, 指数移動平均)**: 新しい値を一気に反映せず、過去の値に少しずつ混ぜながらゆっくり追従させる平滑化の方法です。式は `新平均 = τ × 旧平均 + (1−τ) × 新観測` で、τ(タウ)が1に近いほど動きが鈍くなります。

- **自己教師あり学習(self-supervised learning, SSL)**: 人間がラベルを付けなくても、データ自身から学習信号を作る学習方式です。「画像の一部を隠して残りから当てる」「同じ画像の2つの加工版を近づける」などが典型です。

- **ネガティブサンプル(negative sample)/対照学習(contrastive learning)**: 「同じものの別ビューは近づけ、違うものは遠ざける」という学習方式が対照学習で、その「遠ざける対象」がネガティブサンプルです。遠ざける項があるので「全部同じ値」にすると罰せられ、崩壊しません。ただし大量のネガティブを集める仕組み(大きなバッチやメモリバンク)が必要で、扱いが重いという難点があります。

- **表現崩壊(representation collapse)**: 上の「全部同じ値を出す」ズルのことです。詳しくは [表現崩壊(representation collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0600_representation-collapse) を参照。

非対称設計は、この対照学習の「ネガティブを集める手間」を回避しつつ崩壊を防ぐ、というモチベーションから生まれました。

## 仕組みを詳しく

### 方法1: 凍結バックボーン(frozen target)

ターゲットエンコーダを、事前学習済み(ImageNet 分類や別の SSL タスクで学習済み)の状態のまま一切更新しません。

```python
for p in target_encoder.parameters():
    p.requires_grad = False   # 勾配を流さない = 学習しない
```

実装が最も単純で、追加の更新計算もありません。正解の特徴は学習を通じて固定されたまま、いわば「動かない測量基準点」として働きます。予測側はこの固定基準に向かって予測精度を上げていきます。

### 方法2: EMAターゲットエンコーダ(momentum target)

ターゲットエンコーダの重み θ_target を、学習可能な予測側 backbone の重み θ_online に**ゆっくり**追従させます。毎ステップ、勾配ではなく次の更新式で動かします。

```python
for p_tgt, p_onl in zip(target.parameters(), online.parameters()):
    p_tgt.data = tau * p_tgt.data + (1.0 - tau) * p_onl.data   # tau ≈ 0.996
```

`tau`(τ, momentum coefficient)は追従の遅さを決める係数です。0.996 なら「毎ステップ、古い値を 99.6%・新しい値を 0.4%」混ぜます。ほとんど動きませんが、長期的には予測側に少しずつ近づきます。

具体的な数値で。ある重みが target=1.000、online=2.000 のとき、1ステップ後の target は `0.996×1.000 + 0.004×2.000 = 1.004`。1回では 0.004 しか動きません。だが online が改善していけば、target も遅れて追随し、正解の質そのものが学習とともに向上します。

多くの実装では τ を学習中に増やします。たとえば BYOL は τ_base=0.996 から開始し、コサインスケジュールで 1.0 に近づけます(終盤ほど target を固定に近づけて安定させる)。

### 凍結 vs EMA の違い(before/after)

```
凍結:  正解側は最初から最後までずっと「事前学習の特徴」を出す(動かない)
          時刻0 ──────────────────────────────────── 終了
          [固定特徴] [固定特徴] [固定特徴] ... [固定特徴]

EMA :  正解側は最初は予測側のコピーだが、ゆっくり予測側の改善を取り込む
          時刻0 ──────────────────────────────────── 終了
          [初期特徴] →→→ じわじわ質が上がる →→→ [良い特徴]
```

凍結は「動かない固定的な正解」、EMA は「ゆっくり良くなる移動する正解(moving target)」です。EMA は正解の質が学習とともに上がるので、最終的な表現が良くなりやすい一方、追従が速すぎる(τ が小さすぎる)と予測側に瞬時に追いついて崩壊する危険があります。実験的には τ を 0(=予測側と即同期)にすると BYOL は崩壊します。

### なぜ非対称性だけで崩壊が防げるのか(直感)

崩壊は「予測側と正解側が手を組んで定数に潰れる」過程です。stop-gradient や EMA で正解側を予測側から切り離すと、正解側は予測側の都合で動けません。すると予測側が「定数で逃げる」戦略を取っても、正解側がそこに付いてきてくれないので損失が下がらず、結局は真面目に予測を当てに行くしかなくなります。SimSiam の論文は、この stop-gradient こそが崩壊回避の本質であることを実験で切り分けました。

## 手法の系譜と主要論文

非対称ターゲットの考え方は、対照学習の改良として段階的に確立しました。

- **MoCo (He et al., CVPR 2020, arXiv:1911.05722)**: momentum encoder(=EMA で更新されるキー側エンコーダ)を導入しました。動機は、対照学習で大量のネガティブを安定して供給するために、エンコーダがゆっくり変わる「キュー(queue)」を持ちたかったことです。新規性は、キー側を EMA で更新することで、過去にキューに溜めた特徴と現在の特徴の一貫性を保てる点。これが「EMA エンコーダ = 安定した移動するターゲット」という発想の早い実例です。

- **BYOL (Grill et al., NeurIPS 2020, arXiv:2006.07733)**: online network(予測側、予測器 predictor を持つ)と target network(正解側)を用意し、target を online の EMA で更新する非対称設計を提案しました。最大の主張は「ネガティブサンプルが一切なくても崩壊しない」ことです。動機は、対照学習のネガティブ管理が面倒でバッチサイズに敏感だったため、それを取り除きたかったこと。本トピックの EMA 方式の直接の出典です。

- **SimSiam (Chen & He, CVPR 2021, arXiv:2011.10566)**: BYOL からさらに EMA すら外し、片側に stop-gradient(detach)を入れるだけで崩壊を防げることを示しました。動機は「崩壊回避に必須なのは何か」を最小要素まで切り詰めること。stop-gradient を外すと即座に崩壊することを示し、「正解側に勾配を流さないこと」が本質だと結論づけました。これは「凍結」方式(勾配を流さない)の有効性を裏付ける理論的根拠でもあります。

- **I-JEPA (Assran et al., CVPR 2023, arXiv:2301.08243)**: ピクセルではなく潜在特徴空間で「隠した領域」を予測する Joint-Embedding Predictive Architecture を提案し、ターゲットエンコーダを EMA で更新しました。BYOL/MoCo の非対称設計を、画像の潜在予測(masked prediction)へ持ち込んだ位置づけです。動画版の V-JEPA(Bardes et al., 2024, arXiv:2404.08471)も同じく EMA ターゲットを使います。

系譜をまとめると、MoCo が「EMA でキー側を安定化」→ BYOL が「EMA + 予測器でネガティブを完全に不要に」→ SimSiam が「EMA すら不要、stop-gradient で十分(=凍結に近い)」→ JEPA 系が「潜在予測に非対称設計を適用」と進みました。凍結と EMA は、この一本の流れの両端と理解できます。

## 論文の実験結果(定量データ)

評価の代表的な指標は、SSL で学習した特徴の上に線形分類器だけを足して ImageNet の分類精度を測る**線形プロービング(linear probe)精度**です。これは「特徴がどれだけ意味を保っているか」を測る標準的な代理指標で、数字が高いほど良い特徴です。

- **BYOL**: ResNet-50 バックボーンで線形プローブ top-1 約 74.3%(報告値)。当時のネガティブを使う SimCLR(約 69.3%)を大きく上回り、ネガティブ無しでも最高水準に達することを示しました。さらにアブレーションとして、target を EMA ではなく online と即同期(τ=0)させると性能が崩壊(精度がほぼランダムまで低下)し、EMA の momentum が崩壊回避に必須であることを定量的に示しています。また予測器(predictor)を外しても崩壊します。すなわち「EMA target + predictor」の両方が非対称性に効いています。

- **MoCo / MoCo v2**: ResNet-50 で線形プローブ約 60%(v1)→ 約 71%(v2、predictor と augmentation 改良後)。momentum 係数 m はおよそ 0.999 が最良で、m=0.9 のように小さくすると精度が顕著に落ちると報告されています。これは「EMA は遅いほど安定」という傾向の早い証拠です。

- **SimSiam**: ResNet-50 で線形プローブ約 71.3%。重要なアブレーションは stop-gradient のオン/オフで、stop-gradient を**外すと損失が即座にほぼ最小(崩壊)に落ち**、特徴の標準偏差がゼロに張り付くことを図で示しました。EMA を使わなくても stop-gradient だけで BYOL に迫る精度が出る、というのが核心的な結果です。

- **I-JEPA**: ViT-H バックボーンで、ピクセル再構成型(MAE 等)より少ない計算量で同等以上の下流性能を達成したと報告。EMA ターゲット(同論文では target encoder)を使い、線形・微調整いずれの評価でも競争力を示しました。

これらの数値から読み取れる一般則は、(a) 非対称設計だけでネガティブ無しでも高い表現品質に届く、(b) EMA は遅いほど(τ が大きいほど)安定、(c) stop-gradient/凍結という「勾配を切る」操作が崩壊回避の最小十分条件、という3点です。

## メリット・トレードオフ・限界

凍結方式
- メリット: 実装が最も単純。追加の更新計算もメモリ更新もない。正解が絶対に動かないので崩壊リスクが最小。事前学習済みの強い特徴をそのまま基準にできる。
- トレードオフ: 正解の質が事前学習時点で固定され、学習が進んでも改善しない。事前学習ドメイン(例: 一般物体分類)が対象ドメイン(例: 運転シーン)と乖離していると、的外れな正解を与え続け、予測側の到達点が頭打ちになる(ドメインギャップ問題)。

EMA方式
- メリット: 正解の質が学習とともに向上し、最終表現が良くなりやすい。対象ドメインに自然に適応する。BYOL/MoCo/JEPA で実績豊富。
- トレードオフ: τ の調整が必要で敏感。速すぎる(τ小)と予測側に追いついて崩壊、遅すぎる(τが1に近すぎ)と改善が反映されず学習が遅い。ターゲット用の重みを別途保持し毎ステップ更新するため、メモリと計算がわずかに増える。τ のスケジュール設計も性能に影響する。

共通の限界
- どちらも正解側エンコーダを別に保持するためメモリを食う(予測側とほぼ同サイズ)。
- 完全崩壊は防げても**次元崩壊**(一部の次元しか使われない、[表現崩壊(representation collapse)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0600_representation-collapse) 参照)は残りうるため、特徴の分散監視が別途必要。
- ターゲット特徴の次元・解像度が予測側の出力と合わない場合、間に射影層(projection layer)を挟む必要がある。

未解決の課題として、「凍結と EMA のどちらが優れるかはドメインとデータ量に依存し、事前に決め打ちできない」点が挙げられます。小規模データや事前学習が強い場合は凍結が安全、大規模で対象ドメインが特殊な場合は EMA が伸びる傾向がありますが、理論的な選択基準はまだ確立されていません。

## 発展トピック・研究の最前線

- **DINO / DINOv2 (Caron et al., 2021 / Oquab et al., 2023)**: EMA ターゲット(teacher)に加え、出力分布の centering と sharpening(温度付き softmax)で崩壊を防ぐ蒸留型 SSL。EMA teacher の系譜を ViT に拡張し、教師なしセグメンテーションにも有効な特徴を学びます。
- **正則化型の代替**: VICReg / Barlow Twins は EMA も stop-gradient も使わず、分散・共分散正則化で崩壊を防ぎます。非対称設計とは別系統の崩壊回避策で、設計がシンプルというメリットがあります。
- **崩壊の理論解析**: stop-gradient 構成が暗黙に「予測器を最適化する内側ループ」として振る舞うとする解釈(SimSiam の EM 的解釈)や、EMA が損失地形を平滑化して鞍点を避けるとする分析が進んでいます。なぜ非対称性が効くのかの完全な理論はまだ発展途上です。
