# マップフリー運転

## ひとことで言うと
事前に作り込んだ高精度地図(HD地図)に頼らず、「次の交差点を右」といった粗いナビ命令と、カメラなどのセンサだけで運転しようという考え方です。地図を一切使わないわけではなく、カーナビ並みの粗い地図(SD地図)を最小限のヒントとして使う設計と密接に関係します。人間のドライバーが初めての街でもナビと目だけで運転できるのと同じ発想です。

## 直感的な理解
初めて訪れた街で、あなたはカーナビの「次を右」という案内と、自分の目だけで運転できます。車線の正確な位置や曲がり角の形を、事前に頭の中の精密地図から引き出しているわけではありません。目の前の景色を見て、その場で車線を読み取り、ナビの「右」という意図に従ってハンドルを切っています。

マップフリー運転は、このやり方を機械に真似させようという発想です。なぜそうしたいのかというと、HD地図を前提にした自動運転には構造的な弱点があるからです。地図のある場所でしか走れず、地図の維持コストが莫大で、地図上に自車を正確に合わせる高精度ローカリゼーション([HD地図ローカリゼーション事前情報](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0440_hd-map-localization-prior))が破綻すると一気に危険になります。地図への依存を外せれば、対応できる地域が一気に世界中へ広がります。これを「地図を軽くする = map-light」「地図に頼らない = map-free」と呼びます。

## 基礎: 前提となる概念

- HD地図 / SD地図: HD地図は車線単位の数センチ精度の高精細地図。SD地図(Standard-Definition、OpenStreetMap など)は道路レベルの粗い地図で世界中に既に整備済み。マップフリーは前者を避け、後者やナビ命令を最小限使います。
- ナビ命令(navigation command): これは「直進/右折/左折」の離散コマンド、目的地までの粗い経路、あるいは SD地図上に描いた進むべきルートなど、『だいたいこっち』という高レベルの行き先指示という意味です。車線のどこを走れという精密指示ではありません。
- E2E(end-to-end)運転: センサ入力から運転操作(ステア・アクセル)や軌跡まで、途中をモジュールに分けず1つのモデルで学習する方式。
- 模倣学習(imitation learning)/ 行動クローニング(behavior cloning): 人間の運転ログ(入力と、そのとき人がどう操作したか)を教師に、入力から操作を当てるよう学習する方法。「お手本をまねる」学習です。
- 条件付け(conditioning): モデルの出力を、追加の入力(ここではナビ命令)で切り替えること。同じ交差点の映像でも「右」と言われれば右、「左」と言われれば左を出すよう、命令で振る舞いを変えます。

## 仕組みを詳しく

マップフリー運転の典型的な入力と処理は次の通りです。

1. ナビ命令を受け取る。形は様々で、離散コマンド(直進/右折/左折)、粗い経路、SD地図上のルート描画などです。重要なのは、これが「車線のどこを走れ」ではなく「だいたいこっち」というレベルだという点です。

2. センサで周囲を観測する。主にカメラ、必要なら LiDAR。

3. モデルがナビ命令とセンサ観測を統合し、走り方(軌跡やステア・アクセル)を出力する。車線の正確な位置や曲がり角の形は、地図ではなくセンサから読み取ります。

### ナビ命令の使われ方(交差点の例)

```
HD地図方式:
  地図 → 「この交差点は右折レーンがここ、右折後の車線中心はここ」
  → モデルは地図の指示に沿って曲がる

マップフリー方式:
  ナビ命令 = 「右」（意図だけ）
  カメラ → 交差点の形・空いている経路をその場で読み取る
  → モデルが自力で右へ曲がる経路を生成
```

地図はあくまで「右に行く」という意図だけを与え、操作の中身はセンサ任せにします。これにより、地図に書かれていない新しい交差点でも、カメラさえ見えれば走れます。

### 条件付けの実装

最も基本的な実装は、ナビ命令でネットワークの「枝(branch)」を切り替える方式です。命令が K 種類(例: 直進・左・右・車線維持)なら、出力ヘッドを K 本用意し、与えられた命令に対応するヘッドだけを使います。これにより「同じ映像 + 違う命令 → 違う運転」を実現します。なぜ枝分けが要るかというと、単一のヘッドで学習すると、交差点で「右に行った人」と「左に行った人」のデータが平均化され、出力が直進(両者の中間)に潰れてしまうからです(モード崩壊)。命令で枝を分ければ、各枝が一方向だけを学べます。より柔軟には、命令を埋め込みベクトル(embedding)にして特徴に足し込む、あるいは経路を画像チャンネルとして与える方式もあります。

### 「フリー」の程度

完全にどんな地図も使わない極端な設計もありますが、実用上は「HD地図は使わないが、SD地図やナビ経路という粗い情報は使う」という中間がほとんどです。SD地図は世界中で既に整備済みで、車線単位の精度がなくても「道路のつながり」「次に曲がる場所」を与えられます。これを使えば、目的地に向かう意図をモデルに伝えつつ、HD地図の重さを避けられます。地図を「精密な世界モデル」ではなく「経路ガイド」として最小限に使うのがマップフリーの実像です。

## 手法の系譜と主要論文

CIL / Conditional Imitation Learning (Codevilla, Müller, López, Koltun, Dosovitskiy; ICRA 2018, "End-to-end Driving via Conditional Imitation Learning", arXiv:1710.02410) が原型です。「直進/右折/左折」という高レベルコマンドでモデルの出力ヘッドを切り替える方式を提案しました。動機は、交差点でどちらへ行くかという曖昧な分岐を、HD地図なしでナビ意図だけで解決すること。新規性は、単純な模倣学習だと交差点でモード崩壊して直進してしまう問題を、命令で条件付けることで解いた点です。これがまさにマップフリーの源流です。トレードオフは、コマンドが離散的で表現が粗く、複雑な分岐を表しきれないことでした。

CILRS (Codevilla et al., ICCV 2019, "Exploring the Limitations of Behavior Cloning for Autonomous Driving", arXiv:1904.08980) は CIL を強化し、ResNet バックボーンや速度予測の補助タスクを加えつつ、行動クローニングの限界を実験的に明らかにしました。具体的には、分布シフト(訓練と違う状況で崩れる)と、因果の取り違え(過去の速度から未来の速度を当てるだけで楽をしてしまう inertia problem=慣性問題)を定量的に示し、「人のまねだけで学ぶ」ことの根本的な難しさを突きつけました。

TCP (Wu et al., NeurIPS 2022, "Trajectory-guided Control Prediction for End-to-end Autonomous Driving") は、軌跡予測と直接の制御予測を組み合わせ、ナビ命令条件付けの枠組みで CARLA の閉ループ性能を大きく伸ばした代表例です。単一カメラ入力でも高いスコアを出し、マップフリー系の到達点の一つになりました。

UniAD (Hu, Yang, Chen, et al.; Shanghai AI Lab, CVPR 2023 best paper, "Planning-oriented Autonomous Driving", arXiv:2212.10156) は、知覚・予測・計画を1つの Transformer ネットワークに統合し、最終目標である「計画(どう走るか)」に向けて全体を学習しました。地図構築もモデル内部でオンラインに行い、外部 HD地図に強く依存しません。動機は、モジュールを分けると誤差が積み重なり、HD地図前提だとスケールしないこと。効果は計画性能の大幅改善。トレードオフは、巨大で学習・推論が重く、内部のオンライン地図推定の精度に全体が左右される点です。

VAD (Jiang, Chen, Liang, et al.; Horizon Robotics, ICCV 2023, "VAD: Vectorized Scene Representation for Efficient Autonomous Driving", arXiv:2303.12077) は、シーン(地図要素や他車)をベクトル表現で軽く扱い、HD地図ラスタへの依存を減らしつつ高速な end-to-end 運転を実現しました。動機は重い表現を避けて効率を上げること。効果は精度と速度の両立(UniAD より数倍速い推論を報告)。トレードオフは、ベクトル化の品質がそのまま運転に効く点です。

混同に注意すべき別系統として、Map-free Visual Relocalization (Arnold et al., ECCV 2022) は事前地図なしの再自己位置推定を扱いますが、これは文脈が異なります(ローカリゼーションの話)。このノートの「マップフリー運転」は、運転方策が HD地図に依存しない、という意味です。

## 論文の実験結果(定量データ)

評価は大きく2系統あります。

シミュレータ(CARLA)での閉ループ評価: 実際にエージェントを走らせ、ルート完走率(route completion、与えられた経路をどれだけ進めたか %)や運転スコア(driving score、完走率に違反・衝突のペナルティを掛けた総合指標、高いほど良い)、km あたりの違反数で測ります。閉ループとは「モデルの出力が次の入力に影響する(実際に走らせる)」評価で、1枚ずつ精度を測る開ループより本物に近い評価です。CIL 系はこの CARLA ベンチマークで、ナビ命令による条件付けが交差点の曖昧さを解消し、無条件の行動クローニングより完走率を大きく上げることを示しました。一方 CILRS は、訓練と違う街・天候に変えると性能が落ちる(汎化の限界)ことを定量的に示しました。TCP は CARLA の標準ベンチマークで、当時の単一センサ手法として最高水準の driving score を報告し、ナビ命令条件付け + 軌跡/制御融合の有効性を裏づけました。

nuScenes での開ループ計画評価: 予測軌跡と実際の人の軌跡の L2 誤差(平均距離、メートル、小さいほど良い)と、衝突率(collision rate、予測軌跡が他物体とぶつかる割合 %、小さいほど良い)で測ります。UniAD は、1秒・2秒・3秒先の平均 L2 誤差をおよそ 1.0m 前後(従来のモジュール型より改善)、平均衝突率を 0.3% 程度まで下げたと報告しました。VAD は同等以上の計画精度を、推論速度を数倍速めつつ達成し、「ベクトル表現で軽くしても精度は落ちない」ことを示しました。

アブレーションの示唆として、UniAD は「途中タスク(物体追跡・地図・占有・動き予測)を外すと計画性能が落ちる」ことを示し、各補助タスクが計画に寄与することを裏づけました。CIL 系では「ナビ命令の条件付けを外すと交差点で破綻する」ことが効果の核心として示されています。なお nuScenes の開ループ計画評価には重要な批判があり、後続研究(AD-MLP / BEV-Planner など)は「自車の過去軌跡だけを単純な多層パーセプトロンに入れるだけで、地図やセンサをほとんど見なくても同等の L2 誤差が出てしまう」ことを示しました。つまりこの指標はマップフリー手法の真価を必ずしも測れておらず、閉ループ評価の重要性が近年強く強調されています。これはマップフリー手法の評価における未解決の問題です。

## メリット・トレードオフ・限界

- メリット: HD地図の測量・作図・維持という巨大コストが不要。世界中の未測量の道で動ける可能性があり、スケールする。
- メリット: 高精度ローカリゼーションへの依存が薄く、システムが軽い。GPS が多少粗くてもナビ意図は伝わる。
- メリット: 工事や新設道路など、地図が追いつかない変化にセンサで対応できる。
- トレードオフ: 見通しの悪い先・遮蔽の奥は地図に書いてないので、センサの届く範囲しか分からない。地図が補ってくれる「先読み」が弱くなる。
- トレードオフ: 車線レベルの精密誘導や交通規則(右折禁止など)を地図から保証できず、モデルがセンサから誤読すると危険。安全性の形式的保証が難しい。
- トレードオフ: ナビ命令の表現(離散コマンドか経路か)の設計しだいで、複雑な分岐や多車線で意図が曖昧になりうる。離散コマンドは「3つ目の出口へ」のような複雑な指示を表せない。
- 限界・未解決課題: 模倣学習ベースゆえの分布シフト(訓練にない状況で崩れる)と因果の取り違え(過去軌跡に過度に依存する慣性問題)。閉ループでの安全性をどう保証・検証するか。地図を完全に外すべきか、SD地図を弱い事前情報として残すべきか、という最適バランスも定まっていません。

## 発展トピック・研究の最前線

近年は、SD地図(OpenStreetMap)を弱い事前情報として与え、オンライン知覚を補助する「SD-map-aided」な設計が、マップフリーと HD地図依存の中間として注目されています([オンライン vs オフライン地図](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0456_online-vs-offline-mapping))。また、大規模言語モデルや視覚言語モデル(VLM)をナビ命令の解釈や運転推論・説明生成に使う研究(DriveGPT4 や LMDrive など)が急速に増え、自然言語の指示「次の角を曲がって」から直接運転につなぐ方向が探られています。模倣学習の限界を超えるための強化学習・閉ループ学習(オンライン環境で試行錯誤して方策を改善する)、運転の説明可能性(なぜその操作をしたか)を高める研究も進んでいます。「地図にどこまで頼らず、センサと粗い意図だけでどこまで安全に走れるか」は、自動運転をスケールさせるうえで最も実務的かつ未解決の問いの一つです。
