# Chamfer 距離損失

## ひとことで言うと
2つの「点の集まり(点群)」がどれだけ形として一致しているかを測る損失です。各点について「相手側の一番近い点までの距離」を足し合わせます。点の順番や個数が違っても比べられるのが最大の特徴で、3D 形状生成・LiDAR 点群・車線や地図のベクトル要素・占有予測などの幾何整合に広く使われます。

## 直感的な理解
画像は格子状に並んだピクセルの集まりなので、「左上から順番に i 番目のピクセル」という対応が自明です。だから2枚の画像はピクセルごとに差を取って比べられます(MSE など)。

ところが点群(point cloud)はそうはいきません。点群とは 3 次元(または 2 次元)空間に散らばった点の集まりで、LiDAR(レーザーで距離を測るセンサ)は周囲の物体表面を 1 スキャンで数万〜数十万個の点として記録します。車線も「中心線上に並んだ点の列」として表せます。この点群には画像にない 2 つの厄介な性質があります。

1. 順番がない(順序不変、permutation invariance): 点 P1, P2, P3 と P3, P1, P2 は同じ形を表す。どの点が「1番目」かは決まっていない。
2. 個数が違うことがある: 予測が 1000 点、正解が 1024 点ということが普通に起こる。

この2つのせいで、「1番目の点同士、2番目の点同士…」とペアにして差を取る対応づけ前提の損失(MSE など)が使えません。点 i と点 i がそもそも対応していないからです。

そこで「点の対応を固定せず、形として近いかを測る」損失が必要になります。それが Chamfer 距離(Chamfer distance、シャンファー距離)です。「各点について、相手側で一番近い点を自動で探して、そこまでの距離を測る」という発想で、対応づけ問題を回避します。名前は画像処理で輪郭からの距離変換を計算する「chamfer matching」に由来します。

## 基礎: 前提となる概念
- 点群(point cloud): 空間に散らばった点の集合。各点は座標 (x, y) または (x, y, z) を持つ。物体表面や軌跡を離散点で表現する。
- 順序不変(permutation invariance): 点を並べ替えても同じものを表す性質。点群を扱う損失や層はこの性質を尊重しなければならない。
- 最近傍(nearest neighbor): ある点から最も近い別の点。Chamfer はこの最近傍までの距離を基本単位にする。
- ユークリッド距離(Euclidean distance): 2 点間の直線距離。座標差の二乗和の平方根。Chamfer ではしばしばその二乗(ℓ2 版)、または平方根を取らない代わりに距離そのもの(ℓ1 版)を使う。
- 適合率と再現率(precision / recall): 予測のうち正しい割合と、正解のうち拾えた割合。Chamfer の双方向性はこの2つに対応する(後述)。
- モード崩壊(mode collapse): 生成や予測が多様性を失い、一点や一様な解に潰れてしまう失敗。Chamfer を片方向だけで使うと起きやすい。
- 微分可能(differentiable): 勾配が計算できること。最近傍探索は「どの点が最近傍か」を選ぶ離散操作だが、選ばれた後の距離は座標について微分可能なので、勾配降下に組み込める(選択が切り替わる境界では微分不可だが実用上は問題にならない)。

## 仕組みを詳しく
2つの点群 S1(予測)と S2(正解)があるとき、Chamfer 距離は次のように定義されます。

```
CD(S1, S2) =  Σ_{x ∈ S1} min_{y ∈ S2} ||x - y||²      ← S1 の各点から S2 の最近傍までの距離
            + Σ_{y ∈ S2} min_{x ∈ S1} ||y - x||²      ← S2 の各点から S1 の最近傍までの距離
```

`min_{y ∈ S2} ||x - y||²` は「点 x から見て、S2 の中で一番近い点までの距離(の二乗)」です。これを S1 の全点について足します(第1項)。そして逆向きにも同じことをします(第2項)。

なぜ両方向(双方向)足すのか。片方向だけだと「ズル」ができるからです。例えば予測 S1 の全部の点を、正解 S2 のたった 1 点の真上に重ねると、第 1 項(S1→S2)は 0 になります。でも S2 の他の点はカバーされていません。第 2 項(S2→S1)を足すことで「正解側の全ての点も、予測側の誰かが近くにいること」を強制でき、このズル(モード崩壊)を防げます。第 1 項は「予測の各点が正解の表面に乗っているか(precision に相当)」、第 2 項は「正解の各点が予測でカバーされているか(recall に相当)」を測る、と読むと役割が明確です。

ステップで追います。予測 `pred` 形状 `[B, N, 3]`(B バッチ, N 点, 3=xyz)、正解 `gt` 形状 `[B, M, 3]`(M 点)とします。

1. 全ペアの距離行列を作る。`pred` の各点と `gt` の各点の距離 → 形状 `[B, N, M]`。
2. 各 pred 点について、M 個の gt 点の中の最小距離を取る → `[B, N]`(S1→S2 方向)。
3. 各 gt 点について、N 個の pred 点の中の最小距離を取る → `[B, M]`(S2→S1 方向)。
4. それぞれ合計(または平均)して足す → スカラーの損失。

```python
import torch

def chamfer_distance(pred, gt):
    # pred: [B, N, 3], gt: [B, M, 3]
    dist = torch.cdist(pred, gt, p=2).pow(2)      # 全ペア2乗距離 [B, N, M]
    cd_pred = dist.min(dim=2).values.mean(dim=1)  # 各pred点→最近傍gt の平均 [B]
    cd_gt   = dist.min(dim=1).values.mean(dim=1)  # 各gt点→最近傍pred の平均 [B]
    return (cd_pred + cd_gt).mean()
```

数値例(2 次元、点 1 個ずつの極小例)。`pred=0,0`、`gt=3,4` とすると、最近傍距離の二乗 = 3²+4² = 25。双方向とも同じ点しかないので CD = 25 + 25 = 50(各方向の平均なら 25)。ぴったり重なれば 0 になります。

少し点を増やした例。`pred=[[0,0],[1,0]]`、`gt=[[0,0],[2,0]]`。S1→S2 方向: 点(0,0)の最近傍は(0,0)で距離0、点(1,0)の最近傍は(0,0)か(2,0)で距離1。和=1。S2→S1 方向: 点(0,0)→距離0、点(2,0)→最近傍(1,0)で距離1。和=1。CD=2。ここで分かるのは、Chamfer は「自分に最も都合の良い相手」を毎回選ぶため、一対一の対応を保証しない点です(点(0,0)に2つの最近傍が集まっている)。これが密度の偏りに鈍い原因です。

計算量の注意。全ペア距離 `[B, N, M]` は点が増えると爆発します(N=M=10000 なら 1 サンプルで 1 億要素)。実装では k-d tree や近似最近傍(approximate nearest neighbor)で高速化したり、点をダウンサンプルしたりします。CUDA カーネルで最近傍だけを直接計算し、距離行列を陽に作らない実装も標準的です(PyTorch3D の `chamfer_distance` など)。

似た損失に Earth Mover's Distance(EMD、地球を動かす距離、最適輸送距離)があります。EMD は「点を 1 対 1 で対応づけて移動量の総和を最小化」する厳密な指標で、Chamfer より分布の一致をきれいに測れますが、線形計画的な対応づけ(または Sinkhorn 近似)が要るため計算がずっと重く、点数が同じである必要もあります。Chamfer は近似的だが軽い、というトレードオフで広く使われます。

## 手法の系譜と主要論文
Chamfer 距離を深層学習の点群損失として確立したのが Point Set Generation Network(Fan, Su & Guibas, CVPR 2017, arXiv:1612.00603)です。1 枚の RGB 画像から物体の 3D 点群を生成するネットワークを提案し、その学習に Chamfer 距離と EMD を損失として用いました。点群は順序不変で個数も可変なため通常の損失が使えない、という問題に正面から取り組み、Chamfer と EMD が点群生成の適切な微分可能損失であることを示しました。これにより「画像から 3D 形状を点群表現で直接学習する」道が開けました。

続く点群オートエンコーダ・生成研究でも Chamfer は標準の再構成損失になりました。AtlasNet(Groueix et al., CVPR 2018, arXiv:1802.05384)は複数のパラメトリックパッチ(2D 平面を 3D 表面に写す写像)で表面を表し、その学習に Chamfer を用いました。FoldingNet(Yang et al., CVPR 2018, arXiv:1712.07262)は 2D グリッドを 3D 形状に「折り畳む(folding)」デコーダを提案し、やはり Chamfer を再構成損失にしました。理由は微分可能で軽く、点群に直接適用できるためです。

その後 Chamfer の弱点を補う変種が登場します。Density-aware Chamfer Distance(DCD, Wu et al., NeurIPS 2021, arXiv:2111.12702)は、最近傍へのマッチが何点に集中しているか(密度)を考慮して重み付けし、Chamfer が見逃す密度の偏りを罰するようにしました。これにより EMD ほど重くなく、しかし Chamfer より分布の均一性をよく測れる中間的な指標を実現しています。

自動運転文脈では、車線検出(予測した車線点列と正解点列の整合)、HD マップのベクトル要素予測、占有/フリースペースの点群予測などで Chamfer が使われます。VectorMapNet(Liu et al., ICML 2023, arXiv:2206.08920)や MapTR(Liao et al., ICLR 2023, arXiv:2208.14437)系の地図学習では、ベクトル化された地図要素(車線・横断歩道などを点列で表す)の予測とマッチングに Chamfer 系の距離が登場します。MapTR は順序不変の課題に対し「点列を等価な並べ替えの集合として扱う(permutation-equivalent modeling)」工夫を入れていますが、要素間のマッチングや評価には依然 Chamfer 距離が中心的役割を果たします。狙いは一貫して「点の順序や個数に縛られず、幾何的な形の一致を測る」ことです。

## 論文の実験結果(定量データ)
Point Set Generation 論文では ShapeNet(大規模 3D 物体データセット)で「単一画像からの 3D 復元」を評価しました。評価指標は Chamfer 距離と EMD そのもの(復元点群と正解点群の距離、低いほど良い)です。Chamfer/EMD を損失にした提案手法は、当時のボクセル(3D の格子)ベースの復元手法より細部を保った点群を生成し、両距離指標で優位を示しました。論文の重要な観察は「Chamfer 損失と EMD 損失で生成される形が質的に異なる」ことです。Chamfer は局所的に近い点だけ見るため、形のメインボディは合うが点の密度分布が偏りやすく、表面の薄い部分や細部がぼやけがちです。一方 EMD は分布全体を合わせるため点が均一に広がりますが計算が重い、というトレードオフを定量的・定性的に示しました。

DCD 論文では、点群補完(point cloud completion)ベンチマークで、Chamfer 距離が「視覚的には潰れているのに数値は良い」誤評価を起こすケースを示し、DCD が人間の知覚により合致する評価を与えることを定量化しました。具体的には、Chamfer-ℓ2 で良いスコアを出すが点が偏った結果を、DCD は適切に低く評価することを示し、補完タスクで DCD を損失に使うと出力点群の均一性が改善すると報告しています。

地図学習(MapTR 系)では、Chamfer ベースのマッチング・評価を用いた手法が nuScenes 地図ベンチマークでベクトル要素の mAP(平均適合率、Chamfer 距離のしきい値ごとに計算)を大きく押し上げました。MapTR は従来手法に比べリアルタイムに近い推論速度を保ちつつ mAP を大幅に改善したと報告され、ベクトル要素の幾何整合における Chamfer 系指標の有効性が示されています。アブレーションの典型は「双方向の片側だけを使うと点が一箇所に潰れる(モード崩壊)」「Chamfer だけだと密度が偏るので EMD や DCD を足すと均一性が改善する」「ℓ2 版は外れ値に弱いので ℓ1 版にすると頑健になる」というものです。

## メリット・トレードオフ・限界
メリット
- 点の順序に依存しない(順序不変)ので、点群にそのまま使える。
- 予測と正解の点数が違っても比較できる。
- 微分可能で、勾配降下による学習に組み込める。
- EMD より計算が軽く(対応づけの最適化が不要)、実用的。

トレードオフ・限界
- 全ペア最近傍探索が必要で、点数が増えると計算・メモリが重い(k-d tree、近似最近傍、ダウンサンプルで緩和)。
- 密度の偏り(点が一部に集中)に鈍く、表面の細部が局所的に甘くなる(EMD ほど厳密でない)。これを補う Density-aware Chamfer Distance などが提案されている。
- 外れ値(1 点だけ大きく外れた予測)に二乗距離(ℓ2 版)が強く反応し損失が暴れることがある。ℓ1 版や外れ値に頑健な変種で緩和する。
- 「最近傍」を使うため最適化の地形が非凸で、局所解(モード崩壊)に陥ることがある。双方向にしても完全には防げない。
- 順序情報を捨てるため、ポリライン(順序付きの点列)で「点の並び順」が意味を持つ場合(車線の向きなど)はその情報が損なわれる。順序を考慮した変種や別損失との併用が要る。

## 発展トピック・研究の最前線
Chamfer の弱点(密度の偏り、外れ値感受性)を補う研究が活発です。Density-aware Chamfer Distance(DCD)は近傍密度で重み付けして均一性を改善します。最適輸送の観点から Sinkhorn 近似で EMD を軽量化し Chamfer の代替とする流れもあります。点群補完・生成では Chamfer と EMD を組み合わせたり、表面の法線(normal)整合を加えたりして細部を改善します。地図学習・占有予測では、点列を順序付きポリラインとして扱い順序を考慮した変種を使うか、順序フリーの Chamfer を使うかが設計判断になります。占有を密なグリッドで学ぶなら [Dice 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0379_dice-loss)、点列として学ぶなら Chamfer、という使い分けが定石です。最近は拡散モデル(diffusion model)による点群生成も増え、そこでも品質評価に Chamfer/EMD/DCD が並列して使われており、「単一の万能な点群距離は存在せず、複数指標で多面的に評価する」という認識が研究の前提になっています。
