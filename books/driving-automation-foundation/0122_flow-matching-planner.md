# Flow Matching トラジェクトリプランナ

## ひとことで言うと

自動運転車やロボットが「これから数秒間どう動くか」という軌跡(未来の運動ライン)を出力する部品のひとつです。従来は時刻ごとに1ステップずつ予測する再帰型のやり方が主流でしたが、Flow Matching プランナは「ランダムなノイズから出発し、滑らかな軌跡へ向かって少しずつ変形させていく」生成モデルのやり方で軌跡をまとめて作ります。ノイズから絵を描く拡散モデルの考え方を、軌跡という連続値の系列に応用したものです。

## 直感的な理解

軌跡を作る問題には本質的な難しさがあります。多くの状況で「正解は1つではない」のです。交差点で前に障害物があれば、左から避けても右から避けても正しい。直進しても緩やかに減速しても安全なことがある。このように複数の妥当な未来がある状況をマルチモーダル(multimodal、分布が複数の山を持つ)と呼びます。

従来の回帰型プランナは「入力から1本の軌跡を直接予測し、正解との誤差を最小化」します。ところが正解が左右2本あると、誤差を最小にするのはその平均、つまり「まっすぐ障害物に突っ込む軌跡」になってしまいます。これがモード平均化(mode averaging)の問題で、複数の正解を平均して無難だが危険な1本に潰してしまう現象です。

生成モデルはこの問題に強い枠組みです。1本を決め打ちで出すのではなく、妥当な軌跡の分布そのものを学習し、そこからサンプリングします。左に避ける軌跡と右に避ける軌跡が両方とも分布の山として保たれ、サンプルするたびにどちらか一方が出てきます。Flow Matching はこの分布をノイズから軌跡への連続的な変形として表現する、効率の良い生成モデルです。

## 基礎: 前提となる概念

軌跡(trajectory)の表現。1本の軌跡は時刻ごとの運動量の系列です。よくある表現は `(T, S)` の形で、T は時刻数(たとえば 64)、S は1時刻あたりの信号数です。S=2 なら加速度(acceleration)と曲率(curvature、進行方向の曲がり具合)の2成分がよく使われます。位置 xy で表す流儀もあります。これを平坦化して `[T*S]` 次元のベクトル1本として扱い、学習・推論ではバッチ次元を付けて `[B, T*S]` とします(B はまとめて処理する本数)。

生成モデル(generative model)とは、データの分布 p(x) を学習し、そこから新しいサンプルを生成できるモデルです。拡散モデル(diffusion)は、データにノイズを徐々に加えて純ノイズにする過程を逆向きにたどり、ノイズからデータを復元するよう学習します。Flow Matching はこの拡散の枠組みを連続正規化フロー(continuous normalizing flow)の視点から簡略化したものです。

速度場(velocity field)。Flow Matching の中心概念です。ノイズ分布からデータ分布へ確率質量を「流す」連続的な変形を考え、各地点・各時刻でその流れの速度をベクトルとして表したものが速度場 v(x, t) です。これをニューラルネット v_θ(x, t) で近似します。学習が済めば、ノイズから出発してこの速度に従って進めばデータに到達します。

クロスアテンション(cross-attention)。トランスフォーマの仕組みのひとつで、ある系列(クエリ)が別の系列(キー・バリュー)を参照し、関連する情報を重み付きで取り出します。軌跡生成では、各時刻の軌跡トークンが視覚特徴のどの場所を見るべきかを学習的に決めるのに使います。

条件付き生成(conditional generation)。無条件にデータを生成するのではなく、入力条件(運転ならカメラ・LiDAR から作った BEV 特徴、過去の自車運動、目的地など)を与えて、それに合った軌跡を生成することです。プランナは本質的に条件付き生成です。

## 仕組みを詳しく

Flow Matching の中心アイデアを段階的に組み立てます。

スタートとゴールを決めます。

- スタート x_0: 完全なランダムノイズ。標準正規分布からのサンプル `randn`。意味のない軌跡です。
- ゴール x_1: 正解の軌跡(教師データ)。

この2点を結ぶ経路を考えます。最も単純なのは直線(線形補間)で、途中時刻 t(0 から 1)での位置を次のように定義します。

```
u_t = (1 - t) * x_0 + t * x_1
```

t=0 で純ノイズ、t=1 で正解です。この直線上をノイズからゴールへ進む速度は、直線なので t に依存せず一定で、`x_1 - x_0` です。

ニューラルネット v_θ に「今 u_t という地点に、時刻 t でいる。どっちへどれだけ進めばいい?」と尋ね、その答えが `x_1 - x_0` に一致するよう学習します。学習目標は単純な二乗誤差で、詳細は [条件付き Flow Matching 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0117_conditional-flow-matching-loss) を参照してください。

学習後の推論は、意味のないノイズ x から出発し「今の地点と時刻を v_θ に渡す → 速度をもらう → その方向に少し進む」を数回繰り返して正解らしい軌跡へ到達させます。この常微分方程式(ODE)の数値積分の詳細は [Euler 法による ODE 推論（Flow Matching のサンプリング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0120_euler-ode-inference) にあります。

実用的なプランナでは、速度場ネットワーク v_θ を単なる全結合ではなく、シーンを見ながら計算する作りにします。代表的な流れは次のとおりです。

1. ノイズ軌跡 `u_t` `[B, T*S]` を `[B, T, S]` に並べ替え、各時刻を「アクショントークン」として線形射影で埋め込み次元 D へ投影 → `[B, T, D]`。これがクエリです。
2. BEV 特徴マップ `[B, D, H, W]` を平坦化して `[B, H*W, D]` にする。これがキー・バリュー(情報源)です。
3. クロスアテンションで、各時刻のアクショントークンが BEV のどの場所を見るべきかを学習的に拾う → `[B, T, D]`。これにより「どこに障害物があるか」という空間情報を潰さずに軌跡生成へ反映できます。
4. 時刻 t を正弦波埋め込み(sinusoidal embedding、スカラー t を高次元ベクトルへ変換)し、視覚履歴・自車運動履歴などの条件と足し合わせて、AdaLN(Adaptive Layer Normalization)で各層に注入します。これは画像生成の DiT(Peebles & Xie, ICCV 2023)で確立されたパターンの流用です。
5. velocity head で各時刻を S 次元へ戻し、`[B, T*S]` の速度を出力します。

ここで設計上重要なのは、BEV を1本の条件ベクトルへ圧縮せず、クロスアテンション経由で参照する点です。素朴な設計では BEV を平均プーリングなどで1本のベクトルに潰して条件にしますが、それでは「左前方に歩行者、右奥に停止車両」といった空間的な配置情報が失われます。アテンションを使えば、各時刻の軌跡トークンが必要な場所だけを選んで参照でき、空間の細かさを保てます。

tensor shape の流れを一例で示します(B=8, T=64, S=2, D=256, H=W=32 とする)。

```
u_t       : [8, 128]        ノイズ/中間軌跡(平坦)
           -> reshape -> [8, 64, 2]
queries   : [8, 64, 256]    アクショントークン
bev       : [8, 256, 32, 32]
           -> flatten -> [8, 1024, 256]  キー・バリュー
attn out  : [8, 64, 256]    各時刻が BEV を参照
+ t emb, history (AdaLN)
velocity  : [8, 64, 2] -> [8, 128]
```

## 手法の系譜と主要論文

Flow Matching を軌跡生成へ応用するに至る流れには明確な系譜があります。

Sohl-Dickstein et al.(2015)と Ho et al., DDPM, NeurIPS 2020 が拡散モデルの基礎を築きました。ノイズ除去を繰り返してデータを生成する枠組みで、高品質ですが多数(数百〜千)の推論ステップを要しました。

Chi et al., Diffusion Policy, RSS 2023(arXiv:2303.04137)。拡散モデルをロボットの行動生成(visuomotor policy)に応用した代表例です。観測を条件に、行動列を分布として生成することで、マルチモーダルな実演から無難な平均に潰れずに学習できることを示しました。軌跡・行動を生成モデルで扱う潮流の起点です。

Lipman et al., Flow Matching for Generative Modeling, ICLR 2023(arXiv:2210.02747)。Flow Matching の基礎論文です。拡散が確率微分方程式やスコア関数の理論を要したのに対し、「ノイズとデータを結ぶ経路上の速度場を回帰するだけ」というシンプルでスケールする学習法を提案しました。とりわけ直線経路(linear/OT path)を使う条件付き Flow Matching は、目標速度が定数 `x_1 - x_0` になり実装が極めて簡単です。

Liu et al., Rectified Flow, ICLR 2023(arXiv:2209.03003)。独立に直線経路の定式化を提案し、経路を「まっすぐ」にするほど推論時の積分ステップを減らせる(極端には1ステップに近づけられる)ことを示しました。少ステップ推論の理論的裏付けです。

Black et al., π0, Physical Intelligence, 2024(arXiv:2410.24164)および後継の π0.5。視覚言語モデルに Flow Matching の行動生成ヘッドを接続したロボット基盤方策です。連続的な行動列を高頻度で生成でき、時刻 t を一様ではなくノイズ側へ偏らせるサンプリングが学習を安定させると報告しました。

運転分野では、連続軌跡を離散トークンを経由せずに生成し、低い推論レイテンシ(数ミリ秒)で動かす生成プランナが提案されており、加速度や曲率といった物理量の連続性をそのまま保てることが利点として主張されています。これらは上記の Diffusion Policy / Flow Matching の系譜を自動運転の軌跡計画へ持ち込んだものと位置づけられます。

## 論文の実験結果(定量データ)

Diffusion Policy(RSS 2023)は複数のロボット操作ベンチマークで評価され、従来の決定論的な行動クローニング(behavior cloning)やエネルギーベースの方策に対し、タスク成功率(success rate、エピソードのうち課題を達成できた割合。高いほど良い)を平均でおよそ二桁パーセント単位で改善したと報告されています。この改善の核心はマルチモーダル性の保持で、複数の正解動作がある場面で平均に潰れず、一貫した1つのモードを選べる点にあります。アブレーションでは、行動を1ステップ回帰で出す構成にすると成功率が大きく落ち、生成モデル化の寄与が確認されています。

Lipman らの Flow Matching(ICLR 2023)は画像生成(ImageNet など)で、同等の生成品質を拡散モデルより少ない推論ステップで達成できることを FID(Fréchet Inception Distance、生成分布と実分布の距離。低いほど良い)で示しました。直線経路の条件付き Flow Matching が、学習の安定性とサンプリング効率の両面で優れることが報告されています。

Rectified Flow(ICLR 2023)は、経路を直線化する reflow 手続きにより、わずか1〜数ステップの推論でも高品質な生成が可能になることを示しました。ステップ数を 1 まで減らしても FID の劣化が小さいことが定量的に報告されており、これがレイテンシ制約の厳しい運転・ロボットでの実用性を支えます。

π0(2024)は実機ロボットの多様なタスクで、Flow Matching による高頻度行動生成が滑らかで成功率の高い動作を生むことを報告しました。t をノイズ側へ偏らせる(たとえば Beta 分布で低 t を多めにサンプリングする)ことが学習安定に寄与するというアブレーションが示されています。数値の意味としては、ノイズに近い領域(t が小さい)の速度場をより正確に学ぶことが、最終的な生成品質を左右する、ということです。

## メリット・トレードオフ・限界

メリット。T 時刻分を一括生成でき、自己回帰のような時刻ごとの逐次ループが不要です。推論ステップ数を変えるだけで速度と品質を連続的に調整できます。加速度・曲率などの連続性を保ちやすく滑らかな軌跡になりやすい。BEV をアテンションで直接参照すれば空間情報を保てます。そして最大の利点として、マルチモーダルな未来を平均に潰さず分布として扱えます。

トレードオフと限界。学習が t サンプリングや条件付けの設計に敏感で、素朴な一様サンプリングだと性能が出にくいことがあります。サンプリングするたびに乱数で出力が揺れるため、再現にはシード固定が要り、安全性評価では「出力の分散」をどう保証するかが課題になります。連続生成でもサンプル数が少なければ実質的にマルチモーダル性を活かせず、結局1本に頼ると無難な軌跡へ寄る危険が残ります。推論ステップを増やすほど正確だが遅くなる速度と精度のトレードオフがあります。未解決の課題として、生成した複数候補のうち「どれを実際に実行するか」を安全に選ぶ仕組み(モード選択、不確実性の定量化)はまだ研究途上です。

## 発展トピック・研究の最前線

研究の最前線では、Flow Matching/拡散による軌跡生成をさらに高速化・高品質化する方向が活発です。蒸留(distillation)によって多ステップ生成を1〜数ステップへ圧縮する手法、経路を直線化して少ステップ化する Rectified Flow 系、そして分類器なしガイダンス(classifier-free guidance)で条件への忠実度とサンプル多様性のバランスを制御する手法が運転・ロボットへ持ち込まれています。

また、視覚言語モデル(VLM)に Flow Matching の行動ヘッドを接続する vision-language-action モデルが急速に発展しており、言語指示や経路目標を条件に軌跡・行動を生成する流れが主流になりつつあります。安全側の研究としては、生成軌跡の制約充足(運動学的に実行可能か、衝突しないか)を生成過程に組み込むガイド付きサンプリングや、複数モードを明示的に列挙して評価する手法が進んでいます。
