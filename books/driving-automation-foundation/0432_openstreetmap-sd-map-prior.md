# OpenStreetMap SD地図事前情報

## ひとことで言うと
OpenStreetMap(OSM、誰でも編集できる無料の世界地図)が持つ粗い道路情報(道路網の形状や「これは高速道路」「ここは横断歩道」といったタグ)を、HD 地図構築や軌道予測の「弱いヒント(事前情報=prior)」としてモデルに与える考え方です。prior とは「センサで観測する前から分かっている事前知識」のことです。OSM の SD 地図(Standard Definition map=標準精度地図)は精密ではないけれど、世界中どこでもほぼ無料で手に入る広域カバーが最大の利点です。これを使うと、カメラに今映っていない先の道路構造を補える、というのが核心です。

## 直感的な理解
人間がはじめての街を運転するとき、目で見える範囲だけでなく、頭の中の「だいたいこの先で道が分かれるはず」という大雑把な土地勘を使っています。カーナビの粗い地図を一度見ておくだけで、見通しの悪いカーブの先や、霧で見えない交差点でも「ここは十字路だ」と心の準備ができます。

オンライン HD 地図構築(カメラだけからその場で車線地図を起こす手法)には弱点があります。カーブの先、交差点の全体構造、霧や夜で見えにくい遠方など、センサに今映っていない部分の推定が苦しいのです。そこで「完璧でなくていいから、道路がだいたいどう走っているか」という粗いヒントを外から足せば、精度と安定性が上がるはずだ、というのが SD 地図事前情報(SD-map prior)の発想です。

たとえば交差点に近づくとき、カメラだけだと「十字路か T 字路か」が直前まで分かりません。OSM を見れば「ここは十字路で右に道がある」と粗く但し事前に分かります。これを入力に足しておくと、地図構築や軌道予測が安定します。重要なのは、SD 地図は「正確な車線形状」は教えてくれないが「道路の接続(トポロジー)」は教えてくれる、という非対称性です。接続情報こそ、カメラが最も苦手とする(遠方・遮蔽で見えない)情報なので、両者は補完的です。

## 基礎: 前提となる概念
- HD 地図と SD 地図の違い。HD 地図は車線レベルまで数十 cm 精度。作るのが高価で範囲が限られ、古くなります。SD 地図は道路を中心線 1 本くらいでしか持たず、車線数や正確な幅を持たないことが多い代わりに、世界中をほぼ全域カバーし、OSM なら無料です。詳しくは [ナビ地図 vs HD地図](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0422_nav-map-vs-hd-map)。
- OSM のデータモデル。OSM は地図を「ノード(点)・ウェイ(点をつないだ線、道路はこれ)・リレーション(関係)・タグ(key=value の属性)」で持ちます。`highway=residential`(住宅街の道)、`highway=motorway`(高速)、`crossing`(横断歩道)などのタグが意味を表します。タグの種類は数千に及び、その大半は希少です。データ取得には Overpass API(条件を指定して OSM データを切り出す問い合わせ API)を使うのが一般的です。
- BEV(Bird's-Eye-View=鳥瞰図)。地図やセンサ情報を真上から見た平面のグリッドで表す表現です。BEV のマス目(セル)それぞれが地面の一区画に対応します。オンライン地図構築や運転モデルの多くは、この BEV 上で情報を統合します。
- cross-attention(クロスアテンション)。Transformer の機構の一つで、ある側の各要素(たとえば BEV の各マス)が、別の側の情報(たとえば OSM のトークン)を「どれを・どれだけ参照するか」を学習で決めて取り込む仕組みです。「BEV の各マスが、関係する OSM 情報を選んで吸収する」と理解すれば十分です。
- トークン。Transformer が処理する最小単位のベクトルです。OSM の道を細切れにして点ごとのベクトルにすると、それが「点トークン」になります。
- prior の強さ(weak/strong prior)。prior をどれだけ信じるかの度合いです。SD 地図は不正確なので「弱い prior(weak prior)」として扱い、最終判断は観測(センサ)に委ねるのが定石です。盲信すると古い地図の誤りを引き継ぎます。

## 仕組みを詳しく
SD-prior をモデルに入れる典型的な流れは次のとおりです。

1. 自車のおおよその位置(GPS など)を中心に、OSM から周辺の道路データを取得する。Overpass API で半径数百 m を切り出すのが一般的です。
2. 各道路(way)の中心線を等間隔の点列にリサンプリングし、自車中心の座標系(前方を X、左を Y など)に変換する。位置合わせには自車の姿勢推定が要り、ここの誤差がそのまま prior のズレになります。
3. 各点・各 way のタグを特徴ベクトルに変換する。素朴な方式は「タグを少数の固定カテゴリ(車道/歩道/横断歩道など)に割り当てて one-hot や学習埋め込みにする」やり方です。
4. これらを Transformer のトークン列にし、BEV 特徴へ cross-attention で融合する。BEV の各マスが、自分の近くにある OSM トークンを参照して情報を取り込みます。

タグの扱いには重要な設計分岐があります。固定カテゴリ方式だと、OSM の数千種類にも及ぶ多様なタグのうちごく一部しか活かせず、珍しいタグは「その他」に潰れてしまいます。これを改善する流れが「タグを自然言語として埋め込む」方式です。`highway=residential` という文字列を、事前学習した言語モデル(BERT などの NLP モデル)でベクトル化します。言葉の意味空間に置くので、学習時に見たことのない珍しいタグでも、意味の近い既知タグの近くに配置され、汎化が効くという狙いです。これは「未知のカテゴリは捨てる」のではなく「意味の近さで補間する」という発想の転換です。

実装上よく入る工夫を、形状イメージとともに挙げます。

- OSM エンコーダの出力は概ね `tokens [B, N, C]` と `key_padding_mask [B, N]` の形です。B はバッチ、N はトークン数、C は特徴次元です。padding mask は「どのトークンが本物で、どれが穴埋めか」を示します。
- 連続値の位置エンコーディング(sine continuous positional encoding)で各点の座標を表す。グリッドに丸めず連続座標をそのままエンコードできます。BEV のマス目に量子化しないので、SD 地図の(粗いとはいえ)連続的な道路形状を保てます。
- null トークン(学習可能な穴埋め)。その地点に OSM 情報が無いサンプルでも壊れないよう、空のときは学習可能なダミートークンを入れます。OSM が未整備な地域でも学習・推論がクラッシュしない安全策です。
- ゲートのゼロ初期化。融合の重みをゼロから始めると、学習開始時は「地図なし」と完全に同じ挙動になり、いきなり既存性能を壊さない安全策になります。学習が進むにつれて prior を取り込む度合いが上がります。これは ResNet の恒等写像初期化や、画像生成の ControlNet で使われる「ゼロ畳み込み」と同じ思想で、新しい入力経路を「無害な状態」から徐々に開いていく汎用テクニックです。

```
OSM way → 等間隔点列 → 点トークン [B,N,C] ┐
タグ "highway=residential" → 言語埋め込み ┘→ cross-attn → BEV特徴 [B,C,H,W]
                                  ↑ gate(ゼロ初期化で安全に導入)
```

なぜゲートや attention で「prior をどれだけ信じるか」を学習させるのか。それは prior が間違っていることがあるからです。OSM が古い、自車位置がずれている、といったとき、prior を盲信すると誤りを引き継ぎます。信じる度合いを学習可能にしておけば、信頼できる場面では強く使い、当てにならない場面では無視する、という挙動を獲得できます。研究者視点では、この「学習可能な信頼度」が、固定的なルールベース融合に対する機械学習アプローチの本質的な強みです。

## 手法の系譜と主要論文
- Neural Map Prior(Xiong et al., CVPR 2023, "Neural Map Prior for Autonomous Driving")。厳密には OSM ではなく「過去の走行で作った地図の特徴(ニューラル特徴のグローバルマップ)」を prior にする近縁手法ですが、「事前情報を持って地図構築を補強する」という枠組みの先駆けです。複数回その場所を通るたびに prior を更新していくオンライン更新の発想を導入しました。地図を「一度作って固定」ではなく「通るたびに育てる」という方向性を示した点が重要です。
- P-MapNet(Jiang et al., RA-L 2024, "P-MapNet: Far-seeing Map Generator Enhanced by both SDMap and HDMap Priors")。SD 地図(OSM)と、事前のラスタ HD 地図の両方を prior として使い、オンライン HD 地図構築を強化しました。動機は遠方や遮蔽部の予測を補強することです。OSM の道路骨格を CNN で特徴化し、Transformer の attention で BEV に注入する設計です。さらに HD prior 側は masked autoencoder(地図の一部を隠して復元する自己教師あり学習)で道路の「もっともらしい形」を学習し、出力を実際の地図らしく整えます。新規性は、SD と HD という粒度の異なる二つの prior を同時に活用し、特に遠方(60m 超)の地図品質を上げた点です。
- SMERF(Luo et al., ICRA 2024, "Augmenting Lane Perception and Topology Understanding with Standard Definition Navigation Maps")。SD ナビ地図を Transformer でエンコードし、車線検出と車線トポロジー(車線同士のつながり)推定を補強しました。SD 地図が「どの車線がどこへつながるか」の事前知識として効くことを示した点が重要です。トポロジー(接続関係)は、まさにカメラが最も苦手とする情報なので、SD 地図の長所と知覚の短所が噛み合う設計です。
- SDTagNet(Immel et al., 2025)。OSM のタグを自然言語埋め込みとして扱う方向を推し進めた手法です。動機は、既存の SD-prior 手法がタグを少数の固定カテゴリにしか落とせず、OSM の豊富で多様なタグ情報を活かしきれていなかったこと。新規性は、(1) タグを NLP モデルで言語埋め込みにして珍しいタグへも汎化させること、(2) way を固定点数の点トークン列にして点レベルの細かい構造を保つこと、です。

この系譜を一言でまとめると、「過去走行の prior(NMP)」→「SD と HD の二重 prior で遠方を伸ばす(P-MapNet)」→「SD でトポロジーを補強(SMERF)」→「タグを言語として豊かに使う(SDTagNet)」という、prior の情報量を増やしながら汎化を高める流れです。背後にある共通の問題意識は「カメラだけのオンライン地図構築は遠方・遮蔽・トポロジーで限界がある。安価で広域な SD 地図でそこを埋められないか」という一貫したものです。

## 論文の実験結果(定量データ)
評価の主戦場は nuScenes(自動運転の標準ベンチマークデータセット。米国ボストンとシンガポールの計 1000 シーンを含む)上での HD 地図構築や車線知覚です。代表的な指標は mAP(mean Average Precision=平均適合率。検出・地図要素の予測がどれだけ正確かを 0〜100% で測る、高いほど良い)です。地図構築では、予測した車線ベクトルと正解の距離(Chamfer 距離=2 つの点集合の互いに最近傍までの距離の平均)が一定しきい値以内なら正解、という判定で AP を出すのが一般的です。

- P-MapNet は、SD 地図 prior を加えることで、ベースラインのオンライン HD 地図構築に対して特に遠方領域の地図品質を大きく改善したと報告しています。論文では知覚範囲を広げた設定(前方 60m 級)で、prior 無しに比べ mAP がおよそ十数ポイント規模で向上する結果が示されています(設定により変動)。重要なのは、改善幅が「近く」より「遠く」で大きいことです。これは prior が「センサで見えにくい部分」を埋める、という直感とよく一致します。
- SMERF は、SD 地図を足すことで車線検出とトポロジー推定の両方が改善することを示しました。特に車線同士の接続(トポロジー)の指標で改善が顕著で、これは「SD 地図が道路の接続構造という事前情報を直接持っている」ことの効果です。トポロジー評価には、車線中心線同士の接続が正しいかを測る指標(OpenLane-V2 ベンチマークの TOP スコアなど)が使われ、SD prior でこれが押し上げられます。
- SDTagNet は、タグを言語埋め込みにすることで、固定カテゴリ方式の SD-prior に比べてさらに地図構築精度が上がること、特に多様・希少なタグが現れるシーンで汎化が効くことを報告しています。固定カテゴリでは捨てられていた情報を拾える分、上積みが出るという主張です。
- アブレーション(どの要素を抜くとどうなるか)の典型的な知見として、(1) SD prior 自体を抜くと遠方の mAP が落ちる、(2) タグ情報を抜いて幾何(道路の形)だけにすると改善幅が縮む、(3) 位置合わせ誤差を意図的に増やすと改善幅が小さくなる、が共通して観察されます。とくに (3) は実運用で重要で、GPS 誤差が大きい環境では prior の恩恵が目減りすることを意味します。

指標の意味を補足すると、mAP が十数ポイント上がるというのは「これまで遠方の車線を半分しか正しく描けなかったものが、かなりの割合まで描けるようになる」程度の差で、自動運転の安全余裕に直結する大きさです。一方で、これらの改善は「prior が正しく位置合わせされている」前提に強く依存します。論文の数値はクリーンな位置合わせ前提で出ていることが多く、実車の GPS 誤差(数 m に達することもある)下での頑健性は別途確認が要る、というのが研究者として注意すべき点です。

## メリット・トレードオフ・限界
- メリット。ほぼ無料で世界中をカバーします。HD 地図のような作成コストや範囲制限がありません。
- メリット。センサで見えない交差点構造や道路接続を、弱いヒントとして補えます。効果は遠方ほど大きい。
- メリット。タグの言語埋め込みを使えば、珍しいタグにも汎化しやすくなります。
- トレードオフ。精度が粗く、車線数や正確な幅は不明なことが多い。位置合わせ(GPS 誤差・姿勢推定誤差)でズレます。
- トレードオフ。OSM が古い・未整備の地域では情報が薄い、または間違っています。prior を信じすぎると誤りを引き継ぎます。だからゲートや attention で信頼度を学習させる設計が定石です。
- トレードオフ。データ取得(Overpass)・キャッシュ・座標変換など、データパイプラインが複雑になります。オフライン学習時は事前ダウンロード、車載推論時はローカルキャッシュ、という運用設計が要ります。
- 限界。prior と現実のズレへの頑健性(robustness)は依然未解決の研究課題です。位置合わせ誤差や地図の陳腐化に対して、どれだけ自動で「prior を割り引く」べきかの最適な制御は確立していません。また、OSM の品質は地域差が極端に大きく(欧州の整備度と未整備地域の差)、ある地域で学習した信頼度の使い方が別地域でそのまま通用しない、という汎化の難しさもあります。

## 発展トピック・研究の最前線
最前線は「prior の頑健化」と「prior の言語化」の二方向です。頑健化では、位置合わせ誤差を陽に推定して補正する、あるいは prior と観測の食い違いを検出して prior を動的に割り引く機構が研究されています。学習時に意図的に prior をずらしたり一部を欠落させたりする data augmentation(データ拡張)で、誤った prior への耐性を鍛えるアプローチも一般的です。言語化では、SDTagNet のようにタグを自然言語として扱う流れがさらに進み、地図の意味を大規模言語モデルの知識空間で扱う試みが出てきています。また、SD prior は HD 地図構築だけでなく、軌道予測やプランニングへ直接効かせる方向(道路の接続を予測の制約に使う)にも広がっています。事前作成 HD 地図([Lanelet2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0445_lanelet2))との関係でいえば、SD prior は「HD 地図が無い・古い地域でも最低限の道路構造知識を供給する補完層」として位置づけられつつあります。三層構造(精密だが局所的な HD 地図 / 粗いが広域な SD prior / その場のセンサ観測)を信頼度つきで融合する、という見方が定着しつつあるのが現状です。
