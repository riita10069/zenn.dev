# Timestep Embedding（時刻埋め込み）

## ひとことで言うと

「いま生成プロセスのどの段階にいるか」を表すたった1つの数 t(たとえば 0.0〜1.0)を、ニューラルネットが扱いやすい高次元のベクトル(たとえば128次元)に変換する小さな部品です。さまざまな周波数の sin と cos の波を並べて作ります。Diffusion(拡散モデル)や Flow Matching(フローマッチング)で必ず登場する、いわば「時計の針を読めるようにする変換器」です。

## 直感的な理解

ラジオのダイヤルを思い浮かべてください。たった1本のつまみ(スカラーの t)の位置を、いろいろな周波数の波として表すと、つまみのわずかな違いも、大きな違いも、両方を同時に読み取れます。低い周波数の波はゆっくり変化するので「だいたいどのへんか」を、高い周波数の波は素早く変化するので「微妙な位置の差」を担当します。

なぜこんな変換が要るのか。Flow Matching や Diffusion では、ノイズ(でたらめな点)から本物のデータ(車の走行軌跡や画像)へ少しずつ変形します。その進み具合が時刻 t です(t=0 がノイズ、t=1 が完成)。ネットには「いま t がいくつか」を教える必要があります。t=0.05(ほぼノイズ)のときと t=0.95(ほぼ完成)のときでは、ネットがやるべき仕事がまったく違うからです。前者は大まかな方向決め、後者は細部の微調整です。1つのネットワークで全段階を共有するには、いま何段階目かを正確に伝える「時計の読み取り装置」が要る、というわけです。

## 基礎: 前提となる概念

第一に「スカラーとベクトル」。スカラーは1個の数(t=0.5 など)。ベクトルは数の並び(128個の数)。ネットの層は重み付き和(線形変換)なので、入力の表現が豊かなほど複雑な反応を学びやすくなります。

第二に「なぜ生の t を直接入れるとうまくいかないか」。理由は2つあります。(1) ネットの最初の層は線形変換なので、1次元の生の値 t だけだと t に対してなめらかな直線的反応しかしにくく、「t=0.10 と t=0.11 の僅差」を細かく区別する表現力が乏しい。(2) 生のスカラー1個から「時刻ごとの振る舞いの違い」を引き出すのはネットにとって負担が大きく、学習が進みにくい。Tancik ら(Fourier Features, NeurIPS 2020, arXiv:2006.10739)は、生の低次元座標を直接入れると高周波成分を学べない「スペクトルバイアス」(neural tangent kernel が低周波に偏る性質)が起き、sin/cos のフーリエ特徴に展開すると細かい構造を学べると理論・実験の両面で示しました。時刻埋め込みはこの一例です。

第三に「周波数」。sin(ωt) の ω が大きいほど波は速く振動します。高周波は t の細かい違いに敏感、低周波は大まかな位置に敏感です。両方を混ぜると、細かさと大局を同時に表せます。

第四に「位置符号化(positional encoding)」。Transformer が単語の順番を sin/cos で符号化した手法で、時刻埋め込みの直接の祖先です。あちらは整数の位置、こちらは連続値の時刻、という違いだけです。

第五に「内積と類似度」。sin/cos 符号化の数学的なうまみは、2つの時刻 t と t' の埋め込みの内積が、おおむね差 |t−t'| だけに依存する(近い時刻は似た埋め込み、遠い時刻は似ない)ことです。これによりネットは「いまの時刻が学習で見たどの時刻に近いか」をなめらかに扱えます。

## 仕組みを詳しく

典型的な正弦波時刻埋め込みの計算を段階で分解します。埋め込み次元 D=128 とします。

入力: `t` の形状 `[B]`(B はバッチサイズ。中身は各サンプルの時刻、例 `[0.05, 0.5, …]`)。
出力: `[B, 128]`。

ステップ1: 半分の次元を決める。`half = 128 // 2 = 64`。前半64次元を sin、後半64次元を cos に使います。

ステップ2: 64個の周波数を作る。
```
freqs = exp( -log(10000) * arange(64) / 64 )
```
1.0 から 1/10000 まで、対数スケールでなめらかに小さくなる64個の数です。
- i=0 で exp(0)=1.0(高周波。t が少し変わると大きく変化)。
- i=63 で exp(−log(10000)·63/64) ≈ 1/10000(低周波。t が大きく変わってもゆっくり変化)。

高周波と低周波を混ぜることで、t の細かい違い(高周波)と大まかな位置(低周波)を同時に表せます。基底 10000 は Transformer から受け継いだ慣習値です。なお拡散モデルでは t を 0〜1 ではなく 0〜1000 の整数で扱う実装もあり、その場合は基底や周波数範囲の有効帯が変わるため、時刻のスケールと周波数設計の整合が重要になります。

ステップ3: 各サンプルの t に各周波数を掛ける。
```
args = t[:,None] * freqs[None,:]   # [B,64]
```
t=0.5 なら args = `[0.5, 0.46, …, 0.00005]` の64個。

ステップ4: sin と cos を取って連結する。
```
emb = concat( sin(args), cos(args) )   # [B,128]
```
1個の数 t が128次元のベクトルになりました。

```
t=0.5 →  sin成分(64個)         cos成分(64個)
         [0.48, 0.44, ... 0.00005 | 0.88, 0.90, ... 1.0]
          高周波 ──────→ 低周波   高周波 ──────→ 低周波
```

ステップ5(下流で一般的): 生の埋め込みをさらに小さな MLP に通して埋め込み次元(たとえば256)へ整えます。
```
time_mlp = Linear(128→256) → SiLU → Linear(256→256)
```
SiLU は活性化関数(入力に応じてなめらかに信号を通したり止めたりする関数、x·sigmoid(x))。この MLP の役割は、固定の sin/cos 基底を「このタスクで実際に有用な時刻表現」へ学習で変換することです。生の sin/cos は固定ですが、MLP の重みが学習されるので、結果として時刻表現はタスクに適応します。この最終ベクトルが下流の AdaLN([AdaLN による時刻・文脈の条件付け（Adaptive LayerNorm）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0109_adaln-time-conditioning))に渡され、速度場ネットワークの振る舞いを時刻に応じて変調します。

「なぜ次元は偶数でなければならないか」: sin と cos に半分ずつ割り当てるので、奇数だと割り切れません。実装では奇数次元はエラーにするのが普通です。

## 手法の系譜と主要論文

Vaswani ら(Attention Is All You Need, NeurIPS 2017, arXiv:1706.03762)が sin/cos の位置符号化を提案しました。Transformer は単語の順番を本質的に知らないので位置情報を明示的に足す必要があり、固定の sin/cos を選んだ理由は「学習可能な位置埋め込みでも性能はほぼ同じだったが、sin/cos なら訓練で見たことのない長い系列にも外挿しやすい」と述べています。トレードオフは絶対位置しか直接表現できず、相対位置は層が暗黙に学ぶ必要がある点で、のちの相対位置符号化や RoPE の動機になりました。

Ho, Jain, Abbeel(DDPM, NeurIPS 2020, arXiv:2006.11239)が、ノイズ除去ネットワーク(U-Net)に「いまどのノイズ段階か」を伝えるため Transformer 流の sinusoidal timestep embedding を採用しました。これが連続的な時刻 t を sin/cos で符号化する現在の定番の出発点です。効果は、1つのネットワークで全ノイズ段階を共有して扱える(段階ごとに別ネットを作らなくてよい)こと。

Tancik ら(Fourier Features Let Networks Learn High Frequency Functions, NeurIPS 2020, arXiv:2006.10739)が、低次元入力を sin/cos のフーリエ特徴に写すと MLP が高周波関数を学べるようになることを理論(neural tangent kernel)と実験で示しました。時刻埋め込みが効く根本理由を与えた論文です。ほぼ同時期の Mildenhall ら(NeRF, ECCV 2020, arXiv:2003.08934)も座標の正弦波符号化(positional encoding)で3次元シーンの細部を再現でき、低次元座標の正弦波展開が広く効くことを示しました。

Lipman ら(Flow Matching, ICLR 2023, arXiv:2210.02747)は時刻 t に依存する速度場 v(x,t) をネットで表す枠組みで、Diffusion と同じく t の埋め込みを使います。Peebles & Xie(DiT, ICCV 2023, arXiv:2212.09748)は timestep embedding を MLP に通したのち AdaLN 経由で各 Transformer ブロックに注入する設計を示し、現在の Flow/Diffusion での時刻埋め込みの使い方(埋め込み→MLP→AdaLN の scale/shift)を確立しました。

## 論文の実験結果(定量データ)

Tancik ら(2020)は、座標から色を回帰する画像再構成や3D形状の符号付き距離関数の学習で、生の座標を直接入れた MLP は高周波(細部)を再現できずぼやけるのに対し、フーリエ特徴を入れると PSNR(ピーク信号対雑音比、画質の指標で大きいほど良い)が大幅に向上することを示しました。報告ではタスクによって PSNR が数 dB から十数 dB 改善し、視覚的にも細部が鮮明になりました。これは「正弦波埋め込みが微小な差を区別できるようにする」効果の定量証拠です。NeRF(Mildenhall et al., 2020)でも positional encoding を外すと描画がぼやけ、PSNR が明確に低下することがアブレーションで示されています。

DDPM(Ho et al., 2020)は CIFAR-10 で当時最良級の FID(報告値3.17)と Inception スコアを達成しましたが、その構成要素として timestep embedding が全段階共有の単一ネットを可能にしました。アブレーションの観点では、時刻条件を抜く(全段階を区別しない)とノイズ除去ネットがどの段階かを推定できず品質が崩れます。時刻埋め込みは単体で評価される部品ではなく、抜くと生成プロセス全体が成立しなくなる土台です。

位置符号化(Vaswani et al., 2017)については、論文中で学習可能な位置埋め込みと sin/cos 固定埋め込みを比較し、翻訳品質(BLEU スコア)がほぼ同等だったと報告しています。性能は同じでも外挿性で sin/cos を選んだ、というのが設計判断の根拠です。

## メリット・トレードオフ・限界

メリット:
- 1個のスカラー t を表現力の高い高次元ベクトルに変換でき、ネットが時刻ごとの振る舞いの違いを学びやすくなる(高周波の細かい差も区別できる)。
- sin/cos 部分は学習可能なパラメータを持たない(計算だけ)ので軽量で過学習の心配がない。
- 訓練で見ていない t 値にも比較的なめらかに外挿でき、近い時刻は似た埋め込みになる連続性を持つ。

トレードオフ・限界:
- 周波数の範囲(基底10000)や次元数は手で決めるハイパーパラメータで、タスクによっては調整が要る。基底が不適切だと有効な周波数帯がずれ、時刻のスケール(0〜1 か 0〜1000 か)と噛み合わないと細かい差を区別できなくなる。
- 表現はあくまで滑らかな関数を前提とし、t に対して極端に不連続な振る舞いが必要なタスクでは別の工夫が要ることがある。
- それ単体では何も決定しない。後段の MLP や AdaLN と組み合わせて初めて意味を持つ「素材」である。

未解決の論点として、連続時刻に対して固定 sin/cos が常に最良かは自明でなく、学習可能なフーリエ特徴やランダム周波数(Tancik の random Fourier features)を使うべき場面の切り分けは経験則に依存します。

## 発展トピック・研究の最前線

- 学習可能フーリエ特徴: 周波数自体を学習するランダムフーリエ特徴。座標ベース生成(NeRF 系)で広く使われ、時刻埋め込みにも応用されます。固定基底より柔軟だが過学習しうるトレードオフがあります。
- 相対符号化・RoPE: 位置符号化の系譜で発展した相対位置・回転位置符号化。時刻には直接使わないが、同じトークン列に位置と時刻を併せて与える設計で関連します。
- 条件埋め込みの統一: 時刻だけでなくノイズレベル・ガイダンス強度・クラスなど複数のスカラー条件を同じ正弦波埋め込みで符号化し、まとめて AdaLN に流す設計。
- スケジュールとの相互作用: 時刻サンプリング([Flow Matching の時刻サンプリング戦略](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0125_flow-timestep-sampler))や経路設計と時刻埋め込みの相性。どの周波数帯が学習で実際に使われているかをスペクトル解析する研究が進んでいます。
