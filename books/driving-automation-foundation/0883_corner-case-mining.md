# コーナーケース発掘

## ひとことで言うと

コーナーケース発掘(corner case mining)とは、自動運転などで「めったに起きないが、起きると危険な状況(道路上の落下物、見慣れない車両、急な飛び出しなど)」を、膨大なデータの中から効率よく見つけ出して集め、評価や追加学習に使うための一連の手法です。

## 直感的な理解

まず言葉を噛み砕きます。

コーナーケース(corner case)とは、もともと工学で「複数の条件が極端な値で同時に重なったときに起きる、めったに見ない状況」を指します。自動運転では「学習データにほとんど現れないが、安全に直結する稀なシーン」という意味で使われます。エッジケース(edge case)とほぼ同義で使われることもあります。

ロングテール(long tail、長い尾)とは、データの出現頻度を棒グラフに描いたとき、よく出るもの(歩行者・乗用車)が左端に高くそびえ、稀なもの(横転したトラック、道に出た野生動物、工事の特殊車両)が右へ向かって低く長く尾を引く形を指します。問題は、この長い尾の一つ一つが稀でも、合計すると無視できない頻度になり、しかも事故はしばしばこの尾の部分で起きることです。

```
出現頻度
 │█  歩行者
 │█  乗用車
 │█▁  自転車
 │ ▁▁  ……
 │   ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁  ← ロングテール(稀で危険なケースが延々と続く)
 └──────────────────────────→ クラスやシーンの種類
```

直感的な課題はこうです。普通に大量の走行データを集めると、その99パーセント以上は「何事もない平凡な走行」です。モデルはそういう平凡なシーンには強くなりますが、肝心の稀で危険な状況の例がほとんど含まれないため、そこで失敗します。だから「平凡を捨て、稀を選んで集める」仕組みが要ります。これがコーナーケース発掘です。膨大な干し草の山(平凡な走行)から、針(危険なシーン)を効率よく拾う作業に例えられます。

## 基礎: 前提となる概念

- 分布外(out-of-distribution、OOD): 学習データの「いつもの分布」から外れた入力。見たことのない物体や状況。コーナーケースの多くは OOD です。
- 不確実性(uncertainty): モデルが「自分の予測にどれだけ自信がないか」を表す量。自信がない入力は、未知=コーナーケースである可能性が高い、という発想で発掘に使います。
- 能動学習(active learning): 全データにラベルを付けるのではなく、「ラベルを付けると最も学習効果が高そうなデータ」をモデル自身に選ばせ、人間がそれだけラベル付けする枠組み。コーナーケース発掘の中心的な道具です。
- アノテーション(annotation): データに正解ラベルを付ける作業。高価で時間がかかるので、何にラベルを付けるかの選別が重要になります。
- 偽陰性(false negative): 本当は危険物があるのに「無い」と見逃すこと。安全上、最も避けたい失敗です。コーナーケース発掘は偽陰性を減らすことが究極の目的です。

## 仕組みを詳しく

コーナーケースを「どう見つけるか」には、大きく分けていくつかのアプローチがあります。

第一に、不確実性に基づく発掘です。モデルが出力する予測の確信度を見て、確信度が低い・予測がばらつくサンプルを稀なケースの候補とします。代表的な指標を言葉で説明します。

- エントロピー(entropy): 予測クラスの確率分布の「散らばり具合」。H = -Σ_c p_c log p_c。p_c はクラス c の予測確率。すべてのクラスにまんべんなく確率が散らばっている(=どれか決めきれない)ほど H が大きく、不確実だと判断します。
- アンサンブルの不一致: 複数のモデル(または推論時にニューロンをランダムに落とす MC ドロップアウト)で同じ入力を予測させ、答えがばらつくほど不確実とみなす。
- 再構成誤差: オートエンコーダ(入力を一度圧縮して復元するネットワーク)に通し、うまく復元できない入力を「いつもと違う=OOD」とみなす。

第二に、予測の不整合に基づく発掘です。たとえばカメラとLiDARで答えが食い違う、時間的に連続するフレームで物体が突然消える/出現する、地図と知覚が矛盾する、といった「整合性が崩れる箇所」を稀ケースの兆候とします。複数の情報源が一致しないときは何か異常が起きている、という考え方です。

第三に、ルール・記述に基づく発掘です。「急ブレーキが踏まれた」「ドライバーが介入した(自動運転を解除した)」「衝突に近い接近(near-miss)が記録された」といった、運転データに付随するイベントログを手がかりに、その前後の映像を抜き出します。人間が危険を感じて反応した瞬間こそ、コーナーケースの宝庫だからです。

第四に、生成・シミュレーションによる発掘です。実データで足りない稀ケースを、シミュレータ(CARLA など)や生成モデルで人工的に作り出し、不足を補います。

これらで候補を絞ったら、能動学習のループが回ります。

```
大量の無ラベル走行データ
        │
   ① モデルで全部を推論
        │
   ② 不確実性/不整合スコアで並べ替え
        │
   ③ 上位だけ人間がラベル付け(コスト集中)
        │
   ④ そのデータでモデルを再学習
        │
        └──→ ① へ戻る(賢くなったモデルで再び選別)
```

能動学習の理論的背景は Settles の "Active Learning Literature Survey"(2009)にまとまっており、不確実性サンプリング(uncertainty sampling)、委員会による問い合わせ(query-by-committee、複数モデルの不一致を使う)、多様性を確保する選択(似たものばかり選ばない)などの戦略が整理されています。実務では「不確実かつ多様」を両立させる選び方が重要で、不確実なだけだと似た失敗シーンばかり集まってしまうからです。

## 手法の系譜と主要論文

- Settles, "Active Learning Literature Survey"(技術報告, 2009): 能動学習の戦略を体系化した古典。何にラベルを付けるべきかの理論的土台。
- Heidecker ら, "An Application-Driven Conceptualization of Corner Cases for Perception in Highly Automated Driving"(IV 2021): 自動運転知覚におけるコーナーケースを体系的に分類(ピクセルレベル・物体レベル・シーンレベル・シナリオレベルなど)。「コーナーケースとは何か」の定義づけに貢献。
- Bolte ら, "Towards Corner Case Detection for Autonomous Driving"(IV 2019): 映像から異常・コーナーケースを検出する枠組みの初期研究。
- Li ら, "CODA: A Real-World Road Corner Case Dataset for Object Detection in Autonomous Driving"(arXiv:2203.07724, ECCV 2022): 実世界の道路コーナーケースに特化した評価用データセットを構築し、既存検出器がそこで大きく性能低下することを定量的に暴いた代表作。本ノートの中心。
- OOD 検出の系譜: Hendrycks と Gimpel(ICLR 2017)のソフトマックス確信度ベースライン、Liang らの ODIN(ICLR 2018)、Lee らのマハラノビス距離(NeurIPS 2018)などが、稀ケース検出の基礎技術として転用されます。

## 論文の実験結果(定量データ)

CODA(Li ら, ECCV 2022)が示した数値が、この分野の問題を端的に物語ります。CODA は既存の大規模運転データセット(nuScenes、ONCE、KITTI など)から、稀で危険なコーナーケース(典型クラスに当てはまらない道路上の物体や異常な状況)を含むシーンを集め、検証用に構築されました。報告によれば、データには千数百枚規模のシーンと、多数のコーナーケース物体が含まれます。

核心的な結果は、「通常のベンチマークで高性能な物体検出器が、CODA 上では劇的に性能が落ちる」というものです。論文の報告では、一般的なデータセット(たとえば大規模データで学習した最先端検出器)で良好な平均適合率(mAP、mean Average Precision)を出すモデルでも、CODA のコーナーケース物体に対しては mAP がおよそ 12 パーセント台といった低い水準にとどまる、と示されました。数値の意味を補足すると、mAP が低いとは「危険な未知物体の多くを検出できていない(見逃している)」ことを表します。つまり、平凡なベンチマークでの高得点は、安全に直結する稀ケースでの安全性をまったく保証しない、という警鐘です。

能動学習側の定量的な傾向としては、一般に「ランダムにラベル付けする場合と比べ、不確実性ベースで選んでラベル付けすると、同じ精度に到達するのに必要なラベル数を大きく削減できる(タスクやデータにより数十パーセント規模の削減が報告される)」ことが、多くの実証研究で示されています。具体値はデータセット・タスク・選択戦略に強く依存するため、ここでは「同じ性能をより少ないラベルで達成できる」という傾向として理解するのが安全です。

これらの数値の含意は明確です。第一に、評価データセット自体に稀ケースを意図的に含めないと、モデルの本当の弱点が見えない。第二に、無作為にデータを集めるより、賢く選別したほうがラベル予算を有効に使える、ということです。

## メリット・トレードオフ・限界

メリットは、限られたアノテーション予算を「効果の高いデータ」に集中投下できることです。安全が重要な自動運転では、平凡なデータをいくら積んでも埋まらない稀ケースのギャップを、ピンポイントで埋められる価値があります。評価面では、コーナーケースを含むベンチマークによって、見かけの高精度に隠れた本当の弱点を露呈させられます。

トレードオフと限界もあります。

第一に、「稀」の定義と発見の難しさです。本当に危険なケースは定義上めったに起きず、しかも事前に列挙しきれません(知らない未知の未知、unknown unknowns)。不確実性スコアは「モデルが知らないもの」を捉えますが、「モデルが自信満々に間違える」危険なケース(自信過剰な誤検出)は取りこぼしやすい、という根本的な弱点があります。

第二に、選別バイアスです。能動学習で「モデルが不確実なもの」ばかり集めると、データ分布が偏り、後の学習で別の偏りを生む可能性があります。多様性の確保や、選別過程の評価が必要です。

第三に、アノテーションのコストと一貫性です。稀で曖昧なシーンほど、人間がラベルを付けるのも難しく、アノテータ間でばらつきが出ます。「これはコーナーケースか否か」の判断自体が主観を含みます。

第四に、人工データの妥当性ギャップです。シミュレーションや生成で稀ケースを補っても、それが本物の稀ケースの分布を正しく表しているとは限らず、合成データで上がった指標が実走での安全に直結する保証はありません。

第五に、評価の公平性です。コーナーケースだけで評価すると数値は悲観的に出ますが、それが現実の運行リスクをどれだけ表すかは、稀ケースの実際の出現頻度と重み付け次第で、評価設計が難しい論点です。

## 発展トピック・研究の最前線

OOD 検出との融合: 走行中にリアルタイムで「これは未知物体だ」と検出し、システムを安全側に倒す(減速・ドライバーへ引き継ぎ)研究が進んでいます。発掘(オフラインでデータを集める)と検出(オンラインで危険を察知する)は技術を共有します。

基盤モデル・VLM の活用: 大規模に学習された視覚言語モデル(VLM、画像と言葉を結びつけるモデル)を使い、「道路上の異常」を言葉で記述・検索してコーナーケースを自動抽出する試みが登場しています。「珍しいシーン」を自然言語クエリで掘れる可能性があります。

シナリオ生成と敵対的探索: モデルが失敗しやすいシナリオをシミュレータ上で能動的に探索する(敵対的シナリオ生成)研究があり、安全性検証の自動化につながります。

自動データエンジン: 「収集 → 自動選別 → ラベル付け → 再学習」をループとして高度に自動化する「データエンジン」の構築が、産業界の自動運転開発で重要テーマになっています。コーナーケース発掘はそのループの心臓部です。

評価標準化: コーナーケースの分類体系(Heidecker ら)を踏まえ、稀ケースを含む評価ベンチマークの整備と、安全性指標(見逃し率、最悪ケース性能)の標準化が課題として残っています。
