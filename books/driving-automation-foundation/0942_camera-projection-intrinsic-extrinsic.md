# カメラ射影 (内部・外部パラメータ)

## ひとことで言うと
「世界の中の3次元の点(例: 自車の前方20m・左3m・高さ1mの場所)が、カメラ画像のどのピクセルに写るか」を計算する数式の話です。これは2つの行列、内部パラメータ(intrinsic)と外部パラメータ(extrinsic)の掛け算で表せます。BEV(鳥瞰図)を作るときに「3D参照点をカメラ画像に射影する」操作の土台になり、コンピュータビジョン全体の基礎でもあります。

## 直感的な理解
カメラは3次元の世界を2次元の写真に変換する装置です。BEVを作るには、この変換を逆向きに考える必要があります。つまり「BEVのこのマスに対応する3D点が、各カメラの画像のどこに写っているか」を知り、その位置の画像特徴を拾ってくる。この「3D点 → 画像ピクセル」の対応を計算するのがカメラ射影です。

身近な例で考えます。あなたが部屋のある場所に立ち、ある方向を向いてスマホで写真を撮るとします。「部屋の角の3D位置」が「写真のどのピクセルに写るか」は、2つのことで決まります。1つはスマホのカメラの性質(ズームの強さ=焦点距離、センサーの中心)。もう1つはあなたがどこに立ってどちらを向いていたか(位置と向き)。この2つさえ分かれば、部屋のどの点が写真のどこに来るかは幾何で一意に計算できます。前者がカメラ固有の内部パラメータ、後者が設置情報の外部パラメータです。

## 基礎: 前提となる概念
**ピンホールカメラモデル**: カメラを「1点の穴を通して光が入り、奥のスクリーンに像を結ぶ箱」と理想化したモデル。レンズの歪みを無視する代わりに、3D→2Dの射影を行列の掛け算という単純な形で書けます。

**座標系を3つ区別する**: 射影を理解する鍵は座標系をはっきりさせることです。
- ego/vehicle座標: 自車中心の3次元座標。典型的な規約では X=前方・Y=左・Z=上(メートル単位)。
- カメラ座標: そのカメラを原点にした3次元座標。光軸(レンズが向く方向)を基準にとる。
- ピクセル座標: 画像上の (u, v)。u=横ピクセル(0〜画像幅)、v=縦ピクセル(0〜画像高さ)。

**内部パラメータ(intrinsic)**: カメラ自体の性質。焦点距離(ピクセル単位の fx, fy)と主点(光軸が画像と交わる点、おおよそ画像中心の cx, cy)を含みます。同じカメラなら世界のどこにあっても変わらない、カメラ固有の値です。

**外部パラメータ(extrinsic)**: カメラが世界のどこに、どの向きで取り付けられているか。回転 R(3×3、向き)と並進 t(3×1、位置)で表します。

**カメラ校正(キャリブレーション)**: 内部・外部パラメータを事前に測定する作業。チェッカーボードを様々な角度で撮って解く方法(Zhang法)が代表的で、データセットには校正値が付属します。

**同次座標(homogeneous coordinates)**: 点を1次元増やして表す表記。これにより回転・並進・遠近をすべて1つの行列積にまとめられ、最後に割り戻して本当の座標に戻します。

## 仕組みを詳しく
射影は次の流れで進みます。座標系の変換を一段ずつ追います。

**ステップ1(外部パラメータ extrinsic): ego座標 → カメラ座標。** ego座標の点 p を、回転 R と並進 t でカメラ座標に移します。

```
p_cam = R · p + t        (R: 3×3 回転, t: 3×1 並進)
```

行列で書くと [R | t] という 3×4 の行列になり、同次座標 [X, Y, Z, 1]ᵀ に掛けるとカメラ座標が出ます。

**ステップ2(内部パラメータ intrinsic): カメラ座標 → ピクセル座標。** 内部行列 K を掛けます。

```
      [ fx   0   cx ]
K  =  [  0  fy   cy ]
      [  0   0    1 ]
```

fx, fy は焦点距離(ピクセル単位)。cx, cy は主点。fx が大きいほど望遠(遠くが大きく写る)、小さいほど広角です。

**ステップ3(合成 = ego→pixel): 2つを1つの行列にまとめる。** 多くのBEV実装は、内部と外部を掛け合わせた1個の 3×4 射影行列を使います。

```
P = K · [R | t]          ( = intrinsic @ extrinsic[:3, :] )
```

複数カメラを扱うときは、これを camera_params として形状 [B, V, 3, 4] のtensorで渡します。読み方は「バッチB個、カメラV台、各カメラは3×4の射影行列」です。

**ステップ4(同次座標とパースペクティブ除算): 割り算で遠近を表す。** 3D点 (X,Y,Z) に1を足して同次座標 (X,Y,Z,1) にし、P を掛けると 3要素 [u′, v′, depth] が出ます。最後に depth で割ると本当のピクセル座標になります。

```
[u′]              [X]
[v′]  =  P (3×4) · [Y]          ref_2d = (u′/depth, v′/depth)
[ d ]             [Z]
                  [1]
```

3番目の要素 d = depth は「カメラの光軸方向の距離(正なら前方)」を表します。割り算(perspective division、透視除算)こそが遠近の効果を生む操作で、depth が大きい(遠い)ほど u′/depth が小さくなり、画像中心寄りに縮みます。depth>0 のチェックはカメラの後ろにある点を除外する可視性判定([可視性マスキング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1027_visibility-masking-bev))に使われます。

**ステップ5(正規化): 画像サイズで割って [0,1] に。** 得た (u, v) を画像の幅・高さで割り、[0,1](または [−1,1])の正規化座標にします。これは後段の特徴サンプリング([grid_sampleによる特徴サンプリング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0969_grid-sample-feature-sampling))が正規化座標を要求するためです。

**数値例。** 焦点距離 fx=fy=256、主点 cx=cy=128、画像サイズ256のカメラを考えます。カメラ正面 前方10m・高さ0・横0 の点は、おおよそ画像のど真ん中 (u,v)=(128,128) に写り depth=10。256で割ると正規化 (0.5, 0.5)。これは「真正面の点は画像中央に来る」という直感と一致します。同じ点を前方20mに動かすと depth=20 になり、画像上の見かけの位置は変わらず中央のまま、しかし depth の値が2倍――この depth 情報が、後段で「どの参照点が手前/奥か」を区別するのに使われます。

## 手法の系譜と主要論文
- **Hartley & Zisserman「Multiple View Geometry in Computer Vision」(2003)**。内部・外部パラメータ、ピンホールカメラモデル、同次座標による射影 P = K[R|t] を体系化した基礎文献。動機は、ピンホール幾何を線形代数(行列積)で扱えるようにすること。同次座標を使うと並進も掛け算1回にまとめられます。あらゆる多視点ビジョンの土台ですが、理想ピンホールモデルなので実レンズの歪み(distortion)は別途補正が必要、という限界があります。

- **Zhang (IEEE TPAMI 2000)「A flexible new technique for camera calibration」**。チェッカーボードを数枚撮るだけで内部・外部パラメータを推定できる実用的校正法を確立。今日のほぼすべてのカメラ校正の基盤で、データセットに付属する校正値の多くがこの系譜の手法で得られています。

- **Philion & Fidler (ECCV 2020)「Lift-Splat-Shoot」(arXiv:2008.05711)**。同じ内部・外部パラメータを使い、予測した深度分布に沿って特徴を3D空間へ持ち上げます。射影の数学は古典と共通で、深度の扱い方(確率分布で予測する)が新しい点です。

- **Li, Wang, Lu et al. (ECCV 2022)「BEVFormer」(arXiv:2203.17270)**。各BEVクエリの3D参照点([ピラー参照点 (num_points_in_pillar)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0997_pillar-reference-points))を、各カメラの ego→pixel 射影行列で画像平面に落とし、その位置の特徴を集めます。動機は、深度を陽に予測せず幾何(校正済みカメラ行列)に頼ることで深度推定誤差を回避すること。利点は校正が正確なら幾何的に正しい対応が得られること。限界は校正がずれるとBEV全体がずれる、校正品質への強い依存です。

系譜としては、古典的多視点幾何(射影の数式)→ Zhang法(校正の実用化)→ LSS/BEVFormer(その射影を深層学習のBEV構築に組み込む)という流れで、射影の数学そのものは数十年来不変のまま、その使い方が進化しています。

## 論文の実験結果(定量データ)
カメラ射影自体は数学的に決まる操作なので「精度」を測る対象ではありませんが、それを使うBEV手法の性能と、校正誤差への感度として定量的な知見があります。

BEVFormer(ECCV 2022)はnuScenes(6カメラ)の3D物体検出でNDS(nuScenes Detection Score、0〜1で大きいほど良い)およそ0.51前後を達成し、これは「校正済みカメラ行列で参照点を射影し特徴を集める」幾何ベースの設計が、深度予測ベースの手法と同等以上に機能することを示しています。

校正誤差への感度は研究上重要なテーマです。外部パラメータの回転(カメラの向き)が数度ずれるだけで、遠方の射影位置が大きくずれ、3D検出の位置誤差(nuScenesのmATE、平均並進誤差。小さいほど良い)が悪化することが、ロバスト性評価の論文で報告されています。直感的なオーダー感覚として、カメラの向きが 1度 ずれると、前方 50m の点の射影先は地面換算でおよそ 50m × tan(1度) ≈ 0.87m ずれます。遠方ほどこのズレが線形に拡大するため、100m先では 1.7m 程度、つまり車1台分ずれることになります。実際、nuScenes-C のような外乱を加えたベンチマークでは、外部パラメータに小さな回転ノイズを注入すると BEVFormer 系の NDS が数ポイント低下することが報告されており、これは「射影は幾何的に正確だが、その正確さは校正の正確さに完全に依存する」という、幾何ベース手法の本質的トレードオフを数値で裏づけます。深度予測ベースの手法は校正誤差にやや頑健な代わりに深度予測誤差を抱える、という対照的な弱点を持ちます。

指標の意味を補足します。mATE(mean Average Translation Error)は検出した物体の中心位置が正解からどれだけ離れているか(メートル)の平均で、校正がずれると射影位置がずれ、その結果として物体の推定位置がずれるため、この指標が直接悪化します。

## メリット・トレードオフ・限界
**メリット**: (1) 物理的に正しい3D→2D対応が、行列積1回(と割り算)で計算できる。(2) 内部・外部を分けることで、カメラ交換時は内部だけ、取付変更時は外部だけ更新すればよい。(3) 深度予測を学習しなくてよい(対応が幾何で決まる)。(4) 全自動運転データセットが共通の枠組みで校正値を提供するため、相互運用しやすい。

**トレードオフ・限界**: (1) 校正(キャリブレーション)の精度に強く依存し、ずれるとBEV全体がずれる。(2) レンズ歪みなど非線形効果はこの線形モデルでは表せず、別補正(歪み係数による前処理)が必要。(3) 走行中の振動・温度変化で校正が経時的にずれる(オンライン再校正が課題)。(4) 校正データが無いと真の射影はできない。

**研究上の未解決課題**: 走行データから校正を自動で推定・補正するオンライン自己校正、校正誤差に頑健なBEV構築(誤差を学習で吸収する)、魚眼など強い歪みを持つカメラの統一的な射影モデルなどが活発に研究されています。

## 発展トピック・研究の最前線
広角・魚眼カメラでは線形ピンホールモデルが破綻するため、球面モデルや多項式歪みモデルでの射影が必要になります。最近のサラウンドビュー(車両周囲を魚眼で覆う構成)BEV手法は、こうした非線形射影を学習に組み込む方向に進んでいます。また、校正のずれを学習で吸収する「校正に頑健なBEV」や、複数カメラ間のオーバーラップ領域を使って外部パラメータをオンライン推定する研究が進展しており、量産車のように校正が経時変化する現実環境への適用が焦点になっています。
