# 凍結バックボーンによるターゲット生成

## ひとことで言うと
自動運転モデルに「数秒後の世界がどう見えるか」を予想させて学習させたいとき、未来の「正解」をどう用意するかが問題になります。この手法は、未来の実際のカメラ画像を「学習中は一切変化しない(凍結された)画像認識ネットワーク」に通して特徴ベクトルに変換し、それを正解ターゲットとして使います。生のピクセルではなく抽象的な特徴を予測させることで、モデルがシーンの本質的な動き(車がどこへ動くか、歩行者がどう移動するか)を学べるようにします。ターゲット側を凍結する理由は、学習が「正解を予測しやすいように崩す」インチキ解(崩壊)に陥るのを防ぐためです。

## 直感的な理解
未来予測を学ばせるには、予測と比べる「正解」が必要です。一番素朴な正解は「未来のカメラ画像そのもの(ピクセル)」です。1.6 秒後の実際の映像を持ってきて、モデルの予測した映像と見比べ、ずれを縮めるように学習させる、という発想です。

ところがこの素朴な方法には大きな弱点があります。ピクセル予測は細部に引きずられます。葉っぱの揺れ、影のちらつき、カメラのノイズ、雨粒。これらは運転には無関係なのに、ピクセルを完璧に当てようとするとモデルはこういう細部まで再現しようと労力を割き、肝心の「車がどこへ動くか」という本質を学びそびれます。しかも、450×300 のような画像はピクセル数が 13 万を超え、これを正確に再現させる学習はとても非効率です。さらに未来は本質的に多峰(対向車が来るか来ないか等、複数の可能性がある)なので、ピクセルを 1 枚に決め打ちさせると、モデルは複数の可能性を平均したぼやけた画像を出すという既知の失敗に陥ります。

そこで発想を変えます。「ピクセルを当てる」のではなく「特徴を当てる」。特徴とは、画像認識ネットワークが画像を処理した後に出す抽象化された数値の束で、「この辺に車がいる」「ここは道路」といった意味を圧縮して持っています。未来の画像をネットワークに通して特徴に変えてから、その特徴を予測させれば、細部のノイズは特徴抽出の段階で捨てられ、運転に効く意味的な情報だけが残ります。これが特徴空間での未来予測です。

ただし、ここで一つ落とし穴があります。特徴を作るネットワーク(ターゲット生成器)も学習で一緒に動かしてしまうと、モデルはずるをします。極端には、ターゲットを全部ゼロにしてしまえば「予測 = 正解 = ゼロ」で誤差ゼロになり、何も学んでいないのに損失だけ下がる。これを表現の崩壊(collapse)と呼びます。これを防ぐために、ターゲット生成器を凍結します。

## 基礎: 前提となる概念
- バックボーン(backbone): モデルの土台となる特徴抽出器のこと。画像を入れると特徴マップを出すネットワーク(ResNet、Swin Transformer、ConvNeXt、ViT など)です。

- 特徴空間(feature space): バックボーンが出す抽象化された数値表現の空間。例えば形状 `[B, 256, H, W]` のテンソルで、B はバッチサイズ(同時処理枚数)、256 はチャンネル数(各位置がもつ特徴の種類数)、H/W は空間サイズです。

- 凍結(frozen): 学習中にネットワークの重み(パラメータ)を一切更新しないこと。具体的には、各パラメータの requires_grad を False にし、`torch.no_grad()` の下で実行して勾配(gradient = 重みを更新するための微分情報)を計算・伝播させないことです。

- detach(切り離し): あるテンソルを計算グラフ(勾配を逆向きに流すための記録)から切り離す操作。detach したテンソルから勾配が遡らないので、それより手前のネットワークは更新されません。

- stop-gradient(勾配停止): 片側の経路で勾配を止めること。detach と同義で使われます。自己教師あり学習で崩壊を防ぐ核心的なテクニックです。

- 崩壊(collapse): 自己教師あり学習で、ネットワークが入力によらず同じ(あるいは無意味な)出力を返すようになり、損失は下がるのに有用な表現を学んでいない状態。崩壊には完全崩壊(全部同じベクトル)と次元崩壊(出力が低次元の部分空間に潰れる)があり、ターゲット側を更新可能にすると起きやすい失敗です。

- EMA(指数移動平均, Exponential Moving Average): あるネットワークの重みを、別のネットワークの重みへ少しずつコピーして滑らかに追従させる更新法。`θ_target ← m·θ_target + (1−m)·θ_online`(m は 0.99〜0.999 程度のモメンタム)で更新します。

## 仕組みを詳しく
全体の流れは次の通りです。

1. 現在のフレームからバックボーンで融合特徴 `[B, 256, H, W]` を作ります。複数カメラと地図を統合した BEV 特徴です。

2. 運転意図(ego の隠れ状態, `[B, 256]`)をチャンネルごとのバイアスとして足し込みます。「この軌道を走るつもりなら未来はこう見える」という条件付けが入ります([FutureState 世界モデル](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0013_future-state-world-model))。

3. 数層の畳み込みで未来 4 ステップ分の特徴 `4C` を一気に出し、`torch.chunk` で 4 つに分割します。各々 `[B, 256, H, W]`。これらが +1.6s / +3.2s / +4.8s / +6.4s 後の「予測特徴」です。

ここからが本トピックの核心、「正解(ターゲット)」の作り方です。

- 未来の実フレーム(1.6 秒後、3.2 秒後…の実際のカメラ画像)を用意します。
- それを凍結されたバックボーンのコピーに通します。凍結とは前述の通り、重みを更新せず `torch.no_grad()` 下で実行することです。
- 出てきた特徴(各 `[B, 256, H, W]`)を detach してターゲットにします。

そして予測とターゲットの差を、特徴再構成ロスで測ります。中身は各タイムステップごとの平均二乗誤差(MSE)`mean((pred - target)^2)` で、ステップごとの重み付けは [時系列重み付けロスとゼロ和ガード](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0569_temporal-weighting-loss) が担います。

```
現在フレーム ──backbone──▶ fused_features [B,256,H,W]
                                  │ + ego bias
                                  ▼
                            FutureState (Conv x2)
                                  ▼
        予測: 4 x [B,256,H,W]  (+1.6s .. +6.4s)
                                  │  MSE
未来フレーム(実画像) ─凍結backbone(no_grad,detach)─▶ ターゲット: 4 x [B,256,H,W]
```

なぜ凍結が決定的に重要かをもう一度。ターゲット生成器に勾配を流すと、モデルは「予測しやすいように正解側を崩す」自明解に逃げます。両側を更新可能にすると、ネットワークは入力に依らず一定の特徴(極端にはゼロ)を出すようになり、予測もそれに合わせれば損失ゼロ。表現の崩壊です。ターゲット側を凍結して動かさなければ、正解は「本物の未来画像の意味的な特徴」に固定されるので、予測側はそれを当てるために本当の動態を学ぶしかなくなります。

実務上の注意点。予測特徴とターゲット特徴は、チャンネル数も空間解像度も一致していなければ MSE が取れません。現在フレーム用のバックボーンと、ターゲット用の凍結バックボーンが別の出力次元を持つ場合(例えば features_only 出力の解像度が違う)、両者を揃える射影層(1×1 畳み込みなど)や補間が必要になります。次元の不整合は、この設計を実装するときに最初にぶつかる典型的な落とし穴です。

ターゲット生成器を「完全凍結」にするか「EMA で緩やかに更新」するかは設計の分岐点です。完全凍結は単純で安定ですが、学習が進んでもターゲットの質が上がりません(事前学習済みの固定された表現が予測の上限を決めます)。EMA 更新(BYOL/MoCo/V-JEPA 流)は、予測側の重みを少しずつターゲット側にコピーして「ゆっくり追従する標的」にし、表現の質を伸ばしつつ崩壊も防ぎます。なぜ EMA で崩壊しないのかは完全には解明されていませんが、ターゲットが過去の自分なので「予測すべき的が安定して動く」ことが学習を支える、と理解されています。

## 手法の系譜と主要論文
この「特徴空間で予測し、ターゲットは別ネットワーク(凍結 or EMA)で作る」設計は、JEPA(Joint-Embedding Predictive Architecture)とその源流である自己教師あり学習の系譜です。

- モメンタムエンコーダの導入。MoCo(He et al., FAIR, CVPR 2020, arXiv:1911.05722)は、対照学習(似たペアを近づけ違うペアを遠ざける)で、片側のエンコーダを EMA で更新する「モメンタムエンコーダ」を導入し、ターゲット側を安定させることで大きな辞書(負例の集合)を使えるようにしました。EMA ターゲットという発想の重要な起点です。

- 崩壊回避の源流。BYOL(Grill et al., DeepMind, NeurIPS 2020, "Bootstrap Your Own Latent", arXiv:2006.07733)は、負例(対照学習で使う「違うペア」)を一切使わず、オンライン側とターゲット側(EMA 更新)の二経路 + 片側 stop-gradient だけで崩壊を防ぎ、高品質な表現を学べることを示しました。SimSiam(Chen & He, FAIR, CVPR 2021, arXiv:2011.10566)は、EMA すら不要で「stop-gradient だけ」で崩壊を防げることを実験的に明らかにし、停止勾配が崩壊回避の本質であると整理しました。これが凍結ターゲットの理論的な支えです。DINO(Caron et al., Meta AI, ICCV 2021, arXiv:2104.14294)も EMA ターゲット + centering/sharpening で崩壊を防ぎ、自己教師ありの ViT が物体の境界を教師なしで捉えることを示しました。

- ピクセル再構成型との対比。MAE(He et al., FAIR, CVPR 2022, "Masked Autoencoders", arXiv:2111.06377)は、画像の大部分(75%)を隠して残りからピクセルを再構成する自己教師あり学習で、強力な事前学習として広まりました。JEPA はこれと対比され、「ピクセルではなく特徴を予測する」点が違いです。

- I-JEPA(Assran et al., Meta AI, CVPR 2023, arXiv:2301.08243)。画像の一部を隠し、残りから隠した部分の「特徴」を予測する自己教師あり学習を提案しました。ターゲットは EMA 更新の target encoder が作り、予測は context encoder + predictor が行います。動機は「ピクセル再構成は無関係な細部に労力を割く」こと。効果は「意味的に有用な表現が、ピクセル再構成より少ない計算で速く学べる」ことです。

- V-JEPA(Bardes et al., Meta AI, 2024, arXiv:2404.08471)。I-JEPA を動画に拡張し、時間方向にマスクした領域の特徴を予測します。動画から世界の動態(物がどう動くか)を学ぶ点で、運転の未来予測と問題設定が極めて近い。動機は「ピクセル再構成は動画では特に無駄が多い」ことです。

- 運転の世界モデル。MILE(Hu et al., Wayve, NeurIPS 2022, arXiv:2210.07729)は、専門家の運転データから BEV の将来状態を予測する潜在世界モデルを学び、CARLA で計画に活かしました。これらは生成や潜在状態予測寄りで、JEPA 流の「凍結バックボーン特徴を MSE で合わせる」とはターゲットの作り方が異なりますが、「未来を予測して計画に活かす」という上位の目的を共有します。

系譜の流れ: (1) MoCo がモメンタムエンコーダを導入 → (2) BYOL/SimSiam が stop-gradient による崩壊回避を確立 → (3) I-JEPA が特徴空間予測を画像に適用 → (4) V-JEPA が動画(時間)に拡張 → (5) これを運転の BEV 未来予測に応用、という積み上げです。

## 論文の実験結果(定量データ)
- MoCo(2020)。ImageNet linear probe(凍結表現に線形分類器を乗せる評価)で当時の自己教師あり最先端に並び、検出・分割の転移でも教師あり事前学習を上回る場合があることを示しました。モメンタムエンコーダの実用性を裏づけました。

- BYOL(2020)。ImageNet(物体 1000 種の分類)で、自己教師ありで学んだ表現に線形分類器を乗せる評価(linear probe)で 74.3% の top-1 精度を達成し、当時の対照学習(SimCLR の約 69.3%)を大きく上回りました。重要なのは、負例を全く使わずにこの精度に届いた点で、stop-gradient + EMA ターゲットの有効性を示しました。top-1 精度は「最も確信した予測が正解だった割合」です。

- SimSiam(2021)。BYOL から EMA を外し、stop-gradient だけにしても ImageNet linear probe で同等水準(約 71%)を保てることを示しました。さらに、stop-gradient を外すと精度が一気に崩壊する(表現が無意味になる)ことを実験で示し、停止勾配が崩壊回避の鍵であると明確化しました。これがアブレーション(ある要素を抜くと何が起きるかの検証)として最も明快な例です。

- MAE(2022)。ImageNet で、ViT-Huge をファインチューニングして 87.8% の精度を達成しました。75% という高いマスク率でも(むしろ高マスクのほうが)強い表現が学べることを示し、再構成型の有効性を確立しました。

- I-JEPA(2023)。ImageNet linear probe で MAE 等のピクセル再構成型を上回りつつ、学習に要する計算(GPU 時間)が少ないことを報告しました。ViT-Huge を 16 ノードで約 72 時間など、再構成型より効率的に強い表現に到達できることを示しています。

- V-JEPA(2024)。Kinetics-400(動画の行動分類)や Something-Something-v2(物体操作の細かい違いの認識)で、凍結した特徴に軽い分類器を乗せる評価で動画版 MAE 等を上回りました。とくに Something-Something-v2 のような「動きの細部」を要する課題で優位が大きく、特徴空間予測が動態の把握に向くことを定量的に裏づけました。

これらの数値は、(a) ピクセルより特徴を予測したほうが効率よく強い表現が学べること、(b) stop-gradient(凍結)が崩壊回避に必須であること、を一貫して示しています。運転の未来特徴予測も同じ原理の上に立っています。

## メリット・トレードオフ・限界
メリット
- ピクセルの細部に惑わされず、運転に関係する抽象的なシーン動態を学べます。
- ターゲットを凍結することで自明解への崩壊を防げます(理論的にも BYOL/SimSiam で裏づけ)。
- ターゲットは推論時には不要(学習専用)なので、デプロイ時の計算は増えません。

トレードオフと限界
- 学習時に「もう 1 つのバックボーン分」のメモリと計算が増えます(未来フレームの特徴抽出)。
- 予測特徴とターゲット特徴の次元・解像度を一致させる必要があり、合わないと射影層や補間が要ります。
- 完全凍結だと学習が進んでも正解の質が上がらず、EMA 更新版に比べ表現が伸びにくい可能性があります。逆に EMA はモメンタム m の調整がシビアで、不適切だと崩壊や学習停滞を招きます。
- 時刻整合した未来フレーム(何秒後の実画像かを正確に対応づけたデータ)が必要で、データ整備のコストがかかります。
- ターゲット特徴に運転と無関係な情報(空の色、建物の模様)が大量に含まれると、予測がそれに労力を割いてしまう。特徴空間でも「何を捨て何を残すか」はターゲット生成器の性質に依存します。
- 未解決課題として、(a) どのバックボーンをターゲットに選ぶと運転に有用な特徴になるか、(b) 完全凍結 vs EMA のどちらが運転で良いか、(c) ターゲット特徴に含まれる無関係情報をどう抑えるか、が挙げられます。

## 発展トピック・研究の最前線
- EMA ターゲット(モメンタムエンコーダ)の運転への適用。完全凍結より表現が伸びる可能性があり、崩壊と質のバランスを取る研究。
- マルチスケール・マルチモーダルなターゲット。LiDAR や地図も含めた特徴をターゲットにし、視覚以外の動態も学ばせる方向。
- 予測器の表現力強化。単純な畳み込みではなく Transformer 系の predictor を使い、長期・広域の動態を捉える研究(V-JEPA の predictor 設計が参考)。
- ターゲットの自己回帰生成。1 ステップ先のターゲットを使って次のターゲットを作る連鎖により、長期ロールアウトの学習信号を作る試み。
- 崩壊の理論解明。なぜ stop-gradient と EMA で崩壊が防げるのか(次元崩壊の回避を含む)を厳密に説明する研究は今も進行中です。
