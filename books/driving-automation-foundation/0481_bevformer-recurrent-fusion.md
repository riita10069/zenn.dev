# BEVFormer型 再帰的BEV融合

## ひとことで言うと

前フレームで作ったBEV（真上から見た平面図、Bird's-Eye-View）の特徴マップをそのまま保存しておき、自車の動いたぶんだけ座標をずらして現在に重ね、「時間方向の self-attention（自己注意）」で現在のBEVクエリに混ぜ込む手法です。1つ前のBEVを使い、そのBEVは更にその前を含む、という形で過去が再帰的に伝わります。複数カメラから明示的な深度推定なしにBEV表現を作る代表的方式で、検出・地図・予測・計画を統合した大型モデル（UniAD など）の基盤としても採用されています。

## 直感的な理解

カメラ映像から「真上から見た意味地図（BEV）」を作るとき、1フレームだけでは分からない情報がたくさんあります。代表が物の速度と、いったん隠れた物の存在です。BEVFormer（Li et al., ECCV 2022）はこの問題に対し、「前フレームのBEV特徴マップを丸ごと覚えておき、現在のBEVを作るときの材料として使う」という解き方を提案しました。

ポイントは「何を覚えるか」です。素朴には過去の生画像を覚える手もありますが、それでは毎フレームでゼロから処理し直しになって重い。BEVFormerは「すでに処理済みのBEV特徴マップ」を覚えます。BEV特徴は、シーンを構造化した中間表現（たとえば縦200×横200マス、各マス256次元のテンソル）です。これを次フレームの計算に再利用すれば、過去の処理結果を無駄にせず積み上げられます。これが「再帰的（recurrent）」と呼ばれる理由で、各フレームの計算が1つ前のフレームの結果に依存して連鎖します。リレー走者が前の走者からバトンを受け取って走り続けるイメージです。1つ前のBEVには、さらにその前のBEVの情報が（薄まりながら）含まれているので、明示的に多数フレームを保持しなくても、原理的には過去全体の文脈が尾を引いて伝わります。

## 基礎: 前提となる概念

用語を噛み砕きます。

- BEVクエリ（BEV query）: BEV格子の各マスに割り当てた「問い合わせベクトル」。「このマスは何を見るべきか」という質問役で、学習可能なパラメータとして初期化されます。各マスごとに1本のベクトル（たとえば256次元）を持ち、全体で `[H, W, C]` の格子になります。
- self-attention（自己注意）: 複数の要素が互いに「どれをどれだけ参照するか」を学習で重み付けし、情報を混ぜる仕組み。Transformerの中核です。Query・Key・Value という3つの役割に各要素を写像し、QとKの類似度で重みを決め、Valueを加重平均します。
- cross-attention（交差注意）: 一方の集合（BEVクエリ）が別の集合（カメラ画像特徴）を参照する注意。BEVクエリがカメラ画像から情報を引っ張ってくるのに使います。
- 時間方向 self-attention（Temporal Self-Attention, TSA）: 参照先を「現在のBEV」と「整合済みの過去のBEV」の両方に広げた self-attention。
- deformable attention（変形可能注意）: 画像の全ピクセルを参照する代わりに、クエリ周辺の少数（たとえば各クエリ4〜8点）の「サンプリング点」だけを、位置を学習的にずらしながら参照する効率的な注意機構（Deformable DETR, Zhu et al., ICLR 2021 が起源）。通常の attention は要素数の2乗の計算量だが、deformable は要素数に比例する線形コストで済む。BEVは格子が大きい（200×200=4万マス）ので、全点参照だと計算が爆発します。これを抑えるのに使います。
- ego-motion compensation（自車運動補償）: 自車が動いたぶん、過去BEVの座標をずらして現在に合わせること。
- pillar（柱）: BEVの1マスを、地面から上空へ伸びる縦の柱とみなしたもの。1マスに対し高さ方向に数点を取り、それを3D点として扱います。

## 仕組みを詳しく

処理をステップで分解します。

1. 過去BEVの保持: 前フレームのBEV特徴 `B_{t-1}` を記憶します。形状は例として `[H, W, C]`（たとえば `[200, 200, 256]`）。

2. 自車運動補償: 自車は0.5秒の間に前進・回転しています。前フレームの座標系で描かれた `B_{t-1}` を、現在の自車中心座標へ座標変換します。具体的には、現在のBEVマスの実世界座標（メートル単位）を求め、自車の移動・回転の逆変換を掛けて過去座標へ移し、過去BEVの対応位置を双線形補間（`grid_sample` 等）で取り出します。

   ```
   t-1の自車       tの自車
     ┌─┐    前進+回転    ┌─┐
     │車│  ──────────>  │車│
     └─┘                └─┘
   B_{t-1}を「tの自車中心」に座標変換してから重ねる
   ```

3. 時間方向 self-attention（TSA）: 現在のBEVクエリ `Q_t` が、(a) 整合済みの過去BEV `B'_{t-1}` と (b) 現在の暫定BEV の両方を、deformable attention で参照します。これにより「このマスの物は前フレームではここにいた」という対応を学べます。動く物体は位置がずれているので、deformable attention のサンプリング点が学習的にそのずれ（オフセット）を追いかけます。整合済みでも動く物体だけはずれているので、TSA はその残差を捉えて速度の手がかりにします。

4. 空間 cross-attention（SCA）と空間 self-attention: BEVFormerは時間だけでなく、各BEVマスを3D空間の柱（pillar）に対応づけ、その柱の各高さの3D点を、外部・内部カメラパラメータを使って複数カメラ画像へ投影し、投影先の画像特徴を deformable cross-attention で集約します。1つの柱が複数カメラに跨って写る場合は、写ったカメラからの寄与を平均します。さらにBEVマス間の空間関係も self-attention で混ぜ、最終的に時空間トランスフォーマーとして1つのBEVを出力します。

5. 再帰連鎖: 出力した `B_t` を次フレームの「過去BEV」として保存し、フレームが進むたびに繰り返します。過去はぼやけながらも長く尾を引いて伝わります。

数値例（速度推定）: 前車が0.5秒で1.0m前進。`B_{t-1}` を整合して重ねると、TSAは「前フレームのこのマスの物体が、現在は2マス（解像度0.5m/マスなら1.0m）ぶん前に移った」という対応に高い注意重みを与えます。これで「秒速2m前進」が読めます。

なぜ深度を明示推定しないのか、という設計思想にも触れておきます。LSS（Lift-Splat-Shoot）系は各ピクセルの深度分布を予測してから3Dに持ち上げますが、深度推定が曖昧だとBEVが歪みます（前方への伸び縮みが起きる）。BEVFormerは「BEVマス→3D柱→カメラ投影→画像特徴サンプリング」という逆向きの経路（BEVから画像を引きにいく pull 方式）を取り、注意機構に「どの画像特徴を信じるか」を学習で任せます。これにより深度推定の明示的な誤差伝播を避けています。この設計は DETR3D（Wang et al., CoRL 2021）の「3Dクエリを画像に投影してサンプリングする」発想を、密なBEV格子へ広げたものと位置づけられます。

## 手法の系譜と主要論文

- Deformable DETR（Zhu et al., ICLR 2021, arXiv:2010.04159）: BEVFormerが使う deformable attention の起源。DETR（物体検出Transformer）の収束の遅さと高解像度への弱さを、少数サンプリング点の注意で解決しました。BEVFormerはこれをBEV格子へ応用しています。

- DETR3D（Wang et al., CoRL 2021, arXiv:2110.06922）: 3Dの物体クエリを画像平面に投影して特徴をサンプリングする疎なカメラ3D検出器。BEVFormerの「BEVクエリを画像に投影して引く」発想の直接の先行研究です。

- BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）: 本手法の出典。提案は「空間 cross-attention で複数カメラをBEVに集約」＋「時間 self-attention で前フレームBEVを融合」する時空間トランスフォーマー。動機は明示的な深度推定なしに注意機構で幾何とダイナミクスを学ぶこと。新規性は密なBEVクエリと再帰的な時間 self-attention の組み合わせです。

- BEVFormer v2（Yang et al., CVPR 2023, arXiv:2211.10439）: 視点空間（perspective view）に補助の検出ヘッドを足して、現代の画像バックボーンをBEV検出に適応しやすくし、検出精度を底上げしました。BEVFormerの「バックボーンの恩恵を受けにくい」弱点への改良です。

- UniAD（Hu et al., CVPR 2023, arXiv:2212.10156）: 検出・追跡・地図・予測・計画を1つに統合した「Planning-oriented」な大型モデル。BEVエンコーダとしてBEVFormerをそのまま採用しました。動機は、統合パイプラインの土台に実績ある時間融合BEVが必要だったこと。CVPR 2023 Best Paper。トレードオフは全体が巨大で学習・推論コストが大きいこと。

- VideoBEV（Han et al., 2023, arXiv:2303.05970）: BEVFormer型の「1フレーム再帰」は過去情報が世代を経るごとに薄れる、という弱点を指摘し、長期保持の再帰機構を提案しました。BEVFormer型の限界を補う位置づけです。

共通の論点は「フル特徴マップを覚える表現力 vs 保存コスト」というトレードオフ、そして「1フレーム再帰の情報減衰をどう補うか」です。

## 論文の実験結果（定量データ）

- BEVFormer（ECCV 2022）: nuScenes test セットの3D検出で NDS（nuScenes Detection Score、検出精度・位置・サイズ・向き・速度・属性誤差を総合した0〜1の指標、高いほど良い）56.9%、mAP（mean Average Precision、平均適合率）48.1%を達成し、当時のカメラのみ手法で最高水準でした。地図セグメンテーションでも高いIoU（予測領域と正解領域の重なり率）を報告しています。

  最も重要なアブレーションが時間 self-attention（TSA）の効果です。TSAを外すと、速度推定誤差 mAVE（mean Absolute Velocity Error、検出物体の速度ベクトルの平均誤差 m/s、低いほど良い）が顕著に悪化したと報告されています。これは「速度は時間情報なしには原理的に推定できない」ことの直接的な裏づけです。さらに、隠れた物体の検出再現率（recall）も時間融合ありのほうが高く、オクルージョン耐性が定量的に示されました。低照度・夜間・雨天など難条件でのロバスト性向上も報告されています。空間 cross-attention の参照点数（高さ方向のサンプル数）を減らすと検出が劣化することも示され、複数カメラ集約の設計が効いていることが分かります。

- BEVFormer v2（CVPR 2023）: 視点空間の補助監督を加えることで、強力な画像バックボーン（大規模事前学習済み）をBEV検出に転用でき、NDSをさらに数ポイント押し上げたと報告されています。

- UniAD（CVPR 2023）: BEVFormerを基盤に全タスクを統合し、最終指標である「計画の衝突率（collision rate、低いほど良い）」と「経路逸脱誤差（自車の予測軌跡と正解軌跡のL2距離、低いほど良い）」を、タスクを別々に学習する従来パイプラインより改善しました。報告では1秒・2秒・3秒先の平均L2誤差と平均衝突率の両方で当時のベースラインを上回り、「タスクを統合し共通のBEV時間特徴を全段で共有することが計画性能に効く」ことを示しました。中間タスク（追跡・地図・動き予測）の出力を後段に明示的に渡す設計が、純粋なend-to-endより安全性指標を改善した点が要点です。

- VideoBEV（2023）: 長期再帰により、BEVFormer型の単純1フレーム再帰に対して検出・速度推定の両方で改善を報告し、特に長い時間スパンを要する状況（長い遮蔽からの復帰など）での利得を示しました。

定量的な核心は、「時間 self-attention の有無で mAVE が大きく変わり、それがNDS全体を押し上げる」という点です。NDSはmAVEを構成要素に含むため、速度推定の改善が総合スコアに直結します。

## メリット・トレードオフ・限界

メリット:
- フル解像度のBEV特徴マップを覚えるので表現力が高い。動き・隠れ履歴を細かく扱える。
- 複数カメラを注意機構でBEVに集約するため、明示的な深度推定の誤差伝播を避けられる。
- 密なBEVを出力するので、検出・セグ・地図・占有など多様な下流タスクに同じ特徴を流用できる。UniAD等で実績があり、下流タスク統合の基盤として確立している。

トレードオフと限界:
- 過去BEV特徴マップ全体を保持するメモリコスト。解像度が高いほど重い。
- 自車運動データ（オドメトリ）が必須。整合を誤ると静止物まで動いて見え逆効果。
- deformable attention 等の実装が複雑で、CUDA カスタムカーネルを使うことも多く、移植や量子化が難しい。
- 1フレーム再帰なので、世代を経ると古い情報が薄れる（長期文脈に弱く、VideoBEVが補う論点）。学習時の系列長と推論時のストリーム長が違うことによる挙動差にも注意が要る。
- 計算量が大きく、車載のリアルタイム推論にはモデル縮小や蒸留が要ることが多い。
- 未解決の課題: 過去BEVのどこをどれだけ信用するかの不確実性扱い、誤った過去情報の自己訂正、長期記憶の効率的保持は依然として研究途上です。疎なクエリ伝播型（StreamPETR 系）に対する効率面の劣位も指摘されています。

## 発展トピック・研究の最前線

- 長期メモリ化: VideoBEV のように temporal embedding で多数フレームを保持し、1フレーム再帰の情報減衰を緩和する方向。
- 疎な表現への移行: 密なBEVグリッド全体を保持するコストを避け、物体クエリの伝播（StreamPETR / Sparse4D 系）で時間を扱う流れ。BEVFormer型の重さへの回答です。[BEVDet4D型 特徴キュー連結](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0480_bevdet4d-queue-concat) の軽量化路線とも対比されます。
- 計画指向の統合最適化: UniAD系のように、BEV時間特徴を検出・予測・計画の共通基盤とし、計画性能を最終目的として全体を学習する流れ。
- 4D Occupancy への拡張: BEVを高さ方向にも広げ、時間軸を加えた4D占有予測へ展開する研究。
- バックボーン適応: BEVFormer v2 のように視点空間の補助監督で大規模事前学習済みバックボーンを取り込む方向。
