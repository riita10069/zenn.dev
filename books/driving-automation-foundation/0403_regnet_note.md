# RegNet

## ひとことで言うと
RegNet は、画像から特徴を取り出すニューラルネットワークの設計法であり、その設計法から生まれたモデル群の名前です。普通は「1つの良いネットワークを探す」のですが、RegNet は「良いネットワークが集まっている領域(=設計空間)そのものを探して良くする」という発想をとりました。その結果、ネットワークの各ブロックの幅(チャネル数)を、層が深くなるにつれてどう増やすべきかが「量子化された線形則」という驚くほど単純な数式で表せると発見し、規則的で拡張しやすく、GPU 実機で速いモデル群を作り出しました。

## 直感的な理解
ニューラルネットの構造設計には2つの極があります。1つは人間が経験と勘で1つずつ設計する手作り(VGG・ResNet などはこちら)。もう1つは NAS(自動探索)で、巨大な候補集合の中から最も良い1つを機械に探させる方法です。NAS は強力ですが、見つかる構造はしばしば不規則で複雑、なぜ良いのか人間に説明できず、計算予算を変えたいとき(軽くしたい・重くしたい)にきれいに伸び縮みしません。たとえば「このモデルを半分の計算量にしたい」と思っても、NAS では予算ごとに探索をやり直すことになりがちです。

RegNet はその中間とも違う第3の道を示しました。「1つの最良モデルを探す」のではなく「良いモデルが密集している設計空間を見つけ、その空間を支配する単純な規則を取り出す」のです。たとえるなら、優秀な選手を1人スカウトするのではなく、「優秀な選手が育ちやすい育成環境(空間)とは何か」を統計的に解明し、その環境のレシピを手に入れるようなものです。レシピさえ分かれば、どんな予算でも(軽い選手も重い選手も)同じ規則で量産できます。

## 基礎: 前提となる概念
理解に必要な用語を順に説明します。

設計空間(design space)とは、無数のネットワークを生み出す「パラメータの取りうる範囲の集合」です。例えば「層数は 12〜28、各ステージの幅は 16〜1024、ボトルネック比は 1〜3、グループ幅は 8〜32」のように範囲を決めると、その範囲の組み合わせ全体が1つの設計空間になります。空間の中の1点を選ぶと、具体的な1つのネットワークが定まります。設計空間は「点」ではなく「集合」である点が肝です。

NAS(Neural Architecture Search)とは、設計空間の中から最良の1点(=1つの構造)を自動で探す技術です。RegNet は NAS を否定するのではなく「探す対象を、点(1モデル)から空間(モデルの集合)へ引き上げた」と理解すると分かりやすいです。

幅(width)とはチャネル数、すなわち特徴マップの層の枚数です。深さ(depth)とは層(ブロック)の数です。ボトルネック比(bottleneck ratio)とは、ブロック内でチャネルをいったん絞る/広げる倍率で、グループ幅(group width)とはグループ畳み込みで1グループが受け持つチャネル数です。RegNet はこうした少数の量(自由度)だけでネットワーク全体を記述します。ネットワークは「stem(入力直後の小さな前処理)→ 4 つの stage(各 stage 内は同じ幅のブロックの繰り返し)→ head(分類層)」という ResNet 風の標準骨格を前提にします。

量子化(quantization)とは、連続的な値をきりの良い離散値に丸めることです。例えば 63.7 を 64 に丸めるような操作で、後述する「線形に増える幅」を実際に作れる整数値へ落とし込むのに使います。

誤差分布(error distribution)という概念も鍵です。RegNet は「1つのモデルの精度」ではなく「ある設計空間から多数サンプリングして学習したモデル群の、誤り率の分布(ばらつき方)」を見ます。空間 A の分布が空間 B より全体的に左(低誤差側)に寄っていれば「空間 A のほうが良い空間」と判断します。これを可視化したものが EDF(Error empirical Distribution Function、誤差の経験分布関数)で、横軸が誤り率、縦軸が「その誤り率以下のモデルの割合」を表す累積曲線です。良い空間ほど曲線が左上に寄ります。

## 仕組みを詳しく
RegNet の作り方は「広い設計空間を、統計的な観察に基づいて段階的に絞り込む」プロセスです。

1. 広い初期空間(AnyNet と呼ぶ。ステージ数・各ステージの幅と深さ・ボトルネック比・グループ幅が自由)から、ランダムにネットワークを数百個(論文では各段階 500 モデル)サンプリングし、それぞれを短時間学習する(各 10 エポック程度の低コスト学習)。
2. それらの誤り率の EDF を統計的に観察する。
3. 「ある制約を1つ加えると、分布全体が良い方向(低誤差側)へ動く」なら、その制約を採用して空間を狭める。
4. これを繰り返し、最終的にシンプルな規則だけが残った良い空間 = RegNet を得る。

この絞り込みで核心となった発見が量子化線形則です。良いネットワークでは、各ブロックの幅が、ブロック番号 j に対して直線的に増える、というものです。式で書くと、まず連続的な幅を

u_j = w_0 + w_a × j

で与えます。ここで j はブロック番号(0, 1, 2, …)、w_0 は初期幅(切片)、w_a は幅の増え方(傾き)です。この u_j を、追加パラメータ w_m(量子化の刻みを決める)を使って u_j = w_0 · w_m^(s_j) の形に当てはめ直し、指数 s_j を四捨五入してきりの良い段階値へ量子化し、実際のチャネル数にします。つまり w_0、w_a、w_m、深さ d の4つ程度のパラメータだけで、全ブロックの幅が一意に決まります。

```
ブロック番号 j     : 0    1    2    3    4    5  ...
連続幅 w_0 + w_a×j : 直線で単調増加
   ↓ w_m で量子化(きりの良い段階値へ丸める)
実際の幅(チャネル): 32 → 64 → 64 → 160 → 160 → 400 ...
(わずか数個のパラメータで全層の幅が決まる = 規則的でスケールしやすい)
```

量子化後に「同じ幅が連続するブロックのまとまり」が自然に stage になります。つまり stage の境界も手で決めるのではなく、線形則と量子化から自動的に生まれるのが美しい点です。この単純さの恩恵は大きく、計算予算を1つ決めれば(例えば「400 MFLOPs のモデルが欲しい」)、規則に従って w_0・w_a・w_m・深さを選ぶだけで、その予算に最適な構造が即座に得られます。NAS のように予算ごとに探索をやり直す必要がありません。

RegNet には主に2系統あります。RegNetX は素の構造、RegNetY はそこに Squeeze-and-Excitation(SE、チャネルごとの重要度を動的に学習する小さな仕組み。[Squeeze-and-Excitation (SENet)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1085_squeeze-and-excitation) 参照)を加えたものです。モデル名は計算量で表され、RegNetY-4.0GF なら「計算量 4.0 GFLOPs の RegNetY」を意味します。系列は 200MF(MFLOPs)程度の軽量から 32GF の大型まで連続的に用意されています。

絞り込みの過程で、当時の常識に反する意外な知見も得られました。良いネットワークの深さは計算予算によらずおおむね一定(約 20 ブロック、深さ d≈20)が良いこと、ボトルネック比は 1(=ボトルネックを使わない)が良いこと、グループ幅 w_g は予算が増えるとともに増やすべきこと、などです。「予算が増えたら深くすべき」という直感に反し、予算が増えても深さはあまり変えず、むしろ幅とグループ幅を増やすのが良い、というのが統計的に見えてきた規則でした。

## 手法の系譜と主要論文
直接の土台は Radosavovic ら「On Network Design Spaces for Visual Recognition」(ICCV 2019、Facebook AI Research、arXiv:1905.13214)です。これは設計空間どうしを「サンプリングしたモデル群の誤差分布(EDF)」として比較する方法論を確立しました。空間の良し悪しを単一モデルではなく分布で測る、という発想がここで生まれました。

本命は Radosavovic・Kosaraju・Girshick・He・Dollár「Designing Network Design Spaces」(CVPR 2020、Facebook AI Research、arXiv:2003.13678)です。提案は「設計空間を設計する(design design spaces)」という方法論と、その帰結としての RegNet(量子化線形則)です。動機は「NAS が生む不可解で非スケーラブルな単一モデルではなく、解釈可能で拡張しやすい規則が欲しい」でした。著者陣に Kaiming He(ResNet の主著者)と Piotr Dollár が含まれる点も、設計思想の系譜を象徴しています。SE モジュールは Hu らの Squeeze-and-Excitation Networks(CVPR 2018、arXiv:1709.01507)に由来し、RegNetY に取り込まれています。

その後、RegNet は「強く再現性が高く、実機が速いベースライン」として広く使われ、特に物体検出や自己教師あり学習の比較対象として定番化しました。設計思想の系譜としては、後の ConvNeXt(Liu et al., CVPR 2022)が「設計を統計や ViT の知見から見直す」流れを引き継いでいます。

## 論文の実験結果(定量データ)
評価は ImageNet(1000 クラス、約 128 万枚)の分類精度と、同じ計算量(FLOPs)での比較、そして GPU 上の実速度で行われました。

主要な報告は以下です。FLOPs(理論計算量、掛け算回数)を固定したときの精度比較が中心になります。

```
比較軸: 同じ FLOPs での top-1 精度 と GPU 実速度
- RegNetX/Y は、同じ FLOPs で当時の手作りモデル(ResNet/ResNeXt)を上回る精度。
- RegNetY は EfficientNet と同等以上の精度を、GPU 上で最大 5 倍速く達成したと報告。
  (EfficientNet の depthwise 中心構造が実機で遅いのに対し、RegNet は普通の畳み込み主体で速い)
- 200MF から 32GF まで、軽量〜大型まで連続的にスケールしても規則が崩れず精度効率を保つ。
```

具体的な数値例として、RegNetY-200MF クラスは MobileNet/ShuffleNet 帯の軽量モデルと競合し、RegNetY-8.0GF クラスは ImageNet top-1 で約 81〜82% に達します。ここで重要なのは「FLOPs が同じでも実速度は同じではない」点です。EfficientNet は FLOPs あたりの精度は高いものの、depthwise 畳み込みが GPU で計算密度が低く実機が遅い。RegNet は普通の畳み込み主体なので、同じ FLOPs でも実時間あたりの処理が速い。論文はこの実速度比較を強調し、EfficientNet に対し同精度で GPU 上最大約 5 倍速いと報告しました。FLOPs だけで効率を語る危うさを示した点でも意義があります([ShuffleNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0406_shufflenet) V2 が打ち立てた「実速度第一」の哲学と地続きです)。

アブレーションに相当する分析として、絞り込みの各段階で「制約を加えると誤差分布がどう動くか」が定量的に示されています。ボトルネック比を 1 に固定する制約、グループ幅を共有する制約、幅を線形則に従わせる制約を順に加えるたびに、EDF が左(低誤差)へ寄ることが確認されました。つまり「どの制約がどれだけ空間を改善したか」が見える形で提示されており、線形則が偶然ではなく統計的に裏づけられた規則であることが分かります。深さを増やしても約 20 ブロック以降は精度が頭打ちになる、という分析も同様の手法で示されました。さらに、低コスト学習(短時間)で良かった空間が、本番の長時間学習でも良いままであることを確認しており、絞り込みプロセスの妥当性を支えています。

## メリット・トレードオフ・限界
メリットは、幅が単純な線形則で決まるため構造が規則的で理解しやすいこと、計算予算を決めれば即座に最適構造が得られ軽量から大型まで連続的にスケールできること、普通の畳み込み主体で GPU 実機が速いこと、設計が単純で再現しやすくベースラインとして扱いやすいことです。物体検出や領域分割のバックボーンに差し替えやすく、研究の対照実験で「公平で速い基準線」として重宝されます。

トレードオフと限界もあります。第1に、設計空間の絞り込み自体には多数のネットワークを学習する計算コストがかかります(ただし一度規則が得られれば再利用でき、以後は探索不要)。第2に、得られた規則はあくまで「ImageNet 分類で良かった空間」を反映しており、他のタスクやドメイン、別のハードウェアで同じ規則が最適とは限りません。第3に、RegNet は精度の世界記録を狙う設計ではなく「効率と単純さのバランスを取る」思想なので、最高精度だけを求める場面では大型 Transformer 系などに譲ることがあります。第4に、量子化線形則は「ステージ内で幅を共有し、ステージ間で段階的に増やす」という単調増加・直列の前提に立つため、より自由な接続(分岐や多解像度融合、HRNet のような並列マルチスケール)を必要とするタスクには素直に当てはまらない場合があります。

## 発展トピック・研究の最前線
RegNet の「分布で空間を評価する」方法論は、構造探索だけでなく学習設定(学習率・データ拡張・最適化器)の良し悪しを統計的に比較する研究にも波及しました。設計を勘ではなく統計で語る、という文化的影響は大きいです。

近年は ConvNeXt が ViT の設計知見を CNN に持ち込み、規則的な CNN でも Transformer 級の精度を出せることを示しました。RegNet が確立した「規則的で実機が速い CNN」という路線は、ConvNeXt や RegNetZ といった後続に引き継がれています。また、RegNet は自己教師あり学習(SEER、Goyal et al., 2021, arXiv:2103.01988。10 億枚規模のラベルなし画像で事前学習する研究)のバックボーンとして RegNetY を最大数十億パラメータまで巨大化して採用され、規則的でスケールしやすい性質が大規模学習で活きること、線形則がそのまま超大型まで延長できることが示されました。実機レイテンシを設計指標に直接組み込むハードウェア対応設計の流れも、RegNet の「FLOPs より実速度」という主張と地続きで続いています。
