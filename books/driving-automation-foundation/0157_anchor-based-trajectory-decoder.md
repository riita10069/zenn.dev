# アンカーベース軌道デコーダ

## ひとことで言うと
車がこれから走る道筋(軌道)を予測するとき、白紙の状態からゼロを予測するのではなく、「あらかじめ用意しておいた典型的な道筋の見本」をたくさん持っておき、その中からどれが正しそうかを選び、選んだ見本を少しだけ微調整する、という考え方です。この「典型的な見本」をアンカー(anchor、錨/基準点という意味)と呼びます。難しい予測問題を「64択クイズ + 小さな修正」に分解することで、複数のありえる未来を同時に扱えるようにする、軌道予測・計画の基盤テクニックです。物体検出のアンカーボックスと同じ思想を、画像中の「枠」ではなく時系列の「道筋」に対して適用したものと理解すると見通しがよくなります。

## 直感的な理解
あなたが交差点に近づいた車の運転手だと想像してください。「この先どう進みますか」と聞かれたら、いきなり座標を1ミリ単位で答えるのは難しいですが、「だいたい直進です」「ゆるく右に曲がります」とは即答できます。そのうえで「直進だけど少し右寄りに」と微調整するのは簡単です。アンカーベースの発想はこれと同じで、最初に大まかな「方向(モード)」を選び、あとで細かいズレを足します。

なぜわざわざ2段階にするのか。理由は将来が一つに決まらないからです。交差点の車は直進も右折も左折もありえます。これらは全部「正しい答え」です。もし機械に「未来の軌道を1本だけ数値で出せ」と命じると、機械は学習データに入っている直進例・右折例・左折例の平均を取ろうとして、どこにも行かない不自然な軌道(直進と右折の中間=ガードレールに突っ込む軌道)を出してしまいます。これがアンカーベースが解こうとする核心の問題です。「答えが複数ある問題を、1つの平均で潰してはいけない」のです。

この「平均化してしまう」病は、単純な平均二乗誤差(予測と正解の差の二乗を最小化する標準的な学習目標)を素直に使うと必ず起こります。学習理論的に言えば、複数の正解 y₁, y₂, … に対して二乗誤差を最小化する1点は、それらの平均(期待値)だからです。直進と右折という相反する正解の期待値は「正面の壁」になってしまう。アンカーはこの数学的な落とし穴を、答えの空間をあらかじめ分割しておくことで回避します。

## 基礎: 前提となる概念
いくつか用語を最初に噛み砕いておきます。

軌道(trajectory)とは、時刻ごとの位置を並べたものです。たとえば6秒先まで0.5秒刻みで予測するなら、(x, y) 座標が12個並んだ列になり、tensor(数値を格子状に並べたデータ構造)の形(shape)で書くと `[12, 2]` です。12 が時間ステップ数、2 が x と y の2次元です。これに速度や向き(heading)を加えて `[12, 4]` などにすることもあります。

軌道予測(trajectory prediction)は周りの車や歩行者の未来軌道を当てるタスク、軌道計画(trajectory planning)は自分の車(ego, エゴ)がこれから取るべき軌道を決めるタスクです。アンカーベースの考え方は両方に使えます。両者の数学的な構造はよく似ていますが、計画の場合は「自分が選んだ軌道を実際に実行する」ぶん、安全制約(衝突しない・道をはみ出さない)の組み込みがより重要になります。

多峰性(multimodal、ピークが複数あるという意味)とは、ありえる答えが複数の山(モード)に分かれている状態です。確率分布で言えば、山が1つなら単峰(unimodal)、複数なら多峰です。交差点の例では「直進」「右折」「左折」の3つの山があります。

回帰(regression)は数値そのものを直接予測すること、分類(classification)はどのカテゴリに属するかを当てることです。アンカーベースは「どのアンカーか」を分類で、「アンカーからのズレ」を回帰で扱う、両者のハイブリッドです。

モード平均化(mode averaging、mode collapse とも近縁)は、複数の正解の平均を取ってしまい、どれでもない中途半端な答えになる現象です。単純な回帰の最大の落とし穴で、自動運転では致命的です(直進と右折の平均は安全な軌道ではありません)。

クラスタリング(clustering)は、似たデータ同士をグループに分け、各グループの代表点を求める手法です。アンカーを作るのに使う k-means は、データを K 個のグループに分け各グループの重心を代表とする最も基本的なクラスタリングです。

## 仕組みを詳しく
処理を3ステップに分解します。

ステップ1: アンカー集合を用意する。
事前に「ありえる典型的な軌道」を K 本用意します(たとえば K=64 本)。作り方は、学習データに含まれる大量の本物の軌道を集めてきて、k-means クラスタリングで代表的なパターンに圧縮します。直進・ゆるい右カーブ・急な左折・減速して停止…といった代表的な道筋が64本そろうイメージです。アンカーは固定で、推論のたびに作り直しません。1本のアンカーの shape は軌道と同じ `[12, 2]`、64本まとめると `[64, 12, 2]` です。

```
アンカー集合の例 (上から見た図、原点が自車)
              直進アンカー
               |
   左折アンカー  |  右折アンカー
        \       |      /
         \      |     /
   ───────●─────────────  ← 自車の現在位置
         /      |     \
   急右折       減速停止   急左折
```

ステップ2: 各アンカーの「確率」を分類で出す。
入力(カメラ画像や地図、車の現在状態など)をニューラルネットワークに通して特徴ベクトルを作り、「K本のアンカーのうちどれが正解に近いか」を分類で予測します。出力は K 個の確率で、shape は `[64]`、softmax(全要素を合計1の確率に変換する関数)を通すので合計1になります。たとえば「直進アンカー 0.7、ゆるい右折 0.2、…」のように出ます。この確率分布こそが多峰性の表現です。山が複数あれば、複数のアンカーに確率が分散します。

ステップ3: 選ばれたアンカーからの「ズレ」を回帰で出す。
アンカーはあくまで典型例なので実際の軌道とは少しズレます。そこで各アンカーごとに「このアンカーを基準にすると、実際の各時刻の位置はどれだけズレているか(残差、residual = 差分)」を回帰で予測します。最終的な予測軌道は次の式です。

```
予測軌道 = アンカー(基準の道筋) + 残差(微調整のズレ)
        anchor_k[t]          + Δ_k[t]    （各アンカー k、各時刻 t）
```

たとえば直進アンカーが選ばれ、残差が「全体的にやや右へ0.3mずつ」なら、最終予測は「ほぼ直進だが少し右寄り」になります。

MultiPath はさらに各時刻の位置を1点ではなく2次元ガウス分布(平均ベクトル μ と共分散行列 Σ を持つ釣鐘型の確率分布)で出します。つまり1本のアンカーに対して「平均の軌道 + 各点の不確かさの楕円」を返す。これにより「自信のある直線部分は楕円が小さく、遠い未来や分岐前は楕円が大きい」という、予測の自信のなさまで表現できます。学習はこのガウスの負の対数尤度(predicted distribution が正解をどれだけ高い確率で説明できるか)を最小化します。

なぜこれがうまくいくのか。難しい「ゼロから多峰の軌道を作る」問題を、「64択の分類(どの大まかな方向か)」と「すでに正解の近くにいる状態からの小さなズレの回帰」に分けたからです。回帰が担当するのは小さなズレだけなので、平均化される範囲が1つの山の内側に限定され、モード平均化が起きにくく、学習も安定します。

これは物体検出(画像のどこに何があるか当てるタスク)で使われるアンカーボックスと全く同じ思想です。検出では「典型的な大きさ・形の枠(アンカーボックス)」を画像にばらまき、「どの枠に物体があるか(分類)」と「その枠からの位置・サイズのズレ(回帰)」を出します。軌道版はこの「枠」を「道筋」に置き換えたものだと考えると理解しやすいです(Faster R-CNN や SSD の発想の軌道への移植)。

学習時の重要な工夫として、損失計算では「正解の軌道に一番近いアンカー」を1本選び、そのアンカーに対してだけ残差回帰の損失をかけます(残りのアンカーには分類損失だけ)。これを hard assignment(ハード割り当て)と呼びます。これにより各アンカーが自分の担当する方向の予測に専念でき、責任が分散しません。もし全アンカーに回帰損失をかけると、結局すべてのアンカーが平均的な軌道に引っ張られてモード平均化が復活してしまいます。この「正解に最も近い1本だけを罰する/育てる」学習方式は winner-takes-all(勝者総取り)損失とも呼ばれ、混合密度ネットワーク(MDN)やマルチハイポセシス学習の系譜とも数学的につながっています。

## 手法の系譜と主要論文
アンカー思想は2019年前後に複数の研究グループから同時多発的に提案され、互いに改良し合って発展しました。

MultiPath (Chai, Sapp, Bansal, Anguelov, Waymo, CoRL 2019, arXiv:1910.05449)
アンカーベース軌道予測を世に広めた代表論文です。上で説明した通り「学習データから k-means で抽出した固定アンカー集合を用意し、各アンカーの確率を分類で、アンカーからのオフセットを回帰で出す」枠組みを確立しました。さらに各時刻の位置をガウス分布で出すことで予測の不確かさも表現しました。動機は、当時主流だった「1本だけ回帰」や「占有グリッドを画像として出力する」手法がモード平均化や計算コストの問題を抱えていたことです。

CoverNet (Phan-Minh, Howington, Chen, et al., nuTonomy/Aptiv, CVPR 2020, arXiv:1911.10298)
アンカーを「動的にカバーする軌道集合」として定式化し、軌道予測を純粋な分類問題に落とし込みました。MultiPath が残差回帰を残したのに対し、CoverNet は十分細かい軌道集合を用意すれば回帰なしの分類だけでも実用精度が出ることを示し、「離散化 + 分類」という路線を明確にしました。さらに物理的に到達不能な軌道を除外する dynamic set という工夫も導入しています。詳細は [CoverNet 軌道分類](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0163_covernet-classification-planning) を参照してください。

TNT / DenseTNT (Zhao et al., CoRL 2020 / Gu, Sun, Zhao, ICCV 2021, arXiv:2108.09640)
アンカーを「軌道全体」ではなく「目標地点(target/goal)」として置く発想です。まず数十〜数百の候補ゴール点に確率を付け、選ばれたゴールへ至る軌道を生成します。ゴールという1点に離散化することで、軌道全体を語彙化するよりカバレッジが上がりました。DenseTNT はゴール候補を密に置きアンカーを実質不要にした点で、後述の latent anchor 路線への橋渡しになりました。

MultiPath++ (Varadarajan, Hefny, Srivastava, et al., Waymo, ICRA 2022, arXiv:2111.14973)
MultiPath の発展版です。入力をラスタ画像(地図を画像として描いたもの)ではなくベクトル(道路中心線や他車軌跡を点列・折れ線で表したもの)で表し、軌道出力も「固定アンカー + 残差」から「学習で求める latent anchor(潜在アンカー、明示的に道筋として持つのではなく学習で内部的に獲得する基準)」へ一般化しました。動機は固定アンカーの表現力の限界を超えることと、ラスタ処理の重さを避けることです。

MTR / Motion TRansformer (Shi, Jiang, Dai, Schiele, NeurIPS 2022, arXiv:2209.13508)
アンカーを Transformer の「意図クエリ(intention query)」として持ち、デコーダ層を重ねて軌道を反復的に精緻化する路線を確立しました。各クエリが1つのモードに責任を持ち、層を経るごとにアンカーを動的に更新していく。固定アンカーの硬直性と、エンドツーエンドの注意機構による文脈適応を両立させた点で、現代の軌道予測デコーダのデファクトになりました。

これら「アンカー + 残差」「アンカー + 分類」「アンカー as クエリ」の思想は、その後のベクトル化計画([VAD ベクトル化シーン+プランニング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0210_vad-vectorized-planning) / [VADv2 確率的プランニング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0211_vadv2-probabilistic-planning))や拡散ベース計画([DiffusionDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0171_diffusiondrive))など、現代のエンドツーエンド計画手法の中に形を変えて受け継がれています。

## 論文の実験結果(定量データ)
評価指標をまず説明します。minADE(minimum Average Displacement Error)は、モデルが出した複数候補のうち正解に最も近い1本について、各時刻の予測位置と正解位置の距離を平均したもの(単位はメートル)です。複数候補のうちベストを採るので「少なくとも1本は当たっているか」を測ります。minFDE(minimum Final Displacement Error)は同様にベスト候補の最終時刻だけの誤差で、長期予測のズレに敏感です。Miss Rate は、ベスト候補でも最終位置が一定距離(たとえば2m)以上外れた割合で、低いほど良いです。Waymo の評価では mAP(mean Average Precision、各モードの確率付き予測がどれだけ正確に当たったかを精度・再現率の観点で測る)も使われます。いずれの誤差系も値が小さいほど、mAP は大きいほど優秀です。

MultiPath は nuScenes / Waymo の社内評価で、当時主流の単一回帰や CVAE(条件付き変分オートエンコーダ)ベース手法に対し minADE / minFDE を明確に下げ、特に多峰性が問われる交差点シーンでモード平均化を回避できることを示しました。論文の核心的なアブレーションは「アンカーを使わず単一回帰にすると、交差点で予測が直進方向へ平均化し誤差が跳ね上がる」という比較で、アンカーの効果を端的に示しています。

MultiPath++ は Waymo Open Motion Dataset(WOMD)で当時の公開リーダーボードを大きく更新しました。報告では、ラスタ入力からベクトル入力 + latent anchor へ切り替えることで minADE / minFDE がベースライン比で二桁パーセント改善し、アンサンブルと組み合わせて当時の WOMD で首位を獲得しました。アブレーションでは(1)入力のベクトル化、(2)multi-context gating(エージェント間の相互作用を選択的に取り込むゲート機構)、(3)latent anchor、のそれぞれが独立に効くことが示され、特に latent anchor は固定アンカーより少ない候補数で同等以上のカバレッジを達成しました。これは「アンカーを学習で獲得させると、人手で k-means する固定アンカーより表現効率が上がる」ことの定量的な裏付けです。

CoverNet は nuScenes 予測チャレンジで、軌道集合のサイズ(離散化の細かさ)を上げると Miss Rate が単調に下がること、ただし集合が大きくなりすぎると分類が難しくなり頭打ちになることを示しました。これは「離散化誤差とスコアリング難度のトレードオフ」を定量的に可視化した重要な結果で、後の VADv2 の大語彙設計(数千本)へつながります。

MTR は WOMD のモーション予測リーダーボードで当時の SOTA を更新し、特に soft mAP と minFDE で先行手法を上回りました。意図クエリ + 反復精緻化のアブレーションでは、クエリ数(モード数)を固定したまま精緻化の層を深くするだけで minFDE が改善し、「アンカーを1回選んで終わりにせず、文脈を見ながら更新し続ける」ことの効果が示されました。

## メリット・トレードオフ・限界
メリット
- 多峰性(直進/右折/左折など複数の未来)をきれいに表現でき、モード平均化を防げる。
- 「すでに正解の近く」から微調整するので回帰が安定し、学習が速く収束する。
- 各アンカーの確率が出るので「どの選択肢にどれだけ自信があるか」が分かり、解釈しやすい。下流の安全選択(衝突しない候補を選ぶ等)も組みやすい。
- 物体検出で実績のあるアンカー思想の流用で、設計ノウハウが豊富。

トレードオフと限界
- アンカーが固定だと、学習データに無い珍しい操作(変則的な切り返しなど)を表現しにくい(カバレッジの限界)。残差で吸収できる範囲を超えると精度が落ちる。
- アンカー本数 K の選び方が性能に効くハイパーパラメータ。少なすぎるとカバレッジ不足、多すぎると分類が難しくなる(CoverNet が示した頭打ち)。
- アンカーがデータ分布(地域・道路形状・速度域)に依存するため、別環境では作り直しが必要なことがある。高速道路用アンカーは都市部で機能しない。
- 学習時の hard assignment は「最近傍アンカー」の選択に敏感で、正解がちょうど2つのアンカーの中間にあると割り当てが不安定になる(境界問題)。学習初期にアンカーへの割り当てが偏ると、一部のアンカーが一度も選ばれず「死んだモード」になることもある。
- 未解決課題として、アンカー集合をシーンに応じて動的に変える(context-dependent anchors)、固定アンカーと latent anchor の長所をどう統合するか、が現在も研究テーマです。

## 発展トピック・研究の最前線
固定アンカーの硬直性を超える方向として、latent anchor(MultiPath++)、ゴールベース(DenseTNT)、そしてアンカーを「学習可能なクエリ」として持つ Transformer デコーダ(MTR)が主流になりました。MTR はアンカーを意図クエリとして Transformer に与え、反復的に軌道を精緻化することで WOMD で大幅な精度向上を達成しました。後続の MTR++ や QCNet(query-centric)では、アンカーを座標系に依存しない形で持ち、シーン全体を一度の符号化で多エージェント分予測する効率化も進んでいます。

さらに、離散アンカーに頼らず連続空間で多峰性を扱う方向として、拡散モデルを軌道生成に使う [DiffusionDrive](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0171_diffusiondrive) が登場しています。これは「アンカー周りのノイズから出発して軌道を生成する」もので、固定アンカーと連続生成のハイブリッドと見なせます。アンカーは消えるのではなく、生成モデルの初期化や条件付けの形で生き続けている、というのが現在の構図です。計画側では、語彙(大規模アンカー集合)上の確率分布を学ぶ [VADv2 確率的プランニング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0211_vadv2-probabilistic-planning) が、アンカー思想を「数千本規模」へスケールさせた到達点になっています。
