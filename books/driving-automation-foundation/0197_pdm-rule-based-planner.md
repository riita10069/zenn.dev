# PDM（ルールベース予測駆動プランナー）

## ひとことで言うと
深層学習を使わず、昔ながらの交通ルール（車間距離を保つ追従モデルなど）で複数の候補軌道を作り、それぞれを短時間シミュレーションして「一番安全で良さそうな1本」を選ぶプランナーです。シンプルなのに、nuPlan という大規模ベンチマークで多くの最新の学習ベース手法を閉ループ評価で上回り、「学習系は本当にルールベースより賢いのか？」という問いを投げかけた重要な研究です。2023 年の nuPlan チャレンジで優勝しました。

## 直感的な理解
最先端の深層学習プランナーが、1990 年代生まれの素朴な数式に負ける——そんなことが起きたら驚きますよね。PDM が突きつけたのはまさにこれです。

なぜ起こりうるのか。学習ベースのプランナーは「人間の運転ログに似た軌道」を出すよう訓練されます。これは1回予測して正解と比べる open-loop（開ループ）評価では強い。しかし実際に車を動かすと、自車の小さなズレが次の状況を変え、その状況でまた予測し、を繰り返す closed-loop（閉ループ）になります。学習プランナーは「自分が少しズレた後の、訓練データに無い状況」に弱く、誤差が雪だるま式に積もって車線を外れたり止まったりする。一方ルールベースは「前の車に近づきすぎたら減速」という安全則を毎ステップ律儀に守るので、閉ループで崩れにくい。PDM はこの「閉ループでの頑健さ」を武器に学習系を出し抜いたのです。

## 基礎: 前提となる概念

### 計画（planning）とは
自動運転の計画とは、周囲の状況を踏まえて「自車が次の数秒でどう動くか（自車の軌道）」を決めることです。検知・予測の下流にあり、最終的に車をどう操るかを出力します。

### open-loop と closed-loop
- open-loop（開ループ）評価: 過去ログを与え、1回だけ未来軌道を予測し、人間の実際の軌道と比べる。簡単だが「予測がその後の世界に影響しない」非現実的な設定。
- closed-loop（閉ループ）評価: 予測した軌道で実際に自車を動かし、周囲エージェントも反応する中で、予測 → 実行 → 状況変化 → 再予測を繰り返す。現実に近く、「小さなズレの蓄積」「分布シフト」が初めて顕在化する。

### nuPlan ベンチマーク
nuPlan（Caesar et al., Motional, 2021）は、実車の大規模ログ（約 1,300 時間）と、エージェントが反応するシミュレータをセットにした計画専用ベンチマークです。closed-loop を測れる点が画期的でした。評価モードは3つ:
- OLS（Open-Loop Score）: 人間軌道との一致度。
- CL-NR（Closed-Loop Non-Reactive）: 閉ループだが周囲は録画再生（反応しない）。
- CL-R（Closed-Loop Reactive）: 閉ループで周囲も IDM 等で反応する。
スコアは衝突回避・車線維持・進行・快適性などを総合した 0〜1 の値です。

### IDM（Intelligent Driver Model）
IDM（Treiber, Hennecke, Helbing, 2000）は古典的な追従モデルで、「前の車との距離と速度差から、ぶつからないよう自分の加速・減速を決める」単純な微分方程式です。前が空けば目標速度まで加速、前が詰まれば安全車間を保つよう減速する。PDM のプロポーザル生成の土台になります。

## 仕組みを詳しく

PDM は PDM-Closed（閉ループ用のルールベース本体）と PDM-Open / PDM-Hybrid（学習要素を足した拡張）からなります。中心は PDM-Closed です。

### ステップ1: センターライン抽出
地図から、自車がたどるべき経路（車線中心線、センターライン）を取り出します。これが「基本的にどこを走るべきか」の骨格になります。

### ステップ2: 多数のプロポーザル（候補軌道）を生成
IDM を、いろいろな設定で何通りも走らせます。具体的には、目標速度を複数段階に変え、センターラインからの横オフセット（横ずれ）を複数変える組み合わせで、多数の候補軌道（プロポーザル）を作ります。「速めに走る案」「ゆっくり走る案」「少し左に寄る案」といったバリエーションです。論文の設定では速度 × 横オフセットの組み合わせで十数本程度のプロポーザルが作られます。

### ステップ3: 各候補を短時間シミュレーション
生成した各候補について、その通りに自車を動かしたら数秒後どうなるかを、簡易シミュレータで前方展開（rollout）します。周囲のエージェントは一定の運動（等速など、constant velocity）で動くと仮定します。

### ステップ4: スコアリングして最良を選ぶ
展開した各候補に、nuPlan の評価指標に沿ったスコアをつけます。衝突しないか、車線を逸脱しないか、進行（目的地に向かえているか）、快適さ（急加速・急ハンドルがないか）を総合した点数です。最もスコアの高い1本を選んで実行します。

このステップ2〜4は「たくさん作って、シミュレーションで試して、一番良いものを選ぶ」という生成＋スコアリング構造で、ゴール条件付き計画（[ゴール条件付き計画（先に目標点を予測し、そこへ至る軌道を生成する2段構成）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0179_goal-conditioned-planning)）や生成系プランナー（[CVAE 軌道生成（条件付き変分オートエンコーダによる多峰な未来の生成）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0165_cvae-trajectory-generation) の生成＋ランキング）と発想を共有しますが、PDM は生成も評価も基本ルールベースで学習を必須としない点が特徴です。

### PDM-Hybrid: 学習要素の追加
PDM-Closed は近距離の閉ループ運転に強い一方、長期の経路形状（この先どう曲がるべきか）の見通しは弱めです。PDM-Hybrid は、長期の経路補正（centerline からの将来オフセット予測）だけを軽量な学習モデルに任せ、短期の安全な追従はルールベースに任せる役割分担をしました。重要なのは、学習が予測するのは「数秒以上先の補正」だけで、近距離の安全挙動は触らないこと。これにより open-loop（長期予測の正確さ）と closed-loop（実走の安全さ）の両方で良い成績を出しました。

### 数値・イメージ
```
プロポーザル数 : 目標速度 × 横オフセットの組み合わせで十数本
シミュレーション : 数秒先を 0.1 秒刻みで前方展開 (rollout)
出力 : スコア最大の自車軌道1本 (例 8秒 × xy)
```

## 手法の系譜と主要論文

- IDM（Treiber, Hennecke, Helbing, "Congested traffic states in empirical observations and microscopic simulations", Phys. Rev. E, 2000）: PDM のプロポーザル生成の核となる古典的追従モデル。
- nuPlan（Caesar et al., "nuPlan: A closed-loop ML-based planning benchmark for autonomous vehicles", CVPR ADP3 Workshop 2021、Motional）: PDM が土台にした閉ループ評価ベンチマーク。これがなければ PDM の問題提起は成立しなかった。
- PDM（Dauner, Hallgarten, Geiger, Chitta, "Parting with Misconceptions about Learning-based Vehicle Motion Planning", CoRL 2023、University of Tübingen / NVIDIA）: 原典。IDM で多数候補を作り簡易シミュレーションでスコアリングして選ぶ骨格に、長期見通しだけ学習を足すハイブリッド。2023 nuPlan チャレンジ優勝。
- 論文の主張（misconceptions、誤解を解く）として有名な指摘:
  - open-loop 性能と closed-loop 性能は相関しない（むしろ弱い負相関のことすらある）。人間軌道に似ている（open-loop で良い）からといって実走で安全とは限らない。
  - 単純なルールベースが強力なベースラインになる。学習手法を評価するなら、まず PDM のようなベースラインに勝てているか確認すべき、という規律をもたらした。
- 後続の学習ベース手法: UrbanDriver、GameFormer、PLUTO、Diffusion Planner などが「PDM という強ベースラインを超える」ことを目標に改良を続ける流れ。PDM は「学習系が越えるべき壁」として参照され続けている。
- Hybrid 系の潮流: PDM-Hybrid のように「安全はルール、最適化や長期計画は学習」と役割分担する設計が、実用の現実解として注目されている。

## 論文の実験結果（定量データ）

### 指標の意味
nuPlan の closed-loop スコア（CLS, 0〜100 または 0〜1）は、衝突なし・走行可能領域内・進行・快適性・運転方向遵守などを掛け合わせた総合点です。1つでも致命的違反（衝突等）があるとスコアが大きく下がる、安全重視の設計です。

### PDM の結果
報告によれば、PDM-Closed は nuPlan の closed-loop（CL-NR / CL-R）で、当時の代表的学習手法（UrbanDriver、GC-PGP、IDM 単体など）を明確に上回り、closed-loop スコアでおおむね 90 前後（学習手法群は 70〜80 台が多かった）を記録しました。特筆すべきは、生の人間軌道を「正解」として再生する log-replay すら、closed-loop reactive では PDM に劣る場面があった点で、これは「人間の軌道をそのまま真似ても閉ループで最善とは限らない」ことを示します。

一方 open-loop（OLS）では、PDM-Closed は学習手法に劣ります。人間軌道との一致度を直接最適化していないためです。PDM-Hybrid は長期補正を学習で足すことで OLS を改善し、open-loop と closed-loop の両方で高スコアを達成しました。

### アブレーションが教えること（misconceptions の定量的裏づけ）
- open-loop と closed-loop の非相関: 論文は複数手法の OLS と CLS を散布図にし、両者がほとんど相関しない（open-loop で上位でも closed-loop で下位、その逆もある）ことを定量的に示しました。これが「open-loop だけで評価するな」という規律の根拠です。
- プロポーザル多様化の効果: プロポーザルを単一（速度1通り・オフセット0）にすると closed-loop スコアが落ち、速度・横オフセットを振って多様化するほど改善する。生成＋スコアリングの「生成の多様性」が効いていることを示す。
- 周囲エージェント仮定の影響: 周囲を等速仮定する簡易 rollout でも十分強いが、複雑な交渉局面では読みが浅くなることが定性的に確認された。

## メリット・トレードオフ・限界

メリット
- 学習データ不要・解釈可能。なぜその軌道を選んだかが「スコアが高かったから」と説明でき、デバッグしやすい。
- closed-loop で強く、安全性（衝突回避・車線維持）に直結する挙動が安定。分布シフトに強い。
- 学習手法の強力なベースライン・対照群として、評価の妥当性を担保できる。

トレードオフ・限界
- 想定したルール（IDM 追従 + センターライン）の外の状況（複雑な交渉、譲り合い、創造的な回避）に弱い。
- 周囲エージェントを単純運動と仮定するため相互作用の読みが浅く、混雑・交差点での駆け引きは苦手。
- 多数の候補をシミュレーションするため、候補数を増やすほど計算が増える（精度と計算量のトレードオフ）。
- センターラインに強く依存するため、車線情報が曖昧・欠損する場面（駐車場、複雑な交差点）で破綻しうる。
- 未解決の課題: PDM が示した「単純ルールが学習系を上回る」状況を、学習系がどう本質的に超えるか。closed-loop に強く、かつ複雑な相互作用も扱える学習プランナーの実現は分野の中心課題のまま。

## 発展トピック・研究の最前線
- 学習系の逆襲: PLUTO、GameFormer、Diffusion Planner などは PDM を明示的なベースラインに置き、closed-loop でこれを上回ることを目標に設計された。多くは「PDM 風の提案＋スコアリング」を学習で内製化する方向。
- 評価規律の定着: 「open-loop だけで主張するな、closed-loop と強ルールベースのベースラインを必ず比べよ」という PDM の教訓は、計画研究の事実上の標準作法になった。
- ハイブリッドの一般化: 安全クリティカルな部分はルール・最適化（制御理論、MPC）で保証し、長期計画や好ましさは学習で扱う分業は、実用システムの主流設計の一つ。
- シミュレータ忠実度の課題: closed-loop 評価はシミュレータの周囲エージェント挙動の質に依存する。より現実的な reactive agent（学習ベースの交通シミュレーション）と組み合わせる研究が進む。
