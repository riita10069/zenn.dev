# per-channel バイアスによる条件付け

## ひとことで言うと
「いまシーン全体がどんな状況か」を表す1本のベクトル(グローバル文脈)を、空間特徴マップの各チャネルに足し算という形で混ぜ込む、とても軽い条件付け(conditioning)の手法です。条件付けとは「外から与えた情報に応じてネットワークの内部処理を切り替える」ことを指します。重い再設計をせずに、外部の情報をモデルに教え込めるのがこの手法の値打ちです。シフト(下駄)だけで変調する最小構成なので、追加パラメータも計算もほぼゼロで導入できます。

## 直感的な理解

ニューラルネットワークに「いま起きていること」を補足情報として渡したい場面はとても多くあります。画像を見て質問に答えるモデルなら「質問文」を、文章を生成するモデルなら「書きたいスタイル」を、自動運転モデルなら「自車がいま右折しようとしている」「直前の数秒間ゆっくり走っていた」といった大局的な状況を、空間的な特徴処理の途中に注ぎ込みたくなります。

ところがここに形(shape)の不一致という壁があります。画像系のネットワークが内部で持っているのは「空間特徴マップ」という数値の立方体で、形はたとえば `[B, C, H, W]` です。一方、補足したい文脈は「1本のベクトル」で、形は `[B, C]` です。空間方向(H, W)を持つものと持たないものは、そのままでは足したり混ぜたりできません。

素朴に混ぜようとすると、全部のマス目と文脈ベクトルを総当たりで掛け合わせる巨大な全結合層が必要になり、計算もメモリも爆発します。たとえば 450×300 = 135,000 マス × 256 チャネルという規模を全結合でつなぐのは現実的ではありません。per-channel バイアス条件付けは、この「混ぜ方」を極限まで安くする発想です。各チャネルに「そのチャネル専用のオフセット値」を文脈から作って一律に足すだけ、という割り切りで、ほぼゼロコストの条件付けを実現します。

身近なたとえで言えば、ミキシングコンソールの各チャンネル(楽器ごとのつまみ)を、曲全体の雰囲気を表す指示に応じて一斉に上げ下げするようなものです。個々の音符をいじるのではなく、「ボーカルのチャンネルを全体に少し持ち上げる」といった、チャンネル単位の底上げだけを行います。

## 基礎: 前提となる概念

理解の前提として、いくつかの用語を噛み砕いておきます。

- テンソル(tensor): 多次元の数値配列です。画像は普通「高さ×幅×チャネル」の3次元、それをバッチでまとめると4次元になります。shape(形)とは各次元の大きさのことです。
- バッチサイズ `B`: 同時に処理するデータの本数です。たとえば一度に4シーンをまとめて処理するなら B=4。
- チャネル `C`: 各空間位置(マス目)に持たせる特徴量の本数です。畳み込み層(convolution、近傍のマス目だけを見て計算する層)が出力する「特徴の種類の数」だと思ってよいです。1つのチャネルは「ある種類のパターンへの反応の強さの地図」だと解釈できます。
- 空間サイズ `H, W`: 特徴マップの縦横のマス目数です。元の画像より粗くなっていることが多いです。
- broadcast(放送、ブロードキャスト): 形が違う配列同士で演算するとき、足りない次元を自動的にコピーして引き伸ばし、形を合わせる仕組みです。`[B, C, 1, 1]` を `[B, C, H, W]` に足すとき、最後の2次元(1,1)が(H,W)へ複製され、各マス目に同じ値が足されます。NumPy / PyTorch / TensorFlow いずれにも備わる基本機能です。
- グローバル文脈ベクトル(global context vector): 空間全体を要約した1本のベクトル。たとえば過去フレーム列を要約したり、画像全体を平均プーリング(global average pooling、全マス目の平均を取って1点に潰す)したりして作ります。「どこに何があるか」という空間情報は捨て、「全体としてどういう状況か」だけを残した表現です。
- 非線形活性化(ReLU など): 線形演算(足し算・掛け算)の合間に挟む、入力の符号や大きさで出力を歪める関数。ReLU は負の値をゼロにする関数で、「あるチャネルを通すか遮断するか」というスイッチのように働きます。バイアスの足し算が後段でゲートとして効くのは、この非線形のおかげです。

この前提のうえで、per-channel バイアスは「グローバル文脈ベクトル `[B, C]` を、空間方向に broadcast して空間マップ `[B, C, H, W]` に足す」操作だと一言で言えます。

## 仕組みを詳しく

アイデアの核は「チャネルごとに別の値を、でも空間方向には一律で足す」ことです。グローバル文脈ベクトル `g`(形 `[B, C]`)のチャネル `c` 成分 `g[:, c]` を、そのチャネルのマップ全面(H×W の全マス)へ同じ値として足します。

手順を分解します。

1. グローバル文脈ベクトルを用意する。形 `[B, C]`(例 `[4, 256]`)。元情報の次元が違うなら小さな線形層(MLP)で C 次元に整える。
2. 空間方向に次元を増やす。`[B, C]` → `[B, C, 1, 1]`。
3. 空間マップ `[B, C, H, W]` に足す。broadcast により `[B, C, 1, 1]` が `[B, C, H, W]` へ引き伸ばされ、各チャネル `c` のマップ全面に `g[:, c]` という同じ値が足される。

擬似コードで書くとこれだけです。

```
# feat: [B, 256, 450, 300]   空間特徴マップ
# ctx:  [B, D]               生のグローバル文脈(過去要約など)
g    = linear(ctx)           # [B, 256]  C 次元へ射影
bias = g[:, :, None, None]   # [B, 256, 1, 1]
feat_conditioned = feat + bias  # broadcast で各チャネル全面に同じバイアスが足される
```

数値で見ると分かりやすいです。チャネル0には +0.3、チャネル1には -1.2 … というように各チャネルへ異なるオフセットがかかります。一方、同じチャネルの中ではマップのどのマス目にも同じ値が足されます。だから「per-channel(チャネルごとの)バイアス」と呼びます。

```
before (チャネル c=5 のマップ抜粋):    after (g[:,5] = +0.7 を一律に足す):
 0.1  0.2  0.0                          0.8  0.9  0.7
 0.3 -0.1  0.4              →           1.0  0.6  1.1
-0.2  0.5  0.1                          0.5  1.2  0.8
```

直感的には、各チャネルが「ある種類の特徴を測るセンサー」だとすると、グローバル文脈がそのセンサー全体の基準値(バイアス)をまとめて上下させるイメージです。「いまは右折中」という文脈が来たら、右折に関係するチャネルを底上げし、関係ないチャネルを下げる、といった調整を学習で身につけられます。後段の非線形(ReLU など)を通すと、バイアスの上げ下げは「そのチャネルを活性化させるか抑えるか」のゲートのように働きます。+0.7 されたチャネルは ReLU を通り抜けやすくなり、-2.0 されたチャネルは全面的にゼロへ潰れて事実上オフになります。

なぜこれが軽いか。追加パラメータは「文脈ベクトルを C 次元に整える小さな線形層」くらいで、足し算自体はパラメータゼロ、計算も `O(B·C·H·W)` の単純加算だけです。空間マップを丸ごと作り直したり巨大な全結合をかけたりしないので、メモリ増加は実質ほぼゼロです。実装も既存のネットワークに1行差し込むだけ(drop-in)で済みます。

### FiLM との関係

この「チャネルごとに条件で値を変える」発想は、FiLM (Feature-wise Linear Modulation、特徴ごとの線形変調)として知られる手法の一種です。FiLM は各チャネル `c` に対し、条件ベクトルからスケール `γ_c`(掛ける係数)とシフト `β_c`(足すオフセット)を生成し、

```
out_{c,h,w} = γ_c · feature_{c,h,w} + β_c
```

と変調します(`γ_c, β_c` はチャネルごとに1つの値で、空間(h,w)には broadcast)。per-channel バイアスは、このうちシフト `β` だけを使った最小バージョン(`γ_c = 1` に固定)です。掛け算 `γ` を省くぶんさらに軽く、学習も安定しやすい一方、表現力は一段下がるトレードオフがあります。`γ` があると「そのチャネルを増幅/減衰させる」乗法的な制御ができ、`β` だけだと「下駄を履かせる」加法的な制御に限られます。乗法は「特定の文脈でだけそのチャネルを強烈に立てる/殺す」非線形なゲーティングを表現でき、加法は「常に一定量持ち上げる」線形なバイアスに留まる、という違いです。

### 初期化の作法(恒等変換から始める)

条件付けを安定に導入する定番は、`β` を生成する線形層の重みとバイアスをゼロ初期化し、学習開始時には `β = 0`(つまり条件付け無し=恒等変換)から始めることです。こうすると既存ネットワークの挙動を壊さずに学習が立ち上がり、必要なぶんだけ徐々に変調を効かせるようになります。これは拡散モデルの adaLN-zero(後述)や ReZero、LayerScale と同じ「ゼロから足す」思想で、新しい経路を安全に注入する一般作法として広く使われています。

## 手法の系譜と主要論文

- de Vries et al., "Modulating early visual processing by language" (NeurIPS 2017, arXiv:1707.00683)。Conditional Batch Normalization (CBN) を提案。言語の埋め込みから Batch Normalization のスケールとシフトを予測し、画像特徴を言語で変調しました。動機は、言語の影響を分類器の最後だけでなく視覚処理の初期段階から効かせることです。これが特徴ごとの条件変調の直接の先駆です。
- Perez et al., "FiLM: Visual Reasoning with a General Conditioning Layer" (AAAI 2018, arXiv:1709.07871)。CBN を Batch Norm から切り離して一般化し、「条件から `γ, β` を作って特徴を変調する独立した層」として定式化しました。動機は、質問ごとに別ネットワークを作るのは非効率なので、共通の視覚処理ネットワークの中身を条件に応じてチャネル単位で薄く変調する方が安く柔軟だ、というものです。
- Dumoulin et al., "Feature-wise transformations" (Distill, 2018)。FiLM 系の条件付けを横断的に整理した解説論文。スタイル変換、条件付き生成、マルチタスク、ドメイン適応など多くの場面で「特徴ごとの線形変調」が繰り返し有効だったことをまとめ、安価で汎用的な反面、空間的にきめ細かい条件付けには向かないという限界を明確にしました。
- Park et al., "SPADE: Semantic Image Synthesis with Spatially-Adaptive Normalization" (CVPR 2019, arXiv:1903.07291)。`γ, β` を1チャネルあたり1値ではなく空間マップとして与え、位置ごとに異なる変調を可能にしました。per-channel バイアスの「空間一律」という限界を埋める方向の代表で、空間条件付けと位置一律条件付けの対比を理解する鍵になります。
- Peebles & Xie, "DiT: Scalable Diffusion Models with Transformers" (ICCV 2023, arXiv:2212.09748)。Transformer 時代に同じ「外部条件で特徴を変調する」発想を adaptive layer normalization (adaLN) / adaLN-zero として持ち込みました。adaLN-zero は変調の初期値をゼロにして恒等変換から学習を始める工夫で、per-channel バイアスを「最初は何もしない安全な初期化」で導入する考え方の延長線上にあります。

この系譜は「条件をネットワークの中身へどう注ぐか」を一段ずつ洗練してきた流れです。CBN が「正規化層に条件を埋める」と気づき、FiLM が「正規化から切り離した独立層」に一般化し、SPADE が「空間方向にも変調を広げ」、DiT が「Transformer のノルム層に持ち込んで大規模生成へ」展開しました。per-channel バイアスはこの中で最も軽い末端の実装、と位置づけられます。

## 論文の実験結果(定量データ)

FiLM (Perez et al., 2018) は視覚的推論データセット CLEVR で評価されました。CLEVR は合成画像に対し「赤い立方体の左にある球と同じ色の物体は何個?」のような多段推論を要する質問応答(VQA)のベンチマークで、指標は回答の正解率(accuracy)です。

- FiLM は CLEVR で約 97.7% の正解率を達成しました。人間の正解率(約 92.6% と報告)を上回り、当時の専用設計モデルに匹敵しました。
- 重要なのはモデルの素朴さです。FiLM は「画像 CNN + 質問 GRU + FiLM 変調」という汎用部品の組み合わせで、明示的な推論モジュールを持たないにもかかわらず高精度に届きました。条件付けの混ぜ方ひとつでここまで行ける、という点が示唆的です。
- 汎化テスト(CLEVR-Humans、人間が自由に書いた質問)でも、事前学習した FiLM 層を再利用して fine-tune することで高い精度を保ちました。条件変調が特定の質問表現に過剰適合しにくいことを示します。
- アブレーション: `γ`(スケール)を取り除いて `β`(シフト)だけにしても精度の低下は比較的小さく(報告では数ポイント以内)、変調の効果の多くがシフト項で説明できるとされています。これは per-channel バイアス(=シフトのみ)が実用的に妥当であることの直接の根拠になります。逆に、`β` を取り除いて `γ` だけにすると低下幅が大きく、シフトの寄与が支配的でした。また、変調を浅い層だけに限ると精度が落ち、複数の層に分散して FiLM を挿す方が効くことも示されました。

CBN (de Vries et al., 2017) は VQA データセット(VQAv1)と GuessWhat?! で評価され、言語による視覚特徴の変調がベースラインより精度を改善することを示しました。指標差は数値上は数ポイント規模ですが、「変調を視覚処理の初期から効かせる」設計の有効性を確立した点に意義があります。

これらの数値が伝えるのは、「条件をどう混ぜるか」というアーキテクチャ上の小さな選択が、追加パラメータをほとんど増やさずに下流性能を大きく動かしうる、という事実です。とりわけ「アブレーションでスケール `γ` を抜いてもほとんど落ちない」という FiLM の結果は、加法バイアスのみで十分なケースが多いことを実験的に保証しており、軽量化を狙ううえで安心材料になります。

## メリット・トレードオフ・限界

メリット
- 追加パラメータと計算がごくわずか。空間マップを作り直さないのでメモリ増加がほぼない。
- 既存ネットワークに1行差し込むだけ(drop-in)で導入でき、ゼロ初期化を使えば恒等変換から安全に学習を始められる。
- 足し算のみなので値が暴れにくく、学習が安定しやすい。乗法 `γ` がないぶん、変調が暴走して特徴を発散させる危険が小さい。

トレードオフ・限界
- 空間方向には一律。「右前方のこのマスだけ強める」といった位置ごとの細かい条件付けができない。グローバルな文脈しか伝えられない。
- 個々の動く物体の速度のような、空間位置に紐づいた(per-cell な)情報は表現できない。それが必要なら空間構造を保つ別の機構(畳み込み再帰など)が要る。
- FiLM のスケール `γ` を省いた加法のみの版なので、乗法的なゲーティングが効く問題では表現力が不足しうる。「ある文脈のときだけこのチャネルを完全に殺す」ような鋭い制御は苦手。
- 研究上の未解決課題として、どの層に・いくつ条件変調を挿すのが最適かは問題依存で、理論的な指針は乏しく経験則に頼っている点が挙げられます。また、グローバル文脈の作り方(何を要約に含めるか、どう時間方向に集約するか)が下流性能を左右し、その設計自体が研究テーマになります。

## 発展トピック・研究の最前線

- 空間を保った条件付け: SPADE (Park et al., 2019) は `γ, β` を空間マップとして与え、位置ごとに異なる変調をかけます。per-channel バイアスの「空間一律」の限界を埋める方向です。セグメンテーションマップ条件付き画像生成で、位置ごとの意味を保ったまま変調することで境界の崩れを大きく減らしました。
- Transformer での変調: adaLN / adaLN-zero(DiT)や cross-attention による条件付けは、条件の与え方として加法バイアスより表現力が高い選択肢です。cross-attention は条件の各要素と特徴の各位置を動的に対応づけるため、空間的にも内容的にもきめ細かい一方、計算コストは加法バイアスより桁違いに大きく、コストと表現力のトレードオフを問題に応じて選ぶことになります。
- ゼロ初期化の系譜: 変調や残差の初期値をゼロにして恒等写像から学習を始める設計(adaLN-zero, ReZero, LayerScale)は、条件付けを安定に導入する一般的な作法として広がっています。深いネットワークや事前学習済みモデルへ新経路を後付けする際の標準テクニックです。
- 時系列文脈の集約: グローバル文脈をどう作るかは独立した研究領域で、過去フレーム列を平均する単純法から、注意機構や再帰で重み付き集約する方法まで幅があります。per-channel バイアスは「集約後の1ベクトルをどう注ぐか」を担う末端であり、その上流の集約設計と組み合わせて性能が決まります。
