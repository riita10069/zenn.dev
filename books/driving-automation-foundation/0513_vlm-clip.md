# CLIP (画像-言語対比学習)

## ひとことで言うと
CLIP (Contrastive Language-Image Pre-training、対比的な言語-画像事前学習) は、「画像」と「その画像を説明する文章」を、意味が同じなら近く・違うなら遠くなるように1つの共有ベクトル空間へ並べて覚えさせたモデルです。これにより、あらかじめ決めたラベル一覧 (「犬」「猫」…) に縛られず、好きな文章を投げてその画像が何かを問い合わせられるようになります。今日のほぼすべての視覚言語モデル (VLM、画像も言葉も扱えるモデル) の「目」の部分は、この CLIP 系の表現を土台にしています。CLIP を理解することは、現代のマルチモーダル AI の出発点を理解することと同義です。

## 直感的な理解
従来の画像認識器は、たとえば ImageNet (約120万枚の画像に1000個のクラスラベルを付けたデータセット) で「1000クラスのどれか」を当てるよう学習しました。これは「答えの選択肢が学習時に固定されている」ということです。新しい概念 (「縁石に乗り上げそうな自転車」) を後から問い合わせたければ、その概念のラベル付きデータを集めて再学習しなければなりません。これを closed-vocabulary (語彙が閉じている) と呼びます。

しかし現実の世界、特に運転シーンには無限のバリエーションがあります。「右折待ちの対向車」「工事規制の三角コーン」「逆走する車」など、起こりうる状況を事前にすべてラベルとして列挙するのは不可能です。欲しいのは「自由な文章で問い合わせられる」認識、すなわち open-vocabulary (開いた語彙) です。

CLIP のアイデアは「画像と言葉を同じ地図の上に置く」ことです。地図の上で、ある画像の点と「a photo of a dog」という文章の点が近ければ、その画像は犬らしい、と判定できます。重要なのは、この地図さえ作ってしまえば、後から「逆走する車」という新しい文章の点を打って、画像との距離を測るだけで判定できることです。再学習は要りません。文章を差し替えるだけで分類タスクを無限に変えられます。

なぜこれが画期的だったのか。それまでの「自然言語を教師に使う」試みは、画像から正確なキャプション (説明文) を生成させようとして、難しすぎてうまくいきませんでした。「どんな単語が・どんな順序で・どんな文法で」という生成の自由度が大きすぎたのです。CLIP は問題を「この画像とこの文は本来のペアか?」という○×判定に弱めました。生成より判定の方がはるかに易しく、これが学習効率を一気に高めた発想の核心です。

## 基礎: 前提となる概念
- 埋め込み (embedding): 画像や文章を、固定長の数値ベクトル (例: 512次元の数の並び) に変換したもの。意味の近さがベクトルの近さ (距離やコサイン類似度) として表れるように設計します。
- コサイン類似度 (cosine similarity): 2つのベクトルの「向きの近さ」を測る指標で、-1 (正反対) から +1 (同じ向き) の値をとります。各ベクトルを長さ1に正規化 (単位ベクトル化、ベクトルをその長さで割って大きさを1にすること) しておけば、内積 (対応する要素どうしを掛けて足した値) がそのままコサイン類似度になります。
- 対比学習 (contrastive learning): 「正しい組 (正例、positive) を引き寄せ、間違った組 (負例、negative) を引き離す」学び方。CLIP では、正しい画像-文章ペアを近づけ、それ以外の組み合わせを遠ざけます。負例をたくさん見せるほど「何が違うか」を細かく学べます。
- ゼロショット (zero-shot): あるタスク専用の追加学習を一切せずに、そのタスクをこなすこと。CLIP の「文章を差し替えるだけで分類」がまさにこれです。「いちども見たことのない問題を、その場で解く」と考えるとよいです。
- ViT (Vision Transformer): 画像を 16×16 などの小さなパッチ (区画) に切り、各パッチを単語のように Transformer (系列を一括して相互参照する深層ネットワーク) に流す画像エンコーダ。CLIP の画像側でよく使われます。表記 ViT-L/14 は「Large サイズ・パッチ一辺14画素」を意味します。
- softmax (ソフトマックス): 数のリストを「合計が1の確率」へ変換する関数。最大の値を強調しつつ全体を確率分布にします。CLIP の損失計算で「正解はどれか」を確率として表すのに使います。

## 仕組みを詳しく
CLIP は2つのエンコーダ (符号化器、入力をベクトルに変換する装置) を持ちます。

- 画像エンコーダ: 画像を受け取り、512次元などの1本のベクトル (画像埋め込み) にする。中身は ResNet または ViT。
- テキストエンコーダ: 文章を受け取り、同じ次元のベクトル (テキスト埋め込み) にする。中身は Transformer。

学習のステップを具体的に追います。

1. バッチに N=32768 組の (画像, 文章) を用意する。これらは本来の正しいペア (Web から集めた、画像とそのキャプション)。
2. 画像エンコーダで N 個の画像ベクトル、テキストエンコーダで N 個の文章ベクトルを作る。すべて長さ1に正規化する。形状で言うと `image_emb` は [N, 512]、`text_emb` は [N, 512]。
3. N×N の類似度行列を作る。行 i 列 j の値は「画像 i と文章 j のコサイン類似度」。`logits = image_emb @ text_emb.T / τ` で [N, N]。
4. 対角線の N 個 (画像 i と文章 i、本来の正しい組) を「正解」とみなす。各行で softmax をかけ、対角が最大になるよう学習する。各列でも同様に行う。これを対称な交差エントロピー損失 (正解の確率の対数にマイナスを付けた値を最小化する損失) にする。つまり「画像 i にとって正解の文章は i 番だけ、残り N-1 個は不正解」という N クラス分類を、行方向と列方向の両方で同時に解いています。
5. 温度パラメータ τ (temperature、類似度を何倍に拡大して softmax に入れるかの調整値) を学習可能にして、引き寄せ/引き離しの鋭さを自動調整する。CLIP では τ の対数を学習し、上限 100 でクリップ (頭打ちに) します。τ が小さいほど softmax が鋭くなり、正解を強く際立たせます。

ASCII で損失のイメージを描くと、類似度行列はこうなります (N=4 の例)。

```
        txt0  txt1  txt2  txt3
 img0 [ 0.9   0.1   0.0   0.2 ]   ← 行 softmax の正解は対角 (img0,txt0)
 img1 [ 0.1   0.8   0.1   0.0 ]
 img2 [ 0.0   0.2   0.95  0.1 ]
 img3 [ 0.1   0.0   0.1   0.85]
        ↑列 softmax の正解も対角
```

対角を大きく、非対角を小さくしていくのが学習の中身です。バッチが大きいほど「間違った組 (負例)」が一度にたくさん見られるので、表現がよく鍛えられます。CLIP が N=32768 という巨大バッチを使う理由がここにあります。32768 という1つのバッチを作るには多数の GPU をまたいで埋め込みを集約する必要があり (all-gather と呼ぶ通信)、これが学習インフラ上の大きな負担になりました。後述する SigLIP はこの通信コストを損失の作り替えで取り除きます。

学習後の使い方 (ゼロショット分類) が巧妙です。「この画像は赤信号か青信号か」を判定するなら:

- 文章候補を作る: "a photo of a red traffic light", "a photo of a green traffic light"。この定型文を prompt template (プロンプトの型) と呼び、"a photo of a {}" の形が経験的に効きます。学習データの多くが「a photo of ...」調のキャプションだったため、推論時もその語調に合わせると当たりやすくなります。実際 CLIP では複数のテンプレート (80種) でベクトルを作って平均する prompt ensembling で ImageNet 精度が数ポイント向上しました。
- 各文章をテキストエンコーダでベクトル化。
- 画像をベクトル化し、各文章ベクトルとの類似度を計算。最も近い文章を答えとする。

before/after で言えば、従来は「赤信号クラス (番号5)」と番号で管理し新規クラス追加に再学習が必要でしたが、CLIP では「赤信号という文章」を投げるだけで、学習時に明示的に見ていなくても近い概念なら拾えます。

なぜ生成でなく対比なのか、計算の感覚で言うと: 画像から正確な単語列を予測する (生成) には、その画像に付いた「唯一の正しい文」を当てねばならず、言い回しの違いも誤りとされ学習が遅くなります。対比なら「バッチ内のどの文と最も合うか」を当てればよく、はるかに緩い課題です。論文はこの置き換えで学習効率が約4倍になったと報告しています。

## 手法の系譜と主要論文
- CLIP (Radford et al., "Learning Transferable Visual Models From Natural Language Supervision", ICML 2021, OpenAI, arXiv:2103.00020): 4億の画像-テキストペア (WIT データセット) を Web から集め、対比学習で画像とテキストを共有空間に揃えた起点。動機は「自然言語を教師信号にすれば固定ラベルの制約から逃れ、ゼロショットで多様なタスクに転移できる」。新規性は、画像キャプション生成 (画像から正確な文を作る難問) を解くのではなく、対比学習という「ペアが合うか当てるだけ」の緩い課題に変えたことで、学習が約4倍効率化した点です。
- ALIGN (Jia et al., "Scaling Up Visual and Vision-Language Representation Learning With Noisy Text Supervision", ICML 2021, Google, arXiv:2102.05918): CLIP とほぼ同時期に、より「汚い (フィルタを最小化したノイズの多い)」18億ペアでも、スケールで押し切れば同等以上の表現が学べると示した。動機は「データのきれいさよりも量が効く」。寄与は、丁寧なデータクリーニングがなくても規模で勝負できると実証したこと。ここから「データ量 vs データ質」というその後ずっと続く論点が生まれます。
- OpenCLIP / LAION (Cherti et al., CVPR 2023, arXiv:2212.07143; LAION-5B, Schuhmann et al., NeurIPS 2022): CLIP を公開データ (LAION-400M / LAION-2B / 5B) で再現・拡張し、誰でも使える重みを提供した。これにより研究や運転 VLM の特徴抽出器として CLIP 系が広く流用可能になった。スケーリング則 (データ・計算・モデルを増やすと性能がべき乗で滑らかに伸びる経験法則) を CLIP について体系的に検証し、「モデル・データ・計算のどれを増やすと何が伸びるか」を定量化した点も重要です。
- SigLIP (Zhai et al., "Sigmoid Loss for Language Image Pre-Training", ICCV 2023, Google, arXiv:2303.15343): 対比損失を softmax (バッチ全体での正規化) から sigmoid (各ペアを独立に正例/負例判定する関数) へ変えた。動機は「バッチ全体の正規化のために巨大バッチ間で値を集約する通信が要らなくなり、小バッチでも効率よく学習できる」。各 (i, j) ペアを独立に「正例か負例か」のロジスティック回帰 (二択の確率を出す回帰) として扱うため、N×N 行列の行・列 softmax が不要になります。
- DataComp / DFN (Gadre et al., NeurIPS 2023, arXiv:2304.14108; Fang et al., 2023, arXiv:2309.17425): 「モデルを固定しデータの選び方を競う」という発想で、学習データの取捨選択が性能の主因だと示した。DFN (Data Filtering Networks) は「学習データそのものを選別するネットワーク」を別途学習してデータ質を底上げします。系譜の関心が「アルゴリズム」から「データの質と量の最適化」へ移った流れを象徴します。
- EVA-CLIP / SigLIP 2 (Sun et al., 2023, arXiv:2303.15389; Tschannen et al., 2025, arXiv:2502.14786): EVA-CLIP は MIM (masked image modeling、画像の一部を隠して復元させる事前学習) で鍛えた強い ViT を CLIP 化して効率を上げた。SigLIP 2 は多言語化と密な (パッチ単位の) 特徴・自己蒸留を加え、検出やセグメンテーションなど位置を要する下流に効く表現へ拡張しました。近年の VLM の視覚エンコーダとして SigLIP/SigLIP 2 が CLIP に代わって採用される例が増えています。

## 論文の実験結果 (定量データ)
- ゼロショット ImageNet: CLIP ViT-L/14 (336px 入力) は ImageNet を一切その分類用に追加学習せずに、トップ1精度およそ 76.2% を達成。これは ImageNet で教師あり学習した ResNet-50 (約 76%) に匹敵します。トップ1精度とは「最も確率の高い1つの予測が正解と一致する割合」で、100枚中76枚を当てる感覚です。ラベル付き ImageNet を1枚も追加で使わずこの水準というのが衝撃でした。
- 分布シフトへの頑健性: ImageNet で学習した普通の分類器は、ImageNet-A (分類器が間違えやすい自然画像) や ImageNet-R (絵画・スケッチなど見た目が変わった画像) で精度が大きく落ちます。CLIP はこうした分布シフトに対し、同じ ImageNet 精度の通常モデルより一貫して高い頑健性を示しました (有効頑健性のギャップとして報告)。理由は、固定ラベルへの過剰適合ではなく、言語で結びついた一般的な概念を学んでいるためと解釈されています。安全クリティカルな運転では「学習時と違う見た目」に強いことが死活的に重要で、この性質が CLIP 系を視覚バックボーンに選ぶ実用的な根拠になっています。
- 転移性: 27 のデータセットでのゼロショット/線形プローブ評価で、線形プローブ (特徴を固定し最後の線形層だけ学習する評価法。表現そのものの良さを測る) において従来の自己教師あり/教師あり表現を上回りました。
- スケーリング (OpenCLIP): LAION-2B で学習した ViT-G/14 はゼロショット ImageNet で約 80% 台に達し、データ・計算を増やすほど滑らかに精度が伸びることが確認されました。べき乗則的な改善は「どれだけ投資すればどれだけ伸びるか」を事前に見積もる根拠になります。
- SigLIP のアブレーション的知見: sigmoid 損失により、バッチサイズ 32k 前後で softmax 版と同等以上、かつ小バッチ域 (例えば数千) では softmax 版より明確に高精度になることが報告されました。すなわち「巨大バッチは性能の本質ではなく、損失の正規化方式の問題だった」と示した点が研究上重要です。少ない GPU でも高品質な CLIP 系を学習できる道を開きました。
- 弱点を示すアブレーション: CLIP は「数える」「文字を読む」「左右・前後の空間関係」を要するベンチマーク (たとえば CLEVR 系の計数や空間タスク) で著しく低い精度を示します。対比学習が「だいたい何が写っているか」は捉えても「いくつ・どこに・どの並びで」は捉えにくいことを表しています。運転で「歩行者が何人いるか」「自車の左か右か」を厳密に問う場面では、この弱点が直接効いてきます。

## メリット・トレードオフ・限界
- メリット: 固定ラベル不要で自由文によるゼロショット問い合わせができる。人手アノテーションなしに Web 規模で学習でき、下流タスクへの転移性が高い。多くの VLM の視覚側の事実上の標準初期化になっている。
- メリット: 画像とテキストが同じ空間にあるため、画像検索・テキスト検索・分類を1モデルで統一的に扱える。「テキストで画像を検索」「画像で類似画像を検索」が同じ仕組みで実現します。
- メリット: 分布シフトへの頑健性が高く、学習時と異なる見た目の入力でも性能の崩れが小さい。
- トレードオフ: きめ細かい計数、左右・前後の空間関係、文字の正確な読み取りに弱い。運転で「歩行者が何人いるか」「どちら側にいるか」を厳密に問うと外しやすい。
- トレードオフ: 学習データの偏り (地域・言語・社会的バイアス) をそのまま継承する。Web 由来のため特定の地域・文化に偏った概念分布を持ち、安全が問われる用途で疑似ラベルに使う際は検証が要る。
- 限界・未解決: 学習コストが巨大 (数億〜数十億ペア、数百〜数千 GPU 日) で自前学習は非現実的なため、公開重み (OpenCLIP, SigLIP) に依存する。また「合成性 (compositionality、語の組み合わせ・属性の結びつけ)」、つまり「赤い車の左に青いバイク」のような関係の理解に弱いことが、CREPE や SugarCrepe といったベンチで繰り返し指摘されており、現在も活発な研究課題です。CLIP は「単語の袋 (bag of words) のように扱い、語順や関係を無視しがち」という診断が定着しています。

## 発展トピック・研究の最前線
- グラウンディング/領域対応: 画像全体ではなく領域単位で言語と対応づける RegionCLIP や、open-vocabulary 物体検出 (OWL-ViT, GLIP, Grounding DINO) は CLIP の知識を「どこに何があるか」へ拡張する流れです。運転の物体ローカライズ (位置特定) に直結します。
- 細粒度・合成性の改善: 否定例 (微妙に意味を変えた誤りの文) を意図的に作って学習する手法や、テキスト側を強化する流れ (SigLIP 2 は多言語化と密な特徴を強化) が進んでいます。
- 蒸留と効率化: 巨大 CLIP の知識を小型モデルへ蒸留 (大モデルの出力を教師に小モデルを学習させること) し、車載のような計算制約下でも open-vocabulary 認識を動かす研究があります。MobileCLIP などはモバイル/エッジ向けに高速化と高精度を両立しています。
- 視覚バックボーンとしての世代交代: 後続の [BLIP-2 (Q-Former)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0512_vlm-blip2) や [LLaVA (Visual Instruction Tuning)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0515_vlm-llava)、[Qwen-VL](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0516_vlm-qwen-vl) は、CLIP 系 (近年は SigLIP) の視覚エンコーダを凍結または微調整して言語モデルに接続します。CLIP を理解することは、現代 VLM の「目」を理解することと同義です。
- 自己教師との融合: DINOv2 のような言語を使わない自己教師表現と CLIP を組み合わせ、空間理解 (CLIP が弱い領域) を補う研究 (例: 複数バックボーンの併用) が進んでいます。
