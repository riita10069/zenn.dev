# I-JEPA / V-JEPA(自己教師あり予測表現学習)

## ひとことで言うと

正解ラベルを人手で付けなくても、画像や動画から「良い特徴の作り方」を学ぶ手法群です。やり方は「入力の一部を隠し、その隠れた部分が(ピクセルそのものではなく)抽象的な特徴空間でどう見えるかを予測させる」こと。これを自己教師あり学習(self-supervised learning、SSL)と呼びます。JEPA は「予測を潜在空間で行う」点が新しく、表現崩壊という古典的な落とし穴を非対称設計で巧みに避けます。画像版が I-JEPA、動画版が V-JEPA です。

## 直感的な理解

人間は「世界のすべてのピクセル」を覚えてはいません。机の上のコップを見たとき、私たちは木目の一本一本やコップ表面の細かい反射を再現できるわけではない。しかし「コップがそこにある」「右に押せば落ちる」という構造は理解しています。つまり知能は、再現不能な細部を捨て、行動に効く抽象構造だけを抽出します。

機械にこの「抽象を学ぶ」能力を持たせたいとき、素朴にやると「画像の隠した部分のピクセルを当てさせる」ことになります。しかしこれは、影の濃淡・木目・センサノイズといった、予測しようがない・どうでもいい細部まで再現させてしまいます。学習能力の多くがどうでもいい部分に費やされ、出来上がる特徴も細部に引っ張られます。

JEPA のアイデアは単純です。「ピクセルではなく、エンコーダが作った抽象特徴の方を予測させればいい」。隠した部分の「特徴ベクトル」を当てる課題にすれば、ネットワークは「どうせ細部は当てられない」と知っているので、自然に予測可能な構造(物体の存在・配置・カテゴリ)だけを表現するようになります。これが「予測空間を抽象に置く」ことの効果です。

## 基礎: 前提となる概念

まず関連する用語を噛み砕きます。

- ニューラルネットワーク: 入力(画像など)を受け取り、層を重ねて数値ベクトルに変換する関数。学習とは、出力が望ましくなるよう各層の重みを少しずつ調整することです。
- 教師あり学習(supervised learning): 入力と正解ラベル(「これは猫」)のペアを大量に与えて学ばせる方式。ラベル付けが高コストです。
- 自己教師あり学習(self-supervised learning): データ自身の一部を隠し、それを当てさせることで「正解」を自動生成する。人手のラベルが要りません。「自己」教師ありとは、教師信号をデータ自身から作る、という意味です。
- 表現(representation)・埋め込み(embedding): ネットワークが入力を変換して得る数値ベクトル。良い表現とは、下流のタスク(分類・検出・行動予測)に役立つ情報が整理されて入っているものを指します。
- エンコーダ(encoder): 入力を表現に変換するネットワーク部分。
- 勾配(gradient): 損失(誤差)を下げるには重みをどちらにどれだけ動かせばよいか、という方向と大きさの情報。学習はこれに従って重みを更新します。
- 勾配遮断(stop-gradient / detach): ある経路に勾配を流さないようにする操作。「ここから先は更新の手がかりにしない」と宣言することです。
- Vision Transformer(ViT): 画像を小さなパッチ(例 16×16 ピクセル)に切り分け、各パッチを単語のように扱って Transformer で処理するモデル。JEPA の標準的なバックボーン。

自己教師あり学習には大きく3系統あります。

1. 生成系・再構成系(generative / reconstructive): 隠した部分の「ピクセルそのもの」を復元させる。代表は Masked Autoencoder(He et al., CVPR 2022, arXiv:2111.06377)。問題は前述の通り、予測不能な細部まで再現対象になることです。
2. 対比学習系(contrastive): 同じ画像の2つの見え方(データ拡張で作る)を「近く」、別画像を「遠く」に配置する。代表は SimCLR(Chen et al., ICML 2020)や MoCo(He et al., CVPR 2020)。問題は、大量の「別画像(ネガティブサンプル)」が必要で、バッチサイズや選び方に敏感な点です。
3. 結合埋め込み予測(Joint-Embedding Predictive Architecture、JEPA): ピクセルではなく潜在空間で予測する。ネガティブサンプル不要で、生成系の無駄も避けます。これが本ノートの主役です。

## 仕組みを詳しく

### 表現崩壊という最大の落とし穴

JEPA を理解する鍵は「表現崩壊(representation collapse)」です。予測器と「正解の作り手(target エンコーダ)」の両方を同時に学習で更新すると、ネットワークは致命的な近道を見つけます。「全部の入力に対して同じ定数ベクトルを出せば、予測は必ず当たる(損失ゼロ)」。これでは入力の情報を一切持たない、無意味な表現になります。

JEPA はこれを非対称設計で防ぎます。すなわち、

- オンラインエンコーダ(online encoder): 学習で普通に更新する側。文脈(見える部分)を符号化する。
- target エンコーダ(target encoder): 正解の特徴を作る側。これを勾配で更新しない(frozen)か、オンライン側の重みを指数移動平均(EMA、Exponential Moving Average)でゆっくり真似させるだけにする。

正解側が予測側に引っ張られて一緒に潰れることがなければ、「両方を定数にする」近道が塞がれます。これが BYOL(Grill et al., NeurIPS 2020, arXiv:2006.07733)で発見された非対称性の原理で、JEPA はこれを「予測を潜在空間で行う」枠組みに一般化したものです。

EMA の更新式は次の通りです。target の重みを θ_t、online の重みを θ_o、減衰率を τ(典型的に 0.996〜0.9999)として、

```
θ_t ← τ · θ_t + (1 - τ) · θ_o
```

τ が 1 に近いほど target はゆっくり変わります。「正解が急に動かないので、オンライン側は安定した的を追える」という直感です。τ を学習の進行に応じて 0.996 から 1.0 へ徐々に上げる(コサインスケジュール)のが I-JEPA / V-JEPA の標準的なレシピです。

### I-JEPA(画像版)の流れ

I-JEPA(Assran et al., CVPR 2023)の処理は次の5ステップです。

1. 1枚の画像をパッチに分割し、「文脈ブロック(context、見える部分)」1個と複数の「ターゲットブロック(target、隠す部分)」を切り出す。ターゲットは画像全体に散らばる複数の比較的大きなブロックにし、文脈側からはそのターゲット領域を取り除く。
2. context をオンラインエンコーダに通して特徴を得る。
3. predictor(小さな Transformer)が「target の位置(マスクトークンと位置埋め込みで指定)にある特徴はこうだろう」と潜在空間で予測する。
4. target ブロックを target エンコーダ(EMA、勾配遮断)に通して「正解の特徴」を得る。
5. 予測した特徴と正解の特徴の距離(I-JEPA は L2、すなわち二乗誤差の平均)を損失にして、オンライン側と predictor だけを更新する。

ステップ4が detach されているのが要点です。損失を下げる手段は「予測を正解に合わせる」しかなく、「正解を予測に合わせる(=両方を潰す)」はできません。

形状の数値例(ViT-Huge, パッチ 16, 画像 224×224 の場合):画像は 14×14=196 パッチに分かれ、各パッチが例えば 1280 次元の埋め込みを持ちます。予測器は target 位置のパッチ埋め込み(各 1280 次元)を出力し、L2 で比較します。

### V-JEPA(動画版)との違い

V-JEPA(Bardes et al., 2024, arXiv:2404.08471)は同じ枠組みを動画に拡張し、時空間の隠した領域を潜在空間で予測します。動画は連続フレームをチューブ状(時間×空間)のトークンに分けて扱います。実装上の重要な違いが2つあります。

- マスクの取り方: 単一フレーム内の一部ではなく、複数フレームにまたがる時空間ブロックを隠す。「動きを予測しないと埋められない」マスクにすることで、時間方向の構造(動き・力学)を学ばせます。short-range と long-range の2種のマスクを混ぜるのが標準。
- 損失を L2 から L1(絶対値の平均)に変更: L1 は外れ値に対して頑健(robust)で、動画のように変動が大きいデータで学習が安定しやすい、という経験的知見によります。L2 は誤差を二乗するので大きな外れ値に過剰反応しますが、L1 は線形に効くため一部の予測困難な領域に引きずられにくい。

V-JEPA 2(Bardes et al., 2025, arXiv:2506.09985)はさらにスケールを拡大(報告で 100 万時間超の動画で事前学習)し、ロボット操作の行動条件付き予測(action-conditioned、V-JEPA 2-AC)にまで踏み込みました。これは「世界モデルとして使える表現」という方向で、自動運転の未来予測タスクと地続きの発展です。

### 自動運転の未来予測への移植

JEPA の核心部品は2つです。1つは frozen / EMA な target エンコーダ(勾配遮断つき)、もう1つは予測と正解の距離を測る損失で L1/L2/smooth-L1(Huber)を選べるようにすること。これらをそのまま「現在の特徴から数秒後の特徴を潜在空間で予測する」運転タスクに移植できます。実装上は、target 側を勾配計算対象から外し(`requires_grad_(False)`)かつ評価モード(`eval()`)にし、forward を `torch.no_grad()` 下で実行し出力を `.detach()` することで、「どう損失を組んでも target 側に勾配は流れない」ことをコードレベルで保証します。

```python
# 形状例: 予測も正解も num_future_steps 個の [B, C, H, W]
if loss_type == "l1":
    per_step = (pred - target).abs().mean()      # V-JEPA 流
elif loss_type == "smooth_l1":
    per_step = F.smooth_l1_loss(pred, target)    # Huber、外れ値に頑健
else:  # l2
    per_step = ((pred - target) ** 2).mean()     # I-JEPA 流
```

## 手法の系譜と主要論文

時系列で系譜を描きます。

- 2020 SimCLR / MoCo(対比学習): ネガティブサンプルで近い・遠いを学ぶ。強いが大量のネガティブが要る。
- 2020 BYOL(Grill et al., NeurIPS 2020, arXiv:2006.07733): ネガティブ無しでも、online と EMA target の非対称性だけで崩壊を防げると実証。JEPA 非対称設計の直接の先祖。
- 2021 DINO(Caron et al., ICCV 2021, arXiv:2104.14294): 自己蒸留と中心化(centering+sharpening)で崩壊回避、注意マップが物体を自然に切り出すことを示す。
- 2021 VICReg(Bardes et al., ICLR 2022, arXiv:2105.04906): 分散(variance)・不変性(invariance)・共分散(covariance)の3項正則化で、EMA や stop-gradient に頼らず崩壊を明示的に防ぐ別系統。
- 2022 MAE(He et al., CVPR 2022, arXiv:2111.06377): ピクセル再構成で大規模スケールに成功。生成系の到達点。JEPA はこれの「ピクセルではなく特徴を」という対抗提案。
- 2022 LeCun "A Path Towards Autonomous Machine Intelligence": JEPA という用語と「世界モデルは潜在空間で予測すべき」という哲学を提示した position paper。
- 2023 I-JEPA(Assran et al., CVPR 2023, arXiv:2301.08243): 画像で潜在予測を実証。MAE より計算効率良く強い表現。
- 2024 V-JEPA(Bardes et al., arXiv:2404.08471): 動画版、L1 損失、時空間マスク。
- 2025 V-JEPA 2(Bardes et al., arXiv:2506.09985): 大規模化と行動条件付き予測、ロボットへ。

## 論文の実験結果(定量データ)

評価指標の意味から説明します。SSL の表現の良し悪しは「linear probing(線形評価)」で測るのが定番です。これは、事前学習したエンコーダを凍結し、その出力の上に線形分類器1層だけを足して下流タスク(例: ImageNet 画像分類)で学習し、精度を見る方法です。エンコーダ自体は更新しないので、「表現にどれだけ役立つ情報が整理されて入っているか」を直接測れます。Top-1 精度とは「最も確信度の高い1つの予測が正解と一致する割合」です。

- I-JEPA: ImageNet-1k の linear probing で、ViT-Huge/14 を用い、おおむね 70% 台後半の Top-1 精度を、MAE など生成系より少ない学習計算量(同等精度に到達するための事前学習エポック・GPU 時間)で達成したと報告されています。論文の主張の核は「ピクセル予測より大幅に学習効率が良い(報告では同等精度への到達が数倍速い水準)」という効率面で、これは「予測不能な細部を学ばないので無駄が少ない」ことの定量的裏付けです。色変換やランダムクロップの細かい設計(hand-crafted なデータ拡張)に頼らず良い表現が得られる点も特徴です。
- V-JEPA: Kinetics-400(動画行動認識)や Something-Something-v2(時間的順序が重要な行動認識)で、フローズン評価(エンコーダ固定+軽い attentive probing ヘッド)において、ピクセル再構成ベースの動画モデルを上回ると報告。特に「時間的な順序を入れ替えると意味が変わる」Something-Something のような時間敏感タスクで強く(報告で 70% 前後の精度)、潜在予測が動き・力学を捉えていることを示します。画像由来の SSL(画像をフレーム単位で見るモデル)が時間を扱えず Something-Something で大きく劣るのと対照的です。
- V-JEPA 2: 行動条件付き版(V-JEPA 2-AC)が、わずかなロボット操作データで微調整したのち、未知の環境で pick-and-place(つかんで置く)タスクをゼロショット計画で実行できると報告。世界モデルの表現が下流の制御に直接転用できる証拠です。
- アブレーション(どの要素を抜くとどうなるか): I-JEPA の論文では、(a) target を EMA ではなく online と同じに(=勾配を流す)すると表現が崩壊して精度が大きく落ちる、(b) マスクするターゲットブロックを大きく・複数にするほど(=予測課題を難しくするほど)表現が良くなる、(c) 予測対象をピクセルに戻すと細部に引っ張られ下流精度が下がる、ことが示されています。これは「非対称性」「十分難しい予測課題」「潜在空間予測」の3つが本質的だという証拠です。V-JEPA では L2 を L1 に変えると学習が安定し下流性能が改善したと報告され、損失関数の選択が無視できない要素であることがわかります。

## メリット・トレードオフ・限界

メリット:
- 人手ラベル不要。ラベル付けコストなしで良い特徴を学べる。
- 潜在空間予測で、予測不能な細部を学ばず計算も軽い(I-JEPA の効率優位)。
- 対比学習と違いネガティブサンプル不要で、バッチサイズや拡張設計への依存が小さい。
- frozen/EMA target で表現崩壊を防げる(BYOL 以来の理論的裏付けあり)。

トレードオフ・限界:
- target エンコーダの設計(frozen か EMA か、EMA の τ の値とスケジュール)が結果を左右し、調整が要る。
- 学習対象が抽象表現なので、学習がうまくいっているかの可視化・解釈が難しい(再構成系はピクセルを見れば一目瞭然なのと対照的)。学習の成否は下流タスクの評価まで待たないと分かりにくい。
- frozen は実装が単純だが、オンライン側が成長すると target が相対的に古びて学習信号が弱くなりうる。EMA は新鮮さを保てるが τ 調整が必要。
- 元来は大規模事前学習(pretraining)の手法であり、下流タスクと同時に補助損失として学習する使い方では、補助損失の重みや収束タイミングに依存して効果が文脈依存になりうる。
- 崩壊が起きていても損失は下がる(定数出力でも損失ゼロ)ため、損失曲線だけを見て成功と誤判断する危険がある。表現の分散・特徴のランクを監視するのが実務的な対策。

## 発展トピック・研究の最前線

- 行動条件付き世界モデル: V-JEPA 2 が示したように、潜在予測を「行動を入力に取る」形に拡張すると、計画やロボット制御に使える世界モデルになります。自動運転で「軌道Aを取ったら世界はどう変わるか」を潜在空間で予測する反実仮想ロールアウトはこの延長です。詳細は [未来状態モジュールの設定可能ホライズンと残差(TDV)予測器](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1040_future-state-tdv-residual-predictor)。
- 崩壊回避の理論: VICReg(分散・共分散の正則化で明示的に崩壊を防ぐ)を JEPA の非対称設計と対比して理解すると、崩壊回避の設計空間(暗黙の非対称か明示の正則化か)が見えます。
- 予測空間の選び方: 何を予測対象に置くか(全チャンネルか、圧縮した意味チャンネルか、占有度のようなスカラーか)が下流性能を大きく左右します。詳細は [予測空間の圧縮(forecast space を切り替える調整ノブ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1036_feature-compressor-forecast-space)。
- マルチモーダル JEPA: 画像・動画だけでなく音声・点群・テキストにも JEPA を広げ、共通の潜在空間で複数モダリティを予測し合う方向が探られています。
