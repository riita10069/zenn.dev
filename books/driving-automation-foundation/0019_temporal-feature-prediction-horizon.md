# 多ステップ未来特徴予測と予測ホライズン設計

## ひとことで言うと
多ステップ未来特徴予測とは、現在のシーン特徴から「複数の未来時刻(たとえば +1.6 秒, +3.2 秒, +4.8 秒, +6.4 秒)の特徴」を一度にまとめて予測する設計です。ここで予測するのは生のピクセルではなく、画像をニューラルネットに通して得た意味のベクトル(特徴)です。鍵となる設計判断は二つあります。どれだけ先まで・どれくらいの間隔で予測するか(予測ホライズン horizon の設計)と、近い未来と遠い未来の予測誤差にどう重みを付けるか(時間重み付け損失 temporal weighting)です。このノートはその設計の勘所を、分野共通の一般論として説明します。

## 直感的な理解

未来を 1 ステップだけ予測しても運転には足りません。たとえば「0.1 秒後どうなるか」だけ分かっても、ブレーキやレーンチェンジの計画には使えません。0.1 秒では車はほとんど動いておらず、判断のための「先読み」になっていないからです。逆に、遠すぎる未来(30 秒後)は不確実性が大きすぎて当たりません。前の車が次の交差点で何をするかは、今からは原理的に決まっていないからです。遠い未来を無理に当てようとすると、モデルは「平均的でぼやけた予測」に逃げ、かえって役に立たなくなります。

ちょうど良い「窓」があります。運転判断に効くのは数秒先です。「前の車がブレーキを踏みそう」「歩行者が横断歩道に入りそう」「合流車が割り込みそう」といった事象は、おおむね数秒のスケールで展開します。人間のドライバーも、無意識にこの「数秒先」を読みながらアクセルとブレーキを調整しています。そこで「数秒先までを、適度な間隔で、複数時刻ぶん予測する」のが多ステップ未来特徴予測です。

「どれだけ先まで」「どの間隔で」を決めるのが予測ホライズンの設計で、これは計算コスト・予測精度・運転への有用性のバランスを取る、世界モデル設計の核心的な意思決定です。間隔を細かくすれば滑らかで精密な未来が得られますが計算が爆発し、粗くすれば軽いが急な事象を取りこぼします。この綱引きの落としどころを決めるのが設計者の腕です。

## 基礎: 前提となる概念

特徴(feature)とは、画像をニューラルネット(バックボーンと呼ばれる大きなネットワーク。CNN や Vision Transformer など)に通して得た意味のベクトルです。ここで予測するのは生のピクセル(色や明るさそのもの)ではなく特徴です。理由は [JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note) を参照してください。要するに、反射やノイズや葉の揺れのような運転に無関係な細部まで当てる負担を避け、本質的な意味だけを予測するためです。

BEV(Bird's-Eye View, 鳥瞰図)特徴とは、複数カメラや LiDAR の情報を「上から見た地図」の形に変換した特徴マップです。形は [B, C, H, W](バッチ B、チャンネル C、高さ H、幅 W)で、H×W が地面のグリッド、C が各セルの意味ベクトルです。未来予測を BEV 空間で行うと、車や歩行者の「位置の動き」を素直に表現できます。地面平面で考えるので、画像座標での見かけの大きさ変化(遠近)に惑わされず、メートル単位の動きを扱えるのが利点です。

予測ホライズン(prediction horizon)とは「どれだけ先まで予測するか」の総時間です。これを刻み(time step interval)とステップ数(number of steps)で構成します。ホライズン = 刻み × ステップ数、という関係です。たとえば刻み 1.6 秒 × 4 ステップ = 6.4 秒のホライズンになります。

ロールアウト(rollout)とは、未来を順に展開すること、すなわち「1 ステップ先を予測 → その予測を入力にしてさらに 1 ステップ先を予測 → …」と再帰的に進める方式です。これに対し「1 回の forward で複数時刻を同時に出す」並列予測方式もあり、両者にはトレードオフがあります(後述)。

行動条件付き予測(action-conditioned prediction)とは、「自分がこれからどう動くつもりか」を予測の条件に入れることです。これにより「単なる未来予測」ではなく「この計画をとったらどうなるか」という世界モデルになります。

## 仕組みを詳しく

代表的な実装では、現在のシーン特徴と「自分がこの先どう動くつもりか」を表す計画ベクトル(自車の意図、ego intent)を条件に、複数時刻の未来特徴を生成します。処理を tensor の形(shape)つきで段階分解します。

入力
- 現在のシーン特徴 fused_features: [B, C, H, W]。C は埋め込み次元(たとえば 256)。複数センサを融合して得た「今のシーンの意味」。
- 計画意図 ego_hidden: [B, 256]。軌道計画器(たとえば GRU などの系列モデル)の最終隠れ状態で、「この先どう走るつもりか」を 1 本のベクトルに要約したもの。

ステップ1: 条件付け(conditioning)
ego_hidden を Linear(256→256) で射影し、[B, 256, 1, 1] に変形して fused_features に空間方向へブロードキャストして足します。これで「計画した走り方」がシーン特徴に空間一様なバイアスとして注入されます。

```
bias = Linear(ego_hidden).view(B, 256, 1, 1)
conditioned = fused_features + bias        # [B, 256, H, W]
```

このように action(計画)を条件に入れることで、単なる未来予測ではなく「この計画をとったらどうなるか」という条件付き世界モデルになります(action-conditioned prediction)。空間一様に足すのは最も単純な条件付けで、より凝った実装では FiLM(特徴をスケール・シフトする変調)や空間ごとに異なる条件を入れる方式も使われます。

ステップ2: 複数時刻の未来特徴をまとめて生成
畳み込み層を重ね、チャンネル数を増やします。4 ステップ予測なら 256 → 512 → 1024(= 4 × 256)。

```
x = Conv2d(256 -> 512)(conditioned); x = GELU(x)
x = Conv2d(512 -> 1024)(x)          # [B, 1024, H, W]
```

ステップ3: チャンネル方向に分割
チャンネルを 4 等分し、4 枚の未来特徴マップにします。

```
future = chunk(x, 4, dim=1)
# future[0]: +1.6s [B, 256, H, W]
# future[1]: +3.2s [B, 256, H, W]
# future[2]: +4.8s [B, 256, H, W]
# future[3]: +6.4s [B, 256, H, W]
```

ここで重要なのは「4 ステップを逐次ロールアウトせず、1 回の forward で 4 枚を同時に出す」点です。実装がシンプルで速い反面、ステップ間の因果依存(+3.2 秒の状態が +1.6 秒の状態に基づく、という連鎖)は明示的にはモデル化されません。これは設計上のトレードオフです(後述)。

なぜ刻みを 1.6 秒のように粗く取るのか。仮に 0.1 秒刻みで 6.4 秒先まで予測すると 64 ステップ必要で、計算もメモリも爆発します。1.6 秒という粗い刻みなら 4 ステップで 6.4 秒に届きます。遠い未来は本質的に不確実なので、そこを細かく刻んでも無駄が大きく、粗い刻みで「おおまかな未来」を捉えるほうが理にかなっています。さらに、未来特徴予測のホライズン(6.4 秒)を軌道計画器の計画ホライズンと一致させると、「計画した 6.4 秒の走りに沿った未来特徴」を一貫して予測でき、計画と予測が噛み合います。

### 時間重み付け損失(temporal weighting)

正解特徴は凍結ターゲットエンコーダ([凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))が未来フレームから作ります。損失は各時刻ごとに予測特徴と正解特徴の MSE(Mean Squared Error, 二乗誤差)を取り、時刻ごとの重み(temporal weights)を掛けて合算します。

```
per_step = [ mean((pred_t - target_t)^2) for t in steps ]   # 長さ4
weighted = per_step * temporal_weights                        # 要素ごと
loss     = mean(weighted)
```

temporal_weights は時刻ごとに損失の効き具合を変えるベクトルです。

- [1, 1, 1, 1]: 全ステップ平等(デフォルト)。
- [4, 3, 2, 1]: 近い未来を重視。すぐ先を当てることを優先し、遠い未来は緩く扱う。遠い未来ほど不確実で当てにくく、それに学習が引きずられて不安定になるのを防げる。
- [1, 2, 3, 4]: 遠い未来を重視。長期予測能力を伸ばしたいとき。

実装上の大事な工夫が重みの正規化です。

```
total = sum(weights)
if abs(total) > 1e-8:
    weights = weights / total * num_steps   # 合計を num_steps に揃える
```

たとえば [4,3,2,1] は合計 10 ですが、各要素に 4/10 を掛けて合計 4(= ステップ数)に揃えます。なぜか。正規化しないと、重みを大きくしただけで損失の絶対値が変わり、他の損失(軌道模倣損失など)とのバランスが崩れます。学習は複数の損失の和を最小化するので、片方の損失だけ絶対値が膨らむと、そちらに引っ張られて全体の最適化が歪みます。正規化すれば「重みは配分だけを変え、全体スケールは素の MSE と同じに保つ」ことができます。合計がほぼゼロになる退化ケースで割り算が NaN(非数)になるのを防ぐガード(abs(total) > 1e-8)も入れます。reduction を "none"(平均しない)にすれば各時刻の損失を個別に返せ、「+1.6 秒は当たるが +6.4 秒が外れる」といった時刻別の挙動を可視化できます。

## 手法の系譜と主要論文

特徴空間での未来予測という発想は、Yann LeCun の構想論文「A Path Towards Autonomous Machine Intelligence」(2022)が土台です([JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))。生成的にピクセルを当てるのではなく、特徴空間で未来を予測する世界モデルが知能の中核だ、という主張です。

時間方向の特徴予測を動画で具体化したのが Bardes らの V-JEPA(Meta AI, 2024)です。動画から複数フレーム先の特徴を予測する自己教師あり学習で、「時間方向の予測が自然な教師信号になる」ことを示しました。多ステップ未来特徴予測はこの方向に最も近い系譜です。続く V-JEPA 2(2025)では大規模動画事前学習に行動条件付き予測を接続し、ロボット計画への応用まで広げました。

自動運転に未来予測を持ち込んだ代表が Hu らの MILE(Model-based Imitation LEarning, NeurIPS 2022)です。BEV 空間で未来の状態を潜在的に予測しながら運転方策を学びます。「未来をモデル化すると模倣学習がデータ効率良くなる」という動機で、後述のように CARLA で強い性能を示しました。先行の Hu らの FIERY(ICCV 2021)は BEV 上で将来のインスタンスセグメンテーション(誰がどこに動くか)を確率的に予測し、未来を BEV で扱う流れを作りました。これらは BEV 特徴の未来予測という構造を共有します。

時間重み付けの発想は軌道予測(trajectory prediction)の分野と共通します。Alahi らの Social LSTM(CVPR 2016)以降、歩行者・車両の軌道予測では予測軌道と正解軌道のズレを displacement error(変位誤差)で測りますが、遠い時刻ほど誤差が大きくなるのが自然なので、時刻で重み付けする/遠い時刻の許容を緩める、という扱いが一般的になりました。複数時刻の予測を時刻ごとに重みづけて学習する、という発想を共有します。

これらを並べると、多ステップ未来特徴予測は「JEPA 系の特徴空間予測」「BEV ベースの運転世界モデル(MILE / FIERY)」「軌道予測の時刻重み付け」を組み合わせた設計だと整理できます。

## 論文の実験結果(定量データ)

MILE(Hu et al., NeurIPS 2022)の結果を、指標の意味とともに見ます。評価は CARLA(オープンソースの運転シミュレータ)で行われます。

主指標は Driving Score(運転スコア)で、これはルート完走率(どれだけ目的地に近づけたか)に違反ペナルティ(信号無視・衝突・逸脱など)を掛け合わせた総合点です。0〜100 で高いほど良く、「安全に目的地まで走れたか」を一本化した指標です。報告によると MILE は、当時の代表的手法(画像から直接行動を出す模倣学習や、明示的な中間表現を使う手法)を上回る Driving Score を達成し、とくに新しい町・新しい天候(学習時に見ていない条件)への汎化で優位を示しました。「未来を潜在的にモデル化する」補助が、見たことのない状況での頑健さに効いたと解釈されます。

アブレーション(ある要素を抜くと性能がどう変わるかの検証)で重要なのは、未来予測(world model としての潜在ロールアウト)を外すと Driving Score が低下する、という結果です。これは「未来予測タスクが運転方策学習の有用な補助信号になっている」ことを定量的に示します。逆に言えば、未来予測は最終目的(運転)そのものではなく、表現を良くするための補助タスクとして効いている、という位置づけが読み取れます。

FIERY(Hu et al., ICCV 2021)は nuScenes(都市運転の大規模データセット、約 1000 シーン)で、将来 2 秒程度の BEV インスタンス予測を評価し、Future Video Panoptic Quality(将来の各物体を時間的に一貫して当てられたかを測る指標。高いほど良い)で、点群を使わずカメラのみから既存手法を上回ると報告しました。カメラだけで未来の動きを BEV 上に予測できることを示した点が新規でした。

V-JEPA(Bardes et al., 2024)は動画で評価され、Kinetics-400(400 種の動作分類)や Something-Something v2(物体操作の動作認識)で、凍結エンコーダ + 軽量分類器の評価(エンコーダを固定したまま小さな分類器だけ学習して測る)において、ピクセル予測系の動画自己教師あり手法を上回る、または同等の性能を報告しました。とくに「動きの理解」を要する Something-Something v2 での優位が報告され、時間方向の特徴予測が動的な意味を捉えるのに有効だと示唆されました。

時間重み付けについては、軌道予測の標準指標である minADE / minFDE(複数候補のうち最良の、全時刻平均の変位誤差 / 最終時刻の変位誤差)で、近い時刻の誤差が小さく遠い時刻ほど大きくなるのが一般的であり、これが「遠い時刻の損失を緩める/重み付ける」運用の根拠になっています。たとえば 6 秒予測では最終時刻の FDE が初期時刻の数倍に達するのが普通で、ここを平等に重み付けると学習が遠い時刻の大きな誤差に支配されてしまう、というのが実務上の知見です。逆に、長期予測能力をあえて伸ばしたいベンチマーク設定では、遠い時刻に重みを置く設計が選ばれることもあります。

## メリット・トレードオフ・限界

メリット。粗い刻み(例 1.6 秒)により、少ない計算・メモリで数秒先まで届きます。複数時刻を 1 回の forward で同時生成する並列方式は実装がシンプルで速いです。時間重み付けで近い未来重視・遠い未来重視を柔軟に切り替えられ、正規化により他の損失とのスケールが崩れません。未来予測を補助タスクにすると、本タスク(運転方策)の表現が良くなり汎化が向上する傾向があります(MILE のアブレーション)。

トレードオフと限界。並列予測はステップ間の因果依存を明示的にモデル化しないため、物理的に一貫した連続的な未来(+3.2 秒が +1.6 秒から滑らかにつながる)を保証しにくいです。逐次ロールアウトは因果を表せる反面、誤差が次のステップに伝播して累積し(compounding error, 複利的誤差)、遠い時刻で発散しやすいという別の問題を抱えます。出力チャンネルがステップ数 × 次元(例 4 × 256 = 1024)と大きく、BEV を高解像度にするとメモリを大量消費します(高解像度では bf16 mixed precision や空間ダウンサンプルが事実上必須)。粗い刻みは、それより短い時間スケールの急な事象(直前の飛び出し)を捉えにくいです。研究上の未解決課題として、最適なホライズン・刻み・時間重みが運転状況(高速か市街地か)で変わるため固定値が最適とは限らない点、並列予測と逐次ロールアウトの中間(短い連鎖を並列に組む)設計の探索が挙げられます。

## 発展トピック・研究の最前線

未来予測を本格的な世界モデルへ発展させる方向が活発です。潜在空間でロールアウトして「複数の計画候補それぞれの未来」を予測し、最も安全な計画を選ぶ(Model Predictive Control, 予測しながら制御を決める枠組み的な利用)構成や、確率的予測(未来は一つに決まらないので複数の可能性を分布として出す)への拡張があります。階層化([階層JEPA (H-JEPA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0058_hierarchical-jepa))で、短い刻みの細かい予測と長い刻みの粗い予測を別レイヤに分けると、長期計画と短期制御を同時に扱えます。マルチモーダル(カメラ+LiDAR+地図)での未来特徴予測、occupancy(各セルが将来占有されるかどうか)の未来予測、言語条件付き(「左折する」という指示を条件にした未来)など、入力・出力の拡張も進んでいます。GAIA-1(Wayve, 2023)や DriveDreamer(2023)のような生成型運転世界モデルも登場し、特徴空間予測と生成的予測の使い分けが議論されています。
