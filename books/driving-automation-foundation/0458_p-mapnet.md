# P-MapNet

## ひとことで言うと
カーナビに入っているような大雑把な地図(SD 地図)と、過去に作った高精細地図(HD 地図)の両方を「事前知識(prior)」としてニューラルネットに与え、遠くまで正確な地図をその場で作れるようにした手法です。SD 地図は注意機構で位置のヒントとして融合し、HD 地図はマスクオートエンコーダ(画像の一部を隠して復元させる学習で「形の常識」を覚えたネットワーク)で地図の形を整えるのに使います。特に、車から遠い場所(センサーがよく見えない範囲)の地図精度を上げることに強みがあります。prior をどう安全に取り込むかを明示的に設計した好例です。

## 直感的な理解
あなたが見知らぬ街を歩いていて、霧で 30m 先がよく見えないとします。それでも、手元のスマホのカーナビを見れば「この先は道が右に曲がっていて、片側 2 車線らしい」と分かります。霧で見えない部分を、大雑把だけど信頼できる地図情報で補えるのです。さらに、もし前にこの街を歩いたことがあれば「確かこの辺は車線がこう通っていた」という記憶も使えます。

自動運転の「走りながら地図を作る」技術にも、まったく同じ「遠くが見えない」問題があります。カメラや LiDAR は近くはよく見えますが、30m や 50m 先になると解像度が落ち、白線や物が霞みます。そのため遠距離の地図要素は途切れたり、形がぐにゃぐにゃになったりします。

P-MapNet は、この「見えない遠く」を 2 種類の事前知識で補います。1 つは霧の中のカーナビにあたる SD 地図(大雑把だが広域)、もう 1 つは過去の記憶にあたる HD 地図 prior です。ただし、これらの prior は不正確だったり古かったりするので、そのまま信じると害になります。P-MapNet の工夫は「どこまで prior を信じ、どこからセンサーを信じるか」を学習で調整する点にあります。論文タイトルの "Far-seeing(遠くを見る)" は、まさに遠距離の地図を当てにいくという狙いを表しています。

## 基礎: 前提となる概念

いくつかの言葉を噛み砕きます。

「オンライン HD マップ構築」とは、走行中の車載センサー映像から、その場で高精細地図(車線・境界などの位置を細かく記録した地図)を作る技術です。事前地図の高い作成・更新コストを避けるために生まれました。

「prior(プライア、事前知識)」とは「センサーを見る前から分かっている情報」です。ベイズ的に言えば、観測(センサー)を見る前の信念分布にあたります。P-MapNet は 2 種類の prior を使います。

1. SD 地図(Standard-Definition map、標準精細地図): OpenStreetMap やカーナビの地図がこれです。「この道は片側 2 車線」「ここで交差点」程度の大雑把な情報(道路のセンターラインのトポロジーと種別)しかありませんが、世界中ほぼ全域を無料に近いコストでカバーしています。センサーが見えない遠くでも「この先に道があり、だいたいこの向き」と教えてくれます。

2. HD 地図 prior: 同じ場所を以前に走って作った高精細地図、あるいは大量の HD 地図から学んだ「地図一般の形の常識」です。完璧に最新ではないかもしれませんが、「車線・交差点とはこういう形をしている」という強力なヒントです。

「BEV(Bird's-Eye-View、鳥瞰図)特徴」とは、複数カメラ画像(と LiDAR)を地面の俯瞰格子に貼り直した特徴マップ `[C, H, W]` です。

「ラスタ化(rasterize)」とは、線で書かれた地図(ベクトル)を格子の塗り絵(ラスタ)に変換することです。例えば「道路がある格子は 1、ない格子は 0」のマップ `[1, H, W]` を作ります。SD 地図のセンターラインを BEV 格子に焼き込むのに使います。

「注意機構(attention)」とは、「どの情報をどれだけ重視するか」を入力に応じて動的に決める仕組みです。Transformer の中核技術で、クエリ(問い合わせ)・キー(鍵)・バリュー(値)の内積で重みを計算します。「この部分は SD 地図を信じ、あの部分はセンサーを信じる」といった重みづけを学習で獲得できます。

「マスクオートエンコーダ(MAE、Masked Autoencoder)」とは、入力の一部を隠して、隠した部分を復元するよう訓練したネットワークです。復元を通じて「対象とはこういうものだ」という常識(事前分布)を学びます。He et al.(CVPR 2022)が画像で提案し、入力パッチの 75% 程度を隠しても残り 25% から元画像を復元できることを示して、自己教師あり学習(ラベルなしで学べる学習)の代表になりました。

「prior の不確かさ」とは、prior が当てにならない度合いです。SD 地図はセンサーと位置がずれていることがあります(GPS 誤差で道路が数メートルずれて見える、地図の道路中心が実際の車線中心とずれている)。HD 地図 prior も工事で変わっているかもしれません。これをそのまま信じると逆に害になるため、不確かさへの対処が P-MapNet の中心テーマです。

## 仕組みを詳しく

P-MapNet の核心は 2 つの prior を別々の方法で取り込み、最後にマスクオートエンコーダで地図全体を「もっともらしい形」に整えることです。順を追います。

ステップ1: ベースとなるオンライン地図構築。
ベースは HDMapNet 系(BEV セグメンテーション)です。マルチカメラ(と必要なら LiDAR)から BEV 特徴 `[C, H, W]`(鳥瞰図格子の各マスに C 次元特徴)を作り、地図要素のラスタ/ベクトルを予測します。これだけだと前述のとおり遠距離が弱い。

ステップ2: SD 地図 prior の融合(位置のヒント)。
SD 地図(OpenStreetMap など)を、車の現在位置に合わせて BEV 格子の上にラスタ化します。「道路の中心線がある格子は 1」のようなマップ `[1, H, W]`(あるいは道路種別ごとのチャンネル)を作り、これをセンサー由来の BEV 特徴と結合(concatenate)します。さらに重要な工夫として、cross-attention(センサー特徴をクエリ、SD 地図特徴をキー・バリューとする注意)で融合します。SD 地図がずれている場合でも、attention が「この部分は SD 地図よりセンサーを信じる」と空間的に重みを調整できるようにしています。これが「ずれた prior を安全に使う」ための鍵です。単純に結合するだけだと、ずれた SD 地図がそのままノイズになってしまいますが、attention が信頼度を学習することで害を抑えます。

ステップ3: HD 地図 prior を MAE で取り込む(形の常識)。
ここが P-MapNet の独自性が最も出る部分です。大量の HD 地図データ(ラスタ化した地図)で「地図の一部を隠して残りから復元する」MAE 事前学習をします。すると MAE は「車線はこういうふうにつながる」「交差点はこういう形になる」という地図の常識を覚えます。推論時、オンラインで作った地図(遠距離が途切れている、ノイズが乗っている)を MAE に通すと、MAE が「この途切れ方は不自然。本来こうつながるはず」と、もっともらしい形に補完・整形(refine)してくれます。MAE は「欠損を埋める」性質を持つので、センサーで見えなかった遠距離の途切れを、学習した地図の常識で自然につなぎます。MAE はここでは生成器ではなく、粗い地図を入力にきれいな地図を出す「地図用のデノイザ/補完器」として転用されています。

ステップ4: 出力。
SD prior で位置のヒントを得て、MAE で形の常識を補った地図を最終出力します。結果として、特に遠距離(センサーが弱い範囲)で、途切れの少ない滑らかな地図が得られます。

役割分担を整理すると、SD prior は「遠距離で道がどこにあるか(位置・トポロジー)」に効き、MAE は「地図の形が自然か(形状・連続性)」に効きます。この 2 つを分けて設計したのが P-MapNet の特徴です。位置の prior(外から与える)と形状の prior(データから学ぶ)を分離した、と言い換えられます。

## 手法の系譜と主要論文

P-MapNet は 2 つの研究の流れの交点にあります。

1 つはオンライン地図構築の流れです。HDMapNet(Qi Li, Yue Wang, Yilun Wang, Hang Zhao, ICRA 2022, arXiv:2107.06307)がこの問題設定を定式化し、BEV セグメンテーション + 後処理ベクトル化の枠組みを作りました(詳細は [InstaGraM / インスタンスマスク地図](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0450_maplt-tr-instance-mask))。P-MapNet はこの HDMapNet をベースに採用しています。その後 MapTR(Liao et al., ICLR 2023, arXiv:2208.14437)/ PivotNet / StreamMapNet と発展しましたが、これらに共通する弱点が「遠距離が弱い」ことで、P-MapNet はここに prior で切り込みました。多くの先行研究が近距離(±30m 程度)の評価に閉じていたのに対し、P-MapNet は評価範囲そのものを遠距離へ広げて問題提起した点も重要です。

もう 1 つは自己教師あり表現学習の流れです。MAE(Kaiming He, Xinlei Chen, Saining Xie, Yanghao Li, Piotr Dollár, Ross Girshick, CVPR 2022, arXiv:2111.06377)は、画像パッチの 75% 程度をランダムに隠し、残りから元画像を復元する訓練で、ラベルなしの大量画像から強力な特徴表現を効率よく学べることを示しました。エンコーダは見えるパッチだけ処理するので計算が軽く、「スケーラブルな学習」として大きな影響を与えました。P-MapNet はこの MAE の「欠損を補う」性質を、画像表現学習ではなく地図補完という全く別の用途に転用した点が独創的です。MAE のトレードオフは、復元タスクが「平均的にもっともらしい」答えを出しがちで、珍しい形を苦手とすることで、これは P-MapNet にもそのまま影響します。

P-MapNet(Zhou Jiang, Zhenxin Zhu, Pengfei Li, Huan-ang Gao, Tianyuan Yuan, Yongliang Shi, Hang Zhao, Hao Zhao, RA-L 2024, "P-MapNet: Far-seeing Map Generator Enhanced by both SDMap and HDMap Priors", arXiv:2403.10521)は、SD 地図 prior(attention 融合)と HD 地図 prior(MAE による形の補完)を組み合わせ、遠距離地図構築(far-seeing)を改善しました。新規性は、2 種の prior をそれぞれの不確かさに対応した別々の方法で取り込んだ点です。SD-prior 活用の系統(SD 地図をベクトルのトポロジーとして与える手法や、neural map prior 系のように過去走行で作った特徴地図を蓄積する手法 [Neural Map Prior (NMP)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0453_neural-map-prior) や [SD-map prior融合 (MapEX等)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0449_mapex-mapfusion-sd-prior) など)の中でも、prior の不確かさへの対処を明示的に設計した代表例です。

## 論文の実験結果(定量データ)

評価は nuScenes と Argoverse 2 で行われました。評価指標は IoU(Intersection over Union、予測領域と正解領域の重なり率)と mAP(mean Average Precision、ベクトル精度)です。P-MapNet の鍵となるのは「予測範囲を遠距離まで広げて評価した」点です。

論文では、近距離(例として 60m × 30m 相当の範囲)だけでなく遠距離(120m × 60m 相当)まで予測範囲を広げて評価しました。報告によると、近距離では既存手法との差は小さいものの、遠距離になるほど P-MapNet の優位が大きくなり、IoU や mAP が顕著に改善しました。遠距離設定では既存ベースラインに対して二桁(数十%)規模の IoU 改善が報告されています。これは「遠距離こそ prior が効く」という設計の狙いどおりの結果です。センサーがよく見える近距離では prior の助けはあまり要らず、見えない遠距離で prior の価値が出る、という直感と一致します。

アブレーション(構成要素を分解する実験)が特に示唆に富みます。報告によると、(1) SD prior だけを足すと、遠距離での位置精度(道がどこにあるか)が向上する、(2) MAE(HD prior)だけを足すと、地図の形の自然さ(途切れの少なさ、連続性、ノイズの除去)が向上する、(3) 両方を併用すると最も良い、という結果でした。つまり SD prior と MAE は別々の側面(位置/形)に効いており、互いに代替できない補完的な要素だということが定量的に分解して示されました。これは「prior の取り込みは 1 つの方法では足りず、位置と形を別々に扱うべき」という設計指針を裏づけます。

また、SD 地図がずれている条件での頑健性も検証されています。SD 地図の位置に人工的な誤差(ノイズ)を加える実験で、attention 融合のおかげで誤差がある程度大きくても性能の劣化が緩やかであることが報告されており、「ずれた prior を安全に使う」という主張が裏づけられています。単純な concatenation 融合と比べて、attention 融合の方がずれに頑健だという比較も示されています。

## メリット・トレードオフ・限界

メリット
- 安価で広域な SD 地図を活用し、センサーが弱い遠距離の地図精度を底上げできる。遠距離ほど効果が大きい。
- MAE による形の常識で、途切れた・ノイズの乗った地図を自然な形に補完できる。
- attention 融合により、prior がずれていても害を抑えつつ使える。位置誤差への頑健性がある。
- prior の取り込みを「位置(SD)」と「形(MAE)」に分離して設計した点が、再利用しやすい一般的な設計指針になる。

トレードオフ・限界
- SD 地図の入手・位置合わせ(GPS との対応付け)が前提で、GPS 誤差や地図の古さに弱さが残る。prior が極端にずれている領域では attention でも吸収しきれないことがある。
- MAE の事前学習に大量の HD 地図データが必要で、データ整備コストがかかる。地域や国が変わると地図の常識も変わるため、ある地域で学んだ MAE が別地域で当てはまるか(汎化)に注意が要る。
- MAE の補完は「平均的にもっともらしい」方向に働くため、本当に特殊な道路形状(変則交差点、ラウンドアバウト、複雑な合流)を均してしまう・正解と違う形を自信を持って描いてしまうリスクがある。珍しい形を苦手とする MAE の弱点をそのまま受け継ぐ。
- ベースが HDMapNet 系のセグメンテーションで、最終精度がラスタ解像度に縛られる面がある。最新の純ベクトル手法と表現の細かさで差が出うる。

研究上の未解決課題として、prior の鮮度(古さ)の扱いがあります。HD 地図 prior が工事で変わった場合、古い prior は害になりますが、新旧を区別して更新する仕組みは限定的です。また SD 地図と HD 地図の prior を、不確かさの度合いに応じて動的に重みづけする原理的な(ベイズ的に正当化される)枠組みは、まだ発展途上です。「prior をどれだけ信じるか」を、学習で暗黙に決めるのではなく、明示的な不確実性として推定する方向が課題です。

## 発展トピック・研究の最前線

SD 地図 prior の活用は近年活発な研究分野です。SD 地図をラスタとして与えるだけでなく、ベクトル形式のまま(車線のトポロジーや接続関係を保ったまま)エンコードして地図構築の Transformer に注入する研究や、SD 地図と HD マップ構築・トポロジー推定を同時に学習する研究が進んでいます。SD 地図は世界中で無料に近く手に入るため、コスト面で実用価値が高く、現実の自動運転システムへの組み込みが現実的です。

prior と時間融合の組み合わせも進んでいます。SD 地図 prior(空間方向・外部知識の事前知識)と、過去フレームの記憶([StreamMapNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0463_streammapnet) の時間融合)、さらに過去走行で蓄積した特徴地図([Neural Map Prior (NMP)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0453_neural-map-prior))を統合すれば、遠距離・遮蔽・ちらつきに同時に対処できます。さらに、地図 prior を使った地図構築は、変化検出(prior と現在の観測の差から「地図が変わった」場所を見つける)という新しいタスクにもつながっています。古い地図をそのまま信じるのではなく、「どこが変わったか」を能動的に検出して地図を更新する方向です。

P-MapNet が示した「不確かな prior を、その不確かさに対応した方法で(位置と形に分けて、attention で信頼度を学びながら)安全に取り込む」という設計思想は、地図構築に限らず、不完全な事前知識を持つあらゆる認識タスク(過去の地図・他車からの V2X 情報・粗い衛星画像など)で応用される一般的な原理として参照されています。
