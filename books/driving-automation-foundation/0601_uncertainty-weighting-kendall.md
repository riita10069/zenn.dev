# 不確実性重み付けマルチタスク損失 (Kendall)

## ひとことで言うと

一つのネットワークで複数の仕事(タスク)を同時に学ぶとき、「どの仕事の損失をどれくらい重視するか」を手で決めるのは大変です。Kendall らの手法は、各タスクが「どれくらい当てにくいか(観測ノイズ・不確実性)」をネットワーク自身に学ばせ、その不確実性に応じて損失の重みを自動で決めます。重みを試行錯誤するグリッドサーチが要らなくなり、しかも追加パラメータはタスクごとに数値1個だけ、という軽さが魅力です。確率モデルから自然に導かれるため、理論的な裏付けもあります。

## 直感的な理解

自動運転 E2E では、一つのネットワークが「物体検出」「深度推定」「軌道予測」などを同時にこなします。これをマルチタスク学習(multi-task learning, MTL)と呼びます。それぞれの仕事に損失があり、最終的にはそれらを足し合わせた合計損失を最小化します。なぜ別々のネットワークにしないかというと、タスク間で共通する特徴(車の見え方など)を共有でき、計算が軽く、互いの学習を助け合える(正の転移)からです。

ここで悩ましいのが「足し合わせの重み」です。素朴には

```
合計損失 = w1 × 検出損失 + w2 × 深度損失 + w3 × 軌道損失
```

と書きますが、この w1, w2, w3 をどう決めるか。問題は、タスクごとに損失のスケール(値の大きさの単位)が桁違いに異なることです。分類損失は 0〜数程度ですが、深度の回帰損失はメートルの二乗で数百になることもあります。何も考えず足すと、スケールの大きい深度損失に勾配が支配され、ネットワークは深度ばかり最適化し、検出や軌道がおろそかになります。逆に重みを手で下げすぎると、今度はそのタスクが学習されません。

従来はこの w を人間がグリッドサーチ(候補値の総当たり)で探していました。タスクが3つあれば組合せは膨大で、1回の学習に何時間もかかるため、現実には数えるほどしか試せず、最適な重みに当たる保証もありません。タスクが N 個なら探索空間は指数的に膨らみます。Kendall らは「重みを手で探すのをやめ、各タスクの観測ノイズ(そのタスクが本質的にどれくらい当てにくいか)を学習パラメータにして、最適な重み付けを学習の中で自動的に導く」ことを提案しました。

ここでの不確実性とは「そのタスクは原理的にどれくらい当てにくいか」です。単眼カメラの深度推定は本質的に曖昧で不確実性が高く、明るさの分類などは確実性が高い。直感としては「当てにくいタスクの損失は重視しすぎない方が全体がうまくいく」。なぜなら、当てにくいタスクの損失はノイズが多く、それに引きずられると共有表現が荒れてしまうからです。これを確率モデルから数式に落とし込んだのがこの手法です。

## 基礎: 前提となる概念

確率分布の対数尤度(log-likelihood)。あるモデルがデータをどれだけうまく説明するかを表す量で、これを最大化(=負の対数尤度を最小化)するのが確率的な学習の基本です。Kendall の損失はこの負の対数尤度から自然に導かれます。「損失を最小化する」を「データが生成された確率を最大化する」と読み替えるのが鍵です。

ガウス分布(正規分布)。回帰の誤差がガウス分布に従うと仮定すると、モデル出力 f を平均、σ^2 を分散とする p(y|f) = N(y; f, σ^2) の負の対数尤度は「(y-f)^2 / (2σ^2) + (1/2)log σ^2 + 定数」になります。第1項が「重み付き二乗誤差」、第2項が後で効く正則化項です。

二種類の不確実性。Kendall & Gal(NeurIPS 2017)は不確実性を二つに分けました。一つは認識論的不確実性(epistemic、データが足りないことによる、データを増やせば減る不確実性、モデルの無知)。もう一つは偶然的不確実性(aleatoric、データ自体に内在するノイズで、データを増やしても減らない、世界の本質的なゆらぎ)。本手法が使うのは aleatoric のうち、入力に依らずタスク固有に決まる homoscedastic(ホモスケダスティック、等分散)不確実性です。「このタスクはそもそもこれくらい測りにくい」というタスクの性質を表します。対して入力ごとに変わる不確実性は heteroscedastic(ヘテロスケダスティック)と呼び、別扱いです(画像の暗い領域だけ深度が不確実、など)。

正則化(regularization)。学習が極端な方向(ここでは「全タスクをサボる」)に暴走しないよう、損失にペナルティを加える仕組みです。

## 仕組みを詳しく

各タスク i に対して「観測ノイズの標準偏差」を表す学習可能なパラメータ σ_i(シグマ、大きいほど不確実)を用意します。実装上は数値安定性のため log(σ_i^2) を直接パラメータにします(これを s_i と書くことが多い。σ^2 = exp(s) なので必ず正、かつ広い範囲を表せ、勾配も安定します)。

回帰タスクの合計損失は、各タスクをガウス観測モデルとみなした負の対数尤度から、おおむね次の形になります。

```
L_total = Σ_i [ (1 / (2 σ_i^2)) · L_i + log σ_i ]
```

各記号の意味: L_i はタスク i の生の損失(例えば二乗誤差)、σ_i はタスク i の観測ノイズ標準偏差(学習パラメータ)、Σ_i は全タスクの和です。

直感的な読み方。第1項 (1/(2σ_i^2))·L_i は、σ_i が大きい(不確実な)タスクほど L_i にかかる実効重み 1/(2σ_i^2) が小さくなることを意味します。つまり「当てにくいタスクは控えめに重視する」。第2項 log σ_i は正則化です。これがないと、ネットワークは全タスクの σ_i を無限大にして第1項をゼロにし、何も学ばずに損失を下げられてしまいます。log σ_i は σ_i を大きくしすぎると増える罰なので、この暴走(全タスクのサボり)を防ぎます。第1項が σ を大きくしたがり、第2項が小さくしたがる、この綱引きの釣り合い点が「そのタスクの実際の難しさ」を表す σ に落ち着きます。

分類タスクの場合は softmax の温度スケーリング(出力ロジットを σ^2 で割ってから softmax)から導かれ、おおむね (1/σ^2)·L + log σ という似た形になります(係数 2 が付かない近似形を使うことが多い)。考え方は回帰と同じで、温度が高い=不確実なほど分類損失を軽く見ます。

肝心なのは、σ_i も勾配で自動的に学習される点です。人間は w を一切触りません。学習が進むと、当てにくいタスクの σ は自然に大きくなり、その損失の実効重みが下がります。逆に当てやすいタスクの σ は小さく保たれ重視されます。

数値例で示します。深度タスクの生の損失 L_depth が 200、検出タスクの生の損失 L_det が 2 だったとします。素朴な足し算だと深度が100倍支配的です。学習が進むと深度の不確実性が大きく学ばれ、たとえば σ_depth^2 = 100 になると実効重みは 1/(2×100)=0.005、寄与は 200×0.005=1 に下がります。一方検出は σ_det^2=1 程度なら重み 0.5 で寄与 1。こうして両タスクの実効寄与が自動で釣り合います(数値は説明用)。スケールの違いが σ に吸収され、勾配がどれか一つのタスクに支配されなくなるのが効果の本質です。

```
タスク1損失 L1 ─┐
タスク2損失 L2 ─┼→ 各タスクの σ_i(学習パラメータ)で重み付け
タスク3損失 L3 ─┘        │
                  L_total = Σ [ L_i/(2σ_i²) + log σ_i ]
                         │
                  逆伝播で σ_i もネットワークの重みも同時に学習
```

実装は驚くほど軽く、タスク数ぶんの log σ^2 を nn.Parameter として持ち、上式に従って合計損失を組むだけです(数行)。学習後の σ の値そのものが「どのタスクが当てにくいか」の解釈にも使えます。一点の注意として、log σ は負になりうるため、罰項が負に振れて損失全体が負になることもありますが、最適化上は問題ありません(定数差は勾配に影響しない)。

## 手法の系譜と主要論文

マルチタスクの重み付けは、固定重み → 不確実性ベース → 勾配ベース、と発展してきました。

Kendall & Gal "What Uncertainties Do We Need in Bayesian Deep Learning?"(NeurIPS 2017, arXiv:1703.04977)。aleatoric / epistemic の二分と、ニューラルネットで不確実性を出す枠組み(出力に分散を持たせる、ドロップアウトで epistemic を測る)を整理し、本手法の確率的土台を作りました。

Kendall, Gal, Cipolla "Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics"(CVPR 2018, arXiv:1705.07115)。本トピックの出典です。深度回帰・セマンティックセグメンテーション・インスタンスセグメンテーションを同時に学ぶ設定で、homoscedastic 不確実性に基づく自動重み付けを提案しました。動機は「MTL の性能は損失重みに極端に敏感で、手動探索は高コスト」。

GradNorm(Chen, Badrinarayanan, Lee, Rabinovich, ICML 2018, arXiv:1711.02257)。損失の値ではなく勾配の大きさを各タスクで揃えるよう重みを動的調整します。さらに「学習がどれだけ進んだか(相対的な学習速度)」も見て、遅れているタスクの重みを上げます。Kendall がスケール調整に留まるのに対し、学習速度のバランスまで踏み込みました(詳細は [GradNorm](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0589_gradnorm))。

DWA — Dynamic Weight Averaging(Liu, Johns, Davison, CVPR 2019)。各タスクの損失の「減り具合(直近2エポックの比)」を見て重みを決める軽量法。勾配を計算せず損失値だけで済むため GradNorm より安価です。

PCGrad — Projecting Conflicting Gradients(Yu, Kumar, Gupta, Levine, Hausman, Finn, NeurIPS 2020, arXiv:2001.06782)。タスク勾配が逆方向に引っ張り合う「勾配の競合」を、片方をもう片方に直交する向きへ射影して取り除きます。重み付けでは解けない競合そのものに対処する系統です(詳細は [PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note))。

MGDA — Multiple Gradient Descent Algorithm(Sener & Koltun, NeurIPS 2018)。MTL を多目的最適化と捉え、すべてのタスクを同時に下げられるパレート方向を勾配から計算します。理論的にパレート定常点への収束を保証する点が特徴です。

これらの関係は重要です。Kendall は「損失のスケールを揃える」、GradNorm は「勾配の大きさ(学習速度)を揃える」、PCGrad は「勾配の向きの競合を消す」、MGDA は「多目的最適化として最良の合成方向を解く」。どれが効くかは問題依存で、しばしば組み合わせて使われます。

## 論文の実験結果(定量データ)

Kendall et al. (2018) の主実験は、屋内外シーンのマルチタスク(セマンティックセグメンテーション、インスタンスセグメンテーション、深度回帰)を一つのネットワーク(エンコーダ共有のエンコーダ・デコーダ)で学ぶものです。評価指標を初心者向けに説明します。セグメンテーションは IoU(予測領域と正解領域の重なり率、高いほど良い)や画素正解率、深度は誤差(メートル、小さいほど良い)で測ります。

論文の核心的な結果は、各タスクを単独で学んだ単一タスクモデルよりも、不確実性重み付けで同時学習したモデルの方が、全タスクで性能が向上したことです(セグメンテーション・深度ともに単独学習を上回った)。マルチタスクは負の転移(あるタスクが別タスクの足を引っ張る)で単独学習より悪化することもありますが、適切な重み付けなら相互に助け合えることを示しました。

最も説得力があるのが重みのスイープ実験(アブレーション)です。論文は、手動で固定重みを細かく振って(例えばセグメンテーション:深度の重み比を 0:1 から 1:0 まで)学習し、その性能曲線を描いた上で、不確実性重み付けが「手動で見つけた最良の固定重みとほぼ同等か、それ以上の性能を、一切のチューニングなしで」達成することを示しました。手動探索は重み空間を何点も学習し直す必要があり1点ごとに数時間かかるのに対し、自動手法は1回の学習で済むため、計算コストの削減が劇的です。さらに、不適切な固定重み(あるタスクに偏った重み)では性能が大きく崩れる U 字/谷型の曲線が示され、「重み選びがいかに敏感か=自動化の価値がいかに大きいか」を定量的に裏づけました。

後続研究のアブレーションでは限界も示されています。Kendall の重み付けは本質的に損失のスケール調整に近く、タスク勾配が逆向きに競合する状況では GradNorm や PCGrad に劣ることがある、と報告されています。NYUv2 / CityScapes などの MTL ベンチマークでの比較では、どの手法が勝つかはデータセットとタスクの組合せに依存し、「常に最強の単一手法はない」というのが共通認識です。つまり「スケールの不均衡」が主因の問題には Kendall が強いが、「勾配の方向の対立」が主因の問題には不十分、という棲み分けです。

## メリット・トレードオフ・限界

メリット。損失重みの手動チューニングが不要になり、実験コストを大きく削減します。タスクごとに学習パラメータ σ を1個足すだけで実装が非常に軽く、既存のマルチタスク損失にすぐ組み込めます。スケールの異なる損失(分類と回帰など)を自動でバランスさせ、学習後の σ がタスクの当てにくさの解釈にも使えます(σ が突出して大きいタスクは「データが矛盾している/ラベルが汚い」可能性を示唆する診断にもなる)。

トレードオフと限界。第一に、あくまで損失のスケール調整であり、タスク間の勾配の競合(干渉)そのものは解決しません。第二に、仮定するのは入力非依存の homoscedastic 不確実性で、状況により最適でないことがあります。第三に、σ の学習が不安定だと、特定タスクの重みが極端に下がってそのタスクが事実上学習されなくなる(log 項で緩和されるが完全ではない)危険があります。第四に、確率モデルの仮定(回帰=ガウス、分類=softmax)が現実の損失分布と合わないタスク(例えば最適化済みの complex な複合損失)では導出の前提が崩れ、単なる「学習可能なスケーリング係数」として経験的に使われているのが実情です。第五に、初期値や学習率に σ の収束が依存し、再現性が論点になることもあります。

未解決の課題として、homoscedastic と heteroscedastic を統合した重み付け、重み付けと勾配手術(PCGrad 等)の理論的に整合した統合、タスク数が多い(数十以上)場合のスケーラビリティ、そして「重み付け自体を学習目標(最終性能)から学ぶ」メタ学習的アプローチ(Auto-Lambda など)が研究されています。

## 発展トピック・研究の最前線

マルチタスク重み付けは大きく三系統に整理されます。損失ベース(Kendall の不確実性、損失の減り具合を見る DWA、Random Loss Weighting のような驚くほど強い単純ベースライン)、勾配ベース(GradNorm、PCGrad、CAGrad、勾配を多目的最適化のパレート方向に揃える MGDA / Nash-MTL)、そしてアーキテクチャベース(タスクごとに分岐させる、専門家混合 Mixture-of-Experts でタスク干渉を構造的に避ける、Cross-Stitch / MTAN のようにどの特徴を共有するか自体を学ぶ)です。近年は、これらを多目的最適化(複数の目的を同時に最善化する数学的枠組み)として統一的に捉え、パレート最適(どのタスクも犠牲にせず改善できない点)を狙う研究が主流です。一方で「凝った重み付けは丁寧にチューニングした固定重みやランダム重みに勝てないことも多い」という批判的検証(Xin et al. などの再現研究)もあり、手法選択は慎重さが要ります。自動運転 E2E のように検出・深度・予測・計画を同時に学ぶ大規模マルチタスクでは、Kendall の軽い不確実性重み付けを土台に、競合が強い箇所だけ勾配手術を併用する、あるいは学習を段階分け(知覚を先に学習)する、といった実務的な組み合わせがよく採られます。
