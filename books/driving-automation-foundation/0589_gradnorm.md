# GradNorm

## ひとことで言うと

1つのニューラルネットワークで複数の仕事(タスク)を同時に学ばせると、タスクごとに「学習の進み具合」も「ネットワークに与える勾配の強さ」もバラバラになります。GradNorm は、各タスクが共有部分に流し込む勾配の大きさ(勾配ノルム)を観察し、それを互いに揃える(ただし学習が遅れているタスクには少し大きめに割り当てる)ように損失の重みを動的に調整する手法です。これにより、どのタスクも置いてけぼりにならず、ほぼ同じペースで学習が進むようになります。

## 直感的な理解

複数人で1本のロープを違う方向に引っ張る綱引きを想像してください。力の強い人(勾配の大きいタスク)が一人いると、ロープ(共有された特徴抽出器)はその人の都合のいい場所に寄ってしまい、力の弱い人(勾配の小さいタスク)の望む方向には全く動きません。素朴にすべての損失を足し合わせて学習すると、まさにこれが起きます。値が大きい損失、あるいは勾配が大きいタスクが学習全体を支配し、他のタスクが十分に学ばれないのです。

GradNorm の発想は「全員の引っ張る力(勾配ノルム)を、ほぼ同じ大きさに揃えよう。ただし、まだ目標に遠い人(学習が遅れているタスク)には少しだけ強く引かせてあげよう」というものです。力そのものを直接コントロールするのではなく、各タスクの損失にかける重みを上下させることで、結果的に共有部分へ届く力を均衡させます。

なぜ「損失の値」ではなく「勾配の大きさ」を見るのか。ここが GradNorm の核心です。ネットワークの重みを実際に動かしているのは損失そのものではなく、損失を微分して得られる勾配です。損失の値が大きくても、それが平坦な領域にあって勾配が小さければ重みはほとんど動きません。逆に損失の値は小さくても勾配が急峻ならネットワークを強く引っ張ります。「実際に学習を駆動している量」は勾配なのだから、そこを揃えるべきだ、というのが GradNorm の主張です。

## 基礎: 前提となる概念

マルチタスク学習(multi-task learning、MTL)とは、1つのモデルで複数の予測目標を同時に学ぶ枠組みです。多くの場合、共有エンコーダ(複数タスクで共用する特徴抽出器、shared backbone)と、タスクごとの小さな出力ヘッド(task-specific head)から成ります。共有部分があるので、あるタスクで学んだ表現が別のタスクにも役立つ(正の転移、positive transfer)ことを期待します。

学習では、各タスク i の損失 L_i を1つのスカラーにまとめる必要があります。最も単純なのは重み付き和です。

```
L_total(t) = Σ_i w_i(t) · L_i(t)
```

ここで t は学習ステップ、w_i は各タスクの重みです。固定重み和([固定重み和損失 (Weighted Sum)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0575_fixed-weighted-sum-loss))では w_i を人が決めた定数にしますが、これには2つの困りごとが付きまといます。

第一に、損失のスケールの違いです。回帰損失が数十のオーダー、分類損失が 0.x のオーダー、というようにタスクによって損失の値の桁が違うと、大きい損失が和を支配します。第二に、学習速度の違いです。易しいタスクは早く学び終わり、難しいタスクはなかなか進みません。簡単なタスクの勾配が大きいまま残ると、共有部分が簡単なタスクに最適化され続け、難しいタスクの学習が止まってしまいます。

勾配ノルム(gradient norm)とは、あるタスクの損失が共有部分の重みをどれだけ強く動かそうとしているかを、ベクトルの長さ(L2 ノルム、各成分を二乗して足して平方根を取った値)で測った量です。勾配は重みと同じ次元のベクトルなので、その長さが「引っ張る力の総量」に対応します。

GradNorm の同時期の競合手法に Kendall らの不確実性重み付け([不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall))があります。こちらは各タスクの観測ノイズ(タスク固有の不確かさ)を学習可能パラメータにして損失のスケールを揃えますが、「学習速度のバランス」までは直接見ていません。GradNorm は勾配の大きさと学習進度の両方を見る点で差別化されます。

## 仕組みを詳しく

GradNorm は重み w_i(t) 自体を学習可能なパラメータにし、専用の補助損失(GradNorm loss)でこれを更新します。本体のネットワーク重みは通常どおり L_total を最小化して更新し、w_i だけを別の目的関数で動かす、という二重構造です。ステップで分解します。

ステップ1。共有部分の最後の層(典型的には共有エンコーダと各ヘッドが分岐する直前の重み)を W と置きます。ここを観測点に選ぶのは、全タスクの勾配がここを通って共有部分へ流れ込むからです。各タスク i について、重み付き損失 w_i(t)·L_i(t) を W で微分した勾配の L2 ノルムを計算します。

```
G_i^(W)(t) = || ∇_W ( w_i(t) · L_i(t) ) ||_2
```

ステップ2。全タスクの勾配ノルムの平均を取り、これを「揃えたい目標の基準値」とします。

```
Ḡ_W(t) = E_i [ G_i^(W)(t) ]     (タスクについての平均)
```

ステップ3。各タスクの相対的な学習進度を測ります。タスク i の「現在の損失 ÷ 初期の損失」を逆訓練率(inverse training rate)と呼び、

```
L̃_i(t) = L_i(t) / L_i(0)          (損失がどれだけ下がったかの比)
r_i(t) = L̃_i(t) / E_j [ L̃_j(t) ]  (全タスク平均で正規化した相対進度)
```

と定義します。r_i が 1 より大きいタスクは「平均より学習が遅れている」ことを意味します(損失があまり下がっていない)。

ステップ4。目標勾配ノルムを次のように決めます。

```
目標 G_i(t) = Ḡ_W(t) × ( r_i(t) )^α
```

α(アルファ)は「学習速度の差をどれだけ強く均すか」を決める唯一の主要ハイパーパラメータです。α が大きいほど、遅れているタスクへの引き上げが強くなります。論文では非対称性の強いタスク群で α=1.5、対称的なタスク群で α=0.5 前後を使っています。遅れているタスク(r_i 大)ほど目標ノルムを大きく設定し、もっと強く学ばせます。

ステップ5。GradNorm 損失を、実際のノルムと目標ノルムの差の L1(絶対値)として定義します。

```
L_grad(t) = Σ_i | G_i^(W)(t) − Ḡ_W(t) × r_i(t)^α |
```

重要なのは、この目標値 `Ḡ_W(t) × r_i(t)^α` は定数として扱い(微分を通さない、stop-gradient)、L_grad を w_i についてだけ微分して w_i を更新する点です。こうすることで、w_i を上げ下げして実際の勾配ノルムを目標に近づけます。

ステップ6。更新後、w_i を正規化して Σ_i w_i = T(タスク数)になるように保ちます。これは重みが暴走して全体が発散するのを防ぐためです。

数値例で直感を確認します。検出と深度の2タスクで、検出は早く学べて損失が初期の20%まで下がった(L̃_det=0.2)、深度はまだ初期の80%(L̃_depth=0.8)とします。平均は 0.5 なので、r_det = 0.2/0.5 = 0.4、r_depth = 0.8/0.5 = 1.6。α=1.5 なら目標倍率は検出が 0.4^1.5 ≈ 0.25、深度が 1.6^1.5 ≈ 2.02。深度の目標ノルムが平均の約2倍に設定され、深度の重み w_depth が引き上げられます。結果、共有部分が深度の学習にも貢献するようになり、両タスクの進度が揃っていきます。

```
共有層 W (全タスクの勾配が通る観測点)
   ▲ ▲ ▲
   │ │ │  各タスクの勾配ノルム G_i を測る
 task1 task2 task3
 G1=3.0 G2=1.0 G3=2.0   平均 Ḡ=2.0
   │
 学習進度 r_i から 目標 = Ḡ × r_i^α
 (遅れているタスクは目標を大きく)
   │
 実ノルムを目標に近づけるよう w_i を更新 → 正規化(Σw_i=T)
```

## 手法の系譜と主要論文

マルチタスクの損失バランス問題は2017〜2018年に集中的に研究されました。

GradNorm の出典は Zhao Chen, Vijay Badrinarayanan, Chen-Yu Lee, Andrew Rabinovich による "GradNorm: Gradient Normalization for Adaptive Loss Balancing in Deep Multitask Networks"(ICML 2018, arXiv:1711.02257)です。著者らは当時 Magic Leap に在籍し、AR 向けのマルチタスク認識(深度・法線・セグメンテーション同時推定)を背景にしていました。新規性は「損失そのものではなく、共有層に流れる勾配の大きさを均衡させる」という視点と、それを「学習進度 r_i で重み付けして遅れたタスクを優遇する」非対称化にあります。

ほぼ同時期に Alex Kendall, Yarin Gal, Roberto Cipolla の不確実性重み付け([不確実性重み付けマルチタスク損失 (Kendall)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0601_uncertainty-weighting-kendall), CVPR 2018)が登場しました。こちらは確率的なモデリング(各タスクの観測ノイズ σ_i を学習)から重みを導出し、損失のスケールを自動調整します。GradNorm と Kendall はよく対で比較され、前者は学習速度を、後者はスケールと不確実性を主眼に置く点で対照的です。

翌年、Shikun Liu, Edward Johns, Andrew Davison が Dynamic Weight Averaging([Dynamic Weight Averaging (DWA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0588_dynamic-weight-averaging), CVPR 2019)を提案しました。これは GradNorm の「勾配計算が重い」という欠点を回避し、損失の下降率という値の比だけで重みを近似する軽量版という位置づけです。

向きの競合に着目する系譜も生まれました。Tianhe Yu らの PCGrad([PCGrad (勾配射影)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0598_pcgrad_note), NeurIPS 2020)は勾配の大きさではなく向きの競合(内積が負になる干渉)を射影で取り除きます。GradNorm(大きさ)と PCGrad(向き)は相補的で、両者を組み合わせる研究もあります。さらに後続の CAGrad(Liu et al. NeurIPS 2021)や Nash-MTL(Navon et al. ICML 2022)は、全タスクを公平に改善する更新方向を最適化やゲーム理論で求める方向へ発展しました。

## 論文の実験結果(定量データ)

GradNorm 論文は主に2つの設定で評価しています。

1つ目は NYUv2 を改変したマルチタスク回帰・分類設定です。NYUv2 は屋内シーンの RGB-D データセットで、深度推定・表面法線推定・セマンティックセグメンテーション(各ピクセルが何の物体かを分類)を同時に学びます。論文では入力画像をダウンサンプルした標準版に加え、データ拡張で水増しした版(NYUv2+seg、NYUv2+kpts)を用い、VGG 系の SegNet をバックボーンにしています。評価指標は、深度なら誤差(RMSE などの距離誤差、小さいほど良い)、セグメンテーションなら IoU(Intersection over Union、予測領域と正解領域の重なり率、大きいほど良い)や画素精度、法線なら角度誤差(度、小さいほど良い)です。報告によると、GradNorm(α≈1.5)は等重み(全タスク w=1)のベースラインに対して各タスクの誤差を低減し、とりわけ複数タスクをまとめた平均性能で明確に上回りました。注目すべきは、手動グリッドサーチで時間をかけて見つけた最良の固定重み(論文で grid search と呼ぶ強い比較対象)と同等かそれ以上の性能を、調整パラメータ α 1個だけで達成した点です。固定重みの探索はタスク数 N に対して N 次元の組み合わせ爆発を起こすので、α 1個で済むのは大きな実務的利点です。

論文はさらに、学習中に GradNorm が定めた重み w_i(t) の時間変化を可視化しています。学習初期に損失の下がりが遅いタスクの重みが自動で押し上げられ、そのタスクが追いついてくると重みが下がって落ち着く、という「自己調整」の軌跡が観察されました。これは手動の固定重みでは決して得られない動的な振る舞いで、GradNorm が学習速度の偏りに実際に反応していることの定性的な裏付けになっています。

2つ目は回帰のみの合成・実データタスクで、各タスクの出力スケールを意図的にずらした設定です。等重み学習ではスケールの大きいタスクが支配し、小さいタスクの誤差が大きく残りますが、GradNorm はこれを自動で均衡させ、全タスクで低い誤差を達成しました。

アブレーション(どの要素を抜くとどうなるかの検証)としては、α の効果が示されています。α=0 にすると勾配ノルムを単に等しく揃えるだけ(学習進度を無視)になり、タスク間に難易度差があるケースでは性能が落ちます。α を上げるほど遅れたタスクへの優遇が強まり、適度な α(タスクの非対称性に応じて 0.5〜1.5 程度)で最良になります。これは「スケールを揃えるだけでなく、学習速度の偏りまで補正することに意味がある」ことの直接的な証拠です。また、過学習が減る(検証性能と訓練性能の差が縮む)効果も報告されており、勾配を均衡させることが一種の正則化として働くと解釈されています。

指標の意味を補足します。IoU が数ポイント上がることは、セグメンテーションで物体境界の誤りが目に見えて減ることを意味します。深度誤差が下がることは、各画素の奥行き推定が実測に近づくことを意味し、3次元理解の精度に直結します。マルチタスクでは「あるタスクだけ良くて他が悪い」状態を避け、全タスクの平均を底上げできるかが評価の本質で、GradNorm はその底上げを少ない調整で実現した点が評価されています。

## メリット・トレードオフ・限界

メリットは次の通りです。損失のスケールだけでなくタスク間の学習速度の偏りまで自動で均衡させること。調整すべき主要ハイパーパラメータが実質 α 1個と少なく、固定重みの多次元グリッドサーチを置き換えられること。共有表現が特定タスクに偏るのを防ぎ、全タスクの底上げと正則化効果が期待できること。

トレードオフと限界もあります。毎ステップ各タスクの勾配ノルムを計算するため、追加の逆伝播コストがかかります。タスク数が多いと無視できません。次に「どの共有層 W で勾配ノルムを測るか」の選択が結果に影響し、原理的には恣意性が残ります。また GradNorm は勾配の大きさを揃えるだけで、勾配の向きの競合(あるタスクの改善が別タスクを悪化させる負の転移)は解決しません。これには PCGrad のような射影手法が必要です。さらに、L_i(0)(初期損失)を基準に学習進度を測るため、初期損失の推定がノイジーだと挙動が不安定になることがあり、実装では最初の数ステップの損失を平滑化するなどの工夫が要ります。

未解決の研究課題としては、観測点 W の自動選択、勾配計算コストの削減(近似手法)、そして大きさと向きを統一的に扱う枠組みの構築が挙げられます。

## 発展トピック・研究の最前線

GradNorm 以降、マルチタスク最適化は「重み付け系(損失や勾配の大きさを調整)」と「勾配操作系(向きの競合を解消)」の2系統に分かれ、両者を統合する研究が進みました。CAGrad は全タスクの平均損失の降下を保証しつつ最悪タスクの改善も両立する更新方向を凸最適化で求めます。Nash-MTL は各タスクをプレイヤーと見立て、ナッシュ交渉解として勾配を統合します。IMTL(Liu et al. ICLR 2021)はスケールと向きの両方を1つの枠組みで扱おうとします。

一方で、これら高度な手法の有効性には議論もあります。Kurin らや Xin らの研究(2022 前後)は、適切な正則化と十分なチューニングを施した単純な等重み・固定重みが、多くのベンチマークで高度な MTL 最適化手法と同等に強いことを報告しました。つまり「手法の差はベンチマーク依存で、万能の勝者はいない」というのが現在の理解です。GradNorm を学ぶ意義は、勾配という学習の実体に基づいてバランスを取る考え方そのものにあり、その思想は後続手法の前提になっています。
