# モーション接地型残差ワールドモデル

## ひとことで言うと
「数秒後に周囲はどうなっているか」を予測するとき、変化を2つに分けます。ひとつは「自分(自車)が動いたせいで景色がずれて見えるぶん」、もうひとつは「他の車や歩行者が実際に動いたぶん」。前者は自分の動きさえ分かれば計算だけで正確に出せます。だからニューラルネットには後者の「他者の動き(残差)」だけを学ばせよう、というのがモーション接地型残差ワールドモデルです。学習が楽になり、表現の崩壊も防げます。

## 直感的な理解
電車の窓から外を眺める場面を想像してください。電車が前へ進むと、止まっている建物・電柱・看板はすべて後ろへ流れていきます。これは建物が動いたからではなく、自分が動いたからです。自分の速度が分かっていれば、次の瞬間に建物がどこへ流れるかは完全に計算で予測できます。

一方、すれ違う対向電車や、線路脇を歩く人の動きは、自分の速度だけでは予測できません。相手がどれくらいの速さでどちらへ動くか、観察しなければ分からない。

ワールドモデル(未来の状態を予測する内部モデル)が「次の景色」を当てる難しさは、ほぼ後者にあります。前者は幾何で消せる「易しい大部分」、後者が「難しい小部分」。なのに従来の単純な予測器は、両方をひとつの学習する差分でまとめて背負っていました。易しい部分まで学習に肩代わりさせるのは、表現力の無駄遣いです。モーション接地型残差ワールドモデルは、易しい部分を幾何計算で先に取り除き、ネットを難しい部分へ集中させます。

## 基礎: 前提となる概念

順に噛み砕きます。

- **ワールドモデル(world model)** とは、「いまの状況から少し先の未来がどうなるか」を内部で予測するモデルです。人間が「このまま行くとあの車が来そうだ」と先読みするのに似ています。自動運転では、未来を想像できるモデルほど安全な計画を立てられると考えられています。

- **BEV特徴(Bird's Eye View)**: カメラ映像を上空から見下ろした2次元マップに変換した特徴。各マス(セル)に「そこに何があるか」を表す数十〜数百次元のベクトルが入っています。未来予測はこのBEV特徴を時間発展させる形で行うことが多い。

- **潜在空間での予測(latent prediction)**: ピクセル(画像の生の値)を直接予測するのではなく、ニューラルネットが抽出した「意味のある特徴」の空間で未来を予測すること。テクスチャや影のような予測しても無意味な細部を無視でき、運転に効く意味的な変化だけを扱えます。

- **特徴再構成損失 / JEPA損失(Joint Embedding Predictive Architecture)**: 予測した未来の特徴と、実際にその時刻になったときの特徴(凍結したエンコーダで作る正解)を比べる損失。ピクセル損失ではなく特徴空間で比べるのが肝。

- **表現崩壊(representation collapse)**: 予測器が損失を下げる安易な道として、出力を定数(全部同じ値)にしてしまう病的な解。予測も正解も同じ定数なら損失はゼロだが、何も学んでいない。自己教師あり学習で最大の落とし穴。

ここで着目する問題はこうです。素朴な未来予測器は、いまのBEVから未来のBEVへの変化を「ひとつの学習する差分(additive delta、足し算の差)」として丸ごと予測します。`z_{t+Δ} ≈ z_t + Δ` のイメージです。しかしBEVが変化する理由は性質の全く違う2つが混ざっています。

1. **自車起因の見かけの動き(ego-induced apparent motion)**: 自分が前進・旋回すると、周囲の景色は全体としてずれ・回転します。大きくて支配的な変化ですが、自分の動き(エゴモーション)が分かっていれば完全に既知です。本質は座標をずらす変換であって、単純な足し算では表現しにくい。

2. **動的な動き(dynamic motion)**: 他の車・歩行者が実際に動くぶん。これこそ本当に不確実で予測が難しい部分。

問題は、足し算の差分1つでこの2つを同時に背負わせていること。幾何的に正確に分かる座標変換までネットに学習させるのは容量の無駄であり、何でも予測できる自由度は表現崩壊を招きやすい。

## 仕組みを詳しく

提案は2部構成です。

### Part 1: モーション接地型の残差分解

未来のBEVを、既知の幾何変換と学習する残差に分けます。

```
z_{t+Δ} ≈ Warp_ego(z_t)              + Δ_dynamic
          └ 既知の自車ポーズ変化 ┘   └ ネットはこれだけ予測 ┘
```

- **`Warp_ego(z_t)`**: いまのBEV `z_t`(形 `[B, C, H, W]`)を、未来の自車座標系へワープ(座標に従って値を移し替える再サンプリング)する。動かす量は自車の未来移動量 `(dx, dy, dθ)`(前後移動・左右移動・向きの変化)から決まり、学習しない純粋な幾何変換です。学習時はこの移動量を正解の未来エゴモーション(走行ログに記録された自車の動き)から取り、推論時は計画した軌道から代用します。実装は [ego モーションによるBEVワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0265_ego-motion-bev-warp) と [affine_grid と grid_sample による微分可能ワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0912_affine-grid-grid-sample-warp) を参照。
- **`Δ_dynamic`**: 静的世界のワープからのズレ、つまり他者が実際に動いたぶんだけをネットが予測する。

なぜ効くか。自車の座標変換を先に幾何で除去すれば、ネットは「他者の動き」という本当に難しい部分だけへ容量を使えます。さらにワープ結果が予測の土台(アンカー)になるので、出力がでたらめな定数へ潰れにくくなり、JEPA損失の表現崩壊を防ぐ事前知識として働きます。出力の形(shape)は変わらないため、損失・正解の作り方・学習ループを変えずに差し込める非破壊的な設計です。

入力には自車の運動を表す `(dx, dy, dθ)` を明示的に渡します。これは計画器が出す「意図のベクトル」(条件付けバイアス)とは別物です。前者はワープが消費する幾何そのもの、後者は学習された意図。役割が違うので両方を併存させます。

### Part 2: 視覚履歴で動的残差を条件づけ

もうひとつの着眼点。1枚の現在フレームだけからは他者の速度が分かりません。前の車が止まっているのか、こちらと同じ速さで走っているのかは、一瞬の画像では区別できない(同じ位置に見える)。速度・加速度は複数の過去フレームを見て初めて分かります。そしてこれこそ `Δ_dynamic` が必要とする情報です。

そこで過去の視覚特徴を `[B, T, feat]`(B バッチ、T 過去フレーム数、feat 特徴次元)の形で保持する時間メモリと、低頻度で過去をまとめるエンコーダを使い、視覚的な時間文脈を予測器へ注入します。

- 注入するのは「視覚・シーンの文脈」であって自車の運動履歴ではない点が肝心。自車の動きはすでにワープが扱っているので二重に入れない。
- 段階1(まず手始め): グローバルな視覚文脈ベクトルを、畳み込み前のBEVへチャンネル単位のバイアスとして足す。安価でそのまま組み込める。
- 段階2(将来): セル単位の細かい時間文脈を入れる方向。表現力は上がるがコストも増える。

詳しくは [視覚履歴による動的残差の条件付け](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0479_visual-history-conditioning-future-state)。

## 手法の系譜と主要論文

「既知の運動を先に説明し、難しい残りへ資源を集中する」発想と、「表現崩壊を防ぐ非対称設計」の二系統が交わるところに本トピックがあります。

- **動画予測・運動分解の系譜(2010年代〜)**: オプティカルフローやシーンフローの研究では、「既知のカメラ運動による見かけの流れ」と「物体自身の運動」を分けて推定する研究が古くからあります(motion segmentation, scene flow)。背景の動きを幾何で説明し残りを物体運動とする発想は、Warp + 残差と同じ構造です。

- **BYOL (Grill et al., NeurIPS 2020, arXiv:2006.07733)**: 予測器と目標エンコーダを非対称(目標側を EMA で緩やかに更新、勾配を流さない)にすることで、負例なしでも表現崩壊を防げることを示した自己教師あり学習。凍結/EMAエンコーダを正解に使う根拠であり、ワープによるアンカー発想とも通じます。

- **MILE (Hu et al., NeurIPS 2022, arXiv:2210.07729)**: 都市運転で潜在ワールドモデルを学習し、BEVを潜在空間で時間発展させる。自車運動を座標系へ織り込んで未来をロールアウトする設計で、潜在空間の未来予測が運転に効く根拠。

- **I-JEPA (Assran et al., CVPR 2023, arXiv:2301.08243)**: ピクセルではなく学習した潜在空間で未来(マスクされた領域)を予測する枠組み。意味のある変化だけを予測し、無関係な細部は無視する。表現崩壊を防ぐため予測側と目標側を非対称にする考え方の出典。本トピックは JEPA損失の上に、ワープによるアンカーでさらに崩壊を防ぐ二重の安全装置として位置づけられます。

- **UniAD (Hu et al., CVPR 2023, arXiv:2212.10156)**: 検知・追跡・地図・予測・計画を一つのネットに統合し、未来予測(動き予測・占有予測)モジュールを持つ。幾何的に根拠づけられた自己教師信号が backbone へ良い勾配を与える、という主張の裏づけに引用されます。

- **GAIA-1 / GAIA-2 (Hu et al., 2023〜, arXiv:2309.17080)** や **DriveDreamer (Wang et al., ECCV 2024, arXiv:2309.09777)**: 対極の路線。映像そのものを大規模生成モデル(自己回帰トークン予測や拡散)で多様に生成し、ありうる未来を確率的にサンプリングする。表現力・多様性は圧倒的だが、計算コストが極端に大きく、ego起因の決定的な動きまで生成器に学ばせるため学習効率は悪い。モーション接地型は逆に、決定的に分かる部分を生成から外して学習対象を縮小する。両者は「どこまでを学習・生成に任せ、どこまでを物理で固定するか」というスペクトルの両端に位置づけられます。

これらに対し、モーション接地型残差ワールドモデルの独自性は「ego部分は学習せず幾何計算で消す(パラメータフリー)」割り切りと、「JEPA損失 + ワープアンカー」の崩壊対策の二重化にあります。系譜としては、古典的なモーションセグメンテーション(背景流れと物体流れの分離)が持っていた幾何の知恵を、現代の自己教師あり潜在予測(JEPA)に接ぎ木した位置づけと言えます。

## 論文の実験結果(定量データ)

ワールドモデル・自己教師あり予測の評価は、データセットと指標が多様です。代表的な定量結果を、指標の意味とともに示します。

- **MILE**: CARLA(運転シミュレータ)の標準ベンチマークで、行動を模倣しつつ潜在で未来を予測する設計が、ピクセル予測ベースラインを上回ることを報告。Driving Score(ルート完走率と違反ペナルティを統合した総合指標、高いほど良い)で先行手法を上回り、未到達ルート区間でのロールアウトでも一貫した運転を維持できることを示しました。潜在空間での未来予測が、ピクセル予測より運転性能に直結するという定量的根拠です。

- **I-JEPA**: ImageNet で、ピクセル再構成型(MAE など)と比べて、はるかに少ない事前学習エポックで競合する線形評価精度(linear-probe accuracy、凍結特徴の上に線形分類器だけ載せたときの正解率、高いほど特徴が良い)に到達することを報告。潜在予測がピクセル予測より計算効率良く意味的特徴を学べることを示しました。意味のない細部の予測に資源を割かないことの効果です。

- **BYOL**: ImageNet 線形評価で当時のトップ。重要なのはアブレーションで、目標エンコーダの EMA 更新を止めて予測器と目標を対称にすると、精度がほぼランダム(表現崩壊)まで落ちることを示した点。非対称設計が崩壊防止の本質であるという定量証拠で、本トピックの「アンカーで崩壊を防ぐ」主張の理論的支柱です。

- **残差学習の一般的効果(参考)**: 「予測を恒等写像(何もしない)からの残差として学ばせる」設計が学習を安定させる事実は、ResNet(He et al., CVPR 2016, arXiv:1512.03385)が ImageNet で示した古典的知見と同じ構造です。ResNet は、層が `H(x)` を直接学ぶより `F(x) = H(x) - x` という残差を学ぶ方が、最適解が恒等写像に近いとき圧倒的に学習しやすいことを示し、152層という当時前例のない深さを学習可能にして ImageNet top-5 誤り率を 3.57% まで下げました。ワープ分解はこの発想を時間方向へ持ち込んだもので、「ワープ済みBEV(≒恒等に近い予測)」をスキップ接続のように扱い、ネットには残差 `Δ_dynamic` だけを学ばせます。残差が小さい(大半が静的)シーンほどこの構造の利得が大きい、という直感はここから来ます。

これらは別々の論文の結果ですが、合わせて読むと「潜在予測は効率的に意味を学ぶ」「非対称・アンカーが崩壊を防ぐ」「恒等からの残差は学習を安定させる」という、モーション接地型残差ワールドモデルが依拠する定量的前提を裏づけます。ワープ分解そのものの大規模ベンチでの効果は、提案段階のため未確立であり、今後の検証課題です。特に、激しい動的シーン(交差点の混雑、合流)では `Δ_dynamic` が支配的になり、静的世界が大半という前提が崩れるため、残差分解の利得がどこまで残るかは実データでの定量評価を待つ必要があります。

## メリット・トレードオフ・限界

- メリット: 既知の自車運動を幾何で除去するので、ネットは難しい「他者の動き」だけへ容量を集中でき、学習効率が上がる。
- メリット: ワープがアンカーになり、JEPA損失で起きがちな表現崩壊を抑えられる。
- メリット: 出力形を変えないため、既存の損失・学習ループを改変せず追加できる(非破壊的)。
- メリット: 過去フレームの視覚文脈を入れることで、1枚では分からない他者の速度を推定しやすくなる。
- トレードオフ: 学習時に正解の未来エゴモーションが必要。推論時は計画軌道で代用するなど、入手経路を設計する必要がある(train/test の不一致リスク、[scheduled sampling によるワープ入力の切替](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0580_scheduled-sampling-warp-source))。
- トレードオフ: ワープ(再サンプリング)で格子の端や見えなくなる領域に欠損・補間誤差が出る。
- トレードオフ: 自車運動という新しい入力を学習ループ・データパイプラインへ通す配線が要る。
- トレードオフ: セル単位の条件づけは計算コストが上がる。安価なグローバル条件づけから段階的に進める設計が現実的。
- 限界・未解決課題: 提案段階であり、大規模実データでのワープ分解の効果・安定性、誤差の時間方向蓄積、ワープ前提が崩れる激しい動的シーンでの挙動は、いずれも未検証です。

## 発展トピック・研究の最前線

世界モデル研究は、軽量な幾何ワープ路線(計算で消せる部分は消す)と、大規模生成路線(GAIA-1 系のように映像そのものを多様に生成する)に分かれます。両者は対極で、前者は省力・解釈性重視、後者は表現力・多様性重視。最近は拡散モデルによる運転シーン生成、トークン化したBEVの自己回帰予測、占有(occupancy)の時間予測など、生成と予測の境界が溶けつつあります。一方で、本トピックのように「物理で分かる部分は物理に任せ、学習は難しい残差に集中する」というインダクティブバイアス(事前知識を構造に埋め込む設計)は、データ効率と解釈性の点で根強い価値を持ち続けています。
