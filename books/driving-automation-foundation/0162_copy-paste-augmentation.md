# Copy-Pasteデータ拡張

## ひとことで言うと

Copy-Paste データ拡張とは、ある画像から物体だけを切り抜き、別の画像に貼り付けて新しい学習画像を人工的に作り出すことで、物体検出やインスタンスセグメンテーションの学習データを水増しし、精度を上げる手法です。

## 直感的な理解

まず用語を噛み砕きます。

データ拡張(data augmentation)とは、手元の学習データに変換を加えて見た目の異なるバリエーションを作り、データ量を実質的に増やす技術です。反転・回転・色変更・切り抜きなどが定番です。データが多いほどモデルは過学習(overfitting、訓練データだけに過剰に適応して未知データで性能が落ちる現象)しにくくなります。

物体検出(object detection)は画像中の物体を四角い枠(バウンディングボックス)で囲み、その種類を当てるタスク。インスタンスセグメンテーション(instance segmentation)はさらに細かく、物体を画素単位の輪郭(マスク)で塗り分け、しかも同じ種類でも個体ごとに区別するタスクです。たとえば3人の人を、人クラスとしてまとめず「人1・人2・人3」と別々に塗り分けます。

Copy-Paste のアイデアは拍子抜けするほど単純です。あるシーンに写っている象の輪郭(マスク)を使って象だけを切り抜き、まったく別の街路の画像に貼り付けます。すると「街路に象がいる」という、現実にはまず撮れない新しい学習例が一瞬で作れます。物体のマスクと枠は元画像から正確に分かっているので、貼り付け先での正解ラベルも自動で付きます。

```
元画像A(象が写っている)        元画像B(街路)
   ┌──────────┐                ┌──────────┐
   │   🐘マスク │  ── 切り抜き ─→ │  🐘 ←貼付 │
   │          │                │  街路      │
   └──────────┘                └──────────┘
                                      ↓
                          新しい学習画像(象+街路、ラベル自動付与)
```

これによって、特に「学習データに少ししか出てこない物体」を、いろいろな背景・位置・組み合わせで大量に見せられます。

## 基礎: 前提となる概念

- マスク(mask): 物体がどの画素に属するかを示す、画像と同じ大きさの白黒(0/1)のシート。1 の画素が物体、0 が背景。インスタンスセグメンテーションの正解はこのマスクです。
- バウンディングボックス(bounding box): 物体を囲む最小の長方形。検出の正解として使われます。
- インスタンス(instance): 個々の物体。同じ「車」クラスでも別々の車は別インスタンス。
- 合成(compositing): ある画像の一部を別画像に重ね合わせること。Copy-Paste の貼り付けは画素レベルの合成です。
- ドメインギャップ(domain gap): 学習データと本番データの統計的な違い。貼り付け画像はやや不自然になりがちで、これが本物データとの差(ギャップ)を生む懸念があります。

## 仕組みを詳しく

Ghiasi らの "Simple Copy-Paste"(CVPR 2021)が示した重要な発見は、「凝った工夫をしなくても、ランダムに貼るだけで十分強い」という点でした。それ以前の Copy-Paste 研究は、貼り付け位置を文脈に合わせたり(空に車を貼らない等)、境界をうまくぼかして自然に見せたりする工夫を重視していました。Ghiasi らは、そうした文脈モデリングが必ずしも必要ないことを実験で示しました。

手順を具体的に追います。

1. 画像を2枚選ぶ(貼り付け元と貼り付け先)。両方とも通常のスケールジッタ(scale jitter、画像をランダムに拡大縮小する拡張)をかけておきます。論文は特に大きく揺らす「ラージスケールジッタ(LSJ)」が効くと報告しています。
2. 元画像から、ランダムに選んだいくつかのインスタンス(マスク付き)を取り出す。
3. それらを貼り付け先画像のランダムな位置に重ねる。位置や文脈は気にしない。
4. 貼り付けた物体が覆い隠した分だけ、貼り付け先にもともとあった物体のマスクを更新する(隠れた部分は見えなくなったので正解マスクから削る)。
5. アルファブレンディング(alpha blending、境界を少しなめらかに混ぜる処理)はあってもなくても大差ない、と報告されています。

数式というほどのものはありませんが、合成の核は次のように書けます。新しい画像 I_new は、貼り付け先 I_dst、元画像 I_src、貼り付けるマスク α(物体画素が1、それ以外0)を使って

I_new = α ⊙ I_src + (1 - α) ⊙ I_dst

⊙ は画素ごとの掛け算(要素積)を表します。意味は「マスクが1の画素は元画像の物体を、0の画素は貼り付け先をそのまま使う」というだけです。アルファブレンディングする場合は α を 0 と 1 の間の連続値にして境界をなめらかにします。

なぜ効くのか。第一に、物体・背景・位置の新しい組み合わせが指数的に増えます。N 枚の物体と M 枚の背景があれば、組み合わせは N×M 通りに広がります。第二に、特に稀な物体(出現回数の少ないクラス)を意図的に何度も貼ることで、クラス不均衡(ある種類のデータばかり多く、別の種類が極端に少ない状態)を緩和できます。第三に、オクルージョン(occlusion、物体どうしの重なり)の例を人工的に作れるため、隠れた物体への頑健性が上がります。

実装上の注意点として、貼り付けはオンザフライ(学習中にその場で生成)で行うのが普通です。あらかじめ大量の合成画像をファイルに保存するより、毎エポック違う組み合わせを生成したほうが多様性が増します。

## 手法の系譜と主要論文

- Dwibedi, Misra, Hebert, "Cut, Paste and Learn: Surprisingly Easy Synthesis for Instance Detection"(ICCV 2017): 切り抜いた物体を別背景に貼って検出器を訓練する先駆け。境界のアーティファクト(不自然な縁)を消すため複数のブレンディングを混ぜる工夫を提案。
- Dvornik, Mairal, Schmid, "Modeling Visual Context is Key to Augmenting Object Detection Datasets"(ECCV 2018): 「文脈に合った場所に貼る」ことの重要性を主張(後に Ghiasi らが反証的な結果を示す)。
- Fang ら, "InstaBoost: Boosting Instance Segmentation via Probability Map Guided Copy-Pasting"(ICCV 2019): 確率マップに基づいて貼り付け位置を選ぶ手法。
- Ghiasi ら(Google), "Simple Copy-Paste is a Strong Data Augmentation Method for Instance Segmentation"(arXiv:2012.07177, CVPR 2021): 文脈を考えずランダムに貼るだけで強力、を大規模実験で示した代表作。本ノートの中心。
- 派生として、自動運転の3D検出向けに点群へ物体を貼り付ける GT-Paste / GT-Sampling(SECOND, Yan ら 2018 で導入)があり、LiDAR 点群データの定番拡張になっています。Copy-Paste の発想は2Dにとどまりません。

## 論文の実験結果(定量データ)

評価は主に COCO(約12万枚の画像、80物体クラスの標準ベンチマーク)で、指標はマスク AP(mask Average Precision、画素マスクの正確さを総合した平均適合率。0 から100で大きいほど良い)とボックス AP です。AP とは、検出の正解判定の厳しさ(マスクと正解の重なり度合い IoU のしきい値)をいろいろ変えながら測った精度を平均したもので、検出・セグメンテーションの事実上の標準指標です。

Ghiasi らの主な報告値(概数、構成により変動):

- 強力なベースライン(ラージスケールジッタを使った Mask R-CNN 系)に Copy-Paste を追加すると、マスク AP がおよそ +0.6 から +1.5 ポイント程度向上したと報告されています。すでに強い学習レシピの上での上乗せである点が重要で、「弱いベースラインで派手に効く」のではなく「最先端の上でもなお効く」ことを示しました。
- バックボーン(特徴抽出ネットワーク)を EfficientNet-B7 等に大型化した最良構成では、COCO で当時の最先端級の結果(マスク AP がおよそ 50 前後)に到達したと報告されています。
- 半教師あり設定: ラベルなしデータに擬似ラベル(モデル自身の予測をラベル代わりに使う)を付けてから Copy-Paste を併用すると、さらなる向上が示されました。Copy-Paste が擬似ラベルと相性良く組み合わさることを意味します。
- 希少クラスへの効き: クラスごとに見ると、出現回数の少ないクラスでの改善が相対的に大きい傾向が報告されており、クラス不均衡の緩和という狙いが裏付けられています。

数値の意味を補足します。COCO のマスク AP で +1 ポイントは、見た目は小さいようでも、この分野では1本の論文の主要貢献に値する改善です。しかもデータを買い足したりラベルを増やしたりせず、貼り付けという軽い操作だけで得られる点が実用上の魅力です。

## メリット・トレードオフ・限界

メリットは、追加のアノテーションコストゼロで多様な学習例を生成できることです。実装が単純で、既存の学習パイプラインに数行で組み込めます。希少クラスの底上げとオクルージョン頑健性の向上に効きます。

トレードオフと限界もあります。

第一に、不自然さ(アーティファクト)です。境界に切り抜きの縁が残ったり、照明や影が背景と合わなかったり、スケール(大きさ)が文脈と矛盾したり(巨大な猫が手前に貼られる等)します。モデルがこの「貼られた感」を手がかりに学習してしまうと、本物データへの汎化が損なわれる恐れがあります。アルファブレンディングや適切なスケールジッタはこの緩和に寄与します。

第二に、文脈の破壊です。Ghiasi らは「文脈を気にせず貼ってよい」と示しましたが、これはタスクや分野に依存します。物体どうしの位置関係や共起(一緒に現れやすさ)が意味を持つ場面(例えば運転シーンで歩行者が車道の真ん中の空中に貼られる等)では、非現実的な合成が悪影響を与えうるという指摘もあります。

第三に、マスク品質への依存です。切り抜きの正解マスクが粗いと、その粗さがそのまま貼り付け画像のラベルに混入します。ボックスしかないデータには直接適用しにくいのも制約です(切り抜きには画素単位のマスクが要ります)。

第四に、ハイパーパラメータの調整負担です。何個貼るか、どのくらいスケールを揺らすか、ブレンディングするかなどが性能に影響し、データセットごとに調整が要ることがあります。

## 発展トピック・研究の最前線

3D・点群への展開: LiDAR ベースの3D物体検出では、別シーンの正解物体(の点群)を現在のシーンに挿入する GT-Sampling が標準拡張になっています。地面の高さや遮蔽の整合をどう保つかが研究テーマです。

生成モデルによる合成: 単純な貼り付けの不自然さを、拡散モデル(diffusion model、ノイズから画像を生成する手法)で「貼った物体を背景になじませる」インペインティング(穴埋め生成)で補う研究が進んでいます。照明や影を合成的に整えることで、よりリアルな学習例を作る方向です。

自動運転シーンへの応用: 稀で危険な状況(コーナーケース、たとえば道路上の落下物や倒れた歩行者)を Copy-Paste で人工的に増やし、検出器のロングテール性能を底上げする試みがあります。ただし安全が重要な領域では「人工データで上がった指標が、本物の稀事象にも本当に効くか」の検証が不可欠で、シミュレーションや実走データでの確認とセットで議論されます。

他拡張との組み合わせ: Mosaic(4枚の画像を1枚に貼り合わせる、YOLO 系で普及)や MixUp(2枚を半透明に重ねる)など他の合成系拡張と併用すると相乗効果が出る一方、強すぎる拡張の重ねがけは逆効果になりうるため、バランスが研究対象です。
