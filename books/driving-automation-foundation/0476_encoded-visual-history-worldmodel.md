# Encoded Visual History（世界モデルへの時間文脈の隔離）

## ひとことで言うと
「過去の見え方の記憶(時間文脈)」を、その場で素早く運転判断する反応経路から外し、別系統の遅い予測経路(世界モデル)に持たせる設計です。具体的には、低頻度(たとえば1秒に1回)で多視点カメラ画像を埋め込み、その埋め込みをリングバッファ(古いものから上書きされる固定長の記録)に溜め、未来予測と計画モジュールに供給します。低次元に潰さず、エンコード済み(encoded)の特徴のまま履歴を持つので、過去情報の損失を抑えられます。

## 直感的な理解
人間の運転には2つの思考が同時に走っています。1つは「目の前に飛び出した物を反射的に避ける」速い反応。もう1つは「この先の交差点は混みそうだ」「あの車はたぶん車線変更してくる」という予測的で遅い思考です。前者は0.1秒単位の即応が要りますが、後者は1秒に1回くらいの更新で十分です。心理学者カーネマンの言う「速いシステム1」と「遅いシステム2」の二重過程に対応すると考えると分かりやすいです。

時間文脈(過去の記憶)が一番効くのは後者の予測です。だったら、重い時間融合を速い反応経路に背負わせる必要はありません。記憶は遅い予測経路に隔離し、速い経路は現在フレームに集中させて軽く保つ。これが「時間文脈を世界モデル側へ移す」発想です。Encoded Visual History は、その遅い経路が参照する「エンコード済みの過去の記憶バッファ」を指します。料理にたとえれば、目の前の鍋を見て火加減を即調整するのが反応経路、冷蔵庫の在庫の履歴を見て翌日の献立を計画するのが予測経路、という分業です。

## 基礎: 前提となる概念
反応経路(reactive path)とは、現在の観測から直接、安全な制御(障害物回避、車線維持)を高頻度で出す経路です。レイテンシ(反応の遅れ)が安全に直結するため、軽さが最優先です。10〜20Hz(1秒に10〜20回)で回す必要があります。

世界モデル(world model)とは、環境がこれからどう変化するかを内部で予測するモデルです。「自分がこう動いたら、世界はこうなる」というダイナミクス(時間変化の法則)を学び、それを使って計画します。Ha & Schmidhuber (2018) が概念を整理し、Dreamer系やGAIA-1が大規模化しました。

埋め込み(embedding)とは、画像のような高次元データを、意味を保ったまま固定長のベクトルに写したものです。1フレームの多視点画像を「そのシーンを表す1本の256次元ベクトル」に集約する、といった使い方をします。

リングバッファ(ring buffer)とは、固定長の配列で、新しい要素を追加すると最古の要素が押し出される記録方式です。直近T個の履歴だけを一定メモリで保持できます。先頭ポインタをぐるぐる回して上書きするので「リング(輪)」と呼びます。

JEPA(Joint-Embedding Predictive Architecture)とは、ピクセルを直接予測するのではなく、特徴(埋め込み)空間で隠れた部分や未来を予測する自己教師あり学習の枠組みです。詳細は後述します。

凍結(freeze)とは、あるネットワークの重みを学習中に更新しない(勾配を流さない)ことです。予測のターゲット(正解側の特徴)を安定させたいときに使います。EMA(指数移動平均)はその緩い版で、ゆっくり追従するコピーを正解側に使います。

## 仕組みを詳しく
世界モデル側で時間文脈を持つ典型的なパイプラインを、段階的に分解します。ここではカメラ複数台(たとえば7台)構成を想定します。

ステップ1: 低頻度サンプリング。1秒に1回(1Hz)、多視点画像を取得します。反応経路がたとえば10〜20Hzで回るのに対し、ここは1Hzなので計算負荷は桁違いに小さく、長い履歴を持っても重くなりません。10秒分を持っても10フレームで済みます。

ステップ2: 凍結backboneで特徴抽出。各視点を凍結した事前学習済みbackboneに通します。凍結する理由は2つあります。(a) backboneが反応経路の学習でも共有される場合、世界モデル用の予測損失と勾配が衝突するのを避ける。(b) 後述のJEPAで予測のターゲット(正解の特徴)を固定し、学習を安定させる。

ステップ3: 多視点を1本の埋め込みに集約。複数視点の特徴を、1つの学習されたクエリによるcross-attention(クエリが各視点の特徴を参照して重み付き集約する仕組み)で1本のベクトルにまとめます。テンソルで書くと、各視点を平均プールして `[B, V, 256]`(B=バッチ, V=視点数, 256=チャネル)にし、学習クエリで `[B, 1, 256]` へ集約するイメージです。ここではBEVのような厳密な幾何投影は使わないことが多いです。予測用の表現なので、ピクセル単位の正確な座標対応より、シーンの意味をコンパクトにまとめることが目的だからです。

ステップ4: Encoded Visual History として蓄積。各1Hzフレームの集約埋め込み `[B, 256]` を、固定長Tのリングバッファにrolling(最古を押し出して追加)で溜めます。形は `[B, T, 256]` です。十数次元ではなく256次元の特徴をそのまま時系列で持つため、低次元ボトルネックが解消されます(詳細は [時間文脈の低次元ボトルネック問題](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0474_visual-history-bottleneck))。

```
時刻 t-T+1 ... t-1   t   の埋め込み:
  [256] [256] ... [256] [256]
  └────────── T 本 ──────────┘   ->  バッファ [B, T, 256]
```

ステップ5: 時間融合して未来を予測、計画へ供給。バッファ `[B, T, 256]` を時間方向に融合します。融合方式の候補は、BEVFormer型のtemporal self-attention(過去T本に対する自己注意)や、GRU(Gated Recurrent Unit: 系列を1本のベクトルに畳み込む再帰ネットワーク)などです。融合結果から未来の特徴を予測し、その予測と時間文脈を計画モジュール(planner)へ渡します。

before / after で対比すると、before は「反応経路の中で十数次元×数十ステップの貧弱な履歴を持つ」、after は「世界モデル経路で256次元×Tのリッチな履歴を1Hzで持ち、未来予測に使い、計画へ渡す」です。

学習の中心はステップ5の予測損失です。ピクセル空間ではなく特徴空間で「未来の見え方」を再構成するJEPA流の損失を使います。具体的には、時刻 t までの履歴から時刻 t+k の(凍結backboneが出す)正解特徴を予測し、両者の距離(L2やコサイン)を縮めるよう学習します。これにより、予測ヘッドは細かなピクセルのノイズを当てる無駄を避け、意味のある抽象表現を学べます。ピクセル予測だと「葉っぱ1枚の揺れ」まで当てに行って計算を浪費しますが、特徴予測なら「あの車が前進する」という意味だけを当てればよいのです。

## 手法の系譜と主要論文
この設計は、自己教師あり表現学習と世界モデルという2つの系譜の合流点にあります。

- World Models (Ha & Schmidhuber, NeurIPS 2018, arXiv:1803.10122) — 観測をVAEで潜在に圧縮し、RNNでダイナミクスを予測し、その内部モデル(夢)の中で方策を学ぶ枠組みを提示しました。「速い制御」と「遅い予測」を分け、予測を潜在空間で行うという、本トピックの思想的な源流です。

- BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270) — 固定長キューに対するtemporal self-attentionで特徴レベルの時間融合を確立しました。Encoded Visual Historyの融合段(`[B, T, 256]`に対する時間注意)はこの方式を流用できます。

- I-JEPA (Assran et al., CVPR 2023, arXiv:2301.08243) — 画像の一部を隠し、ピクセルではなく特徴空間でその部分を予測する自己教師あり学習。ターゲット側のencoderをEMA(指数移動平均: 学習中の重みを滑らかに追従させた、ゆっくり動くコピー)で更新し、表現の崩壊を防ぎます。本トピックの「凍結/EMAターゲットで特徴空間予測」の直接の根拠です。

- V-JEPA (Bardes et al., 2024, arXiv:2404.08471) — I-JEPAを動画へ拡張し、時間方向に未来の特徴を予測します。Encoded Visual Historyからの未来特徴再構成は、この系譜にあります。

- DreamerV3 (Hafner et al., 2023, arXiv:2301.04104) / GAIA-1 (Hu et al., 2023, arXiv:2309.17080) — 大規模な世界モデルで、潜在空間の予測から計画・生成を行います。特に運転ドメインのGAIA-1は、多視点動画を潜在で予測する大規模世界モデルとして、この設計方向の到達点の1つです。

系譜としては、(1) 潜在世界モデル(2018)→ (2) 特徴レベル時間融合(2022)→ (3) 特徴空間自己教師あり予測 JEPA(2023)→ (4) 動画・運転ドメインへの大規模化(2024)、という流れで、「履歴を低次元要約ではなく学習された特徴空間で持ち、未来予測に使う」設計が成熟してきました。

## 論文の実験結果(定量データ)
特徴空間予測と世界モデルの有効性は、複数のドメインで定量化されています。

- I-JEPA は ImageNet(1000クラスの画像分類ベンチマーク)で、ピクセル再構成型の自己教師あり(MAE等)より少ない計算量で同等以上の線形評価精度(凍結特徴の上に線形分類器だけを載せたときの正解率)を達成しました。報告では、ViT-Hで線形プローブ精度がおよそ70%台後半に達し、しかもMAEより事前学習が大幅に速い(同等精度に到達するための学習エポックがおよそ数分の1)とされます。意味するところは、「ピクセルの細部を当てるより、特徴空間で予測するほうが、意味的に有用な表現を効率よく学べる」ことです。

- V-JEPA は動画の表現学習で、Kinetics-400(動作認識ベンチマーク)やSomething-Something-v2(時間的因果が重要な動作データセット)で、凍結特徴の評価で強い結果を示しました。特に時間的な手がかりが本質的なSomething-Something-v2で、ピクセル再構成型(動画版MAE等)を明確に上回ったと報告されています。これは「特徴空間での未来予測が、運動・時間構造の理解に効く」ことを示唆します。運転で必要な「他車がどう動くか」の予測と問題構造が近いです。

- DreamerV3 は、Atari、DMLab、ProcGen、Minecraft(ダイヤモンド採掘という長期計画タスク。人間データなしで初めて達成)など150以上の多様な環境を、ハイパーパラメータ固定で解き切りました。潜在世界モデルでの予測ベース計画(想像の中でロールアウトして方策を学ぶ)が、広いドメインで頑健に機能することを示した点が、世界モデル隔離の有効性を裏付けます。

- アブレーションの観点では、JEPA系は一様にターゲットの崩壊防止策(凍結またはEMA)を抜くと性能が崩壊する(全特徴が同一ベクトルに潰れ、線形プローブ精度がほぼランダムまで落ちる)ことが報告されています。つまり「ターゲットを止める」工夫は飾りではなく必須要素です。

数値の読み方として、線形プローブ精度の数ポイント差や、事前学習計算量が数分の1で済むという差は、表現学習では大きな意味を持ちます。特に「少ない計算で良い表現」は、実機やコスト制約のある運転システムで決定的に重要です。

## メリット・トレードオフ・限界
メリット:
- 256次元×Tの特徴で履歴を持つため、低次元ボトルネックが解消される。
- 重い時間融合を遅い経路(1Hz)に隔離でき、速い反応経路を軽く保てる。
- 凍結/EMAターゲットによりJEPAの学習が安定し、崩壊しにくい。
- 計画と予測(reasoning)の両方に時間文脈を供給でき、機能拡張しやすい。

トレードオフ・限界:
- 反応経路と世界モデル経路の2系統を持つぶん、全体アーキテクチャが複雑になる。
- 1Hzの更新頻度が、高速で動く動的物体の追従に十分かは要検証。高速道路で時速100kmなら1秒に約28m進むため、状況により低頻度すぎる恐れがある。
- JEPAは崩壊回避の凍結/EMA設計が前提で、設計の自由度が制約される。EMAの減衰率など追加のハイパーパラメータが増える。
- 「経路を分けたこと」が運転性能を本当に上げるかは、オープンループ(録画データでの予測精度)とクローズドループ(実際に走らせて事故率や乗り心地を測る)の両方で検証が必要で、表現学習の指標が良くても運転が良くなる保証はない。

研究上の未解決課題として、(a) 反応経路と予測経路の最適な更新頻度の組み合わせ、(b) 世界モデルの予測誤差が計画に伝播する compounding error(誤差の累積。1ステップ先の小さな誤差が、それを入力にした2ステップ先でさらに増幅し、長期予測ほど雪だるま式に悪化する現象)の抑制、(c) 特徴空間予測の品質を運転性能と結びつける評価指標、が挙げられます。

## 発展トピック・研究の最前線
最前線では、世界モデルを生成モデル(将来の多視点動画そのものを生成する)として大規模化する流れ(運転ドメインのGAIA-1や後続のドライビング動画生成モデル、拡散モデルベースの生成世界モデル)と、JEPAのように生成せず特徴空間でだけ予測する効率重視の流れが並走しています。前者は解釈性と多様なシナリオ生成(レアケースの合成データ生成にも使える)に強く、後者は計算効率と表現の有用性に強い、という住み分けです。

また、反応と予測の階層分離は、近年の「System 1 / System 2」型の自動運転アーキテクチャ(速い反射と遅い熟慮を分ける)とも響き合います。視覚言語モデル(VLM)による遅い推論(reasoning)を世界モデル経路に接続し、「なぜその行動を選ぶか」を言語で説明させる研究も現れており、Encoded Visual Historyのような時間文脈バッファは、その遅い推論への入力(過去のシーン文脈を言語モデルに渡す橋渡し)として位置づけ直されつつあります。
