# ego モーションによるBEVワープ

## ひとことで言うと
自車(ego = 自分が運転している車)が前進・旋回すると、センサーから見た周りの景色は「自分が動いたぶんだけ」ずれて見えます。このずれは自車の移動量さえ分かれば計算だけで予測できます。ego モーションによるBEVワープとは、その「自分が動いたぶんのずれ」を、学習(ニューラルネットの重み)ではなく純粋な幾何計算(座標の付け替え)で先回りして作り出す処理です。未来BEV予測の「易しい大部分」を計算で片付け、ネットを「難しい残差」へ集中させる土台になります。

## 直感的な理解
カメラを手に持って前へ歩く動画を撮ると想像してください。止まっているテーブルや椅子は、自分が前へ進むぶん、画面の外側へ流れて大きくなっていきます。家具が動いたわけではなく、自分が動いたから。自分の歩いた距離さえ分かれば、次のコマで家具がどこへ流れるかは、定規と分度器だけで計算できます。ニューラルネットに当てさせる必要はありません。

BEV(上空から見下ろした地図状の特徴)でも同じです。自車が1秒で8m前進したなら、止まっている建物・駐車車両・路面標示はすべて8mぶん手前へずれる。この「全体が一様にずれる」変化は、座標を付け替えるだけで再現できます。学習で再現しようとするのは、まるで「足し算の答えを暗記で覚える」ような非効率さ。ego ワープは、この計算で済む部分を計算に任せます。

## 基礎: 前提となる概念

前提語を平易に説明します。

- **BEV(Bird's Eye View、鳥瞰図)**: カメラ画像をそのまま使うのではなく、上空から地面を見下ろした2次元マップへ変換した特徴。典型的には数百×数百のグリッド(マス目)で、1マスが実世界の数十cm(例: 0.4m)を表します。各マスに「そこに何があるか」を表す数百次元の特徴ベクトルが入ります。450×300・0.4m/マスなら実世界 180m × 120m に相当します。

- **特徴(feature)**: 画像を直接扱う代わりに、ニューラルネットが画像から取り出した意味のある数字の並び。「ここは道路っぽい」「車がいそう」といった情報が圧縮されています。

- **エゴモーション(egomotion)**: 自車自身の動き。短い時間Δの間に自車が前後へ `dx`、左右へ `dy`、向きを `dθ` だけ変えた、という3つの量で平面運動を表します。車載では IMU・車輪オドメトリ・GPS などから取得できます。

- **未来予測(future state prediction)**: 「今の景色(時刻 t)から、数秒後のBEVがどうなるか」をモデルに当てさせるタスク。

- **メトリック(実寸)・ego中心グリッド**: BEVの1マスが実世界の決まった距離に対応し、原点が自車にあるグリッド。ワープは実距離 `(dx, dy, dθ)` をマス数へ換算する必要があるので、この性質が必須です。逆に、実寸の定義が無い特徴マップにはワープは適用できません。

ここで問題になるのが、時刻 t から t+Δ にかけてBEVが変化する理由が2種類あることです。

1. **ego起因の見かけの運動**: 自車が前進・旋回したせいで、止まっている景色"全体"がスーッとずれる・回る。自車の動きが分かれば完全に計算できる予測可能な変化。
2. **動的な運動(dynamic motion)**: 他の車・歩行者が"実際に"動いたことによる変化。本当に不確実で予測が難しい部分。

従来の単純な未来予測器は、この2種類を区別せず、ひとつの学習された差分(delta、Δ)でまとめて表現しました。「未来BEV = 今のBEV + 学習した差分」という足し算です。ところがego起因のずれは本質的に「足し算」ではなく「座標の付け替え(空間変換)」です。景色全体が右へ5マスずれる変化を、各マスの値の足し算だけで正確に表すのは無理があります(マス目をまたいで内容が移動するから)。だから ego起因のずれは計算で先に取り除き、ネットには「本当に難しい動的な部分」だけを学ばせる、というのが ego ワープの動機です。

## 仕組みを詳しく

中核は、未来BEVを次のように分解することです。

```
z_{t+Δ}  ≈   Warp_ego(z_t)              +        Δ_dynamic
            └─ 既知の ego 姿勢変化 ─┘     └─ ネットがこれだけ予測 ─┘
```

- `z_t`: 時刻 t の BEV 特徴。形 `[B, C, H, W]` = `[バッチ, 256, 450, 300]`。
- `Warp_ego(z_t)`: 今のBEVを「未来の自車から見た座標系」へ再サンプリング(座標を付け替えて貼り直す)したもの。動かす量は時刻 t から t+Δ までの自車移動量 `(dx, dy, dθ)`。
- `Δ_dynamic`: ワープしただけでは合わない部分、つまり他エージェントが実際に動いた残差。これだけを予測器(畳み込み層など)が出す。

ステップで書くと次のとおりです。

1. 自車の未来移動量 `(dx, dy, dθ)` を用意する。学習時はログに記録された正解の未来エゴモーションを使う。推論時は計画した軌道を使う(詳しくは [scheduled sampling によるワープ入力の切替](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0580_scheduled-sampling-warp-source))。
2. その移動量をBEVのグリッド座標系へ変換し、`Warp_ego(z_t)` を計算する(実装は [affine_grid と grid_sample による微分可能ワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0912_affine-grid-grid-sample-warp)。`affine_grid` + `grid_sample` で微分可能に行う)。
3. 予測器に、ワープ済み特徴あるいは今の特徴を入れて残差 `Δ_dynamic` を出させる。
4. 最終出力 `Warp_ego(z_t) + Δ_dynamic` を、未来の正解BEVと比べて学習する(特徴再構成 / JEPA損失)。

数値イメージ(前進だけのケース): 自車が1秒で8m前進したとします。1マス0.4mなので8m = 20マス。すると止まっている景色は全部20マスぶん手前へずれるのが幾何的に分かる。ワープはこの20マスのずれをそのまま作り、残差ネットは「あの交差点の車が予想より速くて、本来より3マス余分に進んだ」といった"ズレのズレ"だけを担当します。

回転を含む一般のケースでは、`(dx, dy, dθ)` から2次元剛体変換を組み、BEVグリッド全体を回転+並進させてから再サンプリングします。具体的には、現在のBEV座標 `p_t` を未来ego座標 `p_{t+Δ}` へ移す剛体変換は次の形です。

```
p_{t+Δ} = R(dθ) @ (p_t - translation)
          R(dθ) = [ cos dθ  -sin dθ ]
                  [ sin dθ   cos dθ ]
```

ここで `dθ` が向きの変化(ヨー角)、`translation` が `(dx, dy)` をマス数へ換算した並進です。たとえば自車が右へ 5 度旋回しながら 8m 前進したなら、止まっている景色は全体として 5 度逆回転し、かつ 20 マス手前へずれる、という合成変換になります。ワープ(再サンプリング)は実際にはこの逆変換、つまり「未来座標の各マスは現在BEVのどこから値を取るか」を計算します(逆向きマッピングは [affine_grid と grid_sample による微分可能ワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0912_affine-grid-grid-sample-warp) 参照)。重要なのは、この分解で特徴の shape が一切変わらない点です(入力も出力も `[B, 256, 450, 300]`)。だから既存の損失関数も学習ループもそのまま使え、提案が「小さく追加的(opt-in)」と言われるのはこのためです。

設計上の注意として、ワープを置く場所は「メトリックな ego中心 BEV グリッドが存在する側」でなければなりません。実寸の定義が無いマルチビュー特徴や潜在表現に対しては、実距離をマス数へ換算できないためワープは意味をなしません。

## 手法の系譜と主要論文

ego起因の運動を明示的に分離し、ネットには動的残差だけを学ばせる発想は、世界モデル(未来の状態を予測する内部モデル)系の研究に共通します。

- **STN (Spatial Transformer Networks、Jaderberg et al., NeurIPS 2015, arXiv:1506.02025)**: ネットの中に「入力を平行移動・回転・拡大縮小して整える微分可能なモジュール」を埋め込めることを示した。ego ワープはこの機構(grid_sample)だけ借り、変換パラメータは学習せず既知の ego 移動量で与える点が違います([affine_grid と grid_sample による微分可能ワープ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0912_affine-grid-grid-sample-warp))。

- **MILE (Hu et al., NeurIPS 2022, arXiv:2210.07729)**: カメラ映像から BEV を潜在空間(圧縮された内部表現)で予測する世界モデル。BEVを ego中心座標系で表し、未来をロールアウト(連続予測)する設計。自車の動きを座標系へ織り込むことで未来予測を安定させた。トレードオフは潜在世界モデルの学習が重く、誤差が時間方向に蓄積しやすいこと。

- **GAIA-1 (Hu et al., 2023, arXiv:2309.17080)**: 大規模生成モデルで運転シーンの未来を映像として生成。膨大なデータと計算でありえる未来を多様に生成できるのが強みだが、計算コストが極めて大きく、ego ワープのような軽量幾何変換とは方向性が逆(ワープは"計算で消せる部分は消す"省力路線)。

- **UniAD (Hu et al., CVPR 2023, arXiv:2212.10156)**: 検知・追跡・地図・予測・計画を1つのネットへ統合し、未来予測モジュールを持つ。自車運動とエージェント運動を扱う設計で、幾何的に根拠づけられた自己教師信号が backbone へ良い勾配を与える主張の裏づけに引用されます。

これらに対し、ego ワープの独自性は「ego部分は学習せず幾何計算で消す(パラメータフリー)」という割り切りにあります。すべてを学習に任せる代わりに、分かっている物理は計算に任せる、古典的だが堅実な発想です。

## 論文の実験結果(定量データ)

- **MILE**: CARLA の評価で、ego中心BEVの潜在ロールアウトを用いた設計が、ピクセル予測ベースラインを上回ることを報告。総合指標 Driving Score(ルート完走と違反ペナルティを統合、高いほど良い)で先行手法を上回り、学習中に見ていない長いルート区間でも一貫した運転を維持できることを示しました。自車運動を座標系へ織り込む(=ワープ的な発想)ことが、長期ロールアウトの安定性に効くという定量的根拠です。

- **STN**: 歪んだ MNIST(回転・スケール・並進を加えた手書き数字)で、空間変換モジュールを挿入すると分類誤り率が大きく下がることを報告(歪み MNIST で誤り率がベースラインの数分の1まで低下する条件あり)。微分可能な空間変換がCNNの位置・向きへの脆弱さを補えることを示しました。ただし変換パラメータを学習させると不安定になりやすいことも報告されており、ego ワープが変換パラメータを学習せず既知量で与えるのは、この不安定性を回避する選択でもあります。

- 自己教師あり奥行き推定の系譜(SfMLearner 系)では、既知/推定したカメラ運動でフレームをワープして次フレームを再構成する測光損失が、ラベルなしで KITTI 上の奥行き推定を学べることを示しました。動く物体やオクルージョンでワープ前提が崩れる領域では誤差が増えることも報告され、これがまさに ego ワープにおける `Δ_dynamic` の担当範囲に対応します。

- **占有予測の系譜(FIERY, ST-P3, OccWorld)**: FIERY (Hu et al., ICCV 2021, arXiv:2104.10490) は、複数フレームのBEVを共通の現在ego座標系へ揃えてから未来のインスタンス占有を予測する設計で、nuScenes の未来インスタンスセグメンテーションにおいて、過去フレームをego運動で整列(=ワープ)させることが予測精度(Future Video Panoptic Quality など)に寄与することを示しました。整列を行わずフレームをそのまま重ねると、自車運動ぶんのずれが「見かけの動き」として混入し、予測が劣化します。これは ego ワープの効用を間接的に裏づける定量的証拠です。さらに OccWorld (Zheng et al., ECCV 2024, arXiv:2311.16038) のような占有ベース世界モデルでも、ego pose を条件として未来占有を自己回帰予測する際、自車運動を明示的に与えるほど一貫性が増すことが報告されています。

ego ワープ分解そのものの大規模実データでの効果は提案段階で未確立ですが、上の各結果は「自車運動を幾何で扱う(=フレームをego整列する)と予測が安定する」「微分可能ワープは有効だが変換は学習させない方が安定」「ワープ前提は動的物体で崩れる」という、設計を支える定量的事実を提供します。

## メリット・トレードオフ・限界

メリット
- ネットの学習容量を「本当に難しい動的部分」へ集中させられる(ego の予測可能なずれを肩代わりさせない)。
- パラメータフリー(学習する重みが増えない)なので計算も軽く、ego の動きが分かれば常に正確。
- ワープが予測を実世界の幾何へアンカー(固定)するので、自己教師あり予測で起きがちな表現崩壊(全部同じ出力へ潰れる退化)を防ぐ強い事前知識になる([ego起因の見かけ運動と動的運動の分離](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0023_ego-induced-vs-dynamic-motion))。
- shape を変えないので既存の損失・ループを壊さない。

トレードオフ・限界
- BEV がメトリックで ego中心でないと使えない(実寸 m/マス、向きの定義が固定されている必要)。
- 軸・符号の規約(どの軸が前後でどれが左右か、ヨー角の正の向き)をBEV生成側と完全一致させないと、逆向きにワープして壊れる。エラーは出ないので発見しにくい。
- 学習時のワープ元(正解エゴモーションか予測軌道か)の選び方で train/test の不一致が起きうる([scheduled sampling によるワープ入力の切替](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0580_scheduled-sampling-warp-source))。
- ワープで未来座標系へ動かすと、今の視野の外(画面の端より先)の情報は埋められず、パディング領域(欠損)ができる。残差ネットが補えないとそこは情報なしのまま残る。

## 発展トピック・研究の最前線

ego ワープは「占有予測(occupancy forecasting)」とも親和的で、占有グリッドを ego運動で warp してから動的占有を残差で予測する研究が出ています。また長期予測では1ステップのワープを繰り返す自己回帰ロールアウトになり、誤差の蓄積をどう抑えるか(scheduled sampling、複数ステップ損失)が論点です。さらに、ワープ前提が崩れる激しい動的シーン(交差点の混雑など)では残差が支配的になり、幾何分解の利得が薄れる。どこまで幾何で説明し、どこから学習に委ねるかの境界設計が、研究の最前線にあります。
