# Score Matching / Denoising Score Matching

## ひとことで言うと
Score Matching(スコアマッチング)は、「データの確率分布そのもの」ではなく「分布の対数の勾配(=スコア)」を直接学習する技術です。スコアとは「今いる点から、データがありそうな方向はどちらか」を指す矢印のことです。確率分布をきちんと扱おうとすると避けて通れない「全部足して1にするための割り算(正規化)」が高次元ではほぼ計算不可能なのですが、スコアだけを学べばこの厄介な計算を丸ごと回避できます。とくに Denoising Score Matching(デノイジングスコアマッチング、略して DSM)は、「データにノイズを足して、そのノイズを当てる」という単純な回帰問題に置き換える巧妙な方法で、今日の拡散モデル全体の学習則そのものになっています。

## 直感的な理解
山道を霧の中で歩いていると想像してください。目的地は「データが密集している谷底」です。視界はゼロですが、足元の傾き(どちらが下りか)だけは杖で測れるとします。この「足元の傾き」がスコアです。傾きに従って一歩ずつ下りていけば、いつか谷底に着けます。生成モデルでやりたいのはまさにこれです。ランダムな砂嵐(空間のどこか適当な点)から始め、各地点でスコア(=下り方向)を頼りに動いていけば、本物データの多い領域へたどり着けます。

問題は「傾きを測る杖」をどうやって手に入れるかです。本当の傾き(真のスコア)は分からない、本当の地形図(真の分布 p(x))も持っていない。手元にあるのは地形のサンプル点(訓練データ)だけです。Score Matching は「サンプル点だけから、空間のどこででも傾きを答えられる杖(ニューラルネット)を作る」方法だと言えます。重要なのは、これが従来の生成モデルの「分布を直接表して尤度を最大化する」道とまったく違う発想だということです。尤度の道は正規化定数という巨大な壁にぶつかりますが、スコアの道はその壁を最初から避けて通ります。

## 基礎: 前提となる概念

### 確率密度と正規化定数

「確率分布」とは、ある値 x がどれくらい出やすいかを表す関数 p(x) です。連続値だと厳密には「確率密度」と呼び、全空間で積分すると 1 になる、という決まりがあります。ニューラルネットで分布を表すとき、典型的には次のエネルギーベースモデル(EBM, Energy-Based Model)という形になります。

```
p(x) = exp(-E(x)) / Z
```

- E(x) はネットが出す「エネルギー」。値が小さいほど「ありそう」。
- Z は正規化定数(partition function、分配関数)で、Z = ∫ exp(-E(x)) dx。全空間にわたる積分です。

確率は全部足すと 1 でなければならないので、この Z で割って帳尻を合わせます。ところが Z は、x が高次元(たとえば画像 [3,256,256] = 196608 次元)だと、その全空間にわたる多重積分になり、現実には計算できません。これが尤度ベース生成モデルの長年の難所でした。歴史的には、この Z を回避・近似するためにさまざまな手法が考案されてきました。コントラスティブ・ダイバージェンス(CD, Hinton 2002)は MCMC で Z の勾配を近似し、ノイズコントラスティブ推定(NCE, Gutmann & Hyvärinen 2010)はノイズ分布との二値分類問題に変換しました。GAN がこの問題を「分布を陽に書かず、サンプルを出すネットだけ作る」ことで回避したのに対し、Score Matching は「分布の勾配だけ扱う」という第三の道で回避します。

### 勾配・ベクトル場としてのスコア

「スコア」は数式で ∇_x log p(x) と書きます。∇_x は「x の各成分で偏微分してベクトルにする」という記号です。x が d 次元なら、スコアも d 次元のベクトルで、各成分は「その方向に x を少し動かすと log p がどれだけ増えるか」を表します。これを空間の各点に矢印として描くと「ベクトル場(vector field)」になります。データが密集する場所(分布のピーク)ほど、まわりからその場所を指す矢印が集まり、ピークそのものではスコアがゼロ(これ以上登るところがない)になります。注意すべきは、ここでいう「スコア」は統計学で別の意味で使われる「パラメータに関する対数尤度の勾配(Fisher のスコア)」とは異なり、データ変数 x に関する勾配だという点です(Stein score とも呼ばれます)。

### なぜ Z が消えるのか(核心)

Hyvärinen(2005)の着眼点はここです。p(x) = exp(-E(x))/Z の対数を取って x で微分してみます。

```
log p(x) = -E(x) - log Z
∇_x log p(x) = -∇_x E(x) - ∇_x log Z = -∇_x E(x)
```

log Z は x に依存しない定数なので、x で微分すると 0 になり、消えます。つまりスコアには Z が一切現れません。計算不能だった正規化定数を、微分によって完全に回避できる――これがスコアを学ぶことの最大の動機です。言い換えれば、エネルギー関数 E(x) を「定数のずれを除いて」決められれば十分であり、その定数こそが扱えない Z なのです。

## 仕組みを詳しく

### 素朴なスコアマッチング(Hyvärinen 2005)

理想は「ネットが出すスコア s_θ(x)」を「真のスコア ∇_x log p_data(x)」に一致させることです。素直に二乗誤差で書くと次の目的関数(Explicit Score Matching, ESM)になります。

```
J_ESM(θ) = (1/2) E_{x~p_data} [ || s_θ(x) - ∇_x log p_data(x) ||^2 ]
```

しかし真のスコア ∇_x log p_data(x) は未知なので、このままでは計算できません。Hyvärinen は部分積分(積分の境界項が分布の裾で消えるという穏やかな仮定のもとで微分をずらすテクニック)を使い、真のスコアを使わずに済む等価な目的関数を導きました。

```
J_ISM(θ) = E_{x~p_data} [ (1/2) ||s_θ(x)||^2 + tr( ∇_x s_θ(x) ) ] + const
```

これを Implicit Score Matching(ISM)と呼びます。tr( ∇_x s_θ(x) ) は、スコアのヤコビ行列(各出力成分を各入力成分で微分した d×d 行列)の対角和、つまりスコアの「発散(divergence)」 Σ_i ∂s_θ,i/∂x_i です。const は θ に依存しない定数なので最適化では無視できます。J_ISM はデータ x のサンプル平均だけで計算でき、真のスコアを必要としません。理論的に美しい結果です。直感的には、第1項がスコアの大きさを抑えつつ、第2項(発散)がスコア場を「データ点へ向かって収束する」形に整える役割を果たします。

ただし致命的な実用上の問題があります。tr(∇_x s_θ(x)) を厳密に計算するには、d 次元のそれぞれについて、出力 s_θ の対応する成分を入力 x の対応する成分で微分する必要があり、自動微分では d 回の逆伝播に相当します。d が数十万に及ぶ画像では1サンプルあたり数十万回の逆伝播が必要になり、絶望的に重い。だから素朴な ESM/ISM は密度推定や独立成分分析など低次元データにしか使えませんでした。

### Sliced Score Matching(Song et al. 2019)

ヤコビ対角和を回避する一つの道が Sliced Score Matching(SSM, Song, Garg, Shi, Ermon, UAI 2019)です。ランダムな方向ベクトル v(標準正規またはRademacher)を引き、tr(∇_x s_θ) の代わりに v^T (∇_x s_θ) v の v に関する期待値を使う(Hutchinson のトレース推定 E[v^T A v] = tr(A))ことで、1回の方向微分(ベクトル・ヤコビ積、自動微分で1回の逆伝播相当)で済ませます。これにより次元 d によらず一定計算量になり、高次元でも一応学習可能になります。SSM は ISM の不偏推定量なので「ぼかさない真の分布」のスコアを学べる利点がありますが、トレース推定の分散が大きく、次に述べる DSM ほど安定はしません。後に Pang et al.(2020)は有限差分でヤコビ計算自体を近似する Finite-Difference SSM も提案しています。

### Denoising Score Matching(Vincent 2011)

Vincent はヤコビ計算の重さを、まったく別の発想で解消しました。「きれいなデータ x に小さなガウスノイズを足してノイズ入りデータ x̃ を作り、そのノイズ入り分布のスコアを学ぶ」という置き換えです。

ノイズの強さ(標準偏差)を σ とすると、x̃ = x + σ·ε(ε は標準正規乱数 N(0, I))です。ノイズを足したことで分布は p_σ(x̃) = ∫ p_data(x) N(x̃; x, σ²I) dx(真の分布をガウスでぼかした畳み込み)となります。Vincent は、この p_σ のスコアを学ぶ ESM 目的関数が、次の驚くほど単純な形と等価になることを部分積分で証明しました。

```
J_DSM(θ) = E_{x~p_data, ε~N(0,I)} [ || s_θ(x̃) - ( - ε / σ ) ||^2 ]      (x̃ = x + σ·ε)
```

ここで -ε/σ は、条件付き分布のスコア ∇_{x̃} log N(x̃; x, σ²I) = (x - x̃)/σ² = -ε/σ に等しく、「足したノイズを打ち消す方向」です。証明の鍵は、周辺分布のスコア ∇ log p_σ(x̃) が条件付きスコア ∇ log p_σ(x̃|x) の事後期待値になる(E[(x - x̃)/σ² | x̃] に等しい)という関係で、二乗誤差を最小化する解がちょうど条件付き期待値になる、という回帰の基本性質から従います。つまりネットに「x̃ にどんなノイズが乗っているか、どっちへ戻せばきれいになるか」を当てさせているだけ。ヤコビ計算は一切要らず、ただの二乗誤差回帰になります。これなら画像のような高次元データでも GPU で素直に学習できます。

数値例で確かめます。x が [3, 256, 256] の画像、σ = 0.1 とします。ε を同じ形状の標準正規乱数として x̃ = x + 0.1·ε を作る。ネット s_θ(x̃)(同じ形状を出力)が -ε/0.1 = -10ε に近づくよう二乗誤差を最小化する。学習後、ネットは「ノイズ入り画像を見て、全ピクセルについてどう動かせばきれいになるか」を言えるようになります。Tweedie の公式(Robbins 1956; Efron 2011)という別角度の見方をすると、ガウスでぼかした観測 x̃ に対する元データの事後平均が E[x|x̃] = x̃ + σ² ∇ log p_σ(x̃) で与えられ、スコアを学ぶこととノイズ除去(最適なデノイザを作ること)が数学的に同じ営みであることが分かります。

### DSM が学ぶのは「ぼかした分布」のスコアである、という重要な注意

DSM が一致させる先は、真の p_data ではなく、ノイズでぼかした p_σ のスコアです。σ → 0 の極限では p_σ → p_data なので理論的には真のスコアに近づきますが、σ が小さいと教師信号 -ε/σ の分散が σ⁻² で爆発し、学習が不安定になります。逆に σ が大きいと安定するが、ぼけた分布のスコアしか学べず、生成物がぼやけます。「小さすぎると役立たず、大きすぎると元から離れる」というこのジレンマが、後の [NCSN (Noise Conditional Score Networks)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0243_ncsn_note) が「複数の σ を同時に使う」ことで解いた問題そのものです。さらに ESM/ISM/SSM は真の分布のスコアを学ぶ(漸近的に不偏)のに対し、DSM は有限の σ では必ずぼかし分のバイアスを持つ、という質的な違いがあります。実用上は DSM の安定性と計算効率がこのバイアスを補って余りあるため、画像生成では DSM が標準になりました。

### DDPM とのつながり

[DDPM (Denoising Diffusion Probabilistic Models)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0229_ddpm_note) の損失「足したノイズ ε を予測する二乗誤差 E[ ||ε - ε_θ(x_t, t)||² ]」を思い出すと、これはまさに DSM です。DDPM の ε_θ と、ここでの s_θ は、x_t = √(ᾱ_t) x + √(1-ᾱ_t) ε という前向き過程のもとで s_θ(x_t, t) = -ε_θ(x_t, t) / √(1-ᾱ_t) という係数関係で結ばれます。つまり DDPM は変分下限(確率モデルの言葉、ELBO)で導いた手法ですが、その正体はデノイジングスコアマッチングでした。この対応を連続時間で明示的に統一したのが [Score-based Generative Models (SDE定式化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0250_score-based-sde) で、そこでは全ノイズ水準にわたる DSM を時間 t の重み付き積分として書きます。

## 手法の系譜と主要論文

- Hyvärinen (2005), "Estimation of Non-Normalized Statistical Models by Score Matching" (JMLR 6): 原典。正規化定数 Z を計算せず統計モデルを推定する ISM 目的関数を部分積分で導出した。動機は高次元で計算不能な Z の回避。理論的に厳密だが、ヤコビ対角和の計算が高次元で重く画像には非実用的という限界があった。
- Vincent (2011), "A Connection Between Score Matching and Denoising Autoencoders" (Neural Computation 23(7)): DSM を導入し、ノイズ除去オートエンコーダ(denoising autoencoder, DAE)の学習がノイズ入り分布のスコアマッチングと等価であることを証明。Hyvärinen 版のヤコビ計算を回避し、単純な二乗誤差回帰にした。これが今日の拡散モデルの学習則の直接の祖先。
- Raphan & Simoncelli (2011) / Efron (2011): Tweedie の公式を経験ベイズの文脈で整理。スコア推定とノイズ除去の等価性に別角度の理論的裏づけを与えた。
- Song, Garg, Shi, Ermon (2019), "Sliced Score Matching" (UAI 2019): ランダム射影でトレースを推定し、高次元でも ISM を回せるようにした中間的成果。DSM とは別系統(真の分布のスコアを学べる)だが、スコアマッチングを大規模化する流れの一部。
- Song & Ermon (2019), "Generative Modeling by Estimating Gradients of the Data Distribution" (NeurIPS 2019): DSM を「複数ノイズ水準」で運用し、初めて高品質な画像生成を実現した [NCSN (Noise Conditional Score Networks)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0243_ncsn_note)。σ の選び方問題を解いた。
- Song et al. (2021), "Score-Based Generative Modeling through SDEs" (ICLR 2021): DSM を連続時間に拡張し、DDPM と NCSN を統一した [Score-based Generative Models (SDE定式化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0250_score-based-sde)。

## 論文の実験結果(定量データ)

スコアマッチング自体は学習則(損失関数)であり、単体の「ベンチマークスコア」を持つものではありません。その威力は、スコアマッチングを採用した生成モデルの数値に現れます。評価指標 FID(Fréchet Inception Distance)は、生成画像と本物画像を学習済み Inception ネットの特徴空間に写し、両者の分布(平均と共分散)の距離を測るもので、低いほど本物に近いことを意味します。

- Hyvärinen(2005)/Vincent(2011)は、生成画質の数値ではなく、密度推定や表現学習の文脈で提示されました。Vincent の DSM は当初「denoising autoencoder がなぜうまくいくか」の理論的正当化(DAE が局所的にスコアを学んでいる)として登場し、生成品質の数値はまだ示されていませんでした。
- Song, Garg, Shi, Ermon(2019, SSM)は、深層エネルギーモデルの学習で SSM が ISM とほぼ同等の精度を桁違いに少ない計算量で達成できることを示し、CIFAR-10 などでもスコアベース手法を回せることを実証しました。ただし生成画質は次の NCSN ほどではありませんでした。
- Song & Ermon(2019, NCSN)は DSM を複数 σ で運用し、CIFAR-10(32×32 自然画像、10 クラス)で FID およそ 25.3、Inception Score(IS、生成画像の鮮明さと多様性を測る、高いほど良い)8.87 を達成しました。これは当時のスコアベース手法として、強力な GAN に迫る初の数値でした。
- アブレーション的な知見として、Song & Ermon(2020, "Improved Techniques")は σ の最大値をデータ点間の最大距離程度に取る、L(σ の段数)を十分多くする、ネットの出力を 1/σ でスケールする、といった設計を1つずつ外すと品質が崩れることを示し、DSM 運用における σ スケジュールの重要性を定量的に裏づけました。改良版は CIFAR-10 で FID をおよそ 10 前後まで下げています。
- DDPM(Ho et al., 2020)は、DSM と等価な「ノイズ予測 ε-prediction」損失に切り替えることで、変分下限を直接最適化するより安定し品質が上がることを示しました。CIFAR-10 で FID 3.17、IS 9.46 を達成し、当時の最良クラスに到達。これは「DSM 形式の損失(さらに重み 1 の単純化版)」が実用上きわめて効果的であることの強い証拠です。

これらの数値が示すのは、「正規化定数を回避するという理論的トリック(Hyvärinen)」が「単純なノイズ予測回帰(Vincent)」へと実装可能な形に翻訳され、さらに「複数ノイズ水準(NCSN)」と「変分の言葉での再導出(DDPM)」を経て、最終的に GAN を超える画像生成([Classifier Guidance](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0224_classifier-guidance) 系)を支える土台になった、という一本の流れです。重要なのは、損失関数の選択(ESM か ISM か SSM か DSM か)が「何を学ぶか(真の分布か、ぼかした分布か)」と「計算量(d 倍か定数か)」を同時に決めるという点で、画像のような超高次元では計算効率が決定的だったため DSM が勝ち残りました。

## メリット・トレードオフ・限界

メリット
- 計算不能な正規化定数 Z を完全に回避できる。尤度ベースモデルの最大の障壁を取り除いた。
- DSM 版は単純な二乗誤差回帰になり、高次元データでも GPU で素直に学習できる。ヤコビも MCMC も要らない。
- DDPM・NCSN・SDE 拡散モデルすべての理論的土台であり、汎用性がきわめて高い。スコアとノイズ除去の等価性(Tweedie)により、学習済みデノイザがそのまま逆問題ソルバーにも使える。

トレードオフ・限界
- Hyvärinen の素朴版(ESM/ISM)はヤコビ計算が d 回の逆伝播に相当し高次元に非実用的(だから DSM/SSM が必要だった)。
- DSM が学ぶのは「ノイズを足した後」の分布 p_σ のスコアであり、真の p_data とは厳密にはずれる。σ → 0 で一致するが、その極限では教師信号 -ε/σ の分散が爆発する。
- ノイズ強度 σ の選択が品質を強く左右する。単一 σ では「データのない領域でスコアが学べない」問題が残る(NCSN が複数 σ で解決)。
- SSM は真の分布を学べるがトレース推定の分散が大きく、収束が遅い・不安定になりやすい。
- 未解決の課題として、低密度領域でのスコア推定誤差、多峰分布でのモード間の質量配分(モード間を正しい比率で行き来できるか)、離散データへの拡張(勾配が定義できない)、多様体上(球面・回転群など)でのスコアの定義などが研究対象として残っています。

## 発展トピック・研究の最前線

- 連続時間化: 単一 σ や離散的な複数 σ を、時間 t に沿った連続のノイズ過程に拡張したのが [Score-based Generative Models (SDE定式化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0250_score-based-sde)。これにより逆向き SDE / 確率フロー ODE という統一的なサンプリング理論が得られた。学習則は全時刻にわたる重み付き DSM の積分になる。
- Tweedie/経験ベイズの再評価: スコア=最適デノイザという等価性を逆問題(超解像・画像補完・MRI 再構成)に使う diffusion posterior sampling 系が、学習済みスコアを汎用の事前分布として再利用する流れを作っている。
- Consistency Models(Song et al., 2023): スコアではなく「ODE 軌道上の任意の点から終点へ直接写す関数」を学び、1〜数ステップ生成を可能にした。DSM の系譜の最新の高速化。
- Flow Matching / Rectified Flow(Lipman et al., 2023; Liu et al., 2023): スコアの代わりに「ベクトル場(速度場)」を条件付き回帰で学ぶ枠組み。DSM の発想(条件付き目標の回帰が周辺目標の回帰に等価)を一般化し、直線的な輸送経路で少ステップ生成を目指す。
- 離散・多様体上のスコア: 文章やグラフなど離散データ、あるいは球面・回転群などの多様体上でスコアマッチングをどう定義するか(concrete score、Riemannian score matching)は活発な研究領域。
