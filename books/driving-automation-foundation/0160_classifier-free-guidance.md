# Classifier-Free Guidance（CFG／分類器なし誘導）

## ひとことで言うと
Classifier-Free Guidance（CFG, 分類器なし誘導）は、Diffusion モデルや Flow Matching といった「ノイズから少しずつデータを作り出す生成モデル」に対して、「与えた条件（テキスト、ラベル、自動運転なら目的地や地図）にもっと強く従わせたい」ときに使うテクニックです。訓練のときにわざと条件を一定確率で消し、1つのネットワークに「条件あり版の予測」と「条件なし版の予測」の両方を覚えさせます。生成のときには、その2つの予測の差を増幅し、条件が指し示す方向へサンプルを強く押し出します。別途の分類器（classifier, ここでは「画像がどのクラスか判定するネット」のこと）を必要としないので Classifier-Free（分類器なし）と呼ばれます。

## 直感的な理解
画像生成で「赤い車」と指定したのに、出てきたのが少しオレンジっぽい車だった、という経験を想像してください。モデルは条件を「そこそこ」尊重しているものの、もっとビシッと従ってほしい。CFG はこの「もっと従わせる」ためのつまみです。

その仕組みのコアは、地図のたとえで理解できます。あなたが「条件なし」で適当に走ると、なんとなくそれらしい場所へ向かいます。「条件あり」で走ると、目的地の方向へ向かいます。この2つの向きの差ベクトルこそが「条件が生み出している方向（条件があることで進路がどれだけ変わったか）」です。CFG は、この差ベクトルを実際より大げさに掛け算して進むことで、「条件が要求している方向」をより強調します。掛ける倍率（guidance scale, 誘導の強さ）を上げれば条件にぴったり従うようになりますが、上げすぎると現実離れした不自然な結果になります。アクセルとブレーキのように、品質（条件への忠実さ）と多様性のバランスを調整するつまみだと考えてください。

## 基礎: 前提となる概念
CFG を理解するには、「条件付き生成モデル」と「Diffusion / Flow Matching」の基礎を押さえる必要があります。

条件付き生成モデルとは、「条件 c を与えると、それに合ったサンプル x を作る」モデルです。条件 c はテキスト、クラスラベル、自動運転なら自車の運動履歴・地図・周辺物体などです。数式で言えば、データの分布 p(x) ではなく条件付き分布 p(x|c) からサンプリングしたい、ということです。

Diffusion モデルは、データに少しずつノイズを足して最終的に純粋なノイズにする「前向き過程（forward process）」と、逆にノイズから少しずつノイズを取り除いてデータに戻す「逆向き過程（reverse process）」からなります。学習では、各時刻 t においてサンプルに乗っているノイズ ε を予測するネットワーク ε_θ(x_t, t, c) を作ります。θ はネットワークの重み、x_t は時刻 t でノイズが乗った状態です。「ノイズ予測の二乗誤差」を最小化するのが目的です。生成（サンプリング）では、純粋ノイズから出発し、ネットが予測したノイズを少しずつ引いていって、最終的にデータを得ます。

ここで重要な数学的事実があります。学習した ε_θ(x_t, t, c) は、本質的に「ノイズの乗ったデータの対数確率密度の勾配（スコア, score と呼ぶ）」∇_x log p(x_t|c) に比例します。スコアとは「いまの状態から、確率の高い方向はどっちか」を指す矢印のことです。条件付きスコアはベイズの定理から ∇_x log p(x_t|c) = ∇_x log p(x_t) + ∇_x log p(c|x_t) と分解できます。右辺の第1項が「無条件のスコア（とにかくデータらしい方向）」、第2項が「条件方向のスコア（c に合う方向）」です。CFG はこの第2項を w 倍に水増しする操作だ、と理解すると見通しが良くなります。これは後述する Classifier Guidance の理論を裏返したものです。

Flow Matching は、Diffusion を一般化したより新しい枠組みです（Lipman et al., 2023）。ノイズ x_0 から正解データ x_1 までを直線で結ぶ「経路」を考え、その経路上での「速度（velocity, どちらへどれだけ動くか）」を予測するネットワーク v_θ(x_t, t, c) を学習します。生成は、ノイズから出発して速度場を時間積分（Euler 法など）して x_1 に到達します。Diffusion がノイズを予測するのに対し、Flow Matching は速度を予測しますが、「学習目標」と「生成手続き」が別物である点は共通です。

「条件 c が空（null）」という概念も重要です。CFG では、条件を意図的に取り除いた状態を表すため、null 条件（記号 ∅）という特別な入力を用意します。実装上は、学習可能な「空ベクトル（null embedding）」を用意してそれを条件の代わりに差し込む、あるいは条件埋め込みをゼロにする、といった形で表現します。null 条件を「無条件モード」のスイッチだと考えてください。

## 仕組みを詳しく
CFG は訓練と推論の2段階に分かれます。

### 訓練フェーズ（条件ドロップアウト）
通常の条件付き学習に、一工夫加えます。各学習サンプルについて、ある確率 p_uncond（典型的には 0.1〜0.2、つまり 10〜20%）で条件 c を null 条件 ∅ に置き換えます。

```
for each training sample (x, c):
    if random() < p_uncond:
        c <- ∅            # 条件を捨てる（unconditional として学習）
    loss = || ε_θ(x_t, t, c) - ε_true ||^2   # 通常どおりノイズ（or 速度）予測
```

この結果、1つのネットワークが「条件あり ε_θ(x_t, t, c)」と「条件なし ε_θ(x_t, t, ∅)」の両モードを学習します。条件なしモードは「とにかくそれらしいサンプルを作る」一般生成器として、条件ありモードは「c に従う」生成器として機能します。重要なのは、両方を同じパラメータで共有していることです。別ネットを2つ持つのではなく、1つのネットが入力（条件か null か）によって振る舞いを切り替えます。重みを共有しているからこそ、両者の差分が「条件だけの効果」をきれいに切り出せるのです。

### 推論フェーズ（誘導の増幅）
生成の各ステップで、条件あり予測と条件なし予測の両方を計算し、その差を guidance scale w で増幅して合成します。

```
ε_guided = ε_θ(x_t, t, ∅) + w * ( ε_θ(x_t, t, c) - ε_θ(x_t, t, ∅) )
```

ここで `ε_θ(x_t, t, c) - ε_θ(x_t, t, ∅)` が「条件があることで生まれる予測の差」、すなわち条件方向のベクトルです。これを w 倍します。

- w = 0: 条件を完全に無視（unconditional 生成）
- w = 1: 普通の条件付き生成（差を 1 倍、つまり ε_guided = ε_θ(x_t, t, c)）
- w > 1（典型的に 3〜7.5）: 条件方向へ強く押し出す（条件適合が上がるが多様性は下がる）

なお論文によって w の定義が1ずれている（guidance weight を s = w+1 とする流儀がある）ので、他資料と数値を比べるときは「w=1 が素の条件付き生成」か「w=0 が素の条件付き生成」かを確認してください。

数値例: あるステップで条件なし予測が `[0.2, 0.5]`、条件あり予測が `[0.4, 0.6]` だとします。差は `[0.2, 0.1]`。w=3 なら `ε_guided = [0.2, 0.5] + 3*[0.2, 0.1] = [0.8, 0.8]`。元の条件あり予測 `[0.4, 0.6]` よりさらに条件方向へ押し出されているのが分かります。この「押し出しすぎ」が強くなると、後述する過飽和や軌跡なら過剰な機動につながります。

### Flow Matching への適用
Flow Matching では予測対象が速度場 v になりますが、形は全く同じです。

```
v_guided = v_θ(x_t, t, ∅) + w * ( v_θ(x_t, t, c) - v_θ(x_t, t, ∅) )
```

これは速度がスコアと線形に対応する（rectified flow / probability-flow ODE の枠組み）ため、差分増幅という操作がそのまま意味を持つからです。

### 計算コスト
CFG の代償は推論コストです。各サンプリングステップで「条件あり」と「条件なし」の2回ネットワークを呼ぶため、推論計算量が約2倍になります。実装上は、条件ありと条件なしの入力をバッチ次元で連結（B を 2B にする）して1回の forward で両方計算するのが定石ですが、それでも実質的な計算量（FLOPs, 浮動小数点演算回数）は2倍です。リアルタイム性が要求されるタスク（自動運転の運転中推論など、10〜20Hz で軌跡を出し続ける必要がある）では、この2倍は無視できないコストになります。

## 手法の系譜と主要論文
誘導（guidance）の歴史は、次のように展開してきました。

出発点は Ho et al., 2020, "Denoising Diffusion Probabilistic Models"（DDPM, NeurIPS, arXiv:2006.11239）で、Diffusion を実用化し「ノイズ予測の MSE」という単純な学習目標を確立しました。しかしこの段階では「条件への追従を強める」仕組みは未解決でした。

最初の解決策は Classifier Guidance（分類器誘導）でした。Dhariwal & Nichol, 2021, "Diffusion Models Beat GANs on Image Synthesis"（NeurIPS, arXiv:2105.05233）が提案。これは、ノイズだらけの途中画像 x_t を入力として「これはクラス y か？」を判定する専用分類器 p(y|x_t) を別途訓練し、その対数尤度の勾配 ∇_x log p(y|x_t) でサンプリングを条件方向へ押す方法です。前述のスコア分解の第2項を、分類器の勾配で直接計算するイメージです。この論文は、ImageNet 生成で当時の GAN（BigGAN-deep など）を画質指標で上回り、「Diffusion が GAN に勝つ」ことを示した歴史的な仕事です。しかし欠点がありました。(1) ノイズが乗った途中画像でも正しく分類できる専用分類器を、ノイズレベルごとに訓練しなければならず手間とコストが大きい。(2) 分類器の勾配に頼るため、分類器を騙すような不自然・敵対的なサンプルが出るリスクがある。(3) 品質を上げる代わりに多様性が下がる現象（後の CFG にも共通）。

次に Ho & Salimans, 2022, "Classifier-Free Diffusion Guidance"（NeurIPS 2021 Workshop での発表、arXiv:2207.12598 として 2022 公開）が CFG を提案しました。動機は「分類器を別に作るのは面倒で副作用もある。生成モデル自身に条件あり/なしの両方を覚えさせれば、その差で同じ誘導効果が得られる」というものです。やり方は前述の通り、訓練時の条件ドロップアウトと推論時の差分増幅です。効果は Classifier Guidance と同等以上の品質・条件適合を、専用分類器なしで実現することでした。この単純さと効果の高さから、CFG は急速に標準手法となりました。

実際の大規模展開では、Saharia et al., 2022, "Imagen"（NeurIPS, arXiv:2205.11487）が CFG を大きく押し出しました。Imagen は強い CFG が text-to-image の品質を劇的に上げる一方、強すぎると画素が飽和してしまう（saturation）問題を指摘し、その緩和策として「dynamic thresholding（各ステップで予測画素を分位点でクリップして範囲内に戻す）」を提案しました。これは「CFG は効くが副作用があり、別の工夫で抑える」という、その後の研究の典型パターンの先駆けです。現在の主要なテキスト画像生成モデル（Stable Diffusion, Imagen, DALL-E 系など）でも CFG はほぼ例外なく使われています。

Lipman et al., 2023, "Flow Matching for Generative Modeling"（ICLR, arXiv:2210.02747）は Flow Matching の枠組みを提示しました。CFG は元々 Diffusion 文脈で生まれましたが、速度場に対しても同じ差分増幅の形で適用でき、Flow Matching ベースの生成モデルでも CFG は広く併用されます。

近年は CFG 自体の改良が活発です。Karras et al., 2024, "Guiding a Diffusion Model with a Bad Version of Itself"（autoguidance, NeurIPS, arXiv:2406.02507）は、無条件モデルの代わりに「同じ条件だが容量を落とした/学習不足の劣化版モデル」を guidance の基準に使うと、多様性をほとんど犠牲にせず品質だけを上げられることを示しました。ほかに、誘導を一部の時間ステップだけに限定する「guidance interval」（強い誘導を全ステップでかけると弊害が出るため、効く区間だけに絞る）、誘導強度を時刻で変える動的誘導、CFG の2倍コストを蒸留（distillation）で1回推論に畳み込む「guidance distillation」も実用化されています。

自動運転の motion planning 文脈（軌跡生成に Diffusion / Flow を使う研究、2023〜）では、CFG を必須としない設計も多く見られます。理由はメリット・限界の節で述べます。

## 論文の実験結果（定量データ）
ここで使われる主要な評価指標を先に説明します。

- FID（Fréchet Inception Distance）: 生成画像の分布と本物画像の分布が、Inception ネットの特徴空間でどれだけ近いかを測る指標。小さいほど「本物らしく多様」で良い。
- IS（Inception Score）: 生成画像の「明瞭さ（クラスがはっきりしている）」と「多様性」を組み合わせた指標。大きいほど良い。

Dhariwal & Nichol（2021）の Classifier Guidance は、ImageNet 128×128 / 256×256 / 512×512 で当時の最高水準を達成しました。たとえば ImageNet 256×256 で FID 約 4.59 を報告し、BigGAN-deep（FID 約 6.95）を上回りました。重要な発見は「guidance を強めると FID は一旦改善するが、強くしすぎると今度は多様性が落ちて FID が悪化する」というトレードオフ曲線の存在です。一方で IS（条件適合の明瞭さ）は guidance を強めるほど単調に上がる傾向があり、FID と IS が逆方向に動くことが「品質と多様性のトレードオフ」の正体です。

Ho & Salimans（2022）の CFG は、分類器を一切使わずに同等以上の品質・条件適合トレードオフを再現しました。論文では、guidance scale を変えながら FID と IS をプロットし、Classifier Guidance と同じ形のトレードオフ曲線（強めると IS 上昇・FID は一旦改善後に悪化）を示しました。CFG の本質的な貢献は「専用分類器という追加の訓練対象・追加の失敗要因を排除しながら、誘導の効果はそのまま得られる」点にあります。

Imagen（Saharia et al., 2022）は、text-to-image ベンチマーク COCO で zero-shot FID 約 7.27 を報告し、CFG の強度を上げると画像とテキストの整合（CLIP score で測定）が上がる一方で FID が悪化するトレードオフを定量的に示しました。dynamic thresholding を入れると、高い CFG 強度でも飽和を抑えられ、整合と FID を両立しやすくなることを示しました。これが「CFG の強度を上げたいが副作用を抑えたい」というアブレーションの代表例です。

autoguidance（Karras et al., 2024）は、ImageNet 512×512 で FID 約 1.25 という当時のクラス条件生成の最高水準級を達成し、標準 CFG（FID 約 1.5 前後）を更新しました。アブレーションの核心は「guidance の基準を無条件モデルにすると多様性が落ちるが、劣化版の条件付きモデルにすると多様性をほぼ保ったまま品質が上がる」点で、CFG の副作用（多様性低下）の原因が「無条件分布へ引き寄せること」にあると示唆しました。

アブレーション的な観点では、条件ドロップアウト率 p_uncond の選び方が重要です。報告では 10〜20% 程度が広く使われ、低すぎると条件なしモードが十分学習されず誘導が弱く、高すぎると条件ありモードの学習が不足して品質が落ちる、というバランスがあります。w については、画像生成では 3〜7.5 程度がよく使われますが、タスクによって最適値は大きく変わります。

## メリット・トレードオフ・限界

メリット:
- 別途分類器を訓練せずに、条件への追従を後付けで強められる。失敗要因（分類器の訓練・敵対的勾配）を1つ減らせる。
- 推論時の係数 w で「条件をどれだけ尊重するか」を訓練後に調整できる。再訓練不要でつまみを回せる。
- 画像生成では品質・条件適合が大きく向上する実績があり、事実上の標準。

トレードオフ・限界:
- 推論コストが約2倍（各ステップで条件あり/なしの2回推論）。リアルタイム性が重要なタスクでは無視できない。
- w を上げすぎると多様性低下・不自然なサンプル（分布の歪み、彩度過剰、軌跡生成なら過剰・非現実的な機動）。
- 訓練時に条件ドロップアウトを仕込む必要があり、訓練手順がやや複雑化する。
- 条件が「常時必須」のタスクでは調整自由度がそもそも不要なことがある。自動運転の軌跡計画では、自車状態・地図・周辺物体といった条件は「無視すると即座に危険」な情報であり、画像生成のように「条件をどれだけ尊重するか調整する」ニーズが薄い。条件は常にフルに使う前提のため、CFG の追加コスト（推論2倍）に見合う便益が小さく、見送られることが多い。ただし「目的地ナビへの追従だけを強めたい」といった限定的な動機があれば、条件ドロップ＋差分増幅をナビ条件にのみ適用する余地はあります。
- CFG は分布を本来の p(x|c) から歪める（厳密にはサンプリング先が p(x|c)・p(c|x)^(w-1) のような幾何平均的分布になる）ため、尤度や多様性が理論的に保証されません。これが autoguidance のような「副作用の少ない誘導」研究の動機です。

## 発展トピック・研究の最前線
- Autoguidance / 自己誘導: 別の弱い（劣化版）モデルを「基準」役にすることで、CFG の多様性低下を緩和する手法（Karras et al., 2024）。
- Guidance interval / dynamic guidance: 全ステップに一律 w をかけると弊害（過飽和、細部の崩壊）が出るため、誘導を効かせる時間区間や強度を動的に制御する研究が進んでいます。多くの実装で「ノイズの大きい初期ステップだけ強く誘導し、終盤は弱める」のが有効と報告されています。
- Guidance distillation: CFG の2倍コストを蒸留で1回推論に畳み込み、リアルタイム生成を可能にする方向。少ステップ生成（consistency models, rectified flow など）とも組み合わされます。
- 自動運転・ロボット制御での選択的適用: 条件全体ではなく「ルートレベルの意図」だけを CFG で強調する、あるいは安全制約を別の仕組み（最適化・フィルタ）で担保しつつ生成器には CFG を弱くかける、といった応用設計が議論されています。行動生成（Diffusion Policy 系）でも、目標や言語指示への追従を CFG で強める試みがあります。
