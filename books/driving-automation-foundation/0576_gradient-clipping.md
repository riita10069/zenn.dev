# 勾配クリッピング

## ひとことで言うと

学習中、重みを更新する手がかりである「勾配(gradient)」が、ときどき異常に大きな値になることがあります。その大きすぎる値で重みを更新すると、学習が一瞬で壊れて発散します。勾配クリッピング(gradient clipping)は、勾配の大きさに上限を設けて「大きすぎる勾配を切り詰める」ことで学習を安定させる手法です。安価で副作用が小さいため、深いネットワークや系列モデル、大規模 Transformer の学習でほぼ標準の安全装置になっています。

## 直感的な理解

崖を下りるとき、足元の傾斜が急であればあるほど大きく踏み出したくなりますが、傾斜が極端に急な一点で大股を踏み出すと、谷を飛び越えて反対側の壁まで吹き飛んでしまいます。傾斜(勾配)が異常に大きいときだけ、一歩の大きさに上限を設けて飛び過ぎを防ぐ。これが勾配クリッピングの直感です。

数値で見ます。学習率を `1e-3`、ある重みが今 `0.5` だとします。普段の勾配が `2.0` 程度なら更新量は `1e-3 × 2.0 = 0.002` で、重みは `0.498` というごく小さな変化です。ところが、あるステップで勾配が突然 `50000` になったとします。更新量は `1e-3 × 50000 = 50` になり、重みは `0.5 − 50 = −49.5` というかけ離れた値に吹き飛びます。1ステップでネットワークが壊れ、その後の loss が `nan`(数値計算が破綻した状態)になることもあります。何時間もかけた学習が、たった1回の勾配爆発で台無しになる。これを防ぐのがクリッピングです。

## 基礎: 前提となる概念

ニューラルネットの学習は「重み = 重み − 学習率 × 勾配」という更新を繰り返す作業です(歩幅にあたる学習率については [学習率スケジューラ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0578_lr-scheduler) を参照)。勾配は「loss を減らすために重みをどちらにどれだけ動かせばよいか」を表す数値です。

通常、勾配はそれほど大きくなりませんが、ときどき桁外れの値が出ます。これを「勾配爆発(gradient explosion)」と呼びます。なぜ起きるかを押さえておくと、クリッピングがなぜ効くか分かります。

- 深いネットワークや再帰型(時系列を何度も同じネットに通す)構造では、誤差逆伝播の際に各層のヤコビアン(微分の行列)が次々と掛け算されます。その掛け算で生じる係数が 1 より大きいと、層を遡るたびに勾配が指数的に大きくなります。これが爆発です。逆に 1 より小さいと指数的に小さくなり、これは「勾配消失」と呼ばれる別の問題になります。掛け算の係数を支配するのは、おおまかには各重み行列の最大特異値(スペクトルノルム)です。これが 1 を大きく超えると爆発しやすい、という関係が背景にあります。
- 学習率が高すぎる、入力データの前処理が不適切(正規化されていない、外れ値がある)、損失関数が特定の入力で急峻になる、といった条件でも爆発が起きやすくなります。

つまり勾配爆発はモデル構造と学習設定の両方から生じる現象で、クリッピングはその「最後の安全網」です。

もう一つの前提が「ノルム(norm)」です。ノルムとはベクトルの長さのことです。モデルの全パラメータの勾配を1本の長いベクトルとみなし、その長さ(L2 ノルム、各要素を二乗して足してルートを取った値)を計算したものを、ここでは全体ノルム(global norm)と呼びます。式で書くと、各パラメータの勾配を `g_i` として全体ノルムは `sqrt(Σ_i g_i²)` です。これがクリッピングの判定に使う1つの数値になります。

## 仕組みを詳しく

最も広く使われるのが「勾配ノルムによるクリッピング(clip by global norm)」です。上限を `max_norm = 1.0` とした手順は次の通りです。

1. 全勾配を1本のベクトルとみなし、その長さ `g_norm` を計算する。
2. `g_norm <= 1.0` なら何もしない(普通の勾配なのでそのまま使う)。
3. `g_norm > 1.0` なら、全勾配に `1.0 / g_norm` を掛けて、長さがちょうど `1.0` になるよう縮める。

数値例:

```
ケースA: g_norm = 0.7  → 上限1.0以下なので そのまま。勾配は変えない。
ケースB: g_norm = 50.0 → 50倍も長いので 1/50 倍に縮める。
         縮めた後の長さ = 50.0 × (1.0/50.0) = 1.0
```

重要なのは「方向は変えず、長さだけ縮める」点です。勾配が示す「どちらへ動かすか」の方向情報は保ったまま、「一歩の大きさ」だけを安全な範囲に抑えます。これにより、爆発したステップでも重みが吹き飛ばず学習が続けられます。全パラメータをまとめて1本のベクトルとして扱う(global norm)ことがポイントで、層ごとに別々に切ると層間の相対的な勾配バランスが崩れます。

```python
loss.backward()                              # 勾配を計算
clip_grad_norm_(model.parameters(), 1.0)     # 全体ノルムを1.0で上限クリップ
optimizer.step()                             # 縮めた勾配で更新
```

呼ぶ順番が大事です。必ず `backward()`(勾配計算)の後、`optimizer.step()`(更新)の前に挟みます。

クリッピングには別の流儀として「value clipping(各要素ごとに値を [−c, c] に丸める)」もありますが、これは要素ごとに切るため勾配ベクトルの方向が変わってしまう欠点があり、現在は norm clipping が一般的です。混合精度学習(bf16/fp16)と併用する場合、loss スケーリングを使う fp16 では「スケールを戻した本物のノルム」でクリップする必要があるなど、順序に注意が要ります([AMP / bf16混合精度](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0550_amp-bf16-mixed-precision))。具体的には、fp16 では loss を拡大したまま backward した勾配も拡大されているので、まず unscale(スケールを戻す)してから clip しないと、しきい値が拡大係数の分だけずれてクリップが効かなくなります。bf16 は loss scaling を使わないため、この順序問題が起きないのも利点です。

## 手法の系譜と主要論文

- Pascanu, Mikolov, Bengio, "On the difficulty of training Recurrent Neural Networks", ICML 2013(arXiv:1211.5063)。勾配クリッピングを広く知らしめた代表論文です。RNN では時間方向に勾配が何度も掛け算されるため指数的に爆発しやすいことを理論的に示し、「勾配ノルムがしきい値を超えたらしきい値まで縮める」norm clipping を提案しました。狙いは、爆発した1ステップが学習全体を壊すのを防ぐこと。同論文は勾配消失も同時に議論し、爆発と消失が表裏一体であると整理しました。爆発側はクリッピングで対処し、消失側はゲート機構(LSTM/GRU)や初期化・正規化で対処する、という後の役割分担の出発点になっています。

- Vaswani et al., "Attention is All You Need", NeurIPS 2017(arXiv:1706.03762)。直接の主題ではありませんが、Transformer を安定学習させる実務的な工夫として勾配クリッピングが広く併用されるようになりました。warmup(学習率を最初だけ小さくする工夫、[学習率スケジューラ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0578_lr-scheduler))とクリッピングは、深い系列モデルを安定学習させる定番のペアです。

- Zhang et al., "Why Gradient Clipping Accelerates Training: A Theoretical Justification for Adaptivity", ICLR 2020(arXiv:1905.11881)。クリッピングが単に発散を防ぐだけでなく、損失関数のなめらかさが場所によって大きく変わる関数(リプシッツ滑らかさが一定でない、局所的にしか滑らかでない関数)に対して、固定学習率の素朴な勾配降下より速く収束できることを理論的に説明しました。クリッピングは安全装置であると同時に学習を速める効果も持ちうる、という見方を与えた点で重要です。

- Brock et al., "High-Performance Large-Scale Image Recognition Without Normalization"(NFNet), ICML 2021(arXiv:2102.06171)。Adaptive Gradient Clipping(AGC)を提案。各層について「重みのノルムに対する勾配ノルムの比」を見てクリップすることで、Batch Normalization を取り除いても安定学習できる normalizer-free network を実現しました。固定しきい値クリッピングの発展形です。

- Abadi et al., "Deep Learning with Differential Privacy"(DP-SGD), CCS 2016(arXiv:1607.00133)。文脈はプライバシー保護ですが、各サンプルの勾配を per-example でクリップしてからノイズを加えるという形で、クリッピングを「個々のデータの影響を上限で抑える」道具として使いました。安定化目的とは別の用途で勾配の大きさ制御が中心的役割を果たす例です。

## 論文の実験結果(定量データ)

- Pascanu et al.(2013)は、合成タスク(長期依存を要する系列課題)と自然言語のモデリングで実験しました。クリッピングなしでは学習が頻繁に発散して進まないのに対し、クリッピングを入れると安定して学習が完走することを示しました。ここでの主要な「指標」は最終精度より「そもそも学習が成立するか」で、長い系列を扱う RNN/LSTM の学習を現実的に可能にした点が貢献です。

- Zhang et al.(2020)は言語モデリングと画像分類で、クリッピングありの方が同じ反復回数でより低い学習 loss と検証 loss に到達することを実験的に示しました。論文の理論は「勾配が大きい領域では小さい有効学習率、小さい領域では大きい有効学習率」という適応的な振る舞いをクリッピングが生むことを主張し、実験がそれを裏づけています。アブレーションとして、クリッピングのしきい値を変えると収束速度が変わり、しきい値が適切な範囲にあるときに最も速いことが示されています。

- NFNet(Brock et al., 2021)は ImageNet 分類で、Batch Normalization を一切使わずに当時の最先端の精度(NFNet-F1 などで top-1 おおむね 84 パーセント超)を達成し、かつ EfficientNet 系より高速に学習できることを示しました。アブレーションでは、AGC のクリッピング係数 λ を変えると安定性と精度が変わり、特に大きなバッチサイズで AGC を外すと学習が崩れることが報告されています。これは「正規化層を外した代わりに勾配の安定化をクリッピングが担う」ことの定量的な裏づけです。

- 大規模 Transformer の学習報告では、global norm のしきい値を 1.0 に設定するのが定番として共有されています。実務上のアブレーション的知見として、しきい値を外す(クリッピングしない)と学習中にときおり loss が急上昇する「スパイク(loss spike)」が観測され、しきい値を入れるとスパイクが抑えられる、と複数の技術報告で記録されています。

数値の意味を補足します。クリッピングは top-1 精度や BLEU のような最終指標を直接何ポイントも押し上げる手法ではなく、「学習を発散させない」「loss スパイクを抑える」という形で効きます。発散すれば最終指標は無意味になるので、土台として極めて重要です。なお、勾配ノルムそのものを学習中にログとして記録しておくと、いつスパイクが起きたか、どのデータバッチが原因かを後から追跡でき、デバッグの有力な手がかりになります([実験ログ(TensorBoard/WandB/MLflow)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1194_experiment-logging-tensorboard-wandb))。

## メリット・トレードオフ・限界

メリット
- 勾配爆発による発散(loss が nan になる等)を防げる。長時間学習が1回の事故で台無しになるのを避けられる。
- 方向は保ったまま大きさだけ抑えるので、学習の進む向きは壊さない。
- 実装は1行で済み、計算コストもほぼ無視できる。

トレードオフ・限界
- しきい値(max_norm)という調整項目が増える。小さすぎると本来必要な更新まで抑えて学習が遅くなり、大きすぎると効果がない。適切な値はモデル規模・損失スケール・バッチサイズに依存する。
- 勾配が頻繁にクリップされる場合、それは学習率が高すぎる/データ前処理が悪い等の別問題の兆候であり、クリッピングは対症療法にすぎない。根本原因の確認が必要。
- 毎ステップ全パラメータのノルムを計算するわずかなコストがある(通常は無視できるが、分散学習では全 GPU のノルムを集約する通信が必要になり、設計によってはオーバーヘッドになる)。
- 未解決の課題として、最適なしきい値はモデルやデータに依存し、理論から一意に決まらない。固定しきい値は学習が進んで勾配ノルムの自然なスケールが変わると不適切になりうるため、過去の勾配ノルムの統計から動的に決める適応的クリッピングの研究が続いています。

## 発展トピック・研究の最前線

- AGC(Adaptive Gradient Clipping, Brock et al., 2021)。各層の重みのノルムに対する勾配ノルムの比を見てクリップする手法で、固定しきい値より柔軟。正規化層を外した設計を可能にした。
- 大規模言語モデルでは、クリッピングに加えて「loss スパイクが起きたら直前のチェックポイントに戻して問題のバッチをスキップする」といった運用上の対処も併用されます([チェックポイント(保存・再開)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1161_checkpointing-save-resume))。スパイク対策はクリッピング単独では完結しない、という実務的認識が広がっています。
- 勾配クリッピングはプライバシー保護学習(DP-SGD)でも中心的役割を果たし、各サンプルの勾配をクリップしてからノイズを加えることで、個々のデータの影響を上限で抑えます。文脈は異なりますが「勾配の大きさを制御する」という発想は共通です。
- スペクトルノルム正則化や重みの直交初期化など、そもそも勾配が爆発しにくいモデルを作る方向の研究もあり、クリッピング(事後の対処)と相補的に使われます。
