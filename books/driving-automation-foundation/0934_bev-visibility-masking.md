# BEVの可視性マスキング

## ひとことで言うと

多視点カメラから鳥瞰図（BEV, Bird's-Eye-View、車を真上から見下ろした地図のような表現）を作るとき、各カメラには「見える範囲」と「見えない範囲」があります。たとえば前向きカメラに自車の真後ろは写りません。可視性マスキング（visibility masking）とは、「物理的に見えていない 3D 位置」の特徴を誤って拾わないように、無効な点・カメラを除外（マスク）し、有効なものだけで BEV を合成する仕組みです。具体的には「カメラの前にあるか（深度が正か）」「画像の枠内に投影されるか」「サンプル位置をずらした後も枠内か」を判定し、当たったカメラだけで平均します。これを怠ると、見えていない景色が地図に混入して認識・計画を狂わせます。

## 直感的な理解

BEV を作る基本動作は「BEV 地図上の各マス（3D 位置）を、各カメラの画像に投影して、その投影位置の特徴を集めてくる」ことです。ここで素朴に投影計算だけを信じると、重大な間違いが起きます。

例を考えます。「自車の真後ろ 10m」というマスを、前向きカメラに投影してみます。投影は単なる行列計算なので、何かしらの画素座標が出てきます。しかし前向きカメラに後ろの景色は写っていません。出てきた座標が指すのは前方の無関係な画素であり、その特徴は完全なゴミです。これを拾うと、地図の後方マスに前方の景色が貼り付いてしまいます。

もう1つの落とし穴が「深度の符号」です。3D 点を画像に投影するとき、点の深度（カメラからの前後距離）で座標を割り算します（透視除算、[ピンホール投影による3D→2D変換](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0998_pinhole-projection-3d-to-2d) 参照）。カメラの後ろにある点は深度が負ですが、負で割っても数値としては「画像内っぽい座標」が出ることがあります。物理的にはカメラの後ろは写らないので、深度が負の点は必ず捨てる必要があります。これは透視投影の数式 u = f·X/Z が Z の符号を保存しないことに起因する、投影を使うすべての手法に共通の罠です。

さらに、変形アテンション相当の処理では、見る位置を学習オフセットで少しずらします（[F.grid_sample による可搬な変形アテンション](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0968_grid-sample-deformable-attention) 参照）。ずらした結果、もとは枠内だった点が枠外に出てしまうことがあります。だからオフセット後にもう一度チェックが要ります。

可視性マスキングは、こうした「見えていない点を見えているふりして使う」事故を、複数の関門で防ぐフィルタです。マスク（mask）とは「使う／使わないを 0/1 で示す覆い」のことです。

## 基礎: 前提となる概念

**投影（projection）**: 3D 空間の点を、カメラの内部パラメータ（焦点距離など、行列 K）と外部パラメータ（カメラの位置・向き、行列 [R|t]）を使って 2D 画像座標に写す計算です。最後に深度で割る「透視除算」を伴います。

**視錐台（view frustum）**: カメラが見える範囲を表す角錐（ピラミッドを横に倒したような立体）です。カメラの前方の、画角に収まる範囲だけが見えます。視錐台の外（後ろ、画角外）は写りません。可視性マスキングは、本質的には「点が視錐台の中にあるか」を判定する処理です。深度判定が「前方か後方か」、境界判定が「画角内か外か」に対応します。

**正規化座標**: 画像座標を画像サイズで割って 0〜1 に直したものです。投影座標が [0, 1] × [0, 1] の中にあれば「画像の枠内」、外なら「枠外」です。解像度に依らず判定できる利点があります。

**LayerNorm と FFN**: Transformer 系の後処理層です。LayerNorm（層正規化）は特徴ベクトルを平均0・分散1付近に整え、学習可能なスケール γ とシフト β を掛け足します。FFN（Feed-Forward Network、前向きネットワーク）は2層程度の全結合で特徴を変換します。いずれも「バイアス（足し込まれる定数項、β や全結合のバイアス）」を持つため、入力が 0 でも出力は 0 にならない、という性質があとで効いてきます。

## 仕組みを詳しく

典型的な実装は、可視性を多段階の関門で確認します。BEV マス数を N、カメラ台数を V、1マスあたりの柱サンプル点数を num_z（柱に沿った高さ違いの数点）とします。

**関門1: 投影時のマスク（深度＋画像境界）**

3D 参照点をカメラに投影する段階で、2つを確認します。

```
valid_depth = depth > eps          # カメラの前にあるか（深度が正）
in_bounds   = (ref_2d[...,0] >= 0) & (ref_2d[...,0] <= 1) &
              (ref_2d[...,1] >= 0) & (ref_2d[...,1] <= 1)   # 画像枠内か
mask = valid_depth & in_bounds     # 両方満たせば可視  形状 [B, V, N, num_z]
```

`valid_depth` で「カメラの後ろ」を弾き、`in_bounds` で「画像の外」を弾きます。`eps`（例 1e-5）は深度がほぼ0（カメラ面ぎりぎり）の数値不安定を避けるための小さな閾値です。深度が0に近いと透視除算 X/Z が発散し、座標が巨大値になって学習を壊すため、わずかでも前方でない点は早めに切るのが安全です。

**関門2: オフセット後の再マスク**

学習オフセットでサンプル位置をずらした後、もう一度枠内チェックをして関門1のマスクと結合します。

```
sample_in_bounds = (sample_locs[...,0] >= 0) & (sample_locs[...,0] <= 1) &
                   (sample_locs[...,1] >= 0) & (sample_locs[...,1] <= 1)
combined_mask = ref_mask & sample_in_bounds
```

「オフセットを足した後に再計算する」のが要点です。これを忘れると、枠外へずれた点の補間結果（`padding_mode='zeros'` なら 0、それ以外なら端の値）を有効として混ぜてしまいます。

**関門3: アテンション重みの再正規化**

無効な点の重みを 0 にし、残った有効点だけで重みの合計が 1 になるよう割り直します。

```
point_mask = combined_mask.float()
w = attn_weights * point_mask
w = w / w.sum(dim=-1, keepdim=True).clamp(min=eps)   # 有効点で再正規化
```

`clamp(min=eps)` は「全点が無効で合計が 0」のときのゼロ割りを防ぐガードです。これが無いと NaN（非数）が発生し、一度 NaN が出ると逆伝播で全パラメータへ伝染して学習全体が崩壊します。

**関門4: カメラ単位の可視判定と V_hit 平均**

「柱の点が1つでも有効なら、そのカメラは見えている」とみなし、見えたカメラだけを足して、その台数で割って平均します。

```
cam_visible   = combined_mask.any(dim=-1).float()        # どれか1点でも有効
output        = output + weighted * cam_visible
visible_count = visible_count + cam_visible
...
output = output / visible_count.clamp(min=1.0)           # 見えたカメラ数で平均
```

これは BEV 空間クロスアテンションの集約式に対応します。当たったカメラ集合を V_hit とすると、出力は「V_hit に属するカメラの寄与を、その台数 |V_hit| で割った平均」になります。重なり領域（複数カメラに写る点）でも二重カウントせず公平に平均する点が重要です。単純な総和だと、たまたま2台に写る領域だけ値が2倍になり、BEV 全体で明るさムラのような偏りが生じます。

**関門5: 未観測マスのゼロ埋め（順序が肝心）**

どのカメラからも観測されなかったマスを明示的に 0 にします。

```
has_observation = (visible_count > 0).float()
output = output * has_observation                        # 観測ゼロのマスは0埋め
```

ここで決定的に重要なのは、このゼロ埋めを LayerNorm と FFN の「後」に適用することです。理由は前述のとおり、LayerNorm・FFN にはバイアスがあるため、その前にゼロにしても、後段のバイアスで非ゼロに戻ってしまうからです。順序を間違えると「観測ゼロなのに何か写っている」マスが復活します。「無観測=ゼロ」という不変条件を保ちたいなら、不変条件を壊しうる全演算の後ろにマスクを置くのが鉄則です。

**数値例で全体像**: あるマスを 8 台のカメラに投影したところ、前向きと右前の 2 台だけで柱の一部点が有効、残り 6 台は深度負か枠外だったとします。出力はこの 2 台ぶんの重み付き特徴を足し、`visible_count = 2` で割って平均します。0 台しか当たらなければ、関門5で 0 になり「観測なし」を意味します。

```
カメラ:  前  右前  右  右後  後  左後  左  左前
有効?:   ◯   ◯    ✕   ✕   ✕    ✕   ✕    ✕     → visible_count = 2
出力 = (前の寄与 + 右前の寄与) / 2
```

## 手法の系譜と主要論文

**LSS — Lift, Splat, Shoot（Philion & Fidler, ECCV 2020, arXiv:2008.05711）**。各画素に深度分布（その画素が各距離にある確率）を予測し、特徴を 3D に「持ち上げ（lift）」、BEV 平面へ「ばらまく（splat）」手法です。方向は 2D→3D と逆ですが、視錐台の外を扱わない・深度や画角でカバー範囲を決める、という点で可視性の概念を内包します。動機は明示的な幾何投影と微分可能な集約を両立させることで、新規性は深度分布を用いた微分可能な lift にあります。トレードオフは深度を密に予測するためメモリが重いことです。

**DETR3D（Wang et al., CoRL 2021, arXiv:2110.06922）**。3D 物体クエリを画像に投影し、投影位置の特徴をサンプルして集約します。ここでも「投影が画像枠外・カメラ背後に落ちた場合は無効」という扱いが必要で、可視性に基づくマスクの先駆けと言えます。複数カメラに写る点では、有効なカメラの特徴を平均する設計を採っており、本稿の V_hit 平均の原型にあたります。動機は明示的な BEV 中間表現を作らずに 3D 検出を行うことでした。

**BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）**。可視性に基づく集約を空間クロスアテンションの中核に据えました。各 BEV クエリの 3D 参照点（柱に沿った複数高さの点）を全カメラに投影し、hit したカメラ（V_hit）だけで特徴を変形アテンションで集め、その台数で平均します。動機は「各 3D 位置は一部のカメラにしか写らず、カメラ間の重なりも少ない」ことで、写っていないカメラの寄与を混ぜると地図が汚れるからです。新規性は、多視点の境界をまたいでも一貫した BEV を作る集約スキームを定式化した点です。本稿の段階的マスク＋V_hit 平均は、このスキームを忠実に再現したものです。トレードオフは可視判定の計算が増えること、重なり領域での二重カウント回避に配慮が要ることです。

これらに共通するのは「投影は幾何的に無効な領域でも数値を吐く」という事実への対処です。深度符号・画像境界・視錐台のチェックは、投影を伴う BEV/3D 手法すべてに不可欠な実装上の必須処理として確立しています。

## 論文の実験結果（定量データ）

可視性マスキングそのものを単独で切り出したアブレーションは論文では稀ですが、関連する効果は次のように現れます。

**BEVFormer（nuScenes 3D 検出）**: カメラのみで NDS（nuScenes Detection Score、検出精度と位置・速度・属性誤差を総合した 0〜1 のスコア、高いほど良い）約 0.517、mAP（mean Average Precision、検出の平均適合率、高いほど良い）約 0.416 を報告。当時のカメラベース DETR3D（NDS 約 0.425、mAP 約 0.346）を大きく上回りました。各 3D 位置を「写っているカメラだけ」から集約する設計が、多視点融合の精度に寄与しています。もし可視性を無視して全カメラの投影特徴を平均すれば、見えないカメラのゴミ特徴がノイズとして混入し、精度は明確に低下します。直感的には、8台中2台しか写っていない領域で全8台を平均すると、信号対雑音比が 1:3 に薄まる計算になります。

**LSS（nuScenes セグメンテーション）**: 車両 BEV セグメンテーションの IoU（Intersection over Union、予測領域と正解領域の重なり率、0〜1 で高いほど良い、車両クラスでおよそ 0.32 程度を報告）で、当時の投影ベースライン（Cam2BEV 系など）を上回る結果を報告しました。視錐台に基づき有効領域のみを扱うことが、地図のクリーンさと精度に効きます。

**示唆されるトレードオフ**: 可視性判定は境界付近で「有効／無効」が離散的に切り替わるため、勾配が不連続になりやすく、学習の安定性に影響します。研究実装では、ハードな 0/1 マスクの代わりにソフトな重み（連続値の可視度、たとえば境界からの距離をシグモイドに通したもの）を用いて勾配を滑らかにする工夫が検討されることがあります。これは精度と安定性のトレードオフを定量的に扱う研究テーマです。

## メリット・トレードオフ・限界

**メリット**

- 物理的に見えない位置の特徴混入を防ぎ、BEV のクリーンさ（地図としての信頼性）を保つ。
- 深度負・画像外・オフセット後枠外という複数の無効パターンを段階的に確実に弾ける。
- V_hit 平均により、カメラ重なり領域でも二重カウントせず公平に集約できる。
- 観測ゼロのマスを明示的に 0 にするので、後段（検出・計画モジュール）が「未観測」を区別できる。これは「見えていないから空いている」と「見えていて空いている」を取り違えないために重要。

**トレードオフ・限界**

- マスク生成・再正規化・カメラ平均と処理が多く、コードと計算がやや複雑になる。
- 境界付近で有効/無効が急に切り替わり、勾配が不連続になりやすい（学習が不安定になりうる）。
- ゼロ割り防止の `clamp` を要所に入れる必要があり、抜けると NaN で学習全体が崩壊する。
- 多くのマスが未観測（疎な観測）だと、その分の計算が無駄になり効率が落ちる。
- ハードマスクは「見えるか見えないか」を二値で決め打ちするため、半分隠れた領域（部分遮蔽）のような中間状態を表現しにくい。

**未解決の課題（研究視点）**: 自己遮蔽（自車や前方車両に隠れて一部だけ見える領域）の扱い、ソフト可視度による勾配の安定化、複数カメラ間で測光的に矛盾する観測の統合（同じ点が違う色・特徴に見える場合の重み付け）などは、いまも改良の余地があります。とくに「未観測」と「観測したが空」を学習で区別させること自体が難しく、ここを誤ると計画モジュールが見えていない領域を安全だと誤認する危険があります。

## 発展トピック・研究の最前線

近年は、可視性を陽に扱う代わりに、3D 位置エンコーディングをキーに埋め込んで投影マスクを暗黙化する流れ（PETR/PETRv2 系）や、占有グリッド（occupancy）で「観測あり／なし／遮蔽」を3状態以上で表す手法が広がっています。占有予測（occupancy prediction、SurroundOcc・TPVFormer・OccNet など）は、まさに「どこが見えていてどこが未観測か」を学習対象として明示的にモデル化する方向で、可視性の概念を中核データ表現へ昇格させたものと見なせます。また時系列で複数フレームを統合する手法では、過去に見えていた領域の記憶をどう保持・減衰させるか（観測の鮮度）という、時間方向の可視性管理が新しい課題として現れています。占有予測の評価では、可視マスクを使って「見えたボクセルだけ」で IoU を測るか、全ボクセルで測るかで数値が大きく変わるため、可視性は評価指標の定義そのものにも影響を及ぼしています。
