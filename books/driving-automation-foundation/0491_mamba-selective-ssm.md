# Mamba (Selective SSM)

## ひとことで言うと
Mamba は S4 という状態空間モデルを進化させ、「入力の中身を見て、覚えるべき情報と捨てる情報を選べる」ようにしたモデルです。これにより、計算量を系列長に対して線形(系列が 10 倍長くなっても計算も約 10 倍で済む、Transformer の 2 乗とは大違い)に保ったまま、Transformer に匹敵する言語・音声・時系列の性能を出せるようになりました。推論も 1 ステップずつ・一定メモリで進められるため、長系列やストリーミングに向きます。「Transformer の代わりになりうる長系列モデル」として大きな注目を集めています。

## 直感的な理解
前提として [S4 (Structured State Space)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0501_state-space-model-s4) を理解しておくと話が早いです。S4 は長い系列を効率よく扱える優れたモデルでしたが、ひとつ決定的な弱点がありました。線形・時不変(LTI, Linear Time-Invariant)だったことです。

線形・時不変とは、ざっくり言うと「どんな入力が来ても、状態の更新ルール(行列 A, B, C と刻み幅 Δ)が常に同じ」という意味です。これは制御の数式としてはきれいですが、現実のデータ処理では困ります。なぜなら、入力の中身によって「これは大事だから強く覚える」「これはノイズだから無視する」と切り替えたいことが多いからです。

具体例で考えます。文章「私は昨日とても重要な会議に出て…(中略、無関係な雑談が続く)…そこで決めた予算は 500 万円だった」を読むとき、人間は「500 万円」のような重要な数字は強く覚え、途中のどうでもいい接続詞は受け流します。S4 のような時不変モデルは、すべての単語を同じルールで等しく扱うので、こうした「選ぶ」ができません。論文ではこれを「内容に基づく推論(content-based reasoning)が苦手」と表現しています。

象徴的なおもちゃ課題が 2 つあります。Selective Copying(選択的コピー、入力に混ざったノイズトークンを無視して、意味のあるトークンだけを順序通り出力する課題)と Induction Heads(誘導ヘッド、「A の次は B だった」という直前のパターンを記憶し、再び A が来たら B を出す課題)です。どちらも「文脈に応じて記憶を選択・参照する」能力を要求します。時不変な S4 はこれらを苦手としますが、Transformer の自己注意(self-attention、各時点がどこに注目するかを入力から動的に決める仕組み)は得意です。ただし注意の代償は計算量 O(L^2) で、長系列ではメモリも時間も爆発します。Mamba は「S4 の線形計算量」と「注意の選択性」を両立することを狙って提案されました。

## 基礎: 前提となる概念
S4 の離散化後の更新式を再掲します。

```
x_k = Ā x_{k-1} + B̄ u_k
y_k = C x_k
```

ここで x_k は N 次元の隠れ状態(過去の要約)、u_k はその時刻の入力、Ā は状態の時間発展を決める行列、B̄ は入力を状態に取り込む行列、C は状態から出力を読み出す行列です。S4 では Ā, B̄, C はすべての時刻 k で同じ固定値でした(これが時不変ということ)。Δ は離散化の刻み幅で、連続時間の式を 1 ステップごとの差分に直すときの「時間の進み量」を表します。Δ が大きいほど「1 ステップで時間を大きく進める=新しい入力を強く取り込む」、小さいほど「時間をほとんど進めない=過去の状態をそのまま保つ」という効果を持ちます。この Δ の役割が、後で説明する選択性の鍵になります。

もう一つの前提が並列スキャン(parallel scan / prefix scan)です。漸化式は一見「前の結果が出ないと次が計算できない」逐次処理に見えますが、累積和(prefix sum、a1, a1+a2, a1+a2+a3, … を求める計算)と同じ構造を持ち、結合則が成り立つ演算なら木構造で並列化できることが古くから知られています。Mamba の高速化はこの数学的事実を利用します。

## 仕組みを詳しく
Mamba の中心アイデアは選択的状態空間(Selective SSM, S6)です。S4 では固定だったパラメータ(B, C と刻み幅 Δ)を、各時刻の入力から計算して毎ステップ変えます。

```
B_k = Linear_B(u_k)
C_k = Linear_C(u_k)
Δ_k = softplus(Linear_Δ(u_k))
```

つまり入力ごとに「どれだけ取り込むか(B)」「どれだけ出力に反映するか(C)」「時間をどれだけ進めるか(Δ)」が変わります。なお A 自体は構造化したまま固定で、Δ を通じて Ā = exp(Δ A) として入力依存に効きます。

ここで Δ が選択ゲートとして決定的に効きます。Δ_k が大きいと「今の入力を強く取り込み、過去をリセットぎみにする(=今に注目)」、Δ_k が小さいと「今の入力をほぼ無視し、過去の状態を保つ(=この入力をスキップ)」という挙動になります。これがゲート(gate、情報を通すか止めるかの弁)として働き、「重要な入力ではしっかり状態を更新し、どうでもいい入力ではほぼ素通り」を実現します。これが「選択(selective)」の中身です。先ほどの Selective Copying なら、ノイズトークンでは Δ を小さくして素通りさせ、本物のトークンでは Δ を大きくして記憶する、という挙動を学習で獲得できます。

ここで深刻な問題が生じます。S4 が速かったのは、パラメータが固定だったからこそ畳み込み(FFT)で一気に並列計算できたからです。Mamba ではパラメータが時刻ごとに変わる(時変)ので、もはや単一の畳み込みカーネルには書けません。漸化式に戻すしかなく、素朴には逐次計算で遅くなります。

これを救うのがハードウェアに優しい並列スキャン(hardware-aware selective scan)です。前述の通り漸化式は結合則を満たすため並列スキャンで計算でき(L ステップを愚直に逐次処理すると O(L) の段数だが、二分木状にまとめると O(log L) の段数で済む)、Mamba はこれを GPU 向けにカスタム CUDA カーネルとして実装しました。さらに重要なのがメモリ階層の活用で、これは FlashAttention と同じ思想です。GPU には大容量だが遅い HBM(High Bandwidth Memory、数十 GB だが帯域がボトルネック)と、小容量だが極めて速い SRAM(オンチップの数十 MB)があります。SSM の計算は演算量より「大きな中間テンソルを HBM に読み書きする転送量」が律速になりがちです。Mamba は、時刻ごとに変わる大きな中間テンソル(各時刻で離散化したパラメータ Ā・B̄ や、状態を N 次元 × 拡張係数ぶん持つテンソル)を遅い HBM に書き戻さず、SRAM 上で生成・消費して計算を融合(kernel fusion、別々の演算を 1 つのカーネルにまとめて中間結果の往復を省く)し、逆伝播では中間状態を保存せず順方向を再計算(recomputation、メモリと計算量を引き換えにする勾配チェックポイント)することでメモリ使用量を抑えます。この設計のおかげで、時変であっても実用的な速度と省メモリを実現します。

アーキテクチャ全体は、この S6 ブロックを Gated MLP 風のブロックに組み込み、それを積み重ねるだけのシンプルな構成です。Transformer のような自己注意層や別個の MLP 層を持たず、均質な Mamba ブロックの積層で言語モデルなどを構成できます。

```
S4:     パラメータ固定(時不変) → 畳み込みで速い・でも選べない
Mamba:  パラメータが入力依存(時変) → 並列スキャンで速い・選べる
Attn:   全点比較で選べる・でも O(L^2) で長系列に弱い・推論で KV キャッシュが膨らむ
```

## 手法の系譜と主要論文
- Gu & Dao, "Mamba: Linear-Time Sequence Modeling with Selective State Spaces"(2023, arXiv:2312.00752)。提案: SSM のパラメータ(B, C, Δ)を入力依存にする選択機構 S6 と、それを高速計算するハードウェア対応並列スキャン。動機: S4 の弱点だった「内容に基づく取捨選択ができない」を解消しつつ Transformer の O(L^2) を避けるため。新規性: 時変 SSM を実用速度に乗せた点と、注意を一切使わない均質ブロックで Transformer に匹敵させた点。
- Dao & Gu, "Transformers are SSMs: Generalized Models and Efficient Algorithms Through Structured State Space Duality (Mamba-2)"(ICML 2024, arXiv:2405.21060)。提案: 状態空間モデルと注意を統一的に説明する SSD(State Space Duality)と、より大きな状態次元・高速学習を実現する Mamba-2。動機: Mamba の理論的位置づけを明確化し、行列演算(行列積中心)として GPU 効率を上げるため。新規性: 注意と SSM が同じ枠組みの双対であることを示した点で、両者の架け橋になりました。
- Zhu et al., "Vision Mamba (Vim)"(ICML 2024)/ Liu et al., "VMamba"(2024)。提案: 画像をパッチ列とみなして双方向 Mamba を視覚に応用。動機: ViT(Vision Transformer)の O(L^2) を画像でも避けるため。新規性: 双方向スキャンや 2 次元スキャン順序の設計で、画像の順序の曖昧さに対処しました。
- Lieber et al., "Jamba"(2024)。提案: Mamba 層と Transformer 注意層を層ごとに交互配置するハイブリッド大規模言語モデル。動機: 純粋な SSM の検索の弱さと、純粋な注意の長系列コストを補い合うため。新規性: 数十万トークンの長文脈を実用メモリで扱えることを示しました。

## 論文の実験結果(定量データ)
言語モデリングでは、指標として perplexity(困惑度、次の単語を当てる確率の逆数の幾何平均に相当し、「モデルが次の単語を平均何択ぶんの不確かさで迷っているか」を表す。低いほど良い)と下流タスクのゼロショット精度(追加学習なしでそのまま解いたときの正解率)が使われます。報告によると、Mamba は Chinchilla の最適計算量則(同じ計算予算ならパラメータとデータをどう配分すべきかの経験則)に沿った設定で、同じパラメータ数の Transformer(強力な Transformer++ レシピ、回転位置埋め込み RoPE や SwiGLU 等を盛り込んだ近年の最良構成)と同等以上の perplexity を達成しました。特に Mamba-1.4B/2.8B クラスは同サイズの Transformer を上回り、Pile などで学習した Mamba-3B 相当は約 2 倍サイズの Transformer-6B 級に匹敵するゼロショット性能(LAMBADA、HellaSwag、PIQA、ARC、WinoGrande などの常識推論ベンチ平均)を示したとされます。これは「半分のパラメータ数で同等の性能」を意味し、長系列効率と合わせて実用上の魅力が大きいことを示します。

効率面の数値が印象的です。Mamba は系列長に対して線形時間でスケールし、推論スループット(1 秒あたりに生成できるトークン数)は同規模 Transformer の約 5 倍と報告されています。理由は明快で、Transformer は推論時に過去全トークンの Key/Value をメモリに保持(KV キャッシュ、生成済みトークンぶんだけ線形に膨らむ)し続けるため系列が伸びるほど重く・遅くなるのに対し、Mamba は系列長によらず固定サイズの状態しか持たないからです。具体的には、生成 1 トークンあたりの計算もメモリアクセスも系列長に依存せず一定で、これがバッチを大きく取れる(=スループットが上がる)理由でもあります。学習でも、系列長を 1k から数百 k まで伸ばしても計算が線形に増えるだけで破綻せず、ハードウェア対応スキャンのおかげで FlashAttention を用いた Transformer より長系列で高速になる領域があると報告されています。

合成タスクのアブレーションが選択機構の効果を直接示します。Selective Copying では、時不変な S4 系が苦戦するのに対し、選択機構を入れた S6 はほぼ完璧に解けます。Induction Heads では、Mamba は学習時より遥かに長い系列(報告では学習長の数千倍規模の長さ)に外挿しても性能を保ち、長距離の文脈参照能力を示しました。逆に「選択をやめて時不変に戻す」アブレーションでは、これらのタスク性能が大きく落ちます。つまり「効いているのは Δ・B・C の入力依存化(選択性)である」ことが切り分けられています。

応用領域でも汎用性が確認されています。DNA モデリング(ゲノム配列、極端に長い系列)や音声生成では、長系列での効率と精度の両立が報告されました。Vision Mamba/VMamba は ImageNet 分類などで ViT 系に近い精度を、より良いスループット・メモリ効率で達成したと報告しています。

## メリット・トレードオフ・限界
メリット:
- 計算量が系列長に対して線形で、長系列でも破綻しません。推論は固定サイズ状態で一定メモリ・高スループットです。
- 入力依存の選択機構で「重要な情報を覚え、不要を捨てる」ができ、S4 の時不変性の弱点を克服しています。
- 注意を使わない均質な積層でシンプルに大規模化でき、ストリーミング・リアルタイム処理に向きます。

トレードオフ・限界:
- 全時点を直接アクセスする厳密な検索やコピー系の一部タスクでは、状態が固定サイズである以上、Transformer に劣る場合があります(状態の容量が情報量の上限になるため)。Jamba のようなハイブリッドはこの弱点を補う方向です。
- 高速化がカスタム CUDA カーネル(選択スキャンのハードウェア実装)に依存し、環境移植性や他フレームワークへの展開に手間がかかることがあります。
- 比較的新しく、安全クリティカル領域(自動運転・医療など)での長期的な実績はまだ蓄積途上です。
- 未解決の課題として、固定状態の容量限界とタスク要求のバランス、画像など非系列データへの最適なスキャン設計、解釈性(注意のように「どこを見たか」を取り出しにくい)などが挙げられます。

## 発展トピック・研究の最前線
Mamba 以降、状態空間と注意を統合する流れが加速しました。Mamba-2 の SSD は「注意と SSM は同じ構造化行列の双対表現」と示し、両者の良い性質を一つの実装で得る研究を後押ししています。ハイブリッド化(Jamba、Zamba など)は、層ごとに SSM と注意を混ぜることで、長文脈の効率と正確な検索能力を両立する実用解として広がっています。視覚分野では双方向・多方向スキャンの設計が活発で、検出・セグメンテーション・動画など 2 次元/3 次元データへの拡張が進んでいます。さらに、状態の容量を増やす方向(状態次元の拡大、複数状態の併用)や、選択機構の表現力を高める変種、線形注意・RWKV など他の線形時間モデルとの理論的な統一も研究の最前線です。
