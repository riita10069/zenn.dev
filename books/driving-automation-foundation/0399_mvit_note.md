# MViT (Multiscale Vision Transformer)

## ひとことで言うと
MViT(エムビット、Multiscale Vision Transformer)は、画像や動画を理解するモデルで「最初は細かい解像度・少ないチャネル(情報の種類数)で始め、層が深くなるほど解像度を下げてチャネルを増やす」という、CNN で長年使われてきた段階的な作りを Transformer に持ち込んだものです。これにより、ViT が苦手だった「高解像度・動画のような巨大なトークン数」を現実的な計算量で扱えるようにしました。とくに動画認識を主眼に設計された点が特徴で、改良版の MViTv2 は画像分類・物体検出・動画認識を一つのバックボーンで横断的に高性能化しました。

## 直感的な理解
元祖 ViT は、最初から最後まで同じ解像度・同じチャネル数を保ちます。たとえると、入口から出口まで会議室の広さも参加者数も一定のまま、ずっと全員会議を続けるようなものです。動画では「縦 x 横 x 時間」でトークンが膨れ上がり、全員会議の組み合わせが手に負えなくなります。たとえば 16 フレームの動画を 56x56 のトークンで表すと 16x56x56 ≒ 5 万トークンになり、全トークン注意は 5 万の二乗 = 25 億関係という非現実的な量になります。

CNN は昔から賢いやり方を知っていました。浅い層は「色やエッジ」程度しか表さないので、解像度は高くても少ないチャネルで十分。深い層は「これは車」「これは歩行者」といった複雑な意味を表すので、解像度を下げて全体を見渡しつつチャネルを増やす。つまり、進むほど空間を粗く、意味を濃くしていきます。MViT はこの段階的な作り(マルチスケール)を Transformer に持ち込み、注意を計算する直前に Key と Value を間引くことで、巨大なトークン数を現実的な計算量に抑えます。要するに「ViT に CNN のピラミッド構造を取り戻す」のが MViT の核心です。

## 基礎: 前提となる概念
用語を順に噛み砕きます。

チャネル(channel)とは、一つの位置が持つ情報の種類数です。RGB 画像なら 3 チャネル(赤・緑・青)。モデル内部では、浅い層は色やエッジ程度なので少ないチャネルで足り、深い層は複雑な意味を表すため多くのチャネルが要ります。

解像度(resolution)とは、特徴マップの空間的な細かさ(縦 x 横のトークン数、動画なら時間も含む)です。下げると計算が軽くなり、各トークンがより広い範囲を代表するようになります。

マルチスケール(multiscale)とは、複数の解像度を段階的に扱う設計です。CNN(ResNet など)は入力 `[B,3,224,224]` を段ごとに解像度を半分・チャネルを倍にして `[B,2048,7,7]` まで持っていきます。浅い高解像度特徴は細部、深い低解像度特徴は意味、と役割が分かれます。

自己注意(self-attention)は、各トークンが他トークンとの関係の強さを計算し関係の深い相手の情報を取り込む仕組みで、各トークンから Query, Key, Value を作り、Query と Key の内積で関係を測り softmax で正規化して Value を加重平均します。計算量はトークン数 N の二乗 O(N^2)。動画は時間方向にもトークンが広がるため N が特に大きく、ここが最大のボトルネックです。

プーリング(pooling)とは、近隣のトークンをまとめて一つに縮約する操作です。たとえば 2x2 の領域を平均する、あるいはストライド付きの畳み込みで 4 トークンを 1 トークンにまとめる、といった操作で、トークン数を減らします。最大値を取る max pooling、平均する average pooling、学習可能な畳み込みで行う conv pooling などがあります。

元祖 ViT の問題は二つでした。第一に、高解像度や動画ではトークン数が爆発し O(N^2) の注意が非現実的。第二に、浅い層から多チャネルを抱えるのは情報的にも計算的にも無駄が多い(浅い層に意味を表す広いチャネルは過剰)。MViT(Fan, Xiong, Mangalam, Li, Yan, Malik, Feichtenhofer; Facebook AI Research / Berkeley, ICCV 2021)はこれを「ViT にも CNN 的なマルチスケール構造を入れる」ことで解決しました。

## 仕組みを詳しく
MViT の核は Pooling Attention(プーリング注意)です。

### Pooling Attention(プーリング注意)
通常の自己注意は Query, Key, Value を全トークンから作ります。MViT は注意を計算する前に、Key と Value(場合により Query も)をプーリングで間引きます。プーリングは学習可能な深さ方向畳み込み(conv pooling)で実装され、単なる平均より柔軟に重要な情報を残せます。

具体例。あるトークン列が 56x56 = 3136 個あるとき、Key/Value をストライド 2 のプーリングで 28x28 = 784 個に減らします。すると注意の計算は Query 3136 と Key 784 の積で、3136 x 784 ≒ 約 246 万回。間引かない 3136 x 3136 ≒ 約 980 万回に比べて約 4 分の 1 です。

- before(通常注意): Q 3136 × K 3136 ≒ 980 万
- after(プーリング注意、K/V を 1/4 に): Q 3136 × K 784 ≒ 246 万

さらに Query 側もプーリングすれば、出力のトークン数自体が減り、次の段の解像度が下がります。これが「段を進むごとに解像度を下げる」仕組みの正体です。つまり Key/Value のプーリングは「その段の計算を軽くする」効果、Query のプーリングは「段の境界で解像度を落とす(ダウンサンプリング)」効果を担います。両者を分けて制御できるのが MViT の柔軟さです。

```
通常の注意                          Pooling Attention
Q (3136) ─┐                         Q (3136、または段境界で縮約)
K (3136) ─┼─ QK^T = 3136x3136       K' (784、conv pool s=2) ─┐
V (3136) ─┘   ≒ 980万               V' (784) ─────────────────┼ QK'^T = 3136x784
                                                              ≒ 246万(約1/4)
```

### Stage 構成(マルチスケール)
MViT は数段の Stage を持ち、各段で解像度を下げチャネルを上げます(動画では時間方向も扱う)。画像での形の変遷:

- Stage 1: トークン 56x56、チャネル 96(細かい模様・動き)
- Stage 2: トークン 28x28、チャネル 192
- Stage 3: トークン 14x14、チャネル 384
- Stage 4: トークン 7x7、チャネル 768(意味的なまとまり)

浅い段は高解像度・低チャネルで細かいエッジや動きを、深い段は低解像度・高チャネルで物体らしさを捉えます。まさに CNN の階層を Transformer で再現したものです。チャネルを増やすのは、解像度を下げたとき注意の出力次元(head 数)を増やすことで行います。動画の場合はトークンが時空間の立方体(時間 T x 縦 H x 横 W)になり、プーリングも時空間で行います。たとえば時間方向は粗く保ったまま空間だけ縮約する、といったタスクに応じた設計が可能です。

### MViTv2 の改良
MViTv2(Li, Wu, Fan, Mangalam, Xiong, Malik, Feichtenhofer; FAIR / UC Berkeley, CVPR 2022)は主に三点を改良し、初代の動画特化から汎用バックボーンへ脱皮させました。

1. 分解相対位置埋め込み(decomposed relative positional embeddings): トークンの位置関係を「縦の差」「横の差」(動画なら時間差も)に分解して扱い、平行移動に対する頑健性を高めました。位置を内容と切り離して扱うため、物体が画面内で移動しても認識が安定します。全位置ペアを直接持つと位置数の二乗のパラメータが要りますが、軸ごとに分解すると和で済むため、必要なパラメータ・計算が大きく減ります。
2. 残差プーリング接続(residual pooling connection): プーリングで情報を間引く際、プールした Query を残差(ショートカット)として注意出力に足し戻し、間引きで失う情報を補います。学習の安定と精度向上に寄与します。プーリングが情報を捨てる操作である以上、その損失を残差で埋め合わせる発想です。
3. ハイブリッドな段設計と検出向けの工夫(FPN との接続など)で、検出・分類でも使える汎用性を確保しました。

これらにより MViTv2 は動画認識だけでなく画像分類・物体検出でも当時の最先端に並びました。

## 手法の系譜と主要論文
- SlowFast Networks(Feichtenhofer, Fan, Malik, He; ICCV 2019, arXiv:1812.03982): 動画を「ゆっくり見る経路(空間重視、低フレームレート・多チャネル)」と「速く見る経路(動き重視、高フレームレート・少チャネル)」の二経路で処理する CNN。動画認識の CNN 系の到達点で、MViT は同じ著者陣がこの流れを Transformer で引き継ぎました。MViT の「浅く高解像度・深く低解像度」という発想は、SlowFast の二経路思想を一つの階層に畳み込んだものとも読めます。
- ViT(Dosovitskiy et al., ICLR 2021, arXiv:2010.11929): 単一スケール・全トークン注意の起点。高解像度・動画でトークン爆発に苦しむ。
- ViViT / TimeSformer(2021): ViT を動画に拡張した初期手法。時空間注意を分解するなどで対応したが、単一スケールゆえの計算負荷は残りました。MViT は階層化でこれに対抗します。
- Multiscale Vision Transformers(Fan, Xiong, Mangalam, Li, Yan, Malik, Feichtenhofer; FAIR / Berkeley, ICCV 2021, arXiv:2104.11227): Pooling Attention とマルチスケール段構造を提案。動画を主眼に、ImageNet 事前学習なしでも従来 ViT 系を上回りました。
- Swin Transformer(Liu et al., ICCV 2021, arXiv:2103.14030): 同じく階層構造を持つが、解像度削減を「パッチ統合(patch merging、近傍 2x2 を連結して次元削減)」で行う点が MViT と異なります。MViT は注意の内部でプーリングして削減し、Swin は注意の外でトークンを統合します。
- MViTv2(Li, Wu, Fan, Mangalam, Xiong, Malik, Feichtenhofer; CVPR 2022, arXiv:2112.01526): 分解相対位置埋め込みと残差プーリング接続で初代を改良し、画像・検出・動画を一つのバックボーンで横断高性能化しました。

## 論文の実験結果(定量データ)
MViT は動画認識を主舞台に評価されました。

動画認識(Kinetics-400、約 40 万本のクリップ・400 クラス。指標は top-1 accuracy = 最確信予測が正解動画クラスと一致した割合):
- 初代 MViT-B は、ImageNet 事前学習なしでも、同等以上の計算量の従来 ViT 系を上回り、FLOPs(浮動小数点演算回数、計算量の指標)を大幅に削減して同等以上の精度(約 80% 前後)を達成しました。当時の ViT ベース動画モデル(数倍の FLOPs を要する)に対し、数分の一の計算量で並ぶか上回ったのが要点で、「事前学習なし・低計算量で高精度」が初代の主張です。
- Kinetics-600、Something-Something v2(動きの理解がより重要なベンチ)などでも競争力を示しました。

MViTv2 の横断評価:
- 画像分類(ImageNet-1k): MViTv2-L で top-1 約 86% 台を報告。階層型の効率を保ちつつ静止画でも最前線に並びました。
- 物体検出・インスタンスセグメンテーション(COCO、指標は AP = Average Precision、高いほど良い): MViTv2 バックボーンを Mask R-CNN / Cascade に載せた構成が同規模 Swin を上回る AP を報告しました。
- 動画認識(Kinetics-400): MViTv2-L で top-1 約 86% 台に到達し、当時の最前線に並びました。事前学習(ImageNet-21k)を併用してさらに伸ばす設定も示されています。

アブレーション(どの要素を抜くと何が起きるか):
- Pooling Attention を通常の全トークン注意に戻すと、計算量が大きく増えるうえ高解像度・長時間動画では非現実的になります。プーリングが効率の要であることを示します。同じ精度なら FLOPs が数倍、同じ FLOPs なら精度が大きく落ちる、という形で効果が出ます。
- MViTv2 で分解相対位置埋め込みを外すと精度低下。とくに物体の位置がずれるケースや、絶対位置埋め込みでは扱いにくい解像度変更時に効きます。
- 残差プーリング接続を外すと精度低下と学習の不安定化。プーリングで失う情報の補填が重要だと分かります。

数値の意味を補足すると、動画認識の top-1 は静止画分類より一般に低め(動きの曖昧さや時間方向の難しさ、クラス間の見た目の近さのため)で、同じ FLOPs での 1〜2 ポイント差は大きな改善です。MViT の本質的な価値は「精度を保ちつつ FLOPs を削った」点、つまり精度対計算量(accuracy-compute trade-off)の効率にあります。動画は推論コストが特に重いため、この効率改善は実用上の意味が大きいのです。

## メリット・トレードオフ・限界
メリット
- 動画や高解像度のように巨大なトークン数を、プーリングで現実的な計算量に抑えられる。動画で特に効く。
- 階層型で多段特徴を出すため、検出・セグメンテーション・動画認識まで幅広く使える(MViTv2 で汎用化)。
- 浅い層を低チャネルにすることで計算とメモリを節約する。プーリングが学習可能で柔軟に重要情報を残せる。
- Query / Key / Value それぞれのプーリングを独立に設計でき、空間と時間の縮約を別々に制御できる。

トレードオフ・限界
- 各段のプーリング設定(ストライド、カーネル)・チャネル設計などハイパーパラメータが多く、設計の手間が大きい。
- 注意の内部でプーリングするため、純粋 CNN より実装が複雑。プーリングで情報を間引く以上、細部の喪失は原理的に避けられず、残差接続で補っても完全ではない。間引きすぎると細かい動きや小物体を取りこぼします。
- 画像単体タスクでは、後発で広く使われる Swin/ConvNeXt と比べ事前学習済み資産が限定的なことがある。デファクト化の差が実務上の採用しやすさに影響します。
- 動画では時間方向のプーリング設計が性能を左右し、タスク(短いクリップか長尺か、動きが重要か外見が重要か)に応じた調整が要る。

研究上の未解決課題として、空間・時間・チャネルのスケール変化を、タスクとデータに応じて自動設計する一般原理は確立しておらず、依然として経験的なアブレーションに頼ります。また、プーリング(固定的な間引き)が最良の縮約かどうか(トークンの重要度に応じた適応的な間引きとの比較)も開かれた問いです。

## 発展トピック・研究の最前線
MViT が示した「注意の内部でプーリングして階層化する」考えは、効率的な動画 Transformer の主要な一系統です。別系統の Swin がパッチ統合で階層化し、MaxViT がブロック内の多軸注意で局所と大域を両立させたのと対比すると、設計空間の広がりが見えます。階層化の手段が「注意内プーリング(MViT)」「注意外パッチ統合(Swin)」「軸の取り方の切替(MaxViT)」と三者三様である点が、この時代のバックボーン研究の豊かさを物語ります。

最前線では、自己教師あり事前学習(動画版 MAE、すなわち時空間のマスク復元)との組み合わせ、長尺動画への拡張、時空間トークンの効率的な間引き(トークンプルーニングやマージ、重要なトークンだけ残す適応的縮約)、そして検出・追跡・行動認識への統一バックボーン化が進んでいます。

時系列・履歴を扱う応用(過去フレームから将来状態を予測する系、運転シーンの先読みなど)では、MViT のように時間方向を自然に階層化できるバックボーンが参照点になります。空間を粗くしながら時間方向の文脈を蓄える構造は、長い時間スパンの依存関係を効率よく集約するうえで理にかなっており、動画理解と時系列予測の橋渡しとして研究が続いています。
