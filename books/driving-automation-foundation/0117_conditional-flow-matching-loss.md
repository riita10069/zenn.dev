# 条件付き Flow Matching 損失

## ひとことで言うと

Flow Matching の生成モデルを学習させるときの採点ルール(損失関数)です。ノイズと正解データを直線で結び、その途中の点でネットワークに「どっちへ進めばいい?」と尋ね、本当に進むべき向きとの二乗誤差を取ります。やっていることは「予測した速度と、目標とする速度の MSE(平均二乗誤差)」だけという、拍子抜けするほど単純な損失です。それでいて、たくさんのサンプルで平均すると正しい速度場が学習できる、という巧妙な理論的裏付けを持ちます。

## 直感的な理解

ノイズの霧の中からデータの島へ渡るための「地図上の矢印」を学習する作業だと考えてください。霧のどの地点に立っても、矢印は「島へ向かう正しい方向と速さ」を指している必要があります。

問題は、本当に学びたい矢印(データ分布全体に対する速度場)は複雑で、直接の教師信号がないことです。Flow Matching の発想の転換は、「正解データを1つ選んで固定すれば、そのデータへ向かう矢印は単純に計算できる」という点にあります。ノイズ点とその正解データを直線で結べば、矢印はその直線の向き(差ベクトル)そのものです。1つの正解に対する矢印は単純すぎて、それ単体では複雑な速度場になりませんが、無数の正解データについて平均すると、本当に欲しかった複雑な速度場が現れる。これが「条件付き(conditional)」という名前の意味です。

## 基礎: 前提となる概念

速度場(velocity field)v(x, t)。ノイズ分布をデータ分布へ連続的に変形させる流れの、各地点 x・各時刻 t での速度ベクトルです。これに従って点を動かせばノイズはデータに変わります。学習対象はこれを近似するネットワーク v_θ です。

確率経路(probability path)。時刻 t=0 でノイズ分布、t=1 でデータ分布になるよう連続的につなぐ分布の系列 p_t です。各サンプルレベルでは、ノイズ点 x_0 と正解 x_1 を結ぶ軌跡が条件付き経路を定めます。

線形補間と最適輸送(OT, optimal transport)。x_0 と x_1 を結ぶ最も単純な経路は直線で、`(1-t) x_0 + t x_1` です。これは2点間を最短距離で結ぶので、最適輸送の意味で素直な経路であり OT path/linear path と呼ばれます。直線なら速度が一定になるのが計算上の最大の利点です。

MSE(平均二乗誤差、mean squared error)。予測値と目標値の差を要素ごとに二乗して平均する損失です。回帰問題の標準的な採点法で、勾配が素直に計算でき学習が安定します。

## 仕組みを詳しく

学習1ステップの中身を、軌跡が4時刻×2信号=8次元のミニ例で具体的に追います(実用では 64×2=128 次元などになります)。バッチ1本で説明します。

1. 正解 x_1 を用意します。例: `x_1 = [0.1, 0.0, 0.2, 0.0, 0.3, 0.0, 0.4, 0.0]`(加速度がじわっと増える軌跡のつもり)。
2. ノイズ x_0 を作ります。`x_0 = randn_like(x_1)`。例: `x_0 = [-1.2, 0.5, 0.3, -0.8, 1.1, -0.2, 0.0, 0.9]`。
3. 時刻 t を1つサンプリングします。例: `t = 0.3`(t の選び方は [Flow Matching の時刻サンプリング戦略](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0125_flow-timestep-sampler))。
4. 直線上の中間点を計算します。`u_t = (1 - t) * x_0 + t * x_1 = 0.7 * x_0 + 0.3 * x_1`。最初の要素なら `0.7*(-1.2) + 0.3*0.1 = -0.84 + 0.03 = -0.81`。
5. 目標速度を計算します。`target_velocity = x_1 - x_0`。最初の要素なら `0.1 - (-1.2) = 1.3`。これが「u_t から x_1 へ向かう正しい向きと大きさ」で、直線なので t に関係なく一定です。
6. ネットワークに `u_t` と `t`(と条件、運転なら BEV や履歴)を渡して予測速度 `v_θ` を得ます。
7. 損失 `loss = MSE(v_θ, target_velocity)` を計算し、逆伝播で v_θ を更新します。

擬似コードにするとこれだけです。

```
x_0      = randn_like(x_1)
t        = sample_t(batch)            # [B]
u_t      = (1 - t) * x_0 + t * x_1    # 直線上の点
target_v = x_1 - x_0                  # 定数速度(t に依存しない)
loss     = mean((v_theta(u_t, t, cond) - target_v) ** 2)
```

なぜ直線経路を選ぶのか。ノイズと正解を結ぶ経路は無数に選べますが、直線なら目標速度が `x_1 - x_0` という定数になり、計算も実装も最も簡単になります。経路が曲がっていると速度が t によって変化し、損失の計算が複雑になるうえ、推論時の積分も難しくなります。直線経路はさらに、推論時の数値積分のステップ数を減らせるという実用上の利点も持ちます([Euler 法による ODE 推論（Flow Matching のサンプリング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0120_euler-ode-inference))。

なぜこの単純な損失で正しい速度場が学べるのか。本当に学びたいのは周辺速度場(marginal velocity field、データ分布全体に対する複雑な速度場)で、これを直接回帰しようとすると計算困難な積分が現れます。Lipman らの定理は「1サンプル x_1 を条件に固定した条件付き速度(=直線なら定数 x_1 - x_0)を回帰すると、その期待値(多数サンプルでの平均)が周辺速度場に一致する」ことを示しました。つまり、1本ごとに見ると単純な定数回帰なのに、たくさんのサンプルで平均すると複雑な目的が達成される、という仕掛けです。これにより計算困難な周辺速度場を、扱いやすい条件付き速度の回帰へ置き換えられます。

実装上の注意。学習で作る中間テンソル(ノイズ x_0、中間点 u_t、目標速度 target_v)は学習専用で、推論時には登場しません。生成モデルの実装では「推論(完成した軌跡を返す)」と「損失計算(速度を回帰する)」の契約を明確に分けるのが安全です。両者を混同すると、速度ベクトルを軌跡だと取り違えて MSE をかけてしまうバグ(意味のない値を正解として学習してしまう)を生みます。

## 手法の系譜と主要論文

Lipman et al., Flow Matching for Generative Modeling, ICLR 2023(arXiv:2210.02747)。条件付き Flow Matching(CFM)の損失を提案した基礎論文です。サンプル条件付きの速度を回帰すれば期待値として真の周辺速度場が学習できることを理論的に証明し、直線経路では目標が定数 `x_1 - x_0` になって損失が単純な MSE になることを示しました。拡散モデルが必要としたスコアマッチングや重み付けスケジュールを不要にした点が革新です。

Liu et al., Rectified Flow, ICLR 2023(arXiv:2209.03003)。同じく直線経路を使う独立な定式化です。経路を直線化(reflow)すると推論時の積分ステップを劇的に減らせる(極端には1ステップでも近似できる)ことを示しました。少ステップ推論を可能にする理論的支柱です。

Tong et al., Conditional Flow Matching(torchcfm), TMLR 2024(arXiv:2302.00482)。CFM の各種変種(独立カップリング、ミニバッチ最適輸送カップリングなど)を統一的に整理・実装したライブラリと論文です。x_0 と x_1 のペアの作り方(カップリング)を工夫すると経路がより直線的になり学習が安定することを示しました。実装の事実上の標準になっています。

Black et al., π0, Physical Intelligence, 2024(arXiv:2410.24164)。この CFM 損失をロボット行動生成に適用し、時刻 t をノイズ側へ偏らせる(低 t を多めにサンプリングする)と実応用で学習が安定すると報告しました。t サンプリング分布の設計が損失の有効性を左右することを実証しています。

## 論文の実験結果(定量データ)

Lipman らの ICLR 2023 は ImageNet などの画像生成で、CFM が同等の生成品質を拡散より少ない推論ステップで達成できることを FID(Fréchet Inception Distance、生成分布と実分布の距離。低いほど実データに近い)で示しました。直線経路の CFM が、学習曲線の安定性とサンプリング効率の両面で従来のスコアベース拡散を上回ったことが報告されています。

Rectified Flow(ICLR 2023)は、reflow を繰り返して経路を直線化すると、1ステップ推論でも FID の劣化がわずかに収まることを定量的に示しました。具体的には、多ステップ生成と1ステップ生成の FID 差が reflow により大きく縮まることが報告されており、これは「直線経路を学べば速度場がほぼ一定になり、少ない積分ステップでもズレが小さい」という直感を裏付けます。指標の意味としては、推論回数を桁で減らしても画質がほとんど落ちない、すなわちレイテンシ制約の厳しい応用で実用的だということです。

Tong らの torchcfm(TMLR 2024)は、ミニバッチ最適輸送カップリングを使うと、独立カップリングに比べて学習が速く収束し生成品質(FID や対数尤度)が改善することをアブレーションで示しました。x_0 と x_1 を無作為に組むのではなく輸送コストが小さくなるよう組むことで、経路の直線性が増し、結果として速度場の回帰が容易になる、という機序です。

π0(2024)では、t を一様分布ではなくノイズ側へ偏らせる Beta 分布などでサンプリングすると、実機タスクの成功率が向上することが報告されています。低 t 領域(ノイズに近い)の速度場を厚く学習することが、最終的な生成品質に効くという定量的知見です。

## メリット・トレードオフ・限界

メリット。損失が単なる MSE なので実装・デバッグが容易です。目標速度が定数なので数値的に素直で、勾配が安定します。拡散モデルのような複雑な理論や時刻依存の重み付けスケジュールが不要で、コードが短く保てます。

トレードオフと限界。t のサンプリング分布に精度が左右され、一様だと性能が出にくいことがあります(だから Beta 分布などの工夫が要る、[Flow Matching の時刻サンプリング戦略](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0125_flow-timestep-sampler))。1サンプルごとの教師(条件付き速度)は分散が大きく、多くのサンプルで平均して初めて周辺速度場に一致するため、十分なデータとバッチサイズが要ります。直線経路は単純ですが、データ分布が複雑だと中間点 u_t がデータ多様体から大きく外れた非現実的な状態になることがあり、そこでの速度学習が無駄になる場合があります。カップリングの工夫(最適輸送カップリング)はこれを緩和しますが計算コストが増えます。未解決の課題として、低次元データ(軌跡など)では生成モデルの恩恵が高次元データほど明確に出ないことがあり、損失設計と t サンプリングの最適点を見つけるのに試行錯誤が要ります。

## 発展トピック・研究の最前線

研究の最前線では、CFM の経路と目標速度をデータに合わせて設計する方向が活発です。最適輸送カップリングで経路を直線化する手法、確率経路の分散スケジュールを学習する手法、そして CFM を蒸留して1ステップ生成器にする手法が発展しています。

また、損失に物理的・幾何的な制約を組み込む方向もあります。軌跡生成なら、運動学的に実行可能か(加速度や曲率の上限を超えないか)、衝突しないかといった制約を、速度場の回帰損失に補助項として加えたり、ガイド付きサンプリングで推論時に反映したりする研究が進んでいます。理論面では Stochastic Interpolants(Albergo & Vanden-Eijnden)が Flow Matching と拡散を統一的に説明する枠組みとして注目され、決定論的(ODE)と確率的(SDE)なサンプリングを連続的につなぐ視点を与えています。
