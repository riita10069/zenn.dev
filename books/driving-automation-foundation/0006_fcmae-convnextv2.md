# FCMAE (Fully Convolutional MAE)

## ひとことで言うと
FCMAE(エフシーマエ、Fully Convolutional Masked AutoEncoder)は、画像の一部を隠して復元させる「マスク事前学習」を、Transformer ではなく畳み込みネットワーク(CNN)で行うための工夫です。隠した部分の計算を「スパース畳み込み(疎な畳み込み、穴あきデータをそのまま扱える畳み込み)」で飛ばすことで、CNN でも効率よくマスク学習を可能にしました。さらに「特徴の崩壊」を防ぐ GRN という新しい層をセットで導入することで初めて性能が伸び、畳み込みアーキテクチャ ConvNeXt の改良版 ConvNeXt V2 の中核になっています。学習法(FCMAE)と構造(GRN)を一緒に設計する "co-design" の好例です。

## 直感的な理解
マスク事前学習(画像の大半を隠して埋め直させ、ラベルなしで賢くする)は、もともと Transformer 系(ViT や Swin)で大成功しました。ところが、推論が速く実装も枯れていて車載・エッジ機器で重宝される CNN(畳み込みネットワーク)には、このマスク学習がそのまま効きませんでした。理由は CNN の物の見方にあります。

CNN は「画像は隙間なく埋まった 2 次元の格子」だと前提しています。小さなフィルタ(例: 3x3)を画像の上で滑らせ、各位置で周囲の画素を掛け合わせて特徴を作ります。ここで画像の 60% を隠して「穴あき画像」を作るとどうなるか。穴をゼロ(黒)で埋めると、フィルタは「本物の黒い物体」と「隠した場所のゼロ」を区別できず、穴の境界に存在しない模様(アーティファクト)を学んでしまいます。さらに穴の場所も普通に計算するので、隠した意味がなく計算も無駄です。Transformer ならマスクしたパッチを系列から抜き取れますが(MAE の高速化の核)、CNN は格子を崩せないのでそれができません。この壁のせいで、長らく自己教師ありのマスク学習は Transformer の独壇場になっていました。FCMAE は「CNN でもマスク学習を効かせたい」という動機から、この壁を乗り越えるために生まれました。

## 基礎: 前提となる概念

畳み込みネットワーク(CNN, Convolutional Neural Network)とは、小さなフィルタ(カーネル)を画像全体に滑らせて局所パターンを抽出し、それを層を重ねて高次の特徴へ組み上げる画像認識モデルです。計算が規則的で速く、ハードウェア最適化が進んでいるため推論効率に優れます。隣接画素の局所性を前提とする(近くの画素は関係が強い)ことが強みでもあり、マスク学習との相性の悪さの原因でもあります。

MAE(Masked Autoencoder, [MAE (Masked Autoencoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0071_mae-masked-autoencoder))とは、画像の 75% ほどを隠し、見えるパッチだけをエンコーダに通して隠した部分を復元させる自己教師あり手法です。ViT 向けに作られ、マスクパッチを「系列から取り除く」ことで計算を約 1/16 に減らして高速化します。Transformer は画像を「パッチの集まり(可変長の系列)」として扱うため、この除去操作が自然にできます。CNN はこれができないのが問題の出発点です。

スパース畳み込み(sparse convolution)とは、値が存在する場所だけで畳み込みを計算し、存在しない(マスクされた)場所を無視する畳み込みです。元々は 3 次元点群(空間にまばらに散らばる点の集合)を効率処理するために submanifold sparse convolution(Graham et al., 2017)や Minkowski Engine などのライブラリで発展した技術で、FCMAE はその考え方を 2 次元画像に応用します。マスクを「欠損した格子点」ではなく「存在しない場所」として明示的に扱うことで、「穴の境界に偽の模様が生まれない」「穴の場所を計算しないので速い」という 2 つの問題を同時に解きます。

ConvNeXt(Liu et al., CVPR 2022, arXiv:2201.03545)とは、Transformer の設計思想(大きなカーネル 7x7、層正規化 LayerNorm、GELU 活性化、逆ボトルネックなど)を純粋な CNN に取り入れ、Swin に匹敵する性能を出した畳み込みアーキテクチャです。「2020 年代の ConvNet」として、Transformer 全盛のなかで CNN がまだ競争力を持つことを示しました。ただし ConvNeXt(V1)は教師あり学習が前提でした。FCMAE はこの ConvNeXt を土台にし、自己教師あり化します。

## 仕組みを詳しく

FCMAE の事前学習は次の手順で進みます。tensor の形と数値例つきで追います。

1. 入力画像(例: `[B, 3, 224, 224]`)をパッチに区切り、約 60% をランダムにマスクします(MAE の 75% より低いのは、CNN の局所性で復元しやすいため難易度を調整した結果)。
2. ここが核心。マスク領域をゼロで埋めるのではなく、「マスクされていない画素だけが存在する疎(スパース)なデータ」として扱います。スパース畳み込みは値が存在する場所だけで計算し、存在しない場所を無視します。これで穴の境界に偽模様が生まれず、計算も残り 40% の場所だけで済みます。submanifold sparse convolution の考え方を 2D 画像に適用する実装で、マスクのパターンが層を通っても保たれる(穴が勝手に埋まっていかない)よう設計されています。
3. このスパースな入力を ConvNeXt エンコーダに通し、画像全体の特徴を得ます。エンコーダ全体が畳み込みだけで構成されるので "Fully Convolutional"(完全畳み込み)です。
4. 軽量なデコーダ(復元器)で、マスク領域の元ピクセル値を予測します。デコーダは MAE 同様に浅く軽量で、ConvNeXt V2 では単一の ConvNeXt ブロック程度の極小デコーダで十分と報告されています。
5. 損失は MAE と同じく「マスク領域のピクセル再構成誤差(平均二乗誤差 MSE)」だけで計算します。見えている部分は学習対象にしません。

しかし、スパース畳み込みだけでは性能が頭打ちになりました。原因を分析した著者たちは、ConvNeXt の特徴チャンネル(出力の各次元)の間で「特徴の崩壊(feature collapse)」が起きていることを発見しました。これは複数のチャンネルがほぼ同じ反応をしてしまい、せっかく多次元あるのに情報が重複して無駄になる現象です。著者は学習済みモデルのチャンネル間のコサイン類似度を可視化し、多くのチャンネルが冗長に飽和していることを定量的に示しました。マスク学習という難しい課題のもとで、この崩壊が特に顕著に現れました。

これを解決するために導入されたのが GRN(Global Response Normalization、大域的応答正規化)という新しい層です。GRN は次の 3 ステップで動きます。第一に、各チャンネルについて空間方向(画像の縦横全体)の反応の大きさを 1 つの数値に集約します(L2 ノルム = 二乗和の平方根)。第二に、その値を全チャンネルで比較して相対化します(自分は他のチャンネルに比べてどれだけ目立っているか、各チャンネルの集約値を全チャンネル平均で割る)。第三に、目立っているチャンネルは強め、地味なチャンネルは弱めるように元の特徴を掛け算で調整します(さらに学習可能なスケールとバイアスを足して、必要なら効果を抑えられるようにする)。これにより「チャンネル同士が役割分担して多様な特徴を持つ」ようになり、特徴の崩壊が解消されました。GRN は ConvNeXt ブロックの中の MLP 部分に 1 つ挿入するだけで、追加パラメータもごくわずかです。

```
入力 [B,3,224,224]
   │ 約60%マスク → スパースデータ化(ゼロ埋めではない)
   ▼
[ConvNeXtエンコーダ + GRN(スパース畳み込み)] ──► 特徴
   │ 軽量デコーダ
   ▼
マスク領域のピクセル予測 ──► マスク領域だけMSE損失
```

つまり ConvNeXt V2 = ConvNeXt(V1)のアーキテクチャに GRN を追加 + FCMAE で事前学習、という二段構えです。FCMAE という学習方法と GRN という構造改良はセットで効果を発揮しました。論文の実験では、FCMAE だけ、GRN だけでは伸び悩み、両方揃って初めて教師あり学習を大きく上回りました。これが論文タイトルの "Co-designing"(共同設計、学習法と構造を一緒に設計する)の意味です。事前学習が終われば、スパース畳み込みは通常の(密な)畳み込みに戻し、デコーダは捨て、ConvNeXt V2 エンコーダだけを下流タスクに使います。

## 手法の系譜と主要論文

源流は MAE(He et al., CVPR 2022, [MAE (Masked Autoencoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0071_mae-masked-autoencoder))です。マスク自己符号化の元祖で、ViT 前提でマスクパッチを系列から除外して高速化しました。CNN への移植が課題として残りました。並行して SimMIM([SimMIM](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0092_simmim_note))が階層型 Transformer(Swin)向けの MIM を確立し、マスクパッチもエンコーダに入れる方式を示していました。

土台のアーキテクチャは ConvNeXt V1(Liu, Mao, Wu, Feichtenhofer, Darrell, Xie; Meta, CVPR 2022, arXiv:2201.03545)です。Transformer の設計思想を純 CNN に取り込み、Swin に匹敵する性能を出しました。ただし V1 は教師あり学習が前提で、自己教師ありマスク学習とは組み合わせられていませんでした。先行して BEiT 系を CNN に当てはめる試みもありましたが、ゼロ埋めによるアーティファクトと特徴崩壊で素直には効きませんでした。

FCMAE と GRN を提案したのが ConvNeXt V2(Woo, Debnath, Hu, Chen, Liu, Kweon, Xie; Meta AI と KAIST, CVPR 2023, arXiv:2301.00808)です。新規性は 2 点の共同設計です。第一にスパース畳み込みによる FCMAE 事前学習で、CNN でもマスク学習を効かせました。第二に GRN 層で、マスク学習下の特徴崩壊を解消しました。動機は「CNN ではマスク学習が素直に効かず特徴崩壊が起きる」という観察でした。この系譜は、SimMIM([SimMIM](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0092_simmim_note))が階層型 Transformer 向けに、MAE が ViT 向けに切り開いたマスク学習を、ついに CNN にも広げたものと位置づけられます。スパース畳み込みの源流は 3D 点群処理の submanifold sparse convolution(Graham et al., 2017)で、別分野の技術を画像 MIM に転用した点も興味深いところです。

## 論文の実験結果(定量データ)

評価指標。画像分類は top-1 accuracy(トップ1正解率、高いほど良い)で測ります。ベンチマークは ImageNet-1k(1000 クラス・約 128 万枚)です。

ConvNeXt V2 論文の主な結果(報告による):

- ImageNet-1k で ConvNeXt V2 は同サイズの V1 を一貫して上回りました。例として ConvNeXt V2-Base は top-1 約 84.9% で、V1-Base(教師あり、約 83.8%)を約 1.1 ポイント改善しました。1 ポイントの差は飽和に近い高精度帯では大きな意味を持ちます。
- 幅広いスケールでの一貫性。Atto(約 350 万パラメータ)から Huge(約 6.6 億パラメータ)まで、ほぼ全サイズで V1 を上回りました。最小の ConvNeXt V2-Atto でも ImageNet-1k で top-1 約 76.7% に達し、V1 相当の小型モデルを明確に上回りました。小型モデルでも改善が見られた点は、車載・エッジのような軽量モデルが要る場面で価値があります。最大の ConvNeXt V2-Huge は、大規模データ(ImageNet-22k)での追加学習と高解像度微調整を組み合わせると ImageNet-1k で top-1 約 88.9% に到達したと報告されています。
- 下流転移。COCO 物体検出・インスタンスセグメンテーション(指標は AP)や ADE20K セマンティックセグメンテーション(指標は mIoU)で、教師あり事前学習や V1 を上回る転移性能を示しました。分類だけでなく密な予測タスクでも一貫して効くことが確認されています。

アブレーション(どの要素を抜くとどうなるか)。論文の最も重要な実験は「FCMAE と GRN の組み合わせ効果」です。FCMAE だけ(GRN なし)では、自己教師ありにしても教師あり学習をほとんど上回らず、特徴崩壊のせいで頭打ちになりました(改善はおよそ 0.x ポイント程度にとどまる)。GRN だけ(教師あり学習)でも改善は限定的でした。両者を組み合わせて初めて、教師ありスクラッチを明確に(約 1 ポイント超)上回りました。これは「学習法と構造を別々に最適化しても駄目で、共同設計が必要」という論文の中心主張を実証する結果です。さらに、ゼロ埋め畳み込み(穴をゼロで埋める素朴な方法)とスパース畳み込みを比較し、スパース畳み込みの方が偽模様を防いで性能が高いこと、GRN を他の正規化(LayerNorm 追加や BatchNorm など)に置き換えると効果が出ないこと、も示しました。

## メリット・トレードオフ・限界

メリット。CNN(ConvNeXt)でラベルなしマスク事前学習を効かせられます。推論が速い CNN を、自己教師ありの恩恵を受けつつ使えるため、計算資源が限られる車載・エッジ環境に有利です。スパース畳み込みでマスク領域の計算を省くため事前学習が効率的で、GRN により特徴の多様性が保たれて表現力が上がります。Transformer と違い格子構造を保つので、検出・セグメンテーションのような密な予測タスクとも素直に接続できます。小型から巨大まで一貫してスケールするため、用途に応じてモデルサイズを選べます。

トレードオフと限界。スパース畳み込みは標準の畳み込みより実装が複雑で、専用ライブラリ(submanifold sparse convolution / Minkowski Engine 系)を要します。これは導入のハードルになり得ます。事前学習時のスパース畳み込みと、下流での密な畳み込みの切り替えも実装上の注意点です。また生ピクセル再構成が目的のため、色・テクスチャに引っ張られやすく、純粋な意味的特徴では自己蒸留系([DINO (自己蒸留)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0044_dino_note)/[DINOv2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0046_dinov2_note))に劣る場合があります。GRN は ConvNeXt のチャンネル特性に合わせた工夫であり、他の CNN にそのまま効くとは限りません。

研究上の未解決課題。「なぜ CNN ではマスク学習で特徴崩壊が起きやすいのか」の理論的理解はまだ発展途上です。Transformer の self-attention が大域的な情報統合をするのに対し、CNN の局所畳み込みが冗長な反応に陥りやすい、という直感はありますが、定量的な説明は途上です。また、CNN 向け MIM が Transformer 向け MIM や自己蒸留系とどこまで性能で並べるか、特に大規模スケールでの優劣は研究が続いています。スパース畳み込みの計算効率も、ハードウェアによっては不規則メモリアクセスのため理論ほどの高速化が出ないことがあります。

## 発展トピック・研究の最前線

FCMAE は「マスク学習は Transformer 専用ではない」ことを示し、CNN を自己教師あり時代に引き戻しました。ConvNeXt V2 は推論効率と表現品質を両立する選択肢として、特に軽量・高速が求められる実運用(リアルタイム検出、車載知覚など)で重要な位置を占めます。Transformer の高精度と CNN の高速性のどちらを取るかという実務上の選択に、「CNN でも自己教師あり事前学習の恩恵を受けられる」という新しい選択肢を加えました。

研究の流れとしては、第一にスパース畳み込みの効率化と汎用化(専用ライブラリ依存を減らす)、第二に予測対象を生ピクセルから特徴へ変える方向(MIM と自己蒸留の融合、iBOT/DINOv2 の CNN 版に相当する試み)があります。GRN のような「特徴の多様性を保つ正規化」は、マスク学習に限らず一般の表現学習で崩壊(collapse)を防ぐ手段として注目されており、対照学習や自己蒸留の崩壊回避(BYOL や Barlow Twins などが扱う問題)とも通じる論点です。FCMAE は、アーキテクチャ(構造)と学習法を一体で設計する "co-design" の好例として、今後の表現学習設計に影響を与え続けています。
