# VMamba / Vision Mamba (状態空間モデルの視覚応用)

## ひとことで言うと
VMamba と Vision Mamba は、画像から特徴を取り出す土台ネットワーク(バックボーン)の新しい一派です。もともと文章のような時系列データ向けに作られた状態空間モデル(State Space Model, SSM)、特にその高速・高表現力版である Mamba を画像に応用しました。Transformer の自己注意が抱える「画素数の二乗に比例する重い計算」を避け、画素数に対して線形の計算量で、画像全体の長距離の関係(遠く離れた位置どうしの依存)を捉えます。

## 直感的な理解
画像認識で「遠く離れた2か所の関係」を捉えたい場面は多くあります。たとえば運転シーンで、画面左端の対向車と右端の歩行者の位置関係を同時に把握したい、といったときです。

Transformer の自己注意はこれを「全位置ペアの総当たり」で解きます。確実ですが、位置数(画素数)が増えると組み合わせが二乗で爆発し、高解像度画像では計算が重すぎます。

SSM のやり方はこれと根本的に違います。文章を読むように、画像の各位置を端から順番になめていき、「これまで読んだ内容の要約」を内部メモリ(状態)に溜めながら進みます。各位置では、自分の手前までの全情報が要約された状態を参照できるので、遠くの情報も「メモリ経由」で受け取れます。全ペアを総当たりしないので、位置数に対して計算は線形にしか増えません。日記を1ページずつ読みながら要点をノートに書き溜め、最後のページでも最初のページの要点を参照できる、というイメージです。総当たりで全ページを突き合わせる(自己注意)より、はるかに軽く済みます。

ただし、この「要約ノート」は容量が固定です。自己注意は全ページを生のまま保持して何度でも参照できますが、SSM は固定サイズの状態に圧縮します。圧縮するからこそ軽いのですが、何を残し何を捨てるかの選別が下手だと情報が失われます。この選別を賢くしたのが Mamba の核心(選択機構)で、視覚応用の鍵にもなります。

## 基礎: 前提となる概念
いくつかの用語を順に噛み砕きます。

「状態空間モデル(SSM)」は、もともと制御工学や信号処理で時系列を扱う数学モデルです。連続時間では次の微分方程式で書けます。

  h'(t) = A h(t) + B x(t)     (状態の更新)
  y(t)  = C h(t) + D x(t)     (出力の生成)

x(t) が入力、h(t) が内部状態(これまでの履歴の要約)、y(t) が出力です。A は状態がどう時間発展するか、B は入力をどう状態へ取り込むか、C は状態からどう出力を作るかを決める行列です。これを離散化(1ステップずつの漸化式に変換)すると、

  h_t = A_bar h_{t-1} + B_bar x_t
  y_t = C h_t

という形になり、RNN(再帰型ネットワーク)に似た「1ステップずつ状態を更新しながら出力する」計算になります。A_bar, B_bar は連続の A, B をステップ幅 Δ で離散化したものです。

なぜ SSM が注目されたか。第一に、線形の漸化式なので、状態更新を畳み込みカーネルの形に書き直すと、学習時には系列全体を並列に計算できます(RNN は逐次でしか計算できず遅いのが弱点でした)。第二に、推論時は RNN のように1ステップずつ走るので、系列長 N に対し計算量が線形 O(N)、メモリも一定です。長い系列で自己注意の O(N^2) に対し圧倒的に軽くなります。SSM は「学習は畳み込みで並列、推論は RNN で省メモリ」という両方の良さを併せ持つわけです。

「HiPPO 理論(Gu et al., NeurIPS 2020)」は、状態行列 A をどう設計すれば過去の履歴を最もよく圧縮・保持できるかを、直交多項式による関数近似の観点から導いた理論です。これが次の S4 の土台になりました。

「S4 (Structured State Space, Gu et al., ICLR 2022)」は、行列 A を特別な構造(HiPPO に基づく初期化)にして、長距離の履歴を効率よく状態に保持できるようにした SSM です。Long Range Arena という、数千〜1万を超える長い系列での依存関係を測るベンチマークで、当時の Transformer 系を大きく上回り、SSM 復権の起点になりました。

「選択的 SSM(selective SSM)= Mamba の核心」とは、行列 B・C やステップ幅 Δ を入力に応じて動的に変える仕組みです。従来の SSM はこれらが固定で、どの入力も一律に扱っていました(時間不変系)。Mamba は「今読んでいる内容に応じて、状態に残すか・忘れるか・どれだけ取り込むかを変える」ことで、文脈に応じた選択(重要な情報を覚え、不要を捨てる)を可能にし、Transformer 並みの表現力を線形計算量で達成しました。代償として、入力依存パラメータでは前述の畳み込み並列化が使えなくなる(時間不変でないと畳み込みにまとめられない)ため、Mamba は GPU 上で高速に走る専用の並列スキャン実装(hardware-aware parallel scan、selective scan)を用意しています。状態を高速なオンチップメモリ(SRAM)に置いて計算する工夫も含まれます。

「画像を系列にする問題」。SSM は1次元の系列を順になめる仕組みですが、画像は2次元で、どの順に画素をなめるかが自明ではありません。文章は左から右に読む順序が一意ですが、画像はどの位置も周囲の全方向と関係します。この「画像を系列にどう並べるか(スキャン順序)」をどう設計するかが、視覚版 SSM の核心の工夫です。

## 仕組みを詳しく
両モデルとも、まず画像をパッチ(小さなタイル)に分け、各パッチをトークン(系列の1要素)として扱います。たとえば形状 [3, 224, 224] の画像を 16×16 のパッチに分けると 14×14 = 196 個のトークンになります。問題は、この196個をどの順に SSM になめさせるかです。

Vision Mamba (Vim, Zhu et al., ICML 2024) のアプローチ
パッチ列を「前から後ろ」と「後ろから前」の双方向(bidirectional)にスキャンします。文章なら前から読むだけで足りますが、画像はどの位置も前後両側と関係するため、片方向では一方の側の文脈しか取り込めません。双方向にすることで、各パッチが系列の前後両側の情報を得られます。Vim は ViT に近いプレーンな(階層を持たず、最初から最後まで同じ解像度の)構造を採り、クラストークンと位置埋め込みも使います。これにより ViT の素直な置き換えとして位置づけられます。

VMamba (Liu et al., 2024) のアプローチ
SS2D(2D Selective Scan、2次元選択スキャン)を提案しました。画像を1方向だけでなく4方向(左→右、右→左、上→下、下→上)に展開してそれぞれ SSM でスキャンし、4つの結果を統合して2次元の特徴に戻します。こうすると、1次元の SSM でも2次元の空間的な近接関係を多角的に取り込めます。隣り合う画素が「ある走査順では遠いが別の走査順では近い」という偏りを、4方向の併用で緩和するのが狙いです。たとえば縦に隣り合う画素は、左→右の走査では1行分(横幅ぶん)離れてしまいますが、上→下の走査では隣どうしになります。さらに VMamba は Swin / ConvNeXt と同じ4ステージ階層構造(解像度を 1/4 → 1/8 → 1/16 → 1/32 と段階的に下げ、チャンネルを増やす)を採用し、検出・セグメンテーションのヘッドに渡しやすい複数解像度の特徴を出します。

計算量の違いを具体的に言うと、トークン数を N とするとき自己注意は O(N^2)、SSM スキャンは O(N) です。数値例として、入力解像度を 2 倍にすると N は 4 倍になり、自己注意の計算は 16 倍に膨らみますが、SSM は 4 倍に留まります。画像を大きくするほどこの差は劇的に開き、高解像度ほど SSM 系が有利になります。これがメモリ削減にも直結し、自己注意では収まらない高解像度入力を SSM なら扱えることがあります。

ASCII 図で SS2D の発想を示します。

  2D特徴 ─┬─ 左→右に展開 → SSM ─┐
          ├─ 右→左に展開 → SSM ─┤
          ├─ 上→下に展開 → SSM ─┼─統合→ 2D特徴へ戻す
          └─ 下→上に展開 → SSM ─┘

なぜこの設計なのか。SSM 単体は系列順に依存するので、1方向だけだと画像の一部の空間関係が遠くなってしまいます。双方向(Vim)や4方向(VMamba)に走らせて統合することで、2次元構造を持つ画像に SSM の線形コストの利点を活かしつつ、方向バイアス(走査順に起因する近さの偏り)を打ち消しているわけです。これは「1次元のモデルで2次元構造を扱う」という根本的な不整合を、複数走査の冗長性で埋める工夫だと理解できます。

## 手法の系譜と主要論文
系譜は「長距離依存を効率よく扱う SSM の発展」と「それを視覚へ移す試み」の2段で整理できます。

第一段は SSM の進化です。Gu et al. の HiPPO (NeurIPS 2020) が履歴圧縮の理論的基礎を与え、続く S4 (ICLR 2022) が、構造化した状態行列で長距離履歴を保持し、Long Range Arena などの長距離ベンチマークで Transformer を上回って SSM を深層学習の主役候補に押し上げました。続いて Gu & Dao, "Mamba: Linear-Time Sequence Modeling with Selective State Spaces" (2023) が選択的 SSM(入力依存パラメータ)とハードウェア対応並列スキャンを導入し、言語モデリングで同規模 Transformer に匹敵・凌駕する性能を線形計算量で示しました。これが SSM ブームの直接の引き金です。

第二段は視覚への移植で、2024年初頭にほぼ同時に2本が登場しました。Zhu et al., "Vision Mamba" (ICML 2024) は双方向 SSM によるプレーンな視覚バックボーンを提案しました。動機は片方向スキャンでは画像の全方向文脈を取り込めないことです。Liu et al., "VMamba: Visual State Space Model" (2024) は SS2D(4方向スキャン)と階層構造を提案しました。動機は2次元画像の空間関係を1次元 SSM でより忠実に捉え、密予測タスク向けの多解像度特徴を出すことです。両者は「プレーン型(ViT 系)」と「階層型(Swin 系)」という、視覚バックボーン設計の2系統をそれぞれ SSM で再現した格好になっています。

これらは「Transformer 一強の時代に対する第3の選択肢(CNN でも Transformer でもない SSM)」という文脈で、2024年に急速に注目された新興分野です。以降、医用画像(U-Mamba, SegMamba)、点群(PointMamba)、動画、リモートセンシング、3D 検出など多様な領域へ Mamba バックボーンが波及しています。

## 論文の実験結果(定量データ)
両論文の主要評価を、指標の意味とともに説明します。

ImageNet-1K 分類(約120万枚・1000クラス。指標は top-1 accuracy)。Vision Mamba(Vim-S、Small)は ViT 系・DeiT と同等以上の top-1 を、高解像度入力で大幅に低いメモリ・高いスループットで達成したと報告されています。論文の目玉は効率で、高解像度(たとえば 1248×1248 のような大きな入力)では DeiT に対し GPU メモリを大幅に削減(報告ではおよそ数分の1、文献では約 86% 削減という値も挙げられる)し、推論を高速化(報告では複数倍、バッチ推論で約 2〜3 倍規模)しました。これは N が大きいほど O(N) と O(N^2) の差が開くという、SSM の理論的利点を実測で裏付けたものです。

VMamba の ImageNet-1K 分類。VMamba-T(Tiny、約 22〜30M パラメータ帯)は top-1 約 82.5% 前後を達成し、同規模の Swin-T(約 81.3%)を上回りました。VMamba-S / VMamba-B も対応する Swin を上回る精度を報告しています(おおむね 0.5〜1.5 ポイント程度の上乗せ)。階層構造を採ったことで、分類だけでなく密予測タスクにも展開できる点が Vim との違いです。

COCO 物体検出・インスタンスセグメンテーション(指標は box AP / mask AP、0〜100 で大きいほど良い)と ADE20K セマンティックセグメンテーション(指標は mIoU、予測領域と正解領域の重なり率のクラス平均)。VMamba は Mask R-CNN / UperNet のバックボーン差し替えで、同規模の Swin を box AP・mIoU ともに上回ると報告されました(報告では数ポイント程度の改善)。密予測は全画素にわたる広い文脈が効くため、窓分割なしで長距離依存を線形コストで扱える SSM の利点が出やすいタスクです。特に入力解像度を上げたとき、Swin に対する効率・精度の優位が広がる傾向が示されました。

アブレーション。重要な分析は次の通りです。第一に、スキャン方向を1方向に減らすと精度が下がり、Vim の双方向・VMamba の4方向(SS2D)の有効性が確認されました。方向を増やすほど2次元の空間関係を取り込めるためです。第二に、選択的 SSM(入力依存パラメータ)を固定 SSM(従来の時間不変 SSM)に戻すと表現力が落ち、Mamba の選択機構が視覚でも効いていることが示されました。第三に、VMamba では階層構造の段数・チャンネル配分の設計が密予測性能に効くことが示され、Swin 由来の階層設計が SSM とも相性が良いことが確認されました。

## メリット・トレードオフ・限界
メリット
- 計算量・メモリがトークン数に対し線形。高解像度・大画像で自己注意より大幅に軽く、自己注意では収まらない解像度を扱える場合がある。
- 画像全体の長距離依存を、窓分割なしで状態(メモリ)経由で捉えられる。固定サイズの状態に圧縮するため、系列が長くても推論メモリが一定。
- 同規模の Swin / ViT と同等以上の精度。VMamba は検出・分割でも優位を報告。
- 高解像度ほど Transformer に対する効率優位が広がる。大判カメラ・医用・衛星画像などと相性が良い。

トレードオフ・限界
- 高速化は専用 CUDA カーネル(選択的並列スキャン実装)に依存し、移植性・デプロイ性に課題がある。特に車載アクセラレータや一部の推論エンジンでは未対応・最適化不足のことがあり、理論上の速度優位がそのまま実機で出るとは限らない。専用カーネルがないと、むしろ素朴な実装は遅くなりうる。
- 画像のスキャン順序設計(双方向・4方向など)に性能が依存する。SSM は本質的に1次元の系列モデルで、画像の2次元構造を順序で近似しているため、設計の良し悪しが効く。4方向にすると計算も4回分かかり、純粋な O(N) の係数が大きくなる。
- 固定サイズ状態への圧縮は、ある種の細かい局所情報の保持で自己注意に劣る場面がありうる。実際 SSM 単体より局所性を補う畳み込みを併用する設計が多い。
- 2024年登場の新興分野で、エコシステム・採用実績・長期安定性・事前学習済み重みの選択肢が ResNet / Swin に比べ未成熟。
- 小規模モデル・短系列(低解像度・少トークン)では、N が小さく O(N^2) と O(N) の差が小さいため、自己注意や CNN ほどの速度優位が出ないことがある。利点が出るのは主に大 N の領域。
- 未解決の課題として、選択的 SSM の学習安定性(状態行列の初期化や離散化ステップ Δ の感度)、長距離保持の理論的保証、画像の2次元性を順序に頼らず扱う方法(2次元 SSM の定式化)などが、活発に研究されている最中です。

## 発展トピック・研究の最前線
SSM 視覚バックボーンは「CNN でも Transformer でもない第3の軸」として、HorNet の高次相互作用や FocalNet の焦点変調と並ぶ「自己注意を線形コストで代替する」潮流の最新形です。さらに深く学ぶなら、まず HiPPO 理論(どんな初期化が長距離履歴を最もよく保持するか)と S4 の連続-離散変換を押さえると、Mamba の選択機構が何を改良したかが立体的に見えます。研究の最前線では、画像を2次元のまま扱う 2D-SSM の定式化、Mamba と注意のハイブリッド(局所は SSM・大域は疎な注意、あるいは逆など)、Mamba バックボーンの大規模事前学習と転移性、そして車載・組込みでの実機高速化(専用カーネル依存の解消)が主要なテーマです。さらに後継の Mamba-2 では、SSM と注意の数学的な双対性(state space duality)が示され、両者を統一的に最適化する枠組みが提案されています。応用面では、高解像度が本質的に効く動画・医用ボリューム・点群・リモートセンシングへの展開が急速に進んでいます。
