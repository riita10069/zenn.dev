# 行動条件付きワールドモデル

## ひとことで言うと
行動条件付きワールドモデル(action-conditioned world model)とは、「もしこの行動(走行軌道や操作)を取ったら、世界は次にどう見えるか」を予測する仕組みです。普通の未来予測が「現在から未来を当てる」だけなのに対し、行動を入力に加えることで「軌道Aを走った未来」と「軌道Bを走った未来」を別々に引き出せます。これにより、実際には起きていない「もし〜だったら」のシナリオ(反実仮想, counterfactual)を頭の中でロールアウト(roll-out、時間方向に展開してシミュレートすること)でき、計画と予測を一つのモデルに結合できます。

## 直感的な理解
普通の未来予測器は「現在のシーンから、平均的に最もありそうな未来」を出します。しかし自動運転で本当に欲しいのは「自分がどう動くかで未来がどう変わるか」です。

交差点で「直進する」か「右折する」かを迷っている場面を考えます。直進すれば前方の歩行者に近づき、右折すれば対向車線の車に注意が要ります。同じ現在シーンでも、選ぶ行動次第で数秒後の世界はまったく違います。行動を考慮しない予測器は、これらを混ぜた「平均的でぼやけた未来」しか出せません。これは「右にも左にも行ける」を平均して「まっすぐ薄い」になってしまうようなもので、どちらの選択肢の評価にも使えません。統計的には、複数のあり得る未来(マルチモーダルな分布)の平均は、どのモードにも一致しない無意味な点になりがちで、これが行動を入れない予測の根本的な限界です。

行動条件付きにすると、「軌道Aを入れたらこういう未来」「軌道Bを入れたらこういう未来」と、行動ごとに異なる未来を引き出せます。これがあると将来的には、複数の候補軌道それぞれの未来をシミュレートし「最も安全・快適な軌道」を選ぶ、という計画への活用が可能になります。これがワールドモデルを計画に役立てる肝です。

特徴空間の未来予測ヘッド([特徴空間で未来を予測するヘッド(JEPA 風 future-state head)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0011_future-state-jepa-loss))では「プランナーの意図を要約したベクトル」を条件にしていました。行動条件付けはそれをさらに一歩進め、「最終的に決まった軌道そのもの(行動)」を明示的に入れる経路です。意図ベクトルが「だいたいこっちに行きたい」という内部状態だとすれば、行動は「具体的にこの点列を辿る」という外に出した決定で、これを差し替えながら未来を比較できるのが価値です。意図は車のあいまいな気持ち、行動は確定したハンドル操作、と考えると違いが掴みやすいでしょう。

## 基礎: 前提となる概念
ワールドモデル(world model): 環境のダイナミクス、すなわち「現在の状態と取る行動から、次の状態がどうなるか」を学習したモデルです。物理シミュレータの代わりに、ニューラルネットが経験から近似します。記号で書けば、次状態 s' を s'≈ f(s, a) と予測する f を学ぶこと。s が状態、a が行動、f がワールドモデルです。

潜在状態(latent state): 生のピクセルではなく、シーンを圧縮した低次元あるいは特徴的な表現です。多くのワールドモデルはこの潜在空間でダイナミクスを学びます。ピクセルで予測するより軽く、安定するためです。たとえば 1280×720 の画像を 256 次元のベクトルや [256, H, W] の特徴マップに落としてから未来を予測します。

行動(action): エージェントが取る操作です。古典的な強化学習では「アクセル・ハンドル量」のような低次元ベクトル、計画系では「これから T 秒の軌道(点列)を平らに並べたベクトル」を行動とみなします。たとえば 0.5 秒刻みで 6 秒分、各点 (x, y) なら 12×2=24 次元、これを action として扱います。

ロールアウト(roll-out): 現在状態から始め、「次状態を予測 → それを入力に次々状態を予測」と時間方向に繰り返して未来系列を生成することです。行動条件付きなら、各ステップで与える行動を変えることで異なる未来枝が得られます。1 ステップだけ予測するのが one-step、複数ステップ繋ぐのが multi-step ロールアウトです。

反実仮想(counterfactual): 「実際には選ばなかった行動を取っていたらどうなったか」という仮想シナリオです。行動条件付きワールドモデルは、現在を固定して行動だけ差し替えることで、これを安全に(実車を動かさずに)評価できます。これがオフラインで安全に計画を検証する鍵になります。

## 仕組みを詳しく
典型的な実装では、特徴空間の未来予測ヘッドの条件付け経路に「行動を変換する線形層」を追加します。コンストラクタで行動次元 action_dim を指定するとこの層が作られ、指定しなければ(None)作られず通常の無条件挙動になります。設計の意図は「行動を使わない従来の予測モードと、行動条件付きモードを、同じヘッドで切り替えられる」ようにすることです。

```python
self.action_proj = (
    nn.Linear(action_dim, embed_dim) if action_dim is not None else None
)
```

forward では、軌道 trajectory([B, action_dim]、計画軌道を平らに並べたベクトル。例: タイムステップ数 × 信号数を一列にしたもの)を受け取り、線形層で [B, embed_dim] に変換し、[B, embed_dim, 1, 1] に整形して、意図ベクトルの bias に「もう一つの per-channel bias」として足します。

```python
bias = self.ego_proj(ego_hidden).view(-1, embed_dim, 1, 1)
conditioned = fused_features + bias
if trajectory is not None:
    action_bias = self.action_proj(trajectory).view(-1, embed_dim, 1, 1)
    conditioned = conditioned + action_bias   # 行動を追加注入
```

形の流れ(数値例: B=2, embed_dim=256, BEV=8×8, action_dim=40):
```
fused_features [2,256,8,8] ──────────────┐
ego_hidden [2,256] ─ ego_proj ─ [2,256,1,1] ─(broadcast +)
trajectory [2,40] ─ action_proj ─ [2,256,1,1] ─(broadcast +)
                                          ▼
                              conditioned [2,256,8,8] ─▶ Conv×2 ─ chunk×4 ─▶ 未来4枚
```

反実仮想ロールアウトの手順はシンプルです。同じ fused_features と ego_hidden を固定したまま、trajectory だけを軌道A・軌道Bと差し替えて2回 forward すれば、行動ごとの未来予測が2セット得られます。実装上は、行動条件を作っていないのに軌道を渡す、といった誤用に対して例外(ValueError)で弾く安全ガードを入れるのが定石です。これは「action_proj が None なのに trajectory が来た」「逆に action_proj があるのに trajectory が来ない」といった不整合を早期に検出するためです。

ここで設計の素直さと限界の両方を押さえます。行動の注入が「全空間に同じ値を足す per-channel bias」である点です。実装が軽く崩壊もしにくい反面、「行動が画面の特定の場所だけに効く」という空間的にきめ細かい影響は表現しづらい近似です。右折なら本来は画面右側の自由空間や対向車の特徴が大きく変わるはずですが、一律の bias ではその構造を明示的には入れられません。より表現力の高い注入(クロスアテンションで軌道点ごとに対応する空間位置を変調する、あるいは FiLM のように乗算的に変調する)が発展形になります。なぜまず bias 加算を選ぶのかというと、最小の追加パラメータで動き、学習が安定し、表現崩壊しにくいからです。表現力と安定性のトレードオフの、安定側に振った素直な第一歩と言えます。

## 手法の系譜と主要論文
- David Ha & Jürgen Schmidhuber, "World Models" (NeurIPS 2018, arXiv:1803.10122)。行動条件付きワールドモデルの古典です。VAE(画像を潜在ベクトルに圧縮する生成モデル)で観測を潜在に落とし、再帰ネットワーク(MDN-RNN、混合密度ネットワーク付き RNN)が「現在の潜在 + 行動」から次の潜在を予測します。提案動機は「エージェントが頭の中(潜在空間)で行動の結果を夢のようにシミュレートし、そこで方策を学べば実環境の試行を減らせる」こと。潜在空間での行動条件付き予測という、今に続く骨格を与えました。
- Danijar Hafner et al., Dreamer 系(Dreamer: ICLR 2020 / DreamerV2: ICLR 2021 / DreamerV3: 2023, arXiv:2301.04104)。学習した潜在ワールドモデルの中で行動を多数ロールアウトし、想像上の経験だけで方策を改善します。中核は RSSM(Recurrent State-Space Model、決定的な再帰状態と確率的な潜在を組み合わせて未来を予測する構造)。提案理由は実環境より圧倒的に安く未来を試せること。DreamerV3 は同一のハイパーパラメータで多様なタスク(Atari, 連続制御, Minecraft のダイヤ採掘)を解き、ワールドモデルベース強化学習の汎用性を示した点で重要です。
- 自動運転特化では MILE (Hu et al., NeurIPS 2022) が行動と潜在ダイナミクスを同時に学び、行動条件付きで将来の BEV を予測しながら方策を出します。GAIA-1 (Wayve 2023, arXiv:2309.17080) は「行動(操舵・速度・言語指示)を条件に未来の運転映像を生成」する生成系で、見て分かる反面、計算が重く特徴空間方式([特徴空間で未来を予測するヘッド(JEPA 風 future-state head)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0011_future-state-jepa-loss))とは予測する場所が異なります。GameGAN (Kim et al., CVPR 2020) は行動条件付きで「遊べるゲーム画面」を生成した初期例(PAC-MAN を画面ごと学習で再現)で、行動条件付き生成の表現力を印象づけました。

## 論文の実験結果(定量データ)
- World Models (2018): CarRacing-v0 という運転風タスク(トップビューでコースを走る強化学習課題)で、当時の他手法を上回り、初めて 900 点超(平均約 906、これがこのタスクの「解けた」とされる閾値)に到達しました。さらに VizDoom の回避タスクでは「実環境を一切使わず、学習したワールドモデルの夢の中だけで学習した方策」が実環境へ転移して成功し、モデル内学習(model-based)の有効性を実証しました。これは「想像の中の練習が本番で通用する」ことを初めて鮮やかに示した実験です。
- DreamerV3 (2023): 150 以上のタスクを単一設定(タスクごとにハイパーパラメータを調整しない)で解き、特に Minecraft でダイヤモンドを人手データなしで採掘するという、それまで未踏とされた成果を報告しました。これは「行動条件付きワールドモデルの中での想像ロールアウトが、長期・疎報酬(報酬がめったに得られない)のタスクでも方策学習を駆動できる」ことの強い証拠です。サンプル効率(目標性能に到達するまでに必要な実環境ステップ数)でもモデルフリー手法を大きく下回ることが繰り返し示されています。実環境の試行が高価な自動運転にとって、この効率は本質的な利点です。
- MILE (NeurIPS 2022): CARLA のクローズドループ評価で、行動条件付き潜在ワールドモデルを使った計画が、先行手法に対し運転スコア・違反指標を改善したと報告されました。生成系の GAIA-1 は定量精度より「多様で一貫した未来運転映像を行動条件で生成できる」という生成品質・スケーリング(モデルを大きくするほど予測が一貫し長くなる)の知見が中心です。

アブレーション的に重要なのは、Dreamer 系で繰り返し確認される「ワールドモデルの予測精度がそのまま方策性能の上限を決める」という関係です。モデルの予測が不正確だと、その夢の中で最適化された方策は実環境で裏切られます(夢では衝突しないが現実では衝突する)。これは行動条件付きワールドモデルの最大の弱点(誤差の伝播)を端的に表しています。逆に言えば、ワールドモデルの予測を改善する研究はそのまま方策・計画の改善に直結する、というのがこの分野の駆動力です。

## メリット・トレードオフ・限界
- メリット: 「行動を変えたら未来がどう変わるか」を予測でき、計画と予測を一つのモデルに結合できる。
- メリット: 現在を固定して軌道だけ差し替える反実仮想ロールアウトが容易で、実車・実シミュレータを回さず候補軌道を比較できる(モデル内クローズドループ、[クローズドループ vs オープンループ評価](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0746_closed-loop-vs-open-loop-eval))。
- メリット: 行動入力を作らない設定では従来の無条件予測と完全互換にでき、誤用は例外で防げる。
- トレードオフ: 行動注入が全空間一様の bias だと、空間的にきめ細かい行動の影響を表現しづらい。
- トレードオフ: ワールドモデルの予測誤差が長期ロールアウトで累積し、それを信じた計画が誤るリスク(モデルベース手法共通の課題)。誤差が大きいほど反実仮想評価の信頼性が下がる。
- トレードオフ: 学習には「行動と、その行動を取った後の未来観測」が対になったデータが要る。人間運転ログは「人間が選んだ行動」しか含まないため、選ばなかった行動の未来(反実仮想の正解)を直接学べない、という根本的なデータ不足がある。これは off-policy / counterfactual 評価の難しさそのものです。
- 未解決課題: 長期予測の安定化、不確実性の表現(未来は一意でなく分岐する)、空間的にきめ細かい行動条件付け、そして反実仮想予測の正しさを検証する手段の欠如。とくに「実際には起きなかった行動の未来」をどう検証するかは、定義上正解データが存在しないため原理的に難しい問題です。

## 発展トピック・研究の最前線
最前線では、JEPA 系の特徴空間予測を行動条件付きへ拡張する流れ(V-JEPA に行動入力を加え、ロボット操作の計画に使う V-JEPA 2 など)と、生成系ワールドモデルを大規模化して「運転シーンを行動条件で長く一貫して生成する」流れ(GAIA 系、運転ドメインの拡散モデル世界モデル)が並走しています。論点は一貫して「予測の質 対 計算コスト」と「ロールアウトの誤差累積をどう抑えるか」です。

計画への結合では、学習したワールドモデルを使って候補軌道を頭の中で試走させ、衝突や快適性を評価して最良を選ぶ「モデル予測制御(MPC)+ 学習ワールドモデル」の方向や、ワールドモデルの想像ロールアウトを使った安全な強化学習が活発に研究されています。これは [クローズドループ vs オープンループ評価](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0746_closed-loop-vs-open-loop-eval) で述べた「閉ループ性能を学習段階から目標にする」流れと深くつながっています。さらに最近は、大規模言語モデル的なトークン化で運転シーンを系列予測する世界モデルや、行動を言語指示として与える(「次の交差点を左折」)マルチモーダルな条件付けも登場しており、行動の表現自体が点列ベクトルから自然言語・離散トークンへと広がりつつあります。
