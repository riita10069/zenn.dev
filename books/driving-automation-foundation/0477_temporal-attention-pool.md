# temporal attention-pool (時間方向集約)

## ひとことで言うと
過去Nフレームの記憶を1本のベクトルにまとめるとき、「全部を等しく平均する」のではなく「今の判断に効くフレームを学習で重視して」まとめる仕組みです。1個の問い合わせ役(クエリ)がN個のフレームに注意(attention)を向け、重要なものに大きな重みを置いて加重平均します。出力サイズはNに依存せず一定になります。空間方向(マルチカメラ)の集約に使うのと同じ「クエリ+cross-attention」の部品を、時間軸に流用したものです。

## 直感的な理解
過去5秒間の運転シーンを思い出すとき、人は全部の瞬間を均等に重視しているわけではありません。「3秒前に歩行者が縁石から一歩踏み出した」決定的な瞬間は強く記憶し、「何も起きていなかった4秒前」は流します。つまり記憶の重み付けは場面によって変わります。

[RollingHistoryBuffer (FIFO履歴)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0471_rolling-history-buffer-fifo) のように過去フレームをただ連結すると、全フレームが完全に等重みになります。これは単純で頑健ですが、「どの瞬間が今の判断に効くか」をモデルが選べません。また連結だと履歴長Nを変えるたびに出力次元(D×N)が変わり、下流モジュールを作り直すことになります。

temporal attention-pool は、注意機構を使って「N個のフレームから自動的に重要なものを選び出して1本にまとめる」ことでこの2つを同時に解決します。重要度は学習で決まり、出力次元はNに依存しません。空間方向(カメラ)の集約に使うのと同じ部品を時間軸に流用できるので、設計も一貫します。

## 基礎: 前提となる概念
フレーム埋め込み列 (sequence of frame embeddings)。各時刻のシーンを1本のベクトルにした列で、形は `[B, N, D]`。Bはバッチ、Nは履歴フレーム数、Dは1フレームの次元(たとえば224)。各フレーム埋め込みの作り方は [FrameEncoder (GAP+Linear埋め込み)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0354_frame-encoder-gap-linear) を参照。

クエリ (query)。「N個の中から何を取り出したいか」を表す学習可能な1本のベクトル `[B, 1, D]`。固定の問い合わせ役で、学習で中身が決まります。これがN個のフレームに注意を向けて1本に集約する主体です。学習が終わればクエリの値は固定され、推論時は全入力で同じクエリを使います(注意の重みは入力ごとに変わる)。

cross-attention(交差注意)。問い合わせ役(クエリ)と情報源(キー・バリュー)が別物の注意です。クエリと各フレームの相性スコアを計算し、softmax(合計1に正規化する関数)で重みにし、その重みでフレーム群を加重平均します。重要なフレームほど大きい重みになります。式は `softmax(QKᵀ/√d) V` で、Q がクエリ、K と V がフレーム列です。

時間位置埋め込み (temporal positional embedding)。各フレームが「何番目か(=どれくらい昔か)」をモデルに教える、位置ごとの小さなベクトルです。これを足さないと、注意機構は順序の区別がつきません。注意は本質的に「順序を持たない集合演算」だからです。たとえば t-1 と t-5 を入れ替えても、位置埋め込みがなければ結果は同じになってしまいます。位置埋め込みには、固定の正弦波(Transformer 由来。位置 pos と次元 i に対し `sin/cos(pos/10000^(2i/d))` で与える)を使う流儀と、学習可能なベクトルにする流儀があります。

崩壊 (collapse)。学習可能な集約器で起きうる失敗で、クエリが入力を無視して常にほぼ同じ出力(定数や1フレームだけ)を返すようになる現象です。データが少ない・損失が弱いと起きやすく、注意機構が「学習したふりをして何もしていない」状態になります。

## 仕組みを詳しく
処理をステップで説明します。

登場人物:
- フレーム埋め込み列 `[B, N, D]`(例 N=6, D=224)
- 学習可能なクエリ `[B, 1, D]`
- 時間位置埋め込み `[1, N, D]`(各位置に1本)

ステップ:

1. 各フレーム埋め込みに、その位置に対応する時間位置埋め込みを足す。
   - `tok = frame_embeddings + time_pos` → `[B, N, D]`
2. 学習可能なクエリを1本用意し、バッチ分に複製する。
   - `q = history_query.expand(B, -1, -1)` → `[B, 1, D]`
3. クエリでN個のフレームトークンに cross-attention をかける。
   - `out, _ = history_attn(q, tok, tok)` → `[B, 1, D]`
   - 中身: クエリと各フレームの相性スコアを計算し、softmax で0〜1の重みに正規化し、その重みでフレーム群を加重平均。
4. 余分な次元を落として LayerNorm で正規化。
   - `encoded_visual_history = norm(out.squeeze(1))` → `[B, D]`

注意の重みのイメージ(あくまで例):

```
フレーム:        t-5    t-4    t-3    t-2    t-1    t(現在)
attention重み:   0.05   0.05   0.10   0.15   0.25   0.40
→ 直近を重視した加重平均で 1 本のベクトルに集約
```

ただし重みは固定ではありません。歩行者が3秒前に飛び出した場面なら t-3 の重みが跳ね上がる、という具合に入力に応じて変わります。これが等重み連結との決定的な違いです。

出力次元をどう揃えるか。pool した出力は1フレームと同じ D 次元(例 224)になります。一方、下流モジュールが別の次元(たとえば896)を期待することがあります。その場合は attention-pool の後ろに `Linear(D → 896)` を1枚足して、連結方式と同じ出力次元に揃えます。こうすると「同じインターフェース・同じ出力次元」のまま、内部だけ連結と attention を差し替えられます(swappable)。連結方式をデフォルトに残し、attention をオプトイン(opt-in、明示的に有効化したときだけ使う)にするのが安全な導入手順です。

なぜ位置埋め込みが必須か(数値例)。位置埋め込みがないと、`[e_{t-5}, ..., e_t]` を任意に並べ替えても出力は変わりません(注意は集合演算で順序不変)。すると「直近を重視する」という時間的な重み付けは原理的に学習できません。時間位置埋め込みを足して初めて、クエリは「これは最新フレーム」「これは古いフレーム」を区別でき、時間順に応じた注意を学べます。連結方式が位置埋め込みを必要としないのは、連結では「並べた位置そのものが順序を表す」からで、attention に移ると順序情報を明示的に補わなければなりません。

空間集約との統一。この「学習クエリ+cross-attention pool」は [ViewAttentionPool (マルチビュー集約)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0319_view-attention-pool) とまったく同じ構造です。違いは、注意をかける軸がカメラ(V個)か時間(N個)か、足す埋め込みがカメラ識別か時間位置か、だけです。1つの基本演算で空間集約と時間集約の両方を賄えるのは、実装・保守・理解のいずれにも効く設計上の利点です。

## 手法の系譜と主要論文
Transformer (Vaswani et al., NeurIPS 2017, arXiv:1706.03762)。注意機構と positional encoding(位置符号化)の起源です。注意が順序を持たない問題を、位置ごとのベクトルを足すことで解決しました。temporal attention-pool の時間位置埋め込みはこの直系です。

Set Transformer (Lee et al., ICML 2019, arXiv:1810.00825)。学習可能なシード/クエリで集合を1点に要約する PMA (Pooling by Multihead Attention) を提案しました。動機は「平均や最大では集合の重要要素を選べない」こと。新規性は「要素ごとの重要度を学習でき、要素数に対して出力サイズが一定」な集約。時間方向のフレーム集合に適用したのが temporal attention-pool です。

Perceiver (Jaegle et al., ICML 2021, arXiv:2103.03206)。少数の学習可能な潜在クエリで大量の入力に cross-attention して圧縮する考え方で、attention-pool の一般化です。クエリ数を変えることで出力の粒度を制御できます。

BEVFormer (Li et al., ECCV 2022, arXiv:2203.17270)。temporal self-attention(時間方向の自己注意)で過去の BEV 特徴と現在を融合しました。過去フレームの寄与を場面に応じて変えたい、という動機は temporal attention-pool と共通です。違いは、BEVFormer が空間的に構造化された BEV グリッド上で時間融合するのに対し、temporal attention-pool は構造を持たない1ベクトル列に対して軽量に集約する点です。

TimeSformer (Bertasius et al., ICML 2021, arXiv:2102.05095)。ビデオ理解で、空間注意と時間注意をどう組み合わせるか(同時か、分離か)を体系的に比較しました。空間と時間を分けて順に注意する「divided attention」が、計算効率と精度のバランスで良いと報告しています。時間集約を空間集約と分離して段階的に行う設計の理論的背景になります。

系譜としては「単純連結/平均(等重み)→ 再帰的状態(GRU/LSTM)→ 学習可能な注意集約(attention pool)」と進んできました。注意集約は、再帰の「遠い過去を忘れる・並列化しにくい」弱点と、連結の「等重み・次元がNに依存」する弱点を、同時に緩和する位置にあります。

## 論文の実験結果(定量データ)
Set Transformer。集合演算タスク群で、PMA は平均・最大プーリングのベースラインを上回りました。代表例として、点群分類(ModelNet40)や、混合分布のパラメータ推定で、同じエンコーダのまま集約器を PMA にするだけで誤差が減ることが示されています。アブレーションの教訓は「集約器を学習可能にする効果は、対象が複数要素の集合であるほど大きい」こと。

Perceiver。ImageNet で、少数の潜在クエリへ集約しても畳み込み前提のモデルに迫る精度(報告で top-1 およそ78%前後)を達成し、注意による集約が情報を大きく損なわないことを示しました。

BEVFormer。nuScenes で、時間的注意を加えると速度推定 mAVE(mean Average Velocity Error、小さいほど良い)が大きく改善し、総合 NDS(高いほど良い)が数ポイント向上しました(代表値 NDS およそ0.56)。アブレーションで時間融合を外すと速度推定が顕著に劣化します。これは「過去フレームを学習可能な注意で融合すること」の効果を直接示します。

TimeSformer。Kinetics-400 などの行動認識ベンチマークで、divided space-time attention が、空間のみ・時空間同時(joint)の各方式より、精度と計算量のバランスで優れることを報告しました。時間方向の注意を独立に持つことの有効性を支持します。

時間集約を単純連結から注意に替えたときの効き目は、最終的にはタスクの評価(自動運転ならオープンループの軌道予測誤差やクローズドループの運転成功率)で測る必要があります。一般に、(1) データが十分あれば注意集約は連結を上回りやすいが、(2) データが少ないと崩壊や過学習で逆に劣ることがある、という両面が報告されており、「常に注意が勝つ」わけではない点が研究上の注意点です。計算面では、注意集約は N トークンへの注意1回ぶん(`O(N)` 規模)で、時系列を順に処理する再帰エンコーダ(GRU)より軽く並列化しやすく、レイテンシも同程度かそれ以下にできることが多いと報告されています。

## メリット・トレードオフ・限界
メリット
- フレームごとの重要度を学習でき、場面に応じた重み付けができる(等重み連結より表現力が高い)。
- 出力次元がNに依存しないので、履歴長を柔軟に変えられる。
- 空間集約(カメラ方向)と同じ注意プーリング部品を時間集約にも使え、設計が一貫する。
- 時間位置埋め込みにより、不等間隔・欠損フレームにも比較的対応しやすい。

トレードオフ・限界・未解決課題
- 学習パラメータが増える(クエリ、位置埋め込み、注意の重み)。学習データが少ないと過学習や、出力が定数に潰れる崩壊のリスク。
- 連結に比べて計算とデバッグの難易度が上がる。注意の重みが妥当か(本当に決定的フレームを重視しているか)を可視化・定量で別途検証する必要がある。
- どの集約が実際に「運転や予測を良くする」かは、最終タスクの評価で確かめないと分からない。中間指標(注意の鋭さなど)だけでは判断できない。
- 単一クエリは「シーンの1側面」しか取り出せない。複数の関心(直近の障害物と全体の流れ)を同時に持ちたい場合は表現力が不足しうる。

## 発展トピック・研究の最前線
- 複数クエリへの拡張。クエリを数本に増やすと、時間方向の異なる側面(直近の急変・全体のトレンド)を別々に集約できます。Perceiver/Set Transformer の ISAB/PMA 設計が参考になります。
- 時空間同時集約。カメラ軸と時間軸をまとめて1回の注意で集約する(時空間トークンに対する単一クエリ)アプローチもあり、空間と時間の相互作用を1段で捉えられます。ビデオ Transformer(TimeSformer など)の時空間注意が関連します。
- 相対位置 vs 絶対位置。「t-3」のような絶対位置でなく、フレーム間の時間差(秒数)を相対位置として埋め込むと、可変フレームレートに頑健になります。
- 崩壊対策。補助損失(再構成や対比)、注意温度の調整、クエリ正則化、ウォームアップなどが、attention pool の安定学習で研究されています。
- 再帰との併用。直近は明示的バッファ+注意、遠い過去は再帰的状態で要約する、というハイブリッドで「並列化と長期記憶」を両立させる方向もあります。
