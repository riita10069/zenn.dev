# ラベルスムージング

## ひとことで言うと
ラベルスムージング(label smoothing)とは、分類学習で使う正解ラベルを「正解クラスは確率100%、それ以外は0%」という極端な形(one-hot)にせず、「正解は90%くらい、残りの10%を他のクラスに薄く配る」というように少し甘くして学習させる正則化のテクニックです。モデルが正解に過剰な自信を持つこと(自信過剰)を防ぎ、出力する確率の正直さ(校正)を改善し、間違いを含むノイジーなラベルにも頑丈になります。やることは目標分布を少しぼかすだけで、コードはほぼ1行、計算コストはほぼゼロです。

ここで「正則化(regularization)」とは、モデルが学習データに合わせすぎる(過学習する)のを抑え、未知データへの当たり(汎化)を良くするための工夫の総称です。

## 直感的な理解
学校のテストを思い浮かべてください。ある生徒(モデル)に「この問題の答えは絶対にAだ、1ミリの迷いも持つな」と教え込むと、その生徒は自信満々になりますが、問題文が少しひねられたり、教科書の答え自体が誤植だったりすると大崩れします。一方「答えはたぶんAだけど、B・C・Dの可能性もゼロではない」と教えれば、生徒は柔軟に構え、変な問題やミスのある答えにも惑わされにくくなります。

ラベルスムージングはこの「たぶんAだけど他も少しありうる」を機械学習の目標に落とし込んだものです。なぜこれが要るかというと、後で詳しく見るように、通常のクロスエントロピー学習はモデルを際限なく自信過剰にする力を持っているからです。自信過剰なモデルは「99%確信」と言って平気で外します。自動運転や医療のように「自分の確信度がどれだけ信用できるか」が安全に直結する分野では、これは致命的になりえます。たとえば前方に歩行者がいる確率を99%と言い切るモデルが、実際には7割しか当たっていなかったら、下流の判断(止まるか進むか)が崩れます。ラベルスムージングは、その暴走に軽いブレーキをかける最も安価な手段です。

## 基礎: 前提となる概念
理解には3つの概念が必要です。順に噛み砕きます。

第一に one-hot ベクトル。分類では正解を「正解クラスだけ1、他は全部0」というベクトルで表します。5クラスで正解が0番なら `[1, 0, 0, 0, 0]` です。これが普通の正解の表現で、確率分布として見ると「正解クラスに確率の全質量が集中した、最も尖った分布」です。

第二に softmax と logits。ニューラルネットの出力は最初は実数の生スコア(logits、ロジット)で、たとえば `[3.2, -1.0, 0.5, -2.0, 0.1]` のような値です。これを確率に変換するのが softmax で、各 logit を指数関数にかけて合計が1になるよう正規化します。`p_k = exp(z_k) / Σ_j exp(z_j)`。z_k は k 番目の logit、p_k はそのクラスの予測確率です。logit はいくらでも大きく/小さくできる無制約の実数で、softmax を通すと 0〜1 の確率になります。

第三にクロスエントロピー損失。予測確率分布 p と目標分布 q の食い違いを測る指標で、`CE = - Σ_k q_k log(p_k)` と書きます。q が one-hot なら正解クラス y だけ残って `CE = - log(p_y)` となり、「正解クラスの確率を1に近づけよ」という圧力になります。詳しくは [クロスエントロピー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0352_cross-entropy-loss) を参照してください。

ここで本質的な問題が見えます。p_y を完全に1にするには、softmax の性質上、正解クラスの logit を他より無限大に大きくしなければなりません(`exp(∞)/Σ → 1`)。クロスエントロピーは p_y がどれだけ1に近づいても損失が完全に0にはならないので決して満足せず、学習を進めるほどモデルは正解クラスの logit をどこまでも引き上げ続けます。これが自信過剰の正体です。

この自信過剰には具体的な実害が3つあります。

ひとつ、校正(calibration)が悪化します。校正とは「モデルが90%と言ったら本当に約9割当たる」という確率の正直さのことです。自信過剰なモデルは99%と言って外す、ということが頻発します。Guo et al. (ICML 2017) は、現代の深いネットワーク(深い ResNet など)が古い浅いネットワーク(LeNet 級)より精度は高いのに校正は著しく悪い(自信過剰)ことを実証し、この問題を広く知らしめました。彼らはバッチ正規化や深さ・幅の増加が校正を悪化させる主因だと指摘しています。

ふたつ、過学習しやすくなります。正解と他クラスの logit 差を無限に広げようとする過程で、学習データの細部やノイズまで暗記してしまいます。

みっつ、ラベルが間違っているときの罰が重すぎます。誤った正解ラベルに対してもモデルは確率1を目指すので、モデルが「これは怪しい」と正しく疑った場合でも巨大な損失(`-log(小さい確率)` は爆発的に大きい)を受け、ノイズに引きずられます。

## 仕組みを詳しく
平滑化の強さを ε(イプシロン、たとえば 0.1)、クラス数を K とします。one-hot の目標分布 q を次のように作り替えます。

```
正解クラスの目標確率   = 1 - ε
それ以外のクラスの目標 = ε / (K - 1)   (残りを均等に配る)
```

K = 5、ε = 0.1、正解が0番のときの before / after を見ます。

```
before (one-hot)        : [1.00, 0.000, 0.000, 0.000, 0.000]
after  (label smoothing): [0.90, 0.025, 0.025, 0.025, 0.025]
```

配り方には流儀が2つあります。上の「ε を非正解 K-1 個に配る」式のほか、「ε を全 K 個に均等に配って正解は `1 - ε + ε/K`」とする式もあります。多くのディープラーニングフレームワーク(たとえば PyTorch の `cross_entropy(..., label_smoothing=ε)`)は後者に近い実装ですが、効果は同じ「正解を少し下げ、他に少し配る」です。形式的には、目標分布を `q' = (1-ε)·q_onehot + ε·u` と、one-hot と一様分布 u(k)=1/K の混合として書けます。

この平滑化した目標 q' に対してクロスエントロピーを計算します。展開すると、

```
CE_smooth = - (1-ε) log(p_y) - (ε/(K-1)) Σ_{k≠y} log(p_k)
```

第1項は従来どおり正解確率を上げる項ですが、係数が `1-ε` に減っています。第2項が新しく、非正解クラスの確率がゼロに潰れすぎないよう引き戻す力です。この第2項のおかげで、正解クラスの確率を1まで押し上げる圧力が消え、目標の `1-ε`(=0.9)あたりで均衡します。logits が無限大に発散せず、有限の値に落ち着きます。

別の見方として、平滑化クロスエントロピーは「通常のクロスエントロピー + KL距離(予測 p と一様分布 u の食い違い)」の和に分解できます。後者は p を一様分布へ引き寄せる項、すなわち「予測のエントロピー(不確かさ)を大きく保て」という正則化に等しく、過信を罰する仕組みが明示的に見えます。

勾配の符号で見ると直感的です。正解クラスの確率が 0.99 まで上がっているモデルに対し、目標が 1.0(one-hot)ならまだ「もっと上げろ」という勾配がかかりますが、目標が 0.9(平滑化)なら「上げすぎ、少し下げろ」という逆向きの勾配がかかります。これが過信のブレーキです。

理論的にもう一段深く見ると、Müller et al. (2019) は最終層手前の特徴空間で何が起きるかを解析しました。one-hot 学習では、各クラスの特徴がそのクラスの方向へ無限に伸びていきます。一方ラベルスムージングを入れると、同じクラスのサンプルの特徴が「正解クラス中心から等距離の、引き締まったクラスタ」を形成するよう促されます。各クラスが互いに同じくらい離れた正単体(regular simplex)的な配置に整い、これが校正の改善と汎化の向上につながると説明されています。彼らは penultimate(最終手前)層の活性を可視化し、平滑化ありではクラスタが tight に締まる一方、クラス間の細かな類似構造が消えることを示しました。

実装上、平滑化されるのは学習時の目標分布だけです。推論時(実際に使うとき)のネットワーク構造や forward の計算は一切変わりません。学習が終われば、平滑化したことの恩恵(抑えられた logits、整った特徴空間)だけがモデルに残ります。

## 手法の系譜と主要論文
ラベルスムージングは単独の大論文というより、いくつかの研究が積み重なって理解が深まってきたテーマです。

起点は Szegedy, Vanhoucke, Ioffe, Shlens, Wojna による "Rethinking the Inception Architecture for Computer Vision"(CVPR 2016、Inception-v3 の論文、arXiv:1512.00567)です。ここでラベルスムージング正則化(LSR)が正式に提案されました。動機は明快で、one-hot 目標がモデルを自信過剰にし、最大の logit と他の logit の差を際限なく広げようとして汎化を損なうことを問題視しました。彼らは正解one-hot とクラス一様分布 u(k)=1/K を `ε` の割合で混ぜた目標を導入しました。

次に転機となったのが Müller, Kornblith, Hinton の "When Does Label Smoothing Help?"(NeurIPS 2019、arXiv:1906.02629)です。提案ではなく徹底した分析が主眼で、上で述べた「特徴空間でクラスタが引き締まる」可視化と、校正が改善するという定量的証拠を示しました。同時に重要な負の側面も発見しました。平滑化したモデルを教師に使う知識蒸留(大きいモデルから小さいモデルへ知識を移す手法。教師の出力確率を生徒が真似る)では、平滑化がクラス間の細かな類似情報(「3と8は似ている」のような情報)を潰してしまい、蒸留の効果が落ちるという副作用です。つまりラベルスムージングは「常に良い」わけではなく用途次第だと結論しました。

さらに Lukasik, Bhojanapalli, Menon, Kumar の "Does Label Smoothing Mitigate Label Noise?"(ICML 2020、arXiv:2003.02819)が、ノイジーなラベルへの効果を理論・実験の両面で検証しました。結論は「ラベルノイズがある状況ではラベルスムージングが有効で、ノイズに対するロバスト性を高め、ノイズ補正を行う他手法を補完する」というもので、後述するノイジー擬似ラベル環境での利用根拠になっています。彼らは平滑化が損失曲面を平らにし、誤ラベルが勾配に与える過大な影響を緩めることを示しました。

校正の文脈では前述の Guo et al. "On Calibration of Modern Neural Networks"(ICML 2017、arXiv:1706.04599)が背景として重要で、温度スケーリング(学習後に logits を1つのスカラ温度 T で割って確信度を割り引く後処理校正)など他の校正手法との比較対象にもなります。

## 論文の実験結果(定量データ)
具体的な数値を、指標の意味とともに見ます。

Inception-v3 の原論文(Szegedy et al. 2016)では、ImageNet という1000クラス・約128万枚の画像分類ベンチマークで評価しています。評価指標はトップ1誤り率(モデルの第1候補が正解と一致しない割合)とトップ5誤り率(上位5候補に正解が入らない割合)です。ラベルスムージング(ε=0.1)を加えることで、トップ1・トップ5ともにおよそ 0.2% の改善が報告されています。0.2% は小さく聞こえますが、ImageNet で当時の最先端を競う水準では1つの工夫で得られる改善として無視できず、しかもこの工夫はコストがほぼゼロという点が重要です。

Müller et al. (2019) の校正実験はより劇的です。校正の指標として ECE(Expected Calibration Error、期待校正誤差。予測を確信度ごとにビン分けし、各ビンで「平均確信度」と「実際の正解率」の差を取り、サンプル数で重み付け平均した値。小さいほど確信度が正直)を使います。ImageNet 上の ResNet などで、one-hot 学習のモデルは ECE がおよそ数% に達して自信過剰だったのに対し、ラベルスムージングを入れると ECE が温度スケーリングに匹敵する水準まで下がったと報告されています。つまり「学習中の一工夫だけで、後処理校正と同等の校正改善が得られる」ことを示しました。一方で蒸留のアブレーション(平滑化教師 vs one-hot 教師で生徒を訓練する比較)では、平滑化教師から訓練した生徒の精度が明確に劣化することを示し、平滑化が情報を潰す副作用を定量的に裏づけました。彼らはこれを「教師の logits 間の相互情報量が平滑化で減る」と説明しています。

機械翻訳(Transformer 系)でもラベルスムージング(ε=0.1)はほぼ標準装備で、興味深いことに「校正やperplexity(言語モデルの当てにくさ指標、低いほど良い)は一見悪化するのに BLEU(翻訳品質スコア、高いほど良い)は上がる」という、指標により効果の向きが分かれる現象が報告されています。これは平滑化が「確率の正直さ」と「最終タスク性能」をトレードオフしうることを示す好例です。

アブレーションの一般的傾向として、ε を 0(平滑化なし)→ 0.1 → 0.2 → 0.3 と上げていくと、0.05〜0.1 付近で校正・汎化が最も改善し、それを超えると今度は正解への正当な確信まで削られて精度が落ちていきます。だから定番は ε=0.1 前後で、「強くかけるほど良い」単調な関係ではない点がアブレーションの教訓です。

## メリット・トレードオフ・限界
メリット。過信を抑えて出力確率の校正(確信度の正直さ)が良くなり、安全重視の応用と相性が良い。過学習を抑える正則化として汎化が改善することが多い。ノイジーなラベルへの頑健性が上がる(Lukasik et al. 2020)。そして実装は引数1つ、計算コストはほぼゼロという圧倒的な手軽さがあります。

トレードオフと限界。第一に、強くかけすぎると正解への正当な確信まで削り精度が下がります(ε は 0.05〜0.1 が定番で、データやクラス数に応じて調整が要る)。第二に、知識蒸留の教師に使うとクラス間の類似情報が潰れ、蒸留効果が落ちます(Müller et al. 2019)。蒸留パイプラインでは教師は one-hot で訓練し生徒側で必要なら平滑化する、といった使い分けが要ります。第三に、本質的にラベルの「正しさ」自体を直すわけではなく、誤ラベルの影響を和らげるだけです。ノイズが系統的(あるクラスが常に別クラスと取り違えられる等)な場合は、平滑化だけでは不十分で、ノイズ遷移行列の推定など専用手法が要ります。第四に、前述の機械翻訳の例のように、最適化したい最終指標と校正指標が食い違う場合、どちらを優先するかの判断が必要です。

研究上の未解決課題としては、最適な ε をデータ・タスク・クラス数から自動決定する方法、クラスごとに ε を変える適応的平滑化、そして「いつ平滑化が情報を潰し、いつ汎化を助けるのか」を統一的に説明する理論が挙げられます。

## 発展トピック・研究の最前線
ラベルスムージングの一様分布を、より情報のある分布に置き換える発展が活発です。Online Label Smoothing(OLS、Zhang et al., TIP 2021)は、学習中のモデル自身の予測統計から目標分布を動的に作り、クラス間の本当の類似度(混同しやすいクラス対)を反映させます。これにより蒸留で情報が潰れる問題を緩和しつつ平滑化の利点を得ます。OLS は固定の一様分布より分類精度を一貫して上回ったと報告されています。

知識蒸留そのものがラベルスムージングの一般化と見なせる、という視点も重要です。蒸留では生徒が教師のソフトな確率分布を目標にしますが、これは「均一でない、データ依存の平滑化目標」を使う学習と解釈できます。両者の関係を統一的に論じる研究が続いています。

ノイジーなラベルが避けられない実応用、たとえば視覚言語モデル(VLM)やルールベースの自動ラベラが生成した擬似ラベルで分類器を訓練する設定では、ラベルスムージングが事実上の標準的な防御策になっています。擬似ラベルは一定割合で間違っているため、そこに確率1を目指して学習すると誤ラベルに過剰適合します。平滑化で目標を甘くすると、モデルが擬似ラベルと食い違ったときの罰が和らぎ、ノイズに引きずられにくくなります。同じ目的で [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) や [クラス重み付き損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0349_class-weighted-loss) と組み合わせ、クラス不均衡とラベルノイズの両方に同時対処する構成もよく使われます。

校正の最前線では、ラベルスムージングだけでなく、Mixup(複数画像とラベルを線形に混ぜるデータ拡張)、Focal Loss、温度スケーリングなどを組み合わせて、精度と校正を同時に最適化する研究が進んでいます。研究コミュニティでは「どの正則化が、どのデータ規模・モデル規模で校正と精度をどう両立させるか」を体系的に比較する流れがあります。
