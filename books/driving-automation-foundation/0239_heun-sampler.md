# Heun / 2次サンプラ (EDM)

## ひとことで言うと
[Euler ODE サンプラ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0235_euler-ode-sampler) の改良版で、1ステップのあいだに「進む前の速度」と「進んだ先の速度」の2回を測り、その平均で進むことで、より正確に数値積分する解法です。1ステップでネットを2回呼ぶ代わりに、1ステップあたりの誤差が大幅に小さくなり、同じ品質なら総計算量を減らせます。EDM(Elucidating the Design Space of Diffusion-Based Generative Models)という拡散モデルの設計フレームワークで、標準のサンプラとして採用されています。

## 直感的な理解
カーブした下り坂をスキーで滑る場面を想像してください。オイラー法は「スタート地点の傾きだけ」を見て、その向きにまっすぐ一定距離進みます。でも坂は途中で曲がっているので、出発点の傾きだけで進むと曲がりを追いきれず外側にずれます。ヘウン法(Heun's method)は賢く、まず出発点の傾きで仮に滑ってみて到達予定地点を求め、そこでの傾きも測り、出発点とその傾きを平均してから本番の一歩を進みます。区間の入口と出口の傾きを平均するので、途中で坂が曲がっていても、ずれを大きく打ち消せます。

代償は、傾きを2回測ること(=ネットを2回呼ぶこと)です。しかし1ステップの精度がぐっと上がるので、トータルで見れば測定回数を減らしても同じところに正確に着けます。これが「少ない計算で高品質」という生成モデルの目標に直結します。台形の面積を「両端の高さの平均×幅」で求めるのと同じ発想で、ヘウン法は台形公式(trapezoidal rule)を1ステップに当てはめた解法だと言えます。

## 基礎: 前提となる概念
生成モデルのサンプリングは ODE `dx/dt = v(x, t)` を数値積分することです(拡散の [Probability Flow ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0246_probability-flow-ode) や Flow Matching の速度場)。数値積分の誤差を理解するには「離散化誤差」と「次数」という言葉が要ります。

離散化誤差(discretization error)とは、ステップ幅 `Δt` を有限に刻むことで生じる、真の連続解とのずれです。オイラー法は「速度がそのステップのあいだ一定」と仮定するので、実際には速度が変化するぶんだけずれます。

次数(order)とは、ステップ幅を小さくしたとき誤差がどれくらい速く減るかを表す指標です。オイラーは1次で、1ステップあたりの局所誤差は `Δt^2`、N ステップ積んだ大域誤差は `Δt` のオーダー(ステップ数に反比例)です。ヘウンは2次で、局所誤差 `Δt^3`、大域誤差 `Δt^2` のオーダーです。`Δt` が小さい(ステップが多い)ほど、2次の `Δt^2` は1次の `Δt` より圧倒的に速くゼロに近づきます。たとえば `Δt = 0.1` なら、1次の誤差はオーダー 0.1、2次は 0.01 と、10倍の差がつく計算です。つまり同じステップ数なら2次のほうが正確、というのが数学的な根拠です。

生成の現場では、ネットの呼び出し回数(NFE, Number of Function Evaluations)が計算コストそのものです。NFE を減らしたいが品質は保ちたい、という綱引きの中で、「1ステップあたりの精度を上げて少ない NFE で済ませる」高次解法が求められます。その代表がヘウン法です。注意したいのは、2次精度は「ネットを2回呼ぶ」コストとセットだという点で、ヘウンは NFE を1ステップで2消費します。だから「ステップ数を半分にできても NFE は同じ」では意味がなく、「2次精度のおかげで NFE を半分以下に減らせる」ときに初めて得をします。

## 仕組みを詳しく
ヘウン法は「予測子・修正子(predictor-corrector)」の考え方で、1ステップを2段階で進めます。

1. 予測(オイラーの仮の一歩): まず今の速度 v1 で仮に進んだ先 `x_tilde` を求めます。
   ```
   v1 = v(x, t)
   x_tilde = x + Δt · v1
   ```
2. 修正(2点の速度を平均): 仮の到達点での速度 v2 も測り、出発点の速度と平均して、その平均速度で本番の一歩を進めます。
   ```
   v2 = v(x_tilde, t + Δt)
   x_next = x + Δt · (v1 + v2) / 2
   ```

直感的には、オイラーが「出発点の傾きだけ」、ヘウンは「出発点と到達予定点の傾きの平均」で進みます。区間内で速度が変化していても、平均をとることでずれを打ち消せます。なぜこれで2次精度になるのかは、台形公式が1次の(線形に変化する)誤差項を厳密に積分できるためで、残る誤差は速度の曲がり(2階微分)に由来する `Δt^3` まで小さくなります。

代償は1ステップにつきネットを2回(v1 と v2)呼ぶことで、同じステップ数なら計算量は約2倍です。しかし1ステップあたりの誤差が2次まで小さくなるので、トータルでは少ない NFE で同じ品質に届くことが多いのです。数値例で言えば、オイラーで 50 ステップ(NFE 50)要する品質に、ヘウンは 18 ステップ × 2 = NFE 36 程度で到達、といった具合です。

EDM フレームワークでは、ヘウンを単独で使うだけでなく、いくつかの実務的工夫を組み合わせます。

- ノイズスケジュールと前処理(プリコンディショニング): EDM は ODE の積分変数をノイズレベル σ で素直に取り、ネットへの入出力を σ に応じてスケーリングします(入力スケール・出力スケール・スキップ接続の重みを σ の関数で設計)。これにより速度場が滑らかになり、ヘウンの精度が出やすくなります。さらにステップを σ に対して非線形に配置します。具体的には `σ_i ∝ (σ_max^{1/ρ} + i/(N-1)·(σ_min^{1/ρ} - σ_max^{1/ρ}))^ρ` という ρ=7 のべき乗則で、高ノイズ側を細かく刻みます。
- 確率的サンプラ(stochastic sampler): 決定論的ヘウンに加え、各ステップで一度だけ少量のノイズを足し戻すオプションを提供します。具体的には、いったん σ を少し上げて(ノイズを加えて)から、ヘウンの一歩で σ を下げる、という churn(かき混ぜ)操作を入れます。これは決定論版が欠く「誤差の自己補正」を取り戻す工夫で、適度に使うと最終 FID をさらに改善できます。ただし足しすぎると逆に劣化します(EDM は注入量を S_churn, S_min, S_max, S_noise というハイパーパラメータで制御)。
- 最終ステップの省略: 終端では v2 の評価を省く(t + Δt が終点に達したとき)ことで、無駄なネット呼び出しを1回減らします。これにより NFE は「2 × ステップ数 − 1」になります。例えば 18 ステップなら NFE は 35 です。

## 手法の系譜と主要論文
- ヘウン法そのものは古典的な数値解析の手法(改良オイラー法、explicit trapezoidal method とも呼ばれる)で、教科書(Hairer, Nørsett & Wanner, "Solving Ordinary Differential Equations I")に載る基礎です。2次の Runge-Kutta 族の一種です。
- Song et al., "Score-Based Generative Modeling through Stochastic Differential Equations"(ICLR 2021, arXiv:2011.13456): ヘウンが積分する対象である [Probability Flow ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0246_probability-flow-ode) を定義した基盤。
- Karras, Aittala, Aila & Laine, "Elucidating the Design Space of Diffusion-Based Generative Models"(EDM, NeurIPS 2022, arXiv:2206.00364): ヘウン法を拡散サンプリングの標準ソルバとして採用・整理した論文。ノイズスケジュール・前処理・ソルバ・損失重みを独立した設計軸として整理し直し、ソルバには2次ヘウンを推奨。1次オイラーは離散化誤差が大きく少ステップで品質が落ちるため、追加のネット評価1回で誤差を大幅に減らせるヘウンが費用対効果に優れると示しました。
- Karras et al., "Analyzing and Improving the Training Dynamics of Diffusion Models"(EDM2, CVPR 2024, arXiv:2312.02696): EDM の学習側を改良し、重みの大きさや学習率を安定化する手法を導入。サンプラはヘウン系を踏襲しつつ、大規模・高解像度での品質を引き上げました。
- Lu et al., "DPM-Solver"(NeurIPS 2022, arXiv:2206.00927): 同じ「少ステップ高品質」を狙うが、汎用的な高次ソルバ(ヘウン)ではなく、拡散 ODE の半線形構造を専用に使う路線([DPM-Solver](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0232_dpm-solver))。EDM のヘウンは「素直で汎用的」、DPM-Solver は「構造特化で専用」という対比でよく語られます。
- Lipman et al., "Flow Matching for Generative Modeling"(ICLR 2023, arXiv:2210.02747): 直線に近いパスを学習すれば速度の変化が小さく、低次ソルバでも十分という主張で、ヘウンのような高次ソルバの必要性はパスの曲がり具合に依存することを示唆します。

## 論文の実験結果(定量データ)
評価指標は FID(Fréchet Inception Distance、生成画像と本物画像の特徴分布のずれ。小さいほど高品質)です。EDM(Karras et al., NeurIPS 2022)の代表的な成果は、CIFAR-10(32×32)で当時の最先端 FID を、わずか 35〜79 回程度のネットワーク評価(NFE)で達成したことです。具体的には、条件付き CIFAR-10 で FID 1.79 前後、無条件で FID 1.97 前後という、数値解析的に詰めたうえでのトップクラスの結果が報告されています。ImageNet 64×64 でも FID 1.36 前後と、少ない NFE で高品質を示しました。

アブレーションが本質を物語ります。EDM は同じ学習済みモデル・同じノイズスケジュールのもとで、ソルバだけを1次オイラーから2次ヘウンに替えると、同じ NFE での FID が大きく下がることを示しました。逆にオイラーで同じ FID に届くには、はるかに多い NFE が要ります。さらに、ノイズ離散化(ステップをどう配置するか)を σ に対して非線形に取る(ρ=7 のべき乗則)と、線形配置よりも少ない NFE で同等品質になることも示されています(ρ を 1 の線形配置にすると FID が悪化し、3〜10 あたりに最適点があり、7 が頑健と報告)。確率的サンプラについては、適量のノイズ注入が決定論版より FID を改善する一方、過剰だと劣化するという、明確な最適点の存在がアブレーションで確認されています。これらはすべて「PF-ODE をどう数値的に解くか」が最終品質を直接左右することの定量的な証拠です。EDM2(Karras et al., CVPR 2024)では、ImageNet 512×512 で FID 1.81 前後と、より大規模・高解像度でも同系統のサンプラが先端品質を出すことが示されました。

数値の意味を補足すると、NFE 35〜79 は、それまでの拡散モデルが要した数百〜千ステップに対して桁違いに少なく、しかも FID で先端を更新した点が重要です。つまり EDM は「ヘウンという素直な2次解法でも、設計を整えれば構造特化ソルバに迫る効率と最高品質を両立できる」ことを示し、その後のソルバ研究とモデル設計の共通の土台になりました。

## メリット・トレードオフ・限界
メリットは、第一に2次精度で離散化誤差が小さく、オイラーより少ない総 NFE で同等品質に届くこと。第二に、実装が比較的単純(予測子・修正子の2段階)で、DPM-Solver ほど複雑でなく、モデル構造に依存しない汎用性があること。第三に、EDM の確率的サンプラと組み合わせると品質をさらに引き上げられること。

トレードオフと限界は次の通りです。第一に、1ステップにつきネットを2回呼ぶため、ステップ数あたりの計算量はオイラーの約2倍です。レイテンシ制約が厳しい用途(自動運転やロボットのリアルタイム推論など)では、ステップ数を見かけ上は減らせても、1ステップが重くなる点に注意が要ります。NFE 換算で比較しないと判断を誤ります。第二に、パスがほぼ直線(Flow Matching の最適輸送パスなど)だと、そもそも速度の変化が小さいので高次化の恩恵が縮まり、オイラーとの差が小さくなります。直線パスでは「ネット2回」のコストが割に合わないこともあります。第三に、拡散 ODE の半線形構造を専用に使う DPM-Solver と比べると、極端な少ステップ領域(数 NFE)では同じ品質に対する NFE で劣る場合があります。

失敗例としては、確率的サンプラのノイズ量(S_churn)を大きくしすぎて、せっかくの2次精度の利得を打ち消してしまうケースがあります。また、ρ や σ の範囲をモデルの学習設定と合わせずに使うと、前処理の前提が崩れて品質が出ません。研究上の課題は、確率的サンプラのノイズ量を理論的に最適化すること、適応的にステップ次数を切り替えること、そして直線パスと曲がったパスのどちらにも効率よく効く統一的な高次解法を作ることです。

## 発展トピック・研究の最前線
ヘウンは「素直な高次解法」の代表ですが、少ステップ領域では構造特化ソルバ([DPM-Solver](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0232_dpm-solver)、UniPC)や、モデル自体を作り替える蒸留(Consistency Models)に押されつつあります。一方で EDM の貢献は単なるソルバ選択にとどまらず、「ノイズスケジュール・前処理・ソルバ・損失重みを独立に設計する」という方法論そのものにあり、後続の研究の標準的な枠組みになりました。EDM2(Karras et al., CVPR 2024)では学習の安定化と大規模化が進み、ImageNet 512×512 で FID 1.81 前後を達成しています。Flow Matching の普及により「直線化してオイラーで済ます」路線が主流化する中でも、曲がったパスを扱う場面ではヘウン系の2次解法が依然として有効で、両アプローチは用途に応じて使い分けられています。さらに、確率的サンプラの自己補正効果を理論的に裏づける研究(ODE と SDE の品質ギャップの解析)や、ステップごとに次数や churn 量を適応させる自動チューニングの研究が続いており、ヘウンを土台にした実用的な高速サンプリングは今も改良の対象です。
