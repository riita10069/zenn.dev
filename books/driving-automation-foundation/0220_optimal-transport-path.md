# Optimal Transport 確率パス (線形OTパス)

## ひとことで言うと
ノイズ(でたらめな値)から目標(作りたいデータ)へ「まっすぐ最短で移動する直線」を引き、その途中の点と進む向きを学習データとして使う考え方です。式で書くと u_t = (1-t)·x_0 + t·x_1、つまりノイズ x_0 と目標 x_1 を割合 t で混ぜた直線です。この直線パスは、生成モデルの学習を驚くほど単純で安定なものにします。フローマッチングという生成手法の中核であり、近年の画像生成・音声生成・ロボット行動生成で広く使われています。

## 直感的な理解
生成モデルとは「でたらめなノイズから、欲しいもの(画像や音声、ロボットの行動軌道など)を作り出す」仕組みです。出発点はノイズ、ゴールは目標データ。その間をどう移動させるかが問題になります。

ここで「移動の道筋(パス)」を考えます。道の取り方は無限にあります。ぐにゃぐにゃ曲がる道でも、まっすぐな道でもゴールには着けます。学習するモデルには「各時刻でその道のどちらへ進むべきか(向き、速度)」を予測させるので、道が複雑だと予測が難しく、学習が不安定で遅くなります。逆に道がまっすぐなら、向きは常に一定で、予測は格段に楽になります。

身近な例で言えば、目的地まで車で行くとき、曲がりくねった山道より直線の高速道路のほうが「いつどちらへハンドルを切るか」が単純です。生成モデルでも同じで、できるだけまっすぐな道を選ぶと学習が楽になる。その最も単純な「直線の道」が線形 OT パスです。

なぜ「最適輸送(Optimal Transport)」という名前が付くのかというと、後で詳しく述べますが、移動コストを「移動距離の 2 乗」で測ったとき、ある点を別の点へ運ぶ最も効率的な運び方は「直線で等速で運ぶ」ことになるからです。だからノイズと目標を直線で結ぶパスは、1 点を 1 点へ運ぶレベルでは最適輸送そのものです。

## 基礎: 前提となる概念
このトピックの背景には、拡散モデルから発展した「フローマッチング(Flow Matching)」という生成手法があります。順を追って前提を噛み砕きます。

連続正規化フロー(Continuous Normalizing Flow、CNF)とは、データの分布を「時間とともに少しずつ変形するベクトル場(各点に矢印が割り当てられた場、その点が次にどちらへ動くかを表す)」で記述する生成モデルです。ノイズの分布(平均 0・分散 1 のガウス分布など)から出発し、ベクトル場に従って点を動かしていくと、最終的に目標データの分布に変形される、という発想です。CNF は 2018 年頃(Neural ODE, Chen ら)に提案されましたが、当時は「変形の仕方をログ尤度の最大化で学習する」必要があり、その計算が重く(ODE を解いて尤度を逆伝播させる)、大規模化が難しいものでした。フローマッチングはこの「尤度を経由する重い学習」を回避し、もっと直接ベクトル場を回帰で学ぶ方法を与えた点が画期的です。

ベクトル場とは、空間の各点に「進むべき向きと速さ」を表す矢印を割り当てたものです。風の流れの地図(どこでどちらへ風が吹くか)をイメージすると近いです。点はこの矢印に従って移動します。数学的には、点の位置 x の時間変化を dx/dt = v(x, t) という常微分方程式(ODE、ある量の変化率を記述する方程式)で表します。v が速度ベクトル場です。

最適輸送(Optimal Transport、OT)とは、「ある分布(ここではノイズの分布)を別の分布(目標の分布)へ、移動コストが最小になるように運ぶにはどう運べばよいか」という数学の問題です。18 世紀の Monge が「土砂をある形から別の形へ最小労力で運ぶ」問題として定式化し、20 世紀に Kantorovich が「確率的な運び方」へ一般化しました。移動コストを「移動距離の 2 乗」で測るとき(これを 2-Wasserstein 距離と呼びます)、その最適な運び方は「各点を直線で、等速で運ぶ」ことになります。だから「ノイズと目標を直線で結ぶパス」を線形 OT パス(linear optimal transport path)と呼びます。

時刻 t は混ぜ具合を表すスカラー(0 から 1 の数)です。t=0 が完全にノイズ、t=1 が完全に目標です。これは時計の時刻というより「変形がどこまで進んだか」を表す進捗バーだと思うとよいです。

## 仕組みを詳しく
学習で作るデータは 3 つ組です。

```
u_t:             その時刻 t での「途中の状態」(ノイズと目標を混ぜた点)
t:               混ぜ具合(0 なら完全にノイズ、1 なら完全に目標)
target_velocity: その点で進むべき正解の向き(速度ベクトル)
```

線形 OT パスの定義式は次の通りです。

```
x_0 ~ N(0, I)                          # ノイズを標準正規分布からサンプル
t   ~ Uniform(0,1) などからサンプル      # 混ぜ具合
u_t = (1 - t)·x_0 + t·x_1               # 直線上の途中点(x_1 は目標)
target_velocity = x_1 - x_0             # 直線なので向きは t によらず一定
```

1 行ずつ意味を取ります。
1. x_0 を標準正規分布(平均 0・分散 1 のガウス)からランダムに作る。形は目標と同じ、たとえば軌道生成なら [B, trajectory_dim](B はバッチサイズ、trajectory_dim は軌道を平らに並べた次元)。画像なら [B, C, H, W]。
2. t を 0〜1 の間からサンプルする。t は各サンプルごとに 1 つの値で形は [B]。
3. u_t = (1-t)·x_0 + t·x_1 で、ノイズと目標を割合 t で混ぜた直線上の点を作る。実装では t([B])を u_t([B, trajectory_dim])にブロードキャスト(自動的に形を合わせて計算)できるよう次元を 1 つ足します(t を [B, 1] にする)。
4. target_velocity = x_1 - x_0。直線パスなので、t がいくつであっても進むべき向きは常にこの一定値です。

数値例で直感をつかみます。1 次元で x_0 = 0.0、x_1 = 4.0 とします。

```
t = 0.00 → u_t = 0.0   (出発点。完全にノイズ)
t = 0.25 → u_t = 1.0
t = 0.50 → u_t = 2.0   (ちょうど中間)
t = 0.75 → u_t = 3.0
t = 1.00 → u_t = 4.0   (到着点。完全に目標)
target_velocity = x_1 - x_0 = 4.0   ← どの t でも一定
```

学習では、速度ネット v_theta(theta は学習するパラメータ)に「点 u_t と時刻 t と条件 c(画像特徴や地図特徴などの文脈情報)」を入れ、出力が target_velocity に近づくよう MSE(平均二乗誤差、差を 2 乗した平均)で学習します。損失は次の形です。

```
L = E[ || v_theta(u_t, t, c) - (x_1 - x_0) ||^2 ]
```

パスが直線なので正解の向きが一定で、モデルは「各点でどちらへ進むか」を素直に学べます。これが拡散モデルのスコア回帰と比べて学習を安定させる理由です。

### なぜ「条件付き」で学んで全体が正しくなるのか
ここがフローマッチングの理論的な核心です。本当に学びたいのは「分布全体を運ぶベクトル場 u_t(x)」ですが、これは未知の積分を含み直接は計算できません。一方、データ点 x_1 を 1 つ固定したときの「条件付きベクトル場 u_t(x | x_1) = x_1 - x_0」は前述のとおり自明に書けます。Lipman らが示した定理は、条件付きベクトル場を回帰したときの最適解(損失を最小にする v_theta)が、目標である周辺ベクトル場と一致するというものです。直感的には「個々のサンプルに向かう矢印を平均すると、分布全体を運ぶ正しい矢印になる」ということです。だから自明な直線パスで学習データを作るだけで、正しい生成モデルが得られます。

### 時刻 t のサンプリングの工夫
t を一様分布から引くのが基本ですが、研究や実用では「どの t を重点的に学習させるか」を変えることがあります。たとえばノイズ寄り(t が小さい側)を多めにサンプルすると、向きの予測が難しい領域を重点的に学べます。ロボット行動生成の方策(pi-0 系など)は、Beta 分布を変形した分布から t をサンプルし、ノイズ寄りに重み付けします。大規模画像生成では「ロジット正規分布(logit-normal)」で中間の t を重点化する手法(Stable Diffusion 3 の論文で報告)が良い結果を出しています。重要なのは、パス自体は直線(線形 OT パス)のまま変えず、t の重み付けだけを変えている点です。

### 推論(実際に生成するとき)
学習が終わったら、新しいノイズ x_0 を引き、dx/dt = v_theta(x, t, c) という ODE を t=0 から t=1 まで数値的に解いて x_1(生成結果)を得ます。最も素朴な解法はオイラー法(小さな刻みで前進積分する)です。

```
x ← x_0
各ステップ k = 0..K-1:
    t_k = k / K
    x ← x + (1/K) · v_theta(x, t_k, c)
出力 x ≈ x_1
```

直線パスで学習してあると、この積分が少ない刻み数(数ステップから十数ステップ)でも精度が出やすくなります。これは「正解の道がまっすぐなので、大きな刻みで進んでも道を外れにくい」ためで、拡散モデルが数百〜千ステップを要したのに対する大きな速度上の利点です。

## 手法の系譜と主要論文
- Lipman, Chen, Ben-Hamu, Nickel, Le、「Flow Matching for Generative Modeling」(ICLR 2023、arXiv:2210.02747)。拡散モデルの学習を、確率パスに沿った「ベクトル場の回帰」として単純化するフローマッチングを提案しました。鍵となる洞察は前述の Conditional Flow Matching の定理です。さらに「Optimal Transport conditional probability path」として、ノイズと目標を直線で結ぶ条件付きパスを使うと、ベクトル場が一定になり学習がきわめて単純・安定になることを示しました。

- Liu, Gong, Liu、「Flow Straight and Fast: Learning to Generate and Transfer Data with Rectified Flow」(ICLR 2023、arXiv:2209.03003)。ほぼ同時期に、ノイズと目標を直線で結ぶ「rectified flow(整流フロー)」を独立に提案し、パスを直線化すると推論時の積分ステップを大幅に減らせることを示しました。さらに reflow という手続き(学習済みフローで作ったペアを使い再学習し、パスをさらにまっすぐにする)で、最終的に 1 ステップ生成に近づける道筋を示した点が新規です。Lipman の OT パスと Liu の rectified flow は本質的に同じ直線パスを別の動機から導いており、現在は「線形補間に基づくフロー」として統一的に理解されています。

- Albergo & Vanden-Eijnden、「Building Normalizing Flows with Stochastic Interpolants」(ICLR 2023)および「Stochastic Interpolants: A Unifying Framework for Flows and Diffusions」(2023)。直線に限らない一般の補間 I_t(x_0, x_1) を考える枠組みを与え、フローマッチングと拡散を 1 つの理論で扱えるようにしました。決定論的 ODE と確率的 SDE の間を連続的に行き来できる点が重要です。

- Tong ら、「Improving and Generalizing Flow-Based Generative Models with Minibatch Optimal Transport」(TMLR 2024、arXiv:2302.00482)。ノイズと目標を無作為にペアにするのではなく、ミニバッチ内で最適輸送に基づいて賢くペアリングする(輸送コストが小さい対応を選ぶ)ことで、直線同士の交差を減らし、ベクトル場をよりまっすぐにできることを示しました。これは線形 OT パスの「条件付きであって厳密な OT ではない」という弱点に対する一つの解です。

- 応用面では、Stable Diffusion 3(Esser ら, 2024, rectified flow を採用)など大規模画像生成や、ロボティクス・自動運転の行動生成にフローマッチングを用いる方策が増えています。「直線 OT パス + 重み付き時刻サンプリング」が実タスクで有効であることが報告されており、拡散ポリシーの後継として少ステップで滑らかな軌道を生成できる利点を活かしています。

## 論文の実験結果(定量データ)
- 画像生成(Lipman ら、ImageNet 64×64 / 128×128 など)。評価指標は FID(Frechet Inception Distance、生成画像の分布と本物画像の分布の近さを測る。値が小さいほど本物に近く高品質)です。フローマッチングを OT パスで学習すると、同じネットワークを拡散の手法(スコアマッチング)で学習した場合より FID が改善し、かつ少ないサンプリングステップで良い FID に到達することが示されました。特に「同じ計算予算(関数評価回数 NFE、ネットワークを何回通したか)で比べると、OT パスのほうが少ない NFE で低い FID に達する」点が重要で、これは直線パスが少ステップ生成に向くことの定量的な裏付けです。Lipman らは ImageNet 128 で拡散ベースの強いベースラインを上回る FID を報告しています。

- アブレーション(どの要素を抜くとどうなるか)。Lipman らは、同じフローマッチングの枠組みで「拡散型の曲がった確率パス(variance-preserving パス)」と「OT 型の直線パス」を比べました。OT パスのほうが、学習の収束が速く、少 NFE での生成品質が高いことを報告しています。つまり「パスを直線にする」という一点が、速度と品質の両面で効いていることがわかります。曲がったパスでは少 NFE で FID が急増するのに対し、直線パスは緩やかにしか劣化しません。

- 整流フロー(Liu ら、CIFAR-10 で FID を報告)。reflow を繰り返すほど推論ステップを減らせ、最終的に 1 ステップ生成でも実用的な品質(reflow + 蒸留で CIFAR-10 の 1 ステップ FID が一桁台前半)に到達できることを示しました。指標 FID の値が、ステップ数を減らしても急激には悪化しないことが、直線化の効果として定量的に示されています。

- ミニバッチ OT(Tong ら)。無作為ペアリングとミニバッチ OT ペアリングを比べると、後者のほうが学習の勾配分散(ばらつき)が小さく、同じステップ数での生成品質(2-Wasserstein 距離や FID)が向上することが報告されています。これは「ペアの選び方」がパスのまっすぐさに効くことの裏付けです。低次元の合成データ実験では、ミニバッチ OT のほうが少ステップでの誤差が明確に小さくなります。

- Stable Diffusion 3(Esser ら, 2024)は、複数の確率パスと時刻重み付けを大規模に比較し、rectified flow パス + logit-normal 時刻サンプリングが、従来の拡散(EDM など)より一貫して良い FID と人間評価を得ることを報告しました。これは線形パスが大規模・高解像度でも有効であることの強い実証です。

## メリット・トレードオフ・限界
メリット
- パスが直線なので、各点で進むべき向きが一定(x_1 - x_0)になり、学習が単純で安定。
- 推論時の ODE 積分が少ない刻み数でも精度が出やすく、生成が速い。
- 実装が短く明快(数行で学習データを作れる)。バグが入りにくい。
- 拡散と同じネットワーク構造(U-Net や DiT)をそのまま使え、移行コストが低い。

トレードオフと限界
- これは「サンプルごとにノイズと目標を結ぶ」条件付き OT であって、分布全体の厳密な最適輸送ではない。理論上の最短輸送そのものを解いているわけではありません。無作為ペアでは、平均化したベクトル場(周辺場)は一般に直線ではなくなります。
- ランダムに引いたノイズ x_0 と目標 x_1 を無作為にペアにすると、直線同士が交差し、平均的なベクトル場がやや曲がることがある。これがあると少ステップ推論の精度が下がる。reflow やミニバッチ OT ペアリングはこの問題への対策です。
- 多峰性(取りうる正解が複数に分かれる状況、たとえば障害物を右に避けるか左に避けるか)では、無条件に直線パスを学ぶと平均的な解(両者の中間、つまり障害物に突っ込む軌道)に寄りやすい場合があり、条件付けや時刻サンプリングの工夫が要ります。
- 純粋な決定論的 ODE 生成はモード被覆(多様性)で確率的サンプリングに劣ることがあり、stochastic interpolants のように少量のノイズを戻す折衷が使われることもあります。
- 未解決の課題として、厳密 OT を高次元で効率よく解くこと、交差のない理想的なパスを学習段階で保証することなどは、なお研究途上です。

## 発展トピック・研究の最前線
- ストキャスティック補間(stochastic interpolants、Albergo & Vanden-Eijnden)は、フローマッチングと拡散を統一的に扱う枠組みで、決定論的 ODE と確率的 SDE の間を連続的に行き来できます。
- 潜在空間でのフローマッチング(画像を VAE などで圧縮した潜在表現上でフローを学ぶ)は、高解像度生成の計算を減らす標準的な工夫で、最近の大規模生成モデルで広く使われています。
- 一貫性モデルや蒸留と直線パスの融合。直線パスは軌跡が単純なので蒸留(多ステップを 1〜数ステップに圧縮)と相性がよく、1〜2 ステップ生成の研究で中心的役割を果たします。
- ロボット行動・自動運転軌道の生成にフローマッチングを使う流れは活発で、拡散ポリシーよりも少ステップで滑らかに行動を出せる利点が注目されています。条件(センサ特徴や地図)へのクロスアテンションや、DiT 流の条件注入(AdaLN)と組み合わせる構成が一般的です。
