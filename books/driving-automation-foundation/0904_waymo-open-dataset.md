# Waymo Open Dataset

## ひとことで言うと

Waymo Open Dataset (WOD) は、Google 系列の自動運転企業 Waymo が公開した大規模な運転データセットです。複数の LiDAR (レーザーで周囲の 3D 形状を測るセンサ) と複数のカメラの時刻同期されたデータを大量に含み、各物体に正確な 3D バウンディングボックス (3 次元の箱) のラベルが付いています。3D 物体検出 (周囲の車や人を 3 次元の箱で囲んで位置・大きさ・向きを当てる) と 3D 物体追跡 (同じ物体に時間をまたいで一貫した番号を付ける) の研究で、事実上の標準ベンチマークになっています。論文タイトルが示すとおり、知覚アルゴリズムを「データ規模に対してスケールさせる (scale)」ことを実証的に研究するために作られました。

## 直感的な理解

自動運転車が安全に走るには、まず「周りに何があるか」を正確に把握しなければなりません。前の車との距離、横断歩道の歩行者、脇から出てくる自転車 — これらを 3 次元空間の中で正しい位置・大きさ・向きで認識する必要があります。これを機械に学ばせるには、大量の「実際のセンサ記録」と「そこに何がどこにあったかの正解ラベル」が要ります。

なぜ「大規模」であることがそんなに大事なのでしょうか。深層学習は、データが増えるほど性能が上がるという経験則 (スケール則) に従います。小さなデータセットでは、まれな状況 — 夜の雨、工事現場、急な割り込み、変わった形の車両 — を学習できず、いざ現実でそういう場面に出会うと誤認識します。Waymo Open Dataset は「データを大きくしたとき知覚はどこまで良くなるか」を測れるよう、当時としては桁違いに大きく、天候・時間帯・地域の多様性も高いデータを公開しました。これにより研究者は、アルゴリズムだけでなくデータ量という軸でも知覚を議論できるようになりました。

## 基礎: 前提となる概念

LiDAR (Light Detection and Ranging): レーザー光を周囲に照射し、跳ね返って戻るまでの時間から距離を測るセンサです。これを高速に回転・走査することで、周囲を数十万個の点として捉えた「点群 (point cloud)」が得られます。各点は (x, y, z, intensity) を持ち、intensity は反射の強さ (物体の材質や向きで変わる) です。カメラと違い、LiDAR は距離を直接・正確に測れるのが強みで、3D 検出の主役センサです。WOD の LiDAR は距離だけでなく 2 回目の反射 (second return、葉や雨を貫いて奥にも当たった光) も記録します。

3D バウンディングボックス: 物体を囲む 3 次元の直方体です。中心位置 (x, y, z)、寸法 (length, width, height)、そして水平面内の向き (ヨー角) の合計 7 つのパラメータで表します。例えば「中心が自車の前方 15 m・右 2 m・高さ 0.8 m、長さ 4.5 m・幅 1.8 m・高さ 1.5 m、向き 5 度」のように、1 台の車を 7 個の数値で記述します。

キャリブレーション: 各センサ (LiDAR・カメラ) が車のどこに、どの向きで付いていて、カメラなら焦点距離がいくつか、といった幾何情報です。これが正確に分かっていると、LiDAR の 3D 点をカメラ画像の 2D ピクセルに投影でき、点群と画像を対応づけて融合できます。

IoU (Intersection over Union): 予測した箱と正解の箱がどれだけ重なっているかを「重なり領域 ÷ 和集合領域」で測る指標です。検出が「当たり」かどうかを判定する閾値に使います (例: 3D IoU が 0.7 以上なら当たり)。

mAP と APH: 検出性能の指標です。AP (Average Precision、平均適合率) は precision-recall 曲線の下の面積で、「どれだけ正しく (precision)、かつ漏れなく (recall) 検出できたか」を 0〜1 で表します。3D mAP はこれを全カテゴリで平均したもの。Waymo は独自に APH (AP weighted by Heading、向きで重み付けした AP) を導入しました。位置と大きさが合っていても向きを間違えると危険なので、各検出の真偽を判定するときに向きの誤差 (正解ヨーとの角度差) を係数として掛けます。例えば「正しく検出したが向きが 180 度逆」だと AP では加点されても APH ではほとんど加点されません。これにより「向きまで正しく当てる」ことが評価対象になります。

## 仕組みを詳しく

Waymo Open Dataset (Perception 部分) のおおまかな構成を見ていきます。

データの規模と構造:
- セグメント (シーン) 数は約 1,150 (学習 798 / 検証 202 / テスト 150 程度の分割) で、各セグメントは約 20 秒の連続走行です。10 Hz で記録されるので 1 セグメントは約 200 フレームになります。フレーム総数は約 23 万、3D ボックスのラベルは合計で約 1,200 万個に達します。
- センサは LiDAR が 5 台 (天頂に中距離の主 LiDAR 1 台 + 車体周りに短距離 4 台) と、カメラが 5 台 (前方および側方を覆う、各 1920×1280 程度の高解像度) です。すべて時刻同期され、キャリブレーション情報が付きます。
- ラベルは車両・歩行者・自転車・標識などの 3D バウンディングボックスと、フレームをまたぐ追跡 ID です。2D 画像側のボックスも提供されます。
- 自車の軌跡 (ego trajectory) も得られるため、運転計画タスクの教師信号にも転用できます。

1 フレームの中身を具体的に書くと次のようになります。

```
frame = {
  LiDAR点群 : (x, y, z, intensity, ...) が数十万点 (5台のLiDARを統合)
  カメラ画像: 5枚 (高解像度RGB)
  ラベル    : 各物体の 3D box {中心 x,y,z, 寸法 l,w,h, ヨー角, 種別, 追跡ID, 速度}
  キャリブレーション: 各センサの内部/外部パラメータ
  ego pose  : 地図座標系での自車位置・姿勢 (4x4 変換行列)
}
```

3D 検出モデルの典型的な処理は、この点群を入力として 3D ボックスの集合を出力します。LiDAR ベースの代表手法 PointPillars (後述) なら、点群を真上から見て格子状の「柱 (pillar)」に区切り、各柱内の点を小さなニューラルネット (PointNet) でまとめて疑似画像 (BEV 特徴マップ) を作り、その上で通常の 2D CNN と検出ヘッドを走らせます。つまり「3D の点群を 2D の俯瞰画像に変換してから検出する」のが要点で、これにより 3D データを GPU で高速に処理できます。

評価では、検出物体が遠いほど点群が疎で難しいため、Waymo は距離帯ごと (0–30m, 30–50m, 50m+) ・難易度ごとに分けて mAP / APH を報告します。難易度は LEVEL_1 (ラベル点数が多く明瞭なもの) と LEVEL_2 (遠方や遮蔽で点が少ない難しいものを含む。LEVEL_1 を包含する厳しい設定) の 2 段階です。これにより「近距離は得意だが遠距離が弱い」といった手法の癖が見えるようになっています。LEVEL_2 のスコアが公式ランキングの主指標として使われます。

後年には Motion Dataset (軌道予測用、地図と他車の動きを含む) という兄弟データセットも公開されましたが、これは別ノート [Waymo Open Motion Dataset (WOMD)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0905_waymo-open-motion) で扱います。本ノートは知覚 (Perception) 版に焦点を当てます。

## 手法の系譜と主要論文

知覚データセットの系譜は次のように流れます。

KITTI (Geiger, Lenz, Urtasun, CVPR 2012) が 3D 検出ベンチマークの草分けでした。ステレオカメラ + Velodyne LiDAR + GPS/IMU を積み、ドイツの市街地走行を記録しました。キャリブレーションが整い長く標準でしたが、検出用フレームは約 7,481 枚と小規模で、天候は主に晴天・昼間、地域も一都市に限られました。

nuScenes (Caesar et al., CVPR 2020) は 6 カメラで全周囲をカバーし、LiDAR 1 台に加えて 5 台のレーダーと HD マップを持つ点が新しさでした。詳しくは [nuScenes](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0891_nuscenes) を参照してください。

Waymo Open Dataset (Sun, Kretzschmar, Dotiwalla et al., CVPR 2020, "Scalability in Perception for Autonomous Driving: Waymo Open Dataset") は、これらに対し「規模」と「マルチ LiDAR」「高頻度・高品質ラベル」で差別化しました。KITTI の後継として規模と多様性を大幅に拡張し (KITTI の検出フレーム数の桁を大きく超える)、複数 LiDAR・全方位・多天候・多時間帯をカバーします。論文の主眼は「データ規模を増やすと検出性能がどう伸びるか」のスケール分析であり、学習データ量を段階的に増やしたときの mAP の伸びを実験的に示しました。

Waymo Open Dataset 上でよく検証される手法の系譜:
- VoxelNet (Zhou & Tuzel, CVPR 2018): 点群を 3D ボクセル (格子セル) に区切り、各ボクセルを学習可能な特徴に変換 (Voxel Feature Encoding) して 3D 畳み込みで検出。end-to-end 学習の 3D 検出の先駆け。
- PointPillars (Lang, Vora, Caesar et al., CVPR 2019): 点群を Z 方向に区切らず「柱状 (pillar)」に区切って疑似 BEV 画像に変換し、2D CNN で高速に検出する。3D 畳み込みを使わないためリアルタイム性が高い。
- PV-RCNN (Shi et al., CVPR 2020): ボクセル特徴と点単位特徴を組み合わせ (point-voxel fusion)、二段階で精度を上げる。
- CenterPoint (Yin, Zhou, Krähenbühl, CVPR 2021): 物体を「点 (中心)」として検出する、anchor-free (事前に箱の候補を置かない) な手法。BEV 特徴マップ上で中心ヒートマップを予測し、そこから箱を回帰する。Waymo・nuScenes の両方で長くトップ級でした。
- カメラから BEV を作る系譜として BEVFormer (Li et al., ECCV 2022) などがあり、これらも Waymo/nuScenes で性能を報告するのが定番です。

## 論文の実験結果 (定量データ)

Waymo Open Dataset の元論文 (CVPR 2020) は、スケール分析を主要な貢献の 1 つとしました。学習データを段階的 (例えば全体の 10 %・30 %・50 %・100 %) に増やしていくと、車両・歩行者検出の AP が一貫して向上することを示し、「知覚はデータ規模で確かに伸びる、しかも飽和の兆しはまだ見えない」という結論を提示しました。これは「アルゴリズムだけでなくデータ投資にも意味がある」というメッセージで、その後の大規模データ収集の流れを後押ししました。論文ではまた、KITTI で学習したモデルを Waymo に当てると性能が大きく落ち、その逆も成り立つ — つまりデータセット間にドメインギャップがある — ことも示し、規模と多様性の重要性を裏づけました。

ベースライン手法の報告例として、論文では PointPillars を Waymo 上で評価し、車両検出で LEVEL_1 の 3D AP がおよそ 60 %台、歩行者ではそれより低い値が報告されました (バージョンや評価設定で差があります)。歩行者は車より小さく点群が疎なため検出が難しく、車両より低い AP になるのが一般的な傾向です。これは「指標の数値そのもの」より「カテゴリ間・距離帯間でどれだけ差が出るか」を読むのが大切で、遠距離・小物体の難しさが定量的に表れます。50m を超える距離帯では AP が近距離より大きく下がるのが典型で、点群の疎さが直接効いていることが分かります。

その後の手法の伸びも顕著です。CenterPoint (CVPR 2021) は Waymo のリーダーボードで、PointPillars 系のベースラインを車両・歩行者・自転車のすべてで明確に上回り、当時の最高水準を更新しました。アブレーションでは、anchor ベース (事前に多数の箱候補を置く方式) を center ベース (中心点を直接予測) に置き換えるだけで mAPH が数ポイント向上することが示され、「物体を点として表現する設計が回転物体の検出に有利」という知見を定量的に裏づけました。回転した物体に対し軸並行の anchor を正しく合わせるのは難しく、中心点予測なら向きを別途回帰するだけで済むため、この問題を回避できるのです。

マルチモーダル融合の効果も Waymo で繰り返し検証されてきました。カメラと LiDAR を融合した手法は、LiDAR 単独に比べて特に遠距離・小物体・夜間での検出が改善することが報告されています。カメラは色・テクスチャ情報を、LiDAR は正確な距離を提供するため、互いの弱点を補える、というのが定性的な説明であり、それが AP の数値差として現れます。

## メリット・トレードオフ・限界

メリット:
- 大規模・高品質ラベルで、3D 検出・追跡の知覚学習に最適。データ規模によるスケール分析が可能。
- マルチ LiDAR + マルチカメラ + キャリブレーションが揃い、マルチモーダル/マルチビュー融合の検証に向く。
- 自車軌跡 (ego trajectory) もあり、運転計画タスクへの転用余地がある。
- 距離帯・難易度 (LEVEL_1/2) 別の評価と公開リーダーボードがあり、手法の癖まで含めて公平に比較できる。

トレードオフ・限界:
- データ量が膨大で、ストレージ・計算・ダウンロードのコストが高い。1 セグメント単位でもサイズが大きく、全体では数 TB に及ぶ。
- 利用にライセンス (利用規約) への同意が必要で、商用利用には制約がある。
- 収集地域が Waymo の運行都市 (米国の一部) に偏るため、他地域・他交通文化への汎化は別途検証が必要。右側通行前提のデータが中心。
- LiDAR ラベルは高品質だが、遠距離では点群が疎になり小物体の真値そのものが難しくなる。
- APH/mAPH という独自指標は、他データセット (nuScenes の NDS など) と直接比較できないため、データセット横断の議論には注意が要る。

未解決の研究課題: 学習データの地域に過適合せず未知の地域・天候へ汎化する手法、ラベル付けコストを下げる半教師あり・自己教師あり学習、検出だけでなく追跡・予測・計画まで一貫して学ぶエンドツーエンド化、そしてセンサ構成 (LiDAR の有無・本数) が変わっても動くモデルの設計などが活発に研究されています。

## 発展トピック・研究の最前線

近年は「LiDAR を使わずカメラだけで 3D 検出する」カメラオンリー BEV 知覚が大きな流れです。LSS (Lift-Splat-Shoot, Philion & Fidler, ECCV 2020) や BEVFormer (Li et al., ECCV 2022) が代表で、コストの高い LiDAR を減らせる可能性から注目されています。Waymo Open Dataset はこれらカメラ系手法と LiDAR 系手法を同じ土俵で比べる場としても使われ、「カメラだけでどこまで LiDAR に迫れるか」が定量的に追跡されています (現状はまだ LiDAR が AP で優位)。

もう 1 つの最前線は、3D 検出を単独タスクとしてではなく、追跡・予測・運転計画と統合してエンドツーエンドに学習する流れです。知覚で得た物体表現を直接予測・計画に渡すことで、モジュール間で情報が落ちるのを防ごうという発想です。Waymo は知覚版と Motion 版の両方を提供しているため、知覚から予測までを横断する研究の基盤として重宝されています。さらに、占有予測 (Occupancy Flow、空間を格子に区切り各マスが将来どう埋まり動くかを予測) のチャレンジも開かれ、検出という離散的な箱に依存しない安全表現の研究が進んでいます。
