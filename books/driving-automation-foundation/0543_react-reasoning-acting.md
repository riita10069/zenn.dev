# ReAct (Reasoning + Acting)

## ひとことで言うと

言語モデルに「頭の中で考えること(Reasoning、推論)」と「外の世界に対して行動すること(Acting、たとえば検索ツールを呼ぶ)」を交互に繰り返させる進め方です。考えるだけでは知らない事実を補えず、行動するだけでは何のために動いているか見失います。両者を交互に行うことで、モデルは「考えて → 調べて → その結果を踏まえてまた考えて → また調べる」というループを回し、最新情報を取り込みながら賢く問題を解けます。今日の AI エージェント(自分で道具を使って多段階の仕事をこなす AI)の基礎になった考え方で、Princeton と Google の研究者が 2022 年に提案しました。

## 直感的な理解

人が見知らぬ街でレストランを探す場面を考えます。頭の中だけで「たぶんこの辺だろう」と推測し続けても、実際の地理を知らなければ当てずっぽうです。逆に、何も考えずに足の向くまま歩き回るだけでは、目的を見失ってさまよいます。賢い人は「まず駅前で地図を見よう(行動)→ 北口に飲食店街がありそうだ(観測を踏まえた推論)→ そちらへ歩く(行動)→ 看板で確認する(観測)」というように、考えることと調べることを往復します。

言語モデルにも同じ問題があります。学習時に覚えた知識だけで完結させると、最新情報を知らず、覚え違いをそのまま事実として書いてしまいます。一方で行動だけさせると目的を見失います。ReAct(Reason ＝ 推論 ＋ Act ＝ 行動 を合わせた造語)は、この往復をモデルにやらせる枠組みです。重要なのは、これがモデルの再学習をまったく必要とせず、プロンプトに数個のお手本を示すだけで成立する点です。

## 基礎: 前提となる概念

まず Chain-of-Thought(CoT、思考の連鎖)を押さえます。これは言語モデルに、答えだけでなく答えに至る途中の考えを順に書かせてから答えさせる手法で、推論の正答率を上げます。ただし CoT には根本的な弱点があります。モデルの頭の中(学習時に覚えた知識、これを parametric knowledge ＝ パラメータに埋め込まれた知識と呼びます)だけで完結しているため、(1) 学習後に起きた最新の出来事を知らない、(2) 覚え違いをそのまま事実のように書いてしまう、という問題が起きます。この (2) を「ハルシネーション(hallucination、もっともらしい嘘の生成)」と呼びます。長い推論鎖の途中で1つ事実を間違えると、その先が全部崩れます。これを「誤り伝播(error propagation)」と言います。

次に「ツール利用(tool use)」と「外部環境(environment)」です。ツールとは、モデルの外にある機能で、たとえば検索エンジンの API、計算機、データベース照会などです。モデルが `Search[アポロ計画]` のような行動を出力すると、外部システムがそれを実行し、結果(検索文章など)をモデルに返します。環境とは、モデルが行動を起こす対象の世界で、Web サイト、テキストゲーム、シミュレータなどを指します。専門的には、これは強化学習でいう「エージェントが行動 a を取り、環境が観測 o を返す」という相互作用ループと同じ構造です。

当時すでに「言語モデルに行動を出力させて環境を操作する」研究はありました。SayCan(ロボット行動の選択)、WebGPT(検索しながら回答)、WebShop(オンライン買い物環境、Yao ら自身が NeurIPS 2022 で提案)などです。しかしこれらの多くは行動だけで高レベルの推論や計画がないため、「今なぜこの行動をしているか」「次に何をすべきか」を見失い、無駄な行動を繰り返したり迷子になったりしました。逆に CoT 系は推論はできても外界に手を出せませんでした。ReAct はこの2つの系統(内部推論と外部行動)を1つのループに結合します。

## 仕組みを詳しく

ReAct では、モデルの出力を3種類のステップが交互に並ぶ形にします。これを「拡張された行動空間(augmented action space)」と呼び、通常の行動に加えて「考える」という言語行動を許す、という捉え方をします。

- Thought(思考): 今の状況をどう理解し、次に何をすべきかを考える文章。これは環境を変えず、次の行動を導くための内的な計画。
- Action(行動): 外部ツールへの呼び出し。例: `Search[アポロ計画 司令船]`、`Lookup[酸素]`、`Finish[答え]` など。
- Observation(観測): 行動の結果として外部から返ってきた情報。例: 検索結果のテキスト。

これらが `Thought → Action → Observation → Thought → Action → ...` と繰り返され、最後に `Finish[...]` で答えを出します。具体例(Wikipedia 検索ツールを使う質問応答)を見ます。

```
質問: 「コロラド造山運動が及んだ東側の地域の標高はどのくらい?」

Thought 1: コロラド造山運動を調べ、東側の地域を特定し、その標高を調べればよい。
           まず造山運動を検索する。
Action 1:  Search[コロラド造山運動]
Observation 1: コロラド造山運動はコロラド及び周辺の山形成の出来事。東側は
               ハイプレーンズに及ぶ…

Thought 2: 東側はハイプレーンズらしい。ハイプレーンズの標高を調べる。
Action 2:  Search[ハイプレーンズ]
Observation 2: ハイプレーンズは標高およそ 1,800〜2,100m の高原…

Thought 3: 標高は約 1,800〜2,100m と分かった。これが答え。
Action 3:  Finish[約 1,800〜2,100 メートル]
```

ここで重要なのは、Observation(外部から得た事実)が次の Thought の材料になることです。CoT なら「ハイプレーンズの標高」をモデルが記憶頼みで書くしかなく、間違えれば終わりでした。ReAct は実際に検索して事実を取り込むので、ハルシネーションを抑えられます。逆に、Thought があることで「次に何を検索すべきか」が計画的になり、行動が迷走しません。Thought は環境を変えない自由な言語ステップなので、サブゴールへの分解、常識の注入、観測の要約、現在の進捗追跡など、何にでも使えます。

ReAct が解決する失敗パターンを対比すると分かりやすいです。

- CoT だけ(行動なし): 推論は筋が通っているが、途中の事実が誤りで結論も誤る(ハルシネーション)。
- Act だけ(推論なし): 行動はするが何を探しているか見失い、無関係な行動を繰り返す。エラーから抜け出せない。
- ReAct(両方): 推論で行動を導き、行動の結果で推論を正す。相互補完。

実装上は、これらをすべて1つのプロンプトに少数のお手本(few-shot、数個の解答軌跡)として示すだけで成立します。モデルの再学習は不要で、お手本の形式に倣ってモデルが Thought/Action/Observation を生成します。Action のフォーマット(どのツールをどう呼ぶか)と、行動空間(取りうる行動の集合)の設計が肝になります。今日の関数呼び出し(function calling)や JSON 形式のツール指定は、この Action の形式化を進めたものです。

## 手法の系譜と主要論文

起点は Chain-of-Thought Prompting (Wei et al., Google, NeurIPS 2022, arXiv:2201.11903) で、内部推論を引き出す手法です。同じ頃 Self-Ask (Press et al., 2022, arXiv:2210.03350) が「自分に問いを立てて、必要なら検索で答える」という、推論と検索を絡める前駆的な発想を示していました。ReAct: Synergizing Reasoning and Acting in Language Models (Yao, Zhao, Yu, Du, Shafran, Narasimhan, Cao, Princeton & Google, ICLR 2023, arXiv:2210.03629) は、推論と行動を別々のものとして扱う従来の常識を覆し、両者を交互生成する単一のループにまとめました。新規性は、推論と行動の相乗効果(synergy)を一般的なパラダイムとして定式化した点です。動機は、推論だけだとハルシネーション、行動だけだと計画喪失という、両極の弱点を相互に補うことでした。

同時期に Toolformer (Schick et al., Meta, NeurIPS 2023, arXiv:2302.04761) が「モデル自身がいつどのツールを呼ぶべきかを自己教師あり学習で身につける」方向を示し、ReAct のプロンプトベースとは別アプローチ(学習ベース)でツール利用を実現しました。ReAct の直後には Reflexion: Language Agents with Verbal Reinforcement Learning (Shinn et al., NeurIPS 2023, arXiv:2303.11366) が、行動の失敗を言語で振り返って次の試行に活かす「自己反省(self-reflection)」を加え、ReAct ループに学習的なフィードバックを組み込みました。これは重みを更新せず、反省文をメモリに蓄えるという「言語的強化学習」です。さらに同じ Yao らの Tree of Thoughts (NeurIPS 2023, 参照: [Tree of Thoughts (ToT)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0547_tree-of-thoughts)) は、外部行動ではなく内部での木探索による熟慮を扱う、ReAct と対になる系統です。LATS(Language Agent Tree Search, ICML 2024)は両者を MCTS で束ねました。

ReAct は後の LLM エージェント/ツール利用の直接の源流とされ、各種エージェントフレームワーク(LangChain の Agent、AutoGPT 系など)の Agent ループ、関数呼び出し(function calling)、自律エージェントの設計の基礎パラダイムになりました。現在の「ツールを使うエージェント」の多くは、本質的に ReAct ループ(観測→推論→行動の反復)の変種です。

## 論文の実験結果(定量データ)

ReAct 論文は、知識集約型タスクと対話的意思決定タスクの両方で、主に PaLM-540B(当時の Google の大規模モデル)を使って評価しています。

知識集約型では HotpotQA(複数の文書をまたいで答える多段推論の質問応答)と FEVER(主張が事実か虚偽かを検証するタスク)を使いました。HotpotQA の指標は Exact Match(EM、答えの文字列が正解と完全一致した割合)、FEVER の指標は分類正解率(accuracy、SUPPORTS / REFUTES / NOT ENOUGH INFO の判定が当たった割合)です。EM は厳しい指標で、表現が少し違うだけで不正解になるため、本当に正しい答えを出せたかを測れます。結果として、ReAct は Act 単体を明確に上回り、純粋な CoT との比較では一長一短ながら、CoT と ReAct を状況に応じて切り替える併用(CoT-SC → ReAct と ReAct → CoT-SC、片方が確信を持てないときもう片方に切り替える)が最も高い性能を示しました。CoT は内部知識で滑らかに推論できる問題に強く、ReAct は事実の裏取りが必要な問題に強い、という相補関係が数値で確認されています。論文の誤り分析では、CoT の誤りの多くがハルシネーション由来であるのに対し、ReAct はそれを大きく減らす一方、検索結果に引きずられて誤る(検索が役に立たない情報を返すと推論が崩れる)別の失敗様式が現れる、と分析されています。

対話的意思決定では ALFWorld(家事タスクを模したテキスト環境、例「電子レンジを開けてマグカップを入れる」)と WebShop(オンライン買い物環境、指示された商品を探して購入)を使いました。指標は成功率(タスクを達成できたエピソードの割合)です。ALFWorld では、ReAct が模倣学習(人間の操作を真似る学習)や強化学習で何千エピソードも訓練したベースライン(BUTLER など)を、わずか1〜2個のお手本だけで大幅に上回り、報告では成功率がベースラインに対し数十ポイント高い水準(およそ70%前後)に達しました。WebShop でも、模倣学習・模倣+強化学習ベースラインを上回りました。重要なのは、これらをモデルの再学習なし・少数のお手本だけで達成した点です。推論(Thought)を挟む効果のアブレーションとして、Thought を抜いて Act だけにすると ALFWorld の成功率が大きく下がる(数十ポイント低下)ことが示されており、「考えてから動く」ことが性能の本質的要因だと裏づけられています。逆に Thought を入れすぎても冗長になり得るため、適度な頻度が良いという観察もあります。

## メリット・トレードオフ・限界

メリット。外部から最新・正確な情報を取り込めるため、推論だけより事実に忠実(ハルシネーションを抑制)です。推論が行動を方向づけるため、行動が計画的で迷走しにくくなります。Thought/Action/Observation が文章で残るので、何をどう判断したかを人間が追跡・修正でき、解釈性が高いです。再学習不要・少数のお手本で、質問応答から対話的環境操作まで多様なタスクに適用できる汎用性があります。

トレードオフと限界。検索 API や操作環境などの外部ツールが前提で、その品質・可用性に結果が依存します。ツールが誤った情報を返せば、ReAct はそれを信じて誤ります。ループが長くなるほどモデル呼び出し回数・遅延・コストが増え、途中で文脈(コンテキスト)が長くなりすぎてウィンドウ上限に達したり、重要情報が埋もれて破綻することもあります。ツールや行動空間、お手本プロンプトの設計に手間がかかります。さらに、エラーからの回復が弱いという弱点があり、一度悪い行動列に入ると同じ失敗を繰り返しやすい(同じ検索を繰り返すループに陥るなど)。これを補うために Reflexion のような自己反省機構が後に提案されました。低遅延が必須の用途では、毎ステップのツール往復が重く不向きです。

未解決の研究課題として、(1) 長いループでの文脈管理とメモリ設計(何を覚え何を忘れるか)、(2) 失敗からの自動回復・自己修正、(3) ツール呼び出しの誤りを検出して再試行する信頼性向上、(4) 探索的な行動と確実な行動のバランス、(5) 複数ツール・複数エージェントの協調、などがあります。

## 発展トピック・研究の最前線

ReAct のループは、その後のエージェント研究で「観測 → 推論 → 計画 → 行動」の標準サイクルとして定着しました。発展方向としては、行動の振り返りを蓄積するメモリ(Reflexion 系)、複数のエージェントが役割分担して協調するマルチエージェント、ツール選択をモデルが学習で内在化する方向(Toolformer 系、関数呼び出しの学習・強化学習)、そして木探索(ToT / LATS)と組み合わせて「行動の選択肢を分岐評価する」方向があります。近年の推論特化モデルは、ReAct 的な「考える→道具を使う→観測する」ループを強化学習で内在化しつつあり、プロンプトの工夫から学習による獲得へと重心が移っています。

自動運転や具体的なロボット制御の文脈では、ReAct の「観測 → 解釈 → 行動 → また観測」という構造が、知覚・行動サイクルそのものと対応します。ただしツール往復(とくに大規模モデルの呼び出し)の遅延が大きいため、毎秒10回以上回したい高頻度の制御ループにそのまま組み込むのは難しいです。むしろ高レベルの計画立案、運転判断の説明生成、オフラインのシナリオ分析・故障診断といった、遅くても許される層で活きる考え方です。この「遅いが賢い推論ループ」と「速い反射制御」を分けて組み合わせる発想は、二系統運転アーキテクチャ(参照: [Fast-Slow 二系統運転アーキテクチャ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0537_fast-slow-thinking-driving))や認知科学の二重過程理論(参照: [System1/System2 (Kahneman 二重過程理論)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0546_system1-system2-kahneman))と直結します。
