# FPNネックとチャネル射影

## ひとことで言うと

画像から特徴を取り出すバックボーンは、大きさ(解像度)の違う複数枚の特徴マップ(マルチスケール特徴)を出します。これらを「ネック(neck)」と呼ばれる中間部品でまとめ、最後に 1×1 畳み込みで固定次元(よく 256)に落とします。ネックは「高解像度だが意味が浅い特徴」と「低解像度だが意味が深い特徴」をうまく組み合わせる役目を持ち、チャネル射影は「下流が前提とする 1 つの太さに揃える」役目を持ちます。代表的なネックが FPN(Feature Pyramid Network、特徴ピラミッドネットワーク)です。

## 直感的な理解

道路の写真を見ると、物体はいろいろな大きさで写っています。100 m 先の歩行者は数ピクセルの点、目の前の車は画面の半分を占めるかもしれません。これを 1 つの解像度の特徴マップだけで捉えるのは無理があります。細かい点を見るには高解像度が要りますが、高解像度の特徴は「浅い」(エッジや色は分かるが、それが何かまでは分かりにくい)。逆に低解像度の特徴は「深い」(意味は分かるが、位置が粗い)。

例えるなら、地図を「拡大して細部を見る」のと「縮小して全体の意味を掴む」のを同時にやりたい、という要求です。FPN は、深い(意味的)特徴を拡大して浅い(高解像度)特徴に足し込むことで「高解像度なのに意味も豊か」な特徴を作ります。これがネックの仕事です。そして最後に、後続のモジュールが扱いやすいよう特徴の太さを 1 つの値に揃えるのがチャネル射影です。

## 基礎: 前提となる概念

まずマルチスケール特徴を理解します。バックボーン(ResNet、Swin Transformer など)は、画像を処理する過程で段階的に解像度を下げながら、複数の段階(ステージ)の特徴を出します。例えば 256×256 入力の Swin Transformer V2 Tiny は、ざっくり次の 4 段階を出します(解像度は段階ごとに半分になり、チャネル数=特徴の太さは倍増していく)。

```
stage1: [BV,  96, 64, 64]   高解像度・浅い特徴(細かいエッジ、小さい物体に強い)
stage2: [BV, 192, 32, 32]
stage3: [BV, 384, 16, 16]
stage4: [BV, 768,  8,  8]   低解像度・深い特徴(大きい物体、意味的な情報に強い)
```

`BV` はバッチ B とカメラ台数 V を一時的に掛けた数です。チャネル数が 96→192→384→768 と倍々に増え、解像度が 64→32→16→8 と半減していくのが階層的バックボーンの典型です。

次にネック(neck)です。バックボーンを「胴体」、最終的な予測を出すヘッドを「頭」とすると、その間をつなぐ「首」にあたるのでネックと呼びます。ネックの仕事は、胴体が出した複数解像度の特徴を整理・融合して、頭が使いやすい形に整えることです。

最後にチャネル射影(channel projection)です。ネックを通したあと、下流が前提とする 1 つの固定次元(典型的には 256)に特徴の太さを揃える操作です。1×1 畳み込みで行います。なぜ固定次元に揃えるかの理由は [統一埋め込み次元(embed_dim=256)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1051_unified-embed-dim-256) で詳しく扱います。

## 仕組みを詳しく

まず一般的な FPN の発想を説明し、次に簡略版のアプローチを説明します。

FPN(Lin et al. 2017)は次のように動きます。最も深い特徴(stage4、低解像度・意味的)から始め、それを上(高解像度)へ 2 倍に拡大(アップサンプリング)し、1 つ浅い特徴(stage3)に足し込みます。これを stage2、stage1 まで繰り返します。このとき各段の特徴は 1×1 畳み込みでチャネル数を共通値(よく 256)に揃えてから足します(これを lateral connection、横方向接続と呼びます)。結果として「各解像度で、その解像度の細かさを保ちつつ、深い意味も注入された」特徴ピラミッドができます。

```
FPN のトップダウン経路(深い→浅いへ意味を流す)

stage4 [768,8,8] ──1x1──> [256,8,8] ─────────────┐
                              │ 2倍拡大            │ 出力 P4
stage3 [384,16,16]─1x1──>[256,16,16] + 拡大 ──> 融合 → 出力 P3
                              │ 2倍拡大
stage2 [192,32,32]─1x1──>[256,32,32] + 拡大 ──> 融合 → 出力 P2
                              │ 2倍拡大
stage1 [96,64,64] ─1x1──>[256,64,64] + 拡大 ──> 融合 → 出力 P1
```

一方、より軽量な「適応プール → 連結 → 射影」方式もよく使われます。これはフル FPN ほど凝らず、実装が単純です。順に見ます。

ステップ 1: 適応プールで解像度を揃える。

```python
self.pool = nn.AdaptiveMaxPool2d(image_feature_size)  # 例: 8
for i in range(len(features)):
    features[i] = self.pool(features[i])
```

`AdaptiveMaxPool2d(8)` は「入力が何×何でも出力を 8×8 にする最大プーリング」です。プーリングとは小領域の代表値(ここでは最大値)を取って解像度を下げる操作です。これで 64×64, 32×32, 16×16, 8×8 がすべて 8×8 に揃います。FPN のように足し算で混ぜるのではなく、いったん同じ解像度に揃えてしまうのがこの方式の特徴です。

ステップ 2: チャネル方向に連結する。

```python
fused = torch.cat(features, dim=1)   # [BV, 96+192+384+768=1440, 8, 8]
```

`dim=1` はチャネル次元です。4 段階を縦に積み上げて 1440 チャネルにします。これで「全スケールの情報が各位置に同居する」状態になります。

ステップ 3: 1×1 畳み込みで固定次元に射影する(ここがチャネル射影)。

```python
self.channel_proj = nn.Sequential(
    nn.Conv2d(1440, 256, kernel_size=1),  # backbone_channels -> embed_dim
    nn.GELU()
)
fused = self.channel_proj(fused)  # [BV, 256, 8, 8]
```

1440 → 256 への圧縮です。1×1 畳み込みは各位置でチャネル方向だけを線形変換する操作で、空間は混ぜません。

なぜこれが「約 31 倍の軽量化」になるのか。後段の複数カメラ統合や Transformer 処理にある注意機構の主要コストは、おおむね埋め込み次元の 2 乗に比例します。1440 のまま流すと 256 に比べて (1440/256)² ≈ 31.6 倍重い。射影で 256 に落とすことで、その重さを入口で断ち切ります。1440 という具体値は Swin V2 Tiny / ConvNeXt V2 Tiny の 4 段階の合計(96+192+384+768)です。ResNet-50 なら 3840(256+512+1024+2048)になりますが、`channel_proj` の入力を 3840 にするだけで出力は変わらず 256 です。

ステップ 4: 射影後の `[BV, 256, 8, 8]` を下流に渡します。複数カメラ統合の方式によって、BEV グリッドに再投影されたり、そのまま 8×8 のまま統合されたりします。

FPN 方式と簡略版の違いを整理します。FPN は各解像度を保ったまま意味を上から流し込むので、小さい物体の検出に強い高解像度出力(P1)を残せます。簡略版は全部を 8×8 に潰して連結するので実装が単純で計算も軽い反面、高解像度段の細かい位置情報が粗くなります。どちらを選ぶかは「小物体の精度をどこまで重視するか」と「計算予算」のトレードオフです。

## 手法の系譜と主要論文

- Lin et al., "Feature Pyramid Networks for Object Detection" (FPN, CVPR 2017, arXiv:1612.03144)。マルチスケールを安価に扱う標準手法を提案しました。深い特徴を上へ拡大して浅い特徴と足すトップダウン経路と、各段の 1×1 畳み込み(横方向接続)で「高解像度かつ意味的」なピラミッドを構築します。小さい物体の検出精度が大きく上がりました。トレードオフは複数段の予測を扱うぶん実装と推論がやや複雑になること。

- Liu et al., "Path Aggregation Network" (PANet, CVPR 2018, arXiv:1803.01534)。FPN のトップダウン経路に加え、ボトムアップ経路(浅い→深い)も足して情報の往復を強化しました。浅い層の正確な位置情報を深い層へも伝えます。さらに精度が上がる反面、計算量が増えます。YOLO 系の物体検出器が広く採用するネックです。

- He et al., "Deep Residual Learning for Image Recognition" (ResNet, CVPR 2016, arXiv:1512.03385) と Liu et al., "Swin Transformer V2" (CVPR 2022, arXiv:2111.09883)。これらバックボーンが「4 段階の階層的特徴」を出す前提を作りました。ネックはこの 4 段階を消費します。

- Carion et al., "DETR" (ECCV 2020, arXiv:2005.12872) 系で「特徴次元 256」が定着したことが、射影先を 256 にする根拠になっています([統一埋め込み次元(embed_dim=256)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1051_unified-embed-dim-256))。

系譜としては、(1) バックボーンが階層的特徴を出すようになった(ResNet 2016)→(2) それを賢く融合するネック FPN が登場(2017)→(3) ネックの双方向化 PANet(2018)→(4) Transformer 検出器で次元 256 が標準化(2020)、という流れで「マルチスケール特徴の扱い」が成熟しました。

## 論文の実験結果(定量データ)

FPN の効果は物体検出ベンチマーク COCO(80 カテゴリ、約 12 万枚の画像を持つ標準的な物体検出データセット)で示されました。評価指標は AP(Average Precision、平均適合率、検出の正確さの総合点、高いほど良い)です。

Lin らの論文では、Faster R-CNN(当時の標準検出器)に FPN を組み込むと、FPN なしの単一スケール特徴に比べて COCO の AP が顕著に向上しました。特に小さい物体に対する AP(AP_small)の改善が大きく、これは「高解像度かつ意味的」な特徴ピラミッドが小物体検出に効くことを実証しています。論文の代表的な数値では、ResNet-101 バックボーンの Faster R-CNN が FPN により AP で数ポイント(おおむね 2〜3 ポイント前後)改善したと報告されています。AP の 1 ポイントは検出研究では大きな差で、この改善が FPN を標準部品に押し上げました。

PANet は FPN にボトムアップ経路を足し、COCO のインスタンスセグメンテーション(各物体を画素単位で塗り分けるタスク)と物体検出でさらに AP を改善したと報告しています。論文はボトムアップ経路と適応的特徴プーリングの各要素を抜くアブレーションを行い、両方が精度に寄与することを示しました。

簡略版(適応プール+連結+射影)に関しては、フル FPN を省く代わりに計算量を下げる工学的選択であり、精度との明示的なトレードオフがあります。8×8 まで一気に潰すと高解像度段の位置情報が粗くなるため、小物体の精度が重要なタスクではフル FPN のほうが有利、というのが一般的な理解です。逆に、入力解像度が小さい・計算予算が厳しい・物体の大きさのばらつきが小さい状況では、簡略版で十分なことも多い、というのがコミュニティの経験則です。

チャネル射影の効果は計算量で測れます。連結後の 1440 チャネルをそのまま下流の注意機構に流すと、256 に射影した場合に比べ注意計算が約 31.6 倍。射影層自体のパラメータ(1440×256 ≈ 37 万)は下流で削減される計算量に比べれば微小です。

## メリット・トレードオフ・限界

メリット。

- マルチスケール特徴を 1 か所でまとめ、固定次元(256 など)に揃えるので下流が共通化できます。
- 1440→256 の射影で後段の注意機構を約 31.6 倍軽量化できます。連結を空間ごと全結合するような重い射影と比べると、パラメータも大幅に減ります。
- 適応プールにより、入力解像度やバックボーンが変わっても出力サイズが一定になります。

トレードオフと限界。

- 最大プーリングで 8×8 まで一気に落とす簡略版は、高解像度段の細かい位置情報が粗くなります。小さい物体の精度には不利になり得ます。
- 簡略版はフル FPN のトップダウン融合を省くため、スケール間の意味的な伝播は連結に頼るのみで、FPN ほど明示的にはスケールを統合しません。
- 1440→256 の圧縮は情報を捨てる操作です。深層特徴は冗長性が高く実害は小さいことが多いですが、原理的には損失があります。
- フル FPN は精度は高いが、複数解像度の出力を扱うぶん実装と推論が複雑になり、メモリも食います。どのネックを使うかは精度と計算予算のトレードオフで決まり、万能解はありません。

## 発展トピック・研究の最前線

- BiFPN(EfficientDet、Tan et al. 2020)。FPN を双方向にし、各融合に学習可能な重みを付けて「どのスケールをどれだけ混ぜるか」を学習させる発展形です。少ない計算で高精度を出します。
- NAS-FPN(Ghiasi et al. 2019)。ネックの接続構造そのものをニューラルアーキテクチャ探索(NAS)で自動設計する研究。人手設計の FPN を上回る構造を発見しました。
- Transformer 時代のネック。Deformable DETR 以降は、ネックでマルチスケール特徴をトークン列に変換し、変形可能注意でスケール横断的に融合する設計が主流になりつつあります([BEVFormer風の暗黙的投影](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0273_bevformer-deformable-attention))。明示的なピラミッドではなく、注意機構がスケール融合を担う方向です。
- BEV 知覚における特徴融合。多視点カメラの特徴を BEV グリッドに射影する際、各カメラのマルチスケール特徴をどう融合するかが性能を左右します。FPN/簡略版/変形可能注意のどれを使うかは、BEV 知覚研究でも活発に検討されています。
