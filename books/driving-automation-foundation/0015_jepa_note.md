# JEPA (Joint Embedding Predictive Architecture)

## ひとことで言うと
JEPA とは「未来や隠された部分を、ピクセル(色や明るさそのもの)ではなく、抽象的な特徴(意味のかたまり)で予測する」自己教師あり学習(self-supervised learning, 人間が付けた正解ラベルを使わず、データ自身の構造から学ぶ手法)の枠組みです。生成モデルが「次の画像を画素まで描く」のに対し、JEPA は「次に何が起きるかの意味だけを当てる」ことで、予測しても無意味な細部に計算を浪費しません。Yann LeCun が 2022 年に提唱した、世界モデルを作るための中心的なアーキテクチャ構想です。

## 直感的な理解

あなたが運転中、前を走る車を見ているとします。1.6 秒後の世界を「予測せよ」と言われたら、何を予測しますか。おそらく「前の車は少し近づいている」「信号はまだ青」「右の歩行者は横断歩道の手前で止まっている」といった、運転判断に効く事柄でしょう。

ここで仮に、1.6 秒後の景色をカメラ画像そのもの、つまり全ピクセルとして正確に描けと要求されたらどうでしょうか。前の車のフロントガラスにどんな空の反射が映るか、街路樹の葉が風でどちらに揺れるか、路面のアスファルトの粒のノイズがどう並ぶか。これらは運転とは何の関係もないのに、ピクセル予測タスクではすべて当てる責任を負わされます。人間はそんな無駄なことはしません。意味だけを取り出して予測します。

JEPA はこの「人間のやり方」を機械にやらせる枠組みです。生のピクセルを当てるのではなく、画像をいったん意味のベクトル(特徴)に圧縮してから、その特徴空間で未来を予測します。

## 基礎: 前提となる概念

ここでいくつか用語を噛み砕いておきます。

特徴空間(feature space, 潜在空間 latent space とも言う)とは、生のピクセルをニューラルネットに通して得られる中間的なベクトルの世界のことです。たとえば 256×256×3(縦 256 × 横 256 × RGB3色 = 196,608 個の数値)のピクセルを、画像を要約するネットワーク(バックボーン backbone と呼びます。CNN や Vision Transformer など)に通すと、[チャンネル 256, 高さ 8, 幅 8] のような圧縮表現(計 16,384 個の数値)になります。この段階で「ここに車がある」「道がこちらへ伸びている」といった意味は残り、葉一枚の正確な位置のような無意味な細部は失われています。

自己教師あり学習とは、ラベルなしのデータからモデルが学ぶ手法です。たとえば「画像の一部を隠して、隠した部分を予測させる」「ある動画フレームから次のフレームを予測させる」といった、データ自身から自動的に作れる課題(プレテキストタスク pretext task と呼びます)で学習します。これにより、人間が何百万枚もラベルを付ける手間なしに、汎用的な表現を獲得できます。

エンコーダ(encoder)とは、入力(画像など)を特徴ベクトルに変換するネットワークです。プレディクタ(predictor)とは、ある特徴から別の特徴(未来の特徴など)を予測するネットワークです。

生成的予測(generative prediction)とは、ピクセルや単語など、観測そのものを再構成・生成する予測です。画像なら次のフレームを画素単位で描く、言語なら次の単語を当てる、といった形です。JEPA はこれと対比される「特徴空間での予測」を行います。

## 仕組みを詳しく

JEPA の構成要素は 3 つです。

1. コンテキストエンコーダ(context encoder): 今見えている入力 x をエンコードして特徴 s_x を作る。
2. ターゲットエンコーダ(target encoder): 予測したい対象 y(未来の画像、隠された領域など)をエンコードして特徴 s_y を作る。多くの場合これは勾配を更新しない凍結コピー、または指数移動平均(EMA)でゆっくり更新されるコピーです(詳しくは [凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))。
3. プレディクタ(predictor): s_x と条件(action、位置情報、マスク情報など)から s_y を予測する。

学習の流れを tensor の形(shape)つきの数値例で書きます。バッチサイズ B=4 とします。

- 入力画像 x [B, 3, 256, 256] を context encoder に通す → s_x [B, 256, 8, 8]。
- 対象 y [B, 3, 256, 256](未来フレームや隠した領域)を target encoder に通す → s_y [B, 256, 8, 8]。このとき target encoder には勾配を流さない(detach する)。つまり s_y は「動かない正解ラベル」として扱う。
- predictor が s_x(+ 条件)から予測 ŝ_y [B, 256, 8, 8] を作る。
- 損失は L = mean((ŝ_y − s_y)^2)。これは特徴ベクトル同士の二乗誤差(MSE, Mean Squared Error)です。ピクセルではなく特徴で誤差を測っている点が JEPA の核心です。

ASCII で全体像を示します。

```
今の画像 x ─► [context encoder] ─► s_x ─┐
                                        ├─► [predictor] ─► ŝ_y ─┐
       条件(action等) ────────────────┘                        │
                                                          MSE 損失 (特徴空間)
対象 y ─► [target encoder(凍結/EMA)] ─► s_y (detach) ─────────────┘
```

ここで決定的に重要な設計が「ターゲット側に勾配を流さない」ことです。なぜか。もし context と target の両エンコーダを自由に学習させると、学習は最も簡単な近道を見つけます。それは「s_x も s_y も常にゼロベクトルを出す」ことです。両者がゼロなら予測も完璧(0 = 0)で損失はゼロになりますが、その特徴は何の情報も持たない無意味な表現です。この「全部同じ値に潰れて意味を失う」現象を表現崩壊(representation collapse)と呼びます。ターゲット側を固定すれば、その出力 s_y は「動かない本物のラベル」となり、予測側はゼロに逃げられず本物の意味を当てに行かざるを得なくなります(詳細は [凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))。

JEPA を「条件付きの予測」と見ると応用範囲が広がります。条件に action(自分の取る行動)を入れれば、「この行動をとったら世界はどうなるか」を予測する世界モデル(world model)になります。これがロボティクスや自動運転で JEPA が注目される理由です。

## 手法の系譜と主要論文

JEPA は対比学習(contrastive learning)とマスク復元(masked reconstruction)という 2 つの自己教師あり学習の系譜を統合・止揚する位置にあります。順に追います。

対比学習の系譜。Chen et al. の SimCLR(2020)、He et al. の MoCo(CVPR 2020)が代表で、「同じ画像から作った 2 つの見え方(データ拡張)は近づけ、違う画像同士は離す」ことで表現を学びます。問題は「違う画像(ネガティブサンプル)」を大量に必要とすること、バッチサイズやキューの設計に敏感なことでした。

崩壊回避の革新。Grill et al. の BYOL(NeurIPS 2020)は、ネガティブサンプルなしで、ターゲットネットワークを EMA で更新し stop-gradient と組み合わせるだけで崩壊を防げることを示しました。Chen & He の SimSiam(CVPR 2021)はさらに EMA すら不要で stop-gradient だけで足りると示しました。これらは「予測側だけ学習し、ターゲット側は止める/緩やかに追従させる」という JEPA の根幹を準備した研究です(詳細は [凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))。

マスク復元の系譜。He et al. の MAE(Masked Autoencoders, CVPR 2022)は、画像の 75% をマスクし、隠したピクセルを復元させます。シンプルで強力ですが、復元対象がピクセルなので「予測しても無意味な細部」を当てる負担が残ります。

JEPA の提案。LeCun の構想論文「A Path Towards Autonomous Machine Intelligence」(2022)が JEPA を体系的に提示しました。「機械が世界を理解するには、生成的にピクセルを当てるのではなく、特徴空間で予測する自己教師あり世界モデルが必要だ」という主張です。

実装としての具体化。Assran et al. の I-JEPA(Image-JEPA, CVPR 2023)が画像で JEPA を初めて実装しました。1 枚の画像から複数のコンテキストブロックを取り、隠したターゲットブロックの特徴を予測します。続いて Bardes et al. の V-JEPA(Video-JEPA, 2024)が動画に拡張し、時間方向の未来特徴予測を扱いました。さらに 2025 年には V-JEPA 2 が大規模動画事前学習とロボット行動への接続(action-conditioned 予測)を示し、JEPA を世界モデルとして使う方向を具体化しています。

これらを並べると、JEPA は「対比学習が必要としたネガティブサンプルを predictor + stop-gradient で不要にし、MAE のピクセル復元を特徴空間予測に置き換えた」統合形だと整理できます。

## 論文の実験結果(定量データ)

I-JEPA(Assran et al., CVPR 2023)の主要な結果を、指標の意味とともに見ます。

評価には ImageNet-1k(1000 クラス・約 128 万枚の画像分類ベンチマーク)の線形評価(linear probing)がよく使われます。これは「学習済みエンコーダを凍結し、その出力の上に線形分類器(1 層)だけを学習させて分類精度を測る」やり方で、表現そのものの良さを測る標準的な方法です。エンコーダ全体を微調整しないので、特徴が素のままどれだけ分類に役立つかが分かります。

報告によると、I-JEPA は ViT-Huge/14(Vision Transformer の大型版、パッチサイズ 14)を ImageNet で事前学習し、線形評価でおよそ 70% 台後半(報告では ViT-H/14 を 448 解像度まで上げた設定で約 81% 前後)の Top-1 精度(正解クラスが 1 位の割合)を達成しました。重要なのは計算効率で、I-JEPA は MAE や iBOT といった既存手法と同等以上の精度を、より少ない事前学習エポック(GPU 時間)で達成したと報告されています。具体的には、I-JEPA(ViT-H/14)はピクセル再構成系の MAE(ViT-H/14)に対して、同等以上の線形評価精度をおよそ数分の一の計算量(GPU 時間にして 10 分の 1 程度との報告もある)で得た、とされます。これは「無意味な細部を当てなくてよいぶん、学習が効率化する」という JEPA の主張を裏付けます。なお、対比学習系(SimCLR 等)が必要とした大量のネガティブサンプルや強いデータ拡張を、I-JEPA は使わずに済む点も計算効率に寄与しています。

アブレーション(ある要素を抜くと性能がどう変わるかの検証)では、マスク戦略(何を隠して何から予測するか)が性能に大きく効くことが示されました。I-JEPA はターゲットブロックを大きめ(画像の 15〜20% 程度)に、空間的にまとまった領域として取ると良く、小さく散らばったマスクだと意味的な予測にならず性能が落ちると報告されています。また、ターゲットエンコーダを EMA で更新する設定が崩壊回避に必須で、これを外すと表現が崩壊します。

V-JEPA(Bardes et al., 2024)は動画で評価され、Kinetics-400(400 種の動作分類)や Something-Something v2(物体操作の動作認識)で、凍結エンコーダ + 軽量分類器の評価において、ピクセル予測系の動画自己教師あり手法を上回る、または同等の性能を報告しています。とくに「動きの理解」を要する Something-Something v2 での優位が報告され、時間方向の特徴予測が動的な意味を捉えるのに有効だと示唆されました。V-JEPA 2(2025)はさらに大規模化し、ロボット操作のゼロショット/少数ショット計画にまで応用範囲を広げています。

これらの数値が意味するのは「特徴空間予測は、ピクセル復元に比べて計算効率が良く、しかも下流タスク(分類・動作認識・計画)で同等以上の表現を学べる」ということです。

## メリット・トレードオフ・限界

メリット。予測しても無意味な細部(反射、ノイズ、葉の揺れ)を学習段階で自然に無視でき、本質的な情報に計算資源を集中できます。ぼやけた平均画像を出すという生成モデルの典型的失敗を回避できます。ピクセル復元より少ない計算で意味的な表現を学べる傾向があります(I-JEPA の報告)。条件に action を入れれば世界モデルとして計画・制御に使えます。

トレードオフと限界。予測がベクトルなので、人間が「予測結果を画像として目で見て確認する」ことができず、デバッグや解釈性が下がります。ターゲットの凍結・EMA・stop-gradient など崩壊回避の工夫が必須で、設計を誤ると全部ゼロに崩壊して学習が無に帰します。マスク戦略やホライズン(どれだけ先を予測するか)の設計に性能が敏感で、ハイパーパラメータ調整の負担があります。さらに研究上の未解決課題として、なぜ EMA + stop-gradient で崩壊が防げるのかの理論的理解がいまだ不完全である点、特徴空間が「何を捨て何を残すか」をタスクに最適に制御する方法が確立していない点が挙げられます。特徴予測は便利な反面、特徴を作るエンコーダが落とした情報は二度と回復できないため、エンコーダ設計が表現の上限を決めてしまうという本質的限界もあります。

## 発展トピック・研究の最前線

JEPA を世界モデルとして使い、潜在空間でロールアウト(未来を順に展開)して計画する研究が進んでいます。V-JEPA 2 の action-conditioned 予測は、ロボットが「この行動をとったら潜在状態がどうなるか」を予測し、目標状態に近づく行動列を探索する、という Model Predictive Control 的な使い方を示しました。自動運転では、カメラ特徴の未来予測を運転方策学習の補助タスクに使う方向([潜在世界モデルによる未来予測 (FutureState)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0020_world-model-future-prediction) や [多ステップ未来特徴予測と予測ホライズン設計](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0019_temporal-feature-prediction-horizon))が活発です。また、複数の時間スケールで予測を積む階層化([階層JEPA (H-JEPA)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0058_hierarchical-jepa))が長期計画と短期制御の橋渡しとして提案されています。表現崩壊の理論的解明、特徴空間の情報量の制御、マルチモーダル(画像・LiDAR・言語)JEPA への拡張が今後の焦点です。
