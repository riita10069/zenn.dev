# 幾何ベース vs 学習ベースの視点変換

## ひとことで言うと

画像から BEV (鳥瞰図) へ特徴を持ち上げるとき、設計者は 1 つの根本的な判断を迫られます。「カメラの取り付け位置と向き (校正情報) という既知の幾何を明示的に計算に使う」のか、それとも「そういう幾何を一切教えず、画像と BEV の対応関係をネットワークにデータから丸ごと学ばせる」のか。前者を幾何ベース (geometric / calibration-aware)、後者を学習ベース・暗黙 (learned / implicit / calibration-free) と呼びます。本ノートはこの 2 つの流儀のトレードオフを、なぜその分岐が生じるのかから掘り下げます。

## 直感的な理解

地図アプリで「自分が今どこにいるか」を表示する状況を思い浮かべてください。GPS 座標が分かっていれば、地図上のどこに点を打てばよいかは計算式で一発で決まります。これが幾何ベースの発想です。すでに分かっている確かな情報 (座標) を使って、対応関係を計算で求める。

一方、もし GPS がなく「目に見える景色だけから自分の位置を当てよ」と言われたら、膨大な経験 (この建物が見えるならここ、という記憶) から推測するしかありません。これが学習ベースの発想です。確かな手がかりを使わず、対応関係そのものを経験から学ぶ。

視点変換でも同じ構図が成り立ちます。カメラの内部パラメータ (焦点距離・画像中心) と外部パラメータ (車体に対する取り付け位置・向き) が分かっていれば、3 次元空間の点が画像のどのピクセルに写るかは、ピンホールカメラの式で厳密に計算できます。

```
画像座標 = 内部行列 @ 外部行列 @ 3D点(同次座標)
ピクセル(u, v) = 上の結果の最初の2成分 ÷ 3成分目(=深度)   ← 透視除算
```

この式 (@ は行列の掛け算) を使えば「BEV のこのマス目は前カメラのこのあたりに写る」と直接わかるので、ネットワークは対応関係を一から学ぶ必要がありません。学習が楽になり、見たことのない場面への汎化も効きやすい。これが幾何ベースの強みです。

ところがこの幸せには条件があります。校正データが正確に揃っていなければなりません。校正がズレていたり、そもそもデータセットに校正情報が含まれていなかったりすると、幾何ベースは計算の前提が崩れて破綻します。だからこそ「校正に頼らず対応を丸ごと学ぶ」学習ベースという選択肢が、別の状況では合理的になるのです。

## 基礎: 前提となる概念

ピンホールカメラモデル (pinhole camera model): カメラを「小さな穴を通して 3 次元世界を 2 次元の像に投影する装置」として理想化した数学モデルです。穴 (光学中心) を通った光が裏側の面に像を結ぶ、という幾何で、3 次元の点 (X, Y, Z) が画像の (u, v) に写る対応が行列演算で書けます。これが幾何ベース視点変換の数学的土台で、コンピュータビジョンの古典 (Hartley & Zisserman の多視点幾何) で体系化されています。

内部パラメータ (intrinsics): レンズの性質を表す 3×3 の行列で、焦点距離 (どれだけズームしているか) と画像中心 (光軸が画像のどこを貫くか) を含みます。同じ 3D 点でも焦点距離が違えば写る位置が変わるので、この行列が必要です。

外部パラメータ (extrinsics): カメラが世界 (または車体) のどこに、どの向きで置かれているかを表す回転と並進です。3×4 の行列で書け、世界座標をカメラ座標に変換します。内部と外部を掛け合わせると、世界座標から直接ピクセル座標へ飛ばす射影行列ができます。

透視除算 (perspective division): 射影の途中で出てくる 3 成分目 (深度に相当) で残り 2 成分を割る操作です。「遠くのものほど画面で小さく寄る」という遠近法は、まさにこの除算から生まれます。深度がゼロや負 (カメラの背後) だとここで破綻するので、可視判定が要ります。

MLP (多層パーセプトロン): 全結合層を重ねた基本的なニューラルネットワークです。入力ベクトルを重み行列で変換することを繰り返し、任意の写像を近似できます。学習ベース視点変換では、この MLP に「画像特徴 → BEV 特徴」の写像そのものを学ばせます。

これらを押さえると、幾何ベースは「ピンホールモデルの行列演算で対応を計算し、ネットワークには微調整だけ任せる」、学習ベースは「行列演算を使わず、MLP やアテンションに対応関係ごと学ばせる」と整理できます。

## 仕組みを詳しく

### 幾何ベース (geometric, calibration-aware)

BEV の各マス目に対応する 3 次元地点を、既知のカメラ射影行列で画像座標に変換します。典型的な処理を段階で見ます。

1. BEV グリッド (例えば 200×200 のマス目) の各セルに、高さ違いの 3 次元参照点を割り当てる。地面付近だけでなく、低い位置から高い位置まで数点を縦に並べることが多い (これをピラーと呼ぶ)。座標は自車中心の基準 (X が前、Y が左、Z が上、など) で持つ。
2. 各参照点を射影行列 (内部 @ 外部、形は `[3, 4]`) で画像座標に飛ばす。行列演算の後、3 成分目 (深度) で割る透視除算をしてピクセル座標 (u, v) を得る。
3. 深度がゼロ以下 (カメラの背後) や画像の外に出た点は、可視マスクで除外する。後方の BEV セルが前カメラを誤って参照しないようにするためです。
4. 射影先のピクセル位置で画像特徴をサンプリングする (双線形補間で拾う)。

ここでは「どこを見るか」の大枠を幾何が決め、ネットワークは「サンプル位置の微調整 (学習可能オフセット)」と「複数候補の重み付け」だけを学びます。対応の骨格を計算で与え、学習は仕上げに回す、という分業です。

### 学習ベース (learned / implicit, calibration-free)

幾何を一切使わず、画像特徴から BEV 特徴への写像を MLP やアテンションでデータから学習します。最も素朴な形は「画像特徴を平坦化して全結合層に通し、BEV 特徴の形に整形する」ものです。初期の VPN (View Parsing Network) は「画像の各列を BEV の放射方向に対応づける」構造を MLP に学ばせました。PON (Pyramid Occupancy Network) はカメラ画像のボトルネックを経由して BEV 平面の意味マップを生成しました。

学習ベースには「疑似射影行列 (pseudo projection matrix)」という中間的な工夫もあります。これは校正がない状況でも幾何ベースと同じパイプラインを動かすため、本来は固定の校正行列であるべき部分を学習可能なパラメータに置き換えたものです。概念的には次のように書けます。

```python
# 本来は固定の校正行列 (num_views, 3, 4) を、学習で決まるパラメータに置換する
pseudo_projection = nn.Parameter(torch.randn(num_views, 3, 4) * 0.01)
```

これは「校正があるかのように振る舞う学習済みの空間的な事前分布」にすぎず、真のカメラ幾何を復元するわけではありません。意味のある BEV 射影には本物の校正が必要で、疑似射影はあくまでデータセットに校正がない場合のフォールバックです。こうした疑似射影は数値的に不安定になりやすく、ランダムに近い入力では透視除算で小さな深度値が分母に来て値が爆発し、学習が発散して損失が NaN (非数、計算が壊れて意味のある数値でなくなった状態) になることが知られています。これは「対応を保証する幾何の拘束がないと、最適化が暴れやすい」という学習ベースの構造的な弱点を示しています。

### before / after の対比

| 観点 | 幾何ベース | 学習ベース (暗黙) |
|---|---|---|
| 校正データ | 必要 | 不要 |
| 学習負荷 | 軽い (対応は幾何が与える) | 重い (対応も学ぶ) |
| データ要件 | 少なめで済む | 大量に要る |
| 汎化 | 校正が合えば強い | カメラ配置変更に弱い |
| データセット非依存性 | 校正があれば高い | 学習時の配置に依存 |
| 数値安定性 | 高い | 低い (発散しやすい) |
| 代表 | BEVFormer, LSS の投影部 | VPN, PON, 純 MLP 変換 |

幾何ベースと学習ベースは、実は二者択一ではなく連続的なグラデーションです。LSS は深度を学習で予測しつつ、3D へ持ち上げる lift と地面へ落とす splat にはカメラ幾何を使う「幾何 + 学習」の中間例です。多くの現代手法は、幾何で骨格を与えつつ学習で柔軟性を足す、という混合をとっています。

## 手法の系譜と主要論文

時系列で流れを追います。

最初期 (2019-2020 年) は学習ベースが先行しました。Pan ら "Cross-view Semantic Segmentation" (VPN, RA-L 2020) と Roddick & Cipolla "Predicting Semantic Map Representations from Images using Pyramid Occupancy Networks" (PON, CVPR 2020, arXiv:2003.13402) は、校正に依存しない汎用的な image→BEV 変換を模索し、MLP やピラミッド構造で対応を学びました。動機は「カメラ機種や配置が多様で、校正を前提にできない状況でも動く変換が欲しい」というものでした。

転機は幾何を明示的に取り込む手法群です。Philion & Fidler "Lift, Splat, Shoot" (ECCV 2020, arXiv:2008.05711) は深度予測と幾何投影を組み合わせ、Li ら "BEVFormer" (ECCV 2022, arXiv:2203.17270) は BEV クエリの 3D 参照点をカメラ校正で画像に射影し、その射影先だけを変形可能アテンションでサンプリングする「空間クロスアテンション」を提案しました。BEVFormer が幾何を使った動機は明快です。画像全体を総当たりで見るのではなく、幾何が示す「見るべき場所」に注意を集中させて計算を節約し、かつ深度ラベルなしで頑健に融合したかったから。

この系譜の含意は、研究コミュニティが「学習ベースの汎用性」より「幾何ベースの学習効率と汎化」を選ぶ方向に大きく傾いた、ということです。校正情報が運転データセット (nuScenes, Waymo Open Dataset など) で広く利用可能になったことが、この移行を後押ししました。ただし校正のない一般動画や、校正がドリフトする実環境では、学習ベースや自己校正の研究も依然として続いています。

## 論文の実験結果 (定量データ)

評価は主に nuScenes (6 カメラ + LiDAR の都市運転データセット) で行われます。mAP は検出の正確さ (高いほど良い)、NDS は位置・大きさ・向き・速度・属性の誤差を統合した nuScenes 総合指標 (0〜1、高いほど良い)、IoU はセグメンテーションの重なり度合いです。

幾何ベースの優位を示す代表例として、アテンション系で幾何ガイドを外すアブレーションがあります。BEVFormer の論文では、空間クロスアテンション (幾何で射影先を絞る) を素朴な全結合写像 (学習ベース) に置き換えると検出精度が明確に低下することが報告されており、幾何が「見る場所を物理的に正しく絞る」効果の大きさを定量的に示しています。

学習ベースの限界を示すのが、カメラ配置を変えたときの汎化テストです。報告によれば、学習ベースの image→BEV 変換は学習時のカメラ配置に強く依存し、配置を変えると性能が大きく劣化します。対して幾何ベースは校正行列を差し替えるだけで新しい配置に対応でき、再学習なしでの汎化が効きます。LSS が示した「学習時に見ていないカメラを推論時に追加・削除しても性能が緩やかにしか落ちない」という頑健性も、幾何投影を使うことの恩恵です。

数値安定性の面では、校正を持たない疑似射影で学習を回すと、本物の校正を使う場合に比べて損失が発散しやすく、ランダムに近い入力では NaN に至るという観察が広く報告されています。これは幾何拘束がないと最適化の探索空間が広がりすぎて暴れることの表れで、幾何ベースが学習を安定化させる側面を裏づけます。

## メリット・トレードオフ・限界

幾何ベースのメリットは、対応関係を幾何が与えるので学習が軽く、校正が正しければ汎化が強いこと、物理的に意味のある (距離が正しい) BEV になることです。トレードオフは、正確なカメラ校正が必須なこと、校正誤差や「地面は平らである」という仮定のズレがそのまま BEV の歪みになることです。坂道やカメラのキャリブレーションドリフト (走行中の振動や温度変化で取り付け角度が微妙に変わる現象) に弱いのが研究上の未解決課題です。

学習ベースのメリットは、校正不要で動かせること、校正データのないデータセットでも使えることです。トレードオフは、対応も学ぶため大量データが要ること、カメラ配置が変わると学び直しが必要なこと、数値的に不安定になりやすいことです。

実務的には「段階的設計」がよく採られます。データセットや校正が未確定の開発初期は学習ベースのフォールバックでパイプラインを動かし、校正が揃ったら幾何ベースに切り替える、という運用です。本番運用では幾何ベース (本物の校正) を使うのが定石で、学習ベースのフォールバックは校正なしでも壊れずに動く graceful degradation (緩やかな性能低下) を担保するための保険、という役割分担になります。

## 発展トピック・研究の最前線

最前線では「幾何を学習で補正する」方向が伸びています。校正がドリフトする実環境向けに、ネットワークが校正の誤差を推定して補正する自己校正 (self-calibration) や、幾何投影に学習可能な微調整を重ねる手法が研究されています。BEVFormer 系の学習可能オフセットも、幾何が与えた射影点を学習で少しずらすことで校正の小さな誤差を吸収する仕組みで、幾何と学習の融合の一例です。

また、地面平面仮定を捨てて任意の高さ分布を扱う占有グリッド (occupancy) 系の手法や、深度の連続表現で離散ビンの限界を超える手法など、幾何の仮定そのものを緩める研究も進んでいます。長期的には「幾何で確実に分かる部分は幾何に任せ、不確実な部分だけ学習で埋める」というハイブリッドが標準になりつつあり、純粋な幾何ベース・純粋な学習ベースという二分法は、設計の出発点を整理する概念軸として理解しておくのが正しい使い方です。
