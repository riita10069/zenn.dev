# 条件付き最適輸送（OT）パス

## ひとことで言うと

条件付き最適輸送（OT, Optimal Transport）パスとは、Flow Matching の学習で「ノイズ点と本物のデータ点をどう結ぶか」の結び方のうち、いちばん単純に「二点を直線で結ぶ」やり方です。式で書くと u_t = (1 − t)·x_0 + t·x_1（時刻 t での中間点）で、そのときの正解の進む向き（速度）は x_1 − x_0 です。直線なので速度は t に依らず一定で、学ぶ対象が極めて単純になり、生成時の積分も少ないステップで済みます。Flow Matching が拡散モデルより速い理由の核心の一つです。

## 直感的な理解

[Flow Matching（フローマッチング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0121_flow-matching) で見たように、Flow Matching は「ノイズ x_0 から本物 x_1 へ進む速度」をネットに学ばせます。このとき設計者が決めなければならないのが「x_0 から x_1 へ、どういう道筋（パス）で進むことにするか」です。

道筋の選び方は無数にあります。グルッと回り道してもよいし、まっすぐ進んでもよい。どれを選ぶかで「ネットが学ぶべき速度」が変わり、学習の安定性と生成の速さが変わります。たとえば回り道だと、自分が道のどこにいるかで向きがコロコロ変わり、ネットは「位置と時刻ごとに違う向き」を覚えねばならず大変です。一方まっすぐなら、向きは始点から終点へ一直線でずっと一定。学ぶべき速度が単純になります。

身近な比喩でいえば、A 地点から B 地点へ荷物を運ぶとき、最短の直線で運ぶのが移動コスト（距離）の合計を最小にする運び方です。「分布全体を別の分布へ、総移動コスト最小で動かす運び方」を考える数学が「最適輸送（Optimal Transport）」で、移動コストを距離の二乗に取ると最適解は各点を直線で運ぶことになります。Flow Matching で各サンプルペア (x_0, x_1) を直線で結ぶのは、この「最短=直線」という OT の直感を、サンプル一個ずつに「条件付き」で当てはめたものなので「条件付き最適輸送パス」と呼びます。「条件付き」は「真の OT 全体ではなく、与えられたペアごとに直線で近似する」という意味です。

## 基礎: 前提となる概念

**補間（interpolation）**。二点を割合で混ぜて中間点を作ることです。u_t = (1 − t)·x_0 + t·x_1 は線形補間で、t=0 で x_0、t=1 で x_1、その間を直線で結びます。

**速度場とその微分の関係**。パス u_t を時刻 t の関数だと見て微分したもの du_t/dt が、その時刻にその点を進むべき「速度」です。直線補間では du_t/dt = x_1 − x_0 で、t に依りません。これがネットの学習目標（target velocity）になります。一般のパス u_t = α(t)·x_1 + σ(t)·x_0 では速度は du_t/dt = α'(t)·x_1 + σ'(t)·x_0 となり、α・σ の選び方でパスの形（直線か曲線か）が決まります。直線 OT は α(t)=t, σ(t)=1−t という最も単純な選択です。

**最適輸送（OT）と Wasserstein 距離**。OT は「ある確率分布を別の確率分布へ動かす最小コストの輸送計画」を求める問題です（Peyré & Cuturi の総説 arXiv:1803.00567 が標準的な参考文献）。そのコストの最小値が Wasserstein 距離と呼ばれる分布間の距離になります。二次コスト（距離の二乗）のもとでの OT 解は、点を直線で運ぶ「測地線」になることが知られています（Brenier の定理）。Flow Matching の直線パスはこの結果を一ペアに局所適用したものです。

**分散保存パスとの対比**。拡散モデルが暗に使うパスは「分散保存（variance-preserving, VP）」型で、これは曲がっています。VP では中間状態の分散がほぼ一定に保たれるよう α・σ が設計され、ノイズからデータへの軌跡が弧を描きます。直線 OT パスはこの弧をまっすぐに伸ばしたものに相当し、だからこそオイラー法で粗く刻んでも誤差が溜まりにくいのです。

**条件付き Flow Matching と周辺 Flow Matching**。一本ずつの直線の速度（条件付き速度）を回帰することが、全ペアの平均である真の速度場（周辺速度場、実際に分布を運ぶ正しい流れ）を学ぶことと数学的に等価だ、というのが Lipman らの中心定理です。だから「一本ずつ直線」という単純な教師で正しい流れが学べます。証明の鍵は、条件付き確率パスを周辺化（積分）した周辺確率パスが、連続の方程式（continuity equation）を満たす周辺速度場を一意に定める、という点にあります。

## 仕組みを詳しく

データの形を `(B, 128)`（たとえば軌道なら 64 ステップ × 2 信号）とします。

1. **両端を用意**: ノイズ x_0 ~ N(0, I)（ランダム）と本物のデータ x_1（教師データ）。どちらも `(B, 128)`。
2. **時刻 t をサンプル**: t ∈ [0, 1]。t=0 がノイズ端、t=1 が本物端。
3. **直線上の中間点（これがパス）**:
   ```
   u_t = (1 - t)·x_0 + t·x_1
   ```
   例: ある成分で x_0 = −2.0, x_1 = 4.0, t = 0.25 なら
   u_t = 0.75·(−2.0) + 0.25·4.0 = −1.5 + 1.0 = −0.5。
   t を 0→1 と動かすと u_t は −2.0（ノイズ）から 4.0（本物）へ直線的に移動します。
4. **正解の速度（target velocity）**: 直線上を進む向きと速さは時刻 t に依らず一定で
   ```
   target_velocity = x_1 - x_0
   ```
   上の例なら 4.0 − (−2.0) = 6.0。「t がいくつでも速度は同じ 6.0」というのが直線パスの嬉しさです。
5. **学習**: ネット v_θ(u_t, t, c) に u_t・t・条件 c を入れ、出力を target_velocity に MSE で近づける。

```
t=0                              t=1
x_0 ●───────────●───────────● x_1
   (ノイズ)    u_t          (本物のデータ)
   速度はずっと一定 = x_1 - x_0（→ 向き）
```

ここで時刻 t の引き方にも設計余地があります。一様分布 U[0,1] が素直ですが、ロボティクスや軌道生成では、ノイズ側（低 t）に偏らせた Beta 分布（たとえば Beta(1.5, 1)）からサンプルする工夫が知られています。低 t 側は「ほとんどノイズの状態から正しい向きを当てる」ぶん速度場が学びにくいので、そこを重点的にサンプルすると下流タスクの性能が上がる、という経験則に基づきます。大規模画像生成ではロジット正規分布（中間 t を重点）を使う流派もあり、何を重点的に学ぶべきかはタスク依存です。

### 独立ペアとミニバッチ OT の違い

素朴な実装では、ミニバッチ内の x_0 と x_1 を「独立に」対応付けます（ランダムなノイズを各データにそのまま割り当てる）。このとき各ペアの直線は空間中で交差しうるため、無数の直線を平均した周辺速度場は実際には曲がってしまい、推論時の積分軌跡も完全な直線にはなりません。これを緩和するのが「ミニバッチ最適輸送」で、バッチ内の x_0 たちと x_1 たちの間で二次コスト最小の対応付け（割り当て問題、ハンガリアン法や Sinkhorn 法で解く）を求めてからペアを作ると、直線同士の交差が減り、周辺速度場がより直線に近づきます。

## 手法の系譜と主要論文

- **Lipman et al., "Flow Matching for Generative Modeling", ICLR 2023（arXiv:2210.02747）**。条件付き OT パスを提案した中心論文です。各データ点へ向かう条件付きの直線パスと一定速度場を使うと、拡散モデルの曲がった（分散保存）パスより「まっすぐ」で、学習が安定しサンプリングが速いと示しました。新規性は、直線という単純なパスが理論的に正しい（条件付き=周辺の勾配一致）と保証した点です。トレードオフは、独立にサンプルした (x_0, x_1) のペアでは直線同士が交差しうるため、推論時の積分軌跡が完全な直線にはならない点です。

- **Liu et al., "Rectified Flow", ICLR 2023（arXiv:2209.03003）**。同じ「直線で結ぶ」着想を独立に提案しました。さらに reflow という再学習（学習済みの流れで生成したペアを再び直線で結び学習し直す）で軌跡をさらに直線化し、究極的に 1 ステップ生成へ近づけられると示しました。reflow は実質的に、学習済みモデルが作る「ノイズ→データ」の対応を新たなカップリングとして使い、交差を減らす操作で、ミニバッチ OT と同じ「より良い対応付け」の系統にあります。

- **Albergo & Vanden-Eijnden, "Stochastic Interpolants", ICLR 2023（arXiv:2209.15571）**。直線補間を含むより一般の補間子の枠組みを与え、どんな補間が許されるか、決定論（ODE）と確率論（SDE）をどうつなぐかを理論整備しました。直線 OT パスはこの一般枠組みの一特例として位置づけられます。

- **Tong et al., "Improving and Generalizing Flow-Based Generative Models with Minibatch Optimal Transport", TMLR 2024（arXiv:2302.00482）**。素朴な独立ペアの弱点（直線の交差）に対処する改良です。ミニバッチ内で x_0 と x_1 を最適輸送で対応付けてからペアを作ると、交差が減って軌跡がより直線的になり、生成がさらに速く安定すると示しました。条件付き Flow Matching を OT カップリングへ一般化した点が貢献です。

## 論文の実験結果（定量データ）

Lipman らの ICLR 2023 論文で最も示唆的なのは「パスの選び方」のアブレーションです。同じ Flow Matching の枠組みの中で、(a) 拡散と同じ分散保存パス、(b) 提案の条件付き OT 直線パス、を ImageNet などで比較すると、(b) の方が学習損失が安定し、少ない関数評価回数（NFE, ネットを呼ぶ回数）で良い FID（生成品質の指標、小さいほど本物に近い）を達成しました。報告によれば、同じ品質に到達する NFE が分散保存パスより大幅に少なくて済む領域があり、これは「直線パスはオイラー法で粗く刻んでも誤差が溜まりにくい」という直感と一致します。

Rectified Flow（Liu et al.）は CIFAR-10 で、直線化を突き詰めた reflow と蒸留により **1 ステップ生成（NFE=1）で FID ≈ 4 台**という当時驚異的な値を報告しました。通常の拡散が数十〜数百ステップを要したことを思えば、直線パスの威力が定量的に分かります。アブレーションとしては、reflow の回数を増やすほど軌跡がまっすぐになり（直線性の指標が下がり）、少 NFE での FID が改善することが示されています。ただし reflow ごとに再学習・再サンプリングのコストがかかるトレードオフがあります。

Tong らのミニバッチ OT 版は、2 次元のトイデータや画像生成で、独立ペア版より生成軌跡が直線に近づき、同じ NFE での生成品質（2-Wasserstein 距離や FID）が改善すると報告しました。アブレーションとして「OT で対応付ける」工程を抜くと交差が増え軌跡が曲がる、すなわちこの工程が直線性に効いていることが確認されています。トイデータで輸送の交差本数を直接数えると、OT カップリングで明確に減ることが可視化されています。

## メリット・トレードオフ・限界

メリット。速度が時刻 t に依らず一定 x_1 − x_0 で、学ぶ対象が単純です。学習が安定し、生成の積分ステップを減らせます（高速）。実装が数行で済みます（線形補間と引き算だけ）。分散保存パスと違いノイズスケジュールの細かい設計が要らず、パイプラインが単純になります。

トレードオフと限界。独立ペアの直線は互いに交差しうるため、推論時の実際の流れ（周辺速度場）は完全な直線にならず、あくまで近似です。分布が複雑な場合、曲がった輸送やミニバッチ OT の方が良い結果になることがあります。ミニバッチ OT 自体もバッチ内に閉じた近似で、バッチサイズが小さいと真の OT から外れます（割り当て計算のコストも O(B² 〜 B³) で増えます）。直線パス自体は条件 c（シーン情報など）を使わない単純な補間なので、条件の効きは速度場ネットの表現力に依存します。未解決の課題として、データに応じて最適なパスをどう曲げるべきか、分散保存パスと直線パスのどちらが下流タスク（生成品質だけでなく制御・計画など）で有利かは、いまも研究が続いています。

## 発展トピック・研究の最前線

直線パスを起点に、研究は二方向へ伸びています。一つは「さらなる直線化・高速化」で、reflow・蒸留・整合性蒸留（consistency distillation）で 1〜数ステップ生成を狙う流れです。もう一つは「より良い対応付け」で、ミニバッチ OT に加え、データ依存のスケジューリングや、条件 c を考慮したペアリング（条件ごとに OT を解く）で軌跡品質を上げる研究です。さらに、Schrödinger Bridge（確率的な OT、エントロピー正則化付き輸送）と Flow Matching を結びつけ、ノイズだけでなく任意の二分布間（画像→画像変換など）を直線的につなぐ研究も活発です。応用面では、大規模画像生成モデルが内部でこの直線（Rectified Flow）定式化を採用しており、理論的に単純なパスが実用の主流になっています。
