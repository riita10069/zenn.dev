# Swin Transformer (V1)

## ひとことで言うと
Swin Transformer は、画像を小さな窓(ウィンドウ)に区切って、その窓の中だけで自己注意(画像のどの部分どうしが関係が深いかを計算する仕組み)を行うことで、画像用 Transformer の計算量を「画像サイズの2乗」から「画像サイズに比例(線形)」へ一気に下げた手法です。さらに窓の区切り位置を1層おきにずらす「シフト窓(shifted window)」によって、窓と窓の境目をまたいだ情報も少しずつ伝わるようにしました。「Swin」という名前自体が Shifted WINdow の略です。物体検出やセマンティックセグメンテーションといった、画像内の位置が重要なタスクの標準バックボーン(特徴抽出の土台)として広く使われ、後継の SwinV2 の前身でもあります。

## 直感的な理解
人が大きな絵を観察するとき、画面全体のすべての点を同時に見比べることはしません。まず手元の一区画をじっくり見て、次に隣の区画に目を移し、それを繰り返しながら全体像を組み立てます。Swin の発想はこれに近いものです。「全部の点どうしの関係を一度に計算する」のではなく、「小さな区画(窓)の中だけで関係を計算し、層を重ねるうちに少しずつ視野を広げていく」という戦略を取ります。

なぜこんな工夫が要るのか。それは Transformer の自己注意が抱える計算量の問題のためです。自己注意は「すべての要素どうしの関係」を一度に計算します。文章なら単語は数百から数千個なので何とかなりますが、画像では要素=ピクセル(または小さなパッチ)の数が非常に多くなります。たとえば 224x224 の画像を 4x4 のパッチに切ると 56x56=3136 個のトークンになり、その全組み合わせ(3136 の2乗 ≒ 約 1000 万通り)の関係を計算することになります。解像度が上がるとこれが面積の2乗で爆発します。Swin はこれを「窓の中だけ」に限定することで、現実的な計算量に抑えました。

## 基礎: 前提となる概念
このトピックを理解するために、いくつかの言葉を噛み砕いておきます。

- 自己注意(self-attention)とは、入力の各要素が「他のどの要素にどれだけ注目すべきか」を表す重みを計算し、その重みで他要素の情報を集めて自分を更新する仕組みです。各要素から Query(問い合わせ)・Key(鍵)・Value(値)という3つのベクトルを作り、Query と Key の内積で「関連度(注意スコア)」を測り、softmax で正規化した重みで Value を加重平均します。数式では Attention(Q,K,V) = softmax(QKᵀ/√d)V で、Q,K,V は (N, d) 行列(N=トークン数、d=次元)、QKᵀ は (N, N) のスコア行列です。これは「文章のこの単語を理解するには、離れたあの単語が重要だ」という長距離の関係を一発で捉えられるのが強みです。
- 計算量がトークン数の2乗になる、とは、N 個の要素があると注意スコア行列が N×N 個できるという意味です。N が2倍になると計算は4倍になります。画像のように N が大きいと致命的です。
- マルチヘッド注意(multi-head)とは、上記の注意を複数の「ヘッド」に分けて並列に行い、各ヘッドが異なる種類の関係(色の近さ、形の近さ、など)を学べるようにする仕組みです。
- Vision Transformer(ViT)とは、画像を 16x16 などのパッチに分割し、各パッチを1つのトークンとみなして素の Transformer に流す手法(Dosovitskiy et al., ICLR 2021)です。JFT-300M 級の超大量データでは CNN を上回りましたが、全域注意なので高解像度に弱く、出力が単一解像度の特徴しか持たないという欠点がありました。
- 階層的(マルチスケール)特徴とは、CNN が伝統的に持つ性質で、浅い層は細かい模様(エッジ)を高解像度で、深い層は大きな意味(物体全体)を低解像度で表す、という複数の粒度の特徴のことです。物体検出では小さい物も大きい物も検出するため、この多段の特徴がほぼ必須でした。素の ViT はこれを持っていませんでした。

Swin はこの「2乗の計算量」と「階層性の欠如」という2つの弱点を同時に解いたところに価値があります。

## 仕組みを詳しく
Swin の核心は2つの仕組み、W-MSA と SW-MSA です。

### 1. 窓ベースの自己注意(Window-based Multi-head Self-Attention, W-MSA)
特徴マップを重ならない MxM の窓(原論文では M=7、つまり 7x7)に区切り、自己注意を「窓の中だけ」で計算します。全域注意の計算量は画像のパッチ数 hw に対して概ね 2(hw)²C(C はチャンネル数)に比例しますが、窓内注意は 2M²(hw)C に比例します。窓サイズ M を固定すれば、(hw)² の項が M²(hw) に置き換わるので、これは hw(=画像の面積)に対して線形です。

数値で見ると違いは劇的です。56x56=3136 トークンの場合:
- 全域注意のスコア行列: 3136 x 3136 ≒ 約 980 万要素
- 7x7=49 トークンの窓が 64 個あり、各窓のスコア行列は 49 x 49 = 2401 要素。合計 64 x 2401 ≒ 約 15 万要素

同じ画像でも、窓に区切るだけで注意計算が約 60 分の1 になります。解像度を上げてもこの比は窓の数に応じて線形にしか増えません。

### 2. シフト窓自己注意(Shifted Window, SW-MSA)
窓の中だけで注意を取ると、窓の境界をまたぐ関係が捉えられません。隣の窓にある重要な手がかりが永遠に混ざらない、という問題が起きます。

そこで Swin は、層ごとに窓の区切り位置を窓サイズの半分(⌊M/2⌋)だけ右下にずらします。ある層は通常の窓割り(W-MSA)、次の層はずらした窓割り(SW-MSA)、と交互に並べます。こうすると、前の層では別々の窓にいた領域が次の層では同じ窓に入り、情報が窓をまたいで段階的に伝播します。層を重ねるほど実効的な受容野(一度に情報を集められる範囲)が広がり、最終的には全域に近い文脈を獲得できます。

ずらしたことで端に半端な小窓が生じます。これを愚直に処理すると窓の数が 2x2=4 個から 3x3=9 個へ増え、しかも大きさがバラバラになって非効率です。Swin は「巡回シフト(cyclic shift)」という工夫を使います。画像の上端・左端の数行/数列を下端・右端側へ巻き込んで(トーラス状に回転させて)から通常の窓割りを適用し、本来は隣り合っていない領域どうしが同じ窓に混ざらないように注意マスク(関係を計算しないよう特定の組のスコアを −∞ にして softmax で 0 にする)を掛けます。これで窓の数を元の 2x2 のまま保ったままシフトを実現します。

### 階層構造とパッチマージ
処理全体の流れは次の通りです。

```
入力 224x224x3
  │ Patch Partition (4x4ごとに分割し線形埋め込み)
  ▼
Stage1: 56x56, 96ch   ← Swin Block ×2
  │ Patch Merging (隣接2x2をまとめてチャンネル統合)
  ▼
Stage2: 28x28, 192ch  ← Swin Block ×2
  │ Patch Merging
  ▼
Stage3: 14x14, 384ch  ← Swin Block ×6 (Tiny。大型版はここが深い)
  │ Patch Merging
  ▼
Stage4: 7x7, 768ch    ← Swin Block ×2
```

パッチマージ(patch merging)は、隣接する 2x2 のパッチ(4個)を連結して 4C 次元にしてから線形層で 2C 次元に圧縮する操作で、CNN のプーリングに相当します。これで解像度が半分、チャンネルが倍になり、CNN の FPN(Feature Pyramid Network)と同じように4段のマルチスケール特徴が得られます。各サイズのステージ深さは Tiny が (2,2,6,2)、Small/Base が (2,2,18,2) で、第3ステージが最も厚いのは「意味的な処理の主戦場をそこに置く」という Swin の設計思想です。Tiny 版の段ごとのチャンネルは 96, 192, 384, 768、Base 版は 128, 256, 512, 1024 です。各 Swin Block は「W-MSA → MLP → SW-MSA → MLP」を1セットとし、それぞれに LayerNorm と残差接続を伴います(原論文は注意・MLP の前に正規化する Pre-Norm 方式)。

相対位置バイアス(relative position bias)も重要な部品です。窓内の各位置ペアについて「縦横にどれだけ離れているか」を表す学習可能なバイアス項を、softmax 前の注意スコアに足します。Attention = softmax(QKᵀ/√d + B) の B がこれです。M×M の窓なら相対位置の取りうる値は (2M−1)×(2M−1) 通りで、その分の小さなバイアステーブルを学習します。これは「絶対座標」ではなく「相対的なずれ」だけを符号化するので、画像内のどこに窓があっても同じ意味を持ち、並進に対して頑健になります。ただしこのバイアステーブルは特定の窓サイズに紐づくため、別解像度・別窓サイズへ転用すると双線形補間が必要になり、これが後の V2 で改善される弱点になります。

## 手法の系譜と主要論文
- 2017年(NeurIPS): Vaswani らの "Attention Is All You Need"(arXiv:1706.03762)が Transformer を提案。自己注意で系列を処理する枠組みが自然言語処理を席巻しました。
- 2021年(ICLR): Dosovitskiy らの ViT(arXiv:2010.11929)が、画像をパッチ列とみなして素の Transformer に流すことで、JFT-300M 級の超大規模データでは CNN を上回ることを示しました。ただし高解像度・密なタスク(検出・分割)には不向きで、データ効率も悪い(中規模データでは ResNet に負ける)という限界がありました。
- 2021年(ICCV): Liu, Lin, Cao, Hu らの Swin Transformer(arXiv:2103.14030、ICCV 2021 Marr Prize / Best Paper 受賞)が本トピックの原典です。窓ベース注意・シフト窓・階層構造を導入し、Transformer を分類専用から「汎用視覚バックボーン」へ押し上げました。同時期の DeiT(Touvron et al., ICML 2021)は蒸留でデータ効率を改善する別路線で、Swin は「アーキテクチャ側で密タスク対応する」路線です。
- 2022年(CVPR): 同チームの SwinV2([Swin Transformer V2 (SwinV2)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0363_swin-v2-tiny))が、V1 を30億パラメータ級・高解像度へスケールしたときの「学習不安定」「解像度変更で性能低下」を、scaled cosine attention と対数間隔の連続相対位置バイアスで解決しました。
- 同時期(CVPR 2022): Liu らの ConvNeXt([ConvNeXt (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0373_convnext))は、Swin の設計思想を逆に純 CNN へ移植し、「Swin の強さは自己注意そのものより設計の積み重ねによる部分が大きい」ことを示しました。Swin と ConvNeXt は CNN と Transformer の収斂を象徴する一対です。

## 論文の実験結果(定量データ)
評価に使われた主要ベンチマークと指標の意味を押さえます。

- ImageNet-1K 画像分類: 約128万枚・1000クラスの画像を正しく分類する精度(top-1 accuracy、最も確からしい予測が正解と一致する割合)。Swin-T(Tiny、約 2900 万パラメータ、計算量約 4.5 GFLOPs)は top-1 約 81.3%、Swin-S(約 83.0%)、Swin-B(Base、約 8800 万パラメータ)は約 83.5%。同規模の DeiT-S(約 79.8%)や RegNet を明確に上回りました。ImageNet-22K で事前学習した Swin-B/L を 384x384 で評価すると約 86〜87% に達します。
- COCO 物体検出: 画像中の物体の位置を矩形で当てるタスク。指標は box AP(Average Precision、検出枠と正解枠の重なり度合い IoU を複数しきい値で平均したもの。高いほど良い)。Swin-T をバックボーンにした Cascade Mask R-CNN は、同規模の ResNet-50 比で box AP を約 3〜4 ポイント引き上げました。Swin-L を用いた当時の最良構成は COCO test-dev で box AP 約 58.7、mask AP(インスタンス分割の精度)約 51.1 を達成し、当時の最高記録を更新しました。
- ADE20K セマンティックセグメンテーション: 画像の全ピクセルに意味カテゴリを付けるタスク。指標は mIoU(mean Intersection over Union、予測領域と正解領域の重なりをクラス平均したもの)。Swin-L は val mIoU 約 53.5 を達成し、こちらも当時の最高水準でした。

アブレーション(どの要素を抜くとどうなるか)も明快です。シフト窓を外して通常の窓割りだけにすると、ImageNet 分類で約 1.1 ポイント、COCO 検出で box AP 約 2.8、ADE20K で mIoU 約 2.8 ほど低下したと報告されています。窓をまたぐ情報伝達がいかに効いているかが分かります。また、相対位置バイアスを使う方が、位置符号化なし(約 −1.2 ポイント)や絶対位置埋め込み(約 −0.4 ポイント)より一貫して高精度でした。これらの数値の意味は「単に窓に区切るだけでは不十分で、シフトと相対位置という2つの工夫が性能の本体である」ということです。

## メリット・トレードオフ・限界
メリット:
- 窓分割で計算量が画像サイズに線形になり、高解像度を現実的に扱えます。自動運転のような高解像度・多カメラ入力に向きます。
- シフト窓で窓をまたぐ長距離関係も段階的に捉えられ、純粋な窓内注意の弱点を補います。
- 階層的マルチスケール特徴を出すので、検出・分割・各種知覚タスクのバックボーンとして CNN と同じ感覚で差し替えられます。FPN や検出ヘッドとの接続が自然です。

トレードオフ・限界:
- 大規模化すると学習が不安定になりやすい(これが V2 で改善されました)。注意スコアが内積依存のため、特定の位置に注意が集中して活性が暴れる傾向があります。
- 学習時と異なる解像度では相対位置バイアスの補間が必要で、性能が落ちやすい(V2 で改善)。
- 窓・パッチに割り切れる入力サイズが必要で、シフト窓の巡回シフトとマスク処理は実装が複雑です。GPU 上ではマスク付き窓注意のメモリアクセスが不規則になり、理論 FLOPs ほどの速度が出ないこともあります。
- 小〜中規模データでは、CNN の持つ局所性・並進不変性という帰納バイアスがない分、過学習しやすい場面があります。十分なデータか強い事前学習が前提になりがちです。

## 発展トピック・研究の最前線
Swin の登場後、視覚 Transformer は「効率的な注意」をめぐって多様化しました。Twins や PVT(Pyramid Vision Transformer)は窓ではなく空間的に間引いた注意を、Focal Transformer は近くは細かく遠くは粗く見る多粒度注意を提案しました。CSWin Transformer は十字型の窓で受容野を広げる工夫を示しました。また、注意そのものを使わずに大カーネル畳み込みで同等の受容野を得る方向(ConvNeXt、RepLKNet の 31x31 カーネル)も現れ、CNN と Transformer の境界は曖昧になっています。検出・分割では、Swin をバックボーンに据えた DETR 系・Mask2Former が標準化しました。自動運転の文脈では、複数カメラの特徴を鳥瞰図(BEV)空間へ持ち上げる手法(BEVFormer など)のバックボーンとして Swin / SwinV2 が頻繁に使われ、マルチスケール特徴の出しやすさがそのまま活きています。研究上の未解決課題としては、窓注意の固定的な分割が動的なシーン構造に最適かどうか、そして車載のような制約環境で注意のメモリ非効率をどう克服するかが残っています。
