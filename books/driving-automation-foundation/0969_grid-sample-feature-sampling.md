# grid_sampleによる特徴サンプリング

## ひとことで言うと
画像の特徴マップから「ちょうど整数ピクセルの上ではない、中間の位置」の値を滑らかに取り出す操作が、双線形サンプリング(bilinear sampling)です。例えば「横 3.7 番目・縦 5.2 番目のピクセルの値」を、近くの4つのピクセルの値を距離に応じて混ぜ合わせて計算します。深層学習フレームワークでは grid_sample という関数がこれを担います。鳥瞰図(BEV)を作るとき、3次元の参照点を画像に投影した小数位置から画像特徴を拾う、変換処理の中核の道具です。

## 直感的な理解
3次元の点をカメラ画像に投影すると、得られるピクセル座標は (3.7, 5.2) のような小数になります。一方、画像特徴マップは格子状(整数位置)にしか値を持ちません。「(3.7, 5.2) の特徴」は、そのままでは存在しないのです。

素朴な対処は「いちばん近い整数に丸める(最近傍、nearest)」ですが、2つの問題があります。第一に、丸めると位置が少しずれるたびに拾う値がカクッと飛び、入力をわずかに動かしただけで特徴が不連続に変わります。第二に、より致命的なことに、丸め操作は微分できません。深層学習は誤差逆伝播(出力の誤差を入力側へ伝えて重みを更新する仕組み)のために「サンプル位置に関して微分可能」であることを強く要求します。投影位置を学習で動かしたい設計では、ここが微分不能だと学習が成立しません。

そこで「滑らかに、かつ微分可能に、中間位置の値を取り出す」仕組みが要ります。それが双線形補間であり、grid_sample はこれをバッチ・GPU 対応で提供する標準関数です。地図を拡大したときに、ドット絵のようにカクカクさせず、なめらかに色を補間して表示するのと同じ発想です。

## 基礎: 前提となる概念
特徴マップ(feature map): 畳み込みネットワークが画像から抽出した、チャンネル次元を持つ格子状のデータです。形は [C, H, W](チャンネル数、縦、横)で、各格子点が C 次元のベクトルを持ちます。

補間(interpolation): 既知の点の間の未知の点の値を推定することです。線形補間は2点を直線で結んで間を読みます。双線形補間は、これを縦横の2方向に適用して、2次元格子の間を読みます。

微分可能性(differentiability): ある操作の出力が入力に対してなめらかに変化し、勾配(微小変化の比)が定義できる性質です。双線形補間の重みは座標の1次式なので、出力をサンプル座標で微分でき、勾配が座標まで流れます。これにより「どこをサンプルすべきか」自体を学習できます。

正規化座標(normalized coordinates): grid_sample はサンプル位置をピクセル単位ではなく [-1, 1] の範囲で受け取ります。-1 が左端/上端、+1 が右端/下端、0 が中央です。解像度に依存しない座標系なので、画像サイズが変わっても同じ意味を保てます。

## 仕組みを詳しく
双線形補間の中身を分解します。取り出したい位置 (3.7, 5.2) のまわりの4つの整数ピクセル (3,5), (4,5), (3,6), (4,6) を取り、横方向の小数部 0.7 と縦方向の小数部 0.2 で重み付け平均します。位置に近いピクセルほど重みが大きくなります。横 0.7 なら左ピクセルに 0.3、右ピクセルに 0.7 を割り当て、これを縦方向にも適用して4点を混ぜます。位置を連続的に動かすと値も連続的に変わり、各重みは座標の単純な1次式なので微分できます。

grid_sample のインターフェースは次のとおりです。
- 入力特徴 input: 形 [N, C, H_in, W_in](バッチ、チャンネル、縦、横)。
- サンプル位置 grid: 形 [N, H_out, W_out, 2]。最後の 2 は (x, y) で、範囲は [-1, 1] に正規化済みであること。
- 出力: 形 [N, C, H_out, W_out]。grid の各位置で input を補間した値。

BEV 構築での典型的な使い方は次のように書けます。

```
sample_loc  = proj_uv + offset          # [B, N, P, 2]、投影位置 [0,1] に学習オフセットを加算
sample_grid = sample_loc * 2 - 1        # [0,1] を [-1,1] へ線形変換
sampled     = grid_sample(feat_v, sample_grid,
                          mode='bilinear',
                          padding_mode='zeros',
                          align_corners=False)   # [B, C, N, P]
```

ポイントを分解します。
1. sample_loc * 2 - 1: 投影で得た [0,1] 座標を grid_sample が要求する [-1,1] に直します。0→-1、0.5→0、1→+1。
2. mode='bilinear': 双線形補間を使う(微分可能・滑らか)。代替の nearest は不連続で勾配が流れません。
3. padding_mode='zeros': サンプル位置が枠外に出たら、その点の値を 0 として扱います。これは可視性マスキング([可視性マスキング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/1027_visibility-masking-bev))と合わせて「枠外は寄与ゼロ」を実現します。border(端の値で埋める)や reflection(折り返す)もありますが、枠外を無効化したい BEV 用途では zeros が自然です。
4. align_corners=False: ピクセル中心を基準にする座標解釈(近年のフレームワークの推奨設定)。True と False で座標とピクセルの対応が半ピクセルずれます。取り違えると特徴全体が半ピクセルずれて精度が落ちるため、投影座標の計算と一貫させる必要があります。

形の流れ(shape tracking)を追います。
- feat_v: [B, C, H, W]、あるカメラ v の画像特徴。
- sample_grid: [B, N, P, 2]、BEV マス N 個・ピラー点 P 個のサンプル位置。grid_sample は grid の (H_out, W_out) 次元を (N, P) に対応づけます。
- 出力 sampled: [B, C, N, P]。各 BEV マス・各ピラー点に対し、その投影位置の C 次元特徴が補間で得られます。

この後、sampled を [B, N, P, C] に並べ替え、可視点で再正規化した注意重みを掛けてピラー方向に合計し([B, N, C])、カメラ横断で平均して BEV 特徴を作ります。grid_sample は「正しい位置の特徴を微分可能に取り出す」役割だけを担い、重み付けや集約はその外側で行われます。

数値例: 256×256 の特徴で、前方の点が正規化 (0.5, 0.5) に投影されたとします。*2-1 で (0, 0)、これは画像のど真ん中で、grid_sample は中央4ピクセルを補間して特徴を返します。点が (1.1, 0.5) すなわち右に飛び出すと、*2-1 で (1.2, 0)、これは [-1,1] の外なので padding_mode='zeros' によりゼロが返り、その点は寄与しません。

## 手法の系譜と主要論文
Spatial Transformer Networks(STN, Jaderberg et al., NeurIPS 2015, arXiv:1506.02025)が、微分可能な双線形サンプリングを深層学習に持ち込んだ元祖です。動機は、ネットワークが空間変換(回転・拡大・位置合わせ)を自分で学習できるようにすること。そのためにはサンプリング自体が微分可能でなければならず、双線形サンプラー(differentiable image sampling)を定式化しました。位置に関して勾配が流れ、変換パラメータを end-to-end で学習可能にした点が新規性です。grid_sample という標準関数は、この STN のサンプラーをそのまま実装したものです。

Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)は、各クエリが画像の少数のサンプル点だけに注意を向ける可変形アテンション(deformable attention)を提案しました。サンプル点の位置オフセットと注意重みをクエリから予測し、その位置の特徴を双線形補間で取り出します。全画素への注意に比べて計算量を大きく削減しつつ、物体に合わせて適応的に位置をずらせます。

BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)は、3次元参照点を画像に投影した位置から特徴を取り出すのに、この可変形アテンションを使います。複数のサンプル点と位置オフセットを学習し、双線形補間で特徴を拾う点で、grid_sample の発展形です。物体の形に合わせてサンプル位置を適応的にずらせるのが利点ですが、専用 CUDA カーネルを要するなど実装が複雑になります。多くの簡略実装は、フル可変形アテンションの代わりに「grid_sample + 学習オフセット」で近似し、表現力を多少犠牲にしつつ実装を単純化します。

整理すると、STN(微分可能サンプラーの発明)→ Deformable DETR(疎なサンプル点で高速化)→ BEVFormer(BEV 投影への応用)という系譜であり、その全段の基礎が双線形サンプリングです。

## 論文の実験結果(定量データ)
STN は MNIST の歪み・回転を加えた変種(distorted MNIST)で、サンプラーを差し込むだけで誤り率を明確に下げました。報告によると、空間変換を学習することで、通常の CNN より分類誤差が低減し、入力の幾何変形に対する頑健性が向上しました。これは「サンプリングが微分可能だと、ネットワークが入力の位置合わせを自力で学べる」ことの実証です。

Deformable DETR は COCO 物体検出(80カテゴリの標準ベンチマーク、指標は mAP)で、元の DETR に対し収束を約10倍速め(DETR の 500 エポックに対し約 50 エポック)、小物体の AP を改善しました。報告される mAP はおよそ 43〜46 の範囲で、全画素注意の DETR を上回りつつ計算量を抑えました。疎なサンプル点でも双線形補間で十分な特徴が取れることを示した点が重要です。

BEVFormer は nuScenes で3次元物体検出の NDS 約 0.569、mAP 約 0.481(test split、報告値)を達成しました。可変形アテンションによる投影サンプリングが、固定格子からの最近傍サンプリングに比べて精度を押し上げることが、アブレーションで示されています。サンプル位置のオフセット学習を無効化すると、物体の形状に追従できず検出精度が低下する傾向が一般に報告されています。

これらが共通して示すのは、サンプリングを「微分可能・適応的」にすること自体が、最終精度に直結する設計要素だということです。双線形補間という古典的な道具が、現代の知覚モデルの土台に居続けている理由がここにあります。

## メリット・トレードオフ・限界
メリット:
- 微分可能。サンプル位置(学習オフセット込み)に勾配が流れ、end-to-end 学習ができる。
- 滑らか。位置を連続的に動かすと特徴も連続的に変わり、学習が安定する。
- 標準関数として GPU 最適化済みで、実装が簡単。
- padding_mode='zeros' で枠外を自然にゼロ扱いでき、可視性マスキングと相性がよい。

トレードオフと限界:
- 双線形は近傍4点のみを使うため、細かいエッジや高周波な模様はぼける。シャープさが要る用途では精度の上限になる。
- 入力特徴の解像度が低いと、補間しても元情報以上の細かさは出ない。投影サンプリングの精度は元の特徴解像度に縛られる。
- align_corners や正規化方法を取り違えると半ピクセルずれる落とし穴がある。投影座標の計算規約と必ず一致させる必要がある。
- フル可変形アテンション(複数ヘッド・複数サンプル点・適応オフセット)に比べると、単純な grid_sample は表現力が劣る。
- 未解決の論点として、双線形より高次の補間(bicubic)や学習可能な補間カーネルが精度を上げうるが、計算コストとのバランスは用途依存です。

## 発展トピック・研究の最前線
可変形アテンションは Deformable DETR 以降、検出・セグメンテーション・BEV 構築の標準部品になりました。マルチスケール特徴(複数解像度の特徴マップ)からのサンプリングを組み合わせると、大小さまざまな物体に追従しやすくなります。

3次元への拡張では、BEV 平面だけでなく3次元ボクセル空間や球面・円筒座標へのサンプリングが研究されています。占有予測やマルチモーダル融合(カメラと LiDAR の特徴を共通空間でサンプリングして統合)では、座標変換とサンプリングの精度が直接性能に効きます。

実装面では、半精度学習でのサンプリング座標の精度劣化、専用カーネルによる高速化、サンプリング点数とメモリのトレードオフが実務上の論点です。サンプル点を増やすほど表現力は上がりますが、メモリと速度が線形に悪化するため、点数の最適化はモデル設計の重要な調整項目になります。
