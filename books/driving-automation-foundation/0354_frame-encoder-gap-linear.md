# FrameEncoder (GAP+Linear埋め込み)

## ひとことで言うと
1フレーム分の画像から取り出した特徴マップ(縦・横・チャンネルを持つ立体データ)を、Global Average Pooling(大域平均プール、全空間の平均をとる操作)で1本のベクトルに潰し、線形層(Linear)で目的の次元に整える、フレーム単位の埋め込みを作る最小実装です。1枚の画像(または1時刻ぶんの複数カメラ)を、扱いやすい固定長ベクトルに変換する最も素直な方法で、深層学習の分類ネットワークの末尾で長年使われてきた定番の構造です。

## 直感的な理解
本のページを丸ごとコピーするのではなく「このページの要点を1文でメモする」ことを想像してください。要点メモは短く、後で並べて見返すのに便利です。ただし1文に縮める以上、細かい記述は消えます。

ニューラルネットでも同じことが起きます。画像を backbone(画像エンコーダ)に通すと `[256, 8, 8]` のような特徴マップが出ます。これは「8×8の各位置に256次元の特徴が乗った」立体データで、情報は豊富ですが、そのまま時系列に並べたり経路計画モジュールに渡すには大きすぎます。`[N, 256, 8, 8]` を時間方向に並べて attention や GRU に通すのは重い。

そこで「ページを1文にメモする」=「空間(8×8)を平均して1本のベクトルにする」圧縮をします。これが GAP(Global Average Pooling)です。あとは線形層で次元を目的のサイズに揃える。これだけでフレーム単位の埋め込みができます。軽く、頑健で、実装は数行です。下流の時系列モジュールやマルチビュー集約は、この「フレーム1本のベクトル」を入力として受け取ります。

## 基礎: 前提となる概念
特徴マップ (feature map)。画像エンコーダの出力 `[C, h, w]`。`C` はチャンネル数(特徴の種類)、`h×w` は空間グリッド。各マスに `C` 次元のベクトルが乗っています。CNN なら畳み込みの出力、Vision Transformer ならパッチトークンを並べ直したものに相当します。

プーリング (pooling)。空間方向に値をまとめて解像度を下げる操作です。代表は平均プーリング(平均をとる)と最大プーリング(最大値をとる)。「Global」が付くと、空間全体を1点(1×1)に潰すことを指します。

GAP (Global Average Pooling, 大域平均プーリング)。特徴マップの全空間位置をチャンネルごとに平均し、`[C, h, w]` を `[C]` にします。式で書くと、チャンネル `c` の出力は `z_c = (1/(h·w)) Σ_{i,j} feat[c,i,j]` です。これは「チャンネル c の特徴マップ全体の平均値」という意味です。`[256, 8, 8]` → 平均 → `[256]`。学習パラメータはゼロ(ただの平均)です。利点は3つ。(1) パラメータがないので軽く過学習しにくい、(2) 入力の空間サイズが多少変わっても出力次元は `C` のまま一定、(3) 全位置を平等に見るので一部のノイズに振り回されにくい。

GMP (Global Max Pooling, 大域最大プーリング)。平均の代わりに最大値をとります。GAP が「シーン全体の傾向」を捉えるのに対し、GMP は「最も強く反応した1か所」を捉えます。小さく重要な対象を拾いたいときは GMP、全体の要約が欲しいときは GAP、と用途で使い分けます。

線形層 (Linear, 全結合層)。入力ベクトルに重み行列を掛けてバイアスを足す `y = Wx + b` の変換です。次元を変える(たとえば1440→256)役目を担います。`W` のサイズは(出力次元×入力次元)で、ここが学習する重みです。`b` はバイアス(出力次元ぶんの定数ベクトル)です。

## 仕組みを詳しく
最小実装はこうなります。記号は B=バッチ、C_in=backbone 出力チャンネル、D=出力埋め込み次元。

```python
class FrameEncoder(nn.Module):
    def __init__(self, in_channels=1440, embed_dim=256):
        super().__init__()
        self.gap    = nn.AdaptiveAvgPool2d(1)        # 任意のh,w → 1×1 に平均
        self.linear = nn.Linear(in_channels, embed_dim)

    def forward(self, feat):                          # feat: [B, C_in, h, w]
        x = self.gap(feat)                            # [B, C_in, 1, 1]
        x = x.flatten(1)                              # [B, C_in]
        x = self.linear(x)                            # [B, embed_dim]
        return x
```

形(shape)を数値で追います。backbone が `[B, 1440, 8, 8]` を出すとします(多くの実装はマルチスケール特徴を連結して数百〜千超のチャンネルになります)。

1. `gap(feat)` … 8×8の64マスを平均し `[B, 1440, 1, 1]`。学習パラメータなし。
2. `flatten(1)` … 末尾の 1×1 を畳んで `[B, 1440]`。
3. `linear(...)` … 1440 → 256 に射影し `[B, 256]`。ここだけが学習する重み(1440×256 ≈ 37万パラメータ)。

before/after で言うと、入力は「8×8の位置に散らばった1440次元特徴=81920個の数値/サンプル」、出力は「1本の256次元ベクトル=256個の数値/サンプル」。約320分の1に圧縮しています。

なぜ GAP なのか。歴史的には、CNN の最後に巨大な全結合層を置いて分類していました。たとえば `7×7×512=25088` 次元を全結合に入れると重みが数千万個になり、過学習の温床でした。GAP はこの全結合を「チャンネルごとの平均」で置き換え、パラメータを激減させます。さらに「あるチャンネルの平均活性=あるクラスらしさ」という対応がつくため、解釈もしやすくなりました(後述の Network in Network)。

1×1 Conv という代替。Linear の代わりに 1×1 畳み込み(各位置で同じ重みを掛ける Conv)を使う実装もよくあります。`Conv2d(1440, 256, kernel_size=1)` は GAP 前に各位置でチャンネルを256へ射影し、その後に GAP をかける順序です。GAP→Linear と、Conv1×1→GAP は、どちらも線形演算なので、間に非線形(活性化関数)を挟まなければ数学的にはほぼ等価です。GELU など活性化を挟む位置で振る舞いが変わります。

マルチビューでの使われ方。各カメラの特徴マップを GAP で1本ずつにすると `[B, V, C]` のトークン列ができます。これは [ViewAttentionPool (マルチビュー集約)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0319_view-attention-pool) の入力に相当し、ViewAttentionPool 内部の `mean(dim=(2,3))` がまさにこの GAP の役割を果たします。FrameEncoder の Linear は、backbone のチャンネル数を統一の埋め込み次元へ揃える前処理です。

出力次元の選び方。224・256・512 などいくつかの選択肢が現場で議論されます。重要なのは数値そのものより「GAP で空間を畳んで Linear で目的次元に揃える」という構造です。下流モジュール(経路計画・時系列集約など)が期待する入力次元に合わせるのが基本で、互換性のために既存の次元を踏襲するか、すっきりした次元へ作り直すかはトレードオフになります。次元が大きいほど表現力は上がるが、下流の計算とパラメータも増えます。

## 手法の系譜と主要論文
Network in Network (Lin et al., ICLR 2014, arXiv:1312.4400)。GAP を最初に提案した論文です。従来の「畳み込み→巨大な全結合層→分類」を「畳み込み→GAP→分類」に置き換えました。動機は「全結合層はパラメータが多く過学習しやすい」こと。新規性は、最終特徴マップの各チャンネルを平均してそのままクラススコアにする「カテゴリごとの信頼度マップ」という解釈を与えた点です。あわせて 1×1 畳み込み(mlpconv)も導入し、後のネットワーク設計に大きな影響を与えました。

ResNet (He et al., CVPR 2016, arXiv:1512.03385)。残差接続で超深層を学習可能にした金字塔ですが、構成としては最終段で GAP→1つの線形分類層を標準採用しました。以降「畳み込み特徴→GAP→全結合」がデファクトの構成になります。GAP+Linear はこの系譜の最小形です。

Vision Transformer 系のプーリング論争 (Touvron et al. DeiT, ICML 2021, arXiv:2012.12877 ほか)。ViT は分類に CLS トークン(専用の集約トークン)を使いますが、その後の研究で「全トークンを GAP する方が CLS トークンと同等以上に強い、しかも単純」という結果が繰り返し示されました。GAP が追加パラメータなしで安定した表現を与える定番ベースラインであることを再確認した流れです。

GeM (Radenović et al., TPAMI 2018, arXiv:1711.02512)。Generalized Mean Pooling。平均と最大の中間を1つのパラメータ `p` で連続的に補間するプーリングで、`z_c = ((1/(h·w)) Σ feat[c,i,j]^p)^(1/p)` と定義されます。`p=1` で平均、`p→∞` で最大に一致します。画像検索で平均・最大より良い結果を出し、`p` 自体も学習できます。GAP の自然な一般化です。

自己教師あり学習でのプール vs マップ (Assran et al. I-JEPA, CVPR 2023, arXiv:2301.08243 ほか)。表現の目標(ターゲット)として「空間マップ全体」を使うか「プール済みベクトル」を使うかは、性能と計算量のトレードオフになります。プール済みは軽いが空間情報を捨て、マップ全体は重いが豊か。GAP+Linear は「pooled(プール済み)」側の最も軽い実装です。

## 論文の実験結果(定量データ)
Network in Network。GAP の導入により、CIFAR-10 で当時としては低い誤り率(およそ10%前後)を達成し、全結合層を使う構成より過学習が抑えられることを示しました。論文の主張の核は「全結合層を取り除いても GAP で精度を保てる、むしろ汎化が良くなる」ことで、パラメータ数の大幅削減を伴います。これが GAP が広く採用される決定打になりました。

ResNet。ImageNet で、GAP+線形分類の構成のまま、層を深くするほど精度が上がることを示しました(ResNet-152 で top-5 誤り率およそ3.57%、当時の ILSVRC 優勝水準)。GAP がボトルネックにならず、深層化の恩恵をそのまま受けられることの実証です。

DeiT / GAP vs CLS。ImageNet で、ViT 系の最終集約を CLS トークンから GAP に変えても精度はほぼ同等(報告により top-1 で0.1〜0.5ポイント程度の差、しばしば GAP が同等以上)で、実装は単純になることが示されました。アブレーションの教訓は「集約の方式は思ったほど性能を左右せず、GAP で十分なことが多い」点です。

GeM。画像検索ベンチマーク(Oxford/Paris)で、GeM プーリングは平均・最大プーリングを上回る検索精度(mAP)を出し、`p` を学習可能にすることでさらに改善することが報告されています。GAP を1パラメータ拡張するだけで表現力を足せる好例です。

I-JEPA。ImageNet 線形評価(linear probe、凍結した特徴の上に線形分類器だけ学習して測る評価)で、画像の一部から別の一部の表現を予測する自己教師あり学習が、対比学習に匹敵する精度(報告で linear probe およそ70%台)を、生成的手法より少ない計算で達成しました。ここでターゲットを空間マップにするかプール済みにするかは効率と精度の調整点で、軽量重視ならプール済みが選ばれます。

これらが伝えるのは「GAP は安価で頑健、多くのタスクで複雑な集約に劣らない」という経験則です。一方で位置が決定的なタスク(検出・セグメンテーション)では、GAP で空間を捨てると不利になります。

## メリット・トレードオフ・限界
メリット
- GAP は学習パラメータゼロで軽く、入力の空間サイズに頑健(可変解像度でも同じ次元が出る)。
- 過学習しにくく、実装が数行で済む。
- 出力が固定次元の1ベクトルなので、時系列処理や下流モジュールへの受け渡しが容易。

トレードオフ・限界・未解決課題
- 空間情報を平均で捨てるため「物体がどこにあるか」の位置情報は失われる。幾何的な空間理解(俯瞰での物体配置)はできない。
- 平均は外れ値(強い局所反応)を薄めるので、小さく重要な対象を見落とす可能性がある。位置や局所が重要なら GMP・GeM・空間マップ保持を検討。
- Linear で次元を揃える設計は1層ぶんの表現力しかない。backbone を凍結する構成では、フレーム埋め込みの賢さがこの Linear 1枚に縛られる。複雑な集約には [ViewAttentionPool (マルチビュー集約)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0319_view-attention-pool) のような注意機構の方が向く。
- 出力次元の選び方が下流の互換性に影響する。次元を変えると下流モジュールの作り直しが必要になりうる。

## 発展トピック・研究の最前線
- GeM プーリング。GAP を GeM に替えるだけで、平均と最大の中間を学習可能なパラメータで連続的に選べ、表現力を少し足せます。
- Attention pooling への置換。GAP の「全位置等重み」をやめ、学習可能なクエリで加重平均する([ViewAttentionPool (マルチビュー集約)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0319_view-attention-pool))と、重要な位置・カメラを選べます。軽さを失う代わりに表現力が上がります。
- 空間マップ保持との併用。下流が位置情報を要するなら、GAP ベクトルとは別に空間マップも渡すハイブリッド構成が使われます(I-JEPA の pool vs map 論争と同じトレードオフ)。
- 凍結 backbone + 軽量ヘッド。大規模事前学習 backbone を凍結し、GAP+Linear だけ学習する転移学習は、データが少ない領域で標準的な選択です。凍結すれば同じ画像の特徴をキャッシュでき、時系列学習が大幅に軽くなります。
