# 補助タスクによる勾配信号の強化

## ひとことで言うと

軌道を当てる主タスクの損失だけだと、モデルの「入口に近い部分」(画像から特徴を取り出す Backbone や、複数カメラの特徴を統合する Fusion)に届く学習信号が弱くなりがちです。そこで主タスクとは別に「未来の見え方を予測する」などの補助タスク(auxiliary task)を追加し、入口側のモジュールに直接の学習信号(勾配)を送り込む工夫が広く使われています。表現の質を底上げし、主タスクの性能を引き上げるのが狙いです。

「勾配(gradient)」とは、各パラメータを「どちらにどれだけ動かせば損失が減るか」を表す値で、これが学習の指示書になります。勾配が弱い・ぼやけていると、そのパラメータはほとんど更新されず学習が進みません。

## 直感的な理解

深いニューラルネットは、入力(カメラ画像)から出力(軌道)まで多数の層が直列につながっています。end-to-end 運転モデルなら大まかに次の流れです。

```
画像 → Backbone(特徴抽出)→ Fusion(複数カメラ/時刻の統合)→ … → Policy(軌道出力)
```

学習では、出力側で計算した損失を、層をさかのぼって入力側へ「勾配」として伝えます(誤差逆伝播, backpropagation)。ところが、損失が出力側にしかないと、入口に近い Backbone まで勾配が届くころには信号が薄まり(勾配消失に近い現象)、方向もぼやけます。「軌道が少しズレた、だから何層も前の Backbone の重みをどう直せばいいか」という指示が、遠すぎて曖昧になるのです。連鎖律で勾配は各層のヤコビアンの積になるため、層が深いほど積が小さくなったり方向が拡散したりしやすい、というのが数学的背景です。

その結果、Backbone や Fusion は「軌道予測に役立つ良い特徴」を十分に学べないことがあります。これらは複数の下流処理が共有する土台(共有表現, shared representation)なので、ここの質が低いとモデル全体の性能が頭打ちになります。

そこで、入口側のモジュールのすぐ近くに「別の学習目標(補助タスク)」を置きます。補助タスクの損失は入口に近いため、強くはっきりした勾配を Backbone に直接送れます。代表的な補助タスクは「いまの映像から、少し先の未来の見え方(特徴)を予測する」というものです。これは「シーンがこの先どう変化するかを予測できるだけの情報を、特徴の中に詰め込め」という指示になり、結果として土台の特徴の質が上がり、主タスクの軌道予測も改善します。

## 基礎: 前提となる概念

- 多タスク学習(multi-task learning): 1つのネットに複数の学習目標を同時に課すこと。主タスクと補助タスクで Backbone を共有すると、補助タスク由来の勾配も Backbone を鍛えます。共有部分(shared trunk)と各タスク専用の頭(head)に分かれる構造が典型です。
- 自己教師あり学習(self-supervised learning): 人手のラベルを使わず、データ自身から「正解」を作って学ぶこと。たとえば「未来のフレーム」はそのままデータに存在するので、追加アノテーション無しで「未来予測」という教師信号が作れます。
- 特徴マップ(feature map): 画像を Backbone に通して得られる、抽象化された格子状の表現。形は [B, C, H, W](バッチ, チャネル, 高さ, 幅)。生の画素よりも意味のある情報(物体の有無など)を保持します。
- 表現崩壊(representation collapse): 自己教師あり学習で、ネットが「すべての入力に対して同じ定数の特徴を出す」ことで損失をゼロにしてしまう病的な解。予測も正解も同じネットが作ると起きやすい。これを防ぐ非対称設計が重要になります。
- 深層教師(deep supervision): ネットの途中の層にも損失を付けて、中間層へ直接勾配を流す古典的テクニック。補助タスクの一般化された先祖にあたります。
- 負の転移(negative transfer): 補助タスクが主タスクと競合し、主タスク性能をかえって下げてしまう現象。補助タスクは常に効くわけではない、という重要な注意点です。

## 仕組みを詳しく

### 未来特徴を予測するモジュール

代表的な構成では、現在(時刻 t)の映像から「少し先の未来の特徴マップ」を複数予測するモジュールを置きます。たとえば +1.6 秒、+3.2 秒、+4.8 秒、+6.4 秒先の特徴をそれぞれ [B, C, H, W] で予測する、といった形です。これだけでは予測を出すだけで学習に寄与しないので、損失につなぐ必要があります。

### 特徴再構成損失(feature reconstruction loss)

予測した未来の特徴と、実際の未来の特徴を比べる損失です。

- 予測: いま(時刻 t)の映像から予測モジュールが出した、+1.6s 〜 +6.4s の特徴。
- 正解(target): 実際に 1.6s 後の映像を、勾配を流さない凍結バックボーン(frozen backbone)に通して得た特徴。

損失は両者の MSE(二乗誤差)です。実装としては、未来ステップごとに特徴を MSE で比べ、ステップごとの重み temporal_weights をかけて平均する、という形になります。

```python
for pred, target in zip(predicted_features, target_features):
    per_step_losses.append(torch.mean((pred - target) ** 2))
per_step = torch.stack(per_step_losses)        # [num_future_steps]
weighted = per_step * temporal_weights
aux_loss = weighted.mean()
```

### 全体の損失への組み込み

主タスクと補助タスクを重み付きで足し合わせます。

```
total_loss = trajectory_loss + λ * feature_reconstruction_loss
```

`λ`(ラムダ)は2つの目標のバランスを取る重みで、おおよそ 0.5〜1.0 程度が目安とされることが多いです。λ が大きすぎると未来予測ばかり気にして軌道精度が落ち、小さすぎると補助タスクの効果が消えます。最適な λ はデータと構成に依存し、探索が要ります。後述のように、この λ を手で探す代わりに自動でバランスさせる手法(タスク不確実性重み付け、GradNorm)も研究されています。

### なぜ画素でなく「特徴空間」で予測するのか

未来の画像そのもの(画素)を当てさせると、影の位置や路面のテクスチャ、圧縮ノイズといった運転に無関係な細部まで再現させることになり、無駄が多く学習が難しくなります。代わりに「抽象化された特徴」を当てさせると、シーンの意味のある変化(前の車が近づいた、信号が変わった等)だけを学べます。これが後述する JEPA の核心的な主張です。画素再構成(pixel reconstruction)は「予測しても無意味な高周波の細部」にモデルの容量を浪費させるが、特徴空間予測はその浪費を避けられる、という直感です。

### なぜ正解側を凍結バックボーンにするのか

予測する側も正解を作る側も両方学習可能だと、ネットは「特徴を全部同じ定数にする」ことで損失をゼロにできてしまいます(表現崩壊)。正解側を凍結(勾配を流さない)するか、ゆっくり更新する(EMA: 指数移動平均)だけにすると、この手抜きが防げます。予測側と正解側を非対称にすること、これが崩壊回避の鍵です。

```
時刻 t の映像 → Backbone → 予測モジュール → 予測特徴 ┐
                                                    ├→ MSE → aux_loss(勾配は予測側だけに流す)
時刻 t+Δ の映像 → 凍結 Backbone(勾配なし) → 正解特徴 ┘
```

## 手法の系譜と主要論文

### 深層教師(Deeply-Supervised Nets / GoogLeNet)

補助タスクの最も古い形は「中間層に分類器を付けて損失を足す」深層教師です。Lee et al., "Deeply-Supervised Nets", AISTATS 2015 と、Szegedy et al., "Going Deeper with Convolutions (GoogLeNet)", CVPR 2015 の補助分類器(auxiliary classifier)が代表例です。GoogLeNet は深いネット(22 層)で勾配が消失するのを防ぐため、ネットの途中から枝分かれした分類器を2つ追加し、その損失を主損失に重み付き(係数 0.3)で足しました。狙いはまさに「中間層へ直接勾配を送り、学習を安定させる」ことです。本トピックの発想の祖先にあたります。

### JEPA(Assran et al., I-JEPA, CVPR 2023)

「画素ではなく学習された潜在空間(特徴空間)で、欠損部分や未来を予測する」アーキテクチャを提案しました。画素予測が無関係な細部に労力を割くのに対し、特徴空間予測は意味のある構造だけを捉えられる、という主張です。自己教師あり学習で高品質な表現が得られ、画素を再構成する系(MAE 等)より少ない計算で良い表現に達することを示しました。トレードオフは、正解となる特徴の作り方(凍結/EMA)を慎重に設計しないと崩壊する点。未来特徴予測型の補助タスクはこの JEPA の考え方を踏襲しています。

### MILE(Hu et al., NeurIPS 2022, Wayve)

自動運転に特化して、潜在空間の世界モデル(latent world model、シーンの先行きを内部で予測するモデル)を学習しました。画素予測ベースラインを上回る性能を示し、「運転では特徴空間で未来を予測する方が有効」という主張を実証した直接的な根拠です。トレードオフは、世界モデルの学習自体が重く、設計が複雑になること。

### BYOL(Grill et al., NeurIPS 2020)

ターゲット側のネットワークを指数移動平均(EMA、`θ_target ← τ θ_target + (1-τ) θ_online`、τ ≈ 0.996)でゆっくり更新する非対称設計により、ネガティブサンプル無しでも表現崩壊を防げることを示しました。予測側と正解側を非対称にすれば崩壊しない、という発見はこの分野の基盤です。凍結バックボーンの代わりに EMA で正解側をゆっくり追従させる選択肢の根拠になります。

### UniAD(Hu et al., CVPR 2023, Best Paper)

知覚系の補助タスク(物体検出・追跡・地図セグメンテーション・占有予測)を加えることで、共有 BEV 表現の質が上がり、最終的な計画性能が改善することを示しました。「補助タスクで土台の表現を良くする」という多タスク設計の現代的な代表例です。トレードオフは、補助タスクごとにラベルや損失重みの調整が必要になること。

### 損失バランスの自動化(Kendall et al. 2018 / GradNorm)

Kendall et al., "Multi-Task Learning Using Uncertainty to Weigh Losses", CVPR 2018 は、各タスクの不確実性(分散)を学習可能パラメータとして導入し、損失重みを自動調整する手法を提案しました。Chen et al., "GradNorm", ICML 2018 は、各タスクの勾配の大きさを揃えるように重みを動的調整します。手動の λ 探索を減らす方向の代表的アプローチで、補助タスクが複数あるときに効きます。

### GAIA-1(Hu et al., 2023, Wayve, arXiv:2309.17080)

運転データで学んだ生成的世界モデルが、シーンの動きの内部表現を獲得することを示しました。未来予測型の補助損失を推論時に計算して「予測と実際のズレ」を不確実性・異常の指標に使うアイデア(高ズレ = 想定外の状況)とも親和的で、補助タスクが安全シグナルにも転用できることを示唆します。

## 論文の実験結果(定量データ)

- GoogLeNet(Szegedy et al. 2015)は、補助分類器を加えることで非常に深いネット(22 層)を安定して学習でき、ImageNet 分類で当時最高クラスの top-5 誤り率(約 6.7%、アンサンブルで)を達成しました。後の検証(Inception-v3 論文など)では補助分類器の主効果は「勾配を流す」より「正則化」に近いとも報告されましたが、深い層の学習を助ける補助損失の有効性を最初に大規模に示した例です。「主効果が勾配伝播か正則化か」は今も完全には決着しておらず、補助タスクの効き方が一筋縄でないことを示す例でもあります。

- BYOL(Grill et al. 2020)は、ImageNet の線形評価(凍結した特徴の上に線形分類器だけを学習して精度を測る、表現の質の標準指標)で top-1 精度 74.3% を達成し、ネガティブサンプルを使う対照学習(SimCLR 等)に匹敵・凌駕しました。重要なアブレーションは「ターゲットの EMA を止める(オンラインと同一にする)と精度が大きく崩れる(崩壊する)」ことで、非対称設計が崩壊回避に必須だと定量的に示しています。

- I-JEPA(Assran et al. 2023)は、特徴空間で予測する方式により、画素再構成型(MAE)に比べて少ない事前学習計算量で同等以上の線形評価精度に到達することを示しました。「画素の細部を再構成しないことが計算効率と表現品質の両面で得」という主張の定量的裏づけです。

- MILE(Hu et al. 2022)は CARLA の closed-loop 評価で、潜在世界モデルによる予測を学習に組み込むことで、画素再構成や予測無しのベースラインに比べ運転スコア(ルート完走と違反の少なさを総合した指標)を有意に向上させました。「特徴空間での未来予測が運転性能に効く」ことの直接的な証拠です。

- UniAD(Hu et al. 2023)は nuScenes で、知覚補助タスクを統合した場合に計画の軌道誤差(L2 距離、単位メートル)と衝突率がいずれも改善することをアブレーションで示しました。具体的には、知覚・予測モジュールを段階的に外すと最終的な計画性能が劣化することを表で示しており(平均 L2 でおよそ 1 メートル前後の水準を達成)、「補助タスクが共有表現を底上げして計画を助ける」という主張を裏づけています。

これらの数値が共通して示すのは、「補助タスク(特に特徴空間での自己教師あり予測や知覚タスク)を足すと、追加ラベルがほとんど無くても主タスクの性能が上がる。ただし正解側の設計を誤ると崩壊し、タスク間のバランスを誤ると負の転移で悪化する」という点です。

## メリット・トレードオフ・限界

メリット
- 入口に近い Backbone / Fusion に強い勾配を直接送れ、共有表現の質が上がる。
- ラベル不要(未来の自分の映像が正解になる自己教師あり)なので、追加アノテーションが要らない。
- 推論時に予測ズレを測れば、想定外シーンの検知(安全シグナル)に転用できる。

トレードオフ・限界
- 重み λ や EMA 係数 τ などハイパーパラメータが増える。λ が大きすぎると主タスク(軌道)精度を損なう。
- 逐次フレームのペア(時刻 t と t+Δ)をデータ供給側で用意する必要があり、データパイプラインが複雑になる。未来フレームを保持するメモリ・I/O コストも増える。
- 正解側の設計(凍結/EMA)を誤ると表現崩壊で学習が壊れる。
- 未解決寄りの課題として、複数の損失をどう自動でバランスさせるか(GradNorm や不確実性重み付けなどがあるが万能ではない)、補助タスクが主タスクと競合して悪化させる「負の転移(negative transfer)」をどう検知・回避するか、が依然として難しい点です。負の転移はタスクの組み合わせやデータ分布に依存し、事前予測が難しいため、実務では「補助タスクあり/なしの両方を実際に学習して比べる」ことがしばしば必要になります。

## 発展トピック・研究の最前線

- 損失バランスの自動化: Kendall et al. のタスク不確実性に基づく重み付け、GradNorm による勾配大きさの均衡化、勾配の方向衝突を解消する PCGrad など。手動の λ 探索を減らす方向。
- 世界モデルとの融合: 補助タスクを「未来の特徴1枚」から「シーン全体の先行きを生成する世界モデル」へ拡張し、計画・想像(imagination)に使う流れ(MILE、GAIA-1 系)。世界モデルは補助タスクであると同時に、計画器が内部で未来を試算する基盤にもなります。
- 占有予測(occupancy)・3D 補助: 2D 特徴ではなく 3D 占有グリッドの予測を補助に使い、幾何的に整合した表現を学ばせる研究。
- 異常検知への転用: 予測ズレを不確実性・分布外検知の信号として安全機構に組み込む取り組み。学習で使った補助損失を、推論時に「想定外度」のメーターとして再利用する発想です。
