# 軌跡損失の時間減衰重み付け

## ひとことで言うと

運転モデルが「これから数秒間で自車がどう動くか」を時系列で予測したとき、その予測誤差(損失)を時刻ごとに重み付けします。今から近い未来の予測ミスは重く罰し、遠い未来のミスは軽く扱う、という仕組みです。重みは `λ^t`(ラムダの t 乗)という指数的に減る式で決めるのが定番で、λ(ラムダ)は 0 から 1 の間の定数です。狙いは、安全に直結する目先の予測精度を最優先しつつ、本質的に当てづらい遠い未来に学習リソースを浪費しないことです。

## 直感的な理解

あなたが車のハンドルを握っているとします。「次の 0.1 秒でどれだけアクセルを踏むか」を間違えると、車は今この瞬間に間違った加速をして、前の車に追突しかねません。一方「6 秒後にどれだけハンドルを切るか」は、その 6 秒の間に信号が赤に変わるかもしれないし、歩行者が飛び出すかもしれず、そもそも正確に当てるのが原理的に不可能に近い。

ここで、もし両者の予測ミスをまったく同じ重さで罰すると、学習アルゴリズムは「当てづらい 6 秒後」を当てようとして無駄に苦労し、その分「絶対に外せない 0.1 秒後」がおろそかになります。これは試験勉強で、確実に出る基本問題と、出るかどうかも分からない難問に同じ時間をかけるのと同じ非効率です。だから「近い未来を重く、遠い未来を軽く」採点する。これが時間減衰重み付け(temporal decay weighting、テンポラル・ディケイ・ウェイティング、「時間的に減衰する重み付け」の意)の発想です。

もう一段深い動機があります。予測誤差は「今ここで悪い行動を選ばせる」原因になりますが、自動運転で本当に怖いのは、わずかな誤差が次の瞬間の状況をずらし、その新しい状況でさらに大きな誤差を生む連鎖です。近い未来を確実に当てておけば、車が「学習データに出てこなかった変な状態」へ滑り込むのを防げます。つまり近接重視は、単に目先を大事にするだけでなく、長期の安定性を守るための戦略でもあります。

## 基礎: 前提となる概念

いくつか用語を噛み砕きます。

軌跡予測(trajectory prediction)とは、現在から先の自車の動きを時系列で出力することです。たとえば 10Hz(1 秒に 10 回サンプリング)で 6.4 秒先まで予測するなら、64 ステップ分の未来を出します。各ステップで「縦方向の動き(加速度、アクセル/ブレーキに相当)」と「横方向の動き(曲率、ハンドルの切れ具合に相当)」の 2 つの値を出すなら、64 × 2 = 128 個の数字が予測値になります。お手本の人間ドライバーの実際の運転が「正解(target、教師信号)」です。なぜ位置 (x, y) ではなく加速度・曲率を予測するかは [egomotion信号の導出（速度・加速度・ヨーレート・曲率）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0813_egomotion-signal-derivation) で扱いますが、要は車を動かす制御入力に近い量を直接予測する設計です。

損失(loss)とは、予測と正解がどれだけズレているかを表す 1 つの数字です。学習とは、この数字が小さくなるようにモデルの内部パラメータ(重み)を少しずつ調整する作業です。具体的には、損失を各パラメータで微分した勾配(gradient、損失を減らすにはどちらにパラメータを動かせばよいかを示すベクトル)を計算し、勾配の逆方向にパラメータを動かす(勾配降下法)。だから損失の設計を変えると、何を優先して学ぶかが変わります。重みを掛けるとは、その時刻の勾配を強める/弱めることに等しい、と理解してください。

模倣学習(imitation learning)とは、専門家(ここでは人間ドライバー)の行動データを集め、その行動を真似るようにモデルを訓練することです。最も単純な形は行動クローニング(behavior cloning、BC)で、状態を入力、専門家の行動を正解として教師あり回帰を行います。本トピックの損失はこの行動クローニングの損失に時刻ごとの重みを足したものと理解できます。

ブロードキャスト(broadcasting)とは、形の違うテンソル(多次元配列)同士の演算で、小さい方を自動的に引き伸ばして揃える仕組みです。たとえば形 `(64,)` の重みベクトルを `(B, 64)` の損失行列に掛けると、各サンプル(バッチ、まとめて処理する複数データ)に同じ重みが自動適用されます。B はバッチサイズです。

## 仕組みを詳しく

典型的な実装は 2 段階に分かれます。重みの事前計算と、損失計算時の重み適用です。

ステップ 1: 重みの事前計算。減衰率 λ が 1.0 でなければ、次の重みベクトルを作ります。

```
t       = [0, 1, 2, ..., 63]      # 64 個の時刻インデックス
weights = λ ** t                  # 各時刻に λ^t
```

たとえば λ = 0.95 だと:

```
t=0   : 0.95^0  = 1.000   (今すぐ。重み最大)
t=1   : 0.95^1  = 0.950
t=10  : 0.95^10 = 0.599   (約1秒後。6割に減衰)
t=32  : 0.95^32 = 0.194   (約3.2秒後)
t=63  : 0.95^63 = 0.039   (約6.3秒後。最初のほぼ4%)
```

先の時刻ほど重みが指数的に小さくなります。この重みは学習で更新しない定数なので、PyTorch なら `register_buffer` で保持するのが定石です(buffer は「学習対象ではないが、モデルと一緒に GPU へ移動し、保存・復元される定数テンソル」のこと)。λ = 1.0 のときは全要素が 1 になり、「重み付けなし=全時刻平等」を意味します。実装上は重みを総和で正規化して `Σ weights = 1` にしておくと、λ を変えても損失全体のスケール(大きさ)が一定に保たれ、学習率を変えずに λ だけ比較できる利点があります。正規化しないと λ が小さいほど損失全体が小さくなり、実効的に学習率を下げたのと同じ効果が混入してしまいます。

ステップ 2: 損失計算。予測と正解が平らな 128 次元ベクトルなら、まず形を戻します。

```
pred   = trajectory_pred.view(B, 64, 2)         # (B,128) -> (B,64,2)
target = trajectory_target.view(B, 64, 2)
per_element = loss_fn(pred, target)             # (B,64,2) 要素ごとの誤差
per_timestep = per_element.mean(dim=2)          # (B,64)  信号方向(2)を平均
weighted = per_timestep * weights.unsqueeze(0)  # (B,64) * (1,64)
return weighted.mean()                          # スカラー1個
```

`weights.unsqueeze(0)` は `(64,)` を `(1,64)` にし、バッチ方向へブロードキャストさせます。最後に全体平均を取ってスカラーにし、これを `backward()`(誤差逆伝播、各パラメータの勾配を自動計算する処理)に渡します。

before/after の感覚をつかみます。信号方向を平均済みの時刻別誤差が次のようだったとします。

```
時刻 t :   0     1    ...   63
誤差   : 0.2   0.3   ...   2.0   (遠い未来ほど誤差が大きいのは典型的)
```

重みなし(λ=1.0)なら誤差 2.0 がそのまま効きます。λ=0.95 なら t=63 の誤差 2.0 には 0.039 が掛かって 0.078 になり、影響がほぼ消えます。結果として最適化は近い未来を優先します。

要素ごとの損失関数 `loss_fn` には L2(平均二乗誤差、MSE)か SmoothL1(Huber 損失)を選ぶのが一般的です。SmoothL1Loss は、誤差が小さいときは二乗誤差(なめらか)、大きいときは絶対値誤差(外れ値に過敏でない)に切り替わる損失で、ノイズの多い人間運転データに対して頑健です。重要な実装上のコツは、損失関数に `reduction="none"`(平均や総和をとらず要素ごとの値を残す指定)を渡して要素ごとの誤差を残し、平均を自分で取ることです。ライブラリ任せに平均させると、その前に時刻ごとの重みを掛けられないからです。

なぜ指数減衰という形なのか。理由は 3 つあります。第一に、未来の不確実性は時間とともに概ね指数的に増える(各ステップで予測誤差が一定割合で拡大していく)という直感に合うこと。第二に、`λ^t` は再帰的に `weights[t] = λ * weights[t-1]` と計算でき、強化学習の割引率 γ(ガンマ)と同じ数学的構造を持つので、理論的な見通しが良いこと。第三に、ハイパーパラメータが λ ひとつだけで、減衰の速さを連続的に調整できる手軽さです。

## 手法の系譜と主要論文

時刻ごとの重み付けや遠い未来の割引は、軌跡予測・模倣学習・強化学習の各分野で独立に発展し、現在は共通の常識になっています。系譜を時系列で追います。

割引率 γ の起源(強化学習)。Bellman による動的計画法(1957)以来、将来の報酬を `γ^t` で割り引く考え方は強化学習の根幹です(Sutton & Barto, "Reinforcement Learning: An Introduction", 第2版 2018)。理由は「遠い未来の報酬は不確実なので現在価値を下げる」「無限和を収束させる数学的都合」の両方です。本トピックの `λ^t` は、この割引を報酬ではなく予測誤差(損失)に適用した損失版と理解できます。符号が逆(報酬は最大化、損失は最小化)なだけで、形はまったく同じです。

DAgger(Ross, Gordon, Bagnell, AISTATS 2011, arXiv:1011.0686)。模倣学習で「誤差の累積(compounding error)」を理論的に定式化した重要論文です。素の行動クローニングでは、誤差がホライズン T に対して O(T²) で増えうることを示しました(専門家の分布から外れると、外れた先の状態の正解を知らないため、ずれが二乗で膨らむ)。これは「遠い未来の予測は信頼できない」という時間減衰の動機の理論的裏付けです。DAgger 自身は専門家に追加質問するオンライン手法ですが、問題意識は共通します。

ChauffeurNet(Bansal, Krishnan, Ogale, Waymo, RSS 2019, arXiv:1812.03079)。学習で運転方策を作る研究で、1 ステップの予測ミスが次のステップの入力をわずかにずらし、それがさらに次のミスを生む連鎖を実車スケールで指摘しました。ChauffeurNet 自体は合成的な摂動(perturbation、軌跡をわざとずらして「ずれからの復帰」を学ばせる)を訓練データに注入して対処しましたが、根底の問題意識は時間減衰と共通します。

UniAD(Hu Yihan ら、上海 AI Lab / OpenDriveLab、CVPR 2023 Best Paper、arXiv:2212.10156)。知覚から予測・計画までを一気通貫(end-to-end)で学習する大規模モデルです。複数時刻の未来軌跡を出力するモジュールを持ち、各時刻の損失バランスが最終的な計画性能を左右することを示しました。近接時刻を重視する設計思想は時間減衰と整合します。

VAD / VADv2(Jiang ら、ICCV 2023、arXiv:2303.12077)。シーンをベクトル(地図要素やエージェントの折れ線)で表す軽量な end-to-end プランナー。軌跡を確率分布として扱い、時刻ごとの不確実性を明示的にモデル化する方向の先駆けです。さらに DiffusionDrive(2024-2025)など拡散モデルベースのプランナーへ発展しました。時間減衰は「不確実性を固定された指数曲線で近似する」素朴版で、これら新しい手法は不確実性をデータから学習する点が進歩です。

## 論文の実験結果(定量データ)

時間減衰そのものを単独で切り出してアブレーション(ある要素を抜いて効果を測る実験)した論文は多くありませんが、関連する定量データを挙げます。

ChauffeurNet の評価は実車ログとシミュレータで行われ、主指標は「介入なしで走れる距離」や「衝突・路外逸脱の回数」です。報告によると、素の行動クローニングは比較的短い距離で破綻するのに対し、誤差累積への対策(摂動 + 補助損失)を入れると安定走行距離が大幅に伸びることが示されました。これは「遠い未来の信頼性を考慮する設計」が長期安定性に効くことの傍証です。

UniAD は nuScenes データセット([L2D LeRobotデータセットパーサ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0821_l2d-lerobot-dataset-parser) 参照)で評価され、計画の主指標は L2 誤差(予測軌跡と実際の軌跡のメートル単位のズレ)と衝突率(collision rate、予測軌跡が他物体と衝突する割合)です。報告によると UniAD は計画タスクで平均 L2 誤差(1/2/3 秒平均)をおよそ 1.0m 前後に抑え、衝突率を先行手法(例: ST-P3)比でおおむね半減させました。指標の意味は、L2 誤差が小さいほど人間の運転に近く、衝突率が低いほど安全ということです。各時刻の損失を適切に設計することがこれらの数値に直結します。

時間減衰率 λ の選び方については、軌跡予測コンペ(Waymo Open Motion Dataset, Argoverse など)の経験則として、λ をおよそ 0.9〜0.99 の範囲に置くのが穏当とされます。報告によると、λ を 0.5 のように極端に小さくすると t=10 で重みが 0.001 まで落ち、約 1 秒より先をほぼ無視するため、交差点進入時の先読み減速などの長期挙動が学べなくなる失敗が知られています。逆に λ = 1.0(均等)だと近接精度が犠牲になりやすい。λ = 0.95 はおおむね「1 秒ごとに重みが約 6 割、6 秒後に数%」となり、近接重視と長期考慮の保守的なバランス点としてよく採用されます。

指標の意味の補足。L2 誤差は予測点と正解点のユークリッド距離の平均で、単位はメートル。0.1m の差は車幅(約 1.8m)に対して小さいが、衝突を分ける場面では決定的です。minADE/minFDE(複数候補のうち最良の平均/最終変位誤差)もマルチモーダル予測でよく使われ、いずれもメートル単位で「小さいほど良い」。衝突率は「予測軌跡通りに走ったら他者にぶつかる確率」で、0.1% 違うだけでも 1000 シーンに 1 回の事故差になり、安全上きわめて重い指標です。

## メリット・トレードオフ・限界

メリット:
- 安全に直結する近接予測の精度を優先でき、衝突率など実利的な指標の改善が期待できる。
- 遠い未来の本質的な不確実性に振り回されにくく、学習が安定しやすい。
- 重みは事前計算した定数なので追加計算コストはほぼゼロ。λ=1.0 で従来の均等損失に即戻せるため、アブレーションが容易。
- 強化学習の割引率と同じ数学構造を持ち、理論的に扱いやすい。

トレードオフ・限界:
- λ を小さくしすぎると遠い未来をほぼ無視し、先の交差点を見越した減速のような長期計画が弱くなる。
- 指数減衰という形は人為的な仮定であり、真に最適な重み形状(線形減衰、区分的重み、データ依存の不確実性重み)とは限らない。
- λ 自体がハイパーパラメータで、データセットや予測ホライズン(何秒先まで予測するか)ごとに調整が要る。損失を正規化しないと λ が学習率と交絡する点も実装上の落とし穴。
- 未解決の課題として、時間減衰は「不確実性は時間とともに単調かつ指数的に増える」と暗に仮定するが、実際には信号待ちで一旦止まれば数秒先まで挙動が確実になるなど、状況依存で不確実性は変動する。これを固定曲線で扱うのは粗い近似で、不確実性を入力から動的に推定する(異方性のある重み付け)方向が研究の最前線です。

## 発展トピック・研究の最前線

不確実性の明示的モデル化。各時刻の予測に分散(どれだけ自信がないか)を持たせ、ガウス対数尤度(Gaussian NLL)で学習すると、モデルが自分で「遠い未来は自信がない」と学べます。これは固定 λ より柔軟で、Heteroscedastic(分散が入力ごとに変わる)損失と呼ばれます。Kendall & Gal の不確実性推定(NeurIPS 2017)が代表的な基礎で、損失自体が「自信のないサンプルの重みを下げる」形に自動調整されます。

マルチモーダル予測。未来は一通りではない(直進も右折もありうる)ため、複数候補を出してそのうち最良のものだけを罰する winner-takes-all 損失や、混合密度ネットワーク(MDN、出力を混合ガウス分布で表す)、拡散モデルが使われます。時間減衰はこれらと直交し、各候補内の時刻重み付けとして併用できます。

クローズドループ評価。時間減衰の真価はオープンループ(1 ステップ予測の精度)ではなくクローズドループ(予測通り走らせ続けたときの累積安定性)で出ます。nuPlan(Motional, 2021)や CARLA などのシミュレータで「最後まで安全に走れたか」を測る評価が、誤差累積への効果を正しく捉える方向として重視されています。オープンループの L2 誤差だけ見ると時間減衰の利点を見落としやすい、というのが近年の指摘です。
