# スコアベース生成モデル/SDE

## ひとことで言うと
データの分布そのものを式で書き下すのではなく、「データを少しずつノイズで壊していく過程」を学び、それを逆回しすることで、ランダムなノイズから本物そっくりの新しいデータを作り出す生成モデルの枠組みです。壊す/戻す過程を連続時間の確率微分方程式 (SDE) という数式で記述すると、これまで別物として研究されてきた拡散モデルとフローベースの生成が、同じ理論の中の異なる設定として統一的に見えてきます。学習の鍵になるのが「スコア」、すなわち「いまの場所からどちらへ動けばよりデータらしくなるか」を指す勾配ベクトルです。

## 直感的な理解
2 次元の平面に、たくさんの点が雲のように散らばっている様子を想像してください。この雲が「本物のデータの分布」です。たとえば「手書きの数字 3 の画像っぽさ」が高い場所に点が濃く集まっている、と考えてもよいです。

スコアベースのモデルは、平面上のどの場所にいても「ここから濃い場所 (データらしい場所) はどっちか」という矢印を返してくれる関数を学びます。この矢印に従って少しずつ進めば、最初はどんなにデタラメな場所から出発しても、最終的に点が濃い領域、つまり本物らしいデータの近くにたどり着きます。

なぜわざわざ矢印 (勾配) を学ぶのか。直接「この点の確率は何 %」という分布を学ぼうとすると、確率の総和が 1 になるよう全空間で積分して正規化しなければなりません。確率密度を p(x) = exp(−E(x)) / Z と書いたとき、Z = ∫ exp(−E(x)) dx という正規化定数 (分配関数) が立ちはだかります。高次元 (画像なら数万〜数百万次元) では、この積分を計算することは事実上不可能です。ところが「確率の対数の勾配」を取ると ∇_x log p(x) = −∇_x E(x) となり、Z は x に依存しない定数なので微分した瞬間に消えます。つまりスコアを学べば、決して計算できない Z に一度も触れずに済むのです。これがスコアを使う最大の動機です。

## 基礎: 前提となる概念
いくつかの言葉を、知らない人向けに噛み砕いておきます。

生成モデル (generative model) とは、学習データに似た新しいデータを作り出すモデルのことです。画像生成、音声合成、そして連続値の系列予測 (将来の動きの候補生成など) に使われます。

確率密度 p(x) とは、「データ点 x がどれくらいありそうか」を表す関数です。点が濃い場所では大きく、まばらな場所では小さい値を取ります。

スコア (score) とは、確率密度の対数を入力 x で微分した勾配ベクトル ∇_x log p(x) のことです。記号 ∇ (ナブラ) は「各成分で偏微分して並べたベクトル」を意味します。これは「対数確率が最も急に増える方向」、つまり「もっとデータらしくなる方向」を指します。x が D 次元なら、スコアも D 次元のベクトルです。

ウィーナー過程 (Wiener process) / ブラウン運動とは、各瞬間にランダムな微小な揺らぎが加わり続ける連続的なランダムウォークです。記号 w で書き、その微小増分 dw が「ランダムなノイズの一突き」に当たります。dw は平均ゼロ・分散 dt の正規分布に従うと考えてよいです。

確率微分方程式 (SDE, Stochastic Differential Equation) とは、ふつうの微分方程式に上記のランダムな揺らぎを足したものです。「決まった方向への動き (ドリフト)」と「ランダムな揺らぎ (拡散)」を同時に記述できます。普通の微分方程式が「次の瞬間どっちへ動くか」を完全に決めるのに対し、SDE は「平均的にはこっちだが、ランダムにぶれる」という動きを表します。

ガウスノイズ / 正規分布 N(0, I) とは、平均ゼロで各次元が独立な標準的なランダム値の集まりです (I は単位行列で、各次元の分散が 1 で互いに無相関であることを表します)。完全にノイズになった状態 = この分布から取り出した値、というのが生成の出発点になります。

ランジュバン動力学 (Langevin dynamics) とは、スコアの方向へ少し進みつつ、毎回少量のガウスノイズを足す反復です。式で書くと x ← x + (ε/2) ∇_x log p(x) + √ε z (z は標準正規ノイズ、ε は小さなステップ幅)。これを繰り返すと、x は分布 p(x) からのサンプルへ収束します。スコアさえ分かればサンプリングできることを保証してくれる古典的な道具です。

## 仕組みを詳しく
Song らの Score SDE (ICLR 2021) は、それまでバラバラに研究されていた手法を 1 本の連続時間の枠組みにまとめました。中身を段階的に見ていきます。

### 1. 前方過程 (データを壊す): SDE
本物のデータ x(0) に、時刻 t を 0 から 1 へ進めながら少しずつノイズを足していきます。これを SDE で書きます。

```
dx = f(x, t) dt + g(t) dw
```

- f(x, t) dt: ドリフト項。決定的にデータを変形する成分です (例: 値を少しずつ縮める)。
- g(t) dw: 拡散項。dw がランダムノイズの一突き、g(t) がその強さです。

設計として、t=0 では本物のデータ、t=1 ではほぼ完全なガウスノイズ N(0, I) になるようにします。代表的に 2 種類あります。

- VE-SDE (Variance Exploding, 分散発散型): f を持たず、g(t) を時間とともに急速に大きくしてノイズの分散を発散させる。元データはそのまま残るが、その上に巨大なノイズを乗せることで t=1 で実質ノイズだけにする。後述の NCSN/SMLD 系に対応します。
- VP-SDE (Variance Preserving, 分散保存型): f(x,t) = −(1/2) β(t) x で元データを縮めながら g(t) = √β(t) でノイズを足し、全体の分散を一定に保つ。後述の DDPM (拡散モデル) に対応します。

この対応関係こそが論文の核心の一つで、「拡散モデルとは VP-SDE を離散時間で刻んだものだった」「NCSN は VE-SDE だった」と明らかにしました。それまで別々の系譜とされていた二つが、ドリフトと拡散の選び方が違うだけの同じ枠組みの兄弟だったのです。

### 2. 逆方向過程 (ノイズから生成): 逆向き SDE
確率論の古典的な結果 (Anderson 1982) として、上の前方 SDE には必ず対応する「逆向き SDE」が存在します。

```
dx = [ f(x, t) − g(t)² ∇_x log p_t(x) ] dt + g(t) dw̄
```

ここに先ほどのスコア ∇_x log p_t(x) が現れます。p_t は時刻 t における (ノイズが混ざった) データの分布です。t=1 のノイズから出発し、時間を 1→0 へ逆に流せばデータが生成されます。dw̄ は時間を逆向きに流すときのウィーナー過程です。問題は本物のスコアが分からないこと。そこでニューラルネットワーク s_θ(x, t) でスコアを近似します (θ は学習する重み)。実際の生成は、この逆 SDE をオイラー・丸山法などの数値ソルバで t=1 から t=0 まで少しずつ刻んで解くことに相当します。

### 3. 学習: スコアマッチング
各時刻 t でデータにノイズを足し、ネットワークにそのスコアを当てさせます。素朴なスコアマッチング (Hyvärinen 2005) は、真の分布を使わずにスコアを学ぶ巧妙な定式化ですが、ヤコビアンのトレース項を含むため高次元では計算が重いという難点がありました。これを解決したのがデノイジングスコアマッチング (Vincent 2011) です。

鍵は、ノイズを足した条件付き分布 p(x(t)|x(0)) が (前方過程がガウスなので) 解析的に書けることです。たとえば x(t) = α(t) x(0) + σ(t) ε (ε ~ N(0, I)) という形なら、この条件付き分布のスコアは ∇ log p(x(t)|x(0)) = −(x(t) − α(t) x(0)) / σ(t)² = −ε / σ(t) と手で計算できます。つまり正解ラベルを作れます。損失はおおむね次の形です。

```
L = E_t E_{x(0)} E_{x(t)|x(0)} [ λ(t) ‖ s_θ(x(t), t) − ∇_{x(t)} log p(x(t)|x(0)) ‖² ]
```

E は期待値 (平均)、λ(t) は時刻ごとの重みです。実装上はネットワークに「足したノイズ ε を予測させる」形にすることが多く (スコアと ε は σ(t) 倍の関係なので等価)、これがノイズ予測 ε_θ という DDPM の定式化に一致します。学習が安定で実装が容易なのは、各サンプルについて「足したノイズそのもの」という明確な正解があるからです。

### 4. 確率フロー ODE: 拡散とフローの統一
Song らの最も影響力が大きい発見の一つが「確率フロー ODE (probability flow ODE)」です。上の逆 SDE と各時刻でまったく同じ周辺分布 p_t(x) を持つ、ランダム項のない常微分方程式 (ODE) が存在します。

```
dx = [ f(x, t) − (1/2) g(t)² ∇_x log p_t(x) ] dt
```

dw が消え、決定的な軌跡になっています。同じ出発ノイズからは必ず同じデータが生成されます (再現性がある)。係数が逆 SDE の g(t)² から (1/2) g(t)² へ半分になっている点に注目してください。これは「ランダムな拡散をやめる代わりに、ドリフトでその分布の広がりを正確に再現する」ための補正です。

この ODE 視点が後の flow matching や rectified flow (生成経路をまっすぐにする手法) と地続きで、「拡散もフローも、ある速度場 (velocity field) に沿って分布を運ぶ ODE/SDE の一族」という統一的な見方を与えました。つまり拡散モデルと連続正規化フローは別物ではなく、同じ ODE の異なる学習・サンプリングの仕方だったのです。さらに ODE は瞬時変数変換公式 (instantaneous change of variables) を使って厳密な対数尤度を計算できるという理論的なご褒美もあります。確率モデルとして尤度を測れることは、データ圧縮や異常検知など下流タスクで重要です。

### テンソル形状の具体例 (低次元データの生成)
スコアベースの枠組みは画像のような高次元データだけでなく、低次元の連続データにも使えます。たとえば「将来 6 時刻分の 2 次元座標の系列」を生成対象とすると、それは形状 [B, 6, 2] (B はバッチサイズ) のテンソルです。これをノイズ [B, 6, 2] から出発し、条件 c (周辺状況を符号化した特徴) を入れた s_θ(x_t, t, c) を何ステップか ODE/SDE ソルバで適用して、もっともらしい系列へ運びます。画像のような高次元 (数十万次元) と違い、こうした低次元データは数ステップの ODE で十分速く生成でき、リアルタイムに近い用途にも適します。次元が低いと分布の谷が浅く、少ない反復でスコアに沿って収束できるためです。

## 手法の系譜と主要論文
- Hyvärinen, "Estimation of Non-Normalized Statistical Models by Score Matching" (JMLR 2005)。正規化定数を計算せずにスコアを推定する理論的な土台を作りました。
- Vincent, "A Connection Between Score Matching and Denoising Autoencoders" (Neural Computation 2011)。デノイジングスコアマッチングを示し、ノイズ除去の学習がスコア推定と等価であることを明らかにしました。後の拡散モデルの学習則の原点です。
- Song & Ermon, "Generative Modeling by Estimating Gradients of the Data Distribution" (NeurIPS 2019, NCSN/SMLD, arXiv:1907.05600)。複数のノイズレベルでスコアを学び、ランジュバン動力学でサンプリングする原型を提案しました。動機は正規化定数を避けられる利点と、データが薄い領域でもノイズを足せばスコアが学べること。多様で高品質なサンプルを得られる一方、サンプリングに多数ステップを要して遅いのが弱点でした。
- Ho, Jain & Abbeel, "Denoising Diffusion Probabilistic Models" (NeurIPS 2020, DDPM, arXiv:2006.11239)。離散時刻の拡散モデルを定式化し、ノイズ予測 ε_θ という形でスコアマッチングと等価な学習を実用化。画像生成で GAN に匹敵する品質を達成しました。詳しくは [DDPM（拡散モデル基礎）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0230_ddpm-denoising-diffusion)。
- Song et al., "Score-Based Generative Modeling through Stochastic Differential Equations" (ICLR 2021, Score SDE, arXiv:2011.13456)。本ノートの主題。NCSN を VE-SDE、DDPM を VP-SDE として連続時間 SDE に統一し、逆 SDE と確率フロー ODE を導出。条件付き生成 (controllable generation) を可能にし、汎用 ODE ソルバや厳密尤度計算への道を開きました。
- Lipman et al., "Flow Matching for Generative Modeling" (ICLR 2023, arXiv:2210.02747)。確率フロー ODE の速度場を回帰で直接学ぶ flow matching を提案し、学習をさらに単純化・高速化しました。詳しくは [Flow Matching（フローマッチング）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0121_flow-matching)。
- Liu et al., "Rectified Flow" (ICLR 2023) は ODE 経路を直線に近づけて少ステップ生成を狙う発展形です。Song et al., "Consistency Models" (ICML 2023, arXiv:2303.01469) は ODE 軌跡上のどの点からも一発で終点へ飛ぶ写像を学び、1〜数ステップ生成を実現しました。

系譜を一言で言えば、Hyvärinen と Vincent がスコア推定の理論を作り → Song & Ermon が多ノイズレベルで実用化し → Ho らが拡散として整え → Song らが両者を SDE/ODE で統一し → Lipman らがフロー視点で簡潔にし → Song らが consistency で高速化した、という流れです。

## 論文の実験結果 (定量データ)
評価でよく使われるのが FID (Fréchet Inception Distance) です。これは生成画像と本物画像を Inception ネットワークで特徴ベクトルに変換し、その特徴分布 (平均と共分散) の近さを測る指標で、小さいほど本物に近いことを意味します。

- NCSN (Song & Ermon 2019): CIFAR-10 (32×32 の小画像 6 万枚のデータセット) で Inception Score 約 8.87 を報告し、当時の多くの GAN を上回りました。ただしサンプリングに 1000 ステップ規模のランジュバン反復を要しました。
- DDPM (Ho et al. 2020): CIFAR-10 で FID 3.17 を報告。これは当時の有力な GAN に匹敵する水準で、「拡散モデルが GAN と戦える」ことを示した転機でした。
- Score SDE (Song et al. 2021): CIFAR-10 で FID 約 2.20、Inception Score 約 9.89 を報告し、当時の最良クラスを更新しました。Inception Score は生成物の鮮明さと多様性を測る指標で、大きいほど良いとされます。さらに 1024×1024 という高解像度の顔画像 (CelebA-HQ 系) の生成にも成功し、SDE 枠組みの拡張性を示しました。
- Consistency Models (Song et al. 2023): CIFAR-10 で 1 ステップ生成で FID 約 3.55、2 ステップで約 2.93 を報告。多ステップの拡散に迫る品質を 1〜2 ステップで出せることを示し、リアルタイム生成への現実味を与えました。

アブレーション (どの要素を抜くとどうなるか) の観点では、Score SDE 論文は次を示しました。(1) 確率フロー ODE で生成すると逆 SDE よりサンプリングが速くなる一方、品質はわずかに変わる。(2) VE と VP で得意な解像度や安定性が異なる。(3) 予測子修正子 (predictor-corrector, PC) サンプラを併用すると同じステップ数でも品質が上がる。PC とは、逆 SDE で一歩進める「予測子」の後に、ランジュバンの補正を一回挟む「修正子」を入れる合わせ技で、各ステップの精度を底上げします。論文の表では PC 併用で FID が予測子のみより明確に下がることが示されています。

これらの数値が重要なのは、それまで GAN が独占していた高品質生成の領域で、学習が安定し多様性も保てるスコア/拡散系が同等以上を達成できると定量的に裏づけたからです。FID が 3 台から 2 台へ下がるのはわずかに見えますが、この領域では人間が見分けにくいほど高品質な範囲であり、わずかな差が激しい競争を意味します。

## メリット・トレードオフ・限界
メリット
- 正規化定数 Z を計算せずに学習できる。これがスコアを使う本質的な強みです。
- 学習が GAN より安定し、mode collapse (出力の多様性が失われる現象) を起こしにくい。GAN のような敵対的な不安定さがなく、単純な回帰損失で学べます。
- 拡散モデルと連続正規化フローを 1 つの枠組みで説明でき、サンプラ (SDE ソルバ / ODE ソルバ / 予測子修正子) を後から差し替えられる。学習をやり直さずにサンプリング戦略だけ変えられます。
- 条件付き生成が自然に書ける (スコアに条件項を足す)。目標や制約に沿った生成の制御に向きます ([分類器フリーガイダンス (条件付き生成強化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0223_classifier-free-guidance-planning))。

トレードオフと限界
- サンプリングが多ステップになりがちで、素朴には遅い。リアルタイム制御では致命的になりうるため、蒸留 (distillation)、少ステップ ODE、rectified flow、consistency models による高速化が活発に研究されています。
- ノイズスケジュール (どの時刻でどれだけノイズを足すか) や重み λ(t) の設計が品質に効き、調整項目が多い。
- 理論はやや高度で、ランジュバン動力学・SDE・ODE といった確率解析の知識を前提とします。
- 離散データ (テキストなど) には連続なノイズ過程が自然に当てはまらず、拡張に工夫が要ります。
- 未解決課題として、少ステップで品質を落とさないサンプリング、離散データへの自然な拡張、生成の制御性と多様性の両立、誤差が蓄積する数値ソルバの精度などが研究の最前線です。

## 発展トピック・研究の最前線
flow matching と rectified flow は、確率フロー ODE の速度場を直接回帰することで学習を単純化し、より少ないステップでの生成を目指す方向です。蒸留 (consistency models, distillation) は、多ステップの教師サンプラを 1〜数ステップに圧縮する技術で、リアルタイム生成へ近づけます。条件付き生成の制御を強める手法として分類器フリーガイダンス ([分類器フリーガイダンス (条件付き生成強化)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0223_classifier-free-guidance-planning)) があり、スコアの枠組みと自然に接続します。低次元の連続データ (時系列や軌道) への応用では、低次元ゆえ少ステップで高速に生成できる利点が活き、計画・予測タスクで生成的アプローチが広がっています。決定的なクエリデコードと生成的サンプリングを組み合わせる方向 ([クエリベースモーションデコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0200_query-based-motion-decoder)) も、確率的な多様性と効率を両立させる試みとして注目されています。
