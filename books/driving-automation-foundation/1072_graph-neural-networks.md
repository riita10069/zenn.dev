# グラフニューラルネットワーク(GNN)

## ひとことで言うと

グラフニューラルネットワーク(Graph Neural Network, GNN)とは、点(ノード)と線(エッジ)で表される「つながりのデータ」を、隣同士で情報をやり取り(メッセージパッシング)させながら処理するニューラルネットワークです。自動運転では、地図のレーンや周囲の車・歩行者を「ノード」、それらの関係を「エッジ」として表し、誰が誰に影響するかを学習するのに使われます。

## 直感的な理解

画像は格子状にピクセルが並んだ「規則正しい」データで、畳み込みニューラルネットワーク(CNN)が得意です。文章は一列に並んだ「順序のある」データで、RNN や Transformer が得意です。ではソーシャルネットワークの友達関係や、道路のレーンの接続関係のように「不規則につながったデータ」はどう扱うのでしょうか。これがグラフです。

グラフのノード数は対象ごとに変わり(交差点に車が3台のときも10台のときもある)、ノードの並び順に意味はありません(車Aと車Bの登録順を入れ替えても状況は同じはず)。GNN はこの「順序を入れ替えても結果が変わらない」性質、すなわち順列同変性/順列不変性(permutation equivariance / invariance)を満たすように設計されています。

直感的な処理は「噂の伝播」に似ています。各ノードは自分の特徴を持ち、毎ステップ隣のノードから情報を受け取って自分を更新します。これを数回繰り返すと、各ノードは「自分の近所の状況」を集約した表現を持つようになります。

## 基礎: 前提となる概念

グラフを記号で表します。グラフ G は ノード集合 V と エッジ集合 E から成ります。ノード数を N とすると、各ノード i は特徴ベクトル h_i(長さ F の数列。例 F=128)を持ちます。これを縦に積むと特徴行列 H はサイズ [N, F] のテンソルになります。

つながり方は隣接行列(adjacency matrix)A で表します。A は [N, N] の行列で、ノード i と j がつながっていれば A[i,j]=1、なければ 0 です。例えばノード3つで「1-2 と 2-3 がつながる鎖」なら

```
      1  2  3
   1 [ 0  1  0 ]
A= 2 [ 1  0  1 ]
   3 [ 0  1  0 ]
```

「次数(degree)」とは各ノードにつながるエッジの本数で、上の例ではノード2の次数は2です。

CNN の畳み込みが「固定された近傍ピクセルの重み付き和」であるのに対し、GNN は「グラフ上の隣接ノードの重み付き和」を取ります。CNN を不規則なグラフへ一般化したもの、と考えると理解しやすいです。

## 仕組みを詳しく

GNN の中心は メッセージパッシング(message passing、隣接ノードへ情報を送り集約する操作)です。1層の処理は次の3段で書けます。

1. メッセージ生成: 各エッジで、送り手ノードの情報から「メッセージ」を作る。
2. 集約(aggregate): 各ノードが、隣から来たメッセージを順序によらない演算(和・平均・最大など)でまとめる。順序によらない演算を使うのが順列不変性の肝です。
3. 更新(update): まとめたメッセージと自分の現状から、新しいノード特徴を計算する。

代表的な GCN(Graph Convolutional Network)の1層は次式です。

H' = σ( D^(-1/2) A~ D^(-1/2) H W )

記号の意味を言葉で説明します。H はサイズ [N, F] の入力ノード特徴。W は学習する重み行列で、特徴の次元を [F, F'] へ変換します(例 128→64)。A~ は隣接行列に自己ループ(自分自身への接続、A~ = A + I)を足したもので、「自分の情報も集約に含める」ためです。D は A~ の次数を対角に並べた行列で、D^(-1/2) A~ D^(-1/2) は次数の大きいノードの影響が暴れないように正規化(normalize)する役割です。σ は ReLU などの非線形関数。要するに「隣(と自分)の特徴を正規化して平均し、線形変換して非線形をかける」操作です。

GAT(Graph Attention Network)はこの集約に注意機構(attention、どの隣をどれだけ重視するかを学習で決める仕組み)を導入しました。隣 j からノード i へのメッセージに重み α_ij を掛け、α_ij をデータから学習します。固定の正規化係数だった GCN と違い、「重要な隣を強く、無関係な隣を弱く」聞けるのが利点です。

層を L 層重ねると、各ノードは L ホップ先(L 本のエッジでたどれる範囲)までの情報を集約できます。自動運転の軌道予測では、車・歩行者・レーンをノードにし、相互作用をエッジにして、「この車は前方が詰まっているから減速するだろう」といった文脈をモデルに学ばせます。

VectorNet(Gao ら 2020)は自動運転に特化した重要な応用例です。地図のレーンや過去の軌跡を「ポリライン(折れ線)」として点の連なりにベクトル化し、各ポリライン内を1つのサブグラフ、ポリライン同士をグローバルなグラフとして2階層の GNN で処理します。ラスタ画像(地図を画像に描いて CNN で処理する従来手法)よりも、構造を直接扱える効率的な表現として広く参照されています。

## 手法の系譜と主要論文

- Gori, Scarselli ら, 2005/2009年: 「Graph Neural Network」という名称と初期定式化。再帰的に安定点まで反復する重い手法でした。
- Bruna ら, 2014年(arXiv:1312.6203): スペクトルグラフ理論(グラフのフーリエ変換)に基づく畳み込みを提案。理論的だが計算が重い。
- Defferrard ら, 2016年(ChebNet, arXiv:1606.09375): 多項式近似で計算を軽量化。
- Kipf & Welling, 2017年(GCN, ICLR, arXiv:1609.02907): スペクトル手法を1次近似まで簡略化し、上式のシンプルな形に。GNN 普及の決定打。
- Hamilton ら, 2017年(GraphSAGE, NeurIPS, arXiv:1706.02216): 近傍をサンプリングして大規模グラフへスケール、未知ノードへの帰納的(inductive)推論を可能に。
- Veličković ら, 2018年(GAT, ICLR, arXiv:1710.10903): 注意機構を導入。
- Gilmer ら, 2017年(MPNN, ICML, arXiv:1704.01212): 多数の手法を「メッセージパッシング」という統一枠組みで整理。
- Gao ら, 2020年(VectorNet, CVPR, arXiv:2005.04259) / Liang ら, 2020年(LaneGCN, ECCV, arXiv:2007.13732): 自動運転の地図・エージェント関係への応用。

## 論文の実験結果(定量データ)

GCN(Kipf & Welling 2017)は、論文の引用関係を使ったノード分類ベンチマークで評価されました。タスクは「各論文(ノード)がどの分野か」を、一部のラベルと引用グラフだけから当てる半教師あり分類です。報告された分類精度(accuracy、正解率)はおよそ次の通りです。

- Cora データセット: 約81.5%
- Citeseer データセット: 約70.3%
- Pubmed データセット: 約79.0%

これらは当時の従来手法(ラベル伝播やスペクトル埋め込み)を数ポイント上回り、しかも1層あたり1回の行列積で済むため高速でした。「数ポイント」の意味は、例えば Cora で従来の約75%前後から約81%へ、誤分類を相対的に2割ほど減らせたという改善です。

GAT(Veličković ら 2018)は同じ Cora で約83.0%、Citeseer で約72.5%、Pubmed で約79.0%を報告し、注意機構によりさらに数ポイント改善しました。タンパク質間相互作用(PPI)という帰納的タスクでは、マイクロ平均 F1 スコア(適合率と再現率の調和平均)が約0.973と、GraphSAGE の約0.6台を大きく上回ったと報告されています。

VectorNet(Gao ら 2020)は Argoverse 軌道予測ベンチマークで評価され、ラスタ画像ベースの手法より少ない計算量で同等以上の予測誤差(ADE/FDE、平均/最終地点の予測ずれ距離)を達成したと報告されています。

## メリット・トレードオフ・限界

メリット:

- 不規則な関係データを自然に扱え、順列不変性を保証できる。
- 可変個のエージェント(車が何台いても)を統一的に処理できる。自動運転に好適。
- 相互作用(誰が誰に影響するか)を明示的にモデル化できる。

トレードオフと限界:

- 過平滑化(over-smoothing): 層を深く重ねると全ノードの特徴が似通ってしまい、区別がつかなくなる。GNN が浅い層(2〜4層)に留まりがちな主因。
- 過圧縮(over-squashing): 遠いノードの情報が、途中の細いボトルネックを通る際に潰れて伝わりにくい。
- スケーラビリティ: 隣接ノードを毎層集約するため、巨大グラフでは計算とメモリが膨らむ。サンプリング(GraphSAGE)などの工夫が必要。
- グラフ構築自体が設計判断: 「何をノードに、何をエッジにするか」は人が決める前提で、その良し悪しが性能を左右する。自動運転で「どの距離までを隣接とみなすか」は重要なハイパーパラメータ。
- 表現力の理論的上限: 標準的なメッセージパッシング GNN は、グラフの同型判定で1次元 Weisfeiler-Lehman テストと同等の識別力までしか持たないことが示されている(Xu ら 2019, GIN)。

## 発展トピック・研究の最前線

- Graph Transformer: 注意機構を全ノード間に広げ、メッセージパッシングの局所性と過圧縮を緩和する流れ。位置エンコーディングをグラフ用に拡張する研究が活発。
- 時空間 GNN(spatio-temporal GNN): 時間変化するグラフ(刻々と動く交通)を扱う。軌道予測では時系列モデルと GNN を組み合わせる。
- 異種グラフ(heterogeneous graph): ノードやエッジに複数の型(車・歩行者・レーン・信号)がある場合の扱い。
- 自動運転の最前線: VectorNet/LaneGCN の系譜は、近年は Transformer ベースの統合モデル(エージェントと地図を一括で注意機構に通す)へ発展。ただしグラフ的なベクトル表現の思想は今も中核として生き続けている。
