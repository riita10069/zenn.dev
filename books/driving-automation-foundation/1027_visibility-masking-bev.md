# 可視性マスキング

## ひとことで言うと
複数のカメラ画像から鳥瞰図(BEV: Bird's-Eye-View、車を真上から見おろした地図状の平面)を作るとき、3次元空間の点をカメラ画像に投影しても「その点がそのカメラに本当に写っているか」は別の問題です。カメラの背後にある点や、画像の枠からはみ出した点は、計算上は何らかの座標が出てしまうものの、実際には写っていません。可視性マスキングとは、こうした「見えていない点」を集約から除外し、どのカメラからも観測されなかった BEV マスの値をゼロにする処理のことです。数値の暴走を防ぎ、BEV 表現の意味的な正しさを守るための地味だが本質的な仕組みです。

## 直感的な理解
6台のカメラを車のまわりにぐるりと付けた状況を想像してください。前方の道路は前カメラに写りますが、後カメラには写りません。にもかかわらず、3次元の点を「投影する数式」にそのまま通すと、後カメラについても何らかのピクセル座標が返ってきます。投影の数式は「この点は写っていません」とは言ってくれないからです。

もしこの偽の座標を信じて後カメラの画像特徴を拾ってしまうと、前方の道路の情報として、後方のまったく無関係な風景が混ざり込みます。これは BEV の品質を直接壊します。可視性マスキングは「カメラごとに、本当に写っている点だけを採用する」というフィルタを明示的にかけることで、この混入を防ぎます。

身近なたとえを使うなら、複数の証人から事件の証言を集めるときに「その人が現場を実際に見ていたか」を確認せずに証言を平均してしまうと、現場にいなかった人の想像が混ざって真実がぼやけます。可視性マスキングは「現場を見ていた証人だけ」を選んで証言をまとめる作業に相当します。

## 基礎: 前提となる概念
理解に必要な道具をいくつか噛み砕いておきます。

カメラ投影(camera projection): 3次元の点 (X, Y, Z) を、カメラの位置・向き(外部パラメータ、extrinsics)とレンズの焦点距離など(内部パラメータ、intrinsics)を使って、2次元の画像ピクセル (u, v) に対応づける計算です。最終段で「奥行きで割る」操作(透視除算、perspective division)が入ります。点がカメラからどれだけ奥にあるか(depth、光軸方向の距離)を z としたとき、u = f·X/z + cx のように z で割ります。

なぜ depth の符号が重要か: 点がカメラの前にあれば z は正、後ろにあれば z は負です。負の z で割ると符号が反転し、座標が画像の反対側に飛びます。物理的にはレンズの後ろにある被写体が写らないのと同じで、後方の点は本来そのカメラには存在しません。にもかかわらず数式は座標を返すので、明示的に z > 0 を判定して捨てる必要があります。

BEV クエリ・参照点(reference point): BEV 平面を格子に区切り、各マス(セル)に「この地面位置の特徴を知りたい」という問い合わせ役を割り当てます。実際には地面の1点だけでなく、その真上に複数の高さの点(ピラー、pillar)を立てて投影します。歩行者は背が高く、車は低いといった高さの違いを拾うためです。

アテンション(attention)と重みの正規化: 複数のサンプル点から特徴を集めるとき、それぞれにどれだけ注目するかの重み(attention weight)を付け、重みの合計が1になるように正規化(ふつう softmax)してから加重平均します。合計が1であることが「平均」として意味を持つ前提です。一部の点を除外したら、残った点だけで合計が1になるよう正規化し直さなければ、平均の意味が崩れます。

## 仕組みを詳しく
典型的なマルチカメラ BEV 構築では、可視性マスキングは概ね3段階で行われます。tensor の形は [B, V, N, P] を基準に説明します。B はバッチ、V はカメラ台数(例 6)、N は BEV マス数(例 200×200=40000)、P は1マスあたりのピラー点数(例 4)です。

段階1: 投影直後の可視性マスク。各点について「カメラの前にあるか」と「画像の枠内に収まるか」を判定します。

```
valid_depth = depth > eps          # 例 eps = 1e-5。カメラの前(z>0)の点だけ True
in_bounds   = (u >= 0) & (u <= 1) & (v >= 0) & (v <= 1)   # 正規化座標 [0,1] の枠内
mask        = valid_depth & in_bounds                     # 両方満たす点だけ可視
```

ここで座標 (u, v) は画像幅・高さで割って [0,1] に正規化してあるとします。0 が左端/上端、1 が右端/下端です。mask の形は [B, V, N, P]。後方の点や枠外の点は False になり、以後この点は無視されます。透視除算の前に depth を clamp(min=eps) で下限クリップして 0 除算による NaN(Not a Number、不正な数値)を防ぎますが、その点はどのみち mask で除外されるので最終結果には影響しません。「危険な値もまず安全化してから、マスで消す」のが NaN を出さないコツです。

段階2: サンプル位置の微調整後の再マスク。可変形アテンション(deformable attention、後述)の系統では、参照点の投影位置に学習した微小なオフセット(ずらし量)を足してサンプル位置を適応的に動かします。オフセットを足すと枠外に出ることがあるので、足した後でもう一度枠内判定をします。

```
sample_loc = proj_uv + offset                  # 学習したずらしを加算
sample_ok  = (sample_loc in [0,1]^2)            # 枠内か再判定
combined   = original_mask & sample_ok          # 元の可視性 AND 新しい枠内
```

ここで決定的に重要なのが、可視な点だけで重みを再正規化することです。

```
w = attn_weight * point_mask                    # 不可視点の重みを 0 に
w = w / w.sum(dim=last).clamp(min=tiny)         # 残った点だけで合計1へ
```

clamp(min=tiny) は、もし全点が不可視で合計が 0 になっても 0 除算しないための保険です。この再正規化を怠ると「見えていない点に重みが漏れたまま平均」してしまい、その分だけ加重平均の総和が1未満になって特徴の大きさが縮み、学習が不安定になります。

段階3: カメラ横断の平均と未観測マスのゼロ化。各 BEV マスは複数のカメラから観測されうるので、観測できたカメラの数で平均します。

```
cam_visible   = combined.any(dim=P)             # そのカメラで1点でも見えれば True
output        = output + weighted * cam_visible  # 見えたカメラの寄与だけ加算
visible_count = visible_count + cam_visible     # 何台が見たかを数える
...
output = output / visible_count.clamp(min=1)    # 見えたカメラ台数で割って平均
```

最後に、どのカメラからも観測されなかったマス(visible_count が 0)を明示的にゼロ化します。

```
has_obs = (visible_count > 0).float()
output  = output * has_obs                      # 未観測マスを空(ゼロ)に
```

このゼロ化の適用順序が落とし穴です。BEV 特徴を整える後段の層(LayerNorm: 各特徴を平均0・分散1に正規化してからスケールとバイアスを足す層、FFN: 全結合の小ネットワーク)にはバイアス項があり、入力がゼロでも非ゼロを出力します。したがってゼロ化を先にしても、後段の層が値を上書きしてしまいます。未観測を確実に空に保つには、これらの層を通した後で最後にゼロ化する必要があります。

数値例で確認します。前方カメラに写る道路点を後カメラに投影すると depth が負になり valid_depth=False で除外。前方カメラの視野ぎりぎり外を指す点は in_bounds=False で除外。自車の真後ろかつ遠方の死角マスはどのカメラも当たらず visible_count=0 となり出力ゼロ。視野が重なる斜め前方では2台が当たり、その2台分が平均されます。

## 手法の系譜と主要論文
可視性平均という考え方の系譜をたどります。

LSS(Lift-Splat-Shoot, Philion & Fidler, ECCV 2020, arXiv:2008.05711)は、各カメラピクセルに沿って奥行きの確率分布を予測し、特徴を3次元の点群(視錐台、frustum)として「持ち上げ(lift)」、BEV 格子に「投げ落とす(splat)」手法です。ここでは画像の枠外に出る視錐台点は構成上自然に除外され、観測できる領域だけが BEV に寄与します。明示的な可視性マスクという形ではないものの、「観測できる領域だけが BEV に寄与する」という思想の源流です。

Deformable DETR(Zhu et al., ICLR 2021, arXiv:2010.04159)は、画像全画素への注意ではなく、各クエリが少数のサンプル点だけに注意を向ける可変形アテンションを提案しました。サンプル点の位置オフセットを学習する点が、後の BEV 投影での「オフセット後の再マスク」の前提になります。

BEVFormer(Li et al., ECCV 2022, arXiv:2203.17270)が、本トピックの可視性平均を明確に定式化しました。空間クロスアテンション(SCA: Spatial Cross-Attention)の式は、各 BEV クエリ Q_p について次のように書けます。

```
SCA(Q_p) = (1 / |V_hit|) * Σ_{i ∈ V_hit} DeformAttn(Q_p, proj_i(p), F_i)
```

ここで V_hit は「参照点 p の投影が実際に当たったカメラの集合」、|V_hit| はその台数、proj_i(p) は p をカメラ i に投影した位置、F_i はカメラ i の画像特徴です。各 BEV クエリは、当たったカメラだけを使い、その台数で割って平均します。6台構成では、ある BEV 位置を観測できるカメラは通常1〜2台だけなので、見えていないカメラの寄与を入れるのはノイズです。当たったものだけで平均することで、視野の重なり領域では複数カメラが自然に統合され、単独視野では1台分がそのまま使われます。

なお、深度の clamp や softmax の再正規化そのものは特定論文というより深層学習一般のグッドプラクティスであり、数値安定性を確保する定石として広く使われています。

## 論文の実験結果(定量データ)
BEVFormer は nuScenes(自動運転の代表的ベンチマーク。6カメラ・360度・1000シーン)で評価されました。指標は3次元物体検出の NDS(nuScenes Detection Score、検出精度と位置・速度・属性誤差を総合した0〜1のスコア、高いほど良い)と mAP(mean Average Precision、平均適合率)です。

報告によると、BEVFormer は nuScenes test で NDS 約 0.569、mAP 約 0.481 を達成し、当時のカメラのみ手法の上位に立ちました。BEV セグメンテーションの評価では、空間クロスアテンションによる多カメラ融合が単一視点処理を明確に上回ることが示されています。SCA の可視性平均をアブレーション(構成要素を抜いて影響を測る実験)で外すと、視野の重なり領域での融合が崩れ、検出・セグメンテーション双方の精度が低下することが確認されています。具体的には、当たったカメラだけで平均する処理を、全カメラ一律平均に置き換えると、観測のないカメラのノイズが混入して mAP が数ポイント下がるという傾向が一般に報告されています。

LSS は nuScenes と Lyft で BEV セグメンテーションの IoU(Intersection over Union、予測領域と正解領域の重なり面積を合併面積で割った値、0〜1で1が完全一致)を報告し、多カメラを統一的に扱う end-to-end の BEV 表現が、カメラを個別処理する従来法を上回ることを示しました。LSS は車両セグメンテーションで当時のベースラインを確立し、後続研究の比較基準になりました。

これらの数値が意味するのは、可視性に基づく取捨選択と平均化が、単なる数値安定化の小細工ではなく、最終的な知覚精度に効く設計だということです。観測のないカメラを混ぜないこと自体が、測定可能な精度向上につながります。

## メリット・トレードオフ・限界
メリット:
- 後方・枠外の点を除外し、無関係な領域の特徴が幻のように混入するのを防ぐ。
- depth の clamp と重みの再正規化で NaN を回避し、学習を安定させる。
- 未観測マスをゼロで明示することで「ここは見えていない」という情報を下流に正しく伝える。占有を「不明」ではなく「空」として扱える。
- 視野が重なる領域では複数カメラを自然に統合し、単独視野では1台分を使う柔軟な平均ができる。

トレードオフと限界:
- どのカメラからも見えないマスは情報がゼロのまま残り、穴は埋まらない。これを補うには時間方向の融合(過去フレームの特徴を現在に持ち越す)や事前地図が要る。
- マスク計算・再正規化の分だけ計算コストとメモリがわずかに増える。
- 視野の縁でマスが急に途切れるため、境界付近で特徴が不連続になりうる。下流のネットワークがこの不連続をアーティファクトとして拾う可能性がある。
- 未解決の課題として、「ゼロ化された未観測領域」と「観測したが何も無い空き領域」を下流が区別できるかは設計依存です。両者を同じゼロで表すと曖昧さが残るため、別チャンネルで観測有無を渡す設計も検討されます。

## 発展トピック・研究の最前線
時間方向の融合は未観測領域を埋める有力手段です。BEVFormer は過去フレームの BEV 特徴を自己注意で現在に融合し(temporal self-attention)、一時的に隠れた物体や死角を補完します。BEVDet4D や BEVStereo など、複数フレーム・複数視点を使って深度や占有を補強する系統も活発です。

占有予測(occupancy prediction)では、BEV を超えて3次元ボクセル空間で「各立方体が占有されているか」を予測する研究(SurroundOcc, TPVFormer など)が進んでおり、ここでも可視性に基づく集約は中核です。可視性を陽に扱うことで、見えていない領域を「不明」として明示し、安全余裕を計画に渡す方向が研究されています。

数値安定性の観点では、半精度(bf16/fp16)学習が普及するなか、clamp 値や正規化の eps の選び方が学習の収束を左右します。低精度では小さな値が丸められて 0 になりやすく、再正規化の clamp が効きすぎたり効かなさすぎたりする失敗例が知られており、精度別のチューニングが実務上の論点です。
