# Normalizing Flows

## ひとことで言うと
Normalizing Flows(正規化フロー)は、「単純なランダムさ(標準正規分布)」を、可逆(逆向きにも計算できる)な変換を何段も重ねて、複雑な本物のデータの分布に作り変える生成モデルです。[VAE (変分オートエンコーダ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0255_vae_note) や [GAN (敵対的生成ネットワーク)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0236_gan_note) と違って、生成したデータの「本物らしさの確率(尤度)」を近似なしで正確に計算できるのが最大の特徴です。「正規化(normalizing)」は、複雑な分布を逆向きにたどると正規分布(きれいな釣鐘型)に戻る、という意味から来ています。

## 直感的な理解
生成モデルには「そのデータがどれくらいありそうか」という確率(尤度、likelihood)を正確に知りたい場面があります。たとえば異常検知(普段と違うデータを見つける)では、「この入力は学習データの分布から見て確率が極端に低い=異常だ」と判断したい。また、複数のモデルを「どちらがデータをよく説明するか」と比べたいときも、厳密な尤度が物差しになります。ところが、

- [VAE (変分オートエンコーダ)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0255_vae_note): 尤度を直接は計算できず、下限(ELBO)で近似する。本物の尤度との隙間が残る。
- [GAN (敵対的生成ネットワーク)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0236_gan_note): そもそも尤度を計算しない(判定でだますだけ)。密度を一切出せない。

つまり「正確な尤度」が欲しいときどちらも不十分でした。

Normalizing Flows は「正確な尤度を計算できる生成モデル」を実現するために考えられました。鍵は数学の「変数変換の公式(change of variables)」です。これは「ある分布を別の分布へ写すとき、確率密度がどう変わるか」を厳密に与える公式で、変換が可逆でヤコビアン(変換による体積の伸び縮みを表す量)の行列式が計算できれば、変換後の確率を正確に求められます。

身近なたとえ: 紙粘土を伸ばしたり折りたたんだりして形を変えるとき、ある場所がどれだけ「薄く伸ばされたか/分厚く固まったか」が分かれば、各点の「密度」を正確に追跡できます。確率の総量は 1 で保存されるので、ある場所を広げて薄くすれば、別の場所は濃くなる。Normalizing Flows はこの「伸び縮みの量(ヤコビアン行列式)」を計算しやすい形に設計した変換を重ねるのです。

普通のニューラルネットはこの「可逆」と「行列式が計算できる」を満たしません(ReLU のような層は情報を潰してしまうので逆向きに戻せないし、一般の D×D 行列式の計算は O(D³) かかり高次元では重すぎる)。Normalizing Flows の工夫は、わざと可逆かつヤコビアン行列式が簡単に計算できる特殊な層を設計したことです。「表現力」と「計算しやすさ」のせめぎ合いが、この分野の研究の主軸です。

## 基礎: 前提となる概念
- 確率密度: 連続的なデータで「その値の周辺がどれくらい出やすいか」を表す高さです。全体を積分すると 1 になります。離散の「確率」と違い、1点の確率は 0 で、区間の積分で意味を持ちます。
- 変数変換の公式(1次元の直感): y = f(x) と変換すると、x の密度 p(x) と y の密度 q(y) の間には `q(y) = p(x) / |dy/dx|` の関係があります。傾き |dy/dx| が大きい(急に伸ばす)ほど、その範囲に元の確率が薄く広がるので密度は薄まります。多次元ではこの傾きが「ヤコビアン行列式」に置き換わります。
- ヤコビアン行列(Jacobian): 変換の各出力が各入力でどう変化するかを並べた D×D の行列。その行列式(determinant)が「局所的な体積の伸び縮み」を表します。行列式が 2 なら体積が 2 倍に広がる、という意味です。
- 対数尤度: 尤度の対数を取ったもの。掛け算が足し算になり、コンピュータでの数値計算が安定する(極端に小さい確率を掛け続けてもアンダーフローしない)ので、学習では対数尤度を最大化します。
- 全単射(bijection)/微分同相写像(diffeomorphism): 入力と出力が1対1で対応し、順方向も逆方向も微分できる滑らかな写像。フローの各層はこれである必要があります。

これらを使うと、フローの対数尤度は次の1本の式で書けます。
```
log p(x) = log p_z(z) + Σ_k log | det (∂f_k⁻¹ / ∂入力) |
                 └ 正規分布での確率 ┘   └ 各変換の体積補正 ┘
```
ここで z = (逆変換で x を正規分布へ戻した点)。第1項は「標準正規分布のもとで z がどれくらいありそうか」、第2項は各変換が体積をどれだけ伸縮したかの補正です。右辺はすべて計算可能なので、近似なしの尤度が得られます。学習はこの log p(x) を本物データで最大化するだけ(最尤推定)です。GAN のような綱引きも、VAE のような下限近似もありません。

## 仕組みを詳しく
出発点は単純な分布 z(標準正規分布、平均0・分散1のランダムな点)です。これに可逆変換 f を何段も適用して、本物のデータ x にします。

```
 z(標準正規) ─f1→ ─f2→ ─f3→ ... ─fK→ x(本物の分布)
                逆向き(x → z)も計算できる(可逆)
```

- 生成(サンプリング): z を引いて f を順に通せば新しい x が出る。
- 尤度計算: 逆に x を z 方向へ戻し、上の公式で「x の確率密度」を正確に計算する。

問題は「可逆かつ行列式が軽い変換をどう作るか」。代表的な工夫が「カップリング層(coupling layer)」です。

1. 入力ベクトルを前半 x_a と後半 x_b の2つに割る。
2. 前半 x_a はそのまま通す(変えない)。
3. 後半 x_b は、前半 x_a から計算した値で変換する。RealNVP では `y_b = x_b ⊙ exp(s(x_a)) + t(x_a)`(s, t は普通のニューラルネット、⊙ は要素ごとの積)。s はスケール(伸縮)、t はシフト(平行移動)を表します。
4. y_a = x_a、y_b = 上記。

この設計のうまさ:
- 逆変換: y_a(=x_a)から s, t を計算し直せば `x_b = (y_b − t(y_a)) ⊘ exp(s(y_a))`(割り算するだけ)で元に戻せる。s, t がどんな複雑なネットでも、逆向きはこの単純な式で済む。だから可逆。
- ヤコビアン行列式: 前半が変わらず後半だけが前半に依存するので、ヤコビアン行列が下三角になる。下三角行列の行列式は対角成分の積、つまり対角に並ぶ `exp(s)` の積、対数を取れば `Σ s` だけで済む。高次元でも一瞬で計算できる(O(D))。

ただし1段では前半が一切変わらないので、段ごとに前半・後半を入れ替えて(あるいはチャンネルを並べ替えて)何段も重ね、全次元がしっかり変換されるようにします。

形状の例: データ x が `(B, 784)`。各カップリング層は前半 392 次元・後半 392 次元に分け、s と t を出す小さなネットを通す。これを 8〜16 段重ねる構成です。

自己回帰フロー(autoregressive flow)という別系統もあります。i 番目の要素を、それより前の要素 1..i−1 から計算した値で変換します。ヤコビアンが三角行列になる点はカップリング層と同じですが、表現力が高い反面、サンプリングか密度評価のどちらかが逐次的(D 回ループ)になり遅くなる、という非対称性があります。具体的には、MAF(密度評価が1回で速い・サンプリングが D 回で遅い)と IAF(その逆)が双対をなします。用途が「尤度を測りたい」のか「速く生成したい」のかでどちらを選ぶかが変わります。

## 手法の系譜と主要論文
- Dinh, Krueger & Bengio, "NICE: Non-linear Independent Components Estimation" (2014, arXiv:1410.8516)。Normalizing Flows の出発点。加算だけの可逆カップリング層(`y_b = x_b + t(x_a)`)を提案し、ヤコビアン行列式が 1 になる(体積を変えない)設計にした。正確な尤度計算を軽い計算で可能にしたが、体積を変えないぶん表現力が限られる。
- Dinh, Sohl-Dickstein & Bengio, "RealNVP" (ICLR 2017, arXiv:1605.08803)。NICE を拡張し、スケーリング `exp(s)` を加えて表現力を高めた(上記のカップリング層)。multi-scale 構造(段階的に次元を抜き出す)も導入。自然画像をそれなりの品質で生成でき、尤度も計算できる。
- Germain et al., "MADE" (ICML 2015) と Papamakarios et al., "Masked Autoregressive Flow (MAF)" (NeurIPS 2017)。自己回帰的なフローで密度推定の精度を高めた系統。Kingma et al., "IAF"(Inverse Autoregressive Flow, NeurIPS 2016)はサンプリングを速くした逆向きの設計で、VAE の事後分布を柔軟にする用途(より表現力の高い q(z|x))でも使われました。MAF は密度評価が速くサンプリングが遅い、IAF はその逆、という双対の関係です。
- Kingma & Dhariwal, "Glow" (NeurIPS 2018, arXiv:1807.03039)。1×1 可逆畳み込み(チャンネルの並べ替えを学習可能な行列にしたもの)と ActNorm(データ依存の初期化を伴う正規化)を導入し、高解像度の顔画像生成まで到達。フローモデルでも高品質生成が可能と示しましたが、モデルが大きく生成品質では同時期の GAN にやや劣りました。
- Durkan et al., "Neural Spline Flows" (NeurIPS 2019)。カップリングの変換を区分的な有理スプライン(monotonic rational-quadratic spline、単調増加で可逆を保ちつつ柔軟な曲線)にし、表現力を大きく向上させました。

これらは「離散的な段(層)を重ねる」フローですが、段を無限に細かくして連続にした発展形が [Continuous Normalizing Flows (CNF) / Neural ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0226_continuous-normalizing-flows)(CNF / Neural ODE)であり、その学習を劇的に簡単にしたのが Flow Matching です。Normalizing Flows はその源流にあたります。

## 論文の実験結果(定量データ)
フローの主指標は bits/dim(bits per dimension、ピクセルあたり何ビットでデータを符号化できるかを尤度から換算した量。小さいほどデータをよく説明できている)です。これは対数尤度を測る指標で、フロー系が直接最適化できる本来の土俵です。データ圧縮の理論(シャノンの符号化定理)とつながっており、bits/dim が小さいほど「短いビット列で記述できる=よくモデル化できている」ことを意味します。

- RealNVP(2017)は CIFAR-10(32×32 自然画像)で約 3.49 bits/dim、ImageNet 32×32 で約 4.28 bits/dim を報告しました。当時としては尤度ベースモデルとして競争力のある水準です。
- Glow(2018)は CIFAR-10 で約 3.35 bits/dim と RealNVP を改善し、CelebA-HQ 256×256 の高解像度顔画像を生成できることを示しました。アブレーションとして、1×1 可逆畳み込みを単純な固定の並べ替え(reverse や shuffle)に置き換えると bits/dim が悪化することを示し、学習可能な並べ替えの効果を定量化しました。
- MAF(2017)は UCI のテーブルデータや画像で、当時の密度推定手法(MADE 単体や RealNVP)を上回る対数尤度を達成し、自己回帰構造が密度推定で強いことを示しました。
- Neural Spline Flows(2019)は同じパラメータ数でカップリングをスプライン化すると bits/dim が改善することを示し、変換の表現力が尤度を直接押し上げることを定量化しました。アフィン(線形)変換よりスプラインの方が1層あたりの表現力が高いので、少ない層数で同等の尤度に届きます。

ここで重要なのは指標の読み方です。bits/dim の 0.1 の差は対数尤度では大きな差で、データ圧縮率にして無視できません(全ピクセルに効くため画像全体では大きなビット数の差になる)。一方、フロー系は同時期の GAN より FID(見た目の指標)では劣ることが多く、「尤度では強いが見た目では一歩譲る」という、目的次第で評価が分かれるモデルだという点が、定量結果から読み取れる本質です。これは「尤度が高い=見た目が良い」とは限らない、という生成モデル評価の根深い問題(尤度と知覚品質の乖離)を象徴しています。

## メリット・トレードオフ・限界
メリット
- 尤度(本物らしさの確率)を近似なしで正確に計算できる。異常検知や密度推定、モデル比較に強い。
- 学習が安定(対数尤度最大化の1本の損失。GAN の不安定さがない)。
- 可逆なので、生成と尤度評価を同じネットワークで双方向にできる。潜在 z とデータ x が1対1対応するので、潜在補間や属性編集も自然です。

トレードオフ・限界
- 各変換を「可逆かつ行列式が軽い」形に制約するため、ネットワーク構造の自由度が低い。任意の層(プーリングや次元変更)を自由には使えない。
- 制約のぶん表現力が落ちやすく、複雑な分布には多くの段が必要(=計算量・パラメータが増える)。
- 入力と出力の次元が同じでなければならない(VAE のように次元を圧縮できない。高解像度では潜在が巨大になり重い)。
- 生成品質(見た目)では同時期の GAN や後発の拡散モデルに及ばないことが多い。
- 既知の課題: 高次元自然画像で「分布の外(out-of-distribution)のデータにかえって高い尤度を付けてしまう」現象(Nalisnick et al., "Do Deep Generative Models Know What They Don't Know?", ICLR 2019)が報告されています。CIFAR-10 で学習したフローが、まったく別の SVHN 画像により高い尤度を付ける、という直感に反する結果で、フローの尤度を異常検知にそのまま使う際の落とし穴として今も研究されています。

## 発展トピック・研究の最前線
- 連続化への流れ: 離散フローの「段数を決め打ち」「可逆制約のきつさ」を、段を無限小にして連続時間の ODE で表す [Continuous Normalizing Flows (CNF) / Neural ODE](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0226_continuous-normalizing-flows) が緩和します。連続版では任意のニューラルネットを速度の場として使え、可逆性の構造制約から解放されます。
- Flow Matching: CNF を「ODE を解かずに速度の場を回帰で教える」シミュレーションフリー学習で訓練する手法が、拡散モデルと並ぶ生成の主流になりつつあります。離散フロー → CNF → Flow Matching という系譜を押さえると最前線が見通せます。
- 確率推論への応用: 変分推論の近似分布(IAF など)や、シミュレーションベース推論(科学計算でのパラメータ推定、SBI)で、フローは「柔軟で尤度が計算できる分布」として今も活躍しています。物理・天文・生物のパラメータ推定で広く使われます。
- discrete flows / OOD 検出の理論: 離散データ向けのフローや、上記の OOD 尤度問題を typicality(典型集合)の概念で説明し直す研究が続いています。
