# ResNeXt

## ひとことで言うと
ResNeXt は、画像から特徴を取り出す畳み込みニューラルネットワーク ResNet を強化した設計です。「同じ形の小さな変換を何本も並列に並べて足し合わせる」という作りをしていて、その並列本数を「カーディナリティ(cardinality)」と呼びます。ネットワークを深くしたり太くしたりする代わりに、この並列本数を増やすという第3の調整軸で性能を上げられるのが核心です。しかも、その並列処理は実装上「グループ化畳み込み」という1つの単純な操作と等価で、計算量を増やさずに精度を改善できます。

## 直感的な理解
ニューラルネットワークを強くしたいとき、長らく方法は2つでした。深くする(層を増やす)か、太くする(各層のチャンネル数を増やす)かです。どちらもやりすぎると計算量とパラメータが急増し、過学習(訓練データだけに過剰適応して未知データに弱くなる)も起きやすくなります。

ここでInception(GoogLeNet)が示した別の道がヒントになります。Inceptionは「枝分かれして種類の違う変換を並列に走らせ、結果を束ねる」ことで効率を上げました。ただしInceptionは枝ごとにカーネルサイズやチャンネル数を細かく手で設計する必要があり、複雑で他のタスクに転用しにくい欠点がありました。

ResNeXt の発想は「Inceptionの枝分かれの良さと、ResNetの単純さを両取りしよう」です。具体的には、Inceptionのように枝を多数並べるが、枝を全部同じ形にしてしまえば設計は1つの数(枝の本数)で済む、と考えました。料理に例えれば、Inceptionは「素材ごとに違う調理法を職人が個別に設計するコース料理」、ResNeXt は「同じレシピを32人の料理人が並行して作り、最後に合わせる」方式です。後者は人数(カーディナリティ)を変えるだけで規模を調整でき、設計がはるかに単純です。

## 基礎: 前提となる概念
まず残差接続(residual connection)を確認します。ResNet(He ら, CVPR 2016)が導入した仕組みで、変換 F の出力に入力 x を足して y = F(x) + x とします。足し算によって勾配が近道を通って入力側へ流れ、非常に深いネットワークの学習が可能になりました。ResNet のボトルネックブロックは「1×1で256→64にチャンネルを絞る → 3×3で64チャンネルの空間畳み込み → 1×1で64→256に戻す」という3段で、太い畳み込みを安く済ませる定番の形です。ResNeXt はこのブロックを改造します。

次にグループ化畳み込み(grouped convolution)です。これは入力チャンネルをいくつかのグループに分け、各グループ内だけで独立に畳み込みを行う方式です。グループ間ではチャンネルを混ぜません。たとえば256チャンネルを32グループに分ければ、各グループは8チャンネルだけを扱います。通常の畳み込みが全チャンネルを混ぜるのに対し、グループ化は計算量とパラメータをグループ数で割った分だけ削減します。この技術自体は AlexNet(Krizhevsky ら, 2012)が2台のGPU(各3GB)にモデルを分割して載せるために導入したのが起源で、当初は精度向上が目的ではなく実装上の都合でした。ResNeXt はこれを精度向上の道具として再発見した、と言えます。

最後にカーディナリティ(cardinality)です。これは集合論で「集合の要素の個数」を意味する言葉で、ResNeXt では「並列に並べる同形変換の本数」を指します。深さ(層数)・幅(チャンネル数)に続く、ネットワーク容量を制御する第3の次元として提案されました。

## 仕組みを詳しく
ResNeXt のブロックは split-transform-merge(分けて・変換して・束ねる)の3段構えです。これは Inception の枝分かれ、そして単純な全結合ニューロン(入力を各重みで分けて掛けて足す)の両方に共通する抽象構造です。

1. split(分ける): 入力を複数の同じ形の経路に分割する
2. transform(変換する): 各経路で同一構造の小さな変換(1×1 → 3×3 → 1×1)をかける
3. merge(束ねる): 全経路の出力を足し合わせ、最後に入力を残差接続でさらに足す

カーディナリティ32の代表ブロックでは、入力256チャンネルを32本の経路に分け、各経路で「256→4チャンネルに絞る1×1 → 4チャンネルで3×3 → 4→256に戻す1×1」を行い、32本の結果を足し、最後に入力を残差接続で足します。各経路が幅4と細いため、32本あっても全体の計算量とパラメータ数は元の ResNet ボトルネックブロックとほぼ同じになるよう設計されています(32×4d は約25.6M params / 4.2 GFLOPs で ResNet-50 とほぼ一致)。

```
        入力 [256, H, W]
   ┌────┬────┬─ … ─┬────┐   32本の同形経路(カーディナリティ=32)
  1x1  1x1       1x1  1x1   各経路 256→4
   │    │         │    │
  3x3  3x3       3x3  3x3   各経路 4ch で空間畳み込み
   │    │         │    │
  1x1  1x1       1x1  1x1   各経路 4→256
   └────┴────┴─ … ─┴────┘
              ↓ 全経路を足す
              + 入力(残差接続)
              ↓
        出力 [256, H, W]
```

ここで実装上の重要な等価性があります。論文は3つの等価な表現(原論文 Figure 3 の a/b/c)を示しました。(a) 各経路を個別に1×1→3×3→1×1して足す形、(b) 各経路の最初の1×1を先にまとめて1本の1×1にし(256→128 に集約)、中央の3×3だけをグループ化し、最後の1×1も1本にまとめる形、(c) 中央の3×3を「グループ数32のグループ化畳み込み」として書く最も簡潔な形。これら3つは数学的に完全に同じ計算を表します。つまり「32本の同形変換を並列に走らせて足す」操作は「3×3畳み込みを32グループのグループ化畳み込みに置き換えた ResNet」と等価なのです。だから ResNeXt は、Inception のような複雑な手設計なしに、ResNet のコードにグループ数というたった1つのパラメータを足すだけで実装できます。なお、この等価性が厳密に成り立つのは「全経路が同一形(homogeneous)で、各経路がボトルネックの最初に分岐する」という制約を課したからで、Inception のように枝ごとに構造が違うと等価なグループ化には還元できません。ResNeXt の本質は「等価性が成り立つよう、あえて単純化した」点にあります。

代表モデル ResNeXt-50 (32×4d) は「カーディナリティ32、各経路の幅4チャンネル」を意味します。これは ResNet-50 とほぼ同じ約2500万パラメータ・約4.1 GFLOPs でありながら、ImageNet 精度が明確に上回りました。

## 手法の系譜と主要論文
原論文は Xie・Girshick・Dollár・Tu・He「Aggregated Residual Transformations for Deep Neural Networks」(CVPR 2017、arXiv:1611.05431、Facebook AI Research)です。著者には ResNet の生みの親 Kaiming He も含まれます。提案はカーディナリティという新次元で、検証したい仮説は「深さや幅を増やすより、同形変換の本数(カーディナリティ)を増やすほうが、同じ計算予算で精度が上がる」というものでした。

系譜の文脈は明快です。AlexNet が実装都合で導入したグループ化畳み込みを、Inception がマルチブランチの効率化に活かし、ResNet が残差で深さの壁を破り、ResNeXt がこれらを統合しました。ResNeXt の新規性は「Inception のマルチブランチを全て同形にして1パラメータで制御可能にし、それがグループ化畳み込みと等価であることを明示した」点です。これにより、複雑な手設計を規則的でスケールしやすい構造へと整理しました。VGG / ResNet が示した「同じブロックを繰り返す」設計原則を、ブロック内部の構造にも適用した、と捉えると位置づけが鮮明になります。

この思想の発展形が ConvNeXt(Liu ら「A ConvNet for the 2020s」, CVPR 2022)です。ConvNeXt は ResNeXt のグループ化畳み込みを極限まで進めた depthwise convolution(チャンネルごとに独立した畳み込み、グループ数=チャンネル数)を採用しました。論文中で著者らは「ResNeXt 化(グループ化してチャンネルを拡大する)」を、ResNet を Vision Transformer 並みの精度へ近代化する重要なステップの1つに数え、これだけでトップ1精度が約1ポイント上がったと報告しています。ResNeXt の思想が現代CNNに直接受け継がれていることが分かります。

## 論文の実験結果(定量データ)
主評価は ImageNet-1K(120万枚、1000クラス)で、指標はトップ1誤り率(top-1 error、モデルの最有力候補が外れる割合、低いほど良い)とトップ5誤り率です。ResNeXt-50 (32×4d) は ResNet-50 と同等の計算量でトップ1誤り率を約1.7ポイント改善(ResNet-50 の約23.9% に対し ResNeXt-50 は約22.2%)しました。同様に ResNeXt-101 (32×4d) は ResNet-101(約22.0%)を約1ポイント上回り約21.2% を記録しました。ImageNet では当時、1ポイントの改善でも最先端を更新する水準で、計算量を増やさずに得られた点が高く評価されました。

最も重要なアブレーションは「計算量を固定したまま、カーディナリティと幅を交換する」実験です。総計算量をほぼ一定に保ちつつ、カーディナリティを 1→2→4→8→32 と増やし、その分だけ各経路の幅(ボトルネックチャンネル数)を 64→40→24→14→4 と減らしていくと、誤り率が単調に下がりました。カーディナリティ1(=普通の ResNet 相当、64d)から32(32×4d)へ増やすと、同じ計算量でトップ1誤り率が約1.7ポイント改善します。この結果が「カーディナリティを増やすことは、深さや幅を増やすより効果的」という論文の中心的主張の根拠です。

別のアブレーションでは「モデルを大きくする3つの方法」を比較しました。ResNet-101 を起点に計算量を約2倍にする際、(a) より深くする(ResNet-200)、(b) より幅広くする(幅2倍の ResNet-101)、(c) カーディナリティを2倍にする(64×4d)、の3通りで、(c) が最も誤り率を下げ、(a)(b) より約0.5〜0.7ポイント良い結果でした。さらに、ResNeXt はより小さいデータセット CIFAR-10/100 でも一貫して ResNet を上回り、効果がデータ規模に依存しないことを示しました。ResNeXt は ILSVRC 2016 分類タスクで2位に入賞しています。

## メリット・トレードオフ・限界
メリットは次の通りです。第一に、カーディナリティという分かりやすい1つの軸で精度を上げられること。第二に、Inception のような複雑な手設計が不要で構造が規則的なため、深さやステージ数のスケーリングが容易なこと。第三に、同じ計算量でも深さ・幅を増やすより精度効率が良いこと。第四に、残差接続を保つので深くしても学習が安定することです。

トレードオフと限界もあります。グループ化畳み込みは概念は単純でも、GPU やライブラリによっては最適化が不十分で、理論計算量(FLOPs)どおりの速度が出ないことがあります。グループ数が多いとメモリアクセスが断片化し、また各グループの行列積が小さくなって GPU の演算器を埋めきれず、メモリ帯域のボトルネックになりやすいためです。FLOPs と実速度が乖離するという、CNN 設計に共通する課題がここにも現れます。また、計算量を固定したままカーディナリティを上げ続けると、各経路が細くなりすぎて(幅が1〜2チャンネルに近づくと)精度の伸びが頭打ちになります。表現力を確保するための最小限の幅が経路ごとに必要だからです。

研究上の論点としては、カーディナリティが「なぜ効くのか」の理論的説明は完全には確立していません。経験的には、多数の同形経路が一種のアンサンブル(複数の弱い予測器の組み合わせ)として働き、汎化性能を高めると解釈されますが、これは仮説の域を出ません。別の解釈として、グループ化はチャンネル間の不要な相関を断ち、各グループに特化した特徴を学ばせる正則化として働く、という見方もあります。

## 発展トピック・研究の最前線
ResNeXt のグループ化を極限まで進めた depthwise convolution は、MobileNet・EfficientNet・ConvNeXt の中核となり、現代の効率的CNNの標準部品です。「グループ数をチャンネル数まで増やす」と各チャンネルが独立に空間畳み込みを行い、チャンネル混合は別の1×1畳み込みに任せる、という空間とチャンネルの分離(depthwise separable convolution)に行き着きます。

ResNeXt は学習済みモデルとしても長く使われ、特に Mahajan らの弱教師あり大規模事前学習「Exploring the Limits of Weakly Supervised Pretraining」(ECCV 2018)で用いられた ResNeXt-101 32×48d(約8億パラメータ、Instagram の約35億枚のハッシュタグ付き画像で事前学習)は、ImageNet 転移でトップ1精度 約85.4% という当時の最高精度を更新し、「ラベルが多少ノイジーでも、データ量を桁違いに増やせば精度は上がり続ける」という大規模事前学習の有効性を示す代表例になりました。研究の最前線では、グループ化・depthwise の効率を保ちつつ、ハードウェアでの実効速度を最大化する設計(RegNet の設計空間探索や、ハードウェア対応 NAS)へと関心が移っています。RegNet は ResNeXt のグループ幅(group width)を設計パラメータの1つとして取り込み、最適なグループ幅をデータから導いています。
