# ConvNeXt (V1)

## ひとことで言うと
ConvNeXt は、古典的な畳み込みニューラルネットワーク(CNN)である ResNet を出発点に、当時新しく強かった Transformer 系モデル(特に Swin Transformer)の設計上の工夫を1つずつ取り込んで「近代化」した純粋な CNN です。新しい仕組みを発明したのではなく、既存部品の組み合わせ方を丁寧に見直すことで、自己注意を一切使わない純 CNN のまま Swin を上回る精度を出したのが核心です。「画像認識の主役は本当に Transformer の自己注意なのか、それとも周辺の設計の積み重ねなのか」という問いに、実験で答えを出した研究でもあります。

## 直感的な理解
あるレストラン(Transformer)が評判を呼んでいるとき、その人気の理由は「秘伝のソース(自己注意)」なのか、それとも「内装・接客・盛り付けといった細部の総合力」なのか、を切り分けたいとします。ConvNeXt の著者らがやったのは、自分たちの古い店(ResNet)に、評判の店の細部を1つずつ真似して取り入れ、そのたびに評価がどれだけ上がるかを記録する、という地道な検証です。最後まで進めたとき、秘伝のソース(自己注意)を一切使っていないのに評判の店に並んだ、というのが結論でした。つまり「Transformer の勝因の多くは、自己注意そのものより、学習レシピやネットワーク構成といった設計の積み重ねにある」ことを示したのです。

この「近代化のロードマップ」という見せ方そのものが論文の価値で、結果のモデルだけでなく「どの変更が何ポイント効いたか」という過程が再利用可能な知見として残りました。

## 基礎: 前提となる概念
言葉を噛み砕いておきます。

- CNN(畳み込みニューラルネットワーク)とは、小さなフィルタ(例えば 3x3 の窓)を画像の上で滑らせ、局所的な模様(エッジ、テクスチャ)を検出する仕組みのモデルです。フィルタを全域で共有するため、物体が画面のどこにあっても同じように検出でき(並進不変性)、近くのピクセルを重視する性質(局所性)を最初から持ちます。これらを帰納バイアス(inductive bias、モデルがあらかじめ持っている前提)と呼びます。帰納バイアスがあると、少ないデータでも効率よく学べる傾向があります。
- バックボーンとは、生のピクセルから意味のある特徴(車のエッジ、路面のテクスチャなど)を抽出する土台部分のことです。直訳すると「背骨」です。
- ResNet とは、He らが 2016 年に提案した CNN で、残差接続(入力をそのまま出力に足し戻す近道。skip connection、出力 = x + f(x))によって非常に深いネットワークの学習を可能にした金字塔です。ConvNeXt はこの ResNet-50 を土台にします。
- ResNeXt とは、ResNet の畳み込みをグループ化(チャンネルを複数グループに分けて独立に畳み込む)して効率を上げた派生で、ConvNeXt の「深さ方向分離畳み込み」へつながる中間ステップです。
- アブレーション(ablation)とは、要素を1つずつ足したり引いたりして、それぞれが性能にどれだけ寄与するかを測る実験手法のことです。ConvNeXt はこれを「近代化のロードマップ」として正面から使いました。
- 受容野(receptive field)とは、出力の1点が入力画像のどれだけ広い範囲の情報を見て決まっているか、を表す概念です。広いほど大域的な文脈を捉えられます。自己注意は理屈上1層で全域を見られますが、CNN は層を重ねるかカーネルを大きくして広げます。

## 仕組みを詳しく
ConvNeXt は ResNet-50(ImageNet top-1 約 76.1%、ただし当時の素朴な学習レシピ基準)を出発点に、以下の変更を段階的に積み上げ、最終的に約 82.0% へ到達しました。代表的なステップを、論文のロードマップ順に挙げます。

### 1. 学習レシピの近代化(最も効いた)
エポック数(全データを何周学習するか)を 90 から 300 に増やし、最適化器を SGD から AdamW に、データ拡張を Mixup・CutMix・RandAugment・Random Erasing など強いものに、加えてラベルスムージング(正解ラベルを少しぼかして過信を防ぐ)や Stochastic Depth(層をランダムに飛ばす正則化)、Layer Scale(各残差ブロックの出力を小さな学習可能係数でスケールする)を採用しました。アーキテクチャを一切変えずこのレシピだけで、ResNet-50 が約 76.1% → 約 78.8% に上がりました。約 2.7 ポイントの改善で、これは「同じ CNN でも学習法を変えるだけで大きく伸びる」ことの証拠です。Transformer の高精度の少なからぬ部分が、実はこの近代的な学習レシピに由来していたことを示す重要な数字です。

### 2. マクロ設計
- ステージ比の変更: ResNet の各解像度ステージのブロック数を (3,4,6,3) から、Swin の (1:1:3:1) に倣って (3,3,9,3) に変更。深い意味を扱う第3ステージを厚くしました(+0.6 ポイント程度)。
- パッチ化ステム: 入力を最初に処理する層を、ResNet の重なりのある 7x7 stride2 畳み込み + プーリングから、Swin 風に「4x4 の非重複パッチに区切って一気に 1/4 へ縮小する」方式に変更。

### 3. ResNeXt 化と逆ボトルネック
- 深さ方向分離畳み込み(depthwise convolution): チャンネルごとに独立して空間方向の畳み込みを行う軽量な畳み込みで、自己注意の「空間方向の情報混合」に対応する役割を担います。これと 1x1 畳み込み(チャンネル方向の混合)を分けることで、Transformer のブロック構造(空間混合 + チャンネル混合 MLP)に近づけます。これにより FLOPs が下がるので、その分チャンネル幅を 64 から 96 に広げて精度を回復します。
- 逆ボトルネック(inverted bottleneck): Transformer の MLP のように、ブロック内で一度チャンネルを4倍に広げてから戻す構造に変更。MobileNetV2 由来の構造です。

### 4. 大カーネル化
depthwise 畳み込みのフィルタ窓を 3x3 から 7x7 に拡大しました。Transformer の広い受容野を、畳み込みのカーネルを大きくすることで真似たものです。depthwise なので計算量の増加は抑えられます(+0.7 ポイント)。あわせて depthwise 層をブロックの先頭に移動し、Transformer の「注意が先、MLP が後」という順序に揃えます。

### 5. マイクロ設計
- 活性化関数を ReLU から GELU(滑らかな活性化)に変更し、しかも数を減らす(ブロックに1つだけ)。
- 正規化を Batch Normalization から Layer Normalization に変更し、こちらも数を減らす。
- ダウンサンプリングを各ブロック内ではなく専用の層(2x2 stride2 conv + LN)に分離する。

形状の例: 入力 224x224x3 → パッチ化ステム後 56x56x96 → 各ステージで解像度半分・チャンネル倍 → 最終 7x7x768。これを大域平均プーリングして分類します。チャンネルの並び 96, 192, 384, 768 は Swin-T と同一で、設計の収斂を象徴しています。重要なのは、ここまで自己注意を一度も使っていないという点です。

## 手法の系譜と主要論文
- 2016年(CVPR): He らの ResNet(arXiv:1512.03385)が残差接続で超深層 CNN を可能にし、ConvNeXt の骨格そのものを与えました。
- 2017年(CVPR): Xie らの ResNeXt がグループ畳み込みを導入し、ConvNeXt の depthwise 化の前段となりました。
- 2021年(ICCV): Liu らの Swin Transformer([Swin Transformer (V1)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0408_swin-transformer))が窓注意と階層構造で検出・分割を制し、「CNN は時代遅れか」という空気を作りました。ConvNeXt が真似た設計の多く(ステージ比、パッチ化、大受容野、LayerNorm、逆ボトルネック)はここから来ています。
- 2022年(CVPR): Liu, Mao, Wu, Feichtenhofer, Darrell, Xie らの ConvNeXt(arXiv:2201.03545、Facebook AI Research と UC Berkeley)が本トピック。動機は「Transformer の勝因の切り分け」。新規性は新発明ではなく、既存技術の体系的な統合と、それによって純 CNN が Transformer に並ぶことの実証です。
- 2023年(CVPR): 同チームの ConvNeXt V2([ConvNeXt V2](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0351_convnext-v2))が、V1 のアーキテクチャをほぼ引き継ぎつつ、FCMAE という自己教師あり事前学習と GRN 層を追加し、「CNN を MAE 型の自己教師ありに対応させる」という V1 の積み残しを解決しました。

## 論文の実験結果(定量データ)
- ImageNet-1K 分類(top-1 accuracy、最も確からしい予測が正解と一致する割合): ConvNeXt-T(Tiny、約 2900 万パラメータ、約 4.5 GFLOPs)は約 82.1%、ConvNeXt-S は約 83.1%、ConvNeXt-B(Base)は約 83.8%、ConvNeXt-L は約 84.3%。同規模・同学習条件の Swin-T(約 81.3%)/ Swin-B(約 83.5%)を一貫してわずかに(約 0.3〜0.8 ポイント)上回りました。ImageNet-22K で事前学習した ConvNeXt-XL を 384x384 で評価すると約 87.8% に達しました。
- COCO 物体検出・インスタンス分割: ConvNeXt をバックボーンにした Cascade Mask R-CNN は、同規模の Swin を box AP / mask AP で同等以上に。ConvNeXt-L で box AP 約 54〜55 級を達成しました。
- ADE20K セマンティックセグメンテーション: mIoU(予測領域と正解領域の重なりのクラス平均)で Swin と同等以上の水準を記録しました(ConvNeXt-L で val mIoU 約 53.7)。
- 推論スループット: 同精度帯で Swin と同等か、GPU 実機ではしばしば ConvNeXt の方が高速、と報告されています。純畳み込みは行列積として高度に最適化されており、窓シフトやマスクのような不規則な処理が不要なためです。

近代化の各ステップの寄与(アブレーション)が論文の主役です。学習レシピだけで +2.7 ポイント(76.1→78.8)、ステージ比で +0.6、パッチ化ステムで +0.1、depthwise 化はいったん精度を下げるが幅拡張(64→96ch)で +0.7、逆ボトルネックで微増、大カーネル化(7x7)で +0.7、マイクロ設計(GELU/LN/層削減/ダウンサンプル分離)で合計 +0.7 前後、と積み上がり最終 82.0% に達します。この「どの工夫が何ポイント効いたか」を明示したことが研究としての価値で、「Transformer の勝因は自己注意という単一要素ではなく、学習レシピと設計の総合力である」という結論を数値で裏付けました。

## メリット・トレードオフ・限界
メリット:
- 自己注意を使わない純 CNN なので、実装がシンプルで、推論が高速・省メモリ。車載 GPU のような実行環境に向きます。
- ResNet 由来の局所性・並進不変性という帰納バイアスを保ちつつ、Transformer 並みの精度を出します。中規模データでも安定して学習できます。
- depthwise + 大カーネルで、計算量を抑えながら広い受容野を確保できます。
- マルチスケールの階層特徴(96/192/384/768 の4段)を出すため、検出・分割のバックボーンとして差し替えやすい。

トレードオフ・限界:
- 純粋に新しいアイデアというより既存工夫の統合であり、「なぜ効くか」の理論的説明は限定的です。
- V1 時点では MAE 型の自己教師あり事前学習に対応できず、ラベルなし大規模学習の恩恵が薄い(これが V2 で解決)。
- 7x7 の大カーネル depthwise 畳み込みは、ハードウェアや実装(古い cuDNN など)によっては最適化が不十分で、理論計算量ほど速くならない場合があります。
- 超大規模データで青天井に伸びるかという点では、CNN の帰納バイアスがかえって上限になる場面が指摘されており、JFT 級の超大規模では ViT 系に分があるとの報告もあります。

## 発展トピック・研究の最前線
ConvNeXt は「CNN と Transformer は収斂しつつある」という見方を決定づけました。以降、大カーネル畳み込みで Transformer 並みの受容野を狙う研究が活発化し、RepLKNet は 31x31 という巨大カーネルを構造的再パラメータ化(学習時は複数の小カーネルを並列に持ち、推論時に1枚へ畳み込んでまとめる手法)で扱い、SLaK はさらに疎な超大カーネルを提案しました。一方、自己注意と畳み込みを混ぜるハイブリッド(CoAtNet、MaxViT)も主流の一角です。ConvNeXt 自身は V2 で自己教師あり事前学習へ対応し、基盤モデル路線に合流しました。研究上の論点としては、帰納バイアス(CNN の局所性)と汎用性(Transformer の柔軟さ)のどちらをどれだけ持たせるのが最適か、データ規模に応じてその最適点がどう動くか、が未解決のまま残っています。自動運転では、推論レイテンシの読みやすさと省メモリ性から、ConvNeXt 系が車載知覚バックボーンの実用候補として継続的に検討されています。
