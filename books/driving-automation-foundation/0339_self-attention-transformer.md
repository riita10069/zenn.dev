# 自己アテンションとTransformer

## ひとことで言うと

自己アテンション(self-attention)とは、1つの系列(単語の列や画像パッチの列)の中で、各要素が同じ系列内の他のすべての要素との関係を一度に計算する仕組みです。Transformer(トランスフォーマー)は、この自己アテンションを主部品として組み上げたニューラルネットワークの全体構造で、2017年に登場して以来、自然言語処理から画像認識、自動運転の知覚まで、深層学習のほぼあらゆる分野の土台になっています。「再帰も畳み込みも使わず、アテンションだけで系列を処理する」という発想が革命的でした。

## 直感的な理解

「彼は銀行(bank)で釣りをした」と「彼は銀行(bank)でお金を下ろした」という2つの文を考えます。同じ "bank" でも、前者は川岸(土手)、後者は金融機関です。どちらの意味かを決めるには、"bank" が同じ文の中の "釣り" や "お金" という他の単語を参照する必要があります。自己アテンションは、まさにこの「ある単語が同じ文の中の他の単語をどれだけ参照するか」を計算する仕組みです。

自己アテンションの特徴は、参照が全方向・同時であることです。RNNのように左から右へ順番に読むのではなく、文中のすべての単語ペアの関係を1回の行列演算で同時に計算します。「全員が全員を見て、誰を重視するか決める」会議のようなものです。この全結合的な関係づけのおかげで、文の最初の単語と最後の単語も直接結びつけられ、しかもGPUで並列に計算できます。

## 基礎: 前提となる概念

埋め込み(embedding)。単語や記号を、意味を表す数値ベクトルに変換することです。例えば "cat" を [0.2, -0.1, 0.5, ...] のような512次元のベクトルにします。似た意味の単語は似たベクトルになるように学習されます。Transformer の入力は、まずこの埋め込みを通った数値ベクトルの列です。

クエリ・キー・バリュー(Q/K/V)。アテンションの3つの役割で、クエリは「探すための問い合わせ」、キーは「各要素の見出し」、バリューは「各要素の中身」です(詳しくは [アテンション機構](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0320_attention-mechanism) を参照)。自己アテンションでは Q・K・V すべてを同じ入力から作るのがポイントです。

残差接続(residual connection)とレイヤー正規化(layer normalization)。残差接続は、ある層の出力にその層への入力をそのまま足し戻す工夫(output = layer(x) + x)で、深いネットワークでも勾配が伝わりやすくなり学習が安定します。レイヤー正規化は、ベクトルの各要素を平均0・分散1付近に整える処理で、これも学習を安定させます。Transformer はこの2つを各部品の後ろに必ず付けています。

## 仕組みを詳しく

Transformer は「エンコーダ」と「デコーダ」の2つのブロックの積み重ねでできています。翻訳タスクで言えば、エンコーダが入力文(英語)を理解し、デコーダが出力文(日本語)を生成します。元論文ではエンコーダ6層、デコーダ6層を積み重ねました。

エンコーダの1層は次の部品からなります。

1. マルチヘッド自己アテンション。入力系列が自分自身を参照する。「単語間の関係」を捉える。
2. 残差接続 + レイヤー正規化。
3. フィードフォワードネットワーク(FFN)。各位置を独立に通す2層の全結合ネット(例: 512→2048→512次元)。位置ごとの非線形変換を担う。
4. 再び残差接続 + レイヤー正規化。

デコーダの1層は、これに加えて「マスク付き自己アテンション」と「エンコーダ-デコーダ交差アテンション」を持ちます。マスク付き(masked)とは、文を生成するときに「まだ生成していない未来の単語を見ない」ように、未来の位置のスコアを負の無限大にしてソフトマックス後に0にする処理です。これがないと「カンニング」になり、推論時と学習時で挙動が食い違ってしまいます。交差アテンションは、デコーダのクエリでエンコーダの出力(キー・バリュー)を参照し、入力文の情報を取り込みます。

位置エンコーディング(positional encoding)が重要な部品です。自己アテンションは集合演算なので、そのままでは単語の順序を区別できません(「犬が猫を追う」と「猫が犬を追う」が同じ扱いになる)。そこで各位置に固有の信号を埋め込みに足します。元論文では、異なる周波数のサイン・コサイン関数を使いました。

  PE(pos, 2i) = sin(pos / 10000^(2i/d))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

ここで pos は系列内の位置(0,1,2,...)、i は次元のインデックス、d は埋め込み次元です。位置ごとに異なる波形パターンが割り当てられ、ネットワークが相対位置を推測できるようになります。後年の研究では学習可能な位置埋め込みや、相対位置エンコーディング(RoPE など)も広く使われます。

テンソルの形で全体像を追ってみます。入力が「8単語の文」、埋め込み次元 d=512 とすると、入力埋め込みは [8, 512]。位置エンコーディングを足しても [8, 512]。マルチヘッド自己アテンション(8ヘッド、各64次元)を通っても [8, 512]。FFN(512→2048→512)を通っても [8, 512]。つまり1層を通るたびに形は変わらず、中身(各単語の表現)だけが文脈情報を取り込んで豊かになっていきます。これを6回繰り返したものがエンコーダ出力です。

```
入力埋め込み [8,512] + 位置エンコーディング
        │
   ┌────▼─────────────────────┐
   │ マルチヘッド自己アテンション │ ← 単語間の関係
   │   + 残差 + 正規化          │
   │ フィードフォワード         │ ← 位置ごとの変換
   │   + 残差 + 正規化          │
   └────┬─────────────────────┘  ← これを6層積む
        ▼
   エンコーダ出力 [8,512]
```

## 手法の系譜と主要論文

直接の出発点は Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin の "Attention Is All You Need"(NeurIPS 2017, arXiv:1706.03762)です。タイトル通り「アテンションだけあればよい」と宣言し、RNN・CNNを使わずに系列変換タスクで最高性能を出しました。

この構造はすぐに事前学習モデルの基盤になりました。Devlin らの BERT(NAACL 2019, arXiv:1810.04805)はエンコーダだけを使い、大量のテキストで「穴埋め問題」(masked language modeling)を解かせて言語表現を事前学習しました。Radford らの GPT 系列(2018〜)はデコーダだけを使い、次の単語を予測する自己回帰的な生成で大規模言語モデルへと発展しました。現在の対話型AIの直接の祖先です。

画像分野へは Dosovitskiy らの Vision Transformer(ViT, ICLR 2021, arXiv:2010.11929)が持ち込みました。画像を16×16ピクセルのパッチに分割し、各パッチを単語のように埋め込んで自己アテンションを適用しました。Liu らの Swin Transformer(ICCV 2021, arXiv:2103.14030)は、アテンションを局所ウィンドウに区切り、ウィンドウをずらしながら計算量を抑える階層型を提案し、検出やセグメンテーションの基盤になりました。

自動運転では、Carion らの DETR(ECCV 2020, arXiv:2005.12872)が物体検出を「集合予測問題」として Transformer で解き、Li らの BEVFormer(ECCV 2022, arXiv:2203.17270)が空間クエリでマルチカメラ特徴を鳥瞰図(BEV)に集約しました。これらはマルチカメラ・特徴融合の中心的な技術になっています。

## 論文の実験結果(定量データ)

Vaswani ら(2017)の主結果は機械翻訳です。WMT 2014 英独翻訳で Transformer(big)が BLEU 28.4 を達成し、当時のアンサンブルを含む最高記録を2以上更新しました。BLEU は翻訳の良さを0〜100で測る指標で、高いほど良い。英仏では BLEU 41.8。学習コストは、当時の競合が数週間〜膨大な計算を要したのに対し、Transformer(big)は8枚のP100 GPUでおよそ3.5日、base モデルは約12時間と報告され、並列化による効率の高さを示しました。

論文は計算量の比較も提示しています。系列長 n、表現次元 d のとき、自己アテンションの1層あたり計算量は O(n²·d)、RNNは O(n·d²)。系列長 n が表現次元 d より小さい典型的な文(n が数十、d が512)では自己アテンションの方が有利で、しかも逐次的な処理ステップ数が RNN の O(n) に対して自己アテンションは O(1)(全部同時)なので、並列性が圧倒的に高いと示しました。

ViT(2021)の結果も系譜上重要です。大規模データ JFT-300M(約3億枚)で事前学習した ViT は、ImageNet で約88.5%のトップ1精度を報告し、同規模の畳み込みネットと同等以上を、より少ない事前学習計算量で達成しました。一方で ImageNet のみ(約130万枚)で学習すると、同規模のCNNにやや劣ることも示され、「Transformer はデータ量が効く」という教訓が定量的に裏付けられました。

## メリット・トレードオフ・限界

メリットは、長距離依存を1ステップで捉えられること、高い並列性でGPU学習が速いこと、そして同じ構造が言語・画像・音声・センサーなど多様なデータに転用できる汎用性です。実際、Transformer はモダリティを問わない「万能アーキテクチャ」として広がりました。

トレードオフは計算量とメモリです。自己アテンションは系列長 n の2乗に比例するため、長文書や高解像度画像、長い動画では計算が爆発します。また Transformer は強い「帰納バイアス」(モデルが持つ事前の前提)を持たないため、CNN が画像で持つ「近くのピクセルは関連が強い」といった前提がなく、その分を大量のデータで補う必要があります。データが少ない環境では過学習しやすい。

限界として、極端に長い系列での性能劣化(系列の中ほどの情報を取りこぼす lost-in-the-middle 現象)、推論時の自己回帰生成が逐次的で遅いこと、そしてアテンション重みが人間にとって直感的でも因果的説明とは限らないこと、が挙げられます。

## 発展トピック・研究の最前線

効率化のための長系列対応(FlashAttention、線形アテンション、状態空間モデル Mamba など Transformer 代替の探索)が活発です。マルチモーダル化では、画像・テキスト・センサーを共通のトークン列として扱い1つの Transformer で処理する流れが進んでいます。自動運転では、複数時刻・複数カメラの特徴を時空間クエリで融合する時空間 Transformer や、知覚から予測・計画までを1つの Transformer で end-to-end に解く研究(UniAD などの統合型アーキテクチャ)が最前線です。変形可能アテンション(deformable attention)によって、高解像度の特徴マップでも数点のサンプルだけを参照して計算量を抑える工夫が、実用上の鍵になっています。
