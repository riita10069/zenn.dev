# 世界モデルへの意図条件付けと低次元ボトルネックの解消

## ひとことで言うと

自動運転や具現化AIのモデルには「自分がこう動いたら、次の瞬間の世界はこう見えるはず」を予測する部品(世界モデル、world model)を組み込むことがあります。この予測を正しく行うには「自分はこれからどう動くつもりか(意図、intent)」を世界モデルに伝える必要があります。本稿のテーマは、その意図を「ごく少数の数字に潰した低次元ベクトル」で渡すのをやめ、プランナ(軌跡を作る部品)が内部に持つ豊かな高次元の隠れ状態をそのまま渡す、という設計判断です。情報の入り口を細い管から太い管へ付け替える、というイメージです。

## 直感的な理解

人間が「次に何が見えるか」を頭の中で予想するとき、自分の行動計画と密接に結びつけています。右に曲がろうと思えば右側の景色が正面に回り込んでくると予想できますし、ブレーキを踏もうと思えば前方の車との距離が縮まらないと予想します。つまり「未来の見え方の予測」は「自分の行動意図」を入力として初めて成立します。

ここで問題になるのが、その意図をどれだけの情報量で伝えるかです。仮に「曲率」「加速度」といった2つの数字だけ渡したとしても、行動の大枠は伝わります。しかし実際の運転意図はもっと豊かです。どのレーンへ移ろうとしているのか、どの障害物を避けようとしているのか、加速のプロファイルはどうか、横断歩行者への譲り方はどうか。これらを2個や14個の数字に押し込めると、本来予測に役立つはずの情報が削ぎ落とされます。世界モデル側は「曖昧な意図」しか受け取れず、未来予測の質に上限ができてしまいます。これが低次元ボトルネック(low-dimensional bottleneck)です。

解決策は単純で、意図を表す情報をもっと太い帯域で渡すことです。プランナは軌跡を作る過程で、すでに数百次元の内部状態(隠れ状態、hidden state)を計算しています。これは「予測ホライズン全体にわたって自車をどう動かすか」を要約した、意図そのものに近い表現です。これを潰さずにそのまま世界モデルへ渡せば、ボトルネックは解消します。

## 基礎: 前提となる概念

世界モデル(world model)とは、環境の状態遷移を学習したニューラルネットのことです。「現在の状態 s と行動 a を入力すると、次の状態 s' の予測を返す」関数 p(s' | s, a) を近似します。強化学習や自動運転では、これを使ってエージェントが「頭の中でシミュレーション(imagined rollout)」を行い、実環境で試す前に計画を吟味できます。

行動条件付け(action conditioning)とは、世界モデルの予測に「どんな行動を取るか」を条件として与えることです。同じ現在状態でも、アクセルを踏むかブレーキを踏むかで未来は変わります。条件を変えて未来を予測し直すことで「もしこう行動したら(counterfactual、反実仮想)」を比較できます。

BEV(Bird's-Eye-View、鳥瞰図)特徴とは、車を真上から見下ろした座標系に並べた特徴マップのことです。複数のカメラやLiDARの情報を統合し、`[B, C, H, W]` という形(B はバッチ、C はチャネル数=特徴の次元、H と W は鳥瞰格子の縦横)のテンソルで表します。運転の世界モデルは、この BEV 特徴の未来版を予測することが多いです。

隠れ状態(hidden state)とは、再帰型ネットワーク(RNN/GRU)やトランスフォーマが内部に保持する中間表現です。入力系列を処理する過程で「これまで分かったこと」を圧縮して持ち、次の出力を決める材料にします。プランナの隠れ状態は、軌跡生成のために自車運動の文脈を要約したもので、意図のリッチな表現とみなせます。

情報ボトルネック(information bottleneck)とは、Tishby らが定式化した原理で、表現を圧縮するとタスクに不要な情報だけでなく必要な情報まで失う危険がある、という考え方です。圧縮率を上げすぎると下流タスクの性能が頭打ちになります。意図を14次元に潰す設計はまさにこの罠にはまっており、次元を拡張することがボトルネック解消にあたります。

## 仕組みを詳しく

典型的な実装では、世界モデルは次のような処理で未来 BEV 特徴を生成します。意図条件付けの部分を中心に追います。

入力は2つです。

- 融合特徴 `fused_features` `[B, C, H, W]`(C は埋め込み次元、たとえば 256)。現在のシーンを表す BEV 風の特徴マップです。
- 意図ベクトル `ego_hidden` `[B, D]`(D は隠れ次元、たとえば 256)。プランナの隠れ状態です。

意図を空間特徴へ注入する最も単純な方法は、broadcast 加算によるバイアス注入です。

```
bias = ego_proj(ego_hidden)          # Linear: [B, D] -> [B, C]
bias = bias.view(B, C, 1, 1)         # [B, C, 1, 1]
conditioned = fused_features + bias  # [B, C, H, W] に全マス共通で加算
```

`ego_proj` は全結合層(Linear)で、意図ベクトルをチャネル次元 C に合わせます。`view` で空間次元を `1, 1` にしてから加算すると、PyTorch のブロードキャスト規則により H×W のすべての格子点に同じバイアスが足されます。こうして「自車の意図」がシーン全体の特徴に均一に注入されます。

数値例で見ます。`fused_features` のある1チャネルが空間平均でおよそ 0.4 の活性を持っていたとして、意図から計算したバイアスがそのチャネルで +0.8 だとすると、加算後の活性はおよそ 1.2 になります。意図に応じて特徴マップ全体のあるチャネルが押し上げられ、後段の畳み込みがそのチャネルを「意図の手掛かり」として使えるようになります。

もう一段リッチにする場合、より表現力の高い条件注入として AdaLN(Adaptive Layer Normalization、適応的レイヤ正規化)を使います。これは正規化後の特徴に、条件から計算したスケール γ とシフト β を掛け足しする方式で、画像生成の DiT(Diffusion Transformer, Peebles & Xie, ICCV 2023)で広く使われます。単純な加算より、条件が特徴の分散まで制御できるぶん表現力が高いとされます。

行動条件付き(action-conditioned)に拡張する場合は、平坦化した軌跡 `[B, T*S]`(T は時刻数、S は信号数)も第2のバイアスとして射影・加算します。

```
action_bias = action_proj(trajectory.flatten(1)).view(B, C, 1, 1)
conditioned = fused_features + ego_bias + action_bias
```

これにより「軌跡 A で走ったら未来はどう見えるか」「B ならどうか」という反実仮想ロールアウトが可能になります。意図ベクトルが「全体の方針」を、行動ベクトルが「具体的な操作系列」を担う二段構えです。

最後に未来特徴を生成します。条件付けた特徴を畳み込みに通し、複数時刻分の未来 BEV へ展開します。たとえば 6.4 秒先までを 1.6 秒間隔で 4 枚予測するなら、出力は `[B, C, H, W]` を 4 枚分、合わせて `[B, 4*C, H, W]` を 4 分割する形で得ます。

```
future = conv_stack(conditioned)     # [B, 4*C, H, W]
future_frames = future.chunk(4, dim=1)  # [B, C, H, W] × 4
```

メモリ上の注意があります。フル解像度 BEV(たとえば 450×300)で `4*C` チャネルを fp32 で持つと、1サンプルあたり `4*256*450*300*4 byte ≈ 553 MB` に達します。バッチを組むと容易に GPU メモリを溢れさせるため、bf16/fp16 の混合精度や空間方向のダウンサンプリングが事実上必須になります。

なぜ「意図を低次元に潰さず隠れ状態をそのまま渡す」設計が良いのか、その理由を整理します。第一に、視覚の生情報は融合段やプランナの段ですでに高次元のまま処理されており、わざわざ世界モデルの入り口で14次元に潰す必然性がありません。第二に、潰した視覚ベクトルは「見えているもの」を伝えるだけで「これからどう動くか」を含みません。世界モデルが本当に欲しいのは後者です。プランナの隠れ状態はまさに「これからどう動くか」を要約しているので、意味的に正しい条件です。第三に、プランナ非依存になります。隠れ状態という共通インターフェースを入り口にすれば、プランナの中身が GRU でも生成系(拡散・Flow Matching)でも、同じ世界モデルに繋げます。

## 手法の系譜と主要論文

世界モデルへの意図・行動条件付けの考え方には系譜があります。

Ha & Schmidhuber, World Models, NeurIPS 2018(arXiv:1803.10122)。世界モデルの古典です。視覚を VAE で潰した潜在 z と、行動 a を入力に、RNN(MDN-RNN)が次の潜在を予測する構成を示しました。重要なのは「行動を条件に未来を予測すれば、エージェントは頭の中で練習(dream)できる」という発想です。本稿の行動条件付けの源流はここにあります。

Hafner et al., Dreamer 系(Dream to Control, ICLR 2020 ほか)。潜在空間の世界モデルを使い、想像上のロールアウトだけで方策を学習しました。条件として行動を与えて未来潜在を伸ばす点が共通します。

Hu et al., MILE, NeurIPS 2022(arXiv:2210.07729)。運転に特化した世界モデルで、BEV 表現と行動を条件に未来の運転シーンを予測します。意図・行動を豊かに条件付けるほど予測が安定するという潮流の中核です。

Hu et al., GAIA-1, 2023(arXiv:2309.17080)。大規模な生成的運転世界モデルで、テキスト・行動・過去映像を条件に未来の運転映像を生成します。条件の帯域を広く取ることが高品質な未来生成に効くことを示しました。

理論面では Tishby & Zaslavsky, Deep Learning and the Information Bottleneck Principle, 2015(arXiv:1503.02406)が、表現の圧縮率と下流性能のトレードオフを定式化しています。意図を14次元へ過度に圧縮する設計は、この原理が警告する「必要な情報まで捨てる」状態に該当します。

条件注入の技術としては Peebles & Xie, DiT, ICCV 2023(arXiv:2212.09748)が AdaLN による条件付けを普及させ、Perez et al., FiLM, AAAI 2018 が「特徴をスケール・シフトで変調する」一般的枠組みを提供しました。単純な加算バイアスはこれらの最も軽量な特殊形と位置づけられます。

## 論文の実験結果(定量データ)

意図・行動条件の帯域が予測品質に効くことは、運転世界モデルの研究で繰り返し報告されています。指標の意味も併せて説明します。

MILE(NeurIPS 2022)は CARLA シミュレータで評価され、運転の総合スコアであるドライビングスコア(成功率と違反のペナルティを掛け合わせた 0〜100 の指標、高いほど良い)で、従来の模倣学習や条件付けの弱いベースラインを上回ったと報告されています。世界モデルが行動を条件に未来 BEV を予測し、その想像を計画に使うことが寄与しています。

GAIA-1 は生成映像の品質を FID(Fréchet Inception Distance、生成画像分布と実画像分布の距離。低いほど実写らしい)などで評価し、条件(行動・テキスト)を与えた生成が無条件生成より一貫性と制御性で優れると報告しています。条件の帯域を狭めると、生成が指定どおりの行動を反映しなくなる(制御性が落ちる)ことがアブレーションで示されています。

アブレーションの一般的な傾向として、意図表現の次元を極端に小さくすると未来予測の再構成誤差(予測 BEV と実際の BEV の差、たとえば MSE や IoU)が悪化することが観察されます。逆に数百次元の隠れ状態を渡すと、同じ世界モデル構造でも予測誤差が下がり、計画品質が改善します。これは情報ボトルネック原理の予測どおりです。具体的な改善幅は構成依存ですが、報告では「低次元条件から高次元隠れ状態へ切り替えることで未来予測の再構成誤差が二桁パーセント単位で改善する」例が示されています。数値の意味としては、未来 BEV のどこに何があるかをより正確に当てられるようになり、その未来を前提にした計画が安全側に働く、ということです。

## メリット・トレードオフ・限界

メリット。低次元から高次元の隠れ状態へ条件を切り替えることで情報ボトルネックを解消し、未来予測の質の上限を引き上げられます。プランナの意図を直接受け取るので、生成される軌跡と未来予測の整合が取りやすくなります。隠れ状態という共通インターフェースを使えばプランナ非依存になり、行動ベクトルも加えれば反実仮想ロールアウトに拡張できます。

トレードオフと限界。第一に、高次元条件と複数時刻の未来チャネル出力はメモリを大きく消費し、高解像度では混合精度や縮小が必須です。第二に、未来予測の質がプランナの隠れ状態の質に依存します。プランナが未熟だと、その劣った意図が世界モデルへ伝播し、未来予測も劣化する連鎖が起きます。第三に、視覚の生情報を世界モデルへ直接渡す経路を捨てると、プランナを経由しない視覚的手掛かり(プランナが軌跡生成に使わなかったが未来予測には有用な情報)が伝わりにくくなる可能性があります。第四に、未解決の課題として「世界モデルの予測を本当に計画に役立てられているか」の評価が難しい点があります。未来予測ロスが下がっても、それが下流の運転性能に結びつくとは限らず、補助タスクとして付けた予測が主タスクと competing(競合)してしまうこともあります。

## 発展トピック・研究の最前線

近年は世界モデルそのものを大規模化する方向が活発です。GAIA-1 や GAIA-2、各社の運転基盤モデルは、行動・テキスト・過去映像を条件に長時間の未来映像を生成し、レアケースのデータ拡張やシミュレーションに使う研究が進んでいます。条件の帯域を「数値ベクトル」から「言語・地図・経路」へ広げる流れがあります。

また、世界モデルの予測誤差を学習信号として主タスクに還元する自己教師あり的なアプローチ(未来特徴再構成ロス)や、不確実性を持った確率的世界モデル(同じ意図でも複数の未来を分布として予測する)も研究されています。後者はマルチモーダルな未来(歩行者が渡るか止まるか)を1つに平均してしまう問題への対処です。
