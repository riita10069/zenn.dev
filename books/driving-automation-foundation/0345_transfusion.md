# TransFusion

## ひとことで言うと
TransFusionは、LiDARの点群からまず物体候補(クエリ)を作り、その候補がTransformerのアテンション(注目の仕組み)を使ってカメラ画像の特徴を必要なぶんだけ取りに行く、カメラ-LiDAR融合の3D物体検出手法です。最大の特徴は、LiDARとカメラの位置合わせ(校正)が多少ずれていても壊れにくいことです。これは、点と画素を1対1で厳密に紐付けるのをやめ、アテンションで「どの画像領域を見るかを学習で柔らかく決める」設計から生まれます。

## 直感的な理解
LiDARの点に画像の色を貼り付ける従来手法を、「住所を頼りに郵便物を配る」やり方にたとえます。住所(校正で計算した投影先の画素)が正確なら問題ありませんが、住所が数軒ぶんずれていたら、まったく違う家に配達してしまいます。カメラとLiDARは別々の機器で、取り付け位置の測定誤差や、車が動いている間の数十ミリ秒の撮影タイミングずれがあるため、この「住所ずれ」が現実に起こります。時速60kmの車は数十ミリ秒で数十cm進むため、わずかな同期ずれでも投影先が物体からはみ出します。

TransFusionの発想は、住所をピンポイントの一軒に固定するのではなく、「だいたいこのあたり」という地区を見渡して、関連の高い建物から重み付きで情報を集めることです。これがアテンションによる柔らかい対応付け(soft association)です。住所が一軒ずれていても、周辺をまとめて見ているので情報がほとんど狂いません。さらに、配るべき相手(物体)をLiDARが先に確定してくれるので、画像側では「どの地区を見るかの当たり」だけ付ければよく、無駄が少ないのも特徴です。

## 基礎: 前提となる概念
いくつかの言葉を噛み砕きます。

hard associationとsoft association。hard association(ハードな対応付け)は、LiDARの各点を校正で画像上の特定の1画素へ厳密に投影し、その画素の特徴を点に貼り付ける従来方式です。1対1の固定対応なので、対応がずれると即座に誤情報になります。soft association(柔らかい対応付け)は、厳密な1対1を避け、関連の高い複数の場所から重み付きで特徴を集める方式で、TransFusionが採るやり方です。どこを見るかをデータから学習で決めるため、対応のずれを内部で吸収できます。

クロスアテンション(cross-attention)。Transformerの中核操作です。あるクエリ(問い合わせ)が、別の情報源(キーとバリュー)に対して「どこをどれだけ見るか」の重み(注意重み)を計算し、その重みで情報(バリュー)を集めます。重みはクエリとキーの類似度をsoftmaxで正規化して作ります。クエリが画像特徴を見れば、画像のどの領域から特徴を取るかを学習で決められます。soft associationはこのクロスアテンションで実現します。

物体クエリ。1つの物体候補を表すベクトルです。DETR(Carion et al., ECCV 2020、物体検出をクエリ集合の予測として定式化し、二部マッチングでNMSを不要にしたTransformer手法)に由来し、クエリ数が検出できる物体数の上限になります。

ヒートマップ。各格子セルが「物体の中心である確率」を表した地図です。確率の高いセルを選べば、物体がありそうな場所が分かります。CenterPoint(Yin et al., CVPR 2021、LiDAR検出を物体中心のヒートマップ検出として定式化した手法)で広く使われた考え方です。

校正(キャリブレーション)。カメラの内部パラメータ(焦点距離など)と外部パラメータ(取り付け位置・向き)、およびカメラとLiDARの相対位置を測った情報です。これを使って3D点を画像画素へ投影します。実車では振動・温度変化・経年で少しずつずれていきます。

## 仕組みを詳しく
TransFusionはTransformerデコーダを2段(2層)重ねた構造で、おおまかに次の流れです。

第1段階、LiDAR BEV特徴の作成。点群をvoxelやpillarに区切ってBEV特徴を作ります([Pillar/Voxel BEVエンコード](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0337_pillar-and-voxel-bev-encoding))。多くはSECOND系のsparse convバックボーンを使います。shape例 [C, BEV_H, BEV_W]。

第2段階、物体クエリの初期化(input-dependent query)。LiDAR BEV特徴から、物体がありそうな場所をヒートマップで予測し、確率の高い上位N個(例えば200個)をクエリの初期位置にします。ここがTransFusionの工夫です。クエリをランダムや固定位置に置くDETRと違い、データに応じて物体がいそうな場所(category-aware、クラスごとの中心らしさ)に置くため、学習が速く収束し、少ない層数(2層)で済みます。各クエリは位置(BEV上のxy)と特徴ベクトル(例えば128次元)を持ちます。shape [200, 128]。

第3段階、第1デコーダ層(LiDARへのアテンション)。各クエリがLiDAR BEV特徴にクロスアテンションし(位置エンコーディング付き)、まずLiDARだけで箱(中心・大きさ・向き・速度・クラス)を粗く予測します。LiDARは距離が正確なので、ここで物体の幾何(位置と形)がほぼ確定します。クエリ同士のself-attentionで重複候補も抑えます。

第4段階、第2デコーダ層(画像へのアテンション、ここが融合の核心)。各クエリが今度はカメラ画像の特徴にクロスアテンションします。重要なのは、校正で大まかな投影位置の当たりはつけるが、そこに厳密に固定せず、アテンションで周辺の画像領域から重み付きで特徴を集める点です。これがsoft associationの実体です。具体的には、第1層で得た箱の中心を画像へ投影し、その近傍にアテンションを局所化(SMCA、spatially modulated cross attention的に重みを中心付近へ誘導)しつつ、ずれを許容する範囲で集めます。さらに「画像誘導クエリ初期化(image-guided query initialization)」という工夫で、画像のヒートマップからもクエリ候補を補い、LiDARが疎で見落とした物体を画像のヒントで拾えるようにします。遠方の小さな物体はLiDAR点がほとんど当たらないため、カメラのヒントが効きます。

第5段階、予測。融合後の各クエリから、最終的な3D箱とクラスを出力します。二部マッチングで学習するためNMSは不要です。クエリ数が検出上限なので、200クエリなら最大200物体です。

対比で整理します。

```
従来(hard association):
  LiDAR点 → 校正で画素を1つ決める → その特徴を貼る
            （住所が1軒ずれたら即誤り。点が当たらない遠方は情報ゼロ）

TransFusion(soft association):
  クエリ → 校正で見る領域の当たりをつける → アテンションで近傍から柔らかく集める
          （住所が1軒ずれても近傍が吸収。画像ヒントで遠方も拾える）
```

論文では、わざと校正にノイズ(平行移動や回転の誤差)を加えて評価し、hard association型の手法が大きく精度低下するのに対し、TransFusionの劣化が小さいことを示しています。これが「校正ずれに頑健」の定量的な根拠です。

## 手法の系譜と主要論文
カメラ-LiDAR融合は、点に画像を貼るhard association型から、アテンションで柔らかく融合するsoft association型へと進化しました。

PointPainting(Vora et al., CVPR 2020)。画像のセグメンテーション結果をLiDAR点に塗ってからLiDAR検出器に通すhard association型の代表です。実装は単純ですが、校正ずれと密度差に弱く、LiDARが当たらない領域には画像情報を渡せない欠点があります。

CenterPoint(Yin, Zhou, Krähenbühl, CVPR 2021, arXiv:2006.11275)。LiDAR検出を「物体中心のヒートマップ検出」として定式化し、アンカー不要で強力なLiDAR単独検出器になりました。TransFusionのヒートマップによるクエリ初期化はこの考え方を直接取り込んでいます。TransFusionの比較ベースラインとしても頻繁に使われます。

DETR(Carion et al., ECCV 2020)とDeformable DETR(Zhu et al., ICLR 2021)。物体検出をクエリ集合の予測としてTransformerで解く枠組みで、TransFusionの物体クエリとデコーダ構造の源流です。Deformable DETRの局所化アテンションの発想は、画像への局所的なクロスアテンションに通じます。

TransFusion(Bai, Hu, Zhu, et al., CVPR 2022, arXiv:2203.11496)。「TransFusion: Robust LiDAR-Camera Fusion for 3D Object Detection with Transformers」。提案は3点です。第1にLiDAR BEVから作るinput-dependentな物体クエリ、第2にTransformerクロスアテンションによるsoft associationでの画像融合、第3に画像誘導クエリ初期化です。動機は、hard associationが校正・時刻ずれと密度差に弱いことを、アテンションで柔らかく対応付けて頑健化することでした。

並べて見ると、TransFusionは「LiDARで正確な幾何を確定し、カメラをアテンションで柔らかく補う」という、LiDAR主導の中間的な立ち位置にあります。これに対し、カメラとLiDARを共通BEVで対等に足す [BEVFusion (カメラ+LiDAR)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0325_bevfusion) は、より対称的な融合設計です。後続のDeepInteraction(NeurIPS 2022)は両モダリティを最後まで別表現で保ち双方向に相互作用させる方向へ、CMT(Cross Modal Transformer, ICCV 2023)は投影を介さず位置エンコーディングで暗黙に対応づける方向へ進みました。

## 論文の実験結果(定量データ)
評価は主にnuScenesとWaymo Open Dataset(Sun et al., CVPR 2020、自動運転向けの大規模データセット)で行われます。指標を噛み砕きます。
- mAP(mean Average Precision)。検出の総合的な正確さ。中心距離しきい値で正解判定し平均。高いほど良い。
- NDS(nuScenes Detection Score)。mAPに位置・大きさ・向き・速度・属性の誤差を加味した総合スコア。高いほど良い。

TransFusionはnuScenes検証セットおよびテストセットで、当時の上位水準を報告しました。代表的にはテストセットでNDSおよそ0.717、mAPおよそ0.689前後で、LiDAR単独の強力なベースライン(CenterPointはテストでNDSおよそ0.65前後)を明確に上回りました。カメラ融合により、特に遠方・小物体(歩行者・自転車・三角コーンなど)の検出が改善しています。これはLiDAR点が当たりにくい遠方を、画像の手がかりで補えるためです。Waymoでも同様にLiDAR単独を上回る改善を報告しています。

最も重要な実験は校正ロバスト性の評価です。論文では、校正に意図的なノイズ(センサ間の平行移動・回転の誤差)を段階的に加える設定で性能を測ります。hard association型の融合手法はノイズが増えるにつれ大きく精度を落とす(数度の回転ずれでmAPが大幅に低下する)のに対し、TransFusionの劣化は緩やかで、実用上重要な「校正が完璧でない現実」での頑健性を定量的に示しました。これは安全性に直結する結果で、TransFusionの最大のセールスポイントです。

アブレーションから読み取れる傾向は次のとおりです。input-dependentなクエリ初期化(ヒートマップ由来)を、ランダム初期化(DETR流)に置き換えると収束が遅くなり、少ない層数では精度が出にくくなります。これは「物体がいそうな場所にクエリを置く」効果の裏づけです。画像誘導クエリ初期化を外すと、遠方小物体の再現率(recall、見落としの少なさ)が下がり、カメラヒントの貢献が確認されます。第2デコーダ層(画像融合)を外すとカメラの効果が消え、ほぼLiDAR単独相当に戻ります。画像へのアテンションを局所化せず全画像に広げると、関係ない領域の特徴を拾って精度が下がり、局所化の意義が示されます。

## メリット・トレードオフ・限界
メリット。厳密な点-画素対応をやめたことで、カメラ-LiDARの校正ずれ・時刻ずれに頑健です。LiDARで正確な幾何を確定しつつ、画像でLiDARの見落とし物体を補えます(LiDARが当たらない領域にも画像情報を回せる)。input-dependentクエリにより少ない層数(2層)で収束し、計算が比較的軽いです。NMS不要のエンドツーエンド設計です。

トレードオフと限界。Transformerデコーダの計算コストがかかります。クエリ数が検出物体数の上限を決めるため、極端に混雑した場面では不足しえます。LiDARを基盤に据える設計のため、カメラ単独では動かず、LiDAR欠損時の挙動はBEVFusion(NeurIPS版)ほど頑健ではありません。そして、soft associationで吸収できるのはあくまで小〜中程度の校正ずれであり、大まかなレベルの校正は依然として必要です(投影位置の当たりをつけるのに使うため、全くの無校正では機能しません)。

未解決の研究課題としては、LiDAR欠損時の動作(LiDAR前提設計の弱点)、より大きな校正ずれや動的な時刻同期ずれへの対応、計算コストの削減、そして時間情報を取り込んだ複数フレーム融合への拡張が挙げられます。

## 発展トピック・研究の最前線
TransFusionが示した「アテンションによる柔らかい融合」は、その後の融合手法に広く影響しました。発展の方向は次のとおりです。

第1に、より深いクロスモーダルアテンションです。LiDARクエリが画像を見るだけでなく、双方向に相互作用させる設計(DeepInteraction)や、BEV空間と画像空間を行き来する設計、投影を介さず位置エンコーディングで暗黙に対応づける設計(CMT)が研究されています。第2に、共通BEVとの統合です。対等融合の [BEVFusion (カメラ+LiDAR)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0325_bevfusion) と、クエリ主導のTransFusionを組み合わせ、両者の長所(頑健性と意味情報の活用)を取り込む流れがあります。第3に、時間融合です。過去フレームのクエリを持ち越すストリーミング型の時間モデリング(スパースクエリ系の [Sparse4D](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0295_sparse4d) と同じ思想)を融合に取り込むことで、速度推定と遮蔽補完を改善できます。

校正ずれ・センサ欠損・悪天候といった現実の劣化条件への頑健性は、自動運転の安全性に直結するため、TransFusionが切り開いたロバスト融合は今も活発な研究テーマです。特に「どの程度のセンサ劣化まで性能を保証できるか」を体系的に測る評価ベンチマークの整備が、実用化に向けて重視されています。
