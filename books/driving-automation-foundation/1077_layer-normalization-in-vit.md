# ViTの正規化と前処理 (LN/patch embed)

## ひとことで言うと

Vision Transformer(ViT、ヴィジョン・トランスフォーマ)は、画像を小さな正方形の断片(パッチ)に切って Transformer に流す画像認識モデルです。このノートでは「画像をどうやって Transformer が食べられる形に変えるか(パッチ埋め込み)」と「学習が途中で爆発しないように数値を整える仕組み(Layer Normalization、略して LN)」という、ViT を理解する土台になる2つの基礎部品を、前提知識ゼロの人向けに丁寧に説明します。この2つは ViT 固有というより、現代の Transformer 全般を理解する鍵です。

## 直感的な理解

Transformer はもともと自然言語処理(文章を扱う AI)のために作られた仕組みです。文章は「単語の列」なので、Transformer は「ベクトル(数字の並び)の列」を入力として受け取るように設計されています。

ところが画像は「縦×横のピクセルの2次元格子」です。たとえば 224×224 のカラー画像は、数で言えば 224×224×3 = 150,528 個の数の塊です(3 は赤・緑・青の3チャンネル)。これをそのまま Transformer に渡すことはできません。ここで2つの問題が出ます。

1. **形の問題**: 2次元の画像を「ベクトルの列」に変換しないと Transformer に入りません。これを解決するのが「パッチ埋め込み(patch embedding)」です。

2. **学習の安定性の問題**: Transformer は層を何十段も重ねた深いネットワークです。何も対策しないと、層を通るたびに数値がどんどん大きく(または小さく)なり、最終的に学習が破綻します(loss が NaN = 計算不能になる、勾配が爆発・消失する)。これを防ぐのが「正規化(normalization)」、特に Layer Normalization です。

「正規化」とは平たく言うと「数値の集団を、平均がおよそ 0・ばらつきがおよそ 1 になるように揃え直す」操作です。テストの点数を偏差値に直すと、難しいテストでも簡単なテストでも同じ尺度で比較できるのと同じ発想です。深いネットでは各層の出力スケールを揃え直すことで、勾配がうまく流れ続けます。なぜスケールを揃えると勾配が流れるのかというと、層をまたいで信号の大きさが極端に変わらなければ、誤差逆伝播で勾配を掛け算していくときに値が指数的に膨らんだり消えたりしにくいからです。

## 基礎: 前提となる概念

- **トークン(token)**: Transformer が処理する1個の単位ベクトル。言語では1単語、ViT では1パッチがトークンになります。
- **埋め込み次元(embedding dimension)**: 各トークンを表すベクトルの長さ。ViT-Base なら 768。大きいほど表現力が増すが計算も増えます。
- **残差接続(residual connection)**: ある層の入力を、その層の出力にそのまま足し算する「近道」配線。`出力 = 入力 + 層(入力)` の形。これがあると、深いネットでも勾配が入力側まで素直に流れ、学習が安定します。
- **アテンション(self-attention)**: 各トークンが他の全トークンを見て「どれに注目すべきか」を重み付きで集約する仕組み。Transformer の中核です。クエリ(Q、質問)・キー(K、見出し)・バリュー(V、中身)という3つのベクトルを各トークンから作り、Q と K の内積で「注目度」を決め、その重みで V を混ぜます。
- **FFN(Feed-Forward Network)**: 各トークンに独立に適用する小さな全結合ネットワーク。アテンションが「トークン間の混ぜ合わせ」なら、FFN は「各トークン内の変換」を担います。
- **勾配(gradient)**: 損失を各重みについて微分した値。重み更新の方向と大きさを決めます。これが大きすぎる(爆発)と発散し、小さすぎる(消失)と学習が止まります。
- **warmup(ウォームアップ)**: 学習の最初だけ学習率を 0 からゆっくり目標値まで上げるスケジュール。初期の不安定な勾配で重みを壊さないための保険です。

## 仕組みを詳しく

### パッチ埋め込み(patch embedding)

224×224×3 のカラー画像を例にステップで分解します。

1. **画像を格子状のパッチに切る**: 16×16 ピクセルの正方形に区切ります。224 ÷ 16 = 14 なので、縦14×横14 = 196 個のパッチができます。各パッチは 16×16×3 = 768 個の数を持ちます。

2. **各パッチを1本のベクトルに潰す**: 16×16×3 の小さな塊を、768 次元のベクトル(768 個の数の並び)に並べ直します。

3. **線形変換で埋め込み次元に変換**: 768 次元のパッチベクトルに学習可能な重み行列を掛けて、決められた次元(ViT-Base なら 768)のベクトルにします。実装上は「カーネルサイズ 16・ストライド 16 の畳み込み(Conv2d)」1個で、切る・潰す・変換するを同時にやるのが定番です。これは「16ピクセルごとにスタンプを押して特徴を1個ずつ出す」操作と同じです。tensor の形は次のように変わります。

   ```
   入力画像:        [B, 3, 224, 224]    (B = バッチサイズ = 同時処理する画像枚数)
       ↓ Conv2d(3→768, kernel=16, stride=16)
   パッチ特徴マップ: [B, 768, 14, 14]
       ↓ 縦横を1列に潰す (flatten)
   パッチ列:        [B, 196, 768]        (196 パッチ × 768 次元)
   ```

4. **クラストークンを足す(任意)**: 分類タスクでは「画像全体の要約」を担う特別なベクトル([CLS] トークン)を先頭に1個追加します。すると 196 + 1 = 197 個になります。検出やセグメンテーションでは [CLS] を使わず、全パッチトークンを使う設計も一般的です。

5. **位置埋め込み(positional embedding)を足す**: Transformer は順番を本質的に区別できないので、「このパッチは左上」「これは右下」という位置情報を学習可能なベクトルとして各パッチに足します。これがないと、画像をバラバラに切ってシャッフルしたのと区別がつかなくなります。アテンションは「集合」を処理する演算で、入れ替えに対して出力が変わらない(順列同変)ため、位置情報を明示的に注入しないと空間構造が失われるのです。

ここまでで、画像が「197 個のベクトルの列」になり、Transformer が食べられる形になりました。パッチサイズを小さくする(例 16→8)とトークン数が4倍になり、細かい構造を捉えられる代わりにアテンションの計算量(トークン数の2乗に比例)が急増します。ここに解像度と計算コストのトレードオフがあります。

### Layer Normalization(LN)

LN は「1つのベクトルの中の数値たちを、平均0・分散1に揃え直す」操作です。あるパッチの 768 次元ベクトルが `[3.0, -1.0, 5.0, ...]` だったとします。

1. このベクトル 768 個の数の平均 μ と標準偏差 σ を計算する。
2. 各要素から平均を引き、標準偏差で割る: `(x - μ) / σ`。これで平均0・分散1に揃う。
3. 学習可能なスケール γ とシフト β を掛けて足す: `γ · (x - μ) / σ + β`。揃えすぎて情報が消えるのを防ぎ、ネットが必要なら元のばらつきを取り戻せるようにする。

重要なのは「どの軸で平均を取るか」です。LN は**1サンプル内の特徴次元方向**で平均を取ります。バッチ内の他の画像とは一切混ぜません。これが Batch Normalization(BN、バッチ内の同じチャンネルの値を集団で正規化する別手法)との決定的な違いです。BN はバッチサイズが小さいと統計が不安定になりますが、LN は1サンプルだけで完結するのでバッチサイズに依存しません。Transformer のように系列長が長くバッチサイズが小さくなりがちな状況では LN が好まれます。また LN は学習時と推論時で挙動が変わらない(BN は推論時に移動平均統計を使う)ため、実装と再現性の面でも扱いやすい利点があります。

### LN を置く位置: Post-LN vs Pre-LN

ここが学習安定性の核心です。Transformer の1ブロックはアテンションと FFN からなり、それぞれに残差接続が付きます。LN をこの残差接続の「どこに」置くかで2流派があります。

**Post-LN(元祖 Transformer のやり方)**:
```
x → Attention → (+x) → LN → FFN → (+) → LN → 出力
```
残差を足した「後」に LN を掛けます。

**Pre-LN(Xiong et al. 2020 が解析した、現在主流のやり方)**:
```
x → LN → Attention → (+x) → LN → FFN → (+) → 出力
```
各サブ層に入る「前」に LN を掛け、残差はそのまま流します。

before / after で何が変わるか。Post-LN だと、層を深くするほど残差を通る信号の大きさが膨らみ、初期段階の勾配が不安定になります。対策として「warmup(学習率を最初だけゆっくり上げる)」が必須でした。Pre-LN だと残差が正規化を経ずに素通りするため、入力から出力まで勾配がまっすぐ流れる「高速道路」ができ、深い層でも勾配が安定します。結果として warmup なしでも学習でき、学習率も大きく取れます。直感的には、Pre-LN は「各サブ層が残差の流れに小さな補正を足すだけ」になり、最悪でも恒等写像(何もしない)に近い振る舞いから学習を始められる、と理解できます。

## 手法の系譜と主要論文

- **Layer Normalization**(Ba, Kiros, Hinton, 2016)。LN の原典。BN がバッチ統計に依存し RNN や小バッチで使いにくい問題を、1サンプル内で正規化することで解決しました。後の Transformer 全盛時代の基盤になります。

- **Attention Is All You Need**(Vaswani et al., NeurIPS 2017)。Transformer の原典で、Post-LN を採用。warmup 付き学習率スケジュール(Noam スケジュール)が事実上必須でした。

- **On Layer Normalization in the Transformer Architecture(Pre-LN)**(Xiong et al., MSRA ほか, ICML 2020)。LN の位置を理論と実験で解析した重要論文。提案は「Post-LN は初期勾配が層の深さに依存して大きくなり不安定。Pre-LN は勾配の期待値が深さによらず有界なので warmup なしで安定する」という分析と Pre-LN の推奨。動機は、当時 warmup が経験則的に必須なのに理由が不明瞭で調整コストが高かったこと。効果は warmup を省いた学習の高速化・簡略化。トレードオフは、Pre-LN は最終精度が Post-LN にわずかに劣る場合がある(最終層手前で正規化が効きにくい)と後続研究で指摘される点です。

- **ViT: An Image is Worth 16x16 Words**(Dosovitskiy et al., Google, ICLR 2021)。画像を 16×16 パッチに分割し、線形射影でトークン列にして、ほぼ素の Transformer に流すだけで画像分類ができることを示しました。内部で Pre-LN を採用。CNN の帰納バイアス(局所性・並進不変性という前提)がなくても、大量データがあれば Transformer がデータからそれを学べる、という主張です。

- **DeepNorm**(Wang et al., Microsoft, 2022)。Post-LN と Pre-LN の折衷で、残差をスケールする係数を導入し、1000 層級の極端に深い Transformer を安定学習できると示しました。正規化位置の研究が今も続いていることの表れです。

- **Swin Transformer V2**(Liu et al., Microsoft, CVPR 2022)。残差を足した後に LN を掛ける「res-post-norm」を採用し、高解像度・大容量へスケールさせたときの学習発散を抑えました。Post-LN と Pre-LN の中間的な安定化の一例で、画像バックボーンを巨大化する際に正規化位置の選択が効くことを示します。

## 論文の実験結果(定量データ)

ViT の実験結果が示す重要な定量事実は「事前学習データ量と精度の関係」です。指標は ImageNet トップ1精度(1000クラス分類で、最も確信したクラスが正解と一致する割合、0〜100%、高いほど良い)。ViT-Large を ImageNet-1k(約128万枚)だけで学習すると、同規模の CNN(ResNet 系)に負ける一方、ImageNet-21k(約1400万枚)や JFT-300M(約3億枚)で事前学習すると逆転して CNN を上回る、と報告されました。JFT-300M 事前学習後の ViT-Huge は ImageNet で約88.5%のトップ1精度を達成しています。アブレーションの核心は「中規模データでは CNN の帰納バイアスが効くが、データを増やすほど ViT がその差を埋めて追い越す」という交差点の存在で、ViT は大量データ・大量計算を必要とするモデルだと結論づけられます。パッチサイズを 16 から 14 や 8 に小さくするとトークン数が増えて精度が上がる一方、計算量が大幅に増えることも示されています。

Pre-LN の効果については、Xiong et al.(2020)が機械翻訳(WMT)や言語モデルで定量比較しています。Post-LN は warmup を外すと学習が発散しますが、Pre-LN は warmup なしでも安定して収束し、同等以上の最終性能に到達したと報告されています。学習を始める段階での勾配ノルム(勾配の大きさ)を測ると、Post-LN は層が深いほど初期勾配が大きくなる一方、Pre-LN は深さによらず有界に保たれる、という理論予測が実測と一致しました。これは「なぜ warmup が必要だったのか」を勾配スケールの観点から説明した点で価値があります。実務的な含意は、Pre-LN を使えばハイパーパラメータ調整(warmup の長さ・ピーク学習率)が楽になる、ということです。

## メリット・トレードオフ・限界

メリット

- パッチ埋め込みにより、画像を Transformer が扱えるトークン列へ統一的に変換できる。系列長(パッチ数)は解像度とパッチサイズで調整可能。
- LN はバッチサイズに依存せず1サンプルで完結するので、小バッチや系列長が変動する状況に強い。学習時と推論時で挙動が同じで再現性が高い。
- Pre-LN により深い Transformer でも warmup なしで安定学習でき、学習率設計が楽になる。

トレードオフと限界

- パッチ分割は固定サイズなので、パッチをまたぐ細かい局所構造の表現はパッチ内に閉じがち(Swin はウィンドウ+シフトで緩和)。
- ViT は中規模データだと CNN に劣り、大量データ・大量計算前提になりやすい。
- LN は各サンプルで毎回平均・分散を計算するため、わずかな計算オーバーヘッドがある。
- Pre-LN は学習安定の代償に最終精度がわずかに犠牲になる場合があり、正規化位置の選択はモデルごとに検討が要る(res-post-norm や DeepNorm はこの妥協点を探る試み)。
- **未解決の課題**: 「どの正規化位置・どのスケーリングが、どの深さ・解像度・データ規模で最適か」を事前に決める一般理論はまだなく、経験的な探索が残っています。

## 発展トピック・研究の最前線

正規化まわりの研究は活発です。RMSNorm(平均減算を省き分散だけで正規化する軽量版)は大規模言語モデルで広く採用され、計算を減らしつつ安定性を保ちます。QK-Normalization(アテンションのクエリ・キーを正規化してロジットの暴走を防ぐ)は、ViT を巨大化(ViT-22B, Dehghani et al., 2023 など)する際に学習発散を抑える鍵として報告されました。ViT-22B では QK-Norm を入れないと数千億〜数兆パラメータ規模でアテンションのロジットが発散して学習が崩壊することが示され、「正規化はスケールするほど主役になる」ことが裏づけられました。パッチ埋め込み側でも、重なりを持たせる畳み込み的トークン化(CvT、Swin の patch merging)や、解像度を可変にする工夫(FlexiViT のように可変パッチサイズで学習する)が進んでいます。正規化と前処理は地味に見えて、モデルを大規模化・高解像度化するたびに発散しやすくなる現代では、安定学習の最重要因子であり続けています。LN を直接の主役と意識しなくても、アテンションの後に LN を掛けて残差と組み合わせる「LN + アテンション + FFN」の Transformer ブロック構造は、ViT を直接使わない画像・BEV 融合モジュールにも広く浸透しています。
