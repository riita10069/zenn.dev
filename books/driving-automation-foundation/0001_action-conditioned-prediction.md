# 行動条件付き潜在予測 (counterfactual rollout)

## ひとことで言うと
「もし自車がこの軌道Aで走ったら、周りの世界はどう見えるようになるか」を予測させる仕組みです。未来を予測するときに「自分が取る予定の行動(計画した軌道)」を入力として与えて条件付けする点が核心です。これにより「軌道Aを取ったらこうなる」「軌道Bを取ったらこうなる」という、まだ実際には起きていない仮定の未来(反実仮想、counterfactual)を表現でき、行動の良し悪しを実車で試す前に内部で比較できます。

## 直感的な理解
未来は「自分が何をするか」で変わります。交差点を例にすると、自分がブレーキを踏んで止まる軌道を選べば前方の歩行者は安全に横断し続けますが、アクセルを踏んで前進する軌道を選べば歩行者は立ち止まるか、最悪ぶつかります。つまり「未来の世界の見え方」は「自分の行動」次第で分岐します。

この「行動が違えば未来も違う」という関係を陽に扱えないと、安全な意思決定はできません。普通の未来予測モデルは行動を入力に取らず、「今こう見えているからたぶん次はこう」という1通りの未来しか出しません。これでは「軌道Aと軌道Bでどちらが安全か」を比較できません。行動条件付き予測(action-conditioned prediction)は、予測の計算に行動の情報を一緒に入れ、出力が行動に応じて変わるようにすることで、この比較を可能にします。

身近な比喩で言えば、チェスの上級者が「この手を指したら相手はこう来る、別の手なら相手はこう来る」と複数の分岐を頭の中で読むのと同じです。行動条件付き予測は、機械にこの「もし〜なら」の読みを与える仕組みです。

## 基礎: 前提となる概念
世界モデル(world model): 「自分の行動の結果、環境がどう変化するか」を内部でシミュレーションできるモデルです。式で書けば「次の状態 ≒ 関数(今の状態, 取る行動)」を学習したものです(基礎は [潜在世界モデルによる未来予測 (FutureState)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0020_world-model-future-prediction))。

行動 / action: エージェント(ここでは自車)が環境に働きかける操作です。自動運転では「これから数秒間にたどる軌道」(各時刻の x, y 座標や速度・舵角の系列)を行動とみなすことが多いです。

反実仮想(counterfactual): 「実際にはやらなかったが、もしやっていたらどうなっていたか」という仮定の問いです。因果推論(原因と結果の関係を推定する分野)の中心概念で、ここでは「実際には選ばなかった軌道を取った場合の未来」を指します。

ロールアウト(rollout): 行動の系列を順に適用して未来を展開していくことです。1ステップ先だけでなく数ステップ先まで予測を伸ばすと、誤差が積み重なって現実とずれていく複合誤差(compounding error)が問題になります。

潜在表現(latent representation): 観測を低次元の意味あるベクトルに圧縮したもの。ピクセルではなくこの潜在表現で予測すると、運転に無関係な細部に容量を浪費せず軽量になります。

ブロードキャスト加算: 小さなテンソル(例 `[B,C,1,1]`)を大きなテンソル(例 `[B,C,H,W]`)の全空間位置へ同じ値で足す操作です。「ある情報を特徴マップ全体に一様に注入する」のに使います。

## 仕組みを詳しく
行動条件付き未来予測モジュールの典型的な処理を、テンソルの形を追って見ます。入力は次の3つです。

1. 現在のシーン特徴: 今の世界をカメラなどから理解した特徴マップ。形状 `[B, C, H, W]`。B はバッチサイズ、C はチャンネル数(例 256)、H と W は BEV(鳥瞰図)の縦横グリッド。例 `[4, 256, 450, 300]`。
2. 行動意図ベクトル: プランナーの再帰部品(GRU/LSTM)が出す「自車の意図を要約した数百次元ベクトル」。形状 `[B, C]`。「これからどう走るつもりか」の内部状態です。
3. 計画した軌道(オプション): 計画した軌道を平らに並べたベクトル。形状 `[B, action_dim]`。例えば未来4ステップ × 各ステップの信号(x座標, y座標, 速度など)を1列に連結したものです。これが「行動(action)」に当たります。

処理ステップ:

ステップ1(意図の注入): 行動意図ベクトルを全結合層(Linear)で C 次元に変換し、`[B,C,1,1]` に直して特徴マップへブロードキャスト加算します。
```
bias        = Linear(intent).view(B, C, 1, 1)   # [B,C,1,1]
conditioned = scene_features + bias             # [B,C,H,W]
```
これで「自車の意図」が世界の特徴全体ににじみ込みます。

ステップ2(行動条件付けの肝): 計画軌道が渡されたら、それも別の全結合層で C 次元に変換し、もう一段バイアスとして足します。
```
action_bias = Linear(trajectory).view(B, C, 1, 1)
conditioned = conditioned + action_bias
```
行動次元が設定されていない(`action_dim=None`)構成では、この投影層自体を作らず、行動条件付けを行わない設計が一般的です。つまりオプションで有効化する後方互換な作りにできます。設計上重要なのは、「行動を渡したのに行動次元が未設定」という矛盾した呼び方をされたとき、黙って無視せず早期にエラーを出して設定ミスを知らせる防御的な実装にしておくことです。沈黙して行動を捨てると「配管は通っているのに行動条件付けが学習に効いていない」という気づきにくいバグになります。

ステップ3(未来生成): 条件付け済み特徴を畳み込み数段(例 C→2C→4C チャンネル、間に GELU)に通し、チャンネル方向に分割して未来の複数時刻ぶんの特徴マップ `[B, C, H, W]` × 4(例 +1.6/3.2/4.8/6.4s)を出力します。

ここで反実仮想ロールアウトが実現します。同じ現在のシーン特徴でも、軌道を変えれば `action_bias` が変わり、出力(予測される未来)が変わるからです。

before / after で対比すると、
- before(行動入力なし): 未来予測 = 関数(今の世界, 自車意図)。1通りの未来。
- after(行動入力あり): 未来予測 = 関数(今の世界, 自車意図, 取る行動)。行動ごとに違う未来。

メモリの注意点として、450×300 の BEV 解像度で 4×256 チャンネルを fp32 で出すと1サンプルあたり約550MB に達します(`1024×450×300×4 byte`)。混合精度(bf16/fp16)や空間ダウンサンプリングが事実上必須です。

評価・推論時の使い方も重要です。候補軌道を複数用意し、それぞれを行動として入れて未来特徴を予測し、各未来の「安全コスト」(衝突しそうか、車線を逸脱しそうか)を測って最も安全な軌道を選ぶ、という model predictive control(MPC、予測モデルで先読みして最良の制御を選ぶ枠組み)的な利用ができます。これが行動条件付き予測の実戦的価値です。

## 手法の系譜と主要論文
行動条件付き予測は世界モデル研究の中核で、複数の系譜があります。

Action-Conditional Video Prediction(Oh et al., NeurIPS 2015, "Action-Conditional Video Prediction using Deep Networks in Atari Games")。Atari ゲームで「ボタン入力(行動)を条件に次フレームを予測する」初期の代表研究です。CNN + 行動の乗算的相互作用(特徴と行動を掛け合わせる結合)+ 再帰構造で、行動に応じた映像予測を学びました。動機は「行動の効果をモデル化できればエージェントの計画に使える」。長期予測ほど誤差が積もる課題も同時に示しました。

World Models(Ha & Schmidhuber, NeurIPS 2018, "Recurrent World Models Facilitate Policy Evolution", arXiv:1809.01999)。RNN(MDN-RNN)で「行動を入れると次の潜在状態を予測する」遷移モデルを学び、その内部シミュレーション(「夢」)の中で方策を進化させました。動機は実環境での試行を減らすこと。効果は内部で安全に大量試行できること。弱点はロールアウトが長いほど予測誤差が積み重なり現実とずれること。

Dreamer 系(Hafner et al., DreamerV1 2020 / V2 2021 / V3 2023, arXiv:2301.04104)。潜在空間で行動条件付きの未来を予測し、その想像上のロールアウトで強化学習を行う代表系列です。観測を直接予測せず潜在状態で予測することで効率化し、V3 は同一設定で 150 以上の多様なタスクを解きました。教訓は「潜在で予測するとサンプル効率が大きく上がる」。注意点は潜在表現の崩壊と長期予測の不正確さです。

MILE(Hu et al., NeurIPS 2022, Wayve, "Model-Based Imitation Learning for Urban Driving", arXiv:2210.07729)。運転で行動条件付きの潜在世界モデルと模倣学習を組み合わせ、BEV 潜在空間で未来を予測します。「BEV 特徴 + 行動条件 + 未来特徴予測」という構図に最も近い系譜です。CARLA のオフラインデータのみで強い閉ループ性能を達成しました。

GAIA-1(Hu, Russell, Yeo et al., Wayve, 2023, arXiv:2309.17080, "GAIA-1: A Generative World Model for Autonomous Driving")。過去映像・テキスト・行動を条件に未来の運転映像を生成する大規模(約90億パラメータ)生成型世界モデルです。動機は「実車で全パターンを試すのは危険・高コストなので、内部で多様な未来を生成して学習・評価に使いたい」。行動を変えると生成される未来が変わるリアルな映像生成を示しました。代償は巨大計算コストとピクセル生成の重さで、特徴マップ予測型はこの軽量化版にあたります。

これらに共通する考え方は「未来 = 関数(現在, 行動)」であり、特徴マップ予測としての行動条件付けはその最小構成と位置づけられます。

## 論文の実験結果(定量データ)
Oh et al.(2015): Atari の複数ゲームで、行動条件付き予測モデルが数十〜100フレーム先まで妥当な映像を予測できることを示し、予測フレームを使った探索(計画)が実際にゲームスコアを改善することを報告しました。一方、長期予測ではゲーム内オブジェクト(敵や弾)が消えるなどの劣化が観察され、複合誤差の課題を明確化しました。

DreamerV3(2023): 同一ハイパーパラメータで 150 以上のタスクを横断し、Atari・物理制御・Minecraft などで当時の専用手法に匹敵または上回りました。Minecraft のダイヤモンド採取を人間デモなしで達成。重要なアブレーションとして、潜在状態での予測をピクセル予測に置き換えるとサンプル効率が落ちること、対称対数変換などの正規化が安定化に寄与することが示され、「行動条件付きの潜在予測」が効率の源であることを裏付けました。

MILE(2022): CARLA の未知の町・未知の天候という分布外条件での閉ループ評価で、駆動スコア(route completion とインフラクション = 違反のペナルティを組み合わせた総合指標)で先行のモデルフリー模倣手法を上回りました。アブレーションでは、未来予測(世界モデル)成分を外すと汎化が顕著に悪化し、行動条件付きの未来予測が表現と方策の両方を良くすることが示されました。指標の意味は、route completion がルートをどれだけ走破できたか(0〜100%、高いほど良い)、infraction が信号無視・衝突などの違反頻度(少ないほど良い)です。

GAIA-1(2023): 主に生成品質の定性評価ですが、約90億パラメータへのスケーリングでサンプルの一貫性・多様性・行動への忠実度(指示した行動どおりの未来が生成されるか)が向上することを報告しました。同じ過去文脈から行動だけを変えると分岐した未来映像が生成される様子(同じ初期状態から「左に寄る」「直進する」で別々の映像になる)が示され、行動条件付け生成が実際に機能していることを定性的に実証しています。

## メリット・トレードオフ・限界
メリット
- 「行動を変えれば未来が変わる」を陽に表現でき、軌道Aと軌道Bの安全性比較(反実仮想)が可能になる。
- ピクセルではなく特徴マップを予測するので、映像生成型の世界モデルより軽量でリアルタイムに乗せやすい。
- 行動条件付けをオプション化すれば、無効時に従来挙動を完全に保てる安全な設計にできる。

トレードオフ・限界
- 高解像度 BEV ではメモリ消費が大きく、混合精度や空間ダウンサンプリングがほぼ必須。
- 自己回帰ロールアウトは複合誤差で長期予測が現実とずれる。同時生成にすると誤差は積もらないがステップ間因果を明示できない。
- 行動の表現(軌道の平坦化・正規化・座標系の取り方)が予測品質を大きく左右する。設計が雑だと、モデルが行動入力を無視して「平均的な未来」だけを出す行動無視(action-agnostic)に陥り、行動条件付けが実質効かなくなる。
- 予測タスクを学習信号として使うには、未来特徴を正解と突き合わせる損失(特徴再構成損失)や、予測側と目標側の次元整合が必要で、ここを通さないと「配管は通っているが学習に効いていない」状態になりがち。
- 未解決課題: 複数の妥当な未来を確率分布として扱う多峰性、稀少イベント(急な飛び出し)の予測、長期一貫性、そして行動条件付き予測が本当に下流の運転安全性を上げるかの因果的検証。とくに「観測された相関(この行動の後にこの未来が来た)」と「真の因果(この行動がこの未来を引き起こす)」を取り違えると、危険な反実仮想を出しかねない点が研究上の難所です。

## 発展トピック・研究の最前線
拡散モデル(diffusion)で多峰的な未来を生成し行動で条件付ける手法、Transformer による長文脈ロールアウト、言語で行動・状況を指示する言語条件付き世界モデルが活発です。運転では、予測した複数未来から最も安全な軌道を選ぶ MPC 的な推論、生成型と予測型のハイブリッド、シミュレータ生成データと実データの併用が焦点です。因果推論の観点から「真に反実仮想として妥当な予測」を保証する研究(交絡変数を分離する設計、介入の明示的モデル化)も進んでいます。運転特化の大規模生成世界モデル(DriveDreamer、Vista など)も、行動・舵角・速度を条件にした未来映像生成を売りにしており、行動条件付けは生成型・予測型の両系統で共通の中心技術になっています。
