# frozen backboneによるJEPAターゲット

## ひとことで言うと
未来や欠損部分を予測する自己教師あり学習(JEPA)では、「予測の答え合わせに使う正解(ターゲット)」を作る画像エンコーダを、学習中に固定(凍結)する設計がよく採られます。同じ凍結エンコーダで過去(予測の手がかり)も未来(予測の答え)も符号化することで、モデルがズルをして「どんな入力でも同じ無意味な答え」に潰れてしまう表現崩壊(representation collapse、表現崩壊)を構造的に防ぎます。本ノートでは、なぜ凍結が崩壊を防ぐのか、その代償は何か、そして崩壊回避をめぐる自己教師あり学習研究の系譜と定量的証拠を、前提知識ゼロの読者から研究者レベルまで段階的に解説します。

## 直感的な理解
2人で「未来予測ゲーム」をすると想像してください。プレイヤーAは過去の写真を見て「未来の写真はこうなる」と予測カードを出します。プレイヤーBは実際の未来の写真を見て「正解カード」を出します。AとBのカードが近いほど高得点。AはBに近づけようと頑張ります。

ここに落とし穴があります。もしAとBが共謀できたら、ゲームは破綻します。2人で示し合わせて「どんな写真が来ても、必ず真っ白なカードを出そう」と決めれば、AもBも常に真っ白を出すので、2枚は完全に一致して満点。でもこのカードには何の情報もありません。写真の中身を一切見ずに満点を取れてしまうのです。これが表現崩壊です。学習上は満点(損失ゼロ)なのに、得られた特徴は完全に役立たず、という、たちの悪い失敗です。

防ぐ方法は単純で、Bを「共謀できない審判」にすればいい。Bが過去に決めた固定ルールで正解カードを出し、そのルールをゲーム中に変えられないようにすれば、Aは真っ白を出してもBの本物の正解と食い違って減点されます。Aは写真を本当に見て予測するしかなくなる。この「Bを固定する」役割を、JEPA では画像 backbone を凍結することで担わせます。固定された的(まと)に向けて撃つから、的のほうから矢に寄ってくることがなく、本物の照準合わせが学習されるわけです。

## 基礎: 前提となる概念
- 自己教師あり学習(self-supervised learning): 人間のラベルなしで、データ自身から予測ターゲットを作って学ぶ方式。「画像を2通りに加工して近づける」「過去から未来を当てる」などが代表例です。ラベル付けが不要なので大量のデータをそのまま使えます。
- 埋め込み(embedding)/特徴ベクトル: 画像などの入力をニューラルネットが変換した、意味を圧縮した数値ベクトル。「同じ犬の画像どうしは近いベクトルになる」ような空間を学習で作ります。
- JEPA(Joint Embedding Predictive Architecture): ピクセルそのものではなく、この埋め込み空間で未来や欠損部を予測する枠組み。学習の流れは、(1) 過去/コンテキストを符号化して予測器で「未来の特徴はこうなるはず」と予測、(2) 実際の未来をターゲットエンコーダで符号化して正解の特徴を作る、(3) 予測と正解の差(MSE など)を小さくする、の3段階です。
- 表現崩壊(representation collapse): 上で説明した「全入力を同じ定数ベクトルに写す退化解(degenerate solution)」。損失は0になるが特徴は無意味。自己教師あり学習で最も警戒される失敗モードです。
- stop-gradient と detach: 勾配(パラメータ更新の方向を示す微分)を、ある地点から先に流さない操作。`tensor.detach()` や `torch.no_grad()` でターゲット側への勾配を切ると、最適化器(オプティマイザ)がターゲットを動かせなくなります。これが崩壊回避の要です。
- EMA(指数移動平均、exponential moving average): あるネットの重みを、別のネットの重みでゆっくり追従させる更新法。`θ_t ← τ·θ_t + (1−τ)·θ_o`(τ は0.99〜0.999 程度の係数)。BYOL のターゲット更新に使われます。τ が1に近いほどターゲットはゆっくり動きます。

## 仕組みを詳しく
やっていることは驚くほど単純です。まず backbone を作り、勾配を流さないよう凍結します。

```python
backbone = Backbone(arch="swin_v2_tiny", is_pretrained=True)
for p in backbone.parameters():
    p.requires_grad = False
backbone.eval()
```

`is_pretrained=True` は、ImageNet などで先に学習済みの重みを読み込むこと。`eval()` は BatchNorm や Dropout などの「学習時と推論時で挙動が変わる層」を推論モードに固定し、ターゲットの出力を毎回同じにします。そして過去フレームも未来フレームも、同じこの凍結 backbone で符号化します。

```python
# 過去(コンテキスト): 予測器に渡す手がかり
ctx_feats = backbone(past_frames)            # 凍結

# 未来(ターゲット): 答え合わせ用。勾配は切る
with torch.no_grad():
    target_feats = backbone(future_frames)   # 同じ凍結backbone
target_feats = target_feats.detach()

# 予測
pred_feats = predictor(ctx_feats)

# 損失: 予測 vs 凍結backboneが出した未来の正解
loss = mse(pred_feats, target_feats)         # target は固定空間
```

ポイントは3つです。

1. ターゲット空間が時間的に動かない(stationary、定常)。backbone を更新しないので、`target_feats` の「意味」は学習の最初から最後まで変わりません。予測器は「動かない的」に向けて撃てばよく、崩壊しようがありません。動く的(学習で動くターゲット)だと、予測器とターゲットが一緒に楽な点(たとえば原点)へ逃げて崩壊できますが、的が固定なら逃げ場がありません。
2. 過去と未来で同じエンコーダを使う。これにより「過去をこう符号化したら、未来はこう符号化される」という対応が一貫します。別の凍結コピーや EMA コピーを別途用意する必要がなく、実装が単純になります。
3. detach で勾配を切る。`target_feats.detach()` により、損失を小さくする圧力が backbone へ逆流しません。`requires_grad=False` で凍結していても、計算グラフ経由で予期せぬ経路の勾配が混ざる事故を防ぐ二重の安全策になります。

数値例で崩壊を考えます。仮に backbone も学習可能だとして、予測器が「常に零ベクトル `[0,0,...,0]` を出す」、backbone も「どんな画像も零ベクトルにする」と学習したとします。すると loss = MSE(0, 0) = 0。損失は完璧ですが、要約ベクトルも零になり、運転計画には何の役にも立ちません。backbone を凍結しておけば、未来の正解 `target_feats` は画像ごとに違う非自明な値(たとえば歩行者がいるフレームと空いた道路では別のベクトル)になるので、予測器が零を出せば loss は大きいまま残ります。崩壊への道が構造的に塞がれるわけです。

ターゲット凍結 backbone の「凍結の度合い」には設計上の選択肢があります。代表的には2つです。

- 完全凍結(fully frozen): 事前学習の重みのまま一切動かさない。最も単純で、ターゲット空間が完全に固定。
- EMA 更新: 学習可能な online backbone からゆっくり追従(τ≈0.996 程度)。ターゲットが「遅延コピー」として少しずつ良くなる。BYOL / I-JEPA の流派。

「同じ backbone を運転本体でも使い、補助的に未来予測も載せる」という自動運転の文脈では、運転本体との干渉を避けたい事情から完全凍結が選ばれやすい傾向があります(関連: [online encoderとfrozenターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0018_online-vs-frozen-target-encoder))。

自動運転の文脈では、ターゲット特徴を「+1.6s, +3.2s, +4.8s, +6.4s」のように複数の未来時刻(マルチホライズン、multi-horizon)で抽出し、それぞれを別々に予測させる構成がよく取られます。1.6秒は10Hz カメラを1Hz に間引いた周期に対応し、複数ホライズンを持たせることで、短期(すぐ先)と長期(数秒先)の両方のダイナミクスを学ばせます。短期は当てやすく長期は難しいため、ホライズンごとに損失の重みを変える工夫もあります。

## 手法の系譜と主要論文
崩壊回避の系譜をたどると、凍結ターゲットの位置づけがよく分かります。

- 対比学習(時代背景): SimCLR(Chen et al., ICML 2020, arXiv:2002.05709)などは「同じ画像の2つの拡張を近づけ、別画像(ネガティブ)を遠ざける」ことで崩壊を防ぎました。引き離す相手がいるので潰れません。ただし大量のネガティブと巨大バッチ(数千枚規模)が必要で扱いが面倒、という課題がありました。
- BYOL(Grill et al., NeurIPS 2020, "Bootstrap Your Own Latent", arXiv:2006.07733)。ネガティブなしで崩壊を防いだ画期的研究。鍵はターゲットネットワークを online ネットワークの EMA にして、ターゲット側に直接勾配を流さないこと。提案理由は「ネガティブ依存とバッチサイズ敏感性をなくしたい」。効果は「ネガティブなしで高精度」。トレードオフは「EMA 係数 τ の調整が必要で設計がやや複雑」。
- SimSiam(Chen & He, CVPR 2021, "Exploring Simple Siamese Representation Learning", arXiv:2011.10566)。EMA すら使わず、重み共有 + stop-gradient + predictor(予測器)だけで崩壊を防げると実証。崩壊回避の核心が「ターゲットへの stop-gradient」であることを浮き彫りにしました。
- I-JEPA(Assran et al., CVPR 2023, arXiv:2301.08243)。画像の一部から別の部分の特徴を予測。ターゲットエンコーダは EMA で更新する凍結気味のネット。特徴空間での予測がピクセル再構成より意味的に効率的、という JEPA 思想を画像で実装。
- V-JEPA(Bardes et al., 2024)。動画でマスクされた時空間領域の特徴を予測。時間方向の JEPA を本格的に扱い、自動運転の未来予測と発想が直結します。凍結気味のターゲットで時系列を扱う設計の最前線です。
- JEPA 枠組み(LeCun, 2022, "A Path Towards Autonomous Machine Intelligence")。「ターゲットエンコーダは非学習(non-trainable)であるべき」という崩壊回避の要件を一般原理として明文化しました。凍結 backbone はこの要件を最も素直に満たす実装です。
- 自動運転への橋渡し: MILE(Hu et al., NeurIPS 2022, arXiv:2210.07729)。潜在空間での未来予測が都市運転で有効と示し、特徴空間ターゲットの実用性を裏付けました。

## 論文の実験結果(定量データ)
凍結 / stop-gradient ターゲットの効果は、アブレーション(ある要素を抜くと性能がどう変わるかの実験)で最もはっきり示されています。指標の意味とともに見ます。

- BYOL は ImageNet の線形評価(凍結した特徴の上に線形分類器だけを学習し、その分類精度で表現の良さを測る標準手順)で約74.3%のトップ1精度を達成。ここでの最重要アブレーションは「ターゲットを EMA ではなく online と同一(stop-gradient なし)にすると、表現が崩壊して精度がほぼ無意味な水準(数%以下)に落ちる」というものです。つまりターゲット側を更新から切り離す非対称性が、「性能をちょっと上げる小技」ではなく「学習が成立するための条件そのもの」であることが示されました。
- SimSiam は、stop-gradient を外した瞬間に損失が一気にゼロ付近へ落ち、同時に特徴の出力の標準偏差が崩壊する(全サンプルが同じベクトルに集まり、ばらつきが消える)様子を学習曲線で示しました。stop-gradient ありでは ImageNet 線形評価で約68〜71%(設定による)を達成。これは「予測ターゲットへの勾配遮断が崩壊回避の核心」という主張の、最も直接的な定量証拠です。出力の標準偏差を監視すれば崩壊を早期検知できる、という実用的な教訓も得られます。
- I-JEPA は ImageNet で、ピクセル再構成型の MAE と比べて、同等以上の線形評価精度をより少ない事前学習計算量(GPU 時間)で達成したと報告。特徴空間予測の計算効率上の優位を、計算量対精度のトレードオフ曲線で示しました。
- MILE は CARLA の新規タウン・天候設定で、当時最先端の driving score(運転スコア、完走率と違反減点の総合。高いほど良い)を更新。潜在ワールドモデルでの未来予測が、未知環境への汎化を助けることを定量的に示しました。

これらに共通する読み解きは「凍結(または stop-gradient / EMA)は精度を少し上げる小技ではなく、自己教師あり学習が崩壊せずに成立するための必須条件」という点です。完全凍結は EMA に比べてターゲットの更新余地がない分、表現を改善し続けることはできませんが、最も単純で確実に崩壊を防げる選択肢として実務でよく使われます。

## メリット・トレードオフ・限界
メリット

- 表現崩壊を構造的に防げる。ターゲットが固定空間なので予測器が潰れられない
- 過去・未来を同じエンコーダで符号化でき、別の凍結コピーや EMA コピーが不要で実装が単純
- backbone を更新しないので学習が安く、他のタスク(運転本体など)の学習と干渉しない

トレードオフ・限界

- ターゲット空間が「事前学習時点」に固定される。下流ドメイン(運転映像)に合わせて改善されないので、ターゲットの質は事前学習の質に依存する
- 共有 + stop-gradient 方式(非定常だがドメインに追従する)に比べ、ターゲットが新鮮でない可能性がある
- detach / stop-gradient を忘れると勾配が漏れ、崩壊回避が機能しなくなる(静かに崩壊する実装上の落とし穴)
- EMA を使う変種に比べ、ターゲットが一切動かないぶん表現の更新余地がない
- 研究上の未解決課題として、「完全凍結 vs EMA vs 共有+stop-grad」のどれが最良かはタスク・データ規模に依存し、理論的な決着はついていない

## 発展トピック・研究の最前線
- 崩壊回避の別系統: VICReg(Bardes et al., 2022)や Barlow Twins(Zbontar et al., 2021)は、stop-gradient ではなく「特徴の分散を一定以上に保つ」「特徴次元間の相関を消す」という正則化項で崩壊を防ぎます。凍結ターゲットとは異なる崩壊回避の哲学であり、非対称性に頼らずに済む点が対照的です。
- ターゲットの新鮮さ問題: 完全凍結のドメイン非追従を補うため、事前学習後に下流データで backbone を一度だけ追加学習(fine-tune)してから凍結する、あるいは緩やかな EMA を併用するなどのハイブリッドが研究されています。
- 動画・時系列 JEPA の最前線: V-JEPA 系の動画への展開は、自動運転の未来予測とほぼ同じ問題設定であり、時間方向のマスク戦略(どの時刻を隠して予測させるか)が活発な研究対象です。
- ターゲット可視化: 特徴空間予測は人間が当否を直接確認しにくいため、ターゲット/予測特徴をデコーダで画像に戻して検証する研究も進んでいます。「予測が当たっている」を人間が目で確かめられるようにする方向です。
