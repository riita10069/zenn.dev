# マップBEV残差フュージョン（per-channel alpha）

## ひとことで言うと

カメラから作った BEV(俯瞰)特徴に、地図から作った BEV 特徴を `image_bev + alpha * map_bev` という形で足し込む融合方式です。ここで `alpha`(アルファ)はチャンネルごとに別々の学習可能な係数で、最初はすべてゼロから始めます。ゼロ始まりなので学習開始時は地図の枝(ブランチ)が何も影響を与えず、地図なしで学習するのとまったく同じ挙動になります。学習が進むにつれ alpha が育ち、役に立つ地図チャンネルだけが効いてきます。「新しい情報源を、本流を壊さずに後から安全に合流させる」ための、軽くて確実なゲート(門)です。

## 直感的な理解

2つの特徴マップを混ぜる一番素朴な方法は、単純に足すこと(`image_bev + map_bev`)です。ところがこれには学習が不安定になる落とし穴があります。

地図エンコーダは多くの場合ランダム初期化されている(自然写真で事前学習していない)ので、学習の最初は地図特徴がほぼノイズです。これをそのままカメラ特徴に足すと、せっかくカメラ単独でうまく学べていた信号にノイズが上乗せされ、序盤の学習が乱れます。数値で考えると、カメラ特徴の有効な値が ±0.5 程度のスケールのとき、ランダムな地図特徴が ±2 のノイズを足してくれば、信号がノイズに埋もれ、勾配(学習の方向を示す微分)が荒れます。学習が進めばどのみち地図は役に立つのですが、序盤に本流を壊してしまうのが問題なのです。

そこで「地図の枝を、最初は完全にオフにし、有用だと分かってから徐々にオンにする」仕組みが欲しくなります。これを実現するのが、ゼロ初期化したゲート alpha です。学習開始時 alpha=0 なので地図は素通りせず、モデルはまずカメラ単独で安定して学べます。学習が進み「この地図チャンネルは軌跡予測に効く」と分かると、逆伝播(誤差を出力から入力へ逆向きに伝える学習過程)がそのチャンネルの alpha を少しずつ大きくし、地図情報が流れ込みます。これにより「地図を入れたら序盤に壊れる」問題を避けつつ、最終的に地図の恩恵を受けられます。比喩を使わずに言えば、これは「初期値を恒等写像(入力をそのまま出す写像)にしておく」という、深層学習で何度も再発見されてきた安定化の原理です。

## 基礎: 前提となる概念

- 残差接続(residual connection): `出力 = 入力 + 何らかの変換(入力)` の形。He et al. の ResNet(CVPR 2016, arXiv:1512.03385)で広まり、深いネットでも勾配が消えずに学習できるようにする土台技術です。本トピックの融合もこの形の一種です。
- 恒等写像(identity mapping): 入力をそのまま出力に通す写像。学習開始時にネットワークを恒等写像にしておくと「何もしない状態」から始まり、信号を壊さずに少しずつ変換を学べます。これが本トピックのゼロ初期化の狙いです。
- 勾配(gradient): 損失(誤差)を各パラメータで微分した値。学習はこの勾配の向きにパラメータを少し動かす操作の繰り返しです。勾配がゼロのパラメータは更新されません。
- ゲート(gate): 信号を「どれだけ通すか」を 0〜1 などの係数で調整する仕組み。LSTM の入力ゲート/忘却ゲートが古典例です。ここでは alpha がゲートの役割を果たします。
- per-channel(チャンネルごと): 係数を全体共通の1つの数(スカラー)にするのではなく、チャンネルの数だけ別々に用意すること。チャンネルとは特徴マップの「種類の軸」で、各チャンネルが別のパターンを表します。
- ブロードキャスト(broadcasting): 形の違うテンソル同士を、足りない軸を自動で引き伸ばして要素ごとに演算する仕組み。`(1, C, 1, 1)` の係数を `(B, C, H, W)` の特徴に掛けると、全バッチ・全空間位置に同じチャンネル係数が掛かります。
- `nn.Parameter`: 学習で値が更新されるテンソル。これに指定したテンソルは勾配を受け取り、最適化器で更新されます。

## 仕組みを詳しく

典型的な実装は、`embed_dim` 次元のチャンネルごとに alpha を持つ小さなモジュールです。

### alpha は「チャンネルごと(per-channel)」のパラメータ

```python
self.alpha = nn.Parameter(torch.zeros(embed_dim))   # 形 (C,) 例: (256,)
```

重要なのは、alpha が1つのスカラー(全体共通の数)ではなく、チャンネルごとに別々の C 個の値だということです。これが per-channel の意味です。

なぜチャンネルごとか。地図特徴の各チャンネルには、軌跡予測に効くもの(進路の方向を表すチャンネルなど)と、ほとんど無関係なものが混在します。alpha が1つの共通値だと「地図全体を一律に強める/弱める」しかできませんが、チャンネルごとなら「効くチャンネルだけ alpha を育て、無関係なチャンネルは alpha≈0 のまま」という選別ができます。学習が地図の中身を取捨選択できるわけです。これは Squeeze-and-Excitation(後述)の「チャンネル重み付け」と同じ発想を、ゼロ初期化と組み合わせた形と言えます。

### forward: ブロードキャストで足す

```python
def forward(self, image_bev, map_bev):
    gate = self.alpha.view(1, -1, 1, 1)   # (1, C, 1, 1)
    return image_bev + gate * map_bev
```

入力は両方 `(B, C, H, W)`(同形状が前提)。alpha を `(1, C, 1, 1)` に reshape し、ブロードキャストで全バッチ・全空間位置に同じチャンネル係数を掛けます。計算は実質「チャンネルごとのスカラー倍と足し算」だけで、極めて軽量です。

### 数値で挙動を確認(before / after)

学習開始時(alpha = [0, 0, ..., 0]):
```
出力 = image_bev + 0 * map_bev = image_bev   ← 地図は完全に無寄与
```
このとき出力は地図なしの場合と1ビットも違いません。実装では「zero-alpha 保証」として単体テストで検証されることが多いです(地図ありとなしで初期出力が一致するか)。

学習が進んだ後(例: チャンネル5の alpha=0.8、他は≈0):
```
出力[:, 5] = image_bev[:, 5] + 0.8 * map_bev[:, 5]   ← この地図チャンネルだけ効く
出力[:, その他] ≈ image_bev[:, その他]               ← 残りは地図ほぼ無視
```

### 勾配の流れ方

alpha=0 でも、地図エンコーダへの勾配経路 `∂出力/∂map_bev = gate = 0` なので地図ネット本体へはほぼ流れません。しかし alpha 自身への勾配 `∂出力/∂alpha = map_bev` は0ではないので、alpha は更新され得ます。alpha が少しでも非ゼロになると、そこから地図エンコーダへも勾配が流れ始め、地図ブランチが本格的に学習に参加します。この「alpha がまず動き、それを起点に枝全体が起き上がる」連鎖が、ゼロ初期化ゲートの肝です。

## 手法の系譜と主要論文

ゼロ初期化の残差ゲートという考え方は、近年の深層学習で繰り返し再発見されてきました。系譜を追います。

系譜の起点として、ゼロ初期化はもともと「最適化を安定させる初期化の工夫」として現れました。Goyal et al.(2017, arXiv:1706.02677)は、ResNet の各残差ブロック末尾にある BatchNorm の倍率 γ(ガンマ)をゼロ初期化すると、学習開始時に各ブロックが恒等写像になり、大規模ミニバッチ学習でも安定すると報告しました(ImageNet を1時間で学習する有名な研究の一部)。これが「残差枝をゼロから立ち上げる」という発想の早い実用例で、以後さまざまな文脈で再発見されます。

- Hu et al., "Squeeze-and-Excitation Networks"(CVPR 2018, arXiv:1709.01507): チャンネルごとに重みを学習して特徴を再較正する SE ブロックを提案。ImageNet 分類で当時の最先端を更新し、ILSVRC 2017 優勝の核技術になりました。チャンネルごとに「効く/効かない」を学習で調整する発想は、本手法の per-channel alpha と同じ精神です(SE は 0〜1 のゲート、本手法はゼロ初期化の残差係数という違い)。

- Bachlechner et al., "ReZero is All You Need: Fast Convergence at Large Depth"(2020, arXiv:2003.04887): 残差接続を `x + alpha * F(x)` の形にし、alpha をゼロ初期化すると、非常に深いネット(数百〜1万層超)でも学習が安定し収束が速いことを示しました。動機は「初期に恒等写像から始めれば信号が壊れない」。Transformer・ResNet・全結合の各種で収束加速を報告しています。本残差フュージョンは、これを「2つのモダリティ(カメラと地図)の融合ゲート」に応用した形と理解できます。

- Touvron et al., "Going deeper with Image Transformers (LayerScale)"(ICCV 2021, arXiv:2103.17239): 深い Vision Transformer の各残差枝に、チャンネルごとの学習可能なスケール係数(小さな値、例 1e-4 で初期化)を掛けると深層化が安定すると示しました。per-channel のスケールという点で本手法に最も近い先行例です。

- Zhang et al., "Adding Conditional Control to Text-to-Image Diffusion Models (ControlNet)"(ICCV 2023, arXiv:2302.05543): 追加の条件入力(エッジ・姿勢・深度など)を「zero convolution」(重みをゼロ初期化した畳み込み)で事前学習済み本体に足し込み、学習開始時は本体を一切壊さない設計を提案しました。これは本手法とまったく同じ「事前学習済みの本流に、新しい枝を安全に後付けする」パターンです。

これらに共通する原理は「初期値を恒等写像にし、必要になってから枝を立ち上げる」ことで、モダリティ融合・深層化・条件付き生成という異なる文脈で独立に有効性が確認されてきました。

## 論文の実験結果(定量データ)

ReZero(arXiv:2003.04887)は、12層 Transformer の言語モデルで、標準的な post-norm + warmup の設定に比べて収束に要するイテレーションを大きく削減し(報告では収束が約56%速い水準)、warmup なしでも学習が立ち上がることを示しました。さらに、通常は学習困難な極端な深さ(数千層規模)でも発散せずに学習が進むことを実証しています。これは「ゼロ初期化ゲートが序盤の信号崩壊を防ぐ」直接的な証拠です。

ControlNet(ICCV 2023)では、zero convolution を「通常のランダム初期化畳み込み」に置き換えるアブレーションを行うと、学習初期に事前学習済みの生成本体が劣化し、出力品質が一時的に大きく崩れることが報告されています。ゼロ初期化により、学習開始直後の出力は条件なしの本体と完全に一致し、そこから条件制御が徐々に効いてくる挙動が確認されています。これは本手法の「zero-alpha 保証」とまったく同じ性質です。

SE-Net(CVPR 2018)は、ResNet-50 に SE ブロックを足すだけで ImageNet の top-1 誤差を約 1%(報告では 23.30% → 22.28% 前後)改善し、追加計算は約 0.26 GFLOPs 程度とわずかでした。チャンネルごとの重み付けが、わずかなコストで有意な精度改善をもたらすことを示しています。per-channel で地図チャンネルを選別する本手法の設計根拠を支えるデータです。

なお本融合方式そのものの一般的なアブレーションとしては、(a) alpha を per-channel ではなく単一スカラーにすると選別能力が落ちて最終精度が伸び悩む、(b) alpha をゼロでなくランダム初期化すると序盤の学習が不安定になり収束が遅れる、(c) ゲートを外して単純加算すると地図導入時に序盤の損失が跳ねる、という傾向が予想されます(個別の数値は採用するデータセット・タスクに依存し、報告により異なります)。

## メリット・トレードオフ・限界

メリット
- ゼロ初期化で学習序盤が安定。地図を入れても序盤はカメラ単独と同一挙動になり、地図導入のリスクが小さい。
- チャンネルごとの alpha で、効く地図チャンネルだけを選別できる。
- 実装が非常に軽い(パラメータは C 個の alpha だけ、計算はほぼスカラー倍と足し算)。
- いつでも「地図なし」へ連続的に戻せる(alpha→0)。アブレーションや故障時のフォールバックが容易。

トレードオフ・限界
- alpha が育つまで地図の恩恵が遅れる(序盤は実質カメラのみ)。短い学習スケジュールでは地図枝が十分立ち上がらない恐れがある。
- セルごとに足すため、`image_bev[i,j]` と `map_bev[i,j]` が同じ実空間地点を指す前提が要る。この前提は、明示的なカメラ射影と距離グリッドから計量的な BEV を作るモードでのみきれいに成り立ち、抽象的な特徴プールしか行わないモードでは位置対応が崩れる。崩れても「学習で混ぜる仕組み」としては動くが、セル単位の空間整合は失われる。
- 単純な加算なので、位置のズレを補正するような賢い対応付けはできない(それが必要なら [マップクロスアテンションフュージョン](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0310_map-cross-attention-fusion))。
- チャンネルごとの単純ゲートなので、空間的に「画面のここだけ地図を強める」という位置依存の制御はできない。

研究上の未解決課題として、空間依存ゲート(位置ごとに alpha を変える)や、不確実性に応じてゲートを動的に開閉する適応的融合があります。固定の per-channel ゲートは軽さと引き換えに、空間・時間的な文脈に応じた可変の混合ができません。

## 発展トピック・研究の最前線

ゼロ初期化ゲートはマルチモーダル融合一般の安全弁として広く使われています。LLaVA など vision-language モデルでも、視覚特徴を言語モデルへ注入する際にゼロ初期化的なアダプタや射影層が用いられ、事前学習済み言語モデルを序盤で壊さない工夫が定番化しています。Flamingo(Alayrac et al., NeurIPS 2022, arXiv:2204.14198)の gated cross-attention は、視覚特徴を凍結した大規模言語モデルへ注入する枝に `tanh(α)` ゲートを置き、α をゼロ初期化します。学習開始時 `tanh(0)=0` なので言語モデルは一切壊れず、視覚条件が徐々に効いてくる。本残差フュージョンとまったく同じ「凍結した本流に新モダリティを安全に後付けする」設計で、Flamingo はこのゲートにより少数枚の例(few-shot)で画像説明や視覚質問応答に適応できることを示しました。

より進んだ方向として、固定の per-channel から「入力に依存して動的に決まるゲート」へ移る流れがあります。これは Mixture-of-Experts や FiLM(Feature-wise Linear Modulation, Perez et al., AAAI 2018, arXiv:1709.07871)のように、外部信号からチャンネルのスケール・シフトを生成する手法に連なります。FiLM は質問文のような条件から各チャンネルの倍率 γ とシフト β を動的に生成し、視覚推論(CLEVR データセット)で当時の最高精度を達成しました。地図融合でも、シーンの曖昧さ(交差点か直線路か)に応じて地図の効きを動的に変える設計が考えられます。本トピックの固定 per-channel alpha は、この動的変調の最も単純で頑健な特殊形(条件入力に依存しない静的ゲート)と位置づけられます。
