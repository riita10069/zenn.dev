# World Action Model (WAM) の設計

## ひとことで言うと
World Action Model(WAM、ワールドアクションモデル)とは、自動運転車のカメラ映像の「過去の流れ」を比較的ゆっくりした周期(たとえば1秒に1回)で観測し、「この先、世界がどう見えるか」を特徴空間で予測しようとする小さなニューラルモジュールの設計思想です。予測そのものが最終目的ではありません。予測の練習をモデルに課すことで「時間的な世界の理解を圧縮した特徴ベクトル」を獲得させ、それを運転計画(プランナー)の判断材料として渡すのが本当の狙いです。重い画像エンコーダは凍結(更新しない)したまま、その上に乗せた軽いヘッドだけを学習する「安価で再利用しやすい補助モジュール」として設計されるのが典型です。

## 直感的な理解
人間が運転するとき、「今この瞬間の1枚の写真」だけでは正しく判断できません。前の車のブレーキランプが点いた、歩行者が縁石に近づいてきている、対向車が車線をはみ出し始めた——こうした状況は「時間の流れ」を見て初めて分かります。1枚の静止画には「速度」も「相手の意図」も写っていないからです。同じ位置に止まっている車と、これから動き出す車は、1枚の写真の中ではほとんど区別がつきません。

そこで多くの自動運転システムは、過去数秒の映像を1本の要約ベクトル(visual history、視覚履歴)に圧縮し、それを計画に渡したいと考えます。問題は「その要約ベクトルをどう作り、何を学ばせるか」です。単に過去フレームの特徴を平均するだけでは、動きの意味は捉えられません。平均は時間情報を消してしまうからです。

WAM の発想はこうです。「過去から未来を予測できるような要約ベクトルなら、その中には世界のダイナミクス(動きの法則)が詰まっているはずだ」。これは赤ちゃんが世界を学ぶ過程に似ています。ボールを投げると放物線を描いて落ちる、という予測ができるようになった子は、ボールの物理を内部モデル(頭の中のミニチュア世界)として獲得しています。同じように、未来の見え方を予測する練習をさせれば、モデルは「車はこう動く」「歩行者はこう進む」といった世界の規則性を、人間が一枚一枚ラベルを付けなくても(自己教師あり learning で)学べる、というのが核心です。ラベル付けは高価なので、ラベルなしの大量の運転映像をそのまま学習資源にできるのは大きな利点です。

## 基礎: 前提となる概念
WAM を理解するために必要な用語を、知らない人向けに丁寧に説明します。

- バックボーン(backbone): 画像から特徴を取り出す土台のニューラルネット。画像認識で使われる Swin Transformer、ConvNeXt、ResNet などがこれにあたります。生のピクセルを、意味のある高次元の特徴マップ(たとえば [チャネル, 高さ, 幅] という3次元のテンソル)に変換します。
- 凍結(frozen): 学習中にそのネットワークの重み(パラメータ)を更新しないこと。具体的には各パラメータの `requires_grad` 属性を False にして、勾配(更新の方向を示す微分)を流さない状態です。事前学習(大量データで先に学んでおくこと)で得た知識を、そのまま固定して使います。
- 自己教師あり学習(self-supervised learning, SSL): 人間がつけたラベルを使わず、データ自身の中から「予測すべきターゲット(正解)」を作り出して学ぶ方式。「過去から未来を当てる」「画像の一部を隠して残りから埋める」などがその例です。
- JEPA(ジェパ、Joint Embedding Predictive Architecture、結合埋め込み予測アーキテクチャ): 画像のピクセルそのものを予測するのではなく、画像を符号化した「特徴ベクトル(埋め込み)の空間」で未来や欠損部分を予測する自己教師ありの枠組み。LeCun が2022年に提唱しました。ピクセル予測との違いは後述します。
- ワールドモデル(world model): 環境の状態がどう遷移するか(行動を取ると次にどうなるか)を内部的に予測できるモデル。Ha & Schmidhuber が2018年に強化学習の文脈で広めた概念で、自動運転では「次の瞬間のシーンの見え方」を予測するモデルを指します。
- プランナー(planner、計画器): 認識結果と過去の文脈から「これから自車がどう走るか(軌道)」を出力する部品。WAM はこのプランナーに渡す入力の質を上げるための補助モジュールです。
- 1Hz と 10Hz: Hz(ヘルツ)は1秒あたりの回数。カメラは典型的に10Hz(1秒10枚)で画像を出しますが、運転判断に効く「シーンの変化」は数秒スケールでゆっくり起きるため、ワールドモデルは1Hz(1秒1回)に間引いて、その代わり1ステップあたりの情報を厚く持つ、という設計がよく取られます。

## 仕組みを詳しく
代表的な WAM は、次のようなデータの流れを持つ複数の部品から構成されます。記号は B=バッチサイズ、N=過去/未来のフレーム数(例6、1Hz なら約6.4秒分)、V=カメラ台数(例7)、C または D=特徴次元(例256)とします。

```
マルチカメラ画像列 @1Hz   [B, N, V, 3, H, W]
   │
   ▼  (1) Vision Backbone (凍結・再利用)
カメラごと特徴マップ        [B*N*V, C, h, w]
   │
   ▼  (2) Feature Embedding (空間方向の集約)
各フレーム1ベクトル         [B, N, D]
   │
   ▼  (3) Encoded Visual History (時間方向の集約)
過去の要約1ベクトル          [B, D]  ──→ プランナーへ
   │
   ▼  (4) Feature Decoding (JEPA 予測器)
未来Nフレームの予測特徴      [B, N, V*C]
   │
   ▼  (5) JEPA Loss = MSE(予測, 凍結backboneが出す実未来特徴)
```

### 部品1: Vision Backbone(再利用 + 凍結)
過去Nフレームのマルチカメラ画像 `[B, N, V, 3, H, W]` を `[B*N*V, 3, H, W]` に並べ替えて backbone に通し、各画像の特徴マップを得ます(3 は RGB の3チャネル)。この backbone は `requires_grad=False` で凍結します。凍結する理由は3つあります。

1. 勾配の衝突を避ける。同じ backbone が運転本体(BEV 生成や軌道予測)にも使われる場合、WAM の予測損失で backbone を更新すると、「運転に最適な特徴」と「未来予測に最適な特徴」が綱引きになります。凍結すればこの綱引きが消え、運転本体の学習が WAM に乱されません。
2. JEPA のターゲット空間を時間的に固定でき、表現崩壊(representation collapse、特徴が無意味な定数に潰れる失敗)を防げる(詳細は [frozen backboneによるJEPAターゲット](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0024_frozen-backbone-jepa-target))。
3. 計算が安い。backbone の逆伝播(勾配計算)がないので、学習対象は小さなヘッドだけになり、メモリも計算時間も大幅に節約できます。

### 部品2: Feature Embedding(空間方向の集約)
各時刻について、V台のカメラ特徴を「1本のフレーム埋め込みベクトル」に集約します。よく使われるのが、学習可能な1個のクエリベクトルが全カメラに cross-attention(クロスアテンション、注意機構。どのカメラのどの部分を重視するかを学習で動的に決める仕組み)して集約する軽量なアテンションプールです。形の流れは `[B, V, C]`(各カメラを空間平均プールした後)→ アテンション → `[B, C]`(関連: [ViewAttentionPool (マルチビュー集約)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0319_view-attention-pool))。単純平均ではなく学習注意にすることで、「今は左後方のカメラが重要」といった状況依存の重み付けができます。

### 部品3: Encoded Visual History(時間方向の集約)
直近Nフレームの埋め込み `[B, N, D]` を1本の `[B, D]` に圧縮します。これがプランナーに渡る「視覚履歴の要約」です。手法は主に2系統あります。一つは部品2と同じ「学習クエリ + cross-attention」を時間軸に適用する temporal attention-pool(時間方向のアテンションプール)。空間と時間で同じ基本操作(primitive、プリミティブ)を使い回せて設計が統一されます。もう一つは GRU や Transformer などの系列モデルで時間方向に畳み込む方式です。

効率の工夫: backbone が凍結で、各フレームの埋め込みはそのフレームだけで決まる(過去や未来に依存しない)ため、1Hz の毎ステップでは「新しい1フレームだけ」を backbone に通し、過去 N−1 フレームの埋め込みはキャッシュして再利用できます。これにより毎ステップの backbone 計算が「7カメラ × 1フレーム」分で済み、毎回 N フレーム全部を通す素朴な実装に比べて推論が N 倍近く軽くなります。

### 部品4: Feature Decoding(JEPA 予測器)
要約 `[B, D]` から「未来Nフレーム分の特徴」を予測します。1ステップずつ生成していく自己回帰(autoregressive、前の予測を入力に次を予測する方式。誤差が累積しやすい)ではなく、N個の学習クエリが要約に一括で注意する parallel query decoder(並列クエリデコーダ)を使うのが効率的で、`[B, N, V*C]` を一度に出力します。1層の cross-attention と線形層という安価な構成にできます。並列にすることで自己回帰特有の誤差累積を避けられます。

### 部品5: JEPA Loss
正解(ターゲット)は「凍結 backbone を実際の未来Nフレームに通した特徴」で、勾配を切ります(detach、デタッチ。計算グラフからその先を切り離す操作)。予測とターゲットの間で MSE(平均二乗誤差、Mean Squared Error)を取ります。運転全体の総損失は次の形になります。

```
L = L_traj + λ · L_jepa
```

ここで `L_traj` は軌道予測の主損失(プランナーの出力を正解軌道に近づける)、`L_jepa` は未来特徴予測の補助損失、λ(ラムダ)は両者のバランスを取る重みです。λ が大きすぎると未来予測を優先しすぎて運転精度を損ない、小さすぎると WAM がほとんど学習されません。実務では λ を小さめ(主損失を支配させない範囲)から始めてチューニングします。

### 設計の3原則
このようなモジュールはしばしば「Cheap(安い)」「Reuse(既存部品の再利用)」「Decoupled(運転本体から切り離す)」の3原則で設計されます。重い backbone は凍結して再利用、増えるのは小さなヘッドだけ、そして補助損失として運転本体と分離する。これにより既存の運転モデルを壊さずに「世界理解」を後付けでき、もし効果がなければ λ=0 にして無害化できます。

## 手法の系譜と主要論文
WAM は複数の研究系譜が合流した設計です。時系列でたどります。

- ワールドモデルの源流: Ha & Schmidhuber(2018, "World Models", NeurIPS, arXiv:1803.10122)。環境のダイナミクスを潜在空間で学習し、その内部モデルの「夢の中」でエージェントを訓練できると示しました。「未来を予測する内部モデルが行動の役に立つ」という発想の出発点です。
- 自己教師あり崩壊回避: BYOL(Grill et al., NeurIPS 2020, "Bootstrap Your Own Latent", arXiv:2006.07733)。ネガティブサンプル(引き離す相手の画像)なしで、ターゲット側を凍結 / EMA(指数移動平均)にする非対称設計で表現崩壊を防げると示しました。WAM が凍結 backbone を採用する理論的根拠の一つです。
- 特徴空間予測の定式化: JEPA(LeCun, 2022, "A Path Towards Autonomous Machine Intelligence")。ピクセルではなく特徴空間で予測する枠組みを提唱。理由は「ピクセル予測は影やテクスチャ、葉の揺れなど、予測不能で運転に無関係な細部の再現まで強いるため非効率」。効果は「意味的に重要な変化だけを学べる」。トレードオフは「特徴空間なので、予測が当たったかを人間が直接見て確認しにくい」。
- 画像での実証: I-JEPA(Assran et al., CVPR 2023, arXiv:2301.08243)。画像の一部から別部分の特徴を予測する JEPA を実装し、ピクセル再構成より計算効率良く良い表現が得られると示しました。
- 動画への拡張: V-JEPA(Bardes et al., 2024)。動画で時空間マスクされた領域の特徴を予測。時間方向の JEPA をどう設計するかの直接の先行研究であり、WAM の「未来特徴を予測する」発想と直結します。
- 自動運転への応用: MILE(Hu et al., NeurIPS 2022, "Model-Based Imitation Learning for Urban Driving", arXiv:2210.07729)。都市運転で潜在ワールドモデルを模倣学習と組み合わせ、ピクセル予測ベースラインを上回りました。WAM が「特徴空間で未来を予測する」ことの自動運転側の根拠です。
- 生成的ワールドモデル: GAIA-1(Hu et al., 2023, arXiv:2309.17080、Wayve)。大量の運転映像から生成的ワールドモデルを学習し、シーンのダイナミクスの内部表現を獲得。テキストや行動で条件付けて未来映像を生成できます。WAM の将来拡張として「未来予測を行動条件付き(action-conditioned)にする=どこへ運転するかで未来の見え方を変える」案の元ネタになります。初期版では簡単のため行動非条件(unconditioned)にするのが一般的です。

## 論文の実験結果(定量データ)
WAM 自体は設計提案ですが、その根拠となる各論文の定量結果を、指標の意味とともに示します。

- MILE は CARLA(オープンソースの運転シミュレータ)の公式ベンチマークで、訓練で見ていない新しい街・天候(Town05 など)での評価において、当時の最先端を上回りました。評価指標は driving score(運転スコア、ルート完走率と交通違反の減点を掛け合わせた総合点で、高いほど良い)と route completion(ルート完走率)です。MILE は潜在ワールドモデルで未来を予測しながら計画することで、新しい街への汎化(訓練で見ていない環境への対応力)が向上したと報告されています。これは「未来予測の補助タスクが、単なる模倣学習より頑健な表現を生む」ことの実証であり、WAM の中核的な動機を裏付けます。
- BYOL は ImageNet の自己教師あり表現学習において、線形評価(凍結した特徴の上に線形分類器だけを学習し、その分類精度で表現の良さを測る標準的な評価)で約74.3%のトップ1精度を達成し、ネガティブサンプルを使う当時の対比学習(SimCLR など)に匹敵または上回りました。重要なのはアブレーションで、ターゲットネットワークへの勾配を止める(stop-gradient)操作を外すと精度が崩壊(ほぼ役立たない水準に)します。これは「ターゲット側を更新から切り離す非対称性」が崩壊回避の必須要素であることを示し、WAM が backbone を凍結する設計の正当性を裏付けます。
- I-JEPA は ImageNet 上で、ピクセル再構成ベースの手法(MAE、Masked Autoencoder など)と比べて、少ない学習計算量で同等以上の線形評価精度を得たと報告しています。特に「特徴空間での予測は、ピクセル空間より意味的に効率的」という主張を、計算量対精度のトレードオフ曲線で定量的に示しました。WAM がピクセルではなく特徴空間で未来を予測する選択を支えます。
- GAIA-1 は数千時間規模の運転映像で学習し、テキスト・行動プロンプトに応じて多様で時間的に整合した未来シーンを生成できることを、定性・定量(生成品質の指標)の両面で示しました。これは「運転映像から世界のダイナミクスを学べる」ことのスケールでの実証です。

これらを総合すると、「凍結ターゲットへの特徴空間予測」という WAM の中核設計は、画像 SSL と自動運転の両方で再現性ある効果が確認された組み合わせだと言えます。

## メリット・トレードオフ・限界
メリット

- 空っぽになりがちな視覚履歴に、意味のある過去要約を入れられ、プランナーの判断材料が増える
- backbone 凍結で計算が安く、運転本体の学習と干渉しない
- アテンションプールという1つの基本操作を空間・時間の両方で使い回せ、設計がシンプルに保てる
- 1Hz キャッシュで毎ステップ1フレームのみ backbone を通せばよく、推論が軽い

トレードオフ・限界

- 凍結 backbone は「事前学習時点の特徴」しか出せない。運転ドメインに最適化されないため、共有 backbone + stop-gradient(運転本体が更新する非定常ターゲット)と比べると、ターゲットの新鮮さで劣る可能性がある(関連: [online encoderとfrozenターゲットエンコーダ](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0018_online-vs-frozen-target-encoder))
- 未来予測損失はあくまで補助タスク。重み λ を誤ると運転精度を犠牲にする
- 1Hz に間引くため、1秒未満で起きる急な事象(歩行者の飛び出し等)の細かい時間情報は WAM 経路では落ちる。高頻度の反応は別経路(reactive な低レイテンシ経路)で担保する必要がある
- 行動非条件版では「自車が右折するか直進するか」で未来が変わることを表現できない。GAIA 流の行動条件付けは表現力を上げるが、学習が難しくなる
- 研究上の未解決課題として、どの粒度(プール済みベクトルか、空間マップ全体か)で未来を予測するのが運転計画に最も効くか、N をいくつにするのが最適か、複数ホライズン(+1.6s, +3.2s…)を持たせる効果はどれほどか、は依然オープンです

## 発展トピック・研究の最前線
- 行動条件付きワールドモデル(action-conditioned world model): 自車の予定行動を入力に加え「もしこう運転したら世界はこう見える」を予測する。計画(planning)を「複数の行動候補の未来をそれぞれ予測して、最も安全・効率の良いものを選ぶ」問題に変える、強化学習・モデル予測制御(MPC)との接点です。
- 大規模生成ワールドモデル: GAIA-2 や Vista など、運転映像を高解像度で生成し、シミュレーションやデータ拡張(レアな危険シーンの人工生成)に使う研究が活発化しています。
- マルチモーダル融合: カメラだけでなく LiDAR(レーザーで距離を測るセンサ)や地図、自車運動(ego-motion)を条件に加えて未来を予測する方向。時間と空間に加え、センサ間の整合性をどう取るかが課題です。
- 評価の難しさ: 特徴空間予測は人間が直接見て良し悪しを判断しにくいため、下流タスク(実際の運転性能)での評価や、予測特徴を一度デコーダで画像に戻して可視化する研究が進んでいます。
