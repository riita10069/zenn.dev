# L1損失 (平均絶対誤差)

## ひとことで言うと
予測した数値と正解の数値の「差の絶対値」をそのまま損失(罰)とする、最もシンプルな回帰損失の1つです。差が大きくても罰が比例どまりなので、極端に外れたデータ(外れ値)に学習が引きずられにくいのが長所です。複数要素で平均をとった形を平均絶対誤差 (MAE, Mean Absolute Error) と呼びます。統計的には、L1 を最小化する予測は「中央値」に対応します。

## 直感的な理解
クラスの身長を1つの代表値でまとめたいとします。ほとんどが 160cm 前後なのに、入力ミスで1人だけ「1600cm」と記録されていたとしましょう。差を二乗して平均する L2 を最小化すると、代表値は「平均値」になり、その異常な1人に大きく引っ張られて 160cm からずれます。一方、差の絶対値を最小化する L1 を使うと、代表値は「中央値」になり、異常値が1つあっても順番の真ん中は動かないので、ほぼ 160cm のままです。

これが L1 の本質です。L1 にとって、外れ値は「他の点と同じ重みの1票」でしかありません。差が 10 でも罰は 10、二乗の 100 にはなりません。だから外れ値に頑健です。自動運転のように、センサーノイズやラベル付けミスで一瞬だけ飛んだ値が混ざりうるデータでは、この性質が安定した学習につながります。

ただし L1 には弱点もあります。予測が正解にぴったり一致した瞬間、罰の傾き(学習の進む方向と勢いを決める量)が −1 から +1 へジャンプして不連続になります。さらに、正解からどれだけ離れていても傾きの大きさが常に 1 で一定なので、「もうすぐ正解」というときも大股で動こうとして、正解の周りで行ったり来たり振動しがちです。

## 基礎: 前提となる概念

機械学習では、予測と正解の「ズレの大きさ」を1つの数値にまとめ、それを小さくするようモデルを訓練します。ズレの測り方には選択肢があり、選び方でモデルの性格が変わります。学習の更新方向と勢いを決めるのは損失の勾配(微分)なので、損失関数を選ぶことは「どんな勾配で重みを動かすか」を選ぶことと同じです。

L1 損失は、ズレ x = 予測 − 正解 の絶対値 |x| を使います。絶対値とは符号を無視した大きさのことです(+3 も −3 も大きさは 3)。

ここで「最適解が中央値」という性質を、もう少し丁寧に説明します。複数の数 d1, d2, … に対して Σ|c − di| を最小にする1つの値 c は、それらの中央値です。理由は、c を少し右にずらすと「c より左にある点」への距離が全部少し増え、「c より右にある点」への距離が全部少し減る、その差し引きがゼロになるのが、左右の点数が釣り合う中央値だからです。中央値は順位だけで決まり、値の大小に鈍感なので、外れ値に強い代表値になります。これが L1 が外れ値頑健である統計的な根拠です。

対照的に L2(二乗誤差)の最適解は平均値で、平均値は1個の巨大な値に引っ張られます([MSE / L2損失 (平均二乗誤差)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0565_mse-l2-loss))。この「中央値 vs 平均値」の対比は、ノイズ分布とも結びつきます。後述するように、ノイズがラプラス分布(裾の重い分布)に従うとき L1 が最尤推定に一致し、正規分布のとき L2 が最尤推定に一致します。つまり L1 を選ぶことは、暗黙のうちに「ノイズの裾は重い」と仮定していることになります。

## 仕組みを詳しく

要素ごとの損失は次のとおりです。

```
loss(x) = |x|        ただし x = 予測 - 正解
```

複数要素を平均したものが平均絶対誤差 (MAE) です。

数値例(正解 = [2.0, 5.0, 3.0]、予測 = [2.5, 4.0, 8.0]):

- 要素1: |2.5 - 2.0| = 0.5
- 要素2: |4.0 - 5.0| = 1.0
- 要素3: |8.0 - 3.0| = 5.0(外れ値的に大きい差)
- MAE = (0.5 + 1.0 + 5.0) / 3 = 2.17

同じデータを L2(二乗してから平均)で測ると (0.25 + 1.0 + 25.0)/3 = 8.75 となり、要素3の影響が突出します。L1 では要素3の寄与が 5.0 どまりで、他の要素もきちんと損失に効いていることが分かります。要素ごとの寄与率で見ると、L2 では要素3が損失の約 95%(25/26.25)を占めますが、L1 では約 77%(5/6.5)にとどまり、外れ値の支配が緩いことが数字で確認できます。

学習は損失の傾き(微分・勾配)をたどって進みます。L1 の微分は次のようになります。

```
x > 0 のとき:  +1
x < 0 のとき:  -1
x = 0 のとき:  未定義(劣勾配 [-1, +1] のどれか。実装は通常 0 を返す)
```

差がプラスでもマイナスでも傾きの大きさは常に 1 で一定です。これは「正解からどれだけ離れていても更新の勢いが同じ」ことを意味し、正解にぐっと近づいてからも大股で動こうとするため、最適解の周りで振動しやすくなります。x = 0 では傾きが −1 から +1 へ突然ジャンプし、微分が不連続です。深層学習フレームワークでは x = 0 のとき便宜上 0(劣勾配のひとつ)を返す実装が多いですが、不連続性そのものは消えません。劣勾配 (subgradient) とは、折れ曲がった点で微分が一意に決まらないときの「許される傾きの範囲」のことで、|x| の x=0 では [−1, +1] のどれを選んでもよい、という意味です。

この振動を実務で避けるには、学習率を後半で下げる(スケジューリング)か、原点近傍だけ L2 に切り替える Smooth-L1(Huber)を使うのが定番です([Smooth-L1 (Huber) 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0567_smooth-l1-huber-loss))。形状 `[B, T, 2]` のような多次元テンソルにもそのまま適用でき、平均・合計・要素ごと(none)を選べるのは他の回帰損失と同じです。

## 手法の系譜と主要論文

L1 損失(最小絶対偏差、Least Absolute Deviations, LAD)は、最小二乗法より歴史が古いとも言われます。18 世紀の Boscovich が地球の形状(扁平率)を観測データから推定する際に絶対偏差の和を最小化する方法を用い、Laplace がこれを定式化しました。最小二乗法(Gauss、Legendre、19 世紀)が、解が連立一次方程式で閉じた形に書けて計算しやすいという理由で主流になった後も、外れ値への頑健性が必要な場面で LAD は使われ続けました。LAD は線形計画問題として解け、解析解は持たないものの強い頑健性を持つ、という対照が長く知られていました。

近代的な再評価は Koenker & Bassett (1978) "Regression Quantiles" (Econometrica) です。彼らは L1 を一般化し、「分位点回帰 (quantile regression)」を提案しました。これは絶対偏差を非対称に重み付けする(過大予測と過小予測で罰の傾きを変える)ことで、中央値だけでなく任意の分位点(たとえば上位 10% の境界)を推定できる枠組みで、L1 がその対称(分位点 0.5、中央値)の特別な場合に当たります。経済学・計量分析で広く使われ、予測の不確実性を「区間」として出す手法(予測区間)の基礎にもなりました。これは「点予測だけでなく分布の形まで当てたい」という現代の確率的予測へ続く重要な系譜です。

深層学習では、L1 単独より、L1 と L2 を継ぎ目なくつないだ Smooth-L1(Huber)の形で使われることが多いです。Huber (1964) が小さい残差は L2・大きい残差は L1 に切り替える Huber 損失を提案し、Girshick (2015) "Fast R-CNN" がその β=1 版を Smooth L1 と名付けて物体検出の枠回帰に用い、学習を安定化させました(詳細は [Smooth-L1 (Huber) 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0567_smooth-l1-huber-loss))。一方、画像生成・超解像・画像変換の分野では純粋な L1 がむしろ好まれ、Isola et al. "pix2pix"(CVPR 2017、arXiv:1611.07004)など多くの研究が「L1 + 敵対的損失(GAN)」を採用しています。pix2pix の論文は「L1 は低周波(画像の大まかな色・形)を合わせるのが得意で、L2 ほどぼけない。高周波(細部の質感)は敵対的損失に任せる」という役割分担を明確に述べました。

## 論文の実験結果(定量データ)

画像生成・復元の分野には、L1 と L2 を直接比較した定量結果が多くあります。代表的なのは画質指標 PSNR(Peak Signal-to-Noise Ratio、ピーク信号対雑音比。単位 dB、大きいほど元画像に近い)と SSIM(Structural Similarity、構造的類似度。0〜1、大きいほど良い)です。

Zhao et al. (2017) "Loss Functions for Image Restoration with Neural Networks"(IEEE Transactions on Computational Imaging、arXiv:1511.08861)は、同じネットワークを L2 と L1、および MS-SSIM ベースの損失で訓練し直接比較しました。デノイズ・超解像・JPEG ノイズ除去などの復元タスクで、L1 で訓練したネットワークの方が L2 より PSNR・SSIM ともに一貫して良く、視覚的にもアーティファクト(平坦領域の格子状の偽模様 splotchy artifacts など)が少ないと報告しています。論文の説明は明快で、L2 は「誤差の二乗の期待値を最小化する」ため、不確実な領域では複数のもっともらしい解を平均化してぼかすのに対し、L1 は平均化の力が弱く、よりシャープな出力を残すというものです。彼らが最終的に推奨したのは L1 と MS-SSIM を混ぜた損失(Mix と呼ぶ)で、PSNR/SSIM の両方をさらに改善しました。アブレーションとして「L2 → L1 に替えるだけで SSIM が改善する」ことを示した点が、損失選択の重要さを端的に物語っています。

超解像の文脈でも、Lim et al. "EDSR"(CVPRW 2017、NTIRE2017 超解像チャレンジ優勝、arXiv:1707.02921)が「L2 ではなく L1 を使うことで学習が安定し、最終精度(PSNR)も向上した」と明記しています。これらは「外れ値頑健性」というより「平均化によるボケを避ける」という、L1 のもう一つの効用を定量的に示した例です。

統計的な定量直感としては、ノイズがラプラス分布(裾が重い)のときは L1 が最尤推定に一致し、正規分布のときは L2 が最尤推定に一致します。つまり「データのノイズの裾がどれだけ重いか」で、どちらが理論的に正しいかが決まります。実データのノイズは純粋な正規分布より裾が重いことが多く(センサーの飛び値・ラベルミスなど)、これが「現場では L1 系が安定する」経験則の根拠になっています。

## メリット・トレードオフ・限界

メリット
- 外れ値に頑健。差を二乗しないため、ノイズ1点に損失が支配されにくい。
- 最適解が中央値に対応し、極端値に強い推定になる。
- 計算が単純で直感的。生成・復元タスクでは出力がぼやけにくくシャープになりやすい。

トレードオフ・限界
- 原点 (x=0) で微分が不連続。正解近くで学習が振動しやすい。
- 傾きの大きさが常に一定で、誤差が大きくても更新の勢いが増えないため、収束が L2 より遅くなることがある(序盤で大外ししていても更新量が ±1 で頭打ち)。
- 小さい誤差を細かく詰める滑らかな最適化では、L2 や Smooth-L1 に劣る場合がある。
- 多峰性のある正解(複数の正解があり得る状況)では、中央値が「どの正解でもない中間」を指す危険があり、これは L2 が平均でぼかすのと別種だが似た問題。
- 未解決・実務的な注意として、多くの回帰タスクでは純 L1 をそのまま使うより、原点を滑らかにした Smooth-L1 へ置き換える方が安定するため、純 L1 は「概念の土台」として重要だが現場での既定値にはなりにくい。

## 発展トピック・研究の最前線

L1 の拡張として最重要なのが分位点回帰(任意の分位点を推定し、予測の不確実性を区間で出せる)と、それを深層学習に持ち込んだ分位点ネットワーク(強化学習の分布的価値関数 QR-DQN、Dabney et al. AAAI 2018 など)です。QR-DQN は価値を1点の期待値ではなく複数の分位点で表し、報酬の分布そのものを学ぶことで Atari ベンチマークの成績を底上げしました。これは「L1 由来の分位点回帰が現代強化学習の中核に入った」象徴的な例です。また、パラメータの正則化として使う L1(Lasso、Tibshirani 1996)は、損失ではなく「重みのスパース化(多くの重みを完全にゼロにする)」を狙う別文脈の L1 で、特徴選択に使われます。損失としての L1・正則化としての L1 は同じ絶対値でも目的が異なるので、文献を読むときは区別が必要です。生成分野では L1 + 知覚損失(VGG 特徴空間の距離、perceptual loss)+ 敵対的損失の三本立てが今の標準で、純 L1 はその「低周波の骨格を合わせる」役割を担っています。
