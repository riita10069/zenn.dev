# 軌道教師信号 (future window)

## ひとことで言うと
模倣学習(imitation learning = 人間の運転を真似して学ぶ方法)では、「正解=人間がこの後どう運転したか」を教師信号(target、お手本の正解値)としてモデルに与えます。そのために、走行ログのある瞬間 t を選び、t から先の一定区間の未来軌道を切り出して「モデルが当てるべき正解」にします。この切り出した未来の区間を future window(未来ウィンドウ)と呼びます。軌道は生の (x, y) 座標で表すこともあれば、加速度・曲率や操舵角といった「制御に近い量」で表すこともあります。

## 直感的な理解
人間が運転する車にカメラとセンサを積んで、何時間も走行を記録したとします。録画の中の「ある瞬間」に注目し、「この人はこの後6秒間、実際にどう走ったか」を取り出せば、それがそのまま「その状況での正しい運転」のお手本になります。教師あり学習(入力と正解のペアで学ぶ)の正解ラベルを、追加のアノテーション作業なしに、走行ログから自動で大量に作れる——これが軌道教師信号の魅力です。これを自己教師的(self-supervised)なラベル生成と呼ぶこともあります。ラベルは「未来の自分」が勝手に作ってくれるからです。

なぜ「未来」なのかというと、自動運転モデルの仕事が「これからどう動くべきか」を出力することだからです。入力(今の画像・地図・過去の動き)に対し、出力は「未来の軌道」。だから正解も未来でなければ対応しません。未来ウィンドウの長さ(何秒先まで予測するか)は設計者が決める重要なパラメータで、短すぎると先読みが効かず、長すぎると不確実性が大きすぎて学習が不安定になります。研究コミュニティでは数秒(典型的には3〜8秒)を採ることが多く、たとえば 0.1秒刻みで64ステップ(=6.4秒)といった設定が用いられます。

## 基礎: 前提となる概念
- 模倣学習(imitation learning / behavior cloning): 専門家(ここでは人間ドライバー)の行動を真似るように学習する枠組み。最も単純な形が行動クローニング(behavior cloning)で、状態→行動の対応を教師あり回帰で当てる。専門家の方策 π* を観測データだけから真似るので、専門家に追加で質問できない「オフライン」設定が基本です。
- 教師信号 / 監督(supervision): 教師あり学習における「正解ラベル」。軌道予測ではその正解が「未来の軌道」なので「軌道教師信号」と呼びます。
- 自車姿勢(ego pose)と有限差分(finite difference): 走行ログは多くの場合、速度や加速度を直接持たず、6自由度の姿勢(位置 x,y,z と向き)を一定レート(例: 10Hz)で記録しているだけです。そこから速度・加速度・曲率を得るには、隣り合う姿勢の差を時間で割る有限差分(差分÷時間で微分を近似)を使います。
- 曲率(curvature): どれだけ急に曲がっているかを表す量。半径 R の円弧上を走るなら曲率 κ = 1/R。式では κ = yaw_rate / speed(向きの変化率 ÷ 速さ)。速さで割るのは、同じハンドル角でも速いほど大きな円を描く関係を打ち消し、「ハンドルの切り具合」に近い量を取り出すためです。speed が 0 付近だと割り算が発散するので、下限(例: 0.1 m/s)を設けるのが定石です。
- 座標系(frame of reference): 未来軌道を世界座標(地図上の絶対位置)で表すか、自車中心座標(注目時刻の車の位置・向きを原点に取る)で表すかは重要な設計判断です。自車中心座標にすると「どこを走っていても同じ正解の表し方」になり、学習が安定しやすい。多くの実装は注目時刻 t の自車姿勢を原点・前方をx軸に取った座標で未来を表します。
- 共変量シフト(covariate shift): モデルの小さな予測ズレが車の状態を学習データにない領域へずらし、そこでさらに大きくズレる悪循環。純粋な行動クローニングの最大の弱点で、後述の通り対策が研究の中心テーマになりました。

## 仕組みを詳しく
典型的な軌道教師信号の生成手順を分解します。

1. 姿勢列から制御量を導出する。10Hz(0.1秒刻み)の姿勢列に有限差分を適用し、各時刻で次の量を作ります。
   - speed(速さ, m/s): XY平面の位置を時間微分した速度ベクトルの大きさ
   - acceleration(加速度, m/s^2): speed をさらに時間微分(2階差分)
   - yaw_angle(ヨー角, rad): 姿勢のクォータニオン(回転を表す4数)から取り出したZ軸まわりの向き
   - curvature(曲率, 1/m): yaw_rate ÷ speed
   結果は各時刻について `[speed, acceleration, yaw_angle, curvature]` の4信号、形状 `(T, 4)`(T はシーンの総フレーム数)。

2. 注目する瞬間 frame_idx を中心に過去と未来を切り出す。たとえば過去・未来とも64ステップ(=6.4秒)とする設定では次のようになります。
   - 過去(history): frame_idx 直前64ステップ × 4信号 = `(64, 4)` → 平坦化して `(256,)`。これは入力として「ここまでどう動いてきたか」をモデルに教える文脈です。
   - 未来(future window): frame_idx 直後64ステップ分のうち acceleration と curvature の2信号だけ = `(64, 2)` → 平坦化して `(128,)`。これが正解=教師信号です。

   なぜ過去は4信号で未来は2信号かというと、過去は「文脈」として速さや向きも知りたい一方、未来は「これから決める制御」として加速度(縦方向の制御=アクセル/ブレーキ)と曲率(横方向の制御=ステアリング)だけを予測対象にすれば足りるからです。位置は加速度・曲率を時間積分すれば復元できます。この「制御量で表す」流儀は PilotNet 系の系譜で、別の流儀として未来の (x,y) 座標列を直接予測する「軌道点列」表現(UniPlan/nuPlan 系)もあります。どちらを採るかは下流の制御器との接続のしやすさで決めます。

3. 連続走行の長さ制約。中心の前後に64ステップずつ要るので、1サンプルを切り出すには最低 64+64+1 = 129 フレーム(=12.9秒)の連続走行が必要です。これより短い断片は捨てるか除外します。録画の切れ目をまたぐサンプルを作らないよう、シーン境界の管理も必須です。

before/after の形状:
```
before: 10Hzの生の自車姿勢列(位置と向きだけ)  shape (T, 4x4) など
after : 入力用 過去 (256,) + 正解用 未来 (128,)（加速度・曲率）
```

損失(loss)側の工夫。予測軌道と教師軌道の誤差を取るとき、時間減衰(temporal decay)を入れるのが一般的です。たとえば重み w_t = γ^t(γ=0.95 など)で、近い未来の誤差を遠い未来より重く評価します。理由は、すぐ先の予測精度のほうが安全に直結し、かつ遠い未来は本質的に不確実だからです。誤差関数には Smooth L1 Loss(別名 Huber loss。小さい誤差は二乗、大きい誤差は線形に扱い、外れ値に頑健)がよく使われます。L2(二乗誤差)だと大きな外れ値に過剰反応し、L1(絶対値)だと原点で微分が不連続になる——その中間を取る妥協が Smooth L1 です。加速度・曲率を予測する場合、各信号のスケールが大きく違う(加速度は数 m/s^2、曲率は 0.01〜0.1 /m オーダー)ので、信号ごとに正規化(標準化)してから損失を取らないと、スケールの大きい信号に学習が引っ張られます。

## 手法の系譜と主要論文
- PilotNet (Bojarski et al., NVIDIA, 2016, arXiv:1604.07316): カメラ画像から直接ステアリング角を回帰する end-to-end 運転の原点。教師信号が「人間の操舵角」という制御量だった点が、加速度・曲率を予測対象にする発想と直接つながります。ただし瞬間の制御だけを真似たため、誤差が時間とともに累積する共変量シフトに弱いという限界が明確に表れました。
- Conditional Imitation Learning (Codevilla et al., ICRA 2018, arXiv:1710.02410): 「同じ交差点でも左に行くか右に行くかは画像だけでは決まらない」という曖昧さに対し、高レベルの命令(直進/左折/右折)で条件づける枠組みを提案。教師信号としての軌道・制御に「意図」の分岐を持ち込んだ点が新規。出力ヘッドを命令ごとに分け、該当する命令のヘッドだけを学習する branched architecture を導入しました。
- ChauffeurNet (Bansal et al., Waymo, RSS 2019, arXiv:1812.03079): 人間の運転ログから未来の軌道(将来の位置・向き)を教師信号として模倣する枠組みを体系化。最大の貢献は「純粋な模倣だけだと学習データにない失敗状況(車線を大きく外れた等)に弱い」と明示し、わざと悪い状況を合成して回復を教える(摂動・worst case 合成)対策と、衝突・路外逸脱を罰する補助損失を加えたこと。これは「未来軌道を教師にするだけでは不十分」というトレードオフへの正面からの回答です。
- Learning by Cheating (Chen et al., CoRL 2019, arXiv:1912.12294): まず特権情報(俯瞰の真の状態)を見られる教師ネットを訓練し、その後カメラだけを見る生徒ネットに蒸留する2段階アプローチ。教師信号の「質」を上げる工夫として、軌道模倣の系譜で重要な分岐点です。
- UniAD (Hu et al., 上海AIラボ/CVPR 2023 Best Paper, arXiv:2212.10156): 認識・予測・計画を1つのネットワークに統合し、最終的な計画(未来軌道)を全体の目的に据えた。軌道予測を孤立タスクでなく、知覚から計画までの一気通貫の学習信号として位置づけた点が現代的な系譜の到達点です。
- nuPlan (Caesar et al., 2021, arXiv:2106.11810): 軌道計画を「クローズドループで」評価する大規模ベンチマークを整備し、軌道教師信号で学んだ模倣方策がオープンループでは良くてもクローズドループで崩れる現象を再現可能な形で測れるようにしました。

系譜を一言でまとめると、「瞬間の制御を真似る(PilotNet)→ 意図で条件づける(CIL)→ 失敗回復まで教える(ChauffeurNet)→ 教師の質を蒸留で上げる(LBC)→ 知覚〜計画を統合し軌道を最終目標に置く(UniAD)→ クローズドループで正しく測る(nuPlan)」という流れです。

## 論文の実験結果(定量データ)
- ChauffeurNet: 摂動学習(意図的に悪い状態を作って回復軌道を教える)と衝突・逸脱の補助損失を入れると、純粋な行動クローニングに比べてクローズドループ(実際に動かして連鎖的に評価する)での走行が安定し、車線逸脱や衝突が大幅に減ったと報告されています。論文の核心メッセージは「オープンループの当てはまり(1ステップ予測の誤差)が良くても、クローズドループでは破綻しうる」ことで、軌道教師信号を扱う上で必ず意識すべき指標の分離(open-loop vs closed-loop)を示しました。
- Conditional Imitation Learning: CARLA シミュレータで、命令で条件づけたモデルが分岐のある交差点での成功率(目的地まで到達できた割合)を、無条件の行動クローニングより大きく改善。曖昧な状況で教師信号が平均化されて「まっすぐ突っ込む」失敗を防げることを定量的に示しました。
- Learning by Cheating: CARLA ベンチマークで、特権教師からの蒸留により当時の最先端を上回る成功率を達成し、「教師信号の質」が end-to-end 模倣の上限を引き上げることを実証しました。
- 評価指標の意味:
  - ADE(Average Displacement Error, 平均変位誤差): 予測軌道と正解軌道の各時刻の位置差を全時刻で平均した値(メートル)。小さいほど良い。
  - FDE(Final Displacement Error, 最終変位誤差): 予測区間の最終時刻での位置差。先の時刻ほど誤差が大きくなるので、FDE は ADE より大きくなるのが普通。
  - クローズドループ指標: nuPlan の closed-loop score、衝突率、路外逸脱率、進行度(progress)、快適性(jerk = 加速度の変化率)など。ADE/FDE が小さくても危険な衝突を1回起こせば実用上失格——だからクローズドループ指標と併用するのが現代の標準です。実際、模倣ベースラインはオープンループ ADE が良くてもクローズドループ score が大きく下がる(誤差が連鎖蓄積する)ことが nuPlan で繰り返し報告されています。
- 時間減衰の効果(アブレーション的観察): 近い未来を重く評価する重み付けを外すと、遠い未来の不確実な部分に引っ張られて近距離の精度が落ち、クローズドループの安定性が悪化する傾向が広く観察されています。逆に近距離だけ重くしすぎると先読みが効かなくなるため、γ の値はトレードオフのチューニング対象です。

## メリット・トレードオフ・限界
メリット
- 人間の走行ログがそのまま正解になるので、追加の手作業アノテーションなしに大量の教師信号を作れる。スケールしやすい。
- 加速度・曲率という制御に近い量で表すと、出力が車の操作(アクセル/ブレーキ/ステアリング)に直結し、制御器との接続が素直。
- 時間減衰の重み付けで、安全に効く近い未来の精度を優先できる。

トレードオフ・限界
- 共変量シフト: 純粋な模倣は失敗状況の回復を学べない。ChauffeurNet 流の摂動学習や、DAgger(専門家に追加で問い合わせる)、強化学習との併用が要る。
- 微分ノイズの増幅: 姿勢から有限差分で速度・加速度・曲率を導くため、姿勢のノイズが微分で増幅される。特に加速度・曲率(2階微分系)はノイズに敏感で、平滑化フィルタ(移動平均、Savitzky-Golay 等)が必要になりがち。
- 因果性のズレ(causal confusion): 人間は将来を「知って」運転しているわけではないのに、教師信号は実際にたどった未来そのもの。さらにモデルが「直前の自分の速度・加速度」を入力に持つと、それをコピーするだけで低い損失を達成でき、画像を見なくなる(inertia problem / copycat)現象が知られています。過去の自車運動の入力量を制限する設計が要ります。
- マルチモーダル性の潰し: 「直進も左折もありうる」状況で唯一の未来軌道を正解にすると、平均化されて中間(あり得ない軌道)を学ぶ。混合密度ネットワーク(MDN)や複数候補出力(winner-takes-all 損失)、条件づけ(CIL)で対処する。
- 「人間が常に正解」前提: データに含まれる悪い運転癖(急ブレーキ、信号無理通過)も無批判に学んでしまう。データキュレーションでの選別が要る。

## 発展トピック・研究の最前線
- マルチモーダル軌道予測: 単一軌道でなく複数候補とその確率を出す(MultiPath, Trajectron++, MTR など)。長尾の曖昧状況に強い。
- 計画と予測の統合: UniAD/VAD のように、周辺エージェントの予測と自車計画を相互作用させて一貫した軌道を出す方向。
- 強化学習・逆強化学習との融合: 模倣で初期化し、報酬(衝突回避・快適性)で微調整して共変量シフトを緩和する。MaxEnt IRL は「人間の軌道が最適に見える報酬」を逆算する系譜。
- 言語・基盤モデルとの接続: 大規模言語モデル/視覚言語モデルで「なぜその軌道か」を説明・条件づける研究。教師信号に高レベルの意図や常識を注入する流れ。
- クローズドループ評価の標準化: オープンループ指標(ADE/FDE)だけでなく、シミュレータでの衝突率・到達率を主指標に据える動き(nuPlan、CARLA Leaderboard、Bench2Drive 等)。
- 拡散モデルによる軌道生成: 未来軌道を拡散過程で生成し、マルチモーダル性と滑らかさを両立させる Diffusion Planner 系の研究が活発化しています。
