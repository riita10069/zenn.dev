# クラス重み付き損失

## ひとことで言うと
クラス重み付き損失(class-weighted loss)とは、データの中で数が少ない希少クラスの間違いを「重く」、数が多い多数派クラスの間違いを「軽く」数えることで、クラス不均衡(クラスごとのサンプル数の偏り)を補正するテクニックです。クロスエントロピーにクラスごとの重みを掛けるだけで実現でき、データそのものを増減させずに、損失計算の中で少数派の「発言力」を上げます。希少だが重要なクラスの取りこぼしを減らすための、最もシンプルで広く使われる不均衡対策です。

## 直感的な理解
あるクラスの担任が成績をつけるところを想像してください。生徒が40人いて、そのうち38人が同じ平凡な答案、2人だけがまったく違う特別な答案だとします。「全員を平等に扱う」採点では、38人の答案を正しく扱えれば全体の95%が当たるので、特別な2人を無視しても高得点に見えてしまいます。でも本当に注意して見るべきはその2人かもしれません。

機械学習でも同じことが起きます。データの大部分が「ありふれた状況」で、ごく一部が「まれだが決定的に重要な状況」のとき、すべてのサンプルを平等に扱うと、モデルは多数派さえ当てればよいと学習し、希少クラスをほぼ捨ててしまいます。クラス重み付けは「君たち2人の答案は38人分の重みがある」と宣言することで、モデルに少数派を真剣に学ばせます。なぜこれが要るのか。多数派は放っておいても学べますが、安全や異常検知の価値はしばしば少数派の側にあるからです。自動運転で言えば「直進し続けるだけの平凡なシーン」は山ほどあるが、「子どもの飛び出し」のような最重要シーンは滅多に記録されない、という構図です。

## 基礎: 前提となる概念
まずクロスエントロピー損失。分類でモデルの予測確率と正解のズレを測る指標で、サンプル i の損失は `- log(p_i[y_i])` と書けます。p_i[y_i] はそのサンプルの正解クラス y_i にモデルが付けた確率、log は自然対数です。正解確率が1に近いほど損失は0に近づき、外すほど損失が大きくなります。詳細は [クロスエントロピー損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0352_cross-entropy-loss) を参照してください。

次にクラス不均衡(class imbalance)。学習データでクラスごとのサンプル数が大きく偏っている状態です。具体例として、走行シーンの主因を5クラスに分類する課題を考えます。

```
clear        : 8000 サンプル   (特に何もない直進が大多数)
intersection : 1200
traffic_light:  600
obstacle     :  150
pedestrian   :   50    ← いちばん重要なのに最少
```

最後に長尾(ロングテール、long-tail)問題。少数のクラスに大量のサンプルが集中し、多数のクラスがごく少数のサンプルしか持たない、分布の裾が長く伸びた状況を指します。「まれだが重要」なクラスがこの裾にいることが多く、安全や信頼性はしばしばこの裾で決まります。

ここで普通のクロスエントロピー(全サンプル平等)で学習すると何が起きるか。総損失はサンプル数の多いクラスに支配されます。極端な話、モデルが「常に clear と答える」だけでも 8000/10000 = 80% の正解率になり、損失もかなり下がります。結果、モデルは多数派にだけ最適化され、最重要の pedestrian クラスをほとんど学ばないという最悪の事態が起きます。全体の正解率(accuracy)という指標自体が、不均衡下では「常に多数派と答える無能なモデル」を高く評価してしまうという罠もここにあります。だから不均衡下では accuracy ではなく、クラスごとの再現率(recall)を平均した balanced accuracy や、希少クラスの F1 を見るのが定石です。

## 仕組みを詳しく
各クラス c に重み w_c を与え、そのクラスに属するサンプルの損失を w_c 倍します。クラス重み付きクロスエントロピーは、

```
loss_i = - w_{y_i} * log(p_i[y_i])     # y_i はサンプル i の正解クラス
```

となり、希少クラスのサンプルほど大きな w が掛かって総損失への寄与が増えます。勾配も同じ係数で増幅されるので、最適化が希少クラスを当てる方向へ強く引かれます。

重みの決め方には代表的な流儀があります。最も単純なのは逆頻度重み(サンプル数の逆数)です。

```
w_c = N / (K * n_c)
   N   : 全サンプル数
   K   : クラス数
   n_c : クラス c のサンプル数
```

上の分布(N=10000, K=5)で計算すると、

```
clear        : 10000/(5*8000)  = 0.25
intersection : 10000/(5*1200)  = 1.67
traffic_light: 10000/(5*600)   = 3.33
obstacle     : 10000/(5*150)   = 13.33
pedestrian   : 10000/(5*50)    = 40.0   ← 最重要クラスを40倍重く
```

これで「pedestrian を1回間違える」ことが「clear を40回間違える」のと同じ重さになり、モデルは少数派を無視できなくなります。before/after の直感で言うと、重みなし(平等)では総損失に占める pedestrian の割合は 50/10000 = 0.5% にすぎませんが、40倍重みを掛けると存在感が一気に増し、勾配が pedestrian を当てる方向に強く働きます。

ただし逆頻度重みは強すぎることがあります。pedestrian がわずか50サンプルしかないのに40倍も増幅すると、その50サンプルに含まれるノイズや偏りにモデルが過敏に反応し、学習が不安定になります。そこで実務では和らげる工夫をします。

```
w_c = sqrt(N / n_c)        # 平方根で効果をマイルドに
w_c = clip(N/(K*n_c), max=10)   # 上限でクリップ
```

実装はフレームワークの `weight` 引数に長さ K のテンソルを渡すだけで、計算量の増加はありません。

```python
class_weights = torch.tensor([0.25, 1.67, 3.33, 13.33, 40.0])  # [K]
loss = cross_entropy(logits, labels, weight=class_weights)
```

重み設計のより洗練された考え方が「実効サンプル数」です。サンプルが増えるほど新しいサンプルがもたらす情報は重複して減っていく(101枚目の犬の写真は1枚目ほど新規性がない)という観察から、頻度の逆数より緩やかに重みを減らすべきだという発想で、後述の Cui et al. (2019) が定式化しました。直感的には、各クラスのサンプルが特徴空間の小さな領域(体積)を覆っており、新しいサンプルがすでに覆われた領域に落ちる確率を考えると、有効な「新規」サンプル数は頻度より飽和的に伸びる、という幾何的なモデルです。

## 手法の系譜と主要論文
クラス重み付けは特定の1論文の発明というより、コスト考慮型学習(cost-sensitive learning)として古くからある考え方ですが、系譜は明確に辿れます。

統計学の側では King & Zeng の "Logistic Regression in Rare Events Data"(Political Analysis 2001)が、希少事象(戦争や金融危機のような、ほとんど起きないが重要な事象)のロジスティック回帰でクラス頻度の偏りが推定を歪めること、重み補正で是正できることを論じました。機械学習の不均衡問題を体系化したのが Japkowicz & Stephen の "The Class Imbalance Problem: A Systematic Study"(Intelligent Data Analysis 2002)で、不均衡が分類器を多数派へ偏らせる度合いを系統的に測り、対策を「リサンプリング(少数派を複製して増やすオーバーサンプリング、多数派を間引くアンダーサンプリング)」と「コスト調整(=クラス重み付け)」の2系統に整理しました。重み付けの効果は少数派の再現率(取りこぼしの少なさ)向上、副作用は多数派の精度低下や少数派ノイズへの過敏化だと指摘されています。

深層学習時代の代表が Cui, Jia, Lin, Song, Belongie の "Class-Balanced Loss Based on Effective Number of Samples"(CVPR 2019、arXiv:1901.05555)です。単純な逆頻度重みは強すぎるとして、実効サンプル数 `E_n = (1 - β^{n_c}) / (1 - β)` を導入し、重みを `w_c ∝ (1-β) / (1 - β^{n_c})` と設計しました。β はデータ重複度を表すハイパーパラメータ(0.99〜0.9999 程度)で、β→0 なら全クラス等重み、β→1 なら逆頻度重みに近づきます。逆頻度と等重みの中間を連続的に取れるのが新規性です。

もう一つの方向が Cao, Wei, Gaidon, Arechiga, Ma の LDAM-DRW(NeurIPS 2019、arXiv:1906.07413)です。重みを掛けるのではなく、希少クラスほど決定境界からのマージン(余裕)を大きく取るよう損失を設計し(マージンをクラス頻度の `1/n_c^{1/4}` に比例させる)、さらに「学習の後半でだけ重み付けを有効化する」DRW(deferred re-weighting)という学習スケジュールを提案しました。早すぎる重み付けは特徴抽出の学習を乱すため、まず通常学習で良い特徴を学び、後半で再重み付けして分類境界だけ調整する、という二段構えです。

クラス単位ではなく例単位で重み付けする相補的アプローチが [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) です。両者は組み合わせ可能で、クラス重み α と難例重み `(1-p)^γ` を併用する形は物体検出で標準的に使われます。さらに理論的にきれいな代替として、後述するロジット調整(Menon et al. 2021)が「重みを掛ける」のではなく「ロジットに事前確率の対数を足す」アプローチを提案しています。

## 論文の実験結果(定量データ)
Cui et al. (2019) は長尾化した CIFAR(CIFAR-10/100 を人工的に不均衡にしたもの)と iNaturalist(自然界の生物画像で、種ごとのサンプル数が現実に大きく偏った大規模長尾データセット)、ImageNet-LT で評価しました。評価指標はトップ1誤り率です。不均衡度を表す指標 imbalance factor(最多クラスと最少クラスのサンプル比)が 100(=最多が最少の100倍)という強い長尾の Long-Tailed CIFAR-10 で、通常のクロスエントロピーに対し class-balanced loss は誤り率を数ポイント改善したと報告されています。重要なアブレーションとして、β を変えたときの感度を示し、最適 β がデータの不均衡度に依存すること(強い不均衡ほど β を1に近づける=逆頻度寄りが良い)を明らかにしました。これは「逆頻度か等重みか」という二択ではなく中間に最適点があることの実証です。彼らは class-balanced 項を softmax CE だけでなく sigmoid CE や Focal Loss に乗せても効くことを示し、汎用的な重み設計であることを裏づけました。

LDAM-DRW(Cao et al. 2019)は同じ長尾 CIFAR で、単純な再重み付けや再サンプリングを上回る精度を示しました。最も示唆的なアブレーションは DRW の有無で、同じ LDAM 損失でも「最初から重み付け」より「後半でだけ重み付け(DRW)」のほうが明確に高精度でした。これは「重み付けをいつ始めるか」というタイミングが、重みの値そのものと同じくらい重要だという、実務に直結する知見です。具体的には、LDAM 単体に DRW を組み合わせると長尾 CIFAR-10/100 で数ポイントの上乗せがあり、当時の最先端を更新しました。

統計側の King & Zeng (2001) は、希少事象データで補正なしのロジスティック回帰が希少事象の発生確率を系統的に過小評価することを示し、事前確率補正(prior correction)と重み付けの組み合わせで推定バイアスを大きく減らせることを定量的に示しました。

これらを総合すると、クラス重み付けの効果は「希少クラスの再現率が上がる代わりに多数派の精度がいくらか落ちる」というトレードオフが基本で、重みの強さ・設計・適用タイミングのチューニングで最適点を探すのが実務、という像が見えます。

## メリット・トレードオフ・限界
メリット。希少だが重要なクラスの取りこぼしを減らせる。実装は `weight` 引数1つで、データを増減させるリサンプリングと違いデータ量・データローダ構成を変えない。[ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing) や [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) など他の損失と自由に組み合わせられる。計算コストの増加はほぼゼロ。

トレードオフと限界。第一に、重みが強すぎると少数派のわずかなノイズに過敏になり、学習が不安定化したり多数派の精度が落ちます。第二に、適切な重みの決め方(逆頻度・平方根・実効サンプル数・LDAM マージン)はデータ依存で、チューニングが避けられません。第三に、Cao et al. が示したように適用タイミング(最初からか後半からか)も性能を左右します。第四に、これはクラス単位の補正なので、「同じクラス内の易しい例/難しい例」の偏りには無力です(そこは [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss) の領分)。第五に、重み付けは損失のスケールを変えるため、学習率や勾配クリッピングと相互作用し、ハイパーパラメータの再調整が要ることがあります。第六に、訓練データのクラス事前確率とテスト時に期待される事前確率が異なる場合(訓練だけ不均衡で評価は均衡、など)、重みは「テスト分布に合わせるための補正」と考える必要があり、闇雲に逆頻度を使うと逆効果になりえます。

研究上の未解決課題は、重みを自動・適応的に決める方法、特徴学習と分類境界調整を分離する設計(LDAM-DRW の発想の一般化)、そして「再重み付け・再サンプリング・マージン調整・ロジット補正」という乱立する不均衡対策を統一的に理解する枠組みです。近年は学習後にロジットへ事前確率を足し引きする「ロジット調整(logit adjustment)」が理論的にきれいな代替として注目されています。

## 発展トピック・研究の最前線
ロジット調整(Menon et al., ICLR 2021, "Long-tail learning via logit adjustment", arXiv:2007.07314)は、学習や推論時にロジットへ `τ·log(クラス事前確率)` を加減することで、ベイズ最適な不均衡補正を実現する手法で、重み付けより理論的根拠が明快です。学習時に組み込む版(logit-adjusted softmax)と、学習済みモデルの推論時にだけ事前確率を引く版があり、後者は再学習なしで適用できます。

二段階学習(decoupling、Kang et al. ICLR 2020, arXiv:1910.09217)は「表現学習」と「分類器学習」を分け、特徴抽出は通常データ(または instance-balanced サンプリング)で学び、最後に分類器だけを均衡サンプリングで再学習する戦略です。彼らは「良い特徴は不均衡データからでも普通に学べ、問題は分類器の重みノルムが多数派に偏ることだ」と分析し、分類器だけ調整する cRT や、重みノルムを正規化する τ-normalized といった軽い後処理で大きく改善できることを示しました。LDAM-DRW の DRW と同じく「重み付けは特徴学習を乱すので分類段でだけ使う」という洞察に基づきます。

リサンプリング側では、少数派を単純複製せず特徴空間で補間して新サンプルを合成する SMOTE 系や、生成モデルで少数派を増やすアプローチがあります。重み付けと組み合わせることも多いです。

自動運転や異常検知のように「希少クラスこそ本丸」の応用では、クラス重み付け・Focal Loss・ラベルスムージングを組み合わせ、不均衡(頻度の偏り)・難易度(例ごとの易しさ)・ラベルノイズ(誤ラベル)という3つの異なる病に同時に対処する構成が一般的です。それぞれが別の軸を補正するため、相補的に効きます。研究コミュニティでは、これらを場当たり的に重ねるのではなく、ロジット調整のような理論的に裏付けられた一手で代替できないか、という方向の整理が進んでいます。
