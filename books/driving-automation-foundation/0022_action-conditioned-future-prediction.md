# 行動条件付き未来予測 (GAIA風)

## ひとことで言うと
「これから世界がどう見えるか」を予測するとき、「自分(自車)がこれからどう動くか(計画した軌道)」も入力に加えるかどうか、という設計の論点です。行動を与えると「左に曲がったら景色はこう見える」「直進したらこう見える」という、自分の行動次第で変わる未来を予測できるようになります。これを行動条件付き(action-conditioned)予測と呼び、与えない方式を無条件(unconditioned)予測と呼びます。

## 直感的な理解
交差点で停まっているとき、次の瞬間に見える景色は何で決まるでしょうか。一つは外の世界の動き、つまり他車や歩行者がどう動くかです。しかしもう一つ、決定的に効くのが自分の運転です。右折すれば右側の道が、直進すれば前方の道が、左折すれば左側の道が見えてきます。同じ過去・同じ周囲でも、自分の操作次第で次の景色はまったく違うものに分岐します。

ここで未来予測を素朴に「過去だけ見て1つの未来を当てる」と作ると問題が起きます。本来は自分の行動で複数に分岐する未来を、無理に1つの平均像に当てさせることになり、予測が「右折の景色と直進の景色を混ぜたぼやけた像」になってしまいます。これは平均への回帰と呼べる現象で、予測の鋭さが失われます。自分の行動を条件として入れれば、「この軌道で走ったらこの未来」と分岐を解きほぐせるので、予測が鋭くなります。

さらに嬉しいのが反実仮想(counterfactual、「もし〜だったら」)のロールアウトです。「軌道Aで走ったら未来はどう見える?」「軌道Bなら?」を同じモデルで比べられ、これは計画(どの軌道が安全か)に直接使えます。これがワールドモデル(world model、環境のダイナミクスを学んで頭の中でシミュレーションする仕組み)の核心で、行動を入れて初めて「頭の中で運転を試す」ことが可能になります。一方、無条件にも「実装が単純」「軌道の質に予測が引きずられない」「まず世界が勝手にどう動くかだけを学べる」という利点があり、最初の版ではあえて無条件を選ぶ判断もよく取られます。つまりこれは「やるかやらないか、いつやるか」の設計論点です。

## 基礎: 前提となる概念
行動(action)とは、エージェント(ここでは自車)が環境に対して取る操作のことです。自動運転では、ステアリング角・アクセル・ブレーキの時系列や、これから辿る予定の軌道(trajectory、未来の各時刻における自車の位置や速度の系列)で表されます。軌道は典型的には「N個の未来時刻 × (x, y) 座標」のような数値列で、これを1列に平坦化して `[B, action_dim]` のベクトルにします。

条件付け(conditioning)とは、予測や生成の入力に追加の情報を与えて、出力をその情報に応じて変えさせることです。画像生成でテキストを条件に与えると「猫の絵」「犬の絵」を出し分けられるのと同じで、未来予測に行動を条件として与えると「この行動を取ったときの未来」を出し分けられます。条件付けの形式的な意味は、無条件が分布 p(未来 | 過去) を学ぶのに対し、行動条件付きは p(未来 | 過去, 行動) を学ぶ、という違いです。後者のほうが条件が増えるぶん分布が鋭く(分散が小さく)なります。

ワールドモデルとは、環境がどう変化するか(ダイナミクス)を学習したモデルで、現在の状態と行動を入力に「次の状態」を予測します。これがあれば実際に環境で試さなくても、頭の中(モデルの中)でロールアウト(rollout、行動を次々に与えて未来を展開すること)ができ、どの行動系列が良いかを計画できます。

潜在空間(latent space)とは、観測(画像など)を圧縮した抽象的なベクトルの空間です。ピクセルで未来を予測するより、潜在空間で予測するほうが軽く、ダイナミクスを学びやすいことが多いため、多くのワールドモデルは潜在空間で動きます。

## 仕組みを詳しく
行動条件付けの実装は、典型的には「行動を表すベクトルを、特徴に足し込む」という単純な形を取ります。具体例で示します。融合済みの視覚特徴 `fused_features`(形状 `[B, C, H, W]`、C はチャンネル数、H×W は空間解像度)があるとします。

無条件版では、プランナ(計画器)の内部状態 `ego_hidden`(たとえば256次元で、その時刻にプランナが持つ「意図の要約」)を線形変換し、チャンネルごとのバイアス(各チャンネルに一律に足すオフセット)に変えて特徴に加えます。

```
bias = Linear(ego_hidden).view(B, C, 1, 1)
conditioned = fused_features + bias
```

これを畳み込み数層で「N時刻ぶんの未来特徴」に展開し、N分割して `[B, C, H, W]` を N 個返します(たとえば +1.6/3.2/4.8/6.4s)。

行動条件付き版では、計画した軌道を平坦化したベクトル `trajectory`(形状 `[B, action_dim]`、たとえばタイムステップ数 × 信号数を1列に並べたもの)を、もう1枚の線形層でチャンネルバイアスに変換し、上のバイアスに加算します。

```
action_bias = Linear(trajectory).view(B, C, 1, 1)
conditioned = conditioned + action_bias
```

これにより予測した未来特徴が、与えた軌道に応じて変わります。軌道A・Bを差し替えれば反実仮想ロールアウトになります。

ASCII図で条件の入り方を示します。

```
                 ┌── ego_hidden ──linear──┐
fused_features ──┤                         ├──(加算)→ conditioned ─Conv×N→ 未来特徴×N
   [B,C,H,W]     └── trajectory(任意) ─linear──┘
                       行動条件付けのときだけ
```

設計上の良い配慮として、行動経路をオプトイン(明示的に有効化したときだけ動く)にしておく方法があります。`action_dim` が未指定なら action 経路が一切実行されず、無条件版と完全に同一の出力になります。こうしておくと、機能を追加しても既存の挙動を一切壊さず、有効化したときだけ条件付けが効く、という安全な拡張ができます。

なぜ「足し込む」という単純な方式でよいのか。条件情報を特徴に加算するのは、FiLM(Feature-wise Linear Modulation、Perez et al., 2018, AAAI, arXiv:1709.07871)の系譜にある軽量な方法だからです。FiLM は条件からチャンネルごとのスケール γ とシフト β を作り、特徴に γ·x + β を施して条件を効かせます。上の加算はこのうちシフト β だけを使った簡略版で、少ない追加パラメータで条件を効かせられます。より強く効かせたければ、スケール γ も使う完全な FiLM にする、加算ではなく cross-attention で軌道を参照させる、各時刻の軌道点を対応する未来クエリに個別に注入する、といった凝った設計も可能ですが、まずは加算で十分なことが多いです。FiLM が元々 VQA(画像への質問応答)で「質問テキストに応じて画像特徴を変調する」のに使われたように、加算的な条件付けは弱いように見えて実は表現力が高いことが知られています。

## 手法の系譜と主要論文
行動条件付き未来予測は、強化学習のワールドモデルから始まり、運転の生成モデルへと発展しました。

古典は World Models(Ha & Schmidhuber, 2018, NeurIPS, arXiv:1803.10122)です。観測を VAE(変分オートエンコーダ、画像を確率的な潜在ベクトルに圧縮する生成モデル)で潜在に圧縮し、RNN で「現在の潜在と行動から次の潜在」を予測するワールドモデルを学び、その想像のロールアウトの中でエージェントを訓練しました。行動条件付き予測が方策学習に直結することを示した出発点です。続く Dreamer(Hafner et al., 2020, ICLR, arXiv:1912.01603)は、潜在空間のワールドモデル(RSSM、確率的・決定的状態を併せ持つ再帰状態空間モデル)を使い、想像上のロールアウトを通して勾配を流して方策を学ぶ枠組みを洗練させました。DreamerV3(Hafner et al., 2023, arXiv:2301.04104)はこれをさらに安定化させ、共通する弱点である「モデル誤差が想像ロールアウトを重ねるほど累積し、長期予測が信頼できなくなる」問題に、正規化や対称的な変換で対処しました。

運転ドメインでは MILE(Model-based Imitation LEarning、Hu et al., 2022, NeurIPS, arXiv:2210.07729, Wayve)が、行動と観測を潜在空間で結ぶ運転ワールドモデルを学び、想像上のロールアウトで方策を改善しました。行動条件付き予測があれば頭の中で運転を試せる、という発想を運転に持ち込んだ研究で、CARLA(運転シミュレータ)のクローズドループ評価で当時の模倣学習を上回りました。

そして GAIA-1(Hu et al., 2023, Wayve, arXiv:2309.17080)が、運転の生成的ワールドモデルとして注目されました。過去映像に加えてテキストや行動を条件に、未来の運転映像を生成します。映像を離散トークン化(画像を有限個の符号に量子化する)し、Transformer で自己回帰的に次トークンを予測する、言語モデルに近い大規模な構成です。行動を条件にすることで「操作に応じて分岐する多様な未来」を生成でき、本トピックのタイトル「GAIA風」はこの系譜を指します。トレードオフは、巨大で計算が重く、自己回帰生成が遅いことです。

対照的に、I-JEPA / V-JEPA(Assran et al., 2023; Bardes et al., 2024)系の純粋な特徴予測は基本的に無条件で、まず表現を学ぶことに集中します。最初の版で「当面 unconditioned」を選ぶのは、この路線に近い割り切り(まず表現、行動条件付けは後)と言えます。両者の中間として、行動条件付けを後から JEPA 系の軽量予測器に足す研究も増えており、「映像生成はしないが、行動に応じた未来特徴は予測する」という GAIA と JEPA のいいとこ取りが模索されています。

## 論文の実験結果(定量データ)
World Models は、当時の強化学習タスク(CarRacing、VizDoom Take Cover)で、想像のロールアウトの中だけで方策を学んでも実環境で高いスコアを出せることを示しました。CarRacing では当時の報告で 900 点超(その課題で「解けた」とされる基準を超える水準)を達成し、とくに VizDoom では「実環境にほとんど触れずにモデル内の夢の中で訓練し、それを実環境に移して動く」結果が、行動条件付きモデルの計画利用の可能性を印象づけました。

Dreamer 系は、DreamerV2 が Atari でモデルフリー手法に匹敵し、DreamerV3 が Atari 100k(10万フレームという少ないデータでの学習)や連続制御、Minecraft のダイヤ採掘という難題まで、ハイパーパラメータを変えずに当時最高水準を達成したと報告されています。とくに DreamerV3 は、設定を変えずに多数の異なるタスクで安定して高性能を出した点が評価され、行動条件付きの潜在ワールドモデルが汎用的に機能することを示しました。アブレーションでは、ワールドモデルの予測精度(次状態をどれだけ当てられるか)と最終的な方策性能が相関し、行動条件付き予測の質が計画の質を左右することが確認されています。

GAIA-1 は、実走行データで学習した約90億パラメータ規模のモデルで、行動・テキストを条件に数十秒スケールの一貫した運転映像を生成できると報告されました。定量評価は生成映像の品質指標(FID: Frechet Inception Distance、生成画像の分布が本物の分布にどれだけ近いかを測る指標。低いほど良い)などで行われ、条件を与えることで操作に応じた多様で妥当な未来(歩行者の出現、対向車、信号など)が生成されることが示されました。MILE は CARLA のクローズドループ評価(モデルが実際に車を動かし続ける評価)で、当時の模倣学習ベースラインを上回る運転性能(ルート完走率や違反の少なさを統合した driving score)を報告し、行動条件付きワールドモデルが運転方策の改善に寄与することを示しました。

これらを総合すると、行動条件付けは「予測の鋭さ」「計画への直結」という利点を一貫して示す一方、「モデルが大きく重くなる」「想像ロールアウトで誤差が累積する」というコストも一貫して現れます。無条件で表現を学んでから行動条件付けを段階導入する、という慎重な進め方が合理的である根拠になります。

## メリット・トレードオフ・限界
メリットは、自分の行動次第で変わる未来を予測でき、予測のあいまいさ(平均への回帰によるぼやけ)が減ること。反実仮想ロールアウト(軌道A vs B の未来比較)ができ、計画・安全性評価に直結すること。オプトイン設計にすれば無効時は既存挙動と完全に同一に保てることです。

トレードオフと限界もあります。第一に、入力(軌道)が増え、実装と学習が複雑になります。学習初期で軌道がまだ不正確だと、その悪い行動が予測の足を引っ張りうる(鶏と卵の問題)。第二に、「世界が勝手にどう動くか」と「自分の行動効果」が同じ予測に混ざり、モデルが何を学んでいるか切り分けにくくなります。第三に、ワールドモデル一般の問題として、想像ロールアウトを重ねるとモデル誤差が累積し、長期予測が信頼できなくなります(compounding error)。

研究上の未解決課題として、因果の混同(confounding)があります。データ上「ブレーキを踏む」と「前に障害物が現れる」が同時に起きやすいと、モデルが「ブレーキを踏むと障害物が現れる」という逆向きの誤った関係を学びかねません。これは観測データから因果を学ぶ難しさそのもので、模倣学習で知られる causal confusion(過去の行動という「ショートカット」を見て未来を当ててしまい、本当の原因を学ばない)とも通じます。行動が本当に未来の原因として働くよう学ばせる、因果的に正しい条件付けは難問で、行動を強制的に変えたデータ(介入)や、行動経路をわざと一部遮断する正則化などが対策として研究されています。また、安く始めたい初期の方針とは相性が悪く、まず無条件で動かしてから段階導入するのが無難、という判断がよく取られます。

## 発展トピック・研究の最前線
最前線は「重い生成モデル」と「軽い特徴予測」の両極で進んでいます。GAIA-2 や各種の運転拡散モデル(動画拡散で未来フレームを生成する手法)は、行動条件付きでよりリアルで長尺・高解像度の運転映像を生成する方向を追求し、シミュレーション・データ拡張・計画評価への応用を狙います。生成された未来映像は人間が目で検証できるという強みがあり、安全性の議論や希少シーンの合成に向きます。一方で JEPA 系の軽量な特徴予測に行動条件付けを後付けする研究も増えており、「映像を生成せず、行動に応じた未来特徴だけを安く予測して計画に使う」方向が、レイテンシの厳しい実車応用では現実的とされます。

もう一つの潮流は、行動条件付きワールドモデルを使った計画(planning)の高度化です。複数の軌道候補を並列にロールアウトして安全コストを評価し、最良の軌道を選ぶ「モデル予測制御(MPC)的な使い方」や、ワールドモデルの想像の中で強化学習する手法が、実走行データから学んだモデルで試みられています。ここでは、想像の中での誤差累積をどう抑え、計画が信頼できる予測ホライズン(何秒先まで信じてよいか)をどう見極めるかが鍵になります。誤差累積を抑えるため、長期は粗い潜在で・短期は細かい特徴で予測する階層的なロールアウトや、予測の不確実性を明示的に出して「自信のない予測は計画で割り引く」設計が探られています。
