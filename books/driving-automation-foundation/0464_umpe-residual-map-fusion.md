# ゼロ初期化残差によるマップ融合

## ひとことで言うと
地図から取り出した特徴を、画像から取り出した特徴へ「最初はまったく効かせない(ゼロ)状態」から足し始め、学習が進むにつれて効き具合を自動的に強めていく融合のしかたです。式で書くと `出力 = 画像特徴 + α × 地図特徴` で、係数 `α`(アルファ)をゼロで初期化するのが要点です。これにより、地図という新しい入力をモデルに足しても、学習開始直後にそれまでうまく動いていた部分を壊さずに済みます。新人をいきなり前面に立たせず、仕事を覚えた分だけ発言させる、というイメージです。

## 直感的な理解
すでにそこそこ動いている画像中心のネットワークがあるとします。ここに「地図」という別の情報源を後付けしたい。素直にやるなら、画像から得た特徴と地図から得た特徴を足すか、横に並べて(連結して)次の層へ渡します。ところがここに落とし穴があります。

地図を処理する側のネットワーク(地図ブランチ)は、付け足した直後は重みがランダムです。ランダムな重みが吐き出す地図特徴は、現実には意味のないノイズです。そのノイズを、せっかく意味のある画像特徴にいきなり混ぜてしまうと、混合物全体がノイズで汚れます。結果、学習開始直後に損失(loss、予測の悪さを表す数値)が跳ね上がり、それまで学んでいた良い表現が崩れることがあります。これを「学習初期の不安定化」と呼びます。

たとえるなら、長年やってきた熟練チームに、まだ何も分かっていない新人がいきなり大声で口を出し始める状況です。新人の発言は最初はほぼ的外れで、チームの判断をかき乱します。賢いやり方は、最初は新人を黙らせておき、仕事を覚えるにつれて少しずつ発言権を与えることです。

この「最初は黙らせ、徐々に発言させる」を、たった一つの係数 `α` で実現するのがゼロ初期化残差です。`α`(発言の音量つまみ)を 0 から始めます。すると地図特徴は出力にまったく寄与しません。これは「地図ブランチを付けていない状態」と数学的に完全に同じです。そして学習の途中で「地図を効かせた方が損失が下がる」と勾配(gradient、損失を減らす方向を示す微分)が判断したぶんだけ、`α` がじわじわ大きくなり、地図が効き始めます。

## 基礎: 前提となる概念
いくつか言葉をほどいておきます。

「特徴(feature)」とは、ネットワークの途中の層が出力する数値の集まりで、入力(画像や地図)の意味を凝縮した中間表現です。画像なら「ここに車らしきものがある」「この方向に道が伸びている」といった情報が、人間には読めない数百次元のベクトルとして並んでいます。

「BEV(Bird's-Eye View、鳥瞰図)特徴」とは、その特徴を真上から見た平面のマス目(グリッド)上に並べたものです。形は `(チャンネル数 C, 高さ H, 幅 W)` のテンソル(多次元配列)で書けます。マス目の各セルに `C` 次元のベクトルが入っています。自動運転では「自車から見た周囲を上から見下ろした地図状の特徴」がこの形になります。

「融合(fusion)」とは、別々の情報源から来た特徴(ここでは画像由来の `image_bev` と地図由来の `map_bev`)を一つにまとめる処理です。

「残差(residual)」とは「元の値に補正を足し込む」やり方を指します。`y = x + f(x)` の形がそれで、`x` をそのまま通しつつ `f(x)` という補正を加えます。深層学習で広く使われる ResNet(He et al., 2015)の中核アイデアでもあります。残差の利点は、補正 `f(x)` がゼロでも `y = x` が素通しされるので、層を深くしても情報と勾配が消えにくいことです。

「学習可能パラメータ(learnable parameter)」とは、学習(勾配降下)によって値が更新される数値のことです。`α` をただの定数ではなく学習可能パラメータにすると、モデル自身が最適な値を見つけてくれます。

## 仕組みを詳しく
ゼロ初期化残差マップ融合の中身を、典型的な実装の形で書くと次のようになります。

```python
class ResidualMapFusion(nn.Module):
    def __init__(self, embed_dim: int = 256) -> None:
        super().__init__()
        # ゼロ初期化、チャンネルごとに別々の α を持つ
        self.alpha = nn.Parameter(torch.zeros(embed_dim))

    def forward(self, image_bev, map_bev):
        gate = self.alpha.view(1, -1, 1, 1)   # (1, C, 1, 1) に整形
        return image_bev + gate * map_bev
```

要点を順に見ます。

第一に、`alpha` は学習可能パラメータで、形は `(embed_dim,)`、つまりチャンネル数ぶんの要素を持ちます(例として `embed_dim=256`)。スカラー1個ではなく、チャンネルごとに別々の音量つまみを持たせるのが工夫です。これにより「地図のチャンネル A は役に立つから強く効かせる、チャンネル B は不要だから弱いまま」というように、地図情報をチャンネル単位で取捨選択できます。スカラー1個だと地図全体を一律にしか調整できませんが、チャンネルごとなら粒度の細かい制御ができます。

第二に、全要素を 0 に初期化します。だから学習開始時は `gate × map_bev = 0` で、出力は `image_bev` そのものです。これが「地図ブランチを付けていない状態」と完全に一致するという、安定化の核心です。

第三に、`forward` では `alpha` を `(1, C, 1, 1)` の形に整形してから、`(B, C, H, W)`(B はバッチサイズ)の `map_bev` に掛けます。ここで「ブロードキャスト(broadcast)」という仕組みが働きます。形の足りない次元を自動でコピーして揃え、要素ごとに掛け算をするのです。結果、チャンネルごとの音量が、そのチャンネルの全空間位置(全マス)に一律に適用されます。

数値例を出します。`image_bev` のあるセル・あるチャンネルの値が 0.8、`map_bev` の同じ位置が 0.5 だとします。

- 学習開始時(`α=0`): 出力 = 0.8 + 0 × 0.5 = 0.8(地図は完全に無視)。
- 学習が進んだ後(そのチャンネルの `α=0.3` に育った): 出力 = 0.8 + 0.3 × 0.5 = 0.95(地図がわずかに効く)。

勾配の流れも見ておきます。出力が `image_bev + α × map_bev` なので、`α` に関する微分は `∂(出力)/∂α = map_bev` です。重要なのは、`α=0` の地点でもこの微分はゼロではないことです(掛け算の微分は相手側の値になる)。つまり `α=0` から出発しても勾配がちゃんと流れ、`α` が動き出せます。もし `α=0` で勾配まで 0 だったら、`α` は永遠に 0 のまま動かず、地図は一生効きません。掛け算ゲートはこの「初期は無効だが勾配は生きている」という性質を満たします。同時に、地図エンコーダ自身の重みへも `α × (画像側からの勾配)` の形で勾配が伝わるので、`α` が育つほど地図ブランチの学習も加速します。

before / after でまとめます。

```
ナイーブ融合 (image_bev + map_bev):
  学習step 0   ──> ランダム地図特徴が混入 ──> loss 跳ね上がり、表現が崩れる

ゼロ初期化残差 (image_bev + α·map_bev, α=0):
  学習step 0   ──> α=0 なので地図完全無効 ──> 既存表現は無傷
  学習step 中盤 ──> α じわじわ増加     ──> 役立つ地図チャンネルだけ効き始める
  学習step 後半 ──> α 収束             ──> 地図が安定して寄与
```

なぜチャンネルごとなのか、なぜ掛け算ゲートなのか、という設計理由はここまでで説明した通りです。空間方向(マス目方向)には同じ `α` を使う理由は、空間ごとに地図の信頼度を変える必然性が薄く、パラメータと過学習を増やすだけだからです。必要十分な自由度に絞る、という設計判断です。

## 手法の系譜と主要論文
ゼロ初期化(または恒等写像)から新しい枝を足す、という発想は複数の研究で独立に再発見され、共通の設計原則として定着しました。

最初期の重要な仕事は ReZero(Bachlechner et al., "ReZero is All You Need: Fast Convergence at Large Depth", UAI 2021, arXiv:2003.04887)です。各残差ブロックを `x + α·f(x)` と書き、`α` をゼロで初期化します。動機は「学習開始時に各層を恒等写像(入力をそのまま返す写像)にしておけば、勾配が層を素通りして安定に流れ、非常に深いネットワークでも収束が速くなる」というものです。これはまさにゲート係数をゼロから育てるという本トピックの骨格です。

並行して、追加した小さな枝をゼロ(または恒等)から始める設計が、巨大な事前学習モデルへ後付けで能力を足す文脈で広まりました。LoRA(Hu et al., "LoRA: Low-Rank Adaptation of Large Language Models", ICLR 2022, arXiv:2106.09685)は、追加する低ランク行列の片方をゼロ初期化し、適応開始時点で元のモデルと出力が一致するようにします。ControlNet(Zhang et al., "Adding Conditional Control to Text-to-Image Diffusion Models", ICCV 2023, arXiv:2302.05543)は「ゼロ畳み込み(zero convolution)」を導入し、新しい条件入力(エッジ画像や姿勢など)を効かせる制御ブランチをゼロ初期化で接続します。これにより事前学習済みの拡散モデルを保護したまま、条件付き生成の能力を安全に付け足せます。

自動運転やマルチモーダル融合の文脈でも、後から足す地図ブランチやセンサブランチをゼロゲートで接続する設計が使われます。狙いは一貫して「既存の良い表現を壊さず、勾配が必要と判断した分だけ新しい情報源を育てる」ことです。BEVFusion(Liu et al., ICRA 2023, arXiv:2205.13542)のようにカメラと LiDAR を共通 BEV 空間で融合する研究群でも、片方の枝を弱く初期化して学習を安定させる工夫がしばしば併用されます。

これらに共通する設計原則を一文でまとめると、「新しく足す枝はゼロから始めよ。既存表現を破壊せず、勾配が要ると言った分だけ育てよ」となります。

## 論文の実験結果(定量データ)
ゼロ初期化残差の効果は、複数の研究で定量的に示されています。

ReZero の報告では、活性化関数なしの 128 層 Transformer を、ゼロ初期化ゲートにより通常より大幅に少ない反復で収束させられたとされます。同論文は、深さ 12 層の言語モデルで、標準的な学習率ウォームアップ(学習率を最初だけ小さくする慣行)を使わずとも安定に学習でき、収束が速まることを示しました。「ウォームアップが不要になる」という事実が重要なのは、ウォームアップは本来、初期の不安定さを避けるための対症療法だからです。ゼロ初期化はその不安定さを根本から取り除くため、対症療法が要らなくなります。

ControlNet の論文は、ゼロ畳み込みを使うことで、学習の最初のステップで条件ブランチの出力が厳密にゼロになり、元の拡散モデルの生成品質を一切損なわずに学習を始められることを示しました。アブレーション(ある要素を抜いて効果を測る実験)として、ゼロ畳み込みを通常のランダム初期化畳み込みに置き換えると、学習初期に有害なノイズが本体へ流れ込み、生成画像が崩れる「sudden convergence(突然収束)」前の不安定区間が長引くことが報告されています。つまり「ゼロ初期化を抜くと初期が不安定になる」というのが直接のアブレーション結果です。

LoRA の論文は、GPT-3 175B 規模のモデルで、全パラメータを更新する完全ファインチューニングに匹敵する性能を、更新パラメータ数を約 1 万分の 1 に減らして達成したと報告しています。ゼロ初期化により、適応開始時点で元モデルと完全一致する状態から劣化なしに学習を始められることが、この安定性の一因です。

これらの指標が意味するのは次のことです。「収束ステップ数」が減るのは計算コストが下がるということ、「ウォームアップ不要」は学習設定の頑健さが上がるということ、「初期の品質劣化ゼロ」は既存モデルの能力を一切犠牲にせず拡張できるということです。地図融合に置き換えれば、「地図ブランチを足しても初期に性能が落ちず、最終的に地図の恩恵だけを取り込める」という、まさに望ましい挙動に対応します。

## メリット・トレードオフ・限界
メリットは次の通りです。第一に、学習初期に新しいブランチがモデルを壊しません(`α=0` で地図なしと等価)。第二に、チャンネルごとに効き具合を自動調整でき、役立つ情報だけを選択的に取り込めます。第三に、実装が極めて軽量で、追加パラメータは `α` の `embed_dim` 個だけ、計算も掛け算と足し算のみです。第四に、勾配が素直に流れ、収束が安定しやすいです。

トレードオフと限界もあります。最大の限界は、この融合が「空間的に同じ位置どうしを足す」前提に立っていることです。画像 BEV と地図 BEV のマス目が現実の同じ場所を指していないと、足し算は別々の地点の情報を混ぜることになり、幾何的には無意味になります(この論点は [地図BEVと画像BEVの座標整合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0430_map-image-bev-alignment) で詳述)。次に、足し算だけなので、位置をずらして対応づけるような複雑な空間整列はできません。それが必要なら cross-attention(相互注意)のような、より柔軟だが重い融合が要ります。さらに、`α` が育つまで地図情報は効かないため、地図の恩恵が出るまでに学習時間がかかる場合があります。研究上の未解決課題としては、「`α` の育ち方を加速させる正則化や、地図品質が低い領域で `α` を自動的に抑える条件付きゲート」をどう設計するか、といった方向が残されています。

## 発展トピック・研究の最前線
ゲートをスカラーやチャンネルごとの定数にとどめず、入力に応じて動的に決める「条件付きゲート」へ拡張する方向があります。たとえば地図の信頼度(古さ、欠損、位置精度)を推定し、それに応じてゲートを下げる仕組みは、地図品質依存という弱点への直接の対策になります。また、ゼロ初期化の発想は Mixture-of-Experts(専門家混合)のルーティングや、複数の事前学習モデルを安全に合成するモデルマージの研究とも接続します。融合の幾何的妥当性を保証する方向(共通の実座標系に固定した BEV 上でのみ残差を取る)は、次の関連トピックで深掘りします。
