# 潜在世界モデルによる未来予測 (FutureState)

## ひとことで言うと
潜在世界モデル(latent world model)とは、「今のシーンの内部表現」と「これからどう動くつもりか(行動の意図)」を入力に、「未来のシーンの内部表現」を予測するモデルです。明示的なルールを書くのではなく、データから学習して「世界がこれからどう変わるか」を当てる、暗黙の物理シミュレータのようなものです。これは2つの大きな価値を生みます。1つは未来が時間とともに自動的に観測されるので、人間がラベルを付けなくても学習信号が手に入ること。もう1つは「行動Aを取ったら世界はこうなる」という先読み(ロールアウト)を、実車を動かさずに内部で安全に回せることです。

## 直感的な理解
人間が車を運転するとき、頭の中では絶え間なく未来をシミュレーションしています。「いまアクセルを踏めば1秒後にはあの位置に着く」「あの自転車はこのまま進めば自分の進路に入ってくる」といった予測です。この予測能力があるからこそ、まだ起きていない衝突を事前に避けられます。

機械にこの能力を持たせたいというのが世界モデル研究の出発点です。ここで「なぜラベルなしで学習できるのか」が直感的に重要です。教師あり学習では「この画像は猫」という正解札を人間が付けますが、未来予測では「正解」は黙っていても1秒後・数秒後に勝手にやってきます。時刻 t の映像を見せて t+1 秒の世界を予測させ、実際に観測された t+1 秒の映像と突き合わせれば、それだけで学習が成立します。世の中に無尽蔵にある走行ドライブレコーダ映像が、そのまま教材になるわけです。教師なしで信号が自動生成されるこの性質を、自己教師あり学習(self-supervised learning、データ自身から正解を作って学ぶ枠組み)と呼びます。

もう1つの価値が反実仮想(counterfactual、はんじつかそう)の先読みです。「もしブレーキを踏んだら世界はどうなるか」「もし右に避けたらどうなるか」を、実車で試す前に頭の中で複数回シミュレーションし、いちばん安全な行動を選ぶ。これができれば、危険な探索を実車でやらずに済みます。

問題は「未来をどの形式で予測するか」です。素直に未来の画像をピクセル単位で予測すると、空の雲の形や路面のテクスチャといった運転に無関係な細部の予測にモデルの容量を浪費し、しかも本質的に予測不能な揺らぎのせいで出力がぼやけます。そこで近年の多くの手法は、ピクセルではなく特徴空間(潜在空間、内部の圧縮された表現)で未来を予測します。これを潜在世界モデルと呼びます。

## 基礎: 前提となる概念
理解に必要な用語を順に噛み砕きます。

特徴マップ(feature map): ニューラルネットが画像を処理した中間出力です。形状はよく `[B, C, H, W]` と書きます。B はバッチサイズ(同時に処理する場面の数)、C はチャンネル数(各位置を何次元のベクトルで表すか)、H と W は空間方向の縦横グリッド数です。生のピクセルが `[B, 3, 高さ, 幅]`(3 は RGB)なのに対し、特徴マップは「意味のある情報に圧縮された地図」です。

BEV(Bird's Eye View、鳥瞰図): 上空から真下を見下ろした地図的な視点のことです。複数のカメラ映像を1枚の俯瞰図に統合した表現で、「自車の周囲のどこに何があるか」を扱いやすい座標系です。自動運転では計画(どこを走るか)を立てるのに都合が良いので広く使われます。

潜在空間 / 潜在表現(latent space / latent representation): 観測データを低次元の意味あるベクトルに圧縮した空間です。「潜在」は「表面には見えないが背後にある本質」という意味です。

自己回帰ロールアウト(autoregressive rollout): 「予測した未来を入力に戻して、さらにその先を予測する」を繰り返し、長い未来を順に展開することです。1ステップごとの小さな誤差が積み重なって長期予測がずれていく問題(複合誤差、compounding error)が宿命的に付きまといます。

模倣学習(imitation learning): 専門家(人間ドライバー)の運転デモを真似て方策(policy、状況から行動を決める関数)を学ぶ手法です。「この状況ではこのハンドル・アクセル操作をする」を教師データから直接学びます。

表現崩壊(representation collapse): 予測モデルと目標モデルを同時に学習させたとき、両方が「常に同じ定数を出す」という何も予測していない自明解に落ちてしまう現象です。後述する崩壊回避の設計が必要になります。

## 仕組みを詳しく
潜在世界モデルの未来予測モジュール(ここでは慣例に倣い FutureState と呼びます)が典型的に何をするかを、テンソルの形を追いながら見ます。

入力は2つ(オプションで3つ)です。
- 現在のシーン特徴: 形状 `[B, C, H, W]`(例 C=256)。複数カメラを融合した今の BEV 特徴マップです。たとえば `[4, 256, 450, 300]`。
- 行動意図ベクトル: 形状 `[B, C]`(例 `[B, 256]`)。軌道計画器(プランナー)の再帰部品(GRU や LSTM など)が出す「この先どう走るつもりか」を要約した内部状態です。
- (オプション)計画した軌道: 形状 `[B, action_dim]`。後述の行動条件付けに使います。

なぜ行動意図で条件付けるのか。未来は自分の行動次第で分岐するからです。行動の情報を入れない予測は「今こう見えているから次はたぶんこう」という1通りの未来しか出せません。プランナーの隠れ状態は「数秒先まで自車がどう動くつもりか」を凝縮しているので、これを条件として注入することで「自分がこう走るなら世界はこうなる」という行動条件付き予測になります。

処理ステップ(典型的な実装):

1. 行動意図をバイアス(全グリッドに一様に足す加算項)にして現在特徴へ注入する。
```
bias       = Linear(intent).view(B, C, 1, 1)   # [B,C,1,1]
conditioned = scene_features + bias            # [B,C,H,W] にブロードキャスト加算
```
`[B,C,1,1]` を `[B,C,H,W]` の全グリッドへ同じ値で足す(ブロードキャスト)ことで、計画意図がシーン特徴全体に染み込みます。

2. (オプション)計画軌道そのものも第二のバイアスとして足す。
```
action_bias = Linear(trajectory).view(B, C, 1, 1)
conditioned = conditioned + action_bias
```
これにより「軌道Aと軌道Bで未来がどう変わるか」を明示的に比較する反実仮想ロールアウトが可能になります(詳細は [行動条件付き潜在予測 (counterfactual rollout)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0001_action-conditioned-prediction))。

3. 数段の畳み込みで複数時刻の未来特徴を一度に生成する。
```
x      = Conv2d(C, 2C)(conditioned)
x      = GELU(x)
x      = Conv2d(2C, 4C)(x)
future = chunk(x, 4, dim=channel)   # 4枚 [B,C,H,W]
```
出力は等間隔の複数時刻(例: +1.6s, +3.2s, +4.8s, +6.4s)の未来シーン特徴です。ここには2つの設計流派があります。(a) 4ステップを自己回帰的に1枚ずつ展開する設計、(b) チャンネル方向にまとめて同時生成する設計。同時生成は速くて誤差の積み重ねがない反面、ステップ間の因果依存(1.6s の状態が 3.2s を決める、という連鎖)を明示的にモデル化しない、というトレードオフがあります。

学習信号の作り方が肝心です。予測した未来特徴を、凍結した(重みを固定した)ターゲットエンコーダ([凍結ターゲットエンコーダ (Frozen Target Encoder)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0009_frozen-target-encoder))が「実際に観測された未来フレーム」から作った正解特徴と突き合わせ、特徴空間での距離(典型的には MSE、二乗誤差)を最小化します。これが特徴再構成損失(feature reconstruction loss)です。なぜ凍結エンコーダを使うのか。予測側とターゲット側の両方を同時に学習させると、両者が表現崩壊(常に同じ定数を出す自明解)に陥りがちだからです。ターゲット側を固定(または緩やかに更新)することで、予測側が追いかける動かない的を用意し、崩壊を防ぎます(この崩壊回避の系譜は [JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note) と [momentum encoder / EMA target](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0078_momentum-encoder-ema) が詳しい)。

全体像:
```
今のシーン特徴 ──┐
                 ├─[+bias]─► [Conv C→2C→4C] ─► 未来特徴4枚(+1.6/3.2/4.8/6.4s)
計画意図ベクトル ─┘                                          │
                                                             ▼ 特徴空間MSE
実際の未来フレーム ─► [凍結ターゲットエンコーダ] ─► 正解特徴4枚 ──┘
```

数値例で容量感をつかみましょう。BEV 解像度 450×300、出力 4×256=1024 チャンネルを fp32 で持つと、1サンプルあたり `1024 × 450 × 300 × 4 byte ≈ 553 MB`。バッチ数枚で簡単に GPU メモリを食いつぶすため、混合精度(bf16/fp16、数値を半精度で扱いメモリを半減)や空間ダウンサンプリングが事実上必須になります。

## 手法の系譜と主要論文
世界モデルの考え方は強化学習・制御の伝統に根ざしますが、深層学習時代の系譜は次のように整理できます。

World Models(Ha & Schmidhuber, NeurIPS 2018, "Recurrent World Models Facilitate Policy Evolution", arXiv:1809.01999)。世界モデルという言葉を広めた原典です。VAE(変分オートエンコーダ、観測を確率的な潜在ベクトルに圧縮する生成モデル)で観測を潜在ベクトルに圧縮し、MDN-RNN(混合密度出力の再帰ネット)で「行動を入れると次の潜在状態を予測する」遷移モデルを学び、その内部シミュレーション(論文では「夢の中での学習」と表現)で方策を進化させました。動機は「現実より速く・安全に回せる内部シミュレータの中で方策を鍛えれば効率的」。これが「潜在空間で未来を予測する」発想の源流です。

Dreamer 系(Hafner et al., DreamerV1 2020 / V2 2021 / V3 2023)。潜在空間で行動条件付きの未来を予測し、その想像上のロールアウトで強化学習を行う代表系列です。RSSM(Recurrent State Space Model)という決定的成分と確率的成分を併せ持つ潜在遷移モデルが核です。V3 では同一のハイパーパラメータで Atari・物理制御・Minecraft など 150 以上の多様なタスクを解き、Minecraft でダイヤモンド採取を人間データなしで達成したことが話題になりました。教訓は「観測そのものではなく潜在状態で予測すると、サンプル効率が大きく上がる」。弱点は長期ロールアウトの不正確さと潜在表現の崩壊への注意です。

MILE(Hu et al., NeurIPS 2022, Wayve, "Model-Based Imitation Learning for Urban Driving", arXiv:2210.07729)。運転に世界モデルを持ち込んだ重要研究です。BEV 潜在空間で未来状態と運転方策を同時に学び、オフラインの専門家デモから模倣学習します。動機は「未来をモデル化すると、純粋な模倣より少ないデータで頑健な方策が学べる」。CARLA(オープンソースの運転シミュレータ)のオフラインデータのみで学習し、実車を一切使わずに強い閉ループ性能を達成しました。BEV 特徴の未来予測と行動条件付けという構図は、ここで提示されたものに直接連なります。

GAIA-1(Hu, Russell, Yeo et al., Wayve, 2023, arXiv:2309.17080, "GAIA-1: A Generative World Model for Autonomous Driving")。動画・テキスト・行動を入力に未来の運転シーンを生成する大規模(約90億パラメータ)生成型世界モデルです。観測を離散トークン化(画像を有限種類の「視覚の単語」に変換)し、自己回帰 Transformer で次トークンを予測する言語モデル的な定式化を取ります。動機は「多様で稀少な運転シナリオを生成し、世界の挙動そのものを学ぶ」。リアルで多様な未来映像の生成、行動を変えると未来が変わる挙動が示されました。代償は巨大な計算コストと、ピクセル/トークン生成の重さです。

JEPA / I-JEPA(LeCun のビジョン論文 2022 "A Path Towards Autonomous Machine Intelligence"、Assran et al. CVPR 2023)。生成(ピクセル復元)ではなく潜在空間での予測を中心に据える設計思想で、未来予測型世界モデルの理論的支柱になっています(詳細は [JEPA (Joint Embedding Predictive Architecture)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0015_jepa_note))。

これらを一列に並べると、潜在世界モデルの未来予測は「生成型(GAIA-1、ピクセル/トークンを作る)」と「予測型(JEPA/MILE、潜在特徴を当てる)」の2系統に大別でき、軽量さと運転特化を重視する設計は予測型に位置づけられます。

## 論文の実験結果(定量データ)
World Models(2018): CarRacing-v0 という車のレースゲーム環境で、当時としては初めて平均 900 点超(過去手法はおおむね 600-700 点台)を内部シミュレータ内学習だけで達成しました。VizDoom の回避タスクでも、実環境に一切触れず「夢の中」で学んだ方策が転移して高得点を出すことを示し、内部シミュレータ学習の有効性を実証しました。スコアは「どれだけ長く・遠くまで安全に走れたか」を表す指標で、高いほど良いものです。

DreamerV3(2023): 同一設定で 150 以上のタスクを横断し、Atari・DMLab・物理制御などで当時の専用手法を上回るか肩を並べました。とりわけ Minecraft のダイヤモンド採取を人間デモなし・タスク固有チューニングなしで達成した点が、世界モデルの汎用性を象徴する結果として報告されています。アブレーション(構成要素を抜いて効果を測る実験)では、潜在状態予測を観測ピクセル予測に置き換えるとサンプル効率が落ちること、対称対数変換(symlog、報酬や値の桁の幅広さを吸収する正規化)などの技法が安定化に効くことが示されています。

MILE(2022): CARLA の新しい町・新しい天候という未知条件での閉ループ評価で、駆動スコア(route completion と違反ペナルティを組み合わせた総合指標)で先行のモデルフリー模倣手法を上回りました。重要なアブレーションとして、世界モデル(未来予測)成分を外すと汎化性能が顕著に落ちることが報告され、「未来を予測する補助タスクが表現を良くする」という主張を裏付けています。指標の意味としては、route completion はルートをどれだけ完走できたか(0〜100%、高いほど良い)、違反ペナルティは信号無視や衝突をどれだけ犯したか(少ないほど良い)を表します。報告では新規環境でも従来手法より大幅に高い driving score を出したとされています。

GAIA-1(2023): 数値ベンチマークよりも生成品質の定性評価が中心ですが、約90億パラメータへスケールさせるとサンプルの一貫性・多様性・行動への忠実度(指示した行動に沿った未来が生成されるか)が向上することが示されました。短い文脈から数十秒の妥当な運転映像を生成できること、テキスト指示(例: 天候や挙動)で生成を制御できることが報告されています。これは「言語モデル的スケーリング則(モデル・データを大きくするほど性能が伸びる経験則)が運転世界モデルにも当てはまる」可能性を示す結果として注目されました。

## メリット・トレードオフ・限界
メリット
- 未来が自動的に正解になるため、人手ラベルなしで大量の走行映像を学習に使える。
- 特徴空間予測なので、ピクセル予測のぼやけ問題を避け、運転に効く情報へ容量を集中できる。
- 行動意図・軌道で条件付けることで、反実仮想ロールアウト(複数行動の先読み比較)へ自然に発展できる。
- 生成型世界モデル(GAIA-1 等)に比べて圧倒的に軽量で、リアルタイム運転に乗せやすい。

トレードオフ・限界
- 複数時刻を同時生成する設計はステップ間の因果連鎖を明示しない。逆に自己回帰ロールアウトは複合誤差で長期予測がずれる。どちらを取るかは根本的なトレードオフです。
- 高次元出力(例: 4×256=1024 チャンネル)は高解像度 BEV でメモリを大量に消費し、混合精度や空間ダウンサンプリングが事実上必須になります。
- 予測側とターゲット側の整合(出力次元・分布)を取らないと、表現崩壊や勾配の発散が学習を壊します。崩壊回避の設計(凍結/EMA ターゲット、stop-gradient = 目標側に勾配を流さない操作)が必須です。
- 予測が特徴ベクトルなので、生成型と違って結果を人間が画像として目視確認しづらく、デバッグ・解釈性が課題です。
- 未解決の研究課題: 長期一貫性のある潜在ロールアウト、稀少イベント(事故直前など)の予測、不確実性の表現(複数の妥当な未来を確率分布として出す多峰性)、そして予測タスクが本当に運転方策の性能を上げるのかの因果的な検証。「未来予測損失を足したら下流性能が上がった」という相関は数多く報告されていますが、なぜ効くかの理論的説明は途上です。

## 発展トピック・研究の最前線
近年は世界モデルを推論時に「複数の未来候補をサンプルして最良の行動を選ぶ」プランナーとして使う研究(model predictive control 的な利用)や、拡散モデル(diffusion、ノイズから段階的に生成する生成モデル)で多峰的な未来分布を表現する手法、Transformer による長文脈の潜在ロールアウトが活発です。運転分野では、生成型と予測型のハイブリッド、言語条件付き(自然言語で挙動を指示)世界モデル、シミュレータ生成データと実データの併用が焦点になっています。理論面では、潜在空間で予測することが本当に下流の制御性能に効く理由を、表現学習の観点から説明しようとする研究が続いています。さらに最近は、運転に特化した大規模生成世界モデル(Vista、DriveDreamer、GenAD など)が次々と提案され、実走行映像を条件にした長尺・高解像度の未来生成と、それを使ったプランナーの学習・評価が一大潮流になっています。
