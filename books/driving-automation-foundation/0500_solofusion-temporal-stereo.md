# SOLOFusion型 時間ステレオ

## ひとことで言うと

カメラ1台だけでも、自車が動きながら撮った「過去のフレーム」と「今のフレーム」を組み合わせると、左右に並べた2台のカメラ(ステレオカメラ)で撮ったのと同じ効果が得られます。時間ステレオとは、この「時間をまたいだ擬似ステレオ」を使って物体までの距離(深度)を当てる発想です。SOLOFusionはこれを「直前の1〜2フレーム」ではなく「長期間(16フレームなど)」にわたって行い、深度推定を大きく改善しました。深度が正確になると、画像の特徴を鳥瞰図へ正しく貼り付けられ、BEV知覚全体の精度が上がります。「時間こそが深度の最良の手がかりだ(Time Will Tell)」というのが論文の主張です。

## 直感的な理解

人間が両目で距離を測れるのは、左右に2つの目があるからです。指を顔の前に立てて左右の目で交互に見ると、近い指は大きく左右に動いて見え、遠くの壁はほとんど動きません。この「2つの視点で同じ物が見える位置のズレ」を視差(parallax)と呼びます。近い物ほど視差が大きく、遠い物ほど小さい。だから視差から距離を逆算できます。これがステレオ(2眼)による距離測定です。

カメラが1台でも、自車が前に進んでいれば「0.5秒前の位置で撮った画像」と「今の位置で撮った画像」は、別々の場所から撮った2枚になります。つまり時間差を使えば、擬似的に2眼ステレオと同じことができます。さらに嬉しいことに、自車が時速36km(秒速10m)で走っていれば、0.5秒の間に5m進みます。これは左右5m離れた2台のカメラでステレオを組むのと同じで、人間の目(間隔6cm)よりはるかに大きな視差が得られます。視差が大きいほど距離測定は正確になるので、時間差が大きい(=長期の)フレームを使うほど深度が当てやすくなる、というのが「長期」の効く理由です。逆に言えば、それまでの研究が2〜3フレームしか使わなかったのは「安価で強力な長期視差という宝を捨てていた」と論文は指摘します。

## 基礎: 前提となる概念

- 深度推定(depth estimation): 画像の各ピクセルがカメラから何メートル離れているかを当てるタスクです。1枚の写真からの深度推定は本質的に難しく、「小さく写った車」が「本当に小さい車が近くにある」のか「普通の車が遠くにある」のか、1枚では区別がつきません(スケール曖昧性)。奥行きの手がかりが1枚には乏しいためです。

- ステレオと視差(stereo / parallax): 2つの視点から同じ物を見たときの、見える位置のズレが視差です。視差 d と深度 Z は Z = f·B/d の関係にあります(f は焦点距離、B は2視点の間隔=ベースライン)。ベースライン B が大きいほど、同じ深度でも視差 d が大きくなり、深度を細かく区別できます。逆に B が小さいと遠方の視差がほぼゼロになり、距離が当てられません。

- カメラのキャリブレーション(内部・外部パラメータ): 内部パラメータは焦点距離や画像中心(レンズの特性)、外部パラメータはカメラが世界のどこにどの向きで置かれているか(位置・姿勢)です。これらが分かっていると、画像のピクセルと3次元空間の対応を計算できます。

- ego-motion(自車運動): フレーム間で自車がどれだけ動いたか(前進量・横移動量・回転)です。時間ステレオでは、この移動量が2視点のベースラインそのものになります。詳細は [自車運動補償（ego-motion alignment）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0266_ego-motion-compensation)。

- BEV(鳥瞰図): 真上から見下ろした地図のような特徴表現。カメラ画像の特徴を「地面上のどこか」へ深度を使って投影してできます。深度が正確だと投影位置が正確になります。

## 仕組みを詳しく

時間ステレオの中心はコストボリューム(cost volume)という仕組みです。難しそうな名前ですが、中身は単純な「総当たりの照合表」です。段階で分解します。

ステップ1: 深度の候補を何段階か用意する。たとえば「2m, 4m, 6m, ..., 60m」のように深度を D 個の段階(depth bins, 深度ビン)に区切ります。

ステップ2: 過去フレームの特徴を、各深度候補で現在フレームへ投影してみる。「もしこのピクセルが 4m 先にあるなら、過去フレームではこの位置に写っていたはず」という計算を、ego-motion(自車の移動量)とカメラパラメータを使って行います(平面掃引, plane sweep)。これを D 個の深度候補すべてで試します。

ステップ3: 照合スコアを計算する。投影してきた過去フレームの特徴と、現在フレームの特徴がどれだけ似ているかを(内積や相関で)測ります。正しい深度なら過去と現在の見え方が一致して似ているスコアが高く、間違った深度ならズレて似ていない、という性質を使います。

ステップ4: コストボリュームができる。各ピクセル × 各深度候補のスコアを並べた表が出来上がります。tensorの形で書くと `[B, D, H, W]`(Bはバッチ、Dは深度候補数、H×Wは特徴マップの縦横)です。例えば `[4, 88, 16, 44]` のような形です。

ステップ5: 各ピクセルで深度を決める。一番スコアの高い深度を選ぶか、softmaxで深度の期待値 Z = Σ_d softmax(score_d)·z_d を取ります。これで各ピクセルの深度が決まります。

SOLOFusionの核心的な工夫は、このコストボリュームを「短期(short-term)」と「長期(long-term)」の2系統に分けたことです。

- 短期: 直前の数フレームから細かい(高解像度の)コストボリュームを作る。ベースラインは小さいが解像度が高く、動きの速い近くの物に強い。
- 長期: 多くの過去フレーム(報告では最大16フレーム程度)から粗い(低解像度・低コストの)コストボリュームを作る。ベースラインが大きく取れるので、遠くの物の深度に強い。

```
短期: ベースライン小・高解像度  → 近距離に強い、計算は重め
長期: ベースライン大・低解像度  → 遠距離に強い、計算は軽め
両者を組み合わせ → 計算量を抑えつつ近〜遠を広くカバー
```

得られた深度は、画像の特徴を鳥瞰図へ正しく貼り付けるために使われます。深度が正確だと「カメラに写った車が地図上のどこにあるか」を正しく置けるので、BEV投影の精度が上がります。これが「BEV投影精度向上に寄与」の中身です。投影自体は Lift-Splat-Shoot の枠組み(各ピクセルを深度分布つきで3D空間へ持ち上げ、BEVグリッドへ集約)で行われます。

設計の妙は「長期は粗くてよい」と割り切った点です。視差は時間差が大きいほど勝手に大きくなるので、長期側は解像度を落としても深度を区別できます。逆に短期側は視差が小さいぶん高解像度で補う。この役割分担で、全フレームを高解像度で照合する素朴な方式の計算爆発を避けています。長期側の特徴は低解像度のキャッシュとして再帰的に保持できるため、フレーム数を増やしても計算がほぼ一定に保てるのもポイントです。

## 手法の系譜と主要論文

- MVSNet (Yao et al., ECCV 2018, arXiv:1804.02505): コストボリュームで深度を当てる発想の源流。複数視点からの3D復元(Multi-View Stereo)研究の代表で、「深度候補ごとに特徴を投影して照合し、ボリュームを作る」流れを確立しました。SOLOFusionはこの古典的アイデアを「自車の時間方向の動き」に持ち込んだものと言えます。

- Lift-Splat-Shoot (Philion & Fidler, ECCV 2020, arXiv:2008.05711): カメラ画像を深度の確率分布つきで3D空間へ「持ち上げ(lift)」、BEVへ「貼り付ける(splat)」枠組みの原典。カメラのみBEV知覚の基礎で、深度の良し悪しがBEV品質を直接左右する構造を作りました。

- BEVDet (Huang et al., arXiv:2112.11790): Lift-Splat型のBEV検出を体系化した実用系。深度分布を学習するが明示的な深度教師は使わず、深度の不正確さが性能の頭打ち要因になっていました。SOLOFusionが解こうとした問題の出発点です。

- BEVDepth (Li et al., AAAI 2023, arXiv:2206.10092): LiDARから作った深度の正解ラベルで、深度ネットワークを明示的に教師あり学習する手法。深度を正しく当てればBEV知覚が大きく伸びることを示しました。SOLOFusionとの対比点は「明示的な深度ラベルへの依存」です。SOLOFusionは時間視差から自然に深度が出るので、深度ラベルへの依存が小さい。

- STS / BEVStereo (Wang et al., arXiv:2208.10145; Li et al., AAAI 2023, arXiv:2209.10248): 時間ステレオを取り入れた近い時期の手法群。BEVStereoは信頼できる深度候補に動的に絞って効率化します。SOLOFusionと同時期に「時間ステレオで深度を改善する」流れを競い合いました。

- SOLOFusion (Park et al., ICLR 2023, arXiv:2210.02443): 本トピックの中心。タイトルは "Time Will Tell: New Outlooks and A Baseline for Temporal Multi-View 3D Object Detection"。提案の新規性は、短期の高解像度コストボリュームと長期の低解像度コストボリュームを組み合わせる「短期 + 長期の時間ステレオ」。動機は、それ以前の研究が「短い時間窓(2〜3フレーム)」しか使っておらず、長期の視差という安価で強力な手がかりを捨てていたこと。「Time Will Tell(時間が教えてくれる)」というタイトルが、長期の時間こそが深度の鍵だという主張を表しています。

系譜の論点は「深度をどう手に入れるか」です。明示的ラベルで教える(BEVDepth)か、幾何(時間視差)から引き出す(SOLOFusion/BEVStereo)か、という二つの道があります。SOLOFusionの結論は「長期の幾何が、ラベルにかなり迫る、あるいは組み合わせると最良」というものです。

## 論文の実験結果(定量データ)

評価は nuScenes(6カメラ・360度の自動運転ベンチマーク)が中心です。指標を説明します。

- mAP(mean Average Precision): 検出の正確さ。0〜1で大きいほど良い。
- NDS(nuScenes Detection Score): 位置・サイズ・向き・速度・属性の各誤差まで含めた総合スコア。0〜1で大きいほど良い。
- mAVE(mean Average Velocity Error): 速度推定の平均誤差(m/s)。小さいほど良い。
- 深度の絶対誤差(abs. depth error): 各ピクセルの推定深度が正解からどれだけずれているか(m)。小さいほど良い。

報告されている代表的な結果(SOLOFusion論文より、おおよその値):

- SOLOFusionは発表当時、nuScenesのカメラのみ3D検出で最高性能を達成。test setで NDS 0.61前後・mAP 0.54前後と報告され、当時の単一/短期フレーム手法を明確に上回りました。同等バックボーン・解像度で比べた短期のみのベースラインに対して NDS で数ポイントの上積みが示されています。

- 深度推定の改善が顕著。明示的な深度の正解ラベルを使わなくても、長期の時間ステレオだけで深度の絶対誤差が大きく下がることが示されました。これは「ラベルなしで深度が良くなる」点で実用的に重要です。深度が良くなるとBEV投影が正確になり、検出の位置精度がそのまま向上します。

- 速度誤差 mAVE の低下。長期の観測から物体の動きが追えるため、速度推定が安定します。

- アブレーション(どの要素を抜くとどうなるか): 使う過去フレーム数を増やすほど(長期化するほど)深度と検出が改善することが系統的に示されました(報告ではおよそ16フレームまで効果が伸び続ける)。これが論文の主張の核です。一方で、長期側だけ・短期側だけでは性能が落ち、両者の組み合わせが最も良い。長期だけだと近距離・速い動きに弱く(視差が大きすぎて照合が難しい)、短期だけだと遠距離の視差不足で深度が甘くなるためです。さらに、明示的な深度教師(LiDAR由来)を併用するとさらに上がり、幾何と教師は相補的だと示されています。

注意点として、これらの数値は入力解像度・バックボーン・学習設定で変動し、論文間の比較は設定を揃えて見る必要があります。

## メリット・トレードオフ・限界

メリット:

- 深度の正解ラベルが無くても深度推定を改善できる(幾何から引き出す自己教師的な性質)。
- 長期の大きな視差を使うため、遠くの物体の距離も比較的正確になる。
- BEV投影の精度が上がり、下流のBEV知覚・予測・計画が安定する。
- BEVクエリベースの時間融合とは別の層(深度の質)を改善するので、それらと直交的に効く(組み合わせ可能)。

トレードオフ・限界:

- コストボリュームの生成・照合が計算的に重い(特に短期の高解像度系)。長期を粗くする工夫はその緩和策です。
- ego-motion(自車の正確な移動量)が必須。移動量がずれると過去フレームの投影もずれ、照合が崩れます。自車が停止している(ベースライン=0)と時間ステレオが原理的に効かないという弱点もあります。
- 動的物体(動いている車など)が苦手。「自車の動きだけで投影」という前提が崩れ、動く物体の表面は深度候補のどこでも一致しにくくなります。純粋な時間ステレオの根本的な弱点です。
- テクスチャの乏しい面(白い壁、空、ガラス、夜間)では照合スコアに差が出ず、深度が不定になります(ステレオ全般の弱点)。

研究上の未解決課題としては、(1)動的物体の流れ(オプティカルフロー)を取り込んで時間ステレオの「静的前提」を緩和すること、(2)時間視差(自己教師)と明示的深度ラベル(教師あり)の最適な併用、(3)コストボリュームの計算コストをさらに下げる効率的な照合、(4)自車停止時にも頑健な深度の確保などが挙げられます。

## 発展トピック・研究の最前線

時間ステレオは「幾何から深度を引き出す」立場の代表で、近年は単眼深度の基盤モデル(Depth Anything など大規模学習した深度推定器)と組み合わせる方向や、深度を陽に推定せず暗黙的にBEVへ投影する手法(Transformer のクロスアテンションで投影する BEVFormer 系)との比較が進んでいます。BEVクエリベースの時間融合([BEVFormer型 再帰的BEV融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0481_bevformer-recurrent-fusion) など)が「BEV特徴の時間的一貫性」を扱うのに対し、時間ステレオは「投影前の深度の質」を扱うので、両者は補完的です。長期の観測を活かすという問題意識は、再帰で長期文脈を保つ手法([VideoBEV型 長期再帰融合](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0511_videobev-longterm-fusion))とも共通し、SOLOFusionは「並列で長期」の代表格として対比され続けています。さらに、占有グリッド(occupancy)予測でも時間ステレオ由来の深度が幾何的な土台として再利用されつつあります。
