# 時間重み付き損失 (Temporal Decay Weighting)

## ひとことで言うと
未来の系列を予測するとき、「すぐ先の予測」を「ずっと先の予測」より重く扱う仕組みです。時間が進むほど損失への重みを少しずつ減らしていく(時間減衰、temporal decay)ことで、安全に直結する直近の予測精度を優先します。軌跡予測・模倣学習・系列予測で広く使われる、軽量で効果的なテクニックです。

## 直感的な理解
車を運転しているとき、「次の 0.5 秒で何をするか」と「5 秒後に何をしているか」は重要度が全く違います。次の 0.5 秒はハンドルとブレーキの操作に直結し、ここを外せば即座に事故です。一方 5 秒後はあくまで大まかな計画で、状況が変われば直前で何度でも修正できます。だから損失も「手前の誤差は重く、先の誤差は軽く」採点したい。これが時間重み付き損失の発想です。

もし全ステップの誤差を平等に平均してしまうと、困ったことが起きます。遠い未来ほど予測は本質的に難しく、誤差が大きくなりがちです。平等平均だと、当てるのが難しい遠未来の大きな誤差が損失の主役になり、モデルは「遠い未来をなんとか当てよう」とリソースを浪費して、肝心の手前がおろそかになります。重みで優先順位を付けることで、モデルは自然に「まず手前を正確に」学習します。これは強化学習で未来の報酬を割り引く「割引率」と同じ考え方で、遠い未来ほど価値(ここでは精度の重要度)を下げているのです。

## 基礎: 前提となる概念

軌跡 (trajectory) とは「これから物体がどう動くか」を時系列で表したものです。自動運転モデルは未来を一定時間刻みの複数ステップ(たとえば数十ステップ、各 0.1 秒など)に分け、各時刻での位置や進み具合(典型的には縦方向・横方向の 2 つの座標)を予測します。

ここで損失をどう集約するかが問題になります。各ステップの誤差は、回帰損失(L2、L1、Smooth-L1 など。[MSE / L2損失 (平均二乗誤差)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0565_mse-l2-loss) / [Smooth-L1 (Huber) 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0567_smooth-l1-huber-loss))で測れますが、それらをどう「足し合わせて 1 つの数値にするか」で挙動が変わります。

- 平等平均: 全ステップの誤差を同じ重みで平均する。実装は最も単純だが、遠未来の大きな誤差に引っ張られる。
- 時間重み付き: ステップごとに重みを変えてから平均する。近ステップに大きな重み、遠ステップに小さな重みを置く。

時間減衰の代表的な作り方は「指数減衰 (exponential decay)」です。後で詳しく述べますが、減衰率を 1 未満の定数にして、ステップが進むごとに重みを掛け算で減らしていきます。なお、近未来を重視する理由は安全だけでなく、教師データ(お手本の運転)自体が遠未来ほどブレやすく、そこを重く罰するとノイズを学習してしまうという実務的な側面もあります。物理的にも、現在の状態(速度・舵角)が決まれば直後の数百ミリ秒の挙動はほぼ一意ですが、数秒先は運転手の意図次第で何通りにも分岐するため、本質的に「遠未来は当てにくい・当てる価値も相対的に低い」のです。

## 仕組みを詳しく

指数減衰では、減衰率を d(0 < d ≤ 1、たとえば 0.95)、時刻インデックスを t = 0, 1, 2, … とすると、ステップ t の重みは次のとおりです。

```
weight(t) = d ^ t        (d の t 乗)
```

d = 0.95 での重みの並び(冒頭部分):

- t=0:  0.95^0 = 1.000(最も近い未来。最大の重み)
- t=1:  0.95^1 = 0.950
- t=2:  0.95^2 = 0.9025
- t=5:  0.95^5 ≈ 0.774
- t=10: 0.95^10 ≈ 0.599
- t=20: 0.95^20 ≈ 0.358
- t=63: 0.95^63 ≈ 0.039(遠い未来。重みは約 4% まで小さくなる)

遠ざかるほど重みがなだらかに減ります。d = 1.0 を指定すれば全ステップ重み 1(平等)になり、特別な切り替えなしに従来の平等平均へ戻せます。減衰の速さの直感として「重みが半分になるステップ数(半減期)」は log(0.5)/log(d) で求まり、d=0.95 なら約 13.5 ステップ、d=0.9 なら約 6.6 ステップです。減衰率を 0.95 から 0.9 に変えるだけで重みの効き方が倍以上シャープになることが分かります。

実装の典型的な流れは次のとおりです。

1. 初期化時に重みベクトルを作り、学習対象ではない固定値として保存する(深層学習フレームワークでは、デバイス移動に追従させるためバッファとして登録するのが一般的)。

```python
if temporal_decay == 1.0:
    weights = torch.ones(num_timesteps)
else:
    t = torch.arange(num_timesteps, dtype=torch.float32)
    weights = temporal_decay ** t
```

2. forward で、予測と正解を `[B, T, 2]` に整形し、要素ごとの回帰損失(Smooth-L1 か MSE)を計算する。
3. 最後の次元(縦横 2 方向)で平均し、ステップごとの損失 `[B, T]` にする。
4. その `[B, T]` に時間重み `[T]` を掛けてから、全体を平均する。

```python
per_element_loss = self.loss_fn(pred, target)              # [B, T, 2]
per_timestep_loss = per_element_loss.mean(dim=2)            # [B, T]
weighted_loss = per_timestep_loss * weights.unsqueeze(0)    # [B,T] * [1,T]
return weighted_loss.mean()
```

`unsqueeze(0)` は重み `[T]` を `[1, T]` にして、バッチ方向 `[B, T]` とブロードキャスト(自動的に各サンプルへ同じ重みを掛ける)できるようにする操作です。

数値例で効果を見ます。t=0 の誤差が 0.2、t=63 の誤差が 1.0 だったとすると、重み付け後の寄与は t=0 が 0.2 × 1.0 = 0.2、t=63 が 1.0 × 0.039 ≈ 0.039 です。平等平均なら遠未来の 1.0 が主役でしたが、重み付けで近未来の方が損失に効くようになり、モデルは近未来を優先して直すようになります。

設計上の注意が一点あります。重みを掛けると損失の絶対スケールが変わります。指数減衰だと重みの総和が 1 + d + d² + … < ステップ数 となり(d=0.95・64 ステップなら総和は約 18.6 で、平等なら 64)、平等平均より損失値が小さく出ます。複数損失を足し合わせる場面では、重みベクトルを「総和がステップ数に一致する」よう正規化(weights = weights * T / weights.sum())してから掛けることで、平等平均と同じスケールを保つ実装もあります。指数減衰そのものとは別の流儀ですが、「未来ステップごとに重みを変える」という発想は共通です。

重みの作り方は指数減衰以外にもあり、線形減衰(t に比例して減らす)、しきい値より先を完全に無視する打ち切り(遠未来を loss に含めない)、不確実性の推定値に応じて自動で重みを決める方式などが使われます。指数減衰は「減衰率 1 つで挙動を直感的に調整でき、計算が軽い」点で広く採用されています。

## 手法の系譜と主要論文

時間減衰の重み付けは、単独の提案論文があるというより、複数分野で独立に育った共通アイデアです。

強化学習では、未来の報酬を割引率 γ(ガンマ、典型的に 0.95〜0.99)で指数的に割り引く割引報酬 (discounted return) が基礎概念です(Sutton & Barto "Reinforcement Learning: An Introduction")。割引を入れる理由は、無限先まで等価に扱うと和が発散すること、遠未来の不確実性を反映すること、計算を有限に保つことです。時間重み付き損失の指数減衰は、この割引率と数学的に同じ形(γ^t)であり、「遠い未来ほど価値を下げる」という哲学を損失設計に持ち込んだものと言えます。トレードオフも共通で、割引(減衰)を強くしすぎると近視眼的になり、長期計画を軽視します。

軌跡予測・運転計画では、Chai et al. (2019) "MultiPath"(CoRL 2019、arXiv:1910.05449)や Salzmann et al. (2020) "Trajectron++"(ECCV 2020、arXiv:2001.03093)のように、将来を多ステップ予測する研究が「遠未来ほど不確実性が高い」ことを前提に、ステップごとの寄与を調整します。MultiPath はあらかじめ用意したアンカー軌跡ごとにステップ単位のガウス分布を予測し、Trajectron++ は将来の各時刻の分布を出すため、いずれも「遠未来の誤差を確率的に薄める」効果を内包しています。明示的に「指数減衰」と書かなくても、未来ステップの重み付けや不確実性に応じた重み付けは標準的に行われています。

不確実性に応じた重み付けの理論的な定式化は、Kendall & Gal (2017) "What Uncertainties Do We Need in Bayesian Deep Learning?"(NeurIPS 2017、arXiv:1703.04977)に代表されます。彼らは各予測の分散 σ² も同時に出させ、損失を「誤差² / σ² + log σ²」とすることで、モデルが自信のない予測(遠未来など)の重みを自動で下げる仕組みを示しました。固定の指数減衰を「データから学ぶ」一般化と見ることができます。

模倣学習 (imitation learning、お手本の行動を真似て方策を学ぶ枠組み)では、誤差の累積 (compounding error) が中心的な課題です。Ross et al. (2011) "DAgger"(AISTATS 2011、arXiv:1011.0686)は、模倣学習が時間 horizon T に対して誤差を最悪 O(T²) で累積させることを理論的に示しました。これは「序盤で小さく外すと、その後の状態分布がずれて誤差が雪だるま式に増える」ことを意味します。直近の予測を重視する設計は、この雪だるまの起点を抑える発想と整合します。手前を正確にすればそこから先の予測も土台が安定する、というわけです。DAgger 自体は重み付けではなくデータ収集の工夫(専門家に逐次問い合わせて訓練データを増やす)で対処しますが、時間重み付き損失は同じ問題に「損失設計」で挑むアプローチと位置づけられます。

## 論文の実験結果(定量データ)

軌跡予測の標準指標は ADE(Average Displacement Error、全予測ステップでの予測位置と正解位置の平均距離。単位メートル、小さいほど良い)と FDE(Final Displacement Error、最終ステップでの距離。小さいほど良い)です。ADE は経路全体の当たり具合、FDE は終点の当たり具合を測り、両者を併記するのが慣例です。

MultiPath は、自社の大規模運転データセットと公開の nuScenes 上で、単一モード予測のベースラインに対して複数仮説を出す枠組みで ADE/FDE を改善したと報告しています。Trajectron++ は ETH/UCY の歩行者データセットで、当時の最先端に対して FDE を二桁パーセント改善したと報告しており、いずれも「将来ステップを一律ではなく確率的・重み的に扱う」設計の有効性を示しています。ただし、これらは時間減衰そのもののアブレーションというより、不確実性モデリング全体の効果である点に注意が必要です。

時間減衰そのものの定量的直感は、強化学習の割引率の研究から得られます。割引率 γ を大きくすると理論上の最適性は上がる一方、価値推定の分散(ブートストラップの累積)が増えて学習が不安定になり、γ を小さくすると安定するが近視眼的になる、というトレードオフが多くの実験で繰り返し確認されています。時間重み付き損失でも同様に、減衰率 d を強くする(値を小さくする)ほど近未来は鋭く当たるが遠未来の経路が雑になる、という関係が経験的に観察されます。実務では d を 0.9〜0.99 の範囲で振って ADE(全体精度)と直近ステップの誤差のバランスを見て決めるのが定石です。

アブレーション的に言えば、d=1.0(平等)に戻すと直近精度が悪化し、特に急ハンドル・急ブレーキのような直近で大きく動くシーンで遠未来のなだらかな誤差に埋もれてしまう、という挙動が起こりやすい点が、この手法を入れる動機になります。逆に d を 0.8 程度まで下げすぎると、半減期が約 3 ステップとなり遠未来がほぼ無視され、FDE(終点誤差)が悪化して長距離の計画が崩れます。

## メリット・トレードオフ・限界

メリット
- 安全上重要な近未来の予測精度を、損失の設計レベルで明示的に優先できる。
- 遠未来のノイズ・不確実性・教師データのブレに学習が振り回されにくくなる。
- 実装が軽量(重みベクトルを掛けるだけ)で、減衰率 1 つで挙動を直感的に調整できる。模倣学習の誤差累積を抑える方向に働く。

トレードオフ・限界
- 減衰を強くしすぎると遠未来の予測が雑になり、長距離の計画品質(FDE)が落ちる(近視眼化)。
- 減衰率 d は人間が決めるハイパーパラメータで、データやタスク・予測 horizon に応じた調整が要る。
- 重みで損失スケールが変わるため、他の損失と足し合わせる際の重み調整に影響する。正規化を入れないとマルチタスクの相対重みが意図せずずれる。
- 未解決の課題として、固定の減衰率はシーンの難易度や状況(高速道路の直進と交差点の右折では最適な重み配分が違う)に追従できない。状況依存・不確実性依存で重みを動的に決める方が望ましいが、設計と学習が難しく、自信過剰な分散推定(σ² を大きく出して誤差を見かけ上小さくする抜け道)を防ぐ工夫も要る。

## 発展トピック・研究の最前線

固定の指数減衰の次の一手は、重みを「学習・推定」させる方向です。具体的には、(1) 各ステップの予測不確実性(分散)をモデル自身に出させ、その逆数を重みにする不確実性重み付け(Kendall & Gal NeurIPS 2017 の流れ。homoscedastic は全体一律、heteroscedastic は入力ごとに分散を変える)、(2) 複数の将来仮説を出し、最も近い 1 本だけを罰する minimum-over-K 損失と時間重みの併用、(3) 拡散・フローマッチングで未来軌跡を生成する近年の planner では、時刻ごとのノイズスケジュールが実質的に時間重みの役割を担う、といった発展があります。時間重み付き損失は、こうした高度な系列学習の「最も素朴で確実な第一歩」であり、まずこの仕組みと割引率との等価性を理解しておくことが、不確実性モデリングや生成型 planner を読み解く土台になります。
