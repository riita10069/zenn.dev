# 時系列重み付けロスとゼロ和ガード

## ひとことで言うと
未来を複数ステップ先(例 +1.6s, +3.2s, +4.8s, +6.4s)まで予測するとき、「近い未来の予測ミス」と「遠い未来の予測ミス」を同じ重さで扱うか、重み付けして扱うかを選べる仕組みです。重みを使うと損失全体の大きさ(スケール)が変わって学習率の設定が狂うので、それを自動で正規化します。さらに、重みの合計がゼロになると割り算で NaN(計算が壊れた状態)が出るため、それを未然に防ぐ安全装置(ゼロ和ガード)を備えます。地味ですが、設定ミスで学習が静かに壊れるのを防ぐ実務的に重要な工夫です。

## 直感的な理解
世界モデルが未来 4 ステップの特徴マップを予測すると、ステップごとに損失(予測と正解のずれ)が出ます。素朴には 4 つを単純平均すればよいのですが、現場では「ステップごとに重みを変えたい」場面がよくあります。

例えば近い未来は当てやすく、遠い未来は不確実です。遠い未来のミスを強く罰すると学習が不安定になることがあるので、近い未来を重く、遠い未来を軽くしたい。逆に、安全のためにあえて遠い未来を重視したいこともあります。あるいはカリキュラム学習(易しいところから順に学ばせる)として、最初は近い未来だけ重くし、徐々に遠い未来も重くしたい。

ここで素直に重みを掛けると、二つの落とし穴があります。一つ目は、重みの合計が変わると損失全体の大きさが変わってしまうこと。重みが全部 1 なら平均はそのままですが、[2,2,2,2] にすると損失が 2 倍になり、学習率(重みをどれだけ更新するかの設定)が実質的に狂います。複数の損失(模倣損失、未来予測損失など)を足し合わせる構成だと、片方のスケールが暴れるともう片方を押しのけて学習を支配してしまいます。二つ目は、利用者がうっかり重みの合計をゼロにしてしまうと([1,-1,1,-1] や [0,0,0,0] など)、正規化のための割り算でゼロ除算が起き、損失が NaN になって学習が即座に壊れること。

この二つを同時に解決するのが「時系列重み付け + スケール正規化 + ゼロ和ガード」です。

## 基礎: 前提となる概念
- 損失(loss): モデルの予測と正解のずれを 1 つの数値にしたもの。学習はこの数値を小さくする方向にパラメータを動かします。

- MSE(Mean Squared Error、平均二乗誤差): 予測 pred と正解 target の差を二乗して平均したもの。`mean((pred - target)^2)`。ずれが大きいほど大きく罰する、回帰の定番損失です。

- 学習率(learning rate): パラメータを 1 回の更新でどれだけ動かすかの大きさ。損失のスケールが変わると、同じ学習率でも実質の更新量が変わってしまうため、損失の大きさは安定させたいのです。

- NaN(Not a Number、非数): ゼロ除算や 0/0、無限大同士の演算などで生まれる「数でない値」。一度 NaN がパラメータに混入すると、以降の計算がすべて NaN に汚染され、学習が完全に壊れます。

- ブロードキャスト / 要素ごとの積: ステップごとの損失ベクトル `[L1,L2,L3,L4]` に重みベクトル `[w1,w2,w3,w4]` を要素ごとに掛けると `[w1·L1, ...]` になります。

- register_buffer: 学習で更新しないが、モデルと一緒に保存・GPU 移動される定数テンソルを登録する仕組み(PyTorch)。重みは学習対象ではなく固定の係数なので buffer として持つのが自然です。

- カリキュラム学習(curriculum learning): 易しい課題から難しい課題へ段階的に学ばせる手法。「最初は近い未来だけ」という重み調整はこれにあたります。

## 仕組みを詳しく
処理は「初期化時(重みの準備)」と「forward 時(損失計算)」の二段階です。以下の擬似コードは分野で典型的な実装パターンであり、特定のコードベースに固有のものではありません。

初期化時(スケール正規化とガード):

```python
if temporal_weights is None:
    weights = torch.ones(num_future_steps)          # 既定は全部1
else:
    weights = torch.as_tensor(temporal_weights, dtype=torch.float32)
    # 要素数チェック: num_future_steps と一致しないとエラー

total = weights.sum()
if torch.abs(total) > 1e-8:                          # ★ゼロ和ガード
    weights = weights / total * num_future_steps     # スケール正規化
self.register_buffer("temporal_weights", weights)
```

ポイントを分解します。

- スケール正規化 `weights / total * num_future_steps`。重みの合計を `num_future_steps`(ここでは 4)に合わせ直す操作です。これにより、どんな重みを入れても「重み付き平均が、ただの MSE(全部 1 のとき)と同じスケールに保たれる」ようになります。具体例として重み [4,3,2,1] を入れると合計 10、正規化後は [1.6,1.2,0.8,0.4](合計 4)になり、近い未来を重視しつつ全体の大きさは普通の MSE と同程度に保たれます。これで重みを変えても学習率を再調整せずに済みます。

- ゼロ和ガード `if torch.abs(total) > 1e-8`。重みの合計の絶対値が極めて小さい(実質ゼロ)場合は正規化(割り算)をスキップし、重みを入力されたまま使います。これでゼロ除算による NaN を防ぎます。`1e-8` という小さな閾値を使うのは、浮動小数点の丸め誤差で合計が完全な 0.0 でなくても「ほぼ 0」を捕まえるためです。完全一致 `total == 0.0` で判定すると、`1e-12` のような微小な残差をゼロと見なせず割り算して巨大値を生む危険があるため、閾値比較にします。

forward 側(損失計算):

```python
per_step_losses = []
for pred, target in zip(predicted_features, target_features):
    # shape mismatch なら ValueError
    per_step_losses.append(torch.mean((pred - target) ** 2))   # 各ステップMSE
per_step = torch.stack(per_step_losses)        # [num_future_steps]
weighted = per_step * self.temporal_weights    # 重み付け
return weighted.mean()                         # mean reduction
```

`reduction="none"` を指定すると `weighted`(各ステップの重み付き損失、形状 [4])をそのまま返し、ログ記録に使えます。各ステップが今どれくらいズレているかを個別に観察できる、デバッグ向けの出口です。「+6.4s だけ損失が高止まりしている」のように、遠い未来の予測が苦手だと一目で分かります。

数値例。各ステップの MSE が [0.10, 0.20, 0.30, 0.40]、重みが [4,3,2,1]→正規化後 [1.6,1.2,0.8,0.4] のとき、weighted = [0.16, 0.24, 0.24, 0.16]、その平均 = 0.20。重みが全部 1 なら平均 = 0.25。近い未来を重視すると、近い未来のズレが相対的に効くようになります。スケールが 0.20 と 0.25 で大きく跳ねていないことに注目してください。これが正規化の効果です。

なぜ「合計を num_future_steps に合わせる」正規化なのか。仮に「合計を 1 に正規化する」(=重み付き平均)設計だと、損失の絶対値がステップ数によって変わり、ステップ数を増減するたびに学習率を見直す必要が出ます。合計を num_future_steps にすると、全ステップ重み 1 の素朴な平均(各ステップ損失をそのまま平均)とスケールが一致するため、重みを導入する前後で学習率を据え置けます。これは「新機能を入れても既存の学習設定を壊さない」後方互換性のための設計です。

## 手法の系譜と主要論文
時系列の重み付けや正規化ガードそのものを主題にした論文は多くありませんが、背後の考え方は確立されたものです。

- 割引(discounting)による未来の重み付け。強化学習では、未来の報酬に割引率 γ(0 < γ < 1)を掛けて遠い未来を軽くするのが基本です(Sutton & Barto, "Reinforcement Learning: An Introduction", 2nd ed. 2018)。t ステップ先の報酬を γ^t で割り引く形で、「遠い未来は不確実だから軽く見る」を数式化します。本トピックの「近い未来を重く」はこれと同じ動機です。γ=0.99 なら 100 ステップ先で約 0.37 倍、というように指数的に減衰します。

- マルチタスク損失のスケール調整。Kendall et al.(Cambridge, CVPR 2018, "Multi-Task Learning Using Uncertainty to Weigh Losses", arXiv:1705.07115)は、複数の損失を足すとき各損失のスケールが揃っていないと一部のタスクが学習を支配することを示し、各損失の不確実性 σ に基づいて重みを自動学習する手法(`1/(2σ^2)·L + log σ`)を提案しました。深度推定・セマンティック分割・インスタンス分割を同時学習する実験で、手で重みを探した最良の組み合わせと同等以上の性能を、重み探索なしで達成しました。本トピックのスケール正規化(合計を num_future_steps に合わせる)は、これを単純化した固定版の発想です。

- 勾配ベースのマルチタスク調整。GradNorm(Chen et al., ICML 2018, arXiv:1711.02257)は各タスクの勾配の大きさを揃えるように重みを動的調整し、PCGrad(Yu et al., NeurIPS 2020, arXiv:2001.06782)は勾配の向きが衝突する(内積が負)ときに片方を射影して干渉を除きます。時系列ステップ間でも、近い未来と遠い未来の勾配が衝突しうるため、これらの応用が考えられます。

- NaN 防止の数値安定化。深層学習フレームワークでは「分母に ε を足す」「ゼロ割を条件分岐で避ける」のが定石です。LayerNorm や BatchNorm(Ioffe & Szegedy, ICML 2015, arXiv:1502.03167)の分母に入る eps(典型的に 1e-5 〜 1e-8)も同じ系譜の防御で、分散がゼロに近いときの発散を防ぎます。本トピックの `1e-8` 閾値ガードもこの実装上のベストプラクティスです。

- 複数予測対象をもつ世界モデル。V-JEPA(Bardes et al., Meta AI, 2024, arXiv:2404.08471)のように複数の時空間ターゲットを予測する手法では、対象ごとの重み配分や損失のスケール揃えが学習安定性に効きます。未来ステップごとの重み付けはこの一般的な課題の一例です。

## 論文の実験結果(定量データ)
- Kendall et al.(2018)。CityScapes(都市シーンの認識)での 3 タスク同時学習で、単一タスクをそれぞれ別々に学習した場合より、不確実性で自動重み付けしたマルチタスク学習のほうがセマンティック分割の精度(mean IoU、領域の重なり率で高いほど良い)などで改善しました。手で重みを総当たりで探した最良値とほぼ同等の性能に、探索なしで到達できた点が実用的に重要です。これは「損失のスケールを揃えることが、性能に直結する」ことの定量的な証拠です。

- GradNorm(2018)。複数の回帰タスクで、固定の等重みや手調整より、勾配の大きさを揃える動的重み付けのほうが各タスクの誤差を下げられることを示しました。重みを「結果を見て」動かす適応的手法の有効性を裏づけます。

- 強化学習の割引。割引率 γ を変えると学習の安定性と最終性能が大きく変わることは、Atari や連続制御の多数の実験で繰り返し示されています。一般に γ を大きくする(遠い未来を重視する)ほど理論上は最適だが分散が増えて学習が不安定になり、小さくすると安定だが近視眼的になる、というトレードオフが定量的に観察されます。本トピックの時系列重みも、近い未来を重くするほど安定だが遠い未来の予測が甘くなる、という同型のトレードオフを持ちます。

時系列重み付けロスとゼロ和ガードそのものを単独で評価した公開ベンチマークは多くありませんが、効果は二つの軸で測れます。(a) 学習安定性: ゼロ和ガードを外すと、合計ゼロの重み設定で即座に NaN が出て学習が停止することが再現できます(アブレーションとして明快)。(b) スケール不変性: 正規化を入れると、重みを [1,1,1,1] と [4,3,2,1] で切り替えても損失の絶対値が大きく跳ねず、同じ学習率で学習が続けられます。これらは設定ミスや重み変更に対する頑健性として、長い学習を回す上で実務的な価値があります。

## メリット・トレードオフ・限界
メリット
- 近い未来/遠い未来の重要度を自由に調整でき、カリキュラム学習や安全重視のチューニングに使えます。
- スケール正規化により、重みを変えても損失の大きさが暴れず、学習率を再調整せずに済みます。
- ゼロ和ガードで、設定ミス由来の NaN クラッシュを防げます(学習が静かに壊れない)。
- 既定を「全ステップ均等」にしておけば、何も指定しない利用者には従来通りの単純 MSE として働き、後方互換です。

トレードオフと限界
- 重みはハイパーパラメータ(人が決める値)であり、最適値探しに手間がかかります。Kendall 流の自動学習ではないので、本当に良い配分かどうかは試行錯誤に依ります。
- ゼロ和ガードは「合計ゼロ」のとき正規化をスキップするだけで、その重み設定が意味のあるものに自動で直るわけではありません。利用者は警告に頼らず重みを正しく設計する必要があります。
- スケールを num_future_steps に固定する設計は単純で堅い反面、損失間の真の重要度を反映する適応的な重み付けではありません。
- 負の重みを許す設計は柔軟ですが、特定ステップの損失を「下げる」方向に働き、意図しない最適化を招く危険があります。実用上は非負の重みに限るのが安全です。
- 未解決課題として、(a) 重みを学習で自動調整する(Kendall 流や勾配整合に基づく方法)を運転の時系列予測に持ち込むべきか、(b) 重みをカリキュラムとして学習中に動的に変えるスケジュールの設計、が挙げられます。

## 発展トピック・研究の最前線
- 不確実性ベースの自動重み付け(Kendall 流)を時系列予測に適用し、各ステップの予測の不確実性に応じて重みを学習させる方向。
- 勾配ベースのマルチタスク調整(GradNorm、PCGrad など)。複数損失の勾配の大きさや向きの衝突を均す手法で、時系列ステップ間の不均衡にも応用できます。
- カリキュラムスケジューリング。学習の進行に応じて遠い未来の重みを徐々に上げる動的な重み設計。
- 確率的予測との統合。予測を分布として出し、その不確実性(分散)で自然に重み付けする方向(遠い未来は分散が大きく自動的に軽くなる)。
