# 時系列対照学習 (driving/ego-motion)

## ひとことで言うと
動画は「時間的に近いフレームは似ていて、遠いフレームは違う」という構造を持っています。時系列対照学習は、この「近いフレーム同士は正例(似ているべきもの)、遠いフレームは負例(似ていないべきもの)」という関係を利用して、ラベルなしで映像から表現を学ぶ方法です。運転では、連続する映像フレームや自己運動(ego-motion、自車がどう動いたか)を手がかりに、運転シーンの表現を鍛えます。「時間そのものを教師信号にする」という発想が核心です。

## 直感的な理解
運転データは膨大に集まりますが、その大半に人間のラベル(「ここに歩行者」「右折すべき」など)は付いていません。ラベル付けは高コストなので、ラベルなしデータから有用な表現を学ぶ「自己教師あり学習(SSL)」が重要です。

ここで「時間そのものが教師信号になる」という発想が効きます。動画には、人間がタグを付けなくても元から備わった構造があります。

- 時間的に近い 2 フレーム(例えば 0.1 秒差)は、同じ場面のわずかな変化なので「似ているべき」=正例。
- 時間的に大きく離れたフレーム(例えば 30 秒後、別の道路)は「違う場面」=負例。

この「近い=正、遠い=負」を満たすように表現を配置すれば、人間のラベルなしに、シーンの連続性や動きを捉えた表現が学べます。運転の本質は「時間」にあります。前の車が近づいてくる、信号が変わる、自車が交差点に進入する——これらはすべて時間方向の変化で、静止画 1 枚では捉えられない「動き」「因果」「文脈」を学ぶには、時間を明示的に使う学習が必要です。

## 基礎: 前提となる概念
用語を噛み砕きます。

「対照学習(contrastive learning)」とは、正例ペアのベクトルを近づけ、負例とのベクトルを遠ざける学習です。「これとこれは組」「これとあれは別物」という関係だけを教え、表現空間を組み立てます。ラベルそのものではなく「関係」だけで学べるのが利点です。

「InfoNCE 損失」は対照学習で標準的な損失です。アンカー(基準)の埋め込み q、正例の埋め込み k+、負例たち k1..kn があるとき、

L = − log( exp(sim(q, k+)/τ) / Σ_j exp(sim(q, k_j)/τ) )

と書きます。sim はコサイン類似度(2 ベクトルの向きの揃い具合を −1〜1 で測る指標)、τ は温度(類似度の差をどれだけ強調するかのスカラー)です。意味は「多数の選択肢(正例 1 個 + 負例多数)の中から、正例を当てる分類問題」で、正例が相対的に最も似ていれば損失が小さくなります。NCE は Noise-Contrastive Estimation の略で、「本物(正例)とノイズ(負例)を見分ける」発想に由来します。負例が多いほど問題が難しくなり、良い表現が促されます。

「自己運動(ego-motion)」とは、自車自身の動き(前進した距離、回転した角度、姿勢の変化)です。GPS・IMU(慣性計測装置)・オドメトリ(車輪回転からの距離推定)や連続画像から推定でき、これが分かると「同じ物体を別時刻・別位置から見たフレーム」を幾何的に対応づけられます。

「自明解(trivial solution / collapse)」とは、モデルが楽をして「全部同じベクトルを出す」など、損失は下がるが中身が空っぽな解に陥ることです。時間的に近すぎるフレームを正例にすると起きやすい失敗で、対策が要ります。

## 仕組みを詳しく
時系列対照学習の正例・負例の作り方を 3 通り示します。

(1) 連続フレームを正例にする(時間近接)
- 動画から 1 つのフレーム(アンカー)を選び、その直後・直前のフレームを正例にする。
- 同じバッチ内の別シーンのフレームを負例にする。
- 各フレームをエンコーダでベクトル化(例: `[256]`)し、アンカーと正例のコサイン類似度を上げ、アンカーと負例の類似度を下げる(InfoNCE)。

ステップでまとめると:
1. 動画から時刻 t のフレームと t+Δ のフレームを取り出す(正例ペア)。Δ は数フレーム分。
2. バッチ内の他シーンのフレーム N−1 個を負例として用意。
3. すべてをエンコーダに通しベクトル化(`[N, 256]`)。
4. アンカー t と正例 t+Δ の類似度を分子、全負例との類似度を分母にした InfoNCE を計算。
5. 損失を最小化(=正例を相対的に最も似せる)。

```
時刻:  ... t-1   t   t+1 ...        別シーン
            └ 正例 ┘                 frame_x, frame_y (負例)
                │
         アンカー t と t+1 を近づけ、t と frame_x/frame_y を遠ざける
```

(2) 自己運動(ego-motion)を使う
- 自車の動き(前進距離・回転角)が分かれば、「同じ場所を別時刻に見たフレーム」や「動きで対応づくフレーム領域」を正例として作れる。
- 例: 0.5 秒前に正面に見えた物体が、自車の前進で近く・大きく見える。この対応を正例にすると、視点や距離の変化に頑健な表現が得られる。深度と自己運動を同時に学ぶ自己教師あり手法(monodepth2 などの monodepth 系)も、隣接フレームを自己運動で射影して再構成する幾何的整合を学習信号にします。

(3) 複数視点(マルチビュー)を正例にする — TCN の方式
- Sermanet らの Time-Contrastive Networks (TCN) は、同じ動作を複数カメラで同時に撮影し、「同時刻・別視点」を正例、「同視点・別時刻」を負例にした。
- 直感: 同じ瞬間の動作は視点が違っても似ているべき(正)。同じカメラでも時間が違えば動作の段階が違う(負)。これで視点に依存せず、動作の進行段階を捉えた表現が学べる。単視点しかない場合は、時間窓を使い「アンカー近傍を正例、十分離れた時刻を負例」とする時間近接版も提案されました。

予測型の時系列 SSL も近縁です。Contrastive Predictive Coding (CPC) は「過去のフレーム列を要約した文脈から未来フレームの表現を予測し、正しい未来を負例の中から InfoNCE で当てる」形で、対照学習と未来予測を組み合わせます。「対照(どれが正例か当てる)」と「予測(未来を当てる)」は連続したスペクトルにあると理解できます。

設計の勘所:
- 時間間隔 Δ をどう取るか。近すぎると正例がほぼ同じ画像になり自明解に陥り、遠すぎると正例が別物になりすぎる。運転では数百ミリ秒〜数秒のオーダーが妥当な目安。
- 負例の数と質。バッチを大きくするか、MoCo のようなメモリバンク/キュー(過去の埋め込みを溜める辞書)で負例を増やす。
- データ拡張。色変えや切り抜きを併用し、「時間差以外の手がかり(色味の連続性など)」で楽をするのを防ぐ。
- 投影ヘッド。対照損失をかける前に小さな MLP(投影ヘッド)を通し、その出力で損失を取りつつ、下流ではヘッド手前の表現を使うと精度が上がる(SimCLR の発見)。

## 手法の系譜と主要論文
- van den Oord et al., "Representation Learning with Contrastive Predictive Coding" (CPC, 2018, DeepMind)。提案: 過去の系列から未来の潜在表現を予測し、InfoNCE で正しい未来を当てる。新規性: InfoNCE 損失を定式化し、相互情報量の下界を最大化する観点で対照学習を理論づけ、音声・画像・テキストなど多様な系列で有効性を示した。時系列 SSL の理論的支柱。

- Sermanet et al., "Time-Contrastive Networks: Self-Supervised Learning from Video" (ICRA 2018, Google Brain)。提案: 同時刻・別視点を正例、同視点・別時刻を負例にする時間対照学習。動機: ロボットが人間の動作を模倣するため、視点非依存で動作段階を捉えた表現が欲しかった。新規性: 同期マルチカメラ動画だけからロボットの模倣・ポーズ模倣に使える表現を獲得し、表現間距離を報酬関数として強化学習にも利用した。

- Chen et al., "A Simple Framework for Contrastive Learning of Visual Representations" (SimCLR, ICML 2020) / He et al., "Momentum Contrast" (MoCo, CVPR 2020)。提案: 画像 1 枚の異なるデータ拡張(切り抜き・色変え)を正例にする画像対照学習。新規性: SimCLR は大きなバッチと強い拡張・投影ヘッド、MoCo はモメンタムエンコーダ(緩やかに更新する別エンコーダ)とキューで負例を確保。時間構造を使わないが、時系列対照学習はこの枠組みを時間方向に拡張したもの。

- Grill et al., "Bootstrap Your Own Latent" (BYOL, NeurIPS 2020) / Caron et al. "DINO" (ICCV 2021)。提案: 負例を一切使わず、オンライン側がターゲット側(EMA で更新)の出力を予測する自己蒸留型。新規性: stop-gradient と非対称設計で崩壊を防ぎ、負例集めの負担を消した。時系列・動画 SSL でも負例不要路線が広がった。

- 運転寄りの応用: 自己運動や連続フレームを正例にした研究群(深度・自己運動を同時に学ぶ自己教師あり系統、動画フレーム順序の予測やパズル並べ替えなどの pretext タスク)があり、運転表現を実車データから安価に鍛える方向で発展しています。近年は対照学習よりマスク予測や自己蒸留(BYOL/DINO 系、負例不要)へ重心が移りつつありますが、「時間を教師に使う」核は共通です。

## 論文の実験結果(定量データ)
TCN(ICRA 2018): 評価はロボット模倣と表現の質です。同時刻・別視点を正例にした TCN 表現を使うと、人間のデモ動画だけからロボットが液体注ぎ(pouring)などの動作を模倣でき、視点不変性を測る代理タスク(別視点フレームのアラインメント誤差)でベースライン(教師あり ImageNet 特徴や単視点のシャッフル&学習手法)より小さい誤差を達成したと報告しています。ラベルなしの動画から「動作の進行段階」を捉えた表現が得られ、その表現空間の距離がそのまま模倣の報酬として使えた点が成果です。

CPC(2018): 線形評価(学習した表現を凍結し、その上に線形分類器だけ載せて精度を測る、表現の質の標準的測定法)で、音声話者識別や ImageNet など複数ドメインで、当時の自己教師あり手法を上回るか競合する精度を示しました。ImageNet 線形評価で当時の SSL を上回る Top-1 を達成し、「未来予測 + InfoNCE」が汎用に効くことを示しました。アブレーションでは、予測する未来ステップ数や負例数が性能に効くことが報告されています。

SimCLR / MoCo(2020): ImageNet 線形評価 Top-1 で、SimCLR が約 69〜76%(モデル規模・学習長による)、MoCo v2 が約 71% を達成し、教師あり学習(ResNet-50 で約 76%)に肉薄しました。重要なアブレーションは「強いデータ拡張(特に色歪みとランダム切り抜きの併用が単独より大きく効く)」「大きなバッチ/メモリバンクによる負例数(SimCLR はバッチ 4096〜8192 で精度が伸びる)」「投影ヘッド(対照損失をかける前の小さな MLP、付けると数ポイント改善)」がいずれも数ポイント単位で精度に効くことです。時系列版でも同種の要素(拡張・負例数・正例間隔)が性能を左右します。

運転ドメインの一般的傾向: 時系列・自己運動を使った自己教師あり事前学習は、限られたラベルでの下流タスク(セマンティックセグメンテーション、物体検出、深度推定)を、ランダム初期化や ImageNet 初期化より改善することが複数研究で報告されています。改善幅はデータ量・ドメイン一致・正例設計に強く依存し、特に「ラベルが少ない領域」ほど SSL 事前学習の相対的な利得が大きい傾向があります。

## メリット・トレードオフ・限界
- メリット: 人間のラベルが不要。膨大に集まる運転動画をそのまま学習資源にできる。
- メリット: 時間方向の動き・連続性・因果を捉えた表現が学べ、静止画 SSL より運転に適する。
- メリット: 自己運動を使えば視点・距離変化に頑健な表現が得られ、幾何構造を間接的に学べる。
- トレードオフ: 「何を正例・負例にするか」の設計が性能を大きく左右し、調整が難しい。
- トレードオフ: 負例を多く必要とする手法は大きなバッチやメモリバンクが要り、メモリ・計算コストが高い。
- 限界(失敗例): 時間的に近いフレームは似すぎていて(ほぼ同じ画像)、表現が「微小変化を無視する」自明解に陥るリスクがある。データ拡張・時間間隔の工夫・負例不要の自己蒸留(BYOL/DINO)などで回避する。
- 限界(失敗例): 色味や露出の連続性など「時間以外のショートカット」で正例を当ててしまい、意味を学ばないことがある。強い色拡張で潰す必要がある。
- 限界(研究課題): 「時間近接=意味的に近い」という仮定は、急なシーン変化(トンネル出入り、突然の遮蔽、カメラのフラッシュ)では崩れる。動的物体の動きと自車運動を混同しないよう、幾何や運動を明示的に扱う設計が課題。

## 発展トピック・研究の最前線
- 負例不要の自己教師あり: BYOL や DINO は負例なしで(自己蒸留と stop-gradient で)崩壊を防ぎつつ学習でき、運転動画にも適用が進んでいます。負例集めの負担が消える利点があります。
- マスク予測との融合: 動画マスク予測(時空間 MIM、VideoMAE など)が時系列 SSL の主流の一角になり、対照と再構成を組み合わせるハイブリッドも研究されています。
- 自己運動・幾何の活用: 深度・自己運動・オプティカルフローを同時に学ぶ自己教師あり手法は、運転の幾何構造を直接学習信号にでき、3 次元理解の事前学習として有望です。多視点幾何の整合を損失に使うと、対照学習より直接的に「3 次元的に正しい」表現を促せます。
- マルチモーダル時系列対照: カメラと LiDAR、あるいはカメラと運転行動・言語を時間整合で対照学習し、運転の基盤表現を作る方向。CLIP 的なクロスモーダル対照と時系列対照を結ぶ研究が増えています。
- 履歴のアーキテクチャ的集約との関係: 時間を「損失」で学ぶ対照学習に対し、過去フレームを時間方向に集約する履歴エンコーダ(時間畳み込みや GRU)は「時間をアーキテクチャ」で扱います。両者は補完的で、事前学習(対照)→ 履歴集約(アーキテクチャ)という組合せが自然です。
