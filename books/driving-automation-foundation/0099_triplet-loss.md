# Triplet Loss

## ひとことで言うと

Triplet Loss (トリプレット損失, 三つ組損失) は「基準・仲間・他人」の3つを一組にして、基準と仲間は近く、基準と他人は遠くなるように特徴を学習する方法です。さらに「他人は仲間より少なくともこれだけ遠くにいなさい」という余白 (マージン) を設けて、近すぎる位置関係を許しません。顔認証のように「2枚の写真が同じ人か」を距離で判定したい場面で使われます。学習時に見ていない人物でも、距離だけで「同一かどうか」を判定できるのが強みです。

## 直感的な理解

クラスの集合写真から「同じ人を探す」ゲームを考えてください。基準となる1枚 (アンカー) を手に取り、「これと同じ人の別の写真 (仲間)」と「別人の写真 (他人)」を比べます。良い判定をするには、「仲間は手元に引き寄せ、他人は遠くへ押しやる」という配置を頭の中に作ればよいわけです。さらに「他人は仲間より一定以上遠くにいてほしい」という余白を設ければ、仲間と他人が紛らわしく接近する事態を防げます。

なぜこの「相対的な距離」を学ぶのか。素朴には「人物Aクラス、人物Bクラス…」と人ごとにクラスを作って分類器を学習する手があります。しかし人物は世界に無数にいて、新しい人が増えるたびに分類器を作り直すのは非現実的です。さらに、クラス数が数百万に達すると、出力層 (各クラスへのスコアを出す全結合層) のパラメータが巨大になり学習も難しくなります。Triplet Loss は発想を変え、「似たものどうしが近くに集まる空間」を作る関数を学習します。この空間さえできれば、判定は「2点の距離がしきい値より近いか」だけで済み、学習時に見ていない人物にも対応できます。これを距離計量学習 (metric learning) と呼びます。

単に「同じ人は近づける」だけでは、すべてを1点に潰す崩壊が起きます。「違う人は遠ざける」も同時に課す必要があります。Triplet Loss は、この2つを1つの式で、しかも「どれくらい遠ざければ十分か」というマージン付きで同時に表現したものです。ペア型の対照損失が「正ペアの絶対距離」と「負ペアの絶対距離」を別々に縛るのに対し、Triplet は「正と負の距離の差」という相対量を縛るのが本質的な違いです。絶対的な距離スケールに縛られないので、人物ごとの写真の多様さ (ある人は表情が豊か、ある人は地味) に柔軟に対応できます。

## 基礎: 前提となる概念

- 埋め込み (embedding): 画像などを固定長の数値ベクトル (例: 128次元) に写したものです。この空間で「似たものは近い」という性質が成り立てば、距離で類似を判定できます。

- 距離計量学習 (metric learning): 「2点が似ているか」を測る距離そのものを学習目標にする手法群です。分類器の中間層を流用するのではなく、最終的に使う距離を直接最適化する点が特徴です。応用は顔認証・人物再識別・画像検索・推薦・少数ショット学習など、「ラベルが膨大か未知で、距離で照合したい」問題全般です。

- ユークリッド距離 (の二乗): 2点を結ぶまっすぐな距離です。Triplet Loss では計算の都合上、その二乗 d(a,b) = ||f(a)-f(b)||² を使うのが一般的です。埋め込みは通常、長さ1に正規化 (L2 正規化) してから距離を測ります。単位球面上に乗せると、ユークリッド距離とコサイン類似度が単調に対応するため、どちらで考えても同じになります。

- マージン (margin): 「正例との距離と負例との距離の差に、これだけののりしろを要求する」正の定数です。マージンがあることで「負例を無限に遠ざける」無駄を避けつつ、仲間と他人をはっきり分離できます。マージンは決定境界に安全余裕を持たせる役割で、サポートベクターマシン (SVM) のマージンと発想が共通します。

- ハードネガティブ (hard negative): 「別人なのに埋め込みが近い」、つまり間違えやすい負例のことです。学習を進めるには、こうした難しい例を見つけて学ぶことが鍵になります。逆に「最初から十分遠い負例」は損失ゼロで勾配を生まず、学習に寄与しません。

## 仕組みを詳しく

3つの入力を一組 (三つ組, triplet) にします。

- アンカー (anchor, 基準) a: ある人物の写真。
- ポジティブ (positive, 正例) p: 同じ人物の別の写真。
- ネガティブ (negative, 負例) n: 別の人物の写真。

それぞれを同じネットワークに通して埋め込みベクトル f(a), f(p), f(n) を得ます。各ベクトルは長さ1に正規化されます (形状はたとえば `[128]`)。距離 d には通常ユークリッド距離の二乗を使います。損失は次の式です。

```
L = max( 0,  d(a, p) - d(a, n) + margin )
```

意味を分解します。

- d(a, p): アンカーと正例の距離。小さくしたい (同一人物は近い)。
- d(a, n): アンカーと負例の距離。大きくしたい (別人は遠い)。
- margin: 正例と負例の距離差に要求する余白 (例: 0.2)。

max(0, ・) は「条件を満たしていれば損失ゼロ、満たしていなければ違反量を罰する」というヒンジ損失の形です。条件 d(a,n) ≥ d(a,p) + margin を満たすと中身が負になり、max で 0 にクリップされます。

数値例。margin=0.2 とします。

- 例1: d(a,p)=0.1, d(a,n)=0.9。式は max(0, 0.1 - 0.9 + 0.2) = max(0, -0.6) = 0。すでに負例が十分遠いので損失ゼロ、更新不要。
- 例2: d(a,p)=0.5, d(a,n)=0.6。max(0, 0.5 - 0.6 + 0.2) = max(0, 0.1) = 0.1。負例が近すぎる (差が margin に届かない) ので損失が発生し、「正例を近づけ、負例を遠ざける」更新が起きます。
- 例3: d(a,p)=0.7, d(a,n)=0.3 (順序が逆=完全に間違っている)。max(0, 0.7 - 0.3 + 0.2) = max(0, 0.6) = 0.6。最も強い勾配が流れ、急いで配置を直します。

### 三つ組の選び方 (triplet mining) が要

ここで決定的に重要なのが、三つ組をどう選ぶか (triplet mining, トリプレットマイニング) です。ランダムに選ぶと、例1のように最初から損失ゼロの「簡単すぎる三つ組」ばかりになり、学習が進みません。データセットが大きくなるほど、ランダムな三つ組のほとんどが「すでに正しく配置された簡単な組」になり、勾配がほぼゼロになります。難しい組を意図的に選ぶ必要があります。

- ハードネガティブ (hard negative): d(a,n) が最も小さい負例 (別人なのに最も近い)。最も学習信号が強い。
- セミハードネガティブ (semi-hard negative): d(a,p) < d(a,n) < d(a,p) + margin の範囲にある負例。「正例より遠いが、margin の余白には食い込んでいる」中程度の難しさ。

FaceNet は「最も難しい負例だけを選ぶと学習が不安定化・崩壊しやすい」ことを指摘し、セミハードネガティブを選ぶのが安定すると報告しました。難しすぎる例は外れ値やラベルノイズの影響を受けやすく、勾配が暴れ、ときに崩壊 (f が定数に潰れる) を招くためです。マイニングはバッチ内でオンラインに行う (online triplet mining) のが実用的で、全データの三つ組を事前列挙する非現実的な計算 (O(N³) の組合せ) を避けられます。実装上は、バッチに各人物の画像を複数枚入れ (例: P 人 × K 枚)、バッチ内で正例・負例を探す batch-hard / batch-all 戦略が標準です。

数値直感: データセットに 1万人 × 各20枚あると、三つ組は理論上およそ 10000 × 20 × 19 × (10000×20 − 20) ≈ 7.6×10¹³ 通りで、全列挙は不可能です。だからバッチ内オンラインマイニングで「今のバッチに含まれる、いま難しい組」だけを使うのが現実解になります。

## 手法の系譜と主要論文

- Contrastive Loss (Hadsell, Chopra, LeCun, CVPR 2006) が前史です。「2つ一組 (ペア) で、同類なら近づけ、異類なら margin まで遠ざける」損失でした。これは正例と負例を別々のペアで扱うため、両者の相対関係を直接は最適化しません。固定の絶対距離しきい値に縛られる弱点があります。

- FaceNet (Schroff, Kalenichenko, Philbin, Google, CVPR 2015, arXiv:1503.03832) が顔認証のために Triplet Loss を有名にしました。提案: 顔画像を 128次元の埋め込みに写し、同一人物は近く・別人は遠くなる空間を Triplet Loss で直接学習する。動機: 当時主流だった「人物分類器の中間層を特徴として流用する」やり方は、判定に使う距離を直接最適化しておらず非効率だった。新規性: アンカーを共通基準にした三つ組で、正例と負例の相対的距離関係を1つの式で直接扱い、さらにオンラインのセミハードマイニングを導入した。効果: 後述の通り顔認証ベンチで当時最高精度。限界: 三つ組の選び方が性能を大きく左右し、実装が繊細。

- N-pair Loss (Sohn, NEC Labs, NeurIPS 2016) は「負例1個」の Triplet を「バッチ内の多数の負例」に拡張し、softmax 形式で扱いました。これは [InfoNCE 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0062_infonce-loss) への橋渡しに当たります。Triplet Loss は「負例が1つだけの特殊形」と見ることができ、InfoNCE は負例多数への一般化です。動機: 負例1個では各更新で得られる情報が少なく収束が遅いため。

- In Defense of the Triplet Loss (Hermans, Beyer, Leibe, RWTH Aachen, 2017, arXiv:1703.07737) は人物再識別 (person re-identification) で、バッチ内でハード/セミハードな三つ組を効率的に選ぶ batch-hard mining を提案し、Triplet Loss が分類ベース手法より優れうることを再評価しました。動機: 「Triplet は古い・分類の方が良い」という当時の通説への反論。マイニング戦略の重要性を体系的に示した点で重要です。

- Multi-Similarity Loss (Wang ほか, CVPR 2019, arXiv:1904.06627) は、マイニングを陽に行う代わりに、ペアの自己類似度・正例間類似度・負例間類似度の3種を一括で重み付けする損失を提案しました。動機: ハードマイニングの繊細さを、滑らかな重み付けで置き換えるため。深層距離計量学習のベンチで強い結果を示しました。

- Proxy-Anchor Loss (Kim ほか, CVPR 2020, arXiv:2003.13911) は、各クラスに学習可能な代表点 (proxy) を割り当て、サンプルと proxy の関係で損失を定義することで、マイニング不要かつ高速に学習する流れ (proxy ベース metric learning) を代表します。動機: サンプル間の三つ組を数える計算とマイニングの不安定さを、proxy で吸収するため。

系譜は、Contrastive (ペア) → Triplet (三つ組, 相対距離) → N-pair / InfoNCE (負例多数) という、扱う負例の数を増やす方向と、Multi-Similarity / Proxy-Anchor のような「マイニング依存を減らす」方向の、2つの軸で進みました。

## 論文の実験結果 (定量データ)

顔認証で使われる指標を説明します。LFW (Labeled Faces in the Wild) は「2枚が同一人物か」を当てる検証 (verification) ベンチで、精度 (accuracy) は正しく同一/別人を判定できた割合です。1.0 (100%) に近いほど良く、人間の判定精度 (約99%強) が一つの目安です。人物再識別では mAP (mean Average Precision) と Rank-1 精度 (検索結果の1位が正解の割合) が使われます。

報告されているおおよその数値:

- FaceNet は LFW でおよそ 99.63% の検証精度を達成しました。これは当時の最高水準で、しかも128次元という小さなベクトルで実現した点が画期的でした。埋め込みが小さいほど、保存・検索 (大量の顔データベースとの照合) が高速になります。YouTube Faces DB でもおよそ 95.1% を記録しました。アブレーション/分析では、(a) 埋め込み次元を 128 から増やしても精度はほぼ頭打ちで、小さい次元で十分なこと、(b) マイニング戦略が決定的で、最も難しい負例だけを選ぶと学習が崩れやすく、セミハードが安定して高精度に至ること、(c) 学習データ量を 260M 枚規模まで増やすほど精度が向上することが示されました。データ量と精度の関係は、計量学習でもスケールが効くことを示します。

- Hermans ら (Re-ID) は、適切な batch-hard マイニングを施した Triplet Loss が、当時の複雑な分類+検証併用手法を上回ることを Market-1501 や MARS などの再識別ベンチで示しました。Market-1501 で mAP・Rank-1 ともに当時の上位を記録し、「Triplet Loss は古いと言われがちだが、マイニングを正しくやれば強い」というのが論文の主張で、タイトル "In Defense of the Triplet Loss" の通りです。これはマイニングが損失式そのものより重要であることの定量的証拠です。アブレーションでは batch-hard が batch-all やランダム選択より一貫して良いこと、ソフトプラス版のマージン (soft-margin) が固定マージンと同等以上に働くことが示されました。

- Multi-Similarity Loss は CUB-200・Cars196・SOP などの画像検索ベンチで、当時の Triplet/N-pair 系を上回る Recall@1 を報告し、マイニングを陽に行わなくても重み付けで同等以上に到達できることを示しました。

教訓は、Triplet Loss の性能は損失式よりもマイニング戦略 (どの三つ組を選ぶか) に強く依存する、という点です。そして後続研究は、その繊細なマイニングを「滑らかな重み付け」や「proxy」で置き換える方向に進みました。

## メリット・トレードオフ・限界

メリット
- 「同じか違うか」を距離で判定する問題 (顔認証・再識別・画像検索) に直接フィットする。
- 学習時に未知のクラス (新しい人物) でも、距離だけで判定できる。クラスを増やしても再学習が不要 (open-set 認識に強い)。
- マージンによって「どこまで離せば十分か」を明示でき、過剰に離す無駄を避けられる。
- 小さな埋め込み次元でも高精度が得られ、大規模照合が高速。
- 相対距離を縛るため、絶対的な距離スケールの差 (クラスごとの分散の違い) に柔軟。

トレードオフ・限界
- 三つ組の選び方 (mining) が性能を大きく左右し、簡単な組ばかりだと学習が止まる。実装が繊細で、再現性の落とし穴が多い。
- 負例が1つだけなので、InfoNCE のように多数の負例から得られる強い信号に比べると学習効率が劣りやすい。1更新あたりの情報が少ない。
- マージンというハイパーパラメータの調整が必要。大きすぎると満たせず崩壊、小さすぎると分離が甘い。
- 難しすぎる負例ばかり選ぶと、外れ値やラベルノイズに引っ張られ学習が不安定化・崩壊しうる (f が定数に潰れる)。
- ラベル (誰が同一人物か) が必要なので、純粋な自己教師あり学習ではなく、メトリック学習 (弱教師あり) の文脈で使われることが多い。ただし正例をデータ拡張で自動生成すれば自己教師化できる。

## 発展トピック・研究の最前線

Triplet Loss から、扱う負例を増やす方向で N-pair Loss、そして [InfoNCE 損失](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0062_infonce-loss) へと発展しました。現在の大規模表現学習では、温度付き softmax で多数の負例を一度に扱う InfoNCE 系が主流です。一方で Triplet の発想 (アンカー基準の相対距離 + マージン) は、検索・再識別・距離計量学習の現場でいまも生きています。

マイニングを巡る研究は続いており、バッチ全体の構造を使う手法 (Multi-Similarity Loss のように全ペアを滑らかに重み付けする)、各クラスに代表点を置く proxy ベースの metric learning (Proxy-NCA, Proxy-Anchor) が提案されています。proxy ベースはサンプル間の組合せ爆発とマイニングの不安定さを回避し、収束が速い反面、クラス内の細かい多様性の表現は弱まる、というトレードオフがあります。自己教師あり学習との接続では、Triplet を「正例も自動生成 (データ拡張) で作る」ことで疑似ラベル化し、ラベルなしに拡張する流れもあります。

顔認証の最前線では、Triplet の発想を分類の枠組みに取り込んだ角度マージン損失 (SphereFace, CosFace, ArcFace) が主流になりました。これらは「クラス中心との角度にマージンを課す」もので、Triplet のマイニング不要・分類の安定性・マージンによる分離性を兼ね備え、現在の顔認証の標準です。Triplet Loss はこの系譜の出発点として、計量学習の考え方を確立した古典的かつ実用的な損失として位置づけられます。
