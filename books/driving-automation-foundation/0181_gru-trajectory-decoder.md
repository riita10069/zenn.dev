# GRU軌道デコーダ (再帰型の逐次軌道生成)

## ひとことで言うと
GRU という小さな記憶を持つニューラルネットワークを使い、運転軌道を「最初の一歩、次の一歩、その次の一歩…」と時間方向に一歩ずつ順番に作っていく計画方式です。一歩ごとに更新される内部の記憶ベクトル(hidden state、隠れ状態)が、過去の一歩を次の一歩へ橋渡しするので、なめらかにつながった軌道が得られます。最後の隠れ状態は「この先どう運転するつもりか」を 1 本のベクトルに凝縮した要約として、未来予測など下流の部品に渡すこともできます。自動運転や軌道予測で長く使われてきた、シンプルで実装しやすい古典的なデコード方式です。

「GRU(Gated Recurrent Unit、ゲート付き回帰ユニット)」とは、時系列データを順に処理し、過去の情報を「記憶」として持ち越せる再帰型ニューラルネットワーク(RNN)の一種です。「ゲート」とは「過去の記憶をどれだけ残し、新しい情報をどれだけ取り込むか」を学習で調整する仕組みです。

## 直感的な理解
文章を一文字ずつ書く様子を想像してください。「自」と書いたら、次に来るのは「動」が自然で、その次は「運」、そして「転」。一文字ごとに「これまで何を書いたか」を覚えていて、それを踏まえて次の文字を選びます。GRU 軌道デコーダはこれと同じことを運転軌道でやります。「1 歩目で右に少し」と決めたら、その記憶を持ち越して「2 歩目はその続きで右に少し」と出す。記憶が橋渡しになるので、出力が前後でなめらかにつながります。

なぜ一歩ずつ作るのか。運転軌道は「将来数秒間、0.1 秒刻みで数十ステップ分の操作」のような時系列で、各ステップは前のステップと滑らかにつながっている必要があるからです。もし「1 歩目で急加速、2 歩目で急減速」のようにステップ同士がバラバラだと、ガクガクした(ジャーク=加加速度が大きい)乗り心地の悪い、物理的に無理な軌道になります。前の状態を覚えていて次を出す逐次生成は、この時間的なつながり(temporal coherence、時間的整合)を自然に確保できます。

## 基礎: 前提となる概念
土台となるのが「再帰型ニューラルネットワーク(RNN)」です。RNN は系列を 1 要素ずつ処理し、各時刻で「隠れ状態 $h_t$」を更新しながら過去の情報を持ち越します。更新は $h_t = f(h_{t-1}, x_t)$ という形で、前の隠れ状態 $h_{t-1}$ と今の入力 $x_t$ から新しい隠れ状態を作ります。素朴な RNN は長い系列で勾配が消えたり爆発したりして学習が難しい、という弱点がありました。

これを解決するゲート機構を持つのが LSTM と GRU です。GRU は更新ゲート(過去をどれだけ残すか)とリセットゲート(過去をどれだけ忘れるか)の 2 つのゲートで情報の流れを制御します。LSTM より構造が単純でパラメータが少なく、短中期の系列では同等の性能が出やすいため、軌道生成のような数十ステップの系列に向いています。

「自己回帰(autoregressive)」も鍵です。これは「自分が出した前の出力を、次のステップの入力として使う」生成方式です。各時刻で操作を 1 つ出し、それを次のステップに戻して連鎖的に系列を伸ばします。文章生成や音声合成でも使われる汎用的な考え方です。

## 仕組みを詳しく
処理を段階的に分解します。

第 1 段階(文脈を作る)。認識した周囲状況(BEV 特徴や物体・地図の符号化)、過去フレームの視覚履歴、自車の運動履歴(速度・加速度・ヨー角など)をまとめ、GRU の初期入力と初期隠れ状態の材料にします。これが「どんな状況から軌道を生成し始めるか」を決めます。

第 2 段階(隠れ状態を初期化する)。GRU の隠れ状態 $h_0$ を初期化します(例えば 256 次元のベクトル)。これが「今からの運転意図」を時間とともに育てていく器です。文脈特徴から線形層で $h_0$ を作るのが一般的です。

第 3 段階(一歩ずつデコードする = 自己回帰ループ)。$t = 1$ から $T$(例えば 64)まで繰り返します。各ステップで GRU は「直前の出力(または直前の状態)」を入力に取り、隠れ状態 $h_t$ を更新し、その $h_t$ から小さな出力ヘッド(線形層)で時刻 $t$ の操作、例えば (加速度, 曲率) を出します。前のステップの記憶 $h_{t-1}$ が引き継がれるので、出力が滑らかにつながります。

第 4 段階(出力をまとめる)。$T$ ステップ分の操作を並べて軌道 [B, T, 2] とします。出力を「位置 (x, y)」ではなく「加速度・曲率」のような操作量で出す設計もよく使われます。理由は、操作量は車両運動モデル(自転車モデルなど)を通して積分すれば必ず物理的に実現可能な軌道になり、横滑りや瞬間移動のような非現実的な軌道を構造的に防げるからです。

最後の隠れ状態 $h_T$ を「運転意図の要約ベクトル」として別途取り出す設計も一般的です([B, 256] 程度)。このベクトルは「自車がこの先どう動くつもりか」を凝縮しており、未来の視覚状態やシーンを予測する世界モデル的な部品の条件入力に使えます。軌道を生成しつつ、その意図をコンパクトに表現できる点が、逐次デコーダの隠れた利点です。

形状の感覚をまとめると、入力の文脈は [B, 256] など、各ステップ出力は [B, 2]、それを $T$ 回ためて軌道 [B, T, 2](平坦化して [B, 2T])、要約隠れ状態は [B, 256]。

```
文脈 [B,256] → h0
  t=1: GRU(h0, x0) → h1 → 線形 → (a1, κ1)
  t=2: GRU(h1, a1,κ1) → h2 → 線形 → (a2, κ2)
   ...
  t=T: GRU(h_{T-1}, ...) → hT → 線形 → (aT, κT)
軌道 [B,T,2]、要約 hT [B,256]
```

## 手法の系譜と主要論文
GRU 自体の出典は Learning Phrase Representations using RNN Encoder-Decoder for Statistical Machine Translation(Kyunghyun Cho, Bart van Merriënboer, Caglar Gulcehre ら; EMNLP 2014; arXiv:1406.1078)です。機械翻訳のために提案された、LSTM より軽量なゲート付き再帰ユニットで、少ないパラメータで系列を扱える点が、軌道のような短中期の時系列デコードに移植されました。

自動運転での逐次軌道デコードの代表が ChauffeurNet(Mayank Bansal, Alex Krizhevsky, Abhijit Ogale; Waymo; RSS 2019; arXiv:1812.03079)です。鳥瞰の入力から RNN で軌道点を 1 つずつ生成し、模倣学習に「合成された摂動」や補助損失を加えて頑健性を上げました。動機は、純粋な模倣だけでは分布のずれ(自分の小さな誤差が積み重なり訓練で見たことのない状況に入る)に弱いこと。RNN による逐次生成は実装が単純で時間的整合を確保しやすいため、多くの模倣学習系 planner の標準形になりました。

軌道予測の分野でも GRU/LSTM デコーダは定番でした。VectorNet(Gao et al., CVPR 2020)や TNT/DenseTNT(Gu et al., 2021)など多くの予測モデルが、シーン符号化の後段に再帰デコーダや MLP デコーダを置いて将来軌道を生成しました。Social LSTM(Alahi et al., CVPR 2016)は歩行者の将来位置を LSTM で逐次予測し、相互作用を隠れ状態のプーリングで扱った先駆けです。

しかし逐次デコーダには限界があり、近年は生成モデルへの置き換えが進みました。多峰性の問題([アンカー軌道ボキャブラリ計画](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0158_anchor-based-trajectory-vocabulary) や [Diffusion Planner (拡散モデル計画)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0169_diffusion-planner) 参照)が大きく、単一軌道の回帰は mode averaging(複数正解の平均で危険な中間軌道を出す)に弱い。これに対し Flow Matching(Yaron Lipman ら; ICLR 2023; arXiv:2210.02747)は「ノイズから軌道へのベクトル場」を学ぶ生成手法で、多峰分布を扱え、並列に軌道全体を生成できます。多くの近年の planner が、自己回帰 GRU ループをこうした生成ヘッドへ置き換えています。

## 論文の実験結果(定量データ)
ChauffeurNet は社内および実車に近い評価で、純粋な模倣学習(行動を真似るだけ)に対し、摂動データと補助損失を加えることで脱輪・衝突といった失敗が大きく減ることを報告しました。特に「合成摂動」を抜くアブレーションでは、訓練分布から外れた状況での回復が著しく悪化し、逐次模倣が分布シフトに脆弱であることを定量的に示しました。これは逐次デコーダ一般の弱点(exposure bias、自分の誤った出力を次の入力に使うことで誤差が蓄積する現象)を浮き彫りにします。

軌道予測のベンチマーク(nuScenes prediction、Argoverse など)では、評価指標として minADE(複数候補のうち正解に最も近い軌道の平均変位誤差、メートル単位、小さいほど良い)、minFDE(同じく最終点の誤差)、Miss Rate(正解から一定距離以上ずれた割合)が使われます。単一軌道を出す素朴な GRU/LSTM デコーダは、多峰な未来を 1 本に潰すため minADE/minFDE が頭打ちになりやすく、複数候補を出すアンカー方式(MultiPath など)や生成方式に minADE で 1〜数メートル規模の差をつけられることが各種比較で示されています。例えば曲がるか直進するか分岐する交差点では、単一軌道は両者の中間を出してしまい、どちらの正解からも遠い大きな誤差を生みます。

逐次デコードと並列生成の速度比較も重要です。GRU は $T$ ステップを 1 つずつ回すため、ステップ数に比例した逐次計算が必要で GPU の並列性を活かしにくい。例えば $T=64$ ステップの軌道なら、GRU は 64 回の逐次行列演算を直列に並べる必要があり、各演算が軽くてもカーネル起動のオーバーヘッドと依存関係でレイテンシが積み上がります。一方、Flow Matching や拡散の少ステップ版は軌道全体を一度に(あるいは 2〜数ステップで)生成でき、同じ $T$ を 1 回ないし数回の並列演算で吐き出せるため、同等以上の品質をより低レイテンシで達成できることが報告されています。これらの理由から、研究の主流は逐次回帰から生成ヘッドへ移行しました。

逐次デコーダの脆弱性を端的に表すのが exposure bias の積み上がりです。学習時は各ステップの入力に「正解の前ステップ」を与える teacher forcing(教師強制)が一般的ですが、推論時は「自分が出した前ステップ」を入力に使うため、初期の小さな誤差が後段の入力分布をずらし、ステップが進むほど誤差が雪だるま式に増えます。軌道予測の比較では、この影響で minFDE(最終点誤差)が minADE(全点平均誤差)に対して相対的に大きくなる傾向、すなわち系列の後半ほど誤差が開く現象が観測されます。ChauffeurNet が合成摂動と補助損失で対処したのも、本質的にはこの分布シフトへの頑健化でした。並列生成ヘッドは全ステップを同時に出すため、この逐次的な誤差伝播そのものを構造的に持たない点が速度以外の利点になっています。

## メリット・トレードオフ・限界
メリット。隠れ状態を持ち越すのでステップ間の時間的整合(滑らかさ)を取りやすい。実装が単純で軽量、学習も安定しやすい。最後の隠れ状態をそのまま「運転意図の要約」として再利用でき、未来予測や世界モデルへの条件づけに自然につながる。短中期の系列なら GRU の表現力で十分なことが多い。

トレードオフと限界。単一軌道の回帰になりがちで多峰性に弱く、mode averaging で危険な中間軌道を出すリスクがある(複数モードを出す混合密度ネットワーク化などで緩和は可能だが設計が増える)。$T$ ステップを 1 つずつ回す自己回帰なので並列化しづらく、長い系列ではレイテンシが効く。exposure bias により長い系列で誤差が蓄積しやすい。これらの理由から、多峰性と速度が重要な近年の end-to-end 計画では、アンカー分類や Flow Matching / 拡散といった生成ヘッドへの置き換えが進んでいます。研究上は「逐次の時間的整合の良さ」と「並列生成の速度・多峰性」をどう両立するか(例: 生成ヘッドに運動学制約や平滑化を組み込む)が課題です。

## 発展トピック・研究の最前線
逐次デコーダの直接の後継は Transformer デコーダによる並列軌道生成です。クエリベースのデコーダ(複数の軌道クエリを並列にデコードする DETR 的な設計)は、多峰な複数軌道を一度に出しつつアテンションで状況を参照でき、minADE/minFDE で逐次 RNN を上回ります。さらに生成モデル化が進み、[Diffusion Planner (拡散モデル計画)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0169_diffusion-planner) や Flow Matching ヘッドが、ノイズから軌道分布を生成する形で多峰性と速度を両立します。

要約隠れ状態を下流に渡す発想は、世界モデル系の研究([MILE (潜在表現モデルベース模倣)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0074_mile_note)、[Drive-WM (多視点運転World Model)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0051_drive-wm))に通じます。運転意図を 1 本のベクトルに凝縮し、それを条件に未来の観測やシーンを予測する構成は、計画と予測を結びつける手がかりとして今も研究されています。運動学制約を生成過程に組み込み、生成ヘッドでも物理的に無理のない滑らかな軌道を保証する方向も活発です。
