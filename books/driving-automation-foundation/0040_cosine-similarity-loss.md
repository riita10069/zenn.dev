# コサイン類似度損失

## ひとことで言うと
2つのベクトル(数の並び)が「同じ方向を向いているか」を測り、それを揃えるための損失です。大きさは無視して向きだけを比べるので、特徴量の意味を揃える(アライメント)のに向いています。値は −1〜+1 に収まり、+1 が完全一致です。自己教師あり学習・蒸留・マルチモーダル対応の中心部品です。

## 直感的な理解
地図上で2人が指さす方向を考えます。一人が「北東を指している」、もう一人も「北東を指しているが腕がもっと長い」とき、腕の長さが違っても指している方向は同じです。ここで知りたいのは「同じ方向を向いているか」であって「腕の長さ」ではありません。

ニューラルネットは画像や点群を特徴ベクトル(feature vector、数の並び)に変換します。例えば1枚の画像を256個の数に圧縮する、といった具合です。このベクトルが「画像の意味」を表すと考えます。多くの場合、ベクトルの向きが意味を、大きさ(長さ)は明るさやコントラストといった副次的・ノイズ的な情報を担います。

そこで「2つの特徴ベクトルがどれくらい似ているか」を、向きだけで測りたい場面が出てきます。
- 同じ物体を別角度から撮った2枚の画像 → 特徴は似ているべき。
- 教師モデルの特徴と生徒モデルの特徴を揃えたい(蒸留)。
- 自己教師あり学習(正解ラベルなしで学ぶ)で「同じ画像の2つの加工版」の特徴を一致させたい。
- 画像とそのキャプション(テキスト)を同じ向きに揃えたい(マルチモーダル)。

これを実現するのがコサイン類似度です。

## 基礎: 前提となる概念

ユークリッド距離(普通の距離)は各要素の差の二乗和の平方根で、ベクトルの大きさに引きずられます。具体例で見ます。ベクトル A = [3, 4]、B = [6, 8]。B は A をちょうど2倍にしただけで向きは完全に同じです。「同じ意味」と見なしたい。でもユークリッド距離は √((6−3)²+(8−4)²) = √25 = 5 とゼロになりません。大きさが違うだけで「遠い」と判定してしまいます。

内積 (dot product) は2つのベクトルの各要素を掛けて足したもので、a·b = Σ aᵢbᵢ です。幾何的には a·b = |a||b|cosθ(θ は2ベクトルのなす角)という関係があります。ここから、内積をそれぞれの長さで割れば cosθ だけが残る、というのがコサイン類似度の発想です。

ノルム (norm, 長さ) |a| = √(Σ aᵢ²) はベクトルの大きさです。L2正規化(L2 normalization)とは、ベクトルを自分の長さで割って長さ1にそろえる操作で、これをやってから内積を取るとコサイン類似度になります。正規化後のベクトルはすべて単位球面(半径1の球の表面)の上に乗るので、多くの自己教師あり学習は「単位球面上での比較」を行っていると言えます。

## 仕組みを詳しく

コサイン類似度は2ベクトルのなす角のコサインです。

```
cos_sim(a, b) = (a · b) / (|a| * |b|)
```

割ることで長さの影響が消え、向きだけが残ります。値の範囲は −1〜+1。

```
+1 : 完全に同じ方向(角度0度)。最も似ている。
 0 : 直交(角度90度)。無関係。
-1 : 真逆の方向(角度180度)。最も似ていない。
```

A=[3,4], B=[6,8] で計算します。

```
a·b = 3*6 + 4*8 = 18 + 32 = 50
|a| = √(9+16) = 5,  |b| = √(36+64) = 10
cos_sim = 50 / (5*10) = 1.0
```

大きさが2倍違っても向きが同じなので類似度は 1.0。狙い通りです。

これを「損失(小さくしたい量)」にするには、似ているほど小さくなるよう符号を反転します。

```python
import torch.nn.functional as F

def cosine_loss(a, b):
    # a, b の形状: [B, D]  (B サンプル, D 次元の特徴)
    sim = F.cosine_similarity(a, b, dim=-1)  # 形状 [B], 範囲 [-1, 1]
    return (1 - sim).mean()                  # 似ているほど0に近づく
```

`1 - sim` とすると、向き一致(sim=1)で損失0、直交(sim=0)で損失1、真逆(sim=−1)で損失2。学習はこの値を下げる=向きを揃える方向に進みます。

ステップで言うと、
1. 2つの特徴ベクトル a, b を用意(例: 形状 `[8, 256]`、8サンプルそれぞれ256次元)。
2. それぞれを長さ1に L2正規化する(`F.cosine_similarity` が内部で実行)。
3. 正規化後の内積を取る → 各サンプルのコサイン類似度 `[8]`。
4. `1 - sim` を取って平均 → スカラーの損失。

実装上の注意。|a| が0(ゼロベクトル)だと割り算が壊れるので、`F.cosine_similarity` は内部で小さな値 eps を足して安全にしています。

### MSE との等価性

L2正規化した2ベクトルの平均二乗誤差(MSE)を展開すると、コサイン類似度と等価になります。

```
|â - b̂|² = |â|² + |b̂|² - 2(â·b̂) = 1 + 1 - 2 cos_sim = 2(1 - cos_sim)
```

(â, b̂ は正規化済みベクトルで長さ1)。BYOL の損失はまさにこの「正規化した特徴の MSE」で、コサイン類似度の最大化と同じです。実装ごとに「1−cos」か「正規化MSE」かの違いはあれど、本質は同一です。この等価性を知っていると、論文ごとに違う表記を見ても「同じことをやっている」と見抜けます。

### 対照学習での温度付きコサイン類似度

SimCLR のような対照学習では、コサイン類似度を温度 τ で割ってから softmax に入れます。正例(同じ画像の別加工)の類似度を分子に、全候補(負例含む)を分母に置く InfoNCE 損失です。温度 τ は類似度の差をどれだけ強調するかを決め、小さいほど「最も似た相手」に集中します。τ は典型的に 0.07〜0.5 程度で、性能に敏感なハイパーパラメータです。

## 手法の系譜と主要論文

コサイン類似度損失は、自己教師あり表現学習で中心的役割を果たしました。

- 対照学習の土台: SimCLR (Chen et al., ICML 2020, arXiv:2002.05709) は、同じ画像の2つの加工版(正例)のコサイン類似度を上げ、別画像(負例)との類似度を下げる InfoNCE 損失で表現を学びました。温度付きコサイン類似度が損失の核です。大量の負例が必要(大きなバッチサイズ、論文では 4096 まで)なのが難点でした。

- BYOL: Grill et al., "Bootstrap Your Own Latent" (NeurIPS 2020, DeepMind, arXiv:2006.07733)。負例を一切使わず、同じ画像の2つの加工版の特徴の向きを一致させるだけで良い表現を学べることを示し、分野に衝撃を与えました。崩壊(全部同じベクトルを出す手抜き)を防ぐ仕組みは、片方をゆっくり更新する target network(指数移動平均)と、もう片方にだけ付ける predictor という非対称構造です。損失は正規化特徴の MSE = コサイン一致の最大化。

- SimSiam: Chen & He (CVPR 2021, arXiv:2011.10566)。BYOL をさらに単純化し、負例なし・モメンタムなしでも stop-gradient(片側の勾配を止める)だけで崩壊を防げることを示しました。コサイン類似度損失が中心。崩壊回避の本質が stop-gradient にあることを明確にした重要研究です。

- DINO: Caron et al., "Emerging Properties in Self-Supervised Vision Transformers" (ICCV 2021, Meta, arXiv:2104.14294)。教師と生徒ネットワークの出力分布を揃える自己蒸留を行います。生徒の出力を教師の出力(softmax + centering)に近づけ、向きの一致を学びます。教師出力の centering(平均を引く)と sharpening(温度を下げる)の組み合わせが崩壊を防ぎます。

- CLIP: Radford et al. (2021, arXiv:2103.00020)。画像とテキストの特徴をコサイン類似度で対応づけ、4億の画像テキストペアで対照学習しました。コサイン類似度がモダリティをまたぐアライメントの標準手段になった画期です。

## 論文の実験結果(定量データ)

評価は通常、自己教師ありで学習した特徴を固定し、その上に線形分類器だけを乗せて分類精度を測る linear probe(線形評価)で行います。ImageNet(1000クラスの大規模画像分類)の Top-1 精度(最も確信したクラスが正解と一致する割合、高いほど良い)が標準指標です。

- SimCLR は ResNet-50 で ImageNet linear probe 約69.3% を達成し、当時の自己教師あり手法を大きく更新しました。ただし大バッチ・大量の負例・長い学習が前提でした。

- BYOL は ResNet-50 backbone で ImageNet linear probe 74.3% を達成し、SimCLR(約69%)を大きく上回りました。負例を使わずにこの精度に届いたことが画期的でした。さらに大きな backbone(ResNet-200 ×2 など)では79%超も報告されています。

- SimSiam は ResNet-50 で約71%(短い学習設定での比較)を達成し、モメンタムエンコーダも負例も不要でこの水準に到達。アブレーションが重要で、stop-gradient を外すと損失は一気に最小値(−1、全特徴が一点に崩壊)へ落ち、表現が完全に潰れて精度がチャンスレベル(1000クラスなら 0.1% 付近)になることを示しました。これが「崩壊回避の本質は stop-gradient」という結論の根拠です。

- DINO は ViT-S/8 で ImageNet linear probe 約80%、k-NN 評価(最近傍で分類)でも約78%と高精度。注目すべきは定性的結果で、ラベルなし学習にもかかわらず ViT の自己注意マップが物体の境界を教師なしで捉える(セグメンテーションマスクのように物体形状に一致する)ことを示しました。centering を外すと出力が一様分布へ、sharpening を外すと一点へ崩壊することがアブレーションで確認されています。

- CLIP は ImageNet をゼロショット(その分類タスクを一度も学習せず、クラス名のテキストとの類似度だけで分類)で約76% を達成し、ラベルで教師あり学習した ResNet-50 に匹敵しました。コサイン類似度ベースの対照学習が、汎用的で転移しやすい表現を生むことを示しました。

- 指標の意味。linear probe 精度が高いほど「特徴が線形分離できるほど意味的に整理されている」ことを意味します。教師あり学習(ラベルを使う)が同 backbone で76%台のところ、ラベルなしでそれに迫る・並ぶ・一部超えることが、自己教師あり学習の到達点として重要でした。

## メリット・トレードオフ・限界

メリット
- ベクトルの大きさ(スケール)に影響されず、向き(意味)だけを比べられる。
- 値が −1〜+1 に収まり解釈しやすく数値的に安定。
- 負例を集める手間なしに自己教師あり学習で良い表現を学べる(BYOL/DINO/SimSiam)。
- 高次元の特徴(数百〜数千次元)でも素直に効く。

トレードオフ・限界
- 向きしか見ないので、大きさ(強度・自信)に意味がある場合はその情報を捨ててしまう。
- これだけで学習すると「全部同じベクトル」に潰れる崩壊(representational collapse)が起こりうる。回避には stop-gradient、非対称 predictor、target network、centering/sharpening などの工夫が必須で、設計が繊細。
- ゼロベクトルや非常に短いベクトルで不安定になりうる(eps での保護が必要)。
- 角度の尺度であり、絶対的な距離尺度ではない。クラスタの密度や分離度をそのまま測れるわけではない。
- 未解決の課題として「なぜ非対称構造だけで崩壊しないのか」の完全な理論的理解がまだ確立していない。BatchNorm の役割や、predictor が学習中に果たす役割をめぐって議論が続いています。

## 発展トピック・研究の最前線

- 崩壊の理論解析。SimSiam の stop-gradient を EM アルゴリズムや交互最適化として説明する研究、predictor が固有空間を整えることで崩壊を防ぐとする線形モデル解析などが進んでいます。
- 次元崩壊 (dimensional collapse)。完全に一点へ潰れなくても、特徴が低次元の部分空間に偏る弱い崩壊が起こることが分かり、これを検出・防止する正則化が提案されています。VICReg (Bardes et al., ICLR 2022) はコサイン一致に頼らず、分散(各次元が潰れないよう下限を課す)と共分散(次元間の相関を減らす)の明示制約で崩壊を防ぐ別解です。
- マルチモーダルアライメント。CLIP に続き、画像・テキスト・音声・点群など複数モダリティをコサイン類似度で同じ空間に揃える研究が広がっています。コサイン類似度損失はモダリティをまたぐアライメントの標準手段です。
- 自動運転・ロボティクスへの応用。複数カメラや LiDAR・カメラ間で特徴を整合させる、あるいは自己教師あり事前学習で backbone を鍛えてからタスクを微調整する、といった用途でコサイン類似度損失が使われます。大きさを無視して向きだけ揃えたい特徴アライメントに特に向きます。
