# np.unwrapによるヨー角の±π巻き戻し処理

## ひとことで言うと

車の向き(角度)は普通 −π 〜 +π(約 −180度〜+180度)の範囲で記録されます。向きがこの境界をまたぐと、数値が +π から −π へ突然ジャンプします。このジャンプをそのまま差分すると、ほとんど向きが変わっていないのに巨大な角速度が出てしまいます。`np.unwrap`(位相アンラップ)はこのジャンプを取り除き、角度を連続な値にならす関数で、ヨーレートを計算する差分・微分の直前に必ず通すのが定石です。

## 直感的な理解

時計の長針を思い浮かべてください。長針が 59 分から 60 分(= 0 分)へ進むとき、実際にはほんの 1 分しか動いていないのに、目盛りの数字は「59」から「0」へ、見かけ上 −59 ぶん戻ります。この「59 → 0」を真に受けて「1 分で 59 分ぶん逆回転した」と解釈すると大間違いです。

角度でもまったく同じことが起きます。車が真後ろ付近(約 180度)を向いて走っていて、ほんの少し向きが変われば、記録上の角度は +179度から −179度へ飛びます。実際の変化はわずか 2度なのに、数値の差は −358度。これを時間で割ると、ありえない超高速の回転が出てしまう。`np.unwrap` は「これは一周ぶんの折り返しだ」と見抜いて、長針なら「60 分」と読み替えるように、角度を連続な値に直す道具です。

なぜこれが重要かというと、たった 1 点の偽スパイクが学習全体を壊しうるからです。差分で出た非物理的なヨーレートは、そこから作る曲率(= ヨーレート ÷ 速度)も巻き込み、その値が「正解」として学習に流れ込むと、モデルは「真後ろを向いたら突然激しく回れ」というありえない挙動を真似てしまいます。

## 基礎: 前提となる概念

ラジアンと角度の範囲。角度の単位ラジアンでは、一周が 2π(約 6.283)、半周が π(約 3.1416)です。多くのセンサやデータセットは向きを `[−π, +π)` の範囲に正規化(wrap、巻き付け)して記録します。範囲を一定に保てる(メモリや表示の都合が良い)利点がある一方、境界 ±π で不連続が生じます。

周期性(periodicity)。角度には「2π 足しても同じ向き」という性質があります。+179度 と −181度 は物理的に同じ向きです。だから数値としては不連続でも、2π を足し引きすれば連続な別の数値で同じ向きを表せます。`np.unwrap` はこの自由度を使ってジャンプを消します。

ヨーレートと差分。ヨーレート(向きの回転速度)は「向きの時間変化率」で、隣り合う時刻の向きを引いて時間で割って求めます。だから差分の入力である向きの系列に偽のジャンプがあると、その瞬間だけ巨大な偽ヨーレートが出ます。曲率もこのヨーレートから作るので、汚染が連鎖します([egomotion信号の導出（速度・加速度・ヨーレート・曲率）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0813_egomotion-signal-derivation) 参照)。

なぜ差分の「前」なのか。`np.unwrap` は連続した角度系列に対して隣との差を見てジャンプを補正します。差分を取った後の値(=ヨーレート)に unwrap をかけても、すでにスパイクが出た後では意味をなしません。順序が命です。

## 仕組みを詳しく

`np.unwrap` は角度の列を先頭から走査し、隣り合う要素の差がしきい値(既定では π)を超えたら、そこに ±2π(一周ぶん)を足し引きして「連続な軌跡」に戻します。1 周は同じ向きなので 2π 足しても角度としては等価ですが、数値としては飛びがなくなります。アルゴリズムは「累積補正量」を持ち歩き、ジャンプを検出するたびに ±2π を足してそれ以降の全要素に適用する、という一回走査(O(T))の単純なものです。

具体例。真後ろ付近を走り、わずかに向きが変わった場面。

```
時刻 t   : 記録された向き(ラジアン)
t=0      :  3.13    (約 179.3度)
t=1      : -3.13    (約 -179.3度) ← +πを少し超え、-π側へ巻き付いた
```

実際の向きの変化はわずか約 1.4度なのに、記録上の差は `-3.13 - 3.13 = -6.26`(約 −358度)。これを dt=0.1 で割ると −62.6 rad/s という、物理的にありえない回転速度になります。

unwrap をかけると:

```
unwrap 前 :  3.13,  -3.13           (差 -6.26)
unwrap 後 :  3.13,   3.153          (-3.13 に +2π=6.283 を足した。差 +0.023)
```

差が −6.26 から +0.023 に直り、dt=0.1 で割ると 0.23 rad/s という妥当なヨーレートになります。実際の小さな向き変化が正しく復元されました。

実装はヨーレート計算の直前に一行差し込むだけです。等間隔差分の場合:

```
heading  = np.unwrap(headings)        # まず unwrap
yaw_rate = np.zeros_like(heading)
yaw_rate[1:] = np.diff(heading) / 0.1 # そのあと差分
```

不等間隔の中心差分(`np.gradient`)の場合も同様に、gradient の前に unwrap を挟みます。

```
yaw_angle = Rotation.from_quat(quats).as_euler("ZYX")[:, 0]
yaw_rate  = np.gradient(np.unwrap(yaw_angle), t_s)   # gradient の前に unwrap
```

角度の連続化を ASCII 図で示します。

```
unwrap 前 (記録された角度):
 +π ┤      ●         ●          (境界で上下に飛ぶ)
    │     /          /
  0 ┤    /          /
    │   /          /
 -π ┤  ●─────────●              ← 突然 +π から -π へジャンプ

unwrap 後 (連続化した角度):
    │           ●
    │         /
    │       /                   ← 一周ぶん持ち上げて滑らかに繋ぐ
    │     /
    │   ●
```

## 手法の系譜と主要論文

`np.unwrap` 自体は単純な信号処理操作ですが、背景には位相アンラップ(phase unwrapping)という確立した研究分野があります。

位相アンラップの基礎。`[−π, π)` に折り畳まれた位相(wrapped phase)から本来の連続位相を復元する問題で、Itoh(K. Itoh, "Analysis of the phase unwrapping algorithm", Applied Optics, 1982)が一次元の基本アルゴリズムを定式化しました。動機は、干渉計測・合成開口レーダ(SAR)・MRI などで、測定器が原理的に位相を 2π でしか区別できない(折り返してしまう)ため、連続量を取り戻す必要があったことです。`np.unwrap` はこの一次元アンラップの実装です。ヨー角の連続化は、まったく別分野で生まれた道具を運転データに転用した例といえます。

二次元・多次元への拡張。Ghiglia & Pritt("Two-Dimensional Phase Unwrapping: Theory, Algorithms, and Software", Wiley 1998)が、画像全体の位相を整合的にアンラップする手法を体系化しました。一次元では単純な逐次補正で済みますが、二次元ではノイズや特異点(residue)があると経路依存になり難しく、最小ノルム法やネットワークフロー法が研究されています。自動運転のヨー角は時系列の一次元問題なので、この難しさには直面しません。

ロボティクス姿勢推定での慣行。角度を連続に扱うのは姿勢推定・オドメトリの基本で、角度のラップを無視すると軌跡推定が破綻することが教科書的に知られています(SLAM やデッドレコニングの実装で標準的に角度の連続化が行われる)。本処理はこの基礎を素直に踏襲したものです。

回転表現の連続性。Zhou ら("On the Continuity of Rotation Representations in Neural Networks", CVPR 2019, arXiv:1812.07035)は、オイラー角や四元数がニューラルネットの回帰対象としては不連続性を持ち学習を難しくすることを示し、連続な 6 次元表現を提案しました。ヨー角の ±π 問題はこの不連続性の最も単純な一例で、後述のように `(cos θ, sin θ)` 表現で根本回避できます。

## 論文の実験結果(定量データ)

unwrap 単体のベンチマーク論文はありませんが、効果と限界を定量的に説明できます。

補正効果の大きさ。境界をまたぐ 1 ステップでのヨーレート誤差は、補正前で約 2π/dt に達します。dt=0.1s なら約 62.8 rad/s(約 3600度/s、一周/0.1 秒という非物理的な値)。unwrap 後はこれが実際の微小変化(例 0.2 rad/s 程度)に戻ります。つまり該当ステップの誤差を 2〜3 桁削減します。この 1 点の外れ値が学習データに混ざると、SmoothL1 のような頑健な損失でも勾配を大きく乱すため、除去の価値は高い。

エイリアシングの限界(サンプリング定理)。unwrap が正しく機能する前提は「1 ステップあたりの真の角度変化が π 未満」であることです。これはナイキスト条件(信号を正しく復元するには最高周波数の 2 倍以上で標本化が必要)の角度版で、もし 1 ステップで π を超える本物の急回転があると、unwrap はそれを「2π の折り返し」と誤判定して逆に補正をミスします。自動運転では、最も急なケースでも 10Hz サンプリングで 1 ステップ 0.1 秒の間に半回転(180度)することはまずなく(それには 900度/s 超のヨーレートが必要)、実害は小さいと評価できます。車両のヨーレートは通常の運転で数十度/s、スピン時でも数百度/s 程度です。

ノイズの影響。角度ノイズが大きく ±π 付近をちらつく(本物の境界またぎではなくノイズで行き来する)場合、unwrap が過剰に持ち上げて累積ドリフト(系列がどんどん上または下へずれていく)を起こす可能性があります。対策として unwrap の前に軽い平滑化をかける、しきい値を調整するなどが取られます。これは「平滑化を先に強めると本物の急変も鈍る」という [egomotion信号の導出（速度・加速度・ヨーレート・曲率）](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0813_egomotion-signal-derivation) と同じトレードオフです。

## メリット・トレードオフ・限界

メリット:
- ±π 境界での偽の角速度スパイクを除去でき、ヨーレート・曲率が物理的に妥当になる。曲率の汚染連鎖も断てる。
- 一行で済み、計算コストはほぼゼロ(O(T) の一回走査)。
- 「差分・微分の前に通す」という明快な定石で、保守しやすく実装ミスが起きにくい。

トレードオフ・限界:
- サンプリング不足のエイリアシング: 1 ステップで π を超える真の急回転があると誤補正する。10Hz の車載では実害は小さいが、低レート記録や激しい挙動では注意。
- ノイズによる過剰補正: 角度ノイズが ±π 付近でちらつくと過剰に持ち上げる。必要なら事前平滑化。
- 順序依存: 差分の前に置かないと意味がなく、順序ミスは静かなバグになる(エラーは出ないが値が壊れる)。コードレビューで見落とされやすい典型例。
- 未解決寄りの論点として、二次元・多次元の位相アンラップは依然として難問だが、時系列ヨー角は一次元なのでその難しさは関係しない。むしろ実務の課題は「ノイズ平滑化と unwrap の順序・組み合わせをどう設計するか」に集約される。

## 発展トピック・研究の最前線

角度を直接扱わない表現。ヨー角の ±π 問題を根本回避するために、角度そのものではなく `(cos θ, sin θ)` の 2 成分で向きを表す手法が機械学習で広く使われます。これなら境界の不連続が存在せず、ニューラルネットの回帰対象としても滑らかです。向きを復元するときは `atan2(sin, cos)` を使えばよい。Zhou ら(CVPR 2019)の回転の連続表現は、四元数やオイラー角が学習で不連続になりうることを指摘し、3 次元回転には 6 次元表現を提案しました。平面のヨー角に限れば 2 次元の `(cos, sin)` がその最小版に当たります。

確率的アンラップ。観測ノイズを考慮し、アンラップを最尤推定やベイズ推定として定式化する研究があります。SAR 干渉計測の分野では、信頼度マップに基づく重み付きアンラップが標準で、ノイズの大きい画素ほどアンラップ判断を慎重にします。

オンライン処理。リアルタイム制御では未来の値を使えないため、因果的(過去のみ)なアンラップと平滑化を組み合わせる必要があり、遅延と精度のトレードオフが研究されています。後処理で全系列を一括 unwrap できるオフライン学習データ作成と、逐次処理しかできない実車制御では、設計が異なります。
