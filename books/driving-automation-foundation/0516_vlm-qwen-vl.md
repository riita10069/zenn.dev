# Qwen-VL

## ひとことで言うと
Qwen-VL は、画像を見て言葉で答えるオープンな大規模視覚言語モデル (VLM) の系列です。高解像度の画像を細かく扱え、中国語・英語など多言語に対応し、画像中の物体の位置 (座標) まで言葉で指し示せる (グラウンディング) のが特徴です。重みが公開されているため、大量の画像/動画に説明・疑似ラベルを「自前で・低コストで・データを外に出さずに」付ける用途で広く使われます。

## 直感的な理解
クローズドな高性能 VLM (API 経由でしか使えないもの) には、(a) 1枚ごとに課金され大量処理が高コスト、(b) 画像を外部に送るため秘匿データ (自社の走行映像など) を出せない、(c) 細かい挙動を制御できない、という制約があります (詳しくは [GPT-4V / GPT-4o](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0514_vlm-gpt4v))。

研究や産業の現場では、数十万〜数百万枚規模の映像フレームに「シーン説明」「危険要因」「行動理由」を付けたいことが多く、API 課金では非現実的です。そこで「自分の GPU 上で動かせる、重みが公開された強い VLM」が必要になります。Qwen-VL はその有力な選択肢として登場しました。

加えて運転シーンのような現実の映像は「遠くの小さな標識」「画面端の歩行者」など高解像度で初めて見える要素が多く、低解像度に潰す VLM では取りこぼします。Qwen-VL 系は高解像度・可変解像度の扱いに力を入れた点が、こうした用途に合います。さらに「歩行者はここ、その座標は (x,y)」と位置つきで答えられるため、説明だけでなく「どの対象が危ないか」を空間的に紐づけた疑似ラベルが得られます。

## 基礎: 前提となる概念
- グラウンディング (grounding): 言葉と画像内の位置を結びつけること。「歩行者はどこ?」に対し座標つきで答えられると、認識結果を後段の処理 (追跡や危険判定) に渡せます。
- 可変解像度 (dynamic resolution): 画像を固定サイズに潰さず、元のアスペクト比 (縦横比)・解像度に応じてパッチ数を変える方式。縦長や横長の画像を歪めずに扱えます。NaViT (Dehghani et al., NeurIPS 2023) が「複数画像のパッチを1列に詰め込んで可変解像度を ViT で扱う」(patch n' pack) 考え方を示し、Qwen2-VL がこれを採り入れました。
- 位置埋め込み (positional embedding): Transformer は系列の順序を本来持たないため、各トークンが「どこにあるか」を別途与える必要があります。RoPE (Rotary Position Embedding、回転位置埋め込み、Su et al. 2021) はその一種で、ベクトルを位置に応じて回転させ相対位置を表現します。
- OCR (Optical Character Recognition、光学文字認識): 画像中の文字を読み取ること。標識やナンバープレートの読解に相当します。
- 疑似ラベル (pseudo-label): 人手でなくモデルが自動で付けた教師ラベル。安価に大量に作れる反面、誤りを含むため品質管理が課題になります。

## 仕組みを詳しく
Qwen-VL の構成は、VLM の標準的な系譜 ([CLIP (画像-言語対比学習)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0513_vlm-clip) の視覚エンコーダ + 橋 + LLM) に沿いつつ、世代ごとに重要な工夫を重ねています。

- 視覚エンコーダ: ViT (初代は OpenCLIP の ViT-bigG を初期化に使用)。画像をパッチ特徴に変換する。
- 橋 (Vision-Language Adapter): 初代は [BLIP-2 (Q-Former)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0512_vlm-blip2) の Q-Former に似た、学習可能なクエリで画像特徴を一定数 (例: 256) に圧縮するクロスアテンション層。大量のパッチを LLM が扱える長さに縮める。後世代では単純な MLP 併合へ寄せ、可変長を扱う。
- LLM: Qwen 言語モデル。多言語 (特に中国語・英語) に強い。

世代ごとに効く工夫を挙げます。

1. グラウンディング: 「歩行者はどこ?」に対し `<box>(x1,y1),(x2,y2)</box>` のように座標つきで答えられる。学習時にバウンディングボックスの座標を文字列トークンとして扱い、通常の言語生成と同じ枠組みで位置を出力させることで実現。どの物体が危険かを位置つきで疑似ラベル化できます。
2. Naive Dynamic Resolution (Qwen2-VL): 画像を固定サイズに潰さず、解像度に応じてパッチ数を可変にする。縦長のカメラ画像を歪めずに処理でき、遠方の小さな標識も読みやすい。可変長になるため、複数画像をまとめる際は NaViT 流のパッキング (詰め込み) で効率化します。隣接する 2×2 のパッチを1トークンに併合してトークン数を抑える工夫も入っています。
3. M-RoPE (Multimodal RoPE、Qwen2-VL): 位置情報を「時間・縦・横」の3軸に分けて表現し、動画フレームの時系列と画像内の 2D 位置を同時に1つの位置エンコーディングで扱える。連続フレーム理解に向きます。1次元の RoPE を多次元へ自然拡張したものです。
4. 動的フレームレート対応・絶対時刻 (Qwen2.5-VL): 動画の時間軸を実時間 (秒) に対応づけ、長尺動画でも「いつ何が起きたか」を秒単位で捉えやすくしました。文書レイアウト解析や画面操作 (エージェント) も強化されています。

具体的な疑似ラベリングの流れ:
- 映像から1フレーム (例: 1280×720) を取り出す。
- プロンプト「この交差点シーンで自車が注意すべき対象と、減速すべきか否かを理由つきで述べよ」を画像とともに入力。
- Qwen-VL が「右前方に横断中の歩行者 (box 座標)。信号は黄。減速すべき」と座標つきで生成。
- これをそのまま教師ラベル (因果説明 + 位置) として下流モデルの補助学習に使う。誤ラベル混入を抑えるため、信頼度や複数モデルの合議でフィルタするのが実務上の定石です。

## 手法の系譜と主要論文
- Qwen-VL (Bai et al., "Qwen-VL: A Versatile Vision-Language Model for Understanding, Localization, Text Reading, and Beyond", 2023, Alibaba, arXiv:2308.12966): ViT 視覚エンコーダ + クエリ圧縮アダプタ + Qwen LLM の構成で、画像理解・物体位置特定・文字読み取りを1モデルで扱えるオープン VLM を提案。動機は「商用 API に頼らず、位置や文字まで扱える高性能 VLM を公開する」。新規性はグラウンディングと OCR を1つのモデルに統合した点。初代は固定解像度 (448px 程度) で超高解像の細部に限界がありました。
- Qwen2-VL (Wang et al., 2024, Alibaba, arXiv:2409.12191): Naive Dynamic Resolution と M-RoPE を導入。動機は「現実の画像・動画は解像度もアスペクト比もバラバラで、固定処理では情報を失う」。高解像度・長尺動画の理解が大きく向上。可変解像度ゆえバッチ処理やメモリ管理が複雑になる代償があります。2B/7B/72B の複数サイズで提供されました。
- Qwen2.5-VL (Bai et al., 2025, Alibaba, arXiv:2502.13923): 長尺動画の絶対時刻対応、文書/レイアウト解析、画面操作などエージェント的能力を強化。視覚エンコーダにウィンドウアテンション (局所領域内に注意を限定し計算を抑える機構) を導入して高解像度を効率化。実応用での疑似ラベリング基盤としての採用が広がりました。大型版 (72B) は自前推論でも相応の GPU を要します。
- 関連: NaViT (Dehghani et al., NeurIPS 2023, Google) は「パッチ列を詰めて可変解像度・可変アスペクト比を ViT で扱う」基礎を与え、Qwen2-VL の動的解像度の理論的背景になっています。RoPE (Su et al., 2021) は M-RoPE の土台です。

## 論文の実験結果 (定量データ)
- 初代 Qwen-VL: 同規模のオープン VLM に対し、グラウンディング (RefCOCO 系の参照表現位置特定) と OCR/文書理解 (DocVQA, TextVQA, ChartQA など) で当時の上位の精度を報告しました。RefCOCO の精度は「指示された対象を正しく囲めた割合」で、言葉と位置の対応の正確さを測ります。
- Qwen2-VL: 動的解像度の導入で DocVQA や InfoVQA など高解像度を要する文書系ベンチが大きく伸び、当時のオープン VLM で上位の総合性能を示しました。72B 版は MMMU (大学レベルの多分野マルチモーダル推論ベンチ) など総合ベンチで GPT-4o 等の上位クローズドモデルに迫る数値も報告されています (評価条件依存)。長尺動画ベンチでも、固定フレーム数に縛られない設計が効いて改善しました。
- Qwen2.5-VL: 72B 版が文書理解・図表理解・動画理解・グラウンディングの複数ベンチで、同時期のオープン VLM の中で上位に位置づけられたと報告されています。報告によれば、いくつかの文書/OCR 系ベンチでは当時のクローズドな上位モデルに匹敵または上回る数値も示されました (評価条件依存のため断定は避けます)。
- アブレーション的知見: 入力解像度を上げる (固定→動的) ことで、小さな文字や遠方物体を含むタスクの精度が一貫して向上することが各世代で示されています。逆に解像度を絞るとトークン数は減って軽くなるが、細部タスクで顕著に落ちます。M-RoPE の導入は動画ベンチで効果が確認され、時間・空間を分けて位置を表すことの有効性が示されました。これらは VLM 全般に通じる「解像度=情報量=計算コスト」のトレードオフを具体的に裏づけるものです。

## メリット・トレードオフ・限界
- メリット: 重みが公開されており自前 GPU で動かせる。API 課金なしで大量フレームに疑似ラベルを付けられ、秘匿データを外部に出さずに済む。複数のモデルサイズから計算予算に合わせて選べる。
- メリット: 高解像度・可変解像度に対応し、遠方の標識や画面端の小さな対象など細部に強い。
- メリット: グラウンディング (座標つき回答) ができ、対象の位置まで疑似ラベル化できる。多言語対応。
- トレードオフ: 大型版は自前推論でも相応の VRAM・スループットを要し、数百万フレームの処理には時間とコストがかかる。可変解像度はトークン数が画像ごとに変わるため、スループットの見積もりやバッチ設計が難しくなります。
- トレードオフ: 最強クラスのクローズド VLM と比べると、難しい因果推論や曖昧シーンで精度が落ちることがある。幻覚 (画像にない物をもっともらしく述べる現象) も残る。
- 限界・未解決: 可変解像度・動画対応の機能は実装が複雑で、推論パイプラインの作り込みが必要。疑似ラベルの品質保証 (誤ラベルが下流学習を汚染するリスク) は依然として運用上の課題で、信頼度フィルタや人手検証の併用が現実的です。グラウンディング座標も完全ではなく、混雑シーンでの取りこぼし・誤対応が起きえます。

## 発展トピック・研究の最前線
- オープン VLM 競争: Qwen-VL 系は InternVL、Llama-3.2-Vision、Pixtral、MiniCPM-V などと並ぶオープン VLM の主要系列で、解像度・動画・エージェント能力を軸に競っています。系列を追うと「固定解像度 → 動的解像度 → 動画時刻対応 → 画面操作」という能力拡張の流れが見えます。
- 疑似ラベリングと自己改善: 強い VLM で大量データにラベルを付け、それで小型モデルを学習する蒸留/弱教師の流れは、データが高価な領域 (運転・医療・産業検査) で実用的です。誤ラベルの混入を抑えるための信頼度推定や複数モデルの合議が研究されています。
- トークン効率: 高解像度化はトークン爆発を招くため、画像トークンの圧縮・間引き・解像度の動的選択といった効率化が活発な研究テーマです。Qwen2.5-VL のウィンドウアテンションもこの流れで、車載のような計算制約下では特に重要になります。
- 空間・時間理解の強化: グラウンディングと動画時刻対応は、静的な「何が写っているか」から動的な「いつ・どこで・何が起きたか」への拡張を意味し、運転シーンの危険予測や行動説明と直結します。
