# AdamWオプティマイザの選択

## ひとことで言うと
オプティマイザ(optimizer)とは、モデルの重みを「損失が小さくなる方向に、どれくらい・どう動かすか」を決める更新ルールです。AdamW はその中でも、各重みに合った歩幅を自動で調整しつつ、重みが大きくなりすぎないよう抑える処理(weight decay、荷重減衰)を「数学的に正しい位置」で行う改良版です。現在、Transformer をはじめとする深層モデル学習の事実上の標準オプティマイザになっています。Adam に1行加えるだけの小さな修正ですが、汎化性能(未知データでの性能)が改善するため、広く採用されました。

## 直感的な理解
山を下って谷底(損失最小)を目指す登山者を想像してください。素朴な方法(SGD)は「足元の傾き(勾配)を見て、決まった歩幅でその方向に一歩進む」だけです。これだと、細い谷では左右の壁の急な傾きに反応してジグザグに跳ね回り、谷の軸方向にはなかなか進みません。すべての方向に同じ歩幅を使うのが原因です。

Adam の発想は「方向ごとに歩幅を変える」ことです。よく揺れる方向(左右の壁)は歩幅を小さく、安定して下っている方向(谷の軸)は大きく進む。さらに過去の進行方向の慣性(モメンタム)を持たせて、ジグザグを打ち消し滑らかに進みます。これで谷底へ速くたどり着けます。

AdamW はさらに、登山者が荷物(重みの大きさ)を増やしすぎないよう一定の力で軽くする仕組み(weight decay)を、歩幅調整とは独立に効かせる工夫です。Adam ではこの「荷物を軽くする力」が歩幅調整に巻き込まれて方向ごとにバラついてしまう問題があり、それを切り離して直したのが AdamW です。たった一箇所の引き算の位置を変えるだけ、というのが面白いところです。

## 基礎: 前提となる概念

- 勾配降下法(gradient descent): 損失を各重みで微分して得た勾配の逆方向に重みを動かし、損失を減らす基本手法。
- SGD(確率的勾配降下法): 全データでなくバッチごとの勾配で更新する勾配降下法。「確率的」はバッチ抽出のランダム性を指します。源流は Robbins & Monro(1951)の確率近似法です。
- モメンタム(momentum): 過去の更新方向を慣性として加える工夫。ジグザグを抑え収束を速めます。
- 学習率(learning rate): 1回の更新の歩幅。
- 適応的学習率(adaptive learning rate): 重みごと・方向ごとに歩幅を自動で変える考え方。AdaGrad / RMSProp / Adam がこの系譜。
- 正則化(regularization): 過学習を防ぐため、モデルの複雑さに制約を課す工夫。weight decay はその一種。
- L2正則化: 損失に重みの二乗和 `||w||^2` を足し、重みが大きくなることに罰を与える方法。
- weight decay(荷重減衰): 更新のたびに重みを一定割合だけゼロへ引き戻す方法。
- モーメント(moment): Adam が各重みについて保持する統計量。一次モーメントは勾配の移動平均(向きの慣性)、二次モーメントは勾配の二乗の移動平均(揺れの大きさ)。
- 汎化(generalization): 訓練データだけでなく、未知のデータでも良い性能を出すこと。最終的に気にすべきはこちら。

## 仕組みを詳しく

### SGD の弱点

SGD の更新は単純です。重み `w`、勾配 `g`、学習率 `lr` として `w ← w - lr * g`。弱点が2つあります。

1. すべての重みに同じ歩幅を使うので、頻繁に大きく動く重みと、めったに動かない重みを区別できない。
2. 損失の地形が方向で急・緩と異なると、急な方向に振動して(ジグザグ)、緩い方向への前進が遅い。

この弱点に対し、AdaGrad(Duchi et al., 2011)は「過去の勾配二乗の累積」で歩幅を割り、よく動いた重みの歩幅を下げました。しかし累積は単調増加するため歩幅が際限なく小さくなり学習が止まる欠点があり、RMSProp(Tieleman & Hinton, 2012)が「累積」を「移動平均(指数減衰)」に変えてこれを直しました。Adam はこの RMSProp にモメンタムを加えた、いわば合流点です。

### Adam の更新式(直感)

各重み `w` について、勾配 `g` から次を計算します。

```
m = β1 * m + (1 - β1) * g          # 一次モーメント: 勾配の移動平均(向きの慣性)
v = β2 * v + (1 - β2) * g^2        # 二次モーメント: 勾配二乗の移動平均(揺れの大きさ)
m_hat = m / (1 - β1^t)             # バイアス補正(初期の過小評価を補う)
v_hat = v / (1 - β2^t)
w = w - lr * m_hat / (sqrt(v_hat) + ε)   # 揺れが大きい重みほど歩幅を小さく
```

`sqrt(v_hat)` で割ることで、よく揺れる重み(`v` が大きい)は自動的に歩幅が小さくなります。`m` は過去の向きの慣性で、ジグザグを抑えます。バイアス補正(`m_hat`, `v_hat`)は、`m` と `v` が0から始まるため学習初期に値が小さく出すぎる問題を補正します。たとえば `t=1` のとき `1-β1^1 = 0.1` で割るので、初回の `m` を10倍に補正して立ち上げます。定番値は `β1=0.9, β2=0.999, ε=1e-8` です。`t` は更新ステップ数です。`ε`(イプシロン)はゼロ割りを防ぐ小さな数で、`sqrt(v_hat)` がほぼ0のとき歩幅が発散しないようにします。

### L2正則化 vs weight decay(AdamWの核心)

過学習を防ぐため重みを小さく保ちたいとします。やり方は2つあります。

1. L2正則化: 損失に `(λ/2) * ||w||^2` を足す。これを微分すると、勾配に `λw` が加わる。
2. weight decay: 更新の最後に重みを直接 `(1 - lr*λ)` 倍して縮める。

SGD ではこの2つは数学的に等価です(`λw` を勾配に足して `lr` を掛けて引くのと、`lr*λ*w` を引くのは同じ)。ところが Adam では等価になりません。Adam は勾配を `sqrt(v)` で割って歩幅を調整するため、L2正則化(勾配に `λw` を混ぜる方式)だと、weight decay 由来の項 `λw` まで `sqrt(v)` で割られてしまいます。すると、よく揺れる重み(`v` 大)ほど分母が大きく減衰が弱まり、めったに動かない重み(`v` 小)ほど強く減衰する、という重みごとにバラバラな減衰になります。これは「全重みを均等に縮めたい」という当初の意図とずれた、意図しない挙動です。

AdamW は weight decay を勾配から切り離し(decouple)、適応的歩幅の影響を受けない位置で直接 `w` を縮めます。

```
# AdamW
w = w - lr * (m_hat / (sqrt(v_hat) + ε))   # 適応的な更新(勾配由来)
w = w - lr * λ * w                         # weight decay(適応の外で純粋に縮める)
```

これで全重みが意図どおり均等に減衰し、正則化が正しく効きます。差分はこの引き算を `sqrt(v)` の中に入れるか外に出すか、ただそれだけです。

### 典型的な設定

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-4, weight_decay=1e-2)
```

- `lr=1e-4`: Transformer系の学習で安定しやすい控えめな学習率。warmup と組み合わせるのが定番。
- `weight_decay=1e-2`(=0.01): AdamW の「正しく切り離された」減衰として効きます。値はタスク依存で、0.01〜0.1がよく使われます。

なお実務では、bias 項や正規化層(LayerNorm/BatchNorm)のスケール・シフトパラメータには weight decay をかけないのが通例です。これらは特徴のスケールを調整する役割で、ゼロへ縮めると表現力が無駄に落ちるためです。パラメータをグループ分けして decay の有無を切り替えます(no-decay グループと decay グループ)。

## 手法の系譜と主要論文

- Robbins & Monro(1951)の確率近似法が SGD の理論的源流です。Duchi et al., "Adaptive Subgradient Methods (AdaGrad)"(JMLR 2011)が適応的学習率を導入し、Tieleman & Hinton の RMSProp(2012, Coursera 講義)が移動平均で AdaGrad の歩幅消失を直しました。

- Kingma & Ba, "Adam: A Method for Stochastic Optimization"(ICLR 2015, arXiv:1412.6980)が Adam の原典です。一次・二次モーメントから重みごとの歩幅を適応させる手法を提案し、学習率の手調整負担を大きく減らしました。発表後、深層学習のデフォルトオプティマイザとして爆発的に普及しました。

- Reddi et al., "On the Convergence of Adam and Beyond"(ICLR 2018, Best Paper)は、Adam が一部の単純な凸問題で収束しない反例を構成し、二次モーメントの最大値を保持する修正版 AMSGrad を提案しました。Adam の理論的な穴を突いた重要研究です。

- Loshchilov & Hutter, "Decoupled Weight Decay Regularization (AdamW)"(ICLR 2019, arXiv:1711.05101)が AdamW の原典です。「Adam では weight decay を勾配から切り離すべき」という提案で、前述の通り L2正則化と weight decay が Adam では等価でないことが動機です。これにより Adam の汎化を改善し、SGD との汎化ギャップを縮めました。現在では PyTorch などの実装でも AdamW が標準的に提供され、事実上の標準になっています。

- Wilson et al., "The Marginal Value of Adaptive Gradient Methods in Machine Learning"(NeurIPS 2017, arXiv:1705.08292)は逆の立場から、適応的手法(Adam等)はよく調整した SGD+momentum より汎化が劣る場合があると報告しました。AdamW はこの批判への一つの回答でもあり、「適応性の利便さ」と「汎化」のトレードオフを巡る議論の文脈に位置づけられます。

- 実務面では、Vaswani et al., "Attention Is All You Need"(NeurIPS 2017, arXiv:1706.03762)以降の Transformer 系は Adam/AdamW + warmup が定番です。Swin Transformer や ConvNeXt など現代の画像系backboneの原実装も AdamW を用いており、これらをbackboneにする運転モデルで AdamW を選ぶのは素直な選択です。

- 最近では Chen et al., "Symbolic Discovery of Optimization Algorithms (Lion)"(NeurIPS 2023, arXiv:2302.06675)が、探索で見つけた符号ベースの更新則を提案し、状態を一次モーメントのみに減らしつつ一部タスクで AdamW を上回ると報告しました。Adam/AdamW を超える試みが今も続いています。

## 論文の実験結果(定量データ)

- Adam(Kingma & Ba, 2015): MNIST(手書き数字分類)、CIFAR-10(物体画像分類)、IMDB(感情分析)など複数タスクで、SGD やモメンタム法、AdaGrad、RMSProp より速く・安定して訓練損失が下がることを示しました。とくに「学習率の初期設定にあまり敏感でない」点が実務的な利点として強調されました。

- AdamW(Loshchilov & Hutter, 2019): CIFAR-10 の画像分類(ResNet)と Penn Treebank の言語モデルで、Adam(L2正則化)と AdamW(decoupled weight decay)を比較しました。AdamW は同じ計算量でより良いテスト精度に到達し、かつ学習率と正則化係数を独立にチューニングできる(両者の最適値が互いにほぼ独立になり、ハイパーパラメータ探索が格子状にきれいに分離する)ことを示しました。アブレーションの核心は「weight decay を切り離すだけで汎化が改善する」点で、追加コストはほぼゼロです。この低コスト・高効果が普及の決め手になりました。

- 汎化ギャップ(Wilson et al., 2017): いくつかの画像・言語タスクで、Adam は訓練損失こそ速く下げるが、テスト精度では丁寧に調整した SGD+momentum に劣る場合があると定量的に示しました。たとえば画像分類で数ポイントの差が出る例が報告されています。これは「Adam系が万能ではない」ことの根拠であり、AdamW の decoupled weight decay はこのギャップを縮める方向の改良です。

- メモリコストの定量: Adam/AdamW は各重みについて一次モーメント `m` と二次モーメント `v` を保持します。重み本体に対しオプティマイザ状態が約2倍、混合精度学習ではさらに fp32 のマスター重みも加わり、合わせて重みのおよそ3〜4倍のメモリを消費します。大規模モデルでは、このオプティマイザメモリが学習可能なモデルサイズの制約要因になり、状態を分割する技術(ZeRO/FSDP)や状態を量子化する 8bit Adam が必要になります。

- Lion(Chen et al., 2023): ViT の ImageNet 学習や拡散モデルなどで、AdamW と同等以上の精度を、二次モーメントを持たないぶん少ないメモリで達成したと報告されています。一方で学習率・weight decay の最適レンジが AdamW と異なり、チューニングのやり直しが要る点が指摘されています。

## メリット・トレードオフ・限界

- メリット: 重みごとに歩幅を自動調整するので、学習率の手調整がSGDより楽で、初期から安定して収束しやすい。
- メリット: weight decay が適応的歩幅から切り離され、正則化が意図どおり均等に効く(過学習を抑えやすい)。学習率と weight decay を独立に調整できる。
- メリット: Transformer/CNN を問わず幅広く効き、現代の画像backboneの原実装とも整合する。デフォルト設定でそこそこ動く堅牢さがある。
- トレードオフ: 重みごとに `m` と `v` を保持するため、SGDより約2〜3倍のオプティマイザメモリを消費する。大規模モデルでは無視できない。
- トレードオフ: 学習率と weight decay の相互作用はデリケートで、最適値はタスク依存。スケジューラ込みのチューニングが要る。
- トレードオフ: タスクによっては、丁寧に調整した SGD+momentum のほうが最終的な汎化で勝つことがある(特に大規模画像分類の一部報告)。
- 限界: 適応的手法は局所的な勾配統計に依存するため、勾配ノイズが極端な状況では二次モーメント `v` の推定が不安定になりうる(AMSGrad はこの一例への対処)。
- 限界: バッチサイズや総ステップ数を変えると最適なハイパーパラメータも動くため、「ある設定で良かった値」が別設定で最適とは限らない。

## 発展トピック・研究の最前線

- メモリ効率版: Adafactor(二次モーメントを行・列に因数分解して状態メモリを削減)、8bit Adam(状態を量子化して保持)など、大規模学習向けに状態メモリを減らす派生が広く使われています。
- 新世代オプティマイザ: Lion(符号ベースの更新で状態を一次モーメントのみに)、Sophia(対角ヘッシアン近似を使う軽量な二次的手法)など、Adam を超えることを狙う研究が続いています。いずれも AdamW を強力なベースラインとして比較しています。
- パラメータグループ別の設定: backbone とヘッドで学習率を変える(layer-wise learning rate decay、事前学習済みの層ほど小さい学習率)、正規化・bias を decay 対象から外すなど、グループごとに最適化設定を変える実務テクニックが定着しています。
- 学習率と weight decay のスケーリング則: バッチサイズや学習ステップ数に応じて学習率・weight decay をどう合わせるか(線形スケーリング則など)、理論と経験則の両面で研究が進んでいます。
- 最適化と汎化の理論: 適応的手法がなぜ・どんなときに汎化で劣る/勝つのかを、損失地形の平坦性(flat minima)などの観点から説明しようとする理論研究が続いています。
