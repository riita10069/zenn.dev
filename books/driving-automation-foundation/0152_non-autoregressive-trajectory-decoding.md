# 非自己回帰な軌跡デコーディング

## ひとことで言うと

未来の軌跡を表すウェイポイント(経路上の点)を、1 点ずつ順番に作る(自己回帰)のではなく、あらかじめ用意した「時刻クエリ」を使って全点を一度に並列で出す設計のことです。逐次生成の遅さ(レイテンシ)を解消できる一方、点同士の滑らかさ・一貫性を別の仕組みで担保する必要がある、というトレードオフを持ちます。機械翻訳で生まれた発想が、物体検出(DETR)を経て軌跡予測・運転計画に広がってきました。

## 直感的な理解

文章を書くとき、人間は前の単語を見ながら次の単語を選びます。これが自己回帰(autoregressive)です。前の出力が決まらないと次に進めないので、本質的に逐次的(直列)です。

軌跡の計画も同じことができます。「0.1 秒後はここ、それを踏まえて 0.2 秒後はここ、さらにそれを踏まえて…」と、64 点を 1 点ずつ順に決める。前の点に整合した次の点が出せるので軌跡は滑らかになりますが、64 回の処理が一列に並ぶため遅くなります。

ここで GPU の性質が効いてきます。GPU は「たくさんの計算を同時にやる」のが得意で、「前の結果を待ってから次をやる」直列処理は苦手です。64 回の逐次ループは、まさに GPU の苦手分野を踏み抜いています。自動運転は安全のために 1 回の予測を素早く出す(低レイテンシ)ことが命なので、この逐次ループは将来必ず効いてくるボトルネックになります。

非自己回帰(non-autoregressive)の発想は、「全部の点を最初からまとめて出す」ことです。穴埋め問題で、空欄を 1 個ずつ順に埋めるのではなく、全部の空欄を同時に埋めるイメージ。速い代わりに、埋めた答え同士のつじつま(点と点の滑らかさ)が崩れやすいので、それを補う工夫が要ります。

## 基礎: 前提となる概念

自己回帰(autoregressive)。ある出力を、それより前の出力に条件付けて 1 つずつ生成する方式。`p(y) = Π_t p(y_t | y_<t)` と書け、各 y_t は y_1…y_{t-1} を知ってから決まります。再帰ネットワーク(RNN/GRU/LSTM)や、Transformer のデコーダ(因果マスク付き)がこれにあたります。長所は出力間の整合性、短所は逐次でレイテンシが点数 T に比例すること。

非自己回帰(non-autoregressive)。全出力を互いに独立(または弱い依存)に、並列で同時に生成する方式。`p(y) ≈ Π_t p(y_t | x)` のように各 y_t が前の出力に依存しません。長所は並列でレイテンシがほぼ一定、短所は出力間の依存(整合性)が切れること。

クエリ(query)。Transformer の注意機構における「問い合わせ」。学習可能なベクトルを用意し、それが鍵(key)と値(value)、つまり入力特徴に対して「私が欲しい情報はどこ?」と問い合わせ、関連する特徴を集めてきます。「時刻クエリ」とは「時刻 t の経路点を出すための専用の問い合わせ」を時刻ごとに 1 個ずつ用意したものです。

クロスアテンション(cross-attention)。クエリ(計画側)が、別のソース(BEV 特徴などの環境表現)に注意を向けて情報を集める操作。「今この時刻の経路点を決めるには、地図のどこを見るべきか」を学習で決めます。

レイテンシ(latency)とスループット(throughput)。レイテンシは「1 回の予測にかかる時間」、スループットは「単位時間あたりに処理できる量」。自動運転では 1 台の車が次の制御を素早く得る必要があるため、バッチ 1 のレイテンシが特に重要です。

## 仕組みを詳しく

現状の自己回帰デコーダの中身(C を 256、T を 64 とする)。
```
h = h_0                                  # 初期隠れ状態 [B, 256]
for t in range(64):                      # 64回の直列ループ
    q        = h + ego_query             # 現在のメモリと学習クエリから問い合わせ
    attended = cross_attn(q, BEV)        # BEVの注目箇所を抽出 [B, 256]
    _, h     = GRU(attended, h)          # 前のhが必要 → 並列化できない
    waypoint[t] = head(h)                # この時刻の点 [B, 2]
trajectory = stack(waypoint)             # [B, 64, 2]
```
レイテンシは概ねループ回数 64 に比例します。各ステップの計算自体は小さいのに、64 回直列に待つので時間がかかります。

非自己回帰版のイメージ。
```
time_queries = Embedding(64, 256)        # 学習で決まる [64, 256] のクエリ群
                                         #   各時刻に1個ずつ、固定の問い合わせを持つ
# 64個のクエリを一括でBEVにクロスアテンション(バッチ並列)
attended_all = cross_attn(time_queries, BEV)   # [B, 64, 256] を一度に
waypoints    = waypoint_head(attended_all)     # [B, 64, 2] を一度に
trajectory   = waypoints                       # [B, 64, 2]
```
ポイントは「前の点を待つ」依存を切ること。64 時刻ぶんのクエリを最初から全部用意し、各クエリが独立に BEV へ注目し、独立に経路点を出します。GPU はこれを 1 回(または数回)の行列演算群でまとめて処理でき、レイテンシは点数にほとんど比例しなくなります。

before / after を計算の流れで対比すると、
```
before (AR):  step0 → step1 → step2 → ... → step63   (直列, 待ち時間 ∝ 64)
after  (NAR): [step0, step1, ..., step63] を同時に    (並列, 待ち時間 ∝ 1)
```

問題は、点を独立に出すと軌跡がガタつくことです。時刻間の連続性(滑らかさ・物理的妥当性)を担保する代表的な工夫は次の通りです。第一に、出力に多項式やスプラインを当てはめる(各点を生で出すのではなく、係数を出して滑らかな曲線として復元)。第二に、時刻クエリ間に self-attention を一度通す(点同士が「お互いを見る」ことで弱い依存を回復させる。これは「半自己回帰」とも言える中間策)。第三に、反復改良(iterative refinement、一度並列に出した後、数回だけ全体を見直して修正する。機械翻訳の非自己回帰高速化で実証された方法)。

トレードオフの核心は、自己回帰が「前の点に厳密に整合した次の点」を出せるのに対し、非自己回帰は速い代わりに各点を独立に出すため、点同士の整合を別の工夫で担保しないとガタついた軌跡になりやすい、という点です。速度と整合性のバランスをどこに取るかが設計の勝負どころになります。

## 手法の系譜と主要論文

非自己回帰デコーディングは機械翻訳で生まれ、検出・予測・計画へ広がりました。

Gu et al., ICLR 2018, "Non-Autoregressive Neural Machine Translation"(arXiv:1711.02281)。翻訳で全単語を並列生成する非自己回帰の原典です。動機は「自己回帰翻訳の逐次生成が遅い」。効果は推論速度の大幅な向上(後述の数値参照)。トレードオフは、単語間の依存を切るため品質が落ちやすく、潜在変数(fertility)や知識蒸留、反復改良などの補正が必要になること。軌跡の「点同士の整合が崩れる」問題と本質的に同じ課題で、補正手法の系譜(後続の iterative NAR, CMLM など)がそのまま参考になります。

Carion et al., ECCV 2020, "End-to-End Object Detection with Transformers (DETR)"(arXiv:2005.12872)。固定数の学習済みオブジェクトクエリで、全物体を並列に 1 回で検出します。動機は「NMS(重複検出の後処理)などの逐次・ヒューリスティックをなくし、集合予測を直接学習したい」。各クエリが 1 つの物体を担当し、二部マッチング(Hungarian matching)で予測と正解を最適対応づけして学習します。「学習済み時刻クエリで全ウェイポイントを並列に出す」発想は、この DETR のクエリ機構の軌跡版です。クエリベース並列予測の設計思想の起点として極めて重要です。

MTR(Shi et al., NeurIPS 2022, "Motion Transformer with Global Intention Localization and Local Movement Refinement", arXiv:2209.13508)。motion 予測で、意図点(intention point)をクエリとして固定数の軌跡を並列に出し、局所的に refine する設計。Waymo Open Motion Dataset のチャレンジで上位を取り、クエリベース並列予測が予測精度でも有効だと示しました。

VAD(Jiang et al., ICCV 2023, "Vectorized Scene Representation for Efficient Autonomous Driving", arXiv:2303.12077)と VADv2(Chen et al., 2024, arXiv:2402.13243)。ベクトル化したシーン表現とクエリベースの計画で、高速かつ高性能な end-to-end 運転を実現。VAD はラスタ表現を避けることで速度と精度を両立し、クエリで軌跡を並列に出す設計が低レイテンシ計画に有効だと示しました。

要点は、「並列生成で速くする」のは機械翻訳・検出・予測・計画を横断して確立した方向だが、「出力同士の整合をどう保つか」が常に対価として付いてくる、という普遍的な設計トレードオフです。

## 論文の実験結果(定量データ)

非自己回帰機械翻訳の速度と品質。Gu et al. (2018) は、自己回帰の Transformer に対して推論を約 10〜15 倍高速化したと報告しました(GPU 上、文単位)。一方で翻訳品質を測る BLEU(Bilingual Evaluation Understudy、機械訳と参照訳の n-gram 一致度。高いほど良い)は、補正なしの素朴な非自己回帰では自己回帰より数ポイント低下しました。反復改良や知識蒸留を加えるとこの差は縮まり、後続研究(例えば Mask-Predict, Ghazvininejad et al., 2019)は反復回数を増やすことで自己回帰に肉薄する BLEU を、なお高速に達成しています。ここから読み取れる一般則は、「並列化は大きな速度向上を生むが、整合性の補正にどれだけ手間をかけるかで品質差が決まる」ということです。

DETR の効果。DETR は COCO 物体検出で、当時の高度に最適化された Faster R-CNN と同等の AP(Average Precision、検出の正確さの平均。高いほど良い)を、NMS なし・アンカーなしのシンプルな構成で達成しました。報告では大きな物体で特に強く、固定クエリによる集合予測が成立することを示しました。弱点として収束が遅い(学習に長いエポックが必要)ことが指摘され、後続の Deformable DETR(Zhu et al., ICLR 2021)が変形可能注意(deformable attention)で収束を大幅に速め、小物体精度も改善しました。これは「クエリベース並列予測は強力だが、注意の張り方(全体か局所か)で学習効率が大きく変わる」ことを示し、軌跡計画でも局所的な変形可能注意が有用だと示唆します。

軌跡予測での効果。MTR は Waymo Open Motion Dataset で、mAP や miss rate(予測がどれだけ外れたかの割合。低いほど良い)などの指標でトップ級を記録しました。クエリベースの並列予測が、速度だけでなく予測精度でも自己回帰系に対抗できることを定量的に裏付けています。

指標の意味の補足。BLEU は系列出力の品質、AP/mAP は検出・予測の正確さ、miss rate は外れ率、いずれも「並列化しても品質を保てるか」を測る軸です。これらが自己回帰に肉薄・凌駕するという結果が、「整合性を適切に補えば、非自己回帰は速度と品質を両立できる」という結論を支えます。

## メリット・トレードオフ・限界

メリット。64 回の直列ループが消え、推論レイテンシを大幅に削減できます。これは自動運転の安全に直結します。GPU の並列性を活かせるためハードウェア効率も良く、DETR/VAD/MTR 系の確立した設計に乗れるので実装の指針が豊富です。

トレードオフと限界。第一に、各点を独立に出すため、軌跡の滑らかさ・物理的一貫性を別途担保する必要があります(多項式・スプライン当てはめ、クエリ間 self-attention、反復改良など)。これを怠るとガタついた、運動学的に実行不能な軌跡が出ます。第二に、自己回帰ほど「直前の状態に整合した次点」を素直には出せません。急な分岐や障害物回避のような、強い時刻間依存を要する場面で品質が落ちやすい。第三に、学習済み時刻クエリの数・初期化・注意の張り方など、設計の自由度が増えチューニングが要ります。DETR の収束の遅さが示すように、注意機構の選択(全体注意か変形可能注意か)で学習効率が大きく変わります。第四に、これは下流設計の問題ですが、世界モデルへ意図を渡す設計([プランナの隠れ状態による「計画意図→世界モデル」の受け渡し](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0004_ego-hidden-world-model-handoff))と組み合わせる場合、「逐次の最終隠れ状態」がなくなるため、意図ベクトルの作り方を再設計する必要があります(例えばクエリ群を集約したベクトルを意図とするなど)。

研究上のオープンな問い。(a) 完全並列・半自己回帰・反復改良のどこが運転計画にとって最適なレイテンシ/品質のバランスか、(b) 物理的妥当性(運動学制約)を出力側でどう強制するか、(c) マルチモーダル予測([マルチモーダル軌跡予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0151_multimodal-trajectory-prediction))と並列化をどう同時に成立させるか。

## 発展トピック・研究の最前線

並列計画はマルチモーダル予測と自然に結びつきます。クエリを「時刻 × モード」の 2 次元に拡張すれば、複数の軌跡仮説を全時刻ぶん同時に出せます([マルチモーダル軌跡予測](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0151_multimodal-trajectory-prediction))。また、拡散モデル(diffusion model)による軌跡生成(例えば Diffusion Planner や MotionDiffuser, Jiang et al., CVPR 2023)は、ノイズから軌跡全体を反復的に並列復元する枠組みで、非自己回帰と反復改良のハイブリッドと見なせます。これは「全点を同時に扱いつつ、反復で整合性を回復する」という本トピックの理想形に近く、近年急速に発展しています。

物理的妥当性の担保では、運動学的な自転車モデル(kinematic bicycle model)を出力層に組み込み、ネットワークが制御入力(加速度・操舵)を出し、それを積分して軌跡を得る方式が、並列生成と運動学制約の両立に有効だと知られています。
