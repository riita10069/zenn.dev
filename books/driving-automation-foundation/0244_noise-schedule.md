# ノイズスケジュール (linear/cosine)

## ひとことで言うと
拡散モデルは、きれいなデータに少しずつノイズ(でたらめな値)を足していって最後に完全な雑音にし、その逆をたどって生成します。各ステップでどれだけノイズを足すかを決める「時間割」がノイズスケジュールです。この時間割の形(linear=直線的、cosine=コサイン曲線的)を変えるだけで、生成の品質と効率が大きく変わります。学習も生成も同じスケジュールを使うので、ここを間違えると土台から品質が頭打ちになります。

## 直感的な理解
写真を少しずつぼかしていって最後に真っ白なノイズにする動画を想像してください。逆再生すれば「ノイズから写真が現れる」生成過程になります。問題は「どのくらいのペースでぼかすか」です。

最初の何コマかで一気にぼかすと、残りのコマはほとんど真っ白なまま変化がなく、無駄なコマが続きます。逆に最後だけ急にぼかすと、序盤の細かい違いを学ぶ機会が足りません。理想は「全コマがほぼ均等に意味のある変化を運ぶ」ペース配分です。ノイズスケジュールはこのペース配分そのものであり、各ステップが情報をどれだけ運ぶかを決めます。実際、初期の拡散モデルでは終盤のステップが「ほぼ完全ノイズで変化なし」になっていて無駄が多く、これを直したのが cosine スケジュールでした。

## 基礎: 前提となる概念
拡散モデルの前向き過程(きれいなデータにノイズを足す過程)は、各時刻 t(0 から T)で次の量を使って記述します。

α_t(アルファ)とは、各ステップで「元の信号をどれだけ残すか」の係数です。β_t(ベータ)はその逆で「各ステップで足すノイズの分散」で、α_t = 1 - β_t の関係にあります。

ᾱ_t(アルファバー)とは、時刻 0 から t までの α を全部かけ合わせた累積量で、「時刻 t の時点で元データがどれだけ残っているか」を表します。0 から 1 の値を取り、1 なら元のまま、0 なら完全ノイズです。この ᾱ_t が時刻とともにどう減っていくかの曲線こそがノイズスケジュールです。

便利なことに、途中のステップを1つずつ計算しなくても、ᾱ_t を使えば任意の時刻のノイズ入りデータ x_t を一発で作れます。

```
x_t = sqrt(ᾱ_t) * x_0 + sqrt(1 - ᾱ_t) * ε     (ε は平均0分散1のガウスノイズ)
```

x_0 が元データ、ε がノイズです。ᾱ_t=1 なら x_t=x_0(ノイズなし)、ᾱ_t=0 なら x_t=ε(完全ノイズ)になります。

SNR(signal-to-noise ratio、信号対雑音比)とは、信号の強さとノイズの強さの比で、ここでは SNR(t)=ᾱ_t/(1-ᾱ_t) と定義されます。SNR が大きいほど「元データがよく残っている」、小さいほど「ほぼノイズ」を意味します。スケジュールの良し悪しは、この SNR が時間とともにどう変化するかで議論されます。

対数尤度(log-likelihood)とは、モデルが実際のデータをどれだけうまく説明できるかを表す指標で、大きいほど良いモデルです。確率モデルの基本的な評価軸の一つです。

## 仕組みを詳しく
最初の DDPM(Ho et al., 2020)は β_t を t に対して直線的に増やす linear スケジュールを使いました。β_1=0.0001 から β_T=0.02(T=1000)まで等間隔に増やします。これは動いたのですが、後に次の問題が見つかりました。

- 終盤(t が大きい)で ᾱ_t が一気にゼロ近くまで落ちる。つまり「もうほぼ完全ノイズ」という似た状態が長く続き、その区間のステップが情報をほとんど運ばない。
- 小さい画像(32x32 や 64x64)では特にこの無駄が顕著で、終盤の最大20%程度のステップを単純に飛ばしても品質がほとんど落ちなかった。これは「スケジュールが非効率」という直接の証拠でした。

そこで Improved DDPM(Nichol & Dhariwal, 2021)が cosine スケジュールを提案しました。これは ᾱ_t の形を直接コサインの二乗で定義します。

```
ᾱ_t = f(t) / f(0),   f(t) = cos( ( (t/T + s) / (1 + s) ) * (π/2) )^2
```

s は小さなオフセット(論文では 0.008)で、t=0 付近で β が極端に小さくなりすぎるのを防ぎます。この形だと、序盤と終盤で ᾱ_t がゆるやかに変化し、真ん中あたりで最も速く落ちます。終盤に「ほぼ完全ノイズで情報のないステップ」が長く続く無駄を減らせます。

linear と cosine の ᾱ_t の概形を比べます(t を 0 から 1 に正規化、数値はイメージ)。

```
ᾱ_t
1.0 |*.                linear (*): 序盤で急落、終盤はほぼ0で平坦(無駄)
    | * .              cosine (.): 全体になだらか、終盤も急落しない
0.5 |  * .
    |   *  .
    |    *   .
0.0 |_____*____.______  → t
    0    0.5    1.0
```

数値例で違いを見ます。T=1000 として t=900(後半90%地点)での ᾱ_t は、linear だとすでに 0.0001 のような極小値で「中身がほぼ消えている」のに対し、cosine だと 0.03 程度まだ元データが残り、このステップが情報を運びます。言い換えると、cosine は SNR がゼロに落ちる時刻を後ろにずらし、有効なステップ数を増やします。

Improved DDPM はスケジュール改良に加えて、各ステップの分散 Σ をネットワークに学習させる工夫(従来は固定)も入れ、両者あわせて対数尤度を改善しました。なぜ分散の学習が効くかというと、DDPM では逆過程の各ステップを「平均と分散を持つガウス分布」で近似しますが、平均だけネットワークに当てさせ分散は固定値(β_t か、その変種)で置いていました。終盤の SNR が高い領域では理論的な最適分散がこの固定値から大きくずれるため、固定すると尤度が無駄に悪化します。Improved DDPM は分散を β_t と別の値の対数線形補間としてネットワークに出させ、平均(=ノイズ予測)は従来どおり簡単な二乗誤差で学びつつ、分散だけ変分下界(対数尤度の下界)で学ぶハイブリッド損失にしました。これが尤度改善の主因です。

スケジュールを連続時間で見ると、本質は「SNR(t) という単調減少関数をどう設計するか」に尽きます。linear も cosine も、結局は ᾱ_t すなわち SNR の時間プロファイルの違いでしかありません。後年の Variational Diffusion Models はこの見方を徹底し、学習に効くのはスケジュール曲線の「形」そのものではなく両端の SNR(開始 SNR_max と終了 SNR_min)だけであり、中間の通り方は重み付けを通じてしか影響しないことを理論的に示しました。つまり linear/cosine の優劣は「終端でどこまで SNR を落とすか」「どの SNR 帯に学習リソースを割くか」という2点に帰着するのです。

スケジュールは解像度にも依存します。後の研究で「同じスケジュールでも解像度が高いほどノイズが効きにくい」ことが分かりました。直感的には、高解像度画像は隣り合うピクセルが似ているため、少々のノイズを足しても全体の構造(低周波成分)が壊れにくく、結果として SNR が高めに残るからです。そのため simple diffusion(Hoogeboom et al., 2023)などは、解像度に応じて SNR をシフトしてスケジュールをずらすべきと示しました。スケジュールには唯一の正解があるのではなく、データの種類・解像度・ステップ数に合わせて選ぶ設計要素なのです。

## 手法の系譜と主要論文
- Ho, Jain, Abbeel, "Denoising Diffusion Probabilistic Models" (DDPM), NeurIPS 2020 (arXiv:2006.11239)。linear スケジュールの出典。シンプルで実装しやすいが、終盤の効率が悪いことが後に指摘された。
- Nichol & Dhariwal, "Improved Denoising Diffusion Probabilistic Models", ICML 2021 (arXiv:2102.09672)。本ノートの主役。cosine スケジュールと分散の学習を提案し、対数尤度と少ステップ品質を改善した。
- Song, Meng, Ermon, "Denoising Diffusion Implicit Models" (DDIM), ICLR 2021 (arXiv:2010.02502)。学習済みのスケジュールはそのままに、生成時のステップを間引いて高速化する手法。スケジュール設計とサンプリングは分離できるという視点を与えた。
- Kingma et al., "Variational Diffusion Models" (VDM), NeurIPS 2021。スケジュールを SNR で記述し、SNR を学習可能にして対数尤度を最適化する一般化を示した。スケジュールは「SNR(t) の単調減少関数」として抽象化できることを明確にした。
- Karras et al., "Elucidating the Design Space of Diffusion-Based Generative Models" (EDM), NeurIPS 2022 (arXiv:2206.00364)。スケジュールを時刻 t ではなくノイズ標準偏差 σ で直接表し、スケジュールとサンプラを分離。詳細は [EDM (Elucidating Diffusion Models)](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0233_edm-framework)。
- Hoogeboom, Heek, Salimans, "simple diffusion: End-to-end diffusion for high resolution images", ICML 2023 (arXiv:2301.11093)。高解像度では SNR をシフトすべきと示し、スケジュールが解像度依存であることを明確化した。
- Chen, "On the Importance of Noise Scheduling for Diffusion Models", 2023 (arXiv:2301.10972)。スケジュール選択が品質に与える影響を体系的に検証し、解像度に応じた調整の重要性を実験的に裏付けた。

## 論文の実験結果(定量データ)
Improved DDPM の cosine スケジュールの効果は次のように報告されています。

- ImageNet 64x64 と CIFAR-10 で、cosine スケジュールに切り替えると linear と比べて対数尤度(bits/dim、1次元あたりの符号化に要するビット数。小さいほど良い)が改善しました。bits/dim は「データを圧縮するのに必要な情報量」を測る指標で、モデルがデータ分布を正確に捉えているほど小さくなります。
- 少ないステップ数での生成品質が改善しました。論文では、生成ステップを 1000 から大幅に減らしても、cosine の方が FID(生成画像が本物とどれだけ近いかを測る指標。小さいほど良い)の劣化が緩やかでした。たとえば数十ステップ程度に間引いた条件で、linear より cosine が一貫して低い FID を示しました。
- アブレーションとして、linear スケジュールのまま終盤のステップを飛ばしても品質がほとんど変わらないことを示し、「終盤のステップが無駄である」ことを定量的に裏付けました。これが cosine 導入の動機の直接的な証拠になっています。
- 分散の学習(Σ を固定せずネットワークに出させる)を併用すると、対数尤度がさらに改善し、同じステップ数でより少ない誤差で生成できることが示されました。

解像度依存性については、後続研究が「128px や 256px では低解像度向けに最適化されたスケジュールが過剰にノイズを残す/逆に足りない」ことを定量的に示し、SNR を解像度に比例してシフトすると FID が改善することを報告しています。具体的には、解像度を2倍にしたら SNR を約4倍シフトする、といった経験則が提案されています。

## メリット・トレードオフ・限界
- メリット(cosine): 終盤のステップに無駄が少なく、低解像度データで効率が良い。各ステップが均等に情報を運ぶ。
- メリット(linear): 実装が単純で理解しやすく、計算も軽い。歴史的に多くの実装の標準。
- トレードオフ: cosine も万能ではなく、解像度や次元に依存する。データが変わると再調整が要る。
- トレードオフ: スケジュールは学習・サンプラ・前処理と絡み合うため、単独では最適化しにくい(EDM はこれを σ ベースで分離しようとした)。
- 限界: スケジュールを誤ると、生成ステップ数を増やしても品質が頭打ちになる。逆に良いスケジュールは少ステップでも崩れにくい。
- 限界: 画像向けに発達したスケジュール理論を、軌道や時系列など低次元・異なる相関構造のデータへ移すと、最適点が大きくずれる。低次元データでは「終盤の無駄」より「序盤の構造形成」が重要になることもあり、画像の知見がそのまま通用しない。

研究上の未解決課題は、ドメインやデータ統計に応じてスケジュールを自動的に最適化する方法、SNR の単調性を崩さずに学習信号を均等化する設計、超少ステップ生成と整合的なスケジュールの理論などです。

## 発展トピック・研究の最前線
スケジュールは「SNR(t) の単調減少関数」として一般化でき、VDM 以降は SNR を直接設計対象とする見方が主流です。EDM はさらに進んで、時刻ではなくノイズ σ そのものを座標にし、学習時のノイズ分布(log-normal)と生成時の刻み方(ρ=7 のべき乗刻み)を分けて設計しました。Flow Matching や Rectified Flow の系統では、ノイズとデータを直線で結ぶことが事実上のスケジュール選択に相当し、[Stochastic Interpolants](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0252_stochastic-interpolants) がこれらを補間子 α(t)・β(t)・γ(t) の選び方として統一的に説明します。近年は、学習後にスケジュールを事後的に最適化する、あるいはタスク(分類器ガイダンスや条件付け強度)に応じてスケジュールを動的に変える研究も進んでいます。
