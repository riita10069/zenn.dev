# long-tail クラス重み付け

## ひとことで言うと
学習データの中で「めったに出てこない種類のサンプル」を、モデルがちゃんと学べるように特別扱いする仕組みです。具体的には、損失(予測の間違いの大きさを表す数値)を計算するときに、珍しいクラスを間違えたときのペナルティを大きくし、よく出てくるクラスのペナルティを小さくします。これをクラスごとの重み(per-class weight、クラス重み)と呼びます。たった1行で実装できる一方、後述するように「重みを付けすぎると逆に壊れる」という奥の深い問題を抱えていて、長く研究が続いてきたテーマです。

## 直感的な理解

機械学習のモデルは、「与えられた学習データ全体で、平均的に損失が小さくなる」方向に学習します。ここで「損失」とは、モデルの予測がどれくらい間違っているかを表す数値で、これが小さいほど正解に近いという意味です。学習とは、この損失を少しずつ小さくしていく作業のことです。

ここで問題になるのが、現実のデータは「種類ごとの数(出現頻度)が大きく偏っている」ということです。試験勉強にたとえてみます。過去問が1000問あって、そのうち900問が「足し算」、残り100問が「微分・積分・行列・確率…」と多種多様に散らばっているとします。総合点を最大化したい受験生は、自然と足し算ばかり練習し、それだけで90点取れてしまいます。微分を1問落としても総合点はほとんど下がらないので、難しくて数の少ない問題は後回しになります。モデルもまったく同じ挙動をします。

このような「数本の種類が突出して多く、残りの大量の種類が低く長く尾を引く」分布を long-tail(ロングテール、長い尾)分布と呼びます。グラフにすると右側に細く長い尾を引くため、この名前が付いています。

```
出現頻度
 │█
 │█
 │█ █
 │█ █ █
 │█ █ █ █ ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁ ▁  ← 長い尾(tail)
 └───────────────────────────────→ クラス
  頻出クラス(head)    希少クラス(tail)
```

自動運転で具体的に考えます。走行シーンを「交差点」「歩行者」「信号」「障害物」「通常走行(何も特別なことがない)」のように分類するタスクを考えると、実際に車を走らせて記録した時間の大半は「通常走行」になります。たとえば10万シーンを集めて、こんな内訳になったとします。

```
通常走行  : 70,000 (70%)
信号      : 15,000 (15%)
交差点    :  9,000 ( 9%)
歩行者    :  5,000 ( 5%)
障害物    :  1,000 ( 1%)   ← 希少 (long-tail の尾の先)
```

この状態でそのまま学習すると、モデルは「とりあえず全部 通常走行 と答えておけば70%は当たる」という安易な戦略に落ち着きやすくなります。損失の合計はサンプル数の多いクラスの寄与で支配されるため、障害物のような1%しかないクラスを多少間違えても、合計の損失はほとんど動きません。

ところが自動運転では、この「希少だが致命的」なケースこそ正しく扱えないと困ります。事故の多くは、めったに起きない状況で起こるからです。だからこそ、頻度の偏りを人工的に補正して、希少クラスにも学習の力を割り当てる必要があります。

## 基礎: 前提となる概念

ここで使う基本道具を、知らない人向けに噛み砕きます。

クロスエントロピー損失(cross-entropy loss): 分類問題で最もよく使う損失です。モデルが「正解クラスにどれだけ高い確率を割り当てたか」を測ります。1サンプルあたり次のように計算されます。

```
loss_i = -log( p[正解クラス] )
```

ここで p は、モデルが softmax(出力の生スコアを合計1の確率に変換する関数)を通して出した各クラスの確率です。正解クラスの確率 p が1に近ければ `-log(p)` は0に近づき(よくできました、ほぼ損失なし)、p が小さい(自信を持って間違えた)と `-log(p)` は大きくなります。たとえば p=0.9 なら loss≈0.105、p=0.1 なら loss≈2.30 です。

softmax: モデルの最終層は通常、各クラスへの生のスコア(logit、ロジットと呼ぶ)を出します。softmax はそれを `exp(logit) / Σexp(logit)` で正規化し、すべて足すと1になる確率に変えます。

クラス不均衡(class imbalance): クラスごとのサンプル数が大きく違う状態。上記の long-tail はその極端な形です。不均衡率(imbalance ratio)= 最頻クラス数 / 最希少クラス数 で表し、上の例では 70000/1000 = 70 になります。研究用の難しいベンチマークでは 100〜1000 に達します。

これらを踏まえると、クラス重み付けは「クロスエントロピーに、クラスごとの掛け算の係数 w を1つ足すだけ」の素朴な手法だと理解できます。

## 仕組みを詳しく

per-class weight を入れると、クロスエントロピーに正解クラスに応じた重み w が掛かります。

```
loss_i = w[正解クラス] × ( -log( p[正解クラス] ) )
```

重みの決め方の代表例が「逆頻度(inverse frequency)重み付け」です。出現頻度が低いクラスほど大きな重みを与えます。先ほどの例で頻度の逆数を取り、いちばん多い通常走行を基準(1.0)に正規化すると、こうなります。

```
障害物    : 70000/1000  = 70.0  → 強く upweight (重みを上げる)
歩行者    : 70000/5000  = 14.0
交差点    : 70000/9000  ≈  7.8
信号      : 70000/15000 ≈  4.7
通常走行  : 70000/70000 =  1.0
```

数値例で効果を確認します。障害物のシーンでモデルが誤って通常走行と予測し、正解である障害物の確率を p=0.1 と低く見積もったとします。

- 重みなし: loss = -log(0.1) ≈ 2.30
- 重みあり(w=70): loss = 70 × 2.30 = 161.0

同じ間違いでも、障害物を外したときのペナルティが70倍に膨らみます。学習はこの大きな損失を減らそうとするので、自然と希少クラスを正しく当てる方向に圧力がかかります。

ただし70倍はやり過ぎで、学習が暴れて頻出クラスの精度が逆に落ちることが多いです。そこで実務では緩める工夫をします。

平方根による緩和: 重みを `sqrt(逆頻度)` にすると、上の例で障害物は √70 ≈ 8.4 倍に下がり、極端さがやわらぎます。

median frequency balancing(中央値頻度バランシング): Eigen & Fergus (ICCV 2015) が画像のシーンラベリングで使った方法で、`重み = 全クラス頻度の中央値 / そのクラスの頻度` とします。中央値で割るので、最頻でも最希少でもない「真ん中のクラス」の重みが1になり、バランスが取りやすくなります。

テンソルの形で言えば、クラス数を C とすると重みは形状 `[C]` の1次元ベクトルです。多くの深層学習フレームワークのクロスエントロピー実装は `weight` 引数を持ち、内部で「各サンプルの損失に w を掛け、最後に重みの合計で割って平均する」正規化をしてくれるので、バッチ全体の損失スケールが極端に跳ね上がりにくいよう調整されます。

```
入力 logits : [B, C]   (B=バッチサイズ, C=クラス数)
正解ラベル  : [B]      (各サンプルの正解クラス番号 0..C-1)
クラス重み  : [C]      (例: [1.0, 4.7, 7.8, 14.0, 70.0])
出力 loss   : スカラー (重み付き平均)
```

なぜ「重みを掛ける」だけで効くのかを少し深掘りします。学習は損失の勾配(gradient、損失を小さくするためにパラメータをどちらへ動かすべきかの向きと大きさ)に沿ってパラメータを更新します。クラス重み w を掛けると、そのクラスのサンプルから来る勾配が w 倍になります。つまり「希少クラス1サンプルが、頻出クラス w サンプル分の学習圧力を持つ」ようになり、実質的にデータをオーバーサンプリング(希少クラスを水増しする)したのと近い効果が得られます。

## 手法の系譜と主要論文

クラス重み付けと long-tail 学習は、20年近い研究の蓄積があります。系譜を追います。

1. 逆頻度・median frequency balancing(2015年前後)。Eigen & Fergus (ICCV 2015) のシーンラベリング、Badrinarayanan et al. の SegNet (TPAMI 2017) などセマンティックセグメンテーションで広く使われました。考え方は本ノートの説明そのままで、頻度の逆数(または中央値で割った値)を重みにする。実装が簡単で効果も出やすい一方、極端に希少なクラスに巨大な重みが付くと学習が不安定になる弱点が早くから指摘されました。

2. Focal Loss(Lin et al., "Focal Loss for Dense Object Detection", ICCV 2017、RetinaNet)。物体検出で「背景(negative)が圧倒的に多く、検出すべき物体(positive)が少ない」という極端な不均衡(1画像で前景:背景が 1:1000 を超える)を解く損失として提案されました。固定のクラス重みに加えて、すでに正しく自信を持って分類できているサンプルの損失を `(1 - p)^γ` という係数(γ は2程度)で自動的に下げ、まだ間違えている難しいサンプルに学習を集中させます。固定の頻度重みより賢く「難易度」で重み付けする点が新しく、研究上の転換点になりました。詳細は [Focal Loss](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0385_focal-loss)。

3. Class-Balanced Loss(Cui et al., "Class-Balanced Loss Based on Effective Number of Samples", CVPR 2019)。「単純な逆頻度だと、サンプルが増えるほど情報が重複して『実質的な有効サンプル数』は頭打ちになる」という観察から出発します。有効サンプル数 `(1 - β^n) / (1 - β)`(n はサンプル数、β は0.99〜0.9999程度)の逆数を重みにすることで、逆頻度より穏やかで安定した重み付けを実現しました。β→0 なら全クラス均等(重みなし)、β→1 なら逆頻度に近づく、という連続的な調整ができます。

4. LDAM Loss + DRW(Cao et al., "Learning Imbalanced Datasets with Label-Distribution-Aware Margin Loss", NeurIPS 2019)。重みを掛けるのではなく、希少クラスほど「決定境界までのマージン(余裕)を大きく要求する」損失を提案。マージンを `C / n_j^(1/4)` に比例させ(n_j はクラス j のサンプル数)、希少クラスほど境界から離れて分類するよう促します。さらに重要な知見として Deferred Re-Weighting(DRW、遅延再重み付け)を提案しました。これは「学習の前半は重みなしで普通に学習し、後半になってから重み付けを効かせる」二段階戦略です。重み付けを最初から強くかけると特徴抽出(表現学習)が崩れる、という発見は実務で極めて重要です。

5. Decoupling(Kang et al., "Decoupling Representation and Classifier for Long-Tailed Recognition", ICLR 2020)。LDAM の知見をさらに推し進め、「表現(特徴抽出器)と分類器(最後の線形層)を分けて学習すべき」と主張しました。実験から、特徴抽出器は素朴に(重みなし・通常サンプリングで)学習したほうが良い表現が得られ、最後の分類器だけをクラスバランスを取って再学習すれば十分、と示しました。これは「損失の重み付けは表現学習を歪めるリスクがある」という前項の懸念を裏付ける重要な発見です。

6. Logit Adjustment(Menon et al., "Long-Tail Learning via Logit Adjustment", ICLR 2021)。学習ではなく推論時(または損失計算時)に、各クラスの logit を `log(クラス事前確率)` だけ補正するという、理論的に裏付けのある方法を提案しました。ベイズ最適な分類器に対応することを示し、追加の再学習なしで強力なベースラインになることから、現在のlong-tail研究の標準的な比較対象になっています。

この系譜は、「素朴な逆頻度重み付け」が最もシンプルな起点であり、(a) 難易度で動的に重み付けする(Focal)、(b) 有効サンプル数で緩める(Class-Balanced)、(c) 重みではなくマージンや段階的適用で表現を守る(LDAM/DRW)、(d) 表現と分類器を分離する(Decoupling)、(e) 事前確率で logit を補正する(Logit Adjustment)、という多方向の改良へ進んできたことを示しています。

## 論文の実験結果(定量データ)

long-tail研究では、人工的に不均衡にしたベンチマークで比較するのが定番です。指標と数値を初心者向けに説明します。

評価指標: 主に top-1 accuracy(最も確率の高いクラスが正解と一致した割合)を使います。重要なのは、全体精度だけでなく、クラスを Many(頻出)/Medium(中)/Few(希少)の3グループに分けて、それぞれの精度を報告することです。なぜなら全体精度が高くても、Few(希少)が壊滅していたら long-tail対策としては失敗だからです。

代表的なベンチマークとおおよその報告値(論文・後続のサーベイによる)を挙げます。

- CIFAR-100-LT(CIFAR-100 を不均衡率100で人工的に偏らせたもの、100クラス): 素朴なクロスエントロピー(重みなし)の top-1 はおよそ 38〜39%。Class-Balanced Loss(Cui et al., 2019)はおよそ 39〜40% で、緩めた重み付けによってわずかに改善。LDAM+DRW(Cao et al., 2019)はおよそ 42〜43% に達し、特に「学習後半でだけ重みを効かせる」DRW を加えると、DRW なしの LDAM 単体(およそ40%)より2〜3ポイント上がると報告されています。これは「重みをいつ効かせるか」というタイミングだけで数ポイント変わる、という long-tail学習の繊細さを示す代表的な数値です。

- ImageNet-LT(ImageNet を long-tail化、1000クラス、不均衡率約256): 素朴なクロスエントロピーは全体 top-1 がおよそ 41〜45% ですが、Few(希少クラス)の精度は一桁台まで落ち込みます。Decoupling(Kang et al., 2020)の代表手法 cRT/LWS は全体でおよそ 47〜50% に上がり、特に Few グループの精度を大きく底上げします。Logit Adjustment(Menon et al., 2021)も追加学習なしで同程度の改善を報告しています。

- iNaturalist 2018(実世界の生物種、約8000クラス、自然な強い long-tail): Class-Balanced Loss は素朴なベースラインに対して数ポイントの改善を示しました。Decoupling 系はさらに高く、特に希少種の認識を改善します。

アブレーション(どの要素を抜くとどうなるか)の代表的な知見:

- Cao et al. (LDAM) は「重み付けを最初から強くかける(DRW なし)」と、特徴抽出器が早期に希少クラスへ過剰適合して表現が劣化し、最終精度が下がることを示しました。重み付けのタイミングを後半に遅らせる(DRW)だけで回復します。
- Kang et al. (Decoupling) は「特徴抽出器をクラスバランス・サンプリングで学習する」と、むしろ表現の質が落ちて全体精度が下がることを示しました。表現学習は素朴なサンプリングが良く、調整は最後の分類器だけにするのが効く、という反直感的な結果です。
- Focal Loss(Lin et al.)のアブレーションでは、難易度の係数 γ を0(つまり重みのみ)にすると効果が大きく薄れ、γ=2前後で物体検出の AP(平均適合率)が最大になることが示されました。

これらの数値が伝える本質は、「重みを付けるか否か」よりも「どう緩めるか・いつ効かせるか・どこに効かせるか」が精度を左右する、ということです。

## メリット・トレードオフ・限界

メリット

- 実装が「クロスエントロピーに weight 引数を1つ渡すだけ」で済み、追加の計算コストがほぼゼロ。
- 希少だが安全上重要なクラス(障害物・歩行者など)の学習を促進できる。
- ハイパーパラメータが「クラスごとの重み」だけで直感的に理解・調整しやすい。
- データを物理的に増やす(オーバーサンプリング)のと違い、メモリやデータ準備の手間が増えない。

トレードオフ・限界

- 重みを大きくしすぎると損失が暴れて学習が不安定になり、頻出クラスの精度が逆に落ちる(LDAM 論文の指摘)。逆頻度をそのまま使わず、平方根・Class-Balanced で緩める、DRW で後半だけ効かせる、のが定石。
- 重み付けは表現学習そのものを歪めうる(Decoupling の知見)。表現は素朴に学び、分類器だけ調整するほうが良いケースが多い。
- 頻度の偏りは補正できても、希少クラスの「サンプルが種類として少ない・多様性が乏しい」問題は解決しない。根本的にはデータ収集・データ拡張・合成データが必要。
- ラベルがノイズを含む場合(例: 人手でなく自動付与した疑似ラベル)、希少クラスを強く upweight すると「間違ったラベルを強く学んでしまう」リスクがある。[ラベルスムージング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0357_label-smoothing) の併用で自信過剰な学習を抑えるのが定石。
- 評価指標との不一致に注意。重み付けは学習時の損失を変えるだけで、最終的に何を最適化したいか(希少クラスの見逃しを減らしたいのか、全体精度を上げたいのか)を明確にしないと、付けた重みが目的に合わない。

未解決の課題: クラス重み付けは「クラスが明確に分かれている」前提に立ちますが、現実のシーン認識では「交差点かつ歩行者あり」のように複数要因が同時に成立する(マルチラベル)場合が多く、単一クラスへの重み付けでは扱いきれません。また、テスト時のクラス分布が学習時と違う(分布シフト)場合、学習時に最適化した重みが推論時に最適でなくなる問題も残っています。Logit Adjustment 系がこの方向に取り組んでいますが、決定打はまだありません。

## 発展トピック・研究の最前線

- 二段階・分離学習: Decoupling 以降、「重い表現学習」と「軽い分類器調整」を分ける流れが主流になりました。MiSLAS、Bag of Tricks for Long-Tailed Recognition など、複数の小技を組み合わせる研究が続いています。
- 対比学習との融合: 自己教師あり・対比学習(contrastive learning)で得た表現は long-tail に強いという報告があり、Balanced Contrastive Learning などが提案されています。
- 損失ではなくサンプリング・データ側: 重み付けの代わりに、希少クラスを水増しするオーバーサンプリング、頻出クラスを間引くアンダーサンプリング、希少クラスの特徴を合成する手法(feature space augmentation)など、データ側からのアプローチも活発です。
- 検出・セグメンテーションへの展開: 物体検出やセグメンテーションでは「クラスの不均衡」と「前景・背景の不均衡」が両方効くため、Equalization Loss、Seesaw Loss など専用の損失が提案されています。
- 知識蒸留との組み合わせ: 重いモデル(あるいは言語モデルなどの外部知識源)に希少シーンの判断をさせ、その出力を軽量モデルへ蒸留して希少クラスの認識を底上げする、という方向も自動運転で注目されています。
