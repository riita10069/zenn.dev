# F.grid_sample による可搬な変形アテンション

## ひとことで言うと

「変形アテンション（deformable attention）」は、画像のすべての場所を見る代わりに、クエリ（問い合わせ）が自分で指定した少数の位置だけを見る効率的なアテンション機構です。これを高速に動かす標準的な方法は、GPU 専用の手書きプログラム（カスタム CUDA カーネル）を用意することですが、その中核である「任意の小数座標の特徴を補間して取り出す」操作は、ディープラーニングフレームワークに最初から備わっている格子サンプリング関数（PyTorch なら `F.grid_sample`）でほぼ同じことが書けます。速度と表現力はやや落ちますが、追加コンパイルなしにどの環境（CPU でも GPU でも、どの CUDA バージョンでも）でも動く「移植性（portability）」が手に入ります。本稿はこの近似実装の考え方・仕組み・限界を、前提知識ゼロから研究者レベルまで掘り下げます。

## 直感的な理解

普通のアテンション（attention、注意機構）は、ある問い合わせに対して「候補すべて」との関連度を計算し、その重みで候補の中身を平均します。文章なら全単語、画像なら全画素が候補です。候補数を N とすると計算量は N に比例（自己注意なら N の二乗）し、高解像度画像では N が数万〜数十万になって計算が爆発します。

ところが現実を観察すると、ほとんどの問い合わせは「全部」を見る必要がありません。「この物体の輪郭はどこか」を知りたいクエリは、物体の周囲の数点さえ見れば十分です。変形アテンションのアイデアはここにあり、「全画素を見る代わりに、クエリ自身が『どこを見るべきか』を予測し、その数点（典型的には 4 点や 8 点）だけにアテンションを集中させる」ものです。「変形（deformable）」という名前は、見る位置（サンプル点）が固定の格子ではなく、入力に応じて自由に動く（変形する）ことに由来します。

身近なたとえで言えば、本棚から情報を探すとき、全部の本を1ページずつ読む（全画素アテンション）のではなく、目次を見て「だいたいこの数冊のこのあたり」と当たりをつけて開く（変形アテンション）ようなものです。後者は圧倒的に速く、しかも当たりの付け方（どこを開くか）自体を経験から学習できます。

では、なぜ「自由に動く小数座標の特徴を取り出す」のに専用 CUDA が要るのでしょうか。サンプル点は x=12.3, y=45.7 のような連続値で、画像の整数ピクセル位置とはずれます。この中間位置の値を周囲のピクセルから補間し、さらにその操作を「微分可能（勾配が流せる）」にして、多数の点を高速に処理する——これを効率的に行うために原典実装はカスタム CUDA を書きます。`F.grid_sample` は、まさにこの「微分可能な格子サンプリング」をフレームワーク標準で提供しており、専用 CUDA を書かずに同じ中核操作を実現できるのです。

## 基礎: 前提となる概念

**アテンション機構**: クエリ（query, 問い合わせベクトル）、キー（key, 各候補の見出しベクトル）、バリュー（value, 各候補の中身ベクトル）の3つを使います。クエリとキーの類似度から重みを計算し、その重みでバリューを加重平均します。Transformer（Vaswani et al., NeurIPS 2017, "Attention Is All You Need"）がこの定式化を確立しました。変形アテンションでは、キーとの類似度計算を省略し、アテンション重みをクエリから直接予測してしまう点が大きな簡略化になっています（後述）。

**双線形補間（bilinear interpolation）**: 整数格子の間にある小数位置の値を、周囲4ピクセルの距離に応じた重み付き平均で滑らかに推定する方法です。たとえば x=12.3 は、x=12 のピクセルを 0.7、x=13 のピクセルを 0.3 の重みで混ぜます。2次元なら、左上・右上・左下・右下の4ピクセルを使います。重要なのは、この補間が「サンプル位置 x, y について微分可能」であることです。つまり「もう少し右を見たら誤差が減る」という勾配が計算でき、見る位置そのものを学習で動かせます。数式で書くと、サンプル位置 (x, y) に対し f(x,y) = Σ w_i(x,y) · I_i となり、重み w_i が (x,y) の連続関数なので ∂f/∂x が定義できます。

**`F.grid_sample`**: 入力テンソル（特徴マップ）`input` と、サンプルしたい座標の格子 `grid` を渡すと、各格子点の値を補間して返す関数です。`grid` の座標規約は「画像の左上が (-1, -1)、右下が (+1, +1)」に正規化されています（0〜1 の正規化座標を使う場合は `2*x - 1` で変換する必要があります）。`mode='bilinear'`（双線形補間）、`padding_mode='zeros'`（範囲外を 0 埋め）、`align_corners`（座標とピクセル中心の対応規約。True ならコーナーピクセルの中心を -1/+1 に、False ならコーナーピクセルの外縁を -1/+1 に合わせる）といったオプションを取ります。`align_corners` の取り違えは数ピクセル単位のずれを生むため、投影座標とサンプリングで一貫させる必要があります。

**カスタム CUDA カーネル**: GPU 上で動く処理を、CUDA という言語で低レベルに手書きしたものです。高速ですが、コンパイルが必要で、CUDA バージョン・OS・GPU 世代が変わるとビルドに失敗しやすく、CPU では動かない（あるいは別実装が要る）という移植性の問題を抱えます。

## 仕組みを詳しく

変形アテンションを格子サンプリングで近似する典型的な実装は、おおむね次の手順を踏みます。多視点カメラから鳥瞰図（BEV, Bird's-Eye-View）特徴を作る BEV 融合を例に、テンソル形状つきで追います。

入力は、あるカメラ1台ぶんの画像特徴マップ `feat_v`（形状 `[B, C, H, W]`、B=バッチ、C=チャンネル数 例 256、H, W=特徴マップの縦横）です。BEV グリッドの各マス（クエリ）が、自分が映っているはずの画像位置を起点に数点をサンプルします。

**ステップ1: サンプル位置を作る（参照点＋学習オフセット）**

各クエリは、幾何計算で求めた「投影位置（参照点 reference point）」を持ちます。これに学習可能なオフセット（ずらし量）を加えて、実際に見る位置を決めます。

```
offsets      = sampling_offsets(queries)         # クエリから予測 [B, N, num_points, 2]
sample_locs  = ref_2d + offset_scale * offsets   # [B, N, num_points, 2]
```

`N` はクエリ数（BEV マス数、例 450×300=135,000）、`num_points` は1クエリあたりのサンプル点数（例 4）、最後の 2 は (x, y) 座標です。`offset_scale`（例 0.1）は、オフセットが暴れて遠くを見てしまわないよう小さく抑える係数です。この「offsets をクエリから予測する」部分こそが「変形（deformable）」の本体で、ネットワークが入力に応じて見る位置を動かします。

**ステップ2: 0〜1 座標を -1〜1 に変換する**

```
sample_grid = sample_locs * 2 - 1   # [B, N, num_points, 2]
```

正規化座標（左上 0, 右下 1）を `grid_sample` の規約（左上 -1, 右下 +1）に合わせます。

**ステップ3: `grid_sample` で特徴を取り出す**

```
sampled = F.grid_sample(
    feat_v, sample_grid,
    mode='bilinear', padding_mode='zeros', align_corners=False
)   # [B, C, N, num_points]
```

ここで N×num_points 個ぶんの特徴が双線形補間で一気に取れます。`padding_mode='zeros'` により、画像の外を指したサンプル点は 0 が返ります（画像外に特徴は無いので妥当）。

**ステップ4: アテンション重みで合成する**

サンプルした num_points 点を、クエリが予測したアテンション重み `w` で加重平均します。

```
sampled  = sampled.permute(0, 2, 3, 1)              # [B, N, num_points, C]
weighted = (sampled * w.unsqueeze(-1)).sum(dim=2)   # [B, N, C]
```

物理的に見えない点（深度が負、画像枠外など）は重みを 0 にし、残った点で重みを再正規化します（可視性マスキング、[BEVの可視性マスキング](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0934_bev-visibility-masking) 参照）。

この「クエリが見る位置と重みを予測し、数点をサンプルして重み付き和を取る」流れが、変形アテンションの本質を格子サンプリングで再現した部分です。ここで重要なのは、変形アテンションが Q·K^T の内積を一切計算しない点です。アテンション重みはクエリを線形層に通して softmax を取るだけで生成されるため、計算量がサンプル点数 num_points にしか依存せず、候補数 N に依存しなくなります。これが効率の源泉です。

**原典のフル実装との違い**

研究コミュニティのフル実装（原典の CUDA カーネル）と、格子サンプリング近似の差は主に次の点です。

- マルチヘッド: フル実装は複数のアテンションヘッド（それぞれが別の特徴部分空間を独立に見る）を持ちます。たとえば C=256 を 8 ヘッド × 32 次元に分割し、ヘッドごとに別のオフセット・別の重みを予測します。近似実装は単一ヘッドにすることが多く、表現力が落ちます。
- マルチスケール: フル実装は複数解像度の特徴マップ（feature pyramid、例 1/8・1/16・1/32 の3〜4スケール）を横断してサンプルします。低解像度は大物体、高解像度は小物体を捉えやすく、スケール横断で頑健になります。近似実装は単一スケールに簡略化されがちです。
- 処理単位: フル実装は全サンプル点を1つの融合カーネルで処理しますが、近似実装はカメラやスケールごとに Python ループを回すことが多く、ループのオーバーヘッドが生じます。

つまり「機能としては変形アテンション相当」だが「速度・表現力は簡略版」というのが正確な位置づけです。

## 手法の系譜と主要論文

**起点 — Spatial Transformer Networks（Jaderberg et al., NeurIPS 2015, arXiv:1506.02025）**。微分可能な格子サンプリングを初めて一般的な部品として提案しました。動機は「ネットワークに、入力画像の注目領域を自分で切り出し・正規化させたい」というもので、サンプリング格子（変換パラメータ θ から生成）を予測し、その位置の値を双線形補間で取り出します。新規性は、サンプル位置について微分可能にしたことで、誤差逆伝播で「どこを見るか」を学習できる点です。`F.grid_sample` はこの STN のサンプラをフレームワーク標準関数として実装したものです。トレードオフは、双線形補間が局所的（近傍4点しか見ない）なため、大きくずれた初期位置からは勾配が伝わりにくく、最適化が局所解に陥りやすいことです。

**変形畳み込みへの展開 — Deformable Convolution（Dai et al., ICCV 2017, arXiv:1703.06211）**。畳み込みのサンプル格子に学習オフセットを足し、受容野を入力に応じて変形させました。アテンションではなく畳み込みですが、「サンプル位置を学習で動かす」という変形アテンションの思想の直接の先祖にあたります。

**変形アテンション — Deformable DETR（Zhu et al., ICLR 2021, arXiv:2010.04159）**。物体検出 Transformer である DETR（Carion et al., ECCV 2020）は、全画素アテンションのせいで収束に約 500 エポックを要し、小物体に弱いという欠点がありました。Deformable DETR は「各クエリが少数のサンプル点のオフセットとアテンション重みを予測し、その点だけにアテンションをかける」変形アテンションを導入しました。新規性は、Q·K^T を捨てて重みを直接予測することで、計算量を候補数 N に依存しない形（既定 4 点 × 4 スケール × 8 ヘッド）に抑えた点です。この効率的なサンプリングのために原典は専用 CUDA カーネル（`MSDeformAttn`）を実装しています。本稿の格子サンプリング近似が回避したいのが、まさにこの CUDA 依存です。

**BEV への展開 — BEVFormer（Li et al., ECCV 2022, arXiv:2203.17270）**。多視点カメラから BEV 特徴を作るために、変形アテンションを2方向に使います。(1) 空間クロスアテンション（spatial cross-attention）: 各 BEV クエリが、自分の 3D 参照点をカメラ画像に投影し、その投影位置周辺の数点に変形アテンションをかけて画像特徴を集めます。(2) 時間自己アテンション（temporal self-attention）: 過去フレームの BEV からも変形アテンションで情報を引きます。動機は「全画素アテンションでは多視点・高解像度画像で計算が爆発する」ことで、変形アテンションにより実用的な計算量に収めました。実装は Deformable DETR の CUDA カーネルを流用しています。

これらの系譜から見ると、格子サンプリング近似は「STN のサンプラ（フレームワーク標準）を使って、Deformable DETR / BEVFormer の変形アテンションを CUDA 無しで再構成する」位置づけになります。教育・プロトタイプ・移植性重視の文脈で広く使われる実用的な選択肢です。

## 論文の実験結果（定量データ）

**Deformable DETR の効果**（COCO 物体検出、評価指標は AP = Average Precision。検出ボックスがどれだけ正確かを 0〜100 で表し、高いほど良い）。報告によると、DETR が約 500 エポックで AP 42.0 程度に到達するのに対し、Deformable DETR は約 50 エポック（10 分の1）で AP 43.8 程度に達し、同等以上の精度をはるかに少ない学習で実現しました。とくに小物体の AP（APs、小さい物体だけを対象にした AP）で、DETR の 20.5 程度に対し 26 前後へと大きく改善しています。「収束が速い」とは、同じ精度に達するのに必要な学習計算が大幅に少ない、という実務上重要な意味を持ちます。

**BEVFormer の効果**（nuScenes 3D 物体検出ベンチマーク。指標は NDS = nuScenes Detection Score、検出精度と位置・速度・属性誤差を総合した 0〜1 のスコア、高いほど良い。および mAP）。BEVFormer はカメラのみで NDS 約 0.517、mAP 約 0.416 を報告し、当時のカメラベース手法（DETR3D の NDS 約 0.425、mAP 約 0.346 など）を明確に上回りました。変形アテンションによる効率的な多視点融合が、この精度を実用的な計算量で達成する鍵でした。

**アブレーションの示唆**: Deformable DETR の論文では、サンプル点数やマルチスケールの有無を変えた分析があり、点数を 4 から減らすと精度が落ち、マルチスケール（複数解像度のサンプリング）を外すと AP が約 1〜2 ポイント低下し、とくに小物体の検出が悪化することが示されています。これは格子サンプリング近似（単一ヘッド・単一スケール・少点）が、フル実装に比べて表現力・精度で劣る理由を直接裏づけます。つまり近似版で起こりうる精度低下は、これらの要素を簡略化したことの帰結だと理解できます。逆に言えば、近似版でもマルチヘッド化・マルチスケール化を丁寧に実装すれば、フル実装に近づけられることも示唆します。

## メリット・トレードオフ・限界

**メリット**

- カスタム CUDA 不要で、CPU/GPU・各種 CUDA バージョン・各 GPU 世代でそのまま動く（高い移植性）。CI の CPU テストと本番 GPU で同一コードを使える。
- フレームワーク標準関数のみなので依存関係が増えず、保守が容易。ビルド失敗のリスクが無い。
- 双線形補間により連続座標を扱え、オフセットを誤差逆伝播で学習できる（変形アテンションの本質を保持）。
- 実装が読みやすく、教育・実験・新規アーキテクチャの足場として優れる。

**トレードオフ・限界**

- 単一ヘッド・単一スケールにすると、フルの変形アテンションより表現力が劣る（複数の部分空間・複数解像度を同時に見られない）。
- カメラ・スケールごとの Python ループはカーネルフュージョンされた CUDA より遅い。サンプル数が多いと顕著で、サンプル点数が同じでもループ回数ぶんカーネル起動オーバーヘッドが積み上がる。
- 双線形補間は局所的なので、初期オフセットが大きくずれていると勾配が伝わりにくく、サンプル位置の学習が進みにくい（STN 由来の限界）。
- 境界付近では `padding_mode='zeros'` により値が急に 0 になり、勾配が不連続になりやすい。
- フルの BEVFormer / Deformable DETR を完全再現したものではないため、論文の精度数値をそのまま期待することはできない。

**未解決の課題（研究視点）**: 変形アテンションのサンプル点数・初期化・正規化は最適化の安定性に直結し、設計が経験則に依存します（オフセットを予測する線形層は重みを 0、バイアスを格子状に初期化するのが慣例で、これを誤ると収束しないことがあります）。格子サンプリング近似で精度を保ちつつ高速化する（ループのベクトル化、JIT コンパイラによる融合、マルチヘッド化の効率的な実装）方向は、移植性と性能を両立させる実用研究のテーマです。また補間カーネルを双線形以外（バイキュービックや学習可能カーネル）に変えたときの精度・勾配の挙動も検討余地があります。

## 発展トピック・研究の最前線

近年は、変形アテンションを 3D 空間で直接定義する流れ（BEVFormer v2、PETR/PETRv2 の 3D 位置エンコーディング、StreamPETR の時系列伝播）や、サンプリングを疎な点群クエリと組み合わせる手法（Sparse4D、Sparse4Dv2/v3 など）が登場しています。いずれも「全画素を見ない疎なアテンション」という変形アテンションの思想を引き継ぎつつ、CUDA 依存をどう扱うか（独自カーネルか、格子サンプリングベースか）が実装上の分岐点になります。また JIT カーネル生成（PyTorch 2.x の `torch.compile` や Triton による生成）は、「標準関数の組み合わせを書きつつ、実行時に CUDA 並みの速度へ融合する」という第3の道を開きつつあり、移植性と速度のトレードオフを緩和する方向として注目されています。さらに、変形アテンションを言語・マルチモーダルのトークン疎化（重要トークンだけにアテンションを集中させる）へ転用する研究も現れており、「学習可能なサンプリング」という発想自体が視覚を超えて一般化しつつあります。
