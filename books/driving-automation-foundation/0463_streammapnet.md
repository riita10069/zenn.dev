# StreamMapNet

## ひとことで言うと
オンライン HD マップを「1 フレームずつバラバラに」作るのではなく、過去フレームの情報(地面の特徴マップと、地図要素を表すクエリ)を次のフレームへ持ち越しながら作る手法です。動画のように時間をまたいで記憶を引き継ぐことで、一瞬隠れて見えなくなった車線がチラついたり消えたりするのを防ぎ、地図を時間方向に安定させます。時系列を扱うベクトル地図手法の代表で、長距離・遮蔽に強いことを示しました。さらに、時系列手法を不公平に高く見せていた評価データの分割問題を指摘し、地理的に重ならない公正な分割を提案した点でも、分野の評価基準に影響を与えました。

## 直感的な理解
あなたが車を運転していて、前を走る大型トラックに視界の一部が隠れたとします。その瞬間、隠れた先の車線が「目の前から消えた」とは思わないはずです。「さっきまで見えていたから、トラックの向こうにも車線は続いているはず」と、記憶で補って運転を続けます。これが人間の自然な感覚です。

ところが、1 フレームの画像だけから地図を作る手法は、この「記憶による補完」ができません。トラックに隠れた瞬間、その車線は入力画像から消えるので、出力地図からも消えたり大きく崩れたりします。さらに、隠れていなくても、同じ車線がフレームごとに少しずつ違う形・位置で出力され、地図が小刻みに揺れます(ちらつき、temporal inconsistency)。揺れる地図の上で経路を計画すると、計画もブレてしまい、ハンドルがガタつくような挙動につながりかねません。

StreamMapNet は「人間のように過去の観測を記憶として持ち越し、現在の観測と組み合わせて地図を作る」ことで、この問題を解きます。記憶があるので、一瞬隠れても車線は消えず、フレーム間で形がそろい、地図が安定します。

## 基礎: 前提となる概念

まず関連する用語を噛み砕きます。

「オンライン HD マップ構築」とは、走行中の車載センサー映像から、その場で高精細地図(車線・境界・横断歩道などの位置を細かく記録した地図)を作る技術です。事前地図の作成・更新コストを避けるために生まれました。

「単フレーム(single-frame)推論」とは、「その瞬間のカメラ画像 1 枚(正確には同時刻の複数カメラ)」だけから地図を作る方式です。[VectorMapNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0466_vectormapnet)、MapTR、[PivotNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0459_pivotnet) は基本的にこの方式です。実装は単純ですが、時間情報を捨てているため、ちらつきと遮蔽に弱いという弱点があります。

「BEV(Bird's-Eye-View、鳥瞰図)特徴」とは、複数のカメラ画像を「空から見下ろした地面の格子」の上に貼り直して作る特徴マップです。形状は `[C, X, Y]`(縦 X × 横 Y の格子の各マスに C 次元の特徴ベクトル)で、地図要素を地面座標で直接扱えます。

「クエリ(query)」とは、地図要素を表す学習可能なベクトルです。DETR(物体検出の Transformer モデル、Carion et al., ECCV 2020)由来の仕組みで、各クエリが 1 本の地図要素(1 本の車線など)に対応し、デコーダがクエリから線を予測します。クエリは「この場所にこういう車線がある」という抽象的な情報を、座標ではなく高次元ベクトルとして保持しています。

「自車運動(ego motion)」とは、フレーム間で自分の車がどれだけ動いたか(前進量・回転量)です。車載の慣性計測装置(IMU)や走行距離計(オドメトリ)から得られ、4×4 の剛体変換行列として表せます。過去フレームの地図と現在フレームの地図は座標系がずれている(車が動いたぶんだけ世界が後ろに流れる)ので、過去情報を使うにはこの動き量で座標を補正する必要があります。

「時間融合(temporal fusion)」とは、複数時刻の観測を統合して認識を安定させる考え方です。物体検出では BEVFormer(Zhiqi Li et al., ECCV 2022, arXiv:2203.17270)が複数時刻の BEV を統合して検出と速度推定を安定させました。BEVFormer は過去の BEV を全部保持して空間的注意で参照しますが、StreamMapNet はこの発想を「1 つ前の状態だけを再帰的に持ち越す」ストリーミング方式で地図構築に持ち込んだものです。

「再帰(recurrent)」とは、前の時刻の出力(状態)を次の時刻の入力に回す処理形態です。RNN(Recurrent Neural Network)や GRU(Gated Recurrent Unit、ゲート付き再帰ユニット)がその代表で、ゲートが「過去の記憶をどれだけ残すか」を学習で制御します。

## 仕組みを詳しく

StreamMapNet は 2 種類の情報を、フレームをまたいで「ストリーム(流れ)」として伝播させます。すべての過去フレームを保持するのではなく、1 つ前の状態だけを再帰的に更新していくため、フレーム数が増えても計算量・メモリが一定に保たれるのが特徴です(BEVFormer の固定窓方式との違い)。

### 1. BEV 特徴の伝播(streaming BEV / 空間記憶)

前フレームで作った BEV 特徴マップ `[C, X, Y]` を、その間に自車が動いた量(ego motion)に合わせてワープ(座標を平行移動・回転で合わせる変換。具体的にはサンプリング格子を変換行列で動かして双線形補間する)します。自車が前に 2m 進んだら、前フレームの BEV を 2m ぶんずらして現在の座標系に合わせる、という処理です。

位置合わせ後、現フレームの新しい BEV 特徴と融合します。融合には GRU のようなゲート機構を使い、「過去の記憶をどれだけ残し、現在の観測をどれだけ取り込むか」を学習で決めます。これにより、過去に見た地面の情報が現在の BEV に蓄積され、いま見えない領域(トラックの陰など)も過去の記憶で埋まります。ワープで現在視野の外に出た領域の記憶は自然に捨てられ、新しく視野に入った領域は現フレームの観測で埋まります。

### 2. クエリの伝播(streaming query / インスタンス記憶)

地図要素を表すクエリも次フレームへ持ち越します。前フレームで検出した各地図要素のうち、信頼度の高いものをトップ K 個選び、それらのクエリ(と対応する基準点)を現フレームのデコーダの初期クエリとして再利用します。基準点も ego motion で現在座標に合わせます。これには 2 つの効果があります。

第一に、「前フレームの車線 A」と「現フレームの車線 A」を同じインスタンス(個体)として追跡でき、時間をまたいで一貫した ID で地図要素を出せます。第二に、一瞬遮蔽されても、持ち越したクエリが記憶として残るので、車線が出力から消えにくくなります。クエリは「この場所にこういう車線がある」という抽象的な記憶を保持しているため、入力画像にその車線が一瞬映らなくても予測を維持できます。新規に出現した要素は、別途用意した新規クエリ群が拾います。

### 処理の流れ(ストリーミング)

```
フレーム t-1 ──[BEV 特徴]──┐    ┌─ ego motion でワープ ─┐
                           ├──→ │   現フレーム BEV と融合 │ ──→ 地図 t
フレーム t-1 ──[クエリ]─────┘    └─ 初期クエリとして再利用┘
                                        ↑ フレーム t のカメラ入力
                                    (出力した BEV・クエリは次フレーム t+1 へ)
```

各フレームの予測は「現在の観測」+「過去の記憶」の両方に基づくものになり、出力地図のちらつきが減り、遮蔽に強くなります。学習時は、勾配が無限に過去へ流れないよう、数フレームのウィンドウで状態を切る(truncated backpropagation through time)などの工夫がとられます。

### 評価分割の問題提起

StreamMapNet はもう 1 つ、評価データの分割(train/val split)の問題も指摘しました。従来の nuScenes の標準分割は、学習データと評価データで地理的に重なる区間がありました。同じ交差点を学習でも評価でも通っているような状況です。これだと時系列手法が「過去に学習で見た同じ場所を覚えているだけ」で性能が水増しされる恐れがあります。論文は地理的に重ならない新しい分割(geographically disjoint split)を nuScenes と Argoverse 2 の両方に提案し、時間融合の効果をより公正に測れるようにしました。この新分割は後続研究の標準的な検証手段として広く採用され、地図構築研究の評価健全化に寄与しました。

## 手法の系譜と主要論文

オンラインベクトル地図構築は、まず単フレーム手法が発展しました。VectorMapNet(Yicheng Liu, Tianyuan Yuan, Yue Wang, Yilun Wang, Hang Zhao, ICML 2023, arXiv:2206.08920)がエンドツーエンドのベクトル地図出力を確立し、MapTR / MapTRv2(Bencheng Liao et al., ICLR 2023 / IJCV 2024, arXiv:2208.14437)が高速・高精度な固定点集合表現でデファクト基盤になりました。[PivotNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0459_pivotnet)(Ding et al., ICCV 2023)は点表現を可変長に改良しました。これらはいずれも「その瞬間」だけを見る単フレーム手法です。

時間融合の発想は、物体検出側で先行していました。BEVFormer(Zhiqi Li et al., ECCV 2022)が「複数時刻の BEV を統合して認識を安定させる」ことを示し、続く BEVFormer v2 や、よりストリーミング的に状態を持ち越す検出器(StreamPETR など)が「再帰的な時間伝播」の有効性を示していました。StreamMapNet(Tianyuan Yuan, Yicheng Liu, Yue Wang, Yilun Wang, Hang Zhao, WACV 2024, arXiv:2308.12570)は、この時間融合をベクトル地図構築に持ち込んだ代表作です。VectorMapNet と同じ著者グループ(Hang Zhao ら)による時系列拡張という系譜にあります。BEV 特徴とクエリの両方を再帰的に伝播させる二重の記憶機構が新規性で、ほぼ同時期に提案された他の時系列地図手法(例: 履歴のラスタ地図を補助に使う手法など)の中でも、「空間記憶」と「インスタンス記憶」を明確に分けて両方伝播させた点が特徴です。

StreamMapNet の後、時間融合は地図構築の標準的な構成要素になりました。後続には、より長期の記憶を扱う手法や、過去フレームの不確実性を考慮する手法、SD 地図 prior と時間融合を組み合わせる手法([P-MapNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0458_p-mapnet) の系統)などが続いています。「単フレームから時系列へ」という流れの転換点を作ったのが StreamMapNet です。

## 論文の実験結果(定量データ)

評価は nuScenes と Argoverse 2(別の大規模自動運転データセット、6 カメラ + LiDAR、北米 6 都市で収集)で行われました。評価指標は mAP(mean Average Precision、平均適合率)で、予測ポリラインと正解ポリラインの Chamfer 距離(点集合どうしの平均最近傍距離)が一定しきい値以内なら正解とみなし、車線分割線・道路境界・横断歩道について平均します。値が高いほど良く、0〜100% です。

StreamMapNet は、単フレームのベースライン(MapTR など)に対して mAP を有意に改善したと報告しています。特に効果が大きいのは長距離・遮蔽の多い条件で、評価範囲を広げる(たとえば縦方向の予測レンジを通常の ±30m から ±50m や ±60m へ拡張する)ほど時間融合の優位が拡大します。これは「遠距離は単フレームでは見えにくく、過去の記憶が効く」という直感と一致します。

最も重要なのは公正な分割での結果です。論文では、従来の重複ありの標準分割では多くの手法の数値が高めに出ており、地理非重複分割に切り替えると全体に数値が大きく下がる(手法によっては mAP がおよそ半減する)ことが示されました。これは「過去に学習で見た場所を覚えていただけ」の成分がいかに大きかったかを意味します。その厳しい条件下でも StreamMapNet は単フレーム手法に対して時間融合の優位を保ち、新分割上での新たなベースラインとなりました。

アブレーション(構成要素を抜く実験)では、(1) BEV 特徴の伝播だけ、(2) クエリの伝播だけ、(3) 両方、を比較しています。報告によると両方を併用したときに最も精度と時間的一貫性が高く、片方だけでも単フレームより良いものの効果は限定的でした。これは「空間の記憶(BEV)」と「インスタンスの記憶(クエリ)」が補完的に働くことを示しています。BEV 伝播は地面の特徴を蓄積して遮蔽領域を埋め、クエリ伝播は地図要素の同一性を保ってちらつきを抑える、という役割分担です。

時間的一貫性そのものを測る指標(連続フレームで同じ地図要素がどれだけ安定して出力されるか。同一インスタンスの形状の時間方向の分散などで測る)でも、StreamMapNet は単フレーム手法より大幅に揺れが小さいことが示されました。これは mAP だけでは見えない「動画として見たときの安定感」を定量化したもので、下流の経路計画にとって実用上重要です。

## メリット・トレードオフ・限界

メリット
- 過去フレームの記憶を引き継ぐことで、出力地図のちらつき(時間的不一致)を大幅に低減できる。
- 遮蔽や遠距離など「いまは見えない/見えにくい」領域を、過去の観測で補完できる。評価レンジを広げるほど優位が拡大する。
- クエリ伝播により地図要素を時間をまたいで同一インスタンスとして追跡でき、ID の一貫性が保てる。
- 1 つ前の状態だけを再帰的に持ち越すため、フレーム数が増えても計算・メモリが一定。実時間運用に向く。
- 地理非重複の公正な評価分割を提示し、時系列手法の評価を健全化した。研究コミュニティの評価基準に影響を与えた。

トレードオフ・限界
- 状態(BEV 特徴・クエリ)の伝播・管理で、実装と計算が複雑化する。
- 誤った記憶が次フレームに引き継がれ、誤差が時間方向に蓄積・伝播するリスクがある(エラーの自己強化)。一度間違って覚えた車線が、しばらく出力に残り続けることがある。
- ストリーミング前提のため、学習・推論でフレーム順序を保つ必要があり、データを完全にはランダムシャッフルできない。学習バッチ構成が制約され、truncated BPTT などの工夫を要する。
- ego motion(自車運動)推定がずれると BEV ワープが狂い、過去情報の位置合わせが破綻する。低速・停止時や、IMU/オドメトリのノイズが大きい場面で問題になりうる。

研究上の未解決課題として、記憶を「いつ捨てるか」が難しい問題です。地図が本当に変わった(工事で車線が引き直された)場合、古い記憶は害になります。新旧をどう区別して記憶を更新するか、変化検出とのバランスは依然として課題です。また、再帰的に 1 状態だけ持つ方式は超長期(数十秒以上前)の情報を保持しにくく、長期の地図蓄積には別の記憶機構が要ります。

## 発展トピック・研究の最前線

StreamMapNet 以降、時間融合は複数の方向に発展しています。1 つは記憶の長期化で、数フレームの伝播を超えてより長い履歴を保持し、過去に通過した道を「地図の蓄積」として活用する方向です。もう 1 つは不確実性を考慮した記憶で、過去の予測の信頼度を重みづけして伝播し、誤差伝播を抑える研究です。

さらに、時間融合と SD 地図 prior の組み合わせが進んでいます。過去フレームの記憶(時間方向)と、カーナビ地図のような事前知識(prior 方向)の両方を統合することで、遠距離・遮蔽・ちらつきに同時に対処します。[P-MapNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0458_p-mapnet) のような prior 活用と StreamMapNet のような時間融合は、相補的な技術として一緒に使われる傾向にあります。点表現の効率化([PivotNet](https://zenn.dev/riita10069/books/driving-automation-foundation/viewer/0459_pivotnet))と時間融合を同時に取り入れる手法も登場しています。

時間情報をどう扱うかは、地図構築に限らず自動運転モデル全般の重要テーマです。「過去の状態を伝播して一貫性を保つ」という StreamMapNet の発想は、軌跡予測・行動計画・将来状態の推定など、連続フレームを扱うあらゆるタスクの設計指針として参照されています。
